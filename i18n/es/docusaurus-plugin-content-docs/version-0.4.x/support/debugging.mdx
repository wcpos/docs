:::caution
Este artículo está destinado a desarrolladores web, la depuración no es algo que el usuario promedio deba usar.
:::

WCPOS es principalmente una aplicación web de JavaScript, y como tal, la mayor parte de la depuración se realiza a través de la [consola del navegador](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors).
Si la depuración está habilitada, WCPOS registrará varios mensajes en la consola que pueden ayudar a diagnosticar problemas.

![Una Consola JavaScript abierta en Chrome para Mac](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Una Consola JavaScript abierta en Chrome para Mac")

### Activar la depuración usando wp-config.php

Al establecer la bandera [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) en su archivo wp-config.php, puede habilitar la depuración de WCPOS.
Esto cargará los archivos de script sin minimizar y también habilitará los mensajes de depuración en la consola del navegador.

### Activar la depuración usando la consola del navegador

Si no tiene acceso al archivo wp-config.php, puede habilitar los mensajes de depuración agregando una bandera `debug` al Almacenamiento Local.
Esto se puede hacer ingresando `localStorage.setItem('debug', '*')` en la consola.
Ahora recargue la página y la depuración estará activada. El registro de la consola puede consumir recursos del navegador, por lo que probablemente querrá desactivar la depuración una vez que haya terminado.
Para desactivar la depuración, simplemente ingrese `localStorage.removeItem('debug')` en la consola y recargue la página.
