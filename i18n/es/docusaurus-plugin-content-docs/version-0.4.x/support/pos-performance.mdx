---
title: Rendimiento del POS
---

WCPOS es una aplicación web de una sola página en javascript que se comunica con su servidor a través de la [API REST de WooCommerce](http://woothemes.github.io/woocommerce-rest-api-docs/). Los problemas de rendimiento se pueden dividir en dos categorías: del lado del servidor y del lado del cliente.

### Rendimiento del Lado del Servidor

El rendimiento del lado del servidor se refiere a acciones como descargar una página de productos (10 productos) o procesar un pedido. Aspectos que pueden afectar la velocidad de estas acciones incluyen:

- La velocidad de su conexión a internet
- La velocidad de su servidor, es decir: potencia de procesamiento, RAM, carga del servidor, etc.
- Cuántos plugins tiene activos
- Procesamiento de la pasarela de pago

El [sitio de demostración](http://demo.wcpos.com/pos) representa un escenario de 'mejor caso'. El sitio de demostración utiliza [un alojamiento web muy bueno](http://wcpos.com/wpe) y tiene un número muy pequeño de plugins activados.

| Proceso                  | Velocidades promedio para [demo.wcpos.com/pos](http://demo.wcpos.com/pos)<sup>\*</sup> |
| ------------------------ | ------------------------------------------------------------------------------------- |
| Recuperando 10 productos | 1 - 2 segundos                                                                        |
| Procesando una venta en efectivo | 1 - 2 segundos                                                                  |
| Procesando una venta con Stripe   | 2 - 3 segundos                                                                |

<small>
  <sup>*</sup> tiempos promedio para una conexión de banda ancha. Los tiempos pueden variar
  dependiendo de su velocidad de internet.
</small>

Si su POS tarda significativamente más que estos tiempos, puede realizar las siguientes pruebas:

1.  Cambie su tema al tema predeterminado Twenty Fifteen de WordPress
2.  Desactive todos los plugins excepto WooCommerce y WCPOS
3.  [Limpie los datos de almacenamiento local](http://faq.wcpos.com/en/clear-local-data.html)
4.  Ahora, use el POS para ver si hay un aumento de rendimiento
5.  **Si hay un aumento de rendimiento:** reactive su tema y plugins uno por uno para ver qué plugin está impactando su rendimiento
6.  **Si no hay un aumento de rendimiento:** puede considerar cambiar a un mejor alojamiento web o invertir en una conexión a internet más rápida

### Rendimiento del Lado del Cliente

WCPOS utiliza JavaScript, HTML y CSS para mostrar los datos recuperados de la API REST de WooCommerce, es decir: [productos](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) y [pedidos](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order).
Para mejorar el rendimiento del lado del cliente, el POS almacena los datos en el navegador utilizando [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API).
Por ejemplo, cuando un producto se recupera por primera vez, se enviará una solicitud al servidor; una vez que se descarguen los datos del producto, se almacenarán localmente para que las búsquedas posteriores sean instantáneas.

![Un ejemplo de productos almacenados localmente.](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "Un ejemplo de productos almacenados localmente")

Los datos de IndexedDB persisten incluso cuando cierra el navegador o reinicia su computadora.
Si nota que sus datos de productos no están sincronizados por alguna razón, puede [limpiar el almacenamiento local](http://faq.wcpos.com/en/clear-local-data.html) y recuperar un nuevo conjunto de datos del servidor.
