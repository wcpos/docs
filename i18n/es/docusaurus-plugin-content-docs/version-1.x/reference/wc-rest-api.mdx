La **API REST de WooCommerce** es como un conjunto de "canales" estandarizados que permite a los propietarios de tiendas conectar su tienda de WooCommerce con otras aplicaciones y servicios.

Aquí hay una explicación sencilla de cómo funciona:

1. Actúa como un puente entre su tienda de WooCommerce y aplicaciones externas, permitiéndoles comunicarse y compartir datos.

2. La API funciona enviando solicitudes a **endpoints** específicos - piense en estos como "canales" para diferentes tipos de datos de la tienda. Por ejemplo, hay endpoints para productos, pedidos y clientes.

3. Las aplicaciones externas pueden usar estos endpoints para:

- Obtener información (como obtener una lista de productos)
- Agregar nuevos datos (como crear un nuevo pedido)
- Actualizar información existente (como cambiar el precio de un producto)
- Eliminar datos (como eliminar un producto desactualizado)

En lugar de depender de hooks y filtros de PHP (como muchos complementos tradicionales de WordPress), funciona enviando y recibiendo datos en un formato estructurado y predecible, conocido como **JSON**.

## ¿Por qué Algunos Complementos No Funcionan en el POS?

WCPOS descarga los datos de productos, pedidos y clientes de su tienda de WooCommerce a través de la API REST.
Luego mantiene una copia local de esos datos para que pueda continuar funcionando incluso si su conexión a internet es intermitente.
Este enfoque hace que el POS sea más rápido y confiable en un entorno de pago ocupado.

Sin embargo, esto significa que nuestro POS no "ve" los cambios que ocurren dinámicamente en su sitio en tiempo real. Por ejemplo:

      expuestos a través de la API REST.

  </tr>
