# WCPOS WP Multilang {#wcpos-wp-multilang}

Integra [WP Multilang](https://wordpress.org/plugins/wp-multilang/) con WCPOS para que el POS solo muestre productos en un único idioma. A diferencia de Polylang o WPML, WP Multilang almacena todas las traducciones dentro de la misma entrada — este plugin se asegura de que WCPOS lea la traducción correcta y no duplique entradas en los resultados de búsqueda.

## Qué hace {#what-it-does}

- Filtra las respuestas REST de productos y variaciones de WCPOS a un único idioma de WP Multilang.
- Intercepta las rutas de **fast-sync** de WCPOS para que solo el contenido del idioma activo llegue al cliente.
- Las instalaciones gratuitas utilizan el idioma predeterminado de WP Multilang.
- Las instalaciones Pro pueden fijar un idioma por tienda desde el editor de tiendas.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Install WP Multilang">

Instala [WP Multilang](https://wordpress.org/plugins/wp-multilang/) y configura los idiomas de tu sitio, con al menos un idioma establecido como predeterminado.

  </StepCard>
  <StepCard number="2" title="Install WCPOS WP Multilang">

Instálalo desde `WP Admin > POS > Settings > Extensions`, o descarga la última versión desde la [página de versiones de GitHub](https://github.com/wcpos/wcpos-wp-multilang/releases) y súbelo mediante `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="(Pro) Pin a language per store" isLast>

En WCPOS Pro, edita una tienda en `POS > Stores` y selecciona su idioma en la sección **Idioma** de la barra lateral. Déjalo en *Predeterminado* para usar el idioma predeterminado de WP Multilang.

  </StepCard>
</Steps>
