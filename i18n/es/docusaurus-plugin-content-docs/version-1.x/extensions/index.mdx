WCPOS admite extensiones que añaden nuevas funcionalidades a tu punto de venta. El directorio de extensiones te permite explorar las extensiones disponibles, instalarlas directamente desde los ajustes del POS y gestionar las actualizaciones.

:::info Función Pro
Instalar y gestionar extensiones requiere [WCPOS Pro](/getting-started/pro-license). La versión gratuita muestra el catálogo de extensiones, pero desactiva los controles de instalación y activación.
:::

## Explorar extensiones {#browsing-extensions}

Abre el directorio de extensiones desde `POS Settings > Extensions` (también etiquetado como **Plugins** en algunas versiones).

El directorio muestra una cuadrícula de tarjetas con las extensiones disponibles. Cada tarjeta incluye:

- **Icono** (o una pieza de rompecabezas como alternativa si la extensión no proporciona uno)
- **Nombre y versión**
- **Descripción**
- **Insignia de categoría**
- **Estado** — activa, inactiva, actualización disponible o no instalada

### Filtrado y búsqueda {#filtering-and-search}

Utiliza los **botones de categoría** en la parte superior para filtrar las extensiones por categoría. También puedes usar el **campo de búsqueda** para encontrar extensiones por nombre, descripción o etiquetas.

## Instalación de una extensión {#installing-an-extension}

1. Abre `POS Settings > Extensions`.
2. Busca la extensión que quieras y haz clic en **Instalar**.
3. La extensión se descarga e instala mediante el instalador de plugins de WordPress.
4. Una vez instalada, haz clic en **Activar** para habilitarla.

En segundo plano, WCPOS utiliza el `Plugin_Upgrader` nativo de WordPress para gestionar la instalación, por lo que las extensiones siguen el mismo proceso que cualquier plugin de WordPress.

## Activación y desactivación {#activating-and-deactivating}

Cada extensión instalada tiene los botones **Activar** y **Desactivar** en su tarjeta.

- **Activar** habilita la extensión para que pueda ejecutarse en el POS.
- **Desactivar** la deshabilita sin desinstalarla. Los archivos de la extensión permanecen en el servidor y se pueden volver a activar en cualquier momento.

## Actualización de extensiones {#updating-extensions}

Cuando hay una versión más reciente disponible de una extensión instalada, la tarjeta muestra una insignia de **Actualización disponible** y un botón de **Actualizar**.

### Actualizaciones automáticas {#auto-updates}

Las extensiones instaladas desde el directorio tienen las **actualizaciones automáticas habilitadas de forma predeterminada**. Puedes activar o desactivar las actualizaciones automáticas para cada extensión desde su tarjeta en el directorio. Cuando las actualizaciones automáticas están activadas, WordPress aplicará las nuevas versiones automáticamente, igual que lo hace con los plugins que tienen las actualizaciones automáticas habilitadas.

## Solución de problemas {#troubleshooting}

### Mensaje «Requiere Pro» en los botones {#requires-pro-message-on-buttons}

Los botones de instalación, activación y actualización están deshabilitados en la versión gratuita de WCPOS. Actualiza a [WCPOS Pro](/getting-started/pro-license) para gestionar las extensiones.

### La extensión no se instala {#extension-fails-to-install}

- Comprueba que tu servidor de WordPress tenga permisos de escritura en el directorio `wp-content/plugins`.
- Verifica que tu servidor pueda realizar solicitudes HTTPS salientes (algunos hosts bloquean las descargas externas).
- Consulta los detalles del error en `WP Admin > POS > Support > Logs`.

### La extensión no aparece después de instalarla {#extension-not-appearing-after-install}

- Actualiza el POS: la lista de extensiones se almacena en caché hasta 12 horas.
- Confirma que la extensión esté activada (las extensiones instaladas pero inactivas no se ejecutarán).

### El catálogo no carga {#catalog-not-loading}

El catálogo de extensiones se obtiene desde una fuente remota y se almacena en caché localmente durante 12 horas. Si el catálogo no carga:

- Verifica la conectividad a internet de tu servidor.
- Vuelve a intentarlo cuando expire la caché, o borra la caché transitoria de tu servidor.

## Para desarrolladores {#for-developers}

### Crear una extensión para POS {#creating-a-pos-extension}

Una extensión de WCPOS es un plugin estándar de WordPress que se integra con el POS mediante hooks y APIs de WCPOS. Para crear una:

1. **Empieza con un plugin de WordPress.** Tu extensión necesita una cabecera de plugin estándar y un archivo de entrada, igual que cualquier plugin de WooCommerce o WordPress.
2. **Intégrate con WCPOS.** Utiliza los hooks y filtros que proporciona WCPOS para añadir funcionalidad a la interfaz del POS o al backend.
3. **Aloja las versiones en GitHub.** El directorio de extensiones utiliza GitHub Releases para gestionar las versiones y entregar las actualizaciones.

### Envío al directorio {#submitting-to-the-directory}

El catálogo de extensiones se mantiene en el repositorio de GitHub [`wcpos/extensions`](https://github.com/wcpos/extensions). Para incluir tu extensión:

1. Revisa el formato del catálogo y los requisitos de metadatos en el README del repositorio.
2. Abre una pull request para añadir los metadatos de tu extensión a `catalog.json`.
3. Una vez fusionada, tu extensión aparecerá en el directorio para todos los usuarios de WCPOS Pro.

### Convenciones de GitHub Releases {#github-release-conventions}

El ciclo de vida de las actualizaciones se basa en GitHub Releases:

- **Etiqueta las versiones** utilizando versionado semántico (por ejemplo, `v1.0.0`, `v1.2.3`).
- **Adjunta el zip del plugin** como recurso de la versión: este es el archivo que se descarga cuando un usuario instala o actualiza.
- **Publica la versión** (no como borrador) para que el directorio pueda detectarla.

Cuando publiques una nueva versión, los usuarios que tengan tu extensión instalada verán la actualización disponible en su directorio de extensiones. Si la actualización automática está activada, se aplicará automáticamente.

Para más detalles sobre el esquema del catálogo y el proceso de envío, consulta el repositorio [`wcpos/extensions`](https://github.com/wcpos/extensions).
