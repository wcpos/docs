# Traducido: WCPOS Polylang {#wcpos-polylang}

Añade compatibilidad con [Polylang](https://polylang.pro/) a WCPOS para que el POS solo muestre productos de un único idioma, sin traducciones duplicadas en la búsqueda de productos, la cuadrícula del catálogo ni los flujos de caja. Las tiendas con WCPOS Pro pueden fijar un idioma por tienda; las instalaciones gratuitas utilizan el idioma predeterminado de Polylang.

## Qué hace {#what-it-does}

- Filtra las consultas REST de productos y variaciones de WCPOS por idioma.
- Intercepta las rutas de **sincronización rápida** de WCPOS (las solicitudes ligeras `posts_per_page=-1` + `fields` que el POS utiliza para actualizar su índice local) para que los duplicados traducidos nunca lleguen al cliente.
- En instalaciones gratuitas, aplica el idioma predeterminado de Polylang.
- En instalaciones Pro, cada tienda puede elegir su propio idioma desde una nueva sección **Idioma** en el editor de tiendas.
- Respeta la visibilidad de productos **Solo POS** de WCPOS al construir la carga de sincronización rápida.

La integración se desactiva de forma limpia cuando Polylang no está activo; es posible instalar el plugin antes de habilitar Polylang sin que se produzcan errores.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="ES: Instalar Polylang">

Instale [Polylang](https://wordpress.org/plugins/polylang/) (o Polylang Pro) y configure los idiomas del sitio de forma habitual. Asegúrese de que al menos un idioma esté establecido como predeterminado.

  </StepCard>
  <StepCard number="2" title="ES: Instalar WCPOS Polylang">

Instale desde el directorio de extensiones de WCPOS en `WP Admin > POS > Ajustes > Extensiones`, o descargue la última versión desde la [página de lanzamientos de GitHub](https://github.com/wcpos/wcpos-polylang/releases) y cárguela mediante `Plugins > Añadir nuevo > Subir plugin`.

  </StepCard>
  <StepCard number="3" title="ES: (Pro) Establecer un idioma por tienda" isLast>

Si se utilizan [múltiples tiendas](/stores) en WCPOS Pro, vaya a `POS > Tiendas`, edite una tienda y seleccione su idioma en la sección **Idioma** de la barra lateral. Déjelo en *Predeterminado* para usar el idioma predeterminado de Polylang.

  </StepCard>
</Steps>

## Idioma por tienda (Pro) {#per-store-language-pro}

En WCPOS Pro, el plugin añade una sección **Idioma** a la barra lateral del editor de tiendas. Cada tienda puede vincularse a un único slug de idioma de Polylang; los productos servidos a esa tienda se filtran exclusivamente a ese idioma. Las tiendas configuradas como *Predeterminado* utilizan el idioma predeterminado de Polylang.

El valor por tienda se guarda en el post de la tienda como metadato `_wcpos_polylang_language` y se expone a través de la API REST de tiendas de WCPOS Pro (`/wcpos/v1/stores`), por lo que se transmite a través del POS como cualquier otro ajuste de tienda.

## Notas de compatibilidad {#compatibility-notes}

- **Productos exclusivos del POS:** cuando el modo exclusivo del POS está habilitado en los ajustes de WCPOS, los ID de productos exclusivos de la tienda en línea se excluyen de la carga de sincronización rápida para que no aparezcan en el POS.
- **Instalaciones gratuitas:** no hay una interfaz para cambiar el idioma por tienda — el plugin utiliza el idioma predeterminado de Polylang. Para configurar idiomas por tienda, es necesario actualizar a [WCPOS Pro](/getting-started/pro-license).
- **Plugin no disponible:** si Polylang está desactivado, el plugin no realiza ninguna acción. No generará errores ni bloqueará el POS.

## Hooks para desarrolladores {#developer-hooks}

Para uso avanzado, el plugin expone algunos filtros:

| Filtro | Propósito |
| --- | --- |
| `wcpos_polylang_resolved_language` | Permite sobrescribir el idioma utilizado para una solicitud determinada. Recibe el slug resuelto y el `WP_REST_Request`. |
| `wcpos_polylang_default_language` | Permite sobrescribir el idioma de respaldo cuando no se ha definido un valor por tienda. |
| `wcpos_polylang_is_supported` | Permite forzar la activación o desactivación del plugin independientemente de la detección de Polylang. |
| `wcpos_polylang_minimum_version` | Permite requerir una versión mínima de Polylang (por defecto: sin restricción de versión). |
