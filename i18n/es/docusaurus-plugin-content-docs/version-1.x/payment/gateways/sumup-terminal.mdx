# Pasarela del Terminal SumUp {#sumup-terminal-gateway}

La pasarela del Terminal SumUp te permite aceptar pagos con tarjeta utilizando lectores de tarjetas SumUp directamente dentro de WCPOS. Esta pasarela proporciona una integración fluida con el sistema de procesamiento de pagos de SumUp para transacciones presenciales.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e Instalar">

1. Visita la [página de lanzamientos de SumUp Terminal para WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Descarga el último archivo **sumup-terminal-for-woocommerce.zip**
3. En tu administración de WordPress, ve a `Plugins > Añadir Nuevo > Subir Plugin`
4. Sube el archivo zip y activa el plugin

  </StepCard>
  <StepCard number="2" title="Configurar Ajustes de SumUp">

1. Navega a `WP Admin > WooCommerce > Ajustes > Pagos`
2. Encuentra **SumUp Terminal** en la lista de métodos de pago
3. Haz clic en **SumUp Terminal** para acceder a los ajustes
4. Ingresa tu **Clave API de SumUp** (disponible en el panel de control de comerciantes de SumUp)
5. Guarda los ajustes

:::note
No necesitas habilitar la pasarela del Terminal SumUp en los ajustes de WooCommerce. Se habilitará específicamente para el POS en un paso posterior.
:::

  </StepCard>
  <StepCard number="3" title="Emparejar Tu Terminal SumUp">

1. En la misma página de ajustes, localiza la sección **Emparejar Lector**
2. En tu dispositivo SumUp, navega a la pantalla de emparejamiento para mostrar el código de emparejamiento
3. Ingresa el código de emparejamiento que aparece en tu dispositivo SumUp
4. Haz clic en **"Emparejar Lector"** para establecer la conexión
5. Espera la confirmación de que el lector ha sido emparejado con éxito

:::warning Importante
El lector debe estar emparejado con éxito antes de que puedas procesar pagos. Asegúrate de que el proceso de emparejamiento se complete antes de continuar.
:::

  </StepCard>
  <StepCard number="4" title="Habilitar en WCPOS" isLast>

1. Ve a `WP Admin > POS > Ajustes > Pago`
2. Encuentra la pasarela **SumUp Terminal** en la lista
3. Habilita la pasarela para su uso en el POS
4. Guarda tus ajustes

  </StepCard>
</Steps>

## Uso {#usage}

### Procesamiento de Pagos {#processing-payments}

1. **Añadir Artículos**: Añade productos a tu carrito en el POS
2. **Seleccionar Pasarela**: Elige "SumUp Terminal" como método de pago
3. **Iniciar Pago**: Haz clic para iniciar un nuevo pago en tu dispositivo SumUp
4. **Pago del Cliente**: El cliente completa el pago en el terminal SumUp
5. **Finalización Automática**: El pedido se completa automáticamente cuando se detecta el pago

### Controles de Pago {#payment-controls}

Al utilizar la pasarela del Terminal SumUp, tienes las siguientes opciones:

- **Iniciar Nuevo Pago**: Iniciar una solicitud de pago en el terminal conectado
- **Cancelar Pago**: Cancelar un pago que se esté procesando
- **Estado del Pago**: Monitorea el estado actual del proceso de pago

### Gestión de Pedidos {#order-management}

- **Actualizaciones Automáticas**: Los pedidos se marcan automáticamente como completados tras un pago exitoso
- **Seguimiento de Pagos**: Todos los detalles de pago se registran en las notas del pedido
- **Generación de Recibos**: Se generan recibos estándar del POS después de pagos exitosos

## Compatibilidad de Hardware {#hardware-compatibility}

:::important Requisitos de Conectividad
Esta implementación utiliza el SDK de JavaScript de SumUp, lo que significa que funciona a través de aplicaciones web pero requiere **terminales solo conectados a internet**. Los terminales Bluetooth no son compatibles actualmente.
:::

### Terminales Soportados (Conectados a Internet) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Terminal conectado por WiFi/Ethernet

### Terminales No Soportados (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Solo Bluetooth
- **SumUp 3G** ❌ - Solo Bluetooth

:::note Soporte Futuro
El soporte para terminales Bluetooth se añadirá a las aplicaciones iOS y Android en **Q1 2026**. Esto habilitará el soporte para todos los terminales certificados por SumUp, incluyendo los modelos Air y 3G.
:::

## Solución de Problemas {#troubleshooting}

### Problemas Comunes {#common-issues}

<Accordion>
  <AccordionItem question="El lector no se empareja">

  - Asegúrate de estar utilizando un terminal conectado a internet compatible (solo Solo)
  - Verifica que tu dispositivo SumUp Solo esté conectado a WiFi y en línea
  - Comprueba que el dispositivo esté en modo de emparejamiento y que el código de emparejamiento se haya ingresado correctamente
  - Confirma que tu cuenta de SumUp tenga habilitado el acceso a la API
  - Asegúrate de tener una conexión a internet estable durante el emparejamiento

  </AccordionItem>

  <AccordionItem question="El pago no se procesa">

  - Verifica que el lector esté correctamente emparejado
  - Comprueba que tu cuenta de SumUp esté activa y en buen estado
  - Asegúrate de que el dispositivo tenga suficiente batería y conectividad
  - Confirma que la clave API sea correcta y esté activa

  </AccordionItem>

  <AccordionItem question="El pedido no se completa">

  - Verifica la estabilidad de la conexión a internet
  - Verifica la configuración de webhook en tu cuenta de SumUp
  - Asegúrate de que el pago fue exitoso en el terminal SumUp
  - Revisa los registros de errores de WordPress para problemas de comunicación con la API

  </AccordionItem>
</Accordion>

### Límite de Tasa de API {#api-rate-limits}

La pasarela está optimizada para reducir las llamadas a la API y evitar límites de tasa:
- El estado de pago se verifica de forma eficiente
- Se minimizan las solicitudes API innecesarias
- La lógica de reintento automático maneja fallos temporales
