# Pasarela SumUp Terminal {#sumup-terminal-gateway}

La pasarela SumUp Terminal permite aceptar pagos con tarjeta mediante lectores de tarjetas SumUp directamente en WCPOS. Esta pasarela proporciona una integración fluida con el sistema de procesamiento de pagos de SumUp para transacciones presenciales.

## Características {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Integración de hardware">
    Conexión de lectores de tarjetas SumUp al sistema POS mediante conexión a internet
  </FeatureCard>
  <FeatureCard icon="bolt" title="Procesamiento en tiempo real">
    Procesamiento de pagos instantáneo y finalización de pedidos
  </FeatureCard>
  <FeatureCard icon="shield" title="Transacciones seguras">
    Procesamiento de pagos compatible con PCI a través de SumUp
  </FeatureCard>
  <FeatureCard icon="link" title="Emparejamiento sencillo">
    Proceso de emparejamiento sencillo mediante códigos de vinculación
  </FeatureCard>
</FeatureGrid>

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Instalar SumUp Terminal para WooCommerce">

Se puede instalar desde `WP Admin > POS > Ajustes > Extensiones`, o descargar el último **archivo zip del plugin** (no el zip o tarball del código fuente de GitHub) desde la [página de versiones de GitHub](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) y subirlo mediante `Plugins > Añadir nuevo > Subir plugin`.

  </StepCard>
  <StepCard number="2" title="Configurar los ajustes de SumUp">

1. Navegar a `WP Admin > WooCommerce > Ajustes > Pagos`
2. Buscar **SumUp Terminal** en la lista de métodos de pago
3. Hacer clic en **SumUp Terminal** para acceder a los ajustes
4. Introducir la **clave API de SumUp** (disponible en el panel de comerciante de SumUp)
5. Guardar los ajustes

:::note
No es necesario habilitar la pasarela SumUp Terminal en los ajustes de WooCommerce. Se habilitará específicamente para el POS en un paso posterior.
:::

  </StepCard>
  <StepCard number="3" title="Vincular su SumUp Terminal">

1. En la misma página de configuración, localice la sección **Vincular lector**
2. En su dispositivo SumUp, navegue a la pantalla de vinculación para mostrar el código de vinculación
3. Introduzca el código de vinculación que se muestra en su dispositivo SumUp
4. Haga clic en **"Vincular lector"** para establecer la conexión
5. Espere la confirmación de que el lector se ha vinculado correctamente

:::warning Importante
El lector debe estar vinculado correctamente antes de poder procesar pagos. Asegúrese de que el proceso de vinculación se haya completado antes de continuar.
:::

  </StepCard>
  <StepCard number="4" title="Habilitar en WCPOS" isLast>

1. Vaya a `WP Admin > POS > Ajustes > Finalizar compra`
2. Buscar la pasarela **SumUp Terminal** en la lista
3. Activar la pasarela para su uso en el POS
4. Guardar la configuración

  </StepCard>
</Steps>

## Uso {#usage}

### Procesamiento de pagos {#processing-payments}

1. **Agregar artículos**: Agregar productos al carrito en el POS
2. **Seleccionar pasarela**: Elegir "SumUp Terminal" como método de pago
3. **Iniciar pago**: Hacer clic para iniciar un nuevo pago en el dispositivo SumUp
4. **Pago del cliente**: El cliente completa el pago en el terminal SumUp
5. **Completado automático**: El pedido se completa automáticamente cuando se detecta el pago

### Controles de pago {#payment-controls}

Al utilizar la pasarela SumUp Terminal, se dispone de las siguientes opciones:

- **Iniciar nuevo pago**: Iniciar una solicitud de pago en el terminal conectado
- **Cancelar pago**: Cancelar un pago actualmente en proceso
- **Estado del pago**: Supervisar el estado actual del proceso de pago

### Gestión de pedidos {#order-management}

- **Actualizaciones automáticas**: Los pedidos se marcan automáticamente como completados tras un pago exitoso
- **Seguimiento de pagos**: Todos los detalles del pago se registran en las notas del pedido
- **Generación de recibos**: Se generan recibos POS estándar tras los pagos exitosos

## Compatibilidad de hardware {#hardware-compatibility}

:::important Requisitos de conectividad
Esta implementación utiliza el SDK de JavaScript de SumUp, lo que significa que funciona a través de aplicaciones web pero requiere **terminales con conexión a internet únicamente**. Los terminales Bluetooth no son compatibles actualmente.
:::

### Terminales compatibles (con conexión a internet) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Terminal conectado por WiFi/Ethernet

### Terminales no compatibles (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Solo Bluetooth
- **SumUp 3G** ❌ - Solo Bluetooth

:::note Compatibilidad futura
La compatibilidad con terminales Bluetooth está planificada para una futura versión de la aplicación para iOS y Android. Cuando esté disponible, permitirá el uso de todos los terminales certificados por SumUp, incluidos los modelos Air y 3G.
:::

## Solución de problemas {#troubleshooting}

### Problemas comunes {#common-issues}

<Accordion>
  <AccordionItem question="El lector no se empareja">

  - Asegúrese de utilizar un terminal compatible con conexión a internet (únicamente el modelo Solo)
  - Verifique que el dispositivo SumUp Solo esté conectado a WiFi y en línea
  - Verifique que el dispositivo esté en modo de emparejamiento y que el código de emparejamiento se haya introducido correctamente
  - Confirmar que la cuenta de SumUp tiene habilitado el acceso a la API
  - Asegurar una conexión a internet estable durante el emparejamiento

  </AccordionItem>

  <AccordionItem question="El pago no se procesa">

  - Verificar que el lector esté correctamente emparejado
  - Comprobar que la cuenta de SumUp esté activa y en buen estado
  - Asegurar que el dispositivo tenga suficiente batería y conectividad
  - Confirmar que la clave API sea correcta y esté activa

  </AccordionItem>

  <AccordionItem question="El pedido no se completa">

  - Comprobar la estabilidad de la conexión a internet
  - Verificar la configuración de webhooks en la cuenta de SumUp
  - Asegurar que el pago se haya realizado correctamente en el terminal SumUp
  - Revisar los registros de errores de WordPress en busca de problemas de comunicación con la API

  </AccordionItem>
</Accordion>

### Límites de tasa de API {#api-rate-limits}

La pasarela está optimizada para reducir las llamadas a la API y evitar la limitación de tasa:
- El estado del pago se verifica de forma eficiente
- Las solicitudes innecesarias a la API se minimizan
- La lógica de reintento automático gestiona los fallos temporales
