# Pasarela de factura por correo electrónico {#email-invoice-gateway}

La pasarela de factura por correo electrónico permite enviar facturas de pago a los clientes por correo electrónico directamente desde WCPOS. Es ideal para situaciones en las que los clientes prefieren pagar más tarde o necesitan completar el pago en línea utilizando los métodos de pago de la tienda web.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Instalar la pasarela de factura por correo electrónico">

Instalar desde `WP Admin > POS > Ajustes > Extensiones`, o descargar el último **archivo zip del plugin** (no el zip o tarball del código fuente de GitHub) desde la [página de lanzamientos de GitHub](https://github.com/wcpos/email-invoice-gateway/releases) y cargarlo a través de `Plugins > Añadir nuevo > Subir plugin`.

  </StepCard>
  <StepCard number="2" title="Habilitar en WCPOS" isLast>

1. Ir a `WP Admin > POS > Ajustes > Finalizar compra`
2. Buscar la **pasarela de pago por factura de WCPOS** en la lista
3. Habilitar la pasarela para su uso en el POS
4. Guardar los ajustes

:::note
Esta pasarela no requiere claves de API adicionales ni configuración de servicios externos. Utiliza el sistema de correo electrónico existente de WordPress y los métodos de pago de WooCommerce.
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Envío de facturas {#sending-invoices}

1. **Agregar artículos**: Agregar productos al carrito en el POS
2. **Seleccionar pasarela**: Elegir "Factura por correo electrónico" como método de pago
3. **Dirección de correo electrónico**: 
   - Para clientes existentes: el correo electrónico se completa automáticamente a partir de los datos del cliente
   - Para pedidos de invitados: introducir la dirección de correo electrónico del cliente manualmente
4. **Opcional**: Marcar "Guardar en dirección de facturación" para actualizar la información de facturación del cliente
5. **Enviar factura**: Completar el proceso para enviar el correo electrónico de factura

### Experiencia del cliente {#customer-experience}

Al enviar una factura, el cliente recibe un correo electrónico que contiene:

- **Detalles del pedido**: Desglose completo de artículos, cantidades y precios
- **Total del pedido**: Importe final adeudado, incluyendo impuestos y tasas
- **Enlace de pago**: Enlace directo para completar el pago en línea
- **Información del pedido**: Número de pedido y detalles de referencia

### Finalización del pago {#payment-completion}

Los clientes pueden completar el pago de dos formas:

1. **Pago en línea**: Hacer clic en el enlace "Pagar este pedido" en el correo electrónico
2. **Regreso a la tienda**: Volver para completar el pago utilizando otros métodos de pago del POS

El enlace "Pagar este pedido" dirige a los clientes a una página de pago dedicada en el sitio web, donde pueden utilizar cualquiera de las pasarelas de pago web habilitadas (Stripe, PayPal, etc.).

## Gestión de correo electrónico {#email-management}

### Clientes existentes {#existing-customers}

- Las direcciones de correo electrónico se obtienen automáticamente de los perfiles de los clientes
- La información de facturación del cliente se conserva
- El historial de pedidos se mantiene en la cuenta del cliente

### Clientes invitados {#guest-customers}

- Permite introducir manualmente direcciones de correo electrónico para clientes ocasionales
- Opción de guardar el correo electrónico en la dirección de facturación para referencia futura
- Los pedidos de invitados se registran correctamente en WooCommerce

### Plantillas de correo electrónico {#email-templates}

La pasarela utiliza las plantillas de correo electrónico estándar de WooCommerce:

- **Personalizable**: Permite modificar la apariencia del correo electrónico a través de la configuración de correo de WooCommerce
- **Con marca**: Permite incluir el logotipo y la imagen de marca de la tienda
- **Profesional**: Formato de factura limpio y profesional
- **Adaptable a móviles**: Diseño responsivo para todos los dispositivos

## Casos de uso {#use-cases}

### Ideal para {#perfect-for}

- **Ventas B2B**: Clientes empresariales que necesitan procesar pagos a través de sus sistemas contables
- **Pedidos grandes**: Transacciones de alto valor que requieren aprobación o tiempo de procesamiento
- **Clientes remotos**: Clientes que necesitan completar el pago después de abandonar la tienda
- **Clientes con cuenta**: Clientes habituales con condiciones de pago establecidas
- **Presupuesto a pedido**: Conversión de presupuestos o estimaciones en facturas pagaderas

## Solución de problemas {#troubleshooting}

### Problemas comunes {#common-issues}

<Accordion>
  <AccordionItem question="Los correos electrónicos no se envían">

  - Verificar la configuración de correo electrónico de WordPress
  - Verificar la configuración SMTP si se utiliza un servicio de correo electrónico personalizado
  - Realizar pruebas con plugins de prueba de correo electrónico de WordPress
  - Revisar las carpetas de spam en busca de correos electrónicos de prueba

  </AccordionItem>

  <AccordionItem question="Los enlaces de pago no funcionan">

  - Asegurarse de que WooCommerce esté configurado correctamente
  - Verificar que al menos una pasarela de pago esté habilitada para el proceso de pago en línea
  - Verificar que el certificado SSL esté correctamente instalado
  - Confirmar que el estado del pedido permite el pago

  </AccordionItem>

  <AccordionItem question="El cliente no puede completar el pago">

  - Verificar que las pasarelas de pago web estén activas y configuradas
  - Comprobar que el pedido no haya expirado ni sido cancelado
  - Asegurarse de que el cliente tenga límites suficientes en su método de pago
  - Probar el proceso de pago personalmente

  </AccordionItem>
</Accordion>

### Entrega de correo electrónico {#email-delivery}

Para una entrega de correo electrónico confiable, se recomienda considerar:

- **Servicio SMTP**: Utilizar servicios como SendGrid, Mailgun o Amazon SES
- **Plugins de correo electrónico**: Instalar plugins SMTP de WordPress para mejorar la entrega
- **Autenticación de dominio**: Configurar los registros SPF, DKIM y DMARC
- **Monitoreo**: Seguimiento de la entrega de correos electrónicos y tasas de apertura
