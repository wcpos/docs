# Pasarela Stripe Terminal {#stripe-terminal-gateway}

La pasarela Stripe Terminal permite aceptar pagos presenciales mediante lectores de hardware Stripe Terminal directamente en WCPOS. Esta pasarela es compatible tanto con lectores de tarjetas físicos como con el modo simulador para pruebas.

## Características {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Integración de hardware">
    Conexión de lectores Stripe Terminal físicos mediante conexión a internet
  </FeatureCard>
  <FeatureCard icon="desktop" title="Modo simulador">
    Prueba de pagos sin hardware utilizando el simulador de Stripe
  </FeatureCard>
  <FeatureCard icon="bolt" title="Procesamiento en tiempo real">
    Procesamiento y confirmación de pagos instantáneos
  </FeatureCard>
  <FeatureCard icon="shield" title="Transacciones seguras">
    Procesamiento de pagos compatible con PCI a través de Stripe
  </FeatureCard>
  <FeatureCard icon="phone" title="Pedidos telefónicos (MOTO)">
    Aceptar pagos con tarjeta por teléfono ingresando los datos en el lector
  </FeatureCard>
</FeatureGrid>

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Instalar Stripe Terminal para WooCommerce">

Instalar desde `WP Admin > POS > Ajustes > Extensiones`, o descargar el último **archivo zip del plugin** (no el zip o tarball del código fuente de GitHub) desde la [página de lanzamientos de GitHub](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases) y subirlo a través de `Plugins > Añadir nuevo > Subir plugin`.

  </StepCard>
  <StepCard number="2" title="Configurar los ajustes de Stripe">

1. Navegar a `WP Admin > WooCommerce > Ajustes > Pagos`
2. Buscar **Stripe Terminal** en la lista de métodos de pago
3. Hacer clic en **Stripe Terminal** para acceder a los ajustes
4. Introducir la **clave secreta de Stripe** (se puede obtener desde el panel de control de Stripe)
5. Guarde los ajustes

:::note
No es necesario habilitar la pasarela Stripe Terminal en la configuración de WooCommerce. Se habilitará específicamente para el POS en el siguiente paso.
:::

  </StepCard>
  <StepCard number="3" title="Habilitar en WCPOS" isLast>

1. Ir a `WP Admin > POS > Ajustes > Finalizar compra`
2. Buscar la pasarela **Stripe Terminal** en la lista
3. Habilitar la pasarela para su uso en el POS
4. Guardar los ajustes

  </StepCard>
</Steps>

## Uso {#usage}

### Conexión de un lector {#connecting-a-reader}

Al seleccionar la pasarela Stripe Terminal durante el proceso de pago en el POS:

1. **Elegir método de conexión**: Es posible conectar un lector físico o utilizar el simulador
2. **Lector físico**: Siga las instrucciones en pantalla para conectar el dispositivo Stripe Terminal
3. **Simulador**: Seleccione el modo simulador para probar diversos escenarios de pago sin hardware

### Procesamiento de pagos {#processing-payments}

1. **Agregar artículos**: Agregue productos al carrito en el POS
2. **Seleccionar pasarela**: Elija "Stripe Terminal" como método de pago
3. **Conectar lector**: Conecte el lector o elija el modo simulador
4. **Procesar pago**: Siga las indicaciones para completar la transacción
5. **Confirmación**: El pedido se completará automáticamente tras un pago exitoso

### Pruebas con el simulador {#testing-with-simulator}

El simulador permite probar diversos métodos y escenarios de pago:

- **Pagos con tarjeta**: Pruebe diferentes tipos de tarjeta (Visa, Mastercard, etc.)
- **Pagos sin contacto**: Simule transacciones de pago por aproximación
- **Escenarios de error**: Pruebe pagos rechazados y otras condiciones de error
- **Diferentes importes**: Pruebe diversos montos de transacción

### Pedidos telefónicos (MOTO) {#phone-orders-moto}

MOTO (Mail Order/Telephone Order) permite procesar pagos con tarjeta de clientes que no están físicamente presentes, por ejemplo, al recibir un pedido por teléfono. En lugar de acercar o insertar una tarjeta, el comerciante introduce los datos de la tarjeta directamente en la pantalla del lector del terminal.

#### Configuración {#setup}

<Steps>
  <StepCard number="1" title="Solicitar acceso MOTO a Stripe">

MOTO no está habilitado de forma predeterminada. Contacte con el [soporte de Stripe](https://support.stripe.com/contact) y solicite que habiliten los permisos MOTO para su cuenta. Es un proceso rápido, pero requiere aprobación manual por parte de Stripe.

  </StepCard>
  <StepCard number="2" title="Habilitar en los ajustes del plugin">

1. Navegue a `WP Admin > WooCommerce > Ajustes > Pagos > Stripe Terminal`
2. Marque la casilla **Pedidos telefónicos (MOTO)**
3. Guarde los ajustes

  </StepCard>
  <StepCard number="3" title="Conectar un lector compatible" isLast>

MOTO solo funciona con lectores compatibles conectados a internet que se enumeran en [Terminales compatibles](#supported-terminals-internet-connected). El interruptor no aparecerá para otros tipos de lector.

  </StepCard>
</Steps>

#### Recibir un pedido por teléfono {#taking-a-phone-order}

1. Conectar un lector compatible (consultar [Terminales compatibles](#supported-terminals-internet-connected))
2. En la pantalla de pago, activar el interruptor **Pedido por teléfono**
3. Hacer clic en **Cobrar pago con tarjeta** — el lector mostrará una pantalla de ingreso de número de tarjeta en lugar de solicitar acercar/insertar la tarjeta
4. Ingresar el número de tarjeta, la fecha de vencimiento y el CVV del cliente en el lector
5. A partir de ahí, el pago se procesa de forma normal

:::tip
Los pagos MOTO utilizan `card` como tipo de método de pago en lugar de `card_present`. Esto significa que desde la perspectiva de Stripe se tratan más como transacciones en línea, por lo que se aplican las tarifas estándar de procesamiento de tarjetas en línea en lugar de las tarifas presenciales.
:::

:::caution
El interruptor de Pedido por teléfono solo aparece cuando se cumplen las tres condiciones: la configuración MOTO está habilitada en los ajustes del plugin, hay un lector compatible conectado y el lector no es un simulador. Si no se ve el interruptor, verificar estas condiciones.
:::

## Compatibilidad de hardware {#hardware-compatibility}

:::important Requisitos de conectividad
Esta implementación utiliza el SDK de JavaScript de Stripe, lo que significa que funciona a través de aplicaciones web pero requiere **terminales con conexión a internet únicamente**. Los terminales Bluetooth no son compatibles actualmente.
:::

### Terminales compatibles (con conexión a internet) {#supported-terminals-internet-connected}

- **Stripe Reader S700/S710** ✅ - Terminal con conexión Ethernet/WiFi
- **WisePOS E** ✅ - Terminal con conexión WiFi

### Terminales no compatibles (Bluetooth) {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - Solo Bluetooth
- **BBPOS WisePad 3** ❌ - Solo Bluetooth  
- **Verifone P400** ❌ - Solo Bluetooth

:::note Compatibilidad futura
La compatibilidad con terminales Bluetooth está planificada para una futura versión de la aplicación para iOS y Android. Cuando esté disponible, permitirá la compatibilidad con todos los lectores certificados de Stripe Terminal, incluidos el M2 y el WisePad 3.
:::

### Problemas comunes {#common-issues}

<Accordion>
  <AccordionItem question="El lector no se conecta">

  - Asegúrese de utilizar un [terminal con conexión a internet compatible](#supported-terminals-internet-connected)
  - Verifique que el terminal esté conectado a WiFi/Ethernet y en línea
  - Compruebe que su cuenta de Stripe tenga Terminal habilitado
  - Confirme que el terminal esté registrado en su panel de Stripe

  </AccordionItem>

  <AccordionItem question="Pago rechazado">

  - Compruebe que su cuenta de Stripe esté activa y en buen estado
  - Verifique que la tarjeta utilizada sea válida
  - Asegúrese de que haya fondos suficientes disponibles

  </AccordionItem>

  <AccordionItem question="No se muestra el interruptor de pedido telefónico">

  - Verifique que la opción **Pedidos telefónicos (MOTO)** esté habilitada en `WooCommerce > Ajustes > Pagos > Stripe Terminal`
  - Es necesario estar conectado a un lector compatible (consulte [Terminales compatibles](#supported-terminals-internet-connected)) — el interruptor no se muestra para otros tipos de lector
  - El interruptor no aparece cuando se utiliza el simulador

  </AccordionItem>

  <AccordionItem question="El pago MOTO falla con un error">

  - Es necesario confirmar que Stripe ha habilitado los permisos MOTO en la cuenta — se debe contactar con el [soporte de Stripe](https://support.stripe.com/contact) si aún no se ha hecho
  - Se recomienda verificar que los datos de la tarjeta se hayan introducido correctamente en el lector
  - Los pagos MOTO pueden tener controles de fraude más estrictos — es necesario asegurarse de que la tarjeta sea válida y tenga fondos suficientes

  </AccordionItem>

  <AccordionItem question="Errores de certificado SSL">

  - Stripe Terminal requiere HTTPS — es necesario asegurarse de que el sitio tenga un certificado SSL válido
  - Se debe verificar que el certificado SSL esté correctamente configurado

  </AccordionItem>
</Accordion>
