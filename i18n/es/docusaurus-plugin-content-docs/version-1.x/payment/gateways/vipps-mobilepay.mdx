# Pasarela Vipps MobilePay {#vipps-mobilepay-gateway}

La pasarela Vipps MobilePay te permite aceptar pagos basados en teléfono directamente dentro de WCPOS. Los clientes pagan escaneando un código QR en la pantalla o confirmando una notificación push en su teléfono, sin necesidad de hardware de lector de tarjetas.

## Características {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="Pagos con Código QR">
    Muestra un código QR en el proceso de pago para que los clientes lo escaneen y paguen con su teléfono
  </FeatureCard>
  <FeatureCard icon="bolt" title="Notificaciones Push">
    Envía una solicitud de pago directamente al teléfono del cliente ingresando su número
  </FeatureCard>
  <FeatureCard icon="globe" title="Cobertura Nórdica">
    Funciona con Vipps en Noruega, MobilePay en Dinamarca y Finlandia
  </FeatureCard>
  <FeatureCard icon="credit-card" title="Captura Automática">
    Captura fondos inmediatamente después de la autorización, o reserva para captura manual
  </FeatureCard>
</FeatureGrid>

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e Instalar">

1. Visita la [página de lanzamientos de WCPOS Vipps MobilePay](https://github.com/wcpos/wcpos-vipps/releases)
2. Descarga el archivo **wcpos-vipps.zip** más reciente
3. En tu administrador de WordPress, ve a `Plugins > Agregar Nuevo > Subir Plugin`
4. Sube el archivo zip y activa el plugin

  </StepCard>
  <StepCard number="2" title="Configurar Credenciales de Vipps">

1. Navega a `WP Admin > WooCommerce > Configuración > Pagos`
2. Encuentra **WCPOS Vipps MobilePay** en la lista de métodos de pago
3. Haz clic en **WCPOS Vipps MobilePay** para acceder a la configuración
4. Ingresa tus credenciales del [Portal de Vipps](https://portal.vipps.no/):
   - **Número de Serie del Comerciante (MSN)**
   - **ID de Cliente**
   - **Secreto de Cliente**
   - **Clave de Suscripción**
5. Guarda la configuración

:::note
No necesitas habilitar la pasarela aquí para su uso en el POS; se habilitará específicamente para el POS en el siguiente paso. Habilitarla en la configuración de WooCommerce también la haría disponible en el proceso de pago de tu tienda en línea, lo cual puede ser útil para pruebas.
:::

:::tip ¿Ya usas el plugin de Vipps?
Si tienes instalado el plugin oficial [Checkout with Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/), tus credenciales serán importadas automáticamente cuando actives este plugin. Puedes omitir la entrada manual de credenciales.
:::

  </StepCard>
  <StepCard number="3" title="Habilitar en WCPOS" isLast>

1. Ve a `WP Admin > POS > Configuración > Checkout`
2. Encuentra la pasarela **WCPOS Vipps MobilePay** en la lista
3. Habilita la pasarela para su uso en el POS
4. Guarda tus configuraciones

  </StepCard>
</Steps>

## Uso {#usage}

### Procesamiento de Pagos — Código QR {#processing-payments-qr-code}

1. **Agregar Artículos**: Agrega productos a tu carrito en el POS
2. **Seleccionar Pasarela**: Escoge "Vipps MobilePay" como método de pago
3. **Generar Código QR**: Haz clic en el botón "Generar Código QR"
4. **El Cliente Escanea**: El cliente escanea el código QR con su aplicación Vipps o MobilePay
5. **El Cliente Confirma**: El cliente confirma el pago en su aplicación
6. **Finalización Automática**: El pedido se completa automáticamente una vez que se autoriza el pago

### Procesamiento de Pagos — Enviar al Teléfono {#processing-payments-send-to-phone}

1. **Agregar Artículos**: Agrega productos a tu carrito en el POS
2. **Seleccionar Pasarela**: Escoge "Vipps MobilePay" como método de pago
3. **Ingresar Número de Teléfono**: Escribe el número de teléfono del cliente
4. **Enviar al Teléfono**: Haz clic en el botón "Enviar al Teléfono"
5. **El Cliente Confirma**: El cliente recibe una notificación push o es dirigido a la página de destino de Vipps, luego confirma el pago en su aplicación
6. **Finalización Automática**: El pedido se completa automáticamente una vez que se autoriza el pago

El plugin detecta automáticamente el mejor método para enviar la solicitud de pago al teléfono del cliente:

- **Push directo** (preferido): Envía una notificación directamente a la aplicación Vipps del cliente. Esta es la experiencia más rápida, pero requiere que Vipps habilite `PUSH_MESSAGE` en tu unidad de ventas (ver [Habilitando Push Directo](#enabling-direct-push) a continuación).
- **Fallback de página de destino**: Si no se habilita el push directo, el plugin abre la página de destino de Vipps en una nueva pestaña del navegador. La página de destino se encarga de enviar la notificación. Esto funciona inmediatamente sin aprobación especial.

La primera vez que el plugin detecta que el push directo no está disponible, verás un breve mensaje pidiéndote que hagas clic en "Enviar al Teléfono" nuevamente. Después de eso, recuerda el resultado y funciona sin problemas.

### Habilitando el Push Directo {#enabling-direct-push}

El flujo de push directo proporciona la mejor experiencia para pagos con teléfono, sin pestañas adicionales ni páginas de destino. Para habilitarlo:

1. Inicia sesión en [portal.vippsmobilepay.com](https://portal.vippsmobilepay.com)
2. Contacta a tu gerente de cuentas clave de Vipps, gerente de socios o servicio al cliente
3. Diles: **"Necesito que se habilite PUSH_MESSAGE en mi MSN para usarlo con una integración de POS"**

Una vez aprobado, el plugin detecta el cambio automáticamente dentro de las 24 horas y cambia al flujo de push directo. Verás un aviso de recordatorio en la página de configuración de la pasarela mientras estás usando la página de destino como fallback.

### Cancelando un Pago {#cancelling-a-payment}

Mientras esperas a que el cliente confirme, puedes hacer clic en el botón **Cancelar Pago** para abortar la transacción. Esto cancela el pago pendiente del lado de Vipps y restablece la interfaz de pago.

### Reembolsos {#refunds}

Los reembolsos se gestionan a través del proceso estándar de reembolso de WooCommerce. Abre el pedido, haz clic en **Reembolsar**, ingresa el monto y el reembolso se procesa automáticamente a través de la API de Vipps.

## Mercados Soportados {#supported-markets}

Vipps MobilePay opera en la región nórdica bajo dos marcas:

| Región | Marca | Divisas |
|--------|-------|---------|
| Noruega | Vipps | NOK |
| Dinamarca, Finlandia | MobilePay | DKK, EUR |

Tu cuenta de comerciante determina qué mercados y divisas están disponibles. Los clientes en cualquier mercado soportado pueden pagar usando su aplicación local de Vipps o MobilePay.

## Resolución de Problemas {#troubleshooting}

### Problemas Comunes {#common-issues}

<Accordion>
  <AccordionItem question="El pago no se completa">

  - El proceso de pago consulta durante hasta 5 minutos; si el cliente no confirma dentro de ese tiempo, el pago expira
  - Verifica que no estés mezclando el modo de prueba y las credenciales de producción
  - Verifica que tu conexión a Internet sea estable en el dispositivo POS y en el teléfono del cliente

  </AccordionItem>

  <AccordionItem question="Error de credenciales inválidas">

  - Verifica los cuatro campos de credenciales (MSN, ID de Cliente, Secreto de Cliente, Clave de Suscripción)
  - Asegúrate de usar credenciales de prueba cuando el modo de prueba esté habilitado, y credenciales de producción cuando esté deshabilitado
  - Verifica tus credenciales en el [Portal de Vipps](https://portal.vipps.no/)

  </AccordionItem>

  <AccordionItem question="'Enviar al Teléfono' abre una nueva pestaña en lugar de enviar directamente">

  Esto significa que tu cuenta de Vipps aún no tiene habilitado `PUSH_MESSAGE`. El plugin está utilizando el fallback de página de destino, que funciona pero añade un paso adicional. Para obtener una experiencia de push directo más suave, contacta a Vipps y pídeles que habiliten PUSH_MESSAGE en tu unidad de ventas (MSN). Ver [Habilitando Push Directo](#enabling-direct-push) arriba.

  </AccordionItem>

  <AccordionItem question="Código QR no se genera">

  - Confirma que tu sitio está funcionando sobre HTTPS, ya que la API de Vipps rechaza solicitudes de sitios HTTP
  - Verifica los registros de WooCommerce (`WooCommerce > Estado > Registros`) para detalles de error de API
  - Verifica que tu cuenta de comerciante esté activa y tenga habilitada la API de ePayment

  </AccordionItem>
</Accordion>
