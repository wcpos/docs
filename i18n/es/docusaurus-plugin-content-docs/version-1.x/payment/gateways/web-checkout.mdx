# Pasarela de Pago por Web {#web-checkout-gateway}

La pasarela de pago por web permite a los clientes completar sus transacciones de POS utilizando el sistema de pago de su tienda en línea. Esta pasarela crea un puente sin fisuras entre su POS en la tienda y los métodos de pago en línea, perfecto para situaciones en las que los clientes prefieren pagar usando sus propios dispositivos o cuando necesita acceso a métodos de pago solo disponibles en web.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e Instalar">

1. Visite la [página de lanzamientos de la Pasarela de Pago por Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Descargue el archivo más reciente **woocommerce-pos-web-checkout-gateway.zip**
3. En su administración de WordPress, vaya a `Plugins > Agregar Nuevo > Subir Plugin`
4. Cargue el archivo zip y active el plugin

  </StepCard>
  <StepCard number="2" title="Habilitar en WCPOS" isLast>

1. Vaya a `WP Admin > POS > Configuración > Pago`
2. Encuentre la **Pasarela de Pago por Web de WCPOS** en la lista
3. Habilite la pasarela para su uso en el POS
4. Guardar su configuración

:::note
Esta pasarela aprovecha sus métodos de pago existentes de WooCommerce, así que asegúrese de tener al menos una pasarela de pago en línea correctamente configurada (Stripe, PayPal, etc.).
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Iniciando el Pago por Web {#initiating-web-checkout}

1. **Agregar Artículos**: Agregue productos a su carrito en el POS
2. **Seleccionar Pasarela**: Elija "Pago por Web" como método de pago
3. **Generar Enlace**: El sistema crea un enlace de pago único para el pedido
4. **Acceso del Cliente**: El cliente hace clic en el enlace para acceder al pago por web
5. **Pago en Línea**: El cliente completa el pago utilizando los métodos de pago de la tienda en línea

### Proceso de Pago {#payment-process}

La pasarela de pago por web sigue este flujo de trabajo:

1. **Creación del Pedido**: El POS crea un pedido pendiente en WooCommerce
2. **Enlace de Pago**: El sistema genera un enlace de pago único y seguro
3. **Redirección del Cliente**: El cliente es dirigido a la página de pago por web
4. **Selección de Pago**: El cliente elige entre los métodos de pago en línea disponibles
5. **Procesamiento del Pago**: El proceso estándar de pago de WooCommerce maneja el pago
6. **Finalización del Pedido**: El estado del pedido se actualiza automáticamente tras el pago exitoso

### Flujo de Trabajo del POS {#pos-workflow}

Después de iniciar el pago por web:

1. **Monitorear Estado**: Mantenga la pantalla de pedidos del POS abierta para monitorear el estado del pago
2. **Pago del Cliente**: El cliente completa el pago en su dispositivo o en su tableta
3. **Actualización Automática**: El estado del pedido se actualiza en tiempo real cuando se recibe el pago
4. **Procesar Pago**: Haga clic en el botón "Procesar Pago" en el POS para continuar con el recibo
5. **Generación de Recibo**: Genere e imprima el recibo como de costumbre

## Casos de Uso {#use-cases}

- **Preferencia del Cliente**: Clientes que prefieren utilizar sus propias aplicaciones de pago o tarjetas

## Detalles Técnicos {#technical-details}

### Gestión de Pedidos {#order-management}

- **Pedidos Pendientes**: Se crean pedidos con estado de "pago pendiente"
- **Actualizaciones de Estado**: Cambios automáticos de estado cuando se recibe el pago
- **Notas de Pedido**: Los detalles del pago se registran en las notas del pedido
- **Inventario**: El stock se reserva durante el proceso de pago

### Seguridad {#security}

- **Enlaces Únicos**: Cada sesión de pago tiene un URL único y limitado en el tiempo
- **SSL Requerido**: Todo el procesamiento de pagos se realiza a través de conexiones HTTPS seguras
- **Cumplimiento de PCI**: Aprovecha sus pasarelas de pago web existentes cumpliendo con el PCI
- **Gestión de Sesiones**: El manejo seguro de sesiones previene accesos no autorizados

### Métodos de Pago {#payment-methods}

La pasarela proporciona acceso a todos sus métodos de pago configurados en WooCommerce:

- **Tarjetas de Crédito/Débito**: Stripe, Square, Authorize.net, etc.
- **Billeteras Digitales**: PayPal, Apple Pay, Google Pay
- **Transferencias Bancarias**: Métodos de pago bancarios directos
- **Compra Ahora, Paga Después**: Klarna, Afterpay, Sezzle
- **Criptomonedas**: Bitcoin y otras pasarelas de pago en criptomonedas
- **Métodos Locales**: Métodos de pago específicos de la región

## Solución de Problemas {#troubleshooting}

### Problemas Comunes {#common-issues}

<Accordion>
  <AccordionItem question="El enlace de pago no funciona">

  - Verifique que el certificado SSL esté instalado correctamente
  - Revise que los enlaces permanentes de WooCommerce estén configurados correctamente
  - Asegúrese de que las pasarelas de pago web estén activas y configuradas
  - Pruebe con un navegador o dispositivo diferente

  </AccordionItem>

  <AccordionItem question="El pago no se procesa">

  - Confirme que las pasarelas de pago web estén configuradas correctamente
  - Revise los registros de la pasarela de pago en busca de mensajes de error
  - Verifique que el método de pago del cliente sea compatible
  - Pruebe el proceso de pago por web de manera independiente

  </AccordionItem>

  <AccordionItem question="El pedido no se actualiza en el POS">

  - Verifique la estabilidad de la conexión a internet
  - Asegúrese de que los trabajos de cron de WordPress se estén ejecutando correctamente
  - Actualice la pantalla de pedidos del POS
  - Revise el estado del pedido en la administración de WooCommerce

  </AccordionItem>

  <AccordionItem question="El cliente no puede acceder al pago">

  - Asegúrese de que el enlace de pago no haya expirado
  - Verifique que el dispositivo del cliente tenga conectividad a internet
  - Revise que el pedido no haya sido cancelado o completado
  - Pruebe el enlace en un dispositivo diferente

  </AccordionItem>
</Accordion>

### Optimización del Rendimiento {#performance-optimisation}

Para un rendimiento óptimo:

- **Caché**: Configure los plugins de caché para excluir las páginas de pago
- **CDN**: Asegúrese de que la configuración de la CDN no interfiera con el proceso de pago
- **Base de Datos**: Optimice la base de datos para acelerar el procesamiento de pedidos
- **Hosting**: Utilice un hosting confiable con buen tiempo de actividad
