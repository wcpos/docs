# Pasarela de pago web {#web-checkout-gateway}

La pasarela de pago web permite a los clientes completar sus transacciones POS utilizando el sistema de pago de la tienda en línea. Esta pasarela crea un puente transparente entre el POS en tienda y los métodos de pago en línea, ideal para situaciones en las que los clientes prefieren pagar con sus propios dispositivos o cuando se necesita acceso a métodos de pago exclusivos de la web.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e instalar">

1. Visite la [página de versiones de la pasarela de pago web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Descargue el archivo **woocommerce-pos-web-checkout-gateway.zip** más reciente
3. En el administrador de WordPress, vaya a `Plugins > Añadir nuevo > Subir plugin`
4. Suba el archivo zip y active el plugin

  </StepCard>
  <StepCard number="2" title="Habilitar en WCPOS" isLast>

1. Vaya a `WP Admin > POS > Ajustes > Finalizar compra`
2. Busque **Pasarela de pago web de WCPOS** en la lista
3. Habilite la pasarela para su uso en el POS
4. Guarde los ajustes

:::note
Esta pasarela aprovecha los métodos de pago existentes de WooCommerce, por lo que es necesario tener al menos una pasarela de pago web correctamente configurada (Stripe, PayPal, etc.).
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Iniciar el pago web {#initiating-web-checkout}

1. **Agregar artículos**: Agregar productos al carrito en el POS
2. **Seleccionar pasarela**: Elegir "Pago Web" como método de pago
3. **Generar enlace**: El sistema crea un enlace de pago único para el pedido
4. **Acceso del cliente**: El cliente hace clic en el enlace para acceder al pago web
5. **Pago en línea**: El cliente completa el pago utilizando los métodos de pago de la tienda web

### Proceso de pago {#payment-process}

La pasarela de Pago Web sigue este flujo de trabajo:

1. **Creación del pedido**: POS crea un pedido pendiente en WooCommerce
2. **Enlace de pago**: El sistema genera un enlace de pago único y seguro
3. **Redirección del cliente**: El cliente es dirigido a la página de pago web
4. **Selección de pago**: El cliente elige entre los métodos de pago web disponibles
5. **Procesamiento del pago**: El proceso de pago estándar de WooCommerce gestiona el pago
6. **Finalización del pedido**: El estado del pedido se actualiza automáticamente tras un pago exitoso

### Flujo de trabajo en POS {#pos-workflow}

Después de iniciar el pago web:

1. **Supervisar el estado**: Mantenga abierta la pantalla de pedido del POS para supervisar el estado del pago
2. **Pago del cliente**: El cliente completa el pago en su dispositivo o en la tableta
3. **Actualización automática**: El estado del pedido se actualiza en tiempo real cuando se recibe el pago
4. **Procesar pago**: Haga clic en el botón "Process Payment" en POS para continuar al recibo
5. **Generación de recibo**: Se genera e imprime el recibo de forma habitual

## Casos de uso {#use-cases}

### Ideal para {#perfect-for}

- **Preferencia del cliente**: Clientes que prefieren utilizar sus propias aplicaciones de pago o tarjetas
- **Pagos complejos**: Transacciones que requieren métodos de pago no disponibles en POS
- **Pagos divididos**: Clientes que desean utilizar múltiples métodos de pago
- **Programas de fidelización**: Acceso al canje de puntos de fidelización basado en la web
- **Tarjetas de regalo**: Clientes con tarjetas de regalo digitales o crédito en tienda
- **Tarjetas internacionales**: Métodos de pago que funcionan mejor a través de pasarelas web

## Detalles técnicos {#technical-details}

### Gestión de pedidos {#order-management}

- **Pedidos pendientes**: Los pedidos se crean con el estado "Pendiente de pago"
- **Actualizaciones de estado**: Cambios de estado automáticos cuando se recibe el pago
- **Notas del pedido**: Los detalles del pago se registran en las notas del pedido
- **Inventario**: El stock se reserva durante el proceso de pago

### Seguridad {#security}

- **Enlaces únicos**: Cada sesión de pago tiene una URL única con tiempo limitado
- **SSL obligatorio**: Todo el procesamiento de pagos se realiza a través de conexiones HTTPS seguras
- **Cumplimiento PCI**: Aprovecha las pasarelas de pago web compatibles con PCI ya existentes
- **Gestión de sesiones**: El manejo seguro de sesiones previene el acceso no autorizado

### Métodos de pago {#payment-methods}

La pasarela proporciona acceso a todos los métodos de pago de WooCommerce configurados:

- **Tarjetas de crédito/débito**: Stripe, Square, Authorize.net, etc.
- **Billeteras digitales**: PayPal, Apple Pay, Google Pay
- **Transferencias bancarias**: Métodos de pago por transferencia bancaria directa
- **Compre ahora, pague después**: Klarna, Afterpay, Sezzle
- **Criptomonedas**: Bitcoin y otras pasarelas de pago con criptomonedas
- **Métodos locales**: Métodos de pago específicos de cada región

## Solución de problemas {#troubleshooting}

### Problemas comunes {#common-issues}

<Accordion>
  <AccordionItem question="El enlace de pago no funciona">

  - Verificar que el certificado SSL esté correctamente instalado
  - Comprobar que los enlaces permanentes de WooCommerce estén configurados correctamente
  - Asegurarse de que las pasarelas de pago web estén activas y configuradas
  - Probar con un navegador o dispositivo diferente

  </AccordionItem>

  <AccordionItem question="El pago no se procesa">

  - Confirmar que las pasarelas de pago web estén correctamente configuradas
  - Revisar los registros de la pasarela de pago en busca de mensajes de error
  - Verificar que el método de pago del cliente sea compatible
  - Probar el proceso de pago web de forma independiente

  </AccordionItem>

  <AccordionItem question="El pedido no se actualiza en el POS">

  - Comprobar la estabilidad de la conexión a internet
  - Verificar que las tareas cron de WordPress se estén ejecutando correctamente
  - Actualizar la pantalla de pedidos del POS
  - Comprobar el estado del pedido en el administrador de WooCommerce

  </AccordionItem>

  <AccordionItem question="El cliente no puede acceder al proceso de pago">

  - Asegurarse de que el enlace de pago no haya expirado
  - Verificar que el dispositivo del cliente tenga conexión a internet
  - Verificar que el pedido no haya sido cancelado o completado
  - Probar el enlace en un dispositivo diferente

  </AccordionItem>
</Accordion>

### Optimización del rendimiento {#performance-optimisation}

Para un rendimiento óptimo:

- **Caché**: Configurar los plugins de caché para excluir las páginas de pago
- **CDN**: Asegurarse de que la configuración del CDN no interfiera con el proceso de pago
- **Base de datos**: Optimizar la base de datos para un procesamiento de pedidos más rápido
- **Alojamiento**: Utilizar un alojamiento confiable con buena disponibilidad
