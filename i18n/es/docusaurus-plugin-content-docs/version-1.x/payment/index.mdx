WCPOS admite múltiples métodos de pago para satisfacer diferentes necesidades comerciales. El checkout utiliza un iframe/webview para cargar la página WooCommerce Order Pay, lo que significa que cualquier pasarela de pago que funcione con WooCommerce puede funcionar en el POS.

## Métodos de Pago Predeterminados

La versión gratuita incluye dos pasarelas de pago integradas:

### Efectivo

- Ingrese el monto entregado
- Cálculo automático de cambio
- No se requiere configuración adicional

### Tarjeta

- Para usar con terminales de tarjeta externas
- Simplemente marque el pago como completado después de procesar en su terminal
- No se requiere integración directa

## Pasarelas de pago adicionales (Pro)

:::info Pro Feature
Las pasarelas de pago adicionales requieren [WCPOS Pro](/getting-started/pro-license).
:::

Con Pro, puede habilitar pasarelas de pago adicionales de WooCommerce en el checkout de POS:

- **Stripe Terminal** - Integración directa con lectores de tarjetas Stripe
- **SumUp Terminal** - Integración con lectores de tarjetas SumUp
- **Any WooCommerce Gateway** - Habilite cualquier pasarela instalada en su tienda

## Habilitación y deshabilitación de pasarelas

Las pasarelas de pago se administran en el panel de administración de WordPress:

1. Ir a `WP Admin > POS > Settings > Checkout`
2. Verá una lista de todas las puertas de pago WooCommerce instaladas
3. Active o desactive cada puerta de pago para el POS independientemente de su tienda en línea
4. Arrastre para reordenar — la primera puerta de pago habilitada se convierte en la predeterminada al pagar

Esto significa que una puerta de pago puede estar activa en su sitio web pero deshabilitada en el POS, o viceversa. Consulte [Checkout Settings](/settings/wp-admin/checkout) para obtener todos los detalles.

## Prueba de pagos

Antes de activar una nueva puerta de pago:

1. **Cash gateway** — siempre disponible para probar el flujo de pago sin transacciones reales.
2. **Stripe Terminal** — Stripe proporciona un [lector simulado](https://docs.stripe.com/terminal/testing) para el modo de prueba. Primero, habilite el modo de prueba en su panel de Stripe.
3. **Third-party gateways** — consulte la documentación propia de la puerta de pago para obtener instrucciones sobre el modo de prueba/sandbox. La mayoría de las puertas de pago WooCommerce admiten una opción de modo de prueba.

:::tip
Si una puerta de pago recién habilitada no aparece al pagar, intente actualizar el POS. Los cambios de puerta de pago en WP Admin surten efecto en la próxima carga de página.
:::

## Problemas Comunes

### La Pasarela No Aparece en el Checkout

- Confirma que la pasarela está habilitada para POS en `WP Admin > POS > Settings > Checkout`.
- Verifica que la pasarela también esté habilitada en `WP Admin > WooCommerce > Settings > Payments` — una pasarela debe estar activa en WooCommerce antes de que pueda aparecer en el POS.
- Actualiza el POS después de hacer cambios.

### La Pasarela Se Muestra Incorrectamente

El iframe de checkout carga los estilos del tema de tu sitio, lo que a veces puede interferir. Utiliza el botón **Checkout Settings** en la ventana modal de checkout para desabilitar selectivamente estilos o scripts. Consulta [Checkout Troubleshooting](/pos/checkout#checkout-settings-troubleshooting) para más detalles.

### El Procesamiento de Pagos Falla

- Verifica los registros de error de tu sitio (`WP Admin > POS > Support > Logs`) para errores específicos de la pasarela.
- Asegúrate de que tu certificado SSL sea válido — la mayoría de las pasarelas de pago requieren HTTPS.
- Si utilizas un entorno de staging o local, confirma que la pasarela admita modo de prueba/sandbox.
