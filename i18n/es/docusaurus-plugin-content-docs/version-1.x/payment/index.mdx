WCPOS admite múltiples métodos de pago para satisfacer diferentes necesidades comerciales. El checkout utiliza un iframe/webview para cargar la página WooCommerce Order Pay, lo que significa que cualquier puerta de enlace de pago que funcione con WooCommerce puede funcionar en el POS.

## Métodos de Pago Predeterminados

La versión gratuita incluye dos puertas de enlace de pago integradas:

### Efectivo

- Ingrese la cantidad entregada
- Cálculo automático de cambio
- No se requiere configuración adicional

### Tarjeta

- Para usar con terminales de tarjeta externos
- Simplemente marca el pago como completo después de procesar en tu terminal
- No se requiere integración directa

## Pasarelas de Pago Adicionales (Pro)

:::info Característica Pro
Las pasarelas de pago adicionales requieren [WCPOS Pro](/getting-started/pro-license).
:::

Con Pro, puedes habilitar pasarelas de pago adicionales de WooCommerce en el checkout del POS:

- **Stripe Terminal** - Integración directa con lectores de tarjetas Stripe
- **SumUp Terminal** - Integración con lectores de tarjetas SumUp
- **Any WooCommerce Gateway** - Habilita cualquier pasarela instalada en tu tienda

## Habilitación e Inhabilitación de Pasarelas de Pago

Las pasarelas de pago se gestionan en el panel de administración de WordPress:

1. Ve a `WP Admin > POS > Settings > Checkout`
2. Verás una lista de todas las pasarelas de pago de WooCommerce instaladas
3. Activa o desactiva cada pasarela de pago para el POS independientemente de tu tienda en línea
4. Arrastra para reordenar — la primera pasarela de pago habilitada se convierte en la predeterminada en el pago

Esto significa que una pasarela de pago puede estar activa en tu sitio web pero deshabilitada en el POS, o viceversa. Consulta [Configuración de Pago](/settings/wp-admin/checkout) para obtener todos los detalles.

## Prueba de Pagos

Antes de poner en línea una nueva pasarela de pago:

1. **Pasarela de pago de Efectivo** — siempre disponible para probar el flujo de pago sin transacciones reales.
2. **Stripe Terminal** — Stripe proporciona un [lector simulado](https://docs.stripe.com/terminal/testing) para el modo de prueba. Primero habilita el modo de prueba en tu panel de Stripe.
3. **Puertas de enlace de terceros** — consulta la documentación propia de la puerta de enlace para obtener instrucciones del modo sandbox/prueba. La mayoría de las puertas de enlace de WooCommerce admiten un alternador de modo de prueba.

:::tip
Si una puerta de enlace recién habilitada no aparece en el checkout, intenta actualizar el POS. Los cambios de puerta de enlace en WP Admin surten efecto en la próxima carga de página.
:::

## Problemas Comunes

### La puerta de enlace no aparece en el checkout

- Confirma que la puerta de enlace está habilitada para POS en `WP Admin > POS > Settings > Checkout`.
- Verifica que la puerta de enlace también esté habilitada en `WP Admin > WooCommerce > Settings > Payments` — una pasarela debe estar activa en WooCommerce antes de que pueda aparecer en el POS.
- Actualiza el POS después de hacer cambios.

### La pasarela se muestra incorrectamente

El iframe de pago carga los estilos de tema de tu sitio, que a veces pueden interferir. Utiliza el botón **Configuración de pago** en el modal de pago para desactivar de forma selectiva estilos o scripts. Consulta [Solución de problemas de pago](/pos/checkout#checkout-settings-troubleshooting) para obtener más detalles.

### El procesamiento del pago falla

- Comprueba los registros de errores de tu sitio (`WP Admin > POS > Support > Logs`) para obtener errores específicos de la pasarela.
- Asegúrate de que tu certificado SSL sea válido — la mayoría de las pasarelas de pago requieren HTTPS.
- Si utilizas un entorno de prueba o local, confirma que la pasarela admite el modo de prueba/sandbox.
