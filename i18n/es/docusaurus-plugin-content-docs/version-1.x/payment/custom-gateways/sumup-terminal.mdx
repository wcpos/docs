# Pasarela de Terminal SumUp

La pasarela de Terminal SumUp le permite aceptar pagos con tarjeta utilizando lectores de tarjetas SumUp directamente dentro de WooCommerce POS. Esta pasarela proporciona una integración perfecta con el sistema de procesamiento de pagos de SumUp para transacciones en persona.

## Características

- **Integración de Hardware**: Conecte lectores de tarjetas SumUp a su sistema POS a través de conexión a internet
- **Procesamiento en Tiempo Real**: Procesamiento de pagos instantáneo y finalización de pedidos
- **Detección Automática**: Los pedidos se completan automáticamente cuando se detecta el pago
- **Transacciones Seguras**: Procesamiento de pagos compatible con PCI a través de SumUp
- **Emparejamiento Sencillo**: Proceso de emparejamiento de dispositivos simple con códigos de emparejamiento
- **SDK Web**: Utiliza el SDK de JavaScript de SumUp para compatibilidad con aplicaciones web

## Instalación

### Paso 1: Descargar e Instalar

1. Visite la [página de lanzamientos de SumUp Terminal para WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Descargue el último archivo **sumup-terminal-for-woocommerce.zip**
3. En su administrador de WordPress, vaya a `Plugins > Añadir Nuevo > Subir Plugin`
4. Suba el archivo zip y active el plugin

### Paso 2: Configurar Ajustes de SumUp

1. Navegue a `Admin de WP > WooCommerce > Ajustes > Pagos`
2. Encuentre **Terminal SumUp** en la lista de métodos de pago
3. Haga clic en **Terminal SumUp** para acceder a los ajustes
4. Introduzca su **Clave API de SumUp** (disponible en su panel de comerciante de SumUp)
5. Guarde los ajustes

:::note
No es necesario habilitar la pasarela de Terminal SumUp en los ajustes de WooCommerce. Se habilitará específicamente para el POS en un paso posterior.
:::

### Paso 3: Emparejar su Terminal SumUp

1. En la misma página de ajustes, ubique la sección **Emparejar Lector**
2. En su dispositivo SumUp, navegue a la pantalla de emparejamiento para mostrar el código de emparejamiento
3. Introduzca el código de emparejamiento mostrado en su dispositivo SumUp
4. Haga clic en **"Emparejar Lector"** para establecer la conexión
5. Espere la confirmación de que el lector se ha emparejado correctamente

:::warning Importante
El lector debe estar emparejado correctamente antes de poder procesar pagos. Asegúrese de completar el proceso de emparejamiento antes de continuar.
:::

### Paso 4: Habilitar en WooCommerce POS

1. Vaya a `Admin de WP > WooCommerce POS > Ajustes > Pago`
2. Encuentre la pasarela de **Terminal SumUp** en la lista
3. Habilite la pasarela para su uso en el POS
4. Guarde sus ajustes

## Uso

### Procesamiento de Pagos

1. **Añadir Artículos**: Añada productos a su carrito en el POS
2. **Seleccionar Pasarela**: Elija "Terminal SumUp" como método de pago
3. **Iniciar Pago**: Haga clic para iniciar un nuevo pago en su dispositivo SumUp
4. **Pago del Cliente**: El cliente completa el pago en el terminal SumUp
5. **Finalización Automática**: El pedido se completa automáticamente cuando se detecta el pago

### Controles de Pago

Al usar la pasarela de Terminal SumUp, tendrá las siguientes opciones:

- **Iniciar Nuevo Pago**: Inicie una solicitud de pago en el terminal conectado
- **Cancelar Pago**: Cancele un pago en proceso
- **Estado de Pago**: Monitoree el estado actual del proceso de pago

### Gestión de Pedidos

- **Actualizaciones Automáticas**: Los pedidos se marcan automáticamente como completados tras un pago exitoso
- **Seguimiento de Pagos**: Todos los detalles de pago se registran en las notas del pedido
- **Generación de Recibos**: Se generan recibos estándar de POS después de pagos exitosos

## Requisitos

- **Cuenta SumUp**: Cuenta de comerciante SumUp activa
- **Acceso API**: Clave API de SumUp desde su panel de comerciante
- **Hardware Compatible**: Dispositivo lector de tarjetas SumUp
- **WooCommerce POS**: Versión Gratuita o Pro instalada y activada
- **Conexión Estable**: Conexión a internet confiable para comunicación API

## Compatibilidad de Hardware

:::important Requisitos de Conectividad
Esta implementación utiliza el SDK de JavaScript de SumUp, lo que significa que funciona a través de aplicaciones web pero requiere **terminales conectados a internet únicamente**. Los terminales Bluetooth no son compatibles actualmente.
:::

### Terminales Compatibles (Conectados a Internet)

- **SumUp Solo** ✅ - Terminal conectado por WiFi/Ethernet

### Terminales No Compatibles (Bluetooth)

- **SumUp Air** ❌ - Solo Bluetooth
- **SumUp 3G** ❌ - Solo Bluetooth

:::note Soporte Futuro
La compatibilidad con terminales Bluetooth se agregará a las aplicaciones iOS y Android en el **Q1 2026**. Esto permitirá la compatibilidad con todos los terminales certificados por SumUp, incluidos los modelos Air y 3G.
:::

## Solución de Problemas

### Problemas Comunes

**El Lector No Se Empareja**
- Asegúrese de usar un terminal conectado a internet compatible (Solo únicamente)
- Verifique que su dispositivo SumUp Solo esté conectado a WiFi y en línea
- Compruebe que el dispositivo esté en modo de emparejamiento y que el código de emparejamiento se haya introducido correctamente
- Confirme que su cuenta SumUp tenga el acceso API habilitado
- Asegure una conexión a internet estable durante el emparejamiento

**Pago No Procesándose**
- Verifique que el lector esté emparejado correctamente
- Compruebe que su cuenta SumUp esté activa y en buen estado
- Asegúrese de que el dispositivo tenga batería y conectividad suficientes
- Confirme que la clave API sea correcta y esté activa

**Pedido No Completándose**
- Verifique la estabilidad de la conexión a internet
- Compruebe la configuración de webhooks en su cuenta SumUp
- Asegúrese de que el pago haya sido exitoso en el terminal SumUp
- Revise los registros de errores de WordPress para problemas de comunicación API

### Límites de la API

La pasarela está optimizada para reducir las llamadas API y evitar la limitación de tasa:
- El estado del pago se verifica de manera eficiente
- Se minimizan las solicitudes API innecesarias
- La lógica de reintento automático maneja fallos temporales
