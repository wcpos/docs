# Puerta de enlace de Terminal SumUp

La puerta de enlace de Terminal SumUp permite aceptar pagos con tarjeta utilizando lectores de tarjetas SumUp directamente dentro de WooCommerce POS. Esta puerta de enlace proporciona una integración fluida con el sistema de procesamiento de pagos de SumUp para transacciones en persona.

## Características

- **Integración de Hardware**: Conecte los lectores de tarjetas SumUp a su sistema POS a través de una conexión a internet
- **Procesamiento en Tiempo Real**: Procesamiento de pagos instantáneo y finalización de pedidos
- **Detección Automática**: Los pedidos se completan automáticamente cuando se detecta el pago
- **Transacciones Seguras**: Procesamiento de pagos conforme a PCI a través de SumUp
- **Emparejamiento Fácil**: Proceso de emparejamiento simple de dispositivos con códigos de emparejamiento
- **SDK Basado en la Web**: Utiliza el SDK de JavaScript de SumUp para compatibilidad con aplicaciones web

## Instalación

### Paso 1: Descargar e Instalar

1. Visite la [página de lanzamientos de Terminal SumUp para WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Descargue el último archivo **sumup-terminal-for-woocommerce.zip**
3. En su administrador de WordPress, vaya a `Plugins > Añadir Nuevo > Subir Plugin`
4. Suba el archivo zip y active el plugin

### Paso 2: Configurar la Configuración de SumUp

1. Navegue a `WP Admin > WooCommerce > Configuración > Pagos`
2. Encuentre **Terminal SumUp** en la lista de métodos de pago
3. Haga clic en **Terminal SumUp** para acceder a la configuración
4. Ingrese su **Clave API de SumUp** (disponible en su panel de control de comerciante de SumUp)
5. Guarde la configuración

:::note
No es necesario habilitar la puerta de enlace de Terminal SumUp en la configuración de WooCommerce. Se habilitará específicamente para el POS en un paso posterior.
:::

### Paso 3: Emparejar Su Terminal SumUp

1. En la misma página de configuración, localice la sección **Emparejar Lector**
2. En su dispositivo SumUp, navegue a la pantalla de emparejamiento para mostrar el código de emparejamiento
3. Ingrese el código de emparejamiento mostrado en su dispositivo SumUp
4. Haga clic en **"Emparejar Lector"** para establecer la conexión
5. Espere la confirmación de que el lector ha sido emparejado con éxito

:::warning Importante
El lector debe estar emparejado con éxito antes de poder procesar pagos. Asegúrese de que el proceso de emparejamiento se complete antes de continuar.
:::

### Paso 4: Habilitar en WooCommerce POS

1. Vaya a `WP Admin > WooCommerce POS > Configuración > Pago`
2. Encuentre la puerta de enlace **Terminal SumUp** en la lista
3. Habilite la puerta de enlace para usarla en el POS
4. Guarde su configuración

## Uso

### Procesamiento de Pagos

1. **Agregar Artículos**: Agregue productos a su carrito en el POS
2. **Seleccionar Puerta de Enlace**: Elija "Terminal SumUp" como método de pago
3. **Iniciar Pago**: Haga clic para iniciar un nuevo pago en su dispositivo SumUp
4. **Pago del Cliente**: El cliente completa el pago en el terminal SumUp
5. **Finalización Automática**: El pedido se completa automáticamente cuando se detecta el pago

### Controles de Pago

Al utilizar la puerta de enlace de Terminal SumUp, tiene las siguientes opciones:

- **Iniciar Nuevo Pago**: Iniciar una solicitud de pago en el terminal conectado
- **Cancelar Pago**: Cancelar un pago que se está procesando actualmente
- **Estado del Pago**: Monitorear el estado actual del proceso de pago

### Gestión de Pedidos

- **Actualizaciones Automáticas**: Los pedidos se marcan automáticamente como completados tras un pago exitoso
- **Seguimiento de Pagos**: Todos los detalles del pago se registran en las notas del pedido
- **Generación de Recibos**: Los recibos estándar del POS se generan después de pagos exitosos

## Requisitos

- **Cuenta de SumUp**: Cuenta de comerciante de SumUp activa
- **Acceso a la API**: Clave API de SumUp desde su panel de comerciante
- **Hardware Compatible**: Dispositivo lector de tarjetas SumUp
- **WooCommerce POS**: Versión gratuita o Pro instalada y activada
- **Conexión Estable**: Conexión a internet confiable para la comunicación de la API

## Compatibilidad de Hardware

:::important Requisitos de Conectividad
Esta implementación utiliza el SDK de JavaScript de SumUp, lo que significa que funciona a través de aplicaciones web pero requiere **terminales conectadas a internet únicamente**. Actualmente no se admiten terminales Bluetooth.
:::

### Terminales Soportados (Conectados a Internet)

- **SumUp Solo** ✅ - Terminal conectada a WiFi/Ethernet

### Terminales No Soportados (Bluetooth)

- **SumUp Air** ❌ - Solo Bluetooth
- **SumUp 3G** ❌ - Solo Bluetooth

:::note Soporte Futuro
El soporte para terminales Bluetooth se añadirá a las aplicaciones de iOS y Android en **Q1 2026**. Esto habilitará soporte para todos los terminales certificados por SumUp, incluyendo los modelos Air y 3G.
:::

## Solución de Problemas

### Problemas Comunes

**El Lector No se Empareja**
- Asegúrese de que está utilizando un terminal conectado a internet soportado (solo Solo)
- Verifique que su dispositivo SumUp Solo esté conectado a WiFi y en línea
- Asegúrese de que el dispositivo esté en modo de emparejamiento y que el código de emparejamiento se haya ingresado correctamente
- Confirme que su cuenta de SumUp tenga habilitado el acceso a la API
- Asegúrese de que la conexión a internet sea estable durante el emparejamiento

**El Pago No se Procesa**
- Verifique que el lector esté emparejado correctamente
- Asegúrese de que su cuenta de SumUp esté activa y en buen estado
- Asegúrese de que el dispositivo tenga suficiente batería y conectividad
- Confirme que la clave API sea correcta y esté activa

**El Pedido No se Completa**
- Verifique la estabilidad de la conexión a internet
- Verifique la configuración del webhook en su cuenta de SumUp
- Asegúrese de que el pago haya sido exitoso en el terminal SumUp
- Verifique los registros de errores de WordPress para problemas de comunicación de la API

### Límites de Tasa de la API

La puerta de enlace está optimizada para reducir las llamadas a la API y evitar límites de tasa:
- El estado del pago se verifica de manera eficiente
- Se minimizan las solicitudes innecesarias a la API
- La lógica de reintento automático maneja fallos temporales
