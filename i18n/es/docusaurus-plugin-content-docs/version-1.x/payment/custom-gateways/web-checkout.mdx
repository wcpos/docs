# Pasarela de Pago por Web

La pasarela de Pago por Web permite a los clientes completar sus transacciones de POS utilizando el sistema de pago de su tienda web. Esta pasarela crea un puente sin interrupciones entre su POS en la tienda y los métodos de pago en línea, perfecto para situaciones donde los clientes prefieren pagar utilizando sus propios dispositivos o cuando necesita acceso a métodos de pago solo en línea.

## Características

- **Integración Sin Costuras**: Enlace directo del POS al pago web
- **Todos los Métodos de Pago**: Acceso a todas las pasarelas de pago habilitadas en su tienda web
- **Control del Cliente**: Los clientes pueden pagar usando sus propios dispositivos y métodos preferidos
- **Sincronización de Pedidos**: Actualizaciones automáticas de pedidos entre el POS y la tienda web
- **Flujo de Trabajo Flexible**: Completa transacciones en línea mientras mantienes el flujo de trabajo del POS

## Instalación

### Paso 1: Descargar e Instalar

1. Visita la [página de lanzamientos de la Pasarela de Pago por Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Descarga el archivo **woocommerce-pos-web-checkout-gateway.zip** más reciente
3. En tu administración de WordPress, ve a `Plugins > Agregar Nuevo > Subir Plugin`
4. Sube el archivo zip y activa el plugin

### Paso 2: Habilitar en WooCommerce POS

1. Ve a `WP Admin > WooCommerce POS > Configuración > Pago`
2. Busca la **Pasarela de Pago por Web de WooCommerce POS** en la lista
3. Habilita la pasarela para usar en el POS
4. Guarda tus configuraciones

:::note
Esta pasarela aprovecha los métodos de pago de WooCommerce que ya tienes, así que asegúrate de tener al menos una pasarela de pago web configurada correctamente (Stripe, PayPal, etc.).
:::

## Uso

### Iniciando el Pago Web

1. **Agregar Artículos**: Agrega productos a tu carrito en el POS
2. **Seleccionar Pasarela**: Elige "Pago Web" como método de pago
3. **Generar Enlace**: El sistema crea un enlace de pago único para el pedido
4. **Acceso del Cliente**: El cliente hace clic en el enlace para acceder al pago web
5. **Pago en Línea**: El cliente completa el pago utilizando los métodos de pago de la tienda web

### Proceso de Pago

La pasarela de Pago por Web sigue este flujo de trabajo:

1. **Creación de Pedido**: El POS crea un pedido pendiente en WooCommerce
2. **Enlace de Pago**: El sistema genera un enlace de pago seguro y único
3. **Redirección del Cliente**: El cliente es dirigido a la página de pago web
4. **Selección de Pago**: El cliente elige entre los métodos de pago en línea disponibles
5. **Procesamiento de Pago**: El proceso de pago estándar de WooCommerce maneja el pago
6. **Finalización del Pedido**: El estado del pedido se actualiza automáticamente tras el pago exitoso

### Flujo de Trabajo del POS

Después de iniciar el pago web:

1. **Monitorear Estado**: Mantén la pantalla de pedidos del POS abierta para monitorear el estado del pago
2. **Pago del Cliente**: El cliente completa el pago en su dispositivo o en tu tableta
3. **Actualización Automática**: El estado del pedido se actualiza en tiempo real cuando se recibe el pago
4. **Procesar Pago**: Haz clic en el botón "Procesar Pago" en el POS para continuar con el recibo
5. **Generación de Recibo**: Genera e imprime el recibo como de costumbre

## Casos de Uso

### Perfecto Para

- **Preferencia del Cliente**: Clientes que prefieren usar sus propias aplicaciones de pago o tarjetas
- **Pagos Complejos**: Transacciones que requieren métodos de pago no disponibles en el POS
- **Pagos Divididos**: Clientes que desean usar múltiples métodos de pago
- **Programas de Lealtad**: Acceso a la redención de puntos de lealtad en línea
- **Tarjetas Regalo**: Clientes con tarjetas regalo digitales o crédito de tienda
- **Tarjetas Internacionales**: Métodos de pago que funcionan mejor a través de pasarelas web

### Ejemplos de Flujo de Trabajo

**Tienda Minorista**
1. El cliente compra en la tienda y lleva los artículos a la caja
2. El cliente prefiere pagar con su aplicación de billetera móvil
3. El personal selecciona la pasarela de Pago por Web
4. El cliente escanea el código QR o hace clic en el enlace en su teléfono
5. El cliente completa el pago utilizando el método preferido
6. El personal procesa el recibo y completa la transacción

**Negocio de Servicios**
1. Completa el servicio y añade cargos al POS
2. El cliente quiere pagar con tarjeta de crédito empresarial
3. Genera el enlace de pago web
4. El cliente ingresa los detalles de pago en un formulario web seguro
5. El pago se procesa a través de tu pasarela web
6. El pedido se completa y se genera el recibo

## Detalles Técnicos

### Gestión de Pedidos

- **Pedidos Pendientes**: Los pedidos se crean con el estado "pendiente de pago"
- **Actualizaciones de Estado**: Cambios automáticos de estado cuando se recibe el pago
- **Notas del Pedido**: Los detalles del pago se registran en las notas del pedido
- **Inventario**: El stock se reserva durante el proceso de pago

### Seguridad

- **Enlaces Únicos**: Cada sesión de pago tiene una URL única y limitada en el tiempo
- **SSL Requerido**: Todo el procesamiento de pagos se realiza a través de conexiones HTTPS seguras
- **Cumplimiento PCI**: Aprovecha tus pasarelas de pago web compatibles con PCI existentes
- **Gestión de Sesiones**: El manejo seguro de sesiones previene el acceso no autorizado

### Métodos de Pago

La pasarela proporciona acceso a todos tus métodos de pago de WooCommerce configurados:

- **Tarjetas de Crédito/Débito**: Stripe, Square, Authorize.net, etc.
- **Billeteras Digitales**: PayPal, Apple Pay, Google Pay
- **Transferencias Bancarias**: Métodos de pago bancarios directos
- **Compra Ahora, Paga Después**: Klarna, Afterpay, Sezzle
- **Criptomonedas**: Bitcoin y otras pasarelas de pago en criptomonedas
- **Métodos Locales**: Métodos de pago específicos de la región

## Requisitos

- **WooCommerce POS**: Versión gratuita o Pro instalada y activada
- **Pasarelas de Pago Web**: Al menos un método de pago en línea configurado en WooCommerce
- **Certificado SSL**: HTTPS requerido para el procesamiento seguro de pagos
- **Navegador Moderno**: El dispositivo del cliente debe soportar estándares web modernos

## Solución de Problemas

### Problemas Comunes

**Enlace de Pago No Funciona**
- Verifica que el certificado SSL esté instalado correctamente
- Comprueba que los enlaces permanentes de WooCommerce estén configurados correctamente
- Asegúrate de que las pasarelas de pago web estén activas y configuradas
- Prueba con un navegador o dispositivo diferente

**Pago No Procesando**
- Confirma que las pasarelas de pago web estén configuradas correctamente
- Verifica los registros de la pasarela de pago para mensajes de error
- Verifica que el método de pago del cliente sea compatible
- Prueba el proceso de pago web de forma independiente

**El Pedido No Se Actualiza en el POS**
- Comprueba la estabilidad de la conexión a internet
- Verifica que los trabajos cron de WordPress se estén ejecutando correctamente
- Actualiza la pantalla de pedidos del POS
- Verifica el estado del pedido en la administración de WooCommerce

**El Cliente No Puede Acceder al Pago**
- Asegúrate de que el enlace de pago no haya caducado
- Verifica que el dispositivo del cliente tenga conectividad a internet
- Comprueba que el pedido no haya sido cancelado o completado
- Prueba el enlace en un dispositivo diferente

### Optimización del Rendimiento

Para un rendimiento óptimo:

- **Caché**: Configura plugins de caché para excluir páginas de pago
- **CDN**: Asegúrate de que la configuración de la CDN no interfiera con el proceso de pago
- **Base de Datos**: Optimiza la base de datos para un procesamiento de pedidos más rápido
- **Alojamiento**: Utiliza un alojamiento confiable con buen tiempo de actividad
