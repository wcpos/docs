# Puerta de Pago Web {#web-checkout-gateway}

La puerta de pago web permite a los clientes completar sus transacciones de POS utilizando el sistema de pago de su tienda web. Esta puerta crea un puente sin fisuras entre su POS en la tienda y los métodos de pago en línea, ideal para situaciones donde los clientes prefieren pagar utilizando sus propios dispositivos o cuando necesita acceso a métodos de pago solo en la web.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e Instalar">

1. Visite la [página de lanzamientos de la Puerta de Pago Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Descargue el archivo más reciente **woocommerce-pos-web-checkout-gateway.zip**
3. En su administración de WordPress, vaya a `Plugins > Añadir Nuevo > Subir Plugin`
4. Suba el archivo zip y active el plugin

  </StepCard>
  <StepCard number="2" title="Habilitar en WooCommerce POS" isLast>

1. Vaya a `WP Admin > WooCommerce POS > Configuración > Pago`
2. Busque la **Puerta de Pago Web de WooCommerce POS** en la lista
3. Habilite la puerta para su uso en el POS
4. Guarde su configuración

:::note
Esta puerta utiliza sus métodos de pago existentes de WooCommerce, así que asegúrese de tener al menos un método de pago web configurado correctamente (Stripe, PayPal, etc.).
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Iniciando el Pago Web {#initiating-web-checkout}

1. **Agregar Artículos**: Agregue productos a su carrito en el POS
2. **Seleccionar Puerta**: Elija "Pago Web" como método de pago
3. **Generar Enlace**: El sistema crea un enlace de pago único para el pedido
4. **Acceso del Cliente**: El cliente hace clic en el enlace para acceder al pago web
5. **Pago en Línea**: El cliente completa el pago utilizando los métodos de pago de la tienda web

### Proceso de Pago {#payment-process}

La puerta de pago web sigue este flujo de trabajo:

1. **Creación de Pedido**: El POS crea un pedido pendiente en WooCommerce
2. **Enlace de Pago**: El sistema genera un enlace de pago seguro y único
3. **Redirección del Cliente**: El cliente es dirigido a la página de pago web
4. **Selección de Pago**: El cliente elige entre los métodos de pago disponibles en la web
5. **Procesamiento de Pago**: El proceso de pago estándar de WooCommerce maneja el pago
6. **Finalización de Pedido**: El estado del pedido se actualiza automáticamente tras el pago exitoso

### Flujo de Trabajo del POS {#pos-workflow}

Después de iniciar el pago web:

1. **Monitorear Estado**: Mantenga la pantalla de pedidos del POS abierta para monitorear el estado del pago
2. **Pago del Cliente**: El cliente completa el pago en su dispositivo o su tableta
3. **Actualización Automática**: El estado del pedido se actualiza en tiempo real cuando se recibe el pago
4. **Procesar Pago**: Haga clic en el botón "Procesar Pago" en el POS para continuar con el recibo
5. **Generación de Recibo**: Genere e imprima el recibo como de costumbre

## Casos de Uso {#use-cases}

### Perfecto Para {#perfect-for}

- **Preferencia del Cliente**: Clientes que prefieren utilizar sus propias aplicaciones o tarjetas de pago
- **Pagos Complejos**: Transacciones que requieren métodos de pago no disponibles en el POS

- **Tarjetas de Regalo**: Clientes con tarjetas de regalo digitales o crédito en la tienda

## Detalles Técnicos {#technical-details}

### Gestión de Pedidos {#order-management}

- **Pedidos Pendientes**: Los pedidos se crean con el estado "pendiente de pago"
- **Actualizaciones de Estado**: Cambios automáticos de estado cuando se recibe el pago
- **Notas del Pedido**: Los detalles del pago se registran en las notas del pedido
- **Inventario**: El stock se reserva durante el proceso de pago

### Seguridad {#security}

- **Enlaces Únicos**: Cada sesión de pago tiene un URL único y limitado por tiempo
- **SSL Requerido**: Todo el procesamiento de pagos ocurre a través de conexiones HTTPS seguras
- **Cumplimiento PCI**: Utiliza sus puertas de pago web existentes que cumplen con PCI
- **Gestión de Sesiones**: El manejo seguro de sesiones previene el acceso no autorizado

### Métodos de Pago {#payment-methods}

La puerta proporciona acceso a todos sus métodos de pago configurados en WooCommerce:

- **Tarjetas de Crédito/Débito**: Stripe, Square, Authorize.net, etc.
- **Billeteras Digitales**: PayPal, Apple Pay, Google Pay
- **Transferencias Bancarias**: Métodos de pago directos
- **Compra Ahora, Paga Después**: Klarna, Afterpay, Sezzle
- **Criptomonedas**: Bitcoin y otras puertas de pago cripto
- **Métodos Locales**: Métodos de pago específicos de la región

## Resolución de Problemas {#troubleshooting}

### Problemas Comunes {#common-issues}

<Accordion>
  <AccordionItem question="El enlace de pago no funciona">

  - Verifique que el certificado SSL esté instalado correctamente
  - Compruebe que los enlaces permanentes de WooCommerce estén configurados correctamente
  - Asegúrese de que las puertas de pago web estén activas y configuradas
  - Pruebe con un navegador o dispositivo diferente

  </AccordionItem>

  <AccordionItem question="El pago no se procesa">

  - Confirme que las puertas de pago web estén configuradas correctamente
  - Revise los registros de la puerta de pago para obtener mensajes de error
  - Verifique que el método de pago del cliente sea compatible
  - Pruebe el proceso de pago web de forma independiente

  </AccordionItem>

  <AccordionItem question="El pedido no se actualiza en el POS">

  - Revise la estabilidad de la conexión a internet
  - Verifique que las tareas programadas de WordPress estén ejecutándose correctamente
  - Actualice la pantalla de pedidos en el POS
  - Compruebe el estado del pedido en la administración de WooCommerce

  </AccordionItem>

  <AccordionItem question="El cliente no puede acceder al pago">

  - Asegúrese de que el enlace de pago no haya expirado
  - Verifique que el dispositivo del cliente tenga conectividad a internet
  - Compruebe que el pedido no haya sido cancelado o completado
  - Pruebe el enlace en un dispositivo diferente

  </AccordionItem>
</Accordion>

### Optimización del Rendimiento {#performance-optimisation}

Para un rendimiento óptimo:

- **Caché**: Configure los plugins de caché para excluir las páginas de pago
- **CDN**: Asegúrese de que la configuración de CDN no interfiera con el proceso de pago
- **Base de Datos**: Optimice la base de datos para un procesamiento de pedidos más rápido
- **Alojamiento**: Utilice un alojamiento confiable con buen tiempo de actividad
