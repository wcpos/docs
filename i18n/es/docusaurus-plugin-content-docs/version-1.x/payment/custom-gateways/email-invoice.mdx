# Pasarela de Factura por Email {#email-invoice-gateway}

La pasarela de factura por email te permite enviar facturas de pago a los clientes por correo electrónico directamente desde WCPOS. Esto es perfecto para situaciones en las que los clientes prefieren pagar más tarde o necesitan completar el pago en línea usando los métodos de pago de tu tienda web.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e Instalar">

1. Visita la [página de lanzamientos de la Pasarela de Factura por Email](https://github.com/wcpos/email-invoice-gateway/releases)
2. Descarga el último archivo **woocommerce-pos-email-invoice-gateway.zip**
3. En tu administrador de WordPress, ve a `Plugins > Añadir Nuevo > Subir Plugin`
4. Sube el archivo zip y activa el plugin

  </StepCard>
  <StepCard number="2" title="Habilitar en WCPOS" isLast>

1. Ve a `WP Admin > POS > Ajustes > Pago`
2. Busca la **Pasarela de Pago de Factura WCPOS** en la lista
3. Habilita la pasarela para su uso en el TPV
4. Guarda tus ajustes

:::note
Esta pasarela no requiere claves API adicionales ni configuración de servicios externos. Utiliza tu sistema de email de WordPress existente y los métodos de pago de WooCommerce.
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Enviando Facturas {#sending-invoices}

1. **Agregar Artículos**: Agrega productos a tu carrito en el TPV
2. **Seleccionar Pasarela**: Elige "Factura por Email" como método de pago
3. **Dirección de Email**: 
   - Para clientes existentes: El email se completa automáticamente con los datos del cliente
   - Para pedidos de invitados: Ingresa manualmente la dirección de email del cliente
4. **Opcional**: Marca "Guardar en la dirección de facturación" para actualizar la información de facturación del cliente
5. **Enviar Factura**: Completa el proceso para enviar el correo electrónico de la factura

### Experiencia del Cliente {#customer-experience}

Cuando envías una factura, el cliente recibe un email que contiene:

- **Detalles del Pedido**: Desglose completo de artículos, cantidades y precios
- **Total del Pedido**: Monto total a pagar incluyendo impuestos y tarifas
- **Enlace de Pago**: Enlace directo para completar el pago en línea
- **Información del Pedido**: Número de pedido y detalles de referencia

### Finalización del Pago {#payment-completion}

Los clientes pueden completar el pago de dos maneras:

1. **Pago en Línea**: Haz clic en el enlace "Pagar este pedido" en el correo electrónico
2. **Devolución en Tienda**: Regresa para completar el pago utilizando otros métodos de pago del TPV

El enlace "Pagar este pedido" lleva a los clientes a una página de pago dedicada en tu sitio web donde pueden usar cualquiera de tus pasarelas de pago web habilitadas (Stripe, PayPal, etc.).

## Gestión de Emails {#email-management}

### Clientes Existentes {#existing-customers}

- Las direcciones de email se recuperan automáticamente de los perfiles de cliente
- La información de facturación del cliente se conserva
- El historial de pedidos se mantiene bajo la cuenta del cliente

### Clientes Invitados {#guest-customers}

- Ingresa manualmente las direcciones de email para clientes únicos
- Opción para guardar el email en la dirección de facturación para referencia futura
- Los pedidos de invitados se rastrean correctamente en WooCommerce

### Plantillas de Email {#email-templates}

La pasarela utiliza las plantillas de email estándar de WooCommerce:

- **Personalizable**: Modifica la apariencia del email a través de los ajustes de email de WooCommerce
- **De Marca**: Incluye el logotipo y la marca de tu tienda
- **Profesional**: Formato de factura limpio y profesional
- **Optimizada para Móviles**: Diseño responsivo para todos los dispositivos

## Casos de Uso {#use-cases}

### Perfecto Para {#perfect-for}

- **Ventas B2B**: Clientes negocios que necesitan procesar pagos a través de sus sistemas contables
- **Grandes Pedidos**: Transacciones de alto valor que requieren aprobación o tiempo de procesamiento
- **Clientes Remotos**: Clientes que necesiten completar el pago después de salir de la tienda
- **Clientes de Cuenta**: Clientes habituales con términos de pago establecidos
- **Cita a Pedido**: Convertir citas o estimaciones en facturas pagaderas

## Resolución de Problemas {#troubleshooting}

### Problemas Comunes {#common-issues}

<Accordion>
  <AccordionItem question="Emails no se envían">

  - Verifica la configuración de email de WordPress
  - Verifica la configuración de SMTP si usas un servicio de email personalizado
  - Prueba con plugins de prueba de email de WordPress
  - Revisa las carpetas de spam para correos de prueba

  </AccordionItem>

  <AccordionItem question="Enlaces de pago no funcionan">

  - Asegúrate de que WooCommerce esté configurado correctamente
  - Verifica que al menos una pasarela de pago esté habilitada para el pago web
  - Comprueba que el certificado SSL esté correctamente instalado
  - Confirma que el estado del pedido permita el pago

  </AccordionItem>

  <AccordionItem question="El cliente no puede completar el pago">

  - Verifica que las pasarelas de pago web estén activas y configuradas
  - Comprueba que el pedido no haya expirado o sido cancelado
  - Asegúrate de que el cliente tenga límites suficientes en su método de pago
  - Prueba el proceso de pago tú mismo

  </AccordionItem>
</Accordion>

### Entrega de Emails {#email-delivery}

Para una entrega de email confiable, considera:

- **Servicio SMTP**: Usa servicios como SendGrid, Mailgun o Amazon SES
- **Plugins de Email**: Instala plugins SMTP de WordPress para mejor entrega
- **Autenticación de Dominio**: Configura registros SPF, DKIM y DMARC
- **Monitoreo**: Rastrear la entrega de emails y tasas de apertura
