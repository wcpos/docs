# Pasarela de Factura por Email {#email-invoice-gateway}

La pasarela de factura por email te permite enviar facturas de pago a los clientes por correo electrónico directamente desde WooCommerce POS. Esto es perfecto para situaciones en las que los clientes prefieren pagar más tarde o necesitan completar el pago en línea utilizando los métodos de pago de tu tienda web.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e Instalar">

1. Visita la [página de lanzamientos de la Pasarela de Factura por Email](https://github.com/wcpos/email-invoice-gateway/releases)
2. Descarga el archivo **woocommerce-pos-email-invoice-gateway.zip** más reciente
3. En tu administrador de WordPress, ve a `Plugins > Añadir Nuevo > Subir Plugin`
4. Sube el archivo zip y activa el plugin

  </StepCard>
  <StepCard number="2" title="Habilitar en WooCommerce POS" isLast>

1. Ve a `WP Admin > WooCommerce POS > Configuración > Pago`
2. Encuentra la **Pasarela de Pago por Factura de WooCommerce POS** en la lista
3. Habilita la pasarela para su uso en el POS
4. Guarda tus configuraciones

:::note
Esta pasarela no requiere claves API adicionales ni configuración de servicios externos. Utiliza tu sistema de correo electrónico de WordPress existente y los métodos de pago de WooCommerce.
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Envío de Facturas {#sending-invoices}

1. **Agregar Artículos**: Agrega productos a tu carrito en el POS
2. **Seleccionar Pasarela**: Elige "Factura por Email" como método de pago
3. **Dirección de Email**: 
   - Para clientes existentes: El correo electrónico se completa automáticamente a partir de los datos del cliente
   - Para pedidos de invitados: Introduce manualmente la dirección de correo electrónico del cliente
4. **Opcional**: Marca "Guardar en la dirección de facturación" para actualizar la información de facturación del cliente
5. **Enviar Factura**: Completa el proceso para enviar el correo electrónico de factura

### Experiencia del Cliente {#customer-experience}

Cuando envías una factura, el cliente recibe un correo electrónico que contiene:

- **Detalles del Pedido**: Desglose completo de artículos, cantidades y precios
- **Total del Pedido**: Monto final a pagar incluyendo impuestos y tasas
- **Enlace de Pago**: Enlace directo para completar el pago en línea
- **Información del Pedido**: Número de pedido y detalles de referencia

### Finalización del Pago {#payment-completion}

Los clientes pueden completar el pago de dos maneras:

1. **Pago en Línea**: Haz clic en el enlace "Pagar por este pedido" en el correo electrónico
2. **Devolución en Tienda**: Regresa para completar el pago utilizando otros métodos de pago del POS

El enlace "Pagar por este pedido" lleva a los clientes a una página de pago dedicada en tu sitio web donde pueden utilizar cualquiera de tus pasarelas de pago web habilitadas (Stripe, PayPal, etc.).

## Gestión de Emails {#email-management}

### Clientes Existentes {#existing-customers}

- Las direcciones de correo electrónico se recuperan automáticamente de los perfiles de clientes
- La información de facturación del cliente se conserva
- El historial de pedidos se mantiene bajo la cuenta del cliente

### Clientes Invitados {#guest-customers}

- Introduce manualmente direcciones de correo electrónico para clientes ocasionales
- Opción de guardar el correo electrónico en la dirección de facturación para futuras referencias
- Los pedidos de invitados se rastrean correctamente en WooCommerce

### Plantillas de Email {#email-templates}

La pasarela utiliza las plantillas de correo electrónico estándar de WooCommerce:

- **Personalizable**: Modifica la apariencia del correo electrónico a través de la configuración de correo electrónico de WooCommerce
- **Con Marca**: Incluye el logotipo y la marca de tu tienda
- **Profesional**: Formato de factura limpio y profesional
- **Compatible con Dispositivos Móviles**: Diseño receptivo para todos los dispositivos

## Casos de Uso {#use-cases}

### Perfecto Para {#perfect-for}

- **Ventas B2B**: Clientes empresariales que necesitan procesar pagos a través de sus sistemas de contabilidad

## Solución de Problemas {#troubleshooting}

### Problemas Comunes {#common-issues}

<Accordion>
  <AccordionItem question="Emails no enviados">

  - Verifica la configuración del correo electrónico de WordPress
  - Verifica la configuración de SMTP si usas un servicio de correo electrónico personalizado
  - Prueba con plugins de prueba de correo electrónico de WordPress
  - Revisa las carpetas de spam para los correos electrónicos de prueba

  </AccordionItem>

  <AccordionItem question="Enlaces de pago no funcionan">

  - Asegúrate de que WooCommerce esté correctamente configurado
  - Verifica que al menos una pasarela de pago esté habilitada para el pago web
  - Comprueba que el certificado SSL esté correctamente instalado
  - Confirma que el estado del pedido permita el pago

  </AccordionItem>

  <AccordionItem question="El cliente no puede completar el pago">

  - Verifica que las pasarelas de pago web estén activas y configuradas
  - Asegúrate de que el pedido no haya expirado o sido cancelado
  - Asegúrate de que el cliente tenga suficientes límites de método de pago
  - Prueba el proceso de pago tú mismo

  </AccordionItem>
</Accordion>

### Entrega de Email {#email-delivery}

Para una entrega de correo electrónico confiable, considera:

- **Servicio SMTP**: Utiliza servicios como SendGrid, Mailgun o Amazon SES
- **Plugins de Email**: Instala plugins de SMTP de WordPress para una mejor entrega
- **Autenticación de Dominio**: Configura registros SPF, DKIM y DMARC
- **Monitoreo**: Rastrea la entrega de correos electrónicos y las tasas de apertura
