# Email Invoice Gateway {#email-invoice-gateway}

La pasarela de factura por correo electrónico te permite enviar facturas de pago a los clientes directamente desde WooCommerce POS. Esto es perfecto para situaciones en las que los clientes prefieren pagar más tarde o necesitan completar el pago en línea utilizando los métodos de pago de tu tienda web.

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e Instalar">

1. Visita la [página de lanzamientos de la Pasarela de Factura por Email](https://github.com/wcpos/email-invoice-gateway/releases)
2. Descarga el archivo más reciente **woocommerce-pos-email-invoice-gateway.zip**
3. En tu administrador de WordPress, ve a `Plugins > Añadir Nuevo > Subir Plugin`
4. Sube el archivo zip y activa el plugin

  </StepCard>
  <StepCard number="2" title="Habilitar en WooCommerce POS" isLast>

1. Ve a `WP Admin > WooCommerce POS > Settings > Checkout`
2. Busca la **WooCommerce POS Invoice Payment Gateway** en la lista
3. Habilita la pasarela para su uso en el POS
4. Guarda tus configuraciones

:::note
Esta pasarela no requiere claves API adicionales ni configuración de servicios externos. Utiliza tu sistema de correo electrónico de WordPress existente y los métodos de pago de WooCommerce.
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Envío de Facturas {#sending-invoices}

1. **Agregar Artículos**: Agrega productos a tu carrito en el POS
2. **Seleccionar Pasarela**: Elige "Email Invoice" como el método de pago
3. **Dirección de Email**: 
   - Para clientes existentes: El correo electrónico se auto-completa a partir de los datos del cliente
   - Para pedidos de invitados: Introduce la dirección de correo electrónico del cliente manualmente
4. **Opcional**: Marca "Guardar en la dirección de facturación" para actualizar la información de facturación del cliente
5. **Enviar Factura**: Completa el proceso para enviar el correo electrónico de factura

### Experiencia del Cliente {#customer-experience}

Cuando envías una factura, el cliente recibe un correo electrónico que contiene:

- **Detalles del Pedido**: Desglose completo de artículos, cantidades y precios
- **Total del Pedido**: Monto final a pagar incluyendo impuestos y tarifas
- **Enlace de Pago**: Enlace directo para completar el pago en línea
- **Información del Pedido**: Número de pedido y detalles de referencia

### Finalización del Pago {#payment-completion}

Los clientes pueden completar el pago de dos maneras:

1. **Pago en Línea**: Haz clic en el enlace "Pagar por este pedido" en el correo electrónico
2. **Devolución en Tienda**: Regresar para completar el pago utilizando otros métodos de pago del POS

El enlace "Pagar por este pedido" lleva a los clientes a una página de pago dedicada en tu sitio web donde pueden usar cualquiera de tus pasarelas de pago web habilitadas (Stripe, PayPal, etc.).

## Gestión de Emails {#email-management}

### Clientes Existentes {#existing-customers}

- Las direcciones de correo electrónico se recuperan automáticamente de los perfiles de clientes
- Se preserva la información de facturación del cliente
- Se mantiene el historial de pedidos bajo la cuenta del cliente

### Clientes Invitados {#guest-customers}

- Introduce manualmente las direcciones de correo electrónico para clientes ocasionales
- Opción de guardar el correo electrónico en la dirección de facturación para futuras referencias
- Los pedidos de invitados se rastrean correctamente en WooCommerce

### Plantillas de Email {#email-templates}

La pasarela utiliza las plantillas de correo electrónico estándar de WooCommerce:

- **Personalizable**: Modifica la apariencia del correo electrónico a través de la configuración de correo de WooCommerce
- **Con Marca**: Incluye el logotipo y la marca de tu tienda
- **Profesional**: Formato de factura limpio y profesional
- **Adaptable a Móviles**: Diseño responsivo para todos los dispositivos

## Casos de Uso {#use-cases}

### Perfecto Para {#perfect-for}

- **Ventas B2B**: Clientes empresariales que necesitan procesar pagos a través de sus sistemas contables
- **Grandes Pedidos**: Transacciones de alto valor que requieren aprobación o tiempo de procesamiento
- **Clientes Remotos**: Clientes que necesitan completar el pago después de salir de la tienda
- **Clientes de Cuenta**: Clientes regulares con condiciones de pago establecidas
- **Cotización a Pedido**: Convertir cotizaciones o estimaciones en facturas pagaderas

## Solución de Problemas {#troubleshooting}

### Problemas Comunes {#common-issues}

<Accordion>
  <AccordionItem question="Correos electrónicos no enviados">

  - Verifica la configuración de correo de WordPress
  - Verifica los ajustes de SMTP si utilizas un servicio de correo electrónico personalizado
  - Prueba con plugins de prueba de correo de WordPress
  - Revisa las carpetas de spam para correos electrónicos de prueba

  </AccordionItem>

  <AccordionItem question="Enlaces de pago no funcionan">

  - Asegúrate de que WooCommerce esté configurado correctamente
  - Verifica que al menos una pasarela de pago esté habilitada para la compra web
  - Verifica que el certificado SSL esté instalado correctamente
  - Confirma que el estado del pedido permita el pago

  </AccordionItem>

  <AccordionItem question="El cliente no puede completar el pago">

  - Verifica que las pasarelas de pago web estén activas y configuradas
  - Verifica que el pedido no haya expirado o sido cancelado
  - Asegúrate de que el cliente tenga límites de método de pago suficientes
  - Prueba el proceso de pago tú mismo

  </AccordionItem>
</Accordion>

### Entrega de Emails {#email-delivery}

Para una entrega de correo electrónico confiable, considera:

- **Servicio SMTP**: Utiliza servicios como SendGrid, Mailgun o Amazon SES
- **Plugins de Email**: Instala plugins SMTP de WordPress para una mejor entrega
- **Autenticación de Dominio**: Configura registros SPF, DKIM y DMARC
- **Monitoreo**: Realiza un seguimiento de la entrega de correos electrónicos y tasas de apertura
