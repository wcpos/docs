# Gateway de Vipps MobilePay {#vipps-mobilepay-gateway}

El gateway de Vipps MobilePay te permite aceptar pagos basados en teléfonos directamente en WooCommerce POS. Los clientes pueden pagar escaneando un código QR en la pantalla o confirmando una notificación push en su teléfono, sin necesidad de hardware de lector de tarjetas.

## Características {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="Pagos con Código QR">
    Muestra un código QR en el pago para que los clientes lo escaneen y paguen con su teléfono
  </FeatureCard>
  <FeatureCard icon="bolt" title="Notificaciones Push">
    Envía una solicitud de pago directamente al teléfono del cliente ingresando su número
  </FeatureCard>
  <FeatureCard icon="globe" title="Cobertura Nórdica">
    Funciona con Vipps en Noruega, MobilePay en Dinamarca y Finlandia
  </FeatureCard>
  <FeatureCard icon="credit-card" title="Captura Automática">
    Captura fondos inmediatamente después de la autorización, o reserva para captura manual
  </FeatureCard>
</FeatureGrid>

## Instalación {#installation}

<Steps>
  <StepCard number="1" title="Descargar e Instalar">

1. Visita la [página de lanzamientos de WCPOS Vipps MobilePay](https://github.com/wcpos/wcpos-vipps/releases)
2. Descarga el último archivo **wcpos-vipps.zip**
3. En tu administración de WordPress, ve a `Plugins > Añadir Nuevo > Subir Plugin`
4. Sube el archivo zip y activa el plugin

  </StepCard>
  <StepCard number="2" title="Configurar Credenciales de Vipps">

1. Navega a `WP Admin > WooCommerce > Ajustes > Pagos`
2. Encuentra **WCPOS Vipps MobilePay** en la lista de métodos de pago
3. Haz clic en **WCPOS Vipps MobilePay** para acceder a la configuración
4. Ingresa tus credenciales desde el [Portal de Vipps](https://portal.vipps.no/):
   - **Número de Serie del Comerciante (MSN)**
   - **ID de Cliente**
   - **Secreto de Cliente**
   - **Clave de Suscripción**
5. Guarda la configuración

:::note
No necesitas habilitar el gateway aquí para su uso en POS; se habilitará específicamente para el POS en el siguiente paso. Habilitarlo en la configuración de WooCommerce también lo haría disponible en el pago de tu tienda en línea, lo que puede ser útil para pruebas.
:::

:::tip ¿Ya usas el plugin de Vipps?
Si tienes instalado el plugin oficial [Checkout with Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/), tus credenciales se importarán automáticamente cuando actives este plugin. Puedes omitir la entrada manual de credenciales.
:::

  </StepCard>
  <StepCard number="3" title="Habilitar en WooCommerce POS" isLast>

1. Ve a `WP Admin > WooCommerce POS > Ajustes > Pago`
2. Encuentra el gateway **WCPOS Vipps MobilePay** en la lista
3. Habilita el gateway para su uso en el POS
4. Guarda tus configuraciones

  </StepCard>
</Steps>

## Uso {#usage}

### Procesando Pagos — Código QR {#processing-payments-qr-code}

1. **Agregar Artículos**: Agrega productos a tu carrito en el POS
2. **Seleccionar Gateway**: Escoge "Vipps MobilePay" como método de pago
3. **Generar Código QR**: Haz clic en el botón "Generar Código QR"
4. **Cliente Escanea**: El cliente escanea el código QR con su aplicación de Vipps o MobilePay
5. **Cliente Confirma**: El cliente confirma el pago en su aplicación
6. **Finalización Automática**: El pedido se completa automáticamente una vez que el pago es autorizado

### Procesando Pagos — Notificación Push {#processing-payments-push-notification}

1. **Agregar Artículos**: Agrega productos a tu carrito en el POS
2. **Seleccionar Gateway**: Escoge "Vipps MobilePay" como método de pago
3. **Ingresar Número de Teléfono**: Escribe el número de teléfono del cliente
4. **Enviar al Teléfono**: Haz clic en el botón "Enviar al Teléfono"
5. **Cliente Confirma**: El cliente recibe una notificación push y confirma el pago en su aplicación
6. **Finalización Automática**: El pedido se completa automáticamente una vez que el pago es autorizado

### Cancelando un Pago {#cancelling-a-payment}

Mientras esperas que el cliente confirme, puedes hacer clic en el botón **Cancelar Pago** para abortar la transacción. Esto cancela el pago pendiente del lado de Vipps y restablece la interfaz de pago.

### Reembolsos {#refunds}

Los reembolsos se manejan a través del proceso estándar de reembolso de WooCommerce. Abre el pedido, haz clic en **Reembolso**, ingresa el monto, y el reembolso se procesa automáticamente a través de la API de Vipps.

## Mercados Soportados {#supported-markets}

Vipps MobilePay opera en la región nórdica bajo dos marcas:

| Región | Marca | Monedas |
|--------|-------|---------|
| Noruega | Vipps | NOK |
| Dinamarca, Finlandia | MobilePay | DKK, EUR |

Tu cuenta de comerciante determina qué mercados y monedas están disponibles. Los clientes en cualquier mercado soportado pueden pagar utilizando su aplicación local de Vipps o MobilePay.

## Solución de Problemas {#troubleshooting}

### Problemas Comunes {#common-issues}

<Accordion>
  <AccordionItem question="El pago no se completa">

  - El proceso de pago espera hasta 5 minutos; si el cliente no confirma dentro de ese tiempo, el pago caduca
  - Verifica que no estés confundiendo el modo de prueba y las credenciales de producción
  - Verifica que tu conexión a internet sea estable tanto en el dispositivo POS como en el teléfono del cliente

  </AccordionItem>

  <AccordionItem question="Error de credenciales no válidas">

  - Revisa los cuatro campos de credenciales (MSN, ID de Cliente, Secreto de Cliente, Clave de Suscripción)
  - Asegúrate de usar credenciales de prueba cuando el modo de prueba esté habilitado, y credenciales de producción cuando esté deshabilitado
  - Verifica tus credenciales en el [Portal de Vipps](https://portal.vipps.no/)

  </AccordionItem>

  <AccordionItem question="Código QR no se está generando">

  - Confirma que tu sitio esté funcionando sobre HTTPS; la API de Vipps rechaza solicitudes de sitios HTTP
  - Revisa los registros de WooCommerce (`WooCommerce > Estado > Registros`) para obtener detalles del error de la API
  - Verifica que tu cuenta de comerciante esté activa y tenga habilitada la API de ePayment

  </AccordionItem>
</Accordion>
