La pantalla de Logs registra la actividad del sistema dentro del POS. Es una herramienta esencial para depurar problemas y entender lo que está ocurriendo detrás de escena.

{/* Screenshot: logs-screen.png */}

## Accediendo a los Logs

Abre la pantalla de Logs desde el cajón de navegación izquierdo (el ícono del latido).

## Visión General de la Interfaz

### Buscar y Filtrar

- **Buscar Logs** - Encuentra entradas de registro específicas por contenido del mensaje
- **Filtro de Nivel de Registro** - Filtrar por nivel de registro (info, advertencia, error)
- **Configuración de visualización** - Configurar columnas visibles

### Tabla de Logs

Cada entrada de registro muestra:

| Columna | Descripción |
|---------|-------------|
| **Hora** | Cuándo ocurrió el evento |
| **Nivel** | Severidad: info (gris), advertencia (naranja), error (rojo) |
| **Mensaje** | Descripción legible por humanos del evento |
| **Contexto** | JSON expansible con detalles adicionales |
| **Código** | Código de error (enlaces a la documentación) |

## Niveles de Registro

### Info (Gris)

Operaciones normales y eventos exitosos:
- "Se obtuvieron todas las ID para productos"
- "Se sincronizaron nuevos clientes"
- "Se obtuvieron todas las ID para impuestos"

### Advertencia (Naranja)

Problemas no críticos que pueden necesitar atención:
- "No se puede actualizar la sesión"
- "Intento de reintento para la sincronización"

### Error (Rojo)

Problemas que deben ser abordados:
- "Lo sentimos, no puedes listar recursos" (problema de permisos)
- "La solicitud de API falló"
- "Error de sincronización de base de datos"

## Datos de Contexto

Haz clic en la flecha de expansión en cualquier entrada de registro para ver el contexto JSON detallado:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Estos datos incluyen:
- **Conteos de ítems** - Cuántos registros fueron afectados
- **Tiempo de ejecución** - Cuánto tiempo tomó la operación
- **Carga del servidor** - Métricas de rendimiento del servidor
- **Detalles de solicitud/respuesta** - Para depurar problemas de API

## Códigos de Error

Muchas entradas de registro incluyen un **código de error** (por ejemplo, `API02004`, `DB01001`). Estos códigos se vinculan directamente a la [documentación de Códigos de Error](/error-codes), que proporciona:

- Explicación detallada del error
- Causas comunes
- Pasos de resolución de problemas

## Entradas de Registro Comunes

### Operaciones de Sincronización

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

Estos indican actividad normal de sincronización. El contexto muestra cuántos ítems fueron sincronizados.

### Errores de Permiso

```
[error] Sorry, you cannot list resources    {...} API02004
```

Esto generalmente significa que el usuario no tiene permiso para acceder a un recurso. Verifica los roles de usuario en `WP Admin > POS > Settings > Access`.

### Problemas de Sesión

```
[warn] Unable to refresh session    {...} API02007
```

El POS no pudo actualizar la sesión de autenticación. Esto puede requerir que inicies sesión nuevamente.

## Usando los Logs para Soporte

Cuando solicites soporte:

1. **Reproduce el problema** - Realiza la acción que causa el problema
2. **Revisa los logs** - Busca errores (rojo) o advertencias (naranja)
3. **Expande el contexto** - Obtén los detalles JSON completos
4. **Anota el código de error** - Inclúyelo en tu solicitud de soporte
5. **Copia entradas relevantes** - Comparte los mensajes de registro con contexto

## Monitoreo de Rendimiento

Los logs incluyen métricas de tiempo de ejecución y carga del servidor. Si notas:

- **Altos tiempos de ejecución** - El servidor puede estar sobrecargado o lento
- **Altos valores de carga del servidor** - Considera la optimización del servidor
- **Tiempos de espera frecuentes** - Verifica la conectividad de red
