La pantalla de Registros documenta la actividad del sistema dentro del POS. Es una herramienta esencial para depurar problemas y comprender lo que sucede en segundo plano.

{/* Screenshot: logs-screen.png */}

## Acceder a Registros

Abra la pantalla de Registros desde el menú de navegación lateral (el ícono de latido cardíaco).

## Descripción General de la Interfaz

### Buscar y Filtrar

- **Buscar Registros** - Encontrar entradas de registro específicas por contenido del mensaje
- **Filtro de Nivel de Registro** - Filtrar por nivel de registro (información, advertencia, error)
- **Configuración de visualización** - Configurar columnas visibles

### Tabla de Registros

Cada entrada de registro muestra:

| Columna | Descripción |
|--------|-------------|
| **Hora** | Cuándo ocurrió el evento |
| **Nivel** | Gravedad: información (gris), advertencia (naranja), error (rojo) |
| **Mensaje** | Descripción legible del evento |
| **Contexto** | JSON expandible con detalles adicionales |
| **Código** | Código de error (enlace a documentación) |

## Niveles de Registro

### Información (Gris)

Operaciones normales y eventos exitosos:
- "Recuperados todos los ID de productos"
- "Clientes sincronizados"
- "Recuperados todos los ID de impuestos"

### Advertencia (Naranja)

Problemas no críticos que pueden requerir atención:
- "No se puede actualizar la sesión"
- "Intento de reintento de sincronización"

### Error (Rojo)

Problemas que necesitan ser abordados:
- "Lo siento, no puede listar recursos" (problema de permisos)
- "Solicitud de API fallida"
- "Error de sincronización de base de datos"

## Datos de Contexto

Haga clic en la flecha de expansión de cualquier entrada de registro para ver el contexto JSON detallado:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Estos datos incluyen:
- **Recuentos de elementos** - Cuántos registros se vieron afectados
- **Tiempo de ejecución** - Cuánto tardó la operación
- **Carga del servidor** - Métricas de rendimiento del servidor
- **Detalles de solicitud/respuesta** - Para depurar problemas de API

## Códigos de Error

Muchas entradas de registro incluyen un **código de error** (por ejemplo, `API02004`, `DB01001`). Estos códigos enlazan directamente con la documentación de [Códigos de Error](/error-codes), que proporciona:

- Explicación detallada del error
- Causas comunes
- Pasos de resolución de problemas

## Entradas de Registro Comunes

### Operaciones de Sincronización

```
[info] Recuperados todos los ID de productos    {...} 3 elementos
[info] Clientes sincronizados            {...} 1 elemento
[info] Recuperados todos los ID de impuestos       {...} 3 elementos
```

Estos indican actividad normal de sincronización. El contexto muestra cuántos elementos se sincronizaron.

### Errores de Permisos

```
[error] Lo siento, no puede listar recursos    {...} API02004
```

Generalmente significa que el usuario no tiene permiso para acceder a un recurso. Verifique los roles de usuario en `WP Admin > POS > Configuración > Acceso`.

### Problemas de Sesión

```
[warn] No se puede actualizar la sesión    {...} API02007
```

El POS no pudo actualizar la sesión de autenticación. Esto puede requerir cerrar y volver a iniciar sesión.

## Usar Registros para Soporte

Al solicitar soporte:

1. **Reproducir el problema** - Realizar la acción que causa el problema
2. **Verificar los registros** - Buscar errores (rojo) o advertencias (naranja)
3. **Expandir el contexto** - Obtener los detalles JSON completos
4. **Anotar el código de error** - Incluirlo en su solicitud de soporte
5. **Copiar entradas relevantes** - Compartir los mensajes de registro con contexto

## Monitoreo de Rendimiento

Los registros incluyen métricas de tiempo de ejecución y carga del servidor. Si observa:

- **Tiempos de ejecución altos** - El servidor puede estar sobrecargado o ser lento
- **Valores altos de carga del servidor** - Considerar la optimización del servidor
- **Timeouts frecuentes** - Verificar la conectividad de red
