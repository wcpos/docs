El rendimiento del servidor impacta directamente la velocidad y la capacidad de respuesta de su aplicación WCPOS. Esta guía le ayuda a monitorear, diagnosticar y optimizar el rendimiento del servidor utilizando las métricas integradas y técnicas de resolución de problemas.

## Comprendiendo las Métricas del Servidor

WCPOS recopila automáticamente métricas de rendimiento del servidor con cada operación de obtención de datos (productos, pedidos, clientes, etc.). Puede ver estas métricas en la pantalla de [Registros](/support/logs).

### Métricas Típicas del Servidor

```json
{
  "total": "24692",
  "execution_time": "76.64 ms",
  "server_load": "[15.20605469,16.16357422,16.76806641]"
}
```

**Desglose:**
- **`total`** - Número de registros procesados (24,692 ID de productos)
- **`execution_time`** - Tiempo tomado para completar la operación (76.64 milisegundos)
- **`server_load`** - Promedios de carga del servidor para 1, 5 y 15 minutos

## Carga del Servidor Explicada

La carga del servidor representa la carga promedio del sistema durante diferentes períodos de tiempo:

- **Primer valor** - Promedio de 1 minuto (15.21)
- **Segundo valor** - Promedio de 5 minutos (16.16)
- **Tercer valor** - Promedio de 15 minutos (16.77)

### Interpretación de la Carga

Los valores de carga del servidor pueden ser engañosos y deben ser interpretados cuidadosamente:

:::warning Los Valores de Carga Pueden Ser Engañosos
Los promedios de carga del servidor no siempre se correlacionan directamente con el rendimiento. Un servidor con valores de carga altos (15+) aún puede ser muy receptivo si tiene suficientes recursos y está bien optimizado. **Enfóquese en los tiempos de ejecución en lugar de solo en los valores de carga.**
:::

**Directrices Generales:**
- **Carga relativa a los núcleos de CPU** - Una carga de 8.0 en un servidor de 8 núcleos significa uso completo
- **Sostenida vs. temporal** - Breves picos son normales, una carga alta sostenida puede indicar problemas
- **El rendimiento importa más** - Un servidor receptivo con alta carga es mejor que un servidor lento con baja carga

**Qué vigilar:**
- **Tiempos de ejecución en aumento** con el tiempo
- **Carga en crecimiento consistente** sin explicación
- **Tanto alta carga COMO tiempos de ejecución lentos** juntos

## Referencias de Rendimiento

### Directrices de Tiempos de Ejecución

| Operación | Bueno | Aceptable | Pobre | Crítico |
|-----------|------|------------|------|----------|
| **Obtención de productos** | < 100ms | 100-500ms | 500ms-2s | > 2s |
| **Creación de pedidos** | < 200ms | 200-800ms | 800ms-3s | > 3s |

### Consideraciones sobre el Conteo de Registros

El tiempo de ejecución debe escalar razonablemente con el conteo de registros:

```json
// Good scaling examples
{"total": "100", "execution_time": "15.2 ms"}    // 0.15ms per record
{"total": "1000", "execution_time": "89.4 ms"}   // 0.09ms per record
{"total": "10000", "execution_time": "234.1 ms"} // 0.02ms per record

// Poor scaling examples
{"total": "100", "execution_time": "500.0 ms"}   // 5.0ms per record
{"total": "1000", "execution_time": "8000.0 ms"} // 8.0ms per record
```

## Diagnóstico de Problemas de Rendimiento

### Paso 1: Monitorear los Registros

1. Abra **Registros** desde el panel de navegación
2. Realice la operación lenta (sincronizar productos, crear pedido, etc.)
3. Busque la entrada de registro correspondiente
4. Expanda el contexto para ver métricas

### Paso 2: Analizar las Métricas

**Tiempo de ejecución alto + Carga del servidor alta = Problema de recursos del servidor**
```json
{
  "total": "5000",
  "execution_time": "3500.0 ms",
  "server_load": "[12.45, 11.23, 10.87]"
}
```
*Solución: Aumentar los recursos del servidor o optimizar la configuración del servidor*

**Tiempo de ejecución alto + Carga del servidor normal = Problema de plugin/base de datos**
```json
{
  "total": "1000",
  "execution_time": "2800.0 ms",
  "server_load": "[1.23, 1.45, 1.67]"
}
```
*Solución: Identificar plugins lentos u optimizar consultas a la base de datos*

**Tiempo de ejecución normal + Carga del servidor alta = Sobrecarga general del servidor**
```json
{
  "total": "2000",
  "execution_time": "150.0 ms",
  "server_load": "[8.90, 9.12, 8.45]"
}
```
*Solución: Reducir la carga del servidor de otros procesos o aumentar recursos*

## Problemas Comunes de Rendimiento

### 1. Recursos Insuficientes del Servidor

**Síntomas:**
- Carga del servidor consistentemente alta (> 4.0 en la mayoría de los servidores)
- Tiempos de ejecución largos en todas las operaciones
- Frecuentes timeouts

**Soluciones:**
- **Actualizar CPU** - Más núcleos manejan mejor las solicitudes concurrentes
- **Incrementar RAM** - Reduce la I/O de disco y mejora el almacenamiento en caché
- **Usar almacenamiento SSD** - Mejora drásticamente el rendimiento de la base de datos
- **Optimizar los ajustes de PHP** - Aumentar memory_limit, max_execution_time

### 2. Consultas de Base de Datos Lentas

**Síntomas:**
- Altos tiempos de ejecución con carga normal del servidor
- Obtención de productos/pedidos particularmente lenta
- Códigos de error relacionados con la base de datos en los registros

**Soluciones:**
- **Habilitar WooCommerce HPOS** - La mayor mejora de rendimiento de base de datos
- **Usar almacenamiento en caché de objetos** - Redis o Memcached si está disponible por su host
- **Mantener WordPress actualizado** - Las actualizaciones del núcleo a menudo incluyen optimizaciones de la base de datos
- **Limitar revisiones de publicaciones** - Agregar `define('WP_POST_REVISIONS', 3);` a wp-config.php

### 3. Interferencia de Plugins

**Síntomas:**
- Degradación del rendimiento repentina después de actualizaciones de plugins
- Operaciones específicas mucho más lentas que otras
- Altos tiempos de ejecución con carga normal del servidor

**Resolución de problemas:**
1. **Probar en staging** - Desactivar todos los plugins excepto WooCommerce y WCPOS
2. **Medir la línea base** - Registrar tiempos de ejecución con plugins mínimos
3. **Habilitar gradualmente** - Agregar plugins uno por uno para identificar culpables
4. **Revisar los hooks de plugins** - Buscar plugins que se enganchen en acciones de WooCommerce

**Plugins problemáticos comunes:**
- Plugins pesados de SEO durante operaciones con productos
- Sistemas de gestión de inventario complejos
- Plugins de análisis/tracking en tiempo real
- Plugins personalizados con mala codificación

### 4. Configuración de WordPress/WooCommerce

**Síntomas:**
- Rendimiento inconsistente
- Errores relacionados con la memoria en los registros
- Tablero de administración lento

**Lista de verificación de optimización:**
- **Versión de PHP** - Usar PHP 8.0+ para mejor rendimiento
- **WooCommerce HPOS** - Habilitar Almacenamiento de Pedidos de Alto Rendimiento (ver a continuación)
- **Almacenamiento en caché de WordPress** - Habilitar almacenamiento en caché de objetos si está disponible
- **Ajustes de WooCommerce** - Optimizar tamaños de imágenes de productos

### Almacenamiento de Pedidos de Alto Rendimiento de WooCommerce (HPOS)

:::tip Mayor Ganancia de Rendimiento
**HPOS es una de las mejoras más significativas en rendimiento que puede hacer para WooCommerce.** Almacena pedidos en tablas de base de datos personalizadas en lugar de la tabla de publicaciones de WordPress, mejorando drásticamente el rendimiento para tiendas con muchos pedidos.
:::

**Beneficios:**
- **Consultas de pedidos más rápidas** - Pedidos almacenados en una estructura de base de datos optimizada
- **Carga de base de datos reducida** - Separa los pedidos de publicaciones/páginas
- **Mejor escalabilidad** - Maneja grandes cantidades de pedidos de manera eficiente
- **Mejor rendimiento administrativo** - Pantallas de gestión de pedidos más rápidas

**Cómo Habilitar:**
1. Vaya a `WooCommerce > Configuración > Avanzado > Características`
2. Habilite "Almacenamiento de pedidos de alto rendimiento"
3. Siga el proceso de migración

**Aprende Más:**
- [Documentación de WooCommerce HPOS](https://woocommerce.com/document/high-performance-order-storage/)
- [Guía de Migración de HPOS](https://woocommerce.com/document/high-performance-order-storage/#section-3)

## Mejores Prácticas de Monitoreo del Servidor

### 1. Revisiones de Rendimiento Regulares

- **Revisiones semanales** - Revisar registros para tendencias de rendimiento
- **Mediciones de línea base** - Registrar tiempos de ejecución normales
- **Monitoreo en tiempos pico** - Monitorear durante períodos de alta tráfico

### 2. Establecer Alertas de Rendimiento

Monitorear por estas señales de advertencia:
- Tiempos de ejecución > 1000ms consistentemente
- Carga del servidor > 5.0 por períodos prolongados
- Errores de timeout frecuentes en los registros

### 3. Planificación de Capacidad

Rastrear tendencias de crecimiento:
- **Crecimiento del conteo de registros** - Productos, pedidos, clientes
- **Degradación del rendimiento** - Cómo escalan los tiempos de ejecución
- **Utilización de recursos** - CPU, memoria, uso de disco

## Estrategias de Optimización del Servidor

### 1. Mejores Prácticas de WordPress/WooCommerce

**Habilitar HPOS:**
- La mejora más impactante para el rendimiento de WooCommerce
- Consulte la sección HPOS anterior para más detalles

**Configuración de PHP (consulte a su host):**
```ini
memory_limit = 512M
max_execution_time = 300
max_input_vars = 3000
```

**Configuración de WordPress:**
```php
// In wp-config.php - limit post revisions
define('WP_POST_REVISIONS', 3);

// Enable WordPress debug logging if needed
define('WP_DEBUG_LOG', true);
```

### 2. Optimización a Nivel de Hosting

**Almacenamiento en caché de objetos:**
- Pregunte a su host sobre la disponibilidad de Redis o Memcached
- Muchos hosts de WordPress gestionados proporcionan esto automáticamente

**Versión de PHP:**
- Use PHP 8.0+ para mejoras significativas en rendimiento
- La mayoría de los hosts permiten cambiar fácilmente la versión de PHP

**Recursos del servidor:**
- Asegúrese de tener suficiente RAM (mínimo 1GB, preferiblemente 2GB+)
- El almacenamiento SSD ofrece un rendimiento de base de datos mucho mejor que los discos tradicionales

## Cuándo Buscar Ayuda

Contacte a su proveedor de hosting o a un desarrollador de WordPress si:

- **Carga del servidor consistentemente > 8.0** a pesar de los esfuerzos de optimización
- **Tiempos de ejecución > 5000ms** para operaciones simples
- **Errores de memoria** que aparecen frecuentemente en los registros
- **Consultas de base de datos que tardan > 2 segundos** consistentemente

Proporcióneles:
- Métricas del servidor de sus registros
- Lista de plugins activos
- Especificaciones del servidor (CPU, RAM, tipo de almacenamiento)
- Versiones de WordPress y WooCommerce
