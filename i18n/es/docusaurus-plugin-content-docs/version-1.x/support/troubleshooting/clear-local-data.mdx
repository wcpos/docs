WCPOS almacena una copia local de sus productos, pedidos y clientes para que el POS pueda funcionar con rapidez y sin conexión. En ocasiones, estos datos locales pueden perder la sincronización con su tienda WooCommerce. Borrar todos los datos locales elimina la base de datos local y activa una sincronización nueva desde su servidor.

:::warning
Esta es la opción más drástica. Borrar los datos locales elimina todo lo almacenado en el dispositivo y fuerza una resincronización completa. Según el tamaño de su tienda, esto puede tardar un tiempo. Se debe usar solo cuando otras opciones, como una [sincronización](/products/sync) normal, no hayan resuelto el problema.
:::

## Cuándo usarlo {#when-to-use-it}

Puede ser necesario borrar los datos locales cuando:

- **Problemas de sincronización** — los productos, pedidos o clientes no se actualizan correctamente
- **Después de actualizar el plugin** — para garantizar la compatibilidad con las nuevas estructuras de datos
- **Datos obsoletos** — los precios, los niveles de inventario u otra información aparecen desactualizados

## Qué se borra {#what-gets-cleared}

| Borrado | No borrado |
|---------|-------------|
| Base de datos local de productos | Datos del lado del servidor |
| Base de datos local de pedidos | Ajustes de WooCommerce |
| Base de datos local de clientes | |
| Sesión de acceso | |

Los datos de su tienda WooCommerce no se ven afectados; solo se elimina la copia local del dispositivo. Tras el borrado, se inicia automáticamente una sincronización nueva.

:::info Diferencia entre plataformas
- **Aplicación de escritorio:** se cerrará la sesión y será necesario volver a conectar la tienda.
- **Aplicación web:** los datos de acceso se borran, pero como la sesión sigue iniciada en WP Admin, la sesión se rehidrata automáticamente; por tanto, no se notará que se haya cerrado la sesión.
:::

## Cómo borrar los datos locales {#how-to-clear-local-data}

### Web {#web}

1. Haga clic en el **menú de usuario** en la esquina superior derecha para abrir el menú
2. Haga clic en **Borrar todos los datos locales**
3. La base de datos local se eliminará y se iniciará automáticamente una nueva sincronización

### Escritorio {#desktop}

1. Abra el menú de la aplicación **WCPOS** en la barra de menús superior
2. Pase el cursor sobre **Avanzado**
3. Haga clic en **Borrar datos de la aplicación**
4. La base de datos local se eliminará y se iniciará automáticamente una nueva sincronización

:::note
La aplicación de escritorio utiliza una base de datos SQLite, mientras que la aplicación web utiliza IndexedDB. Ambas bases de datos se borran por completo, pero el comportamiento de inicio de sesión difiere — consulte la nota sobre la diferencia de plataforma anterior.
:::
