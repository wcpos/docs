WCPOS incluye un sistema completo de plantillas para recibos, facturas, recibos de regalo, albaranes, tickets de cocina y más — todo gestionado en **WP Admin → POS → Plantillas**.

:::tip ¿Solo necesita cambiar el recibo?
Comience en **[Personalizar el recibo](/receipts/customise)**. Allí se explican las tres formas más sencillas de hacerlo: elegir una plantilla incluida diferente, pedir a la IA que la ajuste o editarla manualmente.
:::

## Motores de plantillas {#template-engines}

WCPOS admite tres motores de plantillas. El motor se elige al crear una plantilla o al hacer clic en **Usar plantilla** en una tarjeta de la galería, y **no se puede cambiar después** — es necesario crear una nueva plantilla para cambiar de motor.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML sin lógica (recomendado)">
    Estilo Mustache <code>{"{{variable}}"}</code> marcadores de posición dentro de HTML simple. Se renderiza en el cliente, por lo que los recibos **funcionan sin conexión**. Se utiliza para impresión desde el navegador y recibos en PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML térmico">
    Plantillas XML que producen tanto una vista previa en pantalla como **comandos ESC/POS** para impresoras térmicas de recibos. Una misma plantilla, dos salidas. Se utiliza para impresoras térmicas Epson, Star y otras.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP heredado">
    Plantillas PHP del lado del servidor que utilizan funciones de WooCommerce. Se mantienen por compatibilidad con las personalizaciones existentes de <code>yourtheme/woocommerce-pos/receipt.php</code> Requiere conexión al servidor para renderizar.
  </FeatureCard>
</FeatureGrid>

## Galería de plantillas {#template-gallery}

La galería incluye plantillas prediseñadas que se pueden usar tal cual o personalizar. Es posible filtrar por **categoría** (Recibo, Factura, Recibo de regalo, Ticket de cocina, Cotización / Orden de compra), **formato** (HTML / ESC/POS) y **dirección** (LTR / RTL); la vista previa utiliza datos reales de la tienda. El pie de cada tarjeta tiene un enlace de **Vista previa** y un botón principal **Usar plantilla** — al hacer clic en Usar plantilla se crea una copia editable en *Sus plantillas*.

### Plantillas de documentos de terceros {#third-party-document-templates}

Si el plugin **PDF Invoices & Packing Slips for WooCommerce** de WP Overnight está activo, WCPOS también muestra **Invoice (WP Overnight)** y **Packing Slip (WP Overnight)** en la lista de plantillas de recibos. Estas plantillas HTML renderizadas en el servidor delegan en la API de documentos de WP Overnight, reutilizando la numeración de facturas y albaranes, el diseño, la imagen de marca, los campos legales/fiscales y las personalizaciones de plantilla configuradas en ese plugin.

Solo están disponibles mientras el plugin WP Overnight esté activo y requieren conexión al servidor para renderizarse. Es recomendable utilizar las plantillas HTML o térmicas incluidas en WCPOS cuando se necesite impresión sin conexión.

### Visualización adaptativa de impuestos {#adaptive-tax-display}

La mayoría de las plantillas incluidas (Standard, Standard RTL, Minimal/Modern, Narrow, Invoice, Quote, variantes Thermal Simple) **se adaptan automáticamente a la configuración de visualización de impuestos de WooCommerce**:

- Las **tiendas con impuestos incluidos** (UE/Reino Unido/AU) muestran precios brutos con una línea de "Impuesto incluido".
- Las **tiendas con impuestos excluidos** (EE. UU./CA) muestran precios netos con el impuesto añadido como una línea separada.
- El total general siempre muestra el importe bruto efectivamente cobrado.

La familia **Detallado** es una factura fiscal formal y siempre desglosa los impuestos con líneas sin impuestos incluidos más un desglose explícito, independientemente de la configuración de la tienda.

## Cómo funciona {#how-it-works}

1. **Agregar una plantilla** — hacer clic en **Usar plantilla** en una tarjeta de la galería, o crear una desde cero.
2. **Personalizar** usando el editor integrado con vista previa en vivo, o pegar el contenido en ChatGPT / Claude y solicitar los cambios deseados.
3. **Activar** — cambiar el interruptor **Activo** en *Sus plantillas*. Cada plantilla de recibo activa aparece en el menú desplegable de la pantalla de recibos en el punto de venta.
4. **Imprimir o mostrar** — los recibos se abren después del proceso de pago con opciones para imprimir, enviar por correo electrónico o ver en pantalla.

Las plantillas se renderizan a partir de un [payload de datos de recibo](/receipts/receipt-data) estandarizado con secciones para información de la tienda, líneas de artículos, totales, impuestos, pagos, reembolsos y más. Todos los campos de moneda incluyen variantes `_display` con formato previo (por ejemplo, `$29.99` en lugar de `29.99`).

## Funciones Pro {#pro-features}

Con [WCPOS Pro](/getting-started/pro-license), cada tienda puede tener sus propias asignaciones de plantillas, marca (logotipo, dirección, datos de contacto) y orden de plantillas — útil para operaciones con múltiples ubicaciones donde cada local imprime su propio membrete.
