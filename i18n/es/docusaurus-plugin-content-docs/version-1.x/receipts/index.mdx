WCPOS incluye un sistema completo de plantillas para recibos, facturas, recibos de regalo, albaranes, comandas de cocina y más, todo gestionado en **WP Admin → POS → Plantillas**.

:::tip ¿Solo quiere cambiar su recibo?
Para empezar, consulte **[Personalizar su recibo](/receipts/customise)**. Explica las tres formas más sencillas de hacerlo: elegir una plantilla incluida diferente, pedir a la IA que ajuste una, o editarla manualmente.
:::

## Motores de plantillas {#template-engines}

WCPOS admite tres motores de plantillas. El motor se elige al crear una plantilla o al hacer clic en **Usar plantilla** en una tarjeta de la galería, y **no se puede cambiar después**; cree una plantilla nueva si desea cambiar de motor.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML sin lógica (recomendado)">
    Estilo Mustache <code>{"{{variable}}"}</code> marcadores de posición dentro de HTML sin formato. Se renderiza en el lado del cliente, por lo que los recibos **funcionan sin conexión**. Se usa para la impresión desde el navegador y recibos en PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML térmico">
    Plantillas XML que generan tanto una vista previa en pantalla como **comandos ESC/POS** para impresoras térmicas de recibos. La misma plantilla, dos salidas. Se usa para Epson, Star y otras impresoras térmicas.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP heredado">
    Plantillas PHP del lado del servidor que usan funciones de WooCommerce. Se conservan por compatibilidad con las sobrescrituras existentes de <code>yourtheme/woocommerce-pos/receipt.php</code> . Requiere una conexión al servidor para renderizarse.
  </FeatureCard>
</FeatureGrid>

## Galería de plantillas {#template-gallery}

La galería incluye plantillas listas para usar tal cual o personalizar. Filtre por **categoría** (Recibo, Factura, Recibo de regalo, Comanda de cocina, Presupuesto / Orden de compra), **formato** (HTML / ESC/POS) y **dirección** (LTR / RTL); obtenga una vista previa con los datos reales de su tienda. El pie de cada tarjeta tiene un enlace **Vista previa** y un botón principal **Usar plantilla**; al hacer clic en Usar plantilla se crea una copia editable en *Sus plantillas*.

| Plantilla | Motor | Descripción |
|----------|--------|-------------|
| Recibo estándar | HTML | Recibo predeterminado: logotipo, identidad de la tienda, artículos, totales, pago |
| Recibo estándar (RTL) | HTML | Variante de derecha a izquierda para árabe, hebreo, persa y urdu |
| Mínimo / Moderno | HTML | Los mismos elementos esenciales que Estándar, comprimidos en menos espacio vertical |
| Recibo detallado | HTML | Factura fiscal completa — columna SKU, precio unitario, desglose de impuestos por tasa, direcciones |
| Recibo de regalo | HTML | Solo artículos, precios ocultos. Incluye mensaje de regalo y política de devoluciones |
| Factura | HTML | Factura de página completa A4/Carta con panel opcional "Cómo pagar" para pedidos no pagados |
| Lista de embalaje | HTML | Documento complementario para almacén — artículos + cantidades, dirección de envío, sin precios |
| Presupuesto / Estimación | HTML | Documento de preventa con precios y condiciones — sin sección de pago |
| Recibo estrecho | HTML | Recibo monoespaciado para papel estrecho o impresoras térmicas compatibles con HTML |
| Recibo térmico simple (58mm) | Térmica | Diseño térmico limpio de 58mm |
| Recibo térmico simple (80mm) | Térmica | Diseño térmico limpio de 80mm — el más común |
| Recibo térmico simple 80mm (RTL) | Térmica | Variante RTL para 80mm. Requiere una impresora con una página de códigos árabe (CP864 / Windows-1256) |
| Recibo térmico detallado (58mm) | Térmica | 58mm completo — direcciones, desglose de impuestos, reembolsos, pagos, condiciones, código de barras |
| Recibo térmico detallado (80mm) | Térmica | 80mm completo — direcciones, desglose de impuestos, reembolsos, pagos, condiciones, código de barras |
| Comanda de cocina | Térmico | Solo artículos, fuente grande, sin precios — para estaciones de preparación |

### Plantillas de documentos de terceros {#third-party-document-templates}

Si **PDF Invoices & Packing Slips for WooCommerce** de WP Overnight está activo, WCPOS también muestra **Factura (WP Overnight)** y **Albarán (WP Overnight)** en la lista de plantillas de recibo. Estas plantillas HTML renderizadas en el servidor delegan en la API de documentos de WP Overnight, reutilizando la numeración de facturas y albaranes, el diseño, la marca, los campos legales/fiscales y las personalizaciones de plantilla configuradas en ese plugin.

Solo están disponibles mientras el plugin WP Overnight esté activo y requieren una conexión con el servidor para renderizarse. Utilice las plantillas HTML o térmicas incluidas con WCPOS cuando se requiera impresión sin conexión.

### Visualización adaptable de impuestos {#adaptive-tax-display}

La mayoría de las plantillas incluidas (Estándar, Estándar RTL, Mínima/Moderna, Estrecha, Factura, Presupuesto, variantes Térmica simple) **se adaptan automáticamente al ajuste de visualización de impuestos de WooCommerce**:

- Las **tiendas con impuestos incluidos** (UE/Reino Unido/AU) muestran precios brutos con una línea de "Impuestos incluidos".
- Las **tiendas con impuestos excluidos** (EE. UU./CA) muestran precios netos con el impuesto añadido como línea separada.
- El total general siempre muestra el importe bruto que se ha cobrado realmente.

La familia **Detallada** es una factura fiscal formal y siempre desglosa los impuestos con líneas sin impuestos incluidos y un desglose explícito, independientemente del ajuste de la tienda.

## Cómo funciona {#how-it-works}

1. **Agregar una plantilla** — haga clic en **Usar plantilla** en una tarjeta de la galería, o cree una propia desde cero.
2. **Personalizarla** con el editor integrado con vista previa en directo, o pegarla en ChatGPT / Claude y solicitar los cambios deseados.
3. **Activarla** — active el interruptor **Activo** en *Sus plantillas*. Cada plantilla de recibo activa aparece en el menú desplegable de la pantalla de recibos en la caja.
4. **Imprimir o mostrar** — los recibos se abren después del pago con opciones para imprimir, enviar por correo electrónico o ver en pantalla.

Las plantillas se renderizan con una [carga de datos de recibo](/receipts/receipt-data) estandarizada, con secciones para información de la tienda, partidas, totales, impuestos, pagos, reembolsos y más. Todos los campos de moneda incluyen variantes `_display` con formato previo (por ejemplo, `$29.99` en lugar de `29.99`).

## Funciones Pro {#pro-features}

Con [WCPOS Pro](/getting-started/pro-license), cada tienda puede tener sus propias asignaciones de plantillas, identidad de marca (logotipo, dirección, datos de contacto) y orden de plantillas, lo que resulta útil para operaciones con varias ubicaciones en las que cada tienda imprime su propio membrete.
