# DB03002: Tipo de Dato Inválido {#db03002-invalid-data-type}

## Qué Significa Esto {#what-this-means}

El tipo de dato no coincide con lo que espera la base de datos. Por ejemplo, se proporcionó texto donde se esperaba un número.

## Causas Comunes {#common-causes}

- **Error de entrada del usuario** — Ingresar texto en un campo numérico
- **Problemas de importación** — Los datos importados tienen formato incorrecto
- **Corrupción de datos** — Valores corruptos durante la transferencia
- **Conflicto de plugins** — Otro plugin modificó los tipos de datos

## Cómo Solucionar {#how-to-fix}

### 1. Verifica Tu Entrada {#1-check-your-input}

Revisa los datos que estás ingresando:
- **Precios** — Deben ser números (por ejemplo, `19.99`)
- **Cantidades** — Deben ser números enteros (por ejemplo, `5`)
- **IDs** — Deben ser numéricos

### 2. Borra y Vuelve a Ingresar {#2-clear-and-re-enter}

Si los datos fueron corruptos:
- Borra el campo
- Ingresa el valor nuevamente de forma manual
- Evita copiar y pegar desde fuentes externas

### 3. Verifica los Datos de Origen {#3-check-source-data}

Si estás sincronizando desde WooCommerce:
- Verifica los datos en WordPress Admin
- Busca campos con formato incorrecto
- Corrige los datos en la fuente

### 4. Re-sincroniza los Datos {#4-re-sync-data}

Obtén datos frescos del servidor:
- Borra la caché local
- Sincroniza los datos nuevamente
- Verifica si el problema se resuelve

### 5. Verifica Conflictos de Plugins {#5-check-for-plugin-conflicts}

Si usas otros plugins de WooCommerce:
- Pueden modificar los datos de maneras inesperadas
- Desactívalos temporalmente para probar
- Reporta incompatibilidades

## Ejemplos Comunes {#common-examples}

| Campo | Esperado | Inválido |
|-------|----------|---------|
| Precio | `19.99` | `$19.99` |
| Cantidad | `5` | `cinco` |
| SKU | `ABC123` | — (cualquier formato está bien) |
| ID del Producto | `42` | `producto-42` |
