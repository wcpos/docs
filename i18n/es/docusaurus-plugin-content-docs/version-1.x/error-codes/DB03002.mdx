# DB03002: Tipo de Datos No Válido

## Qué Significa

El tipo de datos no coincide con lo que la base de datos espera. Por ejemplo, se proporcionó texto donde se esperaba un número.

## Causas Comunes

- **Error de entrada de usuario** — Introducir texto en un campo numérico
- **Problemas de importación** — Los datos importados tienen un formato incorrecto
- **Corrupción de datos** — Valores corruptos durante la transferencia
- **Conflicto de plugins** — Otro plugin modificó los tipos de datos

## Cómo Solucionar

### 1. Verificar la Entrada

Revisa los datos que estás introduciendo:
- **Precios** — Deben ser números (por ejemplo, `19.99`)
- **Cantidades** — Deben ser números enteros (por ejemplo, `5`)
- **IDs** — Deben ser numéricos

### 2. Borrar y Volver a Introducir

Si los datos estaban corruptos:
- Borra el campo
- Introduce el valor manualmente
- Evita copiar y pegar desde fuentes externas

### 3. Verificar los Datos de Origen

Si sincronizas desde WooCommerce:
- Comprueba los datos en el Administrador de WordPress
- Busca campos con formato incorrecto
- Corrige los datos en el origen

### 4. Resincronizar Datos

Obtén datos nuevos del servidor:
- Borra la caché local
- Sincroniza los datos de nuevo
- Comprueba si se resuelve el problema

### 5. Verificar Conflictos de Plugins

Si usas otros plugins de WooCommerce:
- Pueden modificar datos de forma inesperada
- Desactívalos temporalmente para probar
- Reporta incompatibilidades

## Ejemplos Comunes

| Campo | Esperado | No Válido |
|-------|----------|-----------|
| Precio | `19.99` | `$19.99` |
| Cantidad | `5` | `cinco` |
| SKU | `ABC123` | — (formato OK) |
| ID de Producto | `42` | `product-42` |
