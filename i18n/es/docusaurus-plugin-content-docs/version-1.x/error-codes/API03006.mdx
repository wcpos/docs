# API03006: Método No Admitido

## Qué Significa

El método HTTP utilizado (GET, POST, PUT, DELETE, etc.) no está admitido para este punto final. El servidor no acepta este tipo de solicitud para esta URL.

## Causas Comunes

- **Incompatibilidad de versiones** — Las versiones de POS y el plugin son incompatibles
- **Punto final eliminado** — Se ha dejado en desuso un punto final de API
- **Configuración del servidor** — El servidor web está bloqueando ciertos métodos HTTP
- **Conflicto de plugins** — Otro plugin modificando el comportamiento de la API REST

## Cómo Solucionarlo

### 1. Actualizar Ambos Componentes

Asegurar compatibilidad:
- Actualizar el plugin de WordPress de WCPOS
- Actualizar la aplicación POS
- Revisar notas de lanzamiento para cambios importantes

### 2. Verificar Configuración del Servidor

Algunos servidores bloquean ciertos métodos HTTP:
- Asegurarse de que los métodos PUT y DELETE estén permitidos
- Verificar restricciones de métodos en `.htaccess`
- Revisar configuración de Nginx

### 3. Verificar Acceso a la API REST

Probar la API REST de WordPress:
1. Visitar `https://tutienda.com/wp-json/` en un navegador
2. Debe devolver datos JSON
3. Si no, la API REST puede estar deshabilitada o bloqueada

### 4. Verificar Conflictos de Plugins

Deshabilitar plugins temporalmente:
- Plugins de seguridad pueden bloquear métodos
- Otros plugins de API REST pueden causar conflictos
- Volver a habilitar uno por uno para encontrar el problema

### 5. Revisar Restricciones de Hosting

Algunos hosts restringen métodos HTTP:
- Contactar con soporte de hosting
- Solicitar que habiliten todos los métodos estándar
- Considerar cambiar de host si es demasiado restrictivo
