# API06001: Formato de URL no válido

## Qué Significa

La URL configurada para su sitio de WooCommerce no tiene un formato válido. Las URL deben seguir las convenciones estándar de direcciones web.

## Causas Comunes

- **Protocolo faltante** — La URL no comienza con `http://` o `https://`
- **Error de escritura en la URL** — Errores de ortografía en el dominio
- **Caracteres no válidos** — Caracteres especiales que no están permitidos en las URL
- **URL incompleta** — La URL carece de partes como la extensión de dominio

## Cómo Solucionarlo

### 1. Verificar el Formato de la URL

Una URL válida debe tener este aspecto:
- ✅ `https://sutienda.com`
- ✅ `https://www.sutienda.com`
- ✅ `https://tienda.sudominio.com`
- ❌ `sutienda.com` (falta protocolo)
- ❌ `https://sutienda` (falta extensión)
- ❌ `https://su tienda.com` (contiene espacio)

### 2. Incluir el Protocolo

Siempre incluya `https://` o `http://`:
- Preferido: `https://` (seguro)
- Alternativa: `http://` (si HTTPS no está disponible)

### 3. Eliminar Caracteres Extra

Asegúrese de que la URL no tenga:
- Barras diagonales al final (opcional pero sea consistente)
- Espacios extra
- Caracteres especiales
- Parámetros de URL (a menos que sean necesarios)

### 4. Verificar que la URL Funcione

Pruebe la URL en un navegador:
1. Copie la URL que está usando
2. Péguela en un navegador
3. Debe cargar su sitio de WordPress

### 5. Actualizar la Configuración del POS

Introduzca la URL corregida en la configuración del POS:
- Coincida exactamente con lo que funciona en su navegador
- Incluya o excluya `www.` según lo use su sitio

## Ejemplos

| Inválido | Válido |
|---------|-------|
| `mitienda.com` | `https://mitienda.com` |
| `https://mi tienda.com` | `https://mitienda.com` |
| `https://mitienda` | `https://mitienda.com` |
| `htp://mitienda.com` | `https://mitienda.com` |
