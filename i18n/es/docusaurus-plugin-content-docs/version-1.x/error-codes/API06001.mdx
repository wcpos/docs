# API06001: Formato de URL Inválido

## Qué Significa Esto

La URL configurada para su sitio de WooCommerce no está en un formato válido. Las URL deben seguir las convenciones estándar de direcciones web.

## Causas Comunes

- **Falta de protocolo** — La URL no comienza con `http://` o `https://`
- **Error tipográfico en la URL** — Errores ortográficos en el dominio
- **Caracteres inválidos** — Caracteres especiales que no están permitidos en las URL
- **URL incompleta** — La URL carece de partes como la extensión del dominio

## Cómo Solucionar

### 1. Verifique el Formato de la URL

Una URL válida debería lucir así:
- ✅ `https://tuempresa.com`
- ✅ `https://www.tuempresa.com`
- ✅ `https://tienda.tudominio.com`
- ❌ `tuempresa.com` (falta protocolo)
- ❌ `https://tuempresa` (falta extensión)
- ❌ `https://tu empresa.com` (contiene espacio)

### 2. Incluya el Protocolo

Siempre incluya `https://` o `http://`:
- Preferido: `https://` (seguro)
- Alternativa: `http://` (si HTTPS no está disponible)

### 3. Elimine Caracteres Extra

Asegúrese de que la URL no tenga:
- Barras al final (opcional pero sea consistente)
- Espacios extra
- Caracteres especiales
- Parámetros de URL (a menos que sean necesarios)

### 4. Verifique que la URL Funcione

Pruebe la URL en un navegador:
1. Copie la URL que está utilizando
2. Pégela en un navegador
3. Debe cargar su sitio de WordPress

### 5. Actualice la Configuración del POS

Ingrese la URL corregida en la configuración del POS:
- Coincidir exactamente con lo que funciona en su navegador
- Incluir o excluir `www.` según lo que use su sitio

## Ejemplos

| Inválido | Válido |
|----------|--------|
| `mitienda.com` | `https://mitienda.com` |
| `https://mi tienda.com` | `https://mitienda.com` |
| `https://mitienda` | `https://mitienda.com` |
| `htp://mitienda.com` | `https://mitienda.com` |
