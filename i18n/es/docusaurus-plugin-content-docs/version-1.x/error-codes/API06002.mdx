# API06002: URL de API Faltante

## Qué Significa Esto

No se ha configurado ninguna URL para conectar a tu tienda WooCommerce. El POS necesita saber dónde está ubicada tu tienda.

## Causas Comunes

- **Configuración inicial** — URL nunca fue ingresada
- **Configuración borrada** — Ajustes fueron restablecidos o eliminados
- **Datos de aplicación borrados** — Datos de aplicación fueron eliminados
- **Problema de migración** — URL perdida durante actualización o migración

## Cómo Solucionarlo

### 1. Ingresar tu URL de Tienda

En la aplicación POS:
1. Abrir configuración o pantalla de inicio de sesión
2. Encontrar el campo de configuración de URL/Sitio
3. Ingresar la URL de tu sitio WordPress
4. Ejemplo: `https://tutienda.com`

### 2. Encontrar tu URL de WordPress

Si no estás seguro de tu URL:
1. Iniciar sesión en Administración de WordPress
2. Ir a Ajustes → General
3. Buscar "Dirección WordPress (URL)"
4. Usar esta URL en el POS

### 3. Verificar Configuraciones Almacenadas

Si la URL fue configurada previamente:
- Revisar si los datos de la aplicación fueron borrados accidentalmente
- Buscar opciones de respaldo/exportación de configuraciones
- Volver a ingresar la URL si es necesario

### 4. Verificar que la URL sea Correcta

Antes de guardar:
1. Probar la URL en un navegador
2. Asegurarse de que carga tu sitio WordPress
3. Incluir `https://` o `http://`

## Lista de Verificación de Configuración

Al configurar WCPOS por primera vez:

1. ☐ Ingresar la URL de tu sitio WordPress
2. ☐ Verificar que el plugin WCPOS esté instalado y activo
3. ☐ Iniciar sesión con tus credenciales de WordPress
4. ☐ Permitir que la sincronización inicial de datos se complete
