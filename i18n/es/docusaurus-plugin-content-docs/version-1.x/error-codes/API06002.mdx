# API06002: URL de API faltante {#api06002-missing-api-url}

## Qué significa esto {#what-this-means}

No se ha configurado ninguna URL para conectar con su tienda de WooCommerce. El POS necesita saber dónde se encuentra su tienda.

## Causas comunes {#common-causes}

- **Configuración por primera vez** — Nunca se ingresó la URL
- **Configuración borrada** — Las configuraciones fueron restablecidas o borradas
- **Datos de la aplicación borrados** — Se eliminaron los datos de la aplicación
- **Problema de migración** — URL perdida durante la actualización o migración

## Cómo solucionar {#how-to-fix}

### 1. Ingrese la URL de su tienda {#1-enter-your-store-url}

En la aplicación de POS:
1. Abra la configuración o la pantalla de inicio de sesión
2. Busque el campo de configuración de URL/Sitio
3. Ingrese la URL de su sitio de WordPress
4. Ejemplo: `https://sutienda.com`

### 2. Encuentre su URL de WordPress {#2-find-your-wordpress-url}

Si no está seguro de su URL:
1. Inicie sesión en WordPress Admin
2. Vaya a Configuración → General
3. Mire "Dirección de WordPress (URL)"
4. Use esta URL en el POS

### 3. Verifique la configuración almacenada {#3-check-stored-settings}

Si la URL se configuró previamente:
- Verifique si los datos de la aplicación fueron borrados accidentalmente
- Busque opciones de respaldo/exportación de configuraciones
- Vuelva a ingresar la URL si es necesario

### 4. Verifique que la URL sea correcta {#4-verify-url-is-correct}

Antes de guardar:
1. Pruebe la URL en un navegador
2. Asegúrese de que cargue su sitio de WordPress
3. Incluya `https://` o `http://`

## Lista de verificación de configuración {#setup-checklist}

Al configurar WCPOS por primera vez:

1. ☐ Ingrese la URL de su sitio de WordPress
2. ☐ Verifique que el plugin WCPOS esté instalado y activo
3. ☐ Inicie sesión con sus credenciales de WordPress
4. ☐ Permita que la sincronización inicial de datos se complete
