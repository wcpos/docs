# DB03003: Campo Requerido Falta

## Lo que Esto Significa

No se proporcionó un campo requerido. La base de datos no puede guardar el registro porque falta información esencial.

## Causas Comunes

- **Formulario incompleto** — Los campos requeridos no se llenaron
- **Problema de sincronización de datos** — Los datos requeridos no se sincronizaron correctamente
- **Validación omitida** — Formulario enviado sin validación
- **Campo personalizado requerido** — Un campo personalizado requerido está vacío

## Cómo Solucionarlo

### 1. Completa Todos los Campos Requeridos

Verifica el formulario por:
- Campos marcados con asteriscos (*)
- Campos resaltados o marcados con error
- Campos vacíos que deberían tener valores

### 2. Actualiza y Vuelve a Ingresar

Si el estado del formulario es incorrecto:
1. Actualiza la página/pantalla
2. Vuelve a ingresar toda la información
3. Verifica todos los campos antes de enviar

### 3. Revisa la Configuración de WooCommerce

Si se requieren campos personalizados:
1. Revisa la configuración de pago de WooCommerce
2. Verifica los campos personalizados requeridos
3. Asegúrate de que el POS proporcione estos campos

### 4. Sincroniza los Datos Requeridos

Si falta información relacionada:
- Actualiza productos/clientes/etc.
- Espera a que la sincronización se complete
- Vuelve a intentar la operación

### 5. Revisa la Configuración de Campos Requeridos

En WooCommerce Admin:
- Verifica qué campos están marcados como requeridos
- Considera si todos son realmente necesarios para el POS
- Ajusta los requisitos si es necesario

## Campos Comúnmente Requeridos

Típicamente requeridos para pedidos:
- Al menos un artículo de línea
- Método de pago
- Cliente (dependiendo de la configuración)

Típicamente requeridos para productos:
- Nombre del producto
- Precio (para productos simples)
