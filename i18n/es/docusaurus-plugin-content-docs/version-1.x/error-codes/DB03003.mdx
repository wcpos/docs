# DB03003: Campo Obligatorio Faltante

## Qué Significa

No se proporcionó un campo obligatorio. La base de datos no puede guardar el registro porque falta información esencial.

## Causas Comunes

- **Formulario incompleto** — No se completaron los campos obligatorios
- **Problema de sincronización de datos** — Los datos requeridos no se sincronizaron correctamente
- **Validación omitida** — Formulario enviado sin validación
- **Campo personalizado requerido** — Un campo personalizado obligatorio está vacío

## Cómo Solucionar

### 1. Completar Todos los Campos Obligatorios

Revisa el formulario en busca de:
- Campos marcados con asteriscos (*)
- Campos resaltados o marcados con error
- Campos vacíos que deberían tener valores

### 2. Actualizar y Volver a Ingresar

Si el estado del formulario es incorrecto:
1. Actualizar la página/pantalla
2. Volver a ingresar toda la información
3. Verificar todos los campos antes de enviar

### 3. Verificar Configuraciones de WooCommerce

Si hay campos personalizados obligatorios:
1. Revisar configuraciones de pago de WooCommerce
2. Verificar campos personalizados obligatorios
3. Asegurarse de que el POS proporcione estos campos

### 4. Sincronizar Datos Requeridos

Si faltan datos relacionados:
- Actualizar productos/clientes/etc.
- Esperar a que se complete la sincronización
- Reintentar la operación

### 5. Revisar Configuraciones de Campos Obligatorios

En el Administrador de WooCommerce:
- Verificar qué campos están marcados como obligatorios
- Considerar si todos son realmente necesarios para el POS
- Ajustar requisitos si es necesario

## Campos Obligatorios Comunes

Típicamente obligatorios para pedidos:
- Al menos un artículo de línea
- Método de pago
- Cliente (según configuración)

Típicamente obligatorios para productos:
- Nombre del producto
- Precio (para productos simples)
