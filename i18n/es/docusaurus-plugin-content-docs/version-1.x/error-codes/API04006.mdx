# API04006: Recurso No Encontrado

## Qué Significa

El servidor no pudo encontrar el recurso solicitado. Esto corresponde típicamente a un código de estado HTTP 404, indicando que el producto, pedido, cliente u otro recurso que está intentando acceder no existe.

## Causas Comunes

- **Registro eliminado** — El recurso fue eliminado en el servidor
- **ID incorrecto** — Se está usando un ID incorrecto o desactualizado
- **Problemas de sincronización** — Los datos locales hacen referencia a un recurso que ya no existe
- **Configuración de URL incorrecta** — El punto final de la API es incorrecto
- **Problemas de enlaces permanentes** — Es necesario actualizar los enlaces permanentes de WordPress

## Mapeo de Error del Servidor

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|-------------|--------|
| `rest_no_route` | API REST de WordPress |
| HTTP 404 | Cualquier respuesta del servidor |

## Cómo Solucionarlo

### 1. Verificar la Existencia del Recurso

Compruebe si el recurso aún existe en su sitio de WooCommerce:
- Inicie sesión en el administrador de WordPress
- Navegue a la sección relevante (Productos, Pedidos, Clientes)
- Busque el elemento por ID o nombre

### 2. Actualizar Datos Locales

Si el recurso fue eliminado en el servidor:
1. Abra la configuración de POS
2. Navegue a la sección de datos relevante
3. Active una sincronización/actualización para actualizar los datos locales
4. El elemento eliminado debe ser removido localmente

### 3. Verificar los Enlaces Permanentes de WordPress

Si no se encuentran varios recursos:
1. Vaya a **Configuración → Enlaces permanentes** en el administrador de WordPress
2. Haga clic en **Guardar Cambios** (incluso sin hacer cambios)
3. Esto actualiza la estructura de enlaces permanentes

### 4. Verificar Rutas de la API

Pruebe la API REST directamente:
```
https://sutienda.com/wp-json/wc/v3/products
```
Si esto devuelve un 404, puede haber un problema de configuración del servidor.

### 5. Verificar Conflictos de Plugins

Si las rutas de la API REST están ausentes:
1. Asegúrese de que WooCommerce esté activo
2. Asegúrese de que el plugin de WCPOS esté activo
3. Deshabilite temporalmente otros plugins para probar
