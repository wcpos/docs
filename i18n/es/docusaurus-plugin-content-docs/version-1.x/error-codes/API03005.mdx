# API03005: Límite de Tasa Excedido

## Qué Significa

Ha realizado demasiadas solicitudes al servidor en un corto período de tiempo. La limitación de tasa protege al servidor de ser abrumado.

## Causas Comunes

- **Operaciones de sincronización rápidas** — Múltiples dispositivos sincronizando simultáneamente
- **Actualización agresiva** — Actualizando datos con demasiada frecuencia
- **Plugin de seguridad** — Un plugin está imponiendo límites de solicitudes
- **Límites de alojamiento** — Su plan de alojamiento tiene límites de tasa API

## Mapeo de Error del Servidor

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|-------------|--------|
| HTTP 429 | Cualquier respuesta del servidor (respaldo) |

## Cómo Solucionar

### 1. Esperar y Reintentar

Los límites de tasa son temporales:
- Espere 1-5 minutos
- El POS reintentará automáticamente
- Evite actualizar repetidamente de forma manual

### 2. Reducir Operaciones Concurrentes

Si usa varios terminales POS:
- Escalone la sincronización inicial entre dispositivos
- Evite actualizar todos los dispositivos al mismo tiempo
- Deje que uno complete antes de iniciar otro

### 3. Revisar Plugins de Seguridad

Plugins como Wordfence pueden bloquear solicitudes rápidas:
- Incluya en la lista blanca la aplicación POS
- Incluya en la lista blanca las IP de sus dispositivos POS
- Ajuste los umbrales de limitación de tasa

### 4. Revisar Límites de Alojamiento

Algunos hosts limitan las solicitudes API:
- Verifique los detalles de su plan de alojamiento
- Considere actualizar si los límites son muy bajos
- Contacte al soporte sobre el aumento de límites

### 5. Optimizar Frecuencia de Sincronización

En la configuración del POS:
- Ajuste los intervalos de actualización automática
- Use sincronización manual cuando sea apropiado
- Evite actualizaciones de datos innecesarias
