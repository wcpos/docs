# API03005: Límite de Tasa Excedido

## Lo Que Esto Significa

Has realizado demasiadas solicitudes al servidor en un corto período. El límite de tasa protege al servidor de ser abrumado.

## Causas Comunes

- **Operaciones de sincronización rápidas** — Múltiples dispositivos sincronizándose simultáneamente
- **Actualización agresiva** — Actualizando datos con demasiada frecuencia
- **Plugin de seguridad** — Un plugin está haciendo cumplir límites a las solicitudes
- **Límites de hospedaje** — Tu plan de hospedaje tiene límites de tasa de API

## Mapeo de Errores del Servidor

Este código de error se activa cuando el servidor retorna:

| Código del Servidor | Fuente |
|---------------------|--------|
| HTTP 429            | Cualquier respuesta del servidor (fallback) |

## Cómo Solucionarlo

### 1. Espera y Vuelve a Intentarlo

Los límites de tasa son temporales:
- Espera de 1 a 5 minutos
- El POS volverá a intentar automáticamente
- Evita actualizar manualmente de forma repetida

### 2. Reducir Operaciones Concurrentes

Si usas múltiples terminales de POS:
- Escalonar la sincronización inicial entre dispositivos
- Evita actualizar todos los dispositivos a la vez
- Deja que uno termine antes de comenzar otro

### 3. Revisa los Plugins de Seguridad

Plugins como Wordfence pueden bloquear solicitudes rápidas:
- Añade la aplicación POS a la lista blanca
- Añade las IPs de tus dispositivos POS a la lista blanca
- Ajusta los umbrales de límite de tasa

### 4. Revisa los Límites de Hospedaje

Algunos hospedadores limitan las solicitudes de API:
- Revisa los detalles de tu plan de hospedaje
- Considera actualizar si los límites son demasiado bajos
- Contacta soporte para aumentar los límites

### 5. Optimiza la Frecuencia de Sincronización

En la configuración del POS:
- Ajusta los intervalos de auto-actualización
- Usa sincronización manual cuando sea apropiado
- Evita actualizaciones de datos innecesarias
