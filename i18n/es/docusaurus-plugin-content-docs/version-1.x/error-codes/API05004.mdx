# API05004: API de WordPress desactivada

## Qué Significa

El API REST de WordPress está desactivada en tu sitio. Toda la funcionalidad moderna de WordPress, incluidos WooCommerce y WCPOS, depende de esta API.

## Causas Comunes

- **Plugin de seguridad** — Un plugin está bloqueando el acceso al API REST
- **Restricción de alojamiento** — Tu proveedor de host ha desactivado el API REST
- **Código personalizado** — Un tema o plugin ha desactivado el API
- **Reglas de firewall** — WAF bloqueando los endpoints del API REST

## Cómo Solucionarlo

### 1. Probar el API REST

Visita `https://tusite.com/wp-json/` en tu navegador:
- Debería devolver JSON con las rutas disponibles
- Si obtienes un error o nada, está bloqueado

### 2. Revisar Plugins de Seguridad

Plugins comunes que bloquean el API REST:

**Wordfence:**
- Firewall → Todas las opciones de firewall
- Desactivar la opción "Desactivar REST API"

**iThemes Security:**
- Seguridad → Configuración → Ajustes de WordPress
- Activar API REST

**Plugin Desactivar REST API:**
- Desactivar completamente este plugin

### 3. Revisar Código Personalizado

Busca en el `functions.php` de tu tema o plugins personalizados:

```php
// Este código desactiva el API REST - elimínalo
add_filter('rest_authentication_errors', function($result) {
    return new WP_Error('rest_disabled', 'REST API disabled');
});
```

### 4. Revisar .htaccess

Elimina cualquier regla que bloquee `/wp-json/`:

```apache
# Malo - bloquea el API REST
RewriteRule ^wp-json - [F,L]
```

### 5. Contactar con el Proveedor de Alojamiento

Algunos hosts bloquean el API REST por defecto:
- Solicitar que lo habiliten
- Preguntar sobre restricciones de seguridad
- Consultar la documentación de alojamiento

## Por Qué Importa el API REST

El API REST de WordPress es esencial para:
- Aplicaciones móviles
- Integraciones de terceros
- Funciones de WooCommerce
- Funcionamiento de WCPOS

Desactivarlo rompe muchas funcionalidades.
