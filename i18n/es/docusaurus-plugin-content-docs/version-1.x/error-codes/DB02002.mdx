# DB02002: Registro No Encontrado

## Lo Que Esto Significa

El registro solicitado no existe en la base de datos local. El POS intentó acceder a un producto, cliente, pedido u otro registro que no esté almacenado localmente.

## Causas Comunes

- **No sincronizado aún** — Los datos no se han descargado todavía
- **Eliminado en el servidor** — El registro fue eliminado en WooCommerce
- **Sincronización no completa** — La sincronización inicial aún se está ejecutando
- **Filtro aplicado** — Registro excluido por filtros activos

## Cómo Arreglarlo

### 1. Esperar la Sincronización

Si se inició recientemente:
- Esperar a que se complete la sincronización inicial
- Verificar el indicador de progreso de sincronización
- El registro debería aparecer una vez sincronizado

### 2. Actualizar/Sincronizar Nuevamente

Provocar una actualización de datos:
- Usar el botón de actualizar/sincronizar
- Extraer los últimos datos del servidor
- Esperar a que se complete

### 3. Verificar WooCommerce

Verificar que el registro exista en el servidor:
- Iniciar sesión en WordPress Admin
- Verificar Productos, Pedidos o Clientes
- Confirmar que el artículo todavía exista

### 4. Verificar Filtros

El registro podría estar filtrado:
- Revisar los filtros activos en el POS
- Verificar los filtros de categoría
- Confirmar configuraciones de visibilidad

### 5. Limpiar y Sincronizar Nuevamente

Si la sincronización parece detenida:
1. Limpiar los datos locales
2. Iniciar sesión nuevamente
3. Completar una nueva sincronización

## Escenarios Comunes

- **Producto no visible:** Verificar si está publicado y visible
- **Cliente no encontrado:** Puede que aún no se haya sincronizado
- **Pedido faltante:** Podría estar fuera del rango de fechas de sincronización
