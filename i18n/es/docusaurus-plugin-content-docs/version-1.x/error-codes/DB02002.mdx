# DB02002: Registro no encontrado

## Qué significa

El registro solicitado no existe en la base de datos local. El POS intentó acceder a un producto, cliente, pedido u otro registro que no está almacenado localmente.

## Causas comunes

- **Aún no sincronizado** — Los datos aún no se han descargado
- **Eliminado en el servidor** — El registro se eliminó en WooCommerce
- **Sincronización incompleta** — La sincronización inicial aún está en curso
- **Filtro aplicado** — Registro excluido por filtros activos

## Cómo solucionarlo

### 1. Esperar la sincronización

Si se acaba de iniciar:
- Espere a que se complete la sincronización inicial
- Verifique el indicador de progreso de sincronización
- El registro debería aparecer una vez sincronizado

### 2. Actualizar/Resincronizar

Inicie una actualización de datos:
- Use el botón de actualización/sincronización
- Obtenga los últimos datos del servidor
- Espere a que se complete

### 3. Verificar WooCommerce

Confirme que el registro existe en el servidor:
- Inicie sesión en el Administrador de WordPress
- Revise Productos, Pedidos o Clientes
- Confirme que el elemento aún existe

### 4. Verificar filtros

El registro podría estar filtrado:
- Revise los filtros activos en el POS
- Verifique los filtros de categoría
- Compruebe la configuración de visibilidad

### 5. Limpiar y resincronizar

Si la sincronización parece estar atascada:
1. Borre los datos locales
2. Inicie sesión de nuevo
3. Complete una sincronización nueva

## Escenarios comunes

- **Producto no aparece:** Verifique si está publicado y visible
- **Cliente no encontrado:** Puede que aún no se haya sincronizado
- **Pedido faltante:** Podría estar fuera del rango de fecha de sincronización
