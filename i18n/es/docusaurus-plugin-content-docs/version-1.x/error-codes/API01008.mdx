# API01008: Sitio Web No Disponible {#api01008-website-unavailable}

## Qué Significa Esto {#what-this-means}

El POS puede alcanzar Internet, pero tu sitio web específico no está respondiendo. El servidor que aloja tu tienda WooCommerce parece estar caído o inalcanzable.

## Causas Comunes {#common-causes}

- **Mantenimiento del servidor** — Tu proveedor de hosting puede estar realizando actualizaciones
- **Caída del servidor** — El servidor web ha dejado de funcionar inesperadamente
- **Límites de recursos excedidos** — Tu cuenta de hosting ha alcanzado sus límites
- **Ataque DDoS** — El servidor está abrumado por tráfico malicioso
- **Dominio/hosting caducado** — Tu hosting o dominio ha expirado

## Cómo Arreglarlo {#how-to-fix}

### 1. Verifica que el Sitio Esté Caído {#1-verify-the-site-is-down}

Intenta acceder a tu sitio directamente en un navegador. Si no se carga, el problema es del lado del servidor.

### 2. Verifica el Estado de tu Proveedor de Hosting {#2-check-hosting-provider-status}

- Visita la página de estado de tu proveedor de hosting
- Comprueba si hay mantenimiento programado
- Busca reportes de cortes

### 3. Revisa la Cuenta de Hosting {#3-review-hosting-account}

Inicia sesión en tu panel de control de hosting:
- Verifica si tu cuenta está activa
- Asegúrate de no haber excedido los límites de recursos
- Busca avisos de suspensión

### 4. Contacta el Soporte de Hosting {#4-contact-hosting-support}

Si todo parece normal de tu lado, contacta a tu proveedor de hosting. Ellos pueden:
- Verificar el estado del servidor
- Revisar los registros del servidor
- Reiniciar servicios si es necesario

### 5. Espera y Vuelve a Intentarlo {#5-wait-and-retry}

Si es una caída temporal:
- Espera 15-30 minutos
- El POS volverá a intentar las conexiones automáticamente
- Los datos en caché permanecen disponibles sin conexión
