# API01008: Sitio web no disponible {#api01008-website-unavailable}

## Qué significa {#what-this-means}

El POS puede acceder a internet, pero el sitio web específico no responde. El servidor que aloja la tienda WooCommerce parece estar caído o inaccesible.

## Causas comunes {#common-causes}

- **Mantenimiento del servidor** — El proveedor de alojamiento puede estar realizando actualizaciones
- **Caída del servidor** — El servidor web se detuvo inesperadamente
- **Límites de recursos excedidos** — La cuenta de alojamiento alcanzó sus límites
- **Ataque DDoS** — El servidor está saturado por tráfico malicioso
- **Dominio/alojamiento expirado** — El alojamiento o el dominio han caducado

## Cómo solucionarlo {#how-to-fix}

### 1. Verificar si el sitio está caído {#1-verify-the-site-is-down}

Intentar acceder al sitio directamente desde un navegador. Si no carga, el problema está en el servidor.

### 2. Consultar el estado del proveedor de hosting {#2-check-hosting-provider-status}

- Visitar la página de estado del proveedor de hosting
- Verificar si hay mantenimiento programado
- Buscar interrupciones reportadas

### 3. Revisar la cuenta de hosting {#3-review-hosting-account}

Iniciar sesión en el panel de control del hosting:
- Verificar que la cuenta esté activa
- Comprobar que no se hayan excedido los límites de recursos
- Buscar avisos de suspensión

### 4. Contactar con el soporte del hosting {#4-contact-hosting-support}

Si todo parece normal, es recomendable contactar al proveedor de hosting. Ellos pueden:
- Verificar el estado del servidor
- Revisar los registros del servidor
- Reiniciar los servicios si es necesario

### 5. Esperar y reintentar {#5-wait-and-retry}

Si se trata de una interrupción temporal:
- Esperar entre 15 y 30 minutos
- El POS reintentará las conexiones automáticamente
- Los datos en caché permanecen disponibles sin conexión
