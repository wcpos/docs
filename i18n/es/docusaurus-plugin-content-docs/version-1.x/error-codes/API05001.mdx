# API05001: API de WooCommerce desactivada

## Qué significa esto

La API REST de WooCommerce está desactivada en su sitio. WCPOS requiere la API REST para comunicarse con WooCommerce y acceder a los datos de la tienda.

## Causas comunes

- **Configuración de WooCommerce** — La API REST se desactivó intencionalmente
- **Plugin de seguridad** — Un plugin de seguridad está bloqueando el acceso a la API
- **Restricción del hosting** — Su proveedor de hosting ha desactivado el acceso a la API REST
- **Problemas de permalink** — Permalinks no configurados para la API REST

## Cómo solucionar

### 1. Habilitar API REST de WooCommerce

En el Administrador de WordPress:
1. Vaya a WooCommerce → Configuración → Avanzado → API REST
2. Asegúrese de que la API REST esté habilitada
3. Verifique que se hayan creado las claves de API

### 2. Verificar la API REST de WordPress

La API de WooCommerce depende de la API REST de WordPress:
1. Visite `https://su sitio.com/wp-json/` en su navegador
2. Debería devolver datos JSON
3. Si no, consulte [API05004](./API05004)

### 3. Verificar plugins de seguridad

Plugins de seguridad comunes que pueden bloquear la API:
- **Wordfence** — Verifique la configuración del firewall
- **iThemes Security** — Verifique la configuración de la API REST
- **All In One WP Security** — Revise las reglas del firewall

Agregue a la lista blanca los puntos finales de la API REST o la aplicación POS.

### 4. Verificar .htaccess

Busque reglas que bloqueen el acceso a la API:

```apache
# Remove or modify rules blocking /wp-json/
# Ensure mod_rewrite is enabled
```

### 5. Actualizar Permalinks

A veces, la configuración de permalinks necesita ser refrescada:
1. Vaya a Configuración → Permalinks
2. Haga clic en "Guardar cambios" (incluso sin hacer cambios)
3. Esto regenerará las reglas de reescritura
