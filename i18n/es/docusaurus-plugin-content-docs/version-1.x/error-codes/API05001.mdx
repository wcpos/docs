# API05001: API de WooCommerce Desactivada

## Qué Significa

La API REST de WooCommerce está desactivada en tu sitio. WCPOS requiere la API REST para comunicarse con WooCommerce y acceder a los datos de la tienda.

## Causas Comunes

- **Configuración de WooCommerce** — La API REST fue desactivada intencionalmente
- **Plugin de seguridad** — Un plugin de seguridad está bloqueando el acceso a la API
- **Restricción de hosting** — Tu host ha desactivado el acceso a la API REST
- **Problemas de enlaces permanentes** — Los enlaces permanentes no están configurados para la API REST

## Cómo Solucionar

### 1. Activar API REST de WooCommerce

En el Administrador de WordPress:
1. Ve a WooCommerce → Configuración → Avanzado → API REST
2. Asegúrate de que la API REST esté habilitada
3. Verifica que las claves de API estén creadas

### 2. Verificar API REST de WordPress

La API de WooCommerce depende de la API REST de WordPress:
1. Visita `https://tusitio.com/wp-json/` en tu navegador
2. Debe devolver datos JSON
3. Si no lo hace, consulta [API05004](./API05004)

### 3. Verificar Plugins de Seguridad

Plugins de seguridad comunes que pueden bloquear la API:
- **Wordfence** — Revisa la configuración del firewall
- **iThemes Security** — Revisa la configuración de la API REST
- **All In One WP Security** — Revisa las reglas del firewall

Permite las direcciones de los endpoints de la API REST o la aplicación POS.

### 4. Verificar .htaccess

Busca reglas que bloqueen el acceso a la API:

```apache
# Elimina o modifica las reglas que bloquean /wp-json/
# Asegúrate de que mod_rewrite esté habilitado
```

### 5. Actualizar Enlaces Permanentes

A veces los ajustes de enlaces permanentes necesitan actualizarse:
1. Ve a Configuración → Enlaces permanentes
2. Haz clic en "Guardar cambios" (incluso sin hacer cambios)
3. Esto regenera las reglas de reescritura
