# API06003: Configuración de Sitio Inválida

## Qué Significa Esto

La configuración del sitio es inválida o incompleta. Esto podría involucrar URLs incorrectas, configuraciones de autenticación u otros problemas de configuración.

## Causas Comunes

- **Configuración parcial** — Faltan algunas configuraciones
- **Configuraciones desajustadas** — La configuración no coincide con el sitio
- **Configuración corrupta** — Las configuraciones fueron corrompidas
- **Cambios en el sitio** — El sitio fue modificado sin actualizar la configuración de POS

## Cómo Solucionar

### 1. Revisar Todas las Configuraciones

Verifica la configuración completa:
- La URL del sitio es correcta
- La autenticación está configurada correctamente
- Cualquier configuración adicional es correcta

### 2. Reconfigurar desde Cero

Si la configuración está corrompida:
1. Limpia todas las configuraciones almacenadas
2. Inicia el proceso de configuración nuevamente
3. Ingresa la nueva configuración

### 3. Verificar los Requisitos del Sitio

Verifica que tu sitio de WordPress cumpla con los requisitos:
- WordPress está instalado y accesible
- WooCommerce está instalado y activo
- El plugin WCPOS está instalado y activo
- Los enlaces permanentes están habilitados (no "Plain")

### 4. Probar el Acceso al Sitio

Verifica que estas URLs funcionen en un navegador:
- `https://yoursite.com/` — Sitio principal
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. Verificar las Configuraciones de WordPress

En el Admin de WordPress:
1. Configuración → General — Verifica las URLs
2. Configuración → Enlaces Permanentes — Asegúrate de que no sea "Plain"
3. WooCommerce → Estado — Verifica si hay problemas

## Requisitos de Configuración

Una configuración válida necesita:
- **URL del Sitio** — URL completa con protocolo
- **Autenticación** — Credenciales o claves API válidas
- **Permisos** — El usuario tiene acceso al POS
- **Plugins** — Los plugins requeridos están activos
