# API06003: Configuración de Sitio Inválida

## Qué Significa

La configuración del sitio no es válida o está incompleta. Esto podría involucrar URLs incorrectas, configuraciones de autenticación u otros problemas de configuración.

## Causas Comunes

- **Configuración parcial** — Faltan algunos ajustes
- **Configuraciones no coincidentes** — La configuración no coincide con el sitio
- **Configuración dañada** — Los ajustes fueron corrompidos
- **Cambios en el sitio** — El sitio fue modificado sin actualizar la configuración de POS

## Cómo Solucionar

### 1. Revisar Todos los Ajustes

Verifique la configuración completa:
- La URL del sitio es correcta
- La autenticación está configurada correctamente
- Cualquier configuración adicional es correcta

### 2. Reconfigurar desde Cero

Si la configuración está dañada:
1. Borrar todos los ajustes almacenados
2. Iniciar el proceso de configuración de nuevo
3. Ingresar una configuración nueva

### 3. Verificar Requisitos del Sitio

Confirmar que su sitio WordPress cumple los requisitos:
- WordPress está instalado y es accesible
- WooCommerce está instalado y activo
- El plugin WCPOS está instalado y activo
- Los permalinks están habilitados (no "Simple")

### 4. Probar Acceso al Sitio

Verificar que estas URLs funcionen en un navegador:
- `https://tusitio.com/` — Sitio principal
- `https://tusitio.com/wp-json/` — API REST
- `https://tusitio.com/wp-json/wcpos/v1/` — API WCPOS

### 5. Verificar Configuración de WordPress

En Administración de WordPress:
1. Ajustes → General — Verificar URLs
2. Ajustes → Enlaces permanentes — Asegurarse de que no sean "Simple"
3. WooCommerce → Estado — Buscar problemas

## Requisitos de Configuración

Una configuración válida necesita:
- **URL del Sitio** — URL completa con protocolo
- **Autenticación** — Credenciales o claves API válidas
- **Permisos** — El usuario tiene acceso a POS
- **Plugins** — Plugins requeridos están activos
