# API02010: Autenticación Requerida

## Lo Que Significa

La acción que está intentando realizar requiere autenticación, pero actualmente no ha iniciado sesión. El POS necesita credenciales válidas para acceder a este recurso.

## Causas Comunes

- **Sin iniciar sesión** — Aún no se ha autenticado
- **Sesión eliminada** — Su sesión fue borrada o expiró
- **Accediendo a recurso protegido** — El recurso requiere autenticación
- **Datos de la aplicación borrados** — Las credenciales almacenadas fueron eliminadas

## Mapeo de Errores del Servidor

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|-------------------|--------|
| `rest_login_required` | API REST de WordPress |
| `jwt_auth_no_auth_header` | Plugin de Autenticación JWT |
| HTTP 401 | Cualquier respuesta del servidor (respaldo) |

## Cómo Solucionar

### 1. Iniciar Sesión

Si no ha iniciado sesión:
1. Abra la pantalla de inicio de sesión del POS
2. Introduzca sus credenciales de WordPress
3. Complete el proceso de autenticación

### 2. Verificar Estado de Sesión

Si creía que había iniciado sesión:
- Es posible que su sesión haya expirado
- Consulte [API02002](./API02002) (Token Expirado) para más detalles
- Inicie sesión de nuevo para restaurar el acceso

### 3. Verificar Configuración del Servidor

Asegúrese de que los endpoints de la API estén configurados correctamente:
- La API REST de WooCommerce debe estar habilitada
- El plugin de WCPOS debe estar activo
- Los endpoints de autenticación deben ser accesibles

### 4. Revisar Problemas del Navegador/Aplicación

Si está siendo desconectado inesperadamente:
- Borre la caché del navegador (si usa la versión web)
- Verifique que las cookies/almacenamiento local no estén bloqueadas
- Confirme que la aplicación tenga permiso para almacenar datos

## Qué Requiere Autenticación

La mayoría de las operaciones del POS requieren autenticación:
- Ver productos y clientes
- Crear y editar pedidos
- Procesar pagos
- Acceder a informes

Solo la pantalla inicial de inicio de sesión es accesible sin autenticación.
