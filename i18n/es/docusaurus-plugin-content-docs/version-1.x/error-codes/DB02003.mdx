# DB02003: Violación de Restricción

## Qué Significa

Los datos que intenta guardar violan las reglas de la base de datos. Las restricciones garantizan la integridad de los datos al hacer cumplir reglas sobre qué datos se pueden almacenar.

## Causas Comunes

- **Datos requeridos faltantes** — Un campo obligatorio está vacío
- **Relación no válida** — Referenciando un registro que no existe
- **Discrepancia de tipo de datos** — Tipo de datos incorrecto para el campo
- **Valor fuera de rango** — El número excede los límites permitidos

## Cómo Solucionar

### 1. Verificar Campos Obligatorios

Asegúrese de que se proporcionen todos los datos requeridos:
- Información del cliente (si es necesario)
- Detalles del producto
- Líneas de artículos de la orden

### 2. Verificar Referencias

Si el error involucra relaciones:
- Asegúrese de que los productos existan
- Verifique que los ID de cliente sean válidos
- Confirme las asignaciones de categorías

### 3. Revisar Valores de Datos

Revise valores no válidos:
- Cantidades negativas donde no están permitidas
- Precios que exceden límites
- Valores de estado no válidos

### 4. Sincronizar Datos Más Recientes

Los datos referenciados pueden estar desincronizados:
- Actualizar datos desde el servidor
- Esperar a que la sincronización se complete
- Reintentar la operación

### 5. Limpiar y Reintentar

Si los datos están corruptos:
1. Limpiar el formulario problemático
2. Volver a ingresar los datos
3. Enviar nuevamente

## Ejemplos Comunes de Restricciones

- **La cantidad debe ser positiva** — No se pueden agregar 0 o artículos negativos
- **El precio debe ser numérico** — No se permiten textos en campos de precio
- **El cliente debe existir** — No se puede asignar una orden a un cliente inexistente
