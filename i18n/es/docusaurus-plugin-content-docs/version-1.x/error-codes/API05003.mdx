# API05003: Plugin de WCPOS Desactualizado

## Qué Significa

El plugin de WCPOS instalado en su servidor está desactualizado y no es compatible con su versión de la aplicación POS. Necesita actualizar el plugin.

## Causas Comunes

- **Plugin no actualizado** — Las actualizaciones automáticas pueden estar desactivadas
- **Actualización fallida** — Un intento anterior de actualización no tuvo éxito
- **Discrepancia de versiones** — La aplicación POS se actualizó pero el plugin no
- **Sitio de pruebas/desarrollo** — Pruebas en una versión anterior

## Cómo Solucionar

### 1. Actualizar el Plugin

En el Administrador de WordPress:
1. Ir a Plugins → Plugins instalados
2. Buscar "WooCommerce POS"
3. Si hay una actualización disponible, hacer clic en "Actualizar ahora"

### 2. Habilitar Actualizaciones Automáticas

Para prevenir discrepancias futuras:
1. Ir a Plugins → Plugins instalados
2. Buscar "WooCommerce POS"
3. Hacer clic en "Habilitar actualizaciones automáticas"

### 3. Actualización Manual

Si la actualización automática falla:
1. Descargar la última versión de [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/)
2. Desactivar el plugin actual
3. Eliminar los archivos antiguos del plugin
4. Subir y activar la nueva versión

### 4. Verificar Errores de Actualización

Si las actualizaciones están fallando:
- Revisar permisos de archivos en wp-content/plugins
- Verificar el espacio en disco disponible
- Buscar problemas de memoria PHP
- Consultar los registros de errores para obtener detalles

### 5. Downgrade de la Aplicación POS (Temporal)

Si no puede actualizar el plugin inmediatamente:
- Usar una versión anterior de la aplicación POS
- Esta es solo una solución temporal
- Planificar actualizar el plugin pronto

## Compatibilidad de Versiones

Mantener siempre ambos componentes actualizados:
- La aplicación POS
- El plugin de WordPress de WCPOS

Consultar las notas de lanzamiento para cualquier cambio importante al actualizar.
