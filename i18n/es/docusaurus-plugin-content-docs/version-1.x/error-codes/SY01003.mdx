# SY01003: Permiso Denegado

## Qué Significa

La aplicación POS no tiene permiso para acceder a un recurso requerido. Esto podría ser un archivo, carpeta o característica del sistema.

## Causas Comunes

- **Permisos de archivos** — No se puede leer o escribir en archivos necesarios
- **Permisos de aplicaciones** — La aplicación carece de los permisos requeridos
- **Carpeta protegida** — Intentando acceder a una ubicación restringida
- **Software de seguridad** — El antivirus está bloqueando el acceso

## Cómo Solucionar

### 1. Ejecutar como Administrador (Windows)

Haga clic derecho en la aplicación y seleccione "Ejecutar como administrador" si es necesario para la configuración inicial.

### 2. Verificar Permisos de la Aplicación

**macOS:**
- Preferencias del Sistema → Seguridad y Privacidad → Privacidad
- Comprobar que la aplicación tiene los permisos necesarios

**Windows:**
- Configuración → Privacidad
- Revisar permisos de aplicaciones

### 3. Verificar Permisos de Carpetas

Asegúrese de que la aplicación pueda acceder a su carpeta de datos:
- El usuario debe ser propietario del directorio de datos de la aplicación
- Los permisos de escritura deben estar habilitados
- No debe haber marcas de solo lectura en la carpeta

### 4. Revisar Configuración del Antivirus

El software de seguridad puede bloquear el acceso:
- Verificar registros del antivirus para bloqueos
- Agregar la aplicación POS a la lista de permitidos
- Incluir en la lista blanca la carpeta de datos de la aplicación

### 5. Reinstalar la Aplicación

Si los permisos están corruptos:
1. Desinstalar la aplicación POS
2. Reiniciar el dispositivo
3. Reinstalar en una ubicación estándar
4. Ejecutar con permisos normales (no de administrador)

## Permisos del Navegador (Versión Web)

Si usa la versión web:
- Permitir almacenamiento local para el sitio
- Habilitar cookies para el sitio
- No usar configuraciones del navegador demasiado restrictivas
