# API01004: Error en la resolución DNS {#api01004-dns-resolution-failed}

## Qué significa {#what-this-means}

El POS no pudo convertir el nombre de dominio del sitio web (por ejemplo, `yourstore.com`) en una dirección IP. El DNS (Domain Name System) funciona como la guía telefónica de internet: si no puede encontrar el registro, no puede establecer la conexión.

## Causas comunes {#common-causes}

- **Error tipográfico en el nombre de dominio** — La URL puede contener un error de escritura
- **Problemas con el servidor DNS** — El servidor DNS no responde
- **Dominio expirado** — Es posible que el registro del dominio haya caducado
- **Propagación DNS** — Los cambios recientes en el DNS aún no se han propagado
- **Caché DNS local** — Información DNS desactualizada almacenada en la caché del dispositivo

## Cómo solucionarlo {#how-to-fix}

### 1. Verificar el nombre de dominio {#1-verify-the-domain-name}

Revisar la URL en busca de errores tipográficos. Intentar acceder al sitio en un navegador.

### 2. Comprobar el estado del dominio {#2-check-domain-status}

Utilizar una consulta WHOIS para verificar:
- El dominio sigue registrado
- No ha expirado
- Los registros DNS están configurados

### 3. Limpiar la caché DNS {#3-clear-dns-cache}

**En el dispositivo:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Probar un DNS alternativo {#4-try-alternative-dns}

Cambiar a un servidor DNS público como:
- Google: `8.8.8.8` y `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Esperar la propagación de DNS {#5-wait-for-dns-propagation}

Si se modificaron recientemente los ajustes de DNS, es necesario esperar entre 24 y 48 horas para la propagación completa.
