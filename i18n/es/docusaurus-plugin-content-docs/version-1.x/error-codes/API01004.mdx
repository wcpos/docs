# API01004: Fallo en la Resolución de DNS

## Qué Significa Esto

El POS no pudo convertir el nombre de dominio de su sitio web (por ejemplo, `yourstore.com`) en una dirección IP. DNS (Sistema de Nombres de Dominio) es como la guía telefónica de internet; si no puede encontrar el listado, no puede establecer la conexión.

## Causas Comunes

- **Error tipográfico en el nombre de dominio** — La URL puede tener un error ortográfico
- **Problemas con el servidor DNS** — Su servidor DNS no está respondiendo
- **Dominio expirado** — La registración del dominio puede haber caducado
- **Propagación de DNS** — Cambios recientes en DNS aún no se han propagado
- **Caché local de DNS** — Información de DNS obsoleta almacenada en su dispositivo

## Cómo Solucionarlo

### 1. Verifique el Nombre de Dominio

Revise la URL en busca de errores tipográficos. Intente acceder al sitio en un navegador.

### 2. Verifique el Estado del Dominio

Utilice una búsqueda WHOIS para verificar:
- El dominio sigue registrado
- No ha expirado
- Los registros DNS están configurados

### 3. Borre la Caché de DNS

**En su dispositivo:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Pruebe un DNS Alternativo

Cambie a un servidor DNS público como:
- Google: `8.8.8.8` y `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Espere la Propagación de DNS

Si recientemente cambió la configuración de DNS, espere de 24 a 48 horas para la propagación completa.
