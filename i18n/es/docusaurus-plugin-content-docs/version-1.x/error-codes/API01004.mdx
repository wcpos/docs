# API01004: Fallo en la Resolución de DNS

## Qué Significa Esto

El sistema POS no pudo convertir el nombre de dominio de su sitio web (por ejemplo, `sutienda.com`) en una dirección IP. DNS (Sistema de Nombres de Dominio) es como el directorio telefónico de internet; si no puede encontrar el registro, no puede establecer la conexión.

## Causas Comunes

- **Error de escritura en el nombre de dominio** — La URL puede tener un error ortográfico
- **Problemas con el servidor DNS** — Su servidor DNS no está respondiendo
- **Dominio caducado** — El registro del dominio puede haber vencido
- **Propagación de DNS** — Los cambios recientes de DNS aún no se han propagado
- **Caché de DNS local** — Información de DNS desactualizada almacenada en su dispositivo

## Cómo Solucionarlo

### 1. Verificar el Nombre de Dominio

Revise cuidadosamente la URL en busca de errores tipográficos. Intente acceder al sitio en un navegador.

### 2. Comprobar el Estado del Dominio

Use una búsqueda WHOIS para verificar:
- El dominio sigue registrado
- No ha expirado
- Los registros DNS están configurados

### 3. Limpiar Caché de DNS

**En su dispositivo:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Probar DNS Alternativos

Cambie a un servidor DNS público como:
- Google: `8.8.8.8` y `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Esperar la Propagación de DNS

Si ha cambiado recientemente la configuración de DNS, espere de 24 a 48 horas para la propagación completa.
