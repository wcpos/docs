# DB01002: Tiempo de Espera de Consulta

## Qué Significa

Una operación de base de datos tomó demasiado tiempo para completarse. La consulta a la base de datos local superó el tiempo máximo permitido.

## Causas Comunes

- **Conjunto de datos grande** — Número muy alto de productos o registros
- **Consulta compleja** — Consulta que involucra muchas condiciones
- **Rendimiento del dispositivo** — El dispositivo está funcionando lentamente
- **Fragmentación de base de datos** — La base de datos necesita optimización

## Cómo Solucionar

### 1. Esperar y Reintentar

La consulta puede tener éxito al reintentarla:
- Cerrar y reabrir el POS
- Intentar la operación de nuevo
- Evitar otras operaciones intensivas simultáneamente

### 2. Cerrar Otras Aplicaciones

Liberar recursos del dispositivo:
- Cerrar pestañas de navegador no utilizadas
- Cerrar otras aplicaciones
- Reiniciar el dispositivo si es necesario

### 3. Reducir el Alcance de Datos

Al buscar:
- Usar términos de búsqueda más específicos
- Aplicar filtros para reducir resultados
- Dividir operaciones grandes en más pequeñas

### 4. Optimizar la Base de Datos

Limpiar y resincronizar:
1. Borrar datos locales (opción de configuración si está disponible)
2. Iniciar sesión nuevamente
3. Permitir que los datos se sincronicen de nuevo

### 5. Verificar el Rendimiento del Dispositivo

Asegurarse de que el dispositivo cumpla los requisitos:
- RAM suficiente
- Velocidad de procesador adecuada
- SSD preferible sobre HDD

## Catálogos Grandes

Si tiene un catálogo de productos muy grande:
- La sincronización inicial toma más tiempo
- Considere usar filtros en el POS
- Algunas operaciones naturalmente requieren más tiempo
- Contacte con soporte para obtener consejos de optimización
