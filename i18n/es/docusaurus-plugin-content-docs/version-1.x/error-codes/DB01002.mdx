# DB01002: Tiempo de expiración de consulta

## Qué significa esto

Una operación de base de datos tardó demasiado en completarse. La consulta a la base de datos local superó el tiempo máximo permitido.

## Causas comunes

- **Conjunto de datos grande** — Número muy grande de productos o registros
- **Consulta compleja** — Consulta que involucra muchas condiciones
- **Rendimiento del dispositivo** — El dispositivo está funcionando lentamente
- **Fragmentación de la base de datos** — La base de datos necesita optimización

## Cómo arreglar

### 1. Esperar y reintentar

La consulta puede tener éxito en el reintento:
- Cerrar y reabrir el POS
- Intentar la operación de nuevo
- Evitar otras operaciones intensivas simultáneamente

### 2. Cerrar otras aplicaciones

Liberar recursos del dispositivo:
- Cerrar pestañas de navegador no utilizadas
- Cerrar otras aplicaciones
- Reiniciar el dispositivo si es necesario

### 3. Reducir el alcance de datos

Si busca:
- Utilizar términos de búsqueda más específicos
- Aplicar filtros para restringir los resultados
- Dividir operaciones grandes en más pequeñas

### 4. Optimizar la base de datos

Limpiar y re-sincronizar:
1. Limpiar los datos locales (opción de configuración si está disponible)
2. Iniciar sesión nuevamente
3. Dejar que los datos se sincronicen de nuevo

### 5. Verificar el rendimiento del dispositivo

Asegúrese de que su dispositivo cumpla con los requisitos:
- RAM suficiente
- Velocidad del procesador adecuada
- SSD preferido sobre HDD

## Catálogos grandes

Si tiene un catálogo de productos muy grande:
- La sincronización inicial tarda más
- Considere utilizar filtros en el POS
- Algunas operaciones naturalmente toman más tiempo
- Contacte al soporte para obtener consejos de optimización
