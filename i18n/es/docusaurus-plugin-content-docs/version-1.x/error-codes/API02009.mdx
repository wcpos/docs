# API02009: Token de Actualización Expirado

## Lo Que Esto Significa

Su token de actualización ha excedido su tiempo de vida y ya no se puede utilizar para obtener nuevos tokens de acceso. A diferencia de los tokens de acceso (que expiran rápidamente), los tokens de actualización suelen durar días o semanas, pero eventualmente también expiran.

## Causas Comunes

- **Inactividad prolongada** — No ha usado el POS durante mucho tiempo
- **Tiempo de vida corto del token** — Servidor configurado con una corta expiración del token de actualización
- **Desviación del reloj** — La hora del dispositivo o del servidor está significativamente desajustada

## Cómo Solucionarlo

### 1. Inicie Sesión Nuevamente

Cuando el token de actualización expira, debe autenticarse nuevamente:
1. Cierre sesión en el POS
2. Ingrese sus credenciales
3. Se emitirá un nuevo token de actualización

### 2. Use el POS Regularmente

Para evitar esto en el futuro:
- Use el POS regularmente para mantener los tokens frescos
- Los tokens se actualizan automáticamente cuando está activo
- Períodos prolongados de inactividad requerirán un nuevo inicio de sesión

### 3. Verifique la Configuración de Hora

Asegúrese de que la hora sea precisa en todos los sistemas:
- Habilite la sincronización automática de hora en su dispositivo
- Verifique que la hora del servidor sea correcta
- Las diferencias de hora pueden causar expiración prematura

### 4. Ajuste el Tiempo de Vida del Token (Avanzado)

Si los tokens expiran demasiado rápido para sus necesidades, un administrador del servidor puede ajustar el tiempo de vida del token de actualización en la configuración de autenticación.
