# API02009: Token de Actualización Caducado

## Qué Significa

Su token de actualización ha superado su tiempo de vida y ya no se puede utilizar para obtener nuevos tokens de acceso. A diferencia de los tokens de acceso (que caducan rápidamente), los tokens de actualización normalmente duran días o semanas, pero eventualmente caducan.

## Causas Comunes

- **Inactividad prolongada** — No ha utilizado el POS durante mucho tiempo
- **Tiempo de vida corto del token** — Servidor configurado con caducidad corta de tokens de actualización
- **Desajuste de tiempo** — Hora del dispositivo o del servidor está significativamente desincronizada

## Cómo Solucionar

### 1. Iniciar Sesión de Nuevo

Cuando el token de actualización caduca, debe autenticarse de nuevo:
1. Cerrar sesión en el POS
2. Introducir sus credenciales
3. Se emitirá un nuevo token de actualización

### 2. Usar el POS Regularmente

Para evitar esto en el futuro:
- Use el POS con regularidad para mantener los tokens actualizados
- Los tokens se actualizan automáticamente cuando está activo
- Los periodos prolongados de inactividad requerirán volver a iniciar sesión

### 3. Verificar Configuración de Hora

Asegure un tiempo preciso en todos los sistemas:
- Habilite la sincronización automática de hora en su dispositivo
- Verifique que la hora del servidor sea correcta
- Las diferencias de tiempo pueden causar caducidad prematura

### 4. Ajustar Tiempo de Vida del Token (Avanzado)

Si los tokens caducan demasiado rápido para sus necesidades, un administrador del servidor puede ajustar el tiempo de vida del token de actualización en la configuración de autenticación.
