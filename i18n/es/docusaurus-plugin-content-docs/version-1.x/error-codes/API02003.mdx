# API02003: Token Inválido

## Qué Significa

El token de autenticación utilizado no es reconocido por el servidor. Esto es diferente de un token caducado — este token nunca fue válido o ha sido revocado.

## Causas Comunes

- **Token revocado** — Un administrador ha revocado la sesión
- **Token corrupto** — Corrupción de datos durante el almacenamiento o transmisión
- **Reinicio del servidor** — Las claves secretas del servidor cambiaron, invalidando todos los tokens
- **Servidor incorrecto** — Token de una instalación de WordPress diferente

## Mapeo de Errores del Servidor

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|---------------------|--------|
| `jwt_auth_invalid_token` | Plugin de Autenticación JWT |

## Cómo Solucionarlo

### 1. Iniciar Sesión de Nuevo

Borre su sesión y autentíquese nuevamente:
1. Cierre sesión completamente
2. Cierre y vuelva a abrir el POS
3. Inicie sesión con sus credenciales

### 2. Borrar Datos de la Aplicación

Si iniciar sesión no funciona:
- Borre la caché/datos de la aplicación POS
- En web: Borre las cookies y almacenamiento local del sitio
- En escritorio: Busque una opción de "Borrar Datos" en la configuración de la aplicación

### 3. Verificar Configuración del Servidor

Si el problema afecta a todos los usuarios:
- Compruebe si las sales de WordPress han cambiado
- Verifique la configuración del plugin de JWT o autenticación
- Revise los cambios recientes del servidor

### 4. Verificar que Está en el Servidor Correcto

Asegúrese de que el POS está configurado para conectarse al sitio WordPress correcto, especialmente si tiene múltiples instalaciones.
