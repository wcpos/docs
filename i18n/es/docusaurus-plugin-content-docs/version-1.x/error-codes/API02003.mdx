# API02003: Token no válido

## Qué significa este error {#what-this-means}

El token de autenticación utilizado no es reconocido por el servidor. Esto es diferente de un token expirado: este token nunca fue válido o ha sido revocado.

## Causas comunes {#common-causes}

- **Token revocado** — Un administrador revocó la sesión
- **Token dañado** — Corrupción de datos durante el almacenamiento o la transmisión
- **Reinicio del servidor** — Las claves secretas del servidor cambiaron, invalidando todos los tokens
- **Servidor incorrecto** — Token de una instalación de WordPress diferente

## Mapeo de errores del servidor {#server-error-mapping}

Este código de error se activa cuando el servidor devuelve:

| Código del servidor | Origen |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT Authentication plugin |

## Cómo solucionarlo {#how-to-fix}

### 1. Inicia sesión nuevamente {#1-log-in-again}

Borra tu sesión y autentícate de nuevo:
1. Cierra sesión por completo
2. Cierra y vuelve a abrir el POS
3. Inicia sesión con tus credenciales

### 2. Borra los datos de la aplicación {#2-clear-application-data}

Si cerrar sesión no funciona:
- Borra la caché/datos de la app POS
- En web: Borra las cookies del navegador y el almacenamiento local del sitio
- En escritorio: Busca una opción "Borrar datos" en la configuración de la app

### 3. Verifica la configuración del servidor {#3-check-server-configuration}

Si el problema afecta a todos los usuarios:
- Comprueba si se cambiaron las sales (salts) de WordPress
- Verifica la configuración del plugin JWT o de autenticación
- Revisa los cambios recientes en el servidor

### 4. Verifica que estás en el servidor correcto {#4-verify-youre-on-the-correct-server}

Asegúrate de que el POS esté configurado para conectarse al sitio de WordPress correcto, especialmente si tienes varias instalaciones.
