# API02003: Token Inválido {#api02003-token-invalid}

## Qué Significa Esto {#what-this-means}

El token de autenticación que se está usando no es reconocido por el servidor. Esto es diferente de un token expirado: este token nunca fue válido o ha sido revocado.

## Causas Comunes {#common-causes}

- **Token revocado** — Un administrador revocó la sesión
- **Token corrompido** — Corrupción de datos durante el almacenamiento o la transmisión
- **Reinicio del servidor** — Claves secretas del servidor cambiadas, invalidando todos los tokens
- **Servidor incorrecto** — Token de una instalación diferente de WordPress

## Mapeo de Errores del Servidor {#server-error-mapping}

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|---------------------|--------|
| `jwt_auth_invalid_token` | Plugin de Autenticación JWT |

## Cómo Reparar {#how-to-fix}

### 1. Inicia Sesión Nuevamente {#1-log-in-again}

Borra tu sesión y autentícate de nuevo:
1. Cierra la sesión por completo
2. Cierra y vuelve a abrir el POS
3. Inicia sesión con tus credenciales

### 2. Borra los Datos de la Aplicación {#2-clear-application-data}

Si cerrar sesión no funciona:
- Limpia la caché/datos de la app POS
- En la web: Limpia las cookies del navegador y el almacenamiento local para el sitio
- En el escritorio: Revisa la configuración de la app para una opción de "Borrar Datos"

### 3. Verifica la Configuración del Servidor {#3-check-server-configuration}

Si el problema afecta a todos los usuarios:
- Verifica si se cambiaron las sales de WordPress
- Verifica la configuración del plugin JWT o de autenticación
- Revisa cambios recientes en el servidor

### 4. Verifica que Estés en el Servidor Correcto {#4-verify-youre-on-the-correct-server}

Asegúrate de que el POS esté configurado para conectarse al sitio de WordPress correcto, especialmente si tienes múltiples instalaciones.
