# API02003: Token no válido {#api02003-token-invalid}

## Qué significa este error {#what-this-means}

El token de autenticación utilizado no es reconocido por el servidor. Esto es diferente de un token expirado: este token nunca fue válido o ha sido revocado.

## Causas comunes {#common-causes}

- **Token revocado** — Un administrador revocó la sesión
- **Token dañado** — Corrupción de datos durante el almacenamiento o la transmisión
- **Reinicio del servidor** — Las claves secretas del servidor cambiaron, invalidando todos los tokens
- **Servidor incorrecto** — Token de una instalación de WordPress diferente

## Mapeo de errores del servidor {#server-error-mapping}

Este código de error se activa cuando el servidor devuelve:

| Server Code | Source |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT Authentication plugin |

## How to Fix {#how-to-fix}

### 1. Log In Again {#1-log-in-again}

Clear your session and authenticate fresh:
1. Log out completely
2. Close and reopen the POS
3. Log in with your credentials

### 2. Clear Application Data {#2-clear-application-data}

If logging out doesn't work:
- Clear the POS app cache/data
- On web: Clear browser cookies and local storage for the site
- On desktop: Check app settings for a "Clear Data" option

### 3. Check Server Configuration {#3-check-server-configuration}

If the issue affects all users:
- Check if WordPress salts were changed
- Verify JWT or authentication plugin settings
- Review recent server changes

### 4. Verify You're on the Correct Server {#4-verify-youre-on-the-correct-server}

Ensure the POS is configured to connect to the right WordPress site, especially if you have multiple installations.
