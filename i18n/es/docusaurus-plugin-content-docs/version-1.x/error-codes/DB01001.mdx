# DB01001: Conexión Fallida

## Qué Significa

El POS no pudo conectarse a su base de datos local. WCPOS utiliza una base de datos local para almacenar productos, clientes y otros datos para un acceso rápido y funcionalidad sin conexión.

## Causas Comunes

- **Almacenamiento lleno** — El dispositivo no tiene espacio de almacenamiento disponible
- **Base de datos corrupta** — Los archivos de la base de datos local están dañados
- **Problema de permisos** — La aplicación no tiene permisos de almacenamiento
- **Almacenamiento del navegador deshabilitado** — IndexedDB está desactivado (versión web)

## Cómo Solucionar

### 1. Verificar Espacio de Almacenamiento

Asegúrese de que su dispositivo tenga almacenamiento disponible:
- Libere espacio si el almacenamiento está lleno
- Elimine aplicaciones o archivos no utilizados
- Verifique el espacio de disco disponible

### 2. Borrar Datos de la Aplicación

Restablecer la base de datos local:
- **Aplicación de escritorio:** Busque "Borrar datos" en configuración
- **Navegador web:** Borrar datos del sitio para el dominio del POS
- **Nota:** Esto requerirá volver a sincronizar todos los datos

### 3. Verificar Configuración del Navegador (Web)

Si usa la versión web:
1. Asegúrese de que IndexedDB esté habilitado
2. No use modo privado/incógnito (almacenamiento limitado)
3. Permita que el sitio almacene datos

### 4. Verificar Permisos

En escritorio:
- Asegúrese de que la aplicación tenga permiso para escribir en su directorio de datos
- Compruebe si el antivirus está bloqueando el acceso a la base de datos

### 5. Reinstalar la Aplicación

Si otras soluciones fallan:
1. Desinstale la aplicación POS
2. Reinicie su dispositivo
3. Vuelva a instalar la aplicación
4. Inicie sesión y sincronice datos

## Recuperación de Datos

Si tiene datos sin guardar:
- La aplicación puede tener cambios pendientes que no se pudieron sincronizar
- Contacte a soporte si necesita ayuda para recuperar datos
- En la mayoría de los casos, los datos se pueden volver a sincronizar desde el servidor
