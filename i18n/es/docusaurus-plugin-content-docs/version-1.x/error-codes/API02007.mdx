# API02007: Fallo al Refrescar el Token

## Qué Significa Esto

El POS intentó renovar automáticamente tu sesión de autenticación, pero la solicitud de renovación falló. Esto significa típicamente que tendrás que iniciar sesión nuevamente.

## Causas Comunes

- **Token de renovación expirado** — El token de renovación ha expirado
- **Token de renovación revocado** — Un administrador invalidó todas las sesiones
- **Error del servidor** — El servidor no pudo procesar la solicitud de renovación
- **Problema de red** — Problema de conexión durante el intento de renovación

## Cómo Solucionar

### 1. Iniciar Sesión Nuevamente

La solución más confiable:
1. Cerrar sesión en el POS
2. Iniciar sesión nuevamente con tus credenciales
3. Esto crea una nueva sesión con nuevos tokens

### 2. Comprobar la Conexión de Red

Si la renovación está fallando debido a problemas de red:
- Verifica tu conexión a internet
- Intenta acceder a tu sitio de WordPress en un navegador
- Espera un momento y vuelve a intentarlo

### 3. Verificar el Estado del Servidor

Si el servidor está teniendo problemas:
- Verifica que WordPress esté funcionando correctamente
- Busca errores de PHP en los registros de tu servidor
- Asegúrate de que el plugin de autenticación esté activo

### 4. Limpiar los Datos de la Sesión

Si los problemas persisten:
- Limpia la caché de la aplicación POS
- Elimina las credenciales guardadas
- Comienza de nuevo con un nuevo inicio de sesión

## Renovación Automática del Token

WCPOS renueva automáticamente los tokens antes de que expiren para mantener tu sesión. Si este proceso falla repetidamente, puede haber un problema de configuración que necesita atención.
