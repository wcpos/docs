# API02007: Error al Actualizar el Token {#api02007-token-refresh-failed}

## Qué Significa Esto {#what-this-means}

El POS intentó renovar automáticamente tu sesión de autenticación, pero la solicitud de actualización falló. Esto típicamente significa que tendrás que iniciar sesión nuevamente.

## Causas Comunes {#common-causes}

- **Token de actualización expirado** — El token de actualización ha expirado
- **Token de actualización revocado** — Un administrador invalidó todas las sesiones
- **Error del servidor** — El servidor no pudo procesar la solicitud de actualización
- **Problema de red** — Problema de conexión durante el intento de actualización

## Cómo Solucionarlo {#how-to-fix}

### 1. Iniciar Sesión Nuevamente {#1-log-in-again}

La solución más confiable:
1. Cierra sesión en el POS
2. Inicia sesión nuevamente con tus credenciales
3. Esto crea una nueva sesión con nuevos tokens

### 2. Comprobar la Conexión de Red {#2-check-network-connection}

Si la actualización falla debido a problemas de red:
- Verifica tu conexión a internet
- Intenta acceder a tu sitio de WordPress en un navegador
- Espera un momento y vuelve a intentarlo

### 3. Verificar el Estado del Servidor {#3-check-server-status}

Si el servidor está teniendo problemas:
- Verifica que WordPress esté funcionando correctamente
- Busca errores de PHP en los registros de tu servidor
- Asegúrate de que el plugin de autenticación esté activo

### 4. Limpiar los Datos de la Sesión {#4-clear-session-data}

Si los problemas persisten:
- Limpia la caché de la aplicación POS
- Elimina las credenciales guardadas
- Comienza de nuevo con un nuevo inicio de sesión

## Actualización Automática del Token {#automatic-token-refresh}

WCPOS actualiza automáticamente los tokens antes de que expiren para mantener tu sesión. Si este proceso falla repetidamente, puede haber un problema de configuración que necesita atención.
