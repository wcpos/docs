# API02007: Error de Actualización de Token

## Qué Significa

El POS intentó renovar automáticamente su sesión de autenticación, pero la solicitud de actualización falló. Esto típicamente significa que necesitará iniciar sesión nuevamente.

## Causas Comunes

- **Token de actualización caducado** — El token de actualización en sí ha caducado
- **Token de actualización revocado** — Un administrador invalidó todas las sesiones
- **Error del servidor** — El servidor no pudo procesar la solicitud de actualización
- **Problema de red** — Problema de conexión durante el intento de actualización

## Cómo Solucionarlo

### 1. Iniciar Sesión Nuevamente

La solución más confiable:
1. Cerrar sesión en el POS
2. Iniciar sesión nuevamente con sus credenciales
3. Esto crea una sesión nueva con tokens nuevos

### 2. Verificar Conexión de Red

Si la actualización falla por problemas de red:
- Verificar la conexión a internet
- Intentar acceder al sitio de WordPress en un navegador
- Esperar un momento e intentar de nuevo

### 3. Verificar Estado del Servidor

Si el servidor tiene problemas:
- Verificar que WordPress esté funcionando correctamente
- Revisar errores de PHP en los registros del servidor
- Asegurarse de que el plugin de autenticación esté activo

### 4. Limpiar Datos de Sesión

Si los problemas persisten:
- Limpiar la caché de la aplicación POS
- Eliminar credenciales almacenadas
- Comenzar de nuevo con un nuevo inicio de sesión

## Actualización Automática de Token

WCPOS actualiza automáticamente los tokens antes de que caduquen para mantener su sesión. Si este proceso falla repetidamente, puede haber un problema de configuración que requiera atención.
