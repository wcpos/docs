# DB02001: Registro Duplicado

## Lo que Esto Significa

La base de datos intentó insertar un registro con un ID que ya existe. Cada registro debe tener un identificador único.

## Causas Comunes

- **Conflicto de sincronización** — Los mismos datos sincronizados dos veces
- **Condición de carrera** — Múltiples operaciones creando el mismo registro
- **Corrupción de datos** — Los ID fueron corrompidos o duplicados
- **Problema de importación** — Importando datos que ya existen

## Cómo Solucionarlo

### 1. Actualizar los Datos

Re-sincronizar desde el servidor:
- Obtener los datos más recientes
- Dejar que el proceso de sincronización resuelva conflictos
- Los datos más recientes deberían tener prioridad

### 2. Reintentar la Operación

Si está creando nuevos registros:
- El sistema puede que ya lo haya creado
- Verificar si el registro existe
- Evitar hacer clic en enviar varias veces

### 3. Limpiar la Caché Local

Restablecer los datos locales:
1. Limpiar la base de datos/cache local
2. Iniciar sesión nuevamente
3. Sincronizar datos de nuevo

### 4. Verificar Duplicados en WooCommerce

Si el problema persiste:
- Verificar WooCommerce en busca de entradas duplicadas
- Buscar productos/pedidos con los mismos ID
- Limpiar cualquier duplicado en el admin

### 5. Reportar Problemas Persistentes

Si esto sucede con frecuencia:
- Anotar qué acciones lo causan
- Verificar si hay patrones
- Informar al soporte con detalles

## Por Qué Esto Es Importante

Los ID duplicados causarían:
- Confusión sobre cuál registro es correcto
- Sobrescritura de datos
- Problemas de sincronización

El error previene la corrupción de datos.
