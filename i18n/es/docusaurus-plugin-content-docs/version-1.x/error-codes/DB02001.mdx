# DB02001: Registro Duplicado

## Qué Significa

La base de datos intentó insertar un registro con un ID que ya existe. Cada registro debe tener un identificador único.

## Causas Comunes

- **Conflicto de sincronización** — Mismos datos sincronizados dos veces
- **Condición de carrera** — Múltiples operaciones creando el mismo registro
- **Corrupción de datos** — IDs fueron corruptos o duplicados
- **Problema de importación** — Importando datos que ya existen

## Cómo Solucionarlo

### 1. Actualizar Datos

Resincronizar desde el servidor:
- Obtener los datos más recientes
- Permitir que el proceso de sincronización resuelva conflictos
- Los datos más nuevos deben tener prioridad

### 2. Reintentar la Operación

Al crear nuevos registros:
- El sistema puede ya haberlo creado
- Verificar si el registro existe
- Evitar hacer clic en enviar múltiples veces

### 3. Limpiar Caché Local

Reiniciar datos locales:
1. Borrar la base de datos/caché local
2. Iniciar sesión nuevamente
3. Sincronizar datos desde cero

### 4. Verificar Duplicados en WooCommerce

Si el problema persiste:
- Revisar WooCommerce por entradas duplicadas
- Buscar productos/pedidos con IDs iguales
- Limpiar cualquier duplicado en el administrador

### 5. Reportar Problemas Persistentes

Si esto ocurre frecuentemente:
- Anotar qué acciones lo causan
- Buscar patrones
- Reportar al soporte con detalles

## Por Qué Es Importante

Registros con IDs duplicados causarían:
- Confusión sobre qué registro es correcto
- Sobrescrituras de datos
- Problemas de sincronización

El error previene la corrupción de datos.
