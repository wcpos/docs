# API04002: Código de Respuesta Inesperado

## Qué Significa Esto

El servidor devolvió un código de estado HTTP que no era esperado para esta solicitud. Los códigos comunes incluyen 500 (error del servidor), 403 (prohibido), 404 (no encontrado), etc.

## Causas Comunes

- **500 Error Interno del Servidor** — PHP se bloqueó o encontró un error
- **403 Prohibido** — Acceso denegado por configuraciones de seguridad
- **404 No Encontrado** — El endpoint no existe
- **502/503/504** — Problemas de puerta de enlace o disponibilidad del servidor

## Cómo Solucionarlo

### Para Errores 500 (Error del Servidor)

1. Verifica los registros de errores de PHP
2. Aumenta el límite de memoria de PHP
3. Busca conflictos de plugins
4. Revisa `wp-content/debug.log`

### Para Errores 403 (Prohibido)

1. Revisa la configuración del plugin de seguridad (Wordfence, Sucuri, etc.)
2. Añade a la lista blanca el POS o tu IP
3. Revisa `.htaccess` en busca de reglas de bloqueo
4. Verifica que ModSecurity no esté bloqueando solicitudes

### Para Errores 404 (No Encontrado)

1. Verifica que el plugin WCPOS esté activo
2. Limpia los enlaces permanentes de WordPress (Configuración → Enlaces Permanentes → Guardar)
3. Comprueba si la API REST está habilitada
4. Verifica la configuración de la URL

### Para Errores 502/503/504 (Problemas de Puerta de Enlace)

1. Contacta a tu proveedor de hosting
2. Espera a que el servidor se recupere
3. Comprueba si el sitio está bajo una carga pesada
4. Verifica que el servidor esté funcionando

### Solución de Problemas General

1. Intenta acceder a tu sitio directamente
2. Revisa el panel de control de hosting en busca de problemas
3. Revisa los registros de acceso del servidor
4. Contacta con el soporte de hosting si es necesario
