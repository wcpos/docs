# API04002: Código de Respuesta Inesperado

## Qué Significa

El servidor devolvió un código de estado HTTP que no se esperaba para esta solicitud. Los códigos comunes incluyen 500 (error del servidor), 403 (prohibido), 404 (no encontrado), etc.

## Causas Comunes

- **500 Error Interno del Servidor** — PHP se bloqueó o encontró un error
- **403 Prohibido** — Acceso denegado por configuraciones de seguridad
- **404 No Encontrado** — El punto de acceso no existe
- **502/503/504** — Problemas de puerta de enlace o disponibilidad del servidor

## Cómo Solucionarlo

### Para Errores 500 (Error del Servidor)

1. Revisar los registros de errores de PHP
2. Aumentar el límite de memoria de PHP
3. Buscar conflictos de plugins
4. Verificar `wp-content/debug.log`

### Para Errores 403 (Prohibido)

1. Revisar configuraciones de plugins de seguridad (Wordfence, Sucuri, etc.)
2. Incluir en lista blanca el POS o su IP
3. Revisar reglas de bloqueo en `.htaccess`
4. Verificar que ModSecurity no esté bloqueando solicitudes

### Para Errores 404 (No Encontrado)

1. Verificar que el plugin de WCPOS esté activo
2. Restablecer enlaces permanentes de WordPress (Configuración → Enlaces permanentes → Guardar)
3. Comprobar si la API REST está habilitada
4. Verificar la configuración de URL

### Para Errores 502/503/504 (Problemas de Puerta de Enlace)

1. Contactar con el proveedor de alojamiento
2. Esperar a que el servidor se recupere
3. Comprobar si el sitio está bajo una carga pesada
4. Verificar que el servidor esté funcionando

### Resolución de Problemas General

1. Intentar acceder al sitio directamente
2. Revisar el panel de control de alojamiento en busca de problemas
3. Revisar registros de acceso del servidor
4. Contactar con soporte de alojamiento si es necesario
