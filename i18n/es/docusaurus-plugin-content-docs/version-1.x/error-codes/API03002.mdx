# API03002: Parámetros Requeridos Faltantes

## Qué Significa

La solicitud carece de datos que el servidor necesita para completar la acción. No se incluyeron campos requeridos en la solicitud.

## Causas Comunes

- **Formulario incompleto** — No se completaron campos obligatorios
- **Datos no guardados** — Los datos del formulario no se capturaron correctamente
- **Discrepancia de versión** — El complemento espera parámetros diferentes a los que envía el POS
- **Campos personalizados** — Faltan campos personalizados requeridos

## Cómo Solucionarlo

### 1. Completar Campos Requeridos

Verificar que toda la información requerida esté completa:
- Detalles del cliente (si es necesario)
- Información del producto
- Detalles del pedido

### 2. Actualizar e Intentar de Nuevo

El estado del formulario puede estar incompleto:
1. Actualizar el POS
2. Volver a ingresar la información requerida
3. Intentar la acción nuevamente

### 3. Verificar Versiones de Complementos

Asegurar compatibilidad:
- Actualizar el complemento de WCPOS
- Actualizar la aplicación POS
- Ambos deben estar en versiones compatibles

### 4. Verificar Configuración de Campos Requeridos

En WooCommerce, algunos campos pueden estar marcados como obligatorios:
- Revisar requisitos de campos de pago
- Verificar configuraciones de campos personalizados
- Ajustar requisitos si es necesario

### 5. Verificar Integraciones Personalizadas

Si se usan complementos o integraciones personalizadas:
- Pueden requerir campos adicionales
- Revisar la documentación del complemento
- Verificar posibles conflictos
