# API03002: Parámetros Requeridos Faltantes

## Qué significa esto

La solicitud está faltando datos que el servidor necesita para completar la acción. Los campos requeridos no se incluyeron en la solicitud.

## Causas Comunes

- **Formulario incompleto** — No se completaron los campos requeridos
- **Datos no guardados** — No se capturaron correctamente los datos del formulario
- **Desajuste de versión** — El plugin espera parámetros diferentes a los que envía el POS
- **Campos personalizados** — Faltan campos personalizados requeridos

## Cómo Solucionar

### 1. Completar Campos Requeridos

Verifique que toda la información obligatoria esté completada:
- Detalles del cliente (si es requerido)
- Información del producto
- Detalles del pedido

### 2. Refrescar e Intentar de Nuevo

El estado del formulario puede estar incompleto:
1. Refresque el POS
2. Vuelva a ingresar la información requerida
3. Intente la acción nuevamente

### 3. Verificar Versiones del Plugin

Asegure la compatibilidad:
- Actualice el plugin WCPOS
- Actualice la aplicación POS
- Ambos deben estar en versiones compatibles

### 4. Verificar Configuraciones de Campos Requeridos

En WooCommerce, algunos campos pueden estar marcados como requeridos:
- Revise los requisitos de campos de checkout
- Verifique las configuraciones de campos personalizados
- Ajuste los requisitos si es necesario

### 5. Verificar Integraciones Personalizadas

Si utiliza plugins o integraciones personalizadas:
- Pueden requerir campos adicionales
- Revise la documentación del plugin
- Verifique si hay conflictos
