# API05005: Complemento no encontrado

## Qué Significa

Un complemento de WordPress requerido no está instalado o activo. Esto podría ser WooCommerce en sí, una extensión requerida o un complemento de integración.

## Causas Comunes

- **WooCommerce desactivado** — WooCommerce fue deshabilitado
- **Extensión requerida faltante** — Una extensión de WooCommerce necesaria no está instalada
- **Conflicto de complementos** — Otro complemento causó la desactivación de un complemento requerido
- **Actualización fallida** — Una actualización de complemento falló y lo dejó desactivado

## Cómo Solucionarlo

### 1. Verificar que WooCommerce esté Activo

WCPOS requiere WooCommerce:
1. Ir a Complementos → Complementos instalados
2. Buscar "WooCommerce"
3. Asegurarse de que esté activado

### 2. Verificar Extensiones Requeridas

Algunas características de WCPOS pueden requerir extensiones:
- Revisar el mensaje de error para el nombre específico del complemento
- Instalar y activar la extensión requerida

### 3. Revisar Conflictos de Complementos

Si los complementos fueron desactivados por conflictos:
1. Revisar notificaciones de correo electrónico de WordPress
2. Revisar registros de errores
3. Resolver conflictos y reactivar

### 4. Verificar Archivos de Complementos

Si faltan archivos de un complemento:
1. Reinstalar el complemento desde WordPress.org
2. O restaurar desde una copia de seguridad
3. Activar el complemento

### 5. Actualizar Todos los Complementos

Los complementos desactualizados pueden causar problemas:
1. Ir a Panel → Actualizaciones
2. Actualizar todos los complementos
3. Verificar que todos los complementos requeridos estén activos

## Complementos Requeridos

Como mínimo, WCPOS requiere:
- **WooCommerce** — La plataforma de comercio electrónico
- **WCPOS** — El complemento de POS

Características adicionales pueden requerir otros complementos.
