# API05005: Plugin No Encontrado {#api05005-plugin-not-found}

## Qué Significa Esto {#what-this-means}

Un plugin de WordPress requerido no está instalado o activo. Esto podría ser WooCommerce en sí, una extensión requerida, o un plugin de integración.

## Causas Comunes {#common-causes}

- **WooCommerce desactivado** — WooCommerce fue deshabilitado
- **Extensión requerida faltante** — Una extensión de WooCommerce necesaria no está instalada
- **Conflicto de plugins** — Otro plugin causó que un plugin requerido se desactivara
- **Actualización fallida** — Una actualización de plugin falló y dejó el plugin desactivado

## Cómo Arreglarlo {#how-to-fix}

### 1. Verificar que WooCommerce esté Activo {#1-verify-woocommerce-is-active}

WCPOS requiere WooCommerce:
1. Ir a Plugins → Plugins Instalados
2. Buscar "WooCommerce"
3. Asegurarse de que esté activado

### 2. Verificar Extensiones Requeridas {#2-check-for-required-extensions}

Algunas características de WCPOS pueden requerir extensiones:
- Revisar el mensaje de error para el nombre específico del plugin
- Instalar y activar la extensión requerida

### 3. Revisar Conflictos de Plugins {#3-review-plugin-conflicts}

Si los plugins fueron desactivados debido a conflictos:
1. Revisar tu correo electrónico para notificaciones de WordPress
2. Revisar los registros de error
3. Resolver conflictos y reactivar

### 4. Revisar Archivos del Plugin {#4-check-plugin-files}

Si faltan los archivos de un plugin:
1. Reinstalar el plugin desde WordPress.org
2. O restaurar desde la copia de seguridad
3. Activar el plugin

### 5. Actualizar Todos los Plugins {#5-update-all-plugins}

Los plugins desactualizados pueden causar problemas:
1. Ir a Escritorio → Actualizaciones
2. Actualizar todos los plugins
3. Verificar que todos los plugins requeridos estén activos

## Plugins Requeridos {#required-plugins}

Como mínimo, WCPOS requiere:
- **WooCommerce** — La plataforma de comercio electrónico
- **WCPOS** — El plugin POS

Características adicionales pueden requerir otros plugins.
