# API04005: Recuperación de JSON Intentada

## Qué Significa

El servidor envió una respuesta con contenido JSON inválido, pero el POS intentó recuperarse y extraer datos válidos. Este es un aviso informativo y no un error crítico.

## Qué Sucedió

El POS detectó:
1. La respuesta contenía contenido adicional antes o después del JSON
2. Los datos JSON centrales seguían siendo identificables
3. Se intentó la recuperación extrayendo la porción válida de JSON

## Causas Comunes

- **Notificaciones de PHP en la salida** — Advertencias de PHP mezcladas con JSON
- **Salida de depuración** — Depuración de desarrollo dejada habilitada
- **Notificaciones de plugins** — Otros plugins generando notificaciones
- **Problemas de espacios en blanco** — Espacios adicionales alrededor del JSON

## Cómo Solucionar

### 1. Desactivar Mostrar Errores de PHP

En `wp-config.php`:
```php
define('WP_DEBUG_DISPLAY', false);
ini_set('display_errors', 0);
```

### 2. Habilitar Registro de Errores en su Lugar

Mantener errores registrados para depuración:
```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
```

### 3. Verificar Modo de Depuración de Plugins

Algunos plugins tienen modos de depuración que generan contenido adicional:
- Revisar configuraciones de plugins
- Desactivar modos de depuración/desarrollo
- Verificar opciones de registro detallado

### 4. Revisar Cambios Recientes

Si esto comenzó recientemente:
- ¿Qué cambió en su servidor?
- ¿Se actualizaron plugins?
- ¿Se modificaron configuraciones de PHP?

## ¿Es Grave?

Aunque el POS se recuperó de este problema, indica un problema de configuración que debe corregirse. El proceso de recuperación:
- Puede no funcionar siempre
- Añade sobrecarga de procesamiento
- Podría enmascarar otros problemas

Corrija la causa subyacente para garantizar un funcionamiento confiable.
