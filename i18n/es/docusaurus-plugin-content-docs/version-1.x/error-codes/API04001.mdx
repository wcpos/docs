# API04001: Formato de Respuesta No Válido

## Qué Significa

El servidor respondió, pero el formato de respuesta no es el que esperaba el POS. El servidor debería devolver datos JSON, pero se recibió otra cosa.

## Causas Comunes

- **Error de PHP mostrado** — Se está mostrando un error de PHP antes del JSON
- **Conflicto de plugin** — Otro plugin está generando contenido
- **Modo de mantenimiento** — El sitio está mostrando una página de mantenimiento
- **Tipo de contenido incorrecto** — El servidor envía HTML en lugar de JSON
- **Problema de caché** — Se está sirviendo una página de error en caché

## Cómo Solucionar

### 1. Revisar el Sitio

Visite su sitio WordPress en un navegador:
- ¿Se muestra normalmente?
- ¿Hay errores visibles?
- ¿Está en modo de mantenimiento?

### 2. Verificar Errores de PHP

En `wp-config.php`, habilite temporalmente la depuración:

```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```

Revise `wp-content/debug.log` para ver errores.

### 3. Probar la API REST Directamente

Visite `https://yoursite.com/wp-json/` en su navegador:
- Debería devolver datos JSON
- Si ve HTML o errores, hay un problema
- Verifique si hay salida relacionada con plugins

### 4. Deshabilitar Caché Temporalmente

Los plugins de caché pueden servir respuestas obsoletas:
- Limpie todas las cachés
- Deshabilite temporalmente los plugins de caché
- Excluya la API REST de la caché

### 5. Verificar Conflictos de Plugins

Si un plugin genera contenido en cada página:
1. Deshabilite todos los plugins no esenciales
2. Pruebe el POS
3. Vuelva a habilitar los plugins uno por uno
