# API02002: Token Expirado {#api02002-token-expired}

## Lo Que Esto Significa {#what-this-means}

Tu sesión de autenticación ha expirado. Por razones de seguridad, las sesiones de inicio de sesión no duran para siempre. El POS necesita refrescar tu autenticación o puede que necesites iniciar sesión nuevamente.

## Causas Comunes {#common-causes}

- **Larga inactividad** — No has utilizado el POS durante un período prolongado
- **Tiempo de vida del token del servidor** — La expiración del token del servidor está configurada muy corta
- **Desajuste de tiempo del servidor** — Los relojes del servidor y del dispositivo no están sincronizados

## Mapeo de Errores del Servidor {#server-error-mapping}

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|---------------------|--------|
| `jwt_auth_expired_token` | Plugin de Autenticación JWT |

## Cómo Solucionarlo {#how-to-fix}

### 1. Iniciar Sesión Nuevamente {#1-log-in-again}

La solución más simple es cerrar sesión e iniciar sesión nuevamente:
1. Cierra sesión del POS
2. Ingresa tus credenciales nuevamente
3. Esto crea una nueva sesión

### 2. Verificar Refresco del Token {#2-check-token-refresh}

El POS debería refrescar los tokens automáticamente. Si no está funcionando:
- Revisa tu conexión a internet
- Verifica que el servidor sea accesible
- Consulta [API02007](./API02007) si el refresco está fallando

### 3. Verificar el Tiempo del Servidor {#3-check-server-time}

La expiración del token depende de la hora precisa:
- Asegúrate de que el reloj de tu servidor sea correcto
- Verifica que la hora de tu dispositivo sea precisa
- Usa la sincronización automática de hora en ambos

### 4. Revisar la Configuración del Token {#4-review-token-settings}

Si los tokens expiran demasiado rápido, puede que necesites ajustar la configuración del servidor (consulta a tu proveedor de hosting o administrador de WordPress).
