# API02002: Token Caducado

## Qué Significa

Su sesión de autenticación ha caducado. Por razones de seguridad, las sesiones de inicio de sesión no duran para siempre. El POS necesita actualizar su autenticación o es posible que deba iniciar sesión de nuevo.

## Causas Comunes

- **Inactividad prolongada** — No ha utilizado el POS durante un período extendido
- **Duración del token del servidor** — La caducidad del token del servidor está configurada muy corta
- **Discrepancia de hora del servidor** — Los relojes del servidor y del dispositivo no están sincronizados

## Mapeo de Error del Servidor

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|-------------------|--------|
| `jwt_auth_expired_token` | Plugin de Autenticación JWT |

## Cómo Solucionarlo

### 1. Iniciar Sesión de Nuevo

La solución más sencilla es cerrar y volver a iniciar sesión:
1. Cerrar sesión en el POS
2. Introducir sus credenciales de nuevo
3. Esto crea una sesión nueva

### 2. Verificar Actualización de Token

El POS debería actualizar los tokens automáticamente. Si no funciona:
- Comprobar la conexión a internet
- Verificar que el servidor sea accesible
- Consultar [API02007](./API02007) si la actualización está fallando

### 3. Verificar Hora del Servidor

La caducidad del token depende de una hora precisa:
- Asegurarse de que el reloj del servidor sea correcto
- Comprobar que la hora del dispositivo sea precisa
- Usar sincronización automática de hora en ambos

### 4. Revisar Configuración de Tokens

Si los tokens caducan demasiado rápido, es posible que necesite ajustar la configuración del servidor (consulte a su proveedor de alojamiento o al administrador de WordPress).
