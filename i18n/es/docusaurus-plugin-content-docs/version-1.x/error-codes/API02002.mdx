# API02002: Token expirado {#api02002-token-expired}

## Qué significa este error {#what-this-means}

La sesión de autenticación ha expirado. Por motivos de seguridad, las sesiones de inicio de sesión no duran indefinidamente. El POS necesita actualizar la autenticación o puede ser necesario iniciar sesión de nuevo.

## Causas comunes {#common-causes}

- **Inactividad prolongada** — No se ha utilizado el POS durante un período extenso
- **Duración del token en el servidor** — La expiración del token en el servidor está configurada con un valor muy corto
- **Desfase horario del servidor** — Los relojes del servidor y del dispositivo no están sincronizados

## Mapeo de errores del servidor {#server-error-mapping}

Este código de error se activa cuando el servidor devuelve:

| Código del servidor | Origen |
|-------------|--------|
| `jwt_auth_expired_token` | Plugin JWT Authentication |

## Cómo solucionarlo {#how-to-fix}

### 1. Iniciar sesión de nuevo {#1-log-in-again}

La solución más sencilla es cerrar sesión y volver a iniciar sesión:
1. Cerrar sesión en el POS
2. Introducir las credenciales de nuevo
3. Esto crea una sesión nueva

### 2. Verificar la renovación del token {#2-check-token-refresh}

El POS debería renovar los tokens automáticamente. Si no funciona:
- Comprobar la conexión a internet
- Verificar que el servidor es accesible
- Consultar [API02007](./API02007) si la actualización del token está fallando

### 3. Verificar la hora del servidor {#3-check-server-time}

La expiración del token depende de una hora precisa:
- Asegurarse de que el reloj del servidor sea correcto
- Verificar que la hora del dispositivo sea precisa
- Utilizar la sincronización automática de hora en ambos

### 4. Revisar la configuración de tokens {#4-review-token-settings}

Si los tokens expiran demasiado rápido, puede ser necesario ajustar la configuración del servidor (consultar con el proveedor de hosting o el administrador de WordPress).
