# API04004: Datos de Respuesta Faltantes

## Qué Significa

El servidor respondió correctamente, pero la respuesta carece de los datos esperados. El JSON es válido pero no contiene la información que necesita el POS.

## Causas Comunes

- **Resultados vacíos** — No hay datos que coincidan con la consulta
- **Restricciones de permisos** — Datos filtrados por permisos
- **Filtrado de plugins** — Otro plugin filtrando respuestas de API
- **Discrepancia de versiones** — Diferencias en la versión de API
- **Problemas de base de datos** — Datos no presentes en WooCommerce

## Cómo Solucionar

### 1. Verificar Existencia de Datos

Revisar en el Administrador de WordPress:
- ¿Hay productos en WooCommerce?
- ¿Hay clientes para cargar?
- ¿Existe el elemento específico?

### 2. Verificar Permisos de Usuario

Su usuario puede no tener acceso a todos los datos:
- Verificar capacidades del rol de usuario
- Comprobar configuraciones de acceso de POS
- Intentar con una cuenta de administrador

### 3. Verificar Filtros de Respuesta de API

Algunos plugins filtran respuestas REST API:
- Deshabilitar plugins de filtrado temporalmente
- Verificar filtros personalizados de API en el tema
- Revisar configuraciones de plugins de seguridad

### 4. Actualizar Ambos Componentes

Las discrepancias de versiones pueden causar problemas:
- Actualizar plugin de WCPOS
- Actualizar la aplicación POS
- Verificar notas de compatibilidad

### 5. Verificar Datos de WooCommerce

En WooCommerce:
- Verificar que los productos estén publicados (no en borrador)
- Comprobar si los elementos están marcados como visibles
- Asegurar que los datos no estén corruptos

## Vacío vs. Faltante

- **Respuesta vacía** — Respuesta válida sin resultados (puede ser esperado)
- **Campos faltantes** — Respuesta carece de campos de datos requeridos (este error)
