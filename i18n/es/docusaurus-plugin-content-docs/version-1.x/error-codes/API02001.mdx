# API02001: Credenciales no válidas {#api02001-invalid-credentials}

## Qué significa {#what-this-means}

El nombre de usuario o la contraseña introducidos son incorrectos. El servidor rechazó el intento de inicio de sesión porque las credenciales no coinciden con ninguna cuenta válida.

## Causas frecuentes {#common-causes}

- **Error tipográfico en el nombre de usuario o la contraseña** — Verificar si el bloqueo de mayúsculas está activado o si hay espacios adicionales
- **Cuenta incorrecta** — Se están utilizando credenciales de un sitio diferente
- **Contraseña modificada recientemente** — La contraseña fue actualizada, pero el POS aún tiene la anterior
- **Cuenta eliminada** — La cuenta de usuario ya no existe

## Correspondencia de errores del servidor {#server-error-mapping}

Este código de error se activa cuando el servidor devuelve:

| Código del servidor | Origen |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | Plugin JWT Authentication |

## Cómo solucionarlo {#how-to-fix}

### 1. Verificar las credenciales {#1-verify-your-credentials}

- Comprobar el nombre de usuario (generalmente el correo electrónico o el nombre de usuario de WordPress)
- Verificar que la contraseña sea correcta
- Intentar iniciar sesión directamente en el administrador de WordPress para confirmar

### 2. Restablecer la contraseña {#2-reset-your-password}

Si se ha olvidado la contraseña:
1. Ir a la página de inicio de sesión de WordPress
2. Hacer clic en "¿Olvidó su contraseña?"
3. Introducir el correo electrónico para recibir un enlace de restablecimiento
4. Actualizar la contraseña
5. Iniciar sesión en el POS con la nueva contraseña

### 3. Verificar la cuenta de usuario {#3-check-the-user-account}

En el administrador de WordPress:
- Ir a Usuarios → Todos los usuarios
- Verificar que la cuenta existe
- Comprobar que tiene los permisos de acceso de POS adecuados

### 4. Borrar las credenciales guardadas {#4-clear-saved-credentials}

En el POS:
- Cerrar sesión completamente
- Eliminar cualquier información de inicio de sesión guardada o en caché
- Volver a introducir las credenciales desde cero
