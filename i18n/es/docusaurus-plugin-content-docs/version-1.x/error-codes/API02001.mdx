# API02001: Credenciales no válidas

## Lo que significa

El nombre de usuario o contraseña que ha introducido es incorrecto. El servidor ha rechazado el intento de inicio de sesión porque las credenciales no coinciden con ninguna cuenta válida.

## Causas comunes

- **Error de escritura en nombre de usuario o contraseña** — Verifique el bloqueo de mayúsculas o espacios adicionales
- **Cuenta incorrecta** — Usando credenciales de un sitio diferente
- **Contraseña cambiada recientemente** — La contraseña se actualizó pero el POS tiene la antigua
- **Cuenta eliminada** — La cuenta de usuario ya no existe

## Mapeo de errores del servidor

Este código de error se activa cuando el servidor devuelve:

| Código del servidor | Fuente |
|-------------|--------|
| `woocommerce_rest_authentication_error` | API REST de WooCommerce |
| `jwt_auth_failed` | Plugin de Autenticación JWT |

## Cómo solucionarlo

### 1. Verificar sus credenciales

- Verifique su nombre de usuario (generalmente correo electrónico o nombre de usuario de WordPress)
- Confirme que la contraseña sea correcta
- Intente iniciar sesión directamente en el administrador de WordPress para confirmar

### 2. Restablecer su contraseña

Si ha olvidado su contraseña:
1. Vaya a la página de inicio de sesión de WordPress
2. Haga clic en "¿Perdió su contraseña?"
3. Ingrese su correo electrónico para recibir un enlace de restablecimiento
4. Actualice su contraseña
5. Inicie sesión en el POS con la nueva contraseña

### 3. Verificar la cuenta de usuario

En el Administrador de WordPress:
- Vaya a Usuarios → Todos los usuarios
- Verifique que la cuenta exista
- Compruebe que tenga los permisos de acceso de POS apropiados

### 4. Borrar credenciales guardadas

En el POS:
- Cierre sesión completamente
- Borre cualquier información de inicio de sesión guardada o en caché
- Vuelva a introducir las credenciales desde cero
