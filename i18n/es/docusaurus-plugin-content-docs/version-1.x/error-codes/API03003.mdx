# API03003: Valor de Parámetro Inválido

## Qué Significa Esto

Uno de los valores en tu solicitud es inválido. El formato de datos, tipo o valor no coincide con lo que el servidor espera.

## Causas Comunes

- **Tipo de dato incorrecto** — Texto donde se espera un número (o viceversa)
- **Fuera de rango** — El valor excede los límites permitidos
- **Formato inválido** — Correo electrónico, teléfono u otros campos con formato son incorrectos
- **Referencia inválida** — Referenciando un ID que no existe

## Mapeo de Errores del Servidor

Este código de error se activa cuando el servidor retorna:

| Código del Servidor | Fuente |
|---------------------|--------|
| `rest_invalid_param` | API REST de WordPress |
| `woocommerce_rest_invalid_id` | API REST de WooCommerce |

## Cómo Solucionar

### 1. Verificar los Valores de Entrada

Revisa los datos que estás enviando:
- **Precios** — Deben ser números válidos
- **Cantidades** — Deben ser enteros positivos
- **Correos electrónicos** — Deben tener un formato de correo electrónico válido
- **IDs** — Deben referirse a registros existentes

### 2. Buscar Errores en Campos Específicos

El error puede indicar qué campo es inválido:
- Revisa los detalles del mensaje de error
- Corrige el campo específico
- Reintenta la acción

### 3. Verificar que los Datos Referenciados Existan

Si estás referenciando productos, clientes u otros registros:
- Asegúrate de que el ítem exista en WooCommerce
- Verifica si fue eliminado recientemente
- Sincroniza los datos para actualizar los registros locales

### 4. Verificar las Restricciones de Campos

WooCommerce puede tener restricciones:
- Valores máximos/mínimos
- Formatos requeridos
- Opciones permitidas para campos seleccionables

### 5. Borrar y Volver a Ingresar

A veces los datos se corrompen:
- Borra el campo problemático
- Vuelve a ingresar el valor desde cero
- Evita copiar y pegar de otras fuentes
