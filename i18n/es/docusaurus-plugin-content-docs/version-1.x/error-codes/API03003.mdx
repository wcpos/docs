# API03003: Valor de Parámetro No Válido

## Qué Significa

Uno de los valores en su solicitud no es válido. El formato, tipo o valor de los datos no coincide con lo que el servidor espera.

## Causas Comunes

- **Tipo de dato incorrecto** — Texto donde se espera un número (o viceversa)
- **Fuera de rango** — El valor supera los límites permitidos
- **Formato no válido** — Campos de correo electrónico, teléfono u otros con formato incorrecto
- **Referencia no válida** — Haciendo referencia a un ID que no existe

## Mapeo de Errores del Servidor

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|-------------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## Cómo Solucionar

### 1. Verificar Valores de Entrada

Revise los datos que está enviando:
- **Precios** — Deben ser números válidos
- **Cantidades** — Deben ser enteros positivos
- **Correos electrónicos** — Deben tener un formato de correo válido
- **IDs** — Deben hacer referencia a registros existentes

### 2. Buscar Errores de Campos Específicos

El error puede indicar qué campo no es válido:
- Revise los detalles del mensaje de error
- Corrija el campo específico
- Reintente la acción

### 3. Verificar que los Datos Referenciados Existan

Al hacer referencia a productos, clientes u otros registros:
- Asegúrese de que el elemento exista en WooCommerce
- Compruebe si fue eliminado recientemente
- Sincronice los datos para actualizar los registros locales

### 4. Comprobar Restricciones de Campos

WooCommerce puede tener restricciones:
- Valores máximos/mínimos
- Formatos requeridos
- Opciones permitidas para campos de selección

### 5. Limpiar y Volver a Ingresar

A veces los datos pueden corromperse:
- Limpie el campo problemático
- Vuelva a ingresar el valor desde cero
- Evite copiar y pegar de otras fuentes
