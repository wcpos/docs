# API03001: Formato de Solicitud Inválido

## Qué Significa

La solicitud enviada al servidor no estaba en el formato esperado. El servidor no pudo entender lo que el POS estaba solicitando porque la estructura de la solicitud era incorrecta.

## Causas Comunes

- **Datos corruptos** — Los datos se corrompieron antes de enviarlos
- **Error de software** — Un problema en la aplicación POS
- **Interferencia de proxy** — Un proxy o firewall modificó la solicitud
- **Problemas de codificación de caracteres** — Los caracteres especiales no se codificaron correctamente

## Mapeo de Errores del Servidor

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|-------------|--------|
| HTTP 400 | Cualquier respuesta del servidor (alternativa) |

## Cómo Solucionarlo

### 1. Reintentar la Acción

A veces es un problema momentáneo:
- Espere un momento e inténtelo de nuevo
- Actualice el POS y vuelva a intentarlo

### 2. Verificar Caracteres Especiales

Si está ingresando datos con caracteres especiales:
- Intente eliminar emojis o símbolos inusuales
- Use caracteres estándar para nombres de productos, etc.

### 3. Actualizar el POS

Asegúrese de estar ejecutando la versión más reciente:
- Busque actualizaciones de la aplicación
- Actualice el plugin WCPOS en su servidor

### 4. Verificar Configuración de Red

Si está detrás de un proxy:
- Verifique que el proxy no esté modificando solicitudes
- Compruebe las reglas del firewall
- Intente acceder desde una red diferente

### 5. Reportar el Problema

Si esto ocurre consistentemente:
- Anote qué acción desencadena el error
- Revise la consola del navegador para obtener detalles (versión web)
- Reporte en [GitHub](https://github.com/wcpos) con pasos para reproducir
