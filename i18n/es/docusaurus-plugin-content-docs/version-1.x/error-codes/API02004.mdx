# API02004: Usuario No Autorizado

## Qué Significa

Está conectado, pero su cuenta de usuario no tiene permiso para realizar la acción solicitada. Este es un problema de autorización (lo que puede hacer) en lugar de un problema de autenticación (quién es usted).

## Causas Comunes

- **Limitaciones de rol de usuario** — Su rol de WordPress no incluye acceso a POS
- **Acceso a POS deshabilitado** — Su cuenta no recibió permisos de POS
- **Restricciones de características** — Ciertas funciones están limitadas a roles específicos
- **Restricciones de tienda** — Puede que no tenga acceso a esta tienda en particular

## Mapeo de Error del Servidor

Este código de error se activa cuando el servidor devuelve:

| Código del Servidor | Fuente |
|-------------|--------|
| `rest_cannot_view` | API REST de WordPress |
| `woocommerce_rest_cannot_view` | API REST de WooCommerce |

## Cómo Solucionarlo

### 1. Verificar Rol de Usuario

En Administración de WordPress → Usuarios:
1. Busque su cuenta de usuario
2. Verifique el rol (por ejemplo, Administrador de Tienda, Administrador)
3. Asegúrese de que el rol incluya capacidades de WooCommerce

### 2. Habilitar Acceso a POS

En Administración de WordPress → WooCommerce → POS → Acceso:
1. Busque el usuario o rol
2. Habilite los permisos de acceso a POS
3. Guardar cambios

### 3. Solicitar Permisos Adicionales

Contacte al administrador de la tienda para:
- Otorgar acceso a POS a su rol
- Asignarle un rol con permisos apropiados
- Habilitar características específicas que necesite

### 4. Verificar Asignación de Tienda

Si usa múltiples tiendas:
- Verifique que está asignado a la tienda correcta
- Compruebe los permisos específicos de la tienda

## Permisos Requeridos

Diferentes acciones requieren diferentes capacidades:
- **Ver productos**: Acceso de lectura a productos
- **Crear pedidos**: Capacidades de crear/editar pedidos
- **Gestionar clientes**: Capacidades de gestión de clientes
- **Acceder a informes**: Capacidades de ver informes
