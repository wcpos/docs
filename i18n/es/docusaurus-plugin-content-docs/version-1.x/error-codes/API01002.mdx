# API01002: Conexión Rechazada {#api01002-connection-refused}

## Qué Significa Esto {#what-this-means}

El servidor rechazó activamente el intento de conexión. Esto es diferente de un tiempo de espera: el servidor respondió, pero dijo "no".

## Causas Comunes {#common-causes}

- **Servidor caído** — El servidor web (Apache/Nginx) puede no estar funcionando
- **Puerto incorrecto** — Intentando conectarse en el puerto incorrecto
- **Cortafuegos bloqueando** — Un cortafuegos está bloqueando la conexión
- **Restricciones de IP** — Tu dirección IP puede estar bloqueada

## Cómo Solucionarlo {#how-to-fix}

### 1. Verifica que el Servidor Esté en Funcionamiento {#1-verify-the-server-is-running}

Intenta acceder a tu sitio de WordPress directamente en un navegador. Si no se carga, contacta a tu proveedor de hosting.

### 2. Verifica la Configuración de la URL {#2-check-the-url-configuration}

Asegúrate de estar utilizando la URL correcta:
- Verifica `http://` vs `https://`
- Revisa si hay errores tipográficos en el nombre de dominio
- Confirma el puerto correcto si usas uno no estándar

### 3. Verifica la Configuración del Cortafuegos {#3-check-firewall-settings}

Si tienes acceso a tu servidor:
- Verifica que el puerto 80 (HTTP) o 443 (HTTPS) esté abierto
- Comprueba si tu IP está en la lista de permitidos
- Revisa cualquier complemento de seguridad que pueda bloquear el acceso a la API

### 4. Contacta a tu Proveedor de Hosting {#4-contact-your-hosting-provider}

Ellos pueden verificar:
- Estado y registros del servidor
- Configuraciones del cortafuegos
- Si tu IP ha sido bloqueada
