# API01002: Conexión rechazada {#api01002-connection-refused}

## Qué significa {#what-this-means}

El servidor rechazó activamente el intento de conexión. Esto es diferente de un tiempo de espera agotado: el servidor respondió, pero dijo "no".

## Causas comunes {#common-causes}

- **El servidor está caído** — Es posible que el servidor web (Apache/Nginx) no esté en ejecución
- **Puerto incorrecto** — Se está intentando conectar a un puerto incorrecto
- **Bloqueo del cortafuegos** — Un cortafuegos está bloqueando la conexión
- **Restricciones de IP** — Es posible que la dirección IP esté bloqueada

## Cómo solucionarlo {#how-to-fix}

### 1. Verificar que el servidor esté en ejecución {#1-verify-the-server-is-running}

Intente acceder al sitio WordPress directamente en un navegador. Si no carga, contacte a su proveedor de hosting.

### 2. Verificar la configuración de la URL {#2-check-the-url-configuration}

Asegúrese de estar utilizando la URL correcta:
- Verifique `http://` vs `https://`
- Compruebe que no haya errores tipográficos en el nombre del dominio
- Confirme el puerto correcto si se utiliza uno no estándar

### 3. Verificar la configuración del firewall {#3-check-firewall-settings}

Si tiene acceso al servidor:
- Verifique que el puerto 80 (HTTP) o 443 (HTTPS) esté abierto
- Compruebe si su IP está en la lista blanca
- Revise cualquier plugin de seguridad que pueda bloquear el acceso a la API

### 4. Contactar al proveedor de hosting {#4-contact-your-hosting-provider}

Pueden verificar:
- Estado del servidor y registros
- Configuraciones de firewall
- Si la dirección IP ha sido bloqueada
