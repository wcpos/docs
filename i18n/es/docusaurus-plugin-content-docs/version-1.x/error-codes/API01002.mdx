# API01002: Conexión Rechazada

## Qué Significa

El servidor rechazó activamente el intento de conexión. Esto es diferente de un tiempo de espera — el servidor respondió, pero dijo "no".

## Causas Comunes

- **Servidor inactivo** — El servidor web (Apache/Nginx) puede no estar ejecutándose
- **Puerto incorrecto** — Intentando conectar en el puerto equivocado
- **Bloqueo por firewall** — Un firewall está bloqueando la conexión
- **Restricciones de IP** — Su dirección IP puede estar bloqueada

## Cómo Solucionarlo

### 1. Verificar que el Servidor Esté Funcionando

Intente acceder directamente a su sitio WordPress en un navegador. Si no se carga, contacte a su proveedor de alojamiento.

### 2. Verificar la Configuración de URL

Asegúrese de usar la URL correcta:
- Verifique `http://` vs `https://`
- Revise errores de tipeo en el nombre de dominio
- Confirme el puerto correcto si usa uno no estándar

### 3. Revisar Configuraciones del Firewall

Si tiene acceso a su servidor:
- Verifique que el puerto 80 (HTTP) o 443 (HTTPS) esté abierto
- Compruebe si su IP está en la lista blanca
- Revise cualquier plugin de seguridad que pueda bloquear el acceso API

### 4. Contactar al Proveedor de Alojamiento

Pueden verificar:
- Estado del servidor y registros
- Configuraciones del firewall
- Si su IP ha sido bloqueada
