# API02006: Clave API Inválida {#api02006-api-key-invalid}

## Qué Significa Esto {#what-this-means}

La clave de API REST de WooCommerce que se está utilizando es inválida. Las claves API se utilizan para la autenticación de servidor-a-servidor, y la clave proporcionada no coincide con ninguna clave válida en WooCommerce.

## Causas Comunes {#common-causes}

- **Clave eliminada** — La clave API fue eliminada de WooCommerce
- **Error tipográfico en la clave** — La clave fue ingresada incorrectamente
- **Par de claves incorrecto** — La clave de consumidor y el secreto no coinciden
- **Clave de otro sitio diferente** — Uso de claves generadas para otra instalación

## Cómo Solucionar {#how-to-fix}

### 1. Verificar la Clave API {#1-verify-the-api-key}

En el Admin de WordPress → WooCommerce → Configuración → Avanzado → REST API:
1. Verifica si tu clave API existe
2. Asegúrate de que no ha sido revocada
3. Toma nota de los permisos (lectura/escritura/lectura-escritura)

### 2. Generar Nuevas Claves API {#2-generate-new-api-keys}

Si la clave está ausente o es inválida:
1. Ve a WooCommerce → Configuración → Avanzado → REST API
2. Haz clic en "Agregar clave"
3. Ingresa una descripción (p. ej., "WCPOS")
4. Selecciona el usuario
5. Elige permisos de "Lectura/Escritura"
6. Haz clic en "Generar clave API"
7. **Copia tanto la Clave de Consumidor como el Secreto de Consumidor** (se muestra solo una vez)

### 3. Actualizar la Configuración del POS {#3-update-pos-configuration}

Ingresa las nuevas claves API en el POS:
- Clave de Consumidor (comienza con `ck_`)
- Secreto de Consumidor (comienza con `cs_`)

### 4. Verificar los Permisos de la Clave {#4-check-key-permissions}

Asegúrate de que la clave tenga permisos suficientes:
- **Leer** — Solo ver datos
- **Escribir** — Modificar datos solamente
- **Leer/Escribir** — Acceso total (recomendado para el POS)
