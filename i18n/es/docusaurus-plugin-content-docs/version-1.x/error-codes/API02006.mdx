# API02006: Clave de API no válida

## Qué Significa

La clave de la API REST de WooCommerce que se está utilizando no es válida. Las claves de API se usan para la autenticación entre servidores, y la clave proporcionada no coincide con ninguna clave válida en WooCommerce.

## Causas Comunes

- **Clave eliminada** — La clave de API fue eliminada de WooCommerce
- **Error de escritura** — La clave se ingresó incorrectamente
- **Par de claves incorrecto** — La clave de consumo y el secreto no coinciden
- **Clave de otro sitio** — Se están usando claves generadas para otra instalación

## Cómo Solucionarlo

### 1. Verificar la Clave de API

En Administración de WordPress → WooCommerce → Configuración → Avanzado → API REST:
1. Comprobar si existe su clave de API
2. Verificar que no haya sido revocada
3. Tomar nota de los permisos (lectura/escritura/lectura-escritura)

### 2. Generar Nuevas Claves de API

Si la clave está ausente o no es válida:
1. Ir a WooCommerce → Configuración → Avanzado → API REST
2. Hacer clic en "Añadir clave"
3. Introducir una descripción (p. ej., "WCPOS")
4. Seleccionar el usuario
5. Elegir permisos "Lectura/Escritura"
6. Hacer clic en "Generar clave de API"
7. **¡Copiar tanto la Clave de Consumo como el Secreto de Consumo! (¡se muestran solo una vez!)**

### 3. Actualizar Configuración del POS

Introducir las nuevas claves de API en el POS:
- Clave de Consumo (comienza con `ck_`)
- Secreto de Consumo (comienza con `cs_`)

### 4. Verificar Permisos de la Clave

Asegurarse de que la clave tenga permisos suficientes:
- **Lectura** — Solo ver datos
- **Escritura** — Solo modificar datos
- **Lectura/Escritura** — Acceso completo (recomendado para POS)
