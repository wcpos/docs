# API03004: Solicitud Demasiado Grande

## Qué Significa

La solicitud que está enviando supera los límites de tamaño del servidor. Esto típicamente sucede cuando se intenta enviar demasiados datos a la vez.

## Causas Comunes

- **Operaciones por lotes grandes** — Intentando sincronizar demasiados registros a la vez
- **Imágenes grandes** — Subiendo imágenes de tamaño excesivo
- **Demasiados artículos** — Pedido con una cantidad extremadamente alta de líneas de artículos
- **Límites del servidor** — PHP o el servidor web tienen límites de carga bajos

## Cómo Solucionarlo

### 1. Reducir el Tamaño de la Solicitud

Si está sincronizando datos:
- Intente sincronizar en lotes más pequeños
- El POS debería manejar esto automáticamente
- Espere a que la sincronización actual se complete antes de iniciar otra

### 2. Verificar Tamaños de Imágenes

Si está subiendo imágenes:
- Redimensione las imágenes antes de subirlas
- Use formatos comprimidos (JPEG vs BMP)
- La mayoría de las imágenes de productos funcionan bien por debajo de 1MB

### 3. Dividir Pedidos Grandes

Si un pedido tiene muchos artículos:
- Considere dividirlo en múltiples pedidos
- Esto es raro en el uso normal del POS

### 4. Aumentar Límites del Servidor

Contacte a su proveedor de alojamiento o edite la configuración de PHP:

```php
// En php.ini o .htaccess
upload_max_filesize = 64M
post_max_size = 64M
max_input_vars = 5000
```

### 5. Verificar Límites del Servidor Web

Nginx o Apache pueden tener sus propios límites:
- `client_max_body_size` para Nginx
- `LimitRequestBody` para Apache
