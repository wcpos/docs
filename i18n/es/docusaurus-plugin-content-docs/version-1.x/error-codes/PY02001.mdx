# PY02001: Error de Pasarela de Pago

## Lo Que Esto Significa

Ocurrió un error mientras se comunicaba con la pasarela de pago. El procesador de pagos no pudo completar la solicitud.

## Causas Comunes

- **Tiempo de inactividad de la pasarela** — El servicio de pago está experimentando problemas
- **Error de configuración** — La configuración de la pasarela es incorrecta
- **Credenciales de API** — Claves de API inválidas o caducadas
- **Problemas de red** — Problemas de conexión con la pasarela

## Cómo Solucionarlo

### 1. Reintentar la Transacción

A veces, las pasarelas tienen problemas temporales:
- Espera un momento
- Intenta procesar el pago de nuevo
- El problema puede resolverse por sí solo

### 2. Verificar el Estado de la Pasarela

Visita la página de estado de tu proveedor de pagos:
- Stripe: status.stripe.com
- PayPal: Consulta la comunidad o el estado de PayPal
- Square: status.squareup.com
- Revisa si hay interrupciones conocidas

### 3. Verificar la Configuración de la Pasarela

En WooCommerce Admin:
1. Ve a WooCommerce → Configuración → Pagos
2. Haz clic en tu pasarela de pago
3. Verifica que las claves de API sean correctas
4. Comprueba que el modo de prueba/en vivo coincida con tus claves

### 4. Verificar Credenciales de API

Asegúrate de que las credenciales sean válidas:
- Las claves de API no hayan caducado
- Usando claves en vivo para transacciones en vivo
- Usando claves de prueba para transacciones de prueba
- Las credenciales coinciden con la cuenta correcta

### 5. Probar con una Pasarela Diferente

Si está disponible:
- Intenta procesar con una pasarela alternativa
- Usa el pago en efectivo como respaldo
- Completa la venta y concilia más tarde

## Para Propietarios de Tiendas

Problemas de configuración comunes:
- Mezclando claves de API de prueba y en vivo
- Claves de API de diferentes cuentas
- Métodos de pago deshabilitados
- Credenciales caducadas o revocadas

Contacta al soporte de tu proveedor de pagos si el problema persiste.
