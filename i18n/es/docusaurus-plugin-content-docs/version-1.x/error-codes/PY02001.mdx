# PY02001: Error de Pasarela de Pago

## Qué Significa

Ha ocurrido un error al comunicarse con la pasarela de pago. El procesador de pagos no pudo completar la solicitud.

## Causas Comunes

- **Inactividad de la pasarela** — El servicio de pago está experimentando problemas
- **Error de configuración** — La configuración de la pasarela es incorrecta
- **Credenciales de API** — Claves de API inválidas o caducadas
- **Problemas de red** — Problemas de conexión con la pasarela

## Cómo Solucionar

### 1. Reintentar la Transacción

A veces las pasarelas tienen problemas temporales:
- Espere un momento
- Intente procesar el pago de nuevo
- Es posible que el problema se resuelva solo

### 2. Verificar el Estado de la Pasarela

Visite la página de estado de su proveedor de pagos:
- Stripe: status.stripe.com
- PayPal: Consulte la comunidad o estado de PayPal
- Square: status.squareup.com
- Verifique interrupciones conocidas

### 3. Verificar la Configuración de la Pasarela

En el Administrador de WooCommerce:
1. Vaya a WooCommerce → Configuración → Pagos
2. Haga clic en su pasarela de pago
3. Verifique que las claves de API sean correctas
4. Compruebe que el modo de prueba/producción coincida con sus claves

### 4. Verificar Credenciales de API

Asegúrese de que las credenciales sean válidas:
- Las claves de API no han caducado
- Usar claves de producción para transacciones en vivo
- Usar claves de prueba para transacciones de prueba
- Las credenciales coinciden con la cuenta correcta

### 5. Probar con una Pasarela Diferente

Si está disponible:
- Intente procesar con una pasarela alternativa
- Use pago en efectivo como alternativa
- Complete la venta y concilie después

## Para Propietarios de Tiendas

Problemas comunes de configuración:
- Mezclar claves de API de prueba y producción
- Claves de API de diferentes cuentas
- Métodos de pago desactivados
- Credenciales caducadas o revocadas

Contacte con el soporte de su proveedor de pagos si el problema persiste.
