# API01005: Error de Certificado SSL

## Qué Significa

Hay un problema con el certificado SSL/TLS de su sitio web. Este certificado es lo que hace que la conexión sea segura (HTTPS). El POS no se conectará a sitios con certificados no válidos para proteger sus datos.

## Causas Comunes

- **Certificado caducado** — Los certificados SSL necesitan renovarse periódicamente
- **Certificado autofirmado** — No emitido por una autoridad confiable
- **Dominio incorrecto** — El certificado no coincide con el nombre de dominio
- **Cadena de certificados incompleta** — Certificados intermedios faltantes
- **Contenido mixto** — Algunos recursos se cargan por HTTP en lugar de HTTPS

## Cómo Solucionarlo

### 1. Verificar Estado del Certificado

Visite su sitio en un navegador y haga clic en el ícono del candado para ver los detalles del certificado. Busque:
- Fecha de caducidad
- Emitido a (debe coincidir con su dominio)
- Emitido por (debe ser una autoridad reconocida)

### 2. Renovar Certificado Caducado

Si está caducado:
- La mayoría de los proveedores de alojamiento ofrecen certificados gratuitos Let's Encrypt
- Contacte a su proveedor de alojamiento para renovar
- Verifique si la renovación automática está habilitada

### 3. Corregir Discrepancia de Certificado

Asegúrese de que el certificado cubra:
- Su dominio exacto (`sutienda.com`)
- Variante WWW (`www.sutienda.com`) si se usa
- Considere un certificado comodín (`*.sutienda.com`)

### 4. Instalar Certificados Intermedios Faltantes

Use una herramienta de verificación SSL (como SSL Labs) para identificar certificados faltantes. Su proveedor de alojamiento puede ayudar a instalarlos.

### 5. Forzar HTTPS

En WordPress, asegúrese de:
- La URL del sitio use `https://`
- SSL forzado esté habilitado en la configuración de WooCommerce
