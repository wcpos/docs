# API01005: Error de Certificado SSL

## Qué significa esto

Hay un problema con el certificado SSL/TLS de tu sitio web. Este certificado es el que hace que la conexión sea segura (HTTPS). El POS no se conectará a sitios con certificados inválidos para proteger tus datos.

## Causas comunes

- **Certificado expirado** — Los certificados SSL deben renovarse periódicamente
- **Certificado autofirmado** — No emitido por una autoridad de confianza
- **Dominio incorrecto** — El certificado no coincide con el nombre de dominio
- **Cadena de certificados incompleta** — Faltan certificados intermedios
- **Contenido mixto** — Algunos recursos se cargan a través de HTTP en lugar de HTTPS

## Cómo solucionar

### 1. Verificar el estado del certificado

Visita tu sitio en un navegador y haz clic en el ícono de candado para ver los detalles del certificado. Busca:
- Fecha de expiración
- Emitido a (debe coincidir con tu dominio)
- Emitido por (debe ser una autoridad reconocida)

### 2. Renovar el certificado expirado

Si está expirado:
- La mayoría de los proveedores de hosting ofrecen certificados gratuitos de Let's Encrypt
- Contacta a tu proveedor de hosting para renovarlo
- Verifica si la renovación automática está habilitada

### 3. Corregir el desajuste del certificado

Asegúrate de que el certificado cubra:
- Tu dominio exacto (`yourstore.com`)
- Variante WWW (`www.yourstore.com`) si se usa
- Considera un certificado wildcard (`*.yourstore.com`)

### 4. Instalar certificados intermedios faltantes

Utiliza una herramienta de verificación SSL (como SSL Labs) para identificar certificados faltantes. Tu proveedor de hosting puede ayudar a instalarlos.

### 5. Forzar HTTPS

En WordPress, asegúrate de que:
- La URL del sitio use `https://`
- La opción de forzar SSL esté habilitada en la configuración de WooCommerce
