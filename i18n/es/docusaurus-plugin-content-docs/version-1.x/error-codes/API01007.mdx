# API01007: Dispositivo sin conexión {#api01007-device-offline}

## Qué significa {#what-this-means}

El dispositivo no está conectado a internet. El POS detectó que no hay una conexión de red activa.

## Causas comunes {#common-causes}

- **WiFi desconectado** — El dispositivo perdió la conexión WiFi
- **Cable Ethernet desconectado** — El cable de red está desconectado
- **Modo avión activado** — Todas las conexiones inalámbricas están desactivadas
- **Adaptador de red desactivado** — La interfaz de red está apagada

## Cómo solucionarlo {#how-to-fix}

### 1. Verificar la conexión WiFi {#1-check-wifi-connection}

- Verificar que el WiFi esté activado en el dispositivo
- Asegurarse de estar conectado a la red correcta
- Comprobar la intensidad de la señal WiFi

### 2. Comprobar las conexiones físicas {#2-check-physical-connections}

Si se utiliza ethernet:
- Verificar que el cable esté conectado de forma segura
- Probar con un cable o puerto diferente
- Comprobar las luces del switch/router de red

### 3. Desactivar el modo avión {#3-disable-airplane-mode}

Comprobar si el modo avión está activado accidentalmente y desactivarlo.

### 4. Reiniciar el adaptador de red {#4-restart-network-adapter}

**Windows:**
- Abrir Conexiones de red
- Hacer clic derecho en el adaptador → Deshabilitar, luego Habilitar

**macOS:**
- Desactivar y activar el WiFi desde la barra de menú
- O bien: Preferencias del Sistema → Red → Desactivar/activar

### 5. Reiniciar el dispositivo {#5-restart-your-device}

A veces, un simple reinicio resuelve los problemas de detección de red.

## Modo sin conexión {#offline-mode}

Sin conexión, WCPOS aún permite:
- Explorar y buscar productos en la caché local
- Explorar y buscar clientes en la caché local
- Iniciar nuevos pedidos y agregar artículos al carrito
- Ver datos sincronizados previamente

**Requiere conectividad:**
- Completar/finalizar pedidos
- Crear nuevos clientes

Cuando se restablezca la conectividad, el POS reanudará automáticamente las operaciones normales.
