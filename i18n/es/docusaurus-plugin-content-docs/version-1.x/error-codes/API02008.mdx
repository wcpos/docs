# API02008: Refresh Token Invalid

## Qué Significa Esto

El refresh token almacenado por el POS no es reconocido por el servidor. Los refresh tokens se utilizan para obtener nuevos access tokens sin requerir que vuelvas a iniciar sesión.

## Causas Comunes

- **Token revocado** — Un administrador invalidó tu sesión
- **Cambios en los secretos del servidor** — Las claves de seguridad de WordPress fueron actualizadas
- **Corrupción de datos** — El token se corrompió en el almacenamiento
- **Múltiples inicios de sesión** — Iniciar sesión en otro lugar puede haber invalidado este token

## Cómo Solucionarlo

### 1. Inicia Sesión Nuevamente

Dado que el refresh token es inválido, necesitarás autenticarte de nuevo:
1. Cierra sesión en el POS
2. Ingresa tu nombre de usuario y contraseña
3. Esto generará nuevos tokens válidos

### 2. Verifica Sesiones Revocadas

Si un administrador revocó sesiones:
- Esto es una práctica de seguridad normal
- Simplemente inicia sesión nuevamente
- Tu nueva sesión funcionará normalmente

### 3. Verifica Cambios en el Servidor

Si esto afecta a todos los usuarios:
- Las sales/claves de WordPress pueden haber cambiado
- La configuración del plugin JWT puede haber sido modificada
- Contacta a tu administrador de servidor

### 4. Borra Datos Almacenados

Si continúas teniendo problemas:
- Borra los datos de la aplicación POS
- Elimina las credenciales en caché
- Comienza con un inicio de sesión limpio
