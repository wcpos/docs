# API02008: Token de Actualización Inválido

## Qué Significa

El token de actualización almacenado por el POS no es reconocido por el servidor. Los tokens de actualización se utilizan para obtener nuevos tokens de acceso sin necesidad de iniciar sesión de nuevo.

## Causas Comunes

- **Token revocado** — Un administrador ha invalidado su sesión
- **Cambios en los secretos del servidor** — Las claves de seguridad de WordPress se actualizaron
- **Corrupción de datos** — El token se corrompió durante el almacenamiento
- **Múltiples inicios de sesión** — Iniciar sesión en otro lugar puede haber invalidado este token

## Cómo Solucionarlo

### 1. Iniciar Sesión de Nuevo

Como el token de actualización no es válido, necesitará autenticarse nuevamente:
1. Cerrar sesión en el POS
2. Introducir su nombre de usuario y contraseña
3. Esto generará nuevos tokens válidos

### 2. Verificar Sesiones Revocadas

Si un administrador revocó sesiones:
- Esta es una práctica normal de seguridad
- Simplemente inicie sesión de nuevo
- Su nueva sesión funcionará normalmente

### 3. Verificar Cambios del Servidor

Si esto afecta a todos los usuarios:
- Es posible que las sales/claves de WordPress hayan cambiado
- La configuración del complemento JWT puede haber sido modificada
- Contacte con el administrador del servidor

### 4. Borrar Datos Almacenados

Si continúa teniendo problemas:
- Borre los datos de la aplicación POS
- Elimine las credenciales en caché
- Comience con un inicio de sesión limpio
