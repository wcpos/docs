:::info Función Pro
La funcionalidad multitienda requiere [WCPOS Pro](/getting-started/pro-license).
:::

WCPOS Pro permite crear ubicaciones de tienda independientes dentro de una única instalación de WooCommerce. Cada tienda puede tener su propia dirección, logotipo y configuración, lo cual resulta útil para operar desde múltiples ubicaciones físicas.

## Qué ofrecen las tiendas {#what-stores-provide}

Cada tienda puede tener:

- **Nombre y dirección de la tienda** — se utilizan en recibos e informes
- **Logotipo de la tienda** — se muestra en los recibos impresos
- **Tasas de impuestos** — diferentes ubicaciones pueden tener distintos requisitos fiscales
- **Asignación de cajeros** — permite realizar un seguimiento de qué cajeros trabajan en cada tienda

## Configuración de tiendas {#setting-up-stores}

Al activar Pro se añade un submenú **Tiendas** en POS dentro de WP Admin. Desde allí es posible crear cada tienda, establecer su dirección, logotipo, identificadores fiscales y qué cajeros pueden iniciar sesión — consulte **[Configuración de tiendas](/stores/setup)** para obtener la guía completa paso a paso.

Cuando un cajero inicia sesión en el POS, selecciona desde qué tienda está operando. Esto determina los datos de la tienda que aparecen en los recibos y qué filtro de tienda se aplica a los informes.

## Filtrado por tienda {#filtering-by-store}

Una vez que se han configurado varias tiendas, el filtro de tienda aparece en todo el POS:

- **[Pedidos](/orders)** — permite filtrar el historial de pedidos por ubicación de tienda
- **[Informes](/reports)** — permite generar informes de cierre de caja para una tienda específica
- **[Recibos](/receipts/at-checkout)** — los recibos muestran automáticamente la dirección y el logotipo de la tienda correspondiente

## Precios específicos por tienda {#store-specific-pricing}

WCPOS Pro permite establecer **precios diferentes para cada ubicación de tienda**. Es posible definir un precio regular y un precio de oferta independientes por tienda, tanto para productos como para variaciones.

Cuando un cajero opera desde una tienda específica, el POS utiliza automáticamente los precios de esa tienda. Si no se ha definido un precio específico para la tienda, se utiliza el precio predeterminado de WooCommerce.

## Gestión de inventario {#inventory-management}

De forma predeterminada, todas las tiendas comparten los mismos niveles de stock de WooCommerce. Para el seguimiento de inventario por ubicación, WCPOS Pro se integra con el plugin [ATUM Inventory Management](https://www.atum.org/):

1. Instalar y configurar el plugin ATUM en el sitio WooCommerce
2. Crear ubicaciones de inventario en ATUM
3. En WCPOS Pro, asociar cada tienda con una ubicación de inventario de ATUM
4. Los niveles de stock se actualizarán para la ubicación correcta cuando se realicen pedidos

### Alternativa: WordPress Multisite {#alternative-wordpress-multisite}

Para inventarios completamente separados sin ATUM, es posible utilizar [WordPress Multisite](https://developer.wordpress.org/advanced-administration/multisite/create-network/):

1. Crear un sitio WordPress independiente para cada ubicación de tienda
2. Instalar WooCommerce y WCPOS en cada sitio
3. Cada sitio mantiene su propio catálogo de productos y niveles de stock de forma independiente

Esto proporciona una separación completa, pero implica gestionar múltiples instalaciones de WooCommerce.

## Limitaciones conocidas {#known-limitations}

- **Instancia única de WooCommerce** — las tiendas son ubicaciones dentro de una misma instalación de WooCommerce, no comercios independientes
- **Sin visibilidad de productos por tienda** — todas las tiendas ven el mismo catálogo de productos
