## Nota de Pedido

<div class="image-container">
  <Image
    alt="Nota de Pedido en POS"
    img="/img/order-note.png"
    style={{maxHeight: 500}}
  />
  <p class="image-caption">Modal de Nota de Pedido en POS (izquierda) y Nota de Pedido en WP Admin (derecha)</p>

- Abre un diálogo para agregar una nota de cliente al pedido.
- Las notas de cliente son visibles para el cliente y aparecen por defecto en el recibo, en el área de Mi Cuenta del cliente y en `WP Admin > WooCommerce > Pedidos > Pedido` bajo la dirección de envío.

:::note
Las notas de pedido son visibles para los clientes.
:::

## Metadatos del Pedido

- Abre un modal para gestionar metadatos adicionales del pedido.
- Características incluyen:
  - Cambiar la moneda del pedido.
  - Agregar un ID de transacción (por ejemplo, de un terminal de pago externo).
  - Agregar metadatos personalizados para el pedido.

:::note
Se añadirán características adicionales a este modal con el tiempo, incluyendo la integración con otros plugins de WooCommerce.
:::

### ¿Qué son los Metadatos en WooCommerce?

Los metadatos proporcionan información adicional sobre los pedidos y son accesibles a través de la REST API de WooCommerce. Los desarrolladores y plugins los utilizan para extender la funcionalidad, como integrar servicios de terceros.

### Vista JSON

- Ver la representación JSON sin procesar del pedido, útil para depuración o integración.

## Guardar en el Servidor

- Permite guardar carritos abiertos en el servidor, de modo que se almacenan de forma segura y son accesibles para su uso futuro.
- Crea un pedido de WooCommerce con estado `pos-open`.

### ¿Por qué guardar?

Cuando se crea un nuevo pedido en POS, solo existe localmente. 
Guardar el pedido garantiza que persista, incluso si la base de datos local se borra.

Consulte [Pedidos Abiertos](/pos/cart/open-orders) para obtener más detalles sobre la gestión de pedidos guardados.

## Anular

- Dos Escenarios:
  - Pedidos no guardados:
    - Si el pedido no se ha guardado en el servidor, se eliminará permanentemente de la base de datos local.
  - Pedidos guardados:
    - Si el pedido está guardado en el servidor, se moverá a la carpeta de Papelera en WooCommerce y se eliminará de la base de datos local.
- Para recuperar un pedido anulado:
  - Vaya a `WP Admin > WooCommerce > Pedidos > Papelera`.

## Pagar

- Guarda el pedido en el servidor e inicia el proceso de pago.
- Abre la [pantalla de pago](/pos/checkout) para completar el pago.
