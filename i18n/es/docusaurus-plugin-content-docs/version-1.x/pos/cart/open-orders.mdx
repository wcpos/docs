WCPOS le permite trabajar con múltiples órdenes simultáneamente. Esto es útil para manejar reservas de clientes, cambiar entre transacciones y recuperarse de interrupciones.

## Carrusel de Órdenes Abiertas

En la parte inferior del Panel de Carrito, un carrusel horizontal muestra todas las órdenes abiertas:

{/* Screenshot: open-orders-carousel.png */}

- Cada carrito muestra su **monto total**
- La **orden actual** está resaltada
- Haga clic en cualquier carrito para cambiarlo
- Desplace hacia la izquierda/derecha para ver más carritos

## Crear una Nueva Orden

Un nuevo carrito vacío siempre está disponible. Simplemente haga clic en un carrito vacío en el carrusel o comience a agregar productos cuando el carrito actual esté vacío.

## Cambiar Entre Órdenes

Haga clic en cualquier orden en el carrusel para cambiar a ella. El Panel de Carrito se actualiza para mostrar el contenido de la orden seleccionada.

**Casos de uso:**

- El cliente se ausenta para buscar otro artículo
- Necesidad de atender a un cliente rápido mientras se procesa un pedido grande
- Comparar precios o artículos entre órdenes

## Guardar Órdenes en el Servidor

Las órdenes existen en dos estados:

### Solo Local

De manera predeterminada, las nuevas órdenes se almacenan solo en la base de datos local del navegador/aplicación. Persistirán entre actualizaciones de página, pero:

- No son visibles en la administración de WooCommerce
- Se perderán si se borra la base de datos local
- No son accesibles desde otros dispositivos

### Guardadas en el Servidor

Haga clic en **Guardar en el Servidor** para crear una orden en WooCommerce con el estado `pos-open`. Esto:

- Crea una orden real en WooCommerce
- Persiste incluso si se borra la base de datos local
- Puede ser accedida desde otros dispositivos
- Aparece en Admin de WP > WooCommerce > Órdenes

:::tip Cuándo Guardar
Guarde las órdenes en el servidor cuando:
- Un cliente quiere reservar un pedido para recoger después
- Está terminando su turno y otro cajero continuará
- Quiere un respaldo en caso de problemas con la aplicación/navegador
:::

## Recuperar Órdenes Guardadas

Si ha guardado órdenes en el servidor, pueden ser accedidas de nuevo:

1. Abriendo la pantalla de **Órdenes** (función Pro)
2. Filtrando por estado `pos-open`
3. Reabriendo la orden

## Persistencia de Órdenes

### Almacenamiento Local

WCPOS utiliza IndexedDB para almacenar órdenes localmente. Esto proporciona:

- Persistencia entre sesiones de navegador
- Acceso rápido sin solicitudes de red
- Capacidad sin conexión

### Sincronización con el Servidor

Cuando guarda en el servidor o hace checkout:

- La orden se envía a WooCommerce
- Se recibe una confirmación
- Los datos locales y del servidor se sincronizan

## Anular Órdenes

Para eliminar una orden abierta:

1. Cambie a la orden que desea eliminar
2. Haga clic en el botón **Anular**

**Qué sucede:**

- **Órdenes sin guardar:** Eliminadas permanentemente de la base de datos local
- **Órdenes guardadas:** Movidas a Papelera en WooCommerce y eliminadas localmente

Para recuperar una orden guardada anulada:
1. Vaya a `Admin de WP > WooCommerce > Órdenes > Papelera`
2. Restaure la orden

## Consejos

### Mantener Órdenes Organizadas

Con múltiples órdenes abiertas, ayuda a:

- Agregar nombres de clientes a las órdenes para una identificación fácil
- Agregar notas a las órdenes describiendo la razón de la reserva
- Guardar órdenes importantes en el servidor

### Cambios de Turno

Al terminar un turno con órdenes abiertas:

1. Guarde todas las órdenes importantes en el servidor
2. Agregue notas a las órdenes explicando el estado
3. El siguiente cajero puede acceder a ellas desde la pantalla de Órdenes

### Consideraciones Sin Conexión

Si pierde la conectividad:

- Las órdenes locales siguen siendo accesibles y puede continuar agregando artículos
- No puede completar/hacer checkout de órdenes hasta que se restablezca la conectividad
- No puede guardar órdenes en el servidor hasta que se reconecte
- No puede crear nuevos clientes hasta que se reconecte
