Cuando estés listo para completar una venta, haz clic en el botón **Finalizar Compra** para abrir el modal de finalización de compra. Aquí es donde procesas el pago y completas el pedido.

{/* Screenshot: checkout-modal.png */}

## Resumen del Modal de Finalización de Compra

El modal de finalización de compra muestra:

- **Número de Pedido** - La ID de pedido de WooCommerce
- **Monto a Pagar** - Total a pagar
- **Botón de Configuración de Finalización de Compra** - Solucionar problemas de visualización
- **Cajero** - Quien está procesando el pedido
- **Cliente** - El cliente para este pedido (enlace clickeable)
- **Campo de código de cupón** - Aplicar códigos de descuento
- **Resumen del Pedido** - Productos, cantidades y totales
- **Métodos de Pago** - Opciones de pago disponibles
- **Cancelar / Procesar Pago** botones

## Métodos de Pago

### Disponibles en la Versión Gratuita

La versión gratuita de WCPOS incluye dos pasarelas de pago:

- **Efectivo** - Con monto entregado y calculadora de cambio
- **Tarjeta** - Para terminales de tarjeta externos

### Pasarelas Adicionales (Pro)

:::info Función Pro
Las pasarelas de pago adicionales requieren [WCPOS Pro](/getting-started/pro-license).
:::

Con Pro, puedes habilitar:

- **Terminal Stripe** - Integración directa con lectores de tarjetas Stripe
- **Terminal SumUp** - Integración con lectores de tarjetas SumUp
- **Pasarelas Personalizadas** - Crea tus propias integraciones de pago

Consulta [Métodos de Pago](/payment) para más detalles sobre la configuración de pasarelas.

### Seleccionando un Método de Pago

Haz clic en un método de pago para seleccionarlo. El formulario se actualiza para mostrar campos relevantes:

**Efectivo:**

- **Monto Entregado** - Ingresa el monto que el cliente te da
- **Cambio** - Cambio calculado automáticamente a devolver

{/* Screenshot: checkout-cash-payment.png */}

**Tarjeta:**

- Procesa el pago en tu terminal de tarjeta externo
- Haz clic en Procesar Pago para completar

## Códigos de Cupón

Para aplicar un descuento:

1. Ingresa el código del cupón en el campo
2. Haz clic en **Aplicar cupón**
3. El descuento aparece en el resumen del pedido

Los cupones deben estar configurados en WooCommerce (`WP Admin > Marketing > Cupones`).

## Procesando el Pago

1. Selecciona un método de pago
2. Para efectivo, ingresa el monto entregado
3. Haz clic en **Procesar Pago**
4. El pedido se completa y se muestra el [recibo](/pos/checkout/receipts)

## Configuraciones de Finalización de Compra (Solución de Problemas)

El modal de finalización de compra utiliza un iframe/webview para mostrar la página de Pago de Pedido de WooCommerce. Esto aprovecha la infraestructura de pago existente de WooCommerce, lo que significa que cualquier pasarela de pago que funcione con WooCommerce debería funcionar en el POS.

Sin embargo, los scripts de temas y plugins a veces pueden interferir. Haz clic en **Configuraciones de Finalización de Compra** para solucionar problemas:

### Deshabilitar Todos los Estilos y Scripts

:::warning Opción Nuclear
Esta es la opción nuclear y solo debe ser utilizada para pruebas o en raras ocasiones donde el desarrollador sabe lo que está haciendo. Deshabilitar todos los scripts wp_head eliminará incluso los scripts de WooCommerce necesarios para expandir/contraer las pasarelas de pago, pudiendo romper la funcionalidad de pago.
:::

- **Deshabilitar wp_head** - Elimina todos los scripts/styles del encabezado de WordPress
- **Deshabilitar wp_footer** - Elimina todos los scripts/styles del pie de página de WordPress

### Deshabilitar Estilos Seleccionados

Deshabilitar selectivamente CSS que puede causar problemas de visualización:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Deshabilitar Scripts Seleccionados

Deshabilitar selectivamente JavaScript que puede interferir con las pasarelas de pago:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Si una pasarela de pago no se muestra correctamente:

1. Intenta deshabilitar los estilos del tema primero
2. Luego intenta deshabilitar scripts de WooCommerce que no son necesarios
3. Ten cuidado de no deshabilitar scripts requeridos por tu pasarela de pago
:::

## Cancelar

Haz clic en **Cancelar** para cerrar el modal de finalización de compra sin completar el pedido. El pedido permanece como un carrito abierto.
