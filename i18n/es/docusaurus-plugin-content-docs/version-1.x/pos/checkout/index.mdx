Cuando esté listo para completar una venta, haga clic en el botón de **Caja** para abrir el modal de pago. Aquí es donde se procesa el pago y se completa el pedido.

{/* Screenshot: checkout-modal.png */}

## Descripción general del Modal de Caja

El modal de caja muestra:

- **Número de pedido** - El ID de pedido de WooCommerce
- **Monto a Pagar** - Total adeudado
- Botón de **Configuración de Caja** - Solucionar problemas de visualización
- **Cajero** - Quién está procesando el pedido
- **Cliente** - El cliente para este pedido (enlace en el que se puede hacer clic)
- Campo de **Código de cupón** - Aplicar códigos de descuento
- **Resumen del pedido** - Productos, cantidades y totales
- **Métodos de pago** - Opciones de pago disponibles
- Botones de **Cancelar / Procesar Pago**

## Métodos de Pago

### Disponibles en la Versión Gratuita

La versión gratuita de WCPOS incluye dos pasarelas de pago:

- **Efectivo** - Con calculadora de monto entregado y cambio
- **Tarjeta** - Para terminales de tarjeta externos

### Pasarelas Adicionales (Pro)

:::info Característica Pro
Las pasarelas de pago adicionales requieren [WCPOS Pro](/getting-started/pro-license).
:::

Con Pro, puede habilitar:

- **Terminal Stripe** - Integración directa con lectores de tarjetas Stripe
- **Terminal SumUp** - Integración con lectores de tarjetas SumUp
- **Pasarelas Personalizadas** - Crear sus propias integraciones de pago

Consulte [Pago](/payment) para obtener detalles sobre la configuración de pasarelas.

### Selección de un Método de Pago

Haga clic en un método de pago para seleccionarlo. El formulario se actualiza para mostrar los campos relevantes:

**Efectivo:**
- **Monto Entregado** - Ingrese el monto que el cliente le entrega
- **Cambio** - Cambio calculado automáticamente para devolver

{/* Screenshot: checkout-cash-payment.png */}

**Tarjeta:**
- Procesar pago en su terminal de tarjeta externa
- Haga clic en Procesar Pago para completar

## Códigos de Cupón

Para aplicar un descuento:

1. Ingrese el código de cupón en el campo
2. Haga clic en **Aplicar cupón**
3. El descuento aparece en el resumen del pedido

Los cupones deben configurarse en WooCommerce (`WP Admin > Marketing > Cupones`).

## Procesamiento de Pago

1. Seleccione un método de pago
2. Para efectivo, ingrese el monto entregado
3. Haga clic en **Procesar Pago**
4. El pedido se completa y se muestra el [recibo](/pos/checkout/receipts)

## Configuración de Caja (Solución de problemas)

El modal de caja utiliza un iframe/webview para mostrar la página de Pago de Pedido de WooCommerce. Esto aprovecha la infraestructura de pago existente de WooCommerce, lo que significa que cualquier pasarela de pago que funcione con WooCommerce debería funcionar en el POS.

Sin embargo, los scripts de temas y complementos pueden interferir a veces. Haga clic en **Configuración de Caja** para solucionar problemas:

{/* Screenshot: checkout-settings.png */}

### Deshabilitar Todos los Estilos y Scripts

- **Deshabilitar wp_head** - Elimina todos los scripts/estilos del encabezado de WordPress
- **Deshabilitar wp_footer** - Elimina todos los scripts/estilos del pie de página de WordPress

### Deshabilitar Estilos Seleccionados

Deshabilite selectivamente CSS que pueda causar problemas de visualización:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Deshabilitar Scripts Seleccionados

Deshabilite selectivamente JavaScript que pueda interferir con las pasarelas de pago:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Si una pasarela de pago no se muestra correctamente:
1. Primero intente deshabilitar los estilos del tema
2. Luego intente deshabilitar los scripts de WooCommerce que no sean necesarios
3. Tenga cuidado de no deshabilitar scripts requeridos por su pasarela de pago
:::

## Cancelar

Haga clic en **Cancelar** para cerrar el modal de caja sin completar el pedido. El pedido permanece como un carrito abierto.
