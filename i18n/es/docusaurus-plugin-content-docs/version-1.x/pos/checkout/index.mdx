Para completar una venta, haga clic en el botón **Cobro** para abrir el modal de cobro. Aquí se procesa el pago y se completa el pedido.

{/* Screenshot: checkout-modal.png */}

## Descripción general del modal de cobro {#checkout-modal-overview}

El modal de cobro muestra:

- **Número de pedido** - El ID de pedido de WooCommerce
- **Importe a pagar** - Importe total adeudado
- Botón **Configuración de cobro** - Para solucionar problemas de visualización
- **Cajero** - Quién está procesando el pedido
- **Cliente** - El cliente de este pedido (enlace clicable)
- Botón **Agregar cupón** - Aplicar códigos de descuento
- **Resumen del pedido** - Productos, cantidades y totales
- **Métodos de pago** - Opciones de pago disponibles
- Botones **Cancelar / Procesar pago**

## Métodos de pago {#payment-methods}

### Disponible en la versión gratuita {#available-in-free-version}

La versión gratuita de WCPOS incluye dos pasarelas de pago:

- **Efectivo** - Con monto recibido y calculadora de cambio
- **Tarjeta** - Para terminales de tarjeta externos

### Pasarelas adicionales (Pro) {#additional-gateways-pro}

:::info Función Pro
Las pasarelas de pago adicionales requieren [WCPOS Pro](/getting-started/pro-license).
:::

Con Pro, es posible habilitar:

- **Stripe Terminal** - Integración directa con lectores de tarjetas Stripe
- **SumUp Terminal** - Integración con lectores de tarjetas SumUp
- **Pasarelas personalizadas** - Creación de integraciones de pago propias

Consultar [Pagos](/payment) para obtener más información sobre la configuración de pasarelas.

### Selección de un método de pago {#selecting-a-payment-method}

Al hacer clic en un método de pago, el formulario se actualiza para mostrar los campos correspondientes:

**Efectivo:**

- **Monto recibido** - Introducir el monto que entrega el cliente
- **Cambio** - Cálculo automático del cambio a devolver

{/* Screenshot: checkout-cash-payment.png */}

**Tarjeta:**

- Procesar el pago en el terminal de tarjetas externo
- Hacer clic en Procesar pago para completar

## Cupones {#coupon-codes}

El carrito incluye un campo **Agregar cupón** sobre los totales (solo Pro). Es posible escribir el código o buscar por descripción; el cupón se valida localmente y aparece como una etiqueta que se puede eliminar. Los cupones múltiples se aplican de forma secuencial.

Consultar **[Aplicar cupones en la caja](/coupons/applying-coupons)** para el flujo de trabajo completo: búsqueda, etiquetas, descuentos secuenciales y la tabla de errores de validación y resoluciones. Para los tipos de cupones, reglas de validación y configuración en WooCommerce, consultar [Cupones](/coupons).

## Procesamiento del pago {#processing-payment}

1. Seleccionar un método de pago
2. Para efectivo, ingresar el monto entregado
3. Hacer clic en **Procesar pago**
4. El pedido se completa y se muestra el [recibo](/pos/checkout/receipts)

## Configuración de pago (Solución de problemas) {#checkout-settings-troubleshooting}

El modal de pago utiliza un iframe/webview para mostrar la página de pago de pedidos de WooCommerce. Esto aprovecha la infraestructura de pagos existente de WooCommerce, lo que significa que cualquier pasarela de pago compatible con WooCommerce debería funcionar en el POS.

Sin embargo, los scripts de temas y plugins pueden interferir en ocasiones. Haga clic en **Configuración de pago** para solucionar problemas:

### Desactivar todos los estilos y scripts {#disable-all-styles-and-scripts}

:::warning Opción nuclear
Esta es la opción nuclear y solo debe utilizarse para pruebas o en casos excepcionales en los que el desarrollador sabe lo que está haciendo. Desactivar todos los scripts de wp_head eliminará incluso los scripts de WooCommerce necesarios para expandir/contraer las pasarelas de pago, lo que podría interrumpir la funcionalidad de pagos.
:::

- **Desactivar wp_head** - Elimina todos los scripts/estilos del encabezado de WordPress
- **Desactivar wp_footer** - Elimina todos los scripts/estilos del pie de página de WordPress

### Desactivar estilos seleccionados {#disable-selected-styles}

Desactivar selectivamente el CSS que pueda causar problemas de visualización:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Desactivar scripts seleccionados {#disable-selected-scripts}

Desactivar selectivamente JavaScript que pueda interferir con las pasarelas de pago:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Si una pasarela de pago no se muestra correctamente:

1. Intentar deshabilitar primero los estilos del tema
2. A continuación, intentar deshabilitar los scripts de WooCommerce que no sean necesarios
3. Tener cuidado de no deshabilitar los scripts requeridos por la pasarela de pago
   :::

## Cancelar {#cancel}

Hacer clic en **Cancelar** para cerrar el modal de pago sin completar el pedido. El pedido permanece como un carrito abierto.
