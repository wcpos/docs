## Búsqueda de Productos {#product-search}

### Campo de Búsqueda Unificado {#unified-search-field}

WCPOS cuenta con un solo campo de búsqueda que busca simultáneamente en múltiples atributos del producto:

- **Nombre del Producto** - Busca en el título y descripción del producto
- **SKU** - Coincide con el campo postmeta `_sku` del producto
- **Código de Barras** - Busca en el campo de código de barras configurado, que puede ser cualquier campo postmeta. La configuración heredada es `_sku`, pero esto cambiará a `_global_unique_id` en el futuro.

Simplemente escriba su término de búsqueda en el campo "Buscar Productos", y el POS filtrará instantáneamente los resultados en todos estos campos.

### Tecnología de Búsqueda Tokenizada {#tokenized-search-technology}

La funcionalidad de búsqueda utiliza la [biblioteca FlexSearch](https://github.com/nextapps-de/flexsearch) con capacidades avanzadas de tokenización:

- **Tokenización Adelantada** - Coincide con palabras parciales desde el principio (por ejemplo, buscar "azul" encontrará "arándano azul")
- **Optimizado para el Rendimiento** - Utiliza una configuración de rendimiento para resultados de búsqueda rápidos
- **Consciente del Idioma** - Se adapta al idioma configurado de su tienda
- **Inicialización Perezosa** - Optimiza el uso de memoria al cargar índices de búsqueda solo cuando es necesario

### Cómo Funciona la Búsqueda {#how-search-works}

Cuando escribe en el campo de búsqueda, el POS:

1. **Tokeniza** su entrada en términos buscables
2. **Busca localmente** datos de producto almacenados primero para obtener resultados instantáneos
3. **Consulta al servidor** si no se encuentran coincidencias locales, luego descarga y almacena nuevos productos para búsquedas futuras
4. **Actualiza los resultados** en tiempo real mientras escribe

Este enfoque asegura un rendimiento de búsqueda rápido mientras construye progresivamente una base de datos de productos local completa.

## Filtrado de Productos {#product-filtering}

### Barra de Filtros {#filter-bar}

Debajo del campo de búsqueda, encontrará botones de filtro interactivos y menús desplegables que le permiten reducir productos por criterios específicos.

### Filtros Disponibles {#available-filters}

#### Estado de Stock {#stock-status}
Filtrar productos según su estado de inventario:
- **En Stock** - Productos con inventario disponible
- **Sin Stock** - Productos sin inventario
- **Pedido Pendiente** - Productos disponibles para pedido pendiente

#### Productos Destacados {#featured-products}
Alternar para mostrar solo productos marcados como "Destacados" en su tienda WooCommerce.

#### Productos en Oferta {#on-sale-products}
Filtrar para mostrar solo productos actualmente en oferta o con descuentos activos.

#### Categoría {#category}
Utilice el menú desplegable de categoría para filtrar productos por las categorías de producto asignadas. Esto le ayuda a encontrar rápidamente productos dentro de departamentos o líneas de productos específicas.

#### Etiqueta {#tag}
Filtrar por etiquetas de producto para encontrar artículos con atributos o características específicas que ha definido en su tienda WooCommerce.

### Uso de Filtros {#using-filters}

- **Activar Filtros** - Haga clic en cualquier botón de filtro para activarlo (los filtros activos se resaltan)
- **Múltiples Filtros** - Puede combinar múltiples filtros para reducir aún más su búsqueda
- **Limpiar Filtros** - Haga clic nuevamente en un filtro activo para desactivarlo
- **Buscar + Filtrar** - Use filtros junto con el campo de búsqueda para una ubicación precisa del producto

## Configuración del Código de Barras {#barcode-configuration}

### Campos de Búsqueda {#search-fields}

La funcionalidad de búsqueda incluye automáticamente su campo de código de barras configurado. El campo de código de barras utilizado para buscar depende de la configuración de su POS.

## F.A.Q.

<Accordion>
  <AccordionItem question="¿Para qué sirve el campo _global_unique_id para los códigos de barras?">

El campo `_global_unique_id` es un nuevo campo de código de barras que WooCommerce agregó recientemente para proporcionar una mejor estandarización de códigos de barras en las tiendas.

**Puntos Clave:**

- **Estándar Moderno**: Este campo fue diseñado específicamente para la identificación global de códigos de barras
- **Configuración del POS**: Puede configurar el POS para usar `_global_unique_id` como el campo de código de barras en la configuración del POS
- **Herencia vs. Nuevo**: La configuración heredada de código de barras utiliza el campo `_sku`, pero esto cambiará a `_global_unique_id` en versiones futuras
- **Flexibilidad**: Puede configurar cualquier campo meta de producto como su campo de código de barras si está utilizando complementos de código de barras de terceros
- **Valor Predeterminado Futuro**: `_global_unique_id` se convertirá en el campo de código de barras predeterminado en futuras actualizaciones del POS

Para configurar qué campo utiliza el POS para los códigos de barras, visite la configuración de su POS en el área de administración de WordPress.

  </AccordionItem>

  <AccordionItem question="¿Por qué no veo todos mis productos cuando busco?">

WCPOS utiliza descarga progresiva de productos para mantener el rendimiento. Si no ve un producto:

1. **Intente buscarlo** - Esto hará que el POS lo busque en su servidor
2. **Espere la descarga** - El POS descargará el producto y otros en pequeñas cantidades
3. **Busque nuevamente** - Una vez descargado, el producto aparecerá en búsquedas futuras

Este proceso asegura que su POS se mantenga rápido y receptivo incluso con miles de productos. Aprenda más sobre esto en nuestra guía de [Sincronización de Productos](/products/sync).

  </AccordionItem>

  <AccordionItem question="¿Puedo buscar nombres de productos o SKUs parciales?">

¡Sí! La búsqueda tokenizada utiliza coincidencias adelantadas, lo que significa:

- Buscar "azul" encontrará productos con "arándano azul", "camisa azul", etc.
- Buscar "ABC" encontrará SKUs como "ABC123", "ABC-XYZ", etc.
- No necesita escribir palabras completas o códigos

La búsqueda está diseñada para encontrar productos rápidamente con una escritura mínima.

  </AccordionItem>
</Accordion>
