## Búsqueda de Productos

### Campo de Búsqueda Unificado

WCPOS cuenta con un único campo de búsqueda que busca simultáneamente en múltiples atributos de productos:

- **Nombre del Producto** - Busca en el título y la descripción del producto
- **SKU** - Coincide con el campo postmeta `_sku` del producto
- **Código de Barras** - Busca en el campo de código de barras configurado, que puede ser cualquier campo postmeta. La configuración heredada es `_sku`, pero esto cambiará a `_global_unique_id` en el futuro.

Simplemente escribe tu término de búsqueda en el campo "Buscar Productos", y el POS filtrará instantáneamente los resultados en todos estos campos.

### Tecnología de Búsqueda Tokenizada

La funcionalidad de búsqueda utiliza la [biblioteca FlexSearch](https://github.com/nextapps-de/flexsearch) con capacidades avanzadas de tokenización:

- **Tokenización Adelantada** - Coincide palabras parciales desde el comienzo (por ejemplo, buscar "azul" encontrará "arándano")
- **Optimización de Rendimiento** - Utiliza un perfil predeterminado de rendimiento para obtener resultados de búsqueda rápidos
- **Consciente del Idioma** - Se adapta al idioma configurado de tu tienda
- **Inicialización Perezosa** - Optimiza el uso de memoria al cargar índices de búsqueda solo cuando se necesita

### Cómo Funciona la Búsqueda

Cuando escribes en el campo de búsqueda, el POS:

1. **Tokeniza** tu entrada en términos buscables
2. **Busca localmente** en los datos de productos almacenados primero para obtener resultados instantáneos
3. **Consulta al servidor** si no se encuentran coincidencias locales, luego descarga y almacena nuevos productos para futuras búsquedas
4. **Actualiza los resultados** en tiempo real mientras escribes

Este enfoque asegura un rendimiento de búsqueda rápido mientras se construye progresivamente una base de datos local completa de productos.

## Filtrado de Productos

### Barra de Filtros

Debajo del campo de búsqueda, encontrarás interruptores de filtro interactivos y menús desplegables que te permiten reducir productos por criterios específicos.

### Filtros Disponibles

#### Estado de Inventario
Filtra productos según su estado de inventario:
- **Disponible** - Productos con inventario disponible
- **Agotado** - Productos con inventario cero
- **Pedido Pendiente** - Productos disponibles para pedido pendiente

#### Productos Destacados
Activa para mostrar solo productos marcados como "Destacados" en tu tienda de WooCommerce.

#### Productos en Oferta  
Filtra para mostrar solo productos actualmente en oferta o con descuentos activos.

#### Categoría
Utiliza el menú desplegable de categoría para filtrar productos por sus categorías de producto asignadas. Esto te ayuda a encontrar rápidamente productos dentro de departamentos o líneas de productos específicos.

#### Etiqueta
Filtra por etiquetas de productos para encontrar artículos con atributos o características específicas que has definido en tu tienda de WooCommerce.

### Uso de Filtros

- **Activar Filtros** - Haz clic en cualquier botón de filtro para activarlo (los filtros activos aparecen resaltados)
- **Filtros Múltiples** - Puedes combinar múltiples filtros para reducir aún más tu búsqueda
- **Limpiar Filtros** - Haz clic nuevamente en un filtro activo para desactivarlo
- **Buscar + Filtrar** - Utiliza filtros junto con el campo de búsqueda para una ubicación precisa de productos

## Configuración de Códigos de Barras

### Campos de Búsqueda

La funcionalidad de búsqueda incluye automáticamente tu campo de código de barras configurado. El campo de código de barras utilizado para buscar depende de tu configuración de ajustes de POS.

## F.A.Q.

<details>

<summary>
  ¿Para qué sirve el campo _global_unique_id para códigos de barras?
</summary>

El campo `_global_unique_id` es un nuevo campo de código de barras que WooCommerce agregó recientemente para proporcionar una mejor estandarización de códigos de barras en las tiendas.

**Puntos Clave:**

- **Estándar Moderno**: Este campo fue diseñado específicamente para la identificación global de códigos de barras
- **Configuración del POS**: Puedes configurar el POS para usar `_global_unique_id` como el campo de código de barras en la configuración del POS
- **Legado vs. Nuevo**: La configuración de código de barras heredada utiliza el campo `_sku`, pero esto cambiará a `_global_unique_id` en versiones futuras
- **Flexibilidad**: Puedes configurar cualquier campo de meta producto como tu campo de código de barras si utilizas plugins de código de barras de terceros
- **Predeterminado Futuro**: `_global_unique_id` se convertirá en el campo de código de barras predeterminado en futuras actualizaciones del POS

Para configurar qué campo usa el POS para códigos de barras, visita la configuración de tu POS en el área de administración de WordPress.

</details>

<details>

<summary>
  ¿Por qué no veo todos mis productos cuando busco?
</summary>

WCPOS utiliza descarga progresiva de productos para mantener el rendimiento. Si no ves un producto:

1. **Intenta buscarlo** - Esto hará que el POS lo busque en tu servidor
2. **Espera la descarga** - El POS descargará el producto y otros en pequeños lotes
3. **Busca nuevamente** - Una vez descargado, el producto aparecerá en búsquedas futuras

Este proceso asegura que tu POS permanezca rápido y receptivo incluso con miles de productos. Aprende más sobre esto en nuestra guía de [Sincronización de Productos](/products/sync).

</details>

<details>

<summary>
  ¿Puedo buscar nombres de productos o SKUs parciales?
</summary>

¡Sí! La búsqueda tokenizada utiliza coincidencias adelantadas, lo que significa:

- Buscar "azul" encontrará productos con "arándano", "camisa azul", etc.
- Buscar "ABC" encontrará SKUs como "ABC123", "ABC-XYZ", etc.
- No es necesario que escribas palabras o códigos completos

La búsqueda está diseñada para encontrar productos rápidamente con mínima escritura.

</details>
