## Búsqueda de Productos

### Campo de Búsqueda Unificado

WCPOS cuenta con un único campo de búsqueda que busca simultáneamente en varios atributos de producto:

- **Nombre del Producto** - Busca en el título y descripción del producto
- **SKU** - Coincide con el campo postmeta `_sku`
- **Código de Barras** - Busca en el campo de código de barras configurado, que puede ser cualquier campo postmeta. La configuración heredada es `_sku`, pero se cambiará a `_global_unique_id` en el futuro.

Simplemente escribe tu término de búsqueda en el campo "Buscar Productos", y el POS filtrará los resultados instantáneamente en todos estos campos.

### Tecnología de Tokenización de Búsqueda

La funcionalidad de búsqueda utiliza la biblioteca [FlexSearch](https://github.com/nextapps-de/flexsearch) con capacidades avanzadas de tokenización:

- **Tokenización Adelantada** - Coincide con palabras parciales desde el principio (por ejemplo, buscar "azul" encontrará "arándano")
- **Optimizado para Rendimiento** - Usa un ajuste preestablecido de rendimiento para resultados de búsqueda rápidos
- **Consciente del Idioma** - Se adapta al idioma configurado de tu tienda
- **Inicialización Diferida** - Optimiza el uso de memoria cargando índices de búsqueda solo cuando es necesario

### Cómo Funciona la Búsqueda

Cuando escribes en el campo de búsqueda, el POS:

1. **Tokeniza** tu entrada en términos buscables
2. **Busca localmente** los datos de productos almacenados primero para resultados instantáneos
3. **Consulta al servidor** si no se encuentran coincidencias locales, luego descarga y almacena nuevos productos para búsquedas futuras
4. **Actualiza resultados** en tiempo real mientras escribes

Este enfoque garantiza un rendimiento de búsqueda rápido mientras se construye progresivamente una base de datos de productos local completa.

## Filtrado de Productos

### Barra de Filtros

Debajo del campo de búsqueda, encontrarás interruptores de filtro interactivos y menús desplegables que te permiten reducir los productos por criterios específicos.

### Filtros Disponibles

#### Estado de Stock
Filtra productos según su estado de inventario:
- **En Stock** - Productos con inventario disponible
- **Sin Stock** - Productos con inventario cero
- **Pedido Pendiente** - Productos disponibles para pedido pendiente

#### Productos Destacados
Cambia para mostrar solo productos marcados como "Destacados" en tu tienda WooCommerce.

#### Productos en Oferta
Filtra para mostrar solo productos actualmente en oferta o con descuentos activos.

#### Categoría
Usa el menú desplegable de categoría para filtrar productos por sus categorías de producto asignadas. Esto te ayuda a encontrar rápidamente productos dentro de departamentos o líneas de productos específicas.

#### Etiqueta
Filtra por etiquetas de producto para encontrar artículos con atributos o características específicas que hayas definido en tu tienda WooCommerce.

### Uso de Filtros

- **Activar Filtros** - Haz clic en cualquier botón de filtro para activarlo (los filtros activos aparecerán resaltados)
- **Filtros Múltiples** - Puedes combinar varios filtros para reducir aún más la búsqueda
- **Borrar Filtros** - Haz clic en un filtro activo de nuevo para desactivarlo
- **Búsqueda + Filtro** - Usa filtros junto con el campo de búsqueda para ubicar productos con precisión

## Configuración de Código de Barras

### Campos de Búsqueda

La funcionalidad de búsqueda incluye automáticamente tu campo de código de barras configurado. El campo de código de barras utilizado para la búsqueda depende de tu configuración de ajustes de POS.

## Preguntas Frecuentes

<details>

<summary>
  ¿Qué es el campo _global_unique_id para códigos de barras?
</summary>

El campo `_global_unique_id` es un nuevo campo de código de barras que WooCommerce agregó recientemente para proporcionar una mejor estandarización de códigos de barras entre tiendas.

**Puntos Clave:**

- **Estándar Moderno**: Este campo fue diseñado específicamente para la identificación global de códigos de barras
- **Configuración de POS**: Puedes configurar el POS para usar `_global_unique_id` como campo de código de barras en los ajustes de POS
- **Heredado vs. Nuevo**: La configuración heredada de código de barras usa el campo `_sku`, pero esto cambiará a `_global_unique_id` en futuras versiones
- **Flexibilidad**: Puedes configurar cualquier campo meta de producto como tu campo de código de barras si usas complementos de código de barras de terceros
- **Valor Predeterminado Futuro**: `_global_unique_id` se convertirá en el campo de código de barras predeterminado en futuras actualizaciones de POS

Para configurar qué campo usa el POS para códigos de barras, visita tus ajustes de POS en el área de administración de WordPress.

</details>

<details>

<summary>
  ¿Por qué no veo todos mis productos cuando busco?
</summary>

WCPOS usa descarga progresiva de productos para mantener el rendimiento. Si no ves un producto:

1. **Intenta buscarlo** - Esto hará que el POS lo busque en tu servidor
2. **Espera la descarga** - El POS descargará el producto y otros en pequeños lotes
3. **Busca de nuevo** - Una vez descargado, el producto aparecerá en búsquedas futuras

Este proceso garantiza que tu POS se mantenga rápido y receptivo incluso con miles de productos. Aprende más sobre esto en nuestra guía de [Sincronización de Productos](/products/sync).

</details>

<details>

<summary>
  ¿Puedo buscar nombres de productos o SKU parciales?
</summary>

¡Sí! La búsqueda tokenizada usa coincidencia adelantada, lo que significa:

- Buscar "azul" encontrará productos con "arándano", "camisa azul", etc.
- Buscar "ABC" encontrará SKU como "ABC123", "ABC-XYZ", etc.
- No necesitas escribir palabras o códigos completos

La búsqueda está diseñada para encontrar productos rápidamente con un mínimo de escritura.

</details>
