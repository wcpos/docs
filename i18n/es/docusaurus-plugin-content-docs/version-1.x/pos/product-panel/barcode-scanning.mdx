La mayoría de los escáneres de códigos de barras se comportan como un teclado conectado a tu dispositivo.
Cuando escaneas un código de barras, WCPOS detecta que los caracteres fueron ingresados más rápido de lo normal al teclear.
Utiliza estas "presiones de teclas rápidas" para identificar la entrada como un escaneo de código de barras.

## Configuración del Escaneo de Códigos de Barras {#configuring-barcode-scanning}

Dado que un escaneo de código de barras ocurre muy rápido, el POS puede diferenciar entre un código de barras y algo escrito a mano.
En la configuración del POS, encontrarás opciones para ajustar cómo funciona la detección de códigos de barras.

- **Tiempo de entrada promedio:** Cuán rápido debe ser la entrada para ser considerada un código de barras.
- **Longitud mínima:** Cuán larga debe ser la cadena continua de caracteres para ser tratada como un código de barras.
- **Eliminación de prefijos/sufijos:** Si tu escáner añade caracteres extra (como un prefijo o sufijo), puedes eliminarlos automáticamente para que solo permanezca el código de barras principal.

## ¿Qué Ocurre Cuando se Detecta un Código de Barras? {#what-happens-when-a-barcode-is-detected}

Cuando el POS detecta un código de barras, busca en su base de datos local para encontrar un producto o variación de producto coincidente.
Hay tres posibles resultados:

1. **Coincidencia Exacta (Un Producto Encontrado):**
   Si el código de barras coincide exactamente con un solo producto o variación de producto en la base de datos local del POS, ese producto se añade automáticamente a tu carrito.

2. **Múltiples Coincidencias (Más de un Producto Encontrado):**
   Si más de un producto tiene el mismo código de barras, el POS no puede saber cuál añadir.
   En este caso, coloca el código de barras escaneado en la barra de búsqueda para que puedas revisar qué productos comparten ese código.
   Cuando ves múltiples coincidencias, a menudo significa que los datos del producto necesitan una actualización: cada producto debe tener un código de barras único.

3. **Sin Coincidencias Locales (Cero Productos Encontrados):**
   Si el código de barras no coincide con ningún producto almacenado en tu dispositivo, el código escaneado aparecerá en la barra de búsqueda.
   En este punto, el POS intentará encontrar una coincidencia en línea a través de la base de datos de tu tienda WooCommerce.

## Comprendiendo la Sincronización de Productos {#understanding-product-synchronisation}

### Descarga Progresiva de Productos {#progressive-product-downloading}

WCPOS no carga todos tus productos de una vez.
En su lugar, los descarga en pequeños lotes.
Este enfoque previene desaceleraciones y asegura que tu tienda funcione sin problemas.
Con el tiempo, a medida que usas el POS y realizas búsquedas, más de tus productos se almacenan localmente en tu dispositivo.

Consulta [Sincronización de Productos](/products/sync) para más detalles.

### Por Qué es Importante para el Escaneo de Códigos de Barras {#why-it-matters-for-barcode-scanning}

Cuando escaneas un código de barras que aún no está almacenado localmente, el POS "se conecta en línea" a tu tienda WooCommerce para encontrarlo.
Como parte de este proceso, descargará ese producto (y otros en pequeños lotes) y los guardará.
Esto significa que con el tiempo, el POS se vuelve más rápido y eficiente a medida que más productos son almacenados localmente.

### Cómo Acelerar el Proceso {#how-to-speed-up-the-process}

Simplemente buscar productos en el POS ayuda a descargar más de tu inventario.
Cuanto más uses la búsqueda —y cuanto más escanees— más completo se vuelve tu base de datos local.

## PREGUNTAS FRECUENTES {#faq}

<Accordion>
  <AccordionItem question="¿Por qué obtengo '0 productos encontrados localmente' cuando escaneo un código de barras?">

No todos los productos están disponibles localmente desde el principio.
El POS descarga gradualmente productos de tu tienda en línea y los almacena en tu dispositivo.
Si el producto que acabas de escanear aún no está almacenado, la búsqueda activa el POS para buscarlo en línea y luego descargarlo para que esté disponible en el futuro.

  </AccordionItem>

  <AccordionItem question="¿El POS genera e imprime códigos de barras?">

No, no en este momento. Nuestro POS está diseñado para escanear y leer códigos de barras existentes, pero no incluye funcionalidad para crear o imprimirlos.
Si necesitas generar códigos de barras para tus productos, puedes utilizar plugins de WooCommerce de terceros que se especializan en la creación e impresión de códigos de barras. Algunos ejemplos incluyen:

- [EAN for WooCommerce](https://wordpress.org/plugins/ean-for-woocommerce/)
- [A4 Barcode Generator](https://wordpress.org/plugins/a4-barcode-generator/)

Una vez que hayas generado códigos de barras para tus productos, puedes escanearlos fácilmente en la caja para acelerar el proceso de pago en el POS.

  </AccordionItem>
</Accordion>
