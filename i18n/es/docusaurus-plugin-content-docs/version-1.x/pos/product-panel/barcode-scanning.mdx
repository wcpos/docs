La mayoría de los escáneres de códigos de barras se comportan como un teclado conectado a su dispositivo.
Cuando se escanea un código de barras, WCPOS detecta que los caracteres se han ingresado más rápido de lo normal.
Utiliza estas "pulsaciones de teclas rápidas" para identificar la entrada como un escaneo de código de barras.

## Configuración del Escaneo de Códigos de Barras

Dado que un escaneo de código de barras ocurre muy rápidamente, el POS puede diferenciar entre un código de barras y algo escrito a mano.
En la configuración del POS, encontrará opciones para ajustar con precisión cómo funciona la detección de códigos de barras.

- **Tiempo de entrada promedio:** Qué tan rápida debe ser la entrada para contar como un código de barras.
- **Longitud mínima:** Qué tan larga debe ser la cadena continua de caracteres para ser tratada como un código de barras.
- **Eliminación de prefijo/sufijo:** Si su escáner agrega caracteres adicionales (como un prefijo o sufijo), puede eliminarlos automáticamente para que solo quede el código de barras principal.

## Qué Sucede Cuando se Detecta un Código de Barras

Cuando el POS detecta un código de barras, busca en su base de datos local para encontrar un producto o variación de producto coincidente.
Hay tres posibles resultados:

1. **Coincidencia Exacta (Se Encontró Un Producto):**
   Si el código de barras coincide exactamente con un solo producto o variación de producto en la base de datos local del POS, ese producto se agrega automáticamente a su carrito.

2. **Múltiples Coincidencias (Se Encontraron Varios Productos):**
   Si más de un producto tiene el mismo código de barras, el POS no puede saber cuál agregar.
   En este caso, coloca el código de barras escaneado en la barra de búsqueda para que pueda revisar qué productos comparten ese código.
   Cuando ve varias coincidencias, a menudo significa que los datos del producto necesitan actualizarse: cada producto debería tener un código de barras único.

3. **Sin Coincidencias Locales (No se Encontraron Productos):**
   Si el código de barras no coincide con ningún producto almacenado en su dispositivo, el código escaneado aparecerá en la barra de búsqueda.
   En este punto, el POS intentará encontrar una coincidencia en línea a través de la base de datos de su tienda WooCommerce.

## Entendiendo la Sincronización de Productos

### Descarga Progresiva de Productos

WCPOS no carga todos sus productos de una vez.
En su lugar, los descarga en pequeños lotes.
Este enfoque evita ralentizaciones y garantiza que su tienda funcione sin problemas.
Con el tiempo, a medida que usa el POS y realiza búsquedas, más de sus productos se almacenan localmente en su dispositivo.

Consulte [Sincronización de Productos](/products/sync) para más detalles.

### Por Qué Es Importante para el Escaneo de Códigos de Barras

Cuando escanea un código de barras que aún no está almacenado localmente, el POS "se conectará" a su tienda WooCommerce para encontrarlo.
Como parte de este proceso, descargará ese producto (y otros en pequeños lotes) y lo guardará.
Esto significa que con el tiempo, el POS se vuelve más rápido y eficiente a medida que se almacenan más productos localmente.

### Cómo Acelerar el Proceso

Simplemente buscar productos en el POS ayuda a descargar más de su inventario.
Cuanto más use la búsqueda, y cuanto más escanee, más completa se vuelve su base de datos local.

## Preguntas Frecuentes

<details>

<summary>
  ¿Por qué obtengo "0 productos encontrados localmente" cuando escaneo un código de barras?
</summary>

No todos los productos están disponibles localmente desde el principio.
El POS descarga gradualmente productos de su tienda en línea y los almacena en su dispositivo.
Si el producto que acaba de escanear no está almacenado todavía, la búsqueda hace que el POS lo busque en línea y luego lo descargue para que esté disponible en el futuro.

</details>

<details>

<summary>¿Genera e imprime el POS códigos de barras?</summary>

No, por ahora no. Nuestro POS está diseñado para escanear y leer códigos de barras existentes, pero no incluye funcionalidad para crearlos o imprimirlos.
Si necesita generar códigos de barras para sus productos, puede utilizar complementos de WooCommerce de terceros que se especializan en la creación e impresión de códigos de barras. Algunos ejemplos incluyen:

- [EAN for WooCommerce](https://wordpress.org/plugins/ean-for-woocommerce/)
- [A4 Barcode Generator](https://wordpress.org/plugins/a4-barcode-generator/)

Una vez que haya generado códigos de barras para sus productos, puede escanearlos fácilmente en el registro para acelerar el proceso de pago en el POS.

</details>
