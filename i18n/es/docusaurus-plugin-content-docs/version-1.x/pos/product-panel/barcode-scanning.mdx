La mayoría de los escáneres de códigos de barras se comportan como un teclado conectado a su dispositivo. 
Cuando escanea un código de barras, el WCPOS detecta que los caracteres se ingresaron más rápido de lo normal al escribir. 
Utiliza estas "presiones rápidas de teclas" para identificar la entrada como un escaneo de código de barras.

## Configurando el Escaneo de Códigos de Barras

Dado que un escaneo de código de barras sucede muy rápido, el POS puede diferenciar entre un código de barras y algo escrito a mano. 
En las configuraciones del POS, encontrará opciones para ajustar cómo funciona la detección de códigos de barras.

- **Tiempo promedio de entrada:** Qué tan rápido debe ser la entrada para contar como un código de barras.
- **Longitud mínima:** Qué tan larga debe ser la cadena continua de caracteres para ser tratada como un código de barras.
- **Eliminación de prefijos/sufijos:** Si su escáner agrega caracteres extra (como un prefijo o sufijo), puede eliminarlos automáticamente para que solo permanezca el código de barras principal.

## ¿Qué Sucede Cuando se Detecta un Código de Barras?

Cuando el POS detecta un código de barras, busca en su base de datos local para encontrar un producto o variación de producto coincidente. 
Hay tres posibles resultados:

1. **Coincidencia Exacta (Un Producto Encontrado):** 
   Si el código de barras coincide exactamente con un solo producto o variación de producto en la base de datos local del POS, ese producto se agrega automáticamente a su carrito.

2. **Múltiples Coincidencias (Más de un Producto Encontrado):** 
   Si más de un producto tiene el mismo código de barras, el POS no puede saber cuál agregar. 
   En este caso, coloca el código de barras escaneado en la barra de búsqueda para que pueda revisar qué productos comparten ese código. 
   Cuando ve múltiples coincidencias, a menudo significa que los datos del producto necesitan actualizarse: cada producto debe tener un código de barras único.

3. **Sin Coincidencias Locales (Cero Productos Encontrados):** 
   Si el código de barras no coincide con ningún producto almacenado en su dispositivo, el código escaneado aparecerá en la barra de búsqueda. 
   En este punto, el POS intentará encontrar una coincidencia en línea a través de la base de datos de su tienda de WooCommerce.

## Comprendiendo la Sincronización de Productos

### Descarga Progresiva de Productos

WCPOS no carga todos sus productos de una vez. 
En su lugar, los descarga en pequeños lotes. 
Este enfoque evita ralentizaciones y asegura que su tienda funcione sin problemas. 
Con el tiempo, a medida que usa el POS y realiza búsquedas, más de sus productos se almacenan localmente en su dispositivo.

Vea [Sincronización de Productos](/products/sync) para más detalles.

### Por Qué es Importante para el Escaneo de Códigos de Barras

Cuando escanea un código de barras que aún no está almacenado localmente, el POS "se conectará en línea" a su tienda de WooCommerce para encontrarlo. 
Como parte de este proceso, descargará ese producto (y otros en pequeños lotes) y los guardará. 
Esto significa que con el tiempo, el POS se vuelve más rápido y eficiente a medida que más productos se almacenan localmente.

### Cómo Acelerar el Proceso

Simplemente buscar productos en el POS ayuda a que descargue más de su inventario. 
Cuanto más utilice la búsqueda — y cuanto más escanee — más completa se vuelve su base de datos local.

## F.A.Q.

<details>

<summary>
  ¿Por qué obtengo "0 productos encontrados localmente" cuando escaneo un código de barras?
</summary>

No todos los productos están disponibles localmente desde el principio. 
El POS descarga gradualmente productos de su tienda en línea y los almacena en su dispositivo. 
Si el producto que acaba de escanear aún no está almacenado, la búsqueda activa al POS para buscarlo en línea y luego descargarlo para que esté disponible en el futuro.

</details>

<details>

<summary>¿El POS genera e imprime códigos de barras?</summary>

No, en este momento. Nuestro POS está diseñado para escanear y leer códigos de barras existentes, pero no incluye funcionalidad para crear o imprimirlos. 
Si necesita generar códigos de barras para sus productos, puede usar complementos de WooCommerce de terceros que se especializan en la creación e impresión de códigos de barras. Algunos ejemplos incluyen:

- [EAN para WooCommerce](https://wordpress.org/plugins/ean-for-woocommerce/)
- [Generador de Códigos de Barras A4](https://wordpress.org/plugins/a4-barcode-generator/)

Una vez que haya generado códigos de barras para sus productos, puede escanearlos fácilmente en la caja para acelerar el proceso de pago en el POS.

</details>
