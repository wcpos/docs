## Cómo Funciona la Detección de Códigos de Barras

La mayoría de los lectores de códigos de barras se comportan como un teclado conectado a su dispositivo. Cuando se escanea un código de barras, el POS detecta que los caracteres se ingresaron más rápido de lo que se escribe normalmente. Utiliza estas "pulsaciones de teclas rápidas" para identificar la entrada como un escaneo de código de barras.

## Configuraciones

| Configuración | Descripción |
|--------------|-------------|
| **Tiempo promedio de entrada** | Qué tan rápida debe ser la entrada (en milisegundos) para contar como un código de barras. Valores más bajos = entrada más rápida requerida |
| **Longitud mínima** | Qué tan larga debe ser la cadena continua de caracteres para ser tratada como un código de barras |
| **Eliminación de prefijo** | Caracteres a eliminar del inicio de los códigos de barras escaneados |
| **Eliminación de sufijo** | Caracteres a eliminar del final de los códigos de barras escaneados |

## Eliminación de Prefijo/Sufijo

Si su escáner añade caracteres extra (como un prefijo o sufijo), puede eliminarlos automáticamente para que solo quede el código de barras principal.

**Ejemplo:** Si su escáner añade "A" antes de cada código de barras y "Z" después, configure:
- Eliminación de prefijo: `A`
- Eliminación de sufijo: `Z`

## Resolución de Problemas

### Código de barras no detectado

- **Escáner demasiado lento**: Intente aumentar el valor de "Tiempo promedio de entrada"
- **Código de barras demasiado corto**: Intente disminuir el valor de "Longitud mínima"
- **Caracteres extra**: Verifique si su escáner está añadiendo prefijos/sufijos

### Escritura normal detectada como código de barras

- **Escritura demasiado rápida**: Intente disminuir el valor de "Tiempo promedio de entrada"
- **Entrada demasiado larga**: Intente aumentar el valor de "Longitud mínima"
