## Cómo Funciona la Detección de Códigos de Barras

La mayoría de los escáneres de códigos de barras se comportan como un teclado conectado a tu dispositivo. Cuando escaneas un código de barras, el POS detecta que los caracteres fueron ingresados más rápido de lo normal. Utiliza estas "pulsaciones de teclas rápidas" para identificar la entrada como un escaneo de código de barras.

## Configuraciones

| Configuración | Descripción |
|---------------|-------------|
| **Tiempo promedio de entrada** | Qué tan rápido debería ser la entrada (en milisegundos) para contar como un código de barras. Valores más bajos = entrada más rápida requerida |
| **Longitud mínima** | Qué tan largo debe ser la secuencia continua de caracteres para ser tratado como un código de barras |
| **Eliminación de prefijos** | Caracteres a eliminar del inicio de los códigos de barras escaneados |
| **Eliminación de sufijos** | Caracteres a eliminar del final de los códigos de barras escaneados |

## Eliminación de Prefijos/Sufijos

Si tu escáner añade caracteres extras (como un prefijo o sufijo), puedes eliminarlos automáticamente para que solo quede el código de barras principal.

**Ejemplo:** Si tu escáner añade "A" antes de cada código de barras y "Z" después, configura:
- Eliminación de prefijos: `A`
- Eliminación de sufijos: `Z`

## Solución de Problemas

<Accordion>
  <AccordionItem question="Código de barras no detectado">

  - **Escáner demasiado lento**: Intenta aumentar el valor de "Tiempo promedio de entrada"
  - **Código de barras demasiado corto**: Intenta disminuir el valor de "Longitud mínima"
  - **Caracteres extra**: Verifica si tu escáner está añadiendo prefijos/sufijos

  </AccordionItem>

  <AccordionItem question="Escritura regular detectada como código de barras">

  - **Tecleando demasiado rápido**: Intenta disminuir el valor de "Tiempo promedio de entrada"
  - **Entrada demasiado larga**: Intenta aumentar el valor de "Longitud mínima"

  </AccordionItem>
</Accordion>
