## Cómo Funciona la Detección de Códigos de Barras

La mayoría de los escáneres de códigos de barras se comportan como un teclado conectado a tu dispositivo. Cuando escaneas un código de barras, el POS detecta que los caracteres se ingresaron más rápido que la escritura normal. Utiliza estas "pulsaciones rápidas" para identificar la entrada como un escaneo de código de barras.

## Configuraciones

| Configuración | Descripción |
|---------------|-------------|
| **Tiempo de entrada promedio** | Qué tan rápido debe ser la entrada (en milisegundos) para contar como un código de barras. Valores más bajos = entrada más rápida requerida |
| **Longitud mínima** | Qué tan larga debe ser la cadena continua de caracteres para ser tratada como un código de barras |
| **Eliminación de prefijos** | Caracteres a eliminar del principio de los códigos de barras escaneados |
| **Eliminación de sufijos** | Caracteres a eliminar del final de los códigos de barras escaneados |

## Eliminación de Prefijos/Sufijos

Si tu escáner agrega caracteres extra (como un prefijo o sufijo), puedes eliminarlos automáticamente para que solo quede el código de barras principal.

**Ejemplo:** Si tu escáner agrega "A" antes de cada código de barras y "Z" después, configura:
- Eliminación de prefijos: `A`
- Eliminación de sufijos: `Z`

## Solución de Problemas

### Código de barras no detectado

- **Escáner demasiado lento**: Intenta aumentar el valor de "Tiempo de entrada promedio"
- **Código de barras demasiado corto**: Intenta disminuir el valor de "Longitud mínima"
- **Caracteres extra**: Verifica si tu escáner está agregando prefijos/sufijos

### Escritura regular detectada como código de barras

- **Escribiendo demasiado rápido**: Intenta disminuir el valor de "Tiempo de entrada promedio"
- **Entrada demasiado larga**: Intenta aumentar el valor de "Longitud mínima"
