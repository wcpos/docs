La página de Configuración General te permite configurar la funcionalidad principal de WCPOS. Estas configuraciones afectan cómo se muestran los productos, cómo se gestionan las cantidades y cómo funciona el escaneo de códigos de barras.

## Habilitar Productos Solo de POS {#enable-pos-only-products}

Esta configuración añade opciones de visibilidad a tus productos, permitiéndote controlar si aparecen en tu tienda en línea, en el POS o en ambos.

:::warning Impacto en el Rendimiento
Solo habilita esta configuración si necesitas ocultar productos de tu tienda en línea o de POS. Cuando está habilitada, agrega una búsqueda adicional en la base de datos por cada solicitud de producto.
:::

Para información detallada sobre cómo usar esta función, incluyendo cómo establecer visibilidad en productos individuales y edición masiva, consulta [Productos Solo de POS](/products/pos-only-products).

## Habilitar Cantidades Decimales {#enable-decimal-quantities}

Esta configuración permite que los clientes y el personal ingresen cantidades decimales al agregar productos al carrito.

**Ejemplos de cantidades decimales:**
- 0.25 (cuarto de unidad)
- 1.5 (una y media unidades)
- 2.75 (dos y tres cuartos de unidad)

**Útil para:**
- Productos vendidos por peso (por ejemplo, 0.5 kg de granos de café)
- Servicios fraccionarios (por ejemplo, 1.5 horas de consulta)
- Productos al por mayor que pueden ser vendidos en cantidades parciales

## Generar Nombre de Usuario Automáticamente a partir del Correo Electrónico del Cliente {#automatically-generate-username-from-customer-email}

Cuando se habilita, WCPOS creará automáticamente nombres de usuario para nuevos clientes basándose en sus direcciones de correo electrónico durante el proceso de pago. Esto agiliza el proceso de creación de clientes y asegura un formato de nombre de usuario consistente.

## Cliente Predeterminado de POS {#default-pos-customer}

Selecciona el cliente predeterminado que se asignará a los nuevos pedidos de POS. Las opciones comunes incluyen:

- **Invitado**: Los pedidos no están asociados a ninguna cuenta de usuario
- **Cajero**: Los pedidos están asociados a la cuenta del cajero que ha iniciado sesión
- **Cliente Presencial**: Los pedidos están asociados a una cuenta de cliente específica (necesitarás crear esta cuenta primero)

## Usar Cuenta de Cajero {#use-cashier-account}

Cuando está habilitada, la cuenta de usuario del cajero se utilizará como el cliente predeterminado para los pedidos de POS en lugar del Cliente Predeterminado de POS seleccionado.

## Campo de Código de Barras {#barcode-field}

Esta configuración determina qué campo de producto buscará WCPOS al escanear códigos de barras o al usar la funcionalidad de búsqueda de códigos de barras.

:::warning Importante: Limpieza y Actualización Requeridas
Después de cambiar la configuración del campo de código de barras, **debes** limpiar y actualizar tus datos de producto en el POS para que el cambio surta efecto.

**Cómo Limpiar y Actualizar:**
1. Ve a la pantalla de Productos en tu POS
2. **Presiona largo** el ícono de sincronización (↻) en la parte inferior de la pantalla
3. Selecciona **"Limpiar y Refrescar"** del menú contextual
4. Espera a que los productos se recarguen con los nuevos datos del campo de código de barras

Esto es necesario porque el POS almacena en caché los datos del producto localmente, y cambiar la configuración del campo de código de barras no actualiza automáticamente los datos en caché.
:::

### Opción Predeterminada {#default-option}

**`_global_unique_id`** - Este es el campo de ID Único Global de WooCommerce, introducido en WooCommerce 8.2 (julio de 2024). Está diseñado para almacenar códigos GTIN, UPC, EAN o ISBN y es la opción recomendada para nuevas instalaciones.

### Otras Opciones Comunes {#other-common-options}

**`_sku`** - El campo SKU del producto (Unidad de Mantenimiento de Inventario). Esta es una buena opción de respaldo si no estás usando el campo de ID Único Global.

### Claves Meta de Plugins de Terceros {#third-party-plugin-meta-keys}

Si estás utilizando un plugin de gestión de código de barras, es posible que necesites usar su clave meta específica:

| Tipo de Plugin | Claves Meta Comunes | Descripción |
|----------------|---------------------|-------------|
| Plugins EAN    | `_ean`, `_ean13`, `_ean8` | Códigos de Número de Artículo Europeo |
| Plugins UPC    | `_upc`, `_upc_code` | Código de Producto Universal |
| Plugins GTIN   | `_gtin`, `_gtin13`, `_gtin14` | Número Global de Artículo Comercial |
| Código de Barras Genérico | `_barcode`, `_product_barcode` | Campos genéricos de código de barras |
| Códigos de Barras de Productos WooCommerce | `_barcode_text` | Plugin popular de códigos de barras |
| ATUM Inventario | `_atum_barcode` | Gestión de inventario ATUM |

### Encontrar la Clave Meta de tu Plugin {#finding-your-plugins-meta-key}

Para encontrar la clave meta correcta para tu plugin de código de barras:

1. Edita un producto en WP Admin
2. Agrega un código de barras utilizando tu plugin
3. Consulta la documentación del plugin para el nombre de la clave meta
4. O inspecciona la tabla de base de datos `wp_postmeta` para ver qué clave meta fue creada

### Claves Meta Personalizadas {#custom-meta-keys}

También puedes usar cualquier clave meta personalizada que almacene datos de código de barras. Simplemente ingresa el nombre de la clave meta (incluyendo el prefijo de subrayado si corresponde) en este campo.

:::tip
Prueba el escaneo de códigos de barras después de cambiar esta configuración para asegurarte de que esté funcionando correctamente con tu campo elegido. Recuerda limpiar y refrescar tus datos de producto en el POS para que el cambio surta efecto.
:::
