La página de Ajustes generales permite configurar la funcionalidad principal de WCPOS. Estos ajustes afectan la forma en que se muestran los productos, cómo se gestionan las cantidades y cómo funciona el escaneo de códigos de barras.

## Activar productos exclusivos del POS {#enable-pos-only-products}

Este ajuste añade opciones de visibilidad a los productos, permitiendo controlar si aparecen en la tienda en línea, en el POS o en ambos.

:::warning Impacto en el rendimiento
Solo se debe activar este ajuste si es necesario ocultar productos de la tienda en línea o del POS. Al activarlo, se añade una consulta adicional a la base de datos por cada solicitud de producto.
:::

Para obtener información detallada sobre el uso de esta función, incluyendo cómo establecer la visibilidad en productos individuales y la edición masiva, consulte [Productos exclusivos del POS](/products/pos-only-products).

## Habilitar cantidades decimales {#enable-decimal-quantities}

Esta configuración permite a los clientes y al personal introducir cantidades decimales al agregar productos al carrito.

**Ejemplos de cantidades decimales:**
- 0,25 (un cuarto de unidad)
- 1,5 (una unidad y media)
- 2,75 (dos unidades y tres cuartos)

**Útil para:**
- Productos vendidos por peso (p. ej., 0,5 kg de granos de café)
- Servicios fraccionados (p. ej., 1,5 horas de consultoría)
- Productos a granel que pueden venderse en cantidades parciales

## Restaurar stock al eliminar un pedido {#restore-stock-on-order-delete}

Cuando está habilitado, WCPOS restaura automáticamente el stock de los productos cuando un pedido de POS se envía a la papelera o se elimina, de modo que cancelar una venta errónea devuelve los artículos al inventario. Esta configuración está **activada de forma predeterminada**.

Desactivar esta opción si el inventario se concilia de otra manera y no se desea que los pedidos POS eliminados ajusten el stock.

## Generar automáticamente el nombre de usuario a partir del correo electrónico del cliente {#automatically-generate-username-from-customer-email}

Cuando está habilitado, WCPOS creará automáticamente nombres de usuario para los nuevos clientes basándose en sus direcciones de correo electrónico durante el proceso de pago. Esto agiliza la creación de clientes y garantiza un formato de nombre de usuario consistente.

## Cliente POS predeterminado {#default-pos-customer}

Seleccionar el cliente predeterminado que se asignará a los nuevos pedidos POS. Las opciones comunes incluyen:

- **Invitado**: Los pedidos no se asocian con ninguna cuenta de usuario
- **Cajero**: Los pedidos se asocian con la cuenta del cajero que ha iniciado sesión
- **Cliente ocasional**: Los pedidos se asocian con una cuenta de cliente específica (es necesario crear esta cuenta previamente)

## Usar cuenta del cajero {#use-cashier-account}

Cuando está habilitado, la cuenta de usuario del cajero se utilizará como cliente predeterminado para los pedidos POS en lugar del cliente POS predeterminado seleccionado.

## Campo de código de barras {#barcode-field}

Este ajuste determina en qué campo del producto buscará WCPOS al escanear códigos de barras o al utilizar la función de búsqueda por código de barras.

:::warning Importante: Es necesario borrar y actualizar
Después de cambiar la configuración del campo de código de barras, es **necesario** borrar y actualizar los datos de productos en el POS para que el cambio surta efecto.

**Cómo borrar y actualizar:**
1. Ir a la pantalla de Productos en el POS
2. **Mantener presionado** el icono de sincronización (↻) en la parte inferior de la pantalla
3. Seleccionar **"Borrar y actualizar"** en el menú contextual
4. Esperar a que los productos se recarguen con los nuevos datos del campo de código de barras

Esto es necesario porque el POS almacena en caché los datos de productos de forma local, y cambiar la configuración del campo de código de barras no actualiza automáticamente los datos almacenados en caché.
:::

### Opción predeterminada {#default-option}

**`_global_unique_id`** - Este es el campo de ID único global de WooCommerce, introducido en WooCommerce 8.2 (julio de 2024). Está diseñado para almacenar códigos GTIN, UPC, EAN o ISBN y es la opción recomendada para nuevas instalaciones.

### Otras opciones comunes {#other-common-options}

**`_sku`** - El campo SKU (Stock Keeping Unit) del producto. Es una buena opción alternativa si no se utiliza el campo de identificador único global.

### Claves meta de plugins de terceros {#third-party-plugin-meta-keys}

Si se utiliza un plugin de gestión de códigos de barras, es posible que sea necesario usar su clave meta específica:

| Tipo de plugin | Claves meta comunes | Descripción |
|-------------|------------------|-------------|
| Plugins EAN | `_ean`, `_ean13`, `_ean8` | Códigos de número de artículo europeo |
| Plugins UPC | `_upc`, `_upc_code` | Código universal de producto |
| Plugins GTIN | `_gtin`, `_gtin13`, `_gtin14` | Número global de artículo comercial |
| Código de barras genérico | `_barcode`, `_product_barcode` | Campos de código de barras genéricos |
| Códigos de barras de productos de WooCommerce | `_barcode_text` | Plugin de códigos de barras popular |
| Inventario ATUM | `_atum_barcode` | Gestión de inventario ATUM |

### Cómo encontrar la clave meta de un plugin {#finding-your-plugins-meta-key}

Para encontrar la clave meta correcta del plugin de códigos de barras:

1. Editar un producto en WP Admin
2. Agregar un código de barras con el plugin
3. Consultar la documentación del plugin para obtener el nombre de la clave meta
4. O inspeccionar la tabla `wp_postmeta` de la base de datos para ver qué clave meta se creó

### Claves meta personalizadas {#custom-meta-keys}

También es posible utilizar cualquier clave meta personalizada que almacene datos de códigos de barras. Basta con introducir el nombre de la clave meta (incluido el prefijo de guion bajo, si corresponde) en este campo.

:::tip
Se recomienda probar el escaneo de códigos de barras después de cambiar esta configuración para asegurar que funciona correctamente con el campo elegido. Es necesario borrar y actualizar los datos de productos en el POS para que el cambio surta efecto.
:::

## Privacidad {#privacy}

El interruptor **Permitir datos de uso anónimos** controla si WCPOS recopila analíticas de uso anónimas para ayudar a mejorar el producto. Se trata de una opción genuina de aceptación/rechazo: nunca se recopilan datos de clientes, precios ni direcciones de la tienda. El enlace **Más información** junto a la configuración permite ver exactamente qué se recopila y qué no.

## Identificadores fiscales de la tienda {#store-tax-ids}

Permite agregar los identificadores fiscales de la empresa — número de IVA, ABN, GSTIN, EIN u otros números de registro mercantil locales — que WCPOS imprime en los recibos y facturas del POS. WooCommerce no dispone de un campo integrado para estos datos, y muchas regiones requieren más de uno.

Consulte [Identificadores fiscales de la tienda](/settings/wp-admin/store-tax-ids) para obtener la guía completa.

## Identificadores fiscales del cliente {#customer-tax-ids}

WCPOS mantiene un campo de identificador fiscal en los clientes y lo copia en sus pedidos durante el proceso de pago. Dado que diferentes plugins almacenan este campo con distintos nombres, WCPOS detecta qué plugin está instalado y escribe en el campo correcto de forma automática. La sección **Identificadores fiscales** aquí también permite sobrescribir el nombre del campo si es necesario.

Consulte [Identificadores fiscales del cliente](/settings/wp-admin/customer-tax-ids) para obtener la guía completa.
