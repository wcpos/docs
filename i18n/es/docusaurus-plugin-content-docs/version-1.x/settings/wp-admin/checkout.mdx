La página de configuración de caja (`WP Admin > POS > Ajustes > Finalizar compra`) controla qué pasarelas de pago aparecen en el POS, cómo se procesan los pedidos y qué correos electrónicos de notificación se envían para las ventas del POS.

## Notificaciones por correo electrónico {#email-notifications}

En la parte superior de la página se encuentran tres grupos de opciones de correo electrónico — **Correos de administrador**, **Correos de cliente** y **Correos de cajero** — que controlan qué correos electrónicos de notificación de WooCommerce se envían para los pedidos del POS, de forma independiente a la configuración general de correos electrónicos de WooCommerce del sitio.

Consulte **[Notificaciones por correo electrónico](/settings/wp-admin/email-notifications)** para la referencia completa, incluyendo los valores predeterminados de cada correo electrónico, el modelo de destinatario de cajero, escenarios comunes y hooks para desarrolladores.

## Pasarelas de pago {#payment-gateways}

Esta sección enumera todas las pasarelas de pago de WooCommerce instaladas en el sitio. Cada pasarela se puede activar o desactivar de forma independiente para el POS.

| Columna | Descripción |
|--------|-------------|
| **Predeterminada** | Permite elegir qué pasarela aparece preseleccionada cuando un cajero abre la caja |
| **Pasarela** | El nombre de la pasarela de pago |
| **ID de pasarela** | El identificador de pasarela de WooCommerce (útil para desarrolladores) |
| **Habilitada** | Activar o desactivar la pasarela para el cobro en POS |
| **Estado del pedido** | El estado que se asigna a un pedido de POS cuando se paga con esta pasarela (consulte [Estado del pedido](#order-status) a continuación) |

Cada fila también tiene un menú (⋮) con **Ajustes de POS** — para cambiar el **Título** y la **Descripción** exclusivos de POS de la pasarela — y **Ajustes de WooCommerce**, que enlaza directamente a la configuración completa de la pasarela en WooCommerce.

### Habilitar una pasarela {#enabling-a-gateway}

1. Buscar la pasarela en la lista
2. Activar el interruptor **Habilitada**
3. La pasarela aparecerá en el cobro de POS en la siguiente carga de página

:::note Versión gratuita vs Pro
En la versión gratuita solo se pueden editar las pasarelas integradas **Efectivo** y **Tarjeta**. Para habilitar otras pasarelas de WooCommerce instaladas en el POS se requiere [WCPOS Pro](/getting-started/pro-license).
:::

### Reordenar pasarelas {#reordering-gateways}

Arrastre las pasarelas para cambiar su orden de visualización en el proceso de pago del POS. Utilice la columna **Predeterminada** para elegir qué pasarela está preseleccionada cuando un cajero abre el modal de pago.

### POS vs Tienda en línea {#pos-vs-online-store}

La configuración de pasarelas aquí es **independiente** de la configuración de pagos de WooCommerce. Una pasarela puede estar:

- **Habilitada para POS, deshabilitada en línea** — útil para pagos en efectivo que no aplican a pedidos web
- **Habilitada en línea, deshabilitada para POS** — útil para pasarelas como PayPal que no son prácticas para ventas presenciales
- **Habilitada para ambos** — la pasarela funciona en todas partes

:::note
Una pasarela debe estar instalada y activa en `WP Admin > WooCommerce > Ajustes > Pagos` antes de aparecer en la lista de configuración de pago del POS. Si falta una pasarela, verifique que el plugin esté instalado y que la pasarela esté habilitada en WooCommerce.
:::

## Estado del pedido {#order-status}

El estado del pedido se configura **por pasarela**, desde la columna **Estado del pedido** en la tabla de pasarelas — de modo que una venta puede resolverse de forma diferente según cómo se haya pagado:

- **Efectivo** y **Tarjeta** tienen como valor predeterminado **Completado**, ya que el pago se cobra en el punto de venta.
- Las pasarelas en las que el dinero llega después — como **BACS** (transferencia bancaria directa) o **Cheque** — se establecen de forma predeterminada en **En espera**.

Es posible seleccionar cualquier estado de pedido de WooCommerce para cada pasarela. (Las versiones anteriores utilizaban un único valor predeterminado global; ese valor se migró a cada pasarela durante la actualización.)
