La página de Ajustes de pago (`WP Admin > POS > Ajustes > Finalizar compra`) controla qué pasarelas de pago aparecen en el POS, cómo se procesan los pedidos y qué correos electrónicos de notificación se envían para las ventas del POS.

## Notificaciones por correo electrónico {#email-notifications}

En la parte superior de la página se encuentran tres grupos de opciones de correo electrónico — **Correos de administración**, **Correos de cliente** y **Correos de cajero** — que controlan qué correos electrónicos de notificación de WooCommerce se envían para los pedidos del POS, de forma independiente a la configuración de correo electrónico de WooCommerce del sitio.

Consulte **[Notificaciones por correo electrónico](/settings/wp-admin/email-notifications)** para obtener la referencia completa, incluyendo los valores predeterminados de cada opción de correo, el modelo de destinatario del cajero, escenarios comunes y hooks para desarrolladores.

## Pasarelas de pago {#payment-gateways}

Esta sección enumera todas las pasarelas de pago de WooCommerce instaladas en el sitio. Cada pasarela se puede habilitar o deshabilitar de forma independiente para el POS.

| Columna | Descripción |
|--------|-------------|
| **Habilitado** | Permite activar o desactivar la pasarela para el pago en el POS |
| **Pasarela** | El nombre de la pasarela de pago |
| **ID de pasarela** | El identificador de pasarela de WooCommerce (útil para desarrolladores) |

### Activar una pasarela {#enabling-a-gateway}

1. Buscar la pasarela en la lista
2. Activar el interruptor **Habilitado**
3. La pasarela aparecerá en el cobro del POS en la siguiente carga de página

### Reordenar pasarelas {#reordering-gateways}

Es posible arrastrar las pasarelas para cambiar su orden de visualización en el cobro del POS. La primera pasarela habilitada se selecciona de forma predeterminada cuando un cajero abre el modal de cobro.

### POS vs. tienda en línea {#pos-vs-online-store}

La configuración de pasarelas aquí es **independiente** de la configuración de pagos de WooCommerce. Una pasarela puede estar:

- **Habilitada para POS, deshabilitada en línea** — útil para pagos en efectivo que no aplican a pedidos web
- **Habilitada en línea, deshabilitada para POS** — útil para pasarelas como PayPal que no son prácticas para ventas presenciales
- **Habilitada para ambos** — la pasarela funciona en todos los canales

:::note
Una pasarela debe estar instalada y activa en `WP Admin > WooCommerce > Ajustes > Pagos` antes de aparecer en la lista de ajustes de cobro del POS. Si falta una pasarela, es necesario verificar que el plugin esté instalado y que la pasarela esté habilitada en WooCommerce.
:::

## Estado del pedido {#order-status}

Permite configurar el estado predeterminado de los pedidos completados en el POS. De forma predeterminada, los pedidos del POS se establecen como **Completado**, ya que el pago se cobra en el punto de venta.
