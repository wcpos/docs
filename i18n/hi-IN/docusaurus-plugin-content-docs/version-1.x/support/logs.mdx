लॉग स्क्रीन POS के भीतर सिस्टम गतिविधियों को रिकॉर्ड करती है। यह मुद्दों को डिबग करने और पर्दे के पीछे क्या हो रहा है, इसे समझने के लिए एक आवश्यक उपकरण है।

{/* स्क्रीनशॉट: logs-screen.png */}

## लॉग्स तक पहुँचना

बाईं ओर के नेविगेशन ड्रॉअर (हार्टबीट आइकन) से लॉग स्क्रीन खोलें।

## इंटरफ़ेस अवलोकन

### खोजें और फिल्टर करें

- **लॉग खोजें** - संदेश सामग्री द्वारा विशेष लॉग प्रविष्टियाँ खोजें
- **लॉग स्तर फ़िल्टर** - लॉग स्तर (जानकारी, चेतावनी, त्रुटि) द्वारा फ़िल्टर करें
- **प्रदर्शन सेटिंग्स** - दृश्य कॉलम कॉन्फ़िगर करें

### लॉग तालिका

प्रत्येक लॉग प्रविष्टि दिखाती है:

| कॉलम | विवरण |
|--------|-------------|
| **समय** | जब यह घटना घटी |
| **स्तर** | गंभीरता: जानकारी (ग्रे), चेतावनी (नारंगी), त्रुटि (लाल) |
| **संदेश** | घटना का मानव-पठनीय विवरण |
| **संदर्भ** | अतिरिक्त विवरण के साथ विस्तारणीय JSON |
| **कोड** | त्रुटि कोड (दस्तावेज़ीकरण से लिंक करता है) |

## लॉग स्तर

### जानकारी (ग्रे)

सामान्य संचालन और सफल घटनाएँ:
- "उत्पादों के लिए सभी IDs प्राप्त की गईं"
- "नए ग्राहकों को समन्वयित किया गया"
- "करों के लिए सभी IDs प्राप्त की गईं"

### चेतावनी (नारंगी)

गैर-आवश्यक मुद्दे जिन पर ध्यान देने की आवश्यकता हो सकती है:
- "सत्र को ताज़ा करने में असमर्थ"
- "समन्वय के लिए पुनः प्रयास"

### त्रुटि (लाल)

समस्याएँ जिनका समाधान करना आवश्यक है:
- "क्षमा करें, आप संसाधनों को सूचीबद्ध नहीं कर सकते" (अनुमति संबंधी मुद्दा)
- "API अनुरोध विफल"
- "डाटाबेस समन्वय त्रुटि"

## संदर्भ डेटा

किसी भी लॉग प्रविष्टि पर विस्तारणीय तीर पर क्लिक करें ताकि विस्तृत JSON संदर्भ देखा जा सके:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

यह डेटा निम्नलिखित शामिल करता है:
- **आइटम गणनाएँ** - कितने रिकॉर्ड प्रभावित हुए
- **निष्पादन समय** - संचालन में कितना समय लगा
- **सर्वर लोड** - सर्वर प्रदर्शन मेट्रिक्स
- **अनुरोध/प्रतिक्रिया विवरण** - API मुद्दों का डिबग करने के लिए

## त्रुटि कोड

कई लॉग प्रविष्टियों में एक **त्रुटि कोड** शामिल होता है (जैसे, `API02004`, `DB01001`)। ये कोड सीधे [त्रुटि कोड](/error-codes) दस्तावेज़ से लिंक करते हैं, जो प्रदान करता है:

- त्रुटि का विस्तृत स्पष्टीकरण
- सामान्य कारण
- समस्या निवारण के उपाय

## सामान्य लॉग प्रविष्टियाँ

### समन्वय संचालन

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

ये सामान्य समन्वय गतिविधियों का संकेत देते हैं। संदर्भ दिखाता है कि कितने आइटम समन्वयित किए गए थे।

### अनुमति त्रुटियाँ

```
[error] Sorry, you cannot list resources    {...} API02004
```

इसका अर्थ आम तौर पर यह होता है कि उपयोगकर्ता को किसी संसाधन तक पहुँचने की अनुमति नहीं है। `WP Admin > POS > Settings > Access` में उपयोगकर्ता भूमिकाएँ जांचें।

### सत्र मुद्दे

```
[warn] Unable to refresh session    {...} API02007
```

POS प्रमाणीकरण सत्र को ताज़ा नहीं कर सका। इसके लिए लॉग आउट होकर फिर से लॉग इन करने की आवश्यकता हो सकती है।

## समर्थन के लिए लॉग्स का उपयोग

जब सहायता की आवश्यकता हो:

1. **समस्या को पुन: उत्पन्न करें** - उस क्रिया को करें जो समस्या उत्पन्न करती है
2. **लॉग्स की जाँच करें** - त्रुटियों (लाल) या चेतावनियों (नारंगी) की तलाश करें
3. **संदर्भ को विस्तारित करें** - पूर्ण JSON विवरण प्राप्त करें
4. **त्रुटि कोड नोट करें** - इसे अपने सहायता अनुरोध में शामिल करें
5. **संबंधित प्रविष्टियाँ कॉपी करें** - संदर्भ के साथ लॉग संदेश साझा करें

## प्रदर्शन निगरानी

लॉग्स निष्पादन समय और सर्वर लोड मेट्रिक्स शामिल करते हैं। यदि आप ध्यान दें:

- **उच्च निष्पादन समय** - सर्वर अधिभारित या धीमा हो सकता है
- **उच्च सर्वर लोड मान** - सर्वर अनुकूलन पर विचार करें
- **आवर्ती टाइमआउट्स** - नेटवर्क कनेक्टिविटी जांचें
