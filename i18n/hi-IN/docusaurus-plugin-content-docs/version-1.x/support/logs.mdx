लॉग स्क्रीन POS के भीतर सिस्टम गतिविधियों को record करती है। यह मुद्दों को debug करने और पर्दे के पीछे क्या हो रहा है, इसे समझने के लिए एक आवश्यक उपकरण है।

## लॉग्स को एक्सेस करना

बाएं नेविगेशन ड्रॉवर (heartbeat आइकन) से लॉग स्क्रीन खोलें।

## इंटरफ़ेस अवलोकन

### खोजें और फ़िल्टर करें

- **लॉग खोजें** - संदेश सामग्री द्वारा विशिष्ट लॉग प्रविष्टियाँ खोजें
- **लॉग स्तर फ़िल्टर** - लॉग स्तर (info, warn, error) द्वारा फ़िल्टर करें
- **प्रदर्शन सेटिंग्स** - दृश्य कॉलम कॉन्फ़िगर करें

### लॉग तालिका

प्रत्येक लॉग प्रविष्टि दिखाती है:

| कॉलम | विवरण |
|--------|-------------|
| **समय** | जब घटना हुई |
| **स्तर** | गंभीरता: info (ग्रे), warn (नारंगी), error (लाल) |
| **संदेश** | घटना का मानव-पठनीय विवरण |
| **संदर्भ** | अतिरिक्त विवरण के साथ विस्तारीकरण योग्य JSON |
| **कोड** | त्रुटि कोड (प्रलेखन के लिए लिंक) |

## लॉग स्तर

### जानकारी (ग्रे)

सामान्य संचालन और सफल घटनाएँ:
- "सभी उत्पादों के लिए सभी ID प्राप्त की गईं"
- "नए ग्राहकों को समन्वयित किया गया"
- "सभी करों के लिए सभी ID प्राप्त की गईं"

### चेतावनी (नारंगी)

गैर-आवश्यक मुद्दे जिन्हें ध्यान देने की आवश्यकता हो सकती है:
- "सत्र रिफ्रेश करने में असमर्थ"
- "समन्वय के लिए पुनः प्रयास"

### त्रुटि (लाल)

समस्याएँ जिन्हें संबोधित करने की आवश्यकता है:
- "क्षम करें, आप संसाधनों को सूचीबद्ध नहीं कर सकते" (अनुमति समस्या)
- "API अनुरोध विफल"
- "डेटाबेस समन्वय त्रुटि"

## संदर्भ डेटा

किसी भी लॉग प्रविष्टि पर विस्तारण तीर पर क्लिक करें विस्तृत JSON संदर्भ देखने के लिए:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

यह डेटा शामिल है:
- **आइटम की गणना** - कितने रिकॉर्ड प्रभावित हुए
- **कार्यान्वयन समय** - संचालन में कितना समय लगा
- **सर्वर लोड** - सर्वर प्रदर्शन संकेतक
- **अनुरोध/प्रतिक्रिया विवरण** - API मुद्दों को debug करने के लिए

## त्रुटि कोड

कई लॉग प्रविष्टियों में एक **त्रुटि कोड** शामिल होता है (जैसे, `API02004`, `DB01001`)। ये कोड सीधे [त्रुटि कोड](/error-codes) प्रलेखन से जुड़े होते हैं, जो प्रदान करता है:

- त्रुटि का विस्तृत विवरण
- सामान्य कारण
- समस्या निवारण के चरण

## सामान्य लॉग प्रविष्टियाँ

### समन्वय संचालन

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

ये सामान्य समन्वय गतिविधि को दर्शाते हैं। संदर्भ दिखाता है कि कितने आइटम समन्वयित किए गए थे।

### अनुमति त्रुटियाँ

```
[error] Sorry, you cannot list resources    {...} API02004
```

यह आमतौर पर यह दर्शाता है कि उपयोगकर्ता को किसी संसाधन तक पहुँचने की अनुमति नहीं है। उपयोगकर्ता भूमिकाओं की जाँच करें `WP Admin > POS > Settings > Access` में।

### सत्र समस्याएँ

```
[warn] Unable to refresh session    {...} API02007
```

POS प्राधिकरण सत्र को रिफ्रेश नहीं कर सका। इससे लॉग आउट और पुनः लॉग इन करने की आवश्यकता हो सकती है।

## समर्थन के लिए लॉग का उपयोग करना

जब सहायता का अनुरोध करें:

1. **समस्या का पुनरुत्पादन करें** - उस क्रिया को करें जो समस्या का कारण बनती है
2. **लॉग की जांच करें** - त्रुटियों (लाल) या चेतावनियों (नारंगी) की तलाश करें
3. **संदर्भ का विस्तारण करें** - पूर्ण JSON विवरण प्राप्त करें
4. **त्रुटि कोड नोट करें** - इसे अपने समर्थन अनुरोध में शामिल करें
5. **संबंधित प्रविष्टियाँ कॉपी करें** - संदर्भ के साथ लॉग संदेश साझा करें

## प्रदर्शन की निगरानी

लॉग्स में कार्यान्वयन समय और सर्वर लोड मीट्रिक शामिल होते हैं। यदि आप देखते हैं:

- **उच्च कार्यान्वयन समय** - सर्वर ओवरलोड या धीमा हो सकता है
- **उच्च सर्वर लोड मान** - सर्वर अनुकूलन पर विचार करें
- **बार-बार टाइमआउट** - नेटवर्क कनेक्टिविटी की जाँच करें
