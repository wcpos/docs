लॉग स्क्रीन POS के भीतर सिस्टम गतिविधि को रिकॉर्ड करता है। यह समस्याओं को दूर करने और पीछे की घटनाओं को समझने के लिए एक महत्वपूर्ण उपकरण है।

## लॉग तक पहुंच

बाएं नेविगेशन ड्रॉवर (हृदय गति आइकन) से लॉग स्क्रीन खोलें।

## इंटरफ़ेस अवलोकन

### खोज और फ़िल्टर

- **लॉग खोजें** - संदेश सामग्री द्वारा विशिष्ट लॉग प्रविष्टियाँ खोजें
- **लॉग स्तर फ़िल्टर** - लॉग स्तर (जानकारी, चेतावनी, त्रुटि) द्वारा फ़िल्टर करें
- **प्रदर्शन सेटिंग्स** - दृश्य कॉलम कॉन्फ़िगर करें

### लॉग तालिका

प्रत्येक लॉग प्रविष्टि दिखाती है:

| कॉलम | विवरण |
|--------|-------------|
| **समय** | घटना कब हुई |
| **स्तर** | गंभीरता: जानकारी (ग्रे), चेतावनी (नारंगी), त्रुटि (लाल) |
| **संदेश** | घटना का मानव-पठनीय विवरण |
| **संदर्भ** | विस्तारित JSON के साथ अतिरिक्त विवरण |
| **कोड** | त्रुटि कोड (दस्तावेज़ीकरण में लिंक) |

## लॉग स्तर

### जानकारी (ग्रे)

सामान्य संचालन और सफल घटनाएं:
- "उत्पादों के लिए सभी आईडी लाए गए"
- "नए ग्राहक सिंक किए गए"
- "कर के लिए सभी आईडी लाए गए"

### चेतावनी (नारंगी)

गैर-महत्वपूर्ण मुद्दे जिन्हें ध्यान की आवश्यकता हो सकती है:
- "सत्र को ताज़ा करने में असमर्थ"
- "सिंक के लिए पुनः प्रयास"

### त्रुटि (लाल)

समस्याएं जिन्हें संबोधित किया जाना चाहिए:
- "क्षमा करें, आप संसाधन सूचीबद्ध नहीं कर सकते" (अनुमति मुद्दा)
- "API अनुरोध विफल"
- "डेटाबेस सिंक त्रुटि"

## संदर्भ डेटा

किसी भी लॉग प्रविष्टि पर विस्तार तीर पर क्लिक करके विस्तृत JSON संदर्भ देखें:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

यह डेटा शामिल है:
- **आइटम गणना** - कितने रिकॉर्ड प्रभावित हुए
- **निष्पादन समय** - ऑपरेशन में कितना समय लगा
- **सर्वर लोड** - सर्वर प्रदर्शन मेट्रिक्स
- **अनुरोध/प्रतिक्रिया विवरण** - API समस्याओं को दूर करने के लिए

## त्रुटि कोड

कई लॉग प्रविष्टियों में एक **त्रुटि कोड** शामिल होता है (जैसे `API02004`, `DB01001`)। ये कोड सीधे [त्रुटि कोड](/error-codes) दस्तावेज़ीकरण से जुड़े होते हैं, जो प्रदान करता है:

- त्रुटि का विस्तृत स्पष्टीकरण
- सामान्य कारण
- समस्या निवारण चरण

## सामान्य लॉग प्रविष्टियां

### सिंक ऑपरेशन

```
[info] उत्पादों के लिए सभी आईडी लाए गए    {...} 3 आइटम
[info] नए ग्राहक सिंक किए गए            {...} 1 आइटम
[info] कर के लिए सभी आईडी लाए गए       {...} 3 आइटम
```

ये सामान्य सिंक गतिविधि दर्शाते हैं। संदर्भ दिखाता है कि कितने आइटम सिंक किए गए।

### अनुमति त्रुटियां

```
[error] क्षमा करें, आप संसाधन सूची नहीं बना सकते    {...} API02004
```

यह आमतौर पर दर्शाता है कि उपयोगकर्ता के पास संसाधन तक पहुंच की अनुमति नहीं है। `WP Admin > POS > सेटिंग्स > पहुंच` में उपयोगकर्ता भूमिकाओं की जांच करें।

### सत्र मुद्दे

```
[warn] सत्र को ताज़ा करने में असमर्थ    {...} API02007
```

POS प्रमाणीकरण सत्र को ताज़ा नहीं कर पाया। इसके लिए लॉग आउट और फिर से लॉग इन करने की आवश्यकता हो सकती है।

## समर्थन के लिए लॉग का उपयोग

समर्थन के लिए अनुरोध करते समय:

1. **समस्या को दोहराएं** - उस क्रिया को करें जो समस्या पैदा करती है
2. **लॉग की जांच करें** - त्रुटियों (लाल) या चेतावनियों (नारंगी) को देखें
3. **संदर्भ विस्तारित करें** - पूर्ण JSON विवरण प्राप्त करें
4. **त्रुटि कोड नोट करें** - अपने समर्थन अनुरोध में इसे शामिल करें
5. **संबंधित प्रविष्टियाँ कॉपी करें** - संदर्भ के साथ लॉग संदेश साझा करें

## प्रदर्शन निगरानी

लॉग में निष्पादन समय और सर्वर लोड मेट्रिक्स शामिल होते हैं। यदि आप देखते हैं:

- **उच्च निष्पादन समय** - सर्वर अधिभारित या धीमा हो सकता है
- **उच्च सर्वर लोड मान** - सर्वर अनुकूलन पर विचार करें
- **बार-बार टाइमआउट** - नेटवर्क कनेक्टिविटी की जांच करें
