# API06001: अमान्य URL फ़ॉर्मेट

## यह क्या दर्शाता है

आपकी WooCommerce साइट के लिए कॉन्फ़िगर किया गया URL मान्य फ़ॉर्मेट में नहीं है। URLs को मानक वेब पते की परंपराओं का पालन करना चाहिए।

## सामान्य कारण

- **प्रोटोकॉल गायब** — URL `http://` या `https://` से शुरू नहीं होता
- **URL में टाइपो** — डोमेन में वर्तनी की गलतियाँ
- **अवैध वर्ण** — विशेष वर्ण जो URLs में अनुमति नहीं हैं
- **अपूर्ण URL** — URL में ऐसे भाग गायब हैं जैसे डोमेन एक्सटेंशन

## कैसे सुधारें

### 1. URL फ़ॉर्मेट जाँचें

एक मान्य URL ऐसा दिखना चाहिए:
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com` (प्रोटोकॉल गायब)
- ❌ `https://yourstore` (एक्सटेंशन गायब)
- ❌ `https://your store.com` (स्पेस मौजूद)

### 2. प्रोटोकॉल शामिल करें

हमेशा `https://` या `http://` शामिल करें:
- प्राथमिकता: `https://` (सुरक्षित)
- फ़ॉलबैक: `http://` (यदि HTTPS उपलब्ध नहीं है)

### 3. अतिरिक्त वर्ण हटाएँ

सुनिश्चित करें कि URL में नहीं है:
- ट्रेलिंग स्लैश (वैकल्पिक लेकिन सतत रहें)
- अतिरिक्त स्पेस
- विशेष वर्ण
- URL पैरामीटर (जब तक अनिवार्य न हो)

### 4. URL काम करता है यह सत्यापित करें

एक ब्राउज़र में URL का परीक्षण करें:
1. उस URL को कॉपी करें जिसका आप उपयोग कर रहे हैं
2. इसे एक ब्राउज़र में पेस्ट करें
3. यह आपकी WordPress साइट को लोड करना चाहिए

### 5. POS कॉन्फ़िगरेशन अपडेट करें

सही URL को POS सेटिंग्स में दर्ज करें:
- उस URL से मेल खाएँ जो आपके ब्राउज़र में काम करता है
- आपकी साइट के उपयोग को देखते हुए `www.` को शामिल करें या छोड़ें

## उदाहरण

| अमान्य | मान्य |
|---------|-------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
