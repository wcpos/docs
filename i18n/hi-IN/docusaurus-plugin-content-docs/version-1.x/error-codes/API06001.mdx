# API06001: अमान्य URL स्वरूप

## इसका अर्थ

आपकी WooCommerce साइट के लिए कॉन्फ़िगर किया गया URL मान्य स्वरूप में नहीं है। URLs को मानक वेब पते के नियमों का पालन करना चाहिए।

## सामान्य कारण

- **प्रोटोकॉल अनुपस्थित** — URL `http://` या `https://` से शुरू नहीं होता
- **URL में टाइपो** — डोमेन में वर्तनी की त्रुटि
- **अमान्य वर्ण** — URL में अनुमति नहीं दिए गए विशेष वर्ण
- **अधूरा URL** — URL में डोमेन एक्सटेंशन जैसे कुछ भाग गायब हैं

## समाधान कैसे करें

### 1. URL स्वरूप की जाँच करें

एक मान्य URL इस प्रकार दिखना चाहिए:
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com` (प्रोटोकॉल अनुपस्थित)
- ❌ `https://yourstore` (एक्सटेंशन गायब)
- ❌ `https://your store.com` (स्पेस अंतर्निहित)

### 2. प्रोटोकॉल शामिल करें

हमेशा `https://` या `http://` शामिल करें:
- अधिमान्य: `https://` (सुरक्षित)
- बैकअप: `http://` (यदि HTTPS उपलब्ध नहीं है)

### 3. अतिरिक्त वर्णों को हटाएं

यह सुनिश्चित करें कि URL में नहीं हैं:
- ट्रेलिंग स्लैश (वैकल्पिक लेकिन सुसंगत)
- अतिरिक्त स्पेस
- विशेष वर्ण
- URL पैरामीटर (जब तक आवश्यक न हो)

### 4. URL काम करता है यह सत्यापित करें

ब्राउज़र में URL परीक्षण करें:
1. उपयोग किए जा रहे URL की प्रतिलिपि करें
2. इसे ब्राउज़र में पेस्ट करें
3. इसे आपकी WordPress साइट लोड करना चाहिए

### 5. POS कॉन्फ़िगरेशन अपडेट करें

POS सेटिंग्स में सुधारा गया URL दर्ज करें:
- ब्राउज़र में काम करने वाले URL के साथ बिल्कुल मेल खाएं
- `www.` शामिल या बाहर करें जैसा आपकी साइट उपयोग करती है

## उदाहरण

| अमान्य | मान्य |
|---------|-------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
