# API05004: वर्डप्रेस API निष्क्रिय

## इसका अर्थ

वर्डप्रेस REST API आपकी साइट पर निष्क्रिय है। सभी आधुनिक वर्डप्रेस कार्यक्षमताएँ, जिनमें वूकॉमर्स और WCPOS शामिल हैं, इस API पर निर्भर करती हैं।

## सामान्य कारण

- **सुरक्षा प्लगइन** — एक प्लगइन REST API पहुँच को ब्लॉक कर रहा है
- **होस्टिंग प्रतिबंध** — आपके होस्ट ने REST API को निष्क्रिय कर दिया है
- **कस्टम कोड** — एक थीम या प्लगइन ने API को निष्क्रिय कर दिया है
- **फायरवॉल नियम** — WAF REST API एंडपॉइंट को ब्लॉक कर रहा है

## समाधान कैसे करें

### 1. REST API का परीक्षण

अपने ब्राउज़र में `https://yoursite.com/wp-json/` पर जाएँ:
- उपलब्ध मार्गों के साथ JSON वापस आना चाहिए
- यदि आपको त्रुटि या कुछ नहीं मिलता है, तो यह ब्लॉक है

### 2. सुरक्षा प्लगइन जाँचें

सामान्य प्लगइन जो REST API को ब्लॉक करते हैं:

**वर्डफेंस:**
- फायरवॉल → सभी फायरवॉल विकल्प
- "REST API को निष्क्रिय करें" विकल्प को अक्षम करें

**iथीम सुरक्षा:**
- सुरक्षा → सेटिंग्स → वर्डप्रेस ट्वीक्स
- REST API को सक्षम करें

**REST API प्लगइन निष्क्रिय करें:**
- इस प्लगइन को पूरी तरह से निष्क्रिय करें

### 3. कस्टम कोड की जाँच

अपनी थीम के `functions.php` या कस्टम प्लगइन में इसे देखें:

```php
// यह कोड REST API को निष्क्रिय करता है - इसे हटा दें
add_filter('rest_authentication_errors', function($result) {
    return new WP_Error('rest_disabled', 'REST API disabled');
});
```

### 4. .htaccess की जाँच

`/wp-json/` को ब्लॉक करने वाले किसी भी नियम को हटा दें:

```apache
# बुरा - REST API को ब्लॉक करता है
RewriteRule ^wp-json - [F,L]
```

### 5. होस्टिंग प्रदाता से संपर्क करें

कुछ होस्ट REST API को डिफ़ॉल्ट रूप से ब्लॉक करते हैं:
- इसे सक्षम करने का अनुरोध करें
- किसी भी सुरक्षा प्रतिबंध के बारे में पूछें
- होस्टिंग दस्तावेज़ीकरण की जाँच करें

## REST API महत्वपूर्ण क्यों है

वर्डप्रेस REST API आवश्यक है:
- मोबाइल ऐप्स
- तृतीय-पक्ष एकीकरण
- वूकॉमर्स कार्य
- WCPOS संचालन

इसे निष्क्रिय करने से कई सुविधाएँ टूट जाती हैं।
