# API03002: आवश्यक पैरामीटर गायब {#api03002-missing-required-parameters}

## इसका अर्थ {#what-this-means}

अनुरोध में डेटा गायब है जिसकी आवश्यकता सर्वर को क्रिया को पूरा करने के लिए होती है। आवश्यक फ़ील्ड अनुरोध में शामिल नहीं की गई थीं।

## सामान्य कारण {#common-causes}

- **अधूरा फॉर्म** — आवश्यक फ़ील्ड भरी नहीं गई
- **डेटा सहेजा नहीं गया** — फॉर्म डेटा ठीक से कैप्चर नहीं किया गया
- **संस्करण असंगति** — प्लगइन अलग पैरामीटर की अपेक्षा करता है जो POS भेजता है
- **कस्टम फ़ील्ड** — आवश्यक कस्टम फ़ील्ड गायब हैं

## सुधार कैसे करें {#how-to-fix}

### 1. आवश्यक फ़ील्ड भरें {#1-complete-required-fields}

सुनिश्चित करें कि सभी आवश्यक जानकारी भरी गई है:
- ग्राहक विवरण (यदि आवश्यक हो)
- उत्पाद जानकारी
- आदेश विवरण

### 2. ताज़ा करें और पुनः प्रयास करें {#2-refresh-and-retry}

फॉर्म स्थिति अधूरी हो सकती है:
1. POS को ताज़ा करें
2. आवश्यक जानकारी फिर से दर्ज करें
3. एक बार फिर क्रिया का प्रयास करें

### 3. प्लगइन संस्करणों की जांच करें {#3-check-plugin-versions}

संगतता सुनिश्चित करें:
- WCPOS प्लगइन को अपडेट करें
- POS एप्लिकेशन को अपडेट करें
- दोनों संगत संस्करणों पर होने चाहिए

### 4. आवश्यक फ़ील्ड सेटिंग्स की जांच करें {#4-check-required-field-settings}

WooCommerce में, कुछ फ़ील्ड को आवश्यक के रूप में चिह्नित किया जा सकता है:
- चेकआउट फ़ील्ड आवश्यकताओं की समीक्षा करें
- कस्टम फ़ील्ड कॉन्फ़िगरेशन की जांच करें
- आवश्यकताओं को समायोजित करें यदि आवश्यक हो

### 5. कस्टम एकीकरणों की जांच करें {#5-check-custom-integrations}

यदि कस्टम प्लगइन्स या एकीकरण का उपयोग कर रहे हैं:
- उन्हें अतिरिक्त फ़ील्ड की आवश्यकता हो सकती है
- प्लगइन दस्तावेज़ की समीक्षा करें
- संघर्षों की जांच करें
