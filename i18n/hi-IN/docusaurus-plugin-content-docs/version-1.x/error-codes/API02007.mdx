# API02007: टोकन रिफ्रेश विफल

## इसका अर्थ

POS ने स्वचालित रूप से आपके प्रमाणीकरण सत्र को नवीनीकृत करने का प्रयास किया, लेकिन रिफ्रेश अनुरोध विफल हो गया। इसका अर्थ आमतौर पर यह है कि आपको फिर से लॉग इन करना होगा।

## सामान्य कारण

- **रिफ्रेश टोकन समाप्त** — रिफ्रेश टोकन स्वयं समाप्त हो गया है
- **रिफ्रेश टोकन रद्द** — किसी व्यवस्थापक ने सभी सत्रों को अमान्य कर दिया है
- **सर्वर त्रुटि** — सर्वर रिफ्रेश अनुरोध को संसाधित नहीं कर सका
- **नेटवर्क समस्या** — रिफ्रेश प्रयास के दौरान कनेक्शन समस्या

## समाधान कैसे करें

### 1. फिर से लॉग इन करें

सबसे विश्वसनीय समाधान:
1. POS से लॉग आउट करें
2. अपने क्रेडेंशियल्स के साथ फिर से लॉग इन करें
3. यह नए टोकन के साथ एक ताजा सत्र बनाता है

### 2. नेटवर्क कनेक्शन की जाँच करें

यदि नेटवर्क समस्याओं के कारण रिफ्रेश विफल हो रहा है:
- अपने इंटरनेट कनेक्शन की पुष्टि करें
- ब्राउज़र में अपनी WordPress साइट तक पहुंचने का प्रयास करें
- थोड़ी देर प्रतीक्षा करें और फिर से प्रयास करें

### 3. सर्वर स्थिति की जाँच करें

यदि सर्वर में समस्याएं हैं:
- सुनिश्चित करें कि WordPress सही ढंग से चल रहा है
- अपने सर्वर लॉग में PHP त्रुटियों की जाँच करें
- सुनिश्चित करें कि प्रमाणीकरण प्लगइन सक्रिय है

### 4. सत्र डेटा साफ़ करें

यदि समस्याएं बनी रहती हैं:
- POS ऐप कैश साफ़ करें
- संग्रहीत क्रेडेंशियल्स हटाएं
- नए लॉगइन के साथ शुरुआत करें

## स्वचालित टोकन रिफ्रेश

WCPOS स्वचालित रूप से टोकन को उनके समाप्त होने से पहले रिफ्रेश करता है ताकि आपका सत्र बनाए रखा जा सके। यदि यह प्रक्रिया बार-बार विफल होती है, तो हो सकता है कि कॉन्फ़िगरेशन में कोई समस्या हो।
