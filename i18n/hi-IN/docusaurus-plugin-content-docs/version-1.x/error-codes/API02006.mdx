# API02006: API कुंजी अमान्य

## इसका अर्थ

उपयोग की जा रही WooCommerce REST API कुंजी अमान्य है। API कुंजियां सर्वर-से-सर्वर प्रमाणीकरण के लिए उपयोग की जाती हैं, और प्रदान की गई कुंजी WooCommerce में किसी भी वैध कुंजी से मेल नहीं खाती।

## सामान्य कारण

- **कुंजी हटाई गई** — API कुंजी WooCommerce से हटा दी गई थी
- **कुंजी में टाइपो** — कुंजी गलत दर्ज की गई थी
- **गलत कुंजी जोड़ी** — उपभोक्ता कुंजी और गुप्त कुंजी मेल नहीं खाते
- **अन्य साइट की कुंजी** — किसी अन्य इंस्टॉलेशन के लिए बनाई गई कुंजियां उपयोग की जा रही हैं

## समाधान कैसे करें

### 1. API कुंजी की पुष्टि करें

WordPress व्यवस्थापक → WooCommerce → सेटिंग्स → उन्नत → REST API में:
1. जांचें कि आपकी API कुंजी मौजूद है
2. पुष्टि करें कि इसे रद्द नहीं किया गया है
3. अनुमतियों (पढ़ें/लिखें/पढ़ें-लिखें) पर ध्यान दें

### 2. नई API कुंजियां उत्पन्न करें

यदि कुंजी अनुपस्थित या अमान्य है:
1. WooCommerce → सेटिंग्स → उन्नत → REST API पर जाएं
2. "कुंजी जोड़ें" पर क्लिक करें
3. एक विवरण दर्ज करें (जैसे "WCPOS")
4. उपयोगकर्ता चुनें
5. "पढ़ें/लिखें" अनुमतियां चुनें
6. "API कुंजी उत्पन्न करें" पर क्लिक करें
7. **उपभोक्ता कुंजी और उपभोक्ता गुप्त कुंजी दोनों की प्रतिलिपि बनाएं** (केवल एक बार दिखाई जाएंगी!)

### 3. POS कॉन्फ़िगरेशन अपडेट करें

POS में नई API कुंजियां दर्ज करें:
- उपभोक्ता कुंजी (ck_ से शुरू होती है)
- उपभोक्ता गुप्त कुंजी (cs_ से शुरू होती है)

### 4. कुंजी अनुमतियों की जांच करें

यह सुनिश्चित करें कि कुंजी के पास पर्याप्त अनुमतियां हैं:
- **पढ़ें** — केवल डेटा देखें
- **लिखें** — डेटा संशोधित करें
- **पढ़ें/लिखें** — पूर्ण पहुंच (POS के लिए अनुशंसित)
