# API02006: API कुंजी अवैध {#api02006-api-key-invalid}

## इसका क्या अर्थ है {#what-this-means}

जो WooCommerce REST API कुंजी का उपयोग किया जा रहा है वह अवैध है। API कुंजियों का उपयोग सर्वर-से-सर्वर प्रमाणीकरण के लिए किया जाता है, और प्रदान की गई कुंजी WooCommerce में किसी भी वैध कुंजी से मेल नहीं खाती।

## सामान्य कारण {#common-causes}

- **कुंजी हटा दी गई** — API कुंजी WooCommerce से हटा दी गई
- **कुंजी टाइपो** — कुंजी को गलत तरीके से दर्ज किया गया
- **गलत कुंजी जोड़ी** — उपभोक्ता कुंजी और गोपनीयता मेल नहीं खाते
- **अलग साइट की कुंजी** — किसी अन्य इंस्टॉलेशन के लिए उत्पन्न कुंजियों का उपयोग

## कैसे ठीक करें {#how-to-fix}

### 1. API कुंजी की पुष्टि करें {#1-verify-the-api-key}

WordPress Admin → WooCommerce → सेटिंग्स → उन्नत → REST API में:
1. जांचें कि आपकी API कुंजी मौजूद है या नहीं
2. पुष्टि करें कि इसे रद्द नहीं किया गया है
3. अनुमतियों (पढ़ें/लिखें/पढ़ें-लिखें) को नोट करें

### 2. नई API कुंजियाँ उत्पन्न करें {#2-generate-new-api-keys}

यदि कुंजी अनुपलब्ध या अवैध है:
1. WooCommerce → सेटिंग्स → उन्नत → REST API पर जाएं
2. "कुंजी जोड़ें" पर क्लिक करें
3. एक विवरण दर्ज करें (जैसे, "WCPOS")
4. उपयोगकर्ता का चयन करें
5. "पढ़ें/लिखें" अनुमतियाँ चुनें
6. "API कुंजी उत्पन्न करें" पर क्लिक करें
7. **उपभोक्ता कुंजी और उपभोक्ता गोपनीयता दोनों की कॉपी करें** (केवल एक बार दिखाई देती है!)

### 3. POS कॉन्फ़िगरेशन अपडेट करें {#3-update-pos-configuration}

POS में नई API कुंजियों को दर्ज करें:
- उपभोक्ता कुंजी (जो `ck_` से शुरू होती है)
- उपभोक्ता गोपनीयता (जो `cs_` से शुरू होती है)

### 4. कुंजी अनुमतियाँ जांचें {#4-check-key-permissions}

सुनिश्चित करें कि कुंजी के पास पर्याप्त अनुमतियाँ हैं:
- **पढ़ें** — केवल डेटा देखें
- **लिखें** — केवल डेटा संशोधित करें
- **पढ़ें/लिखें** — पूर्ण पहुँच (POS के लिए अनुशंसित)
