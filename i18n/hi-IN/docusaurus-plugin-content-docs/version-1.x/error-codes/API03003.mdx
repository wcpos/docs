# API03003: अवैध मापदंड मान

## इसका अर्थ क्या है

आपके अनुरोध में एक मान अवैध है। डेटा प्रारूप, प्रकार या मान सर्वर की अपेक्षा के अनुरूप नहीं है।

## सामान्य कारण

- **गलत डेटा प्रकार** — जहां संख्या की अपेक्षा है वहां पाठ (या इसके विपरीत)
- **सीमा से बाहर** — मान अनुमत सीमाओं से अधिक
- **अवैध प्रारूप** — ईमेल, फोन या अन्य स्वरूपित फील्ड गलत हैं
- **अवैध संदर्भ** — मौजूद नहीं होने वाली आईडी का संदर्भ

## सर्वर त्रुटि मैपिंग

यह त्रुटि कोड निम्न स्थितियों में ट्रिगर होता है:

| सर्वर कोड | स्रोत |
|-------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## समाधान कैसे करें

### 1. इनपुट मानों की जाँच करें

जमा किए गए डेटा की समीक्षा करें:
- **कीमतें** — वैध संख्याएँ होनी चाहिए
- **मात्राएँ** — सकारात्मक पूर्णांक होनी चाहिए
- **ईमेल** — वैध ईमेल प्रारूप में होना चाहिए
- **आईडी** — मौजूदा रिकॉर्ड का संदर्भ होना चाहिए

### 2. विशिष्ट फील्ड त्रुटियों को देखें

त्रुटि संकेत कर सकती है कि कौन सी फील्ड अवैध है:
- त्रुटि संदेश विवरण की जाँच करें
- विशिष्ट फील्ड को सुधारें
- कार्य को फिर से करें

### 3. संदर्भित डेटा मौजूद है सुनिश्चित करें

उत्पादों, ग्राहकों या अन्य रिकॉर्ड का संदर्भ देते समय:
- सुनिश्चित करें कि आइटम WooCommerce में मौजूद है
- जाँचें कि हाल ही में हटाया नहीं गया है
- स्थानीय रिकॉर्ड को ताजा करने के लिए डेटा सिंक करें

### 4. फील्ड बाधाओं की जाँच करें

WooCommerce में बाधाएँ हो सकती हैं:
- अधिकतम/न्यूनतम मान
- आवश्यक प्रारूप
- चयन फील्ड के लिए अनुमत विकल्प

### 5. साफ करें और फिर से दर्ज करें

कभी-कभी डेटा भ्रष्ट हो जाता है:
- समस्याग्रस्त फील्ड को साफ करें
- मान को शुरू से फिर से दर्ज करें
- अन्य स्रोतों से कॉपी-पेस्ट करने से बचें
