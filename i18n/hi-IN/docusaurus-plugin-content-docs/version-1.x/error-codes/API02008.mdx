# API02008: रिफ्रेश टोकन अमान्य

## इसका अर्थ

POS द्वारा संग्रहित रिफ्रेश टोकन को सर्वर द्वारा पहचाना नहीं जाता है। रिफ्रेश टोकन का उपयोग बिना फिर से लॉग इन किए नए एक्सेस टोकन प्राप्त करने के लिए किया जाता है।

## सामान्य कारण

- **टोकन रद्द किया गया** — एक व्यवस्थापक ने आपका सत्र अमान्य कर दिया
- **सर्वर रहस्य बदले गए** — WordPress सुरक्षा कुंजियाँ अपडेट की गईं
- **डेटा भ्रष्टता** — भंडारण में टोकन भ्रष्ट हो गया
- **बहु लॉगइन** — कहीं और लॉग इन करने से इस टोकन को अमान्य किया जा सकता है

## समाधान कैसे करें

### 1. फिर से लॉग इन करें

चूँकि रिफ्रेश टोकन अमान्य है, आपको फिर से प्रमाणीकृत करना होगा:
1. POS से लॉग आउट करें
2. अपना उपयोगकर्ता नाम और पासवर्ड दर्ज करें
3. यह नए मान्य टोकन उत्पन्न करेगा

### 2. रद्द किए गए सत्रों की जाँच करें

यदि किसी व्यवस्थापक ने सत्र रद्द किए हैं:
- यह सामान्य सुरक्षा अभ्यास है
- बस फिर से लॉग इन करें
- आपका नया सत्र सामान्य रूप से काम करेगा

### 3. सर्वर परिवर्तनों की जाँच करें

यदि यह सभी उपयोगकर्ताओं को प्रभावित करता है:
- WordPress नमक/कुंजियाँ बदल सकती हैं
- JWT प्लगइन सेटिंग्स में संशोधन किया जा सकता है
- अपने सर्वर व्यवस्थापक से संपर्क करें

### 4. संग्रहित डेटा साफ़ करें

यदि आपको लगातार समस्याएँ आ रही हैं:
- POS एप्लिकेशन डेटा साफ़ करें
- कैश किए गए क्रेडेंशियल्स को हटाएँ
- एक साफ़ लॉगइन के साथ शुरुआत करें
