# API02008: रिफ्रेश टोकन अमान्य

## इसका मतलब

POS द्वारा संग्रहीत रिफ्रेश टोकन सर्वर द्वारा मान्यता प्राप्त नहीं है। रिफ्रेश टोकन का उपयोग नए एक्सेस टोकन प्राप्त करने के लिए किया जाता है, बिना आपको फिर से लॉग इन करने की आवश्यकता के।

## सामान्य कारण

- **टोकन रद्द कर दिया गया** — एक व्यवस्थापक ने आपकी सत्र को अमान्य कर दिया
- **सर्वर रहस्य बदले गए** — वर्डप्रेस सुरक्षा कुंजियाँ अपडेट की गईं
- **डेटा भ्रष्टाचार** — संग्रहण में टोकन भ्रष्ट हो गया
- **कई लॉगिन** — अन्यत्र लॉगिन करने से इस टोकन को अमान्य किया जा सकता है

## इसे ठीक करने का तरीका

### 1. फिर से लॉग इन करें

चूंकि रिफ्रेश टोकन अमान्य है, आपको फिर से प्रमाणित करना आवश्यक है:
1. POS से लॉग आउट करें
2. अपना उपयोगकर्ता नाम और पासवर्ड डालें
3. यह नए मान्य टोकन उत्पन्न करता है

### 2. रद्द किए गए सत्रों के लिए जाँच करें

यदि किसी व्यवस्थापक ने सत्र रद्द कर दिए:
- यह सामान्य सुरक्षा प्रक्रिया है
- बस फिर से लॉगिन करें
- आपका नया सत्र सामान्य रूप से काम करेगा

### 3. सर्वर परिवर्तनों की जाँच करें

यदि यह सभी उपयोगकर्ताओं को प्रभावित करता है:
- वर्डप्रेस सॉल्ट/कुंजियाँ बदल गई होंगी
- JWT प्लगइन सेटिंग्स में संशोधन किया गया हो सकता है
- अपने सर्वर व्यवस्थापक से संपर्क करें

### 4. संग्रहीत डेटा को साफ करें

यदि आप समस्याएँ जारी रखते हैं:
- POS एप्लिकेशन का डेटा साफ करें
- कैश की गई क्रेडेंशियल्स को हटा दें
- एक साफ लॉगिन से शुरू करें
