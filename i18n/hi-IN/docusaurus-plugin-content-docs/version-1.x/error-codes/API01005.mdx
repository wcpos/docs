# API01005: SSL प्रमाणपत्र त्रुटि

## इसका अर्थ क्या है

आपकी वेबसाइट के SSL/TLS प्रमाणपत्र में एक समस्या है। यह प्रमाणपत्र कनेक्शन को सुरक्षित (HTTPS) बनाता है। POS अपके डेटा की सुरक्षा के लिए अमान्य प्रमाणपत्रों वाली साइटों से कनेक्ट नहीं होगा।

## सामान्य कारण

- **समाप्त प्रमाणपत्र** — SSL प्रमाणपत्रों को समय-समय पर नवीनीकृत किया जाना चाहिए
- **स्व-हस्ताक्षरित प्रमाणपत्र** — किसी विश्वसनीय अधिकार द्वारा जारी नहीं किया गया
- **गलत डोमेन** — प्रमाणपत्र डोमेन नाम से मेल नहीं खाता
- **अपूर्ण प्रमाणपत्र श्रृंखला** — अंतर्वर्ती प्रमाणपत्र गायब
- **मिश्रित सामग्री** — कुछ संसाधन HTTPS के बजाय HTTP पर लोड हो रहे हैं

## कैसे ठीक करें

### 1. प्रमाणपत्र स्थिति की जाँच करें

एक ब्राउज़र में अपनी साइट पर जाएं और प्रमाणपत्र विवरण देखने के लिए ताले के चिह्न पर क्लिक करें। निम्न को देखें:
- समाप्ति तिथि
- किसके लिए जारी (आपके डोमेन से मेल खाना चाहिए)
- किसके द्वारा जारी (एक मान्यता प्राप्त अधिकार होना चाहिए)

### 2. समाप्त प्रमाणपत्र नवीनीकृत करें

यदि समाप्त हो गया है:
- अधिकांश होस्टिंग प्रदाता मुफ्त Let's Encrypt प्रमाणपत्र प्रदान करते हैं
- नवीनीकरण के लिए अपने होस्टिंग प्रदाता से संपर्क करें
- जाँचें कि स्वचालित नवीनीकरण सक्षम है या नहीं

### 3. प्रमाणपत्र बेमेल को ठीक करें

सुनिश्चित करें कि प्रमाणपत्र शामिल करता है:
- आपका सटीक डोमेन (`yourstore.com`)
- WWW वेरिएंट (`www.yourstore.com`) यदि उपयोग किया जाता है
- वाइल्डकार्ड प्रमाणपत्र (`*.yourstore.com`) पर विचार करें

### 4. गायब अंतर्वर्ती प्रमाणपत्र स्थापित करें

गायब प्रमाणपत्रों की पहचान करने के लिए एक SSL जाँचकर्ता टूल (जैसे SSL Labs) का उपयोग करें। आपका होस्टिंग प्रदाता उन्हें स्थापित करने में मदद कर सकता है।

### 5. HTTPS को लागू करें

WordPress में, सुनिश्चित करें:
- साइट URL `https://` का उपयोग करता है
- WooCommerce सेटिंग्स में SSL को बलपूर्वक लागू किया गया है
