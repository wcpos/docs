# API04002: अप्रत्याशित प्रतिक्रिया कोड

## इसका अर्थ

सर्वर ने एक अप्रत्याशित HTTP स्थिति कोड वापस किया। सामान्य कोड में शामिल हैं 500 (सर्वर त्रुटि), 403 (निषिद्ध), 404 (नहीं मिला), आदि।

## सामान्य कारण

- **500 आंतरिक सर्वर त्रुटि** — PHP क्रैश हुआ या त्रुटि मिली
- **403 निषिद्ध** — सुरक्षा सेटिंग्स द्वारा पहुंच अस्वीकृत
- **404 नहीं मिला** — एंडपॉइंट मौजूद नहीं है
- **502/503/504** — सर्वर गेटवे या उपलब्धता समस्याएं

## समाधान कैसे करें

### 500 त्रुटियों के लिए (सर्वर त्रुटि)

1. PHP त्रुटि लॉग की जाँच करें
2. PHP मेमोरी सीमा बढ़ाएं
3. प्लगइन संघर्षों की जाँच करें
4. `wp-content/debug.log` देखें

### 403 त्रुटियों के लिए (निषिद्ध)

1. सुरक्षा प्लगइन सेटिंग्स की जाँच करें (वर्डफेंस, सुकुरी, आदि)
2. POS या अपने IP को व्हाइटलिस्ट करें
3. `.htaccess` में ब्लॉकिंग नियमों की जाँच करें
4. सुनिश्चित करें कि ModSecurity अनुरोधों को ब्लॉक नहीं कर रहा है

### 404 त्रुटियों के लिए (नहीं मिला)

1. सुनिश्चित करें कि WCPOS प्लगइन सक्रिय है
2. WordPress परमालिंक्स को रीसेट करें (सेटिंग्स → परमालिंक्स → सहेजें)
3. जाँचें कि REST API सक्षम है
4. URL कॉन्फ़िगरेशन की पुष्टि करें

### 502/503/504 त्रुटियों के लिए (गेटवे समस्याएं)

1. होस्टिंग प्रदाता से संपर्क करें
2. सर्वर के ठीक होने की प्रतीक्षा करें
3. जाँचें कि साइट पर भारी लोड है
4. सुनिश्चित करें कि सर्वर चल रहा है

### सामान्य समस्या निवारण

1. सीधे अपनी साइट तक पहुंचने का प्रयास करें
2. समस्याओं के लिए होस्टिंग नियंत्रण पैनल देखें
3. सर्वर एक्सेस लॉग की समीक्षा करें
4. आवश्यकता पड़ने पर होस्टिंग समर्थन से संपर्क करें
