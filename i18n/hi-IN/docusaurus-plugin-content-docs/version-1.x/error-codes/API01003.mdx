# API01003: कनेक्शन रीसेट {#api01003-connection-reset}

## इसका अर्थ {#what-this-means}

सर्वर के साथ कनेक्शन स्थापित किया गया था लेकिन फिर अप्रत्याशित रूप से बंद हो गया। यह आमतौर पर संचार के मध्य होता है जब कुछ कनेक्शन को बाधित करता है।

## सामान्य कारण {#common-causes}

- **सर्वर क्रैश** — सर्वर प्रक्रिया अनुरोध के दौरान क्रैश हो सकती है
- **मेमोरी सीमा** — PHP की मेमोरी खत्म हो गई और इसे समाप्त कर दिया गया
- **अनुरोध के मध्य टाइमआउट** — सर्वर प्रसंस्करण के दौरान टाइमआउट हो गया
- **नेटवर्क अस्थिरता** — अंतराल नेटवर्क समस्याएँ
- **सुरक्षा सॉफ़्टवेयर** — आक्रामक सुरक्षा सेटिंग्स कनेक्शन बंद कर रही हैं

## कैसे ठीक करें {#how-to-fix}

### 1. सर्वर त्रुटि लॉग्स जाँचें {#1-check-server-error-logs}

रीसेट के समय के चारों ओर संबंधित त्रुटियों के लिए अपने वर्डप्रेस त्रुटि लॉग्स या होस्टिंग नियंत्रण पैनल में देखें।

### 2. PHP मेमोरी लिमिट बढ़ाएँ {#2-increase-php-memory-limit}

अपने `wp-config.php` में:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. प्लगइन संघर्षों की जांच करें {#3-check-for-plugin-conflicts}

कोई प्लगइन PHP को क्रैश करने का कारण हो सकता है:
- अस्थायी रूप से प्लगइन्स को बंद करें
- उसे खोजने के लिए एक-एक करके पुनः सक्षम करें

### 4. सर्वर संसाधनों की समीक्षा करें {#4-review-server-resources}

अपने होस्टिंग प्रदाता से जाँच करें:
- PHP प्रक्रिया सीमाएँ
- त्रुटि के समय मेमोरी उपयोग
- सर्वर स्थिरता लॉग्स

### 5. नेटवर्क स्थिरता का परीक्षण करें {#5-test-network-stability}

यदि WiFi पर हैं, तो एक वायर्ड कनेक्शन आज़माएँ। यदि समस्या विभिन्न नेटवर्क्स पर बनी रहती है, तो यह संभवतः सर्वर-साइड है।
