# API02003: टोकन अमान्य

## इसका अर्थ

सर्वर द्वारा उपयोग किया जाने वाला प्रमाणीकरण टोकन पहचाना नहीं जाता है। यह समाप्त टोकन से अलग है - यह टोकन कभी भी मान्य नहीं था या रद्द कर दिया गया है।

## सामान्य कारण

- **टोकन रद्द** — एक व्यवस्थापक ने सत्र समाप्त कर दिया
- **टोकन भ्रष्ट** — संग्रहण या संचरण के दौरान डेटा भ्रष्टाचार
- **सर्वर रीसेट** — सर्वर गोपनीय कुंजियां बदल गईं, सभी टोकन अमान्य हो गए
- **गलत सर्वर** — किसी अन्य WordPress स्थापना का टोकन

## सर्वर त्रुटि मैपिंग

यह त्रुटि कोड तब ट्रिगर होता है जब सर्वर लौटाता है:

| सर्वर कोड | स्रोत |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT प्रमाणीकरण प्लगइन |

## समाधान कैसे करें

### 1. फिर से लॉग इन करें

अपना सत्र साफ़ करें और ताजा रूप से प्रमाणित करें:
1. पूरी तरह से लॉग आउट करें
2. POS को बंद और फिर से खोलें
3. अपने क्रेडेंशियल्स के साथ लॉग इन करें

### 2. एप्लिकेशन डेटा साफ़ करें

यदि लॉग आउट काम नहीं करता:
- POS ऐप कैश/डेटा साफ़ करें
- वेब पर: साइट के लिए ब्राउज़र कुकीज़ और स्थानीय संग्रह साफ़ करें
- डेस्कटॉप पर: "डेटा साफ़ करें" विकल्प के लिए ऐप सेटिंग्स की जाँच करें

### 3. सर्वर कॉन्फ़िगरेशन की जाँच करें

यदि समस्या सभी उपयोगकर्ताओं को प्रभावित करती है:
- जाँच करें कि WordPress सॉल्ट बदले गए हैं
- JWT या प्रमाणीकरण प्लगइन सेटिंग्स सत्यापित करें
- हाल ही में किए गए सर्वर परिवर्तनों की समीक्षा करें

### 4. सही सर्वर पर हैं यह सुनिश्चित करें

यह सुनिश्चित करें कि POS सही WordPress साइट से जुड़ने के लिए कॉन्फ़िगर किया गया है, विशेष रूप से यदि आपके पास कई स्थापनाएँ हैं।
