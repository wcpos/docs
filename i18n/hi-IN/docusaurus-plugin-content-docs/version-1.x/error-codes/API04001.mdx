# API04001: अमान्य प्रतिक्रिया प्रारूप

## इसका अर्थ क्या है

सर्वर ने प्रतिक्रिया दी, लेकिन प्रतिक्रिया प्रारूप POS द्वारा अपेक्षित नहीं था। सर्वर को JSON डेटा वापस करना चाहिए था, लेकिन कुछ और प्राप्त हुआ।

## सामान्य कारण

- **PHP त्रुटि प्रदर्शित** — JSON से पहले एक PHP त्रुटि आउटपुट की जा रही है
- **प्लगइन संघर्ष** — दूसरा प्लगइन सामग्री आउटपुट कर रहा है
- **रखरखाव मोड** — साइट एक रखरखाव पृष्ठ दिखा रही है
- **गलत कंटेंट प्रकार** — सर्वर JSON के बजाय HTML भेज रहा है
- **कैशिंग समस्या** — एक कैश्ड त्रुटि पृष्ठ परोसा जा रहा है

## समाधान कैसे करें

### 1. अपनी साइट की जांच करें

ब्राउज़र में अपनी WordPress साइट पर जाएं:
- क्या यह सामान्य रूप से प्रदर्शित हो रही है?
- क्या कोई दृश्य त्रुटियां हैं?
- क्या यह रखरखाव मोड में है?

### 2. PHP त्रुटियों की जांच करें

`wp-config.php` में, अस्थायी रूप से डीबग सक्षम करें:

```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```

`wp-content/debug.log` में त्रुटियों की जांच करें।

### 3. REST API को सीधे टेस्ट करें

ब्राउज़र में `https://yoursite.com/wp-json/` पर जाएं:
- JSON डेटा वापस आना चाहिए
- यदि आप HTML या त्रुटियां देखते हैं, तो कोई समस्या है
- प्लगइन से संबंधित आउटपुट की जांच करें

### 4. कैशिंग को अस्थायी रूप से अक्षम करें

कैशिंग प्लगइन पुरानी प्रतिक्रियाएं परोस सकते हैं:
- सभी कैश को साफ करें
- कैशिंग प्लगइन को अस्थायी रूप से अक्षम करें
- REST API को कैशिंग से बाहर रखें

### 5. प्लगइन संघर्षों की जांच करें

यदि कोई प्लगइन हर पृष्ठ पर सामग्री आउटपुट करता है:
1. सभी गैर-आवश्यक प्लगइन को अक्षम करें
2. POS का परीक्षण करें
3. प्लगइन को एक-एक करके फिर से सक्षम करें
