# API02010: प्रमाणीकरण आवश्यक

## इसका अर्थ क्या है

आप जिस क्रिया को करने की कोशिश कर रहे हैं, उसके लिए प्रमाणीकरण की आवश्यकता है, लेकिन आवर्तमान में लॉग इन नहीं हैं। POS को इस संसाधन तक पहुंचने के लिए वैध पहचान पत्र की आवश्यकता है।

## सामान्य कारण

- **लॉग इन नहीं किया गया** — आपने अभी तक प्रमाणित नहीं किया है
- **सत्र साफ किया गया** — आपका सत्र साफ किया गया या समाप्त हो गया
- **संरक्षित संसाधन तक पहुंच** — संसाधन को प्रमाणीकरण की आवश्यकता है
- **ऐप डेटा साफ किया गया** — संग्रहीत पहचान पत्र हटा दिए गए

## सर्वर त्रुटि मानचित्रण

यह त्रुटि कोड तब ट्रिगर होता है जब सर्वर वापस करता है:

| सर्वर कोड | स्रोत |
|-------------|--------|
| `rest_login_required` | WordPress REST API |
| `jwt_auth_no_auth_header` | JWT प्रमाणीकरण प्लगइन |
| HTTP 401 | कोई भी सर्वर प्रतिक्रिया (फॉलबैक) |

## समाधान कैसे करें

### 1. लॉग इन करें

यदि आपने लॉग इन नहीं किया है:
1. POS लॉग इन स्क्रीन खोलें
2. अपने WordPress पहचान पत्र दर्ज करें
3. प्रमाणीकरण प्रक्रिया को पूरा करें

### 2. सत्र स्थिति की जाँच करें

यदि आप सोचते हैं कि आप लॉग इन हैं:
- आपका सत्र समाप्त हो सकता है
- अधिक विवरण के लिए [API02002](./API02002) (टोकन समाप्त) देखें
- पहुंच पुनर्स्थापित करने के लिए फिर से लॉग इन करें

### 3. सर्वर कॉन्फ़िगरेशन सत्यापित करें

यह सुनिश्चित करें कि API एंडपॉइंट्स ठीक से कॉन्फ़िगर किए गए हैं:
- WooCommerce REST API सक्षम होना चाहिए
- WCPOS प्लगइन सक्रिय होना चाहिए
- प्रमाणीकरण एंडपॉइंट्स सुलभ होने चाहिए

### 4. ब्राउज़र/ऐप समस्याओं की जाँच करें

यदि आप अप्रत्याशित रूप से लॉग आउट हो रहे हैं:
- ब्राउज़र कैश साफ करें (वेब संस्करण का उपयोग करते समय)
- जाँचें कि कुकीज़/स्थानीय भंडारण अवरुद्ध नहीं हैं
- सत्यापित करें कि ऐप को डेटा संग्रहीत करने की अनुमति है

## क्या प्रमाणीकरण की आवश्यकता है

अधिकांश POS संचालन के लिए प्रमाणीकरण की आवश्यकता होती है:
- उत्पादों और ग्राहकों को देखना
- ऑर्डर बनाना और संपादित करना
- भुगतान प्रोसेस करना
- रिपोर्ट तक पहुंच

केवल प्रारंभिक लॉग इन स्क्रीन तक प्रमाणीकरण के बिना पहुंचा जा सकता है।
