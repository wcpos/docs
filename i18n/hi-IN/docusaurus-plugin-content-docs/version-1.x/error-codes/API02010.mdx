# API02010: प्रमाणीकरण आवश्यक

## इसका अर्थ क्या है

आप जिस क्रिया को करने का प्रयास कर रहे हैं, उसके लिए प्रमाणीकरण की आवश्यकता है, लेकिन आप वर्तमान में लॉगिन नहीं हैं। इस संसाधन तक पहुंचने के लिए POS को वैध क्रेडेंशियल्स की आवश्यकता है।

## सामान्य कारण

- **लॉगिन नहीं किया** — आपने अभी तक प्रमाणीकरण नहीं किया है
- **सत्र साफ किया गया** — आपका सत्र साफ किया गया या समाप्त हो गया
- **संरक्षित संसाधन तक पहुँच** — संसाधन के लिए प्रमाणीकरण की आवश्यकता है
- **ऐप डेटा साफ किया गया** — संग्रहीत क्रेडेंशियल्स हटा दिए गए

## सर्वर त्रुटि मानचित्रण

यह त्रुटि कोड तब सक्रिय होता है जब सर्वर लौटाता है:

| सर्वर कोड | स्रोत |
|-----------|-------|
| `rest_login_required` | वर्डप्रेस REST API |
| `jwt_auth_no_auth_header` | JWT Authentication plugin |
| HTTP 401 | कोई भी सर्वर प्रतिक्रिया (फॉलबैक) |

## इसे कैसे ठीक करें

### 1. लॉगिन करें

यदि आपने लॉगिन नहीं किया है:
1. POS लॉगिन स्क्रीन खोलें
2. अपने वर्डप्रेस क्रेडेंशियल्स दर्ज करें
3. प्रमाणीकरण प्रक्रिया पूरी करें

### 2. सत्र स्थिति जांचें

यदि आपको लगा कि आप लॉगिन हैं:
- आपका सत्र समाप्त हो सकता है
- अधिक जानकारी के लिए [API02002](./API02002) (टोकन समाप्त) देखें
- फिर से लॉगिन करें ताकि आप फिर से एक्सेस बहाल कर सकें

### 3. सर्वर कॉन्फ़िगरेशन सत्यापित करें

सुनिश्चित करें कि API एंडपॉइंट ठीक से कॉन्फ़िगर किए गए हैं:
- वूकॉमर्स REST API सक्षम होना चाहिए
- WCPOS प्लगइन सक्रिय होना चाहिए
- प्रमाणीकरण एंडपॉइंट्स को पहुंच योग्य होना चाहिए

### 4. ब्राउज़र/ऐप समस्याओं की जांच करें

यदि आप अप्रत्याशित रूप से लॉग आउट हो रहे हैं:
- ब्राउज़र कैश साफ करें (यदि वेब संस्करण का उपयोग कर रहे हैं)
- जांचें कि कुकीज़/स्थानीय संग्रह अवरुद्ध नहीं हो रहे हैं
- सत्यापित करें कि ऐप के पास डेटा संगृहीत करने की अनुमति है

## क्या प्रमाणीकरण की आवश्यकता है?

अधिकांश POS संचालन के लिए प्रमाणीकरण की आवश्यकता होती है:
- उत्पादों और ग्राहकों को देखना
- ऑर्डर बनाना और संपादित करना
- भुगतान प्रक्रिया करना
- रिपोर्ट तक पहुंचना

केवल प्रारंभिक लॉगिन स्क्रीन बिना प्रमाणीकरण के उपलब्ध है।
