# API03001: अवैध अनुरोध प्रारूप

## इसका अर्थ क्या है

सर्वर को भेजे गए अनुरोध का प्रारूप अपेक्षित नहीं था। सर्वर यह नहीं समझ पाया कि POS क्या मांग कर रहा था क्योंकि अनुरोध की संरचना गलत थी।

## सामान्य कारण

- **भ्रष्ट डेटा** — डेटा भेजने से पहले भ्रष्ट हो गया
- **सॉफ्टवेयर बग** — POS एप्लिकेशन में एक समस्या
- **प्रॉक्सी हस्तक्षेप** — एक प्रॉक्सी या फ़ायरवॉल ने अनुरोध को संशोधित किया
- **वर्ण एन्कोडिंग समस्याएं** — विशेष अक्षरों को सही ढंग से एन्कोड नहीं किया गया

## सर्वर त्रुटि मैपिंग

यह त्रुटि कोड तब ट्रिगर होता है जब सर्वर लौटाता है:

| सर्वर कोड | स्रोत |
|-------------|--------|
| HTTP 400 | कोई भी सर्वर प्रतिक्रिया (फॉलबैक) |

## समाधान कैसे करें

### 1. क्रिया को फिर से करें

कभी-कभी यह एक बार की समस्या होती है:
- एक क्षण रुकें और फिर से प्रयास करें
- POS को रीफ्रेश करें और फिर से कोशिश करें

### 2. विशेष अक्षरों की जाँच करें

यदि आप विशेष अक्षरों के साथ डेटा दर्ज कर रहे हैं:
- इमोजी या असामान्य प्रतीकों को हटाने का प्रयास करें
- उत्पाद नामों आदि के लिए मानक अक्षरों का उपयोग करें

### 3. POS अपडेट करें

यह सुनिश्चित करें कि आप नवीनतम संस्करण चला रहे हैं:
- ऐप अपडेट की जाँच करें
- अपने सर्वर पर WCPOS प्लगइन को अपडेट करें

### 4. नेटवर्क कॉन्फ़िगरेशन की जाँच करें

यदि आप एक प्रॉक्सी के पीछे हैं:
- सत्यापित करें कि प्रॉक्सी अनुरोधों को संशोधित नहीं कर रहा है
- फ़ायरवॉल नियमों की जाँच करें
- किसी अन्य नेटवर्क से पहुंचने का प्रयास करें

### 5. समस्या की रिपोर्ट करें

यदि यह लगातार होता है:
- नोट करें कि कौन सी क्रिया त्रुटि को ट्रिगर करती है
- विवरण के लिए ब्राउज़र कंसोल की जाँच करें (वेब संस्करण)
- पुनरावृत्ति के चरणों के साथ [GitHub](https://github.com/wcpos) पर रिपोर्ट करें
