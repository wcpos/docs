# API02001: अमान्य क्रेडेंशियल्स

## इसका मतलब

आपके द्वारा प्रविष्ट किया गया उपयोगकर्ता नाम या पासवर्ड गलत है। सर्वर ने लॉगिन प्रयास को अस्वीकार कर दिया क्योंकि क्रेडेंशियल्स किसी मान्य खाते से मेल नहीं खाते हैं।

## सामान्य कारण

- **उपयोगकर्ता नाम या पासवर्ड में टाइपो** — कैप्स लॉक या अतिरिक्त स्थानों की जांच करें
- **गलत खाता** — एक अलग साइट के लिए क्रेडेंशियल्स का उपयोग करना
- **पासवर्ड हाल ही में बदला गया** — पासवर्ड अपडेट किया गया, लेकिन POS में पुराना है
- **खाता हटा दिया गया** — उपयोगकर्ता खाता अब मौजूद नहीं है

## सर्वर त्रुटि मानचित्रण

यह त्रुटि कोड तब सक्रिय होता है जब सर्वर लौटाता है:

| सर्वर कोड | स्रोत |
|-----------|-------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT प्रमाणीकरण प्लगइन |

## इसे कैसे ठीक करें

### 1. अपने क्रेडेंशियल्स की जांच करें

- अपने उपयोगकर्ता नाम की दोबारा जांच करें (आमतौर पर ईमेल या WordPress उपयोगकर्ता नाम)
- सुनिश्चित करें कि पासवर्ड सही है
- पुष्टि करने के लिए सीधे WordPress एडमिन में लॉगिन करने का प्रयास करें

### 2. अपना पासवर्ड रीसेट करें

यदि आपने अपना पासवर्ड भूल गए हैं:
1. अपने WordPress लॉगिन पृष्ठ पर जाएं
2. "क्या आपने अपना पासवर्ड खो दिया?" पर क्लिक करें
3. रीसेट लिंक प्राप्त करने के लिए अपना ईमेल दर्ज करें
4. अपना पासवर्ड अपडेट करें
5. नए पासवर्ड के साथ POS में लॉगिन करें

### 3. उपयोगकर्ता खाता जांचें

WordPress एडमिन में:
- उपयोगकर्ता → सभी उपयोगकर्ताओं पर जाएं
- जांचें कि खाता मौजूद है
- सुनिश्चित करें कि उसके पास उचित POS पहुंच अनुमति है

### 4. सहेजे गए क्रेडेंशियल्स साफ़ करें

POS में:
- पूरी तरह से लॉग आउट करें
- किसी भी सहेजी गई/कैश की गई लॉगिन जानकारी को साफ़ करें
- क्रेडेंशियल्स को ताजा दर्ज करें
