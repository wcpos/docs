# DB02001: डुप्लीकेट रिकॉर्ड

## इसका अर्थ

डेटाबेस ने एक ऐसा रिकॉर्ड डालने का प्रयास किया जिसका आईडी पहले से मौजूद है। प्रत्येक रिकॉर्ड में एक अद्वितीय पहचानकर्ता होना चाहिए।

## सामान्य कारण

- **सिंक संघर्ष** — एक ही डेटा दो बार सिंक किया गया
- **रेस स्थिति** — एक ही रिकॉर्ड को बनाने के कई संचालन
- **डेटा भ्रष्टता** — आईडी भ्रष्ट या डुप्लीकेट हो गए
- **आयात समस्या** — पहले से मौजूद डेटा को आयात किया जा रहा है

## समाधान कैसे करें

### 1. डेटा रीफ्रेश करें

सर्वर से पुनः सिंक करें:
- नवीनतम डेटा प्राप्त करें
- सिंक प्रक्रिया को संघर्ष को हल करने दें
- नया डेटा प्राथमिकता पाएगा

### 2. ऑपरेशन दोहराएं

नए रिकॉर्ड बनाते समय:
- सिस्टम ने पहले से इसे बना दिया हो सकता है
- जांचें कि रिकॉर्ड मौजूद है या नहीं
- सबमिट बटन पर बार-बार क्लिक करने से बचें

### 3. लोकल कैश साफ़ करें

लोकल डेटा रीसेट करें:
1. लोकल डेटाबेस/कैश साफ़ करें
2. फिर से लॉगिन करें
3. डेटा फ्रेश सिंक करें

### 4. WooCommerce में डुप्लीकेट की जाँच करें

यदि समस्या बनी रहती है:
- डुप्लीकेट एंट्री के लिए WooCommerce की जाँच करें
- एक ही आईडी वाले उत्पाद/ऑर्डर देखें
- एडमिन में डुप्लीकेट साफ़ करें

### 5. लगातार समस्याओं की रिपोर्ट करें

यदि यह बार-बार होता है:
- नोट करें कि किन क्रियाओं से यह होता है
- पैटर्न की जाँच करें
- विवरण के साथ समर्थन को रिपोर्ट करें

## यह महत्वपूर्ण क्यों है

डुप्लीकेट आईडी होने पर:
- यह स्पष्ट नहीं होगा कि कौन सा रिकॉर्ड सही है
- डेटा अधिलेखन
- सिंक समस्याएं

यह त्रुटि डेटा भ्रष्टता को रोकती है।
