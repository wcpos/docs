# API01002: कनेक्शन अस्वीकार

## इसका क्या मतलब है

सर्वर ने सक्रिय रूप से कनेक्शन प्रयास को अस्वीकार कर दिया। यह टाइमआउट से अलग है — सर्वर ने उत्तर दिया, लेकिन "नहीं" कहा।

## सामान्य कारण

- **सर्वर बंद है** — वेब सर्वर (Apache/Nginx) चल नहीं रहा हो सकता है
- **गलत पोर्ट** — गलत पोर्ट पर कनेक्ट करने का प्रयास
- **फायरवॉल अवरोध** — एक फायरवॉल कनेक्शन को रोक रहा है
- **आईपी प्रतिबंध** — आपका आईपी पता अवरुद्ध हो सकता है

## कैसे ठीक करें

### 1. सुनिश्चित करें कि सर्वर चालू है

अपने WordPress साइट तक सीधे ब्राउज़र में पहुंचने का प्रयास करें। यदि यह लोड नहीं हो रहा है, तो अपने होस्टिंग प्रदाता से संपर्क करें।

### 2. URL कॉन्फ़िगरेशन की जांच करें

सुनिश्चित करें कि आप सही URL का उपयोग कर रहे हैं:
- `http://` बनाम `https://` की पुष्टि करें
- डोमेन नाम में टाइपो की जांच करें
- यदि गैर-मानक पोर्ट का उपयोग कर रहे हैं, तो सही पोर्ट की पुष्टि करें

### 3. फायरवॉल सेटिंग्स की जांच करें

यदि आपको अपने सर्वर तक पहुंच है:
- यह सुनिश्चित करें कि पोर्ट 80 (HTTP) या 443 (HTTPS) खुला है
- जांचें कि आपकी आईपी सफेद सूची में है
- कोई भी सुरक्षा प्लगइन्स की समीक्षा करें जो API एक्सेस को रोक सकती हैं

### 4. अपने होस्टिंग प्रदाता से संपर्क करें

वे निम्नलिखित की जांच कर सकते हैं:
- सर्वर की स्थिति और लॉग्स
- फायरवॉल कॉन्फ़िगरेशन
- क्या आपकी आईपी अवरुद्ध की गई है
