# API06002: गायब API URL {#api06002-missing-api-url}

## इसका अर्थ {#what-this-means}

आपकी WooCommerce स्टोर से कनेक्ट करने के लिए कोई URL कॉन्फ़िगर नहीं किया गया है। POS को यह जानने की आवश्यकता है कि आपकी स्टोर कहाँ स्थित है।

## सामान्य कारण {#common-causes}

- **पहली बार सेटअप** — URL कभी नहीं डाला गया
- **कॉन्फ़िगरेशन साफ़ किया गया** — सेटिंग्स रीसेट या हटा दी गईं
- **ऐप डेटा साफ़ किया गया** — एप्लिकेशन डेटा हटा दिया गया
- **माइग्रेशन समस्या** — अपडेट या माइग्रेशन के दौरान URL खो गया

## कैसे सुधारें {#how-to-fix}

### 1. अपनी स्टोर का URL दर्ज करें {#1-enter-your-store-url}

POS ऐप में:
1. सेटिंग्स या लॉगिन स्क्रीन खोलें
2. URL/साइट कॉन्फ़िगरेशन फ़ील्ड खोजें
3. अपनी WordPress साइट का URL दर्ज करें
4. उदाहरण: `https://yourstore.com`

### 2. अपने WordPress URL को खोजें {#2-find-your-wordpress-url}

यदि आप अपने URL के बारे में निश्चित नहीं हैं:
1. WordPress व्यवस्थापक में लॉगिन करें
2. सेटिंग्स → सामान्य पर जाएं
3. "WordPress पता (URL)" देखें
4. इस URL का उपयोग POS में करें

### 3. संग्रहीत सेटिंग्स की जांच करें {#3-check-stored-settings}

यदि URL पहले कॉन्फ़िगर किया गया था:
- जांचें कि क्या ऐप डेटा गलती से साफ़ किया गया था
- सेटिंग्स बैकअप/निर्यात विकल्पों की खोज करें
- यदि आवश्यक हो तो URL फिर से दर्ज करें

### 4. URL सही है यह सत्यापित करें {#4-verify-url-is-correct}

सहेजने से पहले:
1. ब्राउज़र में URL का परीक्षण करें
2. सुनिश्चित करें कि यह आपकी WordPress साइट लोड करता है
3. `https://` या `http://` शामिल करें

## सेटअप चेकलिस्ट {#setup-checklist}

जब पहली बार WCPOS को कॉन्फ़िगर कर रहे हों:

1. ☐ अपनी WordPress साइट का URL दर्ज करें
2. ☐ सुनिश्चित करें कि WCPOS प्लगइन स्थापित और सक्रिय है
3. ☐ अपने WordPress क्रेडेंशियल्स के साथ लॉगिन करें
4. ☐ प्रारंभिक डेटा सिंक को पूरा होने देने की अनुमति दें
