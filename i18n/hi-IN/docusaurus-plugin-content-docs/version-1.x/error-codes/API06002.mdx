# API06002: API URL गायब है

## इसका क्या मतलब है

आपके WooCommerce स्टोर से कनेक्ट होने के लिए कोई URL कॉन्फ़िगर नहीं किया गया है। POS को पता होना चाहिए कि आपका स्टोर कहां स्थित है।

## सामान्य कारण

- **पहली बार सेटअप** — URL कभी दर्ज नहीं किया गया
- **कॉन्फ़िगरेशन साफ़ किया गया** — सेटिंग्स रीसेट या साफ़ की गई
- **ऐप डेटा साफ़ किया गया** — एप्लिकेशन डेटा हटाया गया
- **माइग्रेशन समस्या** — अपडेट या माइग्रेशन के दौरान URL खो गया

## कैसे ठीक करें

### 1. अपना स्टोर URL दर्ज करें

POS ऐप में:
1. सेटिंग्स या लॉगिन स्क्रीन खोलें
2. URL/साइट कॉन्फ़िगरेशन फ़ील्ड ढूंढें
3. अपना WordPress साइट URL दर्ज करें
4. उदाहरण: `https://yourstore.com`

### 2. अपना WordPress URL ढूंढें

अगर आपको URL के बारे में निश्चित नहीं है:
1. WordPress व्यवस्थापक में लॉग इन करें
2. सेटिंग्स → सामान्य पर जाएं
3. "WordPress पता (URL)" देखें
4. POS में इस URL का उपयोग करें

### 3. संग्रहीत सेटिंग्स की जाँच करें

अगर URL पहले कॉन्फ़िगर किया गया था:
- जाँच करें कि ऐप डेटा अनजाने में साफ़ नहीं किया गया
- सेटिंग्स बैकअप/निर्यात विकल्प देखें
- आवश्यकता पड़ने पर URL फिर से दर्ज करें

### 4. URL सही है यह सुनिश्चित करें

सहेजने से पहले:
1. ब्राउज़र में URL का परीक्षण करें
2. सुनिश्चित करें कि यह आपकी WordPress साइट लोड होती है
3. `https://` या `http://` शामिल करें

## सेटअप चेकलिस्ट

पहली बार WCPOS कॉन्फ़िगर करते समय:

1. ☐ अपना WordPress साइट URL दर्ज करें
2. ☐ सुनिश्चित करें कि WCPOS प्लगइन इंस्टॉल और सक्रिय है
3. ☐ अपने WordPress क्रेडेंशियल्स के साथ लॉगिन करें
4. ☐ प्रारंभिक डेटा सिंक पूरा होने दें
