# DB01002: डेटाबेस क्वेरी समय समाप्त

## इसका अर्थ

एक डेटाबेस क्रिया पूरी होने में बहुत अधिक समय ले रही है। स्थानीय डेटाबेस क्वेरी अनुमत अधिकतम समय से अधिक हो गई है।

## सामान्य कारण

- **बड़ा डेटा सेट** — बहुत अधिक संख्या में उत्पाद या रिकॉर्ड
- **जटिल क्वेरी** — कई शर्तों वाली क्वेरी
- **डिवाइस प्रदर्शन** — डिवाइस धीमा चल रहा है
- **डेटाबेस विखंडन** — डेटाबेस को अनुकूलित करने की आवश्यकता है

## समाधान कैसे करें

### 1. प्रतीक्षा करें और फिर से प्रयास करें

क्वेरी पुनः प्रयास पर सफल हो सकती है:
- POS को बंद और फिर से खोलें
- संचालन को फिर से करें
- अन्य गहन संचालनों से बचें

### 2. अन्य एप्लिकेशन बंद करें

डिवाइस संसाधन मुक्त करें:
- अप्रयुक्त ब्राउज़र टैब बंद करें
- अन्य एप्लिकेशन बंद करें
- आवश्यकता पड़ने पर डिवाइस को पुनः शुरू करें

### 3. डेटा दायरा कम करें

खोज करते समय:
- अधिक विशिष्ट खोज शब्द का उपयोग करें
- परिणामों को सीमित करने के लिए फ़िल्टर लागू करें
- बड़े संचालनों को छोटे भागों में विभाजित करें

### 4. डेटाबेस को अनुकूलित करें

साफ़ और पुनः सिंक करें:
1. स्थानीय डेटा साफ़ करें (यदि उपलब्ध सेटिंग में हो)
2. फिर से लॉगिन करें
3. डेटा को ताज़ा होने दें

### 5. डिवाइस प्रदर्शन की जाँच करें

यह सुनिश्चित करें कि आपका डिवाइस आवश्यकताओं को पूरा करता है:
- पर्याप्त RAM
- उपयुक्त प्रोसेसर गति
- HDD की तुलना में SSD अधिक पसंद किया जाता है

## बड़े कैटलॉग

यदि आके पास बहुत बड़ा उत्पाद कैटलॉग है:
- प्रारंभिक सिंक में अधिक समय लगता है
- POS में फ़िल्टर का उपयोग करने पर विचार करें
- कुछ संचालन स्वाभाविक रूप से अधिक समय लेते हैं
- अनुकूलन सलाह के लिए समर्थन से संपर्क करें
