# DB01002: क्वेरी टाइमआउट {#db01002-query-timeout}

## इसका मतलब {#what-this-means}

एक डेटाबेस ऑपरेशन को पूरा करने में बहुत समय लग गया। स्थानीय डेटाबेस क्वेरी ने अधिकतम अनुमति प्राप्त समय को पार कर लिया।

## सामान्य कारण {#common-causes}

- **बड़ा डेटा सेट** — उत्पादों या रिकॉर्ड्स की संख्या बहुत अधिक
- **जटिल क्वेरी** — कई शर्तों वाली क्वेरी
- **डिवाइस प्रदर्शन** — डिवाइस धीमा चल रहा है
- **डेटाबेस विखंडन** — डेटाबेस को अनुकूलन की आवश्यकता है

## इसे कैसे ठीक करें {#how-to-fix}

### 1. प्रतीक्षा करें और पुनः प्रयास करें {#1-wait-and-retry}

क्वेरी पुनः प्रयास पर सफल हो सकती है:
- POS को बंद करें और पुनः खोलें
- फिर से ऑपरेशन का प्रयास करें
- एक साथ अन्य गहन कार्यों से बचें

### 2. अन्य अनुप्रयोग बंद करें {#2-close-other-applications}

डिवाइस संसाधनों को मुक्त करें:
- अवांछित ब्राउज़र टैब बंद करें
- अन्य अनुप्रयोग बंद करें
- आवश्यकता पड़ने पर डिवाइस को पुनः प्रारंभ करें

### 3. डेटा दायरा कम करें {#3-reduce-data-scope}

यदि खोज कर रहे हैं:
- अधिक विशिष्ट खोज शब्दों का उपयोग करें
- परिणामों को संकीर्ण करने के लिए फ़िल्टर लागू करें
- बड़े कार्यों को छोटे हिस्सों में विभाजित करें

### 4. डेटाबेस का अनुकूलन करें {#4-optimise-the-database}

क्लियर और फिर से-सिंक करें:
1. स्थानीय डेटा साफ करें (यदि उपलब्ध हो तो सेटिंग्स विकल्प)
2. दोबारा लॉगिन करें
3. डेटा को ताजा समन्वयित करने दें

### 5. डिवाइस प्रदर्शन की जाँच करें {#5-check-device-performance}

सुनिश्चित करें कि आपका डिवाइस आवश्यकताओं को पूरा करता है:
- पर्याप्त RAM
- उचित प्रोसेसर स्पीड
- SSD, HDD पर प्राथमिकता

## बड़े कैटलॉग {#large-catalogues}

यदि आपके पास बहुत बड़ा उत्पाद कैटलॉग है:
- आरंभिक समन्वयन में अधिक समय लगता है
- POS में फ़िल्टर का उपयोग करने पर विचार करें
- कुछ कार्य स्वाभाविक रूप से अधिक समय लेते हैं
- अनुकूलन सलाह के लिए समर्थन से संपर्क करें
