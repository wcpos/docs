# API05001: WooCommerce API अक्षम

## इसका अर्थ

आपकी साइट पर WooCommerce REST API अक्षम है। WCPOS को WooCommerce के साथ संवाद करने और स्टोर डेटा तक पहुंचने के लिए REST API की आवश्यकता होती है।

## सामान्य कारण

- **WooCommerce सेटिंग** — REST API को जानबूझकर अक्षम किया गया
- **सुरक्षा प्लगइन** — एक सुरक्षा प्लगइन API पहुंच को रोक रहा है
- **होस्टिंग प्रतिबंध** — आपके होस्ट ने REST API पहुंच को अक्षम कर दिया है
- **परमालिंक समस्याएं** — REST API के लिए परमालिंक कॉन्फ़िगर नहीं किए गए

## समाधान कैसे करें

### 1. WooCommerce REST API को सक्षम करें

WordPress व्यवस्थापक में:
1. WooCommerce → सेटिंग्स → उन्नत → REST API पर जाएं
2. सुनिश्चित करें कि REST API सक्षम है
3. API कुंजियां बनी हुई हैं, इसकी पुष्टि करें

### 2. WordPress REST API की जांच करें

WooCommerce API WordPress REST API पर निर्भर करता है:
1. अपने ब्राउज़र में `https://yoursite.com/wp-json/` पर जाएं
2. JSON डेटा लौटना चाहिए
3. यदि नहीं, तो [API05004](./API05004) देखें

### 3. सुरक्षा प्लगइन की जांच करें

API को रोक सकने वाले सामान्य सुरक्षा प्लगइन:
- **Wordfence** — फ़ायरवॉल सेटिंग्स की जांच करें
- **iThemes Security** — REST API सेटिंग्स की जांच करें
- **All In One WP Security** — फ़ायरवॉल नियमों की समीक्षा करें

REST API एंडपॉइंट्स या POS एप्लिकेशन को व्हाइटलिस्ट करें।

### 4. .htaccess की जांच करें

API पहुंच को रोकने वाले नियमों की जांच करें:

```apache
# /wp-json/ को रोकने वाले नियमों को हटाएं या संशोधित करें
# सुनिश्चित करें कि mod_rewrite सक्षम है
```

### 5. परमालिंक साफ़ करें

कभी-कभी परमालिंक सेटिंग्स को ताज़ा करने की आवश्यकता होती है:
1. सेटिंग्स → परमालिंक पर जाएं
2. "सहेजें" पर क्लिक करें (बिना कोई परिवर्तन किए)
3. यह रीराइट नियमों को पुनर्जनित करता है
