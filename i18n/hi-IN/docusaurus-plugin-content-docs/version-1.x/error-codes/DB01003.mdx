# DB01003: लेन-देन असफल

## इसका अर्थ

एक डेटाबेस लेन-देन पूरा नहीं किया जा सका। लेन-देन कई संचालनों को एक साथ जोड़ते हैं — यदि कोई भाग विफल होता है, तो डेटा की अखंडता बनाए रखने के लिए सब कुछ वापस कर दिया जाता है।

## सामान्य कारण

- **समवर्ती पहुंच** — एक ही डेटा को संशोधित करने का प्रयास कई संचालन
- **संग्रहण पूर्ण** — नए डेटा को लिखने के लिए कोई स्थान नहीं
- **डेटाबेस लॉक्ड** — कोई अन्य प्रक्रिया डेटाबेस को लॉक कर रही है
- **बिजली बाधा** — संचालन अप्रत्याशित रूप से बाधित

## कैसे ठीक करें

### 1. संचालन को फिर से करें

समस्या अस्थायी हो सकती है:
- थोड़ा इंतजार करें
- संचालन को फिर से करें
- बार-बार तेजी से प्रयास न करें

### 2. संघर्षों की जाँच करें

यदि कई उपकरण या टैब खुले हैं:
- एक बार में एक इंस्टेंस का उपयोग करें
- डुप्लीकेट ब्राउज़र टैब बंद करें
- बहु-उपकरण उपयोग का समन्वय करें

### 3. संग्रहण स्थान की जाँच करें

डेटा के लिए जगह सुनिश्चित करें:
- उपलब्ध डिस्क स्थान की जाँच करें
- जगह खाली करें यदि आवश्यक हो
- ब्राउज़र कैश साफ करें (वेब संस्करण)

### 4. एप्लिकेशन को पुनः शुरू करें

डेटाबेस की स्थिति को रीसेट करें:
1. POS को पूरी तरह बंद करें
2. कुछ सेकंड प्रतीक्षा करें
3. एप्लिकेशन को फिर से खोलें

### 5. साफ करें और फिर से सिंक करें

यदि लेन-देन लगातार विफल होते हैं:
1. स्थानीय डेटा साफ करें
2. फिर से लॉगिन करें
3. सर्वर से ताजा सिंक करें

## लेन-देन सुरक्षा

WCPOS लेन-देन का उपयोग करता है ताकि सुनिश्चित हो:
- डेटा की निरंतरता
- पूर्ण संचालन (सब कुछ या कुछ नहीं)
- आंशिक अपडेट से सुरक्षा

जब एक लेन-देन विफल होता है, तो आपका डेटा अखंड रहता है।
