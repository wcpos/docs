# DB01003: लेनदेन विफल {#db01003-transaction-failed}

## इसका क्या अर्थ है {#what-this-means}

एक डेटाबेस लेनदेन पूरा नहीं किया जा सका। लेनदेन कई ऑपरेशनों को एक साथ समूहित करता है — यदि कोई भी भाग विफल होता है, तो सब कुछ डेटा अखंडता बनाए रखने के लिए वापस रोल किया जाता है।

## सामान्य कारण {#common-causes}

- **समानांतर पहुंच** — कई ऑपरेशन एक ही डेटा को संशोधित करने की कोशिश कर रहे हैं
- **स्टोरेज भरा हुआ** — नए डेटा को लिखने के लिए कोई स्थान नहीं
- **डेटाबेस लॉक** — एक अन्य प्रक्रिया डेटाबेस को लॉक कर रही है
- **पावर विघटन** — ऑपरेशन अप्रत्याशित रूप से बाधित

## इसे कैसे ठीक करें {#how-to-fix}

### 1. ऑपरेशन को फिर से प्रयास करें {#1-retry-the-operation}

समस्या अस्थायी हो सकती है:
- एक पल प्रतीक्षा करें
- ऑपरेशन को फिर से प्रयास करें
- तेजी से पुनरावृत्त प्रयासों से बचें

### 2. संघर्षों की जांच करें {#2-check-for-conflicts}

यदि कई उपकरण या टैब खुले हैं:
- एक समय में एक उदाहरण का उपयोग करें
- डुप्लिकेट ब्राउज़र टैब बंद करें
- बहु-डिवाइस उपयोग का समन्वय करें

### 3. स्टोरेज स्पेस की जांच करें {#3-check-storage-space}

यह सुनिश्चित करें कि डेटा के लिए स्थान है:
- उपलब्ध डिस्क स्थान की जांच करें
- आवश्यकता होने पर स्थान खाली करें
- ब्राउज़र कैश साफ करें (वेब संस्करण)

### 4. एप्लिकेशन को पुनः प्रारंभ करें {#4-restart-the-application}

डेटाबेस स्थिति को रीसेट करें:
1. POS को पूरी तरह से बंद करें
2. कुछ सेकंड प्रतीक्षा करें
3. एप्लिकेशन को फिर से खोलें

### 5. साफ़ करें और फिर से सिंक करें {#5-clear-and-re-sync}

यदि लेनदेन लगातार विफल होते हैं:
1. स्थानीय डेटा साफ़ करें
2. फिर से लॉगिन करें
3. सर्वर से ताजा सिंक करें

## लेनदेन सुरक्षा {#transaction-safety}

WCPOS लेनदेन का उपयोग यह सुनिश्चित करने के लिए करता है:
- डेटा संगतता
- पूर्ण ऑपरेशन (सब कुछ या कुछ नहीं)
- आंशिक अपडेट से सुरक्षा

जब एक लेनदेन विफल होता है, तो आपका डेटा संगत रहता है।
