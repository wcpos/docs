# API04006: संसाधन नहीं मिला

## इसका क्या अर्थ है

सर्वर अनुरोधित संसाधन को नहीं ढूंढ पाया। यह आमतौर पर HTTP 404 स्थिति कोड के अनुरूप होता है, जो दर्शाता है कि आप जिस उत्पाद, ऑर्डर, ग्राहक या अन्य संसाधन को एक्सेस करने की कोशिश कर रहे हैं, वह मौजूद नहीं है।

## सामान्य कारण

- **रिकॉर्ड हटाया गया** — संसाधन सर्वर पर हटा दिया गया था
- **गलत आईडी** — एक गलत या पुरानी आईडी का उपयोग किया जा रहा है
- **सिंक समस्याएं** — स्थानीय डेटा एक ऐसे संसाधन को संदर्भित करता है जो अब मौजूद नहीं है
- **URL विन्यास त्रुटि** — API एंडपॉइंट गलत है
- **परमालिंक समस्याएं** — WordPress परमालिंक को ताज़ा करने की आवश्यकता है

## सर्वर त्रुटि मैपिंग

यह त्रुटि कोड तब ट्रिगर होता है जब सर्वर निम्न प्रतिक्रिया देता है:

| सर्वर कोड | स्रोत |
|-------------|--------|
| `rest_no_route` | WordPress REST API |
| HTTP 404 | कोई भी सर्वर प्रतिक्रिया |

## समाधान कैसे करें

### 1. जांचें कि संसाधन मौजूद है

सत्यापित करें कि संसाधन अभी भी आपकी WooCommerce साइट पर मौजूद है:
- WordPress व्यवस्थापक में लॉग इन करें
- संबंधित अनुभाग (उत्पाद, ऑर्डर, ग्राहक) पर जाएं
- आईडी या नाम से वस्तु की खोज करें

### 2. स्थानीय डेटा ताज़ा करें

यदि संसाधन सर्वर-तरफ हटा दिया गया है:
1. POS सेटिंग्स खोलें
2. संबंधित डेटा अनुभाग पर जाएं
3. डेटा अपडेट करने के लिए सिंक/ताज़ा करें
4. हटाई गई वस्तु को स्थानीय रूप से हटा दिया जाना चाहिए

### 3. WordPress परमालिंक जांचें

यदि कई संसाधन नहीं मिले:
1. WordPress व्यवस्थापक में **सेटिंग्स → परमालिंक** पर जाएं
2. **सेव चेंज** पर क्लिक करें (बिना कोई बदलाव किए)
3. यह परमालिंक संरचना को ताज़ा करता है

### 4. API मार्गों की पुष्टि करें

REST API को सीधे परखें:
```
https://yoursite.com/wp-json/wc/v3/products
```
यदि यह 404 लौटाता है, तो सर्वर कॉन्फ़िगरेशन में समस्या हो सकती है।

### 5. प्लगइन संघर्षों की जांच करें

यदि REST API मार्ग अनुपलब्ध हैं:
1. सुनिश्चित करें कि WooCommerce सक्रिय है
2. सुनिश्चित करें कि WCPOS प्लगइन सक्रिय है
3. परीक्षण के लिए अन्य प्लगइन को अस्थायी रूप से अक्षम करें
