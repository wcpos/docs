# API04005: JSON पुनर्प्राप्ति का प्रयास

## इसका अर्थ क्या है

सर्वर ने कुछ अमान्य JSON सामग्री के साथ एक प्रतिक्रिया भेजी, लेकिन POS ने पुनर्प्राप्ति करने और वैध डेटा निकालने का प्रयास किया। यह एक महत्वपूर्ण त्रुटि नहीं, बल्कि एक सूचनात्मक सूचना है।

## क्या हुआ

POS ने पाया:
1. प्रतिक्रिया में JSON के पहले या बाद में अतिरिक्त सामग्री थी
2. मूल JSON डेटा अभी भी पहचाना जा सकता था
3. वैध JSON हिस्से को निकालने का पुनर्प्राप्ति प्रयास किया गया

## सामान्य कारण

- **PHP नोटिस आउटपुट** — JSON के साथ PHP चेतावनियां मिश्रित
- **डीबग आउटपुट** — विकास डीबगिंग सक्षम रखी गई
- **प्लगइन नोटिस** — अन्य प्लगइन्स नोटिस आउटपुट कर रहे
- **व्हाइटस्पेस समस्याएं** — JSON के आसपास अतिरिक्त व्हाइटस्पेस

## कैसे ठीक करें

### 1. PHP एरर डिस्प्ले बंद करें

`wp-config.php` में:
```php
define('WP_DEBUG_DISPLAY', false);
ini_set('display_errors', 0);
```

### 2. इसके बजाय एरर लॉगिंग सक्षम करें

डीबगिंग के लिए त्रुटियां लॉग करें:
```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
```

### 3. प्लगइन डीबग मोड की जांच करें

कुछ प्लगइन्स में डीबग मोड होते हैं जो अतिरिक्त सामग्री आउटपुट करते हैं:
- प्लगइन सेटिंग्स की समीक्षा करें
- डीबग/विकास मोड बंद करें
- विस्तृत लॉगिंग विकल्पों की जांच करें

### 4. हाल के परिवर्तनों की समीक्षा करें

यदि यह हाल ही में शुरू हुआ है:
- आपके सर्वर पर क्या बदला?
- क्या प्लगइन्स अपडेट किए गए?
- क्या PHP सेटिंग्स संशोधित की गई?

## क्या यह गंभीर है?

हालांकि POS ने इस समस्या से पुनर्प्राप्ति की, यह एक कॉन्फ़िगरेशन समस्या को दर्शाता है जिसे ठीक किया जाना चाहिए। पुनर्प्राप्ति प्रक्रिया:
- हमेशा काम नहीं कर सकती
- प्रोसेसिंग ओवरहेड जोड़ती है
- अन्य मुद्दों को छिपा सकती है

विश्वसनीय संचालन सुनिश्चित करने के लिए मूल कारण को ठीक करें।
