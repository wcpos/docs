# API06003: अवैध साइट कॉन्फ़िगरेशन

## इसका क्या अर्थ है

साइट कॉन्फ़िगरेशन अवैध या अधूरा है। इसमें गलत URL, प्रमाणीकरण सेटिंग्स या अन्य कॉन्फ़िगरेशन समस्याएं शामिल हो सकती हैं।

## सामान्य कारण

- **आंशिक कॉन्फ़िगरेशन** — कुछ सेटिंग्स गायब हैं
- **मेल न खाने वाली सेटिंग्स** — कॉन्फ़िगरेशन साइट से मेल नहीं खाता
- **बिगड़ा हुआ कॉन्फ़िगरेशन** — सेटिंग्स बिगड़ गई थीं
- **साइट परिवर्तनों** — साइट को POS कॉन्फ़िग से अपडेट किए बिना संशोधित किया गया था

## इसे ठीक करने के लिए

### 1. सभी सेटिंग्स की समीक्षा करें

पूर्ण कॉन्फ़िगरेशन की जांच करें:
- साइट URL सही है
- प्रमाणीकरण ठीक से कॉन्फ़िगर किया गया है
- कोई अतिरिक्त सेटिंग्स सही हैं

### 2. फिर से शुरू से कॉन्फ़िगर करें

यदि कॉन्फ़िगरेशन बिगड़ गया है:
1. सभी संग्रहीत सेटिंग्स को साफ़ करें
2. सेटअप प्रक्रिया फिर से शुरू करें
3. नई कॉन्फ़िगरेशन दर्ज करें

### 3. साइट की आवश्यकताओं की जांच करें

सुनिश्चित करें कि आपकी वर्डप्रेस साइट आवश्यकताओं को पूरा करती है:
- वर्डप्रेस स्थापित है और उपयोग में है
- वूकॉमर्स स्थापित और सक्रिय है
- WCPOS प्लगइन स्थापित और सक्रिय है
- पर्मालिंक्स सक्षम हैं (नहीं "सादा")

### 4. साइट एक्सेस का परीक्षण करें

इन URLs को ब्राउज़र में काम करते हुए सत्यापित करें:
- `https://yoursite.com/` — मुख्य साइट
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. वर्डप्रेस सेटिंग्स की जांच करें

वर्डप्रेस प्रशासन में:
1. सेटिंग्स → सामान्य — URLs सत्यापित करें
2. सेटिंग्स → पर्मालिंक्स — सुनिश्चित करें कि "सादा" नहीं है
3. वुचामर्स → स्थिति — समस्याओं के लिए जांच करें

## कॉन्फ़िगरेशन आवश्यकताएँ

वैध कॉन्फ़िगरेशन की आवश्यकता है:
- **साइट URL** — प्रोटोकॉल के साथ पूर्ण URL
- **प्रमाणीकरण** — मान्य क्रेडेंशियल या API कुंजी
- **अनुमतियाँ** — उपयोगकर्ता को POS एक्सेस है
- **प्लगइन** — आवश्यक प्लगइन सक्रिय हैं
