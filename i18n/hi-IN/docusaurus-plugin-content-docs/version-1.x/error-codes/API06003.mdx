# API06003: अमान्य साइट कॉन्फ़िगरेशन

## इसका अर्थ क्या है

साइट कॉन्फ़िगरेशन अमान्य या अपूर्ण है। इसमें गलत URLs, प्रमाणीकरण सेटिंग्स या अन्य कॉन्फ़िगरेशन समस्याएं शामिल हो सकती हैं।

## सामान्य कारण

- **आंशिक कॉन्फ़िगरेशन** — कुछ सेटिंग्स गायब हैं
- **मेल न खाते हुए सेटिंग्स** — कॉन्फ़िगरेशन साइट से मेल नहीं खाता
- **भ्रष्ट कॉन्फ़िगरेशन** — सेटिंग्स भ्रष्ट हो गई हैं
- **साइट में परिवर्तन** — साइट को संशोधित किया गया है बिना POS कॉन्फ़िग को अपडेट किए

## समाधान कैसे करें

### 1. सभी सेटिंग्स की समीक्षा करें

पूर्ण कॉन्फ़िगरेशन की जांच करें:
- साइट URL सही है
- प्रमाणीकरण सही ढंग से कॉन्फ़िगर किया गया है
- अन्य सेटिंग्स सही हैं

### 2. शुरू से फिर से कॉन्फ़िगर करें

यदि कॉन्फ़िगरेशन भ्रष्ट है:
1. सभी संग्रहीत सेटिंग्स को साफ़ करें
2. सेटअप प्रक्रिया को फिर से शुरू करें
3. नई कॉन्फ़िगरेशन दर्ज करें

### 3. साइट आवश्यकताओं की जाँच करें

यह सुनिश्चित करें कि आपकी WordPress साइट आवश्यकताओं को पूरा करती है:
- WordPress इंस्टॉल और सुलभ है
- WooCommerce इंस्टॉल और सक्रिय है
- WCPOS प्लगइन इंस्टॉल और सक्रिय है
- परमालिंक्स सक्षम हैं (सादा नहीं)

### 4. साइट पहुंच की जाँच करें

इन URLs को ब्राउज़र में काम करने की जाँच करें:
- `https://yoursite.com/` — मुख्य साइट
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. WordPress सेटिंग्स की जाँच करें

WordPress व्यवस्थापक में:
1. सेटिंग्स → सामान्य — URLs की पुष्टि करें
2. सेटिंग्स → परमालिंक्स — सुनिश्चित करें कि "सादा" नहीं है
3. WooCommerce → स्थिति — समस्याओं की जाँच करें

## कॉन्फ़िगरेशन आवश्यकताएं

एक मान्य कॉन्फ़िगरेशन के लिए आवश्यक है:
- **साइट URL** — प्रोटोकॉल के साथ पूर्ण URL
- **प्रमाणीकरण** — मान्य क्रेडेंशियल या API कुंजियां
- **अनुमतियां** — उपयोगकर्ता के पास POS पहुंच है
- **प्लगइन्स** — आवश्यक प्लगइन्स सक्रिय हैं
