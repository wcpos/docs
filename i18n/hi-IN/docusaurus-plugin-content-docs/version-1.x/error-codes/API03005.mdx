# API03005: दर सीमा अधिक हो गई {#api03005-rate-limit-exceeded}

## इसका क्या मतलब है {#what-this-means}

आपने सर्वर को अल्प समय में बहुत अधिक अनुरोध किए हैं। दर सीमांकन सर्वर को अत्यधिक भार से बचाता है।

## सामान्य कारण {#common-causes}

- **तेज़ सिंक संचालन** — एक साथ कई डिवाइसों का सिंक करना
- **आक्रामक ताज़ा** — डेटा को बहुत बार ताज़ा करना
- **सुरक्षा प्लगइन** — एक प्लगइन अनुरोध सीमाएँ लागू कर रहा है
- **होस्टिंग सीमाएँ** — आपकी होस्टिंग योजना में API दर सीमाएँ हैं

## सर्वर त्रुटि मैपिंग {#server-error-mapping}

यह त्रुटि कोड तब उत्पन्न होता है जब सर्वर लौटाता है:

| सर्वर कोड | स्रोत |
|-------------|--------|
| HTTP 429 | किसी भी सर्वर प्रतिक्रिया (फाल्बैक) |

## कैसे ठीक करें {#how-to-fix}

### 1. प्रतीक्षा करें और पुनः प्रयास करें {#1-wait-and-retry}

दर सीमाएँ अस्थायी होती हैं:
- 1-5 मिनट प्रतीक्षा करें
- POS स्वचालित रूप से पुनः प्रयास करेगा
- बार-बार मैन्युअल रूप से ताज़ा करने से बचें

### 2. समवर्ती ऑपरेशनों को कम करें {#2-reduce-concurrent-operations}

यदि एक से अधिक POS टर्मिनल का उपयोग कर रहे हैं:
- डिवाइसों के बीच प्रारंभिक सिंक को अंतराल पर करें
- सभी डिवाइसों को एक साथ ताज़ा करने से बचें
- एक के पूर्ण होने पर दूसरे को शुरू करें

### 3. सुरक्षा प्लगइन्स की जांच करें {#3-check-security-plugins}

Wordfence जैसे प्लगइन तेज अनुरोधों को ब्लॉक कर सकते हैं:
- POS एप्लिकेशन को व्हाइटलिस्ट करें
- अपने POS डिवाइस IPs को व्हाइटलिस्ट करें
- दर सीमांकन का थ्रेसहोल्ड समायोजित करें

### 4. होस्टिंग सीमाओं की समीक्षा करें {#4-review-hosting-limits}

कुछ होस्ट API अनुरोधों को सीमित करते हैं:
- अपनी होस्टिंग योजना विवरण की जाँच करें
- यदि सीमाएँ बहुत कम हैं तो अपग्रेड करने पर विचार करें
- सीमाओं को बढ़ाने के बारे में सहायता से संपर्क करें

### 5. सिंक की आवृत्ति को अनुकूलित करें {#5-optimise-sync-frequency}

POS सेटिंग्स में:
- स्वचालित ताज़ा अंतराल समायोजित करें
- उपयुक्त होने पर मैन्युअल सिंक का उपयोग करें
- अनावश्यक डेटा ताज़ा करने से बचें
