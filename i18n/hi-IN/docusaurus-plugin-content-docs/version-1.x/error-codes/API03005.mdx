# API03005: अधिक दर सीमा पार

## इसका अर्थ क्या है

आपने बहुत अधिक समय में सर्वर पर अनुरोध किए हैं। दर सीमा सर्वर को अधिभारित होने से बचाती है।

## सामान्य कारण

- **तीव्र समन्वय संचालन** — एक साथ कई डिवाइस समन्वयन
- **आक्रामक रिफ्रेश** — बहुत अधिक बार डेटा रिफ्रेश करना
- **सुरक्षा प्लगइन** — एक प्लगइन अनुरोध सीमाएं लागू कर रहा है
- **होस्टिंग सीमाएं** — आपकी होस्टिंग योजना में API दर सीमाएं हैं

## सर्वर त्रुटि मैपिंग

यह त्रुटि कोड तब ट्रिगर होता है जब सर्वर वापस करता है:

| सर्वर कोड | स्रोत |
|-------------|--------|
| HTTP 429 | कोई भी सर्वर प्रतिक्रिया (फॉलबैक) |

## कैसे ठीक करें

### 1. प्रतीक्षा करें और फिर से प्रयास करें

दर सीमाएं अस्थायी होती हैं:
- 1-5 मिनट प्रतीक्षा करें
- POS स्वचालित रूप से पुनः प्रयास करेगा
- बार-बार मैनुअल रिफ्रेश से बचें

### 2. समवर्ती संचालन कम करें

यदि कई POS टर्मिनल उपयोग कर रहे हैं:
- डिवाइसों में प्रारंभिक समन्वय को विभाजित करें
- सभी डिवाइसों को एक साथ रिफ्रेश न करें
- एक को पूरा होने दें उससे पहले दूसरा शुरू करें

### 3. सुरक्षा प्लगइन जांचें

Wordfence जैसे प्लगइन तेज अनुरोधों को ब्लॉक कर सकते हैं:
- POS एप्लिकेशन को व्हाइटलिस्ट करें
- अपने POS डिवाइस IP को व्हाइटलिस्ट करें
- दर सीमा की सीमाएं समायोजित करें

### 4. होस्टिंग सीमाओं की समीक्षा करें

कुछ होस्ट API अनुरोधों को सीमित करते हैं:
- अपनी होस्टिंग योजना विवरण की जाँच करें
- अगर सीमाएं बहुत कम हैं तो अपग्रेड पर विचार करें
- सीमाएं बढ़ाने के बारे में सहायता से संपर्क करें

### 5. समन्वय आवृत्ति अनुकूलित करें

POS सेटिंग्स में:
- स्वचालित रिफ्रेश अंतराल समायोजित करें
- उचित समय पर मैनुअल समन्वय का उपयोग करें
- अनावश्यक डेटा रिफ्रेश से बचें
