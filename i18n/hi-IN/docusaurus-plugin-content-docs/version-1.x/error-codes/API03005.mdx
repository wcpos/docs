# API03005: दर सीमा पार हो गई

## इसका क्या मतलब है

आपने एक छोटे समय में सर्वर पर बहुत सारे अनुरोध किए हैं। दर सीमित करना सर्वर को अव्यवस्थित होने से बचाता है।

## सामान्य कारण

- **तेज समन्वय संचालन** — कई उपकरण एक साथ समन्वयित हो रहे हैं
- **आक्रामक रिफ्रेश** — डेटा को बहुत बार रिफ्रेश करना
- **सुरक्षा प्लगइन** — एक प्लगइन अनुरोध सीमाएँ लागू कर रहा है
- **होस्टिंग सीमाएँ** — आपकी होस्टिंग योजना में एपीआई दर सीमाएँ हैं

## सर्वर त्रुटि मैपिंग

यह त्रुटि कोड तब ट्रिगर होता है जब सर्वर वापस करता है:

| सर्वर कोड | स्रोत |
|-----------|-------|
| HTTP 429  | कोई भी सर्वर प्रतिक्रिया (फॉलबैक) |

## ठीक कैसे करें

### 1. प्रतीक्षा करें और पुनः प्रयास करें

दर सीमाएँ अस्थायी होती हैं:
- 1-5 मिनट प्रतीक्षा करें
- पीओएस स्वचालित रूप से पुनः प्रयास करेगा
- बार-बार मैन्युअल रूप से रिफ्रेश करने से बचें

### 2. समानांतर संचालन को कम करें

यदि कई पीओएस टर्मिनल का उपयोग कर रहे हैं:
- उपकरणों के बीच प्रारंभिक समन्वय को इस्क्रित करें
- सभी उपकरणों को एक बार में रिफ्रेश करने से बचें
- एक को पूरा करने दें फिर दूसरे की शुरुआत करें

### 3. सुरक्षा प्लगइनों की जांच करें

Wordfence जैसे प्लगइन तेज अनुरोधों को ब्लॉक कर सकते हैं:
- पीओएस एप्लिकेशन को व्हाइटलिस्ट करें
- अपने पीओएस उपकरण के आईपी को व्हाइटलिस्ट करें
- दर सीमित करने के थ्रेशोल्ड को समायोजित करें

### 4. होस्टिंग सीमाओं की समीक्षा करें

कुछ होस्ट एपीआई अनुरोधों को सीमित करते हैं:
- अपनी होस्टिंग योजना के विवरण की जांच करें
- यदि सीमाएँ बहुत कम हैं तो अपग्रेड पर विचार करें
- सीमाएँ बढ़ाने के लिए समर्थन से संपर्क करें

### 5. समन्वय की आवृत्ति को ऑप्टिमाइज़ करें

पीओएस सेटिंग्स में:
- स्वचालित रिफ्रेश अंतराल को समायोजित करें
- उचित होने पर मैनुअल समन्वय का उपयोग करें
- अनावश्यक डेटा रिफ्रेश से बचें
