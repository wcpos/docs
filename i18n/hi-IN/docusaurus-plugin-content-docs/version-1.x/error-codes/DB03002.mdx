# DB03002: अमान्य डेटा प्रकार

## इसका क्या मतलब है

डेटा प्रकार उस चीज़ से मेल नहीं खाता है जिसकी अपेक्षा डेटाबेस करता है। उदाहरण के लिए, जहाँ संख्या की अपेक्षा की गई थी वहां पाठ प्रदान किया गया था।

## सामान्य कारण

- **उपयोगकर्ता इनपुट त्रुटि** — संख्यात्मक क्षेत्र में पाठ दर्ज करना
- **आयात मुद्दे** — आयातित डेटा का गलत प्रारूप
- **डेटा भ्रष्टाचार** — हस्तांतरण के दौरान मान क्षतिग्रस्त
- **प्लगइन संघर्ष** — किसी अन्य प्लगइन ने डेटा प्रकारों को संशोधित किया

## इसे कैसे ठीक करें

### 1. अपना इनपुट जांचें

आप जो डेटा दर्ज कर रहे हैं उसकी समीक्षा करें:
- **मूल्य** — संख्या होनी चाहिए (जैसे, `19.99`)
- **मात्राएँ** — पूरी संख्याएँ होनी चाहिए (जैसे, `5`)
- **आईडी** — संख्यात्मक होनी चाहिए

### 2. साफ करें और फिर से दर्ज करें

यदि डेटा भ्रष्ट हो गया है:
- क्षेत्र को साफ करें
- मान को फिर से मैन्युअल रूप से दर्ज करें
- बाहरी स्रोतों से कॉपी-पेस्ट करने से बचें

### 3. स्रोत डेटा की जाँच करें

यदि WooCommerce से समन्वय कर रहे हैं:
- WordPress प्रशासन में डेटा जांचें
- गलत प्रारूपित क्षेत्रों की तलाश करें
- स्रोत पर डेटा को ठीक करें

### 4. डेटा को फिर से सिंक करें

सर्वर से ताजा डेटा प्राप्त करें:
- स्थानीय कैश साफ करें
- डेटा को फिर से सिंक करें
- जांचें कि क्या समस्या हल होती है

### 5. प्लगइन संघर्षों की जांच करें

यदि अन्य WooCommerce प्लगइनों का उपयोग कर रहे हैं:
- वे अनपेक्षित तरीके से डेटा को संशोधित कर सकते हैं
- परीक्षण के लिए अस्थायी रूप से निष्क्रिय करें
- असंगतियों की रिपोर्ट करें

## सामान्य उदाहरण

| क्षेत्र | अपेक्षित | अमान्य |
|-------|----------|---------|
| मूल्य | `19.99` | `$19.99` |
| मात्रा | `5` | `five` |
| SKU | `ABC123` | — (कोई भी प्रारूप ठीक है) |
| उत्पाद आईडी | `42` | `product-42` |
