# API03004: अनुरोध बहुत बड़ा

## इसका अर्थ क्या है

आप जो अनुरोध भेज रहे हैं, वह सर्वर की आकार सीमाओं से अधिक है। यह आमतौर पर एक साथ बहुत अधिक डेटा भेजने की कोशिश करने पर होता है।

## सामान्य कारण

- **बड़े बैच ऑपरेशन** — एक साथ बहुत से रिकॉर्ड सिंक करने की कोशिश
- **बड़ी छवियाँ** — बहुत बड़ी छवियाँ अपलोड करना
- **बहुत अधिक आइटम** — अत्यधिक लाइन आइटम वाला ऑर्डर
- **सर्वर सीमाएँ** — PHP या वेब सर्वर के पास कम अपलोड सीमाएँ

## समाधान कैसे करें

### 1. अनुरोध का आकार कम करें

डेटा सिंक करते समय:
- छोटे बैच में सिंक करने का प्रयास करें
- POS को यह स्वचालित रूप से संभालना चाहिए
- वर्तमान सिंक पूरा होने तक प्रतीक्षा करें

### 2. छवि आकारों की जाँच करें

छवियाँ अपलोड करते समय:
- अपलोड करने से पहले छवियों का आकार कम करें
- संपीड़ित प्रारूप का उपयोग करें (JPEG बनाम BMP)
- अधिकांश उत्पाद छवियाँ 1MB से कम में अच्छी काम करती हैं

### 3. बड़े ऑर्डर को विभाजित करें

यदि ऑर्डर में बहुत अधिक आइटम हैं:
- कई ऑर्डर में विभाजित करने पर विचार करें
- सामान्य POS उपयोग में यह दुर्लभ है

### 4. सर्वर सीमाएँ बढ़ाएँ

अपने होस्टिंग प्रदाता से संपर्क करें या PHP सेटिंग्स संपादित करें:

```php
// php.ini या .htaccess में
upload_max_filesize = 64M
post_max_size = 64M
max_input_vars = 5000
```

### 5. वेब सर्वर सीमाओं की जाँच करें

Nginx या Apache के पास अपनी सीमाएँ हो सकती हैं:
- Nginx के लिए `client_max_body_size`
- Apache के लिए `LimitRequestBody`
