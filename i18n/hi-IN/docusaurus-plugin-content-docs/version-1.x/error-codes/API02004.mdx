# API02004: उपयोगकर्ता अधिकृत नहीं

## इसका अर्थ

आप लॉग इन हैं, लेकिन आपके उपयोगकर्ता खाते के पास अनुरोधित कार्य करने की अनुमति नहीं है। यह एक अधिकार मुद्दा है (आप क्या कर सकते हैं) बनाम प्रमाणीकरण मुद्दा (आप कौन हैं)।

## सामान्य कारण

- **उपयोगकर्ता भूमिका सीमाएं** — आपकी WordPress भूमिका में POS पहुंच शामिल नहीं है
- **POS पहुंच अक्षम** — आपके खाते को POS अनुमतियां नहीं दी गई हैं
- **सुविधा प्रतिबंध** — कुछ सुविधाएं विशिष्ट भूमिकाओं तक सीमित हैं
- **स्टोर प्रतिबंध** — आपको इस विशेष स्टोर तक पहुंच नहीं हो सकती

## सर्वर त्रुटि मैपिंग

यह त्रुटि कोड तब ट्रिगर होता है जब सर्वर लौटाता है:

| सर्वर कोड | स्रोत |
|-------------|--------|
| `rest_cannot_view` | WordPress REST API |
| `woocommerce_rest_cannot_view` | WooCommerce REST API |

## समाधान कैसे करें

### 1. उपयोगकर्ता भूमिका जांचें

WordPress एडमिन → उपयोगकर्ता में:
1. अपने उपयोगकर्ता खाते को खोजें
2. भूमिका की पुष्टि करें (जैसे, शॉप प्रबंधक, प्रशासक)
3. सुनिश्चित करें कि भूमिका में WooCommerce क्षमताएं शामिल हैं

### 2. POS पहुंच सक्षम करें

WordPress एडमिन → WooCommerce → POS → पहुंच में:
1. उपयोगकर्ता या भूमिका खोजें
2. POS पहुंच अनुमतियां सक्षम करें
3. परिवर्तन सहेजें

### 3. अतिरिक्त अनुमतियां का अनुरोध करें

अपने स्टोर प्रशासक से संपर्क करें ताकि:
- आपकी भूमिका को POS पहुंच दी जा सके
- आपको उचित अनुमतियों वाली भूमिका असाइन की जा सके
- आवश्यक विशिष्ट सुविधाएं सक्षम की जा सकें

### 4. स्टोर असाइनमेंट की जांच करें

यदि कई स्टोर का उपयोग कर रहे हैं:
- सुनिश्चित करें कि आप सही स्टोर पर हैं
- स्टोर-विशिष्ट अनुमतियों की जांच करें

## आवश्यक अनुमतियां

विभिन्न कार्यों के लिए विभिन्न क्षमताएं आवश्यक हैं:
- **उत्पाद देखें**: उत्पादों की पढ़ने की पहुंच
- **ऑर्डर बनाएं**: ऑर्डर बनाने/संपादित करने की क्षमताएं
- **ग्राहक प्रबंधित करें**: ग्राहक प्रबंधन क्षमताएं
- **रिपोर्ट तक पहुंच**: रिपोर्ट देखने की क्षमताएं
