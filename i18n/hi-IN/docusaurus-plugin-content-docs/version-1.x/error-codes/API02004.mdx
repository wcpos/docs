# API02004: उपयोगकर्ता अधिकृत नहीं

## इसका क्या अर्थ है

आप लॉग इन हैं, लेकिन आपके उपयोगकर्ता खाते में अनुरोधित कार्रवाई करने की अनुमति नहीं है। यह एक अधिकरण (आप क्या कर सकते हैं) की समस्या है न कि प्रमाणन (आप कौन हैं) की समस्या।

## सामान्य कारण

- **उपयोगकर्ता भूमिका सीमाएँ** — आपकी वर्डप्रेस भूमिका में POS एक्सेस शामिल नहीं है
- **POS एक्सेस अक्षम** — आपके खाते को POS अनुमतियाँ नहीं दी गई थीं
- **विशेषता प्रतिबंध** — कुछ विशेषताएँ विशिष्ट भूमिकाओं तक सीमित हैं
- **स्टोर प्रतिबंध** — आपको इस विशेष स्टोर तक पहुँच नहीं हो सकती है

## सर्वर त्रुटि मैपिंग

यह त्रुटि कोड तब उत्पन्न होता है जब सर्वर लौटाता है:

| सर्वर कोड        | स्रोत                      |
|------------------|---------------------------|
| `rest_cannot_view` | वर्डप्रेस REST API       |
| `woocommerce_rest_cannot_view` | वू-कॉमर्स REST API  |

## समाधान कैसे करें

### 1. उपयोगकर्ता भूमिका जांचें

वर्डप्रेस प्रशासन → उपयोगकर्ता:
1. अपने उपयोगकर्ता खाते को खोजें
2. भूमिका की पुष्टि करें (जैसे, दुकान प्रबंधक, प्रशासक)
3. सुनिश्चित करें कि भूमिका में वू-कॉमर्स क्षमताएँ शामिल हैं

### 2. POS एक्सेस सक्षम करें

वर्डप्रेस प्रशासन → वू-कॉमर्स → POS → एक्सेस:
1. उपयोगकर्ता या भूमिका खोजें
2. POS एक्सेस अनुमतियाँ सक्षम करें
3. परिवर्तन सहेजें

### 3. अतिरिक्त अनुमतियाँ मांगें

अपने स्टोर प्रशासक से संपर्क करें:
- आपकी भूमिका को POS एक्सेस देने के लिए
- आपको उचित अनुमतियों वाली भूमिका सौंपने के लिए
- आपकी आवश्यकता की विशिष्ट विशेषताएँ सक्षम करने के लिए

### 4. स्टोर असाइनमेंट चेक करें

यदि आप कई स्टोर का उपयोग कर रहे हैं:
- जांचें कि आपको सही स्टोर में असाइन किया गया है
- स्टोर-विशिष्ट अनुमतियों की जांच करें

## आवश्यक अनुमतियाँ

विभिन्न कार्यों के लिए विभिन्न क्षमताओं की आवश्यकता होती है:
- **उत्पाद देखें**: उत्पादों के लिए पढ़ने की अनुमति
- **आदेश बनाएँ**: आदेश बनाने/edit करने की क्षमताएँ
- **ग्राहकों का प्रबंधन करें**: ग्राहक प्रबंधन क्षमताएँ
- **रिपोर्ट्स तक पहुँच**: रिपोर्ट्स देखना क्षमताएँ
