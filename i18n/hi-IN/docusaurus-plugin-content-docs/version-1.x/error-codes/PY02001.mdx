# PY02001: भुगतान गेटवे त्रुटि

## इसका अर्थ

भुगतान गेटवे के साथ संचार करते समय एक त्रुटि हुई। भुगतान प्रोसेसर अनुरोध को पूरा नहीं कर सका।

## सामान्य कारण

- **गेटवे डाउनटाइम** — भुगतान सेवा में समस्याएं हो रही हैं
- **कॉन्फ़िगरेशन त्रुटि** — गेटवे सेटिंग्स गलत हैं
- **API क्रेडेंशियल** — अमान्य या समाप्त API कुंजियां
- **नेटवर्क समस्याएं** — गेटवे से कनेक्शन में समस्या

## समाधान कैसे करें

### 1. लेनदेन को फिर से प्रयास करें

कभी-कभी गेटवे में अस्थायी समस्याएं होती हैं:
- थोड़ा इंतजार करें
- भुगतान को फिर से प्रोसेस करने का प्रयास करें
- समस्या खुद ही हल हो सकती है

### 2. गेटवे स्थिति की जांच करें

अपने भुगतान प्रदाता के स्थिति पृष्ठ पर जाएं:
- Stripe: status.stripe.com
- PayPal: PayPal समुदाय या स्थिति की जांच करें
- Square: status.squareup.com
- ज्ञात बाधाओं की जांच करें

### 3. गेटवे कॉन्फ़िगरेशन की पुष्टि करें

WooCommerce एडमिन में:
1. WooCommerce → सेटिंग्स → भुगतान पर जाएं
2. अपने भुगतान गेटवे पर क्लिक करें
3. API कुंजियां सही हैं, यह सुनिश्चित करें
4. परीक्षण/लाइव मोड कुंजियों के साथ मेल खाता है, यह जांचें

### 4. API क्रेडेंशियल की जांच करें

क्रेडेंशियल वैध हैं, यह सुनिश्चित करें:
- API कुंजियां समाप्त नहीं हुई हैं
- लाइव लेनदेन के लिए लाइव कुंजियां
- परीक्षण लेनदेन के लिए परीक्षण कुंजियां
- क्रेडेंशियल सही खाते से मेल खाते हैं

### 5. एक अन्य गेटवे के साथ परीक्षण करें

यदि उपलब्ध है:
- एक वैकल्पिक गेटवे के साथ प्रोसेस करने का प्रयास करें
- नकद भुगतान को बैकअप के रूप में उपयोग करें
- बिक्री पूरी करें और बाद में समाधान करें

## स्टोर मालिकों के लिए

सामान्य कॉन्फ़िगरेशन समस्याएं:
- परीक्षण और लाइव API कुंजियों को मिश्रित करना
- अलग-अलग खातों की API कुंजियां
- अक्षम भुगतान विधियां
- समाप्त या रद्द की गई क्रेडेंशियल

यदि समस्या बनी रहती है तो अपने भुगतान प्रदाता के समर्थन से संपर्क करें।
