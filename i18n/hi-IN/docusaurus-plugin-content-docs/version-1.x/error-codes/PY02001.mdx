# PY02001: भुगतान गेटवे त्रुटि

## इसका क्या अर्थ है

भुगतान गेटवे के साथ संचार करते समय एक त्रुटि हुई। भुगतान प्रोसेसर अनुरोध को पूरा नहीं कर सका।

## सामान्य कारण

- **गेटवे डाउनटाइम** — भुगतान सेवा में समस्याएँ आ रही हैं
- **कॉन्फ़िगरेशन त्रुटि** — गेटवे सेटिंग्स गलत हैं
- **एपीआई क्रेडेंशियल्स** — अमान्य या समाप्त एपीआई कुंजी
- **नेटवर्क समस्याएँ** — गेटवे से संबंध संबंधी समस्याएँ

## इसे कैसे ठीक करें

### 1. लेन-देन को फिर से प्रयास करें

कभी-कभी गेटवे में अस्थायी समस्याएँ होती हैं:
- एक क्षण प्रतीक्षा करें
- भुगतान को फिर से प्रोसेस करने की कोशिश करें
- समस्या स्वयं हल हो सकती है

### 2. गेटवे स्थिति की जांच करें

अपने भुगतान प्रदाता के स्थिति पृष्ठ पर जाएँ:
- स्ट्राइप: status.stripe.com
- पेपाल: पेपाल समुदाय या स्थिति की जांच करें
- स्क्वायर: status.squareup.com
- ज्ञात विघटन के लिए जांचें

### 3. गेटवे कॉन्फ़िगरेशन की पुष्टि करें

WooCommerce प्रशासन में:
1. WooCommerce → सेटिंग्स → भुगतान पर जाएँ
2. अपने भुगतान गेटवे पर क्लिक करें
3. एपीआई कुंजियाँ सही हैं यह सुनिश्चित करें
4. जांचें कि परीक्षण/जीवंत मोड आपकी कुंजियों के साथ मेल खाता है

### 4. एपीआई क्रेडेंशियल्स की जांच करें

सुनिश्चित करें कि क्रेडेंशियल्स मान्य हैं:
- एपीआई कुंजी समाप्त नहीं हुई हैं
- जीवंत लेन-देन के लिए जीवंत कुंजी का उपयोग कर रहे हैं
- परीक्षण लेन-देन के लिए परीक्षण कुंजी का उपयोग कर रहे हैं
- क्रेडेंशियल्स सही खाते से मेल खाते हैं

### 5. किसी अन्य गेटवे के साथ परीक्षण करें

यदि उपलब्ध है:
- एक वैकल्पिक गेटवे के साथ प्रोसेस करने की कोशिश करें
- एक बैकअप के रूप में नकद भुगतान का उपयोग करें
- बिक्री पूरी करें और बाद में समायोजित करें

## स्टोर मालिकों के लिए

सामान्य कॉन्फ़िगरेशन मुद्दे:
- परीक्षण और जीवंत एपीआई कुंजियाँ मिलाना
- विभिन्न खातों से एपीआई कुंजियाँ
- अक्षम भुगतान विधियाँ
- समाप्त या रद्द किए गए क्रेडेंशियल्स

यदि समस्या बनी रहती है तो अपने भुगतान प्रदाता के समर्थन से संपर्क करें।
