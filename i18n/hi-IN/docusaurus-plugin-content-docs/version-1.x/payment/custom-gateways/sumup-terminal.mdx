# SumUp टर्मिनल गेटवे {#sumup-terminal-gateway}

SumUp टर्मिनल गेटवे आपको WooCommerce POS के भीतर सीधे SumUp कार्ड रीडर्स का उपयोग करके कार्ड भुगतानों को स्वीकार करने की अनुमति देता है। यह गेटवे व्यक्तिगत लेनदेन के लिए SumUp के भुगतान प्रसंस्करण प्रणाली के साथ सहज इंटीग्रेशन प्रदान करता है।

## इंस्टॉलेशन {#installation}

<Steps>
  <StepCard number="1" title="डाउनलोड और इंस्टॉल करें">

1. [WooCommerce के लिए SumUp टर्मिनल रिलीज़ पृष्ठ](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) पर जाएँ
2. नवीनतम **sumup-terminal-for-woocommerce.zip** फ़ाइल डाउनलोड करें
3. अपने WordPress प्रशासन में, `Plugins > Add New > Upload Plugin` पर जाएँ
4. ज़िप फ़ाइल अपलोड करें और प्लगइन सक्रिय करें

  </StepCard>
  <StepCard number="2" title="SumUp सेटिंग्स कॉन्फ़िगर करें">

1. `WP Admin > WooCommerce > Settings > Payments` पर जाएँ
2. भुगतान विधियों की सूची में **SumUp टर्मिनल** खोजें
3. सेटिंग्स तक पहुँचने के लिए **SumUp टर्मिनल** पर क्लिक करें
4. अपना **SumUp API कुंजी** दर्ज करें (आपके SumUp व्यापारी डैशबोर्ड से उपलब्ध)
5. सेटिंग्स सहेजें

:::note
आपको WooCommerce सेटिंग्स में SumUp टर्मिनल गेटवे सक्षम करने की आवश्यकता नहीं है। यह बाद के चरण में POS के लिए विशेष रूप से सक्षम किया जाएगा।
:::

  </StepCard>
  <StepCard number="3" title="अपने SumUp टर्मिनल को पेयर करें">

1. उसी सेटिंग पृष्ठ पर, **पेयर रीडर** खंड खोजें
2. अपने SumUp उपकरण पर, पेयरिंग कोड प्रदर्शित करने के लिए पेयरिंग स्क्रीन पर जाएँ
3. अपने SumUp उपकरण पर प्रदर्शित पेयरिंग कोड दर्ज करें
4. कनेक्शन स्थापित करने के लिए **"Pair Reader"** पर क्लिक करें
5. पुष्टि का इंतजार करें कि रीडर सफलतापूर्वक पेयर हो गया है

:::warning महत्वपूर्ण
भुगतान प्रसंस्करण करने से पहले रीडर को सफलतापूर्वक पेयर करना आवश्यक है। आगे बढ़ने से पहले पेयरिंग प्रक्रिया पूरी होनी चाहिए।
:::

  </StepCard>
  <StepCard number="4" title="WooCommerce POS में सक्षम करें" isLast>

1. `WP Admin > WooCommerce POS > Settings > Checkout` पर जाएँ
2. सूची में **SumUp टर्मिनल** गेटवे खोजें
3. POS में उपयोग के लिए गेटवे सक्षम करें
4. अपनी सेटिंग्स सहेजें

  </StepCard>
</Steps>

## उपयोग {#usage}

### भुगतानों की प्रक्रिया {#processing-payments}

1. **आइटम जोड़ें**: POS में अपने कार्ट में उत्पाद जोड़ें
2. **गेटवे चुनें**: भुगतान विधि के रूप में "SumUp टर्मिनल" चुनें
3. **भुगतान प्रारंभ करें**: अपने SumUp उपकरण पर एक नया भुगतान प्रारंभ करने के लिए क्लिक करें
4. **ग्राहक भुगतान**: ग्राहक SumUp टर्मिनल पर भुगतान पूरा करता है
5. **स्वचालित पूर्णता**: जब भुगतान का पता लगाया जाता है, तो आदेश स्वचालित रूप से पूरा हो जाता है

### भुगतान नियंत्रण {#payment-controls}

SumUp टर्मिनल गेटवे का उपयोग करते समय, आपके पास निम्नलिखित विकल्प होते हैं:

- **नया भुगतान प्रारंभ करें**: कनेक्टेड टर्मिनल पर भुगतान अनुरोध प्रारंभ करें
- **भुगतान रद्द करें**: प्रक्रिया में चल रहे भुगतान को रद्द करें
- **भुगतान स्थिति**: भुगतान प्रक्रिया की वर्तमान स्थिति की निगरानी करें

### आदेश प्रबंधन {#order-management}

- **स्वचालित अपडेट**: सफल भुगतान पर आदेश स्वचालित रूप से पूर्ण के रूप में चिह्नित किए जाते हैं
- **भुगतान ट्रैकिंग**: सभी भुगतान विवरण आदेश नोट्स में दर्ज किए जाते हैं
- **रसीद निर्माण**: सफल भुगतानों के बाद मानक POS रसीदें उत्पन्न की जाती हैं

## हार्डवेयर संगतता {#hardware-compatibility}

:::important कनेक्टिविटी आवश्यकताएँ
यह कार्यान्वयन SumUp के JavaScript SDK का उपयोग करता है, जिसका अर्थ है कि यह वेब एप्लिकेशन के माध्यम से काम करता है लेकिन केवल **इंटरनेट से जुड़े टर्मिनल** की आवश्यकता होती है। वर्तमान में ब्लूटूथ टर्मिनल का समर्थन नहीं किया गया है।
:::

### समर्थित टर्मिनल (इंटरनेट से जुड़े) {#supported-terminals-internet-connected}

- **SumUp सोलो** ✅ - WiFi/ईथरनेट से जुड़े टर्मिनल

### असमर्थित टर्मिनल (ब्लूटूथ) {#unsupported-terminals-bluetooth}

- **SumUp एयर** ❌ - केवल ब्लूटूथ
- **SumUp 3G** ❌ - केवल ब्लूटूथ

:::note भविष्य का समर्थन
ब्लूटूथ टर्मिनल समर्थन को **Q1 2026** में iOS और Android एप्लिकेशनों में जोड़ा जाएगा। यह एयर और 3G मॉडल सहित सभी SumUp प्रमाणित टर्मिनलों के लिए समर्थन सक्षम करेगा।
:::

## समस्या निवारण {#troubleshooting}

### सामान्य मुद्दे {#common-issues}

<Accordion>
  <AccordionItem question="रीडर पेयर नहीं होता">

  - सुनिश्चित करें कि आप एक समर्थित इंटरनेट से जुड़े टर्मिनल (सोलो केवल) का उपयोग कर रहे हैं
  - सत्यापित करें कि आपका SumUp सोलो उपकरण WiFi से कनेक्टेड और ऑनलाइन है
  - जांचें कि उपकरण पेयरिंग मोड में है और पेयरिंग कोड को सही ढंग से दर्ज किया गया है
  - पुष्टि करें कि आपके SumUp खाते में API पहुँच सक्षम है
  - पेयरिंग के दौरान स्थिर इंटरनेट कनेक्शन सुनिश्चित करें

  </AccordionItem>

  <AccordionItem question="भुगतान प्रक्रिया नहीं हो रही">

  - पुष्टि करें कि रीडर सही ढंग से पेयर किया गया है
  - सुनिश्चित करें कि आपका SumUp खाता सक्रिय है और उसमें कोई समस्या नहीं है
  - जांचें कि उपकरण में पर्याप्त बैटरी और कनेक्टिविटी है
  - पुष्टि करें कि API कुंजी सही और सक्रिय है

  </AccordionItem>

  <AccordionItem question="आदेश पूरा नहीं हो रहा">

  - इंटरनेट कनेक्शन की स्थिरता की जांच करें
  - अपने SumUp खाते में वेबहुक सेटिंग्स की पुष्टि करें
  - सुनिश्चित करें कि SumUp टर्मिनल पर भुगतान सफल था
  - API संचार समस्याओं के लिए WordPress त्रुटि लॉग की जांच करें

  </AccordionItem>
</Accordion>

### API गति सीमाएँ {#api-rate-limits}

गेटवे को API कॉल को कम करने और गति सीमित करने से बचाने के लिए ऑप्टिमाइज किया गया है:
- भुगतान स्थिति को कुशलतापूर्वक चेक किया जाता है
- अनावश्यक API अनुरोध कम किए जाते हैं
- अस्थायी विफलताओं से निपटने के लिए स्वचालित पुनः प्रयास लॉजिक्स लागू है

### मदद प्राप्त करना {#getting-help}

तकनीकी सहायता के लिए:
- मुद्दों की रिपोर्ट करने के लिए [GitHub रिपोजिटरी](https://github.com/wcpos/sumup-terminal-for-woocommerce) पर जाएँ
- API से संबंधित प्रश्नों के लिए [SumUp डेवलपर डोक्यूमेंटेशन](https://developer.sumup.com/) की जांच करें
- खाता और हार्डवेयर समस्याओं के लिए SumUp सहायता से संपर्क करें
