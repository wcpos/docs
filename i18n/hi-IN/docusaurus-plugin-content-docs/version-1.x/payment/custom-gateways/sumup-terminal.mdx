# SumUp टर्मिनल गेटवे {#sumup-terminal-gateway}

SumUp टर्मिनल गेटवे आपको WooCommerce POS के भीतर सीधे SumUp कार्ड रीडर्स का उपयोग करके कार्ड भुगतान स्वीकार करने की अनुमति देता है। यह गेटवे व्यक्तिगत लेनदेन के लिए SumUp के भुगतान प्रसंस्करण प्रणाली के साथ निर्बाध एकीकरण प्रदान करता है।

## स्थापना {#installation}

<Steps>
  <StepCard number="1" title="डाउनलोड और स्थापित करें">

1. [SumUp टर्मिनल के लिए WooCommerce रिलीज़ पृष्ठ](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) पर जाएं
2. नवीनतम **sumup-terminal-for-woocommerce.zip** फ़ाइल डाउनलोड करें
3. अपने WordPress व्यवस्थापक में, `Plugins > Add New > Upload Plugin` पर जाएं
4. ज़िप फ़ाइल अपलोड करें और प्लगइन को सक्रिय करें

  </StepCard>
  <StepCard number="2" title="SumUp सेटिंग्स कॉन्फ़िगर करें">

1. `WP Admin > WooCommerce > Settings > Payments` पर जाएं
2. भुगतान विधियों की सूची में **SumUp टर्मिनल** खोजें
3. सेटिंग्स तक पहुंचने के लिए **SumUp टर्मिनल** पर क्लिक करें
4. अपना **SumUp API कुंजी** दर्ज करें (जो आपके SumUp व्यापार पोर्टल से उपलब्ध है)
5. सेटिंग्स सहेजें

:::note
आपको WooCommerce सेटिंग्स में SumUp टर्मिनल गेटवे को सक्षम करने की आवश्यकता नहीं है। इसे एक बाद के चरण में POS के लिए विशेष रूप से सक्षम किया जाएगा।
:::

  </StepCard>
  <StepCard number="3" title="अपने SumUp टर्मिनल को पेयर करें">

1. उसी सेटिंग पृष्ठ पर **Pair Reader** अनुभाग खोजें
2. अपने SumUp उपकरण पर, पेयरिंग स्क्रीन पर जाएं ताकि पेयरिंग कोड प्रदर्शित हो सके
3. अपने SumUp उपकरण पर प्रदर्शित पेयरिंग कोड दर्ज करें
4. कनेक्शन स्थापित करने के लिए **"Pair Reader"** पर क्लिक करें
5. पढ़ने वाले की सफल पेयरिंग की पुष्टि की प्रतीक्षा करें

:::warning महत्वपूर्ण
भुगतान प्रक्रिया करने से पहले रीडर को सफलतापूर्वक पेयर किया जाना चाहिए। आगे बढ़ने से पहले पेयरिंग प्रक्रिया पूर्ण होने की सुनिश्चितता करें।
:::

  </StepCard>
  <StepCard number="4" title="WooCommerce POS में सक्षम करें" isLast>

1. `WP Admin > WooCommerce POS > Settings > Checkout` पर जाएं
2. सूची में **SumUp टर्मिनल** गेटवे खोजें
3. POS में उपयोग के लिए गेटवे को सक्षम करें
4. अपनी सेटिंग्स सहेजें

  </StepCard>
</Steps>

## उपयोग {#usage}

### भुगतान प्रक्रिया {#processing-payments}

1. **आइटम जोड़ें**: POS में अपने कार्ट में उत्पाद जोड़ें
2. **गेटवे चुनें**: भुगतान विधि के रूप में "SumUp टर्मिनल" चुनें
3. **भुगतान प्रारंभ करें**: अपने SumUp उपकरण पर एक नया भुगतान आरंभ करने के लिए क्लिक करें
4. **ग्राहक भुगतान**: ग्राहक SumUp टर्मिनल पर भुगतान पूरा करता है
5. **स्वत: पूर्णता**: भुगतान का पता चलते ही ऑर्डर स्वतः पूरा हो जाता है

### भुगतान नियंत्रण {#payment-controls}

SumUp टर्मिनल गेटवे का उपयोग करते समय, आपके पास निम्नलिखित विकल्प हैं:

- **नया भुगतान प्रारंभ करें**: जुड़े हुए टर्मिनल पर भुगतान अनुरोध करें
- **भुगतान रद्द करें**: वर्तमान में चल रहे भुगतान को रद्द करें
- **भुगतान स्थिति**: भुगतान प्रक्रिया की वर्तमान स्थिति की निगरानी करें

### आदेश प्रबंधन {#order-management}

- **स्वत: अपडेट**: सफल भुगतान के बाद आदेश को स्वतः पूर्ण के रूप में चिह्नित किया जाता है
- **भुगतान ट्रैकिंग**: सभी भुगतान विवरण आदेश नोट्स में दर्ज होते हैं
- **रसीद निर्माण**: सफल भुगतान के बाद मानक POS रसीदें उत्पन्न होती हैं

## हार्डवेयर संगतता {#hardware-compatibility}

:::important कनेक्टिविटी आवश्यकताएँ
यह कार्यान्वयन SumUp के JavaScript SDK का उपयोग करता है, जिसका अर्थ है कि यह वेब अनुप्रयोगों के माध्यम से कार्य करता है लेकिन **केवल इंटरनेट से जुड़े टर्मिनल की आवश्यकता है**। Bluetooth टर्मिनल वर्तमान में समर्थित नहीं हैं।
:::

### समर्थित टर्मिनल (इंटरनेट-कनेक्टेड) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet से जुड़े टर्मिनल

### असमर्थित टर्मिनल (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - केवल Bluetooth
- **SumUp 3G** ❌ - केवल Bluetooth

:::note भविष्य में समर्थन
Bluetooth टर्मिनल समर्थन **Q1 2026** में iOS और Android अनुप्रयोगों में जोड़ा जाएगा। इससे Air और 3G मॉडलों सहित सभी SumUp प्रमाणित टर्मिनलों का समर्थन सक्षम होगा।
:::

## समस्या निवारण {#troubleshooting}

### सामान्य समस्याएँ {#common-issues}

<Accordion>
  <AccordionItem question="रीडर पेयर नहीं होगा">

  - सुनिश्चित करें कि आप एक समर्थित इंटरनेट-कनेक्टेड टर्मिनल (केवल Solo) का उपयोग कर रहे हैं
  - सुनिश्चित करें कि आपका SumUp Solo उपकरण WiFi से जुड़ा है और ऑनलाइन है
  - जांचें कि उपकरण पेयरिंग मोड में है और पेयरिंग कोड सही ढंग से दर्ज किया गया है
  - सुनिश्चित करें कि आपके SumUp खाते में API एक्सेस सक्षम है
  - पेयरिंग के दौरान स्थिर इंटरनेट कनेक्शन सुनिश्चित करें

  </AccordionItem>

  <AccordionItem question="भुगतान प्रक्रिया नहीं हो रही है">

  - सुनिश्चित करें कि रीडर ठीक से पेयर किया गया है
  - सुनिश्चित करें कि आपका SumUp खाता सक्रिय और अच्छा स्थिति में है
  - सुनिश्चित करें कि उपकरण में पर्याप्त बैटरी और कनेक्टिविटी है
  - सुनिश्चित करें कि API कुंजी सही और सक्रिय है

  </AccordionItem>

  <AccordionItem question="आर्डर पूरा नहीं हो रहा है">

  - इंटरनेट कनेक्शन की स्थिरता की जांच करें
  - अपने SumUp खाते में वेबहुक सेटिंग्स को सत्यापित करें
  - सुनिश्चित करें कि SumUp टर्मिनल पर भुगतान सफल था
  - API संचार समस्याओं के लिए WordPress त्रुटि लॉग की जाँच करें

  </AccordionItem>
</Accordion>

### API दर सीमाएँ {#api-rate-limits}

गेटवे को API कॉल को कम करने और दर सीमाओं से बचने के लिए अनुकूलित किया गया है:
- भुगतान स्थिति को कुशलता से जांचा जाता है
- अनावश्यक API अनुरोधों को न्यूनतम किया जाता है
- अस्थायी विफलताओं को संभालने हेतु स्वतः पुनः प्रयास तर्क है

### मदद प्राप्त करना {#getting-help}

तकनीकी सहायता के लिए:
- मुद्दों की रिपोर्ट करने के लिए [GitHub रिपोजिटरी](https://github.com/wcpos/sumup-terminal-for-woocommerce) पर जाएं
- API से संबंधित प्रश्नों के लिए [SumUp डेवलपर दस्तावेज़](https://developer.sumup.com/) की जाँच करें
- खाता और हार्डवेयर समस्याओं के लिए SumUp सहायता से संपर्क करें
