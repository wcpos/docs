# SumUp टर्मिनल गेटवे {#sumup-terminal-gateway}

SumUp टर्मिनल गेटवे आपको WCPOS के भीतर सीधे SumUp कार्ड रीडर्स का उपयोग करके कार्ड भुगतान स्वीकार करने की अनुमति देता है। यह गेटवे व्यक्तिगत लेनदेन के लिए SumUp के भुगतान प्रसंस्करण प्रणाली के साथ निर्बाध एकीकरण प्रदान करता है।

## स्थापना {#installation}

<Steps>
  <StepCard number="1" title="डाउनलोड और स्थापित करें">

1. [SumUp टर्मिनल के लिए WooCommerce रिलीज़ पृष्ठ](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) पर जाएं
2. नवीनतम **sumup-terminal-for-woocommerce.zip** फ़ाइल डाउनलोड करें
3. अपने WordPress व्यवस्थापक में, जाएं `Plugins > Add New > Upload Plugin`
4. ज़िप फ़ाइल अपलोड करें और प्लगइन सक्रिय करें

  </StepCard>
  <StepCard number="2" title="SumUp सेटिंग्स कॉन्फ़िगर करें">

1. जाएं `WP Admin > WooCommerce > Settings > Payments`
2. भुगतान विधियों की सूची में **SumUp टर्मिनल** खोजें
3. सेटिंग्स तक पहुँचने के लिए **SumUp टर्मिनल** पर क्लिक करें
4. अपना **SumUp API Key** दर्ज करें (जो आपके SumUp व्यापारी डैशबोर्ड से उपलब्ध है)
5. सेटिंग्स सहेजें

:::note
आपको WooCommerce सेटिंग्स में SumUp टर्मिनल गेटवे को सक्षम करने की आवश्यकता नहीं है। यह बाद में POS के लिए विशेष रूप से सक्षम किया जाएगा।
:::

  </StepCard>
  <StepCard number="3" title="अपने SumUp टर्मिनल को जोड़ें">

1. उसी सेटिंग पृष्ठ पर, **पैर रीडर** अनुभाग खोजें
2. अपने SumUp उपकरण पर, पेयरिंग कोड प्रदर्शित करने के लिए पेयरिंग स्क्रीन पर जाएं
3. अपने SumUp उपकरण पर प्रदर्शित पेयरिंग कोड दर्ज करें
4. कनेक्शन स्थापित करने के लिए **"Pair Reader"** पर क्लिक करें
5. पुष्टि के लिए प्रतीक्षा करें कि रीडर सफलतापूर्वक पेयर हो गया है

:::warning महत्वपूर्ण
आप भुगतान प्रक्रिया शुरू करने से पहले रीडर को सफलतापूर्वक पेयर करना होगा। आगे बढ़ने से पहले कृपया पेयरिंग प्रक्रिया पूरी करें।
:::

  </StepCard>
  <StepCard number="4" title="WCPOS में सक्षम करें" isLast>

1. जाएं `WP Admin > POS > Settings > Checkout`
2. सूची में **SumUp टर्मिनल** गेटवे खोजें
3. POS में उपयोग के लिए गेटवे को सक्षम करें
4. अपनी सेटिंग्स सहेजें

  </StepCard>
</Steps>

## उपयोग {#usage}

### भुगतान प्रक्रिया {#processing-payments}

1. **आइटम जोड़ें**: POS में अपने कार्ट में उत्पाद जोड़ें
2. **गेटवे चुनें**: भुगतान विधि के रूप में "SumUp टर्मिनल" चुनें
3. **भुगतान शुरू करें**: अपने SumUp उपकरण पर नए भुगतान को शुरू करने के लिए क्लिक करें
4. **ग्राहक भुगतान**: ग्राहक SumUp टर्मिनल पर भुगतान पूरा करता है
5. **स्वचालित पूर्णता**: भुगतान का पता चलने पर आदेश स्वचालित रूप से पूरा हो जाता है

### भुगतान नियंत्रण {#payment-controls}

SumUp टर्मिनल गेटवे का उपयोग करते समय, आपके पास निम्नलिखित विकल्प होते हैं:

- **नया भुगतान प्रारंभ करें**: जुड़े हुए टर्मिनल पर भुगतान अनुरोध शुरू करें
- **भुगतान रद्द करें**: वर्तमान में प्रक्रिया में एक भुगतान रद्द करें
- **भुगतान स्थिति**: भुगतान प्रक्रिया की वर्तमान स्थिति की निगरानी करें

### आदेश प्रबंधन {#order-management}

- **स्वचालित अपडेट**: सफल भुगतान पर आदेश स्वचालित रूप से पूर्ण के रूप में चिह्नित होते हैं
- **भुगतान ट्रैकिंग**: सभी भुगतान विवरण आदेश नोट्स में दर्ज होते हैं
- **रसीद उत्पादन**: सफल भुगतान के बाद मानक POS रसीदें उत्पन्न होती हैं

## हार्डवेयर संगतता {#hardware-compatibility}

:::important कनेक्टिविटी आवश्यकताएँ
यह कार्यान्वयन SumUp के JavaScript SDK का उपयोग करता है, जिसका अर्थ है कि यह वेब अनुप्रयोगों के माध्यम से काम करता है लेकिन केवल **इंटरनेट से जुड़े टर्मिनलों** की आवश्यकता होती है। Bluetooth टर्मिनल वर्तमान में समर्थित नहीं हैं।
:::

### समर्थित टर्मिनल (इंटरनेट-से-कनेक्टेड) {#supported-terminals-internet-connected}

- **SumUp सोलो** ✅ - वाईफाई/ईथरनेट से जुड़े टर्मिनल

### असमर्थित टर्मिनल (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp एयर** ❌ - केवल Bluetooth
- **SumUp 3G** ❌ - केवल Bluetooth

:::note भविष्य में समर्थन
Bluetooth टर्मिनल समर्थन को iOS और Android अनुप्रयोगों में **Q1 2026** में जोड़ा जाएगा। यह एयर और 3G मॉडल सहित सभी SumUp प्रमाणित टर्मिनलों के लिए समर्थन सक्षम करेगा।
:::

## समस्या निवारण {#troubleshooting}

### सामान्य मुद्दे {#common-issues}

<Accordion>
  <AccordionItem question="रीडर पेयर नहीं हो रहा है">

  - सुनिश्चित करें कि आप एक समर्थित इंटरनेट-से-कनेक्टेड टर्मिनल (सोलो केवल) का उपयोग कर रहे हैं
  - जाँच करें कि आपका SumUp सोलो उपकरण वाईफाई से कनेक्टेड और ऑनलाइन है
  - चेक करें कि उपकरण पेयरिंग मोड में है और पेयरिंग कोड सही ढंग से दर्ज किया गया है
  - पुष्टि करें कि आपके SumUp खाते में API एक्सेस सक्षम है
  - पेयरिंग के दौरान स्थिर इंटरनेट कनेक्शन सुनिश्चित करें

  </AccordionItem>

  <AccordionItem question="भुगतान प्रक्रिया नहीं हो रही है">

  - जांचें कि रीडर सही तरीके से पेयर हो गया है
  - चेक करें कि आपका SumUp खाता सक्रिय और सही स्थिति में है
  - सुनिश्चित करें कि उपकरण में पर्याप्त बैटरी और कनेक्टिविटी है
  - पुष्टि करें कि API कुंजी सही और सक्रिय है

  </AccordionItem>

  <AccordionItem question="आदेश पूरा नहीं हो रहा है">

  - इंटरनेट कनेक्शन की स्थिरता की जाँच करें
  - अपने SumUp खाते में वेबहुक सेटिंग्स की पुष्टि करें
  - सुनिश्चित करें कि SumUp टर्मिनल पर भुगतान सफल रहा
  - API संचार मुद्दों के लिए WordPress त्रुटि लॉग की जाँच करें

  </AccordionItem>
</Accordion>

### API दर सीमाएँ {#api-rate-limits}

गेटवे को API कॉल को कम करने और दर सीमाओं से बचने के लिए अनुकूलित किया गया है:
- भुगतान स्थिति को कुशलता से जांचा जाता है
- अनावश्यक API अनुरोधों को कम किया जाता है
- स्वचालित पुनः प्रयास लॉजिक अस्थायी विफलताओं को संभालता है

### सहायता प्राप्त करना {#getting-help}

तकनीकी सहायता के लिए:
- मुद्दों की रिपोर्ट करने के लिए [GitHub रिपॉजिटरी](https://github.com/wcpos/sumup-terminal-for-woocommerce) पर जाएं
- API से संबंधित प्रश्नों के लिए [SumUp डेवलपर दस्तावेज़](https://developer.sumup.com/) देखें
- खाता और हार्डवेयर मुद्दों के लिए SumUp सहायता से संपर्क करें
