HI: # WCPOS ATUM Integration

HI: Integrates [ATUM Multi-Inventory](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/) with [WCPOS Pro](/getting-started/pro-license), enabling location-based inventory, pricing, and SKUs at the Point of Sale.

HI: ATUM Multi-Inventory lets you split a product's stock across multiple inventory locations — warehouses, retail stores, and so on. This plugin connects those ATUM locations to WCPOS Pro [stores](/stores) so each POS terminal sees the correct stock levels, prices, and SKUs for its physical location.

  </FeatureCard>

HI:     Choose pricing from WooCommerce defaults, WCPOS Pro per-store prices, or ATUM location-specific prices.
  </FeatureCard>

HI:     Orders deduct and restore stock at the correct ATUM location, with full audit trail in `atum_inventory_orders`.
  </FeatureCard>
  <FeatureCard icon="arrow-right" title="HI: Product Edit Write-Back">
HI:     POS edits to stock, price, and SKU sync back to the mapped ATUM inventory row for that location.

## अनुवादित: Installation {#installation}

<Steps>
  <StepCard number="1" title="HI: Install ATUM and Multi-Inventory">

HI: Install [ATUM Inventory Management](https://wordpress.org/plugins/atum-stock-manager-for-woocommerce/) and the [ATUM Multi-Inventory add-on](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/). Configure your inventory locations in ATUM.

  </StepCard>
  <StepCard number="2" title="HI: Install WCPOS ATUM Integration">

HI: Install from `WP Admin > POS > Settings > Extensions`, or download the latest release from the [GitHub releases page](https://github.com/wcpos/wcpos-atum/releases) and upload via `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="HI: Map stores to ATUM locations" isLast>

HI: Go to `POS > Stores`, edit a store, and configure the **ATUM Inventory** sidebar section. Pick the inventory location the store should use, choose a pricing source, and optionally enable SKU overrides.

  </StepCard>
</Steps>

## अनुवादित: Store Configuration {#store-configuration}

HI: The plugin adds an **ATUM Inventory** section to the WCPOS Pro store editor sidebar with three settings per store:

- HI: **Inventory Location** — which ATUM location this store pulls stock from.
- HI: **Pricing Source** — where product prices come from:
  - HI: *Default* — standard WooCommerce prices
  - HI: *WCPOS Pro* — per-store pricing set in WCPOS Pro
  - HI: *ATUM* — location-specific prices from the ATUM inventory
- HI: **SKU Override** — optionally use location-specific SKUs from ATUM instead of the product's main SKU.

## अनुवादित: POS Behavior {#pos-behavior}

HI: When a store has an ATUM location assigned, the product data served to the POS is automatically adjusted:

- HI: **Stock quantities** reflect the specific location's inventory, not the aggregate WooCommerce stock.
- HI: **Stock status** is recalculated based on the location's quantity.
- HI: **Prices** come from the configured pricing source.
- HI: **SKUs** are swapped to the ATUM location SKU if override is enabled.

HI: All adjustments happen transparently through the WCPOS REST API — no changes are needed on the POS app side. Product edits made from the POS are also written back to the mapped ATUM inventory row; see [Product Edit Write-Back](#product-edit-write-back) below.

## अनुवादित: Stock Management {#stock-management}

HI: For POS orders placed at stores with a mapped ATUM location, the plugin lets ATUM's native stock deduction flow handle the write — but steers it to the correct location:

1. HI: **REST payload injection.** When the POS creates or updates an order, the plugin injects a `mi_inventories` entry onto each line item so ATUM knows which location to draw from. Without this, ATUM would fall back to the main inventory.
2. HI: **Location-scoped inventory filter.** The plugin filters ATUM's candidate inventory list to only those linked to the store's mapped location term, ensuring the right one is picked on both reduction and restoration.

HI: ATUM itself performs the actual stock change on order and refund, writing rows to `atum_inventory_orders` with the real `order_id` — preserving ATUM's audit trail.

## अनुवादित: Product Edit Write-Back {#product-edit-write-back}

HI: When a cashier or manager edits a product or variation from the POS, the changes sync back to the mapped ATUM inventory row for that store's location — not just the main WooCommerce product. This keeps each location's stock, price, and SKU in sync with ATUM without manual updates in `WP Admin`.

HI: The write-back is triggered on WCPOS product and variation REST updates (`POST`, `PUT`, `PATCH` to `/wcpos/v1/products/...`) that include a `store_id`. The plugin looks up the store's mapped ATUM location and updates only the inventory row for that location — other locations are untouched.

### अनुवादित: What Syncs {#what-syncs}

HI: The write-back respects each store's configuration so ATUM data only changes when the store actually owns that data:

| HI: Field | HI: When it syncs | HI:
| --- | --- |
| HI: **Stock quantity** | HI: Always — every store with a mapped ATUM location keeps its location row in sync. | HI:
| HI: **Regular price / Sale price / Price** | HI: Only when the store's **Pricing Source** is set to *ATUM*. | HI:
| HI: **SKU** | HI: Only when **SKU Override** is enabled for the store. | HI:

HI: If the store uses *Default* or *WCPOS Pro* pricing, ATUM price fields are left alone so ATUM continues to serve as a reference price rather than the source of truth. The same applies to SKUs when override is off.

### अनुवादित: What Doesn't Trigger Write-Back {#what-doesnt-trigger-write-back}

- HI: Product creation (only updates write back — new products fall through to WooCommerce's normal save path).
- HI: Requests without a `store_id` — the POS has to tell the plugin which location to write to.
- HI: Stores without a mapped ATUM location.
- HI: Products without an existing ATUM inventory row for the store's location — the plugin will not create new inventory rows, only update existing ones.
