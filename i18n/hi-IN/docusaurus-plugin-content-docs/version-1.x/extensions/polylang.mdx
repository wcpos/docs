# WCPOS Polylang {#wcpos-polylang}

WCPOS में [Polylang](https://polylang.pro/) की जानकारी जोड़ता है, ताकि POS केवल एक भाषा के उत्पाद दिखाए। इससे उत्पाद खोज, कैटलॉग ग्रिड और कैशियर workflows में अनुवादित उत्पादों की डुप्लिकेट प्रविष्टियाँ नहीं आतीं। WCPOS Pro स्टोर प्रत्येक स्टोर के लिए भाषा तय कर सकते हैं; निःशुल्क इंस्टॉलेशन Polylang की डिफ़ॉल्ट भाषा पर वापस जाते हैं।

## यह क्या करता है {#what-it-does}

- WCPOS उत्पाद और वैरिएशन REST queries को भाषा के आधार पर फ़िल्टर करता है।
- WCPOS **fast-sync** routes को रोकता है, यानी वे हल्के `posts_per_page=-1` और `fields` requests जिनसे POS अपनी local index refresh करता है, ताकि अनुवादित duplicates client तक न पहुँचें।
- निःशुल्क इंस्टॉलेशन पर Polylang की डिफ़ॉल्ट भाषा लागू करता है।
- Pro इंस्टॉलेशन पर प्रत्येक store को store editor के नए **Language** section से अपनी भाषा चुनने देता है।
- fast-sync payload बनाते समय WCPOS **POS-only** product visibility का सम्मान करता है।

जब Polylang सक्रिय नहीं होता, integration साफ़ तौर पर कुछ नहीं करता। आप Polylang सक्षम करने से पहले भी plugin इंस्टॉल कर सकते हैं और कोई error नहीं आएगा।

## इंस्टॉलेशन {#installation}

<Steps>
  <StepCard number="1" title="Polylang इंस्टॉल करें">

[Polylang](https://wordpress.org/plugins/polylang/) या Polylang Pro इंस्टॉल करें और अपनी site languages सामान्य रूप से configure करें। सुनिश्चित करें कि कम से कम एक भाषा default के रूप में set है।

  </StepCard>
  <StepCard number="2" title="WCPOS Polylang इंस्टॉल करें">

`WP Admin > POS > Settings > Extensions` में WCPOS extensions directory से इंस्टॉल करें, या [GitHub releases page](https://github.com/wcpos/wcpos-polylang/releases) से latest release डाउनलोड करके `Plugins > Add New > Upload Plugin` से upload करें।

  </StepCard>
  <StepCard number="3" title="(Pro) हर store के लिए भाषा सेट करें" isLast>

अगर आप WCPOS Pro पर [multiple stores](/stores) चलाते हैं, तो `POS > Stores` पर जाएँ, store edit करें और **Language** sidebar section से भाषा चुनें। Polylang की default language उपयोग करने के लिए इसे *Default* पर छोड़ दें।

  </StepCard>
</Steps>

## हर store की भाषा (Pro) {#per-store-language-pro}

WCPOS Pro में plugin store editor sidebar में **Language** section जोड़ता है। हर store को एक Polylang language slug से pin किया जा सकता है। उस store को दिए जाने वाले products केवल उसी भाषा तक filtered रहेंगे। *Default* पर छोड़े गए stores Polylang की default language उपयोग करते हैं।

प्रत्येक store का value store post पर `_wcpos_polylang_language` meta के रूप में save होता है और WCPOS Pro stores REST API (`/wcpos/v1/stores`) से expose होता है, इसलिए यह POS के through किसी अन्य store setting की तरह round-trip करता है।

## Compatibility notes {#compatibility-notes}

- **POS-only products:** जब WCPOS settings में POS-only mode enabled होता है, online-only product IDs को fast-sync payload से बाहर रखा जाता है ताकि वे POS में न आएँ।
- **Free installs:** प्रति-store भाषा बदलने के लिए कोई UI नहीं है। plugin Polylang की default language का उपयोग करता है। अगर आपको प्रति-store languages चाहिए, तो [WCPOS Pro](/getting-started/pro-license) पर upgrade करें।
- **Plugin unavailable:** अगर Polylang deactivated है, plugin चुपचाप कुछ नहीं करता। यह errors नहीं throw करता और POS को block नहीं करता।

## Developer hooks {#developer-hooks}

Advanced उपयोग के लिए plugin कुछ filters expose करता है:

| Filter | उद्देश्य |
| --- | --- |
| `wcpos_polylang_resolved_language` | किसी request के लिए उपयोग की जाने वाली language override करता है। resolved slug और `WP_REST_Request` receive करता है। |
| `wcpos_polylang_default_language` | जब कोई per-store value set नहीं है, fallback language override करता है। |
| `wcpos_polylang_is_supported` | Polylang detection से अलग plugin को force on या off करता है। |
| `wcpos_polylang_minimum_version` | न्यूनतम Polylang version मांगता है; default में version gate नहीं है। |
