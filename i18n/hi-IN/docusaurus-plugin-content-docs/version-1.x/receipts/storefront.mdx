आपके WCPOS रसीद टेम्पलेट केवल कैश रजिस्टर तक सीमित नहीं हैं। ऑनलाइन स्टोर के ग्राहक My Account या ऑर्डर पुष्टिकरण पृष्ठ से अपने ऑर्डर की रसीद या इनवॉइस देख और डाउनलोड कर सकते हैं, जो **WP Admin → POS → टेम्पलेट्स** के कस्टम टेम्पलेट से रेंडर होता है।

:::info WCPOS 1.9.11 आवश्यक है
स्टोरफ्रंट रसीद एक्सेस WCPOS 1.9.11 में शामिल है। पुराने संस्करण रसीदों को केवल POS में रेंडर करते हैं।
:::

:::note यह पृष्ठ ग्राहकों द्वारा अपनी रसीदें डाउनलोड करने के बारे में है
कैशियर और कर्मचारियों को इसकी आवश्यकता नहीं है — POS की **ऑर्डर** स्क्रीन में ऑनलाइन ऑर्डर सहित प्रत्येक WooCommerce ऑर्डर पहले से सूचीबद्ध होता है, और तीन-बिंदु वाले मेनू से उसकी रसीद [देखी, प्रिंट या ईमेल की जा सकती है](/orders#receipt-reprinting)। यह पृष्ठ दूसरी आवश्यकता बताता है: **ग्राहकों** को अपनी वेबसाइट से स्वयं रसीद डाउनलोड करने देना।
:::

## My Account में डाउनलोड बटन {#my-account}

जैसे ही कोई रसीद टेम्पलेट **सक्रिय** होता है, लॉग-इन किए हुए ग्राहकों को **मेरा खाता → ऑर्डर** में योग्य ऑर्डर के पास **रसीद** बटन दिखाई देता है। इस पर क्लिक करने से स्टोरफ्रंट के डिफ़ॉल्ट रसीद टेम्पलेट से रेंडर की गई PDF (`receipt-1234.pdf`) डाउनलोड होती है।

डिफ़ॉल्ट रूप से, बटन **प्रसंस्करण में**, **पूर्ण**, और **वापस किए गए** ऑर्डर के लिए दिखाई देता है — वे स्थितियाँ जिनमें ऑर्डर का भुगतान हो चुका है। लंबित, रद्द किए गए और विफल ऑर्डर में यह दिखाई नहीं देता।

कॉन्फ़िगर करने के लिए कुछ नहीं है: टेम्पलेट सक्रिय करें और बटन उपलब्ध हो जाएगा। यदि इसे नहीं दिखाना है, तो डेवलपर इसे [फ़िल्टर से बंद कर सकता है](#developers)।

## `[wcpos_receipt]` शॉर्टकोड {#shortcode}

ऑर्डर पुष्टि ("धन्यवाद") पेज या **मेरा खाता → ऑर्डर देखें** पेज में रसीद का लिंक जोड़ने के लिए, इसका उपयोग करें:

```text
[wcpos_receipt]
```

बिना किसी विशेषता के, शॉर्टकोड दोनों पेज पर वर्तमान ऑर्डर को अपने-आप पहचानता है और *रसीद डाउनलोड करें (PDF)* लिंक प्रदर्शित करता है। अन्य पेज पर यह कुछ भी आउटपुट नहीं करता। स्पष्ट `order_id` केवल उस लॉग-इन ग्राहक के लिए काम करता है, जिसका वह ऑर्डर है।

### विशेषताएँ {#shortcode-attributes}

| विशेषता | डिफ़ॉल्ट | यह क्या करता है |
|---|---|---|
| `order_id` | *(स्वचालित रूप से पता लगाया गया)* | किसी विशिष्ट ऑर्डर का लिंक। केवल उस लॉग-इन ग्राहक के लिए काम करता है, जिसका वह ऑर्डर है। |
| `link_text` | `Download receipt (PDF)` | लिंक का टेक्स्ट। |
| `format` | `pdf` | `pdf` एक PDF डाउनलोड करता है; इसके बजाय `html` प्रिंट करने योग्य रसीद दृश्य खोलता है। |
| `template` | *(स्टोरफ़्रंट डिफ़ॉल्ट)* | स्टोरफ़्रंट डिफ़ॉल्ट के बजाय किसी विशिष्ट टेम्पलेट ID के साथ रेंडर करें। |
| `class` | `wcpos-receipt-link` | लिंक के लिए CSS क्लास। |
| `id` | *(कोई नहीं)* | लिंक के लिए HTML `id`। |

### उदाहरण {#shortcode-examples}

ऑर्डर पुष्टि पृष्ठ पर डाउनलोड लिंक:

```text
[wcpos_receipt link_text="Download your invoice"]
```

PDF के बजाय स्क्रीन पर दिखाई देने वाली (प्रिंट करने योग्य) रसीद का लिंक:

```text
[wcpos_receipt format="html" link_text="View your receipt"]
```

किसी विशिष्ट टेम्पलेट के साथ रेंडर करें — उदाहरण के लिए, अपनी 80mm कैश काउंटर रसीद के बजाय A4 इनवॉइस टेम्पलेट:

```text
[wcpos_receipt template="123" link_text="Download invoice (PDF)"]
```

टेम्पलेट ID, **WP Admin → POS → टेम्पलेट्स** में टेम्पलेट की संपादन-स्क्रीन URL में दिया गया नंबर (`post=123`) है।

## ग्राहकों को कौन-सा टेम्पलेट मिलता है? {#template-choice}

स्टोरफ़्रंट डाउनलोड कॉन्फ़िगर किए गए रसीद टेम्पलेट का उपयोग करते हैं या, यदि कोई कॉन्फ़िगर नहीं है, तो पहले सक्रिय टेम्पलेट का। यह स्टोरफ़्रंट डिफ़ॉल्ट उस टेम्पलेट से स्वतंत्र है जिसे कैशियर कैश काउंटर ड्रॉपडाउन से चुनता है। यदि स्टोरफ़्रंट डिफ़ॉल्ट संकरी थर्मल-शैली की रसीद है, तो PDF भी संकरी होगी।

ऑनलाइन ऑर्डर के लिए पूर्ण-पृष्ठ दस्तावेज़ अधिक उपयुक्त हो सकता है। दो विकल्प हैं:

- स्टोरफ़्रंट डिफ़ॉल्ट के रूप में पूर्ण-पृष्ठ टेम्पलेट का उपयोग करें (गैलरी का **इनवॉइस** टेम्पलेट A4/Letter के लिए डिज़ाइन किया गया है), या
- अपने कैश काउंटर टेम्पलेट सक्रिय रखें और शॉर्टकोड के `template` एट्रिब्यूट का उपयोग करके स्टोरफ़्रंट को किसी भिन्न टेम्पलेट की ओर निर्देशित करें, या नीचे दिए गए [डेवलपर फ़िल्टर](#developers) से साइट-व्यापी रूप से ऐसा करें।

टेम्पलेट POS के समान [रसीद डेटा पेलोड](/receipts/receipt-data) से रेंडर होते हैं, इसलिए आपका लोगो, दुकान की पहचान, कर सेटिंग्स और अनुकूलन सभी लागू रहते हैं।

## सुरक्षा {#security}

रसीद लिंक में ऑर्डर की अद्वितीय कुंजी होती है — वही तंत्र जिसका उपयोग WooCommerce अपने ऑर्डर पुष्टिकरण पृष्ठों के लिए करता है — इसलिए बिना खाते के चेकआउट करने वाले अतिथि भी पुष्टिकरण पृष्ठ से इस लिंक का उपयोग कर सकते हैं। जिस किसी के पास लिंक नहीं है, उसे ऑर्डर के ग्राहक के रूप में लॉग इन होना होगा। स्पष्ट `order_id` शॉर्टकोड केवल उस ऑर्डर के लॉग-इन स्वामी के लिए ही रेंडर होते हैं।

:::note सुंदर परमालिंक आवश्यक
सभी WCPOS चेकआउट पृष्ठों की तरह, रसीद URL WordPress री-राइट नियमों का उपयोग करता है। **सादे** परमालिंक (सेटिंग्स → परमालिंक्स) का उपयोग करने वाली साइटों को किसी अन्य परमालिंक संरचना पर स्विच करना होगा।
:::

## डेवलपर्स के लिए {#developers}

तीन फ़िल्टर स्टोरफ़्रंट इंटरफ़ेस नियंत्रित करते हैं:

```php
// Hide the My Account Receipt button.
add_filter( 'woocommerce_pos_storefront_receipt_my_account_button', '__return_false' );

// Change which order statuses show the button (default: processing, completed, refunded).
add_filter( 'woocommerce_pos_storefront_receipt_order_statuses', function ( $statuses, $order ) {
	$statuses[] = 'on-hold';
	return $statuses;
}, 10, 2 );

// Use a different template for storefront PDF downloads only.
// Receives and returns the full template array (null when none is configured).
add_filter( 'woocommerce_pos_storefront_receipt_template', function ( $template, $order ) {
	return \WCPOS\WooCommercePOS\Templates::get_template( 123 );
}, 10, 2 );
```

PDF रूट स्वयं `/wcpos-checkout/wcpos-receipt/{order_id}?key={order_key}&format=pdf` है — वही ऑर्डर-कुंजी-नियंत्रित एंडपॉइंट जिसका उपयोग POS रसीदों के लिए करता है, जिसमें `format=pdf` ऑन-स्क्रीन दृश्य के बजाय रेंडर किया गया PDF लौटाता है।
