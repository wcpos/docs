जब आप एक बिक्री पूरी करने के लिए तैयार हों, तो चेकआउट मॉडल खोलने के लिए **चेकआउट** बटन पर क्लिक करें। यहाँ आप भुगतान प्रोसेस करते हैं और ऑर्डर को पूरा करते हैं।

{/* Screenshot: checkout-modal.png */}

## चेकआउट मॉडल अवलोकन

चेकआउट मॉडल में दिखाई देता है:

- **ऑर्डर संख्या** - WooCommerce ऑर्डर ID
- **भुगतान राशि** - कुल देय राशि
- **चेकआउट सेटिंग्स** बटन - प्रदर्शन समस्याओं का निवारण
- **कैशियर** - ऑर्डर प्रोसेस करने वाला व्यक्ति
- **ग्राहक** - इस ऑर्डर के लिए ग्राहक (क्लिक करने योग्य लिंक)
- **कूपन कोड** फील्ड - छूट कोड लागू करें
- **ऑर्डर सारांश** - उत्पाद, मात्राएँ, और कुल योग
- **भुगतान विधियाँ** - उपलब्ध भुगतान विकल्प
- **रद्द करें / भुगतान प्रोसेस करें** बटन

## भुगतान विधियाँ

### मुफ्त संस्करण में उपलब्ध

WCPOS के मुफ्त संस्करण में दो भुगतान गेटवे शामिल हैं:

- **नकद** - जमा राशि और चेंज कैलकुलेटर के साथ
- **कार्ड** - बाहरी कार्ड टर्मिनल के लिए

### अतिरिक्त गेटवे (प्रो)

:::info प्रो सुविधा
अतिरिक्त भुगतान गेटवे के लिए [WCPOS प्रो](/getting-started/pro-license) की आवश्यकता होती है।
:::

प्रो के साथ, आप सक्षम कर सकते हैं:

- **स्ट्राइप टर्मिनल** - स्ट्राइप कार्ड रीडर के साथ सीधा एकीकरण
- **सम अप टर्मिनल** - सम अप कार्ड रीडर के साथ एकीकरण
- **कस्टम गेटवे** - अपने भुगतान एकीकरण बनाएं

विवरण के लिए [भुगतान](/payment) देखें।

### भुगतान विधि चुनना

एक भुगतान विधि चुनने के लिए उस पर क्लिक करें। फॉर्म अपडेट होकर संबंधित फील्ड दिखाता है:

**नकद:**
- **जमा राशि** - ग्राहक द्वारा दी गई राशि दर्ज करें
- **चेंज** - स्वचालित रूप से वापस किए जाने वाले परिवर्तन की गणना

{/* Screenshot: checkout-cash-payment.png */}

**कार्ड:**
- अपने बाहरी कार्ड टर्मिनल पर भुगतान प्रोसेस करें
- पूरा करने के लिए भुगतान प्रोसेस करें पर क्लिक करें

## कूपन कोड

छूट लागू करने के लिए:

1. कूपन कोड फील्ड में दर्ज करें
2. **कूपन लागू करें** पर क्लिक करें
3. छूट ऑर्डर सारांश में दिखाई देती है

कूपन WooCommerce में कॉन्फ़िगर किए जाने चाहिए (`WP एडमिन > मार्केटिंग > कूपन`)।

## भुगतान प्रोसेस करना

1. एक भुगतान विधि चुनें
2. नकद के लिए, जमा राशि दर्ज करें
3. **भुगतान प्रोसेस करें** पर क्लिक करें
4. ऑर्डर पूरा हो जाता है और [रसीद](/pos/checkout/receipts) दिखाई जाती है

## चेकआउट सेटिंग्स (समस्या निवारण)

चेकआउट मॉडल WooCommerce ऑर्डर पे पेज प्रदर्शित करने के लिए iframe/वेबव्यू का उपयोग करता है। यह WooCommerce के मौजूदा भुगतान बुनियादी ढांचे का लाभ उठाता है, जिसका अर्थ है कि कोई भी भुगतान गेटवे जो WooCommerce के साथ काम करता है, वह POS में भी काम करना चाहिए।

हालांकि, थीम और प्लगइन स्क्रिप्ट कभी-कभी हस्तक्षेप कर सकती हैं। समस्या निवारण के लिए **चेकआउट सेटिंग्स** पर क्लिक करें:

{/* Screenshot: checkout-settings.png */}

### सभी शैलियाँ और स्क्रिप्ट निष्क्रिय करें

- **wp_head अक्षम करें** - WordPress हेडर से सभी स्क्रिप्ट/शैलियाँ हटाता है
- **wp_footer अक्षम करें** - WordPress फुटर से सभी स्क्रिप्ट/शैलियाँ हटाता है

### चयनित शैलियाँ अक्षम करें

प्रदर्शन समस्याएँ पैदा कर सकने वाली CSS को चुनिंदा रूप से अक्षम करें:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- आदि

### चयनित स्क्रिप्ट अक्षम करें

भुगतान गेटवे में हस्तक्षेप कर सकने वाले जावास्क्रिप्ट को चुनिंदा रूप से अक्षम करें:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- आदि

:::tip
यदि कोई भुगतान गेटवे सही ढंग से प्रदर्शित नहीं होता:
1. पहले थीम शैलियाँ अक्षम करने का प्रयास करें
2. फिर उन WooCommerce स्क्रिप्ट को अक्षम करने का प्रयास करें जिनकी आवश्यकता नहीं है
3. ध्यान रखें कि अपने भुगतान गेटवे के लिए आवश्यक स्क्रिप्ट को अक्षम न करें
:::

## रद्द करें

ऑर्डर पूरा किए बिना चेकआउट मॉडल बंद करने के लिए **रद्द करें** पर क्लिक करें। ऑर्डर एक खुली कार्ट के रूप में रहता है।

## संबंधित दस्तावेज़

- [रसीदें](/pos/checkout/receipts) - चेकआउट के बाद, रसीदें प्रिंट या ईमेल करें
- [भुगतान विधियाँ](/payment) - भुगतान गेटवे कॉन्फ़िगर करें
- [कस्टम गेटवे](/payment/custom-gateways) - कस्टम भुगतान एकीकरण बनाएं
