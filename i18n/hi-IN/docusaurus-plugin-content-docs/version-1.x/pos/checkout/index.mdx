जब आप बिक्री पूरा करने के लिए तैयार हों, तो चेकआउट मोडल खोलने के लिए **Checkout** बटन पर क्लिक करें। यहीं आप भुगतान संसाधित करते हैं और आदेश पूरा करते हैं।

{/* Screenshot: checkout-modal.png */}

## चेकआउट मोडल अवलोकन

चेकआउट मोडल में प्रदर्शित होता है:

- **आदेश संख्या** - WooCommerce आदेश ID
- **भुगतान करने की राशि** - कुल राशि जो देनी है
- **चेकआउट सेटिंग्स** बटन - डिस्प्ले समस्याओं को हल करें
- **कैशियर** - जो आदेश संसाधित कर रहा है
- **ग्राहक** - इस आदेश के लिए ग्राहक (क्लिक करने योग्य लिंक)
- **कूपन कोड** फ़ील्ड - छूट कोड लागू करें
- **आदेश सारांश** - उत्पाद, मात्राएँ, और कुल
- **भुगतान विधियाँ** - उपलब्ध भुगतान विकल्प
- **रद्द करें / भुगतान प्रक्रिया** बटन

## भुगतान विधियाँ

### मुफ्त संस्करण में उपलब्ध

WCPOS के मुफ्त संस्करण में दो भुगतान गेटवे शामिल हैं:

- **नकद** - बताई गई राशि और चालू कैलकुलेटर के साथ
- **कार्ड** - बाहरी कार्ड टर्मिनलों के लिए

### अतिरिक्त गेटवे (Pro)

:::info प्रो सुविधा
अतिरिक्त भुगतान गेटवे के लिए [WCPOS Pro](/getting-started/pro-license) की आवश्यकता होती है।
:::

Pro के साथ, आप सक्षम कर सकते हैं:

- **Stripe टर्मिनल** - Stripe कार्ड रीडर्स के साथ सीधे एकीकरण
- **SumUp टर्मिनल** - SumUp कार्ड रीडर्स के साथ एकीकरण
- **कस्टम गेटवे** - अपनी स्वयं की भुगतान एकीकरण बनाएं

गेटवेज़ कॉन्फ़िगर करने के विवरण के लिए [Payment](/payment) देखें।

### भुगतान विधि चुनना

एक भुगतान विधि पर क्लिक करें ताकि आप इसे चुन सकें। फ़ॉर्म प्रासंगिक फ़ील्ड दिखाने के लिए अपडेट होता है:

**नकद:**
- **जमा की गई राशि** - वह राशि दर्ज करें जो ग्राहक आपको देता है
- **बापसी** - स्वचालित रूप से गणना की गई बापसी

{/* Screenshot: checkout-cash-payment.png */}

**कार्ड:**
- अपने बाहरी कार्ड टर्मिनल पर भुगतान प्रक्रिया करें
- पूरा करने के लिए प्रक्रिया भुगतान पर क्लिक करें

## कूपन कोड

छूट लागू करने के लिए:

1. फ़ील्ड में कूपन कोड दर्ज करें
2. **कूपन लागू करें** पर क्लिक करें
3. छूट आदेश सारांश में दिखाई देती है

कूपन को WooCommerce में कॉन्फ़िगर किया जाना चाहिए (`WP Admin > Marketing > Coupons`)।

## भुगतान प्रक्रिया करना

1. एक भुगतान विधि चुनें
2. नकद के लिए, जमा की गई राशि दर्ज करें
3. **भुगतान प्रक्रिया करें** पर क्लिक करें
4. आदेश पूरा हो जाता है और [रसीद](/pos/checkout/receipts) दिखाई देती है

## चेकआउट सेटिंग्स (समस्या समाधान)

चेकआउट मोडल WooCommerce आदेश भुगतान पृष्ठ दिखाने के लिए एक iframe/webview का उपयोग करता है। यह WooCommerce की मौजूदा भुगतान अवसंरचना का लाभ उठाता है, जिसका अर्थ है कि कोई भी भुगतान गेटवे जो WooCommerce के साथ काम करता है, उसे POS में काम करना चाहिए।

हालांकि, थीम और प्लगइन स्क्रिप्ट कभी-कभी हस्तक्षेप कर सकती हैं। समस्या हल करने के लिए **चेकआउट सेटिंग्स** पर क्लिक करें:

{/* Screenshot: checkout-settings.png */}

### सभी स्टाइल और स्क्रिप्ट्स को अक्षम करें

- **wp_head निष्क्रिय करें** - WordPress शीर्षक से सभी स्क्रिप्ट्स/शैली हटाता है
- **wp_footer निष्क्रिय करें** - WordPress फूटर से सभी स्क्रिप्ट्स/शैली हटाता है

### चयनित स्टाइल्स को अक्षम करें

उन CSS को चुनिंदा रूप से अक्षम करें जो डिस्प्ले समस्याएँ पैदा कर सकती हैं:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- आदि।

### चयनित स्क्रिप्ट्स को अक्षम करें

चुनिंदा रूप से उन JavaScript को अक्षम करें जो भुगतान गेटवे में हस्तक्षेप कर सकती हैं:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- आदि।

:::tip
यदि कोई भुगतान गेटवे सही तरीके से प्रदर्शित नहीं होता है:
1. पहले थीम की स्टाइल को निष्क्रिय करने का प्रयास करें
2. फिर उन WooCommerce स्क्रिप्ट्स को निष्क्रिय करने का प्रयास करें जो आवश्यक नहीं हैं
3. अपने भुगतान गेटवे द्वारा आवश्यक स्क्रिप्ट्स को निष्क्रिय न करने के लिए सावधान रहें
:::

## रद्द करें

आदेश पूरा किए बिना चेकआउट मोडल बंद करने के लिए **रद्द करें** पर क्लिक करें। आदेश एक खुले कार्ट के रूप में रहता है।

## संबंधित दस्तावेज़

- [रसीदें](/pos/checkout/receipts) - चेकआउट के बाद, रसीदें प्रिंट या ईमेल करें
- [भुगतान विधियाँ](/payment) - भुगतान गेटवे कॉन्फ़िगर करें
- [कस्टम गेटवे](/payment/custom-gateways) - कस्टम भुगतान एकीकरण बनाएं
