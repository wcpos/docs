जब आप बिक्री पूरा करने के लिए तैयार हों, तो चेकआउट मोड खोलने के लिए **Checkout** बटन पर क्लिक करें। यहाँ आप भुगतान प्रक्रिया पूरी करते हैं और आदेश को पूरा करते हैं।

{/* Screenshot: checkout-modal.png */}

## चेकआउट मोड का अवलोकन

चेकआउट मोड दिखाता है:

- **आदेश संख्या** - WooCommerce आदेश आईडी
- **भुगतान करने की राशि** - कुल राशि जो देय है
- **चेकआउट सेटिंग्स** बटन - डिस्प्ले समस्या का समाधान करें
- **कैशियर** - जो आदेश प्रसंस्करण कर रहा है
- **ग्राहक** - इस आदेश के लिए ग्राहक (क्लिक करने योग्य लिंक)
- **कूपन कोड** क्षेत्र - छूट कोड लागू करें
- **आदेश सारांश** - उत्पाद, मात्राएँ और कुल
- **भुगतान विधियाँ** - उपलब्ध भुगतान विकल्प
- **रद्द करें / भुगतान प्रक्रिया करें** बटन

## भुगतान विधियाँ

### मुफ्त संस्करण में उपलब्ध

WCPOS के मुफ्त संस्करण में दो भुगतान गेटवे शामिल हैं:

- **नकद** - राशि दिखाई गई और परिवर्तन कैल्क्युलेटर के साथ
- **कार्ड** - बाहरी कार्ड टर्मिनलों के लिए

### अतिरिक्त गेटवे (प्रो)

:::info प्रो फ़ीचर
अतिरिक्त भुगतान गेटवे की आवश्यकता [WCPOS Pro](/getting-started/pro-license) है।
:::

प्रो के साथ, आप सक्षम कर सकते हैं:

- **Stripe टर्मिनल** - Stripe कार्ड रीडर्स के साथ सीधे एकीकरण
- **SumUp टर्मिनल** - SumUp कार्ड रीडर्स के साथ एकीकरण
- **कस्टम गेटवे** - अपने स्वयं के भुगतान एकीकरण बनाएं

गेटवे कॉन्फ़िगर करने के विवरण के लिए [भुगतान](/payment) देखें।

### भुगतान विधि का चयन

किसी भुगतान विधि पर क्लिक करें ताकि इसे चुना जा सके। फॉर्म प्रासंगिक क्षेत्रों के साथ अपडेट होता है:

**नकद:**
- **राशि दिखाई गई** - उस राशि को दर्ज करें जो ग्राहक आपको देता है
- **बदलाव** - लौटाने के लिए स्वतः गणना की गई बदलाव

{/* Screenshot: checkout-cash-payment.png */}

**कार्ड:**
- अपने बाहरी कार्ड टर्मिनल पर भुगतान प्रक्रिया करें
- पूरा करने के लिए प्रक्रिया भुगतान पर क्लिक करें

## कूपन कोड

छूट लागू करने के लिए:

1. क्षेत्र में कूपन कोड दर्ज करें
2. **कूपन लागू करें** पर क्लिक करें
3. छूट आदेश सारांश में दिखाई देती है

कूपन WooCommerce में कॉन्फ़िगर किए जाने चाहिए (`WP Admin > Marketing > Coupons`)।

## भुगतान प्रक्रिया

1. एक भुगतान विधि का चयन करें
2. नकद के लिए, राशि दिखाई गई दर्ज करें
3. **भुगतान प्रक्रिया करें** पर क्लिक करें
4. आदेश पूरा हो जाता है और [रसीद](/pos/checkout/receipts) दिखाई देती है

## चेकआउट सेटिंग्स (समस्या निवारण)

चेकआउट मोड WooCommerce आदेश भुगतान पृष्ठ को प्रदर्शित करने के लिए एक iframe/webview का उपयोग करता है। यह WooCommerce के मौजूदा भुगतान बुनियामूलक संरचना का लाभ उठाता है, जिसका अर्थ है कि कोई भी भुगतान गेटवे जो WooCommerce के साथ काम करता है, वह POS में काम करेगा।

हालाँकि, थीम और प्लगइन स्क्रिप्ट कभी-कभी हस्तक्षेप कर सकती हैं। समस्या निवारण के लिए **चेकआउट सेटिंग्स** पर क्लिक करें:

### सभी शैलियों और स्क्रिप्ट्स को डिसेबल करें

- **wp_head को डिसेबल करें** - WordPress हैडर से सभी स्क्रिप्ट/शैलियाँ हटा देता है
- **wp_footer को डिसेबल करें** - WordPress फ़ुटर से सभी स्क्रिप्ट/शैलियाँ हटा देता है

### चयनित शैलियों को डिसेबल करें

डिस्प्ले समस्याएँ उत्पन्न कर सकती हैं ऐसी CSS को चयनात्मक रूप से डिसेबल करें:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- आदि।

### चयनित स्क्रिप्ट्स को डिसेबल करें

भुगतान गेटवे के साथ हस्तक्षेप कर सकता है ऐसी JavaScript को चयनात्मक रूप से डिसेबल करें:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- आदि।

:::tip
यदि एक भुगतान गेटवे सही ढंग से प्रदर्शित नहीं होता है:
1. सबसे पहले थीम शैलियों को डिसेबल करने का प्रयास करें
2. फिर WooCommerce स्क्रिप्ट्स को डिसेबल करने का प्रयास करें जिनकी आवश्यकता नहीं है
3. अपने भुगतान गेटवे द्वारा आवश्यक स्क्रिप्ट्स को डिसेबल करने में सावधानी बरतें
:::

## रद्द करें

आदेश को पूरा किए बिना चेकआउट मोड बंद करने के लिए **रद्द करें** पर क्लिक करें। आदेश एक खुले कार्ट के रूप में बना रहता है।

- [भुगतान विधियाँ](/payment) - भुगतान गेटवे कॉन्फ़िगर करें
