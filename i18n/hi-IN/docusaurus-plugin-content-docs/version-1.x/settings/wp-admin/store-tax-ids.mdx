स्टोर कर ID वे व्यावसायिक कर पहचानकर्ता हैं जिन्हें WCPOS आपकी POS रसीदों और चालानों पर प्रिंट करता है — आपका VAT नंबर, ABN, GSTIN, EIN, या कोई स्थानीय कंपनी-रजिस्ट्री नंबर।

WooCommerce के पास इनके लिए कोई अंतर्निहित फ़ील्ड नहीं है, और कई क्षेत्रों को एक से अधिक की आवश्यकता होती है। एक जर्मन व्यवसाय एक USt-IdNr., एक Steuernummer, **और** एक HRB नंबर प्रिंट कर सकता है; एक इतालवी व्यवसाय एक Codice Fiscale और एक Partita IVA दोनों प्रिंट करता है; कई कंपनियाँ एक स्थानीय पंजीकरण नंबर के साथ एक VAT ID दिखाती हैं। स्टोर कर ID फ़ील्ड आपको जितनी आवश्यकता हो उतनी प्रविष्टियाँ जोड़ने देता है।

आप इसे **WP Admin > POS > Settings > General** पर, **Store tax IDs** अनुभाग में पाएँगे।

{/* Screenshot: wp-admin-store-tax-ids.png */}

## एक कर ID जोड़ना {#adding-a-tax-id}

1. **WP Admin > POS > Settings > General** पर जाएँ और **Store tax IDs** तक स्क्रॉल करें।
2. **Add tax ID** पर क्लिक करें।
3. एक **Type** चुनें, **Value** दर्ज करें, और वैकल्पिक रूप से एक **Country** और एक कस्टम **Label** सेट करें।
4. प्रत्येक पहचानकर्ता के लिए दोहराएँ जिसे आपको प्रिंट करने की आवश्यकता है।
5. एक पंक्ति हटाने के लिए **Remove** का उपयोग करें।

:::info पंक्तियाँ स्वचालित रूप से सहेजी जाती हैं
इस अनुभाग के लिए कोई Save बटन नहीं है। प्रत्येक पंक्ति तब सहेजी जाती है जब **Value** फ़ील्ड फ़ोकस खो देता है — इसे कमिट करने के लिए फ़ील्ड से बाहर क्लिक करें या टैब करें।
:::

### पहली पंक्ति आपके स्टोर देश का उपयोग करती है {#the-first-row-uses-your-store-country}

जब आप अपना पहला कर ID जोड़ते हैं, तो WCPOS आपके WooCommerce आधार देश के साथ **Country** पूर्व-भरता है और मिलान करने वाला **Type** चुनता है — इसलिए एक व्यवसाय जिसे केवल एकल, स्थानीय कर ID की आवश्यकता है, उसे कुछ भी मैन्युअल रूप से चुनने की आवश्यकता नहीं है। मौजूदा पंक्तियाँ कभी नहीं बदली जातीं।

जैसे ही आप संपादित करते हैं Type और country समन्वय में रहते हैं: **Type** बदलने से **Country** उस प्रकार के मूल देश में अपडेट हो जाता है, और **Country** बदलने से **Type** उस देश के सामान्य पहचानकर्ता में बदल जाता है।

### उपलब्ध प्रकार {#available-types}

Type सूची सामान्य ग्राहक-सामना करने वाले पहचानकर्ताओं के साथ-साथ कंपनी-रजिस्ट्री नंबरों को कवर करती है जिन्हें व्यवसाय रसीदों पर प्रिंट करते हैं:

| क्षेत्र | प्रकार |
|--------|-------|
| यूरोपीय संघ | EU VAT |
| यूनाइटेड किंगडम | GB VAT, GB Company number |
| जर्मनी | USt-IdNr., Steuernummer, HRB |
| फ़्रांस | SIRET, SIREN |
| अन्य यूरोप | Netherlands KVK, Italy Codice Fiscale, Italy Partita IVA, Spain NIF, Switzerland UID |
| अमेरिका | US EIN, Canada GST/HST, Brazil CPF, Brazil CNPJ, Argentina CUIT |
| एशिया और मध्य पूर्व | India GSTIN, Saudi Arabia VAT |
| ऑस्ट्रेलिया | ABN |
| सामान्य | Other |

**Value** फ़ील्ड प्लेसहोल्डर टेक्स्ट के रूप में एक प्रारूप उदाहरण दिखाता है (उदाहरण के लिए US EIN के लिए `12-3456789`)। यह केवल एक संकेत है — WCPOS उन मानों को अस्वीकार नहीं करता जो अपेक्षित आकार से मेल नहीं खाते, इसलिए असामान्य लेकिन मान्य पहचानकर्ता फिर भी सहेजे जाते हैं।

## रसीदों पर क्या प्रिंट होता है {#what-prints-on-receipts}

आपके द्वारा जोड़ा गया प्रत्येक कर ID रसीदों और चालानों पर, क्रम में, प्रत्येक अपनी अलग पंक्ति पर प्रिंट होता है। WCPOS प्रकार के लिए एक स्थानीयकृत लेबल का उपयोग करता है ("VAT ID", "ABN", "GSTIN", इत्यादि) — या आपका कस्टम **Label** यदि आप एक सेट करते हैं। यदि दो पंक्तियों का मान समान है, तो डुप्लिकेट स्वचालित रूप से हटा दिया जाता है।

सभी बंडल किए गए रसीद टेम्पलेट पहले से ही पूरी सूची रेंडर करते हैं। यदि आप एक **कस्टम टेम्पलेट** बनाए रखते हैं जो अभी भी एकल `store.tax_id` मान को संदर्भित करता है, तो यह काम करता रहेगा लेकिन केवल पहली प्रविष्टि प्रिंट करेगा। इसके बजाय इसे सूची पर लूप करने के लिए स्विच करें:

```html
{{#store.tax_ids}}{{label}}: {{value}}{{/store.tax_ids}}
```

## प्रति-स्टोर कर ID (Pro) {#per-store-tax-ids-pro}

:::info Pro सुविधा
[WCPOS Pro](/getting-started/pro-license) के साथ आप पूरी साइट के लिए एक बार के बजाय **प्रति स्टोर** कर ID सेट कर सकते हैं — उपयोगी जब प्रत्येक स्थान का अपना VAT/EIN/GSTIN हो, या जब एक श्रृंखला विभिन्न पहचानकर्ताओं वाले क्षेत्रों में संचालित होती हो।
:::

प्रति-स्टोर कर ID **WP Admin > POS > Stores > Edit Store** पर, साइडबार के **Tax IDs** अनुभाग में सेट किए जाते हैं। जब तक किसी स्टोर की अपनी सूची खाली है, यह यहाँ General टैब पर कॉन्फ़िगर की गई साइट-व्यापी स्टोर कर ID पर वापस आ जाती है। जैसे ही आप किसी स्टोर में एक प्रविष्टि जोड़ते हैं, वह स्टोर केवल अपनी सूची प्रिंट करता है।
