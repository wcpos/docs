रसीद टेम्पलेट संपादक आपको अपने POS सिस्टम से प्रिंट की गई रसीदों की उपस्थिति और सामग्री को अनुकूलित करने की अनुमति देता है। आप टेम्पलेट संपादक तक पहुँचने के लिए नेविगेट कर सकते हैं: `WP Admin > POS > Templates`

## अवलोकन

टेम्पलेट सिस्टम दो प्रकार के टेम्पलेट प्रदान करता है:

- **डिफ़ॉल्ट टेम्पलेट**: आपके प्लगइन और थीम फ़ाइलों से स्वचालित रूप से पाए जाने वाले। इन्हें हटाया नहीं जा सकता लेकिन इन्हें कस्टम संस्करण बनाने के लिए कॉपी किया जा सकता है।
- **कस्टम टेम्पलेट**: उपयोगकर्ता द्वारा निर्मित टेम्पलेट जिन्हें आप पूरी तरह से अनुकूलित, संपादित और प्रबंधित कर सकते हैं।

## टेम्पलेट संपादक तक पहुँच

1. अपने WordPress प्रशासन डैशबोर्ड पर नेविगेट करें
2. **POS > Templates** पर जाएं
3. आप उपलब्ध टेम्पलेट्स की एक सूची देखेंगे जो दो वर्गों में व्यवस्थित हैं:
   - **डिफ़ॉल्ट टेम्पलेट** (प्लगइन/थीम फ़ाइलों से)
   - **कस्टम टेम्पलेट** (आपकी कस्टम रचनाएँ)

## डिफ़ॉल्ट टेम्पलेट

सिस्टम तीन स्रोतों से रसीद टेम्पलेट का स्वतः पता लगाता है:

### टेम्पलेट स्रोत

| स्रोत          | विवरण                                          | स्थिति                               |
| -------------- | ---------------------------------------------- | ------------------------------------ |
| **प्लगइन**    | WooCommerce POS के साथ शामिल मूल रसीद टेम्पलेट  | हमेशा उपलब्ध                         |
| **प्रो प्लगइन**| WooCommerce POS प्रो से संवर्धित टेम्पलेट   | सक्रिय लाइसेंस के साथ उपलब्ध        |
| **थीम**       | आपकी सक्रिय थीम से कस्टम टेम्पलेट           | यदि थीम में टेम्पलेट शामिल है तो उपलब्ध |

### टेम्पलेट क्रियाएँ

प्रत्येक डिफ़ॉल्ट टेम्पलेट के लिए, आप:

- **पूर्वावलोकन**: नमूना डेटा के साथ टेम्पलेट कैसा दिखता है देखें
- **कॉपी**: एक कस्टम कॉपी बनाएं जिसे आप संपादित कर सकें
- **सक्रिय करें**: अपने POS के लिए सक्रिय टेम्पलेट के रूप में सेट करें

## कस्टम टेम्पलेट बनाना

### विधि 1: डिफ़ॉल्ट टेम्पलेट से कॉपी करें

1. **डिफ़ॉल्ट टेम्पलेट** खंड में, उस टेम्पलेट को खोजें जिसे आप अनुकूलित करना चाहते हैं
2. **कॉपी** बटन पर क्लिक करें
3. अपने कस्टम टेम्पलेट के लिए एक नाम दर्ज करें
4. **कॉपी टेम्पलेट** पर क्लिक करें
5. नए बनाए गए कस्टम टेम्पलेट को संपादित करें

### विधि 2: शुरुआत से बनाएँ

1. पृष्ठ के शीर्ष पर **नया टेम्पलेट जोड़ें** पर क्लिक करें
2. एक टेम्पलेट नाम दर्ज करें
3. टेम्पलेट प्रकार के रूप में **रसीद** चुनें
4. संपादक में अपने टेम्पलेट को कोड करना शुरू करें

## टेम्पलेट संपादक की विशेषताएँ

टेम्पलेट संपादक प्रदान करता है:

### कोड संपादक

- **सिंटैक्स हाइलाइटिंग**: बेहतर कोड पठनीयता के लिए PHP सिंटैक्स हाइलाइटिंग
- **लाइन संख्या**: आसान नेविगेशन और डिबगिंग
- **पूर्ण-स्क्रीन संपादन**: विकर्षण-मुक्त संपादन वातावरण

### टेम्पलेट सेटिंग्स पैनल

- **भाषा**: उचित सिंटैक्स हाइलाइटिंग के लिए PHP पर सेट करें
- **टेम्पलेट प्रकार**: रसीद या रिपोर्ट टेम्पलेट के बीच चुनें
- **स्थिति**: प्रकाशन/मसौदा स्थिति प्रबंधन
- **दृश्यता**: टेम्पलेट की दृश्यता नियंत्रित करें

### टेम्पलेट क्रियाएँ

- **पूर्वावलोकन**: देखें कि आपका टेम्पलेट नमूना डेटा के साथ कैसे दिखता है
- **सक्रिय के रूप में सेट करें**: इस टेम्पलेट को आपके POS के लिए डिफ़ॉल्ट बनाएं
- **अपडेट करें**: आपके परिवर्तनों को सहेजें
- **कचरे में ले जाएं**: कस्टम टेम्पलेट हटाएं

## टेम्पलेट संरचना

रसीद के टेम्पलेट PHP फ़ाइलें होती हैं जो HTML आउटपुट उत्पन्न करती हैं। यहाँ मूल संरचना है:

```php
<?php
/**
 * Custom Receipt Template
 */

if ( ! defined( 'ABSPATH' ) ) {
    exit; // Exit if accessed directly
}
?>
<!DOCTYPE html>
<html <?php language_attributes(); ?>>
<head>
    <meta charset="<?php bloginfo( 'charset' ); ?>">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <style>
        /* Your custom CSS styles */
        body { font-family: sans-serif; font-size: 14px; }
        .receipt-header { text-align: center; margin-bottom: 20px; }
        /* Add more styles as needed */
    </style>
</head>
<body>
    <!-- Your receipt content -->
    <div class="receipt-header">
        <h1><?php echo get_bloginfo( 'name' ); ?></h1>
        <p><?php echo get_bloginfo( 'description' ); ?></p>
    </div>

    <!-- Order details, items, totals, etc. -->

</body>
</html>
```

## उपलब्ध टेम्पलेट متغیر

कस्टम टेम्पलेट बनाते समय, आपके पास विभिन्न WordPress और WooCommerce कार्यों और डेटा तक पहुँच होती है:

### स्टोर जानकारी

- `get_bloginfo( 'name' )` - स्टोर का नाम
- `get_bloginfo( 'description' )` - स्टोर टैगलाइन
- `get_bloginfo( 'url' )` - स्टोर URL

### आदेश डेटा

वर्तमान आदेश डेटा टेम्पलेट को प्रस्तुत करते समय मानक WooCommerce आदेश कार्यों के माध्यम से उपलब्ध है।

### WCPOS कार्य

आप POS से संबंधित डेटा और कार्यक्षमता तक पहुँचने के लिए WCPOS-विशिष्ट कार्यों (जो `wcpos_` से प्रारंभ होते हैं) का उपयोग कर सकते हैं।

## सर्वोत्तम प्रथाएं

### टेम्पलेट विकास

1. **कॉपी से शुरू करें**: हमेशा डिफ़ॉल्ट टेम्पलेट की कॉपी करके शुरू करें, न कि शून्य से
2. **पूरी तरह से परीक्षण करें**: सक्रिय करने से पहले अपने टेम्पलेट का परीक्षण करने के लिए पूर्वावलोकन फ़ंक्शन का उपयोग करें
3. **बैकअप रखें**: अपने कस्टम टेम्पलेट का निर्यात करें या अपने कोड की प्रतियाँ रखें
4. **मोबाइल-फ्रेंडली**: सुनिश्चित करें कि आपके टेम्पलेट विभिन्न स्क्रीन आकारों में अच्छी तरह से काम करते हैं

### कोड गुणवत्ता

1. **HTML मान्य करें**: सुनिश्चित करें कि आपका HTML अच्छी तरह से बना हो
2. **आउटपुट को सुरक्षित करें**: सुरक्षा के लिए WordPress.escape फ़ंक्शंस का उपयोग करें
3. **त्रुटि प्रबंधन**: अपने PHP कोड में उचित त्रुटि जांच शामिल करें
4. **टिप्पणियाँ**: भविष्य संदर्भ के लिए अपने अनुकूलन को दस्तावेज करें

### प्रदर्शन

1. **CSS ऑप्टिमाइज़ करें**: शैलियों को न्यूनतम और कुशल रखें
2. **PHP को कम करें**: टेम्पलेट में जटिल तर्क से बचें
3. **प्रिंटिंग का परीक्षण करें**: सुनिश्चित करें कि टेम्पलेट आपके रसीद प्रिंटर के साथ ठीक से काम करते हैं

## समस्या निवारण

### सामान्य समस्याएं

**टेम्पलेट में परिवर्तन नहीं दिखाना**

- सुनिश्चित करें कि आपने अपने परिवर्तन सहेजने के लिए "अपडेट" बटन पर क्लिक किया है
- जांचें कि टेम्पलेट "सक्रिय" के रूप में सेट है
- किसी भी कैशिंग प्लगइन को साफ़ करें

**सिंटैक्स त्रुटियाँ**

- समस्याओं की पहचान के लिए सिंटैक्स हाइलाइटिंग का उपयोग करें
- चूक गए सेमीकोलन, ब्रेस या उद्धरणों की जांच करें
- अपने PHP सिंटैक्स को मान्य करें

**शैली संबंधी समस्याएं**

- पूर्वावलोकन मोड में अपने CSS का परीक्षण करें
- सुनिश्चित करें कि शैलियाँ रसीद प्रिंटर के साथ संगत हैं
- प्रिंट-फ्रेंडली डिज़ाइन सिद्धांतों को ध्यान में रखें

**टेम्पलेट उपलब्ध नहीं है**

- जांचें कि टेम्पलेट प्रकार "रसीद" के रूप में सेट है
- सुनिश्चित करें कि टेम्पलेट की स्थिति "प्रकाशित" है
- सुनिश्चित करें कि आपके पास उचित अनुमतियाँ हैं

### सहायता प्राप्त करना

यदि आप टेम्पलेट अनुकूलन में समस्याएँ का सामना करते हैं:

1. त्रुटि संदेशों के लिए टेम्पलेट पूर्वावलोकन की जाँच करें
2. PHP त्रुटियों के लिए WordPress डिबग लॉग की समीक्षा करें
3. पहले डिफ़ॉल्ट टेम्पलेट की कॉपी के साथ परीक्षण करें
4. **AI सहायता**: आधुनिक AI उपकरण जैसे [Gemini](https://gemini.google.com), ChatGPT, और Claude सरल WordPress टेम्पलेट्स में मदद करने में उत्कृष्ट हैं। आप अपने टेम्पलेट कोड को पेस्ट कर सकते हैं और जो आप हासिल करना चाहते हैं उसे वर्णन कर सकते हैं।
5. उन्नत अनुकूलन के लिए WooCommerce POS दस्तावेज़ों का संदर्भ लें

## उन्नत अनुकूलन

उन उपयोगकर्ताओं के लिए जो रसीद टेम्पलेट्स पर अधिक नियंत्रण की आवश्यकता होती है:

### थीम एकीकरण

आप अपने विषय में एक कस्टम `receipt.php` फ़ाइल रखकर डिफ़ॉल्ट टेम्पलेट को ओवरराइड कर सकते हैं:

```
your-theme/woocommerce-pos/receipt.php
```

### हुक और फ़िल्टर

WooCommerce POS विभिन्न हुक और फ़िल्टर प्रदान करता है टेम्पलेट अनुकूलन के लिए। उपलब्ध विकल्पों के लिए डेवलपर दस्तावेज़ों का संदर्भ लें।

### कस्टम फ़ील्ड

आप उचित WordPress और WooCommerce फ़ंक्शंस का उपयोग करके अपनी रसीद टेम्पलेट्स में कस्टम ऑर्डर फ़ील्ड, उत्पाद विशेषताएँ, और अन्य मेटाडेटा शामिल कर सकते हैं।
