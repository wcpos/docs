रसीद टेम्पलेट संपादक आपको अपने POS सिस्टम से प्रिंट की गई रसीदों की उपस्थिति और सामग्री को कस्टमाइज करने की अनुमति देता है। आप टेम्पलेट संपादक तक पहुंचने के लिए यहां नेविगेट कर सकते हैं: `WP Admin > POS > Templates`

## अवलोकन

टेम्पलेट सिस्टम दो प्रकार के टेम्पलेट प्रदान करता है:

- **डिफ़ॉल्ट टेम्पलेट्स**: आपके प्लगइन और थीम फ़ाइलों से स्वचालित रूप से पता लगाया गया। इन्हें हटाया नहीं जा सकता, लेकिन कस्टम संस्करण बनाने के लिए कॉपी किया जा सकता है।
- **कस्टम टेम्पलेट्स**: उपयोगकर्ता द्वारा बनाए गए टेम्पलेट जिनका आप पूरी तरह से कस्टमाइज, संपादित और प्रबंधित कर सकते हैं।

## टेम्पलेट संपादक तक पहुंचना

1. अपने वर्डप्रेस एडमिन डैशबोर्ड पर नेविगेट करें
2. **POS > Templates** पर जाएं
3. आपको उपलब्ध टेम्पलेट्स की एक सूची दिखाई देगी जिसे दो भागों में वर्गीकृत किया गया है:
   - **डिफ़ॉल्ट टेम्पलेट्स** (प्लगइन/थीम फ़ाइलों से)
   - **कस्टम टेम्पलेट्स** (आपकी कस्टम रचनाएँ)

## डिफ़ॉल्ट टेम्पलेट्स

सिस्टम रसीद टेम्पलेट को तीन स्रोतों से स्वचालित रूप से पहचानता है:

### टेम्पलेट स्रोत

| स्रोत           | विवरण                                                | स्थिति                                |
| --------------- | --------------------------------------------------- | ------------------------------------- |
| **प्लगइन**     | WooCommerce POS के साथ शामिल मुख्य रसीद टेम्पलेट  | हमेशा उपलब्ध                          |
| **प्रो प्लगइन** | WooCommerce POS प्रो से उन्नत टेम्पलेट             | सक्रिय लाइसेंस के साथ उपलब्ध         |
| **थीम**        | आपके सक्रिय थीम से कस्टम टेम्पलेट                  | यदि थीम में टेम्पलेट शामिल हैं तो उपलब्ध |

### टेम्पलेट क्रियाएँ

प्रत्येक डिफ़ॉल्ट टेम्पलेट के लिए, आप:

- **पूर्वावलोकन**: नमूना डेटा के साथ टेम्पलेट कैसे दिखता है, देखें
- **कॉपी**: एक कस्टम प्रति बनाएँ जिसे आप संपादित कर सकें
- **सक्रिय करें**: अपने POS के लिए सक्रिय टेम्पलेट के रूप में सेट करें

## कस्टम टेम्पलेट्स बनाना

### विधि 1: डिफ़ॉल्ट टेम्पलेट से कॉपी करें

1. **डिफ़ॉल्ट टेम्पलेट्स** अनुभाग में, उस टेम्पलेट को ढूंढें जिसे आप कस्टमाइज़ करना चाहते हैं
2. **कॉपी** बटन पर क्लिक करें
3. अपने कस्टम टेम्पलेट का नाम दर्ज करें
4. **कॉपी टेम्पलेट** पर क्लिक करें
5. बनाए गए नए कस्टम टेम्पलेट को संपादित करें

### विधि 2: खरोंच से बनाएँ

1. पृष्ठ के शीर्ष पर **नया टेम्पलेट जोड़ें** पर क्लिक करें
2. एक टेम्पलेट नाम दर्ज करें
3. टेम्पलेट प्रकार के रूप में **रसीद** का चयन करें
4. संपादक में अपने टेम्पलेट को कोड करना शुरू करें

## टेम्पलेट संपादक सुविधाएँ

टेम्पलेट संपादक प्रदान करता है:

### कोड संपादक

- **सिंटैक्स हाइलाइटिंग**: बेहतर कोड पठनीयता के लिए PHP सिंटैक्स हाइलाइटिंग
- **लाइन नंबर**: आसान नेविगेशन और डिबगिंग
- **फुल-स्क्रीन संपादन**: ध्यान भंग न करने वाला संपादन वातावरण

### टेम्पलेट सेटिंग्स पैनल

- **भाषा**: उचित सिंटैक्स हाइलाइटिंग के लिए PHP पर सेट करें
- **टेम्पलेट प्रकार**: रसीद या रिपोर्ट टेम्पलेट के बीच चुनें
- **स्थिति**: प्रकाशन/खरिज़ स्थिति प्रबंधन
- **दृश्यता**: टेम्पलेट की दृश्यता नियंत्रित करें

### टेम्पलेट क्रियाएँ

- **पूर्वावलोकन**: देखें कि आपके टेम्पलेट नमूना डेटा के साथ कैसे रेंडर होता है
- **सक्रिय के रूप में सेट करें**: इस टेम्पलेट को आपके POS के लिए डिफ़ॉल्ट बनाएं
- **अपडेट करें**: अपने परिवर्तनों को सहेजें
- **कचरे में डालें**: कस्टम टेम्पलेट्स को हटाएं

## टेम्पलेट संरचना

रसीद टेम्पलेट PHP फ़ाइलें होती हैं जो HTML आउटपुट उत्पन्न करती हैं। यहाँ बुनियादी संरचना है:

```php
<?php
/**
 * Custom Receipt Template
 */

if ( ! defined( 'ABSPATH' ) ) {
    exit; // Exit if accessed directly
}
?>
<!DOCTYPE html>
<html <?php language_attributes(); ?>>
<head>
    <meta charset="<?php bloginfo( 'charset' ); ?>">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <style>
        /* Your custom CSS styles */
        body { font-family: sans-serif; font-size: 14px; }
        .receipt-header { text-align: center; margin-bottom: 20px; }
        /* Add more styles as needed */
    </style>
</head>
<body>
    <!-- Your receipt content -->
    <div class="receipt-header">
        <h1><?php echo get_bloginfo( 'name' ); ?></h1>
        <p><?php echo get_bloginfo( 'description' ); ?></p>
    </div>

    <!-- Order details, items, totals, etc. -->

</body>
</html>
```

## उपलब्ध टेम्पलेट वेरिएबल्स

कस्टम टेम्पलेट बनाते समय, आपको विभिन्न वर्डप्रेस और वूकॉमर्स फ़ंक्शंस और डेटा तक पहुँच मिलती है:

### स्टोर सूचना

- `get_bloginfo( 'name' )` - स्टोर का नाम
- `get_bloginfo( 'description' )` - स्टोर का टैगलाइन
- `get_bloginfo( 'url' )` - स्टोर का URL

### आदेश डेटा

वर्तमान आदेश डेटा मानक वूकॉमर्स आदेश फ़ंक्शंस के माध्यम से उपलब्ध है जब टेम्पलेट रेंडर किया जाता है।

### WCPOS फ़ंक्शंस

आप POS से संबंधित डेटा और कार्यक्षमता पहुंचने के लिए WCPOS-विशिष्ट फ़ंक्शन (जो `wcpos_` से प्रारंभ होते हैं) का उपयोग कर सकते हैं।

## सर्वोत्तम प्रथाएँ

### टेम्पलेट विकास

1. **कॉपी से शुरू करें**: हमेशा खरोंच से बनाने के बजाय एक डिफ़ॉल्ट टेम्पलेट कॉपी करके शुरू करें
2. **पूरी तरह से टेस्ट करें**: सक्रिय करने से पहले अपने टेम्पलेट का परीक्षण करने के लिए पूर्वावलोकन फ़ंक्शन का उपयोग करें
3. **बैकअप रखें**: अपने कस्टम टेम्पलेट्स का एक्सपोर्ट करें या अपने कोड की प्रतियाँ रखें
4. **मोबाइल-फ्रेंडली**: सुनिश्चित करें कि आपके टेम्पलेट विभिन्न स्क्रीन आकारों पर अच्छी तरह से काम करते हैं

### कोड गुणवत्ता

1. **HTML मान्यता**: सुनिश्चित करें कि आपका HTML अच्छी तरह से बना हुआ है
2. **आउटपुट को एस्केप करें**: सुरक्षा के लिए वर्डप्रेस एस्केपिंग फ़ंक्शंस का उपयोग करें
3. **त्रुटि हैंडलिंग**: अपने PHP कोड में उचित त्रुटि जाँच शामिल करें
4. **टिप्पणियाँ**: भविष्य के संदर्भ के लिए अपनी अनुकूलता का दस्तावेजीकरण करें

### प्रदर्शन

1. **CSS ऑप्टिमाइज़ करें**: स्टाइल को न्यूनतम और कुशल रखें
2. **PHP को कम करें**: टेम्पलेट में जटिल लॉजिक से बचें
3. **प्रिंटिंग का परीक्षण करें**: सत्यापित करें कि टेम्पलेट आपके रसीद प्रिंटर के साथ समानता से काम करते हैं

## समस्या निवारण

### सामान्य समस्याएं

**टेम्पलेट ने बदलाव नहीं दिखाए**

- सुनिश्चित करें कि आपने अपने परिवर्तनों को सहेजने के लिए "अपडेट" पर क्लिक किया है
- जांचें कि टेम्पलेट "सक्रिय" के रूप में सेट है
- किसी भी कैशिंग प्लगइन्स को साफ़ करें

**सिंटैक्स त्रुटियाँ**

- समस्याओं की पहचान के लिए सिंटैक्स हाइलाइटिंग का उपयोग करें
- गायब सेमीकोलोन, ब्रैकेट, या उद्धरणों के लिए चेक करें
- अपने PHP सिंटैक्स को मान्यता दें

**स्टाइलिंग समस्याएं**

- पूर्वावलोकन मोड में अपने CSS का परीक्षण करें
- सुनिश्चित करें कि शैलियाँ रसीद प्रिंटर्स के साथ संगत हैं
- प्रिंट-फ्रेंडली डिज़ाइन सिद्धांतों को ध्यान में रखें

**टेम्पलेट उपलब्ध नहीं है**

- जांचें कि टेम्पलेट प्रकार "रसीद" पर सेट है
- टेम्पलेट स्थिति "प्रकाशित" है या नहीं उसे सत्यापित करें
- सुनिश्चित करें कि आपके पास उचित अनुमतियाँ हैं

### सहायता प्राप्त करना

यदि आपको टेम्पलेट कस्टमाइजेशन के साथ समस्याएँ हैं:

1. त्रुटि संदेशों के लिए टेम्पलेट पूर्वावलोकन की जांच करें
2. PHP त्रुटियों के लिए वर्डप्रेस डेबग लॉग की समीक्षा करें
3. पहले डिफ़ॉल्ट टेम्पलेट की कॉपी के साथ परीक्षण करें
4. **एआई सहायता**: आधुनिक एआई उपकरण जैसे [Gemini](https://gemini.google.com), ChatGPT, और Claude साधारण वर्डप्रेस टेम्पलेट्स के साथ मदद करने में उत्कृष्ट हैं। आप अपने टेम्पलेट कोड को पेस्ट कर सकते हैं और जिस चीज़ को आप हासिल करना चाहते हैं उसे वर्णित कर सकते हैं।
5. उन्नत अनुकूलन के लिए WooCommerce POS दस्तावेज़ की सलाह लें

## उन्नत अनुकूलन

उन उन्नत उपयोगकर्ताओं के लिए जिन्हें रसीद टेम्पलेट्स पर अधिक नियंत्रण की आवश्यकता होती है:

### थीम इंटीग्रेशन

आप अपने थीम में एक कस्टम `receipt.php` फ़ाइल रखकर डिफ़ॉल्ट टेम्पलेट को ओवरराइड कर सकते हैं:

```
your-theme/woocommerce-pos/receipt.php
```

### हुक और फ़िल्टर

WooCommerce POS टेम्पलेट कस्टमाइजेशन के लिए विभिन्न हुक और फ़िल्टर प्रदान करता है। उपलब्ध विकल्पों के लिए डेवलपर दस्तावेज़ की सलाह लें।

### कस्टम फ़ील्ड

आप उचित वर्डप्रेस और वूकॉमर्स फ़ंक्शंस का उपयोग करके अपने रसीद टेम्पलेट्स में कस्टम आदेश फ़ील्ड, उत्पाद विशेषताएँ और अन्य मेटाडेटा जोड़ सकते हैं।
