रसीद टेम्पलेट संपादक आपको अपने POS सिस्टम से छपने वाली रसीदों की उपस्थिति और सामग्री को अनुकूलित करने की अनुमति देता है। आप टेम्पलेट संपादक तक पहुँचने के लिए निम्नलिखित पथ का उपयोग कर सकते हैं: `WP Admin > POS > Templates`

## अवलोकन {#overview}

टेम्पलेट सिस्टम दो प्रकार के टेम्पलेट्स प्रदान करता है:

- **डिफ़ॉल्ट टेम्पलेट्स**: आपके प्लगइन और थीम फ़ाइलों से स्वचालित रूप से पहचाने जाते हैं। इन्हें हटाया नहीं जा सकता लेकिन इन्हें कस्टम संस्करण बनाने के लिए कॉपी किया जा सकता है।
- **कस्टम टेम्पलेट्स**: उपयोगकर्ता द्वारा निर्मित टेम्पलेट्स जिन्हें आप पूरी तरह से अनुकूलित, संपादित और प्रबंधित कर सकते हैं।

## टेम्पलेट संपादक तक पहुँच {#accessing-the-template-editor}

1. अपने WordPress प्रशासन डैशबोर्ड पर जाएँ
2. **POS > Templates** पर जाएँ
3. आपको दो क्षेत्रों में व्यवस्थित उपलब्ध टेम्पलेट्स की एक सूची दिखाई देगी:
   - **डिफ़ॉल्ट टेम्पलेट्स** (प्लगइन/थीम फ़ाइलों से)
   - **कस्टम टेम्पलेट्स** (आपकी कस्टम रचनाएँ)

## डिफ़ॉल्ट टेम्पलेट्स {#default-templates}

सिस्टम तीन स्रोतों से रसीद टेम्पलेट्स का स्वत: पता लगाता है:

### टेम्पलेट स्रोत {#template-sources}

| स्रोत          | विवरण                                              | स्थिति                               |
| -------------- | -------------------------------------------------- | ------------------------------------ |
| **प्लगइन**    | WCPOS के साथ शामिल मूल रसीद टेम्पलेट             | हमेशा उपलब्ध                         |
| **प्रो प्लगइन**| WCPOS प्रो से बेहतर टेम्पलेट                       | सक्रिय लाइसेंस के साथ उपलब्ध         |
| **थीम**       | आपके सक्रिय थीम से कस्टम टेम्पलेट                 | उपलब्ध यदि थीम में टेम्पलेट शामिल हो |

### टेम्पलेट कार्य {#template-actions}

प्रत्येक डिफ़ॉल्ट टेम्पलेट के लिए, आप:

- **पूर्वावलोकन**: नमूना डेटा के साथ टेम्पलेट कैसा दिखता है देखें
- **कॉपी**: एक कस्टम कॉपी बनायें जिसे आप संपादित कर सकें
- **सक्रिय करें**: अपने POS के लिए सक्रिय टेम्पलेट के रूप में सेट करें

## कस्टम टेम्पलेट्स बनाना {#creating-custom-templates}

### विधि 1: डिफ़ॉल्ट टेम्पलेट से कॉपी करें {#method-1-copy-from-default-template}

1. **डिफ़ॉल्ट टेम्पलेट्स** अनुभाग में, उस टेम्पलेट को खोजें जिसे आप अनुकूलित करना चाहते हैं
2. **कॉपी** बटन पर क्लिक करें
3. अपने कस्टम टेम्पलेट का नाम दर्ज करें
4. **टेम्पलेट कॉपी करें** पर क्लिक करें
5. नए बनाए गए कस्टम टेम्पलेट को संपादित करें

### विधि 2: शून्य से बनाएं {#method-2-create-from-scratch}

1. पृष्ठ के शीर्ष पर **नया टेम्पलेट जोड़ें** पर क्लिक करें
2. एक टेम्पलेट नाम दर्ज करें
3. टेम्पलेट प्रकार के रूप में **रसीद** चुनें
4. संपादक में अपने टेम्पलेट को कोड करना शुरू करें

## टेम्पलेट संपादक विशेषताएँ {#template-editor-features}

टेम्पलेट संपादक प्रदान करता है:

### कोड संपादक {#code-editor}

- **वाक्यविन्यास हाइलाइटिंग**: बेहतर कोड पठनीयता के लिए PHP वाक्यविन्यास हाइलाइटिंग
- **लाइन नंबर**: आसान नेविगेशन और डिबगिंग
- **पूर्ण-स्क्रीन संपादन**: बिना अन्य ध्यान भटकाए संपादित करने का वातावरण

### टेम्पलेट सेटिंग्स पैनल {#template-settings-panel}

- **भाषा**: उचित वाक्यविन्यास हाइलाइटिंग के लिए PHP पर सेट करें
- **टेम्पलेट प्रकार**: रसीद या रिपोर्ट टेम्पलेट्स के बीच चयन करें
- **स्थिति**: प्रकाशन/ड्राफ्ट स्थिति प्रबंधन
- **दृश्यता**: टेम्पलेट दृश्यता को नियंत्रित करें

### टेम्पलेट कार्य {#editor-template-actions}

- **पूर्वावलोकन**: देखें कि आपका टेम्पलेट नमूना डेटा के साथ कैसा दिखाई देता है
- **सक्रिय के रूप में सेट करें**: इस टेम्पलेट को अपने POS के लिए डिफ़ॉल्ट बनाएं
- **अपडेट**: अपने परिवर्तनों को सहेजें
- **कचरे में स्थानांतरित करें**: कस्टम टेम्पलेट्स को हटाएं

## टेम्पलेट संरचना {#template-structure}

रसीद टेम्पलेट PHP फ़ाइलें हैं जो HTML आउटपुट उत्पन्न करती हैं। यहाँ मूल संरचना है:

```php
<?php
/**
 * Custom Receipt Template
 */

if ( ! defined( 'ABSPATH' ) ) {
    exit; // Exit if accessed directly
}
?>
<!DOCTYPE html>
<html <?php language_attributes(); ?>>
<head>
    <meta charset="<?php bloginfo( 'charset' ); ?>">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <style>
        /* Your custom CSS styles */
        body { font-family: sans-serif; font-size: 14px; }
        .receipt-header { text-align: center; margin-bottom: 20px; }
        /* Add more styles as needed */
    </style>
</head>
<body>
    <!-- Your receipt content -->
    <div class="receipt-header">
        <h1><?php echo get_bloginfo( 'name' ); ?></h1>
        <p><?php echo get_bloginfo( 'description' ); ?></p>
    </div>

    <!-- Order details, items, totals, etc. -->

</body>
</html>
```

## उपलब्ध टेम्पलेट वेरिएबल्स {#available-template-variables}

कस्टम टेम्पलेट्स बनाते समय, आपके पास विभिन्न WordPress और WooCommerce फ़ंक्शंस और डेटा तक पहुँच होती है:

### स्टोर जानकारी {#store-information}

- `get_bloginfo( 'name' )` - स्टोर का नाम
- `get_bloginfo( 'description' )` - स्टोर का टैगलाइन
- `get_bloginfo( 'url' )` - स्टोर का URL

### ऑर्डर डेटा {#order-data}

वर्तमान ऑर्डर डेटा स्टैंडर्ड WooCommerce ऑर्डर फ़ंक्शंस के माध्यम से उपलब्ध है जब टेम्पलेट प्रस्तुत किया जाता है।

### WCPOS फ़ंक्शंस {#wcpos-functions}

आप POS-संबंधित डेटा और कार्यक्षमताओं तक पहुँचने के लिए WCPOS-विशिष्ट फ़ंक्शंस (जो `wcpos_` से प्रारंभ होते हैं) का उपयोग कर सकते हैं।

## सर्वोत्तम प्रथाएँ {#best-practices}

### टेम्पलेट विकास {#template-development}

1. **कॉपी से शुरू करें**: हमेशा डिफ़ॉल्ट टेम्पलेट की कॉपी करके शुरू करें, न कि शून्य से
2. **गंभीरता से परीक्षण करें**: सक्रिय करने से पहले अपने टेम्पलेट का परीक्षण करने के लिए पूर्वावलोकन फ़ंक्शन का उपयोग करें
3. **बैकअप रखना**: अपने कस्टम टेम्पलेट्स का निर्यात करें या अपने कोड की प्रतियाँ रखें
4. **मोबाइल-फ्रेंडली**: सुनिश्चित करें कि आपके टेम्पलेट विभिन्न स्क्रीन आकारों पर अच्छी तरह से काम करें

### कोड गुणवत्ता {#code-quality}

1. **HTML मान्य करें**: सुनिश्चित करें कि आपका HTML सही है
2. **आउटपुट से सुरक्षा**: सुरक्षा के लिए WordPress एस्केपिंग फ़ंक्शंस का उपयोग करें
3. **त्रुटि प्रबंधन**: अपने PHP कोड में उचित त्रुटि जाँच शामिल करें
4. **टिप्पणियाँ**: भविष्य के संदर्भ के लिए अपने अनुकूलन का दस्तावेजीकरण करें

### प्रदर्शन {#performance}

1. **CSS का अनुकूलन करें**: शैलियों को न्यूनतम और प्रभावी रखें
2. **PHP को कम करें**: टेम्पलेट्स में जटिल तर्क से बचें
3. **प्रिंटिंग का परीक्षण करें**: सुनिश्चित करें कि टेम्पलेट्स आपके रसीद प्रिंटर के साथ अच्छी तरह से काम करते हैं

### सहायता प्राप्त करना {#getting-help}

यदि आपको टेम्पलेट अनुकूलन में समस्याएँ आती हैं:

1. त्रुटि संदेशों के लिए टेम्पलेट पूर्वावलोकन की जाँच करें
2. PHP त्रुटियों के लिए WordPress डिबग लॉग की समीक्षा करें
3. पहले डिफ़ॉल्ट टेम्पलेट की कॉपी के साथ परीक्षण करें
4. **AI सहायता**: आधुनिक AI उपकरण जैसे [Gemini](https://gemini.google.com), ChatGPT, और Claude इन सरल WordPress टेम्पलेट्स के साथ मदद करने में उत्कृष्ट हैं। आप अपने टेम्पलेट कोड को चिपका सकते हैं और जो आप हासिल करना चाहते हैं उसे वर्णित कर सकते हैं।
5. उन्नत अनुकूलन के लिए WCPOS दस्तावेज़ीकरण की परामर्श करें

## उन्नत अनुकूलन {#advanced-customisation}

उन उन्नत उपयोगकर्ताओं के लिए जिन्हें रसीद टेम्पलेट्स पर अधिक नियंत्रण की आवश्यकता है:

### थीम एकीकरण {#theme-integration}

आप अपने विषय में कस्टम `receipt.php` फ़ाइल रखकर डिफ़ॉल्ट टेम्पलेट को ओवरराइड कर सकते हैं:

```
your-theme/woocommerce-pos/receipt.php
```

### हुक्स और फ़िल्टर {#hooks-and-filters}

WCPOS टेम्पलेट अनुकूलन के लिए विभिन्न हुक्स और फ़िल्टर प्रदान करता है। उपलब्ध विकल्पों के लिए डेवलपर दस्तावेज़ीकरण की समीक्षा करें।

### कस्टम फ़ील्ड {#custom-fields}

आप अपने रसीद टेम्पलेट्स में कस्टम ऑर्डर फ़ील्ड, उत्पाद विशेषताएँ, और अन्य मेटाडेटा शामिल कर सकते हैं, उचित WordPress और WooCommerce फ़ंक्शंस का उपयोग करके।
