AR: # WCPOS ATUM Integration

AR: Integrates [ATUM Multi-Inventory](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/) with [WCPOS Pro](/getting-started/pro-license), enabling location-based inventory, pricing, and SKUs at the Point of Sale.

AR: ATUM Multi-Inventory lets you split a product's stock across multiple inventory locations — warehouses, retail stores, and so on. This plugin connects those ATUM locations to WCPOS Pro [stores](/stores) so each POS terminal sees the correct stock levels, prices, and SKUs for its physical location.

  </FeatureCard>
  <FeatureCard icon="store" title="AR: Flexible Pricing">
AR:     Choose pricing from WooCommerce defaults, WCPOS Pro per-store prices, or ATUM location-specific prices.
  </FeatureCard>
  <FeatureCard icon="barcode" title="AR: Location SKUs">
AR:     Optionally swap the product's main SKU for an ATUM location-specific SKU at the POS.
  </FeatureCard>

AR:     Orders deduct and restore stock at the correct ATUM location, with full audit trail in `atum_inventory_orders`.
  </FeatureCard>
  <FeatureCard icon="arrow-right" title="AR: Product Edit Write-Back">
AR:     POS edits to stock, price, and SKU sync back to the mapped ATUM inventory row for that location.
  </FeatureCard>

## مترجم: Installation {#installation}

<Steps>
  <StepCard number="1" title="AR: Install ATUM and Multi-Inventory">

AR: Install [ATUM Inventory Management](https://wordpress.org/plugins/atum-stock-manager-for-woocommerce/) and the [ATUM Multi-Inventory add-on](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/). Configure your inventory locations in ATUM.

  </StepCard>
  <StepCard number="2" title="AR: Install WCPOS ATUM Integration">

AR: Install from `WP Admin > POS > Settings > Extensions`, or download the latest release from the [GitHub releases page](https://github.com/wcpos/wcpos-atum/releases) and upload via `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="AR: Map stores to ATUM locations" isLast>

AR: Go to `POS > Stores`, edit a store, and configure the **ATUM Inventory** sidebar section. Pick the inventory location the store should use, choose a pricing source, and optionally enable SKU overrides.

  </StepCard>
</Steps>

## مترجم: Store Configuration {#store-configuration}

AR: The plugin adds an **ATUM Inventory** section to the WCPOS Pro store editor sidebar with three settings per store:

- AR: **Inventory Location** — which ATUM location this store pulls stock from.
- AR: **Pricing Source** — where product prices come from:
  - AR: *Default* — standard WooCommerce prices
  - AR: *WCPOS Pro* — per-store pricing set in WCPOS Pro
  - AR: *ATUM* — location-specific prices from the ATUM inventory
- AR: **SKU Override** — optionally use location-specific SKUs from ATUM instead of the product's main SKU.

## مترجم: POS Behavior {#pos-behavior}

AR: When a store has an ATUM location assigned, the product data served to the POS is automatically adjusted:

- AR: **Stock quantities** reflect the specific location's inventory, not the aggregate WooCommerce stock.
- AR: **Stock status** is recalculated based on the location's quantity.
- AR: **Prices** come from the configured pricing source.
- AR: **SKUs** are swapped to the ATUM location SKU if override is enabled.

AR: All adjustments happen transparently through the WCPOS REST API — no changes are needed on the POS app side. Product edits made from the POS are also written back to the mapped ATUM inventory row; see [Product Edit Write-Back](#product-edit-write-back) below.

## مترجم: Stock Management {#stock-management}

AR: For POS orders placed at stores with a mapped ATUM location, the plugin lets ATUM's native stock deduction flow handle the write — but steers it to the correct location:

1. AR: **REST payload injection.** When the POS creates or updates an order, the plugin injects a `mi_inventories` entry onto each line item so ATUM knows which location to draw from. Without this, ATUM would fall back to the main inventory.
2. AR: **Location-scoped inventory filter.** The plugin filters ATUM's candidate inventory list to only those linked to the store's mapped location term, ensuring the right one is picked on both reduction and restoration.

AR: ATUM itself performs the actual stock change on order and refund, writing rows to `atum_inventory_orders` with the real `order_id` — preserving ATUM's audit trail.

## مترجم: Product Edit Write-Back {#product-edit-write-back}

AR: When a cashier or manager edits a product or variation from the POS, the changes sync back to the mapped ATUM inventory row for that store's location — not just the main WooCommerce product. This keeps each location's stock, price, and SKU in sync with ATUM without manual updates in `WP Admin`.

AR: The write-back is triggered on WCPOS product and variation REST updates (`POST`, `PUT`, `PATCH` to `/wcpos/v1/products/...`) that include a `store_id`. The plugin looks up the store's mapped ATUM location and updates only the inventory row for that location — other locations are untouched.

### مترجم: What Syncs {#what-syncs}

AR: The write-back respects each store's configuration so ATUM data only changes when the store actually owns that data:

| AR: Field | AR: When it syncs | AR:
| --- | --- |
| AR: **Stock quantity** | AR: Always — every store with a mapped ATUM location keeps its location row in sync. | AR:
| AR: **Regular price / Sale price / Price** | AR: Only when the store's **Pricing Source** is set to *ATUM*. | AR:
| AR: **SKU** | AR: Only when **SKU Override** is enabled for the store. | AR:

AR: If the store uses *Default* or *WCPOS Pro* pricing, ATUM price fields are left alone so ATUM continues to serve as a reference price rather than the source of truth. The same applies to SKUs when override is off.

### مترجم: What Doesn't Trigger Write-Back {#what-doesnt-trigger-write-back}

- AR: Product creation (only updates write back — new products fall through to WooCommerce's normal save path).
- AR: Requests without a `store_id` — the POS has to tell the plugin which location to write to.
- AR: Stores without a mapped ATUM location.
- AR: Products without an existing ATUM inventory row for the store's location — the plugin will not create new inventory rows, only update existing ones.
