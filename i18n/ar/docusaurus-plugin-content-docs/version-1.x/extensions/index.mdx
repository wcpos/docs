AR: WCPOS supports extensions that add new functionality to your point of sale. The extension directory lets you browse available extensions, install them directly from the POS settings, and manage updates.

:::info Pro Feature
AR: Installing and managing extensions requires [WCPOS Pro](/getting-started/pro-license). The free version displays the extension catalog but disables install and activation controls.
:::

## مترجم: Browsing Extensions {#browsing-extensions}

AR: Open the extension directory from `POS Settings > Extensions` (also labeled **Plugins** in some versions).

AR: The directory displays a card grid of available extensions. Each card shows:

- AR: **Icon** (or a puzzle-piece fallback if the extension doesn't provide one)
- AR: **Name and version**
- AR: **Description**
- AR: **Category badge**
- AR: **Status** — active, inactive, update available, or not installed

### مترجم: Filtering and Search {#filtering-and-search}

AR: Use the **category pill buttons** at the top to filter extensions by category. You can also use the **search field** to find extensions by name, description, or tags.

## مترجم: Installing an Extension {#installing-an-extension}

1. AR: Open `POS Settings > Extensions`.
2. AR: Find the extension you want and click **Install**.
3. AR: The extension is downloaded and installed using the WordPress plugin installer.
4. AR: Once installed, click **Activate** to enable it.

AR: Behind the scenes, WCPOS uses WordPress's native `Plugin_Upgrader` to handle installation, so extensions follow the same process as any WordPress plugin.

## مترجم: Activating and Deactivating {#activating-and-deactivating}

AR: Each installed extension has **Activate** and **Deactivate** buttons on its card.

- AR: **Activate** enables the extension so it can run in the POS.
- AR: **Deactivate** disables it without uninstalling. The extension files remain on your server and can be reactivated at any time.

## مترجم: Updating Extensions {#updating-extensions}

AR: When a newer version of an installed extension is available, the card shows an **Update Available** badge and an **Update** button.

### مترجم: Auto-Updates {#auto-updates}

AR: Extensions installed from the directory have **auto-update enabled by default**. You can toggle auto-updates on or off per extension from its card in the directory. When auto-update is on, WordPress will apply new versions automatically, just like it does for plugins with auto-update enabled.

## مترجم: Troubleshooting {#troubleshooting}

### مترجم: "Requires Pro" Message on Buttons {#requires-pro-message-on-buttons}

AR: The install, activate, and update buttons are disabled in the free version of WCPOS. Upgrade to [WCPOS Pro](/getting-started/pro-license) to manage extensions.

### مترجم: Extension Fails to Install {#extension-fails-to-install}

- AR: Check that your WordPress server has write permissions to the `wp-content/plugins` directory.
- AR: Verify that your server can make outbound HTTPS requests (some hosts block external downloads).
- AR: Look at the error details in `WP Admin > POS > Support > Logs`.

### مترجم: Extension Not Appearing After Install {#extension-not-appearing-after-install}

- AR: Refresh the POS — the extension list is cached for up to 12 hours.
- AR: Confirm the extension is activated (installed but inactive extensions won't run).

### مترجم: Catalog Not Loading {#catalog-not-loading}

AR: The extension catalog is fetched from a remote source and cached locally for 12 hours. If the catalog doesn't load:

- AR: Check your server's internet connectivity.
- AR: Try again after the cache expires, or clear your server's transient cache.

## مترجم: For Developers {#for-developers}

### مترجم: Creating a POS Extension {#creating-a-pos-extension}

AR: A WCPOS extension is a standard WordPress plugin that integrates with the POS through WCPOS hooks and APIs. To create one:

1. AR: **Start with a WordPress plugin.** Your extension needs a standard plugin header and entry file, just like any WooCommerce or WordPress plugin.
2. AR: **Integrate with WCPOS.** Use the hooks and filters provided by WCPOS to add functionality to the POS interface or backend.
3. AR: **Host releases on GitHub.** The extension directory uses GitHub Releases to track versions and deliver updates.

### مترجم: Submitting to the Directory {#submitting-to-the-directory}

AR: The extension catalog is maintained in the [`wcpos/extensions`](https://github.com/wcpos/extensions) GitHub repository. To list your extension:

1. AR: Review the catalog format and metadata requirements in the repository's README.
2. AR: Open a pull request to add your extension's metadata to `catalog.json`.
3. AR: Once merged, your extension will appear in the directory for all WCPOS Pro users.

### مترجم: GitHub Release Conventions {#github-release-conventions}

AR: The update lifecycle relies on GitHub Releases:

- AR: **Tag versions** using semantic versioning (e.g., `v1.0.0`, `v1.2.3`).
- AR: **Attach the plugin zip** as a release asset — this is the file that gets downloaded when a user installs or updates.
- AR: **Publish the release** (not draft) so the directory can detect it.

AR: When you publish a new release, users with your extension installed will see the update available in their extension directory. If auto-update is enabled, it will be applied automatically.

AR: For full details on the catalog schema and submission process, see the [`wcpos/extensions`](https://github.com/wcpos/extensions) repository.
