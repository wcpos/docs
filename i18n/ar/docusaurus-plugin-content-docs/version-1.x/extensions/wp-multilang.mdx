AR: # WCPOS WP Multilang

AR: Integrates [WP Multilang](https://wordpress.org/plugins/wp-multilang/) with WCPOS so the POS only serves products for a single language. Unlike Polylang or WPML, WP Multilang stores all translations inside the same post — this plugin makes sure WCPOS reads the correct translation and doesn't duplicate entries in search results.

## مترجم: What It Does {#what-it-does}

- AR: Filters WCPOS product and variation REST responses to a single WP Multilang language.
- AR: Intercepts WCPOS **fast-sync** routes so only the active language's content reaches the client.
- AR: Free installs use the WP Multilang default language.
- AR: Pro installs can pin a language per store from the store editor.

## مترجم: Installation {#installation}

<Steps>
  <StepCard number="1" title="AR: Install WP Multilang">

AR: Install [WP Multilang](https://wordpress.org/plugins/wp-multilang/) and configure your site languages, with at least one language set as the default.

  </StepCard>
  <StepCard number="2" title="AR: Install WCPOS WP Multilang">

AR: Install from `WP Admin > POS > Settings > Extensions`, or download the latest release from the [GitHub releases page](https://github.com/wcpos/wcpos-wp-multilang/releases) and upload via `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="AR: (Pro) Pin a language per store" isLast>

AR: On WCPOS Pro, edit a store under `POS > Stores` and pick its language from the **Language** sidebar section. Leave at *Default* to use the WP Multilang default language.

  </StepCard>
</Steps>
