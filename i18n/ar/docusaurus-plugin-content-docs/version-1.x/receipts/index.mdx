يتضمن WCPOS نظام قوالب كاملاً للإيصالات، والفواتير، وإيصالات الهدايا، وقسائم التعبئة، وتذاكر المطبخ، وغير ذلك — وتتم إدارتها كلها من **WP Admin → POS → القوالب**.

## محركات القوالب {#template-engines}

يدعم WCPOS ثلاثة محركات للقوالب. يتم اختيار المحرك عند إنشاء قالب أو النقر على **استخدام القالب** في بطاقة المعرض، و**لا يمكن تغييره بعد ذلك** — ابدأ قالباً جديداً إذا أردت تبديل المحركات.

## معرض القوالب {#template-gallery}

يأتي المعرض مع قوالب جاهزة يمكن استخدامها كما هي أو تخصيصها. يمكن التصفية حسب **الفئة** (إيصال، فاتورة، إيصال هدية، تذكرة المطبخ، عرض سعر / أمر شراء)، و**التنسيق** (HTML / ESC/POS)، و**الاتجاه** (LTR / RTL)؛ مع المعاينة باستخدام البيانات الفعلية لمتجرك. يحتوي تذييل كل بطاقة على رابط **معاينة** وزر أساسي **استخدام القالب** — يؤدي النقر على استخدام القالب إلى إنشاء نسخة قابلة للتحرير في *قوالبك*.

| القالب | المحرك | الوصف |
|----------|--------|-------------|
| الإيصال القياسي | HTML | الإيصال الافتراضي — الشعار، هوية المتجر، العناصر، الإجماليات، الدفع |
| الإيصال القياسي (RTL) | HTML | النظير من اليمين إلى اليسار للغات العربية والعبرية والفارسية والأردية |
| بسيط / حديث | HTML | نفس الأساسيات الموجودة في القياسي، لكن بمساحة عمودية أقل |
| إيصال مفصل | HTML | فاتورة ضريبية كاملة — عمود SKU، سعر الوحدة، تفصيل الضريبة حسب كل معدل، العناوين |
| إيصال هدية | HTML | العناصر فقط، مع إخفاء الأسعار. يتضمن رسالة الهدية وسياسة الإرجاع |
| فاتورة | HTML | فاتورة بصفحة كاملة بحجم A4/Letter مع لوحة "كيفية الدفع" اختيارية للطلبات غير المدفوعة |
| قسيمة تعبئة | HTML | مستند مرافق للمستودع — العناصر + الكميات، عنوان الشحن، دون أسعار |
| عرض سعر / تقدير | HTML | مستند ما قبل البيع يتضمن الأسعار والشروط — دون قسم للدفع |
| إيصال ضيق | HTML | إيصال بخط ثابت العرض للورق الضيق أو الطابعات الحرارية الداعمة لـ HTML |
| إيصال حراري بسيط (58mm) | حراري | تخطيط حراري واضح بمقاس 58mm |
| إيصال حراري بسيط (80mm) | حراري | تخطيط حراري واضح بمقاس 80mm — الأكثر شيوعًا |
| إيصال حراري بسيط 80mm (RTL) | حراري | النظير RTL لمقاس 80mm. يحتاج إلى طابعة تتضمن صفحة رموز عربية (CP864 / Windows-1256) |
| إيصال حراري مفصل (58mm) | حراري | إيصال شامل بمقاس 58mm — العناوين، تفصيل الضريبة، المبالغ المستردة، المدفوعات، الشروط، الرمز الشريطي |
| إيصال حراري مفصل (80mm) | حراري | إيصال شامل بمقاس 80mm — العناوين، تفصيل الضريبة، المبالغ المستردة، المدفوعات، الشروط، الرمز الشريطي |
| تذكرة المطبخ | حراري | الأصناف فقط، خط كبير، بلا أسعار — لمحطات التحضير |

### قوالب مستندات الجهات الخارجية {#third-party-document-templates}

إذا كانت إضافة **PDF Invoices & Packing Slips for WooCommerce** من WP Overnight مفعّلة، يعرض WCPOS أيضًا **فاتورة (WP Overnight)** و**قسيمة التعبئة (WP Overnight)** في قائمة قوالب الإيصال. تعتمد قوالب HTML هذه، التي تُعرض من الخادم، على API المستندات الخاص بـ WP Overnight، وتعيد استخدام ترقيم الفواتير وقسائم التعبئة، والتخطيط، والعلامة التجارية، والحقول القانونية/الضريبية، وتخصيصات القوالب المضبوطة في تلك الإضافة.

لا تتوفر إلا أثناء تفعيل إضافة WP Overnight، وتتطلب اتصالًا بالخادم للعرض. استخدم قوالب HTML أو القوالب الحرارية المضمنة في WCPOS عندما تكون الطباعة دون اتصال مطلوبة.

### عرض الضريبة التكيفي {#adaptive-tax-display}

معظم القوالب المضمنة (قياسي، قياسي RTL، بسيط/حديث، ضيق، فاتورة، عرض سعر، ومتغيرات الحراري البسيط) **تتكيف تلقائيًا مع إعداد عرض الضريبة في WooCommerce**:

- ترى **المتاجر الشاملة للضريبة** (الاتحاد الأوروبي/المملكة المتحدة/أستراليا) الأسعار الإجمالية مع سطر "الضريبة مشمولة".
- ترى **المتاجر غير الشاملة للضريبة** (الولايات المتحدة/كندا) الأسعار الصافية مع إضافة الضريبة كسطر منفصل.
- يعرض الإجمالي النهائي دائمًا المبلغ الإجمالي الذي تم تحصيله فعليًا.

مجموعة **مفصل** هي فاتورة ضريبية رسمية وتفصل الضريبة دائمًا باستخدام أسطر غير شاملة للضريبة مع تفصيل صريح، بغض النظر عن إعداد المتجر.

## كيف يعمل {#how-it-works}

1. **أضف قالبًا** — انقر على **استخدام القالب** في بطاقة المعرض، أو أنشئ قالبك الخاص من الصفر.
2. **خصّصه** باستخدام المحرر داخل التطبيق مع معاينة مباشرة، أو الصقه في ChatGPT / Claude واطلب التغييرات التي تريدها.
3. **فعّله** — شغّل مفتاح تبديل **نشط** في *قوالبك*. يظهر كل قالب إيصال نشط في القائمة المنسدلة لشاشة الإيصالات عند نقطة البيع.
4. **اطبع أو اعرض** — تُفتح الإيصالات بعد إتمام الدفع مع خيارات للطباعة أو الإرسال بالبريد الإلكتروني أو العرض على الشاشة.

تُعرَض القوالب بناءً على [حمولة بيانات إيصال](/receipts/receipt-data) موحّدة تتضمن أقسامًا لمعلومات المتجر، وبنود الطلب، والإجماليات، والضرائب، والمدفوعات، والمبالغ المستردة، وغير ذلك. تتضمن كل حقول العملة صيغ `_display` منسّقة مسبقًا (مثل `$29.99` بدلًا من `29.99`).

## الميزات الاحترافية {#pro-features}

مع [WCPOS Pro](/getting-started/pro-license)، يمكن أن تكون لكل متجر تعيينات قوالبه الخاصة، وهويته البصرية (الشعار، والعنوان، وتفاصيل الاتصال)، وترتيب القوالب الخاص به — وهذا مفيد للعمليات متعددة المواقع حيث يطبع كل متجر ترويسته الخاصة.
