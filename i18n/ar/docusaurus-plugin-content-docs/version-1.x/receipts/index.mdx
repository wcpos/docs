يتضمن WCPOS نظام قوالب متكاملاً للإيصالات والفواتير وإيصالات الهدايا وقسائم التعبئة وتذاكر المطبخ والمزيد — تتم إدارتها جميعًا من **WP Admin → POS → القوالب**.

:::

## محركات القوالب {#template-engines}

يدعم WCPOS ثلاثة محركات للقوالب. يتم اختيار المحرك عند إنشاء قالب أو النقر على **استخدام القالب** في بطاقة المعرض، و**لا يمكن تغييره لاحقًا** — أنشئ قالبًا جديدًا إذا أردت التبديل بين المحركات.

## معرض القوالب {#template-gallery}

يأتي المعرض مع قوالب جاهزة يمكن استخدامها كما هي أو تخصيصها. يمكن التصفية حسب **الفئة** (إيصال، فاتورة، إيصال هدية، تذكرة مطبخ، عرض سعر / أمر شراء)، و**التنسيق** (HTML / ESC/POS)، و**الاتجاه** (LTR / RTL)؛ مع معاينة باستخدام بيانات المتجر الفعلية. يحتوي أسفل كل بطاقة على رابط **معاينة** وزر **استخدام القالب** الرئيسي — يؤدي النقر على استخدام القالب إلى إنشاء نسخة قابلة للتعديل في *قوالبك*.

| القالب | المحرك | الوصف |
|----------|--------|-------------|
| الإيصال القياسي | HTML | الإيصال الافتراضي — الشعار، هوية المتجر، العناصر، الإجماليات، الدفع |
| إيصال قياسي (من اليمين لليسار) | HTML | نسخة من اليمين لليسار للعربية والعبرية والفارسية والأردية |
| مختصر / عصري | HTML | نفس العناصر الأساسية للإيصال القياسي، بمساحة رأسية أقل |
| إيصال تفصيلي | HTML | فاتورة ضريبية كاملة — عمود SKU، سعر الوحدة، تفصيل الضريبة حسب المعدل، العناوين |
| إيصال هدية | HTML | الأصناف فقط، بدون أسعار. يتضمن رسالة الهدية وسياسة الإرجاع |
| فاتورة | HTML | فاتورة بصفحة كاملة بحجم A4/Letter مع لوحة اختيارية "طريقة الدفع" للطلبات غير المدفوعة |
| بطاقة تعبئة | HTML | مستند مرافق للمستودع — الأصناف + الكميات، عنوان الشحن، بدون أسعار |
| عرض سعر / تقدير | HTML | مستند ما قبل البيع يتضمن الأسعار والشروط — بدون قسم الدفع |
| إيصال ضيق | HTML | إيصال بخط ثابت العرض للورق الضيق أو الطابعات الحرارية التي تدعم HTML |
| إيصال حراري بسيط (58 مم) | حراري | تخطيط حراري نظيف بعرض 58 مم |
| إيصال حراري بسيط (80 مم) | حراري | تخطيط حراري نظيف بعرض 80 مم — الأكثر شيوعًا |
| إيصال حراري بسيط 80 مم (من اليمين لليسار) | حراري | نسخة من اليمين لليسار بعرض 80 مم. يتطلب طابعة تدعم صفحة ترميز عربية (CP864 / Windows-1256) |
| إيصال حراري تفصيلي (58 مم) | حراري | إيصال 58 مم شامل — العناوين، تفصيل الضريبة، المبالغ المستردة، المدفوعات، الشروط، الباركود |
| إيصال حراري مفصّل (80 مم) | حراري | إيصال شامل 80 مم — العناوين، تفصيل الضرائب، المرتجعات، المدفوعات، الشروط، الباركود |
| تذكرة المطبخ | حراري | الأصناف فقط، خط كبير، بدون أسعار — لمحطات التحضير |

### قوالب المستندات من جهات خارجية {#third-party-document-templates}

إذا كانت إضافة **PDF Invoices & Packing Slips for WooCommerce** من WP Overnight مفعّلة، يعرض WCPOS أيضًا **فاتورة (WP Overnight)** و**قسيمة تعبئة (WP Overnight)** في قائمة قوالب الإيصالات. تُفوِّض قوالب HTML المُعالَجة من الخادم إلى واجهة API للمستندات الخاصة بـ WP Overnight، مع إعادة استخدام ترقيم الفواتير وقسائم التعبئة والتخطيط والعلامة التجارية والحقول القانونية/الضريبية وتخصيصات القوالب المُعدَّة في تلك الإضافة.

تتوفر هذه القوالب فقط عندما تكون إضافة WP Overnight مفعّلة وتتطلب اتصالًا بالخادم للعرض. استخدم قوالب HTML أو القوالب الحرارية المُضمَّنة في WCPOS عند الحاجة إلى الطباعة دون اتصال.

### عرض الضرائب التكيّفي {#adaptive-tax-display}

تتكيّف معظم القوالب المُضمَّنة (Standard، Standard RTL، Minimal/Modern، Narrow، Invoice، Quote، متغيرات Thermal Simple) **تلقائيًا مع إعداد عرض الضرائب في WooCommerce**:

- تعرض **المتاجر الشاملة للضريبة** (EU/UK/AU) الأسعار الإجمالية مع سطر "الضريبة مشمولة".
- تعرض **المتاجر غير الشاملة للضريبة** (US/CA) الأسعار الصافية مع إضافة الضريبة كسطر منفصل.
- يعرض الإجمالي الكلي دائمًا المبلغ الإجمالي المُحصَّل فعليًا.

مجموعة **Detailed** هي فاتورة ضريبية رسمية وتُفصِّل الضريبة دائمًا بأسطر غير شاملة للضريبة مع تفصيل صريح، بغض النظر عن إعداد المتجر.

## آلية العمل {#how-it-works}

1. **إضافة قالب** — انقر على **استخدام القالب** في بطاقة المعرض، أو أنشئ قالبًا خاصًا بك من الصفر.
2. **تخصيصه** باستخدام المحرر المدمج في التطبيق مع معاينة مباشرة، أو الصقه في ChatGPT / Claude واطلب التعديلات التي تريدها.
3. **تفعيله** — قم بتبديل مفتاح **نشط** في *القوالب الخاصة بك*. يظهر كل قالب إيصال نشط في القائمة المنسدلة لشاشة الإيصالات عند نقطة البيع.
4. **الطباعة أو العرض** — تُفتح الإيصالات بعد إتمام الدفع مع خيارات للطباعة أو الإرسال بالبريد الإلكتروني أو العرض على الشاشة.

تُعرض القوالب وفقًا لـ[حمولة بيانات الإيصال](/receipts/receipt-data) الموحّدة التي تتضمن أقسامًا لمعلومات المتجر وبنود الطلب والإجماليات والضرائب والمدفوعات والمبالغ المستردة والمزيد. تتضمن جميع حقول العملات متغيرات `_display` مُنسّقة مسبقًا (مثل `$29.99` بدلاً من `29.99`).

## ميزات Pro {#pro-features}

مع [WCPOS Pro](/getting-started/pro-license)، يمكن لكل متجر أن يكون له تعيينات قوالب خاصة به وعلامة تجارية (شعار وعنوان وبيانات الاتصال) وترتيب قوالب مخصص — وهو مفيد للعمليات متعددة المواقع حيث يطبع كل فرع ترويسته الخاصة.
