لا تقتصر قوالب إيصالات WCPOS على نقطة البيع. يمكن لعملاء المتجر الإلكتروني عرض إيصال طلبهم أو فاتورتهم وتنزيله من حسابي أو صفحة تأكيد الطلب، باستخدام قالب مخصص من **WP Admin → POS → القوالب**.

:::info يتطلب WCPOS 1.9.11
تتوفر إمكانية الوصول إلى إيصالات المتجر الإلكتروني مع WCPOS 1.9.11. تعرض الإصدارات السابقة الإيصالات داخل POS فقط.
:::

:::note تتناول هذه الصفحة تنزيل العملاء لإيصالاتهم الخاصة
لا يحتاج أمناء الصندوق والموظفون إلى أي من ذلك — تعرض شاشة **الطلبات** في POS بالفعل كل طلبات WooCommerce، بما في ذلك الطلبات عبر الإنترنت، ويمكنها [عرض إيصالها أو طباعته أو إرساله عبر البريد الإلكتروني](/orders#receipt-reprinting) من قائمة النقاط الثلاث. تتناول هذه الصفحة الجانب الآخر: إتاحة تنزيل **العملاء** للإيصال بأنفسهم من موقعك الإلكتروني.
:::

## زر التنزيل في حسابي {#my-account}

بمجرد أن يصبح قالب الإيصال **نشطًا**، يرى العملاء المسجلون دخولهم زر **إيصال** بجوار الطلبات المؤهلة في **حسابي → الطلبات**. يؤدي النقر عليه إلى تنزيل ملف PDF (`receipt-1234.pdf`) باستخدام قالب الإيصال الافتراضي للمتجر الإلكتروني.

يظهر الزر افتراضيًا للطلبات ذات الحالات **قيد المعالجة** و**مكتمل** و**مسترد**، وهي الحالات التي تم فيها دفع الطلب. لا يظهر للطلبات المعلقة والملغاة والفاشلة.

لا توجد إعدادات مطلوبة: فعِّل قالبًا وسيصبح الزر متاحًا. وإذا لم ترغب في عرضه، يمكن للمطوّر [إيقافه باستخدام عامل تصفية](#developers).

## الرمز المختصر `[wcpos_receipt]` {#shortcode}

لإضافة رابط إيصال إلى صفحة تأكيد الطلب ("شكرًا لك") أو صفحة **حسابي → عرض الطلب**، استخدم:

```text
[wcpos_receipt]
```

من دون سمات، يكتشف الرمز المختصر الطلب الحالي تلقائيًا في أيٍّ من الصفحتين ويعرض رابط *تنزيل الإيصال (PDF)*. أما في الصفحات الأخرى فلا يعرض شيئًا. لا تعمل قيمة `order_id` الصريحة إلا للعميل المسجّل دخوله والمالك لذلك الطلب.

### السمات {#shortcode-attributes}

| السمة | الافتراضي | الوظيفة |
|---|---|---|
| `order_id` | *(يُكتشف تلقائيًا)* | رابط إلى طلب محدد. لا يعمل إلا للعملاء المسجّلين دخولهم الذين يملكون الطلب. |
| `link_text` | `Download receipt (PDF)` | نص الرابط. |
| `format` | `pdf` | ينزّل `pdf` ملف PDF؛ بينما يفتح `html` عرض الإيصال القابل للطباعة. |
| `template` | *(الإعداد الافتراضي للواجهة الأمامية)* | يعرض باستخدام معرّف قالب محدد بدلًا من الإعداد الافتراضي للواجهة الأمامية. |
| `class` | `wcpos-receipt-link` | فئة CSS للرابط. |
| `id` | *(لا شيء)* | معرّف HTML `id` للرابط. |

### أمثلة {#shortcode-examples}

رابط تنزيل في صفحة تأكيد الطلب:

```text
[wcpos_receipt link_text="Download your invoice"]
```

رابط للإيصال المعروض على الشاشة (القابل للطباعة) بدلاً من PDF:

```text
[wcpos_receipt format="html" link_text="View your receipt"]
```

العرض باستخدام قالب محدد — مثل قالب فاتورة A4 بدلاً من إيصال الصندوق بعرض 80mm:

```text
[wcpos_receipt template="123" link_text="Download invoice (PDF)"]
```

معرّف القالب هو الرقم الموجود في URL شاشة تحرير القالب (`post=123`) ضمن **WP Admin → POS → القوالب**.

## أي قالب يحصل عليه العملاء؟ {#template-choice}

تستخدم تنزيلات واجهة المتجر قالب الإيصال المُعدّ، أو أول قالب نشط عند عدم إعداد قالب. هذا الإعداد الافتراضي لواجهة المتجر مستقل عن القالب الذي يختاره أمين الصندوق من القائمة المنسدلة لنقطة البيع. إذا كان الإعداد الافتراضي لواجهة المتجر إيصالاً حرارياً ضيقاً، فسيكون ملف PDF ضيقاً أيضاً.

قد يكون المستند بصفحة كاملة مفضلاً للطلبات عبر الإنترنت. يوجد خياران:

- استخدم قالب صفحة كاملة باعتباره الإعداد الافتراضي لواجهة المتجر (قالب **الفاتورة** في المعرض مصمم لـ A4/Letter)، أو
- أبقِ قوالب نقطة البيع نشطة ووجّه واجهة المتجر إلى قالب مختلف باستخدام السمة `template` الخاصة بالرمز المختصر، أو على مستوى الموقع باستخدام [عامل التصفية للمطورين](#developers) أدناه.

تُعرض القوالب من [حمولة بيانات الإيصال](/receipts/receipt-data) نفسها التي يستخدمها POS، لذا يُنقل شعارك وهوية متجرك وإعدادات الضرائب وتخصيصاتك جميعها.

## الأمان {#security}

تتضمن روابط الإيصالات مفتاح الطلب الفريد — وهي الآلية نفسها التي يستخدمها WooCommerce لصفحات تأكيد الطلب — لذا يظل بإمكان الضيوف الذين أكملوا الشراء دون حساب استخدام الرابط من صفحة التأكيد. يجب على أي شخص لا يملك الرابط تسجيل الدخول بصفته عميل الطلب. لا تُعرض الرموز المختصرة التي تتضمن `order_id` صراحةً إلا لمالك ذلك الطلب الذي سجّل الدخول.

:::note يلزم استخدام الروابط الدائمة الجميلة
مثل جميع صفحات إتمام الشراء في WCPOS، يستخدم رابط URL للإيصال قواعد إعادة الكتابة في WordPress. يجب على المواقع التي تستخدم الروابط الدائمة **العادية** (الإعدادات → الروابط الدائمة) التبديل إلى أي بنية أخرى للروابط الدائمة.
:::

## للمطورين {#developers}

تتحكم ثلاثة عوامل تصفية في واجهات واجهة المتجر:

```php
// Hide the My Account Receipt button.
add_filter( 'woocommerce_pos_storefront_receipt_my_account_button', '__return_false' );

// Change which order statuses show the button (default: processing, completed, refunded).
add_filter( 'woocommerce_pos_storefront_receipt_order_statuses', function ( $statuses, $order ) {
	$statuses[] = 'on-hold';
	return $statuses;
}, 10, 2 );

// Use a different template for storefront PDF downloads only.
// Receives and returns the full template array (null when none is configured).
add_filter( 'woocommerce_pos_storefront_receipt_template', function ( $template, $order ) {
	return \WCPOS\WooCommercePOS\Templates::get_template( 123 );
}, 10, 2 );
```

مسار PDF نفسه هو `/wcpos-checkout/wcpos-receipt/{order_id}?key={order_key}&format=pdf` — وهو نقطة النهاية نفسها المحمية بمفتاح الطلب التي يستخدمها POS للإيصالات، حيث يعيد `format=pdf` ملف PDF المُنشأ بدلاً من العرض على الشاشة.
