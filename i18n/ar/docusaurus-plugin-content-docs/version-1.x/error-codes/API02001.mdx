# API02001: بيانات اعتماد غير صالحة

## ما يعنيه هذا

اسم المستخدم أو كلمة المرور التي أدخلتها غير صحيحة. رفض الخادم محاولة تسجيل الدخول لأن بيانات الاعتماد لا تتطابق مع أي حساب صالح.

## الأسباب الشائعة

- **خطأ مطبعي في اسم المستخدم أو كلمة المرور** — تحقق من مفتاح الأحرف الكبيرة أو المسافات الإضافية
- **الحساب الخطأ** — استخدام بيانات اعتماد لموقع مختلف
- **تغيير كلمة المرور مؤخرًا** — تم تحديث كلمة المرور لكن نظام نقطة البيع لا يزال يحتفظ بالقديمة
- **حذف الحساب** — حساب المستخدم لم يعد موجودًا

## تعيين خطأ الخادم

يتم تشغيل رمز الخطأ هذا عند إرجاع الخادم:

| رمز الخادم | المصدر |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | wtyczka JWT Authentication |

## كيفية الإصلاح

### 1. التحقق من بيانات الاعتماد

- تحقق مرة أخرى من اسم المستخدم (عادة البريد الإلكتروني أو اسم مستخدم WordPress)
- تأكد من صحة كلمة المرور
- حاول تسجيل الدخول مباشرة إلى WordPress للتأكيد

### 2. إعادة تعيين كلمة المرور

إذا نسيت كلمة المرور:
1. انتقل إلى صفحة تسجيل الدخول الخاصة بـ WordPress
2. انقر على "هل نسيت كلمة المرور؟"
3. أدخل بريدك الإلكتروني للحصول على رابط إعادة التعيين
4. قم بتحديث كلمة المرور
5. سجل الدخول إلى نقطة البيع باستخدام كلمة المرور الجديدة

### 3. التحقق من حساب المستخدم

في WordPress Admin:
- انتقل إلى المستخدمون → جميع المستخدمين
- تأكد من وجود الحساب
- تحقق من وجود صلاحيات وصول مناسبة لنقطة البيع

### 4. مسح بيانات الاعتماد المحفوظة

في نظام نقطة البيع:
- تسجيل الخروج بشكل كامل
- مسح أي معلومات تسجيل دخول محفوظة أو مخزنة مؤقتًا
- إعادة إدخال بيانات الاعتماد من جديد
