# API01004: فشل في حل DNS

## ما يعنيه ذلك

لم يتمكن نظام نقاط البيع من تحويل اسم نطاق موقعك (مثل: `yourstore.com`) إلى عنوان IP. يُعتبر DNS (نظام أسماء النطاقات) مثل دليل الهاتف الخاص بالإنترنت — إذا لم يتمكن من العثور على الإدخال، فلن يتمكن من إنشاء الاتصال.

## الأسباب الشائعة

- **خطأ مطبعي في اسم النطاق** — قد يحتوي عنوان URL على خطأ إملائي
- **مشاكل في خادم DNS** — خادم DNS الخاص بك لا يستجيب
- **انتهاء صلاحية النطاق** — قد يكون تسجيل النطاق قد انتهى
- **انتشار DNS** — التغييرات الأخيرة في DNS لم يتم انتشارها بعد
- **ذاكرة تخزين DNS المحلية** — معلومات DNS القديمة مخزنة على جهازك

## كيفية الإصلاح

### 1. تحقق من اسم النطاق

راجِع عنوان URL مرة أخرى للتحقق من الأخطاء المطبعية. حاول الوصول إلى الموقع في متصفح.

### 2. تحقق من حالة النطاق

استخدم خدمة WHOIS للتحقق من:
- أن النطاق لا يزال مسجلاً
- أنه لم ينتهِ
- أن سجلات DNS مُعدة بشكل صحيح

### 3. مسح ذاكرة تخزين DNS

**على جهازك:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. جرب DNS بديل

قم بالتبديل إلى خادم DNS عام مثل:
- Google: `8.8.8.8` و `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. انتظر انتشار DNS

إذا قمت مؤخرًا بتغيير إعدادات DNS، انتظر من 24 إلى 48 ساعة للانتشار الكامل.
