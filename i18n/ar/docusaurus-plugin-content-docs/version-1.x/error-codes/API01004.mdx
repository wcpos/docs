# API01004: فشل في حل اسم النطاق (DNS)

## ما يعنيه هذا

تعذر على نظام نقطة البيع تحويل اسم نطاق موقعك (مثل `yourstore.com`) إلى عنوان IP. يُعد نظام أسماء النطاقات (DNS) بمثابة دليل الإنترنت - إذا لم يتمكن من العثور على السجل، فلن يتمكن من إجراء الاتصال.

## الأسباب الشائعة

- **خطأ إملائي في اسم النطاق** — قد يحتوي عنوان URL على خطأ إملائي
- **مشاكل في خادم DNS** — خادم DNS الخاص بك لا يستجيب
- **انتهاء صلاحية النطاق** — قد تكون تسجيل النطاق قد انتهى
- **انتشار DNS** — التغييرات الحديثة في DNS لم تنتشر بعد
- **ذاكرة التخزين المؤقت المحلية لـ DNS** — معلومات DNS قديمة مخزنة مؤقتًا على جهازك

## كيفية الإصلاح

### 1. التحقق من اسم النطاق

تحقق مرتين من عنوان URL للتأكد من عدم وجود أخطاء إملائية. حاول الوصول إلى الموقع في متصفح.

### 2. التحقق من حالة النطاق

استخدم البحث WHOIS للتحقق من:
- تسجيل النطاق لا يزال ساريًا
- لم تنته صلاحيته
- تكوين سجلات DNS

### 3. مسح ذاكرة التخزين المؤقت لـ DNS

**على جهازك:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. تجربة DNS بديل

التبديل إلى خادم DNS عام مثل:
- Google: `8.8.8.8` و `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. الانتظار لانتشار DNS

إذا قمت بتغيير إعدادات DNS مؤخرًا، انتظر 24-48 ساعة للانتشار الكامل.
