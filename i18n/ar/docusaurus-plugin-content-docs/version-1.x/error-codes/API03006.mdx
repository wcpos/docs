# API03006: طريقة غير مدعومة {#api03006-unsupported-method}

## ما الذي يعنيه ذلك {#what-this-means}

طريقة HTTP المستخدمة (GET، POST، PUT، DELETE، إلخ) غير مدعومة لهذه النقطة النهاية. لا يقبل الخادم هذا النوع من الطلبات لهذا URL.

## الأسباب الشائعة {#common-causes}

- **عدم التوافق في الإصدار** — إصدارات POS والإضافة غير متوافقة
- **إزالة نقطة النهاية** — تم إهمال نقطة نهاية API
- **تكوين الخادم** — خادم الويب يحجب بعض طرق HTTP
- **تعارض الإضافات** — إضافة أخرى تعدل سلوك REST API

## كيفية الإصلاح {#how-to-fix}

### 1. تحديث كلا العنصرين {#1-update-both-components}

ضمان التوافق:
- تحديث إضافة ووردبريس WCPOS
- تحديث تطبيق POS
- التحقق من ملاحظات الإصدار للتغييرات الجذرية

### 2. التحقق من تكوين الخادم {#2-check-server-configuration}

بعض الخوادم تحجب طرق HTTP معينة:
- تأكد من السماح بطرق PUT و DELETE
- تحقق من ملف `.htaccess` لتقييدات الطرق
- مراجعة تكوين Nginx

### 3. التحقق من وصول REST API {#3-verify-rest-api-access}

اختبر REST API الخاص بووردبريس:
1. قم بزيارة `https://yoursite.com/wp-json/` في متصفح
2. يجب أن يعيد بيانات JSON
3. إذا لم يكن كذلك، فقد يتم تعطيل REST API أو حجبه

### 4. التحقق من تعارضات الإضافات {#4-check-for-plugin-conflicts}

قم بتعطيل الإضافات الأخرى مؤقتًا:
- قد تحجب إضافات الأمان الطرق
- قد تتسبب إضافات REST API أخرى في تعارضات
- قم بإعادة تشغيلها واحدة تلو الأخرى للعثور على المشكلة

### 5. مراجعة قيود الاستضافة {#5-review-hosting-restrictions}

بعض مقدمي الخدمة يقيدون طرق HTTP:
- اتصل بدعم الاستضافة
- اطلب منهم تمكين جميع الطرق القياسية
- اعتبر تغيير مقدم الخدمة إذا كانت القيود كثيرة
