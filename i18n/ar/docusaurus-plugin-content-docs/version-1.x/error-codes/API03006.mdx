# API03006: طريقة غير مدعومة

## ما يعنيه هذا

الطريقة HTTP المستخدمة (GET، POST، PUT، DELETE، إلخ) غير مدعومة لهذه النقطة النهائية. الخادم لا يقبل هذا النوع من الطلبات لعنوان URL هذا.

## الأسباب الشائعة

- **عدم توافق الإصدارات** — إصدارات POS والإضافة غير متوافقة
- **إزالة نقطة النهاية** — تم إهمال نقطة نهاية API
- **تهيئة الخادم** — حجب الخادم الويب لبعض طرق HTTP
- **تعارض الإضافات** — إضافة أخرى تعدل سلوك REST API

## كيفية الإصلاح

### 1. تحديث المكونات

تأكد من التوافق:
- قم بتحديث إضافة WCPOS لـ WordPress
- قم بتحديث تطبيق POS
- تحقق من ملاحظات الإصدار للتغييرات الجذرية

### 2. التحقق من تهيئة الخادم

بعض الخوادم تحظر طرق HTTP معينة:
- تأكد من السماح بطرق PUT و DELETE
- تحقق من `.htaccess` من القيود على الطرق
- راجع تهيئة Nginx

### 3. التأكد من وصول REST API

اختبر REST API لـ WordPress:
1. زر `https://yoursite.com/wp-json/` في متصفح
2. يجب أن يعيد بيانات JSON
3. إذا لم يحدث ذلك، فقد يكون REST API معطلاً أو محظوراً

### 4. التحقق من تعارض الإضافات

عطل الإضافات الأخرى مؤقتاً:
- إضافات الأمان قد تحظر الطرق
- إضافات REST API الأخرى قد تسبب تعارضات
- أعد التفعيل واحدة تلو الأخرى للعثور على المشكلة

### 5. مراجعة قيود الاستضافة

بعض المضيفين يقيدون طرق HTTP:
- تواصل مع دعم الاستضافة
- اطلب تمكين جميع الطرق القياسية
- فكر في تغيير المضيف إذا كانت القيود مفرطة
