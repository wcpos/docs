# API04005: محاولة استعادة JSON

## ما يعنيه هذا

أرسل الخادم استجابة تحتوي على محتوى JSON غير صالح، لكن نقطة البيع (POS) حاولت استعادة واستخراج البيانات الصالحة. هذه إشعار معلوماتي وليس خطأ حرجًا.

## ما حدث

اكتشفت نقطة البيع:
1. تحتوي الاستجابة على محتوى إضافي قبل أو بعد JSON
2. كانت البيانات الأساسية لـ JSON لا تزال قابلة للتحديد
3. تمت محاولة الاستعادة عن طريق استخراج الجزء الصالح من JSON

## الأسباب الشائعة

- **إشعارات PHP في المخرجات** — تحذيرات PHP مختلطة مع JSON
- **مخرجات التصحيح** — تم ترك التصحيح في وضع التطوير مفعلًا
- **إشعارات الإضافات** — الإضافات الأخرى التي تخرج إشعارات
- **مشاكل المسافات البيضاء** — مسافات بيضاء إضافية حول JSON

## كيفية الإصلاح

### 1. تعطيل عرض أخطاء PHP

في `wp-config.php`:
```php
define('WP_DEBUG_DISPLAY', false);
ini_set('display_errors', 0);
```

### 2. تفعيل تسجيل الأخطاء بدلاً من ذلك

الاحتفاظ بالأخطاء مسجلة للتصحيح:
```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
```

### 3. التحقق من وضع التصحيح للإضافات

بعض الإضافات لديها أوضاع تصحيح تخرج محتوى إضافي:
- مراجعة إعدادات الإضافات
- تعطيل أوضاع التطوير/التصحيح
- التحقق من خيارات التسجيل التفصيلية

### 4. مراجعة التغييرات الأخيرة

إذا بدأ هذا مؤخرًا:
- ما الذي تغير على الخادم الخاص بك؟
- هل تم تحديث الإضافات؟
- هل تم تعديل إعدادات PHP؟

## هل هذا خطير؟

على الرغم من أن نقطة البيع استعادت من هذه المشكلة، إلا أنها تشير إلى مشكلة في التكوين يجب إصلاحها. عملية الاستعادة:
- قد لا تعمل دائمًا
- تضيف عبئًا على المعالجة
- يمكن أن تخفي مشاكل أخرى

قم بإصلاح السبب الأساسي لضمان التشغيل الموثوق.
