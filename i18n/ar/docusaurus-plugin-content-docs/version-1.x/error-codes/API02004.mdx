# API02004: المستخدم غير مصرح له

## ما يعنيه هذا

أنت مسجل الدخول، لكن حساب المستخدم الخاص بك لا يمتلك الإذن لتنفيذ الإجراء المطلوب. هذه مشكلة تخص التفويض (ما يمكنك فعله) وليست مشكلة مصادقة (من أنت).

## الأسباب الشائعة

- **قيود دور المستخدم** — دورك في WordPress لا يتضمن الوصول إلى نقطة البيع
- **تعطيل الوصول لنقطة البيع** — لم يتم منح حسابك أذونات نقطة البيع
- **قيود الميزات** — بعض الميزات محدودة لأدوار محددة
- **قيود المتجر** — قد لا يكون لديك وصول لهذا المتجر بعينه

## تعيين خطأ الخادم

يتم تشغيل رمز الخطأ هذا عند إرجاع الخادم:

| كود الخادم | المصدر |
|-------------|--------|
| `rest_cannot_view` | WordPress REST API |
| `woocommerce_rest_cannot_view` | WooCommerce REST API |

## كيفية الإصلاح

### 1. التحقق من دور المستخدم

في WordPress Admin → المستخدمون:
1. ابحث عن حساب المستخدم الخاص بك
2. تحقق من الدور (مثل مدير المتجر، المسؤول)
3. تأكد من تضمين الدور لقدرات WooCommerce

### 2. تمكين الوصول لنقطة البيع

في WordPress Admin → WooCommerce → نقطة البيع → الوصول:
1. ابحث عن المستخدم أو الدور
2. قم بتمكين أذونات الوصول لنقطة البيع
3. احفظ التغييرات

### 3. طلب أذونات إضافية

تواصل مع مسؤول المتجر للقيام بما يلي:
- منح دورك الوصول لنقطة البيع
- تعيينك في دور به أذونات مناسبة
- تمكين الميزات المحددة التي تحتاجها

### 4. التحقق من تعيين المتجر

في حالة استخدام متاجر متعددة:
- تأكد من تعيينك للمتجر الصحيح
- تحقق من أذونات المتجر المحددة

## الأذونات المطلوبة

تتطلب الإجراءات المختلفة قدرات مختلفة:
- **عرض المنتجات**: وصول القراءة للمنتجات
- **إنشاء الطلبات**: قدرات إنشاء/تحرير الطلبات
- **إدارة العملاء**: قدرات إدارة العملاء
- **الوصول للتقارير**: قدرات عرض التقارير
