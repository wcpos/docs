# API05004: تعطيل WordPress REST API

## ما يعنيه هذا

تم تعطيل واجهة برمجة التطبيقات REST الخاصة بـ WordPress على موقعك. جميع وظائف WordPress الحديثة، بما في ذلك WooCommerce و WCPOS، تعتمد على هذه الواجهة.

## الأسباب الشائعة

- **البرنامج المساعد للأمان** — برنامج مساعد يحظر الوصول إلى REST API
- **قيود الاستضافة** — مُضيف موقعك عطل REST API
- **كود مخصص** — مظهر أو برنامج مساعد عطل الواجهة
- **قواعد جدار الحماية** — جدار حماية يحظر نقاط نهاية REST API

## كيفية الإصلاح

### 1. اختبار REST API

زيارة `https://yoursite.com/wp-json/` في المتصفح:
- يجب أن يعيد JSON مع المسارات المتاحة
- إذا واجهت خطأ أو لم يظهر شيء، فهو محظور

### 2. التحقق من البرامج المساعدة للأمان

البرامج المساعدة الشائعة التي تحظر REST API:

**Wordfence:**
- جدار الحماية → جميع خيارات جدار الحماية
- تعطيل خيار "تعطيل REST API"

**iThemes Security:**
- الأمان → الإعدادات → تعديلات WordPress
- تمكين REST API

**البرنامج المساعد لتعطيل REST API:**
- إلغاء تنشيط هذا البرنامج المساعد تماماً

### 3. التحقق من الكود المخصص

البحث في `functions.php` للمظهر أو البرامج المساعدة المخصصة عن:

```php
// هذا الكود يعطل REST API - قم بإزالته
add_filter('rest_authentication_errors', function($result) {
    return new WP_Error('rest_disabled', 'REST API disabled');
});
```

### 4. التحقق من .htaccess

إزالة أي قواعد تحظر `/wp-json/`:

```apache
# سيء - يحظر REST API
RewriteRule ^wp-json - [F,L]
```

### 5. التواصل مع مزود الاستضافة

بعض المُضيفين يحظرون REST API افتراضياً:
- طلب تمكينه
- السؤال عن أي قيود أمنية
- التحقق من توثيق الاستضافة

## لماذا تهم REST API

واجهة برمجة التطبيقات REST الخاصة بـ WordPress ضرورية لـ:
- تطبيقات الهاتف المحمول
- التكاملات مع أطراف ثالثة
- وظائف WooCommerce
- تشغيل WCPOS

تعطيلها يكسر العديد من الميزات.
