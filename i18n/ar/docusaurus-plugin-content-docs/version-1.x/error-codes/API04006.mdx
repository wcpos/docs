# API04006: المورد غير موجود {#api04006-resource-not-found}

## ما الذي يعنيه هذا {#what-this-means}

لم يتمكن الخادم من العثور على المورد المطلوب. يتوافق هذا عادةً مع رمز حالة HTTP 404، مما يشير إلى أن المنتج أو الطلب أو العميل أو المورد الآخر الذي تحاول الوصول إليه غير موجود.

## الأسباب الشائعة {#common-causes}

- **تم حذف السجل** — تم حذف المورد على الخادم
- **معرّف خاطئ** — يتم استخدام معرّف غير صحيح أو قديم
- **مشاكل التزامن** — تشير البيانات المحلية إلى مورد لم يعد موجودًا
- **تكوين عنوان URL خاطئ** — نقطة نهاية API غير صحيحة
- **مشاكل الروابط الثابتة** — تحتاج الروابط الثابتة في ووردبريس إلى التحديث

## تخطيط خطأ الخادم {#server-error-mapping}

يتم تفعيل هذا الرمز الخطأ عندما يُرجع الخادم:

| رمز الخادم | المصدر |
|-------------|--------|
| `rest_no_route` | ووردبريس REST API |
| HTTP 404 | أي استجابة خادم |

## كيفية الإصلاح {#how-to-fix}

### 1. تحقق مما إذا كان المورد موجودًا {#1-check-if-the-resource-exists}

تحقق مما إذا كان المورد لا يزال موجودًا على موقع ووكومرس الخاص بك:
- سجل الدخول إلى إدارة ووردبريس
- انتقل إلى القسم المعني (المنتجات، الطلبات، العملاء)
- ابحث عن العنصر حسب المعرّف أو الاسم

### 2. تحديث البيانات المحلية {#2-refresh-local-data}

إذا تم حذف المورد من جانب الخادم:
1. افتح إعدادات نقطة البيع 
2. انتقل إلى قسم البيانات المعني
3. قم بتحفيز مزامنة/تحديث لتحديث البيانات المحلية
4. يجب أن يتم إزالة العنصر المحذوف محليًا

### 3. تحقق من الروابط الثابتة في ووردبريس {#3-check-wordpress-permalinks}

إذا لم يتم العثور على عدة موارد:
1. انتقل إلى **الإعدادات → الروابط الثابتة** في إدارة ووردبريس
2. انقر على **حفظ التغييرات** (حتى بدون إجراء تغييرات)
3. هذا يقوم بتحديث بنية الرابط الثابت

### 4. تحقق من مسارات API {#4-verify-api-routes}

اختبر REST API مباشرة:
```
https://yoursite.com/wp-json/wc/v3/products
```
إذا أعاد هذا 404، فقد تكون هناك مشكلة في تكوين الخادم.

### 5. تحقق من تعارضات المكون الإضافي {#5-check-for-plugin-conflicts}

إذا كانت مسارات REST API مفقودة:
1. تأكد من أن ووكومرس نشط
2. تأكد من أن المكون الإضافي WCPOS نشط
3. قم بتعطيل المكونات الإضافية الأخرى بشكل مؤقت للاختبار
