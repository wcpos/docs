# API04006: المورد غير موجود

## ما يعنيه هذا

لم يتمكن الخادم من العثور على المورد المطلوب. وهذا يتوافق عادةً مع حالة HTTP 404، مما يشير إلى أن المنتج أو الطلب أو العميل أو المورد الآخر الذي تحاول الوصول إليه غير موجود.

## الأسباب الشائعة

- **حذف السجل** — تم حذف المورد على الخادم
- **معرف خاطئ** — يتم استخدام معرف غير صحيح أو قديم
- **مشاكل المزامنة** — المراجع المحلية للبيانات تشير إلى مورد لم يعد موجوداً
- **سوء تكوين URL** — نقطة النهاية للواجهة البرمجية غير صحيحة
- **مشاكل الرابط الدائم** — يجب تحديث روابط WordPress

## تعيين خطأ الخادم

يتم تشغيل رمز الخطأ هذا عند إرجاع الخادم:

| كود الخادم | المصدر |
|-------------|--------|
| `rest_no_route` | WordPress REST API |
| HTTP 404 | أي استجابة خادم |

## كيفية الإصلاح

### 1. التحقق من وجود المورد

تأكد من وجود المورد على موقع WooCommerce الخاص بك:
- تسجيل الدخول إلى لوحة التحكم في WordPress
- الانتقال إلى القسم ذي الصلة (المنتجات، الطلبات، العملاء)
- البحث عن العنصر حسب المعرف أو الاسم

### 2. تحديث البيانات المحلية

إذا تم حذف المورد من جانب الخادم:
1. افتح إعدادات POS
2. انتقل إلى القسم ذي الصلة للبيانات
3. تشغيل المزامنة/التحديث لتحديث البيانات المحلية
4. يجب إزالة العنصر المحذوف محلياً

### 3. التحقق من روابط WordPress الدائمة

إذا لم يتم العثور على موارد متعددة:
1. انتقل إلى **الإعدادات → الروابط الدائمة** في لوحة تحكم WordPress
2. انقر على **حفظ التغييرات** (حتى بدون إجراء تغييرات)
3. هذا يعيد تحديث هيكل الروابط الدائمة

### 4. التحقق من مسارات واجهة برمجة التطبيقات

اختبر واجهة برمجة التطبيقات REST مباشرةً:
```
https://yoursite.com/wp-json/wc/v3/products
```
إذا أرجع 404، قد يكون هناك مشكلة في تكوين الخادم.

### 5. التحقق من تعارضات الإضافات

إذا كانت مسارات واجهة برمجة التطبيقات REST مفقودة:
1. تأكد من تنشيط WooCommerce
2. تأكد من تنشيط إضافة WCPOS
3. قم بتعطيل الإضافات الأخرى مؤقتاً للاختبار
