# API02006: مفتاح API غير صالح

## ما يعنيه هذا

مفتاح REST API الخاص بـ WooCommerce المستخدم غير صالح. يتم استخدام مفاتيح API للمصادقة بين الخوادم، والمفتاح المقدم لا يتطابق مع أي مفتاح صالح في WooCommerce.

## الأسباب الشائعة

- **حذف المفتاح** — تم إزالة مفتاح API من WooCommerce
- **خطأ إملائي في المفتاح** — تم إدخال المفتاح بشكل غير صحيح
- **عدم تطابق أزواج المفاتيح** — المفتاح الرئيسي والسري غير متطابقين
- **مفتاح من موقع مختلف** — استخدام مفاتيح تم إنشاؤها لتثبيت آخر

## كيفية الإصلاح

### 1. التحقق من مفتاح API

في لوحة تحكم WordPress → WooCommerce → الإعدادات → متقدم → REST API:
1. تحقق مما إذا كان مفتاح API موجودًا
2. تأكد من عدم إلغائه
3. لاحظ الأذونات (قراءة/كتابة/قراءة-كتابة)

### 2. إنشاء مفاتيح API جديدة

إذا كان المفتاح مفقودًا أو غير صالح:
1. انتقل إلى WooCommerce → الإعدادات → متقدم → REST API
2. انقر على "إضافة مفتاح"
3. أدخل وصفًا (مثل "WCPOS")
4. حدد المستخدم
5. اختر أذونات "قراءة/كتابة"
6. انقر على "إنشاء مفتاح API"
7. **انسخ المفتاح الرئيسي والسري** (يُعرض مرة واحدة فقط!)

### 3. تحديث إعدادات POS

أدخل مفاتيح API الجديدة في POS:
- المفتاح الرئيسي (يبدأ بـ `ck_`)
- المفتاح السري (يبدأ بـ `cs_`)

### 4. التحقق من أذونات المفتاح

تأكد من وجود أذونات كافية للمفتاح:
- **قراءة** — عرض البيانات فقط
- **كتابة** — تعديل البيانات فقط
- **قراءة/كتابة** — وصول كامل (موصى به للـ POS)
