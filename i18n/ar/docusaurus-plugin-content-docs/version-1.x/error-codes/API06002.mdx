# API06002: عدم وجود رابط URL للواجهة البرمجية

## ما يعنيه هذا

لم يتم تكوين أي رابط URL للاتصال بمتجر WooCommerce الخاص بك. يحتاج نظام نقطة البيع (POS) إلى معرفة موقع متجرك.

## الأسباب الشائعة

- **الإعداد الأولي** — لم يتم إدخال الرابط من قبل
- **مسح الإعدادات** — تمت إعادة تعيين الإعدادات أو مسحها
- **مسح بيانات التطبيق** — تم حذف بيانات التطبيق
- **مشكلة الترحيل** — فقدان الرابط أثناء التحديث أو الترحيل

## كيفية الإصلاح

### 1. إدخال رابط متجرك

في تطبيق نقطة البيع:
1. افتح الإعدادات أو شاشة تسجيل الدخول
2. ابحث عن حقل تكوين الرابط/الموقع
3. أدخل رابط موقع WordPress الخاص بك
4. مثال: `https://متجرك.com`

### 2. العثور على رابط WordPress

إذا كنت غير متأكد من الرابط:
1. سجل الدخول إلى لوحة التحكم في WordPress
2. انتقل إلى الإعدادات → عام
3. ابحث عن "رابط WordPress"
4. استخدم هذا الرابط في نظام نقطة البيع

### 3. التحقق من الإعدادات المخزنة

إذا كان الرابط مكوناً من قبل:
- تحقق مما إذا كانت بيانات التطبيق قد تم مسحها عن طريق الخطأ
- ابحث عن خيارات النسخ الاحتياطي/التصدير للإعدادات
- أعد إدخال الرابط عند الحاجة

### 4. التأكد من صحة الرابط

قبل الحفظ:
1. اختبر الرابط في متصفح
2. تأكد من تحميل موقع WordPress
3. تضمين `https://` أو `http://`

## قائمة التحقق من الإعداد

عند تكوين WCPOS لأول مرة:

1. ☐ أدخل رابط موقع WordPress
2. ☐ تأكد من تثبيت وتنشيط المكون الإضافي لـ WCPOS
3. ☐ سجل الدخول باستخدام بيانات اعتماد WordPress
4. ☐ اسمح باكتمال المزامنة الأولية للبيانات
