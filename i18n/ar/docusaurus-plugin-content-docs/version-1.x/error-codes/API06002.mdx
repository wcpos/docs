# API06002: URL API مفقود {#api06002-missing-api-url}

## ماذا يعني هذا {#what-this-means}

لم يتم تكوين أي URL للاتصال بمتجر WooCommerce الخاص بك. يحتاج نظام نقاط البيع إلى معرفة مكان وجود متجرك.

## الأسباب الشائعة {#common-causes}

- **إعداد للمرة الأولى** — لم يتم إدخال URL أبداً
- **تم مسح الإعدادات** — تم إعادة تعيين أو مسح الإعدادات
- **تم مسح بيانات التطبيق** — تم حذف بيانات التطبيق
- **مشكلة في الترقية** — فقد URL أثناء التحديث أو الانتقال

## كيفية الإصلاح {#how-to-fix}

### 1. أدخل URL متجرك {#1-enter-your-store-url}

في تطبيق نقاط البيع:
1. افتح الإعدادات أو شاشة تسجيل الدخول
2. ابحث عن حقل تكوين URL / الموقع
3. أدخل URL موقع WordPress الخاص بك
4. مثال: `https://yourstore.com`

### 2. ابحث عن URL WordPress الخاص بك {#2-find-your-wordpress-url}

إذا لم تكن متأكداً من URL الخاص بك:
1. قم بتسجيل الدخول إلى إدارة WordPress
2. انتقل إلى الإعدادات → عام
3. انظر إلى "عنوان WordPress (URL)"
4. استخدم هذا URL في نظام نقاط البيع

### 3. تحقق من الإعدادات المخزنة {#3-check-stored-settings}

إذا تم تكوين URL مسبقًا:
- تحقق مما إذا كانت بيانات التطبيق قد تم مسحها عرضياً
- ابحث عن خيارات النسخ الاحتياطي / التصدير للإعدادات
- أعد إدخال URL إذا لزم الأمر

### 4. تحقق من أن URL صحيح {#4-verify-url-is-correct}

قبل الحفظ:
1. اختبر URL في متصفح
2. تأكد من أنه يفتح موقع WordPress الخاص بك
3. تأكد من تضمين `https://` أو `http://`

## قائمة التحقق من الإعداد {#setup-checklist}

عند تكوين WCPOS للمرة الأولى:

1. ☐ أدخل URL موقع WordPress الخاص بك
2. ☐ تحقق من أن إضافة WCPOS مثبتة ومفعلة
3. ☐ سجل الدخول باستخدام بيانات اعتماد WordPress الخاصة بك
4. ☐ اسمح بإكمال مزامنة البيانات الأولية
