# API05001: تعطيل واجهة برمجة تطبيقات WooCommerce

## ما يعنيه هذا

واجهة برمجة تطبيقات REST الخاصة بـ WooCommerce معطلة على موقعك. يتطلب WCPOS واجهة برمجة التطبيقات REST للتواصل مع WooCommerce والوصول إلى بيانات المتجر.

## الأسباب الشائعة

- **إعداد WooCommerce** — تم تعطيل واجهة برمجة التطبيقات REST عمدًا
- **البرنامج الأمني** — يحظر البرنامج الأمني الوصول إلى واجهة برمجة التطبيقات
- **قيود الاستضافة** — قام مُضيفك بتعطيل الوصول إلى واجهة برمجة تطبيقات REST
- **مشاكل الرابط الدائم** — لم يتم تهيئة الروابط الدائمة لواجهة برمجة التطبيقات REST

## كيفية الإصلاح

### 1. تفعيل واجهة برمجة تطبيقات WooCommerce REST

في إدارة WordPress:
1. اذهب إلى WooCommerce → الإعدادات → متقدم → واجهة برمجة التطبيقات REST
2. تأكد من تمكين واجهة برمجة التطبيقات REST
3. تحقق من إنشاء مفاتيح API

### 2. التحقق من واجهة برمجة تطبيقات WordPress REST

تعتمد واجهة برمجة تطبيقات WooCommerce على واجهة برمجة تطبيقات WordPress REST:
1. قم بزيارة `https://yoursite.com/wp-json/` في المتصفح
2. يجب أن يعيد بيانات JSON
3. إذا لم يكن كذلك، راجع [API05004](./API05004)

### 3. التحقق من البرامج الأمنية

البرامج الأمنية الشائعة التي قد تحظر واجهة برمجة التطبيقات:
- **Wordfence** — تحقق من إعدادات جدار الحماية
- **iThemes Security** — تحقق من إعدادات واجهة برمجة التطبيقات REST
- **All In One WP Security** — راجع قواعد جدار الحماية

قم بإضافة نقاط نهاية واجهة برمجة التطبيقات REST أو تطبيق POS إلى القائمة البيضاء.

### 5. إعادة ضبط الروابط الدائمة

أحيانًا تحتاج إعدادات الروابط الدائمة إلى التحديث:
1. اذهب إلى الإعدادات → الروابط الدائمة
2. انقر على "حفظ التغييرات" (حتى بدون إجراء أي تغييرات)
3. سيؤدي هذا إلى إعادة توليد قواعد إعادة الكتابة
