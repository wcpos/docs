# API06003: تكوين الموقع غير صالح

## ما يعنيه هذا

تكوين الموقع غير صالح أو غير مكتمل. قد يتضمن ذلك روابط غير صحيحة، أو إعدادات مصادقة، أو مشاكل تكوين أخرى.

## الأسباب الشائعة

- **تكوين جزئي** — بعض الإعدادات مفقودة
- **إعدادات غير متطابقة** — التكوين لا يتوافق مع الموقع
- **تلف التكوين** — تم إتلاف الإعدادات
- **تغييرات الموقع** — تم تعديل الموقع دون تحديث تكوين نقطة البيع

## كيفية الإصلاح

### 1. مراجعة جميع الإعدادات

تحقق من التكوين الكامل:
- رابط الموقع صحيح
- المصادقة مُكوّنة بشكل صحيح
- جميع الإعدادات الإضافية صحيحة

### 2. إعادة التكوين من الصفر

إذا كان التكوين تالفًا:
1. مسح جميع الإعدادات المخزنة
2. بدء عملية الإعداد من جديد
3. إدخال تكوين جديد

### 3. التحقق من متطلبات الموقع

تأكد من استيفاء موقع WordPress للمتطلبات:
- WordPress مثبت ويمكن الوصول إليه
- WooCommerce مثبت ونشط
- البرنامج المساعد لـ WCPOS مثبت ونشط
- الروابط الدائمة مفعلة (ليست "بسيطة")

### 4. اختبار الوصول للموقع

تأكد من عمل هذه الروابط في المتصفح:
- `https://yoursite.com/` — الموقع الرئيسي
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. التحقق من إعدادات WordPress

في إدارة WordPress:
1. الإعدادات → عام — التحقق من الروابط
2. الإعدادات → الروابط الدائمة — التأكد من عدم كونها "بسيطة"
3. WooCommerce → الحالة — التحقق من وجود مشاكل

## متطلبات التكوين

يحتاج التكوين الصالح إلى:
- **رابط الموقع** — رابط كامل مع البروتوكول
- **المصادقة** — بيانات اعتماد أو مفاتيح API صالحة
- **الأذونات** — المستخدم لديه وصول لنقطة البيع
- **البرامج المساعدة** — البرامج المطلوبة نشطة
