# API06003: إعداد التكوين غير صالح

## ماذا يعني هذا

إعداد التكوين غير صالح أو غير مكتمل. قد يتضمن ذلك عناوين URL غير صحيحة، إعدادات المصادقة، أو مشاكل تكوين أخرى.

## الأسباب الشائعة

- **تكوين جزئي** — بعض الإعدادات مفقودة
- **إعدادات غير متطابقة** — التكوين لا يتطابق مع الموقع
- **تكوين تالف** — تم تلف الإعدادات
- **تغييرات الموقع** — تم تعديل الموقع دون تحديث إعدادات POS

## كيفية الإصلاح

### 1. مراجعة جميع الإعدادات

تحقق من التكوين الكامل:
- عنوان URL للموقع صحيح
- تم تكوين المصادقة بشكل صحيح
- أي إعدادات إضافية صحيحة

### 2. إعادة التكوين من البداية

إذا كان التكوين تالفًا:
1. مسح جميع الإعدادات المخزنة
2. بدء عملية الإعداد مرة أخرى
3. إدخال تكوين جديد

### 3. تحقق من متطلبات الموقع

تحقق من أن موقع WordPress الخاص بك يفي بالمتطلبات:
- تم تثبيت WordPress ويمكن الوصول إليه
- تم تثبيت WooCommerce وهو نشط
- تم تثبيت ملحق WCPOS وهو نشط
- تم تمكين الروابط الدائمة (ليس "Plain")

### 4. اختبار وصول الموقع

تحقق من أن هذه عناوين URL تعمل في متصفح:
- `https://yoursite.com/` — الموقع الرئيسي
- `https://yoursite.com/wp-json/` — واجهة برمجة التطبيقات REST
- `https://yoursite.com/wp-json/wcpos/v1/` — واجهة برمجة تطبيقات WCPOS

### 5. تحقق من إعدادات WordPress

في إدارة WordPress:
1. الإعدادات → عام — التحقق من عناوين URL
2. الإعدادات → الروابط الدائمة — التأكد من أنها ليست "عادية"
3. WooCommerce → الحالة — تحقق من وجود مشاكل

## متطلبات التكوين

تحتاج التكوينات الصالحة إلى:
- **عنوان URL للموقع** — عنوان URL كامل مع البروتوكول
- **المصادقة** — بيانات اعتماد صحيحة أو مفاتيح API
- **الأذونات** — المستخدم لديه وصول إلى POS
- **الملحقات** — الملحقات المطلوبة نشطة
