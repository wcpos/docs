# API02010: المصادقة مطلوبة

## ما يعنيه هذا

الإجراء الذي تحاول تنفيذه يتطلب المصادقة، لكنك غير مسجل الدخول حاليًا. يحتاج نظام نقطة البيع إلى بيانات اعتماد صالحة للوصول إلى هذا المورد.

## الأسباب الشائعة

- **عدم تسجيل الدخول** — لم تقم بالمصادقة بعد
- **مسح الجلسة** — تمت إزالة جلستك أو انتهت صلاحيتها
- **الوصول إلى مورد محمي** — المورد يتطلب المصادقة
- **مسح بيانات التطبيق** — تمت إزالة بيانات الاعتماد المخزنة

## تعيين خطأ الخادم

يتم تشغيل رمز الخطأ هذا عند إرجاع الخادم:

| رمز الخادم | المصدر |
|-------------|--------|
| `rest_login_required` | WordPress REST API |
| `jwt_auth_no_auth_header` | البرنامج المساعد للمصادقة JWT |
| HTTP 401 | أي استجابة خادم (حل بديل) |

## كيفية الإصلاح

### 1. تسجيل الدخول

إذا لم تكن قد سجلت الدخول:
1. افتح شاشة تسجيل الدخول لنقطة البيع
2. أدخل بيانات اعتمادك لـ WordPress
3. أكمل عملية المصادقة

### 2. التحقق من حالة الجلسة

إذا كنت تعتقد أنك مسجل الدخول:
- قد تكون جلستك قد انتهت
- راجع [API02002](./API02002) (انتهاء صلاحية الرمز المميز) للمزيد من التفاصيل
- سجل الدخول مرة أخرى لاستعادة الوصول

### 3. التحقق من إعداد الخادم

تأكد من تكوين نقاط نهاية API بشكل صحيح:
- يجب تفعيل WooCommerce REST API
- يجب أن يكون المكون الإضافي WCPOS نشطًا
- يجب أن تكون نقاط نهاية المصادقة قابلة للوصول

### 4. التحقق من مشاكل المتصفح/التطبيق

إذا كنت تسجل الخروج بشكل غير متوقع:
- امسح ذاكرة التخزين المؤقت للمتصفح (إذا كنت تستخدم الإصدار عبر الويب)
- تأكد من عدم حظر الكوكيز/التخزين المحلي
- تحقق من أن التطبيق لديه إذن لتخزين البيانات

## ما يتطلب المصادقة؟

معظم عمليات نقطة البيع تتطلب المصادقة:
- عرض المنتجات والعملاء
- إنشاء وتحرير الطلبات
- معالجة المدفوعات
- الوصول إلى التقارير

الشاشة الأولية لتسجيل الدخول فقط يمكن الوصول إليها بدون مصادقة.
