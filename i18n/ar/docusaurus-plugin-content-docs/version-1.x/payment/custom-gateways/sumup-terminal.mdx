# بوابة بطاقات SumUp

توفر بوابة SumUp إمكانية قبول مدفوعات البطاقات باستخدام قارئات بطاقات SumUp مباشرة داخل نظام نقطة البيع لـ WooCommerce. توفر هذه البوابة تكاملًا سلسًا مع نظام معالجة المدفوعات الخاص بـ SumUp للمعاملات الشخصية.

## الميزات

- **تكامل الأجهزة**: توصيل قارئات بطاقات SumUp بنظام نقطة البيع عبر اتصال الإنترنت
- **معالجة فورية**: معالجة المدفوعات وإكمال الطلبات فورًا
- **كشف تلقائي**: إكمال الطلبات تلقائيًا عند اكتشاف الدفع
- **معاملات آمنة**: معالجة مدفوعات متوافقة مع PCI من خلال SumUp
- **إقران سهل**: عملية إقران جهاز بسيطة باستخدام رموز الإقران
- **SDK مبني على الويب**: يستخدم SDK JavaScript الخاص بـ SumUp للتوافق مع تطبيقات الويب

## التثبيت

### الخطوة 1: تنزيل وتثبيت

1. قم بزيارة [صفحة إصدارات SumUp Terminal لـ WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. قم بتنزيل أحدث ملف **sumup-terminal-for-woocommerce.zip**
3. في لوحة تحكم WordPress الخاصة بك، انتقل إلى `الإضافات > إضافة جديد > تحميل إضافة`
4. قم بتحميل الملف المضغوط وتفعيل الإضافة

### الخطوة 2: ضبط إعدادات SumUp

1. انتقل إلى `لوحة تحكم WP > WooCommerce > الإعدادات > الدفع`
2. ابحث عن **بوابة بطاقات SumUp** في قائمة طرق الدفع
3. انقر على **بوابة بطاقات SumUp** للوصول إلى الإعدادات
4. أدخل **مفتاح API الخاص بـ SumUp** (متاح من لوحة تحكم تاجر SumUp)
5. احفظ الإعدادات

:::note
لا تحتاج إلى تمكين بوابة SumUp Terminal في إعدادات WooCommerce. سيتم تمكينها خصيصًا لنظام نقطة البيع في خطوة لاحقة.
:::

### الخطوة 3: إقران جهاز SumUp

1. في نفس صفحة الإعدادات، حدد قسم **إقران القارئ**
2. على جهاز SumUp، انتقل إلى شاشة الإقران لعرض رمز الإقران
3. أدخل رمز الإقران المعروض على جهاز SumUp
4. انقر على **"إقران القارئ"** لإنشاء الاتصال
5. انتظر التأكيد بأن القارئ تم إقرانه بنجاح

:::warning مهم
يجب إقران القارئ بنجاح قبل معالجة المدفوعات. تأكد من إكمال عملية الإقران قبل المتابعة.
:::

### الخطوة 4: التمكين في نظام نقطة البيع لـ WooCommerce

1. انتقل إلى `لوحة تحكم WP > نقطة البيع لـ WooCommerce > الإعدادات > الدفع`
2. ابحث عن بوابة **بطاقات SumUp** في القائمة
3. قم بتمكين البوابة للاستخدام في نظام نقطة البيع
4. احفظ الإعدادات

## الاستخدام

### معالجة المدفوعات

1. **إضافة المنتجات**: أضف المنتجات إلى عربة التسوق في نظام نقطة البيع
2. **اختيار البوابة**: اختر "بطاقات SumUp" كطريقة دفع
3. **بدء الدفع**: انقر لبدء دفعة جديدة على جهاز SumUp
4. **دفع العميل**: يكمل العميل الدفع على طرفية SumUp
5. **إكمال تلقائي**: يتم إكمال الطلب تلقائيًا عند اكتشاف الدفع

### أدوات التحكم في الدفع

عند استخدام بوابة بطاقات SumUp، ستتوفر لك الخيارات التالية:

- **بدء دفعة جديدة**: بدء طلب دفع على الطرفية المتصلة
- **إلغاء الدفع**: إلغاء دفعة قيد التنفيذ
- **حالة الدفع**: مراقبة الحالة الحالية لعملية الدفع

### إدارة الطلبات

- **تحديثات تلقائية**: يتم وضع علامة على الطلبات كمكتملة عند نجاح الدفع
- **تتبع الدفع**: يتم تسجيل جميع تفاصيل الدفع في ملاحظات الطلب
- **إنشاء إيصال**: يتم إنشاء إيصالات نظام نقطة البيع القياسية بعد الدفعات الناجحة

## المتطلبات

- **حساب SumUp**: حساب تاجر SumUp نشط
- **وصول API**: مفتاح API SumUp من لوحة تحكم التاجر
- **أجهزة متوافقة**: جهاز قراءة بطاقات SumUp
- **نظام نقطة البيع لـ WooCommerce**: الإصدار المجاني أو المدفوع مثبت ومفعل
- **اتصال مستقر**: اتصال إنترنت موثوق لتواصل API

## توافق الأجهزة

:::important متطلبات الاتصال
يستخدم هذا التنفيذ SDK JavaScript الخاص بـ SumUp، مما يعني أنه يعمل من خلال تطبيقات الويب ولكنه يتطلب **طرفيات متصلة بالإنترنت فقط**. الطرفيات البلوتوث غير مدعومة حاليًا.
:::

### الطرفيات المدعومة (متصلة بالإنترنت)

- **SumUp Solo** ✅ - طرفية متصلة بـ WiFi/Ethernet

### الطرفيات غير المدعومة (بلوتوث)

- **SumUp Air** ❌ - بلوتوث فقط
- **SumUp 3G** ❌ - بلوتوث فقط

:::note الدعم المستقبلي
سيتم إضافة دعم طرفيات البلوتوث في التطبيقات الخاصة بـ iOS وAndroid خلال **الربع الأول من عام 2026**. سيمكن هذا من دعم جميع طرفيات SumUp المعتمدة بما في ذلك طرازات Air و3G.
:::

## استكشاف الأخطاء وإصلاحها

### المشاكل الشائعة

**القارئ لا يتم إقرانه**
- تأكد من استخدام طرفية متصلة بالإنترنت مدعومة (Solo فقط)
- تحقق من اتصال جهاز SumUp Solo بـ WiFi وأنه متصل
- تأكد من وجود الجهاز في وضع الإقران وإدخال رمز الإقران بشكل صحيح
- أكد أن حساب SumUp لديه وصول API مفعل
- تأكد من وجود اتصال إنترنت مستقر أثناء الإقران

**الدفع لا يتم معالجته**
- تحقق من إقران القارئ بشكل صحيح
- تأكد من نشاط حساب SumUp وأنه في حالة جيدة
- تأكد من وجود شحن كافٍ للبطارية واتصال مستقر
- تأكد من صحة مفتاح API وأنه نشط

**الطلب لا يتم إكماله**
- تحقق من استقرار اتصال الإنترنت
- تأكد من إعدادات الويب هوك في حساب SumUp
- تأكد من نجاح الدفع على طرفية SumUp
- تحقق من سجلات أخطاء WordPress لمشاكل تواصل API

### حدود معدل API

تمت مoptimizationة البوابة للحد من استدعاءات API وتجنب وضع الحد:
- يتم التحقق من حالة الدفع بكفاءة
- يتم تقليل طلبات API غير الضرورية
- منطق إعادة المحاولة التلقائي يتعامل مع الأخطاء المؤقتة

### طلب المساعدة
