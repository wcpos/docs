# بوابات الدفع {#payment-gateways}

يدعم WCPOS عدة بوابات دفع مخصصة توسّع وظائف نظام نقاط البيع. وقد صُممت هذه البوابات خصيصًا لبيئات نقاط البيع، وتوفر تكاملاً سلسًا مع مختلف أساليب وخدمات الدفع.

## البوابات المخصصة المتاحة {#available-custom-gateways}

### بوابات أجهزة الدفع الطرفية {#hardware-terminal-gateways}

- **[Stripe Terminal](./stripe-terminal)** - اقبل المدفوعات باستخدام قارئات أجهزة Stripe Terminal
- **[SumUp Terminal](./sumup-terminal)** - عالج المدفوعات عبر قارئات بطاقات SumUp
- **[Square Terminal](./square-terminal)** - اجمع المدفوعات الحضورية على أجهزة Square Terminal
- **[PayPal Reader (Zettle)](./paypal-reader)** - اقبل مدفوعات البطاقات الحضورية على جهاز PayPal Reader (Zettle) الطرفي
- **[Mollie Terminal](./mollie-terminal)** - اقبل المدفوعات الحضورية على أجهزة Mollie Terminal

### بوابات الدفع عبر الجوال {#mobile-payment-gateways}

- **[Vipps MobilePay](./vipps-mobilepay)** - اقبل المدفوعات عبر الهاتف باستخدام رمز QR أو إشعار فوري

### بوابات الدفع الرقمية {#digital-payment-gateways}

- **[فاتورة عبر البريد الإلكتروني](./email-invoice)** - إرسال فواتير الدفع إلى العملاء عبر البريد الإلكتروني
- **[الدفع عبر الويب](./web-checkout)** - إعادة توجيه العملاء لإتمام المدفوعات عبر الإنترنت

## إنشاء بوابتك الخاصة {#creating-your-own-gateway}

إذا كنت بحاجة إلى بوابة دفع مخصصة لمتطلباتك المحددة، يمكنك استخدام **[قالب البوابة](/reference/gateway-template)** لإنشاء بوابة دفع WCPOS خاصة بك.

## نظرة عامة على التثبيت {#installation-overview}

تتبع جميع البوابات المخصصة عملية تثبيت متشابهة:

1. **تنزيل** أحدث إصدار من مستودع GitHub المعني
2. **تثبيت** الملحق عبر `WP Admin > إضافات > إضافة جديد > رفع إضافة`
3. **تهيئة** إعدادات البوابة في `WP Admin > WooCommerce > الإعدادات > المدفوعات`
4. **فعّل** بوابة الدفع في `WP Admin > نقطة البيع > الإعدادات > إتمام الطلب`
