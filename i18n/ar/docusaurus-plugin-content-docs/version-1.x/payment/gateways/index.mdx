# بوابات الدفع {#payment-gateways}

يدعم WCPOS عدة بوابات دفع مخصصة توسّع وظائف نظام نقاط البيع. صُمّمت هذه البوابات خصيصًا لبيئات نقاط البيع وتوفر تكاملًا سلسًا مع مختلف طرق الدفع والخدمات.

## البوابات المخصصة المتاحة {#available-custom-gateways}

### بوابات أجهزة الدفع {#hardware-terminal-gateways}

- **[Stripe Terminal](./stripe-terminal)** - قبول المدفوعات باستخدام أجهزة قراءة Stripe Terminal
- **[SumUp Terminal](./sumup-terminal)** - معالجة المدفوعات عبر أجهزة قراءة البطاقات من SumUp
- **[Square Terminal](./square-terminal)** - تحصيل المدفوعات الحضورية على أجهزة Square Terminal
- **[PayPal Reader (Zettle)](./paypal-reader)** - قبول مدفوعات البطاقات الحضورية عبر جهاز PayPal Reader (Zettle)

### بوابات الدفع عبر الهاتف المحمول {#mobile-payment-gateways}

- **[Vipps MobilePay](./vipps-mobilepay)** - قبول المدفوعات عبر الهاتف من خلال رمز QR أو الإشعارات الفورية

### بوابات الدفع الرقمية {#digital-payment-gateways}

- **[فاتورة بالبريد الإلكتروني](./email-invoice)** - إرسال فواتير الدفع إلى العملاء عبر البريد الإلكتروني
- **[الدفع عبر الويب](./web-checkout)** - إعادة توجيه العملاء لإتمام الدفع عبر الإنترنت

## إنشاء بوابة دفع خاصة بك {#creating-your-own-gateway}

إذا كنت بحاجة إلى بوابة دفع مخصصة تلبي متطلباتك الخاصة، يمكنك استخدام **[قالب البوابة](/reference/gateway-template)** لإنشاء بوابة دفع WCPOS خاصة بك.

## نظرة عامة على التثبيت {#installation-overview}

تتبع جميع البوابات المخصصة عملية تثبيت مشابهة:

1. **تنزيل** أحدث إصدار من مستودع GitHub المعني
2. **تثبيت** الإضافة عبر `WP Admin > إضافات > إضافة جديد > رفع إضافة`
3. **ضبط** إعدادات البوابة في `WP Admin > WooCommerce > الإعدادات > المدفوعات`
4. **تفعيل** البوابة في `WP Admin > نقطة البيع > الإعدادات > إتمام الطلب`
