يتضمن WCPOS نظام قوالب كاملا للإيصالات والفواتير وإيصالات الهدايا وقسائم التعبئة وتذاكر المطبخ وغير ذلك، وتتم إدارته كله من **WP Admin → POS → القوالب**.

:::tip هل تريد فقط تغيير الإيصال؟
ابدأ من **[تخصيص الإيصال](/templates/customise)**. تشرح هذه الصفحة أسهل ثلاث طرق: اختيار قالب مرفق مختلف، أو طلب تعديل من الذكاء الاصطناعي، أو التحرير يدويا.
:::

## محركات القوالب

يدعم WCPOS ثلاثة محركات قوالب. يتم اختيار المحرك عند إنشاء قالب أو عند النقر على **استخدام القالب** في بطاقة من المعرض، و**لا يمكن تغييره بعد ذلك**. ابدأ قالبا جديدا إذا أردت تغيير المحرك.

## معرض القوالب

يأتي المعرض بقوالب جاهزة يمكنك استخدامها كما هي أو تخصيصها. يمكنك التصفية حسب **الفئة** (إيصال، فاتورة، إيصال هدية، تذكرة مطبخ، عرض سعر / أمر شراء)، و**التنسيق** (HTML / ESC/POS)، و**الاتجاه** (LTR / RTL)، مع معاينة تستخدم بيانات متجرك الحقيقية. يحتوي أسفل كل بطاقة على رابط **معاينة** وزر أساسي **استخدام القالب**. يؤدي النقر عليه إلى إنشاء نسخة قابلة للتحرير في *قوالبك*.

| القالب | المحرك | الوصف |
|----------|--------|-------------|
| إيصال قياسي | HTML | الإيصال الافتراضي: الشعار، هوية المتجر، العناصر، الإجماليات، الدفع |
| إيصال قياسي (RTL) | HTML | نسخة من اليمين إلى اليسار للعربية والعبرية والفارسية والأردية |
| بسيط / حديث | HTML | العناصر الأساسية نفسها في الإيصال القياسي لكن بمساحة رأسية أقل |
| إيصال مفصل | HTML | فاتورة ضريبية كاملة: عمود SKU، سعر الوحدة، تفصيل الضرائب حسب النسبة، العناوين |
| إيصال هدية | HTML | العناصر فقط مع إخفاء الأسعار. يتضمن رسالة هدية وسياسة إرجاع |
| فاتورة | HTML | فاتورة صفحة كاملة A4/Letter مع لوحة اختيارية "كيفية الدفع" للطلبات غير المدفوعة |
| قسيمة تعبئة | HTML | مستند للمخزن: العناصر + الكميات، عنوان الشحن، دون أسعار |
| عرض سعر / تقدير | HTML | مستند قبل البيع يتضمن الأسعار والشروط، دون قسم دفع |
| إيصال ضيق | HTML | إيصال بخط ثابت العرض للورق الضيق أو الطابعات الحرارية الداعمة لـ HTML |
| إيصال حراري بسيط (58 مم) | حراري | تخطيط حراري نظيف بقياس 58 مم |
| إيصال حراري بسيط (80 مم) | حراري | تخطيط حراري نظيف بقياس 80 مم، وهو الأكثر شيوعا |
| إيصال حراري بسيط 80 مم (RTL) | حراري | نسخة RTL بقياس 80 مم. تحتاج إلى طابعة بصفحة ترميز عربية (CP864 / Windows-1256) |
| إيصال حراري مفصل (58 مم) | حراري | 58 مم شامل: عناوين، ضرائب، مبالغ مستردة، مدفوعات، شروط، باركود |
| إيصال حراري مفصل (80 مم) | حراري | 80 مم شامل: عناوين، ضرائب، مبالغ مستردة، مدفوعات، شروط، باركود |
| تذكرة مطبخ | حراري | العناصر فقط، خط كبير، بلا أسعار، لمحطات التحضير |

### قوالب مستندات من جهات خارجية

إذا كان **PDF Invoices & Packing Slips for WooCommerce** من WP Overnight مفعلا، يعرض WCPOS أيضا **فاتورة (WP Overnight)** و**قسيمة تعبئة (WP Overnight)** في قائمة قوالب الإيصالات. تفوض قوالب HTML المعروضة على الخادم هذه العمل إلى واجهة مستندات WP Overnight، وتعيد استخدام ترقيم الفواتير وقسائم التعبئة، والتخطيط، والهوية، والحقول القانونية والضريبية، وتخصيصات القوالب المضبوطة في ذلك الملحق.

تتوفر هذه القوالب فقط أثناء تفعيل ملحق WP Overnight، وتحتاج إلى اتصال بالخادم للعرض. استخدم قوالب HTML أو القوالب الحرارية المرفقة مع WCPOS عند الحاجة إلى الطباعة دون اتصال.

### عرض الضرائب التكيفي

تتكيف معظم القوالب المرفقة (القياسي، القياسي RTL، البسيط/الحديث، الضيق، الفاتورة، عرض السعر، والنسخ الحرارية البسيطة) **تلقائيا مع إعداد عرض الضرائب في WooCommerce**:

- **المتاجر التي تشمل الضرائب** (الاتحاد الأوروبي/المملكة المتحدة/أستراليا) تعرض أسعارا إجمالية مع سطر "الضريبة مشمولة".
- **المتاجر التي تستثني الضرائب** (الولايات المتحدة/كندا) تعرض أسعارا صافية مع إضافة الضريبة في سطر منفصل.
- يعرض الإجمالي النهائي دائما المبلغ الإجمالي الذي تم تحصيله فعليا.

عائلة **المفصل** هي فاتورة ضريبية رسمية وتعرض الضرائب دائما بأسطر غير شاملة للضريبة مع تفصيل صريح، بغض النظر عن إعداد المتجر.

## طريقة العمل

1. **أضف قالبا**: انقر على **استخدام القالب** في بطاقة من المعرض، أو أنشئ قالبا جديدا من البداية.
2. **خصصه** باستخدام محرر التطبيق مع المعاينة الحية، أو الصقه في ChatGPT / Claude واطلب التغييرات التي تريدها.
3. **فعله**: شغل مفتاح **نشط** في *قوالبك*. يظهر كل قالب إيصال نشط في قائمة شاشة الإيصال عند نقطة البيع.
4. **اطبع أو اعرض**: تفتح الإيصالات بعد إتمام البيع مع خيارات للطباعة أو الإرسال بالبريد الإلكتروني أو العرض على الشاشة.

يتم عرض القوالب باستخدام [حمولة بيانات إيصال](/templates/receipt-data) موحدة، مع أقسام لمعلومات المتجر، وبنود العناصر، والإجماليات، والضرائب، والمدفوعات، والمبالغ المستردة، وغير ذلك. تتضمن كل حقول العملة نسخ `_display` منسقة مسبقا، مثل `$29.99` بدلا من `29.99`.

## ميزات Pro

مع [WCPOS Pro](/getting-started/pro-license)، يمكن لكل متجر أن يملك تعيينات قوالب وهوية خاصة به (الشعار، العنوان، تفاصيل الاتصال) وترتيب قوالب خاصا به. هذا مفيد للعمليات متعددة المواقع حيث يطبع كل متجر ترويسته الخاصة.
