تسجل شاشة السجلات النشاط في النظام داخل نقطة البيع. إنها أداة أساسية لتصحيح الأخطاء وفهم ما يحدث خلف الكواليس.

## الوصول إلى السجلات

افتح شاشة السجلات من القائمة الجانبية اليسرى (رمز نبض القلب).

## نظرة عامة على الواجهة

### البحث والتصفية

- **بحث السجلات** - ابحث عن إدخالات سجلات محددة بواسطة محتوى الرسالة
- **تصفية مستوى السجل** - تصفية حسب مستوى السجل (معلومات، تحذير، خطأ)
- **إعدادات العرض** - تكوين الأعمدة الظاهرة

### جدول السجل

كل إدخال سجل يظهر:

| العمود | الوصف |
|--------|-------------|
| **الوقت** | متى حدث الحدث |
| **المستوى** | الشدة: معلومات (رمادي)، تحذير (برتقالي)، خطأ (أحمر) |
| **الرسالة** | وصف مفهوم للحدث |
| **السياق** | JSON قابل للتوسع مع تفاصيل إضافية |
| **الكود** | كود الخطأ (روابط إلى الوثائق) |

## مستويات السجل

### معلومات (رمادي)

العمليات العادية والأحداث الناجحة:
- "تم جلب جميع المعرفات للمنتجات"
- "تم مزامنة العملاء الجدد"
- "تم جلب جميع المعرفات للضرائب"

### تحذير (برتقالي)

مشاكل غير حرجة قد تحتاج إلى انتباه:
- "غير قادر على تحديث الجلسة"
- "محاولة إعادة للمزامنة"

### خطأ (أحمر)

مشاكل تحتاج إلى معالجة:
- "عذراً، لا يمكنك سرد الموارد" (مشكلة إذن)
- "فشلت طلب API"
- "خطأ في مزامنة قاعدة البيانات"

## بيانات السياق

انقر على سهم التوسيع في أي إدخال سجل لرؤية تفاصيل السياق JSON:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

تتضمن هذه البيانات:
- **عدد العناصر** - عدد السجلات التي تأثرت
- **وقت التنفيذ** - كم من الوقت استغرقت العملية
- **حمولة الخادم** - مقاييس أداء الخادم
- **تفاصيل الطلب/الاستجابة** - لتصحيح أخطاء API

## أكواد الأخطاء

تتضمن العديد من إدخالات السجل كود خطأ **(مثل، `API02004`، `DB01001`)**. ترتبط هذه الأكواد مباشرة بالوثائق [أكواد الأخطاء](/error-codes)، والتي توفر:

- شرح تفصيلي للخطأ
- الأسباب الشائعة
- خطوات تصحيح الأخطاء

## إدخالات السجل الشائعة

### عمليات المزامنة

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

تشير هذه إلى نشاط مزامنة عادي. يُظهر السياق عدد العناصر التي تم مزامنتها.

### أخطاء الإذن

```
[error] Sorry, you cannot list resources    {...} API02004
```

يعني هذا عادة أن المستخدم ليس لديه إذن للوصول إلى مورد. تحقق من أدوار المستخدم في `WP Admin > POS > Settings > Access`.

### مشاكل الجلسة

```
[warn] Unable to refresh session    {...} API02007
```

لم تتمكن نقطة البيع من تجديد جلسة المصادقة. قد يتطلب ذلك تسجيل الخروج وتسجيل الدخول مرة أخرى.

## استخدام السجلات للدعم

عند طلب الدعم:

1. **إعادة إنتاج المشكلة** - قم بتنفيذ الإجراء الذي يسبب المشكلة
2. **قم بالتحقق من السجلات** - ابحث عن الأخطاء (أحمر) أو التحذيرات (برتقالي)
3. **توسيع السياق** - احصل على تفاصيل JSON الكاملة
4. **لاحظ كود الخطأ** - قم بتضمين هذا في طلب الدعم الخاص بك
5. **نسخ الإدخالات ذات الصلة** - شارك رسائل السجل مع السياق

## مراقبة الأداء

تتضمن السجلات وقت التنفيذ ومقاييس حمل الخادم. إذا لاحظت:

- **أوقات تنفيذ مرتفعة** - قد يكون الخادم مزدحمًا أو بطيئًا
- **قيم حمولة خادم مرتفعة** - النظر في تحسين الخادم
- **تكرار المهلات** - تحقق من اتصال الشبكة
