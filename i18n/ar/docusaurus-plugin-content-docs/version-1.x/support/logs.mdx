شاشة السجلات تسجل النشاط النظامي داخل نظام نقطة البيع. وهي أداة أساسية لتصحيح المشكلات وفهم ما يحدث خلف الكواليس.

{/* Screenshot: logs-screen.png */}

## الوصول إلى السجلات

افتح شاشة السجلات من درج التنقل الأيسر (أيقونة النبض).

## نظرة عامة على الواجهة

### البحث والتصفية

- **البحث في السجلات** - العثور على إدخالات سجل محددة حسب محتوى الرسالة
- **تصفية مستوى السجل** - التصفية حسب مستوى السجل (معلومات، تحذير، خطأ)
- **إعدادات العرض** - تكوين الأعمدة المرئية

### جدول السجلات

يعرض كل إدخال سجل:

| العمود | الوصف |
|--------|-------------|
| **الوقت** | متى حدث الحدث |
| **المستوى** | الشدة: معلومات (رمادي)، تحذير (برتقالي)، خطأ (أحمر) |
| **الرسالة** | وصف يمكن للإنسان قراءته للحدث |
| **السياق** | JSON قابل للتوسيع مع تفاصيل إضافية |
| **الكود** | كود الخطأ (يرتبط بالوثائق) |

## مستويات السجل

### معلومات (رمادي)

العمليات العادية والأحداث الناجحة:
- "تم جلب جميع المعرفات للمنتجات"
- "تم مزامنة العملاء الجدد"
- "تم جلب جميع المعرفات للضرائب"

### تحذير (برتقالي)

مشكلات غير حرجة قد تحتاج إلى اهتمام:
- "تعذر تحديث الجلسة"
- "محاولة إعادة المزامنة"

### خطأ (أحمر)

مشاكل تحتاج إلى معالجة:
- "عذراً، لا يمكنك سرد الموارد" (مشكلة في الإذن)
- "فشل طلب API"
- "خطأ في مزامنة قاعدة البيانات"

## بيانات السياق

انقر على سهم التوسيع في أي إدخال سجل لرؤية سياق JSON التفصيلي:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

تتضمن هذه البيانات:
- **عدد العناصر** - كم عدد السجلات المتأثرة
- **وقت التنفيذ** - مدة العملية
- **حمل الخادم** - مقاييس أداء الخادم
- **تفاصيل الطلب/الاستجابة** - لتصحيح مشاكل API

## أكواد الأخطاء

يتضمن العديد من إدخالات السجل **كود خطأ** (مثل `API02004`, `DB01001`). ترتبط هذه الأكواد مباشرة بوثائق [أكواد الأخطاء](/error-codes)، والتي توفر:

- شرح مفصل للخطأ
- الأسباب الشائعة
- خطوات استكشاف الأخطاء وإصلاحها

## إدخالات السجل الشائعة

### عمليات المزامنة

```
[info] تم جلب جميع المعرفات للمنتجات    {...} 3 عناصر
[info] تمت مزامنة العملاء الجدد            {...} 1 عنصر
[info] تم جلب جميع المعرفات للضرائب       {...} 3 عناصر
```

هذه تشير إلى نشاط مزامنة عادي. يوضح السياق عدد العناصر التي تمت مزامنتها.

### أخطاء الأذونات

```
[error] عذراً، لا يمكنك سرد الموارد    {...} API02004
```

هذا يعني عادة أن المستخدم ليس لديه إذن للوصول إلى مورد. تحقق من أدوار المستخدم في `WP Admin > POS > الإعدادات > الوصول`.

### مشاكل الجلسة

```
[warn] تعذر تحديث الجلسة    {...} API02007
```

تعذر على نظام نقطة البيع تحديث جلسة المصادقة. قد يتطلب ذلك تسجيل الخروج وتسجيل الدخول مرة أخرى.

## استخدام السجلات للدعم

عند طلب الدعم:

1. **إعادة المشكلة** - قم بتنفيذ الإجراء الذي يسبب المشكلة
2. **تحقق من السجلات** - ابحث عن الأخطاء (باللون الأحمر) أو التحذيرات (باللون البرتقالي)
3. **وسع السياق** - احصل على تفاصيل JSON الكاملة
4. **لاحظ كود الخطأ** - ضمنه في طلب الدعم
5. **انسخ الإدخالات ذات الصلة** - شارك رسائل السجل مع السياق

## مراقبة الأداء

تتضمن السجلات مقاييس وقت التنفيذ وحمل الخادم. إذا لاحظت:

- **أوقات تنفيذ عالية** - قد يكون الخادم مثقلاً أو بطيئًا
- **قيم حمل الخادم عالية** - فكر في تحسين الخادم
- **مهل متكررة** - تحقق من اتصال الشبكة
