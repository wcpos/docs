---
title: POS-Leistung
---

WCPOS ist eine Single-Page-JavaScript-Webanwendung, die über die [WooCommerce REST API](http://woothemes.github.io/woocommerce-rest-api-docs/) mit Ihrem Server kommuniziert. Leistungsprobleme können in zwei Kategorien unterteilt werden: serverseitig und clientseitig.

### Serverseitige Leistung

Serverseitige Leistung bezieht sich auf Aktionen wie das Herunterladen einer Produktseite (10 Produkte) oder die Verarbeitung einer Bestellung. Probleme, die die Geschwindigkeit solcher Aktionen beeinflussen können, sind:

- Die Geschwindigkeit Ihrer Internetverbindung
- Die Geschwindigkeit Ihres Servers, d.h. Rechenleistung, RAM, Serverlast usw.
- Wie viele Plugins Sie aktiv haben
- Zahlungsgateway-Verarbeitung

Die [Demo-Website](http://demo.wcpos.com/pos) stellt ein "Best-Case"-Szenario dar. Die Demo-Website verwendet [einen sehr guten Webhost](http://wcpos.com/wpe) und hat nur sehr wenige Plugins aktiviert.

| Vorgang                  | Durchschnittsgeschwindigkeiten für [demo.wcpos.com/pos](http://demo.wcpos.com/pos)<sup>\*</sup> |
| ------------------------ | ---------------------------------------------------------------------------------------------- |
| 10 Produkte laden        | 1 - 2 Sekunden                                                                                 |
| Bargeldverkauf verarbeiten | 1 - 2 Sekunden                                                                                 |
| Stripe-Verkauf verarbeiten | 2 - 3 Sekunden                                                                                 |

<small>
  <sup>*</sup> Durchschnittszeiten für eine Breitbandverbindung. Zeiten können je nach Ihrer Internetgeschwindigkeit variieren.
</small>

Wenn Ihr POS deutlich länger als die oben genannten Zeiten benötigt, sollten Sie folgende Tests durchführen:

1.  Wechseln Sie das Theme zum Standard-WordPress-Theme Twenty Fifteen
2.  Deaktivieren Sie alle Plugins außer WooCommerce und WCPOS
3.  [Löschen Sie die lokalen Speicherdaten](http://faq.wcpos.com/en/clear-local-data.html)
4.  Verwenden Sie nun den POS, um zu sehen, ob es eine Leistungssteigerung gibt
5.  **Wenn es eine Leistungssteigerung gibt:** Aktivieren Sie Ihr Theme und Plugins nacheinander, um zu sehen, welches Plugin die Leistung beeinträchtigt
6.  **Wenn es keine Leistungssteigerung gibt:** Erwägen Sie einen Wechsel zu einem besseren Webhost oder investieren Sie in eine schnellere Internetverbindung

### Clientseitige Leistung

WCPOS verwendet JavaScript, HTML und CSS, um die von der WooCommerce REST API abgerufenen Daten anzuzeigen, d.h. [Produkte](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) und [Bestellungen](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order).
Um die clientseitige Leistung zu verbessern, speichert der POS die Daten im Browser mit [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API).
Wenn beispielsweise ein Produkt zum ersten Mal abgerufen wird, wird eine Anfrage an den Server gesendet. Sobald die Produktdaten heruntergeladen sind, werden sie lokal gespeichert, sodass nachfolgende Suchen sofort erfolgen.

![Ein Beispiel für lokal gespeicherte Produkte.](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "Ein Beispiel für lokal gespeicherte Produkte")

IndexedDB-Daten bleiben auch beim Schließen des Browsers oder Neustarten des Computers erhalten.
Wenn Sie feststellen, dass Ihre Produktdaten aus irgendeinem Grund nicht synchron sind, können Sie [den lokalen Speicher löschen](http://faq.wcpos.com/en/clear-local-data.html) und einen neuen Satz Daten vom Server abrufen.
