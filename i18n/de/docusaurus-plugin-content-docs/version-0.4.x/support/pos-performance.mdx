WCPOS ist eine Single-Page-JavaScript-Webanwendung, die über die [WooCommerce REST API](http://woothemes.github.io/woocommerce-rest-api-docs/) mit Ihrem Server kommuniziert. Leistungsprobleme können in zwei Kategorien unterteilt werden: serverseitig und clientseitig.

### Serverseitige Leistung

Die serverseitige Leistung bezieht sich auf Aktionen wie das Herunterladen einer Seite mit Produkten (10 Produkte) oder das Verarbeiten einer Bestellung. Probleme, die die Geschwindigkeit solcher Aktionen beeinträchtigen können, sind:

- Die Geschwindigkeit Ihrer Internetverbindung
- Die Geschwindigkeit Ihres Servers, d.h.: Verarbeitungsleistung, RAM, Serverlast usw.
- Wie viele Plugins Sie aktiv haben
- Verarbeitung des Zahlungsdienstanbieters

Die [Demoseite](http://demo.wcpos.com/pos) stellt ein 'Best-Case'-Szenario dar. Die Demoseite verwendet [einen sehr guten Webhost](http://wcpos.com/wpe) und hat eine sehr geringe Anzahl an aktivierten Plugins.

| Prozess                   | Durchschnittliche Geschwindigkeiten für [demo.wcpos.com/pos](http://demo.wcpos.com/pos)<sup>\*</sup> |
| ------------------------- | ----------------------------------------------------------------------------------------------------- |
| Abrufen von 10 Produkten  | 1 - 2 Sekunden                                                                                       |
| Verarbeiten eines Barverkaufs | 1 - 2 Sekunden                                                                                   |
| Verarbeiten eines Stripe-Verkaufes | 2 - 3 Sekunden                                                                              |

<small>
  <sup>*</sup> Durchschnittswerte für eine Breitbandverbindung. Die Zeiten können je nach Ihrer Internetgeschwindigkeit variieren.
</small>

Wenn Ihr POS signifikant länger als die oben genannten Zeiten benötigt, sollten Sie die folgenden Tests durchführen:

1.  Wechseln Sie Ihr Theme auf das Standard-Theme Twenty Fifteen von WordPress.
2.  Deaktivieren Sie alle Plugins außer WooCommerce und WCPOS.
3.  [Löschen Sie die lokalen Speicherdaten](http://faq.wcpos.com/en/clear-local-data.html).
4.  Verwenden Sie jetzt den POS, um zu sehen, ob es eine Leistungssteigerung gibt.
5.  **Wenn es eine Leistungssteigerung gibt:** Aktivieren Sie Ihr Theme und Ihre Plugins nacheinander wieder, um zu sehen, welches Plugin Ihre Leistung beeinträchtigt.
6.  **Wenn es keine Leistungssteigerung gibt:** möchten Sie vielleicht zu einem besseren Webhost wechseln oder in eine schnellere Internetverbindung investieren.

### Clientseitige Leistung

WCPOS verwendet JavaScript, HTML und CSS, um die Daten anzuzeigen, die von der WooCommerce REST API abgerufen werden, d.h.: [Produkte](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) und [Bestellungen](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order). 
Um die clientseitige Leistung zu verbessern, speichert der POS die Daten im Browser mithilfe von [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API). 
Wenn ein Produkt zum ersten Mal abgerufen wird, wird eine Anfrage an den Server gesendet. Sobald die Produktdaten heruntergeladen sind, werden sie lokal gespeichert, sodass nachfolgende Suchen sofort erfolgen können.

![Ein Beispiel für lokal gespeicherte Produkte.](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "Ein Beispiel für lokal gespeicherte Produkte.")

IndexedDB-Daten bleiben bestehen, selbst wenn Sie den Browser schließen oder Ihren Computer neu starten. 
Wenn Sie feststellen, dass Ihre Produktdaten aus irgendeinem Grund unsynchronisiert sind, können Sie [den lokalen Speicher löschen](http://faq.wcpos.com/en/clear-local-data.html) und ein frisches Set von Daten vom Server abrufen.
