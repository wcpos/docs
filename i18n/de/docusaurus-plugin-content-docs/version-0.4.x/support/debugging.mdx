:::caution
Dieser Artikel richtet sich an Webentwickler, Fehlerdiagnose ist nicht etwas, das der durchschnittliche Benutzer verwenden sollte.
:::

WCPOS ist größtenteils eine JavaScript-Webanwendung - und als solche - wird die meiste Fehlerdiagnose über die [Browser-Konsole](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors) durchgeführt.
Wenn die Fehlerdiagnose aktiviert ist, protokolliert WCPOS verschiedene Nachrichten in der Konsole, die bei der Diagnose von Problemen helfen können.

![Eine geöffnete JavaScript-Konsole in Chrome für Mac](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Eine geöffnete JavaScript-Konsole in Chrome für Mac")

### Fehlerdiagnose in wp-config.php aktivieren

Durch das Setzen des [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) Flags in Ihrer wp-config.php-Datei können Sie die WCPOS-Fehlerdiagnose aktivieren.
Dies lädt die nicht-minimierten Skriptdateien und aktiviert Debugmeldungen in der Browser-Konsole.

### Fehlerdiagnose über die Browser-Konsole aktivieren

Wenn Sie keinen Zugriff auf die wp-config.php-Datei haben, können Sie Debugmeldungen aktivieren, indem Sie ein `debug`-Flag zum Local Storage hinzufügen.
Dies kann durch Eingabe von `localStorage.setItem('debug', '*')` in die Konsole erfolgen.
Aktualisieren Sie jetzt die Seite und die Fehlerdiagnose ist aktiviert. Konsolenprotokollierung kann Browserressourcen verbrauchen, daher möchten Sie die Fehlerdiagnose wahrscheinlich ausschalten, sobald Sie fertig sind.
Um die Fehlerdiagnose auszuschalten, geben Sie einfach `localStorage.removeItem('debug')` in die Konsole ein und aktualisieren Sie die Seite.
