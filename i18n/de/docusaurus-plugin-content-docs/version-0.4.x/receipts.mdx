Die POS-Quittungsdruckvorlagen befinden sich in der Datei `includes/views/print/tmpl-receipt.php` sowohl im kostenlosen als auch im Pro-Plugin. Quittungsvorlagen können angepasst werden, indem eine Datei `woocommerce-pos/print/tmpl-receipt.php` in Ihrem Theme-Verzeichnis erstellt wird. Der Code für beide Vorlagen ist unten aufgeführt.

### Pro Quittungsvorlage

Die Pro-Quittungsvorlage ermöglicht zusätzliche Informationen aus der [Stores](./stores)-Administration wie Logo, Geschäftsadresse, Öffnungszeiten und spezielle Nachrichten.

### Anpassung des Quittungsdatums

WCPOS verwendet [moment.js](http://momentjs.com/) zur Lokalisierung der Datumsketten. Das Standarddatumsformat ist `"MMMM Do YYYY, h:mm:ss a"`, z.B.: 31. Mai 2015, 19:20:44. Bitte konsultieren Sie die [moment.js-Dokumentation](http://momentjs.com/docs/#/parsing/string-format/) für weitere Informationen zur Anpassung des Datumsformats.

### Bestellungseigenschaften

WCPOS verwendet die JSON-Ausgabe der WC REST API, um die Quittungsvorlage zu füllen. [Die WC REST API-Docs zeigen ein Beispiel](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order) der Standard-JSON-Ausgabe. Einige zusätzliche Eigenschaften wurden von WCPOS mithilfe des `woocommerce_api_order_response`-Filters hinzugefügt.

| Eigenschaft                        | Beschreibung                                                  |
| ---------------------------------- | ------------------------------------------------------------- |
| cart_discount_tax                 | Der Steueranteil des Warenkorb-Rabatts                        |
| cashier.id                        | Benutzer-ID des Kassiers                                      |
| cashier.first_name                | Vorname des Kassiers                                          |
| cashier.last_name                 | Nachname des Kassiers                                         |
| payment_details.result            | Erfolg oder Misserfolg des Zahlungsdienstes                   |
| payment_details.message           | Nachrichten vom Zahlungsdienst                                |
| payment_details.redirect          | Weiterleitungs-URL für Zahlungen auf externen Seiten, z.B. PayPal Standard |
| payment_details.method\_`gateway` | Gateway-spezifische Nachrichten, z.B. Gezahlter Betrag und Wechselgeld |
| shipping_lines[i].total_tax       | Steuerbeträge für jede Versandposition hinzufügen             |
| subtotal_tax                      | Der Steuerbereich des Zwischensumme                           |
