Der Bestellaktionen-Bereich befindet sich am unteren Ende des Warenkorbs in der POS-Schnittstelle. Er bietet schnellen Zugriff auf mehrere wichtige Funktionen zur Bestellverwaltung.

## Bestellnotiz

<div class="image-container">
  <Image

- Öffnet einen Dialog zum Hinzufügen einer Kundennotiz zur Bestellung.
- Kundennotizen sind für den Kunden sichtbar und erscheinen standardmäßig auf dem Kassenzettel, im Kunden-Bereich "Mein Konto" und in `WP Admin > WooCommerce > Bestellungen > Bestellung` unter der Versandadresse.

:::note
Bestellnotizen sind für Kunden sichtbar.
:::

## Bestellmetadaten

- Öffnet ein Modal zur Verwaltung zusätzlicher Metadaten für die Bestellung.
- Funktionen umfassen:
  - Ändern der Bestellungswährung.
  - Hinzufügen einer Transaktions-ID (z.B. von einem externen Zahlungsterminal).
  - Hinzufügen benutzerdefinierter Metadaten für die Bestellung.

:::note
Über die Zeit werden weitere Funktionen zu diesem Modal hinzugefügt, einschließlich Integration mit anderen WooCommerce-Plugins.
:::

### Was sind Metadaten in WooCommerce?

Metadaten liefern zusätzliche Informationen über Bestellungen und sind über die WooCommerce REST API zugänglich. Entwickler und Plugins nutzen diese Daten, um Funktionalität zu erweitern, wie z.B. die Integration von Drittanbieter-Diensten.

### JSON-Ansicht

- Zeigt die rohe JSON-Darstellung der Bestellung, nützlich zum Debuggen oder für Integrationszwecke.

## Auf Server speichern

- Ermöglicht das Speichern offener Warenkörbe auf dem Server, sodass sie sicher aufbewahrt und für die zukünftige Verwendung zugänglich sind.
- Erstellt eine WooCommerce-Bestellung mit dem Status `pos-open`.

### Warum speichern?

Wenn eine neue Bestellung in der POS erstellt wird, existiert sie zunächst nur lokal. 
Das Speichern der Bestellung stellt sicher, dass sie erhalten bleibt, selbst wenn die lokale Datenbank gelöscht wird.

Weitere Details zur Verwaltung gespeicherter Bestellungen finden Sie unter [Offene Bestellungen](/pos/cart/open-orders).

## Stornieren

- Zwei Szenarien:
  - Nicht gespeicherte Bestellungen:
    - Wird dauerhaft aus der lokalen Datenbank gelöscht.
  - Gespeicherte Bestellungen:
    - Wird in den Papierkorb von WooCommerce verschoben und aus der lokalen Datenbank gelöscht.
- So stellen Sie eine stornierte Bestellung wieder her:
  - Gehen Sie zu `WP Admin > WooCommerce > Bestellungen > Papierkorb`.

## Kasse

- Speichert die Bestellung auf dem Server und startet den Kassenprozess.
- Öffnet den [Kassenbildschirm](/pos/checkout) zum Abschluss der Zahlung.
