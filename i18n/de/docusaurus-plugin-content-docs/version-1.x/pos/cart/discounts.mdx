WCPOS bietet verschiedene Möglichkeiten, einen Auftrag spontan zu rabattieren: schnelle prozentuale Schaltflächen, direkte Preisänderungen einzelner Positionen und Rabattgebühren auf Bestellebene. Informationen zu vorkonfigurierten Aktionen mit Nutzungsregeln finden Sie unter [Gutscheine](/coupons) (Pro).

:::info Geändert in v1.9.0
Die Darstellung von Änderungen der POS-Preise in den WooCommerce-Rabattsummen wurde mit v1.9.0 geändert. Aktuelle Belegvorlagen können den ursprünglichen Preis und die Ersparnis im POS getrennt anzeigen, während die native Summe **Rabatt** weiterhin für Gutscheine reserviert bleibt. Details finden Sie unten unter [Was sich in v1.9.0 geändert hat](#what-changed-in-v190).
:::

## Schnellrabatte {#quick-discounts}

Mit Schnellrabatt-Schaltflächen kann mit einem einzigen Tippen ein prozentualer Rabatt auf den gesamten Auftrag angewendet werden.

Öffnen Sie zur Konfiguration die [Anzeigeeinstellungen](/pos/cart#display-settings) des Warenkorbs und geben Sie im Feld **Schnellrabatte** eine durch Kommas getrennte Liste von Prozentsätzen ein. Beispielsweise erstellt `5,10,15,20` vier Schnellzugriffsschaltflächen.

Wenn Sie auf eine Schnellrabatt-Schaltfläche tippen, wird der Prozentsatz auf alle Positionen im Warenkorb angewendet.

## Rabatte auf Positionsebene {#line-item-discounts}

Der Preis jeder einzelnen Position kann direkt im Warenkorb geändert werden:

1. Auf das Feld **Preis** der Position klicken
2. Den neuen Preis eingeben
3. Zum Bestätigen **Eingabetaste** drücken

Dies ist nützlich für Preisanpassungen, Mitarbeiterrabatte oder einmalige Änderungen. Der Gesamtbetrag der Position wird automatisch anhand von Menge × neuem Preis aktualisiert.

Weitere Informationen zum Bearbeiten von Warenkorbpositionen unter [Positionen](/pos/cart/line-items).

:::tip Positionen aufteilen
Wenn ein Kunde unterschiedliche Rabatte auf Teile desselben Produkts erhalten soll (z. B. 3 zum vollen Preis und 2 reduziert), kann die Option **Aufteilen** in den [Anzeigeeinstellungen](/pos/cart#display-settings) des Warenkorbs aktiviert werden, um eine Position in separate Zeilen aufzuteilen.
:::

## Rabatte auf Bestellungsebene {#order-level-discounts}

Um einen festen Rabatt auf die gesamte Bestellung anzuwenden (statt auf einzelne Positionen), eine **negative Gebühr** hinzufügen:

1. Unterhalb der Warenkorbpositionen auf **Gebühr hinzufügen** klicken
2. Geben Sie einen Namen ein (z. B. „Mitarbeiterrabatt“)
3. Geben Sie den Rabattbetrag als negative Zahl ein (z. B. `-5.00`)

Die Gebühr wird als separate Position angezeigt und verringert die Bestellsumme. Bei Bedarf kann der Steuerstatus der Gebühr über das Drei-Punkte-Menü bearbeitet werden.

## POS-Rabatte im Vergleich zu WooCommerce-Gutscheinen {#pos-discounts-vs-woocommerce-coupons}

Die Rabatte auf dieser Seite sind spontane Anpassungen, die Kassierer an der Kasse anwenden; WooCommerce-**Gutscheine** sind vorab konfigurierte Aktionen mit Regeln und Nachverfolgung. Hier ein kurzer Vergleich:

| | POS-Rabatte | WooCommerce-Gutscheine (Pro) |
|---|---|---|
| **Anwendung** | Schneller Rabatt, Bearbeitung des Positionspreises oder negative Gebühr | Gutscheincode im Warenkorb eingeben |
| **Konfiguration** | Direkt durch den Kassierer | Vorab konfiguriert unter **WP Admin → Marketing → Gutscheine** |
| **Nachverfolgung** | Als Positionspreis erfasst (siehe [Änderung in v1.9.0](#what-changed-in-v190)) | Als Gutscheinrabatt in WooCommerce-Berichten nachverfolgt |
| **Einschränkungen** | Keine — der Kassierer legt jeden beliebigen Preis fest | Nutzungslimits, Produkt-/Kategorieeinschränkungen, Mindestbestellwert, Ablaufdatum, E-Mail-Regeln |
| **Am besten geeignet für** | Spontane Anpassungen, Preisanpassungen | Strukturierte Aktionen, nachverfolgbare Rabatte |

:::tip Was sollte verwendet werden?
Für einmalige Preisanpassungen sind die Rabatte auf dieser Seite einfacher. Wenn die Nutzung von Rabatten in WooCommerce-Berichten nachverfolgt oder Regeln wie Nutzungslimits durchgesetzt werden müssen, verwenden Sie [Gutscheine](/coupons).
:::

## Wie Preisänderungen im POS mit Gutscheinen interagieren {#how-pos-price-changes-interact-with-coupons}

Wenn ein Kassenmitarbeiter einen benutzerdefinierten Preis für eine Position festlegt (z. B. von $18 auf $16 reduziert) und anschließend ein Gutschein angewendet wird, wird der Gutschein auf Basis des **im POS rabattierten Preises** ($16) und nicht des ursprünglichen Preises ($18) berechnet. Dadurch erhalten Kunden nicht durch die Kombination eines Kassenrabatts mit einem Gutschein auf Basis des ursprünglichen Preises einen zu hohen Rabatt.

- Im POS rabattierte Artikel werden von WooCommerce als „im Angebot“ behandelt. Wenn für einen Gutschein **Angebotsartikel ausschließen** aktiviert ist, werden im POS rabattierte Artikel übersprungen, genau wie reguläre Angebotsartikel. Entwickler können dies mit dem Filter `woocommerce_pos_item_is_on_sale` überschreiben.
- Beim Entfernen eines Gutscheins bleibt die Position beim im POS rabattierten Preis.

:::note Entwicklerreferenz
Technische Details dazu, wie Preisüberschreibungen im POS gespeichert werden, sowie die verfügbaren Filter finden Sie in der [POS-Rabattreferenz](/reference/pos-discounts).
:::

## Änderungen in v1.9.0 {#what-changed-in-v190}

Wenn Sie von v1.8 aktualisiert haben und festgestellt haben, dass die Summe **Rabatt** auf Ihren Belegen und in Ihren Berichten jetzt **0** beträgt, erklärt dieser Abschnitt warum und welche Optionen verfügbar sind.

### Die Änderung {#the-change}

Vor v1.9.0 speicherte das POS eine Bestellung, wenn eine Kassenkraft den Preis einer Position reduzierte (z. B. $18 → $16), mit `subtotal = $18` und `total = $16`. WooCommerce berechnete dann `discount_total = $2`, was auf dem Beleg und in Berichten als Rabatt erschien.

Dadurch entstanden falsche Summen, wenn zusätzlich ein Gutschein angewendet wurde — der Gutschein wurde auf Grundlage der ursprünglichen $18 berechnet, was zu einem zu hohen Rabatt und einer zu niedrigen Belastung des Kunden führte.

Ab v1.9.0 richtet sich WCPOS danach, wie WooCommerce Angebotspreise behandelt: Der an der Kasse festgelegte Preis **ist** die Zwischensumme der Position. WooCommerce zählt nur Gutscheincodes als „Rabatte“ (dies entspricht dem Verhalten von WC bei reduzierten Produkten, bei denen der Angebotspreis ebenfalls ohne separate Rabattzeile in der Zwischensumme enthalten ist).

### Das bedeutet dies {#what-this-means-for-you}

- **Belege** können den gespeicherten regulären Preis, die Ersparnis zwischen regulärem Preis und Verkaufspreis sowie einen kombinierten Wert **Gesamtersparnis** für diese Ersparnisse und Gutscheine anzeigen. Preisersparnisse bleiben vom WooCommerce-Feld **Rabatt** getrennt.
- **Berichte** zeigen `discount_total = 0`, wenn nur Preisänderungen von POS-Positionen verwendet wurden. Es werden nur Gutscheinrabatte gezählt.
- **Gutscheine** werden jetzt korrekt berechnet, wenn sie mit POS-reduzierten Artikeln kombiniert werden.
- **Die gespeicherten regulären Preise und Verkaufspreise werden in den Metadaten der POS-Positionen gespeichert** (`_woocommerce_pos_data`), sodass der Beleg die Ersparnis ermitteln kann, ohne die Rabattsummen von WooCommerce zu ändern.

### Ursprünglichen Preis und Ersparnis auf Belegen anzeigen {#showing-the-original-price-and-saving-on-receipts}

Die aktuellen gebündelten Vorlagen mit Preisangaben zeigen den gespeicherten regulären Preis und die Ersparnis an, wenn ein Produkt reduziert ist oder sein Preis an der Kasse geändert wird. Sie zeigen außerdem **Gesamtersparnis**, wenn WCPOS einen vollständigen Wert auf Bestellungsebene berechnen kann. Dieser Gesamtwert kombiniert Ersparnisse zum regulären Preis mit Gutscheinrabatten, ohne einen davon doppelt zu zählen.

Aus der Galerie kopierte Vorlagen sind bearbeitbare Momentaufnahmen und werden durch WCPOS-Updates nicht überschrieben. Wenn der Beleg vor dieser Änderung erstellt wurde, stehen folgende Optionen zur Verfügung:

1. Klicken Sie bei einer neuen Kopie der gebündelten Vorlage auf **Vorlage verwenden**; oder
2. Aktualisieren Sie Ihre bestehende Vorlage mit den Feldern in der [Belegdatenreferenz](/receipts/receipt-data#displaying-regular-price-and-savings).

Die wichtigsten Belegfelder sind:

| Feld | Bedeutung |
|-------|---------|
| `lines[].regular_price_display` | Erfasster regulärer Stückpreis |
| `lines[].selling_price_display` | Verkaufspreis pro Einheit vor Gutscheinen |
| `lines[].unit_savings_display` | Ersparnis pro Artikel |
| `lines[].line_regular_total_display` | Regulärer Gesamtpreis der Position |
| `lines[].line_savings_display` | Gesamtersparnis der Position |
| `totals.sale_savings_total_display` | Gesamtersparnis zwischen regulärem Preis und Verkaufspreis aus Katalog-Angeboten und Preisänderungen an der Kasse |
| `totals.discount_total_display` | WooCommerce-Rabattsumme; normalerweise Gutscheine für aktuelle Bestellungen |
| `totals.total_saved_display` | Kombinierte Ersparnis aus regulärem Preis und WooCommerce-Rabatten |
| `totals.total_saved_complete` | Gibt an, ob die kombinierte Gesamtersparnis vollständig ist und sicher angezeigt werden kann |

Benutzerdefinierte Vorlagen sollten die Zeile **Gesamt gespart** absichern, sodass sie ausgeblendet wird, wenn historische Preisdaten unvollständig sind oder nichts gespart wurde — das genaue Muster finden Sie in der [Belegdatenreferenz](/receipts/receipt-data#totals).

WooCommerce-Berichte bleiben unverändert: Die nativen Rabattsummen enthalten weiterhin Gutscheine statt aktueller Einsparungen durch Angebotspreise oder Kassenpreise. Verwenden Sie für Belege `totals.total_saved` und weiterhin [Gutscheine](/coupons), wenn Einsparungen in Berichten auch als WooCommerce-Rabatte erfasst werden müssen.

## Bekannte Einschränkungen {#known-limitations}

- **Keine automatischen Rabattregeln** — das POS unterstützt keine automatischen Rabatte im Stil von "Kaufe 2, erhalte 1 gratis". Verwenden Sie WooCommerce-Gutscheine für strukturierte Werbeaktionen.
- **Schnelle Rabatte sind nur prozentual** — es gibt keine integrierte Schnellschaltfläche für Rabatte mit festem Betrag. Verwenden Sie stattdessen eine negative Gebühr oder bearbeiten Sie einzelne Preise.
