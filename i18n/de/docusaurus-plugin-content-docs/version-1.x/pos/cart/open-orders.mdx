WCPOS ermöglicht es Ihnen, gleichzeitig mit mehreren Bestellungen zu arbeiten. Dies ist nützlich für die Bearbeitung von Kundenanfragen, den Wechsel zwischen Transaktionen und die Wiederherstellung nach Unterbrechungen.

## Karussell Offene Bestellungen {#open-orders-carousel}

Am unteren Ende des Einkaufswagenpanels wird ein horizontales Karussell mit allen offenen Bestellungen angezeigt:

{/* Screenshot: open-orders-carousel.png */}

- Jeder Einkaufswagen zeigt den **Gesamtbetrag**
- Die **aktuelle Bestellung** ist hervorgehoben
- Klicken Sie auf einen beliebigen Einkaufswagen, um zu ihm zu wechseln
- Scrollen Sie nach links/rechts, um weitere Einkaufswagen zu sehen

## Eine Neue Bestellung Erstellen {#creating-a-new-order}

Ein neuer leerer Einkaufswagen ist immer verfügbar. Klicken Sie einfach auf einen leeren Einkaufswagen im Karussell oder beginnen Sie, Produkte hinzuzufügen, wenn der aktuelle Einkaufswagen leer ist.

## Wechsel zwischen Bestellungen {#switching-between-orders}

Klicken Sie auf eine beliebige Bestellung im Karussell, um zu ihr zu wechseln. Das Einkaufswagenpanel wird aktualisiert, um den Inhalt der ausgewählten Bestellung anzuzeigen.

**Anwendungsfälle:**

- Kunde tritt zurück, um einen weiteren Artikel zu holen
- Schnelle Hilfe für einen Kunden, während eine große Bestellung bearbeitet wird
- Preise oder Artikel zwischen Bestellungen vergleichen

## Bestellungen Auf Server Speichern {#saving-orders-to-server}

Bestellungen existieren in zwei Zuständen:

### Nur Lokal {#local-only}

Standardmäßig werden neue Bestellungen nur in der lokalen Browser-/App-Datenbank gespeichert. Sie bleiben bei Seitenaktualisierungen erhalten, aber:

- Sind nicht im WooCommerce-Admin sichtbar
- Gehen verloren, wenn die lokale Datenbank gelöscht wird
- Sind von anderen Geräten aus nicht zugänglich

### Auf Server Gespeichert {#saved-to-server}

Klicken Sie auf **Auf Server speichern**, um eine WooCommerce-Bestellung mit dem Status `pos-open` zu erstellen. Dies:

- Erstellt eine echte Bestellung in WooCommerce
- Bleibt bestehen, auch wenn die lokale Datenbank gelöscht wird
- Kann von anderen Geräten aus zugegriffen werden
- Erscheint in WP Admin > WooCommerce > Bestellungen

:::tip Wann speichern
Speichern Sie Bestellungen auf dem Server, wenn:
- Ein Kunde eine Bestellung für die spätere Abholung zurückhalten möchte
- Sie Ihre Schicht beenden und ein anderer Kassierer fortfahren wird
- Sie eine Sicherungskopie für den Fall von App-/Browserproblemen möchten
:::

## Wiederherstellen Gespeicherter Bestellungen {#recovering-saved-orders}

Wenn Sie Bestellungen auf dem Server gespeichert haben, können sie wieder aufgerufen werden durch:

1. Öffnen des **Bestellungen**-Bildschirms (Pro-Funktion)
2. Filtern nach Status `pos-open`
3. Wiederöffnen der Bestellung

## Bestellpersistenz {#order-persistence}

### Lokaler Speicher {#local-storage}

WCPOS verwendet IndexedDB, um Bestellungen lokal zu speichern. Dies bietet:

- Persistenz über Browsersitzungen hinweg
- Schnellen Zugriff ohne Netzwerkrequests
- Offline-Funktionalität

### Synchronisation mit Server {#sync-with-server}

Wenn Sie auf den Server speichern oder auschecken:

- Wird die Bestellung an WooCommerce gesendet
- Eine Bestätigung wird empfangen
- Lokale und Servicedaten werden synchronisiert

## Bestellungen Stornieren {#voiding-orders}

Um eine offene Bestellung zu entfernen:

1. Wechseln Sie zu der Bestellung, die Sie entfernen möchten
2. Klicken Sie auf die Schaltfläche **Stornieren**

**Was passiert:**

- **Nicht gespeicherte Bestellungen:** Permanent aus der lokalen Datenbank gelöscht
- **Gespeicherte Bestellungen:** In den Papierkorb von WooCommerce verschoben und lokal gelöscht

Um eine stornierte gespeicherte Bestellung wiederherzustellen:
1. Gehen Sie zu `WP Admin > WooCommerce > Bestellungen > Papierkorb`
2. Stellen Sie die Bestellung wieder her

## Tipps {#tips}

### Bestellungen Organisiert Halten {#keep-orders-organized}

Bei mehreren offenen Bestellungen ist es hilfreich, zu:

- Kundennamen zu Bestellungen hinzuzufügen, um die Identifizierung zu erleichtern
- Bestellnotizen hinzuzufügen, die den Grund für die Zurückhaltung beschreiben
- Wichtige Bestellungen auf dem Server zu speichern

### Schichtübergaben {#shift-handoffs}

Beim Beenden einer Schicht mit offenen Bestellungen:

1. Speichern Sie alle wichtigen Bestellungen auf dem Server
2. Fügen Sie Bestellnotizen hinzu, die den Status erläutern
3. Der nächste Kassierer kann von der Bestellansicht darauf zugreifen

### Offline Überlegungen {#offline-considerations}

Wenn Sie die Verbindung verlieren:

- Lokale Bestellungen bleiben zugänglich und Sie können weiterhin Artikel hinzufügen
- Sie können Bestellungen nicht abschließen/ausschecken, bis die Verbindung wiederhergestellt ist
- Sie können Bestellungen nicht auf dem Server speichern, bis die Verbindung wiederhergestellt ist
- Sie können keine neuen Kunden erstellen, bis die Verbindung wiederhergestellt ist
