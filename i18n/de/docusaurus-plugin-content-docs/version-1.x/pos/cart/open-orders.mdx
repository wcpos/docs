WCPOS ermöglicht es Ihnen, mit mehreren Bestellungen gleichzeitig zu arbeiten. Dies ist nützlich für Kundenreservierungen, Transaktionswechsel und Wiederherstellung nach Unterbrechungen.

## Karussell für offene Bestellungen

Am unteren Rand des Warenkorb-Panels befindet sich ein horizontales Karussell mit allen offenen Bestellungen:

{/* Screenshot: open-orders-carousel.png */}

- Jeder Warenkorb zeigt den **Gesamtbetrag**
- Die **aktuelle Bestellung** wird hervorgehoben
- Klicken Sie auf einen Warenkorb, um zu ihm zu wechseln
- Scrollen Sie nach links/rechts, um weitere Warenkörbe zu sehen

## Erstellen einer neuen Bestellung

Ein neuer leerer Warenkorb ist immer verfügbar. Klicken Sie einfach auf einen leeren Warenkorb im Karussell oder beginnen Sie mit dem Hinzufügen von Produkten, wenn der aktuelle Warenkorb leer ist.

## Wechseln zwischen Bestellungen

Klicken Sie auf eine Bestellung im Karussell, um zu ihr zu wechseln. Das Warenkorb-Panel aktualisiert sich, um den Inhalt der ausgewählten Bestellung zu zeigen.

**Anwendungsfälle:**

- Kunde holt einen weiteren Artikel
- Hilfe für einen schnellen Kunden während einer großen Bestellung
- Vergleichen von Preisen oder Artikeln zwischen Bestellungen

## Speichern von Bestellungen auf dem Server

Bestellungen existieren in zwei Zuständen:

### Nur lokal

Standardmäßig werden neue Bestellungen nur in der lokalen Browser-/App-Datenbank gespeichert. Sie bleiben über Seitenaktualisierungen hinweg erhalten, aber:

- Sind nicht im WooCommerce-Admin sichtbar
- Gehen verloren, wenn die lokale Datenbank gelöscht wird
- Sind nicht von anderen Geräten aus zugänglich

### Auf Server gespeichert

Klicken Sie auf **Auf Server speichern**, um eine WooCommerce-Bestellung mit dem Status `pos-open` zu erstellen. Dies:

- Erstellt eine echte Bestellung in WooCommerce
- Bleibt erhalten, auch wenn die lokale Datenbank gelöscht wird
- Kann von anderen Geräten aus zugegriffen werden
- Erscheint in WP Admin > WooCommerce > Bestellungen

:::tip Wann speichern
Speichern Sie Bestellungen auf dem Server, wenn:
- Ein Kunde eine Bestellung zur späteren Abholung reservieren möchte
- Sie Ihre Schicht beenden und eine andere Kassiererin weitermacht
- Sie ein Backup für den Fall von App-/Browser-Problemen benötigen
:::

## Wiederherstellen gespeicherter Bestellungen

Wenn Sie Bestellungen auf dem Server gespeichert haben, können Sie sie erneut aufrufen durch:

1. Öffnen des **Bestellungen**-Bildschirms (Pro-Funktion)
2. Filtern nach Status `pos-open`
3. Wiedereröffnen der Bestellung

## Bestellungspersistenz

### Lokaler Speicher

WCPOS verwendet IndexedDB zum lokalen Speichern von Bestellungen. Dies bietet:

- Persistenz über Browsersitzungen hinweg
- Schnellen Zugriff ohne Netzwerkanfragen
- Offline-Fähigkeit

### Synchronisation mit Server

Wenn Sie auf Server speichern oder zur Kasse gehen:

- Wird die Bestellung an WooCommerce gesendet
- Eine Bestätigung wird empfangen
- Lokale und Server-Daten werden synchronisiert

## Stornieren von Bestellungen

Um eine offene Bestellung zu entfernen:

1. Wechseln Sie zu der Bestellung, die Sie entfernen möchten
2. Klicken Sie auf die Schaltfläche **Stornieren**

**Was passiert:**

- **Ungespeicherte Bestellungen:** Dauerhaft aus der lokalen Datenbank gelöscht
- **Gespeicherte Bestellungen:** In den Papierkorb von WooCommerce verschoben und lokal gelöscht

So stellen Sie eine stornierte gespeicherte Bestellung wieder her:
1. Gehen Sie zu `WP Admin > WooCommerce > Bestellungen > Papierkorb`
2. Stellen Sie die Bestellung wieder her

## Tipps

### Bestellungen organisiert halten

Bei mehreren offenen Bestellungen hilft es:

- Kundennamen zu Bestellungen hinzuzufügen für einfache Identifikation
- Bestellnotizen mit Grund für Reservierung hinzufügen
- Wichtige Bestellungen auf dem Server speichern

### Schichtwechsel

Beim Beenden einer Schicht mit offenen Bestellungen:

1. Alle wichtigen Bestellungen auf dem Server speichern
2. Bestellnotizen mit Statuserklärung hinzufügen
3. Die nächste Kassiererin kann sie vom Bestellungsbildschirm aus aufrufen

### Offline-Überlegungen

Bei Verbindungsverlust:

- Lokale Bestellungen bleiben zugänglich und Sie können weiter Artikel hinzufügen
- Sie können Bestellungen nicht abschließen/zur Kasse gehen, bis die Verbindung wiederhergestellt ist
- Sie können keine Bestellungen auf den Server speichern, bis eine Verbindung besteht
- Sie können keine neuen Kunden erstellen, bis eine Verbindung wiederhergestellt ist
