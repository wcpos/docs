WCPOS ermöglicht es Ihnen, mit mehreren Bestellungen gleichzeitig zu arbeiten. Dies ist nützlich für die Handhabung von Kundenwartezeiten, das Wechseln zwischen Transaktionen und die Wiederherstellung nach Unterbrechungen.

## Karussell für Offene Bestellungen

Am unteren Rand des Warenkorb-Panels wird ein horizontales Karussell mit allen offenen Bestellungen angezeigt:

{/* Screenshot: open-orders-carousel.png */}

- Jeder Warenkorb zeigt seinen **Gesamtbetrag**
- Die **aktuelle Bestellung** ist hervorgehoben
- Klicken Sie auf einen beliebigen Warenkorb, um zu ihm zu wechseln
- Scrollen Sie nach links/rechts, um mehr Warenkörbe zu sehen

## Erstellen einer Neuen Bestellung

Ein neuer leerer Warenkorb ist immer verfügbar. Klicken Sie einfach auf einen leeren Warenkorb im Karussell oder beginnen Sie mit dem Hinzufügen von Produkten, wenn der aktuelle Warenkorb leer ist.

## Wechseln zwischen Bestellungen

Klicken Sie auf eine beliebige Bestellung im Karussell, um zu ihr zu wechseln. Das Warenkorb-Panel aktualisiert sich, um den Inhalt der ausgewählten Bestellung anzuzeigen.

**Anwendungsfälle:**

- Ein Kunde geht weg, um einen weiteren Artikel zu holen
- Sie müssen einem schnellen Kunden helfen, während eine große Bestellung in Bearbeitung ist
- Preise oder Artikel zwischen Bestellungen vergleichen

## Speichern von Bestellungen auf dem Server

Bestellungen existieren in zwei Zuständen:

### Nur Lokal

Standardmäßig werden neue Bestellungen nur in der lokalen Datenbank des Browsers/der App gespeichert. Sie bleiben bei Seitenaktualisierungen bestehen, aber:

- Sind im WooCommerce-Admin nicht sichtbar
- Gehen verloren, wenn die lokale Datenbank gelöscht wird
- Sind von anderen Geräten aus nicht zugänglich

### Auf dem Server gespeichert

Klicken Sie auf **Auf Server speichern**, um eine WooCommerce-Bestellung mit dem Status `pos-open` zu erstellen. Dies:

- Erstellt eine echte Bestellung in WooCommerce
- Besteht auch, wenn die lokale Datenbank gelöscht wird
- Kann von anderen Geräten aus abgerufen werden
- Erscheint in WP Admin > WooCommerce > Bestellungen

:::tip Wann speichern
Speichern Sie Bestellungen auf dem Server, wenn:
- Ein Kunde möchte eine Bestellung für die spätere Abholung halten
- Sie Ihre Schicht beenden und ein anderer Kassierer fortfahren wird
- Sie ein Backup im Falle von App-/Browserproblemen wünschen
:::

## Wiederherstellen Gespeicherter Bestellungen

Wenn Sie Bestellungen auf dem Server gespeichert haben, können sie wieder abgerufen werden durch:

1. Öffnen Sie den Bildschirm **Bestellungen** (Pro-Funktion)
2. Filtern Sie nach Status `pos-open`
3. Öffnen Sie die Bestellung erneut

## Persistenz von Bestellungen

### Lokaler Speicher

WCPOS verwendet IndexedDB, um Bestellungen lokal zu speichern. Dies bietet:

- Persistenz über Browsersitzungen hinweg
- Schnellen Zugriff ohne Netzwerkabfragen
- Offline-Fähigkeit

### Synchronisierung mit dem Server

Wenn Sie auf dem Server speichern oder den Checkout durchführen:

- Wird die Bestellung an WooCommerce gesendet
- Eine Bestätigung wird empfangen
- Lokale und Serverdaten werden synchronisiert

## Stornieren von Bestellungen

Um eine offene Bestellung zu entfernen:

1. Wechseln Sie zu der Bestellung, die Sie entfernen möchten
2. Klicken Sie auf die Schaltfläche **Stornieren**

**Was passiert:**

- **Nicht gespeicherte Bestellungen:** Dauerhaft aus der lokalen Datenbank gelöscht
- **Gespeicherte Bestellungen:** In den Papierkorb in WooCommerce verschoben und lokal gelöscht

Um eine stornierte gespeicherte Bestellung wiederherzustellen:
1. Gehen Sie zu `WP Admin > WooCommerce > Bestellungen > Papierkorb`
2. Stellen Sie die Bestellung wieder her

## Tipps

### Bestellungen Organisiert Halten

Bei mehreren offenen Bestellungen hilft es,:

- Kundennamen zu Bestellungen hinzuzufügen, um die Identifizierung zu erleichtern
- Bestellnotizen hinzuzufügen, die den Grund für die Speicherung beschreiben
- Wichtige Bestellungen auf dem Server zu speichern

### Schichtübergaben

Wenn Sie Ihre Schicht mit offenen Bestellungen beenden:

1. Speichern Sie alle wichtigen Bestellungen auf dem Server
2. Fügen Sie Bestellnotizen hinzu, die den Status erklären
3. Der nächste Kassierer kann auf sie über den Bestellbildschirm zugreifen

### Offline-Überlegungen

Wenn Sie die Verbindung verlieren:

- Lokale Bestellungen bleiben zugänglich und Sie können weiterhin Artikel hinzufügen
- Sie können Bestellungen nicht abschließen/auschecken, bis die Verbindung wiederhergestellt ist
- Sie können Bestellungen nicht auf dem Server speichern, bis Sie wieder verbunden sind
- Sie können keine neuen Kunden erstellen, bis Sie wieder verbunden sind
