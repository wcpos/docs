Muss Geld an der Kasse zurückgegeben werden? Mit WCPOS kann eine WooCommerce-Bestellung zurückerstattet werden, ohne den POS zu verlassen — vollständig oder teilweise, zurück auf die ursprüngliche Zahlungsmethode oder als Bargeld aus der Kasse.

:::info Pro-Funktion
Das Ausstellen von Rückerstattungen aus dem POS erfordert [WCPOS Pro](/getting-started/pro-license). Ohne Pro erfolgt die Rückerstattung über `WP Admin → WooCommerce → Orders`.
:::

## Kurzfassung {#the-short-version}

1. **Rückerstattungsformular öffnen** — in der Bestellliste beim Auftrag auf das Drei-Punkte-Menü klicken und **Rückerstatten** auswählen; alternativ die Bestellung öffnen und in der Fußzeile auf **Rückerstatten** klicken.
2. **Mengen festlegen** — für jede Position eine **Rückerstattungsmenge** eingeben (für die Rückerstattung der gesamten Bestellung jede Position auf die volle Menge setzen, für eine teilweise Rückerstattung nur einige Positionen). Bei Bedarf optional einen **individuellen Betrag** und einen **Grund** hinzufügen.
3. **Auswählen, wohin das Geld zurückgeht:**
   - **An *(gateway)* rückerstatten** — sendet den Betrag zurück an die ursprüngliche Karte oder Wallet, sofern das Zahlungsgateway dies unterstützt (z. B. Stripe Terminal, Vipps MobilePay).
   - **Rückerstattung in bar** — erfasst Bargeld, das aus der Kasse zurückgegeben wurde. Wird automatisch für Barverkäufe verwendet.
4. **Bestätigen** — drücken Sie **Rückerstattung verarbeiten** und bestätigen Sie den Betrag. Die Bestellung wird sofort aktualisiert.

Rückerstattungen erfordern eine aktive Serververbindung — anders als der Checkout können sie nicht offline in die Warteschlange gestellt werden.

## Alle Details? {#want-the-full-details}

Der vollständige Leitfaden behandelt, welche Bestellstatus erstattet werden können, wie das Rückerstattungsformular Steuern berechnet, die Entscheidungsregeln für Gateway oder Bargeld, was danach mit Belegen und Bestellstatus passiert, sowie das Audit-Protokoll für Kassierer und Filiale.
