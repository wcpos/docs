## Produktsuche

### Einheitliches Suchfeld

WCPOS verfügt über ein einziges Suchfeld, das gleichzeitig in mehreren Produktattributen sucht:

- **Produktname** - Durchsucht den Produkttitel und die Beschreibung
- **SKU** - Entspricht dem Produktpostmeta-Feld `_sku`
- **Barcode** - Durchsucht das konfigurierte Barcode-Feld, das ein beliebiges Postmeta-Feld sein kann. Die Legacy-Einstellung ist `_sku`, aber dies wird in Zukunft in `_global_unique_id` geändert.

Geben Sie einfach Ihren Suchbegriff in das Feld „Produkte suchen“ ein, und das POS filtert sofort die Ergebnisse in all diesen Feldern.

### Tokenisierte Suchtechnologie

Die Suchfunktionalität verwendet die [FlexSearch-Bibliothek](https://github.com/nextapps-de/flexsearch) mit fortschrittlichen Tokenisierungsfähigkeiten:

- **Vorwärts-Tokenisierung** - Passt Teilwörter vom Anfang an (z.B. führt die Suche nach "blau" zu "Blaubeere")
- **Leistungsoptimiert** - Verwendet ein Leistungs-Präset für schnelle Suchergebnisse
- **Sprachbewusst** - Passt sich der konfigurierten Sprache Ihres Shops an
- **Lazy Initialization** - Optimiert die Speicherauslastung, indem Suchindizes nur bei Bedarf geladen werden

### Funktionsweise der Suche

Wenn Sie im Suchfeld tippen, macht das POS folgendes:

1. **Tokenisiert** Ihre Eingabe in durchsuchbare Begriffe
2. **Durchsucht lokal** gespeicherte Produktdaten zuerst nach Instant-Ergebnissen
3. **Fragt den Server an**, wenn keine lokalen Übereinstimmungen gefunden werden, und lädt dann neue Produkte für zukünftige Suchen herunter und speichert sie
4. **Aktualisiert Ergebnisse** in Echtzeit, während Sie tippen

Dieser Ansatz gewährleistet eine schnelle Suchleistung, während gleichzeitig eine vollständige lokale Produktdatenbank schrittweise aufgebaut wird.

## Produktfilterung

### Filterleiste

Unter dem Suchfeld finden Sie interaktive Filterwechsel und Dropdown-Menüs, mit denen Sie Produkte anhand spezifischer Kriterien einschränken können.

### Verfügbare Filter

#### Lagerstatus
Filtern Sie Produkte basierend auf ihrem Lagerstatus:
- **Auf Lager** - Produkte mit verfügbarem Lagerbestand
- **Ausverkauft** - Produkte mit null Lagerbestand
- **Rückstand** - Produkte, die zur Rückbestellung verfügbar sind

#### Hervorgehobene Produkte
Umschalten, um nur Produkte anzuzeigen, die in Ihrem WooCommerce-Shop als "Hervorgehoben" markiert sind.

#### Im Angebot befindliche Produkte  
Filter, um nur Produkte anzuzeigen, die derzeit im Angebot sind oder aktive Rabatte haben.

#### Kategorie
Verwenden Sie das Kategoriedropdown, um Produkte nach ihren zugewiesenen Produktkategorien zu filtern. Dies hilft Ihnen, Produkte schnell innerhalb spezifischer Abteilungen oder Produktlinien zu finden.

#### Tag
Filtern Sie nach Produkt-Tags, um Artikel mit spezifischen Attributen oder Merkmalen zu finden, die Sie in Ihrem WooCommerce-Shop definiert haben.

### Verwendung von Filtern

- **Filter umschalten** - Klicken Sie auf eine beliebige Filtertaste, um sie zu aktivieren (aktive Filter erscheinen hervorgehoben)
- **Mehrere Filter** - Sie können mehrere Filter kombinieren, um Ihre Suche weiter einzuschränken
- **Filter löschen** - Klicken Sie erneut auf einen aktiven Filter, um ihn zu deaktivieren
- **Suchen + Filtern** - Verwenden Sie Filter zusammen mit dem Suchfeld für eine präzise Produktsuche

## Barcode-Konfiguration

### Suchfelder

Die Suchfunktionalität schließt automatisch Ihr konfiguriertes Barcode-Feld ein. Das für die Suche verwendete Barcode-Feld hängt von Ihrer POS-Einstellungs-Konfiguration ab.

## F.A.Q.

<details>

<summary>
  Was ist das Feld _global_unique_id für Barcodes?
</summary>

Das Feld `_global_unique_id` ist ein neues Barcode-Feld, das WooCommerce kürzlich hinzugefügt hat, um eine bessere Barcode-Standardisierung über die Läden hinweg zu ermöglichen.

**Wichtige Punkte:**

- **Modernster Standard**: Dieses Feld wurde speziell für die globale Barcode-Identifikation entworfen
- **POS-Konfiguration**: Sie können das POS so konfigurieren, dass es `_global_unique_id` als Barcode-Feld in den POS-Einstellungen verwendet
- **Legacy vs. Neu**: Die Legacy-Barcode-Einstellung verwendet das Feld `_sku`, aber dies wird in zukünftigen Versionen in `_global_unique_id` geändert
- **Flexibilität**: Sie können jedes Produktmeta-Feld als Ihr Barcode-Feld konfigurieren, wenn Sie Drittanbieter-Barcode-Plugins verwenden
- **Künftiger Standard**: `_global_unique_id` wird in zukünftigen POS-Updates zum Standard-Barcodelfeld

Um zu konfigurieren, welches Feld das POS für Barcodes verwendet, besuchen Sie Ihre POS-Einstellungen im WordPress-Adminbereich.

</details>

<details>

<summary>
  Warum sehe ich nicht alle meine Produkte, wenn ich suche?
</summary>

WCPOS verwendet progressive Produktdownloads, um die Leistung aufrechtzuerhalten. Wenn Sie ein Produkt nicht sehen:

1. **Versuchen Sie, danach zu suchen** - Dies wird das POS dazu anregen, danach auf Ihrem Server zu suchen
2. **Warten Sie auf den Download** - Das POS wird das Produkt und andere in kleinen Mengen herunterladen
3. **Suchen Sie erneut** - Sobald heruntergeladen, wird das Produkt in zukünftigen Suchen erscheinen

Dieser Prozess stellt sicher, dass Ihr POS schnell und reaktionsfähig bleibt, selbst bei Tausenden von Produkten. Erfahren Sie mehr darüber in unserem [Produkt-Synchronisation](/products/sync) Leitfaden.

</details>

<details>

<summary>
  Kann ich nach Teilproduktenamen oder SKUs suchen?
</summary>

Ja! Die tokenisierte Suche verwendet eine Vorwärtsübereinstimmung, was bedeutet:

- Die Suche nach "blau" findet Produkte mit "Blaubeere", "blaues Shirt" usw.
- Die Suche nach "ABC" findet SKUs wie "ABC123", "ABC-XYZ" usw.
- Sie müssen keine kompletten Wörter oder Codes eingeben

Die Suche ist so konzipiert, dass Produkte schnell mit minimalem Tippen gefunden werden.

</details>
