Wenn Sie bereit sind, einen Verkauf abzuschließen, klicken Sie auf die Schaltfläche **Checkout**, um das Checkout-Modul zu öffnen. Hier verarbeiten Sie die Zahlung und schließen die Bestellung ab.

{/* Screenshot: checkout-modal.png */}

## Überblick über das Checkout-Modul

Das Checkout-Modul zeigt an:

- **Bestellnummer** - Die WooCommerce-Bestell-ID
- **Zu zahlender Betrag** - Gesamtbetrag fällig
- **Checkout-Einstellungen**-Schaltfläche - Beheben von Anzeigeproblemen
- **Kassierer** - Wer die Bestellung bearbeitet
- **Kunde** - Der Kunde für diese Bestellung (klickbarer Link)
- **Gutschein-Code**-Feld - Rabattcodes anwenden
- **Bestellübersicht** - Produkte, Mengen und Gesamtsummen
- **Zahlungsmethoden** - Verfügbare Zahlungsmöglichkeiten
- **Abbrechen / Zahlung verarbeiten**-Schaltflächen

## Zahlungsmethoden

### Verfügbar in der kostenlosen Version

Die kostenlose Version von WCPOS enthält zwei Zahlungsmethoden:

- **Barzahlung** - Mit eingegebenem Betrag und Wechselgeldberechnung
- **Karte** - Für externe Kartenterminals

### Zusätzliche Gateways (Pro)

:::info Pro Feature
Zusätzliche Zahlungsmethoden erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie aktivieren:

- **Stripe-Terminal** - Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp-Terminal** - Integration mit SumUp-Kartenlesegeräten
- **Benutzerdefinierte Gateways** - Erstellen Sie Ihre eigenen Zahlungsintegrationen

Siehe [Zahlung](/payment) für Details zur Konfiguration von Gateways.

### Auswahl einer Zahlungsmethode

Klicken Sie auf eine Zahlungsmethode, um sie auszuwählen. Das Formular wird aktualisiert, um relevante Felder anzuzeigen:

**Bargeld:**

- **Betrag Barzahlung** - Geben Sie den Betrag ein, den der Kunde Ihnen gibt
- **Wechselgeld** - Automatisch berechnetes Wechselgeld

{/* Screenshot: checkout-cash-payment.png */}

**Karte:**

- Zahlen Sie an Ihrem externen Kartenterminal
- Klicken Sie auf Zahlung verarbeiten, um abzuschließen

## Gutschein-Codes

Um einen Rabatt anzuwenden:

1. Geben Sie den Gutschein-Code in das Feld ein
2. Klicken Sie auf **Gutschein anwenden**
3. Der Rabatt erscheint in der Bestellübersicht

Gutscheine müssen in WooCommerce konfiguriert werden (`WP Admin > Marketing > Gutscheine`).

## Zahlung verarbeiten

1. Wählen Sie eine Zahlungsmethode aus
2. Geben Sie für Bargeld den Betrag ein
3. Klicken Sie auf **Zahlung verarbeiten**
4. Die Bestellung wird abgeschlossen und der [Beleg](/pos/checkout/receipts) wird angezeigt

## Checkout-Einstellungen (Fehlerbehebung)

Das Checkout-Modul verwendet ein iframe/webview, um die WooCommerce Order Pay-Seite anzuzeigen. Dies nutzt die bestehende Zahlungsinfrastruktur von WooCommerce, was bedeutet, dass jedes Zahlungsmethode, die mit WooCommerce funktioniert, im POS funktionieren sollte.

Allerdings können Theme- und Plugin-Skripte manchmal stören. Klicken Sie auf **Checkout-Einstellungen**, um Probleme zu beheben:

### Alle Stile und Skripte deaktivieren

:::warning Nukleare Option
Dies ist die nukleare Option und sollte nur zu Testzwecken oder in seltenen Fällen verwendet werden, in denen der Entwickler weiß, was er tut. Das Deaktivieren aller wp_head-Skripte entfernt sogar die notwendigen WooCommerce-Skripte zum Erweitern/Reduzieren der Zahlungsmethoden, was die Zahlungsfunktionalität beeinträchtigen kann.
:::

- **wp_head deaktivieren** - Entfernt alle Skripte/Stile aus dem WordPress-Header
- **wp_footer deaktivieren** - Entfernt alle Skripte/Stile aus dem WordPress-Footer

### Ausgewählte Stile deaktivieren

Selektiv CSS deaktivieren, das möglicherweise Anzeigeprobleme verursacht:

- wp-emoji-stile
- wp-block-bibliothek
- classic-theme-stile
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-allgemein
- usw.

### Ausgewählte Skripte deaktivieren

Selektiv JavaScript deaktivieren, das mit Zahlungsmethoden interferieren kann:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- usw.

:::tip
Wenn eine Zahlungsmethode nicht korrekt angezeigt wird:

1. Versuchen Sie zuerst, die Theme-Stile zu deaktivieren
2. Versuchen Sie dann, nicht benötigte WooCommerce-Skripte zu deaktivieren
3. Seien Sie vorsichtig, keine Skripte zu deaktivieren, die für Ihre Zahlungsmethode erforderlich sind
   :::

## Abbrechen

Klicken Sie auf **Abbrechen**, um das Checkout-Modul zu schließen, ohne die Bestellung abzuschließen. Die Bestellung bleibt als offener Warenkorb bestehen.
