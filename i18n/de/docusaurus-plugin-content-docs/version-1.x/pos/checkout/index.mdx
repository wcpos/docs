Wenn ein Verkauf abgeschlossen werden soll, klicken Sie auf die Schaltfläche **Checkout**, um das Checkout-Modal zu öffnen. Dort wird die Zahlung verarbeitet und die Bestellung abgeschlossen.

{/* Screenshot: checkout-modal.png */}

## Übersicht über das Checkout-Modal {#checkout-modal-overview}

Das Checkout-Modal zeigt Folgendes an:

- **Bestellnummer** - Die WooCommerce Bestell-ID
- **Zu zahlender Betrag** - Fälliger Gesamtbetrag
- Schaltfläche **Checkout-Einstellungen** - Anzeigeprobleme beheben
- **Kassierer** - Wer die Bestellung verarbeitet
- **Kunde** - Der Kunde für diese Bestellung (anklickbarer Link)
- **Gutschein hinzufügen**-Button - Rabattcodes anwenden
- **Bestellübersicht** - Produkte, Mengen und Summen
- **Zahlungsmethoden** - Verfügbare Zahlungsoptionen
- **Abbrechen**- / **Zahlung verarbeiten**-Buttons

## Zahlungsmethoden {#payment-methods}

### In der kostenlosen Version verfügbar {#available-in-free-version}

Die kostenlose Version von WCPOS enthält zwei Zahlungsgateways:

- **Bargeld** - Mit erhaltenem Betrag und Wechselgeldrechner
- **Karte** - Für externe Kartenterminals

### Zusätzliche Gateways (Pro) {#additional-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie Folgendes aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp Terminal** - Integration mit SumUp-Kartenlesegeräten
- **Benutzerdefinierte Gateways** - Erstellen Sie eigene Zahlungsintegrationen

Weitere Informationen zur Konfiguration von Gateways finden Sie unter [Zahlung](/payment).

### Auswahl einer Zahlungsmethode {#selecting-a-payment-method}

Klicken Sie auf eine Zahlungsmethode, um sie auszuwählen. Das Formular wird aktualisiert und zeigt die relevanten Felder an:

**Bargeld:**

- **Erhaltener Betrag** - Geben Sie den Betrag ein, den der Kunde übergibt
- **Wechselgeld** - Automatisch berechnetes Rückgeld

{/* Screenshot: checkout-cash-payment.png */}

**Karte:**

- Zahlung auf Ihrem externen Kartenterminal verarbeiten
- Zum Abschließen auf Zahlung verarbeiten klicken

## Gutscheine {#coupons}

Der Warenkorb enthält über den Summen ein Eingabefeld **Gutschein hinzufügen** (nur Pro). Geben Sie den Code ein oder suchen Sie nach Beschreibung; der Gutschein wird lokal validiert und als entfernbares Element angezeigt. Mehrere Gutscheine werden nacheinander angewendet.

Den vollständigen Ablauf finden Sie unter **[Gutscheine an der Kasse anwenden](/coupons/applying-coupons)** — Suche, Elemente, aufeinanderfolgende Rabatte sowie die Tabelle mit Validierungsfehlern und Lösungen. Informationen zu Gutscheintypen, Validierungsregeln und Einrichtung in WooCommerce finden Sie unter [Gutscheine](/coupons).

## Zahlung verarbeiten {#processing-payment}

1. Zahlungsmethode auswählen
2. Bei Bargeld den erhaltenen Betrag eingeben
3. Auf **Zahlung verarbeiten** klicken
4. Die Bestellung wird abgeschlossen und der [Beleg](/receipts/at-checkout) wird angezeigt

## Checkout-Einstellungen (Fehlerbehebung) {#checkout-settings-troubleshooting}

Das Checkout-Modal verwendet ein iframe/Webview, um die WooCommerce-Bestellzahlungsseite anzuzeigen. Dadurch wird die vorhandene Zahlungsinfrastruktur von WooCommerce genutzt, sodass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS funktionieren sollte.

Theme- und Plugin-Skripte können jedoch gelegentlich stören. Klicken Sie zur Fehlerbehebung auf **Checkout-Einstellungen**:

### Alle Stile und Skripte deaktivieren {#disable-all-styles-and-scripts}

:::warning Äußerste Option
Dies ist die äußerste Option und sollte nur zu Testzwecken oder in seltenen Fällen verwendet werden, in denen die entwickelnde Person genau weiß, was sie tut. Das Deaktivieren aller wp_head-Skripte entfernt sogar die WooCommerce-Skripte, die zum Ein- und Ausklappen der Zahlungsgateways erforderlich sind, und kann dadurch die Zahlungsfunktion beeinträchtigen.
:::

- **wp_head deaktivieren** - Entfernt alle Skripte/Stile aus dem WordPress-Header
- **wp_footer deaktivieren** - Entfernt alle Skripte/Stile aus dem WordPress-Footer

### Ausgewählte Stile deaktivieren {#disable-selected-styles}

Deaktivieren Sie gezielt CSS, das Anzeigeprobleme verursachen kann:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- usw.

### Ausgewählte Skripte deaktivieren {#disable-selected-scripts}

Deaktivieren Sie gezielt JavaScript, das Zahlungsgateways beeinträchtigen kann:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- usw.

:::tip
Wenn ein Zahlungsgateway nicht korrekt angezeigt wird:

1. Versuchen Sie zuerst, Theme-Stile zu deaktivieren
2. Versuchen Sie anschließend, nicht benötigte WooCommerce-Skripte zu deaktivieren
3. Achten Sie darauf, keine Skripte zu deaktivieren, die von Ihrem Zahlungsgateway benötigt werden
   :::

## Abbrechen {#cancel}

Klicken Sie auf **Abbrechen**, um das Checkout-Modal zu schließen, ohne die Bestellung abzuschließen. Die Bestellung bleibt als offener Warenkorb bestehen.
