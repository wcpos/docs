Wenn Sie bereit sind, einen Verkauf abzuschließen, klicken Sie auf die **Kasse**-Schaltfläche, um das Kassenmodal zu öffnen. Hier verarbeiten Sie die Zahlung und schließen die Bestellung ab.

{/* Screenshot: checkout-modal.png */}

## Übersicht über das Kassenmodal

Das Kassenmodal zeigt an:

- **Bestellnummer** - Die WooCommerce-Bestell-ID
- **Zu zahlender Betrag** - Gesamter fälliger Betrag
- **Kasseneinstellungen**-Schaltfläche - Problemlösungen bei Anzeigeproblemen
- **Kassierer** - Wer die Bestellung bearbeitet
- **Kunde** - Der Kunde für diese Bestellung (anklickbarer Link)
- **Gutschein-Code**-Feld - Rabattcodes anwenden
- **Bestellübersicht** - Produkte, Mengen und Summen
- **Zahlungsmethoden** - Verfügbare Zahlungsoptionen
- **Abbrechen / Zahlung verarbeiten**-Schaltflächen

## Zahlungsmethoden

### In der kostenlosen Version verfügbar

Die kostenlose Version von WCPOS umfasst zwei Zahlungsmethoden:

- **Bargeld** - Mit eingegebener Menge und Wechselgeldberechnung
- **Karte** - Für externe Kartenterminals

### Zusätzliche Zahlungs-Gateways (Pro)

:::info Pro-Feature
Zusätzliche Zahlungsmethoden erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Benutzerdefinierte Gateways** - Erstellen Sie Ihre eigenen Zahlungsintegrationen

Siehe [Zahlung](/payment) für Details zur Konfiguration von Gateways.

### Auswahl einer Zahlungsmethode

Klicken Sie auf eine Zahlungsmethode, um sie auszuwählen. Das Formular wird aktualisiert, um relevante Felder anzuzeigen:

**Bargeld:**

- **Eingegebener Betrag** - Geben Sie den Betrag ein, den der Kunde Ihnen gibt
- **Wechselgeld** - Automatisch berechnetes Wechselgeld

{/* Screenshot: checkout-cash-payment.png */}

**Karte:**

- Zahlung auf Ihrem externen Kartenterminal verarbeiten
- Klicken Sie auf Zahlung verarbeiten, um abzuschließen

## Gutscheincodes

Um einen Rabatt anzuwenden:

1. Geben Sie den Gutscheincode in das Feld ein
2. Klicken Sie auf **Gutschein anwenden**
3. Der Rabatt erscheint in der Bestellübersicht

Gutscheine müssen in WooCommerce konfiguriert werden (`WP Admin > Marketing > Gutscheine`).

## Zahlung verarbeiten

1. Wählen Sie eine Zahlungsmethode aus
2. Geben Sie für Bargeld den eingegebenen Betrag ein
3. Klicken Sie auf **Zahlung verarbeiten**
4. Die Bestellung wird abgeschlossen und der [Beleg](/pos/checkout/receipts) wird angezeigt

## Kassen-Einstellungen (Fehlerbehebung)

Das Kassenmodal verwendet ein iframe/webview, um die WooCommerce-Bestellzahldatei anzuzeigen. Dies nutzt die bestehende Zahlungsinfrastruktur von WooCommerce, was bedeutet, dass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS funktionieren sollte.

Allerdings können Theme- und Plugin-Skripte manchmal stören. Klicken Sie auf **Kasseneinstellungen**, um Fehlersuche zu betreiben:

### Alle Styles und Scripts deaktivieren

:::warning Nukleare Option
Dies ist die nukleare Option und sollte nur zu Testzwecken oder in seltenen Fällen verwendet werden, in denen der Entwickler weiß, was er tut. Das Deaktivieren aller wp_head-Skripte entfernt sogar die WooCommerce-Skripte, die erforderlich sind, um die Zahlungsmethoden zu erweitern oder zu reduzieren, was möglicherweise die Zahlungsfunktionalität beeinträchtigt.
:::

- **wp_head deaktivieren** - Entfernt alle Skripte/Styles aus dem WordPress-Header
- **wp_footer deaktivieren** - Entfernt alle Skripte/Styles aus dem WordPress-Footer

### Ausgewählte Styles deaktivieren

Selektiv CSS deaktivieren, das Anzeigeprobleme verursachen könnte:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- usw.

### Ausgewählte Skripte deaktivieren

Selektiv JavaScript deaktivieren, das möglicherweise mit Zahlungsmethoden interferiert:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- usw.

:::tip
Wenn ein Zahlungsgateway nicht korrekt angezeigt wird:

1. Versuchen Sie zuerst, die Theme-Styles zu deaktivieren
2. Versuchen Sie dann, die WooCommerce-Skripte zu deaktivieren, die nicht benötigt werden
3. Seien Sie vorsichtig, dass Sie keine Skripte deaktivieren, die von Ihrem Zahlungsgateway benötigt werden
   :::

## Abbrechen

Klicken Sie auf **Abbrechen**, um das Kassenmodal zu schließen, ohne die Bestellung abzuschließen. Die Bestellung bleibt als offener Warenkorb.
