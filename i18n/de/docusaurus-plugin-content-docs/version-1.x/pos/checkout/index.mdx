Wenn Sie bereit sind, einen Verkauf abzuschließen, klicken Sie auf die Schaltfläche **Checkout**, um das Checkout-Modal zu öffnen. Hier verarbeiten Sie die Zahlung und schließen die Bestellung ab.

{/* Screenshot: checkout-modal.png */}

## Übersicht des Checkout-Modals

Das Checkout-Modal zeigt an:

- **Bestellnummer** - Die WooCommerce-Bestell-ID
- **Zu zahlender Betrag** - Gesamter fälliger Betrag
- **Checkout-Einstellungen**-Schaltfläche - Probleme mit der Anzeige beheben
- **Kassierer** - Wer die Bestellung verarbeitet
- **Kunde** - Der Kunde für diese Bestellung (anklickbarer Link)
- **Gutscheincode**-Feld - Rabattcodes anwenden
- **Bestellübersicht** - Produkte, Mengen und Summen
- **Zahlungsmethoden** - Verfügbare Zahlungsoptionen
- **Abbrechen / Zahlung verarbeiten**-Schaltflächen

## Zahlungsmethoden

### Verfügbar in der kostenlosen Version

Die kostenlose Version von WCPOS umfasst zwei Zahlungs-Gateways:

- **Bar** - Mit hinterlegtem Betrag und Wechselgeldrechner
- **Karte** - Für externe Kartenlesegeräte

### Zusätzliche Gateways (Pro)

:::info Pro-Funktion
Zusätzliche Zahlungsmethoden erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Benutzerdefinierte Gateways** - Erstellen Sie Ihre eigenen Zahlungsintegrationen

Siehe [Zahlung](/payment) für Details zur Konfiguration von Gateways.

### Auswahl einer Zahlungsmethode

Klicken Sie auf eine Zahlungsmethode, um sie auszuwählen. Das Formular wird aktualisiert, um relevante Felder anzuzeigen:

**Bar:**
- **Hinterlegter Betrag** - Geben Sie den Betrag ein, den der Kunde Ihnen gibt
- **Wechselgeld** - Automatisch berechnetes Wechselgeld zur Rückgabe

{/* Screenshot: checkout-cash-payment.png */}

**Karte:**
- Zahlung auf Ihrem externen Kartenlesegerät verarbeiten
- Klicken Sie auf Zahlung verarbeiten, um abzuschließen

## Gutscheincodes

Um einen Rabatt anzuwenden:

1. Geben Sie den Gutscheincode in das Feld ein
2. Klicken Sie auf **Gutscheincode anwenden**
3. Der Rabatt erscheint in der Bestellübersicht

Gutscheine müssen in WooCommerce konfiguriert werden (`WP Admin > Marketing > Gutscheine`).

## Zahlung verarbeiten

1. Wählen Sie eine Zahlungsmethode aus
2. Geben Sie für Barzahlungen den hinterlegten Betrag ein
3. Klicken Sie auf **Zahlung verarbeiten**
4. Die Bestellung wird abgeschlossen und der [Beleg](/pos/checkout/receipts) angezeigt

## Checkout-Einstellungen (Fehlerbehebung)

Das Checkout-Modal verwendet ein iframe/Webview, um die WooCommerce-Bestellbezahlseite anzuzeigen. Dies nutzt die bestehende Zahlungsinfrastruktur von WooCommerce, was bedeutet, dass jedes Zahlungs-Gateway, das mit WooCommerce funktioniert, auch im POS funktionieren sollte.

Allerdings können Theme- und Plugin-Skripte manchmal interferieren. Klicken Sie auf **Checkout-Einstellungen**, um Probleme zu beheben:

{/* Screenshot: checkout-settings.png */}

### Alle Stile und Skripte deaktivieren

- **wp_head deaktivieren** - Entfernt alle Skripte/Stile aus dem WordPress-Header
- **wp_footer deaktivieren** - Entfernt alle Skripte/Stile aus dem WordPress-Footer

### Ausgewählte Stile deaktivieren

Wählen Sie selektiv CSS aus, das möglicherweise Anzeigeprobleme verursacht:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- usw.

### Ausgewählte Skripte deaktivieren

Wählen Sie selektiv JavaScript aus, das möglicherweise mit Zahlungs-Gateways interferiert:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- usw.

:::tip
Wenn ein Zahlungs-Gateway nicht korrekt angezeigt wird:
1. Versuchen Sie zuerst, die Stile des Themes zu deaktivieren
2. Versuchen Sie dann, WooCommerce-Skripte zu deaktivieren, die nicht benötigt werden
3. Seien Sie vorsichtig, um keine Skripte zu deaktivieren, die von Ihrem Zahlungs-Gateway benötigt werden
:::

## Abbrechen

Klicken Sie auf **Abbrechen**, um das Checkout-Modal zu schließen, ohne die Bestellung abzuschließen. Die Bestellung bleibt als offener Warenkorb.
