Wenn ein Verkauf abgeschlossen werden soll, klicken Sie auf die Schaltfläche **Checkout**, um das Checkout-Fenster zu öffnen. Hier wird die Zahlung abgewickelt und die Bestellung abgeschlossen.

{/* Screenshot: checkout-modal.png */}

## Übersicht des Checkout-Fensters {#checkout-modal-overview}

Das Checkout-Fenster zeigt Folgendes an:

- **Bestellnummer** – Die WooCommerce-Bestell-ID
- **Zu zahlender Betrag** – Der fällige Gesamtbetrag
- Schaltfläche **Checkout-Einstellungen** – Anzeigeprobleme beheben
- **Kassierer** – Wer die Bestellung bearbeitet
- **Kunde** – Der Kunde für diese Bestellung (klickbarer Link)
- **Gutschein hinzufügen**-Schaltfläche – Rabattcodes anwenden
- **Bestellübersicht** – Produkte, Mengen und Summen
- **Zahlungsmethoden** – Verfügbare Zahlungsoptionen
- Schaltflächen **Abbrechen / Zahlung verarbeiten**

## Zahlungsmethoden {#payment-methods}

### In der kostenlosen Version verfügbar {#available-in-free-version}

Die kostenlose Version von WCPOS enthält zwei Zahlungsgateways:

- **Bargeld** – Mit Eingabe des erhaltenen Betrags und Wechselgeldrechner
- **Karte** – Für externe Kartenterminals

### Zusätzliche Gateways (Pro) {#additional-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie Folgendes aktivieren:

- **Stripe Terminal** – Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp Terminal** – Integration mit SumUp-Kartenlesegeräten
- **Benutzerdefinierte Gateways** – Erstellen Sie eigene Zahlungsintegrationen

Weitere Informationen zur Konfiguration von Gateways finden Sie unter [Zahlung](/payment).

### Zahlungsmethode auswählen {#selecting-a-payment-method}

Klicken Sie auf eine Zahlungsmethode, um sie auszuwählen. Das Formular wird aktualisiert und zeigt die relevanten Felder an:

**Bargeld:**

- **Erhaltener Betrag** – Geben Sie den Betrag ein, den der Kunde Ihnen gibt
- **Rückgeld** – Automatisch berechnetes Wechselgeld

{/* Screenshot: checkout-cash-payment.png */}

**Karte:**

- Zahlung auf dem externen Kartenterminal durchführen
- Auf Zahlung verarbeiten klicken, um den Vorgang abzuschließen

## Gutscheine {#coupons}

Der Warenkorb enthält ein Eingabefeld **Gutschein hinzufügen** oberhalb der Summen (nur Pro). Den Code eingeben oder nach Beschreibung suchen; der Gutschein wird lokal validiert und als entfernbare Markierung angezeigt. Mehrere Gutscheine werden nacheinander angewendet.

Siehe **[Gutscheine an der Kasse einlösen](/coupons/applying-coupons)** für den vollständigen Ablauf — Suche, Markierungen, sequenzielle Rabatte und die Tabelle mit Validierungsfehlern und Lösungen. Informationen zu Gutscheinarten, Validierungsregeln und der Einrichtung in WooCommerce finden Sie unter [Gutscheine](/coupons).

## Zahlung verarbeiten {#processing-payment}

1. Eine Zahlungsmethode auswählen
2. Bei Barzahlung den erhaltenen Betrag eingeben
3. Auf **Zahlung verarbeiten** klicken
4. Die Bestellung wird abgeschlossen und der [Beleg](/receipts/at-checkout) wird angezeigt

:::note Eine Zahlungsmethode pro Bestellung
Eine Bestellung wird mit einer einzigen Zahlungsmethode bezahlt — WCPOS unterstützt derzeit keine geteilten oder teilweisen Zahlungen (z. B. teils bar, teils mit Karte) innerhalb einer Bestellung. Unterstützung für geteilte Zahlungen steht auf der [Roadmap](https://github.com/orgs/wcpos/projects/4).
:::

## Checkout-Einstellungen (Fehlerbehebung) {#checkout-settings-troubleshooting}

Das Checkout-Fenster verwendet einen iframe/Webview, um die WooCommerce-Bestellzahlungsseite anzuzeigen. Dadurch wird die bestehende Zahlungsinfrastruktur von WooCommerce genutzt, sodass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS funktionieren sollte.

Allerdings können Theme- und Plugin-Skripte manchmal Störungen verursachen. Klicken Sie auf **Checkout-Einstellungen**, um Fehler zu beheben:

### Alle Styles und Skripte deaktivieren {#disable-all-styles-and-scripts}

:::warning Radikale Maßnahme
Dies ist die radikale Maßnahme und sollte nur zum Testen oder in seltenen Fällen verwendet werden, in denen Entwickler genau wissen, was sie tun. Das Deaktivieren aller wp_head-Skripte entfernt auch die WooCommerce-Skripte, die zum Ein-/Ausklappen der Zahlungsgateways benötigt werden, was die Zahlungsfunktionalität beeinträchtigen kann.
:::

- **wp_head deaktivieren** – Entfernt alle Skripte/Styles aus dem WordPress-Header
- **wp_footer deaktivieren** – Entfernt alle Skripte/Styles aus dem WordPress-Footer

### Ausgewählte Styles deaktivieren {#disable-selected-styles}

CSS selektiv deaktivieren, das zu Anzeigeproblemen führen kann:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- usw.

### Ausgewählte Skripte deaktivieren {#disable-selected-scripts}

JavaScript, das Zahlungsgateways beeinträchtigen kann, gezielt deaktivieren:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- usw.

:::tip
Falls ein Zahlungsgateway nicht korrekt angezeigt wird:

1. Zuerst die Theme-Styles deaktivieren
2. Dann nicht benötigte WooCommerce-Skripte deaktivieren
3. Achten Sie darauf, keine Skripte zu deaktivieren, die von Ihrem Zahlungsgateway benötigt werden
   :::

## Abbrechen {#cancel}

Klicken Sie auf **Abbrechen**, um das Checkout-Fenster zu schließen, ohne die Bestellung abzuschließen. Die Bestellung bleibt als offener Warenkorb erhalten.
