Wenn Sie bereit sind, einen Verkauf abzuschließen, klicken Sie auf die Schaltfläche **Checkout**, um das Checkout-Fenster zu öffnen. Hier verarbeiten Sie die Zahlung und schließen die Bestellung ab.

{/* Screenshot: checkout-modal.png */}

## Übersicht des Checkout-Fensters

Das Checkout-Fenster zeigt an:

- **Bestellnummer** - Die WooCommerce-Bestell-ID
- **Zu zahlender Betrag** - Gesamter fälliger Betrag
- **Checkout-Einstellungen**-Schaltfläche - Lösungsansätze für Anzeigeprobleme
- **Kassierer** - Wer die Bestellung bearbeitet
- **Kunde** - Der Kunde für diese Bestellung (klickbarer Link)
- **Gutscheincode**-Feld - Rabattcodes anwenden
- **Bestellübersicht** - Produkte, Mengen und Gesamtwerte
- **Zahlungsmethoden** - Verfügbare Zahlungsoptionen
- **Abbrechen / Zahlung verarbeiten**-Schaltflächen

## Zahlungsmethoden

### Verfügbar in der kostenlosen Version

Die kostenlose Version von WCPOS umfasst zwei Zahlungsmethoden:

- **Bargeld** - Mit eingegebenem Betrag und Wechselgeldrechner
- **Karte** - Für externe Kartenterminals

### Zusätzliche Gateways (Pro)

:::info Pro-Funktion
Zusätzliche Zahlungsmethoden erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Benutzerdefinierte Gateways** - Erstellen Sie eigene Zahlungsintegrationen

Siehe [Zahlung](/payment) für Details zur Konfiguration von Gateways.

### Auswahl einer Zahlungsmethode

Klicken Sie auf eine Zahlungsmethode, um sie auszuwählen. Das Formular wird aktualisiert, um relevante Felder anzuzeigen:

**Bargeld:**
- **Eingegebener Betrag** - Geben Sie den Betrag an, den der Kunde Ihnen gibt
- **Wechselgeld** - Automatisch berechnetes Wechselgeld, das zurückgegeben werden muss

{/* Screenshot: checkout-cash-payment.png */}

**Karte:**
- Zahlung auf Ihrem externen Kartenterminal verarbeiten
- Klicken Sie auf Zahlung verarbeiten, um den Vorgang abzuschließen

## Gutscheincodes

Um einen Rabatt anzuwenden:

1. Geben Sie den Gutscheincode in das Feld ein
2. Klicken Sie auf **Gutschein anwenden**
3. Der Rabatt wird in der Bestellübersicht angezeigt

Gutscheine müssen in WooCommerce konfiguriert werden (`WP Admin > Marketing > Gutscheine`).

## Zahlung verarbeiten

1. Wählen Sie eine Zahlungsmethode aus
2. Geben Sie für Bargeld den eingegebenen Betrag an
3. Klicken Sie auf **Zahlung verarbeiten**
4. Die Bestellung wird abgeschlossen und der [Beleg](/pos/checkout/receipts) wird angezeigt

## Checkout-Einstellungen (Fehlerbehebung)

Das Checkout-Fenster verwendet ein iframe/webview, um die WooCommerce-Bestellzahlungsseite anzuzeigen. Dies nutzt die vorhandene Zahlungsinfrastruktur von WooCommerce, was bedeutet, dass jede Zahlungsmethode, die mit WooCommerce funktioniert, auch im POS funktionieren sollte.

Themen- und Plugin-Skripte können jedoch manchmal stören. Klicken Sie auf **Checkout-Einstellungen**, um Probleme zu beheben:

### Alle Stile und Skripte deaktivieren

- **wp_head deaktivieren** - Entfernt alle Skripte/Stile aus dem WordPress-Header
- **wp_footer deaktivieren** - Entfernt alle Skripte/Stile aus dem WordPress-Footer

### Ausgewählte Stile deaktivieren

Wählen Sie CSS aus, das möglicherweise Anzeigeprobleme verursacht:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- usw.

### Ausgewählte Skripte deaktivieren

Wählen Sie JavaScript aus, das möglicherweise mit Zahlungsmethoden stört:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- usw.

:::tip
Wenn eine Zahlungsmethode nicht richtig angezeigt wird:
1. Versuchen Sie zuerst, die Stile des Themas zu deaktivieren
2. Versuchen Sie dann, WooCommerce-Skripte, die nicht benötigt werden, zu deaktivieren
3. Seien Sie vorsichtig, dass Sie keine Skripte deaktivieren, die für Ihre Zahlungsmethode erforderlich sind
:::

## Abbrechen

Klicken Sie auf **Abbrechen**, um das Checkout-Fenster zu schließen, ohne die Bestellung abzuschließen. Die Bestellung bleibt als offener Warenkorb.
