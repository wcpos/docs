:::info Pro-Funktion
Der Gutschein-Bildschirm und die Möglichkeit, Gutscheine an der Kasse einzulösen, erfordern [WCPOS Pro](/getting-started/pro-license). Nutzer der kostenlosen Version sehen eine unscharfe Vorschau des Gutschein-Bildschirms, können jedoch keine Gutscheindetails einsehen oder Codes im Warenkorb anwenden.
:::

Der Gutschein-Bildschirm ermöglicht es, WooCommerce-Gutscheine direkt im POS zu durchsuchen und nachzuschlagen. Über die Schaltfläche **Gutschein hinzufügen** im Warenkorb werden diese auf die aktuelle Bestellung angewendet. Gutscheine werden weiterhin in WooCommerce erstellt und verwaltet — das POS synchronisiert sie und validiert sie lokal, um an der Kasse sofortiges Feedback zu liefern.

## Übersicht der Benutzeroberfläche {#interface-overview}

### Kopfzeilen-Aktionen {#header-actions}

Am oberen Rand des Bildschirms:

- **Suchleiste** — Gutschein anhand des Codes suchen
- **Anzeigeeinstellungen** (<Icon name="sliders" />) — Sichtbare Spalten konfigurieren

### Gutschein-Tabelle {#coupons-table}

Im Hauptbereich werden die synchronisierten Gutscheine angezeigt mit:

- **Code** — Der Gutscheincode, den Kunden eingeben
- **Beschreibung** — Interne Beschreibung (wird auch als Belegbezeichnung verwendet, wenn angegeben)
- **Rabattart** — Prozentual, fester Warenkorbrabatt oder fester Produktrabatt
- **Betrag** — Der Rabattwert
- **Verwendet / Limit** — Anzahl der Nutzungen und Gesamtnutzungslimit
- **Ablaufdatum** — Ablaufdatum oder leer, wenn keines festgelegt ist
- **Aktionen** — Drei-Punkte-Menü

### Fußzeile {#footer}

- Gutscheinanzahl mit Synchronisierungsschaltfläche (<Icon name="arrow-rotate-right" />). **Lange drücken** für die Optionen Löschen und Aktualisieren.

## Gutscheinarten {#coupon-types}

WCPOS unterstützt die drei standardmäßigen WooCommerce-Gutscheinarten:

| Art | Beschreibung | Beispiel |
|---|---|---|
| **Prozentualer Rabatt** | Reduziert die Zwischensumme des Warenkorbs um einen Prozentsatz | `10%` Rabatt auf die Bestellung |
| **Fester Warenkorbrabatt** | Reduziert die Warenkorbsumme um einen festen Betrag | `$5` Rabatt auf die Bestellung |
| **Fester Produktrabatt** | Reduziert passende Produktpositionen um einen festen Betrag pro Einheit | `$2` Rabatt pro T-Shirt |

Alle drei Arten unterliegen denselben Validierungsregeln (Ablaufdatum, Nutzungslimits, Produkt-/Kategoriebeschränkungen usw.).

## Gutscheine erstellen {#creating-coupons}

Gutscheine werden in WooCommerce erstellt, nicht im POS. Navigieren Sie zu **WP Admin → Marketing → Gutscheine → Gutschein hinzufügen**:

1. **Gutscheincode** — wird von Kassierern an der Kasse eingegeben. Bei Codes wird nicht zwischen Groß- und Kleinschreibung unterschieden.
2. **Beschreibung** — interner Hinweis. WCPOS verwendet dieses Feld als Rabattbezeichnung auf Belegen. Daher ist eine kundenfreundliche Formulierung wie "Manager-Rabatt 10 %" einem internen Code vorzuziehen.
3. **Rabattart** — prozentual, fester Warenkorbrabatt oder fester Produktrabatt.
4. **Gutscheinbetrag** — der Wert des Rabatts.
5. **Ablaufdatum** — optional. Nach diesem Datum wird der Gutschein abgelehnt.

### Verwendungsbeschränkung {#usage-restriction}

- **Mindest-/Höchstbetrag** — der Gutschein gilt nur oberhalb bzw. unterhalb eines Zwischensummen-Schwellenwerts.
- **Nur einzeln verwendbar** — verhindert, dass dieser Gutschein mit anderen Gutscheinen kombiniert wird.
- **Angebots-Artikel ausschließen** — überspringt bereits reduzierte Artikel (sowie Positionen, bei denen ein Kassierer den Preis an der Kasse gesenkt hat – siehe [POS-Preisüberschreibungen](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)).
- **Produkte / Produkte ausschließen** — beschränkt den Gutschein auf bestimmte Produkte (oder schließt diese aus).
- **Produktkategorien / Kategorien ausschließen** — Einschränkung nach Kategorie. Sonstige/benutzerdefinierte Produkte, die im POS erstellt werden, berücksichtigen auch die über den POS zugewiesenen Kategorien.
- **Erlaubte E-Mails** — beschränkt den Gutschein auf bestimmte Kunden-E-Mail-Adressen (unterstützt `*`-Platzhalter).

### Usage limits {#usage-limits}

- **Usage limit per coupon** — total times this code can be used across all customers.
- **Usage limit per user** — times one customer can use it.
- **Limit usage to X items** — for fixed-product coupons, cap the number of qualifying items discounted per use.

### Tips for POS use {#tips-for-pos-use}

- Create a **"Manager 10%"** coupon (10% off, no minimum spend, no expiry, single-use restriction off) and give the code to managers — staff can apply it at the till for ad-hoc adjustments and you keep a tracked discount in reports.
- Create a **"Loyalty $5"** coupon for repeat-customer rewards.
- For one-off promotions, set a short expiry date so the code can't be reused later by mistake.
- Setting **Description** to the phrase you'd like printed on receipts is the fastest way to brand your discounts.

## Applying a Coupon at the Register {#applying-a-coupon-at-the-register}

In the cart panel:

1. Click **Add Coupon**
2. Enter the coupon code
3. Press **Enter**

The POS validates the coupon against your synced data instantly — no round-trip to the server — and applies the discount to the cart. If you change cart contents afterwards, the discount recalculates automatically.

To remove an applied coupon, click the × next to it in the cart.

## How Validation Works {#how-validation-works}

When a code is entered, the POS checks all the same rules WooCommerce would check on the server:

- Coupon exists and is not expired
- Usage limits aren't exceeded (overall and per-user)
- Minimum/maximum spend is met
- "Individual use" coupons don't conflict with already-applied coupons
- Product/category restrictions match the cart contents
- Email restrictions match the selected customer (or are skipped on Guest orders)
- Sale items are excluded if **Exclude sale items** is enabled

If any check fails, the cashier sees a specific error message (e.g., "This coupon has expired" or "Minimum spend not reached"). The same checks run again on the server when the order is submitted — the local validation is for speed, not for trust.

## Sync Behaviour {#sync-behaviour}

Coupons sync from WooCommerce to the device via the standard WCPOS replication pipeline:

- New coupons created in WP Admin appear in the POS on the next sync.
- Updates to existing coupons (usage count, expiry changes, etc.) sync down automatically.
- Coupons are stored locally in IndexedDB so they remain available offline.

If you just created a coupon in WP Admin and don't see it yet, the sync may not have run. From the Coupons screen footer, tap the sync icon (<Icon name="arrow-rotate-right" />) to refresh — long-press for **Clear and refresh** if you need a fresh fetch.

## Connectivity {#connectivity}

- **Applying coupons** works offline because validation is client-side from synced data.
- **Completing checkout** still requires a server connection — the server re-validates and is the authoritative source for the final order total.
- **Creating or editing coupons** happens in WP Admin and requires a connection to your WordPress site.

## How Coupons Interact with POS Discounts {#how-coupons-interact-with-pos-discounts}

When a cashier lowers a line price at the till and a coupon is then applied, the coupon calculates against the **lowered price**, not the original. POS-lowered lines are treated as "on sale" — coupons with **Exclude sale items** enabled will skip them. See [Discounts](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons) for more.
