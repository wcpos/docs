:::info Pro-Funktion
Die Gutschein-Ansicht und die Möglichkeit, Gutscheine an der Kasse einzulösen, erfordern [WCPOS Pro](/getting-started/pro-license). Nutzer der kostenlosen Version sehen eine unscharfe Vorschau der Gutschein-Ansicht, können jedoch keine Gutscheindetails einsehen oder Codes im Warenkorb anwenden.
:::

Die Gutschein-Ansicht ermöglicht das Durchsuchen und Nachschlagen von WooCommerce-Gutscheinen direkt im POS, und über die Schaltfläche **Gutschein hinzufügen** im Warenkorb werden sie auf die aktuelle Bestellung angewendet. Gutscheine werden weiterhin in WooCommerce erstellt und verwaltet — das POS synchronisiert sie und validiert sie lokal für sofortiges Feedback an der Kasse.

## Übersicht der Benutzeroberfläche {#interface-overview}

### Kopfzeilen-Aktionen {#header-actions}

Am oberen Bildschirmrand:

- **Suchleiste** — Gutschein nach Code suchen
- **Anzeigeeinstellungen** (<Icon name="sliders" />) — Sichtbare Spalten konfigurieren

### Gutscheintabelle {#coupons-table}

Im Hauptbereich werden die synchronisierten Gutscheine angezeigt mit:

- **Code** — Der Gutscheincode, den Kunden eingeben
- **Beschreibung** — Interne Beschreibung (wird auch als Belegtext verwendet, wenn angegeben)
- **Rabattart** — Prozentual, fester Warenkorbrabatt oder fester Produktrabatt
- **Betrag** — Der Rabattwert
- **Verwendet / Limit** — Anzahl der Nutzungen und Gesamtnutzungslimit
- **Ablaufdatum** — Ablaufdatum oder leer, wenn keines festgelegt ist
- **Aktionen** — Drei-Punkte-Menü

### Fußzeile {#footer}

- Gutscheinanzahl mit Sync-Schaltfläche (<Icon name="arrow-rotate-right" />). **Langes Drücken** für die Optionen Leeren und Aktualisieren.

## Gutscheinarten {#coupon-types}

WCPOS unterstützt die drei Standard-Gutscheinarten von WooCommerce:

| Typ | Beschreibung | Beispiel |
|---|---|---|
| **Prozentualer Rabatt** | Reduziert die Warenkorb-Zwischensumme um einen Prozentsatz | `10%` Rabatt auf die Bestellung |
| **Fester Warenkorbrabatt** | Reduziert die Warenkorb-Gesamtsumme um einen festen Betrag | `$5` Rabatt auf die Bestellung |
| **Fester Produktrabatt** | Reduziert passende Produktpositionen um einen festen Betrag pro Stück | `$2` Rabatt auf jedes T-Shirt |

Alle drei Arten unterliegen denselben Validierungsregeln (Ablaufdatum, Nutzungsbeschränkungen, Produkt-/Kategoriebeschränkungen usw.).

## Gutscheine erstellen {#creating-coupons}

Gutscheine werden in WooCommerce erstellt, nicht im POS. Navigieren Sie zu **WP Admin → Marketing → Gutscheine → Gutschein hinzufügen**:

1. **Gutscheincode** — wird von Kassierern an der Kasse eingegeben. Bei Codes wird nicht zwischen Groß- und Kleinschreibung unterschieden.
2. **Beschreibung** — interner Hinweis. WCPOS verwendet dieses Feld als Rabattbezeichnung auf Belegen, daher empfiehlt sich eine kundenfreundliche Formulierung wie „Managerrabatt 10 %“ anstelle eines internen Codes.
3. **Rabattart** — prozentual, fester Warenkorbrabatt oder fester Produktrabatt.
4. **Gutscheinbetrag** — der Wert des Rabatts.
5. **Ablaufdatum** — optional. Nach diesem Datum wird der Gutschein abgelehnt.

### Nutzungsbeschränkung {#usage-restriction}

- **Mindest-/Höchstbetrag** — der Gutschein gilt nur oberhalb bzw. unterhalb eines bestimmten Zwischensummen-Schwellenwerts.
- **Nur einzeln verwendbar** — verhindert, dass dieser Gutschein mit anderen Gutscheinen kombiniert wird.
- **Angebotsartikel ausschließen** — überspringt bereits reduzierte Artikel (sowie Positionen, bei denen ein Kassierer den Preis an der Kasse gesenkt hat – siehe [POS-Preisüberschreibungen](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)).
- **Produkte / Produkte ausschließen** — beschränkt den Gutschein auf bestimmte Produkte oder schließt diese aus.
- **Produktkategorien / Kategorien ausschließen** — Einschränkung nach Kategorie. Sonstige/benutzerdefinierte Produkte, die im POS erstellt werden, berücksichtigen auch die über das POS zugewiesenen Kategorien.
- **Erlaubte E-Mail-Adressen** — beschränkt den Gutschein auf bestimmte Kunden-E-Mail-Adressen (unterstützt `*`-Platzhalter).

### Nutzungslimits {#usage-limits}

- **Nutzungslimit pro Gutschein** — Gesamtanzahl der Einlösungen dieses Codes über alle Kunden hinweg.
- **Nutzungslimit pro Benutzer** — Anzahl der möglichen Einlösungen durch einen einzelnen Kunden.
- **Nutzung auf X Artikel beschränken** — Bei Gutscheinen für bestimmte Produkte wird die Anzahl der rabattierten Artikel pro Einlösung begrenzt.

### Tipps für den POS-Einsatz {#tips-for-pos-use}

- Erstellen Sie einen **"Manager 10%"**-Gutschein (10 % Rabatt, kein Mindestbestellwert, kein Ablaufdatum, Einmalnutzung deaktiviert) und geben Sie den Code an Filialleiter weiter — Mitarbeiter können ihn an der Kasse für spontane Anpassungen einlösen, und Sie behalten einen nachverfolgbaren Rabatt in den Berichten.
- Erstellen Sie einen **"Loyalty $5"**-Gutschein als Prämie für Stammkunden.
- Für einmalige Aktionen empfiehlt es sich, ein kurzes Ablaufdatum festzulegen, damit der Code nicht versehentlich erneut verwendet werden kann.
- Das Eintragen des gewünschten Bondrucktexts im Feld **Beschreibung** ist der schnellste Möglichkeit, Rabatte individuell zu kennzeichnen.

## Einen Gutschein an der Kasse einlösen {#applying-a-coupon-at-the-register}

Tippen Sie im Warenkorb auf **Gutschein hinzufügen**, geben Sie dann den Code ein oder suchen Sie nach der Beschreibung. Der Gutschein wird sofort anhand der synchronisierten Daten validiert und erscheint als entfernbarer Chip über den Warenkorb-Summen.

Den vollständigen Kassenablauf — Suche, Gutschein-Chips, Kombination mehrerer Gutscheine und Behebung von Validierungsfehlern — finden Sie unter **[Gutscheine an der Kasse einlösen](/coupons/applying-coupons)**.

## How Validation Works {#how-validation-works}

When a code is entered, the POS checks all the same rules WooCommerce would check on the server:

- Coupon exists and is not expired
- Usage limits aren't exceeded (overall and per-user)
- Minimum/maximum spend is met
- "Individual use" coupons don't conflict with already-applied coupons
- Product/category restrictions match the cart contents
- Email restrictions match the selected customer (or are skipped on Guest orders)
- Sale items are excluded if **Exclude sale items** is enabled

If any check fails, the cashier sees a specific error message (e.g., "This coupon has expired" or "Minimum spend not reached"). The same checks run again on the server when the order is submitted — the local validation is for speed, not for trust.

## Sync Behaviour {#sync-behaviour}

Coupons sync from WooCommerce to the device via the standard WCPOS replication pipeline:

- New coupons created in WP Admin appear in the POS on the next sync.
- Updates to existing coupons (usage count, expiry changes, etc.) sync down automatically.
- Coupons are stored locally in IndexedDB so they remain available offline.

If you just created a coupon in WP Admin and don't see it yet, the sync may not have run. From the Coupons screen footer, tap the sync icon (<Icon name="arrow-rotate-right" />) to refresh — long-press for **Clear and refresh** if you need a fresh fetch.

## Connectivity {#connectivity}

- **Applying coupons** works offline because validation is client-side from synced data.
- **Completing checkout** still requires a server connection — the server re-validates and is the authoritative source for the final order total.
- **Creating or editing coupons** happens in WP Admin and requires a connection to your WordPress site.

## How Coupons Interact with POS Discounts {#how-coupons-interact-with-pos-discounts}

When a cashier lowers a line price at the till and a coupon is then applied, the coupon calculates against the **lowered price**, not the original. POS-lowered lines are treated as "on sale" — coupons with **Exclude sale items** enabled will skip them. See [Discounts](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons) for more.
