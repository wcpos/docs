:::info Pro-Funktion
Der Gutscheine-Bildschirm und die Möglichkeit, Gutscheine an der Kasse einzulösen, erfordern [WCPOS Pro](/getting-started/pro-license). Nutzer der kostenlosen Version sehen eine unscharfe Vorschau des Gutscheine-Bildschirms, können jedoch keine Gutscheindetails einsehen oder Codes im Warenkorb anwenden.
:::

Der Gutscheine-Bildschirm ermöglicht es, WooCommerce-Gutscheine direkt im POS zu durchsuchen und nachzuschlagen. Die Schaltfläche **Gutschein hinzufügen** im Warenkorb wendet sie auf die aktuelle Bestellung an. Gutscheine werden weiterhin in WooCommerce erstellt und verwaltet — das POS synchronisiert sie und validiert sie lokal für sofortiges Feedback an der Kasse.

## Übersicht der Benutzeroberfläche {#interface-overview}

### Kopfzeilen-Aktionen {#header-actions}

Am oberen Rand des Bildschirms:

- **Suchleiste** — Gutschein anhand des Codes suchen
- **Anzeigeeinstellungen** (<Icon name="sliders" />) — Sichtbare Spalten konfigurieren

### Gutschein-Tabelle {#coupons-table}

Im Hauptbereich werden die synchronisierten Gutscheine angezeigt mit:

- **Code** — Der Gutscheincode, den Kunden eingeben
- **Beschreibung** — Interne Beschreibung (wird auch als Bonbezeichnung verwendet, wenn angegeben)
- **Rabattart** — Prozentual, fester Warenkorb oder festes Produkt
- **Betrag** — Der Rabattwert
- **Verwendet / Limit** — Anzahl der Nutzungen und Gesamtnutzungslimit
- **Ablaufdatum** — Ablaufdatum oder leer, wenn keines festgelegt ist
- **Aktionen** — Drei-Punkte-Menü

### Fußzeile {#footer}

- Gutscheinanzahl mit Synchronisierungsschaltfläche (<Icon name="arrow-rotate-right" />). **Langes Drücken** für die Optionen Leeren und Aktualisieren.

## Gutscheinarten {#coupon-types}

WCPOS unterstützt die drei Standard-Gutscheinarten von WooCommerce:

| Art | Beschreibung | Beispiel |
|---|---|---|
| **Prozentualer Rabatt** | Reduziert die Zwischensumme des Warenkorbs um einen Prozentsatz | `10%` Rabatt auf die Bestellung |
| **Fester Warenkorbrabatt** | Reduziert die Warenkorbsumme um einen festen Betrag | `$5` Rabatt auf die Bestellung |
| **Fester Produktrabatt** | Reduziert passende Produktzeilen um einen festen Betrag pro Einheit | `$2` Rabatt pro T-Shirt |

Alle drei Arten unterliegen denselben Validierungsregeln (Ablaufdatum, Nutzungslimits, Produkt-/Kategoriebeschränkungen usw.).

## Gutscheine erstellen {#creating-coupons}

Gutscheine werden in WooCommerce erstellt, nicht im POS. Navigieren Sie zu **WP Admin → Marketing → Gutscheine → Gutschein hinzufügen**:

1. **Gutscheincode** — wird vom Kassenpersonal an der Kasse eingegeben. Codes unterscheiden nicht zwischen Groß- und Kleinschreibung.
2. **Beschreibung** — interner Hinweis. WCPOS verwendet diesen Text als Rabattbezeichnung auf Belegen, wenn er gesetzt ist. Daher sollte eine kundenfreundliche Formulierung wie „Manager-Rabatt 10 %" anstelle eines internen Codes verwendet werden.
3. **Rabattart** — prozentual, fester Warenkorbrabatt oder fester Produktrabatt.
4. **Gutscheinbetrag** — der Wert des Rabatts.
5. **Ablaufdatum** — optional. Nach diesem Datum wird der Gutschein abgelehnt.

### Verwendungsbeschränkung {#usage-restriction}

- **Mindest-/Höchstbetrag** — Der Gutschein gilt nur oberhalb/unterhalb eines Zwischensummen-Schwellenwerts.
- **Nur einzeln verwendbar** — verhindert, dass dieser Gutschein mit einem anderen Gutschein kombiniert wird.
- **Reduzierte Artikel ausschließen** — überspringt Artikel, die bereits im Angebot sind (sowie alle Positionen, bei denen ein Kassierer den Preis an der Kasse gesenkt hat — siehe [POS-Preisüberschreibungen](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)).
- **Produkte / Produkte ausschließen** — den Gutschein auf bestimmte Produkte beschränken (oder davon ausschließen).
- **Produktkategorien / Kategorien ausschließen** — Einschränkung nach Kategorie. Diverse/benutzerdefinierte Produkte, die im POS erstellt werden, berücksichtigen auch die über den POS zugewiesenen Kategorien.
- **Erlaubte E-Mail-Adressen** — beschränkt den Gutschein auf bestimmte Kunden-E-Mail-Adressen (unterstützt `*`-Platzhalter).

### Nutzungsbeschränkungen {#usage-limits}

- **Nutzungslimit pro Gutschein** — wie oft dieser Code insgesamt von allen Kunden verwendet werden kann.
- **Verwendungslimit pro Benutzer** — wie oft ein Kunde den Gutschein verwenden kann.
- **Nutzung auf X Artikel begrenzen** — bei Festprodukt-Gutscheinen wird die Anzahl der rabattierten qualifizierenden Artikel pro Nutzung begrenzt.

### Tipps für die Nutzung im POS {#tips-for-pos-use}

- Erstellen Sie einen **„Manager 10%"**-Gutschein (10 % Rabatt, kein Mindestbestellwert, kein Ablaufdatum, Einmalnutzung deaktiviert) und geben Sie den Code an Filialleiter weiter — Mitarbeiter können ihn an der Kasse für spontane Anpassungen einlösen, und Sie behalten einen nachverfolgbaren Rabatt in den Berichten.
- Erstellen Sie einen **„Treue 5 $"**-Gutschein für Stammkundenprämien.
- Für einmalige Aktionen empfiehlt es sich, ein kurzes Ablaufdatum festzulegen, damit der Code nicht versehentlich später erneut verwendet werden kann.
- Das Feld **Beschreibung** auf den Text zu setzen, der auf Belegen gedruckt werden soll, ist der schnellste Weg, Ihre Rabatte individuell zu kennzeichnen.

## Einen Gutschein an der Kasse einlösen {#applying-a-coupon-at-the-register}

Tippen Sie auf **Coupon hinzufügen** im Warenkorb und geben Sie den Code ein oder suchen Sie nach Beschreibung. Der Coupon wird sofort anhand der synchronisierten Daten validiert und erscheint als entfernbares Element über den Warenkorb-Summen.

Den vollständigen Kassenablauf — Suche, Gutschein-Pills, Kombinieren mehrerer Gutscheine und Beheben von Validierungsfehlern — finden Sie unter **[Gutscheine an der Kasse einlösen](/coupons/applying-coupons)**.

## So funktioniert die Validierung {#how-validation-works}

Wenn ein Code eingegeben wird, prüft das POS alle Regeln, die auch WooCommerce auf dem Server prüfen würde:

- Gutschein existiert und ist nicht abgelaufen
- Nutzungslimits nicht überschritten sind (insgesamt und pro Benutzer)
- Mindest-/Höchstbetrag ist erreicht
- Gutscheine mit „Individuelle Verwendung" stehen nicht in Konflikt mit bereits angewendeten Gutscheinen
- Produkt-/Kategoriebeschränkungen stimmen mit dem Warenkorbinhalt überein
- E-Mail-Einschränkungen werden mit dem ausgewählten Kunden abgeglichen (oder bei Gastbestellungen übersprungen)
- Angebotsartikel werden ausgeschlossen, wenn **Angebotsartikel ausschließen** aktiviert ist

Wenn eine Prüfung fehlschlägt, wird eine spezifische Fehlermeldung angezeigt (z. B. „Dieser Gutschein ist abgelaufen" oder „Mindestbestellwert nicht erreicht"). Dieselben Prüfungen werden erneut auf dem Server ausgeführt, wenn die Bestellung übermittelt wird — die lokale Validierung dient der Geschwindigkeit, nicht der Absicherung.

## Synchronisierungsverhalten {#sync-behaviour}

Gutscheine werden über die standardmäßige WCPOS-Replikationspipeline von WooCommerce auf das Gerät synchronisiert:

- Neue Gutscheine, die im WP Admin erstellt werden, erscheinen nach der nächsten Synchronisierung im POS.
- Aktualisierungen bestehender Gutscheine (Nutzungsanzahl, Ablaufänderungen usw.) werden automatisch synchronisiert.
- Gutscheine werden lokal in IndexedDB gespeichert und bleiben so auch offline verfügbar.

Wenn ein Gutschein gerade im WP Admin erstellt wurde und noch nicht angezeigt wird, wurde die Synchronisierung möglicherweise noch nicht ausgeführt. In der Fußzeile des Gutschein-Bildschirms kann das Synchronisierungssymbol (<Icon name="arrow-rotate-right" />) angetippt werden, um die Daten zu aktualisieren — durch langes Drücken wird **Löschen und aktualisieren** ausgelöst, falls ein vollständiger Neuabruf erforderlich ist.

## Konnektivität {#connectivity}

- Das **Einlösen von Gutscheinen** funktioniert offline, da die Validierung clientseitig anhand der synchronisierten Daten erfolgt.
- Der **Checkout-Abschluss** erfordert weiterhin eine Serververbindung — der Server validiert erneut und ist die maßgebliche Quelle für den endgültigen Bestellbetrag.
- Das **Erstellen oder Bearbeiten von Gutscheinen** erfolgt im WP Admin und erfordert eine Verbindung zur WordPress-Website.

## Zusammenspiel von Gutscheinen und POS-Rabatten {#how-coupons-interact-with-pos-discounts}

Wenn an der Kasse der Positionspreis gesenkt und anschließend ein Gutschein angewendet wird, berechnet sich der Gutschein auf Basis des **reduzierten Preises**, nicht des Originalpreises. Im POS reduzierte Positionen werden als „im Angebot“ behandelt — Gutscheine mit aktivierter Option **Angebotsartikel ausschließen** überspringen diese. Weitere Informationen unter [Rabatte](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons).
