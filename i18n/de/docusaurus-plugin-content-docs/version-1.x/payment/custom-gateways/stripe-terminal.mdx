# Stripe Terminal Gateway

Das Stripe Terminal-Gateway ermöglicht es Ihnen, persönliche Zahlungen mit physischen Stripe Terminal-Hardwarelesegeräten direkt in WCPOS zu akzeptieren. Dieses Gateway unterstützt sowohl physische Kartenleser als auch den Simulationsmodus zum Testen.

## Funktionen {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Hardware-Integration">
    Physische Stripe Terminal-Lesegeräte über eine Internetverbindung anschließen
  </FeatureCard>
  <FeatureCard icon="desktop" title="Simulator-Modus">
    Zahlungen ohne Hardware mit dem Simulator von Stripe testen
  </FeatureCard>
  <FeatureCard icon="bolt" title="Echtzeitverarbeitung">
    Sofortige Zahlungsabwicklung und Bestätigung
  </FeatureCard>
  <FeatureCard icon="shield" title="Sichere Transaktionen">
    PCI-konforme Zahlungsabwicklung über Stripe
  </FeatureCard>
  <FeatureCard icon="phone" title="Telefonbestellungen (MOTO)">
    Kartenzahlungen über das Telefon annehmen, indem die Details in den Leser eingegeben werden
  </FeatureCard>
</FeatureGrid>

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [Stripe Terminal für WooCommerce Veröffentlichungsseite](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases)
2. Laden Sie die neueste **stripe-terminal-for-woocommerce.zip** Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die ZIP-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="Stripe-Einstellungen konfigurieren">

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Finden Sie **Stripe Terminal** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **Stripe Terminal**, um auf die Einstellungen zuzugreifen
4. Geben Sie Ihren **Stripe Secret Key** ein (diesen können Sie von Ihrem Stripe Dashboard erhalten)
5. Speichern Sie die Einstellungen

:::note
Sie müssen das Stripe Terminal-Gateway in den WooCommerce-Einstellungen nicht aktivieren. Es wird im nächsten Schritt speziell für das POS aktiviert.
:::

  </StepCard>
  <StepCard number="3" title="In WCPOS aktivieren" isLast>

1. Gehen Sie zu `WP Admin > POS > Einstellungen > Kasse`
2. Finden Sie das **Stripe Terminal**-Gateway in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

  </StepCard>
</Steps>

## Verwendung {#usage}

### Verbindung mit einem Leser {#connecting-a-reader}

Wenn Sie das Stripe Terminal-Gateway während der Bestellung im POS auswählen:

1. **Verbindungsmethode wählen**: Sie können entweder einen physischen Leser anschließen oder den Simulator verwenden
2. **Physischer Leser**: Befolgen Sie die Anweisungen auf dem Bildschirm, um Ihr Stripe Terminal-Gerät zu verbinden
3. **Simulator**: Wählen Sie den Simulator-Modus, um verschiedene Zahlungsszenarien ohne Hardware zu testen

### Zahlungen verarbeiten {#processing-payments}

1. **Artikel hinzufügen**: Fügen Sie Produkte zu Ihrem Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "Stripe Terminal" als Zahlungsmethode aus
3. **Leser verbinden**: Schließen Sie Ihren Leser an oder wählen Sie den Simulator-Modus
4. **Zahlung verarbeiten**: Befolgen Sie die Anweisungen, um die Transaktion abzuschließen
5. **Bestätigung**: Die Bestellung wird automatisch nach erfolgreicher Zahlung abgeschlossen

### Testen mit dem Simulator {#testing-with-simulator}

Der Simulator ermöglicht es Ihnen, verschiedene Zahlungsmethoden und Szenarien zu testen:

- **Kartenzahlungen**: Testen Sie verschiedene Kartentypen (Visa, Mastercard usw.)
- **Kontaktlose Zahlungen**: Simulieren Sie kontaktlose Zahlungen
- **Fehler-Szenarien**: Testen Sie abgelehnte Zahlungen und andere Fehlerbedingungen
- **Verschiedene Beträge**: Testen Sie verschiedene Transaktionsbeträge

### Telefonbestellungen (MOTO) {#phone-orders-moto}

MOTO (Mail Order/Telephone Order) ermöglicht es Ihnen, Kartenzahlungen für Kunden zu verarbeiten, die physisch nicht anwesend sind – z. B. wenn Sie eine Bestellung über das Telefon entgegennehmen. Anstatt eine Karte zu tippen oder einzuführen, gibt der Händler die Kartendaten direkt auf dem Bildschirm des Terminallesers ein.

#### Einrichtung {#setup}

<Steps>
  <StepCard number="1" title="MOTO-Zugriff bei Stripe anfordern">

MOTO ist standardmäßig nicht aktiviert. Kontaktieren Sie [den Stripe-Support](https://support.stripe.com/contact) und bitten Sie darum, MOTO-Berechtigungen für Ihr Konto zu aktivieren. Dies ist ein schneller Prozess, erfordert jedoch eine manuelle Genehmigung von Stripe.

  </StepCard>
  <StepCard number="2" title="In den Plugin-Einstellungen aktivieren">

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen > Stripe Terminal`
2. Aktivieren Sie das Kontrollkästchen **Telefonbestellungen (MOTO)**
3. Speichern Sie die Einstellungen

  </StepCard>
  <StepCard number="3" title="Einen kompatiblen Leser anschließen" isLast>

MOTO funktioniert nur mit kompatiblen internetverbundenen Lesegeräten, die in [Unterstützten Terminals](#supported-terminals-internet-connected) aufgeführt sind. Der Schalter erscheint nicht für andere Lesertypen.

  </StepCard>
</Steps>

#### Eine Telefonbestellung annehmen {#taking-a-phone-order}

1. Schließen Sie einen kompatiblen Leser an (siehe [Unterstützte Terminals](#supported-terminals-internet-connected))
2. Aktivieren Sie auf dem Zahlungsbildschirm den Schalter **Telefonbestellung**
3. Klicken Sie auf **Kartenzahlung annehmen** — der Leser zeigt einen Bildschirm zur Eingabe der Kartennummer an, anstatt zum Tippen oder Einstecken der Karte aufzufordern
4. Geben Sie die Kartennummer, das Ablaufdatum und den CVV des Kunden auf dem Leser ein
5. Die Zahlung wird von dort aus wie gewohnt verarbeitet

:::tip
MOTO-Zahlungen verwenden `card` als Zahlungstyp anstelle von `card_present`. Das bedeutet, dass sie aus der Sicht von Stripe eher wie Online-Transaktionen behandelt werden, sodass die Standardgebühren für die Online-Kartenverarbeitung anfallen und nicht die Gebühren für persönliche Zahlungen.
:::

:::caution
Der Schalter für Telefonbestellungen erscheint nur, wenn alle drei Bedingungen erfüllt sind: Die MOTO-Einstellung ist in den Plugin-Einstellungen aktiviert, ein kompatibler Leser ist angeschlossen und der Leser ist kein Simulator. Wenn Sie den Schalter nicht sehen, überprüfen Sie diese Bedingungen.
:::

## Hardware-Kompatibilität {#hardware-compatibility}

:::important Anforderungen an die Konnektivität
Diese Implementierung verwendet das JavaScript SDK von Stripe, was bedeutet, dass es über Webanwendungen funktioniert, jedoch **nur internetverbundene Terminals erfordert**. Bluetooth-Terminals werden derzeit nicht unterstützt.
:::

### Unterstützte Terminals (Internet-Verbunden) {#supported-terminals-internet-connected}

- **Stripe Reader S700/S710** ✅ - Ethernet/WiFi-verbundenes Terminal
- **WisePOS E** ✅ - WiFi-verbundenes Terminal

### Nicht unterstützte Terminals (Bluetooth) {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - Nur Bluetooth
- **BBPOS WisePad 3** ❌ - Nur Bluetooth  
- **Verifone P400** ❌ - Nur Bluetooth

:::note Zukünftige Unterstützung
Die Unterstützung für Bluetooth-Terminals wird in **Q1 2026** zu den iOS- und Android-Anwendungen hinzugefügt. Dies wird die Unterstützung für alle Stripe Terminal-zertifizierten Leser, einschließlich M2 und WisePad 3, ermöglichen.
:::

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Leser lässt sich nicht verbinden">

  - Stellen Sie sicher, dass Sie ein [unterstütztes internetverbundenes Terminal](#supported-terminals-internet-connected) verwenden
  - Vergewissern Sie sich, dass das Terminal mit WiFi/Ethernet verbunden und online ist
  - Überprüfen Sie, ob Ihr Stripe-Konto mit Terminal aktiviert ist
  - Bestätigen Sie, dass das Terminal in Ihrem Stripe-Dashboard registriert ist

  </AccordionItem>

  <AccordionItem question="Zahlung abgelehnt">

  - Überprüfen Sie, ob Ihr Stripe-Konto aktiv und in gutem Zustand ist
  - Vergewissern Sie sich, dass die verwendete Karte gültig ist
  - Stellen Sie sicher, dass ausreichende Mittel verfügbar sind

  </AccordionItem>

  <AccordionItem question="Telefonbestellung-Schalter wird nicht angezeigt">

  - Überprüfen Sie, ob die Einstellung **Telefonbestellungen (MOTO)** in `WooCommerce > Einstellungen > Zahlungen > Stripe Terminal` aktiviert ist
  - Stellen Sie sicher, dass Sie mit einem kompatiblen Leser verbunden sind (siehe [Unterstützte Terminals](#supported-terminals-internet-connected)) — der Schalter ist für andere Lesertypen ausgeblendet
  - Der Schalter erscheint nicht, wenn der Simulator verwendet wird

  </AccordionItem>

  <AccordionItem question="MOTO-Zahlung schlägt mit einem Fehler fehl">

  - Bestätigen Sie, dass Stripe die MOTO-Berechtigungen für Ihr Konto aktiviert hat — kontaktieren Sie [den Stripe-Support](https://support.stripe.com/contact), wenn Sie das noch nicht getan haben
  - Überprüfen Sie, ob die Kartendetails korrekt auf dem Leser eingegeben wurden
  - MOTO-Zahlungen können strengere Betrugsprüfungen haben — stellen Sie sicher, dass die Karte gültig ist und über ausreichende Mittel verfügt

  </AccordionItem>

  <AccordionItem question="SSL-Zertifikatfehler">

  - Stripe Terminal erfordert HTTPS - stellen Sie sicher, dass Ihre Seite ein gültiges SSL-Zertifikat hat
  - Überprüfen Sie, ob Ihr SSL-Zertifikat richtig konfiguriert ist

  </AccordionItem>
</Accordion>
