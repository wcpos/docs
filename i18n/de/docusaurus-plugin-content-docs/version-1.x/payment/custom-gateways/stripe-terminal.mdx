# Stripe Terminal-Gateway

Das Stripe Terminal-Gateway ermöglicht es Ihnen, Zahlungen vor Ort mit Stripe Terminal-Hardwarelesegeräten direkt in WooCommerce POS zu akzeptieren. Dieses Gateway unterstützt sowohl physische Kartenleser als auch den Simulator-Modus zum Testen.

## Funktionen

- **Hardware-Integration**: Schließen Sie physische Stripe Terminal-Lesegeräte über eine Internetverbindung an
- **Simulator-Modus**: Testen Sie Zahlungen ohne Hardware mit Stripes Simulator
- **Mehrere Zahlungsmethoden**: Unterstützung für verschiedene Kartentypen und Zahlungsmethoden
- **Echtzeitverarbeitung**: Instant Zahlungsbearbeitung und Bestätigung
- **Sichere Transaktionen**: PCI-konforme Zahlungsabwicklung über Stripe
- **Web-basiertes SDK**: Verwendet Stripes JavaScript SDK für die Kompatibilität mit Webanwendungen

## Installation

### Schritt 1: Herunterladen und Installieren

1. Besuchen Sie die [Stripe Terminal für WooCommerce Releases-Seite](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases)
2. Laden Sie die neueste **stripe-terminal-for-woocommerce.zip**-Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin-Bereich zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die ZIP-Datei hoch und aktivieren Sie das Plugin

### Schritt 2: Stripe-Einstellungen konfigurieren

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Finden Sie **Stripe Terminal** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **Stripe Terminal**, um auf die Einstellungen zuzugreifen
4. Geben Sie Ihren **Stripe Secret Key** ein (diesen können Sie von Ihrem Stripe-Dashboard abrufen)
5. Speichern Sie die Einstellungen

:::note
Sie müssen das Stripe Terminal-Gateway in den WooCommerce-Einstellungen nicht aktivieren. Es wird speziell für den POS im nächsten Schritt aktiviert.
:::

### Schritt 3: Aktivieren im WooCommerce POS

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Finden Sie das **Stripe Terminal**-Gateway in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

## Verwendung

### Leser anschließen

Wenn Sie das Stripe Terminal-Gateway beim Checkout im POS auswählen:

1. **Verbindungsmethode wählen**: Sie können entweder einen physischen Leser anschließen oder den Simulator verwenden
2. **Physischer Leser**: Folgen Sie den Anweisungen auf dem Bildschirm, um Ihr Stripe Terminal-Gerät anzuschließen
3. **Simulator**: Wählen Sie den Simulator-Modus, um verschiedene Zahlungsszenarien ohne Hardware zu testen

### Zahlungen verarbeiten

1. **Artikel hinzufügen**: Fügen Sie Produkte zu Ihrem Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "Stripe Terminal" als Zahlungsmethode
3. **Leser anschließen**: Schließen Sie Ihren Leser an oder wählen Sie den Simulator-Modus
4. **Zahlung verarbeiten**: Befolgen Sie die Aufforderungen, um die Transaktion abzuschließen
5. **Bestätigung**: Die Bestellung wird automatisch nach erfolgreicher Zahlung abgeschlossen

### Testen mit dem Simulator

Der Simulator ermöglicht es Ihnen, verschiedene Zahlungsmethoden und Szenarien zu testen:

- **Kartenzahlungen**: Testen Sie verschiedene Kartentypen (Visa, Mastercard, etc.)
- **Kontaktlose Zahlungen**: Simulieren Sie Tap-to-Pay-Transaktionen
- **Fehler-Szenarien**: Testen Sie abgelehnte Zahlungen und andere Fehlerszenarien
- **Verschiedene Beträge**: Testen Sie verschiedene Transaktionsbeträge

## Anforderungen

- **Stripe-Konto**: Aktives Stripe-Konto mit aktiviertem Terminal
- **API-Schlüssel**: Stripe Secret Key von Ihrem Dashboard
- **WooCommerce POS**: Kostenlose oder Pro-Version installiert und aktiviert
- **HTTPS**: Ihre Site muss SSL/HTTPS für die Sicherheit verwenden

## Hardwarekompatibilität

:::important Anforderungen an die Konnektivität
Diese Implementierung verwendet Stripes JavaScript SDK, was bedeutet, dass sie über Webanwendungen funktioniert, benötigt aber **internetverbundene Terminals nur**. Bluetooth-Terminals werden derzeit nicht unterstützt.
:::

### Unterstützte Terminals (Internet-verbunden)

- **Stripe Reader S700** ✅ - Ethernet/WiFi verbundenes Terminal
- **WisePOS E** ✅ - WiFi verbundenes Terminal

### Nicht unterstützte Terminals (Bluetooth)

- **BBPOS Chipper 2X BT** ❌ - Bluetooth only
- **BBPOS WisePad 3** ❌ - Bluetooth only  
- **Verifone P400** ❌ - Bluetooth only

:::note Zukünftige Unterstützung
Die Unterstützung für Bluetooth-Terminals wird in **Q1 2026** zu den iOS- und Android-Anwendungen hinzugefügt. Dies ermöglicht die Unterstützung aller Stripe Terminal-zertifizierten Leser, einschließlich des M2 und WisePad 3.
:::

## Fehlersuche

### Häufige Probleme

**Leser lässt sich nicht verbinden**
- Stellen Sie sicher, dass Sie ein unterstütztes internetverbundenes Terminal (S700 oder WisePOS E) verwenden
- Überprüfen Sie, ob das Terminal mit WiFi/Ethernet verbunden und online ist
- Überprüfen Sie, ob Ihr Stripe-Konto das Terminal aktiviert hat
- Bestätigen Sie, dass das Terminal in Ihrem Stripe-Dashboard registriert ist

**Zahlung abgelehnt**
- Stellen Sie sicher, dass Ihr Stripe-Konto aktiv und in gutem Zustand ist
- Überprüfen Sie, ob die verwendete Karte gültig ist
- Stellen Sie sicher, dass ausreichend Geld vorhanden ist

**SSL-Zertifikatfehler**
- Stripe Terminal benötigt HTTPS - stellen Sie sicher, dass Ihre Site ein gültiges SSL-Zertifikat hat
- Überprüfen Sie, ob Ihr SSL-Zertifikat korrekt konfiguriert ist
