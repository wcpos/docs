# Web Checkout Gateway

Das Web Checkout-Gateway ermöglicht es Kunden, ihre POS-Transaktionen über das Checkout-System Ihres Webshops abzuschließen. Dieses Gateway schafft eine nahtlose Verbindung zwischen Ihrem stationären POS und Online-Zahlungsmethoden, ideal für Situationen, in denen Kunden lieber mit ihren eigenen Geräten bezahlen oder wenn Sie Zugriff auf ausschließlich webbasierte Zahlungsmethoden benötigen.

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [Web Checkout Gateway Releases-Seite](https://github.com/wcpos/web-checkout-gateway/releases)
2. Laden Sie die neueste Datei **woocommerce-pos-web-checkout-gateway.zip** herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="In WCPOS aktivieren" isLast>

1. Gehen Sie zu `WP Admin > POS > Einstellungen > Checkout`
2. Suchen Sie das **WCPOS Web Checkout Gateway** in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Gateway nutzt Ihre vorhandenen WooCommerce-Zahlungsmethoden, also stellen Sie sicher, dass Sie mindestens ein Online-Zahlungsgateway korrekt konfiguriert haben (Stripe, PayPal usw.).
:::

  </StepCard>
</Steps>

## Verwendung {#usage}

### Web Checkout Initiieren {#initiating-web-checkout}

1. **Artikel hinzufügen**: Produkte in Ihren Einkaufswagen im POS hinzufügen
2. **Gateway auswählen**: Wählen Sie "Web Checkout" als Zahlungsmethode
3. **Link generieren**: Das System erstellt einen einzigartigen Checkout-Link für die Bestellung
4. **Zugriff für Kunden**: Der Kunde klickt auf den Link, um auf den Web-Checkout zuzugreifen
5. **Online-Zahlung**: Der Kunde schließt die Zahlung mit den Zahlungsmethoden des Webshops ab

### Zahlungsprozess {#payment-process}

Das Web Checkout-Gateway folgt diesem Workflow:

1. **Bestellung erstellen**: POS erstellt eine ausstehende Bestellung in WooCommerce
2. **Checkout-Link**: Das System generiert einen sicheren, einzigartigen Zahlungslink
3. **Kundenweiterleitung**: Der Kunde wird zur Web-Checkout-Seite geleitet
4. **Zahlungsauswahl**: Der Kunde wählt aus den verfügbaren Web-Zahlungsmethoden
5. **Zahlungsabwicklung**: Der Standard-WooCommerce-Checkout-Prozess verarbeitet die Zahlung
6. **Bestellabschluss**: Der Bestellstatus wird automatisch nach erfolgreicher Zahlung aktualisiert

### POS-Workflow

Nach der Initiierung des Web-Checkouts:

1. **Status überwachen**: Halten Sie den POS-Bestellbildschirm geöffnet, um den Zahlungsstatus zu überwachen
2. **Zahlung durch Kunden**: Der Kunde beendet die Zahlung auf seinem Gerät oder Ihrem Tablet
3. **Automatische Aktualisierung**: Der Bestellstatus wird in Echtzeit aktualisiert, wenn die Zahlung eingegangen ist
4. **Zahlung verarbeiten**: Klicken Sie auf die Schaltfläche "Zahlung verarbeiten" im POS, um zu den Quittungen fortzufahren
5. **Quittungserstellung**: Quittung wie gewohnt erstellen und drucken

## Anwendungsfälle {#use-cases}

### Perfekt für {#perfect-for}

- **Kundenpräferenz**: Kunden, die ihr eigenes Zahlungs-Apps oder Karten verwenden möchten
- **Komplexe Zahlungen**: Transaktionen, die Zahlungsmethoden erfordern, die im POS nicht verfügbar sind
- **Teilzahlungen**: Kunden, die mehrere Zahlungsmethoden verwenden möchten
- **Treueprogramme**: Zugriff auf webbasierte loyale Punktetrocknung
- **Geschenkkarten**: Kunden mit digitalen Geschenkkarten oder Guthaben
- **Internationale Karten**: Zahlungsmethoden, die über Web-Gateways besser funktionieren

## Technische Details {#technical-details}

### Bestellmanagement {#order-management}

- **Ausstehende Bestellungen**: Bestellungen werden mit dem Status "Zahlung ausstehend" erstellt
- **Statusupdates**: Automatische Statusänderungen, wenn die Zahlung eingegangen ist
- **Bestellnotizen**: Zahlungsdetails werden in den Bestellnotizen festgehalten
- **Bestand**: Der Lagerbestand wird während des Checkout-Prozesses reserviert

### Sicherheit {#security}

- **Einzigartige Links**: Jede Checkout-Sitzung hat eine einzigartige, zeitlich begrenzte URL
- **SSL erforderlich**: Alle Zahlungsabwicklungen erfolgen über sichere HTTPS-Verbindungen
- **PCI-Konformität**: Nutzt Ihre vorhandenen PCI-konformen Web-Zahlungsgateways
- **Sitzungsmanagement**: Sichere Sitzungsbehandlung verhindert unbefugten Zugriff

### Zahlungsmethoden {#payment-methods}

Das Gateway bietet Zugriff auf alle konfigurierten WooCommerce-Zahlungsmethoden:

- **Kredit-/Debitkarten**: Stripe, Square, Authorize.net usw.
- **Digitale Geldbörsen**: PayPal, Apple Pay, Google Pay
- **Banküberweisungen**: Direkte Bankzahlungsmethoden
- **Jetzt kaufen, später bezahlen**: Klarna, Afterpay, Sezzle
- **Kryptowährung**: Bitcoin und andere Krypto-Zahlungsgateways
- **Lokale Methoden**: Regionale Zahlungsmethoden

## Fehlersuche {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Checkout-Link funktioniert nicht">

  - Überprüfen Sie, ob das SSL-Zertifikat korrekt installiert ist
  - Stellen Sie sicher, dass die WooCommerce-Permalinks korrekt konfiguriert sind
  - Überprüfen Sie, ob die Web-Zahlungsgateways aktiv und konfiguriert sind
  - Testen Sie mit einem anderen Browser oder Gerät

  </AccordionItem>

  <AccordionItem question="Zahlung wird nicht verarbeitet">

  - Bestätigen Sie, dass die Web-Zahlungsgateways korrekt konfiguriert sind
  - Überprüfen Sie die Protokolle der Zahlungsgateways auf Fehlermeldungen
  - Überprüfen Sie, ob die Zahlungsmethode des Kunden unterstützt wird
  - Testen Sie den Web-Checkout-Prozess unabhängig

  </AccordionItem>

  <AccordionItem question="Bestellung wird im POS nicht aktualisiert">

  - Überprüfen Sie die Stabilität der Internetverbindung
  - Überprüfen Sie, ob die WordPress-Cron-Jobs ordnungsgemäß ausgeführt werden
  - Aktualisieren Sie den POS-Bestellbildschirm
  - Überprüfen Sie den Bestellstatus im WooCommerce-Adminbereich

  </AccordionItem>

  <AccordionItem question="Kunde kann nicht auf den Checkout zugreifen">

  - Stellen Sie sicher, dass der Checkout-Link nicht abgelaufen ist
  - Überprüfen Sie, ob das Gerät des Kunden über eine Internetverbindung verfügt
  - Überprüfen Sie, dass die Bestellung nicht storniert oder abgeschlossen wurde
  - Testen Sie den Link auf einem anderen Gerät

  </AccordionItem>
</Accordion>

### Leistungsoptimierung {#performance-optimisation}

Für optimale Leistung:

- **Caching**: Konfigurieren Sie Caching-Plugins, um Checkout-Seiten auszuschließen
- **CDN**: Stellen Sie sicher, dass CDN-Einstellungen den Checkout-Prozess nicht stören
- **Datenbank**: Optimieren Sie die Datenbank für schnellere Bestellbearbeitung
- **Hosting**: Verwenden Sie zuverlässiges Hosting mit guter Verfügbarkeit

### Hilfe erhalten {#getting-help}

- Überprüfen Sie die Dokumentation der WooCommerce-Zahlungsgateways
