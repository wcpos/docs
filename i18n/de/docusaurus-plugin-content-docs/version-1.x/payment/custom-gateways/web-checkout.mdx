# Web Checkout Gateway

Das Web Checkout Gateway ermöglicht es Kunden, ihre POS-Transaktionen über das Checkout-System Ihres Webshops abzuschließen. Dieses Gateway schafft eine nahtlose Verbindung zwischen Ihrem stationären POS und Online-Zahlungsmethoden, ideal für Situationen, in denen Kunden lieber mit ihren eigenen Geräten bezahlen oder wenn Sie Zugang zu webbasierten Zahlungsmethoden benötigen.

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [Web Checkout Gateway Veröffentlichungsseite](https://github.com/wcpos/web-checkout-gateway/releases)
2. Laden Sie die neueste **woocommerce-pos-web-checkout-gateway.zip** Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die ZIP-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="In WooCommerce POS aktivieren" isLast>

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Finden Sie das **WooCommerce POS Web Checkout Gateway** in der Liste
3. Aktivieren Sie das Gateway zur Nutzung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Gateway nutzt Ihre vorhandenen WooCommerce-Zahlungsmethoden, stellen Sie daher sicher, dass Sie mindestens ein Web-Zahlungsgateway ordnungsgemäß konfiguriert haben (Stripe, PayPal, usw.).
:::

  </StepCard>
</Steps>

## Nutzung

### Web Checkout Initiieren

1. **Artikel hinzufügen**: Fügen Sie Produkte in den Einkaufswagen im POS hinzu
2. **Gateway auswählen**: Wählen Sie "Web Checkout" als Zahlungsmethode
3. **Link generieren**: Das System erstellt einen einzigartigen Checkout-Link für die Bestellung
4. **Zugriff für Kunden**: Der Kunde klickt auf den Link, um auf den Web-Checkout zuzugreifen
5. **Online-Zahlung**: Der Kunde schließt die Zahlung mit den Zahlungsmethoden des Webshops ab

### Zahlungsprozess

Das Web Checkout-Gateway folgt diesem Workflow:

1. **Bestelloberfläche**: POS erstellt eine ausstehende Bestellung in WooCommerce
2. **Checkout-Link**: Das System generiert einen sicheren, einzigartigen Zahlungslink
3. **Kundenumleitung**: Der Kunde wird zur Web-Checkout-Seite geleitet
4. **Zahlungsauswahl**: Der Kunde wählt aus den verfügbaren webbasierten Zahlungsmethoden
5. **Zahlungsabwicklung**: Der Standard-Zahlungsprozess von WooCommerce verarbeitet die Zahlung
6. **Bestellabschluss**: Der Bestellstatus wird nach erfolgreicher Zahlung automatisch aktualisiert

### POS-Workflow

Nach der Einleitung des Web-Checkouts:

1. **Status überwachen**: Halten Sie den Bestellbildschirm des POS geöffnet, um den Zahlungsstatus zu überwachen
2. **Zahlung des Kunden**: Der Kunde schließt die Zahlung auf seinem Gerät oder Ihrem Tablet ab
3. **Automatische Updates**: Der Bestellstatus wird in Echtzeit aktualisiert, wenn die Zahlung eingeht
4. **Zahlung verarbeiten**: Klicken Sie auf die Schaltfläche "Zahlung verarbeiten" im POS, um zum Beleg fortzufahren
5. **Belegerstellung**: Erstellen und drucken Sie den Beleg wie gewohnt

## Anwendungsfälle

### Perfekt für

- **Kundenpräferenz**: Kunden, die ihre eigenen Zahlungs-Apps oder Karten verwenden möchten
- **Komplexe Zahlungen**: Transaktionen, die Zahlungsmethoden erfordern, die im POS nicht verfügbar sind
- **Teilzahlungen**: Kunden, die mehrere Zahlungsmethoden verwenden möchten
- **Treueprogramme**: Zugriff auf webbasierte Punkteeinlösungen 
- **Geschenkkarten**: Kunden mit digitalen Geschenkkarten oder Guthaben
- **Internationale Karten**: Zahlungsmethoden, die über Web-Gateways besser funktionieren

## Technische Details

### Auftragsverwaltung

- **Ausstehende Aufträge**: Bestellungen werden mit dem Status "ausstehende Zahlung" erstellt
- **Statusaktualisierungen**: Automatische Statusänderungen, wenn die Zahlung eingeht
- **Auftragsnotizen**: Zahlungsdetails werden in den Auftragsnotizen erfasst
- **Bestand**: Der Lagerbestand wird während des Checkout-Prozesses reserviert

### Sicherheit

- **Eindeutige Links**: Jede Checkout-Sitzung hat eine eindeutige, zeitlich begrenzte URL
- **SSL erforderlich**: Alle Zahlungsabwicklungen erfolgen über sichere HTTPS-Verbindungen
- **PCI-Konformität**: Nutzt Ihre bestehenden PCI-konformen Web-Zahlungsgateways
- **Sitzungsverwaltung**: Sichere Sitzungsverwaltung verhindert unbefugten Zugang

### Zahlungsmethoden

Das Gateway bietet Zugriff auf alle konfigurierten WooCommerce-Zahlungsmethoden:

- **Kredit-/Debitkarten**: Stripe, Square, Authorize.net, usw.
- **Digitale Geldbörsen**: PayPal, Apple Pay, Google Pay
- **Banküberweisungen**: Direkte Bankzahlungsarten
- **Jetzt kaufen, später bezahlen**: Klarna, Afterpay, Sezzle
- **Kryptowährung**: Bitcoin und andere Krypto-Zahlungsgateways
- **Lokale Methoden**: Regionsspezifische Zahlungsmethoden

## Fehlersuche

### Häufige Probleme

<Accordion>
  <AccordionItem question="Checkout-Link funktioniert nicht">

  - Überprüfen Sie, ob das SSL-Zertifikat ordnungsgemäß installiert ist
  - Überprüfen Sie, ob die WooCommerce-Permalinks korrekt konfiguriert sind
  - Stellen Sie sicher, dass die Web-Zahlungsgateways aktiv und konfiguriert sind
  - Testen Sie mit einem anderen Browser oder Gerät

  </AccordionItem>

  <AccordionItem question="Zahlung wird nicht verarbeitet">

  - Bestätigen Sie, dass die Web-Zahlungsgateways ordnungsgemäß konfiguriert sind
  - Überprüfen Sie die Protokolle des Zahlungsgateways auf Fehlermeldungen
  - Überprüfen Sie, ob die Zahlungsmethode des Kunden unterstützt wird
  - Testen Sie den Web-Checkout-Prozess unabhängig

  </AccordionItem>

  <AccordionItem question="Bestellung wird im POS nicht aktualisiert">

  - Überprüfen Sie die Stabilität der Internetverbindung
  - Überprüfen Sie, ob die WordPress-Cron-Jobs ordnungsgemäß ausgeführt werden
  - Aktualisieren Sie den Bestellbildschirm im POS
  - Überprüfen Sie den Bestellstatus im WooCommerce-Admin

  </AccordionItem>

  <AccordionItem question="Kunde kann nicht auf den Checkout zugreifen">

  - Stellen Sie sicher, dass der Checkout-Link nicht abgelaufen ist
  - Überprüfen Sie, ob das Gerät des Kunden eine Internetverbindung hat
  - Überprüfen Sie, ob die Bestellung nicht storniert oder abgeschlossen wurde
  - Testen Sie den Link auf einem anderen Gerät

  </AccordionItem>
</Accordion>

### Leistungsoptimierung

Für eine optimale Leistung:

- **Caching**: Konfigurieren Sie Caching-Plugins, um Checkout-Seiten auszuschließen
- **CDN**: Stellen Sie sicher, dass die CDN-Einstellungen den Checkout-Prozess nicht stören
- **Datenbank**: Optimieren Sie die Datenbank für eine schnellere Auftragsbearbeitung
- **Hosting**: Verwenden Sie zuverlässiges Hosting mit guter Verfügbarkeit
