# Web Checkout Gateway

Das Web Checkout-Gateway ermöglicht es Kunden, ihre POS-Transaktionen über das Checkout-System Ihres Webshops abzuschließen. Dieses Gateway schafft eine nahtlose Verbindung zwischen Ihrem stationären POS und Online-Zahlungsmethoden und ist perfekt für Situationen, in denen Kunden bevorzugen, mit ihren eigenen Geräten zu bezahlen oder wenn Sie Zugriff auf ausschließlich webbasierte Zahlungsmethoden benötigen.

## Funktionen

- **Nahtlose Integration**: Direkte Verbindung vom POS zum Web-Checkout
- **Alle Zahlungsmethoden**: Zugriff auf alle aktivierten Zahlungsmethoden Ihres Webshops
- **Kundenkontrolle**: Kunden können mit ihren eigenen Geräten und bevorzugten Methoden bezahlen
- **Bestellsynchronisierung**: Automatische Bestellaktualisierungen zwischen POS und Webshop
- **Flexibler Workflow**: Transaktionen online abschließen und den POS-Workflow beibehalten

## Installation

### Schritt 1: Herunterladen und Installieren

1. Besuchen Sie die [Web Checkout Gateway Release-Seite](https://github.com/wcpos/web-checkout-gateway/releases)
2. Laden Sie die neueste **woocommerce-pos-web-checkout-gateway.zip**-Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

### Schritt 2: In WooCommerce POS aktivieren

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Finden Sie das **WooCommerce POS Web Checkout Gateway** in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Gateway nutzt Ihre vorhandenen WooCommerce-Zahlungsmethoden, stellen Sie daher sicher, dass Sie mindestens ein Web-Zahlungsgateway ordnungsgemäß konfiguriert haben (Stripe, PayPal usw.).
:::

## Verwendung

### Web Checkout initiieren

1. **Produkte hinzufügen**: Fügen Sie Produkte zu Ihrem Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "Web Checkout" als Zahlungsmethode
3. **Link generieren**: Das System erstellt einen einzigartigen Checkout-Link für die Bestellung
4. **Zugriff des Kunden**: Der Kunde klickt auf den Link, um auf den Web-Checkout zuzugreifen
5. **Online-Zahlung**: Der Kunde schließt die Zahlung über die Zahlungsmethoden des Webshops ab

### Zahlungsprozess

Das Web Checkout-Gateway folgt diesem Workflow:

1. **Bestellerstellung**: Der POS erstellt eine ausstehende Bestellung in WooCommerce
2. **Checkout-Link**: Das System generiert einen sicheren, einzigartigen Zahlungslink
3. **Kundenweiterleitung**: Der Kunde wird zur Web-Checkout-Seite geleitet
4. **Zahlungsauswahl**: Der Kunde wählt aus den verfügbaren Web-Zahlungsmethoden
5. **Zahlungsverarbeitung**: Standardmäßiger WooCommerce-Checkout-Prozess bearbeitet die Zahlung
6. **Bestellabschluss**: Der Bestellstatus wird automatisch nach erfolgreicher Zahlung aktualisiert

### POS-Workflow

Nach der Initiierung des Web-Checkouts:

1. **Status überwachen**: Halten Sie den Bestellbildschirm des POS geöffnet, um den Zahlungsstatus zu überwachen
2. **Zahlung des Kunden**: Der Kunde schließt die Zahlung auf seinem Gerät oder Ihrem Tablet ab
3. **Automatische Aktualisierung**: Der Bestellstatus wird in Echtzeit aktualisiert, wenn die Zahlung eingegangen ist
4. **Zahlung verarbeiten**: Klicken Sie auf die Schaltfläche "Zahlung verarbeiten" im POS, um mit dem Beleg fortzufahren
5. **Belegerstellung**: Generieren und drucken Sie den Beleg wie gewohnt

## Anwendungsfälle

### Perfekt für

- **Kundenpräferenzen**: Kunden, die ihre eigenen Zahlungs-Apps oder Karten verwenden möchten
- **Komplexe Zahlungen**: Transaktionen, die Zahlungsmethoden erfordern, die im POS nicht verfügbar sind
- **Geteilte Zahlungen**: Kunden, die mehrere Zahlungsmethoden verwenden möchten
- **Loyalitätsprogramme**: Zugang zu webbasiertem Loyalitätspunkte-Einlösen
- **Geschenkkarten**: Kunden mit digitalen Geschenkkarten oder Guthaben
- **Internationale Karten**: Zahlungsmethoden, die über Web-Gateways besser funktionieren

### Workflow-Beispiele

**Einzelhandelsgeschäft**
1. Kunde kauft im Geschäft ein und bringt Artikel zur Kasse
2. Kunde möchte mit seiner mobilen Wallet-App bezahlen
3. Mitarbeiter wählt Web Checkout-Gateway aus
4. Kunde scannt den QR-Code oder klickt auf den Link auf seinem Telefon
5. Kunde schließt die Zahlung mit der bevorzugten Methode ab
6. Mitarbeiter verarbeitet den Beleg und schließt die Transaktion ab

**Dienstleistungsunternehmen**
1. Dienstleistung abschließen und Kosten im POS hinzufügen
2. Kunde möchte mit der Geschäftskreditkarte bezahlen
3. Generieren Sie den Web-Checkout-Link
4. Kunde gibt Zahlungsdetails in das sichere Webformular ein
5. Zahlung wird über Ihr Web-Gateway verarbeitet
6. Bestellung wird abgeschlossen und der Beleg wird erstellt

## Technische Details

### Bestellmanagement

- **Ausstehende Bestellungen**: Bestellungen werden mit dem Status "Zahlung ausstehend" erstellt
- **Statusaktualisierungen**: Automatische Statusänderungen, wenn die Zahlung eingegangen ist
- **Bestellnotizen**: Zahlungsdetails werden in den Bestellnotizen aufgezeichnet
- **Inventar**: Bestände werden während des Checkout-Prozesses reserviert

### Sicherheit

- **Einzigartige Links**: Jede Checkout-Sitzung hat eine einzigartige, zeitlich begrenzte URL
- **SSL erforderlich**: Alle Zahlungsabwicklungen erfolgen über sichere HTTPS-Verbindungen
- **PCI-Konformität**: Nutzt Ihre vorhandenen PCI-konformen Web-Zahlungsgateways
- **Sitzungsverwaltung**: Sichere Sitzungsbehandlung verhindert unbefugten Zugriff

### Zahlungsmethoden

Das Gateway bietet Zugriff auf alle Ihre konfigurierten WooCommerce-Zahlungsmethoden:

- **Kredit-/Debitkarten**: Stripe, Square, Authorize.net usw.
- **Digitale Geldbörsen**: PayPal, Apple Pay, Google Pay
- **Banküberweisungen**: Direkte Bankzahlungsarten
- **Jetzt kaufen, später bezahlen**: Klarna, Afterpay, Sezzle
- **Kryptowährung**: Bitcoin und andere Krypto-Zahlungsgateways
- **Lokale Methoden**: Regionsspezifische Zahlungsmethoden

## Anforderungen

- **WooCommerce POS**: Kostenlose oder Pro-Version installiert und aktiviert
- **Web-Zahlungsgateways**: Mindestens eine Online-Zahlungsmethode in WooCommerce konfiguriert
- **SSL-Zertifikat**: HTTPS erforderlich für sichere Zahlungsabwicklung
- **Moderner Browser**: Das Kundengerät muss moderne Webstandards unterstützen

## Fehlersuche

### Häufige Probleme

**Checkout-Link funktioniert nicht**
- Überprüfen Sie, ob das SSL-Zertifikat ordnungsgemäß installiert ist
- Stellen Sie sicher, dass die WooCommerce-Permalinks korrekt konfiguriert sind
- Überprüfen Sie, ob die Web-Zahlungsgateways aktiv und konfiguriert sind
- Testen Sie mit einem anderen Browser oder Gerät

**Zahlung wird nicht verarbeitet**
- Bestätigen Sie, dass die Web-Zahlungsgateways ordnungsgemäß konfiguriert sind
- Überprüfen Sie die Protokolle der Zahlungsgateways auf Fehlermeldungen
- Stellen Sie sicher, dass die Zahlungsmethode des Kunden unterstützt wird
- Testen Sie den Web-Checkout-Prozess unabhängig

**Bestellung wird im POS nicht aktualisiert**
- Überprüfen Sie die Stabilität der Internetverbindung
- Stellen Sie sicher, dass die WordPress-Cron-Jobs ordnungsgemäß ausgeführt werden
- Aktualisieren Sie den Bestellbildschirm im POS
- Überprüfen Sie den Bestellstatus im WooCommerce-Admin

**Kunde kann nicht auf den Checkout zugreifen**
- Stellen Sie sicher, dass der Checkout-Link nicht abgelaufen ist
- Überprüfen Sie, ob das Gerät des Kunden über eine Internetverbindung verfügt
- Überprüfen Sie, ob die Bestellung nicht storniert oder abgeschlossen wurde
- Testen Sie den Link auf einem anderen Gerät

### Leistungsoptimierung

Für optimale Leistung:

- **Caching**: Konfigurieren Sie Caching-Plugins, um Checkout-Seiten auszuschließen
- **CDN**: Stellen Sie sicher, dass die CDN-Einstellungen den Checkout-Prozess nicht stören
- **Datenbank**: Optimieren Sie die Datenbank für eine schnellere Auftragsbearbeitung
- **Hosting**: Verwenden Sie zuverlässiges Hosting mit guter Verfügbarkeit

### Unterstützung erhalten

Für technischen Support:
- Besuchen Sie das [GitHub-Repository](https://github.com/wcpos/web-checkout-gateway), um Probleme zu melden
- Überprüfen Sie die Dokumentation der WooCommerce-Zahlungsgateways
- Testen Sie Ihren Web-Checkout-Prozess regelmäßig
- Überwachen Sie die Protokolle der Zahlungsgateways auf Probleme
