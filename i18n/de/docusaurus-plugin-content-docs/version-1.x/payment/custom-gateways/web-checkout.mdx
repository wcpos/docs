# Web Checkout Gateway

Das Web Checkout-Gateway ermöglicht es Kunden, ihre POS-Transaktionen mit dem Checkout-System Ihres Webshops abzuschließen. Dieses Gateway schafft eine nahtlose Verbindung zwischen Ihrem stationären POS und Online-Zahlungsmethoden, perfekt für Situationen, in denen Kunden lieber mit ihren eigenen Geräten bezahlen oder wenn Sie Zugriff auf ausschließlich webbasierte Zahlungsmethoden benötigen.

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [Web Checkout Gateway Releases-Seite](https://github.com/wcpos/web-checkout-gateway/releases)
2. Laden Sie die neueste **woocommerce-pos-web-checkout-gateway.zip** Datei herunter
3. Gehen Sie in Ihrem WordPress-Adminbereich zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="Aktivieren in WooCommerce POS" isLast>

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Suchen Sie das **WooCommerce POS Web Checkout Gateway** in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Gateway nutzt Ihre vorhandenen WooCommerce-Zahlungsmethoden, stellen Sie also sicher, dass Sie mindestens einen Web-Zahlungsgateway richtig konfiguriert haben (Stripe, PayPal usw.).
:::

  </StepCard>
</Steps>

## Verwendung {#usage}

### Web-Checkout starten {#initiating-web-checkout}

1. **Artikel hinzufügen**: Fügen Sie Produkte in den POS-Warenkorb hinzu
2. **Gateway auswählen**: Wählen Sie „Web Checkout“ als Zahlungsmethode
3. **Link generieren**: Das System erstellt einen einzigartigen Checkout-Link für die Bestellung
4. **Zugriff des Kunden**: Der Kunde klickt auf den Link, um auf den Web-Checkout zuzugreifen
5. **Online-Zahlung**: Der Kunde schließt die Zahlung mit den Zahlungsmethoden des Webshops ab

### Zahlungsprozess {#payment-process}

Das Web Checkout-Gateway folgt diesem Workflow:

1. **Bestellung erstellen**: POS erstellt eine ausstehende Bestellung in WooCommerce
2. **Checkout-Link**: Das System generiert einen sicheren, einzigartigen Zahlungslink
3. **Weiterleitung des Kunden**: Der Kunde wird zur Web-Checkout-Seite geleitet
4. **Zahlungsauswahl**: Der Kunde wählt aus den verfügbaren Web-Zahlungsmethoden
5. **Zahlungsabwicklung**: Der Standard-WooCommerce-Checkoutprozess übernimmt die Zahlung
6. **Bestellabschluss**: Der Bestellstatus wird automatisch nach erfolgreicher Zahlung aktualisiert

### POS-Workflow

Nach dem Start des Web-Checkouts:

1. **Status überwachen**: Halten Sie den POS-Bestellbildschirm geöffnet, um den Zahlungsstatus zu überwachen
2. **Zahlung des Kunden**: Der Kunde schließt die Zahlung auf seinem Gerät oder Ihrem Tablet ab
3. **Automatische Aktualisierung**: Der Bestellstatus wird in Echtzeit aktualisiert, wenn die Zahlung eingeht
4. **Zahlung verarbeiten**: Klicken Sie auf die Schaltfläche "Zahlung verarbeiten" im POS, um zum Beleg zu gelangen
5. **Beleggenerierung**: Generieren und drucken Sie den Beleg wie gewohnt

## Anwendungsfälle {#use-cases}

- **Kundenpräferenz**: Kunden, die ihre eigenen Zahlungsapps oder Karten verwenden möchten

## Technische Details {#technical-details}

### Bestellverwaltung {#order-management}

- **Ausstehende Bestellungen**: Bestellungen werden mit dem Status „ausstehende Zahlung“ erstellt
- **Statusaktualisierungen**: Automatische Statusänderungen, wenn die Zahlung eingeht
- **Bestellnotizen**: Zahlungsdetails werden in den Bestellnotizen aufgezeichnet
- **Inventar**: Lagerbestand wird während des Checkout-Prozesses reserviert

### Sicherheit {#security}

- **Eindeutige Links**: Jede Checkout-Sitzung hat eine eindeutige, zeitlich begrenzte URL
- **SSL erforderlich**: Alle Zahlungsabwicklungen erfolgen über sichere HTTPS-Verbindungen
- **PCI-Compliance**: Nutzt Ihre vorhandenen PCI-konformen Web-Zahlungsgateways
- **Sitzungsmanagement**: Sicheres Sitzungsmanagement verhindert unbefugten Zugriff

### Zahlungsmethoden {#payment-methods}

Das Gateway bietet Zugang zu allen konfigurierten WooCommerce-Zahlungsmethoden:

- **Kredit-/Debitkarten**: Stripe, Square, Authorize.net usw.
- **Digitale Wallets**: PayPal, Apple Pay, Google Pay
- **Banküberweisungen**: Direkte Bankzahlungsmethoden
- **Jetzt kaufen, später zahlen**: Klarna, Afterpay, Sezzle
- **Kryptowährung**: Bitcoin und andere Krypto-Zahlungsgateways
- **Lokale Methoden**: Regionale Zahlungsmethoden

## Fehlersuche {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Checkout-Link funktioniert nicht">

  - Überprüfen Sie, ob das SSL-Zertifikat korrekt installiert ist
  - Stellen Sie sicher, dass die WooCommerce-Permalinks korrekt konfiguriert sind
  - Überprüfen Sie, ob die Web-Zahlungsgateways aktiv und konfiguriert sind
  - Testen Sie es mit einem anderen Browser oder Gerät

  </AccordionItem>

  <AccordionItem question="Zahlung wird nicht verarbeitet">

  - Bestätigen Sie, dass die Web-Zahlungsgateways richtig konfiguriert sind
  - Überprüfen Sie die Protokolle des Zahlungsgateways auf Fehlermeldungen
  - Überprüfen Sie, ob die Zahlungsmethode des Kunden unterstützt wird
  - Testen Sie den Web-Checkout-Prozess unabhängig

  </AccordionItem>

  <AccordionItem question="Bestellung wird im POS nicht aktualisiert">

  - Überprüfen Sie die Stabilität der Internetverbindung
  - Stellen Sie sicher, dass die WordPress-Cronjobs ordnungsgemäß ausgeführt werden
  - Aktualisieren Sie den POS-Bestellbildschirm
  - Überprüfen Sie den Bestellstatus im WooCommerce-Admin

  </AccordionItem>

  <AccordionItem question="Kunde kann nicht auf den Checkout zugreifen">

  - Stellen Sie sicher, dass der Checkout-Link nicht abgelaufen ist
  - Überprüfen Sie, ob das Gerät des Kunden eine Internetverbindung hat
  - Überprüfen Sie, ob die Bestellung nicht annulliert oder abgeschlossen wurde
  - Testen Sie den Link auf einem anderen Gerät

  </AccordionItem>
</Accordion>

### Leistungsoptimierung {#performance-optimisation}

Für optimale Leistung:

- **Caching**: Konfigurieren Sie Caching-Plugins so, dass Checkout-Seiten ausgeschlossen werden
- **CDN**: Stellen Sie sicher, dass die CDN-Einstellungen den Checkout-Prozess nicht stören
- **Datenbank**: Optimieren Sie die Datenbank für schnellere Bestellverarbeitung
- **Hosting**: Verwenden Sie zuverlässiges Hosting mit guter Verfügbarkeit
