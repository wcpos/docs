# SumUp Terminal Gateway

Das SumUp Terminal-Gateway ermöglicht es Ihnen, Kartenzahlungen mit SumUp-Kartenlesern direkt in WCPOS zu akzeptieren. Dieses Gateway bietet eine nahtlose Integration mit dem Zahlungsabwicklungssystem von SumUp für persönliche Transaktionen.

## Funktionen {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Hardware-Integration">
    Verbinden Sie SumUp-Kartenleser über eine Internetverbindung mit Ihrem POS-System
  </FeatureCard>
  <FeatureCard icon="bolt" title="Echtzeitverarbeitung">
    Sofortige Zahlungsabwicklung und Auftragsabschluss
  </FeatureCard>
  <FeatureCard icon="shield" title="Sichere Transaktionen">
    PCI-konforme Zahlungsabwicklung über SumUp
  </FeatureCard>
  <FeatureCard icon="link" title="Einfache Kopplung">
    Einfacher Kopplungsprozess für Geräte mit Kopplungscodes
  </FeatureCard>
</FeatureGrid>

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [SumUp Terminal für WooCommerce Release-Seite](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Laden Sie die neueste **sumup-terminal-for-woocommerce.zip**-Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin-Bereich zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="SumUp-Einstellungen konfigurieren">

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Suchen Sie **SumUp Terminal** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **SumUp Terminal**, um die Einstellungen zuzugreifen
4. Geben Sie Ihren **SumUp API-Schlüssel** ein (verfügbar in Ihrem SumUp-Händlerdashboard)
5. Speichern Sie die Einstellungen

:::note
Sie müssen das SumUp Terminal-Gateway in den WooCommerce-Einstellungen nicht aktivieren. Es wird speziell für das POS in einem späteren Schritt aktiviert.
:::

  </StepCard>
  <StepCard number="3" title="Koppeln Sie Ihr SumUp Terminal">

1. Suchen Sie auf derselben Einstellungsseite den Bereich **Leser koppeln**
2. Navigieren Sie auf Ihrem SumUp-Gerät zum Koppelschirm, um den Koppelschlüssel anzuzeigen
3. Geben Sie den auf Ihrem SumUp-Gerät angezeigten Koppelschlüssel ein
4. Klicken Sie auf **"Leser koppeln"**, um die Verbindung herzustellen
5. Warten Sie auf die Bestätigung, dass der Leser erfolgreich gekoppelt wurde

:::warning Wichtig
Der Leser muss erfolgreich gekoppelt sein, bevor Sie Zahlungen verarbeiten können. Stellen Sie sicher, dass der Kopplungsprozess abgeschlossen ist, bevor Sie fortfahren.
:::

  </StepCard>
  <StepCard number="4" title="Aktivieren in WCPOS" isLast>

1. Gehen Sie zu `WP Admin > POS > Einstellungen > Kasse`
2. Suchen Sie das **SumUp Terminal**-Gateway in der Liste
3. Aktivieren Sie das Gateway zur Nutzung im POS
4. Speichern Sie Ihre Einstellungen

  </StepCard>
</Steps>

## Nutzung {#usage}

### Zahlungen verarbeiten {#processing-payments}

1. **Artikel hinzufügen**: Fügen Sie Produkte zu Ihrem Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "SumUp Terminal" als Zahlungsmethode
3. **Zahlung starten**: Klicken Sie, um eine neue Zahlung auf Ihrem SumUp-Gerät zu starten
4. **Zahlung des Kunden**: Der Kunde schließt die Zahlung am SumUp-Terminal ab
5. **Automatische Vervollständigung**: Der Auftrag wird automatisch abgeschlossen, sobald die Zahlung erkannt wird

### Zahlungssteuerungen {#payment-controls}

Beim Einsatz des SumUp Terminal-Gateways haben Sie die folgenden Optionen:

- **Neue Zahlung starten**: Initiieren Sie eine Zahlungsanforderung auf dem verbundenen Terminal
- **Zahlung abbrechen**: Brechen Sie eine derzeit laufende Zahlung ab
- **Zahlungsstatus**: Überwachen Sie den aktuellen Status des Zahlungsprozesses

### Auftragsverwaltung {#order-management}

- **Automatische Updates**: Aufträge werden automatisch als abgeschlossen markiert, sobald die Zahlung erfolgreich ist
- **Zahlungsverfolgung**: Alle Zahlungsdetails werden in den Auftragsnotizen erfasst
- **Beleggenerierung**: Standard-POS-Belege werden nach erfolgreichen Zahlungen erstellt

## Hardware-Kompatibilität {#hardware-compatibility}

:::important Anforderungen an die Konnektivität
Diese Implementierung verwendet das JavaScript-SDK von SumUp, was bedeutet, dass es über Webanwendungen funktioniert, jedoch **nur über internetverbundene Terminals**. Bluetooth-Terminals werden derzeit nicht unterstützt.
:::

### Unterstützte Terminals (Internet-verbunden) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet-verbundenes Terminal

### Nicht unterstützte Terminals (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Nur Bluetooth
- **SumUp 3G** ❌ - Nur Bluetooth

:::note Zukünftige Unterstützung
Die Unterstützung für Bluetooth-Terminals wird in den iOS- und Android-Anwendungen im **Q1 2026** hinzugefügt. Dadurch wird die Unterstützung für alle SumUp-zertifizierten Terminals, einschließlich der Modelle Air und 3G, ermöglicht.
:::

## Fehlerbehebung {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Leser lässt sich nicht koppeln">

  - Stellen Sie sicher, dass Sie ein unterstütztes, internetverbundenes Terminal verwenden (nur Solo)
  - Überprüfen Sie, ob Ihr SumUp Solo-Gerät mit WiFi verbunden und online ist
  - Stellen Sie sicher, dass das Gerät im Kopplungsmodus ist und der Koppelschlüssel korrekt eingegeben wurde
  - Bestätigen Sie, dass Ihr SumUp-Konto API-Zugang aktiviert hat
  - Stellen Sie eine stabile Internetverbindung während der Kopplung sicher

  </AccordionItem>

  <AccordionItem question="Zahlung wird nicht verarbeitet">

  - Überprüfen Sie, dass der Leser richtig gekoppelt ist
  - Stellen Sie sicher, dass Ihr SumUp-Konto aktiv und in gutem Zustand ist
  - Stellen Sie sicher, dass das Gerät über ausreichend Akku und Konnektivität verfügt
  - Bestätigen Sie, dass der API-Schlüssel korrekt und aktiv ist

  </AccordionItem>

  <AccordionItem question="Bestellung wird nicht abgeschlossen">

  - Überprüfen Sie die Stabilität der Internetverbindung
  - Überprüfen Sie die Webhook-Einstellungen in Ihrem SumUp-Konto
  - Stellen Sie sicher, dass die Zahlung am SumUp-Terminal erfolgreich war
  - Überprüfen Sie die WordPress-Fehlerprotokolle auf API-Kommunikationsprobleme

  </AccordionItem>
</Accordion>

### API-Rate-Limits

Das Gateway ist optimiert, um API-Aufrufe zu reduzieren und Ratenbeschränkungen zu vermeiden:
- Der Zahlungsstatus wird effizient überprüft
- Unnötige API-Anfragen werden minimiert
- Automatische Wiederholungslogik behandelt temporäre Fehler
