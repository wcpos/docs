# SumUp Terminal Gateway

Das SumUp Terminal-Gateway ermöglicht es Ihnen, Kartenzahlungen über SumUp-Kartenlesegeräte direkt innerhalb von WooCommerce POS zu akzeptieren. Dieses Gateway bietet eine nahtlose Integration mit dem Zahlungssystem von SumUp für persönliche Transaktionen.

## Funktionen {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Hardware-Integration">
    Verbinden Sie SumUp-Kartenlesegeräte über eine Internetverbindung mit Ihrem POS-System
  </FeatureCard>
  <FeatureCard icon="bolt" title="Echtzeitverarbeitung">
    Sofortige Zahlungsabwicklung und Auftragsabschluss
  </FeatureCard>

  </FeatureCard>
</FeatureGrid>

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [Seite mit den Releases für SumUp Terminal für WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Laden Sie die neueste **sumup-terminal-for-woocommerce.zip**-Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="SumUp-Einstellungen konfigurieren">

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Finden Sie **SumUp Terminal** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **SumUp Terminal**, um die Einstellungen aufzurufen
4. Geben Sie Ihren **SumUp API-Schlüssel** ein (verfügbar in Ihrem SumUp-Händlersdashboard)
5. Speichern Sie die Einstellungen

:::note
Sie müssen das SumUp Terminal-Gateway in den WooCommerce-Einstellungen nicht aktivieren. Es wird speziell für den POS in einem späteren Schritt aktiviert.
:::

  </StepCard>
  <StepCard number="3" title="Koppeln Sie Ihr SumUp Terminal">

1. Suchen Sie auf derselben Einstellungsseite den Abschnitt **Leser koppeln**
2. Navigieren Sie auf Ihrem SumUp-Gerät zum Kopplungsbildschirm, um den Kopplungscode anzuzeigen
3. Geben Sie den auf Ihrem SumUp-Gerät angezeigten Kopplungscode ein
4. Klicken Sie auf **"Leser koppeln"**, um die Verbindung herzustellen
5. Warten Sie auf die Bestätigung, dass der Leser erfolgreich gekoppelt wurde

:::warning Wichtig
Der Leser muss erfolgreich gekoppelt werden, bevor Sie Zahlungen verarbeiten können. Stellen Sie sicher, dass der Kopplungsprozess abgeschlossen ist, bevor Sie fortfahren.
:::

  </StepCard>
  <StepCard number="4" title="Aktivieren in WooCommerce POS" isLast>

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Finden Sie das **SumUp Terminal**-Gateway in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

  </StepCard>
</Steps>

## Verwendung {#usage}

### Zahlungsverarbeitung {#processing-payments}

1. **Artikel hinzufügen**: Fügen Sie Produkte zu Ihrem Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "SumUp Terminal" als Zahlungsmethode
3. **Zahlung starten**: Klicken Sie, um eine neue Zahlung auf Ihrem SumUp-Gerät einzuleiten
4. **Zahlung des Kunden**: Kunde schließt die Zahlung am SumUp-Terminal ab
5. **Automatische Fertigstellung**: Der Auftrag wird automatisch abgeschlossen, wenn die Zahlung erkannt wird

### Zahlungssteuerungen {#payment-controls}

Bei der Verwendung des SumUp Terminal-Gateways haben Sie die folgenden Optionen:

- **Neue Zahlung starten**: Initiieren Sie eine Zahlungsanfrage auf dem verbundenen Terminal
- **Zahlung stornieren**: Stornieren Sie eine aktuell in Bearbeitung befindliche Zahlung
- **Zahlungsstatus**: Überwachen Sie den aktuellen Status des Zahlungsprozesses

### Auftragsverwaltung {#order-management}

- **Automatische Updates**: Bestellungen werden automatisch als abgeschlossen markiert, sobald die Zahlung erfolgreich ist
- **Zahlungsverfolgung**: Alle Zahlungsdetails werden in den Bestellnotizen aufgezeichnet
- **Quittungserstellung**: Standard-POS-Quittungen werden nach erfolgreichen Zahlungen erstellt

## Hardware-Kompatibilität {#hardware-compatibility}

:::important Verbindungsanforderungen
Diese Implementierung verwendet das JavaScript-SDK von SumUp, was bedeutet, dass es über Webanwendungen funktioniert, jedoch **nur mit internetverbundenen Terminals** arbeitet. Bluetooth-Terminals werden derzeit nicht unterstützt.
:::

### Unterstützte Terminals (Internet-Verbunden) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WLAN-/Ethernet-verbundenes Terminal

### Nicht unterstützte Terminals (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Nur Bluetooth
- **SumUp 3G** ❌ - Nur Bluetooth

:::note Zukünftige Unterstützung
Die Unterstützung für Bluetooth-Terminals wird in eine zukünftige Version der iOS- und Android-Anwendungen in **Q1 2026** integriert. Dies wird die Unterstützung für alle von SumUp zertifizierten Terminals, einschließlich der Modelle Air und 3G, ermöglichen.
:::

## Fehlersuche {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Leser koppelt nicht">

  - Stellen Sie sicher, dass Sie ein unterstütztes internetverbundenes Terminal verwenden (nur Solo)
  - Überprüfen Sie, ob Ihr SumUp Solo-Gerät mit WLAN verbunden und online ist
  - Stellen Sie sicher, dass das Gerät sich im Kopplungsmodus befindet und der Kopplungscode korrekt eingegeben wurde
  - Bestätigen Sie, dass Ihr SumUp-Konto über aktivierte API-Zugriffsrechte verfügt
  - Stellen Sie eine stabile Internetverbindung während des Kopplungsprozesses sicher

  </AccordionItem>

  <AccordionItem question="Zahlung wird nicht verarbeitet">

  - Überprüfen Sie, ob der Leser korrekt gekoppelt ist
  - Stellen Sie sicher, dass Ihr SumUp-Konto aktiv und in gutem Zustand ist
  - Vergewissern Sie sich, dass das Gerät über ausreichend Akku und Konnektivität verfügt
  - Bestätigen Sie, dass der API-Schlüssel korrekt und aktiv ist

  </AccordionItem>

  <AccordionItem question="Bestellung wird nicht abgeschlossen">

  - Überprüfen Sie die Stabilität der Internetverbindung
  - Überprüfen Sie die Webhook-Einstellungen in Ihrem SumUp-Konto
  - Stellen Sie sicher, dass die Zahlung am SumUp-Terminal erfolgreich war
  - Überprüfen Sie die WordPress-Fehlerprotokolle auf Probleme bei der API-Kommunikation

  </AccordionItem>
</Accordion>

### API-Rate-Limits

Das Gateway ist optimiert, um API-Aufrufe zu minimieren und eine Ratenlimitierung zu vermeiden:
- Der Zahlungsstatus wird effizient überprüft
- Unnötige API-Anfragen werden minimiert
- Automatische Wiederholungslogik behandelt temporäre Fehler
