# SumUp Terminal Gateway

Das SumUp Terminal Gateway ermöglicht es Ihnen, Kartenzahlungen mit SumUp-Kartenlesern direkt in WooCommerce POS zu akzeptieren. Dieses Gateway bietet eine nahtlose Integration mit dem Zahlungsverarbeitungssystem von SumUp für Transaktionen vor Ort.

## Funktionen

- **Hardware-Integration**: SumUp-Kartenleser über Internetverbindung mit Ihrem POS-System verbinden
- **Echtzeitverarbeitung**: Sofortige Zahlungsabwicklung und Auftragsabschluss
- **Automatische Erkennung**: Aufträge werden automatisch abgeschlossen, wenn eine Zahlung erkannt wird
- **Sichere Transaktionen**: PCI-konforme Zahlungsverarbeitung über SumUp
- **Einfache Kopplung**: Einfacher Gerätekopplungsprozess mit Kopplungscodes
- **Web-basiertes SDK**: Verwendet SumUp's JavaScript SDK für Webanwendungskompatibilität

## Installation

### Schritt 1: Herunterladen und Installieren

1. Besuchen Sie die [SumUp Terminal für WooCommerce Releases-Seite](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Laden Sie die neueste **sumup-terminal-for-woocommerce.zip**-Datei herunter
3. Gehen Sie in Ihrer WordPress-Administrationsoberfläche zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

### Schritt 2: SumUp-Einstellungen konfigurieren

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Finden Sie **SumUp Terminal** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **SumUp Terminal**, um die Einstellungen zu öffnen
4. Geben Sie Ihren **SumUp API-Schlüssel** ein (verfügbar in Ihrem SumUp-Händler-Dashboard)
5. Speichern Sie die Einstellungen

:::note
Sie müssen das SumUp Terminal Gateway in den WooCommerce-Einstellungen nicht aktivieren. Es wird speziell für den POS in einem späteren Schritt aktiviert.
:::

### Schritt 3: SumUp Terminal koppeln

1. Suchen Sie auf der gleichen Einstellungsseite den Abschnitt **Reader koppeln**
2. Navigieren Sie auf Ihrem SumUp-Gerät zum Kopplungsbildschirm, um den Kopplungscode anzuzeigen
3. Geben Sie den auf Ihrem SumUp-Gerät angezeigten Kopplungscode ein
4. Klicken Sie auf **"Reader koppeln"**, um die Verbindung herzustellen
5. Warten Sie auf die Bestätigung, dass der Reader erfolgreich gekoppelt wurde

:::warning Wichtig
Der Reader muss erfolgreich gekoppelt sein, bevor Sie Zahlungen verarbeiten können. Stellen Sie sicher, dass der Kopplungsprozess abgeschlossen ist, bevor Sie fortfahren.
:::

### Schritt 4: In WooCommerce POS aktivieren

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Finden Sie das **SumUp Terminal** Gateway in der Liste
3. Aktivieren Sie das Gateway für die Verwendung im POS
4. Speichern Sie Ihre Einstellungen

## Nutzung

### Zahlungen verarbeiten

1. **Artikel hinzufügen**: Fügen Sie Produkte zum Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "SumUp Terminal" als Zahlungsmethode
3. **Zahlung starten**: Klicken Sie, um eine neue Zahlung auf Ihrem SumUp-Gerät zu initiieren
4. **Kundenzahlung**: Kunde schließt Zahlung auf dem SumUp-Terminal ab
5. **Automatischer Abschluss**: Der Auftrag wird automatisch abgeschlossen, wenn die Zahlung erkannt wird

### Zahlungssteuerung

Bei Verwendung des SumUp Terminal Gateways haben Sie folgende Optionen:

- **Neue Zahlung starten**: Zahlungsanfrage auf dem verbundenen Terminal initiieren
- **Zahlung abbrechen**: Eine laufende Zahlung abbrechen
- **Zahlungsstatus**: Aktuellen Status des Zahlungsprozesses überwachen

### Auftragsmanagement

- **Automatische Updates**: Aufträge werden bei erfolgreicher Zahlung automatisch als abgeschlossen markiert
- **Zahlungsverfolgung**: Alle Zahlungsdetails werden in den Auftragsnotizen aufgezeichnet
- **Belegegenerierung**: Standardmäßige POS-Belege werden nach erfolgreichen Zahlungen generiert

## Anforderungen

- **SumUp-Konto**: Aktives SumUp-Händlerkonto
- **API-Zugang**: SumUp API-Schlüssel aus Ihrem Händler-Dashboard
- **Kompatible Hardware**: SumUp-Kartenlesegerät
- **WooCommerce POS**: Freie oder Pro-Version installiert und aktiviert
- **Stabile Verbindung**: Zuverlässige Internetverbindung für API-Kommunikation

## Hardware-Kompatibilität

:::important Konnektivitätsanforderungen
Diese Implementierung verwendet SumUp's JavaScript SDK, was bedeutet, dass sie über Webanwendungen funktioniert, aber nur **internetverbundene Terminals** erfordert. Bluetooth-Terminals werden derzeit nicht unterstützt.
:::

### Unterstützte Terminals (Internetverbunden)

- **SumUp Solo** ✅ - WiFi/Ethernet-verbundenes Terminal

### Nicht unterstützte Terminals (Bluetooth)

- **SumUp Air** ❌ - Nur Bluetooth
- **SumUp 3G** ❌ - Nur Bluetooth

:::note Zukünftige Unterstützung
Bluetooth-Terminal-Unterstützung wird in den iOS- und Android-Anwendungen im **Q1 2026** hinzugefügt. Dies wird die Unterstützung für alle von SumUp zertifizierten Terminals, einschließlich Air und 3G-Modelle, ermöglichen.
:::

## Fehlerbehebung

### Häufige Probleme

**Reader lässt sich nicht koppeln**
- Stellen Sie sicher, dass Sie ein unterstütztes internetverbundenes Terminal verwenden (nur Solo)
- Überprüfen Sie, ob Ihr SumUp Solo-Gerät mit WiFi verbunden und online ist
- Prüfen Sie, ob sich das Gerät im Kopplungsmodus befindet und der Kopplungscode korrekt eingegeben wurde
- Bestätigen Sie, dass Ihr SumUp-Konto API-Zugang aktiviert hat
- Sorgen Sie für eine stabile Internetverbindung während der Kopplung

**Zahlung wird nicht verarbeitet**
- Überprüfen Sie, ob der Reader ordnungsgemäß gekoppelt ist
- Prüfen Sie, ob Ihr SumUp-Konto aktiv und in gutem Zustand ist
- Stellen Sie sicher, dass das Gerät ausreichend Akku und Konnektivität hat
- Bestätigen Sie, dass der API-Schlüssel korrekt und aktiv ist

**Auftrag wird nicht abgeschlossen**
- Überprüfen Sie die Stabilität der Internetverbindung
- Verifizieren Sie die Webhook-Einstellungen in Ihrem SumUp-Konto
- Stellen Sie sicher, dass die Zahlung auf dem SumUp-Terminal erfolgreich war
- Überprüfen Sie die WordPress-Fehlerprotokolle auf API-Kommunikationsprobleme

### API-Ratenlimits

Das Gateway ist optimiert, um API-Aufrufe zu reduzieren und Ratenlimits zu vermeiden:
- Zahlungsstatus wird effizient überprüft
- Unnötige API-Anfragen werden minimiert
- Automatische Wiederholungslogik behandelt vorübergehende Ausfälle
