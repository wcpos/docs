# SumUp Terminal Gateway

Das SumUp Terminal Gateway ermöglicht es Ihnen, Kartenzahlungen über SumUp-Kartenlesegeräte direkt in WooCommerce POS zu akzeptieren. Dieses Gateway bietet eine nahtlose Integration mit dem Zahlungssystem von SumUp für persönliche Transaktionen.

## Funktionen

- **Hardware-Integration**: Verbinden Sie SumUp-Kartenlesegeräte über eine Internetverbindung mit Ihrem POS-System
- **Echtzeitverarbeitung**: Sofortige Zahlungsabwicklung und Auftragsabschluss
- **Automatische Erkennung**: Aufträge werden automatisch abgeschlossen, wenn die Zahlung erkannt wird
- **Sichere Transaktionen**: PCI-konforme Zahlungsabwicklung über SumUp
- **Einfache Kopplung**: Einfacher Geräte-Kopplungsprozess mit Kopplungscodes
- **Web-basiertes SDK**: Verwendet das JavaScript-SDK von SumUp für die Kompatibilität mit Webanwendungen

## Installation

### Schritt 1: Herunterladen und Installieren

1. Besuchen Sie die [SumUp Terminal für WooCommerce Releases-Seite](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Laden Sie die neueste **sumup-terminal-for-woocommerce.zip** Datei herunter
3. Gehen Sie in Ihrer WordPress-Administration zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

### Schritt 2: SumUp-Einstellungen konfigurieren

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Finden Sie **SumUp Terminal** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **SumUp Terminal**, um die Einstellungen zuzugreifen
4. Geben Sie Ihren **SumUp API-Schlüssel** ein (verfügbar in Ihrem SumUp-Händler-Dashboard)
5. Speichern Sie die Einstellungen

:::note
Sie müssen das SumUp Terminal Gateway in den WooCommerce-Einstellungen nicht aktivieren. Es wird in einem späteren Schritt speziell für das POS aktiviert.
:::

### Schritt 3: Ihr SumUp Terminal koppeln

1. Auf derselben Einstellungsseite finden Sie den Abschnitt **Reader koppeln**
2. Navigieren Sie auf Ihrem SumUp-Gerät zum Kopplungsbildschirm, um den Kopplungscode anzuzeigen
3. Geben Sie den auf Ihrem SumUp-Gerät angezeigten Kopplungscode ein
4. Klicken Sie auf **"Reader koppeln"**, um die Verbindung herzustellen
5. Warten Sie auf die Bestätigung, dass der Reader erfolgreich gekoppelt wurde

:::warning Wichtig
Der Reader muss erfolgreich gekoppelt werden, bevor Sie Zahlungen verarbeiten können. Stellen Sie sicher, dass der Kopplungsprozess abgeschlossen ist, bevor Sie fortfahren.
:::

### Schritt 4: Aktivieren in WooCommerce POS

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Finden Sie das **SumUp Terminal** Gateway in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

## Verwendung

### Zahlungen verarbeiten

1. **Artikel hinzufügen**: Fügen Sie Produkte zu Ihrem Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "SumUp Terminal" als Zahlungsmethode
3. **Zahlung starten**: Klicken Sie, um eine neue Zahlung auf Ihrem SumUp-Gerät zu initiieren
4. **Zahlung durch den Kunden**: Der Kunde schließt die Zahlung am SumUp Terminal ab
5. **Automatischer Abschluss**: Die Bestellung wird automatisch abgeschlossen, wenn die Zahlung erkannt wird

### Zahlungssteuerung

Bei Verwendung des SumUp Terminal Gateways haben Sie folgende Optionen:

- **Neue Zahlung starten**: Initiieren Sie eine Zahlungsanfrage auf dem verbundenen Terminal
- **Zahlung stornieren**: Stornieren Sie eine aktuell in Bearbeitung befindliche Zahlung
- **Zahlungsstatus**: Überwachen Sie den aktuellen Status des Zahlungsprozesses

### Auftragsverwaltung

- **Automatische Updates**: Aufträge werden nach erfolgreicher Zahlung automatisch als abgeschlossen markiert
- **Zahlungsverfolgung**: Alle Zahlungsdetails werden in den Bestellnotizen festgehalten
- **Belegdruck**: Standard-POS-Belege werden nach erfolgreichen Zahlungen erstellt

## Anforderungen

- **SumUp-Konto**: Aktives SumUp-Händlerkonto
- **API-Zugang**: SumUp-API-Schlüssel aus Ihrem Händler-Dashboard
- **Kompatible Hardware**: SumUp-Kartenlesegerät
- **WooCommerce POS**: Kostenlose oder Pro-Version installiert und aktiviert
- **Stabile Verbindung**: Zuverlässige Internetverbindung für die API-Kommunikation

## Hardware-Kompatibilität

:::important Konnektivitätsanforderungen
Diese Implementierung verwendet das JavaScript-SDK von SumUp, was bedeutet, dass es über Webanwendungen funktioniert, aber nur **Internet-verbundene Terminals** erfordert. Bluetooth-Terminals werden derzeit nicht unterstützt.
:::

### Unterstützte Terminals (Internet-Verbunden)

- **SumUp Solo** ✅ - WLAN-/Ethernet-verbundenes Terminal

### Nicht unterstützte Terminals (Bluetooth)

- **SumUp Air** ❌ - Nur Bluetooth
- **SumUp 3G** ❌ - Nur Bluetooth

:::note Zukünftige Unterstützung
Die Unterstützung für Bluetooth-Terminals wird in den iOS- und Android-Anwendungen in **Q1 2026** hinzugefügt. Dadurch wird die Unterstützung für alle von SumUp zertifizierten Terminals, einschließlich der Modelle Air und 3G, ermöglicht.
:::

## Fehlerbehebung

### Häufige Probleme

**Reader lässt sich nicht koppeln**
- Stellen Sie sicher, dass Sie ein unterstütztes internet-verbundenes Terminal (nur Solo) verwenden
- Vergewissern Sie sich, dass Ihr SumUp Solo-Gerät mit WLAN verbunden und online ist
- Überprüfen Sie, ob das Gerät im Kopplungsmodus ist und der Kopplungscode korrekt eingegeben wurde
- Bestätigen Sie, dass Ihr SumUp-Konto den aktiven API-Zugang hat
- Stellen Sie sicher, dass die Internetverbindung während der Kopplung stabil ist

**Zahlung wird nicht verarbeitet**
- Vergewissern Sie sich, dass der Reader ordnungsgemäß gekoppelt ist
- Überprüfen Sie, ob Ihr SumUp-Konto aktiv und in gutem Zustand ist
- Stellen Sie sicher, dass das Gerät über ausreichend Akku und Verbindung verfügt
- Bestätigen Sie, dass der API-Schlüssel korrekt und aktiv ist

**Bestellung wird nicht abgeschlossen**
- Überprüfen Sie die Stabilität der Internetverbindung
- Verifizieren Sie die Webhook-Einstellungen in Ihrem SumUp-Konto
- Stellen Sie sicher, dass die Zahlung am SumUp Terminal erfolgreich war
- Überprüfen Sie die Fehlerprotokolle von WordPress auf API-Kommunikationsprobleme

### API-Ratenlimits

Das Gateway ist optimiert, um API-Anfragen zu reduzieren und eine Überlastung zu vermeiden:
- Der Zahlungsstatus wird effizient überprüft
- Unnötige API-Anfragen werden minimiert
- Automatische Wiederholungslogik behandelt temporäre Fehler
