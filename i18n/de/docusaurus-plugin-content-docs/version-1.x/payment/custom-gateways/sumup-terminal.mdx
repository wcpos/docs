# SumUp Terminal Gateway

Das SumUp Terminal Gateway ermöglicht es Ihnen, Kartenzahlungen mit SumUp Kartenlesern direkt innerhalb von WooCommerce POS zu akzeptieren. Dieses Gateway bietet eine nahtlose Integration mit dem Zahlungssystem von SumUp für persönliche Transaktionen.

## Funktionen {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Hardware-Integration">
    Verbinden Sie SumUp-Kartenleser über eine Internetverbindung mit Ihrem POS-System
  </FeatureCard>
  <FeatureCard icon="bolt" title="Echtzeitverarbeitung">
    Sofortige Zahlungsabwicklung und Auftragsabschluss
  </FeatureCard>
  <FeatureCard icon="shield" title="Sichere Transaktionen">

  </FeatureCard>

    Einfacher Koppelungsprozess mit Kopplungscodes

</FeatureGrid>

## Installation

<Steps>
  <StepCard number="1" title="Download und Installation">

1. Besuchen Sie die [SumUp Terminal für WooCommerce Releases-Seite](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Laden Sie die neueste **sumup-terminal-for-woocommerce.zip** Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="SumUp-Einstellungen konfigurieren">

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Suchen Sie **SumUp Terminal** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **SumUp Terminal**, um auf die Einstellungen zuzugreifen
4. Geben Sie Ihren **SumUp API-Schlüssel** ein (verfügbar in Ihrem SumUp Händler-Dashboard)
5. Speichern Sie die Einstellungen

:::note
Sie müssen das SumUp Terminal Gateway nicht in den WooCommerce-Einstellungen aktivieren. Es wird zu einem späteren Zeitpunkt speziell für den POS aktiviert.
:::

  </StepCard>
  <StepCard number="3" title="Koppeln Sie Ihr SumUp Terminal">

1. Suchen Sie auf derselben Einstellungsseite den Abschnitt **Reader koppeln**
2. Navigieren Sie auf Ihrem SumUp-Gerät zum Koppelungsbildschirm, um den Koppelungscode anzuzeigen
3. Geben Sie den Koppelungscode ein, der auf Ihrem SumUp-Gerät angezeigt wird
4. Klicken Sie auf **"Reader koppeln"**, um die Verbindung herzustellen
5. Warten Sie auf die Bestätigung, dass der Reader erfolgreich gekoppelt wurde

:::warning Wichtig
Der Reader muss erfolgreich gekoppelt sein, bevor Sie Zahlungen verarbeiten können. Stellen Sie sicher, dass der Koppelungsprozess abgeschlossen ist, bevor Sie fortfahren.
:::

  </StepCard>
  <StepCard number="4" title="Aktivieren in WooCommerce POS" isLast>

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Kasse`
2. Suchen Sie das **SumUp Terminal** Gateway in der Liste
3. Aktivieren Sie das Gateway zur Nutzung im POS
4. Speichern Sie Ihre Einstellungen

  </StepCard>
</Steps>

## Nutzung {#usage}

### Zahlungsvorgänge {#processing-payments}

1. **Artikel hinzufügen**: Fügen Sie Produkte zu Ihrem Einkaufswagen im POS hinzu
2. **Gateway auswählen**: Wählen Sie "SumUp Terminal" als Zahlungsmethode
3. **Zahlung starten**: Klicken Sie, um eine neue Zahlung auf Ihrem SumUp-Gerät einzuleiten
4. **Kundenbezahlung**: Der Kunde schließt die Zahlung am SumUp-Terminal ab
5. **Automatischer Abschluss**: Die Bestellung wird automatisch abgeschlossen, wenn die Zahlung erkannt wird

### Zahlungssteuerung {#payment-controls}

Bei Verwendung des SumUp Terminal Gateways haben Sie folgende Möglichkeiten:

- **Neue Zahlung starten**: Anforderung einer Zahlung auf dem verbundenen Terminal einleiten
- **Zahlung abbrechen**: Eine derzeit verarbeitete Zahlung stornieren
- **Zahlungsstatus**: Überwachen Sie den aktuellen Status des Zahlungsprozesses

### Auftragsverwaltung {#order-management}

- **Automatische Updates**: Bestellungen werden nach erfolgreicher Zahlung automatisch als abgeschlossen markiert
- **Zahlungsverfolgung**: Alle Zahlungsdetails werden in den Bestellnotizen aufgezeichnet
- **Beleggenerierung**: Standard-POS-Belege werden nach erfolgreichen Zahlungen generiert

## Hardware-Kompatibilität {#hardware-compatibility}

:::important Anschlussanforderungen
Diese Implementierung verwendet das JavaScript SDK von SumUp, das bedeutet, dass es über Webanwendungen funktioniert, aber **nur mit internetverbundenen Terminals** funktioniert. Bluetooth-Terminals werden derzeit nicht unterstützt.
:::

### Unterstützte Terminals (Internetverbunden) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet verbundenes Terminal

### Nicht unterstützte Terminals (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Nur Bluetooth
- **SumUp 3G** ❌ - Nur Bluetooth

:::note Zukünftige Unterstützung
Die Unterstützung von Bluetooth-Terminals wird in den iOS- und Android-Anwendungen in **Q1 2026** hinzugefügt. Dies wird die Unterstützung aller von SumUp zertifizierten Terminals, einschließlich der Modelle Air und 3G, ermöglichen.
:::

## Fehlerbehebung {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Reader lässt sich nicht koppeln">

  - Stellen Sie sicher, dass Sie ein unterstütztes, internetverbundenes Terminal (nur Solo) verwenden
  - Überprüfen Sie, ob Ihr SumUp Solo-Gerät mit WiFi verbunden und online ist
  - Überprüfen Sie, ob das Gerät im Koppelungsmodus ist und der Koppelungscode korrekt eingegeben wurde
  - Bestätigen Sie, dass Ihr SumUp-Konto über aktivierten API-Zugriff verfügt
  - Stellen Sie sicher, dass während der Koppelung eine stabile Internetverbindung besteht

  </AccordionItem>

  <AccordionItem question="Zahlung wird nicht verarbeitet">

  - Überprüfen Sie, ob der Reader korrekt gekoppelt ist
  - Stellen Sie sicher, dass Ihr SumUp-Konto aktiv und in gutem Zustand ist
  - Überprüfen Sie, ob das Gerät über ausreichende Batterieleistung und Konnektivität verfügt
  - Bestätigen Sie, dass der API-Schlüssel korrekt und aktiv ist

  </AccordionItem>

  <AccordionItem question="Bestellung wird nicht abgeschlossen">

  - Überprüfen Sie die Stabilität der Internetverbindung
  - Überprüfen Sie die Webhook-Einstellungen in Ihrem SumUp-Konto
  - Stellen Sie sicher, dass die Zahlung am SumUp-Terminal erfolgreich war
  - Überprüfen Sie die WordPress-Fehlerprotokolle auf API-Kommunikationsprobleme

  </AccordionItem>
</Accordion>

### API-Ratenlimits {#api-rate-limits}

Das Gateway ist optimiert, um API-Aufrufe zu reduzieren und Ratenbeschränkungen zu vermeiden:
- Der Zahlungsstatus wird effizient überprüft
- Unnötige API-Anfragen werden minimiert
- Die automatische Wiederholungslogik verarbeitet temporäre Fehler
