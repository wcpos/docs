# Benutzerdefinierte Zahlungsmethoden

WCPOS unterstützt mehrere benutzerdefinierte Zahlungsmethoden, die die Funktionalität Ihres Point of Sale-Systems erweitern. Diese Zahlungsmethoden sind speziell für POS-Umgebungen konzipiert und bieten eine nahtlose Integration mit verschiedenen Zahlungsarten und Services.

## Verfügbare benutzerdefinierte Zahlungsmethoden

### Hardware-Terminal-Zahlungsmethoden

- **[Stripe Terminal](./stripe-terminal)** - Zahlungen mit Stripe Terminal-Hardwarelesegeräten akzeptieren
- **[SumUp Terminal](./sumup-terminal)** - Zahlungen über SumUp-Kartenleser abwickeln

### Digitale Zahlungsmethoden

- **[E-Mail-Rechnung](./email-invoice)** - Zahlungsrechnungen per E-Mail an Kunden senden
- **[Web Checkout](./web-checkout)** - Kunden umleiten, um Zahlungen online abzuschließen

## Erstellen Ihrer eigenen Zahlungsmethode

Wenn Sie eine benutzerdefinierte Zahlungsmethode für Ihre spezifischen Anforderungen benötigen, können Sie unsere **[Gateway-Vorlage](./gateway-template)** verwenden, um Ihre eigene WooCommerce POS-Zahlungsmethode zu erstellen.

## Installationsübersicht

Alle benutzerdefinierten Zahlungsmethoden folgen einem ähnlichen Installationsprozess:

1. **Laden Sie** die neueste Version aus dem entsprechenden GitHub-Repository herunter
2. **Installieren Sie** das Plugin über `WP Admin > Plugins > Neu hinzufügen > Plugin hochladen`
3. **Konfigurieren Sie** die Zahlungsmethoden-Einstellungen in `WP Admin > WooCommerce > Einstellungen > Zahlungen`
4. **Aktivieren Sie** die Zahlungsmethode in `WP Admin > WooCommerce POS > Einstellungen > Checkout`

## Anforderungen

- WooCommerce POS-Plugin (kostenlose oder Pro-Version)
- WordPress mit installiertem WooCommerce
- Geeignete API-Schlüssel oder Zugangsdaten für den jeweiligen Zahlungsdienst

## Unterstützung

Bei Problemen mit benutzerdefinierten Zahlungsmethoden besuchen Sie bitte das entsprechende GitHub-Repository und erstellen Sie ein Problem mit detaillierten Informationen zu Ihrem Anliegen.
