# Vipps MobilePay Gateway

Das Vipps MobilePay-Gateway ermöglicht es Ihnen, telefonbasierte Zahlungen direkt in WooCommerce POS zu akzeptieren. Kunden bezahlen, indem sie einen QR-Code auf dem Bildschirm scannen oder eine Push-Benachrichtigung auf ihrem Telefon bestätigen – kein Kartenlesegerät erforderlich.

## Funktionen {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="QR-Code-Zahlungen">
    Zeigen Sie einen QR-Code an der Kasse an, den die Kunden scannen und mit ihrem Telefon bezahlen können
  </FeatureCard>
  <FeatureCard icon="bolt" title="Push-Benachrichtigungen">
    Senden Sie eine Zahlungsanforderung direkt an das Telefon des Kunden, indem Sie seine Nummer eingeben
  </FeatureCard>
  <FeatureCard icon="globe" title="Nordische Abdeckung">
    Funktioniert mit Vipps in Norwegen, MobilePay in Dänemark und Finnland
  </FeatureCard>
  <FeatureCard icon="credit-card" title="Automatische Erfassung">
    Erfasst Gelder sofort nach der Autorisierung oder reserviert für die manuelle Erfassung
  </FeatureCard>
</FeatureGrid>

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [WCPOS Vipps MobilePay Veröffentlichungsseite](https://github.com/wcpos/wcpos-vipps/releases)
2. Laden Sie die neueste Datei **wcpos-vipps.zip** herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="Vipps-Anmeldeinformationen konfigurieren">

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Finden Sie **WCPOS Vipps MobilePay** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **WCPOS Vipps MobilePay**, um auf die Einstellungen zuzugreifen
4. Geben Sie Ihre Anmeldeinformationen aus dem [Vipps-Portal](https://portal.vipps.no/) ein:
   - **Händler-Seriennummer (MSN)**
   - **Client-ID**
   - **Client Secret**
   - **Abonnement-Schlüssel**
5. Speichern Sie die Einstellungen

:::note
Sie müssen das Gateway hier nicht für die POS-Nutzung aktivieren – es wird spezifisch für das POS im nächsten Schritt aktiviert. Wenn Sie es in den WooCommerce-Einstellungen aktivieren, wäre es auch an der Kasse Ihres Online-Shops verfügbar, was für Tests nützlich sein kann.
:::

:::tip Verwenden Sie bereits das Vipps-Plugin?
Wenn Sie das offizielle [Checkout mit Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/) Plugin installiert haben, werden Ihre Anmeldeinformationen automatisch importiert, wenn Sie dieses Plugin aktivieren. Sie können die manuelle Eingabe der Anmeldeinformationen überspringen.
:::

  </StepCard>
  <StepCard number="3" title="Aktivieren in WooCommerce POS" isLast>

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Kasse`
2. Finden Sie das **WCPOS Vipps MobilePay**-Gateway in der Liste
3. Aktivieren Sie das Gateway zur Nutzung im POS
4. Speichern Sie Ihre Einstellungen

  </StepCard>
</Steps>

## Nutzung {#usage}

### Zahlungen verarbeiten – QR-Code {#processing-payments-qr-code}

1. **Artikel hinzufügen**: Fügen Sie Produkte Ihrem Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "Vipps MobilePay" als Zahlungsmethode
3. **QR-Code generieren**: Klicken Sie auf die Schaltfläche "QR-Code generieren"
4. **Kunde scannt**: Der Kunde scannt den QR-Code mit seiner Vipps oder MobilePay-App
5. **Kunde bestätigt**: Der Kunde bestätigt die Zahlung in seiner App
6. **Automatische Abschluss**: Die Bestellung wird automatisch abgeschlossen, sobald die Zahlung autorisiert ist

### Zahlungen verarbeiten – Push-Benachrichtigung {#processing-payments-push-notification}

1. **Artikel hinzufügen**: Fügen Sie Produkte Ihrem Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "Vipps MobilePay" als Zahlungsmethode
3. **Telefonnummer eingeben**: Geben Sie die Telefonnummer des Kunden ein
4. **An das Telefon senden**: Klicken Sie auf die Schaltfläche "An das Telefon senden"
5. **Kunde bestätigt**: Der Kunde erhält eine Push-Benachrichtigung und bestätigt die Zahlung in seiner App
6. **Automatische Abschluss**: Die Bestellung wird automatisch abgeschlossen, sobald die Zahlung autorisiert ist

### Zahlung stornieren {#cancelling-a-payment}

Während Sie auf die Bestätigung des Kunden warten, können Sie auf die Schaltfläche **Zahlung stornieren** klicken, um die Transaktion abzubrechen. Dadurch wird die ausstehende Zahlung auf der Vipps-Seite storniert und die Kassenoberfläche zurückgesetzt.

### Rückerstattungen {#refunds}

Rückerstattungen werden über den standardmäßigen WooCommerce-Rückerstattungsprozess abgewickelt. Öffnen Sie die Bestellung, klicken Sie auf **Rückerstattung**, geben Sie den Betrag ein und die Rückerstattung wird automatisch über die Vipps-API verarbeitet.

## Unterstützte Märkte {#supported-markets}

Vipps MobilePay ist in der nordischen Region unter zwei Marken tätig:

| Region | Marke | Währungen |
|--------|-------|------------|
| Norwegen | Vipps | NOK |
| Dänemark, Finnland | MobilePay | DKK, EUR |

Ihr Händlerkonto bestimmt, welche Märkte und Währungen verfügbar sind. Kunden in jedem unterstützten Markt können mit ihrer lokalen Vipps oder MobilePay-App bezahlen.

## Fehlerbehebung {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Zahlung wird nicht abgeschlossen">

  - Die Kasse prüft bis zu 5 Minuten – wenn der Kunde innerhalb dieses Zeitrahmens nicht bestätigt, läuft die Zahlung ab
  - Überprüfen Sie, dass Sie Testmodus und Produktionsanmeldeinformationen nicht verwechseln
  - Stellen Sie sicher, dass Ihre Internetverbindung sowohl auf dem POS-Gerät als auch auf dem Telefon des Kunden stabil ist

  </AccordionItem>

  <AccordionItem question="Ungültige Anmeldefehler">

  - Überprüfen Sie alle vier Anmeldefelder (MSN, Client-ID, Client Secret, Abonnement-Schlüssel) noch einmal
  - Stellen Sie sicher, dass Sie Testanmeldeinformationen verwenden, wenn der Testmodus aktiviert ist, und Produktionsanmeldeinformationen, wenn er deaktiviert ist
  - Überprüfen Sie Ihre Anmeldeinformationen im [Vipps-Portal](https://portal.vipps.no/)

  </AccordionItem>

  <AccordionItem question="QR-Code wird nicht generiert">

  - Bestätigen Sie, dass Ihre Website über HTTPS ausgeführt wird – die Vipps-API lehnt Anfragen von HTTP-Websites ab
  - Prüfen Sie die WooCommerce-Protokolle (`WooCommerce > Status > Protokolle`) auf API-Fehlerdetails
  - Vergewissern Sie sich, dass Ihr Händlerkonto aktiv ist und die ePayment-API aktiviert hat

  </AccordionItem>
</Accordion>
