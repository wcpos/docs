# E-Mail-Rechnung Gateway {#email-invoice-gateway}

Das E-Mail-Rechnung-Gateway ermöglicht es Ihnen, Zahlungsrechnungen direkt aus WCPOS per E-Mail an Kunden zu senden. Dies ist perfekt für Situationen, in denen Kunden lieber später bezahlen möchten oder die Zahlung online mit den Zahlungsmethoden Ihres Webshops abschließen müssen.

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [E-Mail-Rechnung Gateway Release-Seite](https://github.com/wcpos/email-invoice-gateway/releases)
2. Laden Sie die neueste **woocommerce-pos-email-invoice-gateway.zip** Datei herunter
3. Gehen Sie im WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="Aktivieren in WCPOS" isLast>

1. Gehen Sie zu `WP Admin > POS > Einstellungen > Checkout`
2. Suchen Sie das **WCPOS Rechnungszahlung Gateway** in der Liste
3. Aktivieren Sie das Gateway zur Nutzung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Gateway erfordert keine zusätzlichen API-Schlüssel oder Konfigurationen von externen Diensten. Es nutzt Ihr vorhandenes WordPress-E-Mail-System und WooCommerce-Zahlungsmethoden.
:::

  </StepCard>
</Steps>

## Nutzung {#usage}

### Rechnungen senden {#sending-invoices}

1. **Artikel hinzufügen**: Fügen Sie Produkte in den POS-Warenkorb hinzu
2. **Gateway auswählen**: Wählen Sie "E-Mail-Rechnung" als Zahlungsmethode
3. **E-Mail-Adresse**: 
   - Für bestehende Kunden: E-Mail wird automatisch aus den Kundendaten ausgefüllt
   - Für Gastbestellungen: Geben Sie die E-Mail-Adresse des Kunden manuell ein
4. **Optional**: Aktivieren Sie "An Rechnungsadresse speichern", um die Rechnungsinformationen des Kunden zu aktualisieren
5. **Rechnung senden**: Schließen Sie den Prozess ab, um die Rechnungs-E-Mail zu senden

### Kundenerfahrung {#customer-experience}

Wenn Sie eine Rechnung senden, erhält der Kunde eine E-Mail mit:

- **Bestelldetails**: Vollständige Aufschlüsselung von Artikeln, Mengen und Preisen
- **Bestellsumme**: Endbetrag einschließlich Steuern und Gebühren
- **Zahlungslink**: Direkter Link zum Abschluss der Zahlung online
- **Bestellinformationen**: Bestellnummer und Referenzdetails

### Zahlungsabschluss {#payment-completion}

Kunden können die Zahlung auf zwei Arten abschließen:

1. **Online-Zahlung**: Klicken Sie auf den Link "Für diese Bestellung bezahlen" in der E-Mail
2. **Zahlung im Geschäft**: Zurückkehren, um die Zahlung mit anderen POS-Zahlungsmethoden abzuschließen

Der Link "Für diese Bestellung bezahlen" führt die Kunden zu einer speziellen Zahlungsseite auf Ihrer Website, wo sie alle aktivierten Web-Zahlungsgateways (Stripe, PayPal usw.) verwenden können.

## E-Mail-Verwaltung {#email-management}

### Bestehende Kunden {#existing-customers}

- E-Mail-Adressen werden automatisch aus den Kundenprofilen abgerufen
- Rechnungsinformationen des Kunden bleiben erhalten
- Bestellhistorie wird im Kundenkonto gespeichert

### Gastkunden {#guest-customers}

- E-Mail-Adressen für einmalige Kunden manuell eingeben
- Möglichkeit, die E-Mail für zukünftige Referenzen an die Rechnungsadresse zu speichern
- Gastbestellungen werden ordnungsgemäß in WooCommerce verfolgt

### E-Mail-Vorlagen {#email-templates}

Das Gateway verwendet die Standard-E-Mail-Vorlagen von WooCommerce:

- **Anpassbar**: Ändern Sie das Erscheinungsbild der E-Mail über die WooCommerce-E-Mail-Einstellungen
- **Markenbildung**: Ihr Logo und Branding einschließen
- **Professionell**: Sauberes, professionelles Rechnungsformat
- **Mobilfreundlich**: Responsives Design für alle Geräte

## Anwendungsfälle {#use-cases}

### Ideal für {#perfect-for}

- **B2B-Verkäufe**: Geschäftskunden, die Zahlungen über ihre Buchhaltungssysteme abwickeln müssen
- **Große Bestellungen**: Hochwertige Transaktionen, die Genehmigungen oder Bearbeitungszeit erfordern
- **Fernkunden**: Kunden, die die Zahlung nach Verlassen des Geschäfts abschließen müssen
- **Kunden mit Konto**: Regelmäßige Kunden mit festgelegten Zahlungsbedingungen
- **Von Angebot zu Rechnung**: Angebote oder Schätzungen in zahlbare Rechnungen umwandeln

## Fehlersuche {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="E-Mails werden nicht gesendet">

  - Überprüfen Sie die WordPress-E-Mail-Konfiguration
  - Überprüfen Sie die SMTP-Einstellungen, wenn Sie einen benutzerdefinierten E-Mail-Dienst verwenden
  - Testen Sie mit WordPress-E-Mail-Test-Plugins
  - Überprüfen Sie die Spam-Ordner nach Test-E-Mails

  </AccordionItem>

  <AccordionItem question="Zahlungslinks funktionieren nicht">

  - Stellen Sie sicher, dass WooCommerce richtig konfiguriert ist
  - Überprüfen Sie, ob mindestens ein Zahlungsgateway für den Web-Checkout aktiviert ist
  - Stellen Sie sicher, dass das SSL-Zertifikat korrekt installiert ist
  - Bestätigen Sie, dass der Bestellstatus eine Zahlung zulässt

  </AccordionItem>

  <AccordionItem question="Kunde kann Zahlung nicht abschließen">

  - Überprüfen Sie, ob die Zahlungs-Gateways aktiv und konfiguriert sind
  - Überprüfen Sie, ob die Bestellung nicht abgelaufen oder storniert wurde
  - Stellen Sie sicher, dass der Kunde über ausreichende Zahlungslimits verfügt
  - Testen Sie den Zahlungsprozess selbst

  </AccordionItem>
</Accordion>

### E-Mail-Zustellung {#email-delivery}

Für zuverlässige E-Mail-Zustellung ziehen Sie in Betracht:

- **SMTP-Dienst**: Verwenden Sie Dienste wie SendGrid, Mailgun oder Amazon SES
- **E-Mail-Plugins**: Installieren Sie WordPress SMTP-Plugins für bessere Zustellung
- **Domänenauthentifizierung**: Richten Sie SPF-, DKIM- und DMARC-Einträge ein
- **Überwachung**: Verfolgen Sie die E-Mail-Zustellung und Öffnungsraten
