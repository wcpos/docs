# E-Mail-Rechnungsgateway

Das E-Mail-Rechnungsgateway ermöglicht es Ihnen, Zahlungsrechnungen direkt aus WooCommerce POS per E-Mail an Kunden zu senden. Dies ist ideal für Situationen, in denen Kunden die Zahlung später bevorzugen oder Zahlungen online über die Zahlungsmethoden Ihres Webshops abschließen möchten.

## Funktionen

- **E-Mail-Integration**: Rechnungen direkt von der POS-Oberfläche senden
- **Kundenverwaltung**: E-Mail-Adresse automatisch aus vorhandenen Kundendaten übernehmen oder neue E-Mail-Adressen hinzufügen
- **Gastbestellungen**: Unterstützung für Gastkunden mit optionaler Aktualisierung der Rechnungsadresse
- **Online-Zahlung**: Kunden erhalten einen Link zur Zahlung der Bestellung online
- **Flexibler Workflow**: Bestellungen im Laden abschließen oder Zahlungsabschluss remote ermöglichen

## Installation

### Schritt 1: Herunterladen und Installieren

1. Besuchen Sie die [E-Mail-Rechnungsgateway-Versionsseite](https://github.com/wcpos/email-invoice-gateway/releases)
2. Laden Sie die neueste **woocommerce-pos-email-invoice-gateway.zip**-Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die ZIP-Datei hoch und aktivieren Sie das Plugin

### Schritt 2: In WooCommerce POS aktivieren

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Finden Sie das **WooCommerce POS Invoice Payment Gateway** in der Liste
3. Aktivieren Sie das Gateway für die Verwendung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Gateway benötigt keine zusätzlichen API-Schlüssel oder externe Dienstkonfiguration. Es verwendet Ihr vorhandenes WordPress-E-Mail-System und WooCommerce-Zahlungsmethoden.
:::

## Nutzung

### Rechnungen senden

1. **Artikel hinzufügen**: Fügen Sie Produkte zum Warenkorb im POS hinzu
2. **Gateway auswählen**: Wählen Sie "E-Mail-Rechnung" als Zahlungsmethode
3. **E-Mail-Adresse**: 
   - Für vorhandene Kunden: E-Mail wird automatisch aus Kundendaten übernommen
   - Für Gastbestellungen: Geben Sie die E-Mail-Adresse des Kunden manuell ein
4. **Optional**: Aktivieren Sie "Zur Rechnungsadresse speichern", um die Rechnungsinformationen des Kunden zu aktualisieren
5. **Rechnung senden**: Schließen Sie den Prozess ab, um die Rechnungs-E-Mail zu senden

### Kundenerlebnis

Wenn Sie eine Rechnung senden, erhält der Kunde eine E-Mail mit:

- **Bestelldetails**: Vollständige Aufschlüsselung der Artikel, Mengen und Preise
- **Gesamtbetrag**: Endbetrag inklusive Steuern und Gebühren
- **Zahlungslink**: Direkter Link zum Abschluss der Zahlung online
- **Bestellinformationen**: Bestellnummer und Referenzdetails

### Zahlungsabschluss

Kunden können die Zahlung auf zwei Arten abschließen:

1. **Online-Zahlung**: Klicken auf den "Für diese Bestellung bezahlen"-Link in der E-Mail
2. **Rückkehr im Laden**: Rückkehr zur Zahlung mit anderen POS-Zahlungsmethoden

Der "Für diese Bestellung bezahlen"-Link führt Kunden zu einer dedizierten Zahlungsseite auf Ihrer Website, wo sie eine der aktivierten Web-Zahlungsgateways (Stripe, PayPal, etc.) nutzen können.
