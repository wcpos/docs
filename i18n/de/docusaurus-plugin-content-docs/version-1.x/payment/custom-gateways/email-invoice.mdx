# E-Mail-Rechnungsgateway {#email-invoice-gateway}

Das E-Mail-Rechnungsgateway ermöglicht es Ihnen, Zahlungsrechnungen direkt aus WooCommerce POS per E-Mail an Kunden zu senden. Dies ist ideal für Situationen, in denen Kunden später bezahlen oder die Zahlung online mit den Zahlungsmethoden Ihres Online-Shops abschließen möchten.

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [E-Mail-Rechnungsgateway Veröffentlichungsseite](https://github.com/wcpos/email-invoice-gateway/releases)
2. Laden Sie die neueste **woocommerce-pos-email-invoice-gateway.zip** Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="Aktivieren in WooCommerce POS" isLast>

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Suchen Sie das **WooCommerce POS Rechnung Zahlungs-Gateway** in der Liste
3. Aktivieren Sie das Gateway zur Nutzung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Gateway erfordert keine zusätzlichen API-Schlüssel oder externe Dienstkonfigurationen. Es verwendet Ihr bestehendes WordPress-E-Mail-System und WooCommerce-Zahlungsmethoden.
:::

  </StepCard>
</Steps>

## Nutzung {#usage}

### Rechnungen senden {#sending-invoices}

1. **Artikel hinzufügen**: Produkte in Ihren Warenkorb im POS hinzufügen
2. **Gateway auswählen**: "E-Mail-Rechnung" als Zahlungsmethode wählen
3. **E-Mail-Adresse**: 
   - Für bestehende Kunden: E-Mail wird automatisch aus den Kundendaten ausgefüllt
   - Für Gastbestellungen: Geben Sie die E-Mail-Adresse des Kunden manuell ein
4. **Optional**: Aktivieren Sie "In Rechnungsadresse speichern", um die Rechnungsinformationen des Kunden zu aktualisieren
5. **Rechnung senden**: Den Prozess zur Versendung der Rechnungs-E-Mail abschließen

### Kundenerfahrung {#customer-experience}

Wenn Sie eine Rechnung senden, erhält der Kunde eine E-Mail mit:

- **Bestelldetails**: Detaillierte Aufschlüsselung der Artikel, Mengen und Preise
- **Bestellsumme**: Endbetrag inklusive Steuern und Gebühren
- **Zahlungslink**: Direkter Link zur vollständigen Zahlung online
- **Bestellinformationen**: Bestellnummer und Referenzdetails

### Zahlungsabschluss {#payment-completion}

Kunden können die Zahlung auf zwei Arten abschließen:

1. **Online-Zahlung**: Klicken Sie auf den Link "Für diese Bestellung bezahlen" in der E-Mail
2. **Zahlung im Geschäft**: Zurückkehren, um die Zahlung mit anderen POS-Zahlungsmethoden abzuschließen

Der Link "Für diese Bestellung bezahlen" führt die Kunden zu einer speziellen Zahlungsseite auf Ihrer Website, wo sie eine der aktivierten Web-Zahlungsgateways (Stripe, PayPal usw.) verwenden können.

## E-Mail-Verwaltung {#email-management}

### Bestehende Kunden {#existing-customers}

- E-Mail-Adressen werden automatisch aus Kundenprofilen abgerufen
- Rechnungsinformationen des Kunden werden gespeichert
- Bestellverlauf wird im Kundenkonto verwaltet

### Gastkunden {#guest-customers}

- E-Mail-Adressen für einmalige Kunden manuell eingeben
- Option zum Speichern der E-Mail in der Rechnungsadresse für zukünftige Referenz
- Gastbestellungen werden korrekt in WooCommerce verfolgt

### E-Mail-Vorlagen {#email-templates}

Das Gateway verwendet die Standard-E-Mail-Vorlagen von WooCommerce:

- **Anpassbar**: E-Mail-Aussehen über die WooCommerce-E-Mail-Einstellungen ändern
- **Branding**: Fügen Sie Ihr Logo und Branding hinzu
- **Professionell**: Sauberes, professionelles Rechnungsformat
- **Mobilfreundlich**: Responsives Design für alle Geräte

## Anwendungsfälle {#use-cases}

### Perfekt für {#perfect-for}

- **B2B-Verkäufe**: Geschäftskunden, die Zahlungen über ihre Buchhaltungssysteme abwickeln müssen
- **Große Bestellungen**: Hochwertige Transaktionen, die Genehmigung oder Bearbeitungszeit benötigen
- **Remote-Kunden**: Kunden, die die Zahlung nach dem Verlassen des Geschäfts abschließen müssen
- **Kunden auf Rechnung**: Regelmäßige Kunden mit vereinbarten Zahlungsbedingungen
- **Angebot zu Bestellung**: Umwandlung von Angeboten oder Kostenvoranschlägen in zahlbare Rechnungen

## Fehlerbehebung {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="E-Mails werden nicht gesendet">

  - Überprüfen Sie die WordPress-E-Mail-Konfiguration
  - Überprüfen Sie die SMTP-Einstellungen, wenn Sie einen benutzerdefinierten E-Mail-Dienst verwenden
  - Testen Sie mit WordPress-E-Mail-Test-Plugins
  - Überprüfen Sie die Spam-Ordner auf Test-E-Mails

  </AccordionItem>

  <AccordionItem question="Zahlungslinks funktionieren nicht">

  - Stellen Sie sicher, dass WooCommerce korrekt konfiguriert ist
  - Überprüfen Sie, ob mindestens ein Zahlungsgateway für den Web-Checkout aktiviert ist
  - Überprüfen Sie, ob das SSL-Zertifikat korrekt installiert ist
  - Bestätigen Sie, dass der Bestellstatus die Zahlung zulässt

  </AccordionItem>

  <AccordionItem question="Kunde kann Zahlung nicht abschließen">

  - Überprüfen Sie, ob die Web-Zahlungsgateways aktiv und konfiguriert sind
  - Überprüfen Sie, dass die Bestellung nicht abgelaufen oder storniert wurde
  - Stellen Sie sicher, dass der Kunde ausreichende Zahlungsmethodenlimits hat
  - Testen Sie den Zahlungsprozess selbst

  </AccordionItem>
</Accordion>

### E-Mail-Zustellung {#email-delivery}

Für eine zuverlässige E-Mail-Zustellung sollten Sie Folgendes berücksichtigen:

- **SMTP-Dienst**: Verwenden Sie Dienste wie SendGrid, Mailgun oder Amazon SES
- **E-Mail-Plugins**: Installieren Sie WordPress SMTP-Plugins für eine bessere Zustellung
- **Domänenauthentifizierung**: SPF-, DKIM- und DMARC-Einträge einrichten
- **Überwachung**: E-Mail-Zustellung und Öffnungsraten verfolgen
