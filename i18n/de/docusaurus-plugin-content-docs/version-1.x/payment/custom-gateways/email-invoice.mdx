# E-Mail-Rechnungs-Gateway

Das E-Mail-Rechnungs-Gateway ermöglicht es Ihnen, Zahlungsrechnungen per E-Mail direkt aus WooCommerce POS an Kunden zu senden. Dies ist perfekt für Situationen, in denen Kunden es vorziehen, später zu zahlen oder die Zahlung online mit den Zahlungsmethoden Ihres Webshops abzuschließen.

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [E-Mail-Rechnungs-Gateway Veröffentlichungsseite](https://github.com/wcpos/email-invoice-gateway/releases)
2. Laden Sie die neueste **woocommerce-pos-email-invoice-gateway.zip** Datei herunter
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die ZIP-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="Im WooCommerce POS aktivieren" isLast>

1. Gehen Sie zu `WP Admin > WooCommerce POS > Einstellungen > Checkout`
2. Suchen Sie das **WooCommerce POS Rechnungs-Zahlungs-Gateway** in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Gateway erfordert keine zusätzlichen API-Schlüssel oder Konfiguration von externen Diensten. Es verwendet Ihr vorhandenes WordPress-E-Mail-System und die Zahlungsmethoden von WooCommerce.
:::

  </StepCard>
</Steps>

## Verwendung

### Rechnungen senden

1. **Artikel hinzufügen**: Produkte in Ihren Warenkorb im POS hinzufügen
2. **Gateway auswählen**: Wählen Sie „E-Mail-Rechnung“ als Zahlungsmethode
3. **E-Mail-Adresse**: 
   - Für bestehende Kunden: E-Mail wird automatisch aus den Kundendaten ausgefüllt
   - Für Gastbestellungen: Geben Sie die E-Mail-Adresse des Kunden manuell ein
4. **Optional**: Aktivieren Sie „In Rechnungsadresse speichern“, um die Rechnungsinformationen des Kunden zu aktualisieren
5. **Rechnung senden**: Schließen Sie den Vorgang ab, um die Rechnungs-E-Mail zu senden

### Kundenerfahrung

Wenn Sie eine Rechnung senden, erhält der Kunde eine E-Mail mit:

- **Bestelldetails**: Vollständige Aufschlüsselung der Artikel, Mengen und Preise
- **Gesamtbetrag**: Endbetrag einschließlich Steuern und Gebühren
- **Zahlungslink**: Direkter Link, um die Zahlung online abzuschließen
- **Bestellinformationen**: Bestellnummer und Referenzdetails

### Zahlungsabschluss

Kunden können die Zahlung auf zwei Arten abschließen:

1. **Online-Zahlung**: Klicken Sie auf den „Für diese Bestellung bezahlen“-Link in der E-Mail
2. **In-Store Rückkehr**: Zurückkehren, um die Zahlung mit anderen POS-Zahlungsmethoden abzuschließen

Der „Für diese Bestellung bezahlen“-Link führt die Kunden zu einer speziellen Zahlungsseite auf Ihrer Website, wo sie alle aktivierten Web-Zahlungsgateways (Stripe, PayPal usw.) verwenden können.

## E-Mail-Verwaltung

### Bestehende Kunden

- E-Mail-Adressen werden automatisch aus Kundenprofilen abgerufen
- Rechnungsinformationen des Kunden werden erhalten
- Bestellhistorie wird im Kundenkonto gespeichert

### Gastkunden

- E-Mail-Adressen für einmalige Kunden manuell eingeben
- Möglichkeit, E-Mail für zukünftige Referenz in der Rechnungsadresse zu speichern
- Gastbestellungen werden in WooCommerce ordnungsgemäß verfolgt

### E-Mail-Vorlagen

Das Gateway verwendet die Standard-E-Mail-Vorlagen von WooCommerce:

- **Anpassbar**: Ändern Sie das Erscheinungsbild der E-Mail über die WooCommerce-E-Mail-Einstellungen
- **Gebrandet**: Fügen Sie Ihr Logo und Branding hinzu
- **Professionell**: Sauberes, professionelles Rechnungsformat
- **Mobilfreundlich**: Responsives Design für alle Geräte

## Anwendungsfälle

### Perfekt für

- **B2B-Verkäufe**: Geschäftskunden, die Zahlungen über ihre Buchhaltungssysteme abwickeln müssen
- **Große Bestellungen**: Hochwertige Transaktionen, die Genehmigung oder Bearbeitungszeit erfordern
- **Fernkunden**: Kunden, die die Zahlung nach Verlassen des Geschäfts abschließen müssen
- **Kunden mit Konto**: Regelmäßige Kunden mit festgelegten Zahlungsbedingungen
- **Angebot zu Bestellung**: Umwandlung von Angeboten oder Kostenschätzungen in zahlbare Rechnungen

## Fehlersuche

### Häufige Probleme

<Accordion>
  <AccordionItem question="E-Mails werden nicht gesendet">

  - Überprüfen Sie die WordPress-E-Mail-Konfiguration
  - Überprüfen Sie die SMTP-Einstellungen, wenn Sie einen benutzerdefinierten E-Mail-Dienst verwenden
  - Testen Sie mit WordPress-E-Mail-Test-Plugins
  - Überprüfen Sie Spam-Ordner auf Test-E-Mails

  </AccordionItem>

  <AccordionItem question="Zahlungslinks funktionieren nicht">

  - Stellen Sie sicher, dass WooCommerce ordnungsgemäß konfiguriert ist
  - Überprüfen Sie, ob mindestens ein Zahlungs-Gateway für den Web-Checkout aktiviert ist
  - Überprüfen Sie, ob das SSL-Zertifikat ordnungsgemäß installiert ist
  - Bestätigen Sie, dass der Bestellstatus die Zahlung zulässt

  </AccordionItem>

  <AccordionItem question="Kunde kann die Zahlung nicht abschließen">

  - Überprüfen Sie, ob die Web-Zahlungsgateways aktiv und konfiguriert sind
  - Überprüfen Sie, ob die Bestellung nicht abgelaufen oder storniert wurde
  - Stellen Sie sicher, dass der Kunde ausreichende Zahlungsmethodenlimits hat
  - Testen Sie den Zahlungsprozess selbst

  </AccordionItem>
</Accordion>

### E-Mail-Zustellung

Für eine zuverlässige E-Mail-Zustellung sollten Sie Folgendes in Betracht ziehen:

- **SMTP-Dienst**: Verwenden Sie Dienste wie SendGrid, Mailgun oder Amazon SES
- **E-Mail-Plugins**: Installieren Sie WordPress-SMTP-Plugins für eine bessere Zustellung
- **Domänenauthentifizierung**: Richten Sie SPF-, DKIM- und DMARC-Einträge ein
- **Überwachung**: Verfolgen Sie die E-Mail-Zustellung und Öffnungsraten

### Unterstützung erhalten

Für technische Unterstützung:
- Besuchen Sie das [GitHub-Repository](https://github.com/wcpos/email-invoice-gateway), um Probleme zu melden
- Überprüfen Sie die WooCommerce-E-Mail-Dokumentation zur Anpassung von Vorlagen
- Testen Sie die E-Mail-Funktionen mit WordPress-E-Mail-Testwerkzeugen
