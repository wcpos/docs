WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedliche Geschäftsanforderungen zu erfüllen. Der Checkout verwendet einen iframe/webview, um die WooCommerce Order Pay-Seite zu laden, was bedeutet, dass jedes Zahlungs-Gateway, das mit WooCommerce funktioniert, auch im POS funktionieren kann.

## Standard-Zahlungsmethoden

Die kostenlose Version umfasst zwei integrierte Zahlungs-Gateways:

### Bargeld

- Gezahlten Betrag eingeben
- Automatische Wechselberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung einfach als abgeschlossen, nachdem Sie sie auf Ihrem Terminal verarbeitet haben
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungs-Gateways (Pro)

:::info Pro Feature
Zusätzliche Zahlungs-Gateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungs-Gateways an der POS-Kasse aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Any WooCommerce Gateway** - Aktivieren Sie ein beliebiges Gateway, das in Ihrem Shop installiert ist

## Gateways aktivieren und deaktivieren

Zahlungs-Gateways werden im WordPress Admin verwaltet:

1. Gehen Sie zu `WP Admin > POS > Settings > Checkout`
2. Sie sehen eine Liste aller installierten WooCommerce-Zahlungsgateways
3. Schalten Sie jedes Gateway für das POS unabhängig von Ihrem Online-Shop ein oder aus
4. Zum Neu-Ordnen ziehen — das erste aktivierte Gateway wird zum Standard bei der Kasse

Das bedeutet, dass ein Gateway auf Ihrer Website aktiv sein kann, aber im POS deaktiviert ist, oder umgekehrt. Weitere Informationen finden Sie unter [Checkout Settings](/settings/wp-admin/checkout).

## Zahlungen testen

Bevor Sie mit einem neuen Gateway live gehen:

1. **Cash gateway** — immer verfügbar zum Testen des Checkout-Ablaufs ohne echte Transaktionen.
2. **Stripe Terminal** — Stripe bietet einen [simulierten Reader](https://docs.stripe.com/terminal/testing) für den Testmodus an. Aktivieren Sie zunächst den Testmodus in Ihrem Stripe-Dashboard.
3. **Drittanbieter-Gateways** — Konsultieren Sie die Dokumentation des Gateways selbst für Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Schalter.

:::tip
Wenn ein neu aktiviertes Gateway nicht an der Kasse angezeigt wird, versuchen Sie, das POS zu aktualisieren. Gateway-Änderungen in WP Admin werden beim nächsten Laden der Seite wirksam.
:::

## Häufige Probleme

### Gateway wird beim Checkout nicht angezeigt

- Bestätigen Sie, dass das Gateway für POS aktiviert ist in `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Gateway auch aktiviert ist in `WP Admin > WooCommerce > Settings > Payments` — ein Gateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt werden kann.
- Aktualisieren Sie das POS nach dem Vornehmen von Änderungen.

### Gateway wird falsch angezeigt

Der Checkout-iframe lädt die Theme-Stile Ihrer Website, was manchmal zu Problemen führen kann. Verwenden Sie die Schaltfläche **Checkout Settings** im Checkout-Modal, um Stile oder Skripte selektiv zu deaktivieren. Siehe [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting) für Details.

### Zahlungsverarbeitung schlägt fehl

- Überprüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) für Gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist – die meisten Payment-Gateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, bestätigen Sie, dass das Gateway den Test-/Sandbox-Modus unterstützt.
