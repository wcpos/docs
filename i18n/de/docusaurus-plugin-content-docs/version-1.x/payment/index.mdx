WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedlichen geschäftlichen Anforderungen gerecht zu werden. Der Checkout verwendet einen iframe/Webview, um die WooCommerce Order Pay-Seite zu laden. Das bedeutet, dass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS verwendet werden kann.

## Standard-Zahlungsmethoden {#default-payment-methods}

Die kostenlose Version enthält zwei integrierte Zahlungsgateways:

### Bargeld {#cash}

- Erhaltenen Betrag eingeben
- Automatische Berechnung des Wechselgelds
- Keine zusätzliche Konfiguration erforderlich

### Karte {#card}

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung nach der Verarbeitung am Terminal einfach als abgeschlossen
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro) {#additional-payment-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungsgateways im POS-Checkout aktivieren:

- **Stripe Terminal** – Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp Terminal** – Integration mit SumUp-Kartenlesegeräten
- **Beliebiges WooCommerce-Gateway** – Aktivieren Sie jedes in Ihrem Shop installierte Gateway

## Zahlungsgateways aktivieren und deaktivieren {#enabling-and-disabling-gateways}

Zahlungsgateways werden im WordPress-Admin verwaltet:

1. Gehe zu `WP Admin > POS > Settings > Checkout`
2. Es wird eine Liste aller installierten WooCommerce-Zahlungsgateways angezeigt
3. Jedes Gateway lässt sich für den POS unabhängig vom Online-Shop ein- oder ausschalten
4. Per Drag-and-drop neu anordnen — das erste aktivierte Gateway wird zur Standardoption an der Kasse

Das bedeutet, dass ein Gateway auf der Website aktiv, im POS aber deaktiviert sein kann – oder umgekehrt. Weitere Details finden sich unter [Checkout-Einstellungen](/settings/wp-admin/checkout).

## Zahlungen testen {#testing-payments}

Vor dem produktiven Einsatz eines neuen Gateways:

1. **Bargeld-Gateway** — immer verfügbar, um den Checkout-Ablauf ohne echte Transaktionen zu testen.
2. **Stripe Terminal** — Stripe stellt einen [simulierten Kartenleser](https://docs.stripe.com/terminal/testing) für den Testmodus bereit. Aktiviere zunächst den Testmodus im Stripe-Dashboard.
3. **Drittanbieter-Gateways** — Hinweise zum Sandbox- bzw. Testmodus finden sich in der jeweiligen Dokumentation des Gateways. Die meisten WooCommerce-Gateways bieten einen Schalter für den Testmodus.

:::tip
Falls ein neu aktiviertes Gateway an der Kasse nicht erscheint, hilft ein Neuladen des POS. Änderungen an Gateways im WP Admin werden beim nächsten Seitenaufruf wirksam.
:::

## Häufige Probleme {#common-issues}

### Gateway erscheint nicht im Checkout {#gateway-not-appearing-at-checkout}

- Stellen Sie sicher, dass das Gateway für den POS aktiviert ist in `WP Admin > POS > Settings > Checkout`.
- Prüfen Sie, ob das Gateway ebenfalls aktiviert ist in `WP Admin > WooCommerce > Settings > Payments` — ein Gateway muss in WooCommerce aktiv sein, bevor es im POS erscheinen kann.
- Aktualisieren Sie den POS nach Änderungen.

### Gateway wird falsch angezeigt {#gateway-displays-incorrectly}

Der Checkout-iframe lädt die Theme-Stile Ihrer Website, was gelegentlich zu Konflikten führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Stile oder Skripte gezielt zu deaktivieren. Weitere Informationen finden Sie unter [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting).

### Zahlungsverarbeitung schlägt fehl {#payment-processing-fails}

- Prüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) für Gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist – die meisten Zahlungsgateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, prüfen Sie, ob das Gateway den Test-/Sandbox-Modus unterstützt.
