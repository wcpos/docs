WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedliche Geschäftsanforderungen zu erfüllen. Der Checkout verwendet einen iframe/webview, um die WooCommerce Order Pay Seite zu laden, was bedeutet, dass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS funktioniert.

## Standard-Zahlungsmethoden

Die kostenlose Version enthält zwei integrierte Zahlungsgateways:

### Bargeld

- Gezahlten Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung einfach nach der Verarbeitung am Terminal als abgeschlossen
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro)

:::info Pro Feature
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungsgateways im POS-Checkout aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Any WooCommerce Gateway** - Aktivieren Sie ein beliebiges auf Ihrem Shop installiertes Gateway

## Gateways aktivieren und deaktivieren

Zahlungsgateways werden im WordPress-Admin verwaltet:

1. Gehen Sie zu `WP Admin > POS > Settings > Checkout`
2. Sie sehen eine Liste aller installierten WooCommerce-Zahlungsgateways
3. Aktivieren oder deaktivieren Sie jeden Gateway für das POS unabhängig von Ihrem Online-Shop
4. Ziehen Sie, um die Reihenfolge zu ändern — der erste aktivierte Gateway wird zum Standard bei der Kasse

Das bedeutet, dass ein Gateway auf Ihrer Website aktiv, aber in der POS deaktiviert sein kann, oder umgekehrt. Weitere Informationen finden Sie unter [Checkout Settings](/settings/wp-admin/checkout).

## Zahlungen testen

Bevor Sie mit einem neuen Gateway live gehen:

1. **Cash gateway** — immer verfügbar zum Testen des Checkout-Flows ohne echte Transaktionen.
2. **Stripe Terminal** — Stripe stellt einen [simulierten Reader](https://docs.stripe.com/terminal/testing) für den Testmodus bereit. Aktivieren Sie zunächst den Testmodus in Ihrem Stripe-Dashboard.
3. **Gateways von Drittanbietern** — Lesen Sie die Dokumentation des Gateways auf Anweisungen zum Sandbox/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Toggle.

:::tip
Wenn ein neu aktiviertes Gateway nicht bei der Kasse angezeigt wird, versuchen Sie, das POS zu aktualisieren. Gateway-Änderungen in WP Admin werden beim nächsten Seitenladevorgang wirksam.
:::

## Häufige Probleme

### Gateway wird an der Kasse nicht angezeigt

- Bestätigen Sie, dass das Gateway für POS in aktiviert ist `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Gateway auch in aktiviert ist `WP Admin > WooCommerce > Settings > Payments` — ein Gateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt werden kann.
- Aktualisieren Sie das POS nach Änderungen.

### Gateway wird nicht korrekt angezeigt

Der Checkout-iframe lädt die Theme-Stile Ihrer Site, was manchmal zu Problemen führen kann. Verwenden Sie die Schaltfläche **Checkout Settings** im Checkout-Modal, um selektiv Styles oder Scripts zu deaktivieren. Weitere Informationen finden Sie unter [Checkout Troubleshooting](/pos/checkout#checkout-settings-troubleshooting).

### Zahlungsverarbeitung schlägt fehl

- Überprüfen Sie die Fehlerprotokolle Ihrer Site (`WP Admin > POS > Support > Logs`) für Gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist — die meisten Payment-Gateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, bestätigen Sie, dass das Gateway den Test-/Sandbox-Modus unterstützt.
