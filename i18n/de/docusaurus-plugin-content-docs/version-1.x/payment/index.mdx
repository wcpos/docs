WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedliche geschäftliche Anforderungen zu erfüllen. Der Checkout verwendet einen iframe/webview, um die WooCommerce Order Pay-Seite zu laden, was bedeutet, dass jedes Zahlungs-Gateway, das mit WooCommerce funktioniert, auch im POS funktionieren kann.

## Standard-Zahlungsmethoden {#default-payment-methods}

Die kostenlose Version enthält zwei integrierte Zahlungs-Gateways:

### Bargeld {#cash}

- Gezahlten Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte {#card}

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung einfach als abgeschlossen, nachdem sie auf Ihrem Terminal verarbeitet wurde
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro) {#additional-payment-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungsgateways in der POS-Kasse aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Beliebiges WooCommerce-Gateway** - Aktivieren Sie jedes auf Ihrem Shop installierte Gateway

## Gateways aktivieren und deaktivieren {#enabling-and-disabling-gateways}

Zahlungsgateways werden im WordPress-Admin verwaltet:

1. Gehen Sie zu `WP Admin > POS > Settings > Checkout`
2. Es wird eine Liste aller installierten WooCommerce-Zahlungsgateways angezeigt.
3. Schalten Sie jedes Gateway für das POS unabhängig von Ihrem Online-Shop ein oder aus.
4. Zum Neu anordnen ziehen – das erste aktivierte Gateway wird an der Kasse zum Standard.

Dies bedeutet, dass ein Gateway auf Ihrer Website aktiv sein kann, aber im POS deaktiviert ist, oder umgekehrt. Weitere Details finden Sie unter [Kasseneinstellungen](/settings/wp-admin/checkout).

## Zahlungen testen {#testing-payments}

Bevor Sie mit einem neuen Gateway live gehen:

1. **Bargeld-Gateway** – immer verfügbar zum Testen des Checkout-Ablaufs ohne echte Transaktionen.
2. **Stripe Terminal** – Stripe bietet einen [simulierten Reader](https://docs.stripe.com/terminal/testing) für den Testmodus. Aktivieren Sie zunächst den Testmodus in Ihrem Stripe-Dashboard.
3. **Gateways von Drittanbietern** – konsultieren Sie die Dokumentation des Gateways für Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Umschalter.

:::tip
Wenn ein neu aktiviertes Gateway an der Kasse nicht angezeigt wird, aktualisieren Sie das POS. Gatewayänderungen in WP Admin werden beim nächsten Laden der Seite wirksam.
:::

## Häufige Probleme {#common-issues}

### Gateway wird beim Checkout nicht angezeigt {#gateway-not-appearing-at-checkout}

- Bestätigen Sie, dass das Gateway für POS aktiviert ist in `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Gateway auch aktiviert ist in `WP Admin > WooCommerce > Settings > Payments` — Ein Gateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt werden kann.
- Aktualisieren Sie das POS nach dem Vornehmen von Änderungen.

### Gateway wird nicht korrekt angezeigt {#gateway-displays-incorrectly}

Der Checkout-iframe lädt die Theme-Stile Ihrer Website, was manchmal zu Problemen führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Stile oder Skripte selektiv zu deaktivieren. Weitere Informationen finden Sie unter [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting).

### Zahlungsverarbeitung schlägt fehl {#payment-processing-fails}

- Überprüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`") für Gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist — die meisten Payment-Gateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, bestätigen Sie, dass das Gateway den Test-/Sandbox-Modus unterstützt.
