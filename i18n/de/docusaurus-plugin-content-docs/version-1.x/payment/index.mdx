WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedlichen geschäftlichen Anforderungen gerecht zu werden. Der Checkout verwendet ein iframe/eine WebView, um die WooCommerce-Seite zur Bezahlung der Bestellung zu laden. Das bedeutet, dass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS funktionieren kann.

## Standard-Zahlungsmethoden {#default-payment-methods}

Die kostenlose Version enthält zwei integrierte Zahlungsgateways:

### Bargeld {#cash}

- Gezahlten Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte {#card}

- Zur Verwendung mit externen Kartenterminals
- Die Zahlung nach der Verarbeitung auf Ihrem Terminal einfach als abgeschlossen markieren
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro) {#additional-payment-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungsgateways im POS-Checkout aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp Terminal** - Integration mit SumUp-Kartenlesegeräten
- **Beliebiges WooCommerce-Zahlungsgateway** - Jedes in Ihrem Shop installierte Gateway aktivieren

## Gateways aktivieren und deaktivieren {#enabling-and-disabling-gateways}

Zahlungsgateways werden im WordPress-Adminbereich verwaltet:

1. Gehen Sie zu `WP Admin > POS > Einstellungen > Checkout`
2. Es wird eine Liste aller installierten WooCommerce-Zahlungsgateways angezeigt
3. Aktivieren oder deaktivieren Sie jedes Gateway für den POS unabhängig von Ihrem Online-Shop
4. Zum Neuordnen ziehen — das erste aktivierte Gateway wird beim Checkout zum Standard

Das bedeutet, dass ein Gateway auf Ihrer Website aktiv, im POS aber deaktiviert sein kann, oder umgekehrt. Vollständige Details finden Sie unter [Checkout-Einstellungen](/settings/wp-admin/checkout).

## Zahlungen testen {#testing-payments}

Vor dem Live-Betrieb mit einem neuen Gateway:

1. **Bargeld-Gateway** — immer verfügbar, um den Checkout-Ablauf ohne echte Transaktionen zu testen.
2. **Stripe Terminal** — Stripe stellt für den Testmodus einen [simulierten Reader](https://docs.stripe.com/terminal/testing) bereit. Aktivieren Sie zuerst den Testmodus in Ihrem Stripe-Dashboard.
3. **Drittanbieter-Gateways** — prüfen Sie die Dokumentation des jeweiligen Gateways auf Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen eine Umschaltung für den Testmodus.

:::tip
Wenn ein neu aktiviertes Gateway beim Checkout nicht angezeigt wird, versuchen Sie, den POS neu zu laden. Änderungen an Gateways in WP Admin werden beim nächsten Laden der Seite wirksam.
:::

## Häufige Probleme {#common-issues}

### Gateway erscheint nicht beim Checkout {#gateway-not-appearing-at-checkout}

- Bestätigen Sie, dass das Gateway für POS unter `WP Admin > POS > Einstellungen > Checkout` aktiviert ist.
- Prüfen Sie, ob das Gateway auch unter `WP Admin > WooCommerce > Einstellungen > Zahlungen` aktiviert ist — ein Gateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt werden kann.
- Aktualisieren Sie den POS, nachdem Sie Änderungen vorgenommen haben.

### Gateway wird falsch angezeigt {#gateway-displays-incorrectly}

Das Checkout-iframe lädt die Theme-Stile Ihrer Website, was gelegentlich zu Konflikten führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Stile oder Skripte gezielt zu deaktivieren. Weitere Details finden Sie unter [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting).

### Zahlungsverarbeitung schlägt fehl {#payment-processing-fails}

- Prüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Protokolle`) auf Gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist — die meisten Zahlungsgateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, bestätigen Sie, dass das Gateway den Test-/Sandbox-Modus unterstützt.
