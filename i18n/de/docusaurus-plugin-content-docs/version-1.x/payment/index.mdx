WCPOS unterstützt mehrere Zahlungsmethoden, um verschiedenen Geschäftsanforderungen gerecht zu werden. Die Kasse verwendet einen iframe/webview zum Laden der WooCommerce-Bestellzahlungsseite, was bedeutet, dass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS funktionieren kann.

## Standard-Zahlungsmethoden {#default-payment-methods}

Die kostenlose Version enthält zwei integrierte Zahlungsgateways:

### Bargeld {#cash}

- Gezahlten Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte {#card}

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung einfach als abgeschlossen, nachdem Sie sie auf Ihrem Terminal bearbeitet haben
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro) {#additional-payment-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungsgateways im POS-Checkout aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Beliebiges WooCommerce-Gateway** - Aktivieren Sie jedes auf Ihrem Shop installierte Gateway

## Gateways aktivieren und deaktivieren {#enabling-and-disabling-gateways}

Zahlungsgateways werden in der WordPress-Verwaltung verwaltet:

1. Gehen Sie zu `WP Admin > POS > Settings > Checkout`
2. Sie sehen eine Liste aller installierten WooCommerce-Zahlungsgateways
3. Aktivieren oder deaktivieren Sie jedes Gateway für den POS unabhängig von Ihrem Online-Shop
4. Zum Neuordnen ziehen — das erste aktivierte Gateway wird zum Standard im POS

Das bedeutet, dass ein Gateway auf Ihrer Website aktiv sein, aber im POS deaktiviert sein kann, oder umgekehrt. Weitere Informationen finden Sie in den [Checkout-Einstellungen](/settings/wp-admin/checkout).

## Zahlungen testen {#testing-payments}

Bevor Sie ein neues Gateway live schalten:

1. **Bargeld-Gateway** — immer verfügbar zum Testen des Checkout-Ablaufs ohne echte Transaktionen.
2. **Stripe Terminal** — Stripe stellt einen [simulierten Reader](https://docs.stripe.com/terminal/testing) für den Testmodus bereit. Aktivieren Sie zunächst den Testmodus in Ihrem Stripe-Dashboard.
3. **Gateways von Drittanbietern** — überprüfen Sie die eigene Dokumentation des Gateways auf Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Umschalter.

:::tip
Wenn ein neu aktiviertes Gateway nicht an der Kasse angezeigt wird, versuchen Sie, den POS zu aktualisieren. Gateway-Änderungen in WP Admin werden beim nächsten Laden der Seite wirksam.
:::

## Häufige Probleme {#common-issues}

### Zahlungsgateway wird beim Checkout nicht angezeigt {#gateway-not-appearing-at-checkout}

- Bestätigen Sie, dass das Zahlungsgateway im POS in `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Zahlungsgateway auch aktiviert ist in `WP Admin > WooCommerce > Settings > Payments` — ein Zahlungsgateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt werden kann.
- Aktualisieren Sie den POS nach Änderungen.

### Zahlungsgateway wird nicht korrekt angezeigt {#gateway-displays-incorrectly}

Der Checkout-iframe lädt die Theme-Stile Ihrer Website, was manchmal zu Interferenzen führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Stile oder Skripte selektiv zu deaktivieren. Siehe [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting) für Details.

### Zahlungsverarbeitung schlägt fehl {#payment-processing-fails}

- Überprüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) für Gateway-spezifische Fehler.
- Das SSL-Zertifikat muss gültig sein — die meisten Zahlungsgateways erfordern HTTPS.
- Bestätigen Sie bei Verwendung einer Staging- oder lokalen Umgebung, dass das Gateway den Test-/Sandbox-Modus unterstützt.
