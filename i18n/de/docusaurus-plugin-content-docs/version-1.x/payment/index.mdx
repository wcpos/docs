WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedlichen geschäftlichen Anforderungen gerecht zu werden. Der Checkout verwendet ein iframe/Webview, um die WooCommerce-Seite „Order Pay“ zu laden, was bedeutet, dass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS funktionieren kann.

## Standard-Zahlungsmethoden {#default-payment-methods}

Die kostenlose Version enthält zwei integrierte Zahlungsgateways:

### Bargeld {#cash}

- Eingabe des erhaltenen Betrags
- Automatische Berechnung des Wechselgeldes
- Keine zusätzliche Konfiguration erforderlich

### Karte {#card}

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung einfach als abgeschlossen, nachdem sie am Terminal verarbeitet wurde
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro) {#additional-payment-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie im POS-Checkout zusätzliche WooCommerce-Zahlungsgateways aktivieren:

- **Stripe Terminal** – Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp Terminal** – Integration mit SumUp-Kartenlesegeräten
- **Beliebiges WooCommerce-Gateway** – Aktivieren Sie ein beliebiges in Ihrem Shop installiertes Gateway

## Gateways aktivieren und deaktivieren {#enabling-and-disabling-gateways}

Zahlungsgateways werden im WordPress-Admin verwaltet:

1. Navigieren Sie zu `WP Admin > POS > Settings > Checkout`
2. Es wird eine Liste aller installierten WooCommerce-Zahlungsgateways angezeigt
3. Jedes Gateway kann unabhängig vom Online-Shop für den POS aktiviert oder deaktiviert werden
4. Per Drag-and-drop neu anordnen – das erste aktivierte Gateway wird zum Standard an der Kasse

Das bedeutet, dass ein Gateway auf der Website aktiv, im POS jedoch deaktiviert sein kann – oder umgekehrt. Weitere Informationen finden Sie unter [Checkout-Einstellungen](/settings/wp-admin/checkout).

## Zahlungen testen {#testing-payments}

Bevor ein neues Gateway in Betrieb genommen wird:

1. **Bargeld-Gateway** – immer verfügbar, um den Checkout-Ablauf ohne echte Transaktionen zu testen.
2. **Stripe Terminal** – Stripe stellt einen [simulierten Reader](https://docs.stripe.com/terminal/testing) für den Testmodus bereit. Aktivieren Sie zuvor den Testmodus im Stripe-Dashboard.
3. **Drittanbieter-Gateways** – Hinweise zu Sandbox- bzw. Testmodus finden Sie in der jeweiligen Dokumentation des Gateways. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Schalter.

:::tip
Falls ein neu aktiviertes Gateway an der Kasse nicht erscheint, sollte der POS aktualisiert werden. Gateway-Änderungen im WP-Admin werden beim nächsten Seitenladen wirksam.
:::

## Häufige Probleme {#common-issues}

### Gateway erscheint nicht im Checkout {#gateway-not-appearing-at-checkout}

- Stellen Sie sicher, dass das Gateway für den POS aktiviert ist unter `WP Admin > POS > Settings > Checkout`.
- Prüfen Sie, ob das Gateway auch aktiviert ist unter `WP Admin > WooCommerce > Settings > Payments` — ein Gateway muss in WooCommerce aktiv sein, bevor es im POS erscheinen kann.
- Laden Sie den POS nach Änderungen neu.

### Gateway wird falsch dargestellt {#gateway-displays-incorrectly}

Der Checkout-iframe lädt die Theme-Stile Ihrer Website, was mitunter zu Konflikten führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Stile oder Skripte gezielt zu deaktivieren. Weitere Informationen finden Sie unter [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting).

### Zahlungsverarbeitung schlägt fehl {#payment-processing-fails}

- Prüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) für gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist – die meisten Zahlungsgateways erfordern HTTPS.
- Bei Verwendung einer Staging- oder lokalen Umgebung ist zu prüfen, ob das Gateway den Test-/Sandbox-Modus unterstützt.
