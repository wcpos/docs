WCPOS unterstützt mehrere Zahlungsmethoden, um verschiedene Geschäftsanforderungen zu erfüllen. Der Checkout verwendet einen iframe/webview, um die WooCommerce Order Pay-Seite zu laden, was bedeutet, dass jedes Zahlungsgateway, das mit WooCommerce funktioniert, im POS funktionieren kann.

## Standardzahlungsmethoden

Die kostenlose Version enthält zwei integrierte Zahlungsgateways:

### Bargeld

- Gezahlten Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung einfach als abgeschlossen, nachdem Sie sie an Ihrem Terminal bearbeitet haben
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro)

:::info Pro Feature
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungsgateways an der POS-Kasse aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Any WooCommerce Gateway** - Aktivieren Sie ein beliebiges auf Ihrem Shop installiertes Gateway

## Aktivieren und Deaktivieren von Gateways

Zahlungsgateways werden im WordPress-Admin verwaltet:

1. Gehen Sie zu `WP Admin > POS > Settings > Checkout`
2. Sie sehen eine Liste aller installierten WooCommerce-Zahlungsgateways
3. Schalten Sie jedes Gateway für das POS unabhängig von Ihrem Online-Shop ein oder aus
4. Ziehen Sie, um neu zu ordnen — das erste aktivierte Gateway wird beim Checkout zur Standardeinstellung

Dies bedeutet, dass ein Gateway auf Ihrer Website aktiv sein kann, aber im POS deaktiviert ist, oder umgekehrt. Siehe [Checkout-Einstellungen](/settings/wp-admin/checkout) für vollständige Details.

## Zahlungen testen

Vor dem Live-Schalten mit einem neuen Gateway:

1. **Cash-Gateway** — immer verfügbar zum Testen des Checkout-Ablaufs ohne echte Transaktionen.
2. **Stripe Terminal** — Stripe bietet ein [simulierten Leser](https://docs.stripe.com/terminal/testing) für Testmodus. Aktivieren Sie zunächst den Testmodus in Ihrem Stripe-Dashboard.
3. **Gateways von Drittanbietern** — Überprüfen Sie die eigene Dokumentation des Gateways auf Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Schalter.

:::tip
Wenn ein neu aktiviertes Gateway beim Checkout nicht angezeigt wird, versuchen Sie, das POS zu aktualisieren. Gateway-Änderungen in WP Admin werden beim nächsten Seitenladevorgang wirksam.
:::

## Häufige Probleme

### Gateway wird beim Checkout nicht angezeigt

- Bestätigen Sie, dass das Gateway für POS aktiviert ist in `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Gateway auch aktiviert ist in `WP Admin > WooCommerce > Settings > Payments` — Ein Gateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt werden kann.
- Aktualisieren Sie das POS nach Änderungen.

### Gateway wird falsch angezeigt

Der Checkout-iFrame lädt die Theme-Styles Ihrer Website, was manchmal zu Konflikten führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Styles oder Skripte selektiv zu deaktivieren. Siehe [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting) für Details.

### Zahlungsverarbeitung schlägt fehl

- Überprüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) auf Gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist — die meisten Payment-Gateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, bestätigen Sie, dass das Gateway den Test-/Sandbox-Modus unterstützt.
