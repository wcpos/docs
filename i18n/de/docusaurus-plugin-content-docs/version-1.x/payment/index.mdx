WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedliche Geschäftsanforderungen zu erfüllen. Der Checkout verwendet einen iframe/webview, um die WooCommerce Order Pay-Seite zu laden, was bedeutet, dass jedes Zahlungs-Gateway, das mit WooCommerce funktioniert, im POS funktionieren kann.

## Standard-Zahlungsmethoden

Die kostenlose Version umfasst zwei integrierte Zahlungs-Gateways:

### Bargeld

- Bezahlten Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung einfach als abgeschlossen, nachdem Sie sie an Ihrem Terminal verarbeitet haben
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungs-Gateways (Pro)

:::info Pro Feature
Zusätzliche Zahlungs-Gateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungs-Gateways an der POS-Kasse aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Any WooCommerce Gateway** - Aktivieren Sie jedes auf Ihrem Store installierte Gateway

## Gateways aktivieren und deaktivieren

Zahlungs-Gateways werden im WordPress-Admin verwaltet:

1. Gehen Sie zu `WP Admin > POS > Settings > Checkout`
2. Sie sehen eine Liste aller installierten WooCommerce-Zahlungsgateways
3. Schalten Sie jedes Gateway für das POS unabhängig von Ihrem Online-Shop ein oder aus
4. Ziehen Sie zum Neuordnen – das erste aktivierte Gateway wird zum Standard beim Checkout

Das bedeutet, dass ein Gateway auf Ihrer Website aktiv sein kann, aber im POS deaktiviert, oder umgekehrt. Weitere Informationen finden Sie in [Checkout Settings](/settings/wp-admin/checkout).

## Zahlungen testen

Bevor Sie mit einem neuen Gateway live gehen:

1. **Cash gateway** – immer verfügbar zum Testen des Checkout-Ablaufs ohne echte Transaktionen.
2. **Stripe Terminal** – Stripe stellt einen [simulated reader](https://docs.stripe.com/terminal/testing) für den Testmodus bereit. Aktivieren Sie zuerst den Testmodus in Ihrem Stripe-Dashboard.
3. **Gateways von Drittanbietern** – überprüfen Sie die Dokumentation des Gateways auf Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Schalter.

:::tip
Wenn ein neu aktiviertes Gateway beim Checkout nicht angezeigt wird, versuchen Sie, das POS zu aktualisieren. Gateway-Änderungen im WP Admin werden beim nächsten Seitenladevorgang wirksam.
:::

## Häufige Probleme

### Gateway wird beim Checkout nicht angezeigt

- Bestätigen Sie, dass das Gateway für POS aktiviert ist in `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Gateway auch aktiviert ist in `WP Admin > WooCommerce > Settings > Payments` — Ein Gateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt wird.
- Aktualisieren Sie das POS nach Änderungen.

### Gateway wird nicht korrekt angezeigt

Der Checkout-iframe lädt die Design-Stile Ihrer Website, was manchmal zu Problemen führen kann. Verwenden Sie die Schaltfläche **Checkout Settings** im Checkout-Modal, um Stile oder Skripte selektiv zu deaktivieren. Weitere Informationen finden Sie unter [Checkout Troubleshooting](/pos/checkout#checkout-settings-troubleshooting).

### Zahlungsverarbeitung fehlgeschlagen

- Überprüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) für Gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist — die meisten Zahlungs-Gateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, bestätigen Sie, dass das Gateway den Test-/Sandbox-Modus unterstützt.
