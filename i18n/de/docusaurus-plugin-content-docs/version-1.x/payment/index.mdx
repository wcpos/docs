WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedliche Geschäftsanforderungen zu erfüllen. Der Checkout verwendet einen iframe/webview, um die WooCommerce Order Pay-Seite zu laden, was bedeutet, dass jedes Zahlungs-Gateway, das mit WooCommerce funktioniert, auch im POS funktionieren kann.

## Standard-Zahlungsmethoden

Die kostenlose Version enthält zwei integrierte Zahlungs-Gateways:

### Bargeld

- Gezahlten Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung nach der Verarbeitung auf Ihrem Terminal einfach als abgeschlossen
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro)

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungsgateways in der POS-Kasse aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesern
- **SumUp Terminal** - Integration mit SumUp-Kartenlesern
- **Any WooCommerce Gateway** - Aktivieren Sie jedes auf Ihrem Shop installierte Gateway

## Aktivieren und Deaktivieren von Gateways

Zahlungsgateways werden in der WordPress-Verwaltung verwaltet:

1. Gehen Sie zu `WP Admin > POS > Settings > Checkout`
2. Sie sehen eine Liste aller installierten WooCommerce-Zahlungsgateways
3. Schalten Sie jedes Gateway für das POS unabhängig von Ihrem Online-Shop ein oder aus
4. Ziehen Sie zum Neu ordnen — das erste aktivierte Gateway wird zum Standard beim Checkout

Dies bedeutet, dass ein Gateway auf Ihrer Website aktiv, aber im POS deaktiviert sein kann, oder umgekehrt. Siehe [Checkout-Einstellungen](/settings/wp-admin/checkout) für vollständige Details.

## Zahlungen testen

Bevor Sie mit einem neuen Gateway live gehen:

1. **Cash Gateway** — immer verfügbar, um den Checkout-Ablauf ohne echte Transaktionen zu testen.
2. **Stripe Terminal** — Stripe bietet ein [simulierten Leser](https://docs.stripe.com/terminal/testing) für Testmodus. Aktivieren Sie zuerst den Testmodus in Ihrem Stripe-Dashboard.
3. **Third-Party-Gateways** — Überprüfen Sie die Dokumentation des Gateways auf Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Schalter.

:::tip
Wenn ein neu aktiviertes Gateway nicht beim Checkout angezeigt wird, versuchen Sie, das POS zu aktualisieren. Änderungen am Gateway in WP Admin treten beim nächsten Seitenladevorgang in Kraft.
:::

## Häufige Probleme

### Gateway wird nicht beim Checkout angezeigt

- Bestätigen Sie, dass das Gateway für POS aktiviert ist in `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Gateway auch aktiviert ist in `WP Admin > WooCommerce > Settings > Payments` — Ein Gateway muss in WooCommerce aktiv sein, bevor es in der POS angezeigt werden kann.
- Aktualisieren Sie die POS nach Änderungen.

### Gateway wird nicht korrekt angezeigt

Der Checkout-iframe lädt die Theme-Stile Ihrer Website, was manchmal zu Interferenzen führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Stile oder Skripte selektiv zu deaktivieren. Siehe [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting) für Details.

### Zahlungsabwicklung schlägt fehl

- Überprüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) auf gatewayspezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist – die meisten Zahlungsgateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, bestätigen Sie, dass das Gateway den Test-/Sandbox-Modus unterstützt.
