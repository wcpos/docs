WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedliche geschäftliche Anforderungen zu erfüllen. Der Checkout verwendet einen iframe/webview zum Laden der WooCommerce Order Pay-Seite, was bedeutet, dass jedes Zahlungsgateway, das mit WooCommerce funktioniert, auch im POS funktionieren kann.

## Standard-Zahlungsmethoden {#default-payment-methods}

Die kostenlose Version enthält zwei integrierte Zahlungsgateways:

### Bargeld {#cash}

- Erhaltenen Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte {#card}

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung nach der Verarbeitung an Ihrem Terminal einfach als abgeschlossen
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungs-Gateways (Pro) {#additional-payment-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungs-Gateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungs-Gateways im POS-Checkout aktivieren:

- **Stripe Terminal** – Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp Terminal** – Integration mit SumUp-Kartenlesegeräten
- **Beliebiges WooCommerce-Gateway** – Aktivieren Sie ein beliebiges auf Ihrem Shop installiertes Gateway

## Aktivieren und Deaktivieren von Gateways {#enabling-and-disabling-gateways}

Zahlungs-Gateways werden in der WordPress-Verwaltung verwaltet:

1. Navigieren Sie zu `WP Admin > POS > Settings > Checkout`
2. Es wird eine Liste aller installierten WooCommerce-Zahlungsgateways angezeigt
3. Aktivieren oder deaktivieren Sie jedes Gateway für den POS unabhängig von Ihrem Online-Shop
4. Ziehen Sie zum Neuordnen — das erste aktivierte Gateway wird beim Checkout zur Standardoption

Das bedeutet, dass ein Gateway auf Ihrer Website aktiv sein kann, aber im POS deaktiviert ist, oder umgekehrt. Vollständige Details finden Sie unter [Checkout-Einstellungen](/settings/wp-admin/checkout).

## Zahlungen testen {#testing-payments}

Vor dem Live-Schalten mit einem neuen Gateway:

1. **Cash-Gateway** — immer verfügbar zum Testen des Checkout-Ablaufs ohne echte Transaktionen.
2. **Stripe Terminal** — Stripe bietet einen [simulierten Reader](https://docs.stripe.com/terminal/testing) für den Testmodus. Aktivieren Sie zunächst den Testmodus in Ihrem Stripe-Dashboard.
3. **Gateways von Drittanbietern** — lesen Sie die eigene Dokumentation des Gateways für Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Schalter.

:::tip
Wenn ein neu aktiviertes Gateway beim Checkout nicht angezeigt wird, versuchen Sie, den POS zu aktualisieren. Gateway-Änderungen im WP Admin werden beim nächsten Laden der Seite wirksam.
:::

## Häufige Probleme {#common-issues}

### Gateway wird nicht beim Checkout angezeigt {#gateway-not-appearing-at-checkout}

- Bestätigen Sie, dass das Gateway für POS aktiviert ist in `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Gateway auch aktiviert ist in `WP Admin > WooCommerce > Settings > Payments` — ein Gateway muss in WooCommerce aktiv sein, bevor es in der POS erscheinen kann.
- Aktualisieren Sie die POS nach Änderungen.

### Gateway wird nicht korrekt angezeigt {#gateway-displays-incorrectly}

Der Checkout-Iframe lädt die Theme-Stile Ihrer Website, was manchmal zu Konflikten führen kann. Verwenden Sie die Schaltfläche **Kassierereinstellungen** im Checkout-Modal, um Stile oder Skripte gezielt zu deaktivieren. Weitere Informationen finden Sie unter [Kassierer-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting).

### Zahlungsverarbeitung fehlgeschlagen {#payment-processing-fails}

- Überprüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) für gatewayspezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist — die meisten Payment-Gateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, bestätigen Sie, dass das Gateway den Test-/Sandbox-Modus unterstützt.
