WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedlichen geschäftlichen Anforderungen gerecht zu werden. Der Checkout verwendet ein Iframe/Webview, um die WooCommerce-Seite zur Bestellzahlung zu laden, sodass jedes mit WooCommerce kompatible Zahlungsgateway auch im POS funktionieren kann.

## Standard-Zahlungsmethoden {#default-payment-methods}

Die kostenlose Version enthält zwei integrierte Zahlungsgateways:

### Bargeld {#cash}

- Erhaltenen Betrag eingeben
- Automatische Berechnung des Wechselgelds
- Keine zusätzliche Konfiguration erforderlich

### Karte {#card}

- Zur Verwendung mit externen Kartenterminals
- Die Zahlung nach der Verarbeitung auf dem Terminal einfach als abgeschlossen markieren
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro) {#additional-payment-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro lassen sich zusätzliche WooCommerce-Zahlungsgateways im POS-Checkout aktivieren:

- **Stripe Terminal** – Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp Terminal** – Integration mit SumUp-Kartenlesegeräten
- **Beliebiges WooCommerce-Gateway** – Aktivierung eines beliebigen im Shop installierten Gateways

## Gateways aktivieren und deaktivieren {#enabling-and-disabling-gateways}

Zahlungsgateways werden im WordPress-Adminbereich verwaltet:

1. Gehen Sie zu `WP Admin > POS > Settings > Checkout`
2. Sie sehen eine Liste aller installierten WooCommerce-Zahlungsgateways
3. Aktivieren oder deaktivieren Sie jedes Gateway für den POS unabhängig von Ihrem Onlineshop
4. Per Drag-and-drop neu anordnen – das erste aktivierte Gateway wird zur Standardoption beim Checkout

Das bedeutet, ein Gateway kann auf Ihrer Website aktiv, aber im POS deaktiviert sein – oder umgekehrt. Vollständige Details finden Sie unter [Checkout-Einstellungen](/settings/wp-admin/checkout).

## Zahlungen testen {#testing-payments}

Bevor Sie ein neues Gateway in den Live-Betrieb nehmen:

1. **Bargeld-Gateway** – immer verfügbar, um den Checkout-Ablauf ohne echte Transaktionen zu testen.
2. **Stripe Terminal** – Stripe stellt für den Testmodus einen [simulierten Reader](https://docs.stripe.com/terminal/testing) bereit. Aktivieren Sie zunächst den Testmodus in Ihrem Stripe-Dashboard.
3. **Drittanbieter-Gateways** – Anweisungen zum Sandbox- bzw. Testmodus finden Sie in der Dokumentation des jeweiligen Gateways. Die meisten WooCommerce-Zahlungsgateways unterstützen einen Schalter für den Testmodus.

:::tip
Wird ein neu aktiviertes Gateway beim Checkout nicht angezeigt, aktualisieren Sie den POS. Gateway-Änderungen im WP Admin werden beim nächsten Laden der Seite wirksam.
:::

## Häufige Probleme {#common-issues}

### Gateway erscheint nicht im Checkout {#gateway-not-appearing-at-checkout}

- Stellen Sie sicher, dass das Gateway für den POS aktiviert ist unter `WP Admin > POS > Settings > Checkout`.
- Prüfen Sie, ob das Gateway auch aktiviert ist unter `WP Admin > WooCommerce > Settings > Payments` — ein Gateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt werden kann.
- Aktualisieren Sie den POS, nachdem Sie Änderungen vorgenommen haben.

### Gateway wird fehlerhaft angezeigt {#gateway-displays-incorrectly}

Der Checkout-iframe lädt die Theme-Styles Ihrer Website, was zu Konflikten führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Styles oder Skripte gezielt zu deaktivieren. Weitere Informationen finden Sie unter [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting).

### Zahlungsabwicklung schlägt fehl {#payment-processing-fails}

- Prüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) für gateway-spezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist – die meisten Zahlungsgateways erfordern HTTPS.
- Wenn Sie eine Staging- oder lokale Umgebung verwenden, prüfen Sie, ob das Gateway den Test-/Sandbox-Modus unterstützt.
