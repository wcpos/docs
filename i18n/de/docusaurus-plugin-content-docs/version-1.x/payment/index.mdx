WCPOS unterstützt mehrere Zahlungsmethoden, um unterschiedliche Geschäftsanforderungen zu erfüllen. Das Checkout nutzt ein iframe/webview zur Anzeige der WooCommerce Order Pay Seite, was bedeutet, dass jedes Payment-Gateway, das mit WooCommerce funktioniert, auch im POS funktioniert.

## Standard-Zahlungsmethoden {#default-payment-methods}

Die kostenlose Version enthält zwei integrierte Payment-Gateways:

### Bargeld {#cash}

- Gezahlten Betrag eingeben
- Automatische Wechselgeldberechnung
- Keine zusätzliche Konfiguration erforderlich

### Karte {#card}

- Zur Verwendung mit externen Kartenterminals
- Markieren Sie die Zahlung nach der Verarbeitung auf Ihrem Terminal einfach als abgeschlossen
- Keine direkte Integration erforderlich

## Zusätzliche Zahlungsgateways (Pro) {#additional-payment-gateways-pro}

:::info Pro-Funktion
Zusätzliche Zahlungsgateways erfordern [WCPOS Pro](/getting-started/pro-license).
:::

Mit Pro können Sie zusätzliche WooCommerce-Zahlungsgateways im POS-Checkout aktivieren:

- **Stripe Terminal** - Direkte Integration mit Stripe-Kartenlesegeräten
- **SumUp Terminal** - Integration mit SumUp-Kartenlesegeräten
- **Any WooCommerce Gateway** - Aktivieren Sie jedes Gateway, das in Ihrem Shop installiert ist

## Aktivieren und Deaktivieren von Gateways {#enabling-and-disabling-gateways}

Zahlungsgateways werden im WordPress-Admin verwaltet:

1. Navigieren Sie zu `WP Admin > POS > Settings > Checkout`
2. Sie sehen eine Liste aller installierten WooCommerce-Zahlungsgateways
3. Aktivieren oder deaktivieren Sie jedes Gateway für das POS unabhängig von Ihrem Online-Shop
4. Ziehen Sie zum Neuordnen — das erste aktivierte Gateway wird zur Standardoption an der Kasse

Das bedeutet, dass ein Gateway auf Ihrer Website aktiv sein kann, aber im POS deaktiviert ist, oder umgekehrt. Weitere Details finden Sie unter [Checkout Settings](/settings/wp-admin/checkout).

## Zahlungen testen {#testing-payments}

Bevor Sie mit einem neuen Gateway live gehen:

1. **Cash Gateway** — immer verfügbar zum Testen des Checkout-Ablaufs ohne echte Transaktionen.
2. **Stripe Terminal** — Stripe stellt einen [simulierten Reader](https://docs.stripe.com/terminal/testing) für den Testmodus bereit. Aktivieren Sie zuerst den Testmodus in Ihrem Stripe-Dashboard.
3. **Gateways von Drittanbietern** — überprüfen Sie die Dokumentation des Gateways auf Anweisungen zum Sandbox-/Testmodus. Die meisten WooCommerce-Gateways unterstützen einen Testmodus-Schalter.

:::tip
Wenn ein neu aktiviertes Gateway an der Kasse nicht angezeigt wird, versuchen Sie, das POS zu aktualisieren. Gateway-Änderungen in WP Admin werden beim nächsten Seitenladevorgang wirksam.
:::

## Häufige Probleme {#common-issues}

### Gateway wird nicht an der Kasse angezeigt {#gateway-not-appearing-at-checkout}

- Bestätigen Sie, dass das Gateway für POS in aktiviert ist `WP Admin > POS > Settings > Checkout`.
- Überprüfen Sie, dass das Gateway auch in aktiviert ist `WP Admin > WooCommerce > Settings > Payments` — Ein Gateway muss in WooCommerce aktiv sein, bevor es im POS angezeigt werden kann.
- Aktualisieren Sie das POS nach Änderungen.

### Gateway wird falsch angezeigt {#gateway-displays-incorrectly}

Der Checkout-iframe lädt die Theme-Stile Ihrer Website, was manchmal zu Störungen führen kann. Verwenden Sie die Schaltfläche **Checkout-Einstellungen** im Checkout-Modal, um Stile oder Skripte selektiv zu deaktivieren. Siehe [Checkout-Fehlerbehebung](/pos/checkout#checkout-settings-troubleshooting) für weitere Informationen.

### Zahlungsverarbeitung schlägt fehl {#payment-processing-fails}

- Überprüfen Sie die Fehlerprotokolle Ihrer Website (`WP Admin > POS > Support > Logs`) für gatewayspezifische Fehler.
- Stellen Sie sicher, dass Ihr SSL-Zertifikat gültig ist – die meisten Zahlungsgateways erfordern HTTPS.
- Bestätigen Sie bei Verwendung einer Staging- oder lokalen Umgebung, dass das Gateway den Test-/Sandbox-Modus unterstützt.
