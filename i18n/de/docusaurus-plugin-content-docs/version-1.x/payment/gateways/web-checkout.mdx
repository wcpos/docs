# Web-Checkout-Gateway {#web-checkout-gateway}

Das Web-Checkout-Gateway ermöglicht es Kunden, ihre POS-Transaktionen über das Checkout-System Ihres Webshops abzuschließen. Dieses Gateway schafft eine nahtlose Verbindung zwischen Ihrem Vor-Ort-POS und Online-Zahlungsmethoden. Es eignet sich besonders für Situationen, in denen Kunden lieber mit ihren eigenen Geräten bezahlen oder Zugriff auf nur im Web verfügbare Zahlungsmethoden benötigt wird.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Herunterladen und installieren">

1. Besuchen Sie die [Release-Seite des Web-Checkout-Gateways](https://github.com/wcpos/web-checkout-gateway/releases)
2. Laden Sie die neueste Datei **woocommerce-pos-web-checkout-gateway.zip** herunter
3. Gehen Sie in Ihrer WordPress-Administration zu `Plugins > Neu hinzufügen > Plugin hochladen`
4. Laden Sie die ZIP-Datei hoch und aktivieren Sie das Plugin

  </StepCard>
  <StepCard number="2" title="In WCPOS aktivieren" isLast>

1. Gehen Sie zu `WP Admin > POS > Einstellungen > Checkout`
2. Suchen Sie das **WCPOS Web-Checkout-Gateway** in der Liste
3. Aktivieren Sie das Gateway zur Verwendung im POS
4. Speichern Sie Ihre Einstellungen

:::note
Dieses Zahlungsgateway nutzt Ihre vorhandenen WooCommerce-Zahlungsmethoden. Stellen Sie daher sicher, dass mindestens ein Web-Zahlungsgateway korrekt konfiguriert ist (Stripe, PayPal usw.).
:::

  </StepCard>
</Steps>

## Nutzung {#usage}

### Web-Checkout starten {#initiating-web-checkout}

1. **Artikel hinzufügen**: Produkte zum Warenkorb im POS hinzufügen
2. **Gateway auswählen**: „Web-Checkout“ als Zahlungsmethode wählen
3. **Link generieren**: Das System erstellt einen eindeutigen Checkout-Link für die Bestellung
4. **Kundenzugriff**: Der Kunde klickt auf den Link, um den Web-Checkout aufzurufen
5. **Online-Zahlung**: Der Kunde schließt die Zahlung mit den Zahlungsmethoden des Webshops ab

### Zahlungsablauf {#payment-process}

Das Web-Checkout-Gateway folgt diesem Ablauf:

1. **Bestellungserstellung**: POS erstellt eine Bestellung mit Status „Zahlung ausstehend“ in WooCommerce
2. **Checkout-Link**: Das System generiert einen sicheren, eindeutigen Zahlungslink
3. **Kundenweiterleitung**: Der Kunde wird zur Web-Checkout-Seite weitergeleitet
4. **Zahlungsauswahl**: Der Kunde wählt aus den verfügbaren Web-Zahlungsmethoden
5. **Zahlungsverarbeitung**: Der Standard-Checkout-Prozess von WooCommerce verarbeitet die Zahlung
6. **Bestellabschluss**: Der Bestellstatus wird nach erfolgreicher Zahlung automatisch aktualisiert

### POS-Workflow {#pos-workflow}

Nach dem Start des Web-Checkouts:

1. **Status überwachen**: Lassen Sie den POS-Bestellbildschirm geöffnet, um den Zahlungsstatus zu überwachen
2. **Kundenzahlung**: Der Kunde schließt die Zahlung auf seinem Gerät oder Ihrem Tablet ab
3. **Automatische Aktualisierung**: Der Bestellstatus wird in Echtzeit aktualisiert, sobald die Zahlung eingegangen ist
4. **Zahlung verarbeiten**: Klicken Sie im POS auf die Schaltfläche "Zahlung verarbeiten", um mit dem Beleg fortzufahren
5. **Belegerstellung**: Beleg wie gewohnt erstellen und drucken

## Technische Details {#technical-details}

### Bestellverwaltung {#order-management}

- **Bestellungen mit ausstehender Zahlung**: Bestellungen werden mit dem Status "Zahlung ausstehend" erstellt
- **Statusaktualisierungen**: Automatische Statusänderungen, wenn die Zahlung eingegangen ist
- **Bestellnotizen**: Zahlungsdetails werden in den Bestellnotizen erfasst
- **Bestand**: Der Bestand wird während des Checkout-Vorgangs reserviert

### Sicherheit {#security}

- **Eindeutige Links**: Jede Checkout-Sitzung verfügt über eine eindeutige, zeitlich begrenzte URL
- **SSL erforderlich**: Die gesamte Zahlungsabwicklung erfolgt über sichere HTTPS-Verbindungen
- **PCI-Konformität**: Nutzt Ihre vorhandenen PCI-konformen Web-Zahlungsgateways
- **Sitzungsverwaltung**: Eine sichere Sitzungsverwaltung verhindert unbefugten Zugriff

### Zahlungsmethoden {#payment-methods}

Das Gateway bietet Zugriff auf alle Ihre konfigurierten WooCommerce-Zahlungsmethoden:

- **Kredit-/Debitkarten**: Stripe, Square, Authorize.net usw.
- **Digitale Wallets**: PayPal, Apple Pay, Google Pay
- **Banküberweisungen**: Direkte Bankzahlungsmethoden
- **Jetzt kaufen, später bezahlen**: Klarna, Afterpay, Sezzle
- **Kryptowährung**: Bitcoin und andere Krypto-Zahlungsgateways
- **Lokale Methoden**: Regionalspezifische Zahlungsmethoden

## Fehlerbehebung {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Kassenlink funktioniert nicht">

  - Prüfen Sie, ob das SSL-Zertifikat ordnungsgemäß installiert ist
  - Prüfen Sie, ob die WooCommerce-Permalinks korrekt konfiguriert sind
  - Stellen Sie sicher, dass Web-Zahlungsgateways aktiv und konfiguriert sind
  - Testen Sie es mit einem anderen Browser oder Gerät

  </AccordionItem>

  <AccordionItem question="Zahlung wird nicht verarbeitet">

  - Bestätigen Sie, dass Web-Zahlungsgateways korrekt konfiguriert sind
  - Zahlungsgateway-Protokolle auf Fehlermeldungen prüfen
  - Überprüfen, ob die Zahlungsmethode des Kunden unterstützt wird
  - Den Web-Checkout-Prozess unabhängig testen

  </AccordionItem>

  <AccordionItem question="Bestellung wird im POS nicht aktualisiert">

  - Stabilität der Internetverbindung prüfen
  - Überprüfen, ob WordPress-Cronjobs ordnungsgemäß ausgeführt werden
  - Bestellansicht im POS aktualisieren
  - Bestellstatus in der WooCommerce-Administration prüfen

  </AccordionItem>

  <AccordionItem question="Kunde kann nicht auf den Checkout zugreifen">

  - Sicherstellen, dass der Checkout-Link nicht abgelaufen ist
  - Überprüfen, ob das Gerät des Kunden über eine Internetverbindung verfügt
  - Prüfen, ob die Bestellung nicht storniert oder abgeschlossen wurde
  - Den Link auf einem anderen Gerät testen

  </AccordionItem>
</Accordion>

### Leistungsoptimierung {#performance-optimisation}

Für optimale Leistung:

- **Caching**: Caching-Plugins so konfigurieren, dass Checkout-Seiten ausgeschlossen werden
- **CDN**: Sicherstellen, dass CDN-Einstellungen den Checkout-Prozess nicht beeinträchtigen
- **Datenbank**: Datenbank für eine schnellere Bestellverarbeitung optimieren
- **Hosting**: Zuverlässiges Hosting mit guter Verfügbarkeit verwenden
