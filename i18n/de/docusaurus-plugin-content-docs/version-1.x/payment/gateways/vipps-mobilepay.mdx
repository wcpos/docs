# Vipps MobilePay Gateway

Das Vipps MobilePay-Gateway ermöglicht es Ihnen, telefonbasierte Zahlungen direkt innerhalb von WCPOS zu akzeptieren. Kunden bezahlen, indem sie einen QR-Code auf dem Bildschirm scannen oder eine Push-Benachrichtigung auf ihrem Smartphone bestätigen – keine Kartenlesehardware erforderlich.

## Funktionen {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="QR-Code-Zahlungen">
    Zeigen Sie einen QR-Code beim Checkout an, den Kunden scannen und mit ihrem Telefon bezahlen können.
  </FeatureCard>
  <FeatureCard icon="bolt" title="Push-Benachrichtigungen">
    Senden Sie eine Zahlungsanforderung direkt an das Telefon des Kunden, indem Sie dessen Nummer eingeben.
  </FeatureCard>
  <FeatureCard icon="globe" title="Nordische Abdeckung">
    Funktioniert mit Vipps in Norwegen, MobilePay in Dänemark und Finnland.
  </FeatureCard>
  <FeatureCard icon="credit-card" title="Automatische Erfassung">
    Erfassen Sie Gelder sofort nach der Genehmigung oder reservieren Sie diese für eine manuelle Erfassung.
  </FeatureCard>
</FeatureGrid>

## Installation

<Steps>
  <StepCard number="1" title="Herunterladen und Installieren">

1. Besuchen Sie die [WCPOS Vipps MobilePay Releases-Seite](https://github.com/wcpos/wcpos-vipps/releases)
2. Laden Sie die neueste **wcpos-vipps.zip**-Datei herunter.
3. Gehen Sie in Ihrem WordPress-Admin zu `Plugins > Neu hinzufügen > Plugin hochladen`.
4. Laden Sie die Zip-Datei hoch und aktivieren Sie das Plugin.

  </StepCard>
  <StepCard number="2" title="Vipps-Anmeldeinformationen konfigurieren">

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`.
2. Suchen Sie **WCPOS Vipps MobilePay** in der Liste der Zahlungsmethoden.
3. Klicken Sie auf **WCPOS Vipps MobilePay**, um die Einstellungen zuzugreifen.
4. Geben Sie Ihre Anmeldeinformationen aus dem [Vipps-Portal](https://portal.vipps.no/) ein:
   - **Händler-Seriennummer (MSN)**
   - **Client-ID**
   - **Client Secret**
   - **Abonnementschlüssel**
5. Speichern Sie die Einstellungen.

:::note
Sie müssen das Gateway hier nicht für die POS-Nutzung aktivieren - es wird im nächsten Schritt speziell für das POS aktiviert. Wenn Sie es in den WooCommerce-Einstellungen aktivieren, ist es auch an Ihrer Online-Store-Checkout verfügbar, was nützlich sein kann, um Tests durchzuführen.
:::

:::tip Verwenden Sie bereits das Vipps-Plugin?
Wenn das offizielle [Checkout mit Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/) Plugin installiert ist, werden Ihre Anmeldeinformationen automatisch importiert, wenn Sie dieses Plugin aktivieren. Sie können die manuelle Eingabe der Anmeldeinformationen überspringen.
:::

  </StepCard>
  <StepCard number="3" title="Im WCPOS aktivieren" isLast>

1. Gehen Sie zu `WP Admin > POS > Einstellungen > Checkout`.
2. Suchen Sie das **WCPOS Vipps MobilePay**-Gateway in der Liste.
3. Aktivieren Sie das Gateway für die Nutzung im POS.
4. Speichern Sie Ihre Einstellungen.

  </StepCard>
</Steps>

## Verwendung {#usage}

### Zahlungen verarbeiten – QR-Code {#processing-payments-qr-code}

1. **Artikel hinzufügen**: Fügen Sie Produkte zu Ihrem Warenkorb im POS hinzu.
2. **Gateway auswählen**: Wählen Sie "Vipps MobilePay" als Zahlungsmethode.
3. **QR-Code generieren**: Klicken Sie auf die Schaltfläche "QR-Code generieren".
4. **Kunde scannt**: Der Kunde scannt den QR-Code mit seiner Vipps oder MobilePay-App.
5. **Kunde bestätigt**: Der Kunde bestätigt die Zahlung in seiner App.
6. **Automatische Vollziehung**: Die Bestellung wird automatisch abgeschlossen, sobald die Zahlung genehmigt ist.

### Zahlungen verarbeiten – An Telefon senden {#processing-payments-send-to-phone}

1. **Artikel hinzufügen**: Fügen Sie Produkte zu Ihrem Warenkorb im POS hinzu.
2. **Gateway auswählen**: Wählen Sie "Vipps MobilePay" als Zahlungsmethode.
3. **Telefonnummer eingeben**: Geben Sie die Telefonnummer des Kunden ein.
4. **An Telefon senden**: Klicken Sie auf die Schaltfläche "An Telefon senden".
5. **Kunde bestätigt**: Der Kunde erhält eine Push-Benachrichtigung oder wird zur Vipps-Startseite geleitet, um die Zahlung in seiner App zu bestätigen.
6. **Automatische Vollziehung**: Die Bestellung wird automatisch abgeschlossen, sobald die Zahlung genehmigt ist.

Das Plugin erkennt automatisch die beste Methode, um die Zahlungsanforderung an das Telefon des Kunden zu senden:

- **Direkte Push-Benachrichtigung** (bevorzugt) - Sendet eine Benachrichtigung direkt an die Vipps-App des Kunden. Dies ist die schnellste Erfahrung, erfordert jedoch, dass Vipps `PUSH_MESSAGE` auf Ihrem Verkaufsgerät aktiviert.
- **Fallback-Seite** - Wenn der direkte Push nicht aktiviert ist, öffnet das Plugin die Vipps-Startseite in einem neuen Browser-Tab. Die Startseite kümmert sich um das Senden der Benachrichtigung. Dies funktioniert sofort ohne spezielle Genehmigung.

Das erste Mal, wenn das Plugin erkennt, dass der direkte Push nicht verfügbar ist, sehen Sie eine kurze Nachricht, die Sie auffordert, erneut auf "An Telefon senden" zu klicken. Danach merkt es sich das Ergebnis und funktioniert nahtlos.

### Direkten Push aktivieren {#enabling-direct-push}

Der direkte Push-Flow bietet die beste Erfahrung für Telefonzahlungen – keine zusätzlichen Tabs, keine Zielseiten. Um es zu aktivieren:

1. Melden Sie sich bei [portal.vippsmobilepay.com](https://portal.vippsmobilepay.com) an.
2. Kontaktieren Sie Ihren Vipps-Key-Account-Manager, Partner-Manager oder den Kundenservice.
3. Sagen Sie ihnen: **"Ich benötige PUSH_MESSAGE, um es mit einer POS-Integration auf meiner MSN zu aktivieren."**

Sobald genehmigt, erkennt das Plugin die Änderung automatisch innerhalb von 24 Stunden und wechselt zum direkten Push-Flow. Sie sehen eine Erinnerungsbenachrichtigung auf der Gateway-Einstellungsseite, während Sie die Fallback-Seite verwenden.

### Zahlung stornieren {#cancelling-a-payment}

Während Sie warten, dass der Kunde bestätigt, können Sie auf die Schaltfläche **Zahlung stornieren** klicken, um die Transaktion abzubrechen. Dies storniert die ausstehende Zahlung auf der Vipps-Seite und setzt die Checkout-Oberfläche zurück.

### Rückerstattungen {#refunds}

Rückerstattungen werden über den normalen WooCommerce-Rückerstattungsprozess abgewickelt. Öffnen Sie die Bestellung, klicken Sie auf **Rückerstattung**, geben Sie den Betrag ein, und die Rückerstattung wird automatisch über die Vipps-API verarbeitet.

## Unterstützte Märkte {#supported-markets}

Vipps MobilePay ist in der nordischen Region unter zwei Marken tätig:

| Region | Marke | Währungen |
|--------|-------|------------|
| Norwegen | Vipps | NOK |
| Dänemark, Finnland | MobilePay | DKK, EUR |

Ihr Händlerkonto bestimmt, welche Märkte und Währungen verfügbar sind. Kunden in jedem unterstützten Markt können mit ihrer lokalen Vipps- oder MobilePay-App bezahlen.

## Fehlersuche {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Zahlung wird nicht abgeschlossen">

  - Der Checkout verfolgt bis zu 5 Minuten - wenn der Kunde innerhalb dieses Zeitraums nicht bestätigt, läuft die Zahlung ab.
  - Überprüfen Sie, dass Sie Testmodus und Produktionsanmeldeinformationen nicht verwechseln.
  - Stellen Sie sicher, dass Ihre Internetverbindung sowohl auf dem POS-Gerät als auch auf dem Telefon des Kunden stabil ist.

  </AccordionItem>

  <AccordionItem question="Fehler bei ungültigen Anmeldeinformationen">

  - Überprüfen Sie alle vier Anmeldefeld (MSN, Client-ID, Client Secret, Abonnementschlüssel) sorgfältig.
  - Stellen Sie sicher, dass Sie Testanmeldeinformationen im Testmodus verwenden und Produktionsanmeldeinformationen, wenn dieser deaktiviert ist.
  - Überprüfen Sie Ihre Anmeldeinformationen im [Vipps-Portal](https://portal.vipps.no/).

  </AccordionItem>

  <AccordionItem question="'An Telefon senden' öffnet einen neuen Tab, anstatt direkt zu senden">

  Das bedeutet, dass Ihr Vipps-Konto `PUSH_MESSAGE` noch nicht aktiviert hat. Das Plugin verwendet die Fallback-Seite, die funktioniert, aber einen zusätzlichen Schritt hinzufügt. Um die reibungslosere direkte Push-Erfahrung zu erhalten, kontaktieren Sie Vipps und bitten Sie sie, PUSH_MESSAGE auf Ihrem Verkaufsgerät (MSN) zu aktivieren. Siehe [Aktivierung des direkten Pushs](#enabling-direct-push) oben.

  </AccordionItem>

  <AccordionItem question="QR-Code wird nicht generiert">

  - Bestätigen Sie, dass Ihre Website über HTTPS läuft - die Vipps-API lehnt Anfragen von HTTP-Websites ab.
  - Überprüfen Sie die WooCommerce-Protokolle (`WooCommerce > Status > Protokolle`) auf API-Fehlerdetails.
  - Überprüfen Sie, ob Ihr Händlerkonto aktiv ist und die ePayment-API aktiviert hat.

  </AccordionItem>
</Accordion>
