# SumUp Terminal Zahlungsgateway {#sumup-terminal-gateway}

Das SumUp Terminal Zahlungsgateway ermöglicht die Annahme von Kartenzahlungen mit SumUp Kartenlesegeräten direkt in WCPOS. Dieses Gateway bietet eine nahtlose Integration mit dem Zahlungsverarbeitungssystem von SumUp für Vor-Ort-Transaktionen.

## Funktionen {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Hardware-Integration">
    SumUp Kartenlesegeräte über eine Internetverbindung mit dem POS-System verbinden
  </FeatureCard>
  <FeatureCard icon="bolt" title="Echtzeitverarbeitung">
    Sofortige Zahlungsverarbeitung und Bestellabschluss
  </FeatureCard>
  <FeatureCard icon="shield" title="Sichere Transaktionen">
    PCI-konforme Zahlungsverarbeitung über SumUp
  </FeatureCard>
  <FeatureCard icon="link" title="Einfache Kopplung">
    Einfacher Geräte-Kopplungsprozess mit Kopplungscodes
  </FeatureCard>
</FeatureGrid>

## Installation {#installation}

<Steps>
  <StepCard number="1" title="SumUp Terminal für WooCommerce installieren">

Installieren Sie es über `WP Admin > POS > Einstellungen > Erweiterungen`, oder laden Sie das neueste **Plugin-Zip-Asset** (nicht die GitHub-Zipdatei oder das Tarball mit dem Quellcode) von der [GitHub-Releases-Seite](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) herunter und laden Sie es über `Plugins > Neu hinzufügen > Plugin hochladen` hoch.

  </StepCard>
  <StepCard number="2" title="SumUp-Einstellungen konfigurieren">

1. Navigieren Sie zu `WP Admin > WooCommerce > Einstellungen > Zahlungen`
2. Suchen Sie **SumUp Terminal** in der Liste der Zahlungsmethoden
3. Klicken Sie auf **SumUp Terminal**, um die Einstellungen zu öffnen
4. Geben Sie Ihren **SumUp API-Schlüssel** ein (verfügbar in Ihrem SumUp-Händler-Dashboard)
5. Speichern Sie die Einstellungen

:::note
Sie müssen das SumUp Terminal-Zahlungsgateway in den WooCommerce-Einstellungen nicht aktivieren. Es wird in einem späteren Schritt speziell für den POS aktiviert.
:::

  </StepCard>
  <StepCard number="3" title="SumUp Terminal koppeln">

1. Suchen Sie auf derselben Einstellungsseite den Abschnitt **Lesegerät koppeln**
2. Öffnen Sie auf Ihrem SumUp Gerät den Kopplungsbildschirm, um den Kopplungscode anzuzeigen
3. Geben Sie den auf Ihrem SumUp Gerät angezeigten Kopplungscode ein
4. Klicken Sie auf **"Lesegerät koppeln"**, um die Verbindung herzustellen
5. Warten Sie auf die Bestätigung, dass das Lesegerät erfolgreich gekoppelt wurde

:::warning Wichtig
Das Lesegerät muss erfolgreich gekoppelt sein, bevor Zahlungen verarbeitet werden können. Stellen Sie sicher, dass der Kopplungsvorgang abgeschlossen ist, bevor Sie fortfahren.
:::

  </StepCard>
  <StepCard number="4" title="In WCPOS aktivieren" isLast>

1. Gehen Sie zu `WP Admin > POS > Einstellungen > Checkout`
2. Das **SumUp Terminal**-Gateway in der Liste suchen
3. Das Gateway für die Verwendung im POS aktivieren
4. Einstellungen speichern

  </StepCard>
</Steps>

## Nutzung {#usage}

### Zahlungen verarbeiten {#processing-payments}

1. **Artikel hinzufügen**: Produkte im POS zum Warenkorb hinzufügen
2. **Gateway auswählen**: "SumUp Terminal" als Zahlungsmethode auswählen
3. **Zahlung starten**: Klicken, um eine neue Zahlung auf dem SumUp-Gerät zu starten
4. **Kundenzahlung**: Der Kunde schließt die Zahlung am SumUp Terminal ab
5. **Automatischer Abschluss**: Die Bestellung wird automatisch abgeschlossen, sobald die Zahlung erkannt wurde

### Zahlungssteuerung {#payment-controls}

Bei Verwendung des SumUp Terminal-Gateways stehen die folgenden Optionen zur Verfügung:

- **Neue Zahlung starten**: Eine Zahlungsanforderung auf dem verbundenen Terminal starten
- **Zahlung abbrechen**: Eine derzeit laufende Zahlung abbrechen
- **Zahlungsstatus**: Den aktuellen Status des Zahlungsvorgangs überwachen

### Bestellverwaltung {#order-management}

- **Automatische Aktualisierungen**: Bestellungen werden nach erfolgreicher Zahlung automatisch als abgeschlossen markiert
- **Zahlungsverfolgung**: Alle Zahlungsdetails werden in den Bestellnotizen erfasst
- **Belegerstellung**: Nach erfolgreichen Zahlungen werden standardmäßige POS-Belege erstellt

## Hardware-Kompatibilität {#hardware-compatibility}

:::important Konnektivitätsanforderungen
Diese Implementierung verwendet das JavaScript-SDK von SumUp. Das bedeutet, dass sie über Webanwendungen funktioniert, aber **nur Terminals mit Internetverbindung** unterstützt. Bluetooth-Terminals werden derzeit nicht unterstützt.
:::

### Unterstützte Terminals (mit Internetverbindung) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Per WLAN/Ethernet verbundenes Terminal

### Nicht unterstützte Terminals (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Nur Bluetooth
- **SumUp 3G** ❌ - Nur Bluetooth

:::note Zukünftige Unterstützung
Die Unterstützung für Bluetooth-Terminals ist für eine zukünftige Version der iOS- und Android-App geplant. Sobald sie verfügbar ist, können alle von SumUp zertifizierten Terminals unterstützt werden, einschließlich der Air- und 3G-Modelle.
:::

## Fehlerbehebung {#troubleshooting}

### Häufige Probleme {#common-issues}

<Accordion>
  <AccordionItem question="Lesegerät lässt sich nicht koppeln">

  - Stellen Sie sicher, dass ein unterstütztes Terminal mit Internetverbindung verwendet wird (nur Solo)
  - Prüfen Sie, ob Ihr SumUp Solo Gerät mit dem WLAN verbunden und online ist
  - Prüfen Sie, ob sich das Gerät im Kopplungsmodus befindet und der Kopplungscode korrekt eingegeben wurde
  - Bestätigen Sie, dass für Ihr SumUp Konto der API-Zugriff aktiviert ist
  - Stellen Sie während der Kopplung eine stabile Internetverbindung sicher

  </AccordionItem>

  <AccordionItem question="Zahlung wird nicht verarbeitet">

  - Überprüfen Sie, ob das Lesegerät korrekt gekoppelt ist
  - Prüfen Sie, ob Ihr SumUp Konto aktiv ist und keine Einschränkungen aufweist
  - Stellen Sie sicher, dass das Gerät über ausreichend Akku und Konnektivität verfügt
  - Bestätigen Sie, dass der API-Schlüssel korrekt und aktiv ist

  </AccordionItem>

  <AccordionItem question="Bestellung wird nicht abgeschlossen">

  - Prüfen Sie die Stabilität der Internetverbindung
  - Überprüfen Sie die Webhook-Einstellungen in Ihrem SumUp Konto
  - Stellen Sie sicher, dass die Zahlung auf dem SumUp Terminal erfolgreich war
  - Prüfen Sie die WordPress Fehlerprotokolle auf Probleme bei der API-Kommunikation

  </AccordionItem>
</Accordion>

### API-Ratenlimits {#api-rate-limits}

Das Gateway ist darauf optimiert, API-Aufrufe zu reduzieren und Ratenbegrenzungen zu vermeiden:
- Der Zahlungsstatus wird effizient geprüft
- Unnötige API-Anfragen werden minimiert
- Eine automatische Wiederholungslogik verarbeitet vorübergehende Fehler
