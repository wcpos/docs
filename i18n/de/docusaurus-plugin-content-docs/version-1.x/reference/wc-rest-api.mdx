Die **WooCommerce REST API** ist wie ein Satz standardisierter "Kanäle", die Store-Besitzern ermöglichen, ihren WooCommerce-Store mit anderen Anwendungen und Diensten zu verbinden.

Hier eine einfache Erklärung, wie es funktioniert:

1. Sie dient als Brücke zwischen Ihrem WooCommerce-Store und externen Anwendungen und ermöglicht ihnen die Kommunikation und den Datenaustausch.

2. Die API funktioniert durch Senden von Anfragen an spezifische **Endpunkte** - denken Sie an diese als "Kanäle" für verschiedene Arten von Store-Daten. Es gibt zum Beispiel Endpunkte für Produkte, Bestellungen und Kunden.

3. Externe Anwendungen können diese Endpunkte nutzen, um:

- Informationen zu erhalten (wie eine Produktliste abrufen)
- Neue Daten hinzuzufügen (wie eine neue Bestellung erstellen)
- Bestehende Informationen zu aktualisieren (wie den Preis eines Produkts zu ändern)
- Daten zu löschen (wie ein veraltetes Produkt zu entfernen)

Anstatt sich auf PHP-Hooks und -Filter zu verlassen (wie viele traditionelle WordPress-Plugins), arbeitet sie durch Senden und Empfangen von Daten in einem strukturierten, vorhersehbaren Format, bekannt als **JSON**.

## Warum funktionieren einige Plugins nicht in der POS?

WCPOS lädt Produkt-, Bestell- undKundendaten aus Ihrem WooCommerce-Store über die REST API herunter.
Anschließend behält es eine lokale Kopie dieser Daten, damit es auch bei einer instabilen Internetverbindung weiterlaufen kann.
Dieser Ansatz macht die POS in einer geschäftigen Kasse schneller und zuverlässiger.

Allerdings bedeutet dies, dass unsere POS Änderungen, die dynamisch auf Ihrer Website in Echtzeit passieren, nicht "sieht". Zum Beispiel:

      die über die REST API bereitgestellt werden.
    </td>
  </tr>
