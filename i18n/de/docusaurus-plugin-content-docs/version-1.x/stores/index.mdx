:::info Pro-Funktion
Die Funktion für mehrere Filialen erfordert [WCPOS Pro](/getting-started/pro-license).
:::

Mit WCPOS Pro lassen sich separate Filialen innerhalb einer einzigen WooCommerce-Installation anlegen. Jede Filiale kann eine eigene Adresse, ein eigenes Logo und eigene Einstellungen haben – nützlich beim Betrieb mehrerer physischer Standorte.

## Was Filialen bieten {#what-stores-provide}

Jede Filiale kann Folgendes haben:

- **Filialname und -adresse** – wird auf Belegen und in Berichten verwendet
- **Filiallogo** – wird auf gedruckten Belegen angezeigt
- **Steuersätze** – unterschiedliche Standorte können unterschiedliche steuerliche Anforderungen haben
- **Kassiererzuweisung** — verfolgen Sie, welche Kassierer in welcher Filiale arbeiten

## Filialen einrichten {#setting-up-stores}

1. Navigieren Sie zu `WP Admin > POS > Settings`
2. Erstellen Sie eine neue Filiale mit Name, Adresse und Logo
3. Konfigurieren Sie filialenspezifische Steuereinstellungen
4. Weisen Sie die Filiale POS-Benutzern zu

Wenn sich ein Kassierer im POS anmeldet, wählt er aus, von welcher Filiale aus er arbeitet. Dies bestimmt die Filialinformationen, die auf Belegen erscheinen, sowie den Filialfilter, der auf Berichte angewendet wird.

## Nach Filiale filtern {#filtering-by-store}

Sobald mehrere Filialen konfiguriert sind, erscheint der Filialfilter im gesamten POS:

- **[Bestellungen](/orders)** — Bestellverlauf nach Filialstandort filtern
- **[Berichte](/reports)** — Tagesabschlussberichte für eine bestimmte Filiale erstellen
- **[Belege](/pos/checkout/receipts)** — Belege zeigen automatisch die korrekte Filialadresse und das Logo an

## Filialspezifische Preise {#store-specific-pricing}

WCPOS Pro unterstützt **unterschiedliche Preise für verschiedene Filialstandorte**. Sie können sowohl für Produkte als auch für Variationen einen separaten regulären Preis und Angebotspreis pro Filiale festlegen.

Wenn ein Kassierer von einer bestimmten Filiale aus arbeitet, verwendet der POS automatisch die Preise dieser Filiale. Falls kein filialspezifischer Preis festgelegt ist, wird der Standardpreis von WooCommerce verwendet.

## Bestandsverwaltung {#inventory-management}

Standardmäßig teilen sich alle Filialen dieselben WooCommerce-Lagerbestände. Für eine standortbezogene Bestandsverwaltung lässt sich WCPOS Pro mit dem Plugin [ATUM Inventory Management](https://www.atum.org/) integrieren:

1. Installieren und konfigurieren Sie das ATUM-Plugin auf Ihrer WooCommerce-Website
2. Erstellen Sie Lagerstandorte in ATUM
3. Verknüpfen Sie in WCPOS Pro jede Filiale mit einem ATUM-Lagerstandort
4. Die Lagerbestände werden beim Aufgeben von Bestellungen für den jeweils richtigen Standort aktualisiert

### Alternative: WordPress Multisite {#alternative-wordpress-multisite}

Für vollständig getrennte Lagerbestände ohne ATUM können Sie [WordPress Multisite](https://developer.wordpress.org/advanced-administration/multisite/create-network/) verwenden:

1. Erstellen Sie für jeden Filialstandort eine separate WordPress-Website
2. WooCommerce und WCPOS auf jeder Website installieren
3. Jede Website verwaltet ihren eigenen unabhängigen Produktkatalog und eigene Lagerbestände

Dies bietet eine vollständige Trennung, bedeutet jedoch die Verwaltung mehrerer WooCommerce-Installationen.

## Bekannte Einschränkungen {#known-limitations}

- **Einzelne WooCommerce-Instanz** — Filialen sind Standorte innerhalb einer WooCommerce-Installation, keine separaten Shops
- **Keine filialspezifische Produktsichtbarkeit** — alle Filialen sehen denselben Produktkatalog
