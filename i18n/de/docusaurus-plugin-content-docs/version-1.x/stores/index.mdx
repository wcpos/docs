:::info Pro-Funktion
Die Multi-Filial-Funktionalität erfordert [WCPOS Pro](/getting-started/pro-license).
:::

Mit WCPOS Pro lassen sich separate Filialstandorte innerhalb einer einzelnen WooCommerce-Installation erstellen. Jede Filiale kann eine eigene Adresse, ein eigenes Logo und eigene Einstellungen haben — nützlich bei Betrieb an mehreren physischen Standorten.

## Was Filialen bieten {#what-stores-provide}

Jede Filiale kann Folgendes haben:

- **Filialname und Adresse** — wird auf Quittungen und Berichten verwendet
- **Filiallogo** — wird auf gedruckten Quittungen angezeigt
- **Steuersätze** — verschiedene Standorte können unterschiedliche steuerliche Anforderungen haben
- **Kassiererzuordnung** — verfolgen, welche Kassierer in welcher Filiale arbeiten

## Filialen einrichten {#setting-up-stores}

Durch die Aktivierung von Pro wird ein Untermenü **Filialen** unter POS in WP Admin hinzugefügt. Dort können Sie jede Filiale anlegen, deren Adresse, Logo, Steuer-IDs und die zugelassenen Kassierer festlegen — die vollständige Schritt-für-Schritt-Anleitung finden Sie unter **[Filialen einrichten](/stores/setup)**.

Wenn sich ein Kassierer im POS anmeldet, wählt er die Filiale aus, von der aus er arbeitet. Dies bestimmt die Filialdetails, die auf Belegen angezeigt werden, und welcher Filialfilter auf Berichte angewendet wird.

## Nach Filiale filtern {#filtering-by-store}

Sobald mehrere Filialen konfiguriert sind, erscheint der Filialfilter im gesamten POS:

- **[Bestellungen](/orders)** — Bestellverlauf nach Filialstandort filtern
- **[Berichte](/reports)** — Tagesabschlussberichte für eine bestimmte Filiale erstellen
- **[Belege](/receipts/at-checkout)** — Belege zeigen automatisch die korrekte Filialadresse und das Logo an

## Filialspezifische Preise {#store-specific-pricing}

WCPOS Pro unterstützt **unterschiedliche Preise für verschiedene Filialstandorte**. Für Produkte und Varianten kann jeweils ein separater regulärer Preis und Aktionspreis pro Filiale festgelegt werden.

Wenn ein Kassierer von einer bestimmten Filiale aus arbeitet, verwendet das POS automatisch die Preise dieser Filiale. Ist kein filialspezifischer Preis festgelegt, wird der standardmäßige WooCommerce-Preis verwendet.

## Bestandsverwaltung {#inventory-management}

Standardmäßig teilen sich alle Filialen dieselben WooCommerce-Lagerbestände. Für eine standortbezogene Bestandsverfolgung lässt sich WCPOS Pro mit dem Plugin [ATUM Inventory Management](https://www.atum.org/) integrieren:

1. Das ATUM-Plugin auf der WooCommerce-Website installieren und konfigurieren
2. Lagerstandorte in ATUM anlegen
3. In WCPOS Pro jede Filiale einem ATUM-Lagerstandort zuordnen
4. Die Lagerbestände werden beim Aufgeben von Bestellungen für den richtigen Standort aktualisiert

### Alternative: WordPress Multisite {#alternative-wordpress-multisite}

Für vollständig getrennte Bestände ohne ATUM kann [WordPress Multisite](https://developer.wordpress.org/advanced-administration/multisite/create-network/) verwendet werden:

1. Für jeden Filialstandort eine separate WordPress-Website erstellen
2. WooCommerce und WCPOS auf jeder Website installieren
3. Jede Website verwaltet einen eigenen, unabhängigen Produktkatalog und eigene Lagerbestände

Dies bietet eine vollständige Trennung, bedeutet jedoch die Verwaltung mehrerer WooCommerce-Installationen.

## Bekannte Einschränkungen {#known-limitations}

- **Einzelne WooCommerce-Instanz** — Filialen sind Standorte innerhalb einer WooCommerce-Installation, keine separaten Shops
- **Keine Produktsichtbarkeit pro Filiale** — alle Filialen sehen denselben Produktkatalog
