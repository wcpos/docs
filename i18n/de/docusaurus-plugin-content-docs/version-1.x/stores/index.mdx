:::info Pro-Funktion
Die Multi-Filialen-Funktionalität erfordert [WCPOS Pro](/getting-started/pro-license).
:::

Mit WCPOS Pro können Sie separate Filialstandorte innerhalb einer einzelnen WooCommerce-Installation erstellen. Jede Filiale kann ihre eigene Adresse, ihr eigenes Logo und eigene Einstellungen haben — nützlich, wenn Sie mehrere physische Standorte betreiben.

## Was Filialen bieten {#what-stores-provide}

Jede Filiale kann Folgendes haben:

- **Name und Adresse der Filiale** — werden auf Belegen und in Berichten verwendet
- **Filiallogo** — wird auf gedruckten Belegen angezeigt
- **Steuersätze** — verschiedene Standorte können unterschiedliche steuerliche Anforderungen haben
- **Kassiererzuweisung** — nachverfolgen, welche Kassierer in welcher Filiale arbeiten

## Filialen einrichten {#setting-up-stores}

Durch die Aktivierung von Pro wird unter POS in WP Admin ein Untermenü **Filialen** hinzugefügt. Dort erstellen Sie jede Filiale, legen Adresse, Logo, Steuernummern und die Kassierer fest, die sich anmelden dürfen — die vollständige Schritt-für-Schritt-Anleitung finden Sie unter **[Filialen einrichten](/stores/setup)**.

Wenn sich ein Kassierer im POS anmeldet, wählt er aus, in welcher Filiale er arbeitet. Dadurch wird festgelegt, welche Filialdaten auf Belegen erscheinen und welcher Filialfilter auf Berichte angewendet wird.

## Nach Filiale filtern {#filtering-by-store}

Sobald mehrere Filialen konfiguriert sind, erscheint der Filialfilter im gesamten POS:

- **[Bestellungen](/orders)** — Bestellhistorie nach Filialstandort filtern
- **[Berichte](/reports)** — Tagesabschlussberichte für eine bestimmte Filiale erstellen
- **[Belege](/receipts/at-checkout)** — Belege zeigen automatisch die richtige Filialadresse und das richtige Logo an

## Filialspezifische Preise {#store-specific-pricing}

WCPOS Pro unterstützt **unterschiedliche Preise für verschiedene Filialstandorte**. Sie können pro Filiale einen separaten regulären Preis und Angebotspreis für Produkte und Varianten festlegen.

Wenn ein Kassierer in einer bestimmten Filiale arbeitet, verwendet der POS automatisch die Preise dieser Filiale. Wenn kein filialspezifischer Preis festgelegt ist, wird der standardmäßige WooCommerce-Preis verwendet.

## Bestandsverwaltung {#inventory-management}

Standardmäßig teilen sich alle Filialen dieselben WooCommerce Lagerbestände. Für die Bestandsverfolgung pro Standort integriert sich WCPOS Pro mit dem Plugin [ATUM Inventory Management](https://www.atum.org/):

1. Installieren und konfigurieren Sie das ATUM Plugin auf Ihrer WooCommerce Website
2. Erstellen Sie Bestandsstandorte in ATUM
3. Ordnen Sie in WCPOS Pro jede Filiale einem ATUM Bestandsstandort zu
4. Die Lagerbestände werden für den richtigen Standort aktualisiert, wenn Bestellungen aufgegeben werden

### Alternative: WordPress Multisite {#alternative-wordpress-multisite}

Für vollständig getrennte Bestände ohne ATUM können Sie [WordPress Multisite](https://developer.wordpress.org/advanced-administration/multisite/create-network/) verwenden:

1. Erstellen Sie eine separate WordPress Website für jeden Filialstandort
2. Installieren Sie WooCommerce und WCPOS auf jeder Website
3. Jede Website verwaltet ihren eigenen unabhängigen Produktkatalog und ihre eigenen Lagerbestände

Dies bietet eine vollständige Trennung, bedeutet jedoch, dass mehrere WooCommerce Installationen verwaltet werden müssen.

## Bekannte Einschränkungen {#known-limitations}

- **Eine einzelne WooCommerce-Instanz** — Filialen sind Standorte innerhalb einer WooCommerce-Installation, keine separaten Shops
- **Keine Produktsichtbarkeit pro Filiale** — alle Filialen sehen denselben Produktkatalog
