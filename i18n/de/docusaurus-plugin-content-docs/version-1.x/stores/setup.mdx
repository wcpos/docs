:::info Pro-Funktion
Mehrere Filialen erfordern [WCPOS Pro](/getting-started/pro-license). Ohne Pro läuft WCPOS als einzelne virtuelle Filiale, die aus den allgemeinen WooCommerce-Einstellungen übernommen wird — es gibt keine Filialen-Ansicht.
:::

Diese Seite führt durch das vollständige Erstellen einer Filiale. Für die konzeptionelle Übersicht (was mehrere Filialen ermöglichen, Preise pro Filiale, ATUM-Integration) beginnen Sie mit [Mehrere Filialen](/stores).

## Vor dem Start {#before-you-start}

- **WCPOS Pro ist installiert und die Lizenz ist aktiviert.** Durch die Aktivierung von Pro wird ein Untermenü **Filialen** unter POS in WP Admin hinzugefügt.
- **Die Kassierer, die in dieser Filiale arbeiten, haben WordPress-Konten** mit einer Rolle, die die Berechtigung *Zugriff auf WooCommerce POS* umfasst. Falls dies noch nicht eingerichtet ist, siehe [Zugriffseinstellungen](/settings/wp-admin/access).
- *(Optional)* **Eine Logo-Datei zum Hochladen** — PNG oder JPG, idealerweise 300×300 oder größer.

## Filiale erstellen {#create-the-store}

<Steps>
  <StepCard number="1" title="Filialliste öffnen">

Gehen Sie in WP Admin zu **POS → Filialen**. Sie sehen eine Liste aller vorhandenen Filialen mit Spalten für **Filiale**, **Adresse**, **Autorisierte Benutzer** und einer Schaltfläche **POS öffnen** pro Zeile.

Wenn dies Ihre erste Filiale ist und Pro gerade aktiviert wurde, ist die Liste leer. Klicken Sie in der Seitenkopfzeile auf **Neue Filiale hinzufügen**.

  </StepCard>
  <StepCard number="2" title="Filiale benennen">

Legen Sie oben im Editor einen **Filialnamen** fest. Dieser erscheint auf Belegen, in der Filialauswahl bei der Anmeldung und in Filialfiltern im gesamten POS. Erforderlich.

Beispiele: *"Main Street Cafe"*, *"Warehouse — Pickup"*, *"Market Stall #3"*.

  </StepCard>
  <StepCard number="3" title="Hauptspalte ausfüllen">

Die linke Seite des Editors gruppiert alles, was auf dem Beleg landet:

- **Filiallogo** — über die WP-Mediathek hochladen. Es gibt einen Umschalter *"Websitelogo verwenden, wenn kein Filiallogo festgelegt ist"*, falls die Website ein Customizer-Logo hat und Sie es als Fallback verwenden möchten.
- **Adresse** — Adresszeile 1, Adresszeile 2, Ort, Bundesland, Postleitzahl, Land. Land ist eine Dropdown-Liste mit Suche während der Eingabe.
- **Kontakt** — Webadresse, Telefon, E-Mail.
- **Belegnachrichten** — *Grußformel*, *Rückgaberichtlinie*, *Fußzeile*. Einfacher Text mit einfachem HTML erlaubt (`<strong>`, `<em>`, `<b>`, `<i>`, `<br>`, `<p>`).
- **Belegvorlagen** — lassen Sie den Schalter *"Filialspezifische Belegvorlagen"* **deaktiviert**, um die websiteweite Vorlagenliste aus **POS → Vorlagen** zu übernehmen, oder schalten Sie ihn **ein**, um ein eigenes Set für diese Filiale auszuwählen. Die Neusortierung per Ziehgriff funktioniert, sobald Sie ein eigenes Set ausgewählt haben.
- **Belegvorschau** — Live-Vorschau mit Vorlagenauswahl und einem Schalter für Beispieldaten / Bestellung, damit Sie vor dem Speichern sehen können, wie der Beleg tatsächlich aussehen wird.

</StepCard>
  <StepCard number="4" title="Seitenleiste konfigurieren">

Die rechte Spalte enthält die filialspezifische Konfiguration, die nicht auf Belegen angezeigt wird:

- **Autorisierte Benutzer** — Kontrollkästchenliste aller WordPress-Benutzer, deren Rolle über die Berechtigung *Zugriff auf WooCommerce POS* verfügt. Wählen Sie die Kassierer aus, die sich in dieser Filiale anmelden dürfen. **Wenn die Liste leer bleibt, sind alle POS-fähigen Benutzer autorisiert** — praktisch für Websites mit nur einer Filiale.
- **Steuersatz** — Dropdown *Steuer berechnen basierend auf*: **Diese Filialadresse** *(Standard)* oder **Basisstandort**. Die darunterliegende Tabelle zeigt die ermittelten Steuersätze je Steuerklasse für die ausgewählte Adresse; sie ist schreibgeschützt, konfigurieren Sie die tatsächlichen Sätze daher unter **WooCommerce → Einstellungen → Steuern**.
- **Steuer-IDs** — wiederholbare Zeilen aus (Typ / Wert / Land / Bezeichnung) für filialbezogene USt-IdNr., ABN, GSTIN usw. Leer lassen, um auf die websiteweiten [Filial-Steuer-IDs](/settings/wp-admin/store-tax-ids) zurückzufallen.
- **Filialeinstellungen** — Währung, Währungsposition, Locale, Zeitzone. Standardmäßig werden die Websitewerte verwendet; nur überschreiben, wenn diese Filiale anders als die Websitevorgabe arbeitet.
- **Öffnungszeiten** — Zeitfenster pro Tag plus ein Freitextfeld für Notizen. Optional, wird von einigen Vorlagen verwendet.

</StepCard>
  <StepCard number="5" title="Speichern" isLast>

Eine ganz neue Filiale zeigt eine sichtbare Schaltfläche **Erstellen**, bis sie zum ersten Mal gespeichert wurde. Klicken Sie auf **Erstellen**.

Nach dem Speichern wechselt die Seite zur automatischen Speicherung — Änderungen werden nach kurzer Verzögerung automatisch gespeichert, mit einer Anzeige *Speichern… → Gespeichert* in der Ecke. Die Seite kann jederzeit verlassen werden und die Änderungen bleiben erhalten; der Browser warnt, wenn versucht wird, die Seite mit ungespeicherten Änderungen zu schließen.

  </StepCard>
</Steps>

## Was Kassierer bei der Anmeldung sehen {#what-cashiers-see-at-login}

Wenn sich ein Kassierer anmeldet, zeigt der POS eine **Filialauswahl** mit allen Filialen an, für die er autorisiert ist:

- **Für eine Filiale autorisiert** — diese Filiale wird automatisch ausgewählt und die Schaltfläche **POS öffnen** ist sofort aktiviert. Die Auswahl wird weiterhin angezeigt, damit der Kassierer sehen kann, welche Filiale er gleich öffnet.
- **Für mehrere Filialen autorisiert** — die Auswahl zeigt eine Radioliste. Wählen Sie eine Filiale aus und anschließend **POS öffnen**.
- **Für keine Filiale autorisiert** — die Auswahl ist leer. Fügen Sie den Kassierer entweder zur Liste *Autorisierte Benutzer* einer Filiale hinzu oder lassen Sie diese Liste leer, um alle zu autorisieren.

Die ausgewählte Filiale wird beim Start des POS als `?store={id}` angehängt und bei jeder während dieser Sitzung aufgegebenen Bestellung gespeichert (die Bestellmeta `_pos_store`). Dies ermöglicht filialbezogene Berichte, den Filialfilter in Bestellungen, das richtige Logo und die richtige Adresse auf Belegen sowie filialbezogene Preise.

## Preisgestaltung pro Filiale {#per-store-pricing}

WCPOS Pro bietet einen nativen Mechanismus für die Preisgestaltung pro Filiale — unabhängig von ATUM. Auf jeder Bearbeitungsseite für Produkte (und Varianten) erhalten die Tabs **Preis** und **Steuer** eine Überschreibung pro Filiale:

1. Öffnen Sie ein Produkt in WP Admin → Produkte → Bearbeiten.
2. Aktivieren Sie auf dem Tab **Preis** das Kontrollkästchen *"Preis pro Filiale festlegen"* für die Filiale, die Sie überschreiben möchten.
3. Geben Sie den regulären Preis (und optional den Aktionspreis) für diese Filiale ein.
4. Speichern Sie das Produkt.

Wenn eine Kassenkraft von dieser Filiale aus arbeitet, verwendet der POS den Preis pro Filiale. Wird von einer Filiale ohne Überschreibung aus gearbeitet, gilt der Standardpreis von WooCommerce.

Dasselbe Muster funktioniert auf dem Tab **Steuer** für den Steuerstatus / die Steuerklasse pro Filiale — nützlich, wenn sich ein Standort in einer steuerbefreiten Zone befindet und ein anderer nicht.

:::tip ATUM im Vergleich zur nativen Preisgestaltung pro Filiale
Die native Preisgestaltung pro Filiale von WCPOS ist das richtige Werkzeug, wenn unterschiedliche Regalpreise an verschiedenen Standorten benötigt werden, der Bestand aber **gemeinsam** genutzt wird. Wenn zusätzlich **Bestandsmengen pro Standort** benötigt werden, installieren Sie die Integration [ATUM Multi-Inventory](/extensions/atum) — sie fügt dem Filialeditor ein Dropdown *Bestandsstandort* und eine Auswahl *Preisquelle* (Standard / WCPOS Pro / ATUM) hinzu, sodass beide Systeme pro Filiale kombiniert werden können.
:::

## Sonderfälle {#edge-cases}

<Accordion>
  <AccordionItem question="Was passiert mit bereits vorhandenen Bestellungen, wenn ich eine neue Filiale hinzufüge?">

Nichts. Bestellungen werden zum Verkaufszeitpunkt mit dem Meta `_pos_store` versehen. Das Hinzufügen einer neuen Filiale weist keine bestehenden Bestellungen rückwirkend neu zu. Bestellungen, die vor der Aktivierung von Pro aufgegeben wurden (oder in einer Einrichtung mit nur einer Filiale), haben kein Meta `_pos_store` und werden weiterhin mit den websiteweiten Standardwerten dargestellt.

Wenn eine bestehende Bestellung einer anderen Filiale zugewiesen werden muss, öffnen Sie sie unter WP Admin → WooCommerce → Bestellungen und verwenden Sie das Dropdown **Filiale**, das Pro dem Bildschirm für einzelne Bestellungen hinzufügt.

  </AccordionItem>

  <AccordionItem question="Wie lösche ich eine Filiale?">

Die Schaltfläche **Papierkorb** im Filialeditor (und die Zeilenaktion in der Filialliste) führt ein WordPress-Soft-Delete aus (`wp_trash_post`). Sie löscht nicht dauerhaft, daher gilt:

- Die Filiale wird in der Auswahl für Kassierer und in Filter-Dropdowns ausgeblendet.
- Bestehende Bestellungen, die auf die in den Papierkorb verschobene Filiale verweisen, werden für den erneuten Belegdruck und Detailansichten der Bestellung weiterhin korrekt aufgelöst — Name, Adresse und Logo der in den Papierkorb verschobenen Filiale bleiben verfügbar.
- Beim Wiederherstellen aus dem Papierkorb wird die Filiale mit allen unveränderten Einstellungen wieder in den aktiven Zustand versetzt.

Nur ein vollständiges dauerhaftes Löschen (über Papierkorb → Endgültig löschen) entfernt den Filialdatensatz. Bestellungen, die auf eine dauerhaft gelöschte Filiale verweisen, fallen auf *"Filiale #ID"* und die websiteweiten Standardwerte zurück.

  </AccordionItem>

  <AccordionItem question="Kann ein Kassierer in mehr als einer Filiale arbeiten?">

Ja — markieren Sie den Namen in der Liste *Autorisierte Benutzer* für jede Filiale, auf die Zugriff bestehen soll. Es gibt kein Konzept einer "primären Filiale" pro Benutzer. Bei jeder Anmeldung wird über die Auswahl festgelegt, von welcher Filiale aus gearbeitet wird.

  </AccordionItem>

  <AccordionItem question="Kann ich pro Filiale unterschiedliche Währungen verwenden?">

Legen Sie die **Währung** und die **Währungsposition** in der Seitenleiste der Filiale fest. Diese überschreiben die websiteweite WooCommerce-Währung für jede Sitzung, die in dieser Filiale geöffnet wird — Belege, Preise und Summen werden alle mit der Währung der Filiale formatiert.

Dies betrifft nur die Anzeigeformatierung; Sie benötigen weiterhin WooCommerce-kompatible Zahlungsgateways für jede Währung, und die Wechselkursverwaltung liegt in der Verantwortung Ihrer Filiale.

  </AccordionItem>

  <AccordionItem question="Warum gibt es kein Untermenü Filialen, wenn ich Pro installiere?">

Zwei häufige Ursachen:

1. **Pro ist nicht aktiviert** — öffnen Sie **POS → Einstellungen → Lizenz** und bestätigen Sie, dass der Lizenzschlüssel als aktiv angezeigt wird.
2. **Altes Layout mit kostenloser Version + Pro** — vor v1.8 war Pro ein Add-on für das kostenlose Plugin. Wenn Sie auf v1.8+ aktualisiert haben, muss das kostenlose Plugin deaktiviert und gelöscht werden (siehe Hinweis zur [Installation von WCPOS Pro](/getting-started/pro-license)) — wenn beide installiert bleiben, kann das Filialen-Menü ausgeblendet werden.

  </AccordionItem>
</Accordion>
