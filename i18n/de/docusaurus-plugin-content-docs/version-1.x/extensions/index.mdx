WCPOS unterstützt Erweiterungen, die dem Kassensystem neue Funktionen hinzufügen. Über das Erweiterungsverzeichnis lassen sich verfügbare Erweiterungen durchsuchen, direkt aus den POS-Einstellungen installieren und Updates verwalten.

:::info Pro-Funktion
Das Installieren und Verwalten von Erweiterungen erfordert [WCPOS Pro](/getting-started/pro-license). In der kostenlosen Version wird der Erweiterungskatalog angezeigt, die Steuerelemente zur Installation und Aktivierung sind jedoch deaktiviert.
:::

## Erweiterungen durchsuchen {#browsing-extensions}

Das Erweiterungsverzeichnis lässt sich über `POS Settings > Extensions` öffnen (in einigen Versionen auch als **Plugins** bezeichnet).

Das Verzeichnis zeigt ein Kartenraster der verfügbaren Erweiterungen an. Jede Karte enthält:

- **Symbol** (oder ein Puzzleteil als Fallback, falls die Erweiterung kein eigenes Symbol bereitstellt)
- **Name und Version**
- **Beschreibung**
- **Kategorie-Badge**
- **Status** – aktiv, inaktiv, Update verfügbar oder nicht installiert

### Filtern und Suchen {#filtering-and-search}

Verwenden Sie die **Kategorie-Pill-Buttons** oben, um Erweiterungen nach Kategorie zu filtern. Sie können auch das **Suchfeld** nutzen, um Erweiterungen nach Name, Beschreibung oder Tags zu finden.

## Eine Erweiterung installieren {#installing-an-extension}

1. Öffnen Sie `POS Settings > Extensions`.
2. Suchen Sie die gewünschte Erweiterung und klicken Sie auf **Installieren**.
3. Die Erweiterung wird über den WordPress-Plugin-Installer heruntergeladen und installiert.
4. Klicken Sie nach der Installation auf **Aktivieren**, um sie zu aktivieren.

Im Hintergrund verwendet WCPOS den nativen `Plugin_Upgrader` von WordPress für die Installation, sodass Erweiterungen demselben Prozess wie jedes andere WordPress-Plugin folgen.

## Aktivieren und Deaktivieren {#activating-and-deactivating}

Jede installierte Erweiterung verfügt auf ihrer Karte über die Schaltflächen **Aktivieren** und **Deaktivieren**.

- **Aktivieren** schaltet die Erweiterung frei, sodass sie im POS ausgeführt werden kann.
- **Deaktivieren** schaltet sie ab, ohne sie zu deinstallieren. Die Dateien der Erweiterung verbleiben auf dem Server und können jederzeit wieder aktiviert werden.

## Erweiterungen aktualisieren {#updating-extensions}

Wenn eine neuere Version einer installierten Erweiterung verfügbar ist, zeigt die Karte das Abzeichen **Update verfügbar** sowie eine Schaltfläche **Aktualisieren** an.

### Automatische Updates {#auto-updates}

Aus dem Verzeichnis installierte Erweiterungen haben **automatische Updates standardmäßig aktiviert**. Automatische Updates lassen sich pro Erweiterung über deren Karte im Verzeichnis ein- oder ausschalten. Wenn automatische Updates aktiviert sind, installiert WordPress neue Versionen automatisch, genau wie bei Plugins mit aktivierten automatischen Updates.

## Fehlerbehebung {#troubleshooting}

### Meldung „Pro erforderlich“ auf Schaltflächen {#requires-pro-message-on-buttons}

Die Schaltflächen zum Installieren, Aktivieren und Aktualisieren sind in der kostenlosen Version von WCPOS deaktiviert. Führen Sie ein Upgrade auf [WCPOS Pro](/getting-started/pro-license) durch, um Erweiterungen zu verwalten.

### Erweiterung lässt sich nicht installieren {#extension-fails-to-install}

- Prüfen Sie, ob Ihr WordPress-Server Schreibrechte für das Verzeichnis `wp-content/plugins` besitzt.
- Stellen Sie sicher, dass Ihr Server ausgehende HTTPS-Anfragen senden kann (einige Hoster blockieren externe Downloads).
- Sehen Sie sich die Fehlerdetails unter `WP Admin > POS > Support > Logs` an.

### Erweiterung wird nach der Installation nicht angezeigt {#extension-not-appearing-after-install}

- Aktualisieren Sie den POS – die Erweiterungsliste wird bis zu 12 Stunden zwischengespeichert.
- Stellen Sie sicher, dass die Erweiterung aktiviert ist (installierte, aber inaktive Erweiterungen werden nicht ausgeführt).

### Katalog wird nicht geladen {#catalog-not-loading}

Der Erweiterungskatalog wird von einer externen Quelle abgerufen und 12 Stunden lang lokal zwischengespeichert. Falls der Katalog nicht geladen wird:

- Prüfen Sie die Internetverbindung Ihres Servers.
- Versuchen Sie es nach Ablauf des Caches erneut oder leeren Sie den Transient-Cache Ihres Servers.

## Für Entwickler {#for-developers}

### Eine POS-Erweiterung erstellen {#creating-a-pos-extension}

Eine WCPOS-Erweiterung ist ein standardmäßiges WordPress-Plugin, das sich über WCPOS-Hooks und -APIs in den POS integriert. So erstellen Sie eine:

1. **Beginnen Sie mit einem WordPress-Plugin.** Ihre Erweiterung benötigt einen Standard-Plugin-Header und eine Eintrittsdatei, genau wie jedes WooCommerce- oder WordPress-Plugin.
2. **Integration in WCPOS.** Verwenden Sie die von WCPOS bereitgestellten Hooks und Filter, um Funktionen zur POS-Oberfläche oder zum Backend hinzuzufügen.
3. **Releases auf GitHub bereitstellen.** Das Erweiterungsverzeichnis nutzt GitHub Releases, um Versionen zu verfolgen und Updates auszuliefern.

### Einreichung im Verzeichnis {#submitting-to-the-directory}

Der Erweiterungskatalog wird im GitHub-Repository [`wcpos/extensions`](https://github.com/wcpos/extensions) gepflegt. So listen Sie Ihre Erweiterung:

1. Prüfen Sie das Katalogformat und die Metadatenanforderungen in der README des Repositorys.
2. Öffnen Sie einen Pull Request, um die Metadaten Ihrer Erweiterung zu `catalog.json` hinzuzufügen.
3. Nach dem Merge erscheint Ihre Erweiterung im Verzeichnis für alle WCPOS Pro-Benutzer.

### Konventionen für GitHub Releases {#github-release-conventions}

Der Update-Lebenszyklus basiert auf GitHub Releases:

- **Versionen taggen** unter Verwendung semantischer Versionierung (z. B. `v1.0.0`, `v1.2.3`).
- **Hängen Sie die Plugin-ZIP-Datei** als Release-Asset an — dies ist die Datei, die heruntergeladen wird, wenn ein Benutzer das Plugin installiert oder aktualisiert.
- **Veröffentlichen Sie das Release** (nicht als Entwurf), damit das Verzeichnis es erkennen kann.

Wenn Sie ein neues Release veröffentlichen, wird Benutzern mit installierter Erweiterung das Update in ihrem Erweiterungsverzeichnis angezeigt. Wenn die automatische Aktualisierung aktiviert ist, wird sie automatisch angewendet.

Vollständige Details zum Katalogschema und zum Einreichungsprozess finden Sie im Repository [`wcpos/extensions`](https://github.com/wcpos/extensions).
