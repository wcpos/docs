WCPOS unterstützt Erweiterungen, die Ihrem Kassensystem neue Funktionen hinzufügen. Über das Erweiterungsverzeichnis lassen sich verfügbare Erweiterungen durchsuchen, direkt aus den POS-Einstellungen installieren und Aktualisierungen verwalten.

:::info Pro-Funktion
Das Installieren und Verwalten von Erweiterungen erfordert [WCPOS Pro](/getting-started/pro-license). In der kostenlosen Version wird der Erweiterungskatalog zwar angezeigt, die Steuerelemente für Installation und Aktivierung sind jedoch deaktiviert.
:::

## Erweiterungen durchsuchen {#browsing-extensions}

Öffnen Sie das Erweiterungsverzeichnis im WordPress-Adminbereich über `POS Settings > Extensions` (in einigen Versionen auch als **Plugins** bezeichnet).

Das Verzeichnis zeigt ein Kartenraster der verfügbaren Erweiterungen an. Jede Karte enthält:

- **Symbol** (oder ein Puzzleteil als Ersatz, wenn die Erweiterung kein Symbol bereitstellt)
- **Name und Version**
- **Beschreibung**
- **Kategorie-Badge**
- **Status** – aktiv, inaktiv, Update verfügbar oder nicht installiert

### Filterung und Suche {#filtering-and-search}

Verwenden Sie die **Kategorie-Pill-Schaltflächen** oben, um Erweiterungen nach Kategorie zu filtern. Sie können auch das **Suchfeld** verwenden, um Erweiterungen nach Name, Beschreibung oder Tags zu finden.

## Installation einer Erweiterung {#installing-an-extension}

1. Öffnen `POS Settings > Extensions`.
2. Suchen Sie die gewünschte Erweiterung und klicken Sie auf **Installieren**.
3. Die Erweiterung wird über das native WordPress-Plugin-Installationsprogramm heruntergeladen und installiert.
4. Klicken Sie nach der Installation auf **Aktivieren**, um sie zu aktivieren.

Im Hintergrund verwendet WCPOS WordPress' natives `Plugin_Upgrader` für die Installation zuständig, sodass Erweiterungen denselben Prozess wie jedes WordPress-Plugin durchlaufen.

## Aktivieren und Deaktivieren {#activating-and-deactivating}

Jede installierte Erweiterung verfügt auf ihrer Karte über die Schaltflächen **Aktivieren** und **Deaktivieren**.

- **Aktivieren** schaltet die Erweiterung frei, sodass sie im POS ausgeführt werden kann.
- **Deaktivieren** schaltet sie ab, ohne sie zu deinstallieren. Die Dateien der Erweiterung bleiben auf dem Server und können jederzeit wieder aktiviert werden.

## Erweiterungen aktualisieren {#updating-extensions}

Sobald eine neuere Version einer installierten Erweiterung verfügbar ist, zeigt die Karte ein Abzeichen **Update verfügbar** sowie eine Schaltfläche **Aktualisieren** an.

### Automatische Updates {#auto-updates}

Bei Erweiterungen, die aus dem WordPress-Verzeichnis installiert wurden, sind **automatische Updates standardmäßig aktiviert**. Die automatischen Updates lassen sich pro Erweiterung über deren Karte im Verzeichnis ein- oder ausschalten. Ist die Funktion aktiviert, installiert WordPress neue Versionen automatisch – genauso wie bei Plugins mit aktivierten automatischen Updates.

## Fehlerbehebung {#troubleshooting}

### Meldung „Erfordert Pro“ auf Schaltflächen {#requires-pro-message-on-buttons}

Die Schaltflächen zum Installieren, Aktivieren und Aktualisieren sind in der kostenlosen Version von WCPOS deaktiviert. Mit einem Upgrade auf [WCPOS Pro](/getting-started/pro-license) lassen sich Erweiterungen verwalten.

### Erweiterung lässt sich nicht installieren {#extension-fails-to-install}

- Prüfen Sie, ob Ihr WordPress-Server Schreibrechte für das Verzeichnis `wp-content/plugins` hat.
- Stellen Sie sicher, dass Ihr Server ausgehende HTTPS-Anfragen senden kann (manche Hoster blockieren externe Downloads).
- Sehen Sie sich die Fehlerdetails in `WP Admin > POS > Support > Logs`.

### Erweiterung erscheint nach der Installation nicht {#extension-not-appearing-after-install}

- Aktualisieren Sie den POS – die Erweiterungsliste wird bis zu 12 Stunden zwischengespeichert.
- Vergewissern Sie sich, dass die Erweiterung aktiviert ist (installierte, aber inaktive Erweiterungen werden nicht ausgeführt).

### Katalog lädt nicht {#catalog-not-loading}

Der Erweiterungskatalog wird von einer externen Quelle abgerufen und lokal für 12 Stunden zwischengespeichert. Falls der Katalog nicht geladen wird:

- Prüfen Sie die Internetverbindung Ihres Servers.
- Versuchen Sie es erneut, nachdem der Cache abgelaufen ist, oder leeren Sie den Transient-Cache Ihres Servers.

## Für Entwickler {#for-developers}

### Eine POS-Erweiterung erstellen {#creating-a-pos-extension}

Eine WCPOS-Erweiterung ist ein reguläres WordPress-Plugin, das über WCPOS-Hooks und -APIs in den POS integriert wird. So erstellen Sie eine:

1. **Beginnen Sie mit einem WordPress-Plugin.** Ihre Erweiterung benötigt einen Standard-Plugin-Header und eine Einstiegsdatei, genau wie jedes WooCommerce- oder WordPress-Plugin.
2. **Integration mit WCPOS.** Nutzen Sie die von WCPOS bereitgestellten Hooks und Filter, um Funktionen zur POS-Oberfläche oder zum Backend hinzuzufügen.
3. **Releases auf GitHub hosten.** Das Erweiterungsverzeichnis verwendet GitHub Releases, um Versionen zu verfolgen und Updates auszuliefern.

### Einreichung im Verzeichnis {#submitting-to-the-directory}

Der Erweiterungskatalog wird im GitHub-Repository [`wcpos/extensions`](https://github.com/wcpos/extensions) gepflegt. So listen Sie Ihre Erweiterung auf:

1. Lesen Sie das Katalogformat und die Metadatenanforderungen in der README des Repositorys nach.
2. Öffnen Sie einen Pull Request, um die Metadaten Ihrer Erweiterung hinzuzufügen zu `catalog.json`.
3. Nach dem Merge erscheint Ihre Erweiterung im Verzeichnis für alle WCPOS Pro-Nutzer.

### GitHub-Release-Konventionen {#github-release-conventions}

Der Update-Lebenszyklus basiert auf GitHub Releases:

- **Versionen taggen** mit semantischer Versionierung (z. B. `v1.0.0`, `v1.2.3`).
- **Plugin-ZIP anhängen** als Release-Asset – dies ist die Datei, die beim Installieren oder Aktualisieren durch den Nutzer heruntergeladen wird.
- **Release veröffentlichen** (nicht als Entwurf), damit das Verzeichnis es erkennen kann.

Wenn Sie ein neues Release veröffentlichen, sehen Nutzer mit installierter Erweiterung das Update in ihrem Erweiterungsverzeichnis. Bei aktivierter automatischer Aktualisierung wird es automatisch angewendet.

Vollständige Informationen zum Katalogschema und zum Einreichungsprozess finden Sie im [`wcpos/extensions`](https://github.com/wcpos/extensions)-Repository.
