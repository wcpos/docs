WCPOS unterstützt Erweiterungen, die Ihrem Kassensystem neue Funktionen hinzufügen. Über das Erweiterungsverzeichnis lassen sich verfügbare Erweiterungen durchsuchen, direkt aus den POS-Einstellungen installieren und Aktualisierungen verwalten.

:::info Pro-Funktion
Für das Installieren und Verwalten von Erweiterungen ist [WCPOS Pro](/getting-started/pro-license) erforderlich. In der kostenlosen Version wird der Erweiterungskatalog zwar angezeigt, die Steuerelemente zur Installation und Aktivierung sind jedoch deaktiviert.
:::

## Erweiterungen durchsuchen {#browsing-extensions}

Öffnen Sie das Erweiterungsverzeichnis über `POS Settings > Extensions` (in einigen Versionen auch als **Plugins** bezeichnet).

Das Verzeichnis zeigt ein Kartenraster der verfügbaren Erweiterungen an. Jede Karte enthält:

- **Symbol** (oder ein Puzzleteil als Ersatz, falls die Erweiterung keines bereitstellt)
- **Name und Version**
- **Beschreibung**
- **Kategorie-Badge**
- **Status** – aktiv, inaktiv, Update verfügbar oder nicht installiert

### Filtern und Suchen {#filtering-and-search}

Verwenden Sie die **Kategorie-Pill-Buttons** oben, um Erweiterungen nach Kategorie zu filtern. Sie können auch das **Suchfeld** nutzen, um Erweiterungen nach Name, Beschreibung oder Tags zu finden.

## Eine Erweiterung installieren {#installing-an-extension}

1. Öffnen Sie `POS Settings > Extensions`.
2. Suchen Sie die gewünschte Erweiterung und klicken Sie auf **Installieren**.
3. Die Erweiterung wird über den WordPress-Plugin-Installer heruntergeladen und installiert.
4. Klicken Sie nach der Installation auf **Aktivieren**, um sie zu aktivieren.

Im Hintergrund nutzt WCPOS den nativen `Plugin_Upgrader` von WordPress für die Installation, sodass Erweiterungen demselben Prozess wie jedes WordPress-Plugin folgen.

## Aktivieren und Deaktivieren {#activating-and-deactivating}

Jede installierte Erweiterung verfügt über **Aktivieren**- und **Deaktivieren**-Schaltflächen auf ihrer Karte.

- **Aktivieren** schaltet die Erweiterung frei, damit sie im POS ausgeführt werden kann.
- **Deaktivieren** schaltet sie ab, ohne sie zu deinstallieren. Die Dateien der Erweiterung verbleiben auf dem Server und können jederzeit wieder aktiviert werden.

## Erweiterungen aktualisieren {#updating-extensions}

Wenn eine neuere Version einer installierten Erweiterung verfügbar ist, zeigt die Karte ein **Update verfügbar**-Abzeichen sowie eine Schaltfläche **Aktualisieren** an.

### Automatische Updates {#auto-updates}

Aus dem Verzeichnis installierte Erweiterungen haben **automatische Updates standardmäßig aktiviert**. Automatische Updates lassen sich pro Erweiterung über deren Karte im Verzeichnis ein- oder ausschalten. Bei aktivierten automatischen Updates wendet WordPress neue Versionen automatisch an, genauso wie bei Plugins mit aktivierten automatischen Updates.

## Fehlerbehebung {#troubleshooting}

### Meldung „Pro erforderlich“ auf Schaltflächen {#requires-pro-message-on-buttons}

In der kostenlosen Version von WCPOS sind die Schaltflächen zum Installieren, Aktivieren und Aktualisieren deaktiviert. Für die Verwaltung von Erweiterungen ist ein Upgrade auf [WCPOS Pro](/getting-started/pro-license) erforderlich.

### Erweiterung lässt sich nicht installieren {#extension-fails-to-install}

- Prüfen Sie, ob Ihr WordPress-Server Schreibrechte für das Verzeichnis `wp-content/plugins` besitzt.
- Stellen Sie sicher, dass Ihr Server ausgehende HTTPS-Anfragen senden kann (einige Hoster blockieren externe Downloads).
- Sehen Sie sich die Fehlerdetails unter `WP Admin > POS > Support > Logs` an.

### Erweiterung wird nach der Installation nicht angezeigt {#extension-not-appearing-after-install}

- Aktualisieren Sie den POS — die Erweiterungsliste wird bis zu 12 Stunden zwischengespeichert.
- Stellen Sie sicher, dass die Erweiterung aktiviert ist (installierte, aber inaktive Erweiterungen werden nicht ausgeführt).

### Katalog wird nicht geladen {#catalog-not-loading}

Der Erweiterungskatalog wird von einer entfernten Quelle abgerufen und lokal für 12 Stunden zwischengespeichert. Falls der Katalog nicht geladen wird:

- Überprüfen Sie die Internetverbindung Ihres Servers.
- Versuchen Sie es nach Ablauf des Caches erneut oder leeren Sie den Transient-Cache Ihres Servers.

## Für Entwickler {#for-developers}

### Erstellen einer POS-Erweiterung {#creating-a-pos-extension}

Eine WCPOS-Erweiterung ist ein standardmäßiges WordPress-Plugin, das über WCPOS-Hooks und -APIs in den POS integriert wird. So erstellen Sie eine:

1. **Beginnen Sie mit einem WordPress-Plugin.** Ihre Erweiterung benötigt einen standardmäßigen Plugin-Header und eine Einstiegsdatei, genau wie jedes WooCommerce- oder WordPress-Plugin.
2. **Integration mit WCPOS.** Nutzen Sie die von WCPOS bereitgestellten Hooks und Filter, um Funktionen zur POS-Oberfläche oder zum Backend hinzuzufügen.
3. **Releases auf GitHub hosten.** Das Erweiterungsverzeichnis nutzt GitHub Releases, um Versionen zu verfolgen und Updates auszuliefern.

### Einreichung im Verzeichnis {#submitting-to-the-directory}

Der Erweiterungskatalog wird im GitHub-Repository [`wcpos/extensions`](https://github.com/wcpos/extensions) gepflegt. So listen Sie Ihre Erweiterung:

1. Lesen Sie das Katalogformat und die Metadaten-Anforderungen in der README des Repositories nach.
2. Öffnen Sie einen Pull Request, um die Metadaten Ihrer Erweiterung zur `catalog.json` hinzuzufügen.
3. Nach dem Merge erscheint Ihre Erweiterung im Verzeichnis für alle WCPOS Pro-Nutzer.

### Konventionen für GitHub Releases {#github-release-conventions}

Der Update-Lebenszyklus basiert auf GitHub Releases:

- **Versionen taggen** mittels semantischer Versionierung (z. B. `v1.0.0`, `v1.2.3`).
- **Plugin-ZIP anhängen** als Release-Asset – dies ist die Datei, die heruntergeladen wird, wenn Nutzer das Plugin installieren oder aktualisieren.
- **Veröffentlichen Sie das Release** (kein Entwurf), damit das Verzeichnis es erkennen kann.

Wenn Sie ein neues Release veröffentlichen, sehen Benutzer mit installierter Erweiterung das Update in ihrem Erweiterungsverzeichnis. Wenn die automatische Aktualisierung aktiviert ist, wird sie automatisch angewendet.

Vollständige Details zum Katalogschema und zum Einreichungsprozess finden Sie im Repository [`wcpos/extensions`](https://github.com/wcpos/extensions).
