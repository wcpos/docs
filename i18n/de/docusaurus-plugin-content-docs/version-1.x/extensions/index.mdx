WCPOS unterstützt Erweiterungen, die Ihrem Point of Sale neue Funktionen hinzufügen. Im Erweiterungsverzeichnis können Sie verfügbare Erweiterungen durchsuchen, direkt aus den POS-Einstellungen installieren und Aktualisierungen verwalten.

:::info Pro-Funktion
Das Installieren und Verwalten von Erweiterungen erfordert [WCPOS Pro](/getting-started/pro-license). Die kostenlose Version zeigt den Erweiterungskatalog an, deaktiviert jedoch die Installations- und Aktivierungssteuerelemente.
:::

## Erweiterungen durchsuchen {#browsing-extensions}

Öffnen Sie das Erweiterungsverzeichnis über `POS Settings > Extensions` (in einigen Versionen auch als **Plugins** bezeichnet).

Das Verzeichnis zeigt ein Kartenraster der verfügbaren Erweiterungen an. Jede Karte enthält:

- **Symbol** (oder ein Puzzleteil als Ersatz, wenn die Erweiterung keines bereitstellt)
- **Name und Version**
- **Beschreibung**
- **Kategorie-Badge**
- **Status** – aktiv, inaktiv, Update verfügbar oder nicht installiert

### Filterung und Suche {#filtering-and-search}

Verwenden Sie die **Kategorie-Filterschaltflächen** oben, um Erweiterungen nach Kategorie zu filtern. Sie können auch das **Suchfeld** nutzen, um Erweiterungen nach Name, Beschreibung oder Tags zu finden.

## Eine Erweiterung installieren {#installing-an-extension}

1. Öffnen Sie `POS Settings > Extensions` hinzuzufügen.
2. Suchen Sie die gewünschte Erweiterung und klicken Sie auf **Installieren**.
3. Die Erweiterung wird über den WordPress-Plugin-Installer heruntergeladen und installiert.
4. Klicken Sie nach der Installation auf **Aktivieren**, um die Erweiterung zu aktivieren.

Im Hintergrund verwendet WCPOS den nativen WordPress- `Plugin_Upgrader` für die Installation, sodass Erweiterungen demselben Prozess wie jedes WordPress-Plugin folgen.

## Aktivieren und Deaktivieren {#activating-and-deactivating}

Jede installierte Erweiterung hat auf ihrer Karte die Schaltflächen **Aktivieren** und **Deaktivieren**.

- **Aktivieren** schaltet die Erweiterung frei, sodass sie im POS ausgeführt werden kann.
- **Deaktivieren** schaltet die Erweiterung ab, ohne sie zu deinstallieren. Die Erweiterungsdateien bleiben auf dem Server und können jederzeit wieder aktiviert werden.

## Erweiterungen aktualisieren {#updating-extensions}

Wenn eine neuere Version einer installierten Erweiterung verfügbar ist, zeigt die Karte ein **Update verfügbar**-Abzeichen und eine **Aktualisieren**-Schaltfläche an.

### Automatische Updates {#auto-updates}

Bei Erweiterungen, die aus dem Verzeichnis installiert wurden, sind **automatische Updates standardmäßig aktiviert**. Die automatischen Updates lassen sich je Erweiterung über deren Karte im Verzeichnis ein- oder ausschalten. Wenn automatische Updates aktiviert sind, übernimmt WordPress neue Versionen automatisch, genauso wie bei Plugins mit aktivierten automatischen Updates.

## Fehlerbehebung {#troubleshooting}

### Meldung „Requires Pro" auf Schaltflächen {#requires-pro-message-on-buttons}

Die Schaltflächen zum Installieren, Aktivieren und Aktualisieren sind in der kostenlosen Version von WCPOS deaktiviert. Für die Verwaltung von Erweiterungen ist ein Upgrade auf [WCPOS Pro](/getting-started/pro-license) erforderlich.

### Installation einer Erweiterung schlägt fehl {#extension-fails-to-install}

- Prüfen Sie, ob der WordPress-Server Schreibrechte für das `wp-content/plugins` -Verzeichnis hat.
- Stellen Sie sicher, dass der Server ausgehende HTTPS-Anfragen ausführen kann (manche Hoster blockieren externe Downloads).
- Sehen Sie sich die Fehlerdetails an unter `WP Admin > POS > Support > Logs` hinzuzufügen.

### Erweiterung wird nach der Installation nicht angezeigt {#extension-not-appearing-after-install}

- Aktualisieren Sie den POS – die Erweiterungsliste wird bis zu 12 Stunden zwischengespeichert.
- Stellen Sie sicher, dass die Erweiterung aktiviert ist (installierte, aber inaktive Erweiterungen werden nicht ausgeführt).

### Katalog wird nicht geladen {#catalog-not-loading}

Der Erweiterungskatalog wird von einer externen Quelle abgerufen und 12 Stunden lang lokal zwischengespeichert. Falls der Katalog nicht geladen wird:

- Überprüfen Sie die Internetverbindung Ihres Servers.
- Versuchen Sie es nach Ablauf des Caches erneut oder leeren Sie den Transient-Cache Ihres Servers.

## Für Entwickler {#for-developers}

### Eine POS-Erweiterung erstellen {#creating-a-pos-extension}

Eine WCPOS-Erweiterung ist ein standardmäßiges WordPress-Plugin, das über WCPOS-Hooks und -APIs in den POS integriert wird. So erstellen Sie eine Erweiterung:

1. **Mit einem WordPress-Plugin beginnen.** Ihre Erweiterung benötigt einen standardmäßigen Plugin-Header und eine Einstiegsdatei, genau wie jedes WooCommerce- oder WordPress-Plugin.
2. **In WCPOS integrieren.** Verwenden Sie die von WCPOS bereitgestellten Hooks und Filter, um Funktionen zur POS-Oberfläche oder zum Backend hinzuzufügen.
3. **Releases auf GitHub hosten.** Das Erweiterungsverzeichnis nutzt GitHub Releases, um Versionen zu verfolgen und Updates bereitzustellen.

### Einreichung im Verzeichnis {#submitting-to-the-directory}

Der Erweiterungskatalog wird im GitHub-Repository [`wcpos/extensions`](https://github.com/wcpos/extensions) gepflegt. So listen Sie Ihre Erweiterung:

1. Lesen Sie das Katalogformat und die Metadatenanforderungen in der README-Datei des Repositorys.
2. Öffnen Sie einen Pull Request, um die Metadaten Ihrer Erweiterung zu `catalog.json` hinzuzufügen.
3. Sobald die Änderung zusammengeführt wurde, erscheint Ihre Erweiterung im Verzeichnis für alle WCPOS Pro-Nutzer.

### Konventionen für GitHub Releases {#github-release-conventions}

Der Update-Lebenszyklus basiert auf GitHub Releases:

- **Versionen taggen** mittels semantischer Versionierung (z. B. `v1.0.0`, `v1.2.3`).
- **Plugin-ZIP anhängen** als Release-Asset – dies ist die Datei, die beim Installieren oder Aktualisieren durch den Benutzer heruntergeladen wird.
- **Release veröffentlichen** (nicht als Entwurf), damit das Verzeichnis es erkennen kann.

Wenn ein neues Release veröffentlicht wird, sehen Benutzer mit installierter Erweiterung das verfügbare Update in ihrem Erweiterungsverzeichnis. Ist die automatische Aktualisierung aktiviert, wird sie automatisch angewendet.

Vollständige Informationen zum Katalogschema und zum Einreichungsprozess finden Sie im Repository [`wcpos/extensions`](https://github.com/wcpos/extensions).
