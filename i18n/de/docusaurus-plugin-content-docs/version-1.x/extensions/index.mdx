WCPOS unterstützt Erweiterungen, die Ihrem Kassensystem neue Funktionen hinzufügen. Über das Erweiterungsverzeichnis lassen sich verfügbare Erweiterungen durchsuchen, direkt aus den POS-Einstellungen installieren und Aktualisierungen verwalten.

:::info Pro-Funktion
Das Installieren und Verwalten von Erweiterungen erfordert [WCPOS Pro](/getting-started/pro-license). In der kostenlosen Version wird der Erweiterungskatalog zwar angezeigt, die Installations- und Aktivierungssteuerung ist jedoch deaktiviert.
:::

## Erweiterungen durchsuchen {#browsing-extensions}

Öffnen Sie das Erweiterungsverzeichnis über `POS Settings > Extensions` (in einigen Versionen auch als **Plugins** bezeichnet).

Das Verzeichnis zeigt ein Kartenraster der verfügbaren Erweiterungen an. Jede Karte enthält:

- **Symbol** (oder ein Puzzleteil-Platzhalter, falls die Erweiterung keines bereitstellt)
- **Name und Version**
- **Beschreibung**
- **Kategorie-Badge**
- **Status** – aktiv, inaktiv, Update verfügbar oder nicht installiert

### Filtern und Suchen {#filtering-and-search}

Verwenden Sie die **Kategorie-Pill-Buttons** oben, um Erweiterungen nach Kategorie zu filtern. Sie können auch das **Suchfeld** nutzen, um Erweiterungen nach Name, Beschreibung oder Tags zu finden.

## Eine Erweiterung installieren {#installing-an-extension}

1. Öffnen Sie `POS Settings > Extensions`.
2. Suchen Sie die gewünschte Erweiterung und klicken Sie auf **Installieren**.
3. Die Erweiterung wird über den WordPress-Plugin-Installer heruntergeladen und installiert.
4. Klicken Sie nach der Installation auf **Aktivieren**, um sie zu aktivieren.

Im Hintergrund verwendet WCPOS den nativen `Plugin_Upgrader` von WordPress, um die Installation abzuwickeln, sodass Erweiterungen demselben Ablauf folgen wie jedes WordPress-Plugin.

## Aktivieren und Deaktivieren {#activating-and-deactivating}

Jede installierte Erweiterung verfügt auf ihrer Karte über die Schaltflächen **Aktivieren** und **Deaktivieren**.

- **Aktivieren** schaltet die Erweiterung frei, sodass sie im POS ausgeführt werden kann.
- **Deaktivieren** schaltet sie ab, ohne sie zu deinstallieren. Die Erweiterungsdateien verbleiben auf dem Server und können jederzeit wieder aktiviert werden.

## Erweiterungen aktualisieren {#updating-extensions}

Wenn eine neuere Version einer installierten Erweiterung verfügbar ist, zeigt die Karte ein **Update verfügbar**-Abzeichen sowie eine Schaltfläche **Aktualisieren** an.

### Automatische Updates {#auto-updates}

Bei Erweiterungen, die aus dem Verzeichnis installiert wurden, sind **automatische Updates standardmäßig aktiviert**. Automatische Updates lassen sich pro Erweiterung über deren Karte im Verzeichnis ein- oder ausschalten. Ist die Funktion aktiviert, installiert WordPress neue Versionen automatisch, genau wie bei Plugins mit aktivierten Auto-Updates.

## Fehlerbehebung {#troubleshooting}

### Meldung „Pro erforderlich“ auf Schaltflächen {#requires-pro-message-on-buttons}

Die Schaltflächen zum Installieren, Aktivieren und Aktualisieren sind in der kostenlosen Version von WCPOS deaktiviert. Führen Sie ein Upgrade auf [WCPOS Pro](/getting-started/pro-license) durch, um Erweiterungen zu verwalten.

### Erweiterung lässt sich nicht installieren {#extension-fails-to-install}

- Stellen Sie sicher, dass Ihr WordPress-Server über Schreibrechte für das Verzeichnis `wp-content/plugins` verfügt.
- Prüfen Sie, ob Ihr Server ausgehende HTTPS-Anfragen stellen kann (einige Hoster blockieren externe Downloads).
- Sehen Sie sich die Fehlerdetails in `WP Admin > POS > Support > Logs` an.

### Erweiterung erscheint nach der Installation nicht {#extension-not-appearing-after-install}

- Aktualisieren Sie den POS – die Liste der Erweiterungen wird bis zu 12 Stunden zwischengespeichert.
- Stellen Sie sicher, dass die Erweiterung aktiviert ist (installierte, aber inaktive Erweiterungen werden nicht ausgeführt).

### Katalog wird nicht geladen {#catalog-not-loading}

Der Erweiterungskatalog wird aus einer entfernten Quelle abgerufen und 12 Stunden lang lokal zwischengespeichert. Falls der Katalog nicht geladen wird:

- Prüfen Sie die Internetverbindung Ihres Servers.
- Versuchen Sie es erneut, nachdem der Cache abgelaufen ist, oder leeren Sie den Transient-Cache Ihres Servers.

## Für Entwickler {#for-developers}

### Eine POS-Erweiterung erstellen {#creating-a-pos-extension}

Eine WCPOS-Erweiterung ist ein standardmäßiges WordPress-Plugin, das über WCPOS-Hooks und -APIs in den POS integriert wird. Um eine zu erstellen:

1. **Mit einem WordPress-Plugin beginnen.** Ihre Erweiterung benötigt einen standardmäßigen Plugin-Header und eine Einstiegsdatei, genau wie jedes WooCommerce- oder WordPress-Plugin.
2. **In WCPOS integrieren.** Verwenden Sie die von WCPOS bereitgestellten Hooks und Filter, um der POS-Oberfläche oder dem Backend Funktionen hinzuzufügen.
3. **Releases auf GitHub hosten.** Das Erweiterungsverzeichnis verwendet GitHub Releases, um Versionen zu verfolgen und Updates auszuliefern.

### Einreichung im Verzeichnis {#submitting-to-the-directory}

Der Erweiterungskatalog wird im GitHub-Repository [`wcpos/extensions`](https://github.com/wcpos/extensions) gepflegt. So listen Sie Ihre Erweiterung auf:

1. Prüfen Sie das Katalogformat und die Metadatenanforderungen in der README des Repositorys.
2. Öffnen Sie einen Pull Request, um die Metadaten Ihrer Erweiterung zu `catalog.json` hinzuzufügen.
3. Nach dem Merge erscheint Ihre Erweiterung im Verzeichnis für alle WCPOS Pro-Nutzer.

### Konventionen für GitHub-Releases {#github-release-conventions}

Der Update-Lebenszyklus basiert auf GitHub Releases:

- **Versionen taggen** mittels semantischer Versionierung (z. B. `v1.0.0`, `v1.2.3`).
- **Plugin-ZIP anhängen** als Release-Asset — dies ist die Datei, die beim Installieren oder Aktualisieren durch einen Nutzer heruntergeladen wird.
- **Veröffentlichen Sie das Release** (nicht als Entwurf), damit das Verzeichnis es erkennen kann.

Wenn Sie ein neues Release veröffentlichen, sehen Benutzer mit installierter Erweiterung das verfügbare Update in ihrem Erweiterungsverzeichnis. Wenn automatische Updates aktiviert sind, wird es automatisch angewendet.

Vollständige Informationen zum Katalog-Schema und zum Einreichungsprozess finden Sie im Repository [`wcpos/extensions`](https://github.com/wcpos/extensions).
