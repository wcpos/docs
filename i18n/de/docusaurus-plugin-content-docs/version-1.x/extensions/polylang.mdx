# WCPOS Polylang {#wcpos-polylang}

Ergänzt WCPOS um [Polylang](https://polylang.pro/)-Unterstützung, sodass im POS nur Produkte einer einzigen Sprache angezeigt werden – keine doppelten Übersetzungen in der Produktsuche, im Katalograster oder in Kassiervorgängen. WCPOS-Pro-Shops können eine Sprache pro Shop festlegen; in kostenlosen Installationen wird auf die Polylang-Standardsprache zurückgegriffen.

## Funktionsweise {#what-it-does}

- Filtert WCPOS-REST-Abfragen für Produkte und Varianten nach Sprache.
- Fängt WCPOS-**Fast-Sync**-Routen ab (die leichtgewichtigen `posts_per_page=-1` + `fields` Anfragen, die der POS zum Aktualisieren seines lokalen Index verwendet), sodass übersetzte Duplikate niemals den Client erreichen.
- In kostenlosen Installationen wird die Polylang-Standardsprache angewendet.
- In Pro-Installationen kann jeder Shop seine eigene Sprache in einem neuen Abschnitt **Sprache** im Shop-Editor auswählen.
- Berücksichtigt die WCPOS-Sichtbarkeitseinstellung **Nur POS** für Produkte beim Erstellen der Fast-Sync-Datenstruktur.

Die Integration verhält sich neutral, wenn Polylang nicht aktiv ist – das Plugin kann ohne Fehler bereits vor der Aktivierung von Polylang installiert werden.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Install Polylang">

[Polylang](https://wordpress.org/plugins/polylang/) (oder Polylang Pro) wie gewohnt installieren und die Sprachen der Website konfigurieren. Mindestens eine Sprache muss als Standard festgelegt sein.

  </StepCard>
  <StepCard number="2" title="Install WCPOS Polylang">

Installation über das WCPOS-Erweiterungsverzeichnis unter `WP Admin > POS > Settings > Extensions`, oder das aktuelle Release über die [GitHub-Releases-Seite](https://github.com/wcpos/wcpos-polylang/releases) herunterladen und hochladen über `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="(Pro) Set a per-store language" isLast>

Bei Verwendung von [mehreren Stores](/stores) unter WCPOS Pro wechseln Sie zu `POS > Stores`, bearbeiten Sie einen Store und wählen Sie dessen Sprache im Seitenleisten-Abschnitt **Sprache** aus. Belassen Sie die Einstellung auf *Standard*, um die Polylang-Standardsprache zu verwenden.

  </StepCard>
</Steps>

## Sprache pro Store (Pro) {#per-store-language-pro}

In WCPOS Pro fügt das Plugin der Seitenleiste des Store-Editors einen Abschnitt **Sprache** hinzu. Jeder Store kann an einen einzelnen Polylang-Sprachslug gebunden werden – Produkte, die an diesen Store ausgeliefert werden, werden ausschließlich auf diese Sprache gefiltert. Stores, die auf *Standard* belassen werden, verwenden die Polylang-Standardsprache.

Der Wert pro Store wird am Store-Post als `_wcpos_polylang_language` -Meta gespeichert und über die WCPOS Pro Stores-REST-API bereitgestellt (`/wcpos/v1/stores`), sodass er wie jede andere Store-Einstellung durch den POS hin- und zurückgereicht wird.

## Hinweise zur Kompatibilität {#compatibility-notes}

- **Nur-POS-Produkte:** Wenn der Nur-POS-Modus in den WCPOS-Einstellungen aktiviert ist, werden IDs reiner Online-Produkte aus der Fast-Sync-Nutzlast ausgeschlossen, sodass sie nicht in den POS gelangen.
- **Kostenlose Installationen:** Es gibt keine Benutzeroberfläche, um die Sprache pro Store zu ändern – das Plugin verwendet die Standardsprache von Polylang. Wenn Sie Sprachen pro Store benötigen, führen Sie ein Upgrade auf [WCPOS Pro](/getting-started/pro-license) durch.
- **Plugin nicht verfügbar:** Wenn Polylang deaktiviert ist, bleibt das Plugin stillschweigend untätig. Es löst keine Fehler aus und blockiert den POS nicht.

## Entwickler-Hooks {#developer-hooks}

Für erweiterte Anwendungsfälle stellt das Plugin einige Filter bereit:

| Filter | Zweck |
| --- | --- |
| `wcpos_polylang_resolved_language` | Überschreibt die für eine bestimmte Anfrage verwendete Sprache. Erhält den ermittelten Slug und das `WP_REST_Request`. |
| `wcpos_polylang_default_language` | Überschreibt die Fallback-Sprache, wenn kein Wert pro Store festgelegt ist. |
| `wcpos_polylang_is_supported` | Erzwingt das Aktivieren oder Deaktivieren des Plugins unabhängig von der Polylang-Erkennung. |
| `wcpos_polylang_minimum_version` | Erfordert eine Mindestversion von Polylang (Standard: keine Versionsprüfung). |
