# WCPOS ATUM Integration {#wcpos-atum-integration}

Integriert [ATUM Multi-Inventory](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/) mit [WCPOS Pro](/getting-started/pro-license) und ermöglicht standortbasierte Bestände, Preise und SKUs am Point of Sale.

ATUM Multi-Inventory ermöglicht es, den Lagerbestand eines Produkts auf mehrere Inventarstandorte aufzuteilen — Lager, Einzelhandelsgeschäfte und so weiter. Dieses Plugin verbindet diese ATUM-Standorte mit den WCPOS Pro [Filialen](/stores), sodass jedes POS-Terminal die korrekten Lagerbestände, Preise und SKUs für seinen physischen Standort anzeigt.

## Funktionen {#features}

<FeatureGrid>
  <FeatureCard icon="box" title="Bestand pro Standort">
    Jede Filiale bezieht die Lagerbestände vom zugewiesenen ATUM-Inventarstandort statt vom aggregierten WooCommerce-Lagerbestand.
  </FeatureCard>
  <FeatureCard icon="store" title="Flexible Preisgestaltung">
    Wählen Sie zwischen WooCommerce-Standardpreisen, filialspezifischen Preisen aus WCPOS Pro oder standortspezifischen Preisen aus ATUM.
  </FeatureCard>
  <FeatureCard icon="barcode" title="Standort-SKUs">
    Optional kann im POS die Haupt-SKU des Produkts gegen eine ATUM-standortspezifische SKU ausgetauscht werden.
  </FeatureCard>
  <FeatureCard icon="sync" title="Audit-sichere Bestandsbewegung">
    Bestellungen reduzieren und stellen den Bestand am korrekten ATUM-Standort wieder her, mit vollständigem Audit-Trail in `atum_inventory_orders`.
  </FeatureCard>
  <FeatureCard icon="arrow-right" title="Rückschreiben bei Produktbearbeitung">
    Bearbeitungen von Bestand, Preis und SKU im POS werden in die zugeordnete ATUM-Bestandszeile für diesen Standort zurücksynchronisiert.
  </FeatureCard>
</FeatureGrid>

## Installation {#installation}

<Steps>
  <StepCard number="1" title="ATUM und Multi-Inventory installieren">

Installieren Sie [ATUM Inventory Management](https://wordpress.org/plugins/atum-stock-manager-for-woocommerce/) und das [ATUM Multi-Inventory Add-on](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/). Konfigurieren Sie Ihre Lagerstandorte in ATUM.

  </StepCard>
  <StepCard number="2" title="WCPOS ATUM Integration installieren">

Installieren Sie über `WP Admin > POS > Settings > Extensions` oder laden Sie die neueste Version von der [GitHub-Releases-Seite](https://github.com/wcpos/wcpos-atum/releases) herunter und installieren Sie sie über `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="Filialen ATUM-Standorten zuordnen" isLast>

Gehen Sie zu `POS > Stores`, bearbeiten Sie eine Filiale und konfigurieren Sie den Seitenleistenbereich **ATUM Inventory**. Wählen Sie den Lagerstandort, den die Filiale verwenden soll, wählen Sie eine Preisquelle und aktivieren Sie optional SKU-Überschreibungen.

  </StepCard>
</Steps>

## Filialkonfiguration {#store-configuration}

Das Plugin fügt einen Bereich **ATUM Inventory** zur Seitenleiste des WCPOS Pro Filialeditors hinzu, mit drei Einstellungen pro Filiale:

- **Lagerort** — von welchem ATUM-Standort diese Filiale ihren Bestand bezieht.
- **Preisquelle** — woher die Produktpreise stammen:
  - *Standard* — Standard-WooCommerce-Preise
  - *WCPOS Pro* — filialspezifische Preise, die in WCPOS Pro festgelegt werden
  - *ATUM* — standortspezifische Preise aus dem ATUM-Bestand
- **SKU-Überschreibung** — optional standortspezifische SKUs aus ATUM anstelle der Haupt-SKU des Produkts verwenden.

## POS-Verhalten {#pos-behavior}

Wenn einer Filiale ein ATUM-Standort zugewiesen ist, werden die an den POS gesendeten Produktdaten automatisch angepasst:

- **Bestandsmengen** spiegeln den Bestand des jeweiligen Standorts wider, nicht den gesamten WooCommerce-Bestand.
- **Bestandsstatus** wird basierend auf der Menge des Standorts neu berechnet.
- **Preise** stammen aus der konfigurierten Preisquelle.
- **SKUs** werden durch die SKU des ATUM-Standorts ersetzt, wenn die Überschreibung aktiviert ist.

Alle Anpassungen erfolgen transparent über die WCPOS REST API – auf der Seite der POS-App sind keine Änderungen erforderlich. Produktbearbeitungen, die im POS vorgenommen werden, werden ebenfalls in die zugeordnete ATUM-Bestandszeile zurückgeschrieben; siehe [Rückschreibung von Produktbearbeitungen](#product-edit-write-back) unten.

## Bestandsverwaltung {#stock-management}

Für POS-Bestellungen, die in Filialen mit einem zugeordneten ATUM-Standort aufgegeben werden, überlässt das Plugin den Schreibvorgang dem nativen Bestandsabzugsablauf von ATUM – steuert ihn jedoch zum richtigen Standort:

1. **REST-Payload-Injektion.** Wenn der POS eine Bestellung erstellt oder aktualisiert, fügt das Plugin jeder Position einen Eintrag `mi_inventories` hinzu, damit ATUM weiß, von welchem Standort abgezogen werden soll. Ohne diesen würde ATUM auf den Hauptbestand zurückgreifen.
2. **Standortbezogener Bestandsfilter.** Das Plugin filtert die Liste der ATUM-Kandidatenbestände auf jene, die mit dem zugeordneten Standortbegriff der Filiale verknüpft sind, sodass sowohl bei der Reduzierung als auch bei der Wiederherstellung der richtige ausgewählt wird.

ATUM selbst führt die eigentliche Bestandsänderung bei Bestellung und Rückerstattung durch und schreibt Zeilen in `atum_inventory_orders` mit der echten `order_id` – wodurch der Prüfpfad von ATUM erhalten bleibt.

## Rückschreibung von Produktbearbeitungen {#product-edit-write-back}

Wenn ein Kassierer oder Manager ein Produkt oder eine Variation im POS bearbeitet, werden die Änderungen in die zugeordnete ATUM-Bestandszeile für den Standort der jeweiligen Filiale zurücksynchronisiert – nicht nur in das WooCommerce-Hauptprodukt. So bleiben Bestand, Preis und SKU jeder Filiale mit ATUM synchron, ohne manuelle Aktualisierungen im `WP Admin`.

Die Rückschreibung erfolgt bei WCPOS-Produkt- und Variations-REST-Aktualisierungen (`POST`, `PUT`, `PATCH` an `/wcpos/v1/products/...`) mit einer `store_id`. Das Plugin sucht den zugeordneten ATUM-Standort der Filiale und aktualisiert nur die Bestandszeile für diesen Standort – andere Standorte bleiben unberührt.

### Was synchronisiert wird {#what-syncs}

Das Zurückschreiben berücksichtigt die Konfiguration jeder Filiale, sodass ATUM-Daten nur dann geändert werden, wenn die Filiale die jeweiligen Daten tatsächlich besitzt:

| Feld | Wann es synchronisiert wird |
| --- | --- |
| **Lagerbestand** | Immer – jede Filiale mit einem zugeordneten ATUM-Standort hält ihre Standortzeile synchron. |
| **Regulärer Preis / Angebotspreis / Preis** | Nur wenn die **Preisquelle** der Filiale auf *ATUM* gesetzt ist. |
| **SKU** | Nur wenn **SKU-Überschreibung** für die Filiale aktiviert ist. |

Wenn die Filiale die Preisgestaltung *Standard* oder *WCPOS Pro* verwendet, werden die ATUM-Preisfelder nicht angetastet, sodass ATUM weiterhin als Referenzpreis und nicht als verbindliche Quelle dient. Dasselbe gilt für SKUs, wenn die Überschreibung deaktiviert ist.

### Was kein Zurückschreiben auslöst {#what-doesn-t-trigger-write-back}

- Produkterstellung (nur Aktualisierungen werden zurückgeschrieben – neue Produkte durchlaufen den normalen Speicherpfad von WooCommerce).
- Anfragen ohne `store_id` – der POS muss dem Plugin mitteilen, an welchen Standort geschrieben werden soll.
- Filialen ohne zugeordneten ATUM-Standort.
- Produkte ohne vorhandene ATUM-Bestandszeile für den Standort der Filiale – das Plugin erstellt keine neuen Bestandszeilen, sondern aktualisiert nur bestehende.
