# Ubersetzt: WCPOS WPML {#wcpos-wpml}

Integriert [WPML](https://wpml.org/) mit WCPOS, sodass das POS nur Produkte für eine einzelne Sprache liefert — übersetzte Duplikate erscheinen nicht mehr in der Produktsuche und im Katalograster. Mit WCPOS Pro kann eine Sprache pro Filiale festgelegt werden.

## Funktionsweise {#what-it-does}

- Filtert WCPOS-Produkt- und Varianten-REST-Abfragen auf eine einzelne WPML-Sprache.
- Fängt die **Fast-Sync**-Routen von WCPOS ab (die leichtgewichtigen Anfragen, die das POS zur Aktualisierung seines lokalen Index verwendet), sodass übersetzte Duplikate nie den Client erreichen.
- Die kostenlose Version verwendet die WPML-Standardsprache.
- Die Pro-Version ermöglicht es, die Sprache pro Filiale im Filial-Editor zu überschreiben.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="DE: WPML installieren">

Installieren und konfigurieren Sie [WPML](https://wpml.org/) wie gewohnt, wobei mindestens eine Sprache als Standard festgelegt sein muss.

  </StepCard>
  <StepCard number="2" title="DE: WCPOS WPML installieren">

Installation über `WP Admin > POS > Einstellungen > Erweiterungen`, oder laden Sie die neueste Version von der [GitHub-Releases-Seite](https://github.com/wcpos/wcpos-wpml/releases) herunter und installieren Sie sie über `Plugins > Neu hinzufügen > Plugin hochladen`.

  </StepCard>
  <StepCard number="3" title="DE: (Pro) Sprache pro Filiale festlegen" isLast>

In WCPOS Pro kann eine Filiale unter `POS > Geschäfte` bearbeitet und die Sprache im Seitenleistenbereich **Sprache** ausgewählt werden. Bei *Standard* wird die WPML-Standardsprache verwendet.

  </StepCard>
</Steps>

## Bekannte WPML-Kompatibilitätsprobleme {#known-wpml-compatibility-issues}

Dies sind Verhaltensweisen von WPML selbst und nicht der Integration — es lohnt sich, diese vor der Einführung von Mehrsprachigkeit in der Produktivumgebung zu kennen:

- **Benutzerdefinierte POS-Felder werden nicht über Übersetzungen hinweg übernommen.** WPML übersetzt Produkte, kopiert jedoch standardmäßig keine benutzerdefinierten WCPOS-Felder in übersetzte Versionen. Ein Produkt, das in der Standardsprache als „Nur POS" markiert ist, kann diese Einstellung in seinen Übersetzungen verlieren. Konfigurieren Sie WPML so, dass benutzerdefinierte WCPOS-Felder bei der Übersetzung kopiert werden.
- **Nur-POS-Produkte und 404-Fehler in der Storefront.** Da WPML für jede Sprache Storefront-Seiten generiert, können Nur-POS-Produkte beim Zugriff über die Website als 404-Fehler angezeigt werden. Dies ist eine bekannte WPML-Interaktion und kein WCPOS-Fehler.

Siehe [Nur-POS-Produkte](/products/pos-only-products) für die zugehörigen POS-Sichtbarkeitseinstellungen.
