# WCPOS WPML {#wcpos-wpml}

Integriert [WPML](https://wpml.org/) in WCPOS, sodass im POS nur Produkte einer einzigen Sprache bereitgestellt werden – übersetzte Duplikate erscheinen nicht mehr in der Produktsuche und im Katalograster. Mit WCPOS Pro lässt sich eine Sprache pro Filiale festlegen.

## Funktionsweise {#what-it-does}

- Filtert REST-Abfragen für WCPOS-Produkte und -Varianten auf eine einzige WPML-Sprache.
- Fängt die **Fast-Sync**-Routen von WCPOS ab (die leichtgewichtigen Anfragen, mit denen der POS seinen lokalen Index aktualisiert), sodass übersetzte Duplikate niemals den Client erreichen.
- Bei Free-Installationen wird die WPML-Standardsprache verwendet.
- Bei Pro-Installationen lässt sich die Sprache pro Filiale im Filialeditor überschreiben.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Install WPML">

[WPML](https://wpml.org/) wie gewohnt installieren und konfigurieren, wobei mindestens eine Sprache als Standardsprache festgelegt sein muss.

  </StepCard>
  <StepCard number="2" title="Install WCPOS WPML">

Installation über `WP Admin > POS > Settings > Extensions`oder das neueste Release von der [GitHub-Releases-Seite](https://github.com/wcpos/wcpos-wpml/releases) herunterladen und hochladen über `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="(Pro) Pin a language per store" isLast>

In WCPOS Pro einen Store bearbeiten unter `POS > Stores` und dessen Sprache im Seitenleistenbereich **Sprache** auswählen. Auf *Standard* belassen, um die WPML-Standardsprache zu verwenden.

  </StepCard>
</Steps>

## Bekannte WPML-Kompatibilitätsprobleme {#known-wpml-compatibility-issues}

Hierbei handelt es sich um Verhaltensweisen von WPML selbst und nicht der Integration – sollte vor dem produktiven Einsatz mehrsprachiger Inhalte bekannt sein:

- **POS-Benutzerdefinierte Felder werden nicht in Übersetzungen übernommen.** WPML übersetzt Produkte, kopiert WCPOS-benutzerdefinierte Felder jedoch standardmäßig nicht in die übersetzten Versionen. Ein Produkt, das in der Standardsprache als „POS Only“ markiert ist, kann diese Einstellung in seinen Übersetzungen verlieren. WPML so konfigurieren, dass WCPOS-benutzerdefinierte Felder während der Übersetzung kopiert werden.
- **Nur-POS-Produkte und 404-Fehler im Shop.** Da WPML Shop-Seiten für jede Sprache generiert, können Nur-POS-Produkte beim Aufruf auf der Website als 404 angezeigt werden. Dies ist eine bekannte WPML-Interaktion und kein Fehler von WCPOS.

Siehe [Nur-POS-Produkte](/products/pos-only-products) für die zugehörigen POS-Sichtbarkeitseinstellungen.
