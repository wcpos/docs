# WCPOS WP Multilang {#wcpos-wp-multilang}

Integriert [WP Multilang](https://wordpress.org/plugins/wp-multilang/) in WCPOS, sodass der POS nur Produkte für eine einzige Sprache ausliefert. Im Gegensatz zu Polylang oder WPML speichert WP Multilang alle Übersetzungen innerhalb desselben Beitrags – dieses Plugin stellt sicher, dass WCPOS die korrekte Übersetzung liest und keine doppelten Einträge in den Suchergebnissen erzeugt.

## Funktionsweise {#what-it-does}

- Filtert die REST-Antworten für WCPOS-Produkte und -Varianten auf eine einzige WP-Multilang-Sprache.
- Fängt die **Fast-Sync**-Routen von WCPOS ab, sodass nur die Inhalte der aktiven Sprache den Client erreichen.
- Free-Installationen verwenden die Standardsprache von WP Multilang.
- Pro-Installationen können im Store-Editor eine Sprache pro Store festlegen.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Install WP Multilang">

Installieren Sie [WP Multilang](https://wordpress.org/plugins/wp-multilang/) und konfigurieren Sie die Sprachen Ihrer Website, wobei mindestens eine Sprache als Standard festgelegt sein muss.

  </StepCard>
  <StepCard number="2" title="Install WCPOS WP Multilang">

Installation über `WP Admin > POS > Settings > Extensions`oder laden Sie die neueste Version von der [GitHub-Releases-Seite](https://github.com/wcpos/wcpos-wp-multilang/releases) herunter und laden Sie sie hoch über `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="(Pro) Pin a language per store" isLast>

Bearbeiten Sie in WCPOS Pro einen Store unter `POS > Stores` und wählen Sie dessen Sprache im Seitenleistenbereich **Sprache** aus. Belassen Sie die Einstellung auf *Standard*, um die Standardsprache von WP Multilang zu verwenden.

  </StepCard>
</Steps>
