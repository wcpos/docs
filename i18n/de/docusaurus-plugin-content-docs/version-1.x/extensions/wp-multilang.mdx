# Ubersetzt: WCPOS WP Multilang {#wcpos-wp-multilang}

Integriert [WP Multilang](https://wordpress.org/plugins/wp-multilang/) mit WCPOS, sodass das POS nur Produkte für eine einzelne Sprache ausliefert. Im Gegensatz zu Polylang oder WPML speichert WP Multilang alle Übersetzungen im selben Beitrag – dieses Plugin stellt sicher, dass WCPOS die korrekte Übersetzung liest und keine doppelten Einträge in Suchergebnissen anzeigt.

## Funktionsweise {#what-it-does}

- Filtert WCPOS-Produkt- und Varianten-REST-Antworten auf eine einzelne WP-Multilang-Sprache.
- Fängt die **Fast-Sync**-Routen von WCPOS ab, sodass nur die Inhalte der aktiven Sprache den Client erreichen.
- Kostenlose Installationen verwenden die WP-Multilang-Standardsprache.
- Pro-Installationen können über den Filial-Editor eine Sprache pro Filiale festlegen.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="DE: WP Multilang installieren">

Installieren Sie [WP Multilang](https://wordpress.org/plugins/wp-multilang/) und konfigurieren Sie die Sprachen Ihrer Website, wobei mindestens eine Sprache als Standard festgelegt sein muss.

  </StepCard>
  <StepCard number="2" title="DE: WCPOS WP Multilang installieren">

Installation über `WP Admin > POS > Einstellungen > Erweiterungen`, oder laden Sie die neueste Version von der [GitHub-Releases-Seite](https://github.com/wcpos/wcpos-wp-multilang/releases) herunter und laden Sie sie über `Plugins > Neu hinzufügen > Plugin hochladen` hoch.

  </StepCard>
  <StepCard number="3" title="DE: (Pro) Sprache pro Filiale festlegen" isLast>

In WCPOS Pro kann eine Filiale unter `POS > Geschäfte` bearbeitet und die Sprache im Seitenleistenbereich **Sprache** ausgewählt werden. Bei der Einstellung *Standard* wird die Standardsprache von WP Multilang verwendet.

  </StepCard>
</Steps>
