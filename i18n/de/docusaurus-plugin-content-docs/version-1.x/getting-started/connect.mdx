Dieser Bildschirm wird nur in den Desktop- und Mobil-Apps angezeigt. Web-Benutzer greifen direkt unter `yourdomain.com/pos` auf das POS zu und melden sich mit ihren WordPress-Zugangsdaten an.

## Übersicht des Verbindungsbildschirms {#connect-screen-overview}

Beim Öffnen der WCPOS Desktop- oder Mobil-App wird der Verbindungsbildschirm angezeigt. Hier werden Filialverbindungen und Benutzeranmeldungen verwaltet.

{/* Screenshot: connect-screen.png */}

## Eine neue Filiale hinzufügen {#adding-a-new-store}

1. Geben Sie die URL Ihres WooCommerce-Shops in das Textfeld ein (z. B. `https://mystore.com`)
2. Klicken Sie auf **Verbinden**
3. Sie werden zur Anmeldung mit Ihren WordPress-Zugangsdaten weitergeleitet
4. Nach erfolgreicher Anmeldung werden Sie zur App zurückgeleitet

## Mehrere Filialen {#multiple-stores}

Sie können sich mit beliebig vielen WooCommerce-Shops verbinden. Jeder Shop wird als separate Karte auf dem Verbindungsbildschirm angezeigt, mit folgenden Informationen:

- **Shop-Name** und Favicon
- **Shop-URL**
- **Angemeldete Benutzer** für diesen Shop

Dies ist nützlich, wenn Sie mehrere Standorte oder Unternehmen verwalten.

## Mehrere Benutzer pro Filiale {#multiple-users-per-store}

In jeder Filiale können mehrere Benutzer gleichzeitig angemeldet sein. Das ist hilfreich für:

- **Schichtwechsel** – Neue Kassierer können sich anmelden, bevor sich die vorherigen abmelden
- **Mehrere Kassen** – Verschiedene Kassierer an verschiedenen Geräten
- **Schnelles Wechseln** – Einfaches Wechseln zwischen Benutzerkonten

### Benutzer hinzufügen {#adding-a-user}

Klicken Sie auf die <Icon name="plus" /> Schaltfläche neben „Angemeldete Benutzer", um einen weiteren Benutzer zu dieser Filiale hinzuzufügen.

### Benutzer wechseln {#switching-users}

Klicken Sie auf ein Benutzer-Badge (z.\u00a0B. „Brenda"), um den POS als dieser Benutzer zu öffnen.

### Einen Benutzer entfernen {#removing-a-user}

Klicken Sie auf das **×** an einem Benutzer-Badge, um diesen Benutzer aus der Filiale abzumelden.

## Eine Filiale entfernen {#removing-a-store}

Klicken Sie auf die rote **×**-Schaltfläche auf der Filialkarte, um sie aus der Liste zu entfernen. Dadurch werden alle Benutzer abgemeldet und die Filialverbindung aus der App entfernt.

## Demo-Filiale {#demo-store}

Am unteren Bildschirmrand wird möglicherweise ein Link „Demo-Filiale öffnen" angezeigt. Dieser verbindet Sie mit einer Demo-WooCommerce-Filiale, um WCPOS-Funktionen auszuprobieren, ohne Ihre eigenen Filialdaten zu beeinträchtigen.

## Fehlerbehebung {#troubleshooting}

:::tip Erste Überprüfung: `X-Frame-Options`
Die Desktop- und Mobil-Apps verwenden **iframes** für Anmeldung, Zahlung und Belege. **Jeder** Server-Header oder jedes Plugin, das `X-Frame-Options: DENY` oder `SAMEORIGIN` sendet, verhindert die Anmeldung. Dies ist die häufigste Ursache für fehlgeschlagene App-Anmeldungen — überprüfen Sie die Antwort-Header der Anmeldeseite (Browser-Entwicklertools oder `curl -I https://yourstore.com/wp-login.php`), bevor Sie etwas anderes tun.
:::

<Accordion>
  <AccordionItem question="Verbindung zur Filiale nicht möglich?">

  - Stellen Sie sicher, dass das WCPOS-Plugin auf Ihrer WordPress-Website installiert und aktiviert ist
  - Überprüfen Sie, ob die korrekte URL verwendet wird (einschließlich `https://` — die WooCommerce REST API erfordert SSL)
  - Versuchen Sie zunächst, `yourdomain.com/pos` in einem Webbrowser zu öffnen, um zu bestätigen, dass das Plugin funktioniert
  - Überprüfen Sie, ob die WooCommerce REST API erreichbar ist
  - Überprüfen Sie, ob Ihr Benutzerkonto über POS-Zugriffsberechtigungen verfügt

  </AccordionItem>

  <AccordionItem question="Anmeldung in der Desktop- oder Mobil-App schlägt fehl">

  Die meisten fehlgeschlagenen App-Anmeldungen werden durch ein Sicherheits- oder Caching-Plugin verursacht, das den Anmelde-iFrame blockiert:

  - **`X-Frame-Options`-Header** (gesetzt durch ein Sicherheits-Plugin oder den Server) blockieren den Login-iFrame — siehe den Hinweis oben. Deaktivieren Sie das Sicherheits-Plugin vorübergehend, melden Sie sich an und aktivieren Sie es dann wieder (die Sitzung bleibt etwa eine Woche bestehen).
  - **Sicherheits-Plugins** — Wordfence, Really Simple Security, WPS Hide Login, iThemes/Solid Security und Defender Pro sind häufige Ursachen. Die vollständige Liste und Lösungen finden Sie unter [Plugin-Konflikte](/support/troubleshooting/plugin-conflicts#security-and-login-plugins).
  - **Wordfence 2FA** — das 2FA-Codefeld wird im Login-iFrame nicht dargestellt. Deaktivieren Sie 2FA vorerst für POS-Benutzer.
  - **Benutzerdefinierte Login-URL** (z. B. WPS Hide Login) — die App kann die Anmeldeseite nicht finden. Verwenden Sie die Standard-URL `/wp-admin/`.
  - **Caching-Plugins** können das blockierte Anmeldeformular weiterhin ausliefern, auch nachdem das verursachende Plugin deaktiviert wurde — leeren Sie den Cache oder leeren Sie den App-Cache bzw. installieren Sie die Desktop-App neu.

  </AccordionItem>

  <AccordionItem question='"REST API requires authentication" oder ein Sicherheits-Plugin-Fehler auf dem Verbindungsbildschirm'>

  Ein Plugin (z. B. Force Login oder ein JWT-Auth-Plugin) erfordert eine Authentifizierung für alle REST-API-Anfragen, sodass die App die öffentlichen Informationen der Website nicht lesen kann. Die App zeigt jetzt die tatsächliche Meldung des Servers an (z. B. _„Nur authentifizierte Benutzer können auf die REST-API zugreifen"_), anstatt den Website-Typ falsch anzuzeigen.

  **Lösung:** Konfigurieren Sie das Sicherheits-Plugin so, dass nicht authentifizierter Zugriff auf `/wp-json/wcpos/` und `/wp-json/wc/v3/` erlaubt ist, oder deaktivieren Sie es so lange, bis die erste Verbindung hergestellt wurde.

  </AccordionItem>

  <AccordionItem question='"Scheint keine WordPress-Website zu sein" (Desktop-App)'>

  Die Desktop-App erkennt die REST API über den HTTP-`Link`-Header. Wenn ein Plugin (häufig **Image Prioritizer** oder andere Performance-Plugins) diesen Header überflutet oder abschneidet, schlägt die Erkennung fehl. **Lösung:** Bildoptimierungs- bzw. Header-modifizierende Performance-Plugins deaktivieren und erneut versuchen.

  </AccordionItem>

  <AccordionItem question="Die App meldet, dass ein Update erforderlich ist / stürzt nach einem Update ab">

  Prüfen Sie, ob eine Versionsabweichung zwischen der App und dem Server-Plugin besteht — der App Store hat möglicherweise ein App-Update bereitgestellt, während das WCPOS-Plugin noch aktualisiert werden muss (oder umgekehrt). Stellen Sie sicher, dass die App und das WCPOS-Plugin die **gleiche Hauptversion** verwenden. WCPOS v1.9.0+ Apps erfordern das WCPOS-Plugin in **Version 1.8.0 oder höher**.

  </AccordionItem>

  <AccordionItem question='Fehler „Cannot create fast store database"'>

  Dies ist eine Race Condition beim ersten Anmelden. **Schließen Sie die App vollständig und versuchen Sie es erneut** – beim zweiten Versuch funktioniert es in der Regel.

  </AccordionItem>

  <AccordionItem question="Hängt auf dem Benutzerauswahl-Bildschirm (Desktop-App)">

  Nach der Anmeldung wird Ihr Benutzername angezeigt, aber es gibt keinen offensichtlichen Weg weiter. **Klicken Sie auf Ihren Benutzernamen/Namen**, um in das POS zu gelangen – der Name selbst ist die Schaltfläche.

  </AccordionItem>

  <AccordionItem question="Verbindung schlägt wiederholt fehl?">

  - Rufen Sie zunächst `yourdomain.com/pos` in einem Webbrowser auf, um zu überprüfen, ob das Plugin funktioniert
  - Prüfen Sie die Fehlerprotokolle Ihrer Website auf mögliche Probleme
  - Stellen Sie sicher, dass Ihr Hoster die REST API nicht blockiert – siehe [Hosting-spezifische Hinweise](/support/performance/server#hosting-specific-notes)
  - Stellen Sie sicher, dass Ihr Server die [Mindestanforderungen](/support/performance/server#minimum-server-requirements) erfüllt

  </AccordionItem>
</Accordion>
