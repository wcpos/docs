# API05002: WCPOS-Plugin Nicht Gefunden {#api05002-wcpos-plugin-not-found}

## Was Das Bedeutet {#what-this-means}

Das WCPOS-Plugin ist auf Ihrer WordPress-Seite nicht installiert oder nicht aktiv. Die POS-Anwendung benötigt dieses Plugin, um mit WooCommerce zu kommunizieren.

## Häufige Ursachen {#common-causes}

- **Plugin nicht installiert** — Das Plugin wurde nie installiert
- **Plugin deaktiviert** — Das Plugin wurde deaktiviert
- **Plugin gelöscht** — Die Plugin-Dateien wurden entfernt
- **Falsche Seite** — Verbunden mit einer Seite ohne WCPOS

## So Beheben Sie Es {#how-to-fix}

### 1. Installieren Sie das Plugin {#1-install-the-plugin}

Falls nicht installiert:
1. Gehen Sie zu WordPress Admin → Plugins → Neu Hinzufügen
2. Suchen Sie nach "WooCommerce POS"
3. Installieren und aktivieren Sie das Plugin

Oder laden Sie es von [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/) herunter.

### 2. Aktivieren Sie das Plugin {#2-activate-the-plugin}

Falls installiert, aber nicht aktiv:
1. Gehen Sie zu Plugins → Installierte Plugins
2. Finden Sie "WooCommerce POS"
3. Klicken Sie auf "Aktivieren"

### 3. Auf Fehler Überprüfen {#3-check-for-errors}

Falls das Plugin sich nicht aktivieren lässt:
- Überprüfen Sie auf PHP-Fehler in Ihrem Fehlerprotokoll
- Vergewissern Sie sich, dass WooCommerce installiert und aktiv ist
- Überprüfen Sie die PHP-Version-Anforderungen
- Suchen Sie nach Plugin-Konflikten

### 4. Die URL Überprüfen {#4-verify-the-url}

Stellen Sie sicher, dass die POS so konfiguriert ist, dass sie sich mit der richtigen WordPress-Seite verbindet:
- Überprüfen Sie die URL in den POS-Einstellungen
- Vergewissern Sie sich, dass sie mit Ihrer WordPress-Installation übereinstimmt

### 5. Bei Bedarf Neu Installieren {#5-reinstall-if-necessary}

Falls das Plugin beschädigt ist:
1. Deaktivieren und löschen Sie das Plugin
2. Installieren Sie es erneut von WordPress.org
3. Aktivieren Sie das Plugin
