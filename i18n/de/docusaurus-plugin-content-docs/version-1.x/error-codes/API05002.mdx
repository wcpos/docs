# API05002: WCPOS-Plugin nicht gefunden {#api05002-wcpos-plugin-not-found}

## Was das bedeutet {#what-this-means}

Das WCPOS-Plugin ist auf Ihrer WordPress-Website nicht installiert oder nicht aktiv. Die POS-Anwendung benötigt dieses Plugin, um mit WooCommerce zu kommunizieren.

## Häufige Ursachen {#common-causes}

- **Plugin nicht installiert** — Das Plugin wurde nie installiert
- **Plugin deaktiviert** — Das Plugin wurde deaktiviert
- **Plugin gelöscht** — Die Plugin-Dateien wurden entfernt
- **Falsche Website** — Verbindung zu einer Website ohne WCPOS

## So beheben Sie das Problem {#how-to-fix}

### 1. Plugin installieren {#1-install-the-plugin}

Falls nicht installiert:
1. Gehen Sie zu WordPress-Administration → Plugins → Installieren
2. Suchen Sie nach "WCPOS"
3. Installieren und aktivieren Sie das Plugin

Oder von [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/) herunterladen

### 2. Das Plugin aktivieren {#2-activate-the-plugin}

Falls installiert, aber nicht aktiv:
1. Gehen Sie zu Plugins → Installierte Plugins
2. Suchen Sie "WCPOS"
3. Klicken Sie auf "Aktivieren"

### 3. Auf Fehler prüfen {#3-check-for-errors}

Falls sich das Plugin nicht aktivieren lässt:
- Prüfen Sie Ihr Fehlerprotokoll auf PHP-Fehler
- Stellen Sie sicher, dass WooCommerce installiert und aktiviert ist
- Prüfen Sie die PHP-Versionsanforderungen
- Suchen Sie nach Plugin-Konflikten

### 4. URL überprüfen {#4-verify-the-url}

Stellen Sie sicher, dass der POS für die Verbindung mit der richtigen WordPress-Website konfiguriert ist:
- Prüfen Sie die URL in den POS-Einstellungen
- Stellen Sie sicher, dass sie mit Ihrer WordPress-Installation übereinstimmt

### 5. Bei Bedarf neu installieren {#5-reinstall-if-necessary}

Wenn das Plugin beschädigt ist:
1. Deaktivieren und löschen Sie das Plugin
2. Installieren Sie es erneut von WordPress.org
3. Plugin aktivieren
