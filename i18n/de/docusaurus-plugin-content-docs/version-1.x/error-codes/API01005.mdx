# API01005: SSL-Zertifikatfehler

## Was das bedeutet

Es gibt ein Problem mit dem SSL/TLS-Zertifikat Ihrer Website. Dieses Zertifikat sorgt dafür, dass die Verbindung sicher ist (HTTPS). Das POS verbindet sich nicht mit Webseiten, die ungültige Zertifikate haben, um Ihre Daten zu schützen.

## Häufige Ursachen

- **Abgelaufenes Zertifikat** — SSL-Zertifikate müssen periodisch erneuert werden
- **Selbstsigniertes Zertifikat** — Nicht von einer vertrauenswürdigen Behörde ausgestellt
- **Falsche Domain** — Zertifikat stimmt nicht mit dem Domänennamen überein
- **Unvollständige Zertifikatskette** — Fehlende Zwischenzertifikate
- **Gemischte Inhalte** — Einige Ressourcen werden über HTTP anstelle von HTTPS geladen

## So beheben Sie das Problem

### 1. Zertifikatstatus überprüfen

Besuchen Sie Ihre Website in einem Browser und klicken Sie auf das Vorhängeschloss-Symbol, um die Zertifikatsdetails anzuzeigen. Achten Sie auf:
- Ablaufdatum
- Ausgestellt an (sollte mit Ihrer Domain übereinstimmen)
- Ausgestellt von (sollte eine anerkannte Behörde sein)

### 2. Abgelaufenes Zertifikat erneuern

Falls abgelaufen:
- Die meisten Hosting-Anbieter bieten kostenlose Let's Encrypt-Zertifikate an
- Kontaktieren Sie Ihren Hosting-Anbieter zur Erneuerung
- Überprüfen Sie, ob die automatische Erneuerung aktiviert ist

### 3. Zertifikatsfehler beheben

Stellen Sie sicher, dass das Zertifikat folgende abdeckt:
- Ihre genaue Domain (`yourstore.com`)
- WWW-Variante (`www.yourstore.com`), falls verwendet
- Erwägen Sie ein Wildcard-Zertifikat (`*.yourstore.com`)

### 4. Fehlende Zwischenzertifikate installieren

Verwenden Sie ein SSL-Checker-Tool (wie SSL Labs), um fehlende Zertifikate zu identifizieren. Ihr Hosting-Anbieter kann Ihnen bei der Installation helfen.

### 5. HTTPS erzwingen

Stellen Sie in WordPress sicher:
- Die Website-URL verwendet `https://`
- SSL erzwingen ist in den WooCommerce-Einstellungen aktiviert
