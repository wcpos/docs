# API01005: SSL-Zertifikatsfehler

## Was Dies Bedeutet

Es gibt ein Problem mit dem SSL/TLS-Zertifikat Ihrer Website. Dieses Zertifikat stellt eine sichere Verbindung (HTTPS) her. Das POS verbindet sich nicht mit Websites mit ungültigen Zertifikaten, um Ihre Daten zu schützen.

## Häufige Ursachen

- **Abgelaufenes Zertifikat** — SSL-Zertifikate müssen regelmäßig erneuert werden
- **Selbstsigniertes Zertifikat** — Nicht von einer vertrauenswürdigen Stelle ausgestellt
- **Falsche Domain** — Zertifikat stimmt nicht mit dem Domainnamen überein
- **Unvollständige Zertifikatskette** — Fehlende Zwischenzertifikate
- **Gemischte Inhalte** — Einige Ressourcen werden über HTTP statt HTTPS geladen

## Problemlösung

### 1. Zertifikatsstatus prüfen

Besuchen Sie Ihre Website in einem Browser und klicken Sie auf das Schloss-Symbol, um Zertifikatsdetails anzuzeigen. Achten Sie auf:
- Ablaufdatum
- Ausgestellt für (sollte Ihre Domain übereinstimmen)
- Ausgestellt von (sollte eine anerkannte Stelle sein)

### 2. Abgelaufenes Zertifikat erneuern

Bei Ablauf:
- Die meisten Hosting-Anbieter bieten kostenlose Let's Encrypt-Zertifikate
- Kontaktieren Sie Ihren Hosting-Anbieter zur Erneuerung
- Überprüfen Sie, ob automatische Erneuerung aktiviert ist

### 3. Zertifikatsdiskrepanz beheben

Stellen Sie sicher, dass das Zertifikat abdeckt:
- Ihre exakte Domain (`ihreseite.com`)
- WWW-Variante (`www.ihreseite.com`) falls verwendet
- Erwägen Sie ein Wildcard-Zertifikat (`*.ihreseite.com`)

### 4. Fehlende Zwischenzertifikate installieren

Verwenden Sie ein SSL-Prüftool (wie SSL Labs), um fehlende Zertifikate zu identifizieren. Ihr Hosting-Anbieter kann bei der Installation helfen.

### 5. HTTPS erzwingen

In WordPress sicherstellen:
- Website-URL verwendet `https://`
- SSL-Erzwingung in WooCommerce-Einstellungen aktiviert
