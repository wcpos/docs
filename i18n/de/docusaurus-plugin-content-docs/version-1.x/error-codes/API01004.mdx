# API01004: DNS-Übertragung fehlgeschlagen

## Was das bedeutet

Das POS konnte den Domainnamen Ihrer Website (z. B. `yourstore.com`) nicht in eine IP-Adresse umwandeln. DNS (Domain Name System) ist wie das Telefonbuch des Internets — wenn es den Eintrag nicht finden kann, kann es die Verbindung nicht herstellen.

## Häufige Ursachen

- **Tippfehler im Domainnamen** — Die URL könnte einen Schreibfehler enthalten
- **Probleme mit dem DNS-Server** — Ihr DNS-Server antwortet nicht
- **Domain abgelaufen** — Die Domainregistrierung könnte abgelaufen sein
- **DNS-Propagation** — Jüngste DNS-Änderungen sind möglicherweise noch nicht propagiert
- **Lokaler DNS-Cache** — Veraltete DNS-Informationen auf Ihrem Gerät zwischengespeichert

## So beheben Sie das Problem

### 1. Überprüfen Sie den Domainnamen

Überprüfen Sie die URL auf Tippfehler. Versuchen Sie, die Seite in einem Browser zu erreichen.

### 2. Überprüfen Sie den Status der Domain

Verwenden Sie eine WHOIS-Abfrage, um Folgendes zu überprüfen:
- Die Domain ist weiterhin registriert
- Sie ist nicht abgelaufen
- Die DNS-Einträge sind konfiguriert

### 3. DNS-Cache leeren

**Auf Ihrem Gerät:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Alternativen DNS ausprobieren

Wechseln Sie zu einem öffentlichen DNS-Server wie:
- Google: `8.8.8.8` und `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Warten auf DNS-Propagation

Wenn Sie kürzlich die DNS-Einstellungen geändert haben, warten Sie 24-48 Stunden auf die vollständige Propagation.
