# API01004: DNS-Auflösung fehlgeschlagen

## Was das bedeutet

Das POS konnte den Domainnamen Ihrer Website (z.B. `yourstore.com`) nicht in eine IP-Adresse umwandeln. DNS (Domain Name System) ist wie das Telefonbuch des Internets — wenn es den Eintrag nicht finden kann, kann es keine Verbindung herstellen.

## Häufige Ursachen

- **Tippfehler im Domainnamen** — Die URL enthält möglicherweise einen Rechtschreibfehler
- **DNS-Server-Probleme** — Ihr DNS-Server antwortet nicht
- **Domain abgelaufen** — Die Domainregistrierung ist möglicherweise abgelaufen
- **DNS-Propagierung** — Kürzliche DNS-Änderungen wurden noch nicht vollständig verteilt
- **Lokaler DNS-Cache** — Veraltete DNS-Informationen auf Ihrem Gerät

## Problemlösung

### 1. Domainnamen überprüfen

Überprüfen Sie die URL auf Tippfehler. Versuchen Sie, die Seite in einem Browser zu öffnen.

### 2. Domain-Status prüfen

Verwenden Sie eine WHOIS-Suche, um zu überprüfen:
- Die Domain ist noch registriert
- Sie ist nicht abgelaufen
- DNS-Einträge sind konfiguriert

### 3. DNS-Cache leeren

**Auf Ihrem Gerät:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Alternativen DNS-Server ausprobieren

Wechseln Sie zu einem öffentlichen DNS-Server wie:
- Google: `8.8.8.8` und `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Auf DNS-Propagierung warten

Wenn Sie kürzlich DNS-Einstellungen geändert haben, warten Sie 24-48 Stunden auf vollständige Verteilung.
