# API03003: Ungültiger Parameterwert

## Was Dies Bedeutet

Einer der Werte in Ihrer Anfrage ist ungültig. Das Datenformat, der Typ oder der Wert entspricht nicht den Erwartungen des Servers.

## Häufige Ursachen

- **Falscher Datentyp** — Text, wo eine Zahl erwartet wird (oder umgekehrt)
- **Außerhalb des Wertebereichs** — Wert überschreitet zulässige Grenzen
- **Ungültiges Format** — E-Mail, Telefon oder andere formatierte Felder sind fehlerhaft
- **Ungültige Referenz** — Verweist auf eine nicht existierende ID

## Server-Fehler-Zuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## Wie Sie Es Beheben Können

### 1. Eingabewerte Überprüfen

Überprüfen Sie die Daten, die Sie übermitteln:
- **Preise** — Sollten gültige Zahlen sein
- **Mengen** — Sollten positive Ganzzahlen sein
- **E-Mails** — Müssen gültiges E-Mail-Format haben
- **IDs** — Müssen auf vorhandene Datensätze verweisen

### 2. Nach Spezifischen Felderfehlern Suchen

Die Fehlermeldung kann angeben, welches Feld ungültig ist:
- Überprüfen Sie die Fehlermeldedetails
- Korrigieren Sie das spezifische Feld
- Wiederholen Sie die Aktion

### 3. Referenzierte Daten Überprüfen

Bei Verweis auf Produkte, Kunden oder andere Datensätze:
- Stellen Sie sicher, dass der Artikel in WooCommerce existiert
- Prüfen Sie, ob er kürzlich gelöscht wurde
- Synchronisieren Sie Daten, um lokale Datensätze zu aktualisieren

### 4. Feldeinschränkungen Prüfen

WooCommerce kann Einschränkungen haben:
- Maximal-/Minimalwerte
- Erforderliche Formate
- Erlaubte Optionen für Auswahlfelder

### 5. Löschen und Neu Eingeben

Manchmal werden Daten beschädigt:
- Löschen Sie das problematische Feld
- Geben Sie den Wert von Grund auf neu ein
- Vermeiden Sie das Kopieren aus anderen Quellen
