# API03003: Ungültiger Parameterwert

## Was das bedeutet

Einer der Werte in Ihrer Anfrage ist ungültig. Das Datenformat, der Typ oder der Wert entsprechen nicht den Erwartungen des Servers.

## Häufige Ursachen

- **Falscher Datentyp** — Text, wo eine Zahl erwartet wird (oder umgekehrt)
- **Außerhalb des Bereichs** — Wert überschreitet zulässige Grenzen
- **Ungültiges Format** — E-Mail, Telefon oder andere formatierte Felder sind inkorrekt
- **Ungültiger Verweis** — Verweis auf eine ID, die nicht existiert

## Serverfehlerzuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Servercode | Quelle |
|------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## So beheben Sie es

### 1. Überprüfen Sie die Eingabewerte

Überprüfen Sie die Daten, die Sie übermitteln:
- **Preise** — Sollten gültige Zahlen sein
- **Mengen** — Sollten positive ganze Zahlen sein
- **E-Mails** — Müssen im gültigen E-Mail-Format vorliegen
- **IDs** — Müssen auf vorhandene Datensätze verweisen

### 2. Suchen Sie nach spezifischen Feldfehlern

Der Fehler kann angeben, welches Feld ungültig ist:
- Überprüfen Sie die Details der Fehlermeldung
- Korrigieren Sie das spezifische Feld
- Versuchen Sie die Aktion erneut

### 3. Überprüfen Sie, ob verwiesene Daten existieren

Wenn Sie auf Produkte, Kunden oder andere Datensätze verweisen:
- Stellen Sie sicher, dass der Artikel in WooCommerce existiert
- Überprüfen Sie, ob er kürzlich gelöscht wurde
- Synchronisieren Sie die Daten, um lokale Datensätze zu aktualisieren

### 4. Überprüfen Sie die Feldbeschränkungen

WooCommerce kann Einschränkungen haben:
- Höchst-/Niedrigstwerte
- Erforderliche Formate
- Zulässige Optionen für Auswahlfelder

### 5. Löschen und neu eingeben

Manchmal werden Daten beschädigt:
- Löschen Sie das problematische Feld
- Geben Sie den Wert von Grund auf neu ein
- Vermeiden Sie das Kopieren und Einfügen aus anderen Quellen
