# DB01002: Abfrage-Zeitüberschreitung {#db01002-query-timeout}

## Was das bedeutet {#what-this-means}

Eine Datenbankoperation hat zu lange gedauert, um abgeschlossen zu werden. Die lokale Datenbankabfrage hat die maximal zulässige Zeit überschritten.

## Häufige Ursachen {#common-causes}

- **Großer Datensatz** — Sehr große Anzahl von Produkten oder Datensätzen
- **Komplexe Abfrage** — Abfrage, die viele Bedingungen enthält
- **Geräteleistung** — Gerät läuft langsam
- **Datenbankfragmentierung** — Datenbank benötigt Optimierung

## So beheben Sie es {#how-to-fix}

### 1. Warten und erneut versuchen {#1-wait-and-retry}

Die Abfrage kann bei einem erneuten Versuch erfolgreich sein:
- Schließen und öffnen Sie die POS erneut
- Versuchen Sie die Operation erneut
- Vermeiden Sie gleichzeitig andere ressourcenintensive Operationen

### 2. Schließen Sie andere Anwendungen {#2-close-other-applications}

Geben Sie Geräte-Ressourcen frei:
- Schließen Sie ungenutzte Browser-Tabs
- Schließen Sie andere Anwendungen
- Starten Sie das Gerät gegebenenfalls neu

### 3. Datenumfang reduzieren {#3-reduce-data-scope}

Wenn Sie suchen:
- Verwenden Sie spezifischere Suchbegriffe
- Wenden Sie Filter an, um die Ergebnisse einzugrenzen
- Brechen Sie große Operationen in kleinere auf

### 4. Datenbank optimieren {#4-optimise-the-database}

Löschen und erneut synchronisieren:
1. Lokale Daten löschen (Einstellungsoption, wenn verfügbar)
2. Melden Sie sich erneut an
3. Lassen Sie die Daten neu synchronisieren

### 5. Überprüfen Sie die Geräteleistung {#5-check-device-performance}

Stellen Sie sicher, dass Ihr Gerät die Anforderungen erfüllt:
- Ausreichend RAM
- Angemessene Prozessorgeschwindigkeit
- SSD bevorzugt gegenüber HDD

## Große Kataloge {#large-catalogues}

Wenn Sie einen sehr großen Produktkatalog haben:
- Die anfängliche Synchronisierung dauert länger
- Ziehen Sie in Betracht, Filter im POS zu verwenden
- Einige Operationen dauern von Natur aus länger
- Kontaktieren Sie den Support für Optimierungsberatung
