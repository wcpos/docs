# DB01002: Abfrage-Zeitüberschreitung

## Was Dies Bedeutet

Eine Datenbankoperation hat zu lange zur Ausführung gebraucht. Die lokale Datenbankabfrage hat die maximal zulässige Zeit überschritten.

## Häufige Ursachen

- **Großer Datensatz** — Sehr große Anzahl von Produkten oder Datensätzen
- **Komplexe Abfrage** — Abfrage mit vielen Bedingungen
- **Gerärteleistung** — Gerät läuft langsam
- **Datenbankfragmentierung** — Datenbank benötigt Optimierung

## Wie man es behebt

### 1. Warten und erneut versuchen

Die Abfrage kann beim erneuten Versuch erfolgreich sein:
- POS schließen und wieder öffnen
- Operation erneut versuchen
- Gleichzeitige intensive Operationen vermeiden

### 2. Andere Anwendungen schließen

Geräteressourcen freigeben:
- Ungenutzte Browser-Tabs schließen
- Andere Anwendungen schließen
- Gerät bei Bedarf neu starten

### 3. Datenumfang reduzieren

Bei der Suche:
- Spezifischere Suchbegriffe verwenden
- Filter zur Ergebniseinschränkung anwenden
- Große Operationen in kleinere unterteilen

### 4. Datenbank optimieren

Löschen und neu synchronisieren:
1. Lokale Daten löschen (falls Einstellung verfügbar)
2. Erneut anmelden
3. Daten neu synchronisieren lassen

### 5. Gerärteleistung überprüfen

Sicherstellen, dass Gerät Anforderungen erfüllt:
- Ausreichend RAM
- Ausreichende Prozessorgeschwindigkeit
- SSD bevorzugt gegenüber HDD

## Große Kataloge

Bei sehr großen Produktkatalogen:
- Initiale Synchronisation dauert länger
- Filter im POS verwenden
- Einige Operationen benötigen naturgemäß mehr Zeit
- Support für Optimierungshinweise kontaktieren
