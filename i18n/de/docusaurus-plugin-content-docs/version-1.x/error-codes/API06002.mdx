# API06002: Fehlende API-URL

## Was das bedeutet

Es wurde keine URL konfiguriert, um eine Verbindung zu Ihrem WooCommerce-Shop herzustellen. Das POS muss wissen, wo sich Ihr Geschäft befindet.

## Häufige Ursachen

- **Ersteinrichtung** — URL wurde nie eingegeben
- **Konfiguration gelöscht** — Einstellungen wurden zurückgesetzt oder gelöscht
- **App-Daten gelöscht** — Anwendungsdaten wurden gelöscht
- **Migrationsproblem** — URL während des Updates oder der Migration verloren

## So beheben Sie es

### 1. Geben Sie Ihre Shop-URL ein

In der POS-App:
1. Öffnen Sie die Einstellungen oder den Anmeldebildschirm
2. Suchen Sie das Feld für die URL/Website-Konfiguration
3. Geben Sie die URL Ihrer WordPress-Website ein
4. Beispiel: `https://yourstore.com`

### 2. Finden Sie Ihre WordPress-URL

Wenn Sie sich nicht sicher sind, was Ihre URL ist:
1. Melden Sie sich im WordPress-Admin-Bereich an
2. Gehen Sie zu Einstellungen → Allgemein
3. Schauen Sie nach "WordPress-Adresse (URL)"
4. Verwenden Sie diese URL im POS

### 3. Überprüfen Sie die gespeicherten Einstellungen

Wenn die URL zuvor konfiguriert war:
- Überprüfen Sie, ob die App-Daten versehentlich gelöscht wurden
- Suchen Sie nach Optionen für die Sicherung/Export der Einstellungen
- Geben Sie die URL bei Bedarf erneut ein

### 4. Überprüfen Sie, ob die URL korrekt ist

Vor dem Speichern:
1. Testen Sie die URL in einem Browser
2. Stellen Sie sicher, dass Ihre WordPress-Website geladen wird
3. Fügen Sie `https://` oder `http://` hinzu

## Einrichtungscheckliste

Bei der erstmaligen Konfiguration von WCPOS:

1. ☐ Geben Sie die URL Ihrer WordPress-Website ein
2. ☐ Überprüfen Sie, ob das WCPOS-Plugin installiert und aktiv ist
3. ☐ Melden Sie sich mit Ihren WordPress-Anmeldeinformationen an
4. ☐ Lassen Sie die erste Datensynchronisierung abschließen
