# API06002: Fehlende API-URL

## Was Das Bedeutet

Es wurde keine URL für die Verbindung zu Ihrem WooCommerce-Shop konfiguriert. Das POS-System muss wissen, wo sich Ihr Shop befindet.

## Häufige Ursachen

- **Ersteinrichtung** — URL wurde noch nie eingegeben
- **Konfiguration gelöscht** — Einstellungen wurden zurückgesetzt oder gelöscht
- **App-Daten gelöscht** — Anwendungsdaten wurden gelöscht
- **Migrationsproblem** — URL ging während Update oder Migration verloren

## Wie Man Es Behebt

### 1. Geben Sie Ihre Shop-URL Ein

In der POS-App:
1. Öffnen Sie Einstellungen oder Anmeldebildschirm
2. Suchen Sie das URL/Site-Konfigurationsfeld
3. Geben Sie Ihre WordPress-Site-URL ein
4. Beispiel: `https://ihrshop.com`

### 2. Finden Sie Ihre WordPress-URL

Wenn Sie sich nicht sicher sind:
1. Melden Sie sich an WordPress Admin an
2. Gehen Sie zu Einstellungen → Allgemein
3. Suchen Sie nach „WordPress-Adresse (URL)"
4. Verwenden Sie diese URL im POS

### 3. Gespeicherte Einstellungen Überprüfen

Wenn die URL zuvor konfiguriert war:
- Überprüfen Sie, ob App-Daten versehentlich gelöscht wurden
- Suchen Sie nach Einstellungs-Backup/Exportoptionen
- Geben Sie die URL bei Bedarf erneut ein

### 4. URL-Richtigkeit Überprüfen

Vor dem Speichern:
1. Testen Sie die URL im Browser
2. Stellen Sie sicher, dass Ihre WordPress-Site lädt
3. Fügen Sie `https://` oder `http://` hinzu

## Einrichtungs-Checkliste

Bei der Erstkonfiguration von WCPOS:

1. ☐ Geben Sie Ihre WordPress-Site-URL ein
2. ☐ Stellen Sie sicher, dass das WCPOS-Plugin installiert und aktiviert ist
3. ☐ Melden Sie sich mit Ihren WordPress-Zugangsdaten an
4. ☐ Initialen Datensync abschließen lassen
