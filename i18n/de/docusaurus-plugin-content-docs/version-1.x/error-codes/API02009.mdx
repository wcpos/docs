# API02009: Refresh-Token abgelaufen

## Was das bedeutet

Ihr Refresh-Token hat seine Lebensdauer überschritten und kann nicht mehr verwendet werden, um neue Zugriffstoken zu erhalten. Im Gegensatz zu Zugriffstokens (die schnell ablaufen), haben Refresh-Tokens typischerweise eine Lebensdauer von Tagen oder Wochen, laufen aber letztendlich ab.

## Häufige Ursachen

- **Längere Inaktivität** — Sie haben das POS längere Zeit nicht genutzt
- **Kurze Token-Lebensdauer** — Server ist mit kurzer Refresh-Token-Ablaufzeit konfiguriert
- **Zeitabweichung** — Geräte- oder Serverzeit weicht erheblich ab

## Wie Sie das beheben

### 1. Erneut Anmelden

Wenn der Refresh-Token abläuft, müssen Sie sich erneut authentifizieren:
1. Vom POS abmelden
2. Zugangsdaten eingeben
3. Ein neuer Refresh-Token wird ausgestellt

### 2. POS regelmäßig nutzen

Um dies in Zukunft zu vermeiden:
- Nutzen Sie das POS regelmäßig, um Token aktuell zu halten
- Token werden automatisch aktualisiert, wenn Sie aktiv sind
- Längere Inaktivitätszeiträume erfordern erneute Anmeldung

### 3. Zeiteinstellungen Überprüfen

Stellen Sie genaue Zeit auf allen Systemen sicher:
- Aktivieren Sie die automatische Zeitsynchronisation auf Ihrem Gerät
- Überprüfen Sie die Serverzeit
- Zeitunterschiede können vorzeitigen Ablauf verursachen

### 4. Token-Lebensdauer Anpassen (Fortgeschritten)

Wenn Token zu schnell ablaufen, kann ein Serveradministrator die Lebensdauer der Refresh-Tokens in den Authentifizierungseinstellungen anpassen.
