# API02009: Refresh-Token abgelaufen

## Was das bedeutet

Ihr Refresh-Token hat seine Lebensdauer überschritten und kann nicht mehr verwendet werden, um neue Zugriffstokens zu erhalten. Im Gegensatz zu Zugriffstokens (die schnell ablaufen), halten Refresh-Tokens in der Regel Tage oder Wochen, laufen aber schließlich ab.

## Häufige Ursachen

- **Erweiterte Inaktivität** — Sie haben das POS über einen längeren Zeitraum nicht genutzt
- **Kurze Token-Lebensdauer** — Server mit kurzer Ablaufzeit für das Refresh-Token konfiguriert
- **Uhrzeitabweichung** — Geräte- oder Serverzeit ist erheblich abweichend

## So beheben Sie das Problem

### 1. Melden Sie sich erneut an

Wenn das Refresh-Token abläuft, müssen Sie sich erneut authentifizieren:
1. Melden Sie sich vom POS ab
2. Geben Sie Ihre Anmeldedaten ein
3. Ein neues Refresh-Token wird ausgestellt

### 2. POS regelmäßig verwenden

Um dies in Zukunft zu vermeiden:
- Verwenden Sie das POS regelmäßig, um Tokens frisch zu halten
- Tokens werden automatisch erneuert, wenn Sie aktiv sind
- Längere Inaktivitätszeiten erfordern eine erneute Anmeldung

### 3. Zeiteinstellungen überprüfen

Stellen Sie die genaue Uhrzeit auf allen Systemen sicher:
- Aktivieren Sie die automatische Zeitsynchronisierung auf Ihrem Gerät
- Überprüfen Sie, ob die Serverzeit korrekt ist
- Zeitunterschiede können zu einer vorzeitigen Ablauffrist führen

### 4. Token-Lebensdauer anpassen (Erweitert)

Wenn Tokens zu schnell ablaufen, kann ein Serveradministrator die Lebensdauer des Refresh-Tokens in den Authentifizierungseinstellungen anpassen.

## Verwandte Fehler

- [API02002](./API02002) — Token abgelaufen
- [API02007](./API02007) — Token Refresh fehlgeschlagen
