# DB01001: Verbindung fehlgeschlagen

## Was dies bedeutet

Der POS konnte keine Verbindung zur lokalen Datenbank herstellen. WCPOS verwendet eine lokale Datenbank, um Produkte, Kunden und andere Daten für schnellen Zugriff und Offline-Funktionalität zu speichern.

## Häufige Ursachen

- **Speicher voll** — Gerät hat keinen Speicherplatz mehr
- **Datenbank beschädigt** — Lokale Datenbankdateien sind beschädigt
- **Berechtigungsproblem** — App hat keine Speicherberechtigungen
- **Browserspeicher deaktiviert** — IndexedDB ist deaktiviert (Webversion)

## Wie man es behebt

### 1. Speicherplatz überprüfen

Stellen Sie sicher, dass Ihr Gerät verfügbaren Speicher hat:
- Speicher freimachen, wenn Speicher voll ist
- Ungenutzte Apps oder Dateien löschen
- Verfügbaren Festplattenspeicher überprüfen

### 2. App-Daten löschen

Lokale Datenbank zurücksetzen:
- **Desktop-App:** Nach "Daten löschen" in den Einstellungen suchen
- **Webbrowser:** Sitedaten für die POS-Domain löschen
- **Hinweis:** Dies erfordert erneutes Synchronisieren aller Daten

### 3. Browser-Einstellungen überprüfen (Web)

Bei Verwendung der Webversion:
1. Sicherstellen, dass IndexedDB aktiviert ist
2. Keine privaten/Inkognito-Modi verwenden (begrenzter Speicher)
3. Der Website erlauben, Daten zu speichern

### 4. Berechtigungen überprüfen

Auf Desktop:
- Sicherstellen, dass die App Berechtigungen zum Schreiben in ihr Datenverzeichnis hat
- Prüfen, ob Antivirus den Datenbankzugriff blockiert

### 5. App neu installieren

Wenn andere Lösungen fehlschlagen:
1. POS-App deinstallieren
2. Gerät neu starten
3. App neu installieren
4. Anmelden und Daten synchronisieren

## Datenwiederherstellung

Wenn Sie ungespeicherte Daten haben:
- Die App kann ausstehende Änderungen haben, die nicht synchronisiert werden konnten
- Kontaktieren Sie den Support, wenn Sie Hilfe bei der Datenwiederherstellung benötigen
- In den meisten Fällen können Daten vom Server erneut synchronisiert werden
