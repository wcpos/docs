# API02001: Ungültige Anmeldeinformationen

## Was das bedeutet

Der von Ihnen eingegebene Benutzername oder das Passwort ist falsch. Der Server hat den Anmeldeversuch abgelehnt, weil die Anmeldeinformationen mit keinem gültigen Konto übereinstimmen.

## Häufige Ursachen

- **Tippfehler im Benutzernamen oder Passwort** — Überprüfen Sie die Feststelltaste oder zusätzliche Leerzeichen
- **Falsches Konto** — Verwendung von Anmeldeinformationen für eine andere Seite
- **Passwort kürzlich geändert** — Das Passwort wurde aktualisiert, aber das POS hat das alte
- **Konto gelöscht** — Das Benutzerkonto existiert nicht mehr

## Serverfehlerzuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Servercode | Quelle |
|------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT-Authentifizierungs-Plugin |

## So beheben Sie das Problem

### 1. Überprüfen Sie Ihre Anmeldeinformationen

- Überprüfen Sie Ihren Benutzernamen (normalerweise E-Mail oder WordPress-Benutzername)
- Stellen Sie sicher, dass das Passwort korrekt ist
- Versuchen Sie, sich direkt im WordPress-Admin-Bereich anzumelden, um dies zu bestätigen

### 2. Setzen Sie Ihr Passwort zurück

Wenn Sie Ihr Passwort vergessen haben:
1. Gehen Sie zu Ihrer WordPress-Login-Seite
2. Klicken Sie auf "Passwort vergessen?"
3. Geben Sie Ihre E-Mail ein, um einen Link zum Zurücksetzen zu erhalten
4. Aktualisieren Sie Ihr Passwort
5. Melden Sie sich mit dem neuen Passwort im POS an

### 3. Überprüfen Sie das Benutzerkonto

Im WordPress-Admin:
- Gehen Sie zu Benutzer → Alle Benutzer
- Überprüfen Sie, ob das Konto existiert
- Stellen Sie sicher, dass es die entsprechenden POS-Zugriffsberechtigungen hat

### 4. Gespeicherte Anmeldeinformationen löschen

Im POS:
- Melden Sie sich vollständig ab
- Löschen Sie alle gespeicherten/cached Anmeldeinformationen
- Geben Sie die Anmeldeinformationen erneut ein
