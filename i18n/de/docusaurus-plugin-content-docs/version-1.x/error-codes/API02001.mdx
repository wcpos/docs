# API02001: Ungültige Anmeldedaten

## Was Das Bedeutet

Der Benutzername oder das Passwort, das Sie eingegeben haben, ist falsch. Der Server hat den Anmeldeversuch abgelehnt, da die Anmeldedaten mit keinem gültigen Konto übereinstimmen.

## Häufige Ursachen

- **Tippfehler im Benutzernamen oder Passwort** — Überprüfen Sie die Feststelltaste oder Leerzeichen
- **Falsches Konto** — Verwendung von Anmeldedaten für eine andere Website
- **Passwort kürzlich geändert** — Das Passwort wurde aktualisiert, aber die POS hat das alte Passwort
- **Konto gelöscht** — Das Benutzerkonto existiert nicht mehr

## Server-Fehlerzuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server Folgendes zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT Authentication Plugin |

## Wie Sie Es Beheben

### 1. Überprüfen Sie Ihre Anmeldedaten

- Überprüfen Sie Ihren Benutzernamen (normalerweise E-Mail oder WordPress-Benutzername)
- Stellen Sie sicher, dass das Passwort korrekt ist
- Versuchen Sie, sich direkt im WordPress-Admin anzumelden, um es zu bestätigen

### 2. Passwort Zurücksetzen

Wenn Sie Ihr Passwort vergessen haben:
1. Gehen Sie zur WordPress-Anmeldeseite
2. Klicken Sie auf „Passwort vergessen?"
3. Geben Sie Ihre E-Mail ein, um einen Zurücksetzungslink zu erhalten
4. Aktualisieren Sie Ihr Passwort
5. Melden Sie sich mit dem neuen Passwort bei der POS an

### 3. Benutzerkonto Überprüfen

Im WordPress-Admin:
- Gehen Sie zu Benutzer → Alle Benutzer
- Überprüfen Sie, ob das Konto existiert
- Prüfen Sie, ob es über die entsprechenden POS-Zugriffsberechtigungen verfügt

### 4. Gespeicherte Anmeldedaten Löschen

In der POS:
- Vollständig abmelden
- Alle gespeicherten/zwischengespeicherten Anmeldeinformationen löschen
- Anmeldedaten neu eingeben
