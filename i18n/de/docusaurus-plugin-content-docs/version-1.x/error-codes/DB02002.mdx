# DB02002: Datensatz nicht gefunden

## Was das bedeutet

Der angeforderte Datensatz existiert nicht in der lokalen Datenbank. Das POS-System hat versucht, auf ein Produkt, einen Kunden, eine Bestellung oder einen anderen Datensatz zuzugreifen, der nicht lokal gespeichert ist.

## Häufige Ursachen

- **Noch nicht synchronisiert** — Daten wurden noch nicht heruntergeladen
- **Auf dem Server gelöscht** — Datensatz wurde in WooCommerce entfernt
- **Synchronisierung nicht abgeschlossen** — Erste Synchronisierung läuft noch
- **Filter angewendet** — Datensatz durch aktive Filter ausgeschlossen

## So beheben Sie das Problem

### 1. Warten Sie auf die Synchronisierung

Wenn kürzlich gestartet:
- Warten Sie auf den Abschluss der ersten Synchronisierung
- Überprüfen Sie den Fortschrittsindikator der Synchronisierung
- Der Datensatz sollte nach der Synchronisierung angezeigt werden

### 2. Aktualisieren/Synchronisieren

Lösen Sie eine Datenaktualisierung aus:
- Verwenden Sie die Aktualisierungs-/Synchronisierungsschaltfläche
- Ziehen Sie die neuesten Daten vom Server
- Warten Sie auf den Abschluss

### 3. Überprüfen Sie WooCommerce

Stellen Sie sicher, dass der Datensatz auf dem Server existiert:
- Melden Sie sich im WordPress-Adminbereich an
- Überprüfen Sie Produkte, Bestellungen oder Kunden
- Bestätigen Sie, dass der Artikel noch existiert

### 4. Überprüfen Sie die Filter

Der Datensatz könnte herausgefiltert sein:
- Überprüfen Sie die aktiven Filter im POS
- Überprüfen Sie die Kategoriefilter
- Bestätigen Sie die Sichtbarkeitseinstellungen

### 5. Löschen und neu synchronisieren

Wenn die Synchronisierung feststeckt:
1. Lokale Daten löschen
2. Wieder anmelden
3. Eine frische Synchronisierung abschließen

## Häufige Szenarien

- **Produkt wird nicht angezeigt:** Überprüfen Sie, ob es veröffentlicht und sichtbar ist
- **Kunde nicht gefunden:** Möglicherweise wurde er noch nicht synchronisiert
- **Bestellung fehlt:** Könnte außerhalb des Synchronisierungszeitraums liegen
