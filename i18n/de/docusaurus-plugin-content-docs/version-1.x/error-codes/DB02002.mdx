# DB02002: Datensatz nicht gefunden

## Was Das Bedeutet

Der angeforderte Datensatz existiert nicht in der lokalen Datenbank. Das POS-System versuchte, auf ein Produkt, einen Kunden, eine Bestellung oder einen anderen Datensatz zuzugreifen, der lokal nicht gespeichert ist.

## Häufige Ursachen

- **Noch nicht synchronisiert** — Daten wurden noch nicht heruntergeladen
- **Auf dem Server gelöscht** — Datensatz wurde in WooCommerce entfernt
- **Synchronisation nicht abgeschlossen** — Erste Synchronisation läuft noch
- **Filter angewendet** — Datensatz durch aktive Filter ausgeschlossen

## Behebung

### 1. Auf Synchronisation warten

Bei kürzlich gestarteter Anwendung:
- Warten Sie, bis die erste Synchronisation abgeschlossen ist
- Überprüfen Sie den Synchronisationsfortschritt
- Der Datensatz sollte nach der Synchronisation erscheinen

### 2. Aktualisieren/Neu-Synchronisieren

Lösen Sie eine Datenaktualisierung aus:
- Verwenden Sie die Aktualisieren/Synchronisieren-Taste
- Laden Sie die neuesten Daten vom Server
- Warten Sie auf den Abschluss

### 3. WooCommerce überprüfen

Überprüfen Sie, ob der Datensatz auf dem Server existiert:
- Melden Sie sich im WordPress-Admin an
- Überprüfen Sie Produkte, Bestellungen oder Kunden
- Bestätigen Sie, dass der Artikel noch existiert

### 4. Filter prüfen

Der Datensatz könnte herausgefiltert worden sein:
- Überprüfen Sie aktive Filter im POS
- Prüfen Sie Kategoriefilter
- Verifizieren Sie Sichtbarkeitseinstellungen

### 5. Löschen und Neu-Synchronisieren

Wenn die Synchronisation stecken geblieben ist:
1. Lokale Daten löschen
2. Erneut anmelden
3. Eine neue Synchronisation durchführen

## Häufige Szenarien

- **Produkt nicht sichtbar:** Überprüfen Sie, ob es veröffentlicht und sichtbar ist
- **Kunde nicht gefunden:** Möglicherweise noch nicht synchronisiert
- **Bestellung fehlt:** Könnte außerhalb des Synchronisationsdatumsbereichs liegen
