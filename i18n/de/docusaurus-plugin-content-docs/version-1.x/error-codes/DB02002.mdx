# DB02002: Datensatz nicht gefunden

## Was das bedeutet

Der angeforderte Datensatz existiert nicht in der lokalen Datenbank. Das POS hat versucht, auf ein Produkt, einen Kunden, eine Bestellung oder einen anderen Datensatz zuzugreifen, der nicht lokal gespeichert ist.

## Häufige Ursachen

- **Noch nicht synchronisiert** — Daten wurden noch nicht heruntergeladen
- **Auf dem Server gelöscht** — Datensatz wurde in WooCommerce entfernt
- **Sync nicht abgeschlossen** — Die erste Synchronisation ist noch im Gange
- **Filter angewendet** — Datensatz von aktiven Filtern ausgeschlossen

## So beheben Sie das Problem

### 1. Auf die Sync warten

Falls kürzlich gestartet:
- Warten Sie, bis die erste Sync abgeschlossen ist
- Überprüfen Sie den Fortschrittsindikator der Sync
- Der Datensatz sollte nach der Sync erscheinen

### 2. Aktualisieren/Re-sync

Ein Daten-Refresh auslösen:
- Verwenden Sie die Aktualisieren/Re-sync Taste (<Icon name="arrow-rotate-right" />) - kurzer Druck für die Sync
- **Lange drücken** Sie die Sync-Taste für die Option Löschen und Aktualisieren
- Ziehen Sie die neuesten Daten vom Server
- Warten Sie, bis die Verarbeitung abgeschlossen ist

### 3. WooCommerce überprüfen

Stellen Sie sicher, dass der Datensatz auf dem Server existiert:
- Melden Sie sich im WordPress-Admin an
- Überprüfen Sie Produkte, Bestellungen oder Kunden
- Bestätigen Sie, dass der Artikel noch existiert

### 4. Filter überprüfen

Der Datensatz könnte herausgefiltert sein:
- Überprüfen Sie die aktiven Filter im POS
- Überprüfen Sie die Kategoriefilter
- Überprüfen Sie die Sichtbarkeitseinstellungen

### 5. Löschen und Re-sync

Falls die Sync anscheinend feststeckt:
1. Lokale Daten löschen
2. Erneut anmelden
3. Eine frische Sync durchführen

## Häufige Szenarien

- **Produkt wird nicht angezeigt:** Überprüfen, ob es veröffentlicht und sichtbar ist
- **Kunde nicht gefunden:** Könnte noch nicht synchronisiert worden sein
- **Bestellung fehlt:** Könnte außerhalb des Sync-Zeitraums liegen
