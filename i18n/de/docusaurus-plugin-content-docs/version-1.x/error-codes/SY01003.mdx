# SY01003: Zugriff verweigert

## Was das bedeutet

Die POS-Anwendung hat keine Berechtigung, auf eine erforderliche Ressource zuzugreifen. Dies kann eine Datei, ein Ordner oder eine Systemfunktion sein.

## Häufige Ursachen

- **Dateiberechtigungen** — Kann nicht auf erforderliche Dateien lesen oder schreiben
- **App-Berechtigungen** — Anwendung verfügt nicht über erforderliche Berechtigungen
- **Geschützter Ordner** — Versucht, auf einen eingeschränkten Speicherort zuzugreifen
- **Sicherheitssoftware** — Antivirenprogramm blockiert den Zugriff

## Wie man es behebt

### 1. Als Administrator ausführen (Windows)

Rechtsklick auf die App und "Als Administrator ausführen" wählen, falls für die Ersteinrichtung erforderlich.

### 2. Anwendungsberechtigungen überprüfen

**macOS:**
- Systemeinstellungen → Sicherheit & Datenschutz → Datenschutz
- Überprüfen, ob die App die notwendigen Berechtigungen hat

**Windows:**
- Einstellungen → Datenschutz
- App-Berechtigungen überprüfen

### 3. Ordnerberechtigungen überprüfen

Sicherstellen, dass die App auf ihren Datenordner zugreifen kann:
- Der Benutzer sollte Besitzer des App-Datenverzeichnisses sein
- Schreibberechtigungen sollten aktiviert sein
- Keine schreibgeschützten Markierungen am Ordner

### 4. Antivireneinstellungen überprüfen

Sicherheitssoftware kann den Zugriff blockieren:
- Antivirenprotokolle auf Blockierungen prüfen
- POS-App zur Zulassungsliste hinzufügen
- App-Datenordner auf die Whitelist setzen

### 5. Anwendung neu installieren

Wenn Berechtigungen beschädigt sind:
1. POS-Anwendung deinstallieren
2. Gerät neustarten
3. An einem Standardspeicherort neu installieren
4. Mit normalen (Nicht-Admin-) Berechtigungen ausführen

## Browser-Berechtigungen (Webversion)

Bei Verwendung der Webversion:
- Lokalen Speicher für die Website erlauben
- Cookies für die Website aktivieren
- Nicht zu restriktive Browsereinstellungen verwenden
