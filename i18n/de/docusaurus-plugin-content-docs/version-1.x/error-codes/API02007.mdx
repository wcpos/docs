# API02007: Token-Aktualisierung fehlgeschlagen

## Was das bedeutet

Das POS-System hat versucht, Ihre Authentifizierungssitzung automatisch zu erneuern, aber die Aktualisierungsanfrage ist fehlgeschlagen. Dies bedeutet normalerweise, dass Sie sich erneut anmelden müssen.

## Häufige Ursachen

- **Erneuerungstoken abgelaufen** — Das Erneuerungstoken selbst ist abgelaufen
- **Erneuerungstoken widerrufen** — Ein Administrator hat alle Sitzungen ungültig gemacht
- **Serverfehler** — Der Server konnte die Aktualisierungsanfrage nicht verarbeiten
- **Netzwerkproblem** — Verbindungsproblem während des Aktualisierungsversuchs

## Wie man es behebt

### 1. Erneut anmelden

Die zuverlässigste Lösung:
1. Vom POS abmelden
2. Mit Ihren Anmeldedaten erneut anmelden
3. Dies erstellt eine neue Sitzung mit neuen Tokens

### 2. Netzwerkverbindung prüfen

Wenn die Aktualisierung aufgrund von Netzwerkproblemen fehlschlägt:
- Überprüfen Sie Ihre Internetverbindung
- Versuchen Sie, Ihre WordPress-Seite in einem Browser zu öffnen
- Warten Sie einen Moment und versuchen Sie es erneut

### 3. Serverstatus prüfen

Wenn der Server Probleme hat:
- Stellen Sie sicher, dass WordPress ordnungsgemäß läuft
- Überprüfen Sie PHP-Fehler in Ihren Serverprotokollen
- Stellen Sie sicher, dass das Authentifizierungs-Plugin aktiv ist

### 4. Sitzungsdaten löschen

Wenn Probleme bestehen bleiben:
- POS-App-Cache löschen
- Gespeicherte Anmeldedaten entfernen
- Mit einer neuen Anmeldung neu beginnen

## Automatische Token-Aktualisierung

WCPOS aktualisiert Token automatisch vor deren Ablauf, um Ihre Sitzung aufrechtzuerhalten. Wenn dieser Prozess wiederholt fehlschlägt, kann es ein Konfigurationsproblem geben, das Aufmerksamkeit erfordert.
