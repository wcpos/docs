# API02007: Token-Aktualisierung fehlgeschlagen

## Was Das Bedeutet

Der POS hat versucht, Ihre Authentifizierungssitzung automatisch zu erneuern, aber die Aktualisierungsanfrage ist fehlgeschlagen. Dies bedeutet in der Regel, dass Sie sich erneut anmelden müssen.

## Häufige Ursachen

- **Aktualisierungstoken abgelaufen** — Das Aktualisierungstoken selbst ist abgelaufen
- **Aktualisierungstoken widerrufen** — Ein Administrator hat alle Sitzungen ungültig gemacht
- **Serverfehler** — Der Server konnte die Aktualisierungsanfrage nicht verarbeiten
- **Netzwerkproblem** — Verbindungsproblem während des Aktualisierungsversuchs

## So Beheben Sie Es

### 1. Erneut Anmelden

Die zuverlässigste Lösung:
1. Melden Sie sich vom POS ab
2. Melden Sie sich mit Ihren Zugangsdaten wieder an
3. Dadurch wird eine neue Sitzung mit neuen Token erstellt

### 2. Netzwerkverbindung Überprüfen

Wenn die Aktualisierung aufgrund von Netzwerkproblemen fehlschlägt:
- Überprüfen Sie Ihre Internetverbindung
- Versuchen Sie, Ihre WordPress-Website in einem Browser zu öffnen
- Warten Sie einen Moment und versuchen Sie es erneut

### 3. Serverstatus Überprüfen

Wenn der Server Probleme hat:
- Stellen Sie sicher, dass WordPress ordnungsgemäß läuft
- Überprüfen Sie auf PHP-Fehler in Ihren Serverprotokollen
- Stellen Sie sicher, dass das Authentifizierungs-Plugin aktiv ist

### 4. Sitzungsdaten Löschen

Wenn die Probleme bestehen bleiben:
- Leeren Sie den Cache der POS-App
- Entfernen Sie gespeicherte Anmeldeinformationen
- Beginnen Sie mit einer neuen Anmeldung von vorne

## Automatische Token-Aktualisierung

WCPOS aktualisiert Token automatisch, bevor sie ablaufen, um Ihre Sitzung aufrechtzuerhalten. Wenn dieser Prozess wiederholt fehlschlägt, kann ein Konfigurationsproblem vorliegen, das Aufmerksamkeit erfordert.
