# API02007: Token-Aktualisierung fehlgeschlagen {#api02007-token-refresh-failed}

## Was Dies Bedeutet {#what-this-means}

Der POS hat versucht, Ihre Authentifizierungssitzung automatisch zu erneuern, aber die Refresh-Anfrage ist fehlgeschlagen. Dies bedeutet in der Regel, dass Sie sich erneut anmelden müssen.

## Häufige Ursachen {#common-causes}

- **Refresh-Token abgelaufen** — Der Refresh-Token selbst ist abgelaufen
- **Refresh-Token widerrufen** — Ein Administrator hat alle Sitzungen ungültig gemacht
- **Serverfehler** — Der Server konnte die Refresh-Anfrage nicht verarbeiten
- **Netzwerkproblem** — Verbindungsprobleme während des Refresh-Versuchs

## So Beheben Sie Es {#how-to-fix}

### 1. Erneut Anmelden {#1-log-in-again}

Die zuverlässigste Lösung:
1. Melden Sie sich vom POS ab
2. Melden Sie sich mit Ihren Zugangsdaten wieder an
3. Dadurch wird eine neue Sitzung mit neuen Tokens erstellt

### 2. Netzwerkverbindung Überprüfen {#2-check-network-connection}

Wenn der Refresh aufgrund von Netzwerkproblemen fehlschlägt:
- Überprüfen Sie Ihre Internetverbindung
- Versuchen Sie, auf Ihre WordPress-Site zuzugreifen
- Warten Sie einen Moment und versuchen Sie es erneut

### 3. Serverstatus Überprüfen {#3-check-server-status}

Wenn der Server Probleme hat:
- Stellen Sie sicher, dass WordPress ordnungsgemäß läuft
- Überprüfen Sie auf PHP-Fehler in Ihren Serverprotokollen
- Stellen Sie sicher, dass das Authentifizierungs-Plugin aktiv ist

### 4. Sitzungsdaten Löschen {#4-clear-session-data}

Wenn die Probleme bestehen bleiben:
- Leeren Sie den Cache der POS-App
- Entfernen Sie gespeicherte Anmeldeinformationen
- Starten Sie mit einer neuen Anmeldung von vorne

## Automatische Token-Aktualisierung {#automatic-token-refresh}

WCPOS aktualisiert Tokens automatisch, bevor sie ablaufen, um Ihre Sitzung aufrechtzuerhalten. Wenn dieser Prozess wiederholt fehlschlägt, kann ein Konfigurationsproblem vorliegen, das Aufmerksamkeit erfordert.
