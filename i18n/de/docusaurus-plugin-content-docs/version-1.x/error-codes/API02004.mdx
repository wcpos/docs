# API02004: Benutzer nicht autorisiert

## Was Dies Bedeutet

Sie sind angemeldet, aber Ihr Benutzerkonto verfügt nicht über die Berechtigung, die angeforderte Aktion auszuführen. Dies ist ein Autorisierungsproblem (was Sie tun können) und kein Authentifizierungsproblem (wer Sie sind).

## Häufige Ursachen

- **Einschränkungen der Benutzerrolle** — Ihre WordPress-Rolle umfasst keinen POS-Zugriff
- **POS-Zugriff deaktiviert** — Ihrem Konto wurden keine POS-Berechtigungen erteilt
- **Funktionsbeschränkungen** — Bestimmte Funktionen sind auf bestimmte Rollen beschränkt
- **Speicherbeschränkungen** — Sie haben möglicherweise keinen Zugriff auf diesen bestimmten Laden

## Server-Fehler-Zuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server Folgendes zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| `rest_cannot_view` | WordPress REST API |
| `woocommerce_rest_cannot_view` | WooCommerce REST API |

## So beheben Sie das Problem

### 1. Benutzerrolle überprüfen

In WordPress Admin → Benutzer:
1. Suchen Sie Ihr Benutzerkonto
2. Überprüfen Sie die Rolle (z. B. Shop-Manager, Administrator)
3. Stellen Sie sicher, dass die Rolle WooCommerce-Funktionen umfasst

### 2. POS-Zugriff aktivieren

In WordPress Admin → WooCommerce → POS → Zugriff:
1. Suchen Sie den Benutzer oder die Rolle
2. Aktivieren Sie POS-Zugriffsberechtigungen
3. Änderungen speichern

### 3. Zusätzliche Berechtigungen anfordern

Kontaktieren Sie den Store-Administrator, um:
- Ihrer Rolle POS-Zugriff zu gewähren
- Ihnen eine Rolle mit entsprechenden Berechtigungen zuzuweisen
- Bestimmte benötigte Funktionen zu aktivieren

### 4. Store-Zuweisung überprüfen

Bei Verwendung mehrerer Stores:
- Überprüfen Sie, ob Sie dem richtigen Store zugewiesen sind
- Überprüfen Sie Store-spezifische Berechtigungen

## Erforderliche Berechtigungen

Verschiedene Aktionen erfordern unterschiedliche Fähigkeiten:
- **Produkte anzeigen**: Lesezugriff auf Produkte
- **Bestellungen erstellen**: Erstellen/Bearbeiten von Bestellungsfunktionen
- **Kunden verwalten**: Kundenmanagement-Funktionen
- **Berichte aufrufen**: Berichtsfunktionen anzeigen
