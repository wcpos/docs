# PY02001: Zahlungsgateway-Fehler

## Was Dies Bedeutet

Bei der Kommunikation mit dem Zahlungsgateway ist ein Fehler aufgetreten. Der Zahlungsanbieter konnte die Anfrage nicht abschließen.

## Häufige Ursachen

- **Gateway-Ausfallzeit** — Der Zahlungsdienst hat Probleme
- **Konfigurationsfehler** — Gateway-Einstellungen sind falsch
- **API-Zugangsdaten** — Ungültige oder abgelaufene API-Schlüssel
- **Netzwerkprobleme** — Verbindungsprobleme zum Gateway

## Wie Sie Das Beheben

### 1. Transaktion Erneut Versuchen

Gateways haben manchmal vorübergehende Probleme:
- Warten Sie einen Moment
- Versuchen Sie, die Zahlung erneut zu verarbeiten
- Das Problem löst sich möglicherweise von selbst

### 2. Gateway-Status Überprüfen

Besuchen Sie die Statusseite Ihres Zahlungsanbieters:
- Stripe: status.stripe.com
- PayPal: PayPal-Community oder Status überprüfen
- Square: status.squareup.com
- Nach bekannten Ausfällen suchen

### 3. Gateway-Konfiguration Überprüfen

In WooCommerce Admin:
1. Zu WooCommerce → Einstellungen → Zahlungen gehen
2. Auf Ihr Zahlungsgateway klicken
3. API-Schlüssel auf Korrektheit prüfen
4. Sicherstellen, dass Test-/Live-Modus den Schlüsseln entspricht

### 4. API-Zugangsdaten Überprüfen

Stellen Sie sicher, dass Zugangsdaten gültig sind:
- API-Schlüssel nicht abgelaufen
- Live-Schlüssel für Live-Transaktionen verwenden
- Test-Schlüssel für Test-Transaktionen verwenden
- Zugangsdaten dem richtigen Konto zugeordnet

### 5. Mit Einem Anderen Gateway Testen

Falls verfügbar:
- Versuchen Sie, mit einem alternativen Gateway zu bezahlen
- Barzahlung als Ausweichlösung nutzen
- Verkauf abschließen und später abstimmen

## Für Ladenbesitzer

Häufige Konfigurationsprobleme:
- Vermischen von Test- und Live-API-Schlüsseln
- API-Schlüssel von verschiedenen Konten
- Deaktivierte Zahlungsmethoden
- Abgelaufene oder widerrufene Zugangsdaten

Kontaktieren Sie den Support Ihres Zahlungsanbieters, wenn das Problem weiterhin besteht.
