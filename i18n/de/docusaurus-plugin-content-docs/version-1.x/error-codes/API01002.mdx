# API01002: Verbindung verweigert

## Was das bedeutet

Der Server hat den Verbindungsversuch aktiv abgelehnt. Dies unterscheidet sich von einem Timeout — der Server hat geantwortet, aber "Nein" gesagt.

## Häufige Ursachen

- **Server ist offline** — Der Webserver (Apache/Nginx) läuft möglicherweise nicht
- **Falscher Port** — Verbindungsversuch über den falschen Port
- **Firewall blockiert** — Eine Firewall blockiert die Verbindung
- **IP-Einschränkungen** — Ihre IP-Adresse wurde möglicherweise gesperrt

## So beheben Sie das Problem

### 1. Serverstart überprüfen

Versuchen Sie, Ihre WordPress-Seite direkt im Browser aufzurufen. Wenn sie nicht lädt, kontaktieren Sie Ihren Hosting-Anbieter.

### 2. URL-Konfiguration prüfen

Stellen Sie sicher, dass Sie die korrekte URL verwenden:
- Überprüfen Sie `http://` vs `https://`
- Achten Sie auf Tippfehler im Domainnamen
- Bestätigen Sie den korrekten Port bei Verwendung eines nicht-standardmäßigen Ports

### 3. Firewall-Einstellungen überprüfen

Falls Sie Serverzugriff haben:
- Stellen Sie sicher, dass Port 80 (HTTP) oder 443 (HTTPS) geöffnet ist
- Überprüfen Sie, ob Ihre IP auf der Whitelist steht
- Überprüfen Sie Sicherheits-Plugins, die möglicherweise API-Zugriff blockieren

### 4. Hosting-Anbieter kontaktieren

Sie können Folgendes überprüfen:
- Serverstatus und Logs
- Firewall-Konfigurationen
- Ob Ihre IP gesperrt wurde
