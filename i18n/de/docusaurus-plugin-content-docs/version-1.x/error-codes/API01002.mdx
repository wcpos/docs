# API01002: Verbindung Verweigert

## Was Das Bedeutet

Der Server hat den Verbindungsversuch aktiv verweigert. Dies unterscheidet sich von einem Timeout — der Server hat geantwortet, aber "nein" gesagt.

## Häufige Ursachen

- **Server ist down** — Der Webserver (Apache/Nginx) läuft möglicherweise nicht
- **Falscher Port** — Versuch, sich auf den falschen Port zu verbinden
- **Firewall blockiert** — Eine Firewall blockiert die Verbindung
- **IP-Beschränkungen** — Ihre IP-Adresse könnte blockiert sein

## So Beheben Sie Es

### 1. Überprüfen Sie, ob der Server läuft

Versuchen Sie, Ihre WordPress-Website direkt in einem Browser aufzurufen. Wenn sie nicht lädt, kontaktieren Sie Ihren Hosting-Anbieter.

### 2. Überprüfen Sie die URL-Konfiguration

Stellen Sie sicher, dass Sie die korrekte URL verwenden:
- Überprüfen Sie `http://` vs `https://`
- Achten Sie auf Tippfehler im Domänennamen
- Bestätigen Sie den korrekten Port, wenn Sie einen nicht standardmäßigen verwenden

### 3. Überprüfen Sie die Firewall-Einstellungen

Wenn Sie Zugriff auf Ihren Server haben:
- Überprüfen Sie, ob Port 80 (HTTP) oder 443 (HTTPS) geöffnet ist
- Überprüfen Sie, ob Ihre IP auf der Whitelist steht
- Überprüfen Sie alle Sicherheits-Plugins, die den API-Zugriff blockieren könnten

### 4. Kontaktieren Sie Ihren Hosting-Anbieter

Sie können überprüfen:
- Serverstatus und Protokolle
- Firewall-Konfigurationen
- Ob Ihre IP blockiert wurde
