# API06001: Ungültige URL-Format

## Was Dies Bedeutet

Die für Ihre WooCommerce-Website konfigurierte URL entspricht nicht einem gültigen Format. URLs müssen den Standard-Webadress-Konventionen folgen.

## Häufige Ursachen

- **Fehlendes Protokoll** — URL beginnt nicht mit `http://` oder `https://`
- **Tippfehler in URL** — Rechtschreibfehler in der Domain
- **Ungültige Zeichen** — Sonderzeichen, die in URLs nicht erlaubt sind
- **Unvollständige URL** — URL fehlen Teile wie die Domainerweiterung

## Wie Sie Es Beheben

### 1. URL-Format Überprüfen

Eine gültige URL sollte so aussehen:
- ✅ `https://ihreseite.com`
- ✅ `https://www.ihreseite.com`
- ✅ `https://shop.ihredomain.com`
- ❌ `ihreseite.com` (fehlendes Protokoll)
- ❌ `https://ihreseite` (fehlende Erweiterung)
- ❌ `https://ihre seite.com` (enthält Leerzeichen)

### 2. Protokoll Hinzufügen

Fügen Sie immer `https://` oder `http://` hinzu:
- Bevorzugt: `https://` (sicher)
- Ausweichlösung: `http://` (wenn HTTPS nicht verfügbar)

### 3. Extra-Zeichen Entfernen

Stellen Sie sicher, dass die URL keine folgenden Elemente enthält:
- Abschließende Schrägstriche (optional, aber konsistent)
- Zusätzliche Leerzeichen
- Sonderzeichen
- URL-Parameter (es sei denn, sie sind erforderlich)

### 4. URL-Funktionalität Überprüfen

URL im Browser testen:
1. URL kopieren
2. In Browser einfügen
3. Es sollte Ihre WordPress-Website laden

### 5. POS-Konfiguration Aktualisieren

Korrigierte URL in den POS-Einstellungen eingeben:
- Exakt wie im Browser
- `www.` ein- oder ausschließen, wie Ihre Seite es verwendet

## Beispiele

| Ungültig | Gültig |
|----------|--------|
| `meineseite.com` | `https://meineseite.com` |
| `https://meine seite.com` | `https://meineseite.com` |
| `https://meineseite` | `https://meineseite.com` |
| `htp://meineseite.com` | `https://meineseite.com` |
