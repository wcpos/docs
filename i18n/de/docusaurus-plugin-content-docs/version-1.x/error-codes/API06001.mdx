# API06001: Ungültiges URL-Format {#api06001-invalid-url-format}

## Was Dies Bedeutet {#what-this-means}

Die für Ihre WooCommerce-Seite konfigurierte URL hat ein ungültiges Format. URLs müssen den standardmäßigen Konventionen für Webadressen folgen.

## Häufige Ursachen {#common-causes}

- **Fehlendes Protokoll** — URL beginnt nicht mit `http://` oder `https://`
- **Tippfehler in der URL** — Rechtschreibfehler in der Domain
- **Ungültige Zeichen** — Sonderzeichen, die in URLs nicht erlaubt sind
- **Unvollständige URL** — URL fehlen Teile wie die Domain-Erweiterung

## So Beheben {#how-to-fix}

### 1. Überprüfen Sie das URL-Format {#1-check-url-format}

Eine gültige URL sollte folgendermaßen aussehen:
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com` (fehlendes Protokoll)
- ❌ `https://yourstore` (fehlende Erweiterung)
- ❌ `https://your store.com` (enthält Leerzeichen)

### 2. Fügen Sie das Protokoll Hinzu {#2-include-the-protocol}

Fügen Sie immer `https://` oder `http://` hinzu:
- Bevorzugt: `https://` (sicher)
- Fallback: `http://` (wenn HTTPS nicht verfügbar ist)

### 3. Entfernen Sie Zusätzliche Zeichen {#3-remove-extra-characters}

Stellen Sie sicher, dass die URL keine hat:
- Nachgestellte Schrägstriche (optional, aber konsistent sein)
- Zusätzliche Leerzeichen
- Sonderzeichen
- URL-Parameter (es sei denn, sie sind erforderlich)

### 4. Überprüfen Sie, Ob Die URL Funktioniert {#4-verify-the-url-works}

Testen Sie die URL in einem Browser:
1. Kopieren Sie die verwendete URL
2. Fügen Sie sie in einen Browser ein
3. Sie sollte Ihre WordPress-Seite laden

### 5. Aktualisieren Sie Die POS-Konfiguration {#5-update-pos-configuration}

Geben Sie die korrigierte URL in den POS-Einstellungen ein:
- Genau das abgleichen, was in Ihrem Browser funktioniert
- `www.` je nach Nutzung Ihrer Seite einfügen oder weglassen

## Beispiele {#examples}

| Ungültig | Gültig |
|---------|-------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
