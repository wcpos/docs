# API06003: Ungültige Site-Konfiguration {#api06003-invalid-site-configuration}

## Was dies bedeutet {#what-this-means}

Die Site-Konfiguration ist ungültig oder unvollständig. Dies könnte falsche URLs, Authentifizierungseinstellungen oder andere Konfigurationsprobleme umfassen.

## Häufige Ursachen {#common-causes}

- **Teilweise Konfiguration** — Einige Einstellungen fehlen
- **Nicht übereinstimmende Einstellungen** — Die Konfiguration stimmt nicht mit der Site überein
- **Beschädigte Konfiguration** — Einstellungen wurden beschädigt
- **Änderungen an der Site** — Die Site wurde geändert, ohne die POS-Konfiguration zu aktualisieren

## So beheben Sie das Problem {#how-to-fix}

### 1. Überprüfen Sie alle Einstellungen {#1-review-all-settings}

Überprüfen Sie die vollständige Konfiguration:
- Die Site-URL ist korrekt
- Die Authentifizierung ist ordnungsgemäß konfiguriert
- Alle zusätzlichen Einstellungen sind korrekt

### 2. Neu konfigurieren von Grund auf {#2-reconfigure-from-scratch}

Wenn die Konfiguration beschädigt ist:
1. Löschen Sie alle gespeicherten Einstellungen
2. Starten Sie den Setup-Prozess erneut
3. Geben Sie eine neue Konfiguration ein

### 3. Überprüfen Sie die Site-Anforderungen {#3-check-site-requirements}

Überprüfen Sie, ob Ihre WordPress-Site die Anforderungen erfüllt:
- WordPress ist installiert und zugänglich
- WooCommerce ist installiert und aktiv
- WCPOS-Plugin ist installiert und aktiv
- Permalinks sind aktiviert (nicht "Einfach")

### 4. Testen Sie den Site-Zugriff {#4-test-site-access}

Überprüfen Sie, ob diese URLs in einem Browser funktionieren:
- `https://yoursite.com/` — Hauptsite
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. Überprüfen Sie die WordPress-Einstellungen {#5-check-wordpress-settings}

Im WordPress-Admin:
1. Einstellungen → Allgemein — URLs überprüfen
2. Einstellungen → Permalinks — Sicherstellen, dass nicht "Einfach"
3. WooCommerce → Status — Auf Probleme überprüfen

## Konfigurationsanforderungen {#configuration-requirements}

Eine gültige Konfiguration benötigt:
- **Site-URL** — Vollständige URL mit Protokoll
- **Authentifizierung** — Gültige Anmeldeinformationen oder API-Schlüssel
- **Berechtigungen** — Benutzer hat POS-Zugriff
- **Plugins** — Erforderliche Plugins sind aktiv
