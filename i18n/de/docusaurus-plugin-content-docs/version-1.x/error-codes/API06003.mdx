# API06003: Ungültige Standortkonfiguration

## Was Dies Bedeutet

Die Standortkonfiguration ist ungültig oder unvollständig. Dies kann fehlerhafte URLs, Authentifizierungseinstellungen oder andere Konfigurationsprobleme umfassen.

## Häufige Ursachen

- **Teilweise Konfiguration** — Einige Einstellungen fehlen
- **Nicht übereinstimmende Einstellungen** — Konfiguration stimmt nicht mit der Website überein
- **Beschädigte Konfiguration** — Einstellungen wurden beschädigt
- **Websiteänderungen** — Die Website wurde modifiziert, ohne die POS-Konfiguration zu aktualisieren

## Problemlösung

### 1. Alle Einstellungen überprüfen

Überprüfen Sie die vollständige Konfiguration:
- Site-URL ist korrekt
- Authentifizierung ist ordnungsgemäß konfiguriert
- Alle zusätzlichen Einstellungen sind korrekt

### 2. Von Grund auf neu konfigurieren

Wenn die Konfiguration beschädigt ist:
1. Alle gespeicherten Einstellungen löschen
2. Setup-Prozess erneut starten
3. Neue Konfiguration eingeben

### 3. Site-Anforderungen prüfen

Überprüfen Sie, ob Ihre WordPress-Website die Anforderungen erfüllt:
- WordPress ist installiert und zugänglich
- WooCommerce ist installiert und aktiv
- WCPOS-Plugin ist installiert und aktiv
- Permalinks sind aktiviert (nicht "Einfach")

### 4. Site-Zugriff testen

Überprüfen Sie, ob diese URLs im Browser funktionieren:
- `https://ihresite.com/` — Hauptseite
- `https://ihresite.com/wp-json/` — REST API
- `https://ihresite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. WordPress-Einstellungen prüfen

In WordPress Admin:
1. Einstellungen → Allgemein — URLs überprüfen
2. Einstellungen → Permalinks — Sicherstellen, dass nicht "Einfach"
3. WooCommerce → Status — Auf Probleme prüfen

## Konfigurationsanforderungen

Eine gültige Konfiguration benötigt:
- **Site-URL** — Vollständige URL mit Protokoll
- **Authentifizierung** — Gültige Anmeldedaten oder API-Schlüssel
- **Berechtigungen** — Benutzer hat POS-Zugriff
- **Plugins** — Erforderliche Plugins sind aktiv
