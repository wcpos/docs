# API02005: Unzureichende Berechtigungen

## Was Dies Bedeutet

Ihr Benutzerkonto verfügt nicht über die spezifischen WordPress-Fähigkeiten, die für diese Aktion erforderlich sind. Während Sie grundlegenden Zugriff haben, benötigt der von Ihnen durchgeführte Vorgang zusätzliche Berechtigungen.

## Häufige Ursachen

- **Eingeschränkte Benutzerrolle** — Ihre Rolle umfasst nicht alle benötigten Fähigkeiten
- **Fähigkeit nicht zugewiesen** — Einer Rolle fehlt eine bestimmte Fähigkeit
- **Plugin-Einschränkungen** — Ein Sicherheits-Plugin beschränkt Fähigkeiten
- **Probleme mit benutzerdefinierten Rollen** — Benutzerdefinierte Rollen fehlen möglicherweise Fähigkeiten

## Server-Fehler-Zuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server Folgendes zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| `rest_forbidden` | WordPress REST API |
| `rest_cannot_create` | WordPress REST API |
| `rest_cannot_edit` | WordPress REST API |
| `rest_cannot_delete` | WordPress REST API |
| `woocommerce_rest_cannot_create` | WooCommerce REST API |
| `woocommerce_rest_cannot_edit` | WooCommerce REST API |
| `woocommerce_rest_cannot_delete` | WooCommerce REST API |
| HTTP 403 | Beliebige Serverantwort (Fallback) |

## Wie Sie Dies Beheben

### 1. Erforderliche Fähigkeiten Überprüfen

Häufig benötigte Fähigkeiten für POS-Vorgänge:
- `manage_woocommerce` — Allgemeine WooCommerce-Verwaltung
- `edit_shop_orders` — Bestellungen erstellen und bearbeiten
- `edit_products` — Produktinformationen ändern
- `edit_users` — Kundenkonten verwalten

### 2. Benutzerrolle Aufwerten

Bitten Sie einen Administrator, eine leistungsfähigere Rolle zuzuweisen:
- **Shop-Manager** — Vollständiger WooCommerce-Zugriff
- **Administrator** — Vollständiger Website-Zugriff

### 3. Bestimmte Fähigkeiten Hinzufügen

Wenn Sie eine benutzerdefinierte Rolle benötigen, fügen Sie erforderliche Fähigkeiten hinzu:

```php
// Beispiel: POS-Fähigkeiten zu einer benutzerdefinierten Rolle hinzufügen
$role = get_role('ihre_benutzerdefinierte_rolle');
$role->add_cap('manage_woocommerce');
$role->add_cap('edit_shop_orders');
```

### 4. Plugin-Konflikte Prüfen

Einige Sicherheits- oder Rollen-Management-Plugins können Fähigkeiten einschränken:
- Plugin-Einstellungen überprüfen
- Nach Fähigkeitsfiltern suchen
- Vorübergehend deaktivieren zum Testen
