# API02006: API-Schlüssel ungültig

## Was das bedeutet

Der verwendete WooCommerce REST API-Schlüssel ist ungültig. API-Schlüssel werden für die Authentifizierung von Server zu Server verwendet, und der bereitgestellte Schlüssel stimmt mit keinem gültigen Schlüssel in WooCommerce überein.

## Häufige Ursachen

- **Schlüssel gelöscht** — Der API-Schlüssel wurde aus WooCommerce entfernt
- **Tippfehler im Schlüssel** — Der Schlüssel wurde falsch eingegeben
- **Falsches Schlüsselpaar** — Verbraucherschlüssel und Geheimnis stimmen nicht überein
- **Schlüssel von einer anderen Site** — Verwendung von Schlüsseln, die für eine andere Installation generiert wurden

## So beheben Sie das Problem

### 1. Bestätigen Sie den API-Schlüssel

In WordPress Admin → WooCommerce → Einstellungen → Erweitert → REST API:
1. Überprüfen Sie, ob Ihr API-Schlüssel existiert
2. Stellen Sie sicher, dass er nicht widerrufen wurde
3. Notieren Sie sich die Berechtigungen (lesen/schreiben/lesen-schreiben)

### 2. Neue API-Schlüssel generieren

Wenn der Schlüssel fehlt oder ungültig ist:
1. Gehen Sie zu WooCommerce → Einstellungen → Erweitert → REST API
2. Klicken Sie auf "Schlüssel hinzufügen"
3. Geben Sie eine Beschreibung ein (z.B. "WCPOS")
4. Wählen Sie den Benutzer aus
5. Wählen Sie die Berechtigungen "Lesen/Schreiben"
6. Klicken Sie auf "API-Schlüssel generieren"
7. **Kopieren Sie sowohl den Verbraucherschlüssel als auch das Verbrauchergeheimnis** (wird nur einmal angezeigt!)

### 3. POS-Konfiguration aktualisieren

Geben Sie die neuen API-Schlüssel im POS ein:
- Verbraucherschlüssel (beginnt mit `ck_`)
- Verbrauchergeheimnis (beginnt mit `cs_`)

### 4. Schlüsselberechtigungen überprüfen

Stellen Sie sicher, dass der Schlüssel über ausreichende Berechtigungen verfügt:
- **Lesen** — Nur Daten anzeigen
- **Schreiben** — Nur Daten ändern
- **Lesen/Schreiben** — Vollzugriff (empfohlen für POS)
