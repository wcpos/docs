# API02006: API-Key ungültig

## Was das bedeutet {#what-this-means}

Der verwendete WooCommerce REST API Key ist ungültig. API Keys werden für die Authentifizierung von Server zu Server verwendet, und der bereitgestellte Key stimmt mit keinem gültigen Key in WooCommerce überein.

## Häufige Ursachen {#common-causes}

- **Key gelöscht** — Der API Key wurde aus WooCommerce entfernt
- **Tippfehler im Key** — Der Key wurde falsch eingegeben
- **Falsches Key-Paar** — Consumer Key und Secret stimmen nicht überein
- **Key von einer anderen Site** — Verwendung von Keys, die für eine andere Installation generiert wurden

## So beheben Sie das Problem {#how-to-fix}

### 1. Den API Key überprüfen {#1-verify-the-api-key}

In WordPress Admin → WooCommerce → Settings → Advanced → REST API:
1. Überprüfen Sie, ob Ihr API Key existiert
2. Stellen Sie sicher, dass er nicht widerrufen wurde
3. Notieren Sie sich die Berechtigungen (read/write/read-write)

### 2. Neue API Keys generieren {#2-generate-new-api-keys}

Wenn der Key fehlt oder ungültig ist:
1. Gehen Sie zu WooCommerce → Settings → Advanced → REST API
2. Klicken Sie auf "Add key"
3. Geben Sie eine Beschreibung ein (z. B. "WCPOS")
4. Wählen Sie den Benutzer aus
5. Wählen Sie die Berechtigungen "Read/Write"
6. Klicken Sie auf "Generate API key"
7. **Kopieren Sie sowohl den Consumer Key als auch das Consumer Secret** (wird nur einmal angezeigt!)

### 3. POS-Konfiguration aktualisieren {#3-update-pos-configuration}

Geben Sie die neuen API Keys im POS ein:
- Consumer Key (beginnt mit `ck_`)
- Consumer Secret (beginnt mit `cs_`)

### 4. Key-Berechtigungen prüfen {#4-check-key-permissions}

Stellen Sie sicher, dass der Key über ausreichende Berechtigungen verfügt:
- **Read** — Nur Daten anzeigen
- **Write** — Nur Daten ändern
- **Read/Write** — Vollzugriff (empfohlen für POS)
