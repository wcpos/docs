# API02006: API-Schlüssel ungültig

## Was Das Bedeutet

Der verwendete WooCommerce REST-API-Schlüssel ist ungültig. API-Schlüssel werden für die Server-zu-Server-Authentifizierung verwendet, und der angegebene Schlüssel stimmt mit keinem gültigen Schlüssel in WooCommerce überein.

## Häufige Ursachen

- **Schlüssel gelöscht** — Der API-Schlüssel wurde aus WooCommerce entfernt
- **Schlüsseltippfehler** — Der Schlüssel wurde falsch eingegeben
- **Falsches Schlüsselpaar** — Consumer-Schlüssel und -Geheimnis stimmen nicht überein
- **Schlüssel von anderer Seite** — Verwendung von Schlüsseln, die für eine andere Installation generiert wurden

## Wie Man Es Behebt

### 1. API-Schlüssel überprüfen

In WordPress-Admin → WooCommerce → Einstellungen → Erweitert → REST-API:
1. Prüfen, ob der API-Schlüssel existiert
2. Verifizieren, dass er nicht widerrufen wurde
3. Berechtigungen notieren (lesen/schreiben/lesen-schreiben)

### 2. Neue API-Schlüssel generieren

Wenn der Schlüssel fehlt oder ungültig ist:
1. Zu WooCommerce → Einstellungen → Erweitert → REST-API gehen
2. Auf "Schlüssel hinzufügen" klicken
3. Eine Beschreibung eingeben (z.B. "WCPOS")
4. Benutzer auswählen
5. "Lesen/Schreiben"-Berechtigungen wählen
6. Auf "API-Schlüssel generieren" klicken
7. **Consumer-Schlüssel und Consumer-Geheimnis kopieren** (werden nur einmal angezeigt!)

### 3. POS-Konfiguration aktualisieren

Neue API-Schlüssel im POS eingeben:
- Consumer-Schlüssel (beginnt mit `ck_`)
- Consumer-Geheimnis (beginnt mit `cs_`)

### 4. Schlüsselberechtigungen prüfen

Ausreichende Berechtigungen sicherstellen:
- **Lesen** — Nur Daten anzeigen
- **Schreiben** — Daten nur ändern
- **Lesen/Schreiben** — Vollzugriff (für POS empfohlen)
