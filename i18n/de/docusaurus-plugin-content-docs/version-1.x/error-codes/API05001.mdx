# API05001: WooCommerce-API deaktiviert

## Was das bedeutet

Die WooCommerce-REST-API ist auf Ihrer Website deaktiviert. WCPOS benötigt die REST-API, um mit WooCommerce zu kommunizieren und auf Geschäftsdaten zuzugreifen.

## Häufige Ursachen

- **WooCommerce-Einstellung** — Die REST-API wurde absichtlich deaktiviert
- **Sicherheits-Plugin** — Ein Sicherheits-Plugin blockiert den API-Zugriff
- **Hosting-Einschränkung** — Ihr Hosting-Anbieter hat den REST-API-Zugriff deaktiviert
- **Permalink-Probleme** — Permalinks nicht für REST-API konfiguriert

## Behebung

### 1. WooCommerce-REST-API aktivieren

Im WordPress-Admin:
1. Gehen Sie zu WooCommerce → Einstellungen → Erweitert → REST-API
2. Stellen Sie sicher, dass die REST-API aktiviert ist
3. API-Schlüssel überprüfen

### 2. WordPress-REST-API überprüfen

Die WooCommerce-API hängt von der WordPress-REST-API ab:
1. Besuchen Sie `https://ihreseite.com/wp-json/` im Browser
2. Sollte JSON-Daten zurückgeben
3. Falls nicht, siehe [API05004](./API05004)

### 3. Sicherheits-Plugins überprüfen

Häufige Sicherheits-Plugins, die die API blockieren können:
- **Wordfence** — Firewall-Einstellungen prüfen
- **iThemes Security** — REST-API-Einstellungen überprüfen
- **All In One WP Security** — Firewall-Regeln überprüfen

REST-API-Endpunkte oder die POS-Anwendung auf die Whitelist setzen.

### 4. .htaccess überprüfen

Nach Regeln suchen, die den API-Zugriff blockieren:

```apache
# Regeln entfernen oder modifizieren, die /wp-json/ blockieren
# Sicherstellen, dass mod_rewrite aktiviert ist
```

### 5. Permalinks zurücksetzen

Manchmal müssen Permalink-Einstellungen aktualisiert werden:
1. Gehen Sie zu Einstellungen → Permalinks
2. Klicken Sie auf "Änderungen speichern" (auch ohne Änderungen)
3. Dies regeneriert Rewrite-Regeln

## Verwandte Fehler

- [API05004](./API05004) — WordPress-API deaktiviert
- [API02006](./API02006) — API-Schlüssel ungültig
