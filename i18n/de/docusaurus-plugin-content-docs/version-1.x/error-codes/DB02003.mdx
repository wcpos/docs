# DB02003: Einschränkungsverletzung

## Was Dies Bedeutet

Die Daten, die Sie speichern möchten, verstoßen gegen Datenbankregeln. Einschränkungen stellen die Datenintegrität sicher, indem sie Regeln darüber durchsetzen, welche Daten gespeichert werden können.

## Häufige Ursachen

- **Fehlende erforderliche Daten** — Ein Pflichtfeld ist leer
- **Ungültige Beziehung** — Verweis auf einen nicht existierenden Datensatz
- **Datentypkonflikt** — Falscher Datentyp für das Feld
- **Wert außerhalb des Bereichs** — Zahl überschreitet zulässige Grenzen

## Wie Sie Es Beheben

### 1. Überprüfen Sie Pflichtfelder

Stellen Sie sicher, dass alle erforderlichen Daten bereitgestellt werden:
- Kundeninformationen (falls erforderlich)
- Produktdetails
- Auftragsposition

### 2. Referenzen Überprüfen

Wenn der Fehler Beziehungen betrifft:
- Stellen Sie sicher, dass referenzierte Produkte existieren
- Überprüfen Sie, ob Kunden-IDs gültig sind
- Kategoriezuweisungen prüfen

### 3. Datenwerte Überprüfen

Auf ungültige Werte prüfen:
- Negative Mengen, wo nicht erlaubt
- Preise, die Grenzen überschreiten
- Ungültige Statuswerte

### 4. Letzte Daten Synchronisieren

Die referenzierten Daten könnten nicht synchron sein:
- Daten vom Server aktualisieren
- Synchronisation abwarten
- Vorgang erneut versuchen

### 5. Löschen und Erneut Versuchen

Wenn Daten beschädigt sind:
1. Problematisches Formular löschen
2. Daten erneut eingeben
3. Erneut absenden

## Typische Einschränkungsbeispiele

- **Menge muss positiv sein** — Keine 0 oder negativen Artikel hinzufügen
- **Preis muss numerisch sein** — Kein Text in Preisfeldern erlaubt
- **Kunde muss existieren** — Kein Auftrag an nicht existierenden Kunden

- [DB02001](./DB02001) — Doppelter Datensatz
- [DB03003](./DB03003) — Erforderliches Feld fehlt
