# API01006: Netzwerk nicht erreichbar

## Was dies bedeutet

Der POS kann keine Route zum Server herstellen. Dies ist ein Problem auf Netzwerkebene — die Datenpakete können keinen Weg zum Server finden.

## Häufige Ursachen

- **Netzwerkkonfigurationsprobleme** — Falsche Gateway- oder Routingeinstellungen
- **ISP-Probleme** — Ihr Internetdienstanbieter hat Probleme
- **VPN-Interferenz** — Ein VPN leitet Datenverkehr möglicherweise falsch
- **Server-Netzwerkprobleme** — Das Netzwerk des Servers ist möglicherweise ausgefallen

## Problembehebung

### 1. Überprüfen Sie Ihre Internetverbindung

- Versuchen Sie, andere Websites zu erreichen
- Starten Sie Ihren Router/Modem neu
- Überprüfen Sie, ob Sie mit dem Netzwerk verbunden sind

### 2. VPN vorübergehend deaktivieren

Bei Verwendung eines VPN:
- Trennen Sie die Verbindung und versuchen Sie es erneut
- Versuchen Sie einen anderen VPN-Server
- Überprüfen Sie, ob das VPN Zugriff auf den Serverstandort erlaubt

### 3. Kontaktieren Sie Ihren Internetdienstanbieter

Kontaktieren Sie Ihren Internetdienstanbieter, wenn:
- Andere Websites ebenfalls nicht erreichbar sind
- Das Problem nach dem Neustarten der Netzwerkgeräte weiterbesteht

### 4. Server-Status überprüfen

- Prüfen Sie, ob Ihr Hosting-Anbieter Netzwerkprobleme meldet
- Versuchen Sie, die Website über ein anderes Netzwerk (z.B. Mobilfunkdaten) zu erreichen
- Verwenden Sie Online-Tools, um zu überprüfen, ob die Website für alle ausgefallen ist

### 5. Firewalleinstellungen überprüfen

Stellen Sie sicher, dass die Firewall Ihres Geräts keine ausgehenden Verbindungen zum Server blockiert.
