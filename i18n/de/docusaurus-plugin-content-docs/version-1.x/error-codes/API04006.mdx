# API04006: Ressource nicht gefunden

## Was Dies Bedeutet

Der Server konnte die angeforderte Ressource nicht finden. Dies entspricht typischerweise einem HTTP-404-Statuscode und zeigt an, dass das Produkt, die Bestellung, der Kunde oder eine andere Ressource, auf die Sie zugreifen möchten, nicht existiert.

## Häufige Ursachen

- **Datensatz gelöscht** — Die Ressource wurde auf dem Server gelöscht
- **Falsche ID** — Es wird eine falsche oder veraltete ID verwendet
- **Synchronisationsprobleme** — Lokale Daten verweisen auf eine nicht mehr existierende Ressource
- **URL-Fehlkonfiguration** — Der API-Endpunkt ist falsch
- **Permalink-Probleme** — WordPress-Permalinks müssen aktualisiert werden

## Server-Fehlerzuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| `rest_no_route` | WordPress REST API |
| HTTP 404 | Beliebige Serverantwort |

## Behebung

### 1. Überprüfen Sie, ob die Ressource existiert

Stellen Sie sicher, dass die Ressource auf Ihrer WooCommerce-Website noch vorhanden ist:
- Melden Sie sich im WordPress-Admin an
- Navigieren Sie zum relevanten Bereich (Produkte, Bestellungen, Kunden)
- Suchen Sie das Element nach ID oder Namen

### 2. Lokale Daten aktualisieren

Wenn die Ressource serverseitig gelöscht wurde:
1. Öffnen Sie die POS-Einstellungen
2. Navigieren Sie zum relevanten Datenbereich
3. Lösen Sie eine Synchronisation/Aktualisierung aus
4. Das gelöschte Element sollte lokal entfernt werden

### 3. WordPress-Permalinks überprüfen

Wenn mehrere Ressourcen nicht gefunden werden:
1. Gehen Sie zu **Einstellungen → Permalinks** im WordPress-Admin
2. Klicken Sie auf **Änderungen speichern** (auch ohne Änderungen)
3. Dies aktualisiert die Permalink-Struktur

### 4. API-Routen überprüfen

Testen Sie die REST-API direkt:
```
https://ihreseite.com/wp-json/wc/v3/products
```
Wenn dies einen 404 zurückgibt, könnte ein Serverkonfigurationsproblem vorliegen.

### 5. Plugin-Konflikte überprüfen

Wenn REST-API-Routen fehlen:
1. Stellen Sie sicher, dass WooCommerce aktiv ist
2. Stellen Sie sicher, dass das WCPOS-Plugin aktiv ist
3. Deaktivieren Sie andere Plugins vorübergehend zum Testen
