# API03005: Rate Limit Überschreitet

## Was Das Bedeutet

Sie haben zu viele Anfragen an den Server in kurzer Zeit gesendet. Die Ratenbegrenzung schützt den Server vor Überlastung.

## Häufige Ursachen

- **Schnelle Synchronisationsvorgänge** — Mehrere Geräte synchronisieren gleichzeitig
- **Aggressives Aktualisieren** — Daten zu häufig aktualisieren
- **Sicherheitsplugin** — Ein Plugin setzt Anfragelimits durch
- **Hostinglimits** — Ihr Hosting-Plan hat API-Ratenlimits

## Serverfehlerzuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Servercode | Quelle |
|------------|--------|
| HTTP 429   | Jede Serverantwort (Fallback) |

## So Fixen Sie Es

### 1. Warten und Erneut Versuchen

Ratenlimits sind vorübergehend:
- Warten Sie 1-5 Minuten
- Das POS wird automatisch erneut versuchen
- Vermeiden Sie manuelles wiederholtes Aktualisieren

### 2. Reduzieren Sie Gleichzeitige Vorgänge

Wenn Sie mehrere POS-Terminals verwenden:
- Versetzen Sie die anfängliche Synchronisation auf verschiedene Geräte
- Vermeiden Sie, alle Geräte auf einmal zu aktualisieren
- Lassen Sie eines abschließen, bevor Sie ein anderes starten

### 3. Überprüfen Sie Sicherheitsplugins

Plugins wie Wordfence können schnelle Anfragen blockieren:
- Setzen Sie die POS-Anwendung auf die Whitelist
- Setzen Sie die IPs Ihrer POS-Geräte auf die Whitelist
- Passen Sie die Ratenbegrenzungsschwellenwerte an

### 4. Überprüfen Sie Hostinglimits

Einige Hosts begrenzen API-Anfragen:
- Überprüfen Sie die Einzelheiten Ihres Hosting-Plans
- Ziehen Sie in Betracht, ein Upgrade durchzuführen, wenn die Grenzen zu niedrig sind
- Kontaktieren Sie den Support, um die Limits zu erhöhen

### 5. Optimieren Sie die Synchronisationsfrequenz

In den POS-Einstellungen:
- Passen Sie die automatischen Aktualisierungsintervalle an
- Verwenden Sie manuelle Synchronisation, wenn angebracht
- Vermeiden Sie unnötige Datenaktualisierungen
