# API03005: Ratenlimit überschritten

## Was Das Bedeutet

Sie haben zu viele Anfragen in kurzer Zeit an den Server gesendet. Die Ratenbegrenzung schützt den Server vor Überlastung.

## Häufige Ursachen

- **Schnelle Synchronisationsvorgänge** — Mehrere Geräte synchronisieren gleichzeitig
- **Aggressive Aktualisierung** — Zu häufiges Aktualisieren von Daten
- **Sicherheits-Plugin** — Ein Plugin erzwingt Anfragelimits
- **Hosting-Limits** — Ihr Hosting-Tarif hat API-Ratenlimits

## Server-Fehler-Zuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| HTTP 429 | Beliebige Serverantwort (Fallback) |

## Wie Sie Es Beheben

### 1. Warten und Erneut Versuchen

Ratenlimits sind temporär:
- 1-5 Minuten warten
- Die POS wird automatisch erneut versuchen
- Vermeiden Sie manuelles Wiederholen

### 2. Gleichzeitige Vorgänge Reduzieren

Bei Nutzung mehrerer POS-Terminals:
- Synchronisation über Geräte verteilen
- Vermeiden Sie gleichzeitiges Aktualisieren
- Einen Vorgang abschließen, bevor der nächste beginnt

### 3. Sicherheits-Plugins Überprüfen

Plugins wie Wordfence können schnelle Anfragen blockieren:
- POS-Anwendung auf die Whitelist setzen
- POS-Geräte-IPs auf die Whitelist setzen
- Ratenlimit-Schwellenwerte anpassen

### 4. Hosting-Limits Überprüfen

Einige Hosts begrenzen API-Anfragen:
- Hosting-Tarif-Details überprüfen
- Upgrade erwägen, falls Limits zu niedrig
- Support bezüglich Erhöhung der Limits kontaktieren

### 5. Synchronisationshäufigkeit Optimieren

In POS-Einstellungen:
- Auto-Aktualisierungsintervalle anpassen
- Manuelle Synchronisation verwenden
- Unnötige Datenaktualisierungen vermeiden
