# DB03002: Ungültiger Datentyp

## Was Dies Bedeutet

Der Datentyp stimmt nicht mit den Erwartungen der Datenbank überein. Zum Beispiel wurde Text eingegeben, wo eine Zahl erwartet wurde.

## Häufige Ursachen

- **Benutzereingabefehler** — Text in einem numerischen Feld eingeben
- **Import-Probleme** — Importierte Daten haben falsches Format
- **Datenbeschädigung** — Werte während der Übertragung korrumpiert
- **Plugin-Konflikt** — Ein anderes Plugin hat Datentypen verändert

## Wie Sie Es Beheben

### 1. Überprüfen Sie Ihre Eingabe

Überprüfen Sie die Daten, die Sie eingeben:
- **Preise** — Sollten Zahlen sein (z.B. `19.99`)
- **Mengen** — Sollten ganze Zahlen sein (z.B. `5`)
- **IDs** — Sollten numerisch sein

### 2. Löschen und Neu Eingeben

Wenn Daten beschädigt wurden:
- Feld löschen
- Wert manuell neu eingeben
- Vermeiden Sie das Kopieren aus externen Quellen

### 3. Quelldaten Überprüfen

Beim Synchronisieren von WooCommerce:
- Daten in WordPress Admin überprüfen
- Nach falsch formatierten Feldern suchen
- Daten an der Quelle korrigieren

### 4. Daten Erneut Synchronisieren

Frische Daten vom Server holen:
- Lokalen Cache löschen
- Daten erneut synchronisieren
- Prüfen, ob das Problem behoben ist

### 5. Nach Plugin-Konflikten Suchen

Bei Verwendung anderer WooCommerce-Plugins:
- Sie können Daten auf unerwartete Weise verändern
- Vorübergehend deaktivieren zum Testen
- Inkompatibilitäten melden

## Häufige Beispiele

| Feld | Erwartet | Ungültig |
|-------|----------|---------|
| Preis | `19.99` | `$19.99` |
| Menge | `5` | `fünf` |
| SKU | `ABC123` | — (jedes Format OK) |
| Produkt-ID | `42` | `produkt-42` |
