# DB03002: Ungültiger Datentyp {#db03002-invalid-data-type}

## Was Das Bedeutet {#what-this-means}

Der Datentyp stimmt nicht mit dem überein, was die Datenbank erwartet. Beispielsweise wurde Text bereitgestellt, wo eine Zahl erwartet wurde.

## Häufige Ursachen {#common-causes}

- **Benutzereingabefehler** — Eingabe von Text in ein numerisches Feld
- **Importprobleme** — Importierte Daten haben das falsche Format
- **Datenkorruption** — Werte während des Transfers beschädigt
- **Plugin-Konflikt** — Ein anderes Plugin hat Datentypen geändert

## So Beheben Sie Es {#how-to-fix}

### 1. Überprüfen Sie Ihre Eingabe {#1-check-your-input}

Überprüfen Sie die Daten, die Sie eingeben:
- **Preise** — Sollten Zahlen sein (z.B. `19.99`)
- **Mengen** — Sollten ganze Zahlen sein (z.B. `5`)
- **IDs** — Sollten numerisch sein

### 2. Löschen und Wieder Eingeben {#2-clear-and-re-enter}

Wenn Daten beschädigt waren:
- Löschen Sie das Feld
- Geben Sie den Wert manuell erneut ein
- Vermeiden Sie das Kopieren und Einfügen aus externen Quellen

### 3. Überprüfen Sie die Quelldaten {#3-check-source-data}

Wenn Sie von WooCommerce synchronisieren:
- Überprüfen Sie die Daten im WordPress-Admin
- Suchen Sie nach falsch formatierten Feldern
- Beheben Sie die Daten an der Quelle

### 4. Daten Neu Synchronisieren {#4-re-sync-data}

Holen Sie frische Daten vom Server:
- Leeren Sie den lokalen Cache
- Synchronisieren Sie die Daten erneut
- Überprüfen Sie, ob das Problem behoben wird

### 5. Überprüfen Sie auf Plugin-Konflikte {#5-check-for-plugin-conflicts}

Wenn Sie andere WooCommerce-Plugins verwenden:
- Diese können Daten auf unerwartete Weise ändern
- Vorübergehend deaktivieren, um zu testen
- Inkompatibilitäten melden

## Häufige Beispiele {#common-examples}

| Feld | Erwartet | Ungültig |
|-------|----------|---------|
| Preis | `19.99` | `$19.99` |
| Menge | `5` | `fünf` |
| SKU | `ABC123` | — (jedes Format OK) |
| Produkt-ID | `42` | `produkt-42` |
