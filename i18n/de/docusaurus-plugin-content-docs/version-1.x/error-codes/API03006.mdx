# API03006: Nicht unterstützte Methode

## Was Dies Bedeutet

Die verwendete HTTP-Methode (GET, POST, PUT, DELETE, etc.) wird für diesen Endpunkt nicht unterstützt. Der Server akzeptiert diese Art von Anfrage nicht für diese URL.

## Häufige Ursachen

- **Versionskonflikt** — POS und Plugin-Versionen sind nicht kompatibel
- **Endpunkt entfernt** — Ein API-Endpunkt wurde veraltet
- **Serverkonfiguration** — Webserver blockiert bestimmte HTTP-Methoden
- **Plugin-Konflikt** — Ein anderes Plugin verändert das REST-API-Verhalten

## Behebung

### 1. Komponenten aktualisieren

Kompatibilität sicherstellen:
- WCPOS WordPress-Plugin aktualisieren
- POS-Anwendung aktualisieren
- Versionshinweise auf Breaking Changes prüfen

### 2. Serverkonfiguration prüfen

Manche Server blockieren bestimmte HTTP-Methoden:
- Sicherstellen, dass PUT- und DELETE-Methoden erlaubt sind
- `.htaccess` auf Methodenbeschränkungen prüfen
- Nginx-Konfiguration überprüfen

### 3. REST-API-Zugriff verifizieren

WordPress REST-API testen:
1. `https://yoursite.com/wp-json/` im Browser aufrufen
2. Es sollten JSON-Daten zurückgegeben werden
3. Falls nicht, ist die REST-API möglicherweise deaktiviert oder blockiert

### 4. Plugin-Konflikte überprüfen

Andere Plugins vorübergehend deaktivieren:
- Sicherheits-Plugins können Methoden blockieren
- Andere REST-API-Plugins können Konflikte verursachen
- Einzeln wieder aktivieren, um die Ursache zu finden

### 5. Hosting-Einschränkungen überprüfen

Manche Hosts beschränken HTTP-Methoden:
- Hosting-Support kontaktieren
- Anfragen, alle Standardmethoden zu aktivieren
- Bei zu restriktiven Bedingungen Hostanbieterwechsel in Betracht ziehen
