# API01003: Verbindung zurückgesetzt {#api01003-connection-reset}

## Was Das Bedeutet {#what-this-means}

Die Verbindung zum Server wurde hergestellt, aber dann unerwartet geschlossen. Dies passiert typischerweise mitten in der Kommunikation, wenn etwas die Verbindung unterbricht.

## Häufige Ursachen {#common-causes}

- **Serverabsturz** — Der Serverprozess könnte während der Anfrage abgestürzt sein
- **Speicherlimits** — PHP ist der Speicher ausgegangen und wurde beendet
- **Timeout mitten in der Anfrage** — Der Server hat beim Verarbeiten ein Timeout erreicht
- **Netzwerkinstabilität** — Intermittierende Netzwerkprobleme
- **Sicherheitssoftware** — Aggressive Sicherheitseinstellungen schließen Verbindungen

## So Beheben Sie Es {#how-to-fix}

### 1. Überprüfen Sie die Serverfehlerprotokolle {#1-check-server-error-logs}

Schauen Sie in Ihre WordPress-Fehlerprotokolle oder das Hosting-Kontrollpanel auf verwandte Fehler zur Zeit des Zurücksetzens.

### 2. Erhöhen Sie das PHP-Speicherlimit {#2-increase-php-memory-limit}

In Ihrer `wp-config.php`:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. Überprüfen Sie auf Plugin-Konflikte {#3-check-for-plugin-conflicts}

Ein Plugin könnte PHP zum Absturz bringen:
- Deaktivieren Sie vorübergehend die Plugins
- Aktivieren Sie sie nacheinander wieder, um den Übeltäter zu identifizieren

### 4. Überprüfen Sie die Serverressourcen {#4-review-server-resources}

Kontaktieren Sie Ihren Hosting-Anbieter, um Folgendes zu prüfen:
- PHP-Prozesslimits
- Speicherverbrauch zur Zeit des Fehlers
- Protokolle zur Serverstabilität

### 5. Testen Sie die Netzwerkstabilität {#5-test-network-stability}

Wenn Sie WLAN nutzen, versuchen Sie eine kabelgebundene Verbindung. Wenn das Problem in verschiedenen Netzwerken weiterhin besteht, liegt es wahrscheinlich am Server.
