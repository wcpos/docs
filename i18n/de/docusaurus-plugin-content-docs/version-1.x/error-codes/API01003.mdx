# API01003: Verbindungsabbruch

## Was Das Bedeutet

Die Verbindung zum Server wurde hergestellt, aber dann unerwartet geschlossen. Dies geschieht typischerweise während der Kommunikation, wenn etwas die Verbindung unterbricht.

## Häufige Ursachen

- **Serverabsturz** — Der Serverprozess ist möglicherweise während der Anfrage abgestürzt
- **Speicherlimits** — PHP hat den Arbeitsspeicher erschöpft und wurde beendet
- **Timeout während der Anfrage** — Der Server hat beim Verarbeiten das Zeitlimit überschritten
- **Netzwerkinstabilität** — Intermittierende Netzwerkprobleme
- **Sicherheitssoftware** — Aggressive Sicherheitseinstellungen, die Verbindungen schließen

## Fehlerbehebung

### 1. Serverfehlerlogs überprüfen

Schauen Sie in Ihre WordPress-Fehlerprotokolle oder das Hosting-Kontrollpanel nach verwandten Fehlern um den Zeitpunkt des Verbindungsabbruchs.

### 2. PHP-Speicherlimit erhöhen

In Ihrer `wp-config.php`:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. Plugin-Konflikte überprüfen

Ein Plugin könnte PHP zum Absturz bringen:
- Plugins vorübergehend deaktivieren
- Nach und nach wieder aktivieren, um den Verursacher zu finden

### 4. Serverressourcen überprüfen

Kontaktieren Sie Ihren Hosting-Anbieter, um zu prüfen:
- PHP-Prozessgrenzen
- Speichernutzung zum Zeitpunkt des Fehlers
- Serverstabilitätsprotokolle

### 5. Netzwerkstabilität testen

Versuchen Sie bei WLAN eine kabelgebundene Verbindung. Wenn das Problem auf verschiedenen Netzwerken auftritt, ist es wahrscheinlich serverseitig.
