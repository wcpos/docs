# API01003: Verbindung zurückgesetzt

## Was Das Bedeutet

Die Verbindung zum Server wurde hergestellt, dann jedoch unerwartet geschlossen. Dies passiert typischerweise mitten in der Kommunikation, wenn etwas die Verbindung unterbricht.

## Häufige Ursachen

- **Serverabsturz** — Der Serverprozess könnte während der Anfrage abgestürzt sein
- **Speicherlimits** — PHP ist der Speicher ausgegangen und wurde beendet
- **Timeout während der Anfrage** — Der Server hat beim Verarbeiten ein Timeout erreicht
- **Netzwerkinstabilität** — Intermittierende Netzwerkprobleme
- **Sicherheitssoftware** — Aggressive Sicherheitseinstellungen schließen Verbindungen

## So Beheben Sie Es

### 1. Überprüfen Sie die Serverfehlerprotokolle

Überprüfen Sie Ihre WordPress-Fehlerprotokolle oder das Hosting-Kontrollpanel auf verwandte Fehler zur Zeit des Zurücksetzens.

### 2. Erhöhen Sie das PHP-Speicherlimit

In Ihrer `wp-config.php`:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. Überprüfen Sie auf Plugin-Konflikte

Ein Plugin könnte den PHP-Absturz verursachen:
- Deaktivieren Sie vorübergehend die Plugins
- Aktivieren Sie sie nacheinander wieder, um den Übeltäter zu finden

### 4. Überprüfen Sie die Serverressourcen

Kontaktieren Sie Ihren Hosting-Anbieter, um Folgendes zu überprüfen:
- PHP-Prozesslimits
- Speicherverbrauch zur Zeit des Fehlers
- Serverstabilitätsprotokolle

### 5. Testen Sie die Netzwerkstabilität

Wenn Sie WiFi nutzen, versuchen Sie eine kabelgebundene Verbindung. Wenn das Problem in verschiedenen Netzwerken weiterhin besteht, liegt es wahrscheinlich am Server.
