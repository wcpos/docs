# API02010: Auth Required

## Was das bedeutet

Die Aktion, die Sie ausführen möchten, erfordert eine Authentifizierung, aber Sie sind derzeit nicht angemeldet. Das POS benötigt gültige Anmeldeinformationen, um auf diese Ressource zuzugreifen.

## Häufige Ursachen

- **Nicht eingeloggt** – Sie haben sich noch nicht authentifiziert
- **Sitzung zurückgesetzt** – Ihre Sitzung wurde gelöscht oder ist abgelaufen
- **Zugriff auf geschützte Ressource** – Die Ressource erfordert eine Authentifizierung
- **App-Daten zurückgesetzt** – Gespeicherte Anmeldeinformationen wurden entfernt

## Serverfehlerzuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server Folgendes zurückgibt:

| Servercode | Quelle |
|------------|--------|
| `rest_login_required` | WordPress REST API |
| `jwt_auth_no_auth_header` | JWT-Authentifizierungs-Plugin |
| HTTP 401 | Jede Serverantwort (Fallback) |

## So beheben Sie das

### 1. Anmelden

Wenn Sie sich nicht angemeldet haben:
1. Öffnen Sie den Anmeldebildschirm des POS
2. Geben Sie Ihre WordPress-Anmeldeinformationen ein
3. Schließen Sie den Authentifizierungsprozess ab

### 2. Sitzungsstatus überprüfen

Wenn Sie dachten, Sie wären angemeldet:
- Ihre Sitzung könnte abgelaufen sein
- Suchen Sie nach [API02002](./API02002) (Token Expired) für weitere Details
- Melden Sie sich erneut an, um den Zugriff wiederherzustellen

### 3. Serverkonfiguration überprüfen

Stellen Sie sicher, dass die API-Endpunkte ordnungsgemäß konfiguriert sind:
- WooCommerce REST API sollte aktiviert sein
- WCPOS-Plugin sollte aktiv sein
- Authentifizierungsendpunkte sollten zugänglich sein

### 4. Auf Browser/App-Probleme prüfen

Wenn Sie unerwartet abgemeldet werden:
- Löschen Sie den Browser-Cache (falls Sie die Webversion verwenden)
- Überprüfen Sie, ob Cookies/der lokale Speicher blockiert werden
- Stellen Sie sicher, dass die App die Berechtigung hat, Daten zu speichern

## Was erfordert eine Authentifizierung?

Die meisten POS-Operationen erfordern eine Authentifizierung:
- Produkte und Kunden anzeigen
- Bestellungen erstellen und bearbeiten
- Zahlungen verarbeiten
- Auf Berichte zugreifen

Nur der erste Anmeldebildschirm ist ohne Authentifizierung zugänglich.

## Verwandte Fehler

- [API02001](./API02001) — Ungültige Anmeldeinformationen
- [API02002](./API02002) — Token Expired
