# API02010: Authentifizierung erforderlich

## Was dies bedeutet

Die Aktion, die Sie durchführen möchten, erfordert eine Authentifizierung, aber Sie sind derzeit nicht angemeldet. Die POS-Anwendung benötigt gültige Anmeldedaten, um auf diese Ressource zuzugreifen.

## Häufige Ursachen

- **Nicht angemeldet** — Sie haben sich noch nicht authentifiziert
- **Sitzung gelöscht** — Ihre Sitzung wurde gelöscht oder ist abgelaufen
- **Zugriff auf geschützte Ressource** — Für die Ressource ist eine Authentifizierung erforderlich
- **App-Daten gelöscht** — Gespeicherte Anmeldedaten wurden entfernt

## Server-Fehlerzuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server Folgendes zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| `rest_login_required` | WordPress REST-API |
| `jwt_auth_no_auth_header` | JWT-Authentifizierungs-Plugin |
| HTTP 401 | Beliebige Server-Antwort (Fallback) |

## Wie man es behebt

### 1. Anmelden

Wenn Sie sich noch nicht angemeldet haben:
1. Öffnen Sie den POS-Anmeldebildschirm
2. Geben Sie Ihre WordPress-Anmeldedaten ein
3. Schließen Sie den Authentifizierungsprozess ab

### 2. Sitzungsstatus überprüfen

Wenn Sie dachten, Sie seien angemeldet:
- Ihre Sitzung ist möglicherweise abgelaufen
- Weitere Informationen finden Sie unter [API02002](./API02002) (Token abgelaufen)
- Melden Sie sich erneut an, um den Zugriff wiederherzustellen

### 3. Serverkonfiguration überprüfen

Stellen Sie sicher, dass die API-Endpunkte korrekt konfiguriert sind:
- WooCommerce REST-API sollte aktiviert sein
- WCPOS-Plugin sollte aktiv sein
- Authentifizierungs-Endpunkte sollten erreichbar sein

### 4. Browser-/App-Probleme prüfen

Wenn Sie unerwartet abgemeldet werden:
- Browser-Cache löschen (bei Webversion)
- Überprüfen, ob Cookies/lokaler Speicher nicht blockiert werden
- Sicherstellen, dass die App Berechtigung zum Speichern von Daten hat

## Was Authentifizierung erfordert

Die meisten POS-Vorgänge erfordern eine Authentifizierung:
- Produkte und Kunden anzeigen
- Aufträge erstellen und bearbeiten
- Zahlungen abwickeln
- Berichte aufrufen

Nur der erste Anmeldebildschirm ist ohne Authentifizierung zugänglich.

## Verwandte Fehler

- [API02001](./API02001) — Ungültige Anmeldedaten
- [API02002](./API02002) — Token abgelaufen
