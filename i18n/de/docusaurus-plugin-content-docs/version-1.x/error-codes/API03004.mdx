# API03004: Anfrage zu groß

## Was Dies Bedeutet

Die Anfrage, die Sie senden, überschreitet die Größenbeschränkungen des Servers. Dies tritt typischerweise auf, wenn zu viele Daten auf einmal gesendet werden.

## Häufige Ursachen

- **Große Batchvorgänge** — Versuch, zu viele Datensätze gleichzeitig zu synchronisieren
- **Große Bilder** — Hochladen von überdimensionierten Bildern
- **Zu viele Artikel** — Bestellung mit extrem vielen Positionen
- **Serverlimits** — PHP oder Webserver haben niedrige Upload-Grenzen

## Behebung

### 1. Anfragegröße reduzieren

Bei Datensynchronisation:
- Versuchen Sie, in kleineren Batches zu synchronisieren
- Die Kasse sollte dies automatisch handhaben
- Warten Sie, bis die aktuelle Synchronisation abgeschlossen ist, bevor Sie eine neue starten

### 2. Bildgrößen überprüfen

Beim Bildupload:
- Bilder vor dem Upload verkleinern
- Komprimierte Formate verwenden (JPEG statt BMP)
- Die meisten Produktbilder funktionieren gut unter 1 MB

### 3. Große Bestellungen aufteilen

Wenn eine Bestellung sehr viele Artikel hat:
- Erwägen Sie die Aufteilung in mehrere Bestellungen
- Dies ist bei normaler Kassennutzung selten

### 4. Serverlimits erhöhen

Kontaktieren Sie Ihren Hosting-Anbieter oder bearbeiten Sie PHP-Einstellungen:

```php
// In php.ini oder .htaccess
upload_max_filesize = 64M
post_max_size = 64M
max_input_vars = 5000
```

### 5. Webserver-Limits überprüfen

Nginx oder Apache können eigene Limits haben:
- `client_max_body_size` für Nginx
- `LimitRequestBody` für Apache
