# API02002: Token Abgelaufen {#api02002-token-expired}

## Was Das Bedeutet {#what-this-means}

Ihre Authentifizierungssitzung ist abgelaufen. Aus Sicherheitsgründen dauern Anmeldesitzungen nicht ewig. Das POS muss Ihre Authentifizierung aktualisieren oder Sie müssen sich möglicherweise erneut anmelden.

## Häufige Ursachen {#common-causes}

- **Lange Inaktivität** — Sie haben das POS längere Zeit nicht verwendet
- **Server-Token-Lifetime** — Die Ablaufzeit des Tokens auf dem Server ist sehr kurz eingestellt
- **Serverzeit-Abgleich** — Die Uhren von Server und Gerät sind nicht synchron

## Serverfehlerzuordnung {#server-error-mapping}

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Servercode | Quelle |
|------------|--------|
| `jwt_auth_expired_token` | JWT Authentication-Plugin |

## So Beheben Sie Es {#how-to-fix}

### 1. Melden Sie Sich Wieder An {#1-log-in-again}

Die einfachste Lösung ist, sich abzumelden und wieder anzumelden:
1. Melden Sie sich vom POS ab
2. Geben Sie Ihre Anmeldedaten erneut ein
3. Dies erstellt eine neue Sitzung

### 2. Überprüfen Sie Die Token-Aktualisierung {#2-check-token-refresh}

Das POS sollte Token automatisch aktualisieren. Wenn es nicht funktioniert:
- Überprüfen Sie Ihre Internetverbindung
- Vergewissern Sie sich, dass der Server zugänglich ist
- Siehe [API02007](./API02007), wenn die Aktualisierung fehlschlägt

### 3. Überprüfen Sie Die Serverzeit {#3-check-server-time}

Die Token-Ablaufzeit hängt von einer genauen Uhrzeit ab:
- Stellen Sie sicher, dass die Uhrzeit des Servers korrekt ist
- Überprüfen Sie, dass die Uhrzeit Ihres Geräts genau ist
- Verwenden Sie die automatische Zeitsynchronisierung auf beiden

### 4. Überprüfen Sie Die Token-Einstellungen {#4-review-token-settings}

Wenn Tokens zu schnell ablaufen, müssen Sie möglicherweise die Servereinstellungen anpassen (fragen Sie Ihren Hosting-Anbieter oder WordPress-Administrator).

## Verwandte Fehler {#related-errors}

- [API02007](./API02007) — Token-Aktualisierung fehlgeschlagen
- [API02009](./API02009) — Aktualisierungstoken abgelaufen
