# API02002: Token abgelaufen

## Was das bedeutet

Ihre Authentifizierungssitzung ist abgelaufen. Aus Sicherheitsgründen sind Anmeldesitzungen nicht ewig gültig. Die POS muss Ihre Authentifizierung aktualisieren oder Sie müssen sich möglicherweise erneut anmelden.

## Häufige Ursachen

- **Lange Inaktivität** — Sie haben das POS über einen längeren Zeitraum nicht genutzt
- **Servertokenlaufzeit** — Die Tokenablaufzeit des Servers ist sehr kurz eingestellt
- **Zeitabweichung des Servers** — Serveruhr und Geräteuhr sind nicht synchron

## Server-Fehler-Zuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Servercode | Quelle |
|-------------|--------|
| `jwt_auth_expired_token` | JWT-Authentifizierungs-Plugin |

## Wie man es behebt

### 1. Erneut anmelden

Die einfachste Lösung ist, sich ab- und wieder anzumelden:
1. Vom POS abmelden
2. Zugangsdaten erneut eingeben
3. Damit wird eine neue Sitzung erstellt

### 2. Token-Aktualisierung prüfen

Der POS sollte Token automatisch aktualisieren. Falls dies nicht funktioniert:
- Internetverbindung überprüfen
- Sicherstellen, dass der Server erreichbar ist
- Konsultieren Sie [API02007](./API02007), wenn die Aktualisierung fehlschlägt

### 3. Serverzeit prüfen

Die Tokenablaufzeit hängt von der korrekten Zeit ab:
- Sicherstellen, dass die Serveruhr korrekt ist
- Überprüfen, ob die Zeit des Geräts stimmt
- Automatische Zeitsynchronisation auf beiden aktivieren

### 4. Token-Einstellungen überprüfen

Wenn Token zu schnell ablaufen, müssen möglicherweise Servereinstellungen angepasst werden (Konsultieren Sie Ihren Hosting-Anbieter oder WordPress-Administrator).

## Verwandte Fehler

- [API02007](./API02007) — Token-Aktualisierung fehlgeschlagen
- [API02009](./API02009) — Aktualisierungs-Token abgelaufen
