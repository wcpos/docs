# API04001: Ungültiges Antwortformat

## Was Dies Bedeutet

Der Server hat geantwortet, aber das Antwortformat entspricht nicht den Erwartungen des POS. Der Server sollte JSON-Daten zurückgeben, aber etwas anderes wurde empfangen.

## Häufige Ursachen

- **PHP-Fehler angezeigt** — Ein PHP-Fehler wird vor dem JSON ausgegeben
- **Plugin-Konflikt** — Ein anderes Plugin gibt Inhalt aus
- **Wartungsmodus** — Die Seite zeigt eine Wartungsseite
- **Falscher Inhaltstyp** — Server sendet HTML statt JSON
- **Caching-Problem** — Eine zwischengespeicherte Fehlerseite wird ausgegeben

## Behebung

### 1. Überprüfen Sie Ihre Website

Besuchen Sie Ihre WordPress-Website in einem Browser:
- Wird sie normal angezeigt?
- Gibt es sichtbare Fehler?
- Ist sie im Wartungsmodus?

### 2. Auf PHP-Fehler prüfen

In `wp-config.php` das Debugging temporär aktivieren:

```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```

Überprüfen Sie `wp-content/debug.log` auf Fehler.

### 3. REST API direkt testen

Besuchen Sie `https://yoursite.com/wp-json/` in Ihrem Browser:
- Sollte JSON-Daten zurückgeben
- Wenn Sie HTML oder Fehler sehen, liegt ein Problem vor
- Überprüfen Sie auf pluginbedingte Ausgaben

### 4. Caching temporär deaktivieren

Caching-Plugins können veraltete Antworten liefern:
- Alle Caches leeren
- Caching-Plugins vorübergehend deaktivieren
- REST API von Caching ausnehmen

### 5. Plugin-Konflikte prüfen

Wenn ein Plugin auf jeder Seite Inhalt ausgibt:
1. Alle nicht wesentlichen Plugins deaktivieren
2. POS testen
3. Plugins einzeln wieder aktivieren
