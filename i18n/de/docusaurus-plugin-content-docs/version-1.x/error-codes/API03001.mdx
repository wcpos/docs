# API03001: Ungültige Anforderungsformat

## Was Dies Bedeutet

Die an den Server gesendete Anforderung hatte nicht das erwartete Format. Der Server konnte nicht verstehen, was das POS anfragte, da die Anforderungsstruktur nicht korrekt war.

## Häufige Ursachen

- **Beschädigte Daten** — Daten wurden vor dem Senden beschädigt
- **Softwarefehler** — Ein Problem in der POS-Anwendung
- **Proxy-Interferenz** — Ein Proxy oder eine Firewall hat die Anforderung verändert
- **Zeichenkodierungsprobleme** — Sonderzeichen wurden nicht richtig kodiert

## Server-Fehler-Zuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| HTTP 400 | Beliebige Serverantwort (Fallback) |

## So Beheben Sie Das

### 1. Aktion Wiederholen

Manchmal handelt es sich um einen einmaligen Fehler:
- Warten Sie einen Moment und versuchen Sie es erneut
- Aktualisieren Sie das POS und wiederholen Sie den Vorgang

### 2. Auf Sonderzeichen Prüfen

Wenn Sie Daten mit Sonderzeichen eingeben:
- Versuchen Sie, Emojis oder ungewöhnliche Symbole zu entfernen
- Verwenden Sie Standardzeichen für Produktnamen usw.

### 3. POS Aktualisieren

Stellen Sie sicher, dass Sie die neueste Version ausführen:
- Prüfen Sie auf App-Updates
- Aktualisieren Sie das WCPOS-Plugin auf Ihrem Server

### 4. Netzwerkkonfiguration Prüfen

Wenn Sie sich hinter einem Proxy befinden:
- Überprüfen Sie, ob der Proxy die Anforderungen nicht verändert
- Überprüfen Sie Firewallregeln
- Versuchen Sie, von einem anderen Netzwerk aus zuzugreifen

### 5. Problem Melden

Wenn dies konsistent auftritt:
- Notieren Sie, welche Aktion den Fehler auslöst
- Überprüfen Sie die Browserkonsole auf Details (Webversion)
- Melden Sie es auf [GitHub](https://github.com/wcpos) mit Reproduktionsschritten
