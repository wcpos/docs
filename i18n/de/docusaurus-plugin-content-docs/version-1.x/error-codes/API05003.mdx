# API05003: WCPOS-Plugin Veraltet {#api05003-wcpos-plugin-outdated}

## Was das Bedeutet {#what-this-means}

Das auf Ihrem Server installierte WCPOS-Plugin ist veraltet und nicht mit Ihrer Version der POS-Anwendung kompatibel. Sie müssen das Plugin aktualisieren.

## Häufige Ursachen {#common-causes}

- **Plugin nicht aktualisiert** — Automatische Updates sind möglicherweise deaktiviert
- **Update fehlgeschlagen** — Ein vorheriger Aktualisierungsversuch ist fehlgeschlagen
- **Versionsinkonsistenz** — POS-App aktualisiert, aber Plugin nicht
- **Staging-/Entwicklungsseite** — Testen mit einer älteren Version

## So beheben Sie es {#how-to-fix}

### 1. Plugin aktualisieren {#1-update-the-plugin}

In WordPress Admin:
1. Gehe zu Plugins → Installierte Plugins
2. Finden Sie "WooCommerce POS"
3. Wenn ein Update verfügbar ist, klicken Sie auf "Jetzt aktualisieren"

### 2. Automatische Updates aktivieren {#2-enable-auto-updates}

Um zukünftige Inkonsistenzen zu verhindern:
1. Gehe zu Plugins → Installierte Plugins
2. Finden Sie "WooCommerce POS"
3. Klicken Sie auf "Automatische Updates aktivieren"

### 3. Manuelles Update {#3-manual-update}

Wenn das automatische Update fehlschlägt:
1. Laden Sie die neueste Version von [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/) herunter
2. Deaktivieren Sie das aktuelle Plugin
3. Löschen Sie die alten Plugin-Dateien
4. Laden Sie die neue Version hoch und aktivieren Sie sie

### 4. Nach Update-Fehlern suchen {#4-check-for-update-errors}

Wenn Updates fehlschlagen:
- Überprüfen Sie die Dateiberechtigungen im wp-content/plugins
- Stellen Sie sicher, dass Sie genügend Festplattenspeicher haben
- Überprüfen Sie auf PHP-Speicherprobleme
- Prüfen Sie die Fehlerprotokolle auf Details

### 5. Downgrade der POS-App (vorübergehend) {#5-downgrade-pos-app-temporary}

Wenn Sie das Plugin nicht sofort aktualisieren können:
- Verwenden Sie eine ältere Version der POS-App
- Dies ist nur eine vorübergehende Lösung
- Planen Sie bald ein Update des Plugins

## Versionskompatibilität {#version-compatibility}

Halten Sie immer beide Komponenten auf dem neuesten Stand:
- Die POS-Anwendung
- Das WCPOS-WordPress-Plugin

Überprüfen Sie die Release-Notes auf möglicherweise breaking changes beim Aktualisieren.
