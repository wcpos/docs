# API05003: WCPOS-Plugin veraltet

## Was Das Bedeutet

Das auf Ihrem Server installierte WCPOS-Plugin ist veraltet und nicht kompatibel mit Ihrer Version der POS-Anwendung. Sie müssen das Plugin aktualisieren.

## Häufige Ursachen

- **Plugin nicht aktualisiert** — Automatische Updates sind möglicherweise deaktiviert
- **Update fehlgeschlagen** — Ein vorheriger Aktualisierungsversuch ist gescheitert
- **Versionskonflikt** — POS-App wurde aktualisiert, aber Plugin nicht
- **Staging-/Entwicklungsumgebung** — Test einer älteren Version

## Wie Sie Es Beheben

### 1. Plugin aktualisieren

In WordPress-Administration:
1. Gehen Sie zu Plugins → Installierte Plugins
2. Suchen Sie "WooCommerce POS"
3. Wenn ein Update verfügbar ist, klicken Sie auf "Jetzt aktualisieren"

### 2. Automatische Updates aktivieren

Um zukünftige Konflikte zu vermeiden:
1. Gehen Sie zu Plugins → Installierte Plugins
2. Suchen Sie "WooCommerce POS"
3. Klicken Sie auf "Automatische Updates aktivieren"

### 3. Manuelle Aktualisierung

Wenn automatische Aktualisierung fehlschlägt:
1. Laden Sie die neueste Version von [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/) herunter
2. Deaktivieren Sie das aktuelle Plugin
3. Löschen Sie die alten Plugindateien
4. Laden Sie die neue Version hoch und aktivieren Sie sie

### 4. Auf Aktualisierungsfehler prüfen

Wenn Updates fehlschlagen:
- Überprüfen Sie Dateiberechtigungen in wp-content/plugins
- Stellen Sie sicher, dass Speicherplatz verfügbar ist
- Prüfen Sie auf PHP-Speicherprobleme
- Konsultieren Sie Fehlerprotokolle für Details

### 5. POS-App vorübergehend herabstufen

Wenn Sie das Plugin nicht sofort aktualisieren können:
- Verwenden Sie eine ältere Version der POS-App
- Dies ist nur eine temporäre Lösung
- Planen Sie zeitnah eine Plugin-Aktualisierung

## Versionskompatibilität

Halten Sie beide Komponenten stets aktuell:
- Die POS-Anwendung
- Das WCPOS WordPress-Plugin

Prüfen Sie Versionshinweise auf mögliche Breaking Changes bei Updates.
