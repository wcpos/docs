# API03002: Fehlende erforderliche Parameter

## Was das bedeutet

Die Anfrage enthält nicht die Daten, die der Server benötigt, um die Aktion abzuschließen. Erforderliche Felder wurden nicht in die Anfrage aufgenommen.

## Häufige Ursachen

- **Unvollständiges Formular** — Erforderliche Felder wurden nicht ausgefüllt
- **Daten nicht gespeichert** — Formulardaten wurden nicht richtig erfasst
- **Versionskonflikt** — Plugin erwartet andere Parameter als das POS sendet
- **Benutzerdefinierte Felder** — Erforderliche benutzerdefinierte Felder fehlen

## So beheben Sie das Problem

### 1. Erforderliche Felder ausfüllen

Überprüfen Sie, ob alle erforderlichen Informationen ausgefüllt sind:
- Kundendaten (falls erforderlich)
- Produktinformationen
- Bestelldetails

### 2. Aktualisieren und erneut versuchen

Der Formularstatus könnte unvollständig sein:
1. Aktualisieren Sie das POS
2. Geben Sie die erforderlichen Informationen erneut ein
3. Versuchen Sie die Aktion noch einmal

### 3. Plugin-Versionen überprüfen

Stellen Sie die Kompatibilität sicher:
- Aktualisieren Sie das WCPOS-Plugin
- Aktualisieren Sie die POS-Anwendung
- Beide sollten auf kompatiblen Versionen sein

### 4. Einstellungen der erforderlichen Felder überprüfen

In WooCommerce können einige Felder als erforderlich markiert sein:
- Überprüfen Sie die Anforderungen der Checkout-Felder
- Überprüfen Sie die Konfigurationen benutzerdefinierter Felder
- Passen Sie die Anforderungen bei Bedarf an

### 5. Benutzerdefinierte Integrationen überprüfen

Wenn benutzerdefinierte Plugins oder Integrationen verwendet werden:
- Möglicherweise sind zusätzliche Felder erforderlich
- Überprüfen Sie die Plugin-Dokumentation
- Überprüfen Sie auf Konflikte
