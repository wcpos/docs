# API01008: Website Nicht Verfügbar

## Was Das Bedeutet

Das POS kann auf das Internet zugreifen, aber Ihre spezifische Website reagiert nicht. Der Server, der Ihren WooCommerce-Shop hostet, scheint heruntergefahren oder unerreichbar zu sein.

## Häufige Ursachen

- **Serverwartung** — Ihr Hosting-Anbieter führt möglicherweise Updates durch
- **Serverabsturz** — Der Webserver hat unerwartet aufgehört zu arbeiten
- **Ressourcengrenzen überschritten** — Ihr Hosting-Konto hat seine Grenzen erreicht
- **DDoS-Angriff** — Der Server ist mit bösartigem Traffic überlastet
- **Domain/Hosting abgelaufen** — Ihr Hosting oder Ihre Domain ist abgelaufen

## So Beheben Sie Das

### 1. Überprüfen Sie, ob die Website Nicht Verfügbar Ist

Versuchen Sie, Ihre Website direkt in einem Browser aufzurufen. Wenn sie nicht lädt, liegt das Problem serverseitig.

### 2. Überprüfen Sie Den Status Des Hosting-Anbieters

- Besuchen Sie die Statusseite Ihres Hosting-Anbieters
- Überprüfen Sie, ob Wartungsarbeiten geplant sind
- Suchen Sie nach gemeldeten Ausfällen

### 3. Überprüfen Sie Ihr Hosting-Konto

Melden Sie sich in Ihrem Hosting-Kontrollpanel an:
- Überprüfen Sie, ob Ihr Konto aktiv ist
- Stellen Sie sicher, dass Sie die Ressourcengrenzen nicht überschritten haben
- Achten Sie auf etwaige Suspendierungsbenachrichtigungen

### 4. Kontaktieren Sie Den Hosting-Support

Wenn alles auf Ihrer Seite normal aussieht, kontaktieren Sie Ihren Hosting-Anbieter. Sie können:
- Den Serverstatus überprüfen
- Die Serverprotokolle einsehen
- Dienste bei Bedarf neu starten

### 5. Warten Und Erneut Versuchen

Wenn es sich um einen vorübergehenden Ausfall handelt:
- Warten Sie 15-30 Minuten
- Das POS wird automatisch versuchen, die Verbindungen erneut herzustellen
- Zwischengespeicherte Daten bleiben offline verfügbar
