# API01008: Website nicht verfügbar

## Was Dies Bedeutet

Die POS kann das Internet erreichen, aber Ihre spezifische Website reagiert nicht. Der Server, der Ihren WooCommerce-Shop hostet, scheint ausgefallen oder nicht erreichbar zu sein.

## Häufige Ursachen

- **Serverwartung** — Ihr Hosting-Anbieter führt möglicherweise Updates durch
- **Serverabsturz** — Der Webserver ist unerwartet gestoppt
- **Ressourcenlimits überschritten** — Ihr Hosting-Konto hat seine Grenzen erreicht
- **DDoS-Angriff** — Der Server ist von bösartigem Datenverkehr überlastet
- **Domain/Hosting abgelaufen** — Ihr Hosting oder Ihre Domain ist ausgelaufen

## So beheben Sie das Problem

### 1. Überprüfen Sie, ob die Seite down ist

Versuchen Sie, Ihre Seite direkt im Browser zu öffnen. Wenn sie nicht lädt, liegt das Problem auf Serverseite.

### 2. Überprüfen Sie den Status des Hosting-Anbieters

- Besuchen Sie die Statusseite Ihres Hosting-Anbieters
- Prüfen Sie auf geplante Wartungsarbeiten
- Suchen Sie nach gemeldeten Ausfällen

### 3. Überprüfen Sie Ihr Hosting-Konto

Melden Sie sich in Ihrer Hosting-Kontrollleiste an:
- Prüfen Sie, ob Ihr Konto aktiv ist
- Überprüfen Sie, ob Sie keine Ressourcenlimits überschritten haben
- Suchen Sie nach Aussetzungshinweisen

### 4. Kontaktieren Sie den Hosting-Support

Wenn auf Ihrer Seite alles normal aussieht, kontaktieren Sie Ihren Hosting-Anbieter. Sie können:
- Serverstatus prüfen
- Serverprotokolle überprüfen
- Dienste bei Bedarf neu starten

### 5. Warten und erneut versuchen

Bei einem vorübergehenden Ausfall:
- Warten Sie 15-30 Minuten
- Die POS wird Verbindungen automatisch erneut versuchen
- Zwischengespeicherte Daten bleiben offline verfügbar
