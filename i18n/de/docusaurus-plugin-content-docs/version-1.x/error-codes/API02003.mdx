# API02003: Token ungültig

## Was das bedeutet

Der verwendete Authentifizierungs-Token wird vom Server nicht erkannt. Dies unterscheidet sich von einem abgelaufenen Token — dieser Token war nie gültig oder wurde widerrufen.

## Häufige Ursachen

- **Token widerrufen** — Ein Administrator hat die Sitzung beendet
- **Token beschädigt** — Datenbeschädigung während Speicherung oder Übertragung
- **Serverreset** — Serversicherheitsschlüssel wurden geändert, wodurch alle Token ungültig wurden
- **Falscher Server** — Token von einer anderen WordPress-Installation

## Server-Fehler-Zuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Server-Code | Quelle |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT-Authentifizierungs-Plugin |

## Problemlösung

### 1. Erneut anmelden

Sitzung löschen und neu authentifizieren:
1. Vollständig abmelden
2. POS-App schließen und wieder öffnen
3. Mit Ihren Anmeldedaten erneut anmelden

### 2. Anwendungsdaten löschen

Wenn Abmelden nicht funktioniert:
- POS-App-Cache/Daten löschen
- Im Web: Browser-Cookies und lokalen Speicher der Website löschen
- Auf Desktop: App-Einstellungen nach Option "Daten löschen" überprüfen

### 3. Serverkonfiguration überprüfen

Wenn das Problem alle Benutzer betrifft:
- Prüfen, ob WordPress-Salts geändert wurden
- JWT- oder Authentifizierungs-Plugin-Einstellungen verifizieren
- Kürzliche Serveränderungen überprüfen

### 4. Korrekten Server überprüfen

Sicherstellen, dass die POS mit der richtigen WordPress-Website verbunden ist, besonders wenn mehrere Installationen existieren.
