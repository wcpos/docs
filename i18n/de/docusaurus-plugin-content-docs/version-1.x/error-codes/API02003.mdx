# API02003: Token Ungültig

## Was Das Bedeutet

Das verwendete Authentifizierungstoken wird vom Server nicht erkannt. Dies unterscheidet sich von einem abgelaufenen Token – dieses Token war niemals gültig oder wurde widerrufen.

## Häufige Ursachen

- **Token widerrufen** – Ein Administrator hat die Sitzung widerrufen
- **Token beschädigt** – Datenkorruption während der Speicherung oder Übertragung
- **Server zurückgesetzt** – Servergeheimschlüssel wurden geändert, was alle Tokens ungültig macht
- **Falscher Server** – Token von einer anderen WordPress-Installation

## Serverfehlerzuordnung

Dieser Fehlercode wird ausgelöst, wenn der Server zurückgibt:

| Servercode | Quelle |
|------------|--------|
| `jwt_auth_invalid_token` | JWT-Authentifizierungs-Plugin |

## So Beheben Sie Es

### 1. Wieder Einloggen

Löschen Sie Ihre Sitzung und authentifizieren Sie sich neu:
1. Komplett ausloggen
2. POS schließen und erneut öffnen
3. Mit Ihren Zugangsdaten einloggen

### 2. Anwendungsdaten Löschen

Wenn das Ausloggen nicht funktioniert:
- Cache/Daten der POS-App löschen
- Im Web: Browser-Cookies und lokalen Speicher der Website löschen
- Auf dem Desktop: App-Einstellungen nach einer Option "Daten löschen" überprüfen

### 3. Serverkonfiguration Überprüfen

Wenn das Problem alle Benutzer betrifft:
- Überprüfen Sie, ob die WordPress-Salze geändert wurden
- Überprüfen Sie die Einstellungen des JWT- oder Authentifizierungs-Plugins
- Überprüfen Sie kürzliche Serveränderungen

### 4. Überprüfen Sie, Ob Sie Auf Dem Richtigen Server Sind

Stellen Sie sicher, dass die POS so konfiguriert ist, dass sie sich mit der richtigen WordPress-Website verbindet, insbesondere wenn Sie mehrere Installationen haben.
