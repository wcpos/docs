# API05004: WordPress-REST-API deaktiviert

## Was dies bedeutet

Die WordPress-REST-API ist auf Ihrer Website deaktiviert. Alle modernen WordPress-Funktionen, einschließlich WooCommerce und WCPOS, hängen von dieser API ab.

## Häufige Ursachen

- **Sicherheits-Plugin** — Ein Plugin blockiert den Zugriff auf die REST-API
- **Hosting-Einschränkung** — Ihr Host hat die REST-API deaktiviert
- **Eigener Code** — Ein Theme oder Plugin hat die API deaktiviert
- **Firewall-Regeln** — WAF blockiert REST-API-Endpunkte

## Problemlösung

### 1. REST-API testen

Besuchen Sie `https://yoursite.com/wp-json/` in Ihrem Browser:
- Sollte JSON mit verfügbaren Routen zurückgeben
- Wenn Sie einen Fehler oder nichts sehen, ist er blockiert

### 2. Sicherheits-Plugins überprüfen

Häufige Plugins, die die REST-API blockieren:

**Wordfence:**
- Firewall → Alle Firewall-Optionen
- Option "REST-API deaktivieren" deaktivieren

**iThemes Security:**
- Sicherheit → Einstellungen → WordPress-Anpassungen
- REST-API aktivieren

**Disable REST API Plugin:**
- Plugin komplett deaktivieren

### 3. Eigenen Code überprüfen

Suchen Sie in der `functions.php` Ihres Themes oder in benutzerdefinierten Plugins nach:

```php
// Dieser Code deaktiviert die REST-API - entfernen Sie ihn
add_filter('rest_authentication_errors', function($result) {
    return new WP_Error('rest_disabled', 'REST-API deaktiviert');
});
```

### 4. .htaccess überprüfen

Entfernen Sie alle Regeln, die `/wp-json/` blockieren:

```apache
# Schlecht - blockiert REST-API
RewriteRule ^wp-json - [F,L]
```

### 5. Hosting-Anbieter kontaktieren

Manche Hosts blockieren die REST-API standardmäßig:
- Bitten Sie sie, sie zu aktivieren
- Fragen Sie nach Sicherheitseinschränkungen
- Überprüfen Sie die Hosting-Dokumentation

## Warum die REST-API wichtig ist

Die WordPress-REST-API ist wesentlich für:
- Mobile Apps
- Drittanbieter-Integrationen
- WooCommerce-Funktionen
- WCPOS-Betrieb

Die Deaktivierung unterbricht viele Funktionen.
