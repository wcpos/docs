# DB02001: Doppelter Datensatz

## Was Das Bedeutet

Die Datenbank versuchte, einen Datensatz mit einer ID einzufügen, die bereits existiert. Jeder Datensatz muss einen eindeutigen Identifikator haben.

## Häufige Ursachen

- **Synchronisierungskonflikt** — Gleiche Daten zweimal synchronisiert
- **Wettlaufbedingung** — Mehrere Vorgänge erstellen denselben Datensatz
- **Datenbeschädigung** — IDs wurden beschädigt oder dupliziert
- **Importproblem** — Daten werden importiert, die bereits existieren

## Wie Man Es Behebt

### 1. Daten Aktualisieren

Vom Server neu synchronisieren:
- Aktuelle Daten abrufen
- Synchronisierungsprozess Konflikte lösen lassen
- Neuere Daten sollten Vorrang haben

### 2. Operation Wiederholen

Beim Erstellen neuer Datensätze:
- Das System hat ihn möglicherweise bereits erstellt
- Prüfen, ob der Datensatz existiert
- Vermeiden, Absenden mehrmals zu klicken

### 3. Lokalen Cache Löschen

Lokale Daten zurücksetzen:
1. Lokale Datenbank/Cache löschen
2. Erneut anmelden
3. Daten neu synchronisieren

### 4. Duplikate in WooCommerce Überprüfen

Wenn das Problem anhält:
- WooCommerce auf doppelte Einträge prüfen
- Nach Produkten/Bestellungen mit gleichen IDs suchen
- Duplikate im Admin bereinigen

### 5. Wiederkehrende Probleme Melden

Wenn dies häufig auftritt:
- Notieren, welche Aktionen es verursachen
- Nach Mustern suchen
- Supportteam mit Details informieren

## Warum Dies Wichtig Ist

Doppelte IDs würden verursachen:
- Verwirrung darüber, welcher Datensatz korrekt ist
- Datenüberschreibungen
- Synchronisierungsprobleme

Der Fehler verhindert Datenbeschädigung.

## Verwandte Fehler

- [DB02002](./DB02002) — Datensatz nicht gefunden
- [DB02003](./DB02003) — Einschränkungsverletzung
