# DB02001: Duplikatdatensatz

## Was Das Bedeutet

Die Datenbank hat versucht, einen Datensatz mit einer ID einzufügen, die bereits existiert. Jeder Datensatz muss einen eindeutigen Identifikator haben.

## Häufige Ursachen

- **Sync-Konflikt** — Gleiche Daten wurden zweimal synchronisiert
- **Race Condition** — Mehrere Vorgänge erstellen denselben Datensatz
- **Datenbeschädigung** — IDs wurden beschädigt oder dupliziert
- **Importproblem** — Importieren von Daten, die bereits existieren

## So Beheben Sie Es

### 1. Daten Aktualisieren

Daten vom Server neu synchronisieren:
- Neueste Daten abrufen
- Den Synchronisierungsprozess Konflikte lösen lassen
- Die neueren Daten sollten Vorrang haben

### 2. Vorgang Wiederholen

Beim Erstellen neuer Datensätze:
- Das System hat ihn möglicherweise bereits erstellt
- Überprüfen, ob der Datensatz existiert
- Vermeiden, mehrmals auf "Absenden" zu klicken

### 3. Lokalen Cache Leeren

Lokale Daten zurücksetzen:
1. Lokale Datenbank/Cache leeren
2. Erneut einloggen
3. Daten neu synchronisieren

### 4. Auf Duplikate in WooCommerce Überprüfen

Falls das Problem weiterhin besteht:
- WooCommerce auf doppelte Einträge überprüfen
- Nach Produkten/Bestellungen mit denselben IDs suchen
- Alle Duplikate im Admin bereinigen

### 5. Anhaltende Probleme Meldem

Wenn dies häufig passiert:
- Notieren, welche Aktionen es verursachen
- Auf Muster überprüfen
- Dem Support mit Details berichten

## Warum Das Wichtig Ist

Duplikate IDs würden verursachen:
- Verwirrung darüber, welcher Datensatz korrekt ist
- Datenüberschreibungen
- Synchronisierungsprobleme

Der Fehler verhindert Datenbeschädigungen.

## Verwandte Fehler

- [DB02002](./DB02002) — Datensatz Nicht Gefunden
- [DB02003](./DB02003) — Verstoß Gegen Einschränkungen
