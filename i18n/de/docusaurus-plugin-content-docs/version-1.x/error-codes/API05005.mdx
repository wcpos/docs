# API05005: Plugin nicht gefunden

## Was Dies Bedeutet

Ein erforderliches WordPress-Plugin ist nicht installiert oder aktiviert. Dies könnte WooCommerce selbst, eine erforderliche Erweiterung oder ein Integrations-Plugin sein.

## Häufige Ursachen

- **WooCommerce deaktiviert** — WooCommerce wurde deaktiviert
- **Erforderliche Erweiterung fehlt** — Eine benötigte WooCommerce-Erweiterung ist nicht installiert
- **Plugin-Konflikt** — Ein anderes Plugin hat ein erforderliches Plugin deaktiviert
- **Update fehlgeschlagen** — Ein Plugin-Update ist fehlgeschlagen und hat es deaktiviert

## Behebung

### 1. WooCommerce-Aktivierung überprüfen

WCPOS benötigt WooCommerce:
1. Gehen Sie zu Plugins → Installierte Plugins
2. Suchen Sie nach "WooCommerce"
3. Stellen Sie sicher, dass es aktiviert ist

### 2. Erforderliche Erweiterungen prüfen

Einige WCPOS-Funktionen benötigen möglicherweise Erweiterungen:
- Prüfen Sie die Fehlermeldung auf einen spezifischen Plugin-Namen
- Installieren und aktivieren Sie die erforderliche Erweiterung

### 3. Plugin-Konflikte überprüfen

Wenn Plugins aufgrund von Konflikten deaktiviert wurden:
1. Überprüfen Sie Ihre E-Mails auf WordPress-Benachrichtigungen
2. Prüfen Sie Fehlerprotokolle
3. Lösen Sie Konflikte und aktivieren Sie die Plugins wieder

### 4. Plugin-Dateien überprüfen

Wenn Plugin-Dateien fehlen:
1. Installieren Sie das Plugin erneut von WordPress.org
2. Oder stellen Sie es aus einer Sicherungskopie wieder her
3. Aktivieren Sie das Plugin

### 5. Alle Plugins aktualisieren

Veraltete Plugins können Probleme verursachen:
1. Gehen Sie zum Dashboard → Updates
2. Aktualisieren Sie alle Plugins
3. Überprüfen Sie, dass alle erforderlichen Plugins aktiviert sind

## Erforderliche Plugins

Mindestens benötigt WCPOS:
- **WooCommerce** — Die E-Commerce-Plattform
- **WCPOS** — Das POS-Plugin

Zusätzliche Funktionen können andere Plugins erfordern.

- [API05002](./API05002) — WCPOS-Plugin nicht gefunden
- [API05001](./API05001) — WooCommerce-API deaktiviert
