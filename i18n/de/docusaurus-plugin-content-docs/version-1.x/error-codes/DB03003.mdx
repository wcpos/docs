# DB03003: Erforderliches Feld fehlt

## Was Dies Bedeutet

Ein erforderliches Feld wurde nicht angegeben. Die Datenbank kann den Datensatz nicht speichern, da wesentliche Informationen fehlen.

## Häufige Ursachen

- **Unvollständiges Formular** — Erforderliche Felder wurden nicht ausgefüllt
- **Datensynchronisierungsproblem** — Erforderliche Daten wurden nicht richtig synchronisiert
- **Validierung umgangen** — Formular ohne Validierung übermittelt
- **Benutzerdefiniertes Feld erforderlich** — Ein benutzerdefiniertes Pflichtfeld ist leer

## Wie Sie Es Beheben

### 1. Alle Pflichtfelder Ausfüllen

Überprüfen Sie das Formular auf:
- Felder mit Asterisken (*)
- Hervorgehobene oder mit Fehler markierte Felder
- Leere Felder, die einen Wert haben sollten

### 2. Aktualisieren und Erneut Eingeben

Wenn der Formularzustand nicht korrekt ist:
1. Seite/Bildschirm aktualisieren
2. Alle Informationen erneut eingeben
3. Vor der Übermittlung alle Felder überprüfen

### 3. WooCommerce-Einstellungen Prüfen

Wenn benutzerdefinierte Felder erforderlich sind:
1. WooCommerce-Checkout-Einstellungen überprüfen
2. Auf erforderliche benutzerdefinierte Felder prüfen
3. Sicherstellen, dass der POS diese Felder bereitstellt

### 4. Erforderliche Daten Synchronisieren

Wenn zugehörige Daten fehlen:
- Produkte/Kunden/etc. aktualisieren
- Synchronisation abwarten
- Vorgang erneut versuchen

### 5. Einstellungen für Pflichtfelder Überprüfen

In WooCommerce Admin:
- Überprüfen, welche Felder als Pflichtfelder markiert sind
- Überlegen, ob alle für POS wirklich erforderlich sind
- Anforderungen bei Bedarf anpassen

## Häufige Pflichtfelder

Typischerweise für Bestellungen erforderlich:
- Mindestens ein Positionsartikel
- Zahlungsmethode
- Kunde (abhängig von den Einstellungen)

Typischerweise für Produkte erforderlich:
- Produktname
- Preis (für einfache Produkte)

- [DB02003](./DB02003) — Einschränkungsverletzung
- [API03002](./API03002) — Fehlende erforderliche Parameter
