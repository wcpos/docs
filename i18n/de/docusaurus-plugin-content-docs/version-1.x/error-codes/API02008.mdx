# API02008: Refresh Token Ungültig {#api02008-refresh-token-invalid}

## Was das bedeutet {#what-this-means}

Der Refresh Token, der vom POS gespeichert wurde, wird vom Server nicht erkannt. Refresh Tokens werden verwendet, um neue Access Tokens zu erhalten, ohne dass Sie sich erneut anmelden müssen.

## Häufige Ursachen {#common-causes}

- **Token widerrufen** — Ein Administrator hat Ihre Sitzung ungültig gemacht
- **Servergeheimnisse geändert** — Die Sicherheitskeys von WordPress wurden aktualisiert
- **Datenkorruption** — Der Token wurde im Speicher beschädigt
- **Mehrere Anmeldungen** — Das Anmelden an einem anderen Ort könnte diesen Token ungültig gemacht haben

## So beheben Sie das Problem {#how-to-fix}

### 1. Erneut anmelden {#1-log-in-again}

Da der Refresh Token ungültig ist, müssen Sie sich erneut authentifizieren:
1. Melden Sie sich vom POS ab
2. Geben Sie Ihren Benutzernamen und Ihr Passwort ein
3. Dies generiert neue gültige Tokens

### 2. Widerrufene Sitzungen überprüfen {#2-check-for-revoked-sessions}

Wenn ein Administrator Sitzungen widerrufen hat:
- Dies ist eine normale Sicherheitspraktik
- Melden Sie sich einfach erneut an
- Ihre neue Sitzung wird normal funktionieren

### 3. Serveränderungen überprüfen {#3-check-server-changes}

Wenn dies alle Benutzer betrifft:
- Die WordPress Salzkonstrukte/Keys könnten geändert worden sein
- Die Einstellungen des JWT-Plugins könnten modifiziert worden sein
- Kontaktieren Sie Ihren Serveradministrator

### 4. Gespeicherte Daten löschen {#4-clear-stored-data}

Wenn Sie weiterhin Probleme haben:
- Löschen Sie die Anwendungsdaten des POS
- Entfernen Sie zwischengespeicherte Anmeldedaten
- Beginnen Sie mit einer sauberen Anmeldung
