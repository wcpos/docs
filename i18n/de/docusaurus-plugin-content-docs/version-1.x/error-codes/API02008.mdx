# API02008: Ungültiges Aktualisierungstoken

## Was Dies Bedeutet

Das vom POS gespeicherte Aktualisierungstoken wird vom Server nicht erkannt. Aktualisierungstokens werden verwendet, um neue Zugriffstokens zu erhalten, ohne sich erneut anmelden zu müssen.

## Häufige Ursachen

- **Token widerrufen** — Ein Administrator hat Ihre Sitzung ungültig gemacht
- **Serversecrets geändert** — WordPress-Sicherheitsschlüssel wurden aktualisiert
- **Datenbeschädigung** — Das Token wurde bei der Speicherung beschädigt
- **Mehrere Anmeldungen** — Die Anmeldung an anderer Stelle kann dieses Token ungültig gemacht haben

## Wie Sie Dies Beheben

### 1. Erneut Anmelden

Da das Aktualisierungstoken ungültig ist, müssen Sie sich erneut authentifizieren:
1. Melden Sie sich vom POS ab
2. Geben Sie Ihren Benutzernamen und Ihr Passwort ein
3. Dies generiert neue gültige Tokens

### 2. Widerrufene Sitzungen Überprüfen

Wenn ein Administrator Sitzungen widerrufen hat:
- Dies ist eine normale Sicherheitspraxis
- Melden Sie sich einfach erneut an
- Ihre neue Sitzung wird normal funktionieren

### 3. Serveränderungen Überprüfen

Wenn dies alle Benutzer betrifft:
- WordPress-Salze/Schlüssel könnten geändert worden sein
- JWT-Plugin-Einstellungen wurden möglicherweise modifiziert
- Kontaktieren Sie Ihren Serveradministrator

### 4. Gespeicherte Daten Löschen

Wenn weiterhin Probleme auftreten:
- Löschen Sie die POS-Anwendungsdaten
- Entfernen Sie zwischengespeicherte Anmeldedaten
- Beginnen Sie mit einer sauberen Anmeldung
