# API04002: Unerwarteter Antwortcode

## Was Dies Bedeutet

Der Server hat einen HTTP-Statuscode zurückgegeben, der für diese Anfrage nicht erwartet wurde. Häufige Codes sind 500 (Serverfehler), 403 (Verboten), 404 (Nicht gefunden) usw.

## Häufige Ursachen

- **500 Interner Serverfehler** — PHP ist abgestürzt oder auf einen Fehler gestoßen
- **403 Verboten** — Zugriff durch Sicherheitseinstellungen verweigert
- **404 Nicht gefunden** — Der Endpunkt existiert nicht
- **502/503/504** — Server-Gateway- oder Verfügbarkeitsprobleme

## Behebung

### Für 500-Fehler (Serverfehler)

1. PHP-Fehlerprotokolle überprüfen
2. PHP-Speicherlimit erhöhen
3. Nach Plugin-Konflikten suchen
4. `wp-content/debug.log` überprüfen

### Für 403-Fehler (Verboten)

1. Einstellungen des Sicherheits-Plugins überprüfen (Wordfence, Sucuri, etc.)
2. POS oder IP auf die Whitelist setzen
3. `.htaccess` auf Blockierungsregeln prüfen
4. Überprüfen, ob ModSecurity Anfragen blockiert

### Für 404-Fehler (Nicht gefunden)

1. Überprüfen, ob WCPOS-Plugin aktiviert ist
2. WordPress-Permalinks leeren (Einstellungen → Permalinks → Speichern)
3. Prüfen, ob REST-API aktiviert ist
4. URL-Konfiguration verifizieren

### Für 502/503/504-Fehler (Gateway-Probleme)

1. Hostinganbieter kontaktieren
2. Auf Wiederherstellung des Servers warten
3. Überprüfen, ob die Website überlastet ist
4. Sicherstellen, dass der Server läuft

### Allgemeine Fehlersuche

1. Versuchen, die Website direkt aufzurufen
2. Hosting-Kontrollpanel auf Probleme prüfen
3. Serverzugriffsprotokoll überprüfen
4. Hosting-Support bei Bedarf kontaktieren
