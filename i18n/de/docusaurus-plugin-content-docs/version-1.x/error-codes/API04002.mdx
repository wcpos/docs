# API04002: Unerwarteter Antwortcode {#api04002-unexpected-response-code}

## Was das bedeutet {#what-this-means}

Der Server hat einen HTTP-Statuscode zurückgegeben, der für diese Anfrage nicht erwartet wurde. Häufige Codes sind 500 (Serverfehler), 403 (verboten), 404 (nicht gefunden) usw.

## Häufige Ursachen {#common-causes}

- **500 Interner Serverfehler** — PHP ist abgestürzt oder hat einen Fehler festgestellt
- **403 Verboten** — Zugriff durch Sicherheitseinstellungen verweigert
- **404 Nicht gefunden** — Der Endpunkt existiert nicht
- **502/503/504** — Probleme mit dem Server-Gateway oder der Verfügbarkeit

## So beheben Sie es {#how-to-fix}

### Bei 500-Fehlern (Serverfehler) {#for-500-errors-server-error}

1. Überprüfen Sie die PHP-Fehlerprotokolle
2. Erhöhen Sie das PHP-Speicherlimit
3. Suchen Sie nach Plugin-Konflikten
4. Überprüfen Sie `wp-content/debug.log`

### Bei 403-Fehlern (Verboten) {#for-403-errors-forbidden}

1. Überprüfen Sie die Einstellungen des Sicherheits-Plugins (Wordfence, Sucuri usw.)
2. Fügen Sie POS oder Ihre IP zur Whitelist hinzu
3. Überprüfen Sie `.htaccess` auf blockierende Regeln
4. Stellen Sie sicher, dass ModSecurity keine Anfragen blockiert

### Bei 404-Fehlern (Nicht gefunden) {#for-404-errors-not-found}

1. Überprüfen Sie, ob das WCPOS-Plugin aktiv ist
2. Leeren Sie die WordPress-Permalinks (Einstellungen → Permalinks → Speichern)
3. Überprüfen Sie, ob die REST API aktiviert ist
4. Verifizieren Sie die URL-Konfiguration

### Bei 502/503/504-Fehlern (Gateway-Probleme) {#for-502503504-errors-gateway-issues}

1. Kontaktieren Sie Ihren Hosting-Anbieter
2. Warten Sie, bis sich der Server erholt
3. Überprüfen Sie, ob die Website stark ausgelastet ist
4. Verifizieren Sie, dass der Server läuft

### Allgemeine Fehlersuche {#general-troubleshooting}

1. Versuchen Sie, direkt auf Ihre Seite zuzugreifen
2. Überprüfen Sie das Hosting-Kontrollpanel auf Probleme
3. Überprüfen Sie die Serverzugriffsprotokolle
4. Kontaktieren Sie den Hosting-Support, falls notwendig
