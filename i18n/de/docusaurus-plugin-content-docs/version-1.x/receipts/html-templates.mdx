HTML-Vorlagen verwenden eine logiklose Mustache-Syntax mit `{{variable}}`-Platzhaltern. Sie werden clientseitig im POS gerendert, was bedeutet, dass sie **offline funktionieren** — keine Serververbindung erforderlich, um einen Beleg anzuzeigen oder zu drucken.

Dies ist die empfohlene Engine für die meisten Benutzer.

## Syntax {#syntax}

### Variablen {#variables}

Daten werden mit doppelten geschweiften Klammern eingefügt:

```html
<h1>{{store.name}}</h1>
<p>Order #{{order.number}}</p>
<p>Total: {{totals.total_display}}</p>
```

Verwenden Sie `_display`-Varianten für vorformatierte Währungswerte (z. B. `$12.50` statt `12.5`). Alle verfügbaren Felder finden Sie in der [Belegdaten-Referenz](/receipts/receipt-data).

### Abschnitte (Schleifen und Bedingungen) {#sections-loops-and-conditionals}

Verwenden Sie `{{#section}}...{{/section}}`, um über Arrays zu iterieren oder Inhalte bedingt anzuzeigen:

```html
{{#lines}}
<div class="line-item">
  <span>{{name}} × {{qty}}</span>
  <span>{{line_total_display}}</span>
</div>
{{/lines}}
```

### Invertierte Abschnitte {#inverted-sections}

Verwenden Sie `{{^section}}...{{/section}}`, um Inhalte anzuzeigen, wenn ein Wert leer oder falsch ist:

```html
{{#customer.id}}
  <p>Customer: {{customer.name}}</p>
{{/customer.id}}
{{^customer.id}}
  <p>Guest checkout</p>
{{/customer.id}}
```

### Lokalisierte Bezeichnungen {#localised-labels}

Verwenden Sie `{{i18n.key}}` für übersetzbare Bezeichnungen, die sich an die Sprache der Filiale anpassen:

```html
<th>{{i18n.subtotal}}</th>
<td>{{totals.subtotal_display}}</td>
```

## Gängige Muster {#common-patterns}

### Filialkopf {#store-header}

Das Array `store.address_lines` ist bereits für Belege formatiert — verwenden Sie eine Schleife darüber, anstatt einzelne Adressfelder zusammenzusetzen.

```html
<div style="text-align: center;">
  {{#store.logo}}<img src="{{store.logo}}" alt="{{store.name}}" style="max-width: 200px;" />{{/store.logo}}
  <h1 style="margin: 8px 0;">{{store.name}}</h1>
  {{#store.address_lines}}<div>{{.}}</div>{{/store.address_lines}}
  {{#store.phone}}<div>{{store.phone}}</div>{{/store.phone}}
  {{#store.tax_ids}}<div>{{#label}}{{label}} {{/label}}{{value}}</div>{{/store.tax_ids}}
</div>
```

### Positionen mit Rabatten {#line-items-with-discounts}

Das Feld `discounts` einer Position enthält den Rabattbetrag als positive Zahl oder `0`, wenn kein Rabatt vorhanden ist. Mustache behandelt `0` als falsy, sodass `{{#discounts}}...{{/discounts}}` die richtige Abfrage ist.

```html
{{#lines}}
<div class="line-item" style="display: flex; justify-content: space-between;">
  <div>
    <div>{{name}} × {{qty}}</div>
    {{#discounts}}
      <div style="font-size: 0.9em; color: #6b7280;">
        <s>{{unit_subtotal_display}}</s> {{unit_price_display}}
      </div>
    {{/discounts}}
  </div>
  <div>{{line_total_display}}</div>
</div>
{{/lines}}
```

### Steuerübersicht {#tax-summary}

```html
{{#tax_summary}}
<div class="tax-line">
  <span>{{label}} ({{rate}}%)</span>
  <span>{{tax_amount_display}}</span>
</div>
{{/tax_summary}}
```

### Zahlungsdetails {#payment-details}

```html
{{#payments}}
<div class="payment" style="display: flex; justify-content: space-between;">
  <span>{{method_title}}</span>
  <span>{{amount_display}}</span>
</div>
{{#tendered}}
  <div style="display: flex; justify-content: space-between;">
    <span>{{i18n.tendered}}</span><span>{{tendered_display}}</span>
  </div>
  <div style="display: flex; justify-content: space-between;">
    <span>{{i18n.change}}</span><span>{{change_display}}</span>
  </div>
{{/tendered}}
{{/payments}}
```

### Barcodes und QR-Codes {#barcodes-and-qr-codes}

Verwenden Sie die `<barcode>` Element — dieselbe Syntax wie bei [Thermovorlagen](/receipts/thermal-templates). Der Wert wird innerhalb des Elements platziert; die Symbologie wird über das `type` -Attribut festgelegt. Der Renderer ersetzt jedes Element durch ein Inline-SVG.

```html
<!-- Code 128 barcode of the order number -->
<barcode type="code128" height="40">{{order.number}}</barcode>

<!-- QR code -->
<barcode type="qrcode" scale="3">{{order.payment_url}}</barcode>

<!-- EAN-13 -->
<barcode type="ean13">{{order.number}}</barcode>
```

Das `type`-Attribut muss **wörtlich** in der Vorlage gesetzt werden — es kann nicht aus einem Platzhalter stammen. Ein `type` von `qr` oder `qrcode` wird sowohl in HTML- als auch in Thermovorlagen als QR-Code gerendert.

Wenn der ausgewählte Barcode-Typ den Wert nicht kodieren kann (z. B. nicht-numerische oder falsch lange EAN-13-Daten), zeigt die Vorschau einen `Barcode error`- oder `QR code error`-Block mit dem ursprünglichen Wert an, sodass der Wert korrigiert oder ein kompatibler Typ gewählt werden kann.

Alle [bwip-js-Symbologien](https://github.com/metafloor/bwip-js/wiki/Supported-Barcode-Types) werden unterstützt — `code128`, `qrcode`, `ean13`, `ean8`, `upca`, `pdf417`, `datamatrix` und viele mehr.

## Gestaltung {#styling}

Logikfreie Vorlagen werden vor dem Rendern durch die WordPress-Funktion `wp_kses_post` bereinigt, die **alle `<style>`, `<head>` und Dokument-Level-Tags entfernt**. Verwenden Sie **Inline-Styles** für jedes Element:

```html
<div style="font-family: monospace; font-size: 12px; max-width: 300px; margin: 0 auto;">
  <div style="text-align: center; margin-bottom: 16px;">
    <strong style="font-size: 16px;">{{store.name}}</strong>
  </div>
  <div style="display: flex; justify-content: space-between; font-weight: 700; border-top: 1px solid black; padding-top: 8px;">
    <span>{{i18n.total}}</span>
    <span>{{totals.total_display}}</span>
  </div>
</div>
```

Dieselbe Einschränkung gilt für `<script>`, `<link>` und andere nicht-inhaltsbezogene Tags — sie werden entfernt, daher sollte nicht auf externes CSS oder JavaScript zurückgegriffen werden.

:::tip Für Schwarzweiß-Druck konzipiert
Belege werden in der Regel auf Schwarzweiß-Thermo- oder Laserdruckern gedruckt. Berücksichtigen Sie dies beim Design — verwenden Sie **Schriftstärke, Leerraum, Schriftkontrast und horizontale Linien** für die Hierarchie anstelle von farbigen Flächen. Farbige Hintergründe und grauer Text verschwinden auf Schwarzweiß-Ausgaben oder werden als unsaubere Halbtöne gedruckt.
:::

:::tip Thermopapier-Optik ohne Thermodrucker gewünscht?
Klicken Sie auf **Vorlage verwenden** auf der Karte **Schmaler Beleg** in der Galerie — es handelt sich um ein Monospace-Layout mit 72 mm Breite, das sauber über den Browserdruck, HTML-fähige Thermodrucker (Sunmi/Imin-WebView-Geräte) und Standardpapier gedruckt wird. Nicht für reine ESC/POS-Thermodrucker geeignet — verwenden Sie dafür die XML-Vorlagen **Einfacher Thermobeleg**.
:::

## Bewährte Vorgehensweisen {#best-practices}

- **`_display`-Felder verwenden** für alle Währungswerte — sie übernehmen die sprachabhängige Formatierung automatisch
- **`{{i18n.*}}`-Bezeichnungen verwenden** anstatt Text fest zu codieren, damit Vorlagen sprachübergreifend funktionieren
- **Abschnitte für Bedingungen verwenden** — optionale Felder in `{{#field}}...{{/field}}` einschließen, damit sie bei leerem Inhalt ausgeblendet werden
- **Mit einer Galerie-Vorlage beginnen**, anstatt alles von Grund auf neu zu erstellen
- **Mit der Vorschau testen** im Vorlagen-Editor, bevor die Vorlage aktiviert wird
