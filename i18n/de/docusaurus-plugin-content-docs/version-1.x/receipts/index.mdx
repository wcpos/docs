WCPOS enthält ein vollständiges Vorlagensystem für Belege, Rechnungen, Geschenkbelege, Lieferscheine, Küchentickets und mehr — alles verwaltet in **WP Admin → POS → Vorlagen**.

:::tip Möchten Sie nur Ihren Beleg ändern?
Beginnen Sie mit **[Beleg anpassen](/receipts/customise)**. Dort werden die drei einfachsten Möglichkeiten erklärt — eine andere mitgelieferte Vorlage auswählen, eine Vorlage per KI anpassen lassen oder sie manuell bearbeiten.
:::

## Vorlagen-Engines {#template-engines}

WCPOS unterstützt drei Vorlagen-Engines. Die Engine wird ausgewählt, wenn eine Vorlage erstellt oder auf einer Galerie-Karte **Vorlage verwenden** geklickt wird, und kann **später nicht mehr geändert werden** — erstellen Sie eine neue Vorlage, wenn Sie die Engine wechseln möchten.

<FeatureGrid>
  <FeatureCard icon="code" title="Logikfreies HTML (empfohlen)">
    Mustache-Stil <code>{"{{variable}}"}</code> Platzhalter in einfachem HTML. Wird clientseitig gerendert, sodass Belege **offline funktionieren**. Für Browserdruck und PDF-Belege verwenden.
  </FeatureCard>
  <FeatureCard icon="print" title="Thermo-XML">
    XML-Vorlagen, die sowohl eine Bildschirmvorschau als auch **ESC/POS-Befehle** für Thermo-Belegdrucker erzeugen. Dieselbe Vorlage, zwei Ausgaben. Für Epson, Star und andere Thermodrucker verwenden.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP für ältere Vorlagen">
    Serverseitige PHP-Vorlagen mit WooCommerce-Funktionen. Aus Gründen der Abwärtskompatibilität mit bestehenden <code>yourtheme/woocommerce-pos/receipt.php</code> Überschreibungen beibehalten. Zum Rendern ist eine Serververbindung erforderlich.
  </FeatureCard>
</FeatureGrid>

## Vorlagengalerie {#template-gallery}

Die Galerie enthält fertige Vorlagen, die unverändert verwendet oder angepasst werden können. Filtern Sie nach **Kategorie** (Beleg, Rechnung, Geschenkbeleg, Küchenticket, Angebot / Bestellung), **Format** (HTML / ESC/POS) und **Richtung** (LTR / RTL); zeigen Sie eine Vorschau mit den echten Daten Ihrer Filiale an. Die Fußzeile jeder Karte enthält einen Link **Vorschau** und eine primäre Schaltfläche **Vorlage verwenden** — ein Klick auf Vorlage verwenden erstellt eine bearbeitbare Kopie in *Ihre Vorlagen*.

| Vorlage | Vorlagen-Engine | Beschreibung |
|----------|--------|-------------|
| Standardbeleg | HTML | Standardbeleg — Logo, Filialidentität, Artikel, Summen, Zahlung |
| Standardbeleg (RTL) | HTML | Rechts-nach-links-Variante für Arabisch, Hebräisch, Persisch, Urdu |
| Minimal / Modern | HTML | Dieselben Kerndaten wie Standard, auf weniger vertikalem Platz untergebracht |
| Detaillierter Beleg | HTML | Vollständige Steuerrechnung — SKU-Spalte, Einzelpreis, Steueraufschlüsselung nach Steuersatz, Adressen |
| Geschenkbeleg | HTML | Nur Artikel, Preise ausgeblendet. Enthält Geschenknachricht und Rückgaberichtlinie |
| Rechnung | HTML | Ganzseitige Rechnung im A4/Letter-Format mit optionalem Panel "Zahlungsanleitung" für unbezahlte Bestellungen |
| Packzettel | HTML | Begleitdokument für das Lager — Artikel + Mengen, Lieferadresse, keine Preise |
| Angebot / Kostenvoranschlag | HTML | Vorverkaufsdokument mit Preisen und Bedingungen — kein Zahlungsabschnitt |
| Schmaler Beleg | HTML | Beleg in Monospace-Schrift für schmales Papier oder HTML-fähige Thermodrucker |
| Einfacher Thermobeleg (58mm) | Thermodruck | Sauberes Thermodruck-Layout für 58mm |
| Einfacher Thermobeleg (80mm) | Thermodruck | Sauberes Thermodruck-Layout für 80mm — am häufigsten verwendet |
| Einfacher Thermobeleg 80mm (RTL) | Thermodruck | RTL-Gegenstück für 80mm. Benötigt einen Drucker mit arabischer Codepage (CP864 / Windows-1256) |
| Detaillierter Thermobeleg (58mm) | Thermodruck | Umfassendes Layout für 58mm — Adressen, Steueraufschlüsselung, Rückerstattungen, Zahlungen, Bedingungen, Barcode |
| Detaillierter Thermobeleg (80mm) | Thermodruck | Umfassendes Layout für 80mm — Adressen, Steueraufschlüsselung, Rückerstattungen, Zahlungen, Bedingungen, Barcode |
| Küchenbon | Thermisch | Nur Artikel, große Schrift, keine Preise — für Vorbereitungsstationen |

### Dokumentvorlagen von Drittanbietern {#third-party-document-templates}

Wenn **PDF Invoices & Packing Slips for WooCommerce** von WP Overnight aktiv ist, zeigt WCPOS in der Belegvorlagenliste auch **Rechnung (WP Overnight)** und **Lieferschein (WP Overnight)** an. Diese serverseitig gerenderten HTML-Vorlagen verwenden die Dokument-API von WP Overnight und übernehmen die in diesem Plugin konfigurierten Nummerierungen für Rechnungen und Lieferscheine, Layouts, Markenangaben, rechtlichen und steuerlichen Felder sowie Vorlagenanpassungen.

Sie sind nur verfügbar, solange das WP Overnight-Plugin aktiv ist, und benötigen zum Rendern eine Serververbindung. Verwenden Sie die mitgelieferten HTML- oder Thermovorlagen von WCPOS, wenn Offline-Druck erforderlich ist.

### Adaptive Steueranzeige {#adaptive-tax-display}

Die meisten mitgelieferten Vorlagen (Standard, Standard RTL, Minimal/Modern, Schmal, Rechnung, Angebot, Varianten von Thermisch Einfach) **passen sich automatisch an Ihre WooCommerce-Einstellung zur Steueranzeige an**:

- **Filialen mit Preisen inklusive Steuer** (EU/UK/AU) zeigen Bruttopreise mit einer Zeile "Steuer enthalten" an.
- **Filialen mit Preisen exklusive Steuer** (US/CA) zeigen Nettopreise an, wobei die Steuer als separate Zeile hinzugefügt wird.
- Die Gesamtsumme zeigt immer den tatsächlich berechneten Bruttobetrag an.

Die **Detailliert**-Familie ist eine formale Steuerrechnung und weist die Steuer unabhängig von der Filialeinstellung immer mit steuerexklusiven Zeilen und einer ausdrücklichen Aufschlüsselung aus.

## Funktionsweise {#how-it-works}

1. **Vorlage hinzufügen** — klicken Sie auf einer Galeriekarte auf **Vorlage verwenden**, oder erstellen Sie eine eigene Vorlage von Grund auf.
2. **Anpassen** mit dem In-App-Editor mit Live-Vorschau, oder in ChatGPT / Claude einfügen und die gewünschten Änderungen anfordern.
3. **Aktivieren** — schalten Sie den Umschalter **Aktiv** in *Ihre Vorlagen* ein. Jede aktive Belegvorlage erscheint im Dropdown auf dem Belegbildschirm an der Kasse.
4. **Drucken oder anzeigen** — Belege werden nach dem Checkout geöffnet, mit Optionen zum Drucken, per E-Mail senden oder Anzeigen auf dem Bildschirm.

Vorlagen werden mit einer standardisierten [Belegdaten-Payload](/receipts/receipt-data) gerendert, die Abschnitte für Filialinformationen, Positionen, Summen, Steuern, Zahlungen, Rückerstattungen und mehr enthält. Währungsfelder enthalten alle vorformatierte `_display`-Varianten (z. B. `$29.99` statt `29.99`).

## Pro-Funktionen {#pro-features}

Mit [WCPOS Pro](/getting-started/pro-license) kann jede Filiale eigene Vorlagenzuweisungen, eigenes Branding (Logo, Adresse, Kontaktdaten) und eine eigene Vorlagenreihenfolge haben — nützlich für den Betrieb mit mehreren Standorten, bei dem jede Filiale ihren eigenen Briefkopf druckt.
