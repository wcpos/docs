WCPOS enthält ein vollständiges Vorlagensystem für Belege, Rechnungen, Geschenkbelege, Lieferscheine, Küchentickets und mehr — alles verwaltet unter **WP Admin → POS → Vorlagen**.

:::tip Möchten Sie nur Ihren Beleg ändern?
Beginnen Sie mit **[Beleg anpassen](/receipts/customise)**. Dort werden die drei einfachsten Möglichkeiten erläutert — eine andere mitgelieferte Vorlage auswählen, KI um Anpassungen bitten oder manuell bearbeiten.
:::

## Template-Engines {#template-engines}

WCPOS unterstützt drei Template-Engines. Die Engine wird ausgewählt, wenn eine Vorlage erstellt oder auf einer Galeriekarte auf **Vorlage verwenden** geklickt wird, und kann **danach nicht mehr geändert werden** — erstellen Sie eine neue Vorlage, wenn Sie die Engine wechseln möchten.

<FeatureGrid>
  <FeatureCard icon="code" title="Logikloses HTML (empfohlen)">
    Mustache-Stil <code>{"{{variable}}"}</code> Platzhalter in einfachem HTML. Wird clientseitig gerendert, daher funktionieren Belege **offline**. Für Browserdruck und PDF-Belege verwenden.
  </FeatureCard>
  <FeatureCard icon="print" title="Thermo-XML">
    XML-Vorlagen, die sowohl eine Bildschirmvorschau als auch **ESC/POS-Befehle** für Thermo-Belegdrucker erzeugen. Dieselbe Vorlage, zwei Ausgaben. Für Epson, Star und andere Thermodrucker verwenden.
  </FeatureCard>
  <FeatureCard icon="terminal" title="Legacy-PHP">
    Serverseitige PHP-Vorlagen mit WooCommerce-Funktionen. Zur Abwärtskompatibilität mit vorhandenen <code>yourtheme/woocommerce-pos/receipt.php</code> Überschreibungen beibehalten. Zum Rendern ist eine Serververbindung erforderlich.
  </FeatureCard>
</FeatureGrid>

## Vorlagengalerie {#template-gallery}

Die Galerie enthält vorgefertigte Vorlagen, die unverändert verwendet oder angepasst werden können. Filtern Sie nach **Kategorie** (Beleg, Rechnung, Geschenkbeleg, Küchenbon, Angebot / Bestellung), **Format** (HTML / ESC/POS) und **Richtung** (LTR / RTL); zeigen Sie eine Vorschau mit den echten Daten Ihrer Filiale an. Die Fußzeile jeder Karte enthält einen Link **Vorschau** und die primäre Schaltfläche **Vorlage verwenden** — durch Klicken auf Vorlage verwenden wird eine bearbeitbare Kopie in *Ihre Vorlagen* erstellt.

### Dokumentvorlagen von Drittanbietern {#third-party-document-templates}

Wenn **PDF Invoices & Packing Slips for WooCommerce** von WP Overnight aktiv ist, zeigt WCPOS in der Belegvorlagenliste auch **Rechnung (WP Overnight)** und **Lieferschein (WP Overnight)** an. Diese serverseitig gerenderten HTML-Vorlagen delegieren an die Dokument-API von WP Overnight und verwenden die in diesem Plugin konfigurierten Rechnungs- und Lieferschein-Nummerierungen, Layouts, Markenanpassungen, Rechts-/Steuerfelder und Vorlagenanpassungen wieder.

Sie sind nur verfügbar, solange das WP Overnight-Plugin aktiv ist, und benötigen zum Rendern eine Serververbindung. Verwenden Sie die gebündelten HTML- oder Thermovorlagen von WCPOS, wenn Offline-Druck erforderlich ist.

### Adaptive Steueranzeige {#adaptive-tax-display}

Die meisten gebündelten Vorlagen (Standard, Standard RTL, Minimal/Modern, Schmal, Rechnung, Angebot, Varianten von Thermo Einfach) **passen sich automatisch an Ihre WooCommerce-Einstellung zur Steueranzeige an**:

- **Filialen mit Bruttopreisen** (EU/UK/AU) sehen Bruttopreise mit einer Zeile "Steuer enthalten".
- **Filialen mit Nettopreisen** (US/CA) sehen Nettopreise, bei denen die Steuer als separate Zeile hinzugefügt wird.
- Die Gesamtsumme zeigt immer den tatsächlich berechneten Bruttobetrag.

Die **Detailliert**-Familie ist eine formelle Steuerrechnung und weist die Steuer immer einzeln aus, mit Nettopositionen und einer ausdrücklichen Aufschlüsselung, unabhängig von der Filialeinstellung.

## Funktionsweise {#how-it-works}

1. **Vorlage hinzufügen** — klicken Sie auf einer Galerie-Karte auf **Vorlage verwenden**, oder erstellen Sie eine eigene Vorlage von Grund auf.
2. **Anpassen** mit dem In-App-Editor mit Live-Vorschau, oder in ChatGPT / Claude einfügen und die gewünschten Änderungen anfordern.
3. **Aktivieren** — schalten Sie den Umschalter **Aktiv** unter *Ihre Vorlagen* ein. Jede aktive Belegvorlage erscheint im Dropdown des Belegbildschirms am POS.
4. **Drucken oder anzeigen** — Belege öffnen sich nach dem Checkout mit Optionen zum Drucken, E-Mail-Versand oder Anzeigen auf dem Bildschirm.

Vorlagen werden mit einer standardisierten [Belegdaten-Nutzlast](/receipts/receipt-data) gerendert, die Abschnitte für Filialinformationen, Positionen, Summen, Steuern, Zahlungen, Rückerstattungen und mehr enthält. Alle Währungsfelder enthalten vorformatierte `_display`-Varianten (z. B. `$29.99` statt `29.99`).

## Pro-Funktionen {#pro-features}

Mit [WCPOS Pro](/getting-started/pro-license) kann jede Filiale eigene Vorlagenzuweisungen, Branding (Logo, Adresse, Kontaktdaten) und eine eigene Vorlagensortierung haben — nützlich für Betriebe mit mehreren Standorten, bei denen jede Filiale ihren eigenen Briefkopf druckt.
