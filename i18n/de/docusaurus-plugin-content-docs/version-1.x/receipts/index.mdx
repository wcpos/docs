WCPOS enthält ein vollständiges Vorlagensystem für Belege, Rechnungen, Geschenkbelege, Packzettel, Küchenzettel und mehr – alles verwaltet unter **WP Admin → POS → Vorlagen**.

:::tip Möchten Sie nur Ihren Beleg ändern?
Beginnen Sie mit **[Beleg anpassen](/receipts/customise)**. Dort werden die drei einfachsten Möglichkeiten beschrieben – eine andere mitgelieferte Vorlage auswählen, die KI um eine Anpassung bitten oder manuell bearbeiten.
:::

## Vorlagen-Engines {#template-engines}

WCPOS unterstützt drei Vorlagen-Engines. Die Engine wird beim Erstellen einer Vorlage oder beim Klicken auf **Vorlage verwenden** auf einer Galeriekarte festgelegt und **kann danach nicht mehr geändert werden** – erstellen Sie eine neue Vorlage, wenn Sie die Engine wechseln möchten.

<FeatureGrid>
  <FeatureCard icon="code" title="Logikloses HTML (empfohlen)">
    Mustache-Stil <code>{"{{variable}}"}</code> Platzhalter innerhalb von einfachem HTML. Wird clientseitig gerendert, sodass Belege **offline funktionieren**. Geeignet für Browser-Druck und PDF-Belege.
  </FeatureCard>
  <FeatureCard icon="print" title="Thermal-XML">
    XML-Vorlagen, die sowohl eine Bildschirmvorschau als auch **ESC/POS-Befehle** für Thermobon-Drucker erzeugen. Eine Vorlage, zwei Ausgaben. Geeignet für Epson, Star und andere Thermodrucker.
  </FeatureCard>
  <FeatureCard icon="terminal" title="Legacy-PHP">
    Serverseitige PHP-Vorlagen mit WooCommerce-Funktionen. Werden aus Gründen der Abwärtskompatibilität mit bestehenden <code>yourtheme/woocommerce-pos/receipt.php</code> Überschreibungen beibehalten. Zum Rendern ist eine Serververbindung erforderlich.
  </FeatureCard>
</FeatureGrid>

## Vorlagengalerie {#template-gallery}

Die Galerie enthält fertige Vorlagen, die direkt verwendet oder angepasst werden können. Filtern Sie nach **Kategorie** (Beleg, Rechnung, Geschenkbeleg, Küchenbon, Angebot / Bestellung), **Format** (HTML / ESC/POS) und **Richtung** (LTR / RTL); die Vorschau verwendet die echten Daten Ihrer Filiale. In der Fußzeile jeder Karte befinden sich ein **Vorschau**-Link und eine primäre Schaltfläche **Vorlage verwenden** — ein Klick auf Vorlage verwenden erstellt eine bearbeitbare Kopie unter *Ihre Vorlagen*.

| Vorlage | Engine | Beschreibung |
|----------|--------|-------------|
| Standardbeleg | HTML | Standardbeleg — Logo, Filialidentität, Artikel, Summen, Zahlung |
| Standardbeleg (RTL) | HTML | Rechts-nach-links-Variante für Arabisch, Hebräisch, Persisch und Urdu |
| Minimal / Modern | HTML | Dieselben Kerninformationen wie Standard, aber mit weniger vertikalem Platz |
| Detaillierter Beleg | HTML | Vollständige Steuerrechnung — SKU-Spalte, Einzelpreis, Steueraufschlüsselung nach Satz, Adressen |
| Geschenkbeleg | HTML | Nur Artikel, Preise ausgeblendet. Enthält Geschenknachricht und Rückgabebedingungen |
| Rechnung | HTML | Ganzseitige A4/Letter-Rechnung mit optionalem Bereich "So bezahlen Sie" für unbezahlte Bestellungen |
| Lieferschein | HTML | Lagerbegleitdokument — Artikel + Mengen, Lieferadresse, keine Preise |
| Angebot / Kostenvoranschlag | HTML | Vorverkaufsdokument mit Preisen und Bedingungen — kein Zahlungsbereich |
| Schmaler Beleg | HTML | Monospace-Beleg für schmales Papier oder HTML-fähige Thermodrucker |
| Einfacher Thermobeleg (58 mm) | Thermal | Übersichtliches 58-mm-Thermolayout |
| Einfacher Thermobeleg (80 mm) | Thermal | Übersichtliches 80-mm-Thermolayout — am häufigsten verwendet |
| Einfacher Thermobeleg 80 mm (RTL) | Thermal | RTL-Variante für 80 mm. Benötigt einen Drucker mit arabischer Codepage (CP864 / Windows-1256) |
| Detaillierter Thermobeleg (58 mm) | Thermal | Umfangreiches 58-mm-Layout — Adressen, Steueraufschlüsselung, Erstattungen, Zahlungen, Bedingungen, Barcode |
| Detaillierter Thermobeleg (80 mm) | Thermal | Umfangreiches 80-mm-Layout — Adressen, Steueraufschlüsselung, Erstattungen, Zahlungen, Bedingungen, Barcode |
| Küchenbon | Thermal | Nur Artikel, große Schrift, keine Preise — für Zubereitungsstationen |

### Dokumentvorlagen von Drittanbietern {#third-party-document-templates}

Wenn **PDF Invoices & Packing Slips for WooCommerce** von WP Overnight aktiv ist, zeigt WCPOS in der Belegvorlagenliste auch **Rechnung (WP Overnight)** und **Lieferschein (WP Overnight)** an. Diese serverseitig gerenderten HTML-Vorlagen delegieren an die Dokument-API von WP Overnight und verwenden die in diesem Plugin konfigurierten Rechnungs- und Lieferscheinnummern, Layouts, Branding, rechtlichen/steuerlichen Felder und Vorlagenanpassungen wieder.

Sie sind nur verfügbar, solange das WP-Overnight-Plugin aktiv ist, und benötigen zum Rendern eine Serververbindung. Verwenden Sie die mitgelieferten HTML- oder Thermovorlagen von WCPOS, wenn Offline-Druck erforderlich ist.

### Adaptive Steueranzeige {#adaptive-tax-display}

Die meisten mitgelieferten Vorlagen (Standard, Standard RTL, Minimal/Modern, Schmal, Rechnung, Angebot, einfache Thermovarianten) **passen sich automatisch an Ihre WooCommerce-Steueranzeige an**:

- **Filialen mit Preisen inklusive Steuer** (EU/UK/AU) sehen Bruttopreise mit einer Zeile "Steuer enthalten".
- **Filialen mit Preisen exklusive Steuer** (US/CA) sehen Nettopreise mit separat hinzugefügter Steuer.
- Die Gesamtsumme zeigt immer den tatsächlich berechneten Bruttobetrag.

Die **Detailliert**-Familie ist eine formelle Steuerrechnung und weist die Steuer unabhängig von der Filialeinstellung immer mit steuerexklusiven Zeilen plus ausdrücklicher Aufschlüsselung aus.

## So funktioniert es {#how-it-works}

1. **Vorlage hinzufügen** — auf einer Galeriekarte auf **Vorlage verwenden** klicken oder eine eigene Vorlage von Grund auf erstellen.
2. **Anpassen** — den integrierten Editor mit Live-Vorschau verwenden oder die Vorlage in ChatGPT / Claude einfügen und die gewünschten Änderungen anfordern.
3. **Aktivieren** — den Schalter **Aktiv** in *Ihre Vorlagen* umlegen. Jede aktive Belegvorlage erscheint im Dropdown der Belegansicht an der Kasse.
4. **Drucken oder anzeigen** — Belege werden nach dem Checkout geöffnet, mit Optionen zum Drucken, Versenden per E-Mail oder Anzeigen auf dem Bildschirm.

Vorlagen werden anhand eines standardisierten [Belegdaten-Payloads](/receipts/receipt-data) gerendert, der Abschnitte für Filialinformationen, Einzelpositionen, Summen, Steuern, Zahlungen, Rückerstattungen und mehr enthält. Alle Währungsfelder enthalten vorformatierte `_display`-Varianten (z. B. `$29.99` statt `29.99`).

## Pro-Funktionen {#pro-features}

Mit [WCPOS Pro](/getting-started/pro-license) kann jede Filiale eigene Vorlagenzuweisungen, eigenes Branding (Logo, Adresse, Kontaktdaten) und eine eigene Vorlagenreihenfolge erhalten — nützlich für Betriebe mit mehreren Standorten, bei denen jede Filiale ihren eigenen Briefkopf druckt.
