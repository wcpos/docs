WCPOS enthält ein vollständiges Vorlagensystem für Belege, Rechnungen, Geschenkbelege, Lieferscheine, Küchentickets und mehr. Alles wird unter **WP Admin → POS → Vorlagen** verwaltet.

:::tip Sie möchten nur Ihren Beleg ändern?
Beginnen Sie mit **[Beleg anpassen](/templates/customise)**. Dort werden die drei einfachsten Wege beschrieben: eine andere mitgelieferte Vorlage wählen, eine KI um Anpassungen bitten oder die Vorlage manuell bearbeiten.
:::

## Vorlagen-Engines

WCPOS unterstützt drei Vorlagen-Engines. Die Engine wird gewählt, wenn Sie eine Vorlage erstellen oder in der Galerie auf **Vorlage verwenden** klicken. Sie **kann danach nicht mehr geändert werden**. Erstellen Sie eine neue Vorlage, wenn Sie die Engine wechseln möchten.

<FeatureGrid>
  <FeatureCard icon="code" title="Logikloses HTML (empfohlen)">
    Mustache-ähnliche Platzhalter <code>{"{{variable}}"}</code> in einfachem HTML. Die Darstellung erfolgt clientseitig, daher **funktionieren Belege offline**. Nutzen Sie dies für Browserdruck und PDF-Belege.
  </FeatureCard>
  <FeatureCard icon="print" title="Thermisches XML">
    XML-Vorlagen, die sowohl eine Bildschirmvorschau als auch **ESC/POS-Befehle** für Thermo-Belegdrucker erzeugen. Eine Vorlage, zwei Ausgaben. Geeignet für Epson, Star und andere Thermodrucker.
  </FeatureCard>
  <FeatureCard icon="terminal" title="Veraltetes PHP">
    Serverseitige PHP-Vorlagen mit WooCommerce-Funktionen. Sie bleiben aus Kompatibilitätsgründen für bestehende Overrides in <code>yourtheme/woocommerce-pos/receipt.php</code> erhalten. Zum Rendern ist eine Serververbindung erforderlich.
  </FeatureCard>
</FeatureGrid>

## Vorlagengalerie

Die Galerie enthält fertige Vorlagen, die Sie direkt verwenden oder anpassen können. Filtern Sie nach **Kategorie** (Beleg, Rechnung, Geschenkbeleg, Küchenticket, Angebot / Bestellung), **Format** (HTML / ESC/POS) und **Richtung** (LTR / RTL). Die Vorschau verwendet echte Daten Ihrer Filiale. Jede Karte hat unten einen Link **Vorschau** und die Hauptaktion **Vorlage verwenden**. Dadurch wird eine bearbeitbare Kopie unter *Ihre Vorlagen* erstellt.

| Vorlage | Engine | Beschreibung |
|----------|--------|--------------|
| Standardbeleg | HTML | Standardbeleg mit Logo, Filialdaten, Artikeln, Summen und Zahlung |
| Standardbeleg (RTL) | HTML | Rechts-nach-links-Variante für Arabisch, Hebräisch, Persisch und Urdu |
| Minimal / Modern | HTML | Dieselben Kerndaten wie Standard, aber kompakter |
| Detaillierter Beleg | HTML | Vollständige Steuerrechnung mit SKU-Spalte, Einzelpreis, Steueraufschlüsselung und Adressen |
| Geschenkbeleg | HTML | Nur Artikel, Preise ausgeblendet. Enthält Geschenktext und Rückgaberichtlinie |
| Rechnung | HTML | A4/Letter-Rechnung mit optionalem Bereich "So bezahlen Sie" für unbezahlte Bestellungen |
| Packzettel | HTML | Begleitdokument für Lager: Artikel + Mengen, Lieferadresse, keine Preise |
| Angebot / Kostenvoranschlag | HTML | Vorverkaufsdokument mit Preisen und Bedingungen, ohne Zahlungsbereich |
| Schmaler Beleg | HTML | Monospace-Beleg für schmales Papier oder HTML-fähige Thermodrucker |
| Einfacher Thermobeleg (58 mm) | Thermisch | Sauberes 58-mm-Thermolayout |
| Einfacher Thermobeleg (80 mm) | Thermisch | Sauberes 80-mm-Thermolayout, die häufigste Variante |
| Einfacher Thermobeleg 80 mm (RTL) | Thermisch | RTL-Variante für 80 mm. Benötigt einen Drucker mit arabischer Codepage (CP864 / Windows-1256) |
| Detaillierter Thermobeleg (58 mm) | Thermisch | Umfangreicher 58-mm-Beleg mit Adressen, Steuern, Erstattungen, Zahlungen, Bedingungen und Barcode |
| Detaillierter Thermobeleg (80 mm) | Thermisch | Umfangreicher 80-mm-Beleg mit Adressen, Steuern, Erstattungen, Zahlungen, Bedingungen und Barcode |
| Küchenticket | Thermisch | Nur Artikel, große Schrift, keine Preise, für Zubereitungsstationen |

### Vorlagen für Drittanbieter-Dokumente

Wenn **PDF Invoices & Packing Slips for WooCommerce** von WP Overnight aktiv ist, zeigt WCPOS auch **Rechnung (WP Overnight)** und **Packzettel (WP Overnight)** in der Belegvorlagenliste an. Diese serverseitig gerenderten HTML-Vorlagen delegieren an die Dokument-API von WP Overnight und verwenden Rechnungs- und Packzettelnummern, Layout, Branding, rechtliche und steuerliche Felder sowie die dort konfigurierten Anpassungen weiter.

Sie sind nur verfügbar, solange das WP-Overnight-Plugin aktiv ist, und benötigen zum Rendern eine Serververbindung. Verwenden Sie die mitgelieferten HTML- oder Thermovorlagen von WCPOS, wenn Offline-Druck erforderlich ist.

### Adaptive Steueranzeige

Die meisten mitgelieferten Vorlagen (Standard, Standard RTL, Minimal/Modern, Schmal, Rechnung, Angebot und einfache Thermovarianten) **passen sich automatisch an die WooCommerce-Einstellung zur Steueranzeige an**:

- **Shops mit Steuer inklusive** (EU/UK/AU) sehen Bruttopreise mit einer Zeile "Steuer enthalten".
- **Shops mit Steuer exklusive** (US/CA) sehen Nettopreise, wobei die Steuer als separate Zeile ergänzt wird.
- Die Gesamtsumme zeigt immer den tatsächlich berechneten Bruttobetrag.

Die Familie **Detailliert** ist eine formale Steuerrechnung und weist Steuern immer mit Nettozeilen und einer ausdrücklichen Aufschlüsselung aus, unabhängig von der Shop-Einstellung.

## So funktioniert es

1. **Vorlage hinzufügen**: Klicken Sie in der Galerie auf **Vorlage verwenden** oder erstellen Sie eine eigene Vorlage von Grund auf.
2. **Anpassen**: Nutzen Sie den integrierten Editor mit Live-Vorschau oder fügen Sie die Vorlage in ChatGPT / Claude ein und bitten Sie um die gewünschten Änderungen.
3. **Aktivieren**: Schalten Sie den Regler **Aktiv** in *Ihre Vorlagen* ein. Jede aktive Belegvorlage erscheint im Beleg-Dropdown an der Kasse.
4. **Drucken oder anzeigen**: Belege öffnen sich nach dem Checkout mit Optionen zum Drucken, E-Mail-Versand oder Anzeigen auf dem Bildschirm.

Vorlagen werden gegen ein standardisiertes [Belegdaten-Payload](/templates/receipt-data) gerendert, mit Abschnitten für Filialdaten, Positionen, Summen, Steuern, Zahlungen, Erstattungen und mehr. Währungsfelder enthalten vorformatierte `_display`-Varianten, zum Beispiel `$29.99` statt `29.99`.

## Pro-Funktionen

Mit [WCPOS Pro](/getting-started/pro-license) kann jede Filiale eigene Vorlagenzuweisungen, Branding (Logo, Adresse, Kontaktdaten) und eine eigene Reihenfolge haben. Das ist nützlich für mehrere Standorte, bei denen jedes Geschäft seinen eigenen Briefkopf druckt.
