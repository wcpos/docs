WCPOS enthält ein vollständiges Vorlagensystem für Belege, Rechnungen, Geschenkbelege, Packzettel, Küchenzettel und mehr – alles verwaltet unter **WP Admin → POS → Vorlagen**.

:::tip Möchten Sie nur Ihren Beleg ändern?
Beginnen Sie mit **[Beleg anpassen](/templates/customise)**. Dort werden die drei einfachsten Möglichkeiten beschrieben – eine andere mitgelieferte Vorlage auswählen, die KI um eine Anpassung bitten oder manuell bearbeiten.
:::

## Vorlagen-Engines {#template-engines}

WCPOS unterstützt drei Vorlagen-Engines. Die Engine wird beim Erstellen einer Vorlage oder beim Klicken auf **Vorlage verwenden** auf einer Galeriekarte festgelegt und **kann danach nicht mehr geändert werden** – erstellen Sie eine neue Vorlage, wenn Sie die Engine wechseln möchten.

<FeatureGrid>
  <FeatureCard icon="code" title="Logikloses HTML (empfohlen)">
    Mustache-Stil <code>{"{{variable}}"}</code> Platzhalter innerhalb von einfachem HTML. Wird clientseitig gerendert, sodass Belege **offline funktionieren**. Geeignet für Browser-Druck und PDF-Belege.
  </FeatureCard>
  <FeatureCard icon="print" title="Thermal-XML">
    XML-Vorlagen, die sowohl eine Bildschirmvorschau als auch **ESC/POS-Befehle** für Thermobon-Drucker erzeugen. Eine Vorlage, zwei Ausgaben. Geeignet für Epson, Star und andere Thermodrucker.
  </FeatureCard>
  <FeatureCard icon="terminal" title="Legacy-PHP">
    Serverseitige PHP-Vorlagen mit WooCommerce-Funktionen. Werden aus Gründen der Abwärtskompatibilität mit bestehenden <code>yourtheme/woocommerce-pos/receipt.php</code> Überschreibungen beibehalten. Zum Rendern ist eine Serververbindung erforderlich.
  </FeatureCard>
</FeatureGrid>

## Vorlagengalerie {#template-gallery}

Die Galerie enthält fertige Vorlagen, die direkt verwendet oder angepasst werden können. Filtern Sie nach **Kategorie** (Beleg, Rechnung, Geschenkbeleg, Küchenbon, Angebot / Bestellung), **Format** (HTML / ESC/POS) und **Richtung** (LTR / RTL); die Vorschau verwendet die echten Daten Ihrer Filiale. In der Fußzeile jeder Karte befinden sich ein **Vorschau**-Link und eine primäre Schaltfläche **Vorlage verwenden** — ein Klick auf Vorlage verwenden erstellt eine bearbeitbare Kopie unter *Ihre Vorlagen*.

| Vorlage | Engine | Beschreibung |
|----------|--------|-------------|
| Standardbeleg | HTML | Standardbeleg — Logo, Filialidentität, Artikel, Summen, Zahlung |
| Standard Receipt (RTL) | HTML | Right-to-left counterpart for Arabic, Hebrew, Persian, Urdu |
| Minimal / Modern | HTML | Same essentials as Standard, packed into less vertical space |
| Detailed Receipt | HTML | Full tax invoice — SKU column, unit price, per-rate tax breakdown, addresses |
| Gift Receipt | HTML | Items only, prices hidden. Includes gift message and return policy |
| Invoice | HTML | Full-page A4/Letter invoice with optional "How to pay" panel for unpaid orders |
| Packing Slip | HTML | Warehouse companion — items + quantities, ship-to, no prices |
| Quote / Estimate | HTML | Pre-sale document with pricing and terms — no payment section |
| Narrow Receipt | HTML | Monospace receipt for narrow paper or HTML-capable thermal printers |
| Simple Thermal Receipt (58mm) | Thermal | Clean 58mm thermal layout |
| Simple Thermal Receipt (80mm) | Thermal | Clean 80mm thermal layout — most common |
| Simple Thermal Receipt 80mm (RTL) | Thermal | RTL counterpart for 80mm. Needs a printer with an Arabic codepage (CP864 / Windows-1256) |
| Detailed Thermal Receipt (58mm) | Thermal | Kitchen-sink 58mm — addresses, tax breakdown, refunds, payments, terms, barcode |
| Detailed Thermal Receipt (80mm) | Thermal | Kitchen-sink 80mm — addresses, tax breakdown, refunds, payments, terms, barcode |
| Kitchen Ticket | Thermal | Items only, large font, no pricing — for prep stations |

### Third-party document templates {#third-party-document-templates}

If **PDF Invoices & Packing Slips for WooCommerce** by WP Overnight is active, WCPOS also exposes **Invoice (WP Overnight)** and **Packing Slip (WP Overnight)** in the receipt template list. These server-rendered HTML templates delegate to WP Overnight's document API, reusing the invoice and packing-slip numbering, layout, branding, legal/tax fields, and template customisations configured in that plugin.

They are available only while the WP Overnight plugin is active and require a server connection to render. Use WCPOS's bundled HTML or thermal templates when offline printing is required.

### Adaptive tax display {#adaptive-tax-display}

Most bundled templates (Standard, Standard RTL, Minimal/Modern, Narrow, Invoice, Quote, Thermal Simple variants) **adapt automatically to your WooCommerce tax-display setting**:

- **Tax-inclusive stores** (EU/UK/AU) see gross prices with a "Tax included" line.
- **Tax-exclusive stores** (US/CA) see net prices with tax added as a separate line.
- The grand total always shows the gross amount actually charged.

The **Detailed** family is a formal tax invoice and always itemises tax with tax-exclusive lines plus an explicit breakdown, regardless of the store setting.

## How it works {#how-it-works}

1. **Vorlage hinzufügen** — auf einer Galeriekarte auf **Vorlage verwenden** klicken oder eine eigene Vorlage von Grund auf erstellen.
2. **Anpassen** — den integrierten Editor mit Live-Vorschau verwenden oder die Vorlage in ChatGPT / Claude einfügen und die gewünschten Änderungen beschreiben.
3. **Aktivieren** — den Schalter **Aktiv** unter *Ihre Vorlagen* umlegen. Jede aktive Belegvorlage erscheint im Dropdown der Belegansicht an der Kasse.
4. **Drucken oder anzeigen** — Belege werden nach dem Checkout mit Optionen zum Drucken, per E-Mail versenden oder auf dem Bildschirm anzeigen geöffnet.

Vorlagen werden anhand eines standardisierten [Belegdaten-Payloads](/templates/receipt-data) gerendert, der Abschnitte für Filialinformationen, Einzelpositionen, Summen, Steuern, Zahlungen, Rückerstattungen und mehr enthält. Alle Währungsfelder enthalten vorformatierte `_display`-Varianten (z. B. `$29.99` statt `29.99`).

## Pro-Funktionen {#pro-features}

Mit [WCPOS Pro](/getting-started/pro-license) kann jede Filiale eigene Vorlagenzuweisungen, eigenes Branding (Logo, Adresse, Kontaktdaten) und eine eigene Vorlagenreihenfolge erhalten — nützlich für Betriebe mit mehreren Standorten, bei denen jede Filiale ihren eigenen Briefkopf druckt.
