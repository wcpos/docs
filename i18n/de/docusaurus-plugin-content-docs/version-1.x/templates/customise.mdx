Wenn Sie das Aussehen Ihres Belegs ändern möchten, haben Sie drei Möglichkeiten. Wählen Sie die einfachste, die Ihren Anforderungen entspricht — die meisten Filialen kommen mit der ersten Option aus.

## Drei Wege zur Anpassung {#three-ways-to-customise}

<FeatureGrid>
  <FeatureCard icon="grid" title="1. Eine andere Vorlage wählen">
    Verwenden Sie eine der fertigen Vorlagen aus der Galerie. **Kein Code erforderlich.** Geeignet für: ein anderes Layout, Preise ausblenden, eine A4-Rechnung, einen Küchenbon.
  </FeatureCard>
  <FeatureCard icon="sparkles" title="2. KI um Anpassungen bitten">
    Fügen Sie die Vorlage in ChatGPT oder Claude ein und beschreiben Sie, was Sie ändern möchten. **Keine Programmierkenntnisse erforderlich** — beschreiben Sie es einfach in Ihren eigenen Worten. Geeignet für: kleine Änderungen wie Formulierungen, Farben oder das Verschieben von Elementen.
  </FeatureCard>
  <FeatureCard icon="code" title="3. Manuell bearbeiten">
    Der integrierte Editor ermöglicht es, die Vorlage direkt zu bearbeiten. Geeignet für: präzise Kontrolle oder wenn Sie bereits HTML-Kenntnisse haben.
  </FeatureCard>
</FeatureGrid>

Alle drei beginnen am selben Ort: **WP Admin → POS → Vorlagen**. Die Seite besteht aus zwei Bereichen — **Ihre Vorlagen** oben (die aktuell verwendeten) und die **Vorlagengalerie** darunter (die Starter-Bibliothek).

## Option 1 — Eine andere Vorlage auswählen {#option-1-pick-a-different-template}

Dies ist der einfachste Weg und deckt die meisten Anforderungen ab.

<Steps>
  <StepCard number="1" title="Die Vorlagengalerie öffnen">

Navigieren Sie in WP Admin zu **POS → Vorlagen**. Scrollen Sie an *Ihre Vorlagen* vorbei zum Abschnitt **Vorlagengalerie** — das ist die Startbibliothek.

  </StepCard>
  <StepCard number="2" title="Durchsuchen und Vorschau anzeigen">

Filtern Sie nach **Kategorie** (Beleg, Rechnung, Geschenkbeleg, Küchenbon, Angebot / Bestellung), **Format** (HTML für Browserdruck, ESC/POS für Thermodrucker) oder **Ausrichtung** (Links-nach-rechts oder Rechts-nach-links). Klicken Sie auf die Miniaturansicht einer Karte — oder den Link **Vorschau** in der Fußzeile — um eine Live-Vorschau mit den realen Daten Ihrer Filiale zu öffnen.

  </StepCard>
  <StepCard number="3" title="Verwenden" isLast>

Klicken Sie auf der Karte auf **Vorlage verwenden**. WCPOS erstellt eine bearbeitbare Kopie und fügt sie oben auf der Seite unter **Ihre Vorlagen** hinzu. Aktivieren Sie den Schalter **Aktiv** in der Zeile, um die Vorlage für Belege zu verwenden; ziehen Sie am Anfasser der Zeile, um die Reihenfolge zu ändern. Es können mehrere gleichzeitig aktiv sein — die Auswahl erfolgt an der Kasse.

  </StepCard>
</Steps>

:::tip Vorlage verwenden ersetzt nichts
Durch Klicken auf **Vorlage verwenden** wird immer eine neue Kopie erstellt. Die ursprüngliche Galerievorlage bleibt unverändert, sodass jederzeit ein anderer Ausgangspunkt gewählt werden kann. Wenn mehrere Belegvorlagen aktiv sind, zeigt der Belegbildschirm ein Dropdown-Menü an, über das an der Kasse zwischen ihnen gewechselt werden kann.
:::

### Die mitgelieferten Vorlagen {#the-bundled-templates}

| Vorlage | Format | Wofür sie gedacht ist |
|---|---|---|
| **Standardbeleg** | HTML | Standard — Logo, Artikel, Summen, Zahlung. Deckt die meisten Filialen ab |
| **Standardbeleg (RTL)** | HTML | Wie Standard, gespiegelt für Arabisch / Hebräisch / Persisch / Urdu |
| **Minimal / Modern** | HTML | Gleiche Angaben wie Standard, auf weniger vertikalen Raum komprimiert |
| **Detaillierter Beleg** | HTML | Vollständige Steuerrechnung — SKU-Spalte, Stückpreis, Steueraufschlüsselung pro Satz, Adressen |
| **Geschenkbeleg** | HTML | Nur Artikel — Preise ausgeblendet. Enthält Geschenknachricht und Rückgaberichtlinie |
| **Rechnung** | HTML | Ganzseitige Rechnung im A4-/Letter-Format. Fügt bei unbezahlten Bestellungen ein „So bezahlen Sie“-Feld hinzu |
| **Lieferschein** | HTML | Begleitdokument für das Lager — Artikel + Mengen, Lieferadresse, keine Preise |
| **Angebot / Kostenvoranschlag** | HTML | Vorverkaufsdokument mit Preisen und Bedingungen — kein Zahlungsabschnitt |
| **Schmaler Beleg** | HTML | Beleg in Festbreitenschrift für schmales Papier oder HTML-fähige Thermodrucker |
| **Einfacher Thermobeleg (58 mm)** | Thermisch | Klares 58-mm-Thermolayout |
| **Einfacher Thermobeleg (80 mm)** | Thermisch | Klares 80-mm-Thermolayout — am gebräuchlichsten |
| **Einfacher Thermobeleg 80 mm (RTL)** | Thermisch | RTL-Variante für 80 mm. Erfordert einen Drucker mit arabischer Codepage |
| **Detaillierter Thermobeleg (58 mm / 80 mm)** | Thermisch | Fügt Steueraufschlüsselung, Adressen, Erstattungen, Zahlungen, Bedingungen und Barcode hinzu |
| **Küchenbon** | Thermisch | Nur Artikel, große Schrift, keine Preise — für Zubereitungsstationen |

Die meisten der mitgelieferten Vorlagen **passen sich automatisch an die Steuereinstellungen Ihrer Filiale an** — bei Filialen mit Bruttopreisen werden Bruttopreise und eine Zeile „Inkl. Steuer“ angezeigt; bei Filialen mit Nettopreisen werden Nettopreise mit separat ausgewiesener Steuer angezeigt. Die **Detailliert**-Familie zeigt unabhängig von dieser Einstellung stets eine vollständige Steueraufschlüsselung.

## Filialspezifische Zuweisungen {#per-store-assignments}

Wenn Sie mehr als eine [Filiale](/stores) (Pro) haben, kann jede Filiale ihre **eigene Vorlagenauswahl und -reihenfolge** haben, getrennt von den websiteweiten Standardwerten. Das Café um die Ecke kann einen kleinen Thermobeleg mit anderem Logo und anderer Adresse verwenden; das Lager kann einen Lieferschein nutzen; der Hauptladen kann beim Standardbeleg bleiben — alles aus derselben Vorlagengalerie.

Richten Sie dies unter **WP Admin → POS → Filialen** ein und öffnen Sie dann die zu konfigurierende Filiale. Die Seite **Filiale bearbeiten** enthält einen Abschnitt **Belegvorlagen** mit einem Schalter *„Filialspezifische Belegvorlagen“*:

- **Schalter aus** *(Standard)* — die Filiale übernimmt die websiteweite Vorlagenliste von der Hauptseite **POS → Vorlagen**.
- **Schalter ein** — die Filiale erhält ihre eigene Vorlagenauswahl und -reihenfolge, getrennt von den websiteweiten Standardwerten. Das Umsortieren per Anfasser funktioniert auf dieselbe Weise.

Auf derselben Seite „Filiale bearbeiten“ befindet sich auch der **Briefkopf** jeder Filiale (Logo, Adresse, Kontaktdaten und der Block *Belegnachrichten* — Grußformel, Rückgaberichtlinie, Fußzeile). Die mitgelieferten Vorlagen greifen auf diese filialspezifischen Felder zu, sodass eine einzelne Vorlage „Standardbeleg“ an verschiedenen Standorten unterschiedliches Branding tragen kann.

Wenn sich ein Kassierer in einer Filiale anmeldet, erscheinen im Beleg-Dropdown nur die aktiven Vorlagen dieser Filiale.

:::tip Websiteweit vs. pro Filiale
Die Seite **Vorlagen** in WP Admin legt die Standardwerte für die gesamte Website fest. Die filialspezifische Überschreibung ermöglicht es, dass eine einzelne Vorlage (z. B. ein Standardbeleg) an verschiedenen Standorten unterschiedliches Branding tragen kann oder dass ein Standort ein Layout verwenden kann, das die anderen nicht nutzen. Wenn alle Filialen dieselben Vorlagen verwenden sollen, lassen Sie die filialspezifischen Zuweisungen einfach leer – die websiteweiten Standardwerte werden dann übernommen.
:::

## Option 2 — KI um Anpassungen bitten {#option-2-ask-ai-to-tweak-it}

Wenn die Galerie fast, aber nicht ganz passt, kann ein KI-Assistent die Anpassungen in wenigen Minuten vornehmen – HTML-Kenntnisse sind dafür nicht erforderlich.

<Steps>
  <StepCard number="1" title="Vorlage kopieren">

Öffnen Sie die gewünschte Ausgangsvorlage unter **WP Admin → POS → Vorlagen**, klicken Sie in den Editor und markieren Sie den gesamten Text auf der linken Seite (Strg/Cmd + A). Kopieren Sie ihn.

  </StepCard>
  <StepCard number="2" title="In ChatGPT oder Claude einfügen">

Öffnen Sie [ChatGPT](https://chat.openai.com) oder [Claude](https://claude.ai). Fügen Sie die Vorlage ein und beschreiben Sie dann in einfachen Worten, was Sie ändern möchten:

  </StepCard>
  <StepCard number="3" title="Gewünschte Änderungen beschreiben">

Beschreiben Sie der KI genau, was Sie möchten. Beispiele, die gut funktionieren:

- *„Den Filialnamen größer und zentriert darstellen.“*
- *„Unten eine Dankesnachricht in Kursivschrift hinzufügen.“*
- *„Den Kundennamen ausblenden. Stattdessen die Telefonnummer unter der Bestellnummer einfügen.“*
- *„Den Barcode in einen QR-Code umwandeln, der auf meine Rückgabeseite verweist.“*
- *„Einen Slogan ‚Familienbetrieb seit 1987‘ unter dem Filialnamen hinzufügen.“*

Die KI gibt Ihnen eine angepasste Vorlage zurück.

  </StepCard>
  <StepCard number="4" title="Wieder einfügen" isLast>

Kopieren Sie die Antwort der KI. Markieren Sie zurück im WCPOS-Vorlageneditor alles (Strg/Cmd + A), fügen Sie die neue Version ein und klicken Sie auf **Aktualisieren**. Die Vorschau auf der rechten Seite wird aktualisiert, sodass Sie das Ergebnis sehen können. Sieht es nicht richtig aus, bitten Sie die KI um Korrektur — beschreiben Sie, was schiefgelaufen ist.

  </StepCard>
</Steps>

:::tip Empfohlene Vorgehensweise
Jeder Klick auf **Vorlage verwenden** in der Galerie erstellt eine neue bearbeitbare Kopie, sodass das Original geschützt bleibt. Wenn Sie experimentieren, können Sie dieselbe Galerievorlage mehrfach verwenden — benennen Sie Ihre Kopien um *(Beleg v1, Beleg v2)* und wechseln Sie zwischen ihnen, während Sie sich entscheiden.
:::

:::info Was ist mit Variablen?
Bestandteile wie `{{store.name}}` und `{{order.number}}` sind **Platzhalter** für Ihre echten Daten. Die KI versteht diese — Sie müssen das nicht. Wenn Sie alle verfügbaren Platzhalter kennenlernen möchten, siehe [Belegdaten-Referenz](/templates/receipt-data).
:::

## Option 3 — Manuell bearbeiten {#option-3-edit-it-by-hand}

Wenn Sie etwas HTML kennen (oder mit einem Entwickler zusammenarbeiten), können Sie die Vorlage direkt im integrierten Editor bearbeiten. Der Editor bietet eine Live-Vorschau, Syntaxhervorhebung, eine durchsuchbare Feldauswahl, Rückgängig/Wiederherstellen und Suchen-und-Ersetzen.

Wählen Sie Ihre Engine:

- **[HTML-Vorlagen](/templates/html-templates)** — Mustache-artige `{{variable}}`-Platzhalter. Wird clientseitig gerendert und funktioniert offline. **Empfohlen für die meisten Filialen.**
- **[Thermovorlagen](/templates/thermal-templates)** — XML für ESC/POS-Thermodrucker. Dieselbe Vorlage erzeugt sowohl die Bildschirmvorschau als auch die Druckerausgabe.
- **[Belegdaten-Referenz](/templates/receipt-data)** — Alle verfügbaren Platzhalter, nach Abschnitt gruppiert.

:::note Ältere PHP-Vorlagen
Wenn Sie den Beleg früher mit einer PHP-Datei in Ihrem Theme überschrieben haben (`yourtheme/woocommerce-pos/receipt.php`), funktioniert das weiterhin. Diese Option ist jetzt in der Galerie als **Ältere PHP-Vorlage** gekennzeichnet und steht neben den neuen logiklosen und Thermo-Engines. Neue Anpassungen sollten stattdessen die Galerie oder den integrierten Editor verwenden — sie funktionieren offline, bieten eine Live-Vorschau und benötigen keinen Server-Roundtrip.
:::

## Häufige Anpassungen {#common-customisations}

Schnelle Antworten auf die am häufigsten gestellten Fragen.

<Accordion>
  <AccordionItem question="Wie füge ich mein Filiallogo hinzu?">

Logos stammen aus Ihren Filialeinstellungen, nicht aus der Vorlage selbst. Gehen Sie zu **WP Admin → POS → Einstellungen → Filialen**, bearbeiten Sie Ihre Filiale und laden Sie dort ein Logo hoch. Jede mitgelieferte Vorlage, die ein Logo anzeigt, verwendet es automatisch.

Wenn Sie ändern möchten, *wo* das Logo in der Vorlage erscheint, bearbeiten Sie die Vorlage und verschieben Sie den `{{#store.logo}}<img src="{{store.logo}}" ...>{{/store.logo}}` Block an die gewünschte Stelle.

  </AccordionItem>

  <AccordionItem question="Wie ändere ich den Fußzeilentext (z. B. „Vielen Dank für Ihren Einkauf!“)?">

Zwei Möglichkeiten:

1. **Am einfachsten** – legen Sie den Text einmalig für alle Belege fest unter **WP Admin → POS → Einstellungen → Filialen → Filialdetails → Bonfußzeile / persönliche Notiz**. Mitgelieferte Vorlagen übernehmen diesen automatisch; ist keine Fußzeile festgelegt, wird ein freundlicher Standardtext wie *"Vielen Dank für Ihren Einkauf!"* verwendet.
2. **In einer einzelnen Vorlage** – bearbeiten Sie die Vorlage und ersetzen Sie den Fußzeilentext direkt. Suchen Sie nach `{{store.personal_notes}}` oder der wörtlichen Dankeszeile.

  </AccordionItem>

  <AccordionItem question="Wie füge ich einen Slogan unter dem Filialnamen hinzu?">

Bearbeiten Sie die Vorlage und fügen Sie eine Zeile unter `{{store.name}}` hinzu:

```html
<div style="font-style: italic; color: #6b7280;">Family-owned since 1987</div>
```

In einer Thermodruck-Vorlage:

```xml
<align mode="center"><text>Family-owned since 1987</text></align>
```

  </AccordionItem>

  <AccordionItem question="Wie blende ich Preise aus (für einen Geschenkbeleg)?">

Klicken Sie auf **Vorlage verwenden** auf der Karte **Geschenkbeleg** in der Galerie – alle Preise und Summen werden ausgeblendet, während Artikel, SKU, Attribute und die Geschenknachricht weiterhin angezeigt werden. Keine Bearbeitung erforderlich.

Wenn Sie lieber einen eigenen preisfreien Beleg erstellen möchten, kopieren Sie eine beliebige Vorlage und löschen Sie die Blöcke `{{...total...}}`, `{{...price...}}` und `{{#totals}}...{{/totals}}`.

  </AccordionItem>

  <AccordionItem question="Wie wandle ich den Barcode in einen QR-Code um?">

Suchen Sie das `<barcode>` Element in Ihrer Vorlage und ändern Sie das `type` Attribut:

```html
<!-- Default: Code 128 barcode of the order number -->
<barcode type="code128">{{order.number}}</barcode>

<!-- Switch to a QR code -->
<barcode type="qrcode">{{order.number}}</barcode>

<!-- Or encode something else, like a returns URL -->
<barcode type="qrcode">https://example.com/returns?order={{order.number}}</barcode>
```

Dieselbe `<barcode>` Syntax funktioniert sowohl in HTML- als auch in Thermovorlagen. Weitere unterstützte Typen umfassen `ean13`, `ean8`, `upca`, `pdf417` und [alles, was bwip-js unterstützt](https://github.com/metafloor/bwip-js/wiki/Supported-Barcode-Types).

  </AccordionItem>

  <AccordionItem question="Wie sende ich eine andere Vorlage an einen bestimmten Drucker?">

Navigieren Sie in der POS-App zu **Einstellungen → Druckerzuordnung**. Dort werden alle aktiven Vorlagen mit einem Drucker-Dropdown angezeigt. Wählen Sie den gewünschten Drucker aus oder belassen Sie die Einstellung auf **Automatisch**.

- **Automatisch** ordnet Vorlagen automatisch Druckern zu — Thermovorlagen werden an Thermodrucker gesendet, HTML-Vorlagen an den Systemdruckdialog.
- Ein **bestimmter Drucker** überschreibt die automatische Zuordnung und sendet die Vorlage immer an diesen Drucker.
- Beim Drucken kann der Kassierer beide oben genannten Optionen über das Drucker-Dropdown auf dem Belegbildschirm überschreiben.

Die Zuordnung wird pro Gerät gespeichert, sodass jedes iPad oder jeder Computer eine eigene Konfiguration haben kann.

  </AccordionItem>

  <AccordionItem question="Mein Beleg zeigt nach dem Bearbeiten immer noch die alte Version">

Klicken Sie auf dem Vorlagen-Bearbeitungsbildschirm auf die WordPress-Schaltfläche **Aktualisieren**. Der Editor speichert nicht automatisch — Ihre Änderungen bleiben nur erhalten, wenn Sie auf Aktualisieren klicken.

Bei **älteren PHP-Vorlagen** zeigt die Vorschau im Editor die *zuletzt gespeicherte* Version, nicht das, was Sie gerade eingeben. Speichern Sie zuerst und aktualisieren Sie dann die Vorschau.

  </AccordionItem>

  <AccordionItem question="Die Vorschau ist leer oder zeigt „Keine POS-Bestellungen gefunden“">

Dies tritt nur bei **älteren PHP-Vorlagen** auf, die zur Vorschau eine echte Bestellung benötigen. Verarbeiten Sie eine einzelne POS-Bestellung — auch ein Testverkauf über 0 $ — und die Vorschau funktioniert.

Logiklose (HTML) und Thermovorlagen verfügen immer über Beispieldaten als Rückfallebene, sodass die Vorschau auch in einer brandneuen Filiale problemlos funktioniert.

  </AccordionItem>

  <AccordionItem question="Ich habe alles durcheinandergebracht — wie fange ich neu an?">

Drei Sicherheitsnetze:

1. Der Editor verfügt über **Rückgängig** (Strg/Cmd + Z) für Änderungen innerhalb der Sitzung.
2. Jedes Speichern erstellt eine WordPress-**Revision** — öffnen Sie **Revisionen** auf dem Bearbeitungsbildschirm, um frühere Versionen zu vergleichen und wiederherzustellen.
3. Wenn Sie von einer Galerievorlage ausgegangen sind, klicken Sie bei Ihrer Kopie unter *Ihre Vorlagen* auf **Löschen** und anschließend erneut auf **Vorlage verwenden** auf derselben Galeriekarte. Sie erhalten eine frische, unveränderte Kopie.

  </AccordionItem>
</Accordion>

## Wann Sie Hilfe anfordern sollten {#when-to-ask-for-help}

- Der Vorlageneditor lädt nicht oder Speicherungen werden nicht übernommen.
- Der Beleg wird auf einem Gerät einwandfrei gedruckt, auf einem anderen jedoch nicht.
- Sie benötigen ein fiskalisches/gesetzliches Layout für ein bestimmtes Land (Italien, Brasilien, Spanien usw.) — diese werden in der Regel von [WCPOS Pro](/getting-started/pro-license) oder einer länderspezifischen Integration bereitgestellt.
- Sie versuchen etwas Individuelles umzusetzen und die KI liefert nicht ganz das richtige Ergebnis.

Erstellen Sie ein [Support-Ticket](https://wcpos.com/support) und fügen Sie die Vorlage ein, an der Sie arbeiten — so haben wir alles, was wir zur Unterstützung benötigen.
