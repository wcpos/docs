Wenn Sie das Aussehen Ihres Belegs ändern möchten, stehen Ihnen drei Möglichkeiten zur Verfügung. Wählen Sie die einfachste, die Ihren Anforderungen entspricht — die meisten Filialen benötigen nur die erste.

## Drei Möglichkeiten zur Anpassung {#three-ways-to-customise}

<FeatureGrid>
  <FeatureCard icon="grid" title="1. Eine andere Vorlage wählen">
    Verwenden Sie eine der fertigen Vorlagen aus der Galerie. **Kein Code erforderlich.** Ideal für: ein anderes Layout, Preise ausblenden, eine A4-Rechnung, einen Küchenbon.
  </FeatureCard>
  <FeatureCard icon="sparkles" title="2. KI um Anpassung bitten">
    Fügen Sie die Vorlage in ChatGPT oder Claude ein und beschreiben Sie, was Sie möchten. **Keine Programmierkenntnisse erforderlich** — die Beschreibung erfolgt in einfacher Sprache. Ideal für: kleine Anpassungen wie Formulierungen, Farben oder das Verschieben von Elementen.
  </FeatureCard>
  <FeatureCard icon="code" title="3. Manuell bearbeiten">
    Der integrierte Editor ermöglicht die direkte Bearbeitung der Vorlage. Ideal für: präzise Kontrolle oder wenn Sie bereits HTML-Kenntnisse haben.
  </FeatureCard>
</FeatureGrid>

Alle drei beginnen am selben Ort: **WP Admin → POS → Vorlagen**. Die Seite besteht aus zwei Bereichen — **Ihre Vorlagen** oben (die derzeit verwendeten) und die **Vorlagengalerie** darunter (die Starter-Bibliothek).

## Option 1 — Pick a different template {#option-1-pick-a-different-template}

This is the easiest path and covers most needs.

<Steps>
  <StepCard number="1" title="Open the template gallery">

In WP Admin go to **POS → Templates**. Scroll past *Your Templates* to the **Template Gallery** section — that's the starter library.

  </StepCard>
  <StepCard number="2" title="Browse and preview">

Filter by **category** (Receipt, Invoice, Gift Receipt, Kitchen Ticket, Quote / Purchase Order), **format** (HTML for browser print, ESC/POS for thermal printers), or **direction** (Left-to-right or Right-to-left). Click any card's thumbnail — or the **Preview** link in its footer — to open a live preview with your store's real data.

  </StepCard>
  <StepCard number="3" title="Use it" isLast>

Click **Use Template** on the card. WCPOS makes you an editable copy and adds it to **Your Templates** at the top of the page. Flip the **Active** toggle on the row to start using it on receipts; drag the row's grip handle to reorder. You can have several active at once — the cashier picks at the till.

  </StepCard>
</Steps>

:::tip Use Template never replaces anything
Clicking **Use Template** always creates a fresh copy. The original gallery template is left untouched, so you can come back and pick a different starting point any time. If multiple receipt templates are active, the receipt screen shows a dropdown so the cashier can switch between them on the fly.
:::

### The bundled templates {#the-bundled-templates}

| Vorlage | Format | Verwendungszweck |
|---|---|---|
| **Standardbeleg** | HTML | Standard — Logo, Artikel, Summen, Zahlung. Geeignet für die meisten Filialen |
| **Standardbeleg (RTL)** | HTML | Wie Standard, gespiegelt für Arabisch / Hebräisch / Persisch / Urdu |
| **Minimal / Modern** | HTML | Gleiche Informationen wie Standard, kompakter in der Höhe |
| **Detaillierter Beleg** | HTML | Vollständige Steuerrechnung — SKU-Spalte, Einzelpreis, Steueraufschlüsselung nach Satz, Adressen |
| **Geschenkbeleg** | HTML | Nur Artikel — Preise ausgeblendet. Enthält Geschenknachricht und Rückgabebedingungen |
| **Rechnung** | HTML | Ganzseitige A4/Letter-Rechnung. Enthält einen "So bezahlen Sie"-Bereich für unbezahlte Bestellungen |
| **Lieferschein** | HTML | Lagerbegleitdokument — Artikel + Mengen, Lieferadresse, keine Preise |
| **Angebot / Kostenvoranschlag** | HTML | Vorverkaufsdokument mit Preisen und Bedingungen — kein Zahlungsbereich |
| **Schmaler Beleg** | HTML | Monospace-Beleg für schmales Papier oder HTML-fähige Thermodrucker |
| **Einfacher Thermobeleg (58 mm)** | Thermal | Übersichtliches 58-mm-Thermolayout |
| **Simple Thermal Receipt (80mm)** | Thermal | Clean 80mm thermal layout — most common |
| **Simple Thermal Receipt 80mm (RTL)** | Thermal | RTL counterpart for 80mm. Requires a printer with an Arabic codepage |
| **Detailed Thermal Receipt (58mm / 80mm)** | Thermal | Adds tax breakdown, addresses, refunds, payments, terms, barcode |
| **Kitchen Ticket** | Thermal | Items only, large font, no prices — for prep stations |

Most of the bundled templates **adapt to your store's tax settings automatically** — tax-inclusive stores see gross prices and a "Tax included" line; tax-exclusive stores see net prices with tax added as a separate line. The **Detailed** family always shows a full tax breakdown regardless of the setting.

### WP Overnight invoice and packing slip templates {#wp-overnight-invoice-and-packing-slip-templates}

If your site also uses [PDF Invoices & Packing Slips for WooCommerce](https://wordpress.org/plugins/woocommerce-pdf-invoices-packing-slips/) by WP Overnight, WCPOS automatically adds two extra templates to **Your Templates**:

| Vorlage | Format | Verwendungszweck |
|---|---|---|
| **Invoice (WP Overnight)** | Server-rendered HTML | Uses WP Overnight's configured invoice document for the POS order |
| **Packing Slip (WP Overnight)** | Server-rendered HTML | Uses WP Overnight's configured packing-slip document for the POS order |

These templates do not copy WCPOS's built-in invoice or packing-slip layouts. They ask WP Overnight to render the document for the POS order, so your existing invoice numbers, branding, legal/tax fields, and WP Overnight template customisations stay consistent between online and in-store orders.

They appear only while the WP Overnight plugin is active. The output opens as HTML in the WCPOS print screen rather than as a separate PDF download. Because the document is rendered on the server, the POS needs a connection to your site when printing these templates; use the bundled HTML or thermal templates for offline printing.

## Per-store assignments {#per-store-assignments}

If you have more than one [store](/stores) (Pro), each store can have its **own template selection and ordering**, separate from the site-wide defaults. The cafe down the road can run a small thermal receipt with a different logo and address; the warehouse can use a packing slip; the main shop can keep the standard receipt — all from the same template gallery.

Set it up from **WP Admin → POS → Stores**, then open the store you want to configure. The **Edit Store** page has a **Receipt Templates** section with a *"Store specific receipt templates"* toggle:

- **Toggle off** *(default)* — the store inherits the site-wide template list from the main **POS → Templates** page.
- **Toggle on** — the store gets its own template selection and ordering, separate from the site-wide defaults. Drag-handle reorder works the same way.

The same Edit Store page is also where each store's **letterhead** lives (logo, address, contact details, and the *Receipt Messages* block — Complimentary Close, Returns Policy, Footer). The bundled templates pull from these per-store fields, so a single "Standard Receipt" template can carry different branding at different locations.

When a cashier signs in at a store, only that store's active templates appear in the receipt dropdown.

:::tip Site-wide vs per-store
The **Templates** page in WP Admin sets the default for the whole site. The per-store override exists so a single template (e.g. a Standard Receipt) can carry different branding at different locations, or so one location can use a layout the others don't. If all your stores want the same templates, just leave per-store assignments empty and the site-wide defaults apply.
:::

## Option 2 — Ask AI to tweak it {#option-2-ask-ai-to-tweak-it}

Wenn die Galerie fast, aber nicht ganz passend ist, kann ein KI-Assistent sie in wenigen Minuten anpassen — HTML-Kenntnisse sind dafür nicht erforderlich.

<Steps>
  <StepCard number="1" title="Vorlage kopieren">

Die gewünschte Ausgangsvorlage unter **WP Admin → POS → Vorlagen** öffnen, in den Editor klicken und den gesamten Text auf der linken Seite markieren (Ctrl/Cmd + A). Dann kopieren.

  </StepCard>
  <StepCard number="2" title="In ChatGPT oder Claude einfügen">

[ChatGPT](https://chat.openai.com) oder [Claude](https://claude.ai) öffnen. Die Vorlage einfügen und in einfachen Worten beschreiben, was geändert werden soll:

  </StepCard>
  <StepCard number="3" title="Gewünschte Änderungen beschreiben">

Der KI die gewünschten Änderungen genau beschreiben. Beispiele, die gut funktionieren:

- *„Den Filialnamen größer und zentriert darstellen.“*
- *„Eine Dankesnachricht in Kursivschrift am Ende hinzufügen.“*
- *„Den Kundennamen ausblenden. Stattdessen die Telefonnummer unter der Bestellnummer hinzufügen.“*
- *„Den Barcode durch einen QR-Code ersetzen, der auf die Rückgabeseite verlinkt.“*
- *„Einen Slogan 'Familienunternehmen seit 1987' unter dem Filialnamen hinzufügen.“*

Die KI gibt eine überarbeitete Vorlage zurück.

  </StepCard>
  <StepCard number="4" title="Wieder einfügen" isLast>

Kopieren Sie die Antwort der KI. Wechseln Sie zurück zum WCPOS-Vorlageneditor, markieren Sie alles (Strg/Cmd + A), fügen Sie die neue Version ein und klicken Sie auf **Aktualisieren**. Die Vorschau auf der rechten Seite wird aktualisiert, sodass Sie das Ergebnis sehen können. Wenn es nicht richtig aussieht, bitten Sie die KI um eine Korrektur — beschreiben Sie, was nicht stimmt.

  </StepCard>
</Steps>

:::tip Bewährte Vorgehensweise
Jeder Klick auf **Vorlage verwenden** in der Galerie erstellt eine neue bearbeitbare Kopie, sodass das Original unverändert bleibt. Beim Experimentieren kann dieselbe Galerievorlage mehrfach verwendet werden — benennen Sie die Kopien um *(Beleg v1, Beleg v2)* und wechseln Sie zwischen ihnen, bis Sie sich entschieden haben.
:::

:::info Was sind die Variablen?
Die Ausdrücke wie `{{store.name}}` und `{{order.number}}` sind **Platzhalter** für Ihre tatsächlichen Daten. Die KI versteht diese — Sie müssen es nicht. Eine vollständige Übersicht aller verfügbaren Platzhalter finden Sie in der [Belegdaten-Referenz](/templates/receipt-data).
:::

## Option 3 — Manuell bearbeiten {#option-3-edit-it-by-hand}

Mit HTML-Grundkenntnissen (oder in Zusammenarbeit mit einem Entwickler) kann die Vorlage direkt im integrierten Editor bearbeitet werden. Der Editor bietet Live-Vorschau, Syntaxhervorhebung, eine durchsuchbare Feldauswahl, Rückgängig/Wiederherstellen und Suchen-und-Ersetzen.

Wählen Sie die gewünschte Engine:

- **[HTML-Vorlagen](/templates/html-templates)** — Platzhalter im Mustache-Stil mit `{{variable}}`. Werden clientseitig gerendert und funktionieren offline. **Für die meisten Filialen empfohlen.**
- **[Thermovorlagen](/templates/thermal-templates)** — XML für ESC/POS-Thermodrucker. Dieselbe Vorlage erzeugt sowohl die Bildschirmvorschau als auch die Druckerausgabe.
- **[Belegdaten-Referenz](/templates/receipt-data)** — Alle verfügbaren Platzhalter, nach Abschnitt gruppiert.

:::note Alte PHP-Vorlagen
Falls zuvor der Beleg mit einer PHP-Datei im Theme überschrieben wurde (`yourtheme/woocommerce-pos/receipt.php`), funktioniert das weiterhin. Diese Option wird nun in der Galerie als **Alte PHP-Vorlage** angezeigt und steht neben den neuen logiklosen und Thermo-Engines. Die WP-Overnight-Integration nutzt ebenfalls den serverseitig gerenderten Pfad, da die Dokument-API des Drittanbieters HTML auf dem Server rendert. Neue Anpassungen sollten stattdessen die Galerie oder den In-App-Editor verwenden – sie funktionieren offline, bieten eine Live-Vorschau und benötigen keinen Server-Roundtrip.
:::

## Häufige Anpassungen {#common-customisations}

Schnelle Antworten auf die am häufigsten gestellten Fragen.

<Accordion>
  <AccordionItem question="How do I add my store logo?">

Logos stammen aus den Filialeinstellungen, nicht aus der Vorlage selbst. Unter **WP Admin → POS → Einstellungen → Filialen** kann die Filiale bearbeitet und dort ein Logo hochgeladen werden. Jede mitgelieferte Vorlage, die ein Logo anzeigt, verwendet es automatisch.

Um die *Position* des Logos in der Vorlage zu ändern, kann die Vorlage bearbeitet und der `{{#store.logo}}<img src="{{store.logo}}" ...>{{/store.logo}}` Block an die gewünschte Stelle verschoben werden.

  </AccordionItem>

  <AccordionItem question="How do I change the footer text (e.g. 'Thank you for your purchase!')?">

Zwei Möglichkeiten:

1. **Am einfachsten** — einmalig für alle Belege festlegen unter **WP Admin → POS → Einstellungen → Filialen → Filialdetails → Belegfußzeile / persönliche Notiz**. Mitgelieferte Vorlagen übernehmen dies automatisch; ist keine Fußzeile festgelegt, wird ein freundlicher Standardtext wie *"Vielen Dank für Ihren Einkauf!"* verwendet.
2. **In einer einzelnen Vorlage** — die Vorlage bearbeiten und den Fußzeilentext direkt ersetzen. Nach `{{store.personal_notes}}` oder der wörtlichen Dankeszeile suchen.

  </AccordionItem>

  <AccordionItem question="How do I add a tagline or slogan under the store name?">

Die Vorlage bearbeiten und eine Zeile unter `{{store.name}}` hinzufügen:

```html
<div style="font-style: italic; color: #6b7280;">Family-owned since 1987</div>
```

In einer Thermodruck-Vorlage:

```xml
<align mode="center"><text>Family-owned since 1987</text></align>
```

  </AccordionItem>

  <AccordionItem question="How do I hide prices (for a gift receipt)?">

Auf der **Geschenkbeleg**-Karte in der Galerie auf **Vorlage verwenden** klicken — alle Preise und Summen werden ausgeblendet, während Artikel, SKU, Attribute und die Geschenknachricht weiterhin angezeigt werden. Keine Bearbeitung erforderlich.

Um einen eigenen preisfreien Beleg zu erstellen, eine beliebige Vorlage kopieren und die Blöcke `{{...total...}}`, `{{...price...}}` und `{{#totals}}...{{/totals}}` entfernen.

  </AccordionItem>

  <AccordionItem question="How do I change the barcode to a QR code?">

Das `<barcode>` -Element in der Vorlage suchen und das `type` -Attribut ändern:

```html
<!-- Default: Code 128 barcode of the order number -->
<barcode type="code128">{{order.number}}</barcode>

<!-- Switch to a QR code -->
<barcode type="qrcode">{{order.number}}</barcode>

<!-- Or encode something else, like a returns URL -->
<barcode type="qrcode">https://example.com/returns?order={{order.number}}</barcode>
```

Dieselbe `<barcode>` -Syntax funktioniert sowohl in HTML- als auch in Thermodruck-Vorlagen. Weitere unterstützte Typen sind `ean13`, `ean8`, `upca`, `pdf417`, and [everything bwip-js supports](https://github.com/metafloor/bwip-js/wiki/Supported-Barcode-Types).

  </AccordionItem>

  <AccordionItem question="How do I send a different template to a specific printer?">

In the POS app, go to **Settings → Print Routing**. You'll see each of your active templates with a printer dropdown next to it. Pick the printer you want, or leave it as **Auto**.

- **Auto** matches templates to printers automatically — thermal templates go to thermal printers, HTML templates go to the system print dialog.
- A **specific printer** overrides Auto and always sends that template there.
- At print time, the cashier can override either of the above with the printer dropdown on the receipt screen.

Routing is stored per-device, so each iPad or computer can have its own setup.

  </AccordionItem>

  <AccordionItem question="My receipt still shows the old version after I edit it">

Click the WordPress **Update** button on the template edit screen. The editor doesn't auto-save — your changes only persist when you Update.

For **Legacy PHP templates**, the preview in the editor shows the *last saved* version, not what you're typing. Save first, then refresh the preview.

  </AccordionItem>

  <AccordionItem question="The preview is blank or shows 'No POS orders found'">

Dies tritt nur bei **Legacy-PHP-Vorlagen** auf, die eine echte Bestellung für die Vorschau benötigen. Führen Sie eine einzelne POS-Bestellung durch — auch einen $0-Testverkauf — und die Vorschau wird funktionieren.

Logiklose (HTML) und Thermovorlagen verfügen immer über Beispieldaten als Fallback, sodass die Vorschau auch bei einem brandneuen Shop problemlos funktioniert.

  </AccordionItem>

  <AccordionItem question="I made a mess — how do I start over?">

Drei Sicherheitsnetze:

1. Der Editor verfügt über **Rückgängig** (Strg/Cmd + Z) für Änderungen innerhalb der Sitzung.
2. Jedes Speichern erstellt eine WordPress-**Revision** — öffnen Sie **Revisionen** auf dem Bearbeitungsbildschirm, um frühere Versionen zu vergleichen und wiederherzustellen.
3. Wenn Sie mit einer Galerievor­lage begonnen haben, klicken Sie auf **Löschen** bei Ihrer Kopie unter *Ihre Vorlagen* und dann erneut auf **Vorlage verwenden** auf derselben Galeriekarte. Sie erhalten eine neue, unveränderte Kopie.

  </AccordionItem>
</Accordion>

## Wann Sie um Hilfe bitten sollten {#when-to-ask-for-help}

- Der Vorlageneditor wird nicht geladen oder Änderungen werden nicht gespeichert.
- Der Beleg wird auf einem Gerät korrekt gedruckt, auf einem anderen jedoch nicht.
- Sie benötigen ein steuerliches/rechtliches Layout für ein bestimmtes Land (Italien, Brasilien, Spanien usw.) — diese werden in der Regel über [WCPOS Pro](/getting-started/pro-license) oder eine länderspezifische Integration bereitgestellt.
- Sie versuchen etwas Individuelles umzusetzen und KI liefert nicht ganz das richtige Ergebnis.

Erstellen Sie ein [Support-Ticket](https://wcpos.com/support) und fügen Sie die Vorlage ein, an der Sie arbeiten — so haben wir alles, was wir zur Unterstützung benötigen.
