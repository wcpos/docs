## Enable POS Only Products {#enable-pos-only-products}

This setting adds visibility options to your products, allowing you to control whether they appear in your online store, POS, or both.

:::warning Performance Impact
Only enable this setting if you need to hide products from either your online store or POS. When enabled, it adds an extra database lookup for every product request.
:::

For detailed information on using this feature, including how to set visibility on individual products and bulk editing, see [POS Only Products](/products/pos-only-products).

## Enable Decimal Quantities {#enable-decimal-quantities}

Diese Einstellung ermöglicht es Kunden und Mitarbeitern, beim Hinzufügen von Produkten zum Warenkorb Dezimalmengen einzugeben.

**Beispiele für Dezimalmengen:**
- 0,25 (Vierteleinheit)
- 1,5 (eineinhalb Einheiten)
- 2,75 (zweidreiviertel Einheiten)

**Nützlich für:**
- Produkte, die nach Gewicht verkauft werden (z. B. 0,5 kg Kaffeebohnen)
- Anteilige Dienstleistungen (z. B. 1,5 Stunden Beratung)
- Großmengenprodukte, die in Teilmengen verkauft werden können

## Benutzernamen automatisch aus Kunden-E-Mail generieren {#automatically-generate-username-from-customer-email}

Wenn aktiviert, erstellt WCPOS beim Checkout automatisch Benutzernamen für neue Kunden basierend auf deren E-Mail-Adressen. Dies vereinfacht den Kundenerstellungsprozess und gewährleistet eine einheitliche Formatierung der Benutzernamen.

## Standard-POS-Kunde {#default-pos-customer}

Wählen Sie den Standardkunden aus, der neuen POS-Bestellungen zugewiesen wird. Gängige Optionen sind:

- **Gast**: Bestellungen werden keinem Benutzerkonto zugeordnet
- **Kassierer**: Bestellungen werden dem Konto des angemeldeten Kassierers zugeordnet
- **Laufkunde**: Bestellungen werden einem bestimmten Kundenkonto zugeordnet (dieses Konto muss zuvor erstellt werden)

## Kassiererkonto verwenden {#use-cashier-account}

Wenn aktiviert, wird das Benutzerkonto des Kassierers anstelle des ausgewählten Standard-POS-Kunden als Standardkunde für POS-Bestellungen verwendet.

## Barcode-Feld {#barcode-field}

Diese Einstellung legt fest, welches Produktfeld WCPOS beim Scannen von Barcodes oder bei der Barcode-Suchfunktion durchsucht.

:::warning Wichtig: Löschen und Aktualisieren erforderlich
Nach dem Ändern der Barcode-Feld-Einstellung **müssen** die Produktdaten im POS gelöscht und aktualisiert werden, damit die Änderung wirksam wird.

**So werden die Daten gelöscht und aktualisiert:**
1. Navigieren Sie zum Produkte-Bildschirm in Ihrem POS
2. **Long press** the sync icon (↻) at the bottom of the screen
3. Select **"Clear and Refresh"** from the context menu
4. Wait for the products to reload with the new barcode field data

This is necessary because the POS caches product data locally, and changing the barcode field setting doesn't automatically update the cached data.
:::

### Default Option {#default-option}

**`_global_unique_id`** - This is WooCommerce's Global Unique ID field, introduced in WooCommerce 8.2 (July 2024). It's designed to store GTIN, UPC, EAN, or ISBN codes and is the recommended option for new installations.

### Other Common Options {#other-common-options}

**`_sku`** - The product SKU (Stock Keeping Unit) field. This is a good fallback option if you're not using the Global Unique ID field.

### Third-Party Plugin Meta Keys {#third-party-plugin-meta-keys}

If you're using a barcode management plugin, you may need to use their specific meta key:

| Plugin Type | Common Meta Keys | Description |
|-------------|------------------|-------------|
| EAN-Plugins | `_ean`, `_ean13`, `_ean8` | Europäische Artikelnummer-Codes |
| UPC-Plugins | `_upc`, `_upc_code` | Universeller Produktcode |
| GTIN-Plugins | `_gtin`, `_gtin13`, `_gtin14` | Globale Handelsartikelnummer |
| Generischer Barcode | `_barcode`, `_product_barcode` | Generische Barcode-Felder |
| WooCommerce Product Barcodes | `_barcode_text` | Beliebtes Barcode-Plugin |
| ATUM Inventory | `_atum_barcode` | ATUM-Bestandsverwaltung |

### Den Meta-Key Ihres Plugins finden {#finding-your-plugins-meta-key}

So finden Sie den richtigen Meta-Key für Ihr Barcode-Plugin:

1. Bearbeiten Sie ein Produkt im WP Admin
2. Fügen Sie einen Barcode über Ihr Plugin hinzu
3. Prüfen Sie die Dokumentation des Plugins für den Meta-Schlüssel-Namen
4. Alternativ kann die Datenbanktabelle `wp_postmeta` geprüft werden, um zu sehen, welcher Meta-Schlüssel erstellt wurde

### Benutzerdefinierte Meta-Schlüssel {#custom-meta-keys}

Es kann auch jeder benutzerdefinierte Meta-Schlüssel verwendet werden, der Barcode-Daten speichert. Geben Sie einfach den Namen des Meta-Schlüssels (einschließlich des Unterstrich-Präfixes, falls zutreffend) in dieses Feld ein.

:::tip
Testen Sie das Barcode-Scannen nach Änderung dieser Einstellung, um sicherzustellen, dass es mit dem gewählten Feld korrekt funktioniert. Denken Sie daran, die Produktdaten im POS zu löschen und zu aktualisieren, damit die Änderung wirksam wird.
:::

## Steuer-IDs der Filiale {#store-tax-ids}

Hier werden die steuerlichen Identifikationsnummern des Unternehmens hinterlegt — USt-IdNr., ABN, GSTIN, EIN oder lokale Handelsregisternummern —, die WCPOS auf POS-Belegen und Rechnungen ausdruckt. WooCommerce verfügt über kein integriertes Feld für diese Angaben, und in vielen Regionen wird mehr als eine Nummer benötigt.

Die vollständige Anleitung finden Sie unter [Steuer-IDs der Filiale](/settings/wp-admin/store-tax-ids).

## Kunden-Steuer-IDs {#customer-tax-ids}

WCPOS speichert ein Steuer-ID-Feld bei den Kunden und übernimmt es beim Checkout in deren Bestellungen. Da verschiedene Plugins dieses Feld unter unterschiedlichen Namen speichern, erkennt WCPOS automatisch, welches Plugin installiert ist, und schreibt in das richtige Feld. Im Abschnitt **Steuer-IDs** kann der Feldname bei Bedarf auch manuell überschrieben werden.

Die vollständige Anleitung finden Sie unter [Kunden-Steuer-IDs](/settings/wp-admin/customer-tax-ids).
