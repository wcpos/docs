Die Seite Checkout-Einstellungen (`WP Admin > POS > Einstellungen > Checkout`) steuert, welche Zahlungsgateways im POS angezeigt werden, wie Bestellungen verarbeitet werden und welche Benachrichtigungs-E-Mails für POS-Verkäufe gesendet werden.

## E-Mail-Benachrichtigungen {#email-notifications}

Oben auf der Seite befinden sich drei Gruppen von E-Mail-Umschaltern — **Admin-E-Mails**, **Kunden-E-Mails** und **Kassierer-E-Mails** — die steuern, welche WooCommerce-Benachrichtigungs-E-Mails für POS-Bestellungen gesendet werden, unabhängig von den seitenweiten WooCommerce-E-Mail-Einstellungen.

Die vollständige Referenz, einschließlich Standardwerte der Umschalter pro E-Mail, Kassierer-Empfängermodell, häufige Szenarien und Entwickler-Hooks, finden Sie unter **[E-Mail-Benachrichtigungen](/settings/wp-admin/email-notifications)**.

## Zahlungsgateways {#payment-gateways}

Dieser Abschnitt listet alle auf Ihrer Website installierten WooCommerce-Zahlungsgateways auf. Jedes Gateway kann für den POS unabhängig aktiviert oder deaktiviert werden.

| Spalte | Beschreibung |
|--------|-------------|
| **Aktiviert** | Gateway für den POS-Checkout ein- oder ausschalten |
| **Zahlungsgateway** | Der Name des Zahlungsgateways |
| **Gateway-ID** | Die WooCommerce-Gateway-Kennung (nützlich für Entwickler) |

### Ein Gateway aktivieren {#enabling-a-gateway}

1. Das Gateway in der Liste suchen
2. Den Schalter **Aktiviert** einschalten
3. Das Gateway erscheint beim nächsten Laden der Seite im POS-Checkout

### Gateways neu anordnen {#reordering-gateways}

Ziehen Sie Gateways, um ihre Anzeigereihenfolge im POS-Checkout zu ändern. Das erste aktivierte Gateway wird standardmäßig ausgewählt, wenn eine Kassenkraft das Checkout-Modal öffnet.

### POS vs. Online-Shop {#pos-vs-online-store}

Gateway-Einstellungen hier sind **unabhängig** von Ihren WooCommerce-Zahlungseinstellungen. Ein Gateway kann sein:

- **Für POS aktiviert, online deaktiviert** — nützlich für Barzahlungen, die nicht für Webbestellungen gelten
- **Online aktiviert, für POS deaktiviert** — nützlich für Gateways wie PayPal, die für Vor-Ort-Verkäufe nicht praktikabel sind
- **Für beides aktiviert** — das Gateway funktioniert überall

:::note
Ein Zahlungsgateway muss zuerst in `WP Admin > WooCommerce > Einstellungen > Zahlungen` installiert und aktiv sein, bevor es in der Liste der POS-Checkout-Einstellungen erscheint. Wenn ein Gateway fehlt, prüfen Sie, ob das Plugin installiert und das Gateway in WooCommerce aktiviert ist.
:::

## Bestellstatus {#order-status}

Konfigurieren Sie den Standard-Bestellstatus für abgeschlossene POS-Bestellungen. Standardmäßig werden POS-Bestellungen auf **Abgeschlossen** gesetzt, da die Zahlung am Point of Sale eingezogen wird.
