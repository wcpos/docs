Die Seite Checkout-Einstellungen (`WP Admin > POS > Einstellungen > Checkout`) steuert, welche Zahlungsgateways im POS angezeigt werden, wie Bestellungen verarbeitet werden und welche Benachrichtigungs-E-Mails bei POS-Verkäufen ausgelöst werden.

## E-Mail-Benachrichtigungen {#email-notifications}

Am oberen Seitenrand befinden sich drei Gruppen von E-Mail-Schaltern — **Admin-E-Mails**, **Kunden-E-Mails** und **Kassierer-E-Mails** — die steuern, welche WooCommerce-Benachrichtigungs-E-Mails für POS-Bestellungen gesendet werden, unabhängig von den seitenweiten WooCommerce-E-Mail-Einstellungen.

Weitere Informationen finden Sie unter **[E-Mail-Benachrichtigungen](/settings/wp-admin/email-notifications)** mit der vollständigen Referenz, einschließlich Standard-Schaltereinstellungen pro E-Mail, dem Kassierer-Empfängermodell, häufigen Szenarien und Entwickler-Hooks.

## Zahlungsgateways {#payment-gateways}

Dieser Abschnitt listet alle auf der Website installierten WooCommerce-Zahlungsgateways auf. Jedes Gateway kann für den POS unabhängig aktiviert oder deaktiviert werden.

| Spalte | Beschreibung |
|--------|-------------|
| **Aktiviert** | Gateway für den POS-Checkout ein- oder ausschalten |
| **Gateway** | Der Name des Zahlungsgateways |
| **Gateway-ID** | Die WooCommerce-Gateway-Kennung (nützlich für Entwickler) |

### Ein Gateway aktivieren {#enabling-a-gateway}

1. Das Gateway in der Liste suchen
2. Den Schalter **Aktiviert** einschalten
3. Das Gateway wird beim nächsten Seitenaufruf im POS-Checkout angezeigt

### Gateways neu anordnen {#reordering-gateways}

Gateways per Drag-and-Drop verschieben, um die Anzeigereihenfolge im POS-Checkout zu ändern. Das erste aktivierte Gateway wird standardmäßig ausgewählt, wenn ein Kassierer das Checkout-Fenster öffnet.

### POS vs. Onlineshop {#pos-vs-online-store}

Die Gateway-Einstellungen hier sind **unabhängig** von den WooCommerce-Zahlungseinstellungen. Ein Gateway kann:

- **Im POS aktiviert, online deaktiviert** — nützlich für Barzahlungen, die bei Webbestellungen nicht relevant sind
- **Online aktiviert, im POS deaktiviert** — nützlich für Gateways wie PayPal, die für Vor-Ort-Verkäufe nicht praktikabel sind
- **Für beide aktiviert** — das Gateway funktioniert überall

:::note
Ein Zahlungsgateway muss zunächst unter `WP Admin > WooCommerce > Einstellungen > Zahlungen` installiert und aktiviert sein, bevor es in der Liste der POS-Checkout-Einstellungen erscheint. Falls ein Gateway fehlt, prüfen Sie, ob das Plugin installiert und das Gateway in WooCommerce aktiviert ist.
:::

## Bestellstatus {#order-status}

Konfigurieren Sie den Standard-Bestellstatus für abgeschlossene POS-Bestellungen. Standardmäßig werden POS-Bestellungen auf **Abgeschlossen** gesetzt, da die Zahlung am Point of Sale erfolgt.
