WCPOS speichert ein Steuer-ID-Feld für Ihre Kunden, etwa eine Umsatzsteuer-ID, ABN, CPF, GSTIN, NIF und so weiter, für B2B-Verkäufe, bei denen die Steuerkennung des Käufers auf der Rechnung erscheinen muss. Wenn eine Bestellung für einen Kunden aufgegeben wird, kopiert WCPOS dessen Steuer-ID auf die Bestellung, damit sie auf dem Beleg gedruckt werden kann.

Der Abschnitt **Steuer-IDs** unter **WP Admin > POS > Einstellungen > Allgemein** steuert, wie dieses Feld erkannt und gespeichert wird.

## Plugins und Feldnamen {#plugins-and-field-names}

Viele WooCommerce-Plugins fügen Kunden bereits ein Feld für eine Umsatzsteuer-ID oder Steuernummer hinzu, und sie verwenden dafür jeweils unterschiedliche Datenbank-Feldnamen (Meta-Schlüssel). Damit Ihre vorhandenen Kundendaten ohne Migration weiter funktionieren, erkennt WCPOS, was installiert ist, und schreibt automatisch in das richtige Feld.

WCPOS ermittelt den Feldnamen in dieser Reihenfolge:

1. **Ein erkanntes Plugin ist aktiv** — WCPOS schreibt in das Feld dieses Plugins zurück. Zu den erkannten Plugins gehören:
   - WooCommerce EU VAT Number
   - Aelia EU VAT Assistant (einschließlich seines Prüfstatus)
   - EU VAT for WooCommerce (WPFactory)
   - WooCommerce Germanized / Germanized Pro
   - Brazilian Market on WooCommerce / Extra Checkout Fields for Brazil
   - NIF/CIF Spain
2. **Kein erkanntes Plugin** — WCPOS durchsucht aktuelle Bestellungen nach dem bereits verwendeten Feld für USt-IdNr./CPF usw. und schreibt dorthin.
3. **Nichts gefunden** — WCPOS greift auf sinnvolle Standardwerte zurück (`_billing_vat_number` für IDs im USt-Stil, `_billing_cpf` / `_billing_cnpj` für Brasilien, `_billing_gstin` für Indien usw.).

### Feldnamen selbst auswählen {#choosing-the-field-name-yourself}

Um den erkannten Feldnamen zu überschreiben, gehen Sie zu **WP Admin > POS > Einstellungen > Allgemein**, öffnen Sie den Abschnitt **Steuer-IDs** und klappen Sie **Erweitert: anderen Feldnamen verwenden** auf.

Ein Banner zeigt dort, welches Plugin automatisch erkannt wurde (oder „kein Plugin erkannt“, wenn die integrierten Standardwerte verwendet werden). Das Eingabefeld zum Überschreiben verfügt über eine Vorauswahl, die Metaschlüssel vorschlägt, die andere Plugins häufig verwenden. Leeren Sie das Eingabefeld, um die Überschreibung zu entfernen und zum automatischen Feldnamen zurückzukehren.

:::note
In diesem Editor erscheinen nur die kundenbezogenen Steuer-ID-Typen. Unternehmensregister-Kennungen wie die deutsche USt-IdNr. oder die niederländische KVK-Nummer sind Felder auf Filialebene — siehe [Filial-Steuer-IDs](/settings/wp-admin/store-tax-ids).
:::

## Steuer-IDs werden in Bestellungen kopiert {#tax-ids-are-copied-onto-orders}

Wenn eine Bestellung aufgegeben wird, werden die Steuer-IDs des Kunden als **Momentaufnahme** in die Bestellung übernommen — die Bestellung behält die Werte, die zum Verkaufszeitpunkt hinterlegt waren.

- Das spätere Bearbeiten der Steuer-ID eines Kunden ändert dessen frühere Bestellungen **nicht**.
- Eine Steuer-ID, die ein Kassierer beim Checkout eingibt, überschreibt für diese Bestellung den Wert aus dem Kundendatensatz.
- Das Bearbeiten einer bestehenden Bestellung ändert ihre Steuer-IDs nur, wenn die Aktualisierung sie ausdrücklich enthält; WCPOS kopiert sie nie erneut aus dem Kundendatensatz.

So bleiben historische Rechnungen korrekt, auch wenn sich Kundendaten später ändern.

## Unterstützte Typen {#supported-types}

EU-USt-IdNr., GB VAT, SA VAT, australische ABN, brasilianische CPF / CNPJ, indische GSTIN, italienischer Codice Fiscale und Partita IVA, spanische NIF, argentinische CUIT, kanadische GST/HST und US EIN — plus eine generische **Steuer-ID** für alles andere.

## Kunden anhand der Steuer-ID finden {#finding-customers-by-tax-id}

Die Kundensuche im POS gleicht auch Steuer-ID-Werte ab, einschließlich der älteren Plugin-Feldnamen, sodass ein Kassierer einen Kunden anhand seiner USt-IdNr., CPF oder NIF finden kann.

## Auf Belegen {#on-receipts}

Steuer-IDs von Kunden erscheinen auf Belegen, die Kundendetails anzeigen, mit der jeweils korrekten lokalisierten Bezeichnung für jeden Typ. USt-IdNrn. erhalten ein Länderpräfix (zum Beispiel `DE123456789`).
