WCPOS führt ein Steuer-ID-Feld bei Kunden — eine USt-IdNr., ABN, CPF, GSTIN, NIF usw. — für B2B-Verkäufe, bei denen die Steuerkennung des Käufers auf der Rechnung erscheinen muss. Wenn eine Bestellung für einen Kunden aufgegeben wird, kopiert WCPOS die Steuer-ID auf die Bestellung, damit sie auf dem Beleg gedruckt werden kann.

Der Abschnitt **Steuer-IDs** unter **WP Admin > POS > Einstellungen > Allgemein** steuert, wie dieses Feld erkannt und gespeichert wird.

## Plugins und Feldnamen {#plugins-and-field-names}

Viele WooCommerce-Plugins fügen bereits ein USt- oder Steuernummernfeld bei Kunden hinzu und verwenden dabei jeweils unterschiedliche Datenbankfeldnamen (Meta-Keys). Damit die vorhandenen Kundendaten weiterhin funktionieren — ohne Migration — erkennt WCPOS, was installiert ist, und schreibt automatisch in das richtige Feld.

WCPOS ermittelt den Feldnamen in dieser Reihenfolge:

1. **Ein erkanntes Plugin ist aktiv** — WCPOS schreibt in das Feld dieses Plugins zurück. Zu den erkannten Plugins gehören:
   - WooCommerce EU VAT Number
   - Aelia EU VAT Assistant (einschließlich des Verifizierungsstatus)
   - EU VAT for WooCommerce (WPFactory)
   - WooCommerce Germanized / Germanized Pro
   - Brazilian Market on WooCommerce / Extra Checkout Fields for Brazil
   - NIF/CIF Spain
2. **No recognised plugin** — WCPOS scans recent orders for whatever VAT/CPF/etc. field is already in use and writes to that.
3. **Nothing found** — WCPOS falls back to sensible defaults (`_billing_vat_number` for VAT-style IDs, `_billing_cpf` / `_billing_cnpj` for Brazil, `_billing_gstin` for India, and so on).

### Choosing the field name yourself {#choosing-the-field-name-yourself}

To override the detected field name, go to **WP Admin > POS > Settings > General**, open the **Tax IDs** section, and expand **Advanced: use a different field name**.

A banner there shows which plugin was auto-detected (or "no plugin detected", with the built-in defaults in use). The override input has a typeahead that suggests the meta keys other plugins commonly use. Clear the input to drop your override and go back to the automatic field name.

:::note
Only the customer-facing tax-ID types appear in this editor. Company-registry identifiers like the German USt-IdNr. or Dutch KVK number are store-level fields — see [Store Tax IDs](/settings/wp-admin/store-tax-ids).
:::

## Steuer-IDs werden auf Bestellungen übernommen {#tax-ids-are-copied-onto-orders}

Bei einer Bestellung werden die Steuer-IDs des Kunden als **Momentaufnahme** auf die Bestellung übertragen — die Bestellung behält die Werte, die zum Zeitpunkt des Verkaufs hinterlegt waren.

- Eine spätere Änderung der Steuer-ID eines Kunden überschreibt **nicht** dessen frühere Bestellungen.
- Eine Steuer-ID, die an der Kasse eingegeben wird, überschreibt für diese Bestellung den im Kundendatensatz hinterlegten Wert.
- Beim Bearbeiten einer bestehenden Bestellung werden die Steuer-IDs nur geändert, wenn die Aktualisierung diese explizit enthält; WCPOS kopiert sie niemals erneut aus dem Kundendatensatz.

So bleiben historische Rechnungen auch nach einer Änderung der Kundendaten korrekt.

## Unterstützte Typen {#supported-types}

EU-USt-IdNr., GB VAT, SA VAT, australische ABN, brasilianische CPF / CNPJ, indische GSTIN, italienische Codice Fiscale und Partita IVA, spanische NIF, argentinische CUIT, kanadische GST/HST und US EIN — sowie eine generische **Steuer-ID** für alle übrigen Fälle.

## Kunden anhand der Steuer-ID finden {#finding-customers-by-tax-id}

Die Kundensuche im POS gleicht auch Steuer-ID-Werte ab, einschließlich der älteren Plugin-Feldnamen, sodass ein Kassierer einen Kunden anhand seiner USt-IdNr., CPF oder NIF nachschlagen kann.

## Auf Belegen {#on-receipts}

Steuer-IDs des Kunden erscheinen auf Belegen, die Kundendaten anzeigen, mit der jeweils korrekten lokalisierten Bezeichnung für jeden Typ. USt-IdNr. werden mit Länderpräfix angegeben (zum Beispiel `DE123456789`).
