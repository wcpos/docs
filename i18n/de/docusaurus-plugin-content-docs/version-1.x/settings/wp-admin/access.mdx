Um den Zugriff auf die WCPOS-Funktionalität zu steuern, können WordPress-Berechtigungen und Benutzerrollen genutzt werden. Durch das Zuweisen bestimmter Rollen zu Benutzern und das Konfigurieren ihrer Berechtigungen lassen sich die Zugriffsebenen für verschiedene POS-Benutzer anpassen. Das WCPOS-Plugin integriert sich in die vorhandenen WordPress-Rollen und führt eine spezielle Rolle für Kassierer ein.

Die folgende Tabelle zeigt die wichtigsten Benutzerrollen im POS und ihre jeweiligen Berechtigungen:

| Rolle          | Erstellt von      | Beschreibung                                                                                                                                                                                                                                                                                                                                                    |
| ------------- | --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Administrierende Person | WordPress       | Der Website-Inhaber, bei dem standardmäßig alle Berechtigungen aktiviert sind.                                                                                                                                                                                                                                                                                                    |
| Shop-Manager  | WooCommerce     | Diese Rolle ist für Benutzer vorgesehen, die Zugriff auf den POS benötigen und POS-Einstellungen verwalten können. Shop-Manager haben möglicherweise keine Berechtigung, Plugins zu installieren oder andere administrative Aufgaben auszuführen.                                                                                                                                                             |
| Kassierer       | WCPOS | Diese Rolle wird vom WCPOS-Plugin hinzugefügt und ist für Benutzer vorgesehen, die hauptsächlich den POS verwenden. Kassierer haben in der Regel eingeschränkte Berechtigungen und möglicherweise keinen Zugriff auf WP Admin, um POS-Einstellungen zu ändern, Plugins zu installieren oder Blog-Beiträge zu erstellen. In einigen Fällen kann es sogar sinnvoll sein, Kassierer daran zu hindern, Kunden im POS hinzuzufügen oder zu bearbeiten. |

Zum Zuweisen und Verwalten von Benutzerrollen und Berechtigungen navigieren Sie zu **`WP Admin > POS > Access`**.

Die folgende Tabelle bietet einen Überblick über die WordPress-Berechtigungen, die mit dem WCPOS-Plugin verbunden sind:

| Berechtigung                  | Beschreibung                                                            | Beispielverwendung                                                                                                                                    |
| --------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `access_woocommerce_pos`    | Erlaubt den Zugriff auf die WCPOS-Oberfläche.                        | Wenn diese Option aktiviert ist, erhält das Kassenpersonal die Berechtigung, auf den POS zuzugreifen und ihn zu verwenden, um POS-Transaktionen durchzuführen.          |
| `manage_woocommerce_pos`    | Ermöglicht die Verwaltung von WCPOS-Einstellungen und -Konfigurationen. | Wenn diese Option aktiviert ist, kann das Kassenpersonal Einstellungen im Zusammenhang mit dem POS ändern, z. B. Zahlungsmethoden oder Belegvorlagen.     |
| `create_users`              | Erlaubt dem Kassenpersonal, neue WordPress-Benutzerkonten zu erstellen.              | Wenn diese Option deaktiviert ist, kann das Kassenpersonal keine neuen Kundenkonten im POS erstellen.                                                  |
| `edit_others_shop_orders`   | Ermöglicht das Bearbeiten von Bestellungen, die von anderem Kassenpersonal erstellt wurden.              | Wenn diese Option deaktiviert ist, kann das Kassenpersonal keine Bestellungen ändern, die von anderem Kassenpersonal im POS erstellt wurden.                              |
| `edit_product`              | Erlaubt dem Kassenpersonal, Produktdetails zu bearbeiten.                            | Wenn diese Option deaktiviert ist, kann das Kassenpersonal die Details der im POS verfügbaren Produkte nicht ändern.                                     |
| `edit_published_products`   | Erlaubt dem Kassenpersonal, veröffentlichte Produkte zu bearbeiten.                         | Wenn diese Option deaktiviert ist, kann das Kassenpersonal keine Änderungen an Produkten vornehmen, die im POS bereits veröffentlicht sind.                           |
| `edit_shop_orders`          | Erlaubt dem Kassenpersonal, Bestellungen zu bearbeiten.                                | Wenn diese Option deaktiviert ist, kann das Kassenpersonal die Details von Bestellungen, z. B. Bestellstatus oder Kundendaten, im POS nicht ändern. |
| `edit_users`                | Erlaubt dem Kassenpersonal, bestehende Benutzerkonten zu bearbeiten.                     | Wenn diese Option deaktiviert ist, kann das Kassenpersonal die Details von Kundenkonten, z. B. die Rechnungsadresse, im POS nicht ändern.            |
| `list_users`                | Erlaubt dem Kassenpersonal, eine Liste von Benutzerkonten anzuzeigen.                    | Wenn diese Option deaktiviert ist, kann das Kassenpersonal im POS nicht nach Kunden suchen oder Kunden auswählen.                                                |
| `manage_product_terms`      | Erlaubt dem Kassenpersonal, Produktkategorien und Schlagwörter zu verwalten.              | Wenn diese Option deaktiviert ist, kann das Kassenpersonal im POS keine Produktkategorien oder Schlagwörter erstellen, bearbeiten oder löschen.                       |
| `publish_shop_orders`       | Erlaubt dem Kassenpersonal, Bestellungen zu veröffentlichen.                             | Wenn diese Option deaktiviert ist, kann das Kassenpersonal neue Bestellungen im POS nicht abschließen und verarbeiten.                                       |
| `read_private_products`     | Erlaubt dem Kassenpersonal, auf die Details privater Produkte zuzugreifen.          | Wenn diese Option deaktiviert ist, kann das Kassenpersonal die Details privater Produkte, die im POS verfügbar sind, nicht anzeigen.                               |
| `read_private_shop_coupons` | Ermöglicht der Kassenkraft den Zugriff auf die Details privater Shop-Gutscheine.      | Durch Deaktivieren dieser Option wird verhindert, dass die Kassenkraft die Details privater Shop-Gutscheine im POS anzeigen kann.                                     |
| `read_private_shop_orders`  | Ermöglicht der Kassenkraft den Zugriff auf die Details privater Shop-Bestellungen.       | Durch Deaktivieren dieser Option wird verhindert, dass die Kassenkraft die Details privater Shop-Bestellungen im POS anzeigen kann.                                      |
| `read`                      | Ermöglicht den Zugriff auf den WordPress-Adminbereich.                                  | Durch Deaktivieren wird verhindert, dass die Kassenkraft auf das WordPress-Dashboard und die Benutzerprofilseite zugreifen kann.                                                   |

Durch das Zuweisen der passenden Berechtigungen zu jeder Rolle lässt sich der Zugriff und die Berechtigungen für verschiedene Benutzer im POS fein abstimmen. So können Sie steuern, wer bestimmte Aktionen ausführen, sensible Informationen anzeigen und Änderungen an Ihrer Filiale vornehmen kann.

Ausführlichere Informationen zum Verwalten von Benutzerrollen und Berechtigungen in WordPress finden Sie im [WordPress Codex](https://codex.wordpress.org/Roles_and_Capabilities).

## Was jede Standardrolle auf einen Blick tun kann {#what-each-default-role-can-do-at-a-glance}

Mit den Standardberechtigungen verhalten sich die drei Rollen im POS wie folgt:

- **Administrator/-in** — vollständiger POS- und Verwaltungszugriff.
- **Shop-Manager** — vollständiger POS- und Verwaltungszugriff, mit Ausnahme von Aktionen, die die Berechtigung zum Installieren von Plugins erfordern (z. B. das Installieren oder Aktivieren von POS-Erweiterungen).
- **Kassenkraft** — der normale POS-Verkaufsablauf (Produkte durchsuchen, Bestellungen aufnehmen, Zahlungen verarbeiten, Belege drucken, Kunden verwalten), jedoch kein Zugriff auf Einstellungen, Protokolle oder die Filialverwaltung.

Eine vollständige Aufschlüsselung pro Endpunkt — einschließlich der Endpunkte des Pro-Plugins, des Verhaltens beim Ablauf von Tokens und Tipps zur Unterscheidung von Berechtigungsfehlern und Token-Fehlern — finden Sie in der [Referenz zum Rollen-Endpunktzugriff](/reference/role-endpoint-access).
