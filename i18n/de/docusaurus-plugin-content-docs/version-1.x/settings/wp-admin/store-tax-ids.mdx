Steuer-IDs der Filiale sind die geschäftlichen Steuerkennungen, die WCPOS auf Ihren POS-Belegen und -Rechnungen druckt — Ihre VAT-Nummer, ABN, GSTIN, EIN oder eine lokale Handelsregisternummer.

WooCommerce verfügt über kein integriertes Feld dafür, und in vielen Regionen wird mehr als eine benötigt. Ein deutsches Unternehmen druckt möglicherweise eine USt-IdNr., eine Steuernummer **und** eine HRB-Nummer; ein italienisches Unternehmen druckt sowohl eine Codice Fiscale als auch eine Partita IVA; viele Unternehmen zeigen eine VAT-ID zusammen mit einer lokalen Registernummer an. Das Feld „Steuer-IDs der Filiale“ ermöglicht es Ihnen, so viele Einträge wie nötig hinzuzufügen.

Sie finden es unter **WP Admin > POS > Settings > General** im Abschnitt **Store tax IDs**.

{/* Screenshot: wp-admin-store-tax-ids.png */}

## Eine Steuer-ID hinzufügen {#adding-a-tax-id}

1. Gehen Sie zu **WP Admin > POS > Settings > General** und scrollen Sie zu **Store tax IDs**.
2. Klicken Sie auf **Add tax ID**.
3. Wählen Sie einen **Typ**, geben Sie den **Wert** ein und legen Sie optional ein **Land** und eine benutzerdefinierte **Bezeichnung** fest.
4. Wiederholen Sie dies für jede Kennung, die Sie drucken müssen.
5. Verwenden Sie **Entfernen**, um eine Zeile zu löschen.

:::info Zeilen werden automatisch gespeichert
Für diesen Bereich gibt es keine Speichern-Schaltfläche. Jede Zeile wird gespeichert, sobald das Feld **Wert** den Fokus verliert — klicken oder tabben Sie aus dem Feld, um die Eingabe zu übernehmen.
:::

### Die erste Zeile verwendet Ihr Filialland {#the-first-row-uses-your-store-country}

Wenn Sie Ihre erste Steuer-ID hinzufügen, füllt WCPOS das Feld **Land** automatisch mit Ihrem WooCommerce-Basisland und wählt den passenden **Typ** aus — ein Unternehmen, das nur eine einzige lokale Steuer-ID benötigt, muss also nichts manuell auswählen. Bestehende Zeilen werden nie geändert.

Typ und Land bleiben beim Bearbeiten synchron: Wenn Sie den **Typ** ändern, wird das **Land** auf das Herkunftsland dieses Typs aktualisiert, und wenn Sie das **Land** ändern, wird der **Typ** auf die übliche Kennung dieses Landes umgestellt.

### Verfügbare Typen {#available-types}

Die Typ-Liste umfasst die gängigen kundenseitigen Kennungen sowie die Handelsregisternummern, die Unternehmen auf Belegen aufdrucken:

| Region | Typen |
|--------|-------|
| Europäische Union | EU VAT |
| Vereinigtes Königreich | GB VAT, GB Company number |
| Deutschland | USt-IdNr., Steuernummer, HRB |
| Frankreich | SIRET, SIREN |
| Übriges Europa | Niederlande KVK, Italien Codice Fiscale, Italien Partita IVA, Spanien NIF, Schweiz UID |
| Amerika | US EIN, Kanada GST/HST, Brasilien CPF, Brasilien CNPJ, Argentinien CUIT |
| Asien & Naher Osten | Indien GSTIN, Saudi-Arabien VAT |
| Australien | ABN |
| Allgemein | Sonstige |

Das Feld **Wert** zeigt einen Formathinweis als Platzhaltertext an (z. B. `12-3456789` für eine US EIN). Dies dient nur zur Orientierung – WCPOS lehnt Werte, die nicht dem erwarteten Format entsprechen, nicht ab, sodass auch ungewöhnliche, aber gültige Kennungen gespeichert werden.

## Was auf Belegen gedruckt wird {#what-prints-on-receipts}

Jede hinzugefügte Steuer-ID wird in der festgelegten Reihenfolge auf Belegen und Rechnungen gedruckt, jeweils in einer eigenen Zeile. WCPOS verwendet eine lokalisierte Bezeichnung für den Typ ("VAT ID", "ABN", "GSTIN" usw.) – oder die benutzerdefinierte **Bezeichnung**, falls eine festgelegt wurde. Haben zwei Einträge denselben Wert, wird das Duplikat automatisch entfernt.

Alle mitgelieferten Belegvorlagen geben bereits die vollständige Liste aus. Falls eine **benutzerdefinierte Vorlage** verwendet wird, die noch einen einzelnen `store.tax_id`-Wert referenziert, funktioniert sie weiterhin, druckt jedoch nur den ersten Eintrag. Stattdessen sollte die Vorlage so angepasst werden, dass sie die Liste durchläuft:

```html
{{#store.tax_ids}}{{label}}: {{value}}{{/store.tax_ids}}
```

## Steuer-IDs pro Filiale (Pro) {#per-store-tax-ids-pro}

:::info Pro-Funktion
Mit [WCPOS Pro](/getting-started/pro-license) können Sie Steuer-IDs **pro Filiale** festlegen, anstatt einmal für die gesamte Website — nützlich, wenn jeder Standort eine eigene VAT/EIN/GSTIN hat oder wenn eine Kette über Regionen mit unterschiedlichen Kennungen hinweg betrieben wird.
:::

Steuer-IDs pro Filiale werden unter **WP Admin > POS > Stores > Edit Store** im Abschnitt **Tax IDs** der Seitenleiste festgelegt. Solange die eigene Liste einer Filiale leer ist, greift sie auf die hier auf dem Reiter „General“ konfigurierten websiteweiten Steuer-IDs der Filiale zurück. Sobald Sie einer Filiale einen Eintrag hinzufügen, druckt diese Filiale ausschließlich ihre eigene Liste.
