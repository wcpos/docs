Der Fehler "Kann Eigenschaften von undefined (lesen von 'data')" tritt auf, wenn das Point of Sale (POS) System erwartet, Daten vom Server zu empfangen, stattdessen jedoch eine leere Antwort erhält.

## Mögliche Ursachen

- **Plugin- oder Theme-Konflikte:** Konflikte mit anderen WordPress-Plugins oder Themen können die Datenabfrage stören und leere Antworten verursachen.
- **Veraltete Softwareversionen:** Die Verwendung veralteter Versionen von WordPress, Plugins oder Themen kann zu Kompatibilitätsproblemen und unerwartetem Verhalten führen. Bitte stellen Sie sicher, dass Sie die [Mindestanforderungen](/getting-started/installation) erfüllt haben.
- **Serverfehler oder -konfiguration:** Es kann eine falsche Serverkonfiguration oder Probleme wie unzureichender Arbeitsspeicher, begrenzte Ausführungszeit oder serverseitige Fehler vorliegen, die die ordnungsgemäße Erstellung und Bereitstellung von Daten verhindern.

## Fehlerbehebungsmaßnahmen

### 1. Überprüfen Sie den Netzwerk-Tab im Browser

1. Öffnen Sie die POS-Anwendung in Ihrem Browser.
2. Öffnen Sie die Entwicklertools, indem Sie mit der rechten Maustaste auf die Seite klicken und "Untersuchen" auswählen oder `Ctrl + Shift + I` (oder `Cmd + Option + I` auf macOS) drücken.
3. Navigieren Sie zum Tab "Netzwerk" in den Entwicklertools.
4. Suchen Sie die fehlgeschlagene Anfrage, die normalerweise rot markiert oder mit einem entsprechenden HTTP-Statuscode (z. B. 500 Internal Server Error) versehen ist.
5. Wählen Sie die fehlgeschlagene Anfrage aus, um deren Details anzuzeigen.
6. Überprüfen Sie den Antwortbereich der Anfrage auf Fehlermeldungen oder Probleme mit den JSON-Daten.
7. Kopieren Sie relevante Fehlermeldungen oder machen Sie einen Screenshot.

### 2. Überprüfen Sie die WooCommerce-Fehlerprotokolle

<div class="image-container">
  <Image
    alt="Fehlerprotokolle"
    img="https://woocommerce.com/wp-content/uploads/2018/11/wc_log_fatal_errors.png"
  />
  <p class="image-caption"><a target="_blank" rel="noopener noreferrer" href="https://woocommerce.com/document/finding-php-error-logs/">Beispiel für WooCommerce-Fehlerprotokolle</a></p>
</div>

1. Melden Sie sich in Ihrem WordPress-Adminbereich an.
2. Navigieren Sie im WordPress-Seitenmenü zu "WooCommerce" > "Status" > "Protokolle".
3. Suchen Sie den Abschnitt "fatal_error" Protokolle.
4. Klicken Sie auf den Protokolleintrag, der dem Tag entspricht, an dem der Fehler aufgetreten ist (normalerweise das heutige Datum), um die Fehlerdetails anzuzeigen.
5. Identifizieren Sie die Ursache des Fehlers, wie sie in diesem Abschnitt protokolliert ist.

### 3. Überprüfen Sie die POS-Protokolle

Der POS [Protokollbildschirm](/support/logs) kann ebenfalls relevante Fehlermeldungen und Fehlercodes enthalten, die bei der Identifizierung des Problems helfen können.
