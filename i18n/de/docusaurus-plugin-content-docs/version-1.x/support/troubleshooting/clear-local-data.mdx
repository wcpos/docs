WCPOS speichert eine lokale Kopie Ihrer Produkte, Bestellungen und Kunden, damit das POS schnell arbeiten und offline funktionieren kann. Gelegentlich kann es vorkommen, dass diese lokalen Daten mit Ihrem WooCommerce-Shop nicht mehr synchron sind. Das Löschen aller lokalen Daten entfernt die lokale Datenbank und löst eine frische Synchronisierung von Ihrem Server aus.

:::warning
Dies ist die nukleare Option. Das Löschen lokaler Daten entfernt alles, was auf dem Gerät gespeichert ist, und zwingt zu einer vollständigen Resynchronisierung. Je nach Größe Ihres Shops kann dies einige Zeit in Anspruch nehmen. Verwenden Sie es nur, wenn andere Optionen - wie eine normale [Synchronisierung](/products/sync) - das Problem nicht gelöst haben.
:::

## Wann Sie es verwenden sollten {#when-to-use-it}

Sie müssen möglicherweise die lokalen Daten löschen, wenn:

- **Synchronisierungsprobleme** - Produkte, Bestellungen oder Kunden werden nicht ordnungsgemäß aktualisiert
- **Nach einem Plugin-Update** - um die Kompatibilität mit neuen Datenstrukturen sicherzustellen
- **Veraltete Daten** - Preise, Bestände oder andere Informationen erscheinen veraltet

## Was gelöscht wird {#what-gets-cleared}

| Gelöscht | Nicht gelöscht |
|----------|----------------|
| Lokale Produktdatenbank | Ihre Anmeldesitzung |
| Lokale Bestelldatenbank  | Daten auf Serverseite |
| Lokale Kundendatenbank   | WooCommerce-Einstellungen |

Sie werden **nicht** abgemeldet. Ihre WooCommerce-Shop-Daten sind nicht betroffen - nur die lokale Kopie auf Ihrem Gerät wird entfernt. Eine frische Synchronisierung beginnt automatisch nach dem Löschen.

## So löschen Sie lokale Daten {#how-to-clear-local-data}

### Web

1. Klicken Sie auf Ihr **Benutzermenü** in der oberen rechten Ecke, um das Menü zu öffnen
2. Klicken Sie auf **Alle lokalen Daten löschen**
3. Die lokale Datenbank wird gelöscht und eine frische Synchronisierung beginnt automatisch

### Desktop

1. Öffnen Sie das **WCPOS** Anwendungsmenü in der oberen Menüleiste
2. Fahren Sie mit der Maus über **Erweitert**
3. Klicken Sie auf **App-Daten löschen**
4. Die lokale Datenbank wird gelöscht und eine frische Synchronisierung beginnt automatisch

:::note
Die Desktop-Anwendung verwendet eine SQLite-Datenbank, während die Webanwendung IndexedDB verwendet, aber das Verhalten ist auf beiden Plattformen dasselbe.
:::
