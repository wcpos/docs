Der Protokollbildschirm zeichnet Systemaktivitäten innerhalb der POS auf. Es ist ein unverzichtbares Werkzeug zum Debuggen von Problemen und zum Verständnis der Vorgänge hinter den Kulissen.

{/* Screenshot: logs-screen.png */}

## Zugriff auf Protokolle

Öffnen Sie den Protokollbildschirm über die linke Navigationsleiste (das Herzschlag-Symbol).

## Schnittstellenübersicht

### Suche & Filter

- **Protokolle durchsuchen** - Spezifische Protokolleinträge nach Nachrichteninhalt finden
- **Protokollebenen-Filter** - Nach Protokollebene filtern (Info, Warnung, Fehler)
- **Anzeigeeinstellungen** - Sichtbare Spalten konfigurieren

### Protokolltabelle

Jeder Protokolleintrag zeigt:

| Spalte | Beschreibung |
|--------|-------------|
| **Zeit** | Zeitpunkt des Ereignisses |
| **Ebene** | Schweregrad: Info (grau), Warnung (orange), Fehler (rot) |
| **Nachricht** | Menschenlesbare Beschreibung des Ereignisses |
| **Kontext** | Erweiterbares JSON mit zusätzlichen Details |
| **Code** | Fehlercode (verlinkt zur Dokumentation) |

## Protokollebenen

### Info (Grau)

Normale Abläufe und erfolgreiche Ereignisse:
- "Alle IDs für Produkte abgerufen"
- "Neue Kunden synchronisiert"
- "Alle IDs für Steuern abgerufen"

### Warnung (Orange)

Nicht-kritische Probleme, die möglicherweise Aufmerksamkeit erfordern:
- "Sitzung konnte nicht aktualisiert werden"
- "Wiederholungsversuch für Synchronisierung"

### Fehler (Rot)

Probleme, die behoben werden müssen:
- "Entschuldigung, Sie können keine Ressourcen auflisten" (Berechtigungsproblem)
- "API-Anfrage fehlgeschlagen"
- "Datenbank-Synchronisierungsfehler"

## Kontextdaten

Klicken Sie auf den Erweiterungspfeil bei einem Protokolleintrag, um detaillierte JSON-Kontexte zu sehen:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Diese Daten umfassen:
- **Elementanzahl** - Wie viele Datensätze betroffen waren
- **Ausführungszeit** - Wie lange die Operation dauerte
- **Serverlast** - Server-Leistungskennzahlen
- **Anfrage-/Antwortdetails** - Zum Debuggen von API-Problemen

## Fehlercodes

Viele Protokolleinträge enthalten einen **Fehlercode** (z.B. `API02004`, `DB01001`). Diese Codes verlinken direkt zur [Fehlercode-Dokumentation](/error-codes), die Folgendes bietet:

- Detaillierte Erklärung des Fehlers
- Häufige Ursachen
- Fehlerbehebungsschritte

## Häufige Protokolleinträge

### Synchronisierungsoperationen

```
[info] Alle IDs für Produkte abgerufen    {...} 3 Elemente
[info] Neue Kunden synchronisiert         {...} 1 Element
[info] Alle IDs für Steuern abgerufen     {...} 3 Elemente
```

Diese zeigen normale Synchronisierungsaktivitäten. Der Kontext zeigt, wie viele Elemente synchronisiert wurden.

### Berechtigungsfehler

```
[error] Entschuldigung, Sie können keine Ressourcen auflisten    {...} API02004
```

Dies bedeutet normalerweise, dass der Benutzer keine Berechtigung hat, auf eine Ressource zuzugreifen. Überprüfen Sie Benutzerrollen in `WP Admin > POS > Einstellungen > Zugriff`.

### Sitzungsprobleme

```
[warn] Sitzung konnte nicht aktualisiert werden    {...} API02007
```

Die POS konnte die Authentifizierungssitzung nicht aktualisieren. Dies erfordert möglicherweise das Aus- und Einloggen.

## Verwendung von Protokollen für Support

Bei Supportanfragen:

1. **Problem reproduzieren** - Führen Sie die Aktion aus, die das Problem verursacht
2. **Protokolle überprüfen** - Nach Fehlern (rot) oder Warnungen (orange) suchen
3. **Kontext erweitern** - Vollständige JSON-Details abrufen
4. **Fehlercode notieren** - Diesen in Ihrer Supportanfrage angeben
5. **Relevante Einträge kopieren** - Protokollmeldungen mit Kontext teilen

## Leistungsüberwachung

Protokolle enthalten Ausführungszeit- und Serverlast-Metriken. Wenn Sie Folgendes bemerken:

- **Hohe Ausführungszeiten** - Server möglicherweise überlastet oder langsam
- **Hohe Serverlast-Werte** - Server-Optimierung in Betracht ziehen
- **Häufige Timeouts** - Netzwerkverbindung überprüfen
