Der Logs-Bildschirm zeichnet die Systemaktivität innerhalb des POS auf. Es ist ein essentielles Werkzeug zur Fehlersuche und zum Verständnis dessen, was im Hintergrund passiert.

{/* Screenshot: logs-screen.png */}

## Protokolle Zugreifen

Öffnen Sie den Logs-Bildschirm aus dem linken Navigationsbereich (das Herzsymbol).

## Übersicht der Benutzeroberfläche

### Suchen & Filtern

- **Protokolle durchsuchen** - Bestimmte Protokolleinträge nach Nachrichteninhalt finden
- **Protokollstufenfilter** - Nach Protokollstufe filtern (info, warn, error)
- **Anzeigeeinstellungen** - Sichtbare Spalten konfigurieren

### Protokolltabelle

Jeder Protokolleintrag zeigt:

| Spalte | Beschreibung |
|--------|-------------|
| **Zeit** | Wann das Ereignis aufgetreten ist |
| **Stufe** | Schweregrad: info (grau), warn (orange), error (rot) |
| **Nachricht** | Verständliche Beschreibung des Ereignisses |
| **Kontext** | Ausklappbares JSON mit zusätzlichen Details |
| **Code** | Fehlercode (Links zur Dokumentation) |

## Protokollstufen

### Info (Grau)

Normale Vorgänge und erfolgreiche Ereignisse:
- "Alle IDs für Produkte abgerufen"
- "Neue Kunden synchronisiert"
- "Alle IDs für Steuern abgerufen"

### Warnung (Orange)

Nicht kritische Probleme, die möglicherweise Aufmerksamkeit erfordern:
- "Konnte Sitzung nicht aktualisieren"
- "Wiederholungsversuch für Synchronisierung"

### Fehler (Rot)

Probleme, die behoben werden müssen:
- "Entschuldigung, Sie können Ressourcen nicht auflisten" (Berechtigungsproblem)
- "API-Anforderung fehlgeschlagen"
- "Datenbanksynchronisierungsfehler"

## Kontextdaten

Klicken Sie auf den Ausklapppfeil bei einem Protokolleintrag, um den detaillierten JSON-Kontext zu sehen:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Diese Daten enthalten:
- **Elementzahlen** - Wie viele Datensätze betroffen waren
- **Ausführungszeit** - Wie lange die Operation dauerte
- **Serverlast** - Serverleistungsmesswerte
- **Anfrage-/Antwortdetails** - Zur Fehlersuche bei API-Problemen

## Fehlercodes

Viele Protokolleinträge enthalten einen **Fehlercode** (z.B. `API02004`, `DB01001`). Diese Codes verlinken direkt zur [Fehlercodes](/error-codes) Dokumentation, die bietet:

- Detaillierte Erläuterung des Fehlers
- Häufige Ursachen
- Schritte zur Fehlersuche

## Häufige Protokolleinträge

### Synchronisierungsoperationen

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

Diese zeigen normale Synchronisierungsaktivitäten an. Der Kontext zeigt, wie viele Elemente synchronisiert wurden.

### Berechtigungsfehler

```
[error] Sorry, you cannot list resources    {...} API02004
```

Das bedeutet normalerweise, dass der Benutzer keine Berechtigung hat, auf eine Ressource zuzugreifen. Überprüfen Sie die Benutzerrollen in `WP Admin > POS > Einstellungen > Zugriff`.

### Sitzungsprobleme

```
[warn] Unable to refresh session    {...} API02007
```

Der POS konnte die Authentifizierungssitzung nicht aktualisieren. Dies könnte erfordern, dass Sie sich aus- und wieder einloggen.

## Verwendung von Protokollen für den Support

Bei der Anfrage von Support:

1. **Reproduzieren Sie das Problem** - Führen Sie die Aktion aus, die das Problem verursacht
2. **Überprüfen Sie die Protokolle** - Suchen Sie nach Fehlern (rot) oder Warnungen (orange)
3. **Erweitern Sie den Kontext** - Erhalten Sie die vollständigen JSON-Details
4. **Notieren Sie den Fehlercode** - Fügen Sie dies in Ihre Supportanfrage ein
5. **Kopieren Sie relevante Einträge** - Teilen Sie die Protokollnachrichten mit Kontext

## Leistungsüberwachung

Protokolle enthalten Ausführungszeiten und Serverlastkennzahlen. Wenn Sie Folgendes bemerken:

- **Hohe Ausführungszeiten** - Server könnte überlastet oder langsam sein
- **Hohe Serverlastwerte** - Erwägen Sie die Serveroptimierung
- **Häufige Zeitüberschreitungen** - Überprüfen Sie die Netzwerkverbindung
