WCPOS 是一个单页 JavaScript Web 应用程序，通过 [WooCommerce REST API](http://woothemes.github.io/woocommerce-rest-api-docs/) 与服务器通信。性能问题可以分为两类：服务器端和客户端。

### 服务器端性能

服务器端性能指下载产品页面（10 个产品）或处理订单等操作。影响这些操作速度的因素包括：

- 互联网连接速度
- 服务器性能，如：处理能力、RAM、服务器负载等
- 活跃插件数量
- 支付网关处理速度

[演示站点](http://demo.wcpos.com/pos) 代表了"最佳情况"。演示站点使用[非常好的虚拟主机](http://wcpos.com/wpe)并且只激活了很少的插件。

| 进程                     | [demo.wcpos.com/pos](http://demo.wcpos.com/pos) 的平均速度<sup>\*</sup> |
| ------------------------ | ------------------------------------------------------------------------ |
| 获取 10 个产品           | 1 - 2 秒                                                                |
| 处理现金销售             | 1 - 2 秒                                                                |
| 处理 Stripe 支付         | 2 - 3 秒                                                                |

<small>
  <sup>*</sup> 宽带连接的平均时间。时间可能因您的网速而异。
</small>

如果您的 POS 系统比上述时间明显更慢，您可以进行以下测试：

1.  将主题切换到 WordPress 默认的 Twenty Fifteen 主题
2.  禁用除 WooCommerce 和 WCPOS 之外的所有插件
3.  [清除本地存储数据](http://faq.wcpos.com/en/clear-local-data.html)
4.  使用 POS 查看是否有性能提升
5.  **如果性能提升：** 逐一重新激活您的主题和插件，找出影响性能的插件
6.  **如果没有性能提升：** 您可能需要更换更好的虚拟主机或投资更快的互联网连接

### 客户端性能

WCPOS 使用 JavaScript、HTML 和 CSS 显示从 WooCommerce REST API 检索的数据，如 [产品](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) 和 [订单](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order)。
为了提高客户端性能，POS 使用 [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API) 在浏览器中存储数据。
例如，当第一次获取产品时，将向服务器发送请求，下载产品数据后，它将在本地存储，以便后续搜索可以立即完成。

![本地存储产品示例。](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "本地存储产品示例")

即使关闭浏览器或重启计算机，IndexedDB 数据也会保持持久。
如果您发现产品数据出现任何不同步的情况，可以[清除本地存储](http://faq.wcpos.com/en/clear-local-data.html)并从服务器获取新的数据集。
