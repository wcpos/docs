:::caution
本文面向Web开发人员，调试不是普通用户需要使用的功能。
:::

WCPOS主要是一个JavaScript Web应用程序 - 因此 - 大多数调试都是通过[浏览器控制台](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors)完成的。
如果启用调试，WCPOS将在控制台中记录各种消息，这可能有助于诊断问题。

![Mac上Chrome中打开的JavaScript控制台](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Mac上Chrome中打开的JavaScript控制台")

### 使用 wp-config.php 开启调试

通过在 wp-config.php 文件中设置 [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) 标志，您可以启用WCPOS调试。
这将加载未压缩的脚本文件，并在浏览器控制台中启用调试消息。

### 使用浏览器控制台开启调试

如果您无法访问 wp-config.php 文件，可以通过在本地存储中添加 `debug` 标志来启用调试消息。
可以在控制台中输入 `localStorage.setItem('debug', '*')` 来实现。
现在刷新页面，调试就会开启。控制台日志可能会消耗浏览器资源，所以您可能希望在完成调试后关闭调试。
要关闭调试，只需在控制台中输入 `localStorage.removeItem('debug')` 并刷新页面。
