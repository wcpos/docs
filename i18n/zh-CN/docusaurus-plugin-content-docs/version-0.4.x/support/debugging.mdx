:::caution
本文面向网页开发者，调试不是普通用户应该使用的功能。
:::

WCPOS 基本上是一个 JavaScript 网络应用 - 因此 - 大多数调试通过 [浏览器控制台](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors) 完成。
如果启用了调试，WCPOS 将会将各种日志记录到控制台，这可能有助于诊断问题。

![Mac 上 Chrome 的打开的 JavaScript 控制台](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Mac 上 Chrome 的打开的 JavaScript 控制台")

### 使用 wp-config.php 启用调试

通过在 wp-config.php 文件中设置 [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) 标志，您可以启用 WCPOS 调试。
这将加载未缩小的脚本文件，并在浏览器控制台中启用调试消息。

### 使用浏览器控制台启用调试

如果您无法访问 wp-config.php 文件，可以通过将 `debug` 标志添加到本地存储来启用调试消息。
可以通过在控制台中输入 `localStorage.setItem('debug', '*')` 来完成此操作。
现在刷新页面，调试已启用。控制台记录可能消耗浏览器资源，因此您可能希望在使用完后关闭调试。
要关闭调试，只需在控制台中输入 `localStorage.removeItem('debug')` 并刷新页面。
