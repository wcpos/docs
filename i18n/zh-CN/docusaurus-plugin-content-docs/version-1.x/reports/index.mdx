报告页面提供销售报告和每日对账工具。它主要用于**日终报告**，帮助收银员和经理结束一天的工作。

{/* Screenshot: reports-overview.png */}

:::tip 综合分析
有关任何日期范围的详细分析，请使用**WP Admin > Analytics**。POS报告页面优化用于快速的日终摘要和每日对账。
:::

## 界面概述

### 过滤器

在页面顶部：

- **状态过滤器** - 根据订单状态过滤（已完成、处理中等）
- **客户过滤器** - 根据客户过滤
- **收银员过滤器** - 根据处理订单的人过滤
- **商店过滤器** - 根据商店过滤（多商店配置）
- **日期范围** - 选择报告期间（今天、本周、自定义）

### 三面板布局

报告页面分为三个区域：

#### 1. 销售图表（顶部）

显示销售随时间变化的条形图：

- 每小时细分的每日报告
- 销售模式的可视化表示
- 帮助识别销售高峰时间

#### 2. 订单表（左下）

报告中包含的订单列表：

- **复选框** - 选择/取消选择包含的订单
- **状态** - 订单状态图标
- **订单号** - 唯一的订单ID
- **创建日期** - 下单时间
- **支付日期** - 收到付款的时间
- **总计** - 订单金额

点击显示设置图标自定义列。

#### 3. 报告摘要（右下）

计算的报告总计：

- **报告类型** 下拉菜单（默认/离线）
- **商店信息**（名称、ID）
- **报告期间**（开始/结束时间戳）
- **收银员**（如果有过滤）

**销售摘要：**
- 总订单数
- 总净销售额
- 收取的总税额
- **总销售额**
- 总折扣

**支付方式：**
- 按支付类型细分（现金、卡等）

**打印** 按钮生成物理报告。

## 报告类型

### 默认（离线）

默认报告是从本地存储的数据生成的。这：

- 无需服务器连接
- 提供快速的报告生成
- 使用已同步到设备的数据

这非常适合快速的日终摘要，因为所有相关的订单已经同步。

## 日终工作流程

### 典型的每日对账

1. **设置日期范围** - 选择“今天”或特定日期
2. **按收银员过滤**（可选） - 针对个人收银员的报告
3. **审查订单** - 检查订单表的准确性
4. **验证总计** - 与实际现金计数和卡终端进行比较
5. **打印报告** - 生成一个物理记录

### 现金对账

1. 过滤仅显示现金支付
2. 注意报告中的现金总额
3. 计算抽屉中的实际现金
4. 比较总计（应该匹配）
5. 记录任何差异

### 卡对账

1. 过滤仅显示卡支付
2. 注意报告中的卡总额
3. 与卡终端的批次总额进行比较
4. 在结算批次之前验证金额匹配

## 打印报告

点击**打印**按钮生成可打印的报告。这包括：

- 商店信息
- 报告期间
- 销售摘要
- 支付方式细分
- 订单详情（如果已选择）
