:::info 专业功能
报告屏幕需要 [WCPOS Pro](/getting-started/pro-license)。
:::

报告屏幕提供销售报告和每日对账工具。它主要用于 **每日结束报告**，帮助收银员和经理结算一天的交易。

{/* 截图: reports-overview.png */}

:::tip 综合数据分析
要获取任何日期范围内的详细数据分析，请使用 **WP 管理 > 数据分析**。POS 报告屏幕优化了快速的每日总结和每日对账。
:::

## 界面概述

### 过滤器

在屏幕顶部：

- **状态过滤器** - 按订单状态过滤（已完成、处理中等）
- **客户过滤器** - 按客户过滤
- **收银员过滤器** - 按处理订单的人员过滤
- **商店过滤器** - 按商店过滤（多商店设置）
- **日期范围** - 选择报告周期（今天、本周、自定义）

### 三面板布局

报告屏幕分为三个区域：

#### 1. 销售图表（顶部）

显示一段时间内销售情况的柱状图：

- 每日报告的小时细分
- 销售模式的可视化表示
- 帮助识别销售高峰时段

#### 2. 订单表（左下）

报告中包含的订单列表：

- **复选框** - 选择/取消选择包含的订单
- **状态** - 订单状态图标
- **订单编号** - 唯一订单 ID
- **创建日期** - 订单下单时间
- **付款日期** - 收到付款的时间
- **总额** - 订单金额

单击显示设置图标以自定义列。

#### 3. 报告摘要（右下）

计算的报告总额：

- **报告类型** 下拉菜单（默认/离线）
- **商店信息**（名称，ID）
- **报告周期**（开始/结束时间戳）
- **收银员**（如果过滤过）

**销售摘要：**
- 总订单数
- 总净销售额
- 收取的总税额
- **总销售额**
- 总折扣

**支付方式：**
- 按支付类型细分（现金、卡等）

**打印** 按钮以生成物理报告。

## 报告类型

### 默认（离线）

默认报告是从本地存储的数据生成的。这：

- 在没有服务器连接的情况下工作
- 提供快速的报告生成
- 使用已经同步到设备的数据

这非常适合快速的日终总结，因为所有相关订单已经同步。

## 日终工作流程

### 一般每日对账

1. **设置日期范围** - 选择“今天”或特定日期
2. **按收银员过滤**（可选） - 针对单独收银员的报告
3. **查看订单** - 检查订单表的准确性
4. **验证总额** - 与物理现金数和信用卡终端进行比较
5. **打印报告** - 生成物理记录

### 现金对账

1. 过滤以仅显示现金支付
2. 注意报告中的现金总额
3. 计算抽屉中的物理现金
4. 比较总额（应匹配）
5. 记录任何差异

### 卡片对账

1. 过滤以仅显示卡支付
2. 注意报告中的卡总额
3. 与卡片终端的批次总额进行比较
4. 在结算批次之前确认金额匹配

## 打印报告

单击 **打印** 按钮以生成可打印的报告。这包括：

- 商店信息
- 报告周期
- 销售摘要
- 支付方式细分
- 订单详细信息（如果选择）
