当您准备完成销售时，点击**结账**按钮以打开结账模态框。在这里，您可以处理付款并完成订单。

{/* 截图: checkout-modal.png */}

## 结账模态框概述

结账模态框显示：

- **订单编号** - WooCommerce 订单 ID
- **应付款项** - 应付总额
- **结账设置**按钮 - 排查显示问题
- **收银员** - 正在处理订单的人
- **客户** - 此订单的客户（可点击链接）
- **优惠券代码**字段 - 申请折扣代码
- **订单摘要** - 产品、数量和总计
- **支付方式** - 可用的支付选项
- **取消 / 处理付款**按钮

## 支付方式

### 免费版本可用

WCPOS 的免费版本包含两个支付网关：

- **现金** - 包括金额输入和找零计算器
- **卡** - 用于外部卡终端

### 附加网关（专业版）

:::info 专业功能
附加支付网关需要 [WCPOS Pro](/getting-started/pro-license)。
:::

使用专业版，您可以启用：

- **Stripe 终端** - 与 Stripe 卡阅读器的直接集成
- **SumUp 终端** - 与 SumUp 卡阅读器的集成
- **自定义网关** - 创建您自己的支付集成

有关配置网关的详细信息，请参见 [支付](/payment)。

### 选择支付方式

点击支付方式以选择它。表单会更新以显示相关字段：

**现金：**
- **现金金额** - 输入客户给您的金额
- **找零** - 自动计算的找零金额

{/* 截图: checkout-cash-payment.png */}

**卡：**
- 在您的外部卡终端上处理付款
- 点击处理付款以完成

## 优惠券代码

要申请折扣：

1. 在字段中输入优惠券代码
2. 点击**申请优惠券**
3. 折扣出现在订单摘要中

优惠券必须在 WooCommerce 中配置（`WP Admin > 营销 > 优惠券`）。

## 处理付款

1. 选择支付方式
2. 对于现金，输入给定的金额
3. 点击**处理付款**
4. 订单完成并显示 [收据](/pos/checkout/receipts)

## 结账设置（故障排除）

结账模态框使用 iframe/webview 来显示 WooCommerce 订单支付页面。这利用了 WooCommerce 的现有支付基础设施，意味着任何与 WooCommerce 兼容的支付网关都应该在 POS 中工作。

然而，主题和插件脚本有时可能会干扰。点击**结账设置**以进行故障排除：

{/* 截图: checkout-settings.png */}

### 禁用所有样式和脚本

- **禁用 wp_head** - 从 WordPress 头部移除所有脚本/样式
- **禁用 wp_footer** - 从 WordPress 页脚移除所有脚本/样式

### 禁用选定样式

选择性禁用可能导致显示问题的 CSS：

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- 等等。

### 禁用选定脚本

选择性禁用可能干扰支付网关的 JavaScript：

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- 等等。

:::tip
如果支付网关显示不正确：
1. 首先尝试禁用主题样式
2. 然后尝试禁用不需要的 WooCommerce 脚本
3. 小心不要禁用您的支付网关所需的脚本
:::

## 取消

点击**取消**以在不完成订单的情况下关闭结账模态框。订单将作为开放购物车保留。
