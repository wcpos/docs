WCPOS 为收银员提供多种即时为订单应用折扣的方式：快速百分比按钮、直接编辑商品行价格，以及订单级折扣费用。有关带使用规则的预配置促销，请参阅[优惠券](/coupons)（Pro）。

:::info 在 v1.9.0 中更改
在 v1.9.0 中，POS 价格调整在 WooCommerce 折扣总计中的显示方式已更改。当前收据模板可分别显示原价和 POS 节省金额，而原生的**折扣**总计仍保留给优惠券使用。详情请参阅下方的[在 v1.9.0 中更改了什么](#what-changed-in-v190)。
:::

## 快速折扣 {#quick-discounts}

快速折扣按钮可一键为整个订单应用百分比折扣。

要进行配置，请打开购物车的[显示设置](/pos/cart#display-settings)，然后在**快速折扣**字段中输入以逗号分隔的百分比列表。例如，`5,10,15,20`会创建四个快捷按钮。

轻触快速折扣按钮后，该百分比将应用于购物车中的所有商品行。

## 商品行折扣 {#line-item-discounts}

可直接在购物车中更改任意单个商品行的价格：

1. 点击商品行中的**价格**字段
2. 输入新价格
3. 按 **Enter** 确认

此功能适用于价格匹配、员工折扣或一次性调整。商品行总额会根据数量 × 新价格自动更新。

有关编辑购物车商品的更多信息，请参阅[商品行](/pos/cart/line-items)。

:::tip 拆分商品
如果顾客希望同一商品的不同部分使用不同折扣（例如，3 件按原价销售，2 件享受折扣），请在购物车[显示设置](/pos/cart#display-settings)中启用**拆分**选项，将一个商品行拆分为多个单独的行。
:::

## 订单级折扣 {#order-level-discounts}

如需对整个订单应用固定折扣（而非单个商品），请添加**负费用**：

1. 点击购物车商品下方的**添加费用**
2. 输入名称（例如，“员工折扣”）
3. 以负数输入折扣金额（例如，`-5.00`）

该费用会显示为单独的订单项目，并减少订单总额。如有需要，可通过三点菜单编辑该费用的税务状态。

## POS 折扣与 WooCommerce 优惠券 {#pos-discounts-vs-woocommerce-coupons}

本页的折扣是收银员在收银台临时应用的调整；WooCommerce **优惠券**则是预先配置的促销活动，带有规则和跟踪功能。以下是两者的简要对比：

| | POS 折扣 | WooCommerce 优惠券（Pro） |
|---|---|---|
| **应用方式** | 快速折扣、编辑商品行价格或负费用 | 在购物车中输入优惠券代码 |
| **配置位置** | 由收银员即时设置 | 在 **WP Admin → 营销 → 优惠券** 中预先配置 |
| **跟踪** | 记录为商品行价格（参见 [v1.9.0 变更](#what-changed-in-v190)） | 在 WooCommerce 报告中作为优惠券折扣跟踪 |
| **限制** | 无——收银员可设置任意价格 | 使用次数限制、商品/类别限制、最低消费额、有效期、电子邮件规则 |
| **最适合** | 临时调整、价格匹配 | 结构化促销活动、可跟踪的折扣 |

:::tip 应该使用哪一种？
对于一次性的价格调整，本页的折扣更简单。如果需要在 WooCommerce 报告中跟踪折扣使用情况，或强制执行使用次数限制等规则，请使用[优惠券](/coupons)。
:::

## POS 价格变更如何与优惠券交互 {#how-pos-price-changes-interact-with-coupons}

当收银员为订单项设置自定义价格（例如，将 $18 降至 $16）后再应用优惠券时，优惠券会基于 **POS 折后价格**（$16）计算，而非原价（$18）。这样可避免收银员折扣和优惠券同时按原价计算，导致顾客获得过多折扣。

- WooCommerce 会将 POS 折扣商品视为“促销商品”。如果优惠券启用了 **排除促销商品**，则会跳过 POS 折扣商品，与跳过常规促销商品的方式相同。开发人员可通过 `woocommerce_pos_item_is_on_sale` 过滤器覆盖此行为。
- 移除优惠券后，订单项将保留其 POS 折后价格。

:::note 开发人员参考
有关 POS 价格覆盖的存储方式和可用过滤器的技术详情，请参阅 [POS 折扣参考](/reference/pos-discounts)。
:::

## v1.9.0 中的变更 {#what-changed-in-v190}

如果从 v1.8 升级后发现收据和报告中的 **折扣** 总额现在显示为 **0**，本节将说明原因及可选方案。

### 此变更 {#the-change}

在 v1.9.0 之前，当收银员降低商品行价格时（例如，$18 → $16），POS 会以 `subtotal = $18` 和 `total = $16` 记录订单。WooCommerce 随后计算 `discount_total = $2`，并将其作为折扣显示在收据和报表中。

当在此基础上再应用优惠券时，这会导致总额不正确——优惠券会按原价 $18 计算，造成折扣过高，客户被少收款。

从 v1.9.0 开始，WCPOS 与 WooCommerce 对促销价的处理方式保持一致：在收银台设置的价格**就是**商品行小计。WooCommerce 仅将优惠券代码计为“折扣”（这与 WC 对促销商品的处理一致：促销价也会计入小计，不会单独显示折扣行）。

### 这对您意味着什么 {#what-this-means-for-you}

- **收据**可显示记录的常规价格、常规价与售价之间的节省金额，以及将这些节省金额与优惠券合并计算的 **总节省金额**。价格节省金额仍与 WooCommerce 的 **折扣** 字段分开显示。
- **报表**会在仅使用 POS 商品行价格调整时显示 `discount_total = 0`。只有优惠券折扣会被计入。
- **优惠券**现在可在 POS 折扣商品上叠加使用并正确计算。
- **记录的常规价格和销售价格存储在** POS 商品行元数据 (`_woocommerce_pos_data`), 因此收据可以在不更改 WooCommerce 折扣总额的情况下计算节省金额。

### 在收据上显示原价和节省金额 {#showing-the-original-price-and-saving-on-receipts}

当前随附的含价格模板会在商品促销或在收银台修改价格时显示记录的常规价格和节省金额。当 WCPOS 可以计算完整的订单级金额时，模板还会显示 **总节省金额**。该总额会合并常规价节省金额和优惠券折扣，且不会重复计算任一项。

从模板库复制的模板是可编辑的快照，不会被 WCPOS 更新覆盖。如果在此更改之前创建了收据，可选择以下任一方式：

1. 在随附模板的新副本上点击 **使用模板**；或
2. 使用[收据数据参考](/receipts/receipt-data#displaying-regular-price-and-savings)中的字段更新现有模板。

主要收据字段如下：

| 字段 | 含义 |
|-------|---------|
| `lines[].regular_price_display` | 记录的常规单价 |
| `lines[].selling_price_display` | 使用优惠券前的销售单价 |
| `lines[].unit_savings_display` | 每件节省金额 |
| `lines[].line_regular_total_display` | 该行的常规价格总计 |
| `lines[].line_savings_display` | 该行的节省总额 |
| `totals.savings_total_display` | 商品目录促销和收银台价格变更带来的常规价与销售价之间的节省总额 |
| `totals.discount_total_display` | WooCommerce 折扣总额，通常为当前订单中的优惠券折扣 |
| `totals.total_saved_display` | 常规价格节省金额与 WooCommerce 折扣的合计 |
| `totals.total_saved_complete` | 合计是否完整且可安全显示 |

对于自定义模板，请将 `totals.total_saved_display` 同时包裹在 `{{#totals.total_saved_complete}}` 和 `{{#totals.total_saved}}` 区段中。这样可在历史价格数据不完整或客户未节省任何金额时隐藏该行。有关完整示例，请参阅[收据数据参考](/receipts/receipt-data#totals)。

WooCommerce 报表保持不变：原生折扣总额仍包含优惠券，而非当前促销价或收银价格节省的金额。收据请使用 `totals.total_saved`；如果还需要在报表中将节省金额作为 WooCommerce 折扣进行追踪，请继续使用[优惠券](/coupons)。

## 已知限制 {#known-limitations}

- **没有自动折扣规则** — POS 不支持“买 2 件，送 1 件”这类自动折扣。结构化促销请使用 WooCommerce 优惠券。
- **快速折扣仅支持百分比** — 没有用于固定金额折扣的内置快捷按钮。请改用负费用或编辑单个价格。
