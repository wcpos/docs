- 打开模态框以管理订单的其他元数据。
- 功能包括：
  - 更改订单货币。
  - 添加交易ID（例如，来自外部支付终端）。
  - 为订单添加自定义元数据。

:::note
这个模态框将随着时间的推移添加更多功能，包括与其他WooCommerce插件的集成。
:::

### WooCommerce中的元数据是什么？

元数据提供有关订单的额外信息，并可以通过WooCommerce REST API访问。开发人员和插件使用这些数据来扩展功能，例如集成第三方服务。

### JSON 视图

- 查看订单的原始 JSON 表示，非常有助于调试或集成目的。

## 保存到服务器

- 允许您将打开的购物车保存到服务器，从而安全存储并便于将来使用。
- 创建状态为`pos-open`的WooCommerce订单。

### 为什么要保存？

当在POS中创建新订单时，它仅在本地存在。 保存订单确保它能持续存在，即使本地数据库被清除。

有关管理已保存订单的更多详细信息，请参见[打开的订单](/pos/cart/open-orders)。

## 作废

- 两种情况：
  - 未保存的订单：
    - 如果订单未保存到服务器，它将从本地数据库中永久删除。
  - 已保存的订单：
    - 如果订单已保存到服务器，它将被移动到WooCommerce中的垃圾箱文件夹，并从本地数据库中删除。
- 要恢复作废的订单：
  - 转到`WP Admin > WooCommerce > Orders > Trash`。

## 结账

- 将订单保存到服务器并启动结账流程。
- 打开[结账界面](/pos/checkout)以完成付款。
