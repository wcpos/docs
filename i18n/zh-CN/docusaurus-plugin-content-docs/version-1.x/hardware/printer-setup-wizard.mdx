  <WizardStep id="network-setup" title="连接网络打印机">

前往 **POS &gt; 设置 &gt; 打印 &gt; 添加打印机**。输入打印机的 **IP 地址**，然后保存。

要查找 IP 地址，请打印打印机的网络自检页：

- **Star：** 在打印机已开机的状态下，按住 **FEED** 按钮约 5 秒。
- **Epson：** 在开机时按住 **FEED** 按钮。

我们强烈建议为打印机分配一个**固定地址**（在路由器上设置 DHCP 保留），这样它就不会变化并在日后中断打印。

  <WizardFix id="fix-not-printing" title="当测试打印没有出来时">

首先做快速检查：是否有纸、机盖是否合上、纸张是否装反了？热敏纸只有一面可以打印——如果装反了，打印机就会走空白纸。做刮擦测试：用指甲刮一下纸张；有痕迹的那一面就是必须朝向打印头的那一面。

然后确认打印机可以访问：在 Web 浏览器中打开 `http://<printer-ip>/`（如果打不开，尝试 `https://<printer-ip>/`）。如果你看到打印机自己的配置页面，说明它已在网络中，问题出在设置上；如果两者都打不开，那就是网络/IP 问题（地址错误，或打印机未连接）。

  <WizardFix id="net-deep" title="更深入的网络检查">
如果基础项都没问题，但仍然无法打印，通常是以下某个网络问题导致的：

**IP 错误，或已改变。** 确认打印机自检页上的 IP 与你输入的一致——很容易误用了另一台打印机的地址。使用 DHCP 的打印机可能在重启后更改地址；请在路由器上设置 **DHCP 保留**，让它保持不变。以 `169.254` 开头的地址表示打印机根本没有连上网络。

**曾经可用，然后当天第一次打印失败。** 打印机的网卡进入了休眠；第一个作业将其唤醒并丢失了。再发送一次测试打印，并考虑为固定收银台使用有线 **Ethernet** 连接。

**两台收银机，一台打印机。** 许多 Star 网络打印机默认一次只接受一个连接——第二台收银机会超时。请在打印机的 Web 配置中启用 **9100 多会话**。

**打印机的网络设置使用了错误的协议。** WCPOS 通过 **端口 9100 上的原始 TCP** 发送作业。在浏览器中打开打印机的配置页面（`http://<printer-ip>/`，如果打不开则用 `https://<printer-ip>/`），并确保已启用 **raw / 端口 9100** 打印——如果打印机仅设置为 LPR、IPP 或 WSD，作业将永远无法送达。

**Wi-Fi 无法加入，或不断断开。** 收据打印机通常**仅支持 2.4GHz**——请将它们连到 2.4GHz 网络，而不是仅支持 5GHz 或使用频段引导的网络。对于不稳定的连接，有线 Ethernet 是可靠的解决方案。

  <WizardStep id="web-net-setup" title="从 Web 应用连接网络打印机">

**从 Web 浏览器**打印到网络打印机是最棘手的设置，因为浏览器只能通过 HTTP/HTTPS 与打印机的内置 Web 服务器通信——而安全（HTTPS）的 POS 也会强制要求与打印机建立安全连接。

首先，确认打印机可以访问：在新的浏览器标签页中打开 `http://<printer-ip>/`（如果打不开，尝试 `https://<printer-ip>/`）。如果你看到打印机自己的配置页面，说明它已在网络中；如果两者都打不开，请先修复网络/IP 问题。

然后在 **POS &gt; 设置 &gt; 打印** 中用打印机的 IP 地址添加打印机。如果你的 POS 通过 **HTTPS** 运行，与打印机的连接会被强制使用 HTTPS（Epson 端口 8043，Star 443）——只有当打印机拥有受信任的 **SSL 证书**时才能正常工作。

  <WizardFix id="web-net-fix" title="当 Web 网络打印无法工作时">

这是最常见的死胡同，而且通常无法从浏览器中解决。原因及应对方法如下：

打印打印机的**自检页**，查看 SSL/TLS 部分。如果它显示证书**不存在**，那么在你生成一个证书之前，HTTPS 打印无法工作——而且除非你访问一次 `https://<printer-ip>` 并在每台设备上接受它，否则浏览器不会信任自签名证书。

如果出现“发送成功”的消息但没有任何打印，说明浏览器悄悄拦截了请求（打印机使用 HTTP 而你的 POS 是 HTTPS——即“混合内容”拦截），或者在没有证书的情况下强制使用了 HTTPS 端口（Epson 为 8043，Star 为 443）。

Web 应用还只能通过打印机的**内置 Web 服务器**——Epson 的 **ePOS-Print** 或 Star 的 **WebPRNT**——访问打印机，因此请确保该服务在打印机的配置中已**开启**。只有 Epson 的“intelligent”/ **-NT** 型号和 Star **WebPRNT** 型号才配有该服务；如果你的型号没有，浏览器就无法访问。

**可靠的解决方案是桌面应用**，它可通过网络直接连接到打印机（端口 9100），完全没有浏览器的 HTTPS 限制。从[打印机设置](/hardware/printers)下载它，并在那里添加同一台打印机。

  <WizardStep id="bt-setup" title="通过 Bluetooth 连接">

Bluetooth 收据打印使用打印机厂商自己的软件，因此打印机必须是 **Epson** 或 **Star** 型号——通用 Bluetooth 打印机在任何平台上都不支持通过 Bluetooth 连接。

首先，**在你的计算机或设备的系统 Bluetooth 设置中配对打印机**——而不是在 WCPOS 中。Bluetooth 打印机一次只记住一台设备，因此如果它曾与另一台收银机配对，请先在那里“忘记”它。在 **iPhone/iPad** 上，还要在提示时同时授予 WCPOS 的 **Bluetooth** 和 **本地网络** 权限。在 **Android** 上，将 WCPOS 设置为**不受限制**的电池使用，以免系统断开连接。在 **Windows 或 Mac 桌面**上，先在操作系统的 Bluetooth 设置中配对，然后在 WCPOS 中扫描。

然后在 **POS &gt; 设置 &gt; 打印** 中，点按 **扫描打印机** 并选择你的打印机——它的名称和品牌会自动填入。

  <WizardFix id="bt-fix" title="Bluetooth 无法配对或不断断开">

先在设备的**系统 Bluetooth 设置中配对打印机**，然后在 WCPOS 中再次扫描。如果它之前配对过但无法重新连接，请在另一台设备上“忘记”它（并重置打印机的 Bluetooth），使它不再绑定在那里。确保它已开机且在连接范围内。在 iPhone/iPad 上，确认 WCPOS 同时拥有 Bluetooth 和本地网络权限；在 Android 上，将该应用设置为不受限制的电池使用。请记住 Bluetooth **仅支持 Epson 和 Star**——通用 Bluetooth 打印机无法连接。

  <WizardFix id="usb-fix" title="USB 无法连接">

在 **Android** 上，重新插入线缆并批准 USB 权限提示。在**桌面应用（Windows）**上，出现 `LIBUSB_ERROR_NOT_SUPPORTED` 错误意味着 Windows 已将该设备锁定到它自己的打印驱动程序——最简单的解决办法是**改为通过网络连接打印机（端口 9100）**，这样可以完全避开 USB 驱动程序。（高级用户可以用 WinUSB 工具替换驱动程序，但这会禁用该设备的正常 Windows 打印队列——只有在你清楚自己在做什么时才这样做。）
