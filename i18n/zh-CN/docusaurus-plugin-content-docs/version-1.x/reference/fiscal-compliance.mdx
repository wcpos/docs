一些国家/地区法律要求销售点软件必须经过“税务欺诈认证”，或与国家财政机构整合。本页总结了 WCPOS 的立场，以便您可以在依赖它之前**检查它是否适合您的管辖范围。

:::warning WCPOS 不是经过认证的财务 POS
WCPOS 目前 **不** 持有任何受监管市场的财政认证。如果您所在的国家/地区需要经过认证的 POS 软件，请在使用 WCPOS 进行实时销售之前确认您当地的要求。 WCPOS 充当 WooCommerce 的中间件 — 发票和 VAT 最终在 WooCommerce 级别处理。
:::

:::info 我们正在努力
财政合规性是一个活跃、持续的发展领域 - 我们的目标是使 WCPOS 在我们的用户运营所在的所有国家/地区都符合要求。下面的状态反映的是**今天**，而不是我们的发展方向。如果您所在地区的合规性对您很重要，请在[社区](https://wcpos.com/discord) 中告诉我们 — 这有助于我们确定下一步要应对哪些市场的优先顺序。
:::

## WCPOS 如何处理税收 {#how-wcpos-handles-tax}

WCPOS 使用从 WooCommerce 下载的费率在应用程序中本地计算税费，然后在 WooCommerce 中创建订单。这速度很快并且可以离线工作，但这意味着经过认证的/服务器端税务逻辑（例如某些合规性插件）仅在订单到达服务器时运行。 **“保存到服务器”** 操作和 **Web 结帐** / **电子邮件发票** 网关通过标准 WooCommerce 结帐发送订单，其中适用服务器端税务和合规性规则。

## 国家状况 {#country-status}

### 法国 — NF525 {#france-nf525}

WCPOS 对于 NF525 **未经过认证**。法国法律要求 POS 系统经过防欺诈认证，因此这会影响在法国的销售活力。由于 WCPOS 是 WooCommerce 的中间件，因此开票和 VAT 合规性在 WooCommerce 级别处理 - 请参阅 WooCommerce 有关 [法国 VAT 欺诈预防法](https://woocommerce.com/posts/french-vat-fraud-prevention-laws-how-to-certify-invoices-with-woocommerce/) 的指南。

### 西班牙 — VERIFACTU {#spain-verifactu}

WCPOS 对于 VERIFACTU **未经过认证**。西班牙要求POS软件经过认证； VERIFACTU 截止日期已**推迟至 2027 年**。该团队已了解该要求，但尚未承诺认证时间表。

### 挪威 — Kassasystemloven {#norway-kassasystemloven}

WCPOS **未经过挪威收银法 (_Kassasystemloven_) 认证**，并且没有可用的供应商声明。挪威的一些商家确实使用 POS，但您应该确认自己的法律义务。

### 智利 — SII {#chile-sii}

**没有与智利 Servicio de Impuestos Internos (SII) 集成**以验证销售收据 (_boletas_)。收据模板是标准的、可自定义的 WooCommerce 模板，但没有内置 SII 合规性。

### 欧盟 — 一站式服务 (OSS) {#eu-oss}

POS 本身**不**支持用于跨境销售的欧盟一站式服务 (OSS) VAT 方案，因为它根据下载的费率在本地计算税费。

**解决方法：**
- 结帐前使用 **保存到服务器** 通过 OSS 插件触发服务器端税务重新计算，或者
- 使用 **Web Checkout** 或 **Email Invoice** 网关通过标准 WooCommerce 结账进行付款，其中 OSS 规则适用。

## 其他受监管的市场 {#other-regulated-markets}

对于需要经过认证的 POS 软件的任何司法管辖区，请首先检查您当地的要求。 WCPOS 目前不持有受监管市场的认证。如果认证状态对您的购买决定很重要，请在做出决定之前在[社区](https://wcpos.com/discord) 中询问。
