# API06001: 无效的 URL 格式

## 这意味着什么

为您的 WooCommerce 网站配置的 URL 格式无效。URL 必须遵循标准的网页地址惯例。

## 常见原因

- **缺少协议** — URL 没有以 `http://` 或 `https://` 开头
- **URL 中的拼写错误** — 域名中的拼写错误
- **无效字符** — URL 中不允许的特殊字符
- **不完整的 URL** — URL 缺少部分，例如域名扩展

## 如何修复

### 1. 检查 URL 格式

有效的 URL 应该如下所示：
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com`（缺少协议）
- ❌ `https://yourstore`（缺少扩展）
- ❌ `https://your store.com`（包含空格）

### 2. 包括协议

始终包括 `https://` 或 `http://`：
- 优先选择：`https://`（安全）
- 备用：`http://`（如果 HTTPS 不可用）

### 3. 删除多余字符

确保 URL 没有：
- 结尾斜杠（可选但要保持一致）
- 多余的空格
- 特殊字符
- URL 参数（除非必要）

### 4. 验证 URL 是否有效

在浏览器中测试 URL：
1. 复制您使用的 URL
2. 粘贴到浏览器中
3. 应加载您的 WordPress 网站

### 5. 更新 POS 配置

在 POS 设置中输入正确的 URL：
- 与浏览器中有效的 URL 完全匹配
- 根据您的网站使用情况包括或排除 `www.`

## 示例

| 无效 | 有效 |
|------|------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
