# API06001：无效的 URL 格式

## 这意味着什么

为您的 WooCommerce 站点配置的 URL 不符合有效的格式。URL 必须遵循标准的网页地址约定。

## 常见原因

- **缺少协议** — URL 未以 `http://` 或 `https://` 开头
- **URL 拼写错误** — 域名中有拼写错误
- **无效字符** — 使用了不允许的特殊字符
- **URL 不完整** — URL 缺少域名扩展名等部分

## 如何修复

### 1. 检查 URL 格式

有效的 URL 应该是：
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com`（缺少协议）
- ❌ `https://yourstore`（缺少扩展名）
- ❌ `https://your store.com`（包含空格）

### 2. 包含协议

始终包含 `https://` 或 `http://`：
- 首选：`https://`（安全）
- 备选：`http://`（如果 HTTPS 不可用）

### 3. 删除额外字符

确保 URL 没有：
- 尾随斜杠（可选，但要保持一致）
- 额外空格
- 特殊字符
- URL 参数（除非必需）

### 4. 验证 URL 可用性

在浏览器中测试 URL：
1. 复制您正在使用的 URL
2. 将其粘贴到浏览器中
3. 应能加载您的 WordPress 站点

### 5. 更新 POS 配置

在 POS 设置中输入更正后的 URL：
- 与在浏览器中有效的 URL 完全匹配
- 根据您的站点情况包含或排除 `www.`

## 示例

| 无效 | 有效 |
|---------|-------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
