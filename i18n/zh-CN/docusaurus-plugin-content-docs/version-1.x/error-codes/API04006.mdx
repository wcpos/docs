# API04006：资源未找到

## 这意味着什么

服务器无法找到请求的资源。这通常对应于HTTP 404状态码，表示您尝试访问的产品、订单、客户或其他资源不存在。

## 常见原因

- **记录已删除** — 资源已在服务器上被删除
- **ID错误** — 使用了不正确或过时的ID
- **同步问题** — 本地数据引用了不再存在的资源
- **URL配置错误** — API端点不正确
- **永久链接问题** — WordPress永久链接需要刷新

## 服务器错误映射

当服务器返回以下情况时触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `rest_no_route` | WordPress REST API |
| HTTP 404 | 任何服务器响应 |

## 如何修复

### 1. 检查资源是否存在

验证资源是否仍存在于您的WooCommerce站点：
- 登录WordPress管理后台
- 导航到相关部分（产品、订单、客户）
- 通过ID或名称搜索项目

### 2. 刷新本地数据

如果资源在服务器端已删除：
1. 打开POS设置
2. 导航到相关数据部分
3. 触发同步/刷新以更新本地数据
4. 已删除的项目应在本地被移除

### 3. 检查WordPress永久链接

如果多个资源未找到：
1. 在WordPress管理后台转到**设置 → 永久链接**
2. 点击**保存更改**（即使未做任何更改）
3. 这将刷新永久链接结构

### 4. 验证API路由

直接测试REST API：
```
https://yoursite.com/wp-json/wc/v3/products
```
如果返回404，可能存在服务器配置问题。

### 5. 检查插件冲突

如果REST API路由丢失：
1. 确保WooCommerce已激活
2. 确保WCPOS插件已激活
3. 暂时禁用其他插件以测试
