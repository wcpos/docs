# API01004: DNS解析失败 {#api01004-dns-resolution-failed}

## 这意味着什么 {#what-this-means}

POS无法将您网站的域名（例如，`yourstore.com`）转换为IP地址。DNS（域名系统）就像互联网的电话簿 — 如果找不到条目，它就无法建立连接。

## 常见原因 {#common-causes}

- **域名拼写错误** — URL可能有拼写错误
- **DNS服务器问题** — 您的DNS服务器未响应
- **域名过期** — 域名注册可能已失效
- **DNS传播** — 最近的DNS更改尚未传播
- **本地DNS缓存** — 您设备上缓存的DNS信息已过时

## 如何修复 {#how-to-fix}

### 1. 核对域名 {#1-verify-the-domain-name}

仔细检查URL是否有拼写错误。尝试在浏览器中访问该网站。

### 2. 检查域名状态 {#2-check-domain-status}

使用WHOIS查询进行验证：
- 确认域名仍然注册
- 没有过期
- DNS记录已配置

### 3. 清除DNS缓存 {#3-clear-dns-cache}

**在您的设备上：**

macOS：
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows：
```bash
ipconfig /flushdns
```

### 4. 尝试备用DNS {#4-try-alternative-dns}

切换到公共DNS服务器，例如：
- Google: `8.8.8.8` 和 `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. 等待DNS传播 {#5-wait-for-dns-propagation}

如果您最近更改了DNS设置，请等待24-48小时以便完全传播。
