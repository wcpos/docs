# API01004：DNS解析失败

## 这意味着什么

POS无法将您网站的域名（例如 `yourstore.com`）转换为IP地址。DNS（域名系统）就像互联网的电话簿 — 如果找不到记录，就无法建立连接。

## 常见原因

- **域名输入错误** — URL可能有拼写错误
- **DNS服务器问题** — 您的DNS服务器未响应
- **域名过期** — 域名注册可能已失效
- **DNS传播** — 最近的DNS更改尚未完全传播
- **本地DNS缓存** — 设备上缓存的DNS信息已过时

## 如何修复

### 1. 验证域名

仔细检查URL是否有拼写错误。尝试在浏览器中访问该站点。

### 2. 检查域名状态

使用WHOIS查询验证：
- 域名仍在注册
- 未过期
- DNS记录已配置

### 3. 清除DNS缓存

**在您的设备上：**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. 尝试备用DNS

切换到公共DNS服务器，如：
- Google: `8.8.8.8` 和 `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. 等待DNS传播

如果您最近更改了DNS设置，请等待24-48小时以完成完全传播。
