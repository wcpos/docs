# DB01001：连接失败

## 这意味着什么

POS 无法连接到其本地数据库。WCPOS 使用本地数据库来存储产品、客户和其他数据，以便快速访问和离线功能。

## 常见原因

- **存储空间已满** — 设备没有剩余存储空间
- **数据库损坏** — 本地数据库文件已损坏
- **权限问题** — 应用没有存储权限
- **浏览器存储被禁用** — IndexedDB 已禁用（网页版）

## 如何修复

### 1. 检查存储空间

确保您的设备有可用存储空间：
- 如果存储空间已满，请释放空间
- 删除未使用的应用或文件
- 检查可用磁盘空间

### 2. 清除应用数据

重置本地数据库：
- **桌面应用：** 在设置中查找"清除数据"
- **网页浏览器：** 清除 POS 域的站点数据
- **注意：** 这将需要重新同步所有数据

### 3. 检查浏览器设置（网页版）

如果使用网页版：
1. 确保 IndexedDB 已启用
2. 不要使用隐私/无痕模式（存储受限）
3. 允许站点存储数据

### 4. 检查权限

在桌面上：
- 确保应用有权写入其数据目录
- 检查防病毒软件是否阻止数据库访问

### 5. 重新安装应用

如果其他解决方案失败：
1. 卸载 POS 应用
2. 重启设备
3. 重新安装应用
4. 登录并同步数据

## 数据恢复

如果您有未保存的数据：
- 应用可能有尚未同步的待处理更改
- 如需帮助恢复数据，请联系支持
- 在大多数情况下，可以从服务器重新同步数据
