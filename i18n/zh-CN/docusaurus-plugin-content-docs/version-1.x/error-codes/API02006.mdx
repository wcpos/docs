# API02006：API密钥无效

## 这意味着什么

正在使用的WooCommerce REST API密钥是无效的。API密钥用于服务器间身份验证，提供的密钥与WooCommerce中任何有效密钥都不匹配。

## 常见原因

- **密钥已删除** — API密钥已从WooCommerce中移除
- **密钥拼写错误** — 密钥输入不正确
- **错误的密钥对** — 消费者密钥和密钥不匹配
- **来自其他站点的密钥** — 使用为另一个安装生成的密钥

## 如何修复

### 1. 验证API密钥

在WordPress管理后台 → WooCommerce → 设置 → 高级 → REST API：
1. 检查您的API密钥是否存在
2. 确认密钥没有被吊销
3. 记下权限（读取/写入/读写）

### 2. 生成新的API密钥

如果密钥丢失或无效：
1. 转到WooCommerce → 设置 → 高级 → REST API
2. 点击"添加密钥"
3. 输入描述（例如"WCPOS"）
4. 选择用户
5. 选择"读/写"权限
6. 点击"生成API密钥"
7. **复制消费者密钥和消费者密码**（仅显示一次！）

### 3. 更新POS配置

在POS中输入新的API密钥：
- 消费者密钥（以 `ck_` 开头）
- 消费者密码（以 `cs_` 开头）

### 4. 检查密钥权限

确保密钥具有足够的权限：
- **读取** — 仅查看数据
- **写入** — 仅修改数据
- **读/写** — 完全访问（建议用于POS）
