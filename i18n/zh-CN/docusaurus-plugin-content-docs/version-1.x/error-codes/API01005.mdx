# API01005：SSL 证书错误

## 这意味着什么

您网站的 SSL/TLS 证书存在问题。该证书是确保连接安全（HTTPS）的关键。为了保护您的数据，POS 系统不会连接到具有无效证书的站点。

## 常见原因

- **证书过期** — SSL 证书需要定期续期
- **自签名证书** — 未由受信任的机构颁发
- **域名不匹配** — 证书与域名不一致
- **证书链不完整** — 缺少中间证书
- **混合内容** — 部分资源通过 HTTP 而非 HTTPS 加载

## 如何修复

### 1. 检查证书状态

在浏览器中访问您的网站，点击锁图标查看证书详情。注意：
- 到期日期
- 颁发对象（应匹配您的域名）
- 颁发机构（应为可信机构）

### 2. 续期过期证书

如果已过期：
- 大多数主机提供商提供免费的 Let's Encrypt 证书
- 联系您的主机提供商续期
- 检查是否启用自动续期

### 3. 修复证书不匹配问题

确保证书覆盖：
- 精确域名（`yourstore.com`）
- WWW 变体（`www.yourstore.com`，如果使用）
- 考虑使用通配符证书（`*.yourstore.com`）

### 4. 安装缺失的中间证书

使用 SSL 检查工具（如 SSL Labs）识别缺失的证书。您的主机提供商可帮助安装。

### 5. 强制使用 HTTPS

在 WordPress 中，确保：
- 站点 URL 使用 `https://`
- WooCommerce 设置中启用强制 SSL
