# DB02002: 记录未找到

## 这意味着什么

请求的记录在本地数据库中不存在。POS尝试访问未存储在本地的产品、客户、订单或其他记录。

## 常见原因

- **尚未同步** — 数据尚未下载
- **服务器上已删除** — 记录在WooCommerce中被移除
- **同步未完成** — 初始同步仍在运行
- **应用了过滤器** — 记录被活动过滤器排除

## 如何修复

### 1. 等待同步

如果最近开始：
- 等待初始同步完成
- 检查同步进度指示器
- 记录应该在同步后出现

### 2. 刷新/重新同步

触发数据刷新：
- 使用刷新/同步按钮 (<Icon name="arrow-rotate-right" />) - 短按以进行同步
- **长按**同步按钮以选择清除和刷新选项
- 从服务器拉取最新数据
- 等待完成

### 3. 检查WooCommerce

验证记录在服务器上存在：
- 登录WordPress Admin
- 检查产品、订单或客户
- 确认项目仍然存在

### 4. 检查过滤器

记录可能被过滤掉：
- 查看POS中的活动过滤器
- 检查类别过滤器
- 验证可见性设置

### 5. 清除并重新同步

如果同步似乎卡住：
1. 清除本地数据
2. 再次登录
3. 完成新的同步

## 常见场景

- **产品未显示：** 检查其是否已发布且可见
- **客户未找到：** 可能尚未同步
- **订单丢失：** 可能超出了同步日期范围
