# DB02002: 记录未找到

## 这意味着什么

请求的记录在本地数据库中不存在。POS尝试访问一个未在本地存储的产品、客户、订单或其他记录。

## 常见原因

- **尚未同步** — 数据尚未下载
- **在服务器上删除** — 记录已在WooCommerce中移除
- **同步未完成** — 初始同步仍在进行中
- **应用了过滤器** — 记录被活动过滤器排除

## 如何解决

### 1. 等待同步

如果最近开始：
- 等待初始同步完成
- 检查同步进度指示器
- 同步后记录应会出现

### 2. 刷新/重新同步

触发数据刷新：
- 使用刷新/同步按钮
- 从服务器提取最新数据
- 等待完成

### 3. 检查WooCommerce

验证记录在服务器上存在：
- 登录WordPress管理后台
- 检查产品、订单或客户
- 确认该项目仍然存在

### 4. 检查过滤器

记录可能被过滤掉：
- 查看POS中的活动过滤器
- 检查类别过滤器
- 验证可见性设置

### 5. 清除并重新同步

如果同步似乎卡住：
1. 清除本地数据
2. 重新登录
3. 完成新同步

## 常见场景

- **产品未显示：** 检查它是否已发布并可见
- **客户未找到：** 可能尚未同步
- **订单缺失：** 可能在同步日期范围之外
