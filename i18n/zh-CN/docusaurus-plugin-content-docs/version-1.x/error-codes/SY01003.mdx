# SY01003: 权限被拒绝

## 这意味着什么

POS 应用程序没有权限访问所需资源。这可能是文件、文件夹或系统功能。

## 常见原因

- **文件权限** — 无法读取或写入所需文件
- **应用程序权限** — 应用程序缺少所需权限
- **受保护的文件夹** — 尝试访问受限制的位置
- **安全软件** — 杀毒软件阻止访问

## 如何修复

### 1. 以管理员身份运行（Windows）

右键单击应用程序，选择“以管理员身份运行”，以便进行初始设置。

### 2. 检查应用程序权限

**macOS:**
- 系统偏好设置 → 安全性与隐私 → 隐私
- 检查该应用是否具有必要的权限

**Windows:**
- 设置 → 隐私
- 审查应用权限

### 3. 检查文件夹权限

确保应用可以访问其数据文件夹：
- 用户应该拥有应用数据目录
- 应启用写入权限
- 文件夹上不得有只读标志

### 4. 审查杀毒软件设置

安全软件可能会阻止访问：
- 检查杀毒软件日志以查看阻止情况
- 将 POS 应用添加到允许列表中
- 白名单应用数据文件夹

### 5. 重新安装应用程序

如果权限已损坏：
1. 卸载 POS 应用程序
2. 重启设备
3. 安装到标准位置
4. 以普通（非管理员）权限运行

## 浏览器权限（Web 版本）

如果使用 Web 版本：
- 允许网站使用本地存储
- 启用网站的 cookies
- 不要使用过于严格的浏览器设置
