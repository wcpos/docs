# API06003: 无效的网站配置

## 这意味着什么

网站配置无效或不完整。这可能涉及错误的 URL、身份验证设置或其他配置问题。

## 常见原因

- **部分配置** — 某些设置缺失
- **设置不匹配** — 配置与网站不匹配
- **损坏的配置** — 设置已损坏
- **网站更改** — 网站已被修改而未更新 POS 配置

## 如何修复

### 1. 检查所有设置

检查完整配置：
- 网站 URL 正确
- 身份验证配置正确
- 任何附加设置均正确

### 2. 从头开始重新配置

如果配置已损坏：
1. 清除所有存储的设置
2. 重新开始设置过程
3. 输入新的配置

### 3. 检查网站要求

验证您的 WordPress 网站满足要求：
- WordPress 已安装并可访问
- WooCommerce 已安装并处于活动状态
- WCPOS 插件已安装并处于活动状态
- 启用永久链接（不是“简单”）

### 4. 测试网站访问

验证这些 URL 在浏览器中有效：
- `https://yoursite.com/` — 主要网站
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. 检查 WordPress 设置

在 WordPress 管理后台：
1. 设置 → 常规 — 验证 URL
2. 设置 → 永久链接 — 确保不是“简单”
3. WooCommerce → 状态 — 检查问题

## 配置要求

有效的配置需要：
- **网站 URL** — 带协议的完整 URL
- **身份验证** — 有效的凭据或 API 密钥
- **权限** — 用户具有 POS 访问权限
- **插件** — 所需插件处于活动状态
