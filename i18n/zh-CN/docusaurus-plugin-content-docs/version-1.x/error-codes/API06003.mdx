# API06003：站点配置无效

## 这意味着什么

站点配置无效或不完整。这可能涉及不正确的 URL、认证设置或其他配置问题。

## 常见原因

- **部分配置** — 某些设置缺失
- **设置不匹配** — 配置与站点不符
- **配置损坏** — 设置已被破坏
- **站点变更** — 站点已修改但未更新 POS 配置

## 如何修复

### 1. 检查所有设置

检查完整配置：
- 站点 URL 正确
- 认证正确配置
- 所有其他设置正确

### 2. 从头重新配置

如果配置已损坏：
1. 清除所有存储的设置
2. 重新开始设置过程
3. 输入全新配置

### 3. 检查站点要求

验证您的 WordPress 站点是否满足要求：
- WordPress 已安装并可访问
- WooCommerce 已安装并激活
- WCPOS 插件已安装并激活
- 已启用固定链接（非"简单"模式）

### 4. 测试站点访问

在浏览器中验证这些 URL 是否有效：
- `https://yoursite.com/` — 主站点
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. 检查 WordPress 设置

在 WordPress 管理后台：
1. 设置 → 常规 — 验证 URL
2. 设置 → 固定链接 — 确保不是"简单"模式
3. WooCommerce → 状态 — 检查问题

## 配置要求

有效配置需要：
- **站点 URL** — 包含协议的完整 URL
- **认证** — 有效的凭据或 API 密钥
- **权限** — 用户有 POS 访问权限
- **插件** — 必需插件已激活
