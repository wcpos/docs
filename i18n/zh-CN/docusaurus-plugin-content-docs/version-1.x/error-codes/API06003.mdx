# API06003: 无效的站点配置 {#api06003-invalid-site-configuration}

## 这意味着什么 {#what-this-means}

站点配置无效或不完整。这可能涉及错误的 URL、身份验证设置或其他配置问题。

## 常见原因 {#common-causes}

- **部分配置** — 某些设置缺失
- **设置不匹配** — 配置与站点不匹配
- **损坏的配置** — 设置已损坏
- **站点更改** — 站点已被修改而未更新 POS 配置

## 如何修复 {#how-to-fix}

### 1. 检查所有设置 {#1-review-all-settings}

检查完整配置：
- 站点 URL 正确
- 身份验证配置正确
- 任何附加设置均正确

### 2. 从头开始重新配置 {#2-reconfigure-from-scratch}

如果配置已损坏：
1. 清除所有存储的设置
2. 重新开始设置过程
3. 输入新的配置

### 3. 检查站点要求 {#3-check-site-requirements}

验证您的 WordPress 站点满足要求：
- WordPress 已安装并可访问
- WooCommerce 已安装并处于活动状态
- WCPOS 插件已安装并处于活动状态
- 启用永久链接（不是“普通”）

### 4. 测试站点访问 {#4-test-site-access}

验证这些 URL 在浏览器中可用：
- `https://yoursite.com/` — 主站点
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. 检查 WordPress 设置 {#5-check-wordpress-settings}

在 WordPress 管理后台：
1. 设置 → 常规 — 验证 URL
2. 设置 → 永久链接 — 确保不是“普通”
3. WooCommerce → 状态 — 检查问题

## 配置要求 {#configuration-requirements}

有效的配置需要：
- **站点 URL** — 带协议的完整 URL
- **身份验证** — 有效的凭据或 API 密钥
- **权限** — 用户具有 POS 访问权限
- **插件** — 所需插件处于活动状态
