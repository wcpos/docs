# API02007：令牌刷新失败

## 这意味着什么

POS尝试自动续新您的身份验证会话，但刷新请求失败。这通常意味着您需要重新登录。

## 常见原因

- **刷新令牌已过期** — 刷新令牌本身已过期
- **刷新令牌已撤销** — 管理员使所有会话失效
- **服务器错误** — 服务器无法处理刷新请求
- **网络问题** — 刷新尝试期间的连接问题

## 如何解决

### 1. 重新登录

最可靠的修复方法：
1. 从POS注销
2. 使用您的凭据重新登录
3. 这将创建一个带有新令牌的新会话

### 2. 检查网络连接

如果由于网络问题导致刷新失败：
- 验证您的互联网连接
- 尝试在浏览器中访问您的WordPress站点
- 等待片刻后重试

### 3. 检查服务器状态

如果服务器出现问题：
- 验证WordPress正常运行
- 检查服务器日志中的PHP错误
- 确保身份验证插件处于活动状态

### 4. 清除会话数据

如果问题持续：
- 清除POS应用缓存
- 删除存储的凭据
- 使用新登录重新开始

## 自动令牌刷新

WCPOS在令牌过期之前自动刷新令牌以维持您的会话。如果此过程反复失败，可能存在需要关注的配置问题。
