# API02007: 令牌刷新失败 {#api02007-token-refresh-failed}

## 这意味着什么 {#what-this-means}

POS 尝试自动续订您的身份验证会话，但刷新请求失败。这通常意味着您需要再次登录。

## 常见原因 {#common-causes}

- **刷新令牌已过期** — 刷新令牌本身已过期
- **刷新令牌已被撤销** — 管理员使所有会话失效
- **服务器错误** — 服务器无法处理刷新请求
- **网络问题** — 刷新尝试期间的连接问题

## 如何修复 {#how-to-fix}

### 1. 再次登录 {#1-log-in-again}

最可靠的修复方法：
1. 登出 POS
2. 使用您的凭据重新登录
3. 这将创建一个带有新令牌的新会话

### 2. 检查网络连接 {#2-check-network-connection}

如果刷新因网络问题而失败：
- 验证您的互联网连接
- 尝试在浏览器中访问您的 WordPress 网站
- 等待片刻并再试一次

### 3. 检查服务器状态 {#3-check-server-status}

如果服务器存在问题：
- 验证 WordPress 是否正常运行
- 检查服务器日志中的 PHP 错误
- 确保身份验证插件处于活动状态

### 4. 清除会话数据 {#4-clear-session-data}

如果问题持续：
- 清除 POS 应用缓存
- 删除存储的凭据
- 通过新的登录开始新的会话

## 自动令牌刷新 {#automatic-token-refresh}

WCPOS 会在令牌过期前自动刷新令牌以维持您的会话。如果此过程反复失败，可能存在需要注意的配置问题。
