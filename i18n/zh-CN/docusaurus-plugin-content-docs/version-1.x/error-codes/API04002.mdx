# API04002: 意外的响应代码 {#api04002-unexpected-response-code}

## 这意味着什么 {#what-this-means}

服务器返回了一个不符合该请求预期的HTTP状态代码。常见代码包括500（服务器错误）、403（禁止）、404（未找到）等。

## 常见原因 {#common-causes}

- **500 内部服务器错误** — PHP崩溃或遇到了错误
- **403 禁止** — 安全设置拒绝访问
- **404 未找到** — 端点不存在
- **502/503/504** — 服务器网关或可用性问题

## 如何修复 {#how-to-fix}

### 对于500错误（服务器错误） {#for-500-errors-server-error}

1. 检查PHP错误日志
2. 增加PHP内存限制
3. 查看插件冲突
4. 检查 `wp-content/debug.log`

### 对于403错误（禁止） {#for-403-errors-forbidden}

1. 检查安全插件设置（Wordfence、Sucuri等）
2. 将POS或您的IP加入白名单
3. 检查`.htaccess`中的阻止规则
4. 验证ModSecurity没有阻止请求

### 对于404错误（未找到） {#for-404-errors-not-found}

1. 确保WCPOS插件已激活
2. 刷新WordPress的固定链接（设置 → 固定链接 → 保存）
3. 检查REST API是否已启用
4. 验证URL配置

### 对于502/503/504错误（网关问题） {#for-502503504-errors-gateway-issues}

1. 联系您的主机提供商
2. 等待服务器恢复
3. 检查站点是否处于高负载
4. 验证服务器运行状态

### 一般故障排除 {#general-troubleshooting}

1. 尝试直接访问您的站点
2. 检查主机控制面板是否有问题
3. 查看服务器访问日志
4. 如有需要，请联系主机支持
