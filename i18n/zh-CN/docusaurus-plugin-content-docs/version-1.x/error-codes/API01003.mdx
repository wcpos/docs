# API01003: 连接重置

## 这意味着什么

与服务器的连接已建立，但随后意外关闭。这通常发生在通信中途，当某些事情中断连接时。

## 常见原因

- **服务器崩溃** — 服务器进程可能在请求期间崩溃
- **内存限制** — PHP 用尽内存并被终止
- **请求中超时** — 服务器在处理时超时
- **网络不稳定** — 间歇性网络问题
- **安全软件** — 激进的安全设置关闭连接

## 如何修复

### 1. 检查服务器错误日志

查看您的 WordPress 错误日志或托管控制面板，寻找重置时相关的错误。

### 2. 增加 PHP 内存限制

在您的 `wp-config.php` 中：

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. 检查插件冲突

某个插件可能导致 PHP 崩溃：
- 暂时禁用插件
- 一一重新启用以找出罪魁祸首

### 4. 检查服务器资源

联系您的托管提供商检查：
- PHP 进程限制
- 发生错误时的内存使用
- 服务器稳定性日志

### 5. 测试网络稳定性

如果使用 WiFi，请尝试使用有线连接。如果在不同网络上问题仍然存在，可能是服务器端的问题。
