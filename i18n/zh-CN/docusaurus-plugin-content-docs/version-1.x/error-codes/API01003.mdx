# API01003：连接重置

## 这意味着什么

到服务器的连接已建立，但随后意外关闭。这通常发生在通信过程中连接被中断的情况。

## 常见原因

- **服务器崩溃** — 服务器进程在请求期间可能崩溃
- **内存限制** — PHP耗尽内存并被终止
- **请求超时** — 服务器在处理过程中超时
- **网络不稳定** — 间歇性网络问题
- **安全软件** — aggressive安全设置关闭连接

## 如何修复

### 1. 检查服务器错误日志

查看WordPress错误日志或主机控制面板，找出重置时间附近的相关错误。

### 2. 增加PHP内存限制

在 `wp-config.php` 中：

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. 检查插件冲突

插件可能导致PHP崩溃：
- 暂时禁用插件
- 逐一重新启用以找出问题插件

### 4. 检查服务器资源

联系主机提供商，检查：
- PHP进程限制
- 错误发生时的内存使用情况
- 服务器稳定性日志

### 5. 测试网络稳定性

如果在WiFi上，尝试有线连接。如果在不同网络上问题仍然存在，很可能是服务器端的问题。
