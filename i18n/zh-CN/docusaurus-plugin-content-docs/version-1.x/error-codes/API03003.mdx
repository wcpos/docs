# API03003：无效的参数值

## 这意味着什么

你的请求中的一个值无效。数据格式、类型或值与服务器预期的不匹配。

## 常见原因

- **错误的数据类型** — 在预期数字的地方使用了文本（反之亦然）
- **超出范围** — 值超过了允许的限制
- **无效格式** — 电子邮件、电话或其他格式化字段不正确
- **无效引用** — 引用了不存在的ID

## 服务器错误映射

当服务器返回以下内容时触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## 如何修复

### 1. 检查输入值

审查你提交的数据：
- **价格** — 应为有效数字
- **数量** — 应为正整数
- **电子邮件** — 必须为有效的电子邮件格式
- **ID** — 必须引用现有记录

### 2. 查找特定字段错误

错误可能指示哪个字段无效：
- 检查错误消息详情
- 更正特定字段
- 重试操作

### 3. 验证引用的数据是否存在

如果引用产品、客户或其他记录：
- 确保该项目存在于WooCommerce中
- 检查是否最近被删除
- 同步数据以刷新本地记录

### 4. 检查字段约束

WooCommerce可能有约束：
- 最大/最小值
- 必需的格式
- 选择字段的允许选项

### 5. 清除并重新输入

有时数据会损坏：
- 清除有问题的字段
- 从头重新输入值
- 避免从其他来源复制粘贴
