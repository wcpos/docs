# API02001: 无效凭据

## 这意味着什么

您输入的用户名或密码不正确。由于凭据与任何有效账户不匹配，服务器拒绝了登录尝试。

## 常见原因

- **用户名或密码拼写错误** — 检查是否开启了大写锁定或是否有多余空格
- **错误的账户** — 使用了不同网站的凭据
- **密码最近更改** — 密码已更新，但POS使用的是旧密码
- **账户已删除** — 用户账户不再存在

## 服务器错误映射

当服务器返回以下内容时，会触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT 身份验证插件 |

## 如何修复

### 1. 验证您的凭据

- 仔细检查您的用户名（通常是电子邮件或WordPress用户名）
- 验证密码是否正确
- 尝试直接登录WordPress管理后台以确认

### 2. 重置您的密码

如果您忘记了密码：
1. 前往您的WordPress登录页面
2. 点击“忘记您的密码？”
3. 输入您的电子邮件接收重置链接
4. 更新您的密码
5. 使用新密码登录POS

### 3. 检查用户账户

在WordPress管理后台：
- 前往 用户 → 所有用户
- 验证账户是否存在
- 检查其是否有适当的POS访问权限

### 4. 清除保存的凭据

在POS中：
- 完全注销
- 清除任何已保存/缓存的登录信息
- 重新输入凭据以获得新鲜登录
