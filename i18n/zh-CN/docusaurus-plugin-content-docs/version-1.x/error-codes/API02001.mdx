# API02001：无效凭据

## 这意味着什么

您输入的用户名或密码不正确。服务器拒绝了登录尝试，因为凭据与任何有效帐户都不匹配。

## 常见原因

- **用户名或密码输入错误** — 检查大写锁定或额外空格
- **错误的帐户** — 使用了不同站点的凭据
- **密码recently更改** — 密码已更新，但POS仍保存旧密码
- **帐户已删除** — 用户帐户不再存在

## 服务器错误映射

当服务器返回以下内容时触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT身份验证插件 |

## 如何修复

### 1. 验证您的凭据

- 仔细检查您的用户名（通常是电子邮件或WordPress用户名）
- 验证密码是否正确
- 尝试直接登录WordPress管理员以确认

### 2. 重置密码

如果您忘记了密码：
1. 转到WordPress登录页面
2. 点击"忘记密码？"
3. 输入您的电子邮件以接收重置链接
4. 更新密码
5. 使用新密码登录POS

### 3. 检查用户帐户

在WordPress管理员中：
- 转到"用户" → "所有用户"
- 验证帐户是否存在
- 检查是否有适当的POS访问权限

### 4. 清除保存的凭据

在POS中：
- 完全退出登录
- 清除任何保存/缓存的登录信息
- 重新输入凭据
