# API05002: 找不到 WCPOS 插件 {#api05002-wcpos-plugin-not-found}

## 这意味着什么 {#what-this-means}

WCPOS 插件未安装或未在您的 WordPress 网站上激活。POS 应用程序需要此插件与 WooCommerce 通信。

## 常见原因 {#common-causes}

- **插件未安装** — 从未安装该插件
- **插件已停用** — 已禁用该插件
- **插件已删除** — 插件文件已被移除
- **错误的网站** — 连接到没有 WCPOS 的网站

## 如何修复 {#how-to-fix}

### 1. 安装插件 {#1-install-the-plugin}

如果未安装：
1. 转到 WordPress 管理员 → 插件 → 添加新插件
2. 搜索 "WooCommerce POS"
3. 安装并激活插件

或从 [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/) 下载

### 2. 激活插件 {#2-activate-the-plugin}

如果已安装但未激活：
1. 转到插件 → 已安装的插件
2. 找到 "WooCommerce POS"
3. 点击 "激活"

### 3. 检查错误 {#3-check-for-errors}

如果插件无法激活：
- 检查错误日志中的 PHP 错误
- 验证 WooCommerce 是否已安装并激活
- 检查 PHP 版本要求
- 查找插件冲突

### 4. 验证 URL {#4-verify-the-url}

确保 POS 配置为连接到正确的 WordPress 网站：
- 检查 POS 设置中的 URL
- 验证其是否与您的 WordPress 安装匹配

### 5. 如有必要，重新安装 {#5-reinstall-if-necessary}

如果插件损坏：
1. 禁用并删除插件
2. 从 WordPress.org 重新安装
3. 激活插件
