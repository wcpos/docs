# API02004：用户未授权

## 含义

你已登录，但你的用户账户没有执行所请求操作的权限。这是一个授权问题（你能做什么），而不是认证问题（你是谁）。

## 常见原因

- **用户角色限制** — 你的WordPress角色不包含POS访问权限
- **POS访问已禁用** — 你的账户未被授予POS权限
- **功能限制** — 某些功能仅限特定角色
- **店铺限制** — 你可能无法访问特定店铺

## 服务器错误映射

当服务器返回以下内容时，触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `rest_cannot_view` | WordPress REST API |
| `woocommerce_rest_cannot_view` | WooCommerce REST API |

## 如何修复

### 1. 检查用户角色

在WordPress管理后台 → 用户：
1. 找到你的用户账户
2. 验证角色（例如，店铺管理员，管理员）
3. 确保角色包含WooCommerce权限

### 2. 启用POS访问

在WordPress管理后台 → WooCommerce → POS → 访问：
1. 找到用户或角色
2. 启用POS访问权限
3. 保存更改

### 3. 请求额外权限

联系你的店铺管理员：
- 为你的角色授予POS访问权限
- 为你分配具有适当权限的角色
- 启用你需要的特定功能

### 4. 检查店铺分配

如果使用多个店铺：
- 验证你被分配到正确的店铺
- 检查店铺特定权限

## 必需权限

不同操作需要不同的权限：
- **查看产品**：产品读取权限
- **创建订单**：创建/编辑订单权限
- **管理客户**：客户管理权限
- **访问报告**：查看报告权限
