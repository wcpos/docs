# DB03002：无效的数据类型

## 这意味着什么

数据类型与数据库预期的不匹配。例如，在需要数字的地方提供了文本。

## 常见原因

- **用户输入错误** — 在数值字段中输入文本
- **导入问题** — 导入的数据格式错误
- **数据损坏** — 数据传输过程中值被破坏
- **插件冲突** — 其他插件修改了数据类型

## 如何修复

### 1. 检查您的输入

检查您输入的数据：
- **价格** — 应为数字（例如 `19.99`）
- **数量** — 应为整数（例如 `5`）
- **ID** — 应为数字

### 2. 清除并重新输入

如果数据已损坏：
- 清空字段
- 手动重新输入值
- 避免从外部来源复制粘贴

### 3. 检查源数据

如果从 WooCommerce 同步：
- 在 WordPress 管理后台检查数据
- 查找格式不正确的字段
- 在源头修复数据

### 4. 重新同步数据

从服务器获取新的数据：
- 清除本地缓存
- 重新同步数据
- 检查问题是否解决

### 5. 检查插件冲突

如果使用其他 WooCommerce 插件：
- 它们可能以意外的方式修改数据
- 暂时禁用以测试
- 报告不兼容性

## 常见示例

| 字段 | 预期 | 无效 |
|-------|----------|---------|
| 价格 | `19.99` | `$19.99` |
| 数量 | `5` | `five` |
| SKU | `ABC123` | — (任何格式均可) |
| 产品ID | `42` | `product-42` |
