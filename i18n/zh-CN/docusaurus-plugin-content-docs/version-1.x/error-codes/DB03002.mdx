# DB03002: 无效的数据类型 {#db03002-invalid-data-type}

## 这意味着 {#what-this-means}

数据类型与数据库期望的类型不匹配。例如，在应填入数字的地方填入了文本。

## 常见原因 {#common-causes}

- **用户输入错误** — 在数字字段中输入文本
- **导入问题** — 导入的数据格式错误
- **数据损坏** — 在传输过程中值被损坏
- **插件冲突** — 其他插件修改了数据类型

## 如何修复 {#how-to-fix}

### 1. 检查输入 {#1-check-your-input}

审核您输入的数据：
- **价格** — 应为数字（例如，`19.99`）
- **数量** — 应为整数（例如，`5`）
- **ID** — 应为数字

### 2. 清除并重新输入 {#2-clear-and-re-enter}

如果数据被损坏：
- 清除字段
- 手动重新输入值
- 避免从外部来源复制粘贴

### 3. 检查源数据 {#3-check-source-data}

如果从 WooCommerce 同步：
- 检查 WordPress 管理中的数据
- 查找格式不正确的字段
- 在源头修复数据

### 4. 重新同步数据 {#4-re-sync-data}

从服务器获取新数据：
- 清除本地缓存
- 再次同步数据
- 检查问题是否解决

### 5. 检查插件冲突 {#5-check-for-plugin-conflicts}

如果使用其他 WooCommerce 插件：
- 它们可能以意想不到的方式修改数据
- 暂时禁用以进行测试
- 报告不兼容性

## 常见示例 {#common-examples}

| 字段 | 预期 | 无效 |
|------|------|------|
| 价格 | `19.99` | `$19.99` |
| 数量 | `5` | `five` |
| SKU | `ABC123` | — （任何格式均可） |
| 产品 ID | `42` | `product-42` |
