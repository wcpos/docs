# API06002：缺少API地址

## 这意味着什么

没有配置连接到您的WooCommerce商店的地址。POS需要知道您的商店位置。

## 常见原因

- **首次设置** — 从未输入地址
- **配置已清除** — 设置已重置或清空
- **应用数据已清除** — 应用程序数据被删除
- **迁移问题** — 更新或迁移过程中丢失地址

## 如何修复

### 1. 输入您的商店地址

在POS应用中：
1. 打开设置或登录界面
2. 找到URL/站点配置字段
3. 输入您的WordPress站点地址
4. 示例：`https://yourstore.com`

### 2. 查找您的WordPress地址

如果不确定地址：
1. 登录WordPress管理后台
2. 转到设置 → 常规
3. 查看"WordPress地址（URL）"
4. 在POS中使用此地址

### 3. 检查已存储的设置

如果之前已配置地址：
- 检查应用数据是否意外清除
- 查找设置备份/导出选项
- 必要时重新输入地址

### 4. 验证地址正确性

保存前：
1. 在浏览器中测试地址
2. 确保加载您的WordPress站点
3. 包括 `https://` 或 `http://`

## 设置检查清单

首次配置WCPOS时：

1. ☐ 输入您的WordPress站点地址
2. ☐ 验证WCPOS插件已安装并激活
3. ☐ 使用您的WordPress凭证登录
4. ☐ 允许初始数据同步完成
