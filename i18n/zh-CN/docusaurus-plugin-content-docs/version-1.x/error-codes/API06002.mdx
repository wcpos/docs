# API06002: 缺少 API URL

## 这意味着什么

没有配置用于连接到您的 WooCommerce 商店的 URL。POS 需要知道您的商店位于何处。

## 常见原因

- **首次设置** — 从未输入 URL
- **配置已清除** — 设置已重置或清除
- **应用数据已清除** — 应用程序数据已被删除
- **迁移问题** — 更新或迁移过程中丢失 URL

## 如何修复

### 1. 输入您的商店 URL

在 POS 应用中：
1. 打开设置或登录屏幕
2. 找到 URL/网站配置字段
3. 输入您的 WordPress 网站 URL
4. 示例：`https://yourstore.com`

### 2. 查找您的 WordPress URL

如果您不确定您的 URL：
1. 登录到 WordPress 管理后台
2. 转到 设置 → 一般
3. 查看“WordPress 地址 (URL)”
4. 在 POS 中使用此 URL

### 3. 检查存储的设置

如果 URL 之前已配置：
- 检查应用数据是否意外清除
- 查找设置备份/导出选项
- 如有需要，重新输入 URL

### 4. 验证 URL 是否正确

在保存之前：
1. 在浏览器中测试 URL
2. 确保它可以加载您的 WordPress 网站
3. 包含 `https://` 或 `http://`

## 设置清单

首次配置 WCPOS 时：

1. ☐ 输入您的 WordPress 网站 URL
2. ☐ 验证 WCPOS 插件已安装并处于活动状态
3. ☐ 使用您的 WordPress 凭据登录
4. ☐ 允许初始数据同步完成
