# DB02001: 重复记录 {#db02001-duplicate-record}

## 这意味着什么 {#what-this-means}

数据库尝试插入一个已存在的 ID 的记录。每条记录必须有一个唯一的标识符。

## 常见原因 {#common-causes}

- **同步冲突** — 相同数据同步了两次
- **竞态条件** — 多个操作创建相同记录
- **数据损坏** — ID 被损坏或重复
- **导入问题** — 导入已存在的数据

## 如何修复 {#how-to-fix}

### 1. 刷新数据 {#1-refresh-data}

从服务器重新同步：
- 拉取最新数据
- 让同步过程解决冲突
- 更新的数据应优先使用

### 2. 重试操作 {#2-retry-the-operation}

如果创建新记录：
- 系统可能已经创建了它
- 检查记录是否存在
- 避免多次点击提交

### 3. 清除本地缓存 {#3-clear-local-cache}

重置本地数据：
1. 清除本地数据库/缓存
2. 重新登录
3. 新鲜同步数据

### 4. 检查 WooCommerce 中的重复项 {#4-check-for-duplicates-in-woocommerce}

如果问题依然存在：
- 检查 WooCommerce 中的重复条目
- 查找具有相同 ID 的产品/订单
- 在管理后台清理任何重复项

### 5. 报告持续问题 {#5-report-persistent-issues}

如果频繁发生：
- 记录导致此问题的操作
- 检查模式
- 向支持部门报告详细信息

## 为什么这很重要 {#why-this-matters}

重复的 ID 会导致：
- 不确定哪个记录是正确的
- 数据覆盖
- 同步问题

该错误可防止数据损坏。
