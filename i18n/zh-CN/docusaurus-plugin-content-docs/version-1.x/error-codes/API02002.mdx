# API02002: Token Expired

## 这意味着什么 {#what-this-means}

您的身份验证会话已过期。出于安全原因，登录会话不会永久持续。POS需要刷新您的身份验证，或者您可能需要重新登录。

## 常见原因 {#common-causes}

- **长时间不活动** — 您在较长时间内没有使用POS
- **服务器令牌生命周期** — 服务器的令牌过期时间设置得非常短
- **服务器时间不匹配** — 服务器和设备时钟不同步

## 服务器错误映射 {#server-error-mapping}

当服务器返回以下代码时，会触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `jwt_auth_expired_token` | JWT 认证插件 |

## 如何修复 {#how-to-fix}

### 1. 重新登录 {#1-log-in-again}

最简单的解决方案是注销并重新登录：
1. 从POS注销
2. 再次输入您的凭据
3. 这会创建一个新的会话

### 2. 检查令牌刷新 {#2-check-token-refresh}

POS应该自动刷新令牌。如果未能正常工作：
- 检查您的互联网连接
- 验证服务器是否可访问
- 如果刷新失败，请查看 [API02007](./API02007)

### 3. 检查服务器时间 {#3-check-server-time}

令牌过期取决于准确的时间：
- 确保您的服务器时钟是准确的
- 检查您的设备时间是否准确
- 在两者上使用自动时间同步

### 4. 审查令牌设置 {#4-review-token-settings}

如果令牌过期得太快，您可能需要调整服务器设置（请咨询您的托管服务提供商或WordPress管理员）。
