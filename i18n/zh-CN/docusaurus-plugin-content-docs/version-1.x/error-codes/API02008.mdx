# API02008：刷新令牌无效

## 这意味着什么

存储在POS中的刷新令牌无法被服务器识别。刷新令牌用于获取新的访问令牌，而无需重新登录。

## 常见原因

- **令牌被吊销** — 管理员使你的会话失效
- **服务器密钥已更改** — WordPress安全密钥已更新
- **数据损坏** — 令牌在存储过程中被损坏
- **多次登录** — 在其他地方登录可能导致此令牌失效

## 如何修复

### 1. 重新登录

由于刷新令牌无效，你需要重新进行身份验证：
1. 从POS中注销
2. 输入你的用户名和密码
3. 这将生成新的有效令牌

### 2. 检查已撤销的会话

如果管理员撤销了会话：
- 这是正常的安全做法
- 只需重新登录
- 你的新会话将正常工作

### 3. 检查服务器变更

如果这影响所有用户：
- WordPress盐/密钥可能已更改
- JWT插件设置可能已修改
- 联系你的服务器管理员

### 4. 清除存储的数据

如果你仍然遇到问题：
- 清除POS应用程序数据
- 移除缓存的凭据
- 重新开始干净的登录
