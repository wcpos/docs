# API02008: 刷新令牌无效 {#api02008-refresh-token-invalid}

## 这意味着什么 {#what-this-means}

POS存储的刷新令牌未被服务器识别。刷新令牌用于在不需要您重新登录的情况下获取新的访问令牌。

## 常见原因 {#common-causes}

- **令牌已撤销** — 管理员使您的会话失效
- **服务器密钥已更改** — WordPress安全密钥已更新
- **数据损坏** — 令牌在存储中被损坏
- **多次登录** — 在其他地方登录可能使此令牌失效

## 如何修复 {#how-to-fix}

### 1. 再次登录 {#1-log-in-again}

由于刷新令牌无效，您需要再次进行身份验证：
1. 从POS注销
2. 输入您的用户名和密码
3. 这将生成新的有效令牌

### 2. 检查是否有已撤销的会话 {#2-check-for-revoked-sessions}

如果管理员撤销了会话：
- 这是正常的安全做法
- 只需再次登录
- 您的新会话将正常工作

### 3. 检查服务器更改 {#3-check-server-changes}

如果这影响所有用户：
- WordPress盐值/密钥可能已更改
- JWT插件设置可能已被修改
- 联系您的服务器管理员

### 4. 清除存储的数据 {#4-clear-stored-data}

如果您继续遇到问题：
- 清除POS应用程序数据
- 删除缓存的凭据
- 从干净的登录开始
