# API01002：连接被拒绝

## 这意味着什么

服务器主动拒绝了连接尝试。这与超时不同 — 服务器已响应，但说"不"。

## 常见原因

- **服务器已关闭** — Web服务器（Apache/Nginx）可能未运行
- **错误端口** — 尝试在错误的端口上连接
- **防火墙阻止** — 防火墙阻止了连接
- **IP限制** — 您的IP地址可能已被阻止

## 如何修复

### 1. 验证服务器是否运行

直接在浏览器中尝试访问您的WordPress网站。如果无法加载，请联系您的托管提供商。

### 2. 检查URL配置

确保使用正确的URL：
- 验证 `http://` 与 `https://`
- 检查域名是否有拼写错误
- 如果使用非标准端口，请确认正确的端口

### 3. 检查防火墙设置

如果您可以访问服务器：
- 验证端口80（HTTP）或443（HTTPS）是否开放
- 检查您的IP是否已被列入白名单
- 查看可能阻止API访问的任何安全插件

### 4. 联系您的托管提供商

他们可以检查：
- 服务器状态和日志
- 防火墙配置
- 您的IP是否已被阻止
