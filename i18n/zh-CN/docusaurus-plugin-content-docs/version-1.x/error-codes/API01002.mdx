# API01002: 连接被拒绝 {#api01002-connection-refused}

## 这意味着什么 {#what-this-means}

服务器主动拒绝了连接尝试。这与超时不同——服务器响应了，但表示“否”。

## 常见原因 {#common-causes}

- **服务器宕机** — 网络服务器（Apache/Nginx）可能未运行
- **错误端口** — 尝试在错误端口上连接
- **防火墙阻止** — 防火墙阻止了连接
- **IP限制** — 您的IP地址可能被阻止

## 如何修复 {#how-to-fix}

### 1. 验证服务器是否在运行 {#1-verify-the-server-is-running}

尝试在浏览器中直接访问您的WordPress网站。如果无法加载，请联系您的主机提供商。

### 2. 检查URL配置 {#2-check-the-url-configuration}

确保您使用的是正确的URL：
- 验证`http://`与`https://`
- 检查域名中的拼写错误
- 如果使用非标准端口，请确认正确的端口

### 3. 检查防火墙设置 {#3-check-firewall-settings}

如果您可以访问您的服务器：
- 验证端口80（HTTP）或443（HTTPS）是否打开
- 检查您的IP是否在白名单中
- 审核可能会阻止API访问的任何安全插件

### 4. 联系您的主机提供商 {#4-contact-your-hosting-provider}

他们可以检查：
- 服务器状态和日志
- 防火墙配置
- 您的IP是否被阻止
