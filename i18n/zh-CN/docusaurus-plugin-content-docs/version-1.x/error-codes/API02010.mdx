# API02010：需要身份验证

## 这意味着什么

您正在尝试执行的操作需要身份验证，但您目前尚未登录。POS需要有效的凭据才能访问此资源。

## 常见原因

- **未登录** — 您尚未进行身份验证
- **会话已清除** — 您的会话已被清除或过期
- **访问受保护的资源** — 该资源需要身份验证
- **应用数据已清除** — 存储的凭据已被删除

## 服务器错误映射

当服务器返回以下内容时，将触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `rest_login_required` | WordPress REST API |
| `jwt_auth_no_auth_header` | JWT身份验证插件 |
| HTTP 401 | 任何服务器响应（回退） |

## 如何修复

### 1. 登录

如果您尚未登录：
1. 打开POS登录界面
2. 输入您的WordPress凭据
3. 完成身份验证过程

### 2. 检查会话状态

如果您认为已登录：
- 您的会话可能已过期
- 查看[API02002](./API02002)（令牌过期）了解更多详细信息
- 重新登录以恢复访问权限

### 3. 验证服务器配置

确保API端点配置正确：
- WooCommerce REST API应已启用
- WCPOS插件应已激活
- 身份验证端点应可访问

### 4. 检查浏览器/应用程序问题

如果您意外注销：
- 清除浏览器缓存（如果使用Web版本）
- 检查Cookie/本地存储是否未被阻止
- 验证应用程序是否有权存储数据

## 需要身份验证的操作

大多数POS操作需要身份验证：
- 查看产品和客户
- 创建和编辑订单
- 处理付款
- 访问报告

仅初始登录界面无需身份验证可访问。
