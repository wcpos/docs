# API03001：无效的请求格式

## 这意味着什么

发送到服务器的请求不是预期的格式。服务器无法理解POS系统的请求，因为请求结构不正确。

## 常见原因

- **数据损坏** — 数据在发送前已损坏
- **软件错误** — POS应用程序中的问题
- **代理干扰** — 代理或防火墙修改了请求
- **字符编码问题** — 特殊字符未正确编码

## 服务器错误映射

当服务器返回以下内容时触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| HTTP 400 | 任何服务器响应（回退） |

## 如何修复

### 1. 重试操作

有时这是一次性故障：
- 等待片刻后重试
- 刷新POS系统并重试

### 2. 检查特殊字符

如果您正在输入包含特殊字符的数据：
- 尝试删除表情符号或unusual符号
- 对于产品名称等使用标准字符

### 3. 更新POS系统

确保运行最新版本：
- 检查应用程序更新
- 更新服务器上的WCPOS插件

### 4. 检查网络配置

如果您在代理后面：
- 验证代理是否未修改请求
- 检查防火墙规则
- 尝试从不同网络访问

### 5. 报告问题

如果此错误持续发生：
- 记录触发错误的操作
- 检查浏览器控制台的详细信息（Web版本）
- 在[GitHub](https://github.com/wcpos)上报告，并提供重现步骤
