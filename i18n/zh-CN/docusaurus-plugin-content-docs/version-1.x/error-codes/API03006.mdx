# API03006：不支持的方法

## 这意味着什么

使用的 HTTP 方法（GET、POST、PUT、DELETE 等）不被此终端节点支持。服务器不接受对该 URL 的这种类型的请求。

## 常见原因

- **版本不匹配** — POS 和插件版本不兼容
- **终端节点已移除** — API 终端节点已被弃用
- **服务器配置** — Web 服务器阻止某些 HTTP 方法
- **插件冲突** — 其他插件修改了 REST API 行为

## 如何修复

### 1. 更新所有组件

确保兼容性：
- 更新 WCPOS WordPress 插件
- 更新 POS 应用程序
- 检查发行说明中的重大变更

### 2. 检查服务器配置

一些服务器会阻止某些 HTTP 方法：
- 确保允许 PUT 和 DELETE 方法
- 检查 `.htaccess` 中的方法限制
- 检查 Nginx 配置

### 3. 验证 REST API 访问

测试 WordPress REST API：
1. 在浏览器中访问 `https://yoursite.com/wp-json/`
2. 应返回 JSON 数据
3. 如果没有，REST API 可能已禁用或被阻止

### 4. 检查插件冲突

暂时禁用其他插件：
- 安全插件可能会阻止方法
- 其他 REST API 插件可能会造成冲突
- 逐一重新启用以找出问题

### 5. 检查主机限制

某些主机限制 HTTP 方法：
- 联系主机支持
- 请求他们启用所有标准方法
- 如果限制过多，考虑更换主机
