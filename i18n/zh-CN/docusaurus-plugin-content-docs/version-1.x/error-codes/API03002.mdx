# API03002: 缺少必需参数 {#api03002-missing-required-parameters}

## 这意味着什么 {#what-this-means}

请求缺少服务器完成操作所需的数据。请求中未包含必填字段。

## 常见原因 {#common-causes}

- **表单不完整** — 必填字段未填写
- **数据未保存** — 表单数据未正确捕获
- **版本不匹配** — 插件期待的参数与 POS 发送的不同
- **自定义字段** — 缺少必填自定义字段

## 如何修复 {#how-to-fix}

### 1. 完成必填字段 {#1-complete-required-fields}

检查所有必需信息是否已填写：
- 客户详情（如果需要）
- 产品信息
- 订单详情

### 2. 刷新并重试 {#2-refresh-and-retry}

表单状态可能不完整：
1. 刷新该 POS
2. 重新输入所需信息
3. 再次尝试该操作

### 3. 检查插件版本 {#3-check-plugin-versions}

确保兼容性：
- 更新 WCPOS 插件
- 更新 POS 应用程序
- 二者应在兼容版本上

### 4. 检查必填字段设置 {#4-check-required-field-settings}

在 WooCommerce 中，一些字段可能被标记为必填：
- 审核结账字段要求
- 检查自定义字段配置
- 如有需要，调整要求

### 5. 检查自定义集成 {#5-check-custom-integrations}

如果使用自定义插件或集成：
- 可能需要额外字段
- 查看插件文档
- 检查冲突
