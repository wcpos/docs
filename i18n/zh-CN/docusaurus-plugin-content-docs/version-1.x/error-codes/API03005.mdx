# API03005: 超过速率限制

## 这意味着什么

您在短时间内向服务器发送了过多的请求。速率限制保护服务器不被压垮。

## 常见原因

- **快速同步操作** — 多个设备同时同步
- **频繁刷新** — 数据刷新得太频繁
- **安全插件** — 某个插件正在强制执行请求限制
- **托管限制** — 您的托管计划具有 API 速率限制

## 服务器错误映射

当服务器返回以下内容时，会触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| HTTP 429 | 任何服务器响应（后备） |

## 如何解决

### 1. 等待并重试

速率限制是暂时的：
- 等待 1-5 分钟
- POS 将自动重试
- 避免手动反复刷新

### 2. 减少并发操作

如果使用多个 POS 终端：
- 在设备之间错开初始同步
- 避免同时刷新所有设备
- 让一个完成后再开始另一个

### 3. 检查安全插件

像 Wordfence 这样的插件可能会阻止快速请求：
- 将 POS 应用列入白名单
- 将您的 POS 设备 IP 列入白名单
- 调整速率限制阈值

### 4. 审查托管限制

某些主机限制 API 请求：
- 检查您的托管计划细节
- 如果限制太低，考虑升级
- 联系支持以增加限制

### 5. 优化同步频率

在 POS 设置中：
- 调整自动刷新间隔
- 在适当的时候使用手动同步
- 避免不必要的数据刷新
