# API03005: 速率限制已超出 {#api03005-rate-limit-exceeded}

## 这意味着什么 {#what-this-means}

您在短时间内向服务器发送了太多请求。速率限制是为了保护服务器免于被压垮。

## 常见原因 {#common-causes}

- **快速同步操作** — 多个设备同时进行同步
- **激进刷新** — 刷新数据过于频繁
- **安全插件** — 插件正在强制执行请求限制
- **托管限制** — 您的托管计划有 API 速率限制

## 服务器错误映射 {#server-error-mapping}

当服务器返回以下响应时，会触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| HTTP 429 | 任何服务器响应（后备） |

## 如何修复 {#how-to-fix}

### 1. 等待并重试 {#1-wait-and-retry}

速率限制是暂时的：
- 等待 1-5 分钟
- POS 将自动重试
- 避免手动重复刷新

### 2. 减少并发操作 {#2-reduce-concurrent-operations}

如果使用多个 POS 终端：
- 在设备之间错开初始同步
- 避免同时刷新所有设备
- 让一个设备完成后再开始另一个

### 3. 检查安全插件 {#3-check-security-plugins}

像 Wordfence 这样的插件可能阻止快速请求：
- 将该 POS 应用程序加入白名单
- 将您的 POS 设备 IP 加入白名单
- 调整速率限制阈值

### 4. 审查托管限制 {#4-review-hosting-limits}

某些主机限制 API 请求：
- 检查您的托管计划详细信息
- 如果限制过低，考虑升级
- 联系支持以增加限制

### 5. 优化同步频率 {#5-optimise-sync-frequency}

在 POS 设置中：
- 调整自动刷新间隔
- 在适当的时候使用手动同步
- 避免不必要的数据刷新
