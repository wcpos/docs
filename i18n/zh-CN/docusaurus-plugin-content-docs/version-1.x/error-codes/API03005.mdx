# API03005：超出请求频率限制

## 含义

您在短时间内向服务器发送了过多的请求。频率限制是为了保护服务器免于过载。

## 常见原因

- **快速同步操作** — 多个设备同时同步
- **频繁刷新** — 过于频繁地刷新数据
- **安全插件** — 插件正在执行请求限制
- **主机限制** — 您的主机方案有API请求频率限制

## 服务器错误映射

当服务器返回以下内容时触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| HTTP 429 | 任何服务器响应（备用） |

## 解决方法

### 1. 等待并重试

频率限制是临时的：
- 等待1-5分钟
- POS将自动重试
- 避免重复手动刷新

### 2. 减少并发操作

如果使用多个POS终端：
- 在设备间错开初始同步
- 避免同时刷新所有设备
- 等待一个完成后再开始另一个

### 3. 检查安全插件

像Wordfence这样的插件可能会阻止快速请求：
- 将POS应用程序加入白名单
- 将您的POS设备IP加入白名单
- 调整频率限制阈值

### 4. 查看主机限制

一些主机限制API请求：
- 检查您的主机方案详情
- 考虑升级如果限制太低
- 联系支持询问增加限制

### 5. 优化同步频率

在POS设置中：
- 调整自动刷新间隔
- 在适当时使用手动同步
- 避免不必要的数据刷新
