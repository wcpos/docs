# PY01002: 资金不足 {#py01002-insufficient-funds}

## 这意味着什么 {#what-this-means}

支付被拒绝，因为账户中没有足够的钱来支付交易。

## 常见原因 {#common-causes}

- **低账户余额** — 可用资金不足
- **待处理交易** — 其他待处理交易减少了可用余额
- **信用额度已达上限** — 对于信用卡，已达到额度上限
- **资金被保留** — 资金因其他交易而被保留

## 如何解决 {#how-to-fix}

### 1. 使用不同的支付方法 {#1-use-a-different-payment-method}

最快的解决方案：
- 尝试不同的卡
- 使用现金
- 在多种方式之间分摊付款（如果支持）

### 2. 减少订单金额 {#2-reduce-the-order-amount}

如果可以部分付款：
- 删除一些商品
- 应用折扣或优惠券
- 完成较小的交易

### 3. 客户操作 {#3-customer-actions}

客户可能需要：
- 向账户转账
- 等待待处理交易清算
- 偿还信用卡余额
- 请求增加信用额度

### 4. 拆分交易 {#4-split-the-transaction}

如果您的POS支持分拆支付：
- 用一张卡支付部分费用
- 用另一种方式支付剩余金额

## 隐私说明 {#privacy-note}

与客户沟通时：
- 对具体拒绝原因要谨慎
- 仅说明“卡片被拒绝”
- 允许他们私下尝试其他支付方式
