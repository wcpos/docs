# DB01002：查询超时

## 含义

数据库操作耗时过长。本地数据库查询超过了最大允许时间。

## 常见原因

- **大型数据集** — 产品或记录数量非常多
- **复杂查询** — 涉及多个条件的查询
- **设备性能** — 设备运行缓慢
- **数据库碎片** — 需要优化数据库

## 如何解决

### 1. 等待并重试

查询可能在重试时成功：
- 关闭并重新打开POS
- 尝试再次执行操作
- 避免同时进行其他密集型操作

### 2. 关闭其他应用程序

释放设备资源：
- 关闭未使用的浏览器标签
- 关闭其他应用程序
- 必要时重启设备

### 3. 缩小数据范围

如果在搜索：
- 使用更具体的搜索词
- 应用筛选器缩小结果范围
- 将大型操作拆分为较小的操作

### 4. 优化数据库

清除并重新同步：
1. 清除本地数据（如果有设置选项）
2. 重新登录
3. 重新同步数据

### 5. 检查设备性能

确保您的设备满足要求：
- 足够的内存
- 足够的处理器速度
- 优先使用SSD而非HDD

## 大型目录

如果您有非常大的产品目录：
- 初始同步需要更长时间
- 考虑在POS中使用筛选器
- 某些操作自然需要更多时间
- 联系支持人员获取优化建议
