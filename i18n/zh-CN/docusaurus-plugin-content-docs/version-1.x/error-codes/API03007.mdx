# API03007：请求队列已满

## 这意味着什么

POS系统有太多等待发送的挂起请求。内部队列已满，在一些请求完成之前无法接受更多请求。

## 常见原因

- **网络缓慢** — 由于连接慢导致请求积压
- **服务器无响应** — 服务器响应时间过长
- **快速操作** — 执行的操作比处理速度快
- **同步过载** — 大型同步操作阻塞队列

## 如何解决

### 1. 等待队列清空

给挂起的请求时间完成：
- 等待30-60秒
- 观察同步指示器是否完成
- 避免在清空之前执行额外操作

### 2. 检查网络连接

缓慢的连接会导致积压：
- 测试网络速度
- 尝试使用更快的连接
- 移近WiFi路由器

### 3. 检查服务器响应时间

如果服务器很慢：
- 在浏览器中测试访问站点
- 如果站点缓慢，请联系主机提供商
- 等待服务器负载减轻

### 4. 重启POS

如果队列卡住：
1. 关闭POS应用程序
2. 稍等片刻
3. 重新打开POS
4. 可能需要重新发起挂起的请求

### 5. 避免快速操作

在同步或网络缓慢期间：
- 等待一个操作完成后再开始另一个
- 不要重复点击按钮
- 对网络操作保持耐心

## 离线行为

当队列因离线条件而填满时：
- POS将保留挂起的请求直到恢复在线
- 你仍可浏览缓存的产品和客户
- 可以开始新订单并向购物车添加商品
- 完成订单需要恢复连接
