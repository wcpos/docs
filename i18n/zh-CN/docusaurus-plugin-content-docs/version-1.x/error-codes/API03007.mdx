# API03007：请求队列已满

## 含义

POS有太多待发送的请求。内部队列已满，无法接受更多请求，直到一些请求完成。

## 常见原因

- **网络缓慢** — 由于连接缓慢，请求堆积
- **服务器无响应** — 服务器响应时间过长
- **快速操作** — 执行许多操作的速度超过处理速度
- **同步过载** — 大型同步操作阻塞队列

## 如何修复

### 1. 等待队列清除

给待处理请求一些时间完成：
- 等待30-60秒
- 观察同步指示灯完成
- 避免追加操作直到清除

### 2. 检查网络连接

慢速连接会导致积压：
- 测试你的互联网速度
- 尝试更快的连接
- 移动更靠近WiFi路由器

### 3. 检查服务器响应时间

如果服务器响应慢：
- 在浏览器中测试访问你的网站
- 如果网站慢，联系主机服务提供商
- 等待服务器负载减少

### 4. 重启POS

如果队列卡住：
1. 关闭POS应用程序
2. 等待片刻
3. 重新打开POS
4. 待处理请求可能需要重新发起

### 5. 避免快速操作

在同步或缓慢期间：
- 等待一个操作完成再开始另一个
- 不要反复点击按钮
- 对网络操作保持耐心

## 离线行为

当队列因离线条件而填满时：
- POS将持有待处理请求，直到重新上线
- 你仍然可以浏览缓存的产品和客户
- 你可以开始新订单并将商品添加到购物车
- 完成订单需要恢复连接
