# API02003: 令牌无效

## 这意味着什么

所使用的身份验证令牌未被服务器识别。这与过期令牌不同——此令牌从未有效或已被撤销。

## 常见原因

- **令牌被撤销** — 管理员撤销了会话
- **令牌损坏** — 存储或传输过程中数据损坏
- **服务器重置** — 服务器密钥更改，导致所有令牌无效
- **错误的服务器** — 来自不同 WordPress 安装的令牌

## 服务器错误映射

当服务器返回时，会触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT 身份验证插件 |

## 如何修复

### 1. 再次登录

清除您的会话并重新进行身份验证：
1. 完全注销
2. 关闭并重新打开 POS
3. 使用您的凭据登录

### 2. 清除应用数据

如果注销无效：
- 清除 POS 应用缓存/数据
- 在网页上：清除网站的浏览器 Cookies 和本地存储
- 在桌面上：检查应用设置是否有“清除数据”选项

### 3. 检查服务器配置

如果问题影响所有用户：
- 检查 WordPress 盐值是否已更改
- 验证 JWT 或身份验证插件的设置
- 查看近期的服务器更改

### 4. 确保您连接到正确的服务器

确保 POS 配置为连接到正确的 WordPress 站点，尤其是在您有多个安装的情况下。
