# API02003：令牌无效

## 这意味着什么

正在使用的身份验证令牌未被服务器识别。这与过期令牌不同 — 此令牌从未有效或已被撤销。

## 常见原因

- **令牌被撤销** — 管理员撤销了会话
- **令牌损坏** — 存储或传输过程中的数据损坏
- **服务器重置** — 服务器密钥发生变化，使所有令牌失效
- **错误的服务器** — 来自不同WordPress安装的令牌

## 服务器错误映射

当服务器返回以下内容时触发此错误代码：

| 服务器代码 | 来源 |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT身份验证插件 |

## 如何修复

### 1. 重新登录

清除会话并重新验证：
1. 完全退出登录
2. 关闭并重新打开POS
3. 使用您的凭据登录

### 2. 清除应用程序数据

如果注销不起作用：
- 清除POS应用缓存/数据
- 在网页上：清除网站的浏览器Cookie和本地存储
- 在桌面端：检查应用设置中的"清除数据"选项

### 3. 检查服务器配置

如果问题影响所有用户：
- 检查WordPress密钥是否已更改
- 验证JWT或身份验证插件设置
- 审查最近的服务器更改

### 4. 验证您连接的服务器正确

确保POS配置连接到正确的WordPress站点，尤其是在有多个安装的情况下。
