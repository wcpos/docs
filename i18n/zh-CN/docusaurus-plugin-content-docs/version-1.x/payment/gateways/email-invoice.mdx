# 电子邮件发票网关 {#email-invoice-gateway}

电子邮件发票网关允许直接从 WCPOS 通过电子邮件向客户发送付款发票。适用于客户希望稍后付款，或需要使用您的网上店铺付款方式在线完成付款的情况。

## 功能 {#features}

  </FeatureCard>
  <FeatureCard icon="users" title="客户管理">

  </FeatureCard>

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="安装电子邮件发票网关">

从 `WP Admin > POS > 设置 > 扩展` 安装，或从 [GitHub 发布页面](https://github.com/wcpos/email-invoice-gateway/releases)下载最新的**插件 zip 资产**（不是 GitHub 源代码 zip 或 tarball），并通过 `插件 > 新增 > 上传插件` 上传。

  </StepCard>
  <StepCard number="2" title="在 WCPOS 中启用" isLast>

1. 前往 `WP Admin > POS > 设置 > 结账`
2. 在列表中找到 **WCPOS 发票支付网关**
3. 启用该网关以便在 POS 中使用
4. 保存设置

:::note
此网关不需要额外的 API 密钥或外部服务配置。它使用现有的 WordPress 电子邮件系统和 WooCommerce 支付方式。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### 发送发票 {#sending-invoices}

1. **添加商品**：在 POS 中将产品添加到购物车
2. **选择网关**：选择“电子邮件发票”作为付款方式
3. **电子邮件地址**： 
   - 对于现有客户：电子邮件会根据客户数据自动填充
   - 对于访客订单：手动输入客户的电子邮件地址
4. **可选**：勾选“保存到账单地址”以更新客户的账单信息
5. **发送发票**：完成流程以发送发票电子邮件

### 客户体验 {#customer-experience}

发送发票时，客户会收到一封包含以下内容的电子邮件：

- **订单详情**：商品、数量和价格的完整明细
- **订单总额**：应付的最终金额，包括税费和费用
- **付款链接**：用于在线完成付款的直接链接
- **订单信息**：订单号和参考详情

### 完成付款 {#payment-completion}

客户可以通过两种方式完成付款：

1. **在线付款**：点击电子邮件中的“支付此订单”链接
2. **返回店内付款**：返回店内使用其他 POS 付款方式完成付款

“支付此订单”链接会将客户带到网站上的专用付款页面，他们可以在该页面使用已启用的任何网页付款网关（Stripe、PayPal 等）。

## 电子邮件管理 {#email-management}

### 现有客户 {#existing-customers}

- 电子邮件地址会自动从客户资料中检索
- 客户账单信息会被保留
- 订单历史记录保存在客户账户下

### 访客客户 {#guest-customers}

- 为一次性客户手动输入电子邮件地址
- 可选择将电子邮件保存到账单地址，便于将来参考
- 访客订单会在 WooCommerce 中正确跟踪

### 电子邮件模板 {#email-templates}

该网关使用 WooCommerce 的标准电子邮件模板：

- **可自定义**：通过 WooCommerce 电子邮件设置修改邮件外观
- **品牌化**：包含店铺徽标和品牌元素
- **专业**：简洁、专业的发票格式
- **移动设备友好**：适用于所有设备的响应式设计

## 使用场景 {#use-cases}

### 适用场景 {#perfect-for}

- **B2B 销售**：需要通过其会计系统处理付款的企业客户
- **大额订单**：需要审批或处理时间的高价值交易
- **远程客户**：离开店铺后需要完成付款的客户
- **账期客户**：已有约定付款条款的常客
- **报价转订单**：将报价或估算转换为可付款的发票

## 故障排查 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="电子邮件未发送">

  - 检查 WordPress 电子邮件配置
  - 如果使用自定义电子邮件服务，请验证 SMTP 设置
  - 使用 WordPress 电子邮件测试插件进行测试
  - 检查测试电子邮件是否进入垃圾邮件文件夹

  </AccordionItem>

  <AccordionItem question="付款链接无法使用">

  - 确保 WooCommerce 已正确配置
  - 验证至少启用了一个用于网页结账的支付网关
  - 检查 SSL 证书是否已正确安装
  - 确认订单状态允许付款

  </AccordionItem>

  <AccordionItem question="客户无法完成付款">

  - 验证网页支付网关是否已启用并配置完成
  - 检查订单是否未过期或未取消
  - 确保客户的支付方式限额充足
  - 自行测试付款流程

  </AccordionItem>
</Accordion>

### 电子邮件发送 {#email-delivery}

为确保电子邮件可靠送达，请考虑：

- **SMTP 服务**：使用 SendGrid、Mailgun 或 Amazon SES 等服务
- **电子邮件插件**：安装 WordPress SMTP 插件以提升送达率
- **域名身份验证**：设置 SPF、DKIM 和 DMARC 记录
- **监控**：跟踪电子邮件送达率和打开率
