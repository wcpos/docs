# SumUp 终端网关 {#sumup-terminal-gateway}

SumUp 终端网关使您能够直接在 WCPOS 中使用 SumUp 卡读取器接受卡支付。此网关与 SumUp 的支付处理系统无缝集成，适用于面对面的交易。

## 功能 {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="硬件集成">
    通过互联网连接将 SumUp 卡读取器连接到您的 POS 系统
  </FeatureCard>
  <FeatureCard icon="bolt" title="实时处理">
    即时支付处理和订单完成
  </FeatureCard>
  <FeatureCard icon="shield" title="安全交易">
    通过 SumUp 进行符合 PCI 标准的支付处理
  </FeatureCard>
  <FeatureCard icon="link" title="简单配对">
    通过配对代码进行简单设备配对过程
  </FeatureCard>
</FeatureGrid>

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="下载和安装">

1. 访问 [SumUp Terminal for WooCommerce 发布页面](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. 下载最新的 **sumup-terminal-for-woocommerce.zip** 文件
3. 在您的 WordPress 管理后台，转到 `插件 > 添加新 > 上传插件`
4. 上传 zip 文件并激活插件

  </StepCard>
  <StepCard number="2" title="配置 SumUp 设置">

1. 导航到 `WP 管理 > WooCommerce > 设置 > 支付`
2. 在支付方式列表中找到 **SumUp 终端**
3. 点击 **SumUp 终端** 以访问设置
4. 输入您的 **SumUp API 密钥**（可以在您的 SumUp 商户仪表板上获得）
5. 保存设置

:::note
您无需在 WooCommerce 设置中启用 SumUp 终端网关。它将在后面的步骤中特别为 POS 启用。
:::

  </StepCard>
  <StepCard number="3" title="配对您的 SumUp 终端">

1. 在同一设置页面上，找到 **配对读取器** 部分
2. 在您的 SumUp 设备上，导航到配对屏幕以显示配对代码
3. 输入您在 SumUp 设备上显示的配对代码
4. 点击 **"配对读取器"** 以建立连接
5. 等待确认读取器已成功配对

:::warning 注意
在您处理支付之前，读取器必须成功配对。确保在继续之前完成配对过程。
:::

  </StepCard>
  <StepCard number="4" title="在 WCPOS 中启用" isLast>

1. 转到 `WP 管理 > POS > 设置 > 结账`
2. 在列表中找到 **SumUp 终端** 网关
3. 启用该网关以供 POS 使用
4. 保存您的设置

  </StepCard>
</Steps>

## 使用 {#usage}

### 处理支付 {#processing-payments}

1. **添加商品**：在 POS 中将商品添加到购物车
2. **选择网关**：选择 "SumUp 终端" 作为支付方式
3. **开始支付**：点击以在您的 SumUp 设备上启动新支付
4. **客户支付**：客户在 SumUp 终端上完成支付
5. **自动完成**：一旦检测到支付，订单自动完成

### 支付控制 {#payment-controls}

使用 SumUp 终端网关时，您有以下选项：

- **开始新支付**：在连接的终端上发起支付请求
- **取消支付**：取消当前正在处理的支付
- **支付状态**：监控支付过程的当前状态

### 订单管理 {#order-management}

- **自动更新**：订单在支付成功后自动标记为已完成
- **支付追踪**：所有支付详情都记录在订单备注中
- **收据生成**：支付成功后生成标准 POS 收据

## 硬件兼容性 {#hardware-compatibility}

:::important 连接要求
此实现使用 SumUp 的 JavaScript SDK，这意味着它通过 Web 应用程序工作，但仅需 **连接到互联网的终端**。目前不支持蓝牙终端。
:::

### 支持的终端（连接互联网） {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet 连接终端

### 不支持的终端（蓝牙） {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - 仅支持蓝牙
- **SumUp 3G** ❌ - 仅支持蓝牙

:::note 未来支持
蓝牙终端支持将在 **2026年第1季度** 添加到 iOS 和 Android 应用程序中。这将支持所有 SumUp 认证终端，包括 Air 和 3G 型号。
:::

## 故障排除 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="读取器无法配对">

  - 确保您使用的是受支持的互联网连接终端（仅限 Solo）
  - 验证您的 SumUp Solo 设备已连接到 WiFi 和在线
  - 检查设备是否处于配对模式且配对代码输入正确
  - 确认您的 SumUp 账户已启用 API 访问
  - 确保在配对过程中保持稳定的互联网连接

  </AccordionItem>

  <AccordionItem question="支付未处理">

  - 验证读取器是否已正确配对
  - 检查您的 SumUp 账户是否活跃并且状态良好
  - 确保设备电量充足且连接正常
  - 确认 API 密钥正确且有效

  </AccordionItem>

  <AccordionItem question="订单未完成">

  - 检查互联网连接稳定性
  - 验证您在 SumUp 账户中的 webhook 设置
  - 确保在 SumUp 终端上支付成功
  - 检查 WordPress 错误日志以获取 API 通信问题

  </AccordionItem>
</Accordion>

### API 速率限制 {#api-rate-limits}

该网关经过优化以减少 API 调用并避免速率限制：
- 以高效方式检查支付状态
- 最小化不必要的 API 请求
- 自动重试逻辑处理临时故障
