# SumUp Terminal 网关 {#sumup-terminal-gateway}

SumUp Terminal 网关支持直接在 WCPOS 中使用 SumUp 读卡器接受银行卡付款。此网关与 SumUp 的付款处理系统无缝集成，适用于现场交易。

## 功能 {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="硬件集成">
    通过互联网连接将 SumUp 读卡器连接到 POS 系统
  </FeatureCard>
  <FeatureCard icon="bolt" title="实时处理">
    即时处理付款并完成订单
  </FeatureCard>
  <FeatureCard icon="shield" title="安全交易">
    通过 SumUp 进行符合 PCI 标准的付款处理
  </FeatureCard>
  <FeatureCard icon="link" title="轻松配对">
    使用配对码即可完成简单的设备配对流程
  </FeatureCard>
</FeatureGrid>

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="安装适用于 WooCommerce 的 SumUp Terminal">

从 `WP Admin > POS > 设置 > 扩展` 安装，或从 [GitHub releases 页面](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)下载最新的**插件 zip 资源文件**（不是 GitHub 源代码 zip 或 tarball），然后通过 `插件 > 新增 > 上传插件` 上传。

  </StepCard>
  <StepCard number="2" title="配置 SumUp 设置">

1. 导航到 `WP Admin > WooCommerce > 设置 > 支付`
2. 在付款方式列表中找到 **SumUp Terminal**
3. 点击 **SumUp Terminal** 以访问设置
4. 输入你的 **SumUp API 密钥**（可从 SumUp 商户仪表盘获取）
5. 保存设置

:::note
无需在 WooCommerce 设置中启用 SumUp Terminal 网关。稍后的步骤会专门为 POS 启用它。
:::

  </StepCard>
  <StepCard number="3" title="配对 SumUp Terminal">

1. 在同一设置页面中，找到 **配对读卡器** 部分
2. 在 SumUp 设备上，前往配对屏幕以显示配对码
3. 输入 SumUp 设备上显示的配对码
4. 点击 **"配对读卡器"** 以建立连接
5. 等待确认读卡器已成功配对

:::warning 重要
读卡器必须成功配对后才能处理付款。请确保完成配对流程后再继续。
:::

  </StepCard>
  <StepCard number="4" title="在 WCPOS 中启用" isLast>

1. 前往 `WP Admin > POS > 设置 > 结账`
2. 在列表中找到 **SumUp Terminal** 网关
3. 启用该网关，以便在 POS 中使用
4. 保存设置

  </StepCard>
</Steps>

## 使用 {#usage}

### 处理付款 {#processing-payments}

1. **添加商品**：在 POS 中将产品添加到购物车
2. **选择网关**：选择“SumUp Terminal”作为付款方式
3. **开始付款**：点击以在 SumUp 设备上发起新的付款
4. **顾客付款**：顾客在 SumUp terminal 上完成付款
5. **自动完成**：检测到付款后，订单会自动完成

### 付款控制 {#payment-controls}

使用 SumUp Terminal 网关时，可以使用以下选项：

- **开始新付款**：在已连接的终端上发起付款请求
- **取消付款**：取消当前正在处理的付款
- **付款状态**：监控付款流程的当前状态

### 订单管理 {#order-management}

- **自动更新**：付款成功后，订单会自动标记为已完成
- **付款跟踪**：所有付款详情都会记录在订单备注中
- **收据生成**：付款成功后会生成标准 POS 收据

## 硬件兼容性 {#hardware-compatibility}

:::important 连接要求
此实现使用 SumUp 的 JavaScript SDK，这意味着它可通过 Web 应用程序工作，但要求**仅使用联网终端**。目前不支持蓝牙终端。
:::

### 支持的终端（联网） {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - 通过 WiFi/以太网连接的终端

### 不支持的终端（蓝牙） {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - 仅支持蓝牙
- **SumUp 3G** ❌ - 仅支持蓝牙

:::note 未来支持
计划在未来的 iOS 和 Android 应用版本中支持蓝牙终端。可用后，将支持所有 SumUp 认证终端，包括 Air 和 3G 型号。
:::

## 故障排除 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="读卡器无法配对">

  - 确保使用受支持且已连接互联网的终端（仅 Solo）
  - 确认 SumUp Solo 设备已连接 WiFi 并处于在线状态
  - 检查设备是否处于配对模式，并确认配对代码输入正确
  - 确认您的 SumUp 账户已启用 API 访问权限
  - 确保配对期间互联网连接稳定

  </AccordionItem>

  <AccordionItem question="付款无法处理">

  - 验证读卡器是否已正确配对
  - 检查您的 SumUp 账户是否处于活动状态且状态良好
  - 确保设备电量充足且连接正常
  - 确认 API 密钥正确且处于活动状态

  </AccordionItem>

  <AccordionItem question="订单未完成">

  - 检查互联网连接稳定性
  - 验证您的 SumUp 账户中的 webhook 设置
  - 确保付款已在 SumUp Terminal 上成功完成
  - 检查 WordPress 错误日志，了解 API 通信问题

  </AccordionItem>
</Accordion>

### API 速率限制 {#api-rate-limits}

该网关经过优化，可减少 API 调用并避免触发速率限制：
- 高效检查付款状态
- 尽量减少不必要的 API 请求
- 自动重试逻辑会处理临时故障
