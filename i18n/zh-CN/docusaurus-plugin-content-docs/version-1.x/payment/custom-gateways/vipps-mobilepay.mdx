# Vipps MobilePay 网关 {#vipps-mobilepay-gateway}

Vipps MobilePay 网关让您可以直接在 WCPOS 中接受手机支付。顾客通过扫描屏幕上的二维码或在手机上确认推送通知进行支付—无需卡片读取硬件。

## 功能 {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="二维码支付">
    在结账时显示二维码，供顾客扫描以使用手机支付
  </FeatureCard>
  <FeatureCard icon="bolt" title="推送通知">
    通过输入顾客的号码直接向其手机发送支付请求
  </FeatureCard>
  <FeatureCard icon="globe" title="北欧覆盖">
    在挪威使用 Vipps，在丹麦和芬兰使用 MobilePay
  </FeatureCard>
  <FeatureCard icon="credit-card" title="自动捕获">
    在授权后立即捕获资金，或预留以供手动捕获
  </FeatureCard>
</FeatureGrid>

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="下载并安装">

1. 访问 [WCPOS Vipps MobilePay 发布页面](https://github.com/wcpos/wcpos-vipps/releases)
2. 下载最新的 **wcpos-vipps.zip** 文件
3. 在您的 WordPress 管理中，转到 `插件 > 添加新插件 > 上传插件`
4. 上传 zip 文件并激活插件

  </StepCard>
  <StepCard number="2" title="配置 Vipps 证书">

1. 导航到 `WP 管理 > WooCommerce > 设置 > 支付`
2. 在支付方式列表中找到 **WCPOS Vipps MobilePay**
3. 点击 **WCPOS Vipps MobilePay** 以访问设置
4. 输入您在 [Vipps 门户](https://portal.vipps.no/) 中的证书：
   - **商户序列号 (MSN)**
   - **客户端 ID**
   - **客户端密钥**
   - **订阅密钥**
5. 保存设置

:::note
您无需在此处启用ゲートウェイ供POS使用——它将在下一步专门为POS启用。在 WooCommerce 设置中启用它也会使其可用于您的在线商店结账，这对于测试非常有用。
:::

:::tip 已在使用 Vipps 插件？
如果您已安装官方的 [Checkout with Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/) 插件，则在激活此插件时，您的凭据将自动导入。您可以跳过手动输入凭据的步骤。
:::

  </StepCard>
  <StepCard number="3" title="在 WCPOS 中启用" isLast>

1. 转到 `WP 管理 > WCPOS > 设置 > 结账`
2. 在列表中找到 **WCPOS Vipps MobilePay** 网关
3. 启用网关以供 POS 使用
4. 保存您的设置

  </StepCard>
</Steps>

## 使用 {#usage}

### 处理支付 — 二维码 {#processing-payments-qr-code}

1. **添加商品**：在 POS 中将商品添加到购物车
2. **选择网关**：选择“Vipps MobilePay”作为支付方式
3. **生成二维码**：点击“生成二维码”按钮
4. **顾客扫描**：顾客使用其 Vipps 或 MobilePay 应用扫描二维码
5. **顾客确认**：顾客在应用中确认支付
6. **自动完成**：支付一旦授权，订单自动完成

### 处理支付 — 推送通知 {#processing-payments-push-notification}

1. **添加商品**：在 POS 中将产品添加到购物车
2. **选择网关**：选择“Vipps MobilePay”作为支付方式
3. **输入电话号码**：输入顾客的电话号码
4. **发送到手机**：点击“发送到手机”按钮
5. **顾客确认**：顾客收到推送通知并在其应用中确认支付
6. **自动完成**：支付一旦授权，订单自动完成

### 取消支付 {#cancelling-a-payment}

在等待顾客确认时，您可以点击 **取消支付** 按钮以中止交易。这将取消 Vipps 端的待处理付款，并重置结账界面。

### 退款 {#refunds}

退款通过标准 WooCommerce 退款流程处理。打开订单，点击 **退款**，输入金额，退款将通过 Vipps API 自动处理。

## 支持的市场 {#supported-markets}

Vipps MobilePay 在北欧地区以两个品牌运营：

| 地区 | 品牌 | 货币 |
|------|------|------|
| 挪威 | Vipps | NOK |
| 丹麦, 芬兰 | MobilePay | DKK, EUR |

您的商户账户决定可用的市场和货币。任何支持的市场的顾客都可以使用其本地的 Vipps 或 MobilePay 应用进行付款。

## 需求 {#requirements}

<RequirementsList>
  <RequirementItem label="Vipps 账户">已激活的 Vipps MobilePay 商户账户及 API 凭据</RequirementItem>

## 故障排除 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="支付未完成">

  - 结账最多等待 5 分钟——如果顾客在此时间段内未确认，支付将超时
  - 检查您是否没有混淆测试模式和生产凭据
  - 验证 POS 设备和顾客手机的互联网连接是否稳定

  </AccordionItem>

  <AccordionItem question="无效凭据错误">

  - 仔细检查所有四个凭据字段（MSN、客户端 ID、客户端密钥、订阅密钥）
  - 确保在测试模式启用时您使用的是测试凭据，禁用时使用的是生产凭据
  - 在 [Vipps 门户](https://portal.vipps.no/) 验证您的凭据

  </AccordionItem>

  <AccordionItem question="二维码未生成">

  - 确认您的网站通过 HTTPS 运行——Vipps API 拒绝来自 HTTP 网站的请求
  - 检查 WooCommerce 日志（`WooCommerce > 状态 > 日志`）以获取 API 错误详情
  - 验证您的商户账户是否激活并启用了电子支付 API

  </AccordionItem>
</Accordion>
