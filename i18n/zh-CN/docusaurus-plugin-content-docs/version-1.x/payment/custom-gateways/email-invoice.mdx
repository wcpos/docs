# 电子邮件发票网关 {#email-invoice-gateway}

电子邮件发票网关允许您直接从 WooCommerce POS 向客户发送付款发票。这非常完美，适用于客户希望稍后付款或需要使用您的在线商店付款方式完成支付的情况。

## 功能 {#features}

  <FeatureCard icon="envelope" title="电子邮件集成">

  </FeatureCard>
  <FeatureCard icon="users" title="客户管理">

    在店内完成订单或允许远程付款
  </FeatureCard>
</FeatureGrid>

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="下载并安装">

1. 访问 [电子邮件发票网关发布页面](https://github.com/wcpos/email-invoice-gateway/releases)
2. 下载最新的 **woocommerce-pos-email-invoice-gateway.zip** 文件
3. 在您的 WordPress 管理中，转到 `插件 > 添加新 > 上传插件`
4. 上传 zip 文件并激活插件

  </StepCard>
  <StepCard number="2" title="在 WooCommerce POS 中启用" isLast>

1. 转到 `WP 管理 > WooCommerce POS > 设置 > 结账`
2. 在列表中找到 **WooCommerce POS 发票支付网关**
3. 启用此网关以便在 POS 中使用
4. 保存设置

:::note
此网关不需要额外的 API 密钥或外部服务配置。它使用您现有的 WordPress 电子邮件系统和 WooCommerce 支付方法。
:::

  </StepCard>
</Steps>

## 使用 {#usage}

### 发送发票 {#sending-invoices}

1. **添加项目**：在 POS 中将产品添加到购物车
2. **选择网关**：选择“电子邮件发票”作为支付方式
3. **电子邮件地址**：
   - 对于现有客户：电子邮件自动从客户数据中填充
   - 对于访客订单：手动输入客户的电子邮件地址
4. **可选**：勾选“保存到账单地址”以更新客户的账单信息
5. **发送发票**：完成流程以发送发票电子邮件

### 客户体验 {#customer-experience}

当您发送发票时，客户会收到包含以下内容的电子邮件：

- **订单详情**：详细列出商品、数量和价格
- **订单总额**：包括税费和费用的最终应付金额
- **支付链接**：直接链接以在线完成支付
- **订单信息**：订单号和参考详情

### 完成支付 {#payment-completion}

客户可以通过两种方式完成支付：

1. **在线支付**：点击电子邮件中的“为此订单付款”链接
2. **店内返回**：返回以使用其他 POS 支付方式完成支付

“为此订单付款”链接将客户带到您网站上的专用支付页面，在该页面上他们可以使用您启用的任何在线支付网关（Stripe、PayPal 等）。

## 电子邮件管理 {#email-management}

### 现有客户 {#existing-customers}

- 自动从客户档案中检索电子邮件地址
- 保留客户的账单信息
- 在客户账户下维护订单历史

### 访客客户 {#guest-customers}

- 手动输入一次性客户的电子邮件地址
- 选择将电子邮件保存到账单地址以供将来参考
- 访客订单在 WooCommerce 中得到正确跟踪

### 电子邮件模板 {#email-templates}

该网关使用 WooCommerce 的标准电子邮件模板：

- **可定制**：通过 WooCommerce 电子邮件设置修改电子邮件外观
- **品牌化**：包括您的商店徽标和品牌
- **专业**：干净、专业的发票格式
- **移动友好**：适应所有设备的响应式设计

## 使用案例 {#use-cases}

### 完美适合 {#perfect-for}

- **B2B 销售**：需要通过其会计系统处理付款的商业客户
- **大订单**：需要批准或处理时间的高价值交易
- **远程客户**：需要在离开商店后完成支付的客户
- **账户客户**：具有既定付款条款的常规客户
- **报价转订单**：将报价或估算转为应付发票

## 故障排除 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="电子邮件未发送">

  - 检查 WordPress 电子邮件配置
  - 如果使用自定义电子邮件服务，验证 SMTP 设置
  - 使用 WordPress 电子邮件测试插件进行测试
  - 检查垃圾邮件文件夹查看测试电子邮件

  </AccordionItem>

  <AccordionItem question="支付链接无法使用">

  - 确保 WooCommerce 配置正确
  - 验证至少启用一种支付网关用于网络结账
  - 检查 SSL 证书是否正确安装
  - 确认订单状态允许支付

  </AccordionItem>

  <AccordionItem question="客户无法完成支付">

  - 验证网络支付网关是否处于活动状态且已配置
  - 检查订单是否未过期或被取消
  - 确保客户具备充足的支付方式限额
  - 自己测试支付流程

  </AccordionItem>
</Accordion>

### 电子邮件投递 {#email-delivery}

为确保可靠的电子邮件投递，考虑：

- **SMTP 服务**：使用 SendGrid、Mailgun 或 Amazon SES 等服务
- **电子邮件插件**：安装 WordPress SMTP 插件以提高投递质量
- **域名认证**：设置 SPF、DKIM 和 DMARC 记录
- **监控**：跟踪电子邮件投递和开启率
