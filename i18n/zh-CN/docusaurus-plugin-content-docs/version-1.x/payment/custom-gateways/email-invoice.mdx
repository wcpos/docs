# 邮件发票网关 {#email-invoice-gateway}

邮件发票网关允许您直接从WCPOS通过电子邮件向客户发送付款发票。这非常适合客户希望稍后付款或需要使用您的网络商店付款方式在线完成支付的情况。

  </FeatureCard>
</FeatureGrid>

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="下载和安装">

1. 访问 [邮件发票网关发布页面](https://github.com/wcpos/email-invoice-gateway/releases)
2. 下载最新的 **woocommerce-pos-email-invoice-gateway.zip** 文件
3. 在您的WordPress管理中，前往 `插件 > 添加新 > 上传插件`
4. 上传zip文件并激活插件

  </StepCard>
  <StepCard number="2" title="在WCPOS中启用" isLast>

1. 转到 `WP Admin > POS > 设置 > 结账`
2. 在列表中找到 **WCPOS发票付款网关**
3. 启用该网关以供POS使用
4. 保存您的设置

:::note
该网关不需要额外的API密钥或外部服务配置。它利用您现有的WordPress电子邮件系统和WooCommerce付款方式。
:::

  </StepCard>
</Steps>

## 使用 {#usage}

### 发送发票 {#sending-invoices}

1. **添加商品**：在POS中将产品添加到购物车
2. **选择网关**：选择“邮件发票”作为付款方式
3. **电子邮件地址**：
   - 对于现有客户：电子邮件会自动从客户数据中填写
   - 对于访客订单：手动输入客户的电子邮件地址
4. **可选**：勾选“保存到账单地址”以更新客户的账单信息
5. **发送发票**：完成流程以发送发票电子邮件

### 客户体验 {#customer-experience}

当您发送发票时，客户会收到一封包含以下内容的电子邮件：

- **订单详情**：完整的商品、数量和价格明细
- **订单总额**：包括税费在内的最终应付金额
- **付款链接**：直接链接以在线完成支付
- **订单信息**：订单号和参考详情

### 完成付款 {#payment-completion}

客户可以通过两种方式完成付款：

1. **在线支付**：点击电子邮件中的“为此订单付款”链接
2. **店内返回**：返回以使用其他POS支付方式完成付款

“为此订单付款”链接将客户引导至您网站上的专用付款页面，在该页面上他们可以使用您启用的任何网络支付网关（Stripe、PayPal等）。

## 电子邮件管理 {#email-management}

### 现有客户 {#existing-customers}

- 电子邮件地址会自动从客户资料中提取
- 保留客户账单信息
- 订单历史记录保留在客户账户下

### 访客客户 {#guest-customers}

- 手动输入一次性客户的电子邮件地址
- 选项将电子邮件保存到账单地址以供将来参考
- 在WooCommerce中正确跟踪访客订单

### 邮件模板 {#email-templates}

该网关使用WooCommerce的标准电子邮件模板：

- **可定制**：通过WooCommerce电子邮件设置修改电子邮件外观
- **品牌化**：添加您的商店徽标和品牌
- **专业**：清晰、专业的发票格式
- **移动友好**：适用于所有设备的响应式设计

## 用例 {#use-cases}

### 完美适合 {#perfect-for}

- **B2B销售**：需要通过其会计系统处理付款的商业客户
- **大订单**：需要批准或处理时间的高价值交易
- **远程客户**：需要在离开商店后完成付款的客户
- **账户客户**：有固定付款条件的常客
- **报价到订单**：将报价或估算转换为可付款发票

## 故障排除 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="电子邮件未发送">

  - 检查WordPress电子邮件配置
  - 如果使用自定义电子邮件服务，请验证SMTP设置
  - 使用WordPress电子邮件测试插件进行测试
  - 检查垃圾邮件文件夹以寻找测试邮件

  </AccordionItem>

  <AccordionItem question="付款链接无法工作">

  - 确保WooCommerce配置正确
  - 验证至少启用了一种支付网关用于网络结账
  - 检查SSL证书是否正确安装
  - 确认订单状态允许付款

  </AccordionItem>

  <AccordionItem question="客户无法完成付款">

  - 验证网络支付网关是否处于活动状态并已配置
  - 检查订单是否过期或已取消
  - 确保客户有足够的付款方式限制
  - 自己测试付款流程

  </AccordionItem>
</Accordion>

### 电子邮件投递 {#email-delivery}

为了可靠送达电子邮件，请考虑：

- **SMTP服务**：使用如SendGrid、Mailgun或Amazon SES等服务
- **电子邮件插件**：安装WordPress SMTP插件以改进投递
- **域名验证**：设置SPF、DKIM和DMARC记录
- **监控**：跟踪电子邮件投递和打开率
