# 电子邮件发票网关 {#email-invoice-gateway}

电子邮件发票网关允许您直接从 WooCommerce POS 向客户发送付款发票。这非常适合客户希望稍后付款或需要使用您的网络商店支付方式在线完成付款的情况。

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="下载并安装">

1. 访问 [电子邮件发票网关发布页面](https://github.com/wcpos/email-invoice-gateway/releases)
2. 下载最新的 **woocommerce-pos-email-invoice-gateway.zip** 文件
3. 在您的 WordPress 管理后台，转到 `插件 > 添加新插件 > 上传插件`
4. 上传 zip 文件并激活插件

  </StepCard>
  <StepCard number="2" title="在 WooCommerce POS 中启用" isLast>

1. 转到 `WP 管理 > WooCommerce POS > 设置 > 结账`
2. 在列表中找到 **WooCommerce POS 发票支付网关**
3. 启用该网关以便在 POS 中使用
4. 保存您的设置

:::note
此网关无需额外的 API 密钥或外部服务配置。它利用您现有的 WordPress 电子邮件系统和 WooCommerce 支付方式。
:::

  </StepCard>
</Steps>

## 使用 {#usage}

### 发送发票 {#sending-invoices}

1. **添加商品**：在 POS 中将产品添加到购物车
2. **选择网关**：选择“电子邮件发票”作为付款方式
3. **电子邮件地址**： 
   - 对于现有客户：电子邮件自动从客户数据中填写
   - 对于访客订单：手动输入客户的电子邮件地址
4. **可选**：勾选“保存为账单地址”以更新客户的账单信息
5. **发送发票**：完成流程以发送发票电子邮件

### 客户体验 {#customer-experience}

当您发送发票时，客户会收到一封包含以下内容的电子邮件：

- **订单详情**：物品、数量和价格的完整清单
- **订单总计**：包括税费的应付总金额
- **付款链接**：直接链接以在线完成付款
- **订单信息**：订单号码和参考详情

### 付款完成 {#payment-completion}

客户可以通过两种方式完成付款：

1. **在线付款**：点击电子邮件中的“为该订单付款”链接
2. **店内返回**：返回以使用其他 POS 付款方式完成付款

“为该订单付款”链接将客户带到您网站上的专用付款页面，在该页面上他们可以使用您启用的任何网络支付网关（Stripe、PayPal 等）。

## 电子邮件管理 {#email-management}

### 现有客户 {#existing-customers}

- 自动从客户档案中检索电子邮件地址
- 保留客户的账单信息
- 订单历史记录保存在客户帐户下

### 访客客户 {#guest-customers}

- 为一次性客户手动输入电子邮件地址
- 有选择地将电子邮件保存到账单地址以供将来参考
- 访客订单在 WooCommerce 中得到妥善跟踪

### 电子邮件模板 {#email-templates}

该网关使用 WooCommerce 的标准电子邮件模板：

- **可定制**：通过 WooCommerce 电子邮件设置修改电子邮件外观
- **品牌化**：包含您的商店徽标和品牌
- **专业**：清晰、专业的发票格式
- **移动友好**：针对所有设备的响应式设计

## 用例 {#use-cases}

### 最适合 {#perfect-for}

- **B2B 销售**：需要通过其会计系统处理付款的商业客户
- **大订单**：需要批准或处理时间的高价值交易
- **远程客户**：需要在离开商店后完成付款的客户
- **账户客户**：与已建立的付款条款的常规客户
- **从报价到订单**：将报价或估算转换为应付款的发票

## 故障排除 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="邮件未发送">

  - 检查 WordPress 电子邮件配置
  - 如果使用自定义电子邮件服务，验证 SMTP 设置
  - 使用 WordPress 电子邮件测试插件进行测试
  - 检查垃圾邮件文件夹中的测试邮件

  </AccordionItem>

  <AccordionItem question="付款链接无法工作">

  - 确保 WooCommerce 正确配置
  - 验证至少启用了一个用于网络结账的支付网关
  - 检查 SSL 证书是否正确安装
  - 确认订单状态允许付款

  </AccordionItem>

  <AccordionItem question="客户无法完成付款">

  - 验证网络支付网关是否激活且已配置
  - 检查订单是否未过期或被取消
  - 确保客户具有足够的付款方式限额
  - 自己测试付款流程

  </AccordionItem>
</Accordion>

### 电子邮件投递 {#email-delivery}

为了确保可靠的电子邮件投递，请考虑：

- **SMTP 服务**：使用 SendGrid、Mailgun 或 Amazon SES 等服务
- **电子邮件插件**：安装 WordPress SMTP 插件以改善投递
- **域名认证**：设置 SPF、DKIM 和 DMARC 记录
- **监控**：跟踪电子邮件投递和打开率
