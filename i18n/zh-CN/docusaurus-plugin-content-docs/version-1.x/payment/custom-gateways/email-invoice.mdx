# 电子邮件发票网关

电子邮件发票网关允许您直接从 WooCommerce POS 通过电子邮件向客户发送付款发票。这非常适合客户希望稍后支付或需要使用网上商店的支付方式完成支付的情况。

## 功能

- **电子邮件集成**：直接从 POS 界面发送发票
- **客户管理**：从现有客户数据中自动填充电子邮件或添加新的电子邮件地址
- **游客订单**：支持游客客户，可选择更新账单地址
- **在线支付**：客户收到"支付此订单"的链接以在线完成支付
- **灵活工作流**：可在店内完成订单或允许远程完成支付

## 安装

### 第 1 步：下载和安装

1. 访问 [Email Invoice Gateway 发布页面](https://github.com/wcpos/email-invoice-gateway/releases)
2. 下载最新的 **woocommerce-pos-email-invoice-gateway.zip** 文件
3. 在 WordPress 管理后台，转到 `插件 > 添加新的 > 上传插件`
4. 上传 zip 文件并激活插件

### 第 2 步：在 WooCommerce POS 中启用

1. 转到 `WP 管理后台 > WooCommerce POS > 设置 > 结账`
2. 在列表中找到 **WooCommerce POS 发票支付网关**
3. 启用 POS 中使用的网关
4. 保存设置

:::note
此网关不需要额外的 API 密钥或外部服务配置。它使用您现有的 WordPress 电子邮件系统和 WooCommerce 支付方式。
:::

## 使用

### 发送发票

1. **添加商品**：在 POS 中将商品添加到购物车
2. **选择网关**：选择"电子邮件发票"作为支付方式
3. **电子邮件地址**：
   - 对于现有客户：从客户数据自动填充电子邮件
   - 对于游客订单：手动输入客户的电子邮件地址
4. **可选**：勾选"保存到账单地址"以更新客户的账单信息
5. **发送发票**：完成发送发票电子邮件的流程

### 客户体验

发送发票时，客户将收到包含以下内容的电子邮件：

- **订单详情**：商品、数量和价格的完整明细
- **订单总计**：包括税费的最终应付金额
- **支付链接**：直接链接以在线完成支付
- **订单信息**：订单号和参考详情

### 支付完成

客户可以通过两种方式完成支付：

1. **在线支付**：点击电子邮件中的"支付此订单"链接
2. **店内返回**：返回使用其他 POS 支付方式完成支付

"支付此订单"链接会将客户带到您网站上的专用支付页面，他们可以使用任何已启用的网上支付网关（如 Stripe、PayPal 等）。

## 电子邮件管理

### 现有客户

- 电子邮件地址自动从客户档案中检索
- 保留客户账单信息
- 订单历史记录保存在客户账户下

### 游客客户

- 为一次性客户手动输入电子邮件地址
- 选择保存电子邮件到账单地址以便将来参考
- 游客订单在 WooCommerce 中正常追踪

### 电子邮件模板

网关使用 WooCommerce 的标准电子邮件模板：

- **可自定义**：通过 WooCommerce 电子邮件设置修改电子邮件外观
- **品牌化**：包含您的店铺标志和品牌元素
- **专业**：干净、专业的发票格式
- **移动友好**：适用于所有设备的响应式设计

## 使用场景

### 非常适合

- **B2B 销售**：需要通过其会计系统处理付款的商业客户
- **大额订单**：需要审批或处理时间的高价值交易
- **远程客户**：需要在离开店后完成支付的客户
- **账户客户**：有既定支付条款的固定客户
- **报价转订单**：将报价或估价转换为可支付的发票

### 工作流程示例

**零售店**
1. 客户选择商品但需要获得购买批准
2. 发送包含所有详情的发票电子邮件
3. 客户在准备好时在线完成支付
4. 系统自动更新订单

**服务业**
1. 完成服务工作并将商品添加到 POS
2. 向客户的会计部门发送发票
3. 客户使用首选支付方式在线支付
4. 接收支付确认并完成订单

## 要求

- **WooCommerce POS**：已安装并激活免费或专业版
- **电子邮件系统**：正常运行的 WordPress 电子邮件配置
- **网上支付网关**：在 WooCommerce 中至少启用一种在线支付方式
- **SSL 证书**：需要 HTTPS 以进行安全支付处理

## 故障排除

### 常见问题

**电子邮件未发送**
- 检查 WordPress 电子邮件配置
- 验证 SMTP 设置（如果使用自定义电子邮件服务）
- 使用 WordPress 电子邮件测试插件进行测试
- 检查测试电子邮件的垃圾邮件文件夹

**支付链接不起作用**
- 确保 WooCommerce 正确配置
- 验证至少启用了一个网上结账支付网关
- 检查 SSL 证书是否正确安装
- 确认订单状态允许支付

**客户无法完成支付**
- 验证网上支付网关是否活跃且已配置
- 检查订单是否已过期或已取消
- 确保客户的支付方式限额充足
- 自行测试支付流程

### 电子邮件投递

为获得可靠的电子邮件投递，请考虑：

- **SMTP 服务**：使用 SendGrid、Mailgun 或 Amazon SES 等服务
- **电子邮件插件**：安装 WordPress SMTP 插件以提高投递率
- **域名认证**：设置 SPF、DKIM 和 DMARC 记录
- **监控**：跟踪电子邮件投递和打开率
