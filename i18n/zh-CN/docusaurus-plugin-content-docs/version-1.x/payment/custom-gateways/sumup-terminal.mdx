# SumUp终端网关

SumUp终端网关使您能够直接在WooCommerce POS中使用SumUp读卡器接受卡支付。此网关与SumUp的支付处理系统无缝集成，适用于面对面的交易。

## 安装

<Steps>
  <StepCard number="1" title="下载并安装">

1. 访问[SumUp终端用于WooCommerce的发布页面](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. 下载最新的**sumup-terminal-for-woocommerce.zip**文件
3. 在您的WordPress管理后台，转到`插件 > 添加新 > 上传插件`
4. 上传zip文件并激活插件

  </StepCard>
  <StepCard number="2" title="配置SumUp设置">

1. 导航到`WP管理 > WooCommerce > 设置 > 支付`
2. 在支付方式列表中找到**SumUp终端**
3. 点击**SumUp终端**以访问设置
4. 输入您的**SumUp API密钥**（可在您的SumUp商户仪表板上获取）
5. 保存设置

:::note
您无需在WooCommerce设置中启用SumUp终端网关。它将在后面的步骤中专门为POS启用。
:::

  </StepCard>
  <StepCard number="3" title="配对您的SumUp终端">

1. 在同一设置页面，找到**配对读卡器**部分
2. 在您的SumUp设备上，转到配对屏幕以显示配对代码
3. 输入您在SumUp设备上显示的配对代码
4. 点击**“配对读卡器”**以建立连接
5. 等待确认读卡器已成功配对

:::warning 重要
在您可以处理支付之前，读卡器必须成功配对。确保配对过程完成后再继续。
:::

  </StepCard>
  <StepCard number="4" title="在WooCommerce POS中启用" isLast>

1. 转到`WP管理 > WooCommerce POS > 设置 > 结账`
2. 在列表中找到**SumUp终端**网关
3. 启用该网关以在POS中使用
4. 保存您的设置

  </StepCard>
</Steps>

## 使用

### 处理支付

1. **添加商品**：在POS中将产品添加到购物车中
2. **选择网关**：选择“SumUp终端”作为支付方式
3. **开始支付**：点击在您的SumUp设备上启动新支付
4. **客户支付**：客户在SumUp终端上完成支付
5. **自动完成**：当检测到支付时，订单自动完成

### 支付控制

使用SumUp终端网关时，您有以下选项：

- **开始新支付**：在连接的终端上发起支付请求
- **取消支付**：取消当前正在进行的支付
- **支付状态**：监控支付流程的当前状态

### 订单管理

- **自动更新**：成功支付后，订单会自动标记为完成
- **支付跟踪**：所有支付细节均记录在订单备注中
- **收据生成**：成功支付后生成标准POS收据

## 硬件兼容性

:::important 连接要求
此实现使用SumUp的JavaScript SDK，这意味着它通过Web应用程序工作，但仅支持**连接互联网的终端**。目前不支持蓝牙终端。
:::

### 支持的终端（连接互联网）

- **SumUp Solo** ✅ - 通过WiFi/以太网连接的终端

### 不支持的终端（蓝牙）

- **SumUp Air** ❌ - 仅蓝牙
- **SumUp 3G** ❌ - 仅蓝牙

:::note 未来支持
在**2026年第一季度**，将为iOS和Android应用程序添加蓝牙终端支持。这将支持所有SumUp认证终端，包括Air和3G型号。
:::

## 疑难解答

### 常见问题

<Accordion>
  <AccordionItem question="读卡器无法配对">

  - 确保您使用的是支持的互联网连接终端（仅限Solo）
  - 验证您的SumUp Solo设备已连接到WiFi并在线
  - 检查设备是否在配对模式，并为配对代码输入正确
  - 确认您的SumUp账户已启用API访问
  - 确保配对过程中互联网连接稳定

  </AccordionItem>

  <AccordionItem question="支付未处理">

  - 验证读卡器已正确配对
  - 检查您的SumUp账户是否激活且状态良好
  - 确保设备电量充足且连接正常
  - 确认API密钥正确且有效

  </AccordionItem>

  <AccordionItem question="订单未完成">

  - 检查互联网连接稳定性
  - 验证您在SumUp账户中的Webhook设置
  - 确保在SumUp终端上支付成功
  - 检查WordPress错误日志中的API通信问题

  </AccordionItem>
</Accordion>
