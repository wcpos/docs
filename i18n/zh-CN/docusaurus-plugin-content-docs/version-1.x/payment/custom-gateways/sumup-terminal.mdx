# SumUp终端网关

SumUp终端网关使您能够通过SumUp读卡器直接在WooCommerce POS中接受卡片支付。此网关与SumUp的支付处理系统无缝集成，适用于面对面交易。

## 特性

- **硬件集成**：通过互联网连接将SumUp读卡器连接到您的POS系统
- **即时处理**：即时支付处理和订单完成
- **自动检测**：当检测到付款时订单会自动完成
- **安全交易**：通过SumUp进行PCI合规支付处理
- **简易配对**：使用配对代码进行简单的设备配对过程
- **基于网络的SDK**：使用SumUp的JavaScript SDK以兼容Web应用程序

## 安装

### 第一步：下载和安装

1. 访问[SumUp终端WooCommerce发布页面](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. 下载最新的**sumup-terminal-for-woocommerce.zip**文件
3. 在您的WordPress管理后台，转到`插件 > 添加新 > 上传插件`
4. 上传zip文件并激活插件

### 第二步：配置SumUp设置

1. 导航到`WP Admin > WooCommerce > 设置 > 支付`
2. 在付款方式列表中找到**SumUp终端**
3. 点击**SumUp终端**以访问设置
4. 输入您的**SumUp API密钥**（可从您的SumUp商户仪表板上获得）
5. 保存设置

:::note
您无需在WooCommerce设置中启用SumUp终端网关。它将在稍后的步骤中专门为POS启用。
:::

### 第三步：配对您的SumUp终端

1. 在同一设置页面，找到**配对读卡器**部分
2. 在您的SumUp设备上，导航至配对屏幕以显示配对代码
3. 输入在您的SumUp设备上显示的配对代码
4. 点击**“配对读卡器”**以建立连接
5. 等待确认读卡器已成功配对

:::warning 重要
在处理支付之前，读卡器必须成功配对。请确保配对过程完成后再继续。
:::

### 第四步：在WooCommerce POS中启用

1. 转到`WP Admin > WooCommerce POS > 设置 > 结账`
2. 在列表中找到**SumUp终端**网关
3. 启用该网关以便在POS中使用
4. 保存您的设置

## 使用

### 处理支付

1. **添加商品**：在POS中将产品添加到购物车
2. **选择网关**：选择“SumUp终端”作为支付方式
3. **开始支付**：点击以在您的SumUp设备上启动新支付
4. **客户支付**：客户在SumUp终端上完成支付
5. **自动完成**：当检测到付款时，订单会自动完成

### 支付控制

使用SumUp终端网关时，您有以下选项：

- **开始新支付**：在连接的终端上发起付款请求
- **取消支付**：取消当前正在进行的支付
- **支付状态**：监控当前支付过程的状态

### 订单管理

- **自动更新**：订单在成功付款后自动标记为已完成
- **付款追踪**：所有付款详细信息都记录在订单备注中
- **生成收据**：成功支付后生成标准POS收据

## 需求

- **SumUp账户**：激活的SumUp商户账户
- **API访问**：来自您的商户仪表板的SumUp API密钥
- **兼容硬件**：SumUp读卡器设备
- **WooCommerce POS**：安装并激活的免费或专业版本
- **稳定连接**：用于API通信的可靠互联网连接

## 硬件兼容性

:::important 连接要求
此实现使用SumUp的JavaScript SDK，这意味着它通过Web应用程序工作，但仅需要**连接互联网的终端**。目前不支持蓝牙终端。
:::

### 支持的终端（连接互联网）

- **SumUp Solo** ✅ - WiFi/以太网连接的终端

### 不支持的终端（蓝牙）

- **SumUp Air** ❌ - 仅支持蓝牙
- **SumUp 3G** ❌ - 仅支持蓝牙

:::note 未来支持
蓝牙终端的支持将于**2026年第一季度**添加到iOS和Android应用程序中。这将使所有SumUp认证的终端，包括Air和3G型号获得支持。
:::

## 故障排除

### 常见问题

**读卡器无法配对**
- 确保您使用的是支持的互联网连接终端（仅限Solo）
- 确认您的SumUp Solo设备已连接至WiFi并在线
- 检查设备是否处于配对模式，以及配对代码是否输入正确
- 确认您的SumUp账户已启用API访问
- 在配对过程中确保稳定的互联网连接

**支付未处理**
- 确认读卡器是否已正确配对
- 检查您的SumUp账户是否活跃并且状态良好
- 确保设备具有足够的电池和连接性
- 确认API密钥是否正确并有效

**订单未完成**
- 检查互联网连接的稳定性
- 确认您的SumUp账户中的Webhook设置
- 确保在SumUp终端上支付已成功
- 检查WordPress错误日志以获取API通信问题

### API速率限制

此网关经过优化，以减少API调用并避免速率限制：
- 支付状态的检查效率高
- 最小化不必要的API请求
- 自动重试逻辑处理临时故障
