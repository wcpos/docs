# 网页结账网关 {#web-checkout-gateway}

网页结账网关允许客户使用您网上商店的结账系统完成POS交易。这个网关在您的店内POS和网上支付方式之间创造了无缝的桥梁，适用于客户更愿意使用自己的设备支付或当您需要访问仅限网上的支付方法时的情况。

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="下载并安装">

1. 访问[网页结账网关发布页面](https://github.com/wcpos/web-checkout-gateway/releases)
2. 下载最新的**woocommerce-pos-web-checkout-gateway.zip**文件
3. 在您的WordPress管理后台，前往`插件 > 添加新 > 上传插件`
4. 上传zip文件并激活插件

  </StepCard>
  <StepCard number="2" title="在WooCommerce POS中启用" isLast>

1. 前往`WP Admin > WooCommerce POS > 设置 > 结账`
2. 在列表中找到**WooCommerce POS Web Checkout Gateway**
3. 启用该网关以在POS中使用
4. 保存您的设置

:::note
该网关使用您现有的WooCommerce支付方式，因此确保您至少有一个网上支付网关已正确配置（Stripe、PayPal等）。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### 启动网页结账 {#initiating-web-checkout}

1. **添加商品**：在POS中将产品添加到购物车
2. **选择网关**：选择“网页结账”作为支付方式
3. **生成链接**：系统为订单创建一个独特的结账链接
4. **客户访问**：客户点击链接访问网上结账
5. **在线支付**：客户使用网上商店的支付方式完成支付

### 支付过程 {#payment-process}

网页结账网关遵循以下工作流程：

1. **订单创建**：POS在WooCommerce中创建一个待处理订单
2. **结账链接**：系统生成一个安全且独特的支付链接
3. **客户重定向**：客户被引导到网上结账页面
4. **支付选择**：客户从可用的网上支付方式中选择
5. **支付处理**：标准的WooCommerce结账流程处理支付
6. **订单完成**：支付成功后订单状态自动更新

### POS工作流程 {#pos-workflow}

启动网页结账后：

1. **监控状态**：保持POS订单屏幕打开以监控支付状态
2. **客户支付**：客户在自己的设备或您的平板上完成支付
3. **自动更新**：支付收到时，订单状态实时更新
4. **处理支付**：点击POS中的“处理支付”按钮以继续收据生成
5. **收据生成**：按常规生成和打印收据

## 技术详情 {#technical-details}

### 订单管理 {#order-management}

- **待处理订单**：订单以“待处理支付”状态创建
- **状态更新**：支付收到时自动状态更改
- **订单备注**：支付详情记录在订单备注中
- **库存**：在结账过程中保留库存

### 安全性 {#security}

- **唯一链接**：每个结账会话都有一个唯一的、时间限制的URL
- **需SSL**：所有支付处理通过安全的HTTPS连接进行
- **PCI合规**：利用您现有的符合PCI标准的网络支付网关
- **会话管理**：安全的会话处理防止未经授权的访问

### 支付方式 {#payment-methods}

该网关提供访问您所有配置的WooCommerce支付方式：

- **信用/借记卡**：Stripe、Square、Authorize.net等
- **数字钱包**：PayPal、Apple Pay、Google Pay
- **银行转账**：直接银行支付方式
- **先买后付**：Klarna、Afterpay、Sezzle
- **加密货币**：比特币和其他加密支付网关
- **本地支付方式**：地区特定的支付方式

## 故障排除 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="结账链接无法使用">

  - 验证SSL证书是否正确安装
  - 检查WooCommerce永久链接是否配置正确
  - 确保网络支付网关处于活动状态并已配置
  - 在不同浏览器或设备上进行测试

  </AccordionItem>

  <AccordionItem question="支付未处理">

  - 确认网络支付网关已正确配置
  - 检查支付网关日志中的错误信息
  - 验证客户的支付方式是否受支持
  - 独立测试网上结账流程

  </AccordionItem>

  <AccordionItem question="POS中订单未更新">

  - 检查互联网连接的稳定性
  - 验证WordPress定时任务是否正常运行
  - 刷新POS订单屏幕
  - 在WooCommerce管理中检查订单状态

  </AccordionItem>

  <AccordionItem question="客户无法访问结账">

  - 确保结账链接未过期
  - 验证客户设备是否有互联网连接
  - 检查订单是否未被取消或已完成
  - 在不同设备上测试链接

  </AccordionItem>
</Accordion>
