# 网页结账网关 {#web-checkout-gateway}

网页结账网关允许客户使用您的网上商店结账系统完成他们的 POS 交易。此网关在您的店内 POS 和在线支付方式之间建立了无缝连接，非常适合客户更喜欢使用他们自己的设备付款或您需要访问仅限网络的支付方式的情况。

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="下载并安装">

1. 访问 [网页结账网关发布页面](https://github.com/wcpos/web-checkout-gateway/releases)
2. 下载最新的 **woocommerce-pos-web-checkout-gateway.zip** 文件
3. 在您的 WordPress 管理后台，转到 `插件 > 添加新插件 > 上传插件`
4. 上传 zip 文件并激活插件

  </StepCard>
  <StepCard number="2" title="在 WooCommerce POS 中启用" isLast>

1. 转到 `WP 管理后台 > WooCommerce POS > 设置 > 结账`
2. 在列表中找到 **WooCommerce POS 网页结账网关**
3. 启用该网关以在 POS 中使用
4. 保存您的设置

:::note
此网关利用您现有的 WooCommerce 支付方式，因此确保您至少配置了一个正确的网上支付网关（Stripe、PayPal 等）。
:::

  </StepCard>
</Steps>

## 使用 {#usage}

### 启动网页结账 {#initiating-web-checkout}

1. **添加商品**：在 POS 中将产品添加到购物车
2. **选择网关**：选择“网页结账”作为支付方式
3. **生成链接**：系统为订单生成一个独特的结账链接
4. **客户访问**：客户点击链接访问网页结账
5. **在线支付**：客户使用网上商店的支付方式完成支付

### 支付流程 {#payment-process}

网页结账网关遵循以下工作流程：

1. **订单创建**：POS 在 WooCommerce 中创建一个待处理订单
2. **结账链接**：系统生成一个安全的、独特的支付链接
3. **客户重定向**：客户被引导到网页结账页面
4. **支付选择**：客户从可用的网络支付方式中选择
5. **支付处理**：标准 WooCommerce 结账流程处理支付
6. **订单完成**：订单状态在成功支付后自动更新

### POS 工作流程 {#pos-workflow}

在启动网页结账后：

1. **监控状态**：保持 POS 订单屏幕打开以监控支付状态
2. **客户支付**：客户在他们的设备或您的平板上完成支付
3. **自动更新**：支付到账时订单状态实时更新
4. **处理支付**：在 POS 中点击“处理支付”按钮继续打印收据
5. **收据生成**：按正常流程生成并打印收据

## 技术细节 {#technical-details}

### 订单管理 {#order-management}

- **待处理订单**：订单以“待支付”状态创建
- **状态更新**：收到支付后自动状态更改
- **订单备注**：支付细节记录在订单备注中
- **库存**：在结账过程中保留库存

### 安全性 {#security}

- **唯一链接**：每个结账会话都有唯一的、时间有限的 URL
- **需要 SSL**：所有支付处理通过安全的 HTTPS 连接进行
- **PCI 合规性**：利用您现有的符合 PCI 标准的网络支付网关
- **会话管理**：安全的会话处理防止未经授权的访问

### 付款方式 {#payment-methods}

该网关可访问您配置的所有 WooCommerce 支付方式：

- **信用卡/借记卡**：Stripe、Square、Authorize.net 等
- **数字钱包**：PayPal、Apple Pay、Google Pay
- **银行转账**：直接银行支付方式
- **先购后付**：Klarna、Afterpay、Sezzle
- **加密货币**：比特币和其他加密支付网关
- **本地方法**：地区特定的支付方式

## 排错 {#troubleshooting}

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="结账链接无法使用">

  - 确保 SSL 证书已正确安装
  - 检查 WooCommerce 固定链接是否配置正确
  - 确保网络支付网关已激活并配置
  - 使用不同的浏览器或设备进行测试

  </AccordionItem>

  <AccordionItem question="支付无法处理">

  - 确认网络支付网关已正确配置
  - 检查支付网关日志是否有错误消息
  - 确保客户的支付方式被支持
  - 独立测试网页结账流程

  </AccordionItem>

  <AccordionItem question="POS 中的订单未更新">

  - 检查互联网连接的稳定性
  - 验证 WordPress 定时作业是否正常运行
  - 刷新 POS 订单屏幕
  - 在 WooCommerce 管理后台检查订单状态

  </AccordionItem>

  <AccordionItem question="客户无法访问结账">

  - 确保结账链接未过期
  - 验证客户的设备是否有互联网连接
  - 检查订单是否未被取消或完成
  - 在不同的设备上测试链接

  </AccordionItem>
</Accordion>
