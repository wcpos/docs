# Web Checkout Gateway

Web Checkout Gateway 允许客户使用您的网上商店结账系统完成其 POS 交易。此网关在您的店内 POS 和在线支付方式之间创建一个无缝的桥梁，非常适合客户希望使用自己的设备进行支付的情况，或当您需要访问仅限网络的支付方式时。

## 功能

- **无缝集成**：直接从 POS 链接到网络结账
- **所有支付方式**：访问您网上商店启用的所有支付网关
- **客户控制**：客户可以使用自己的设备和首选方式进行支付
- **订单同步**：POS 和网上商店之间自动更新订单
- **灵活工作流程**：在线完成交易同时保持 POS 工作流程

## 安装

### 步骤 1：下载和安装

1. 访问 [Web Checkout Gateway 发布页面](https://github.com/wcpos/web-checkout-gateway/releases)
2. 下载最新的 **woocommerce-pos-web-checkout-gateway.zip** 文件
3. 在您的 WordPress 管理后台，转到 `插件 > 添加新 > 上传插件`
4. 上传 zip 文件并激活插件

### 步骤 2：在 WooCommerce POS 中启用

1. 转到 `WP 管理 > WooCommerce POS > 设置 > 结账`
2. 在列表中找到 **WooCommerce POS Web Checkout Gateway**
3. 启用该网关以在 POS 中使用
4. 保存您的设置

:::note
此网关利用您现有的 WooCommerce 支付方式，因此请确保至少有一个正确配置的网络支付网关（如 Stripe、PayPal 等）。
:::

## 使用方法

### 启动 Web Checkout

1. **添加商品**：在 POS 中将产品添加到购物车
2. **选择网关**：选择“Web Checkout”作为支付方式
3. **生成链接**：系统为订单创建一个唯一的结账链接
4. **客户访问**：客户点击链接进入 Web Checkout
5. **在线支付**：客户使用网上商店支付方式完成支付

### 支付流程

Web Checkout Gateway 遵循以下工作流程：

1. **订单创建**：POS 在 WooCommerce 中创建一个待处理的订单
2. **结账链接**：系统生成一个安全的、唯一的支付链接
3. **客户重定向**：客户被引导到网络结账页面
4. **支付选择**：客户从可用的网络支付方式中选择
5. **支付处理**：标准的 WooCommerce 结账流程处理支付
6. **订单完成**：付款成功后，订单状态自动更新

### POS 工作流程

在启动 Web Checkout 后：

1. **监控状态**：保持 POS 订单屏幕打开以监控支付状态
2. **客户支付**：客户在他们的设备或您的平板电脑上完成支付
3. **自动更新**：在收到支付时，订单状态实时更新
4. **处理支付**：点击 POS 中的“处理支付”按钮继续到收据
5. **收据生成**：如常生成并打印收据

## 使用案例

### 完美适用

- **客户偏好**：希望使用自己的支付应用或卡的客户
- **复杂支付**：需要 POS 中不可用的支付方式的交易
- **分期支付**：希望使用多种支付方式的客户
- **忠诚计划**：访问基于网络的积分兑换
- **礼品卡**：拥有数字礼品卡或商店信用的客户
- **国际卡**：通过网络网关更好工作的支付方式

### 工作流程示例

**零售商店**
1. 客户在店内购物并将商品带到结账
2. 客户希望使用他们的移动钱包应用进行支付
3. 员工选择 Web Checkout Gateway
4. 客户扫描二维码或在手机上点击链接
5. 客户使用首选方式完成支付
6. 员工处理收据并完成交易

**服务行业**
1. 完成服务并将费用添加到 POS
2. 客户希望使用企业信用卡支付
3. 生成 Web Checkout 链接
4. 客户在安全的网络表单中输入支付详情
5. 通过您的网络网关处理支付
6. 订单完成并生成收据

## 技术细节

### 订单管理

- **待处理订单**：订单以“待支付”状态创建
- **状态更新**：付款收到时自动状态更改
- **订单备注**：付款详情记录在订单备注中
- **库存**：在结账过程中保留库存

### 安全性

- **唯一链接**：每个结账会话都有一个唯一的、时间限制的 URL
- **需要 SSL**：所有支付处理通过安全的 HTTPS 连接进行
- **PCI 合规性**：利用您现有的符合 PCI 的网络支付网关
- **会话管理**：安全的会话处理防止未经授权的访问

### 支付方式

该网关提供对您所有配置的 WooCommerce 支付方式的访问：

- **信用卡/借记卡**：Stripe、Square、Authorize.net 等
- **数字钱包**：PayPal、Apple Pay、Google Pay
- **银行转账**：直接银行支付方式
- **先买后付**：Klarna、Afterpay、Sezzle
- **加密货币**：比特币及其他加密支付网关
- **本地方式**：区域特定的支付方式

## 系统要求

- **WooCommerce POS**：已安装并激活的免费或专业版本
- **Web Payment Gateways**：在 WooCommerce 中配置至少一种在线支付方式
- **SSL 证书**：需要 HTTPS 进行安全支付处理
- **现代浏览器**：客户设备必须支持现代网络标准

## 故障排除

### 常见问题

**结账链接无效**
- 验证 SSL 证书是否正确安装
- 检查 WooCommerce 固定链接是否配置正确
- 确保 Web Payment Gateways 已激活并配置
- 使用不同的浏览器或设备进行测试

**支付未处理**
- 确认 Web Payment Gateways 是否正确配置
- 检查支付网关日志以获取错误消息
- 验证客户的支付方式是否受支持
- 独立测试 Web Checkout 过程

**POS 中订单没有更新**
- 检查互联网连接稳定性
- 验证 WordPress 定时任务是否正常运行
- 刷新 POS 订单屏幕
- 检查 WooCommerce 管理中订单状态

**客户无法访问结账**
- 确保结账链接未过期
- 验证客户的设备是否具有互联网连接
- 检查订单是否未被取消或完成
- 在不同设备上测试链接

### 性能优化

为了获得最佳性能：

- **缓存**：配置缓存插件以排除结账页面
- **CDN**：确保 CDN 设置不干扰结账流程
- **数据库**：优化数据库以加快订单处理速度
- **托管**：使用具有良好正常运行时间的可靠托管
