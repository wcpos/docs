# Stripe 终端网关 {#stripe-terminal-gateway}

Stripe 终端网关允许您通过 WCPOS 直接使用 Stripe 终端硬件读取器接受面对面付款。此网关支持物理卡读取器和测试用的模拟器模式。

## 特性 {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="硬件集成">
    通过互联网连接连接物理 Stripe 终端读取器
  </FeatureCard>
  <FeatureCard icon="desktop" title="模拟器模式">
    使用 Stripe 的模拟器无硬件测试付款
  </FeatureCard>
  <FeatureCard icon="bolt" title="实时处理">
    即时付款处理和确认
  </FeatureCard>
  <FeatureCard icon="shield" title="安全交易">
    通过 Stripe 进行 PCI 合规的付款处理
  </FeatureCard>
  <FeatureCard icon="phone" title="电话订单 (MOTO)">
    通过将详细信息输入读取器来接受电话中的卡付款
  </FeatureCard>
</FeatureGrid>

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="下载并安装">

1. 访问 [WooCommerce 的 Stripe Terminal 发布页面](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases)
2. 下载最新的 **stripe-terminal-for-woocommerce.zip** 文件
3. 在 WordPress 管理中，转到 `插件 > 添加新 > 上传插件`
4. 上传 zip 文件并激活插件

  </StepCard>
  <StepCard number="2" title="配置 Stripe 设置">

1. 导航到 `WP 管理 > WooCommerce > 设置 > 付款`
2. 在支付方式列表中找到 **Stripe Terminal**
3. 点击 **Stripe Terminal** 以访问设置
4. 输入您的 **Stripe 密钥**（您可以从 Stripe 仪表板获取）
5. 保存设置

:::note
您无需在 WooCommerce 设置中启用 Stripe 终端网关。它将在下一步中专门为 POS 启用。
:::

  </StepCard>
  <StepCard number="3" title="在 WCPOS 中启用" isLast>

1. 转到 `WP 管理 > POS > 设置 > 结账`
2. 在列表中找到 **Stripe Terminal** 网关
3. 启用该网关以便在 POS 中使用
4. 保存您的设置

  </StepCard>
</Steps>

## 使用 {#usage}

### 连接读取器 {#connecting-a-reader}

当您在 POS 中结账时选择 Stripe 终端网关：

1. **选择连接方式**：您可以选择连接物理读取器或使用模拟器
2. **物理读取器**：按照屏幕上的说明连接您的 Stripe 终端设备
3. **模拟器**：选择模拟器模式以在没有硬件的情况下测试各种付款场景

### 处理付款 {#processing-payments}

1. **添加商品**：在 POS 中将产品添加到购物车
2. **选择网关**：选择 "Stripe Terminal" 作为付款方式
3. **连接读取器**：连接您的读取器或选择模拟器模式
4. **处理付款**：按照提示完成交易
5. **确认**：成功付款后，订单将自动完成

### 使用模拟器测试 {#testing-with-simulator}

模拟器允许您测试各种付款方式和场景：

- **卡付款**：测试不同类型的卡（Visa、Mastercard 等）
- **非接触式付款**：模拟感应式支付交易
- **错误场景**：测试拒绝付款和其他错误条件
- **不同金额**：测试各种交易金额

### 电话订单 (MOTO) {#phone-orders-moto}

MOTO（邮寄订单/电话订单）允许您为未亲自到场的客户处理卡付款——例如，当通过电话下订单时。商家不是通过刷卡或插卡，而是直接在终端读取器的屏幕上输入卡的详细信息。

#### 设置 {#setup}

<Steps>
  <StepCard number="1" title="向 Stripe 请求 MOTO 访问权限">

MOTO 默认未启用。联系 [Stripe 支持](https://support.stripe.com/contact) 并请求为您的账户启用 MOTO 权限。这是一个快速的过程，但需要 Stripe 的手动批准。

  </StepCard>
  <StepCard number="2" title="在插件设置中启用">

1. 导航到 `WP 管理 > WooCommerce > 设置 > 付款 > Stripe Terminal`
2. 勾选 **电话订单 (MOTO)** 复选框
3. 保存设置

  </StepCard>
  <StepCard number="3" title="连接兼容的读取器" isLast>

MOTO 仅适用于连接到互联网的兼容读取器，详见 [支持的终端](#supported-terminals-internet-connected)。其他类型的读取器将不会显示切换选项。

  </StepCard>
</Steps>

#### 接受电话订单 {#taking-a-phone-order}

1. 连接兼容的读取器（见 [支持的终端](#supported-terminals-internet-connected)）
2. 在付款屏幕上，打开 **电话订单** 切换
3. 点击 **收集卡付款** — 读取器将显示卡号输入屏幕，而不是提示刷卡/插卡
4. 在读取器上输入客户的卡号、过期日期和 CVV 
5. 从那里付款会像往常一样处理

:::tip
MOTO 付款使用 `card` 作为付款方式类型，而不是 `card_present`。这意味着从 Stripe 的角度来看，它们更像是在线交易，因此适用标准的在线卡处理费用，而不是亲自费率。
:::

:::caution
仅在满足三个条件时，电话订单切换才会出现：插件设置中启用了 MOTO 设置、连接了兼容的读取器，并且该读取器不是模拟器。如果您未看到切换，请检查这些条件。
:::

## 硬件兼容性 {#hardware-compatibility}

:::important 连接要求
此实现使用 Stripe 的 JavaScript SDK，这意味着它通过网络应用程序工作，但仅需要 **连接互联网的终端**。目前不支持蓝牙终端。
:::

### 支持的终端（连接互联网） {#supported-terminals-internet-connected}

- **Stripe Reader S700/S710** ✅ - 以太网/WiFi 连接终端
- **WisePOS E** ✅ - WiFi 连接终端

### 不支持的终端（蓝牙） {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - 仅蓝牙
- **BBPOS WisePad 3** ❌ - 仅蓝牙  
- **Verifone P400** ❌ - 仅蓝牙

:::note 将来支持
蓝牙终端的支持将在 **2026 年第一季度** 加入 iOS 和 Android 应用程序。这将支持所有经过 Stripe 终端认证的读取器，包括 M2 和 WisePad 3。
:::

### 常见问题 {#common-issues}

<Accordion>
  <AccordionItem question="读取器无法连接">

  - 确保您正在使用 [支持的互联网连接终端](#supported-terminals-internet-connected)
  - 验证终端已连接到 WiFi/Ethernet 并在线
  - 检查您的 Stripe 账户是否启用了终端
  - 确认终端已在您的 Stripe 仪表板中注册

  </AccordionItem>

  <AccordionItem question="付款被拒绝">

  - 检查您的 Stripe 账户是否处于活动状态并良好
  - 验证正在使用的卡是否有效
  - 确保有足够的资金可用

  </AccordionItem>

  <AccordionItem question="电话订单切换未显示">

  - 验证 `WooCommerce > 设置 > 付款 > Stripe Terminal` 中的 **电话订单 (MOTO)** 设置已启用
  - 确保您连接了兼容的读取器（见 [支持的终端](#supported-terminals-internet-connected)）— 对于其他类型的读取器，该切换将被隐藏
  - 使用模拟器时不会出现切换

  </AccordionItem>

  <AccordionItem question="MOTO 付款失败并出现错误">

  - 确认 Stripe 已在您的账户上启用 MOTO 权限——如果尚未，联系 [Stripe 支持](https://support.stripe.com/contact)
  - 确保在读取器上正确输入了卡的详细信息
  - MOTO 付款可能有更严格的欺诈检查——确保卡有效且有足够的资金

  </AccordionItem>

  <AccordionItem question="SSL 证书错误">

  - Stripe 终端需要 HTTPS - 确保您的网站具有有效的 SSL 证书
  - 检查您的 SSL 证书是否正确配置

  </AccordionItem>
</Accordion>
