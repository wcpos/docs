:::warning 重要
在初始加载时，POS 将从 WooCommerce 获取默认的税务设置。一般来说，用户应保持与 WooCommerce 相同的设置，除非有特定的需求。与服务器的税务设置不同几乎肯定会导致 POS 和 WooCommerce 计算出不同的税务金额，这可能会导致订单同步的问题和差异。
:::

## 查看所有税率 {#view-all-tax-rates}

点击 **查看所有税率** 按钮以查看所有已下载到 POS 的税率。此操作显示：

- 税率名称
- 税率百分比
- 适用的地区/位置

这对于验证正确的税率是否已加载以及调试税务计算问题非常有用。

## 税务计算设置 {#tax-calculation-settings}

### 输入含税价格 {#prices-entered-with-tax}

确定产品价格是否包含税：

| 选项 | 描述 |
|--------|-------------|
| **是** | 产品价格包含税。POS 通过从总价格向后计算税 |
| **否** | 产品价格不包含税。POS 在输入价格上加上税 |

### 基于...计算税 {#calculate-tax-based-on}

确定用于税务计算的地址：

| 选项 | 描述 |
|--------|-------------|
| **顾客送货地址** | 税率基于订单将寄送的地点 |
| **顾客账单地址** | 税率基于顾客的账单地址 |
| **商店基本地址** | 税率基于您商店的物理位置 |

### 运费税类 {#shipping-tax-class}

确定运费如何征税：

| 选项 | 描述 |
|--------|-------------|
| **基于购物车商品的运费税类** | 运费税根据购物车中商品的税类计算 |
| **标准** | 运费按标准税率征税 |
| **减税率** | 运费按减税率征税 |
| **零税率** | 运费不征收税 |

### 四舍五入 {#rounding}

| 选项 | 描述 |
|--------|-------------|
| **在小计级别上四舍五入税** | 税是针对整个小计计算，然后四舍五入，而不是对每个单独的项目进行四舍五入 |

## 显示设置 {#display-settings}

### 在商店中显示价格 {#display-prices-in-the-shop}

| 选项 | 描述 |
|--------|-------------|
| **包括税** | 显示的价格包含税款 |
| **不包括税** | 显示的价格不包含税 |

### 在购物车和结账过程中显示价格 {#display-prices-during-cart-and-checkout}

| 选项 | 描述 |
|--------|-------------|
| **包括税** | 购物车和结账价格包含税 |
| **不包括税** | 购物车和结账价格不包含税 |

### 价格显示后缀 {#price-display-suffix}

出现在价格后面以指示税务状态的文本（例如，“含增值税”或“不含增值税”）。

### 显示税务总额 {#display-tax-totals}

| 选项 | 描述 |
|--------|-------------|
| **作为单一总额** | 所有税款合并为一条总额 |
| **逐项显示** | 每个税率作为单独的行项目显示 |

## 恢复服务器设置 {#restore-server-settings}

点击以从您的服务器获取并应用当前的 WooCommerce 税务设置。

:::tip 最佳实践
定期使用“恢复服务器设置”以保持您的 POS 税务配置与 WooCommerce 商店同步，尤其是在 WordPress 后台更改税务设置后。
:::
