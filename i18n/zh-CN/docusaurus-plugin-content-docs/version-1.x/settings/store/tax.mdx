:::warning 重要
在初始加载时，POS 将从 WooCommerce 获取默认税务设置。一般来说，用户应保持与 WooCommerce 相同的设置，除非有特定需求。与服务器的税务设置不同几乎肯定会导致 POS 和 WooCommerce 计算出不同的税额，从而产生不一致和订单同步问题。
:::

## 查看所有税率

点击 **查看所有税率** 按钮以查看已下载到 POS 的所有税率。这显示：

- 税率名称
- 税率百分比
- 适用地区/位置

这对于验证正确的税率是否已加载以及调试税款计算问题非常有用。

## 税款计算设置

### 包含税的价格

确定产品价格是否包含税：

| 选项 | 描述 |
|------|------|
| **是** | 产品价格包含税。POS 通过从总价格向后推算来计算税 |
| **否** | 产品价格不包含税。POS 在输入价格上加上税 |

### 根据以下内容计算税

确定用于税款计算的地址：

| 选项 | 描述 |
|------|------|
| **客户送货地址** | 根据订单将发往何处的税率 |
| **客户账单地址** | 根据客户的账单地址的税率 |
| **商店基础地址** | 根据您商店的实际位置的税率 |

### 运费税类别

确定运费如何征税：

| 选项 | 描述 |
|------|------|
| **根据购物车商品的运费税类别** | 根据购物车中商品的税种计算运费税 |
| **标准** | 运费按标准税率纳税 |
| **减免税率** | 运费按减免税率纳税 |
| **零税率** | 运费不收税 |

### 四舍五入

| 选项 | 描述 |
|------|------|
| **在小计级别四舍五入税** | 对整个小计计算税款，然后进行四舍五入，而不是对每个单独的条目进行税款四舍五入 |

## 显示设置

### 在商店中显示价格

| 选项 | 描述 |
|------|------|
| **包含税** | 显示的价格包括税额 |
| **不含税** | 显示的价格不包括税 |

### 在购物车和结账时显示价格

| 选项 | 描述 |
|------|------|
| **包含税** | 购物车和结账时价格包括税 |
| **不含税** | 购物车和结账时价格不包括税 |

### 价格显示后缀

显示在价格后以指示税务状态的文本（例如：“含税”或“未含税”）。

### 显示税款总额

| 选项 | 描述 |
|------|------|
| **作为一个总额** | 所有税款合并为一个总额行 |
| **逐项列出** | 每个税率显示为单独的行项目 |

## 恢复服务器设置

点击以从您的服务器获取并应用当前的 WooCommerce 税务设置。

:::tip 最佳实践
定期使用“恢复服务器设置”以保持您的 POS 税务配置与 WooCommerce 商店同步，特别是在您在 WordPress 管理中更改税务设置后。
:::
