:::warning 重要提示
在初始加载时，POS将从WooCommerce获取默认税收设置。通常，用户应保持与WooCommerce相同的设置，除非有特定需求。如果税收设置与服务器不同，POS和WooCommerce计算的税值可能会有差异，这可能导致订单同步问题。
:::

## 查看所有税率

点击**查看所有税率**按钮，查看已下载到POS的所有税率。这将显示：

- 税率名称
- 税率百分比
- 适用区域/地点

这对于验证加载的税率是否正确和调试税收计算问题很有帮助。

## 税收计算设置

### 输入含税价格

确定产品价格是否包含税：

| 选项 | 描述 |
|--------|-------------|
| **是** | 产品价格包含税。POS通过从总价格向后计算税额 |
| **否** | 产品价格不包含税。POS在输入价格的基础上添加税额 |

### 基于以下地址计算税收

确定用于税收计算的地址：

| 选项 | 描述 |
|--------|-------------|
| **客户送货地址** | 基于订单将要发送的地方的税率 |
| **客户账单地址** | 基于客户的账单地址的税率 |
| **商店基础地址** | 基于您商店的物理位置的税率 |

### 配送税类

确定如何对配送成本征税：

| 选项 | 描述 |
|--------|-------------|
| **基于购物车商品的配送税类** | 配送税根据购物车中商品的税类计算 |
| **标准** | 配送按标准税率征税 |
| **减免税率** | 配送按减免税率征税 |
| **零税率** | 配送成本不征税 |

### 四舍五入

| 选项 | 描述 |
|--------|-------------|
| **在小计级别四舍五入** | 对整个小计计算税额，然后四舍五入，而不是对每个单独的明细行项目四舍五入 |

## 显示设置

### 在商店中显示价格

| 选项 | 描述 |
|--------|-------------|
| **包含税** | 显示的价格包含税额 |
| **不含税** | 显示的价格不包含税额 |

### 在购物车和结账期间显示价格

| 选项 | 描述 |
|--------|-------------|
| **包含税** | 购物车和结账价格包含税额 |
| **不含税** | 购物车和结账价格不包含税额 |

### 价格显示后缀

出现在价格后的文本，用于指示税务状态（如"含增值税"或"不含税"）。

### 显示税收总额

| 选项 | 描述 |
|--------|-------------|
| **作为单一总额** | 所有税收合并为一个总计行 |
| **明细** | 每个税率显示为单独的明细项 |

## 恢复服务器设置

点击从您的服务器获取并应用当前的WooCommerce税收设置。

:::tip 最佳实践
定期使用"恢复服务器设置"，保持您的POS税收配置与WooCommerce商店同步，尤其是在WordPress管理中更改税收设置后。
:::
