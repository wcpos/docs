## 启用仅限 POS 的产品

此设置为您的产品添加可见性选项，允许您控制它们是否出现在您的在线商店、POS 或两者。

:::warning 性能影响
只有在您需要从在线商店或 POS 隐藏产品时，才启用此设置。启用后，它会为每个产品请求增加额外的数据库查找。
:::

有关如何使用此功能的详细信息，包括如何设置单个产品的可见性和批量编辑，请参见 [仅限 POS 的产品](/products/pos-only-products)。

## 启用小数数量

此设置允许客户和员工在将产品添加到购物车时输入小数数量。

**小数数量示例：**
- 0.25（四分之一单位）
- 1.5（一个半单位）
- 2.75（两个三分之四单位）

**适用于：**
- 按重量出售的产品（例如，0.5kg 咖啡豆）
- 部分服务（例如，1.5 小时咨询）
- 可按部分数量销售的散装产品

## 从客户电子邮件自动生成用户名

当启用时，WCPOS 将根据客户在结账时的电子邮件地址自动为新客户创建用户名。这简化了客户创建过程并确保一致的用户名格式。

## 默认 POS 客户

选择将分配给新 POS 订单的默认客户。常见选项包括：

- **访客**：订单不与任何用户帐户相关联
- **收银员**：订单与登录的收银员帐户相关联
- **进店客户**：订单与特定客户帐户相关联（您需要先创建此帐户）

## 使用收银员帐户

启用后，收银员的用户帐户将用作 POS 订单的默认客户，而不是所选的默认 POS 客户。

## 条形码字段

此设置决定了 WCPOS 在扫描条形码或使用条形码搜索功能时将搜索哪个产品字段。

:::warning 重要：需要清除和刷新
更改条形码字段设置后，您 **必须** 在 POS 中清除并刷新产品数据，以使更改生效。

**如何清除和刷新：**
1. 转到 POS 中的产品屏幕
2. **长按** 屏幕底部的同步图标 (↻)
3. 从上下文菜单中选择 **“清除并刷新”**
4. 等待产品以新条形码字段数据重新加载

这是必要的，因为 POS 在本地缓存产品数据，更改条形码字段设置不会自动更新缓存的数据。
:::

### 默认选项

**`_global_unique_id`** - 这是 WooCommerce 的全球唯一 ID 字段，在 WooCommerce 8.2（2024 年 7 月）中引入。它旨在存储 GTIN、UPC、EAN 或 ISBN 代码，并且是新安装的推荐选项。

### 其他常见选项

**`_sku`** - 产品 SKU（库存单位）字段。如果您不使用全球唯一 ID 字段，这是一个不错的备用选项。

### 第三方插件元键

如果您使用条形码管理插件，您可能需要使用其特定的元键：

| 插件类型 | 常见元键 | 描述 |
|----------|----------|------|
| EAN 插件 | `_ean`, `_ean13`, `_ean8` | 欧洲商品编号代码 |
| UPC 插件 | `_upc`, `_upc_code` | 通用产品代码 |
| GTIN 插件 | `_gtin`, `_gtin13`, `_gtin14` | 全球贸易项目编号 |
| 通用条形码 | `_barcode`, `_product_barcode` | 通用条形码字段 |
| WooCommerce 产品条形码 | `_barcode_text` | 流行的条形码插件 |
| ATUM 库存 | `_atum_barcode` | ATUM 库存管理 |

### 查找您的插件的元键

要查找您的条形码插件的正确元键：

1. 在 WP 管理中编辑一个产品
2. 使用您的插件添加一个条形码
3. 检查插件的文档以获取元键名称
4. 或检查数据库 `wp_postmeta` 表以查看创建了哪个元键

### 自定义元键

您还可以使用任何存储条形码数据的自定义元键。只需在此字段中输入元键名称（如果适用，包括下划线前缀）。

:::tip
更改此设置后，请测试您的条形码扫描，以确保它与您选择的字段正常工作。请记得在 POS 中清除并刷新您的产品数据，以使更改生效。
:::
