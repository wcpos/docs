## 启用仅限 POS 的产品 {#enable-pos-only-products}

此设置为您的产品添加可见性选项，允许您控制它们是否出现在您的在线商店、POS 或两者。

:::warning 性能影响
只有在您需要从在线商店或 POS 隐藏产品时，才启用此设置。启用后，它会为每个产品请求增加额外的数据库查找。
:::

有关如何使用此功能的详细信息，包括如何设置单个产品的可见性和批量编辑，请参见 [仅限 POS 的产品](/products/pos-only-products)。

## 启用小数数量 {#enable-decimal-quantities}

此设置允许客户和员工在将产品添加到购物车时输入小数数量。

**小数数量示例：**
- 0.25（四分之一单位）
- 1.5（一个半单位）
- 2.75（两个又四分之三单位）

**适用于：**
- 按重量出售的产品（例如，0.5kg 咖啡豆）
- 部分服务（例如，1.5 小时咨询）
- 可按部分数量出售的散装产品

## 删除订单时恢复库存 {#restore-stock-on-order-delete}

启用后，当 POS 订单被移入回收站或删除时，WCPOS 会自动恢复产品库存——因此取消一笔错误的销售会将商品退回库存。此设置 **默认开启**。

如果您以其他方式对账库存，并且不希望已删除的 POS 订单调整库存，请将其关闭。

## 从客户电子邮件自动生成用户名 {#automatically-generate-username-from-customer-email}

启用时，WCPOS 将根据顾客在结账时的电子邮件地址自动为新顾客创建用户名。这简化了顾客创建过程，并确保用户名格式一致。

## 默认 POS 客户 {#default-pos-customer}

选择将分配给新 POS 订单的默认客户。常见选项包括：

- **访客**：订单不与任何用户帐户关联
- **收银员**：订单与登录的收银员帐户关联
- **进店客户**：订单与特定客户帐户关联（需先创建此帐户）

## 使用收银员帐户 {#use-cashier-account}

启用后，收银员的用户帐户将被用作 POS 订单的默认客户，而不是选择的默认 POS 客户。

## 条形码字段 {#barcode-field}

此设置决定 WCPOS 在扫描条形码或使用条形码搜索功能时将搜索哪个产品字段。

<a id="default-option"></a>

:::note 默认元键
使用 **`_global_unique_id`**——WooCommerce 的全球唯一 ID 字段，在 WooCommerce 8.2（2024 年 7 月）中引入。它旨在存储 GTIN、UPC、EAN 或 ISBN 代码，并且是新安装的推荐选项。如果您不使用全球唯一 ID 字段，**`_sku`**（产品 SKU / 库存单位字段）是一个不错的替代选项。
:::

### 常见插件 → 元键 {#common-plugins-meta-key}

如果您使用条形码管理插件，您可能需要使用其特定的元键：

| 插件 / 字段类型 | 元键 |
|---|---|
| WooCommerce 全球唯一 ID *（默认，推荐）* | `_global_unique_id` |
| 产品 SKU *（备用）* | `_sku` |
| EAN 插件（欧洲商品编号） | `_ean`, `_ean13`, `_ean8` |
| UPC 插件（通用产品代码） | `_upc`, `_upc_code` |
| GTIN 插件（全球贸易项目编号） | `_gtin`, `_gtin13`, `_gtin14` |
| 通用条形码字段 | `_barcode`, `_product_barcode` |
| WooCommerce Product Barcodes | `_barcode_text` |
| ATUM Inventory | `_atum_barcode` |

<a id="custom-meta-keys"></a>

:::tip 自定义元键
您可以使用 **任何** 存储条形码数据的自定义元键。只需在此字段中输入元键名称（如果适用，包括下划线前缀）。
:::

### 查找您插件的元键 {#finding-your-plugins-meta-key}

如果您的插件未在上面列出，可按以下方式找到正确的元键：

<Steps>
  <StepCard number="1" title="为产品添加条形码">

在 WP Admin 中编辑产品，并使用您的插件添加条形码。

  </StepCard>
  <StepCard number="2" title="查看插件文档">

在插件的文档中查找它使用的元键名称。

  </StepCard>
  <StepCard number="3" title="或检查数据库" isLast>

检查数据库 `wp_postmeta` 表，查看为该产品创建了哪个元键。

  </StepCard>
</Steps>

:::warning 更改此设置后请清除并刷新
更改条形码字段设置后，您 **必须** 在 POS 中清除并刷新您的产品数据，以使更改生效。POS 会在本地缓存产品数据，更改条形码字段设置不会自动更新缓存的数据。

**如何清除和刷新：**

1. 转到 POS 中的产品屏幕
2. **长按** 屏幕底部的同步图标 (↻)
3. 从上下文菜单中选择 **“清除并刷新”**
4. 等待产品数据以新条形码字段重新加载

然后测试您的条形码扫描，以确认它与您选择的字段正常工作。
:::

## 隐私 {#privacy}

**允许匿名使用数据** 开关控制 WCPOS 是否收集匿名使用分析数据以帮助改进产品。这是真正的自愿加入/退出——绝不会收集任何客户数据、价格或店铺地址。使用该设置旁边的 **了解更多** 链接，查看具体收集和不收集哪些内容。

## 店铺税号 {#store-tax-ids}

添加企业税务标识符——VAT 号、ABN、GSTIN、EIN 或当地公司注册号——WCPOS 会将其打印在 POS 收据和发票上。WooCommerce 没有这些字段的内置选项，而许多地区需要不止一个。

有关完整指南，请参见 [店铺税号](/settings/wp-admin/store-tax-ids)。

## 客户税号 {#customer-tax-ids}

WCPOS 会在您的客户上保留一个税号字段，并在结账时将其复制到他们的订单中。由于不同插件以不同名称存储此字段，WCPOS 会检测您安装了哪个插件，并自动写入正确的字段。此处的 **税号** 部分还允许您在需要时覆盖字段名称。

有关完整指南，请参见 [客户税号](/settings/wp-admin/customer-tax-ids)。
