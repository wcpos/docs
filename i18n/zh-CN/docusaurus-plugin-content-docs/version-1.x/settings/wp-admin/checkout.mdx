结账设置页面（`WP Admin > POS > Settings > Checkout`）控制哪些支付网关显示在 POS 中、订单如何处理，以及 POS 销售会触发哪些通知电子邮件。

## 电子邮件通知 {#email-notifications}

页面顶部有三组电子邮件开关——**管理员电子邮件**、**客户电子邮件** 和 **收银员电子邮件**——它们控制 POS 订单会发送哪些 WooCommerce 通知电子邮件，独立于您的站点级 WooCommerce 电子邮件设置。

完整参考请参见 **[电子邮件通知](/settings/wp-admin/email-notifications)**，其中包括每封电子邮件的开关默认值、收银员收件人模型、常见场景以及开发者钩子。

## 支付网关 {#payment-gateways}

此部分列出了安装在您站点上的所有 WooCommerce 支付网关。每个网关都可以针对 POS 独立启用或禁用。

| 列 | 描述 |
|--------|-------------|
| **已启用** | 为 POS 结账打开或关闭该网关 |
| **网关** | 支付网关名称 |
| **网关 ID** | WooCommerce 网关标识符（对开发者有用） |

### 启用网关 {#enabling-a-gateway}

1. 在列表中找到该网关
2. 打开 **已启用** 开关
3. 该网关将在下次页面加载时出现在 POS 结账中

### 重新排序网关 {#reordering-gateways}

拖动网关以更改它们在 POS 结账中的显示顺序。当收银员打开结账弹窗时，默认选中第一个已启用的网关。

### POS 与在线商店 {#pos-vs-online-store}

此处的网关设置 **独立于** 您的 WooCommerce 支付设置。一个网关可以是：

- **为 POS 启用，在线禁用**——适用于不适合网店订单的现金支付
- **在线启用，为 POS 禁用**——适用于像 PayPal 这样不便于面对面销售的网关
- **两者都启用**——该网关在任何地方都可用

:::note
网关必须先在 `WP Admin > WooCommerce > Settings > Payments` 中安装并激活，然后才会出现在 POS 结账设置列表中。如果某个网关缺失，请检查该插件是否已安装且该网关已在 WooCommerce 中启用。
:::

## 订单状态 {#order-status}

为已完成的 POS 订单配置默认订单状态。默认情况下，由于在销售点已收取付款，POS 订单会被设置为 **已完成**。
