WCPOS 存储您产品、订单和客户的本地副本，以便 POS 可以快速工作并离线功能。偶尔，这些本地数据可能与您的 WooCommerce 商店不同步。清除所有本地数据将删除本地数据库并触发从服务器的新同步。

:::warning
这是一种最后手段。清除本地数据将删除保存在设备上的所有内容，并强制进行完全重新同步。根据您的商店规模，这可能需要一些时间。仅在其他选项（如正常的 [同步](/products/sync)）未解决问题时使用。
:::

## 何时使用 {#when-to-use-it}

您可能需要清除本地数据的情况包括：

- **同步问题** — 产品、订单或客户未正确更新
- **插件更新后** — 确保与新数据结构的兼容性
- **过时数据** — 价格、库存水平或其他信息显示过时

## 清除的内容 {#what-gets-cleared}

| 清除的内容 | 未清除的内容 |
|------------|---------------|
| 本地产品数据库 | 您的登录会话 |
| 本地订单数据库 | 服务器端数据 |
| 本地客户数据库 | WooCommerce 设置 |

您将**不会**被注销。您的 WooCommerce 商店数据不受影响 — 仅删除您设备上的本地副本。清除后将自动开始新同步。

## 如何清除本地数据 {#how-to-clear-local-data}

### 网站 {#web}

1. 点击右上角的 **用户菜单** 打开菜单
2. 点击 **清除所有本地数据**
3. 本地数据库将被删除，并将自动开始新的同步

### 桌面 {#desktop}

1. 在顶部菜单栏中打开 **WCPOS** 应用程序菜单
2. 悬停在 **高级** 上
3. 点击 **清除应用程序数据**
4. 本地数据库将被删除，并将自动开始新的同步

:::note
桌面应用使用 SQLite 数据库，而网页应用使用 IndexedDB，但两种平台的行为相同。
:::
