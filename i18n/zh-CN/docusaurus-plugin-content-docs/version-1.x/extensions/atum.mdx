ZH: # WCPOS ATUM Integration

ZH: Integrates [ATUM Multi-Inventory](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/) with [WCPOS Pro](/getting-started/pro-license), enabling location-based inventory, pricing, and SKUs at the Point of Sale.

ZH: ATUM Multi-Inventory lets you split a product's stock across multiple inventory locations — warehouses, retail stores, and so on. This plugin connects those ATUM locations to WCPOS Pro [stores](/stores) so each POS terminal sees the correct stock levels, prices, and SKUs for its physical location.

## 已翻译: Installation {#installation}

<Steps>
  <StepCard number="1" title="ZH: Install ATUM and Multi-Inventory">

ZH: Install [ATUM Inventory Management](https://wordpress.org/plugins/atum-stock-manager-for-woocommerce/) and the [ATUM Multi-Inventory add-on](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/). Configure your inventory locations in ATUM.

  </StepCard>
  <StepCard number="2" title="ZH: Install WCPOS ATUM Integration">

ZH: Install from `WP Admin > POS > Settings > Extensions`, or download the latest release from the [GitHub releases page](https://github.com/wcpos/wcpos-atum/releases) and upload via `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="ZH: Map stores to ATUM locations" isLast>

  </StepCard>
</Steps>

## 已翻译: Store Configuration {#store-configuration}

ZH: The plugin adds an **ATUM Inventory** section to the WCPOS Pro store editor sidebar with three settings per store:

- ZH: **Inventory Location** — which ATUM location this store pulls stock from.
- ZH: **Pricing Source** — where product prices come from:
  - ZH: *Default* — standard WooCommerce prices
  - ZH: *WCPOS Pro* — per-store pricing set in WCPOS Pro
  - ZH: *ATUM* — location-specific prices from the ATUM inventory
- ZH: **SKU Override** — optionally use location-specific SKUs from ATUM instead of the product's main SKU.

## 已翻译: POS Behavior {#pos-behavior}

ZH: When a store has an ATUM location assigned, the product data served to the POS is automatically adjusted:

- ZH: **Stock quantities** reflect the specific location's inventory, not the aggregate WooCommerce stock.
- ZH: **Stock status** is recalculated based on the location's quantity.
- ZH: **Prices** come from the configured pricing source.
- ZH: **SKUs** are swapped to the ATUM location SKU if override is enabled.

ZH: All adjustments happen transparently through the WCPOS REST API — no changes are needed on the POS app side. Product edits made from the POS are also written back to the mapped ATUM inventory row; see [Product Edit Write-Back](#product-edit-write-back) below.

## 已翻译: Stock Management {#stock-management}

ZH: For POS orders placed at stores with a mapped ATUM location, the plugin lets ATUM's native stock deduction flow handle the write — but steers it to the correct location:

1. ZH: **REST payload injection.** When the POS creates or updates an order, the plugin injects a `mi_inventories` entry onto each line item so ATUM knows which location to draw from. Without this, ATUM would fall back to the main inventory.
2. ZH: **Location-scoped inventory filter.** The plugin filters ATUM's candidate inventory list to only those linked to the store's mapped location term, ensuring the right one is picked on both reduction and restoration.

ZH: ATUM itself performs the actual stock change on order and refund, writing rows to `atum_inventory_orders` with the real `order_id` — preserving ATUM's audit trail.

## 已翻译: Product Edit Write-Back {#product-edit-write-back}

ZH: When a cashier or manager edits a product or variation from the POS, the changes sync back to the mapped ATUM inventory row for that store's location — not just the main WooCommerce product. This keeps each location's stock, price, and SKU in sync with ATUM without manual updates in `WP Admin`.

ZH: The write-back is triggered on WCPOS product and variation REST updates (`POST`, `PUT`, `PATCH` to `/wcpos/v1/products/...`) that include a `store_id`. The plugin looks up the store's mapped ATUM location and updates only the inventory row for that location — other locations are untouched.

### 已翻译: What Syncs {#what-syncs}

ZH: The write-back respects each store's configuration so ATUM data only changes when the store actually owns that data:

| ZH: Field | ZH: When it syncs | ZH:
| --- | --- |
| ZH: **Stock quantity** | ZH: Always — every store with a mapped ATUM location keeps its location row in sync. | ZH:
| ZH: **Regular price / Sale price / Price** | ZH: Only when the store's **Pricing Source** is set to *ATUM*. | ZH:
| ZH: **SKU** | ZH: Only when **SKU Override** is enabled for the store. | ZH:

ZH: If the store uses *Default* or *WCPOS Pro* pricing, ATUM price fields are left alone so ATUM continues to serve as a reference price rather than the source of truth. The same applies to SKUs when override is off.

### 已翻译: What Doesn't Trigger Write-Back {#what-doesnt-trigger-write-back}

- ZH: Product creation (only updates write back — new products fall through to WooCommerce's normal save path).
- ZH: Requests without a `store_id` — the POS has to tell the plugin which location to write to.
- ZH: Stores without a mapped ATUM location.
- ZH: Products without an existing ATUM inventory row for the store's location — the plugin will not create new inventory rows, only update existing ones.
