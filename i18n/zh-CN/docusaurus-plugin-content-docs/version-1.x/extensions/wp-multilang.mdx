# WCPOS WP Multilang 集成 {#wcpos-wp-multilang}

将 [WP Multilang](https://wordpress.org/plugins/wp-multilang/) 与 WCPOS 集成，使 POS 只提供单一语言的产品。与 Polylang 或 WPML 不同，WP Multilang 将所有翻译存储在同一篇文章中——此插件可确保 WCPOS 读取正确的翻译，并且不会在搜索结果中重复显示条目。

## 功能 {#what-it-does}

- 将 WCPOS 产品和变体 REST 响应筛选为单一 WP Multilang 语言。
- 拦截 WCPOS **fast-sync** 路由，使只有当前语言的内容到达客户端。
- 免费版安装使用 WP Multilang 默认语言。
- Pro 版安装可在店铺编辑器中为每个店铺固定一种语言。

## 安装 {#installation}

<Steps>
  <StepCard number="1" title="安装 WP Multilang">

安装 [WP Multilang](https://wordpress.org/plugins/wp-multilang/) 并配置站点语言，至少将一种语言设为默认语言。

  </StepCard>
  <StepCard number="2" title="安装 WCPOS WP Multilang">

从 `WP Admin > POS > 设置 > 扩展` 安装，或从 [GitHub 发布页面](https://github.com/wcpos/wcpos-wp-multilang/releases) 下载最新版本，并通过 `插件 > 新增 > 上传插件` 上传。

  </StepCard>
  <StepCard number="3" title="(Pro) 为每个店铺固定语言" isLast>

在 WCPOS Pro 中，在 `POS > 商店` 下编辑店铺，并从 **语言** 侧边栏部分选择其语言。保留为 *默认* 可使用 WP Multilang 的默认语言。

  </StepCard>
</Steps>
