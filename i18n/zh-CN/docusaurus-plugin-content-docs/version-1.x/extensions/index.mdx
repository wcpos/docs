WCPOS 支持通过扩展为您的销售点添加新功能。扩展目录可让您浏览可用扩展、直接从 POS 设置中安装扩展并管理更新。

:::info 专业版功能
安装和管理扩展需要 [WCPOS Pro](/getting-started/pro-license)。免费版会显示扩展目录,但安装和激活控件将被禁用。
:::

## 浏览扩展 {#browsing-extensions}

从 `POS Settings > Extensions`（在某些版本中也标记为 **插件**）打开扩展目录。

该目录以卡片网格形式显示可用的扩展。每张卡片显示：

- **图标**（如果扩展未提供图标，则显示拼图块作为替代）
- **名称和版本**
- **描述**
- **分类徽章**
- **状态** — 已启用、已停用、有可用更新或未安装

### 筛选与搜索 {#filtering-and-search}

使用顶部的**分类胶囊按钮**按类别筛选扩展。您还可以使用**搜索框**按名称、描述或标签查找扩展。

## 安装扩展 {#installing-an-extension}

1. 打开 `POS Settings > Extensions`。
2. 找到所需的扩展，点击**安装**。
3. 扩展将通过 WordPress 插件安装程序进行下载和安装。
4. 安装完成后，点击**启用**即可使用。

在后台，WCPOS 使用 WordPress 原生的 `Plugin_Upgrader` 处理安装，因此扩展的安装流程与任何 WordPress 插件相同。

## 启用与停用 {#activating-and-deactivating}

每个已安装的扩展都在其卡片上提供**启用**和**停用**按钮。

- **激活**会启用扩展，使其可在 POS 中运行。
- **停用**会禁用扩展但不会卸载它。扩展文件仍保留在你的服务器上，可随时重新激活。

## 更新扩展 {#updating-extensions}

当已安装的扩展有新版本可用时，卡片会显示**有可用更新**徽章和**更新**按钮。

### 自动更新 {#auto-updates}

从目录安装的扩展**默认启用自动更新**。你可以在目录中的扩展卡片上单独打开或关闭其自动更新。当自动更新开启时，WordPress 会自动应用新版本，与启用了自动更新的插件行为一致。

## 故障排查 {#troubleshooting}

### 按钮上出现“需要 Pro”提示 {#requires-pro-message-on-buttons}

在 WCPOS 免费版中，安装、激活和更新按钮均被禁用。升级到 [WCPOS Pro](/getting-started/pro-license) 即可管理扩展。

### 扩展安装失败 {#extension-fails-to-install}

- 请检查 WordPress 服务器是否对 `wp-content/plugins` 目录具有写入权限。
- 确认你的服务器可以发起对外的 HTTPS 请求（部分主机会阻止外部下载）。
- 在 `WP Admin > POS > Support > Logs` 中查看错误详情。

### 安装后扩展未显示 {#extension-not-appearing-after-install}

- 刷新 POS — 扩展列表的缓存时间最长为 12 小时。
- 确认扩展已激活（已安装但未激活的扩展不会运行）。

### 目录无法加载 {#catalog-not-loading}

扩展目录从远程源获取，并在本地缓存 12 小时。如果目录无法加载：

- 检查服务器的网络连接。
- 等待缓存过期后重试，或清除服务器的瞬态缓存。

## 面向开发者 {#for-developers}

### 创建 POS 扩展 {#creating-a-pos-extension}

WCPOS 扩展是一个标准的 WordPress 插件，通过 WCPOS 钩子和 API 与 POS 集成。创建方法：

1. **从 WordPress 插件起步。** 您的扩展需要标准的插件头部和入口文件，与任何 WooCommerce 或 WordPress 插件一样。
2. **与 WCPOS 集成。** 使用 WCPOS 提供的钩子和过滤器，为 POS 界面或后端添加功能。
3. **在 GitHub 上托管发布版本。** 扩展目录使用 GitHub Releases 跟踪版本并交付更新。

### 提交至扩展目录 {#submitting-to-the-directory}

扩展目录在 [`wcpos/extensions`](https://github.com/wcpos/extensions) GitHub 仓库中维护。要将您的扩展列入目录：

1. 在仓库的 README 中查阅目录格式和元数据要求。
2. 提交拉取请求，将您扩展的元数据添加到 `catalog.json`。
3. 合并后，您的扩展将出现在目录中，供所有 WCPOS Pro 用户使用。

### GitHub 发布约定 {#github-release-conventions}

更新生命周期依赖于 GitHub Releases：

- **标记版本** 使用语义化版本号（例如 `v1.0.0`、`v1.2.3`）。
- **附加插件压缩包** 作为发布资源——这是用户安装或更新时下载的文件。
- **发布该版本**（非草稿），以便目录能够检测到。

发布新版本后，已安装您扩展的用户将在其扩展目录中看到可用更新。如果启用了自动更新，更新将自动应用。

有关目录架构和提交流程的完整说明，请参阅 [`wcpos/extensions`](https://github.com/wcpos/extensions) 代码仓库。
