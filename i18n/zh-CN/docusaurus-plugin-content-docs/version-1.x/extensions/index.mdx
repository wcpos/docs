WCPOS 支持通过扩展为销售点添加新功能。扩展目录可用于浏览可用扩展、直接从 POS 设置中进行安装，以及管理更新。

:::info 专业版功能
安装和管理扩展需要 [WCPOS 专业版](/getting-started/pro-license)。免费版会显示扩展目录，但会禁用安装和启用控件。
:::

## 浏览扩展 {#browsing-extensions}

通过 `POS Settings > Extensions`（在某些版本中也标注为 **插件**）打开扩展目录。

该目录以卡片网格的形式展示可用扩展。每张卡片显示：

- **图标**（如果扩展未提供图标，则显示拼图块作为占位图）
- **名称和版本**
- **描述**
- **分类标签**
- **状态** — 已启用、未启用、有可用更新或未安装

### 筛选与搜索 {#filtering-and-search}

使用顶部的**分类胶囊按钮**按类别筛选扩展。你也可以使用**搜索栏**按名称、描述或标签查找扩展。

## 安装扩展 {#installing-an-extension}

1. 打开 `POS Settings > Extensions`。
2. 找到所需的扩展并点击**安装**。
3. 扩展将通过 WordPress 插件安装程序进行下载和安装。
4. 安装完成后,点击**启用**即可使用。

在底层,WCPOS 使用 WordPress 原生的 `Plugin_Upgrader` 处理安装过程,因此扩展的安装流程与其他 WordPress 插件完全一致。

## 启用与停用 {#activating-and-deactivating}

每个已安装的扩展卡片上都有**启用**和**停用**按钮。

- **激活**会启用该扩展，使其可在 POS 中运行。
- **停用**会禁用该扩展，但不会卸载它。扩展文件仍保留在您的服务器上，随时可以重新激活。

## 更新扩展 {#updating-extensions}

当已安装的扩展有新版本可用时，扩展卡片会显示**有可用更新**徽章和**更新**按钮。

### 自动更新 {#auto-updates}

从扩展目录安装的扩展**默认启用自动更新**。您可以在目录中的扩展卡片上为每个扩展单独开启或关闭自动更新。开启自动更新后，WordPress 会像处理已启用自动更新的插件一样自动应用新版本。

## 故障排查 {#troubleshooting}

### 按钮上显示“需要 Pro”提示 {#requires-pro-message-on-buttons}

在 WCPOS 免费版中，安装、激活和更新按钮均被禁用。升级到 [WCPOS Pro](/getting-started/pro-license) 即可管理扩展。

### 扩展安装失败 {#extension-fails-to-install}

- 请检查您的 WordPress 服务器是否对 `wp-content/plugins` 目录具有写入权限。
- 请确认您的服务器可以发出对外的 HTTPS 请求（部分主机会阻止外部下载）。
- 在 `WP Admin > POS > Support > Logs` 中查看错误详情。

### 安装后扩展未显示 {#extension-not-appearing-after-install}

- 刷新 POS——扩展列表的缓存最长可达 12 小时。
- 确认扩展已启用（已安装但未启用的扩展不会运行）。

### 目录无法加载 {#catalog-not-loading}

扩展目录从远程源获取，并在本地缓存 12 小时。如果目录无法加载：

- 检查服务器的互联网连接。
- 等待缓存过期后重试，或清除服务器的瞬态缓存。

## 面向开发者 {#for-developers}

### 创建 POS 扩展 {#creating-a-pos-extension}

WCPOS 扩展是一个标准的 WordPress 插件，通过 WCPOS 钩子和 API 与 POS 集成。创建步骤如下：

1. **从 WordPress 插件开始。**与任何 WooCommerce 或 WordPress 插件一样，您的扩展需要标准的插件头部信息和入口文件。
2. **与 WCPOS 集成。**使用 WCPOS 提供的钩子和过滤器，为 POS 界面或后端添加功能。
3. **在 GitHub 上托管发行版本。**扩展目录使用 GitHub Releases 来跟踪版本并交付更新。

### 提交至目录 {#submitting-to-the-directory}

扩展目录在 [`wcpos/extensions`](https://github.com/wcpos/extensions) GitHub 仓库中维护。要将您的扩展列入目录：

1. 请查阅仓库 README 中的目录格式和元数据要求。
2. 提交一个 pull request，将您扩展的元数据添加到 `catalog.json`。
3. 合并后，您的扩展将显示在目录中，供所有 WCPOS Pro 用户使用。

### GitHub 发行版本约定 {#github-release-conventions}

更新生命周期依赖于 GitHub Releases：

- **使用语义化版本号标记版本**（例如 `v1.0.0`、`v1.2.3`）。
- **附加插件压缩包**作为发行版本资源——用户安装或更新时下载的就是该文件。
- **发布版本**（非草稿），以便目录能够检测到。

当你发布新版本时，已安装你扩展的用户将在其扩展目录中看到可用更新。如果启用了自动更新，更新将自动应用。

有关目录架构和提交流程的完整详情，请参见 [`wcpos/extensions`](https://github.com/wcpos/extensions) 仓库。
