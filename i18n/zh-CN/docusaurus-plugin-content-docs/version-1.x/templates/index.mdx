WCPOS 包含一套完整的模板系统，可用于收据、发票、礼品收据、装箱单、厨房票据等，全部在 **WP Admin → POS → Templates** 中管理。

:::tip 只想更改收据？
请从 **[自定义收据](/templates/customise)** 开始。该页面会介绍三种最简单的方法：选择另一个内置模板、让 AI 调整某个模板，或手动编辑。
:::

## 模板引擎 {#template-types}

WCPOS 支持三种模板引擎。创建模板或在模板库卡片上点击 **使用模板** 时会选择引擎，并且 **之后不能更改**。如果要切换引擎，请新建一个模板。

<FeatureGrid>
  <FeatureCard icon="code" title="无逻辑 HTML（推荐）">
    在普通 HTML 中使用 Mustache 风格的占位符 <code>{"{{variable}}"}</code>。它在客户端渲染，因此收据 **可离线工作**。适用于浏览器打印和 PDF 收据。
  </FeatureCard>
  <FeatureCard icon="print" title="热敏 XML">
    XML 模板会同时生成屏幕预览和热敏收据打印机的 **ESC/POS 命令**。同一个模板产生两种输出。适用于 Epson、Star 和其他热敏打印机。
  </FeatureCard>
  <FeatureCard icon="terminal" title="旧版 PHP">
    使用 WooCommerce 函数的服务器端 PHP 模板。保留它是为了兼容现有的 <code>yourtheme/woocommerce-pos/receipt.php</code> 覆盖。渲染时需要服务器连接。
  </FeatureCard>
</FeatureGrid>

## 模板库 {#template-gallery}

模板库随附可直接使用或自定义的现成模板。可按 **类别**（收据、发票、礼品收据、厨房票据、报价 / 采购订单）、**格式**（HTML / ESC/POS）和 **方向**（LTR / RTL）筛选，并使用店铺真实数据预览。每张卡片底部都有 **预览** 链接和主要按钮 **使用模板**；点击后会在 *您的模板* 中创建一个可编辑副本。

| 模板 | 引擎 | 描述 |
|----------|--------|-------------|
| 标准收据 | HTML | 默认收据：徽标、店铺身份、商品、合计、付款 |
| 标准收据（RTL） | HTML | 阿拉伯语、希伯来语、波斯语、乌尔都语的从右到左版本 |
| 极简 / 现代 | HTML | 与标准版相同的核心信息，占用更少垂直空间 |
| 详细收据 | HTML | 完整税务发票：SKU 列、单价、按税率拆分的税额、地址 |
| 礼品收据 | HTML | 仅商品，隐藏价格。包含礼品留言和退货政策 |
| 发票 | HTML | A4/Letter 整页发票，可为未付款订单显示可选的“如何付款”面板 |
| 装箱单 | HTML | 仓库辅助文档：商品 + 数量、收货地址、无价格 |
| 报价 / 估价 | HTML | 售前文档，包含价格和条款，无付款部分 |
| 窄版收据 | HTML | 适用于窄纸或支持 HTML 的热敏打印机的等宽收据 |
| 简单热敏收据（58 mm） | 热敏 | 干净的 58 mm 热敏布局 |
| 简单热敏收据（80 mm） | 热敏 | 干净的 80 mm 热敏布局，最常用 |
| 简单热敏收据 80 mm（RTL） | 热敏 | 80 mm 的 RTL 对应版本。需要支持阿拉伯代码页（CP864 / Windows-1256）的打印机 |
| 详细热敏收据（58 mm） | 热敏 | 完整 58 mm：地址、税额拆分、退款、付款、条款、条码 |
| 详细热敏收据（80 mm） | 热敏 | 完整 80 mm：地址、税额拆分、退款、付款、条款、条码 |
| 厨房票据 | 热敏 | 仅商品，大字号，无价格；用于备餐工位 |

### 第三方文档模板 {#default-templates}

如果启用了 WP Overnight 的 **PDF Invoices & Packing Slips for WooCommerce**，WCPOS 还会在收据模板列表中显示 **发票（WP Overnight）** 和 **装箱单（WP Overnight）**。这些服务器渲染的 HTML 模板会委托给 WP Overnight 的文档 API，复用该插件中配置的发票和装箱单编号、布局、品牌、法律/税务字段以及模板自定义。

它们只在 WP Overnight 插件处于启用状态时可用，并且渲染需要服务器连接。需要离线打印时，请使用 WCPOS 随附的 HTML 或热敏模板。

### 自适应税费显示 {#custom-templates}

大多数内置模板（标准、标准 RTL、极简/现代、窄版、发票、报价、简单热敏变体）**会自动适配 WooCommerce 的税费显示设置**：

- **含税店铺**（欧盟/英国/澳大利亚）会显示含税总价和“已含税”行。
- **不含税店铺**（美国/加拿大）会显示净价，并在单独一行添加税额。
- 总计始终显示实际收取的含税总额。

**详细** 系列是正式税务发票，无论店铺设置如何，都会始终以不含税行和明确拆分来逐项列出税费。

## 工作方式 {#how-it-works}

1. **添加模板** — 在模板库卡片上点击 **使用模板**，或从零开始创建自己的模板。
2. **自定义模板** — 使用带实时预览的应用内编辑器，或将其粘贴到 ChatGPT / Claude 并说明想要的更改。
3. **启用模板** — 在 *您的模板* 中打开 **启用** 开关。每个启用的收据模板都会出现在收银台收据屏幕的下拉菜单中。
4. **打印或显示** — 结账后收据会打开，并提供打印、通过电子邮件发送或在屏幕上查看的选项。

模板会根据标准化的 [收据数据载荷](/templates/receipt-data) 渲染，其中包含店铺信息、商品行、合计、税费、付款、退款等部分。所有货币字段都包含预格式化的 `_display` 变体，例如 `$29.99` 而不是 `29.99`。

## Pro 功能 {#pro-features}

使用 [WCPOS Pro](/getting-started/pro-license)，每个店铺都可以拥有自己的模板分配、品牌信息（徽标、地址、联系方式）和模板顺序。这对于每家门店都要打印自己信头的多地点运营很有用。
