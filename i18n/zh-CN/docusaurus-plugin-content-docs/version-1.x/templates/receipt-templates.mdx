收据模板编辑器允许您自定义从 POS 系统打印的收据的外观和内容。您可以通过以下路径访问模板编辑器：`WP Admin > POS > Templates`

## 概述 {#overview}

模板系统提供两种类型的模板：

- **默认模板**：从您的插件和主题文件中自动检测。这些不能被删除，但可以复制以创建自定义版本。
- **自定义模板**：用户创建的模板，您可以完全自定义、编辑和管理。

## 访问模板编辑器 {#accessing-the-template-editor}

1. 导航到您的 WordPress 管理仪表板
2. 进入 **POS > Templates**
3. 您将看到可用模板的列表，分为两个部分：
   - **默认模板**（来自插件/主题文件）
   - **自定义模板**（您的自定义创建）

## 默认模板 {#default-templates}

系统自动从三个来源检测收据模板：

### 模板来源 {#template-sources}

| 来源           | 描述                                         | 状态                               |
| -------------- | -------------------------------------------- | ---------------------------------- |
| **插件**       | 包含在 WCPOS 中的核心收据模板              | 始终可用                           |
| **Pro 插件**   | 来自 WCPOS Pro 的增强模板                  | 在激活许可证下可用                |
| **主题**       | 来自您活动主题的自定义模板                | 如果主题包含模板，则可用          |

### 模板操作 {#template-actions}

对于每个默认模板，您可以：

- **预览**：查看模板使用示例数据的样子
- **复制**：创建您可以编辑的自定义副本
- **激活**：设置为您的 POS 的活动模板

## 创建自定义模板 {#creating-custom-templates}

### 方法 1：从默认模板复制 {#method-1-copy-from-default-template}

1. 在 **默认模板** 部分，找到您想要自定义的模板
2. 单击 **复制** 按钮
3. 输入您的自定义模板的名称
4. 单击 **复制模板**
5. 编辑新创建的自定义模板

### 方法 2：从头创建 {#method-2-create-from-scratch}

1. 单击页面顶部的 **添加新模板**
2. 输入模板名称
3. 选择 **收据** 作为模板类型
4. 开始在编辑器中编写您的模板

## 模板编辑器功能 {#template-editor-features}

模板编辑器提供：

### 代码编辑器 {#code-editor}

- **语法高亮**：PHP 语法高亮，使代码更易读
- **行号**：便于导航和调试
- **全屏编辑**：无干扰的编辑环境

### 模板设置面板 {#template-settings-panel}

- **语言**：设置为 PHP 以确保正确的语法高亮
- **模板类型**：选择收据或报告模板
- **状态**：发布/草稿状态管理
- **可见性**：控制模板的可见性

### 模板操作 {#template-actions}

- **预览**：查看模板与示例数据的渲染效果
- **设为活动**：将此模板设为您的 POS 默认模板
- **更新**：保存您的更改
- **移动到垃圾箱**：删除自定义模板

## 模板结构 {#template-structure}

收据模板是生成 HTML 输出的 PHP 文件。以下是基本结构：

```php
<?php
/**
 * Custom Receipt Template
 */

if ( ! defined( 'ABSPATH' ) ) {
    exit; // Exit if accessed directly
}
?>
<!DOCTYPE html>
<html <?php language_attributes(); ?>>
<head>
    <meta charset="<?php bloginfo( 'charset' ); ?>">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <style>
        /* Your custom CSS styles */
        body { font-family: sans-serif; font-size: 14px; }
        .receipt-header { text-align: center; margin-bottom: 20px; }
        /* Add more styles as needed */
    </style>
</head>
<body>
    <!-- Your receipt content -->
    <div class="receipt-header">
        <h1><?php echo get_bloginfo( 'name' ); ?></h1>
        <p><?php echo get_bloginfo( 'description' ); ?></p>
    </div>

    <!-- Order details, items, totals, etc. -->

</body>
</html>
```

## 可用的模板变量 {#available-template-variables}

在创建自定义模板时，您可以访问各种 WordPress 和 WooCommerce 函数和数据：

### 商店信息 {#store-information}

- `get_bloginfo( 'name' )` - 商店名称
- `get_bloginfo( 'description' )` - 商店标语
- `get_bloginfo( 'url' )` - 商店网址

### 订单数据 {#order-data}

当前订单数据可通过标准的 WooCommerce 订单函数在模板渲染时访问。

### WCPOS 函数 {#wcpos-functions}

您可以使用以 `wcpos_` 为前缀的 WCPOS 特定函数访问与 POS 相关的数据和功能。

## 最佳实践 {#best-practices}

### 模板开发 {#template-development}

1. **从复制开始**：始终从复制默认模板开始，而不是从头创建
2. **完全测试**：使用预览功能在激活前测试您的模板
3. **保持备份**：导出您的自定义模板或保留代码的副本
4. **移动友好**：确保您的模板在不同屏幕尺寸上工作良好

### 代码质量 {#code-quality}

1. **验证 HTML**：确保您的 HTML 结构良好
2. **输出转义**：使用 WordPress 转义函数以提高安全性
3. **错误处理**：在 PHP 代码中包括适当的错误检查
4. **注释**：为将来的参考记录您的自定义内容

### 性能 {#performance}

1. **优化 CSS**：保持样式简洁高效
2. **减少 PHP**：避免在模板中使用复杂逻辑
3. **测试打印**：验证模板在您的收据打印机上工作正常

### 获取帮助 {#getting-help}

如果您在模板自定义中遇到问题：

1. 检查模板预览以获取错误消息
2. 查看 WordPress 调试日志以查找 PHP 错误
3. 首先使用默认模板的副本进行测试
4. **AI 协助**：现代 AI 工具如 [Gemini](https://gemini.google.com)、ChatGPT 和 Claude 非常适合帮助处理这些简单的 WordPress 模板。您可以粘贴您的模板代码，并描述您想实现的目标。
5. 查阅 WCPOS 文档以获取高级自定义帮助

## 高级自定义 {#advanced-customisation}

对于需要更高控制权的高级用户：

### 主题集成 {#theme-integration}

您可以通过将自定义 `receipt.php` 文件放入主题中来覆盖默认模板：

```
your-theme/woocommerce-pos/receipt.php
```

### 钩子和过滤器 {#hooks-and-filters}

WCPOS 提供各种钩子和过滤器以进行模板自定义。请查阅开发者文档以获取可用选项。

### 自定义字段 {#custom-fields}

您可以使用适当的 WordPress 和 WooCommerce 函数在收据模板中包含自定义订单字段、产品属性和其他元数据。
