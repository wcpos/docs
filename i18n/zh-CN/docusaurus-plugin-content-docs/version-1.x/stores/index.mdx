:::info 专业版功能
多店铺功能需要 [WCPOS Pro](/getting-started/pro-license)。
:::

WCPOS Pro 允许您在单个 WooCommerce 安装中创建独立的店铺位置。每个店铺可以拥有自己的地址、标志和设置——适用于在多个实体店铺运营的场景。

## 店铺提供的功能 {#what-stores-provide}

每个店铺可以拥有：

- **店铺名称和地址** ——用于收据和报表
- **店铺标志** ——显示在打印收据上
- **税率** ——不同位置可能有不同的税务要求
- **收银员分配** — 跟踪哪些收银员在哪家店铺工作

## 设置店铺 {#setting-up-stores}

激活 Pro 后，WP Admin 的 POS 下会新增一个**店铺**子菜单。在此可以创建每家店铺，设置地址、Logo、税号以及允许登录的收银员——完整的分步操作指南请参阅**[设置店铺](/stores/setup)**。

收银员登录 POS 时，需要选择所在的店铺。这将决定小票上显示的店铺信息以及报表所应用的店铺筛选条件。

## 按店铺筛选 {#filtering-by-store}

配置多家店铺后，店铺筛选器会出现在 POS 的各个模块中：

- **[订单](/orders)** — 按店铺位置筛选订单历史
- **[报表](/reports)** — 为指定店铺生成日结报表
- **[小票](/pos/checkout/receipts)** — 小票会自动显示对应店铺的地址和 Logo

## 店铺专属定价 {#store-specific-pricing}

WCPOS Pro 支持**为不同店铺设置不同价格**。可以按店铺分别为商品和变体设置常规价格和促销价格。

收银员在特定店铺操作时，POS 会自动使用该店铺的定价。如果未设置店铺专属价格，则使用 WooCommerce 的默认价格。

## 库存管理 {#inventory-management}

默认情况下，所有店铺共享相同的 WooCommerce 库存水平。如需按位置跟踪库存，WCPOS Pro 可与 [ATUM Inventory Management](https://www.atum.org/) 插件集成：

1. 在 WooCommerce 站点上安装并配置 ATUM 插件
2. 在 ATUM 中创建库存位置
3. 在 WCPOS Pro 中，将每个店铺关联到一个 ATUM 库存位置
4. 下单时，库存水平将根据对应位置自动更新

### 替代方案：WordPress 多站点 {#alternative-wordpress-multisite}

如果不使用 ATUM 而需要完全独立的库存，可以使用 [WordPress 多站点](https://developer.wordpress.org/advanced-administration/multisite/create-network/)：

1. 为每个店铺位置创建一个独立的 WordPress 站点
2. 在每个站点上安装 WooCommerce 和 WCPOS
3. 每个站点维护各自独立的产品目录和库存水平

这种方式可实现完全隔离，但需要管理多个 WooCommerce 安装。

## 已知限制 {#known-limitations}

- **单一 WooCommerce 实例** — 店铺是同一个 WooCommerce 安装中的不同位置，而非独立的商城
- **不支持按店铺设置商品可见性** — 所有店铺共享相同的商品目录
