- Opent een dialoogvenster om een klantnotitie aan de bestelling toe te voegen.
- Klantnotities zijn zichtbaar voor de klant en verschijnen standaard op het bonnetje, in het gedeelte Mijn account van de klant en in `WP Admin > WooCommerce > Orders > Order` onder het verzendadres.

:::note
Bestelnotities zijn zichtbaar voor klanten.
:::

- Opent een modal om aanvullende metadata voor de bestelling te beheren.
- Functies zijn onder andere:
  - De valuta van de bestelling wijzigen.
  - Een transactie-ID toevoegen (bijv. van een externe betaalterminal).
  - Aangepaste metadata aan de bestelling toevoegen.

:::note
Er worden in de loop van de tijd aanvullende functies aan deze modal toegevoegd, waaronder integratie met andere WooCommerce-plugins.
:::

### Wat is metadata in WooCommerce? {#what-is-meta-data-in-woocommerce}

Metadata biedt extra informatie over bestellingen en is toegankelijk via de WooCommerce REST API. Ontwikkelaars en plugins gebruiken deze gegevens om functionaliteit uit te breiden, zoals het integreren van externe services.

### JSON-weergave {#json-view}

- Bekijk de ruwe JSON-weergave van de bestelling, handig voor debuggen of integratiedoeleinden.

## Opslaan op de server {#save-to-server}

- Hiermee kun je openstaande winkelwagens op de server opslaan, zodat ze veilig worden bewaard en toegankelijk zijn voor toekomstig gebruik.
- Maakt een WooCommerce-bestelling aan met de status `pos-open`.

### Waarom opslaan? {#why-save}

Wanneer er een nieuwe bestelling in de POS wordt aangemaakt, bestaat deze alleen lokaal. 
Door de bestelling op te slaan, zorg je ervoor dat deze behouden blijft, zelfs als de lokale database wordt gewist.

Zie [Openstaande bestellingen](/pos/cart/open-orders) voor meer details over het beheren van opgeslagen bestellingen.

## Annuleren {#void}

- Twee scenario's:
  - Niet-opgeslagen bestellingen:
    - Als de bestelling niet op de server is opgeslagen, wordt deze permanent verwijderd uit de lokale database.
  - Opgeslagen bestellingen:
    - Als de bestelling op de server is opgeslagen, wordt deze verplaatst naar de Prullenbak in WooCommerce en verwijderd uit de lokale database.
- Om een geannuleerde bestelling te herstellen:
  - Ga naar `WP Admin > WooCommerce > Orders > Trash`.

## Afrekenen {#checkout}

- Slaat de bestelling op de server op en start het afrekenproces.
- Opent het [afrekenscherm](/pos/checkout) om de betaling te voltooien.
