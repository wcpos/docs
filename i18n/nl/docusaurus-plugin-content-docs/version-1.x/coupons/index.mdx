:::info Pro Feature
NL: The Coupons screen and the ability to apply coupons at the register require [WCPOS Pro](/getting-started/pro-license). Free users see a blurred preview of the Coupons screen but cannot view coupon details or apply codes in the cart.
:::

NL: The Coupons screen lets you browse and look up your WooCommerce coupons from inside the POS, and the **Add Coupon** button in the cart applies them to the current order. Coupons themselves are still created and managed in WooCommerce — the POS syncs them down and validates them locally for instant feedback at the till.

## Vertaald: Interface Overview {#interface-overview}

### Vertaald: Header Actions {#header-actions}

NL: At the top of the screen:

- NL: **Search bar** — Find a coupon by code
- NL: **Display settings** (<Icon name="sliders" />) — Configure visible columns

### Vertaald: Coupons Table {#coupons-table}

NL: The main area displays your synced coupons with:

- NL: **Code** — The coupon code customers enter
- NL: **Description** — Internal description (also used as the receipt label when set)
- NL: **Discount Type** — Percentage, fixed cart, or fixed product
- NL: **Amount** — The discount value
- NL: **Used / Limit** — Times used and overall usage limit
- NL: **Expires** — Expiry date, or blank if none
- NL: **Actions** — Three-dot menu

### Vertaald: Footer {#footer}

- NL: Coupon count with sync button (<Icon name="arrow-rotate-right" />). **Long press** for Clear and Refresh options.

## Vertaald: Coupon Types {#coupon-types}

NL: WCPOS supports the three standard WooCommerce coupon types:

| NL: Type | NL: What it does | NL: Example | NL:
|---|---|---|
| NL: **Percentage discount** | NL: Reduces the cart subtotal by a percentage | NL: `10%` off the order | NL:
| NL: **Fixed cart discount** | NL: Reduces the cart total by a fixed amount | NL: `$5` off the order | NL:
| NL: **Fixed product discount** | NL: Reduces matching product lines by a fixed per-unit amount | NL: `$2` off each T-shirt | NL:

NL: All three types respect the same validation rules (expiry, usage limits, product/category restrictions, etc.).

## Vertaald: Creating Coupons {#creating-coupons}

NL: Coupons are created in WooCommerce, not in the POS. Go to **WP Admin → Marketing → Coupons → Add coupon**:

1. NL: **Coupon code** — what cashiers type at the register. Codes are case-insensitive.
2. NL: **Description** — internal note. WCPOS uses this as the discount label on receipts when set, so prefer a customer-friendly phrase like "Manager Discount 10%" over an internal code.
3. NL: **Discount type** — percentage, fixed cart, or fixed product.
4. NL: **Coupon amount** — the value of the discount.
5. NL: **Expiry date** — optional. After this date the coupon will be rejected.

### Vertaald: Usage restriction {#usage-restriction}

- NL: **Minimum / maximum spend** — coupon only applies above/below a subtotal threshold.
- NL: **Individual use only** — prevents this coupon from being combined with any other coupon.
- NL: **Exclude sale items** — skips items already on sale (and any line where a cashier has lowered the price at the till — see [POS price overrides](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)).
- NL: **Products / Exclude products** — restrict the coupon to (or away from) specific products.
- NL: **Product categories / Exclude categories** — restrict by category. Misc/custom products created at the POS respect categories assigned through the POS as well.
- NL: **Allowed emails** — limit the coupon to specific customer email addresses (supports `*` wildcards).

### Vertaald: Usage limits {#usage-limits}

- NL: **Usage limit per coupon** — total times this code can be used across all customers.
- NL: **Usage limit per user** — times one customer can use it.
- NL: **Limit usage to X items** — for fixed-product coupons, cap the number of qualifying items discounted per use.

### Vertaald: Tips for POS use {#tips-for-pos-use}

- NL: Create a **"Manager 10%"** coupon (10% off, no minimum spend, no expiry, single-use restriction off) and give the code to managers — staff can apply it at the till for ad-hoc adjustments and you keep a tracked discount in reports.
- NL: Create a **"Loyalty $5"** coupon for repeat-customer rewards.
- NL: For one-off promotions, set a short expiry date so the code can't be reused later by mistake.
- NL: Setting **Description** to the phrase you'd like printed on receipts is the fastest way to brand your discounts.

## Vertaald: Applying a Coupon at the Register {#applying-a-coupon-at-the-register}

NL: Tap **Add Coupon** in the cart, then type the code or search by description. The coupon validates instantly against synced data and appears as a removable pill above the cart totals.

NL: For the full cashier workflow — search, coupon pills, stacking multiple coupons, and resolving validation errors — see **[Applying Coupons at the Till](/coupons/applying-coupons)**.

## Vertaald: How Validation Works {#how-validation-works}

NL: When a code is entered, the POS checks all the same rules WooCommerce would check on the server:

- NL: Coupon exists and is not expired
- NL: Usage limits aren't exceeded (overall and per-user)
- NL: Minimum/maximum spend is met
- NL: "Individual use" coupons don't conflict with already-applied coupons
- NL: Product/category restrictions match the cart contents
- NL: Email restrictions match the selected customer (or are skipped on Guest orders)
- NL: Sale items are excluded if **Exclude sale items** is enabled

NL: If any check fails, the cashier sees a specific error message (e.g., "This coupon has expired" or "Minimum spend not reached"). The same checks run again on the server when the order is submitted — the local validation is for speed, not for trust.

## Vertaald: Sync Behaviour {#sync-behaviour}

NL: Coupons sync from WooCommerce to the device via the standard WCPOS replication pipeline:

- NL: New coupons created in WP Admin appear in the POS on the next sync.
- NL: Updates to existing coupons (usage count, expiry changes, etc.) sync down automatically.
- NL: Coupons are stored locally in IndexedDB so they remain available offline.

NL: If you just created a coupon in WP Admin and don't see it yet, the sync may not have run. From the Coupons screen footer, tap the sync icon (<Icon name="arrow-rotate-right" />) to refresh — long-press for **Clear and refresh** if you need a fresh fetch.

## Vertaald: Connectivity {#connectivity}

- NL: **Applying coupons** works offline because validation is client-side from synced data.
- NL: **Completing checkout** still requires a server connection — the server re-validates and is the authoritative source for the final order total.
- NL: **Creating or editing coupons** happens in WP Admin and requires a connection to your WordPress site.

## Vertaald: How Coupons Interact with POS Discounts {#how-coupons-interact-with-pos-discounts}

NL: When a cashier lowers a line price at the till and a coupon is then applied, the coupon calculates against the **lowered price**, not the original. POS-lowered lines are treated as "on sale" — coupons with **Exclude sale items** enabled will skip them. See [Discounts](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons) for more.
