# Web Checkout Gateway

Met de Web Checkout-gateway kunnen klanten hun POS-transacties voltooien via het afrekensysteem van je webwinkel. Deze gateway vormt een naadloze brug tussen je POS in de winkel en online betaalmethoden, perfect voor situaties waarin klanten liever met hun eigen apparaten betalen of wanneer je toegang nodig hebt tot betaalmethoden die alleen op het web beschikbaar zijn.

## Installatie {#installation}

<Steps>
  <StepCard number="1" title="Downloaden en installeren">

1. Bezoek de [releases-pagina van de Web Checkout Gateway](https://github.com/wcpos/web-checkout-gateway/releases)
2. Download het nieuwste **woocommerce-pos-web-checkout-gateway.zip**-bestand
3. Ga in je WordPress-beheer naar `Plugins > Add New > Upload Plugin`
4. Upload het zip-bestand en activeer de plugin

  </StepCard>
  <StepCard number="2" title="Inschakelen in WCPOS" isLast>

1. Ga naar `WP Admin > POS > Settings > Checkout`
2. Zoek de **WCPOS Web Checkout Gateway** in de lijst
3. Schakel de gateway in voor gebruik in de POS
4. Sla je instellingen op

:::note
Deze gateway maakt gebruik van je bestaande WooCommerce-betaalmethoden, dus zorg dat je ten minste één webbetaalgateway correct hebt geconfigureerd (Stripe, PayPal, enz.).
:::

  </StepCard>
</Steps>

## Gebruik {#usage}

### Afrekenen op het web starten {#initiating-web-checkout}

1. **Artikelen toevoegen**: Voeg producten toe aan je winkelwagen in de POS
2. **Gateway selecteren**: Kies "Web Checkout" als betaalmethode
3. **Link genereren**: Het systeem maakt een unieke afrekenlink voor de bestelling
4. **Klanttoegang**: De klant klikt op de link om het afrekenen op het web te openen
5. **Online betaling**: De klant voltooit de betaling met de betaalmethoden van de webwinkel

### Betalingsproces {#payment-process}

De Web Checkout-gateway volgt deze workflow:

1. **Bestelling aanmaken**: De POS maakt een openstaande bestelling aan in WooCommerce
2. **Afrekenlink**: Het systeem genereert een veilige, unieke betaallink
3. **Klant doorverwijzen**: De klant wordt naar de afrekenpagina op het web geleid
4. **Betaling selecteren**: De klant kiest uit de beschikbare webbetaalmethoden
5. **Betalingsverwerking**: Het standaard WooCommerce-afrekenproces handelt de betaling af
6. **Bestelling voltooien**: De bestelstatus wordt automatisch bijgewerkt na een geslaagde betaling

### POS-workflow {#pos-workflow}

Na het starten van het afrekenen op het web:

1. **Status monitoren**: Houd het POS-bestelscherm open om de betalingsstatus te volgen
2. **Klantbetaling**: De klant voltooit de betaling op zijn eigen apparaat of op je tablet
3. **Automatische update**: De bestelstatus wordt in realtime bijgewerkt wanneer de betaling is ontvangen
4. **Betaling verwerken**: Klik op de knop "Process Payment" in de POS om door te gaan naar het bonnetje
5. **Bonnetje genereren**: Genereer en print het bonnetje zoals gebruikelijk

## Technische details {#technical-details}

### Bestelbeheer {#order-management}

- **Openstaande bestellingen**: Bestellingen worden aangemaakt met de status "pending payment"
- **Statusupdates**: Automatische statuswijzigingen wanneer de betaling is ontvangen
- **Bestelnotities**: Betalingsgegevens worden vastgelegd in de bestelnotities
- **Voorraad**: De voorraad wordt gereserveerd tijdens het afrekenproces

### Beveiliging {#security}

- **Unieke links**: Elke afrekensessie heeft een unieke, in tijd beperkte URL
- **SSL vereist**: Alle betalingsverwerking verloopt via veilige HTTPS-verbindingen
- **PCI-compliance**: Maakt gebruik van je bestaande PCI-conforme webbetaalgateways
- **Sessiebeheer**: Veilig sessiebeheer voorkomt ongeautoriseerde toegang

### Betaalmethoden {#payment-methods}

De gateway biedt toegang tot al je geconfigureerde WooCommerce-betaalmethoden:

- **Krediet-/debetkaarten**: Stripe, Square, Authorize.net, enz.
- **Digitale portemonnees**: PayPal, Apple Pay, Google Pay
- **Bankoverschrijvingen**: Directe bankbetaalmethoden
- **Nu kopen, later betalen**: Klarna, Afterpay, Sezzle
- **Cryptovaluta**: Bitcoin en andere crypto-betaalgateways
- **Lokale methoden**: Regiospecifieke betaalmethoden

## Probleemoplossing {#troubleshooting}

### Veelvoorkomende problemen {#common-issues}

<Accordion>
  <AccordionItem question="Afrekenlink werkt niet">

  - Controleer of het SSL-certificaat correct is geïnstalleerd
  - Controleer of de WooCommerce-permalinks correct zijn geconfigureerd
  - Zorg dat de webbetaalgateways actief en geconfigureerd zijn
  - Test met een andere browser of een ander apparaat

  </AccordionItem>

  <AccordionItem question="Betaling wordt niet verwerkt">

  - Bevestig dat de webbetaalgateways correct zijn geconfigureerd
  - Controleer de logs van de betaalgateway op foutmeldingen
  - Controleer of de betaalmethode van de klant wordt ondersteund
  - Test het afrekenproces op het web afzonderlijk

  </AccordionItem>

  <AccordionItem question="Bestelling wordt niet bijgewerkt in de POS">

  - Controleer de stabiliteit van de internetverbinding
  - Controleer of de WordPress-cronjobs correct draaien
  - Vernieuw het POS-bestelscherm
  - Controleer de bestelstatus in de WooCommerce-beheeromgeving

  </AccordionItem>

  <AccordionItem question="Klant heeft geen toegang tot het afrekenen">

  - Zorg dat de afrekenlink niet is verlopen
  - Controleer of het apparaat van de klant internetverbinding heeft
  - Controleer of de bestelling niet is geannuleerd of voltooid
  - Test de link op een ander apparaat

  </AccordionItem>
</Accordion>

### Prestatieoptimalisatie {#performance-optimisation}

Voor optimale prestaties:

- **Caching**: Configureer cachingplugins om afrekenpagina's uit te sluiten
- **CDN**: Zorg dat de CDN-instellingen het afrekenproces niet verstoren
- **Database**: Optimaliseer de database voor snellere bestelverwerking
- **Hosting**: Gebruik betrouwbare hosting met een goede uptime
