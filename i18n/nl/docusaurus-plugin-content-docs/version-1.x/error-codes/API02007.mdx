# API02007: Tokenvernieuwing mislukt

## Wat dit betekent {#what-this-means}

De POS heeft geprobeerd je authenticatiesessie automatisch te vernieuwen, maar het vernieuwingsverzoek is mislukt. Dit betekent meestal dat je opnieuw moet inloggen.

## Veelvoorkomende oorzaken {#common-causes}

- **Vernieuwingstoken verlopen** — Het vernieuwingstoken zelf is verlopen
- **Vernieuwingstoken ingetrokken** — Een beheerder heeft alle sessies ongeldig gemaakt
- **Serverfout** — De server kon het vernieuwingsverzoek niet verwerken
- **Netwerkprobleem** — Verbindingsprobleem tijdens de vernieuwingspoging

## Oplossen {#how-to-fix}

### 1. Opnieuw inloggen {#1-log-in-again}

De betrouwbaarste oplossing:
1. Log uit van de POS
2. Log opnieuw in met je gegevens
3. Dit maakt een nieuwe sessie met nieuwe tokens aan

### 2. Netwerkverbinding controleren {#2-check-network-connection}

Als de vernieuwing mislukt door netwerkproblemen:
- Controleer je internetverbinding
- Probeer je WordPress-site in een browser te openen
- Wacht even en probeer het opnieuw

### 3. Serverstatus controleren {#3-check-server-status}

Als de server problemen heeft:
- Controleer of WordPress goed werkt
- Controleer op PHP-fouten in je serverlogs
- Zorg dat de authenticatieplugin actief is

### 4. Sessiegegevens wissen {#4-clear-session-data}

Als de problemen blijven bestaan:
- Wis de cache van de POS-app
- Verwijder opgeslagen inloggegevens
- Begin opnieuw met een nieuwe aanmelding

## Automatische tokenvernieuwing {#automatic-token-refresh}

WCPOS vernieuwt tokens automatisch voordat ze verlopen om je sessie te behouden. Als dit proces herhaaldelijk mislukt, is er mogelijk een configuratieprobleem dat aandacht nodig heeft.
