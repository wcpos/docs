# API02008: Vernieuwingstoken ongeldig

## Wat dit betekent {#what-this-means}

Het vernieuwingstoken dat door de POS is opgeslagen, wordt niet herkend door de server. Vernieuwingstokens worden gebruikt om nieuwe toegangstokens te verkrijgen zonder dat je opnieuw hoeft in te loggen.

## Veelvoorkomende oorzaken {#common-causes}

- **Token ingetrokken** — Een beheerder heeft je sessie ongeldig gemaakt
- **Servergeheimen gewijzigd** — De WordPress-beveiligingssleutels zijn bijgewerkt
- **Gegevensbeschadiging** — Het token is beschadigd geraakt in de opslag
- **Meerdere aanmeldingen** — Ergens anders inloggen kan dit token ongeldig hebben gemaakt

## Oplossen {#how-to-fix}

### 1. Opnieuw inloggen {#1-log-in-again}

Omdat het vernieuwingstoken ongeldig is, moet je opnieuw authenticeren:
1. Log uit van de POS
2. Voer je gebruikersnaam en wachtwoord in
3. Dit genereert nieuwe, geldige tokens

### 2. Controleren op ingetrokken sessies {#2-check-for-revoked-sessions}

Als een beheerder sessies heeft ingetrokken:
- Dit is een normale beveiligingsmaatregel
- Log gewoon opnieuw in
- Je nieuwe sessie werkt dan normaal

### 3. Serverwijzigingen controleren {#3-check-server-changes}

Als dit alle gebruikers treft:
- WordPress-salts/sleutels zijn mogelijk gewijzigd
- De instellingen van de JWT-plugin zijn mogelijk aangepast
- Neem contact op met je serverbeheerder

### 4. Opgeslagen gegevens wissen {#4-clear-stored-data}

Als je problemen blijft ondervinden:
- Wis de gegevens van de POS-toepassing
- Verwijder gecachte inloggegevens
- Begin met een schone aanmelding
