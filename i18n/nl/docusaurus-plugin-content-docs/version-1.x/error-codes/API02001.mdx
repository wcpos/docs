# API02001: Ongeldige inloggegevens

## Wat dit betekent {#what-this-means}

De gebruikersnaam of het wachtwoord dat je hebt ingevoerd is onjuist. De server heeft de inlogpoging geweigerd omdat de inloggegevens niet overeenkomen met een geldig account.

## Veelvoorkomende oorzaken {#common-causes}

- **Typefout in gebruikersnaam of wachtwoord** — Controleer op caps lock of extra spaties
- **Verkeerd account** — Inloggegevens van een andere site gebruiken
- **Wachtwoord recent gewijzigd** — Het wachtwoord is bijgewerkt maar de POS heeft het oude
- **Account verwijderd** — Het gebruikersaccount bestaat niet meer

## Toewijzing serverfouten {#server-error-mapping}

Deze foutcode wordt geactiveerd wanneer de server het volgende teruggeeft:

| Servercode | Bron |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT Authentication plugin |

## Hoe op te lossen {#how-to-fix}

### 1. Verifieer je inloggegevens {#1-verify-your-credentials}

- Controleer je gebruikersnaam nogmaals (meestal e-mail of WordPress-gebruikersnaam)
- Controleer of het wachtwoord correct is
- Probeer rechtstreeks in te loggen op WordPress Admin om dit te bevestigen

### 2. Reset je wachtwoord {#2-reset-your-password}

Als je je wachtwoord bent vergeten:
1. Ga naar je WordPress-inlogpagina
2. Klik op "Lost your password?"
3. Voer je e-mailadres in om een resetlink te ontvangen
4. Werk je wachtwoord bij
5. Log in op de POS met het nieuwe wachtwoord

### 3. Controleer het gebruikersaccount {#3-check-the-user-account}

In WordPress Admin:
- Ga naar Users → All Users
- Verifieer dat het account bestaat
- Controleer of het de juiste POS-toegangsmachtigingen heeft

### 4. Wis opgeslagen inloggegevens {#4-clear-saved-credentials}

In de POS:
- Log volledig uit
- Wis alle opgeslagen/gecachete inloggegevens
- Voer de inloggegevens opnieuw in
