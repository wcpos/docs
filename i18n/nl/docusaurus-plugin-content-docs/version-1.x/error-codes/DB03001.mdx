# DB03001: Syntaxfout in query

## Wat dit betekent {#what-this-means}

De databasequery bevat syntaxfouten. Dit duidt doorgaans op een bug in de applicatie en niet op een gebruikersfout.

## Veelvoorkomende oorzaken {#common-causes}

- **Softwarefout** — Een probleem in de POS-applicatie
- **Versieverschil** — Incompatibel dataschema
- **Beschadigde gegevens** — Gegevens bevatten onverwachte tekens
- **Coderingsproblemen** — Problemen met tekencodering

## Hoe op te lossen {#how-to-fix}

### 1. Werk de applicatie bij {#1-update-the-application}

Zorg ervoor dat je de nieuwste versie gebruikt:
- Controleer op updates van de POS-app
- Werk de WCPOS WordPress-plugin bij
- Herstart na het bijwerken

### 2. Wis applicatiegegevens {#2-clear-application-data}

Reset naar een schone staat:
- Wis lokale gegevens/cache
- Log opnieuw in
- Synchroniseer de gegevens opnieuw

### 3. Controleer op speciale tekens {#3-check-for-special-characters}

Als de fout optreedt bij specifieke gegevens:
- Controleer op ongebruikelijke tekens in productnamen
- Let op emoji's of speciale symbolen
- Probeer het met eenvoudigere gegevens

### 4. Meld de bug {#4-report-the-bug}

Als het probleem aanhoudt:
1. Noteer welke actie de fout veroorzaakt
2. Leg eventuele foutdetails vast
3. Meld het op [GitHub](https://github.com/wcpos)

## Dit is meestal een bug {#this-is-usually-a-bug}

Syntaxfouten in query's zijn doorgaans bugs in de software, geen gebruikersfouten. Als je dit tegenkomt:
- Je doet niets verkeerd
- Het ontwikkelteam moet hiervan op de hoogte zijn
- Een oplossing wordt uitgebracht in een update

## Tijdelijke oplossing {#workaround}

Totdat er een oplossing beschikbaar is:
- Probeer alternatieve manieren om je taak uit te voeren
- Gebruik de WooCommerce-beheeromgeving voor de getroffen bewerkingen
- Controleer regelmatig op updates
