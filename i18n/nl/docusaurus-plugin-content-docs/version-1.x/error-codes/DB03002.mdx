# DB03002: Ongeldig gegevenstype

## Wat dit betekent {#what-this-means}

Het gegevenstype komt niet overeen met wat de database verwacht. Er werd bijvoorbeeld tekst opgegeven waar een getal werd verwacht.

## Veelvoorkomende oorzaken {#common-causes}

- **Invoerfout van gebruiker** — Tekst invoeren in een numeriek veld
- **Importproblemen** — Geïmporteerde gegevens hebben het verkeerde formaat
- **Gegevensbeschadiging** — Waarden beschadigd tijdens overdracht
- **Pluginconflict** — Een andere plugin heeft gegevenstypen gewijzigd

## Hoe op te lossen {#how-to-fix}

### 1. Controleer je invoer {#1-check-your-input}

Controleer de gegevens die je invoert:
- **Prijzen** — Moeten getallen zijn (bijv. `19.99`)
- **Aantallen** — Moeten hele getallen zijn (bijv. `5`)
- **ID's** — Moeten numeriek zijn

### 2. Wissen en opnieuw invoeren {#2-clear-and-re-enter}

Als gegevens beschadigd waren:
- Wis het veld
- Voer de waarde opnieuw handmatig in
- Vermijd kopiëren en plakken uit externe bronnen

### 3. Controleer de brongegevens {#3-check-source-data}

Bij synchronisatie vanuit WooCommerce:
- Controleer de gegevens in WordPress Admin
- Let op onjuist opgemaakte velden
- Corrigeer de gegevens bij de bron

### 4. Synchroniseer gegevens opnieuw {#4-re-sync-data}

Haal verse gegevens op van de server:
- Wis de lokale cache
- Synchroniseer de gegevens opnieuw
- Controleer of het probleem is opgelost

### 5. Controleer op pluginconflicten {#5-check-for-plugin-conflicts}

Als je andere WooCommerce-plugins gebruikt:
- Ze kunnen gegevens op onverwachte manieren wijzigen
- Schakel ze tijdelijk uit om te testen
- Meld incompatibiliteiten

## Veelvoorkomende voorbeelden {#common-examples}

| Veld | Verwacht | Ongeldig |
|-------|----------|---------|
| Prijs | `19.99` | `$19.99` |
| Aantal | `5` | `vijf` |
| SKU | `ABC123` | — (elk formaat is OK) |
| Product-ID | `42` | `product-42` |
