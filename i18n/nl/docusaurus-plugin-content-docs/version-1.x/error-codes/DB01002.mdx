# DB01002: Query-time-out

## Wat dit betekent {#what-this-means}

Een databasebewerking duurde te lang om te voltooien. De lokale databasequery overschreed de maximaal toegestane tijd.

## Veelvoorkomende oorzaken {#common-causes}

- **Grote gegevensset** — Een zeer groot aantal producten of records
- **Complexe query** — Een query met veel voorwaarden
- **Prestaties van het apparaat** — Het apparaat werkt traag
- **Databasefragmentatie** — De database moet worden geoptimaliseerd

## Hoe op te lossen {#how-to-fix}

### 1. Wacht en probeer opnieuw {#1-wait-and-retry}

De query slaagt mogelijk bij een nieuwe poging:
- Sluit de POS en open deze opnieuw
- Probeer de bewerking opnieuw
- Vermijd andere intensieve bewerkingen tegelijkertijd

### 2. Sluit andere applicaties {#2-close-other-applications}

Maak apparaatbronnen vrij:
- Sluit ongebruikte browsertabbladen
- Sluit andere applicaties
- Start het apparaat opnieuw op indien nodig

### 3. Beperk de gegevensomvang {#3-reduce-data-scope}

Bij het zoeken:
- Gebruik specifiekere zoektermen
- Pas filters toe om de resultaten te beperken
- Splits grote bewerkingen op in kleinere

### 4. Optimaliseer de database {#4-optimise-the-database}

Wissen en opnieuw synchroniseren:
1. Wis de lokale gegevens (instellingsoptie indien beschikbaar)
2. Log opnieuw in
3. Laat de gegevens opnieuw synchroniseren

### 5. Controleer de prestaties van het apparaat {#5-check-device-performance}

Zorg ervoor dat je apparaat aan de vereisten voldoet:
- Voldoende RAM
- Adequate processorsnelheid
- SSD heeft de voorkeur boven HDD

## Grote catalogi {#large-catalogues}

Als je een zeer grote productcatalogus hebt:
- De eerste synchronisatie duurt langer
- Overweeg filters te gebruiken in de POS
- Sommige bewerkingen nemen van nature meer tijd in beslag
- Neem contact op met support voor optimalisatieadvies
