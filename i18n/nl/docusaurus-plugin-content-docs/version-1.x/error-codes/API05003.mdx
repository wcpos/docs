# API05003: WCPOS-plugin verouderd

## Wat dit betekent {#what-this-means}

De WCPOS-plugin die op je server is geïnstalleerd, is verouderd en niet compatibel met jouw versie van de POS-applicatie. Je moet de plugin bijwerken.

## Veelvoorkomende oorzaken {#common-causes}

- **Plugin niet bijgewerkt** — Automatische updates zijn mogelijk uitgeschakeld
- **Update mislukt** — Een eerdere updatepoging is mislukt
- **Versieverschil** — De POS-app is bijgewerkt maar de plugin niet
- **Staging-/dev-site** — Testen op een oudere versie

## Hoe op te lossen {#how-to-fix}

### 1. Werk de plugin bij {#1-update-the-plugin}

In WordPress-beheer:
1. Ga naar Plugins → Geïnstalleerde plugins
2. Zoek "WCPOS"
3. Als er een update beschikbaar is, klik je op "Nu bijwerken"

### 2. Schakel automatische updates in {#2-enable-auto-updates}

Om toekomstige versieverschillen te voorkomen:
1. Ga naar Plugins → Geïnstalleerde plugins
2. Zoek "WCPOS"
3. Klik op "Automatische updates inschakelen"

### 3. Handmatige update {#3-manual-update}

Als de automatische update mislukt:
1. Download de nieuwste versie van [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/)
2. Deactiveer de huidige plugin
3. Verwijder de oude pluginbestanden
4. Upload en activeer de nieuwe versie

### 4. Controleer op updatefouten {#4-check-for-update-errors}

Als updates mislukken:
- Controleer de bestandsrechten op wp-content/plugins
- Verifieer dat je schijfruimte hebt
- Zoek naar PHP-geheugenproblemen
- Controleer de foutlogs voor details

### 5. Downgrade de POS-app (tijdelijk) {#5-downgrade-pos-app-temporary}

Als je de plugin niet meteen kunt bijwerken:
- Gebruik een oudere versie van de POS-app
- Dit is slechts een tijdelijke oplossing
- Plan om de plugin binnenkort bij te werken

## Versiecompatibiliteit {#version-compatibility}

Houd beide componenten altijd bijgewerkt:
- De POS-applicatie
- De WCPOS WordPress-plugin

Controleer de release notes op breaking changes bij het bijwerken.
