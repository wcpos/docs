# API01004: DNS-resolutie mislukt

## Wat dit betekent {#what-this-means}

De POS kon de domeinnaam van je website (bijv. `yourstore.com`) niet omzetten in een IP-adres. DNS (Domain Name System) is als het telefoonboek van het internet — als het de vermelding niet kan vinden, kan het geen verbinding maken.

## Veelvoorkomende oorzaken {#common-causes}

- **Typefout in domeinnaam** — De URL bevat mogelijk een spelfout
- **Problemen met DNS-server** — Je DNS-server reageert niet
- **Domein verlopen** — De domeinregistratie is mogelijk verlopen
- **DNS-propagatie** — Recente DNS-wijzigingen zijn nog niet doorgevoerd
- **Lokale DNS-cache** — Verouderde DNS-informatie in de cache van je apparaat

## Hoe op te lossen {#how-to-fix}

### 1. Verifieer de domeinnaam {#1-verify-the-domain-name}

Controleer de URL nogmaals op typefouten. Probeer de site in een browser te openen.

### 2. Controleer de domeinstatus {#2-check-domain-status}

Gebruik een WHOIS-zoekopdracht om te verifiëren:
- Het domein is nog steeds geregistreerd
- Het is niet verlopen
- De DNS-records zijn geconfigureerd

### 3. Wis de DNS-cache {#3-clear-dns-cache}

**Op je apparaat:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Probeer een alternatieve DNS {#4-try-alternative-dns}

Schakel over naar een openbare DNS-server zoals:
- Google: `8.8.8.8` en `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Wacht op DNS-propagatie {#5-wait-for-dns-propagation}

Als je onlangs de DNS-instellingen hebt gewijzigd, wacht dan 24-48 uur tot de volledige propagatie.
