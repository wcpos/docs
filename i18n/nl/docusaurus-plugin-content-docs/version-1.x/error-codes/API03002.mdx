# API03002: Vereiste parameters ontbreken

## Wat dit betekent {#what-this-means}

In het verzoek ontbreken gegevens die de server nodig heeft om de actie te voltooien. Vereiste velden waren niet in het verzoek opgenomen.

## Veelvoorkomende oorzaken {#common-causes}

- **Onvolledig formulier** — Vereiste velden waren niet ingevuld
- **Gegevens niet opgeslagen** — De formuliergegevens zijn niet goed vastgelegd
- **Versieverschil** — De plugin verwacht andere parameters dan de POS verstuurt
- **Aangepaste velden** — Vereiste aangepaste velden ontbreken

## Oplossen {#how-to-fix}

### 1. Vereiste velden invullen {#1-complete-required-fields}

Controleer of alle vereiste informatie is ingevuld:
- Klantgegevens (indien vereist)
- Productinformatie
- Bestelgegevens

### 2. Vernieuwen en opnieuw proberen {#2-refresh-and-retry}

De status van het formulier is mogelijk onvolledig:
1. Vernieuw de POS
2. Voer de vereiste informatie opnieuw in
3. Probeer de actie opnieuw

### 3. Pluginversies controleren {#3-check-plugin-versions}

Zorg voor compatibiliteit:
- Werk de WCPOS-plugin bij
- Werk de POS-toepassing bij
- Beide moeten compatibele versies zijn

### 4. Instellingen voor vereiste velden controleren {#4-check-required-field-settings}

In WooCommerce kunnen sommige velden als vereist zijn gemarkeerd:
- Bekijk de vereisten voor afrekenvelden
- Controleer de configuraties van aangepaste velden
- Pas de vereisten zo nodig aan

### 5. Aangepaste integraties controleren {#5-check-custom-integrations}

Als je aangepaste plugins of integraties gebruikt:
- Deze vereisen mogelijk extra velden
- Bekijk de plugindocumentatie
- Controleer op conflicten
