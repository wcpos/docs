# API01008: Website niet beschikbaar

## Wat dit betekent {#what-this-means}

De POS kan het internet bereiken, maar je specifieke website reageert niet. De server die je WooCommerce-winkel host lijkt offline of onbereikbaar te zijn.

## Veelvoorkomende oorzaken {#common-causes}

- **Serveronderhoud** — Je hostingprovider voert mogelijk updates uit
- **Servercrash** — De webserver is onverwacht gestopt
- **Resourcelimieten overschreden** — Je hostingaccount heeft zijn limieten bereikt
- **DDoS-aanval** — De server wordt overweldigd door kwaadaardig verkeer
- **Domein/hosting verlopen** — Je hosting of domein is verlopen

## Hoe op te lossen {#how-to-fix}

### 1. Controleer of de site offline is {#1-verify-the-site-is-down}

Probeer je site rechtstreeks in een browser te openen. Als deze niet laadt, ligt het probleem aan de serverzijde.

### 2. Controleer de status van de hostingprovider {#2-check-hosting-provider-status}

- Bezoek de statuspagina van je hostingprovider
- Controleer op gepland onderhoud
- Zoek naar gemelde storingen

### 3. Bekijk het hostingaccount {#3-review-hosting-account}

Log in op het bedieningspaneel van je hosting:
- Controleer of je account actief is
- Verifieer dat je de resourcelimieten niet hebt overschreden
- Zoek naar eventuele opschortingsmeldingen

### 4. Neem contact op met de hostingsupport {#4-contact-hosting-support}

Als alles aan jouw kant normaal lijkt, neem dan contact op met je hostingprovider. Zij kunnen:
- De serverstatus controleren
- De serverlogboeken bekijken
- Indien nodig services opnieuw opstarten

### 5. Wacht en probeer opnieuw {#5-wait-and-retry}

Als het een tijdelijke storing is:
- Wacht 15-30 minuten
- De POS probeert automatisch opnieuw verbinding te maken
- Gecachete gegevens blijven offline beschikbaar
