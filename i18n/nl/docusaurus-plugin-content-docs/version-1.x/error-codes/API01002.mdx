# API01002: Verbinding geweigerd

## Wat dit betekent {#what-this-means}

De server heeft de verbindingspoging actief geweigerd. Dit is anders dan een time-out — de server reageerde, maar zei "nee".

## Veelvoorkomende oorzaken {#common-causes}

- **Server is offline** — De webserver (Apache/Nginx) draait mogelijk niet
- **Verkeerde poort** — Er wordt geprobeerd verbinding te maken op de verkeerde poort
- **Firewall blokkeert** — Een firewall blokkeert de verbinding
- **IP-beperkingen** — Je IP-adres is mogelijk geblokkeerd

## Hoe op te lossen {#how-to-fix}

### 1. Controleer of de server draait {#1-verify-the-server-is-running}

Probeer je WordPress-site rechtstreeks in een browser te openen. Als deze niet laadt, neem dan contact op met je hostingprovider.

### 2. Controleer de URL-configuratie {#2-check-the-url-configuration}

Zorg ervoor dat je de juiste URL gebruikt:
- Controleer `http://` versus `https://`
- Controleer op typefouten in de domeinnaam
- Bevestig de juiste poort als je een niet-standaard poort gebruikt

### 3. Controleer de firewall-instellingen {#3-check-firewall-settings}

Als je toegang hebt tot je server:
- Controleer of poort 80 (HTTP) of 443 (HTTPS) open is
- Controleer of je IP op de witte lijst staat
- Bekijk eventuele beveiligingsplugins die de API-toegang kunnen blokkeren

### 4. Neem contact op met je hostingprovider {#4-contact-your-hosting-provider}

Zij kunnen controleren:
- De serverstatus en -logboeken
- De firewall-configuraties
- Of je IP is geblokkeerd
