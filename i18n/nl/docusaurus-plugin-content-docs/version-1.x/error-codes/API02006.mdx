# API02006: API-sleutel ongeldig

## Wat dit betekent {#what-this-means}

De WooCommerce REST API-sleutel die wordt gebruikt, is ongeldig. API-sleutels worden gebruikt voor server-naar-serverauthenticatie, en de opgegeven sleutel komt niet overeen met een geldige sleutel in WooCommerce.

## Veelvoorkomende oorzaken {#common-causes}

- **Sleutel verwijderd** — De API-sleutel is uit WooCommerce verwijderd
- **Typefout in sleutel** — De sleutel is onjuist ingevoerd
- **Verkeerd sleutelpaar** — Consumer key en secret komen niet overeen
- **Sleutel van andere site** — Er worden sleutels gebruikt die voor een andere installatie zijn gegenereerd

## Oplossen {#how-to-fix}

### 1. De API-sleutel controleren {#1-verify-the-api-key}

In WordPress Admin → WooCommerce → Instellingen → Geavanceerd → REST API:
1. Controleer of je API-sleutel bestaat
2. Controleer of deze niet is ingetrokken
3. Noteer de rechten (lezen/schrijven/lezen-schrijven)

### 2. Nieuwe API-sleutels genereren {#2-generate-new-api-keys}

Als de sleutel ontbreekt of ongeldig is:
1. Ga naar WooCommerce → Instellingen → Geavanceerd → REST API
2. Klik op "Add key"
3. Voer een beschrijving in (bijv. "WCPOS")
4. Selecteer de gebruiker
5. Kies "Read/Write"-rechten
6. Klik op "Generate API key"
7. **Kopieer zowel de Consumer Key als de Consumer Secret** (worden maar één keer getoond!)

### 3. POS-configuratie bijwerken {#3-update-pos-configuration}

Voer de nieuwe API-sleutels in de POS in:
- Consumer Key (begint met `ck_`)
- Consumer Secret (begint met `cs_`)

### 4. Sleutelrechten controleren {#4-check-key-permissions}

Zorg dat de sleutel voldoende rechten heeft:
- **Read** — Alleen gegevens bekijken
- **Write** — Alleen gegevens wijzigen
- **Read/Write** — Volledige toegang (aanbevolen voor POS)
