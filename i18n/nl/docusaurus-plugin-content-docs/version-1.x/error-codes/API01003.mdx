# API01003: Verbinding gereset

## Wat dit betekent {#what-this-means}

De verbinding met de server werd tot stand gebracht maar daarna onverwacht verbroken. Dit gebeurt meestal midden in de communicatie wanneer iets de verbinding onderbreekt.

## Veelvoorkomende oorzaken {#common-causes}

- **Servercrash** — Het serverproces is mogelijk gecrasht tijdens het verzoek
- **Geheugenlimieten** — PHP raakte zonder geheugen en werd beëindigd
- **Time-out tijdens verzoek** — De server kreeg een time-out tijdens de verwerking
- **Netwerkinstabiliteit** — Intermitterende netwerkproblemen
- **Beveiligingssoftware** — Agressieve beveiligingsinstellingen die verbindingen sluiten

## Hoe op te lossen {#how-to-fix}

### 1. Controleer de foutlogboeken van de server {#1-check-server-error-logs}

Kijk in je WordPress-foutlogboeken of het bedieningspaneel van je hosting naar gerelateerde fouten rond het tijdstip van de reset.

### 2. Verhoog de PHP-geheugenlimiet {#2-increase-php-memory-limit}

In je `wp-config.php`:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. Controleer op pluginconflicten {#3-check-for-plugin-conflicts}

Een plugin kan ervoor zorgen dat PHP crasht:
- Schakel plugins tijdelijk uit
- Schakel ze één voor één weer in om de boosdoener te vinden

### 4. Bekijk de serverbronnen {#4-review-server-resources}

Neem contact op met je hostingprovider om te controleren:
- De PHP-proceslimieten
- Het geheugengebruik op het moment van de fout
- De stabiliteitslogboeken van de server

### 5. Test de netwerkstabiliteit {#5-test-network-stability}

Als je WiFi gebruikt, probeer dan een bekabelde verbinding. Als het probleem zich voordoet op verschillende netwerken, ligt het waarschijnlijk aan de serverzijde.
