# API01001: Verbinding time-out

## Wat dit betekent {#what-this-means}

De POS-applicatie probeerde verbinding te maken met je WooCommerce-server, maar de server deed er te lang over om te antwoorden. Dit duidt meestal op netwerklatentieproblemen of een overbelaste server.

## Veelvoorkomende oorzaken {#common-causes}

- **Trage internetverbinding** — Je netwerkverbinding ondervindt mogelijk hoge latentie
- **Serveroverbelasting** — Je WordPress-server verwerkt mogelijk te veel verzoeken
- **Grote gegevensverzoeken** — Te veel gegevens tegelijk opvragen kan time-outs veroorzaken
- **Firewall- of proxyvertragingen** — Netwerkbeveiligingsapparaten kunnen latentie toevoegen

## Hoe op te lossen {#how-to-fix}

### 1. Controleer je internetverbinding {#1-check-your-internet-connection}

Test de snelheid en stabiliteit van je internet. Probeer je WooCommerce-beheerpaneel in een browser te laden om te zien of het traag reageert.

### 2. Verminder de serverbelasting {#2-reduce-server-load}

Als je server overbelast is:
- Overweeg je hostingpakket te upgraden
- Schakel onnodige plugins tijdelijk uit
- Controleer op zware cron-taken of achtergrondprocessen

### 3. Optimaliseer de serverresponstijd {#3-optimise-server-response-time}

Neem contact op met je hostingprovider over:
- Het inschakelen van PHP opcache
- Het verhogen van de PHP-geheugenlimieten
- Het gebruik van caching aan de serverzijde

### 4. Verhoog de time-outinstellingen {#4-increase-timeout-settings}

Als je server legitiem meer tijd nodig heeft:
- Controleer of je hostingprovider het verhogen van de PHP-uitvoeringstijd toestaat
- Overweeg een CDN te gebruiken om de latentie te verminderen
