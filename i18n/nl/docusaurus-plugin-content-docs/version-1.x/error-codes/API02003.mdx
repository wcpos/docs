# API02003: Token ongeldig

## Wat dit betekent {#what-this-means}

Het authenticatietoken dat wordt gebruikt, wordt niet herkend door de server. Dit verschilt van een verlopen token — dit token was nooit geldig of is ingetrokken.

## Veelvoorkomende oorzaken {#common-causes}

- **Token ingetrokken** — Een beheerder heeft de sessie ingetrokken
- **Token beschadigd** — Gegevensbeschadiging tijdens opslag of verzending
- **Server gereset** — De geheime serversleutels zijn gewijzigd, waardoor alle tokens ongeldig zijn geworden
- **Verkeerde server** — Token van een andere WordPress-installatie

## Toewijzing van serverfouten {#server-error-mapping}

Deze foutcode wordt geactiveerd wanneer de server het volgende retourneert:

| Servercode | Bron |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT Authentication plugin |

## Oplossen {#how-to-fix}

### 1. Opnieuw inloggen {#1-log-in-again}

Wis je sessie en authenticeer opnieuw:
1. Log volledig uit
2. Sluit de POS en open hem opnieuw
3. Log in met je gegevens

### 2. Toepassingsgegevens wissen {#2-clear-application-data}

Als uitloggen niet werkt:
- Wis de cache/gegevens van de POS-app
- Op het web: wis browsercookies en lokale opslag voor de site
- Op desktop: zoek in de app-instellingen naar een optie "Gegevens wissen"

### 3. Serverconfiguratie controleren {#3-check-server-configuration}

Als het probleem alle gebruikers treft:
- Controleer of de WordPress-salts zijn gewijzigd
- Controleer de instellingen van de JWT- of authenticatieplugin
- Bekijk recente serverwijzigingen

### 4. Controleren of je op de juiste server zit {#4-verify-youre-on-the-correct-server}

Zorg dat de POS is ingesteld om verbinding te maken met de juiste WordPress-site, vooral als je meerdere installaties hebt.
