# API06001: Ongeldige URL-indeling

## Wat dit betekent {#what-this-means}

De URL die voor je WooCommerce-site is geconfigureerd, heeft geen geldige indeling. URL's moeten voldoen aan de standaardconventies voor webadressen.

## Veelvoorkomende oorzaken {#common-causes}

- **Ontbrekend protocol** — De URL begint niet met `http://` of `https://`
- **Typefout in de URL** — Spelfouten in het domein
- **Ongeldige tekens** — Speciale tekens die niet zijn toegestaan in URL's
- **Onvolledige URL** — In de URL ontbreken onderdelen zoals de domeinextensie

## Hoe op te lossen {#how-to-fix}

### 1. Controleer de URL-indeling {#1-check-url-format}

Een geldige URL ziet er als volgt uit:
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com` (protocol ontbreekt)
- ❌ `https://yourstore` (extensie ontbreekt)
- ❌ `https://your store.com` (bevat een spatie)

### 2. Vermeld het protocol {#2-include-the-protocol}

Voeg altijd `https://` of `http://` toe:
- Voorkeur: `https://` (beveiligd)
- Terugval: `http://` (als HTTPS niet beschikbaar is)

### 3. Verwijder overbodige tekens {#3-remove-extra-characters}

Zorg ervoor dat de URL het volgende niet bevat:
- Afsluitende slashes (optioneel, maar wees consistent)
- Extra spaties
- Speciale tekens
- URL-parameters (tenzij vereist)

### 4. Controleer of de URL werkt {#4-verify-the-url-works}

Test de URL in een browser:
1. Kopieer de URL die je gebruikt
2. Plak deze in een browser
3. Deze zou je WordPress-site moeten laden

### 5. Werk de POS-configuratie bij {#5-update-pos-configuration}

Voer de gecorrigeerde URL in bij de POS-instellingen:
- Zorg dat deze exact overeenkomt met wat werkt in je browser
- Voeg `www.` toe of laat het weg, afhankelijk van hoe je site werkt

## Voorbeelden {#examples}

| Ongeldig | Geldig |
|---------|-------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
