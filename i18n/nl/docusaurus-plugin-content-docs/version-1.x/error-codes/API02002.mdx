# API02002: Token verlopen

## Wat dit betekent {#what-this-means}

Je authenticatiesessie is verlopen. Om veiligheidsredenen blijven aanmeldsessies niet eeuwig geldig. De POS moet je authenticatie vernieuwen of je moet mogelijk opnieuw inloggen.

## Veelvoorkomende oorzaken {#common-causes}

- **Lange inactiviteit** — Je hebt de POS lange tijd niet gebruikt
- **Levensduur van servertoken** — De verloopduur van het servertoken is erg kort ingesteld
- **Verkeerde servertijd** — De klokken van de server en het apparaat lopen niet synchroon

## Toewijzing van serverfouten {#server-error-mapping}

Deze foutcode wordt geactiveerd wanneer de server het volgende retourneert:

| Servercode | Bron |
|-------------|--------|
| `jwt_auth_expired_token` | JWT Authentication plugin |

## Oplossen {#how-to-fix}

### 1. Opnieuw inloggen {#1-log-in-again}

De eenvoudigste oplossing is uitloggen en opnieuw inloggen:
1. Log uit van de POS
2. Voer je gegevens opnieuw in
3. Dit maakt een nieuwe sessie aan

### 2. Tokenvernieuwing controleren {#2-check-token-refresh}

De POS zou tokens automatisch moeten vernieuwen. Als dat niet werkt:
- Controleer je internetverbinding
- Controleer of de server bereikbaar is
- Zie [API02007](./API02007) als de vernieuwing mislukt

### 3. Servertijd controleren {#3-check-server-time}

Het verlopen van tokens hangt af van een nauwkeurige tijd:
- Zorg dat de klok van je server correct is
- Controleer of de tijd van je apparaat klopt
- Gebruik automatische tijdsynchronisatie op beide

### 4. Tokeninstellingen bekijken {#4-review-token-settings}

Als tokens te snel verlopen, moet je mogelijk de serverinstellingen aanpassen (raadpleeg je hostingprovider of WordPress-beheerder).
