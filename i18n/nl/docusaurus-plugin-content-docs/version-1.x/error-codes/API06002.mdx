# API06002: Ontbrekende API-URL

## Wat dit betekent {#what-this-means}

Er is geen URL geconfigureerd voor de verbinding met je WooCommerce-winkel. De POS moet weten waar je winkel zich bevindt.

## Veelvoorkomende oorzaken {#common-causes}

- **Eerste installatie** — De URL is nooit ingevoerd
- **Configuratie gewist** — De instellingen zijn gereset of gewist
- **App-gegevens gewist** — De applicatiegegevens zijn verwijderd
- **Migratieprobleem** — De URL is verloren gegaan tijdens een update of migratie

## Hoe op te lossen {#how-to-fix}

### 1. Voer de URL van je winkel in {#1-enter-your-store-url}

In de POS-app:
1. Open het instellingen- of inlogscherm
2. Zoek het configuratieveld voor de URL/Site
3. Voer de URL van je WordPress-site in
4. Voorbeeld: `https://yourstore.com`

### 2. Vind je WordPress-URL {#2-find-your-wordpress-url}

Als je niet zeker weet wat je URL is:
1. Log in op het WordPress-beheer
2. Ga naar Instellingen → Algemeen
3. Bekijk "WordPress-adres (URL)"
4. Gebruik deze URL in de POS

### 3. Controleer de opgeslagen instellingen {#3-check-stored-settings}

Als de URL eerder was geconfigureerd:
- Controleer of de app-gegevens per ongeluk zijn gewist
- Zoek naar opties voor back-up/export van instellingen
- Voer de URL indien nodig opnieuw in

### 4. Controleer of de URL correct is {#4-verify-url-is-correct}

Voordat je opslaat:
1. Test de URL in een browser
2. Zorg ervoor dat deze je WordPress-site laadt
3. Voeg `https://` of `http://` toe

## Installatiechecklist {#setup-checklist}

Bij het voor de eerste keer configureren van WCPOS:

1. ☐ Voer de URL van je WordPress-site in
2. ☐ Controleer of de WCPOS-plugin is geïnstalleerd en actief is
3. ☐ Log in met je WordPress-inloggegevens
4. ☐ Laat de eerste gegevenssynchronisatie voltooien
