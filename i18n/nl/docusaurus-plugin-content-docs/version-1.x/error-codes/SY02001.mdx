# SY02001: Ongeldige configuratie

## Wat dit betekent {#what-this-means}

De systeemconfiguratie is ongeldig of beschadigd. De POS-applicatie heeft configuratie-instellingen aangetroffen die geen logische waarden hebben of onvolledig zijn.

## Veelvoorkomende oorzaken {#common-causes}

- **Beschadigde instellingen** — Het configuratiebestand is beschadigd geraakt
- **Onvolledige installatie** — Het installatieproces is niet voltooid
- **Fout bij handmatige bewerking** — Configuratiebestanden zijn onjuist gewijzigd
- **Probleem bij versie-upgrade** — Oude configuratie is incompatibel met de nieuwe versie

## Hoe op te lossen {#how-to-fix}

### 1. Terugzetten naar standaardinstellingen {#1-reset-to-default-settings}

Indien beschikbaar in de app:
- Zoek naar "Instellingen resetten" of "Standaardwaarden herstellen"
- Hiermee worden aangepaste instellingen gewist, maar wordt de beschadiging hersteld

### 2. De installatie voltooien {#2-complete-the-setup}

Als de installatie niet is voltooid:
1. Start het installatieproces opnieuw
2. Voltooi alle vereiste stappen
3. Controleer de configuratie voordat je afrondt

### 3. Applicatiegegevens wissen {#3-clear-application-data}

Verwijder de beschadigde configuratie:
- **Desktop:** Verwijder de app-gegevensmap, herstart de app
- **Browser:** Wis de sitegegevens, herlaad de pagina

### 4. De applicatie opnieuw installeren {#4-reinstall-the-application}

Bij aanhoudende problemen:
1. Verwijder de app volledig
2. Verwijder eventuele achtergebleven gegevensmappen
3. Installeer opnieuw vanaf nul
4. Voltooi de installatie helemaal opnieuw

### 5. Controleren op updates {#5-check-for-updates}

Configuratie-indelingen kunnen veranderen:
- Werk de POS-applicatie bij
- Werk de WCPOS-plugin bij
- Configureer opnieuw na de update

## Wat wordt er gereset {#what-gets-reset}

Wanneer je de configuratie reset:
- Moet je opnieuw inloggen
- Gaan aangepaste instellingen verloren
- Moeten gegevens opnieuw worden gesynchroniseerd
- Maar je WooCommerce-gegevens zijn veilig op de server
