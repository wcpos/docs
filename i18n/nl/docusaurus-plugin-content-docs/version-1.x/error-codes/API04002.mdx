# API04002: Onverwachte responscode

## Wat dit betekent {#what-this-means}

De server retourneerde een HTTP-statuscode die niet werd verwacht voor dit verzoek. Veelvoorkomende codes zijn onder andere 500 (serverfout), 403 (verboden), 404 (niet gevonden), enz.

## Veelvoorkomende oorzaken {#common-causes}

- **500 Internal Server Error** — PHP is gecrasht of liep tegen een fout aan
- **403 Forbidden** — Toegang geweigerd door beveiligingsinstellingen
- **404 Not Found** — Het eindpunt bestaat niet
- **502/503/504** — Problemen met de servergateway of beschikbaarheid

## Hoe op te lossen {#how-to-fix}

### Voor 500-fouten (serverfout) {#for-500-errors-server-error}

1. Controleer de PHP-foutlogs
2. Verhoog de PHP-geheugenlimiet
3. Zoek naar pluginconflicten
4. Controleer `wp-content/debug.log`

### Voor 403-fouten (verboden) {#for-403-errors-forbidden}

1. Controleer de instellingen van de beveiligingsplugin (Wordfence, Sucuri, enz.)
2. Zet de POS of je IP op de witte lijst
3. Controleer `.htaccess` op blokkeerregels
4. Verifieer dat ModSecurity de verzoeken niet blokkeert

### Voor 404-fouten (niet gevonden) {#for-404-errors-not-found}

1. Verifieer dat de WCPOS-plugin actief is
2. Vernieuw de WordPress-permalinks (Instellingen → Permalinks → Opslaan)
3. Controleer of de REST API is ingeschakeld
4. Verifieer de URL-configuratie

### Voor 502/503/504-fouten (gatewayproblemen) {#for-502503504-errors-gateway-issues}

1. Neem contact op met je hostingprovider
2. Wacht tot de server is hersteld
3. Controleer of de site zwaar belast is
4. Verifieer dat de server actief is

### Algemene probleemoplossing {#general-troubleshooting}

1. Probeer je site rechtstreeks te openen
2. Controleer het hostingcontrolepaneel op problemen
3. Bekijk de toegangslogs van de server
4. Neem indien nodig contact op met de hostingsupport
