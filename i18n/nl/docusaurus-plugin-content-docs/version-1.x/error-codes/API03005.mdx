# API03005: Snelheidslimiet overschreden

## Wat dit betekent {#what-this-means}

Je hebt in korte tijd te veel verzoeken naar de server gestuurd. Snelheidsbeperking beschermt de server tegen overbelasting.

## Veelvoorkomende oorzaken {#common-causes}

- **Snelle synchronisatiebewerkingen** — Meerdere apparaten die tegelijk synchroniseren
- **Agressief vernieuwen** — Data te vaak vernieuwen
- **Beveiligingsplugin** — Een plugin handhaaft verzoeklimieten
- **Hostinglimieten** — Je hostingpakket heeft API-snelheidslimieten

## Servercodetoewijzing {#server-error-mapping}

Deze foutcode wordt geactiveerd wanneer de server het volgende retourneert:

| Servercode | Bron |
|-------------|--------|
| HTTP 429 | Elke serverrespons (terugval) |

## Hoe op te lossen {#how-to-fix}

### 1. Wachten en opnieuw proberen {#1-wait-and-retry}

Snelheidslimieten zijn tijdelijk:
- Wacht 1-5 minuten
- De POS probeert het automatisch opnieuw
- Vermijd herhaaldelijk handmatig vernieuwen

### 2. Verminder gelijktijdige bewerkingen {#2-reduce-concurrent-operations}

Bij gebruik van meerdere POS-terminals:
- Spreid de eerste synchronisatie over de apparaten
- Vermijd het tegelijk vernieuwen van alle apparaten
- Laat er één voltooien voordat je een andere start

### 3. Controleer beveiligingsplugins {#3-check-security-plugins}

Plugins zoals Wordfence kunnen snelle verzoeken blokkeren:
- Zet de POS-applicatie op de witte lijst
- Zet de IP-adressen van je POS-apparaten op de witte lijst
- Pas de drempels voor snelheidsbeperking aan

### 4. Bekijk hostinglimieten {#4-review-hosting-limits}

Sommige hosts beperken API-verzoeken:
- Controleer de details van je hostingpakket
- Overweeg een upgrade als de limieten te laag zijn
- Neem contact op met support over het verhogen van limieten

### 5. Optimaliseer de synchronisatiefrequentie {#5-optimise-sync-frequency}

In de POS-instellingen:
- Pas de intervallen voor automatisch vernieuwen aan
- Gebruik handmatige synchronisatie wanneer dat gepast is
- Vermijd onnodige dataverversingen
