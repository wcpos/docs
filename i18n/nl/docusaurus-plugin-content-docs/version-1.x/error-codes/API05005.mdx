# API05005: Plugin niet gevonden

## Wat dit betekent {#what-this-means}

Een vereiste WordPress-plugin is niet geïnstalleerd of niet actief. Dit kan WooCommerce zelf zijn, een vereiste uitbreiding of een integratieplugin.

## Veelvoorkomende oorzaken {#common-causes}

- **WooCommerce gedeactiveerd** — WooCommerce was uitgeschakeld
- **Vereiste uitbreiding ontbreekt** — Een benodigde WooCommerce-uitbreiding is niet geïnstalleerd
- **Pluginconflict** — Een andere plugin heeft ervoor gezorgd dat een vereiste plugin werd gedeactiveerd
- **Mislukte update** — Een plugin-update is mislukt en heeft de plugin gedeactiveerd achtergelaten

## Hoe op te lossen {#how-to-fix}

### 1. Controleer of WooCommerce actief is {#1-verify-woocommerce-is-active}

WCPOS vereist WooCommerce:
1. Ga naar Plugins → Geïnstalleerde plugins
2. Zoek "WooCommerce"
3. Zorg ervoor dat deze geactiveerd is

### 2. Controleer op vereiste uitbreidingen {#2-check-for-required-extensions}

Sommige WCPOS-functies vereisen mogelijk uitbreidingen:
- Controleer de foutmelding op een specifieke pluginnaam
- Installeer en activeer de vereiste uitbreiding

### 3. Bekijk pluginconflicten {#3-review-plugin-conflicts}

Als plugins werden gedeactiveerd vanwege conflicten:
1. Controleer je e-mail op WordPress-meldingen
2. Bekijk de foutlogboeken
3. Los de conflicten op en activeer opnieuw

### 4. Controleer de pluginbestanden {#4-check-plugin-files}

Als de bestanden van een plugin ontbreken:
1. Installeer de plugin opnieuw vanaf WordPress.org
2. Of herstel vanuit een back-up
3. Activeer de plugin

### 5. Update alle plugins {#5-update-all-plugins}

Verouderde plugins kunnen problemen veroorzaken:
1. Ga naar Dashboard → Updates
2. Update alle plugins
3. Controleer of alle vereiste plugins actief zijn

## Vereiste plugins {#required-plugins}

WCPOS vereist minimaal:
- **WooCommerce** — Het e-commerceplatform
- **WCPOS** — De POS-plugin

Aanvullende functies vereisen mogelijk andere plugins.
