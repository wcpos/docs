:::info Pro Feature
NL: The Customers screen requires [WCPOS Pro](/getting-started/pro-license). Free users can select existing customers for orders but cannot view the full customer list or edit customer details.
:::

NL: The Customers screen provides comprehensive customer management directly within the POS. View, edit, and create customers without switching to the WooCommerce admin.

## Vertaald: Interface Overview {#interface-overview}

### Vertaald: Header Actions {#header-actions}

NL: At the top of the screen:

- NL: **Search bar** - Find customers by name, email, etc.
- NL: **Add Customer** (<Icon name="plus" />) - Create a new customer
- NL: **Display settings** (<Icon name="sliders" />) - Configure visible columns

### Vertaald: Customer Table {#customer-table}

NL: The main area displays customers with:

- NL: **Avatar** - Customer profile picture (or placeholder)
- NL: **First Name** - Customer's first name
- NL: **Last Name** - Customer's last name (sortable)
- NL: **Email** - Contact email address
- NL: **Billing Address** - Full billing address
- NL: **Date Created** - When the customer was added
- NL: **Actions** - Three-dot menu

### Vertaald: Footer {#footer}

- NL: Customer count with sync button (<Icon name="arrow-rotate-right" />). **Long press** for Clear and Refresh option

## Vertaald: Key Features {#key-features}

### Vertaald: Customer Search {#customer-search}

NL: Find customers quickly by:

- NL: First or last name
- NL: Email address
- NL: Address information

### Vertaald: Add New Customer {#add-new-customer}

NL: Create customers directly from the POS:

1. NL: Click the <Icon name="plus" /> icon in the header
2. NL: Fill in customer details
3. NL: Save the new customer

NL: The customer is created in WooCommerce and immediately available for orders.

### Vertaald: Edit Customer {#edit-customer}

NL: Update customer information:

1. NL: Click the three-dot menu on a customer
2. NL: Select **Edit**
3. NL: Modify details (name, email, addresses)
4. NL: Save changes

NL: Changes sync to WooCommerce automatically.

## Vertaald: Customer Actions {#customer-actions}

NL: Click the three-dot menu (⋮) for options:

- NL: **Edit** - Modify customer details
- NL: **Sync** - Refresh customer from server
- NL: **Delete** - Remove from local database

:::note
NL: Deleting a customer from the POS only removes them locally. The customer remains in WooCommerce and will reappear on the next sync.
:::

## Vertaald: Default Customer (Guest) {#default-customer-guest}

NL: When no customer is selected, orders are placed as **Guest** orders. Guest orders:

- NL: Have no customer name, email, or address attached
- NL: Cannot be looked up by customer in order history
- NL: Are still visible in the Orders screen and WooCommerce admin

NL: To avoid Guest orders, select a customer from the Cart Panel before checkout. If your business requires a customer on every order, consider training cashiers to always assign one — there is no built-in setting to enforce this.

## Vertaald: Using Customers in Orders {#using-customers-in-orders}

NL: To assign a customer to an order:

1. NL: In the [Cart Panel](/pos/cart), click the customer badge
2. NL: Search for the customer by name, email, or phone number
3. NL: Select the customer
4. NL: The customer is now associated with the order

NL: Customer information (billing/shipping addresses) is automatically used for the order.

## Vertaald: Synchronization {#synchronization}

NL: Customer data syncs between the POS and WooCommerce:

- NL: **Real-time updates** - Changes sync across all devices
- NL: **WooCommerce integration** - Customers are stored in WooCommerce
- NL: **Offline capability** - View customers even when offline
