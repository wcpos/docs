---
title: Serveurs Legacy
description: Si votre hébergeur web bloque les méthodes HTTP RESTful modernes, vous devrez activer la prise en charge des serveurs legacy.
---

HTTP (et HTTPS) sont les protocoles utilisés par tous les navigateurs pour communiquer avec les applications web.
Un élément clé du protocole est la [méthode HTTP](https://tools.ietf.org/html/rfc2616#section-5.1.1).
Traditionnellement, les applications web utilisaient un ensemble très limité de méthodes HTTP comme GET (récupérer une page web) et POST (soumettre un formulaire web).

Les applications web modernes, comme WCPOS, utilisent une approche REST/HTTP avec les méthodes HTTP suivantes :

- GET
- POST
- PUT
- PATCH
- DELETE

Dans de rares cas, un hébergeur web peut choisir de bloquer certaines de ces méthodes HTTP.
Si votre hébergeur web bloque les méthodes HTTP RESTful modernes, vous devrez activer la prise en charge des serveurs legacy.

![Prise en charge des serveurs legacy](https://wcpos.com/wp-content/uploads/2016/06/enable-legacy-server-support.png "Administration WP > POS > Paramètres > Outils > Activer la prise en charge des serveurs legacy")
