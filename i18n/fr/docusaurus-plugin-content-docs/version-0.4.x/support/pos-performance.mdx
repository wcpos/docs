---
title: Performance du POS
---

WCPOS est une application web JavaScript à page unique qui communique avec votre serveur via l'[API REST WooCommerce](http://woothemes.github.io/woocommerce-rest-api-docs/). Les problèmes de performance peuvent être divisés en deux catégories : côté serveur et côté client.

### Performance côté serveur

La performance côté serveur fait référence à des actions telles que le téléchargement d'une page de produits (10 produits) ou le traitement d'une commande. Les problèmes pouvant affecter la vitesse de ces actions incluent :

- La vitesse de votre connexion internet
- La vitesse de votre serveur, c'est-à-dire : puissance de traitement, RAM, charge du serveur, etc.
- Le nombre de plugins actifs
- Le traitement de la passerelle de paiement

Le [site de démonstration](http://demo.wcpos.com/pos) représente un scénario optimal. Le site de démonstration utilise [un très bon hébergeur web](http://wcpos.com/wpe) et a un très petit nombre de plugins activés.

| Processus                  | Vitesses moyennes pour [demo.wcpos.com/pos](http://demo.wcpos.com/pos)<sup>\*</sup> |
| ------------------------ | ------------------------------------------------------------------------------------ |
| Récupération de 10 produits | 1 - 2 secondes                                                                      |
| Traitement d'une vente en espèces | 1 - 2 secondes                                                                |
| Traitement d'une vente Stripe | 2 - 3 secondes                                                                    |

<small>
  <sup>*</sup> temps moyens pour une connexion haut débit. Les temps peuvent varier
  selon votre vitesse internet.
</small>

Si votre POS prend significativement plus de temps que les temps ci-dessus, vous pouvez effectuer les tests suivants :

1.  Basculer le thème vers le thème Twenty Fifteen par défaut de WordPress
2.  Désactiver tous les plugins sauf WooCommerce et WCPOS
3.  [Effacer les données du stockage local](http://faq.wcpos.com/en/clear-local-data.html)
4.  Utiliser le POS pour voir s'il y a une augmentation des performances
5.  **Si les performances augmentent :** réactiver votre thème et les plugins un par un pour voir quel plugin impacte les performances
6.  **S'il n'y a pas d'augmentation des performances :** vous pouvez envisager de migrer vers un meilleur hébergeur ou investir dans une connexion internet plus rapide

### Performance côté client

WCPOS utilise JavaScript, HTML et CSS pour afficher les données récupérées à partir de l'API REST WooCommerce, c'est-à-dire : [produits](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) et [commandes](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order).
Pour améliorer les performances côté client, le POS stocke les données dans le navigateur en utilisant [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API).
Par exemple, lorsqu'un produit est récupéré pour la première fois, une requête sera envoyée au serveur, une fois les données du produit téléchargées, elles seront stockées localement pour que les recherches suivantes soient instantanées.

![Un exemple de produit stocké localement.](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "Un exemple de produits stockés localement")

Les données IndexedDB persistent même lorsque vous fermez le navigateur ou redémarrez votre ordinateur.
Si vous remarquez que vos données de produit ne sont pas synchronisées pour une raison quelconque, vous pouvez [effacer le stockage local](http://faq.wcpos.com/en/clear-local-data.html) et récupérer un nouvel ensemble de données à partir du serveur.
