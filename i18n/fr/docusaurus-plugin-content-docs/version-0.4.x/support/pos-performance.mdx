---
title: Performance du POS
---

WCPOS est une application web JavaScript monopage qui communique avec votre serveur via l'[API REST WooCommerce](http://woothemes.github.io/woocommerce-rest-api-docs/). Les problèmes de performance peuvent être séparés en deux catégories : côté serveur et côté clients.

### Performance côté serveur {#server-side-performance}

La performance côté serveur fait référence à des actions telles que le téléchargement d'une page de produits (10 produits) ou le traitement d'une commande. Les problèmes qui peuvent affecter la vitesse de ces actions comprennent :

- La vitesse de votre connexion Internet
- La vitesse de votre serveur, c'est-à-dire : puissance de traitement, RAM, charge du serveur, etc.
- Combien de plugins vous avez activés
- Traitement par la passerelle de paiement

Le [site de démonstration](http://demo.wcpos.com/pos) représente un scénario de 'meilleur cas'. Le site de démonstration utilise [un très bon hébergeur](http://wcpos.com/wpe) et a un très petit nombre de plugins activés.

| Processus                | Vitesses moyennes pour [demo.wcpos.com/pos](http://demo.wcpos.com/pos)<sup>\*</sup> |
| ------------------------ | ------------------------------------------------------------------------------- |
| Récupération de 10 produits | 1 - 2 secondes                                                                   |
| Traitement d'une vente au comptant | 1 - 2 secondes                                                                   |
| Traitement d'une vente Stripe | 2 - 3 secondes                                                                   |

<small>
  <sup>*</sup> temps moyens pour une connexion haut débit. Les temps peuvent varier
  en fonction de votre vitesse Internet.
</small>

Si votre POS prend significativement plus de temps que les délais ci-dessus, vous pouvez souhaiter effectuer les tests suivants :

1.  Passez votre thème au thème par défaut Twenty Fifteen de WordPress
2.  Désactivez tous les plugins sauf WooCommerce et WCPOS
3.  [Effacez les données de stockage local](http://faq.wcpos.com/en/clear-local-data.html)
4.  Maintenant, utilisez le POS pour voir s'il y a une augmentation de performance
5.  **S'il y a une augmentation de performance :** réactivez votre thème et vos plugins un par un pour voir quel plugin impacte vos performances
6.  **S'il n'y a pas d'augmentation de performance :** vous voudrez peut-être passer à un meilleur hébergeur ou investir dans une connexion Internet plus rapide

### Performance côté clients {#client-side-performance}

WCPOS utilise JavaScript, HTML et CSS pour afficher les données récupérées de l'API REST WooCommerce, c'est-à-dire : [produits](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) et [commandes](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order).
Pour améliorer la performance côté clients, le POS stocke les données dans le navigateur en utilisant [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API).
Par exemple, lorsque qu'un produit est récupéré pour la première fois, une demande sera envoyée au serveur, une fois que les données du produit sont téléchargées, elles seront stockées localement afin que les recherches suivantes soient instantanées.

![Un exemple de produit stocké localement.](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "Un exemple de produits stockés localement")

Les données d'IndexedDB persistent même lorsque vous fermez le navigateur ou redémarrez votre ordinateur.
Si vous constatez que vos données produit ne sont pas synchronisées pour une raison quelconque, vous pouvez [effacer le stockage local](http://faq.wcpos.com/en/clear-local-data.html) et récupérer un nouvel ensemble de données du serveur.
