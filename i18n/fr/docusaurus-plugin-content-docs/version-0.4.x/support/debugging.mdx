:::caution
Cet article est destiné aux développeurs web, le débogage n'est pas quelque chose que l'utilisateur moyen devrait avoir besoin d'utiliser.
:::

WCPOS est principalement une application web JavaScript - et en tant que telle - le débogage se fait principalement via la [console du navigateur](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors).
Si le débogage est activé, WCPOS enregistrera divers messages dans la console qui peuvent aider à diagnostiquer des problèmes.

![Une console JavaScript ouverte dans Chrome sur Mac](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Une console JavaScript ouverte dans Chrome sur Mac")

### Activer le débogage à l'aide de wp-config.php

En définissant le drapeau [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) dans votre fichier wp-config.php, vous pouvez activer le débogage de WCPOS.
Cela chargera les fichiers de script non minifiés et activera également les messages de débogage dans la console du navigateur.

### Activer le débogage à l'aide de la console du navigateur

Si vous n'avez pas accès au fichier wp-config.php, vous pouvez activer les messages de débogage en ajoutant un drapeau `debug` dans le stockage local.
Cela peut être fait en entrant `localStorage.setItem('debug', '*')` dans la console.
Actualisez maintenant la page et le débogage sera activé. La journalisation de console peut consommer des ressources du navigateur, vous voudrez donc probablement désactiver le débogage une fois que vous aurez terminé.
Pour désactiver le débogage, entrez simplement `localStorage.removeItem('debug')` dans la console et actualisez la page.
