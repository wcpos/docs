# Passerelle Facture par e-mail {#email-invoice-gateway}

La passerelle Facture par e-mail permet d'envoyer des factures de paiement aux clients par e-mail directement depuis WCPOS. Cette option est idéale lorsque les clients préfèrent payer ultérieurement ou doivent finaliser le paiement en ligne via les moyens de paiement de votre boutique en ligne.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Installer la passerelle de facture par e-mail">

Installez depuis `WP Admin > POS > Réglages > Extensions`, ou téléchargez le dernier **fichier zip de l'extension** (et non le zip ou l'archive du code source GitHub) depuis la [page des releases GitHub](https://github.com/wcpos/email-invoice-gateway/releases) et téléversez-le via `Extensions > Ajouter > Téléverser une extension`.

  </StepCard>
  <StepCard number="2" title="Activer dans WCPOS" isLast>

1. Accédez à `WP Admin > POS > Réglages > Validation de la commande`
2. Recherchez la **Passerelle de paiement par facture WCPOS** dans la liste
3. Activez la passerelle pour l'utiliser dans le POS
4. Enregistrez vos réglages

:::note
Cette passerelle ne nécessite aucune clé API supplémentaire ni configuration de service externe. Elle utilise votre système de messagerie WordPress existant et les méthodes de paiement WooCommerce.
:::

  </StepCard>
</Steps>

## Utilisation {#usage}

### Envoi de factures {#sending-invoices}

1. **Ajouter des articles** : ajoutez des produits à votre panier dans le PDV
2. **Sélectionner la passerelle** : choisissez « Facture par e-mail » comme méthode de paiement
3. **Adresse e-mail** : 
   - Pour les clients existants : l'adresse e-mail est automatiquement renseignée à partir des données du client
   - Pour les commandes invité : saisissez manuellement l'adresse e-mail du client
4. **Facultatif** : cochez « Enregistrer dans l'adresse de facturation » pour mettre à jour les informations de facturation du client
5. **Envoyer la facture** : finalisez le processus pour envoyer la facture par e-mail

### Expérience client {#customer-experience}

Lorsque vous envoyez une facture, le client reçoit un e-mail contenant :

- **Détails de la commande** : récapitulatif complet des articles, quantités et prix
- **Total de la commande** : montant final dû, taxes et frais inclus
- **Lien de paiement** : lien direct pour finaliser le paiement en ligne
- **Informations de commande** : numéro de commande et détails de référence

### Finalisation du paiement {#payment-completion}

Le client peut finaliser le paiement de deux manières :

1. **Paiement en ligne** : cliquer sur le lien « Payer cette commande » dans l'e-mail
2. **Retour en boutique** : revenir en boutique pour finaliser le paiement avec d'autres méthodes de paiement du PDV

Le lien « Payer cette commande » dirige le client vers une page de paiement dédiée sur votre site web, où il peut utiliser n'importe quelle passerelle de paiement en ligne activée (Stripe, PayPal, etc.).

## Gestion des e-mails {#email-management}

### Clients existants {#existing-customers}

- Les adresses e-mail sont automatiquement récupérées depuis les profils clients
- Les informations de facturation du client sont conservées
- L'historique des commandes est conservé dans le compte client

### Clients invités {#guest-customers}

- Saisie manuelle des adresses e-mail pour les clients occasionnels
- Possibilité d'enregistrer l'adresse e-mail dans l'adresse de facturation pour référence ultérieure
- Les commandes invités sont correctement suivies dans WooCommerce

### Modèles d'e-mails {#email-templates}

La passerelle utilise les modèles d'e-mails standard de WooCommerce :

- **Personnalisable** : modification de l'apparence des e-mails via les paramètres d'e-mails de WooCommerce
- **Personnalisé** : intégration du logo et de l'identité visuelle de la boutique
- **Professionnel** : format de facture clair et soigné
- **Adapté aux mobiles** : design responsive pour tous les appareils

## Cas d'utilisation {#use-cases}

### Idéal pour {#perfect-for}

- **Ventes B2B** : les clients professionnels qui doivent traiter les paiements via leur système comptable
- **Commandes importantes** : les transactions de montant élevé nécessitant une approbation ou un délai de traitement
- **Clients à distance** : les clients qui doivent finaliser le paiement après avoir quitté la boutique
- **Clients avec compte** : les clients réguliers bénéficiant de conditions de paiement établies
- **Devis vers commande** : convertir des devis ou estimations en factures payables

## Dépannage {#troubleshooting}

### Problèmes courants {#common-issues}

<Accordion>
  <AccordionItem question="Les e-mails ne s'envoient pas">

  - Vérifier la configuration des e-mails de WordPress
  - Vérifier les paramètres SMTP en cas d'utilisation d'un service de messagerie personnalisé
  - Tester avec des extensions WordPress de test d'e-mails
  - Vérifier les dossiers de courrier indésirable pour les e-mails de test

  </AccordionItem>

  <AccordionItem question="Les liens de paiement ne fonctionnent pas">

  - S'assurer que WooCommerce est correctement configuré
  - Vérifier qu'au moins une passerelle de paiement est activée pour le paiement en ligne
  - Vérifier que le certificat SSL est correctement installé
  - Confirmer que le statut de la commande autorise le paiement

  </AccordionItem>

  <AccordionItem question="Le client ne peut pas finaliser le paiement">

  - Vérifier que les passerelles de paiement web sont actives et configurées
  - Vérifier que la commande n'a pas expiré ou été annulée
  - S'assurer que le client dispose de limites de moyen de paiement suffisantes
  - Tester le processus de paiement soi-même

  </AccordionItem>
</Accordion>

### Livraison des e-mails {#email-delivery}

Pour une livraison fiable des e-mails, envisager :

- **Service SMTP** : Utiliser des services tels que SendGrid, Mailgun ou Amazon SES
- **Extensions d'e-mail** : Installer des extensions SMTP WordPress pour une meilleure livraison
- **Authentification du domaine** : Configurer les enregistrements SPF, DKIM et DMARC
- **Suivi** : suivre la livraison des e-mails et les taux d'ouverture
