# Passerelle SumUp Terminal {#sumup-terminal-gateway}

La passerelle SumUp Terminal permet d'accepter les paiements par carte à l'aide de lecteurs de cartes SumUp directement dans WCPOS. Cette passerelle offre une intégration transparente avec le système de traitement des paiements SumUp pour les transactions en personne.

## Fonctionnalités {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Intégration matérielle">
    Connecter les lecteurs de cartes SumUp au système de point de vente via une connexion internet
  </FeatureCard>
  <FeatureCard icon="bolt" title="Traitement en temps réel">
    Traitement instantané des paiements et finalisation des commandes

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Installer SumUp Terminal pour WooCommerce">

Installez depuis `WP Admin > POS > Réglages > Extensions`, ou téléchargez le dernier **fichier zip du plugin** (et non le zip ou tarball du code source GitHub) depuis la [page des releases GitHub](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) et importez-le via `Extensions > Ajouter > Téléverser une extension`.

  </StepCard>
  <StepCard number="2" title="Configurer les réglages SumUp">

1. Accédez à `WP Admin > WooCommerce > Réglages > Paiements`
2. Recherchez **SumUp Terminal** dans la liste des moyens de paiement
3. Cliquez sur **SumUp Terminal** pour accéder aux réglages
4. Saisissez votre **clé API SumUp** (disponible depuis votre tableau de bord marchand SumUp)
5. Enregistrez les réglages

:::note
Il n'est pas nécessaire d'activer la passerelle SumUp Terminal dans les réglages WooCommerce. Elle sera activée spécifiquement pour le PDV lors d'une étape ultérieure.
:::

  </StepCard>
  <StepCard number="3" title="Associer votre SumUp Terminal">

1. Sur la même page de réglages, repérez la section **Associer le lecteur**
2. Sur votre appareil SumUp, accédez à l'écran d'association pour afficher le code d'association
3. Saisissez le code d'association affiché sur votre appareil SumUp
4. Cliquez sur **« Associer le lecteur »** pour établir la connexion
5. Attendez la confirmation que le lecteur a été associé avec succès

:::warning Important
Le lecteur doit être associé avec succès avant de pouvoir traiter des paiements. Assurez-vous que le processus d'association est terminé avant de continuer.
:::

  </StepCard>
  <StepCard number="4" title="Activer dans WCPOS" isLast>

1. Accédez à `WP Admin > POS > Réglages > Validation de la commande`
2. Recherchez la passerelle **SumUp Terminal** dans la liste
3. Activez la passerelle pour l'utiliser dans le PDV
4. Enregistrez vos paramètres

  </StepCard>
</Steps>

## Utilisation {#usage}

### Traitement des paiements {#processing-payments}

1. **Ajouter des articles** : ajoutez des produits à votre panier dans le PDV
2. **Sélectionner la passerelle** : choisissez « SumUp Terminal » comme moyen de paiement
3. **Lancer le paiement** : cliquez pour initier un nouveau paiement sur votre appareil SumUp
4. **Paiement du client** : le client effectue le paiement sur le terminal SumUp
5. **Finalisation automatique** : la commande est automatiquement finalisée lorsque le paiement est détecté

### Contrôles de paiement {#payment-controls}

Lorsque vous utilisez la passerelle SumUp Terminal, les options suivantes sont disponibles :

- **Démarrer un nouveau paiement** : Initier une demande de paiement sur le terminal connecté
- **Annuler le paiement** : Annuler un paiement en cours de traitement
- **Statut du paiement** : Suivre l'état actuel du processus de paiement

### Gestion des commandes {#order-management}

- **Mises à jour automatiques** : Les commandes sont automatiquement marquées comme terminées après un paiement réussi
- **Suivi des paiements** : Tous les détails du paiement sont enregistrés dans les notes de commande
- **Génération de reçus** : Les reçus PDV standard sont générés après les paiements réussis

## Compatibilité matérielle {#hardware-compatibility}

:::important Exigences de connectivité
Cette implémentation utilise le SDK JavaScript de SumUp, ce qui signifie qu'elle fonctionne via des applications web mais nécessite **uniquement des terminaux connectés à Internet**. Les terminaux Bluetooth ne sont pas pris en charge actuellement.
:::

### Terminaux pris en charge (connectés à Internet) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Terminal connecté en WiFi/Ethernet

### Terminaux non pris en charge (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Bluetooth uniquement
- **SumUp 3G** ❌ - Bluetooth uniquement

:::note Prise en charge future
La prise en charge des terminaux Bluetooth est prévue pour une future version de l'application iOS et Android. Une fois disponible, cela permettra la prise en charge de tous les terminaux certifiés SumUp, y compris les modèles Air et 3G.
:::

## Dépannage {#troubleshooting}

### Problèmes courants {#common-issues}

<Accordion>
  <AccordionItem question="Le lecteur ne s'appaire pas">

  - Assurez-vous d'utiliser un terminal connecté à Internet pris en charge (Solo uniquement)
  - Vérifiez que votre appareil SumUp Solo est connecté au WiFi et en ligne
  - Vérifiez que l'appareil est en mode d'appairage et que le code d'appairage est saisi correctement
  - Vérifiez que l'accès API est activé sur votre compte SumUp
  - Assurez-vous de disposer d'une connexion internet stable pendant l'appairage

  </AccordionItem>

  <AccordionItem question="Le paiement ne s'exécute pas">

  - Vérifiez que le lecteur est correctement appairé
  - Vérifiez que votre compte SumUp est actif et en règle
  - Assurez-vous que l'appareil dispose d'une batterie suffisante et d'une connectivité adéquate
  - Vérifiez que la clé API est correcte et active

  </AccordionItem>

  <AccordionItem question="La commande ne se finalise pas">

  - Vérifiez la stabilité de la connexion internet
  - Vérifiez les paramètres de webhook dans votre compte SumUp
  - Assurez-vous que le paiement a bien été effectué sur le terminal SumUp
  - Consultez les journaux d'erreurs WordPress pour détecter d'éventuels problèmes de communication avec l'API

  </AccordionItem>
</Accordion>

### Limites de débit API {#api-rate-limits}

La passerelle est optimisée pour réduire les appels API et éviter la limitation de débit :
- Le statut de paiement est vérifié de manière efficace
- Les requêtes API inutiles sont minimisées
- La logique de nouvelle tentative automatique gère les échecs temporaires
