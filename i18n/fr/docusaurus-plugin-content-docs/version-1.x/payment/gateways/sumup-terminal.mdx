# Passerelle Terminal SumUp {#sumup-terminal-gateway}

La passerelle Terminal SumUp vous permet d'accepter les paiements par carte en utilisant les lecteurs de cartes SumUp directement au sein de WCPOS. Cette passerelle offre une intégration transparente avec le système de traitement des paiements de SumUp pour les transactions en personne.

## Installation

<Steps>
  <StepCard number="1" title="Télécharger et Installer">

1. Visitez la [page des versions du Terminal SumUp pour WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Téléchargez le fichier **sumup-terminal-for-woocommerce.zip** le plus récent
3. Dans votre espace d'administration WordPress, allez à `Plugins > Ajouter Nouveau > Télécharger le Plugin`
4. Téléchargez le fichier zip et activez le plugin

  </StepCard>
  <StepCard number="2" title="Configurer les Paramètres SumUp">

1. Naviguez vers `WP Admin > WooCommerce > Paramètres > Paiements`
2. Trouvez **Terminal SumUp** dans la liste des méthodes de paiement
3. Cliquez sur **Terminal SumUp** pour accéder aux paramètres
4. Entrez votre **Clé API SumUp** (disponible depuis votre tableau de bord marchand SumUp)
5. Enregistrez les paramètres

:::note
Vous n'avez pas besoin d'activer la passerelle Terminal SumUp dans les paramètres WooCommerce. Elle sera activée spécifiquement pour le PDV dans une étape ultérieure.
:::

  </StepCard>
  <StepCard number="3" title="Appairer Votre Terminal SumUp">

1. Sur la même page de paramètres, localisez la section **Appairer le Lecteur**
2. Sur votre appareil SumUp, allez à l'écran d'appariement pour afficher le code d'appariement
3. Entrez le code d'appariement affiché sur votre appareil SumUp
4. Cliquez sur **"Appairer le Lecteur"** pour établir la connexion
5. Attendez la confirmation que le lecteur a été apparié avec succès

:::warning Important
Le lecteur doit être apparié avec succès avant que vous puissiez traiter les paiements. Assurez-vous que le processus d'appariement est terminé avant de continuer.
:::

  </StepCard>
  <StepCard number="4" title="Activer dans WCPOS" isLast>

1. Allez à `WP Admin > PDV > Paramètres > Paiement`
2. Trouvez la passerelle **Terminal SumUp** dans la liste
3. Activez la passerelle pour une utilisation dans le PDV
4. Enregistrez vos paramètres

  </StepCard>
</Steps>

## Utilisation {#usage}

### Traitement des Paiements {#processing-payments}

1. **Ajouter des Articles**: Ajoutez des produits à votre panier dans le PDV
2. **Sélectionner la Passerelle**: Choisissez "Terminal SumUp" comme méthode de paiement
3. **Commencer le Paiement**: Cliquez pour initier un nouveau paiement sur votre appareil SumUp
4. **Paiement Client**: Le client complète le paiement sur le terminal SumUp
5. **Achèvement Automatique**: La commande se termine automatiquement lorsque le paiement est détecté

### Contrôles de Paiement {#payment-controls}

Lorsque vous utilisez la passerelle Terminal SumUp, vous avez les options suivantes :

- **Démarrer un Nouveau Paiement**: Initier une demande de paiement sur le terminal connecté
- **Annuler le Paiement**: Annuler un paiement actuellement en cours
- **Statut du Paiement**: Surveiller l'état actuel du processus de paiement

### Gestion des Commandes {#order-management}

- **Mises à Jour Automatiques**: Les commandes sont automatiquement marquées comme complètes après un paiement réussi
- **Suivi des Paiements**: Tous les détails de paiement sont enregistrés dans les notes de commande
- **Génération de Reçus**: Les reçus standard de PDV sont générés après des paiements réussis

## Compatibilité Matérielle {#hardware-compatibility}

:::important Exigences de Connectivité
Cette mise en œuvre utilise le SDK JavaScript de SumUp, ce qui signifie qu'elle fonctionne via des applications web mais nécessite **uniquement des terminaux connectés à Internet**. Les terminaux Bluetooth ne sont pas actuellement supportés.
:::

### Terminaux Supportés (Connectés à Internet) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Terminal connecté par WiFi/Ethernet

### Terminaux Non Supportés (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Bluetooth uniquement
- **SumUp 3G** ❌ - Bluetooth uniquement

:::note Support Futur
Le support des terminaux Bluetooth sera ajouté aux applications iOS et Android au **Q1 2026**. Cela permettra le support de tous les terminaux certifiés SumUp, y compris les modèles Air et 3G.
:::

## Dépannage {#troubleshooting}

### Problèmes Courants {#common-issues}

<Accordion>
  <AccordionItem question="Le lecteur ne s'apaire pas">

  - Assurez-vous d'utiliser un terminal connecté à Internet compatible (Solo uniquement)
  - Vérifiez que votre appareil SumUp Solo est connecté au WiFi et en ligne
  - Assurez-vous que l'appareil est en mode d'appariement et que le code d'appariement est correctement saisi
  - Confirmez que votre compte SumUp a l'accès API activé
  - Assurez-vous d'une connexion Internet stable lors de l'appariement

  </AccordionItem>

  <AccordionItem question="Paiement non traité">

  - Vérifiez que le lecteur est correctement apparié
  - Vérifiez que votre compte SumUp est actif et en règle
  - Assurez-vous que l'appareil a suffisamment de batterie et de connectivité
  - Confirmez que la clé API est correcte et active

  </AccordionItem>

  <AccordionItem question="Commande non complétée">

  - Vérifiez la stabilité de la connexion Internet
  - Vérifiez les paramètres de webhook dans votre compte SumUp
  - Assurez-vous que le paiement a été réussi sur le terminal SumUp
  - Vérifiez les journaux d'erreurs WordPress pour les problèmes de communication API

  </AccordionItem>
</Accordion>

### Limites de Taux API {#api-rate-limits}

La passerelle est optimisée pour réduire les appels API et éviter les limites de taux :
- Le statut des paiements est vérifié de manière efficace
- Les demandes API inutiles sont minimisées
- La logique de nouvelle tentative automatique gère les échecs temporaires
