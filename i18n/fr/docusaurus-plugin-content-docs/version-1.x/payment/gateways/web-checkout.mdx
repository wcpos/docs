# Passerelle de paiement web {#web-checkout-gateway}

La passerelle de paiement web permet aux clients de finaliser leurs transactions POS avec le système de paiement de votre boutique en ligne. Cette passerelle crée un lien fluide entre votre POS en magasin et les moyens de paiement en ligne, ce qui est idéal lorsque les clients préfèrent payer avec leurs propres appareils ou lorsque vous avez besoin d’accéder à des moyens de paiement disponibles uniquement sur le web.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Télécharger et installer">

1. Consultez la [page des versions de la passerelle de commande web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Téléchargez le dernier fichier **woocommerce-pos-web-checkout-gateway.zip**
3. Dans l’administration WordPress, accédez à `Extensions > Ajouter > Téléverser une extension`
4. Téléversez le fichier zip et activez l’extension

  </StepCard>
  <StepCard number="2" title="Activer dans WCPOS" isLast>

1. Accédez à `WP Admin > POS > Réglages > Validation de la commande`
2. Recherchez la **passerelle de commande web WCPOS** dans la liste
3. Activez la passerelle pour l’utiliser dans le POS
4. Enregistrez les réglages

:::note
Cette passerelle utilise vos moyens de paiement WooCommerce existants. Veillez donc à disposer d’au moins une passerelle de paiement web correctement configurée (Stripe, PayPal, etc.).
:::

  </StepCard>
</Steps>

## Utilisation {#usage}

### Lancer le paiement web {#initiating-web-checkout}

1. **Ajouter des articles** : ajoutez des produits à votre panier dans le POS
2. **Sélectionner la passerelle** : choisissez « Paiement web » comme moyen de paiement
3. **Générer le lien** : le système crée un lien de paiement unique pour la commande
4. **Accès client** : le client clique sur le lien pour accéder au paiement web
5. **Paiement en ligne** : le client finalise le paiement à l’aide des moyens de paiement de la boutique en ligne

### Processus de paiement {#payment-process}

La passerelle Paiement web suit ce flux de travail :

1. **Création de la commande** : POS crée une commande en attente de paiement dans WooCommerce
2. **Lien de paiement**: le système génère un lien de paiement sécurisé et unique
3. **Redirection du client**: le client est dirigé vers la page de paiement web
4. **Sélection du paiement**: le client choisit parmi les moyens de paiement web disponibles
5. **Traitement du paiement**: le processus de paiement standard de WooCommerce prend en charge le paiement
6. **Finalisation de la commande**: le statut de la commande est mis à jour automatiquement une fois le paiement réussi

### Flux de travail POS {#pos-workflow}

Après le lancement du paiement web:

1. **Surveiller le statut** : gardez l’écran de commande du POS ouvert pour surveiller le statut du paiement
2. **Paiement du client**: le client effectue le paiement sur son appareil ou sur votre tablette
3. **Mise à jour automatique**: le statut de la commande est mis à jour en temps réel lorsque le paiement est reçu
4. **Traiter le paiement** : cliquez sur le bouton "Traiter le paiement" dans le POS pour passer au reçu
5. **Génération du reçu** : générer et imprimer le reçu comme d’habitude

## Cas d’utilisation {#use-cases}

### Idéal pour {#perfect-for}

- **Préférence du client** : clients qui préfèrent utiliser leurs propres applications ou cartes de paiement
- **Paiements complexes** : transactions nécessitant des moyens de paiement non disponibles dans le POS
- **Paiements fractionnés** : clients souhaitant utiliser plusieurs moyens de paiement
- **Programmes de fidélité** : accès à l’utilisation de points de fidélité sur le Web
- **Cartes-cadeaux** : clients disposant de cartes-cadeaux numériques ou d’un avoir en boutique
- **Cartes internationales** : moyens de paiement qui fonctionnent mieux avec les passerelles Web

## Détails techniques {#technical-details}

### Gestion des commandes {#order-management}

- **Commandes en attente de paiement** : les commandes sont créées avec l’état "En attente de paiement"
- **Mises à jour de l’état** : l’état change automatiquement lorsque le paiement est reçu
- **Notes de commande** : les détails du paiement sont enregistrés dans les notes de commande
- **Inventaire** : le stock est réservé pendant le processus de validation de commande

### Sécurité {#security}

- **Liens uniques** : chaque session de validation de commande dispose d’une URL unique et limitée dans le temps
- **SSL requis** : tout le traitement des paiements s’effectue via des connexions HTTPS sécurisées
- **Conformité PCI** : utilise vos passerelles de paiement web existantes conformes PCI
- **Gestion des sessions** : la gestion sécurisée des sessions empêche les accès non autorisés

### Moyens de paiement {#payment-methods}

La passerelle donne accès à tous vos moyens de paiement WooCommerce configurés :

- **Cartes de crédit/débit** : Stripe, Square, Authorize.net, etc.
- **Portefeuilles numériques** : PayPal, Apple Pay, Google Pay
- **Virements bancaires** : moyens de paiement par virement bancaire direct
- **Acheter maintenant, payer plus tard** : Klarna, Afterpay, Sezzle
- **Cryptomonnaie** : Bitcoin et autres passerelles de paiement en cryptomonnaie
- **Moyens locaux** : moyens de paiement propres à certaines régions

## Dépannage {#troubleshooting}

### Problèmes courants {#common-issues}

<Accordion>
  <AccordionItem question="Le lien de paiement ne fonctionne pas">

  - Vérifier que le certificat SSL est correctement installé
  - Vérifier que les permaliens WooCommerce sont correctement configurés
  - S’assurer que les passerelles de paiement web sont actives et configurées
  - Tester avec un autre navigateur ou appareil

  </AccordionItem>

  <AccordionItem question="Le paiement n’est pas traité">

  - Confirmer que les passerelles de paiement web sont correctement configurées
  - Vérifier les journaux des passerelles de paiement pour rechercher des messages d’erreur
  - Vérifier que le moyen de paiement du client est pris en charge
  - Tester le processus de validation de commande web indépendamment

  </AccordionItem>

  <AccordionItem question="La commande ne se met pas à jour dans le POS">

  - Vérifier la stabilité de la connexion Internet
  - Vérifier que les tâches cron de WordPress s’exécutent correctement
  - Actualiser l’écran des commandes POS
  - Vérifier l’état de la commande dans l’administration WooCommerce

  </AccordionItem>

  <AccordionItem question="Le client ne peut pas accéder à la validation de commande">

  - S’assurer que le lien de validation de commande n’a pas expiré
  - Vérifier que l’appareil du client dispose d’une connexion Internet
  - Vérifier que la commande n’a pas été annulée ou terminée
  - Tester le lien sur un autre appareil

  </AccordionItem>
</Accordion>

### Optimisation des performances {#performance-optimisation}

Pour des performances optimales :

- **Mise en cache** : configurez les extensions de cache pour exclure les pages de paiement
- **CDN** : assurez-vous que les réglages du CDN n’interfèrent pas avec le processus de paiement
- **Base de données** : optimisez la base de données pour accélérer le traitement des commandes
- **Hébergement** : utilisez un hébergement fiable avec une bonne disponibilité
