# Passerelle Stripe Terminal {#stripe-terminal-gateway}

La passerelle Stripe Terminal permet d'accepter les paiements en personne à l'aide de lecteurs Stripe Terminal directement dans WCPOS. Cette passerelle prend en charge les lecteurs de cartes physiques ainsi que le mode simulateur pour les tests.

## Fonctionnalités {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Intégration matérielle">
    Connecter des lecteurs Stripe Terminal physiques via une connexion Internet
  </FeatureCard>
  <FeatureCard icon="desktop" title="Mode simulateur">
    Tester les paiements sans matériel à l'aide du simulateur Stripe
  </FeatureCard>
  <FeatureCard icon="bolt" title="Traitement en temps réel">
    Traitement et confirmation instantanés des paiements
  </FeatureCard>
  <FeatureCard icon="shield" title="Transactions sécurisées">
    Traitement des paiements conforme à la norme PCI via Stripe
  </FeatureCard>
  <FeatureCard icon="phone" title="Commandes par téléphone (MOTO)">
    Acceptez les paiements par carte par téléphone en saisissant les informations sur le lecteur
  </FeatureCard>
</FeatureGrid>

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Installer Stripe Terminal pour WooCommerce">

Installez depuis `WP Admin > POS > Réglages > Extensions`, ou téléchargez le dernier **fichier zip de l'extension** (et non le zip ou le tarball du code source GitHub) depuis la [page des versions GitHub](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases) et téléversez-le via `Extensions > Ajouter > Téléverser une extension`.

  </StepCard>
  <StepCard number="2" title="Configurer les réglages Stripe">

1. Accédez à `WP Admin > WooCommerce > Réglages > Paiements`
2. Recherchez **Stripe Terminal** dans la liste des moyens de paiement
3. Cliquez sur **Stripe Terminal** pour accéder aux réglages
4. Saisissez votre **clé secrète Stripe** (disponible depuis votre tableau de bord Stripe)
5. Enregistrez les réglages

:::note
Il n'est pas nécessaire d'activer la passerelle Stripe Terminal dans les réglages de WooCommerce. Elle sera activée spécifiquement pour le PDV à l'étape suivante.
:::

  </StepCard>
  <StepCard number="3" title="Activer dans WCPOS" isLast>

1. Accédez à `WP Admin > POS > Réglages > Validation de la commande`
2. Recherchez la passerelle **Stripe Terminal** dans la liste
3. Activez la passerelle pour l'utiliser dans le PDV
4. Enregistrez vos réglages

  </StepCard>
</Steps>

## Utilisation {#usage}

### Connexion d'un lecteur {#connecting-a-reader}

Lorsque vous sélectionnez la passerelle Stripe Terminal lors du paiement dans le PDV :

1. **Choisir la méthode de connexion** : vous pouvez connecter un lecteur physique ou utiliser le simulateur
2. **Lecteur physique** : Suivez les instructions à l'écran pour connecter votre appareil Stripe Terminal
3. **Simulateur** : Sélectionnez le mode simulateur pour tester différents scénarios de paiement sans matériel

### Traitement des paiements {#processing-payments}

1. **Ajouter des articles** : Ajoutez des produits à votre panier dans le PDV
2. **Sélectionner la passerelle** : Choisissez « Stripe Terminal » comme méthode de paiement
3. **Connecter le lecteur** : Connectez votre lecteur ou choisissez le mode simulateur
4. **Traiter le paiement** : Suivez les instructions pour finaliser la transaction
5. **Confirmation** : La commande sera automatiquement finalisée après un paiement réussi

### Test avec le simulateur {#testing-with-simulator}

Le simulateur vous permet de tester différentes méthodes de paiement et différents scénarios :

- **Paiements par carte** : Testez différents types de cartes (Visa, Mastercard, etc.)
- **Paiements sans contact** : Simulez des transactions par paiement sans contact
- **Scénarios d'erreur** : Testez les paiements refusés et autres conditions d'erreur
- **Montants différents** : Testez différents montants de transaction

### Commandes téléphoniques (MOTO) {#phone-orders-moto}

MOTO (Mail Order/Telephone Order) permet de traiter les paiements par carte pour les clients qui ne sont pas physiquement présents — par exemple, lors d'une prise de commande par téléphone. Au lieu de présenter ou d'insérer une carte, le commerçant saisit les informations de la carte directement sur l'écran du lecteur de terminal.

#### Configuration {#setup}

<Steps>
  <StepCard number="1" title="Demander l'accès MOTO auprès de Stripe">

MOTO n'est pas activé par défaut. Contactez le [support Stripe](https://support.stripe.com/contact) et demandez-leur d'activer les autorisations MOTO pour votre compte. Il s'agit d'un processus rapide mais nécessitant une approbation manuelle de la part de Stripe.

  </StepCard>
  <StepCard number="2" title="Activer dans les réglages de l'extension">

1. Accédez à `WP Admin > WooCommerce > Réglages > Paiements > Stripe Terminal`
2. Cochez la case **Commandes téléphoniques (MOTO)**
3. Enregistrez les réglages

  </StepCard>
  <StepCard number="3" title="Connecter un lecteur compatible" isLast>

MOTO fonctionne uniquement avec les lecteurs compatibles connectés à Internet répertoriés dans [Terminaux pris en charge](#supported-terminals-internet-connected). Le bouton bascule n'apparaîtra pas pour les autres types de lecteurs.

  </StepCard>
</Steps>

#### Prise de commande par téléphone {#taking-a-phone-order}

1. Connectez un lecteur compatible (voir [Terminaux pris en charge](#supported-terminals-internet-connected))
2. Sur l'écran de paiement, activez le bouton **Commande par téléphone**
3. Cliquez sur **Encaisser le paiement par carte** — le lecteur affichera un écran de saisie du numéro de carte au lieu de demander un paiement sans contact ou par insertion
4. Saisissez le numéro de carte, la date d'expiration et le CVV du client sur le lecteur
5. Le paiement se poursuit ensuite normalement

:::tip
Les paiements MOTO utilisent `card` comme type de méthode de paiement plutôt que `card_present`. Cela signifie qu'ils sont traités comme des transactions en ligne du point de vue de Stripe, et les frais standard de traitement de carte en ligne s'appliquent plutôt que les tarifs en personne.
:::

:::caution
Le bouton Commande par téléphone n'apparaît que lorsque les trois conditions suivantes sont réunies : le paramètre MOTO est activé dans les réglages de l'extension, un lecteur compatible est connecté et le lecteur n'est pas un simulateur. Si le bouton n'apparaît pas, vérifiez ces conditions.
:::

## Compatibilité matérielle {#hardware-compatibility}

:::important Exigences de connectivité
Cette implémentation utilise le SDK JavaScript de Stripe, ce qui signifie qu'elle fonctionne via des applications web mais nécessite **des terminaux connectés à Internet uniquement**. Les terminaux Bluetooth ne sont pas actuellement pris en charge.
:::

### Terminaux pris en charge (connectés à Internet) {#supported-terminals-internet-connected}

- **Stripe Reader S700/S710** ✅ - Terminal connecté par Ethernet/WiFi
- **WisePOS E** ✅ - Terminal connecté par WiFi

### Terminaux non pris en charge (Bluetooth) {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - Bluetooth uniquement
- **BBPOS WisePad 3** ❌ - Bluetooth uniquement  
- **Verifone P400** ❌ - Bluetooth uniquement

:::note Prise en charge future
La prise en charge des terminaux Bluetooth est prévue pour une future version de l'application iOS et Android. Une fois disponible, cela permettra la prise en charge de tous les lecteurs certifiés Stripe Terminal, y compris le M2 et le WisePad 3.
:::

### Problèmes courants {#common-issues}

<Accordion>
  <AccordionItem question="Le lecteur ne se connecte pas">

  - Vérifiez que vous utilisez un [terminal connecté à Internet pris en charge](#supported-terminals-internet-connected)
  - Vérifiez que le terminal est connecté au WiFi/Ethernet et en ligne
  - Vérifiez que votre compte Stripe a Terminal activé
  - Confirmez que le terminal est enregistré dans votre tableau de bord Stripe

  </AccordionItem>

  <AccordionItem question="Paiement refusé">

  - Vérifiez que votre compte Stripe est actif et en règle
  - Vérifiez que la carte utilisée est valide
  - Vérifiez que des fonds suffisants sont disponibles

  </AccordionItem>

  <AccordionItem question="Le bouton Commande téléphonique ne s'affiche pas">

  - Vérifiez que le paramètre **Commandes téléphoniques (MOTO)** est activé dans `WooCommerce > Réglages > Paiements > Stripe Terminal`
  - Assurez-vous d'être connecté à un lecteur compatible (voir [Terminaux compatibles](#supported-terminals-internet-connected)) — le bouton est masqué pour les autres types de lecteurs
  - Le bouton n'apparaît pas lors de l'utilisation du simulateur

  </AccordionItem>

  <AccordionItem question="Le paiement MOTO échoue avec une erreur">

  - Vérifiez que Stripe a activé les autorisations MOTO sur votre compte — contactez le [support Stripe](https://support.stripe.com/contact) si ce n'est pas encore fait
  - Vérifiez que les informations de la carte ont été saisies correctement sur le lecteur
  - Les paiements MOTO peuvent être soumis à des contrôles anti-fraude plus stricts — assurez-vous que la carte est valide et dispose de fonds suffisants

  </AccordionItem>

  <AccordionItem question="Erreurs de certificat SSL">

  - Stripe Terminal nécessite HTTPS — assurez-vous que votre site dispose d'un certificat SSL valide
  - Vérifiez que votre certificat SSL est correctement configuré

  </AccordionItem>
</Accordion>
