# Passerelle de Paiement Web {#web-checkout-gateway}

La passerelle de paiement Web permet aux clients de finaliser leurs transactions POS en utilisant le système de paiement de votre magasin en ligne. Cette passerelle crée un pont sans couture entre votre POS en magasin et les méthodes de paiement en ligne, parfait pour les situations où les clients préfèrent payer en utilisant leurs propres appareils ou lorsque vous avez besoin d'accéder à des méthodes de paiement uniquement sur le web.

## Installation

<Steps>
  <StepCard number="1" title="Télécharger et Installer">

1. Visitez la [page des versions de la Passerelle de Paiement Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Téléchargez le fichier **woocommerce-pos-web-checkout-gateway.zip** le plus récent
3. Dans votre admin WordPress, allez à `Plugins > Ajouter Nouveau > Téléverser un Plugin`
4. Téléversez le fichier zip et activez le plugin

  </StepCard>
  <StepCard number="2" title="Activer dans WooCommerce POS" isLast>

1. Allez à `WP Admin > WooCommerce POS > Paramètres > Paiement`
2. Trouvez la **Passerelle de Paiement Web WooCommerce POS** dans la liste
3. Activez la passerelle pour utilisation dans le POS
4. Enregistrez vos paramètres

:::note
Cette passerelle utilise vos méthodes de paiement WooCommerce existantes, alors assurez-vous d'avoir au moins une passerelle de paiement en ligne correctement configurée (Stripe, PayPal, etc.).
:::

  </StepCard>
</Steps>

## Utilisation {#usage}

### Initiation du Paiement Web {#initiating-web-checkout}

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Paiement Web" comme méthode de paiement
3. **Générer un Lien** : Le système crée un lien de paiement unique pour la commande
4. **Accès Client** : Le client clique sur le lien pour accéder au paiement web
5. **Paiement en Ligne** : Le client finalise le paiement en utilisant les méthodes de paiement du magasin en ligne

### Processus de Paiement {#payment-process}

La passerelle de paiement Web suit ce flux de travail :

1. **Création de la Commande** : Le POS crée une commande en attente dans WooCommerce
2. **Lien de Paiement** : Le système génère un lien de paiement sécurisé et unique
3. **Redirection du Client** : Le client est dirigé vers la page de paiement web
4. **Sélection de Paiement** : Le client choisit parmi les méthodes de paiement en ligne disponibles
5. **Traitement du Paiement** : Le processus de paiement standard de WooCommerce gère le paiement
6. **Finalisation de la Commande** : L'état de la commande se met à jour automatiquement après un paiement réussi

### Flux de Travail POS {#pos-workflow}

Après avoir initié le paiement web :

1. **Surveiller l'État** : Gardez l'écran de commande POS ouvert pour surveiller l'état du paiement
2. **Paiement Client** : Le client finalise le paiement sur son appareil ou votre tablette
3. **Mise à Jour Automatique** : L'état de la commande se met à jour en temps réel lorsque le paiement est reçu
4. **Traiter le Paiement** : Cliquez sur le bouton "Traiter le Paiement" dans le POS pour continuer vers le reçu
5. **Génération de Reçu** : Générez et imprimez le reçu comme d'habitude

### Idéal Pour {#perfect-for}

- **Préférence du Client** : Clients qui préfèrent utiliser leurs propres applications ou cartes de paiement

- **Cartes-Cadeaux** : Clients avec des cartes-cadeaux numériques ou un crédit en magasin
- **Cartes Internationales** : Méthodes de paiement qui fonctionnent mieux via des passerelles web

## Détails Techniques {#technical-details}

### Gestion des Commandes {#order-management}

- **Commandes en Attente** : Les commandes sont créées avec le statut "paiement en attente"
- **Mises à Jour d'État** : Changements d'état automatiques lorsque le paiement est reçu
- **Notes de Commande** : Les détails de paiement sont enregistrés dans les notes de commande
- **Inventaire** : Le stock est réservé durant le processus de paiement

### Sécurité {#security}

- **Liens Uniques** : Chaque session de paiement a une URL unique et limitée dans le temps
- **SSL Requis** : Tous les traitements de paiement se font sur des connexions HTTPS sécurisées
- **Conformité PCI** : Utilise vos passerelles de paiement web conformes PCI existantes
- **Gestion des Sessions** : La gestion sécurisée des sessions empêche l'accès non autorisé

### Méthodes de Paiement {#payment-methods}

La passerelle fournit l'accès à toutes vos méthodes de paiement WooCommerce configurées :

- **Cartes de Crédit/Débit** : Stripe, Square, Authorize.net, etc.
- **Portefeuilles Numériques** : PayPal, Apple Pay, Google Pay
- **Transferts Bancaires** : Méthodes de paiement directes bancaires
- **Acheter Maintenant, Payer Plus Tard** : Klarna, Afterpay, Sezzle
- **Cryptomonnaie** : Bitcoin et autres passerelles de paiement crypto
- **Méthodes Locales** : Méthodes de paiement spécifiques à la région

## Dépannage {#troubleshooting}

### Problèmes Courants {#common-issues}

<Accordion>
  <AccordionItem question="Le lien de paiement ne fonctionne pas">

  - Vérifiez que le certificat SSL est correctement installé
  - Assurez-vous que les permaliens WooCommerce sont configurés correctement
  - Assurez-vous que les passerelles de paiement web sont actives et configurées
  - Testez avec un autre navigateur ou dispositif

  </AccordionItem>

  <AccordionItem question="Le paiement ne se traite pas">

  - Confirmez que les passerelles de paiement web sont correctement configurées
  - Vérifiez les journaux de la passerelle de paiement pour des messages d'erreur
  - Vérifiez que la méthode de paiement du client est supportée
  - Testez le processus de paiement web indépendamment

  </AccordionItem>

  <AccordionItem question="La commande ne se met pas à jour dans le POS">

  - Vérifiez la stabilité de la connexion internet
  - Assurez-vous que les tâches cron WordPress s'exécutent correctement
  - Rafraîchissez l'écran de commande du POS
  - Vérifiez l'état de la commande dans l'administration WooCommerce

  </AccordionItem>

  <AccordionItem question="Le client ne peut pas accéder au paiement">

  - Assurez-vous que le lien de paiement n'a pas expiré
  - Vérifiez que le dispositif du client dispose d'une connexion internet
  - Vérifiez que la commande n'a pas été annulée ou complétée
  - Testez le lien sur un autre dispositif

  </AccordionItem>
</Accordion>

### Optimisation des Performances {#performance-optimisation}

Pour des performances optimales :

- **Mise en Cache** : Configurez les plugins de mise en cache pour exclure les pages de paiement
- **CDN** : Assurez-vous que les paramètres CDN n'interfèrent pas avec le processus de paiement
- **Base de Données** : Optimisez la base de données pour un traitement des commandes plus rapide
- **Hébergement** : Utilisez un hébergement fiable avec un bon temps de disponibilité
