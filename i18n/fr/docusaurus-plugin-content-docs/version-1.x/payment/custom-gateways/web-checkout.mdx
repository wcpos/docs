# Passerelle de Paiement Web

La passerelle de paiement Web permet aux clients de finaliser leurs transactions POS en utilisant le système de paiement de votre boutique en ligne. Cette passerelle crée un pont fluide entre votre POS en magasin et les méthodes de paiement en ligne, parfait pour les situations où les clients préfèrent payer en utilisant leurs propres appareils ou lorsque vous avez besoin d'accéder à des méthodes de paiement uniquement en ligne.

## Caractéristiques

- **Intégration Transparente** : Lien direct du POS au paiement en ligne
- **Tous les Méthodes de Paiement** : Accès à toutes les passerelles de paiement activées de votre boutique en ligne
- **Contrôle Client** : Les clients peuvent payer en utilisant leurs propres appareils et méthodes préférées
- **Synchronisation des Commandes** : Mises à jour automatiques des commandes entre le POS et la boutique en ligne
- **Flux de Travail Flexible** : Finalisez les transactions en ligne tout en maintenant le flux de travail POS

## Installation

### Étape 1 : Télécharger et Installer

1. Visitez la [page des versions de la Passerelle de Paiement Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Téléchargez le dernier fichier **woocommerce-pos-web-checkout-gateway.zip**
3. Dans votre admin WordPress, allez à `Extensions > Ajouter > Téléverser une extension`
4. Téléversez le fichier zip et activez l'extension

### Étape 2 : Activer dans WooCommerce POS

1. Allez à `WP Admin > WooCommerce POS > Paramètres > Paiement`
2. Trouvez la **Passerelle de Paiement Web WooCommerce POS** dans la liste
3. Activez la passerelle pour une utilisation dans le POS
4. Enregistrez vos paramètres

:::note
Cette passerelle utilise vos méthodes de paiement WooCommerce existantes, assurez-vous d'avoir au moins une passerelle de paiement en ligne configurée correctement (Stripe, PayPal, etc.).
:::

## Utilisation

### Initiation du Paiement Web

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Paiement Web" comme méthode de paiement
3. **Générer le Lien** : Le système crée un lien de paiement unique pour la commande
4. **Accès Client** : Le client clique sur le lien pour accéder au paiement en ligne
5. **Paiement en Ligne** : Le client finalise le paiement en utilisant les méthodes de paiement de la boutique en ligne

### Processus de Paiement

La passerelle de paiement Web suit ce flux de travail :

1. **Création de Commande** : Le POS crée une commande en attente dans WooCommerce
2. **Lien de Paiement** : Le système génère un lien de paiement sécurisé et unique
3. **Redirection du Client** : Le client est dirigé vers la page de paiement en ligne
4. **Sélection de Paiement** : Le client choisit parmi les méthodes de paiement en ligne disponibles
5. **Traitement du Paiement** : Le processus de paiement standard WooCommerce gère le paiement
6. **Finalisation de la Commande** : L'état de la commande se met à jour automatiquement après un paiement réussi

### Flux de Travail POS

Après avoir initié le paiement en ligne :

1. **Surveiller le Statut** : Gardez l'écran de la commande POS ouvert pour surveiller l'état du paiement
2. **Paiement Client** : Le client termine le paiement sur son appareil ou votre tablette
3. **Mise à Jour Automatique** : L'état de la commande se met à jour en temps réel lorsque le paiement est reçu
4. **Traiter le Paiement** : Cliquez sur le bouton "Traiter le Paiement" dans le POS pour continuer vers le reçu
5. **Génération de Reçu** : Générez et imprimez le reçu comme d'habitude

## Cas d'Utilisation

### Parfait Pour

- **Préférence Client** : Clients préférant utiliser leurs propres applications ou cartes de paiement
- **Paiements Complexes** : Transactions nécessitant des méthodes de paiement non disponibles dans le POS
- **Paiements Fractionnés** : Clients souhaitant utiliser plusieurs méthodes de paiement
- **Programmes de Fidélité** : Accès à la restitution de points de fidélité en ligne
- **Cartes-Cadeaux** : Clients avec des cartes-cadeaux numériques ou des crédits en magasin
- **Cartes Internationales** : Méthodes de paiement qui fonctionnent mieux via des passerelles en ligne

### Exemples de Flux de Travail

**Magasin de Détail**
1. Le client fait ses achats en magasin et apporte des articles à la caisse
2. Le client préfère payer avec son application de portefeuille mobile
3. Le personnel sélectionne la passerelle de paiement Web
4. Le client scanne le code QR ou clique sur le lien depuis son téléphone
5. Le client finalize le paiement en utilisant sa méthode préférée
6. Le personnel traite le reçu et finalise la transaction

**Entreprise de Services**
1. Finalisez le service et ajoutez les frais au POS
2. Le client souhaite payer avec une carte de crédit professionnelle
3. Générez un lien de paiement Web
4. Le client saisit les détails de paiement sur un formulaire web sécurisé
5. Le paiement est traité via votre passerelle web
6. La commande se termine et le reçu est généré

## Détails Techniques

### Gestion des Commandes

- **Commandes en Attente** : Les commandes sont créées avec le statut "paiement en attente"
- **Mises à Jour de Statut** : Changements de statut automatiques lorsque le paiement est reçu
- **Notes de Commande** : Détails de paiement enregistrés dans les notes de commande
- **Inventaire** : Le stock est réservé pendant le processus de paiement

### Sécurité

- **Liens Uniques** : Chaque session de paiement a une URL unique et à durée limitée
- **SSL Requis** : Tous les traitements de paiement se font via des connexions HTTPS sécurisées
- **Conformité PCI** : Utilise vos passerelles de paiement en ligne conformes PCI existantes
- **Gestion de Session** : Gestion sécurisée des sessions pour empêcher l'accès non autorisé

### Méthodes de Paiement

La passerelle donne accès à toutes vos méthodes de paiement WooCommerce configurées :

- **Cartes de Crédit/Débit** : Stripe, Square, Authorize.net, etc.
- **Portefeuilles Numériques** : PayPal, Apple Pay, Google Pay
- **Virements Bancaires** : Méthodes de paiement directes
- **Acheter Maintenant, Payer Plus Tard** : Klarna, Afterpay, Sezzle
- **Cryptomonnaie** : Bitcoin et autres passerelles de paiement cryptographiques
- **Méthodes Locales** : Méthodes de paiement spécifiques à la région

## Exigences

- **WooCommerce POS** : Version gratuite ou Pro installée et activée
- **Passerelles de Paiement Web** : Au moins une méthode de paiement en ligne configurée dans WooCommerce
- **Certificat SSL** : HTTPS requis pour un traitement de paiement sécurisé
- **Navigateur Moderne** : L'appareil du client doit supporter les normes web modernes

## Dépannage

### Problèmes Courants

**Lien de Paiement Ne Fonctionne Pas**
- Vérifiez que le certificat SSL est correctement installé
- Vérifiez que les permaliens WooCommerce sont configurés correctement
- Assurez-vous que les passerelles de paiement en ligne sont actives et configurées
- Testez avec un autre navigateur ou appareil

**Paiement Ne Se Traite Pas**
- Confirmez que les passerelles de paiement en ligne sont correctement configurées
- Vérifiez les journaux de passerelle de paiement pour les messages d'erreur
- Vérifiez que la méthode de paiement du client est supportée
- Testez le processus de paiement en ligne indépendamment

**Commande Ne Se Met Pas à Jour dans le POS**
- Vérifiez la stabilité de la connexion Internet
- Vérifiez que les tâches cron de WordPress fonctionnent correctement
- Rafraichissez l'écran de commande POS
- Vérifiez l'état de la commande dans l'admin WooCommerce

**Le Client Ne Peut Pas Accéder au Paiement**
- Assurez-vous que le lien de paiement n'a pas expiré
- Vérifiez que l'appareil du client a une connectivité Internet
- Vérifiez que la commande n'a pas été annulée ou complétée
- Testez le lien sur un autre appareil

### Optimisation des Performances

Pour des performances optimales :

- **Mise en Cache** : Configurez les plugins de mise en cache pour exclure les pages de paiement
- **CDN** : Assurez-vous que les paramètres CDN n'interfèrent pas avec le processus de paiement
- **Base de Données** : Optimisez la base de données pour un traitement des commandes plus rapide
- **Hébergement** : Utilisez un hébergement fiable avec une bonne disponibilité
