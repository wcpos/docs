# Passerelle de Facture par Email

La passerelle de Facture par Email vous permet d'envoyer des factures de paiement aux clients par email directement depuis WooCommerce POS. Cela est parfait pour les situations où les clients préfèrent payer plus tard ou ont besoin de compléter le paiement en ligne en utilisant les méthodes de paiement de votre boutique en ligne.

## Caractéristiques

- **Intégration Email** : Envoyez des factures directement depuis l'interface POS
- **Gestion des Clients** : Remplissage automatique de l'email à partir des données client existantes ou ajout de nouvelles adresses email
- **Commandes Invitées** : Prise en charge des clients invités avec mises à jour d'adresse de facturation facultatives
- **Paiement En Ligne** : Les clients reçoivent un lien "Payer pour cette commande" pour compléter le paiement en ligne
- **Flux de Travail Flexible** : Complétez les commandes en magasin ou permettez la finalisation du paiement à distance

## Installation

### Étape 1 : Télécharger et Installer

1. Visitez la [page des versions de la passerelle de facture par email](https://github.com/wcpos/email-invoice-gateway/releases)
2. Téléchargez le dernier fichier **woocommerce-pos-email-invoice-gateway.zip**
3. Dans votre administration WordPress, allez à `Plugins > Ajouter Nouveau > Télécharger le Plugin`
4. Téléchargez le fichier zip et activez le plugin

### Étape 2 : Activer dans WooCommerce POS

1. Allez à `WP Admin > WooCommerce POS > Réglages > Paiement`
2. Trouvez la **Passerelle de Paiement par Facture WooCommerce POS** dans la liste
3. Activez la passerelle pour une utilisation dans le POS
4. Enregistrez vos paramètres

:::note
Cette passerelle ne nécessite pas de clés API supplémentaires ou de configuration de service externe. Elle utilise votre système d'email WordPress existant et les méthodes de paiement de WooCommerce.
:::

## Utilisation

### Envoi de Factures

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Facture par Email" comme méthode de paiement
3. **Adresse Email** : 
   - Pour les clients existants : L'email est automatiquement rempli à partir des données client
   - Pour les commandes invitées : Entrez manuellement l'adresse email du client
4. **Optionnel** : Cochez "Sauvegarder l'adresse de facturation" pour mettre à jour les informations de facturation du client
5. **Envoyer la Facture** : Complétez le processus pour envoyer l'email de facture

### Expérience Client

Lorsque vous envoyez une facture, le client reçoit un email contenant :

- **Détails de la Commande** : Détail complet des articles, quantités et prix
- **Total de la Commande** : Montant final dû incluant taxes et frais
- **Lien de Paiement** : Lien direct pour compléter le paiement en ligne
- **Informations sur la Commande** : Numéro de commande et détails de référence

### Finalisation du Paiement

Les clients peuvent finaliser le paiement de deux façons :

1. **Paiement en Ligne** : Cliquez sur le lien "Payer pour cette commande" dans l'email
2. **Retour en Magasin** : Retournez pour compléter le paiement en utilisant d'autres méthodes de paiement POS

Le lien "Payer pour cette commande" redirige les clients vers une page de paiement dédiée sur votre site web où ils peuvent utiliser l'une de vos passerelles de paiement en ligne activées (Stripe, PayPal, etc.).

## Gestion des Emails

### Clients Existants

- Les adresses email sont automatiquement récupérées à partir des profils clients
- Les informations de facturation des clients sont préservées
- L'historique des commandes est maintenu sous le compte client

### Clients Invités

- Entrez manuellement les adresses email pour les clients occasionnels
- Option pour sauvegarder l'email dans l'adresse de facturation pour référence future
- Les commandes invitées sont correctement suivies dans WooCommerce

### Modèles d'Email

La passerelle utilise les modèles d'email standards de WooCommerce :

- **Personnalisable** : Modifiez l'apparence des emails via les paramètres de courrier électronique de WooCommerce
- **Personnalisé** : Incluez votre logo et branding de boutique
- **Professionnel** : Format de facture propre et professionnel
- **Compatible Mobile** : Design responsive pour tous les appareils

## Cas d'Utilisation

### Parfait Pour

- **Ventes B2B** : Clients professionnels qui ont besoin de traiter des paiements via leurs systèmes comptables
- **Grandes Commandes** : Transactions de grande valeur nécessitant approbation ou temps de traitement
- **Clients à Distance** : Clients qui doivent finaliser leur paiement après avoir quitté le magasin
- **Clients Comptes** : Clients réguliers avec des modalités de paiement établies
- **Devis à Commande** : Conversion des devis ou estimations en factures payables

### Exemples de Flux de Travail

**Magasin de Détail**
1. Le client sélectionne des articles mais doit obtenir une approbation pour l'achat
2. Envoyez un email de facture avec tous les détails
3. Le client complète le paiement en ligne quand il est prêt
4. La commande se met automatiquement à jour dans votre système

**Entreprise de Services**
1. Complétez le travail de service et ajoutez des articles au POS
2. Envoyez la facture au service comptable du client
3. Le client paie en ligne en utilisant la méthode de paiement préférée
4. Recevez la confirmation de paiement et complétez la commande

## Exigences

- **WooCommerce POS** : Version gratuite ou Pro installée et activée
- **Système d'Email** : Configuration d'email WordPress fonctionnelle
- **Passerelles de Paiement Web** : Au moins une méthode de paiement en ligne activée dans WooCommerce
- **Certificat SSL** : HTTPS requis pour le traitement sécurisé des paiements

## Dépannage

### Problèmes Courants

**Emails Ne Sont Pas Envoyés**
- Vérifiez la configuration des emails WordPress
- Vérifiez les paramètres SMTP si vous utilisez un service email personnalisé
- Testez avec des plugins de test d'email WordPress
- Vérifiez les dossiers de spam pour les emails de test

**Liens de Paiement Ne Fonctionnent Pas**
- Assurez-vous que WooCommerce est correctement configuré
- Vérifiez qu'au moins une passerelle de paiement est activée pour le paiement web
- Vérifiez que le certificat SSL est correctement installé
- Confirmez que le statut de la commande permet le paiement

**Le Client Ne Peut Pas Compléter le Paiement**
- Vérifiez que les passerelles de paiement web sont actives et configurées
- Vérifiez que la commande n'a pas expiré ou été annulée
- Assurez-vous que le client dispose de limites de méthode de paiement suffisantes
- Testez le processus de paiement vous-même

### Livraison des Emails

Pour une livraison fiable des emails, considérez :

- **Service SMTP** : Utilisez des services comme SendGrid, Mailgun ou Amazon SES
- **Plugins Email** : Installez des plugins SMTP WordPress pour une meilleure livraison
- **Authentification de Domaine** : Configurez des enregistrements SPF, DKIM et DMARC
- **Surveillance** : Suivez la livraison et les taux d'ouverture des emails
