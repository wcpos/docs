# Passerelle de Facturation par Email {#email-invoice-gateway}

La passerelle de facturation par email vous permet d'envoyer des factures de paiement aux clients par email directement depuis WooCommerce POS. Cela est parfait pour les situations où les clients préfèrent payer plus tard ou ont besoin de finaliser le paiement en ligne en utilisant les méthodes de paiement de votre boutique en ligne.

## Installation

<Steps>
  <StepCard number="1" title="Télécharger et Installer">

1. Visitez la [page des versions de la Passerelle de Facturation par Email](https://github.com/wcpos/email-invoice-gateway/releases)
2. Téléchargez le dernier fichier **woocommerce-pos-email-invoice-gateway.zip**
3. Dans votre admin WordPress, allez à `Plugins > Ajouter Nouveau > Télécharger le Plugin`
4. Téléchargez le fichier zip et activez le plugin

  </StepCard>
  <StepCard number="2" title="Activer dans WooCommerce POS" isLast>

1. Allez à `WP Admin > WooCommerce POS > Paramètres > Checkout`
2. Trouvez la **Passerelle de Paiement par Facture WooCommerce POS** dans la liste
3. Activez la passerelle pour utilisation dans le POS
4. Sauvegardez vos paramètres

:::note
Cette passerelle n'exige pas de clés API supplémentaires ni de configuration de service externe. Elle utilise votre système d'email WordPress existant et les méthodes de paiement WooCommerce.
:::

  </StepCard>
</Steps>

## Utilisation {#usage}

### Envoi de Factures {#sending-invoices}

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Facture par Email" comme méthode de paiement
3. **Adresse Email** : 
   - Pour les clients existants : L'email est automatiquement rempli à partir des données client
   - Pour les commandes invitées : Saisissez l'adresse email du client manuellement
4. **Optionnel** : Cochez "Sauvegarder à l'adresse de facturation" pour mettre à jour les informations de facturation du client
5. **Envoyer la Facture** : Complétez le processus pour envoyer l'email de facture

### Expérience Client {#customer-experience}

Lorsque vous envoyez une facture, le client reçoit un email contenant :

- **Détails de Commande** : Détail complet des articles, quantités et prix
- **Total de la Commande** : Montant final dû, y compris les taxes et frais
- **Lien de Paiement** : Lien direct pour compléter le paiement en ligne
- **Informations de Commande** : Numéro de commande et détails de référence

### Complétion de Paiement {#payment-completion}

Les clients peuvent compléter le paiement de deux manières :

1. **Paiement en Ligne** : Cliquez sur le lien "Pay for this order" dans l'email
2. **Retour en Magasin** : Revenez pour terminer le paiement en utilisant d'autres méthodes de paiement POS

Le lien "Pay for this order" redirige les clients vers une page de paiement dédiée sur votre site où ils peuvent utiliser n'importe laquelle de vos passerelles de paiement web activées (Stripe, PayPal, etc.).

## Gestion des Emails {#email-management}

### Clients Existants {#existing-customers}

- Les adresses email sont automatiquement récupérées des profils clients
- Les informations de facturation du client sont préservées
- L'historique des commandes est maintenu sous le compte client

### Clients Invités {#guest-customers}

- Entrez manuellement les adresses email pour les clients occasionnels
- Option de sauvegarder l'email à l'adresse de facturation pour référence future
- Les commandes invitées sont correctement suivies dans WooCommerce

### Modèles d'Email {#email-templates}

La passerelle utilise les modèles d'email standards de WooCommerce :

- **Personnalisable** : Modifiez l'apparence des emails via les paramètres d'email WooCommerce
- **Marqué** : Incluez le logo et le branding de votre boutique
- **Professionnel** : Format de facture propre et professionnel
- **Compatible Mobile** : Design réactif pour tous les appareils

## Cas d'Utilisation {#use-cases}

### Parfait Pour {#perfect-for}

- **Ventes B2B** : Clients professionnels qui ont besoin de traiter des paiements via leurs systèmes comptables
- **Commandes Importantes** : Transactions de grande valeur nécessitant approbation ou temps de traitement
- **Clients à Distance** : Clients qui ont besoin de compléter le paiement après avoir quitté le magasin
- **Clients de Compte** : Clients réguliers avec des conditions de paiement établies
- **Devis à Commande** : Conversion de devis ou estimations en factures payables

## Dépannage {#troubleshooting}

### Problèmes Courants {#common-issues}

<Accordion>
  <AccordionItem question="Emails non envoyés">

  - Vérifiez la configuration de l'email WordPress
  - Vérifiez les paramètres SMTP si vous utilisez un service email personnalisé
  - Testez avec des plugins de test d'email WordPress
  - Vérifiez les dossiers de spam pour les emails de test

  </AccordionItem>

  <AccordionItem question="Liens de paiement non fonctionnels">

  - Assurez-vous que WooCommerce est correctement configuré
  - Vérifiez qu'au moins une passerelle de paiement est activée pour le paiement en ligne
  - Vérifiez que le certificat SSL est correctement installé
  - Confirmez que le statut de la commande permet le paiement

  </AccordionItem>

  <AccordionItem question="Le client ne peut pas compléter le paiement">

  - Vérifiez que les passerelles de paiement web sont actives et configurées
  - Vérifiez que la commande n'a pas expiré ou été annulée
  - Assurez-vous que le client a suffisamment de limites de méthode de paiement
  - Testez le processus de paiement vous-même

  </AccordionItem>
</Accordion>

### Livraison des Emails {#email-delivery}

Pour une livraison d'email fiable, considérez :

- **Service SMTP** : Utilisez des services comme SendGrid, Mailgun ou Amazon SES
- **Plugins d'Email** : Installez des plugins SMTP WordPress pour une meilleure livraison
- **Authentification de Domaine** : Mettez en place des enregistrements SPF, DKIM et DMARC
- **Suivi** : Suivez la livraison d'email et les taux d'ouverture
