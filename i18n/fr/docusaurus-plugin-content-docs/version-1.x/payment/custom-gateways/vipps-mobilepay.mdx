# Passerelle Vipps MobilePay {#vipps-mobilepay-gateway}

La passerelle Vipps MobilePay vous permet d'accepter des paiements par téléphone directement dans WooCommerce POS. Les clients paient en scannant un QR code à l'écran ou en confirmant une notification push sur leur téléphone, sans besoin de matériel de lecteur de carte.

## Caractéristiques {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="Paiements par QR Code">
    Affichez un QR code au moment du paiement pour que les clients le scannent et paient avec leur téléphone
  </FeatureCard>
  <FeatureCard icon="bolt" title="Notifications Push">
    Envoyez une demande de paiement directement au téléphone du client en entrant son numéro
  </FeatureCard>
  <FeatureCard icon="globe" title="Couverture Nordique">
    Fonctionne avec Vipps en Norvège, MobilePay au Danemark et en Finlande
  </FeatureCard>
  <FeatureCard icon="credit-card" title="Capture Automatique">
    Capturez les fonds immédiatement après autorisation, ou réservez pour une capture manuelle
  </FeatureCard>
</FeatureGrid>

## Installation

<Steps>
  <StepCard number="1" title="Télécharger et Installer">

1. Visitez la [page des versions WCPOS Vipps MobilePay](https://github.com/wcpos/wcpos-vipps/releases)
2. Téléchargez le dernier fichier **wcpos-vipps.zip**
3. Dans votre administrateur WordPress, allez à `Plugins > Ajouter Nouveau > Télécharger Plugin`
4. Téléchargez le fichier zip et activez le plugin

  </StepCard>
  <StepCard number="2" title="Configurer les Identifiants Vipps">

1. Accédez à `WP Admin > WooCommerce > Paramètres > Paiements`
2. Trouvez **WCPOS Vipps MobilePay** dans la liste des méthodes de paiement
3. Cliquez sur **WCPOS Vipps MobilePay** pour accéder aux paramètres
4. Entrez vos identifiants provenant du [Portail Vipps](https://portal.vipps.no/):
   - **Numéro de Série du Détail (MSN)**
   - **ID Client**
   - **Clé Secrète du Client**
   - **Clé d’Abonnement**
5. Enregistrez les paramètres

:::note
Vous n'avez pas besoin d'activer la passerelle ici pour une utilisation POS — elle sera activée spécifiquement pour le POS à l'étape suivante. L'activer dans les paramètres WooCommerce la rendrait également disponible lors du passage à la caisse de votre boutique en ligne, ce qui peut être utile pour les tests.
:::

:::tip Vous utilisez déjà le plugin Vipps ?
Si vous avez installé le plugin officiel [Checkout with Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/), vos identifiants seront importés automatiquement lors de l'activation de ce plugin. Vous pouvez sauter l'entrée manuelle des identifiants.
:::

  </StepCard>
  <StepCard number="3" title="Activer dans WooCommerce POS" isLast>

1. Allez à `WP Admin > WooCommerce POS > Paramètres > Paiement`
2. Trouvez la passerelle **WCPOS Vipps MobilePay** dans la liste
3. Activez la passerelle pour une utilisation dans le POS
4. Enregistrez vos paramètres

  </StepCard>
</Steps>

## Utilisation {#usage}

### Traitement des Paiements — QR Code {#processing-payments-qr-code}

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Vipps MobilePay" comme méthode de paiement
3. **Générer le QR Code** : Cliquez sur le bouton "Générer le QR Code"
4. **Le Client Scanne** : Le client scanne le QR code avec son application Vipps ou MobilePay
5. **Le Client Confirme** : Le client confirme le paiement dans son application
6. **Achèvement Automatique** : La commande se complète automatiquement lorsque le paiement est autorisé

### Traitement des Paiements — Notification Push {#processing-payments-push-notification}

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Vipps MobilePay" comme méthode de paiement
3. **Entrer le Numéro de Téléphone** : Tapez le numéro de téléphone du client
4. **Envoyer au Téléphone** : Cliquez sur le bouton "Envoyer au Téléphone"
5. **Le Client Confirme** : Le client reçoit une notification push et confirme le paiement dans son application
6. **Achèvement Automatique** : La commande se complète automatiquement lorsque le paiement est autorisé

### Annulation d'un Paiement {#cancelling-a-payment}

En attendant que le client confirme, vous pouvez cliquer sur le bouton **Annuler le Paiement** pour annuler la transaction. Cela annule le paiement en attente du côté de Vipps et réinitialise l'interface de paiement.

### Remboursements {#refunds}

Les remboursements sont gérés par le processus standard de remboursement de WooCommerce. Ouvrez la commande, cliquez sur **Rembourser**, entrez le montant et le remboursement est traité automatiquement via l'API Vipps.

## Marchés Pris en Charge {#supported-markets}

Vipps MobilePay opère dans toute la région nordique sous deux marques :

| Région | Marque | Devises |
|--------|-------|---------|
| Norvège | Vipps | NOK |
| Danemark, Finlande | MobilePay | DKK, EUR |

Votre compte marchand détermine quels marchés et devises sont disponibles. Les clients dans n'importe quel marché pris en charge peuvent payer en utilisant leur application Vipps ou MobilePay locale.

## Exigences {#requirements}

<RequirementsList>
  <RequirementItem label="Compte Vipps">Compte marchand Vipps MobilePay actif avec identifiants API</RequirementItem>
  <RequirementItem label="Identifiants API">Numéro de Série du Détail, ID Client, Clé Secrète du Client et Clé d’Abonnement</RequirementItem>

## Dépannage {#troubleshooting}

### Problèmes Courants {#common-issues}

<Accordion>
  <AccordionItem question="Paiement ne se complétant pas">

  - Le passage à la caisse effectue des requêtes pendant jusqu'à 5 minutes — si le client ne confirme pas dans ce délai, le paiement expire
  - Vérifiez que vous ne mélangez pas les identifiants en mode test et en production
  - Assurez-vous que votre connexion Internet est stable à la fois sur l'appareil POS et sur le téléphone du client

  </AccordionItem>

  <AccordionItem question="Erreur d'identifiants invalides">

  - Vérifiez tous les quatre champs d'identifiants (MSN, ID Client, Clé Secrète du Client, Clé d’Abonnement)
  - Assurez-vous que vous utilisez des identifiants de test lorsque le mode test est activé, et des identifiants de production lorsque le mode test est désactivé
  - Vérifiez vos identifiants dans le [Portail Vipps](https://portal.vipps.no/)

  </AccordionItem>

  <AccordionItem question="QR code ne se générant pas">

  - Assurez-vous que votre site fonctionne sur HTTPS — l'API Vipps rejette les requêtes provenant de sites HTTP
  - Vérifiez les journaux WooCommerce (`WooCommerce > Statut > Journaux`) pour des détails d'erreur de l'API
  - Vérifiez que votre compte marchand est actif et que l'API ePayment est activée

  </AccordionItem>
</Accordion>
