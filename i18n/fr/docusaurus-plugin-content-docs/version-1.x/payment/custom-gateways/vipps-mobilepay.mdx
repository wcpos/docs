# Passerelle Vipps MobilePay {#vipps-mobilepay-gateway}

La passerelle Vipps MobilePay vous permet d'accepter des paiements par téléphone directement au sein de WCPOS. Les clients paient en scannant un code QR à l'écran ou en confirmant une notification push sur leur téléphone — aucun matériel de lecteur de carte n'est nécessaire.

## Fonctionnalités {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="Paiements par code QR">
    Affichez un code QR à la caisse pour que les clients puissent le scanner et payer avec leur téléphone
  </FeatureCard>
  <FeatureCard icon="bolt" title="Notifications Push">
    Envoyez une demande de paiement directement sur le téléphone du client en entrant son numéro
  </FeatureCard>
  <FeatureCard icon="globe" title="Couverture nordique">
    Fonctionne avec Vipps en Norvège, MobilePay au Danemark et en Finlande
  </FeatureCard>
  <FeatureCard icon="credit-card" title="Capture automatique">
    Capturez les fonds immédiatement après autorisation, ou réservez pour une capture manuelle
  </FeatureCard>
</FeatureGrid>

## Installation

<Steps>
  <StepCard number="1" title="Télécharger et installer">

1. Rendez-vous sur la [page des versions WCPOS Vipps MobilePay](https://github.com/wcpos/wcpos-vipps/releases)
2. Téléchargez le dernier fichier **wcpos-vipps.zip**
3. Dans votre admin WordPress, allez dans `Plugins > Ajouter Nouveau > Télécharger un Plugin`
4. Téléchargez le fichier zip et activez le plugin

  </StepCard>
  <StepCard number="2" title="Configurer les identifiants Vipps">

1. Accédez à `WP Admin > WooCommerce > Paramètres > Paiements`
2. Trouvez **WCPOS Vipps MobilePay** dans la liste des méthodes de paiement
3. Cliquez sur **WCPOS Vipps MobilePay** pour accéder aux paramètres
4. Entrez vos identifiants du [Portail Vipps](https://portal.vipps.no/) :
   - **Numéro de série commerçant (MSN)**
   - **ID Client**
   - **Secret Client**
   - **Clé d'abonnement**
5. Enregistrez les paramètres

:::note
Vous n'avez pas besoin d'activer la passerelle ici pour une utilisation POS — elle sera activée spécifiquement pour le POS à l'étape suivante. L'activer dans les paramètres WooCommerce la rendrait également disponible sur votre caisse en ligne, ce qui peut être utile pour le test.
:::

:::tip Vous utilisez déjà le plugin Vipps ?
Si vous avez installé le plugin officiel [Checkout with Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/), vos identifiants seront importés automatiquement lorsque vous activerez ce plugin. Vous pouvez ignorer l'entrée manuelle des identifiants.
:::

  </StepCard>
  <StepCard number="3" title="Activer dans WCPOS" isLast>

1. Allez dans `WP Admin > POS > Réglages > Caisse`
2. Trouvez la passerelle **WCPOS Vipps MobilePay** dans la liste
3. Activez la passerelle pour une utilisation dans le POS
4. Enregistrez vos réglages

  </StepCard>
</Steps>

## Utilisation {#usage}

### Traitement des paiements — Code QR {#processing-payments-qr-code}

1. **Ajouter des articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la passerelle** : Choisissez "Vipps MobilePay" comme méthode de paiement
3. **Générer le code QR** : Cliquez sur le bouton "Générer le code QR"
4. **Le client scanne** : Le client scanne le code QR avec son application Vipps ou MobilePay
5. **Le client confirme** : Le client confirme le paiement dans son application
6. **Complétion automatique** : La commande se complète automatiquement une fois le paiement autorisé

### Traitement des paiements — Envoyer au téléphone {#processing-payments-send-to-phone}

1. **Ajouter des articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la passerelle** : Choisissez "Vipps MobilePay" comme méthode de paiement
3. **Entrer le numéro de téléphone** : Tapez le numéro de téléphone du client
4. **Envoyer au téléphone** : Cliquez sur le bouton "Envoyer au téléphone"
5. **Le client confirme** : Le client reçoit une notification push ou est dirigé vers la page d'atterrissage Vipps, puis confirme le paiement dans son application
6. **Complétion automatique** : La commande se complète automatiquement une fois le paiement autorisé

Le plugin détecte automatiquement la meilleure méthode pour envoyer la demande de paiement au téléphone du client :

- **Push direct** (préféré) — Envoie une notification directement à l'application Vipps du client. C'est l'expérience la plus rapide mais nécessite que Vipps active `PUSH_MESSAGE` sur votre unité de vente (voir [Activation du Push Direct](#enabling-direct-push) ci-dessous).
- **Fallback sur la page d'atterrissage** — Si le push direct n'est pas activé, le plugin ouvre la page d'atterrissage Vipps dans un nouvel onglet du navigateur. La page d'atterrissage gère l'envoi de la notification. Cela fonctionne immédiatement sans approbation spéciale.

La première fois que le plugin détecte que le push direct n'est pas disponible, vous verrez un bref message vous demandant de cliquer à nouveau sur "Envoyer au téléphone". Après cela, il se souvient du résultat et fonctionne sans problème.

### Activation du Push Direct {#enabling-direct-push}

Le flux de push direct fournit la meilleure expérience pour les paiements par téléphone — pas d'onglets supplémentaires, pas de pages d'atterrissage. Pour l'activer :

1. Connectez-vous à [portal.vippsmobilepay.com](https://portal.vippsmobilepay.com)
2. Contactez votre responsable de compte clé Vipps, ou le service client
3. Dites-leur : **"J'ai besoin de PUSH_MESSAGE activé sur mon MSN pour une utilisation avec une intégration POS"**

Une fois approuvé, le plugin détecte le changement automatiquement dans les 24 heures et passe au flux de push direct. Vous verrez un avis de rappel sur la page des paramètres de la passerelle tant que vous utilisez le fallback sur la page d'atterrissage.

### Annuler un paiement {#cancelling-a-payment}

En attendant que le client confirme, vous pouvez cliquer sur le bouton **Annuler le paiement** pour interrompre la transaction. Cela annule le paiement en attente côté Vipps et réinitialise l'interface de caisse.

### Remboursements {#refunds}

Les remboursements sont gérés via le processus de remboursement standard de WooCommerce. Ouvrez la commande, cliquez sur **Rembourser**, entrez le montant, et le remboursement est traité via l'API Vipps automatiquement.

## Marchés supportés {#supported-markets}

Vipps MobilePay opère dans la région nordique sous deux marques :

| Région | Marque | Devises |
|--------|-------|------------|
| Norvège | Vipps | NOK |
| Danemark, Finlande | MobilePay | DKK, EUR |

Votre compte commerçant détermine quels marchés et devises sont disponibles. Les clients de tout marché supporté peuvent payer en utilisant leur application Vipps ou MobilePay locale.

## Dépannage {#troubleshooting}

### Problèmes courants {#common-issues}

<Accordion>
  <AccordionItem question="Le paiement ne se termine pas">

  - La caisse interroge pendant jusqu'à 5 minutes — si le client ne confirme pas dans cette fenêtre, le paiement expire
  - Vérifiez que vous ne mélangez pas les identifiants de mode test et de production
  - Vérifiez que votre connexion Internet est stable sur l'appareil POS et le téléphone du client

  </AccordionItem>

  <AccordionItem question="Erreur d'identifiants invalides">

  - Vérifiez tous les quatre champs d'identifiants (MSN, ID Client, Secret Client, Clé d'Abonnement)
  - Assurez-vous d'utiliser des identifiants de test lorsque le mode test est activé, et des identifiants de production lorsque celui-ci est désactivé
  - Vérifiez vos identifiants dans le [Portail Vipps](https://portal.vipps.no/)

  </AccordionItem>

  <AccordionItem question="'Envoyer au téléphone' ouvre un nouvel onglet au lieu d'envoyer directement">

  Cela signifie que votre compte Vipps n'a pas encore `PUSH_MESSAGE` activé. Le plugin utilise le fallback sur la page d'atterrissage, ce qui fonctionne mais ajoute une étape supplémentaire. Pour obtenir une expérience de push direct plus fluide, contactez Vipps et demandez-leur d'activer PUSH_MESSAGE sur votre unité de vente (MSN). Voir [Activation du Push Direct](#enabling-direct-push) ci-dessus.

  </AccordionItem>

  <AccordionItem question="Code QR non généré">

  - Confirmez que votre site fonctionne sous HTTPS — l'API Vipps rejette les demandes des sites HTTP
  - Vérifiez les journaux de WooCommerce (`WooCommerce > Statut > Journaux`) pour des détails d'erreurs API
  - Vérifiez que votre compte commerçant est actif et que l'API ePayment est activée

  </AccordionItem>
</Accordion>
