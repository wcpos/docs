# Passerelle Stripe Terminal {#stripe-terminal-gateway}

La passerelle Stripe Terminal vous permet d'accepter des paiements en personne en utilisant des lecteurs matériels Stripe Terminal directement dans WCPOS. Cette passerelle prend en charge à la fois les lecteurs de cartes physiques et le mode simulateur pour les tests.

## Installation

<Steps>
  <StepCard number="1" title="Télécharger et Installer">

1. Visitez la [page des versions Stripe Terminal pour WooCommerce](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases)
2. Téléchargez le fichier **stripe-terminal-for-woocommerce.zip** le plus récent
3. Dans votre admin WordPress, allez à `Plugins > Ajouter Nouveau > Télécharger Plugin`
4. Téléchargez le fichier zip et activez le plugin

  </StepCard>
  <StepCard number="2" title="Configurer les Paramètres Stripe">

1. Accédez à `WP Admin > WooCommerce > Paramètres > Paiements`
2. Trouvez **Stripe Terminal** dans la liste des méthodes de paiement
3. Cliquez sur **Stripe Terminal** pour accéder aux paramètres
4. Entrez votre **Clé Secrète Stripe** (vous pouvez l'obtenir depuis votre tableau de bord Stripe)
5. Enregistrez les paramètres

:::note
Vous n'avez pas besoin d'activer la passerelle Stripe Terminal dans les paramètres WooCommerce. Elle sera spécifiquement activée pour le POS à l'étape suivante.
:::

  </StepCard>
  <StepCard number="3" title="Activer dans WCPOS" isLast>

1. Allez à `WP Admin > POS > Paramètres > Caisse`
2. Trouvez la passerelle **Stripe Terminal** dans la liste
3. Activez la passerelle pour utilisation dans le POS
4. Enregistrez vos paramètres

  </StepCard>
</Steps>

## Utilisation {#usage}

### Connexion d'un Lecteur {#connecting-a-reader}

Lorsque vous sélectionnez la passerelle Stripe Terminal lors du paiement dans le POS :

1. **Choisir la Méthode de Connexion** : Vous pouvez soit connecter un lecteur physique, soit utiliser le simulateur
2. **Lecteur Physique** : Suivez les instructions à l'écran pour connecter votre appareil Stripe Terminal
3. **Simulateur** : Sélectionnez le mode simulateur pour tester divers scénarios de paiement sans matériel

### Traitement des Paiements {#processing-payments}

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Stripe Terminal" comme méthode de paiement
3. **Connecter le Lecteur** : Connectez votre lecteur ou choisissez le mode simulateur
4. **Traiter le Paiement** : Suivez les instructions pour compléter la transaction
5. **Confirmation** : La commande sera automatiquement finalisée lors d'un paiement réussi

### Test avec le Simulateur {#testing-with-simulator}

Le simulateur vous permet de tester diverses méthodes et scénarios de paiement :

- **Paiements par Carte** : Testez différents types de cartes (Visa, Mastercard, etc.)
- **Paiements Sans Contact** : Simulez des transactions par paiement tactile
- **Scénarios d'Erreur** : Testez les paiements refusés et d'autres conditions d'erreur
- **Montants Différents** : Testez divers montants de transaction

### Commandes Téléphoniques (MOTO) {#phone-orders-moto}

MOTO (Commande par correspondance/Commande par Téléphone) vous permet de traiter les paiements par carte pour des clients qui ne sont pas physiquement présents — par exemple, lors de la prise d'une commande par téléphone. Au lieu de taper ou d'insérer une carte, le marchand saisit les détails de la carte directement sur l'écran du lecteur terminal.

#### Configuration {#setup}

<Steps>
  <StepCard number="1" title="Demander l'accès MOTO à Stripe">

MOTO n'est pas activé par défaut. Contactez le [support Stripe](https://support.stripe.com/contact) et demandez-leur d'activer les autorisations MOTO pour votre compte. C'est un processus rapide mais nécessite une approbation manuelle de Stripe.

  </StepCard>
  <StepCard number="2" title="Activer dans les paramètres du plugin">

1. Accédez à `WP Admin > WooCommerce > Paramètres > Paiements > Stripe Terminal`
2. Cochez la case **Commandes Téléphoniques (MOTO)**
3. Enregistrez les paramètres

  </StepCard>
  <StepCard number="3" title="Connecter un lecteur compatible" isLast>

MOTO ne fonctionne qu'avec des lecteurs connectés à Internet compatibles répertoriés dans [Terminaux pris en charge](#supported-terminals-internet-connected). Le commutateur n'apparaîtra pas pour d'autres types de lecteurs.

  </StepCard>
</Steps>

#### Prendre une Commande par Téléphone {#taking-a-phone-order}

1. Connectez un lecteur compatible (voir [Terminaux pris en charge](#supported-terminals-internet-connected))
2. À l'écran de paiement, activez le commutateur **Commande Téléphonique**
3. Cliquez sur **Collecter le Paiement par Carte** — le lecteur affichera un écran de saisie du numéro de carte au lieu de demander un paiement par tap/d'insertion
4. Saisissez le numéro de carte du client, la date d'expiration et le CVV sur le lecteur
5. Le paiement se traite normalement à partir de là

:::tip
Les paiements MOTO utilisent `card` comme type de méthode de paiement plutôt que `card_present`. Cela signifie qu'ils sont traités davantage comme des transactions en ligne du point de vue de Stripe, donc les frais de traitement de carte en ligne standard s'appliquent plutôt que les tarifs en personne.
:::

:::caution
Le commutateur Commande Téléphonique n'apparaît que lorsque les trois conditions suivantes sont remplies : le paramètre MOTO est activé dans les paramètres du plugin, un lecteur compatible est connecté et le lecteur n'est pas un simulateur. Si vous ne voyez pas le commutateur, vérifiez ces conditions.
:::

## Compatibilité Matérielle {#hardware-compatibility}

:::important Exigences de Connectivité
Cette implémentation utilise le SDK JavaScript de Stripe, ce qui signifie qu'elle fonctionne via des applications Web mais nécessite **uniquement des terminaux connectés à Internet**. Les terminaux Bluetooth ne sont pas pris en charge actuellement.
:::

### Terminaux pris en charge (Connectés à Internet) {#supported-terminals-internet-connected}

- **Lecteur Stripe S700/S710** ✅ - terminal connecté par Ethernet/WiFi
- **WisePOS E** ✅ - terminal connecté par WiFi

### Terminaux Non Supportés (Bluetooth) {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - uniquement Bluetooth
- **BBPOS WisePad 3** ❌ - uniquement Bluetooth  
- **Verifone P400** ❌ - obsolète, plus pris en charge par Stripe

:::note Support Futur
La prise en charge des terminaux Bluetooth sera ajoutée aux applications iOS et Android dans **Q1 2026**. Cela permettra la prise en charge de tous les lecteurs certifiés Stripe Terminal, y compris le M2 et le WisePad 3.
:::

### Problèmes Courants {#common-issues}

<Accordion>
  <AccordionItem question="Le lecteur ne se connecte pas">

  - Assurez-vous que vous utilisez un [terminal connecté à Internet pris en charge](#supported-terminals-internet-connected)
  - Vérifiez que le terminal est connecté au WiFi/Ethernet et en ligne
  - Vérifiez que votre compte Stripe a le Terminal activé
  - Confirmez que le terminal est enregistré dans votre tableau de bord Stripe

  </AccordionItem>

  <AccordionItem question="Paiement refusé">

  - Vérifiez que votre compte Stripe est actif et en règle
  - Vérifiez que la carte utilisée est valide
  - Assurez-vous que des fonds suffisants sont disponibles

  </AccordionItem>

  <AccordionItem question="Le commutateur de commande par téléphone ne s'affiche pas">

  - Vérifiez que le paramètre **Commandes Téléphoniques (MOTO)** est activé dans `WooCommerce > Paramètres > Paiements > Stripe Terminal`
  - Assurez-vous que vous êtes connecté à un lecteur compatible (voir [Terminaux pris en charge](#supported-terminals-internet-connected)) — le commutateur est caché pour d'autres types de lecteurs
  - Le commutateur n'apparaît pas lors de l'utilisation du simulateur

  </AccordionItem>

  <AccordionItem question="Le paiement MOTO échoue avec une erreur">

  - Confirmez que Stripe a activé les autorisations MOTO sur votre compte — contactez [le support Stripe](https://support.stripe.com/contact) si ce n'est pas déjà fait
  - Vérifiez que les détails de la carte ont été saisis correctement sur le lecteur
  - Les paiements MOTO peuvent avoir des vérifications de fraude plus strictes — assurez-vous que la carte est valide et dispose de fonds suffisants

  </AccordionItem>

  <AccordionItem question="Erreurs de certificat SSL">

  - Stripe Terminal nécessite HTTPS - assurez-vous que votre site dispose d'un certificat SSL valide
  - Vérifiez que votre certificat SSL est correctement configuré

  </AccordionItem>
</Accordion>
