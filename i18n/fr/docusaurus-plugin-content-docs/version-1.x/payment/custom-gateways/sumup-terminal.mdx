# Passerelle de Terminal SumUp

La passerelle SumUp Terminal vous permet d'accepter les paiements par carte en utilisant des lecteurs de cartes SumUp directement dans WooCommerce POS. Cette passerelle offre une intégration transparente avec le système de traitement des paiements de SumUp pour les transactions en personne.

## Fonctionnalités

- **Intégration Matérielle** : Connectez des lecteurs de cartes SumUp à votre système de point de vente via une connexion internet
- **Traitement en Temps Réel** : Traitement des paiements et achèvement de la commande instantanés
- **Détection Automatique** : Les commandes se complètent automatiquement lorsque le paiement est détecté
- **Transactions Sécurisées** : Traitement des paiements conforme PCI via SumUp
- **Appairage Facile** : Processus d'appairage de l'appareil simple avec des codes d'appairage
- **SDK Web** : Utilise le SDK JavaScript de SumUp pour la compatibilité des applications web

## Installation

### Étape 1 : Télécharger et Installer

1. Visitez la [page de versions de SumUp Terminal pour WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Téléchargez le dernier fichier **sumup-terminal-for-woocommerce.zip**
3. Dans votre administration WordPress, allez à `Plugins > Ajouter > Téléverser un plugin`
4. Téléversez le fichier zip et activez le plugin

### Étape 2 : Configurer les Paramètres SumUp

1. Accédez à `Admin WP > WooCommerce > Paramètres > Paiements`
2. Trouvez **SumUp Terminal** dans la liste des méthodes de paiement
3. Cliquez sur **SumUp Terminal** pour accéder aux paramètres
4. Entrez votre **Clé API SumUp** (disponible depuis votre tableau de bord marchand SumUp)
5. Enregistrez les paramètres

:::note
Vous n'avez pas besoin d'activer la passerelle SumUp Terminal dans les paramètres WooCommerce. Elle sera activée spécifiquement pour le POS à une étape ultérieure.
:::

### Étape 3 : Appairer Votre Terminal SumUp

1. Sur la même page de paramètres, localisez la section **Appairer un Lecteur**
2. Sur votre appareil SumUp, naviguez jusqu'à l'écran d'appairage pour afficher le code d'appairage
3. Entrez le code d'appairage affiché sur votre appareil SumUp
4. Cliquez sur **"Appairer le Lecteur"** pour établir la connexion
5. Attendez la confirmation que le lecteur a été correctement appairé

:::warning Important
Le lecteur doit être correctement appairé avant de pouvoir traiter des paiements. Assurez-vous que le processus d'appairage est terminé avant de continuer.
:::

### Étape 4 : Activer dans WooCommerce POS

1. Allez à `Admin WP > WooCommerce POS > Paramètres > Caisse`
2. Trouvez la passerelle **SumUp Terminal** dans la liste
3. Activez la passerelle pour une utilisation dans le POS
4. Enregistrez vos paramètres

## Utilisation

### Traitement des Paiements

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "SumUp Terminal" comme méthode de paiement
3. **Démarrer le Paiement** : Cliquez pour lancer un nouveau paiement sur votre appareil SumUp
4. **Paiement Client** : Le client termine le paiement sur le terminal SumUp
5. **Achèvement Automatique** : La commande se termine automatiquement lorsque le paiement est détecté

### Contrôles de Paiement

Lors de l'utilisation de la passerelle SumUp Terminal, vous disposez des options suivantes :

- **Démarrer un Nouveau Paiement** : Lancer une demande de paiement sur le terminal connecté
- **Annuler le Paiement** : Annuler un paiement en cours
- **Statut du Paiement** : Surveiller le statut actuel du processus de paiement

### Gestion des Commandes

- **Mises à Jour Automatiques** : Les commandes sont automatiquement marquées comme terminées après un paiement réussi
- **Suivi des Paiements** : Tous les détails de paiement sont enregistrés dans les notes de commande
- **Génération de Reçu** : Des reçus standard sont générés après les paiements réussis

## Configuration Requise

- **Compte SumUp** : Compte marchand SumUp actif
- **Accès API** : Clé API SumUp de votre tableau de bord marchand
- **Matériel Compatible** : Appareil de lecteur de cartes SumUp
- **WooCommerce POS** : Version gratuite ou pro installée et activée
- **Connexion Stable** : Connexion internet fiable pour la communication API

## Compatibilité Matérielle

:::important Exigences de Connectivité
Cette implémentation utilise le SDK JavaScript de SumUp, ce qui signifie qu'elle fonctionne via des applications web mais nécessite des **terminaux connectés à internet uniquement**. Les terminaux Bluetooth ne sont pas actuellement pris en charge.
:::

### Terminaux Pris en Charge (Connectés à Internet)

- **SumUp Solo** ✅ - Terminal connecté WiFi/Ethernet

### Terminaux Non Pris en Charge (Bluetooth)

- **SumUp Air** ❌ - Bluetooth uniquement
- **SumUp 3G** ❌ - Bluetooth uniquement

:::note Support Futur
Le support des terminaux Bluetooth sera ajouté aux applications iOS et Android au **1er trimestre 2026**. Cela permettra la prise en charge de tous les terminaux certifiés SumUp, y compris les modèles Air et 3G.
:::

## Dépannage

### Problèmes Courants

**Le Lecteur ne s'Appaire Pas**
- Assurez-vous d'utiliser un terminal connecté à internet pris en charge (Solo uniquement)
- Vérifiez que votre appareil SumUp Solo est connecté au WiFi et en ligne
- Vérifiez que l'appareil est en mode appairage et que le code d'appairage est entré correctement
- Confirmez que votre compte SumUp a l'accès API activé
- Assurez-vous d'avoir une connexion internet stable pendant l'appairage

**Paiement non Traité**
- Vérifiez que le lecteur est correctement appairé
- Vérifiez que votre compte SumUp est actif et en bon état
- Assurez-vous que l'appareil a une batterie suffisante et une connectivité
- Confirmez que la clé API est correcte et active

**Commande non Terminée**
- Vérifiez la stabilité de la connexion internet
- Vérifiez les paramètres de webhook dans votre compte SumUp
- Assurez-vous que le paiement a été réussi sur le terminal SumUp
- Consultez les journaux d'erreurs WordPress pour les problèmes de communication API

### Limites de l'API

La passerelle est optimisée pour réduire les appels API et éviter la limitation :
- Le statut du paiement est vérifié efficacement
- Les requêtes API inutiles sont minimisées
- La logique de nouvelle tentative automatique gère les défaillances temporaires
