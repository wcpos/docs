# Passerelle de Terminal SumUp {#sumup-terminal-gateway}

La passerelle Terminal SumUp vous permet d'accepter des paiements par carte en utilisant les lecteurs de cartes SumUp directement dans WooCommerce POS. Cette passerelle offre une intégration transparente avec le système de traitement des paiements de SumUp pour les transactions en personne.

## Installation

<Steps>
  <StepCard number="1" title="Télécharger et Installer">

1. Visitez la [page des releases de Terminal SumUp pour WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Téléchargez le fichier **sumup-terminal-for-woocommerce.zip** le plus récent
3. Dans votre admin WordPress, allez à `Plugins > Ajouter Nouveau > Téléverser Plugin`
4. Téléversez le fichier zip et activez le plugin

  </StepCard>
  <StepCard number="2" title="Configurer les Paramètres SumUp">

1. Naviguez vers `WP Admin > WooCommerce > Paramètres > Paiements`
2. Trouvez **Terminal SumUp** dans la liste des méthodes de paiement
3. Cliquez sur **Terminal SumUp** pour accéder aux paramètres
4. Entrez votre **Clé API SumUp** (disponible depuis votre tableau de bord marchand SumUp)
5. Enregistrez les paramètres

:::note
Vous n'avez pas besoin d'activer la passerelle Terminal SumUp dans les paramètres WooCommerce. Elle sera spécifiquement activée pour le POS dans une étape ultérieure.
:::

  </StepCard>
  <StepCard number="3" title="Appairer Votre Terminal SumUp">

1. Sur la même page de paramètres, localisez la section **Appairer le Lecteur**
2. Sur votre appareil SumUp, naviguez vers l'écran d'appairage pour afficher le code d'appairage
3. Entrez le code d'appairage affiché sur votre appareil SumUp
4. Cliquez sur **"Appairer le Lecteur"** pour établir la connexion
5. Attendez la confirmation que le lecteur a été apparié avec succès

:::warning Important
Le lecteur doit être apparié avec succès avant que vous puissiez traiter des paiements. Assurez-vous que le processus d'appairage est terminé avant de continuer.
:::

  </StepCard>
  <StepCard number="4" title="Activer dans WooCommerce POS" isLast>

1. Allez à `WP Admin > WooCommerce POS > Paramètres > Paiement`
2. Trouvez la passerelle **Terminal SumUp** dans la liste
3. Activez la passerelle pour une utilisation dans le POS
4. Enregistrez vos paramètres

  </StepCard>
</Steps>

## Utilisation {#usage}

### Traitement des Paiements {#processing-payments}

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Terminal SumUp" comme méthode de paiement
3. **Démarrer le Paiement** : Cliquez pour initier un nouveau paiement sur votre appareil SumUp
4. **Paiement du Client** : Le client complète le paiement sur le terminal SumUp
5. **Achèvement Automatique** : La commande se complète automatiquement lorsque le paiement est détecté

### Contrôles de Paiement {#payment-controls}

Lorsque vous utilisez la passerelle Terminal SumUp, vous avez les options suivantes :

- **Démarrer un Nouveau Paiement** : Initier une demande de paiement sur le terminal connecté
- **Annuler le Paiement** : Annuler un paiement actuellement en cours
- **État du Paiement** : Surveiller l'état actuel du processus de paiement

### Gestion des Commandes {#order-management}

- **Mises à Jour Automatiques** : Les commandes sont automatiquement marquées comme complètes après un paiement réussi
- **Suivi des Paiements** : Tous les détails de paiement sont enregistrés dans les notes de commande
- **Génération de Reçus** : Des reçus standard POS sont générés après des paiements réussis

## Compatibilité Matériel {#hardware-compatibility}

:::important Exigences de Connectivité
Cette implémentation utilise le SDK JavaScript de SumUp, ce qui signifie qu'elle fonctionne via des applications web mais nécessite **uniquement des terminaux connectés à Internet**. Les terminaux Bluetooth ne sont actuellement pas pris en charge.
:::

### Terminaux Prise en Charge (Connectés à Internet) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Terminal connecté par WiFi/Ethernet

### Terminaux Non Supportés (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Bluetooth uniquement
- **SumUp 3G** ❌ - Bluetooth uniquement

:::note Support Futur
La prise en charge des terminaux Bluetooth sera ajoutée aux applications iOS et Android dans **le T1 2026**. Cela permettra la prise en charge de tous les terminaux certifiés SumUp, y compris les modèles Air et 3G.
:::

## Dépannage {#troubleshooting}

### Problèmes Courants {#common-issues}

<Accordion>
  <AccordionItem question="Le lecteur ne veut pas s'appairer">

  - Assurez-vous que vous utilisez un terminal connecté à Internet pris en charge (Solo uniquement)
  - Vérifiez que votre appareil SumUp Solo est connecté au WiFi et en ligne
  - Assurez-vous que l'appareil est en mode d'appairage et que le code d'appairage est entré correctement
  - Confirmez que votre compte SumUp a l'accès API activé
  - Assurez-vous d'une connexion Internet stable pendant l'appairage

  </AccordionItem>

  <AccordionItem question="Paiement non traité">

  - Vérifiez que le lecteur est correctement apparié
  - Assurez-vous que votre compte SumUp est actif et en règle
  - Assurez-vous que l'appareil a une batterie suffisante et est connecté
  - Confirmez que la clé API est correcte et active

  </AccordionItem>

  <AccordionItem question="Commande ne se complétant pas">

  - Vérifiez la stabilité de la connexion Internet
  - Vérifiez les paramètres du webhook dans votre compte SumUp
  - Assurez-vous que le paiement a été réussi sur le terminal SumUp
  - Vérifiez les journaux d'erreurs WordPress pour les problèmes de communication API

  </AccordionItem>
</Accordion>

### Limites de Taux API {#api-rate-limits}

La passerelle est optimisée pour réduire les appels API et éviter le plafonnement des taux :
- L'état du paiement est vérifié efficacement
- Les demandes API inutiles sont minimisées
- La logique de reprise automatique gère les échecs temporaires
