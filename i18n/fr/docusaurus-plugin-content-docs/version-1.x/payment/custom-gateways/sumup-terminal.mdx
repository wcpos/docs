# Passerelle Terminal SumUp

La passerelle Terminal SumUp vous permet d'accepter les paiements par carte en utilisant les lecteurs de cartes SumUp directement dans WooCommerce POS. Cette passerelle offre une intégration transparente avec le système de traitement des paiements de SumUp pour les transactions en personne.

## Fonctionnalités

- **Intégration Matérielle** : Connectez les lecteurs de cartes SumUp à votre système POS via une connexion Internet
- **Traitement en Temps Réel** : Traitement instantané des paiements et finalisation des commandes
- **Détection Automatique** : Les commandes se complètent automatiquement lorsque le paiement est détecté
- **Transactions Sécurisées** : Traitement des paiements conforme à la norme PCI via SumUp
- **Appairage Facile** : Processus d'appairage simple avec des codes d'appairage
- **SDK Basé sur le Web** : Utilise le SDK JavaScript de SumUp pour la compatibilité avec les applications Web

## Installation

### Étape 1 : Télécharger et Installer

1. Visitez la [page des releases de SumUp Terminal pour WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Téléchargez le dernier fichier **sumup-terminal-for-woocommerce.zip**
3. Dans votre admin WordPress, allez à `Extensions > Ajouter Nouveau > Téléverser Extension`
4. Téléversez le fichier zip et activez l'extension

### Étape 2 : Configurer les Paramètres SumUp

1. Accédez à `WP Admin > WooCommerce > Paramètres > Paiements`
2. Trouvez **Terminal SumUp** dans la liste des méthodes de paiement
3. Cliquez sur **Terminal SumUp** pour accéder aux paramètres
4. Entrez votre **SumUp API Key** (disponible dans votre tableau de bord marchand SumUp)
5. Enregistrez les paramètres

:::note
Vous n'avez pas besoin d'activer la passerelle Terminal SumUp dans les paramètres de WooCommerce. Elle sera activée spécifiquement pour le POS à une étape ultérieure.
:::

### Étape 3 : Appairer Votre Terminal SumUp

1. Sur la même page de paramètres, localisez la section **Appairer le Lecteur**
2. Sur votre appareil SumUp, accédez à l'écran d'appairage pour afficher le code d'appairage
3. Entrez le code d'appairage affiché sur votre appareil SumUp
4. Cliquez sur **"Appairer le Lecteur"** pour établir la connexion
5. Attendez la confirmation que le lecteur a été appairé avec succès

:::warning Important
Le lecteur doit être appairé avec succès avant que vous puissiez traiter les paiements. Assurez-vous que le processus d'appairage est terminé avant de continuer.
:::

### Étape 4 : Activer dans WooCommerce POS

1. Allez à `WP Admin > WooCommerce POS > Paramètres > Caisse`
2. Trouvez la passerelle **Terminal SumUp** dans la liste
3. Activez la passerelle pour une utilisation dans le POS
4. Enregistrez vos paramètres

## Utilisation

### Traitement des Paiements

1. **Ajouter des Articles** : Ajoutez des produits à votre panier dans le POS
2. **Sélectionner la Passerelle** : Choisissez "Terminal SumUp" comme méthode de paiement
3. **Démarrer le Paiement** : Cliquez pour initier un nouveau paiement sur votre appareil SumUp
4. **Paiement du Client** : Le client termine le paiement sur le terminal SumUp
5. **Finalisation Automatique** : La commande se complète automatiquement lorsque le paiement est détecté

### Contrôles de Paiement

Lorsque vous utilisez la passerelle Terminal SumUp, vous avez les options suivantes :

- **Démarrer un Nouveau Paiement** : Initiez une demande de paiement sur le terminal connecté
- **Annuler le Paiement** : Annulez un paiement en cours
- **Statut du Paiement** : Surveillez l'état actuel du processus de paiement

### Gestion des Commandes

- **Mises à Jour Automatiques** : Les commandes sont automatiquement marquées comme complètes après un paiement réussi
- **Suivi des Paiements** : Tous les détails du paiement sont enregistrés dans les notes de commande
- **Génération de Reçus** : Les reçus standard du POS sont générés après des paiements réussis

## Exigences

- **Compte SumUp** : Compte marchand SumUp actif
- **Accès API** : SumUp API key depuis votre tableau de bord marchand
- **Matériel Compatible** : Appareil lecteur de carte SumUp
- **WooCommerce POS** : Version gratuite ou Pro installée et activée
- **Connexion Stable** : Connexion Internet fiable pour la communication API

## Compatibilité Matérielle

:::important Exigences de Connectivité
Cette implémentation utilise le SDK JavaScript de SumUp, ce qui signifie qu'il fonctionne à travers des applications Web mais nécessite **uniquement des terminaux connectés à Internet**. Les terminaux Bluetooth ne sont actuellement pas pris en charge.
:::

### Terminaux Supportés (Connectés à Internet)

- **SumUp Solo** ✅ - Terminal connecté en WiFi/Ethernet

### Terminaux Non Supportés (Bluetooth)

- **SumUp Air** ❌ - Bluetooth uniquement
- **SumUp 3G** ❌ - Bluetooth uniquement

:::note Support Futur
Le support des terminaux Bluetooth sera ajouté aux applications iOS et Android au **T1 2026**. Cela permettra de prendre en charge tous les terminaux certifiés SumUp, y compris les modèles Air et 3G.
:::

## Résolution de Problèmes

### Problèmes Courants

**Le Lecteur ne S’appaire Pas**
- Assurez-vous que vous utilisez un terminal connecté à Internet pris en charge (Solo uniquement)
- Vérifiez que votre appareil SumUp Solo est connecté au WiFi et en ligne
- Assurez-vous que l'appareil est en mode d'appairage et que le code d'appairage est saisi correctement
- Confirmez que votre compte SumUp a l'accès API activé
- Assurez-vous d'une connexion Internet stable lors de l'appairage

**Le Paiement ne se Traite Pas**
- Vérifiez que le lecteur est correctement appairé
- Assurez-vous que votre compte SumUp est actif et en règle
- Vérifiez que l'appareil a suffisamment de batterie et de connectivité
- Confirmez que la SumUp API Key est correcte et active

**La Commande ne se Termine Pas**
- Vérifiez la stabilité de la connexion Internet
- Vérifiez les paramètres des webhooks dans votre compte SumUp
- Assurez-vous que le paiement a été réussi sur le terminal SumUp
- Consultez les journaux d'erreurs WordPress pour les problèmes de communication API

### Limites de Taux API

La passerelle est optimisée pour réduire les appels API et éviter la limitation de taux :
- L'état du paiement est vérifié de manière efficace
- Les requêtes API inutiles sont minimisées
- La logique de nouvelle tentative automatique gère les échecs temporaires
