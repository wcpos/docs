WCPOS prend en charge plusieurs méthodes de paiement pour s'adapter aux différents besoins commerciaux. Le processus de paiement utilise une iframe/webview pour charger la page de paiement de commande (Order Pay) de WooCommerce, ce qui signifie que toute passerelle de paiement compatible avec WooCommerce peut fonctionner dans le POS.

## Méthodes de Paiement Par Défaut {#default-payment-methods}

La version gratuite inclut deux passerelles de paiement intégrées :

### Espèces {#cash}

- Saisir le montant remis
- Calcul automatique de la monnaie
- Aucune configuration supplémentaire requise

### Carte {#card}

- À utiliser avec des terminaux de carte externes
- Il suffit de marquer le paiement comme complet après traitement sur votre terminal
- Aucune intégration directe requise

## Passerelles de Paiement Supplémentaires (Pro) {#additional-payment-gateways-pro}

:::info Fonction Pro
Les passerelles de paiement supplémentaires nécessitent [WCPOS Pro](/getting-started/pro-license).
:::

Avec Pro, vous pouvez activer des passerelles de paiement WooCommerce supplémentaires dans le processus de paiement du POS :

- **Stripe Terminal** - Intégration directe avec les lecteurs de cartes Stripe
- **SumUp Terminal** - Intégration avec les lecteurs de cartes SumUp
- **Toute Passerelle WooCommerce** - Activez toute passerelle installée sur votre boutique

## Activer et Désactiver les Passerelles {#enabling-and-disabling-gateways}

Les passerelles de paiement sont gérées dans l'administration WordPress :

1. Accédez à `WP Admin > POS > Settings > Checkout`
2. Vous verrez une liste de toutes les passerelles de paiement WooCommerce installées
3. Activez ou désactivez chaque passerelle pour le POS indépendamment de votre boutique en ligne
4. Faites glisser pour réorganiser — la première passerelle activée devient celle par défaut au moment du paiement

Cela signifie qu'une passerelle peut être active sur votre site web mais désactivée dans le POS, ou inversement. Consultez les [Paramètres de Paiement](/settings/wp-admin/checkout) pour tous les détails.

## Tester les Paiements {#testing-payments}

Avant de mettre en service une nouvelle passerelle :

1. **Passerelle Espèces** — toujours disponible pour tester le flux de paiement sans transactions réelles.
2. **Stripe Terminal** — Stripe fournit un [lecteur simulé](https://docs.stripe.com/terminal/testing) pour le mode test. Activez d'abord le mode test dans votre tableau de bord Stripe.
3. **Passerelles tierces** — consultez la propre documentation de la passerelle pour les instructions du mode bac à sable/test. La plupart des passerelles WooCommerce prennent en charge une option de mode test.

:::tip
Si une passerelle nouvellement activée n'apparaît pas au moment du paiement, essayez de rafraîchir le POS. Les modifications de passerelle dans WP Admin prennent effet au prochain chargement de page.
:::

## Problèmes Courants {#common-issues}

### La Passerelle N'apparaît Pas au Paiement {#gateway-not-appearing-at-checkout}

- Confirmez que la passerelle est activée pour le POS dans `WP Admin > POS > Settings > Checkout`.
- Vérifiez que la passerelle est également activée dans `WP Admin > WooCommerce > Settings > Payments` — une passerelle doit être active dans WooCommerce avant de pouvoir apparaître dans le POS.
- Rafraîchissez le POS après avoir effectué des modifications.

### La Passerelle S'affiche Incorrectement {#gateway-displays-incorrectly}

L'iframe de paiement charge les styles du thème de votre site, ce qui peut parfois interférer. Utilisez le bouton **Checkout Settings** dans la fenêtre modale de paiement pour désactiver de manière sélective les styles ou les scripts. Consultez [Dépannage du Paiement](/pos/checkout#checkout-settings-troubleshooting) pour plus de détails.

### Le Traitement du Paiement Échoue {#payment-processing-fails}

- Vérifiez les journaux d'erreurs de votre site (`WP Admin > POS > Support > Logs`) pour les erreurs spécifiques à la passerelle.
- Assurez-vous que votre certificat SSL est valide — la plupart des passerelles de paiement nécessitent HTTPS.
- Si vous utilisez un environnement de préproduction ou local, confirmez que la passerelle prend en charge le mode test/bac à sable.
