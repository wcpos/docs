WCPOS prend en charge plusieurs méthodes de paiement pour répondre aux différents besoins commerciaux. Le paiement utilise un iframe/webview pour charger la page WooCommerce Order Pay, ce qui signifie que toute passerelle de paiement qui fonctionne avec WooCommerce peut fonctionner dans le POS.

## Méthodes de paiement par défaut

La version gratuite comprend deux passerelles de paiement intégrées :

### Espèces

- Entrez le montant payé
- Calcul automatique du change
- Aucune configuration supplémentaire requise

### Carte

- À utiliser avec des terminaux de paiement externes
- Marquez simplement le paiement comme terminé après le traitement sur votre terminal
- Aucune intégration directe requise

## Passerelles de paiement supplémentaires (Pro)

:::info Fonctionnalité Pro
Les passerelles de paiement supplémentaires nécessitent [WCPOS Pro](/getting-started/pro-license).
:::

Avec Pro, vous pouvez activer des passerelles de paiement WooCommerce supplémentaires dans le paiement du POS:

- **Stripe Terminal** - Intégration directe avec les lecteurs de cartes Stripe
- **SumUp Terminal** - Intégration avec les lecteurs de cartes SumUp
- **Any WooCommerce Gateway** - Activez n'importe quelle passerelle installée sur votre magasin

## Activation et désactivation des passerelles de paiement

Les passerelles de paiement sont gérées dans l'administration WordPress :

1. Allez à `WP Admin > POS > Settings > Checkout`
2. Vous verrez une liste de toutes les passerelles de paiement WooCommerce installées
3. Activez ou désactivez chaque passerelle pour le POS indépendamment de votre boutique en ligne
4. Faites glisser pour réorganiser — la première passerelle activée devient la passerelle par défaut au moment du paiement

Cela signifie qu'une passerelle peut être active sur votre site web mais désactivée dans le POS, ou vice versa. Voir [Paramètres de paiement](/settings/wp-admin/checkout) pour plus de détails.

## Test des paiements

Avant de mettre en ligne une nouvelle passerelle :

1. **Passerelle de paiement en espèces** — toujours disponible pour tester le flux de paiement sans transactions réelles.
2. **Stripe Terminal** — Stripe fournit [lecteur simulé](https://docs.stripe.com/terminal/testing) pour le mode test. Activez d'abord le mode test dans votre tableau de bord Stripe.
3. **Passerelles tierces** — consultez la documentation propre de la passerelle pour les instructions du mode sandbox/test. La plupart des passerelles WooCommerce supportent un bouton de basculement du mode test.

:::tip
Si une passerelle nouvellement activée n'apparaît pas à la caisse, essayez d'actualiser le POS. Les modifications de passerelle dans WP Admin prennent effet au prochain chargement de page.
:::

## Problèmes courants

### Passerelle n'apparaissant pas à la caisse

- Confirmez que la passerelle est activée pour POS dans `WP Admin > POS > Settings > Checkout`.
- Vérifiez que la passerelle est également activée dans `WP Admin > WooCommerce > Settings > Payments` — une passerelle doit être active dans WooCommerce avant de pouvoir apparaître au PDV.
- Actualisez le PDV après avoir apporté des modifications.

### La passerelle s'affiche de manière incorrecte

L'iframe de paiement charge les styles de thème de votre site, ce qui peut parfois interférer. Utilisez le bouton **Paramètres de paiement** dans la fenêtre modale de paiement pour désactiver sélectivement les styles ou les scripts. Voir [Dépannage du paiement](/pos/checkout#checkout-settings-troubleshooting) pour plus de détails.

### Le traitement du paiement échoue

- Vérifiez les journaux d'erreurs de votre site (`WP Admin > POS > Support > Logs`) pour les erreurs spécifiques à la passerelle.
- Assurez-vous que votre certificat SSL est valide — la plupart des passerelles de paiement nécessitent HTTPS.
- Si vous utilisez un environnement de staging ou local, confirmez que la passerelle supporte le mode test/sandbox.
