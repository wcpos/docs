WCPOS prend en charge plusieurs méthodes de paiement pour répondre aux différents besoins professionnels. Le paiement utilise une iframe/webview pour charger la page WooCommerce Order Pay, ce qui signifie que toute passerelle de paiement compatible avec WooCommerce peut fonctionner dans le POS.

## Méthodes de Paiement par Défaut

La version gratuite inclut deux passerelles de paiement intégrées :

### Espèces

- Entrez le montant remis
- Calcul automatique de la monnaie
- Aucune configuration supplémentaire requise

### Carte

- Pour une utilisation avec des terminaux de cartes externes
- Marquez simplement le paiement comme terminé après le traitement sur votre terminal
- Aucune intégration directe requise

## Passerelles de paiement supplémentaires (Pro)

:::info Pro Feature
Les passerelles de paiement supplémentaires nécessitent [WCPOS Pro](/getting-started/pro-license).
:::

Avec Pro, vous pouvez activer des passerelles de paiement WooCommerce supplémentaires à la caisse POS :

- **Stripe Terminal** - Intégration directe avec les lecteurs de cartes Stripe
- **SumUp Terminal** - Intégration avec les lecteurs de cartes SumUp
- **Any WooCommerce Gateway** - Activez n'importe quelle passerelle installée sur votre magasin

## Activation et désactivation des passerelles

Les passerelles de paiement sont gérées dans l'administration WordPress :

1. Aller à `WP Admin > POS > Settings > Checkout`
2. Vous verrez une liste de toutes les passerelles de paiement WooCommerce installées
3. Activez ou désactivez chaque passerelle pour le POS indépendamment de votre magasin en ligne
4. Glissez pour réorganiser — la première passerelle activée devient la passerelle par défaut à la caisse

Cela signifie qu'une passerelle peut être active sur votre site Web mais désactivée dans le POS, ou vice versa. Consultez [Checkout Settings](/settings/wp-admin/checkout) pour plus de détails.

## Test des paiements

Avant de passer en direct avec une nouvelle passerelle :

1. **Cash gateway** — toujours disponible pour tester le flux de caisse sans transactions réelles.
2. **Stripe Terminal** — Stripe fournit un [lecteur simulé](https://docs.stripe.com/terminal/testing) pour le mode test. Activez d'abord le mode test dans votre tableau de bord Stripe.
3. **Third-party gateways** — consultez la documentation propre de la passerelle pour les instructions du mode bac à sable/test. La plupart des passerelles WooCommerce prennent en charge un bouton de mode test.

:::tip
Si une passerelle nouvellement activée n'apparaît pas à la caisse, essayez d'actualiser le POS. Les modifications de passerelle dans WP Admin prennent effet au prochain chargement de page.
:::

## Problèmes courants

### Passerelle absente de la caisse

- Confirmez que la passerelle est activée pour le POS dans `WP Admin > POS > Settings > Checkout`.
- Vérifiez que la passerelle est également activée dans `WP Admin > WooCommerce > Settings > Payments` — une passerelle doit être active dans WooCommerce avant de pouvoir apparaître dans le POS.
- Actualisez le POS après avoir apporté les modifications.

### Affichage incorrect de la passerelle

L'iframe de paiement charge les styles de thème de votre site, ce qui peut parfois causer des interférences. Utilisez le bouton **Checkout Settings** dans la fenêtre modale de paiement pour désactiver sélectivement les styles ou les scripts. Consultez [Checkout Troubleshooting](/pos/checkout#checkout-settings-troubleshooting) pour plus de détails.

### Échec du traitement du paiement

- Vérifiez les journaux d'erreurs de votre site (`WP Admin > POS > Support > Logs`) pour les erreurs spécifiques à la passerelle.
- Assurez-vous que votre certificat SSL est valide — la plupart des passerelles de paiement nécessitent HTTPS.
- Si vous utilisez un environnement de test ou local, confirmez que la passerelle supporte le mode test/sandbox.
