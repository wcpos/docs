L'écran des logs enregistre l'activité du système au sein du POS. C'est un outil essentiel pour déboguer les problèmes et comprendre ce qui se passe en arrière-plan.

{/* Capture d'écran : logs-screen.png */}

## Accéder aux Logs

Ouvrez l'écran des logs depuis le tiroir de navigation gauche (l'icône du rythme cardiaque).

## Présentation de l'Interface

### Recherche & Filtre

- **Recherche Logs** - Trouvez des entrées de log spécifiques par contenu de message
- **Filtre de Niveau de Log** - Filtrer par niveau de log (info, warn, error)
- **Display settings** - Configurer les colonnes visibles

### Tableau des Logs

Chaque entrée de log montre :

| Colonne | Description |
|---------|-------------|
| **Temps** | Quand l'événement s'est produit |
| **Niveau** | Sévérité : info (gris), warn (orange), error (rouge) |
| **Message** | Description lisible par l'homme de l'événement |
| **Contexte** | JSON extensible avec des détails supplémentaires |
| **Code** | Code d'erreur (lien vers la documentation) |

## Niveaux de Log

### Info (Gris)

Opérations normales et événements réussis :
- "Tous les IDs de produits récupérés"
- "Nouveaux clients synchronisés"
- "Tous les IDs de taxes récupérés"

### Warning (Orange)

Problèmes non critiques qui peuvent nécessiter une attention :
- "Impossible de rafraîchir la session"
- "Nouvelle tentative de synchronisation"

### Error (Rouge)

Problèmes qui doivent être résolus :
- "Désolé, vous ne pouvez pas lister les ressources" (problème de permission)
- "La requête API a échoué"
- "Erreur de synchronisation de la base de données"

## Données de Contexte

Cliquez sur la flèche d'extension de n'importe quelle entrée de log pour voir le contexte JSON détaillé :

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Ces données incluent :
- **Comptes d'articles** - Combien d'enregistrements ont été affectés
- **Temps d'exécution** - Combien de temps l'opération a duré
- **Charge du serveur** - Métriques de performance du serveur
- **Détails de la requête/réponse** - Pour déboguer les problèmes d'API

## Codes d'Erreur

De nombreuses entrées de log incluent un **code d'erreur** (par exemple, `API02004`, `DB01001`). Ces codes renvoient directement à la documentation [Codes d'Erreur](/error-codes), qui fournit :

- Explication détaillée de l'erreur
- Causes courantes
- Étapes de dépannage

## Entrées de Log Courantes

### Opérations de Synchronisation

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

Ceci indique une activité de synchronisation normale. Le contexte montre combien d'articles ont été synchronisés.

### Erreurs de Permission

```
[error] Sorry, you cannot list resources    {...} API02004
```

Cela signifie généralement que l'utilisateur n'a pas la permission d'accéder à une ressource. Vérifiez les rôles des utilisateurs dans `WP Admin > POS > Settings > Access`.

### Problèmes de Session

```
[warn] Unable to refresh session    {...} API02007
```

Le POS n'a pas pu rafraîchir la session d'authentification. Cela peut nécessiter de se déconnecter et de se reconnecter.

## Utiliser les Logs pour le Support

Lors de la demande de support :

1. **Reproduisez le problème** - Effectuez l'action qui cause le problème
2. **Vérifiez les logs** - Recherchez des erreurs (rouge) ou des warnings (orange)
3. **Développez le contexte** - Obtenez les détails JSON complets
4. **Notez le code d'erreur** - Incluez ceci dans votre demande de support
5. **Copiez les entrées pertinentes** - Partagez les messages de log avec le contexte

## Surveillance des Performances

Les logs incluent les temps d'exécution et les métriques de charge du serveur. Si vous remarquez :

- **Temps d'exécution élevés** - Le serveur peut être surchargé ou lent
- **Valeurs de charge serveur élevées** - Envisagez d'optimiser le serveur
- **Délai d'attente fréquent** - Vérifiez la connectivité réseau
