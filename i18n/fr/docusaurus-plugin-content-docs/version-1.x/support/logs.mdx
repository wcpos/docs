L'écran des Journaux enregistre l'activité système au sein du POS. C'est un outil essentiel pour déboguer les problèmes et comprendre ce qui se passe en coulisses.

{/* Screenshot: logs-screen.png */}

## Accès aux Journaux

Ouvrez l'écran des Journaux depuis le tiroir de navigation de gauche (l'icône de battement cardiaque).

## Vue d'Ensemble de l'Interface

### Recherche et Filtrage

- **Rechercher des Journaux** - Trouver des entrées de journal spécifiques par contenu de message
- **Filtre de Niveau de Journal** - Filtrer par niveau de journal (info, avertissement, erreur)
- **Paramètres d'affichage** - Configurer les colonnes visibles

### Tableau des Journaux

Chaque entrée de journal affiche :

| Colonne | Description |
|--------|-------------|
| **Heure** | Quand l'événement s'est produit |
| **Niveau** | Gravité : info (gris), avertissement (orange), erreur (rouge) |
| **Message** | Description lisible par l'homme de l'événement |
| **Contexte** | JSON extensible avec des détails supplémentaires |
| **Code** | Code d'erreur (lien vers la documentation) |

## Niveaux de Journal

### Info (Gris)

Opérations normales et événements réussis :
- "Récupéré tous les identifiants des produits"
- "Clients synchronisés"
- "Récupéré tous les identifiants des taxes"

### Avertissement (Orange)

Problèmes non critiques pouvant nécessiter une attention :
- "Impossible de rafraîchir la session"
- "Tentative de nouvelle synchronisation"

### Erreur (Rouge)

Problèmes devant être résolus :
- "Désolé, vous ne pouvez pas lister les ressources" (problème de permission)
- "Échec de la requête API"
- "Erreur de synchronisation de base de données"

## Données de Contexte

Cliquez sur la flèche d'expansion de toute entrée de journal pour voir le contexte JSON détaillé :

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Ces données incluent :
- **Nombre d'éléments** - Combien d'enregistrements ont été affectés
- **Temps d'exécution** - Durée de l'opération
- **Charge du serveur** - Métriques de performance du serveur
- **Détails de requête/réponse** - Pour déboguer les problèmes API

## Codes d'Erreur

De nombreuses entrées de journal incluent un **code d'erreur** (par exemple, `API02004`, `DB01001`). Ces codes sont directement liés à la documentation des [Codes d'Erreur](/error-codes), qui fournit :

- Explication détaillée de l'erreur
- Causes courantes
- Étapes de dépannage

## Entrées de Journal Courantes

### Opérations de Synchronisation

```
[info] Récupéré tous les identifiants des produits    {...} 3 éléments
[info] Clients synchronisés            {...} 1 élément
[info] Récupéré tous les identifiants des taxes       {...} 3 éléments
```

Celles-ci indiquent une activité de synchronisation normale. Le contexte montre combien d'éléments ont été synchronisés.

### Erreurs de Permission

```
[error] Désolé, vous ne pouvez pas lister les ressources    {...} API02004
```

Cela signifie généralement que l'utilisateur n'a pas la permission d'accéder à une ressource. Vérifiez les rôles utilisateur dans `WP Admin > POS > Paramètres > Accès`.

### Problèmes de Session

```
[warn] Impossible de rafraîchir la session    {...} API02007
```

Le POS n'a pas pu actualiser la session d'authentification. Cela peut nécessiter une déconnexion et une reconnexion.

## Utilisation des Journaux pour le Support

Lors d'une demande de support :

1. **Reproduire le problème** - Effectuez l'action qui cause le problème
2. **Vérifier les journaux** - Recherchez des erreurs (rouge) ou des avertissements (orange)
3. **Développer le contexte** - Obtenir les détails JSON complets
4. **Noter le code d'erreur** - L'inclure dans votre demande de support
5. **Copier les entrées pertinentes** - Partager les messages de journal avec leur contexte

## Surveillance des Performances

Les journaux incluent des métriques de temps d'exécution et de charge du serveur. Si vous remarquez :

- **Temps d'exécution élevés** - Le serveur peut être surchargé ou lent
- **Valeurs de charge du serveur élevées** - Envisager l'optimisation du serveur
- **Dépassements de délai fréquents** - Vérifier la connectivité réseau
