## Note de commande

<div class="image-container">

<p class="image-caption">Modalité Note de commande dans le POS (à gauche) et Note de commande dans l'administrateur WP (à droite)</p>

- Ouvre une boîte de dialogue pour ajouter une note client à la commande.
- Les notes du client sont visibles par le client et apparaissent sur le reçu par défaut, dans la zone Mon compte du client et dans « WP Admin > WooCommerce > Commandes > Commande » sous l'adresse de livraison.

:::note
Les notes de commande sont visibles par les clients.
:::

## Métadonnées de commande

- Ouvre une fenêtre modale pour gérer des métadonnées supplémentaires pour la commande.
- Les fonctionnalités incluent :
- Modification de la devise de la commande.
- Ajout d'un identifiant de transaction (par exemple, à partir d'un terminal de paiement externe).
- Ajout de métadonnées personnalisées pour la commande.

:::note
Des fonctionnalités supplémentaires seront ajoutées à cette fenêtre modale au fil du temps, notamment l'intégration avec d'autres plugins WooCommerce.
:::

### Qu'est-ce que les métadonnées dans WooCommerce ?

Les métadonnées fournissent des informations supplémentaires sur les commandes et sont accessibles via l'API REST de WooCommerce. Les développeurs et les plugins utilisent ces données pour étendre les fonctionnalités, telles que l'intégration de services tiers.

### Affichage JSON

- Affichez la représentation JSON brute de la commande, utile à des fins de débogage ou d'intégration.

## Enregistrer sur le serveur

- Vous permet d'enregistrer les paniers ouverts sur le serveur, afin qu'ils soient stockés en toute sécurité et accessibles pour une utilisation future.

### Pourquoi enregistrer ?

Lorsqu'une nouvelle commande est créée dans le POS, elle n'existe que localement.
L'enregistrement de la commande garantit sa persistance, même si la base de données locale est effacée.

## Annulation

- Deux scénarios :
- Commandes non enregistrées :
- Si la commande n'a pas été enregistrée sur le serveur, elle sera définitivement supprimée de la base de données locale.
- Commandes enregistrées :
- Si la commande est enregistrée sur le serveur, elle sera déplacée vers le dossier Corbeille de WooCommerce et supprimée de la base de données locale.
- Pour récupérer une commande annulée :
- Accédez à « WP Admin > WooCommerce > Commandes > Corbeille ».

## Paiement

- Enregistre la commande sur le serveur et lance le processus de paiement.
- Ouvre l'écran de paiement pour terminer le paiement.
