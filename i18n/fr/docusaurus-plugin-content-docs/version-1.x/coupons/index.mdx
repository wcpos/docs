:::info Fonctionnalité Pro
L'écran Coupons et la possibilité d'appliquer des coupons en caisse nécessitent [WCPOS Pro](/getting-started/pro-license). Les utilisateurs de la version gratuite voient un aperçu flouté de l'écran Coupons, mais ne peuvent ni consulter les détails des coupons ni appliquer de codes dans le panier.
:::

L'écran Coupons permet de parcourir et de rechercher vos coupons WooCommerce depuis le point de vente, et le bouton **Ajouter un coupon** dans le panier les applique à la commande en cours. Les coupons sont toujours créés et gérés dans WooCommerce — le PDV les synchronise et les valide localement pour un retour instantané en caisse.

## Aperçu de l'interface {#interface-overview}

### Actions de l'en-tête {#header-actions}

En haut de l'écran :

- **Barre de recherche** — Rechercher un coupon par code
- **Paramètres d'affichage** (<Icon name="sliders" />) — Configurer les colonnes visibles

### Tableau des coupons {#coupons-table}

La zone principale affiche les coupons synchronisés avec :

- **Code** — Le code de coupon saisi par les clients
- **Description** — Description interne (également utilisée comme libellé sur le reçu lorsqu'elle est définie)
- **Type de remise** — Pourcentage, montant fixe sur le panier ou montant fixe sur le produit
- **Montant** — La valeur de la remise
- **Utilisé / Limite** — Nombre d'utilisations et limite d'utilisation globale
- **Expiration** — Date d'expiration, ou vide si aucune
- **Actions** — Menu à trois points

### Pied de page {#footer}

- Nombre de coupons avec bouton de synchronisation (<Icon name="arrow-rotate-right" />). **Appui long** pour les options Effacer et Actualiser.

## Types de coupons {#coupon-types}

WCPOS prend en charge les trois types de coupons standard de WooCommerce :

| Type | Description | Exemple |
|---|---|---|
| **Remise en pourcentage** | Réduit le sous-total du panier d'un pourcentage | `10%` de réduction sur la commande |
| **Remise fixe sur le panier** | Réduit le total du panier d'un montant fixe | `$5` de réduction sur la commande |
| **Remise fixe par produit** | Réduit les lignes de produits correspondants d'un montant fixe par unité | `$2` de réduction sur chaque T-shirt |

Les trois types respectent les mêmes règles de validation (expiration, limites d'utilisation, restrictions de produit/catégorie, etc.).

## Création de coupons {#creating-coupons}

Les coupons sont créés dans WooCommerce, et non dans le PDV. Accédez à **WP Admin → Marketing → Coupons → Ajouter un coupon** :

1. **Code promo** — ce que les caissiers saisissent en caisse. Les codes ne sont pas sensibles à la casse.
2. **Description** — note interne. WCPOS utilise ce champ comme libellé de remise sur les reçus lorsqu'il est renseigné ; préférez donc une formulation adaptée au client, par exemple « Remise responsable 10 % », plutôt qu'un code interne.
3. **Type de remise** — pourcentage, remise fixe sur le panier ou remise fixe sur le produit.
4. **Montant du coupon** — la valeur de la remise.
5. **Date d'expiration** — facultative. Passée cette date, le coupon sera refusé.

### Restriction d'utilisation {#usage-restriction}

- **Montant minimum / maximum** — le coupon ne s'applique que si le sous-total est supérieur ou inférieur au seuil défini.
- **Utilisation individuelle uniquement** — empêche ce coupon d'être combiné avec tout autre coupon.
- **Exclure les articles en promotion** — ignore les articles déjà en promotion (ainsi que toute ligne dont le prix a été modifié en caisse par un caissier — voir [Modifications de prix au PDV](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)).
- **Produits / Exclure des produits** — restreint le coupon à des produits spécifiques (ou les exclut).
- **Catégories de produits / Exclure des catégories** — restriction par catégorie. Les produits divers ou personnalisés créés au point de vente respectent également les catégories assignées via le PDV.
- **Adresses e-mail autorisées** — limite le coupon à des adresses e-mail de clients spécifiques (les caractères génériques `*` sont acceptés).

### Limites d'utilisation {#usage-limits}

- **Limite d'utilisation par coupon** — nombre total de fois où ce code peut être utilisé par l'ensemble des clients.
- **Limite d'utilisation par utilisateur** — nombre de fois qu'un même client peut l'utiliser.
- **Limiter l'utilisation à X articles** — pour les coupons à produit fixe, limite le nombre d'articles éligibles bénéficiant de la remise par utilisation.

### Conseils pour l'utilisation en point de vente {#tips-for-pos-use}

- Créez un coupon **« Responsable 10 % »** (10 % de réduction, sans minimum d'achat, sans date d'expiration, restriction d'utilisation unique désactivée) et communiquez le code aux responsables — le personnel peut l'appliquer en caisse pour des ajustements ponctuels, et vous conservez une remise traçable dans les rapports.
- Créez un coupon **« Fidélité 5 $ »** pour récompenser les clients réguliers.
- Pour les promotions ponctuelles, définissez une date d'expiration courte afin que le code ne puisse pas être réutilisé par erreur.
- Renseigner le champ **Description** avec le texte souhaité sur les reçus est le moyen le plus rapide de personnaliser vos remises.

## Appliquer un coupon en caisse {#applying-a-coupon-at-the-register}

Appuyez sur **Ajouter un coupon** dans le panier, puis saisissez le code ou recherchez par description. Le coupon est validé instantanément à partir des données synchronisées et apparaît sous forme de pastille supprimable au-dessus des totaux du panier.

Pour le flux de travail complet en caisse — recherche, pastilles de coupon, cumul de plusieurs coupons et résolution des erreurs de validation — consultez **[Appliquer des coupons en caisse](/coupons/applying-coupons)**.

## Fonctionnement de la validation {#how-validation-works}

Lorsqu'un code est saisi, le PDV vérifie toutes les mêmes règles que WooCommerce vérifierait sur le serveur :

- Le coupon existe et n'est pas expiré
- Les limites d'utilisation ne sont pas dépassées (globales et par utilisateur)
- Le montant minimum/maximum d'achat est atteint
- Les coupons « Utilisation individuelle » ne sont pas en conflit avec les coupons déjà appliqués
- Les restrictions de produit/catégorie correspondent au contenu du panier
- Les restrictions par e-mail correspondent au client sélectionné (ou sont ignorées pour les commandes d'invités)
- Les articles en promotion sont exclus si **Exclure les articles en promotion** est activé

Si une vérification échoue, le caissier voit un message d'erreur spécifique (par ex., « Ce coupon a expiré » ou « Montant minimum non atteint »). Les mêmes vérifications sont effectuées à nouveau sur le serveur lors de la soumission de la commande — la validation locale sert à la rapidité, pas à la confiance.

## Comportement de synchronisation {#sync-behaviour}

Les coupons se synchronisent depuis WooCommerce vers l'appareil via le pipeline de réplication standard de WCPOS :

- Les nouveaux coupons créés dans WP Admin apparaissent dans le PDV lors de la prochaine synchronisation.
- Les modifications apportées aux coupons existants (nombre d'utilisations, changements de date d'expiration, etc.) sont synchronisées automatiquement.
- Les coupons sont stockés localement dans IndexedDB et restent donc disponibles hors ligne.

Si vous venez de créer un coupon dans WP Admin et qu'il n'apparaît pas encore, la synchronisation n'a peut-être pas encore eu lieu. Depuis le pied de page de l'écran Coupons, appuyez sur l'icône de synchronisation (<Icon name="arrow-rotate-right" />) pour actualiser — appuyez longuement pour **Effacer et actualiser** si une récupération complète est nécessaire.

## Connectivité {#connectivity}

- **L'application des coupons** fonctionne hors ligne, car la validation est effectuée côté client à partir des données synchronisées.
- **La finalisation du paiement** nécessite toujours une connexion au serveur — celui-ci procède à une nouvelle validation et constitue la source de référence pour le montant total final de la commande.
- **La création ou la modification des coupons** s'effectue dans WP Admin et nécessite une connexion à votre site WordPress.

## Interaction entre les coupons et les remises PDV {#how-coupons-interact-with-pos-discounts}

Lorsqu'un caissier réduit le prix d'une ligne en caisse et qu'un coupon est ensuite appliqué, le coupon est calculé sur le **prix réduit**, et non sur le prix d'origine. Les lignes dont le prix a été réduit via le PDV sont traitées comme « en promotion » — les coupons avec l'option **Exclure les articles en promotion** activée les ignoreront. Consultez la section [Remises](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons) pour en savoir plus.
