:::info Fonctionnalité Pro
L'écran Coupons et la possibilité d'appliquer des coupons en caisse nécessitent [WCPOS Pro](/getting-started/pro-license). Les utilisateurs de la version gratuite voient un aperçu flouté de l'écran Coupons, mais ne peuvent ni consulter les détails des coupons ni appliquer de codes dans le panier.
:::

L'écran Coupons permet de parcourir et de rechercher vos coupons WooCommerce depuis le point de vente, et le bouton **Ajouter un coupon** dans le panier les applique à la commande en cours. Les coupons sont toujours créés et gérés dans WooCommerce — le PDV les synchronise et les valide localement pour un retour instantané en caisse.

## Aperçu de l'interface {#interface-overview}

### Actions de l'en-tête {#header-actions}

En haut de l'écran :

- **Barre de recherche** — Rechercher un coupon par code
- **Paramètres d'affichage** (<Icon name="sliders" />) — Configurer les colonnes visibles

### Tableau des coupons {#coupons-table}

La zone principale affiche les coupons synchronisés avec :

- **Code** — Le code de coupon saisi par les clients
- **Description** — Description interne (également utilisée comme libellé sur le reçu lorsqu'elle est définie)
- **Type de remise** — Pourcentage, panier fixe ou produit fixe
- **Montant** — La valeur de la remise
- **Utilisé / Limite** — Nombre d'utilisations et limite d'utilisation globale
- **Expiration** — Date d'expiration, ou vide si aucune
- **Actions** — Menu à trois points

### Pied de page {#footer}

- Nombre de coupons avec bouton de synchronisation (<Icon name="arrow-rotate-right" />). **Appui long** pour les options Effacer et Actualiser.

## Types de coupons {#coupon-types}

WCPOS prend en charge les trois types de coupons standard de WooCommerce :

| Type | Description | Exemple |
|---|---|---|
| **Remise en pourcentage** | Réduit le sous-total du panier d'un pourcentage | `10%` de réduction sur la commande |
| **Remise fixe sur le panier** | Réduit le total du panier d'un montant fixe | `$5` de réduction sur la commande |
| **Remise fixe par produit** | Réduit les lignes de produits correspondants d'un montant fixe par unité | `$2` de réduction par T-shirt |

Les trois types respectent les mêmes règles de validation (expiration, limites d'utilisation, restrictions de produits/catégories, etc.).

## Créer des coupons {#creating-coupons}

Les coupons sont créés dans WooCommerce, pas dans le PDV. Accédez à **WP Admin → Marketing → Coupons → Ajouter un coupon** :

1. **Code du coupon** — ce que les caissiers saisissent en caisse. Les codes ne sont pas sensibles à la casse.
2. **Description** — note interne. WCPOS utilise ce champ comme libellé de remise sur les reçus lorsqu'il est renseigné ; préférez donc une formulation adaptée au client, comme « Remise responsable 10 % », plutôt qu'un code interne.
3. **Type de remise** — pourcentage, montant fixe sur le panier ou montant fixe sur le produit.
4. **Montant du coupon** — la valeur de la remise.
5. **Date d'expiration** — facultative. Au-delà de cette date, le coupon sera refusé.

### Restriction d'utilisation {#usage-restriction}

- **Montant minimum / maximum** — le coupon ne s'applique que si le sous-total est supérieur ou inférieur au seuil défini.
- **Utilisation individuelle uniquement** — empêche ce coupon d'être combiné avec tout autre coupon.
- **Exclure les articles en promotion** — ignore les articles déjà en promotion (ainsi que toute ligne dont le prix a été abaissé en caisse par un caissier — voir [Modifications de prix au point de vente](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)).
- **Produits / Exclure des produits** — restreint le coupon à des produits spécifiques ou les en exclut.
- **Catégories de produits / Exclure des catégories** — restriction par catégorie. Les produits divers ou personnalisés créés au point de vente respectent également les catégories attribuées via le point de vente.
- **Adresses e-mail autorisées** — limite le coupon à des adresses e-mail de clients spécifiques (les caractères génériques `*` sont pris en charge).

### Limites d'utilisation {#usage-limits}

- **Limite d'utilisation par coupon** — nombre total de fois où ce code peut être utilisé, tous clients confondus.
- **Limite d'utilisation par utilisateur** — nombre de fois qu'un même client peut l'utiliser.
- **Limiter l'utilisation à X articles** — pour les coupons à produit fixe, limite le nombre d'articles éligibles bénéficiant de la remise par utilisation.

### Conseils pour l'utilisation en point de vente {#tips-for-pos-use}

- Créez un coupon **« Responsable 10 % »** (10 % de remise, sans montant minimum d'achat, sans date d'expiration, restriction d'utilisation unique désactivée) et communiquez le code aux responsables — le personnel pourra l'appliquer en caisse pour des ajustements ponctuels, et vous conserverez une remise traçable dans les rapports.
- Créez un coupon **« Fidélité 5 $ »** pour récompenser les clients réguliers.
- Pour les promotions ponctuelles, définissez une date d'expiration courte afin que le code ne puisse pas être réutilisé par erreur.
- Renseigner le champ **Description** avec la phrase que vous souhaitez imprimer sur les reçus est le moyen le plus rapide de personnaliser vos remises.

## Appliquer un coupon en caisse {#applying-a-coupon-at-the-register}

Dans le panneau du panier :

1. Cliquez sur **Ajouter un coupon**
2. Saisir le code du coupon
3. Appuyer sur **Entrée**

Le PDV valide le coupon instantanément à partir des données synchronisées — sans aller-retour avec le serveur — et applique la remise au panier. Si le contenu du panier est modifié par la suite, la remise est recalculée automatiquement.

Pour retirer un coupon appliqué, cliquer sur le × à côté de celui-ci dans le panier.

## Fonctionnement de la validation {#how-validation-works}

Lorsqu'un code est saisi, le PDV vérifie toutes les règles que WooCommerce vérifierait sur le serveur :

- Le coupon existe et n'est pas expiré
- Les limites d'utilisation ne sont pas dépassées (globales et par utilisateur)
- Le montant minimum/maximum est atteint
- Les coupons à « utilisation individuelle » ne sont pas en conflit avec les coupons déjà appliqués
- Les restrictions par produit/catégorie correspondent au contenu du panier
- Les restrictions par e-mail correspondent au client sélectionné (ou sont ignorées pour les commandes invité)
- Les articles en promotion sont exclus si **Exclure les articles en promotion** est activé

Si une vérification échoue, le caissier voit un message d'erreur spécifique (par ex., « Ce coupon a expiré » ou « Montant minimum non atteint »). Les mêmes vérifications sont exécutées à nouveau sur le serveur lors de la soumission de la commande — la validation locale sert à la rapidité, pas à la sécurité.

## Comportement de synchronisation {#sync-behaviour}

Les coupons se synchronisent depuis WooCommerce vers l'appareil via le pipeline de réplication standard de WCPOS :

- Les nouveaux coupons créés dans WP Admin apparaissent dans le PDV lors de la prochaine synchronisation.
- Les modifications apportées aux coupons existants (nombre d'utilisations, changements d'expiration, etc.) se synchronisent automatiquement.
- Les coupons sont stockés localement dans IndexedDB afin de rester disponibles hors connexion.

Si un coupon vient d'être créé dans WP Admin et n'apparaît pas encore, la synchronisation n'a peut-être pas encore eu lieu. Depuis le pied de page de l'écran Coupons, appuyez sur l'icône de synchronisation (<Icon name="arrow-rotate-right" />) pour actualiser — appuyez longuement pour **Effacer et actualiser** si une récupération complète est nécessaire.

## Connectivité {#connectivity}

- **L'application de coupons** fonctionne hors connexion car la validation s'effectue côté client à partir des données synchronisées.
- **La finalisation de la commande** nécessite toujours une connexion au serveur — le serveur revalide les données et constitue la source de référence pour le montant total final de la commande.
- **La création ou la modification de coupons** s'effectue dans WP Admin et nécessite une connexion à votre site WordPress.

## Interaction entre les coupons et les remises PDV {#how-coupons-interact-with-pos-discounts}

Lorsqu'un caissier réduit le prix d'un article en caisse et qu'un coupon est ensuite appliqué, le coupon est calculé sur le **prix réduit**, et non sur le prix d'origine. Les articles dont le prix a été modifié au PDV sont traités comme « en promotion » — les coupons avec l'option **Exclure les articles en promotion** activée les ignoreront. Consultez la section [Remises](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons) pour en savoir plus.
