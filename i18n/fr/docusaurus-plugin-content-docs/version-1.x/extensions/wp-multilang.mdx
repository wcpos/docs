# WCPOS WP Multilang {#wcpos-wp-multilang}

Intègre [WP Multilang](https://wordpress.org/plugins/wp-multilang/) à WCPOS afin que le POS ne serve les produits que pour une seule langue. Contrairement à Polylang ou WPML, WP Multilang stocke toutes les traductions dans le même article — ce plugin garantit que WCPOS lit la traduction correcte et ne duplique pas les entrées dans les résultats de recherche.

## Fonctionnement {#what-it-does}

- Filtre les réponses REST des produits et variations WCPOS pour une seule langue WP Multilang.
- Intercepte les routes **fast-sync** de WCPOS afin que seul le contenu de la langue active parvienne au client.
- Les installations gratuites utilisent la langue par défaut de WP Multilang.
- Les installations Pro peuvent fixer une langue par boutique depuis l'éditeur de boutique.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Install WP Multilang">

Installez [WP Multilang](https://wordpress.org/plugins/wp-multilang/) et configurez les langues de votre site, avec au moins une langue définie comme langue par défaut.

  </StepCard>
  <StepCard number="2" title="Install WCPOS WP Multilang">

Installer depuis `WP Admin > POS > Settings > Extensions`, ou télécharger la dernière version depuis la [page des versions GitHub](https://github.com/wcpos/wcpos-wp-multilang/releases) et envoyer via `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="(Pro) Pin a language per store" isLast>

Sur WCPOS Pro, modifier une boutique sous `POS > Stores` et choisir sa langue dans la section **Langue** de la barre latérale. Laisser sur *Par défaut* pour utiliser la langue par défaut de WP Multilang.

  </StepCard>
</Steps>
