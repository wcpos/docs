# WCPOS Polylang {#wcpos-polylang}

Ajoute la prise en charge de [Polylang](https://polylang.pro/) à WCPOS afin que le POS n'affiche que les produits d'une seule langue — sans doublons de traductions dans la recherche de produits, la grille du catalogue ou les flux de caisse. Les boutiques WCPOS Pro peuvent fixer une langue par boutique ; les installations gratuites utilisent par défaut la langue par défaut de Polylang.

## Fonctionnement {#what-it-does}

- Filtre les requêtes REST de produits et de variations WCPOS par langue.
- Intercepte les routes **fast-sync** de WCPOS (les requêtes légères `posts_per_page=-1` + `fields` que le POS utilise pour rafraîchir son index local) afin que les doublons traduits n'atteignent jamais le client.
- Pour les installations gratuites, applique la langue par défaut de Polylang.
- Pour les installations Pro, chaque boutique peut choisir sa propre langue depuis une nouvelle section **Langue** dans l'éditeur de boutique.
- Respecte la visibilité des produits **POS uniquement** de WCPOS lors de la construction de la charge utile de synchronisation rapide.

L'intégration reste inactive proprement lorsque Polylang n'est pas activé — vous pouvez installer l'extension avant d'activer Polylang sans erreur.

## Installation {#installation}

<Steps>
  <StepCard number="1" title="Install Polylang">

Installez [Polylang](https://wordpress.org/plugins/polylang/) (ou Polylang Pro) et configurez les langues de votre site comme d'habitude. Assurez-vous qu'au moins une langue est définie par défaut.

  </StepCard>
  <StepCard number="2" title="Install WCPOS Polylang">

Installez depuis le répertoire des extensions WCPOS à l'adresse `WP Admin > POS > Settings > Extensions`, ou téléchargez la dernière version depuis la [page des versions GitHub](https://github.com/wcpos/wcpos-polylang/releases) et téléversez via `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="(Pro) Set a per-store language" isLast>

Si vous gérez [plusieurs boutiques](/stores) sur WCPOS Pro, accédez à `POS > Stores`, modifiez une boutique et choisissez sa langue dans la section **Language** de la barre latérale. Laissez-la sur *Default* pour utiliser la langue par défaut de Polylang.

  </StepCard>
</Steps>

## Langue par boutique (Pro) {#per-store-language-pro}

Sur WCPOS Pro, l'extension ajoute une section **Language** à la barre latérale de l'éditeur de boutique. Chaque boutique peut être associée à un identifiant de langue Polylang unique — les produits servis à cette boutique sont alors filtrés selon cette langue uniquement. Les boutiques laissées sur *Default* utilisent la langue par défaut de Polylang.

La valeur propre à chaque boutique est enregistrée sur le post de la boutique en tant que `_wcpos_polylang_language` meta et est exposée via l'API REST des boutiques de WCPOS Pro (`/wcpos/v1/stores`), ce qui lui permet de circuler dans le POS comme n'importe quel autre paramètre de boutique.

## Notes de compatibilité {#compatibility-notes}

- **Produits réservés au POS :** lorsque le mode POS-only est activé dans les paramètres WCPOS, les identifiants des produits réservés à la boutique en ligne sont exclus de la charge utile de synchronisation rapide afin qu'ils ne soient pas exposés dans le POS.
- **Installations gratuites :** il n'existe pas d'interface pour changer la langue par boutique — l'extension utilise la langue par défaut de Polylang. Si vous avez besoin d'une langue par boutique, passez à [WCPOS Pro](/getting-started/pro-license).
- **Extension indisponible :** si Polylang est désactivé, l'extension ne fait rien en silence. Elle ne génère pas d'erreurs et ne bloque pas le POS.

## Hooks pour développeurs {#developer-hooks}

Pour des cas d'usage avancés, l'extension expose quelques filtres :

| Filtre | Rôle |
| --- | --- |
| `wcpos_polylang_resolved_language` | Remplace la langue utilisée pour une requête donnée. Reçoit le slug résolu ainsi que le `WP_REST_Request`. |
| `wcpos_polylang_default_language` | Remplace la langue de repli lorsqu'aucune valeur par boutique n'est définie. |
| `wcpos_polylang_is_supported` | Force l'activation ou la désactivation de l'extension, indépendamment de la détection de Polylang. |
| `wcpos_polylang_minimum_version` | Exige une version minimale de Polylang (par défaut : aucune contrainte de version). |
