WCPOS prend en charge des extensions qui ajoutent de nouvelles fonctionnalités à votre point de vente. Le répertoire d'extensions vous permet de parcourir les extensions disponibles, de les installer directement depuis les réglages du POS et de gérer les mises à jour.

:::info Fonctionnalité Pro
L'installation et la gestion des extensions nécessitent [WCPOS Pro](/getting-started/pro-license). La version gratuite affiche le catalogue d'extensions mais désactive les commandes d'installation et d'activation.
:::

## Parcourir les extensions {#browsing-extensions}

Ouvrez le répertoire des extensions depuis `POS Settings > Extensions` (également intitulé **Plugins** dans certaines versions).

Le répertoire affiche une grille de cartes des extensions disponibles. Chaque carte présente :

- **Icône** (ou une pièce de puzzle par défaut si l'extension n'en fournit pas)
- **Nom et version**
- **Description**
- **Badge de catégorie**
- **Statut** — active, inactive, mise à jour disponible ou non installée

### Filtrage et recherche {#filtering-and-search}

Utilisez les **boutons de catégorie** en haut pour filtrer les extensions par catégorie. Vous pouvez également utiliser le **champ de recherche** pour trouver des extensions par nom, description ou étiquettes.

## Installation d'une extension {#installing-an-extension}

1. Ouvrez `POS Settings > Extensions`.
2. Trouvez l'extension souhaitée et cliquez sur **Installer**.
3. L'extension est téléchargée et installée à l'aide du programme d'installation des extensions de WordPress.
4. Une fois installée, cliquez sur **Activer** pour l'activer.

En arrière-plan, WCPOS utilise le `Plugin_Upgrader` natif de WordPress pour gérer l'installation, de sorte que les extensions suivent le même processus que toute extension WordPress.

## Activation et désactivation {#activating-and-deactivating}

Chaque extension installée dispose de boutons **Activer** et **Désactiver** sur sa fiche.

- **Activer** active l'extension afin qu'elle puisse fonctionner dans le POS.
- **Désactiver** la désactive sans la désinstaller. Les fichiers de l'extension restent sur votre serveur et peuvent être réactivés à tout moment.

## Mise à jour des extensions {#updating-extensions}

Lorsqu'une nouvelle version d'une extension installée est disponible, la carte affiche un badge **Mise à jour disponible** et un bouton **Mettre à jour**.

### Mises à jour automatiques {#auto-updates}

Les extensions installées depuis le répertoire ont les **mises à jour automatiques activées par défaut**. Vous pouvez activer ou désactiver les mises à jour automatiques pour chaque extension depuis sa carte dans le répertoire. Lorsque les mises à jour automatiques sont activées, WordPress applique les nouvelles versions automatiquement, comme il le fait pour les extensions dont les mises à jour automatiques sont activées.

## Dépannage {#troubleshooting}

### Message « Pro requis » sur les boutons {#requires-pro-message-on-buttons}

Les boutons d'installation, d'activation et de mise à jour sont désactivés dans la version gratuite de WCPOS. Passez à [WCPOS Pro](/getting-started/pro-license) pour gérer les extensions.

### Échec de l'installation d'une extension {#extension-fails-to-install}

- Vérifiez que votre serveur WordPress dispose des autorisations d'écriture sur le répertoire `wp-content/plugins`.
- Vérifiez que votre serveur peut effectuer des requêtes HTTPS sortantes (certains hébergeurs bloquent les téléchargements externes).
- Consultez les détails de l'erreur dans `WP Admin > POS > Support > Logs`.

### L'extension n'apparaît pas après l'installation {#extension-not-appearing-after-install}

- Actualisez le POS — la liste des extensions est mise en cache jusqu'à 12 heures.
- Vérifiez que l'extension est activée (les extensions installées mais inactives ne s'exécutent pas).

### Le catalogue ne se charge pas {#catalog-not-loading}

Le catalogue d'extensions est récupéré depuis une source distante et mis en cache localement pendant 12 heures. Si le catalogue ne se charge pas :

- Vérifiez la connectivité Internet de votre serveur.
- Réessayez après l'expiration du cache, ou videz le cache transient de votre serveur.

## Pour les développeurs {#for-developers}

### Créer une extension POS {#creating-a-pos-extension}

Une extension WCPOS est un plugin WordPress standard qui s'intègre au POS via les hooks et les API WCPOS. Pour en créer une :

1. **Commencez par une extension WordPress.** Votre extension a besoin d'un en-tête de plugin standard et d'un fichier d'entrée, comme n'importe quelle extension WooCommerce ou WordPress.
2. **Intégrez-la à WCPOS.** Utilisez les hooks et filtres fournis par WCPOS pour ajouter des fonctionnalités à l'interface POS ou au backend.
3. **Hébergez les versions sur GitHub.** Le répertoire d'extensions utilise GitHub Releases pour suivre les versions et diffuser les mises à jour.

### Soumission au répertoire {#submitting-to-the-directory}

Le catalogue d'extensions est maintenu dans le dépôt GitHub [`wcpos/extensions`](https://github.com/wcpos/extensions). Pour référencer votre extension :

1. Consultez le format du catalogue et les exigences de métadonnées dans le README du dépôt.
2. Ouvrez une pull request pour ajouter les métadonnées de votre extension à `catalog.json`.
3. Une fois la fusion effectuée, votre extension apparaîtra dans le répertoire pour tous les utilisateurs de WCPOS Pro.

### Conventions des versions GitHub {#github-release-conventions}

Le cycle de vie des mises à jour repose sur GitHub Releases :

- **Étiquetez les versions** en utilisant le versionnement sémantique (par exemple, `v1.0.0`, `v1.2.3`).
- **Joignez le zip de l'extension** comme ressource de la version — c'est le fichier qui est téléchargé lorsqu'un utilisateur l'installe ou la met à jour.
- **Publiez la version** (et non un brouillon) afin que le répertoire puisse la détecter.

Lorsque vous publiez une nouvelle version, les utilisateurs ayant installé votre extension verront la mise à jour disponible dans leur répertoire d'extensions. Si la mise à jour automatique est activée, elle sera appliquée automatiquement.

Pour tous les détails sur le schéma du catalogue et le processus de soumission, consultez le dépôt [`wcpos/extensions`](https://github.com/wcpos/extensions).
