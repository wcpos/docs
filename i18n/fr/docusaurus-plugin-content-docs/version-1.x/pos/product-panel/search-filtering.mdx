## Recherche de Produits

### Champ de Recherche Unifié

WCPOS dispose d’un seul champ de recherche qui explore simultanément plusieurs attributs de produit :

- **Nom du Produit** - Recherche dans le titre et la description du produit
- **SKU** - Correspond au champ postmeta `_sku` du produit
- **Code-barres** - Recherche dans le champ de code-barres configuré, qui peut être n'importe quel champ postmeta. Le paramètre hérité est `_sku`, mais cela sera changé en `_global_unique_id` à l’avenir.

Il suffit de taper votre terme de recherche dans le champ "Rechercher des Produits", et le POS filtrera instantanément les résultats dans tous ces champs.

### Technologie de Recherche Tokenisée

La fonctionnalité de recherche utilise la [bibliothèque FlexSearch](https://github.com/nextapps-de/flexsearch) avec des capacités de tokenisation avancées :

- **Tokenisation Forward** - Correspond aux mots partiels dès le début (par exemple, rechercher "bleu" trouvera "myrtille")
- **Optimisé pour la Performance** - Utilise un préréglage de performance pour des résultats de recherche rapides
- **Sensible à la Langue** - S'adapte à la langue configurée de votre magasin
- **Initialisation Paresseuse** - Optimise l'utilisation de la mémoire en chargeant les index de recherche uniquement lorsque cela est nécessaire

### Comment Fonctionne la Recherche

Lorsque vous tapez dans le champ de recherche, le POS :

1. **Tokenise** votre saisie en termes recherchables
2. **Recherche Localement** les données produit stockées en premier pour des résultats instantanés
3. **Interroge le Serveur** si aucune correspondance locale n'est trouvée, puis télécharge et stocke de nouveaux produits pour de futures recherches
4. **Met à Jour les Résultats** en temps réel pendant que vous tapez

Cette approche garantit une performance de recherche rapide tout en construisant progressivement une base de données locale de produits complète.

## Filtrage de Produits

### Barre de Filtrage

Sous le champ de recherche, vous trouverez des filtres interactifs et des menus déroulants qui vous permettent de restreindre les produits par critères spécifiques.

### Filtres Disponibles

#### Statut de Stock
Filtrer les produits en fonction de leur statut d'inventaire :
- **En Stock** - Produits avec un inventaire disponible
- **Rupture de Stock** - Produits avec zéro inventaire
- **En Réapprovisionnement** - Produits disponibles en réapprovisionnement

#### Produits en Vedette
Basculez pour n'afficher que les produits marqués comme "En Vedette" dans votre magasin WooCommerce.

#### Produits en Solde  
Filtrer pour afficher uniquement les produits actuellement en solde ou avec des remises actives.

#### Catégorie
Utilisez le menu déroulant de catégorie pour filtrer les produits par leurs catégories de produits assignées. Cela vous aide à trouver rapidement des produits dans des départements ou des gammes de produits spécifiques.

#### Étiquette
Filtrer par étiquettes de produit pour trouver des articles avec des attributs ou des caractéristiques spécifiques que vous avez définis dans votre magasin WooCommerce.

### Utilisation des Filtres

- **Basculez les Filtres** - Cliquez sur n'importe quel bouton de filtre pour l'activer (les filtres actifs apparaissent en surbrillance)
- **Filtres Multiples** - Vous pouvez combiner plusieurs filtres pour restreindre encore votre recherche
- **Effacer les Filtres** - Cliquez à nouveau sur un filtre actif pour le désactiver
- **Recherche + Filtre** - Utilisez les filtres avec le champ de recherche pour une localisation précise des produits

## Configuration des Codes-Barres

### Champs de Recherche

La fonctionnalité de recherche inclut automatiquement votre champ de code-barres configuré. Le champ de code-barres utilisé pour la recherche dépend de la configuration de vos paramètres POS.

## F.A.Q.

<details>

<summary>
  À quoi sert le champ _global_unique_id pour les codes-barres ?
</summary>

Le champ `_global_unique_id` est un nouveau champ de code-barres que WooCommerce a récemment ajouté pour fournir une meilleure standardisation des codes-barres entre les magasins.

**Points Clés :**

- **Norme Moderne** : Ce champ a été conçu spécifiquement pour l'identification globale des codes-barres
- **Configuration POS** : Vous pouvez configurer le POS pour utiliser `_global_unique_id` comme champ de code-barres dans les paramètres du POS
- **Hérité vs. Nouveau** : Le paramètre de code-barres hérité utilise le champ `_sku`, mais cela changera en `_global_unique_id` dans les futures versions
- **Flexibilité** : Vous pouvez configurer n'importe quel champ méta produit comme votre champ de code-barres si vous utilisez des plugins de code-barres tiers
- **Par Défaut Futur** : `_global_unique_id` deviendra le champ de code-barres par défaut dans les futures mises à jour du POS

Pour configurer quel champ le POS utilise pour les codes-barres, rendez-vous dans vos paramètres POS dans l'interface admin de WordPress.

</details>

<details>

<summary>
  Pourquoi ne vois-je pas tous mes produits lorsque je recherche ?
</summary>

WCPOS utilise le téléchargement progressif des produits pour maintenir la performance. Si vous ne voyez pas un produit :

1. **Essayez de le Rechercher** - Cela déclenchera le POS pour le rechercher sur votre serveur
2. **Attendez le Téléchargement** - Le POS téléchargera le produit et d'autres par petits lots
3. **Recherchez à Nouveau** - Une fois téléchargé, le produit apparaîtra dans les recherches futures

Ce processus garantit que votre POS reste rapide et réactif même avec des milliers de produits. Découvrez-en plus sur cela dans notre guide [Synchronisation des Produits](/products/sync).

</details>

<details>

<summary>
  Puis-je rechercher des noms de produits ou des SKU partiels ?
</summary>

Oui ! La recherche tokenisée utilise une correspondance forward, ce qui signifie :

- Rechercher "bleu" trouvera des produits avec "myrtille", "chemise bleue", etc.
- Rechercher "ABC" trouvera des SKU comme "ABC123", "ABC-XYZ", etc.
- Vous n'avez pas besoin de taper des mots ou des codes complets

La recherche est conçue pour trouver rapidement des produits avec un minimum de saisie.

</details>
