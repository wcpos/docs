Lorsque vous êtes prêt à finaliser une vente, cliquez sur le bouton **Passage en caisse** pour ouvrir la fenêtre modale de paiement. C'est ici que vous traitez le paiement et terminez la commande.

{/* Screenshot: checkout-modal.png */}

## Aperçu de la fenêtre de passage en caisse

La fenêtre modale affiche :

- **Numéro de commande** - L'identifiant de commande WooCommerce
- **Montant à payer** - Montant total dû
- **Bouton Paramètres de paiement** - Résoudre les problèmes d'affichage
- **Caissier** - Qui traite la commande
- **Client** - Le client de cette commande (lien cliquable)
- **Champ de code de coupon** - Appliquer des codes de réduction
- **Résumé de la commande** - Produits, quantités et totaux
- **Modes de paiement** - Options de paiement disponibles
- **Boutons Annuler / Traiter le paiement**

## Modes de paiement

### Disponibles dans la version gratuite

La version gratuite de WCPOS inclut deux passerelles de paiement :

- **Espèces** - Avec calcul du montant versé et du rendu
- **Carte** - Pour les terminaux de carte externes

### Passerelles supplémentaires (Pro)

:::info Fonctionnalité Pro
Des passerelles de paiement supplémentaires nécessitent [WCPOS Pro](/getting-started/pro-license).
:::

Avec Pro, vous pouvez activer :

- **Stripe Terminal** - Intégration directe avec les lecteurs de carte Stripe
- **Lecteur SumUp** - Intégration avec les lecteurs de carte SumUp
- **Passerelles personnalisées** - Créer vos propres intégrations de paiement

Consultez [Paiement](/payment) pour plus de détails sur la configuration des passerelles.

### Sélection d'un mode de paiement

Cliquez sur un mode de paiement pour le sélectionner. Le formulaire se met à jour pour afficher les champs pertinents :

**Espèces :**
- **Montant versé** - Entrez le montant donné par le client
- **Rendu** - Calcul automatique du rendu à restituer

{/* Screenshot: checkout-cash-payment.png */}

**Carte :**
- Traiter le paiement sur votre terminal de carte externe
- Cliquez sur Traiter le paiement pour terminer

## Codes de coupon

Pour appliquer une réduction :

1. Entrez le code de coupon dans le champ
2. Cliquez sur **Appliquer le coupon**
3. La réduction apparaît dans le résumé de la commande

Les coupons doivent être configurés dans WooCommerce (`WP Admin > Marketing > Coupons`).

## Traitement du paiement

1. Sélectionnez un mode de paiement
2. Pour les espèces, entrez le montant versé
3. Cliquez sur **Traiter le paiement**
4. La commande est terminée et le [reçu](/pos/checkout/receipts) est affiché

## Paramètres de paiement (Dépannage)

La fenêtre modale de paiement utilise un iframe/webview pour afficher la page de paiement de commande WooCommerce. Cela s'appuie sur l'infrastructure de paiement existante de WooCommerce, ce qui signifie que toute passerelle de paiement fonctionnant avec WooCommerce devrait fonctionner dans le POS.

Cependant, les scripts de thème et de plugin peuvent parfois interférer. Cliquez sur **Paramètres de paiement** pour résoudre les problèmes :

{/* Screenshot: checkout-settings.png */}

### Désactiver tous les styles et scripts

- **Désactiver wp_head** - Supprime tous les scripts/styles de l'en-tête WordPress
- **Désactiver wp_footer** - Supprime tous les scripts/styles du pied de page WordPress

### Désactiver les styles sélectionnés

Désactivez sélectivement les CSS pouvant causer des problèmes d'affichage :

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Désactiver les scripts sélectionnés

Désactivez sélectivement les scripts JavaScript pouvant interférer avec les passerelles de paiement :

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Si une passerelle de paiement ne s'affiche pas correctement :
1. Essayez de désactiver d'abord les styles de thème
2. Ensuite, essayez de désactiver les scripts WooCommerce non nécessaires
3. Faites attention à ne pas désactiver les scripts requis par votre passerelle de paiement
:::

## Annuler

Cliquez sur **Annuler** pour fermer la fenêtre modale de paiement sans terminer la commande. La commande reste un panier ouvert.
