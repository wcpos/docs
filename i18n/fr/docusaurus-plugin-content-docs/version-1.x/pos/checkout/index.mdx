Lorsque la vente est prête à être finalisée, cliquez sur le bouton **Paiement** pour ouvrir la fenêtre modale de paiement. C’est ici que le paiement est traité et que la commande est finalisée.

{/* Screenshot: checkout-modal.png */}

## Vue d’ensemble de la fenêtre modale de paiement {#checkout-modal-overview}

La fenêtre modale de paiement affiche :

- **Numéro de commande** - L’ID de commande WooCommerce
- **Montant à payer** - Montant total dû
- Bouton **Réglages du paiement** - Résoudre les problèmes d’affichage
- **Caissier** - Personne qui traite la commande
- **Client** - Le client associé à cette commande (lien cliquable)
- Bouton **Ajouter un code promo** - Appliquer des codes de réduction
- **Récapitulatif de la commande** - Produits, quantités et totaux
- **Moyens de paiement** - Options de paiement disponibles
- Boutons **Annuler / Traiter le paiement**

## Moyens de paiement {#payment-methods}

### Disponible dans la version gratuite {#available-in-free-version}

La version gratuite de WCPOS inclut deux passerelles de paiement :

- **Espèces** - Avec montant reçu et calcul de la monnaie à rendre
- **Carte** - Pour les terminaux de carte externes

### Passerelles supplémentaires (Pro) {#additional-gateways-pro}

:::info Fonctionnalité Pro
Les passerelles de paiement supplémentaires nécessitent [WCPOS Pro](/getting-started/pro-license).
:::

Avec Pro, vous pouvez activer :

- **Stripe Terminal** - Intégration directe avec les lecteurs de cartes Stripe
- **SumUp Terminal** - Intégration avec les lecteurs de cartes SumUp
- **Passerelles personnalisées** - Créez vos propres intégrations de paiement

Consultez [Paiement](/payment) pour plus de détails sur la configuration des passerelles.

### Sélection d’un mode de paiement {#selecting-a-payment-method}

Cliquez sur un mode de paiement pour le sélectionner. Le formulaire se met à jour pour afficher les champs pertinents :

**Espèces :**

- **Montant remis** - Saisissez le montant donné par le client
- **Monnaie** - Monnaie à rendre calculée automatiquement

{/* Screenshot: checkout-cash-payment.png */}

**Carte :**

- Traitez le paiement sur votre terminal de carte externe
- Cliquez sur Traiter le paiement pour terminer

## Codes promo {#coupons}

Le panier inclut un champ **Ajouter un code promo** au-dessus des totaux (Pro uniquement). Saisissez le code ou effectuez une recherche par description ; le code promo est validé localement et apparaît sous forme de pastille amovible. Plusieurs codes promo se cumulent séquentiellement.

Consultez **[Application de codes promo à la caisse](/coupons/applying-coupons)** pour le flux de travail complet : recherche, pastilles, remises séquentielles, ainsi que le tableau des erreurs de validation et de leurs résolutions. Pour les types de codes promo, les règles de validation et la configuration dans WooCommerce, consultez [Codes promo](/coupons).

## Traitement du paiement {#processing-payment}

1. Sélectionnez un moyen de paiement
2. Pour les espèces, saisissez le montant remis
3. Cliquez sur **Traiter le paiement**
4. La commande est terminée et le [reçu](/receipts/at-checkout) s’affiche

## Réglages du paiement (dépannage) {#checkout-settings-troubleshooting}

La fenêtre modale de paiement utilise une iframe/webview pour afficher la page de paiement de commande WooCommerce. Elle s’appuie sur l’infrastructure de paiement existante de WooCommerce, ce qui signifie que toute passerelle de paiement compatible avec WooCommerce devrait fonctionner dans le POS.

Toutefois, les scripts de thèmes et d’extensions peuvent parfois interférer. Cliquez sur **Réglages du paiement** pour résoudre le problème :

### Désactiver tous les styles et scripts {#disable-all-styles-and-scripts}

:::warning Option radicale
Il s’agit d’une option radicale qui ne doit être utilisée que pour des tests ou dans de rares cas où le développeur sait ce qu’il fait. La désactivation de tous les scripts wp_head supprimera même les scripts WooCommerce nécessaires pour développer/réduire les passerelles de paiement, ce qui peut compromettre la fonctionnalité de paiement.
:::

- **Désactiver wp_head** - Supprime tous les scripts/styles de l’en-tête WordPress
- **Désactiver wp_footer** - Supprime tous les scripts/styles du pied de page WordPress

### Désactiver les styles sélectionnés {#disable-selected-styles}

Désactivez de manière sélective les CSS susceptibles de provoquer des problèmes d’affichage :

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Désactiver les scripts sélectionnés {#disable-selected-scripts}

Désactivez de manière sélective le JavaScript susceptible d’interférer avec les passerelles de paiement :

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Si une passerelle de paiement ne s’affiche pas correctement :

1. Essayez d’abord de désactiver les styles du thème
2. Essayez ensuite de désactiver les scripts WooCommerce qui ne sont pas nécessaires
3. Veillez à ne pas désactiver les scripts requis par votre passerelle de paiement
   :::

## Annuler {#cancel}

Cliquez sur **Annuler** pour fermer la fenêtre modale de validation de commande sans terminer la commande. La commande reste sous forme de panier ouvert.
