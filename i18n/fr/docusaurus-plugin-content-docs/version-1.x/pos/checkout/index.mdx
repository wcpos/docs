Lorsque vous êtes prêt à finaliser une vente, cliquez sur le bouton **Checkout** pour ouvrir la fenêtre modale de checkout. C'est ici que vous traitez le paiement et finalisez la commande.

{/* Screenshot: checkout-modal.png */}

## Vue d'ensemble de la fenêtre modale de Checkout {#checkout-modal-overview}

La fenêtre modale de checkout affiche :

- **Numéro de commande** - L'ID de commande WooCommerce
- **Montant à payer** - Montant total dû
- Bouton **Paramètres de Checkout** - Résoudre les problèmes d'affichage
- **Caissier** - Qui traite la commande
- **Client** - Le client pour cette commande (lien cliquable)
- Champ **Codes de coupon** - Appliquer les codes de réduction
- **Résumé de la commande** - Produits, quantités et totaux
- **Méthodes de paiement** - Options de paiement disponibles
- Boutons **Annuler / Process Payment**

## Méthodes de paiement {#payment-methods}

### Disponibles dans la version gratuite {#available-in-free-version}

La version gratuite de WCPOS comprend deux passerelles de paiement :

- **Espèces** - Avec montant donné et calculateur de monnaie
- **Carte** - Pour les terminaux de carte externes

### Passerelles supplémentaires (Pro) {#additional-gateways-pro}

:::info Fonction Pro
Des passerelles de paiement supplémentaires nécessitent [WCPOS Pro](/getting-started/pro-license).
:::

Avec Pro, vous pouvez activer :

- **Terminal Stripe** - Intégration directe avec les lecteurs de carte Stripe
- **Terminal SumUp** - Intégration avec les lecteurs de carte SumUp
- **Passerelles personnalisées** - Créez vos propres intégrations de paiement

Voir [Payment](/payment) pour plus de détails sur la configuration des passerelles.

### Sélection d'une méthode de paiement {#selecting-a-payment-method}

Cliquez sur une méthode de paiement pour la sélectionner. Le formulaire se met à jour pour montrer les champs pertinents :

**Espèces :**

- **Montant donné** - Entrez le montant que le client vous donne
- **Monnaie** - Monnaie à rendre calculée automatiquement

{/* Screenshot: checkout-cash-payment.png */}

**Carte :**

- Traitez le paiement sur votre terminal de carte externe
- Cliquez sur Process Payment pour finaliser

## Codes de coupon {#coupon-codes}

Pour appliquer une réduction :

1. Entrez le code de coupon dans le champ
2. Cliquez sur **Apply coupon**
3. La réduction apparaît dans le résumé de la commande

Les coupons doivent être configurés dans WooCommerce (`WP Admin > Marketing > Coupons`).

## Traitement du paiement {#processing-payment}

1. Sélectionnez une méthode de paiement
2. Pour les espèces, entrez le montant donné
3. Cliquez sur **Process Payment**
4. La commande est complétée et le [reçu](/pos/checkout/receipts) est affiché

## Paramètres de Checkout (Dépannage) {#checkout-settings-troubleshooting}

La fenêtre modale de checkout utilise un iframe/webview pour afficher la page de paiement de commande WooCommerce. Cela tire parti de l'infrastructure de paiement existante de WooCommerce, ce qui signifie que toute passerelle de paiement qui fonctionne avec WooCommerce devrait fonctionner dans le POS.

Cependant, les scripts de thème et de plugin peuvent parfois interférer. Cliquez sur **Paramètres de Checkout** pour résoudre les problèmes :

### Désactiver tous les styles et scripts {#disable-all-styles-and-scripts}

:::warning Option nucléaire
Ceci est l'option nucléaire et ne doit être utilisé que pour des tests ou dans de rares cas où le développeur sait ce qu'il fait. Désactiver tous les scripts wp_head supprimera même les scripts WooCommerce nécessaires pour étendre/contracter les passerelles de paiement, ce qui pourrait perturber la fonctionnalité de paiement.
:::

- **Désactiver wp_head** - Supprime tous les scripts/styles de l'en-tête de WordPress
- **Désactiver wp_footer** - Supprime tous les scripts/styles du pied de page de WordPress

### Désactiver des styles sélectionnés {#disable-selected-styles}

Désactiver sélectivement le CSS qui peut causer des problèmes d'affichage :

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Désactiver des scripts sélectionnés {#disable-selected-scripts}

Désactiver sélectivement le JavaScript qui peut interférer avec les passerelles de paiement :

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Si une passerelle de paiement ne s'affiche pas correctement :

1. Essayez d'abord de désactiver les styles du thème
2. Ensuite, essayez de désactiver les scripts WooCommerce qui ne sont pas nécessaires
3. Faites attention à ne pas désactiver les scripts requis par votre passerelle de paiement
   :::

## Annuler {#cancel}

Cliquez sur **Annuler** pour fermer la fenêtre modale de checkout sans finaliser la commande. La commande reste en tant que panier ouvert.
