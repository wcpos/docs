Lorsque vous êtes prêt à finaliser une vente, cliquez sur le bouton **Caisse** pour ouvrir la fenêtre de caisse. C'est ici que vous traitez le paiement et finalisez la commande.

{/* Screenshot: checkout-modal.png */}

## Aperçu de la fenêtre de caisse {#checkout-modal-overview}

La fenêtre de caisse affiche :

- **Numéro de commande** – L'identifiant de commande WooCommerce
- **Montant à payer** – Montant total dû
- Bouton **Paramètres de caisse** – Résoudre les problèmes d'affichage
- **Caissier** – La personne qui traite la commande
- **Client** – Le client de cette commande (lien cliquable)
- Bouton **Ajouter un coupon** - Appliquer des codes de réduction
- **Récapitulatif de commande** - Produits, quantités et totaux
- **Modes de paiement** - Options de paiement disponibles
- Boutons **Annuler / Traiter le paiement**

## Modes de paiement {#payment-methods}

### Disponible dans la version gratuite {#available-in-free-version}

La version gratuite de WCPOS inclut deux passerelles de paiement :

- **Espèces** - Avec montant remis et calcul de la monnaie
- **Carte** - Pour les terminaux de carte externes

### Passerelles supplémentaires (Pro) {#additional-gateways-pro}

:::info Fonctionnalité Pro
Les passerelles de paiement supplémentaires nécessitent [WCPOS Pro](/getting-started/pro-license).
:::

Avec Pro, vous pouvez activer :

- **Stripe Terminal** - Intégration directe avec les lecteurs de cartes Stripe
- **SumUp Terminal** - Intégration avec les lecteurs de cartes SumUp
- **Passerelles personnalisées** - Créez vos propres intégrations de paiement

Consultez [Paiement](/payment) pour plus de détails sur la configuration des passerelles.

### Sélection d'un mode de paiement {#selecting-a-payment-method}

Cliquez sur un mode de paiement pour le sélectionner. Le formulaire se met à jour pour afficher les champs correspondants :

**Espèces :**

- **Montant remis** - Saisissez le montant donné par le client
- **Monnaie à rendre** - Calcul automatique de la monnaie à remettre

{/* Screenshot: checkout-cash-payment.png */}

**Carte :**

- Traitez le paiement sur votre terminal de carte externe
- Cliquez sur Traiter le paiement pour finaliser

## Coupons {#coupon-codes}

Le panier comprend un champ **Ajouter un coupon** au-dessus des totaux (Pro uniquement). Saisissez le code ou recherchez par description ; le coupon est validé localement et apparaît sous forme de pastille supprimable. Plusieurs coupons s'appliquent séquentiellement.

Consultez **[Appliquer des coupons en caisse](/coupons/applying-coupons)** pour le processus complet — recherche, pastilles, remises séquentielles et tableau des erreurs de validation et résolutions. Pour les types de coupons, les règles de validation et la configuration dans WooCommerce, consultez [Coupons](/coupons).

## Traitement du paiement {#processing-payment}

1. Sélectionnez un moyen de paiement
2. Pour les espèces, saisissez le montant remis
3. Cliquez sur **Traiter le paiement**
4. La commande est finalisée et le [reçu](/receipts/at-checkout) s'affiche

## Paramètres de passage en caisse (Dépannage) {#checkout-settings-troubleshooting}

La fenêtre de paiement utilise un iframe/webview pour afficher la page de paiement de commande WooCommerce. Cela s'appuie sur l'infrastructure de paiement existante de WooCommerce, ce qui signifie que toute passerelle de paiement compatible avec WooCommerce devrait fonctionner dans le PDV.

Cependant, les scripts de thèmes et d'extensions peuvent parfois interférer. Cliquez sur **Paramètres de paiement** pour résoudre les problèmes :

### Désactiver tous les styles et scripts {#disable-all-styles-and-scripts}

:::warning Option radicale
Il s'agit de l'option radicale, à utiliser uniquement pour les tests ou dans de rares cas où le développeur sait ce qu'il fait. Désactiver tous les scripts wp_head supprimera même les scripts WooCommerce nécessaires pour déplier/replier les passerelles de paiement, ce qui risque de compromettre la fonctionnalité de paiement.
:::

- **Désactiver wp_head** – Supprime tous les scripts/styles de l'en-tête WordPress
- **Désactiver wp_footer** – Supprime tous les scripts/styles du pied de page WordPress

### Désactiver les styles sélectionnés {#disable-selected-styles}

Désactivez sélectivement les CSS susceptibles de provoquer des problèmes d'affichage :

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Désactiver les scripts sélectionnés {#disable-selected-scripts}

Désactiver sélectivement le JavaScript susceptible d'interférer avec les passerelles de paiement :

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Si une passerelle de paiement ne s'affiche pas correctement :

1. Essayez d'abord de désactiver les styles du thème
2. Ensuite, essayez de désactiver les scripts WooCommerce qui ne sont pas nécessaires
3. Veillez à ne pas désactiver les scripts requis par votre passerelle de paiement
   :::

## Annuler {#cancel}

Cliquez sur **Annuler** pour fermer la fenêtre de paiement sans finaliser la commande. La commande reste sous forme de panier ouvert.
