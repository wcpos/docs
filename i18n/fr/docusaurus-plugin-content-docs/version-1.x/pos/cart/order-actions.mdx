## Note de Commande

<div class="image-container">
  <Image
    alt="Note de Commande dans le POS"
    img="/img/order-note.png"
    style={{maxHeight: 500}}
  />
  <p class="image-caption">Modal de Note de Commande dans le POS (à gauche) et Note de Commande dans le WP Admin (à droite)</p>

- Ouvre une boîte de dialogue pour ajouter une note client à la commande.
- Les notes clients sont visibles par le client et apparaissent par défaut sur le reçu, dans la zone Mon Compte du client, et dans le `WP Admin > WooCommerce > Commandes > Commande` sous l'adresse de livraison.

:::note
Les notes de commande sont visibles par les clients.
:::

## Métadonnées de Commande

<div class="image-container">
  <Image

    style={{maxHeight: 500}}
  />
  <p class="image-caption">Métadonnées de Commande et Vue JSON dans le POS</p>

- Ouvre une modal pour gérer des métadonnées supplémentaires pour la commande.
- Les fonctionnalités incluent :
  - Changer la devise de la commande.
  - Ajouter un ID de transaction (par exemple, d'un terminal de paiement externe).
  - Ajouter des données méta personnalisées pour la commande.

:::note
Des fonctionnalités supplémentaires seront ajoutées à cette modal au fil du temps, y compris l'intégration avec d'autres plugins WooCommerce.
:::

### Qu'est-ce que les Métadonnées dans WooCommerce ?

Les métadonnées fournissent des informations supplémentaires sur les commandes et sont accessibles via l'API REST de WooCommerce. Les développeurs et les plugins utilisent ces données pour étendre les fonctionnalités, comme l'intégration de services tiers.

### Vue JSON

- Affiche la représentation JSON brute de la commande, utile pour le débogage ou les besoins d'intégration.

## Sauvegarder sur le Serveur

- Vous permet de sauvegarder les paniers ouverts sur le serveur, afin qu'ils soient stockés de manière sécurisée et accessibles pour une utilisation future.
- Crée une commande WooCommerce avec le statut `pos-open`.

### Pourquoi sauvegarder ?

Lorsqu'une nouvelle commande est créée dans le POS, elle n'existe que localement. 
Sauvegarder la commande garantit qu'elle persiste, même si la base de données locale est effacée.

Voir [Commandes Ouvertes](/pos/cart/open-orders) pour plus de détails sur la gestion des commandes sauvegardées.

## Annuler

- Deux Scénarios :
  - Commandes Non Sauvegardées :
    - Si la commande n'a pas été sauvegardée sur le serveur, elle sera définitivement supprimée de la base de données locale.
  - Commandes Sauvegardées :
    - Si la commande est sauvegardée sur le serveur, elle sera déplacée dans le dossier Corbeille dans WooCommerce et supprimée de la base de données locale.
- Pour récupérer une commande annulée :
  - Allez dans `WP Admin > WooCommerce > Commandes > Corbeille`.

## Paiement

- Sauvegarde la commande sur le serveur et lance le processus de paiement.
- Ouvre l'[écran de paiement](/pos/checkout) pour compléter le paiement.
