## Note de commande {#order-note}

<div class="image-container">
  <Image
    alt="Note de commande dans le POS"
    img="/img/order-note.png"
    style={{maxHeight: 500}}
  />

- Ouvre une boîte de dialogue permettant d’ajouter une note client à la commande.
- Les notes client sont visibles par le client et apparaissent par défaut sur le reçu, dans l’espace Mon compte du client, ainsi que dans `WP Admin > WooCommerce > Orders > Order`, sous l’adresse de livraison.

:::note
Les notes de commande sont visibles par les clients.
:::

## Métadonnées de commande {#order-meta}

  <p class="image-caption">Métadonnées de commande et vue JSON dans le POS</p>

- Ouvre une fenêtre modale pour gérer les métadonnées supplémentaires de la commande.
- Les fonctionnalités comprennent :
  - La modification de la devise de la commande.
  - L’ajout d’un ID de transaction (par exemple, depuis un terminal de paiement externe).
  - L’ajout de métadonnées personnalisées pour la commande.

:::note
Des fonctionnalités supplémentaires seront ajoutées à cette fenêtre modale au fil du temps, notamment l’intégration avec d’autres extensions WooCommerce.
:::

### Que sont les métadonnées dans WooCommerce ? {#what-is-meta-data-in-woocommerce}

Les métadonnées fournissent des informations supplémentaires sur les commandes et sont accessibles via l’API REST WooCommerce. Les développeurs et les extensions utilisent ces données pour étendre les fonctionnalités, par exemple pour intégrer des services tiers.

### Vue JSON {#json-view}

- Affichez la représentation JSON brute de la commande, utile à des fins de débogage ou d’intégration.

## Enregistrer sur le serveur {#save-to-server}

- Permet d’enregistrer les paniers ouverts sur le serveur, afin qu’ils soient stockés en toute sécurité et accessibles pour une utilisation ultérieure.
- Crée une commande WooCommerce avec le statut `pos-open`.

### Pourquoi enregistrer ? {#why-save}

Lorsqu’une nouvelle commande est créée dans le POS, elle n’existe que localement. 
L’enregistrement de la commande garantit sa conservation, même si la base de données locale est effacée.

Consultez [Commandes ouvertes](/pos/cart/open-orders) pour plus de détails sur la gestion des commandes enregistrées.

## Annuler {#void}

- Deux scénarios :
  - Commandes non enregistrées :
    - Si la commande n’a pas été enregistrée sur le serveur, elle sera définitivement supprimée de la base de données locale.
  - Commandes enregistrées :
    - Si la commande est enregistrée sur le serveur, elle sera déplacée vers le dossier Corbeille dans WooCommerce et supprimée de la base de données locale.
- Pour récupérer une commande annulée :
  - Accédez à `WP Admin > WooCommerce > Orders > Trash`.

## Paiement {#checkout}

- Enregistre la commande sur le serveur et lance le processus de paiement.
- Ouvre l’[écran de paiement](/pos/checkout) pour finaliser le paiement.
