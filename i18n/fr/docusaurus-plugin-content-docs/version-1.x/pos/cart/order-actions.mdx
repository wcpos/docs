## Note de Commande

<div class="image-container">
  <Image
    alt="Note de Commande dans le POS"

    style={{maxHeight: 500}}

  <p class="image-caption">Modale de Note de Commande dans le POS (à gauche) et Note de Commande dans l'Admin WP (à droite)</p>

- Ouvre un dialogue pour ajouter une note client à la commande.
- Les notes client sont visibles par le client et apparaissent par défaut sur le ticket de caisse, dans la zone Mon Compte du client et dans `WP Admin > WooCommerce > Commandes > Commande` sous l'adresse de livraison.

:::note
Les notes de commande sont visibles par les clients.
:::

## Méta de Commande

- Ouvre une modale pour gérer des métadonnées supplémentaires pour la commande.
- Fonctionnalités incluent :
  - Changer la devise de la commande.
  - Ajouter un ID de transaction (par exemple, depuis un terminal de paiement externe).
  - Ajouter des métadonnées personnalisées pour la commande.

:::note
Des fonctionnalités supplémentaires seront ajoutées à cette modale au fil du temps, y compris l'intégration avec d'autres plugins WooCommerce.
:::

### Qu'est-ce que sont les Métadonnées dans WooCommerce ?

Les métadonnées fournissent des informations supplémentaires sur les commandes et sont accessibles via l'API REST de WooCommerce. Les développeurs et les plugins utilisent ces données pour étendre les fonctionnalités, comme l'intégration de services tiers.

### Vue JSON

- Affiche la représentation JSON brute de la commande, utile pour le débogage ou l'intégration.

## Enregistrer sur le Serveur

- Permet d'enregistrer des paniers ouverts sur le serveur, afin qu'ils soient stockés en toute sécurité et accessibles pour une utilisation ultérieure.
- Crée une commande WooCommerce avec le statut `pos-open`.

### Pourquoi enregistrer ?

Lorsqu'une nouvelle commande est créée dans le POS, elle n'existe que localement.
L'enregistrement de la commande garantit sa persistance, même si la base de données locale est effacée.

Consultez [Commandes Ouvertes](/pos/cart/open-orders) pour plus de détails sur la gestion des commandes enregistrées.

## Annuler

- Deux Scénarios :
  - Commandes Non Enregistrées :
    - Si la commande n'a pas été enregistrée sur le serveur, elle sera définitivement supprimée de la base de données locale.
  - Commandes Enregistrées :
    - Si la commande est enregistrée sur le serveur, elle sera déplacée dans le dossier Corbeille dans WooCommerce et supprimée de la base de données locale.
- Pour récupérer une commande annulée :
  - Allez à `WP Admin > WooCommerce > Commandes > Corbeille`.

## Paiement

- Enregistre la commande sur le serveur et lance le processus de paiement.
- Ouvre l'[écran de paiement](/pos/checkout) pour terminer le paiement.
