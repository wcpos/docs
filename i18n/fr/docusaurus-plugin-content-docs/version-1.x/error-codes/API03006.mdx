# API03006 : Méthode Non Prise en Charge

## Ce Que Cela Signifie

La méthode HTTP utilisée (GET, POST, PUT, DELETE, etc.) n'est pas prise en charge pour ce point de terminaison. Le serveur n'accepte pas ce type de requête pour cette URL.

## Causes Courantes

- **Incompatibilité de version** — Les versions de POS et du plugin sont incompatibles
- **Point de terminaison supprimé** — Un point de terminaison d'API a été abandonné
- **Configuration du serveur** — Serveur web bloquant certaines méthodes HTTP
- **Conflit de plugin** — Un autre plugin modifiant le comportement de l'API REST

## Comment Résoudre

### 1. Mettre à Jour les Deux Composants

Assurer la compatibilité :
- Mettre à jour le plugin WordPress WCPOS
- Mettre à jour l'application POS
- Vérifier les notes de version concernant les changements majeurs

### 2. Vérifier la Configuration du Serveur

Certains serveurs bloquent certaines méthodes HTTP :
- S'assurer que les méthodes PUT et DELETE sont autorisées
- Vérifier les restrictions de méthode dans `.htaccess`
- Examiner la configuration Nginx

### 3. Vérifier l'Accès à l'API REST

Tester l'API REST WordPress :
1. Visiter `https://votresite.com/wp-json/` dans un navigateur
2. Cela devrait renvoyer des données JSON
3. Si ce n'est pas le cas, l'API REST peut être désactivée ou bloquée

### 4. Vérifier les Conflits de Plugins

Désactiver temporairement d'autres plugins :
- Les plugins de sécurité peuvent bloquer des méthodes
- D'autres plugins d'API REST peuvent causer des conflits
- Réactiver un par un pour identifier le problème

### 5. Examiner les Restrictions de l'Hébergement

Certains hébergeurs restreignent les méthodes HTTP :
- Contacter le support de l'hébergeur
- Demander l'activation de toutes les méthodes standard
- Envisager de changer d'hébergeur si trop restrictif
