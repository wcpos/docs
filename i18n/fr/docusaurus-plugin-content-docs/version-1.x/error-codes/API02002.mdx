# API02002 : Jeton Expiré

## Ce Que Cela Signifie

Votre session d'authentification a expiré. Pour des raisons de sécurité, les sessions de connexion ne durent pas éternellement. Le POS doit actualiser votre authentification ou vous devrez peut-être vous reconnecter.

## Causes Courantes

- **Longue inactivité** — Vous n'avez pas utilisé le POS pendant une période prolongée
- **Durée de vie du jeton du serveur** — L'expiration du jeton sur le serveur est définie très court
- **Désynchronisation de l'heure du serveur** — Les horloges du serveur et de l'appareil ne sont pas synchronisées

## Mapping de l'Erreur Serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code Serveur | Source |
|-------------|--------|
| `jwt_auth_expired_token` | Plugin d'Authentification JWT |

## Comment Résoudre

### 1. Se Reconnecter

La solution la plus simple est de se déconnecter et se reconnecter :
1. Se déconnecter du POS
2. Saisir à nouveau vos identifiants
3. Cela crée une nouvelle session

### 2. Vérifier l'Actualisation du Jeton

Le POS devrait automatiquement actualiser les jetons. S'il ne fonctionne pas :
- Vérifier votre connexion internet
- Confirmer l'accessibilité du serveur
- Consulter [API02007](./API02007) si l'actualisation échoue

### 3. Vérifier l'Heure du Serveur

L'expiration du jeton dépend d'un temps précis :
- Assurez-vous que l'horloge de votre serveur est correcte
- Vérifiez que l'heure de votre appareil est précise
- Utilisez la synchronisation automatique de l'heure sur les deux

### 4. Examiner les Paramètres du Jeton

Si les jetons expirent trop rapidement, vous devrez peut-être ajuster les paramètres du serveur (consultez votre hébergeur ou l'administrateur WordPress).
