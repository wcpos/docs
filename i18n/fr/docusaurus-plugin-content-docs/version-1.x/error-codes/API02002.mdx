# API02002 : Token Expiré

## Ce que cela signifie

Votre session d'authentification a expiré. Pour des raisons de sécurité, les sessions de connexion ne durent pas éternellement. Le POS doit actualiser votre authentification ou vous devrez peut-être vous reconnecter à nouveau.

## Causes courantes

- **Longue inactivité** — Vous n'avez pas utilisé le POS pendant une période étendue
- **Durée de vie du token serveur** — L'expiration du token du serveur est fixée très courte
- **Désynchronisation de l'heure du serveur** — Les horloges du serveur et de l'appareil ne sont pas synchronisées

## Mappage des erreurs du serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code Serveur | Source |
|--------------|--------|
| `jwt_auth_expired_token` | Plugin d'authentification JWT |

## Comment corriger

### 1. Se connecter de nouveau

La solution la plus simple est de se déconnecter et de se reconnecter :
1. Déconnectez-vous du POS
2. Entrez à nouveau vos identifiants
3. Cela crée une nouvelle session

### 2. Vérifier le rafraîchissement du token

Le POS devrait actualiser automatiquement les tokens. Si cela ne fonctionne pas :
- Vérifiez votre connexion Internet
- Vérifiez que le serveur est accessible
- Consultez [API02007](./API02007) si le rafraîchissement échoue

### 3. Vérifier l'heure du serveur

L'expiration du token dépend d'une heure précise :
- Assurez-vous que l'horloge de votre serveur est correcte
- Vérifiez que l'heure de votre appareil est précise
- Utilisez la synchronisation horaire automatique sur les deux

### 4. Vérifier les paramètres du token

Si les tokens expirent trop rapidement, vous devrez peut-être ajuster les paramètres du serveur (consultez votre fournisseur d'hébergement ou l'administrateur WordPress).
