# DB03001 : Erreur de syntaxe de requête

## Ce que cela signifie

La requête de base de données présente des erreurs de syntaxe. Cela indique généralement un bug dans l'application plutôt qu'une erreur utilisateur.

## Causes courantes

- **Bug logiciel** — Un problème dans l'application POS
- **Incompatibilité de version** — Schéma de données incompatible
- **Données corrompues** — Les données contiennent des caractères inattendus
- **Problèmes d'encodage** — Problèmes d'encodage de caractères

## Comment réparer

### 1. Mettre à jour l'application

Assurez-vous d'utiliser la dernière version :
- Vérifiez les mises à jour de l'application POS
- Mettez à jour le plugin WordPress WCPOS
- Redémarrez après la mise à jour

### 2. Effacer les données de l'application

Réinitialisez à un état propre :
- Effacez les données/local cache
- Connectez-vous à nouveau
- Resynchronisez les données

### 3. Vérifier les caractères spéciaux

Si l'erreur se produit avec des données spécifiques :
- Vérifiez les caractères inhabituels dans les noms de produits
- Recherchez des emojis ou des symboles spéciaux
- Essayez avec des données plus simples

### 4. Signaler le bug

Si le problème persiste :
1. Notez quelle action cause l'erreur
2. Capturez les détails de l'erreur
3. Signalez sur [GitHub](https://github.com/wcpos)

## Il s'agit généralement d'un bug

Les erreurs de syntaxe de requête sont généralement des bugs dans le logiciel, pas des erreurs d'utilisateur. Si vous rencontrez cela :
- Vous ne faites rien de mal
- L'équipe de développement doit le savoir
- Un correctif sera publié dans une mise à jour

## Contournement

Jusqu'à ce qu'un correctif soit disponible :
- Essayez d'autres approches pour accomplir votre tâche
- Utilisez l'administration WooCommerce pour les opérations affectées
- Vérifiez régulièrement les mises à jour
