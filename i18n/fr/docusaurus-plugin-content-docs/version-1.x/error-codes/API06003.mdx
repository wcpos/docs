# API06003 : Configuration du site invalide

## Ce que cela signifie

La configuration du site est invalide ou incomplète. Cela pourrait impliquer des URL incorrectes, des paramètres d'authentification ou d'autres problèmes de configuration.

## Causes courantes

- **Configuration partielle** — Certains paramètres sont manquants
- **Paramètres non correspondants** — La configuration ne correspond pas au site
- **Configuration corrompue** — Les paramètres ont été corrompus
- **Modifications du site** — Le site a été modifié sans mettre à jour la configuration du POS

## Comment corriger

### 1. Vérifiez tous les paramètres

Vérifiez la configuration complète :
- L'URL du site est correcte
- L'authentification est correctement configurée
- Tous les paramètres supplémentaires sont corrects

### 2. Reconfigurer depuis le début

Si la configuration est corrompue :
1. Effacez tous les paramètres enregistrés
2. Redémarrez le processus de configuration
3. Entrez une nouvelle configuration

### 3. Vérifiez les exigences du site

Vérifiez que votre site WordPress répond aux exigences :
- WordPress est installé et accessible
- WooCommerce est installé et actif
- Le plugin WCPOS est installé et actif
- Les permaliens sont activés (pas "Simple")

### 4. Testez l'accès au site

Vérifiez que ces URL fonctionnent dans un navigateur :
- `https://votresite.com/` — Site principal
- `https://votresite.com/wp-json/` — REST API
- `https://votresite.com/wp-json/wcpos/v1/` — API WCPOS

### 5. Vérifiez les paramètres WordPress

Dans l'administration WordPress :
1. Paramètres → Général — Vérifiez les URL
2. Paramètres → Permaliens — Assurez-vous que ce n'est pas "Simple"
3. WooCommerce → Statut — Vérifiez les problèmes

## Exigences de configuration

Une configuration valide nécessite :
- **URL du site** — URL complète avec protocole
- **Authentification** — Identifiants valides ou clés API
- **Permissions** — L'utilisateur a accès au POS
- **Plugins** — Les plugins requis sont actifs
