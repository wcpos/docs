# API06003 : Configuration de site non valide

## Ce que cela signifie

La configuration du site est non valide ou incomplète. Cela peut concerner des URL incorrectes, des paramètres d'authentification ou d'autres problèmes de configuration.

## Causes courantes

- **Configuration partielle** — Certains paramètres sont manquants
- **Paramètres incompatibles** — La configuration ne correspond pas au site
- **Configuration corrompue** — Les paramètres ont été corrompus
- **Modifications du site** — Le site a été modifié sans mettre à jour la configuration POS

## Comment résoudre

### 1. Examiner tous les paramètres

Vérifiez la configuration complète :
- L'URL du site est correcte
- L'authentification est correctement configurée
- Tous les paramètres supplémentaires sont corrects

### 2. Reconfigurer à partir de zéro

Si la configuration est corrompue :
1. Effacez tous les paramètres enregistrés
2. Recommencez le processus de configuration
3. Entrez une nouvelle configuration

### 3. Vérifier les exigences du site

Vérifiez que votre site WordPress répond aux exigences :
- WordPress est installé et accessible
- WooCommerce est installé et actif
- Le plugin WCPOS est installé et actif
- Les permaliens sont activés (pas "Simple")

### 4. Tester l'accès au site

Vérifiez que ces URL fonctionnent dans un navigateur :
- `https://votresite.com/` — Site principal
- `https://votresite.com/wp-json/` — API REST
- `https://votresite.com/wp-json/wcpos/v1/` — API WCPOS

### 5. Vérifier les paramètres WordPress

Dans l'administration WordPress :
1. Paramètres → Général — Vérifier les URL
2. Paramètres → Permaliens — S'assurer que ce n'est pas "Simple"
3. WooCommerce → État — Vérifier les problèmes

## Exigences de configuration

Une configuration valide nécessite :
- **URL du site** — URL complète avec protocole
- **Authentification** — Identifiants ou clés API valides
- **Autorisations** — L'utilisateur a accès au POS
- **Plugins** — Plugins requis actifs
