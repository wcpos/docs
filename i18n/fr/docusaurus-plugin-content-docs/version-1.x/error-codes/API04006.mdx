# API04006 : Ressource non trouvée

## Ce que cela signifie

Le serveur n'a pas pu trouver la ressource demandée. Cela correspond généralement à un code de statut HTTP 404, indiquant que le produit, la commande, le client ou une autre ressource que vous essayez d'accéder n'existe pas.

## Causes courantes

- **Enregistrement supprimé** — La ressource a été supprimée sur le serveur
- **ID incorrect** — Un ID incorrect ou obsolète est utilisé
- **Problèmes de synchronisation** — Les données locales font référence à une ressource qui n'existe plus
- **Mauvaise configuration d'URL** — Le point de terminaison de l'API est incorrect
- **Problèmes de permaliens** — Les permaliens WordPress doivent être actualisés

## Mappage des erreurs serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code serveur | Source |
|-------------|--------|
| `rest_no_route` | API REST WordPress |
| HTTP 404 | Toute réponse du serveur |

## Comment résoudre

### 1. Vérifier l'existence de la ressource

Vérifiez si la ressource existe toujours sur votre site WooCommerce :
- Connectez-vous à l'administration WordPress
- Accédez à la section pertinente (Produits, Commandes, Clients)
- Recherchez l'élément par ID ou nom

### 2. Actualiser les données locales

Si la ressource a été supprimée côté serveur :
1. Ouvrez les paramètres du POS
2. Accédez à la section de données pertinente
3. Déclenchez une synchronisation/actualisation pour mettre à jour les données locales
4. L'élément supprimé devrait être retiré localement

### 3. Vérifier les permaliens WordPress

Si plusieurs ressources sont introuvables :
1. Allez dans **Réglages → Permaliens** dans l'administration WordPress
2. Cliquez sur **Enregistrer les modifications** (même sans faire de changements)
3. Cela actualise la structure des permaliens

### 4. Vérifier les routes de l'API

Testez directement l'API REST :
```
https://yoursite.com/wp-json/wc/v3/products
```
Si cela renvoie une erreur 404, il peut y avoir un problème de configuration du serveur.

### 5. Vérifier les conflits de plugins

Si les routes de l'API REST sont manquantes :
1. Assurez-vous que WooCommerce est actif
2. Assurez-vous que le plugin WCPOS est actif
3. Désactivez temporairement d'autres plugins pour tester
