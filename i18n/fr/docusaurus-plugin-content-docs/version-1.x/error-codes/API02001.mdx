# API02001 : Identifiants Invalides

## Ce que Cela Signifie

Le nom d'utilisateur ou le mot de passe que vous avez saisi est incorrect. Le serveur a rejeté la tentative de connexion car les identifiants ne correspondent à aucun compte valide.

## Causes Courantes

- **Faute de frappe dans le nom d'utilisateur ou le mot de passe** — Vérifiez le verrouillage des majuscules ou les espaces supplémentaires
- **Mauvais compte** — Utilisation d'identifiants pour un site différent
- **Mot de passe récemment modifié** — Le mot de passe a été mis à jour mais le POS conserve l'ancien
- **Compte supprimé** — Le compte utilisateur n'existe plus

## Mappage des Erreurs Serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code Serveur | Source |
|-------------|--------|
| `woocommerce_rest_authentication_error` | API REST WooCommerce |
| `jwt_auth_failed` | Plugin d'Authentification JWT |

## Comment Résoudre

### 1. Vérifier Vos Identifiants

- Vérifiez votre nom d'utilisateur (généralement e-mail ou nom d'utilisateur WordPress)
- Assurez-vous que le mot de passe est correct
- Essayez de vous connecter directement à l'administration WordPress pour confirmer

### 2. Réinitialiser Votre Mot de Passe

Si vous avez oublié votre mot de passe :
1. Allez sur la page de connexion WordPress
2. Cliquez sur « Mot de passe perdu ? »
3. Saisissez votre e-mail pour recevoir un lien de réinitialisation
4. Mettez à jour votre mot de passe
5. Connectez-vous au POS avec le nouveau mot de passe

### 3. Vérifier le Compte Utilisateur

Dans l'Administration WordPress :
- Allez dans Utilisateurs → Tous les utilisateurs
- Vérifiez que le compte existe
- Assurez-vous qu'il dispose des autorisations d'accès POS appropriées

### 4. Effacer les Identifiants Enregistrés

Dans le POS :
- Déconnectez-vous complètement
- Effacez toutes les informations de connexion enregistrées ou en cache
- Ressaisissez les identifiants
