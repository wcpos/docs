# API02001: Identifiants invalides

## Ce que cela signifie

Le nom d'utilisateur ou le mot de passe que vous avez saisi est incorrect. Le serveur a rejeté la tentative de connexion car les identifiants ne correspondent à aucun compte valide.

## Causes courantes

- **Erreur de saisie dans le nom d'utilisateur ou le mot de passe** — Vérifiez si les majuscules sont activées ou s'il y a des espaces supplémentaires
- **Mauvais compte** — Utilisation d'identifiants pour un site différent
- **Mot de passe récemment changé** — Le mot de passe a été mis à jour mais le POS a l'ancien
- **Compte supprimé** — Le compte utilisateur n'existe plus

## Cartographie des erreurs du serveur

Ce code d'erreur est déclenché lorsque le serveur retourne :

| Code Serveur | Source |
|--------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | Plugin d'authentification JWT |

## Comment réparer

### 1. Vérifiez vos identifiants

- Vérifiez votre nom d'utilisateur (généralement email ou nom d'utilisateur WordPress)
- Vérifiez que le mot de passe est correct
- Essayez de vous connecter directement à l'administration WordPress pour confirmer

### 2. Réinitialisez votre mot de passe

Si vous avez oublié votre mot de passe :
1. Allez sur votre page de connexion WordPress
2. Cliquez sur "Mot de passe oublié ?"
3. Saisissez votre email pour recevoir un lien de réinitialisation
4. Mettez à jour votre mot de passe
5. Connectez-vous au POS avec le nouveau mot de passe

### 3. Vérifiez le compte utilisateur

Dans l'administration WordPress :
- Allez dans Utilisateurs → Tous les utilisateurs
- Vérifiez que le compte existe
- Vérifiez qu'il a les autorisations d'accès au POS appropriées

### 4. Effacer les identifiants enregistrés

Dans le POS :
- Déconnectez-vous complètement
- Effacez toutes les informations de connexion enregistrées/cachées
- Saisissez à nouveau les identifiants frais
