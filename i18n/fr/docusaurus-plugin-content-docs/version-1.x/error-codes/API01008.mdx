# API01008 : Site Web Indisponible {#api01008-website-unavailable}

## Ce que cela signifie {#what-this-means}

Le POS peut accéder à Internet, mais votre site web spécifique ne répond pas. Le serveur hébergeant votre boutique WooCommerce semble être hors service ou inaccessible.

## Causes Courantes {#common-causes}

- **Maintenance du serveur** — Votre fournisseur d'hébergement peut effectuer des mises à jour
- **Crash du serveur** — Le serveur web a arrêté de fonctionner de manière inattendue
- **Limites de ressources dépassées** — Votre compte d'hébergement a atteint ses limites
- **Attaque DDoS** — Le serveur est submergé par un trafic malveillant
- **Domaine/hébergement expiré** — Votre hébergement ou domaine a expiré

## Comment Réparer {#how-to-fix}

### 1. Vérifiez que le Site est Indisponible {#1-verify-the-site-is-down}

Essayez d'accéder à votre site directement dans un navigateur. S'il ne se charge pas, le problème vient du serveur.

### 2. Vérifiez l'État du Fournisseur d'Hébergement {#2-check-hosting-provider-status}

- Visitez la page d'état de votre fournisseur d'hébergement
- Vérifiez les maintenances programmées
- Recherchez des pannes signalées

### 3. Examinez le Compte d'Hébergement {#3-review-hosting-account}

Connectez-vous à votre panneau de contrôle d'hébergement :
- Vérifiez si votre compte est actif
- Vérifiez que vous n'avez pas dépassé les limites de ressources
- Recherchez d'éventuelles notifications de suspension

### 4. Contactez le Support d'Hébergement {#4-contact-hosting-support}

Si tout semble normal de votre côté, contactez votre fournisseur d'hébergement. Il peut :
- Vérifier l'état du serveur
- Examiner les journaux du serveur
- Redémarrer les services si nécessaire

### 5. Attendez et Réessayez {#5-wait-and-retry}

S'il s'agit d'une panne temporaire :
- Attendez 15 à 30 minutes
- Le POS tentera automatiquement de se reconnecter
- Les données mises en cache restent disponibles hors ligne
