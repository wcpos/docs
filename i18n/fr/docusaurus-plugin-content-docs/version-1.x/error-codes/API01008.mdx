# API01008 : Site Web Indisponible

## Ce Que Cela Signifie

Le système POS peut se connecter à Internet, mais votre site web spécifique ne répond pas. Le serveur hébergeant votre boutique WooCommerce semble être hors service ou inaccessible.

## Causes Courantes

- **Maintenance du serveur** — Votre hébergeur peut effectuer des mises à jour
- **Plantage du serveur** — Le serveur web s'est arrêté de manière inattendue
- **Limites de ressources dépassées** — Votre compte d'hébergement a atteint ses limites
- **Attaque DDoS** — Le serveur est submergé par du trafic malveillant
- **Expiration du domaine/hébergement** — Votre hébergement ou domaine a expiré

## Comment Résoudre

### 1. Vérifier que le Site est Hors Ligne

Essayez d'accéder à votre site directement dans un navigateur. S'il ne se charge pas, le problème est côté serveur.

### 2. Vérifier le Statut du Fournisseur d'Hébergement

- Consultez la page de statut de votre hébergeur
- Vérifiez s'il y a une maintenance programmée
- Recherchez des pannes signalées

### 3. Examiner le Compte d'Hébergement

Connectez-vous au panneau de contrôle de votre hébergement :
- Vérifiez que votre compte est actif
- Confirmez que vous n'avez pas dépassé les limites de ressources
- Recherchez d'éventuels avis de suspension

### 4. Contacter le Support de l'Hébergement

Si tout semble normal de votre côté, contactez votre fournisseur d'hébergement. Ils peuvent :
- Vérifier le statut du serveur
- Examiner les journaux du serveur
- Redémarrer les services si nécessaire

### 5. Attendre et Réessayer

En cas de panne temporaire :
- Attendez 15-30 minutes
- Le POS réessaiera automatiquement les connexions
- Les données en cache restent disponibles hors ligne
