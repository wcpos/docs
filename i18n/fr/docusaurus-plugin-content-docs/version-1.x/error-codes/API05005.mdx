# API05005 : Plugin non trouvé

## Ce que cela signifie

Un plugin WordPress requis n'est pas installé ou actif. Cela peut concerner WooCommerce lui-même, une extension requise ou un plugin d'intégration.

## Causes courantes

- **WooCommerce désactivé** — WooCommerce a été désactivé
- **Extension requise manquante** — Une extension WooCommerce nécessaire n'est pas installée
- **Conflit de plugins** — Un autre plugin a provoqué la désactivation d'un plugin requis
- **Mise à jour échouée** — Une mise à jour de plugin a échoué et l'a laissé désactivé

## Comment résoudre

### 1. Vérifier que WooCommerce est actif

WCPOS nécessite WooCommerce :
1. Aller à Plugins → Plugins installés
2. Rechercher "WooCommerce"
3. S'assurer qu'il est activé

### 2. Vérifier les extensions requises

Certaines fonctionnalités de WCPOS peuvent nécessiter des extensions :
- Vérifier le message d'erreur pour le nom du plugin spécifique
- Installer et activer l'extension requise

### 3. Examiner les conflits de plugins

Si des plugins ont été désactivés à cause de conflits :
1. Vérifier vos notifications par e-mail de WordPress
2. Consulter les journaux d'erreurs
3. Résoudre les conflits et réactiver

### 4. Vérifier les fichiers du plugin

Si les fichiers d'un plugin sont manquants :
1. Réinstaller le plugin depuis WordPress.org
2. Ou restaurer à partir d'une sauvegarde
3. Activer le plugin

### 5. Mettre à jour tous les plugins

Les plugins obsolètes peuvent causer des problèmes :
1. Aller au Tableau de bord → Mises à jour
2. Mettre à jour tous les plugins
3. Vérifier que tous les plugins requis sont actifs

## Plugins requis

Au minimum, WCPOS nécessite :
- **WooCommerce** — La plateforme de commerce électronique
- **WCPOS** — Le plugin POS

Des fonctionnalités supplémentaires peuvent nécessiter d'autres plugins.
