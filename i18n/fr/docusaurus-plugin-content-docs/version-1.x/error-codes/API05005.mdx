# API05005 : Plugin Non Trouvé

## Ce Que Cela Signifie

Un plugin WordPress requis n'est pas installé ou actif. Cela peut être WooCommerce lui-même, une extension requise ou un plugin d'intégration.

## Causes Courantes

- **WooCommerce désactivé** — WooCommerce a été désactivé
- **Extension requise manquante** — Une extension WooCommerce nécessaire n'est pas installée
- **Conflit de plugin** — Un autre plugin a provoqué la désactivation d'un plugin requis
- **Mise à jour échouée** — Une mise à jour de plugin a échoué et l'a laissé désactivé

## Comment Résoudre

### 1. Vérifier que WooCommerce est Actif

WCPOS nécessite WooCommerce :
1. Allez dans Plugins → Plugins installés
2. Trouvez "WooCommerce"
3. Assurez-vous qu'il est activé

### 2. Vérifier les Extensions Requises

Certaines fonctionnalités de WCPOS peuvent nécessiter des extensions :
- Vérifiez le message d'erreur pour le nom spécifique du plugin
- Installez et activez l'extension requise

### 3. Réviser les Conflits de Plugins

Si des plugins ont été désactivés en raison de conflits :
1. Vérifiez votre e-mail pour les notifications WordPress
2. Consultez les journaux d'erreur
3. Résolvez les conflits et réactivez

### 4. Vérifier les Fichiers du Plugin

Si des fichiers d'un plugin manquent :
1. Réinstallez le plugin depuis WordPress.org
2. Ou restaurez depuis une sauvegarde
3. Activez le plugin

### 5. Mettre à Jour Tous les Plugins

Des plugins obsolètes peuvent causer des problèmes :
1. Allez dans Dashboard → Mises à jour
2. Mettez à jour tous les plugins
3. Vérifiez que tous les plugins requis sont actifs

## Plugins Requis

Au minimum, WCPOS nécessite :
- **WooCommerce** — La plateforme de commerce électronique
- **WCPOS** — Le plugin de point de vente

Des fonctionnalités supplémentaires peuvent nécessiter d'autres plugins.
