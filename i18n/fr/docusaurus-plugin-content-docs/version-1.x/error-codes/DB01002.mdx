# DB01002 : Délai de requête dépassé

## Ce que cela signifie

Une opération de base de données a pris trop de temps pour se terminer. La requête de base de données locale a dépassé le temps maximum autorisé.

## Causes courantes

- **Grand ensemble de données** — Très grand nombre de produits ou d'enregistrements
- **Requête complexe** — Requête impliquant de nombreuses conditions
- **Performance de l'appareil** — L'appareil fonctionne lentement
- **Fragmentation de la base de données** — La base de données nécessite une optimisation

## Comment résoudre

### 1. Attendre et réessayer

La requête peut réussir en réessayant :
- Fermer et rouvrir le POS
- Réessayer l'opération
- Éviter d'autres opérations intensives simultanément

### 2. Fermer d'autres applications

Libérer les ressources de l'appareil :
- Fermer les onglets de navigateur inutilisés
- Fermer les autres applications
- Redémarrer l'appareil si nécessaire

### 3. Réduire la portée des données

Lors de la recherche :
- Utiliser des termes de recherche plus spécifiques
- Appliquer des filtres pour réduire les résultats
- Diviser les grandes opérations en opérations plus petites

### 4. Optimiser la base de données

Effacer et resynchroniser :
1. Effacer les données locales (option de paramètres si disponible)
2. Se reconnecter
3. Laisser les données se synchroniser à nouveau

### 5. Vérifier les performances de l'appareil

S'assurer que votre appareil répond aux exigences :
- RAM suffisante
- Vitesse de processeur adéquate
- SSD préférable à HDD

## Catalogues volumineux

Si vous avez un catalogue de produits très volumineux :
- La synchronisation initiale prend plus de temps
- Envisager d'utiliser des filtres dans le POS
- Certaines opérations prennent naturellement plus de temps
- Contacter le support pour des conseils d'optimisation
