# DB01002 : Délai de Requête

## Ce que Cela Signifie

Une opération de base de données a pris trop de temps à se compléter. La requête de base de données locale a dépassé le temps maximum autorisé.

## Causes Courantes

- **Grand ensemble de données** — Nombre très élevé de produits ou d'enregistrements
- **Requête complexe** — Requête impliquant de nombreuses conditions
- **Performance de l'appareil** — L'appareil fonctionne lentement
- **Fragmentation de la base de données** — La base de données a besoin d'optimisation

## Comment Réparer

### 1. Attendre et Réessayer

La requête peut réussir lors d'une nouvelle tentative :
- Fermez et rouvrir le POS
- Essayez à nouveau l'opération
- Évitez d'autres opérations intensives simultanément

### 2. Fermer D'autres Applications

Libérez les ressources de l'appareil :
- Fermez les onglets de navigateur inutilisés
- Fermez d'autres applications
- Redémarrez l'appareil si nécessaire

### 3. Réduire la Portée des Données

Si vous recherchez :
- Utilisez des termes de recherche plus spécifiques
- Appliquez des filtres pour affiner les résultats
- Divisez les grandes opérations en plus petites

### 4. Optimiser la Base de Données

Effacer et re-synchroniser :
1. Effacez les données locales (option de paramètres si disponible)
2. Connectez-vous à nouveau
3. Laissez les données se synchroniser fraîchement

### 5. Vérifier la Performance de l'Appareil

Assurez-vous que votre appareil répond aux exigences :
- RAM suffisante
- Vitesse de processeur adéquate
- SSD préféré au HDD

## Grands Catalogues

Si vous avez un très grand catalogue de produits :
- La synchronisation initiale prend plus de temps
- Envisagez d'utiliser des filtres dans le POS
- Certaines opérations prennent naturellement plus de temps
- Contactez le support pour des conseils d'optimisation
