# DB02003 : Violation de contrainte

## Que signifie cette erreur

Les données que vous essayez d'enregistrer violent les règles de base de données. Les contraintes garantissent l'intégrité des données en appliquant des règles sur ce qui peut être stocké.

## Causes courantes

- **Données requises manquantes** — Un champ obligatoire est vide
- **Relation invalide** — Référence à un enregistrement inexistant
- **Incompatibilité de type de données** — Mauvais type de données pour le champ
- **Valeur hors limites** — Nombre dépassant les limites autorisées

## Comment résoudre

### 1. Vérifier les champs obligatoires

Assurez-vous que toutes les données requises sont fournies :
- Informations du client (si nécessaire)
- Détails du produit
- Lignes de commande

### 2. Vérifier les références

Si l'erreur concerne des relations :
- Vérifier que les produits référencés existent
- Vérifier la validité des identifiants de clients
- Vérifier les assignations de catégories

### 3. Examiner les valeurs de données

Vérifier les valeurs invalides :
- Quantités négatives non autorisées
- Prix dépassant les limites
- Valeurs de statut invalides

### 4. Synchroniser les dernières données

Les données référencées peuvent être désynchronisées :
- Actualiser les données du serveur
- Attendre la fin de la synchronisation
- Réessayer l'opération

### 5. Effacer et réessayer

Si les données sont corrompues :
1. Effacer le formulaire problématique
2. Ressaisir les données
3. Soumettre à nouveau

## Exemples courants de contraintes

- **La quantité doit être positive** — Impossible d'ajouter 0 ou des éléments négatifs
- **Le prix doit être numérique** — Texte non autorisé dans les champs de prix
- **Le client doit exister** — Impossible d'assigner une commande à un client inexistant
