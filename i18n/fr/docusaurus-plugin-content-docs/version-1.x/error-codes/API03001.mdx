# API03001 : Format de requête invalide

## Ce que cela signifie

La requête envoyée au serveur n'était pas dans le format attendu. Le serveur n'a pas pu comprendre ce que le POS demandait en raison d'une structure de requête incorrecte.

## Causes courantes

- **Données corrompues** — Les données ont été corrompues avant l'envoi
- **Bogue logiciel** — Un problème dans l'application POS
- **Interférence de proxy** — Un proxy ou un pare-feu a modifié la requête
- **Problèmes d'encodage de caractères** — Les caractères spéciaux n'ont pas été encodés correctement

## Mappage des erreurs du serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code du serveur | Source |
|-------------|--------|
| HTTP 400 | Toute réponse du serveur (repli) |

## Comment résoudre

### 1. Réessayer l'action

Parfois, il s'agit d'un problème ponctuel :
- Attendez un moment et réessayez
- Actualisez le POS et réessayez

### 2. Vérifier les caractères spéciaux

Si vous entrez des données avec des caractères spéciaux :
- Essayez de supprimer les emojis ou les symboles inhabituels
- Utilisez des caractères standard pour les noms de produits, etc.

### 3. Mettre à jour le POS

Assurez-vous d'exécuter la dernière version :
- Vérifiez les mises à jour de l'application
- Mettez à jour le plugin WCPOS sur votre serveur

### 4. Vérifier la configuration réseau

Si vous êtes derrière un proxy :
- Vérifiez que le proxy ne modifie pas les requêtes
- Vérifiez les règles du pare-feu
- Essayez d'accéder depuis un autre réseau

### 5. Signaler le problème

Si cela se produit régulièrement :
- Notez l'action qui déclenche l'erreur
- Vérifiez la console du navigateur pour plus de détails (version web)
- Signalez sur [GitHub](https://github.com/wcpos) avec les étapes de reproduction
