# PY02002 : Timeout de paiement {#py02002-payment-timeout}

## Ce que cela signifie {#what-this-means}

Le traitement du paiement a pris trop de temps et a expiré. La passerelle de paiement n'a pas répondu dans le délai attendu.

## Causes courantes {#common-causes}

- **Surcharge de la passerelle** — Le processeur de paiement connaît un volume élevé
- **Latence réseau** — Connexion lente au service de paiement
- **Retard de traitement** — Vérifications ou validations de fraude complexes
- **Problèmes de serveur** — La passerelle de paiement rencontre des problèmes de performance

## Comment résoudre {#how-to-fix}

### 1. Vérifiez si le paiement a réussi {#1-check-if-payment-succeeded}

**Important :** Avant de réessayer, vérifiez le statut du paiement :
- Vérifiez le tableau de bord de votre passerelle de paiement
- Recherchez la transaction dans WooCommerce
- Ne traitez pas à nouveau si elle a réellement réussi

### 2. Réessayez avec précaution {#2-retry-with-caution}

Si le paiement n'a définitivement pas abouti :
- Attendez 30 à 60 secondes
- Essayez à nouveau
- Faites attention aux doubles prélèvements

### 3. Vérifiez votre connexion {#3-check-your-connection}

Assurez-vous d'une connectivité stable :
- Testez votre connexion Internet
- Essayez une connexion câblée si vous êtes sur WiFi
- Vérifiez les problèmes de réseau

### 4. Essayez à un moment moins chargé {#4-try-at-a-less-busy-time}

Si la passerelle est surchargée :
- Les heures de pointe au détail peuvent entraîner des ralentissements
- Réessayez sous peu
- Utilisez une méthode de paiement alternative

### 5. Contactez le fournisseur de paiement {#5-contact-payment-provider}

Si les timeouts persistent :
- Vérifiez leur page d'état
- Contactez leur support
- Signalez les problèmes en cours

## Éviter les doubles prélèvements {#avoiding-duplicate-charges}

Lorsqu'un timeout se produit :
1. **Ne réessayez pas immédiatement** — Vérifiez le statut d'abord
2. **Examinez les transactions** — Vérifiez à la fois la passerelle et WooCommerce
3. **Remboursez les doublons** — Traitez les remboursements pour tout doublon accidentel
4. **Informez le client** — Expliquez la situation s'il y a un retard

## Actions alternatives {#alternative-actions}

Si le paiement continue de rencontrer un timeout :
- Acceptez le paiement en espèces
- Enregistrez comme "payer plus tard" si cela est permis
- Utilisez une méthode de paiement différente
