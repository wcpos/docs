# API03005: Limite de Taux Dépassée

## Que Cela Signifie

Vous avez fait trop de demandes au serveur en peu de temps. La limitation de taux protège le serveur d'une surcharge.

## Causes Courantes

- **Opérations de synchronisation rapides** — Plusieurs appareils se synchronisent simultanément
- **Rafraîchissement agressif** — Rafraîchir les données trop fréquemment
- **Plugin de sécurité** — Un plugin impose des limites de demande
- **Limites d'hébergement** — Votre plan d'hébergement a des limites de taux API

## Mappage des Erreurs Serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code Serveur | Source |
|--------------|--------|
| HTTP 429 | Toute réponse du serveur (secours) |

## Comment Corriger

### 1. Attendre et Réessayer

Les limites de taux sont temporaires :
- Attendez 1 à 5 minutes
- Le POS tentera automatiquement de nouveau
- Évitez de rafraîchir manuellement de manière répétée

### 2. Réduire les Opérations Concurrentes

Si vous utilisez plusieurs terminaux POS :
- Échelonnez la synchronisation initiale entre les appareils
- Évitez de rafraîchir tous les appareils en même temps
- Laissez un se terminer avant d'en commencer un autre

### 3. Vérifier les Plugins de Sécurité

Des plugins comme Wordfence peuvent bloquer les demandes rapides :
- Mettez l'application POS sur liste blanche
- Mettez les adresses IP de votre appareil POS sur liste blanche
- Ajustez les seuils de limitation de taux

### 4. Réviser les Limites d'Hébergement

Certains hébergeurs limitent les demandes API :
- Vérifiez les détails de votre plan d'hébergement
- Envisagez une mise à niveau si les limites sont trop basses
- Contactez le support concernant l'augmentation des limites

### 5. Optimiser la Fréquence de Synchronisation

Dans les paramètres POS :
- Ajustez les intervalles de rafraîchissement automatique
- Utilisez la synchronisation manuelle lorsque c'est approprié
- Évitez les rafraîchissements de données inutiles
