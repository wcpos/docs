# API03005 : Limite de taux dépassée

## Ce que cela signifie

Vous avez effectué trop de requêtes sur le serveur en peu de temps. La limitation du taux protège le serveur contre la surcharge.

## Causes courantes

- **Opérations de synchronisation rapide** — Plusieurs appareils se synchronisant simultanément
- **Actualisation agressive** — Actualisation des données trop fréquemment
- **Plugin de sécurité** — Un plugin impose des limites de requêtes
- **Limites d'hébergement** — Votre plan d'hébergement a des limites de taux d'API

## Mappage des erreurs serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code serveur | Source |
|-------------|--------|
| HTTP 429 | Toute réponse serveur (repli) |

## Comment résoudre

### 1. Attendre et réessayer

Les limites de taux sont temporaires :
- Attendre 1 à 5 minutes
- Le POS réessaiera automatiquement
- Éviter d'actualiser manuellement à plusieurs reprises

### 2. Réduire les opérations concurrentes

Si vous utilisez plusieurs terminaux POS :
- Échelonner la synchronisation initiale sur les appareils
- Éviter d'actualiser tous les appareils en même temps
- Laisser un appareil terminer avant d'en démarrer un autre

### 3. Vérifier les plugins de sécurité

Les plugins comme Wordfence peuvent bloquer les requêtes rapides :
- Autoriser l'application POS
- Autoriser les adresses IP de votre appareil POS
- Ajuster les seuils de limitation de taux

### 4. Examiner les limites d'hébergement

Certains hébergeurs limitent les requêtes API :
- Vérifier les détails de votre plan d'hébergement
- Envisager une mise à niveau si les limites sont trop basses
- Contacter le support pour augmenter les limites

### 5. Optimiser la fréquence de synchronisation

Dans les paramètres POS :
- Ajuster les intervalles d'actualisation automatique
- Utiliser la synchronisation manuelle si nécessaire
- Éviter les actualisations de données superflues
