# API03004 : Requête Trop Volumineuse

## Ce Que Cela Signifie

La requête que vous envoyez dépasse les limites de taille du serveur. Cela se produit généralement lorsque vous essayez d'envoyer trop de données à la fois.

## Causes Courantes

- **Opérations par lots importantes** — Tentative de synchronisation de trop d'enregistrements simultanément
- **Images volumineuses** — Téléchargement d'images trop grandes
- **Nombre d'articles excessif** — Commande avec un nombre de lignes d'articles extrêmement élevé
- **Limites du serveur** — Limites de téléchargement PHP ou du serveur web faibles

## Comment Résoudre

### 1. Réduire la Taille de la Requête

Lors de la synchronisation de données :
- Essayez de synchroniser par petits lots
- Le POS devrait gérer cela automatiquement
- Attendez que la synchronisation en cours soit terminée avant d'en commencer une autre

### 2. Vérifier les Tailles d'Images

Lors du téléchargement d'images :
- Redimensionnez les images avant le téléchargement
- Utilisez des formats compressés (JPEG vs BMP)
- La plupart des images de produits fonctionnent bien sous 1 Mo

### 3. Diviser les Grandes Commandes

Si une commande contient de nombreux articles :
- Envisagez de la diviser en plusieurs commandes
- Cela est rare dans une utilisation normale du POS

### 4. Augmenter les Limites du Serveur

Contactez votre hébergeur ou modifiez les paramètres PHP :

```php
// Dans php.ini ou .htaccess
upload_max_filesize = 64M
post_max_size = 64M
max_input_vars = 5000
```

### 5. Vérifier les Limites du Serveur Web

Nginx ou Apache peuvent avoir leurs propres limites :
- `client_max_body_size` pour Nginx
- `LimitRequestBody` pour Apache
