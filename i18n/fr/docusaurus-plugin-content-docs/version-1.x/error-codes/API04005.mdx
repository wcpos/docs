# API04005 : Tentative de récupération JSON

## Ce que Cela Signifie

Le serveur a envoyé une réponse contenant du contenu JSON invalide, mais le POS a tenté de récupérer et d'extraire des données valides. Il s'agit d'un avis d'information plutôt que d'une erreur critique.

## Ce qui s'est Passé

Le POS a détecté :
1. La réponse contenait du contenu supplémentaire avant ou après le JSON
2. Les données JSON principales étaient toujours identifiables
3. Une récupération a été tentée en extrayant la portion JSON valide

## Causes Courantes

- **Notifications PHP dans la sortie** — Avertissements PHP mélangés avec JSON
- **Sortie de débogage** — Débogage de développement laissé activé
- **Notifications de plugins** — Autres plugins émettant des notifications
- **Problèmes d'espaces blancs** — Espaces supplémentaires autour du JSON

## Comment Résoudre

### 1. Désactiver l'Affichage des Erreurs PHP

Dans `wp-config.php` :
```php
define('WP_DEBUG_DISPLAY', false);
ini_set('display_errors', 0);
```

### 2. Activer la Journalisation des Erreurs à la Place

Conserver les erreurs dans les journaux pour le débogage :
```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
```

### 3. Vérifier le Mode de Débogage des Plugins

Certains plugins ont des modes de débogage qui produisent du contenu supplémentaire :
- Examiner les paramètres des plugins
- Désactiver les modes de développement/débogage
- Vérifier les options de journalisation détaillée

### 4. Examiner les Modifications Récentes

Si ce problème est récent :
- Qu'a-t-on changé sur votre serveur ?
- Des plugins ont-ils été mis à jour ?
- Des paramètres PHP ont-ils été modifiés ?

## Est-ce Grave ?

Bien que le POS ait récupéré de ce problème, cela indique un problème de configuration qui devrait être corrigé. Le processus de récupération :
- Peut ne pas toujours fonctionner
- Ajoute une surcharge de traitement
- Pourrait masquer d'autres problèmes

Corrigez la cause sous-jacente pour garantir un fonctionnement fiable.
