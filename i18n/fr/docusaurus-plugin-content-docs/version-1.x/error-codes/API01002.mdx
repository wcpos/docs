# API01002 : Connexion refusée

## Ce que Cela Signifie

Le serveur a activement refusé la tentative de connexion. Cela diffère d'un délai d'attente — le serveur a répondu, mais a dit « non ».

## Causes Courantes

- **Serveur hors ligne** — Le serveur web (Apache/Nginx) peut ne pas être en cours d'exécution
- **Mauvais port** — Tentative de connexion sur le mauvais port
- **Blocage par pare-feu** — Un pare-feu bloque la connexion
- **Restrictions IP** — Votre adresse IP peut être bloquée

## Comment Résoudre

### 1. Vérifier le Fonctionnement du Serveur

Essayez d'accéder directement à votre site WordPress dans un navigateur. S'il ne se charge pas, contactez votre hébergeur.

### 2. Vérifier la Configuration de l'URL

Assurez-vous d'utiliser l'URL correcte :
- Vérifiez `http://` vs `https://`
- Vérifiez l'absence de fautes de frappe dans le nom de domaine
- Confirmez le port correct si vous utilisez un port non standard

### 3. Vérifier les Paramètres du Pare-feu

Si vous avez accès à votre serveur :
- Vérifiez que les ports 80 (HTTP) ou 443 (HTTPS) sont ouverts
- Vérifiez que votre IP est sur liste blanche
- Examinez les plugins de sécurité qui pourraient bloquer l'accès à l'API

### 4. Contacter Votre Hébergeur

Ils peuvent vérifier :
- État du serveur et journaux
- Configurations du pare-feu
- Si votre IP a été bloquée
