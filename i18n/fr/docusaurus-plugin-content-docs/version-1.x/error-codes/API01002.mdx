# API01002 : Connection Refusée

## Ce que cela signifie

Le serveur a activement refusé la tentative de connexion. Cela est différent d'un délai d'attente — le serveur a répondu, mais a dit "non".

## Causes fréquentes

- **Serveur est en panne** — Le serveur web (Apache/Nginx) peut ne pas être en marche
- **Mauvais port** — Tentative de connexion sur le mauvais port
- **Blocage par le pare-feu** — Un pare-feu bloque la connexion
- **Restrictions IP** — Votre adresse IP peut être bloquée

## Comment corriger

### 1. Vérifiez que le serveur est en marche

Essayez d'accéder directement à votre site WordPress dans un navigateur. Si cela ne se charge pas, contactez votre fournisseur d'hébergement.

### 2. Vérifiez la configuration de l'URL

Assurez-vous d'utiliser la bonne URL :
- Vérifiez `http://` vs `https://`
- Vérifiez les fautes de frappe dans le nom de domaine
- Confirmez le port correct si vous utilisez un port non standard

### 3. Vérifiez les paramètres du pare-feu

Si vous avez accès à votre serveur :
- Vérifiez que le port 80 (HTTP) ou 443 (HTTPS) est ouvert
- Vérifiez si votre IP est sur liste blanche
- Passez en revue les plugins de sécurité pouvant bloquer l'accès à l'API

### 4. Contactez votre fournisseur d'hébergement

Ils peuvent vérifier :
- L'état du serveur et les journaux
- Les configurations du pare-feu
- Si votre IP a été bloquée
