# API05001 : API WooCommerce désactivée

## Ce que cela signifie

L'API REST WooCommerce est désactivée sur votre site. WCPOS nécessite l'API REST pour communiquer avec WooCommerce et accéder aux données du magasin.

## Causes courantes

- **Paramètre WooCommerce** — L'API REST a été intentionnellement désactivée
- **Plugin de sécurité** — Un plugin de sécurité bloque l'accès à l'API
- **Restriction d'hébergement** — Votre hébergeur a désactivé l'accès à l'API REST
- **Problèmes de permaliens** — Les permaliens ne sont pas configurés pour l'API REST

## Comment résoudre

### 1. Activer l'API REST WooCommerce

Dans l'administration WordPress :
1. Aller à WooCommerce → Paramètres → Avancé → API REST
2. Assurez-vous que l'API REST est activée
3. Vérifiez que les clés API sont créées

### 2. Vérifier l'API REST WordPress

L'API WooCommerce dépend de l'API REST WordPress :
1. Visitez `https://votresite.com/wp-json/` dans votre navigateur
2. Doit renvoyer des données JSON
3. Si ce n'est pas le cas, consultez [API05004](./API05004)

### 3. Vérifier les plugins de sécurité

Plugins de sécurité courants pouvant bloquer l'API :
- **Wordfence** — Vérifier les paramètres du pare-feu
- **iThemes Security** — Vérifier les paramètres de l'API REST
- **All In One WP Security** — Examiner les règles du pare-feu

Autoriser les points de terminaison de l'API REST ou l'application POS.

### 4. Vérifier .htaccess

Rechercher des règles bloquant l'accès à l'API :

```apache
# Supprimer ou modifier les règles bloquant /wp-json/
# Assurez-vous que mod_rewrite est activé
```

### 5. Actualiser les permaliens

Les paramètres de permaliens nécessitent parfois une actualisation :
1. Aller à Paramètres → Permaliens
2. Cliquer sur "Enregistrer les modifications" (même sans faire de changements)
3. Cela régénère les règles de réécriture
