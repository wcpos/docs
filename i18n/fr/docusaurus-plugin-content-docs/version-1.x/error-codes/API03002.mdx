# API03002 : Paramètres requis manquants {#api03002-missing-required-parameters}

## Ce que cela signifie {#what-this-means}

La demande manque de données que le serveur nécessite pour accomplir l'action. Des champs requis n'ont pas été inclus dans la demande.

## Causes communes {#common-causes}

- **Formulaire incomplet** — Les champs requis n'ont pas été remplis
- **Données non sauvegardées** — Les données du formulaire n'ont pas été capturées correctement
- **Incompatibilité de version** — Le plugin attend des paramètres différents de ceux que le POS envoie
- **Champs personnalisés** — Des champs personnalisés obligatoires sont manquants

## Comment résoudre {#how-to-fix}

### 1. Compléter les champs requis {#1-complete-required-fields}

Vérifiez que toutes les informations requises sont remplies :
- Détails du client (si requis)
- Informations sur le produit
- Détails de la commande

### 2. Actualiser et réessayer {#2-refresh-and-retry}

L'état du formulaire peut être incomplet :
1. Actualisez le POS
2. Réentrez les informations requises
3. Essayez à nouveau l'action

### 3. Assurez-vous de la compatibilité {#3-check-plugin-versions}

Assurez-vous de la compatibilité :
- Mettez à jour le plugin WCPOS
- Mettez à jour l'application POS
- Les deux doivent être sur des versions compatibles

### 4. Vérifiez les exigences des champs de commande {#4-check-required-field-settings}

Dans WooCommerce, certains champs peuvent être marqués comme obligatoires :
- Examinez les exigences des champs de commande
- Vérifiez les configurations de champs personnalisés
- Ajustez les exigences si nécessaire

### 5. Vérifiez les intégrations personnalisées {#5-check-custom-integrations}

Si vous utilisez des plugins ou des intégrations personnalisées :
- Ils peuvent nécessiter des champs supplémentaires
- Consultez la documentation du plugin
- Vérifiez les conflits
