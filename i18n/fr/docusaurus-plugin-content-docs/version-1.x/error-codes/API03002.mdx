# API03002 : Paramètres requis manquants

## Ce que cela signifie

La requête manque de données dont le serveur a besoin pour terminer l'action. Les champs requis n'ont pas été inclus dans la requête.

## Causes courantes

- **Formulaire incomplet** — Les champs requis n'ont pas été remplis
- **Données non enregistrées** — Les données du formulaire n'ont pas été capturées correctement
- **Incompatibilité de version** — Le plugin attend des paramètres différents de ceux envoyés par le POS
- **Champs personnalisés** — Des champs personnalisés requis sont manquants

## Comment résoudre

### 1. Remplir les champs requis

Vérifiez que toutes les informations requises sont renseignées :
- Détails du client (si nécessaire)
- Informations sur le produit
- Détails de la commande

### 2. Actualiser et réessayer

L'état du formulaire peut être incomplet :
1. Actualiser le POS
2. Ressaisir les informations requises
3. Réessayer l'action

### 3. Vérifier les versions des plugins

Assurer la compatibilité :
- Mettre à jour le plugin WCPOS
- Mettre à jour l'application POS
- S'assurer que les versions sont compatibles

### 4. Vérifier les paramètres des champs requis

Dans WooCommerce, certains champs peuvent être marqués comme obligatoires :
- Examiner les exigences des champs de paiement
- Vérifier les configurations des champs personnalisés
- Ajuster les exigences si nécessaire

### 5. Vérifier les intégrations personnalisées

Si vous utilisez des plugins ou des intégrations personnalisés :
- Ils peuvent nécessiter des champs supplémentaires
- Consulter la documentation du plugin
- Vérifier les éventuels conflits
