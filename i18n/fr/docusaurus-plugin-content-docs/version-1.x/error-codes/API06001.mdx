# API06001 : Format d'URL invalide

## Ce que cela signifie

L'URL configurée pour votre site WooCommerce n'est pas au format valide. Les URL doivent suivre les conventions standard des adresses web.

## Causes courantes

- **Protocole manquant** — L'URL ne commence pas par `http://` ou `https://`
- **Erreur de frappe dans l'URL** — Erreurs d'orthographe dans le domaine
- **Caractères invalides** — Caractères spéciaux non autorisés dans les URL
- **URL incomplète** — L'URL manque de parties comme l'extension de domaine

## Comment réparer

### 1. Vérifiez le format de l'URL

Une URL valide devrait ressembler à :
- ✅ `https://votreboutique.com`
- ✅ `https://www.votreboutique.com`
- ✅ `https://boutique.votredomaine.com`
- ❌ `votreboutique.com` (protocole manquant)
- ❌ `https://votreboutique` (extension manquante)
- ❌ `https://votre boutique.com` (contient un espace)

### 2. Inclure le protocole

Incluez toujours `https://` ou `http://` :
- Préféré : `https://` (sécurisé)
- De secours : `http://` (si HTTPS n'est pas disponible)

### 3. Supprimer les caractères supplémentaires

Assurez-vous que l'URL ne contient pas :
- Des barres obliques finales (optionnelles mais soyez cohérent)
- Des espaces supplémentaires
- Des caractères spéciaux
- Des paramètres d'URL (sauf si nécessaire)

### 4. Vérifiez que l'URL fonctionne

Testez l'URL dans un navigateur :
1. Copiez l'URL que vous utilisez
2. Collez-la dans un navigateur
3. Cela devrait charger votre site WordPress

### 5. Mettez à jour la configuration du POS

Entrez l'URL corrigée dans les paramètres du POS :
- Correspond exactement à ce qui fonctionne dans votre navigateur
- Incluez ou excluez `www.` selon ce que votre site utilise

## Exemples

| Invalide | Valide |
|----------|--------|
| `maboutique.com` | `https://maboutique.com` |
| `https://ma boutique.com` | `https://maboutique.com` |
| `https://maboutique` | `https://maboutique.com` |
| `htp://maboutique.com` | `https://maboutique.com` |
