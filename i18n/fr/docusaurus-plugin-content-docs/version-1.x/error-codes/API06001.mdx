# API06001 : Format d'URL Invalide

## Ce Que Cela Signifie

L'URL configurée pour votre site WooCommerce n'est pas dans un format valide. Les URL doivent suivre les conventions standard des adresses web.

## Causes Courantes

- **Protocole manquant** — L'URL ne commence pas par `http://` ou `https://`
- **Faute de frappe dans l'URL** — Erreurs d'orthographe dans le domaine
- **Caractères non valides** — Caractères spéciaux non autorisés dans les URL
- **URL incomplète** — URL manquant des parties comme l'extension de domaine

## Comment Corriger

### 1. Vérifier le Format de l'URL

Une URL valide doit ressembler à :
- ✅ `https://votremagasin.com`
- ✅ `https://www.votremagasin.com`
- ✅ `https://magasin.votredomaine.com`
- ❌ `votremagasin.com` (protocole manquant)
- ❌ `https://votremagasin` (extension manquante)
- ❌ `https://votre magasin.com` (contient un espace)

### 2. Inclure le Protocole

Toujours inclure `https://` ou `http://` :
- Recommandé : `https://` (sécurisé)
- Alternative : `http://` (si HTTPS n'est pas disponible)

### 3. Supprimer les Caractères Supplémentaires

Assurez-vous que l'URL ne contient pas :
- Barres obliques de fin (optionnelles mais soyez cohérent)
- Espaces supplémentaires
- Caractères spéciaux
- Paramètres d'URL (sauf si nécessaires)

### 4. Vérifier le Fonctionnement de l'URL

Testez l'URL dans un navigateur :
1. Copiez l'URL que vous utilisez
2. Collez-la dans un navigateur
3. Elle doit charger votre site WordPress

### 5. Mettre à Jour la Configuration du POS

Entrez l'URL corrigée dans les paramètres du POS :
- Correspondre exactement à ce qui fonctionne dans votre navigateur
- Inclure ou exclure `www.` selon votre site

## Exemples

| Invalide | Valide |
|---------|-------|
| `monmagasin.com` | `https://monmagasin.com` |
| `https://mon magasin.com` | `https://monmagasin.com` |
| `https://monmagasin` | `https://monmagasin.com` |
| `htp://monmagasin.com` | `https://monmagasin.com` |
