# DB02001 : Enregistrement en Double

## Ce Que Cela Signifie

La base de données a essayé d'insérer un enregistrement avec un ID qui existe déjà. Chaque enregistrement doit avoir un identifiant unique.

## Causes Communes

- **Conflit de sync** — Même données synchronisées deux fois
- **Condition de course** — Plusieurs opérations créant le même enregistrement
- **Corruption des données** — Les IDs ont été corrompus ou dupliqués
- **Problème d'importation** — Importation de données qui existent déjà

## Comment Réparer

### 1. Rafraîchir les Données

Re-sync depuis le serveur :
- Tirer les dernières données
- Laisser le processus de sync résoudre les conflits
- Les données les plus récentes devraient prévaloir

### 2. Réessayer l'Opération

Si vous créez de nouveaux enregistrements :
- Le système a peut-être déjà créé celui-ci
- Vérifiez si l'enregistrement existe
- Évitez de cliquer sur soumettre plusieurs fois

### 3. Vider le Cache Local

Réinitialiser les données locales :
1. Vider la base de données/cache local
2. Se connecter à nouveau
3. Synchroniser les données fraîches

### 4. Vérifier les Doublons dans WooCommerce

Si le problème persiste :
- Vérifiez WooCommerce pour des entrées dupliquées
- Recherchez des produits/commandes avec les mêmes IDs
- Nettoyez tout doublon dans l'administration

### 5. Signaler les Problèmes Persistants

Si cela se produit fréquemment :
- Notez quelles actions le causent
- Vérifiez s'il y a des motifs
- Signalez au support avec des détails

## Pourquoi Cela Compte

Des IDs dupliqués causeraient :
- Confusion sur quel enregistrement est correct
- Écrasement de données
- Problèmes de synchronisation

L'erreur empêche la corruption des données.
