# DB02001 : Enregistrement en double

## Ce que cela signifie

La base de données a tenté d'insérer un enregistrement avec un identifiant qui existe déjà. Chaque enregistrement doit avoir un identifiant unique.

## Causes courantes

- **Conflit de synchronisation** — Mêmes données synchronisées deux fois
- **Condition de course** — Plusieurs opérations créant le même enregistrement
- **Corruption de données** — Les identifiants ont été corrompus ou dupliqués
- **Problème d'importation** — Importation de données déjà existantes

## Comment résoudre

### 1. Actualiser les données

Synchroniser depuis le serveur :
- Récupérer les données les plus récentes
- Laisser le processus de synchronisation résoudre les conflits
- Les données les plus récentes doivent avoir la priorité

### 2. Réessayer l'opération

Lors de la création de nouveaux enregistrements :
- Le système a peut-être déjà créé l'enregistrement
- Vérifier si l'enregistrement existe
- Éviter de cliquer plusieurs fois sur le bouton de soumission

### 3. Effacer le cache local

Réinitialiser les données locales :
1. Effacer la base de données/cache local
2. Se reconnecter
3. Synchroniser les données à nouveau

### 4. Vérifier les doublons dans WooCommerce

Si le problème persiste :
- Vérifier les entrées en double dans WooCommerce
- Rechercher des produits/commandes avec des identifiants identiques
- Nettoyer les doublons dans l'administration

### 5. Signaler les problèmes persistants

Si cela se produit fréquemment :
- Noter les actions qui le déclenchent
- Rechercher des modèles
- Signaler au support avec des détails

## Pourquoi c'est important

Des identifiants en double entraîneraient :
- Une confusion sur l'enregistrement correct
- Des écrasements de données
- Des problèmes de synchronisation

L'erreur empêche la corruption des données.
