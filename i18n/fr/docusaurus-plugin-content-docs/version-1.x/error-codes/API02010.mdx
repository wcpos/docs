# API02010 : Authentification requise

## Ce que cela signifie

L'action que vous essayez d'effectuer nécessite une authentification, mais vous n'êtes pas actuellement connecté. Le POS a besoin d'identifiants valides pour accéder à cette ressource.

## Causes courantes

- **Non connecté** — Vous n'avez pas encore été authentifié
- **Session effacée** — Votre session a été effacée ou a expiré
- **Accès à une ressource protégée** — La ressource nécessite une authentification
- **Données de l'application effacées** — Les identifiants stockés ont été supprimés

## Correspondance d'erreur serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code serveur | Source |
|-------------|--------|
| `rest_login_required` | API REST WordPress |
| `jwt_auth_no_auth_header` | Plugin d'authentification JWT |
| HTTP 401 | Toute réponse du serveur (secours) |

## Comment résoudre

### 1. Se connecter

Si vous n'êtes pas connecté :
1. Ouvrez l'écran de connexion du POS
2. Entrez vos identifiants WordPress
3. Terminez le processus d'authentification

### 2. Vérifier le statut de la session

Si vous pensiez être connecté :
- Votre session a peut-être expiré
- Consultez [API02002](./API02002) (Token expiré) pour plus de détails
- Reconnectez-vous pour restaurer l'accès

### 3. Vérifier la configuration du serveur

Assurez-vous que les points de terminaison API sont correctement configurés :
- L'API REST WooCommerce doit être activée
- Le plugin WCPOS doit être actif
- Les points de terminaison d'authentification doivent être accessibles

### 4. Vérifier les problèmes de navigateur/application

Si vous êtes déconnecté de manière inattendue :
- Effacez le cache du navigateur (si vous utilisez la version web)
- Vérifiez que les cookies/stockage local ne sont pas bloqués
- Vérifiez que l'application a la permission de stocker des données

## Ce qui nécessite une authentification

La plupart des opérations du POS nécessitent une authentification :
- Affichage des produits et des clients
- Création et édition de commandes
- Traitement des paiements
- Accès aux rapports

Seul l'écran de connexion initial est accessible sans authentification.
