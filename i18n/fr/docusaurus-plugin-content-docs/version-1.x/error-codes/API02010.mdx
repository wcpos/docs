# API02010 : Authentification requise

## Ce que cela signifie

L'action que vous essayez d'effectuer nécessite une authentification, mais vous n'êtes actuellement pas connecté. Le POS a besoin de références valides pour accéder à cette ressource.

## Causes courantes

- **Non connecté** — Vous ne vous êtes pas encore authentifié
- **Session effacée** — Votre session a été effacée ou a expiré
- **Accès à une ressource protégée** — La ressource nécessite une authentification
- **Données de l'application effacées** — Les références stockées ont été supprimées

## Cartographie des erreurs du serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code Serveur | Source |
|--------------|--------|
| `rest_login_required` | API REST WordPress |
| `jwt_auth_no_auth_header` | Plugin d'authentification JWT |
| HTTP 401 | Toute réponse du serveur (fallback) |

## Comment réparer

### 1. Connectez-vous

Si vous ne vous êtes pas connecté :
1. Ouvrez l'écran de connexion du POS
2. Saisissez vos références WordPress
3. Terminez le processus d'authentification

### 2. Vérifiez l'état de la session

Si vous pensiez être connecté :
- Votre session a peut-être expiré
- Recherchez [API02002](./API02002) (Token expiré) pour plus de détails
- Connectez-vous à nouveau pour restaurer l'accès

### 3. Vérifiez la configuration du serveur

Assurez-vous que les points de terminaison de l'API sont correctement configurés :
- L'API REST de WooCommerce doit être activée
- Le plugin WCPOS doit être actif
- Les points de terminaison d'authentification doivent être accessibles

### 4. Vérifiez les problèmes de navigateur/application

Si vous êtes déconnecté de manière inattendue :
- Effacez le cache du navigateur (si vous utilisez la version web)
- Vérifiez que les cookies/le stockage local ne sont pas bloqués
- Vérifiez que l'application a la permission de stocker des données

## Qu'est-ce qui nécessite une authentification ?

La plupart des opérations POS nécessitent une authentification :
- Consultation des produits et des clients
- Création et modification de commandes
- Traitement des paiements
- Accès aux rapports

Seul l'écran de connexion initial est accessible sans authentification.
