# API03003: Valeur de Paramètre Invalide {#api03003-invalid-parameter-value}

## Ce Que Cela Signifie {#what-this-means}

Une des valeurs dans votre demande est invalide. Le format de données, le type ou la valeur ne correspondent pas à ce que le serveur attend.

## Causes Courantes {#common-causes}

- **Mauvais type de données** — Texte là où un nombre est attendu (ou vice versa)
- **Hors limites** — La valeur dépasse les limites autorisées
- **Format invalide** — Les champs formatés comme l'email, le téléphone, etc. sont incorrects
- **Référence invalide** — Référence à un ID qui n'existe pas

## Mappage des Erreurs du Serveur {#server-error-mapping}

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code Serveur | Source |
|--------------|--------|
| `rest_invalid_param` | API REST WordPress |
| `woocommerce_rest_invalid_id` | API REST WooCommerce |

## Comment Réparer {#how-to-fix}

### 1. Vérifiez les Valeurs d'Entrée {#1-check-input-values}

Examinez les données que vous soumettez :
- **Prix** — Doivent être des nombres valides
- **Quantités** — Doivent être des entiers positifs
- **Emails** — Doivent être au format email valide
- **IDs** — Doivent référencer des enregistrements existants

### 2. Recherchez des Erreurs de Champ Spécifiques {#2-look-for-specific-field-errors}

L'erreur peut indiquer quel champ est invalide :
- Vérifiez les détails du message d'erreur
- Corrigez le champ spécifique
- Réessayez l'action

### 3. Vérifiez que les Données Référencées Existent {#3-verify-referenced-data-exists}

Si vous faites référence à des produits, clients ou autres enregistrements :
- Assurez-vous que l'élément existe dans WooCommerce
- Vérifiez s'il a été récemment supprimé
- Synchronisez les données pour actualiser les enregistrements locaux

### 4. Vérifiez les Contraintes des Champs {#4-check-field-constraints}

WooCommerce peut avoir des contraintes :
- Valeurs maximum/minimum
- Formats requis
- Options autorisées pour les champs sélectionnés

### 5. Effacer et Ré-entrer {#5-clear-and-re-enter}

Parfois, les données se corrompent :
- Effacez le champ problématique
- Ré-entrez la valeur depuis le début
- Évitez de copier-coller depuis d'autres sources
