# API03003 : Valeur de Paramètre Invalide

## Ce Que Cela Signifie

Une des valeurs dans votre requête est invalide. Le format, le type ou la valeur des données ne correspond pas à ce que le serveur attend.

## Causes Courantes

- **Mauvais type de données** — Texte là où un nombre est attendu (ou vice versa)
- **Hors limites** — La valeur dépasse les limites autorisées
- **Format incorrect** — Emails, numéros de téléphone ou autres champs formatés sont incorrects
- **Référence invalide** — Référencement d'un ID qui n'existe pas

## Mappage des Erreurs Serveur

Ce code d'erreur est déclenché lorsque le serveur retourne :

| Code Serveur | Source |
|-------------|--------|
| `rest_invalid_param` | API REST WordPress |
| `woocommerce_rest_invalid_id` | API REST WooCommerce |

## Comment Corriger

### 1. Vérifier les Valeurs d'Entrée

Examinez les données que vous soumettez :
- **Prix** — Doivent être des nombres valides
- **Quantités** — Doivent être des entiers positifs
- **Emails** — Doivent être dans un format d'email valide
- **IDs** — Doivent référencer des enregistrements existants

### 2. Rechercher des Erreurs de Champ Spécifiques

L'erreur peut indiquer quel champ est invalide :
- Vérifier les détails du message d'erreur
- Corriger le champ spécifique
- Réessayer l'action

### 3. Vérifier l'Existence des Données Référencées

Si vous référencez des produits, clients ou autres enregistrements :
- Assurez-vous que l'élément existe dans WooCommerce
- Vérifiez s'il n'a pas été récemment supprimé
- Synchronisez les données pour actualiser les enregistrements locaux

### 4. Vérifier les Contraintes de Champ

WooCommerce peut avoir des contraintes :
- Valeurs maximales/minimales
- Formats requis
- Options autorisées pour les champs de sélection

### 5. Effacer et Ressaisir

Parfois, les données peuvent être corrompues :
- Effacez le champ problématique
- Ressaisissez la valeur à partir de zéro
- Évitez de copier-coller à partir d'autres sources
