# API06002 : URL API manquante {#api06002-missing-api-url}

## Ce que cela signifie {#what-this-means}

Aucune URL n'a été configurée pour se connecter à votre boutique WooCommerce. Le POS doit savoir où se trouve votre boutique.

## Causes courantes {#common-causes}

- **Première configuration** — L'URL n'a jamais été saisie
- **Configuration effacée** — Les paramètres ont été réinitialisés ou effacés
- **Données de l'application effacées** — Les données de l'application ont été supprimées
- **Problème de migration** — L'URL perdue lors de la mise à jour ou de la migration

## Comment résoudre {#how-to-fix}

### 1. Entrez l'URL de votre boutique {#1-enter-your-store-url}

Dans l'application POS :
1. Ouvrez l'écran des paramètres ou de connexion
2. Trouvez le champ de configuration de l'URL/site
3. Entrez l'URL de votre site WordPress
4. Exemple : `https://votreboutique.com`

### 2. Trouvez votre URL WordPress {#2-find-your-wordpress-url}

Si vous n'êtes pas sûr de votre URL :
1. Connectez-vous à l'administration WordPress
2. Allez dans Réglages → Général
3. Regardez "Adresse WordPress (URL)"
4. Utilisez cette URL dans le POS

### 3. Vérifiez les paramètres enregistrés {#3-check-stored-settings}

Si l'URL a été configurée précédemment :
- Vérifiez si les données de l'application ont été accidentellement effacées
- Cherchez des options de sauvegarde/exportation des paramètres
- Réentrez l'URL si nécessaire

### 4. Vérifiez que l'URL est correcte {#4-verify-url-is-correct}

Avant de sauvegarder :
1. Testez l'URL dans un navigateur
2. Assurez-vous qu'elle charge votre site WordPress
3. Incluez `https://` ou `http://`

## Liste de contrôle de configuration {#setup-checklist}

Lors de la configuration de WCPOS pour la première fois :

1. ☐ Entrez l'URL de votre site WordPress
2. ☐ Vérifiez que le plugin WCPOS est installé et actif
3. ☐ Connectez-vous avec vos identifiants WordPress
4. ☐ Autorisez la synchronisation initiale des données à se terminer
