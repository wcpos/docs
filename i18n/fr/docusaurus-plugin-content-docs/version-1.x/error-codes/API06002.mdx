# API06002 : URL d'API manquante

## Ce que cela signifie

Aucune URL n'a été configurée pour se connecter à votre boutique WooCommerce. Le système de caisse doit connaître l'emplacement de votre boutique.

## Causes courantes

- **Configuration initiale** — URL n'a jamais été saisie
- **Configuration effacée** — Les paramètres ont été réinitialisés ou supprimés
- **Données de l'application effacées** — Les données de l'application ont été supprimées
- **Problème de migration** — URL perdue lors d'une mise à jour ou d'une migration

## Comment résoudre

### 1. Saisissez l'URL de votre boutique

Dans l'application de caisse :
1. Ouvrez les paramètres ou l'écran de connexion
2. Trouvez le champ de configuration de l'URL/du site
3. Saisissez l'URL de votre site WordPress
4. Exemple : `https://votreboutique.com`

### 2. Trouver votre URL WordPress

Si vous n'êtes pas sûr de votre URL :
1. Connectez-vous à l'administration WordPress
2. Allez dans Paramètres → Général
3. Recherchez "Adresse WordPress (URL)"
4. Utilisez cette URL dans la caisse

### 3. Vérifier les paramètres enregistrés

Si l'URL était précédemment configurée :
- Vérifiez si les données de l'application ont été accidentellement effacées
- Recherchez des options de sauvegarde/exportation des paramètres
- Ressaisissez l'URL si nécessaire

### 4. Vérifier que l'URL est correcte

Avant d'enregistrer :
1. Testez l'URL dans un navigateur
2. Assurez-vous qu'elle charge votre site WordPress
3. Incluez `https://` ou `http://`

## Liste de vérification de configuration

Lors de la configuration initiale de WCPOS :

1. ☐ Saisissez l'URL de votre site WordPress
2. ☐ Vérifiez que le plugin WCPOS est installé et actif
3. ☐ Connectez-vous avec vos identifiants WordPress
4. ☐ Autoriser la synchronisation initiale des données
