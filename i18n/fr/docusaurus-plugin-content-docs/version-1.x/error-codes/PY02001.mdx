# PY02001 : Erreur de passerelle de paiement

## Ce que cela signifie

Une erreur s'est produite lors de la communication avec la passerelle de paiement. Le processeur de paiement n'a pas pu terminer la demande.

## Causes courantes

- **Temps d'arrêt de la passerelle** — Le service de paiement rencontre des problèmes (expérimente des problèmes)
- **Erreur de configuration** — Les paramètres de la passerelle sont incorrects
- **Identifiants API** — Clés API invalides ou expirées
- **Problèmes de réseau** — Problèmes de connexion à la passerelle

## Comment corriger

### 1. Réessayez la transaction

Parfois, les passerelles rencontrent des problèmes temporaires :
- Attendez un moment
- Essayez de traiter à nouveau le paiement
- Le problème peut se résoudre de lui-même

### 2. Vérifiez l'état de la passerelle

Visitez la page de statut de votre fournisseur de paiement :
- Stripe : status.stripe.com
- PayPal : Vérifiez la communauté ou le statut de PayPal
- Square : status.squareup.com
- Vérifiez les pannes connues

### 3. Vérifiez la configuration de la passerelle

Dans l'administration de WooCommerce :
1. Allez dans WooCommerce → Paramètres → Paiements
2. Cliquez sur votre passerelle de paiement
3. Vérifiez que les clés API sont correctes
4. Vérifiez que le mode test/en direct correspond

### 4. Vérifiez les identifiants API

Assurez-vous que les identifiants sont valides :
- Les clés API n'ont pas expiré
- Utilisation des clés en direct pour les transactions en direct
- Utilisation des clés de test pour les transactions de test
- Les identifiants correspondent au bon compte

### 5. Testez avec une passerelle différente

Si disponible :
- Essayez de traiter avec une passerelle alternative
- Utilisez le paiement en espèces comme solution de secours
- Terminez la vente et réconciliez plus tard

## Pour les propriétaires de magasin

Problèmes de configuration courants :
- Mélange de clés API de test et de production
- Clés API provenant de comptes différents
- Méthodes de paiement désactivées
- Identifiants expirés ou révoqués

Contactez le support de votre fournisseur de paiement si le problème persiste.
