# PY02001 : Erreur de Passerelle de Paiement

## Ce Que Cela Signifie

Une erreur s'est produite lors de la communication avec la passerelle de paiement. Le processeur de paiement n'a pas pu terminer la requête.

## Causes Courantes

- **Interruption de la passerelle** — Le service de paiement connaît des problèmes
- **Erreur de configuration** — Les paramètres de la passerelle sont incorrects
- **Identifiants API** — Clés API invalides ou expirées
- **Problèmes réseau** — Problèmes de connexion à la passerelle

## Comment Résoudre

### 1. Réessayer la Transaction

Parfois, les passerelles ont des problèmes temporaires :
- Attendez un moment
- Essayez de nouveau de traiter le paiement
- Le problème pourrait se résoudre de lui-même

### 2. Vérifier le Statut de la Passerelle

Visitez la page de statut de votre fournisseur de paiement :
- Stripe : status.stripe.com
- PayPal : Vérifiez la communauté PayPal ou son statut
- Square : status.squareup.com
- Vérifiez les interruptions connues

### 3. Vérifier la Configuration de la Passerelle

Dans l'Administration WooCommerce :
1. Allez à WooCommerce → Paramètres → Paiements
2. Cliquez sur votre passerelle de paiement
3. Vérifiez que les clés API sont correctes
4. Assurez-vous que le mode test/réel corresponde à vos clés

### 4. Vérifier les Identifiants API

Assurez-vous que les identifiants sont valides :
- Les clés API n'ont pas expiré
- Utilisation de clés de production pour les transactions réelles
- Utilisation de clés de test pour les transactions de test
- Les identifiants correspondent au bon compte

### 5. Tester avec une Autre Passerelle

Si disponible :
- Essayez de traiter avec une passerelle alternative
- Utilisez un paiement en espèces comme solution de repli
- Terminez la vente et rapprochez plus tard

## Pour les Propriétaires de Boutique

Problèmes de configuration courants :
- Mélange des clés API de test et de production
- Clés API de différents comptes
- Méthodes de paiement désactivées
- Identifiants expirés ou révoqués

Contactez le support de votre fournisseur de paiement si le problème persiste.
