# API02007: Échec de la mise à jour du token

## Signification

Le POS a tenté de renouveler automatiquement votre session d'authentification, mais la demande de mise à jour a échoué. Cela signifie généralement que vous devrez vous reconnecter.

## Causes courantes

- **Token de mise à jour expiré** — Le token de mise à jour lui-même a expiré
- **Token de mise à jour révoqué** — Un administrateur a invalidé toutes les sessions
- **Erreur du serveur** — Le serveur n'a pas pu traiter la demande de mise à jour
- **Problème de réseau** — Problème de connexion lors de la tentative de mise à jour

## Comment résoudre

### 1. Se reconnecter

La solution la plus fiable :
1. Déconnectez-vous du POS
2. Reconnectez-vous avec vos identifiants
3. Cela crée une nouvelle session avec de nouveaux tokens

### 2. Vérifier la connexion réseau

Si la mise à jour échoue en raison de problèmes de réseau :
- Vérifiez votre connexion internet
- Essayez d'accéder à votre site WordPress dans un navigateur
- Attendez un moment et réessayez

### 3. Vérifier l'état du serveur

Si le serveur a des problèmes :
- Vérifiez que WordPress fonctionne correctement
- Vérifiez les erreurs PHP dans vos journaux de serveur
- Assurez-vous que le plugin d'authentification est actif

### 4. Effacer les données de session

Si les problèmes persistent :
- Effacez le cache de l'application POS
- Supprimez les identifiants enregistrés
- Commencez à nouveau avec une nouvelle connexion

## Mise à jour automatique du token

WCPOS met automatiquement à jour les tokens avant qu'ils n'expirent pour maintenir votre session. Si ce processus échoue à plusieurs reprises, il peut y avoir un problème de configuration qui nécessite une attention particulière.
