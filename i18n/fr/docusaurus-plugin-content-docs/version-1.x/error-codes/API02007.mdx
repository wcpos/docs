# API02007: Échec du rafraîchissement du token {#api02007-token-refresh-failed}

## Signification {#what-this-means}

Le POS a tenté de renouveler automatiquement votre session d'authentification, mais la demande de rafraîchissement a échoué. Cela signifie généralement que vous devrez vous reconnecter.

## Causes courantes {#common-causes}

- **Refresh token expiré** — Le refresh token lui-même a expiré
- **Refresh token révoqué** — Un administrateur a invalidé toutes les sessions
- **Erreur du serveur** — Le serveur n'a pas pu traiter la demande de rafraîchissement
- **Problème de réseau** — Problème de connexion lors de la tentative de rafraîchissement

## Comment résoudre {#how-to-fix}

### 1. Se reconnecter {#1-log-in-again}

La correction la plus fiable :
1. Déconnectez-vous du POS
2. Reconnectez-vous avec vos identifiants
3. Cela crée une nouvelle session avec de nouveaux tokens

### 2. Vérifier la connexion réseau {#2-check-network-connection}

Si le rafraîchissement échoue en raison de problèmes de réseau :
- Vérifiez votre connexion internet
- Essayez d'accéder à votre site WordPress dans un navigateur
- Attendez un moment et réessayez

### 3. Vérifier l'état du serveur {#3-check-server-status}

Si le serveur a des problèmes :
- Vérifiez que WordPress fonctionne correctement
- Vérifiez les erreurs PHP dans vos journaux de serveur
- Assurez-vous que le plugin d'authentification est actif

### 4. Effacer les données de session {#4-clear-session-data}

Si les problèmes persistent :
- Effacez le cache de l'application POS
- Supprimez les identifiants enregistrés
- Commencez à nouveau avec une nouvelle connexion

## Rafraîchissement automatique du token {#automatic-token-refresh}

WCPOS rafraîchit automatiquement les tokens avant qu'ils n'expirent pour maintenir votre session. Si ce processus échoue à plusieurs reprises, il peut y avoir un problème de configuration qui nécessite une attention particulière.
