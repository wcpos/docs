# API02007 : Échec de l'actualisation du jeton

## Ce que cela signifie

Le système POS a essayé de renouveler automatiquement votre session d'authentification, mais la demande d'actualisation a échoué. Cela signifie généralement que vous devrez vous reconnecter.

## Causes courantes

- **Jeton d'actualisation expiré** — Le jeton d'actualisation lui-même a expiré
- **Jeton d'actualisation révoqué** — Un administrateur a invalidé toutes les sessions
- **Erreur de serveur** — Le serveur n'a pas pu traiter la demande d'actualisation
- **Problème réseau** — Problème de connexion lors de la tentative d'actualisation

## Comment résoudre

### 1. Se reconnecter

La solution la plus fiable :
1. Se déconnecter du POS
2. Se reconnecter avec vos identifiants
3. Cela crée une nouvelle session avec de nouveaux jetons

### 2. Vérifier la connexion réseau

Si l'actualisation échoue en raison de problèmes réseau :
- Vérifier votre connexion internet
- Essayer d'accéder à votre site WordPress dans un navigateur
- Attendre un moment et réessayer

### 3. Vérifier l'état du serveur

Si le serveur rencontre des problèmes :
- Vérifier que WordPress fonctionne correctement
- Rechercher des erreurs PHP dans vos journaux de serveur
- S'assurer que le plugin d'authentification est actif

### 4. Effacer les données de session

Si les problèmes persistent :
- Effacer le cache de l'application POS
- Supprimer les identifiants stockés
- Recommencer avec une nouvelle connexion

## Actualisation automatique du jeton

WCPOS actualise automatiquement les jetons avant leur expiration pour maintenir votre session. Si ce processus échoue à plusieurs reprises, il peut y avoir un problème de configuration qui nécessite une attention particulière.
