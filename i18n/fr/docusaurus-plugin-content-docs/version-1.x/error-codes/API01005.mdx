# API01005 : Erreur de Certificat SSL

## Ce Que Cela Signifie

Il y a un problème avec le certificat SSL/TLS de votre site web. Ce certificat garantit une connexion sécurisée (HTTPS). Le POS ne se connectera pas aux sites avec des certificats invalides pour protéger vos données.

## Causes Courantes

- **Certificat expiré** — Les certificats SSL doivent être renouvelés périodiquement
- **Certificat auto-signé** — Non émis par une autorité de confiance
- **Domaine incorrect** — Le certificat ne correspond pas au nom de domaine
- **Chaîne de certificat incomplète** — Certificats intermédiaires manquants
- **Contenu mixte** — Certaines ressources chargées via HTTP au lieu de HTTPS

## Comment Corriger

### 1. Vérifier le Statut du Certificat

Visitez votre site dans un navigateur et cliquez sur l'icône de cadenas pour afficher les détails du certificat. Recherchez :
- Date d'expiration
- Émis à (doit correspondre à votre domaine)
- Émis par (doit être une autorité reconnue)

### 2. Renouveler un Certificat Expiré

En cas d'expiration :
- La plupart des hébergeurs offrent des certificats Let's Encrypt gratuits
- Contactez votre hébergeur pour renouveler
- Vérifiez si le renouvellement automatique est activé

### 3. Corriger la Non-Concordance du Certificat

Assurez-vous que le certificat couvre :
- Votre domaine exact (`votresite.com`)
- La variante WWW (`www.votresite.com`) si utilisée
- Envisagez un certificat générique (`*.votresite.com`)

### 4. Installer les Certificats Intermédiaires Manquants

Utilisez un outil de vérification SSL (comme SSL Labs) pour identifier les certificats manquants. Votre hébergeur peut vous aider à les installer.

### 5. Forcer HTTPS

Dans WordPress, assurez-vous :
- L'URL du site utilise `https://`
- Le forçage SSL est activé dans les paramètres WooCommerce
