# DB03002 : Type de Données Invalide

## Ce Que Cela Signifie

Le type de données ne correspond pas à ce que la base de données attend. Par exemple, du texte a été fourni là où un nombre était attendu.

## Causes Courantes

- **Erreur de saisie utilisateur** — Saisie de texte dans un champ numérique
- **Problèmes d'importation** — Données importées avec un format incorrect
- **Corruption de données** — Valeurs corrompues lors du transfert
- **Conflit de plugin** — Un autre plugin a modifié les types de données

## Comment Résoudre

### 1. Vérifier Votre Saisie

Examinez les données que vous saisissez :
- **Prix** — Doivent être des nombres (par exemple, `19,99`)
- **Quantités** — Doivent être des nombres entiers (par exemple, `5`)
- **ID** — Doivent être numériques

### 2. Effacer et Ressaisir

Si les données sont corrompues :
- Effacez le champ
- Saisissez à nouveau la valeur manuellement
- Évitez de copier-coller à partir de sources externes

### 3. Vérifier les Données Source

Si synchronisation depuis WooCommerce :
- Vérifiez les données dans l'Administration WordPress
- Recherchez des champs mal formatés
- Corrigez les données à la source

### 4. Re-synchroniser les Données

Obtenir des données fraîches du serveur :
- Effacez le cache local
- Synchronisez à nouveau les données
- Vérifiez si le problème est résolu

### 5. Vérifier les Conflits de Plugins

Si vous utilisez d'autres plugins WooCommerce :
- Ils peuvent modifier les données de manière inattendue
- Désactivez temporairement pour tester
- Signalez les incompatibilités

## Exemples Courants

| Champ | Attendu | Invalide |
|-------|----------|----------|
| Prix | `19,99` | `19,99 €` |
| Quantité | `5` | `cinq` |
| SKU | `ABC123` | — (format OK) |
| ID Produit | `42` | `produit-42` |
