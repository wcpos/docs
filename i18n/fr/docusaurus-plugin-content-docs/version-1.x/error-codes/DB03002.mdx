# DB03002: Type de Données Invalide {#db03002-invalid-data-type}

## Ce Que Cela Signifie {#what-this-means}

Le type de données ne correspond pas à ce que la base de données attend. Par exemple, du texte a été fourni là où un nombre était attendu.

## Causes Courantes {#common-causes}

- **Erreur de saisie utilisateur** — Saisie de texte dans un champ numérique
- **Problèmes d'importation** — Données importées au mauvais format
- **Corruption des données** — Valeurs corrompues lors du transfert
- **Conflit de plugin** — Un autre plugin a modifié les types de données

## Comment Réparer {#how-to-fix}

### 1. Vérifiez Votre Saisie {#1-check-your-input}

Examinez les données que vous saisissez :
- **Prix** — Doivent être des nombres (par exemple, `19.99`)
- **Quantités** — Doivent être des nombres entiers (par exemple, `5`)
- **IDs** — Doivent être numériques

### 2. Effacer et Réintroduire {#2-clear-and-re-enter}

Si les données étaient corrompues :
- Effacez le champ
- Saisissez à nouveau manuellement la valeur
- Évitez de copier-coller à partir de sources externes

### 3. Vérifiez les Données Sources {#3-check-source-data}

Si vous synchronisez depuis WooCommerce :
- Vérifiez les données dans le WordPress Admin
- Recherchez les champs mal formatés
- Corrigez les données à la source

### 4. Resynchroniser les Données {#4-re-sync-data}

Obtenez des données fraîches du serveur :
- Effacez le cache local
- Synchronisez à nouveau les données
- Vérifiez si le problème est résolu

### 5. Vérifiez les Conflits de Plugins {#5-check-for-plugin-conflicts}

Si vous utilisez d'autres plugins WooCommerce :
- Ils peuvent modifier les données de manière inattendue
- Désactivez temporairement pour tester
- Signalez les incompatibilités

## Exemples Courants {#common-examples}

| Champ | Attendu | Invalide |
|-------|---------|----------|
| Prix | `19.99` | `$19.99` |
| Quantité | `5` | `cinq` |
| SKU | `ABC123` | — (format quelconque OK) |
| ID Produit | `42` | `produit-42` |
