# API05003 : Plugin WCPOS Périmé

## Ce que Cela Signifie

Le plugin WCPOS installé sur votre serveur est obsolète et n'est pas compatible avec votre version de l'application POS. Vous devez mettre à jour le plugin.

## Causes Courantes

- **Plugin non mis à jour** — Les mises à jour automatiques peuvent être désactivées
- **Mise à jour échouée** — Une tentative précédente de mise à jour a échoué
- **Différence de version** — Application POS mise à jour mais pas le plugin
- **Site de test/développement** — Test sur une version plus ancienne

## Comment Résoudre

### 1. Mettre à Jour le Plugin

Dans l'Administration WordPress :
1. Aller à Plugins → Plugins installés
2. Trouver "WooCommerce POS"
3. Si une mise à jour est disponible, cliquer sur "Mettre à jour maintenant"

### 2. Activer les Mises à Jour Automatiques

Pour éviter de futurs désalignements :
1. Aller à Plugins → Plugins installés
2. Trouver "WooCommerce POS"
3. Cliquer sur "Activer les mises à jour automatiques"

### 3. Mise à Jour Manuelle

Si la mise à jour automatique échoue :
1. Télécharger la dernière version depuis [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/)
2. Désactiver le plugin actuel
3. Supprimer les anciens fichiers du plugin
4. Téléverser et activer la nouvelle version

### 4. Vérifier les Erreurs de Mise à Jour

Si les mises à jour échouent :
- Vérifier les permissions de fichiers sur wp-content/plugins
- Vérifier l'espace disque disponible
- Rechercher des problèmes de mémoire PHP
- Consulter les journaux d'erreurs pour plus de détails

### 5. Rétrograder l'Application POS (Temporaire)

Si vous ne pouvez pas mettre à jour le plugin immédiatement :
- Utiliser une version plus ancienne de l'application POS
- Ce n'est qu'une solution temporaire
- Prévoir de mettre à jour le plugin rapidement

## Compatibilité des Versions

Toujours maintenir les deux composants à jour :
- L'application POS
- Le plugin WordPress WCPOS

Consulter les notes de version pour tout changement majeur lors de la mise à jour.
