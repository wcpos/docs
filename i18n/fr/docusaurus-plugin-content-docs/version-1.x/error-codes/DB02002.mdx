# DB02002 : Enregistrement non trouvé

## Ce que cela signifie

L'enregistrement demandé n'existe pas dans la base de données locale. Le POS a essayé d'accéder à un produit, un client, une commande ou un autre enregistrement qui n'est pas stocké localement.

## Causes courantes

- **Pas encore synchronisé** — Les données n'ont pas encore été téléchargées
- **Supprimé sur le serveur** — L'enregistrement a été supprimé dans WooCommerce
- **Synchronisation incomplète** — La synchronisation initiale est toujours en cours
- **Filtre appliqué** — Enregistrement exclu par des filtres actifs

## Comment corriger

### 1. Attendre la synchronisation

Si démarré récemment :
- Attendre que la synchronisation initiale soit complète
- Vérifier l'indicateur de progression de la synchronisation
- L'enregistrement devrait apparaître une fois synchronisé

### 2. Actualiser/Ré-synchroniser

Déclencher une actualisation des données :
- Utiliser le bouton d'actualisation/synchronisation
- Récupérer les dernières données du serveur
- Attendre la fin du processus

### 3. Vérifier WooCommerce

Vérifier que l'enregistrement existe sur le serveur :
- Se connecter à l'administration de WordPress
- Vérifier les produits, les commandes ou les clients
- Confirmer que l'élément existe toujours

### 4. Vérifier les filtres

L'enregistrement pourrait être filtré :
- Réviser les filtres actifs dans le POS
- Vérifier les filtres de catégorie
- Vérifier les paramètres de visibilité

### 5. Effacer et ré-synchroniser

Si la synchronisation semble bloquée :
1. Effacer les données locales
2. Se reconnecter
3. Compléter une nouvelle synchronisation

## Scénarios courants

- **Produit non affiché :** Vérifier s'il est publié et visible
- **Client non trouvé :** Peut ne pas avoir été encore synchronisé
- **Commande manquante :** Pourrait être en dehors de la plage de dates de synchronisation
