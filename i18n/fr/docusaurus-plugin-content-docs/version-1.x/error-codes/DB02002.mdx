# DB02002 : Enregistrement non trouvé {#db02002-record-not-found}

## Ce que cela signifie {#what-this-means}

L'enregistrement demandé n'existe pas dans la base de données locale. Le POS a tenté d'accéder à un produit, un client, une commande ou un autre enregistrement qui n'est pas stocké localement.

## Causes courantes {#common-causes}

- **Pas encore synchronisé** — Les données n'ont pas encore été téléchargées
- **Supprimé sur le serveur** — L'enregistrement a été supprimé dans WooCommerce
- **Synchronisation incomplète** — La synchronisation initiale est encore en cours
- **Filtre appliqué** — Enregistrement exclu par des filtres actifs

## Comment résoudre {#how-to-fix}

### 1. Attendre la synchronisation {#1-wait-for-sync}

Si récemment commencé :
- Attendez que la synchronisation initiale soit terminée
- Vérifiez l'indicateur de progression de la synchronisation
- L'enregistrement devrait apparaître une fois synchronisé

### 2. Actualiser/Re-synchroniser {#2-refreshre-sync}

Déclenchez un rafraîchissement des données :
- Utilisez le bouton de rafraîchissement/synchronisation (<Icon name="arrow-rotate-right" />) - appui court pour synchroniser
- **Appui long** sur le bouton de synchronisation pour l'option Effacer et Rafraîchir
- Récupérez les dernières données du serveur
- Attendez la fin

### 3. Vérifier WooCommerce {#3-check-woocommerce}

Vérifiez que l'enregistrement existe sur le serveur :
- Connectez-vous à l'administration WordPress
- Vérifiez Produits, Commandes ou Clients
- Confirmez que l'élément existe toujours

### 4. Vérifier les filtres {#4-check-filters}

L'enregistrement pourrait être filtré :
- Vérifiez les filtres actifs dans le POS
- Vérifiez les filtres de catégorie
- Vérifiez les paramètres de visibilité

### 5. Effacer et Re-synchroniser {#5-clear-and-re-sync}

Si la synchronisation semble bloquée :
1. Effacez les données locales
2. Connectez-vous à nouveau
3. Complétez une nouvelle synchronisation

## Scénarios courants {#common-scenarios}

- **Produit non affiché :** Vérifiez s'il est publié et visible
- **Client non trouvé :** Peut ne pas avoir encore été synchronisé
- **Commande manquante :** Pourrait être en dehors de la plage de dates de synchronisation
