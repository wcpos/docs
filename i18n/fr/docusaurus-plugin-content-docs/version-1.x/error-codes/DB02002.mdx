# DB02002 : Enregistrement Non Trouvé

## Ce Que Cela Signifie

L'enregistrement demandé n'existe pas dans la base de données locale. Le système POS a tenté d'accéder à un produit, un client, une commande ou un autre enregistrement qui n'est pas stocké localement.

## Causes Courantes

- **Pas encore synchronisé** — Les données n'ont pas encore été téléchargées
- **Supprimé sur le serveur** — L'enregistrement a été retiré de WooCommerce
- **Synchronisation incomplète** — La synchronisation initiale est toujours en cours
- **Filtre appliqué** — L'enregistrement est exclu par des filtres actifs

## Comment Résoudre

### 1. Attendre la Synchronisation

Si récemment démarré :
- Attendre que la synchronisation initiale se termine
- Vérifier l'indicateur de progression de la synchronisation
- L'enregistrement devrait apparaître une fois synchronisé

### 2. Actualiser/Resynchroniser

Déclencher une actualisation des données :
- Utiliser le bouton d'actualisation/synchronisation
- Récupérer les dernières données du serveur
- Attendre la fin du processus

### 3. Vérifier WooCommerce

Vérifier que l'enregistrement existe sur le serveur :
- Se connecter à l'administration WordPress
- Vérifier les Produits, Commandes ou Clients
- Confirmer que l'élément existe toujours

### 4. Vérifier les Filtres

L'enregistrement pourrait être filtré :
- Examiner les filtres actifs dans le POS
- Vérifier les filtres de catégorie
- Confirmer les paramètres de visibilité

### 5. Effacer et Resynchroniser

Si la synchronisation semble bloquée :
1. Effacer les données locales
2. Se reconnecter
3. Effectuer une nouvelle synchronisation

## Scénarios Courants

- **Produit non visible :** Vérifier s'il est publié et visible
- **Client non trouvé :** Peut ne pas avoir été synchronisé
- **Commande manquante :** Pourrait être hors de la plage de synchronisation
