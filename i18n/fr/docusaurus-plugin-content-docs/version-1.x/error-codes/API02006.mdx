# API02006 : Clé API invalide {#api02006-api-key-invalid}

## Ce que cela signifie {#what-this-means}

La clé API REST WooCommerce utilisée est invalide. Les clés API sont utilisées pour l'authentification serveur à serveur, et la clé fournie ne correspond à aucune clé valide dans WooCommerce.

## Causes courantes {#common-causes}

- **Clé supprimée** — La clé API a été retirée de WooCommerce
- **Erreur de frappe dans la clé** — La clé a été saisie incorrectement
- **Mauvaise paire de clés** — La clé de consommateur et le secret ne correspondent pas
- **Clé d'un site différent** — Utilisation de clés générées pour une autre installation

## Comment résoudre {#how-to-fix}

### 1. Vérifiez la clé API {#1-verify-the-api-key}

Dans l'administration WordPress → WooCommerce → Paramètres → Avancé → REST API :
1. Vérifiez si votre clé API existe
2. Vérifiez qu'elle n'a pas été révoquée
3. Notez les autorisations (lecture/écriture/lecture-écriture)

### 2. Générer de nouvelles clés API {#2-generate-new-api-keys}

Si la clé est manquante ou invalide :
1. Allez dans WooCommerce → Paramètres → Avancé → REST API
2. Cliquez sur "Ajouter une clé"
3. Saisissez une description (par exemple, "WCPOS")
4. Sélectionnez l'utilisateur
5. Choisissez les autorisations "Lecture/Écriture"
6. Cliquez sur "Générer la clé API"
7. **Copiez à la fois la clé de consommateur et le secret de consommateur** (affichés une seule fois !)

### 3. Mettre à jour la configuration du POS {#3-update-pos-configuration}

Entrez les nouvelles clés API dans le POS :
- Clé de consommateur (commence par `ck_`)
- Secret de consommateur (commence par `cs_`)

### 4. Vérifiez les autorisations de la clé {#4-check-key-permissions}

Assurez-vous que la clé dispose d'autorisations suffisantes :
- **Lecture** — Voir les données uniquement
- **Écriture** — Modifier les données uniquement
- **Lecture/Écriture** — Accès complet (recommandé pour le POS)
