# API02006 : Clé API Invalide

## Ce Que Cela Signifie

La clé de l'API REST WooCommerce utilisée est invalide. Les clés API sont utilisées pour l'authentification serveur à serveur, et la clé fournie ne correspond à aucune clé valide dans WooCommerce.

## Causes Courantes

- **Clé supprimée** — La clé API a été supprimée de WooCommerce
- **Erreur de frappe** — La clé a été saisie incorrectement
- **Paire de clés incorrecte** — La clé de consommation et le secret ne correspondent pas
- **Clé d'un autre site** — Utilisation de clés générées pour une autre installation

## Comment Résoudre

### 1. Vérifier la Clé API

Dans l'Administration WordPress → WooCommerce → Paramètres → Avancé → API REST :
1. Vérifiez si votre clé API existe
2. Assurez-vous qu'elle n'a pas été révoquée
3. Notez les autorisations (lecture/écriture/lecture-écriture)

### 2. Générer de Nouvelles Clés API

Si la clé est manquante ou invalide :
1. Allez à WooCommerce → Paramètres → Avancé → API REST
2. Cliquez sur "Ajouter une clé"
3. Entrez une description (par exemple, "WCPOS")
4. Sélectionnez l'utilisateur
5. Choisissez les autorisations "Lecture/Écriture"
6. Cliquez sur "Générer une clé API"
7. **Copiez la Clé de Consommation et le Secret de Consommation** (affichés une seule fois !)

### 3. Mettre à Jour la Configuration du POS

Entrez les nouvelles clés API dans le POS :
- Clé de Consommation (commence par `ck_`)
- Secret de Consommation (commence par `cs_`)

### 4. Vérifier les Autorisations de la Clé

Assurez-vous que la clé a des autorisations suffisantes :
- **Lecture** — Afficher les données uniquement
- **Écriture** — Modifier les données uniquement
- **Lecture/Écriture** — Accès complet (recommandé pour le POS)
