# API04002 : Code de Réponse Inattendu

## Ce Que Cela Signifie

Le serveur a renvoyé un code de statut HTTP qui n'était pas attendu pour cette requête. Les codes courants incluent 500 (erreur serveur), 403 (interdit), 404 (non trouvé), etc.

## Causes Courantes

- **500 Erreur Interne du Serveur** — PHP a planté ou a rencontré une erreur
- **403 Interdit** — Accès refusé par les paramètres de sécurité
- **404 Non Trouvé** — Le point de terminaison n'existe pas
- **502/503/504** — Problèmes de passerelle serveur ou de disponibilité

## Comment Résoudre

### Pour les Erreurs 500 (Erreur Serveur)

1. Vérifier les journaux d'erreurs PHP
2. Augmenter la limite de mémoire PHP
3. Rechercher des conflits de plugins
4. Consulter `wp-content/debug.log`

### Pour les Erreurs 403 (Interdit)

1. Vérifier les paramètres du plugin de sécurité (Wordfence, Sucuri, etc.)
2. Mettre le POS ou votre IP sur liste blanche
3. Vérifier les règles de blocage dans `.htaccess`
4. Vérifier que ModSecurity ne bloque pas les requêtes

### Pour les Erreurs 404 (Non Trouvé)

1. Vérifier que le plugin WCPOS est actif
2. Vider les permaliens WordPress (Paramètres → Permaliens → Enregistrer)
3. Vérifier si l'API REST est activée
4. Vérifier la configuration de l'URL

### Pour les Erreurs 502/503/504 (Problèmes de Passerelle)

1. Contacter votre hébergeur
2. Attendre que le serveur se rétablisse
3. Vérifier si le site est sous forte charge
4. Vérifier que le serveur fonctionne

### Dépannage Général

1. Essayer d'accéder directement à votre site
2. Vérifier le panneau de contrôle de l'hébergement
3. Examiner les journaux d'accès du serveur
4. Contacter le support de l'hébergement si nécessaire
