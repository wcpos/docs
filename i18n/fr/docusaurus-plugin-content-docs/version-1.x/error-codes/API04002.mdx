# API04002: Code de réponse inattendu

## Ce que cela signifie

Le serveur a retourné un code d'état HTTP qui n'était pas attendu pour cette requête. Les codes courants incluent 500 (erreur de serveur), 403 (interdit), 404 (non trouvé), etc.

## Causes courantes

- **500 Erreur interne du serveur** — PHP a planté ou a rencontré une erreur
- **403 Interdit** — Accès refusé par des paramètres de sécurité
- **404 Non trouvé** — Le point de terminaison n'existe pas
- **502/503/504** — Problèmes de passerelle ou de disponibilité du serveur

## Comment corriger

### Pour les erreurs 500 (Erreur du serveur)

1. Vérifiez les journaux d'erreurs PHP
2. Augmentez la limite de mémoire PHP
3. Recherchez des conflits de plugins
4. Vérifiez `wp-content/debug.log`

### Pour les erreurs 403 (Interdit)

1. Vérifiez les paramètres des plugins de sécurité (Wordfence, Sucuri, etc.)
2. Mettez sur liste blanche le POS ou votre adresse IP
3. Vérifiez le fichier `.htaccess` pour des règles de blocage
4. Vérifiez que ModSecurity ne bloque pas les requêtes

### Pour les erreurs 404 (Non trouvé)

1. Vérifiez que le plugin WCPOS est actif
2. Videz les permaliens de WordPress (Réglages → Permaliens → Enregistrer)
3. Vérifiez si l'API REST est activée
4. Vérifiez la configuration de l'URL

### Pour les erreurs 502/503/504 (Problèmes de passerelle)

1. Contactez votre fournisseur d'hébergement
2. Attendez que le serveur se rétablisse
3. Vérifiez si le site est sous une forte charge
4. Vérifiez que le serveur est en marche

### Dépannage général

1. Essayez d'accéder directement à votre site
2. Vérifiez le panneau de contrôle d'hébergement pour des problèmes
3. Passez en revue les journaux d'accès du serveur
4. Contactez le support d'hébergement si nécessaire
