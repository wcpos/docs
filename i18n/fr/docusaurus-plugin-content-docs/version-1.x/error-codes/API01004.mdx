# API01004 : Échec de la résolution DNS

## Ce que cela signifie

Le POS n'a pas pu convertir le nom de domaine de votre site web (par exemple, `yourstore.com`) en une adresse IP. Le DNS (Système de Noms de Domaine) est comme un annuaire téléphonique de l'internet : s'il ne peut pas trouver l'inscription, il ne peut pas établir la connexion.

## Causes courantes

- **Erreur de frappe dans le nom de domaine** — L'URL peut contenir une faute de frappe
- **Problèmes de serveur DNS** — Votre serveur DNS ne répond pas
- **Domaine expiré** — L'enregistrement du domaine peut avoir expiré
- **Propagation DNS** — Les changements DNS récents n'ont pas encore été propagés
- **Cache DNS local** — Informations DNS obsolètes mises en cache sur votre appareil

## Comment corriger

### 1. Vérifiez le nom de domaine

Vérifiez à nouveau l'URL pour les fautes de frappe. Essayez d'accéder au site dans un navigateur.

### 2. Vérifiez l'état du domaine

Utilisez une recherche WHOIS pour vérifier :
- Que le domaine est toujours enregistré
- Qu'il n'a pas expiré
- Que les enregistrements DNS sont configurés

### 3. Effacez le cache DNS

**Sur votre appareil :**

macOS :
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows :
```bash
ipconfig /flushdns
```

### 4. Essayez un DNS alternatif

Changez pour un serveur DNS public comme :
- Google : `8.8.8.8` et `8.8.4.4`
- Cloudflare : `1.1.1.1`

### 5. Attendez la propagation DNS

Si vous avez récemment changé les paramètres DNS, attendez 24 à 48 heures pour la propagation complète.
