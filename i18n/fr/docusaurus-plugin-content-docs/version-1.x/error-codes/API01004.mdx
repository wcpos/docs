# API01004 : Échec de la résolution DNS {#api01004-dns-resolution-failed}

## Ce que cela signifie {#what-this-means}

Le POS n'a pas pu convertir le nom de domaine de votre site web (par exemple, `yourstore.com`) en une adresse IP. Le DNS (Domain Name System) est comme l'annuaire téléphonique de l'internet : s'il ne peut pas trouver l'inscription, il ne peut pas établir la connexion.

## Causes courantes {#common-causes}

- **Erreur de frappe dans le nom de domaine** — L'URL peut avoir une faute de frappe
- **Problèmes de serveur DNS** — Votre serveur DNS ne répond pas
- **Domaine expiré** — L'enregistrement du domaine peut avoir expiré
- **Propagation DNS** — Les changements DNS récents n'ont pas encore été propagés
- **Cache DNS local** — Informations DNS obsolètes mises en cache sur votre appareil

## Comment corriger {#how-to-fix}

### 1. Vérifier le nom de domaine {#1-verify-the-domain-name}

Vérifiez soigneusement l'URL pour les fautes de frappe. Essayez d'accéder au site dans un navigateur.

### 2. Vérifier l'état du domaine {#2-check-domain-status}

Utilisez une recherche WHOIS pour vérifier :
- Que le domaine est toujours enregistré
- Qu'il n'a pas expiré
- Que les enregistrements DNS sont configurés

### 3. Effacer le cache DNS {#3-clear-dns-cache}

**Sur votre appareil :**

macOS :
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows :
```bash
ipconfig /flushdns
```

### 4. Essayer un DNS alternatif {#4-try-alternative-dns}

Changez pour un serveur DNS public comme :
- Google : `8.8.8.8` et `8.8.4.4`
- Cloudflare : `1.1.1.1`

### 5. Attendre la propagation DNS {#5-wait-for-dns-propagation}

Si vous avez récemment changé les paramètres DNS, attendez 24 à 48 heures pour la propagation complète.
