# API01004 : Échec de la résolution DNS

## Ce que cela signifie

Le système POS n'a pas pu convertir le nom de domaine de votre site web (par exemple, `votremagasin.com`) en adresse IP. Le DNS (Domain Name System) est comme l'annuaire téléphonique d'Internet — s'il ne trouve pas la liste, il ne peut pas établir la connexion.

## Causes courantes

- **Faute de frappe dans le nom de domaine** — L'URL peut contenir une erreur d'orthographe
- **Problèmes de serveur DNS** — Votre serveur DNS ne répond pas
- **Domaine expiré** — L'enregistrement du domaine a peut-être expiré
- **Propagation DNS** — Les modifications DNS récentes n'ont pas encore été propagées
- **Cache DNS local** — Informations DNS obsolètes en cache sur votre appareil

## Comment résoudre

### 1. Vérifier le nom de domaine

Vérifiez attentivement l'URL pour détecter toute faute de frappe. Essayez d'accéder au site dans un navigateur.

### 2. Vérifier le statut du domaine

Utilisez une recherche WHOIS pour vérifier :
- Le domaine est toujours enregistré
- Il n'a pas expiré
- Les enregistrements DNS sont configurés

### 3. Vider le cache DNS

**Sur votre appareil :**

macOS :
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows :
```bash
ipconfig /flushdns
```

### 4. Essayer un DNS alternatif

Passez à un serveur DNS public comme :
- Google : `8.8.8.8` et `8.8.4.4`
- Cloudflare : `1.1.1.1`

### 5. Attendre la propagation DNS

Si vous avez récemment modifié les paramètres DNS, attendez 24 à 48 heures pour une propagation complète.
