# API01003 : Réinitialisation de la connexion

## Ce que cela signifie

La connexion au serveur a été établie mais a été ensuite fermée de manière inattendue. Cela se produit généralement en cours de communication lorsqu'un problème interrompt la connexion.

## Causes courantes

- **Panne du serveur** — Le processus du serveur a pu se bloquer pendant la demande
- **Limites de mémoire** — PHP a manqué de mémoire et a été tué
- **Délai d'attente pendant la demande** — Le serveur a dépassé le délai d'attente lors du traitement
- **Instabilité du réseau** — Problèmes intermittents de réseau
- **Logiciel de sécurité** — Paramètres de sécurité agressifs fermant les connexions

## Comment corriger

### 1. Vérifiez les journaux d'erreurs du serveur

Consultez les journaux d'erreurs de WordPress ou le panneau de contrôle d'hébergement pour des erreurs liées autour de l'heure de la réinitialisation.

### 2. Augmentez la limite de mémoire PHP

Dans votre `wp-config.php` :

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. Vérifiez les conflits de plugins

Un plugin peut provoquer un crash de PHP :
- Désactivez temporairement les plugins
- Réactivez-les un par un pour trouver le coupable

### 4. Examinez les ressources du serveur

Contactez votre fournisseur d'hébergement pour vérifier :
- Les limites de processus PHP
- L'utilisation de la mémoire au moment de l'erreur
- Les journaux de stabilité du serveur

### 5. Testez la stabilité du réseau

Si vous êtes sur Wi-Fi, essayez une connexion filaire. Si le problème persiste sur différents réseaux, il est probable qu'il soit du côté du serveur.
