# API02003 : Token Invalide {#api02003-token-invalid}

## Ce que cela signifie {#what-this-means}

Le token d'authentification utilisé n'est pas reconnu par le serveur. Cela diffère d'un token expiré — ce token n'a jamais été valide ou a été révoqué.

## Causes courantes {#common-causes}

- **Token révoqué** — Un administrateur a révoqué la session
- **Token corrompu** — Corruption des données lors du stockage ou de la transmission
- **Réinitialisation du serveur** — Les clés secrètes du serveur ont changé, rendant tous les tokens invalides
- **Mauvais serveur** — Token d'une autre installation de WordPress

## Cartographie des erreurs serveur {#server-error-mapping}

Ce code d'erreur est déclenché lorsque le serveur retourne :

| Code serveur | Source |
|--------------|--------|
| `jwt_auth_invalid_token` | JWT Authentication plugin |

## Comment corriger {#how-to-fix}

### 1. Connectez-vous à nouveau {#1-log-in-again}

Effacez votre session et authentifiez-vous à nouveau :
1. Déconnectez-vous complètement
2. Fermez et rouvrez le POS
3. Connectez-vous avec vos identifiants

### 2. Effacez les données de l'application {#2-clear-application-data}

Si la déconnexion ne fonctionne pas :
- Vider le cache/données de l'application POS
- Sur le web : Effacer les cookies et le stockage local du navigateur pour le site
- Sur desktop : Vérifiez les paramètres de l'application pour une option "Clear Data"

### 3. Vérifiez la configuration du serveur {#3-check-server-configuration}

Si le problème affecte tous les utilisateurs :
- Vérifiez si les sels de WordPress ont été changés
- Vérifiez les paramètres du plugin JWT ou d'authentification
- Passez en revue les modifications récentes du serveur

### 4. Vérifiez que vous êtes sur le bon serveur {#4-verify-youre-on-the-correct-server}

Assurez-vous que le POS est configuré pour se connecter au bon site WordPress, surtout si vous avez plusieurs installations.
