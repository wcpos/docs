# API02003 : Jeton Non Valide

## Ce Que Cela Signifie

Le jeton d'authentification utilisé n'est pas reconnu par le serveur. Ceci est différent d'un jeton expiré — ce jeton n'a jamais été valide ou a été révoqué.

## Causes Courantes

- **Jeton révoqué** — Un administrateur a mis fin à la session
- **Jeton corrompu** — Corruption de données pendant le stockage ou la transmission
- **Réinitialisation du serveur** — Les clés secrètes du serveur ont changé, invalidant tous les jetons
- **Mauvais serveur** — Jeton provenant d'une autre installation WordPress

## Correspondance des Erreurs Serveur

Ce code d'erreur est déclenché lorsque le serveur renvoie :

| Code Serveur | Source |
|-------------|--------|
| `jwt_auth_invalid_token` | Plugin d'Authentification JWT |

## Comment Résoudre

### 1. Se Reconnecter

Effacez votre session et authentifiez-vous à nouveau :
1. Déconnectez-vous complètement
2. Fermez et rouvrez le POS
3. Connectez-vous avec vos identifiants

### 2. Effacer les Données de l'Application

Si la déconnexion ne fonctionne pas :
- Effacez le cache/données de l'application POS
- Sur web : Effacez les cookies et le stockage local du site
- Sur desktop : Vérifiez les paramètres de l'application pour une option "Effacer les Données"

### 3. Vérifier la Configuration du Serveur

Si le problème affecte tous les utilisateurs :
- Vérifiez si les sel WordPress ont été modifiés
- Vérifiez les paramètres du plugin JWT ou d'authentification
- Passez en revue les récents changements du serveur

### 4. Vérifier le Bon Serveur

Assurez-vous que le POS est configuré pour se connecter au bon site WordPress, en particulier si vous avez plusieurs installations.
