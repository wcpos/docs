## Aperçu

Lors du chargement initial, le POS récupère automatiquement les réglages par défaut à partir de la configuration de votre magasin WooCommerce. Dans la plupart des cas, vous voudrez garder ces réglages synchronisés avec votre magasin WooCommerce pour maintenir la cohérence sur tous les canaux de vente.

:::tip Meilleure Pratique
En général, les utilisateurs devraient maintenir les mêmes réglages que leur magasin WooCommerce, à moins qu'il n'y ait un besoin commercial spécifique de les personnaliser pour le POS, comme changer de devise pour un marché différent ou définir un client par défaut différent pour les transactions en magasin.
:::

## Informations sur le magasin

### Nom du magasin

Le nom de votre magasin tel qu'il apparaît dans l'interface POS et sur les reçus. Cela est généralement synchronisé avec le nom de votre magasin WooCommerce.

### Adresse du magasin

Les champs d'adresse du magasin comprennent :

- **Ville de base du magasin** : La ville où se trouve votre magasin
- **Code postal de base du magasin** : Le code postal pour l'emplacement de votre magasin
- **État de base du magasin** : L'état ou la province où votre magasin opère
- **Pays de base du magasin** : Le pays où se trouve votre magasin

:::warning Réglages de l'adresse désactivés
Les champs d'adresse du magasin sont actuellement désactivés dans les réglages POS car des changements dans l'adresse du magasin peuvent impacter les calculs de taxes et peuvent causer des problèmes avec le traitement des commandes. Ces réglages restent synchronisés avec la configuration de votre magasin WooCommerce.
:::

## Réglages de localisation

### Langue

Définit la langue d'affichage pour l'interface POS. Cela contrôle la langue utilisée pour :

- Les éléments de menu et la navigation
- Les étiquettes des boutons et le texte de l'interface
- Les messages et notifications du système
- Le formatage de la date et de l'heure

### Devise

Définit la devise principale utilisée pour les transactions et l'affichage des prix. Ce réglage affecte :

- L'affichage des prix des produits
- Les totaux et calculs de commande
- Le formatage des reçus
- Les rapports financiers

### Position de la devise

Contrôle où le symbole de la devise apparaît par rapport au montant du prix :

- **Gauche** : $10,00 (symbole avant le montant)
- **Droite** : 10,00$ (symbole après le montant)
- **Gauche avec espace** : $ 10,00
- **Droite avec espace** : 10,00 $

## Formatage des nombres

### Séparateur décimal

Le caractère utilisé pour séparer le nombre entier de la partie décimale (par exemple, "." ou ",").

### Nombre de décimales

Spécifie combien de chiffres décimaux afficher pour les prix et les valeurs monétaires (généralement 2 pour la plupart des devises).

### Séparateur de milliers

Le caractère utilisé pour séparer les milliers dans les grands nombres (par exemple, "," pour 1 000 ou "." pour 1.000).

### Style de regroupement des milliers

Détermine comment les milliers sont groupés et affichés :

- **mille** : Groupement occidental standard (123 456 789) - groupe tous les 3 chiffres
- **lakh** : Système de numération indien (12,34,56,789) - groupe les 3 premiers chiffres, puis tous les 2 chiffres
- **wan** : Système de numération est-asiatique (1,2345,6789) - groupe tous les 4 chiffres

## Réglages des clients

### Client par défaut

Définit le client qui est automatiquement sélectionné pour les nouvelles commandes. Les options incluent généralement :

- **Invité** : Pour les clients sans compte
- **Client spécifique** : Un compte client prédéfini
- **Caissier** : Le caissier connecté en tant que client

### Le client par défaut est le caissier

Lorsque cela est activé, le caissier actuellement connecté est automatiquement défini comme le client pour les nouvelles commandes.

## Restaurer les réglages du serveur

Le bouton "Restaurer les réglages du serveur" vous permet de réinitialiser tous les réglages généraux aux valeurs par défaut de votre magasin WooCommerce.

:::info Synchronisation
Cliquer sur "Restaurer les réglages du serveur" récupérera les valeurs par défaut de la boutique WooCommerce actuelle et remplacera tous les réglages personnalisés du POS.
:::
