:::warning Important
Au premier chargement, le PDV récupère les paramètres de taxe par défaut depuis WooCommerce. En général, il est recommandé de conserver les mêmes paramètres que WooCommerce, sauf en cas de besoin spécifique. Des paramètres de taxe différents de ceux du serveur entraîneront presque certainement des calculs de taxe différents entre le PDV et WooCommerce, ce qui peut provoquer des écarts et des problèmes de synchronisation des commandes.
:::

## Afficher tous les taux de taxe {#view-all-tax-rates}

Cliquez sur le bouton **Afficher tous les taux de taxe** pour voir tous les taux de taxe téléchargés sur le PDV. Celui-ci affiche :

- Noms des taux de taxe
- Pourcentages des taux de taxe
- Régions/emplacements applicables

Ceci est utile pour vérifier que les taux de taxe corrects sont chargés et pour résoudre les problèmes de calcul des taxes.

## Paramètres de calcul des taxes {#tax-calculation-settings}

### Prix saisis avec taxe {#prices-entered-with-tax}

Détermine si les prix des produits incluent la taxe :

| Option | Description |
|--------|-------------|
| **Oui** | Les prix des produits incluent la taxe. Le PDV calcule la taxe en partant du prix total |
| **Non** | Les prix des produits n'incluent pas la taxe. Le PDV ajoute la taxe en plus du prix saisi |

### Calculer la taxe en fonction de {#calculate-tax-based-on}

Détermine l'adresse utilisée pour le calcul des taxes :

| Option | Description |
|--------|-------------|
| **Adresse de livraison du client** | Taux de taxe basés sur le lieu de livraison de la commande |
| **Adresse de facturation du client** | Taux de taxe basés sur l'adresse de facturation du client |
| **Adresse de base de la boutique** | Taux de taxe basés sur l'emplacement physique de la boutique |

### Classe de taxe de livraison {#shipping-tax-class}

Détermine la taxation des frais de livraison :

| Option | Description |
|--------|-------------|
| **Classe de taxe de livraison basée sur les articles du panier** | La taxe de livraison est calculée en fonction de la classe de taxe des articles du panier |
| **Taux standard** | La livraison est taxée au taux de taxe standard |
| **Taux réduit** | La livraison est taxée à un taux de taxe réduit |
| **Taux zéro** | Aucune taxe n'est appliquée aux frais de livraison |

### Arrondi {#rounding}

| Option | Description |
|--------|-------------|
| **Arrondir la taxe au niveau du sous-total** | La taxe est calculée sur l'ensemble du sous-total puis arrondie, au lieu d'arrondir la taxe pour chaque article individuellement |

## Paramètres d'affichage {#display-settings}

### Affichage des prix dans la boutique {#display-prices-in-the-shop}

| Option | Description |
|--------|-------------|
| **Taxes incluses** | Les prix affichés incluent les taxes |
| **Hors taxes** | Les prix affichés n'incluent pas les taxes |

### Affichage des prix dans le panier et lors du paiement {#display-prices-during-cart-and-checkout}

| Option | Description |
|--------|-------------|
| **Taxes incluses** | Les prix du panier et du paiement incluent les taxes |
| **Hors taxes** | Les prix du panier et du paiement n'incluent pas les taxes |

### Suffixe d'affichage des prix {#price-display-suffix}

Texte affiché après les prix pour indiquer le statut fiscal (par exemple, « TTC » ou « HT »).

### Affichage des totaux de taxes {#display-tax-totals}

| Option | Description |
|--------|-------------|
| **Total unique** | Toutes les taxes sont regroupées en une seule ligne de total |
| **Détaillé** | Chaque taux de taxe est affiché sur une ligne distincte |

## Restaurer les paramètres du serveur {#restore-server-settings}

Cliquez pour récupérer et appliquer les paramètres de taxes WooCommerce actuels depuis votre serveur.

:::tip Quand et comment l'utiliser
WooCommerce est la source de référence pour les paramètres de taxes — le PDV doit les refléter. **Restaurer les paramètres du serveur** est le moyen sûr de récupérer la dernière configuration depuis votre boutique, et c'est l'action à effectuer chaque fois que vous avez modifié les paramètres de taxes dans `WP Admin > WooCommerce > Réglages > Tax` et souhaitez que le PDV les prenne en compte.

Le seul cas où il faut y réfléchir à deux fois : si vous avez **délibérément** modifié un paramètre de taxe côté PDV pour qu'il diffère du serveur (une configuration avancée), Restaurer les paramètres du serveur écrasera cette modification. Si vous n'êtes pas sûr de l'avoir fait, c'est très probablement que non.
:::

## Problèmes courants {#common-issues}

### Le PDV et WooCommerce affichent des totaux différents {#pos-and-woocommerce-showing-different-totals}

Le problème de taxe le plus fréquent est une différence entre les paramètres de taxe du PDV et ceux de WooCommerce. Lorsqu'ils divergent, la même commande donnera des totaux différents selon l'endroit où elle est passée. Gardez toujours les paramètres du PDV synchronisés — utilisez **Restaurer les paramètres du serveur** après toute modification dans `WP Admin > WooCommerce > Réglages > Tax`.

### Confusion liée aux prix TTC {#tax-inclusive-pricing-confusion}

Si « Prix saisis avec taxe » est défini sur **Oui** mais que « Afficher les prix dans la boutique » est défini sur **Hors taxe**, le prix affiché sera inférieur au prix saisi. C'est le comportement attendu — le PDV retire la part de taxe pour l'affichage — mais cela peut surprendre les caissiers qui s'attendent à voir le prix affiché sur l'étiquette.

### Adresse du client et calcul de la taxe {#customer-address-and-tax-calculation}

Lorsque « Calculer la taxe selon » est défini sur **Adresse de livraison du client** ou **Adresse de facturation du client**, le PDV a besoin d'une adresse valide pour déterminer le taux applicable. Si un client de passage n'a aucune adresse enregistrée, WCPOS utilise l'**adresse de base de la boutique** pour le calcul. C'est généralement le comportement approprié pour les ventes en magasin, mais si vous constatez des taux inattendus pour certains clients, vérifiez les détails de leur adresse.

## Dépannage {#troubleshooting}

### Les montants de taxe sont incorrects {#tax-amounts-are-incorrect}

1. Accédez aux **Paramètres de taxe** dans le PDV et cliquez sur **Restaurer les paramètres du serveur** pour récupérer la dernière configuration depuis WooCommerce.
2. Cliquez sur **Voir tous les taux de taxe** et confirmez que les taux attendus apparaissent. Si des taux sont manquants, vérifiez qu'ils sont correctement configurés dans `WP Admin > WooCommerce > Réglages > Tax > Standard rates`.
3. Vérifiez la classe de taxe du produit — les produits assignés à une classe de taxe « Taux zéro » ne seront pas taxés, quels que soient les paramètres.

### La taxe s'affiche à zéro {#tax-is-showing-as-zero}

- Vérifiez que la taxe est activée dans WooCommerce (`WP Admin > WooCommerce > Réglages > General > Enable tax rates and calculations`).
- Utilisez **Voir tous les taux de taxe** dans le PDV pour confirmer que les taux ont été téléchargés. Si la liste est vide, essayez une synchronisation complète.
- Vérifiez que le statut fiscal du produit est défini sur « Taxable » (et non « Aucun » ou « Livraison uniquement »).

### Écarts d'arrondi {#rounding-discrepancies}

De légères différences d'arrondi (un ou deux centimes) entre le PDV et WooCommerce peuvent survenir lorsque « Arrondir la taxe au niveau du sous-total » est activé dans un système mais pas dans l'autre. Assurez-vous que ce paramètre est identique dans les deux systèmes.
