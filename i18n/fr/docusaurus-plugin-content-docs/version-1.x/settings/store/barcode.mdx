## Comment fonctionne la détection de code-barres

La plupart des scanners de code-barres se comportent comme un clavier connecté à votre appareil. Lorsque vous scannez un code-barres, le POS détecte que les caractères ont été saisis plus rapidement qu'une saisie normale. Il utilise ces "appuis rapides sur les touches" pour identifier l'entrée comme un scan de code-barres.

## Paramètres

| Setting | Description |
|---------|-------------|
| **Average input time** | À quelle vitesse l'entrée doit être (en millisecondes) pour être considérée comme un code-barres. Valeurs plus basses = saisie plus rapide requise |
| **Minimum length** | Quelle doit être la longueur de la chaîne continue de caractères pour être traitée comme un code-barres |
| **Prefix removal** | Caractères à supprimer du début des codes-barres scannés |
| **Suffix removal** | Caractères à supprimer de la fin des codes-barres scannés |

## Suppression de préfixe/suffixe

Si votre scanner ajoute des caractères supplémentaires (comme un préfixe ou un suffixe), vous pouvez les supprimer automatiquement afin que seul le code-barres principal reste.

**Exemple :** Si votre scanner ajoute "A" avant chaque code-barres et "Z" après, configurez :
- Prefix removal : `A`
- Suffix removal : `Z`

## Dépannage

### Code-barres non détecté

- **Scanner trop lent** : Essayez d'augmenter la valeur du "Average input time"
- **Code-barres trop court** : Essayez de diminuer la valeur de la "Minimum length"
- **Caractères supplémentaires** : Vérifiez si votre scanner ajoute des préfixes/suffixes

### Saisie normale détectée comme code-barres

- **Saisie trop rapide** : Essayez de diminuer la valeur du "Average input time"
- **Entrée trop longue** : Essayez d'augmenter la valeur de la "Minimum length"
