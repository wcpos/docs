---
title: Accès au POS
sidebar_label: Accès
---

Pour contrôler l'accès aux fonctionnalités de WCPOS, vous pouvez utiliser les capacités et les rôles d'utilisateurs WordPress. En attribuant des rôles spécifiques aux utilisateurs et en configurant leurs capacités, vous pouvez adapter les niveaux d'accès pour différents utilisateurs du POS. Le plugin WCPOS s'intègre aux rôles WordPress existants et introduit un rôle spécifique pour les caissiers.

Le tableau suivant présente les principaux rôles d'utilisateurs dans le POS et leurs capacités respectives :

| Rôle           | Créé par      | Description                                                                                                                                                                                                                                                                                                                                                    |
| ------------- | --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Administrateur | WordPress       | Le propriétaire du site qui dispose de toutes les capacités activées par défaut.                                                                                                                                                                                                                                                                                |
| Gestionnaire de boutique  | WooCommerce     | Ce rôle est conçu pour les utilisateurs qui ont besoin d'accéder au POS et de gérer les paramètres du POS. Les gestionnaires de boutique peuvent ne pas avoir la permission d'installer des plugins ou d'effectuer d'autres tâches administratives.                                                                                                                 |
| Caissier       | WCPOS | Ce rôle est ajouté par le plugin WCPOS et est destiné aux utilisateurs qui utiliseront principalement le POS. Les caissiers ont généralement des capacités limitées et peuvent ne pas avoir accès à l'administration WordPress pour modifier les paramètres du POS, installer des plugins ou créer des articles de blog. Dans certains cas, vous pouvez même vouloir restreindre un caissier de l'ajout ou de la modification de clients dans le POS. |

Pour attribuer et gérer les rôles et capacités des utilisateurs, vous pouvez naviguer vers **`WP Admin > POS > Accès`**.

Le tableau suivant fournit un aperçu des capacités WordPress associées au plugin WCPOS :

| Capacité                   | Description                                                            | Exemple d'utilisation                                                                                                                                    |
| --------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `access_woocommerce_pos`    | Permet l'accès à l'interface WCPOS.                        | L'activation de cette option donnerait au caissier la permission d'accéder et d'utiliser le POS, lui permettant d'effectuer des transactions de point de vente.          |
| `manage_woocommerce_pos`    | Permet la gestion des paramètres et configurations de WCPOS. | L'activation de cette option donnerait au caissier la capacité de modifier les paramètres liés au POS, tels que les méthodes de paiement ou les modèles de reçus.     |
| `create_users`              | Permet au caissier de créer de nouveaux comptes d'utilisateurs WordPress.              | La désactivation de cette option empêcherait le caissier de créer de nouveaux comptes clients dans le POS.                                                  |
| `edit_others_shop_orders`   | Permet de modifier les commandes de boutique créées par d'autres caissiers.              | La désactivation de cette option empêcherait un caissier de modifier les commandes de boutique créées par un autre caissier dans le POS.                              |
| `edit_product`              | Permet au caissier de modifier les détails des produits.                            | La désactivation de cette option empêcherait le caissier de modifier les détails des produits disponibles dans le POS.                                     |
| `edit_published_products`   | Permet au caissier de modifier les produits publiés.                         | La désactivation de cette option empêcherait le caissier d'apporter des modifications aux produits déjà publiés dans le POS.                           |
| `edit_shop_orders`          | Permet au caissier de modifier les commandes de boutique.                                | La désactivation de cette option empêcherait le caissier de modifier les détails des commandes de boutique, tels que le statut de la commande ou les détails du client, dans le POS. |
| `edit_users`                | Permet au caissier de modifier les comptes d'utilisateurs existants.                     | La désactivation de cette option empêcherait le caissier de modifier les détails des comptes clients, tels que l'adresse de facturation, dans le POS.            |
| `list_users`                | Permet au caissier de voir une liste des comptes d'utilisateurs.                    | La désactivation de cette option empêcherait le caissier de rechercher ou de sélectionner des clients dans le POS.                                                |
| `manage_product_terms`      | Permet au caissier de gérer les catégories et les étiquettes de produits.              | La désactivation de cette option empêcherait le caissier de créer, modifier ou supprimer des catégories ou des étiquettes de produits dans le POS.                       |
| `publish_shop_orders`       | Permet au caissier de publier des commandes de boutique.                             | La désactivation de cette option empêcherait le caissier de finaliser et de traiter de nouvelles commandes de boutique dans le POS.                                       |
| `read_private_products`     | Permet au caissier d'accéder aux détails des produits privés.          | La désactivation de cette option empêcherait le caissier de voir les détails des produits privés disponibles dans le POS.                               |
| `read_private_shop_coupons` | Permet au caissier d'accéder aux détails des coupons de boutique privés.      | La désactivation de cette option empêcherait le caissier de voir les détails des coupons de boutique privés dans le POS.                                     |
| `read_private_shop_orders`  | Permet au caissier d'accéder aux détails des commandes de boutique privées.       | La désactivation de cette option empêcherait le caissier de voir les détails des commandes de boutique privées dans le POS.                                      |
| `read`                      | Permet l'accès à l'administration WordPress.                                  | La désactivation empêche le caissier d'accéder au tableau de bord WordPress et à la page de profil utilisateur.                                                   |

En attribuant les capacités appropriées à chaque rôle, vous pouvez affiner l'accès et les autorisations pour différents utilisateurs dans le POS. Cela vous permet de contrôler qui peut effectuer certaines actions, voir des informations sensibles et apporter des modifications à votre boutique.

Pour plus d'informations détaillées sur la gestion des rôles et des capacités des utilisateurs dans WordPress, vous pouvez vous référer au [WordPress Codex](https://codex.wordpress.org/Roles_and_Capabilities).
