## Activer les produits réservés au POS {#enable-pos-only-products}

Ce paramètre ajoute des options de visibilité à vos produits, vous permettant de contrôler s'ils apparaissent dans votre boutique en ligne, dans le POS, ou dans les deux.

:::warning Impact sur la performance
Activez ce paramètre uniquement si vous devez cacher des produits de votre boutique en ligne ou du POS. Lorsqu'il est activé, il ajoute une recherche supplémentaire dans la base de données pour chaque demande de produit.
:::

Pour des informations détaillées sur l'utilisation de cette fonctionnalité, y compris comment définir la visibilité sur des produits individuels et l'édition en masse, voir [Produits réservés au POS](/products/pos-only-products).

## Activer les quantités décimales {#enable-decimal-quantities}

Ce paramètre permet aux clients et au personnel d'entrer des quantités décimales lors de l'ajout de produits au panier.

**Exemples de quantités décimales :**
- 0,25 (un quart d'unité)
- 1,5 (une unité et demie)
- 2,75 (deux unités et trois quarts)

**Utile pour :**
- Produits vendus au poids (par exemple, 0,5 kg de grains de café)
- Services fractionnaires (par exemple, 1,5 heure de consultation)
- Produits en vrac pouvant être vendus en quantités partielles

## Générer automatiquement un nom d'utilisateur à partir de l'email du client {#automatically-generate-username-from-customer-email}

Lorsqu'il est activé, WCPOS créera automatiquement des noms d'utilisateur pour les nouveaux clients en fonction de leurs adresses e-mail lors du passage à la caisse. Cela simplifie le processus de création des clients et garantit une mise en forme cohérente des noms d'utilisateur.

## Client POS par défaut {#default-pos-customer}

Sélectionnez le client par défaut qui sera attribué aux nouvelles commandes POS. Les options courantes comprennent :

- **Invité** : Les commandes ne sont pas associées à aucun compte utilisateur
- **Caissier** : Les commandes sont associées au compte utilisateur du caissier connecté
- **Client de passage** : Les commandes sont associées à un compte client spécifique (vous devrez d'abord créer ce compte)

## Utiliser le compte de caissier {#use-cashier-account}

Lorsqu'il est activé, le compte utilisateur du caissier sera utilisé comme client par défaut pour les commandes POS au lieu du Client POS par défaut sélectionné.

## Champ de code-barres {#barcode-field}

Ce paramètre détermine quel champ de produit WCPOS recherchera lors de la numérisation des codes-barres ou en utilisant la fonctionnalité de recherche par code-barres.

:::warning Important : Effacement et actualisation requis
Après avoir modifié le paramètre du champ de code-barres, vous **devez** effacer et actualiser vos données produit dans le POS pour que le changement prenne effet.

**Comment effacer et actualiser :**
1. Allez à l'écran des Produits dans votre POS
2. **Appuyez longuement** sur l'icône de synchronisation (↻) en bas de l'écran
3. Sélectionnez **"Effacer et actualiser"** dans le menu contextuel
4. Attendez que les produits se rechargent avec les nouvelles données de champ de code-barres

Ceci est nécessaire car le POS met en cache les données de produit localement, et le changement du paramètre de champ de code-barres ne met pas automatiquement à jour les données en cache.
:::

### Option par défaut {#default-option}

**`_global_unique_id`** - Il s'agit du champ d'ID Unique Global de WooCommerce, introduit dans WooCommerce 8.2 (juillet 2024). Il est conçu pour stocker des codes GTIN, UPC, EAN ou ISBN et constitue l'option recommandée pour les nouvelles installations.

### Autres options courantes {#other-common-options}

**`_sku`** - Le champ SKU du produit (Stock Keeping Unit). C'est une bonne option de repli si vous n'utilisez pas le champ d'ID Unique Global.

### Clés Méta de Plugins Tierce Partie {#third-party-plugin-meta-keys}

Si vous utilisez un plugin de gestion des codes-barres, vous devrez peut-être utiliser leur clé méta spécifique :

| Type de plugin | Clés méta courantes | Description |
|----------------|---------------------|-------------|
| Plugins EAN | `_ean`, `_ean13`, `_ean8` | Codes de Numéro d'Article Européen |
| Plugins UPC | `_upc`, `_upc_code` | Code Produit Universel |
| Plugins GTIN | `_gtin`, `_gtin13`, `_gtin14` | Numéro d'Article Commercial Global |
| Code-barres générique | `_barcode`, `_product_barcode` | Champs de code-barres génériques |
| Codes-barres de produits WooCommerce | `_barcode_text` | Plugin de code-barres populaire |
| ATUM Inventory | `_atum_barcode` | Gestion des éléments d'inventaire ATUM |

### Trouver la clé méta de votre plugin {#finding-your-plugins-meta-key}

Pour trouver la clé méta correcte pour votre plugin de code-barres :

1. Éditez un produit dans l'admin WP
2. Ajoutez un code-barres en utilisant votre plugin
3. Consultez la documentation du plugin pour le nom de la clé méta
4. Ou inspectez la table de base de données `wp_postmeta` pour voir quelle clé méta a été créée

### Clés méta personnalisées {#custom-meta-keys}

Vous pouvez également utiliser n'importe quelle clé méta personnalisée qui stocke des données de code-barre. Il vous suffit d'entrer le nom de la clé méta (y compris le préfixe de soulignement si applicable) dans ce champ.

:::tip
Testez votre numérisation de code-barres après avoir changé ce paramètre pour vous assurer qu'elle fonctionne correctement avec le champ choisi. N'oubliez pas d'effacer et d'actualiser vos données produit dans le POS pour que le changement prenne effet.
:::
