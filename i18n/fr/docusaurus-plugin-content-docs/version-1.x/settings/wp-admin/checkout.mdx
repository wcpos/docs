La page Paramètres de caisse (`WP Admin > POS > Paramètres > Validation de la commande`) contrôle les passerelles de paiement affichées dans le PDV, le traitement des commandes et les e-mails de notification envoyés pour les ventes en PDV.

## Notifications par e-mail {#email-notifications}

En haut de la page se trouvent trois groupes de bascules d'e-mails — **E-mails administrateur**, **E-mails client** et **E-mails caissier** — qui contrôlent les e-mails de notification WooCommerce envoyés pour les commandes PDV, indépendamment des paramètres d'e-mails WooCommerce globaux du site.

Consultez **[Notifications par e-mail](/settings/wp-admin/email-notifications)** pour la référence complète, y compris les valeurs par défaut des bascules par e-mail, le modèle de destinataire caissier, les scénarios courants et les hooks développeur.

## Passerelles de paiement {#payment-gateways}

Cette section répertorie toutes les passerelles de paiement WooCommerce installées sur le site. Chaque passerelle peut être activée ou désactivée indépendamment pour le PDV.

| Colonne | Description |
|--------|-------------|
| **Activé** | Activer ou désactiver la passerelle pour la caisse PDV |
| **Passerelle** | Le nom de la passerelle de paiement |
| **Identifiant de passerelle** | L'identifiant de passerelle WooCommerce (utile pour les développeurs) |

### Activer une passerelle {#enabling-a-gateway}

1. Recherchez la passerelle dans la liste
2. Activez le commutateur **Activé**
3. La passerelle apparaîtra à l'encaissement du PDV au prochain chargement de page

### Réorganiser les passerelles {#reordering-gateways}

Glissez les passerelles pour modifier leur ordre d'affichage à l'encaissement du PDV. La première passerelle activée est sélectionnée par défaut lorsqu'un caissier ouvre la fenêtre d'encaissement.

### PDV vs boutique en ligne {#pos-vs-online-store}

Les paramètres de passerelle ici sont **indépendants** de vos paramètres de paiement WooCommerce. Une passerelle peut être :

- **Activée pour le PDV, désactivée en ligne** — utile pour les paiements en espèces qui ne s'appliquent pas aux commandes web
- **Activée en ligne, désactivée pour le PDV** — utile pour les passerelles comme PayPal qui ne sont pas adaptées aux ventes en personne
- **Activée pour les deux** — la passerelle fonctionne partout

:::note
Une passerelle doit d'abord être installée et active dans `WP Admin > WooCommerce > Réglages > Paiements` avant d'apparaître dans la liste des réglages de paiement du PDV. Si une passerelle est manquante, vérifiez que l'extension est installée et que la passerelle est activée dans WooCommerce.
:::

## Statut de commande {#order-status}

Configurez le statut de commande par défaut pour les commandes PDV terminées. Par défaut, les commandes PDV sont définies sur **Terminée** puisque le paiement est encaissé au point de vente.
