La page Paramètres de la caisse (`WP Admin > POS > Settings > Checkout`) détermine les passerelles de paiement qui apparaissent dans le POS ainsi que la manière dont les commandes sont traitées.

## Passerelles de paiement {#payment-gateways}

Cette section répertorie toutes les passerelles de paiement WooCommerce installées sur le site. Chaque passerelle peut être activée ou désactivée indépendamment pour le POS.

| Colonne | Description |
|--------|-------------|
| **Activée** | Activer ou désactiver la passerelle pour la caisse du POS |
| **Passerelle** | Le nom de la passerelle de paiement |
| **ID de la passerelle** | L'identifiant de la passerelle WooCommerce (utile pour les développeurs) |

### Activer une passerelle {#enabling-a-gateway}

1. Repérez la passerelle dans la liste
2. Activez le commutateur **Activé**
3. La passerelle apparaîtra dans le paiement du POS au prochain chargement de la page

### Réorganiser les passerelles {#reordering-gateways}

Faites glisser les passerelles pour modifier leur ordre d'affichage dans le paiement du POS. La première passerelle activée est sélectionnée par défaut lorsqu'un caissier ouvre la fenêtre de paiement.

### POS et boutique en ligne {#pos-vs-online-store}

Les paramètres des passerelles ici sont **indépendants** de vos paramètres de paiement WooCommerce. Une passerelle peut être :

- **Activée pour le POS, désactivée en ligne** — utile pour les paiements en espèces qui ne s'appliquent pas aux commandes web
- **Activée en ligne, désactivée pour le POS** — utile pour des passerelles comme PayPal qui ne sont pas pratiques pour les ventes en personne
- **Activée pour les deux** — la passerelle fonctionne partout

:::note
Une passerelle doit d'abord être installée et active dans `WP Admin > WooCommerce > Settings > Payments` avant d'apparaître dans la liste des paramètres de paiement du POS. Si une passerelle est absente, vérifiez que l'extension est installée et que la passerelle est activée dans WooCommerce.
:::

## Statut de la commande {#order-status}

Configurez le statut par défaut des commandes POS terminées. Par défaut, les commandes POS sont définies sur **Terminée** puisque le paiement est encaissé au point de vente.
