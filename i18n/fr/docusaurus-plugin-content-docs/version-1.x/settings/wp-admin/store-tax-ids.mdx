Les identifiants fiscaux de la boutique sont les identifiants fiscaux d'entreprise que WCPOS imprime sur vos reçus et factures du point de vente — votre numéro de TVA, ABN, GSTIN, EIN ou tout numéro de registre du commerce local.

WooCommerce ne dispose pas de champ intégré pour ces identifiants, et de nombreuses régions en nécessitent plusieurs. Une entreprise allemande peut imprimer un USt-IdNr., un Steuernummer **et** un numéro HRB ; une entreprise italienne imprime à la fois un Codice Fiscale et une Partita IVA ; de nombreuses entreprises affichent un numéro de TVA en plus d'un numéro d'immatriculation local. Le champ Identifiants fiscaux de la boutique permet d'ajouter autant d'entrées que nécessaire.

Ce réglage se trouve dans **WP Admin > PDV > Réglages > Général**, dans la section **Identifiants fiscaux de la boutique**.

{/* Screenshot: wp-admin-store-tax-ids.png */}

## Ajouter un identifiant fiscal {#adding-a-tax-id}

1. Accédez à **WP Admin > PDV > Réglages > Général** et faites défiler jusqu'à **Identifiants fiscaux de la boutique**.
2. Cliquez sur **Ajouter un identifiant fiscal**.
3. Choisissez un **Type**, saisissez la **Valeur**, et définissez éventuellement un **Pays** et un **Libellé** personnalisé.
4. Répétez l'opération pour chaque identifiant à imprimer.
5. Utilisez **Supprimer** pour retirer une ligne.

:::info Les lignes sont enregistrées automatiquement
Il n'y a pas de bouton Enregistrer pour cette section. Chaque ligne est enregistrée lorsque le champ **Valeur** perd le focus — cliquez ou tabulez hors du champ pour valider.
:::

### La première ligne utilise le pays de votre boutique {#the-first-row-uses-your-store-country}

Lorsque vous ajoutez votre premier identifiant fiscal, WCPOS pré-remplit le **Pays** avec le pays de base de votre WooCommerce et sélectionne le **Type** correspondant — ainsi, une entreprise qui n'a besoin que d'un seul identifiant fiscal local n'a rien à choisir manuellement. Les lignes existantes ne sont jamais modifiées.

Le type et le pays restent synchronisés lors de la modification : changer le **Type** met à jour le **Pays** vers le pays d'origine de ce type, et changer le **Pays** bascule le **Type** vers l'identifiant habituel de ce pays.

### Types disponibles {#available-types}

La liste des types couvre les identifiants courants destinés aux clients ainsi que les numéros de registre d'entreprise imprimés sur les reçus :

| Région | Types |
|--------|-------|
| Union européenne | EU VAT |
| Royaume-Uni | GB VAT, GB Company number |
| Allemagne | USt-IdNr., Steuernummer, HRB |
| France | SIRET, SIREN |
| Autres pays européens | Pays-Bas KVK, Italie Codice Fiscale, Italie Partita IVA, Espagne NIF, Suisse UID |
| Amériques | US EIN, Canada GST/HST, Brésil CPF, Brésil CNPJ, Argentine CUIT |
| Asie et Moyen-Orient | Inde GSTIN, Arabie saoudite VAT |
| Australie | ABN |
| Générique | Autre |

Le champ **Valeur** affiche un exemple de format en texte indicatif (par exemple `12-3456789` pour un US EIN). Il s'agit uniquement d'une indication — WCPOS ne rejette pas les valeurs qui ne correspondent pas au format attendu, de sorte que les identifiants inhabituels mais valides sont tout de même enregistrés.

## Ce qui apparaît sur les reçus {#what-prints-on-receipts}

Chaque numéro d'identification fiscale ajouté est imprimé sur les reçus et les factures, dans l'ordre, chacun sur sa propre ligne. WCPOS utilise un libellé localisé pour le type (« VAT ID », « ABN », « GSTIN », etc.) — ou votre **Libellé** personnalisé si vous en avez défini un. Si deux lignes contiennent la même valeur, le doublon est automatiquement supprimé.

Tous les modèles de reçus fournis affichent déjà la liste complète. Si vous utilisez un **modèle personnalisé** qui référence encore une seule valeur `store.tax_id`, il continuera de fonctionner mais n'imprimera que la première entrée. Modifiez-le pour parcourir la liste à la place :

```html
{{#store.tax_ids}}{{label}}: {{value}}{{/store.tax_ids}}
```

## Numéros d'identification fiscale par boutique (Pro) {#per-store-tax-ids-pro}

:::info Fonctionnalité Pro
Avec [WCPOS Pro](/getting-started/pro-license), vous pouvez définir des numéros d'identification fiscale **par boutique** au lieu d'une seule fois pour l'ensemble du site — utile lorsque chaque emplacement possède son propre numéro de TVA/EIN/GSTIN, ou lorsqu'une chaîne opère dans des régions avec des identifiants différents.
:::

Les numéros d'identification fiscale par boutique se configurent dans **WP Admin > PDV > Boutiques > Modifier la boutique**, dans la section **Numéros d'identification fiscale** de la barre latérale. Tant que la liste propre à une boutique est vide, elle utilise par défaut les numéros d'identification fiscale du site configurés ici dans l'onglet Général. Dès qu'une entrée est ajoutée à une boutique, celle-ci n'affiche que sa propre liste.
