WCPOS conserve un champ d'identifiant fiscal sur vos clients — numéro de TVA, ABN, CPF, GSTIN, NIF, etc. — pour les ventes B2B où l'identifiant fiscal de l'acheteur doit figurer sur la facture. Lorsqu'une commande est passée pour un client, WCPOS copie son identifiant fiscal sur la commande afin qu'il puisse être imprimé sur le reçu.

La section **Identifiants fiscaux** dans **WP Admin > PDV > Réglages > Général** contrôle la manière dont ce champ est détecté et stocké.

## Extensions et noms de champs {#plugins-and-field-names}

De nombreuses extensions WooCommerce ajoutent déjà un champ de TVA ou de numéro fiscal aux clients, et chacune utilise un nom de champ de base de données (clé meta) différent. Pour que vos données clients existantes continuent de fonctionner — sans migration — WCPOS détecte ce qui est installé et écrit automatiquement dans le champ approprié.

WCPOS résout le nom de champ dans l'ordre suivant :

1. **Une extension reconnue est active** — WCPOS écrit dans le champ de cette extension. Les extensions détectées incluent :
   - WooCommerce EU VAT Number
   - Aelia EU VAT Assistant (y compris son statut de vérification)
   - EU VAT for WooCommerce (WPFactory)
   - WooCommerce Germanized / Germanized Pro
   - Brazilian Market on WooCommerce / Extra Checkout Fields for Brazil
   - NIF/CIF Spain
2. **Aucun plugin reconnu** — WCPOS analyse les commandes récentes pour détecter le champ TVA/CPF/etc. déjà utilisé et écrit dans celui-ci.
3. **Aucun résultat** — WCPOS utilise des valeurs par défaut appropriées (`_billing_vat_number` pour les identifiants de type TVA, `_billing_cpf` / `_billing_cnpj` pour le Brésil, `_billing_gstin` pour l'Inde, etc.).

### Choisir soi-même le nom du champ {#choosing-the-field-name-yourself}

Pour remplacer le nom de champ détecté, accédez à **WP Admin > PDV > Réglages > Général**, ouvrez la section **Identifiants fiscaux** et développez **Avancé : utiliser un nom de champ différent**.

Un bandeau indique le plugin détecté automatiquement (ou « aucun plugin détecté », avec les valeurs par défaut intégrées). Le champ de remplacement propose une saisie semi-automatique suggérant les clés méta couramment utilisées par d'autres plugins. Videz le champ pour annuler le remplacement et revenir au nom de champ automatique.

:::note
Seuls les types d'identifiants fiscaux destinés aux clients apparaissent dans cet éditeur. Les identifiants de registre d'entreprise tels que l'USt-IdNr. allemand ou le numéro KVK néerlandais sont des champs au niveau de la boutique — voir [Identifiants fiscaux de la boutique](/settings/wp-admin/store-tax-ids).
:::

## Les identifiants fiscaux sont copiés sur les commandes {#tax-ids-are-copied-onto-orders}

Lors du passage d'une commande, les identifiants fiscaux du client sont **figés** sur la commande — celle-ci conserve les valeurs enregistrées au moment de la vente.

- Modifier ultérieurement l'identifiant fiscal d'un client ne modifie **pas** ses commandes passées.
- Un identifiant fiscal saisi par le caissier lors de l'encaissement remplace la valeur issue de la fiche client pour cette commande.
- La modification d'une commande existante ne change ses identifiants fiscaux que si la mise à jour les inclut explicitement ; WCPOS ne les recopie jamais depuis la fiche client.

Cela garantit l'exactitude des factures historiques même après la modification des coordonnées d'un client.

## Types pris en charge {#supported-types}

TVA UE, TVA GB, TVA SA, ABN australien, CPF / CNPJ brésilien, GSTIN indien, Codice Fiscale et Partita IVA italiens, NIF espagnol, CUIT argentin, GST/HST canadien et EIN américain — ainsi qu'un **identifiant fiscal** générique pour tout autre cas.

## Rechercher un client par identifiant fiscal {#finding-customers-by-tax-id}

La recherche de clients dans le PDV prend également en compte les identifiants fiscaux, y compris les anciens noms de champs du plugin, ce qui permet au caissier de retrouver un client par son numéro de TVA, CPF ou NIF.

## Sur les reçus {#on-receipts}

Les identifiants fiscaux des clients apparaissent sur les reçus affichant les coordonnées du client, avec le libellé localisé correspondant à chaque type. Les numéros de TVA sont préfixés du code pays (par exemple `DE123456789`).
