WCPOS conserve un champ d'identifiant fiscal pour vos clients — numéro de TVA, ABN, CPF, GSTIN, NIF, etc. — destiné aux ventes B2B où l'identifiant fiscal de l'acheteur doit figurer sur la facture. Lorsqu'une commande est passée pour un client, WCPOS copie son identifiant fiscal sur la commande afin qu'il puisse être imprimé sur le reçu.

La section **Identifiants fiscaux** dans **WP Admin > POS > Réglages > Général** contrôle la détection et le stockage de ce champ.

## Extensions et noms de champs {#plugins-and-field-names}

De nombreuses extensions WooCommerce ajoutent déjà un champ de TVA ou de numéro fiscal aux clients, et chacune utilise un nom de champ (clé meta) différent dans la base de données. Afin que vos données clients existantes continuent de fonctionner — sans aucune migration — WCPOS détecte ce qui est installé et écrit automatiquement dans le bon champ.

WCPOS résout le nom du champ dans cet ordre :

1. **Une extension reconnue est active** — WCPOS écrit dans le champ de cette extension. Les extensions détectées incluent :
   - WooCommerce EU VAT Number
   - Aelia EU VAT Assistant (y compris son statut de vérification)
   - EU VAT for WooCommerce (WPFactory)
   - WooCommerce Germanized / Germanized Pro
   - Brazilian Market on WooCommerce / Extra Checkout Fields for Brazil
   - NIF/CIF Spain
2. **Aucune extension reconnue** — WCPOS analyse les commandes récentes pour identifier le champ TVA/CPF/etc. déjà utilisé et écrit dans celui-ci.
3. **Aucun résultat trouvé** — WCPOS utilise des valeurs par défaut appropriées (`_billing_vat_number` pour les identifiants de type TVA, `_billing_cpf` / `_billing_cnpj` pour le Brésil, `_billing_gstin` pour l'Inde, etc.).

### Choisir le nom du champ manuellement {#choosing-the-field-name-yourself}

Pour remplacer le nom de champ détecté, accédez à **WP Admin > PDV > Réglages > Général**, ouvrez la section **Identifiants fiscaux** et développez **Avancé : utiliser un nom de champ différent**.

Un bandeau indique quelle extension a été détectée automatiquement (ou « aucune extension détectée », avec les valeurs par défaut intégrées). Le champ de remplacement dispose d'une saisie semi-automatique qui suggère les clés méta couramment utilisées par d'autres extensions. Videz le champ pour supprimer votre remplacement et revenir au nom de champ automatique.

:::note
Seuls les types d'identifiants fiscaux côté client apparaissent dans cet éditeur. Les identifiants de registre commercial tels que le USt-IdNr. allemand ou le numéro KVK néerlandais sont des champs au niveau de la boutique — consultez [Identifiants fiscaux de la boutique](/settings/wp-admin/store-tax-ids).
:::

## Les identifiants fiscaux sont copiés sur les commandes {#tax-ids-are-copied-onto-orders}

Lors du passage d'une commande, les identifiants fiscaux du client sont **figés** sur la commande — celle-ci conserve les valeurs enregistrées au moment de la vente.

- Modifier ultérieurement l'identifiant fiscal d'un client ne **modifie pas** ses commandes passées.
- Un identifiant fiscal saisi par le caissier lors de l'encaissement remplace la valeur de la fiche client pour cette commande.
- La modification d'une commande existante ne change ses identifiants fiscaux que si la mise à jour les inclut explicitement ; WCPOS ne les recopie jamais depuis la fiche client.

Cela garantit l'exactitude des factures historiques même après modification des coordonnées du client.

## Types pris en charge {#supported-types}

TVA UE, TVA GB, TVA SA, ABN australien, CPF / CNPJ brésilien, GSTIN indien, Codice Fiscale et Partita IVA italiens, NIF espagnol, CUIT argentin, GST/HST canadien et EIN américain — ainsi qu'un **identifiant fiscal** générique pour tout autre cas.

## Recherche de clients par identifiant fiscal {#finding-customers-by-tax-id}

La recherche de clients dans le PDV prend également en compte les identifiants fiscaux, y compris les anciens noms de champs du plugin, ce qui permet au caissier de retrouver un client par son numéro de TVA, CPF ou NIF.

## Sur les reçus {#on-receipts}

Les identifiants fiscaux des clients apparaissent sur les reçus qui affichent les informations client, avec le libellé localisé correspondant à chaque type. Les numéros de TVA sont préfixés du code pays (par exemple `DE123456789`).
