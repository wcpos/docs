WCPOS conserve un champ d'identifiant fiscal pour vos clients — un numéro de TVA, ABN, CPF, GSTIN, NIF, etc. — pour les ventes B2B où l'identifiant fiscal de l'acheteur doit apparaître sur la facture. Lorsqu'une commande est passée pour un client, WCPOS copie son identifiant fiscal sur la commande afin qu'il puisse être imprimé sur le reçu.

La section **Identifiants fiscaux** dans **WP Admin > POS > Réglages > Général** contrôle la manière dont ce champ est détecté et stocké.

## Extensions et noms de champs {#plugins-and-field-names}

De nombreuses extensions WooCommerce ajoutent déjà un champ de TVA ou de numéro fiscal aux clients, et elles utilisent toutes des noms de champs de base de données différents (clés méta) pour celui-ci. Pour que vos données client existantes continuent de fonctionner — sans migration — WCPOS détecte ce qui est installé et écrit automatiquement dans le bon champ.

WCPOS résout le nom du champ dans cet ordre :

1. **Une extension reconnue est active** — WCPOS réécrit dans le champ de cette extension. Les extensions détectées incluent :
   - WooCommerce EU VAT Number
   - Aelia EU VAT Assistant (y compris son statut de vérification)
   - EU VAT for WooCommerce (WPFactory)
   - WooCommerce Germanized / Germanized Pro
   - Brazilian Market on WooCommerce / Extra Checkout Fields for Brazil
   - NIF/CIF Spain
2. **Aucune extension reconnue** — WCPOS analyse les commandes récentes pour trouver le champ TVA/CPF/etc. déjà utilisé et y écrit les données.
3. **Rien trouvé** — WCPOS se rabat sur des valeurs par défaut pertinentes (`_billing_vat_number` pour les identifiants de type TVA, `_billing_cpf` / `_billing_cnpj` pour le Brésil, `_billing_gstin` pour l’Inde, etc.).

### Choisir soi-même le nom du champ {#choosing-the-field-name-yourself}

Pour remplacer le nom de champ détecté, accédez à **WP Admin > POS > Réglages > Général**, ouvrez la section **Identifiants fiscaux**, puis déployez **Avancé : utiliser un nom de champ différent**.

Une bannière indique quelle extension a été détectée automatiquement (ou « aucune extension détectée », avec les valeurs par défaut intégrées utilisées). Le champ de remplacement propose une saisie semi-automatique avec les clés méta couramment utilisées par d’autres extensions. Effacez le champ pour supprimer le remplacement et revenir au nom de champ automatique.

:::note
Seuls les types d’identifiants fiscaux destinés aux clients apparaissent dans cet éditeur. Les identifiants de registre des entreprises, comme l’USt-IdNr. allemand ou le numéro KVK néerlandais, sont des champs au niveau de la boutique — consultez [Identifiants fiscaux de la boutique](/settings/wp-admin/store-tax-ids).
:::

## Les identifiants fiscaux sont copiés dans les commandes {#tax-ids-are-copied-onto-orders}

Lorsqu’une commande est passée, les identifiants fiscaux du client sont **capturés sous forme d’instantané** dans la commande — celle-ci conserve les valeurs enregistrées au moment de la vente.

- Modifier ultérieurement l’identifiant fiscal d’un client ne réécrit **pas** ses commandes passées.
- Un identifiant fiscal saisi par un caissier lors du passage en caisse remplace la valeur de la fiche client pour cette commande.
- Modifier une commande existante ne change ses identifiants fiscaux que si la mise à jour les inclut explicitement ; WCPOS ne les recopie jamais depuis la fiche client.

Cela permet de conserver des factures historiques exactes, même après la modification des informations d’un client.

## Types pris en charge {#supported-types}

TVA UE, TVA GB, TVA SA, ABN australien, CPF / CNPJ brésilien, GSTIN indien, Codice Fiscale et Partita IVA italiens, NIF espagnol, CUIT argentin, GST/HST canadienne et EIN américain — ainsi qu’un **identifiant fiscal** générique pour tout le reste.

## Recherche de clients par identifiant fiscal {#finding-customers-by-tax-id}

La recherche de clients dans le POS correspond également aux valeurs d’identifiant fiscal, y compris les anciens noms de champs de l’extension, afin qu’un caissier puisse retrouver un client par son numéro de TVA, son CPF ou son NIF.

## Sur les reçus {#on-receipts}

Les identifiants fiscaux des clients apparaissent sur les reçus qui affichent les informations client, avec le libellé localisé approprié pour chaque type. Les numéros de TVA sont préfixés par le code pays (par exemple `DE123456789`).
