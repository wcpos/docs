L'**API REST WooCommerce** est comme un ensemble de "canaux" standardisés qui permet aux propriétaires de boutiques de connecter leur magasin WooCommerce à d'autres applications et services.

Voici une explication simple de son fonctionnement :

1. Elle agit comme un pont entre votre boutique WooCommerce et des applications externes, permettant leur communication et le partage de données.

2. L'API fonctionne en envoyant des requêtes à des **points de terminaison** spécifiques - pensez à ces points comme des "canaux" pour différents types de données de magasin. Par exemple, il existe des points de terminaison pour les produits, les commandes et les clients.

3. Les applications externes peuvent utiliser ces points de terminaison pour :

- Obtenir des informations (comme récupérer une liste de produits)
- Ajouter de nouvelles données (comme créer une nouvelle commande)
- Mettre à jour des informations existantes (comme modifier le prix d'un produit)
- Supprimer des données (comme retirer un produit obsolète)

Au lieu de s'appuyer sur des hooks et des filtres PHP (comme le font de nombreux plugins WordPress traditionnels), elle fonctionne en envoyant et recevant des données dans un format structuré et prévisible, appelé **JSON**.

## Pourquoi Certains Plugins Ne Fonctionnent Pas dans le POS ?

WCPOS télécharge les données de produits, de commandes et de clients depuis votre boutique WooCommerce via l'API REST.
Il conserve ensuite une copie locale de ces données pour pouvoir continuer à fonctionner même avec une connexion internet instable.
Cette approche rend le POS plus rapide et plus fiable dans un environnement de caisse occupé.

Cependant, cela signifie que notre POS ne "voit" pas les changements qui se produisent dynamiquement sur votre site en temps réel. Par exemple :

<table>
  <tr>
    <th style={{ width: "200px" }}>Type de Plugin</th>
    <th>Pourquoi Ça Ne Fonctionne Pas</th>
  </tr>
  <tr>
    <td>Tarification Dynamique</td>
    <td>
      Si un autre plugin de votre boutique calcule un prix unique pour chaque
      client connecté, le POS n'obtiendra pas automatiquement ce prix personnalisé.
    </td>
  </tr>
  <tr>
    <td>Modules Complémentaires de Produits</td>
    <td>
      Si un autre plugin de votre boutique ajoute des données de produit supplémentaires, 
      le POS n'obtiendra pas automatiquement ces données. Le POS ne reçoit que les données 
      exposées via l'API REST.
    </td>
  </tr>
</table>
