WCPOS inclut un système complet de modèles pour les reçus, factures, reçus cadeaux, bordereaux d'expédition, tickets de cuisine et plus encore — le tout géré dans **WP Admin → PDV → Modèles**.

:::tip Vous souhaitez simplement modifier votre reçu ?
Commencez par **[Personnaliser votre reçu](/templates/customise)**. Ce guide présente les trois méthodes les plus simples : choisir un modèle fourni différent, demander à l'IA d'en ajuster un ou le modifier manuellement.
:::

## Moteurs de modèles {#template-engines}

WCPOS prend en charge trois moteurs de modèles. Le moteur est choisi lors de la création d'un modèle ou en cliquant sur **Utiliser le modèle** sur une carte de la galerie, et **ne peut pas être modifié par la suite** — créez un nouveau modèle si vous souhaitez changer de moteur.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML sans logique (recommandé)">
    Style Mustache <code>{"{{variable}}"}</code> des espaces réservés dans du HTML simple. Le rendu s'effectue côté client, les reçus **fonctionnent hors ligne**. À utiliser pour l'impression navigateur et les reçus PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML thermique">
    Modèles XML qui produisent à la fois un aperçu à l'écran et des **commandes ESC/POS** pour les imprimantes thermiques de reçus. Un seul modèle, deux sorties. À utiliser pour les imprimantes thermiques Epson, Star et autres.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP hérité">
    Modèles PHP côté serveur utilisant les fonctions WooCommerce. Conservés pour la compatibilité ascendante avec les <code>yourtheme/woocommerce-pos/receipt.php</code> surcharges existantes. Nécessite une connexion au serveur pour le rendu.
  </FeatureCard>
</FeatureGrid>

## Galerie de modèles {#template-gallery}

La galerie propose des modèles prêts à l'emploi, utilisables tels quels ou personnalisables. Filtrez par **catégorie** (Reçu, Facture, Reçu cadeau, Ticket cuisine, Devis / Bon de commande), **format** (HTML / ESC/POS) et **direction** (LTR / RTL) ; prévisualisez avec les données réelles de votre boutique. Le pied de chaque carte contient un lien **Aperçu** et un bouton principal **Utiliser le modèle** — cliquer sur Utiliser le modèle crée une copie modifiable dans *Vos modèles*.

| Modèle | Moteur | Description |
|--------|--------|-------------|
| Reçu standard | HTML | Reçu par défaut — logo, identité de la boutique, articles, totaux, paiement |
| Reçu standard (RTL) | HTML | Version droite-à-gauche pour l'arabe, l'hébreu, le persan et l'ourdou |
| Minimal / Moderne | HTML | Mêmes informations essentielles que le reçu standard, dans un format plus compact |
| Reçu détaillé | HTML | Facture fiscale complète — colonne SKU, prix unitaire, ventilation des taxes par taux, adresses |
| Reçu cadeau | HTML | Articles uniquement, prix masqués. Inclut le message cadeau et la politique de retour |
| Facture | HTML | Facture pleine page A4/Letter avec panneau optionnel « Comment payer » pour les commandes impayées |
| Bon de livraison | HTML | Document d'entrepôt — articles + quantités, adresse de livraison, sans prix |
| Devis / Estimation | HTML | Document de pré-vente avec tarification et conditions — sans section de paiement |
| Reçu étroit | HTML | Reçu monospace pour papier étroit ou imprimantes thermiques compatibles HTML |
| Reçu thermique simple (58 mm) | Thermal | Mise en page thermique 58 mm épurée |
| Reçu thermique simple (80 mm) | Thermal | Mise en page thermique 80 mm épurée — le plus courant |
| Reçu thermique simple 80 mm (RTL) | Thermal | Version RTL pour 80 mm. Nécessite une imprimante avec une page de code arabe (CP864 / Windows-1256) |
| Reçu thermique détaillé (58 mm) | Thermal | 58 mm tout inclus — adresses, ventilation des taxes, remboursements, paiements, conditions, code-barres |
| Reçu thermique détaillé (80 mm) | Thermique | 80 mm complet — adresses, ventilation des taxes, remboursements, paiements, conditions, code-barres |
| Ticket de cuisine | Thermique | Articles uniquement, grande police, sans prix — pour les postes de préparation |

### Modèles de documents tiers {#third-party-document-templates}

Si **PDF Invoices & Packing Slips for WooCommerce** de WP Overnight est actif, WCPOS propose également **Invoice (WP Overnight)** et **Packing Slip (WP Overnight)** dans la liste des modèles de reçus. Ces modèles HTML rendus côté serveur délèguent à l'API de documents de WP Overnight, réutilisant la numérotation des factures et des bordereaux d'expédition, la mise en page, l'image de marque, les champs légaux/fiscaux et les personnalisations de modèles configurées dans cette extension.

Ils ne sont disponibles que lorsque l'extension WP Overnight est active et nécessitent une connexion au serveur pour le rendu. Utilisez les modèles HTML ou thermiques intégrés à WCPOS lorsque l'impression hors ligne est requise.

### Affichage adaptatif des taxes {#adaptive-tax-display}

La plupart des modèles intégrés (Standard, Standard RTL, Minimal/Modern, Narrow, Invoice, Quote, variantes Thermal Simple) **s'adaptent automatiquement au paramètre d'affichage des taxes de WooCommerce** :

- **Boutiques taxes incluses** (UE/Royaume-Uni/Australie) : les prix TTC sont affichés avec une ligne « Taxes incluses ».
- **Boutiques hors taxes** (États-Unis/Canada) : les prix HT sont affichés avec les taxes ajoutées sur une ligne distincte.
- Le total général affiche toujours le montant TTC effectivement facturé.

La famille **Détaillé** est une facture fiscale formelle et détaille toujours les taxes avec des lignes hors taxes et une ventilation explicite, quel que soit le paramètre de la boutique.

## Fonctionnement {#how-it-works}

1. **Ajouter un modèle** — cliquez sur **Utiliser le modèle** sur une carte de la galerie, ou créez le vôtre de zéro.
2. **Personnaliser** à l'aide de l'éditeur intégré avec aperçu en direct, ou collez-le dans ChatGPT / Claude et demandez les modifications souhaitées.
3. **Activer** — basculez le bouton **Actif** dans *Vos modèles*. Chaque modèle de reçu actif apparaît dans le menu déroulant de l'écran des reçus à la caisse.
4. **Imprimer ou afficher** — les reçus s'ouvrent après le passage en caisse avec des options pour imprimer, envoyer par e-mail ou afficher à l'écran.

Les modèles sont générés à partir d'un [payload de données de reçu](/templates/receipt-data) standardisé comprenant des sections pour les informations de la boutique, les articles, les totaux, les taxes, les paiements, les remboursements, et plus encore. Les champs de devise incluent tous des variantes pré-formatées `_display` (par ex., `$29.99` au lieu de `29.99`).

## Fonctionnalités Pro {#pro-features}

Avec [WCPOS Pro](/getting-started/pro-license), chaque boutique peut disposer de ses propres attributions de modèles, de son identité visuelle (logo, adresse, coordonnées) et de l'ordre des modèles — utile pour les opérations multi-sites où chaque point de vente imprime son propre en-tête.
