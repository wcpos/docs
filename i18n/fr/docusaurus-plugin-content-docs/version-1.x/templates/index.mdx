WCPOS inclut un système complet de templates pour les reçus, factures, reçus cadeaux, bons de livraison, tickets de cuisine et plus encore, le tout géré dans **WP Admin → POS → Templates**.

:::tip Vous voulez seulement modifier votre reçu ?
Commencez par **[Personnaliser votre reçu](/templates/customise)**. Cette page présente les trois façons les plus simples de le faire : choisir un autre template inclus, demander à l'IA de l'ajuster ou le modifier à la main.
:::

## Moteurs de templates

WCPOS prend en charge trois moteurs de templates. Le moteur est choisi lorsque vous créez un template ou cliquez sur **Utiliser le template** sur une carte de la galerie, et **il ne peut plus être modifié ensuite**. Créez un nouveau template si vous voulez changer de moteur.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML sans logique (recommandé)">
    Des espaces réservés de style Mustache <code>{"{{variable}}"}</code> dans du HTML simple. Le rendu se fait côté client, donc les reçus **fonctionnent hors ligne**. À utiliser pour l'impression navigateur et les reçus PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML thermique">
    Des templates XML qui produisent à la fois un aperçu à l'écran et des **commandes ESC/POS** pour les imprimantes thermiques de reçus. Un même template, deux sorties. À utiliser avec Epson, Star et les autres imprimantes thermiques.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP hérité">
    Des templates PHP côté serveur utilisant les fonctions WooCommerce. Conservés pour la compatibilité avec les remplacements existants dans <code>yourtheme/woocommerce-pos/receipt.php</code>. Nécessite une connexion au serveur pour le rendu.
  </FeatureCard>
</FeatureGrid>

## Galerie de templates

La galerie fournit des templates prêts à l'emploi que vous pouvez utiliser tels quels ou personnaliser. Filtrez par **catégorie** (Reçu, Facture, Reçu cadeau, Ticket de cuisine, Devis / Bon de commande), **format** (HTML / ESC/POS) et **direction** (LTR / RTL) ; l'aperçu utilise les données réelles de votre boutique. Le pied de chaque carte contient un lien **Aperçu** et un bouton principal **Utiliser le template**. Cliquer sur Utiliser le template crée une copie modifiable dans *Vos templates*.

| Template | Moteur | Description |
|----------|--------|-------------|
| Reçu standard | HTML | Reçu par défaut : logo, identité de la boutique, articles, totaux, paiement |
| Reçu standard (RTL) | HTML | Équivalent de droite à gauche pour l'arabe, l'hébreu, le persan et l'ourdou |
| Minimal / Moderne | HTML | Les mêmes éléments essentiels que le standard, avec moins de hauteur |
| Reçu détaillé | HTML | Facture fiscale complète : colonne SKU, prix unitaire, taxes par taux, adresses |
| Reçu cadeau | HTML | Articles uniquement, prix masqués. Inclut un message cadeau et la politique de retour |
| Facture | HTML | Facture pleine page A4/Letter avec panneau facultatif "Comment payer" pour les commandes impayées |
| Bon de livraison | HTML | Document d'entrepôt : articles + quantités, adresse d'expédition, sans prix |
| Devis / Estimation | HTML | Document avant-vente avec prix et conditions, sans section de paiement |
| Reçu étroit | HTML | Reçu en police monospace pour papier étroit ou imprimantes thermiques compatibles HTML |
| Reçu thermique simple (58 mm) | Thermique | Mise en page thermique propre en 58 mm |
| Reçu thermique simple (80 mm) | Thermique | Mise en page thermique propre en 80 mm, la plus courante |
| Reçu thermique simple 80 mm (RTL) | Thermique | Version RTL pour 80 mm. Nécessite une imprimante avec page de code arabe (CP864 / Windows-1256) |
| Reçu thermique détaillé (58 mm) | Thermique | 58 mm complet : adresses, taxes, remboursements, paiements, conditions, code-barres |
| Reçu thermique détaillé (80 mm) | Thermique | 80 mm complet : adresses, taxes, remboursements, paiements, conditions, code-barres |
| Ticket de cuisine | Thermique | Articles uniquement, grande police, sans prix, pour les postes de préparation |

### Templates de documents tiers

Si **PDF Invoices & Packing Slips for WooCommerce** de WP Overnight est actif, WCPOS expose aussi **Facture (WP Overnight)** et **Bon de livraison (WP Overnight)** dans la liste des templates de reçus. Ces templates HTML rendus côté serveur délèguent à l'API de documents de WP Overnight et réutilisent la numérotation, la mise en page, la marque, les champs légaux/fiscaux et les personnalisations configurées dans cette extension.

Ils sont disponibles uniquement lorsque l'extension WP Overnight est active et nécessitent une connexion au serveur pour le rendu. Utilisez les templates HTML ou thermiques inclus dans WCPOS lorsque l'impression hors ligne est requise.

### Affichage adaptatif des taxes

La plupart des templates inclus (Standard, Standard RTL, Minimal/Moderne, Étroit, Facture, Devis, variantes thermiques simples) **s'adaptent automatiquement au réglage d'affichage des taxes de WooCommerce** :

- Les **boutiques avec taxes incluses** (UE/Royaume-Uni/Australie) voient des prix bruts avec une ligne "Taxes incluses".
- Les **boutiques avec taxes exclues** (États-Unis/Canada) voient des prix nets avec la taxe ajoutée sur une ligne séparée.
- Le total général affiche toujours le montant brut réellement facturé.

La famille **Détaillé** est une facture fiscale formelle et détaille toujours les taxes avec des lignes hors taxes et un récapitulatif explicite, quel que soit le réglage de la boutique.

## Fonctionnement

1. **Ajouter un template** : cliquez sur **Utiliser le template** sur une carte de la galerie, ou créez le vôtre de zéro.
2. **Le personnaliser** avec l'éditeur intégré et son aperçu en direct, ou collez-le dans ChatGPT / Claude et demandez les changements souhaités.
3. **L'activer** : activez le bouton **Actif** dans *Vos templates*. Chaque template de reçu actif apparaît dans la liste déroulante de l'écran de reçu à la caisse.
4. **Imprimer ou afficher** : les reçus s'ouvrent après l'encaissement avec des options pour imprimer, envoyer par e-mail ou afficher à l'écran.

Les templates sont rendus à partir d'un [payload de données de reçu](/templates/receipt-data) standardisé avec des sections pour les informations de la boutique, les lignes d'articles, les totaux, les taxes, les paiements, les remboursements et plus encore. Tous les champs monétaires incluent des variantes `_display` déjà formatées, par exemple `$29.99` au lieu de `29.99`.

## Fonctionnalités Pro

Avec [WCPOS Pro](/getting-started/pro-license), chaque boutique peut avoir ses propres affectations de templates, sa marque (logo, adresse, coordonnées) et son ordre de templates. C'est utile pour les opérations multi-sites où chaque magasin imprime son propre en-tête.
