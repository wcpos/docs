WCPOS inclut un système complet de modèles pour les reçus, les factures, les reçus cadeaux, les bons de préparation, les tickets cuisine et plus encore — tous gérés dans **WP Admin → POS → Modèles**.

:::tip Vous souhaitez simplement modifier votre reçu ?
Commencez par **[Personnaliser votre reçu](/receipts/customise)**. Cette page présente les trois méthodes les plus simples pour le faire : choisir un autre modèle inclus, demander à l’IA d’en ajuster un, ou le modifier à la main.
:::

## Moteurs de modèles {#template-engines}

WCPOS prend en charge trois moteurs de modèles. Le moteur est choisi lorsque vous créez un modèle ou cliquez sur **Utiliser le modèle** dans une carte de la galerie, et **ne peut pas être modifié par la suite** — créez un nouveau modèle si vous souhaitez changer de moteur.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML sans logique (recommandé)">
    Style Mustache <code>{"{{variable}}"}</code> espaces réservés dans du HTML simple. Le rendu se fait côté client, donc les reçus **fonctionnent hors ligne**. À utiliser pour l’impression depuis le navigateur et les reçus PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML thermique">
    Modèles XML qui produisent à la fois un aperçu à l’écran et des **commandes ESC/POS** pour les imprimantes de reçus thermiques. Un même modèle, deux sorties. À utiliser pour Epson, Star et d’autres imprimantes thermiques.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP hérité">
    Modèles PHP côté serveur utilisant les fonctions WooCommerce. Conservés pour la compatibilité ascendante avec les <code>yourtheme/woocommerce-pos/receipt.php</code> remplacements. Nécessite une connexion au serveur pour le rendu.
  </FeatureCard>
</FeatureGrid>

## Galerie de modèles {#template-gallery}

La galerie est fournie avec des modèles prêts à l’emploi que vous pouvez utiliser tels quels ou personnaliser. Filtrez par **catégorie** (Reçu, Facture, Reçu cadeau, Ticket de cuisine, Devis / Bon de commande), **format** (HTML / ESC/POS) et **direction** (LTR / RTL) ; prévisualisez avec les données réelles de votre boutique. Le pied de chaque carte comporte un lien **Aperçu** et un bouton principal **Utiliser le modèle** : cliquer sur Utiliser le modèle crée une copie modifiable dans *Vos modèles*.

### Modèles de documents tiers {#third-party-document-templates}

Si **PDF Invoices & Packing Slips for WooCommerce** par WP Overnight est actif, WCPOS expose également **Facture (WP Overnight)** et **Bon de livraison (WP Overnight)** dans la liste des modèles de reçus. Ces modèles HTML rendus côté serveur délèguent le rendu à l’API de documents de WP Overnight, en réutilisant la numérotation des factures et bons de livraison, la mise en page, l’image de marque, les champs légaux/fiscaux et les personnalisations de modèles configurés dans cette extension.

Ils sont disponibles uniquement lorsque l’extension WP Overnight est active et nécessitent une connexion au serveur pour le rendu. Utilisez les modèles HTML ou thermiques intégrés à WCPOS lorsque l’impression hors ligne est requise.

### Affichage adaptatif des taxes {#adaptive-tax-display}

La plupart des modèles intégrés (Standard, Standard RTL, Minimal/Moderne, Étroit, Facture, Devis, variantes thermiques simples) **s’adaptent automatiquement au réglage d’affichage des taxes de WooCommerce** :

- Les **boutiques avec taxes incluses** (UE/R.-U./AU) voient les prix TTC avec une ligne « Taxes incluses ».
- Les **boutiques avec taxes exclues** (US/CA) voient les prix nets avec la taxe ajoutée sur une ligne distincte.
- Le total général affiche toujours le montant TTC réellement facturé.

La famille **Détaillé** est une facture fiscale formelle et détaille toujours les taxes avec des lignes hors taxes et une ventilation explicite, quel que soit le paramètre de la boutique.

## Fonctionnement {#how-it-works}

1. **Ajouter un modèle** — cliquez sur **Utiliser le modèle** sur une carte de la galerie, ou créez le vôtre à partir de zéro.
2. **Personnalisez-le** avec l’éditeur intégré à l’application et son aperçu en direct, ou collez-le dans ChatGPT / Claude et demandez les modifications souhaitées.
3. **Activez-le** — activez l’option **Actif** dans *Vos modèles*. Chaque modèle de reçu actif apparaît dans la liste déroulante de l’écran des reçus à la caisse.
4. **Imprimer ou afficher** — les reçus s’ouvrent après le passage en caisse, avec des options pour imprimer, envoyer par e-mail ou afficher à l’écran.

Les modèles sont rendus à partir d’une [charge utile de données de reçu](/receipts/receipt-data) standardisée, avec des sections pour les informations de la boutique, les lignes d’articles, les totaux, les taxes, les paiements, les remboursements, etc. Tous les champs de devise incluent des variantes `_display` préformatées (par exemple, `$29.99` au lieu de `29.99`).

## Fonctionnalités Pro {#pro-features}

Avec [WCPOS Pro](/getting-started/pro-license), chaque boutique peut avoir ses propres attributions de modèles, son image de marque (logo, adresse, coordonnées) et son ordre de modèles — pratique pour les activités sur plusieurs points de vente, où chaque boutique imprime son propre en-tête.
