:::caution
Questo articolo è destinato agli sviluppatori web, il debug non è qualcosa che l'utente medio dovrebbe dover utilizzare.
:::

WCPOS è principalmente un'applicazione web JavaScript - e come tale - il debug viene svolto principalmente attraverso la [console del browser](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors).
Se il debug è abilitato, WCPOS registrerà vari messaggi nella console che possono aiutare a diagnosticare problemi.

![Una Console JavaScript aperta in Chrome per Mac](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Una Console JavaScript aperta in Chrome per Mac")

### Attivare il debug utilizzando wp-config.php

Impostando il flag [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) nel file wp-config.php, puoi abilitare il debug di WCPOS.
Questo caricherà i file di script non minimizzati e abiliterà anche i messaggi di debug nella console del browser.

### Attivare il debug utilizzando la console del browser

Se non hai accesso al file wp-config.php, puoi abilitare i messaggi di debug aggiungendo un flag `debug` all'Archivio Locale.
Questo può essere fatto immettendo `localStorage.setItem('debug', '*')` nella console.
Ora aggiorna la pagina e il debug sarà attivo. La registrazione nella console può consumare risorse del browser, quindi vorrai probabilmente disattivare il debug una volta terminato.
Per disattivare il debug, immetti semplicemente `localStorage.removeItem('debug')` nella console e aggiorna la pagina.
