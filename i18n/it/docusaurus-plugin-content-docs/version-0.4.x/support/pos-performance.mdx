title: Prestazioni POS
---

WCPOS è un'applicazione web javascript a pagina singola che comunica con il tuo server tramite l'[API REST di WooCommerce](http://woothemes.github.io/woocommerce-rest-api-docs/). I problemi di prestazioni possono essere suddivisi in due categorie: lato server e lato client.

### Prestazioni Lato Server

Le prestazioni lato server si riferiscono ad azioni come il download di una pagina di prodotti (10 prodotti) o l'elaborazione di un ordine. Problemi che possono influire sulla velocità di tali azioni includono:

- La velocità della connessione internet
- La velocità del server, cioè: potenza di elaborazione, RAM, carico del server ecc.
- Quanti plugin hai attivi
- Elaborazione del gateway di pagamento

Il [sito demo](http://demo.wcpos.com/pos) rappresenta uno scenario "migliore". Il sito demo utilizza [un ottimo hosting web](http://wcpos.com/wpe) e ha un numero molto limitato di plugin attivati.

| Processo                 | Velocità medie per [demo.wcpos.com/pos](http://demo.wcpos.com/pos)<sup>\*</sup> |
| ------------------------ | ------------------------------------------------------------------------------- |
| Recupero di 10 prodotti  | 1 - 2 secondi                                                                   |
| Elaborazione vendita Cash| 1 - 2 secondi                                                                   |
| Elaborazione vendita Stripe | 2 - 3 secondi                                                               |

<small>
  <sup>*</sup> tempi medi per una connessione a banda larga. I tempi possono variare
  a seconda della velocità di internet.
</small>

Se il tuo POS richiede significativamente più tempo dei tempi sopra indicati, potresti voler effettuare i seguenti test:

1.  Cambia il tema nel tema predefinito Twenty Fifteen di WordPress
2.  Disattiva tutti i plugin tranne WooCommerce e WCPOS
3.  [Cancella i dati di archiviazione locale](http://faq.wcpos.com/en/clear-local-data.html)
4.  Ora, usa il POS per vedere se c'è un aumento delle prestazioni
5.  **Se c'è un aumento delle prestazioni:** riattiva il tuo tema e i plugin uno per uno per vedere quale plugin sta impattando le prestazioni
6.  **Se non c'è aumento delle prestazioni:** potresti voler passare a un hosting web migliore o investire in una connessione internet più veloce

### Prestazioni Lato Client

WCPOS utilizza JavaScript, HTML e CSS per visualizzare i dati recuperati dall'API REST di WooCommerce, ovvero [prodotti](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) e [ordini](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order).
Per migliorare le prestazioni lato client, il POS memorizza i dati nel browser utilizzando [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API).
Ad esempio, quando un prodotto viene recuperato per la prima volta, verrà inviata una richiesta al server; una volta scaricati i dati del prodotto, verranno memorizzati localmente in modo che le ricerche successive siano istantanee.

![Un esempio di prodotto memorizzato localmente.](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "Un esempio di prodotti memorizzati localmente")

I dati di IndexedDB persistono anche quando si chiude il browser o si riavvia il computer.
Se si nota che i dati dei prodotti non sono sincronizzati per qualsiasi motivo, è possibile [cancellare l'archiviazione locale](http://faq.wcpos.com/en/clear-local-data.html) e recuperare un nuovo set di dati dal server.
