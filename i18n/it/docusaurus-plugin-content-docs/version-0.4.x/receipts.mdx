I modelli di stampa delle ricevute POS risiedono nel file `includes/views/print/tmpl-receipt.php` sia del plugin gratuito che del Pro. I modelli di ricevuta possono essere personalizzati creando un file `woocommerce-pos/print/tmpl-receipt.php` nella directory del tuo tema. Il codice per entrambi i modelli è incluso di seguito.

### Personalizzazione della data della ricevuta {#customising-the-receipt-date}

WCPOS utilizza [moment.js](http://momentjs.com/) per localizzare le stringhe di data. Il formato di data predefinito è `"MMMM Do YYYY, h:mm:ss a"`, ad esempio: 31 maggio 2015, 19:20:44. Si prega di consultare la [documentazione di moment.js](http://momentjs.com/docs/#/parsing/string-format/) per ulteriori informazioni sulla personalizzazione del formato della data.

### Proprietà dell'Ordine {#order-properties}

WCPOS utilizza l'output json dell'API REST di WC per popolare il modello di ricevuta dell'ordine. [La documentazione dell'API REST di WC mostra un esempio](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order) dell'output json standard. Alcune proprietà aggiuntive sono state aggiunte da WCPOS utilizzando il filtro `woocommerce_api_order_response`.

| Proprietà                         | Descrizione                                                 |
| --------------------------------- | ----------------------------------------------------------- |
| cart_discount_tax                 | La parte fiscale dello sconto sul carrello                 |
| cashier.id                        | ID utente del Cassiere                                     |
| cashier.first_name                | Nome del Cassiere                                           |
| cashier.last_name                 | Cognome del Cassiere                                        |
| payment_details.result            | Successo o fallimento del metodo di pagamento              |
| payment_details.message           | Qualsiasi messaggio dal gateway di pagamento                |
| payment_details.redirect          | URL di reindirizzamento per pagamenti esterni, ad esempio: PayPal Standard |
| payment_details.method\_`gateway` | Qualsiasi messaggio specifico del gateway, ad esempio: Importo Versato e Cambio |
| shipping_lines[i].total_tax       | Aggiungi l'importo fiscale per ciascuna linea di spedizione |
| subtotal_tax                      | La parte fiscale del subtotale                              |
