WCPOS memorizza una copia locale dei tuoi prodotti, ordini e clienti in modo che il POS possa funzionare rapidamente e operare offline. Occasionalmente, questi dati locali possono diventare fuori sincrono con il tuo negozio WooCommerce. Cancellare tutti i dati locali rimuove il database locale e attiva una nuova sincronizzazione dal tuo server.

:::warning
Questa è l'opzione nucleare. Cancellare i dati locali elimina tutto ciò che è memorizzato sul dispositivo e costringe una piena ri-sincronizzazione. A seconda della grandezza del tuo negozio, questo potrebbe richiedere del tempo. Usala solo quando altre opzioni — come una normale [sync](/products/sync) — non hanno risolto il problema.
:::

## Quando Usarlo

Potresti dover cancellare i dati locali quando:

- **Problemi di sincronizzazione** — prodotti, ordini o clienti non si aggiornano correttamente
- **Dopo un aggiornamento del plugin** — per garantire la compatibilità con le nuove strutture di dati
- **Dati obsoleti** — prezzi, livelli di stock, o altre informazioni sembrano obsolete

## Cosa Viene Cancellato

| Cancellato | Non Cancellato |
|------------|----------------|
| Database locale dei prodotti | La tua sessione di accesso |
| Database locale degli ordini | Dati lato server |
| Database locale dei clienti | Impostazioni di WooCommerce |

Non verrai **disconnesso**. I dati del tuo negozio WooCommerce non sono influenzati — solo la copia locale sul tuo dispositivo viene rimossa. Una nuova sincronizzazione inizia automaticamente dopo la cancellazione.

## Come Cancellare i Dati Locali

### Web

1. Clicca sul tuo **menu utente** in alto a destra per aprire il menu
2. Clicca su **Cancella Tutti i Dati Locali**
3. Il database locale verrà eliminato e una nuova sincronizzazione inizierà automaticamente

### Desktop

1. Apri il menu dell'app **WCPOS** nella barra del menu in alto
2. Passa il mouse su **Avanzato**
3. Clicca su **Cancella Dati App**
4. Il database locale verrà eliminato e una nuova sincronizzazione inizierà automaticamente

L'app desktop utilizza un database SQLite mentre l'app web utilizza IndexedDB, ma il comportamento è lo stesso su entrambe le piattaforme.
:::
