La schermata dei Log registra l'attività del sistema all'interno del POS. È uno strumento essenziale per il debug dei problemi e per comprendere cosa sta accadendo dietro le quinte.

{/* Screenshot: logs-screen.png */}

## Accesso ai Log

Apri la schermata dei Log dal pannello di navigazione a sinistra (l'icona del battito cardiaco).

## Panoramica dell'Interfaccia

### Cerca & Filtra

- **Cerca Log** - Trova voci di log specifiche per contenuto del messaggio
- **Filtro Livello Log** - Filtra per livello di log (info, avviso, errore)
- **Impostazioni di visualizzazione** - Configura le colonne visibili

### Tabella dei Log

Ogni voce di log mostra:

| Colonna | Descrizione |
|---------|-------------|
| **Ora** | Quando si è verificato l'evento |
| **Livello** | Severità: info (grigio), avviso (arancione), errore (rosso) |
| **Messaggio** | Descrizione leggibile dall'uomo dell'evento |
| **Contesto** | JSON espandibile con dettagli aggiuntivi |
| **Codice** | Codice di errore (collegamenti alla documentazione) |

## Livelli di Log

### Info (Grigio)

Operazioni normali e eventi riusciti:
- "Recuperati tutti gli ID per i prodotti"
- "Sincronizzati nuovi clienti"
- "Recuperati tutti gli ID per le tasse"

### Avviso (Arancione)

Problemi non critici che potrebbero necessitare attenzione:
- "Impossibile aggiornare la sessione"
- "Tentativo di ripetizione per la sincronizzazione"

### Errore (Rosso)

Problemi che necessitano di essere risolti:
- "Spiacente, non puoi elencare le risorse" (problema di autorizzazione)
- "Richiesta API fallita"
- "Errore di sincronizzazione del database"

## Dati di Contesto

Clicca sulla freccia di espansione su qualsiasi voce di log per vedere il contesto JSON dettagliato:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Questi dati includono:
- **Conteggi degli elementi** - Quanti record sono stati interessati
- **Tempo di esecuzione** - Quanto tempo ha impiegato l'operazione
- **Carico del server** - Metriche delle prestazioni del server
- **Dettagli richiesta/risposta** - Per il debug dei problemi API

## Codici di Errore

Molte voci di log includono un **codice di errore** (ad esempio, `API02004`, `DB01001`). Questi codici collegano direttamente alla documentazione [Codici di Errore](/error-codes), che fornisce:

- Spiegazione dettagliata dell'errore
- Cause comuni
- Passi per la risoluzione dei problemi

## Voci di Log Comuni

### Operazioni di Sincronizzazione

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

Questi indicano un'attività di sincronizzazione normale. Il contesto mostra quanti elementi sono stati sincronizzati.

### Errori di Permesso

```
[error] Sorry, you cannot list resources    {...} API02004
```

Questo di solito significa che l'utente non ha il permesso di accedere a una risorsa. Controlla i ruoli utente in `WP Admin > POS > Settings > Access`.

### Problemi di Sessione

```
[warn] Unable to refresh session    {...} API02007
```

Il POS non è riuscito ad aggiornare la sessione di autenticazione. Questo potrebbe richiedere di disconnettersi e riconnettersi.

## Utilizzare i Log per il Supporto

Quando richiedi supporto:

1. **Riproduci il problema** - Esegui l'azione che causa il problema
2. **Controlla i log** - Cerca errori (rossi) o avvisi (arancioni)
3. **Espandi il contesto** - Ottieni i dettagli JSON completi
4. **Annota il codice di errore** - Includilo nella tua richiesta di supporto
5. **Copia le voci rilevanti** - Condividi i messaggi di log con il contesto

## Monitoraggio delle Prestazioni

I log includono il tempo di esecuzione e le metriche del carico del server. Se noti:

- **Tempi di esecuzione elevati** - Il server potrebbe essere sovraccarico o lento
- **Valori di carico del server elevati** - Considera l'ottimizzazione del server
- **Timeout frequenti** - Controlla la connettività di rete
