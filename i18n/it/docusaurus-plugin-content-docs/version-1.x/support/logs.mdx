La schermata dei Log registra l'attività di sistema all'interno del POS. È uno strumento essenziale per identificare problemi e comprendere cosa accade dietro le quinte.

{/* Screenshot: logs-screen.png */}

## Accesso ai Log

Aprire la schermata dei Log dal cassetto di navigazione di sinistra (icona del battito cardiaco).

## Panoramica dell'Interfaccia

### Ricerca e Filtri

- **Ricerca Log** - Trovare voci di log specifiche per contenuto del messaggio
- **Filtro Livello Log** - Filtrare per livello di log (info, avviso, errore)
- **Impostazioni di visualizzazione** - Configurare le colonne visibili

### Tabella dei Log

Ogni voce di log mostra:

| Colonna | Descrizione |
|--------|-------------|
| **Tempo** | Quando si è verificato l'evento |
| **Livello** | Gravità: info (grigio), avviso (arancione), errore (rosso) |
| **Messaggio** | Descrizione leggibile dell'evento |
| **Contesto** | JSON espandibile con dettagli aggiuntivi |
| **Codice** | Codice errore (collegamento alla documentazione) |

## Livelli di Log

### Info (Grigio)

Operazioni normali ed eventi riusciti:
- "Recuperati tutti gli ID per i prodotti"
- "Clienti sincronizzati"
- "Recuperati tutti gli ID per le tasse"

### Avviso (Arancione)

Problemi non critici che potrebbero richiedere attenzione:
- "Impossibile aggiornare la sessione"
- "Tentativo di ripetizione della sincronizzazione"

### Errore (Rosso)

Problemi che devono essere risolti:
- "Spiacenti, non è possibile elencare le risorse" (problema di autorizzazione)
- "Richiesta API non riuscita"
- "Errore di sincronizzazione del database"

## Dati di Contesto

Fare clic sulla freccia di espansione in qualsiasi voce di log per vedere il contesto JSON dettagliato:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Questi dati includono:
- **Conteggi degli elementi** - Quanti record sono stati interessati
- **Tempo di esecuzione** - Quanto tempo ha richiesto l'operazione
- **Carico del server** - Metriche delle prestazioni del server
- **Dettagli richiesta/risposta** - Per il debug dei problemi API

## Codici Errore

Molte voci di log includono un **codice errore** (es. `API02004`, `DB01001`). Questi codici si collegano direttamente alla documentazione [Codici Errore](/error-codes), che fornisce:

- Spiegazione dettagliata dell'errore
- Cause comuni
- Passaggi per la risoluzione dei problemi

## Voci di Log Comuni

### Operazioni di Sincronizzazione

```
[info] Recuperati tutti gli ID per i prodotti    {...} 3 elementi
[info] Clienti sincronizzati                     {...} 1 elemento
[info] Recuperati tutti gli ID per le tasse      {...} 3 elementi
```

Questi indicano l'attività di sincronizzazione normale. Il contesto mostra quanti elementi sono stati sincronizzati.

### Errori di Autorizzazione

```
[error] Spiacenti, non è possibile elencare le risorse    {...} API02004
```

Questo di solito significa che l'utente non ha il permesso di accedere a una risorsa. Controllare i ruoli utente in `WP Admin > POS > Impostazioni > Accesso`.

### Problemi di Sessione

```
[warn] Impossibile aggiornare la sessione    {...} API02007
```

Il POS non è riuscito ad aggiornare la sessione di autenticazione. Potrebbe essere necessario effettuare il logout e il login.

## Utilizzo dei Log per il Supporto

Quando si richiede supporto:

1. **Riprodurre il problema** - Eseguire l'azione che causa il problema
2. **Controllare i log** - Cercare errori (rossi) o avvisi (arancioni)
3. **Espandere il contesto** - Ottenere i dettagli JSON completi
4. **Annotare il codice errore** - Includerlo nella richiesta di supporto
5. **Copiare le voci rilevanti** - Condividere i messaggi di log con il contesto

## Monitoraggio delle Prestazioni

I log includono metriche di tempo di esecuzione e carico del server. Se si nota:

- **Tempi di esecuzione elevati** - Il server potrebbe essere sovraccarico o lento
- **Valori di carico del server elevati** - Considerare l'ottimizzazione del server
- **Timeout frequenti** - Verificare la connettività di rete
