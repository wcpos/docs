WCPOS include un sistema completo di modelli per ricevute, fatture, ricevute regalo, distinte di imballaggio, comande di cucina e altro ancora, tutti gestiti in **WP Admin → POS → Modelli**.

:::tip Vuoi solo modificare la ricevuta?
Inizia da **[Personalizzare la ricevuta](/receipts/customise)**. La guida illustra i tre modi più semplici per farlo: scegliere un altro modello incluso, chiedere all'AI di modificarne uno oppure intervenire manualmente.
:::

## Motori dei modelli {#template-engines}

WCPOS supporta tre motori di modelli. Il motore viene scelto quando crei un modello o fai clic su **Usa modello** in una scheda della galleria e **non può essere modificato in seguito**: crea un nuovo modello se vuoi cambiare motore.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML senza logica (consigliato)">
    Stile Mustache <code>{"{{variable}}"}</code> segnaposto all'interno di HTML semplice. Viene renderizzato lato client, quindi le ricevute **funzionano offline**. Da usare per la stampa dal browser e le ricevute PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML termico">
    Modelli XML che producono sia un'anteprima a schermo sia **comandi ESC/POS** per stampanti termiche per ricevute. Stesso modello, due output. Da usare per Epson, Star e altre stampanti termiche.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP legacy">
    Modelli PHP lato server che usano le funzioni WooCommerce. Mantenuti per retrocompatibilità con gli override esistenti <code>yourtheme/woocommerce-pos/receipt.php</code> override. Richiedono una connessione al server per il rendering.
  </FeatureCard>
</FeatureGrid>

## Galleria dei modelli {#template-gallery}

La galleria include modelli pronti all'uso che si possono usare così come sono o personalizzare. Filtra per **categoria** (Ricevuta, Fattura, Ricevuta regalo, Comanda cucina, Preventivo / Ordine di acquisto), **formato** (HTML / ESC/POS) e **direzione** (LTR / RTL); visualizza l'anteprima con i dati reali del negozio. Il piè di pagina di ogni scheda contiene un link **Anteprima** e un pulsante principale **Usa modello**: facendo clic su Usa modello viene creata una copia modificabile in *I tuoi modelli*.

| Modello | Motore | Descrizione |
|----------|--------|-------------|
| Ricevuta standard | HTML | Ricevuta predefinita — logo, identità del negozio, articoli, totali, pagamento |
| Ricevuta standard (RTL) | HTML | Versione da destra a sinistra per arabo, ebraico, persiano, urdu |
| Minimale / Moderna | HTML | Gli stessi elementi essenziali della Standard, compattati in meno spazio verticale |
| Ricevuta dettagliata | HTML | Fattura fiscale completa — colonna SKU, prezzo unitario, dettaglio delle imposte per aliquota, indirizzi |
| Ricevuta regalo | HTML | Solo articoli, prezzi nascosti. Include messaggio regalo e politica di reso |
| Fattura | HTML | Fattura a pagina intera A4/Letter con pannello opzionale "Come pagare" per gli ordini non pagati |
| Distinta di imballaggio | HTML | Documento di accompagnamento per magazzino — articoli + quantità, indirizzo di spedizione, nessun prezzo |
| Preventivo / Stima | HTML | Documento pre-vendita con prezzi e termini — nessuna sezione di pagamento |
| Ricevuta stretta | HTML | Ricevuta a spaziatura fissa per carta stretta o stampanti termiche compatibili con HTML |
| Ricevuta termica semplice (58mm) | Termico | Layout termico essenziale da 58mm |
| Ricevuta termica semplice (80mm) | Termico | Layout termico essenziale da 80mm — il più comune |
| Ricevuta termica semplice 80mm (RTL) | Termico | Versione RTL corrispondente per 80mm. Richiede una stampante con codepage araba (CP864 / Windows-1256) |
| Ricevuta termica dettagliata (58mm) | Termico | 58mm completa di tutto — indirizzi, dettaglio delle imposte, rimborsi, pagamenti, termini, codice a barre |
| Ricevuta termica dettagliata (80mm) | Termico | 80mm completa di tutto — indirizzi, dettaglio delle imposte, rimborsi, pagamenti, termini, codice a barre |
| Comanda cucina | Termico | Solo articoli, carattere grande, senza prezzi — per le postazioni di preparazione |

### Modelli di documenti di terze parti {#third-party-document-templates}

Se **PDF Invoices & Packing Slips for WooCommerce** di WP Overnight è attivo, WCPOS mostra anche **Fattura (WP Overnight)** e **Documento di trasporto (WP Overnight)** nell'elenco dei modelli di ricevuta. Questi modelli HTML generati dal server delegano all'API documentale di WP Overnight, riutilizzando la numerazione di fatture e documenti di trasporto, il layout, il branding, i campi legali/fiscali e le personalizzazioni dei modelli configurati in quel plugin.

Sono disponibili solo mentre il plugin WP Overnight è attivo e richiedono una connessione al server per il rendering. Utilizzare i modelli HTML o termici inclusi in WCPOS quando è necessaria la stampa offline.

### Visualizzazione adattiva delle imposte {#adaptive-tax-display}

La maggior parte dei modelli inclusi (Standard, Standard RTL, Minimale/Moderno, Stretto, Fattura, Preventivo, varianti Termico semplice) **si adatta automaticamente all'impostazione di WooCommerce per la visualizzazione delle imposte**:

- I **negozi con imposte incluse** (UE/UK/AU) visualizzano i prezzi lordi con una riga "Imposte incluse".
- I **negozi con imposte escluse** (US/CA) visualizzano i prezzi netti con l'imposta aggiunta come riga separata.
- Il totale complessivo mostra sempre l'importo lordo effettivamente addebitato.

La famiglia **Dettagliato** è una fattura fiscale formale e dettaglia sempre le imposte con righe al netto delle imposte più una ripartizione esplicita, indipendentemente dall'impostazione del negozio.

## Come funziona {#how-it-works}

1. **Aggiungi un modello** — fare clic su **Usa modello** in una scheda della galleria oppure crearne uno da zero.
2. **Personalizzarlo** usando l’editor integrato nell’app con anteprima in tempo reale, oppure incollarlo in ChatGPT / Claude e richiedere le modifiche desiderate.
3. **Attivarlo** — abilitare l’interruttore **Attivo** in *I tuoi modelli*. Ogni modello di ricevuta attivo appare nel menu a discesa della schermata delle ricevute alla cassa.
4. **Stampa o visualizzazione** — le ricevute si aprono dopo il checkout con opzioni per stamparle, inviarle via email o visualizzarle sullo schermo.

I modelli vengono renderizzati usando un [payload di dati della ricevuta](/receipts/receipt-data) standardizzato, con sezioni per informazioni sul negozio, righe degli articoli, totali, imposte, pagamenti, rimborsi e altro ancora. Tutti i campi valuta includono varianti `_display` già formattate (ad esempio, `$29.99` invece di `29.99`).

## Funzionalità Pro {#pro-features}

Con [WCPOS Pro](/getting-started/pro-license), ogni negozio può avere le proprie assegnazioni dei modelli, il proprio branding (logo, indirizzo, recapiti) e il proprio ordinamento dei modelli — utile per attività con più punti vendita, in cui ogni negozio stampa la propria carta intestata.
