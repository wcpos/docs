# Gateway Checkout Web {#web-checkout-gateway}

Il gateway Checkout Web consente ai clienti di completare le transazioni POS utilizzando il sistema di checkout del negozio online. Questo gateway crea un collegamento diretto tra il POS in negozio e i metodi di pagamento online, ideale per situazioni in cui i clienti preferiscono pagare con i propri dispositivi o quando è necessario accedere a metodi di pagamento disponibili solo sul web.

## Installazione {#installation}

<Steps>
  <StepCard number="1" title="Scarica e installa">

1. Visita la [pagina delle release del gateway Checkout Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Scarica il file **woocommerce-pos-web-checkout-gateway.zip** più recente
3. Nel pannello di amministrazione WordPress, vai su `Plugin > Aggiungi Nuovo > Carica plugin`
4. Carica il file zip e attiva il plugin

  </StepCard>
  <StepCard number="2" title="Attiva in WCPOS" isLast>

1. Vai su `WP Admin > POS > Impostazioni > Pagamento`
2. Trova **WCPOS Gateway Checkout Web** nell'elenco
3. Attiva il gateway per l'utilizzo nel POS
4. Salva le impostazioni

:::note
Questo gateway sfrutta i metodi di pagamento WooCommerce esistenti, quindi assicurati di avere almeno un gateway di pagamento web configurato correttamente (Stripe, PayPal, ecc.).
:::

  </StepCard>
</Steps>

## Utilizzo {#usage}

### Avvio del Checkout Web {#initiating-web-checkout}

1. **Aggiungi articoli**: Aggiungi i prodotti al carrello nel POS
2. **Seleziona il gateway**: Scegli "Checkout Web" come metodo di pagamento
3. **Genera il link**: Il sistema crea un link di checkout univoco per l'ordine
4. **Accesso del cliente**: Il cliente clicca il link per accedere al checkout web
5. **Pagamento online**: Il cliente completa il pagamento utilizzando i metodi di pagamento del negozio online

### Processo di pagamento {#payment-process}

Il gateway Checkout Web segue questo flusso di lavoro:

1. **Creazione ordine**: il POS crea un ordine in attesa di pagamento in WooCommerce
2. **Link di pagamento**: il sistema genera un link di pagamento sicuro e univoco
3. **Reindirizzamento del cliente**: il cliente viene indirizzato alla pagina di checkout web
4. **Selezione del pagamento**: il cliente sceglie tra i metodi di pagamento web disponibili
5. **Elaborazione del pagamento**: il processo di checkout standard di WooCommerce gestisce il pagamento
6. **Completamento dell'ordine**: lo stato dell'ordine si aggiorna automaticamente al completamento del pagamento

### Flusso di lavoro del POS {#pos-workflow}

Dopo aver avviato il checkout web:

1. **Monitorare lo stato**: mantenere aperta la schermata dell'ordine nel POS per monitorare lo stato del pagamento
2. **Pagamento del cliente**: il cliente completa il pagamento sul proprio dispositivo o sul tablet
3. **Aggiornamento automatico**: lo stato dell'ordine si aggiorna in tempo reale alla ricezione del pagamento
4. **Elaborare il pagamento**: fare clic sul pulsante "Elabora pagamento" nel POS per procedere alla ricevuta
5. **Generazione ricevuta**: generazione e stampa della ricevuta come di consueto

## Dettagli tecnici {#technical-details}

### Gestione degli ordini {#order-management}

- **Ordini in attesa**: Gli ordini vengono creati con lo stato "in attesa di pagamento"
- **Aggiornamenti di stato**: Cambio automatico dello stato alla ricezione del pagamento
- **Note dell'ordine**: I dettagli del pagamento vengono registrati nelle note dell'ordine
- **Inventario**: Le scorte vengono riservate durante il processo di checkout

### Sicurezza {#security}

- **Link univoci**: Ogni sessione di checkout ha un URL univoco con scadenza temporale
- **SSL obbligatorio**: Tutte le operazioni di pagamento avvengono tramite connessioni HTTPS sicure
- **Conformità PCI**: Utilizza i gateway di pagamento web già conformi allo standard PCI
- **Gestione delle sessioni**: La gestione sicura delle sessioni impedisce accessi non autorizzati

### Metodi di pagamento {#payment-methods}

Il gateway fornisce accesso a tutti i metodi di pagamento WooCommerce configurati:

- **Carte di credito/debito**: Stripe, Square, Authorize.net, ecc.
- **Portafogli digitali**: PayPal, Apple Pay, Google Pay
- **Bonifici bancari**: metodi di pagamento tramite bonifico bancario
- **Compra ora, paga dopo**: Klarna, Afterpay, Sezzle
- **Criptovalute**: Bitcoin e altri gateway di pagamento in criptovaluta
- **Metodi locali**: metodi di pagamento specifici per area geografica

## Risoluzione dei problemi {#troubleshooting}

### Problemi comuni {#common-issues}

<Accordion>
  <AccordionItem question="Il link di checkout non funziona">

  - Verificare che il certificato SSL sia installato correttamente
  - Controllare che i permalink di WooCommerce siano configurati correttamente
  - Assicurarsi che i gateway di pagamento web siano attivi e configurati
  - Provare con un browser o dispositivo diverso

  </AccordionItem>

  <AccordionItem question="Il pagamento non viene elaborato">

  - Verificare che i gateway di pagamento web siano configurati correttamente
  - Controllare i log del gateway di pagamento per eventuali messaggi di errore
  - Verificare che il metodo di pagamento del cliente sia supportato
  - Testare il processo di checkout web in modo indipendente

  </AccordionItem>

  <AccordionItem question="L'ordine non si aggiorna nel POS">

  - Controllare la stabilità della connessione internet
  - Verificare che i cron job di WordPress funzionino correttamente
  - Aggiornare la schermata degli ordini del POS
  - Controllare lo stato dell'ordine nell'amministrazione di WooCommerce

  </AccordionItem>

  <AccordionItem question="Il cliente non riesce ad accedere al checkout">

  - Assicurarsi che il link di checkout non sia scaduto
  - Verificare che il dispositivo del cliente disponga di connessione internet
  - Verificare che l'ordine non sia stato annullato o completato
  - Testare il link su un dispositivo diverso

  </AccordionItem>
</Accordion>

### Ottimizzazione delle prestazioni {#performance-optimisation}

Per prestazioni ottimali:

- **Cache**: configurare i plugin di cache per escludere le pagine di checkout
- **CDN**: assicurarsi che le impostazioni del CDN non interferiscano con il processo di checkout
- **Database**: ottimizzare il database per un'elaborazione degli ordini più rapida
- **Hosting**: utilizzare un hosting affidabile con un buon uptime
