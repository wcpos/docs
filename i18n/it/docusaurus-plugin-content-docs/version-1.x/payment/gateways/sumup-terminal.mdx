# Gateway SumUp Terminal {#sumup-terminal-gateway}

Il gateway SumUp Terminal consente di accettare pagamenti con carta utilizzando i lettori di carte SumUp direttamente in WCPOS. Questo gateway offre un'integrazione completa con il sistema di elaborazione pagamenti di SumUp per le transazioni in presenza.

## Funzionalità {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Integrazione hardware">
    Collega i lettori di carte SumUp al sistema POS tramite connessione internet
  </FeatureCard>
  <FeatureCard icon="bolt" title="Elaborazione in tempo reale">
    Elaborazione istantanea dei pagamenti e completamento degli ordini
  </FeatureCard>
  <FeatureCard icon="shield" title="Transazioni sicure">
    Elaborazione dei pagamenti conforme allo standard PCI tramite SumUp
  </FeatureCard>
  <FeatureCard icon="link" title="Abbinamento semplice">
    Procedura di associazione semplice tramite codici di abbinamento
  </FeatureCard>
</FeatureGrid>

## Installazione {#installation}

<Steps>
  <StepCard number="1" title="Installare SumUp Terminal per WooCommerce">

Installare da `WP Admin > POS > Impostazioni > Estensioni`, oppure scaricare l'ultimo **file zip del plugin** (non il file zip o tarball del codice sorgente GitHub) dalla [pagina delle release su GitHub](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) e caricarlo tramite `Plugin > Aggiungi Nuovo > Carica plugin`.

  </StepCard>
  <StepCard number="2" title="Configurare le impostazioni di SumUp">

1. Navigare in `WP Admin > WooCommerce > Impostazioni > Pagamenti`
2. Individuare **SumUp Terminal** nell'elenco dei metodi di pagamento
3. Fare clic su **SumUp Terminal** per accedere alle impostazioni
4. Inserire la propria **chiave API SumUp** (disponibile nel pannello di controllo esercente SumUp)
5. Salvare le impostazioni

:::note
Non è necessario abilitare il gateway SumUp Terminal nelle impostazioni di WooCommerce. Verrà abilitato specificamente per il POS in un passaggio successivo.
:::

  </StepCard>
  <StepCard number="3" title="Associa il tuo SumUp Terminal">

1. Nella stessa pagina delle impostazioni, individua la sezione **Associa lettore**
2. Sul dispositivo SumUp, accedi alla schermata di associazione per visualizzare il codice di abbinamento
3. Inserisci il codice di abbinamento visualizzato sul dispositivo SumUp
4. Fai clic su **"Associa lettore"** per stabilire la connessione
5. Attendi la conferma che il lettore è stato associato correttamente

:::warning Importante
Il lettore deve essere associato correttamente prima di poter elaborare i pagamenti. Assicurati che il processo di associazione sia completato prima di procedere.
:::

  </StepCard>
  <StepCard number="4" title="Attiva in WCPOS" isLast>

1. Vai su `WP Admin > POS > Impostazioni > Pagamento`
2. Trovare il gateway **SumUp Terminal** nell'elenco
3. Abilitare il gateway per l'utilizzo nel POS
4. Salvare le impostazioni

  </StepCard>
</Steps>

## Utilizzo {#usage}

### Elaborazione dei pagamenti {#processing-payments}

1. **Aggiungere articoli**: aggiungere i prodotti al carrello nel POS
2. **Selezionare il gateway**: scegliere "SumUp Terminal" come metodo di pagamento
3. **Avviare il pagamento**: fare clic per avviare un nuovo pagamento sul dispositivo SumUp
4. **Pagamento del cliente**: il cliente completa il pagamento sul terminale SumUp
5. **Completamento automatico**: l'ordine viene completato automaticamente quando il pagamento viene rilevato

### Controlli di pagamento {#payment-controls}

Quando si utilizza il gateway SumUp Terminal, sono disponibili le seguenti opzioni:

- **Avvia nuovo pagamento**: Avvia una richiesta di pagamento sul terminale collegato
- **Annulla pagamento**: Annulla un pagamento attualmente in corso
- **Stato del pagamento**: Monitora lo stato attuale del processo di pagamento

### Gestione degli ordini {#order-management}

- **Aggiornamenti automatici**: Gli ordini vengono automaticamente contrassegnati come completati al termine di un pagamento riuscito
- **Tracciamento dei pagamenti**: Tutti i dettagli del pagamento vengono registrati nelle note dell'ordine
- **Generazione ricevute**: Le ricevute POS standard vengono generate dopo i pagamenti riusciti

## Compatibilità hardware {#hardware-compatibility}

:::important Requisiti di connettività
Questa implementazione utilizza il JavaScript SDK di SumUp, il che significa che funziona tramite applicazioni web ma richiede **solo terminali connessi a internet**. I terminali Bluetooth non sono attualmente supportati.
:::

### Terminali supportati (connessi a internet) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Terminale connesso tramite WiFi/Ethernet

### Terminali non supportati (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Solo Bluetooth
- **SumUp 3G** ❌ - Solo Bluetooth

:::note Supporto futuro
Il supporto per i terminali Bluetooth è previsto in una futura versione dell'app iOS e Android. Una volta disponibile, consentirà il supporto per tutti i terminali certificati SumUp, inclusi i modelli Air e 3G.
:::

## Risoluzione dei problemi {#troubleshooting}

### Problemi comuni {#common-issues}

<Accordion>
  <AccordionItem question="Il lettore non si associa">

  - Assicurarsi di utilizzare un terminale supportato con connessione a internet (solo Solo)
  - Verificare che il dispositivo SumUp Solo sia connesso al WiFi e online
  - Controllare che il dispositivo sia in modalità di associazione e che il codice di associazione sia stato inserito correttamente
  - Verificare che l'account SumUp abbia l'accesso API abilitato
  - Assicurarsi di avere una connessione internet stabile durante l'associazione

  </AccordionItem>

  <AccordionItem question="Pagamento non elaborato">

  - Verificare che il lettore sia correttamente associato
  - Verificare che l'account SumUp sia attivo e in regola
  - Assicurarsi che il dispositivo abbia batteria sufficiente e connettività
  - Verificare che la chiave API sia corretta e attiva

  </AccordionItem>

  <AccordionItem question="Ordine non completato">

  - Verificare la stabilità della connessione internet
  - Verificare le impostazioni webhook nell'account SumUp
  - Assicurarsi che il pagamento sia andato a buon fine sul terminale SumUp
  - Controllare i log degli errori di WordPress per problemi di comunicazione API

  </AccordionItem>
</Accordion>

### Limiti di frequenza API {#api-rate-limits}

Il gateway è ottimizzato per ridurre le chiamate API ed evitare i limiti di frequenza:
- Lo stato del pagamento viene verificato in modo efficiente
- Le richieste API non necessarie vengono ridotte al minimo
- La logica di ripetizione automatica gestisce gli errori temporanei
