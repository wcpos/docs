# Gateway Terminal SumUp

Il gateway Terminal SumUp ti consente di accettare pagamenti con carta utilizzando i lettori di carte SumUp direttamente all'interno di WooCommerce POS. Questo gateway offre un'integrazione senza soluzione di continuità con il sistema di elaborazione dei pagamenti di SumUp per transazioni di persona.

## Caratteristiche

- **Integrazione Hardware**: Collega i lettori di carte SumUp al tuo sistema POS tramite connessione a Internet
- **Elaborazione in tempo reale**: Elaborazione dei pagamenti istantanea e completamento degli ordini
- **Rilevamento automatico**: Gli ordini si completano automaticamente quando viene rilevato il pagamento
- **Transazioni sicure**: Elaborazione dei pagamenti conforme PCI tramite SumUp
- **Abbinamento semplice**: Processo di abbinamento del dispositivo semplice con codici di abbinamento
- **SDK basato su web**: Utilizza l'SDK JavaScript di SumUp per la compatibilità con le applicazioni web

## Installazione

### Passo 1: Scarica e Installa

1. Visita la [pagina delle versioni di SumUp Terminal per WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Scarica l'ultimo file **sumup-terminal-for-woocommerce.zip**
3. Nel tuo admin di WordPress, vai su `Plugin > Aggiungi Nuovo > Carica Plugin`
4. Carica il file zip e attiva il plugin

### Passo 2: Configura le Impostazioni di SumUp

1. Naviga su `WP Admin > WooCommerce > Impostazioni > Pagamenti`
2. Trova **Terminal SumUp** nell'elenco dei metodi di pagamento
3. Clicca su **Terminal SumUp** per accedere alle impostazioni
4. Inserisci la tua **API Key di SumUp** (disponibile nella tua dashboard merchant di SumUp)
5. Salva le impostazioni

:::note
Non è necessario abilitare il gateway Terminal SumUp nelle impostazioni di WooCommerce. Sarà abilitato specificamente per il POS in un passaggio successivo.
:::

### Passo 3: Abbina il Tuo Terminal SumUp

1. Nella stessa pagina delle impostazioni, individua la sezione **Abbina Lettore**
2. Sul tuo dispositivo SumUp, vai alla schermata di abbinamento per visualizzare il codice di abbinamento
3. Inserisci il codice di abbinamento visualizzato sul tuo dispositivo SumUp
4. Clicca su **"Abbina Lettore"** per stabilire la connessione
5. Attendi la conferma che il lettore è stato abbinato con successo

:::warning Importante
Il lettore deve essere abbinato con successo prima di poter elaborare i pagamenti. Assicurati che il processo di abbinamento sia completato prima di procedere.
:::

### Passo 4: Abilita in WooCommerce POS

1. Vai su `WP Admin > WooCommerce POS > Impostazioni > Checkout`
2. Trova il gateway **Terminal SumUp** nell'elenco
3. Abilita il gateway per l'utilizzo nel POS
4. Salva le tue impostazioni

## Utilizzo

### Elaborazione Pagamenti

1. **Aggiungi Articoli**: Aggiungi prodotti al tuo carrello nel POS
2. **Seleziona Gateway**: Scegli "Terminal SumUp" come metodo di pagamento
3. **Inizia Pagamento**: Clicca per avviare un nuovo pagamento sul tuo dispositivo SumUp
4. **Pagamento del Cliente**: Il cliente completa il pagamento sul terminal SumUp
5. **Completamento Automatico**: L'ordine si completa automaticamente quando viene rilevato il pagamento

### Controlli di Pagamento

Quando utilizzi il gateway Terminal SumUp, hai le seguenti opzioni:

- **Inizia Nuovo Pagamento**: Avvia una richiesta di pagamento sul terminale collegato
- **Annulla Pagamento**: Annulla un pagamento attualmente in elaborazione
- **Stato di Pagamento**: Monitora lo stato attuale del processo di pagamento

### Gestione degli Ordini

- **Aggiornamenti Automatici**: Gli ordini vengono automaticamente contrassegnati come completati dopo un pagamento riuscito
- **Tracciamento Pagamenti**: Tutti i dettagli dei pagamenti sono registrati nelle note dell'ordine
- **Generazione della Ricevuta**: Le ricevute standard del POS vengono generate dopo pagamenti riusciti

## Requisiti

- **Account SumUp**: Account merchant SumUp attivo
- **Accesso API**: Chiave API di SumUp dalla tua dashboard merchant
- **Hardware Compatibile**: Dispositivo lettore di carte SumUp
- **WooCommerce POS**: Versione gratuita o Pro installata e attivata
- **Connessione Stabile**: Connessione Internet affidabile per la comunicazione API

## Compatibilità Hardware

:::important Requisiti di Connettività
Questa implementazione utilizza l'SDK JavaScript di SumUp, il che significa che funziona tramite applicazioni web ma richiede **solo terminali connessi a Internet**. I terminali Bluetooth non sono attualmente supportati.
:::

### Terminali Supportati (Connessi a Internet)

- **SumUp Solo** ✅ - Terminale connesso tramite WiFi/Ethernet

### Terminali Non Supportati (Bluetooth)

- **SumUp Air** ❌ - Solo Bluetooth
- **SumUp 3G** ❌ - Solo Bluetooth

:::note Supporto Futuro
Il supporto per terminali Bluetooth sarà aggiunto alle applicazioni iOS e Android nel **Q1 2026**. Questo abiliterà il supporto per tutti i terminali certificati SumUp, inclusi i modelli Air e 3G.
:::

## Risoluzione dei Problemi

### Problemi Comuni

**Il Lettore Non Si Abbina**
- Assicurati di utilizzare un terminale connesso a Internet supportato (solo Solo)
- Verifica che il tuo dispositivo SumUp Solo sia connesso a WiFi e online
- Controlla che il dispositivo sia in modalità di abbinamento e che il codice di abbinamento sia inserito correttamente
- Conferma che il tuo account SumUp abbia attivato l'accesso API
- Assicurati di avere una connessione a Internet stabile durante l'abbinamento

**Pagamento Non Elaborato**
- Verifica che il lettore sia correttamente abbinato
- Controlla che il tuo account SumUp sia attivo e in regola
- Assicurati che il dispositivo abbia sufficiente carica della batteria e connettività
- Conferma che la chiave API sia corretta e attiva

**Ordine Non Completato**
- Controlla la stabilità della connessione a Internet
- Verifica le impostazioni del webhook nel tuo account SumUp
- Assicurati che il pagamento sia stato completato con successo sul terminal SumUp
- Controlla i log degli errori di WordPress per problemi di comunicazione API

### Limiti di Velocità API

Il gateway è ottimizzato per ridurre le chiamate API e evitare la limitazione della velocità:
- Lo stato del pagamento viene controllato in modo efficiente
- Le richieste API non necessarie sono ridotte al minimo
- La logica di ripetizione automatica gestisce i guasti temporanei
