# Gateway di Checkout Web {#web-checkout-gateway}

Il gateway di Checkout Web consente ai clienti di completare le proprie transazioni POS utilizzando il sistema di checkout del tuo store web. Questo gateway crea un ponte senza soluzione di continuità tra il tuo POS in negozio e i metodi di pagamento online, perfetto per situazioni in cui i clienti preferiscono pagare utilizzando i propri dispositivi o quando hai bisogno di accesso a metodi di pagamento disponibili solo online.

## Installazione {#installation}

<Steps>
  <StepCard number="1" title="Scarica e Installa">

1. Visita la [pagina delle versioni del Gateway di Checkout Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Scarica l'ultimo file **woocommerce-pos-web-checkout-gateway.zip**
3. Nel tuo admin di WordPress, vai su `Plugin > Aggiungi Nuovo > Carica Plugin`
4. Carica il file zip e attiva il plugin

  </StepCard>
  <StepCard number="2" title="Abilita in WCPOS" isLast>

1. Vai su `WP Admin > POS > Impostazioni > Checkout`
2. Trova il **Gateway di Checkout Web WCPOS** nell'elenco
3. Abilita il gateway per l'uso nel POS
4. Salva le tue impostazioni

:::note
Questo gateway sfrutta i metodi di pagamento WooCommerce esistenti, quindi assicurati di avere almeno un gateway di pagamento web configurato correttamente (Stripe, PayPal, ecc.).
:::

  </StepCard>
</Steps>

## Utilizzo {#usage}

### Iniziare il Checkout Web {#initiating-web-checkout}

1. **Aggiungi Articoli**: Aggiungi prodotti al tuo carrello nel POS
2. **Seleziona Gateway**: Scegli "Checkout Web" come metodo di pagamento
3. **Genera Link**: Il sistema crea un link di checkout unico per l'ordine
4. **Accesso del Cliente**: Il cliente clicca il link per accedere al checkout web
5. **Pagamento Online**: Il cliente completa il pagamento utilizzando i metodi di pagamento dello store web

### Processo di Pagamento {#payment-process}

Il gateway di Checkout Web segue questo flusso di lavoro:

1. **Creazione dell'Ordine**: Il POS crea un ordine in attesa in WooCommerce
2. **Link di Checkout**: Il sistema genera un link di pagamento unico e sicuro
3. **Reindirizzamento del Cliente**: Il cliente è diretto alla pagina di checkout web
4. **Selezione del Pagamento**: Il cliente sceglie tra i metodi di pagamento web disponibili
5. **Elaborazione del Pagamento**: Il processo standard di checkout WooCommerce gestisce il pagamento
6. **Completamento dell'Ordine**: Lo stato dell'ordine viene aggiornato automaticamente al pagamento riuscito

### Flusso di Lavoro POS {#pos-workflow}

Dopo aver iniziato il checkout web:

1. **Monitora Stato**: Tieni aperta la schermata degli ordini POS per monitorare lo stato del pagamento
2. **Pagamento del Cliente**: Il cliente completa il pagamento sul proprio dispositivo o sul tuo tablet
3. **Aggiornamento Automatico**: Lo stato dell'ordine si aggiorna in tempo reale quando il pagamento viene ricevuto
4. **Elabora Pagamento**: Clicca il pulsante "Elabora Pagamento" nel POS per continuare con la ricevuta
5. **Generazione della Ricevuta**: Genera e stampa la ricevuta come di consueto

## Dettagli Tecnici {#technical-details}

### Gestione degli Ordini {#order-management}

- **Ordini in Attesa**: Gli ordini vengono creati con stato "in attesa di pagamento"
- **Aggiornamenti di Stato**: Cambiamenti di stato automatici quando il pagamento viene ricevuto
- **Note dell'Ordine**: I dettagli del pagamento vengono registrati nelle note dell'ordine
- **Inventario**: Le scorte vengono riservate durante il processo di checkout

### Sicurezza {#security}

- **Link Unici**: Ogni sessione di checkout ha un URL unico e con tempo limitato
- **SSL Richiesto**: Tutti i processi di pagamento avvengono su connessioni HTTPS sicure
- **Conformità PCI**: Sfrutta i tuoi gateway di pagamento web conformi PCI esistenti
- **Gestione delle Sessioni**: La gestione sicura delle sessioni previene accessi non autorizzati

### Metodi di Pagamento {#payment-methods}

Il gateway fornisce accesso a tutti i tuoi metodi di pagamento WooCommerce configurati:

- **Carte di Credito/Debito**: Stripe, Square, Authorize.net, ecc.
- **Portafogli Digitali**: PayPal, Apple Pay, Google Pay
- **Bonifici Bancari**: Metodi di pagamento bancari diretti
- **Compra Ora, Paga Dopo**: Klarna, Afterpay, Sezzle
- **Criptovalute**: Bitcoin e altri gateway di pagamento crypto
- **Metodi Locali**: Metodi di pagamento specifici per la regione

## Risoluzione dei Problemi {#troubleshooting}

### Problemi Comuni {#common-issues}

<Accordion>
  <AccordionItem question="Link di checkout non funzionante">

  - Verifica che il certificato SSL sia installato correttamente
  - Controlla che i permalink di WooCommerce siano configurati correttamente
  - Assicurati che i gateway di pagamento web siano attivi e configurati
  - Prova con un browser o dispositivo diverso

  </AccordionItem>

  <AccordionItem question="Pagamento non elaborato">

  - Conferma che i gateway di pagamento web siano configurati correttamente
  - Controlla i log del gateway di pagamento per messaggi di errore
  - Verifica che il metodo di pagamento del cliente sia supportato
  - Prova il processo di checkout web in modo indipendente

  </AccordionItem>

  <AccordionItem question="L'ordine non si aggiorna nel POS">

  - Controlla la stabilità della connessione internet
  - Verifica che i lavori cron di WordPress siano in esecuzione correttamente
  - Aggiorna la schermata degli ordini del POS
  - Controlla lo stato dell'ordine nell'amministrazione di WooCommerce

  </AccordionItem>

  <AccordionItem question="Il cliente non può accedere al checkout">

  - Assicurati che il link di checkout non sia scaduto
  - Verifica che il dispositivo del cliente abbia connettività internet
  - Controlla che l'ordine non sia stato annullato o completato
  - Testa il link su un dispositivo diverso

  </AccordionItem>
</Accordion>

### Ottimizzazione delle Prestazioni {#performance-optimisation}

Per prestazioni ottimali:

- **Caching**: Configura i plugin di caching per escludere le pagine di checkout
- **CDN**: Assicurati che le impostazioni del CDN non interferiscano con il processo di checkout
- **Database**: Ottimizza il database per un'elaborazione degli ordini più veloce
- **Hosting**: Utilizza un hosting affidabile con buona disponibilità
