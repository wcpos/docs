# Gateway di Checkout Web

Il gateway di checkout web consente ai clienti di completare le loro transazioni POS utilizzando il sistema di checkout del tuo negozio web. Questo gateway crea un ponte senza soluzione di continuità tra il tuo POS in negozio e i metodi di pagamento online, perfetto per situazioni in cui i clienti preferiscono pagare utilizzando i propri dispositivi o quando hai bisogno di accedere a metodi di pagamento disponibili solo online.

## Caratteristiche

- **Integrazione Senza Soluzione di Continuità**: Collegamento diretto dal POS al checkout web
- **Tutti i Metodi di Pagamento**: Accesso a tutti i gateway di pagamento abilitati del tuo negozio web
- **Controllo del Cliente**: I clienti possono pagare utilizzando i propri dispositivi e metodi preferiti
- **Sincronizzazione degli Ordini**: Aggiornamenti automatici degli ordini tra POS e negozio web
- **Flusso di Lavoro Flessibile**: Completa le transazioni online mantenendo il flusso di lavoro del POS

## Installazione

### Passo 1: Scarica e Installa

1. Visita la [pagina delle versioni del Gateway di Checkout Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Scarica il file **woocommerce-pos-web-checkout-gateway.zip** più recente
3. Nel tuo admin di WordPress, vai su `Plugin > Aggiungi Nuovo > Carica Plugin`
4. Carica il file zip e attiva il plugin

### Passo 2: Abilita in WooCommerce POS

1. Vai su `WP Admin > WooCommerce POS > Impostazioni > Checkout`
2. Trova il **Gateway di Checkout Web WooCommerce POS** nell'elenco
3. Abilita il gateway per l'uso nel POS
4. Salva le tue impostazioni

:::note
Questo gateway sfrutta i metodi di pagamento esistenti di WooCommerce, quindi assicurati di avere almeno un gateway di pagamento web configurato correttamente (Stripe, PayPal, ecc.).
:::

## Utilizzo

### Iniziare il Checkout Web

1. **Aggiungi Articoli**: Aggiungi prodotti al tuo carrello nel POS
2. **Seleziona Gateway**: Scegli "Checkout Web" come metodo di pagamento
3. **Genera Link**: Il sistema crea un link di checkout unico per l'ordine
4. **Accesso del Cliente**: Il cliente clicca sul link per accedere al checkout web
5. **Pagamento Online**: Il cliente completa il pagamento utilizzando i metodi di pagamento del negozio web

### Processo di Pagamento

Il gateway di checkout web segue questo flusso di lavoro:

1. **Creazione dell'Ordine**: Il POS crea un ordine in attesa in WooCommerce
2. **Link di Checkout**: Il sistema genera un link di pagamento sicuro e unico
3. **Reindirizzamento del Cliente**: Il cliente viene indirizzato alla pagina di checkout web
4. **Selezione del Pagamento**: Il cliente sceglie tra i metodi di pagamento web disponibili
5. **Elaborazione del Pagamento**: Il processo di checkout standard di WooCommerce gestisce il pagamento
6. **Completamento dell'Ordine**: Lo stato dell'ordine si aggiorna automaticamente al termine del pagamento

### Flusso di Lavoro del POS

Dopo aver avviato il checkout web:

1. **Monitora lo Stato**: Mantieni aperta la schermata degli ordini POS per monitorare lo stato del pagamento
2. **Pagamento del Cliente**: Il cliente completa il pagamento sul proprio dispositivo o sul tuo tablet
3. **Aggiornamento Automatico**: Lo stato dell'ordine viene aggiornato in tempo reale quando il pagamento viene ricevuto
4. **Elaborare il Pagamento**: Clicca sul pulsante "Elabora Pagamento" nel POS per continuare verso la ricevuta
5. **Generazione della Ricevuta**: Genera e stampa la ricevuta come di consueto

## Casi d'Uso

### Perfetto per

- **Preferenza del Cliente**: Clienti che preferiscono utilizzare le proprie app di pagamento o carte
- **Pagamenti Complessi**: Transazioni che richiedono metodi di pagamento non disponibili nel POS
- **Pagamenti Splittati**: Clienti che vogliono utilizzare più metodi di pagamento
- **Programmi di Fedeltà**: Accesso al riscatto di punti fedeltà basato sul web
- **Buoni Regalo**: Clienti con buoni regalo digitali o credito del negozio
- **Carte Internazionali**: Metodi di pagamento che funzionano meglio tramite gateway web

### Esempi di Flusso di Lavoro

**Negozio al Dettaglio**
1. Il cliente fa acquisti in negozio e porta gli articoli alla cassa
2. Il cliente preferisce pagare con la propria app di portafoglio mobile
3. Il personale seleziona il gateway di Checkout Web
4. Il cliente scansiona il codice QR o clicca sul link sul proprio telefono
5. Il cliente completa il pagamento utilizzando il metodo preferito
6. Il personale elabora la ricevuta e completa la transazione

**Attività di Servizio**
1. Completa il servizio e aggiungi costi al POS
2. Il cliente desidera pagare con carta di credito aziendale
3. Genera un link di checkout web
4. Il cliente inserisce i dettagli di pagamento su un modulo web sicuro
5. Il pagamento viene elaborato tramite il tuo gateway web
6. L'ordine viene completato e viene generata la ricevuta

## Dettagli Tecnici

### Gestione degli Ordini

- **Ordini in Attesa**: Gli ordini vengono creati con lo stato "pagamento in attesa"
- **Aggiornamenti di Stato**: Cambiamenti di stato automatici quando il pagamento viene ricevuto
- **Note dell'Ordine**: I dettagli di pagamento vengono registrati nelle note dell'ordine
- **Inventario**: Lo stock viene riservato durante il processo di checkout

### Sicurezza

- **Link Unici**: Ogni sessione di checkout ha un URL unico e limitato nel tempo
- **SSL Richiesto**: Tutte le elaborazioni di pagamento avvengono su connessioni HTTPS sicure
- **Conformità PCI**: Sfrutta i tuoi gateway di pagamento web conformi al PCI
- **Gestione della Sessione**: La gestione sicura delle sessioni previene accessi non autorizzati

### Metodi di Pagamento

Il gateway fornisce accesso a tutti i tuoi metodi di pagamento WooCommerce configurati:

- **Carte di Credito/Debito**: Stripe, Square, Authorize.net, ecc.
- **Portafogli Digitali**: PayPal, Apple Pay, Google Pay
- **Bonifici Bancari**: Metodi di pagamento bancari diretti
- **Compra Ora, Paga Dopo**: Klarna, Afterpay, Sezzle
- **Cryptocurrency**: Bitcoin e altri gateway di pagamento crypto
- **Metodi Locali**: Metodi di pagamento specifici per regione

## Requisiti

- **WooCommerce POS**: Versione gratuita o Pro installata e attivata
- **Gateway di Pagamento Web**: Almeno un metodo di pagamento online configurato in WooCommerce
- **Certificato SSL**: HTTPS richiesto per un'elaborazione di pagamento sicura
- **Browser Moderno**: Il dispositivo del cliente deve supportare standard web moderni

## Risoluzione dei Problemi

### Problemi Comuni

**Link di Checkout Non Funzionante**
- Verifica che il certificato SSL sia correttamente installato
- Controlla che i permalink di WooCommerce siano configurati correttamente
- Assicurati che i gateway di pagamento web siano attivi e configurati
- Prova con un browser o un dispositivo diverso

**Pagamento Non Elaborato**
- Conferma che i gateway di pagamento web siano configurati correttamente
- Controlla i log del gateway di pagamento per messaggi di errore
- Verifica che il metodo di pagamento del cliente sia supportato
- Prova il processo di checkout web in modo indipendente

**Ordine Non Aggiornato nel POS**
- Controlla la stabilità della connessione Internet
- Verifica che i lavori cron di WordPress stiano funzionando correttamente
- Aggiorna la schermata degli ordini POS
- Controlla lo stato dell'ordine nell'amministratore di WooCommerce

**Il Cliente Non Può Accedere al Checkout**
- Assicurati che il link di checkout non sia scaduto
- Verifica che il dispositivo del cliente abbia connettività Internet
- Controlla che l'ordine non sia stato cancellato o completato
- Prova il link su un dispositivo diverso

### Ottimizzazione delle Prestazioni

Per prestazioni ottimali:

- **Caching**: Configura i plugin di caching per escludere le pagine di checkout
- **CDN**: Assicurati che le impostazioni del CDN non interferiscano con il processo di checkout
- **Database**: Ottimizza il database per un'elaborazione degli ordini più veloce
- **Hosting**: Utilizza un hosting affidabile con buon uptime
