WCPOS supporta più metodi di pagamento per soddisfare le diverse esigenze aziendali. Il checkout utilizza un iframe/webview per caricare la pagina WooCommerce Order Pay, il che significa che qualsiasi gateway di pagamento che funziona con WooCommerce può funzionare nel POS.

## Metodi di Pagamento Predefiniti

La versione gratuita include due gateway di pagamento integrati:

### Contanti

- Inserisci l'importo fornito
- Calcolo automatico del resto
- Nessuna configurazione aggiuntiva richiesta

### Carta

- Per l'uso con terminali di carta esterni
- Contrassegna semplicemente il pagamento come completato dopo l'elaborazione sul tuo terminale
- Non è richiesta alcuna integrazione diretta

## Gateway di Pagamento Aggiuntivi (Pro)

:::info Funzionalità Pro
I gateway di pagamento aggiuntivi richiedono [WCPOS Pro](/getting-started/pro-license).
:::

Con Pro, puoi abilitare gateway di pagamento WooCommerce aggiuntivi nel checkout POS:

- **Stripe Terminal** - Integrazione diretta con i lettori di carte Stripe
- **SumUp Terminal** - Integrazione con i lettori di carte SumUp
- **Any WooCommerce Gateway** - Abilita qualsiasi gateway installato nel tuo negozio

## Abilitazione e Disabilitazione dei Gateway

I gateway di pagamento sono gestiti nell'amministrazione di WordPress:

1. Vai a `WP Admin > POS > Settings > Checkout`
2. Vedrai un elenco di tutti i gateway di pagamento WooCommerce installati
3. Attiva o disattiva ogni gateway per il POS indipendentemente dal tuo negozio online
4. Trascina per riordinare — il primo gateway abilitato diventa l'impostazione predefinita al checkout

Ciò significa che un gateway può essere attivo sul tuo sito Web ma disabilitato nel POS, o viceversa. Vedi [Impostazioni di Checkout](/settings/wp-admin/checkout) per tutti i dettagli.

## Test dei Pagamenti

Prima di andare in diretta con un nuovo gateway:

1. **Gateway di cassa** — sempre disponibile per testare il flusso di checkout senza transazioni reali.
2. **Stripe Terminal** — Stripe fornisce un [lettore simulato](https://docs.stripe.com/terminal/testing) per la modalità test. Abilita la modalità test nel tuo dashboard Stripe per primo.
3. **Gateway di terze parti** — consulta la documentazione del gateway per le istruzioni sulla modalità sandbox/test. La maggior parte dei gateway WooCommerce supporta un interruttore della modalità test.

:::tip
Se un gateway appena abilitato non appare al checkout, prova ad aggiornare il POS. Le modifiche ai gateway in WP Admin hanno effetto al caricamento della pagina successiva.
:::

## Problemi comuni

### Gateway non visualizzato al checkout

- Conferma che il gateway sia abilitato per POS in `WP Admin > POS > Settings > Checkout`.
- Verifica che il gateway sia abilitato anche in `WP Admin > WooCommerce > Settings > Payments` — un gateway deve essere attivo in WooCommerce prima di poter apparire nel POS.
- Aggiorna il POS dopo aver apportato modifiche.

### Il Gateway si Visualizza in Modo Non Corretto

L'iframe di checkout carica gli stili del tema del tuo sito, che a volte possono interferire. Utilizzare il pulsante **Checkout Settings** nella finestra modale di checkout per disabilitare selettivamente gli stili o gli script. Vedere [Risoluzione dei Problemi di Checkout](/pos/checkout#checkout-settings-troubleshooting) per i dettagli.

### Elaborazione dei Pagamenti Non Riuscita

- Controllare i registri degli errori del sito (`WP Admin > POS > Support > Logs`) per errori specifici del gateway.
- Assicurarsi che il certificato SSL sia valido — la maggior parte dei gateway di pagamento richiede HTTPS.
- Se si utilizza un ambiente di staging o locale, confermare che il gateway supporta la modalità test/sandbox.
