WCPOS supporta diversi metodi di pagamento per soddisfare le diverse esigenze aziendali. Il checkout utilizza un iframe/webview per caricare la pagina Paga ordine di WooCommerce, il che significa che qualsiasi gateway di pagamento compatibile con WooCommerce può funzionare nel POS.

## Metodi di pagamento predefiniti {#default-payment-methods}

La versione gratuita include due gateway di pagamento integrati:

### Contanti {#cash}

- Inserimento dell'importo ricevuto
- Calcolo automatico del resto
- Nessuna configurazione aggiuntiva necessaria

### Carta {#card}

- Per l'uso con terminali di pagamento esterni
- Basta contrassegnare il pagamento come completato dopo l'elaborazione sul terminale
- Non è richiesta un'integrazione diretta

## Gateway di pagamento aggiuntivi (Pro) {#additional-payment-gateways-pro}

:::info Funzionalità Pro
I gateway di pagamento aggiuntivi richiedono [WCPOS Pro](/getting-started/pro-license).
:::

Con Pro è possibile abilitare gateway di pagamento WooCommerce aggiuntivi nella cassa del POS:

- **Stripe Terminal** — Integrazione diretta con i lettori di carte Stripe
- **SumUp Terminal** — Integrazione con i lettori di carte SumUp
- **Qualsiasi gateway WooCommerce** — Possibilità di abilitare qualsiasi gateway installato nel negozio

## Abilitazione e disabilitazione dei gateway {#enabling-and-disabling-gateways}

I gateway di pagamento vengono gestiti nel pannello di amministrazione WordPress:

1. Vai su `WP Admin > POS > Impostazioni > Pagamento`
2. Verrà visualizzato un elenco di tutti i gateway di pagamento WooCommerce installati
3. Attiva o disattiva ciascun gateway per il POS indipendentemente dal negozio online
4. Trascina per riordinare — il primo gateway abilitato diventa quello predefinito al checkout

Questo significa che un gateway può essere attivo sul sito web ma disabilitato nel POS, o viceversa. Consulta [Impostazioni checkout](/settings/wp-admin/checkout) per tutti i dettagli.

## Test dei pagamenti {#testing-payments}

Prima di passare in produzione con un nuovo gateway:

1. **Gateway contanti** — sempre disponibile per testare il flusso di checkout senza transazioni reali.
2. **Stripe Terminal** — Stripe fornisce un [lettore simulato](https://docs.stripe.com/terminal/testing) per la modalità di test. Abilitare prima la modalità di test nella dashboard di Stripe.
3. **Gateway di terze parti** — consultare la documentazione del gateway per le istruzioni sulla modalità sandbox/test. La maggior parte dei gateway WooCommerce supporta un'opzione per la modalità di test.

:::tip
Se un gateway appena abilitato non compare al checkout, provare ad aggiornare il POS. Le modifiche ai gateway in WP Admin hanno effetto al successivo caricamento della pagina.
:::

## Problemi comuni {#common-issues}

### Il gateway non appare al checkout {#gateway-not-appearing-at-checkout}

- Verificare che il gateway sia abilitato per il POS in `WP Admin > POS > Impostazioni > Pagamento`.
- Verificare che il gateway sia abilitato anche in `WP Admin > WooCommerce > Impostazioni > Pagamenti` — un gateway deve essere attivo in WooCommerce prima di poter apparire nel POS.
- Aggiornare il POS dopo aver apportato le modifiche.

### Il gateway viene visualizzato in modo errato {#gateway-displays-incorrectly}

L'iframe del checkout carica gli stili del tema del sito, che a volte possono interferire. Utilizzare il pulsante **Impostazioni checkout** nella finestra del checkout per disabilitare selettivamente stili o script. Consultare [Risoluzione problemi checkout](/pos/checkout#checkout-settings-troubleshooting) per i dettagli.

### L'elaborazione del pagamento non riesce {#payment-processing-fails}

- Controllare i log degli errori del sito (`WP Admin > POS > Supporto > Log`) per eventuali errori specifici del gateway.
- Assicurarsi che il certificato SSL sia valido — la maggior parte dei gateway di pagamento richiede HTTPS.
- Se si utilizza un ambiente di staging o locale, verificare che il gateway supporti la modalità test/sandbox.
