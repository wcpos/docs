WCPOS supporta più metodi di pagamento per soddisfare diverse esigenze aziendali. Il checkout utilizza un iframe/webview per caricare la pagina WooCommerce Order Pay, il che significa che qualsiasi gateway di pagamento che funziona con WooCommerce può funzionare nel POS.

## Metodi di Pagamento Predefiniti

La versione gratuita include due gateway di pagamento integrati:

### Contanti

- Inserisci l'importo versato
- Calcolo automatico del resto
- Nessuna configurazione aggiuntiva richiesta

### Carta

- Per l'utilizzo con terminali di carte esterne
- Contrassegna semplicemente il pagamento come completato dopo l'elaborazione sul tuo terminale
- Nessuna integrazione diretta richiesta

## Gateway di Pagamento Aggiuntivi (Pro)

:::info Pro Feature
I gateway di pagamento aggiuntivi richiedono [WCPOS Pro](/getting-started/pro-license).
:::

Con Pro, puoi abilitare gateway di pagamento WooCommerce aggiuntivi nel checkout POS:

- **Stripe Terminal** - Integrazione diretta con i lettori di carte Stripe
- **SumUp Terminal** - Integrazione con i lettori di carte SumUp
- **Any WooCommerce Gateway** - Abilita qualsiasi gateway installato sul tuo negozio

## Abilitazione e Disabilitazione dei Gateway

I gateway di pagamento vengono gestiti nell'admin di WordPress:

1. Vai a `WP Admin > POS > Settings > Checkout`
2. Vedrai un elenco di tutti i gateway di pagamento WooCommerce installati
3. Attiva o disattiva ogni gateway per il POS indipendentemente dal tuo negozio online
4. Trascina per riordinare — il primo gateway abilitato diventa quello predefinito al checkout

Ciò significa che un gateway può essere attivo sul tuo sito web ma disabilitato nel POS, o viceversa. Vedi [Checkout Settings](/settings/wp-admin/checkout) per i dettagli completi.

## Test dei pagamenti

Prima di andare in diretta con un nuovo gateway:

1. **Cash gateway** — sempre disponibile per testare il flusso di checkout senza transazioni reali.
2. **Stripe Terminal** — Stripe fornisce un [lettore simulato](https://docs.stripe.com/terminal/testing) per la modalità di test. Abilita prima la modalità di test nel tuo dashboard Stripe.
3. **Gateway di terze parti** — controlla la documentazione del gateway per le istruzioni sulla modalità sandbox/test. La maggior parte dei gateway WooCommerce supporta un toggle della modalità di test.

:::tip
Se un gateway appena abilitato non appare al checkout, prova ad aggiornare il POS. Le modifiche ai gateway in WP Admin hanno effetto al caricamento della pagina successiva.
:::

## Problemi comuni

### Il gateway non appare al checkout

- Conferma che il gateway sia abilitato per il POS in `WP Admin > POS > Settings > Checkout`.
- Verifica che il gateway sia abilitato anche in `WP Admin > WooCommerce > Settings > Payments` — un gateway deve essere attivo in WooCommerce prima di poter apparire nel POS.
- Aggiorna il POS dopo aver apportato modifiche.

### Il gateway viene visualizzato in modo non corretto

L'iframe del checkout carica gli stili del tema del tuo sito, che a volte possono interferire. Utilizza il pulsante **Checkout Settings** nella finestra modale di checkout per disabilitare selettivamente gli stili o gli script. Consulta [Risoluzione dei problemi di checkout](/pos/checkout#checkout-settings-troubleshooting) per i dettagli.

### Elaborazione del pagamento non riuscita

- Controlla i log degli errori del tuo sito (`WP Admin > POS > Support > Logs`) per errori specifici del gateway.
- Assicurati che il tuo certificato SSL sia valido — la maggior parte dei gateway di pagamento richiede HTTPS.
- Se utilizzi un ambiente di staging o locale, conferma che il gateway supporta la modalità test/sandbox.
