WCPOS include un sistema completo di modelli per ricevute, fatture, ricevute regalo, distinte di imballaggio, comande per la cucina e altro — il tutto gestito in **WP Admin → POS → Modelli**.

:::tip Vuoi solo modificare la ricevuta?
Inizia da **[Personalizza la tua ricevuta](/templates/customise)**. Illustra i tre modi più semplici per farlo: scegliere un modello predefinito diverso, chiedere all'IA di modificarne uno o intervenire manualmente.
:::

## Motori di template {#template-engines}

WCPOS supporta tre motori di template. Il motore viene scelto al momento della creazione di un modello o facendo clic su **Usa modello** nella scheda della galleria e **non può essere modificato in seguito** — per cambiare motore è necessario creare un nuovo modello.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML senza logica (consigliato)">
    Stile Mustache <code>{"{{variable}}"}</code> segnaposto all'interno di HTML semplice. Esegue il rendering lato client, quindi le ricevute **funzionano offline**. Da utilizzare per la stampa da browser e le ricevute PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML termico">
    Modelli XML che producono sia un'anteprima a schermo sia **comandi ESC/POS** per stampanti termiche di ricevute. Stesso modello, due output. Da utilizzare con stampanti termiche Epson, Star e altre.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP legacy">
    Modelli PHP lato server che utilizzano le funzioni di WooCommerce. Mantenuti per retrocompatibilità con gli <code>yourtheme/woocommerce-pos/receipt.php</code> override esistenti. Richiede una connessione al server per il rendering.
  </FeatureCard>
</FeatureGrid>

## Galleria dei modelli {#template-gallery}

La galleria include modelli pronti all'uso che è possibile utilizzare così come sono o personalizzare. È possibile filtrare per **categoria** (Ricevuta, Fattura, Ricevuta regalo, Comanda cucina, Preventivo / Ordine d'acquisto), **formato** (HTML / ESC/POS) e **direzione** (LTR / RTL); l'anteprima utilizza i dati reali del negozio. Nel piè di pagina di ogni scheda sono presenti un link **Anteprima** e un pulsante principale **Usa modello** — facendo clic su Usa modello viene creata una copia modificabile in *I tuoi modelli*.

| Modello | Motore | Descrizione |
|----------|--------|-------------|
| Ricevuta standard | HTML | Ricevuta predefinita — logo, identità del negozio, articoli, totali, pagamento |
| Ricevuta standard (RTL) | HTML | Versione da destra a sinistra per arabo, ebraico, persiano, urdu |
| Minimalista / Moderna | HTML | Stesse informazioni essenziali della Standard, in meno spazio verticale |
| Ricevuta dettagliata | HTML | Fattura fiscale completa — colonna SKU, prezzo unitario, dettaglio imposte per aliquota, indirizzi |
| Ricevuta regalo | HTML | Solo articoli, prezzi nascosti. Include messaggio regalo e politica di reso |
| Fattura | HTML | Fattura a pagina intera A4/Letter con pannello opzionale "Come pagare" per ordini non saldati |
| Documento di trasporto | HTML | Documento per il magazzino — articoli + quantità, indirizzo di spedizione, senza prezzi |
| Preventivo / Stima | HTML | Documento pre-vendita con prezzi e condizioni — senza sezione di pagamento |
| Ricevuta stretta | HTML | Ricevuta a spaziatura fissa per carta stretta o stampanti termiche compatibili HTML |
| Ricevuta termica semplice (58mm) | Termico | Layout termico pulito da 58mm |
| Ricevuta termica semplice (80mm) | Termico | Layout termico pulito da 80mm — il più comune |
| Ricevuta termica semplice 80mm (RTL) | Termico | Versione RTL per 80mm. Richiede una stampante con codepage araba (CP864 / Windows-1256) |
| Ricevuta termica dettagliata (58mm) | Termico | 58mm completa — indirizzi, dettaglio imposte, rimborsi, pagamenti, condizioni, codice a barre |
| Ricevuta termica dettagliata (80mm) | Termica | 80mm completa — indirizzi, dettaglio imposte, rimborsi, pagamenti, termini, codice a barre |
| Comanda cucina | Termica | Solo articoli, carattere grande, senza prezzi — per le postazioni di preparazione |

### Modelli di documento di terze parti {#third-party-document-templates}

Se **PDF Invoices & Packing Slips for WooCommerce** di WP Overnight è attivo, WCPOS espone anche **Fattura (WP Overnight)** e **Bolla di accompagnamento (WP Overnight)** nell'elenco dei modelli di ricevuta. Questi modelli HTML generati dal server si appoggiano all'API documenti di WP Overnight, riutilizzando numerazione di fatture e bolle di accompagnamento, layout, branding, campi fiscali/legali e personalizzazioni dei modelli configurate nel plugin.

Sono disponibili solo quando il plugin WP Overnight è attivo e richiedono una connessione al server per la generazione. Utilizzare i modelli HTML o termici integrati in WCPOS quando è necessaria la stampa offline.

### Visualizzazione adattiva delle imposte {#adaptive-tax-display}

La maggior parte dei modelli integrati (Standard, Standard RTL, Minimale/Moderno, Stretto, Fattura, Preventivo, varianti Termica Semplice) **si adatta automaticamente all'impostazione di visualizzazione imposte di WooCommerce**:

- **Negozi con imposte incluse** (UE/UK/AU) visualizzano i prezzi lordi con una riga "Imposte incluse".
- **Negozi con imposte escluse** (US/CA) visualizzano i prezzi netti con l'imposta aggiunta come riga separata.
- Il totale generale mostra sempre l'importo lordo effettivamente addebitato.

La famiglia **Dettagliata** è una fattura fiscale formale e dettaglia sempre le imposte con righe al netto più un prospetto esplicito, indipendentemente dall'impostazione del negozio.

## Come funziona {#how-it-works}

1. **Aggiungi un modello** — fai clic su **Usa modello** su una scheda della galleria, oppure creane uno da zero.
2. **Personalizzalo** utilizzando l'editor integrato con anteprima in tempo reale, oppure incollalo in ChatGPT / Claude e richiedi le modifiche desiderate.
3. **Attivalo** — attiva l'interruttore **Attivo** in *I tuoi modelli*. Ogni modello di ricevuta attivo viene visualizzato nel menu a discesa della schermata ricevute alla cassa.
4. **Stampa o visualizza** — le ricevute si aprono dopo il pagamento con opzioni per stampare, inviare via email o visualizzare a schermo.

I modelli vengono renderizzati a partire da un [payload dati ricevuta](/templates/receipt-data) standardizzato con sezioni per informazioni sul negozio, righe articolo, totali, imposte, pagamenti, rimborsi e altro. Tutti i campi valuta includono varianti `_display` pre-formattate (ad es., `$29.99` anziché `29.99`).

## Funzionalità Pro {#pro-features}

Con [WCPOS Pro](/getting-started/pro-license), ogni negozio può avere le proprie assegnazioni di modelli, la propria identità visiva (logo, indirizzo, recapiti) e il proprio ordinamento dei modelli — utile per operazioni multi-sede in cui ogni punto vendita stampa la propria intestazione.
