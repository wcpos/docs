WCPOS include un sistema completo di template per ricevute, fatture, ricevute regalo, documenti di trasporto, comande cucina e altro ancora, tutto gestito in **WP Admin → POS → Template**.

:::tip Vuoi solo modificare la ricevuta?
Inizia da **[Personalizzare la ricevuta](/templates/customise)**. Spiega i tre modi più semplici: scegliere un template incluso diverso, chiedere all'IA di modificarlo o intervenire manualmente.
:::

## Motori di template

WCPOS supporta tre motori di template. Il motore viene scelto quando crei un template o fai clic su **Usa template** in una scheda della galleria, e **non può essere cambiato in seguito**. Crea un nuovo template se vuoi cambiare motore.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML senza logica (consigliato)">
    Segnaposto in stile Mustache <code>{"{{variable}}"}</code> dentro semplice HTML. Il rendering avviene lato client, quindi le ricevute **funzionano offline**. Usalo per la stampa dal browser e le ricevute PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML termico">
    Template XML che producono sia un'anteprima a schermo sia **comandi ESC/POS** per stampanti termiche per ricevute. Un unico template, due output. Usalo per Epson, Star e altre stampanti termiche.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP legacy">
    Template PHP lato server che usano funzioni WooCommerce. Sono mantenuti per compatibilità con override esistenti in <code>yourtheme/woocommerce-pos/receipt.php</code>. Richiedono una connessione al server per il rendering.
  </FeatureCard>
</FeatureGrid>

## Galleria dei template

La galleria contiene template pronti che puoi usare così come sono o personalizzare. Filtra per **categoria** (Ricevuta, Fattura, Ricevuta regalo, Comanda cucina, Preventivo / Ordine di acquisto), **formato** (HTML / ESC/POS) e **direzione** (LTR / RTL); l'anteprima usa i dati reali del negozio. Il piè di ogni scheda contiene un link **Anteprima** e il pulsante principale **Usa template**. Facendo clic, viene creata una copia modificabile in *I tuoi template*.

| Template | Motore | Descrizione |
|----------|--------|-------------|
| Ricevuta standard | HTML | Ricevuta predefinita con logo, identità del negozio, articoli, totali e pagamento |
| Ricevuta standard (RTL) | HTML | Variante da destra a sinistra per arabo, ebraico, persiano e urdu |
| Minimal / Moderno | HTML | Gli stessi elementi essenziali della standard, in meno spazio verticale |
| Ricevuta dettagliata | HTML | Fattura fiscale completa con colonna SKU, prezzo unitario, imposte per aliquota e indirizzi |
| Ricevuta regalo | HTML | Solo articoli, prezzi nascosti. Include messaggio regalo e politica di reso |
| Fattura | HTML | Fattura A4/Letter a pagina intera con pannello opzionale "Come pagare" per ordini non pagati |
| Documento di trasporto | HTML | Documento per magazzino: articoli + quantità, indirizzo di spedizione, nessun prezzo |
| Preventivo / Stima | HTML | Documento pre-vendita con prezzi e condizioni, senza sezione pagamento |
| Ricevuta stretta | HTML | Ricevuta monospace per carta stretta o stampanti termiche compatibili con HTML |
| Ricevuta termica semplice (58 mm) | Termico | Layout termico pulito da 58 mm |
| Ricevuta termica semplice (80 mm) | Termico | Layout termico pulito da 80 mm, il più comune |
| Ricevuta termica semplice 80 mm (RTL) | Termico | Variante RTL per 80 mm. Richiede una stampante con codepage araba (CP864 / Windows-1256) |
| Ricevuta termica dettagliata (58 mm) | Termico | 58 mm completa: indirizzi, imposte, rimborsi, pagamenti, termini e codice a barre |
| Ricevuta termica dettagliata (80 mm) | Termico | 80 mm completa: indirizzi, imposte, rimborsi, pagamenti, termini e codice a barre |

### Template di documenti di terze parti

Se **PDF Invoices & Packing Slips for WooCommerce** di WP Overnight è attivo, WCPOS mostra anche **Fattura (WP Overnight)** e **Documento di trasporto (WP Overnight)** nell'elenco dei template ricevuta. Questi template HTML renderizzati sul server delegano all'API documenti di WP Overnight, riutilizzando numerazione, layout, branding, campi legali/fiscali e personalizzazioni configurate in quel plugin.

Sono disponibili solo mentre il plugin WP Overnight è attivo e richiedono una connessione al server per il rendering. Usa i template HTML o termici inclusi in WCPOS quando serve la stampa offline.

### Visualizzazione adattiva delle imposte

La maggior parte dei template inclusi (Standard, Standard RTL, Minimal/Moderno, Stretta, Fattura, Preventivo e varianti termiche semplici) **si adatta automaticamente all'impostazione WooCommerce di visualizzazione delle imposte**:

- I **negozi con imposte incluse** (UE/Regno Unito/AU) vedono prezzi lordi con una riga "Imposte incluse".
- I **negozi con imposte escluse** (USA/CA) vedono prezzi netti con l'imposta aggiunta su una riga separata.
- Il totale finale mostra sempre l'importo lordo effettivamente addebitato.

La famiglia **Dettagliata** è una fattura fiscale formale e mostra sempre le imposte con righe al netto e una ripartizione esplicita, indipendentemente dall'impostazione del negozio.

## Come funziona

1. **Aggiungi un template**: fai clic su **Usa template** in una scheda della galleria oppure creane uno da zero.
2. **Personalizzalo** con l'editor integrato e l'anteprima live, oppure incollalo in ChatGPT / Claude e chiedi le modifiche desiderate.
3. **Attivalo**: abilita l'interruttore **Attivo** in *I tuoi template*. Ogni template ricevuta attivo appare nel menu a discesa della schermata ricevuta al registratore.
4. **Stampa o mostra**: le ricevute si aprono dopo il checkout con opzioni per stampare, inviare via e-mail o visualizzare a schermo.

I template vengono renderizzati usando un [payload dati ricevuta](/templates/receipt-data) standardizzato, con sezioni per informazioni del negozio, righe articolo, totali, imposte, pagamenti, rimborsi e altro. Tutti i campi valuta includono varianti `_display` già formattate, ad esempio `$29.99` invece di `29.99`.

## Funzionalità Pro

Con [WCPOS Pro](/getting-started/pro-license), ogni negozio può avere assegnazioni di template, branding (logo, indirizzo, contatti) e ordine dei template propri. È utile per attività multi-sede in cui ogni punto vendita stampa la propria intestazione.
