IT: # WCPOS WPML

IT: Integrates [WPML](https://wpml.org/) with WCPOS so the POS only serves products for a single language — translated duplicates stop appearing in product search and the catalog grid. On WCPOS Pro, you can pin a language per store.

## Tradotto: What It Does {#what-it-does}

- IT: Filters WCPOS product and variation REST queries to a single WPML language.
- IT: Intercepts WCPOS **fast-sync** routes (the lightweight requests the POS uses to refresh its local index) so translated duplicates never reach the client.
- IT: Free installs use the WPML default language.
- IT: Pro installs can override the language per store from the store editor.

## Tradotto: Installation {#installation}

<Steps>
  <StepCard number="1" title="IT: Install WPML">

IT: Install and configure [WPML](https://wpml.org/) as normal with at least one language set as the default.

  </StepCard>
  <StepCard number="2" title="IT: Install WCPOS WPML">

IT: Install from `WP Admin > POS > Settings > Extensions`, or download the latest release from the [GitHub releases page](https://github.com/wcpos/wcpos-wpml/releases) and upload via `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="IT: (Pro) Pin a language per store" isLast>

IT: On WCPOS Pro, edit a store under `POS > Stores` and pick its language from the **Language** sidebar section. Leave at *Default* to use the WPML default language.

  </StepCard>
</Steps>

## Tradotto: Known WPML Compatibility Issues {#known-wpml-compatibility-issues}

IT: These are behaviours of WPML itself rather than the integration — worth knowing before rolling out multilingual in production:

- IT: **POS custom fields don't carry across translations.** WPML translates products but does not copy WCPOS custom fields to translated versions by default. A product marked "POS Only" in the default language may lose that setting on its translations. Configure WPML to copy WCPOS custom fields during translation.
- IT: **POS-only products and 404s on the storefront.** Because WPML generates storefront pages for each language, POS-only products may render as 404s when accessed on the website. This is a known WPML interaction, not a WCPOS bug.

IT: See [POS-Only Products](/products/pos-only-products) for the related POS visibility controls.
