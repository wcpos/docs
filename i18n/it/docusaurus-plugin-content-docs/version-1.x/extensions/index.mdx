WCPOS supporta estensioni che aggiungono nuove funzionalità al tuo punto vendita. La directory delle estensioni consente di sfogliare le estensioni disponibili, installarle direttamente dalle impostazioni del POS e gestire gli aggiornamenti.

:::info Funzionalità Pro
L'installazione e la gestione delle estensioni richiedono [WCPOS Pro](/getting-started/pro-license). La versione gratuita mostra il catalogo delle estensioni ma disabilita i controlli di installazione e attivazione.
:::

## Esplorare le estensioni {#browsing-extensions}

Apri la directory delle estensioni da `POS Settings > Extensions` (in alcune versioni indicata anche come **Plugins**).

La directory mostra una griglia di card con le estensioni disponibili. Ogni card include:

- **Icona** (o un pezzo di puzzle come segnaposto se l'estensione non ne fornisce una)
- **Nome e versione**
- **Descrizione**
- **Badge categoria**
- **Stato** — attiva, inattiva, aggiornamento disponibile o non installata

### Filtri e ricerca {#filtering-and-search}

Utilizza i **pulsanti pill delle categorie** in alto per filtrare le estensioni per categoria. Puoi anche utilizzare il **campo di ricerca** per trovare le estensioni per nome, descrizione o tag.

## Installazione di un'estensione {#installing-an-extension}

1. Apri `POS Settings > Extensions`.
2. Individua l'estensione desiderata e fai clic su **Installa**.
3. L'estensione viene scaricata e installata utilizzando il programma di installazione dei plugin di WordPress.
4. Una volta installata, fai clic su **Attiva** per abilitarla.

Dietro le quinte, WCPOS utilizza il `Plugin_Upgrader` nativo di WordPress per gestire l'installazione, quindi le estensioni seguono lo stesso processo di qualsiasi plugin WordPress.

## Attivazione e disattivazione {#activating-and-deactivating}

Ogni estensione installata dispone dei pulsanti **Attiva** e **Disattiva** sulla propria scheda.

- **Attiva** abilita l'estensione in modo che possa essere eseguita nel POS.
- **Disattiva** la disabilita senza disinstallarla. I file dell'estensione rimangono sul server e possono essere riattivati in qualsiasi momento.

## Aggiornamento delle estensioni {#updating-extensions}

Quando è disponibile una versione più recente di un'estensione installata, la scheda mostra un badge **Aggiornamento disponibile** e un pulsante **Aggiorna**.

### Aggiornamenti automatici {#auto-updates}

Le estensioni installate dalla directory hanno gli **aggiornamenti automatici abilitati per impostazione predefinita**. È possibile attivare o disattivare gli aggiornamenti automatici per ciascuna estensione dalla relativa scheda nella directory. Quando l'aggiornamento automatico è attivo, WordPress applica automaticamente le nuove versioni, esattamente come avviene per i plugin con aggiornamento automatico abilitato.

## Risoluzione dei problemi {#troubleshooting}

### Messaggio "Richiede Pro" sui pulsanti {#requires-pro-message-on-buttons}

I pulsanti di installazione, attivazione e aggiornamento sono disabilitati nella versione gratuita di WCPOS. Esegui l'upgrade a [WCPOS Pro](/getting-started/pro-license) per gestire le estensioni.

### L'estensione non si installa {#extension-fails-to-install}

- Verifica che il server WordPress disponga dei permessi di scrittura sulla directory `wp-content/plugins`.
- Verifica che il server possa effettuare richieste HTTPS in uscita (alcuni hosting bloccano i download esterni).
- Consulta i dettagli dell'errore in `WP Admin > POS > Support > Logs`.

### L'estensione non appare dopo l'installazione {#extension-not-appearing-after-install}

- Aggiorna il POS — l'elenco delle estensioni viene memorizzato nella cache per un massimo di 12 ore.
- Verifica che l'estensione sia attivata (le estensioni installate ma non attive non vengono eseguite).

### Il catalogo non si carica {#catalog-not-loading}

Il catalogo delle estensioni viene recuperato da una fonte remota e memorizzato nella cache locale per 12 ore. Se il catalogo non si carica:

- Verifica la connettività Internet del server.
- Riprova dopo la scadenza della cache oppure svuota la cache transient del server.

## Per gli sviluppatori {#for-developers}

### Creare un'estensione POS {#creating-a-pos-extension}

Un'estensione WCPOS è un plugin WordPress standard che si integra con il POS tramite hook e API di WCPOS. Per crearne una:

1. **Inizia con un plugin WordPress.** L'estensione necessita di un header standard del plugin e di un file di ingresso, esattamente come qualsiasi plugin WooCommerce o WordPress.
2. **Integra con WCPOS.** Utilizza gli hook e i filtri forniti da WCPOS per aggiungere funzionalità all'interfaccia POS o al backend.
3. **Ospita le release su GitHub.** La directory delle estensioni utilizza le Release di GitHub per tenere traccia delle versioni e distribuire gli aggiornamenti.

### Invio alla directory {#submitting-to-the-directory}

Il catalogo delle estensioni è mantenuto nel repository GitHub [`wcpos/extensions`](https://github.com/wcpos/extensions). Per inserire la tua estensione:

1. Consulta il formato del catalogo e i requisiti dei metadati nel README del repository.
2. Apri una pull request per aggiungere i metadati della tua estensione a `catalog.json`.
3. Una volta effettuato il merge, l'estensione comparirà nella directory per tutti gli utenti WCPOS Pro.

### Convenzioni per le release GitHub {#github-release-conventions}

Il ciclo di vita degli aggiornamenti si basa sulle Release di GitHub:

- **Assegna i tag delle versioni** utilizzando il versionamento semantico (ad esempio, `v1.0.0`, `v1.2.3`).
- **Allega lo zip del plugin** come asset della release — questo è il file che viene scaricato quando un utente installa o aggiorna.
- **Pubblica la release** (non come bozza) in modo che la directory possa rilevarla.

Quando pubblichi una nuova release, gli utenti che hanno installato la tua estensione vedranno l'aggiornamento disponibile nella loro directory delle estensioni. Se l'aggiornamento automatico è abilitato, verrà applicato automaticamente.

Per tutti i dettagli sullo schema del catalogo e sul processo di invio, consulta il repository [`wcpos/extensions`](https://github.com/wcpos/extensions).
