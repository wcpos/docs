WCPOS supporta estensioni che aggiungono nuove funzionalità al punto vendita. La directory delle estensioni consente di sfogliare le estensioni disponibili, installarle direttamente dalle impostazioni del POS e gestire gli aggiornamenti.

:::info Funzionalità Pro
L'installazione e la gestione delle estensioni richiedono [WCPOS Pro](/getting-started/pro-license). La versione gratuita visualizza il catalogo delle estensioni ma disabilita i controlli di installazione e attivazione.
:::

## Esplorare le estensioni {#browsing-extensions}

Aprire la directory delle estensioni da `POS Settings > Estensioni` (indicata anche come **Plugin** in alcune versioni).

La directory mostra una griglia di schede con le estensioni disponibili. Ogni scheda riporta:

- **Icona** (o un'icona segnaposto a forma di puzzle se l'estensione non ne fornisce una)
- **Nome e versione**
- **Descrizione**
- **Badge categoria**
- **Stato** — attiva, inattiva, aggiornamento disponibile o non installata

### Filtri e ricerca {#filtering-and-search}

Utilizzare i **pulsanti a pillola delle categorie** in alto per filtrare le estensioni per categoria. È inoltre possibile utilizzare il **campo di ricerca** per trovare estensioni per nome, descrizione o tag.

## Installazione di un'estensione {#installing-an-extension}

1. Aprire `POS Settings > Estensioni`.
2. Trovare l'estensione desiderata e fare clic su **Installa**.
3. L'estensione viene scaricata e installata tramite il programma di installazione dei plugin di WordPress.
4. Una volta installata, fare clic su **Attiva** per abilitarla.

Dietro le quinte, WCPOS utilizza il `Plugin_Upgrader` nativo di WordPress per gestire l'installazione, quindi le estensioni seguono lo stesso processo di qualsiasi plugin WordPress.

## Attivazione e disattivazione {#activating-and-deactivating}

Ogni estensione installata ha i pulsanti **Attiva** e **Disattiva** sulla propria scheda.

- **Attiva** abilita l'estensione in modo che possa funzionare nel POS.
- **Disattiva** la disabilita senza disinstallarla. I file dell'estensione rimangono sul server e possono essere riattivati in qualsiasi momento.

## Aggiornamento delle estensioni {#updating-extensions}

Quando è disponibile una versione più recente di un'estensione installata, la scheda mostra un badge **Aggiornamento disponibile** e un pulsante **Aggiorna**.

### Aggiornamenti automatici {#auto-updates}

Le estensioni installate dalla directory hanno l'**aggiornamento automatico abilitato per impostazione predefinita**. È possibile attivare o disattivare gli aggiornamenti automatici per ogni estensione dalla relativa scheda nella directory. Quando l'aggiornamento automatico è attivo, WordPress applica automaticamente le nuove versioni, esattamente come avviene per i plugin con aggiornamento automatico abilitato.

## Risoluzione dei problemi {#troubleshooting}

### Messaggio "Richiede Pro" sui pulsanti {#requires-pro-message-on-buttons}

I pulsanti di installazione, attivazione e aggiornamento sono disabilitati nella versione gratuita di WCPOS. Effettuare l'upgrade a [WCPOS Pro](/getting-started/pro-license) per gestire le estensioni.

### Installazione dell'estensione non riuscita {#extension-fails-to-install}

- Verificare che il server WordPress abbia i permessi di scrittura nella directory `wp-content/plugins`.
- Verificare che il server possa effettuare richieste HTTPS in uscita (alcuni hosting bloccano i download esterni).
- Consultare i dettagli dell'errore in `WP Admin > POS > Supporto > Log`.

### Estensione non visibile dopo l'installazione {#extension-not-appearing-after-install}

- Aggiornare il POS — l'elenco delle estensioni è memorizzato nella cache per un massimo di 12 ore.
- Verificare che l'estensione sia attivata (le estensioni installate ma non attive non verranno eseguite).

### Catalogo non caricato {#catalog-not-loading}

Il catalogo delle estensioni viene recuperato da una fonte remota e memorizzato nella cache locale per 12 ore. Se il catalogo non si carica:

- Verificare la connessione a Internet del server.
- Riprovare dopo la scadenza della cache oppure svuotare la cache dei transienti del server.

## Per sviluppatori {#for-developers}

### Creare un'estensione POS {#creating-a-pos-extension}

Un'estensione WCPOS è un plugin WordPress standard che si integra con il POS tramite hook e API di WCPOS. Per crearne una:

1. **Partire da un plugin WordPress.** L'estensione richiede un header standard e un file di ingresso, come qualsiasi plugin WooCommerce o WordPress.
2. **Integrarsi con WCPOS.** Utilizzare gli hook e i filtri forniti da WCPOS per aggiungere funzionalità all'interfaccia POS o al backend.
3. **Pubblicare le release su GitHub.** La directory delle estensioni utilizza le GitHub Releases per tracciare le versioni e distribuire gli aggiornamenti.

### Invio alla directory {#submitting-to-the-directory}

Il catalogo delle estensioni è mantenuto nel repository GitHub [`wcpos/extensions`](https://github.com/wcpos/extensions). Per inserire la propria estensione:

1. Consultare il formato del catalogo e i requisiti dei metadati nel README del repository.
2. Aprire una pull request per aggiungere i metadati dell'estensione a `catalog.json`.
3. Una volta effettuato il merge, l'estensione apparirà nella directory per tutti gli utenti WCPOS Pro.

### Convenzioni per le GitHub Release {#github-release-conventions}

Il ciclo di aggiornamento si basa sulle GitHub Releases:

- **Taggare le versioni** utilizzando il versionamento semantico (es. `v1.0.0`, `v1.2.3`).
- **Allegare lo zip del plugin** come asset della release — questo è il file che viene scaricato quando un utente installa o aggiorna.
- **Pubblicare la release** (non bozza) affinché la directory possa rilevarla.

Quando si pubblica una nuova release, gli utenti con l'estensione installata vedranno l'aggiornamento disponibile nella loro directory delle estensioni. Se l'aggiornamento automatico è abilitato, verrà applicato automaticamente.

Per tutti i dettagli sullo schema del catalogo e sul processo di invio, consultare il repository [`wcpos/extensions`](https://github.com/wcpos/extensions).
