WCPOS consente di lavorare con più ordini contemporaneamente. Questo è utile per gestire attese dei clienti, passare tra transazioni e riprendersi da interruzioni.

## Carosello Ordini Aperti

Nella parte inferiore del Pannello Carrello, un carosello orizzontale mostra tutti gli ordini aperti:

{/* Screenshot: open-orders-carousel.png */}

- Ogni carrello mostra l'**importo totale**
- L'**ordine corrente** è evidenziato
- Fare clic su un carrello per passare ad esso
- Scorrere a sinistra/destra per vedere più carrelli

## Creazione di un Nuovo Ordine

Un nuovo carrello vuoto è sempre disponibile. Semplicemente fare clic su un carrello vuoto nel carosello o iniziare ad aggiungere prodotti quando il carrello corrente è vuoto.

## Passare tra gli Ordini

Fare clic su qualsiasi ordine nel carosello per passare ad esso. Il Pannello Carrello si aggiorna per mostrare il contenuto dell'ordine selezionato.

**Casi d'uso:**

- Il cliente si allontana per prendere un altro articolo
- Necessità di assistere un cliente veloce mentre è in corso un ordine grande
- Confrontare prezzi o articoli tra ordini

## Salvare gli Ordini sul Server

Gli ordini esistono in due stati:

### Solo Locali

Per impostazione predefinita, i nuovi ordini sono archiviati solo nel database del browser/app locale. Persisteranno attraverso gli aggiornamenti della pagina ma:

- Non sono visibili nell'admin di WooCommerce
- Andranno persi se il database locale viene cancellato
- Non sono accessibili da altri dispositivi

### Salvati sul Server

Fare clic su **Salva sul Server** per creare un ordine WooCommerce con stato `pos-open`. Questo:

- Crea un ordine reale in WooCommerce
- Persiste anche se il database locale viene cancellato
- Può essere accessibile da altri dispositivi
- Appare in WP Admin > WooCommerce > Ordini

:::tip Quando Salvare
Salvare gli ordini sul server quando:
- Un cliente vuole mettere da parte un ordine per un ritiro successivo
- Si sta finendo il turno e un altro cassiere continuerà
- Si vuole un backup in caso di problemi con l'app/browser
:::

## Recuperare Ordini Salvati

Se hai salvato ordini sul server, possono essere nuovamente accessibili:

1. Aprendo la schermata **Ordini** (funzione Pro)
2. Filtrando per stato `pos-open`
3. Riaprendo l'ordine

## Persistenza dell'Ordine

### Archiviazione Locale

WCPOS usa IndexedDB per archiviare gli ordini localmente. Questo fornisce:

- Persistenza tra sessioni del browser
- Accesso rapido senza richieste di rete
- Capacità offline

### Sincronizzazione con il Server

Quando si salva sul server o si effettua il checkout:

- L'ordine viene inviato a WooCommerce
- Viene ricevuta una conferma
- Dati locali e server sono sincronizzati

## Annullamento Ordini

Per rimuovere un ordine aperto:

1. Passare all'ordine che si desidera rimuovere
2. Fare clic sul pulsante **Annulla**

**Cosa succede:**

- **Ordini non salvati:** Eliminati permanentemente dal database locale
- **Ordini salvati:** Spostati nel Cestino di WooCommerce ed eliminati localmente

Per recuperare un ordine salvato annullato:
1. Andare a `WP Admin > WooCommerce > Ordini > Cestino`
2. Ripristinare l'ordine

## Suggerimenti

### Mantenere gli Ordini Organizzati

Con più ordini aperti, è utile:

- Aggiungere nomi dei clienti agli ordini per un'identificazione facile
- Aggiungere note all'ordine che descrivono il motivo dell'attesa
- Salvare ordini importanti sul server

### Passaggi di Turno

Quando si termina un turno con ordini aperti:

1. Salvare tutti gli ordini importanti sul server
2. Aggiungere note agli ordini che spiegano lo stato
3. Il prossimo cassiere può accedervi dalla schermata Ordini

### Considerazioni Offline

In caso di perdita di connettività:

- Gli ordini locali rimangono accessibili e si possono continuare ad aggiungere articoli
- Non è possibile completare/effettuare il checkout degli ordini fino al ripristino della connettività
- Non è possibile salvare ordini sul server fino alla riconnessione
- Non è possibile creare nuovi clienti fino alla riconnessione
