WCPOS ti consente di lavorare con più ordini simultaneamente. Questo è utile per gestire le attese dei clienti, passare da una transazione all'altra e recuperare da interruzioni.

## Carousel degli Ordini Aperti

In fondo al Pannello Carrello, un carosello orizzontale visualizza tutti gli ordini aperti:

{/* Screenshot: open-orders-carousel.png */}

- Ogni carrello mostra il suo **importo totale**
- L'**ordine corrente** è evidenziato
- Clicca su qualsiasi carrello per passare ad esso
- Scorri a sinistra/destra per vedere più carrelli

## Creazione di un Nuovo Ordine

Un nuovo carrello vuoto è sempre disponibile. Basta cliccare su un carrello vuoto nel carosello o iniziare ad aggiungere prodotti quando il carrello corrente è vuoto.

## Passare tra Ordini

Clicca su qualsiasi ordine nel carosello per passare ad esso. Il Pannello Carrello si aggiorna per mostrare il contenuto dell'ordine selezionato.

**Casi d'uso:**

- Il cliente si allontana per prendere un altro articolo
- È necessario aiutare un cliente veloce mentre è in corso un grande ordine
- Confrontare prezzi o articoli tra ordini

## Salvataggio Ordini nel Server

Gli ordini esistono in due stati:

### Solo Locale

Per impostazione predefinita, i nuovi ordini vengono memorizzati solo nel database locale del browser/app. Essi persisteranno attraverso i refresh della pagina ma:

- Non sono visibili nell'amministrazione di WooCommerce
- Saranno persi se il database locale viene cancellato
- Non sono accessibili da altri dispositivi

### Salvati nel Server

Clicca su **Salva nel Server** per creare un ordine WooCommerce con lo stato `pos-open`. Questo:

- Crea un ordine reale in WooCommerce
- Persiste anche se il database locale viene cancellato
- Può essere accessibile da altri dispositivi
- Appare in WP Admin > WooCommerce > Ordini

:::tip Quando Salvare
Salva gli ordini nel server quando:
- Un cliente desidera tenere un ordine per il ritiro successivo
- Stai terminando il tuo turno e un altro cassiere continuerà
- Vuoi un backup in caso di problemi con l'app/browser
:::

## Recupero di Ordini Salvati

Se hai salvato ordini nel server, possono essere nuovamente accessibili mediante:

1. Apertura della schermata **Ordini** (funzione Pro)
2. Filtraggio per stato `pos-open`
3. Riapertura dell'ordine

## Persistenza degli Ordini

### Memoria Locale

WCPOS utilizza IndexedDB per memorizzare gli ordini localmente. Questo fornisce:

- Persistenza attraverso le sessioni del browser
- Accesso veloce senza richieste di rete
- Capacità offline

### Sincronizzazione con il Server

Quando salvi nel server o fai il checkout:

- L'ordine viene inviato a WooCommerce
- Viene ricevuta una conferma
- I dati locali e del server vengono sincronizzati

## Annullamento degli Ordini

Per rimuovere un ordine aperto:

1. Passa all'ordine che vuoi rimuovere
2. Clicca sul pulsante **Annulla**

**Cosa succede:**

- **Ordini non salvati:** Cancellati permanentemente dal database locale
- **Ordini salvati:** Spostati nel Cestino in WooCommerce e cancellati localmente

Per recuperare un ordine salvato annullato:
1. Vai su `WP Admin > WooCommerce > Ordini > Cestino`
2. Ripristina l'ordine

## Suggerimenti

### Mantieni Ordinati gli Ordini

Con più ordini aperti, è utile:

- Aggiungere nomi dei clienti agli ordini per una facile identificazione
- Aggiungere note sugli ordini che descrivono il motivo della sospensione
- Salvare ordini importanti nel server

### Passaggio di Turni

Quando termini un turno con ordini aperti:

1. Salva tutti gli ordini importanti nel server
2. Aggiungi note sugli ordini che spiegano lo stato
3. Il prossimo cassiere può accedervi dalla schermata Ordini

### Considerazioni Offline

Se perdi connettività:

- Gli ordini locali rimangono accessibili e puoi continuare ad aggiungere articoli
- Non puoi completare/fare checkout degli ordini fino a quando la connettività non viene ripristinata
- Non puoi salvare ordini nel server fino a quando non sei riconnesso
- Non puoi creare nuovi clienti fino a quando non sei riconnesso
