- Apre un dialogo per aggiungere una nota cliente all'ordine.
- Le note dei clienti sono visibili al cliente e appaiono di default sulla ricevuta, nell'area Il Mio Account del cliente e in `WP Admin > WooCommerce > Ordini > Ordine` sotto l'indirizzo di spedizione.

:::note
Le note dell'ordine sono visibili ai clienti.
:::

- Apre una modale per gestire metadati aggiuntivi per l'ordine.
- Funzionalità incluse:
  - Cambiare la valuta dell'ordine.
  - Aggiungere un ID transazione (ad es. da un terminale di pagamento esterno).
  - Aggiungere metadati personalizzati per l'ordine.

:::note
Ulteriori funzionalità verranno aggiunte a questa modale nel tempo, inclusa l'integrazione con altri plugin WooCommerce.
:::

### Cosa sono i Metadati in WooCommerce?

I metadati forniscono informazioni aggiuntive sugli ordini ed è possibile accedervi tramite l'API REST di WooCommerce. Sviluppatori e plugin utilizzano questi dati per estendere la funzionalità, come l'integrazione con servizi di terze parti.

### Vista JSON

- Visualizza la rappresentazione JSON grezza dell'ordine, utile per debug o scopi di integrazione.

## Salva sul Server

- Consente di salvare carrelli aperti sul server, in modo che siano archiviati in modo sicuro e accessibili per un uso futuro.
- Crea un ordine WooCommerce con stato `pos-open`.

### Perché salvare?

Quando viene creato un nuovo ordine nel POS, esiste solo localmente. 
Salvare l'ordine garantisce che persista, anche se il database locale viene cancellato.

Consulta [Ordini Aperti](/pos/cart/open-orders) per maggiori dettagli sulla gestione degli ordini salvati.

## Annulla

- Due Scenari:
  - Ordini Non Salvati:
    - Se l'ordine non è stato salvato sul server, verrà eliminato permanentemente dal database locale.
  - Ordini Salvati:
    - Se l'ordine è salvato sul server, verrà spostato nella cartella Cestino di WooCommerce ed eliminato dal database locale.
- Per recuperare un ordine annullato:
  - Vai a `WP Admin > WooCommerce > Ordini > Cestino`.

## Checkout

- Salva l'ordine sul server e avvia il processo di checkout.
- Apre la [schermata di checkout](/pos/checkout) per completare il pagamento.
