# API02010: Autenticazione Richiesta

## Cosa Significa

L'azione che si sta cercando di eseguire richiede l'autenticazione, ma non si è attualmente collegati. Il POS necessita di credenziali valide per accedere a questa risorsa.

## Cause Comuni

- **Non collegato** — Non è stata ancora eseguita l'autenticazione
- **Sessione cancellata** — La sessione è stata cancellata o è scaduta
- **Accesso a risorsa protetta** — La risorsa richiede l'autenticazione
- **Dati dell'app cancellati** — Le credenziali memorizzate sono state rimosse

## Mapping Errore Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Origine |
|--------------|---------|
| `rest_login_required` | WordPress REST API |
| `jwt_auth_no_auth_header` | Plugin Autenticazione JWT |
| HTTP 401 | Qualsiasi risposta del server (fallback) |

## Come Risolvere

### 1. Accedi

Se non hai ancora effettuato l'accesso:
1. Apri la schermata di accesso del POS
2. Inserisci le tue credenziali WordPress
3. Completa il processo di autenticazione

### 2. Controlla lo Stato della Sessione

Se pensavi di essere già collegato:
- La tua sessione potrebbe essere scaduta
- Consulta [API02002](./API02002) (Token Scaduto) per ulteriori dettagli
- Accedi nuovamente per ripristinare l'accesso

### 3. Verifica la Configurazione del Server

Assicurati che gli endpoint API siano configurati correttamente:
- La REST API di WooCommerce dovrebbe essere abilitata
- Il plugin WCPOS dovrebbe essere attivo
- Gli endpoint di autenticazione dovrebbero essere accessibili

### 4. Controlla Problemi del Browser/App

Se vieni disconnesso inaspettatamente:
- Cancella la cache del browser (se usi la versione web)
- Verifica che cookie e archivio locale non siano bloccati
- Conferma che l'app abbia il permesso di memorizzare dati

## Cosa Richiede Autenticazione?

La maggior parte delle operazioni POS richiede l'autenticazione:
- Visualizzazione di prodotti e clienti
- Creazione e modifica di ordini
- Elaborazione pagamenti
- Accesso a report

Solo la schermata di accesso iniziale è accessibile senza autenticazione.
