# API02010: Autenticazione Richiesta {#api02010-auth-required}

## Cosa Significa {#what-this-means}

L'azione che stai cercando di eseguire richiede autenticazione, ma attualmente non sei connesso. Il POS ha bisogno di credenziali valide per accedere a questa risorsa.

## Cause Comuni {#common-causes}

- **Non connesso** — Non ti sei ancora autenticato
- **Sessione cancellata** — La tua sessione è stata cancellata o è scaduta
- **Accesso a risorsa protetta** — La risorsa richiede autenticazione
- **Dati dell'app eliminati** — Le credenziali memorizzate sono state rimosse

## Mappatura degli Errori del Server {#server-error-mapping}

Codice di errore attivato quando il server restituisce:

| Codice Server | Fonte |
|---------------|-------|
| `rest_login_required` | WordPress REST API |
| `jwt_auth_no_auth_header` | Plugin di Autenticazione JWT |
| HTTP 401 | Qualsiasi risposta del server (fallback) |

## Come Risolvere {#how-to-fix}

### 1. Effettua il Login {#1-log-in}

Se non hai effettuato il login:
1. Apri la schermata di accesso del POS
2. Inserisci le tue credenziali di WordPress
3. Completa il processo di autenticazione

### 2. Controlla lo Stato della Sessione {#2-check-session-status}

Se pensavi di essere connesso:
- La tua sessione potrebbe essere scaduta
- Cerca [API02002](./API02002) (Token Scaduto) per ulteriori dettagli
- Effettua nuovamente il login per ripristinare l'accesso

### 3. Verifica la Configurazione del Server {#3-verify-server-configuration}

Assicurati che gli endpoint API siano configurati correttamente:
- WooCommerce REST API dovrebbe essere abilitata
- Il plugin WCPOS dovrebbe essere attivo
- Gli endpoint di autenticazione dovrebbero essere accessibili

### 4. Controlla Problemi con Browser/App {#4-check-for-browserapp-issues}

Se vieni disconnesso improvvisamente:
- Cancella la cache del browser (se utilizzi la versione web)
- Verifica che i cookie/storage locale non siano bloccati
- Verifica che l'app abbia il permesso di memorizzare dati

## Cosa Richiede Autenticazione? {#what-requires-authentication}

La maggior parte delle operazioni POS richiede autenticazione:
- Visualizzazione di prodotti e clienti
- Creazione e modifica di ordini
- Elaborazione dei pagamenti
- Accesso ai report

Solo la schermata di login iniziale è accessibile senza autenticazione.
