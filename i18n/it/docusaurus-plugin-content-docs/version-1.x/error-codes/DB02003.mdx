# DB02003: Violazione del Vincolo

## Cosa Significa

I dati che stai cercando di salvare violano le regole del database. I vincoli garantiscono l'integrità dei dati applicando regole su cosa può essere memorizzato.

## Cause Comuni

- **Dati obbligatori mancanti** — Un campo obbligatorio è vuoto
- **Relazione non valida** — Riferimento a un record inesistente
- **Tipo di dati non corrispondente** — Tipo di dati errato per il campo
- **Valore fuori intervallo** — Numero che supera i limiti consentiti

## Come Risolvere

### 1. Verificare i Campi Obbligatori

Assicurati che tutti i dati obbligatori siano forniti:
- Informazioni cliente (se richieste)
- Dettagli prodotto
- Voci dell'ordine

### 2. Verificare i Riferimenti

Se l'errore riguarda le relazioni:
- Assicurarsi che i prodotti referenziati esistano
- Controllare che gli ID cliente siano validi
- Verificare le assegnazioni di categorie

### 3. Rivedere i Valori dei Dati

Controllare valori non validi:
- Quantità negative dove non consentite
- Prezzi che superano i limiti
- Valori di stato non validi

### 4. Sincronizzare i Dati Più Recenti

I dati referenziati potrebbero non essere sincronizzati:
- Aggiornare i dati dal server
- Attendere il completamento della sincronizzazione
- Riprovare l'operazione

### 5. Cancellare e Riprovare

Se i dati sono corrotti:
1. Cancellare il modulo problematico
2. Reinserire i dati
3. Inviare nuovamente

## Esempi Comuni di Vincoli

- **La quantità deve essere positiva** — Non è possibile aggiungere 0 o elementi negativi
- **Il prezzo deve essere numerico** — Non è consentito testo nei campi prezzo
- **Il cliente deve esistere** — Non è possibile assegnare un ordine a un cliente inesistente
