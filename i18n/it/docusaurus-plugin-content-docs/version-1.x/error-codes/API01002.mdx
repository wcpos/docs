# API01002: Connessione Rifiutata

## Cosa Significa

Il server ha rifiutato attivamente il tentativo di connessione. Questo è diverso da un timeout — il server ha risposto, ma ha detto "no".

## Cause Comuni

- **Server non funzionante** — Il server web (Apache/Nginx) potrebbe non essere in esecuzione
- **Porta errata** — Tentativo di connessione sulla porta sbagliata
- **Firewall che blocca** — Un firewall sta bloccando la connessione
- **Restrizioni IP** — Il tuo indirizzo IP potrebbe essere bloccato

## Come Risolvere

### 1. Verificare il Funzionamento del Server

Prova ad accedere al tuo sito WordPress direttamente in un browser. Se non si carica, contatta il tuo fornitore di hosting.

### 2. Controllare la Configurazione dell'URL

Assicurati di usare l'URL corretto:
- Verifica `http://` vs `https://`
- Controlla eventuali errori di battitura nel nome di dominio
- Conferma la porta corretta se si usa una porta non standard

### 3. Controllare le Impostazioni del Firewall

Se hai accesso al tuo server:
- Verifica che le porte 80 (HTTP) o 443 (HTTPS) siano aperte
- Controlla se il tuo IP è in whitelist
- Rivedi eventuali plugin di sicurezza che potrebbero bloccare l'accesso all'API

### 4. Contatta il Tuo Fornitore di Hosting

Possono verificare:
- Stato del server e log
- Configurazioni del firewall
- Se il tuo IP è stato bloccato
