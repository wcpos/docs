# DB01001: Connessione Fallita

## Cosa Significa

Il POS non è riuscito a connettersi al database locale. WCPOS utilizza un database locale per archiviare prodotti, clienti e altri dati per un accesso rapido e funzionalità offline.

## Cause Comuni

- **Spazio di archiviazione esaurito** — Il dispositivo non ha più spazio di archiviazione disponibile
- **Database danneggiato** — I file del database locale sono danneggiati
- **Problema di autorizzazione** — L'app non ha i permessi di archiviazione
- **Archiviazione browser disabilitata** — IndexedDB è disabilitato (versione web)

## Come Risolvere

### 1. Controllare lo Spazio di Archiviazione

Assicurarsi che il dispositivo abbia spazio di archiviazione disponibile:
- Liberare spazio se l'archiviazione è piena
- Eliminare app o file non utilizzati
- Verificare lo spazio del disco disponibile

### 2. Cancellare i Dati dell'App

Reimpostare il database locale:
- **App desktop:** Cercare "Cancella Dati" nelle impostazioni
- **Browser web:** Cancellare i dati del sito per il dominio POS
- **Nota:** Sarà necessario risincronizzare tutti i dati

### 3. Controllare le Impostazioni del Browser (Web)

Se si utilizza la versione web:
1. Assicurarsi che IndexedDB sia abilitato
2. Non usare la modalità privata/in incognito (archiviazione limitata)
3. Consentire al sito di archiviare dati

### 4. Controllare i Permessi

Sul desktop:
- Assicurarsi che l'app abbia il permesso di scrivere nella sua directory dati
- Verificare se l'antivirus sta bloccando l'accesso al database

### 5. Reinstallare l'App

Se altre soluzioni falliscono:
1. Disinstallare l'app POS
2. Riavviare il dispositivo
3. Reinstallare l'app
4. Accedere e sincronizzare i dati

## Recupero Dati

Se si hanno dati non salvati:
- L'app potrebbe avere modifiche in sospeso che non sono state sincronizzate
- Contattare il supporto se si necessita aiuto per recuperare i dati
- Nella maggior parte dei casi, i dati possono essere risincronizzati dal server
