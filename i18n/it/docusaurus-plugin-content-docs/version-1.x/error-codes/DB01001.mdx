# DB01001: Connessione Fallita

## Cosa Significa

Il POS non è riuscito a connettersi al proprio database locale. WCPOS utilizza un database locale per memorizzare prodotti, clienti e altri dati per un accesso rapido e funzionalità offline.

## Cause Comuni

- **Memoria piena** — Il dispositivo non ha spazio di archiviazione disponibile
- **Database corrotto** — I file del database locale sono danneggiati
- **Problema di autorizzazione** — L'app non ha autorizzazioni per l'archiviazione
- **Archiviazione del browser disabilitata** — IndexedDB è disabilitato (versione web)

## Come Risolvere

### 1. Controlla lo Spazio di Archiviazione

Assicurati che il tuo dispositivo abbia spazio di archiviazione disponibile:
- Libera spazio se la memoria è piena
- Elimina app o file non utilizzati
- Controlla lo spazio su disco disponibile

### 2. Cancella i Dati dell'App

Reimposta il database locale:
- **App desktop:** Cerca "Cancella Dati" nelle impostazioni
- **Browser web:** Cancella i dati del sito per il dominio POS
- **Nota:** Questo richiederà la re-sincronizzazione di tutti i dati

### 3. Controlla le Impostazioni del Browser (Web)

Se utilizzi la versione web:
1. Assicurati che IndexedDB sia abilitato
2. Non utilizzare la modalità privata/incognito (memoria limitata)
3. Consenti al sito di memorizzare dati

### 4. Controlla le Autorizzazioni

Su desktop:
- Assicurati che l'app abbia l'autorizzazione per scrivere nella sua directory di dati
- Controlla se l'antivirus blocca l'accesso al database

### 5. Reinstalla l'App

Se altre soluzioni non funzionano:
1. Disinstalla l'app POS
2. Riavvia il dispositivo
3. Reinstalla l'app
4. Accedi e sincronizza i dati

## Recupero Dati

Se hai dati non salvati:
- L'app potrebbe avere modifiche in sospeso che non possono essere sincronizzate
- Contatta il supporto se hai bisogno di aiuto per recuperare i dati
- Nella maggior parte dei casi, i dati possono essere re-sincronizzati dal server
