# API05001: API WooCommerce Disattivata

## Cosa Significa

L'API REST di WooCommerce è disattivata sul tuo sito. WCPOS richiede l'API REST per comunicare con WooCommerce e accedere ai dati del negozio.

## Cause Comuni

- **Impostazione WooCommerce** — L'API REST è stata disattivata intenzionalmente
- **Plugin di sicurezza** — Un plugin di sicurezza sta bloccando l'accesso all'API
- **Restrizione dell'hosting** — Il tuo host ha disabilitato l'accesso all'API REST
- **Problemi dei permalink** — I permalink non sono configurati per l'API REST

## Come Risolvere

### 1. Attivare l'API REST di WooCommerce

In Amministrazione WordPress:
1. Vai su WooCommerce → Impostazioni → Avanzate → API REST
2. Assicurati che l'API REST sia abilitata
3. Verifica che siano state create le chiavi API

### 2. Verificare l'API REST di WordPress

L'API di WooCommerce dipende dall'API REST di WordPress:
1. Visita `https://tuosito.com/wp-json/` nel browser
2. Dovrebbe restituire dati JSON
3. Se non funziona, vedi [API05004](./API05004)

### 3. Controllare i Plugin di Sicurezza

Plugin di sicurezza comuni che potrebbero bloccare l'API:
- **Wordfence** — Controlla le impostazioni del firewall
- **iThemes Security** — Controlla le impostazioni dell'API REST
- **All In One WP Security** — Rivedi le regole del firewall

Autorizza gli endpoint dell'API REST o l'applicazione POS.

### 4. Controllare .htaccess

Cerca regole che bloccano l'accesso all'API:

```apache
# Rimuovi o modifica le regole che bloccano /wp-json/
# Assicurati che mod_rewrite sia abilitato
```

### 5. Svuotare i Permalink

A volte le impostazioni dei permalink devono essere aggiornate:
1. Vai su Impostazioni → Permalink
2. Fai clic su "Salva Modifiche" (anche senza fare cambiamenti)
3. Questo rigenera le regole di riscrittura
