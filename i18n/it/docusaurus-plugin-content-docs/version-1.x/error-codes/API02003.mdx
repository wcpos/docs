# API02003: Token Non Valido

## Cosa Significa

Il token di autenticazione utilizzato non è riconosciuto dal server. Questo è diverso da un token scaduto — questo token non è mai stato valido o è stato revocato.

## Cause Comuni

- **Token revocato** — Un amministratore ha annullato la sessione
- **Token corrotto** — Corruzione dei dati durante l'archiviazione o la trasmissione
- **Ripristino del server** — Le chiavi segrete del server sono cambiate, invalidando tutti i token
- **Server errato** — Token di un'altra installazione di WordPress

## Mappatura Errore del Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Origine |
|-------------|--------|
| `jwt_auth_invalid_token` | Plugin di Autenticazione JWT |

## Come Risolvere

### 1. Accedere Nuovamente

Cancellare la sessione e autenticarsi di nuovo:
1. Disconnettersi completamente
2. Chiudere e riaprire il POS
3. Accedere con le proprie credenziali

### 2. Cancellare i Dati dell'Applicazione

Se la disconnessione non funziona:
- Cancellare la cache/dati dell'app POS
- Sul web: Cancellare i cookie e l'archiviazione locale del sito
- Sul desktop: Verificare nelle impostazioni dell'app l'opzione "Cancella Dati"

### 3. Verificare la Configurazione del Server

Se il problema riguarda tutti gli utenti:
- Controllare se i salt di WordPress sono stati modificati
- Verificare le impostazioni del plugin JWT o di autenticazione
- Rivedere recenti modifiche al server

### 4. Verificare di Essere sul Server Corretto

Assicurarsi che il POS sia configurato per connettersi al sito WordPress giusto, soprattutto se si hanno più installazioni.
