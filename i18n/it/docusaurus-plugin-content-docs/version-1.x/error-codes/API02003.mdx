# API02003: Token Invalid

## Cosa Significa {#what-this-means}

Il token di autenticazione utilizzato non è riconosciuto dal server. Questo è diverso da un token scaduto: questo token non è mai stato valido o è stato revocato.

## Cause Comuni {#common-causes}

- **Token revocato** — Un amministratore ha revocato la sessione
- **Token danneggiato** — Corruzione dei dati durante lo storage o la trasmissione
- **Reset del server** — Le chiavi segrete del server sono cambiate, invalidando tutti i token
- **Server sbagliato** — Token da un'altra installazione di WordPress

## Mappatura degli Errori del Server {#server-error-mapping}

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Fonte |
|---------------|-------|
| `jwt_auth_invalid_token` | Plugin di Autenticazione JWT |

## Come Risolvere {#how-to-fix}

### 1. Accedi di Nuovo {#1-log-in-again}

Pulisci la tua sessione e autentica di nuovo:
1. Esci completamente
2. Chiudi e riapri il POS
3. Accedi con le tue credenziali

### 2. Pulisci i Dati dell'Applicazione {#2-clear-application-data}

Se il logout non funziona:
- Pulisci la cache/dati dell'app POS
- Sul web: Pulisci i cookie del browser e lo storage locale per il sito
- Su desktop: Controlla le impostazioni dell'app per un'opzione "Pulisci Dati"

### 3. Controlla la Configurazione del Server {#3-check-server-configuration}

Se il problema riguarda tutti gli utenti:
- Controlla se i sali di WordPress sono stati cambiati
- Verifica le impostazioni del plugin JWT o di autenticazione
- Rivedi le recenti modifiche al server

### 4. Verifica di Essere sul Server Corretto {#4-verify-youre-on-the-correct-server}

Assicurati che il POS sia configurato per connettersi al sito WordPress corretto, specialmente se hai installazioni multiple.
