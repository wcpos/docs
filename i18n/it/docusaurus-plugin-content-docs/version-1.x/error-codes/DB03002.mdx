# DB03002: Tipo di Dati Non Valido

## Cosa Significa

Il tipo di dati non corrisponde a quello atteso dal database. Ad esempio, è stato fornito testo dove era previsto un numero.

## Cause Comuni

- **Errore di input utente** — Inserimento di testo in un campo numerico
- **Problemi di importazione** — Dati importati con formato errato
- **Corruzione dei dati** — Valori corrotti durante il trasferimento
- **Conflitto di plugin** — Un altro plugin ha modificato i tipi di dati

## Come Risolvere

### 1. Controlla il Tuo Input

Rivedi i dati che stai inserendo:
- **Prezzi** — Devono essere numeri (es. `19.99`)
- **Quantità** — Devono essere numeri interi (es. `5`)
- **ID** — Devono essere numerici

### 2. Cancella e Reinserisci

Se i dati sono stati corrotti:
- Cancella il campo
- Inserisci manualmente il valore
- Evita copia e incolla da fonti esterne

### 3. Controlla i Dati Sorgente

Se si sincronizza da WooCommerce:
- Controlla i dati nell'Amministrazione WordPress
- Cerca campi formattati in modo errato
- Correggi i dati alla fonte

### 4. Risincronizza i Dati

Ottieni dati freschi dal server:
- Cancella la cache locale
- Risincronizza i dati
- Verifica se il problema si risolve

### 5. Controlla Conflitti di Plugin

Se usi altri plugin WooCommerce:
- Potrebbero modificare i dati in modi inaspettati
- Disabilita temporaneamente per testare
- Segnala incompatibilità

## Esempi Comuni

| Campo | Atteso | Non Valido |
|-------|----------|------------|
| Prezzo | `19.99` | `$19.99` |
| Quantità | `5` | `cinque` |
| SKU | `ABC123` | — (formato qualsiasi OK) |
| ID Prodotto | `42` | `prodotto-42` |
