# DB03002: Tipo di Dato Non Valido

## Cosa Significa

Il tipo di dato non corrisponde a quello che si aspetta il database. Ad esempio, è stato fornito un testo quando era atteso un numero.

## Cause Comuni

- **Errore di input dell'utente** — Inserimento di testo in un campo numerico
- **Problemi di importazione** — I dati importati hanno un formato errato
- **Corruzione dei dati** — Valori corrotti durante il trasferimento
- **Conflitto di plugin** — Un altro plugin ha modificato i tipi di dato

## Come Risolvere

### 1. Controlla il Tuo Input

Esamina i dati che stai inserendo:
- **Prezzi** — Dovrebbero essere numeri (es., `19.99`)
- **Quantità** — Dovrebbero essere numeri interi (es., `5`)
- **ID** — Dovrebbero essere numerici

### 2. Cancella e Reinserisci

Se i dati sono stati corrotti:
- Cancella il campo
- Inserisci nuovamente il valore manualmente
- Evita di copiare e incollare da fonti esterne

### 3. Controlla i Dati Sorgente

Se stai sincronizzando da WooCommerce:
- Controlla i dati nell'Amministrazione di WordPress
- Cerca campi formattati in modo errato
- Correggi i dati alla fonte

### 4. Riesegui la Sincronizzazione dei Dati

Ottieni dati freschi dal server:
- Cancella la cache locale
- Sincronizza nuovamente i dati
- Controlla se il problema si risolve

### 5. Controlla i Conflitti di Plugin

Se stai usando altri plugin di WooCommerce:
- Potrebbero modificare i dati in modi inaspettati
- Disabilita temporaneamente per testare
- Riporta le incompatibilità

## Esempi Comuni

| Campo | Atteso | Non Valido |
|-------|--------|------------|
| Prezzo | `19.99` | `$19.99` |
| Quantità | `5` | `cinque` |
| SKU | `ABC123` | — (qualsiasi formato OK) |
| ID Prodotto | `42` | `prodotto-42` |
