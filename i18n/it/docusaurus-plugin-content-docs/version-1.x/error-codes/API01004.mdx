# API01004: Risoluzione DNS non riuscita {#api01004-dns-resolution-failed}

## Cosa Significa {#what-this-means}

Il POS non è riuscito a convertire il nome di dominio del tuo sito web (ad es., `tuoNegozio.com`) in un indirizzo IP. Il DNS (Domain Name System) è come la rubrica telefonica di internet: se non trova l'elenco, non può stabilire la connessione.

## Cause Comuni {#common-causes}

- **Errore di battitura nel nome di dominio** — L'URL potrebbe contenere un errore di ortografia
- **Problemi del server DNS** — Il tuo server DNS non sta rispondendo
- **Dominio scaduto** — La registrazione del dominio potrebbe essere scaduta
- **Propagazione DNS** — Le recenti modifiche DNS non sono ancora propagate
- **Cache DNS locale** — Informazioni DNS obsolete memorizzate sul tuo dispositivo

## Come Risolvere {#how-to-fix}

### 1. Verifica il Nome di Dominio {#1-verify-the-domain-name}

Controlla di nuovo l'URL per eventuali errori di battitura. Prova ad accedere al sito in un browser.

### 2. Controlla lo Stato del Dominio {#2-check-domain-status}

Usa una consultazione WHOIS per verificare:
- Il dominio è ancora registrato
- Non è scaduto
- I record DNS sono configurati

### 3. Pulisci la Cache DNS {#3-clear-dns-cache}

**Sul tuo dispositivo:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Prova DNS Alternativi {#4-try-alternative-dns}

Passa a un server DNS pubblico come:
- Google: `8.8.8.8` e `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Aspetta la Propagazione DNS {#5-wait-for-dns-propagation}

Se hai recentemente modificato le impostazioni DNS, aspetta 24-48 ore per la completa propagazione.
