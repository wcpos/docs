# API01004: Risoluzione DNS Fallita

## Cosa Significa

Il POS non è riuscito a convertire il nome di dominio del tuo sito web (ad esempio, `yourstore.com`) in un indirizzo IP. Il DNS (Domain Name System) è come la rubrica telefonica di Internet — se non riesce a trovare la voce, non può stabilire la connessione.

## Cause Comuni

- **Errore di battitura nel nome di dominio** — L'URL potrebbe avere un errore di ortografia
- **Problemi del server DNS** — Il tuo server DNS non risponde
- **Dominio scaduto** — La registrazione del dominio potrebbe essere scaduta
- **Propagazione DNS** — Le modifiche recenti al DNS non sono ancora state propagate
- **Cache DNS locale** — Informazioni DNS obsolete memorizzate sul dispositivo

## Come Risolvere

### 1. Verificare il Nome di Dominio

Controllare attentamente l'URL per eventuali errori di battitura. Provare ad accedere al sito in un browser.

### 2. Controllare lo Stato del Dominio

Utilizzare una ricerca WHOIS per verificare:
- Il dominio è ancora registrato
- Non è scaduto
- I record DNS sono configurati

### 3. Cancellare la Cache DNS

**Sul tuo dispositivo:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Provare un DNS Alternativo

Cambiare a un server DNS pubblico come:
- Google: `8.8.8.8` e `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Attendere la Propagazione DNS

Se di recente hai modificato le impostazioni DNS, attendi 24-48 ore per la piena propagazione.
