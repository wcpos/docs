# DB01002: Timeout Query

## Cosa Significa

Un'operazione di database ha richiesto troppo tempo per essere completata. La query del database locale ha superato il tempo massimo consentito.

## Cause Comuni

- **Set di dati ampio** — Numero molto elevato di prodotti o record
- **Query complessa** — Query che coinvolge molte condizioni
- **Prestazioni del dispositivo** — Dispositivo funziona lentamente
- **Frammentazione database** — Database necessita di ottimizzazione

## Come Risolvere

### 1. Attendere e Riprovare

La query potrebbe riuscire nel nuovo tentativo:
- Chiudere e riaprire il POS
- Provare nuovamente l'operazione
- Evitare operazioni intensive contemporanee

### 2. Chiudere Altre Applicazioni

Liberare risorse del dispositivo:
- Chiudere schede del browser inutilizzate
- Chiudere altre applicazioni
- Riavviare il dispositivo se necessario

### 3. Ridurre l'Ambito dei Dati

Durante la ricerca:
- Usare termini di ricerca più specifici
- Applicare filtri per restringere i risultati
- Suddividere operazioni grandi in più piccole

### 4. Ottimizzare il Database

Cancellare e risincronizzare:
1. Cancellare dati locali (opzione nelle impostazioni se disponibile)
2. Effettuare nuovamente l'accesso
3. Permettere una nuova sincronizzazione dei dati

### 5. Verificare le Prestazioni del Dispositivo

Assicurarsi che il dispositivo soddisfi i requisiti:
- RAM sufficiente
- Velocità del processore adeguata
- Preferibile SSD rispetto a HDD

## Cataloghi Estesi

In caso di catalogo prodotti molto ampio:
- La sincronizzazione iniziale richiede più tempo
- Considerare l'uso di filtri nel POS
- Alcune operazioni richiedono naturalmente più tempo
- Contattare l'assistenza per consigli di ottimizzazione
