# DB01002: Timeout della Query

## Cosa Significa

Un'operazione del database ha impiegato troppo tempo per essere completata. La query del database locale ha superato il tempo massimo consentito.

## Cause Comuni

- **Set di dati molto grande** — Numero molto elevato di prodotti o record
- **Query complessa** — Query che coinvolge molte condizioni
- **Prestazioni del dispositivo** — Il dispositivo è lento
- **Frammentazione del database** — Il database necessita di ottimizzazione

## Come Risolvere

### 1. Aspetta e Ripeti

La query può avere successo al secondo tentativo:
- Chiudi e riapri il POS
- Riprova l'operazione
- Evita altre operazioni intensive simultaneamente

### 2. Chiudi Altre Applicazioni

Libera le risorse del dispositivo:
- Chiudi le schede del browser non utilizzate
- Chiudi altre applicazioni
- Riavvia il dispositivo se necessario

### 3. Riduci l'Ambito dei Dati

Se stai cercando:
- Usa termini di ricerca più specifici
- Applica filtri per restringere i risultati
- Suddividi le operazioni grandi in operazioni più piccole

### 4. Ottimizza il Database

Pulire e risincronizzare:
1. Cancella i dati locali (opzione di impostazione se disponibile)
2. Accedi nuovamente
3. Lascia che i dati si sincronizzino freschi

### 5. Controlla le Prestazioni del Dispositivo

Assicurati che il tuo dispositivo soddisfi i requisiti:
- RAM sufficiente
- Velocità del processore adeguata
- SSD preferito rispetto a HDD

## Cataloghi Grandi

Se hai un catalogo prodotti molto grande:
- La sincronizzazione iniziale richiede più tempo
- Considera di usare filtri nel POS
- Alcune operazioni richiedono naturalmente più tempo
- Contatta il supporto per consigli sull'ottimizzazione
