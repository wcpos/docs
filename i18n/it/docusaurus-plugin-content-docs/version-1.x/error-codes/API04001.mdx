# API04001: Formato Risposta Non Valido

## Cosa Significa

Il server ha risposto, ma il formato della risposta non è quello che il POS si aspettava. Il server dovrebbe restituire dati JSON, ma è stato ricevuto qualcos'altro.

## Cause Comuni

- **Errore PHP visualizzato** — Un errore PHP viene stampato prima del JSON
- **Conflitto plugin** — Un altro plugin sta generando contenuto
- **Modalità manutenzione** — Il sito mostra una pagina di manutenzione
- **Tipo di contenuto errato** — Il server invia HTML invece di JSON
- **Problema di cache** — Viene servita una pagina di errore cached

## Come Risolvere

### 1. Controlla il Tuo Sito

Visita il tuo sito WordPress in un browser:
- Viene visualizzato normalmente?
- Sono presenti errori visibili?
- È in modalità manutenzione?

### 2. Verifica Errori PHP

In `wp-config.php`, abilita temporaneamente il debug:

```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```

Controlla `wp-content/debug.log` per gli errori.

### 3. Testa Direttamente l'API REST

Visita `https://yoursite.com/wp-json/` nel browser:
- Dovrebbe restituire dati JSON
- Se vedi HTML o errori, c'è un problema
- Controlla l'output correlato ai plugin

### 4. Disabilita Temporaneamente la Cache

I plugin di caching potrebbero servire risposte obsolete:
- Cancella tutte le cache
- Disabilita temporaneamente i plugin di caching
- Escludi l'API REST dalla cache

### 5. Verifica Conflitti Plugin

Se un plugin genera contenuto su ogni pagina:
1. Disabilita tutti i plugin non essenziali
2. Testa il POS
3. Riabilita i plugin uno per volta
