# API02004: Utente Non Autorizzato

## Cosa Significa

Hai effettuato l'accesso, ma il tuo account utente non ha i permessi per eseguire l'azione richiesta. Questo è un problema di autorizzazione (cosa puoi fare) piuttosto che di autenticazione (chi sei).

## Cause Comuni

- **Limiti del ruolo utente** — Il tuo ruolo WordPress non include l'accesso al POS
- **Accesso POS disabilitato** — Al tuo account non sono stati concessi i permessi POS
- **Restrizioni delle funzionalità** — Alcune funzioni sono limitate a ruoli specifici
- **Restrizioni del negozio** — Potresti non avere accesso a questo specifico negozio

## Mappatura Errori Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Origine |
|-------------|--------|
| `rest_cannot_view` | WordPress REST API |
| `woocommerce_rest_cannot_view` | WooCommerce REST API |

## Come Risolvere

### 1. Verifica Ruolo Utente

In WordPress Admin → Utenti:
1. Trova il tuo account utente
2. Verifica il ruolo (es. Gestore Negozio, Amministratore)
3. Assicurati che il ruolo includa le capacità WooCommerce

### 2. Abilita Accesso POS

In WordPress Admin → WooCommerce → POS → Accesso:
1. Trova l'utente o il ruolo
2. Abilita i permessi di accesso POS
3. Salva le modifiche

### 3. Richiedi Permessi Aggiuntivi

Contatta l'amministratore del negozio per:
- Concedere l'accesso POS al tuo ruolo
- Assegnarti un ruolo con autorizzazioni appropriate
- Abilitare funzionalità specifiche di cui hai bisogno

### 4. Verifica Assegnazione Negozio

Se utilizzi più negozi:
- Verifica di essere assegnato al negozio corretto
- Controlla i permessi specifici del negozio

## Permessi Richiesti

Azioni diverse richiedono capacità diverse:
- **Visualizzare prodotti**: Accesso in lettura ai prodotti
- **Creare ordini**: Capacità di creare/modificare ordini
- **Gestire clienti**: Capacità di gestione clienti
- **Accedere a report**: Capacità di visualizzare report
