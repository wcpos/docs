# API01003: Ripristino Connessione

## Cosa Significa

La connessione al server è stata stabilita ma poi chiusa inaspettatamente. Questo accade tipicamente a metà comunicazione quando qualcosa interrompe la connessione.

## Cause Comuni

- **Crash del server** — Il processo del server potrebbe essere crashato durante la richiesta
- **Limiti di memoria** — PHP ha esaurito la memoria ed è stato terminato
- **Timeout durante la richiesta** — Il server ha raggiunto il timeout durante l'elaborazione
- **Instabilità di rete** — Problemi intermittenti di rete
- **Software di sicurezza** — Impostazioni di sicurezza aggressive che chiudono le connessioni

## Come Risolvere

### 1. Controllare i Log degli Errori del Server

Cerca nei log degli errori di WordPress o nel pannello di controllo dell'hosting eventuali errori correlati intorno al momento del ripristino.

### 2. Aumentare il Limite di Memoria PHP

Nel file `wp-config.php`:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. Verificare Conflitti tra Plugin

Un plugin potrebbe causare il crash di PHP:
- Disabilita temporaneamente i plugin
- Riattivali uno per uno per trovare il responsabile

### 4. Verificare le Risorse del Server

Contatta il tuo fornitore di hosting per controllare:
- Limiti dei processi PHP
- Utilizzo della memoria al momento dell'errore
- Log di stabilità del server

### 5. Testare la Stabilità della Rete

Se sei su WiFi, prova una connessione cablata. Se il problema persiste su reti diverse, è probabile che sia lato server.
