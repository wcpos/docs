# API05005: Plugin non trovato

## Cosa significa

Un plugin WordPress richiesto non è installato o attivo. Potrebbe trattarsi di WooCommerce stesso, di un'estensione richiesta o di un plugin di integrazione.

## Cause comuni

- **WooCommerce disattivato** — WooCommerce è stato disabilitato
- **Estensione richiesta mancante** — Un'estensione WooCommerce necessaria non è installata
- **Conflitto plugin** — Un altro plugin ha causato la disattivazione di un plugin richiesto
- **Aggiornamento fallito** — Un aggiornamento del plugin non è riuscito e lo ha lasciato disattivato

## Come risolvere

### 1. Verificare che WooCommerce sia attivo

WCPOS richiede WooCommerce:
1. Vai a Plugin → Plugin installati
2. Trova "WooCommerce"
3. Assicurati che sia attivato

### 2. Verificare le estensioni richieste

Alcune funzionalità di WCPOS potrebbero richiedere estensioni:
- Controlla il messaggio di errore per il nome specifico del plugin
- Installa e attiva l'estensione richiesta

### 3. Verificare i conflitti dei plugin

Se i plugin sono stati disattivati a causa di conflitti:
1. Controlla le notifie e-mail di WordPress
2. Rivedi i registri degli errori
3. Risolvi i conflitti e riattiva

### 4. Controllare i file dei plugin

Se i file di un plugin sono mancanti:
1. Reinstalla il plugin da WordPress.org
2. O ripristina da un backup
3. Attiva il plugin

### 5. Aggiornare tutti i plugin

Plugin obsoleti possono causare problemi:
1. Vai a Dashboard → Aggiornamenti
2. Aggiorna tutti i plugin
3. Verifica che tutti i plugin richiesti siano attivi

## Plugin richiesti

Come minimo, WCPOS richiede:
- **WooCommerce** — La piattaforma e-commerce
- **WCPOS** — Il plugin POS

Funzionalità aggiuntive possono richiedere altri plugin.
