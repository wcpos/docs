# API05005: Plugin Non Trovato

## Cosa Significa

Un plugin WordPress richiesto non è installato o attivo. Questo potrebbe essere WooCommerce stesso, un'estensione necessaria o un plugin di integrazione.

## Cause Comuni

- **WooCommerce disattivato** — WooCommerce è stato disabilitato
- **Estensione richiesta mancante** — Un'estensione WooCommerce necessaria non è installata
- **Conflitto tra plugin** — Un altro plugin ha causato la disattivazione di un plugin richiesto
- **Aggiornamento non riuscito** — Un aggiornamento di un plugin non è riuscito e lo ha lasciato disattivato

## Come Risolvere

### 1. Verifica che WooCommerce sia Attivo

WCPOS richiede WooCommerce:
1. Vai su Plugin → Plugin Installati
2. Trova "WooCommerce"
3. Assicurati che sia attivato

### 2. Controlla le Estensioni Richieste

Alcune funzionalità di WCPOS potrebbero richiedere delle estensioni:
- Controlla il messaggio di errore per il nome specifico del plugin
- Installa e attiva l'estensione richiesta

### 3. Rivedi i Conflitti tra Plugin

Se i plugin sono stati disattivati a causa di conflitti:
1. Controlla la tua email per notifiche di WordPress
2. Rivedi i log degli errori
3. Risolvi i conflitti e riattiva

### 4. Controlla i File del Plugin

Se i file di un plugin sono mancanti:
1. Reinstalla il plugin da WordPress.org
2. Oppure ripristina dal backup
3. Attiva il plugin

### 5. Aggiorna Tutti i Plugin

I plugin obsoleti possono causare problemi:
1. Vai su Dashboard → Aggiornamenti
2. Aggiorna tutti i plugin
3. Verifica che tutti i plugin richiesti siano attivi

## Plugin Richiesti

Al minimo, WCPOS richiede:
- **WooCommerce** — La piattaforma di e-commerce
- **WCPOS** — Il plugin POS

Funzionalità aggiuntive potrebbero richiedere altri plugin.
