# API03001: Formato di Richiesta Non Valido

## Cosa Significa

La richiesta inviata al server non era nel formato previsto. Il server non è riuscito a capire cosa stesse chiedendo il POS perché la struttura della richiesta era errata.

## Cause Comuni

- **Dati corrotti** — I dati sono stati corrotti prima dell'invio
- **Bug del software** — Un problema nell'applicazione POS
- **Interferenza del proxy** — Un proxy o un firewall ha modificato la richiesta
- **Problemi di codifica dei caratteri** — Caratteri speciali non codificati correttamente

## Mappatura degli Errori del Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Origine |
|-------------|--------|
| HTTP 400 | Qualsiasi risposta del server (fallback) |

## Come Risolvere

### 1. Ripetere l'Azione

A volte è solo un problema temporaneo:
- Attendere un momento e riprovare
- Aggiornare il POS e ripetere

### 2. Verificare i Caratteri Speciali

Se si stanno inserendo dati con caratteri speciali:
- Provare a rimuovere emoji o simboli inusuali
- Utilizzare caratteri standard per i nomi dei prodotti, ecc.

### 3. Aggiornare il POS

Assicurarsi di utilizzare l'ultima versione:
- Verificare gli aggiornamenti dell'app
- Aggiornare il plugin WCPOS sul proprio server

### 4. Controllare la Configurazione di Rete

Se si è dietro a un proxy:
- Verificare che il proxy non modifichi le richieste
- Controllare le regole del firewall
- Provare ad accedere da una rete diversa

### 5. Segnalare il Problema

Se si verifica costantemente:
- Annotare quale azione attiva l'errore
- Controllare la console del browser per i dettagli (versione web)
- Segnalare su [GitHub](https://github.com/wcpos) con i passaggi per riprodurre l'errore
