# API01005: Errore del Certificato SSL

## Cosa Significa

C'è un problema con il certificato SSL/TLS del tuo sito web. Questo certificato è ciò che rende la connessione sicura (HTTPS). Il POS non si connetterà a siti con certificati non validi per proteggere i tuoi dati.

## Cause Comuni

- **Certificato scaduto** — I certificati SSL devono essere rinnovati periodicamente
- **Certificato autofirmato** — Non emesso da un'autorità attendibile
- **Dominio errato** — Il certificato non corrisponde al nome di dominio
- **Catena di certificati incompleta** — Certificati intermedi mancanti
- **Contenuto misto** — Alcune risorse caricate su HTTP invece di HTTPS

## Come Risolvere

### 1. Verifica lo Stato del Certificato

Visita il tuo sito in un browser e fai clic sull'icona del lucchetto per visualizzare i dettagli del certificato. Controlla:
- Data di scadenza
- Emesso a (deve corrispondere al tuo dominio)
- Emesso da (deve essere un'autorità riconosciuta)

### 2. Rinnova Certificato Scaduto

Se scaduto:
- La maggior parte dei provider di hosting offre certificati Let's Encrypt gratuiti
- Contatta il tuo provider di hosting per il rinnovo
- Verifica se il rinnovo automatico è abilitato

### 3. Correggi Discrepanza Certificato

Assicurati che il certificato copra:
- Il tuo dominio esatto (`negozio.com`)
- Variante WWW (`www.negozio.com`) se utilizzata
- Considera un certificato wildcard (`*.negozio.com`)

### 4. Installa Certificati Intermedi Mancanti

Usa uno strumento di verifica SSL (come SSL Labs) per identificare i certificati mancanti. Il tuo provider di hosting può aiutarti a installarli.

### 5. Forza HTTPS

In WordPress, assicurati:
- L'URL del sito utilizzi `https://`
- SSL forzato sia abilitato nelle impostazioni di WooCommerce
