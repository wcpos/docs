# API03002: Parametri Richiesti Mancanti

## Cosa Significa

La richiesta manca di dati che il server necessita per completare l'azione. I campi richiesti non sono stati inclusi nella richiesta.

## Cause Comuni

- **Modulo incompleto** — I campi richiesti non sono stati compilati
- **Dati non salvati** — I dati del modulo non sono stati acquisiti correttamente
- **Disallineamento delle versioni** — Il plugin si aspetta parametri diversi da quelli inviati dal POS
- **Campi personalizzati** — Mancano campi personalizzati richiesti

## Come Risolvere

### 1. Completare i Campi Richiesti

Verifica che tutte le informazioni richieste siano compilate:
- Dati del cliente (se richiesti)
- Informazioni sul prodotto
- Dettagli dell'ordine

### 2. Aggiorna e Ripeti

Lo stato del modulo potrebbe essere incompleto:
1. Ricarica il POS
2. Reinserisci le informazioni richieste
3. Prova di nuovo l'azione

### 3. Controllare le Versioni del Plugin

Assicurati della compatibilità:
- Aggiorna il plugin WCPOS
- Aggiorna l'applicazione POS
- Entrambi dovrebbero essere su versioni compatibili

### 4. Controllare le Impostazioni dei Campi Richiesti

In WooCommerce, alcuni campi potrebbero essere contrassegnati come richiesti:
- Rivedi i requisiti dei campi di checkout
- Controlla le configurazioni dei campi personalizzati
- Modifica i requisiti se necessario

### 5. Controllare Integrazioni Personalizzate

Se si utilizzano plugin o integrazioni personalizzate:
- Potrebbero richiedere campi aggiuntivi
- Rivedi la documentazione del plugin
- Controlla i conflitti
