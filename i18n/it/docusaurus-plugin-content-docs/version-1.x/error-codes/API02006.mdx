# API02006: Chiave API non valida {#api02006-api-key-invalid}

## Cosa significa {#what-this-means}

La chiave API REST di WooCommerce in uso non è valida. Le chiavi API sono utilizzate per l'autenticazione server-to-server e la chiave fornita non corrisponde a nessuna chiave valida in WooCommerce.

## Cause comuni {#common-causes}

- **Chiave eliminata** — La chiave API è stata rimossa da WooCommerce
- **Errore di battitura nella chiave** — La chiave è stata inserita in modo errato
- **Coppia di chiavi errata** — La chiave del consumatore e il segreto non corrispondono
- **Chiave da un sito diverso** — Utilizzo di chiavi generate per un'altra installazione

## Come risolvere {#how-to-fix}

### 1. Verifica la chiave API {#1-verify-the-api-key}

In WordPress Admin → WooCommerce → Impostazioni → Avanzate → REST API:
1. Controlla se la tua chiave API esiste
2. Verifica che non sia stata revocata
3. Nota le autorizzazioni (lettura/scrittura/lettura-scrittura)

### 2. Genera nuove chiavi API {#2-generate-new-api-keys}

Se la chiave è mancante o non valida:
1. Vai su WooCommerce → Impostazioni → Avanzate → REST API
2. Clicca su "Aggiungi chiave"
3. Inserisci una descrizione (ad es., "WCPOS")
4. Seleziona l'utente
5. Scegli le autorizzazioni "Lettura/Scrittura"
6. Clicca su "Genera chiave API"
7. **Copia sia la Chiave del Consumatore che il Segreto del Consumatore** (mostrati solo una volta!)

### 3. Aggiorna la configurazione del POS {#3-update-pos-configuration}

Inserisci le nuove chiavi API nel POS:
- Chiave del Consumatore (inizia con `ck_`)
- Segreto del Consumatore (inizia con `cs_`)

### 4. Controlla le autorizzazioni della chiave {#4-check-key-permissions}

Assicurati che la chiave abbia autorizzazioni sufficienti:
- **Lettura** — Visualizza solo i dati
- **Scrittura** — Modifica solo i dati
- **Lettura/Scrittura** — Accesso completo (consigliato per il POS)
