# API02006: Chiave API Non Valida

## Cosa Significa

La chiave dell'API REST di WooCommerce utilizzata non è valida. Le chiavi API vengono utilizzate per l'autenticazione tra server, e la chiave fornita non corrisponde ad alcuna chiave valida in WooCommerce.

## Cause Comuni

- **Chiave eliminata** — La chiave API è stata rimossa da WooCommerce
- **Errore di battitura** — La chiave è stata inserita in modo errato
- **Coppia di chiavi errata** — La chiave consumer e il segreto non corrispondono
- **Chiave di un sito diverso** — Utilizzo di chiavi generate per un'altra installazione

## Come Risolvere

### 1. Verificare la Chiave API

In Amministrazione WordPress → WooCommerce → Impostazioni → Avanzate → REST API:
1. Controllare se la chiave API esiste
2. Verificare che non sia stata revocata
3. Notare i permessi (lettura/scrittura/lettura-scrittura)

### 2. Generare Nuove Chiavi API

Se la chiave è mancante o non valida:
1. Andare su WooCommerce → Impostazioni → Avanzate → REST API
2. Fare clic su "Aggiungi chiave"
3. Inserire una descrizione (es. "WCPOS")
4. Selezionare l'utente
5. Scegliere i permessi "Lettura/Scrittura"
6. Fare clic su "Genera chiave API"
7. **Copiare sia la Chiave Consumer che il Segreto Consumer** (mostrati solo una volta!)

### 3. Aggiornare la Configurazione POS

Inserire le nuove chiavi API nel POS:
- Chiave Consumer (inizia con `ck_`)
- Segreto Consumer (inizia con `cs_`)

### 4. Controllare i Permessi della Chiave

Assicurarsi che la chiave abbia permessi sufficienti:
- **Lettura** — Solo visualizzazione dati
- **Scrittura** — Solo modifica dati
- **Lettura/Scrittura** — Accesso completo (consigliato per POS)
