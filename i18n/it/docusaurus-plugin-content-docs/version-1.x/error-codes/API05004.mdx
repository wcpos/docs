# API05004: API WordPress Disattivata

## Cosa Significa

L'API REST di WordPress è disattivata sul tuo sito. Tutte le funzionalità moderne di WordPress, inclusi WooCommerce e WCPOS, dipendono da questa API.

## Cause Comuni

- **Plugin di sicurezza** — Un plugin sta bloccando l'accesso all'API REST
- **Restrizione hosting** — Il tuo host ha disattivato l'API REST
- **Codice personalizzato** — Un tema o un plugin ha disattivato l'API
- **Regole firewall** — WAF sta bloccando gli endpoint dell'API REST

## Come Risolvere

### 1. Verifica l'API REST

Visita `https://yoursite.com/wp-json/` nel tuo browser:
- Dovrebbe restituire JSON con le rotte disponibili
- Se ottieni un errore o nulla, è bloccato

### 2. Controlla i Plugin di Sicurezza

Plugin comuni che bloccano l'API REST:

**Wordfence:**
- Firewall → Tutte le Opzioni Firewall
- Disabilita l'opzione "Disabilita REST API"

**iThemes Security:**
- Sicurezza → Impostazioni → Modifiche WordPress
- Abilita l'API REST

**Plugin Disabilita REST API:**
- Disattiva completamente questo plugin

### 3. Controlla il Codice Personalizzato

Cerca nel `functions.php` del tuo tema o nei plugin personalizzati:

```php
// Questo codice disabilita l'API REST - rimuoverlo
add_filter('rest_authentication_errors', function($result) {
    return new WP_Error('rest_disabled', 'REST API disabled');
});
```

### 4. Controlla .htaccess

Rimuovi eventuali regole che bloccano `/wp-json/`:

```apache
# Non valido - blocca l'API REST
RewriteRule ^wp-json - [F,L]
```

### 5. Contatta il Provider di Hosting

Alcuni host bloccano l'API REST per impostazione predefinita:
- Richiedi che venga abilitata
- Chiedi informazioni su eventuali restrizioni di sicurezza
- Controlla la documentazione dell'hosting

## Perché l'API REST è Importante

L'API REST di WordPress è essenziale per:
- App mobile
- Integrazioni di terze parti
- Funzioni di WooCommerce
- Funzionamento di WCPOS

Disabilitarla interrompe molte funzionalità.
