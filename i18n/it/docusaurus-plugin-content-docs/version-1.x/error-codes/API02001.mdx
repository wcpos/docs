# API02001: Credenziali non valide

## Cosa Significa

Il nome utente o la password inseriti non sono corretti. Il server ha rifiutato il tentativo di accesso perché le credenziali non corrispondono ad alcun account valido.

## Cause Comuni

- **Errore di battitura nel nome utente o password** — Controllare il blocco maiuscole o spazi extra
- **Account errato** — Utilizzo di credenziali per un sito diverso
- **Password modificata di recente** — La password è stata aggiornata ma il POS ha ancora la vecchia
- **Account eliminato** — L'account utente non esiste più

## Mapping degli Errori del Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Origine |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | Plugin di Autenticazione JWT |

## Come Risolvere

### 1. Verificare le Proprie Credenziali

- Controllare attentamente il nome utente (solitamente email o username WordPress)
- Verificare che la password sia corretta
- Provare ad accedere direttamente all'admin di WordPress per confermare

### 2. Reimpostare la Password

Se hai dimenticato la password:
1. Andare alla pagina di accesso di WordPress
2. Fare clic su "Password dimenticata?"
3. Inserire l'email per ricevere un link di reimpostazione
4. Aggiornare la password
5. Accedere al POS con la nuova password

### 3. Controllare l'Account Utente

Nell'Admin di WordPress:
- Andare su Utenti → Tutti gli Utenti
- Verificare che l'account esista
- Controllare che abbia i permessi di accesso al POS appropriati

### 4. Cancellare le Credenziali Salvate

Nel POS:
- Disconnettersi completamente
- Cancellare tutte le informazioni di accesso salvate/memorizzate
- Reinserire le credenziali da zero
