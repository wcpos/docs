# API02007: Rinnovo Token Fallito

## Cosa Significa

Il POS ha cercato di rinnovare automaticamente la tua sessione di autenticazione, ma la richiesta di rinnovo non è riuscita. Questo significa tipicamente che dovrai effettuare nuovamente l'accesso.

## Cause Comuni

- **Token di rinnovo scaduto** — Il token di rinnovo stesso è scaduto
- **Token di rinnovo revocato** — Un amministratore ha invalidato tutte le sessioni
- **Errore del server** — Il server non è riuscito a elaborare la richiesta di rinnovo
- **Problema di rete** — Problema di connessione durante il tentativo di rinnovo

## Come Risolvere

### 1. Accedi Nuovamente

La soluzione più affidabile:
1. Esci dal POS
2. Accedi nuovamente con le tue credenziali
3. Questo creerà una nuova sessione con nuovi token

### 2. Controlla la Connessione di Rete

Se il rinnovo non riesce a causa di problemi di rete:
- Verifica la tua connessione internet
- Prova ad accedere al tuo sito WordPress in un browser
- Attendi un momento e riprova

### 3. Controlla lo Stato del Server

Se il server sta avendo problemi:
- Verifica che WordPress sia in esecuzione correttamente
- Controlla gli errori PHP nei registri del server
- Assicurati che il plugin di autenticazione sia attivo

### 4. Cancella i Dati della Sessione

Se i problemi persistono:
- Cancella la cache dell'app POS
- Rimuovi le credenziali archiviate
- Inizia di nuovo con un nuovo accesso

## Rinnovo Automatico del Token

WCPOS rinnova automaticamente i token prima che scadano per mantenere la tua sessione. Se questo processo fallisce ripetutamente, potrebbe esserci un problema di configurazione che richiede attenzione.
