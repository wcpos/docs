# API02007: Aggiornamento del Token Fallito {#api02007-token-refresh-failed}

## Cosa Significa {#what-this-means}

Il POS ha tentato di rinnovare automaticamente la tua sessione di autenticazione, ma la richiesta di aggiornamento è fallita. Questo significa tipicamente che dovrai accedere di nuovo.

## Cause Comuni {#common-causes}

- **Token di aggiornamento scaduto** — Il token di aggiornamento stesso è scaduto
- **Token di aggiornamento revocato** — Un amministratore ha invalidato tutte le sessioni
- **Errore del server** — Il server non è riuscito a elaborare la richiesta di aggiornamento
- **Problemi di rete** — Problema di connessione durante il tentativo di aggiornamento

## Come Risolvere {#how-to-fix}

### 1. Accedi di Nuovo {#1-log-in-again}

La soluzione più affidabile:
1. Disconnettiti dal POS
2. Accedi di nuovo con le tue credenziali
3. Questo crea una nuova sessione con nuovi token

### 2. Controlla la Connessione di Rete {#2-check-network-connection}

Se l'aggiornamento fallisce a causa di problemi di rete:
- Verifica la tua connessione a internet
- Prova ad accedere al tuo sito WordPress in un browser
- Aspetta un momento e riprova

### 3. Controlla lo Stato del Server {#3-check-server-status}

Se il server ha problemi:
- Verifica che WordPress funzioni correttamente
- Controlla eventuali errori PHP nei log del server
- Assicurati che il plugin di autenticazione sia attivo

### 4. Cancella i Dati della Sessione {#4-clear-session-data}

Se i problemi persistono:
- Cancella la cache dell'app POS
- Rimuovi le credenziali memorizzate
- Inizia di nuovo con un nuovo accesso

## Aggiornamento Automatico del Token {#automatic-token-refresh}

WCPOS aggiorna automaticamente i token prima che scadano per mantenere la tua sessione. Se questo processo fallisce ripetutamente, potrebbe esserci un problema di configurazione che necessita attenzione.
