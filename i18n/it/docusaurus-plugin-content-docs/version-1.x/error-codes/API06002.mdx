# API06002: URL API mancante

## Cosa significa

Nessun URL è stato configurato per connettersi al tuo negozio WooCommerce. Il POS deve sapere dove si trova il tuo negozio.

## Cause comuni

- **Configurazione iniziale** — URL non è mai stato inserito
- **Configurazione cancellata** — Impostazioni ripristinate o cancellate
- **Dati dell'app cancellati** — Dati dell'applicazione eliminati
- **Problema di migrazione** — URL perso durante un aggiornamento o una migrazione

## Come risolvere

### 1. Inserisci l'URL del tuo negozio

Nell'app POS:
1. Apri le impostazioni o la schermata di accesso
2. Trova il campo di configurazione URL/Sito
3. Inserisci l'URL del tuo sito WordPress
4. Esempio: `https://tuonegozio.com`

### 2. Trova l'URL di WordPress

Se non sei sicuro dell'URL:
1. Accedi all'Amministrazione di WordPress
2. Vai su Impostazioni → Generali
3. Cerca "Indirizzo WordPress (URL)"
4. Usa questo URL nel POS

### 3. Controlla le impostazioni memorizzate

Se l'URL era stato precedentemente configurato:
- Controlla se i dati dell'app sono stati accidentalmente cancellati
- Cerca opzioni di backup/esportazione delle impostazioni
- Reinserisci l'URL se necessario

### 4. Verifica la correttezza dell'URL

Prima di salvare:
1. Testa l'URL in un browser
2. Assicurati che carichi il tuo sito WordPress
3. Includi `https://` o `http://`

## Checklist di configurazione

Quando configuri WCPOS per la prima volta:

1. ☐ Inserisci l'URL del tuo sito WordPress
2. ☐ Verifica che il plugin WCPOS sia installato e attivo
3. ☐ Accedi con le tue credenziali WordPress
4. ☐ Consenti il completamento della sincronizzazione iniziale dei dati
