# API06002: URL API mancante {#api06002-missing-api-url}

## Cosa significa {#what-this-means}

Nessun URL è stato configurato per collegarsi al tuo negozio WooCommerce. Il POS ha bisogno di sapere dove si trova il tuo negozio.

## Cause comuni {#common-causes}

- **Impostazione iniziale** — L'URL non è mai stato inserito
- **Configurazione cancellata** — Le impostazioni sono state ripristinate o cancellate
- **Dati dell'app cancellati** — I dati dell'applicazione sono stati eliminati
- **Problema di migrazione** — L'URL perso durante l'aggiornamento o la migrazione

## Come risolvere {#how-to-fix}

### 1. Inserisci il tuo URL del negozio {#1-enter-your-store-url}

Nell'app POS:
1. Apri le impostazioni o la schermata di accesso
2. Trova il campo di configurazione URL/Sito
3. Inserisci l'URL del tuo sito WordPress
4. Esempio: `https://tuo-negozio.com`

### 2. Trova il tuo URL WordPress {#2-find-your-wordpress-url}

Se non sei sicuro del tuo URL:
1. Accedi a WordPress Admin
2. Vai a Impostazioni → Generale
3. Controlla "Indirizzo WordPress (URL)"
4. Usa questo URL nel POS

### 3. Controlla le impostazioni salvate {#3-check-stored-settings}

Se l'URL era precedentemente configurato:
- Controlla se i dati dell'app sono stati accidentalmente cancellati
- Cerca opzioni di backup/esportazione delle impostazioni
- Reinserisci l'URL se necessario

### 4. Verifica che l'URL sia corretto {#4-verify-url-is-correct}

Prima di salvare:
1. Prova l'URL in un browser
2. Assicurati che carichi il tuo sito WordPress
3. Includi `https://` o `http://`

## Elenco di controllo per la configurazione {#setup-checklist}

Quando configuri WCPOS per la prima volta:

1. ☐ Inserisci l'URL del tuo sito WordPress
2. ☐ Verifica che il plugin WCPOS sia installato e attivo
3. ☐ Accedi con le tue credenziali WordPress
4. ☐ Consenti il completamento della sincronizzazione iniziale dei dati
