# DB02001: Record Duplicato {#db02001-duplicate-record}

## Cosa Significa {#what-this-means}

Il database ha tentato di inserire un record con un ID già esistente. Ogni record deve avere un identificatore unico.

## Cause Comuni {#common-causes}

- **Conflitto di sincronizzazione** — Dati identici sincronizzati due volte
- **Condizione di corsa** — Operazioni multiple che creano lo stesso record
- **Danneggiamento dei dati** — Gli ID sono stati danneggiati o duplicati
- **Problema di importazione** — Importazione di dati già esistenti

## Come Risolvere {#how-to-fix}

### 1. Aggiorna i Dati {#1-refresh-data}

Ri-sincronizza dal server:
- Estrai i dati più recenti
- Lascia che il processo di sincronizzazione risolva i conflitti
- I dati più recenti devono avere la precedenza

### 2. Ripeti l'Operazione {#2-retry-the-operation}

Se stai creando nuovi record:
- Il sistema potrebbe averlo già creato
- Controlla se il record esiste
- Evita di fare clic su invia più volte

### 3. Cancella la Cache Locale {#3-clear-local-cache}

Reimposta i dati locali:
1. Pulisci il database/cache locale
2. Accedi di nuovo
3. Sincronizza i dati freschi

### 4. Controlla i Duplicati in WooCommerce {#4-check-for-duplicates-in-woocommerce}

Se il problema persiste:
- Controlla WooCommerce per voci duplicate
- Cerca prodotti/ordini con gli stessi ID
- Pulisci eventuali duplicati nell'amministratore

### 5. Segnala Problemi Persistenti {#5-report-persistent-issues}

Se questo accade frequentemente:
- Nota quali azioni lo provocano
- Controlla eventuali schemi
- Riporta al supporto con dettagli

## Perché Questo è Importante {#why-this-matters}

ID duplicati causerebbero:
- Confusione su quale record sia corretto
- Sovrascritture dei dati
- Problemi di sincronizzazione

L'errore previene la corruzione dei dati.
