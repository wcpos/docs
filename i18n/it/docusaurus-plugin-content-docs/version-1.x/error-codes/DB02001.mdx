# DB02001: Record Duplicato

## Cosa Significa

Il database ha tentato di inserire un record con un ID già esistente. Ogni record deve avere un identificatore univoco.

## Cause Comuni

- **Conflitto di sincronizzazione** — Stessi dati sincronizzati due volte
- **Condizione di gara** — Operazioni multiple che creano lo stesso record
- **Corruzione dati** — Gli ID sono stati corrotti o duplicati
- **Problema di importazione** — Importazione di dati già esistenti

## Come Risolvere

### 1. Aggiorna Dati

Risincronizza dal server:
- Recupera gli ultimi dati
- Lascia che il processo di sincronizzazione risolva i conflitti
- I dati più recenti dovrebbero prevalere

### 2. Riprova l'Operazione

Durante la creazione di nuovi record:
- Il sistema potrebbe averlo già creato
- Verifica se il record esiste
- Evita di fare clic su invio più volte

### 3. Cancella Cache Locale

Reimposta i dati locali:
1. Cancella il database/cache locale
2. Effettua nuovamente l'accesso
3. Sincronizza nuovamente i dati

### 4. Controlla Duplicati in WooCommerce

Se il problema persiste:
- Controlla in WooCommerce eventuali voci duplicate
- Cerca prodotti/ordini con ID identici
- Pulisci eventuali duplicati nell'admin

### 5. Segnala Problemi Persistenti

Se accade frequentemente:
- Annota le azioni che lo causano
- Cerca eventuali pattern
- Segnala al supporto con dettagli

## Perché È Importante

Record duplicati causerebbero:
- Confusione su quale record sia corretto
- Sovrascritture di dati
- Problemi di sincronizzazione

L'errore previene la corruzione dei dati.
