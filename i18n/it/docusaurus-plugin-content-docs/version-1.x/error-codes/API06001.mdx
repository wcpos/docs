# API06001: Formato URL non valido

## Cosa Significa

L'URL configurato per il tuo sito WooCommerce non è in un formato valido. Gli URL devono seguire le convenzioni standard degli indirizzi web.

## Cause Comuni

- **Protocollo mancante** — L'URL non inizia con `http://` o `https://`
- **Errore di battitura nell'URL** — Errori di ortografia nel dominio
- **Caratteri non validi** — Caratteri speciali non consentiti negli URL
- **URL incompleto** — All'URL mancano parti come l'estensione del dominio

## Come Risolvere

### 1. Controllare il Formato dell'URL

Un URL valido dovrebbe essere:
- ✅ `https://negozio.com`
- ✅ `https://www.negozio.com`
- ✅ `https://store.tuodominio.com`
- ❌ `negozio.com` (protocollo mancante)
- ❌ `https://negozio` (estensione mancante)
- ❌ `https://negozio web.com` (contiene uno spazio)

### 2. Includere il Protocollo

Includere sempre `https://` o `http://`:
- Preferito: `https://` (sicuro)
- Alternativa: `http://` (se HTTPS non è disponibile)

### 3. Rimuovere Caratteri Extra

Assicurarsi che l'URL non contenga:
- Barre finali (opzionali ma essere coerenti)
- Spazi extra
- Caratteri speciali
- Parametri URL (a meno che non siano necessari)

### 4. Verificare il Funzionamento dell'URL

Testare l'URL in un browser:
1. Copiare l'URL in uso
2. Incollarlo in un browser
3. Dovrebbe caricare il sito WordPress

### 5. Aggiornare la Configurazione POS

Inserire l'URL corretto nelle impostazioni POS:
- Corrispondere esattamente a ciò che funziona nel browser
- Includere o escludere `www.` come utilizzato dal sito

## Esempi

| Non Valido | Valido |
|---------|-------|
| `negozio.com` | `https://negozio.com` |
| `https://negozio web.com` | `https://negozio.com` |
| `https://negozio` | `https://negozio.com` |
| `htp://negozio.com` | `https://negozio.com` |
