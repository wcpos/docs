# API02002: Token Scaduto

## Cosa Significa

La tua sessione di autenticazione è scaduta. Per motivi di sicurezza, le sessioni di accesso non durano per sempre. Il POS deve aggiornare la tua autenticazione oppure potresti dover accedere nuovamente.

## Cause Comuni

- **Lunga inattività** — Non hai utilizzato il POS per un periodo prolungato
- **Durata del token del server** — L'espirazione del token del server è impostata su un tempo molto breve
- **Disallineamento dell'orario del server** — Gli orologi del server e del dispositivo non sono sincronizzati

## Mappatura degli Errori del Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice del Server | Fonte |
|------------------|-------|
| `jwt_auth_expired_token` | Plugin di Autenticazione JWT |

## Come Risolvere

### 1. Accedi Nuovamente

La soluzione più semplice è disconnettersi e accedere di nuovo:
1. Disconnettiti dal POS
2. Inserisci nuovamente le tue credenziali
3. Questo crea una nuova sessione

### 2. Controlla il Rinnovo del Token

Il POS dovrebbe aggiornare automaticamente i token. Se non funziona:
- Verifica la tua connessione a Internet
- Assicurati che il server sia accessibile
- Consulta [API02007](./API02007) se il rinnovo non riesce

### 3. Controlla l'Ora del Server

L'espirazione del token dipende da un orario preciso:
- Assicurati che l'orologio del tuo server sia corretto
- Controlla che l'orario del tuo dispositivo sia accurato
- Usa la sincronizzazione automatica dell'ora su entrambi

### 4. Rivedi le Impostazioni del Token

Se i token scadono troppo rapidamente, potrebbe essere necessario regolare le impostazioni del server (consulta il tuo fornitore di hosting o l'amministratore di WordPress).
