# API02002: Token Scaduto

## Cosa Significa

La tua sessione di autenticazione è scaduta. Per ragioni di sicurezza, le sessioni di accesso non durano all'infinito. Il POS deve aggiornare la tua autenticazione o potrebbe essere necessario effettuare nuovamente l'accesso.

## Cause Comuni

- **Inattività prolungata** — Non hai utilizzato il POS per un periodo di tempo prolungato
- **Durata del token del server** — Il server ha impostato un tempo di scadenza del token molto breve
- **Discrepanza dell'ora del server** — Gli orologi del server e del dispositivo non sono sincronizzati

## Mappatura dell'Errore del Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice del Server | Origine |
|------------------|---------|
| `jwt_auth_expired_token` | Plugin di Autenticazione JWT |

## Come Risolvere

### 1. Accedere Nuovamente

La soluzione più semplice è disconnettersi e riconnettersi:
1. Esci dal POS
2. Inserisci nuovamente le tue credenziali
3. Questo creerà una nuova sessione

### 2. Verificare l'Aggiornamento del Token

Il POS dovrebbe aggiornare automaticamente i token. Se non funziona:
- Controlla la tua connessione internet
- Verifica l'accessibilità del server
- Consulta [API02007](./API02007) se l'aggiornamento non riesce

### 3. Controllare l'Ora del Server

La scadenza del token dipende da un orario preciso:
- Assicurati che l'orologio del server sia corretto
- Verifica che l'ora del tuo dispositivo sia accurata
- Usa la sincronizzazione automatica dell'ora su entrambi

### 4. Rivedere le Impostazioni del Token

Se i token scadono troppo rapidamente, potrebbe essere necessario modificare le impostazioni del server (consulta il tuo provider di hosting o l'amministratore WordPress).
