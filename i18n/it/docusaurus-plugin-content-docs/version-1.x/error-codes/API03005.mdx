# API03005: Limite di Richieste Superato

## Cosa Significa

Hai effettuato troppe richieste al server in un breve periodo. Il rate limiting protegge il server da sovraccarichi.

## Cause Comuni

- **Operazioni di sincronizzazione rapide** — Più dispositivi che si sincronizzano simultaneamente
- **Aggiornamento aggressivo** — Aggiornamento dei dati troppo frequentemente
- **Plugin di sicurezza** — Un plugin sta imponendo limiti alle richieste
- **Limiti di hosting** — Il tuo piano di hosting ha limiti di tasso API

## Mappatura degli Errori del Server

Questo codice di errore si attiva quando il server restituisce:

| Codice Server | Origine |
|---------------|---------|
| HTTP 429      | Qualsiasi risposta del server (fallback) |

## Come Risolvere

### 1. Aspetta e Ripeti

I limiti di richiesta sono temporanei:
- Aspetta 1-5 minuti
- Il POS tenterà automaticamente di riprovare
- Evita di aggiornare manualmente ripetutamente

### 2. Riduci le Operazioni Concomitanti

Se utilizzi più terminali POS:
- Scaglionare la sincronizzazione iniziale tra i dispositivi
- Evita di aggiornare tutti i dispositivi contemporaneamente
- Lascia completare uno prima di iniziarne un altro

### 3. Controlla i Plugin di Sicurezza

Plugin come Wordfence possono bloccare richieste rapide:
- Aggiungi l'applicazione POS alla lista bianca
- Aggiungi gli IP dei tuoi dispositivi POS alla lista bianca
- Regola le soglie di rate limiting

### 4. Rivedi i Limiti di Hosting

Alcuni host limitano le richieste API:
- Controlla i dettagli del tuo piano di hosting
- Considera di effettuare un upgrade se i limiti sono troppo bassi
- Contatta il supporto per chiedere un aumento dei limiti

### 5. Ottimizza la Frequenza di Sincronizzazione

Nelle impostazioni POS:
- Regola gli intervalli di aggiornamento automatico
- Usa la sincronizzazione manuale quando appropriato
- Evita aggiornamenti non necessari dei dati
