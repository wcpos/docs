# API03005: Limite di Richiesta Superato

## Cosa Significa

Hai effettuato troppe richieste al server in un breve periodo. Il limitazione delle richieste protegge il server dall'essere sopraffatto.

## Cause Comuni

- **Operazioni di sincronizzazione rapide** — Dispositivi multipli che sincronizzano simultaneamente
- **Aggressivo aggiornamento** — Aggiornamento dei dati troppo frequentemente
- **Plugin di sicurezza** — Un plugin sta applicando limiti alle richieste
- **Limiti di hosting** — Il tuo piano di hosting ha limiti sulle API

## Mappatura degli Errori del Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Origine |
|---------------|---------|
| HTTP 429 | Qualsiasi risposta del server (fallback) |

## Come Risolvere

### 1. Aspetta e Ripeti

I limiti di richiesta sono temporanei:
- Aspetta 1-5 minuti
- Il POS riproverà automaticamente
- Evita di aggiornare manualmente ripetutamente

### 2. Ridurre le Operazioni Concomitanti

Se utilizzi più terminali POS:
- Distribuisci la sincronizzazione iniziale tra i dispositivi
- Evita di aggiornare tutti i dispositivi contemporaneamente
- Lascia completare uno prima di iniziarne un altro

### 3. Controlla i Plugin di Sicurezza

Plugin come Wordfence potrebbero bloccare richieste rapide:
- Aggiungi l'applicazione POS alla lista bianca
- Aggiungi gli indirizzi IP dei tuoi dispositivi POS alla lista bianca
- Regola le soglie di limitazione delle richieste

### 4. Rivedi i Limiti di Hosting

Alcuni host limitano le richieste API:
- Controlla i dettagli del tuo piano di hosting
- Considera di aggiornarlo se i limiti sono troppo bassi
- Contatta il supporto per aumentare i limiti

### 5. Ottimizza la Frequenza di Sincronizzazione

Nelle impostazioni del POS:
- Regola gli intervalli di aggiornamento automatico
- Usa la sincronizzazione manuale quando appropriato
- Evita aggiornamenti dei dati non necessari
