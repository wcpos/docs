# API03005: Limite di Richieste Superato

## Cosa Significa

Hai effettuato troppe richieste al server in un breve periodo. La limitazione delle richieste protegge il server dal sovraffollamento.

## Cause Comuni

- **Operazioni di sincronizzazione rapide** — Più dispositivi sincronizzano contemporaneamente
- **Aggiornamento aggressivo** — Aggiornamento dei dati troppo frequente
- **Plugin di sicurezza** — Un plugin sta applicando limiti di richieste
- **Limiti di hosting** — Il tuo piano di hosting ha limiti di richieste API

## Mappatura Errore Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Origine |
|-------------|--------|
| HTTP 429 | Qualsiasi risposta del server (fallback) |

## Come Risolvere

### 1. Attendere e Riprovare

I limiti di richieste sono temporanei:
- Attendere 1-5 minuti
- Il POS riproverà automaticamente
- Evitare di aggiornare ripetutamente manualmente

### 2. Ridurre Operazioni Concorrenti

Se si utilizzano più terminali POS:
- Scaglionare la sincronizzazione iniziale tra i dispositivi
- Evitare di aggiornare tutti i dispositivi contemporaneamente
- Attendere che uno completi prima di avviarne un altro

### 3. Verificare Plugin di Sicurezza

Plugin come Wordfence possono bloccare richieste rapide:
- Inserire l'applicazione POS nella whitelist
- Inserire gli indirizzi IP dei dispositivi POS nella whitelist
- Modificare le soglie di limitazione delle richieste

### 4. Rivedere Limiti di Hosting

Alcuni host limitano le richieste API:
- Controllare i dettagli del proprio piano di hosting
- Considerare l'upgrade se i limiti sono troppo bassi
- Contattare il supporto per aumentare i limiti

### 5. Ottimizzare la Frequenza di Sincronizzazione

Nelle impostazioni POS:
- Regolare gli intervalli di aggiornamento automatico
- Utilizzare la sincronizzazione manuale quando appropriato
- Evitare aggiornamenti di dati non necessari
