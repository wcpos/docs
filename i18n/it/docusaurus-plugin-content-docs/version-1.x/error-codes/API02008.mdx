# API02008: Token di Aggiornamento Non Valido

## Cosa Significa

Il token di aggiornamento memorizzato dal POS non è riconosciuto dal server. I token di aggiornamento vengono utilizzati per ottenere nuovi token di accesso senza richiedere nuovamente di effettuare il login.

## Cause Comuni

- **Token revocato** — Un amministratore ha invalidato la tua sessione
- **Secret del server cambiati** — Le chiavi di sicurezza di WordPress sono state aggiornate
- **Corruzione dei dati** — Il token è stato corrotto durante la memorizzazione
- **Accessi multipli** — Accedere altrove potrebbe aver invalidato questo token

## Come Risolvere

### 1. Accedi Nuovamente

Poiché il token di aggiornamento è non valido, è necessario autenticarsi nuovamente:
1. Disconnettiti dal POS
2. Inserisci il tuo nome utente e la tua password
3. Questo genera nuovi token validi

### 2. Controlla le Sessioni Revocate

Se un amministratore ha revocato le sessioni:
- Questa è una pratica di sicurezza normale
- Accedi semplicemente di nuovo
- La tua nuova sessione funzionerà normalmente

### 3. Controlla le Modifiche del Server

Se questo influisce su tutti gli utenti:
- Le sali/chiavi di WordPress potrebbero essere cambiate
- Le impostazioni del plugin JWT potrebbero essere state modificate
- Contatta l'amministratore del server

### 4. Cancella i Dati Memorizzati

Se continui ad avere problemi:
- Cancella i dati dell'applicazione POS
- Rimuovi le credenziali memorizzate nella cache
- Inizia con un accesso pulito
