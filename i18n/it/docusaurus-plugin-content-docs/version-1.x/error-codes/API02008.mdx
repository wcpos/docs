# API02008: Token di Aggiornamento Non Valido

## Cosa Significa

Il token di aggiornamento memorizzato dal POS non è riconosciuto dal server. I token di aggiornamento vengono utilizzati per ottenere nuovi token di accesso senza dover effettuare nuovamente l'accesso.

## Cause Comuni

- **Token revocato** — Un amministratore ha invalidato la tua sessione
- **Segreti del server modificati** — Le chiavi di sicurezza di WordPress sono state aggiornate
- **Corruzione dei dati** — Il token è stato danneggiato durante la memorizzazione
- **Accessi multipli** — Accedere da un altro dispositivo potrebbe avere invalidato questo token

## Come Risolvere

### 1. Accedi Nuovamente

Poiché il token di aggiornamento non è valido, dovrai autenticarti nuovamente:
1. Esci dal POS
2. Inserisci il tuo nome utente e password
3. Questo genererà nuovi token validi

### 2. Verifica Sessioni Revocate

Se un amministratore ha revocato le sessioni:
- Questa è una normale prassi di sicurezza
- Accedi semplicemente di nuovo
- La tua nuova sessione funzionerà normalmente

### 3. Controlla Modifiche al Server

Se questo riguarda tutti gli utenti:
- I salt/chiavi di WordPress potrebbero essere cambiati
- Le impostazioni del plugin JWT potrebbero essere state modificate
- Contatta l'amministratore del server

### 4. Cancella Dati Memorizzati

Se continui ad avere problemi:
- Cancella i dati dell'applicazione POS
- Rimuovi le credenziali memorizzate nella cache
- Inizia con un nuovo accesso
