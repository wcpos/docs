# API03007: Coda di Richiesta Piena

## Cosa Significa

Il POS ha troppe richieste in attesa di invio. La coda interna è piena e non può accettare ulteriori richieste fino a quando alcune non vengono completate.

## Cause Comuni

- **Rete lenta** — Richieste in attesa a causa di una connessione lenta
- **Server non risponde** — Server che impiega troppo tempo per rispondere
- **Azioni rapide** — Esecuzione di molte azioni più velocemente di quanto possano essere elaborate
- **Sovraccarico di sincronizzazione** — Grande operazione di sincronizzazione che blocca la coda

## Come Risolvere

### 1. Attendere che la Coda si Svuoti

Dai tempo alle richieste in attesa di completarsi:
- Aspetta 30-60 secondi
- Osserva gli indicatori di sincronizzazione fino a completamento
- Evita ulteriori azioni fino a quando non si è svuotata

### 2. Controlla la Connessione di Rete

Una connessione lenta provoca un arretrato:
- Testa la tua velocità di internet
- Prova una connessione più veloce
- Avvicinati al router WiFi

### 3. Controlla il Tempo di Risposta del Server

Se il server è lento:
- Testa l'accesso al tuo sito in un browser
- Contatta l'hosting se il sito è lento
- Aspetta che il carico del server diminuisca

### 4. Riavvia il POS

Se la coda è bloccata:
1. Chiudi l'applicazione POS
2. Aspetta un momento
3. Riapri il POS
4. Le richieste in attesa potrebbero dover essere ri-iniziate

### 5. Evita Azioni Rapide

Durante i periodi di sincronizzazione o lenti:
- Aspetta che un'azione si completi prima di iniziarne un'altra
- Non cliccare ripetutamente sui pulsanti
- Sii paziente con le operazioni di rete

## Comportamento Offline

Quando la coda si riempie a causa di condizioni offline:
- Il POS terrà le richieste in attesa fino a quando non sarà di nuovo online
- Puoi comunque sfogliare i prodotti e i clienti nella cache
- Puoi iniziare nuovi ordini e aggiungere articoli al carrello
- Completare gli ordini richiede che la connettività venga ripristinata
