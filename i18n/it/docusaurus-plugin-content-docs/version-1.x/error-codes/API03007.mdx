# API03007: Coda di Richieste Piena

## Cosa Significa

Il POS ha troppe richieste in sospeso in attesa di essere inviate. La coda interna è piena e non può accettare ulteriori richieste finché alcune non vengono completate.

## Cause Comuni

- **Rete lenta** — Richieste in accumulo a causa di una connessione lenta
- **Server non reattivo** — Server che impiega troppo tempo a rispondere
- **Azioni rapide** — Esecuzione di molte azioni più velocemente di quanto possano essere elaborate
- **Sovraccarico sincronizzazione** — Grande operazione di sincronizzazione che blocca la coda

## Come Risolvere

### 1. Attendere lo Svuotamento della Coda

Dare tempo alle richieste in sospeso di completarsi:
- Attendere 30-60 secondi
- Osservare gli indicatori di sincronizzazione fino al completamento
- Evitare azioni aggiuntive finché non è tutto chiaro

### 2. Verificare la Connessione di Rete

Una connessione lenta causa accumulo:
- Testare la velocità di Internet
- Provare una connessione più veloce
- Spostarsi più vicino al router WiFi

### 3. Controllare il Tempo di Risposta del Server

Se il server è lento:
- Testare l'accesso al sito in un browser
- Contattare l'hosting se il sito è lento
- Attendere la diminuzione del carico del server

### 4. Riavviare il POS

Se la coda è bloccata:
1. Chiudere l'applicazione POS
2. Attendere un momento
3. Riaprire il POS
4. Le richieste in sospeso potrebbero dover essere ri-avviate

### 5. Evitare Azioni Rapide

Durante la sincronizzazione o in periodi lenti:
- Attendere il completamento di un'azione prima di iniziarne un'altra
- Non fare clic ripetutamente sui pulsanti
- Essere pazienti con le operazioni di rete

## Comportamento Offline

Quando la coda si riempie in condizioni offline:
- Il POS manterrà le richieste in sospeso fino a quando non sarà di nuovo online
- È possibile sfogliare prodotti e clienti memorizzati nella cache
- È possibile avviare nuovi ordini e aggiungere articoli al carrello
- Completare gli ordini richiede il ripristino della connettività
