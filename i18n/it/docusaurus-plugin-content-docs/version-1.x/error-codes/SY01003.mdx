# SY01003: Autorizzazione Negata

## Cosa Significa

L'applicazione POS non ha l'autorizzazione per accedere a una risorsa richiesta. Potrebbe trattarsi di un file, una cartella o una funzione di sistema.

## Cause Comuni

- **Permessi dei file** — Impossibile leggere o scrivere i file necessari
- **Autorizzazioni dell'app** — L'applicazione non ha le autorizzazioni richieste
- **Cartella protetta** — Tentativo di accedere a una posizione riservata
- **Software di sicurezza** — Antivirus che blocca l'accesso

## Come Risolvere

### 1. Esegui come Amministratore (Windows)

Fare clic con il tasto destro sull'app e selezionare "Esegui come amministratore" se necessario per la configurazione iniziale.

### 2. Controlla i Permessi dell'Applicazione

**macOS:**
- Preferenze di Sistema → Sicurezza e Privacy → Privacy
- Verificare che l'app abbia le autorizzazioni necessarie

**Windows:**
- Impostazioni → Privacy
- Rivedere i permessi delle app

### 3. Controlla i Permessi della Cartella

Assicurarsi che l'app possa accedere alla sua cartella dati:
- L'utente dovrebbe possedere la directory dei dati dell'app
- Abilitare i permessi di scrittura
- Nessun flag di sola lettura sulla cartella

### 4. Rivedi le Impostazioni Antivirus

Il software di sicurezza potrebbe bloccare l'accesso:
- Controllare i registri dell'antivirus per i blocchi
- Aggiungere l'app POS all'elenco di eccezioni
- Inserire nella whitelist la cartella dei dati dell'app

### 5. Reinstalla l'Applicazione

Se i permessi sono corrotti:
1. Disinstallare l'applicazione POS
2. Riavviare il dispositivo
3. Reinstallare in una posizione standard
4. Eseguire con permessi normali (non amministratore)

## Permessi del Browser (Versione Web)

Se si utilizza la versione web:
- Consentire l'archiviazione locale per il sito
- Abilitare i cookie per il sito
- Non utilizzare impostazioni del browser eccessivamente restrittive
