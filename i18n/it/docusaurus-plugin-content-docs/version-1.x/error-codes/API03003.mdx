# API03003: Valore Parametro Non Valido

## Cosa Significa

Uno dei valori nella tua richiesta non è valido. Il formato, il tipo o il valore dei dati non corrisponde a quanto previsto dal server.

## Cause Comuni

- **Tipo di dati errato** — Testo dove è previsto un numero (o viceversa)
- **Fuori intervallo** — Valore che supera i limiti consentiti
- **Formato non valido** — Campi email, telefono o altri formattati in modo incorreto
- **Riferimento non valido** — Riferimento a un ID che non esiste

## Mapping Errori Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Fonte |
|-------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## Come Risolvere

### 1. Controllare i Valori di Input

Rivedere i dati che si stanno inviando:
- **Prezzi** — Devono essere numeri validi
- **Quantità** — Devono essere interi positivi
- **Email** — Devono essere in un formato email valido
- **ID** — Devono fare riferimento a record esistenti

### 2. Cercare Errori Specifici dei Campi

L'errore può indicare quale campo non è valido:
- Controllare i dettagli del messaggio di errore
- Correggere il campo specifico
- Riprovare l'azione

### 3. Verificare l'Esistenza dei Dati Referenziati

Se si fa riferimento a prodotti, clienti o altri record:
- Assicurarsi che l'elemento esista in WooCommerce
- Verificare se è stato eliminato di recente
- Sincronizzare i dati per aggiornare i record locali

### 4. Controllare i Vincoli dei Campi

WooCommerce può avere vincoli:
- Valori massimi/minimi
- Formati richiesti
- Opzioni consentite per campi di selezione

### 5. Cancellare e Reinserire

A volte i dati possono corrompersi:
- Cancellare il campo problematico
- Reinserire il valore da zero
- Evitare di copiare e incollare da altre fonti
