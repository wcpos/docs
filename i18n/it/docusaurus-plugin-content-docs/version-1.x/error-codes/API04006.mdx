# API04006: Risorsa Non Trovata

## Cosa Significa

Il server non ha trovato la risorsa richiesta. Questo corrisponde tipicamente a un codice di stato HTTP 404, che indica che il prodotto, l'ordine, il cliente o altra risorsa che si sta cercando di accedere non esiste.

## Cause Comuni

- **Record eliminato** — La risorsa è stata eliminata sul server
- **ID errato** — Viene utilizzato un ID non corretto o obsoleto
- **Problemi di sincronizzazione** — I dati locali fanno riferimento a una risorsa che non esiste più
- **Errore di configurazione URL** — L'endpoint API non è corretto
- **Problemi di permalink** — I permalink di WordPress devono essere aggiornati

## Mappatura Errori del Server

Questo codice di errore viene attivato quando il server restituisce:

| Codice Server | Origine |
|--------------|---------|
| `rest_no_route` | WordPress REST API |
| HTTP 404 | Qualsiasi risposta del server |

## Come Risolvere

### 1. Verificare l'Esistenza della Risorsa

Verificare che la risorsa esista ancora sul sito WooCommerce:
- Accedere all'amministrazione di WordPress
- Navigare nella sezione pertinente (Prodotti, Ordini, Clienti)
- Cercare l'elemento per ID o nome

### 2. Aggiornare i Dati Locali

Se la risorsa è stata eliminata lato server:
1. Aprire le impostazioni del POS
2. Navigare nella sezione dati pertinente
3. Attivare una sincronizzazione/aggiornamento per aggiornare i dati locali
4. L'elemento eliminato dovrebbe essere rimosso localmente

### 3. Controllare i Permalink di WordPress

Se non vengono trovate più risorse:
1. Andare su **Impostazioni → Permalink** nell'amministrazione di WordPress
2. Fare clic su **Salva Modifiche** (anche senza modificare nulla)
3. Questo aggiorna la struttura dei permalink

### 4. Verificare le Rotte API

Testare direttamente l'API REST:
```
https://tuosito.com/wp-json/wc/v3/products
```
Se restituisce un 404, potrebbe esserci un problema di configurazione del server.

### 5. Controllare Conflitti di Plugin

Se le rotte REST API sono mancanti:
1. Assicurarsi che WooCommerce sia attivo
2. Assicurarsi che il plugin WCPOS sia attivo
3. Disattivare temporaneamente altri plugin per testare
