# API03004: Richiesta Troppo Grande

## Cosa Significa

La richiesta che stai inviando supera i limiti di dimensione del server. Questo accade tipicamente quando si tenta di inviare troppi dati contemporaneamente.

## Cause Comuni

- **Operazioni batch massive** — Tentativo di sincronizzare troppi record in una volta
- **Immagini grandi** — Caricamento di immagini di dimensioni eccessive
- **Troppi articoli** — Ordine con un numero estremamente elevato di articoli
- **Limiti del server** — Limiti di caricamento bassi di PHP o del server web

## Come Risolvere

### 1. Ridurre le Dimensioni della Richiesta

Se si sincronizzano dati:
- Prova a sincronizzare in batch più piccoli
- Il POS dovrebbe gestire questo automaticamente
- Attendi il completamento della sincronizzazione corrente prima di avviarne un'altra

### 2. Controllare le Dimensioni delle Immagini

Se carichi immagini:
- Ridimensiona le immagini prima del caricamento
- Usa formati compressi (JPEG vs BMP)
- La maggior parte delle immagini di prodotti funziona bene sotto 1MB

### 3. Dividere Ordini Grandi

Se un ordine ha molti articoli:
- Considera di dividere in più ordini
- Questo è raro nell'uso normale del POS

### 4. Aumentare i Limiti del Server

Contatta il tuo provider di hosting o modifica le impostazioni PHP:

```php
// In php.ini o .htaccess
upload_max_filesize = 64M
post_max_size = 64M
max_input_vars = 5000
```

### 5. Controllare i Limiti del Server Web

Nginx o Apache possono avere i propri limiti:
- `client_max_body_size` per Nginx
- `LimitRequestBody` per Apache
