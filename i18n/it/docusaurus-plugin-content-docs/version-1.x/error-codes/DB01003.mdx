# DB01003: Transazione Fallita

## Cosa Significa

Una transazione del database non è stata completata. Le transazioni raggruppano più operazioni insieme — se una parte fallisce, tutto viene ripristinato per mantenere l'integrità dei dati.

## Cause Comuni

- **Accesso simultaneo** — Più operazioni che cercano di modificare gli stessi dati
- **Memoria piena** — Nessuno spazio per scrivere nuovi dati
- **Database bloccato** — Un altro processo sta bloccando il database
- **Interruzione di alimentazione** — Operazione interrotta inaspettatamente

## Come Risolvere

### 1. Ripetere l'Operazione

Il problema potrebbe essere temporaneo:
- Attendere un momento
- Riprovare l'operazione
- Evitare tentativi ripetuti rapidamente

### 2. Verificare Conflitti

Se sono aperti più dispositivi o schede:
- Utilizzare una sola istanza alla volta
- Chiudere schede del browser duplicate
- Coordinare l'uso multi-dispositivo

### 3. Controllare lo Spazio di Archiviazione

Assicurarsi che ci sia spazio per i dati:
- Verificare lo spazio su disco disponibile
- Liberare spazio se necessario
- Cancellare la cache del browser (versione web)

### 4. Riavviare l'Applicazione

Reimpostare lo stato del database:
1. Chiudere completamente il POS
2. Attendere alcuni secondi
3. Riaprire l'applicazione

### 5. Cancellare e Risincronizzare

Se le transazioni falliscono costantemente:
1. Cancellare i dati locali
2. Accedere nuovamente
3. Sincronizzare nuovamente dal server

## Sicurezza delle Transazioni

WCPOS utilizza transazioni per garantire:
- Coerenza dei dati
- Operazioni complete (tutto o niente)
- Protezione da aggiornamenti parziali

Quando una transazione fallisce, i tuoi dati rimangono coerenti.
