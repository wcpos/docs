# API04005: Tentativo di Recupero JSON

## Cosa Significa

Il server ha inviato una risposta con alcuni contenuti JSON non validi, ma il POS ha tentato di recuperare ed estrarre dati validi. Questa è una notifica informativa piuttosto che un errore critico.

## Cosa è Successo

Il POS ha rilevato:
1. La risposta conteneva contenuti aggiuntivi prima o dopo il JSON
2. I dati JSON principali erano ancora identificabili
3. È stato tentato il recupero estraendo la porzione JSON valida

## Cause Comuni

- **Avvisi PHP in output** — Avvertenze PHP miscelate con JSON
- **Output di debug** — Debug di sviluppo lasciato abilitato
- **Avvisi di plugin** — Altri plugin che generano notifiche
- **Problemi di spazi bianchi** — Spazi aggiuntivi intorno al JSON

## Come Risolvere

### 1. Disattivare la Visualizzazione Errori PHP

In `wp-config.php`:
```php
define('WP_DEBUG_DISPLAY', false);
ini_set('display_errors', 0);
```

### 2. Abilitare la Registrazione Errori

Mantenere gli errori registrati per il debug:
```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
```

### 3. Verificare la Modalità Debug del Plugin

Alcuni plugin hanno modalità di debug che generano contenuti aggiuntivi:
- Rivedere le impostazioni del plugin
- Disattivare modalità di debug/sviluppo
- Verificare le opzioni di registrazione dettagliate

### 4. Rivedere Modifiche Recenti

Se questo è iniziato di recente:
- Cosa è cambiato sul tuo server?
- Sono stati aggiornati dei plugin?
- Sono state modificate le impostazioni PHP?

## È Grave?

Sebbene il POS abbia recuperato da questo problema, indica un problema di configurazione che dovrebbe essere risolto. Il processo di recupero:
- Potrebbe non funzionare sempre
- Aggiunge overhead di elaborazione
- Potrebbe mascherare altri problemi

Risolvi la causa alla base per garantire un funzionamento affidabile.
