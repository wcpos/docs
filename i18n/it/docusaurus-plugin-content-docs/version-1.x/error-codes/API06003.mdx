# API06003: Configurazione del sito non valida

## Cosa Significa

La configurazione del sito è invalida o incompleta. Questo potrebbe comportare URL errati, impostazioni di autenticazione o altri problemi di configurazione.

## Cause Comuni

- **Configurazione parziale** — Alcune impostazioni mancano
- **Impostazioni discordanti** — La configurazione non corrisponde al sito
- **Configurazione danneggiata** — Le impostazioni sono state corrotte
- **Modifiche al sito** — Il sito è stato modificato senza aggiornare la configurazione del POS

## Come Risolvere

### 1. Rivedere tutte le impostazioni

Controlla la configurazione completa:
- L'URL del sito è corretto
- L'autenticazione è correttamente configurata
- Eventuali impostazioni aggiuntive sono corrette

### 2. Riconfigurare da zero

Se la configurazione è danneggiata:
1. Cancella tutte le impostazioni memorizzate
2. Inizia nuovamente il processo di configurazione
3. Inserisci una nuova configurazione

### 3. Controlla i requisiti del sito

Verifica che il tuo sito WordPress soddisfi i requisiti:
- WordPress è installato e accessibile
- WooCommerce è installato e attivo
- Il plugin WCPOS è installato e attivo
- I permalink sono abilitati (non "Semplice")

### 4. Testa l'accesso al sito

Verifica che questi URL funzionino in un browser:
- `https://yoursite.com/` — Sito principale
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. Controlla le impostazioni di WordPress

Nell'amministrazione di WordPress:
1. Impostazioni → Generale — Verifica gli URL
2. Impostazioni → Permalink — Assicurati che non sia "Semplice"
3. WooCommerce → Stato — Controlla eventuali problemi

## Requisiti di Configurazione

Una configurazione valida necessita di:
- **URL del sito** — URL completo con protocollo
- **Autenticazione** — Credenziali valide o chiavi API
- **Permessi** — L'utente ha accesso al POS
- **Plugin** — I plugin richiesti sono attivi
