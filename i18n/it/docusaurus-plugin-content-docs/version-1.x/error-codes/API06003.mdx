# API06003: Configurazione del Sito Non Valida

## Cosa Significa

La configurazione del sito non è valida o incompleta. Questo potrebbe riguardare URL errati, impostazioni di autenticazione o altri problemi di configurazione.

## Cause Comuni

- **Configurazione parziale** — Alcune impostazioni sono mancanti
- **Impostazioni non corrispondenti** — La configurazione non corrisponde al sito
- **Configurazione corrotta** — Le impostazioni sono state danneggiate
- **Modifiche del sito** — Il sito è stato modificato senza aggiornare la configurazione POS

## Come Risolvere

### 1. Rivedere Tutte le Impostazioni

Controllare la configurazione completa:
- L'URL del sito è corretto
- L'autenticazione è configurata correttamente
- Tutte le impostazioni aggiuntive sono corrette

### 2. Riconfigurare da Zero

Se la configurazione è corrotta:
1. Cancellare tutte le impostazioni memorizzate
2. Avviare nuovamente il processo di configurazione
3. Inserire una nuova configurazione

### 3. Verificare i Requisiti del Sito

Verificare che il sito WordPress soddisfi i requisiti:
- WordPress è installato e accessibile
- WooCommerce è installato e attivo
- Il plugin WCPOS è installato e attivo
- I permalink sono abilitati (non "Semplice")

### 4. Testare l'Accesso al Sito

Verificare che questi URL funzionino in un browser:
- `https://iltuosito.com/` — Sito principale
- `https://iltuosito.com/wp-json/` — REST API
- `https://iltuosito.com/wp-json/wcpos/v1/` — API WCPOS

### 5. Controllare le Impostazioni di WordPress

Nell'Amministrazione di WordPress:
1. Impostazioni → Generali — Verificare gli URL
2. Impostazioni → Permalink — Assicurarsi che non siano "Semplici"
3. WooCommerce → Stato — Verificare eventuali problemi

## Requisiti di Configurazione

Una configurazione valida necessita di:
- **URL del Sito** — URL completo con protocollo
- **Autenticazione** — Credenziali o chiavi API valide
- **Autorizzazioni** — Utente con accesso POS
- **Plugin** — Plugin richiesti attivi
