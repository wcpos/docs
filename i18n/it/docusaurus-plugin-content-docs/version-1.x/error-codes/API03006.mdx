# API03006: Metodo Non Supportato

## Cosa Significa

Il metodo HTTP utilizzato (GET, POST, PUT, DELETE, ecc.) non è supportato per questo endpoint. Il server non accetta questo tipo di richiesta per questo URL.

## Cause Comuni

- **Discrepanza di versione** — Versioni di POS e plugin non compatibili
- **Endpoint rimosso** — Un endpoint API è stato deprecato
- **Configurazione server** — Server web che blocca alcuni metodi HTTP
- **Conflitto plugin** — Un altro plugin che modifica il comportamento dell'API REST

## Come Risolvere

### 1. Aggiornare Entrambi i Componenti

Assicurare la compatibilità:
- Aggiornare il plugin WordPress di WCPOS
- Aggiornare l'applicazione POS
- Verificare le note di rilascio per modifiche sostanziali

### 2. Controllare la Configurazione del Server

Alcuni server bloccano certi metodi HTTP:
- Assicurarsi che i metodi PUT e DELETE siano consentiti
- Verificare le restrizioni dei metodi in `.htaccess`
- Rivedere la configurazione Nginx

### 3. Verificare l'Accesso all'API REST

Testare l'API REST di WordPress:
1. Visitare `https://yoursite.com/wp-json/` in un browser
2. Dovrebbe restituire dati JSON
3. Se non funziona, l'API REST potrebbe essere disabilitata o bloccata

### 4. Verificare Conflitti tra Plugin

Disabilitare temporaneamente altri plugin:
- Plugin di sicurezza potrebbero bloccare metodi
- Altri plugin API REST potrebbero causare conflitti
- Riabilitare uno alla volta per identificare il problema

### 5. Verificare le Restrizioni di Hosting

Alcuni host limitano i metodi HTTP:
- Contattare il supporto hosting
- Richiedere l'abilitazione di tutti i metodi standard
- Considerare il cambio di host se troppo restrittivo
