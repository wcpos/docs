WCPOS mantiene un campo ID fiscale per i clienti — un numero di partita IVA, ABN, CPF, GSTIN, NIF e così via — per le vendite B2B in cui l'identificativo fiscale dell'acquirente deve comparire sulla fattura. Quando viene effettuato un ordine per un cliente, WCPOS copia il suo ID fiscale nell'ordine, così può essere stampato sulla ricevuta.

La sezione **ID fiscali** in **WP Admin > POS > Impostazioni > Generale** controlla come questo campo viene rilevato e archiviato.

## Plugin e nomi dei campi {#plugins-and-field-names}

Molti plugin WooCommerce aggiungono gia ai clienti un campo per la partita IVA o il codice fiscale, e ciascuno usa nomi di campo del database (meta key) diversi. Per mantenere funzionanti i dati cliente esistenti — senza alcuna migrazione — WCPOS rileva cio che e installato e scrive automaticamente nel campo corretto.

WCPOS risolve il nome del campo in questo ordine:

1. **E attivo un plugin riconosciuto** — WCPOS scrive nel campo di quel plugin. I plugin rilevati includono:
   - WooCommerce EU VAT Number
   - Aelia EU VAT Assistant (incluso il relativo stato di verifica)
   - EU VAT for WooCommerce (WPFactory)
   - WooCommerce Germanized / Germanized Pro
   - Brazilian Market on WooCommerce / Extra Checkout Fields for Brazil
   - NIF/CIF Spain
2. **Nessun plugin riconosciuto** — WCPOS analizza gli ordini recenti per individuare il campo IVA/CPF/ecc. già in uso e scrive in quello.
3. **Non è stato trovato nulla** — WCPOS usa valori predefiniti appropriati (`_billing_vat_number` for VAT-style IDs, `_billing_cpf` / `_billing_cnpj` for Brazil, `_billing_gstin` for India, and so on).

### Scegliere personalmente il nome del campo {#choosing-the-field-name-yourself}

Per sostituire il nome del campo rilevato, vai a **WP Admin > POS > Impostazioni > Generale**, apri la sezione **ID fiscali** ed espandi **Avanzate: usa un nome campo diverso**.

Un banner mostra quale plugin è stato rilevato automaticamente (oppure "nessun plugin rilevato", con i valori predefiniti integrati in uso). Il campo di sostituzione include un completamento automatico che suggerisce le meta key comunemente usate da altri plugin. Svuota il campo per rimuovere la sostituzione e tornare al nome campo automatico.

:::note
In questo editor compaiono solo i tipi di ID fiscale rivolti al cliente. Gli identificativi del registro imprese, come il numero USt-IdNr. tedesco o il numero KVK olandese, sono campi a livello di negozio — vedi [ID fiscali del negozio](/settings/wp-admin/store-tax-ids).
:::

## Gli ID fiscali vengono copiati negli ordini {#tax-ids-are-copied-onto-orders}

Quando viene effettuato un ordine, gli ID fiscali del cliente vengono **registrati come istantanea** nell'ordine: l'ordine conserva i valori presenti in archivio al momento della vendita.

- La modifica successiva dell'ID fiscale di un cliente **non** riscrive i suoi ordini precedenti.
- Un ID fiscale inserito dal cassiere alla cassa sostituisce, per quell'ordine, il valore presente nella scheda cliente.
- La modifica di un ordine esistente cambia i suoi ID fiscali solo se l'aggiornamento li include esplicitamente; WCPOS non li copia mai di nuovo dalla scheda cliente.

Questo mantiene accurate le fatture storiche anche dopo la modifica dei dati del cliente.

## Tipi supportati {#supported-types}

IVA UE, IVA GB, IVA SA, ABN australiano, CPF / CNPJ brasiliano, GSTIN indiano, Codice Fiscale e Partita IVA italiani, NIF spagnolo, CUIT argentino, GST/HST canadese ed EIN statunitense, oltre a un **ID fiscale** generico per qualsiasi altro caso.

## Trovare i clienti tramite ID fiscale {#finding-customers-by-tax-id}

La ricerca clienti nel POS corrisponde anche ai valori degli ID fiscali, inclusi i nomi dei campi dei plugin precedenti, così il cassiere può cercare un cliente tramite numero di partita IVA, CPF o NIF.

## Sulle ricevute {#on-receipts}

Gli ID fiscali dei clienti compaiono sulle ricevute che mostrano i dettagli del cliente, con l'etichetta localizzata corretta per ogni tipo. I numeri di partita IVA sono preceduti dal prefisso del Paese (ad esempio `DE123456789`).
