:::info Funzione Pro
La schermata Report richiede [WCPOS Pro](/getting-started/pro-license).
:::

La schermata Report fornisce strumenti di reporting delle vendite e riconciliazione giornaliera. È progettata principalmente per i **report di fine giornata** per aiutare i cassieri e i manager a chiudere la giornata.

{/* Screenshot: reports-overview.png */}

:::tip Analisi Approfondite
Per analisi dettagliate su qualsiasi intervallo di date, utilizza **WP Admin > Analytics**. La schermata Report POS è ottimizzata per riepiloghi rapidi di fine giornata e riconciliazione giornaliera.
:::

## Panoramica dell'Interfaccia {#interface-overview}

### Filtri {#filters}

Nella parte superiore della schermata:

- **Filtro di stato** - Filtra per stato dell'ordine (Completato, Processing, ecc.)
- **Filtro cliente** - Filtra per cliente
- **Filtro cassiere** - Filtra per chi ha elaborato gli ordini
- **Filtro negozio** - Filtra per negozio (configurazioni multi-negozio)
- **Intervallo di date** - Seleziona il periodo di reporting (Oggi, Questa settimana, Personalizzato)

### Layout a Tre Pannelli {#three-panel-layout}

La schermata Report è divisa in tre aree:

#### 1. Grafico delle Vendite (Parte Superiore) {#1-sales-chart-top}

Un grafico a barre che mostra le vendite nel tempo:

- Suddivisione oraria per report giornalieri
- Rappresentazione visiva dei modelli di vendita
- Aiuta a identificare i tempi di picco delle vendite

#### 2. Tabella degli Ordini (Parte Inferiore Sinistra) {#2-orders-table-bottom-left}

Un elenco di ordini inclusi nel report:

- **Checkbox** - Seleziona/deseleziona ordini per inclusione
- **Stato** - Icona di stato dell'ordine
- **Numero Ordine** - ID ordine unico
- **Date Created** - Quando è stato effettuato l'ordine
- **Date Paid** - Quando è stato ricevuto il pagamento
- **Totale** - Importo dell'ordine

Clicca sull'icona delle impostazioni di visualizzazione per personalizzare le colonne.

#### 3. Riepilogo del Report (Parte Inferiore Destra) {#3-report-summary-bottom-right}

I totali calcolati del report:

- **Tipo di report** a discesa (Predefinito/Offline)
- **Informazioni sul negozio** (nome, ID)
- **Periodo del report** (timestamp di inizio/fine)
- **Cassiere** (se filtrato)

**Riepilogo Vendite:**
- Totale Ordini
- Totale Vendite Nette
- Totale Tasse Raccolte
- **Totale Vendite**
- Totale Sconti

**Metodi di Pagamento:**
- Suddivisione per tipo di pagamento (Contante, Carta, ecc.)

**Stampa** pulsante per generare un report fisico.

## Tipi di Rapporti {#report-types}

### Predefinito (Offline) {#default-offline}

Il report predefinito viene generato da dati memorizzati localmente. Questo:

- Funziona senza connessione al server
- Fornisce generazione rapida di report
- Utilizza dati già sincronizzati con il dispositivo

Questo è ideale per riepiloghi rapidi di fine giornata, in cui tutti gli ordini pertinenti sono già sincronizzati.

## Flusso di Lavoro di Fine Giornata {#end-of-day-workflow}

### Riconciliazione Giornaliera Tipica {#typical-daily-reconciliation}

1. **Imposta intervallo di date** - Seleziona "Oggi" o la data specifica
2. **Filtra per cassiere** (opzionale) - Per report di cassiere individuali
3. **Esamina gli ordini** - Controlla la tabella degli ordini per accuratezza
4. **Verifica i totali** - Confronta con il conteggio fisico del denaro e del terminale di carte
5. **Stampa report** - Genera un registro fisico

### Riconciliazione in Contante {#cash-reconciliation}

1. Filtra per mostrare solo i pagamenti in contante
2. Annotare il totale in contante dal report
3. Conta il contante fisico nel cassetto
4. Confronta i totali (dovrebbero corrispondere)
5. Documenta eventuali discrepanze

### Riconciliazione delle Carte {#card-reconciliation}

1. Filtra per mostrare solo i pagamenti con carta
2. Annotare il totale delle carte dal report
3. Confronta con il totale batch del tuo terminale di carte
4. Verifica che gli importi corrispondano prima di chiudere il batch

## Stampa Report {#printing-reports}

Clicca sul pulsante **Stampa** per generare un report stampabile. Questo include:

- Informazioni sul negozio
- Periodo del report
- Riepilogo delle vendite
- Suddivisione dei metodi di pagamento
- Dettagli degli ordini (se selezionati)
