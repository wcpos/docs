La schermata Report richiede [WCPOS Pro](/getting-started/pro-license).
:::

La schermata Report fornisce strumenti di reporting delle vendite e riconciliazione giornaliera. È principalmente progettata per **report di fine giornata** per aiutare cassieri e manager a chiudere la giornata.

{/* Screenshot: reports-overview.png */}

:::tip Analisi Complete
Per analisi dettagliate in qualsiasi intervallo di date, usa **WP Admin > Analisi**. La schermata Report del POS è ottimizzata per rapidi sommari di fine giornata e riconciliazione giornaliera.
:::

## Panoramica dell'Interfaccia

### Filtri

Nella parte superiore dello schermo:

- **Filtro stato** - Filtra per stato dell'ordine (Completato, In elaborazione, ecc.)
- **Filtro cliente** - Filtra per cliente
- **Filtro cassiere** - Filtra per chi ha elaborato gli ordini
- **Filtro negozio** - Filtra per negozio (impostazioni multi-negozio)
- **Intervallo Date** - Seleziona il periodo di reporting (Oggi, Questa Settimana, Personalizzato)

### Layout a Tre Pannelli

La schermata Report è divisa in tre aree:

#### 1. Grafico Vendite (Alto)

Un grafico a barre che mostra le vendite nel tempo:

- Dettaglio orario per report giornalieri
- Rappresentazione visiva degli andamenti delle vendite
- Aiuta a identificare i momenti di picco delle vendite

#### 2. Tabella Ordini (In Basso a Sinistra)

Un elenco degli ordini inclusi nel report:

- **Casella di spunta** - Seleziona/deseleziona ordini per l'inclusione
- **Stato** - Icona dello stato dell'ordine
- **Numero Ordine** - ID ordine univoco
- **Data Creazione** - Quando l'ordine è stato effettuato
- **Data Pagamento** - Quando è stato ricevuto il pagamento
- **Totale** - Importo dell'ordine

Fai clic sull'icona delle impostazioni di visualizzazione per personalizzare le colonne.

#### 3. Riepilogo Report (In Basso a Destra)

I totali calcolati del report:

- Dropdown **tipo di report** (Predefinito/Offline)
- **Informazioni negozio** (nome, ID)
- **Periodo report** (timestamp di inizio/fine)
- **Cassiere** (se filtrato)

**Riepilogo Vendite:**
- Totale Ordini
- Totale Vendite Nette
- Totale Tasse Raccolte
- **Totale Vendite**
- Totale Sconti

**Metodi di Pagamento:**
- Dettaglio per tipo di pagamento (Contanti, Carta, ecc.)

Pulsante **Stampa** per generare un report cartaceo.

## Tipi di Report

### Predefinito (Offline)

Il report predefinito è generato da dati memorizzati localmente. Questo:

- Funziona senza connessione al server
- Fornisce generazione rapida del report
- Utilizza dati già sincronizzati sul dispositivo

Ideale per rapidi sommari di fine giornata in cui tutti gli ordini rilevanti sono già sincronizzati.

## Flusso di Lavoro di Fine Giornata

### Riconciliazione Giornaliera Tipica

1. **Imposta intervallo date** - Seleziona "Oggi" o la data specifica
2. **Filtra per cassiere** (opzionale) - Per report individuali del cassiere
3. **Rivedi ordini** - Controlla la tabella degli ordini per l'accuratezza
4. **Verifica totali** - Confronta con il conteggio del denaro fisico e il terminale di pagamento
5. **Stampa report** - Genera un record cartaceo

### Riconciliazione Contanti

1. Filtra per mostrare solo pagamenti in contanti
2. Annota l'importo totale in contanti dal report
3. Conta i contanti fisici nel cassetto
4. Confronta i totali (dovrebbero corrispondere)
5. Documenta eventuali discrepanze

### Riconciliazione Carta

1. Filtra per mostrare solo pagamenti con carta
2. Annota l'importo totale della carta dal report
3. Confronta con il totale batch del tuo terminale di pagamento
4. Verifica che gli importi corrispondano prima di chiudere il batch

## Stampa Report

Fai clic sul pulsante **Stampa** per generare un report stampabile. Questo include:

- Informazioni negozio
- Periodo report
- Riepilogo vendite
- Dettaglio metodi di pagamento
- Dettagli ordine (se selezionati)
