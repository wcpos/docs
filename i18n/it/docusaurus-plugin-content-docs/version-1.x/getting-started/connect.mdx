:::note Solo desktop e mobile
Questa schermata viene mostrata solo nelle app desktop e mobile. Gli utenti web accedono direttamente al POS all'indirizzo `yourdomain.com/pos` ed effettuano l'accesso con le proprie credenziali WordPress.
:::

## Panoramica della schermata Connetti {#connect-screen-overview}

Quando apri l'app desktop o mobile WCPOS, viene visualizzata la schermata Connetti. Da qui puoi gestire le connessioni ai negozi e gli accessi degli utenti.

{/* Screenshot: connect-screen.png */}

## Aggiunta di un nuovo negozio {#adding-a-new-store}

1. Inserisci l'URL del tuo negozio WooCommerce nel campo di testo (ad es. `https://mystore.com`)
2. Fai clic su **Connetti**
3. Verrai reindirizzato per accedere con le tue credenziali WordPress
4. Dopo l'accesso riuscito, tornerai all'app

## Più negozi {#multiple-stores}

Puoi collegare tutti i negozi WooCommerce di cui hai bisogno. Ogni negozio appare come una scheda separata nella schermata Connetti, mostrando:

- **Nome del negozio** e favicon
- **URL del negozio**
- **Utenti connessi** per quel negozio

Questo è utile se gestisci più sedi o attività.

## Più utenti per negozio {#multiple-users-per-store}

Ogni negozio può avere più utenti connessi contemporaneamente. Questo è utile per:

- **Cambi turno** - Il nuovo cassiere può accedere prima che quello precedente esca
- **Configurazioni con più registratori di cassa** - Cassieri diversi su dispositivi diversi
- **Cambio rapido** - Passa facilmente da un account utente all'altro

### Aggiunta di un utente {#adding-a-user}

Fare clic sul <Icon name="plus" /> pulsante accanto a "Utenti connessi" per aggiungere un altro utente a quel negozio.

### Cambio utente {#switching-users}

Fare clic sul badge di un utente (ad esempio, "Brenda") per aprire il POS come quell'utente.

### Rimozione di un utente {#removing-a-user}

Fare clic sulla **×** sul badge di un utente per disconnettere quell'utente dal negozio.

## Rimozione di un negozio {#removing-a-store}

Fare clic sul pulsante rosso **×** sulla scheda del negozio per rimuoverlo dall'elenco. Questa operazione disconnette tutti gli utenti e rimuove la connessione del negozio dall'app.

## Negozio demo {#demo-store}

Nella parte inferiore dello schermo potrebbe essere visualizzato un link "Accedi al negozio demo". Questo collega a un negozio WooCommerce demo per provare le funzionalità di WCPOS senza influire sui dati del proprio negozio.

## Risoluzione dei problemi {#troubleshooting}

:::tip Prima cosa da controllare: `X-Frame-Options`
Le app desktop e mobile usano **iframe** per accesso, pagamenti e ricevute. **Qualsiasi** intestazione del server o plugin che invii `X-Frame-Options: DENY` o `SAMEORIGIN` interromperà l'accesso. Questa è la causa più comune degli errori di accesso all'app: controlla le intestazioni di risposta della pagina di accesso (strumenti di sviluppo del browser oppure `curl -I https://yourstore.com/wp-login.php`) prima di qualsiasi altra cosa.
:::

<Accordion>
  <AccordionItem question="Non riesci a connetterti al negozio?">

  - Assicurati che il plugin WCPOS sia installato e attivato sul sito WordPress
  - Verifica di usare l'URL corretto (includi `https://` — l'API REST di WooCommerce richiede SSL)
  - Prova prima ad aprire `yourdomain.com/pos` in un browser web per confermare che il plugin funzioni
  - Verifica che l'API REST di WooCommerce sia accessibile
  - Controlla che l'account utente disponga delle autorizzazioni di accesso a POS

  </AccordionItem>

  <AccordionItem question="L'accesso non riesce nell'app desktop o mobile">

  La maggior parte degli errori di accesso all'app è causata da un plugin di sicurezza o di cache che blocca l'iframe di accesso:

  - **Intestazioni `X-Frame-Options`** (impostate da un plugin di sicurezza o dal server) bloccano l'iframe di accesso — vedere la nota sopra. Disattivare temporaneamente il plugin di sicurezza, effettuare l'accesso, quindi riattivarlo (la sessione dura circa una settimana).
  - **Plugin di sicurezza** — Wordfence, Really Simple Security, WPS Hide Login, iThemes/Solid Security e Defender Pro sono responsabili comuni. Consultare l'elenco completo e le soluzioni in [Conflitti tra plugin](/support/troubleshooting/plugin-conflicts).
  - **2FA di Wordfence** — il campo del codice 2FA non viene visualizzato nell'iframe di accesso. Per ora disattivare la 2FA per gli utenti POS.
  - **URL di accesso personalizzato** (ad esempio WPS Hide Login) — l'app non riesce a trovare la pagina di accesso. Utilizzare l'URL standard `/wp-admin/`.
  - **Plugin di cache** possono continuare a servire il modulo di accesso bloccato anche dopo aver disattivato il plugin responsabile — svuotare la cache, oppure svuotare la cache dell'app / reinstallare l'app desktop.

  </AccordionItem>

  <AccordionItem question='"REST API richiede l&#39;autenticazione" o un errore del plugin di sicurezza nella schermata di connessione'>

  Un plugin (ad esempio Force Login o un plugin di autenticazione JWT) richiede l'autenticazione per tutte le richieste REST API, quindi l'app non riesce a leggere le informazioni pubbliche del sito. Ora l'app mostra il messaggio effettivo del server (ad esempio _"Solo gli utenti autenticati possono accedere alla REST API"_) invece di indicare erroneamente il tipo di sito.

  **Correzione:** configurare il plugin di sicurezza per consentire l'accesso non autenticato a `/wp-json/wcpos/` e `/wp-json/wc/v3/`, oppure disattivarlo solo per il tempo necessario a completare la prima connessione.

  </AccordionItem>

  <AccordionItem question='"Non sembra essere un sito WordPress" (app desktop)'>

  L'app desktop rileva la REST API tramite l'intestazione HTTP `Link`. Se un plugin (spesso **Image Prioritizer** o altri plugin per le prestazioni) sovraccarica o tronca quell'intestazione, il rilevamento non riesce. **Correzione:** disattivare i plugin per l'ottimizzazione delle immagini / per le prestazioni che modificano le intestazioni e riprovare.

  </AccordionItem>

  <AccordionItem question="L'app indica che è necessario un aggiornamento / si arresta in modo anomalo dopo un aggiornamento">

  Verificare che non ci sia una mancata corrispondenza di versione tra l'app e il plugin server — lo store dell'app potrebbe aver distribuito un aggiornamento dell'app mentre il plugin WCPOS deve ancora essere aggiornato (o viceversa). Assicurarsi che l'app e il plugin WCPOS siano sulla **stessa versione principale**. Le app WCPOS v1.9.0+ richiedono che il plugin WCPOS sia alla versione **v1.8.0 o superiore**.

  </AccordionItem>

  <AccordionItem question='Errore "Impossibile creare il database rapido del negozio"'>

  Questa e una race condition al primo accesso. **Chiudi completamente l'app e riprova**: di solito riesce al secondo tentativo.

  </AccordionItem>

  <AccordionItem question="Bloccato nella schermata di selezione utente (app desktop)">

  Dopo l'accesso viene visualizzato il tuo nome utente, ma non c'e un modo evidente per proseguire. **Fai clic sul tuo nome utente/nome** per accedere al POS: il nome stesso e il pulsante.

  </AccordionItem>

  <AccordionItem question="La connessione continua a non riuscire?">

  - Prova prima ad accedere a `yourdomain.com/pos` in un browser web per verificare che il plugin funzioni
  - Controlla i log degli errori del sito per eventuali problemi
  - Conferma che il tuo host non blocchi la REST API: consulta le [note specifiche per hosting](/support/performance/server#hosting-specific-notes)
  - Assicurati che il server soddisfi i [requisiti minimi](/support/performance/server#minimum-server-requirements)

  </AccordionItem>
</Accordion>
