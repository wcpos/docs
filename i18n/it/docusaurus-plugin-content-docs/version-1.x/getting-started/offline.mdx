WCPOS archivia i dati di prodotti e clienti localmente su ciascun dispositivo usando un database del browser (IndexedDB). Questo significa che alcune parti del POS funzionano senza connessione Internet, mentre altre richiedono connettività.

## Cosa funziona offline {#what-works-offline}

- **Navigazione dei prodotti** — cerca, filtra e visualizza i dettagli dei prodotti dai dati in cache
- **Navigazione dei clienti** — consulta nomi, email e indirizzi dei clienti
- **Creazione del carrello** — aggiungi articoli, modifica quantità, cambia prezzi e applica sconti POS
- **Scansione dei codici a barre** — scansiona codici a barre per trovare prodotti nel database locale
- **Visualizzazione dei report** — il tipo di report predefinito (offline) genera report dagli ordini archiviati localmente

## Cosa richiede una connessione {#what-requires-a-connection}

- **Completare il checkout** — elaborare il pagamento e creare l'ordine in WooCommerce richiede comunicazione con il server
- **Applicare codici coupon** — la convalida dei coupon avviene sul server
- **Sincronizzare i dati** — scaricare nuovi prodotti, prezzi aggiornati o nuovi clienti da WooCommerce
- **Accedere** — l'autenticazione iniziale richiede una connessione al tuo sito WordPress
- **Attivare la licenza** — i controlli della licenza Pro devono raggiungere il server licenze WCPOS
- **Elaborare rimborsi** — i rimborsi non possono essere accodati offline; sia il gateway sia il negozio devono essere raggiungibili (vedi [Rimborsi](/orders/refunds))

## Come funziona il database locale {#how-the-local-database-works}

Quando apri WCPOS per la prima volta, inizia a scaricare in background i prodotti e i clienti WooCommerce. Il processo è progressivo: più usi il POS, più completi diventano i dati locali.

Il database locale:

- **Persiste tra le sessioni** — i dati sopravvivono al riavvio del browser e del dispositivo
- **È per dispositivo** — ogni dispositivo mantiene la propria copia locale
- **Resta sincronizzato** — WCPOS controlla periodicamente le modifiche sul server e scarica gli aggiornamenti

Per maggiori dettagli tecnici, consulta il riferimento [Architettura](/reference/architecture).

## Indicatore di connettività {#connectivity-indicator}

L'intestazione del POS mostra un punto colorato che indica lo stato della connessione:

- **Verde** — connesso al server, tutte le funzioni disponibili
- **Giallo** — connessione intermittente, alcune operazioni potrebbero essere lente
- **Rosso** — offline, limitato alla navigazione dei dati in cache

## Cosa accade durante una perdita di connettività {#what-happens-during-connectivity-loss}

Se perdi la connessione Internet mentre usi il POS:

1. **Prodotti e clienti rimangono consultabili** dai dati in cache.
2. **Puoi continuare a creare carrelli** e modificare articoli.
3. **Il checkout fallirà** se tentato — il POS deve raggiungere il server WooCommerce per elaborare l'ordine.
4. **Gli ordini aperti vengono conservati** nel database locale finché la connettività non torna.

## Quando la connessione viene ripristinata {#when-connection-restores}

Quando la connessione torna:

- L'indicatore di connettività diventa verde.
- Puoi procedere al checkout e completare eventuali ordini parcheggiati.
- La sincronizzazione in background riprende, scaricando eventuali modifiche a prodotti o clienti avvenute mentre eri offline.
- Non è richiesta alcuna azione manuale — WCPOS gestisce automaticamente la riconnessione.

## Consigli per connessioni instabili {#tips-for-unreliable-connections}

- **Usa "Salva sul server" per gli ordini importanti** — invia subito l'ordine a WooCommerce, così non viene perso se il database locale del dispositivo viene cancellato.
- **Sincronizza regolarmente** — se sai che la connettività è intermittente, sincronizza il catalogo prodotti quando hai una buona connessione, così i dati locali restano aggiornati.
- **Considera le app native** — le app desktop e mobili (quando disponibili) possono offrire una migliore resilienza offline rispetto alla versione browser.
