O WCPOS inclui um sistema completo de templates para recibos, faturas, recibos de presente, romaneios, tickets de cozinha e muito mais, tudo gerenciado em **WP Admin → POS → Templates**.

:::tip Quer apenas alterar seu recibo?
Comece em **[Personalizar seu recibo](/templates/customise)**. A página mostra as três formas mais fáceis de fazer isso: escolher outro template incluído, pedir para a IA ajustar um template ou editar manualmente.
:::

## Mecanismos de template

O WCPOS oferece suporte a três mecanismos de template. O mecanismo é escolhido quando você cria um template ou clica em **Usar template** em um cartão da galeria, e **não pode ser alterado depois**. Crie um novo template se quiser trocar de mecanismo.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML sem lógica (recomendado)">
    Placeholders no estilo Mustache <code>{"{{variable}}"}</code> dentro de HTML simples. A renderização acontece no cliente, então os recibos **funcionam offline**. Use para impressão pelo navegador e recibos em PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML térmico">
    Templates XML que produzem uma prévia na tela e **comandos ESC/POS** para impressoras térmicas de recibo. O mesmo template gera duas saídas. Use com Epson, Star e outras impressoras térmicas.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP legado">
    Templates PHP no servidor usando funções do WooCommerce. Mantidos para compatibilidade com substituições existentes em <code>yourtheme/woocommerce-pos/receipt.php</code>. Exigem conexão com o servidor para renderizar.
  </FeatureCard>
</FeatureGrid>

## Galeria de templates

A galeria vem com templates prontos que você pode usar como estão ou personalizar. Filtre por **categoria** (Recibo, Fatura, Recibo de presente, Ticket de cozinha, Cotação / Pedido de compra), **formato** (HTML / ESC/POS) e **direção** (LTR / RTL); a prévia usa os dados reais da sua loja. O rodapé de cada cartão tem um link **Prévia** e o botão principal **Usar template**. Ao clicar, o WCPOS cria uma cópia editável em *Seus templates*.

| Template | Mecanismo | Descrição |
|----------|-----------|-----------|
| Recibo padrão | HTML | Recibo padrão com logotipo, identidade da loja, itens, totais e pagamento |
| Recibo padrão (RTL) | HTML | Versão da direita para a esquerda para árabe, hebraico, persa e urdu |
| Minimalista / Moderno | HTML | Os mesmos itens essenciais do padrão, ocupando menos espaço vertical |
| Recibo detalhado | HTML | Fatura fiscal completa com coluna SKU, preço unitário, impostos por alíquota e endereços |
| Recibo de presente | HTML | Somente itens, preços ocultos. Inclui mensagem de presente e política de devolução |
| Fatura | HTML | Fatura A4/Letter de página inteira com painel opcional "Como pagar" para pedidos não pagos |
| Romaneio | HTML | Documento de depósito: itens + quantidades, endereço de entrega, sem preços |
| Cotação / Estimativa | HTML | Documento pré-venda com preços e termos, sem seção de pagamento |
| Recibo estreito | HTML | Recibo monoespaçado para papel estreito ou impressoras térmicas compatíveis com HTML |
| Recibo térmico simples (58 mm) | Térmico | Layout térmico limpo de 58 mm |

### Templates de documentos de terceiros

Se o **PDF Invoices & Packing Slips for WooCommerce** da WP Overnight estiver ativo, o WCPOS também mostra **Fatura (WP Overnight)** e **Romaneio (WP Overnight)** na lista de templates de recibo. Esses templates HTML renderizados no servidor delegam para a API de documentos da WP Overnight, reutilizando numeração, layout, identidade visual, campos legais/fiscais e personalizações configuradas nesse plugin.

Eles ficam disponíveis somente enquanto o plugin WP Overnight estiver ativo e exigem conexão com o servidor para renderizar. Use os templates HTML ou térmicos incluídos no WCPOS quando precisar de impressão offline.

### Exibição adaptável de impostos

A maioria dos templates incluídos (Padrão, Padrão RTL, Minimalista/Moderno, Estreito, Fatura, Cotação e variantes térmicas simples) **se adapta automaticamente à configuração de exibição de impostos do WooCommerce**:

- **Lojas com impostos inclusos** (UE/Reino Unido/AU) veem preços brutos com uma linha "Impostos incluídos".
- **Lojas com impostos excluídos** (EUA/CA) veem preços líquidos com o imposto adicionado em uma linha separada.
- O total geral sempre mostra o valor bruto realmente cobrado.

A família **Detalhado** é uma fatura fiscal formal e sempre discrimina impostos com linhas sem imposto e uma divisão explícita, independentemente da configuração da loja.

## Como funciona

1. **Adicione um template**: clique em **Usar template** em um cartão da galeria ou crie um do zero.
2. **Personalize** usando o editor no app com prévia ao vivo, ou cole em ChatGPT / Claude e peça as alterações desejadas.
3. **Ative**: ligue o botão **Ativo** em *Seus templates*. Todo template de recibo ativo aparece no menu suspenso da tela de recibo no caixa.
4. **Imprima ou exiba**: os recibos abrem depois do checkout com opções para imprimir, enviar por e-mail ou ver na tela.

Os templates são renderizados usando um [payload de dados do recibo](/templates/receipt-data) padronizado, com seções para informações da loja, linhas de itens, totais, impostos, pagamentos, reembolsos e mais. Todos os campos monetários incluem variantes `_display` pré-formatadas, por exemplo `$29.99` em vez de `29.99`.

## Recursos Pro

Com o [WCPOS Pro](/getting-started/pro-license), cada loja pode ter suas próprias atribuições de templates, identidade visual (logotipo, endereço, contatos) e ordem de templates. Isso é útil para operações com várias unidades em que cada loja imprime seu próprio cabeçalho.
