O WCPOS inclui um sistema completo de modelos para recibos, faturas, recibos de presente, romaneios de envio, comandas de cozinha e mais — todos gerenciados em **WP Admin → POS → Modelos**.

:::tip Quer apenas alterar seu recibo?
Comece em **[Personalize seu recibo](/templates/customise)**. Ele apresenta as três maneiras mais fáceis de fazer isso — escolher um modelo incluso diferente, pedir à IA para ajustá-lo ou editar manualmente.
:::

## Mecanismos de modelo {#template-engines}

O WCPOS suporta três mecanismos de modelo. O mecanismo é escolhido ao criar um modelo ou ao clicar em **Usar modelo** em um cartão da galeria, e **não pode ser alterado posteriormente** — inicie um novo modelo se quiser trocar de mecanismo.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML sem lógica (recomendado)">
    Estilo Mustache <code>{"{{variable}}"}</code> placeholders dentro de HTML simples. Renderiza no lado do cliente, então os recibos **funcionam offline**. Use para impressão no navegador e recibos em PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML Térmico">
    Modelos XML que produzem tanto uma pré-visualização na tela quanto **comandos ESC/POS** para impressoras térmicas de recibos. Mesmo modelo, duas saídas. Use para impressoras térmicas Epson, Star e outras.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP Legado">
    Modelos PHP renderizados no servidor usando funções do WooCommerce. Mantidos para compatibilidade com <code>yourtheme/woocommerce-pos/receipt.php</code> substituições existentes. Requer conexão com o servidor para renderizar.
  </FeatureCard>
</FeatureGrid>

## Galeria de modelos {#template-gallery}

A galeria inclui modelos prontos que podem ser usados como estão ou personalizados. Filtre por **categoria** (Recibo, Nota Fiscal, Recibo de Presente, Comanda de Cozinha, Orçamento / Ordem de Compra), **formato** (HTML / ESC/POS) e **direção** (LTR / RTL); visualize com os dados reais da sua loja. O rodapé de cada cartão tem um link de **Pré-visualização** e um botão principal **Usar Modelo** — clicar em Usar Modelo cria uma cópia editável em *Seus Modelos*.

| Modelo | Motor | Descrição |
|----------|--------|-------------|
| Recibo Padrão | HTML | Recibo padrão — logo, identidade da loja, itens, totais, pagamento |
| Recibo Padrão (RTL) | HTML | Versão da direita para a esquerda para árabe, hebraico, persa e urdu |
| Minimalista / Moderno | HTML | Mesmas informações essenciais do Padrão, em menos espaço vertical |
| Recibo Detalhado | HTML | Nota fiscal completa — coluna de SKU, preço unitário, discriminação de impostos por alíquota, endereços |
| Recibo de Presente | HTML | Somente itens, sem preços. Inclui mensagem de presente e política de devolução |
| Fatura | HTML | Fatura em página inteira A4/Carta com painel opcional "Como pagar" para pedidos não pagos |
| Romaneio | HTML | Documento para o depósito — itens + quantidades, endereço de entrega, sem preços |
| Orçamento / Estimativa | HTML | Documento pré-venda com preços e condições — sem seção de pagamento |
| Recibo Estreito | HTML | Recibo monoespaçado para papel estreito ou impressoras térmicas compatíveis com HTML |
| Recibo Térmico Simples (58mm) | Térmico | Layout térmico limpo de 58mm |
| Recibo Térmico Simples (80mm) | Térmico | Layout térmico limpo de 80mm — o mais comum |

### Modelos de documentos de terceiros {#third-party-document-templates}

Se o **PDF Invoices & Packing Slips for WooCommerce** da WP Overnight estiver ativo, o WCPOS também exibe **Invoice (WP Overnight)** e **Packing Slip (WP Overnight)** na lista de modelos de recibo. Esses modelos HTML renderizados no servidor delegam para a API de documentos da WP Overnight, reutilizando a numeração de faturas e guias de remessa, o layout, a identidade visual, os campos legais/fiscais e as personalizações de modelo configuradas nesse plugin.

Eles estão disponíveis apenas enquanto o plugin WP Overnight estiver ativo e requerem conexão com o servidor para renderizar. Use os modelos HTML ou térmicos incluídos no WCPOS quando for necessário imprimir offline.

### Exibição adaptativa de impostos {#adaptive-tax-display}

A maioria dos modelos incluídos (Standard, Standard RTL, Minimal/Modern, Narrow, Invoice, Quote, variantes Thermal Simple) **se adapta automaticamente à configuração de exibição de impostos do WooCommerce**:

- **Lojas com impostos inclusos** (UE/Reino Unido/AU) exibem preços brutos com uma linha "Impostos inclusos".
- **Lojas com impostos exclusos** (EUA/CA) exibem preços líquidos com o imposto adicionado como uma linha separada.
- O total geral sempre exibe o valor bruto efetivamente cobrado.

A família **Detalhado** é uma fatura fiscal formal e sempre discrimina os impostos com linhas de valores líquidos e um detalhamento explícito, independentemente da configuração da loja.

## Como funciona {#how-it-works}

1. **Adicionar um modelo** — clique em **Usar Modelo** em um card da galeria, ou crie o seu do zero.
2. **Personalizar** usando o editor integrado com pré-visualização ao vivo, ou cole no ChatGPT / Claude e solicite as alterações desejadas.
3. **Ativar** — alterne o botão **Ativo** em *Seus Modelos*. Cada modelo de recibo ativo aparece no menu suspenso da tela de recibos no caixa.
4. **Imprimir ou exibir** — os recibos são abertos após o checkout com opções para imprimir, enviar por e-mail ou visualizar na tela.

Os modelos são renderizados com base em um [payload de dados de recibo](/templates/receipt-data) padronizado, com seções para informações da loja, itens de linha, totais, impostos, pagamentos, reembolsos e mais. Todos os campos de moeda incluem variantes `_display` pré-formatadas (por exemplo, `$29.99` em vez de `29.99`).

## Recursos Pro {#pro-features}

Com o [WCPOS Pro](/getting-started/pro-license), cada loja pode ter suas próprias atribuições de modelo, identidade visual (logotipo, endereço, dados de contato) e ordenação de modelos — útil para operações com múltiplas localidades onde cada estabelecimento imprime seu próprio cabeçalho.
