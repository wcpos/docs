:::warning Importante
No carregamento inicial, o POS obtém as configurações padrão de impostos do WooCommerce. Em geral, os usuários devem manter as mesmas configurações do WooCommerce, a menos que tenham uma necessidade específica. Ter configurações de impostos diferentes das do servidor quase certamente fará com que o POS e o WooCommerce calculem valores de impostos diferentes, o que pode causar discrepâncias e problemas com a sincronização de pedidos.
:::

## Ver Todas as Alíquotas de Impostos {#view-all-tax-rates}

Clique no botão **Ver todas as alíquotas de impostos** para visualizar todas as alíquotas de impostos que foram baixadas para o POS. São exibidos:

- Nomes das alíquotas de impostos
- Percentuais das alíquotas de impostos
- Regiões/localizações aplicáveis

Isso é útil para verificar se as alíquotas de imposto corretas estão carregadas e para depurar problemas de cálculo de impostos.

## Configurações de Cálculo de Impostos {#tax-calculation-settings}

### Preços Inseridos com Imposto {#prices-entered-with-tax}

Determina se os preços dos produtos incluem imposto:

| Opção | Descrição |
|--------|-------------|
| **Sim** | Os preços dos produtos incluem imposto. O POS calcula o imposto a partir do preço total |
| **Não** | Os preços dos produtos não incluem imposto. O POS adiciona o imposto sobre o preço inserido |

### Calcular Imposto com Base em {#calculate-tax-based-on}

Determina o endereço utilizado para o cálculo de impostos:

| Opção | Descrição |
|--------|-------------|
| **Endereço de entrega do cliente** | Alíquotas de imposto baseadas no destino de envio do pedido |
| **Endereço de cobrança do cliente** | Alíquotas de imposto baseadas no endereço de cobrança do cliente |
| **Endereço base da loja** | Alíquotas de imposto baseadas na localização física da sua loja |

### Classe de Imposto de Frete {#shipping-tax-class}

Determina como os custos de frete são tributados:

| Opção | Descrição |
|--------|-------------|
| **Classe de imposto do frete baseada nos itens do carrinho** | O imposto do frete é calculado com base na classe de imposto dos itens no carrinho |
| **Padrão** | O frete é tributado pela alíquota padrão |
| **Alíquota reduzida** | O frete é tributado por uma alíquota reduzida |
| **Alíquota zero** | Nenhum imposto é aplicado aos custos de frete |

### Arredondamento {#rounding}

| Opção | Descrição |
|--------|-------------|
| **Arredondar imposto no nível do subtotal** | O imposto é calculado sobre o subtotal inteiro e então arredondado, em vez de arredondar o imposto de cada item de linha individualmente |

## Configurações de Exibição {#display-settings}

### Exibir Preços na Loja {#display-prices-in-the-shop}

| Opção | Descrição |
|--------|-------------|
| **Com imposto** | Os preços exibidos incluem os valores de imposto |
| **Sem imposto** | Os preços exibidos não incluem imposto |

### Exibir Preços no Carrinho e Finalização de Compra {#display-prices-during-cart-and-checkout}

| Opção | Descrição |
|--------|-------------|
| **Incluindo impostos** | Os preços no carrinho e no checkout incluem impostos |
| **Excluindo impostos** | Os preços no carrinho e no checkout não incluem impostos |

### Sufixo de Exibição de Preço {#price-display-suffix}

Texto exibido após os preços para indicar a situação tributária (por exemplo, "com imposto" ou "sem imposto").

### Exibição dos Totais de Impostos {#display-tax-totals}

| Opção | Descrição |
|--------|-------------|
| **Como total único** | Todos os impostos são combinados em uma única linha de total |
| **Detalhado** | Cada alíquota de imposto é exibida como um item separado |

## Restaurar Configurações do Servidor {#restore-server-settings}

Clique para buscar e aplicar as configurações de impostos atuais do WooCommerce no seu servidor.

:::tip Como e quando usar
O WooCommerce é a fonte de verdade para as configurações de impostos — o POS deve espelhá-lo. **Restaurar Configurações do Servidor** é a maneira segura de obter a configuração mais recente da sua loja, e é a ação correta sempre que você alterar as configurações de impostos em `WP Admin > WooCommerce > Configurações > Tax` e quiser que o POS as aplique.

O único momento para pensar duas vezes: se você **deliberadamente** substituiu uma configuração de imposto no POS para diferir do servidor (uma configuração avançada), Restaurar Configurações do Servidor sobrescreverá essa substituição. Se você não tem certeza se fez isso, quase certamente não fez.
:::

## Problemas Comuns {#common-issues}

### POS e WooCommerce Exibindo Totais Diferentes {#pos-and-woocommerce-showing-different-totals}

O problema de imposto mais frequente é uma divergência entre as configurações de imposto do POS e as configurações de imposto do WooCommerce. Quando essas configurações divergem, o mesmo pedido calculará totais diferentes dependendo de onde é realizado. Sempre mantenha as configurações do POS sincronizadas — use **Restaurar Configurações do Servidor** após alterar qualquer coisa em `WP Admin > WooCommerce > Configurações > Tax`.

### Confusão com Preços com Impostos Inclusos {#tax-inclusive-pricing-confusion}

Se "Preços Inseridos com Imposto" estiver definido como **Sim** mas "Exibir Preços na Loja" estiver definido como **Sem impostos**, o preço exibido será menor que o preço inserido. Esse é o comportamento correto — o POS está removendo o componente de imposto para exibição — mas pode surpreender os operadores de caixa se estiverem esperando ver o preço da etiqueta.

### Endereço do Cliente e Cálculo de Impostos {#customer-address-and-tax-calculation}

Quando "Calcular Imposto Com Base Em" estiver definido como **Endereço de entrega do cliente** ou **Endereço de cobrança do cliente**, o POS precisa de um endereço válido para consultar a alíquota correta. Se um cliente avulso não tiver endereço cadastrado, o WCPOS usará o **endereço base da loja** para o cálculo. Esse geralmente é o comportamento correto para vendas presenciais, mas se você notar alíquotas inesperadas para clientes específicos, verifique os dados de endereço deles.

## Solução de Problemas {#troubleshooting}

### Os valores de imposto estão incorretos {#tax-amounts-are-incorrect}

1. Acesse **Configurações de Impostos** no POS e clique em **Restaurar Configurações do Servidor** para obter a configuração mais recente do WooCommerce.
2. Clique em **Ver todas as alíquotas de imposto** e confirme se as alíquotas esperadas aparecem. Se estiverem faltando alíquotas, verifique se estão configuradas corretamente em `WP Admin > WooCommerce > Configurações > Tax > Standard rates`.
3. Verifique a classe de imposto do produto — produtos atribuídos a uma classe de imposto "Taxa zero" não terão imposto aplicado independentemente das configurações.

### O imposto está aparecendo como zero {#tax-is-showing-as-zero}

- Verifique se os impostos estão habilitados no WooCommerce (`WP Admin > WooCommerce > Configurações > General > Enable tax rates and calculations`).
- Use **Ver todas as alíquotas de imposto** no POS para confirmar se as alíquotas foram baixadas. Se a lista estiver vazia, tente uma sincronização completa.
- Confirme se o status de imposto do produto está definido como "Tributável" (não "Nenhum" ou "Somente frete").

### Discrepâncias de arredondamento {#rounding-discrepancies}

Pequenas diferenças de arredondamento (um ou dois centavos) entre o POS e o WooCommerce podem ocorrer quando "Arredondar imposto no nível do subtotal" está habilitado em um sistema, mas não no outro. Certifique-se de que essa configuração seja igual em ambos os locais.
