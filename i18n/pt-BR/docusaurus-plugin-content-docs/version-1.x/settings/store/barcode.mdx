## Como Funciona a Detecção de Código de Barras

A maioria dos leitores de código de barras se comporta como um teclado conectado ao seu dispositivo. Quando você digitaliza um código de barras, o POS detecta que os caracteres foram inseridos mais rapidamente do que a digitação normal. Ele usa essas "teclas pressionadas rapidamente" para identificar a entrada como uma digitalização de código de barras.

## Configurações

| Configuração | Descrição |
|---------|-------------|
| **Tempo médio de entrada** | Quão rápida a entrada deve ser (em milissegundos) para ser considerada um código de barras. Valores menores = entrada mais rápida necessária |
| **Comprimento mínimo** | Qual deve ser o comprimento da sequência contínua de caracteres para ser tratada como um código de barras |
| **Remoção de prefixo** | Caracteres a serem removidos do início dos códigos de barras digitalizados |
| **Remoção de sufixo** | Caracteres a serem removidos do final dos códigos de barras digitalizados |

## Remoção de Prefixo/Sufixo

Se seu leitor adiciona caracteres extras (como um prefixo ou sufixo), você pode removê-los automaticamente para que reste apenas o código de barras principal.

**Exemplo:** Se seu leitor adiciona "A" antes de cada código de barras e "Z" depois, configure:
- Remoção de prefixo: `A`
- Remoção de sufixo: `Z`

## Solução de Problemas

### Código de barras não detectado

- **Leitor muito lento**: Tente aumentar o valor de "Tempo médio de entrada"
- **Código de barras muito curto**: Tente diminuir o valor de "Comprimento mínimo"
- **Caracteres extras**: Verifique se o leitor está adicionando prefixos/sufixos

### Digitação normal detectada como código de barras

- **Digitação muito rápida**: Tente diminuir o valor de "Tempo médio de entrada"
- **Entrada muito longa**: Tente aumentar o valor de "Comprimento mínimo"
