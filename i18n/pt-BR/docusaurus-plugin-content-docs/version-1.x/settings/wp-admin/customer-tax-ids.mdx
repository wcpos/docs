O WCPOS mantém um campo de identificação fiscal nos seus clientes — número de CNPJ, CPF, VAT, ABN, GSTIN, NIF, entre outros — para vendas B2B em que o identificador fiscal do comprador precisa constar na nota fiscal. Quando um pedido é realizado para um cliente, o WCPOS copia a identificação fiscal dele para o pedido, para que possa ser impressa no recibo.

A seção **Identificações Fiscais** em **WP Admin > POS > Configurações > Geral** controla como esse campo é detectado e armazenado.

## Plugins e nomes de campos {#plugins-and-field-names}

Muitos plugins do WooCommerce já adicionam um campo de VAT ou número fiscal aos clientes, e todos usam nomes de campo (meta keys) diferentes no banco de dados. Para que os dados existentes dos seus clientes continuem funcionando — sem necessidade de migração — o WCPOS detecta o que está instalado e grava no campo correto automaticamente.

O WCPOS resolve o nome do campo na seguinte ordem:

1. **Um plugin reconhecido está ativo** — o WCPOS grava no campo desse plugin. Os plugins detectados incluem:
   - WooCommerce EU VAT Number
   - Aelia EU VAT Assistant (incluindo o status de verificação)
   - EU VAT for WooCommerce (WPFactory)
   - WooCommerce Germanized / Germanized Pro
   - Brazilian Market on WooCommerce / Extra Checkout Fields for Brazil
   - NIF/CIF Spain
2. **Nenhum plugin reconhecido** — o WCPOS escaneia pedidos recentes em busca do campo de VAT/CPF/etc. já em uso e grava nesse campo.
3. **Nenhum campo encontrado** — o WCPOS utiliza padrões sensatos (`_billing_vat_number` para IDs do tipo VAT, `_billing_cpf` / `_billing_cnpj` para o Brasil, `_billing_gstin` para a Índia, e assim por diante).

### Escolhendo o nome do campo manualmente {#choosing-the-field-name-yourself}

Para substituir o nome do campo detectado, acesse **WP Admin > POS > Configurações > Geral**, abra a seção **IDs Fiscais** e expanda **Avançado: usar um nome de campo diferente**.

Um aviso exibido ali mostra qual plugin foi detectado automaticamente (ou "nenhum plugin detectado", com os padrões integrados em uso). O campo de substituição possui autocompletar que sugere as meta keys comumente usadas por outros plugins. Limpe o campo para remover a substituição e voltar ao nome de campo automático.

:::note
Apenas os tipos de ID fiscal voltados ao cliente aparecem neste editor. Identificadores de registro empresarial, como o USt-IdNr. alemão ou o número KVK holandês, são campos no nível da loja — consulte [IDs Fiscais da Loja](/settings/wp-admin/store-tax-ids).
:::

## Os IDs fiscais são copiados para os pedidos {#tax-ids-are-copied-onto-orders}

Quando um pedido é realizado, os IDs fiscais do cliente são **registrados como snapshot** no pedido — o pedido mantém os valores que estavam cadastrados no momento da venda.

- Editar o ID fiscal de um cliente posteriormente **não** altera os pedidos anteriores.
- Um ID fiscal informado pelo operador de caixa no checkout substitui o valor do cadastro do cliente apenas para aquele pedido.
- Editar um pedido existente altera seus IDs fiscais somente se a atualização os incluir explicitamente; o WCPOS nunca recopia os dados do cadastro do cliente.

Isso mantém as notas fiscais históricas precisas mesmo após a alteração dos dados do cliente.

## Tipos suportados {#supported-types}

EU VAT, GB VAT, SA VAT, ABN australiano, CPF / CNPJ brasileiro, GSTIN indiano, Codice Fiscale e Partita IVA italianos, NIF espanhol, CUIT argentino, GST/HST canadense e EIN americano — além de um **ID Fiscal** genérico para qualquer outro caso.

## Busca de clientes por ID fiscal {#finding-customers-by-tax-id}

A busca de clientes no POS também localiza valores de ID fiscal, incluindo os nomes de campo antigos do plugin, permitindo que o operador de caixa encontre um cliente pelo número de VAT, CPF ou NIF.

## Nos recibos {#on-receipts}

Os IDs fiscais do cliente aparecem nos recibos que exibem os dados do cliente, com o rótulo localizado correto para cada tipo. Os números de VAT são prefixados pelo país (por exemplo, `DE123456789`).
