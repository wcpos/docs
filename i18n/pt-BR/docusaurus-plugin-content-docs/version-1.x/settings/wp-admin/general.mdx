## Ativar Produtos Apenas para POS {#enable-pos-only-products}

Esta configuração adiciona opções de visibilidade aos seus produtos, permitindo que você controle se eles aparecem na sua loja online, no POS ou em ambos.

:::warning Impacto na Performance
Ative esta configuração apenas se precisar ocultar produtos de sua loja online ou do POS. Quando ativada, ela adiciona uma pesquisa extra no banco de dados para cada solicitação de produto.
:::

Para informações detalhadas sobre como usar este recurso, incluindo como definir a visibilidade em produtos individuais e edição em massa, consulte [Produtos Apenas para POS](/products/pos-only-products).

## Ativar Quantidades Decimais {#enable-decimal-quantities}

Esta configuração permite que clientes e funcionários insiram quantidades decimais ao adicionar produtos ao carrinho.

**Exemplos de quantidades decimais:**
- 0.25 (um quarto de unidade)
- 1.5 (uma unidade e meia)
- 2.75 (duas unidades e três quartos)

**Útil para:**
- Produtos vendidos por peso (por exemplo, 0,5kg de grãos de café)
- Serviços fracionários (por exemplo, 1,5 horas de consulta)
- Produtos a granel que podem ser vendidos em quantidades parciais

## Restaurar Estoque ao Excluir Pedido {#restore-stock-on-order-delete}

Quando ativada, o WCPOS restaura automaticamente o estoque do produto quando um pedido do POS é movido para a lixeira ou excluído — assim, cancelar uma venda equivocada devolve os itens ao inventário. Esta configuração está **ativada por padrão**.

Desative-a se você reconcilia o estoque de outra forma e não deseja que pedidos do POS excluídos ajustem o inventário.

## Gerar Nome de Usuário Automaticamente a partir do E-mail do Cliente {#automatically-generate-username-from-customer-email}

Quando ativada, o WCPOS criará automaticamente nomes de usuário para novos clientes com base em seus endereços de e-mail durante o checkout. Isso agiliza o processo de criação de clientes e garante um formato consistente de nome de usuário.

## Cliente Padrão do POS {#default-pos-customer}

Selecione o cliente padrão que será atribuído a novos pedidos do POS. Opções comuns incluem:

- **Convidado**: Pedidos não estão associados a nenhuma conta de usuário
- **Caixa**: Pedidos estão associados à conta do caixa logado
- **Cliente Presencial**: Pedidos estão associados a uma conta de cliente específica (você precisará criar esta conta primeiro)

## Usar Conta de Caixa {#use-cashier-account}

Quando ativada, a conta de usuário do caixa será usada como o cliente padrão para pedidos do POS em vez do Cliente Padrão do POS selecionado.

## Campo de Código de Barras {#barcode-field}

Esta configuração determina qual campo de produto o WCPOS irá pesquisar ao escanear códigos de barras ou usar a funcionalidade de busca por código de barras.

<a id="default-option"></a>

:::note Chave meta padrão
Use **`_global_unique_id`** — o campo Global Unique ID do WooCommerce, introduzido no WooCommerce 8.2 (Julho de 2024). Ele foi projetado para armazenar códigos GTIN, UPC, EAN ou ISBN e é a opção recomendada para novas instalações. Se você não estiver usando o campo Global Unique ID, **`_sku`** (o campo SKU / Stock Keeping Unit do produto) é uma boa alternativa.
:::

<a id="other-common-options"></a>
<a id="third-party-plugin-meta-keys"></a>

### Plugins comuns → chave meta {#common-plugins-meta-key}

Se você estiver usando um plugin de gestão de códigos de barras, talvez precise usar a chave meta específica dele:

| Plugin / tipo de campo | Chave meta |
|---|---|
| WooCommerce Global Unique ID *(padrão, recomendado)* | `_global_unique_id` |
| Product SKU *(alternativa)* | `_sku` |
| Plugins EAN (European Article Number) | `_ean`, `_ean13`, `_ean8` |
| Plugins UPC (Universal Product Code) | `_upc`, `_upc_code` |
| Plugins GTIN (Global Trade Item Number) | `_gtin`, `_gtin13`, `_gtin14` |
| Campos de código de barras genéricos | `_barcode`, `_product_barcode` |
| WooCommerce Product Barcodes | `_barcode_text` |
| ATUM Inventory | `_atum_barcode` |

<a id="custom-meta-keys"></a>

:::tip Chaves meta personalizadas
Você pode usar **qualquer** chave meta personalizada que armazene dados de código de barras. Basta inserir o nome da chave meta (incluindo o prefixo de sublinhado, se aplicável) neste campo.
:::

### Encontrando a chave meta do seu plugin {#finding-your-plugins-meta-key}

Se o seu plugin não estiver listado acima, encontre a chave meta correta da seguinte forma:

<Steps>
  <StepCard number="1" title="Adicione um código de barras a um produto">

Edite um produto no WP Admin e adicione um código de barras usando seu plugin.

  </StepCard>
  <StepCard number="2" title="Verifique a documentação do plugin">

Procure na documentação do plugin pelo nome da chave meta que ele utiliza.

  </StepCard>
  <StepCard number="3" title="Ou inspecione o banco de dados" isLast>

Inspecione a tabela `wp_postmeta` do banco de dados para ver qual chave meta foi criada para o produto.

  </StepCard>
</Steps>

:::warning Limpe e atualize após alterar esta configuração
Após alterar a configuração do campo de código de barras, você **deve** limpar e atualizar seus dados de produto no POS para que a alteração tenha efeito. O POS armazena em cache os dados dos produtos localmente, e alterar a configuração do campo de código de barras não atualiza automaticamente os dados em cache.

**Como limpar e atualizar:**

1. Vá para a tela de Produtos no seu POS
2. **Pressione longamente** o ícone de sincronização (↻) na parte inferior da tela
3. Selecione **"Limpar e Atualizar"** no menu de contexto
4. Aguarde os produtos serem recarregados com os novos dados do campo de código de barras

Em seguida, teste a leitura de código de barras para confirmar que está funcionando corretamente com o campo escolhido.
:::

## Privacidade {#privacy}

A alternância **Permitir dados de uso anônimos** controla se o WCPOS coleta análises de uso anônimas para ajudar a melhorar o produto. É uma adesão/recusa genuína — nenhum dado de cliente, preços ou endereço da loja é coletado. Use o link **Saiba mais** ao lado da configuração para ver exatamente o que é e o que não é coletado.

## IDs Fiscais da Loja {#store-tax-ids}

Adicione os identificadores fiscais da empresa — número de VAT, ABN, GSTIN, EIN ou números de registro de empresas locais — que o WCPOS imprime nos recibos e faturas do POS. O WooCommerce não tem um campo nativo para esses dados, e muitas regiões precisam de mais de um.

Consulte [IDs Fiscais da Loja](/settings/wp-admin/store-tax-ids) para o guia completo.

## IDs Fiscais do Cliente {#customer-tax-ids}

O WCPOS mantém um campo de ID fiscal em seus clientes e o copia para os pedidos deles na finalização da compra. Como diferentes plugins armazenam esse campo com nomes distintos, o WCPOS detecta qual plugin você tem instalado e grava no campo correto automaticamente. A seção **Tax IDs** aqui também permite que você substitua o nome do campo, se necessário.

Consulte [IDs Fiscais do Cliente](/settings/wp-admin/customer-tax-ids) para o guia completo.
