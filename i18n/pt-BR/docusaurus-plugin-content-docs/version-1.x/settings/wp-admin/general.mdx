## Ativar Produtos Apenas para POS

Essa configuração adiciona opções de visibilidade aos seus produtos, permitindo que você controle se eles aparecem na sua loja online, no POS ou em ambos.

:::warning Impacto na Performance
Ative esta configuração apenas se precisar ocultar produtos da sua loja online ou do POS. Quando ativado, ela adiciona uma pesquisa extra no banco de dados para cada solicitação de produto.
:::

Para informações detalhadas sobre como usar este recurso, incluindo como definir a visibilidade em produtos individuais e edição em massa, consulte [Produtos Apenas para POS](/products/pos-only-products).

## Ativar Quantidades Decimais

Essa configuração permite que clientes e funcionários insiram quantidades decimais ao adicionar produtos ao carrinho.

**Exemplos de quantidades decimais:**
- 0.25 (um quarto de unidade)
- 1.5 (uma unidade e meia)
- 2.75 (duas unidades e três quartos)

**Útil para:**
- Produtos vendidos por peso (por exemplo, 0.5kg de grãos de café)
- Serviços fracionários (por exemplo, 1.5 horas de consulta)
- Produtos a granel que podem ser vendidos em quantidades parciais

## Gerar Nome de Usuário Automaticamente a partir do E-mail do Cliente

Quando ativado, o WCPOS criará automaticamente nomes de usuário para novos clientes com base em seus endereços de e-mail durante o checkout. Isso agiliza o processo de criação de clientes e garante um formato consistente de nome de usuário.

## Cliente Padrão do POS

Selecione o cliente padrão que será atribuído a novos pedidos do POS. Opções comuns incluem:

- **Convidado**: Pedidos não estão associados a nenhuma conta de usuário
- **Caixa**: Pedidos estão associados à conta do caixa logado
- **Cliente Presencial**: Pedidos estão associados a uma conta de cliente específica (você precisará criar esta conta primeiro)

## Usar Conta de Caixa

Quando ativado, a conta de usuário do caixa será usada como o cliente padrão para pedidos do POS em vez do Cliente Padrão do POS selecionado.

## Campo de Código de Barras

Essa configuração determina qual campo de produto o WCPOS irá pesquisar ao escanear códigos de barras ou usar a funcionalidade de busca por código de barras.

:::warning Importante: Limpeza e Atualização Necessárias
Após alterar a configuração do campo de código de barras, você **deve** limpar e atualizar seus dados de produto no POS para que a alteração tenha efeito.

**Como Limpar e Atualizar:**
1. Vá para a tela de Produtos no seu POS
2. **Pressione longamente** o ícone de sincronização (↻) na parte inferior da tela
3. Selecione **"Limpar e Atualizar"** no menu de contexto
4. Aguarde os produtos serem recarregados com os novos dados do campo de código de barras

Isso é necessário porque o POS armazena em cache os dados dos produtos localmente e mudar a configuração do campo de código de barras não atualiza automaticamente os dados em cache.
:::

### Opção Padrão

**`_global_unique_id`** - Este é o campo de ID Único Global do WooCommerce, introduzido no WooCommerce 8.2 (Julho de 2024). É projetado para armazenar códigos GTIN, UPC, EAN ou ISBN e é a opção recomendada para novas instalações.

### Outras Opções Comuns

**`_sku`** - O campo SKU do produto (Stock Keeping Unit). Esta é uma boa opção alternativa se você não estiver usando o campo de ID Único Global.

### Chaves Meta de Plugins de Terceiros

Se você estiver usando um plugin de gerenciamento de códigos de barras, talvez precise usar a chave meta específica deles:

| Tipo de Plugin      | Chaves Meta Comuns        | Descrição                                      |
|---------------------|---------------------------|------------------------------------------------|
| Plugins EAN         | `_ean`, `_ean13`, `_ean8` | Códigos de Número de Artigo Europeu           |
| Plugins UPC         | `_upc`, `_upc_code`      | Código de Produto Universal                    |
| Plugins GTIN        | `_gtin`, `_gtin13`, `_gtin14` | Número de Item de Comércio Global          |
| Código de Barras Genérico | `_barcode`, `_product_barcode` | Campos de código de barras genéricos |
| Códigos de Barras do Produto WooCommerce | `_barcode_text` | Plugin popular de código de barras     |
| Inventário ATUM     | `_atum_barcode`          | Gerenciamento de inventário ATUM               |

### Encontrando a Chave Meta do Seu Plugin

Para encontrar a chave meta correta para o seu plugin de código de barras:

1. Edite um produto no WP Admin
2. Adicione um código de barras usando seu plugin
3. Verifique a documentação do plugin para o nome da chave meta
4. Ou inspecione a tabela `wp_postmeta` no banco de dados para ver qual chave meta foi criada

### Chaves Meta Personalizadas

Você também pode usar qualquer chave meta personalizada que armazene dados de código de barras. Basta inserir o nome da chave meta (incluindo o prefixo de sublinhado, se aplicável) neste campo.

:::tip
Teste a leitura de código de barras após alterar esta configuração para garantir que esteja funcionando corretamente com o campo escolhido. Lembre-se de limpar e atualizar seus dados de produto no POS para que a alteração tenha efeito.
:::
