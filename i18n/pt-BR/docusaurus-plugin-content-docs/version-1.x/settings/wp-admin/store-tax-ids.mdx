Os IDs Fiscais da Loja são os identificadores fiscais empresariais que o WCPOS imprime nos seus recibos e faturas do POS — seu número de IVA, ABN, GSTIN, EIN ou qualquer número de registro comercial local.

O WooCommerce não possui um campo nativo para esses dados, e muitas regiões precisam de mais de um. Uma empresa alemã pode imprimir um USt-IdNr. (número de identificação de IVA), um Steuernummer (número fiscal) **e** um número HRB (registro comercial); uma empresa italiana imprime tanto o Codice Fiscale quanto a Partita IVA; muitas empresas exibem um número de IVA junto com um número de registro local. O campo IDs Fiscais da Loja permite adicionar quantas entradas forem necessárias.

Você encontra em **WP Admin > POS > Configurações > Geral**, na seção **IDs Fiscais da Loja**.

{/* Screenshot: wp-admin-store-tax-ids.png */}

## Adicionando um ID fiscal {#adding-a-tax-id}

1. Acesse **WP Admin > POS > Configurações > Geral** e role até **IDs Fiscais da Loja**.
2. Clique em **Adicionar ID fiscal**.
3. Escolha um **Tipo**, insira o **Valor** e, opcionalmente, defina um **País** e um **Rótulo** personalizado.
4. Repita para cada identificador que precisar imprimir.
5. Use **Remover** para excluir uma linha.

:::info As linhas são salvas automaticamente
Não há botão Salvar nesta seção. Cada linha é salva quando o campo **Valor** perde o foco — clique fora ou pressione Tab para confirmar.
:::

### A primeira linha usa o país da sua loja {#the-first-row-uses-your-store-country}

Ao adicionar seu primeiro ID fiscal, o WCPOS preenche automaticamente o **País** com o país base do WooCommerce e seleciona o **Tipo** correspondente — assim, uma empresa que precisa apenas de um único ID fiscal local não precisa escolher nada manualmente. Linhas existentes nunca são alteradas.

Tipo e país permanecem sincronizados durante a edição: alterar o **Tipo** atualiza o **País** para o país de origem daquele tipo, e alterar o **País** muda o **Tipo** para o identificador habitual daquele país.

### Tipos disponíveis {#available-types}

A lista de Tipos abrange os identificadores comuns voltados ao cliente, além dos números de registro empresarial que as empresas imprimem nos recibos:

| Região | Tipos |
|--------|-------|
| União Europeia | IVA da UE |
| Reino Unido | IVA do Reino Unido, Número de registro da empresa (Reino Unido) |
| Alemanha | USt-IdNr. (nº de identificação do IVA), Steuernummer (nº fiscal), HRB (registro comercial) |
| França | SIRET, SIREN |
| Outros países europeus | Holanda KVK, Itália Codice Fiscale, Itália Partita IVA, Espanha NIF, Suíça UID |
| Américas | EIN (EUA), Canadá GST/HST, Brasil CPF, Brasil CNPJ, Argentina CUIT |
| Ásia e Oriente Médio | Índia GSTIN, Arábia Saudita IVA |
| Austrália | ABN |
| Genérico | Outro |

O campo **Valor** exibe um exemplo de formato como texto de placeholder (por exemplo, `12-3456789` para um EIN dos EUA). Isso é apenas uma referência — o WCPOS não rejeita valores que não correspondam ao formato esperado, portanto identificadores incomuns, mas válidos, são salvos normalmente.

## O que é impresso nos recibos {#what-prints-on-receipts}

Cada identificação fiscal adicionada é impressa nos recibos e faturas, em ordem, cada uma em sua própria linha. O WCPOS utiliza um rótulo localizado para o tipo ("VAT ID", "ABN", "GSTIN" e assim por diante) — ou o **Rótulo** personalizado, caso definido. Se duas linhas tiverem o mesmo valor, a duplicata é removida automaticamente.

Todos os modelos de recibo inclusos já renderizam a lista completa. Se você mantém um **modelo personalizado** que ainda referencia um único valor `store.tax_id`, ele continuará funcionando, mas imprimirá apenas a primeira entrada. Em vez disso, altere-o para iterar sobre a lista:

```html
{{#store.tax_ids}}{{label}}: {{value}}{{/store.tax_ids}}
```

## Identificações fiscais por loja (Pro) {#per-store-tax-ids-pro}

:::info Recurso Pro
Com o [WCPOS Pro](/getting-started/pro-license) você pode definir IDs fiscais **por loja** em vez de uma única vez para todo o site — útil quando cada localidade possui seu próprio VAT/EIN/GSTIN, ou quando uma rede opera em regiões com identificadores diferentes.
:::

Os IDs fiscais por loja são definidos em **WP Admin > POS > Lojas > Editar Loja**, na seção **IDs Fiscais** da barra lateral. Enquanto a lista própria de uma loja estiver vazia, ela utiliza os IDs fiscais da loja configurados aqui na aba Geral, válidos para todo o site. Assim que você adicionar uma entrada a uma loja, essa loja passará a imprimir apenas sua própria lista.
