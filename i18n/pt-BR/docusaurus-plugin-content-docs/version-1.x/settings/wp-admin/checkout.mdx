A página de Configurações de Finalização de Compra (`WP Admin > POS > Configurações > Finalização de compra`) controla quais gateways de pagamento aparecem no POS, como os pedidos são processados e quais e-mails de notificação são disparados para vendas no POS.

## Notificações por E-mail {#email-notifications}

No topo da página há três grupos de opções de e-mail — **E-mails do administrador**, **E-mails do cliente** e **E-mails do operador de caixa** — que controlam quais e-mails de notificação do WooCommerce são enviados para pedidos do POS, independentemente das configurações de e-mail do WooCommerce para todo o site.

Consulte **[Notificações por E-mail](/settings/wp-admin/email-notifications)** para a referência completa, incluindo padrões de cada opção de e-mail, o modelo de destinatário do operador de caixa, cenários comuns e hooks para desenvolvedores.

## Gateways de Pagamento {#payment-gateways}

Esta seção lista todos os gateways de pagamento do WooCommerce instalados no seu site. Cada gateway pode ser habilitado ou desabilitado independentemente para o POS.

| Coluna | Descrição |
|--------|-------------|
| **Habilitado** | Ativar ou desativar o gateway para a finalização de compra do POS |
| **Gateway** | O nome do gateway de pagamento |
| **ID do Gateway** | O identificador do gateway no WooCommerce (útil para desenvolvedores) |

### Ativando um Gateway {#enabling-a-gateway}

1. Encontre o gateway na lista
2. Ative o botão **Habilitado**
3. O gateway aparecerá no checkout do POS no próximo carregamento da página

### Reordenando Gateways {#reordering-gateways}

Arraste os gateways para alterar a ordem de exibição no checkout do POS. O primeiro gateway habilitado é selecionado por padrão quando o operador de caixa abre o modal de checkout.

### POS vs Loja Online {#pos-vs-online-store}

As configurações de gateway aqui são **independentes** das configurações de pagamento do WooCommerce. Um gateway pode ser:

- **Habilitado para o POS, desabilitado online** — útil para pagamentos em dinheiro que não se aplicam a pedidos pela web
- **Habilitado online, desabilitado para o POS** — útil para gateways como PayPal que não são práticos para vendas presenciais
- **Habilitado para ambos** — o gateway funciona em todos os lugares

:::note
Um gateway deve primeiro ser instalado e ativado em `WP Admin > WooCommerce > Configurações > Pagamentos` antes de aparecer na lista de configurações de checkout do POS. Se um gateway estiver ausente, verifique se o plugin está instalado e se o gateway está habilitado no WooCommerce.
:::

## Status do Pedido {#order-status}

Configure o status padrão dos pedidos concluídos no POS. Por padrão, os pedidos do POS são definidos como **Concluído**, pois o pagamento é coletado no ponto de venda.
