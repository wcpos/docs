A página Configurações de Checkout (`WP Admin > POS > Settings > Checkout`) controla quais gateways de pagamento aparecem no POS e como os pedidos são processados.

## Gateways de Pagamento {#payment-gateways}

Esta seção lista todos os gateways de pagamento do WooCommerce instalados no site. Cada gateway pode ser ativado ou desativado de forma independente para o POS.

| Coluna | Descrição |
|--------|-------------|
| **Ativado** | Liga ou desliga o gateway para o checkout do POS |
| **Gateway** | O nome do gateway de pagamento |
| **ID do Gateway** | O identificador do gateway no WooCommerce (útil para desenvolvedores) |

### Habilitando um gateway {#enabling-a-gateway}

1. Localize o gateway na lista
2. Ative o botão **Habilitado**
3. O gateway aparecerá no checkout do POS no próximo carregamento da página

### Reordenando gateways {#reordering-gateways}

Arraste os gateways para alterar a ordem de exibição no checkout do POS. O primeiro gateway habilitado é selecionado por padrão quando o caixa abre o modal de checkout.

### POS vs. loja online {#pos-vs-online-store}

As configurações de gateway aqui são **independentes** das configurações de pagamento do WooCommerce. Um gateway pode estar:

- **Habilitado para o POS, desabilitado online** — útil para pagamentos em dinheiro, que não se aplicam a pedidos pela web
- **Habilitado online, desabilitado para o POS** — útil para gateways como o PayPal, que não são práticos para vendas presenciais
- **Habilitado em ambos** — o gateway funciona em todos os lugares

:::note
Um gateway precisa primeiro estar instalado e ativo no `WP Admin > WooCommerce > Settings > Payments` para aparecer na lista de configurações de checkout do POS. Se um gateway estiver ausente, verifique se o plugin está instalado e se o gateway está habilitado no WooCommerce.
:::

## Status do pedido {#order-status}

Configure o status padrão dos pedidos concluídos no POS. Por padrão, os pedidos do POS são definidos como **Concluído**, pois o pagamento é recebido no ponto de venda.
