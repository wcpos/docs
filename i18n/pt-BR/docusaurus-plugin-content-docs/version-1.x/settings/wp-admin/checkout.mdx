A página Configurações de finalização da compra (`WP Admin > POS > Configurações > Finalização de compra`) controla quais gateways de pagamento aparecem no POS, como os pedidos são processados e quais e-mails de notificação são disparados para vendas no POS.

## Notificações por e-mail {#email-notifications}

No topo da página há três grupos de alternâncias de e-mail — **E-mails do admin**, **E-mails do cliente** e **E-mails do operador de caixa** — que controlam quais e-mails de notificação do WooCommerce são enviados para pedidos do POS, independentemente das configurações de e-mail do WooCommerce em todo o site.

Consulte **[Notificações por e-mail](/settings/wp-admin/email-notifications)** para a referência completa, incluindo os padrões de alternância por e-mail, o modelo de destinatário do operador de caixa, cenários comuns e hooks para desenvolvedores.

## Gateways de pagamento {#payment-gateways}

Esta seção lista todos os gateways de pagamento do WooCommerce instalados no seu site. Cada gateway pode ser ativado ou desativado de forma independente para o POS.

| Coluna | Descrição |
|--------|-------------|
| **Padrão** | Escolha qual gateway será pré-selecionado quando um operador de caixa abrir a finalização da compra |
| **Gateway de pagamento** | O nome do gateway de pagamento |
| **ID do método de pagamento** | O identificador do método de pagamento do WooCommerce (útil para desenvolvedores) |
| **Ativado** | Ative ou desative o método de pagamento para o checkout do POS |
| **Status do pedido** | O status definido para um pedido do POS quando ele é pago com este método de pagamento (veja [Status do pedido](#order-status) abaixo) |

Cada linha também tem um menu (⋮) com **Configurações do POS** — altere o **Título** e a **Descrição** exclusivos do POS para o método de pagamento — e **Configurações do WooCommerce**, que abre diretamente as configurações completas do método de pagamento no WooCommerce.

### Ativar um método de pagamento {#enabling-a-gateway}

1. Encontre o método de pagamento na lista
2. Ative a opção **Ativado**
3. O método de pagamento aparecerá no checkout do POS no próximo carregamento da página

:::note Gratuito x Pro
Na versão gratuita, apenas os métodos de pagamento integrados **Dinheiro** e **Cartão** podem ser editados. Ativar outros métodos de pagamento do WooCommerce instalados para o POS requer o [WCPOS Pro](/getting-started/pro-license).
:::

### Reordenar métodos de pagamento {#reordering-gateways}

Arraste os métodos de pagamento para alterar a ordem de exibição deles na finalização de compra do POS. Use a coluna **Padrão** para escolher qual método de pagamento será pré-selecionado quando um operador de caixa abrir o modal de finalização de compra.

### POS x loja online {#pos-vs-online-store}

As configurações de métodos de pagamento aqui são **independentes** das suas configurações de pagamento do WooCommerce. Um método de pagamento pode estar:

- **Ativado para POS, desativado online** — útil para pagamentos em dinheiro, que não se aplicam a pedidos pela web
- **Ativado online, desativado para POS** — útil para métodos de pagamento como PayPal, que não são práticos para vendas presenciais
- **Ativado para ambos** — o método de pagamento funciona em todos os lugares

:::note
Um método de pagamento deve primeiro estar instalado e ativo em `WP Admin > WooCommerce > Configurações > Pagamentos` antes de aparecer na lista de configurações de finalização de compra do POS. Se um método de pagamento estiver ausente, verifique se o plugin está instalado e se o método de pagamento está ativado no WooCommerce.
:::

## Status do pedido {#order-status}

O status do pedido é definido **por método de pagamento**, na coluna **Status do pedido** da tabela de métodos de pagamento — portanto, uma venda pode ser resolvida de maneiras diferentes dependendo de como foi paga:

- **Dinheiro** e **Cartão** usam **Concluído** como padrão, pois o pagamento é recebido no ponto de venda.
- Métodos de pagamento em que o dinheiro chega posteriormente — como **BACS** (transferência bancária direta) ou **Cheque** — usam **Em espera** por padrão.

Selecione qualquer status de pedido do WooCommerce para cada método de pagamento. (Versões mais antigas usavam um único padrão global; esse valor foi migrado para cada método de pagamento durante a atualização.)
