- Abre um diálogo para adicionar uma nota ao pedido do cliente.
- Notas de clientes são visíveis para o cliente e aparecem no recibo por padrão, na área Minha Conta do cliente, e em `WP Admin > WooCommerce > Orders > Order` sob o endereço de entrega.

:::note
Notas de pedidos são visíveis para os clientes.
:::

- Abre um modal para gerenciar metadados adicionais para o pedido.
- Os recursos incluem:
  - Alterar a moeda do pedido.
  - Adicionar um ID de transação (por exemplo, de um terminal de pagamento externo).
  - Adicionar metadados personalizados para o pedido.

:::note
Recursos adicionais serão adicionados a este modal ao longo do tempo, incluindo integração com outros plugins do WooCommerce.
:::

### O que são Metadados no WooCommerce? {#what-is-meta-data-in-woocommerce}

Metadados fornecem informações extras sobre os pedidos e são acessíveis via a API REST do WooCommerce. Desenvolvedores e plugins usam esses dados para expandir a funcionalidade, como integrar serviços de terceiros.

### Visualização JSON {#json-view}

- Visualize a representação JSON bruta do pedido, útil para depuração ou integração.

## Salvar no Servidor {#save-to-server}

- Permite que você salve carrinhos abertos no servidor, para que sejam armazenados de forma segura e acessíveis para uso futuro.
- Cria um pedido no WooCommerce com o status `pos-open`.

### Por que salvar? {#why-save}

Quando um novo pedido é criado no POS, ele existe apenas localmente. 
Salvar o pedido garante que ele persista, mesmo que o banco de dados local seja limpo.

Veja [Pedidos Abertos](/pos/cart/open-orders) para mais detalhes sobre como gerenciar pedidos salvos.

## Anular {#void}

- Dois Cenários:
  - Pedidos Não Salvos:
    - Se o pedido não foi salvo no servidor, ele será permanentemente excluído do banco de dados local.
  - Pedidos Salvos:
    - Se o pedido estiver salvo no servidor, ele será movido para a pasta Lixeira no WooCommerce e excluído do banco de dados local.
- Para recuperar um pedido anulado:
  - Vá para `WP Admin > WooCommerce > Orders > Trash`.

## Finalizar Compra {#checkout}

- Salva o pedido no servidor e inicia o processo de finalização de compra.
- Abre a [tela de finalização de compra](/pos/checkout) para concluir o pagamento.
