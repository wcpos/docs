WCPOS permite que você trabalhe com múltiplos pedidos simultaneamente. Isso é útil para gerenciar retenções de clientes, alternar entre transações e recuperar-se de interrupções.

## Carrossel de Pedidos Abertos

Na parte inferior do Painel de Carrinho, um carrossel horizontal exibe todos os pedidos abertos:

{/* Screenshot: open-orders-carousel.png */}

- Cada carrinho mostra seu **valor total**
- O **pedido atual** está em destaque
- Clique em qualquer carrinho para alternar para ele
- Role para a esquerda/direita para ver mais carrinhos

## Criando um Novo Pedido

Um novo carrinho vazio está sempre disponível. Basta clicar em um carrinho vazio no carrossel ou começar a adicionar produtos quando o carrinho atual estiver vazio.

## Alternando Entre Pedidos

Clique em qualquer pedido no carrossel para alternar para ele. O Painel de Carrinho é atualizado para mostrar o conteúdo do pedido selecionado.

**Casos de uso:**

- O cliente se afasta para pegar outro item
- É necessário ajudar um cliente rápido enquanto um pedido grande está em andamento
- Comparar preços ou itens entre pedidos

## Salvando Pedidos no Servidor

Os pedidos existem em dois estados:

### Apenas Local

Por padrão, novos pedidos são armazenados apenas no banco de dados local do navegador/app. Eles persistirão entre atualizações de página, mas:

- Não são visíveis no admin do WooCommerce
- Serão perdidos se o banco de dados local for limpo
- Não são acessíveis de outros dispositivos

### Salvo no Servidor

Clique em **Salvar no Servidor** para criar um pedido no WooCommerce com o status `pos-open`. Isso:

- Cria um pedido real no WooCommerce
- Persiste mesmo se o banco de dados local for limpo
- Pode ser acessado de outros dispositivos
- Aparece em WP Admin > WooCommerce > Pedidos

:::tip Quando Salvar
Salve pedidos no servidor quando:
- Um cliente deseja segurar um pedido para retirada posterior
- Você está encerrando seu turno e outro caixa continuará
- Você quer um backup em caso de problemas no app/navegador
:::

## Recuperando Pedidos Salvos

Se você salvou pedidos no servidor, eles podem ser acessados novamente por:

1. Abrindo a tela de **Pedidos** (recurso Pro)
2. Filtrando por status `pos-open`
3. Reabrindo o pedido

## Persistência de Pedidos

### Armazenamento Local

WCPOS usa IndexedDB para armazenar pedidos localmente. Isso fornece:

- Persistência entre sessões do navegador
- Acesso rápido sem requisições de rede
- Capacidade offline

### Sincronização com o Servidor

Quando você salva no servidor ou finaliza a compra:

- O pedido é enviado para o WooCommerce
- Uma confirmação é recebida
- Os dados locais e do servidor são sincronizados

## Cancelamento de Pedidos

Para remover um pedido aberto:

1. Altere para o pedido que você deseja remover
2. Clique no botão **Anular**

**O que acontece:**

- **Pedidos não salvos:** Excluídos permanentemente do banco de dados local
- **Pedidos salvos:** Movidos para a Lixeira no WooCommerce e excluídos localmente

Para recuperar um pedido salvo que foi anulado:
1. Vá para `WP Admin > WooCommerce > Pedidos > Lixeira`
2. Restaure o pedido

## Dicas

### Mantenha os Pedidos Organizados

Com múltiplos pedidos abertos, é útil:

- Adicionar nomes de clientes aos pedidos para fácil identificação
- Adicionar notas ao pedido descrevendo o motivo da retenção
- Salvar pedidos importantes no servidor

### Transferência de Turno

Ao encerrar um turno com pedidos abertos:

1. Salve todos os pedidos importantes no servidor
2. Adicione notas ao pedido explicando o status
3. O próximo caixa pode acessá-los a partir da tela de Pedidos

### Considerações Offline

Se você perder a conectividade:

- Os pedidos locais permanecem acessíveis e você pode continuar adicionando itens
- Você não pode completar/finalizar pedidos até que a conectividade seja restaurada
- Você não pode salvar pedidos no servidor até ser reconectado
- Você não pode criar novos clientes até ser reconectado
