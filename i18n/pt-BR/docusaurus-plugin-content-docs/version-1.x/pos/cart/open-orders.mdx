O WCPOS permite que você trabalhe com múltiplos pedidos simultaneamente. Isso é útil para gerenciar reservas de clientes, alternar entre transações e recuperar-se de interrupções.

## Carrossel de Pedidos Abertos

Na parte inferior do Painel do Carrinho, um carrossel horizontal exibe todos os pedidos abertos:

{/* Screenshot: open-orders-carousel.png */}

- Cada carrinho mostra seu **valor total**
- O **pedido atual** é destacado
- Clique em qualquer carrinho para alternar para ele
- Role para a esquerda/direita para ver mais carrinhos

## Criando um Novo Pedido

Um novo carrinho vazio está sempre disponível. Simplesmente clique em um carrinho vazio no carrossel ou comece a adicionar produtos quando o carrinho atual estiver vazio.

## Alternando Entre Pedidos

Clique em qualquer pedido no carrossel para alternar para ele. O Painel do Carrinho é atualizado para mostrar o conteúdo do pedido selecionado.

**Casos de uso:**

- Cliente se afasta para pegar outro item
- Necessidade de atender um cliente rápido enquanto um pedido grande está em andamento
- Comparar preços ou itens entre pedidos

## Salvando Pedidos no Servidor

Os pedidos existem em dois estados:

### Apenas Local

Por padrão, novos pedidos são armazenados apenas no banco de dados local do navegador/aplicativo. Eles persistirão entre atualizações de página, mas:

- Não são visíveis no admin do WooCommerce
- Serão perdidos se o banco de dados local for limpo
- Não são acessíveis de outros dispositivos

### Salvos no Servidor

Clique em **Salvar no Servidor** para criar um pedido do WooCommerce com o status `pos-open`. Isso:

- Cria um pedido real no WooCommerce
- Persiste mesmo se o banco de dados local for limpo
- Pode ser acessado de outros dispositivos
- Aparece em WP Admin > WooCommerce > Pedidos

:::tip Quando Salvar
Salve pedidos no servidor quando:
- Um cliente quiser reservar um pedido para retirada posterior
- Você estiver terminando seu turno e outro caixa continuará
- Quiser um backup em caso de problemas no aplicativo/navegador
:::

## Recuperando Pedidos Salvos

Se você salvou pedidos no servidor, eles podem ser acessados novamente:

1. Abrindo a tela de **Pedidos** (recurso Pro)
2. Filtrando por status `pos-open`
3. Reabrindo o pedido

## Persistência de Pedidos

### Armazenamento Local

O WCPOS usa IndexedDB para armazenar pedidos localmente. Isso fornece:

- Persistência entre sessões do navegador
- Acesso rápido sem solicitações de rede
- Capacidade offline

### Sincronização com Servidor

Quando você salva no servidor ou finaliza:

- O pedido é enviado para o WooCommerce
- Uma confirmação é recebida
- Dados locais e do servidor são sincronizados

## Anulando Pedidos

Para remover um pedido aberto:

1. Alterne para o pedido que deseja remover
2. Clique no botão **Anular**

**O que acontece:**

- **Pedidos não salvos:** Excluídos permanentemente do banco de dados local
- **Pedidos salvos:** Movidos para a Lixeira no WooCommerce e excluídos localmente

Para recuperar um pedido salvo anulado:
1. Vá para `WP Admin > WooCommerce > Pedidos > Lixeira`
2. Restaure o pedido

## Dicas

### Mantenha os Pedidos Organizados

Com múltiplos pedidos abertos, é útil:

- Adicionar nomes de clientes aos pedidos para identificação fácil
- Adicionar notas de pedido descrevendo o motivo da reserva
- Salvar pedidos importantes no servidor

### Passagem de Turno

Ao encerrar um turno com pedidos abertos:

1. Salve todos os pedidos importantes no servidor
2. Adicione notas de pedido explicando o status
3. O próximo caixa poderá acessá-los na tela de Pedidos

### Considerações Offline

Se você perder a conectividade:

- Pedidos locais permanecem acessíveis e você pode continuar adicionando itens
- Você não pode finalizar/finalizar pedidos até que a conectividade seja restaurada
- Você não pode salvar pedidos no servidor até reconectar
- Você não pode criar novos clientes até reconectar
