Quando estiver tudo pronto para concluir uma venda, clique no botão **Finalizar compra** para abrir o modal de finalização de compra. É nele que o pagamento é processado e o pedido é concluído.

{/* Screenshot: checkout-modal.png */}

## Visão geral do modal de finalização de compra {#checkout-modal-overview}

O modal de finalização de compra exibe:

- **Número do pedido** - O ID do pedido do WooCommerce
- **Valor a pagar** - Valor total devido
- Botão **Configurações de finalização de compra** - Solucione problemas de exibição
- **Caixa** - Quem está processando o pedido
- **Cliente** - O cliente deste pedido (link clicável)
- botão **Adicionar cupom** - Aplicar códigos de desconto
- **Resumo do pedido** - Produtos, quantidades e totais
- **Métodos de pagamento** - Opções de pagamento disponíveis
- botões **Cancelar / Processar pagamento**

## Métodos de pagamento {#payment-methods}

### Disponível na versão gratuita {#available-in-free-version}

A versão gratuita do WCPOS inclui dois gateways de pagamento:

- **Dinheiro** - Com valor recebido e calculadora de troco
- **Cartão** - Para terminais de cartão externos

### Gateways adicionais (Pro) {#additional-gateways-pro}

:::info Recurso Pro
Gateways de pagamento adicionais exigem o [WCPOS Pro](/getting-started/pro-license).
:::

Com o Pro, você pode habilitar:

- **Stripe Terminal** - Integração direta com leitores de cartão Stripe
- **SumUp Terminal** - Integração com leitores de cartão SumUp
- **Gateways personalizados** - Crie suas próprias integrações de pagamento

Consulte [Pagamento](/payment) para obter detalhes sobre a configuração de gateways.

### Selecionar uma forma de pagamento {#selecting-a-payment-method}

Clique em uma forma de pagamento para selecioná-la. O formulário é atualizado para mostrar os campos relevantes:

**Dinheiro:**

- **Valor recebido** - Informe o valor que o cliente entrega
- **Troco** - Troco a devolver calculado automaticamente

{/* Screenshot: checkout-cash-payment.png */}

**Cartão:**

- Processe o pagamento no seu terminal de cartão externo
- Clique em Processar pagamento para concluir

## Cupons {#coupons}

O carrinho inclui um campo **Adicionar cupom** acima dos totais (somente Pro). Digite o código ou pesquise pela descrição; o cupom é validado localmente e aparece como um marcador removível. Vários cupons são aplicados em sequência.

Consulte **[Aplicação de cupons no caixa](/coupons/applying-coupons)** para ver o fluxo completo — pesquisa, marcadores, descontos sequenciais e a tabela de erros de validação e resoluções. Para tipos de cupom, regras de validação e configuração no WooCommerce, consulte [Cupons](/coupons).

## Processamento do pagamento {#processing-payment}

1. Selecione uma forma de pagamento
2. Para dinheiro, informe o valor recebido
3. Clique em **Processar pagamento**
4. O pedido é concluído e o [recibo](/receipts/at-checkout) é exibido

## Configurações de checkout (Solução de problemas) {#checkout-settings-troubleshooting}

O modal de finalização da compra usa um iframe/webview para exibir a página de pagamento do pedido do WooCommerce. Isso aproveita a infraestrutura de pagamento existente do WooCommerce, o que significa que qualquer gateway de pagamento que funcione com o WooCommerce deve funcionar no POS.

No entanto, scripts de temas e plugins às vezes podem interferir. Clique em **Configurações de finalização da compra** para solucionar problemas:

### Desativar todos os estilos e scripts {#disable-all-styles-and-scripts}

:::warning Opção nuclear
Esta é a opção nuclear e deve ser usada apenas para testes ou em casos raros em que o desenvolvedor sabe o que está fazendo. Desativar todos os scripts wp_head removerá até mesmo os scripts do WooCommerce necessários para expandir/recolher os gateways de pagamento, possivelmente interrompendo a funcionalidade de pagamento.
:::

- **Desativar wp_head** - Remove todos os scripts/estilos do cabeçalho do WordPress
- **Desativar wp_footer** - Remove todos os scripts/estilos do rodapé do WordPress

### Desativar estilos selecionados {#disable-selected-styles}

Desative seletivamente o CSS que pode causar problemas de exibição:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Desativar scripts selecionados {#disable-selected-scripts}

Desative seletivamente o JavaScript que pode interferir nos gateways de pagamento:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Se um gateway de pagamento não for exibido corretamente:

1. Tente desativar primeiro os estilos do tema
2. Depois, tente desativar os scripts do WooCommerce que não são necessários
3. Tenha cuidado para não desativar scripts exigidos pelo seu gateway de pagamento
   :::

## Cancelar {#cancel}

Clique em **Cancelar** para fechar o modal de finalização da compra sem concluir o pedido. O pedido permanece como um carrinho aberto.
