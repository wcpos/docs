Quando estiver pronto para concluir uma venda, clique no botão **Finalizar Compra** para abrir o modal de finalização. É aqui que o pagamento é processado e o pedido é concluído.

{/* Screenshot: checkout-modal.png */}

## Visão Geral do Modal de Finalização de Compra {#checkout-modal-overview}

O modal de finalização de compra exibe:

- **Número do pedido** - O ID do pedido no WooCommerce
- **Valor a Pagar** - Valor total devido
- Botão **Configurações de Finalização de Compra** - Solucionar problemas de exibição
- **Caixa** - Quem está processando o pedido
- **Cliente** - O cliente deste pedido (link clicável)
- Botão **Adicionar Cupom** - Aplicar códigos de desconto
- **Resumo do pedido** - Produtos, quantidades e totais
- **Métodos de pagamento** - Opções de pagamento disponíveis
- Botões **Cancelar / Processar Pagamento**

## Métodos de Pagamento {#payment-methods}

### Disponível na Versão Gratuita {#available-in-free-version}

A versão gratuita do WCPOS inclui dois gateways de pagamento:

- **Dinheiro** - Com valor recebido e calculadora de troco
- **Cartão** - Para terminais de cartão externos

### Gateways Adicionais (Pro) {#additional-gateways-pro}

:::info Recurso Pro
Gateways de pagamento adicionais requerem o [WCPOS Pro](/getting-started/pro-license).
:::

Com o Pro, você pode habilitar:

- **Stripe Terminal** - Integração direta com leitores de cartão Stripe
- **SumUp Terminal** - Integração com leitores de cartão SumUp
- **Gateways Personalizados** - Crie suas próprias integrações de pagamento

Consulte [Pagamento](/payment) para detalhes sobre a configuração de gateways.

### Selecionando um Método de Pagamento {#selecting-a-payment-method}

Clique em um método de pagamento para selecioná-lo. O formulário é atualizado para exibir os campos relevantes:

**Dinheiro:**

- **Valor Recebido** - Insira o valor que o cliente entregou
- **Troco** - Troco calculado automaticamente a ser devolvido

{/* Screenshot: checkout-cash-payment.png */}

**Cartão:**

- Processe o pagamento no seu terminal de cartão externo
- Clique em Processar Pagamento para concluir

## Cupons {#coupon-codes}

O carrinho inclui um campo **Adicionar Cupom** acima dos totais (somente Pro). Digite o código ou pesquise pela descrição; o cupom é validado localmente e aparece como uma etiqueta removível. Múltiplos cupons são aplicados sequencialmente.

Consulte **[Aplicando Cupons no Caixa](/coupons/applying-coupons)** para o fluxo completo — pesquisa, etiquetas, descontos sequenciais e a tabela de erros de validação e resoluções. Para tipos de cupons, regras de validação e configuração no WooCommerce, consulte [Cupons](/coupons).

## Processando Pagamento {#processing-payment}

1. Selecione um método de pagamento
2. Para dinheiro, insira o valor recebido
3. Clique em **Processar Pagamento**
4. O pedido é concluído e o [recibo](/receipts/at-checkout) é exibido

## Configurações de Finalização de Compra (Solução de Problemas) {#checkout-settings-troubleshooting}

O modal de finalização de compra utiliza um iframe/webview para exibir a página de Pagamento de Pedido do WooCommerce. Isso aproveita a infraestrutura de pagamento existente do WooCommerce, o que significa que qualquer gateway de pagamento compatível com o WooCommerce deve funcionar no POS.

No entanto, scripts de temas e plugins podem interferir em alguns casos. Clique em **Configurações de Finalização de Compra** para solucionar problemas:

### Desabilitar Todos os Estilos e Scripts {#disable-all-styles-and-scripts}

:::warning Opção Nuclear
Esta é a opção nuclear e deve ser usada apenas para testes ou em casos raros em que o desenvolvedor sabe o que está fazendo. Desabilitar todos os scripts do wp_head removerá até mesmo os scripts do WooCommerce necessários para expandir/contrair os gateways de pagamento, podendo comprometer a funcionalidade de pagamento.
:::

- **Desabilitar wp_head** - Remove todos os scripts/estilos do cabeçalho do WordPress
- **Desabilitar wp_footer** - Remove todos os scripts/estilos do rodapé do WordPress

### Desabilitar Estilos Selecionados {#disable-selected-styles}

Desabilite seletivamente CSS que possa causar problemas de exibição:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Desabilitar Scripts Selecionados {#disable-selected-scripts}

Desabilitar seletivamente JavaScript que pode interferir nos gateways de pagamento:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Se um gateway de pagamento não for exibido corretamente:

1. Tente desabilitar os estilos do tema primeiro
2. Em seguida, tente desabilitar os scripts do WooCommerce que não são necessários
3. Tenha cuidado para não desabilitar scripts necessários para o seu gateway de pagamento
   :::

## Cancelar {#cancel}

Clique em **Cancelar** para fechar o modal de finalização de compra sem concluir o pedido. O pedido permanece como um carrinho aberto.
