Quando você estiver pronto para concluir uma venda, clique no botão **Checkout** para abrir o modal de checkout. É aqui que você processa o pagamento e conclui o pedido.

{/* Screenshot: checkout-modal.png */}

## Visão Geral do Modal de Checkout {#checkout-modal-overview}

O modal de checkout exibe:

- **Número do pedido** - O ID do pedido do WooCommerce
- **Valor a pagar** - Valor total devido
- **Botão de Configurações de Checkout** - Solucionar problemas de exibição
- **Caixa** - Quem está processando o pedido
- **Cliente** - O cliente para este pedido (link clicável)
- **Campo de código do cupom** - Aplicar códigos de desconto
- **Resumo do pedido** - Produtos, quantidades e totais
- **Métodos de pagamento** - Opções de pagamento disponíveis
- **Cancelar / Processar Pagamento** - botões

## Métodos de Pagamento {#payment-methods}

### Disponível na Versão Gratuita {#available-in-free-version}

A versão gratuita do WCPOS inclui duas gateways de pagamento:

- **Dinheiro** - Com valor entregue e calculadora de troco
- **Cartão** - Para terminais de cartão externos

### Gateways Adicionais (Pro) {#additional-gateways-pro}

:::info Recurso Pro
Gateways de pagamento adicionais requerem [WCPOS Pro](/getting-started/pro-license).
:::

Com o Pro, você pode habilitar:

- **Terminal Stripe** - Integração direta com leitores de cartão Stripe
- **Terminal SumUp** - Integração com leitores de cartão SumUp
- **Gateways Personalizados** - Crie suas próprias integrações de pagamento

Veja [Pagamento](/payment) para detalhes sobre como configurar gateways.

### Selecionando um Método de Pagamento {#selecting-a-payment-method}

Clique em um método de pagamento para selecioná-lo. O formulário é atualizado para mostrar campos relevantes:

**Dinheiro:**

- **Valor Entregue** - Insira o valor que o cliente lhe dá
- **Troco** - Troco calculado automaticamente a devolver

{/* Screenshot: checkout-cash-payment.png */}

**Cartão:**

- Processar pagamento no seu terminal de cartão externo
- Clique em Processar Pagamento para concluir

## Códigos de Cupom {#coupon-codes}

Para aplicar um desconto:

1. Insira o código do cupom no campo
2. Clique em **Aplicar cupom**
3. O desconto aparece no resumo do pedido

Os cupons devem ser configurados no WooCommerce (`WP Admin > Marketing > Coupons`).

## Processando Pagamento {#processing-payment}

1. Selecione um método de pagamento
2. Para dinheiro, insira o valor entregue
3. Clique em **Processar Pagamento**
4. O pedido é concluído e o [recibo](/pos/checkout/receipts) é exibido

## Configurações de Checkout (Solução de Problemas) {#checkout-settings-troubleshooting}

O modal de checkout usa um iframe/webview para exibir a página de pagamento do Pedido do WooCommerce. Isso aproveita a infraestrutura de pagamento existente do WooCommerce, significando que qualquer gateway de pagamento que funcione com o WooCommerce deve funcionar no POS.

No entanto, scripts de tema e plugins podem às vezes interferir. Clique em **Configurações de Checkout** para solucionar problemas:

### Desabilitar Todos os Estilos e Scripts {#disable-all-styles-and-scripts}

:::warning Opção Nuclear
Esta é a opção nuclear e deve ser usada apenas para testes ou em casos raros onde o desenvolvedor sabe o que está fazendo. Desabilitar todos os scripts wp_head removerá até mesmo os scripts do WooCommerce necessários para expandir/contrair os gateways de pagamento, potencialmente quebrando a funcionalidade de pagamento.
:::

- **Desabilitar wp_head** - Remove todos os scripts/estilos do cabeçalho do WordPress
- **Desabilitar wp_footer** - Remove todos os scripts/estilos do rodapé do WordPress

### Desabilitar Estilos Selecionados {#disable-selected-styles}

Desabilitar seletivamente CSS que pode causar problemas de exibição:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- etc.

### Desabilitar Scripts Selecionados {#disable-selected-scripts}

Desabilitar seletivamente JavaScript que pode interferir com gateways de pagamento:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- etc.

:::tip
Se um gateway de pagamento não for exibido corretamente:

1. Tente desabilitar os estilos do tema primeiro
2. Depois tente desabilitar scripts do WooCommerce que não são necessários
3. Tenha cuidado para não desabilitar scripts exigidos pelo seu gateway de pagamento
   :::

## Cancelar {#cancel}

Clique em **Cancelar** para fechar o modal de checkout sem concluir o pedido. O pedido permanece como um carrinho aberto.
