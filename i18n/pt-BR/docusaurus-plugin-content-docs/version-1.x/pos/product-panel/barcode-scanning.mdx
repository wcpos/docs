A maioria dos scanners de código de barras se comporta como um teclado conectado ao seu dispositivo.
Quando você escaneia um código de barras, o WCPOS detecta que os caracteres foram inseridos mais rapidamente do que a digitação normal.
Ele usa essas "pressionamentos rápidos de tecla" para identificar a entrada como um escaneamento de código de barras.

## Configurando o Escaneamento de Código de Barras {#configuring-barcode-scanning}

Como o escaneamento de um código de barras acontece muito rápido, o POS consegue diferenciar entre um código de barras e algo digitado manualmente.
Nas configurações do POS, você encontrará opções para ajustar como a detecção de código de barras funciona.

- **Tempo médio de entrada:** Quão rápido a entrada deve ser para contar como um código de barras.
- **Comprimento mínimo:** Quanto tempo a sequência contínua de caracteres deve ter para ser tratada como um código de barras.
- **Remoção de prefixo/sufixo:** Se seu scanner adiciona caracteres extras (como um prefixo ou sufixo), você pode removê-los automaticamente para que apenas o código de barras principal permaneça.

## O Que Acontece Quando um Código de Barras é Detectado? {#what-happens-when-a-barcode-is-detected}

Quando o POS detecta um código de barras, ele procura em seu banco de dados local para encontrar um produto ou variação de produto correspondente.
Existem três resultados possíveis:

1. **Correspondência Exata (Um Produto Encontrado):**
   Se o código de barras corresponder exatamente a um único produto ou variação de produto no banco de dados local do POS, esse produto é adicionado automaticamente ao seu carrinho.

2. **Múltiplas Correspondências (Mais de Um Produto Encontrado):**
   Se mais de um produto tiver o mesmo código de barras, o POS não pode saber qual deles adicionar.
   Nesse caso, ele coloca o código de barras escaneado na barra de pesquisa para que você possa revisar quais produtos compartilham esse código.
   Quando você vê múltiplas correspondências, geralmente significa que os dados do produto precisam ser atualizados—cada produto deve ter um código de barras exclusivo.

3. **Sem Correspondência Local (Zero Produtos Encontrados):**
   Se o código de barras não corresponder a nenhum produto armazenado no seu dispositivo, o código escaneado aparecerá na barra de pesquisa.
   Nesse momento, o POS tentará encontrar uma correspondência online através do banco de dados da sua loja WooCommerce.

## Entendendo a Sincronização de Produtos {#understanding-product-synchronisation}

### Download Progressivo de Produtos {#progressive-product-downloading}

O WCPOS não carrega todos os seus produtos de uma vez.
Em vez disso, ele os baixa em pequenos lotes.
Essa abordagem evita lentidões e garante que sua loja funcione suavemente.
Com o tempo, à medida que você usa o POS e realiza buscas, mais de seus produtos são armazenados localmente no seu dispositivo.

Veja [Sincronização de Produtos](/products/sync) para mais detalhes.

### Por Que Isso é Importante para o Escaneamento de Código de Barras {#why-it-matters-for-barcode-scanning}

Quando você escaneia um código de barras que ainda não está armazenado localmente, o POS "vai online" para a sua loja WooCommerce para encontrá-lo.
Como parte desse processo, ele irá baixar esse produto (e outros em pequenos lotes) e salvá-los.
Isso significa que, com o tempo, o POS se torna mais rápido e eficiente à medida que mais produtos são armazenados localmente.

### Como Acelerar o Processo {#how-to-speed-up-the-process}

Simplesmente buscar produtos no POS ajuda a baixar mais do seu estoque.
Quanto mais você usar a pesquisa — e mais você escanear — mais completa sua base de dados local se torna.

## Dúvidas Frequentes (F.A.Q.) {#faq}

<Accordion>
  <AccordionItem question="Por que recebo '0 produtos encontrados localmente' quando escaneio um código de barras?">

Nem todos os produtos estão disponíveis localmente desde o início.
O POS baixa gradualmente produtos da sua loja online e os armazena no seu dispositivo.
Se o produto que você acabou de escanear ainda não está armazenado, a pesquisa faz com que o POS procure online e então o baixe para que esteja disponível no futuro.

  </AccordionItem>

  <AccordionItem question="O POS gera e imprime códigos de barras?">

Não, no momento. Nosso POS é projetado para escanear e ler códigos de barras existentes, mas não inclui funcionalidade para criá-los ou imprimi-los.
Se você precisar gerar códigos de barras para seus produtos, pode usar plugins de terceiros do WooCommerce que se especializam na criação e impressão de códigos de barras. Alguns exemplos incluem:

- [EAN for WooCommerce](https://wordpress.org/plugins/ean-for-woocommerce/)
- [A4 Barcode Generator](https://wordpress.org/plugins/a4-barcode-generator/)

Depois de gerar códigos de barras para seus produtos, você pode escaneá-los facilmente no caixa para acelerar o processo de checkout no POS.

  </AccordionItem>
</Accordion>
