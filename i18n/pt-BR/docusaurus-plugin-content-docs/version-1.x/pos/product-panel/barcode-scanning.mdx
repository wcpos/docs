A maioria dos scanners de código de barras se comporta como um teclado conectado ao seu dispositivo.
Quando você escaneia um código de barras, o WCPOS detecta que os caracteres foram inseridos mais rapidamente do que a digitação normal.
Ele usa essas "teclas pressionadas rapidamente" para identificar a entrada como uma leitura de código de barras.

## Configurando a Leitura de Código de Barras

Como uma leitura de código de barras acontece muito rapidamente, o POS consegue diferenciar entre um código de barras e algo digitado à mão.
Nas configurações do POS, você encontrará opções para ajustar como a detecção de código de barras funciona.

- **Tempo médio de entrada:** Quão rápido a entrada deve ser para contar como um código de barras.
- **Comprimento mínimo:** Quão longa a sequência contínua de caracteres deve ser para ser tratada como um código de barras.
- **Remoção de prefixo/sufixo:** Se o seu scanner adicionar caracteres extras (como um prefixo ou sufixo), você pode removê-los automaticamente para que apenas o código de barras principal permaneça.

## O Que Acontece Quando Um Código de Barras é Detectado?

Quando o POS detecta um código de barras, ele verifica em seu banco de dados local para encontrar um produto ou variação de produto correspondente.
Existem três resultados possíveis:

1. **Correspondência Exata (Um Produto Encontrado):**
   Se o código de barras corresponder exatamente a um único produto ou variação de produto no banco de dados local do POS, esse produto é automaticamente adicionado ao seu carrinho.

2. **Múltiplas Correspondências (Mais de Um Produto Encontrado):**
   Se mais de um produto tiver o mesmo código de barras, o POS não pode saber qual adicionar.
   Nesse caso, ele coloca o código de barras escaneado na barra de busca para que você possa revisar quais produtos compartilham esse código.
   Quando você vê múltiplas correspondências, isso geralmente significa que os dados do produto precisam ser atualizados—cada produto deve ter um código de barras único.

3. **Sem Correspondências Locais (Zero Produtos Encontrados):**
   Se o código de barras não corresponder a nenhum produto armazenado no seu dispositivo, o código escaneado aparecerá na barra de busca.
   Nesse ponto, o POS tentará encontrar uma correspondência online através do banco de dados da sua loja WooCommerce.

## Compreendendo a Sincronização de Produtos

### Download Progressivo de Produtos

O WCPOS não carrega todos os seus produtos de uma vez.
Em vez disso, ele os baixa em pequenos lotes.
Essa abordagem evita lentidões e garante que sua loja funcione sem problemas.
Com o tempo, conforme você usa o POS e realiza pesquisas, mais dos seus produtos são armazenados localmente no seu dispositivo.

Veja [Sincronização de Produtos](/products/sync) para mais detalhes.

### Por Que Isso Importa Para a Leitura de Código de Barras

Quando você escaneia um código de barras que ainda não está armazenado localmente, o POS "vai online" para a sua loja WooCommerce para encontrá-lo.
Como parte desse processo, ele baixará esse produto (e outros em pequenos lotes) e os salvará.
Isso significa que, com o tempo, o POS se torna mais rápido e eficiente à medida que mais produtos são armazenados localmente.

### Como Acelerar o Processo

Simplesmente pesquisar produtos no POS ajuda-o a baixar mais do seu inventário.
Quanto mais você usa a busca — e mais você escaneia — mais completa sua base de dados local se torna.

## F.A.Q.

<details>

<summary>
  Por que recebo "0 produtos encontrados localmente" ao escanear um código de barras?
</summary>

Nem todos os produtos estão disponíveis localmente desde o início.
O POS baixa gradualmente produtos da sua loja online e os armazena no seu dispositivo.
Se o produto que você acabou de escanear ainda não está armazenado, a pesquisa aciona o POS para procurá-lo online e então baixá-lo para que esteja disponível no futuro.

</details>

<details>

<summary>O POS gera e imprime códigos de barras?</summary>

Não, no momento. Nosso POS é projetado para escanear e ler códigos de barras existentes, mas não inclui funcionalidade para criá-los ou imprimi-los.
Se você precisar gerar códigos de barras para seus produtos, pode usar plugins de terceiros do WooCommerce que se especializam na criação e impressão de códigos de barras. Alguns exemplos incluem:

- [EAN for WooCommerce](https://wordpress.org/plugins/ean-for-woocommerce/)
- [Gerador de Código de Barras A4](https://wordpress.org/plugins/a4-barcode-generator/)

Depois de gerar códigos de barras para seus produtos, você pode escaneá-los facilmente no caixa para acelerar o processo de checkout no POS.

</details>
