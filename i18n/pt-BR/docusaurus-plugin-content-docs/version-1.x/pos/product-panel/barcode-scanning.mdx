A maioria dos leitores de código de barras se comporta como um teclado conectado ao seu dispositivo.
Quando você lê um código de barras, o WCPOS detecta que os caracteres foram inseridos mais rapidamente do que a digitação normal.
Ele usa essas "teclas pressionadas rapidamente" para identificar a entrada como uma leitura de código de barras.

## Configurando a Leitura de Código de Barras

Como uma leitura de código de barras acontece muito rapidamente, o PDV pode diferenciar um código de barras de algo digitado manualmente.
Nas configurações do PDV, você encontrará opções para ajustar finamente como a detecção de código de barras funciona.

- **Tempo médio de entrada:** Quão rápida a entrada deve ser para ser considerada um código de barras.
- **Comprimento mínimo:** Qual o tamanho mínimo da sequência contínua de caracteres para ser tratada como um código de barras.
- **Remoção de prefixo/sufixo:** Se o seu leitor adicionar caracteres extras (como um prefixo ou sufixo), você pode removê-los automaticamente para que apenas o código de barras principal permaneça.

## O Que Acontece Quando um Código de Barras é Detectado?

Quando o PDV detecta um código de barras, ele procura em seu banco de dados local para encontrar um produto ou variação de produto correspondente.
Existem três possíveis resultados:

1. **Correspondência Exata (Um Produto Encontrado):**
   Se o código de barras corresponder exatamente a um único produto ou variação de produto no banco de dados local do PDV, esse produto é automaticamente adicionado ao seu carrinho.

2. **Múltiplas Correspondências (Mais de um Produto Encontrado):**
   Se mais de um produto tiver o mesmo código de barras, o PDV não pode saber qual adicionar.
   Neste caso, ele coloca o código de barras lido na barra de pesquisa para que você possa revisar quais produtos compartilham esse código.
   Quando você vê múltiplas correspondências, geralmente significa que os dados do produto precisam ser atualizados—cada produto deve ter um código de barras único.

3. **Sem Correspondências Locais (Zero Produtos Encontrados):**
   Se o código de barras não corresponder a nenhum produto armazenado no seu dispositivo, o código lido aparecerá na barra de pesquisa.
   Neste momento, o PDV tentará encontrar uma correspondência online através do banco de dados da sua loja WooCommerce.

## Entendendo a Sincronização de Produtos

### Download Progressivo de Produtos

O WCPOS não carrega todos os seus produtos de uma vez.
Em vez disso, ele baixa os produtos em pequenos lotes.
Esta abordagem evita atrasos e garante que sua loja funcione suavemente.
Com o tempo, à medida que você usa o PDV e realiza pesquisas, mais produtos são armazenados localmente no seu dispositivo.

Consulte [Sincronização de Produtos](/products/sync) para mais detalhes.

### Por Que Isso Importa para a Leitura de Código de Barras

Quando você lê um código de barras que ainda não está armazenado localmente, o PDV "irá online" para sua loja WooCommerce para encontrá-lo.
Como parte deste processo, ele baixará esse produto (e outros em pequenos lotes) e o salvará.
Isso significa que, com o tempo, o PDV se torna mais rápido e eficiente à medida que mais produtos são armazenados localmente.

### Como Acelerar o Processo

Simplesmente pesquisar produtos no PDV ajuda a baixar mais do seu inventário.
Quanto mais você usar a pesquisa — e quanto mais você ler — mais completo se torna o seu banco de dados local.

## Perguntas Frequentes

<details>

<summary>
  Por que recebo "0 produtos encontrados localmente" quando leio um código de barras?
</summary>

Nem todos os produtos estão disponíveis localmente desde o início.
O PDV baixa gradualmente os produtos da sua loja online e os armazena no seu dispositivo.
Se o produto que você acabou de ler ainda não está armazenado, a pesquisa faz o PDV procurá-lo online e baixá-lo para que esteja disponível no futuro.

</details>

<details>

<summary>O PDV gera e imprime códigos de barras?</summary>

Não, no momento não. Nosso PDV é projetado para ler códigos de barras existentes, mas não inclui funcionalidade para criá-los ou imprimi-los.
Se você precisar gerar códigos de barras para seus produtos, pode usar plugins de terceiros do WooCommerce que se especializam na criação e impressão de códigos de barras. Alguns exemplos incluem:

- [EAN for WooCommerce](https://wordpress.org/plugins/ean-for-woocommerce/)
- [A4 Barcode Generator](https://wordpress.org/plugins/a4-barcode-generator/)

Depois de gerar códigos de barras para seus produtos, você pode facilmente lê-los no caixa para acelerar o processo de checkout no PDV.

</details>
