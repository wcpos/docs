## Pesquisa de Produtos {#product-search}

### Campo de Pesquisa Unificado {#unified-search-field}

O WCPOS possui um único campo de pesquisa que busca simultaneamente em múltiplos atributos de produto:

- **Nome do Produto** - Busca no título e na descrição do produto
- **SKU** - Corresponde ao campo postmeta `_sku` do produto
- **Código de Barras** - Pesquisa no campo de código de barras configurado, que pode ser qualquer campo postmeta. A configuração antiga é `_sku`, mas isso será alterado para `_global_unique_id` no futuro.

Basta digitar seu termo de pesquisa no campo "Pesquisar Produtos", e o POS filtrará instantaneamente os resultados em todos esses campos.

### Tecnologia de Pesquisa Tokenizada {#tokenized-search-technology}

A funcionalidade de pesquisa utiliza a [biblioteca FlexSearch](https://github.com/nextapps-de/flexsearch) com avançadas capacidades de tokenização:

- **Tokenização Direta** - Corresponde a palavras parciais a partir do início (por exemplo, pesquisar "azul" encontrará "blueberry")
- **Otimizada para Performance** - Usa uma configuração de desempenho para resultados de pesquisa rápidos
- **Consciente da Linguagem** - Adapta-se à linguagem configurada da sua loja
- **Inicialização Preguiçosa** - Otimiza o uso de memória carregando índices de pesquisa apenas quando necessário

### Como Funciona a Pesquisa {#how-search-works}

Quando você digita no campo de pesquisa, o POS:

1. **Tokeniza** sua entrada em termos pesquisáveis
2. **Pesquisa dados de produtos** armazenados localmente primeiro para resultados instantâneos
3. **Consulta o servidor** se nenhuma correspondência local for encontrada, depois baixa e armazena novos produtos para pesquisas futuras
4. **Atualiza resultados** em tempo real enquanto você digita

Essa abordagem garante um desempenho de pesquisa rápido enquanto constrói progressivamente um banco de dados local completo de produtos.

## Filtragem de Produtos {#product-filtering}

### Barra de Filtros {#filter-bar}

Abaixo do campo de pesquisa, você encontrará alternâncias de filtro interativas e menus suspensos que permitem restringir os produtos por critérios específicos.

### Filtros Disponíveis {#available-filters}

#### Status do Estoque {#stock-status}
Filtre produtos com base em seu status de inventário:
- **Em Estoque** - Produtos com inventário disponível
- **Fora de Estoque** - Produtos sem inventário
- **Pré-encomenda** - Produtos disponíveis para pré-encomenda

#### Produtos em Destaque {#featured-products}
Alternância para mostrar apenas produtos marcados como "Destaque" em sua loja WooCommerce.

#### Produtos em Promoção {#on-sale-products}
Filtre para exibir apenas produtos atualmente em promoção ou com descontos ativos.

#### Categoria {#category}
Use o menu suspenso de categoria para filtrar produtos por suas categorias de produtos atribuídas. Isso ajuda você a encontrar rapidamente produtos dentro de departamentos ou linhas de produtos específicos.

#### Tag
Filtre por tags de produtos para encontrar itens com atributos ou características específicas que você definiu em sua loja WooCommerce.

### Usando Filtros {#using-filters}

- **Alternar Filtros** - Clique em qualquer botão de filtro para ativá-lo (filtros ativos aparecem destacados)
- **Múltiplos Filtros** - Você pode combinar vários filtros para restringir ainda mais sua pesquisa
- **Limpar Filtros** - Clique em um filtro ativo novamente para desativá-lo
- **Pesquisar + Filtrar** - Use filtros juntamente com o campo de pesquisa para localização precisa de produtos

## Configuração de Código de Barras {#barcode-configuration}

### Campos de Pesquisa {#search-fields}

A funcionalidade de pesquisa inclui automaticamente seu campo de código de barras configurado. O campo de código de barras usado para pesquisa depende da configuração das suas configurações POS.

## F.A.Q.

<Accordion>
  <AccordionItem question="Para que serve o campo _global_unique_id para códigos de barras?">

O campo `_global_unique_id` é um novo campo de código de barras que o WooCommerce adicionou recentemente para fornecer melhor padronização de códigos de barras em lojas.

**Pontos Principais:**

- **Padrão Moderno**: Este campo foi projetado especificamente para identificação global de códigos de barras
- **Configuração do POS**: Você pode configurar o POS para usar `_global_unique_id` como campo de código de barras nas configurações do POS
- **Legado vs. Novo**: A configuração de código de barras legado usa o campo `_sku`, mas isso mudará para `_global_unique_id` em versões futuras
- **Flexibilidade**: Você pode configurar qualquer campo meta de produto como seu campo de código de barras se estiver usando plugins de códigos de barras de terceiros
- **Padrão Futuro**: `_global_unique_id` se tornará o campo padrão de código de barras em futuras atualizações do POS

Para configurar qual campo o POS usa para códigos de barras, visite suas configurações de POS na área administrativa do WordPress.

  </AccordionItem>

  <AccordionItem question="Por que não vejo todos os meus produtos quando procuro?">

O WCPOS usa download progressivo de produtos para manter o desempenho. Se você não vê um produto:

1. **Tente pesquisá-lo** - Isso acionará o POS para procurá-lo em seu servidor
2. **Aguarde o download** - O POS baixará o produto e outros em pequenos lotes
3. **Pesquise novamente** - Uma vez baixado, o produto aparecerá em pesquisas futuras

Esse processo garante que seu POS permaneça rápido e responsivo, mesmo com milhares de produtos. Saiba mais sobre isso em nosso guia de [Sincronização de Produtos](/products/sync).

  </AccordionItem>

  <AccordionItem question="Posso pesquisar por nomes de produtos ou SKUs parciais?">

Sim! A pesquisa tokenizada usa correspondência direta, o que significa:

- Pesquisar "azul" encontrará produtos com "blueberry", "camisa azul", etc.
- Pesquisar "ABC" encontrará SKUs como "ABC123", "ABC-XYZ", etc.
- Você não precisa digitar palavras ou códigos completos

A pesquisa é projetada para encontrar produtos rapidamente com digitação mínima.

  </AccordionItem>
</Accordion>
