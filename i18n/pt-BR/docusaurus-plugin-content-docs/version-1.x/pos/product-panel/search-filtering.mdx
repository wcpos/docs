## Pesquisa de Produtos

### Campo de Pesquisa Unificado

O WCPOS possui um único campo de pesquisa que busca simultaneamente em vários atributos de produtos:

- **Nome do Produto** - Busca no título e na descrição do produto
- **SKU** - Corresponde ao campo postmeta `_sku` do produto
- **Código de Barras** - Busca no campo de código de barras configurado, que pode ser qualquer campo postmeta. A configuração legada é `_sku`, mas isso será alterado para `_global_unique_id` no futuro.

Basta digitar seu termo de pesquisa no campo "Pesquisar Produtos", e o POS filtrará instantaneamente os resultados em todos esses campos.

### Tecnologia de Pesquisa Tokenizada

A funcionalidade de pesquisa utiliza a [biblioteca FlexSearch](https://github.com/nextapps-de/flexsearch) com capacidades avançadas de tokenização:

- **Tokenização Direcionada** - Corresponde palavras parciais desde o início (por exemplo, pesquisar "azul" encontrará "mirtilo")
- **Otimização de Performance** - Utiliza uma pré-configuração de performance para resultados de pesquisa rápidos
- **Consciente de Idioma** - Adapta-se ao idioma configurado da sua loja
- **Inicialização Lenta** - Otimiza o uso de memória carregando índices de pesquisa somente quando necessário

### Como a Pesquisa Funciona

Quando você digita no campo de pesquisa, o POS:

1. **Tokeniza** sua entrada em termos pesquisáveis
2. **Busca** primeiro nos dados de produtos armazenados localmente para resultados instantâneos
3. **Consulta o servidor** se nenhuma correspondência local for encontrada e, em seguida, baixa e armazena novos produtos para futuras pesquisas
4. **Atualiza resultados** em tempo real enquanto você digita

Essa abordagem garante um desempenho rápido de pesquisa enquanto constrói progressivamente um banco de dados completo de produtos local.

## Filtragem de Produtos

### Barra de Filtros

Abaixo do campo de pesquisa, você encontrará alternâncias de filtros interativas e menus suspensos que permitem restringir os produtos por critérios específicos.

### Filtros Disponíveis

#### Status do Estoque
Filtre produtos com base no status do seu estoque:
- **Em Estoque** - Produtos com estoque disponível
- **Fora de Estoque** - Produtos com estoque zero
- **Sob Encomenda** - Produtos disponíveis para encomenda

#### Produtos em Destaque
Ative para mostrar apenas produtos marcados como "Em Destaque" na sua loja WooCommerce.

#### Produtos em Promoção  
Filtre para exibir apenas produtos atualmente em promoção ou com descontos ativos.

#### Categoria
Use o menu suspenso de categoria para filtrar produtos por suas categorias de produtos atribuídas. Isso ajuda a encontrar rapidamente produtos dentro de departamentos ou linhas de produtos específicos.

#### Tag
Filtre por tags de produtos para encontrar itens com atributos ou características específicas que você definiu em sua loja WooCommerce.

### Usando Filtros

- **Alternar Filtros** - Clique em qualquer botão de filtro para ativá-lo (filtros ativos aparecem em destaque)
- **Múltiplos Filtros** - Você pode combinar vários filtros para restringir ainda mais sua pesquisa
- **Limpar Filtros** - Clique novamente em um filtro ativo para desativá-lo
- **Pesquisar + Filtrar** - Use filtros juntos com o campo de pesquisa para uma localização precisa de produtos

## Configuração de Código de Barras

### Campos de Pesquisa

A funcionalidade de pesquisa inclui automaticamente o campo de código de barras configurado. O campo de código de barras usado para pesquisa depende da configuração de suas definições de POS.

## Dúvidas Frequentes

<details>

<summary>
  Qual é o campo _global_unique_id para códigos de barras?
</summary>

O campo `_global_unique_id` é um novo campo de código de barras que o WooCommerce adicionou recentemente para fornecer melhor padronização de código de barras entre as lojas.

**Principais Pontos:**

- **Padrão Moderno**: Este campo foi projetado especificamente para identificação global de código de barras
- **Configuração de POS**: Você pode configurar o POS para usar `_global_unique_id` como o campo de código de barras nas configurações do POS
- **Legado vs. Novo**: A configuração legada de código de barras utiliza o campo `_sku`, mas isso mudará para `_global_unique_id` em versões futuras
- **Flexibilidade**: Você pode configurar qualquer campo meta de produto como seu campo de código de barras se estiver usando plugins de código de barras de terceiros
- **Padrão Futuro**: `_global_unique_id` se tornará o campo de código de barras padrão em futuras atualizações do POS

Para configurar qual campo o POS usa para códigos de barras, visite suas configurações de POS na área administrativa do WordPress.

</details>

<details>

<summary>
  Por que não vejo todos os meus produtos quando pesquiso?
</summary>

O WCPOS utiliza download progressivo de produtos para manter o desempenho. Se você não vê um produto:

1. **Tente pesquisá-lo** - Isso fará com que o POS procure por ele em seu servidor
2. **Aguarde o download** - O POS fará o download do produto e outros em pequenos lotes
3. **Pesquise novamente** - Uma vez baixado, o produto aparecerá em futuras pesquisas

Esse processo garante que seu POS permaneça rápido e responsivo, mesmo com milhares de produtos. Saiba mais sobre isso em nosso guia de [Sincronização de Produtos](/products/sync).

</details>

<details>

<summary>
  Posso pesquisar por nomes de produtos ou SKUs parciais?
</summary>

Sim! A pesquisa tokenizada usa correspondência direta, o que significa:

- Pesquisar "azul" encontrará produtos como "mirtilo", "camisa azul", etc.
- Pesquisar "ABC" encontrará SKUs como "ABC123", "ABC-XYZ", etc.
- Você não precisa digitar palavras ou códigos completos

A pesquisa é projetada para encontrar produtos rapidamente com digitação mínima.

</details>
