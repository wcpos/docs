## Pesquisa de Produtos

### Campo de Pesquisa Unificado

O WCPOS possui um único campo de pesquisa que busca simultaneamente em vários atributos de produtos:

- **Nome do Produto** - Busca no título e descrição do produto
- **SKU** - Corresponde ao campo postmeta `_sku`
- **Código de Barras** - Busca no campo de código de barras configurado, que pode ser qualquer campo postmeta. A configuração atual é `_sku`, mas será alterada para `_global_unique_id` no futuro.

Basta digitar seu termo de pesquisa no campo "Pesquisar Produtos" que o PDV filtrará instantaneamente os resultados em todos esses campos.

### Tecnologia de Tokenização de Pesquisa

A funcionalidade de pesquisa usa a [biblioteca FlexSearch](https://github.com/nextapps-de/flexsearch) com recursos avançados de tokenização:

- **Tokenização Progressiva** - Corresponde a palavras parciais desde o início (por exemplo, pesquisar "azul" encontrará "ameixa-azul")
- **Otimizado para Desempenho** - Usa uma predefinição de desempenho para resultados de busca rápidos
- **Ciente de Idioma** - Adapta-se ao idioma configurado em sua loja
- **Inicialização Preguiçosa** - Otimiza o uso de memória carregando índices de pesquisa apenas quando necessário

### Como a Pesquisa Funciona

Quando você digita no campo de pesquisa, o PDV:

1. **Tokeniza** sua entrada em termos pesquisáveis
2. **Pesquisa primeiro** nos dados de produtos armazenados localmente para resultados instantâneos
3. **Consulta o servidor** se nenhuma correspondência local for encontrada, então baixa e armazena novos produtos para futuras pesquisas
4. **Atualiza resultados** em tempo real enquanto você digita

Esta abordagem garante desempenho de pesquisa rápido enquanto constrói progressivamente um banco de dados de produtos local completo.

## Filtragem de Produtos

### Barra de Filtros

Abaixo do campo de pesquisa, você encontrará alternadores de filtro interativos e menus suspensos que permitem restringir produtos por critérios específicos.

### Filtros Disponíveis

#### Status de Estoque
Filtre produtos com base em seu status de inventário:
- **Em Estoque** - Produtos com estoque disponível
- **Fora de Estoque** - Produtos com estoque zero
- **Pedido em Atraso** - Produtos disponíveis para pedido em atraso

#### Produtos Destacados
Alternar para mostrar apenas produtos marcados como "Destacados" em sua loja WooCommerce.

#### Produtos em Promoção
Filtre para exibir apenas produtos atualmente em promoção ou com descontos ativos.

#### Categoria
Use o menu suspenso de categoria para filtrar produtos por suas categorias de produtos atribuídas. Isso ajuda a encontrar rapidamente produtos dentro de departamentos ou linhas de produtos específicos.

#### Etiqueta
Filtre por etiquetas de produtos para encontrar itens com atributos ou características específicas definidos em sua loja WooCommerce.

### Usando Filtros

- **Alternar Filtros** - Clique em qualquer botão de filtro para ativá-lo (filtros ativos aparecem destacados)
- **Múltiplos Filtros** - Você pode combinar vários filtros para refinar ainda mais sua busca
- **Limpar Filtros** - Clique em um filtro ativo novamente para desativá-lo
- **Pesquisa + Filtro** - Use filtros junto com o campo de pesquisa para localização precisa de produtos

## Configuração de Código de Barras

### Campos de Pesquisa

A funcionalidade de pesquisa inclui automaticamente seu campo de código de barras configurado. O campo de código de barras usado para pesquisa depende da configuração das suas definições de PDV.

## Perguntas Frequentes

<details>

<summary>
  O que é o campo _global_unique_id para códigos de barras?
</summary>

O campo `_global_unique_id` é um novo campo de código de barras que o WooCommerce adicionou recentemente para fornecer melhor padronização de códigos de barras entre lojas.

**Pontos Principais:**

- **Padrão Moderno**: Este campo foi projetado especificamente para identificação global de códigos de barras
- **Configuração de PDV**: Você pode configurar o PDV para usar `_global_unique_id` como campo de código de barras nas configurações de PDV
- **Legado vs. Novo**: A configuração de código de barras antiga usa o campo `_sku`, mas isso mudará para `_global_unique_id` em versões futuras
- **Flexibilidade**: Você pode configurar qualquer campo de meta de produto como seu campo de código de barras se estiver usando plugins de código de barras de terceiros
- **Padrão Futuro**: `_global_unique_id` será o campo de código de barras padrão em futuras atualizações de PDV

Para configurar qual campo o PDV usa para códigos de barras, visite as configurações de PDV na área de administração do WordPress.

</details>

<details>

<summary>
  Por que não vejo todos os meus produtos na pesquisa?
</summary>

O WCPOS usa download progressivo de produtos para manter o desempenho. Se você não encontra um produto:

1. **Tente pesquisá-lo** - Isso fará o PDV procurar no seu servidor
2. **Aguarde o download** - O PDV baixará o produto e outros em pequenos lotes
3. **Pesquise novamente** - Uma vez baixado, o produto aparecerá em pesquisas futuras

Este processo garante que seu PDV permaneça rápido e responsivo, mesmo com milhares de produtos. Saiba mais sobre isso no nosso guia de [Sincronização de Produtos](/products/sync).

</details>

<details>

<summary>
  Posso pesquisar nomes ou SKUs parciais de produtos?
</summary>

Sim! A pesquisa tokenizada usa correspondência progressiva, o que significa:

- Pesquisar "azul" encontrará produtos com "ameixa-azul", "camisa azul", etc.
- Pesquisar "ABC" encontrará SKUs como "ABC123", "ABC-XYZ", etc.
- Você não precisa digitar palavras ou códigos completos

A pesquisa é projetada para encontrar produtos rapidamente com o mínimo de digitação.

</details>
