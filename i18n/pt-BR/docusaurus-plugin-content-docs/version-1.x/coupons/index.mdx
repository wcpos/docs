:::info Recurso Pro
A tela de Cupons e a capacidade de aplicar cupons no caixa requerem o [WCPOS Pro](/getting-started/pro-license). Usuários gratuitos veem uma prévia desfocada da tela de Cupons, mas não podem visualizar detalhes dos cupons nem aplicar códigos no carrinho.
:::

A tela de Cupons permite navegar e consultar os cupons do WooCommerce diretamente no POS, e o botão **Adicionar Cupom** no carrinho os aplica ao pedido atual. Os cupons continuam sendo criados e gerenciados no WooCommerce — o POS os sincroniza e valida localmente para feedback instantâneo no caixa.

## Visão Geral da Interface {#interface-overview}

### Ações do Cabeçalho {#header-actions}

No topo da tela:

- **Barra de pesquisa** — Encontre um cupom pelo código
- **Configurações de exibição** (<Icon name="sliders" />) — Configurar colunas visíveis

### Tabela de Cupons {#coupons-table}

A área principal exibe os cupons sincronizados com:

- **Código** — O código do cupom inserido pelos clientes
- **Descrição** — Descrição interna (também usada como rótulo no recibo quando definida)
- **Tipo de Desconto** — Porcentagem, valor fixo no carrinho ou valor fixo no produto
- **Valor** — O valor do desconto
- **Usado / Limite** — Vezes utilizado e limite total de uso
- **Expira** — Data de expiração, ou em branco se não houver
- **Ações** — Menu de três pontos

### Rodapé {#footer}

- Contagem de cupons com botão de sincronização (<Icon name="arrow-rotate-right" />). **Pressão longa** para opções de Limpar e Atualizar.

## Tipos de Cupons {#coupon-types}

O WCPOS suporta os três tipos padrão de cupons do WooCommerce:

| Tipo | O que faz | Exemplo |
|---|---|---|
| **Desconto percentual** | Reduz o subtotal do carrinho por uma porcentagem | `10%` de desconto no pedido |
| **Desconto fixo no carrinho** | Reduz o total do carrinho por um valor fixo | `$5` de desconto no pedido |
| **Desconto fixo no produto** | Reduz os itens de produto correspondentes por um valor fixo por unidade | `$2` de desconto em cada camiseta |

Os três tipos respeitam as mesmas regras de validação (validade, limites de uso, restrições de produto/categoria, etc.).

## Criando Cupons {#creating-coupons}

Os cupons são criados no WooCommerce, não no POS. Vá para **WP Admin → Marketing → Cupons → Adicionar cupom**:

1. **Código do cupom** — o que os operadores de caixa digitam no registro. Os códigos não diferenciam maiúsculas de minúsculas.
2. **Descrição** — nota interna. O WCPOS utiliza este campo como rótulo do desconto nos recibos quando preenchido, portanto prefira uma frase amigável ao cliente, como "Desconto Gerente 10%", em vez de um código interno.
3. **Tipo de desconto** — porcentagem, valor fixo no carrinho ou valor fixo no produto.
4. **Valor do cupom** — o valor do desconto.
5. **Data de expiração** — opcional. Após esta data, o cupom será recusado.

### Restrição de uso {#usage-restriction}

- **Gasto mínimo / máximo** — o cupom só se aplica acima/abaixo de um limite de subtotal.
- **Uso individual apenas** — impede que este cupom seja combinado com qualquer outro cupom.
- **Excluir itens em promoção** — ignora itens que já estão em promoção (e qualquer linha em que o operador de caixa tenha reduzido o preço no ponto de venda — consulte [Substituições de preço no POS](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)).
- **Produtos / Excluir produtos** — restringe o cupom a (ou exclui de) produtos específicos.
- **Categorias de produtos / Excluir categorias** — restringe por categoria. Produtos diversos/personalizados criados no POS também respeitam as categorias atribuídas pelo POS.
- **E-mails permitidos** — limita o cupom a endereços de e-mail específicos de clientes (suporta curingas `*`).

### Limites de uso {#usage-limits}

- **Limite de uso por cupom** — número total de vezes que este código pode ser utilizado entre todos os clientes.
- **Limite de uso por usuário** — número de vezes que um cliente pode utilizá-lo.
- **Limitar uso a X itens** — para cupons de produto fixo, limita o número de itens qualificados com desconto por uso.

### Dicas para uso no POS {#tips-for-pos-use}

- Crie um cupom **"Gerente 10%"** (10% de desconto, sem valor mínimo de compra, sem data de expiração, restrição de uso único desativada) e forneça o código aos gerentes — a equipe pode aplicá-lo no caixa para ajustes pontuais e você mantém um desconto rastreado nos relatórios.
- Crie um cupom **"Fidelidade R$5"** para recompensas de clientes recorrentes.
- Para promoções pontuais, defina uma data de expiração curta para que o código não seja reutilizado por engano.
- Definir a **Descrição** com a frase que deseja imprimir nos recibos é a maneira mais rápida de personalizar seus descontos.

## Aplicando um Cupom no Caixa {#applying-a-coupon-at-the-register}

Toque em **Adicionar Cupom** no carrinho, depois digite o código ou pesquise pela descrição. O cupom é validado instantaneamente com os dados sincronizados e aparece como uma etiqueta removível acima dos totais do carrinho.

Para o fluxo completo do caixa — pesquisa, etiquetas de cupom, acúmulo de múltiplos cupons e resolução de erros de validação — consulte **[Aplicando Cupons no Caixa](/coupons/applying-coupons)**.

## Como Funciona a Validação {#how-validation-works}

Quando um código é inserido, o POS verifica todas as mesmas regras que o WooCommerce verificaria no servidor:

- O cupom existe e não está expirado
- Os limites de uso não foram excedidos (geral e por usuário)
- O valor mínimo/máximo de compra foi atingido
- Cupons de "uso individual" não entram em conflito com cupons já aplicados
- As restrições de produto/categoria correspondem ao conteúdo do carrinho
- As restrições de e-mail correspondem ao cliente selecionado (ou são ignoradas em pedidos de visitante)
- Itens em promoção são excluídos se **Excluir itens em promoção** estiver ativado

Se alguma verificação falhar, o operador de caixa verá uma mensagem de erro específica (por exemplo, "Este cupom expirou" ou "Valor mínimo de compra não atingido"). As mesmas verificações são executadas novamente no servidor quando o pedido é enviado — a validação local é para agilidade, não para confiança.

## Comportamento de Sincronização {#sync-behaviour}

Os cupons são sincronizados do WooCommerce para o dispositivo por meio do pipeline de replicação padrão do WCPOS:

- Novos cupons criados no WP Admin aparecem no POS na próxima sincronização.
- Atualizações em cupons existentes (contagem de uso, alterações de validade, etc.) são sincronizadas automaticamente.
- Os cupons são armazenados localmente no IndexedDB para que permaneçam disponíveis offline.

Se você acabou de criar um cupom no WP Admin e ainda não o vê, a sincronização pode não ter sido executada. No rodapé da tela de Cupons, toque no ícone de sincronização (<Icon name="arrow-rotate-right" />) para atualizar — mantenha pressionado para **Limpar e atualizar** se precisar de uma busca completa.

## Conectividade {#connectivity}

- **Aplicar cupons** funciona offline porque a validação é feita no lado do cliente a partir dos dados sincronizados.
- **Finalizar a compra** ainda requer uma conexão com o servidor — o servidor revalida e é a fonte oficial do total final do pedido.
- **Criar ou editar cupons** acontece no WP Admin e requer uma conexão com o seu site WordPress.

## Como os Cupons Interagem com os Descontos do POS {#how-coupons-interact-with-pos-discounts}

Quando um operador de caixa reduz o preço de um item no caixa e um cupom é então aplicado, o cupom é calculado sobre o **preço reduzido**, e não sobre o original. Itens com preço reduzido no POS são tratados como "em promoção" — cupons com **Excluir itens em promoção** ativado irão ignorá-los. Consulte [Descontos](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons) para mais informações.
