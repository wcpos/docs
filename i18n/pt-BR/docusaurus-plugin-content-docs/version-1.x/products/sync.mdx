Um dos aspectos significativos do sistema WCPOS (Ponto de Venda) é sua abordagem à gestão de dados. Ele fornece uma maneira eficiente de lidar com um grande volume de dados de produtos. Este guia explica como os produtos são baixados e sincronizados dentro do WCPOS.

## Armazenamento Local de Produtos {#local-storage-of-products}

O WCPOS busca produtos da sua loja WooCommerce e os armazena localmente no seu dispositivo. Essa abordagem significa que, uma vez que um produto é baixado, não precisa ser buscado repetidamente. À medida que você continua usando o POS, ele progressivamente baixa todos os produtos da sua loja WooCommerce, criando um banco de dados local dos seus produtos.

## Download em Lote de Produtos {#batch-downloading-of-products}

Para a maioria das lojas, tentar baixar mais de cerca de 50 produtos de uma só vez pode levar a uma alta carga no servidor, tempos de resposta mais lentos ou até mesmo quedas do servidor. Para evitar isso, o WCPOS foi projetado para baixar produtos em pequenos lotes, geralmente em torno de 50 produtos por vez. Esse processo continua até que todo o inventário tenha sido armazenado localmente, garantindo que um grande volume de dados possa ser gerenciado sem colocar estresse excessivo no seu servidor.

## Usando a Pesquisa de Produtos {#using-the-product-search}

Uma maneira eficaz de iniciar o download em lote de produtos é usando a função de pesquisa de produtos. Cada vez que você faz uma pesquisa de produtos, o sistema POS busca e armazena mais um lote de produtos. Essa estratégia não apenas ajuda a baixar gradualmente novos produtos, mas também melhora a velocidade e a eficiência da função de pesquisa à medida que mais produtos são armazenados localmente.

## Conclusão {#conclusion}

É importante compreender que todos os seus produtos não aparecerão de uma só vez no seu sistema POS, e isso é completamente normal. Com o tempo, usando a pesquisa de produtos e outras operações do POS, você baixará todos os seus produtos. Para informações mais detalhadas sobre como o WCPOS opera, veja [Arquitetura](/reference/architecture).

O WCPOS foi projetado para otimizar a carga do servidor e a eficiência operacional. A funcionalidade de download progressivo de produtos garante uma experiência de gestão de varejo suave, eficiente e amigável ao usuário.

## F.A.Q.

<Accordion>
  <AccordionItem question="Como faço para carregar mais produtos?">

  A sincronização de produtos no POS pode ser acionada de várias maneiras:

  - Quando o POS é carregado pela primeira vez.
  - Cada vez que você executa uma pesquisa de produtos.
  - Aplicando um filtro, por exemplo, por categoria ou etiqueta.
  - Rolando até o final da lista de produtos.
  - Pressionando o botão de sincronização manualmente (pressão curta para sincronizar, **pressão longa** para a opção Limpar e Atualizar).
  - Além disso, o POS sincronizará automaticamente a cada 10 minutos.

  Uma vez que todos os produtos sejam baixados, sincronizações futuras buscarão apenas os produtos que foram atualizados desde a última sincronização.

  </AccordionItem>

  <AccordionItem question="Eu rolei até o final e ainda não está mostrando todos os produtos">

  Podem haver algumas razões pelas quais nem todos os produtos estão sendo exibidos:

  - O POS ocultará itens fora de estoque por padrão. Essa configuração pode ser alterada nas configurações de exibição de produtos se você desejar mostrar itens fora de estoque.
  - A API REST do WooCommerce é compatível apenas com os tipos de produtos padrão do WooCommerce, que são simples, variáveis, agrupados e externos. Se você estiver usando um tipo de produto personalizado, eles podem não ser exibidos no POS.

  </AccordionItem>
</Accordion>
