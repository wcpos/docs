Um dos aspectos mais importantes do sistema WCPOS (Ponto de Venda) é sua abordagem ao gerenciamento de dados. Ele oferece uma maneira eficiente de lidar com um grande volume de dados de produtos. Este guia explica como os produtos são baixados e sincronizados no WCPOS.

## Armazenamento Local de Produtos {#local-storage-of-products}

O WCPOS busca produtos da sua loja WooCommerce e os armazena localmente no seu dispositivo. Essa abordagem significa que, uma vez que um produto é baixado, ele não precisa ser buscado repetidamente. Conforme você continua usando o POS, ele baixará progressivamente todos os produtos da sua loja WooCommerce, criando um banco de dados local dos seus produtos.

## Download de Produtos em Lotes {#batch-downloading-of-products}

Para a maioria das lojas, tentar baixar mais de cerca de 50 produtos de uma vez pode gerar alta carga no servidor, tempos de resposta mais lentos ou até mesmo falhas no servidor. Para evitar isso, o WCPOS é projetado para baixar produtos em pequenos lotes, geralmente cerca de 50 produtos por vez. Esse processo continua até que todo o inventário tenha sido armazenado localmente, garantindo que um grande volume de dados possa ser gerenciado sem sobrecarregar o servidor.

## Usando a Busca de Produtos {#using-the-product-search}

Uma maneira eficaz de iniciar o download em lotes de produtos é usando a função de busca de produtos. Cada vez que uma busca de produtos é realizada, o sistema POS busca e armazena outro lote de produtos. Essa estratégia não apenas ajuda a baixar gradualmente novos produtos, mas também melhora a velocidade e a eficiência da função de busca à medida que mais produtos são armazenados localmente.

## Conclusão {#conclusion}

É importante entender que todos os seus produtos não aparecerão de uma vez no sistema POS, e isso é completamente normal. Com o tempo, usando a busca de produtos e outras operações do POS, todos os seus produtos serão baixados. Para informações mais detalhadas sobre como o WCPOS funciona, consulte [Arquitetura](/reference/architecture).

O WCPOS foi projetado para otimizar a carga do servidor e a eficiência operacional. O recurso de download progressivo de produtos garante uma experiência de gestão de varejo fluida, eficiente e fácil de usar.

:::tip
Se você estiver enfrentando problemas persistentes de sincronização que uma sincronização normal não consegue resolver, é possível [limpar todos os dados locais](/support/troubleshooting/clear-local-data) para forçar um novo download completo da sua loja.
:::

## Perguntas Frequentes {#faq}

<Accordion>
  <AccordionItem question="Como faço para carregar mais produtos?">

  A sincronização de produtos no POS pode ser acionada de várias maneiras:

  - Quando o POS é carregado pela primeira vez.
  - Cada vez que você realiza uma busca de produtos.
  - Ao aplicar um filtro, por exemplo, por categoria ou tag.
  - Ao rolar até o final da lista de produtos.
  - Ao pressionar o botão de sincronização manualmente (toque curto para sincronizar, **toque longo** para a opção Limpar e Atualizar).
  - Além disso, o POS sincroniza automaticamente a cada 5 minutos.

  Depois que todos os produtos forem baixados, as sincronizações futuras buscarão apenas os produtos que foram atualizados desde a última sincronização.

  </AccordionItem>

  <AccordionItem question="Rolei até o final e ainda não mostra todos os produtos">

  Pode haver alguns motivos pelos quais nem todos os produtos estão sendo exibidos:

  - O POS oculta itens fora de estoque por padrão. Essa configuração pode ser alterada nas configurações de exibição de produtos, caso deseje mostrar itens fora de estoque.
  - A API REST do WooCommerce é compatível apenas com os tipos de produto padrão do WooCommerce, que são simples, variável, agrupado e externo. Se você estiver usando um tipo de produto personalizado, ele pode não ser exibido no POS.

  </AccordionItem>
</Accordion>
