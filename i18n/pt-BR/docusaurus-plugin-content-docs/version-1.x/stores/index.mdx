:::info Recurso Pro
A funcionalidade de múltiplas lojas requer o [WCPOS Pro](/getting-started/pro-license).
:::

O WCPOS Pro permite criar lojas separadas dentro de uma única instalação do WooCommerce. Cada loja pode ter seu próprio endereço, logotipo e configurações — útil se você opera em múltiplos locais físicos.

## O que as Lojas Oferecem {#what-stores-provide}

Cada loja pode ter:

- **Nome e endereço da loja** — usados em recibos e relatórios
- **Logotipo da loja** — exibido em recibos impressos
- **Alíquotas de impostos** — diferentes locais podem ter diferentes requisitos fiscais
- **Atribuição de operadores de caixa** — acompanhe quais operadores de caixa trabalham em cada loja

## Configurando Lojas {#setting-up-stores}

Ao ativar o Pro, um submenu **Lojas** é adicionado em POS no WP Admin. A partir dele, você cria cada loja, define endereço, logotipo, identificações fiscais e quais operadores de caixa podem fazer login — consulte **[Configurando Lojas](/stores/setup)** para o passo a passo completo.

Quando um operador de caixa faz login no POS, ele seleciona a loja em que está operando. Isso determina os dados da loja exibidos nos recibos e qual filtro de loja é aplicado aos relatórios.

## Filtrando por Loja {#filtering-by-store}

Depois de configurar várias lojas, o filtro de loja aparece em todo o POS:

- **[Pedidos](/orders)** — filtre o histórico de pedidos por localização da loja
- **[Relatórios](/reports)** — gere relatórios de fechamento de caixa para uma loja específica
- **[Recibos](/receipts/at-checkout)** — os recibos exibem automaticamente o endereço e o logotipo corretos da loja

## Preços por Loja {#store-specific-pricing}

O WCPOS Pro permite **preços diferentes para diferentes lojas**. É possível definir um preço normal e um preço promocional separados por loja, tanto para produtos quanto para variações.

Quando um operador de caixa está operando em uma loja específica, o POS utiliza automaticamente os preços daquela loja. Se nenhum preço específico estiver definido, o preço padrão do WooCommerce é utilizado.

## Gerenciamento de Estoque {#inventory-management}

Por padrão, todas as lojas compartilham os mesmos níveis de estoque do WooCommerce. Para rastreamento de estoque por localização, o WCPOS Pro integra-se com o plugin [ATUM Inventory Management](https://www.atum.org/):

1. Instale e configure o plugin ATUM no seu site WooCommerce
2. Crie localizações de estoque no ATUM
3. No WCPOS Pro, associe cada loja a uma localização de estoque do ATUM
4. Os níveis de estoque serão atualizados para a localização correta quando os pedidos forem realizados

### Alternativa: WordPress Multisite {#alternative-wordpress-multisite}

Para estoques completamente separados sem o ATUM, você pode usar o [WordPress Multisite](https://developer.wordpress.org/advanced-administration/multisite/create-network/):

1. Crie um site WordPress separado para cada localização de loja
2. Instale o WooCommerce e o WCPOS em cada site
3. Cada site mantém seu próprio catálogo de produtos e níveis de estoque independentes

Isso proporciona separação completa, mas significa gerenciar múltiplas instalações do WooCommerce.

## Limitações Conhecidas {#known-limitations}

- **Instância única do WooCommerce** — as lojas são locais dentro de uma única instalação do WooCommerce, não lojas separadas
- **Sem visibilidade de produtos por loja** — todas as lojas veem o mesmo catálogo de produtos
