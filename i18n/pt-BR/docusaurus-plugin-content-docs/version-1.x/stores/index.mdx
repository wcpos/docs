:::info Recurso Pro
A funcionalidade de multilojas requer o [WCPOS Pro](/getting-started/pro-license).
:::

O WCPOS Pro permite criar lojas separadas dentro de uma única instalação do WooCommerce. Cada loja pode ter seu próprio endereço, logotipo e configurações — útil para quem opera em vários locais físicos.

## O que as lojas oferecem {#what-stores-provide}

Cada loja pode ter:

- **Nome e endereço da loja** — usados em recibos e relatórios
- **Logotipo da loja** — exibido em recibos impressos
- **Alíquotas de imposto** — locais diferentes podem ter exigências fiscais diferentes
- **Atribuição de caixas** — acompanhe quais caixas trabalham em cada loja

## Configurando lojas {#setting-up-stores}

1. Acesse `WP Admin > POS > Settings`
2. Crie uma nova loja com nome, endereço e logotipo
3. Configure quaisquer ajustes de impostos específicos da loja
4. Atribua a loja aos usuários do POS

Quando um caixa faz login no POS, ele seleciona em qual loja está operando. Isso determina os dados da loja exibidos nos recibos e qual filtro de loja é aplicado aos relatórios.

## Filtrando por loja {#filtering-by-store}

Depois que várias lojas estiverem configuradas, o filtro de loja aparece em todo o POS:

- **[Pedidos](/orders)** — filtre o histórico de pedidos pela localização da loja
- **[Relatórios](/reports)** — gere relatórios de fechamento do dia para uma loja específica
- **[Recibos](/pos/checkout/receipts)** — os recibos exibem automaticamente o endereço e o logotipo corretos da loja

## Preços por loja {#store-specific-pricing}

O WCPOS Pro oferece suporte a **preços diferentes para diferentes lojas**. É possível definir um preço normal e um preço promocional separados por loja, tanto para produtos quanto para variações.

Quando um caixa está operando a partir de uma loja específica, o POS utiliza automaticamente os preços daquela loja. Se nenhum preço específico da loja estiver definido, será usado o preço padrão do WooCommerce.

## Gestão de estoque {#inventory-management}

Por padrão, todas as lojas compartilham os mesmos níveis de estoque do WooCommerce. Para o controle de estoque por localidade, o WCPOS Pro integra-se ao plugin [ATUM Inventory Management](https://www.atum.org/):

1. Instale e configure o plugin ATUM no seu site WooCommerce
2. Crie locais de estoque no ATUM
3. No WCPOS Pro, associe cada loja a um local de estoque do ATUM
4. Os níveis de estoque serão atualizados para a localidade correta quando os pedidos forem realizados

### Alternativa: WordPress Multisite {#alternative-wordpress-multisite}

Para estoques totalmente separados sem o ATUM, você pode usar o [WordPress Multisite](https://developer.wordpress.org/advanced-administration/multisite/create-network/):

1. Crie um site WordPress separado para cada loja
2. Instale o WooCommerce e o WCPOS em cada site
3. Cada site mantém seu próprio catálogo de produtos e níveis de estoque independentes

Isso oferece separação completa, mas implica gerenciar várias instalações do WooCommerce.

## Limitações conhecidas {#known-limitations}

- **Instância única do WooCommerce** — as lojas são locais dentro de uma única instalação do WooCommerce, não lojas separadas
- **Sem visibilidade de produtos por loja** — todas as lojas veem o mesmo catálogo de produtos
