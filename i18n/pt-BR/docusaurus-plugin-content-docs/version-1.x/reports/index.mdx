:::info Recurso Pro
A tela de Relatórios requer [WCPOS Pro](/getting-started/pro-license).
:::

A tela de Relatórios fornece ferramentas de relatórios de vendas e conciliação diária. É projetada principalmente para **relatórios de fim de dia** para ajudar caixas e gerentes a fecharem o dia.

{/* Screenshot: reports-overview.png */}

:::tip Análises Abrangentes
Para análises detalhadas em qualquer intervalo de datas, use **WP Admin > Analytics**. A tela de Relatórios do POS é otimizada para resumos rápidos de fim de dia e conciliação diária.
:::

## Visão Geral da Interface

### Filtros

No topo da tela:

- **Filtro de Status** - Filtrar por status do pedido (Concluído, Processando, etc.)
- **Filtro de Cliente** - Filtrar por cliente
- **Filtro de Caixa** - Filtrar por quem processou os pedidos
- **Filtro de Loja** - Filtrar por loja (configurações de múltiplas lojas)
- **Intervalo de Datas** - Selecionar o período do relatório (Hoje, Esta Semana, Personalizado)

### Layout de Três Painéis

A tela de Relatórios é dividida em três áreas:

#### 1. Gráfico de Vendas (Topo)

Um gráfico de barras mostrando vendas ao longo do tempo:

- Detalhamento por hora para relatórios diários
- Representação visual de padrões de vendas
- Ajuda a identificar horários de pico de vendas

#### 2. Tabela de Pedidos (Inferior Esquerdo)

Uma lista de pedidos incluídos no relatório:

- **Caixa de Seleção** - Selecionar/desmarcar pedidos para inclusão
- **Status** - Ícone de status do pedido
- **Número do Pedido** - ID único do pedido
- **Data de Criação** - Quando o pedido foi feito
- **Data de Pagamento** - Quando o pagamento foi recebido
- **Total** - Valor do pedido

Clique no ícone de configurações de exibição para personalizar colunas.

#### 3. Resumo do Relatório (Inferior Direito)

Os totais calculados do relatório:

- Dropdown de **tipo de relatório** (Padrão/Offline)
- **Informações da loja** (nome, ID)
- **Período do relatório** (timestamps de início/fim)
- **Caixa** (se filtrado)

**Resumo de Vendas:**
- Total de Pedidos
- Total de Vendas Líquidas
- Total de Impostos Coletados
- **Total de Vendas**
- Total de Descontos

**Métodos de Pagamento:**
- Detalhamento por tipo de pagamento (Dinheiro, Cartão, etc.)

Botão de **Impressão** para gerar um relatório físico.

## Tipos de Relatórios

### Padrão (Offline)

O relatório padrão é gerado a partir de dados armazenados localmente. Isso:

- Funciona sem conexão com o servidor
- Fornece geração rápida de relatórios
- Usa dados já sincronizados com o dispositivo

Ideal para resumos rápidos de fim de dia onde todos os pedidos relevantes já estão sincronizados.

## Fluxo de Trabalho de Fim de Dia

### Conciliação Diária Típica

1. **Definir intervalo de datas** - Selecionar "Hoje" ou data específica
2. **Filtrar por caixa** (opcional) - Para relatórios individuais de caixa
3. **Revisar pedidos** - Verificar a precisão da tabela de pedidos
4. **Verificar totais** - Comparar com contagem física de dinheiro e terminal de cartão
5. **Imprimir relatório** - Gerar um registro físico

### Conciliação de Dinheiro

1. Filtrar para mostrar apenas pagamentos em dinheiro
2. Anotar o valor total em dinheiro do relatório
3. Contar dinheiro físico na gaveta
4. Comparar totais (devem corresponder)
5. Documentar quaisquer discrepâncias

### Conciliação de Cartão

1. Filtrar para mostrar apenas pagamentos de cartão
2. Anotar o valor total de cartão do relatório
3. Comparar com o total de lote do terminal de cartão
4. Verificar se os valores correspondem antes de liquidar o lote

## Imprimindo Relatórios

Clique no botão **Imprimir** para gerar um relatório imprimível. Isso inclui:

- Informações da loja
- Período do relatório
- Resumo de vendas
- Detalhamento de métodos de pagamento
- Detalhes do pedido (se selecionado)
