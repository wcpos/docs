# API04006: Recurso Não Encontrado

## O Que Isso Significa

O servidor não conseguiu encontrar o recurso solicitado. Isso normalmente corresponde a um código de status HTTP 404, indicando que o produto, pedido, cliente ou outro recurso que você está tentando acessar não existe.

## Causas Comuns

- **Registro excluído** — O recurso foi excluído no servidor
- **ID incorreto** — Um ID incorreto ou desatualizado está sendo usado
- **Problemas de sincronização** — Os dados locais fazem referência a um recurso que não existe mais
- **Configuração de URL incorreta** — O endpoint da API está incorreto
- **Problemas de permalink** — Os permalinks do WordPress precisam ser atualizados

## Mapeamento de Erro do Servidor

Este código de erro é disparado quando o servidor retorna:

| Código do Servidor | Fonte |
|-------------|--------|
| `rest_no_route` | WordPress REST API |
| HTTP 404 | Qualquer resposta do servidor |

## Como Corrigir

### 1. Verificar se o Recurso Existe

Verifique se o recurso ainda existe no seu site WooCommerce:
- Faça login no admin do WordPress
- Navegue até a seção relevante (Produtos, Pedidos, Clientes)
- Procure o item por ID ou nome

### 2. Atualizar Dados Locais

Se o recurso foi excluído no lado do servidor:
1. Abra as configurações do POS
2. Navegue até a seção de dados relevante
3. Acione uma sincronização/atualização para atualizar os dados locais
4. O item excluído deve ser removido localmente

### 3. Verificar Permalinks do WordPress

Se vários recursos não forem encontrados:
1. Vá para **Configurações → Permalinks** no admin do WordPress
2. Clique em **Salvar Alterações** (mesmo sem fazer alterações)
3. Isso atualiza a estrutura de permalinks

### 4. Verificar Rotas da API

Teste a REST API diretamente:
```
https://seusite.com/wp-json/wc/v3/products
```
Se isso retornar 404, pode haver um problema de configuração do servidor.

### 5. Verificar Conflitos de Plugins

Se as rotas da REST API estiverem ausentes:
1. Certifique-se de que o WooCommerce está ativo
2. Certifique-se de que o plugin WCPOS está ativo
3. Desative temporariamente outros plugins para testar
