# API01007: Dispositivo Offline

## O Que Isso Significa

Seu dispositivo não está conectado à internet. O PDV detectou que não há conexão de rede ativa.

## Causas Comuns

- **WiFi desconectado** — Seu dispositivo perdeu a conexão WiFi
- **Ethernet desconectado** — O cabo de rede está desconectado
- **Modo avião ativado** — Todas as conexões sem fio estão desativadas
- **Adaptador de rede desativado** — A interface de rede está desligada

## Como Corrigir

### 1. Verificar Conexão WiFi

- Verifique se o WiFi está habilitado no seu dispositivo
- Certifique-se de estar conectado à rede correta
- Verifique a força do sinal WiFi

### 2. Verificar Conexões Físicas

Se estiver usando ethernet:
- Verifique se o cabo está conectado firmemente
- Tente um cabo ou porta diferentes
- Verifique as luzes do switch/roteador de rede

### 3. Desativar Modo Avião

Verifique se o modo avião foi ativado acidentalmente e desative-o.

### 4. Reiniciar Adaptador de Rede

**Windows:**
- Abra Conexões de Rede
- Clique com o botão direito no adaptador → Desabilitar, depois Habilitar

**macOS:**
- Desligue e ligue o WiFi da barra de menu
- Ou: Preferências do Sistema → Rede → Desligar/ligar

### 5. Reiniciar Seu Dispositivo

Às vezes, um simples reinício resolve problemas de detecção de rede.

## Modo Offline

Enquanto estiver offline, o WCPOS ainda pode:
- Navegar e pesquisar produtos em cache localmente
- Navegar e pesquisar clientes em cache localmente
- Iniciar novos pedidos e adicionar itens ao carrinho
- Visualizar dados sincronizados anteriormente

**Requer conectividade:**
- Concluir/finalizar pedidos
- Criar novos clientes

Quando a conectividade for restaurada, o PDV retomará automaticamente as operações normais.
