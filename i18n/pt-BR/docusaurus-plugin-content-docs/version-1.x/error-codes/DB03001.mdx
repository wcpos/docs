# DB03001: Erro de Sintaxe de Consulta

## O Que Isso Significa

A consulta ao banco de dados contém erros de sintaxe. Isso geralmente indica um bug no aplicativo em vez de um erro do usuário.

## Causas Comuns

- **Bug de software** — Um problema no aplicativo POS
- **Incompatibilidade de versão** — Esquema de dados incompatível
- **Dados corrompidos** — Dados contêm caracteres inesperados
- **Problemas de codificação** — Problemas de codificação de caracteres

## Como Corrigir

### 1. Atualizar o Aplicativo

Certifique-se de estar executando a versão mais recente:
- Verificar atualizações do aplicativo POS
- Atualizar o plugin WCPOS do WordPress
- Reiniciar após a atualização

### 2. Limpar Dados do Aplicativo

Redefinir para um estado limpo:
- Limpar dados/cache local
- Fazer login novamente
- Ressincronizar dados

### 3. Verificar Caracteres Especiais

Se o erro ocorrer com dados específicos:
- Verificar caracteres incomuns em nomes de produtos
- Procurar emojis ou símbolos especiais
- Tentar com dados mais simples

### 4. Reportar o Bug

Se o problema persistir:
1. Anotar qual ação causa o erro
2. Capturar detalhes do erro
3. Reportar no [GitHub](https://github.com/wcpos)

## Geralmente É um Bug

Erros de sintaxe de consulta são tipicamente bugs no software, não erros do usuário. Se você encontrar isso:
- Você não está fazendo nada de errado
- A equipe de desenvolvimento precisa saber
- Uma correção será lançada em uma atualização

## Solução Alternativa

Até que uma correção esteja disponível:
- Tentar abordagens alternativas para realizar sua tarefa
- Usar o admin do WooCommerce para operações afetadas
- Verificar atualizações regularmente
