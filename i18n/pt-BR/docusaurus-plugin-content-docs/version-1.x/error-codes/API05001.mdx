# API05001: API do WooCommerce Desativada

## O Que Isso Significa

A REST API do WooCommerce está desativada no seu site. O WCPOS requer a REST API para se comunicar com o WooCommerce e acessar dados da loja.

## Causas Comuns

- **Configuração do WooCommerce** — A REST API foi desativada intencionalmente
- **Plugin de segurança** — Um plugin de segurança está bloqueando o acesso à API
- **Restrição de hospedagem** — Seu host desativou o acesso à REST API
- **Problemas de permalink** — Permalinks não configurados para REST API

## Como Corrigir

### 1. Ativar REST API do WooCommerce

No Painel do WordPress:
1. Vá para WooCommerce → Configurações → Avançado → REST API
2. Certifique-se de que a REST API esteja ativada
3. Verifique se as chaves de API foram criadas

### 2. Verificar REST API do WordPress

A API do WooCommerce depende da REST API do WordPress:
1. Visite `https://seusite.com/wp-json/` no navegador
2. Deve retornar dados JSON
3. Se não retornar, consulte [API05004](./API05004)

### 3. Verificar Plugins de Segurança

Plugins de segurança comuns que podem bloquear a API:
- **Wordfence** — Verifique as configurações do firewall
- **iThemes Security** — Verifique as configurações da REST API
- **All In One WP Security** — Revise as regras do firewall

Adicione os endpoints da REST API ou do aplicativo POS à lista de permissões.

### 4. Verificar .htaccess

Procure por regras bloqueando o acesso à API:

```apache
# Remova ou modifique regras bloqueando /wp-json/
# Certifique-se de que mod_rewrite esteja ativado
```

### 5. Limpar Permalinks

Às vezes, as configurações de permalink precisam ser atualizadas:
1. Vá para Configurações → Permalinks
2. Clique em "Salvar Alterações" (mesmo sem fazer alterações)
3. Isso regenera as regras de reescrita
