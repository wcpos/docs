# API01005: Erro de Certificado SSL

## O Que Isso Significa

Há um problema com o certificado SSL/TLS do seu site. Este certificado é o que torna a conexão segura (HTTPS). O POS não irá se conectar a sites com certificados inválidos para proteger seus dados.

## Causas Comuns

- **Certificado expirado** — Certificados SSL precisam ser renovados periodicamente
- **Certificado autoassinado** — Não emitido por uma autoridade confiável
- **Domínio incorreto** — Certificado não corresponde ao nome de domínio
- **Cadeia de certificados incompleta** — Certificados intermediários ausentes
- **Conteúdo misto** — Alguns recursos carregados via HTTP em vez de HTTPS

## Como Corrigir

### 1. Verificar Status do Certificado

Visite seu site em um navegador e clique no ícone de cadeado para ver os detalhes do certificado. Observe:
- Data de expiração
- Emitido para (deve corresponder ao seu domínio)
- Emitido por (deve ser uma autoridade reconhecida)

### 2. Renovar Certificado Expirado

Se expirado:
- A maioria dos provedores de hospedagem oferece certificados Let's Encrypt gratuitos
- Entre em contato com seu provedor de hospedagem para renovar
- Verifique se a renovação automática está habilitada

### 3. Corrigir Incompatibilidade de Certificado

Garanta que o certificado cubra:
- Seu domínio exato (`sualoja.com.br`)
- Variante WWW (`www.sualoja.com.br`) se usada
- Considere um certificado curinga (`*.sualoja.com.br`)

### 4. Instalar Certificados Intermediários Ausentes

Use uma ferramenta de verificação SSL (como SSL Labs) para identificar certificados ausentes. Seu provedor de hospedagem pode ajudar a instalá-los.

### 5. Forçar HTTPS

No WordPress, certifique-se de:
- URL do site use `https://`
- SSL forçado esteja habilitado nas configurações do WooCommerce
