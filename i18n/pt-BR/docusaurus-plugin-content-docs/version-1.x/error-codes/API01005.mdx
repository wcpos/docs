# API01005: Erro de Certificado SSL

## O que Isso Significa

Há um problema com o certificado SSL/TLS do seu site. Este certificado é o que torna a conexão segura (HTTPS). O POS não se conectará a sites com certificados inválidos para proteger seus dados.

## Causas Comuns

- **Certificado expirado** — Certificados SSL precisam ser renovados periodicamente
- **Certificado autoassinado** — Não emitido por uma autoridade de confiança
- **Dominio errado** — Certificado não corresponde ao nome do domínio
- **Cadeia de certificados incompleta** — Faltando certificados intermediários
- **Conteúdo misto** — Alguns recursos carregados via HTTP em vez de HTTPS

## Como Corrigir

### 1. Verifique o Status do Certificado

Visite seu site em um navegador e clique no ícone do cadeado para ver os detalhes do certificado. Procure por:
- Data de expiração
- Emitido para (deve corresponder ao seu domínio)
- Emitido por (deve ser uma autoridade reconhecida)

### 2. Renove o Certificado Expirado

Se expirou:
- A maioria dos provedores de hospedagem oferece certificados gratuitos do Let's Encrypt
- Entre em contato com seu provedor de hospedagem para renovar
- Verifique se a renovação automática está habilitada

### 3. Corrija a Divergência do Certificado

Certifique-se de que o certificado cobre:
- Seu domínio exato (`yourstore.com`)
- Variante WWW (`www.yourstore.com`) se utilizada
- Considere um certificado curinga (`*.yourstore.com`)

### 4. Instale Certificados Intermediários Faltantes

Use uma ferramenta de verificação SSL (como SSL Labs) para identificar certificados faltantes. Seu provedor de hospedagem pode ajudar a instalá-los.

### 5. Forçar HTTPS

No WordPress, certifique-se de que:
- A URL do site usa `https://`
- Forçar SSL está habilitado nas configurações do WooCommerce
