# API02003: Token Inválido

## O Que Isso Significa

O token de autenticação que está sendo usado não é reconhecido pelo servidor. Isso é diferente de um token expirado — este token nunca foi válido ou foi revogado.

## Causas Comuns

- **Token revogado** — Um administrador revogou a sessão
- **Token corrompido** — Corrupção de dados durante o armazenamento ou transmissão
- **Reinicialização do servidor** — Chaves secretas do servidor mudaram, invalidando todos os tokens
- **Servidor errado** — Token de uma instalação WordPress diferente

## Mapeamento de Erro do Servidor

Este código de erro é acionado quando o servidor retorna:

| Código do Servidor | Fonte |
|--------------------|-------|
| `jwt_auth_invalid_token` | Plugin de Autenticação JWT |

## Como Corrigir

### 1. Faça Login Novamente

Limpe sua sessão e autentique-se novamente:
1. Desconecte-se completamente
2. Feche e reabra o POS
3. Faça login com suas credenciais

### 2. Limpe os Dados do Aplicativo

Se sair não funcionar:
- Limpe o cache/dados do aplicativo POS
- Na web: Limpe os cookies do navegador e o armazenamento local para o site
- No desktop: Verifique as configurações do aplicativo para uma opção de "Limpar Dados"

### 3. Verifique a Configuração do Servidor

Se o problema afetar todos os usuários:
- Verifique se os sais do WordPress foram alterados
- Verifique as configurações do JWT ou do plugin de autenticação
- Revise as mudanças recentes no servidor

### 4. Verifique se Está no Servidor Correto

Garanta que o POS esteja configurado para se conectar ao site WordPress correto, especialmente se você tiver múltiplas instalações.
