# API02003: Token Inválido

## O Que Isso Significa

O token de autenticação usado não é reconhecido pelo servidor. Isso é diferente de um token expirado — este token nunca foi válido ou foi revogado.

## Causas Comuns

- **Token revogado** — Um administrador encerrou a sessão
- **Token corrompido** — Corrupção de dados durante armazenamento ou transmissão
- **Reinício do servidor** — Chaves secretas do servidor foram alteradas, invalidando todos os tokens
- **Servidor incorreto** — Token de uma instalação diferente do WordPress

## Mapeamento de Erro do Servidor

Este código de erro é acionado quando o servidor retorna:

| Código do Servidor | Origem |
|-------------|--------|
| `jwt_auth_invalid_token` | Plugin de Autenticação JWT |

## Como Corrigir

### 1. Fazer Login Novamente

Limpe sua sessão e faça autenticação novamente:
1. Saia completamente
2. Feche e reabra o POS
3. Faça login com suas credenciais

### 2. Limpar Dados do Aplicativo

Se fazer logout não funcionar:
- Limpe o cache/dados do aplicativo POS
- Na web: Limpe os cookies e armazenamento local do site no navegador
- Na versão desktop: Verifique as configurações do aplicativo para uma opção "Limpar Dados"

### 3. Verificar Configuração do Servidor

Se o problema afetar todos os usuários:
- Verifique se os salts do WordPress foram alterados
- Confirme as configurações do plugin de JWT ou autenticação
- Revise alterações recentes no servidor

### 4. Verificar o Servidor Correto

Certifique-se de que o POS está configurado para se conectar ao site WordPress correto, especialmente se você tiver múltiplas instalações.
