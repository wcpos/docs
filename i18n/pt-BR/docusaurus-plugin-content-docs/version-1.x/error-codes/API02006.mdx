# API02006: Chave de API Inválida

## O Que Isso Significa

A chave da API REST do WooCommerce que está sendo usada é inválida. As chaves de API são usadas para autenticação entre servidores, e a chave fornecida não corresponde a nenhuma chave válida no WooCommerce.

## Causas Comuns

- **Chave excluída** — A chave de API foi removida do WooCommerce
- **Erro de digitação** — A chave foi digitada incorretamente
- **Par de chaves incorreto** — Chave de consumo e segredo não correspondem
- **Chave de outro site** — Usando chaves geradas para outra instalação

## Como Corrigir

### 1. Verificar a Chave de API

No Administrador do WordPress → WooCommerce → Configurações → Avançado → API REST:
1. Verifique se sua chave de API existe
2. Confirme que não foi revogada
3. Observe as permissões (leitura/gravação/leitura-gravação)

### 2. Gerar Novas Chaves de API

Se a chave estiver ausente ou inválida:
1. Vá para WooCommerce → Configurações → Avançado → API REST
2. Clique em "Adicionar chave"
3. Digite uma descrição (por exemplo, "WCPOS")
4. Selecione o usuário
5. Escolha permissões "Leitura/Gravação"
6. Clique em "Gerar chave de API"
7. **Copie a Chave de Consumo e o Segredo de Consumo** (mostrados apenas uma vez!)

### 3. Atualizar Configuração do POS

Digite as novas chaves de API no POS:
- Chave de Consumo (começa com `ck_`)
- Segredo de Consumo (começa com `cs_`)

### 4. Verificar Permissões da Chave

Certifique-se de que a chave tenha permissões suficientes:
- **Leitura** — Visualizar dados apenas
- **Gravação** — Modificar dados apenas
- **Leitura/Gravação** — Acesso completo (recomendado para POS)
