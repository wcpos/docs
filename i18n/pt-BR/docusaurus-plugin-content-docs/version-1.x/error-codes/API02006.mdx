# API02006: Chave API Inválida {#api02006-api-key-invalid}

## O Que Isso Significa {#what-this-means}

A chave da API REST do WooCommerce utilizada é inválida. As chaves da API são usadas para autenticação de servidor para servidor, e a chave fornecida não corresponde a nenhuma chave válida no WooCommerce.

## Causas Comuns {#common-causes}

- **Chave excluída** — A chave da API foi removida do WooCommerce
- **Erro de digitação na chave** — A chave foi digitada incorretamente
- **Par de chaves errado** — A chave do consumidor e o segredo não coincidem
- **Chave de site diferente** — Usando chaves geradas para outra instalação

## Como Corrigir {#how-to-fix}

### 1. Verifique a Chave da API {#1-verify-the-api-key}

No Admin do WordPress → WooCommerce → Configurações → Avançado → REST API:
1. Verifique se sua chave da API existe
2. Confirme se não foi revogada
3. Observe as permissões (somente leitura/leitura/gravação)

### 2. Gere Novas Chaves da API {#2-generate-new-api-keys}

Se a chave estiver faltando ou inválida:
1. Acesse WooCommerce → Configurações → Avançado → REST API
2. Clique em "Adicionar chave"
3. Insira uma descrição (por exemplo, "WCPOS")
4. Selecione o usuário
5. Escolha permissões "Leitura/Gravação"
6. Clique em "Gerar chave da API"
7. **Copie tanto a Chave do Consumidor quanto o Segredo do Consumidor** (mostrado apenas uma vez!)

### 3. Atualize a Configuração do POS {#3-update-pos-configuration}

Insira as novas chaves da API no POS:
- Chave do Consumidor (começa com `ck_`)
- Segredo do Consumidor (começa com `cs_`)

### 4. Verifique as Permissões da Chave {#4-check-key-permissions}

Certifique-se de que a chave possui permissões suficientes:
- **Leitura** — Visualizar dados apenas
- **Gravação** — Modificar dados apenas
- **Leitura/Gravação** — Acesso total (recomendado para POS)
