# API01002: Conexão Recusada

## O Que Isso Significa

O servidor recusou ativamente a tentativa de conexão. Isso é diferente de um tempo limite — o servidor respondeu, mas disse "não".

## Causas Comuns

- **Servidor offline** — O servidor web (Apache/Nginx) pode não estar em execução
- **Porta incorreta** — Tentativa de conexão na porta errada
- **Bloqueio por firewall** — Um firewall está bloqueando a conexão
- **Restrições de IP** — Seu endereço IP pode estar bloqueado

## Como Corrigir

### 1. Verificar se o Servidor Está Funcionando

Tente acessar seu site WordPress diretamente em um navegador. Se não estiver carregando, entre em contato com seu provedor de hospedagem.

### 2. Verificar a Configuração da URL

Certifique-se de usar a URL correta:
- Verificar `http://` vs `https://`
- Verificar erros de digitação no nome de domínio
- Confirmar a porta correta se estiver usando uma porta não padrão

### 3. Verificar Configurações do Firewall

Se você tiver acesso ao seu servidor:
- Verificar se as portas 80 (HTTP) ou 443 (HTTPS) estão abertas
- Verificar se seu IP está na lista de permissões
- Revisar plugins de segurança que possam bloquear o acesso à API

### 4. Contatar o Provedor de Hospedagem

Eles podem verificar:
- Status e logs do servidor
- Configurações de firewall
- Se seu IP foi bloqueado
