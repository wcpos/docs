# API01002: Conexão Recusada

## O Que Isso Significa

O servidor recusou ativamente a tentativa de conexão. Isso é diferente de um timeout — o servidor respondeu, mas disse "não".

## Causas Comuns

- **Servidor fora do ar** — O servidor web (Apache/Nginx) pode não estar em execução
- **Porta errada** — Tentativa de conexão na porta errada
- **Firewall bloqueando** — Um firewall está bloqueando a conexão
- **Restrições de IP** — Seu endereço IP pode estar bloqueado

## Como Corrigir

### 1. Verifique se o Servidor Está em Execução

Tente acessar seu site WordPress diretamente em um navegador. Se não estiver carregando, entre em contato com seu provedor de hospedagem.

### 2. Verifique a Configuração da URL

Certifique-se de que está usando a URL correta:
- Verifique `http://` vs `https://`
- Verifique se há erros de digitação no nome do domínio
- Confirme a porta correta se estiver usando uma não padrão

### 3. Verifique as Configurações do Firewall

Se você tiver acesso ao seu servidor:
- Verifique se a porta 80 (HTTP) ou 443 (HTTPS) está aberta
- Verifique se seu IP está na lista de permissões
- Revise quaisquer plugins de segurança que possam bloquear o acesso à API

### 4. Entre em Contato com Seu Provedor de Hospedagem

Eles podem verificar:
- Status e registros do servidor
- Configurações do firewall
- Se o seu IP foi bloqueado
