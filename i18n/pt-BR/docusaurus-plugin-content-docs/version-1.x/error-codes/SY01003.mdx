# SY01003: Permissão Negada

## O Que Isso Significa

O aplicativo POS não tem permissão para acessar um recurso necessário. Isso pode ser um arquivo, pasta ou recurso do sistema.

## Causas Comuns

- **Permissões de arquivo** — Não é possível ler ou gravar arquivos necessários
- **Permissões do aplicativo** — O aplicativo não possui permissões necessárias
- **Pasta protegida** — Tentativa de acessar um local restrito
- **Software de segurança** — Antivírus bloqueando o acesso

## Como Corrigir

### 1. Executar como Administrador (Windows)

Clique com o botão direito no aplicativo e selecione "Executar como administrador" se necessário para configuração inicial.

### 2. Verificar Permissões do Aplicativo

**macOS:**
- Preferências do Sistema → Segurança e Privacidade → Privacidade
- Verificar se o aplicativo possui as permissões necessárias

**Windows:**
- Configurações → Privacidade
- Revisar permissões do aplicativo

### 3. Verificar Permissões de Pasta

Certifique-se de que o aplicativo possa acessar sua pasta de dados:
- O usuário deve ser proprietário do diretório de dados do aplicativo
- Permissões de gravação devem estar habilitadas
- Sem sinalizadores de somente leitura na pasta

### 4. Revisar Configurações de Antivírus

O software de segurança pode bloquear o acesso:
- Verificar logs do antivírus para bloqueios
- Adicionar o aplicativo POS à lista de permissões
- Incluir a pasta de dados do aplicativo na lista de permissões

### 5. Reinstalar o Aplicativo

Se as permissões estiverem corrompidas:
1. Desinstalar o aplicativo POS
2. Reiniciar o dispositivo
3. Reinstalar em um local padrão
4. Executar com permissões normais (não administrativas)

## Permissões do Navegador (Versão Web)

Ao usar a versão web:
- Permitir armazenamento local para o site
- Habilitar cookies para o site
- Não usar configurações do navegador excessivamente restritivas
