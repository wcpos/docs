# SY01003: Permissão Negada {#sy01003-permission-denied}

## O Que Isso Significa {#what-this-means}

O aplicativo POS não tem permissão para acessar um recurso necessário. Isso pode ser um arquivo, pasta ou recurso do sistema.

## Causas Comuns {#common-causes}

- **Permissões de arquivo** — Não é possível ler ou gravar nos arquivos necessários
- **Permissões de aplicativo** — O aplicativo não possui as permissões necessárias
- **Pasta protegida** — Tentando acessar um local restrito
- **Software de segurança** — Antivírus bloqueando o acesso

## Como Corrigir {#how-to-fix}

### 1. Executar como Administrador (Windows) {#1-run-as-administrator-windows}

Clique com o botão direito no aplicativo e selecione "Executar como administrador" se necessário para a configuração inicial.

### 2. Verificar Permissões do Aplicativo {#2-check-application-permissions}

**macOS:**
- Preferências do Sistema → Segurança e Privacidade → Privacidade
- Verifique se o aplicativo possui as permissões necessárias

**Windows:**
- Configurações → Privacidade
- Revise as permissões do aplicativo

### 3. Verificar Permissões da Pasta {#3-check-folder-permissions}

Certifique-se de que o aplicativo pode acessar sua pasta de dados:
- O usuário deve ser o proprietário do diretório de dados do aplicativo
- As permissões de gravação devem estar habilitadas
- Não deve haver flags de somente leitura na pasta

### 4. Revisar Configurações do Antivírus {#4-review-antivirus-settings}

O software de segurança pode bloquear o acesso:
- Verifique os logs do antivírus para bloqueios
- Adicione o aplicativo POS à lista de permissões
- Coloque a pasta de dados do aplicativo na lista branca

### 5. Reinstalar o Aplicativo {#5-reinstall-the-application}

Se as permissões estiverem corrompidas:
1. Desinstale o aplicativo POS
2. Reinicie o dispositivo
3. Reinstale em um local padrão
4. Execute com permissões normais (não administrativas)

## Permissões do Navegador (Versão Web) {#browser-permissions-web-version}

Se estiver usando a versão web:
- Permita o armazenamento local para o site
- Habilite cookies para o site
- Não use configurações de navegador excessivamente restritivas
