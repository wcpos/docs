# API06001: Formato de URL Inválido

## O Que Isso Significa

A URL configurada para o seu site WooCommerce não está em um formato válido. As URLs devem seguir as convenções padrão de endereços web.

## Causas Comuns

- **Protocolo ausente** — A URL não começa com `http://` ou `https://`
- **Erro de digitação na URL** — Erros de ortografia no domínio
- **Caracteres inválidos** — Caracteres especiais não permitidos em URLs
- **URL incompleta** — URL está faltando partes como a extensão de domínio

## Como Corrigir

### 1. Verificar Formato da URL

Uma URL válida deve se parecer com:
- ✅ `https://sujaloja.com`
- ✅ `https://www.sujaloja.com`
- ✅ `https://loja.seudominio.com`
- ❌ `sujaloja.com` (protocolo ausente)
- ❌ `https://sujaloja` (faltando extensão)
- ❌ `https://sua loja.com` (contém espaço)

### 2. Incluir o Protocolo

Sempre inclua `https://` ou `http://`:
- Preferido: `https://` (seguro)
- Alternativa: `http://` (se HTTPS não estiver disponível)

### 3. Remover Caracteres Extras

Certifique-se de que a URL não tenha:
- Barras no final (opcional, mas seja consistente)
- Espaços extras
- Caracteres especiais
- Parâmetros de URL (a menos que sejam necessários)

### 4. Verificar se a URL Funciona

Teste a URL em um navegador:
1. Copie a URL que está usando
2. Cole em um navegador
3. Deve carregar seu site WordPress

### 5. Atualizar Configuração do POS

Insira a URL corrigida nas configurações do POS:
- Corresponda exatamente ao que funciona no seu navegador
- Inclua ou exclua `www.` conforme seu site usa

## Exemplos

| Inválido | Válido |
|---------|-------|
| `minhaloja.com` | `https://minhaloja.com` |
| `https://minha loja.com` | `https://minhaloja.com` |
| `https://minhaloja` | `https://minhaloja.com` |
| `htp://minhaloja.com` | `https://minhaloja.com` |
