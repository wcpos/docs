# API06001: Formato de URL Inválido {#api06001-invalid-url-format}

## O que Isso Significa {#what-this-means}

A URL configurada para o seu site WooCommerce não está em um formato válido. As URLs devem seguir as convenções padrão de endereços web.

## Causas Comuns {#common-causes}

- **Protocolo ausente** — URL não começa com `http://` ou `https://`
- **Erro de digitação na URL** — Erros de ortografia no domínio
- **Caracteres inválidos** — Caracteres especiais que não são permitidos em URLs
- **URL incompleta** — A URL está faltando partes como a extensão do domínio

## Como Corrigir {#how-to-fix}

### 1. Verifique o Formato da URL {#1-check-url-format}

Uma URL válida deve ser parecida com:
- ✅ `https://sualoja.com`
- ✅ `https://www.sualoja.com`
- ✅ `https://loja.seudominio.com`
- ❌ `sualoja.com` (falta protocolo)
- ❌ `https://sualoja` (falta extensão)
- ❌ `https://sua loja.com` (contém espaço)

### 2. Inclua o Protocolo {#2-include-the-protocol}

Sempre inclua `https://` ou `http://`:
- Preferencial: `https://` (seguro)
- Alternativa: `http://` (se HTTPS não estiver disponível)

### 3. Remova Caracteres Extras {#3-remove-extra-characters}

Certifique-se de que a URL não tenha:
- Barras no final (opcional, mas seja consistente)
- Espaços extras
- Caracteres especiais
- Parâmetros de URL (a menos que sejam necessários)

### 4. Verifique se a URL Funciona {#4-verify-the-url-works}

Teste a URL em um navegador:
1. Copie a URL que você está usando
2. Cole-a em um navegador
3. Deve carregar o seu site WordPress

### 5. Atualize a Configuração do POS {#5-update-pos-configuration}

Insira a URL corrigida nas configurações do POS:
- Combine exatamente o que funciona no seu navegador
- Inclua ou exclua `www.` conforme o uso do seu site

## Exemplos {#examples}

| Inválido | Válido |
|----------|--------|
| `mystore.com` | `https://mystore.com` |
| `https://minha loja.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
