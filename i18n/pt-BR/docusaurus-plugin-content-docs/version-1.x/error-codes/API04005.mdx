# API04005: Tentativa de Recuperação de JSON

## O Que Isso Significa

O servidor enviou uma resposta com algum conteúdo JSON inválido, mas o POS tentou recuperar e extrair dados válidos. Este é um aviso informativo e não um erro crítico.

## O Que Aconteceu

O POS detectou:
1. A resposta continha conteúdo extra antes ou depois do JSON
2. Os dados JSON principais ainda eram identificáveis
3. Foi tentada a recuperação extraindo a parte válida do JSON

## Causas Comuns

- **Notícias PHP na saída** — Avisos PHP misturados com JSON
- **Saída de depuração** — Depuração de desenvolvimento deixada ativa
- **Notícias de plugins** — Outros plugins emitindo notificações
- **Problemas de espaço em branco** — Espaço em branco extra ao redor do JSON

## Como Corrigir

### 1. Desativar Exibição de Erros PHP

No `wp-config.php`:
```php
define('WP_DEBUG_DISPLAY', false);
ini_set('display_errors', 0);
```

### 2. Ativar Registro de Erros

Manter erros registrados para depuração:
```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
```

### 3. Verificar Modo de Depuração de Plugins

Alguns plugins têm modos de depuração que geram conteúdo extra:
- Revisar configurações do plugin
- Desativar modos de depuração/desenvolvimento
- Verificar opções de registro detalhado

### 4. Revisar Alterações Recentes

Se isso começou recentemente:
- O que mudou no seu servidor?
- Plugins foram atualizados?
- Configurações PHP foram modificadas?

## Isso É Grave?

Embora o POS tenha se recuperado deste problema, indica um problema de configuração que deve ser corrigido. O processo de recuperação:
- Pode nem sempre funcionar
- Adiciona sobrecarga de processamento
- Pode mascarar outros problemas

Corrija a causa subjacente para garantir operação confiável.
