# API04002: Código de Resposta Inesperado

## O Que Isso Significa

O servidor retornou um código de status HTTP que não era esperado para esta solicitação. Códigos comuns incluem 500 (erro do servidor), 403 (proibido), 404 (não encontrado), etc.

## Causas Comuns

- **500 Erro Interno do Servidor** — O PHP falhou ou encontrou um erro
- **403 Proibido** — Acesso negado pelas configurações de segurança
- **404 Não Encontrado** — O endpoint não existe
- **502/503/504** — Problemas de gateway ou disponibilidade do servidor

## Como Resolver

### Para Erros 500 (Erro do Servidor)

1. Verifique os logs de erro do PHP
2. Aumente o limite de memória do PHP
3. Procure por conflitos de plugins
4. Verifique `wp-content/debug.log`

### Para Erros 403 (Proibido)

1. Verifique as configurações do plugin de segurança (Wordfence, Sucuri, etc.)
2. Coloque o POS ou seu IP na lista de permissões
3. Verifique o `.htaccess` em busca de regras de bloqueio
4. Verifique se o ModSecurity não está bloqueando solicitações

### Para Erros 404 (Não Encontrado)

1. Verifique se o plugin WCPOS está ativo
2. Limpe os links permanentes do WordPress (Configurações → Links Permanentes → Salvar)
3. Verifique se a API REST está habilitada
4. Verifique a configuração da URL

### Para Erros 502/503/504 (Problemas de Gateway)

1. Entre em contato com seu provedor de hospedagem
2. Aguarde a recuperação do servidor
3. Verifique se o site está sob carga pesada
4. Verifique se o servidor está funcionando

### Resolução Geral de Problemas

1. Tente acessar seu site diretamente
2. Verifique o painel de controle de hospedagem em busca de problemas
3. Revise os logs de acesso do servidor
4. Entre em contato com o suporte de hospedagem, se necessário
