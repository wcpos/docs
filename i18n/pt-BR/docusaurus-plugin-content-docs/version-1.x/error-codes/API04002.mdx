# API04002: Código de Resposta Inesperado

## O Que Isso Significa

O servidor retornou um código de status HTTP que não era esperado para esta solicitação. Códigos comuns incluem 500 (erro do servidor), 403 (proibido), 404 (não encontrado), etc.

## Causas Comuns

- **500 Erro Interno do Servidor** — PHP travou ou encontrou um erro
- **403 Proibido** — Acesso negado por configurações de segurança
- **404 Não Encontrado** — O endpoint não existe
- **502/503/504** — Problemas de gateway ou disponibilidade do servidor

## Como Corrigir

### Para Erros 500 (Erro do Servidor)

1. Verificar logs de erro do PHP
2. Aumentar o limite de memória do PHP
3. Procurar conflitos de plugins
4. Verificar `wp-content/debug.log`

### Para Erros 403 (Proibido)

1. Verificar configurações do plugin de segurança (Wordfence, Sucuri, etc.)
2. Incluir o POS ou seu IP na lista de permissões
3. Verificar regras de bloqueio no `.htaccess`
4. Verificar se o ModSecurity não está bloqueando solicitações

### Para Erros 404 (Não Encontrado)

1. Verificar se o plugin WCPOS está ativo
2. Limpar permalinks do WordPress (Configurações → Permalinks → Salvar)
3. Verificar se a API REST está habilitada
4. Verificar configuração de URL

### Para Erros 502/503/504 (Problemas de Gateway)

1. Contatar o provedor de hospedagem
2. Aguardar a recuperação do servidor
3. Verificar se o site está sob alta carga
4. Verificar se o servidor está funcionando

### Solução de Problemas Geral

1. Tentar acessar o site diretamente
2. Verificar painel de controle de hospedagem em busca de problemas
3. Revisar logs de acesso do servidor
4. Contatar suporte de hospedagem se necessário
