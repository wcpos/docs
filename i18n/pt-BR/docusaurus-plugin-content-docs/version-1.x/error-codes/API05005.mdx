# API05005: Plugin Não Encontrado

## O Que Isso Significa

Um plugin do WordPress necessário não está instalado ou ativo. Pode ser o próprio WooCommerce, uma extensão obrigatória ou um plugin de integração.

## Causas Comuns

- **WooCommerce desativado** — O WooCommerce foi desabilitado
- **Extensão necessária ausente** — Uma extensão do WooCommerce necessária não está instalada
- **Conflito de plugin** — Outro plugin causou a desativação de um plugin obrigatório
- **Atualização falhou** — A atualização de um plugin falhou e o deixou desativado

## Como Corrigir

### 1. Verificar se o WooCommerce Está Ativo

O WCPOS requer o WooCommerce:
1. Vá para Plugins → Plugins Instalados
2. Localize "WooCommerce"
3. Certifique-se de que está ativado

### 2. Verificar Extensões Obrigatórias

Alguns recursos do WCPOS podem exigir extensões:
- Verifique a mensagem de erro para o nome específico do plugin
- Instale e ative a extensão necessária

### 3. Revisar Conflitos de Plugins

Se os plugins foram desativados devido a conflitos:
1. Verifique seus e-mails de notificação do WordPress
2. Revise os registros de erros
3. Resolva os conflitos e reative

### 4. Verificar Arquivos do Plugin

Se os arquivos de um plugin estiverem ausentes:
1. Reinstale o plugin do WordPress.org
2. Ou restaure a partir de um backup
3. Ative o plugin

### 5. Atualizar Todos os Plugins

Plugins desatualizados podem causar problemas:
1. Vá para Painel → Atualizações
2. Atualize todos os plugins
3. Verifique se todos os plugins necessários estão ativos

## Plugins Obrigatórios

No mínimo, o WCPOS requer:
- **WooCommerce** — A plataforma de comércio eletrônico
- **WCPOS** — O plugin de PDV

Recursos adicionais podem exigir outros plugins.
