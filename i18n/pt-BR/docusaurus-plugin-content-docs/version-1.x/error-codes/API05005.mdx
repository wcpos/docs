# API05005: Plugin Não Encontrado

## O Que Isso Significa

Um plugin WordPress necessário não está instalado ou ativo. Isso pode ser o próprio WooCommerce, uma extensão necessária ou um plugin de integração.

## Causas Comuns

- **WooCommerce desativado** — O WooCommerce foi desativado
- **Extensão necessária ausente** — Uma extensão necessária do WooCommerce não está instalada
- **Conflito de plugin** — Outro plugin causou a desativação de um plugin necessário
- **Atualização falhou** — Uma atualização de plugin falhou e deixou-o desativado

## Como Corrigir

### 1. Verifique se o WooCommerce está Ativo

O WCPOS requer o WooCommerce:
1. Vá para Plugins → Plugins Instalados
2. Encontre "WooCommerce"
3. Certifique-se de que está ativado

### 2. Verifique as Extensões Necessárias

Alguns recursos do WCPOS podem exigir extensões:
- Verifique a mensagem de erro para o nome específico do plugin
- Instale e ative a extensão necessária

### 3. Revise Conflitos de Plugins

Se os plugins foram desativados devido a conflitos:
1. Verifique seu email para notificações do WordPress
2. Revise os registros de erro
3. Resolva os conflitos e reative

### 4. Verifique os Arquivos do Plugin

Se os arquivos de um plugin estiverem ausentes:
1. Reinstale o plugin a partir do WordPress.org
2. Ou restaure do backup
3. Ative o plugin

### 5. Atualize Todos os Plugins

Plugins desatualizados podem causar problemas:
1. Vá para o Painel → Atualizações
2. Atualize todos os plugins
3. Verifique se todos os plugins necessários estão ativos

## Plugins Necessários

No mínimo, o WCPOS requer:
- **WooCommerce** — A plataforma de e-commerce
- **WCPOS** — O plugin de POS

Recursos adicionais podem exigir outros plugins.
