# API02009: Token de Atualização Expirado {#api02009-refresh-token-expired}

## O Que Isso Significa {#what-this-means}

Seu token de atualização excedeu sua vida útil e não pode mais ser usado para obter novos tokens de acesso. Ao contrário dos tokens de acesso (que expiram rapidamente), os tokens de atualização geralmente duram dias ou semanas, mas eventualmente expiram.

## Causas Comuns {#common-causes}

- **Inatividade prolongada** — Você não usou o POS por um longo tempo
- **Vida útil do token curta** — Servidor configurado com expiração curta do token de atualização
- **Desvio de relógio** — O tempo do dispositivo ou servidor está significativamente incorreto

## Como Corrigir {#how-to-fix}

### 1. Faça o Login Novamente {#1-log-in-again}

Quando o token de atualização expira, você deve se autenticar novamente:
1. Faça logout do POS
2. Insira suas credenciais
3. Um novo token de atualização será emitido

### 2. Use o POS Regularmente {#2-use-pos-regularly}

Para evitar isso no futuro:
- Use o POS regularmente para manter os tokens atualizados
- Os tokens são atualizados automaticamente quando você está ativo
- Períodos prolongados de inatividade exigirão novo login

### 3. Verifique as Configurações de Hora {#3-check-time-settings}

Garanta que a hora esteja correta em todos os sistemas:
- Habilite a sincronização automática de hora em seu dispositivo
- Verifique se a hora do servidor está correta
- Diferenças de horário podem causar expiração prematura

### 4. Ajuste a Vida Útil do Token (Avançado) {#4-adjust-token-lifetime-advanced}

Se os tokens expirarem muito rapidamente para suas necessidades, um administrador do servidor pode ajustar a vida útil do token de atualização nas configurações de autenticação.
