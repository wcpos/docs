# DB01001: Falha na Conexão

## O Que Significa

O POS não conseguiu conectar ao seu banco de dados local. O WCPOS usa um banco de dados local para armazenar produtos, clientes e outros dados para acesso rápido e funcionalidade offline.

## Causas Comuns

- **Armazenamento cheio** — Dispositivo sem espaço de armazenamento disponível
- **Banco de dados corrompido** — Arquivos do banco de dados local estão danificados
- **Problema de permissão** — O aplicativo não tem permissões de armazenamento
- **Armazenamento do navegador desativado** — IndexedDB está desabilitado (versão web)

## Como Corrigir

### 1. Verificar Espaço de Armazenamento

Garanta que seu dispositivo tenha armazenamento disponível:
- Libere espaço se o armazenamento estiver cheio
- Exclua aplicativos ou arquivos não utilizados
- Verifique o espaço em disco disponível

### 2. Limpar Dados do Aplicativo

Redefina o banco de dados local:
- **Aplicativo desktop:** Procure por "Limpar Dados" nas configurações
- **Navegador web:** Limpe os dados do site para o domínio do POS
- **Nota:** Isso exigirá nova sincronização de todos os dados

### 3. Verificar Configurações do Navegador (Web)

Se estiver usando a versão web:
1. Certifique-se de que o IndexedDB esteja habilitado
2. Não use modo privado/incógnito (armazenamento limitado)
3. Permita que o site armazene dados

### 4. Verificar Permissões

No desktop:
- Certifique-se de que o aplicativo tenha permissão para gravar no seu diretório de dados
- Verifique se o antivírus está bloqueando o acesso ao banco de dados

### 5. Reinstalar o Aplicativo

Se outras soluções falharem:
1. Desinstale o aplicativo POS
2. Reinicie seu dispositivo
3. Reinstale o aplicativo
4. Faça login e sincronize os dados

## Recuperação de Dados

Se você tiver dados não salvos:
- O aplicativo pode ter alterações pendentes que não puderam ser sincronizadas
- Entre em contato com o suporte se precisar de ajuda para recuperar dados
- Na maioria dos casos, os dados podem ser sincronizados novamente a partir do servidor
