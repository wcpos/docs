# DB01002: Tempo Limite de Consulta

## O Que Isso Significa

Uma operação de banco de dados demorou muito para ser concluída. A consulta local ao banco de dados excedeu o tempo máximo permitido.

## Causas Comuns

- **Conjunto de dados grande** — Número muito grande de produtos ou registros
- **Consulta complexa** — Consulta envolvendo muitas condições
- **Desempenho do dispositivo** — Dispositivo está funcionando lentamente
- **Fragmentação do banco de dados** — Banco de dados precisa de otimização

## Como Corrigir

### 1. Aguarde e Tente Novamente

A consulta pode ser bem-sucedida na nova tentativa:
- Feche e reabra o PDV
- Tente a operação novamente
- Evite outras operações intensivas simultaneamente

### 2. Feche Outros Aplicativos

Libere recursos do dispositivo:
- Feche guias do navegador não utilizadas
- Feche outros aplicativos
- Reinicie o dispositivo se necessário

### 3. Reduza o Escopo dos Dados

Ao pesquisar:
- Use termos de pesquisa mais específicos
- Aplique filtros para reduzir resultados
- Divida operações grandes em menores

### 4. Otimize o Banco de Dados

Limpe e sincronize novamente:
1. Limpe dados locais (opção de configurações, se disponível)
2. Faça login novamente
3. Deixe os dados sincronizarem do zero

### 5. Verifique o Desempenho do Dispositivo

Verifique se o dispositivo atende aos requisitos:
- RAM suficiente
- Velocidade de processador adequada
- SSD preferível a HDD

## Catálogos Grandes

Se você tiver um catálogo de produtos muito grande:
- Sincronização inicial leva mais tempo
- Considere usar filtros no PDV
- Algumas operações naturalmente levam mais tempo
- Entre em contato com o suporte para orientações de otimização
