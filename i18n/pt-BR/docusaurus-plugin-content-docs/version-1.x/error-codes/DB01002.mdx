# DB01002: Tempo de Consulta Excedido {#db01002-query-timeout}

## O Que Isso Significa {#what-this-means}

Uma operação de banco de dados levou tempo demais para ser concluída. A consulta ao banco de dados local excedeu o tempo máximo permitido.

## Causas Comuns {#common-causes}

- **Conjunto de dados grande** — Número muito grande de produtos ou registros
- **Consulta complexa** — Consulta envolvendo muitas condições
- **Desempenho do dispositivo** — Dispositivo está funcionando lentamente
- **Fragmentação do banco de dados** — O banco de dados precisa de otimização

## Como Corrigir {#how-to-fix}

### 1. Aguarde e Tente Novamente {#1-wait-and-retry}

A consulta pode ter sucesso na nova tentativa:
- Feche e reabra o POS
- Tente a operação novamente
- Evite outras operações intensivas simultaneamente

### 2. Feche Outros Aplicativos {#2-close-other-applications}

Libere recursos do dispositivo:
- Feche guias de navegador não utilizadas
- Feche outros aplicativos
- Reinicie o dispositivo se necessário

### 3. Reduza o Escopo de Dados {#3-reduce-data-scope}

Se estiver pesquisando:
- Use termos de busca mais específicos
- Aplique filtros para restringir resultados
- Divida grandes operações em menores

### 4. Otimize o Banco de Dados {#4-optimise-the-database}

Limpe e re-sincronize:
1. Limpe os dados locais (opção de configurações se disponível)
2. Faça login novamente
3. Deixe os dados se re-sincronizarem

### 5. Verifique o Desempenho do Dispositivo {#5-check-device-performance}

Garanta que seu dispositivo atenda aos requisitos:
- RAM suficiente
- Velocidade de processador adequada
- SSD preferível ao HDD

## Catálogos Grandes {#large-catalogues}

Se você tiver um catálogo de produtos muito grande:
- A sincronização inicial leva mais tempo
- Considere usar filtros no POS
- Algumas operações naturalmente levam mais tempo
- Entre em contato com o suporte para dicas de otimização
