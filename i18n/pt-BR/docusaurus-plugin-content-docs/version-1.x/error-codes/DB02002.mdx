# DB02002: Registro Não Encontrado

## O Que Isso Significa

O registro solicitado não existe no banco de dados local. O POS tentou acessar um produto, cliente, pedido ou outro registro que não está armazenado localmente.

## Causas Comuns

- **Ainda não sincronizado** — Os dados ainda não foram baixados
- **Deletado no servidor** — Registro foi removido no WooCommerce
- **Sincronização não completa** — Sincronização inicial ainda está em andamento
- **Filtro aplicado** — Registro excluído por filtros ativos

## Como Consertar

### 1. Aguarde a Sincronização

Se iniciou recentemente:
- Aguarde a sincronização inicial ser concluída
- Verifique o indicador de progresso da sincronização
- O registro deve aparecer uma vez sincronizado

### 2. Atualizar/Re-sincronizar

Acione uma atualização de dados:
- Use o botão de atualizar/sincronizar (<Icon name="arrow-rotate-right" />) - pressão breve para sincronização
- **Pressione longamente** o botão de sincronização para a opção Limpar e Atualizar
- Puxe os últimos dados do servidor
- Aguarde a conclusão

### 3. Verifique o WooCommerce

Verifique se o registro existe no servidor:
- Faça login no WordPress Admin
- Verifique Produtos, Pedidos ou Clientes
- Confirme se o item ainda existe

### 4. Verifique os Filtros

O registro pode estar filtrado:
- Reveja os filtros ativos no POS
- Verifique os filtros de categoria
- Verifique as configurações de visibilidade

### 5. Limpar e Re-sincronizar

Se a sincronização parecer travada:
1. Limpe os dados locais
2. Faça login novamente
3. Complete uma nova sincronização

## Cenários Comuns

- **Produto não exibido:** Verifique se está publicado e visível
- **Cliente não encontrado:** Pode não ter sido sincronizado ainda
- **Pedido ausente:** Pode estar fora do intervalo de datas de sincronização
