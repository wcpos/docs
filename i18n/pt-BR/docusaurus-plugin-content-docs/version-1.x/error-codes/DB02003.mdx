# DB02003: Violação de Restrição

## O Que Isso Significa

Os dados que você está tentando salvar violam as regras do banco de dados. As restrições garantem a integridade dos dados, aplicando regras sobre o que pode ser armazenado.

## Causas Comuns

- **Dados obrigatórios ausentes** — Um campo obrigatório está vazio
- **Relacionamento inválido** — Referenciando um registro que não existe
- **Incompatibilidade de tipo de dados** — Tipo de dados incorreto para o campo
- **Valor fora do intervalo** — Número excede os limites permitidos

## Como Corrigir

### 1. Verificar Campos Obrigatórios

Certifique-se de que todos os dados obrigatórios sejam fornecidos:
- Informações do cliente (se obrigatório)
- Detalhes do produto
- Itens de linha do pedido

### 2. Verificar Referências

Se o erro envolver relacionamentos:
- Certifique-se de que os produtos existam
- Verifique se os IDs de clientes são válidos
- Confirme as atribuições de categorias

### 3. Revisar Valores de Dados

Verifique valores inválidos:
- Quantidades negativas onde não são permitidas
- Preços que excedem limites
- Valores de status inválidos

### 4. Sincronizar Dados Mais Recentes

Os dados referenciados podem estar dessincronizados:
- Atualizar dados do servidor
- Aguardar a sincronização ser concluída
- Tentar a operação novamente

### 5. Limpar e Tentar Novamente

Se os dados estiverem corrompidos:
1. Limpar o formulário problemático
2. Reinserir os dados
3. Enviar novamente

## Exemplos Comuns de Restrições

- **Quantidade deve ser positiva** — Não é possível adicionar 0 ou itens negativos
- **Preço deve ser numérico** — Texto não é permitido em campos de preço
- **Cliente deve existir** — Não é possível atribuir pedido a um cliente inexistente
