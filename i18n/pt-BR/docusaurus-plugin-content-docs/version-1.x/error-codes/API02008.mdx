# API02008: Token de Atualização Inválido

## O Que Isso Significa

O token de atualização armazenado pelo POS não é reconhecido pelo servidor. Os tokens de atualização são usados para obter novos tokens de acesso sem a necessidade de fazer login novamente.

## Causas Comuns

- **Token revogado** — Um administrador invalidou sua sessão
- **Segredos do servidor alterados** — As chaves de segurança do WordPress foram atualizadas
- **Corrupção de dados** — O token foi corrompido no armazenamento
- **Múltiplos logins** — Fazer login em outro lugar pode ter invalidado este token

## Como Corrigir

### 1. Faça Login Novamente

Uma vez que o token de atualização é inválido, você precisará se autenticar novamente:
1. Faça logout do POS
2. Digite seu nome de usuário e senha
3. Isso gera novos tokens válidos

### 2. Verifique Sessões Revogadas

Se um administrador revogou sessões:
- Isso é uma prática de segurança normal
- Simplesmente faça login novamente
- Sua nova sessão funcionará normalmente

### 3. Verifique Alterações no Servidor

Se isso afetar todos os usuários:
- As salgadas/chaves do WordPress podem ter mudado
- As configurações do plugin JWT podem ter sido modificadas
- Entre em contato com o administrador do seu servidor

### 4. Limpe os Dados Armazenados

Se você continuar tendo problemas:
- Limpe os dados do aplicativo POS
- Remova credenciais em cache
- Comece com um login limpo
