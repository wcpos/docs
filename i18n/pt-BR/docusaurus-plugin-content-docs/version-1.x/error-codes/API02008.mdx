# API02008: Token de Atualização Inválido

## O Que Isso Significa

O token de atualização armazenado pelo POS não é reconhecido pelo servidor. Os tokens de atualização são usados para obter novos tokens de acesso sem exigir que você faça login novamente.

## Causas Comuns

- **Token revogado** — Um administrador invalidou sua sessão
- **Segredos do servidor alterados** — Chaves de segurança do WordPress foram atualizadas
- **Corrupção de dados** — O token foi corrompido no armazenamento
- **Múltiplos logins** — Fazer login em outro lugar pode ter invalidado este token

## Como Corrigir

### 1. Fazer Login Novamente

Como o token de atualização é inválido, você precisará autenticar novamente:
1. Sair do POS
2. Inserir nome de usuário e senha
3. Isso gerará novos tokens válidos

### 2. Verificar Sessões Revogadas

Se um administrador revogou sessões:
- Esta é uma prática normal de segurança
- Simplesmente faça login novamente
- Sua nova sessão funcionará normalmente

### 3. Verificar Alterações no Servidor

Se isso afetar todos os usuários:
- Os salts/chaves do WordPress podem ter sido alterados
- As configurações do plugin JWT podem ter sido modificadas
- Entre em contato com o administrador do servidor

### 4. Limpar Dados Armazenados

Se você continuar tendo problemas:
- Limpe os dados do aplicativo POS
- Remova as credenciais em cache
- Comece com um login limpo
