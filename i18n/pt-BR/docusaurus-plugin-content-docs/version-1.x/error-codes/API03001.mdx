# API03001: Formato de Solicitação Inválido

## O Que Isso Significa

A solicitação enviada ao servidor não estava no formato esperado. O servidor não conseguiu entender o que o PDV estava solicitando porque a estrutura da solicitação estava incorreta.

## Causas Comuns

- **Dados corrompidos** — Dados foram corrompidos antes do envio
- **Bug de software** — Um problema no aplicativo de PDV
- **Interferência de proxy** — Um proxy ou firewall modificou a solicitação
- **Problemas de codificação de caracteres** — Caracteres especiais não foram codificados corretamente

## Mapeamento de Erro do Servidor

Este código de erro é acionado quando o servidor retorna:

| Código do Servidor | Fonte |
|-------------------|-------|
| HTTP 400 | Qualquer resposta do servidor (padrão) |

## Como Corrigir

### 1. Tentar a Ação Novamente

Às vezes, isso é um problema momentâneo:
- Espere um momento e tente novamente
- Atualize o PDV e tente novamente

### 2. Verificar Caracteres Especiais

Se você está inserindo dados com caracteres especiais:
- Tente remover emojis ou símbolos incomuns
- Use caracteres padrão para nomes de produtos, etc.

### 3. Atualizar o PDV

Certifique-se de estar executando a versão mais recente:
- Verifique se há atualizações do aplicativo
- Atualize o plugin WCPOS no seu servidor

### 4. Verificar Configuração de Rede

Se você está atrás de um proxy:
- Verifique se o proxy não está modificando solicitações
- Verifique as regras do firewall
- Tente acessar de uma rede diferente

### 5. Reportar o Problema

Se isso acontecer consistentemente:
- Anote qual ação aciona o erro
- Verifique o console do navegador para obter detalhes (versão web)
- Reporte no [GitHub](https://github.com/wcpos) com etapas de reprodução
