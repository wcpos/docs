# API06003: Configuração de Site Inválida

## O Que Isso Significa

A configuração do site é inválida ou incompleta. Isso pode envolver URLs incorretas, configurações de autenticação ou outros problemas de configuração.

## Causas Comuns

- **Configuração parcial** — Algumas configurações estão ausentes
- **Configurações incompatíveis** — A configuração não corresponde ao site
- **Configuração corrompida** — As configurações foram corrompidas
- **Mudanças no site** — O site foi modificado sem atualizar a configuração do POS

## Como Corrigir

### 1. Revise Todas as Configurações

Verifique a configuração completa:
- A URL do site está correta
- A autenticação está configurada corretamente
- Quaisquer configurações adicionais estão corretas

### 2. Reconfigure do Zero

Se a configuração estiver corrompida:
1. Limpe todas as configurações armazenadas
2. Inicie o processo de configuração novamente
3. Insira uma nova configuração

### 3. Verifique os Requisitos do Site

Verifique se seu site WordPress atende aos requisitos:
- WordPress está instalado e acessível
- WooCommerce está instalado e ativo
- O plugin WCPOS está instalado e ativo
- Os links permanentes estão habilitados (não "Simples")

### 4. Teste o Acesso ao Site

Verifique se essas URLs funcionam em um navegador:
- `https://yoursite.com/` — Site principal
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. Verifique as Configurações do WordPress

No Admin do WordPress:
1. Configurações → Geral — Verifique as URLs
2. Configurações → Links Permanentes — Certifique-se de que não está "Simples"
3. WooCommerce → Status — Verifique se há problemas

## Requisitos de Configuração

Uma configuração válida precisa de:
- **URL do Site** — URL completa com protocolo
- **Autenticação** — Credenciais ou chaves de API válidas
- **Permissões** — O usuário tem acesso ao POS
- **Plugins** — Plugins necessários estão ativos
