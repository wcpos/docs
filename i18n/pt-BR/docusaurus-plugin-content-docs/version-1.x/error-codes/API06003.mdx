# API06003: Configuração de Site Inválida

## O Que Isso Significa

A configuração do site é inválida ou incompleta. Isso pode envolver URLs incorretos, configurações de autenticação ou outros problemas de configuração.

## Causas Comuns

- **Configuração parcial** — Algumas configurações estão faltando
- **Configurações incompatíveis** — A configuração não corresponde ao site
- **Configuração corrompida** — As configurações foram corrompidas
- **Alterações no site** — O site foi modificado sem atualizar a configuração do POS

## Como Corrigir

### 1. Revisar Todas as Configurações

Verifique a configuração completa:
- URL do site está correto
- Autenticação está configurada corretamente
- Todas as configurações adicionais estão corretas

### 2. Reconfigurar do Zero

Se a configuração estiver corrompida:
1. Limpe todas as configurações armazenadas
2. Inicie o processo de configuração novamente
3. Insira uma nova configuração

### 3. Verificar Requisitos do Site

Verifique se o site WordPress atende aos requisitos:
- WordPress está instalado e acessível
- WooCommerce está instalado e ativo
- Plugin WCPOS está instalado e ativo
- Permalinks estão habilitados (não "Simples")

### 4. Testar Acesso ao Site

Verifique se esses URLs funcionam no navegador:
- `https://seusite.com/` — Site principal
- `https://seusite.com/wp-json/` — REST API
- `https://seusite.com/wp-json/wcpos/v1/` — API WCPOS

### 5. Verificar Configurações do WordPress

No Administrador do WordPress:
1. Configurações → Geral — Verificar URLs
2. Configurações → Permalinks — Garantir que não esteja "Simples"
3. WooCommerce → Status — Verificar problemas

## Requisitos de Configuração

Uma configuração válida precisa de:
- **URL do Site** — URL completa com protocolo
- **Autenticação** — Credenciais válidas ou chaves de API
- **Permissões** — Usuário tem acesso ao POS
- **Plugins** — Plugins necessários estão ativos
