# API03004: Solicitação Muito Grande

## O Que Isso Significa

A solicitação que você está enviando excede os limites de tamanho do servidor. Isso geralmente acontece ao tentar enviar dados em grande quantidade de uma vez.

## Causas Comuns

- **Operações em lote grandes** — Tentativa de sincronizar muitos registros de uma vez
- **Imagens grandes** — Envio de imagens com tamanho excessivo
- **Muitos itens** — Pedido com número extremamente alto de linhas de item
- **Limites do servidor** — Limites de upload baixos no PHP ou servidor web

## Como Corrigir

### 1. Reduzir Tamanho da Solicitação

Ao sincronizar dados:
- Tente sincronizar em lotes menores
- O POS deve lidar com isso automaticamente
- Espere a sincronização atual ser concluída antes de iniciar outra

### 2. Verificar Tamanhos de Imagens

Ao enviar imagens:
- Redimensione imagens antes de enviar
- Use formatos comprimidos (JPEG vs BMP)
- A maioria das imagens de produtos funciona bem com menos de 1MB

### 3. Dividir Pedidos Grandes

Se um pedido tem muitos itens:
- Considere dividir em múltiplos pedidos
- Isso é raro no uso normal do POS

### 4. Aumentar Limites do Servidor

Entre em contato com seu provedor de hospedagem ou edite as configurações do PHP:

```php
// Em php.ini ou .htaccess
upload_max_filesize = 64M
post_max_size = 64M
max_input_vars = 5000
```

### 5. Verificar Limites do Servidor Web

Nginx ou Apache podem ter seus próprios limites:
- `client_max_body_size` para Nginx
- `LimitRequestBody` para Apache
