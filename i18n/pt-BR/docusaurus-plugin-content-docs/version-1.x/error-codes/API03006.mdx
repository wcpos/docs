# API03006: Método Não Suportado

## O Que Isso Significa

O método HTTP utilizado (GET, POST, PUT, DELETE, etc.) não é suportado para este endpoint. O servidor não aceita este tipo de solicitação para esta URL.

## Causas Comuns

- **Incompatibilidade de versão** — Versões do POS e do plugin incompatíveis
- **Endpoint removido** — Um endpoint da API foi descontinuado
- **Configuração do servidor** — Servidor web bloqueando certos métodos HTTP
- **Conflito de plugin** — Outro plugin modificando o comportamento da REST API

## Como Corrigir

### 1. Atualizar Ambos os Componentes

Garantir compatibilidade:
- Atualizar o plugin WordPress do WCPOS
- Atualizar o aplicativo POS
- Verificar notas de lançamento para mudanças significativas

### 2. Verificar Configuração do Servidor

Alguns servidores bloqueiam certos métodos HTTP:
- Garantir que métodos PUT e DELETE estejam permitidos
- Verificar restrições de método no `.htaccess`
- Revisar configuração do Nginx

### 3. Verificar Acesso à REST API

Testar a REST API do WordPress:
1. Visitar `https://seusite.com/wp-json/` em um navegador
2. Deve retornar dados JSON
3. Se não, a REST API pode estar desativada ou bloqueada

### 4. Verificar Conflitos de Plugin

Desativar outros plugins temporariamente:
- Plugins de segurança podem bloquear métodos
- Outros plugins de REST API podem causar conflitos
- Reativar um por um para encontrar o problema

### 5. Revisar Restrições de Hospedagem

Alguns hosts restringem métodos HTTP:
- Contatar suporte de hospedagem
- Solicitar que habilitem todos os métodos padrão
- Considerar trocar de host se for muito restritivo
