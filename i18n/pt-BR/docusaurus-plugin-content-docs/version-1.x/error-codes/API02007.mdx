# API02007: Falha na Atualização do Token {#api02007-token-refresh-failed}

## O que Isso Significa {#what-this-means}

O POS tentou renovar automaticamente sua sessão de autenticação, mas a solicitação de atualização falhou. Isso geralmente significa que você precisará fazer login novamente.

## Causas Comuns {#common-causes}

- **Token de atualização expirado** — O token de atualização em si expirou
- **Token de atualização revogado** — Um administrador invalidou todas as sessões
- **Erro no servidor** — O servidor não conseguiu processar a solicitação de atualização
- **Problema na rede** — Problema de conexão durante a tentativa de atualização

## Como Corrigir {#how-to-fix}

### 1. Faça Login Novamente {#1-log-in-again}

A correção mais confiável:
1. Faça logout do POS
2. Faça login novamente com suas credenciais
3. Isso cria uma nova sessão com novos tokens

### 2. Verifique a Conexão de Rede {#2-check-network-connection}

Se a atualização estiver falhando devido a problemas na rede:
- Verifique sua conexão com a internet
- Tente acessar seu site WordPress em um navegador
- Aguarde um momento e tente novamente

### 3. Verifique o Status do Servidor {#3-check-server-status}

Se o servidor estiver apresentando problemas:
- Verifique se o WordPress está funcionando corretamente
- Verifique se há erros de PHP nos logs do seu servidor
- Certifique-se de que o plugin de autenticação esteja ativo

### 4. Limpe os Dados da Sessão {#4-clear-session-data}

Se os problemas persistirem:
- Limpe o cache do aplicativo POS
- Remova as credenciais armazenadas
- Comece novamente com um novo login

## Atualização Automática do Token {#automatic-token-refresh}

O WCPOS atualiza automaticamente os tokens antes que eles expirem para manter sua sessão. Se esse processo falhar repetidamente, pode haver um problema de configuração que precisa de atenção.
