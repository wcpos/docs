# API02007: Falha na Atualização do Token

## O Que Isso Significa

O POS tentou renovar automaticamente sua sessão de autenticação, mas a solicitação de atualização falhou. Isso normalmente significa que você precisará fazer login novamente.

## Causas Comuns

- **Token de atualização expirado** — O próprio token de atualização expirou
- **Token de atualização revogado** — Um administrador invalidou todas as sessões
- **Erro de servidor** — O servidor não conseguiu processar a solicitação de atualização
- **Problema de rede** — Problema de conexão durante a tentativa de atualização

## Como Corrigir

### 1. Fazer Login Novamente

A correção mais confiável:
1. Sair do POS
2. Fazer login novamente com suas credenciais
3. Isso cria uma sessão nova com tokens novos

### 2. Verificar Conexão de Rede

Se a atualização está falhando por problemas de rede:
- Verifique sua conexão com a internet
- Tente acessar seu site WordPress em um navegador
- Espere um momento e tente novamente

### 3. Verificar Status do Servidor

Se o servidor está tendo problemas:
- Verifique se o WordPress está funcionando corretamente
- Verifique erros de PHP nos logs do servidor
- Certifique-se de que o plugin de autenticação está ativo

### 4. Limpar Dados da Sessão

Se os problemas persistirem:
- Limpe o cache do aplicativo POS
- Remova as credenciais armazenadas
- Comece com um novo login

## Atualização Automática de Token

O WCPOS atualiza tokens automaticamente antes que expirem para manter sua sessão. Se esse processo falhar repetidamente, pode haver um problema de configuração que precisa de atenção.
