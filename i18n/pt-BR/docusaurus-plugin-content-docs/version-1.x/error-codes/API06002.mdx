# API06002: URL da API Ausente {#api06002-missing-api-url}

## O Que Isso Significa {#what-this-means}

Nenhuma URL foi configurada para conectar-se à sua loja WooCommerce. O POS precisa saber onde sua loja está localizada.

## Causas Comuns {#common-causes}

- **Configuração inicial** — URL nunca foi inserida
- **Configuração limpa** — As configurações foram redefinidas ou limpas
- **Dados do aplicativo limpos** — Dados do aplicativo foram excluídos
- **Problema de migração** — URL perdida durante atualização ou migração

## Como Corrigir {#how-to-fix}

### 1. Insira a URL da Sua Loja {#1-enter-your-store-url}

No aplicativo POS:
1. Abra as configurações ou a tela de login
2. Encontre o campo de configuração de URL/Site
3. Insira a URL do seu site WordPress
4. Exemplo: `https://sualoja.com`

### 2. Encontre Sua URL do WordPress {#2-find-your-wordpress-url}

Se você não tiver certeza da sua URL:
1. Faça login no Admin do WordPress
2. Vá para Configurações → Geral
3. Olhe para "Endereço do WordPress (URL)"
4. Use essa URL no POS

### 3. Verifique as Configurações Armazenadas {#3-check-stored-settings}

Se a URL já foi configurada anteriormente:
- Verifique se os dados do aplicativo foram acidentalmente limpos
- Procure opções de backup/exportação de configurações
- Reinserir a URL, se necessário

### 4. Verifique se a URL Está Correta {#4-verify-url-is-correct}

Antes de salvar:
1. Teste a URL em um navegador
2. Certifique-se de que carregue seu site WordPress
3. Inclua `https://` ou `http://`

## Lista de Verificação de Configuração {#setup-checklist}

Ao configurar o WCPOS pela primeira vez:

1. ☐ Insira a URL do seu site WordPress
2. ☐ Verifique se o plugin WCPOS está instalado e ativo
3. ☐ Faça login com suas credenciais do WordPress
4. ☐ Permita que a sincronização inicial de dados seja concluída
