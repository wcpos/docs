# API06002: URL da API Ausente

## O Que Isso Significa

Nenhuma URL foi configurada para conectar-se à sua loja WooCommerce. O POS precisa saber onde sua loja está localizada.

## Causas Comuns

- **Configuração inicial** — URL nunca foi inserida
- **Configuração limpa** — Configurações foram redefinidas ou apagadas
- **Dados do aplicativo limpos** — Dados do aplicativo foram excluídos
- **Problema de migração** — URL perdida durante atualização ou migração

## Como Corrigir

### 1. Insira a URL da Sua Loja

No aplicativo POS:
1. Abra as configurações ou tela de login
2. Localize o campo de configuração de URL/Site
3. Insira a URL do seu site WordPress
4. Exemplo: `https://sualoja.com`

### 2. Encontre Sua URL do WordPress

Se não tiver certeza da URL:
1. Faça login no Administrador do WordPress
2. Vá para Configurações → Geral
3. Procure por "Endereço do WordPress (URL)"
4. Use esta URL no POS

### 3. Verifique as Configurações Armazenadas

Se a URL foi configurada anteriormente:
- Verifique se os dados do aplicativo foram limpos acidentalmente
- Procure opções de backup/exportação de configurações
- Reinsira a URL, se necessário

### 4. Verifique se a URL está Correta

Antes de salvar:
1. Teste a URL em um navegador
2. Verifique se carrega seu site WordPress
3. Inclua `https://` ou `http://`

## Lista de Verificação de Configuração

Ao configurar o WCPOS pela primeira vez:

1. ☐ Insira a URL do seu site WordPress
2. ☐ Verifique se o plugin WCPOS está instalado e ativo
3. ☐ Faça login com suas credenciais do WordPress
4. ☐ Permita que a sincronização inicial de dados seja concluída
