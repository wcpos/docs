# API05004: API do WordPress Desativada

## O Que Isso Significa

A API REST do WordPress está desativada no seu site. Toda a funcionalidade moderna do WordPress, incluindo WooCommerce e WCPOS, depende dessa API.

## Causas Comuns

- **Plugin de segurança** — Um plugin está bloqueando o acesso à API REST
- **Restrição de hospedagem** — Seu provedor desabilitou a API REST
- **Código personalizado** — Um tema ou plugin desativou a API
- **Regras de firewall** — WAF bloqueando endpoints da API REST

## Como Corrigir

### 1. Testar a API REST

Visite `https://seusite.com/wp-json/` no seu navegador:
- Deve retornar JSON com rotas disponíveis
- Se aparecer um erro ou nada, está bloqueada

### 2. Verificar Plugins de Segurança

Plugins comuns que bloqueiam a API REST:

**Wordfence:**
- Firewall → Todas as Opções de Firewall
- Desabilitar opção "Desativar API REST"

**iThemes Security:**
- Segurança → Configurações → Ajustes do WordPress
- Ativar API REST

**Plugin Desativar API REST:**
- Desativar completamente este plugin

### 3. Verificar Código Personalizado

Procure no `functions.php` do tema ou em plugins personalizados:

```php
// Este código desativa a API REST - remova-o
add_filter('rest_authentication_errors', function($result) {
    return new WP_Error('rest_disabled', 'REST API disabled');
});
```

### 4. Verificar .htaccess

Remova qualquer regra bloqueando `/wp-json/`:

```apache
# Ruim - bloqueia API REST
RewriteRule ^wp-json - [F,L]
```

### 5. Contatar Provedor de Hospedagem

Alguns provedores bloqueiam a API REST por padrão:
- Solicite que ativem
- Pergunte sobre restrições de segurança
- Verifique a documentação da hospedagem

## Por Que a API REST É Importante

A API REST do WordPress é essencial para:
- Aplicativos móveis
- Integrações de terceiros
- Funções do WooCommerce
- Operação do WCPOS

Desativá-la quebra muitas funcionalidades.
