# API03002: Parâmetros Requeridos Ausentes

## O Que Isso Significa

A solicitação está faltando dados que o servidor precisa para concluir a ação. Campos requeridos não foram incluídos na solicitação.

## Causas Comuns

- **Formulário incompleto** — Campos requeridos não foram preenchidos
- **Dados não salvos** — Os dados do formulário não foram capturados corretamente
- **Incompatibilidade de versão** — O plugin espera parâmetros diferentes do que o POS envia
- **Campos personalizados** — Campos personalizados requeridos estão ausentes

## Como Corrigir

### 1. Completar Campos Requeridos

Verifique se todas as informações requeridas estão preenchidas:
- Detalhes do cliente (se necessário)
- Informações do produto
- Detalhes do pedido

### 2. Atualizar e Tentar Novamente

O estado do formulário pode estar incompleto:
1. Recarregue o POS
2. Reinsira as informações requeridas
3. Tente a ação novamente

### 3. Verificar Versões do Plugin

Assegure a compatibilidade:
- Atualize o plugin WCPOS
- Atualize o aplicativo POS
- Ambos devem estar em versões compatíveis

### 4. Verificar Configurações de Campos Requeridos

No WooCommerce, alguns campos podem ser marcados como requeridos:
- Reveja os requisitos dos campos de checkout
- Verifique as configurações de campos personalizados
- Ajuste os requisitos se necessário

### 5. Verificar Integrações Personalizadas

Se utilizando plugins ou integrações personalizadas:
- Eles podem exigir campos adicionais
- Reveja a documentação do plugin
- Verifique se há conflitos
