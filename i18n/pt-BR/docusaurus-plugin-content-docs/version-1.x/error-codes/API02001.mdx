# API02001: Credenciais Inválidas

## O Que Isso Significa

O nome de usuário ou senha que você inseriu está incorreto. O servidor rejeitou a tentativa de login porque as credenciais não correspondem a nenhuma conta válida.

## Causas Comuns

- **Erro de digitação no nome de usuário ou senha** — Verifique o bloqueio de maiúsculas ou espaços extras
- **Conta errada** — Usando credenciais de um site diferente
- **Senha alterada recentemente** — A senha foi atualizada, mas o POS ainda tem a antiga
- **Conta excluída** — A conta de usuário não existe mais

## Mapeamento de Erro do Servidor

Este código de erro é acionado quando o servidor retorna:

| Código do Servidor | Fonte |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | Plugin de Autenticação JWT |

## Como Corrigir

### 1. Verificar Suas Credenciais

- Verifique novamente seu nome de usuário (geralmente email ou nome de usuário do WordPress)
- Confirme se a senha está correta
- Tente fazer login diretamente no admin do WordPress para confirmar

### 2. Redefinir Sua Senha

Se você esqueceu sua senha:
1. Vá para a página de login do WordPress
2. Clique em "Perdeu a senha?"
3. Digite seu email para receber um link de redefinição
4. Atualize sua senha
5. Faça login no POS com a nova senha

### 3. Verificar a Conta de Usuário

No Admin do WordPress:
- Vá para Usuários → Todos os Usuários
- Verifique se a conta existe
- Verifique se possui permissões de acesso adequadas para o POS

### 4. Limpar Credenciais Salvas

No POS:
- Saia completamente
- Limpe todas as informações de login salvas ou em cache
- Insira as credenciais novamente
