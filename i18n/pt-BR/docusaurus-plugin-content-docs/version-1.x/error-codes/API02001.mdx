# API02001: Credenciais Inválidas

## O Que Isso Significa

O nome de usuário ou a senha que você inseriu estão incorretos. O servidor rejeitou a tentativa de login porque as credenciais não coincidem com nenhuma conta válida.

## Causas Comuns

- **Erro de digitação no nome de usuário ou na senha** — Verifique se o caps lock está ativado ou se há espaços extras
- **Conta errada** — Usando credenciais de um site diferente
- **Senha alterada recentemente** — A senha foi atualizada, mas o POS tem a antiga
- **Conta excluída** — A conta de usuário não existe mais

## Mapeamento de Erros do Servidor

Esse código de erro é acionado quando o servidor retorna:

| Código do Servidor | Fonte |
|--------------------|-------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | Plugin de Autenticação JWT |

## Como Resolver

### 1. Verifique Suas Credenciais

- Verifique novamente seu nome de usuário (geralmente e-mail ou nome de usuário do WordPress)
- Verifique se a senha está correta
- Tente fazer login diretamente no admin do WordPress para confirmar

### 2. Redefina Sua Senha

Se você esqueceu sua senha:
1. Vá para a página de login do WordPress
2. Clique em "Esqueceu sua senha?"
3. Insira seu e-mail para receber um link de redefinição
4. Atualize sua senha
5. Faça login no POS com a nova senha

### 3. Verifique a Conta de Usuário

No Admin do WordPress:
- Vá para Usuários → Todos os Usuários
- Verifique se a conta existe
- Confira se ela tem as permissões de acesso ao POS adequadas

### 4. Limpe Credenciais Salvas

No POS:
- Faça logout completamente
- Limpe qualquer informação de login salva/cache
- Reinsira as credenciais do zero
