# API01004: Falha na Resolução de DNS

## O Que Isso Significa

O POS não conseguiu converter o nome de domínio do seu site (por exemplo, `seusite.com.br`) em um endereço IP. O DNS (Sistema de Nomes de Domínio) é como a lista telefônica da internet — se não encontrar o registro, não pode estabelecer a conexão.

## Causas Comuns

- **Erro de digitação no nome de domínio** — A URL pode conter um erro de ortografia
- **Problemas no servidor DNS** — Seu servidor DNS não está respondendo
- **Domínio expirado** — O registro do domínio pode ter vencido
- **Propagação de DNS** — Alterações recentes de DNS ainda não foram propagadas
- **Cache de DNS local** — Informações de DNS desatualizadas no seu dispositivo

## Como Corrigir

### 1. Verificar o Nome de Domínio

Verifique cuidadosamente a URL em busca de erros de digitação. Tente acessar o site em um navegador.

### 2. Verificar Status do Domínio

Use uma consulta WHOIS para verificar:
- O domínio ainda está registrado
- Não expirou
- Os registros de DNS estão configurados

### 3. Limpar Cache de DNS

**No seu dispositivo:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Tentar DNS Alternativo

Mude para um servidor DNS público como:
- Google: `8.8.8.8` e `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Aguardar Propagação de DNS

Se você fez alterações recentes nas configurações de DNS, aguarde 24-48 horas para propagação completa.
