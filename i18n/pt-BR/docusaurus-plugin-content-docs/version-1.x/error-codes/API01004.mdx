# API01004: Falha na Resolução de DNS {#api01004-dns-resolution-failed}

## O que isso significa {#what-this-means}

O POS não conseguiu converter o nome de domínio do seu site (por exemplo, `yourstore.com`) em um endereço IP. O DNS (Sistema de Nomes de Domínio) é como a lista telefônica da internet — se não conseguir encontrar o registro, não poderá fazer a conexão.

## Causas Comuns {#common-causes}

- **Erro de digitação no nome de domínio** — A URL pode ter um erro de escrita
- **Problemas no servidor DNS** — Seu servidor DNS não está respondendo
- **Domínio expirado** — O registro do domínio pode ter expirado
- **Propagação de DNS** — Mudanças recentes no DNS ainda não foram propagadas
- **Cache DNS local** — Informações DNS desatualizadas armazenadas no seu dispositivo

## Como Corrigir {#how-to-fix}

### 1. Verifique o Nome de Domínio {#1-verify-the-domain-name}

Verifique a URL em busca de erros de digitação. Tente acessar o site em um navegador.

### 2. Verifique o Status do Domínio {#2-check-domain-status}

Use uma consulta WHOIS para verificar:
- O domínio ainda está registrado
- Não expirou
- Os registros DNS estão configurados

### 3. Limpe o Cache DNS {#3-clear-dns-cache}

**No seu dispositivo:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. Tente DNS Alternativo {#4-try-alternative-dns}

Mude para um servidor DNS público como:
- Google: `8.8.8.8` e `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. Aguarde a Propagação do DNS {#5-wait-for-dns-propagation}

Se você alterou recentemente as configurações de DNS, aguarde de 24 a 48 horas para a propagação completa.
