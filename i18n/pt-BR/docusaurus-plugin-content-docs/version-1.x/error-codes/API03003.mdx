# API03003: Valor de Parâmetro Inválido

## O Que Isso Significa

Um dos valores na sua solicitação é inválido. O formato dos dados, tipo ou valor não corresponde ao que o servidor espera.

## Causas Comuns

- **Tipo de dados incorreto** — Texto onde um número é esperado (ou vice-versa)
- **Fora do intervalo** — Valor excede os limites permitidos
- **Formato inválido** — Campos de e-mail, telefone ou outros estão incorretos
- **Referência inválida** — Referenciando um ID que não existe

## Mapeamento de Erro do Servidor

Este código de erro é acionado quando o servidor retorna:

| Código do Servidor | Fonte |
|-------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## Como Corrigir

### 1. Verificar Valores de Entrada

Revise os dados que você está enviando:
- **Preços** — Devem ser números válidos
- **Quantidades** — Devem ser inteiros positivos
- **E-mails** — Devem estar em formato de e-mail válido
- **IDs** — Devem referenciar registros existentes

### 2. Procurar Erros em Campos Específicos

O erro pode indicar qual campo é inválido:
- Verifique os detalhes da mensagem de erro
- Corrija o campo específico
- Tente a ação novamente

### 3. Verificar Existência de Dados Referenciados

Ao referenciar produtos, clientes ou outros registros:
- Certifique-se de que o item existe no WooCommerce
- Verifique se não foi excluído recentemente
- Sincronize os dados para atualizar os registros locais

### 4. Verificar Restrições de Campos

O WooCommerce pode ter restrições:
- Valores máximos/mínimos
- Formatos obrigatórios
- Opções permitidas para campos de seleção

### 5. Limpar e Redigitar

Às vezes, os dados podem ser corrompidos:
- Limpe o campo problemático
- Redigite o valor do zero
- Evite copiar e colar de outras fontes
