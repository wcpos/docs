# API02010: Autenticação Necessária

## O Que Isso Significa

A ação que você está tentando realizar requer autenticação, mas você não está atualmente conectado. O POS precisa de credenciais válidas para acessar este recurso.

## Causas Comuns

- **Não conectado** — Você ainda não se autenticou
- **Sessão limpa** — Sua sessão foi limpa ou expirou
- **Acessando recurso protegido** — O recurso requer autenticação
- **Dados do aplicativo limpos** — As credenciais armazenadas foram removidas

## Mapeamento de Erro do Servidor

Este código de erro é disparado quando o servidor retorna:

| Código do Servidor | Origem |
|-------------|--------|
| `rest_login_required` | WordPress REST API |
| `jwt_auth_no_auth_header` | Plugin de Autenticação JWT |
| HTTP 401 | Qualquer resposta do servidor (contingência) |

## Como Corrigir

### 1. Fazer Login

Se você não fez login:
1. Abra a tela de login do POS
2. Digite suas credenciais do WordPress
3. Complete o processo de autenticação

### 2. Verificar Status da Sessão

Se você achava que estava conectado:
- Sua sessão pode ter expirado
- Procure por [API02002](./API02002) (Token Expirado) para mais detalhes
- Faça login novamente para restaurar o acesso

### 3. Verificar Configuração do Servidor

Certifique-se de que os endpoints da API estejam configurados corretamente:
- A REST API do WooCommerce deve estar habilitada
- O plugin WCPOS deve estar ativo
- Os endpoints de autenticação devem estar acessíveis

### 4. Verificar Problemas no Navegador/Aplicativo

Se você está sendo desconectado inesperadamente:
- Limpe o cache do navegador (se estiver usando a versão web)
- Verifique se cookies/armazenamento local não estão sendo bloqueados
- Verifique se o aplicativo tem permissão para armazenar dados

## O Que Requer Autenticação?

A maioria das operações do POS requer autenticação:
- Visualizar produtos e clientes
- Criar e editar pedidos
- Processar pagamentos
- Acessar relatórios

Apenas a tela inicial de login é acessível sem autenticação.
