# DB03002: Tipo de Dado Inválido

## O Que Isso Significa

O tipo de dado não corresponde ao que o banco de dados espera. Por exemplo, texto foi fornecido onde um número era esperado.

## Causas Comuns

- **Erro de entrada do usuário** — Inserindo texto em um campo numérico
- **Problemas de importação** — Dados importados com formato incorreto
- **Corrupção de dados** — Valores corrompidos durante transferência
- **Conflito de plugin** — Outro plugin modificou tipos de dados

## Como Corrigir

### 1. Verifique Sua Entrada

Revise os dados que você está inserindo:
- **Preços** — Devem ser números (por exemplo, `19.99`)
- **Quantidades** — Devem ser números inteiros (por exemplo, `5`)
- **IDs** — Devem ser numéricos

### 2. Limpar e Redigitar

Se os dados foram corrompidos:
- Limpe o campo
- Digite o valor novamente manualmente
- Evite copiar e colar de fontes externas

### 3. Verifique os Dados de Origem

Ao sincronizar do WooCommerce:
- Verifique os dados no Painel Administrativo do WordPress
- Procure campos com formatação incorreta
- Corrija os dados na origem

### 4. Ressincronizar Dados

Obtenha dados novos do servidor:
- Limpe o cache local
- Sincronize os dados novamente
- Verifique se o problema é resolvido

### 5. Verificar Conflitos de Plugins

Se estiver usando outros plugins do WooCommerce:
- Eles podem modificar dados de maneira inesperada
- Desative temporariamente para testar
- Relate incompatibilidades

## Exemplos Comuns

| Campo | Esperado | Inválido |
|-------|----------|----------|
| Preço | `19.99` | `$19.99` |
| Quantidade | `5` | `cinco` |
| SKU | `ABC123` | — (qualquer formato OK) |
| ID do Produto | `42` | `product-42` |
