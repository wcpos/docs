# DB03002: Tipo de Dados Inválido {#db03002-invalid-data-type}

## O Que Isso Significa {#what-this-means}

O tipo de dado não corresponde ao que o banco de dados espera. Por exemplo, um texto foi fornecido onde um número era esperado.

## Causas Comuns {#common-causes}

- **Erro de entrada do usuário** — Inserindo texto em um campo numérico
- **Problemas de importação** — Dados importados com formato incorreto
- **Corrupção de dados** — Valores corrompidos durante a transferência
- **Conflito de plugins** — Outro plugin modificou os tipos de dados

## Como Corrigir {#how-to-fix}

### 1. Verifique Sua Entrada {#1-check-your-input}

Revise os dados que você está inserindo:
- **Preços** — Devem ser números (por exemplo, `19.99`)
- **Quantidades** — Devem ser números inteiros (por exemplo, `5`)
- **IDs** — Devem ser numéricos

### 2. Limpar e Reinsira {#2-clear-and-re-enter}

Se os dados foram corrompidos:
- Limpe o campo
- Insira o valor novamente manualmente
- Evite copiar e colar de fontes externas

### 3. Verifique os Dados de Origem {#3-check-source-data}

Se sincronizando do WooCommerce:
- Verifique os dados no WordPress Admin
- Procure campos formatados incorretamente
- Corrija os dados na origem

### 4. Re-sincronize os Dados {#4-re-sync-data}

Obtenha dados atualizados do servidor:
- Limpe o cache local
- Sincronize os dados novamente
- Verifique se o problema foi resolvido

### 5. Verifique Conflitos de Plugins {#5-check-for-plugin-conflicts}

Se estiver usando outros plugins do WooCommerce:
- Eles podem modificar os dados de maneiras inesperadas
- Desative temporariamente para testar
- Relate incompatibilidades

## Exemplos Comuns {#common-examples}

| Campo       | Esperado | Inválido      |
|-------------|----------|---------------|
| Preço       | `19.99`  | `$19.99`      |
| Quantidade  | `5`      | `cinco`       |
| SKU         | `ABC123` | — (qualquer formato OK) |
| ID do Produto | `42`   | `product-42`  |
