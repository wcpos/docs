# API03005: Limite de Taxa Excedido

## O Que Isso Significa

Você fez muitas requisições ao servidor em um curto período. O limite de taxa protege o servidor de ser sobrecarregado.

## Causas Comuns

- **Operações de sincronização rápidas** — Vários dispositivos sincronizando simultaneamente
- **Atualização agressiva** — Atualizando dados com muita frequência
- **Plugin de segurança** — Um plugin está impondo limites de requisição
- **Limites de hospedagem** — Seu plano de hospedagem tem limites de taxa de API

## Mapeamento de Erros do Servidor

Este código de erro é acionado quando o servidor retorna:

| Código do Servidor | Fonte |
|-------------------|-------|
| HTTP 429          | Qualquer resposta do servidor (fallback) |

## Como Resolver

### 1. Aguarde e Tente Novamente

Limites de taxa são temporários:
- Aguarde 1-5 minutos
- O POS tentará automaticamente novamente
- Evite atualizar manualmente repetidamente

### 2. Reduza Operações Concorrentes

Se estiver usando múltiplos terminais POS:
- Espalhe a sincronização inicial entre os dispositivos
- Evite atualizar todos os dispositivos ao mesmo tempo
- Deixe um concluir antes de iniciar outro

### 3. Verifique Plugins de Segurança

Plugins como Wordfence podem bloquear requisições rápidas:
- Coloque o aplicativo POS na lista de permissões
- Coloque os IPs do seu dispositivo POS na lista de permissões
- Ajuste os limites de taxa

### 4. Revise Limites de Hospedagem

Alguns provedores limitam requisições de API:
- Verifique os detalhes do seu plano de hospedagem
- Considere uma atualização se os limites forem muito baixos
- Contate o suporte sobre o aumento dos limites

### 5. Otimize a Frequência de Sincronização

Nas configurações do POS:
- Ajuste os intervalos de atualização automática
- Use sincronização manual quando apropriado
- Evite atualizações de dados desnecessárias
