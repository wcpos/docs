# API03005: Limite de Taxa Excedido

## O Que Isso Significa

Você fez solicitações demais ao servidor em um curto período. O limite de taxa protege o servidor de ser sobrecarregado.

## Causas Comuns

- **Operações de sincronização rápidas** — Múltiplos dispositivos sincronizando simultaneamente
- **Atualização agressiva** — Atualizando dados com muita frequência
- **Plugin de segurança** — Um plugin está aplicando limites de solicitações
- **Limites de hospedagem** — Seu plano de hospedagem tem limites de taxa de API

## Mapeamento de Erro do Servidor

Este código de erro é acionado quando o servidor retorna:

| Código do Servidor | Fonte |
|-------------|--------|
| HTTP 429 | Qualquer resposta do servidor (contingência) |

## Como Corrigir

### 1. Aguarde e Tente Novamente

Limites de taxa são temporários:
- Aguarde 1-5 minutos
- O POS tentará automaticamente novamente
- Evite atualizar repetidamente manualmente

### 2. Reduza Operações Simultâneas

Se estiver usando múltiplos terminais POS:
- Distribua a sincronização inicial entre dispositivos
- Evite atualizar todos os dispositivos ao mesmo tempo
- Deixe um completar antes de iniciar outro

### 3. Verifique Plugins de Segurança

Plugins como Wordfence podem bloquear solicitações rápidas:
- Adicione o aplicativo POS à lista de permissões
- Adicione os IPs do seu dispositivo POS à lista de permissões
- Ajuste os limites de taxa

### 4. Revise Limites de Hospedagem

Alguns hosts limitam solicitações de API:
- Verifique os detalhes do seu plano de hospedagem
- Considere atualizar se os limites forem muito baixos
- Entre em contato com o suporte sobre aumento de limites

### 5. Otimize a Frequência de Sincronização

Nas configurações do POS:
- Ajuste os intervalos de atualização automática
- Use sincronização manual quando apropriado
- Evite atualizações de dados desnecessárias
