# API03005: Limite de Taxa Excedido

## O Que Isso Significa

Você fez solicitações demais ao servidor em um curto período. O limite de taxa protege o servidor de ser sobrecarregado.

## Causas Comuns

- **Operações de sincronização rápidas** — Múltiplos dispositivos sincronizando simultaneamente
- **Atualização agressiva** — Atualizando dados com muita frequência
- **Plugin de segurança** — Um plugin está impondo limites de solicitações
- **Limites de hospedagem** — Seu plano de hospedagem tem limites de taxa da API

## Mapeamento de Erros do Servidor

Este código de erro é acionado quando o servidor retorna:

| Código do Servidor | Fonte |
|--------------------|-------|
| HTTP 429           | Qualquer resposta do servidor (fallback) |

## Como Corrigir

### 1. Aguarde e Tente Novamente

Os limites de taxa são temporários:
- Aguarde 1-5 minutos
- O POS irá automaticamente tentar novamente
- Evite atualizar manualmente repetidamente

### 2. Reduzir Operações Concorrentes

Se estiver usando múltiplos terminais POS:
- Espalhe a sincronização inicial entre os dispositivos
- Evite atualizar todos os dispositivos ao mesmo tempo
- Deixe um completar antes de iniciar outro

### 3. Verifique os Plugins de Segurança

Plugins como o Wordfence podem bloquear solicitações rápidas:
- Coloque o aplicativo POS na lista de permissões
- Coloque os IPs do seu dispositivo POS na lista de permissões
- Ajuste os limites de taxa

### 4. Revisar Limites de Hospedagem

Alguns provedores limitam as solicitações da API:
- Verifique os detalhes do seu plano de hospedagem
- Considere fazer upgrade se os limites forem muito baixos
- Entre em contato com o suporte a respeito do aumento dos limites

### 5. Otimize a Frequência de Sincronização

Nas configurações do POS:
- Ajuste os intervalos de autoatualização
- Use a sincronização manual quando apropriado
- Evite atualizações desnecessárias de dados
