# API02002: Token Expirado {#api02002-token-expired}

## O Que Isso Significa {#what-this-means}

Sua sessão de autenticação expirou. Por razões de segurança, as sessões de login não duram para sempre. O POS precisa atualizar sua autenticação ou você pode precisar fazer login novamente.

## Causas Comuns {#common-causes}

- **Longa inatividade** — Você não usou o POS por um período prolongado
- **Tempo de vida do token do servidor** — A expiração do token do servidor está configurada para um período muito curto
- **Desalinhamento do horário do servidor** — Os relógios do servidor e do dispositivo estão fora de sincronia

## Mapeamento de Erros do Servidor {#server-error-mapping}

Este código de erro é ativado quando o servidor retorna:

| Código do Servidor | Fonte |
|--------------------|-------|
| `jwt_auth_expired_token` | Plugin de Autenticação JWT |

## Como Corrigir {#how-to-fix}

### 1. Faça Login Novamente {#1-log-in-again}

A solução mais simples é sair e entrar novamente:
1. Saia do POS
2. Insira suas credenciais novamente
3. Isso cria uma nova sessão

### 2. Verifique a Atualização do Token {#2-check-token-refresh}

O POS deve atualizar os tokens automaticamente. Se não estiver funcionando:
- Verifique sua conexão com a internet
- Verifique se o servidor está acessível
- Veja [API02007](./API02007) se a atualização falhar

### 3. Verifique o Horário do Servidor {#3-check-server-time}

A expiração do token depende de um horário preciso:
- Certifique-se de que o relógio do seu servidor esteja correto
- Verifique se o horário do seu dispositivo está preciso
- Use a sincronização automática de horário em ambos

### 4. Revise as Configurações do Token {#4-review-token-settings}

Se os tokens expirarem muito rapidamente, você pode precisar ajustar as configurações do servidor (consulte seu provedor de hospedagem ou administrador do WordPress).
