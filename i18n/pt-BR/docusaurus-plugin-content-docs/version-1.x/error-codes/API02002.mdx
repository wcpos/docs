# API02002: Token Expirado

## O Que Isso Significa

Sua sessão de autenticação expirou. Por razões de segurança, as sessões de login não duram para sempre. O POS precisa atualizar sua autenticação ou você pode precisar fazer login novamente.

## Causas Comuns

- **Inatividade prolongada** — Você não usou o POS por um período extenso
- **Tempo de vida do token do servidor** — O tempo de expiração do token do servidor está definido muito curto
- **Dessincronização do horário do servidor** — Os relógios do servidor e do dispositivo estão dessincronizados

## Mapeamento de Erro do Servidor

Este código de erro é disparado quando o servidor retorna:

| Código do Servidor | Fonte |
|-------------|--------|
| `jwt_auth_expired_token` | Plugin de Autenticação JWT |

## Como Corrigir

### 1. Fazer Login Novamente

A solução mais simples é sair e fazer login novamente:
1. Saia do POS
2. Digite suas credenciais novamente
3. Isso criará uma nova sessão

### 2. Verificar Atualização do Token

O POS deve atualizar tokens automaticamente. Se não estiver funcionando:
- Verifique sua conexão com a internet
- Confirme se o servidor está acessível
- Consulte [API02007](./API02007) se a atualização estiver falhando

### 3. Verificar Horário do Servidor

A expiração do token depende de um horário preciso:
- Certifique-se de que o relógio do seu servidor esteja correto
- Verifique se o horário do seu dispositivo está preciso
- Use sincronização automática de horário em ambos

### 4. Revisar Configurações do Token

Se os tokens expirarem muito rapidamente, pode ser necessário ajustar as configurações do servidor (consulte seu provedor de hospedagem ou administrador do WordPress).
