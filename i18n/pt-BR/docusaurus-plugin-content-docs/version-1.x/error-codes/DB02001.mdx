# DB02001: Registro Duplicado

## O que Isso Significa

O banco de dados tentou inserir um registro com um ID que já existe. Cada registro deve ter um identificador único.

## Causas Comuns

- **Conflito de sincronização** — Os mesmos dados foram sincronizados duas vezes
- **Condição de corrida** — Várias operações criando o mesmo registro
- **Corrupção de dados** — IDs foram corrompidos ou duplicados
- **Problema de importação** — Importando dados que já existem

## Como Corrigir

### 1. Atualizar os Dados

Re-sincronizar a partir do servidor:
- Puxar os dados mais recentes
- Deixar o processo de sincronização resolver conflitos
- Os dados mais novos devem ter precedência

### 2. Tentar a Operação Novamente

Se estiver criando novos registros:
- O sistema pode já tê-lo criado
- Verifique se o registro existe
- Evite clicar em enviar várias vezes

### 3. Limpar o Cache Local

Redefinir os dados locais:
1. Limpar o banco de dados/cache local
2. Fazer login novamente
3. Sincronizar dados atualizados

### 4. Verificar Duplicatas no WooCommerce

Se o problema persistir:
- Verifique o WooCommerce em busca de entradas duplicadas
- Procure por produtos/pedidos com os mesmos IDs
- Elimine quaisquer duplicatas no admin

### 5. Relatar Problemas Persistentes

Se isso acontecer com frequência:
- Anote quais ações causam isso
- Verifique padrões
- Relate ao suporte com detalhes

## Por que Isso É Importante

IDs duplicados causariam:
- Confusão sobre qual registro é correto
- Sobrescritas de dados
- Problemas de sincronização

O erro previne a corrupção de dados.
