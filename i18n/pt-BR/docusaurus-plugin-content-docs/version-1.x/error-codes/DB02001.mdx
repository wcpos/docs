# DB02001: Registro Duplicado

## O Que Isso Significa

O banco de dados tentou inserir um registro com um ID que já existe. Cada registro deve ter um identificador único.

## Causas Comuns

- **Conflito de sincronização** — Mesmos dados sincronizados duas vezes
- **Condição de corrida** — Múltiplas operações criando o mesmo registro
- **Corrupção de dados** — IDs foram corrompidos ou duplicados
- **Problema de importação** — Importando dados que já existem

## Como Corrigir

### 1. Atualizar Dados

Ressincronizar do servidor:
- Puxar os dados mais recentes
- Deixar o processo de sincronização resolver conflitos
- Os dados mais novos devem ter precedência

### 2. Tentar a Operação Novamente

Ao criar novos registros:
- O sistema pode já ter criado
- Verificar se o registro existe
- Evitar clicar em enviar várias vezes

### 3. Limpar Cache Local

Redefinir dados locais:
1. Limpar banco de dados/cache local
2. Fazer login novamente
3. Sincronizar dados do zero

### 4. Verificar Duplicatas no WooCommerce

Se o problema persistir:
- Verificar duplicatas no WooCommerce
- Procurar produtos/pedidos com IDs iguais
- Limpar quaisquer duplicatas no painel administrativo

### 5. Reportar Problemas Persistentes

Se isso ocorrer frequentemente:
- Anotar quais ações causam
- Verificar padrões
- Reportar ao suporte com detalhes

## Por Que Isso Importa

IDs duplicados causariam:
- Confusão sobre qual registro está correto
- Substituições de dados
- Problemas de sincronização

O erro previne corrupção de dados.
