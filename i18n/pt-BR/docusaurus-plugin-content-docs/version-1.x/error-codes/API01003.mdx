# API01003: Reinício de Conexão

## O Que Isso Significa

A conexão com o servidor foi estabelecida, mas então foi inesperadamente fechada. Isso tipicamente acontece durante a comunicação quando algo interrompe a conexão.

## Causas Comuns

- **Queda do servidor** — O processo do servidor pode ter travado durante a solicitação
- **Limites de memória** — PHP ficou sem memória e foi encerrado
- **Tempo limite durante a solicitação** — O servidor excedeu o tempo limite ao processar
- **Instabilidade de rede** — Problemas intermitentes de rede
- **Software de segurança** — Configurações de segurança agressivas fechando conexões

## Como Corrigir

### 1. Verificar Logs de Erro do Servidor

Procure nos logs de erro do WordPress ou no painel de hospedagem por erros relacionados próximos ao momento do reinício.

### 2. Aumentar Limite de Memória do PHP

No seu `wp-config.php`:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. Verificar Conflitos de Plugins

Um plugin pode estar causando a queda do PHP:
- Desative temporariamente os plugins
- Reative um por um para identificar o culpado

### 4. Revisar Recursos do Servidor

Entre em contato com seu provedor de hospedagem para verificar:
- Limites de processos PHP
- Uso de memória no momento do erro
- Logs de estabilidade do servidor

### 5. Testar Estabilidade da Rede

Se estiver em WiFi, tente uma conexão com cabo. Se o problema persistir em diferentes redes, provavelmente é do lado do servidor.
