# API01003: Conexão Resetada

## O Que Isso Significa

A conexão com o servidor foi estabelecida, mas foi fechada inesperadamente. Isso geralmente acontece durante a comunicação, quando algo interrompe a conexão.

## Causas Comuns

- **Queda do servidor** — O processo do servidor pode ter falhado durante a requisição
- **Limites de memória** — O PHP ficou sem memória e foi encerrado
- **Timeout durante a requisição** — O servidor excedeu o tempo limite enquanto processava
- **Instabilidade na rede** — Problemas de rede intermitentes
- **Software de segurança** — Configurações de segurança agressivas que fecham conexões

## Como Corrigir

### 1. Verifique os Registros de Erro do Servidor

Verifique os registros de erro do WordPress ou o painel de controle de hospedagem em busca de erros relacionados ao momento do reset.

### 2. Aumente o Limite de Memória do PHP

No seu `wp-config.php`:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. Verifique por Conflitos de Plugin

Um plugin pode estar fazendo o PHP falhar:
- Desative os plugins temporariamente
- Reative um por um para encontrar o culpado

### 4. Revise os Recursos do Servidor

Entre em contato com seu provedor de hospedagem para verificar:
- Limites do processo PHP
- Uso de memória no momento do erro
- Registros de estabilidade do servidor

### 5. Teste a Estabilidade da Rede

Se estiver usando WiFi, tente uma conexão com fio. Se o problema persistir em diferentes redes, é provável que seja do lado do servidor.
