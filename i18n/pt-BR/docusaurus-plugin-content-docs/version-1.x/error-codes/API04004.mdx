# API04004: Dados de Resposta Ausentes

## O Que Isso Significa

O servidor respondeu com sucesso, mas a resposta está sem os dados esperados. O JSON é válido, mas não contém as informações necessárias para o PDV.

## Causas Comuns

- **Resultados vazios** — Nenhum dado corresponde à consulta
- **Restrições de permissão** — Dados filtrados devido a permissões
- **Filtragem de plugin** — Outro plugin filtrando respostas da API
- **Incompatibilidade de versão** — Diferenças na versão da API
- **Problemas de banco de dados** — Dados não presentes no WooCommerce

## Como Corrigir

### 1. Verificar Existência dos Dados

Verifique no Administrador do WordPress:
- Existem produtos no WooCommerce?
- Existem clientes para carregar?
- O item específico existe?

### 2. Verificar Permissões de Usuário

Seu usuário pode não ter acesso a todos os dados:
- Verificar recursos da função de usuário
- Verificar configurações de acesso do PDV
- Tentar com uma conta de administrador

### 3. Verificar Filtros de Resposta da API

Alguns plugins filtram respostas da API REST:
- Desativar plugins de filtragem temporariamente
- Verificar filtros personalizados de API no tema
- Revisar configurações de plugin de segurança

### 4. Atualizar Ambos os Componentes

Incompatibilidades de versão podem causar problemas:
- Atualizar plugin WCPOS
- Atualizar o aplicativo PDV
- Verificar notas de compatibilidade

### 5. Verificar Dados do WooCommerce

No WooCommerce:
- Verificar se os produtos estão publicados (não em rascunho)
- Verificar se os itens estão marcados como visíveis
- Garantir que os dados não estejam corrompidos

## Vazio vs. Ausente

- **Resposta vazia** — Resposta válida sem resultados (pode ser esperado)
- **Campos ausentes** — Resposta sem campos de dados necessários (este erro)
