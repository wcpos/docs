:::note Somente Desktop e Mobile
Esta tela é exibida apenas nos aplicativos Desktop e Mobile. Usuários da Web acessam o POS diretamente em `yourdomain.com/pos` e fazem login com suas credenciais do WordPress.

## Visão geral da tela de conexão {#connect-screen-overview}

Ao abrir o aplicativo WCPOS Desktop ou Mobile, será exibida a tela de conexão. É nela que você gerencia as conexões da sua loja e os logins de usuários.

{/* Screenshot: connect-screen.png */}

## Adicionando uma nova loja {#adding-a-new-store}

1. Insira a URL da sua loja WooCommerce no campo de texto (por exemplo, `https://mystore.com`)
2. Clique em **Conectar**
3. Você será redirecionado para entrar com suas credenciais do WordPress
4. Após o login bem-sucedido, você retornará ao aplicativo

## Várias lojas {#multiple-stores}

Você pode se conectar a quantas lojas WooCommerce precisar. Cada loja aparece como um cartão separado na tela Conectar, mostrando:

- **Nome da loja** e favicon
- **URL da loja**
- **Usuários conectados** nessa loja

Isso é útil se você gerencia vários locais ou negócios.

## Vários usuários por loja {#multiple-users-per-store}

Cada loja pode ter vários usuários conectados simultaneamente. Isso é útil para:

- **Trocas de turno** - O novo caixa pode entrar antes que o anterior saia
- **Configurações com vários caixas registradores** - Diferentes caixas em diferentes dispositivos
- **Alternância rápida** - Alterne facilmente entre contas de usuário

### Adicionando um usuário {#adding-a-user}

Clique no <Icon name="plus" /> botão ao lado de "Usuários conectados" para adicionar outro usuário a essa loja.

### Alternando usuários {#switching-users}

Clique em um emblema de usuário (por exemplo, "Brenda") para abrir o POS como esse usuário.

### Removendo um usuário {#removing-a-user}

Clique no **×** em um emblema de usuário para desconectar esse usuário da loja.

## Removendo uma loja {#removing-a-store}

Clique no botão vermelho **×** no cartão da loja para removê-la da sua lista. Isso desconecta todos os usuários e remove a conexão da loja do aplicativo.

## Loja de demonstração {#demo-store}

Na parte inferior da tela, você pode ver um link "Entrar na loja de demonstração". Ele conecta você a uma loja WooCommerce de demonstração para testar os recursos do WCPOS sem afetar os dados da sua própria loja.

## Solução de problemas {#troubleshooting}

:::tip Primeira coisa a verificar: `X-Frame-Options`
Os apps para desktop e dispositivos móveis usam **iframes** para login, pagamento e recibos. **Qualquer** cabeçalho de servidor ou plugin que envie `X-Frame-Options: DENY` ou `SAMEORIGIN` interromperá o login. Essa é a causa mais comum de falhas de login no app: verifique os cabeçalhos de resposta da página de login (ferramentas de desenvolvedor do navegador ou `curl -I https://yourstore.com/wp-login.php`) antes de qualquer outra coisa.
:::

<Accordion>
  <AccordionItem question="Não consegue se conectar à minha loja?">

  - Verifique se o plugin WCPOS está instalado e ativado no seu site WordPress
  - Verifique se você está usando a URL correta (inclua `https://` — a WooCommerce REST API requer SSL)
  - Tente abrir `yourdomain.com/pos` em um navegador da web primeiro para confirmar que o plugin funciona
  - Verifique se a WooCommerce REST API está acessível
  - Verifique se sua conta de usuário tem permissões de acesso ao POS

  </AccordionItem>

  <AccordionItem question="O login falha no app para desktop ou dispositivos móveis">

  A maioria das falhas de login no app é causada por um plugin de segurança ou cache que bloqueia o iframe de login:

  - Os cabeçalhos **`X-Frame-Options`** (definidos por um plugin de segurança ou pelo servidor) bloqueiam o iframe de login — veja a observação acima. Desative temporariamente o plugin de segurança, faça login e depois reative-o (sua sessão dura cerca de uma semana).
  - **Plugins de segurança** — Wordfence, Really Simple Security, WPS Hide Login, iThemes/Solid Security e Defender Pro são causas comuns. Veja a lista completa e as correções em [Conflitos de plugins](/support/troubleshooting/plugin-conflicts#security-and-login-plugins).
  - **2FA do Wordfence** — o campo do código 2FA não é renderizado no iframe de login. Por enquanto, desative a 2FA para usuários do POS.
  - **URL de login personalizada** (por exemplo, WPS Hide Login) — o aplicativo não consegue encontrar a página de login. Use a URL padrão `/wp-admin/`.
  - **Plugins de cache** podem continuar servindo o formulário de login bloqueado mesmo depois que o plugin problemático é desativado — limpe o cache, ou limpe o cache do aplicativo / reinstale o aplicativo para desktop.

  </AccordionItem>

  <AccordionItem question='"A REST API requer autenticação" ou um erro de plugin de segurança na tela de conexão'>

  Um plugin (por exemplo, Force Login ou um plugin de autenticação JWT) está exigindo autenticação para todas as solicitações da REST API, então o aplicativo não consegue ler as informações públicas do seu site. O aplicativo agora mostra a mensagem real do servidor (por exemplo, _"Somente usuários autenticados podem acessar a REST API"_) em vez de informar incorretamente o tipo de site.

  **Correção:** configure o plugin de segurança para permitir acesso não autenticado a `/wp-json/wcpos/` e `/wp-json/wc/v3/`, ou desative-o apenas pelo tempo necessário para concluir a primeira conexão.

  </AccordionItem>

  <AccordionItem question='"Não parece ser um site WordPress" (aplicativo para desktop)'>

  O aplicativo para desktop descobre a REST API por meio do cabeçalho HTTP `Link`. Se um plugin (comumente **Image Prioritizer** ou outros plugins de desempenho) sobrecarregar ou truncar esse cabeçalho, a descoberta falhará. **Correção:** desative plugins de desempenho que otimizam imagens / modificam cabeçalhos e tente novamente.

  </AccordionItem>

  <AccordionItem question="O aplicativo diz que precisa de uma atualização / trava após uma atualização">

  Verifique se há incompatibilidade de versão entre o aplicativo e o plugin do servidor — a loja de aplicativos pode ter enviado uma atualização do aplicativo enquanto o plugin WCPOS ainda precisa ser atualizado (ou vice-versa). Certifique-se de que o aplicativo e o plugin WCPOS estejam na **mesma versão principal**. Aplicativos WCPOS v1.9.0+ exigem que o plugin WCPOS seja **v1.8.0 ou superior**.

  </AccordionItem>

  <AccordionItem question='Erro "Não foi possível criar o banco de dados rápido da loja"'>

  Esta é uma condição de corrida no primeiro login. **Feche o aplicativo completamente e tente novamente** — geralmente funciona na segunda tentativa.

  </AccordionItem>

  <AccordionItem question="Travado na tela de seleção de usuário (aplicativo para desktop)">

  Após fazer login, o nome de usuário aparece, mas não há uma forma óbvia de avançar. **Clique no seu nome de usuário/nome** para entrar no POS — o próprio nome é o botão.

  </AccordionItem>

  <AccordionItem question="A conexão continua falhando?">

  - Tente acessar `yourdomain.com/pos` em um navegador primeiro para verificar se o plugin está funcionando
  - Verifique os logs de erro do site para identificar possíveis problemas
  - Confirme se o provedor de hospedagem não está bloqueando a REST API — consulte as [observações específicas sobre hospedagem](/support/performance/server#hosting-specific-notes)
  - Verifique se o servidor atende aos [requisitos mínimos](/support/performance/server#minimum-server-requirements)

  </AccordionItem>
</Accordion>
