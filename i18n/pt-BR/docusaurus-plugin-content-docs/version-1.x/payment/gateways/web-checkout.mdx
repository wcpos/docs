# Gateway de checkout web {#web-checkout-gateway}

O gateway de checkout web permite que os clientes concluam suas transações de POS usando o sistema de checkout da sua loja virtual. Este gateway cria uma ponte contínua entre o POS na loja e os métodos de pagamento online, ideal para situações em que os clientes preferem pagar usando seus próprios dispositivos ou quando é necessário acessar métodos de pagamento disponíveis apenas na web.

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Baixar e instalar">

1. Acesse a [página de versões do gateway de checkout web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Baixe o arquivo **woocommerce-pos-web-checkout-gateway.zip** mais recente
3. No admin do WordPress, acesse `Plugins > Adicionar novo > Enviar plugin`
4. Envie o arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Ativar no WCPOS" isLast>

1. Acesse `WP Admin > POS > Configurações > Finalização de compra`
2. Encontre o **Gateway de finalização de compra pela web do WCPOS** na lista
3. Ative o gateway para uso no POS
4. Salve suas configurações

:::note
Este gateway usa os métodos de pagamento WooCommerce existentes, portanto, certifique-se de ter pelo menos um gateway de pagamento web configurado corretamente (Stripe, PayPal etc.).
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Como iniciar o checkout web {#initiating-web-checkout}

1. **Adicionar itens**: adicione produtos ao carrinho no POS
2. **Selecionar gateway**: escolha "Checkout web" como método de pagamento
3. **Gerar link**: o sistema cria um link de checkout exclusivo para o pedido
4. **Acesso do cliente**: o cliente clica no link para acessar o checkout web
5. **Pagamento online**: o cliente conclui o pagamento usando os métodos de pagamento da loja virtual

### Processo de pagamento {#payment-process}

O gateway de Checkout web segue este fluxo de trabalho:

1. **Criação do pedido**: o POS cria um pedido com status Aguardando pagamento no WooCommerce
2. **Link de finalização de compra**: o sistema gera um link de pagamento seguro e exclusivo
3. **Redirecionamento do cliente**: o cliente é direcionado para a página de finalização de compra na web
4. **Seleção de pagamento**: o cliente escolhe entre os métodos de pagamento disponíveis na web
5. **Processamento do pagamento**: o processo padrão de finalização de compra do WooCommerce gerencia o pagamento
6. **Conclusão do pedido**: o status do pedido é atualizado automaticamente após o pagamento bem-sucedido

### Fluxo de trabalho do POS {#pos-workflow}

Depois de iniciar a finalização de compra na web:

1. **Monitorar status**: mantenha a tela do pedido no POS aberta para monitorar o status do pagamento
2. **Pagamento do cliente**: o cliente conclui o pagamento no próprio dispositivo ou no seu tablet
3. **Atualização automática**: o status do pedido é atualizado em tempo real quando o pagamento é recebido
4. **Processar pagamento**: clique no botão "Processar pagamento" no POS para continuar para o recibo
5. **Geração de recibo**: Gere e imprima o recibo normalmente

## Casos de uso {#use-cases}

### Ideal para {#perfect-for}

- **Preferência do cliente**: Clientes que preferem usar seus próprios aplicativos de pagamento ou cartões
- **Pagamentos complexos**: Transações que exigem métodos de pagamento não disponíveis no POS
- **Pagamentos divididos**: Clientes que desejam usar vários métodos de pagamento
- **Programas de fidelidade**: Acesso ao resgate de pontos de fidelidade pela web
- **Cartões-presente**: Clientes com cartões-presente digitais ou crédito da loja
- **Cartões internacionais**: Métodos de pagamento que funcionam melhor por meio de gateways web

## Detalhes técnicos {#technical-details}

### Gerenciamento de pedidos {#order-management}

- **Pedidos aguardando pagamento**: os pedidos são criados com o status "Aguardando pagamento"
- **Atualizações de status**: alterações automáticas de status quando o pagamento é recebido
- **Notas do pedido**: os detalhes do pagamento são registrados nas notas do pedido
- **Estoque**: o estoque é reservado durante o processo de finalização da compra

### Segurança {#security}

- **Links exclusivos**: cada sessão de finalização da compra tem uma URL exclusiva e com tempo limitado
- **SSL obrigatório**: todo o processamento de pagamentos ocorre por conexões HTTPS seguras
- **Conformidade com PCI**: utiliza seus gateways de pagamento web existentes em conformidade com PCI
- **Gerenciamento de sessão**: o tratamento seguro da sessão impede acesso não autorizado

### Métodos de pagamento {#payment-methods}

O gateway fornece acesso a todos os seus métodos de pagamento WooCommerce configurados:

- **Cartões de crédito/débito**: Stripe, Square, Authorize.net, etc.
- **Carteiras digitais**: PayPal, Apple Pay, Google Pay
- **Transferências bancárias**: métodos de pagamento bancário direto
- **Compre agora, pague depois**: Klarna, Afterpay, Sezzle
- **Criptomoeda**: Bitcoin e outros gateways de pagamento com criptomoedas
- **Métodos locais**: métodos de pagamento específicos da região

## Solução de problemas {#troubleshooting}

### Problemas comuns {#common-issues}

<Accordion>
  <AccordionItem question="Link de finalização de compra não funciona">

  - Verifique se o certificado SSL está instalado corretamente
  - Verifique se os links permanentes do WooCommerce estão configurados corretamente
  - Certifique-se de que os gateways de pagamento pela web estejam ativos e configurados
  - Teste com outro navegador ou dispositivo

  </AccordionItem>

  <AccordionItem question="Pagamento não está sendo processado">

  - Confirme se os gateways de pagamento web estão configurados corretamente
  - Verifique se há mensagens de erro nos logs do gateway de pagamento
  - Verifique se o método de pagamento do cliente é compatível
  - Teste o processo de finalização da compra na web de forma independente

  </AccordionItem>

  <AccordionItem question="Pedido não é atualizado no POS">

  - Verifique a estabilidade da conexão com a internet
  - Verifique se as tarefas cron do WordPress estão sendo executadas corretamente
  - Atualize a tela de pedidos do POS
  - Verifique o status do pedido no admin do WooCommerce

  </AccordionItem>

  <AccordionItem question="Cliente não consegue acessar a finalização da compra">

  - Certifique-se de que o link de finalização da compra não expirou
  - Verifique se o dispositivo do cliente tem conectividade com a internet
  - Verifique se o pedido não foi cancelado ou concluído
  - Teste o link em um dispositivo diferente

  </AccordionItem>
</Accordion>

### Otimização de desempenho {#performance-optimisation}

Para obter o melhor desempenho:

- **Cache**: Configure plugins de cache para excluir páginas de finalização de compra
- **CDN**: Certifique-se de que as configurações de CDN não interfiram no processo de finalização de compra
- **Banco de dados**: Otimize o banco de dados para um processamento de pedidos mais rápido
- **Hospedagem**: Use uma hospedagem confiável com boa disponibilidade

### Como obter ajuda {#getting-help}

Para suporte técnico:
- Acesse o [repositório do GitHub](https://github.com/wcpos/web-checkout-gateway) para relatar problemas
- Consulte a documentação de gateways de pagamento do WooCommerce
- Teste regularmente o processo de checkout na web
- Monitore os logs do gateway de pagamento para identificar problemas
