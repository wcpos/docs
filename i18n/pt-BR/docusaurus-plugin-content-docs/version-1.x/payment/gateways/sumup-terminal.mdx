# Método de pagamento SumUp Terminal {#sumup-terminal-gateway}

O método de pagamento SumUp Terminal permite aceitar pagamentos com cartão usando leitores de cartão SumUp diretamente no WCPOS. Esse método oferece integração contínua com o sistema de processamento de pagamentos da SumUp para transações presenciais.

## Recursos {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="Integração com hardware">
    Conecte leitores de cartão SumUp ao seu sistema POS por meio de uma conexão com a internet
  </FeatureCard>
  <FeatureCard icon="bolt" title="Processamento em tempo real">
    Processamento instantâneo de pagamentos e conclusão de pedidos
  </FeatureCard>
  <FeatureCard icon="shield" title="Transações seguras">
    Processamento de pagamentos compatível com PCI por meio da SumUp
  </FeatureCard>
  <FeatureCard icon="link" title="Pareamento fácil">
    Processo simples de pareamento de dispositivos com códigos de pareamento
  </FeatureCard>
</FeatureGrid>

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Instalar o SumUp Terminal for WooCommerce">

Instale por `WP Admin > POS > Configurações > Extensões` ou baixe o **arquivo zip do plugin** mais recente (não o zip ou tarball do código-fonte do GitHub) na [página de versões do GitHub](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) e envie-o por `Plugins > Adicionar novo > Enviar plugin`.

  </StepCard>
  <StepCard number="2" title="Configurar as definições da SumUp">

1. Acesse `WP Admin > WooCommerce > Configurações > Pagamentos`
2. Localize **SumUp Terminal** na lista de métodos de pagamento
3. Clique em **SumUp Terminal** para acessar as configurações
4. Insira sua **chave de API da SumUp** (disponível no painel do comerciante da SumUp)
5. Salve as configurações

:::note
Não é necessário ativar o meio de pagamento SumUp Terminal nas configurações do WooCommerce. Ele será ativado especificamente para o POS em uma etapa posterior.
:::

  </StepCard>
  <StepCard number="3" title="Pareie o SumUp Terminal">

1. Na mesma página de configurações, localize a seção **Parear leitor**
2. No dispositivo SumUp, navegue até a tela de pareamento para exibir o código de pareamento
3. Insira o código de pareamento exibido no dispositivo SumUp
4. Clique em **"Parear leitor"** para estabelecer a conexão
5. Aguarde a confirmação de que o leitor foi pareado com sucesso

:::warning Importante
O leitor deve ser pareado com sucesso antes que pagamentos possam ser processados. Certifique-se de concluir o processo de pareamento antes de prosseguir.
:::

  </StepCard>
  <StepCard number="4" title="Ativar no WCPOS" isLast>

1. Acesse `WP Admin > POS > Configurações > Finalização de compra`
2. Encontre o gateway **SumUp Terminal** na lista
3. Ative o gateway para uso no POS
4. Salve suas configurações

  </StepCard>
</Steps>

## Uso {#usage}

### Processamento de pagamentos {#processing-payments}

1. **Adicionar itens**: adicione produtos ao carrinho no POS
2. **Selecionar gateway**: escolha "SumUp Terminal" como forma de pagamento
3. **Iniciar pagamento**: clique para iniciar um novo pagamento no seu dispositivo SumUp
4. **Pagamento do cliente**: o cliente conclui o pagamento no terminal SumUp
5. **Conclusão automática**: o pedido é concluído automaticamente quando o pagamento é detectado

### Controles de pagamento {#payment-controls}

Ao usar o gateway SumUp Terminal, você tem as seguintes opções:

- **Iniciar novo pagamento**: Inicia uma solicitação de pagamento no terminal conectado
- **Cancelar pagamento**: Cancela um pagamento atualmente em andamento
- **Status do pagamento**: Monitora o status atual do processo de pagamento

### Gerenciamento de pedidos {#order-management}

- **Atualizações automáticas**: Os pedidos são marcados automaticamente como concluídos após o pagamento bem-sucedido
- **Rastreamento de pagamento**: Todos os detalhes do pagamento são registrados nas notas do pedido
- **Geração de recibo**: Recibos padrão do POS são gerados após pagamentos bem-sucedidos

## Compatibilidade de hardware {#hardware-compatibility}

:::important Requisitos de conectividade
Esta implementação usa o SDK JavaScript da SumUp, o que significa que funciona por meio de aplicações web, mas requer **apenas terminais conectados à internet**. Terminais Bluetooth não são compatíveis no momento.
:::

### Terminais compatíveis (conectados à internet) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Terminal conectado por Wi-Fi/Ethernet

### Terminais não compatíveis (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - somente Bluetooth
- **SumUp 3G** ❌ - somente Bluetooth

:::note Suporte futuro
O suporte a terminais Bluetooth está planejado para uma versão futura do aplicativo para iOS e Android. Quando disponível, isso permitirá suporte a todos os terminais certificados pela SumUp, incluindo os modelos Air e 3G.
:::

## Solução de problemas {#troubleshooting}

### Problemas comuns {#common-issues}

<Accordion>
  <AccordionItem question="O leitor não emparelha">

  - Verifique se está usando um terminal compatível conectado à internet (somente Solo)
  - Verifique se o dispositivo SumUp Solo está conectado ao WiFi e online
  - Verifique se o dispositivo está no modo de emparelhamento e se o código de emparelhamento foi inserido corretamente
  - Confirme se a sua conta SumUp tem acesso à API ativado
  - Garanta uma conexão estável com a internet durante o pareamento

  </AccordionItem>

  <AccordionItem question="Pagamento não está sendo processado">

  - Verifique se o leitor está pareado corretamente
  - Verifique se a sua conta SumUp está ativa e em situação regular
  - Garanta que o dispositivo tenha bateria e conectividade suficientes
  - Confirme se a chave de API está correta e ativa

  </AccordionItem>

  <AccordionItem question="Pedido não está sendo concluído">

  - Verifique a estabilidade da conexão com a internet
  - Verifique as configurações de webhook na sua conta SumUp
  - Garanta que o pagamento tenha sido bem-sucedido no terminal SumUp
  - Verifique os logs de erro do WordPress para identificar problemas de comunicação da API

  </AccordionItem>
</Accordion>

### Limites de taxa da API {#api-rate-limits}

O gateway é otimizado para reduzir chamadas de API e evitar limitação de taxa:
- O status do pagamento é verificado com eficiência
- Solicitações de API desnecessárias são minimizadas
- A lógica de nova tentativa automática lida com falhas temporárias
