# Portal de Pagamento SumUp {#sumup-terminal-gateway}

O portal de pagamento SumUp permite que você aceite pagamentos com cartão usando leitores de cartões SumUp diretamente no WCPOS. Este portal fornece integração perfeita com o sistema de processamento de pagamentos da SumUp para transações presenciais.

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Acesse a [página de lançamentos do SumUp Terminal para WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Baixe o arquivo mais recente **sumup-terminal-for-woocommerce.zip**
3. No seu painel do WordPress, vá para `Plugins > Adicionar Novo > Fazer Upload do Plugin`
4. Faça o upload do arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Configurar as Configurações da SumUp">

1. Navegue até `WP Admin > WooCommerce > Configurações > Pagamentos`
2. Encontre **SumUp Terminal** na lista de métodos de pagamento
3. Clique em **SumUp Terminal** para acessar as configurações
4. Digite sua **Chave da API SumUp** (disponível no painel do comerciante SumUp)
5. Salve as configurações

:::note
Você não precisa habilitar o portal de pagamento SumUp nas configurações do WooCommerce. Ele será habilitado especificamente para o POS em uma etapa posterior.
:::

  </StepCard>
  <StepCard number="3" title="Emparelhe seu Terminal SumUp">

1. Na mesma página de configurações, localize a seção **Emparelhar Leitor**
2. No seu dispositivo SumUp, acesse a tela de emparelhamento para exibir o código de emparelhamento
3. Digite o código de emparelhamento exibido no seu dispositivo SumUp
4. Clique em **"Emparelhar Leitor"** para estabelecer a conexão
5. Aguarde a confirmação de que o leitor foi emparelhado com sucesso

:::warning Importante
O leitor deve ser emparelhado com sucesso antes que você possa processar pagamentos. Certifique-se de que o processo de emparelhamento seja concluído antes de prosseguir.
:::

  </StepCard>
  <StepCard number="4" title="Habilitar no WCPOS" isLast>

1. Vá para `WP Admin > POS > Configurações > Finalização`
2. Encontre o portal **SumUp Terminal** na lista
3. Habilite o portal para uso no POS
4. Salve suas configurações

  </StepCard>
</Steps>

## Uso {#usage}

### Processando Pagamentos {#processing-payments}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Portal**: Escolha "SumUp Terminal" como método de pagamento
3. **Iniciar Pagamento**: Clique para iniciar um novo pagamento no seu dispositivo SumUp
4. **Pagamento do Cliente**: O cliente completa o pagamento no terminal SumUp
5. **Finalização Automática**: O pedido é finalizado automaticamente quando o pagamento é detectado

### Controles de Pagamento {#payment-controls}

Ao usar o portal SumUp Terminal, você tem as seguintes opções:

- **Iniciar Novo Pagamento**: Inicie uma solicitação de pagamento no terminal conectado
- **Cancelar Pagamento**: Cancele um pagamento que está em processamento
- **Status do Pagamento**: Monitore o status atual do processo de pagamento

### Gerenciamento de Pedidos {#order-management}

- **Atualizações Automáticas**: Os pedidos são automaticamente marcados como concluídos após o pagamento bem-sucedido
- **Rastreamento de Pagamento**: Todos os detalhes do pagamento são registrados nas notas do pedido
- **Geração de Recibos**: Recibos padrão do POS são gerados após pagamentos bem-sucedidos

## Compatibilidade de Hardware {#hardware-compatibility}

:::important Requisitos de Conectividade
Esta implementação utiliza o SDK JavaScript da SumUp, o que significa que funciona através de aplicações web, mas requer **terminais conectados à internet apenas**. Terminais Bluetooth não são suportados atualmente.
:::

### Terminais Suportados (Conectados à Internet) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - Terminal conectado via WiFi/Ethernet

### Terminais Não Suportados (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Somente Bluetooth
- **SumUp 3G** ❌ - Somente Bluetooth

:::note Suporte Futuro
O suporte para terminais Bluetooth será adicionado aos aplicativos iOS e Android em **Q1 2026**. Isso permitirá suporte para todos os terminais certificados pela SumUp, incluindo os modelos Air e 3G.
:::

## Solução de Problemas {#troubleshooting}

### Problemas Comuns {#common-issues}

<Accordion>
  <AccordionItem question="Leitor não emparelha">

  - Certifique-se de que você está usando um terminal conectado à internet compatível (somente Solo)
  - Verifique se seu dispositivo SumUp Solo está conectado ao WiFi e online
  - Confira se o dispositivo está no modo de emparelhamento e se o código de emparelhamento foi inserido corretamente
  - Confirme que sua conta SumUp tem acesso à API habilitado
  - Certifique-se de que a conexão de internet esteja estável durante o emparelhamento

  </AccordionItem>

  <AccordionItem question="Pagamento não processando">

  - Verifique se o leitor está devidamente emparelhado
  - Confira se sua conta SumUp está ativa e em boas condições
  - Certifique-se de que o dispositivo tenha bateria suficiente e conectividade
  - Confirme se a chave da API está correta e ativa

  </AccordionItem>

  <AccordionItem question="Pedido não completando">

  - Verifique a estabilidade da conexão de internet
  - Confirme as configurações do webhook na sua conta SumUp
  - Certifique-se de que o pagamento foi bem-sucedido no terminal SumUp
  - Verifique os logs de erro do WordPress para problemas de comunicação com a API

  </AccordionItem>
</Accordion>

### Limites de Taxa da API {#api-rate-limits}

O portal é otimizado para reduzir chamadas à API e evitar limitação de taxa:
- O status do pagamento é verificado de maneira eficiente
- Chamadas desnecessárias à API são minimizadas
- Lógica de reexibição automática lida com falhas temporárias
