# Gateway de Fatura por Email {#email-invoice-gateway}

O gateway de Fatura por Email permite que você envie faturas de pagamento para clientes via email diretamente do WCPOS. Isso é perfeito para situações em que os clientes preferem pagar depois ou precisam completar o pagamento online usando os métodos de pagamento da sua loja virtual.

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Visite a [página de lançamentos do Gateway de Fatura por Email](https://github.com/wcpos/email-invoice-gateway/releases)
2. Baixe o arquivo **woocommerce-pos-email-invoice-gateway.zip** mais recente
3. No admin do WordPress, vá para `Plugins > Adicionar Novo > Fazer Upload do Plugin`
4. Faça upload do arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Habilitar no WCPOS" isLast>

1. Vá para `WP Admin > PDV > Configurações > Checkout`
2. Encontre o **Gateway de Pagamento por Fatura WCPOS** na lista
3. Habilite o gateway para uso no PDV
4. Salve suas configurações

:::note
Este gateway não requer chaves de API adicionais ou configuração de serviços externos. Ele utiliza o sistema de email existente do WordPress e os métodos de pagamento do WooCommerce.
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Enviando Faturas {#sending-invoices}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no PDV
2. **Selecionar Gateway**: Escolha "Fatura por Email" como o método de pagamento
3. **Endereço de Email**: 
   - Para clientes existentes: O email é preenchido automaticamente a partir dos dados do cliente
   - Para pedidos de convidados: Insira o endereço de email do cliente manualmente
4. **Opcional**: Marque "Salvar como endereço de cobrança" para atualizar as informações de cobrança do cliente
5. **Enviar Fatura**: Complete o processo para enviar o email da fatura

### Experiência do Cliente {#customer-experience}

Quando você envia uma fatura, o cliente recebe um email contendo:

- **Detalhes do Pedido**: Detalhamento completo de itens, quantidades e preços
- **Total do Pedido**: Valor total devido incluindo impostos e taxas
- **Link de Pagamento**: Link direto para completar o pagamento online
- **Informações do Pedido**: Número do pedido e detalhes de referência

### Conclusão do Pagamento {#payment-completion}

Os clientes podem completar o pagamento de duas maneiras:

1. **Pagamento Online**: Clique no link "Pagar por este pedido" no email
2. **Retorno na Loja**: Retorne para completar o pagamento usando outros métodos de pagamento do PDV

O link "Pagar por este pedido" leva os clientes a uma página de pagamento dedicada em seu site onde eles podem usar qualquer um de seus gateways de pagamento web habilitados (Stripe, PayPal, etc.).

## Gerenciamento de Email {#email-management}

### Clientes Existentes {#existing-customers}

- Os endereços de email são recuperados automaticamente dos perfis dos clientes
- As informações de cobrança do cliente são preservadas
- O histórico de pedidos é mantido sob a conta do cliente

### Clientes Convidados {#guest-customers}

- Insira manualmente endereços de email para clientes ocasionais
- Opção de salvar o email no endereço de cobrança para referência futura
- Os pedidos de convidados são devidamente rastreados no WooCommerce

### Modelos de Email {#email-templates}

O gateway usa os modelos padrão de email do WooCommerce:

- **Customizável**: Modifique a aparência do email através das configurações de email do WooCommerce
- **Personalizado**: Inclua o logotipo e a marca da sua loja
- **Profissional**: Formato de fatura limpo e profissional
- **Amigável para Dispositivos Móveis**: Design responsivo para todos os dispositivos

## Casos de Uso {#use-cases}

### Perfeito Para {#perfect-for}

- **Vendas B2B**: Clientes empresariais que precisam processar pagamentos através de seus sistemas contábeis
- **Grandes Pedidos**: Transações de alto valor que requerem aprovação ou tempo de processamento
- **Clientes Remotos**: Clientes que precisam completar o pagamento após sair da loja
- **Clientes com Conta**: Clientes regulares com termos de pagamento estabelecidos
- **Orçamento para Pedido**: Convertendo orçamentos ou estimativas em faturas pagáveis

## Solução de Problemas {#troubleshooting}

### Problemas Comuns {#common-issues}

<Accordion>
  <AccordionItem question="Emails não estão sendo enviados">

  - Verifique a configuração de email do WordPress
  - Verifique as configurações de SMTP se estiver usando um serviço de email personalizado
  - Teste com plugins de teste de email do WordPress
  - Verifique as pastas de spam para emails de teste

  </AccordionItem>

  <AccordionItem question="Links de pagamento não funcionam">

  - Certifique-se de que o WooCommerce está configurado corretamente
  - Verifique se pelo menos um gateway de pagamento está habilitado para checkout web
  - Verifique se o certificado SSL está instalado corretamente
  - Confirme se o status do pedido permite pagamento

  </AccordionItem>

  <AccordionItem question="Cliente não consegue completar o pagamento">

  - Verifique se os gateways de pagamento web estão ativos e configurados
  - Verifique se o pedido não expirou ou foi cancelado
  - Certifique-se de que o cliente tem limites suficientes para o método de pagamento
  - Teste o processo de pagamento você mesmo

  </AccordionItem>
</Accordion>

### Entrega de Email {#email-delivery}

Para uma entrega de email confiável, considere:

- **Serviço de SMTP**: Use serviços como SendGrid, Mailgun ou Amazon SES
- **Plugins de Email**: Instale plugins SMTP do WordPress para melhor entrega
- **Autenticação de Domínio**: Configure registros SPF, DKIM e DMARC
- **Monitoramento**: Acompanhe a entrega de emails e as taxas de abertura
