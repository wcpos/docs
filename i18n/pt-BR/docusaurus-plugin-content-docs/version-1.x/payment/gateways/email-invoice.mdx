# Gateway de fatura por e-mail {#email-invoice-gateway}

O gateway de fatura por e-mail permite enviar faturas de pagamento aos clientes por e-mail diretamente pelo WCPOS. Isso é ideal para situações em que os clientes preferem pagar depois ou precisam concluir o pagamento online usando os métodos de pagamento da sua loja virtual.

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Instalar gateway de fatura por e-mail">

Instale em `WP Admin > POS > Configurações > Extensões` ou baixe o **asset ZIP do plugin** mais recente (não o ZIP ou tarball do código-fonte do GitHub) na [página de releases do GitHub](https://github.com/wcpos/email-invoice-gateway/releases) e envie-o por meio de `Plugins > Adicionar novo > Enviar plugin`.

  </StepCard>
  <StepCard number="2" title="Ativar no WCPOS" isLast>

1. Acesse `WP Admin > POS > Configurações > Finalização de compra`
2. Encontre o **gateway de pagamento por fatura WCPOS** na lista
3. Ative o gateway para uso no POS
4. Salve suas configurações

:::note
Este gateway não exige chaves de API adicionais nem configuração de serviço externo. Ele usa o sistema de e-mail existente do WordPress e os métodos de pagamento do WooCommerce.
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Envio de faturas {#sending-invoices}

1. **Adicionar itens**: adicione produtos ao carrinho no POS
2. **Selecionar gateway**: escolha "Fatura por e-mail" como forma de pagamento
3. **Endereço de e-mail**: 
   - Para clientes existentes: o e-mail é preenchido automaticamente a partir dos dados do cliente
   - Para pedidos de visitantes: insira manualmente o endereço de e-mail do cliente
4. **Opcional**: marque "Salvar no endereço de cobrança" para atualizar as informações de cobrança do cliente
5. **Enviar fatura**: conclua o processo para enviar o e-mail da fatura

### Experiência do cliente {#customer-experience}

Ao enviar uma fatura, o cliente recebe um e-mail contendo:

- **Detalhes do pedido**: detalhamento completo de itens, quantidades e preços
- **Total do pedido**: Valor final devido, incluindo impostos e taxas
- **Link de pagamento**: Link direto para concluir o pagamento online
- **Informações do pedido**: Número do pedido e detalhes de referência

### Conclusão do pagamento {#payment-completion}

Os clientes podem concluir o pagamento de duas formas:

1. **Pagamento online**: Clicar no link "Pagar por este pedido" no e-mail
2. **Retorno à loja**: Retornar para concluir o pagamento usando outros métodos de pagamento do POS

O link "Pagar por este pedido" leva os clientes a uma página de pagamento dedicada no seu site, onde podem usar qualquer um dos gateways de pagamento web habilitados (Stripe, PayPal etc.).

## Gerenciamento de e-mails {#email-management}

### Clientes existentes {#existing-customers}

- Os endereços de e-mail são recuperados automaticamente dos perfis dos clientes
- As informações de cobrança do cliente são preservadas
- O histórico de pedidos é mantido na conta do cliente

### Clientes convidados {#guest-customers}

- Insira manualmente endereços de e-mail para clientes ocasionais
- Opção para salvar o e-mail no endereço de cobrança para referência futura
- Os pedidos de convidados são rastreados corretamente no WooCommerce

### Modelos de e-mail {#email-templates}

O gateway usa os modelos de e-mail padrão do WooCommerce:

- **Personalizável**: modifique a aparência dos e-mails nas configurações de e-mail do WooCommerce
- **Com marca**: inclua o logotipo e a identidade visual da sua loja
- **Profissional**: formato de fatura limpo e profissional
- **Compatível com dispositivos móveis**: design responsivo para todos os dispositivos

## Casos de uso {#use-cases}

### Ideal para {#perfect-for}

- **Vendas B2B**: Clientes empresariais que precisam processar pagamentos por meio de seus sistemas contábeis
- **Pedidos grandes**: Transações de alto valor que exigem aprovação ou tempo de processamento
- **Clientes remotos**: Clientes que precisam concluir o pagamento depois de sair da loja
- **Clientes com conta**: Clientes recorrentes com condições de pagamento estabelecidas
- **De orçamento a pedido**: Conversão de orçamentos ou estimativas em faturas pagáveis

## Solução de problemas {#troubleshooting}

### Problemas comuns {#common-issues}

<Accordion>
  <AccordionItem question="E-mails não estão sendo enviados">

  - Verifique a configuração de e-mail do WordPress
  - Verifique as configurações de SMTP se estiver usando um serviço de e-mail personalizado
  - Teste com plugins de teste de e-mail do WordPress
  - Verifique as pastas de spam para e-mails de teste

  </AccordionItem>

  <AccordionItem question="Links de pagamento não estão funcionando">

  - Certifique-se de que o WooCommerce esteja configurado corretamente
  - Verifique se pelo menos um gateway de pagamento está ativado para a finalização de compra na web
  - Verifique se o certificado SSL está instalado corretamente
  - Confirme se o status do pedido permite pagamento

  </AccordionItem>

  <AccordionItem question="O cliente não consegue concluir o pagamento">

  - Verifique se os gateways de pagamento na web estão ativos e configurados
  - Verifique se o pedido não expirou ou foi cancelado
  - Certifique-se de que o cliente tenha limites suficientes no método de pagamento
  - Teste o processo de pagamento você mesmo

  </AccordionItem>
</Accordion>

### Entrega de e-mails {#email-delivery}

Para uma entrega confiável de e-mails, considere:

- **Serviço SMTP**: Use serviços como SendGrid, Mailgun ou Amazon SES
- **Plugins de e-mail**: Instale plugins SMTP do WordPress para melhorar a entrega
- **Autenticação de domínio**: Configure registros SPF, DKIM e DMARC
- **Monitoramento**: acompanhe a entrega de emails e as taxas de abertura
