# Gateway Vipps MobilePay {#vipps-mobilepay-gateway}

O gateway Vipps MobilePay permite que você aceite pagamentos via telefone diretamente no WCPOS. Os clientes pagam escaneando um QR code na tela ou confirmando uma notificação push em seu telefone — não é necessário hardware de leitor de cartão.

## Recursos {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="Pagamentos por QR Code">
    Exiba um QR code na finalização da compra para que os clientes possam escanear e pagar com seu telefone
  </FeatureCard>
  <FeatureCard icon="bolt" title="Notificações Push">
    Envie uma solicitação de pagamento diretamente para o telefone do cliente inserindo seu número
  </FeatureCard>
  <FeatureCard icon="globe" title="Cobertura Nórdica">
    Funciona com Vipps na Noruega, MobilePay na Dinamarca e Finlandia
  </FeatureCard>
  <FeatureCard icon="credit-card" title="Captura Automática">
    Capture os fundos imediatamente após a autorização ou reserve para captura manual
  </FeatureCard>
</FeatureGrid>

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Visite a [página de lançamentos do WCPOS Vipps MobilePay](https://github.com/wcpos/wcpos-vipps/releases)
2. Baixe o arquivo mais recente **wcpos-vipps.zip**
3. No seu admin do WordPress, vá para `Plugins > Adicionar Novo > Fazer Upload do Plugin`
4. Faça o upload do arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Configurar Credenciais Vipps">

1. Navegue para `WP Admin > WooCommerce > Configurações > Pagamentos`
2. Encontre **WCPOS Vipps MobilePay** na lista de métodos de pagamento
3. Clique em **WCPOS Vipps MobilePay** para acessar as configurações
4. Insira suas credenciais do [Portal Vipps](https://portal.vipps.no/):
   - **Número de Série do Comerciante (MSN)**
   - **ID do Cliente**
   - **Segredo do Cliente**
   - **Chave de Assinatura**
5. Salve as configurações

:::note
Você não precisa habilitar o gateway aqui para uso no POS — ele será habilitado especificamente para o POS na próxima etapa. Habilitá-lo nas configurações do WooCommerce também o tornaria disponível na finalização da compra da sua loja online, o que pode ser útil para testes.
:::

:::tip Já está usando o plugin Vipps?
Se você tem o plugin oficial [Checkout com Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/) instalado, suas credenciais serão importadas automaticamente quando você ativar este plugin. Você pode pular a entrada manual das credenciais.
:::

  </StepCard>
  <StepCard number="3" title="Habilitar no WCPOS" isLast>

1. Vá para `WP Admin > POS > Configurações > Finalização da Compra`
2. Encontre o gateway **WCPOS Vipps MobilePay** na lista
3. Habilite o gateway para uso no POS
4. Salve suas configurações

  </StepCard>
</Steps>

## Uso {#usage}

### Processando Pagamentos — QR Code {#processing-payments-qr-code}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Vipps MobilePay" como o método de pagamento
3. **Gerar QR Code**: Clique no botão "Gerar QR Code"
4. **Cliente Escaneia**: O cliente escaneia o QR code com seu aplicativo Vipps ou MobilePay
5. **Cliente Confirma**: O cliente confirma o pagamento em seu aplicativo
6. **Conclusão Automática**: O pedido é concluído automaticamente assim que o pagamento é autorizado

### Processando Pagamentos — Enviar para o Telefone {#processing-payments-send-to-phone}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Vipps MobilePay" como o método de pagamento
3. **Inserir Número de Telefone**: Digite o número de telefone do cliente
4. **Enviar para o Telefone**: Clique no botão "Enviar para o Telefone"
5. **Cliente Confirma**: O cliente recebe uma notificação push ou é direcionado para a página de destino do Vipps, então confirma o pagamento em seu aplicativo
6. **Conclusão Automática**: O pedido é concluído automaticamente assim que o pagamento é autorizado

O plugin detecta automaticamente o melhor método para enviar a solicitação de pagamento para o telefone do cliente:

- **Push direto** (preferido) — Envia uma notificação diretamente para o aplicativo Vipps do cliente. Esta é a experiência mais rápida, mas requer que o Vipps habilite `PUSH_MESSAGE` na sua unidade de venda (veja [Habilitando Push Direto](#enabling-direct-push) abaixo).
- **Fallback da página de destino** — Se o push direto não estiver habilitado, o plugin abre a página de destino do Vipps em uma nova aba do navegador. A página de destino lida com o envio da notificação. Isso funciona imediatamente sem aprovação especial.

Na primeira vez que o plugin detectar que o push direto não está disponível, você verá uma mensagem breve pedindo que clique em "Enviar para o Telefone" novamente. Após isso, ele lembrará o resultado e funcionará sem problemas.

### Habilitando Push Direto {#enabling-direct-push}

O fluxo de push direto oferece a melhor experiência para pagamentos via telefone — sem abas extras, sem páginas de destino. Para habilitá-lo:

1. Faça login em [portal.vippsmobilepay.com](https://portal.vippsmobilepay.com)
2. Contate seu gerente de conta principal, gerente de parceira ou serviço ao cliente do Vipps
3. Diga a eles: **"Preciso que o PUSH_MESSAGE seja habilitado no meu MSN para uso com uma integração POS"**

Assim que aprovado, o plugin detecta a alteração automaticamente dentro de 24 horas e muda para o fluxo de push direto. Você verá um aviso de lembrete na página de configurações do gateway enquanto estiver usando o fallback da página de destino.

### Cancelando um Pagamento {#cancelling-a-payment}

Enquanto aguarda a confirmação do cliente, você pode clicar no botão **Cancelar Pagamento** para abortar a transação. Isso cancela o pagamento pendente no lado do Vipps e redefine a interface de finalização da compra.

### Reembolsos {#refunds}

Os reembolsos são tratados através do processo padrão de reembolso do WooCommerce. Abra o pedido, clique em **Reembolso**, insira o valor e o reembolso é processado através da API do Vipps automaticamente.

## Mercados Suportados {#supported-markets}

O Vipps MobilePay opera na região Nórdica sob duas marcas:

| Região | Marca | Moedas |
|--------|-------|--------|
| Noruega | Vipps | NOK |
| Dinamarca, Finlândia | MobilePay | DKK, EUR |

Sua conta de comerciante determina quais mercados e moedas estão disponíveis. Clientes em qualquer mercado suportado podem pagar usando seu aplicativo local Vipps ou MobilePay.

## Resolução de Problemas {#troubleshooting}

### Problemas Comuns {#common-issues}

<Accordion>
  <AccordionItem question="Pagamento não completando">

  - A finalização da compra aguarda por até 5 minutos — se o cliente não confirmar dentro desse período, o pagamento expira
  - Verifique se você não está misturando modo de teste e credenciais de produção
  - Verifique se sua conexão à internet é estável tanto no dispositivo POS quanto no telefone do cliente

  </AccordionItem>

  <AccordionItem question="Erro de credenciais inválidas">

  - Verifique novamente todos os quatro campos de credenciais (MSN, ID do Cliente, Segredo do Cliente, Chave de Assinatura)
  - Certifique-se de que você está usando credenciais de teste quando o modo de teste estiver habilitado e credenciais de produção quando desativado
  - Verifique suas credenciais no [Portal Vipps](https://portal.vipps.no/)

  </AccordionItem>

  <AccordionItem question="'Enviar para o Telefone' abre uma nova aba em vez de enviar diretamente">

  Isso significa que sua conta Vipps ainda não tem `PUSH_MESSAGE` habilitado. O plugin está utilizando o fallback da página de destino, que funciona, mas adiciona um passo extra. Para obter a experiência de push direto mais suave, entre em contato com o Vipps e peça para habilitar o PUSH_MESSAGE na sua unidade de vendas (MSN). Veja [Habilitando Push Direto](#enabling-direct-push) acima.

  </AccordionItem>

  <AccordionItem question="QR code não gerando">

  - Confirme que seu site está operando sobre HTTPS — a API do Vipps rejeita solicitações de sites HTTP
  - Verifique os logs do WooCommerce (`WooCommerce > Status > Logs`) para detalhes de erro da API
  - Verifique se sua conta de comerciante está ativa e tem a API de ePayment habilitada

  </AccordionItem>
</Accordion>
