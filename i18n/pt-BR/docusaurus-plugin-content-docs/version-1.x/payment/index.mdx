O WCPOS oferece suporte a vários métodos de pagamento para atender a diferentes necessidades comerciais. O checkout usa um iframe/webview para carregar a página de pagamento do pedido do WooCommerce, o que significa que qualquer gateway de pagamento que funcione com WooCommerce pode funcionar no POS.

## Métodos de pagamento padrão {#default-payment-methods}

A versão gratuita inclui dois gateways de pagamento integrados:

### Dinheiro {#cash}

- Informe o valor recebido
- Cálculo automático do troco
- Nenhuma configuração adicional necessária

### Cartão {#card}

- Para uso com terminais de cartão externos
- Basta marcar o pagamento como concluído após processá-lo no seu terminal
- Nenhuma integração direta necessária

## Gateways de pagamento adicionais (Pro) {#additional-payment-gateways-pro}

:::info Recurso Pro
Gateways de pagamento adicionais exigem [WCPOS Pro](/getting-started/pro-license).
:::

Com o Pro, é possível ativar gateways de pagamento WooCommerce adicionais no checkout do POS:

- **Stripe Terminal** - Integração direta com leitores de cartão Stripe
- **SumUp Terminal** - Integração com leitores de cartão SumUp
- **Qualquer gateway WooCommerce** - Ative qualquer gateway instalado na sua loja

## Ativação e desativação de gateways {#enabling-and-disabling-gateways}

Os gateways de pagamento são gerenciados na administração do WordPress:

1. Acesse `WP Admin > POS > Configurações > Finalização de compra`
2. Será exibida uma lista de todos os métodos de pagamento WooCommerce instalados
3. Ative ou desative cada método de pagamento para o POS independentemente da sua loja online
4. Arraste para reordenar — o primeiro método de pagamento ativado se torna o padrão na finalização da compra

Isso significa que um método de pagamento pode estar ativo no seu site, mas desativado no POS, ou vice-versa. Consulte [Configurações de finalização de compra](/settings/wp-admin/checkout) para obter todos os detalhes.

## Teste de pagamentos {#testing-payments}

Antes de colocar um novo método de pagamento em produção:

1. **Método de pagamento em dinheiro** — sempre disponível para testar o fluxo de finalização de compra sem transações reais.
2. **Stripe Terminal** — a Stripe oferece um [leitor simulado](https://docs.stripe.com/terminal/testing) para o modo de teste. Primeiro, ative o modo de teste no painel da Stripe.
3. **Métodos de pagamento de terceiros** — consulte a documentação do próprio método de pagamento para obter instruções sobre sandbox/modo de teste. A maioria dos métodos de pagamento WooCommerce oferece suporte a uma opção de modo de teste.

:::tip
Se um método de pagamento recém-ativado não aparecer na finalização da compra, tente atualizar o POS. Alterações de métodos de pagamento no WP Admin entram em vigor no próximo carregamento da página.
:::

## Problemas comuns {#common-issues}

### Gateway não aparece no checkout {#gateway-not-appearing-at-checkout}

- Confirme se o gateway está ativado para o POS em `WP Admin > POS > Configurações > Finalização de compra`.
- Verifique se o gateway também está ativado em `WP Admin > WooCommerce > Configurações > Pagamentos` — um gateway precisa estar ativo no WooCommerce antes de aparecer no POS.
- Atualize o POS depois de fazer alterações.

### Gateway exibido incorretamente {#gateway-displays-incorrectly}

O iframe de checkout carrega os estilos do tema do site, o que às vezes pode causar interferências. Use o botão **Configurações de checkout** no modal de checkout para desativar estilos ou scripts seletivamente. Consulte [Solução de problemas do checkout](/pos/checkout#checkout-settings-troubleshooting) para obter detalhes.

### Falha no processamento do pagamento {#payment-processing-fails}

- Verifique os registros de erro do site (`WP Admin > POS > Suporte > Logs`) para encontrar erros específicos do gateway.
- Certifique-se de que o certificado SSL seja válido — a maioria dos gateways de pagamento exige HTTPS.
- Ao usar um ambiente de staging ou local, confirme se o gateway oferece suporte ao modo de teste/sandbox.
