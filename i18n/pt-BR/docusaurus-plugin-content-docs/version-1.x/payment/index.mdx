O WCPOS oferece suporte a vários métodos de pagamento para atender a diferentes necessidades comerciais. O checkout usa um iframe/webview para carregar a página WooCommerce Order Pay, o que significa que qualquer gateway de pagamento que funcione com WooCommerce pode funcionar no POS.

## Métodos de Pagamento Padrão

A versão gratuita inclui dois gateways de pagamento integrados:

### Dinheiro

- Insira o valor pago
- Cálculo automático de troco
- Nenhuma configuração adicional necessária

### Cartão

- Para uso com terminais de cartão externas
- Simplesmente marque o pagamento como completo após processar no seu terminal
- Nenhuma integração direta necessária

## Gateways de Pagamento Adicionais (Pro)

:::info Pro Feature
Gateways de pagamento adicionais requerem [WCPOS Pro](/getting-started/pro-license).
:::

Com o Pro, você pode ativar gateways de pagamento adicionais do WooCommerce no checkout do POS:

- **Stripe Terminal** - Integração direta com leitores de cartão Stripe
- **SumUp Terminal** - Integração com leitores de cartão SumUp
- **Any WooCommerce Gateway** - Ative qualquer gateway instalado na sua loja

## Ativando e Desativando Gateways

Os gateways de pagamento são gerenciados no painel administrativo do WordPress:

1. Vá para `WP Admin > POS > Settings > Checkout`
2. Você verá uma lista de todos os gateways de pagamento WooCommerce instalados
3. Ative ou desative cada gateway para o POS independentemente de sua loja online
4. Arraste para reordenar — o primeiro gateway ativado se torna o padrão no checkout

Isso significa que um gateway pode estar ativo em seu site, mas desativado no POS, ou vice-versa. Consulte [Configurações de Checkout](/settings/wp-admin/checkout) para obter detalhes completos.

## Testando Pagamentos

Antes de ativar um novo gateway:

1. **Cash gateway** — sempre disponível para testar o fluxo de checkout sem transações reais.
2. **Stripe Terminal** — Stripe fornece um [leitor simulado](https://docs.stripe.com/terminal/testing) para o modo de teste. Ative o modo de teste no seu painel do Stripe primeiro.
3. **Gateways de terceiros** — verifique a documentação própria do gateway para instruções de modo sandbox/teste. A maioria dos gateways WooCommerce suporta uma alternância de modo de teste.

:::tip
Se um gateway recém-ativado não aparecer no checkout, tente atualizar o POS. As mudanças de gateway no WP Admin entram em vigor no próximo carregamento de página.
:::

## Problemas Comuns

### Gateway Não Aparecendo no Checkout

- Confirme se o gateway está ativado para POS em `WP Admin > POS > Settings > Checkout`.
- Verifique se o gateway também está ativado em `WP Admin > WooCommerce > Settings > Payments` — um gateway deve estar ativo no WooCommerce antes de aparecer no POS.
- Atualize o POS após fazer alterações.

### Gateway Exibido Incorretamente

O iframe de checkout carrega os estilos do tema do seu site, o que às vezes pode interferir. Use o botão **Checkout Settings** na modal de checkout para desabilitar seletivamente estilos ou scripts. Veja [Solução de Problemas de Checkout](/pos/checkout#checkout-settings-troubleshooting) para obter mais detalhes.

### Falha no Processamento de Pagamento

- Verifique os logs de erro do seu site (`WP Admin > POS > Support > Logs`) para erros específicos do gateway.
- Certifique-se de que seu certificado SSL é válido — a maioria dos gateways de pagamento requer HTTPS.
- Se estiver usando um ambiente de staging ou local, confirme que o gateway suporta modo de teste/sandbox.
