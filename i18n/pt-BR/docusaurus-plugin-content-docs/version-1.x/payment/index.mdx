O WCPOS suporta múltiplos métodos de pagamento para atender diferentes necessidades comerciais. O checkout usa um iframe/webview para carregar a página WooCommerce Order Pay, o que significa que qualquer gateway de pagamento que funciona com WooCommerce pode funcionar no POS.

## Métodos de Pagamento Padrão

A versão gratuita inclui dois gateways de pagamento integrados:

### Dinheiro

- Insira o valor recebido
- Cálculo automático de troco
- Nenhuma configuração adicional necessária

### Cartão

- Para uso com terminais de cartão externos
- Simplesmente marque o pagamento como concluído após o processamento no seu terminal
- Nenhuma integração direta necessária

## Gateways de Pagamento Adicionais (Pro)

:::info Pro Feature
Gateways de pagamento adicionais requerem [WCPOS Pro](/getting-started/pro-license).
:::

Com o Pro, você pode ativar gateways de pagamento WooCommerce adicionais no checkout do POS:

- **Stripe Terminal** - Integração direta com leitoras de cartão Stripe
- **SumUp Terminal** - Integração com leitoras de cartão SumUp
- **Any WooCommerce Gateway** - Ative qualquer gateway instalado na sua loja

## Ativando e Desativando Gateways

Os gateways de pagamento são gerenciados no admin do WordPress:

1. Acesse `WP Admin > POS > Settings > Checkout`
2. Você verá uma lista de todos os gateways de pagamento WooCommerce instalados
3. Ative ou desative cada gateway para o POS independentemente de sua loja online
4. Arraste para reordenar — o primeiro gateway ativado se torna o padrão no checkout

Isso significa que um gateway pode estar ativo em seu site, mas desativado no POS, ou vice-versa. Veja [Checkout Settings](/settings/wp-admin/checkout) para detalhes completos.

## Testando Pagamentos

Antes de usar um novo gateway em produção:

1. **Cash gateway** — sempre disponível para testar o fluxo de checkout sem transações reais.
2. **Stripe Terminal** — Stripe fornece um [simulated reader](https://docs.stripe.com/terminal/testing) para modo de teste. Ative o modo de teste em seu painel do Stripe primeiro.
3. **Third-party gateways** — verifique a documentação do próprio gateway para instruções de modo sandbox/teste. A maioria dos gateways WooCommerce suporta uma alternância de modo de teste.

:::tip
Se um gateway recém-ativado não aparecer no checkout, tente atualizar o POS. As alterações de gateway no WP Admin entram em vigor no próximo carregamento de página.
:::

## Problemas Comuns

### Gateway Não Aparecendo no Checkout

- Confirme que o gateway está ativado para POS em `WP Admin > POS > Settings > Checkout`.
- Verifique se o gateway também está ativado em `WP Admin > WooCommerce > Settings > Payments` — um gateway deve estar ativo no WooCommerce antes de aparecer no POS.
- Atualize o POS após fazer alterações.

### Gateway Exibido Incorretamente

O iframe de checkout carrega os estilos do tema do seu site, que às vezes podem interferir. Use o botão **Checkout Settings** na modal de checkout para desabilitar seletivamente estilos ou scripts. Veja [Solução de Problemas de Checkout](/pos/checkout#checkout-settings-troubleshooting) para detalhes.

### Falha no Processamento de Pagamento

- Verifique os logs de erro do seu site (`WP Admin > POS > Support > Logs`) para erros específicos do gateway.
- Certifique-se de que seu certificado SSL é válido — a maioria dos gateways de pagamento exige HTTPS.
- Se estiver usando um ambiente de staging ou local, confirme que o gateway suporta modo de teste/sandbox.
