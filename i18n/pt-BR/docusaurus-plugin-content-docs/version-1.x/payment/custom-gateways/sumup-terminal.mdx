# Gateway do Terminal SumUp

O gateway do Terminal SumUp permite que você aceite pagamentos com cartão usando leitores de cartão SumUp diretamente dentro do WooCommerce POS. Este gateway proporciona integração perfeita com o sistema de processamento de pagamentos da SumUp para transações presenciais.

## Recursos

- **Integração de Hardware**: Conecte os leitores de cartão SumUp ao seu sistema POS via conexão de internet
- **Processamento em Tempo Real**: Processamento de pagamentos instantâneo e conclusão de pedidos
- **Detecção Automática**: Os pedidos são concluídos automaticamente quando o pagamento é detectado
- **Transações Seguras**: Processamento de pagamentos em conformidade com PCI através da SumUp
- **Emparelhamento Fácil**: Processo simples de emparelhamento de dispositivos com códigos de emparelhamento
- **SDK Baseado na Web**: Usa o SDK JavaScript da SumUp para compatibilidade com aplicativos web

## Instalação

### Passo 1: Baixar e Instalar

1. Visite a [página de lançamentos do SumUp Terminal para WooCommerce](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)
2. Baixe o arquivo mais recente **sumup-terminal-for-woocommerce.zip**
3. Em seu painel do WordPress, vá para `Plugins > Adicionar Novo > Enviar Plugin`
4. Faça o upload do arquivo zip e ative o plugin

### Passo 2: Configurar as Configurações da SumUp

1. Navegue até `WP Admin > WooCommerce > Configurações > Pagamentos`
2. Encontre **Terminal SumUp** na lista de métodos de pagamento
3. Clique em **Terminal SumUp** para acessar as configurações
4. Insira sua **Chave API da SumUp** (disponível no seu painel de comerciante da SumUp)
5. Salve as configurações

:::note
Você não precisa habilitar o gateway do Terminal SumUp nas configurações do WooCommerce. Ele será habilitado especificamente para o POS em um passo posterior.
:::

### Passo 3: Emparelhar Seu Terminal SumUp

1. Na mesma página de configurações, localize a seção **Emparelhar Leitor**
2. Em seu dispositivo SumUp, navegue até a tela de emparelhamento para exibir o código de emparelhamento
3. Insira o código de emparelhamento exibido em seu dispositivo SumUp
4. Clique em **"Emparelhar Leitor"** para estabelecer a conexão
5. Aguarde a confirmação de que o leitor foi emparelhado com sucesso

:::warning Importante
O leitor deve ser emparelhado com sucesso antes que você possa processar pagamentos. Certifique-se de que o processo de emparelhamento esteja concluído antes de prosseguir.
:::

### Passo 4: Habilitar no WooCommerce POS

1. Vá para `WP Admin > WooCommerce POS > Configurações > Checkout`
2. Encontre o gateway **Terminal SumUp** na lista
3. Habilite o gateway para uso no POS
4. Salve suas configurações

## Uso

### Processamento de Pagamentos

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Terminal SumUp" como método de pagamento
3. **Iniciar Pagamento**: Clique para iniciar um novo pagamento em seu dispositivo SumUp
4. **Pagamento do Cliente**: O cliente conclui o pagamento no terminal SumUp
5. **Conclusão Automática**: O pedido é concluído automaticamente quando o pagamento é detectado

### Controles de Pagamento

Ao usar o gateway do Terminal SumUp, você tem as seguintes opções:

- **Iniciar Novo Pagamento**: Inicie uma solicitação de pagamento no terminal conectado
- **Cancelar Pagamento**: Cancele um pagamento que está atualmente em processamento
- **Status do Pagamento**: Monitore o status atual do processo de pagamento

### Gerenciamento de Pedidos

- **Atualizações Automáticas**: Os pedidos são automaticamente marcados como concluídos após o pagamento bem-sucedido
- **Rastreamento de Pagamento**: Todos os detalhes do pagamento são registrados nas notas do pedido
- **Geração de Recibos**: Recibos padrão do POS são gerados após pagamentos bem-sucedidos

## Requisitos

- **Conta SumUp**: Conta de comerciante SumUp ativa
- **Acesso à API**: Chave da API SumUp do seu painel de comerciante
- **Hardware Compatível**: Dispositivo leitor de cartão SumUp
- **WooCommerce POS**: Versão gratuita ou Pro instalada e ativada
- **Conexão Estável**: Conexão de internet confiável para comunicação com a API

## Compatibilidade de Hardware

:::important Requisitos de Conectividade
Esta implementação utiliza o SDK JavaScript da SumUp, o que significa que funciona através de aplicativos web, mas requer **terminais apenas conectados à internet**. Terminais Bluetooth não são suportados atualmente.
:::

### Terminais Suportados (Conectados à Internet)

- **SumUp Solo** ✅ - Terminal conectado via WiFi/Ethernet

### Terminais Não Suportados (Bluetooth)

- **SumUp Air** ❌ - Apenas Bluetooth
- **SumUp 3G** ❌ - Apenas Bluetooth

:::note Suporte Futuro
O suporte para terminais Bluetooth será adicionado aos aplicativos iOS e Android no **1º trimestre de 2026**. Isso permitirá o suporte para todos os terminais certificados pela SumUp, incluindo os modelos Air e 3G.
:::

## Solução de Problemas

### Problemas Comuns

**Leitor Não Emparelha**
- Certifique-se de que você está usando um terminal conectado à internet suportado (apenas Solo)
- Verifique se o seu dispositivo SumUp Solo está conectado ao WiFi e online
- Confira se o dispositivo está no modo de emparelhamento e se o código de emparelhamento foi inserido corretamente
- Confirme que sua conta SumUp tem o acesso à API habilitado
- Certifique-se de que a conexão de internet está estável durante o emparelhamento

**Pagamento Não Processando**
- Verifique se o leitor está devidamente emparelhado
- Confira se sua conta SumUp está ativa e em situação regular
- Certifique-se de que o dispositivo tem bateria suficiente e conectividade
- Confirme que a chave da API está correta e ativa

**Pedido Não Conclui**
- Verifique a estabilidade da conexão com a internet
- Confira as configurações do webhook na sua conta SumUp
- Certifique-se de que o pagamento foi bem-sucedido no terminal SumUp
- Verifique os logs de erro do WordPress para problemas de comunicação com a API

### Limites de Taxa da API

O gateway é otimizado para reduzir chamadas à API e evitar limitação de taxa:
- O status do pagamento é verificado de maneira eficiente
- Chamadas de API desnecessárias são minimizadas
- A lógica de tentativa automática lida com falhas temporárias
