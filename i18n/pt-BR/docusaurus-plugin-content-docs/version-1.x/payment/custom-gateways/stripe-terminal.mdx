# Gateway do Terminal Stripe {#stripe-terminal-gateway}

O gateway do Stripe Terminal permite que você aceite pagamentos presenciais usando leitores de hardware Stripe Terminal diretamente dentro do WCPOS. Este gateway suporta tanto leitores de cartão físicos quanto o modo simulador para testes.

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Visite a [página de lançamentos do Stripe Terminal para WooCommerce](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases)
2. Baixe o arquivo mais recente **stripe-terminal-for-woocommerce.zip**
3. No seu WordPress admin, vá para `Plugins > Adicionar Novo > Fazer Upload do Plugin`
4. Faça o upload do arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Configurar as Configurações do Stripe">

1. Navegue até `WP Admin > WooCommerce > Configurações > Pagamentos`
2. Encontre **Stripe Terminal** na lista de métodos de pagamento
3. Clique em **Stripe Terminal** para acessar as configurações
4. Insira sua **Chave Secreta do Stripe** (você pode obtê-la no seu Painel do Stripe)
5. Salve as configurações

:::note
Você não precisa ativar o gateway do Stripe Terminal nas configurações do WooCommerce. Ele será ativado especificamente para o POS na próxima etapa.
:::

  </StepCard>
  <StepCard number="3" title="Ativar no WCPOS" isLast>

1. Vá para `WP Admin > POS > Configurações > Checkout`
2. Encontre o gateway **Stripe Terminal** na lista
3. Ative o gateway para uso no POS
4. Salve suas configurações

  </StepCard>
</Steps>

## Uso {#usage}

### Conectando um Leitor {#connecting-a-reader}

Quando você seleciona o gateway do Stripe Terminal durante o checkout no POS:

1. **Escolha o Método de Conexão**: Você pode conectar um leitor físico ou usar o simulador
2. **Leitor Físico**: Siga as instruções na tela para conectar seu dispositivo Stripe Terminal
3. **Simulador**: Selecione o modo simulador para testar vários cenários de pagamento sem hardware

### Processando Pagamentos {#processing-payments}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Stripe Terminal" como o método de pagamento
3. **Conectar Leitor**: Conecte seu leitor ou escolha o modo simulador
4. **Processar Pagamento**: Siga as instruções para concluir a transação
5. **Confirmação**: O pedido será automaticamente concluído após um pagamento bem-sucedido

### Testando com Simulador {#testing-with-simulator}

O simulador permite que você teste vários métodos e cenários de pagamento:

- **Pagamentos com Cartão**: Teste diferentes tipos de cartão (Visa, Mastercard, etc.)
- **Pagamentos Sem Contato**: Simule transações de pagamento por aproximação
- **Cenários de Erro**: Teste pagamentos recusados e outras condições de erro
- **Valores Diferentes**: Teste vários valores de transação

### Pedidos por Telefone (MOTO) {#phone-orders-moto}

MOTO (Mail Order/Telephone Order) permite que você processe pagamentos com cartão para clientes que não estão fisicamente presentes — por exemplo, ao fazer um pedido por telefone. Em vez de passar ou inserir um cartão, o comerciante insere os detalhes do cartão diretamente na tela do leitor de terminal.

#### Configuração {#setup}

<Steps>
  <StepCard number="1" title="Solicitar acesso MOTO do Stripe">

MOTO não está habilitado por padrão. Entre em contato com o [suporte do Stripe](https://support.stripe.com/contact) e peça para habilitar as permissões MOTO para sua conta. Este é um processo rápido, mas requer aprovação manual do Stripe.

  </StepCard>
  <StepCard number="2" title="Habilitar nas configurações do plugin">

1. Navegue até `WP Admin > WooCommerce > Configurações > Pagamentos > Stripe Terminal`
2. Marque a caixa **Pedidos por Telefone (MOTO)**
3. Salve as configurações

  </StepCard>
  <StepCard number="3" title="Conectar um leitor compatível" isLast>

MOTO só funciona com leitores compatíveis conectados à internet listados em [Terminais Suportados](#supported-terminals-internet-connected). O botão de alternância não aparecerá para outros tipos de leitores.

  </StepCard>
</Steps>

#### Fazendo um Pedido por Telefone {#taking-a-phone-order}

1. Conecte um leitor compatível (consulte [Terminais Suportados](#supported-terminals-internet-connected))
2. Na tela de pagamento, ative **Pedido por Telefone**
3. Clique em **Coletar Pagamento com Cartão** — o leitor exibirá uma tela de entrada do número do cartão em vez de solicitar um toque/inserção
4. Digite o número do cartão do cliente, validade e CVV no leitor
5. O pagamento será processado normalmente a partir daí

:::tip
Pagamentos MOTO usam `card` como o tipo de método de pagamento em vez de `card_present`. Isso significa que são tratados mais como transações online do ponto de vista do Stripe, então as taxas de processamento de cartão online padrão se aplicam em vez das taxas de presença física.
:::

:::caution
O botão de alternância de Pedido por Telefone só aparece quando todas as três condições são atendidas: a configuração MOTO está habilitada nas configurações do plugin, um leitor compatível está conectado e o leitor não é um simulador. Se você não vê o botão, verifique essas condições.
:::

## Compatibilidade de Hardware {#hardware-compatibility}

:::important Requisitos de Conectividade
Esta implementação usa o SDK JavaScript do Stripe, o que significa que funciona através de aplicações web, mas requer **terminais conectados à internet apenas**. Terminais Bluetooth não são suportados atualmente.
:::

### Terminais Suportados (Conectados à Internet) {#supported-terminals-internet-connected}

- **Leitor Stripe S700/S710** ✅ - terminal conectado por Ethernet/WiFi
- **WisePOS E** ✅ - terminal conectado por WiFi

### Terminais Não Suportados (Bluetooth) {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - apenas Bluetooth
- **BBPOS WisePad 3** ❌ - apenas Bluetooth  
- **Verifone P400** ❌ - descontinuado, não mais suportado pelo Stripe

:::note Suporte Futuro
O suporte para terminais Bluetooth será adicionado aos aplicativos iOS e Android em uma **futura atualização**. Isso permitirá o suporte a todos os leitores certificados do Stripe Terminal, incluindo o M2 e WisePad 3.
:::

### Problemas Comuns {#common-issues}

<Accordion>
  <AccordionItem question="Leitor não conecta">

  - Verifique se você está usando um [terminal conectado à internet suportado](#supported-terminals-internet-connected)
  - Verifique se o terminal está conectado ao WiFi/Ethernet e online
  - Certifique-se de que sua conta Stripe tenha o Terminal habilitado
  - Confirme se o terminal está registrado no seu Painel do Stripe

  </AccordionItem>

  <AccordionItem question="Pagamento recusado">

  - Verifique se sua conta Stripe está ativa e em boas condições
  - Certifique-se de que o cartão usado é válido
  - Verifique se há fundos suficientes disponíveis

  </AccordionItem>

  <AccordionItem question="Botão de Pedido por Telefone não aparece">

  - Verifique se a configuração **Pedidos por Telefone (MOTO)** está habilitada em `WooCommerce > Configurações > Pagamentos > Stripe Terminal`
  - Certifique-se de que você está conectado a um leitor compatível (veja [Terminais Suportados](#supported-terminals-internet-connected)) — o botão de alternância é oculto para outros tipos de leitores
  - O botão de alternância não aparece ao usar o simulador

  </AccordionItem>

  <AccordionItem question="Pagamento MOTO falha com um erro">

  - Confirme se o Stripe habilitou as permissões MOTO na sua conta — entre em contato com o [suporte do Stripe](https://support.stripe.com/contact) se ainda não o fez
  - Verifique se os detalhes do cartão foram inseridos corretamente no leitor
  - Pagamentos MOTO podem ter verificações de fraude mais rigorosas — certifique-se de que o cartão é válido e tem fundos suficientes

  </AccordionItem>

  <AccordionItem question="Erros de certificado SSL">

  - O Stripe Terminal requer HTTPS - verifique se seu site tem um certificado SSL válido
  - Verifique se seu certificado SSL está configurado corretamente

  </AccordionItem>
</Accordion>
