# Gateway Vipps MobilePay {#vipps-mobilepay-gateway}

O gateway Vipps MobilePay permite que você aceite pagamentos baseados em telefone diretamente no WCPOS. Os clientes pagam escaneando um QR code na tela ou confirmando uma notificação push em seu telefone — sem necessidade de hardware de leitor de cartão.

## Recursos {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="Pagamentos via QR Code">
    Exiba um QR code no checkout para que os clientes escaneiem e paguem com seu telefone
  </FeatureCard>
  <FeatureCard icon="bolt" title="Notificações Push">
    Envie um pedido de pagamento diretamente para o telefone do cliente inserindo seu número
  </FeatureCard>
  <FeatureCard icon="globe" title="Cobertura Nórdica">
    Funciona com Vipps na Noruega, MobilePay na Dinamarca e Finlândia
  </FeatureCard>
  <FeatureCard icon="credit-card" title="Captura Automática">
    Capture fundos imediatamente após a autorização, ou reserve para captura manual
  </FeatureCard>
</FeatureGrid>

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Visite a [página de lançamentos do WCPOS Vipps MobilePay](https://github.com/wcpos/wcpos-vipps/releases)
2. Baixe o arquivo mais recente **wcpos-vipps.zip**
3. No seu WordPress admin, vá para `Plugins > Adicionar Novo > Enviar Plugin`
4. Faça o upload do arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Configurar Credenciais Vipps">

1. Navegue até `WP Admin > WooCommerce > Configurações > Pagamentos`
2. Encontre **WCPOS Vipps MobilePay** na lista de métodos de pagamento
3. Clique em **WCPOS Vipps MobilePay** para acessar as configurações
4. Insira suas credenciais do [Portal Vipps](https://portal.vipps.no/):
   - **Número de Série do Comerciante (MSN)**
   - **ID do Cliente**
   - **Segredo do Cliente**
   - **Chave de Assinatura**
5. Salve as configurações

:::note
Você não precisa habilitar o gateway aqui para uso no POS — ele será habilitado especificamente para o POS na próxima etapa. Habilitá-lo nas configurações do WooCommerce também o tornará disponível no checkout da sua loja online, o que pode ser útil para testes.
:::

:::tip Já está usando o plugin Vipps?
Se você tiver o plugin oficial [Checkout com Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/) instalado, suas credenciais serão importadas automaticamente quando você ativar este plugin. Você pode pular a entrada manual de credenciais.
:::

  </StepCard>
  <StepCard number="3" title="Habilitar no WCPOS" isLast>

1. Vá para `WP Admin > WCPOS > Configurações > Checkout`
2. Encontre o gateway **WCPOS Vipps MobilePay** na lista
3. Habilite o gateway para uso no POS
4. Salve suas configurações

  </StepCard>
</Steps>

## Uso {#usage}

### Processamento de Pagamentos — QR Code {#processing-payments-qr-code}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Vipps MobilePay" como método de pagamento
3. **Gerar QR Code**: Clique no botão "Gerar QR Code"
4. **Cliente Escaneia**: O cliente escaneia o QR code com seu aplicativo Vipps ou MobilePay
5. **Cliente Confirma**: O cliente confirma o pagamento em seu aplicativo
6. **Conclusão Automática**: O pedido é concluído automaticamente assim que o pagamento é autorizado

### Processamento de Pagamentos — Notificação Push {#processing-payments-push-notification}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Vipps MobilePay" como método de pagamento
3. **Digite o Número do Telefone**: Digite o número de telefone do cliente
4. **Enviar para o Telefone**: Clique no botão "Enviar para o Telefone"
5. **Cliente Confirma**: O cliente recebe uma notificação push e confirma o pagamento em seu aplicativo
6. **Conclusão Automática**: O pedido é concluído automaticamente assim que o pagamento é autorizado

### Cancelando um Pagamento {#cancelling-a-payment}

Enquanto aguarda a confirmação do cliente, você pode clicar no botão **Cancelar Pagamento** para abortar a transação. Isso cancela o pagamento pendente no lado do Vipps e redefine a interface de checkout.

### Reembolsos {#refunds}

Os reembolsos são tratados através do processo padrão de reembolso do WooCommerce. Abra o pedido, clique em **Reembolsar**, insira o valor e o reembolso será processado automaticamente através da API do Vipps.

## Mercados Suportados {#supported-markets}

O Vipps MobilePay opera na região nórdica sob duas marcas:

| Região | Marca | Moedas |
|--------|-------|--------|
| Noruega | Vipps | NOK |
| Dinamarca, Finlândia | MobilePay | DKK, EUR |

Sua conta de comerciante determina quais mercados e moedas estão disponíveis. Clientes em qualquer mercado suportado podem pagar usando seu aplicativo local do Vipps ou MobilePay.

## Requisitos {#requirements}

<RequirementsList>
  <RequirementItem label="Conta Vipps">Conta de comerciante Vipps MobilePay ativa com credenciais de API</RequirementItem>
  <RequirementItem label="Credenciais de API">Número de Série do Comerciante, ID do Cliente, Segredo do Cliente e Chave de Assinatura</RequirementItem>
  <RequirementItem label="WCPOS">Versão Pro necessária para checkout no POS. O gateway também funciona no checkout web padrão do WooCommerce sem Pro.</RequirementItem>
  <RequirementItem label="HTTPS">Seu site deve usar SSL/HTTPS (exigido pela API do Vipps)</RequirementItem>
</RequirementsList>

## Solução de Problemas {#troubleshooting}

### Problemas Comuns {#common-issues}

<Accordion>
  <AccordionItem question="Pagamento não concluído">

  - O checkout faz polling por até 5 minutos — se o cliente não confirmar dentro desse intervalo, o pagamento expira
  - Verifique se você não está misturando credenciais de modo de teste e produção
  - Verifique se sua conexão com a internet está estável tanto no dispositivo POS quanto no telefone do cliente

  </AccordionItem>

  <AccordionItem question="Erro de credenciais inválidas">

  - Verifique novamente todos os quatro campos de credencial (MSN, ID do Cliente, Segredo do Cliente, Chave de Assinatura)
  - Certifique-se de que está usando credenciais de teste quando o modo de teste está habilitado e credenciais de produção quando está desativado
  - Verifique suas credenciais no [Portal Vipps](https://portal.vipps.no/)

  </AccordionItem>

  <AccordionItem question="QR code não gerando">

  - Confirme que seu site está rodando sob HTTPS — a API do Vipps rejeita requisições de sites HTTP
  - Verifique os logs do WooCommerce (`WooCommerce > Status > Logs`) para detalhes sobre erros da API
  - Verifique se sua conta de comerciante está ativa e tem a API de ePayment habilitada

  </AccordionItem>
</Accordion>
