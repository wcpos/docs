# Gateway de Fatura por Email

O gateway de Fatura por Email permite que você envie faturas de pagamento para clientes via email diretamente do WooCommerce POS. Isso é perfeito para situações em que os clientes preferem pagar mais tarde ou precisam concluir o pagamento online usando os métodos de pagamento da sua loja online.

## Recursos

- **Integração de Email**: Envie faturas diretamente da interface do POS
- **Gerenciamento de Clientes**: Preenchimento automático de email a partir dos dados de clientes existentes ou adição de novos endereços de email
- **Pedidos de Convidados**: Suporte para clientes convidados com atualizações opcionais de endereço de cobrança
- **Pagamento Online**: Os clientes recebem um link "Pagar por este pedido" para concluir o pagamento online
- **Fluxo de Trabalho Flexível**: Complete pedidos na loja ou permita a conclusão do pagamento remotamente

## Instalação

### Passo 1: Baixe e Instale

1. Visite a [página de lançamentos do Gateway de Fatura por Email](https://github.com/wcpos/email-invoice-gateway/releases)
2. Baixe o arquivo mais recente **woocommerce-pos-email-invoice-gateway.zip**
3. No seu administrador WordPress, vá para `Plugins > Adicionar Novo > Carregar Plugin`
4. Carregue o arquivo zip e ative o plugin

### Passo 2: Ativar no WooCommerce POS

1. Vá para `WP Admin > WooCommerce POS > Configurações > Checkout`
2. Encontre o **Gateway de Pagamento de Fatura do WooCommerce POS** na lista
3. Ative o gateway para uso no POS
4. Salve suas configurações

:::note
Este gateway não requer chaves de API adicionais ou configuração de serviços externos. Ele utiliza seu sistema de email WordPress existente e métodos de pagamento do WooCommerce.
:::

## Uso

### Enviando Faturas

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Fatura por Email" como método de pagamento
3. **Endereço de Email**: 
   - Para clientes existentes: O email é preenchido automaticamente a partir dos dados do cliente
   - Para pedidos de convidados: Insira o endereço de email do cliente manualmente
4. **Opcional**: Marque "Salvar para endereço de cobrança" para atualizar as informações de cobrança do cliente
5. **Enviar Fatura**: Complete o processo para enviar o email da fatura

### Experiência do Cliente

Quando você envia uma fatura, o cliente recebe um email contendo:

- **Detalhes do Pedido**: Detalhamento completo de itens, quantidades e preços
- **Total do Pedido**: Valor final devido, incluindo impostos e taxas
- **Link de Pagamento**: Link direto para concluir o pagamento online
- **Informações do Pedido**: Número do pedido e detalhes de referência

### Conclusão do Pagamento

Os clientes podem concluir o pagamento de duas maneiras:

1. **Pagamento Online**: Clique no link "Pagar por este pedido" no email
2. **Retorno à Loja**: Retorne para concluir o pagamento usando outros métodos de pagamento do POS

O link "Pagar por este pedido" leva os clientes a uma página de pagamento dedicada em seu site, onde podem usar qualquer um dos seus gateways de pagamento online ativados (Stripe, PayPal, etc.).

## Gerenciamento de Email

### Clientes Existentes

- Endereços de email são recuperados automaticamente dos perfis de cliente
- As informações de cobrança do cliente são preservadas
- O histórico de pedidos é mantido sob a conta do cliente

### Clientes Convidados

- Insira manualmente endereços de email para clientes únicos
- Opção de salvar email para o endereço de cobrança para referência futura
- Pedidos de convidados são devidamente rastreados no WooCommerce

### Modelos de Email

O gateway utiliza os modelos de email padrão do WooCommerce:

- **Personalizável**: Modifique a aparência do email por meio das configurações de email do WooCommerce
- **Personalizado**: Inclua o logotipo e a marca da sua loja
- **Profissional**: Formato de fatura limpo e profissional
- **Compatível com Dispositivos Móveis**: Design responsivo para todos os dispositivos

## Casos de Uso

### Perfeito Para

- **Vendas B2B**: Clientes empresariais que precisam processar pagamentos por meio de seus sistemas contábeis
- **Grandes Pedidos**: Transações de alto valor que exigem aprovação ou tempo de processamento
- **Clientes Remotos**: Clientes que precisam concluir o pagamento após sair da loja
- **Clientes com Conta**: Clientes regulares com termos de pagamento estabelecidos
- **Citação para Pedido**: Converter cotações ou estimativas em faturas pagáveis

### Exemplos de Fluxo de Trabalho

**Loja de Varejo**
1. O cliente seleciona itens, mas precisa obter aprovação para a compra
2. Enviar email de fatura com todos os detalhes
3. O cliente conclui o pagamento online quando estiver pronto
4. O pedido é automaticamente atualizado em seu sistema

**Negócio de Serviços**
1. Complete o trabalho de serviço e adicione itens ao POS
2. Envie a fatura para o departamento contábil do cliente
3. O cliente paga online usando o método de pagamento preferido
4. Receba a confirmação de pagamento e conclua o pedido

## Requisitos

- **WooCommerce POS**: Versão gratuita ou Pro instalada e ativada
- **Sistema de Email**: Configuração de email WordPress funcionando
- **Gateways de Pagamento Web**: Pelo menos um método de pagamento online ativado no WooCommerce
- **Certificado SSL**: HTTPS necessário para o processamento seguro de pagamentos

## Solução de Problemas

### Problemas Comuns

**Emails Não Enviados**
- Verifique a configuração de email do WordPress
- Verifique as configurações SMTP se estiver usando um serviço de email personalizado
- Teste com plugins de teste de email do WordPress
- Verifique pastas de spam para emails de teste

**Links de Pagamento Não Funcionando**
- Certifique-se de que o WooCommerce esteja corretamente configurado
- Verifique se pelo menos um gateway de pagamento está ativado para checkout web
- Verifique se o certificado SSL está instalado corretamente
- Confirme se o status do pedido permite pagamento

**Cliente Não Consegue Concluir o Pagamento**
- Verifique se os gateways de pagamento web estão ativos e configurados
- Verifique se o pedido não expirou ou foi cancelado
- Certifique-se de que o cliente tenha limites suficientes para o método de pagamento
- Teste o processo de pagamento você mesmo

### Entrega de Email

Para entrega confiável de email, considere:

- **Serviço SMTP**: Use serviços como SendGrid, Mailgun ou Amazon SES
- **Plugins de Email**: Instale plugins SMTP do WordPress para melhor entrega
- **Autenticação de Domínio**: Configure registros SPF, DKIM e DMARC
- **Monitoramento**: Acompanhe a entrega de emails e taxas de abertura
