# Gateway de Fatura por Email

O gateway de Fatura por Email permite enviar faturas de pagamento para clientes por email diretamente do WooCommerce POS. Isso é perfeito para situações em que os clientes preferem pagar depois ou precisam concluir o pagamento online usando os métodos de pagamento da sua loja virtual.

## Instalação

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Visite a [página de lançamentos do Gateway de Fatura por Email](https://github.com/wcpos/email-invoice-gateway/releases)
2. Baixe o arquivo mais recente **woocommerce-pos-email-invoice-gateway.zip**
3. No seu admin do WordPress, vá para `Plugins > Adicionar Novo > Fazer Upload do Plugin`
4. Faça o upload do arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Ativar no WooCommerce POS" isLast>

1. Vá para `WP Admin > WooCommerce POS > Configurações > Finalização`
2. Encontre o **Gateway de Pagamento por Fatura do WooCommerce POS** na lista
3. Ative o gateway para uso no POS
4. Salve suas configurações

:::note
Este gateway não requer chaves de API adicionais ou configuração de serviço externo. Ele utiliza o sistema de email existente do WordPress e os métodos de pagamento do WooCommerce.
:::

  </StepCard>
</Steps>

## Uso

### Enviando Faturas

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Fatura por Email" como o método de pagamento
3. **Endereço de Email**: 
   - Para clientes existentes: O email é preenchido automaticamente com os dados do cliente
   - Para pedidos de convidados: Insira manualmente o endereço de email do cliente
4. **Opcional**: Marque "Salvar no endereço de cobrança" para atualizar as informações de cobrança do cliente
5. **Enviar Fatura**: Conclua o processo para enviar o email da fatura

### Experiência do Cliente

Quando você envia uma fatura, o cliente recebe um email contendo:

- **Detalhes do Pedido**: Detalhamento completo de itens, quantidades e preços
- **Total do Pedido**: Valor final devido, incluindo impostos e taxas
- **Link de Pagamento**: Link direto para concluir o pagamento online
- **Informações do Pedido**: Número do pedido e detalhes de referência

### Conclusão do Pagamento

Os clientes podem concluir o pagamento de duas maneiras:

1. **Pagamento Online**: Clique no link "Pagar por este pedido" no email
2. **Retorno na Loja**: Retorne para concluir o pagamento usando outros métodos de pagamento do POS

O link "Pagar por este pedido" leva os clientes a uma página de pagamento dedicada em seu site onde eles podem usar qualquer um dos gateways de pagamento online ativados (Stripe, PayPal, etc.).

## Gerenciamento de Email

### Clientes Existentes

- Endereços de email são recuperados automaticamente dos perfis dos clientes
- As informações de cobrança do cliente são preservadas
- O histórico de pedidos é mantido sob a conta do cliente

### Clientes Convidados

- Insira manualmente endereços de email para clientes ocasionais
- Opção de salvar o email no endereço de cobrança para referência futura
- Pedidos de convidados são devidamente rastreados no WooCommerce

### Modelos de Email

O gateway utiliza os modelos de email padrão do WooCommerce:

- **Personalizável**: Modifique a aparência do email através das configurações de email do WooCommerce
- **Personalizado**: Inclua o logotipo e a marca da sua loja
- **Profissional**: Formato de fatura limpo e profissional
- **Adaptado para Dispositivos Móveis**: Design responsivo para todos os dispositivos

## Casos de Uso

### Perfeito Para

- **Vendas B2B**: Clientes empresariais que precisam processar pagamentos através de seus sistemas de contabilidade
- **Grandes Pedidos**: Transações de alto valor que exigem aprovação ou tempo de processamento
- **Clientes Remotos**: Clientes que precisam concluir o pagamento após sair da loja
- **Clientes com Conta**: Clientes regulares com termos de pagamento estabelecidos
- **Citação para Pedido**: Convertendo cotações ou estimativas em faturas pagáveis

## Resolução de Problemas

### Problemas Comuns

<Accordion>
  <AccordionItem question="Emails não estão sendo enviados">

  - Verifique a configuração de email do WordPress
  - Verifique as configurações SMTP se estiver usando um serviço de email personalizado
  - Teste com plugins de teste de email do WordPress
  - Verifique pastas de spam para emails de teste

  </AccordionItem>

  <AccordionItem question="Links de pagamento não estão funcionando">

  - Assegure-se de que o WooCommerce esteja corretamente configurado
  - Verifique se pelo menos um gateway de pagamento está habilitado para finalização web
  - Verifique se o certificado SSL está devidamente instalado
  - Confirme se o status do pedido permite o pagamento

  </AccordionItem>

  <AccordionItem question="Cliente não consegue concluir o pagamento">

  - Verifique se os gateways de pagamento web estão ativos e configurados
  - Verifique se o pedido não expirou ou foi cancelado
  - Assegure-se de que o cliente tenha limites de método de pagamento suficientes
  - Teste o processo de pagamento você mesmo

  </AccordionItem>
</Accordion>

### Entrega de Email

Para entrega confiável de emails, considere:

- **Serviço SMTP**: Use serviços como SendGrid, Mailgun ou Amazon SES
- **Plugins de Email**: Instale plugins SMTP do WordPress para melhor entrega
- **Autenticação de Domínio**: Configure registros SPF, DKIM e DMARC
- **Monitoramento**: Acompanhe a entrega de emails e taxas de abertura
