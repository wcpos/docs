# Gateway de Checkout Web

O gateway de Checkout Web permite que os clientes concluam suas transações de POS usando o sistema de checkout da sua loja web. Esse gateway cria uma ponte perfeita entre seu POS em loja e os métodos de pagamento online, ideal para situações onde os clientes preferem pagar usando seus próprios dispositivos ou quando você precisa de acesso a métodos de pagamento disponíveis apenas na web.

## Instalação

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Acesse a [página de lançamentos do Gateway de Checkout Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Baixe o arquivo mais recente **woocommerce-pos-web-checkout-gateway.zip**
3. No seu admin do WordPress, vá para `Plugins > Adicionar Novo > Enviar Plugin`
4. Envie o arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Habilitar no WooCommerce POS" isLast>

1. Vá para `WP Admin > WooCommerce POS > Configurações > Checkout`
2. Encontre o **Gateway de Checkout Web do WooCommerce POS** na lista
3. Habilite o gateway para uso no POS
4. Salve suas configurações

:::note
Esse gateway utiliza seus métodos de pagamento existentes do WooCommerce, então garanta que você tenha pelo menos um gateway de pagamento web configurado corretamente (Stripe, PayPal, etc.).
:::

  </StepCard>
</Steps>

## Uso

### Iniciando o Checkout Web

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Checkout Web" como método de pagamento
3. **Gerar Link**: O sistema cria um link de checkout único para o pedido
4. **Acesso do Cliente**: O cliente clica no link para acessar o checkout web
5. **Pagamento Online**: O cliente conclui o pagamento usando os métodos de pagamento da loja web

### Processo de Pagamento

O gateway de Checkout Web segue este fluxo de trabalho:

1. **Criação do Pedido**: O POS cria um pedido pendente no WooCommerce
2. **Link de Checkout**: O sistema gera um link de pagamento seguro e único
3. **Redirecionamento do Cliente**: O cliente é direcionado para a página de checkout web
4. **Seleção de Pagamento**: O cliente escolhe entre os métodos de pagamento web disponíveis
5. **Processamento de Pagamento**: O processo padrão de checkout do WooCommerce lida com o pagamento
6. **Conclusão do Pedido**: O status do pedido é atualizado automaticamente após o pagamento bem-sucedido

### Fluxo de Trabalho do POS

Após iniciar o checkout web:

1. **Monitorar Status**: Mantenha a tela de pedidos do POS aberta para monitorar o status do pagamento
2. **Pagamento do Cliente**: O cliente conclui o pagamento em seu dispositivo ou no seu tablet
3. **Atualização Automática**: O status do pedido é atualizado em tempo real quando o pagamento é recebido
4. **Processar Pagamento**: Clique no botão "Processar Pagamento" no POS para continuar com o recibo
5. **Geração do Recibo**: Gere e imprima o recibo normalmente

## Casos de Uso

### Perfeito Para

- **Preferência do Cliente**: Clientes que preferem usar seus próprios aplicativos ou cartões de pagamento
- **Pagamentos Complexos**: Transações que exigem métodos de pagamento não disponíveis no POS
- **Pagamentos Compartilhados**: Clientes desejando usar múltiplos métodos de pagamento
- **Programas de Fidelidade**: Acesso à troca de pontos de fidelidade baseados na web
- **Cartões Presente**: Clientes com cartões presente digitais ou crédito na loja
- **Cartões Internacionais**: Métodos de pagamento que funcionam melhor através de gateways web

## Detalhes Técnicos

### Gerenciamento de Pedidos

- **Pedidos Pendentes**: Os pedidos são criados com status "pagamento pendente"
- **Atualizações de Status**: Mudanças de status automáticas quando o pagamento é recebido
- **Notas de Pedido**: Os detalhes do pagamento são registrados nas notas do pedido
- **Inventário**: O estoque é reservado durante o processo de checkout

### Segurança

- **Links Únicos**: Cada sessão de checkout possui uma URL única e com limite de tempo
- **SSL Necessário**: Todo o processamento de pagamentos ocorre através de conexões HTTPS seguras
- **Conformidade com PCI**: Aproveita seus gateways de pagamento web já compatíveis com PCI
- **Gerenciamento de Sessão**: O manuseio seguro de sessões previne acesso não autorizado

### Métodos de Pagamento

O gateway fornece acesso a todos os seus métodos de pagamento WooCommerce configurados:

- **Cartões de Crédito/Débito**: Stripe, Square, Authorize.net, etc.
- **Wallets Digitais**: PayPal, Apple Pay, Google Pay
- **Transferências Bancárias**: Métodos de pagamento bancário direto
- **Compre Agora, Pague Depois**: Klarna, Afterpay, Sezzle
- **Criptomoeda**: Bitcoin e outros gateways de pagamento em criptomoeda
- **Métodos Locais**: Métodos de pagamento específicos da região

## Solução de Problemas

### Problemas Comuns

<Accordion>
  <AccordionItem question="Link de checkout não funcionando">

  - Verifique se o certificado SSL está instalado corretamente
  - Verifique se os permalinks do WooCommerce estão configurados corretamente
  - Assegure que os gateways de pagamento web estão ativos e configurados
  - Teste com um navegador ou dispositivo diferente

  </AccordionItem>

  <AccordionItem question="Pagamento não processando">

  - Confirme se os gateways de pagamento web estão configurados corretamente
  - Verifique os logs do gateway de pagamento para mensagens de erro
  - Verifique se o método de pagamento do cliente é suportado
  - Teste o processo de checkout web de forma independente

  </AccordionItem>

  <AccordionItem question="Pedido não atualizando no POS">

  - Verifique a estabilidade da conexão de internet
  - Verifique se os trabalhos cron do WordPress estão rodando corretamente
  - Atualize a tela de pedidos do POS
  - Verifique o status do pedido no admin do WooCommerce

  </AccordionItem>

  <AccordionItem question="Cliente não consegue acessar o checkout">

  - Assegure que o link de checkout não expirou
  - Verifique se o dispositivo do cliente tem conectividade com a internet
  - Verifique se o pedido não foi cancelado ou concluído
  - Teste o link em um dispositivo diferente

  </AccordionItem>
</Accordion>

### Otimização de Desempenho

Para desempenho ideal:

- **Cache**: Configure plugins de cache para excluir páginas de checkout
- **CDN**: Assegure que as configurações do CDN não interfiram no processo de checkout
- **Banco de Dados**: Otimize o banco de dados para um processamento de pedidos mais rápido
- **Hospedagem**: Utilize uma hospedagem confiável com boa disponibilidade

### Obtendo Ajuda

Para suporte técnico:
- Visite o [repositório do GitHub](https://github.com/wcpos/web-checkout-gateway) para relatar problemas
- Verifique a documentação do gateway de pagamento do WooCommerce
- Teste regularmente o seu processo de checkout web
- Monitore os logs do gateway de pagamento para problemas
