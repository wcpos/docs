# Gateway de Checkout na Web

O gateway de Checkout na Web permite que os clientes concluam suas transações de POS usando o sistema de checkout da sua loja web. Este gateway cria uma ponte perfeita entre seu POS na loja e métodos de pagamento online, ideal para situações em que os clientes preferem pagar usando seus próprios dispositivos ou quando você precisa de acesso a métodos de pagamento disponíveis apenas na web.

## Recursos

- **Integração Transparente**: Link direto do POS para o checkout na web
- **Todos os Métodos de Pagamento**: Acesso a todos os gateways de pagamento habilitados na sua loja web
- **Controle do Cliente**: Os clientes podem pagar usando seus próprios dispositivos e métodos preferidos
- **Sincronização de Pedidos**: Atualizações automáticas de pedidos entre o POS e a loja web
- **Fluxo de Trabalho Flexível**: Conclua transações online enquanto mantém o fluxo de trabalho do POS

## Instalação

### Passo 1: Baixar e Instalar

1. Visite a [página de lançamentos do Gateway de Checkout na Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Baixe o arquivo **woocommerce-pos-web-checkout-gateway.zip** mais recente
3. No seu painel do WordPress, vá para `Plugins > Adicionar Novo > Enviar Plugin`
4. Envie o arquivo zip e ative o plugin

### Passo 2: Habilitar no WooCommerce POS

1. Vá para `WP Admin > WooCommerce POS > Configurações > Checkout`
2. Encontre o **Gateway de Checkout na Web do WooCommerce POS** na lista
3. Habilite o gateway para uso no POS
4. Salve suas configurações

:::note
Este gateway utiliza seus métodos de pagamento existentes do WooCommerce, então certifique-se de ter pelo menos um gateway de pagamento na web configurado corretamente (Stripe, PayPal, etc.).
:::

## Uso

### Iniciando o Checkout na Web

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Checkout na Web" como o método de pagamento
3. **Gerar Link**: O sistema cria um link de checkout único para o pedido
4. **Acesso do Cliente**: O cliente clica no link para acessar o checkout na web
5. **Pagamento Online**: O cliente completa o pagamento usando os métodos de pagamento da loja web

### Processo de Pagamento

O gateway de Checkout na Web segue este fluxo de trabalho:

1. **Criação do Pedido**: O POS cria um pedido pendente no WooCommerce
2. **Link de Checkout**: O sistema gera um link de pagamento único e seguro
3. **Redirecionamento do Cliente**: O cliente é direcionado para a página de checkout na web
4. **Selecionar Pagamento**: O cliente escolhe entre os métodos de pagamento web disponíveis
5. **Processamento de Pagamento**: O processo de checkout padrão do WooCommerce gerencia o pagamento
6. **Conclusão do Pedido**: O status do pedido é atualizado automaticamente após o pagamento bem-sucedido

### Fluxo de Trabalho do POS

Após iniciar o checkout na web:

1. **Monitorar Status**: Mantenha a tela do pedido do POS aberta para monitorar o status do pagamento
2. **Pagamento do Cliente**: O cliente completa o pagamento em seu dispositivo ou em seu tablet
3. **Atualização Automática**: O status do pedido é atualizado em tempo real quando o pagamento é recebido
4. **Processar Pagamento**: Clique no botão "Processar Pagamento" no POS para continuar para o recibo
5. **Geração de Recibo**: Gere e imprima o recibo como de costume

## Casos de Uso

### Perfeito Para

- **Preferência do Cliente**: Clientes que preferem usar seus próprios aplicativos ou cartões de pagamento
- **Pagamentos Complexos**: Transações que requerem métodos de pagamento não disponíveis no POS
- **Pagamentos Divididos**: Clientes que desejam usar múltiplos métodos de pagamento
- **Programas de Fidelidade**: Acesso ao resgate de pontos de fidelidade baseados na web
- **Cartões Presente**: Clientes com cartões presente digitais ou créditos da loja
- **Cartões Internacionais**: Métodos de pagamento que funcionam melhor através de gateways web

### Exemplos de Fluxo de Trabalho

**Loja de Varejo**
1. O cliente faz compras na loja e traz itens para o checkout
2. O cliente prefere pagar com seu aplicativo de carteira móvel
3. A equipe seleciona o gateway de Checkout na Web
4. O cliente escaneia o código QR ou clica no link em seu telefone
5. O cliente completa o pagamento usando o método preferido
6. A equipe processa o recibo e conclui a transação

**Negócio de Serviços**
1. Complete o serviço e adicione as cobranças ao POS
2. O cliente deseja pagar com o cartão de crédito da empresa
3. Gere o link de checkout na web
4. O cliente insere os detalhes do pagamento em um formulário web seguro
5. O pagamento é processado através do seu gateway web
6. O pedido é concluído e o recibo é gerado

## Detalhes Técnicos

### Gerenciamento de Pedidos

- **Pedidos Pendentes**: Pedidos são criados com o status "pagamento pendente"
- **Atualizações de Status**: Mudanças automáticas de status quando o pagamento é recebido
- **Notas do Pedido**: Detalhes do pagamento são registrados nas notas do pedido
- **Inventário**: Estoque é reservado durante o processo de checkout

### Segurança

- **Links Exclusivos**: Cada sessão de checkout possui uma URL única e com limite de tempo
- **SSL Necessário**: Todo o processamento de pagamento ocorre em conexões HTTPS seguras
- **Conformidade PCI**: Utiliza seus gateways de pagamento web existentes que estão em conformidade com PCI
- **Gerenciamento de Sessão**: O manuseio seguro de sessão evita acesso não autorizado

### Métodos de Pagamento

O gateway oferece acesso a todos os seus métodos de pagamento configurados no WooCommerce:

- **Cartões de Crédito/Débito**: Stripe, Square, Authorize.net, etc.
- **Carteiras Digitais**: PayPal, Apple Pay, Google Pay
- **Transferências Bancárias**: Métodos de pagamento diretos
- **Compre Agora, Pague Depois**: Klarna, Afterpay, Sezzle
- **Criptomoeda**: Bitcoin e outros gateways de pagamento em criptomoeda
- **Métodos Locais**: Métodos de pagamento específicos da região

## Requisitos

- **WooCommerce POS**: Versão gratuita ou Pro instalada e ativada
- **Gateways de Pagamento Web**: Pelo menos um método de pagamento online configurado no WooCommerce
- **Certificado SSL**: HTTPS necessário para processamento de pagamento seguro
- **Navegador Moderno**: O dispositivo do cliente deve suportar padrões web modernos

## Solução de Problemas

### Problemas Comuns

**Link de Checkout Não Funciona**
- Verifique se o certificado SSL está instalado corretamente
- Confira se os links permanentes do WooCommerce estão configurados corretamente
- Assegure-se de que os gateways de pagamento da web estejam ativos e configurados
- Teste com um navegador ou dispositivo diferente

**Pagamento Não Está Processando**
- Confirme se os gateways de pagamento da web estão configurados corretamente
- Verifique os logs do gateway de pagamento em busca de mensagens de erro
- Verifique se o método de pagamento do cliente é suportado
- Teste o processo de checkout na web independentemente

**Pedido Não Atualiza no POS**
- Verifique a estabilidade da conexão com a internet
- Verifique se as tarefas cron do WordPress estão sendo executadas corretamente
- Atualize a tela de pedidos do POS
- Verifique o status do pedido no painel do WooCommerce

**Cliente Não Consegue Acessar o Checkout**
- Assegure-se de que o link de checkout não tenha expirado
- Verifique se o dispositivo do cliente tem conectividade com a internet
- Confira se o pedido não foi cancelado ou concluído
- Teste o link em um dispositivo diferente

### Otimização de Desempenho

Para desempenho ideal:

- **Cache**: Configure plugins de cache para excluir páginas de checkout
- **CDN**: Assegure-se de que as configurações do CDN não interfiram no processo de checkout
- **Banco de Dados**: Otimize o banco de dados para processamento de pedidos mais rápido
- **Hospedagem**: Use hospedagem confiável com bom tempo de atividade

### Obtendo Ajuda

Para suporte técnico:
- Visite o [repositório do GitHub](https://github.com/wcpos/web-checkout-gateway) para relatar problemas
- Verifique a documentação do gateway de pagamento do WooCommerce
- Teste seu processo de checkout na web regularmente
- Monitore os logs do gateway de pagamento em busca de problemas
