# Portal de Checkout na Web {#web-checkout-gateway}

O portal de checkout na web permite que os clientes concluam suas transações de POS usando o sistema de checkout da sua loja online. Este portal cria uma ponte sem costura entre seu POS em loja e os métodos de pagamento online, perfeito para situações em que os clientes preferem pagar usando seus próprios dispositivos ou quando você precisa de acesso a métodos de pagamento exclusivamente online.

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Visite a [página de lançamentos do Portal de Checkout na Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Baixe o arquivo mais recente **woocommerce-pos-web-checkout-gateway.zip**
3. No seu painel do WordPress, vá para `Plugins > Adicionar Novo > Enviar Plugin`
4. Faça o upload do arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Habilitar no WCPOS" isLast>

1. Vá para `WP Admin > POS > Configurações > Checkout`
2. Encontre o **Portal de Checkout na Web do WCPOS** na lista
3. Habilite o portal para uso no POS
4. Salve suas configurações

:::note
Este portal aproveita seus métodos de pagamento existentes no WooCommerce, então assegure-se de ter pelo menos um gateway de pagamento online configurado corretamente (Stripe, PayPal, etc.).
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Iniciando o Checkout na Web {#initiating-web-checkout}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Portal**: Escolha "Checkout na Web" como o método de pagamento
3. **Gerar Link**: O sistema cria um link de checkout único para o pedido
4. **Acesso do Cliente**: O cliente clica no link para acessar o checkout na web
5. **Pagamento Online**: O cliente completa o pagamento usando os métodos de pagamento da loja online

### Processo de Pagamento {#payment-process}

O portal de checkout na web segue este fluxo de trabalho:

1. **Criação do Pedido**: O POS cria um pedido pendente no WooCommerce
2. **Link de Checkout**: O sistema gera um link de pagamento seguro e único
3. **Redirecionamento do Cliente**: O cliente é direcionado para a página de checkout na web
4. **Escolha do Pagamento**: O cliente escolhe entre os métodos de pagamento online disponíveis
5. **Processamento do Pagamento**: O processo padrão de checkout do WooCommerce lida com o pagamento
6. **Conclusão do Pedido**: O status do pedido é atualizado automaticamente após o pagamento bem-sucedido

### Fluxo de Trabalho do POS {#pos-workflow}

Após iniciar o checkout na web:

1. **Monitorar Status**: Mantenha a tela de pedidos do POS aberta para monitorar o status do pagamento
2. **Pagamento do Cliente**: O cliente completa o pagamento em seu dispositivo ou no seu tablet
3. **Atualização Automática**: O status do pedido é atualizado em tempo real quando o pagamento é recebido
4. **Processar Pagamento**: Clique no botão "Processar Pagamento" no POS para continuar até o recibo
5. **Geração do Recibo**: Gere e imprima o recibo como de costume

## Casos de Uso {#use-cases}

### Perfeito Para {#perfect-for}

- **Preferência do Cliente**: Clientes que preferem usar seus próprios aplicativos de pagamento ou cartões
- **Pagamentos Complexos**: Transações que exigem métodos de pagamento não disponíveis no POS

- **Cartões Presente**: Clientes com cartões presente digitais ou crédito na loja
- **Cartões Internacionais**: Métodos de pagamento que funcionam melhor através de gateways web

## Detalhes Técnicos {#technical-details}

### Gestão de Pedidos {#order-management}

- **Pedidos Pendentes**: Pedidos são criados com o status "pagamento pendente"
- **Atualizações de Status**: Mudanças automáticas de status quando o pagamento é recebido
- **Notas do Pedido**: Detalhes do pagamento são registrados nas notas do pedido
- **Inventário**: Estoque é reservado durante o processo de checkout

### Segurança {#security}

- **Links Únicos**: Cada sessão de checkout tem um URL único e com limite de tempo
- **SSL Necessário**: Todo o processamento de pagamento ocorre sobre conexões HTTPS seguras
- **Conformidade PCI**: Aproveita seus gateways de pagamento web já compatíveis com PCI
- **Gestão de Sessões**: O manuseio de sessões seguras previne acesso não autorizado

### Métodos de Pagamento {#payment-methods}

O portal fornece acesso a todos os seus métodos de pagamento do WooCommerce configurados:

- **Cartões de Crédito/Débito**: Stripe, Square, Authorize.net, etc.
- **Carteiras Digitais**: PayPal, Apple Pay, Google Pay
- **Transferências Bancárias**: Métodos de pagamento bancário direto
- **Compre Agora, Pague Depois**: Klarna, Afterpay, Sezzle
- **Criptomoeda**: Bitcoin e outros gateways de pagamento em criptomoedas
- **Métodos Locais**: Métodos de pagamento específicos da região

## Solução de Problemas {#troubleshooting}

### Problemas Comuns {#common-issues}

<Accordion>
  <AccordionItem question="Link de checkout não funcionando">

  - Verifique se o certificado SSL está instalado corretamente
  - Verifique se os links permanentes do WooCommerce estão configurados corretamente
  - Assegure-se de que os gateways de pagamento web estão ativos e configurados
  - Teste com um navegador ou dispositivo diferente

  </AccordionItem>

  <AccordionItem question="Pagamento não processando">

  - Confirme se os gateways de pagamento web estão configurados corretamente
  - Verifique os logs do gateway de pagamento para mensagens de erro
  - Verifique se o método de pagamento do cliente é suportado
  - Teste o processo de checkout na web de forma independente

  </AccordionItem>

  <AccordionItem question="Pedido não atualizando no POS">

  - Verifique a estabilidade da conexão com a internet
  - Verifique se as tarefas agendadas do WordPress estão sendo executadas corretamente
  - Atualize a tela de pedidos do POS
  - Verifique o status do pedido no admin do WooCommerce

  </AccordionItem>

  <AccordionItem question="Cliente não consegue acessar o checkout">

  - Assegure-se de que o link de checkout não expirou
  - Verifique se o dispositivo do cliente tem conectividade com a internet
  - Verifique se o pedido não foi cancelado ou concluído
  - Teste o link em um dispositivo diferente

  </AccordionItem>
</Accordion>

### Otimização de Performance {#performance-optimisation}

Para desempenho ideal:

- **Cache**: Configure plugins de cache para excluir páginas de checkout
- **CDN**: Assegure-se de que as configurações da CDN não interfiram no processo de checkout
- **Banco de Dados**: Otimize o banco de dados para um processamento de pedidos mais rápido
- **Hospedagem**: Utilize hospedagem confiável com boa disponibilidade

### Obtendo Ajuda {#getting-help}

Para suporte técnico:
- Visite o [repositório do GitHub](https://github.com/wcpos/web-checkout-gateway) para relatar problemas
- Verifique a documentação do gateway de pagamento do WooCommerce
- Teste seu processo de checkout na web regularmente
- Monitore os logs do gateway de pagamento para problemas
