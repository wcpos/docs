# Gateway de Checkout Web

O gateway de Checkout Web permite que os clientes concluam suas transações de POS usando o sistema de checkout da sua loja online. Este gateway cria uma ponte perfeita entre o POS da loja física e os métodos de pagamento online, ideal para situações em que os clientes preferem pagar usando seus próprios dispositivos ou quando você precisa acessar métodos de pagamento exclusivos da web.

## Recursos

- **Integração Perfeita**: Link direto do POS para checkout web
- **Todos os Métodos de Pagamento**: Acesso a todos os gateways de pagamento habilitados na loja web
- **Controle do Cliente**: Clientes podem pagar usando seus próprios dispositivos e métodos preferidos
- **Sincronização de Pedidos**: Atualizações automáticas de pedidos entre POS e loja web
- **Fluxo de Trabalho Flexível**: Conclua transações online mantendo o fluxo de trabalho do POS

## Instalação

### Etapa 1: Baixar e Instalar

1. Visite a [página de lançamentos do Gateway de Checkout Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Baixe o arquivo **woocommerce-pos-web-checkout-gateway.zip** mais recente
3. No admin do WordPress, vá para `Plugins > Adicionar Novo > Enviar Plugin`
4. Faça upload do arquivo zip e ative o plugin

### Etapa 2: Habilitar no WooCommerce POS

1. Vá para `Admin do WP > WooCommerce POS > Configurações > Checkout`
2. Encontre o **Gateway de Checkout Web do WooCommerce POS** na lista
3. Habilite o gateway para uso no POS
4. Salve suas configurações

:::note
Este gateway utiliza seus métodos de pagamento existentes do WooCommerce, portanto, certifique-se de ter pelo menos um gateway de pagamento web configurado corretamente (Stripe, PayPal, etc.).
:::

## Uso

### Iniciando o Checkout Web

1. **Adicionar Itens**: Adicione produtos ao carrinho no POS
2. **Selecionar Gateway**: Escolha "Checkout Web" como método de pagamento
3. **Gerar Link**: O sistema cria um link de checkout único para o pedido
4. **Acesso do Cliente**: O cliente clica no link para acessar o checkout web
5. **Pagamento Online**: O cliente conclui o pagamento usando métodos de pagamento da loja web

### Processo de Pagamento

O gateway de Checkout Web segue este fluxo de trabalho:

1. **Criação do Pedido**: O POS cria um pedido pendente no WooCommerce
2. **Link de Checkout**: O sistema gera um link de pagamento seguro e único
3. **Redirecionamento do Cliente**: O cliente é direcionado para a página de checkout web
4. **Seleção de Pagamento**: O cliente escolhe entre os métodos de pagamento web disponíveis
5. **Processamento de Pagamento**: O processo de checkout padrão do WooCommerce lida com o pagamento
6. **Conclusão do Pedido**: O status do pedido é atualizado automaticamente após o pagamento bem-sucedido

### Fluxo de Trabalho do POS

Após iniciar o checkout web:

1. **Monitorar Status**: Mantenha a tela de pedidos do POS aberta para monitorar o status do pagamento
2. **Pagamento do Cliente**: O cliente conclui o pagamento em seu dispositivo ou tablet
3. **Atualização Automática**: O status do pedido é atualizado em tempo real quando o pagamento é recebido
4. **Processar Pagamento**: Clique no botão "Processar Pagamento" no POS para continuar para o recibo
5. **Geração de Recibo**: Gere e imprima o recibo normalmente

## Casos de Uso

### Perfeito Para

- **Preferência do Cliente**: Clientes que preferem usar seus próprios aplicativos ou cartões de pagamento
- **Pagamentos Complexos**: Transações que requerem métodos de pagamento não disponíveis no POS
- **Pagamentos Divididos**: Clientes que desejam usar múltiplos métodos de pagamento
- **Programas de Fidelidade**: Acesso ao resgate de pontos de fidelidade baseado na web
- **Cartões-presente**: Clientes com cartões-presente digitais ou crédito de loja
- **Cartões Internacionais**: Métodos de pagamento que funcionam melhor através de gateways web

### Exemplos de Fluxo de Trabalho

**Loja de Varejo**
1. O cliente compra na loja física e leva os itens para o caixa
2. O cliente prefere pagar com seu aplicativo de carteira móvel
3. A equipe seleciona o gateway de Checkout Web
4. O cliente escaneia o código QR ou clica no link em seu telefone
5. O cliente conclui o pagamento usando o método preferido
6. A equipe processa o recibo e conclui a transação

**Negócio de Serviços**
1. Concluir o serviço e adicionar encargos ao POS
2. O cliente quer pagar com cartão de crédito corporativo
3. Gerar link de checkout web
4. O cliente insere os detalhes de pagamento no formulário web seguro
5. O pagamento é processado através do seu gateway web
6. O pedido é concluído e o recibo é gerado

## Detalhes Técnicos

### Gerenciamento de Pedidos

- **Pedidos Pendentes**: Pedidos são criados com status "pagamento pendente"
- **Atualizações de Status**: Alterações automáticas de status quando o pagamento é recebido
- **Notas de Pedido**: Detalhes de pagamento são registrados em notas de pedido
- **Estoque**: Estoque é reservado durante o processo de checkout

### Segurança

- **Links Únicos**: Cada sessão de checkout tem uma URL única e com tempo limitado
- **SSL Obrigatório**: Todo processamento de pagamento ocorre em conexões HTTPS seguras
- **Conformidade PCI**: Utiliza seus gateways de pagamento web já compatíveis com PCI
- **Gerenciamento de Sessão**: Manipulação de sessão segura previne acesso não autorizado

### Métodos de Pagamento

O gateway fornece acesso a todos os métodos de pagamento do WooCommerce configurados:

- **Cartões de Crédito/Débito**: Stripe, Square, Authorize.net, etc.
- **Carteiras Digitais**: PayPal, Apple Pay, Google Pay
- **Transferências Bancárias**: Métodos de pagamento bancário direto
- **Compre Agora, Pague Depois**: Klarna, Afterpay, Sezzle
- **Criptomoedas**: Gateways de pagamento de Bitcoin e outras criptomoedas
- **Métodos Locais**: Métodos de pagamento específicos da região

## Requisitos

- **WooCommerce POS**: Versão gratuita ou pro instalada e ativada
- **Gateways de Pagamento Web**: Pelo menos um método de pagamento online configurado no WooCommerce
- **Certificado SSL**: HTTPS obrigatório para processamento de pagamento seguro
- **Navegador Moderno**: O dispositivo do cliente deve suportar padrões web modernos

## Solução de Problemas

### Problemas Comuns

**Link de Checkout Não Funciona**
- Verificar se o certificado SSL está instalado corretamente
- Verificar se os permalinks do WooCommerce estão configurados corretamente
- Garantir que os gateways de pagamento web estejam ativos e configurados
- Testar com um navegador ou dispositivo diferente

**Pagamento Não Processando**
- Confirmar se os gateways de pagamento web estão configurados corretamente
- Verificar logs de gateway de pagamento em busca de mensagens de erro
- Verificar se o método de pagamento do cliente é suportado
- Testar o processo de checkout web independentemente

**Pedido Não Atualizando no POS**
- Verificar a estabilidade da conexão de internet
- Verificar se os trabalhos cron do WordPress estão funcionando corretamente
- Atualizar a tela de pedidos do POS
- Verificar o status do pedido no admin do WooCommerce

**Cliente Não Consegue Acessar o Checkout**
- Garantir que o link de checkout não tenha expirado
- Verificar se o dispositivo do cliente tem conectividade de internet
- Verificar se o pedido não foi cancelado ou concluído
- Testar o link em um dispositivo diferente

### Otimização de Desempenho

Para desempenho ideal:

- **Cache**: Configurar plugins de cache para excluir páginas de checkout
- **CDN**: Garantir que as configurações de CDN não interfiram no processo de checkout
- **Banco de Dados**: Otimizar banco de dados para processamento de pedidos mais rápido
- **Hospedagem**: Usar hospedagem confiável com boa disponibilidade

### Obtendo Ajuda

Para suporte técnico:
- Visite o [repositório do GitHub](https://github.com/wcpos/web-checkout-gateway) para relatar problemas
- Consulte a documentação de gateway de pagamento do WooCommerce
- Testar o processo de checkout web regularmente
- Monitorar logs de gateway de pagamento para problemas
