# Portal de Pagamento Web {#web-checkout-gateway}

O portal de pagamento web permite que os clientes concluam suas transações de POS usando o sistema de checkout da sua loja online. Este gateway cria uma ponte contínua entre o seu POS na loja e os métodos de pagamento online, perfeito para situações onde os clientes preferem pagar usando seus próprios dispositivos ou quando você precisa de acesso a métodos de pagamento disponíveis apenas na web.

## Instalação {#installation}

<Steps>
  <StepCard number="1" title="Baixar e Instalar">

1. Visite a [página de lançamentos do Portal de Pagamento Web](https://github.com/wcpos/web-checkout-gateway/releases)
2. Baixe o arquivo mais recente **woocommerce-pos-web-checkout-gateway.zip**
3. No seu admin do WordPress, vá para `Plugins > Adicionar Novo > Enviar Plugin`
4. Faça o upload do arquivo zip e ative o plugin

  </StepCard>
  <StepCard number="2" title="Habilitar no WooCommerce POS" isLast>

1. Vá para `WP Admin > WooCommerce POS > Configurações > Checkout`
2. Encontre o **Portal de Pagamento Web do WooCommerce POS** na lista
3. Habilite o gateway para uso no POS
4. Salve suas configurações

:::note
Este gateway aproveita seus métodos de pagamento existentes do WooCommerce, então assegure-se de que você tenha pelo menos um gateway de pagamento web configurado corretamente (Stripe, PayPal, etc.).
:::

  </StepCard>
</Steps>

## Uso {#usage}

### Iniciando o Checkout Web {#initiating-web-checkout}

1. **Adicionar Itens**: Adicione produtos ao seu carrinho no POS
2. **Selecionar Gateway**: Escolha "Checkout Web" como o método de pagamento
3. **Gerar Link**: O sistema cria um link de checkout único para o pedido
4. **Acesso do Cliente**: O cliente clica no link para acessar o checkout web
5. **Pagamento Online**: O cliente completa o pagamento usando os métodos de pagamento da loja web

### Processo de Pagamento {#payment-process}

O portal de pagamento web segue este fluxo de trabalho:

1. **Criação do Pedido**: O POS cria um pedido pendente no WooCommerce
2. **Link de Checkout**: O sistema gera um link de pagamento seguro e único
3. **Redirecionamento do Cliente**: O cliente é direcionado para a página de checkout web
4. **Seleção de Pagamento**: O cliente escolhe entre os métodos de pagamento web disponíveis
5. **Processamento de Pagamento**: O processo de checkout padrão do WooCommerce lida com o pagamento
6. **Conclusão do Pedido**: O status do pedido é atualizado automaticamente após o pagamento bem-sucedido

### Fluxo de Trabalho do POS {#pos-workflow}

Após iniciar o checkout web:

1. **Monitorar Status**: Mantenha a tela de pedidos do POS aberta para monitorar o status do pagamento
2. **Pagamento do Cliente**: O cliente completa o pagamento em seu dispositivo ou no seu tablet
3. **Atualização Automática**: O status do pedido é atualizado em tempo real quando o pagamento é recebido
4. **Processar Pagamento**: Clique no botão "Processar Pagamento" no POS para continuar para o recibo
5. **Geração de Recibo**: Gere e imprima o recibo como de costume

## Casos de Uso {#use-cases}

### Perfeito Para {#perfect-for}

- **Preferência do Cliente**: Clientes que preferem usar seus próprios aplicativos ou cartões de pagamento
- **Pagamentos Complexos**: Transações que exigem métodos de pagamento não disponíveis no POS
- **Pagamentos Divididos**: Clientes que desejam usar múltiplos métodos de pagamento
- **Programas de Fidelidade**: Acesso à troca de pontos de fidelidade baseados na web
- **Cartões Presente**: Clientes com cartões-presente digitais ou créditos na loja
- **Cartões Internacionais**: Métodos de pagamento que funcionam melhor através de gateways web

## Detalhes Técnicos {#technical-details}

### Gerenciamento de Pedidos {#order-management}

- **Pedidos Pendentes**: Os pedidos são criados com status "pendente de pagamento"
- **Atualizações de Status**: Mudanças automáticas de status quando o pagamento é recebido
- **Notas de Pedido**: Detalhes do pagamento são registrados nas notas do pedido
- **Inventário**: Estoque é reservado durante o processo de checkout

### Segurança {#security}

- **Links Únicos**: Cada sessão de checkout possui uma URL única e com limite de tempo
- **SSL Requerido**: Todo o processamento de pagamento ocorre por conexões HTTPS seguras
- **Conformidade PCI**: Aproveita seus gateways de pagamento web existentes que são compatíveis com PCI
- **Gerenciamento de Sessão**: O tratamento seguro de sessão previne acesso não autorizado

### Métodos de Pagamento {#payment-methods}

O gateway fornece acesso a todos os seus métodos de pagamento configurados no WooCommerce:

- **Cartões de Crédito/Debito**: Stripe, Square, Authorize.net, etc.
- **Carteiras Digitais**: PayPal, Apple Pay, Google Pay
- **Transferências Bancárias**: Métodos de pagamento bancário direto
- **Compre Agora, Pague Depois**: Klarna, Afterpay, Sezzle
- **Criptomoedas**: Bitcoin e outros gateways de pagamento cripto
- **Métodos Locais**: Métodos de pagamento específicos da região

## Solução de Problemas {#troubleshooting}

### Problemas Comuns {#common-issues}

<Accordion>
  <AccordionItem question="Link de checkout não funciona">

  - Verifique se o certificado SSL está instalado corretamente
  - Verifique se os permalinks do WooCommerce estão configurados corretamente
  - Certifique-se de que os gateways de pagamento web estão ativos e configurados
  - Teste com um navegador ou dispositivo diferente

  </AccordionItem>

  <AccordionItem question="Pagamento não está processando">

  - Confirme que os gateways de pagamento web estão configurados corretamente
  - Verifique os logs do gateway de pagamento para mensagens de erro
  - Verifique se o método de pagamento do cliente é suportado
  - Teste o processo de checkout web independentemente

  </AccordionItem>

  <AccordionItem question="Pedido não atualizando no POS">

  - Verifique a estabilidade da conexão com a internet
  - Verifique se as tarefas cron do WordPress estão sendo executadas corretamente
  - Atualize a tela de pedidos do POS
  - Verifique o status do pedido no admin do WooCommerce

  </AccordionItem>

  <AccordionItem question="Cliente não consegue acessar o checkout">

  - Certifique-se de que o link de checkout não expirou
  - Verifique se o dispositivo do cliente tem conectividade com a internet
  - Verifique se o pedido não foi cancelado ou concluído
  - Teste o link em um dispositivo diferente

  </AccordionItem>
</Accordion>

### Otimização de Desempenho {#performance-optimisation}

Para desempenho ideal:

- **Cache**: Configure plugins de cache para excluir páginas de checkout
- **CDN**: Assegure que as configurações de CDN não interfiram com o processo de checkout
- **Banco de Dados**: Otimize o banco de dados para processamento de pedidos mais rápido
- **Hospedagem**: Utilize hospedagem confiável com boa disponibilidade

### Obtendo Ajuda {#getting-help}

Para suporte técnico:
- Visite o [repositório do GitHub](https://github.com/wcpos/web-checkout-gateway) para relatar problemas
- Verifique a documentação do gateway de pagamento do WooCommerce
- Teste regularmente seu processo de checkout web
- Monitore logs do gateway de pagamento para problemas
