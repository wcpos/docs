A tela de Logs registra a atividade do sistema dentro do POS. É uma ferramenta essencial para solucionar problemas e entender o que está acontecendo nos bastidores.

{/* Screenshot: logs-screen.png */}

## Acessando os Logs

Abra a tela de Logs no painel de navegação à esquerda (o ícone de batimento cardíaco).

## Visão Geral da Interface

### Pesquisa e Filtro

- **Pesquisar Logs** - Encontrar entradas de log específicas pelo conteúdo da mensagem
- **Filtro de Nível de Log** - Filtrar pelo nível de log (info, aviso, erro)
- **Configurações de Exibição** - Configurar colunas visíveis

### Tabela de Logs

Cada entrada de log mostra:

| Coluna | Descrição |
|--------|-------------|
| **Hora** | Quando o evento ocorreu |
| **Nível** | Severidade: info (cinza), aviso (laranja), erro (vermelho) |
| **Mensagem** | Descrição legível por humanos do evento |
| **Contexto** | JSON expansível com detalhes adicionais |
| **Código** | Código de erro (links para a documentação) |

## Níveis de Log

### Info (Cinza)

Operações normais e eventos bem-sucedidos:
- "Buscou todos os IDs para produtos"
- "Sincronizou novos clientes"
- "Buscou todos os IDs para impostos"

### Aviso (Laranja)

Questões não críticas que podem necessitar de atenção:
- "Não foi possível atualizar a sessão"
- "Tentativa de repetição para sincronização"

### Erro (Vermelho)

Problemas que precisam ser resolvidos:
- "Desculpe, você não pode listar recursos" (problema de permissão)
- "Solicitação da API falhou"
- "Erro de sincronização do banco de dados"

## Dados de Contexto

Clique na seta de expansão em qualquer entrada de log para ver o contexto JSON detalhado:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Esses dados incluem:
- **Contagem de itens** - Quantos registros foram afetados
- **Tempo de execução** - Quanto tempo a operação levou
- **Carga do servidor** - Métricas de desempenho do servidor
- **Detalhes do pedido/resposta** - Para depuração de problemas de API

## Códigos de Erro

Muitas entradas de log incluem um **código de erro** (por exemplo, `API02004`, `DB01001`). Esses códigos se conectam diretamente à documentação de [Códigos de Erro](/error-codes), que fornece:

- Explicação detalhada do erro
- Causas comuns
- Etapas de solução de problemas

## Entradas de Log Comuns

### Operações de Sincronização

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

Essas indicam atividade de sincronização normal. O contexto mostra quantos itens foram sincronizados.

### Erros de Permissão

```
[error] Sorry, you cannot list resources    {...} API02004
```

Isso geralmente significa que o usuário não tem permissão para acessar um recurso. Verifique os papéis do usuário em `WP Admin > POS > Configurações > Acesso`.

### Problemas de Sessão

```
[warn] Unable to refresh session    {...} API02007
```

O POS não conseguiu atualizar a sessão de autenticação. Isso pode exigir que você saia e entre novamente.

## Usando os Logs para Suporte

Ao solicitar suporte:

1. **Reproduza o problema** - Execute a ação que causa o problema
2. **Verifique os logs** - Procure erros (vermelhos) ou avisos (laranjas)
3. **Expanda o contexto** - Obtenha os detalhes JSON completos
4. **Anote o código de erro** - Inclua isso em sua solicitação de suporte
5. **Copie entradas relevantes** - Compartilhe as mensagens de log com o contexto

## Monitoramento de Desempenho

Os logs incluem tempo de execução e métricas de carga do servidor. Se você notar:

- **Altos tempos de execução** - O servidor pode estar sobrecarregado ou lento
- **Altos valores de carga do servidor** - Considere otimização do servidor
- **Tempo limite frequente** - Verifique a conectividade da rede
