A tela de Registros grava a atividade do sistema dentro do POS. É uma ferramenta essencial para depurar problemas e entender o que está acontecendo nos bastidores.

{/* Screenshot: logs-screen.png */}

## Acessando Registros

Abra a tela de Registros no menu de navegação lateral (ícone de batimento cardíaco).

## Visão Geral da Interface

### Busca e Filtro

- **Buscar Registros** - Encontrar entradas de registro específicas por conteúdo da mensagem
- **Filtro de Nível de Registro** - Filtrar por nível de registro (informação, aviso, erro)
- **Configurações de exibição** - Configurar colunas visíveis

### Tabela de Registros

Cada entrada de registro mostra:

| Coluna | Descrição |
|--------|-------------|
| **Hora** | Quando o evento ocorreu |
| **Nível** | Severidade: informação (cinza), aviso (laranja), erro (vermelho) |
| **Mensagem** | Descrição legível por humanos do evento |
| **Contexto** | JSON expansível com detalhes adicionais |
| **Código** | Código de erro (links para documentação) |

## Níveis de Registro

### Informação (Cinza)

Operações normais e eventos bem-sucedidos:
- "Buscou todos os IDs para produtos"
- "Sincronizou novos clientes"
- "Buscou todos os IDs para impostos"

### Aviso (Laranja)

Problemas não críticos que podem precisar de atenção:
- "Não foi possível atualizar a sessão"
- "Tentativa de repetição de sincronização"

### Erro (Vermelho)

Problemas que precisam ser resolvidos:
- "Desculpe, você não pode listar recursos" (problema de permissão)
- "Falha na solicitação de API"
- "Erro de sincronização de banco de dados"

## Dados de Contexto

Clique na seta de expansão em qualquer entrada de registro para ver o contexto JSON detalhado:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

Estes dados incluem:
- **Contagem de itens** - Quantos registros foram afetados
- **Tempo de execução** - Quanto tempo a operação levou
- **Carga do servidor** - Métricas de desempenho do servidor
- **Detalhes de solicitação/resposta** - Para depurar problemas de API

## Códigos de Erro

Muitas entradas de registro incluem um **código de erro** (por exemplo, `API02004`, `DB01001`). Estes códigos se vinculam diretamente à documentação de [Códigos de Erro](/error-codes), que fornece:

- Explicação detalhada do erro
- Causas comuns
- Etapas de solução de problemas

## Entradas de Registro Comuns

### Operações de Sincronização

```
[info] Buscou todos os IDs para produtos    {...} 3 itens
[info] Sincronizou novos clientes            {...} 1 item
[info] Buscou todos os IDs para impostos       {...} 3 itens
```

Estes indicam atividade normal de sincronização. O contexto mostra quantos itens foram sincronizados.

### Erros de Permissão

```
[error] Desculpe, você não pode listar recursos    {...} API02004
```

Isso geralmente significa que o usuário não tem permissão para acessar um recurso. Verifique as funções de usuário em `Admin do WP > POS > Configurações > Acesso`.

### Problemas de Sessão

```
[warn] Não foi possível atualizar a sessão    {...} API02007
```

O POS não conseguiu atualizar a sessão de autenticação. Pode ser necessário fazer logout e login novamente.

## Usando Registros para Suporte

Ao solicitar suporte:

1. **Reproduza o problema** - Realize a ação que causa o problema
2. **Verifique os registros** - Procure por erros (vermelho) ou avisos (laranja)
3. **Expanda o contexto** - Obtenha os detalhes completos do JSON
4. **Anote o código de erro** - Inclua isso em sua solicitação de suporte
5. **Copie as entradas relevantes** - Compartilhe as mensagens de registro com contexto

## Monitoramento de Desempenho

Os registros incluem métricas de tempo de execução e carga do servidor. Se você notar:

- **Tempos de execução altos** - O servidor pode estar sobrecarregado ou lento
- **Valores altos de carga do servidor** - Considere a otimização do servidor
- **Timeouts frequentes** - Verifique a conectividade de rede
