O WCPOS inclui um sistema completo de modelos para recibos, faturas, recibos de presente, romaneios, pedidos de cozinha e muito mais — tudo gerenciado em **WP Admin → POS → Modelos**.

:::tip Só precisa alterar o recibo?
Comece em **[Personalize seu recibo](/receipts/customise)**. A página explica as três formas mais simples de fazer isso — escolher outro modelo incluído, pedir à IA para ajustar um modelo ou editar manualmente.
:::

## Mecanismos de modelo {#template-engines}

O WCPOS oferece suporte a três mecanismos de modelo. O mecanismo é escolhido ao criar um modelo ou clicar em **Usar modelo** em um cartão da galeria, e **não pode ser alterado depois** — crie um novo modelo se quiser trocar de mecanismo.

<FeatureGrid>
  <FeatureCard icon="code" title="HTML sem lógica (recomendado)">
    Estilo Mustache <code>{"{{variable}}"}</code> espaços reservados dentro de HTML simples. Renderiza no lado do cliente, portanto os recibos **funcionam offline**. Use para impressão pelo navegador e recibos em PDF.
  </FeatureCard>
  <FeatureCard icon="print" title="XML térmico">
    Modelos XML que produzem tanto uma prévia na tela quanto **comandos ESC/POS** para impressoras térmicas de recibos. O mesmo modelo, duas saídas. Use para Epson, Star e outras impressoras térmicas.
  </FeatureCard>
  <FeatureCard icon="terminal" title="PHP legado">
    Modelos PHP no lado do servidor usando funções do WooCommerce. Mantidos para compatibilidade retroativa com os <code>yourtheme/woocommerce-pos/receipt.php</code> existentes. Requer uma conexão com o servidor para renderizar.
  </FeatureCard>
</FeatureGrid>

## Galeria de modelos {#template-gallery}

A galeria inclui modelos prontos que podem ser usados como estão ou personalizados. Filtre por **categoria** (Recibo, Fatura, Recibo de presente, Comanda de cozinha, Orçamento / Pedido de compra), **formato** (HTML / ESC/POS) e **direção** (LTR / RTL); visualize uma prévia com os dados reais da sua loja. O rodapé de cada cartão tem um link **Prévia** e um botão principal **Usar modelo** — clicar em Usar modelo cria uma cópia editável em *Seus modelos*.

| Modelo | Mecanismo | Descrição |
|----------|--------|-------------|
| Recibo padrão | HTML | Recibo padrão — logotipo, identidade da loja, itens, totais, pagamento |
| Recibo padrão (RTL) | HTML | Contraparte da direita para a esquerda para árabe, hebraico, persa, urdu |
| Minimalista / Moderno | HTML | Os mesmos elementos essenciais do Padrão, compactados em menos espaço vertical |
| Recibo detalhado | HTML | Fatura fiscal completa — coluna de SKU, preço unitário, detalhamento de impostos por alíquota, endereços |
| Recibo de presente | HTML | Apenas itens, preços ocultos. Inclui mensagem de presente e política de devolução |

### Modelos de documentos de terceiros {#third-party-document-templates}

Se **PDF Invoices & Packing Slips for WooCommerce** da WP Overnight estiver ativo, o WCPOS também exibirá **Fatura (WP Overnight)** e **Romaneio de embalagem (WP Overnight)** na lista de modelos de recibo. Esses modelos HTML renderizados no servidor delegam à API de documentos da WP Overnight, reutilizando a numeração, o layout, a identidade visual, os campos legais/fiscais e as personalizações de modelo de faturas e romaneios de embalagem configurados nesse plugin.

Eles ficam disponíveis apenas enquanto o plugin WP Overnight estiver ativo e exigem uma conexão com o servidor para renderizar. Use os modelos HTML ou térmicos incluídos no WCPOS quando for necessário imprimir offline.

### Exibição adaptativa de impostos {#adaptive-tax-display}

A maioria dos modelos incluídos (Padrão, Padrão RTL, Minimalista/Moderno, Estreito, Fatura, Orçamento, variantes Térmico simples) **se adapta automaticamente à configuração de exibição de impostos do WooCommerce**:

- **Lojas com impostos inclusos** (UE/Reino Unido/Austrália) exibem preços brutos com uma linha "Imposto incluído".
- **Lojas com impostos não inclusos** (EUA/Canadá) exibem preços líquidos com o imposto adicionado em uma linha separada.
- O total geral sempre mostra o valor bruto efetivamente cobrado.

A família **Detalhado** é uma fatura fiscal formal e sempre discrimina os impostos com linhas sem impostos inclusos mais um detalhamento explícito, independentemente da configuração da loja.

## Como funciona {#how-it-works}

1. **Adicionar um modelo** — clique em **Usar modelo** em um cartão da galeria ou crie o seu do zero.
2. **Personalize-o** usando o editor no aplicativo com pré-visualização ao vivo, ou cole-o no ChatGPT / Claude e peça as alterações desejadas.
3. **Ative-o** — habilite o botão **Ativo** em *Seus modelos*. Todo modelo de recibo ativo aparece no menu suspenso da tela de recibo no caixa.
4. **Imprima ou exiba** — os recibos são abertos após a finalização da compra, com opções para imprimir, enviar por e-mail ou visualizar na tela.

Os modelos são renderizados com base em uma [carga de dados de recibo](/receipts/receipt-data) padronizada, com seções para informações da loja, itens de linha, totais, impostos, pagamentos, reembolsos e muito mais. Todos os campos de moeda incluem variantes `_display` pré-formatadas (por exemplo, `$29.99` em vez de `29.99`).

## Recursos Pro {#pro-features}

Com o [WCPOS Pro](/getting-started/pro-license), cada loja pode ter suas próprias atribuições de modelos, identidade visual (logotipo, endereço, dados de contato) e ordenação de modelos — útil para operações com várias unidades em que cada loja imprime seu próprio cabeçalho.
