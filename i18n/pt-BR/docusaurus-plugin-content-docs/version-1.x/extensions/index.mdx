O WCPOS oferece suporte a extensões que adicionam novas funcionalidades ao seu ponto de venda. O diretório de extensões permite navegar pelas extensões disponíveis, instalá-las diretamente nas configurações do POS e gerenciar atualizações.

:::info Recurso Pro
A instalação e o gerenciamento de extensões exigem o [WCPOS Pro](/getting-started/pro-license). A versão gratuita exibe o catálogo de extensões, mas desativa os controles de instalação e ativação.
:::

## Navegando pelas extensões {#browsing-extensions}

Abra o diretório de extensões em `POS Settings > Extensions` (também identificado como **Plugins** em algumas versões).

O diretório exibe uma grade de cartões com as extensões disponíveis. Cada cartão mostra:

- **Ícone** (ou uma peça de quebra-cabeça como alternativa, caso a extensão não forneça um)
- **Nome e versão**
- **Descrição**
- **Selo de categoria**
- **Status** — ativo, inativo, atualização disponível ou não instalado

### Filtros e Busca {#filtering-and-search}

Use os **botões de categoria** no topo para filtrar as extensões por categoria. Você também pode usar o **campo de busca** para encontrar extensões por nome, descrição ou tags.

## Instalando uma Extensão {#installing-an-extension}

1. Abra `POS Settings > Extensions`.
2. Encontre a extensão desejada e clique em **Instalar**.
3. A extensão é baixada e instalada usando o instalador de plugins do WordPress.
4. Após a instalação, clique em **Ativar** para habilitá-la.

Nos bastidores, o WCPOS utiliza o `Plugin_Upgrader` nativo do WordPress para realizar a instalação, portanto as extensões seguem o mesmo processo de qualquer plugin do WordPress.

## Ativando e Desativando {#activating-and-deactivating}

Cada extensão instalada possui os botões **Ativar** e **Desativar** em seu card.

- **Ativar** habilita a extensão para que ela possa ser executada no POS.
- **Desativar** a desabilita sem desinstalá-la. Os arquivos da extensão permanecem no servidor e podem ser reativados a qualquer momento.

## Atualizando extensões {#updating-extensions}

Quando uma nova versão de uma extensão instalada estiver disponível, o cartão exibirá um selo **Atualização disponível** e um botão **Atualizar**.

### Atualizações automáticas {#auto-updates}

As extensões instaladas pelo diretório vêm com **atualização automática habilitada por padrão**. Você pode ativar ou desativar a atualização automática de cada extensão no respectivo cartão do diretório. Quando a atualização automática está ativa, o WordPress aplica as novas versões automaticamente, da mesma forma que faz com os plugins com atualização automática habilitada.

## Solução de problemas {#troubleshooting}

### Mensagem "Requires Pro" nos botões {#requires-pro-message-on-buttons}

Os botões de instalar, ativar e atualizar ficam desabilitados na versão gratuita do WCPOS. Faça upgrade para o [WCPOS Pro](/getting-started/pro-license) para gerenciar extensões.

### A extensão não é instalada {#extension-fails-to-install}

- Verifique se o seu servidor WordPress tem permissões de escrita no diretório `wp-content/plugins`.
- Verifique se o seu servidor pode realizar requisições HTTPS de saída (alguns provedores bloqueiam downloads externos).
- Verifique os detalhes do erro em `WP Admin > POS > Support > Logs`.

### Extensão não aparece após a instalação {#extension-not-appearing-after-install}

- Atualize o POS — a lista de extensões fica em cache por até 12 horas.
- Confirme se a extensão está ativada (extensões instaladas mas inativas não serão executadas).

### O catálogo não carrega {#catalog-not-loading}

O catálogo de extensões é obtido de uma fonte remota e armazenado em cache localmente por 12 horas. Se o catálogo não carregar:

- Verifique a conectividade de internet do seu servidor.
- Tente novamente após o cache expirar ou limpe o cache de transients do seu servidor.

## Para desenvolvedores {#for-developers}

### Como criar uma extensão para o POS {#creating-a-pos-extension}

Uma extensão do WCPOS é um plugin padrão do WordPress que se integra ao POS por meio de hooks e APIs do WCPOS. Para criar uma:

1. **Comece com um plugin do WordPress.** Sua extensão precisa de um cabeçalho de plugin padrão e um arquivo de entrada, assim como qualquer plugin do WooCommerce ou WordPress.
2. **Integre-se ao WCPOS.** Use os hooks e filtros fornecidos pelo WCPOS para adicionar funcionalidades à interface do POS ou ao backend.
3. **Hospede as versões no GitHub.** O diretório de extensões usa GitHub Releases para controlar versões e entregar atualizações.

### Envio para o diretório {#submitting-to-the-directory}

O catálogo de extensões é mantido no repositório do GitHub [`wcpos/extensions`](https://github.com/wcpos/extensions). Para listar sua extensão:

1. Consulte o formato do catálogo e os requisitos de metadados no README do repositório.
2. Abra um pull request para adicionar os metadados da sua extensão ao `catalog.json`.
3. Após o merge, sua extensão aparecerá no diretório para todos os usuários do WCPOS Pro.

### Convenções de releases no GitHub {#github-release-conventions}

O ciclo de vida de atualizações depende dos GitHub Releases:

- **Marque as versões** usando versionamento semântico (por exemplo, `v1.0.0`, `v1.2.3`).
- **Anexe o zip do plugin** como um asset da release — este é o arquivo baixado quando um usuário instala ou atualiza.
- **Publique a release** (não como rascunho) para que o diretório possa detectá-la.

Quando você publicar uma nova release, os usuários com sua extensão instalada verão a atualização disponível no diretório de extensões. Se a atualização automática estiver ativada, ela será aplicada automaticamente.

Para detalhes completos sobre o esquema do catálogo e o processo de submissão, consulte o repositório [`wcpos/extensions`](https://github.com/wcpos/extensions).
