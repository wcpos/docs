Alguns países exigem legalmente que o software de ponto de venda seja **certificado** contra fraude fiscal ou integrado a uma autoridade fiscal nacional. Esta página resume a posição do WCPOS para que você possa verificar se ele se adapta à sua jurisdição **antes** de confiar nele.

:::warning WCPOS não é um POS fiscal certificado
WCPOS **não** atualmente possui certificações fiscais para nenhum mercado regulamentado. Se o seu país exigir software POS certificado, confirme os requisitos locais antes de usar o WCPOS para vendas ao vivo. WCPOS atua como middleware para WooCommerce – o faturamento e o VAT são tratados no nível WooCommerce.
:::

:::info Estamos trabalhando nisso
A conformidade fiscal é uma área ativa e contínua de desenvolvimento — nosso objetivo é tornar compatível com WCPOS em todos os países onde nossos usuários operam. O status abaixo reflete **hoje**, e não para onde estamos indo. Se a conformidade na sua região for importante para você, informe-nos na [comunidade](https://wcpos.com/discord) — isso nos ajuda a priorizar quais mercados abordar em seguida.
:::

## Como WCPOS lida com impostos {#how-wcpos-handles-tax}

WCPOS calcula o imposto localmente no aplicativo usando taxas baixadas de WooCommerce e, em seguida, cria o pedido em WooCommerce. Isso é rápido e funciona offline, mas significa que a lógica fiscal certificada/do lado do servidor (como alguns plug-ins de conformidade) só é executada quando o pedido chega ao servidor. A ação **'Salvar no servidor'** e os gateways **Web Checkout** / **Email Invoice** enviam o pedido por meio do checkout padrão WooCommerce, onde se aplicam regras fiscais e de conformidade do lado do servidor.

## Situação do país {#country-status}

### França — NF525 {#france-nf525}

WCPOS **não é certificado** para NF525. A lei francesa exige que os sistemas POS sejam certificados contra fraude, o que afeta a viabilidade das vendas em França. Como WCPOS é um middleware para WooCommerce, o faturamento e a conformidade com VAT são tratados no nível WooCommerce — consulte as orientações do WooCommerce sobre [leis francesas de prevenção de fraude VAT](https://woocommerce.com/posts/french-vat-fraud-prevention-laws-how-to-certify-invoices-with-woocommerce/).

### Espanha — VERIFACTU {#spain-verifactu}

WCPOS **não é certificado** para VERIFACTU. A Espanha exige que o software POS seja certificado; o prazo VERIFACTU foi **adiado para 2027**. A equipe está ciente do requisito, mas nenhum cronograma de certificação foi comprometido.

### Noruega – Kassasystemloven {#norway-kassasystemloven}

WCPOS **não é certificado** pela Lei Norueguesa de Caixas Registradoras (_Kassasystemloven_) e nenhuma declaração de fornecedor está disponível. Alguns comerciantes na Noruega usam o POS, mas você deve confirmar suas próprias obrigações legais.

### Chile – SII {#chile-sii}

**Não há integração** com o Servicio de Impuestos Internos (SII) do Chile para recibos de vendas validados (_boletas_). Os modelos de recibo são modelos WooCommerce padrão e personalizáveis, mas nenhuma conformidade com SII está integrada.

### UE — Balcão Único (OSS) {#eu-oss}

O POS **não** suporta nativamente o esquema VAT do EU One Stop Shop (OSS) para vendas internacionais, porque calcula impostos localmente a partir de taxas baixadas.

**Soluções alternativas:**
- Use **Salvar no servidor** antes da finalização da compra para acionar um recálculo de impostos no servidor por meio do plug-in OSS ou
- Use o gateway **Web Checkout** ou **Email Invoice** para receber o pagamento por meio do checkout WooCommerce padrão, onde as regras OSS se aplicam.

## Outros mercados regulamentados {#other-regulated-markets}

Para qualquer jurisdição que exija software POS certificado, verifique primeiro os requisitos locais. WCPOS atualmente não possui certificações para mercados regulamentados. Se o status da certificação for importante para sua decisão de compra, pergunte na [comunidade](https://wcpos.com/discord) antes de se comprometer.
