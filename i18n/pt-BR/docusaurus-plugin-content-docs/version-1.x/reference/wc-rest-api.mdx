A **REST API do WooCommerce** é como um conjunto de "canais" padronizados que permite aos proprietários de lojas conectar sua loja WooCommerce a outros aplicativos e serviços.

Aqui está uma explicação simples de como funciona:

1. Atua como uma ponte entre sua loja WooCommerce e aplicativos externos, permitindo que eles se comuniquem e compartilhem dados.

2. A API funciona enviando solicitações para **endpoints** específicos - pense neles como "canais" para diferentes tipos de dados da loja. Por exemplo, existem endpoints para produtos, pedidos e clientes.

3. Aplicativos externos podem usar esses endpoints para:

- Obter informações (como buscar uma lista de produtos)
- Adicionar novos dados (como criar um novo pedido)
- Atualizar informações existentes (como alterar o preço de um produto)
- Excluir dados (como remover um produto desatualizado)

Em vez de depender de hooks e filtros PHP (como muitos plugins tradicionais do WordPress), funciona enviando e recebendo dados em um formato estruturado e previsível, conhecido como **JSON**.

## Por que alguns plugins não funcionam no PDV?

O WCPOS faz o download dos dados de produtos, pedidos e clientes da sua loja WooCommerce através da REST API.
Em seguida, mantém uma cópia local desses dados para poder continuar funcionando mesmo com uma conexão de internet instável.
Essa abordagem torna o PDV mais rápido e confiável em um ambiente de checkout movimentado.

No entanto, isso significa que nosso PDV não "vê" alterações que acontecem dinamicamente em seu site em tempo real. Por exemplo:
