:::caution
Este artigo é destinado a desenvolvedores web, a depuração não é algo que o usuário médio deva precisar usar.
:::

WCPOS é essencialmente um aplicativo web JavaScript - e como tal - a maioria da depuração é feita através do [console do navegador](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors).
Se a depuração estiver ativada, o WCPOS registrará várias mensagens no console que podem ajudar a diagnosticar problemas.

![Um Console JavaScript aberto no Chrome para Mac](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Um Console JavaScript aberto no Chrome para Mac")

### Ativar depuração usando wp-config.php

Definindo o sinalizador [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) no arquivo wp-config.php, você pode ativar a depuração do WCPOS.
Isso carregará os arquivos de script não minimificados e também habilitará as mensagens de depuração no console do navegador.

### Ativar depuração usando o console do navegador

Se você não tiver acesso ao arquivo wp-config.php, pode habilitar mensagens de depuração adicionando um sinalizador `debug` no Armazenamento Local.
Isso pode ser feito digitando `localStorage.setItem('debug', '*')` no console.
Agora atualize a página e a depuração estará ativada. O registro de console pode consumir recursos do navegador, então você provavelmente vai querer desativar a depuração quando terminar.
Para desativar a depuração, simplesmente digite `localStorage.removeItem('debug')` no console e atualize a página.
