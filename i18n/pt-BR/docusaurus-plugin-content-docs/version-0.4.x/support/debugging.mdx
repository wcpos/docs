:::caution
Este artigo é destinado a desenvolvedores web, a depuração não é algo que o usuário médio deve precisar usar.
:::

WCPOS é em grande parte uma aplicação web em JavaScript - e, como tal - a maior parte da depuração é feita através do [console do navegador](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors).
Se a depuração estiver habilitada, o WCPOS registrará várias mensagens no console que podem ajudar a diagnosticar problemas.

![Um console JavaScript aberto no Chrome para Mac](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Um console JavaScript aberto no Chrome para Mac")

### Ativar depuração usando wp-config.php {#turn-on-debugging-using-wp-configphp}

Definindo a flag [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) no seu arquivo wp-config.php, você pode habilitar a depuração do WCPOS.
Isso carregará os arquivos de script não minificados e também ativará mensagens de depuração no console do navegador.

### Ativar depuração usando o console do navegador {#turn-on-debugging-using-the-browser-console}

Se você não tiver acesso ao arquivo wp-config.php, pode habilitar mensagens de depuração adicionando uma flag `debug` ao Armazenamento Local.
Isso pode ser feito digitando `localStorage.setItem('debug', '*')` no console.
Agora atualize a página e a depuração estará ativada. O registro no console pode consumir recursos do navegador, então você provavelmente desejará desativar a depuração assim que terminar com ela.
Para desativar a depuração, basta digitar `localStorage.removeItem('debug')` no console e atualizar a página.
