---
title: Desempenho do PDV
---

WCPOS é um aplicativo web JavaScript de página única que se comunica com seu servidor através da [API REST do WooCommerce](http://woothemes.github.io/woocommerce-rest-api-docs/). Os problemas de desempenho podem ser divididos em duas categorias: lado do servidor e lado do cliente.

### Desempenho do Lado do Servidor

O desempenho do lado do servidor refere-se a ações como baixar uma página de produtos (10 produtos) ou processar um pedido. Problemas que podem afetar a velocidade dessas ações incluem:

- A velocidade da sua conexão de internet
- A velocidade do seu servidor, ou seja: poder de processamento, RAM, carga do servidor etc.
- Quantidade de plugins ativos
- Processamento do gateway de pagamento

O [site de demonstração](http://demo.wcpos.com/pos) representa um cenário de 'melhor caso'. O site de demonstração usa [um provedor de hospedagem muito bom](http://wcpos.com/wpe) e tem um número muito pequeno de plugins ativados.

| Processo                  | Velocidades médias para [demo.wcpos.com/pos](http://demo.wcpos.com/pos)<sup>\*</sup> |
| ------------------------ | ------------------------------------------------------------------------------------ |
| Buscando 10 produtos     | 1 - 2 segundos                                                                       |
| Processando venda em dinheiro   | 1 - 2 segundos                                                                   |
| Processando venda no Stripe | 2 - 3 segundos                                                                     |

<small>
  <sup>*</sup> tempos médios para uma conexão de banda larga. Os tempos podem variar
  dependendo da sua velocidade de internet.
</small>

Se seu PDV estiver levando significativamente mais tempo do que os tempos acima, você pode realizar os seguintes testes:

1.  Altere o tema para o tema padrão Twenty Fifteen do WordPress
2.  Desative todos os plugins, exceto WooCommerce e WCPOS
3.  [Limpe os dados de armazenamento local](http://faq.wcpos.com/en/clear-local-data.html)
4.  Agora, use o PDV para ver se há um aumento de desempenho
5.  **Se houver um aumento de desempenho:** reative seu tema e plugins um por um para ver qual plugin está impactando o desempenho
6.  **Se não houver aumento de desempenho:** você pode querer mudar para um provedor de hospedagem melhor ou investir em uma conexão de internet mais rápida

### Desempenho do Lado do Cliente

WCPOS usa JavaScript, HTML e CSS para exibir os dados recuperados da API REST do WooCommerce, ou seja, [produtos](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) e [pedidos](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order).
Para melhorar o desempenho do lado do cliente, o PDV armazena os dados no navegador usando [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API).
Por exemplo, quando um produto é buscado pela primeira vez, uma solicitação será enviada ao servidor; uma vez que os dados do produto sejam baixados, eles serão armazenados localmente para que as buscas subsequentes sejam instantâneas.

![Um exemplo de produto armazenado localmente.](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "Um exemplo de produtos sendo armazenados localmente")

Os dados do IndexedDB persistem mesmo quando você fecha o navegador ou reinicia o computador.
Se você notar que seus dados de produto estão dessincronizados por qualquer motivo, você pode [limpar o armazenamento local](http://faq.wcpos.com/en/clear-local-data.html) e buscar um novo conjunto de dados do servidor.
