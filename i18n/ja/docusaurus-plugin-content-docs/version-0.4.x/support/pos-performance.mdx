---
title: POSのパフォーマンス
---

WCPOSは、[WooCommerce REST API](http://woothemes.github.io/woocommerce-rest-api-docs/)を介してサーバーと通信するシングルページのJavaScriptウェブアプリケーションです。パフォーマンスの問題は、サーバーサイドとクライアントサイドの2つのカテゴリに分けることができます。

### サーバーサイドのパフォーマンス

サーバーサイドのパフォーマンスとは、製品のページ（10製品）をダウンロードしたり、注文を処理したりするなどのアクションを指します。そのようなアクションの速度に影響を与える可能性のある問題は以下の通りです：

- インターネット接続の速度
- サーバーの速度（処理能力、RAM、サーバー負荷など）
- アクティブなプラグインの数
- 決済ゲートウェイの処理

[デモサイト](http://demo.wcpos.com/pos)は「最良の」シナリオを表しています。デモサイトは[非常に優れたウェブホスト](http://wcpos.com/wpe)を使用し、アクティブなプラグインの数が非常に少ないです。

| プロセス                 | [demo.wcpos.com/pos](http://demo.wcpos.com/pos)の平均速度<sup>\*</sup> |
| ------------------------ | ----------------------------------------------------------------------- |
| 10製品の取得            | 1 - 2秒                                                               |
| 現金売上の処理           | 1 - 2秒                                                               |
| Stripe売上の処理         | 2 - 3秒                                                               |

<small>
  <sup>*</sup> ブロードバンド接続の平均時間。インターネット速度によって変動する可能性があります。
</small>

上記の時間よりも明らかに長い場合は、以下のテストを実施することをお勧めします：

1.  WordPressのデフォルトテーマ Twenty Fifteenに切り替える
2.  WooCommerceとWCPOSを除くすべてのプラグインを無効にする
3.  [ローカルストレージデータをクリア](http://faq.wcpos.com/en/clear-local-data.html)
4.  POSを使用してパフォーマンスの改善を確認する
5.  **パフォーマンスが改善された場合：** テーマとプラグインを1つずつ再有効化し、パフォーマンスに影響を与えているプラグインを特定する
6.  **パフォーマンスが改善されない場合：** より良いウェブホストに移行するか、より高速なインターネット接続に投資することを検討する

### クライアントサイドのパフォーマンス

WCPOSは、WooCommerce REST APIから取得したデータ（[製品](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product)や[注文](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order)）を表示するために、JavaScriptとHTML、CSSを使用します。
クライアントサイドのパフォーマンスを向上させるため、POSは[IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API)を使用してブラウザにデータを保存します。
例えば、製品を初めて取得する際にサーバーにリクエストが送信され、製品データがダウンロードされると、ローカルに保存されるため、subsequent検索が瞬時に行われます。

![ローカルに保存された製品の例。](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "ローカルに保存された製品の例")

IndexedDBデータは、ブラウザを閉じたり、コンピューターを再起動しても保持されます。
何らかの理由で製品データが同期していないことに気付いた場合は、[ローカルストレージをクリア](http://faq.wcpos.com/en/clear-local-data.html)し、サーバーから新しいデータセットを取得できます。
