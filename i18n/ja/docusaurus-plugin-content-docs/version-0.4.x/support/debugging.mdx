:::caution
この記事はWeb開発者向けです。デバッグは一般的なユーザーが使用する必要のあるものではありません。
:::

WCPOSは主にJavaScriptウェブアプリケーションであり、ほとんどのデバッグは[ブラウザコンソール](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors)を通じて行われます。
デバッグが有効になっている場合、WCPOSはコンソールにさまざまなメッセージをログ出力し、問題の診断に役立つ可能性があります。

![MacのChromeで開かれたJavaScriptコンソール](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "MacのChromeで開かれたJavaScriptコンソール")

### wp-config.phpを使用してデバッグを有効にする

wp-config.phpファイルで[SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG)フラグを設定することで、WCPOSのデバッグを有効にできます。
これにより、縮小されていないスクリプトファイルが読み込まれ、ブラウザコンソールでデバッグメッセージが有効になります。

### ブラウザコンソールを使用してデバッグを有効にする

wp-config.phpファイルにアクセスできない場合、ローカルストレージに`debug`フラグを追加することでデバッグメッセージを有効にできます。
これは、コンソールに`localStorage.setItem('debug', '*')`と入力することで行えます。
ページを更新すると、デバッグがオンになります。コンソールログはブラウザリソースを消費するため、終了したらデバッグをオフにすることをお勧めします。
デバッグをオフにするには、コンソールに`localStorage.removeItem('debug')`と入力し、ページを更新するだけです。
