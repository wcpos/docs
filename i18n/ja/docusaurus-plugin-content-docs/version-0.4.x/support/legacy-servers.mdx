---
title: レガシーサーバー
description: ウェブホストが最新のRESTful HTTPメソッドをブロックしている場合、レガシーサーバーサポートを有効にする必要があります。
---

HTTP（およびHTTPS）は、すべてのブラウザがウェブアプリケーションと通信するために使用するプロトコルです。
プロトコルの重要な部分は[HTTPメソッド](https://tools.ietf.org/html/rfc2616#section-5.1.1)です。
従来、ウェブアプリケーションはGET（ウェブページを取得）やPOST（ウェブフォームを送信）などの非常に限られたHTTPメソッドのセットを使用していました。

WCPOSのような最新のウェブアプリケーションは、次のHTTPメソッドを使用したREST/HTTPアプローチを採用しています。

- GET
- POST
- PUT
- PATCH
- DELETE

まれに、ウェブホストがこれらのHTTPメソッドの一部をブロックすることがあります。
ウェブホストが最新のRESTful HTTPメソッドをブロックしている場合、レガシーサーバーサポートを有効にする必要があります。
