:::info プロ機能
レポート画面には[WCPOS Pro](/getting-started/pro-license)が必要です。
:::

レポート画面は、売上レポートと日次精算ツールを提供します。主に**日々の終了時レポート**を作成し、レジ担当者や管理者が1日の締めを行うために設計されています。

{/* Screenshot: reports-overview.png */}

:::tip 包括的な分析
任意の日付範囲の詳細な分析には、**WP Admin > 分析**を使用してください。POSレポート画面は、迅速な日次サマリーと日次精算に最適化されています。
:::

## インターフェース概要

### フィルター

画面上部：

- **状態フィルター** - 注文状態によるフィルタリング（完了、処理中など）
- **顧客フィルター** - 顧客によるフィルタリング
- **レジ担当者フィルター** - 注文処理者によるフィルタリング
- **店舗フィルター** - 店舗によるフィルタリング（複数店舗設定）
- **日付範囲** - 報告期間の選択（今日、今週、カスタム）

### 3つのパネルレイアウト

レポート画面は3つのエリアに分かれています：

#### 1. 売上チャート（上部）

時系列の売上をバーチャートで表示：

- 日次レポートの時間単位の内訳
- 売上パターンの視覚的表現
- ピーク販売時間の特定に役立つ

#### 2. 注文テーブル（左下）

レポートに含まれる注文のリスト：

- **チェックボックス** - 注文の選択/非選択
- **状態** - 注文状態アイコン
- **注文番号** - ユニークな注文ID
- **作成日** - 注文が行われた日時
- **支払日** - 支払いが受領された日時
- **合計** - 注文金額

表示設定アイコンをクリックして列をカスタマイズ。

#### 3. レポートサマリー（右下）

計算されたレポート合計：

- **レポートタイプ**ドロップダウン（デフォルト/オフライン）
- **店舗情報**（名前、ID）
- **レポート期間**（開始/終了タイムスタンプ）
- **レジ担当者**（フィルターされている場合）

**売上サマリー：**
- 総注文数
- 総売上純額
- 総税額
- **総売上**
- 総割引額

**支払い方法：**
- 支払いタイプ別の内訳（現金、カードなど）

**印刷**ボタンで物理的なレポートを生成。

## レポートタイプ

### デフォルト（オフライン）

デフォルトのレポートは、ローカルに保存されたデータから生成されます。これは：

- サーバー接続なしで動作
- 高速なレポート生成
- デバイスに既に同期されているデータを使用

すべての関連注文が既に同期されている場合の、迅速な日次サマリーに最適です。

## 日次精算ワークフロー

### 典型的な日次精算

1. **日付範囲を設定** - 「今日」または特定の日付を選択
2. **レジ担当者でフィルタリング**（オプション） - 個々のレジ担当者レポート用
3. **注文を確認** - 注文テーブルの正確性を確認
4. **合計を検証** - 実際の現金カウントとカードターミナルと比較
5. **レポートを印刷** - 物理的な記録を生成

### 現金精算

1. 現金支払いのみを表示するようフィルタリング
2. レポートからの総現金額を確認
3. レジの現金を数える
4. 合計を比較（一致するはず）
5. 不一致を文書化

### カード精算

1. カード支払いのみを表示するようフィルタリング
2. レポートからの総カード金額を確認
3. カードターミナルのバッチ総額と比較
4. バッチ決済前に金額が一致することを確認

## レポートの印刷

**印刷**ボタンをクリックして印刷可能なレポートを生成。以下を含む：

- 店舗情報
- レポート期間
- 売上サマリー
- 支払い方法の内訳
- 注文詳細（選択された場合）
