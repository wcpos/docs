:::info Pro 機能
マルチストア機能を使用するには [WCPOS Pro](/getting-started/pro-license) が必要です。
:::

WCPOS Pro を使用すると、単一の WooCommerce インストール内で複数の店舗拠点を作成できます。各店舗には個別の住所、ロゴ、設定を割り当てられるため、複数の実店舗を運営する場合に便利です。

## 店舗で利用できる機能 {#what-stores-provide}

各店舗には以下を設定できます。

- **店舗名と住所** — レシートやレポートに使用されます
- **店舗ロゴ** — 印刷されるレシートに表示されます
- **税率** — 拠点ごとに異なる税要件に対応できます
- **レジ担当者の割り当て** — どのレジ担当者がどの店舗で勤務しているかを管理

## 店舗のセットアップ {#setting-up-stores}

1. `WP Admin > POS > Settings` を開きます
2. 店舗名、住所、ロゴを入力して新しい店舗を作成します
3. 店舗ごとの税設定を構成します
4. 店舗を POS ユーザーに割り当てます

レジ担当者が POS にログインすると、どの店舗で業務を行うかを選択します。これにより、レシートに表示される店舗情報や、レポートに適用される店舗フィルターが決定されます。

## 店舗別フィルタリング {#filtering-by-store}

複数の店舗を設定すると、POS 全体で店舗フィルターが利用できるようになります:

- **[注文](/orders)** — 注文履歴を店舗ごとに絞り込み
- **[レポート](/reports)** — 特定の店舗の日次レポートを生成
- **[レシート](/pos/checkout/receipts)** — レシートには該当する店舗の住所とロゴが自動的に表示されます

## 店舗別価格設定 {#store-specific-pricing}

WCPOS Pro は**店舗ごとに異なる価格**に対応しています。商品およびバリエーションのいずれについても、店舗ごとに通常価格とセール価格を個別に設定できます。

レジ担当者が特定の店舗で操作している場合、POS は自動的にその店舗の価格を使用します。店舗別価格が設定されていない場合は、WooCommerce のデフォルト価格が使用されます。

## 在庫管理 {#inventory-management}

デフォルトでは、すべての店舗が同じ WooCommerce の在庫数を共有します。拠点ごとの在庫管理を行うには、WCPOS Pro が [ATUM Inventory Management](https://www.atum.org/) プラグインと連携します:

1. WooCommerce サイトに ATUM プラグインをインストールして設定します
2. ATUM で在庫拠点を作成します
3. WCPOS Pro で、各店舗を ATUM の在庫拠点に関連付けます
4. 注文が行われると、該当する拠点の在庫数が更新されます

### 代替案: WordPress マルチサイト {#alternative-wordpress-multisite}

ATUM を使用せずに在庫を完全に分離したい場合は、[WordPress マルチサイト](https://developer.wordpress.org/advanced-administration/multisite/create-network/) を利用できます:

1. 店舗ごとに個別の WordPress サイトを作成します
2. 各サイトに WooCommerce と WCPOS をインストールします
3. 各サイトは独立した商品カタログと在庫数を保持します

この方法では完全な分離が実現できますが、複数の WooCommerce インストールを管理する必要があります。

## 既知の制限事項 {#known-limitations}

- **単一の WooCommerce インスタンス** — 店舗は別個のショップではなく、1 つの WooCommerce インストール内のロケーションとして扱われます
- **店舗ごとの商品表示制御は不可** — すべての店舗で同じ商品カタログが表示されます
