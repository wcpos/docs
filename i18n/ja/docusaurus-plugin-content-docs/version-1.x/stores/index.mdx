:::info Pro機能
マルチ店舗機能には[WCPOS Pro](/getting-started/pro-license)が必要です。
:::

WCPOS Proでは、単一のWooCommerceインストール内に個別の店舗拠点を作成できます。各店舗には独自の住所、ロゴ、設定を持たせることができ、複数の実店舗を運営している場合に便利です。

## 店舗で利用できる機能 {#what-stores-provide}

各店舗には以下を設定できます：

- **店舗名と住所** — レシートやレポートに使用されます
- **店舗ロゴ** — 印刷されるレシートに表示されます
- **税率** — 拠点ごとに異なる税要件に対応できます
- **レジ担当者の割り当て** — どのレジ担当者がどの店舗で勤務するかを管理

## 店舗の設定 {#setting-up-stores}

Proを有効にすると、WP AdminのPOSメニュー配下に**店舗**サブメニューが追加されます。そこから各店舗を作成し、住所、ロゴ、税務ID、サインインできるレジ担当者を設定できます。詳細な手順については**[店舗の設定](/stores/setup)**を参照してください。

レジ担当者がPOSにログインすると、操作する店舗を選択します。これにより、レシートに表示される店舗情報と、レポートに適用される店舗フィルターが決まります。

## 店舗別フィルタリング {#filtering-by-store}

複数の店舗を設定すると、POS全体に店舗フィルターが表示されます：

- **[注文](/orders)** — 店舗ごとに注文履歴をフィルタリング
- **[レポート](/reports)** — 特定の店舗の日次レポートを生成
- **[レシート](/receipts/at-checkout)** — レシートに正しい店舗の住所とロゴが自動的に表示されます

## 店舗別価格設定 {#store-specific-pricing}

WCPOS Proは**店舗ごとに異なる価格設定**に対応しています。商品とバリエーションの両方で、店舗ごとに通常価格とセール価格を個別に設定できます。

レジ担当者が特定の店舗で操作している場合、POSはその店舗の価格を自動的に使用します。店舗別価格が設定されていない場合は、WooCommerceのデフォルト価格が使用されます。

## 在庫管理 {#inventory-management}

デフォルトでは、すべての店舗が同じWooCommerceの在庫数を共有します。店舗ごとの在庫追跡を行うには、WCPOS Proが[ATUM Inventory Management](https://www.atum.org/)プラグインと連携します：

1. WooCommerceサイトにATUMプラグインをインストールして設定します
2. ATUMで在庫ロケーションを作成します
3. WCPOS Proで各店舗をATUMの在庫ロケーションに関連付けます
4. 注文が行われると、該当するロケーションの在庫数が更新されます

### 代替方法：WordPressマルチサイト {#alternative-wordpress-multisite}

ATUMを使用せずに在庫を完全に分離するには、[WordPressマルチサイト](https://developer.wordpress.org/advanced-administration/multisite/create-network/)を利用できます：

1. 店舗ごとに個別のWordPressサイトを作成します
2. 各サイトにWooCommerceとWCPOSをインストールします
3. 各サイトが独立した商品カタログと在庫数を管理します

この方法では完全な分離が実現できますが、複数のWooCommerceインストールを管理する必要があります。

## 既知の制限事項 {#known-limitations}

- **単一の WooCommerce インスタンス** — 店舗は1つの WooCommerce インストール内のロケーションであり、個別のショップではありません
- **店舗ごとの商品表示設定なし** — すべての店舗で同じ商品カタログが表示されます
