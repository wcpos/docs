:::info Pro機能
マルチ店舗機能には[WCPOS Pro](/getting-started/pro-license)が必要です。Proがない場合、WCPOSはWooCommerceの一般設定から構成された単一の仮想店舗として動作し、店舗画面は表示されません。
:::

このページでは、店舗の作成手順を最初から最後まで説明します。概念的な概要（マルチ店舗の機能、店舗ごとの価格設定、ATUM連携）については、[マルチ店舗](/stores)をご覧ください。

## 始める前に {#before-you-start}

- **WCPOS Proがインストールされ、ライセンスが有効化されていること。** Proを有効化すると、WP AdminのPOS内に**店舗**サブメニューが追加されます。
- **この店舗で働くキャッシャーがWordPressアカウントを持っていること。** *Access WooCommerce POS* 権限を含むロールが割り当てられている必要があります。まだ設定していない場合は、[アクセス設定](/settings/wp-admin/access)を参照してください。
- *(任意)* **アップロード用のロゴファイル** — PNGまたはJPG、300×300以上を推奨。

## 店舗を作成する {#create-the-store}

<Steps>
  <StepCard number="1" title="店舗一覧を開く">

WP Admin で **POS → 店舗** に移動します。既存の店舗一覧が表示され、**店舗**、**住所**、**認証ユーザー**の列と、各行に **POS を開く**ボタンがあります。

初めての店舗で Pro を有効化したばかりの場合、一覧は空になっています。ページヘッダーの**新規店舗を追加**をクリックしてください。

  </StepCard>
  <StepCard number="2" title="店舗名を設定する">

エディター上部で**店舗名**を設定します。この名前はレシート、ログイン時の店舗選択、POS 全体の店舗フィルターに表示されます。必須項目です。

例：*「本通りカフェ」*、*「倉庫 — ピックアップ」*、*「マーケット出店 #3」*

  </StepCard>
  <StepCard number="3" title="メインカラムを入力する">

エディター左側には、レシートに印刷される情報がまとめられています。

- **店舗ロゴ** — WP メディアライブラリからアップロードします。サイトにカスタマイザーロゴが設定されている場合、フォールバックとして*「店舗ロゴ未設定時にサイトロゴを使用」*トグルを利用できます。
- **住所** — 住所1、住所2、市区町村、都道府県、郵便番号、国。国は入力に応じて候補が絞り込まれるドロップダウンです。
- **連絡先** — Web アドレス、電話番号、メールアドレス。
- **レシートメッセージ** — *結びの挨拶*、*返品ポリシー*、*フッター*。プレーンテキストで、簡単な HTML が使用できます（`<strong>`、 `<em>`、 `<b>`、 `<i>`、 `<br>`、 `<p>`）。
- **レシートテンプレート** — *「店舗固有のレシートテンプレート」*トグルを**オフ**のままにすると、**POS → テンプレート**のサイト全体のテンプレートリストが継承されます。**オン**にすると、この店舗用のカスタムセットを選択できます。カスタムセットを選択すると、ドラッグハンドルによる並べ替えが可能になります。
- **レシートプレビュー** — テンプレートセレクターとサンプルデータ／注文の切り替えを備えたライブプレビューで、保存前にレシートの実際の表示を確認できます。

</StepCard>
  <StepCard number="4" title="サイドバーの設定">

右カラムは、レシートには表示されない店舗ごとの設定です：

- **認可ユーザー** — *Access WooCommerce POS* 権限を持つロールのすべてのWordPressユーザーのチェックボックスリストです。この店舗にサインインできるレジ担当者にチェックを入れてください。**リストを空のままにすると、POS権限を持つすべてのユーザーが認可されます** — 単一店舗のサイトに便利です。
- **税率** — *税金の計算基準*ドロップダウン：**この店舗の住所**（*デフォルト*）または**基本所在地**。下の表には、選択した住所に対する税クラスごとの適用税率が表示されます。この表は読み取り専用のため、実際の税率は**WooCommerce → 設定 → 税**で設定してください。
- **税ID** — 店舗ごとのVAT、ABN、GSTINなどを登録するための（種類 / 値 / 国 / ラベル）の繰り返し行です。空のままにすると、サイト全体の[店舗税ID](/settings/wp-admin/store-tax-ids)にフォールバックします。
- **店舗設定** — 通貨、通貨の表示位置、ロケール、タイムゾーン。デフォルトではサイトの値が使用されます。この店舗がサイトのデフォルトと異なる運用をする場合のみ上書きしてください。
- **営業時間** — 曜日ごとの時間枠と、自由記述のメモ欄です。任意項目で、一部のテンプレートで使用されます。

</StepCard>
  <StepCard number="5" title="保存" isLast>

新しく作成した店舗は、初回保存するまで**作成**ボタンが表示されます。**作成**をクリックしてください。

保存後、ページは自動保存モードに切り替わります。編集内容は短い遅延の後に自動的に保存され、画面の隅に*保存中… → 保存済み*のインジケーターが表示されます。いつでもページを離れることができ、変更は保持されます。未保存の変更がある状態でブラウザを閉じようとすると、警告が表示されます。

  </StepCard>
</Steps>

## ログイン時にキャッシャーに表示される内容 {#what-cashiers-see-at-login}

キャッシャーがサインインすると、POSには権限のあるすべての店舗を一覧表示する**店舗選択画面**が表示されます：

- **権限のある店舗が1つの場合** — その店舗が自動的に選択され、**POSを開く**ボタンがすぐに有効になります。キャッシャーがこれから入る店舗を確認できるよう、選択画面は引き続き表示されます。
- **権限のある店舗が複数の場合** — ラジオボタンのリストが表示されます。店舗を選択してから、**POSを開く**をクリックしてください。
- **権限のある店舗がない場合** — 選択画面は空です。店舗の*認可ユーザー*リストにキャッシャーを追加するか、リストを空のままにして全員を認可してください。

選択した店舗はPOS起動時に`?store={id}`としてURLに付加され、そのセッション中に作成されたすべての注文に記録されます(`_pos_store` 注文メタ)。これにより、店舗別レポート、注文一覧の店舗フィルター、レシートへの適切なロゴと住所の表示、店舗別価格設定が実現されます。

## 店舗別価格設定 {#per-store-pricing}

WCPOS Proには、ATUMとは独立したネイティブの店舗別価格設定機能があります。すべての商品（およびバリエーション）の編集画面で、**価格**タブと**税金**タブに店舗別の上書き設定が追加されます：

1. WP Admin → 商品 → 編集 で商品を開きます。
2. **価格**タブで、上書きしたい店舗の*「店舗別価格を設定」*ボックスにチェックを入れます。
3. その店舗の通常価格（および任意のセール価格）を入力します。
4. 商品を保存します。

レジ担当者がその店舗で操作している場合、POSは店舗別価格を使用します。上書き設定のない店舗で操作している場合は、WooCommerceの標準価格が適用されます。

同じ方法が**税金**タブでも適用でき、店舗別の課税ステータス／税区分を設定できます。ある店舗が免税区域にあり、別の店舗がそうでない場合に便利です。

:::tip ATUMとネイティブ店舗別価格設定の比較
WCPOSのネイティブ店舗別価格設定は、店舗ごとに異なる販売価格を設定しつつ在庫は**共有**したい場合に適したツールです。**店舗別の在庫数**も必要な場合は、[ATUM Multi-Inventory](/extensions/atum)連携をインストールしてください。店舗エディターに*在庫ロケーション*ドロップダウンと*価格ソース*セレクター（デフォルト / WCPOS Pro / ATUM）が追加され、店舗ごとに両方のシステムを組み合わせて使用できます。
:::

## エッジケース {#edge-cases}

<Accordion>
  <AccordionItem question="新しい店舗を追加したとき、既存の注文はどうなりますか？">

何も起こりません。注文は販売時に `_pos_store` メタが付与されます。新しい店舗を追加しても、既存の注文が遡って再割り当てされることはありません。Pro を有効化する前（または単一店舗構成時）に作成された注文には `_pos_store` メタがなく、引き続きサイトレベルのデフォルト設定で表示されます。

既存の注文を別の店舗に再割り当てする必要がある場合は、WP Admin → WooCommerce → 注文で該当注文を開き、Proが単一注文画面に追加する**店舗**ドロップダウンを使用してください。

  </AccordionItem>

  <AccordionItem question="店舗を削除するには？">

店舗エディタの**ゴミ箱**ボタン（および店舗一覧の行アクション）は、WordPressのソフトデリート(`wp_trash_post`)を実行します。完全削除ではないため、以下の動作になります：

- レジ担当者の店舗選択画面およびフィルタードロップダウンから非表示になります。
- ゴミ箱に移動した店舗を参照している既存の注文は、レシートの再印刷や注文詳細の表示で引き続き正しく解決されます。ゴミ箱内の店舗の名前、住所、ロゴは引き続き利用可能です。
- ゴミ箱から復元すると、すべての設定がそのままの状態で店舗がアクティブ状態に戻ります。

完全な削除（ゴミ箱 → 完全に削除する）を実行した場合のみ、店舗レコードが削除されます。完全削除された店舗を参照する注文は、*「店舗 #ID」* およびサイト全体のデフォルト設定にフォールバックします。

  </AccordionItem>

  <AccordionItem question="1人のキャッシャーが複数の店舗で働くことはできますか？">

はい。アクセスを許可する各店舗の*認可ユーザー*リストで該当ユーザーの名前にチェックを入れてください。ユーザーごとの「プライマリ店舗」という概念はありません。サインインするたびに、店舗選択画面からどの店舗で業務を行うかを選択します。

  </AccordionItem>

  <AccordionItem question="店舗ごとに異なる通貨を使用できますか？">

店舗のサイドバーで**通貨**と**通貨の位置**を設定します。これらの設定は、その店舗で開かれたセッションに対してサイト全体の WooCommerce 通貨設定を上書きします。レシート、価格、合計金額はすべて店舗の通貨でフォーマットされます。

これは表示フォーマットのみの設定です。各通貨に対応する WooCommerce 互換の決済ゲートウェイが別途必要であり、為替レートの処理は店舗側の責任となります。

  </AccordionItem>

  <AccordionItem question="Proをインストールしても店舗サブメニューが表示されないのはなぜですか？">

よくある原因は2つあります：

1. **Pro が有効化されていない** — **POS → 設定 → ライセンス** に移動し、ライセンスキーがアクティブと表示されていることを確認してください。
2. **旧無料版 + Pro レイアウト** — v1.8 以前の Pro は無料プラグインへのアドオンでした。v1.8 以降にアップグレードした場合、無料プラグインを無効化して削除する必要があります（[WCPOS Pro のインストール](/getting-started/pro-license)の注記を参照）。両方をインストールしたままにすると、店舗メニューが表示されなくなることがあります。

  </AccordionItem>
</Accordion>
