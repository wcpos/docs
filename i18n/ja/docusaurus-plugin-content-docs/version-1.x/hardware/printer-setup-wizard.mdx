  <WizardStep id="network-setup" title="ネットワークプリンターを接続する">

**POS &gt; 設定 &gt; プリンター &gt; プリンターを追加**を開きます。プリンターの **IP アドレス**を入力し、保存します。

IP アドレスを確認するには、プリンターのネットワークセルフテストを印刷してください：

- **Star：** プリンターの電源が入った状態で、**FEED** ボタンを約 5 秒間押し続けます。
- **Epson：** **FEED** ボタンを押しながらプリンターの電源を入れます。

後で印刷ができなくならないよう、プリンターには**固定アドレス**（ルーターの DHCP 予約）を割り当てることを強くおすすめします。

<WizardFigure src="/img/printer-wizard/star-selftest.png" alt="IP アドレスと開いているポートが強調表示されたプリンターのネットワークセルフテスト印刷" summary="セルフテストには、プリンターの現在の IP アドレスと開いているネットワークポートが表示されます。" />

  <WizardFix id="fix-not-printing" title="テスト印刷が出てこないとき">

まずは簡単なチェックから：用紙は入っていますか、カバーは閉まっていますか、用紙は正しい向きでセットされていますか。感熱紙は片面にしか印刷されません — 上下逆に入れるとプリンターは白紙のまま紙送りします。スクラッチテストをしてみましょう：爪で用紙をこすり、跡がつく面が印字ヘッドに向いていなければならない面です。

次に、プリンターに到達できるか確認します：Web ブラウザで `http://<printer-ip>/` を開きます（読み込まれない場合は `https://<printer-ip>/` を試してください）。プリンター自身の設定ページが表示されれば、ネットワーク上にあり問題は設定側にあります。どちらも読み込まれない場合は、ネットワーク／IP の問題です（アドレスが間違っているか、プリンターが接続されていない）。

  <WizardFix id="net-deep" title="より詳しいネットワークチェック">
基本が正しくてもまだ印刷されない場合、次のいずれかのネットワークの問題が原因であることがほとんどです：

**IP が間違っている、または変わった。** プリンターのセルフテストに表示された IP が入力したものと一致しているか確認してください — 別のプリンターのアドレスを使ってしまうのはよくあることです。DHCP のプリンターは再起動でアドレスが変わることがあるので、ルーターに **DHCP 予約**を設定して固定してください。`169.254` で始まるアドレスは、プリンターがそもそもネットワークに接続できていないことを意味します。

**動いていたのに、その日の最初の印刷が失敗する。** プリンターのネットワークカードがスリープしていて、最初のジョブがそれを起こして失われたのです。2 回目のテストを送信し、固定のレジでは有線の **Ethernet** 接続を検討してください。

**2 台のレジ、1 台のプリンター。** 多くの Star ネットワークプリンターは、既定で一度に 1 接続しか受け付けません — 2 台目のレジはタイムアウトします。プリンターの Web 設定で **9100 マルチセッション**を有効にしてください。

**プリンターのネットワーク設定が間違ったプロトコルを使っている。** WCPOS は **ポート 9100 の raw TCP** でジョブを送信します。ブラウザでプリンターの設定ページ（`http://<printer-ip>/`、読み込まれない場合は `https://<printer-ip>/`）を開き、**raw／ポート 9100** 印刷が有効になっていることを確認してください — プリンターが LPR、IPP、WSD のみに設定されていると、ジョブは届きません。

**Wi-Fi に接続できない、または頻繁に切れる。** レシートプリンターはたいてい **2.4GHz のみ**です — 5GHz 専用やバンドステアリングのネットワークではなく、2.4GHz のネットワークに接続してください。接続が不安定な場合は、有線 Ethernet が確実な解決策です。

  <WizardInvalid id="web-generic-invalid" title="汎用プリンターにはデスクトップアプリが必要です">

Web ブラウザは raw ネットワーク接続を開けないため、印刷サーバーを内蔵した **Epson** および **Star** のプリンターにしか到達できません。汎用の ESC/POS ネットワークプリンターには、ネットワーク経由（ポート 9100）で直接接続する**デスクトップアプリ**を使用してください — [プリンターの設定](/hardware/printers)ページからダウンロードできます。

  </WizardInvalid>

  <WizardStep id="web-net-setup" title="Web アプリからネットワークプリンターを接続する">

**Web ブラウザから**ネットワークプリンターへ印刷するのは最も難しい設定です。ブラウザは HTTP／HTTPS 経由でプリンターの内蔵 Web サーバーとしか通信できず、さらにセキュア（HTTPS）な POS はプリンターへの接続もセキュアにするよう強制するためです。

まず、プリンターに到達できるか確認します：新しいブラウザタブで `http://<printer-ip>/` を開きます（読み込まれない場合は `https://<printer-ip>/` を試してください）。プリンター自身の設定ページが表示されればネットワーク上にあります。どちらも読み込まれない場合は、先にネットワーク／IP を修正してください。

次に、**POS &gt; 設定 &gt; プリンター**でプリンターの IP アドレスを指定して追加します。POS が **HTTPS** で動作している場合、プリンターへの接続は HTTPS（Epson はポート 8043、Star は 443）に強制されます — これはプリンターが信頼された **SSL 証明書**を持っている場合にのみ機能します。

  <WizardFix id="web-net-fix" title="Web でのネットワーク印刷がうまくいかないとき">

これは最も多い行き詰まりで、たいていブラウザからは解決できません。その理由と対処法は次のとおりです：

プリンターの**セルフテスト**を印刷し、SSL/TLS のセクションを確認してください。証明書が**存在しない**と表示されている場合、証明書を生成するまで HTTPS 印刷は機能しません — そして自己署名証明書の場合、各デバイスで一度 `https://<printer-ip>` にアクセスして承認しない限り、ブラウザは信頼しません。

「正常に送信されました」と表示されるのに何も印刷されない場合は、ブラウザがリクエストを密かにブロックしたか（プリンターは HTTP を話すのに POS は HTTPS — 「混合コンテンツ」ブロック）、証明書のない状態で HTTPS ポート（Epson は 8043、Star は 443）が強制されたことを意味します。

また、Web アプリはプリンターの**内蔵 Web サーバー** — Epson の **ePOS-Print** または Star の **WebPRNT** — を通じてしかプリンターに到達できないため、そのサービスがプリンターの設定で**オン**になっていることを確認してください。内蔵 Web サーバーを持つのは Epson の「インテリジェント」／**-NT** モデルと Star **WebPRNT** モデルだけです。お使いのプリンターにない場合、ブラウザからは到達できません。

**確実な解決策はデスクトップアプリです。** ブラウザの HTTPS 制限を一切受けず、ネットワーク経由（ポート 9100）でプリンターへ直接接続します。[プリンターの設定](/hardware/printers)からダウンロードし、同じプリンターをそこに追加してください。

      <WizardChoice value="yes" label="はい" goTo="g-width" />
      <WizardChoice value="no" label="まだ印刷されません" goTo="support" />

  <WizardStep id="bt-setup" title="Bluetooth で接続する">

Bluetooth のレシート印刷はプリンターメーカー独自のソフトウェアを使用するため、プリンターは **Epson** または **Star** のモデルである必要があります — 汎用の Bluetooth プリンターは、どのプラットフォームでも Bluetooth 経由ではサポートされません。

まず、**コンピューターまたはデバイスのシステム Bluetooth 設定でプリンターをペアリングしてください** — WCPOS 内ではありません。Bluetooth プリンターは一度に 1 台のデバイスしか記憶しないので、別のレジとペアリングされていた場合は、まずそちらで「登録解除」してください。**iPhone／iPad** では、確認を求められたら WCPOS に **Bluetooth** と**ローカルネットワーク**の両方の権限を許可してください。**Android** では、システムが接続を切らないよう WCPOS のバッテリー使用を**制限なし**に設定してください。**Windows または Mac のデスクトップ**では、まず OS の Bluetooth 設定でペアリングしてから、WCPOS でスキャンしてください。

次に、**POS &gt; 設定 &gt; プリンター**で**プリンターをスキャン**をタップし、お使いのプリンターを選択します — 名前とブランドが自動的に入力されます。

      <WizardChoice value="no" label="いいえ／ペアリングまたは接続できません" goTo="bt-fix" />
    </WizardGate>
  </WizardStep>

  <WizardFix id="bt-fix" title="Bluetooth がペアリングできない、または頻繁に切れる">

まずデバイスの**システム Bluetooth 設定でプリンターをペアリング**してから、WCPOS でもう一度スキャンしてください。以前にペアリングしていて再接続できない場合は、他のデバイスで「登録解除」（およびプリンターの Bluetooth をリセット）して、そちらに紐づいたままにならないようにしてください。電源が入っていて通信範囲内にあることを確認してください。iPhone／iPad では WCPOS に Bluetooth とローカルネットワークの両方の権限があることを確認し、Android ではアプリのバッテリー使用を制限なしに設定してください。Bluetooth は **Epson と Star のみ**対応であることをお忘れなく — 汎用の Bluetooth プリンターは接続できません。

  <WizardFix id="usb-fix" title="USB が接続できない">

**Android** では、ケーブルを差し直して USB の権限プロンプトを承認してください。**デスクトップアプリ（Windows）**で `LIBUSB_ERROR_NOT_SUPPORTED` エラーが出る場合は、Windows がデバイスを自身の印刷ドライバーに固定していることを意味します — 最も簡単な解決策は、USB ドライバーを一切使わない**ネットワーク（ポート 9100）接続に切り替える**ことです。（上級者は WinUSB ツールでドライバーを置き換えることもできますが、それによりそのデバイスの通常の Windows 印刷キューが無効になります — 内容を理解している場合のみ行ってください。）

**Web アプリ**に Bluetooth または USB でプリンターを接続できるのは **Google Chrome または Microsoft Edge** のみで（Safari と Firefox では不可）、プリンターは **Epson** または **Star** のモデルである必要があります。**POS &gt; 設定 &gt; プリンター**で**接続**をクリックし、ブラウザのデバイス選択画面でプリンターを選択します。

Chrome または Edge を使っていない場合 — または接続できない場合 — Bluetooth と USB には**デスクトップアプリ**が最も確実な選択肢です。[プリンターの設定](/hardware/printers)からダウンロードしてください。
