:::warning 重要
初回読み込み時、POSはWooCommerceからデフォルトの税設定を取得します。一般的に、特定のニーズがない限り、ユーザーはWooCommerceと同じ設定を維持するべきです。サーバーと異なる税設定があると、POSとWooCommerceが異なる税額を計算し、差異や注文同期の問題を引き起こす可能性があります。
:::

## すべての税率を表示 {#view-all-tax-rates}

**すべての税率を表示**ボタンをクリックすると、POSにダウンロードされたすべての税率が表示されます。表示内容は以下の通りです:

- 税率名
- 税率パーセンテージ
- 適用される地域/場所

これは、正しい税率が読み込まれているかを確認し、税金計算の問題をデバッグする際に役立ちます。

## 税金計算設定 {#tax-calculation-settings}

### 税込価格の入力 {#prices-entered-with-tax}

商品価格が税を含むかどうかを決定します:

| オプション | 説明 |
|------------|------|
| **はい**   | 商品価格は税を含みます。POSは総価格から逆算して税金を計算します |
| **いいえ** | 商品価格は税を含みません。POSは入力された価格に税を追加します |

### 税計算の基準 {#calculate-tax-based-on}

税金計算に使用する住所を決定します:

| オプション | 説明 |
|------------|------|
| **顧客の配送先住所** | 注文が配送される場所に基づいた税率 |
| **顧客の請求先住所** | 顧客の請求先住所に基づいた税率 |
| **店舗の基本住所**    | あなたの店舗の物理的な場所に基づいた税率 |

### 配送料の税クラス {#shipping-tax-class}

配送料がどのように課税されるかを決定します:

| オプション | 説明 |
|------------|------|
| **カートアイテムに基づく配送料の税クラス** | 配送料はカート内のアイテムの税クラスに基づいて計算されます |
| **標準**      | 配送は標準税率で課税されます |
| **軽減税率**  | 配送料は軽減税率で課税されます |
| **ゼロ税率**  | 配送料には税金が適用されません |

### 丸め処理 {#rounding}

| オプション | 説明 |
|------------|------|
| **小計レベルでの税金の丸め** | 税金は全体の小計に対して計算されてから丸められ、各個別の項目に対しては丸められません |

## 表示設定 {#display-settings}

### 店舗内での価格表示 {#display-prices-in-the-shop}

| オプション | 説明 |
|------------|------|
| **税込**    | 表示される価格には税額が含まれています |
| **税抜**    | 表示される価格には税額が含まれていません |

### カートおよびチェックアウト中の価格表示 {#display-prices-during-cart-and-checkout}

| オプション | 説明 |
|------------|------|
| **税込**    | カートとチェックアウトの価格には税が含まれています |
| **税抜**    | カートとチェックアウトの価格には税が含まれていません |

### 価格表示のサフィックス {#price-display-suffix}

価格の後に表示されるテキスト（例: "税込" や "税抜"）。

### 税金合計の表示 {#display-tax-totals}

| オプション | 説明 |
|------------|------|
| **合計として表示** | すべての税金が1つの合計行にまとめられます |
| **項目明細**     | 各税率が別々の行アイテムとして表示されます |

## サーバー設定の復元 {#restore-server-settings}

サーバーから現在のWooCommerce税設定を取得して適用するには、クリックしてください。

:::tip ベストプラクティス
「サーバー設定の復元」を定期的に使用して、POSの税設定をWooCommerceストアと同期させておくことをお勧めします。特に、WordPress管理画面で税設定を変更した後に行うと良いでしょう。
:::
