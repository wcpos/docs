---
title: POSへのアクセス
sidebar_label: アクセス
---

WordPressの権限とユーザーロールを活用して、WCPOSの機能へのアクセスを制御できます。ユーザーに特定のロールを割り当て、その権限を設定することで、異なるPOSユーザーのアクセスレベルをカスタマイズできます。WCPOSプラグインは既存のWordPressロールと統合され、レジ担当者向けの特定のロールを導入しています。

以下の表は、POSの主要なユーザーロールとその権限の概要です：

| ロール          | 作成元      | 説明                                                                                                                                                                                                                                                                                                                                                    |
| ------------- | --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 管理者 | WordPress       | サイト所有者で、デフォルトですべての権限が有効になっています。                                                                                                                                                                                                                                                                                                    |
| ショップ管理者  | WooCommerce     | POSにアクセスし、POSの設定を管理する必要があるユーザー向けのロールです。ショップ管理者はプラグインのインストールなどの管理タスクを実行する権限がない場合があります。                                                                                                                                                             |
| レジ担当者       | WCPOS | WCPOSプラグインによって追加されるロールで、主にPOSを使用するユーザー向けです。レジ担当者は通常、制限された権限を持ち、WP管理画面でPOS設定の変更、プラグインのインストール、ブログ投稿の作成などができない場合があります。場合によっては、レジ担当者が顧客の追加や編集を制限したいこともあります。 |

ユーザーロールと権限を割り当て、管理するには、**`WP管理画面 > POS > アクセス`**に移動します。

以下の表は、WCPOSプラグインに関連するWordPressの権限の概要です：

| 権限                  | 説明                                                            | 使用例                                                                                                                                    |
| --------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `access_woocommerce_pos`    | WCPOSインターフェースへのアクセスを許可します。                        | このオプションを有効にすると、レジ担当者にPOSへのアクセスと使用を許可し、販売取引を実行できるようになります。          |
| `manage_woocommerce_pos`    | WCPOSの設定と構成の管理を有効にします。 | このオプションを有効にすると、レジ担当者に支払い方法や領収書テンプレートなどのPOS関連設定の変更を許可します。     |
| `create_users`              | 新しいWordPressユーザーアカウントの作成を許可します。              | このオプションを無効にすると、レジ担当者はPOSで新しい顧客アカウントを作成できなくなります。                                                  |
| `edit_others_shop_orders`   | 他のレジ担当者が作成したショップ注文の編集を有効にします。              | このオプションを無効にすると、レジ担当者は別のレジ担当者がPOSで作成したショップ注文を変更できなくなります。                              |
| `edit_product`              | 商品の詳細を編集することをレジ担当者に許可します。                            | このオプションを無効にすると、レジ担当者はPOSで利用可能な商品の詳細を変更できなくなります。                                     |
| `edit_published_products`   | 公開済み商品の編集をレジ担当者に許可します。                         | このオプションを無効にすると、レジ担当者はPOSで既に公開されている商品に変更を加えられなくなります。                           |
| `edit_shop_orders`          | ショップ注文の編集をレジ担当者に許可します。                                | このオプションを無効にすると、レジ担当者はPOSで注文状況や顧客詳細などのショップ注文の詳細を変更できなくなります。 |
| `edit_users`                | 既存のユーザーアカウントの編集をレジ担当者に許可します。                     | このオプションを無効にすると、レジ担当者は顧客アカウントの請求先住所などの詳細を変更できなくなります。            |
| `list_users`                | ユーザーアカウントの一覧表示をレジ担当者に許可します。                    | このオプションを無効にすると、レジ担当者は顧客の検索や選択ができなくなります。                                                |
| `manage_product_terms`      | 商品のカテゴリーとタグの管理をレジ担当者に許可します。              | このオプションを無効にすると、レジ担当者はPOSで商品カテゴリーやタグの作成、編集、削除ができなくなります。                       |
| `publish_shop_orders`       | ショップ注文の公開をレジ担当者に許可します。                             | このオプションを無効にすると、レジ担当者は新しいショップ注文の確定と処理ができなくなります。                                       |
| `read_private_products`     | プライベート商品の詳細へのアクセスをレジ担当者に許可します。          | このオプションを無効にすると、レジ担当者はPOSで利用可能なプライベート商品の詳細を表示できなくなります。                               |
| `read_private_shop_coupons` | プライベートショップクーポンの詳細へのアクセスをレジ担当者に許可します。      | このオプションを無効にすると、レジ担当者はPOSでプライベートショップクーポンの詳細を表示できなくなります。                                     |
| `read_private_shop_orders`  | プライベートショップ注文の詳細へのアクセスをレジ担当者に許可します。       | このオプションを無効にすると、レジ担当者はPOSでプライベートショップ注文の詳細を表示できなくなります。                                      |
| `read`                      | WordPress管理画面へのアクセスを許可します。                                  | これを無効にすると、レジ担当者はWordPressダッシュボードとユーザープロファイルページにアクセスできなくなります。                                                   |

各ロールに適切な権限を割り当てることで、POSの異なるユーザーのアクセスと権限を微調整できます。これにより、特定のアクションの実行、機密情報の表示、ストアへの変更を制御できます。

WordPress におけるユーザーロールと権限の管理の詳細については、[WordPress Codex](https://codex.wordpress.org/Roles_and_Capabilities)を参照してください。
