---
title: POSへのアクセス
sidebar_label: アクセス
description: WCPOSへのアクセスのためのユーザー役割と機能を設定します。これには管理者、ショップマネージャー、キャッシャーの権限が含まれます。
---

WCPOS機能へのアクセスを制御するには、WordPressの機能とユーザー役割を利用できます。特定の役割をユーザーに割り当て、その機能を設定することで、異なるPOSユーザーのアクセスレベルを調整できます。WCPOSプラグインは、既存のWordPressの役割と統合し、キャッシャー用の特定の役割を導入します。

以下の表は、POS内の主なユーザー役割とそれぞれの機能を示しています。

| 役割           | Created By          | 説明                                                                                                                                                                                                                                                                                                                                                                   |
| -------------- | ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 管理者         | WordPress          | すべての機能がデフォルトで有効化されているサイト所有者。                                                                                                                                                                                                                                                                                                            |
| ショップマネージャー     | WooCommerce        | この役割は、POSにアクセスし、POS設定を管理する必要があるユーザーのために設計されています。ショップマネージャーは、プラグインをインストールしたり、他の管理作業を行ったりする権限を持っていない場合があります。                                                                                                                                                               |
| キャッシャー    | WCPOS              | この役割はWCPOSプラグインによって追加され、主にPOSを使用するユーザーを対象としています。キャッシャーは通常、限られた機能を持ち、POS設定を変更したり、プラグインをインストールしたり、ブログ投稿を作成したりするためにWP Adminにアクセスすることができません。場合によっては、キャッシャーがPOS内で顧客を追加または編集することを制限したい場合もあります。 |

ユーザー役割と機能を割り当て、管理するには、**`WP Admin > POS > Access`** に移動します。

以下の表は、WCPOSプラグインに関連するWordPressの機能の概要を提供します。

| 機能                                      | 説明                                                           | 使用例                                                                                                                                                              |
| ----------------------------------------- | -------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `access_woocommerce_pos`                  | WCPOSインターフェースへのアクセスを許可します。              | このオプションを有効にすると、キャッシャーにPOSへのアクセスと使用の権限が付与され、販売時点取引を行えるようになります。                                                     |
| `manage_woocommerce_pos`                  | WCPOSの設定と構成を管理する機能を有効にします。               | このオプションを有効にすると、キャッシャーはPOSに関連する設定（支払い方法や領収書テンプレートなど）を変更できるようになります。                                          |
| `create_users`                            | キャッシャーが新しいWordPressユーザーアカウントを作成できるようにします。 | このオプションを無効にすると、キャッシャーはPOS内で新しい顧客アカウントを作成できなくなります。                                                                      |
| `edit_others_shop_orders`                 | 他のキャッシャーによって作成されたショップ注文の編集を可能にします。  | このオプションを無効にすると、キャッシャーはPOS内で異なるキャッシャーによって作成されたショップ注文を変更できなくなります。                                            |
| `edit_product`                            | キャッシャーが製品の詳細を編集できるようにします。              | このオプションを無効にすると、キャッシャーはPOSにある製品の詳細を変更できなくなります。                                                                            |
| `edit_published_products`                 | キャッシャーが公開された製品を編集できるようにします。         | このオプションを無効にすると、キャッシャーはPOSにすでに公開されている製品を変更できなくなります。                                                                  |
| `edit_shop_orders`                        | キャッシャーがショップ注文を編集できるようにします。            | このオプションを無効にすると、キャッシャーはPOS内のショップ注文の詳細（注文状況や顧客の詳細など）を変更できなくなります。                                               |
| `edit_users`                              | キャッシャーが既存のユーザーアカウントを編集できるようにします。  | このオプションを無効にすると、キャッシャーはPOS内の顧客アカウントの詳細（請求先住所など）を変更できなくなります。                                                  |
| `list_users`                              | キャッシャーがユーザーアカウントのリストを表示できるようにします。 | このオプションを無効にすると、キャッシャーはPOS内で顧客を検索または選択できなくなります。                                                                          |
| `manage_product_terms`                    | キャッシャーが製品のカテゴリーとタグを管理できるようにします。  | このオプションを無効にすると、キャッシャーはPOS内で製品のカテゴリーやタグを作成、編集、または削除できなくなります。                                                |
| `publish_shop_orders`                     | キャッシャーがショップ注文を公開できるようにします。            | このオプションを無効にすると、キャッシャーはPOS内で新しいショップ注文を最終化および処理できなくなります。                                                            |
| `read_private_products`                   | キャッシャーがプライベート製品の詳細にアクセスできるようにします。 | このオプションを無効にすると、キャッシャーはPOS内で利用可能なプライベート製品の詳細を表示できなくなります。                                                        |
| `read_private_shop_coupons`               | キャッシャーがプライベートショップクーポンの詳細にアクセスできるようにします。 | このオプションを無効にすると、キャッシャーはPOS内でプライベートショップクーポンの詳細を表示できなくなります。                                                      |
| `read_private_shop_orders`                | キャッシャーがプライベートショップ注文の詳細にアクセスできるようにします。 | このオプションを無効にすると、キャッシャーはPOS内でプライベートショップ注文の詳細を表示できなくなります。                                                            |
| `read`                                    | WordPress Adminへのアクセスを許可します。                      | このオプションを無効にすると、キャッシャーはWordPressダッシュボードおよびユーザープロフィールページにアクセスできなくなります。                                          |

各役割に適切な機能を割り当てることで、POS内の異なるユーザーに対するアクセスと権限を微調整することができます。これにより、誰が特定のアクションを実行できるか、機密情報を表示できるか、ストアに変更を加えられるかを制御できます。

WordPressでのユーザー役割と機能の管理に関する詳細な情報については、[WordPress Codex](https://codex.wordpress.org/Roles_and_Capabilities)を参照してください。
