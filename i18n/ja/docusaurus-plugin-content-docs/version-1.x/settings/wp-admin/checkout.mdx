チェックアウト設定ページ (`WP Admin > POS > Settings > Checkout`) では、POS に表示する決済ゲートウェイと注文処理方法を制御します。

## 決済ゲートウェイ {#payment-gateways}

このセクションには、サイトにインストールされているすべての WooCommerce 決済ゲートウェイが表示されます。各ゲートウェイは POS ごとに個別に有効化または無効化できます。

| 列 | 説明 |
|--------|-------------|
| **有効** | POS チェックアウトでのゲートウェイの有効・無効を切り替えます |
| **ゲートウェイ** | 決済ゲートウェイ名 |
| **ゲートウェイ ID** | WooCommerce のゲートウェイ識別子（開発者向け） |

### ゲートウェイを有効化する {#enabling-a-gateway}

1. 一覧からゲートウェイを見つけます
2. **有効** スイッチをオンに切り替えます
3. 次回のページ読み込み時に、ゲートウェイがPOSチェックアウトに表示されます

### ゲートウェイの並び替え {#reordering-gateways}

ゲートウェイをドラッグして、POSチェックアウトでの表示順序を変更できます。レジ担当者がチェックアウトモーダルを開いたとき、有効化されている最初のゲートウェイがデフォルトで選択されます。

### POSとオンラインストアの違い {#pos-vs-online-store}

ここでのゲートウェイ設定は、WooCommerceの決済設定とは**独立しています**。ゲートウェイは次のように設定できます：

- **POSで有効、オンラインで無効** — Web注文には適用されない現金支払いに便利です
- **オンラインで有効、POSで無効** — 対面販売には実用的でないPayPalなどのゲートウェイに便利です
- **両方で有効** — ゲートウェイがどちらでも動作します

:::note
ゲートウェイは、まず以下にインストールして有効化する必要があります: `WP Admin > WooCommerce > Settings > Payments` その後、POS のチェックアウト設定リストに表示されます。ゲートウェイが見当たらない場合は、プラグインがインストールされ、WooCommerce でそのゲートウェイが有効になっていることを確認してください。
:::

## 注文ステータス {#order-status}

完了した POS 注文のデフォルトの注文ステータスを設定します。POS 注文では販売時点で支払いが回収されるため、デフォルトでは **完了済み** に設定されます。
