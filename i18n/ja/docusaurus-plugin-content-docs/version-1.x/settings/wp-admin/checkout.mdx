チェックアウト設定ページ (`WP Admin > POS > Settings > Checkout`) では、POS に表示する決済ゲートウェイと注文処理の方法を制御します。

## 決済ゲートウェイ {#payment-gateways}

このセクションには、サイトにインストールされているすべての WooCommerce 決済ゲートウェイが一覧表示されます。各ゲートウェイは POS ごとに個別に有効化または無効化できます。

| 列 | 説明 |
|--------|-------------|
| **有効** | POS チェックアウトでゲートウェイの有効・無効を切り替えます |
| **ゲートウェイ** | 決済ゲートウェイ名 |
| **ゲートウェイ ID** | WooCommerce ゲートウェイの識別子 (開発者向け) |

### ゲートウェイを有効化する {#enabling-a-gateway}

1. リストからゲートウェイを見つけます
2. **有効** スイッチをオンに切り替えます
3. 次回のページ読み込み時に、POSチェックアウトにゲートウェイが表示されます

### ゲートウェイの並べ替え {#reordering-gateways}

ゲートウェイをドラッグして、POSチェックアウトでの表示順を変更します。レジ係がチェックアウトモーダルを開いたとき、最初に有効化されているゲートウェイがデフォルトで選択されます。

### POSとオンラインストアの比較 {#pos-vs-online-store}

ここでのゲートウェイ設定は、WooCommerceの決済設定とは**独立**しています。ゲートウェイは次のように構成できます。

- **POSで有効、オンラインで無効** — Web注文には適用されない現金払いなどに便利です
- **オンラインで有効、POSで無効** — 対面販売には適さないPayPalなどのゲートウェイに便利です
- **両方で有効** — ゲートウェイがどこでも動作します

:::note
ゲートウェイは、まず以下にインストールされ、有効化されている必要があります `WP Admin > WooCommerce > Settings > Payments` そうすることで、POSのチェックアウト設定リストに表示されます。ゲートウェイが見つからない場合は、プラグインがインストールされていること、およびWooCommerceでゲートウェイが有効になっていることを確認してください。
:::

## 注文ステータス {#order-status}

完了したPOS注文のデフォルトの注文ステータスを設定します。デフォルトでは、販売時点で支払いが回収されるため、POS注文は**完了**に設定されます。
