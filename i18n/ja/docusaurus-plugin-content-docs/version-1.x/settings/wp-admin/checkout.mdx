チェックアウト設定ページ (`WP Admin > POS > 設定 > 支払い`) では、POS に表示される決済ゲートウェイ、注文の処理方法、POS 販売で送信される通知メールを制御します。

## メール通知 {#email-notifications}

ページ上部には、**管理者メール**、**顧客メール**、**キャッシャーメール**の 3 つのメール切り替えグループがあります。これらは、サイト全体の WooCommerce メール設定とは独立して、POS 注文で送信される WooCommerce 通知メールを制御します。

メールごとの切り替えのデフォルト、キャッシャー受信者モデル、一般的なシナリオ、開発者フックを含む完全なリファレンスについては、**[メール通知](/settings/wp-admin/email-notifications)**を参照してください。

## 決済ゲートウェイ {#payment-gateways}

このセクションには、サイトにインストールされているすべての WooCommerce 決済ゲートウェイが一覧表示されます。各ゲートウェイは、POS で個別に有効化または無効化できます。

| 列 | 説明 |
|--------|-------------|
| **有効** | POS チェックアウトでゲートウェイをオンまたはオフに切り替えます |
| **ゲートウェイ** | 決済ゲートウェイ名 |
| **ゲートウェイ ID** | WooCommerce ゲートウェイ識別子（開発者向け） |

### ゲートウェイの有効化 {#enabling-a-gateway}

1. 一覧でゲートウェイを探します
2. **有効** スイッチをオンにします
3. 次回ページを読み込むと、ゲートウェイが POS チェックアウトに表示されます

### ゲートウェイの並べ替え {#reordering-gateways}

ゲートウェイをドラッグして、POS チェックアウトでの表示順を変更します。レジ担当者がチェックアウトモーダルを開くと、最初に有効なゲートウェイがデフォルトで選択されます。

### POS とオンラインストア {#pos-vs-online-store}

ここでのゲートウェイ設定は、WooCommerce の支払い設定から **独立** しています。ゲートウェイは次のように設定できます。

- **POS で有効、オンラインでは無効** — Web 注文には適用されない現金支払いに便利です
- **オンラインで有効、POS では無効** — 対面販売には実用的でない PayPal のようなゲートウェイに便利です
- **両方で有効** — ゲートウェイはすべての場所で動作します

:::note
決済方法は、POSチェックアウト設定の一覧に表示される前に、まず `WP Admin > WooCommerce > 設定 > 支払い` でインストールされ、有効化されている必要があります。決済方法が見つからない場合は、プラグインがインストールされていること、およびその決済方法が WooCommerce で有効になっていることを確認してください。
:::

## 注文ステータス {#order-status}

完了したPOS注文のデフォルト注文ステータスを設定します。初期設定では、POS注文は販売時に支払いが回収されるため、**完了**に設定されます。
