チェックアウト設定ページ (`WP Admin > POS > 設定 > 支払い`) では、POSに表示する決済ゲートウェイ、注文の処理方法、POS売上時に送信する通知メールを制御します。

## メール通知 {#email-notifications}

ページ上部には、**管理者メール**、**顧客メール**、**レジ担当者メール**の3グループのメールトグルがあり、サイト全体のWooCommerceメール設定とは独立して、POS注文に対するWooCommerce通知メールの送信を制御します。

メールごとのトグルのデフォルト値、レジ担当者宛メールモデル、一般的なシナリオ、開発者向けフックなど、詳細については**[メール通知](/settings/wp-admin/email-notifications)**を参照してください。

## 決済ゲートウェイ {#payment-gateways}

このセクションには、サイトにインストールされているすべてのWooCommerce決済ゲートウェイが一覧表示されます。各ゲートウェイは、POSに対して個別に有効・無効を切り替えることができます。

| 列 | 説明 |
|--------|-------------|
| **有効** | POSチェックアウトでゲートウェイのオン・オフを切り替えます |
| **ゲートウェイ** | 決済ゲートウェイ名 |
| **ゲートウェイ ID** | WooCommerce のゲートウェイ識別子（開発者向け） |

### ゲートウェイの有効化 {#enabling-a-gateway}

1. 一覧からゲートウェイを探します
2. **有効** スイッチをオンに切り替えます
3. 次回のページ読み込み時に、POS の会計画面にゲートウェイが表示されます

### ゲートウェイの並べ替え {#reordering-gateways}

ゲートウェイをドラッグして、POS 会計画面での表示順を変更できます。最初に有効化されたゲートウェイが、レジ担当者が会計モーダルを開いた際にデフォルトで選択されます。

### POS とオンラインストア {#pos-vs-online-store}

ここでのゲートウェイ設定は、WooCommerce の決済設定とは**独立**しています。ゲートウェイは以下のように設定できます：

- **POS で有効、オンラインで無効** — ウェブ注文には適用されない現金決済に便利です
- **オンラインで有効、POS で無効** — 対面販売には実用的でない PayPal などのゲートウェイに便利です
- **両方で有効** — ゲートウェイはすべての環境で使用可能です

:::note
ゲートウェイは、POSのチェックアウト設定一覧に表示される前に、まず `WP Admin > WooCommerce > 設定 > 支払い` でインストールおよび有効化されている必要があります。ゲートウェイが表示されない場合は、プラグインがインストールされており、WooCommerceでゲートウェイが有効になっていることを確認してください。
:::

## 注文ステータス {#order-status}

POS注文完了時のデフォルト注文ステータスを設定します。POS注文は販売時点で決済が完了するため、デフォルトでは**完了**に設定されています。
