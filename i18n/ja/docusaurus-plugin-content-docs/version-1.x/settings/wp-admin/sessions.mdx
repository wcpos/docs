**セッション** 画面では、誰がどのデバイスで現在 POS にログインしているかを確認し、個々のデバイスをリモートでサインアウトできます。デバイスを紛失または盗難された場合、レジ係が離れた場合、または単にアクティブなログインを確認したい場合に、ここにアクセスします。

**`WP Admin > POS > Settings > Sessions`** にあります。

> _アクティブなユーザー セッションを管理します。ログインしているすべてのユーザーを表示したり、ユーザーのデバイスを確認したり、必要に応じてセッションを終了したりできます。_

## 仕組み {#how-it-works}

画面には 2 つのペインがあります。

- **ユーザー** (左) — 少なくとも 1 つのアクティブなセッションを持つすべてのユーザー。過去数分間にアクティブだったユーザーには緑色のドットが表示されます。自分のアカウントの **You** チップと、各ユーザーが持つセッション数のカウント。フィルター ボックスを使用すると、名前またはユーザー名で検索できます。
- **セッション** (右) — ユーザーを選択すると、アクティブなセッションごとに 1 枚のカードが表示されます。

各セッション カードには次の内容が表示されます。

| 分野 | それが何を意味するか |
| --- | --- |
| **応用** | ユーザーがログインしている場所 — **iOS アプリケーション**、**Android アプリケーション**、**デスクトップ アプリケーション**、または **Web アプリケーション** – およびブラウザ/バージョン |
| **現在のデバイス** | 現在使用しているセッションをマークするチップ |
| **最後にアクティブだった** | セッションが最近リクエストを行った時期 (例: _今_、_5 分前_) |
| **IP** | セッションの接続元の IP アドレス (または _N/A_) |
| **作成されました** | セッションが最初にサインインしたとき |
| **期限切れ** | セッショントークンの有効期限が切れたとき |
| **ユーザーエージェント** | 展開すると、生のブラウザ/デバイスのユーザー エージェント文字列が表示されます。 |

過去 5 分以内にリクエストを行ったセッションは、**現在アクティブです** と表示されます。

## デバイスのサインアウト {#signing-devices-out}

それぞれ確認ダイアログの背後にある 3 つのアクションが利用可能です。

- **終了** — 単一セッションをサインアウトします。そのセッションのカード上のボタンを使用します。
- **他のデバイスをログアウト** — 選択したユーザーを、**現在使用しているデバイスを除く**すべてのデバイスからサインアウトします。ユーザーが複数のセッションを持っている場合に表示されます。
- **すべてログアウト** — 現在のデバイスを含む**すべて**のデバイスから選択したユーザーをサインアウトします。

:::tip デバイスの紛失または盗難
POS が搭載された携帯電話、タブレット、またはラップトップが紛失した場合は、セッションを開き、影響を受けるユーザーを選択し、そのデバイスのセッションで **終了** を使用します (または、安全のため **すべてログアウト**)。デバイスは次のリクエストでアクセスを失います。
:::

:::note
現在のセッションを終了すると、現在使用しているデバイスからサインアウトされます。その前に確認を求められます。
:::
