**セッション**画面では、現在 POS にログインしているユーザー、使用中のデバイスを確認し、個別のデバイスをリモートでサインアウトできます。デバイスの紛失や盗難、キャッシャーの退職、または有効なログインを確認したい場合に使用します。

**`WP Admin > POS > 設定 > Sessions`** にあります。

> _有効なユーザーセッションを管理します。ログイン中のすべてのユーザーを表示し、使用中のデバイスを確認して、必要に応じてセッションを終了できます。_

## 仕組み {#how-it-works}

画面は2つのペインで構成されています。

- **ユーザー**（左）— 少なくとも1つの有効なセッションがあるすべてのユーザーが表示されます。直近数分以内にアクティブだったユーザーには緑のドットが付き、自分のアカウントには **あなた** チップが表示され、各ユーザーのセッション数も確認できます。フィルターボックスでは、名前またはユーザー名で検索できます。
- **セッション**（右）— ユーザーを選択すると、有効なセッションごとに1枚のカードが表示されます。

各セッションカードには次の情報が表示されます。

| フィールド | 意味 |
| --- | --- |
| **アプリケーション** | ユーザーがログインしている場所 — **iOSアプリケーション**、**Androidアプリケーション**、**デスクトップアプリケーション**、または **Webアプリケーション** — とブラウザー/バージョン |
| **現在のデバイス** | 現在使用しているセッションを示すチップ |
| **最終アクティブ** | セッションが最後にリクエストを行ってからの経過時間（例: _たった今_、_5分前_） |
| **IP** | セッションの接続元IPアドレス（または _N/A_） |
| **作成日時** | セッションで最初にサインインした日時 |
| **有効期限** | セッショントークンの有効期限が切れる日時 |
| **ユーザーエージェント** | 展開すると、ブラウザー/デバイスの生のユーザーエージェント文字列を表示します |

過去5分以内にリクエストを行ったセッションは、**現在アクティブ**として表示されます。

## デバイスからサインアウトする {#signing-devices-out}

3つの操作を利用できます。いずれも確認ダイアログが表示されます。

- **終了** — 1つのセッションからサインアウトします。そのセッションのカードにあるボタンを使用します。
- **他のデバイスからログアウト** — 選択したユーザーを、現在使用しているデバイス**以外**のすべてのデバイスからサインアウトします。ユーザーに複数のセッションがある場合に表示されます。
- **すべてログアウト** — 選択したユーザーを、現在のデバイスを含む**すべての**デバイスからログアウトさせます。

:::tip デバイスの紛失または盗難
POS が入った電話、タブレット、またはノートパソコンを紛失した場合は、セッションを開き、対象のユーザーを選択して、そのデバイスのセッションで **終了** を使用します（安全のために **すべてログアウト** を使用することもできます）。そのデバイスは次回のリクエスト時にアクセスできなくなります。
:::

:::note
現在の自分のセッションを終了すると、今使用しているデバイスからログアウトされます。その前に確認を求められます。
:::
