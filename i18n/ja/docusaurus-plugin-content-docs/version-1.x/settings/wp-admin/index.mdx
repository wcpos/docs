これらの設定では、PHP と WooCommerce の構成を含む POS アプリケーションのバックエンド機能を構成します。`WP Admin > POS > 設定` に移動してアクセスできます。

## 設定の概要 {#settings-at-a-glance}

| 設定 | 機能 | 詳細 |
|---|---|---|
| **一般** | すべてのキャッシャーに対するデフォルトのカート顧客、POS専用商品、小数数量、商品のバーコードフィールド。 | [一般設定](./general) |
| **チェックアウト** | POSで利用可能な決済ゲートウェイと、POS注文の処理方法。 | [チェックアウト設定](./checkout) |
| **メール通知** | メールごとの管理者、顧客、キャッシャー向け切り替えで、POS注文に対して送信される WooCommerce メールを制御します。 | [メール通知](./email-notifications) |
| **アクセス** | 専用のキャッシャー権限グループを含め、POSを使用できるユーザーの権限グループと権限を設定します。 | [POSへのアクセス](./access) |
| **セッション** | ユーザーおよびデバイスごとのアクティブなPOSログインセッションを表示し、デバイスをリモートでサインアウトさせます。 | [セッション](./sessions) |
| **店舗税務ID** | レシートや請求書に印刷される事業者の税務識別番号（VAT、ABN、GSTIN、EIN、会社登録番号）。 | [店舗税務ID](./store-tax-ids) |
| **顧客の税務ID** | 顧客の税務IDの保存方法、書き込み先のプラグインフィールド、チェックアウト時に注文へコピーされる仕組み。 | [顧客の税務ID](./customer-tax-ids) |
