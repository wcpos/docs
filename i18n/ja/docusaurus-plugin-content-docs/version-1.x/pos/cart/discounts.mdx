WCPOSでは、クイックパーセントボタン、明細価格の直接編集、注文全体の割引手数料など、レジ係がその場で注文を値引きするための複数の方法を提供します。使用ルールを設定した事前構成済みプロモーションについては、[クーポン](/coupons)（Pro）を参照してください。

:::info v1.9.0 で変更
POSの価格変更がWooCommerceの割引合計に表示される方法は、v1.9.0で変更されました。現在のレシートテンプレートでは元の価格とPOSでの値引きを別々に表示できますが、ネイティブの**割引**合計は引き続きクーポン用に確保されています。詳細は、以下の[ v1.9.0 での変更点](#what-changed-in-v190)を参照してください。
:::

## クイック割引 {#quick-discounts}

クイック割引ボタンを使用すると、1回のタップで注文全体にパーセント割引を適用できます。

設定するには、カートの[表示設定](/pos/cart#display-settings)を開き、**クイック割引**フィールドにパーセント値をカンマ区切りで入力します。たとえば、`5,10,15,20`と入力すると、4つのショートカットボタンが作成されます。

クイック割引ボタンをタップすると、カート内のすべての明細にパーセント割引が適用されます。

## 明細ごとの割引 {#line-item-discounts}

カート内の任意の明細の価格を直接変更できます:

1. 明細の **価格** フィールドをクリックします
2. 新しい価格を入力します
3. **Enter** を押して確定します

価格合わせ、スタッフ割引、または一度限りの調整に便利です。明細の合計は、数量 × 新しい価格に基づいて自動的に更新されます。

カート内の商品の編集について詳しくは、[明細](/pos/cart/line-items)を参照してください。

:::tip 商品の分割
同じ商品の一部に異なる割引を適用する場合（例: 3個は通常価格、2個は割引価格）、カートの[表示設定](/pos/cart#display-settings)で **分割** オプションを有効にすると、明細を別々の行に分割できます。
:::

## 注文全体の割引 {#order-level-discounts}

個々の商品ではなく注文全体に定額割引を適用するには、**マイナスの手数料**を追加します:

1. カート内の商品一覧の下にある **手数料を追加** をクリックします
2. 名前を入力します（例: "スタッフ割引"）
3. 割引額を負の数で入力します（例: `-5.00`）

手数料は個別の明細行として表示され、注文合計を減額します。必要に応じて、三点メニューから手数料の税区分を編集できます。

## POSの割引とWooCommerceクーポンの比較 {#pos-discounts-vs-woocommerce-coupons}

このページの割引は、レジ担当者が販売時に適用する都度調整です。一方、WooCommerceの**クーポン**は、ルールとトラッキングを備えた事前設定済みのプロモーションです。主な違いは次のとおりです。

| | POSの割引 | WooCommerceクーポン（Pro） |
|---|---|---|
| **適用方法** | クイック割引、明細価格の編集、または負の手数料 | カートでクーポンコードを入力 |
| **設定場所** | レジ担当者がその場で設定 | **WP Admin → マーケティング → クーポン**で事前設定 |
| **トラッキング** | 明細価格として記録（[v1.9.0の変更](#what-changed-in-v190)を参照） | WooCommerceレポートでクーポン割引として追跡 |
| **制限** | なし — レジ担当者が任意の価格を設定 | 利用回数制限、商品・カテゴリーの制限、最低利用額、有効期限、メールルール |
| **最適な用途** | 都度調整、価格合わせ | 構造化されたプロモーション、追跡可能な割引 |

:::tip どちらを使用すべきですか？
一時的な価格調整には、このページの値引き機能の方が簡単です。WooCommerceのレポートで割引の使用状況を追跡する必要がある場合や、使用回数制限などのルールを適用する場合は、[クーポン](/coupons)を使用してください。
:::

## POSの価格変更とクーポンの連携 {#how-pos-price-changes-interact-with-coupons}

レジ担当者が明細の価格をカスタム価格に設定し（例: $18から$16に値下げ）、その後クーポンを適用すると、クーポンは元の価格（$18）ではなく、**POS値引き後の価格**（$16）に対して計算されます。これにより、レジ担当者による値引きとクーポンを元の価格に重ねて適用することで、顧客に過剰な割引が適用されるのを防ぎます。

- POS値引きが適用された商品は、WooCommerceでは「セール中」として扱われます。クーポンで**セール商品を除外**を有効にしている場合、通常のセール商品と同様に、POS値引きが適用された商品は対象外になります。開発者は、`woocommerce_pos_item_is_on_sale`フィルターでこの動作を上書きできます。
- クーポンを削除しても、明細はPOS値引き後の価格のままです。

:::note 開発者向けリファレンス
POS価格の上書きの保存方法と利用可能なフィルターの技術的な詳細については、[POS値引きリファレンス](/reference/pos-discounts)を参照してください。
:::

## v1.9.0での変更点 {#what-changed-in-v190}

v1.8からアップグレード後、レシートやレポートの**値引き**合計が**0**と表示されるようになった場合は、このセクションで理由と対応方法を説明します。

### 変更内容 {#the-change}

v1.9.0 より前は、レジ担当者が明細価格を値下げした場合（例: $18 → $16）、POS は注文を `subtotal = $18` および `total = $16` として記録していました。WooCommerce はその後 `discount_total = $2` を計算し、これはレシートとレポートに割引として表示されていました。

このため、さらにクーポンを適用すると合計が正しくなくなっていました。クーポンが元の $18 を基準に計算されるため、割引額が過大になり、顧客への請求額が少なくなっていました。

v1.9.0 以降、WCPOS は WooCommerce のセール価格の扱いに合わせています。レジで設定した価格 **が** 明細の小計になります。WooCommerce が「割引」として扱うのはクーポンコードのみです（これはセール商品の WooCommerce の動作と同じで、セール価格も個別の割引明細なしで小計に組み込まれます）。

### 利用者への影響 {#what-this-means-for-you}

- **レシート**には、記録された通常価格、通常価格から販売価格への値引き額、およびその値引きとクーポンを合わせた **合計節約額** を表示できます。価格の値引きは、WooCommerce の **割引** フィールドとは別に扱われます。
- **レポート**では、POS の明細価格変更のみを使用した場合、`discount_total = 0` と表示されます。クーポンによる割引のみが計上されます。
- **クーポン**は、POS で値引きされた商品に重ねて適用した場合も正しく計算されるようになりました。
- **記録された通常価格と販売価格は保存されます**。これらは POS の明細メタデータ (`_woocommerce_pos_data`) に保存されるため、WooCommerce の割引合計を変更せずに、レシートで値引き額を算出できます。

### レシートに元の価格と値引き額を表示する {#showing-the-original-price-and-saving-on-receipts}

現在同梱されている価格表示対応テンプレートでは、商品がセール中の場合やレジで価格を変更した場合に、記録された通常価格と値引き額を表示します。WCPOS で注文全体の金額を完全に算出できる場合は、**合計節約額** も表示されます。この合計には通常価格からの値引きとクーポン割引が含まれ、どちらも二重計上されません。

ギャラリーからコピーしたテンプレートは編集可能なスナップショットであり、WCPOS の更新によって上書きされません。この変更前にレシートを作成した場合は、次のいずれかを行ってください。

1. 新しい同梱テンプレートのコピーで **テンプレートを使用** をクリックする。
2. [レシートデータリファレンス](/receipts/receipt-data#displaying-regular-price-and-savings)のフィールドで既存のテンプレートを更新します。

主なレシートフィールドは次のとおりです。

| フィールド | 意味 |
|-------|---------|
| `lines[].regular_price_display` | 記録された通常単価 |
| `lines[].selling_price_display` | クーポン適用前の販売単価 |
| `lines[].unit_savings_display` | 商品ごとの節約額 |
| `lines[].line_regular_total_display` | 行の通常価格合計 |
| `lines[].line_savings_display` | 行の節約額合計 |
| `totals.savings_total_display` | カタログセールとレジ価格の変更による通常価格と販売価格の差額合計 |
| `totals.discount_total_display` | WooCommerceの割引合計。通常は現在の注文に適用されたクーポンです |
| `totals.total_saved_display` | 通常価格からの節約額とWooCommerce割引の合計 |
| `totals.total_saved_complete` | 合計値が完全で、安全に表示できるかどうか |

カスタムテンプレートでは、履歴価格データが不完全な場合や値引き額がない場合に **合計割引** 行が表示されないようにしてください。正確なパターンについては、[レシートデータリファレンス](/receipts/receipt-data#totals)を参照してください。

WooCommerce レポートは変更されません。標準の割引合計には、現在のセール価格またはレジ価格による節約額ではなく、引き続きクーポンが含まれます。レシートには `totals.total_saved` を使用し、レポートでも節約額を WooCommerce の割引として追跡する必要がある場合は、引き続き[クーポン](/coupons)を使用してください。

## 既知の制限事項 {#known-limitations}

- **自動割引ルールは未対応** — POS では「2個購入で1個無料」のような自動割引はサポートされていません。体系的なプロモーションには WooCommerce クーポンを使用してください。
- **クイック割引はパーセントのみ** — 固定額割引用のクイックボタンは組み込まれていません。代わりにマイナスの手数料を使用するか、個別の価格を編集してください。
