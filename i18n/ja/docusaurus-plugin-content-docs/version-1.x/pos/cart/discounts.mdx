WCPOSでは、クイックパーセントボタン、明細価格の直接編集、注文レベルの割引手数料により、レジ担当者がその場で注文を割引できます。利用ルールを伴う事前設定済みのプロモーションについては、[クーポン](/coupons)（Pro）を参照してください。

:::info v1.9.0で変更
POSの価格変更がWooCommerceの割引合計に表示される方法がv1.9.0で変更されました。現在のレシートテンプレートでは、元の価格とPOSの節約額を別々に表示できます。一方、標準の**割引**合計は引き続きクーポン用に予約されています。詳細は以下の[ v1.9.0での変更内容](#what-changed-in-v190)を参照してください。
:::

## クイック割引 {#quick-discounts}

クイック割引ボタンを使用すると、タップ1回で注文全体にパーセント割引を適用できます。

設定するには、カートの[表示設定](/pos/cart#display-settings)を開き、**クイック割引**フィールドにパーセント値をカンマ区切りで入力します。たとえば、`5,10,15,20`と入力すると、4つのショートカットボタンが作成されます。

クイック割引ボタンをタップすると、そのパーセント値がカート内のすべての明細に適用されます。

## 明細ごとの割引 {#line-item-discounts}

カート内の個々の商品行の価格は、直接変更できます。

1. 商品行の **価格** フィールドをクリックします
2. 新しい価格を入力します
3. **Enter** を押して確定します

これは価格の照合、スタッフ割引、または個別の調整に便利です。商品行の合計は、数量 × 新しい価格に基づいて自動的に更新されます。

カート内の商品を編集する方法については、[商品行](/pos/cart/line-items)を参照してください。

:::tip 商品の分割
顧客が同じ商品の一部に異なる割引を適用したい場合（例: 3点を通常価格、2点を割引価格にする場合）、カートの[表示設定](/pos/cart#display-settings)で **分割** オプションを有効にすると、商品行を別々の行に分割できます。
:::

## 注文全体への割引 {#order-level-discounts}

個々の商品ではなく注文全体に固定額の割引を適用するには、**負の手数料**を追加します。

1. カート内の商品一覧の下にある **手数料を追加** をクリックします
2. 名前を入力します（例: "スタッフ割引"）
3. 割引額を負の数で入力します（例: `-5.00`）

手数料は別の明細行として表示され、注文合計を減額します。必要に応じて、三点メニューから手数料の税区分を編集できます。

## POS 割引と WooCommerce クーポンの比較 {#pos-discounts-vs-woocommerce-coupons}

このページの割引は、レジ担当者が会計時に適用する都度調整です。WooCommerce の**クーポン**は、ルールと追跡機能を備えた事前設定済みのプロモーションです。主な違いは次のとおりです。

| | POS 割引 | WooCommerce クーポン（Pro） |
|---|---|---|
| **適用方法** | クイック割引、明細価格の編集、または負の手数料 | カートでクーポンコードを入力 |
| **設定場所** | レジ担当者がその場で設定 | **WP Admin → マーケティング → クーポン**で事前設定 |
| **追跡** | 明細価格として記録（[v1.9.0 の変更](#what-changed-in-v190)を参照） | WooCommerce レポートでクーポン割引として追跡 |
| **制限** | なし。レジ担当者が任意の価格を設定 | 使用回数制限、商品・カテゴリーの制限、最低利用額、有効期限、メールルール |
| **最適な用途** | 都度調整、価格合わせ | 構造化されたプロモーション、追跡可能な割引 |

:::tip どちらを使うべきですか？
単発の価格調整には、このページの割引の方が簡単です。WooCommerce のレポートで割引の使用状況を追跡したり、使用回数制限などのルールを適用したりする必要がある場合は、[クーポン](/coupons)を使用してください。
:::

## POS の価格変更とクーポンの連携 {#how-pos-price-changes-interact-with-coupons}

レジ担当者が明細項目にカスタム価格を設定し（例: $18 を $16 に値下げ）、その後クーポンを適用すると、クーポンは元の価格（$18）ではなく **POS 割引後価格**（$16）に対して計算されます。これにより、レジ担当者による割引とクーポンを元の価格に重ねて適用することで、顧客が過剰に割引されるのを防ぎます。

- POS 割引済みの商品は、WooCommerce では「セール中」として扱われます。クーポンで **セール品を除外** を有効にすると、通常のセール品と同様に POS 割引済みの商品は対象外になります。開発者は `woocommerce_pos_item_is_on_sale` フィルターでこの動作を上書きできます。
- クーポンを削除しても、明細は POS 割引後の価格のままです。

:::note 開発者向けリファレンス
POS の価格上書きの保存方法と利用可能なフィルターの技術的な詳細については、[POS 割引リファレンス](/reference/pos-discounts)を参照してください。
:::

## v1.9.0 での変更点 {#what-changed-in-v190}

v1.8 からアップグレード後、レシートやレポートの **割引** 合計が **0** と表示されるようになった場合は、このセクションで理由と対応方法を説明します。

### 変更内容 {#the-change}

v1.9.0 より前は、レジ担当者が明細価格を値下げした場合（例: $18 → $16）、POS は注文を `subtotal = $18`、`total = $16` として記録していました。WooCommerce はその後 `discount_total = $2` を計算し、レシートやレポートではこれが割引として表示されていました。

このため、クーポンをさらに適用すると合計額が正しくなくなっていました。クーポンは元の $18 を基準に計算されるため、割引額が過大になり、顧客への請求額が少なくなっていました。

v1.9.0 以降、WCPOS は WooCommerce のセール価格の扱いに合わせています。レジで設定した価格**が**明細の小計になります。WooCommerce で「割引」としてカウントされるのはクーポンコードのみです（これはセール中の商品における WC の動作と一致します。セール価格も小計に含まれ、個別の割引明細は作成されません）。

### この変更による影響 {#what-this-means-for-you}

- **レシート**には、記録された通常価格、通常価格から販売価格への値引き額、およびそれらの値引きとクーポンを合わせた**節約額合計**を表示できます。価格による値引きは、WooCommerce の**割引**項目とは別に扱われます。
- **レポート**では、POS の明細価格変更のみを使用した場合、`discount_total = 0` と表示されます。カウントされるのはクーポン割引のみです。
- **クーポン**は、POS で値引きされた商品に重ねて適用した場合も正しく計算されるようになりました。
- **記録された通常価格と販売価格は** POS の明細メタデータ (`_woocommerce_pos_data`) に保存されるため、WooCommerce の割引合計を変更せずに、レシートで節約額を算出できます。

### レシートに元の価格と節約額を表示する {#showing-the-original-price-and-saving-on-receipts}

現在同梱されている価格表示対応テンプレートでは、商品がセール中の場合、またはレジで価格が変更された場合に、記録された通常価格と節約額を表示します。WCPOS で注文全体の金額を完全に計算できる場合は、**節約額合計**も表示されます。この合計は、通常価格からの値引きとクーポン割引を、どちらも重複してカウントせずに合算します。

ギャラリーからコピーしたテンプレートは編集可能なスナップショットであり、WCPOS の更新によって上書きされません。この変更より前にレシートを作成した場合は、次のいずれかを行ってください。

1. 同梱テンプレートの新しいコピーで**テンプレートを使用**をクリックする。または
2. 既存のテンプレートを、[レシートデータリファレンス](/receipts/receipt-data#displaying-regular-price-and-savings)のフィールドで更新します。

主なレシートフィールドは次のとおりです。

| フィールド | 意味 |
|-------|---------|
| `lines[].regular_price_display` | 記録された通常の単価 |
| `lines[].selling_price_display` | クーポン適用前の販売単価 |
| `lines[].unit_savings_display` | 商品1点あたりの節約額 |
| `lines[].line_regular_total_display` | 明細の通常価格合計 |
| `lines[].line_savings_display` | 明細の節約額合計 |
| `totals.sale_savings_total_display` | カタログのセールとレジでの価格変更による、通常価格から販売価格への節約額合計 |
| `totals.discount_total_display` | WooCommerceの割引合計。通常は現在の注文に適用されたクーポン |
| `totals.total_saved_display` | 通常価格からの節約額とWooCommerce割引の合計 |
| `totals.total_saved_complete` | 合計額が完全で、安全に表示できるかどうか |

カスタムテンプレートでは、履歴価格データが不完全な場合や値引き額がない場合に **合計節約額** 行が表示されないようにしてください。正確なパターンについては [レシートデータリファレンス](/receipts/receipt-data#totals) を参照してください。

WooCommerce レポートは変更されません。標準の割引合計には、現在のセール価格またはレジ価格による節約額ではなく、引き続きクーポンが含まれます。レシートには `totals.total_saved` を使用し、レポートで節約額を WooCommerce の割引としても追跡する必要がある場合は、引き続き [クーポン](/coupons) を使用してください。

## 既知の制限事項 {#known-limitations}

- **自動割引ルールはありません** — POS は「2個購入で1個無料」形式の自動割引に対応していません。体系的なプロモーションには WooCommerce クーポンを使用してください。
- **クイック割引はパーセントのみです** — 定額割引用の組み込みクイックボタンはありません。代わりに、負の手数料を使用するか、個別の価格を編集してください。
