WCPOSでは、レジ担当者がその場で注文に割引を適用するための方法がいくつか用意されています。パーセント割引のクイックボタン、明細行の価格の直接編集、注文レベルの割引手数料です。利用条件付きの事前設定プロモーションについては、[クーポン](/coupons)（Pro）を参照してください。

:::info v1.9.0での変更
POSでの価格変更がレシートやレポートに表示される方法がv1.9.0で変更されました。レシートの「割引」合計がゼロと表示されるようになった場合は、以下の[v1.9.0での変更点](#what-changed-in-v190)を参照してください。
:::

## クイック割引 {#quick-discounts}

クイック割引ボタンを使用すると、ワンタップで注文全体にパーセント割引を適用できます。

設定するには、カートの[表示設定](/pos/cart#display-settings)を開き、**クイック割引**フィールドにカンマ区切りのパーセンテージを入力します。たとえば、`5,10,15,20` と入力すると4つのショートカットボタンが作成されます。

クイック割引ボタンをタップすると、カート内のすべての明細行にパーセント割引が適用されます。

## 明細行割引 {#line-item-discounts}

カート内の個々の明細行の価格を直接変更できます：

1. 明細行の**価格**フィールドをクリックします
2. 新しい価格を入力します
3. **Enter** キーを押して確定します

価格のマッチング、スタッフ割引、個別の調整などに便利です。明細行の合計は、数量 × 新価格に基づいて自動的に更新されます。

カート内の商品の編集について詳しくは、[明細行](/pos/cart/line-items)を参照してください。

:::tip 商品の分割
同じ商品の一部に異なる割引を適用したい場合（例：3個は通常価格、2個は割引価格）、カートの[表示設定](/pos/cart#display-settings)で**分割**オプションを有効にすると、1つの明細行を複数の行に分けることができます。
:::

## 注文レベルの割引 {#order-level-discounts}

個々の商品ではなく注文全体に一律の割引を適用するには、**マイナス手数料**を追加します：

1. カート商品の下にある**手数料を追加**をクリックします
2. 名前を入力します（例：「スタッフ割引」）
3. 割引額を負の数で入力します（例：`-5.00`）

手数料は別の明細行として表示され、注文合計から差し引かれます。必要に応じて、三点メニューから手数料の税ステータスを編集できます。

## POS割引とWooCommerceクーポンの比較 {#pos-discounts-vs-woocommerce-coupons}

このページの割引はアドホック（その場限り）のもので、レジ担当者が会計時に適用します。WooCommerceの**クーポン**は、ルールやトラッキング機能を備えた事前設定のプロモーションです。どちらにもそれぞれの用途があります。

| | POS割引 | WooCommerceクーポン（Pro） |
|---|---|---|
| **適用方法** | クイック割引、明細価格の編集、またはマイナス手数料 | カートにクーポンコードを入力 |
| **設定場所** | レジ担当者がその場で設定 | **WP Admin → マーケティング → クーポン**で事前設定 |
| **トラッキング** | 明細価格として記録（[v1.9.0の変更点](#what-changed-in-v190)を参照） | WooCommerceレポートでクーポン割引として追跡 |
| **制限** | なし — レジ担当者が任意の価格を設定可能 | 使用回数制限、商品／カテゴリ制限、最低購入額、有効期限、メールルール |
| **最適な用途** | アドホックな価格調整、プライスマッチ | 体系的なプロモーション、追跡可能な割引 |

WooCommerceのレポートで割引の使用状況を追跡したり、使用回数の制限などのルールを適用したりする必要がある場合は、[クーポン](/coupons)を使用してください。一回限りの価格調整であれば、このページの割引機能の方がシンプルです。

## POSでの価格変更とクーポンの相互作用 {#how-pos-price-changes-interact-with-coupons}

レジ担当者が明細の価格を変更した場合（例：$18から$16に値下げ）、その後クーポンが適用されると、クーポンは元の価格（$18）ではなく、**POS割引後の価格**（$16）に対して計算されます。これにより、レジ担当者による値引きとクーポンが元の価格に対して二重に適用されることを防ぎます。

- POS割引が適用された商品は、WooCommerceでは「セール中」として扱われます。クーポンで**セール中の商品を除外**が有効になっている場合、通常のセール商品をスキップするのと同様に、POS割引済みの商品もスキップされます。開発者は `woocommerce_pos_item_is_on_sale` フィルターでこの動作をオーバーライドできます。
- クーポンを削除しても、明細はPOS割引後の価格のままになります。

:::note 開発者向けリファレンス
POS価格オーバーライドの保存方法と利用可能なフィルターの技術的な詳細については、[POS割引リファレンス](/reference/pos-discounts)を参照してください。
:::

## v1.9.0での変更点 {#what-changed-in-v1-9-0}

v1.8からアップグレードした後、レシートやレポートの**割引**合計が**0**と表示されるようになった場合、このセクションでその理由と対処方法を説明します。

### 変更内容 {#the-change}

v1.9.0以前は、レジ担当者が明細の価格を値下げした場合（例：$18 → $16）、POSは `subtotal = $18`、`total = $16` として注文を記録していました。WooCommerceはこれを基に `discount_total = $2` を算出し、この金額がレシートやレポートに割引として表示されていました。

この仕組みでは、クーポンを重ねて適用した際に合計金額が正しく計算されませんでした。クーポンが元の$18に対して計算されるため、割引が過剰に適用され、顧客への請求が不足する問題がありました。

v1.9.0以降、WCPOSはWooCommerceのセール価格の扱いに準拠するようになりました。レジで設定した価格が**そのまま**明細小計になります。WooCommerceではクーポンコードのみを「割引」としてカウントします（これはセール中の商品に対するWCの動作と同じで、セール価格は小計に組み込まれ、別途割引行は表示されません）。

### この変更の影響 {#what-this-means-for-you}

- **レシート**には、明細単位の価格変更に対する「割引」行が表示されなくなります。変更後の価格がそのまま販売価格となります。
- **レポート**では、POSの明細単位の価格変更のみが使用された場合、`discount_total = 0` と表示されます。クーポンによる割引のみがカウントされます。
- **クーポン**は、POSで割引された商品に重ねて適用しても正しく計算されるようになりました。
- **割引データは引き続き保存されます**。すべての注文の明細メタデータ(`_woocommerce_pos_data`)に記録されるため、必要に応じて過去のデータを参照できます。

### 割引の表示を復元したい場合 {#if-you-want-discount-visibility-back}

日次精算で「本日の割引合計」を確認する必要がある場合は、明細単位の価格編集ではなく[クーポン](/coupons)を使用することを推奨します。**マーケティング → クーポン**で「店長10%」や「ロイヤルティ $5」などのクーポンを作成し、カートで適用してください。レポートにそのまま反映されます。

また、クーポン割引とは別に明細単位の価格変更を表示するPOS専用の「合計節約額」機能の導入も検討しています。この機能がワークフローにとって重要な場合は、[お知らせください](https://wcpos.com/support)。

## 既知の制限事項 {#known-limitations}

- **自動割引ルールは未対応** — POSでは「2つ買うと1つ無料」のような自動割引には対応していません。構造化されたプロモーションにはWooCommerceのクーポンを使用してください。
- **クイック割引はパーセンテージのみです** — 固定金額の割引用のクイックボタンは組み込まれていません。代わりに、マイナス手数料を使用するか、個別の価格を編集してください。
