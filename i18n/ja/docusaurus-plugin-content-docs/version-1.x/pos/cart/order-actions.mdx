- 注文にカスタマーメモを追加するダイアログを開きます。
- カスタマーメモは、デフォルトで領収書、顧客のマイアカウントエリア、`WP管理画面 > WooCommerce > 注文 > 注文`の配送先住所の下に表示されます。

:::note
注文メモは顧客に表示されます。
:::

## 注文メタデータ

  <Image
    alt="POSの注文メタデータとJSONビュー"
    img="/img/order-meta.png"
    style={{maxHeight: 500}}
  />
  <p class="image-caption">POSの注文メタデータとJSONビュー</p>
</div>

- 注文の追加メタデータを管理するモーダルを開きます。
- 機能には以下が含まれます：
  - 注文通貨の変更
  - トランザクションIDの追加（外部決済端末からなど）
  - 注文のカスタムメタデータの追加

:::note
このモーダルには今後、他のWooCommerceプラグインとの統合を含む追加機能が追加される予定です。
:::

### WooCommerceのメタデータとは

メタデータは注文に関する追加情報を提供し、WooCommerce REST APIを介してアクセス可能です。開発者やプラグインはこのデータを使用して、サードパーティサービスの統合など、機能を拡張します。

### JSONビュー

- 注文の生のJSON表現を表示。デバッグや統合目的に役立ちます。

## サーバーに保存

- 開いているカートをサーバーに保存し、将来使用するために安全に保存できます。
- ステータス`pos-open`のWooCommerce注文を作成します。

### なぜ保存するのか？

POSで新しい注文が作成されると、最初はローカルにのみ存在します。
注文を保存することで、ローカルデータベースがクリアされても注文が残ります。

詳細は[オープン注文](/pos/cart/open-orders)を参照してください。

## 無効化

- 2つのシナリオ：
  - 未保存の注文：
    - サーバーに保存されていない場合、ローカルデータベースから完全に削除されます。
  - 保存済みの注文：
    - サーバーに保存されている場合、WooCommerceのゴミ箱フォルダに移動し、ローカルデータベースから削除されます。
- 無効化された注文を復元するには：
  - `WP管理画面 > WooCommerce > 注文 > ゴミ箱` に移動します。

## チェックアウト

- 注文をサーバーに保存し、チェックアウトプロセスを開始します。
- 支払いを完了するための[チェックアウト画面](/pos/checkout)を開きます。
