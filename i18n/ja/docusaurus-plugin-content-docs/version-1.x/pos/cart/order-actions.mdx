## 注文メモ {#order-note}

<div class="image-container">
  <Image
    alt="POSの注文メモ"
    img="/img/order-note.png"
    style={{maxHeight: 500}}
  />
  <p class="image-caption">POSの注文メモモーダル（左）とWP管理画面の注文メモ（右）</p>

- 注文に顧客メモを追加する対話ウィンドウが開きます。
- 顧客メモは、顧客に表示され、デフォルトでレシートに、顧客のマイアカウントエリアに、そして`WP Admin > WooCommerce > Orders > Order`の配送先住所の下に表示されます。

:::note
注文メモは顧客に表示されます。
:::

## 注文メタ {#order-meta}

<div class="image-container">
  <Image
    alt="POSの注文メタおよびJSONビュー"
    img="/img/order-meta.png"
    style={{maxHeight: 500}}
  />
  <p class="image-caption">POSの注文メタおよびJSONビュー</p>
</div>

- 注文に関する追加のメタデータを管理するためのモーダルが開きます。
- 機能には以下が含まれます：
  - 注文の通貨を変更する。
  - 取引ID（例：外部支払い端末から）を追加する。
  - 注文のカスタムメタデータを追加する。

:::note
このモーダルには、他のWooCommerceプラグインとの統合を含む追加機能が時間の経過とともに追加されます。
:::

### WooCommerceにおけるメタデータとは？ {#what-is-meta-data-in-woocommerce}

メタデータは、注文に関する追加情報を提供し、WooCommerce REST APIを介してアクセス可能です。開発者やプラグインはこのデータを使用して機能を拡張し、サードパーティサービスとの統合などを実現します。

### JSONビュー {#json-view}

- 注文の生のJSON表現を表示し、デバッグや統合目的に役立ちます。

## サーバーに保存 {#save-to-server}

- 開いているカートをサーバーに保存することができ、将来の使用のために安全に保存され、アクセス可能になります。
- ステータス`pos-open`のWooCommerce注文を作成します。

### なぜ保存するのか？ {#why-save}

POSで新しい注文が作成されると、それはローカルにのみ存在します。 
注文を保存することで、ローカルデータベースがクリアされても、注文が持続します。

保存された注文の管理についての詳細は[オープン注文](/pos/cart/open-orders)をご覧ください。

## 無効化 {#void}

- 2つのシナリオ：
  - 保存されていない注文：
    - 注文がサーバーに保存されていない場合、ローカルデータベースから永久に削除されます。
  - 保存された注文：
    - 注文がサーバーに保存されている場合、WooCommerceのゴミ箱フォルダに移動し、ローカルデータベースから削除されます。
- 無効化された注文を回復するには：
  - `WP Admin > WooCommerce > Orders > Trash`に移動します。

## チェックアウト {#checkout}

- 注文をサーバーに保存し、チェックアウトプロセスを開始します。
- 支払いを完了するために[チェックアウト画面](/pos/checkout)を開きます。
