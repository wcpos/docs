支払いを正常に処理した後、レシートモーダルが表示されます。これは、完了した注文の概要を示し、レシートを印刷するか、メールで送信するオプションを提供します。

{/* Screenshot: receipt-modal.png */}

## レシート内容

レシートには以下が表示されます：

### 店舗情報
- **店舗名**
- **住所**
- **電話番号**
- **メールアドレス**
- **ウェブサイト**
- **ロゴ**（設定されている場合）

### レシートヘッダー
- **レシートタイプ**（例： "税務レシート"）
- **注文番号**
- **日時**
- **キャッシャー名**
- **支払い方法**

### 注文詳細
- **アイテム明細**には：
  - 商品名
  - バリエーション属性（該当する場合）
  - 数量
  - 価格
  - 合計

### 合計
- **小計**
- **税金**（該当する場合）
- **合計**
- **支払い金額**（現金の場合）
- **お釣り**（現金の場合）

### 顧客情報
- **請求先住所**
- **配送先住所**（異なる場合）

## レシートアクション

### 閉じる
レシートモーダルを閉じて、POSに戻ります。

### レシートをメールする
レシートを顧客のメールアドレスに送信します：

1. **レシートをメールする**をクリック
2. メールアドレスを確認
3. WooCommerceのメールシステムを介してレシートが送信されます

顧客は注文に関連付けられたメールアドレスを持っている必要があります。

### レシートを印刷する
レシートの物理コピーを印刷します：

1. **レシートを印刷する**をクリック
2. ブラウザの印刷ダイアログが開きます
3. レシートプリンターを選択し印刷する

:::tip レシートプリンター
WCPOSは、ブラウザからアクセス可能な任意のプリンターと連携します。最良の結果を得るためには、以下を使用してください：
- 専用のレシートプリンター（サーマルプリンターが一般的です）
- プリンター設定で用紙サイズを設定
- 迅速な印刷のために印刷プリセットを設定
:::

## 自動レシートオプション

[カート表示設定](/pos/cart#display-settings)で次のように設定できます：

- **チェックアウト後にレシートを自動表示** - 支払い後にレシートモーダルがすぐに開きます
- **チェックアウト後にレシートを自動印刷** - レシートが表示されたときに自動的に印刷がトリガーされます

これらのオプションは、高ボリュームのチェックアウトワークフローを迅速化します。

## レシートのカスタマイズ

レシートの外観はWordPressのテンプレートで制御されています：

`WP Admin > POS > テンプレート`

カスタマイズの詳細については、[レシートテンプレート](/templates/receipt-templates)を参照してください：

- 店舗情報の表示
- ロゴとブランド
- レイアウトとフォーマット
- 含める情報

## レシートの再印刷

過去の注文のレシートを再印刷するには：

1. [注文画面](/orders)に移動（Pro機能）
2. 注文を見つける
3. 注文を開くのをクリック
4. レシートオプションを使用して再印刷
