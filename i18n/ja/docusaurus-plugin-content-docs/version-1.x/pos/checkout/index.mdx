販売を完了する準備ができたら、**チェックアウト**ボタンをクリックしてチェックアウトモーダルを開きます。ここで支払いを処理し、注文を完了します。

{/* Screenshot: checkout-modal.png */}

## チェックアウトモーダルの概要 {#checkout-modal-overview}

チェックアウトモーダルには次の情報が表示されます。

- **注文番号** - WooCommerce の注文 ID
- **支払金額** - 支払いが必要な合計金額
- **チェックアウト設定**ボタン - 表示の問題をトラブルシューティングします
- **レジ担当者** - 注文を処理している担当者
- **顧客** - この注文の顧客（クリック可能なリンク）
- **クーポンを追加**ボタン - 割引コードを適用
- **注文概要** - 商品、数量、合計
- **支払い方法** - 利用可能な支払いオプション
- **キャンセル / 支払いを処理**ボタン

## 支払い方法 {#payment-methods}

### 無料版で利用可能 {#available-in-free-version}

WCPOS の無料版には、2 つの支払いゲートウェイが含まれています:

- **現金** - 預かり金額とお釣り計算機能付き
- **カード** - 外部カード端末用

### 追加ゲートウェイ (Pro) {#additional-gateways-pro}

:::info Pro 機能
追加の支払いゲートウェイには [WCPOS Pro](/getting-started/pro-license) が必要です。
:::

Pro では、次を有効にできます:

- **Stripe Terminal** - Stripe カードリーダーとの直接連携
- **SumUp Terminal** - SumUp カードリーダーとの連携
- **カスタムゲートウェイ** - 独自の支払い連携を作成できます

ゲートウェイの設定について詳しくは、[支払い](/payment)を参照してください。

### 支払い方法の選択 {#selecting-a-payment-method}

支払い方法をクリックして選択します。フォームが更新され、関連するフィールドが表示されます:

**現金:**

- **預かり金額** - 顧客から受け取る金額を入力します
- **お釣り** - 返すお釣りが自動計算されます

{/* スクリーンショット: checkout-cash-payment.png */}

**カード:**

- 外部カード端末で支払いを処理します
- 「支払いを処理」をクリックして完了します

## クーポン {#coupons}

カートには、合計の上に **クーポンを追加** 入力欄があります（Pro のみ）。コードを入力するか、説明で検索します。クーポンはローカルで検証され、削除可能なピルとして表示されます。複数のクーポンは順番に適用されます。

検索、ピル、順次割引、検証エラーと解決方法の表を含む全体のワークフローについては、**[レジでクーポンを適用する](/coupons/applying-coupons)**を参照してください。クーポンの種類、検証ルール、WooCommerce での設定については、[クーポン](/coupons)を参照してください。

## 支払いの処理 {#processing-payment}

1. 支払い方法を選択します
2. 現金の場合は、預かり金額を入力します
3. **支払いを処理** をクリックします
4. 注文が完了し、[レシート](/receipts/at-checkout)が表示されます

## チェックアウト設定（トラブルシューティング） {#checkout-settings-troubleshooting}

チェックアウトモーダルでは、iframe/webview を使用して WooCommerce の注文支払いページを表示します。これにより WooCommerce の既存の決済インフラを活用できるため、WooCommerce で動作する決済ゲートウェイは POS でも動作するはずです。

ただし、テーマやプラグインのスクリプトが干渉する場合があります。トラブルシューティングを行うには、**チェックアウト設定**をクリックします。

### すべてのスタイルとスクリプトを無効化 {#disable-all-styles-and-scripts}

:::warning 最終手段
これは最終手段であり、テスト時、または開発者が影響を十分に理解しているまれな場合にのみ使用してください。すべての wp_head スクリプトを無効化すると、決済ゲートウェイの展開/折りたたみに必要な WooCommerce スクリプトまで削除され、決済機能が動作しなくなる可能性があります。
:::

- **wp_head を無効化** - WordPress ヘッダーからすべてのスクリプト/スタイルを削除します
- **wp_footer を無効化** - WordPress フッターからすべてのスクリプト/スタイルを削除します

### 選択したスタイルを無効化 {#disable-selected-styles}

表示の問題を引き起こす可能性がある CSS を選択的に無効化します。

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- など。

### 選択したスクリプトを無効化 {#disable-selected-scripts}

決済ゲートウェイと干渉する可能性がある JavaScript を選択的に無効化します。

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- など。

:::tip
決済ゲートウェイが正しく表示されない場合:

1. まずテーマスタイルを無効化してみてください
2. 次に、不要なWooCommerceスクリプトを無効化してみてください
3. 決済ゲートウェイに必要なスクリプトを無効化しないよう注意してください
   :::

## キャンセル {#cancel}

**キャンセル**をクリックすると、注文を完了せずにチェックアウトモーダルを閉じます。注文は未確定のカートとして残ります。
