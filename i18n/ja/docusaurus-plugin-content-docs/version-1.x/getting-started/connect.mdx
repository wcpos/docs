この画面はデスクトップアプリとモバイルアプリでのみ表示されます。ウェブユーザーは `yourdomain.com/pos` でPOSに直接アクセスし、WordPressの資格情報でログインします。
:::

## 接続画面の概要

WCPOSのデスクトップまたはモバイルアプリを開くと、接続画面が表示されます。ここでは、ストアの接続およびユーザーログインを管理します。

{/* スクリーンショット: connect-screen.png */}

## 新しいストアの追加

1. テキストフィールドにWooCommerceストアのURLを入力します（例: `https://mystore.com`）
2. **接続**をクリックします
3. WordPressの資格情報でログインするためにリダイレクトされます
4. 成功したログイン後、アプリに戻ります

## 複数のストア

必要に応じて、複数のWooCommerceストアに接続できます。各ストアは接続画面に別々のカードとして表示され、以下を示します。

- **ストア名**とファビコン
- **ストアURL**
- 当該ストアの**ログイン中のユーザー**

これは、複数の場所やビジネスを管理している場合に便利です。

## ストアごとの複数のユーザー

各ストアには、同時に複数のユーザーがログインできます。これは以下のために便利です。

- **シフト交代** - 新しいキャッシャーが前の人がログアウトする前にログインできます
- **多層レジ設定** - 異なるデバイスで異なるキャッシャー
- **迅速な切り替え** - ユーザーアカウント間での簡単な切り替え

### ユーザーの追加

ストアの「ログイン中のユーザー」の隣にある <Icon name="plus" /> ボタンをクリックして、そのストアに別のユーザーを追加します。

### ユーザーの切り替え

ユーザーのバッジ（例: "ブレンダ"）をクリックして、そのユーザーとしてPOSを開きます。

### ユーザーの削除

ユーザーバッジの**×**をクリックして、そのユーザーをストアからログアウトします。

## ストアの削除

ストアカードの赤い **×** ボタンをクリックして、リストから削除します。これにより、すべてのユーザーがログアウトし、アプリからストア接続が削除されます。

## デモストア

画面の下部に「デモストアに入る」というリンクが表示される場合があります。これは、WCPOSの機能を試してみるためにデモのWooCommerceストアに接続し、自分のストアデータには影響を与えません。

## トラブルシューティング

### ストアに接続できませんか？

- WordPressサイトにWCPOSプラグインがインストールされ、有効化されているか確認します
- 正しいURLを使用しているか確認します（サイトがSSLを使用している場合は `https://` を含める）
- WooCommerce REST APIにアクセス可能か確認します
- ユーザーアカウントにPOSアクセス権限があるか確認します

### 接続が失敗しますか？

- まず、ウェブブラウザで `yourdomain.com/pos` にアクセスして、プラグインが機能しているか確認してください
- サイトのエラーログに問題がないか確認します
- サーバーが [最小要件](/getting-started/installation) を満たしているか確認します
