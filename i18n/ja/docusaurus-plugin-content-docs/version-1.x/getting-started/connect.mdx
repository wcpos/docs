## 接続画面の概要

WCPOSデスクトップまたはモバイルアプリを開くと、接続画面が表示されます。ここでストア接続とユーザーログインを管理します。

{/* Screenshot: connect-screen.png */}

## 新しいストアの追加

1. テキストフィールドにWooCommerceストアのURLを入力します（例：`https://mystore.com`）
2. **接続**をクリック
3. WordPressの認証情報でログインするページにリダイレクトされます
4. ログイン成功後、アプリに戻ります

## 複数のストア

必要な数のWooCommerceストアに接続できます。各ストアは接続画面に別のカードとして表示され、以下が示されます：

- **ストア名**とファビコン
- **ストアURL**
- そのストアの**ログイン中のユーザー**

これは、複数の拠点やビジネスを管理する場合に便利です。

## ストアごとの複数ユーザー

各ストアには、同時に複数のユーザーがログインできます。これは以下の場合に便利です：

- **シフト交代** - 前のユーザーがログアウトする前に新しいレジ担当者がログイン
- **マルチレジセットアップ** - 異なるデバイス上の異なるレジ担当者
- **簡単な切り替え** - ユーザーアカウント間を簡単に切り替え

### ユーザーの追加

"ログイン中のユーザー"の横にある <Icon name="plus" /> ボタンをクリックして、そのストアに別のユーザーを追加します。

### ユーザーの切り替え

ユーザーバッジ（例：「Brenda」）をクリックして、そのユーザーとしてPOSを開きます。

### ユーザーの削除

ユーザーバッジの**×**をクリックして、そのユーザーをストアからログアウトさせます。

## ストアの削除

ストアカードの赤い**×**ボタンをクリックして、リストからそのストアを削除します。これにより、すべてのユーザーがログアウトし、アプリからストア接続が削除されます。

## デモストア

画面の下部に「デモストアに入る」リンクが表示される場合があります。これにより、独自のストアデータに影響を与えることなく、WCPOSの機能を試すためのデモWooCommerceストアに接続できます。

## トラブルシューティング

### ストアに接続できない場合

- WCPOSプラグインがWordPressサイトにインストールされ、有効化されていることを確認
- 正しいURLを使用していることを確認（サイトがSSLを使用している場合は`https://`を含める）
- WooCommerce REST APIにアクセスできることを確認
- ユーザーアカウントにPOSアクセス権限があることを確認

### 接続が繰り返し失敗する場合

- まずWebブラウザで`yourdomain.com/pos`にアクセスし、プラグインが機能していることを確認
- サイトのエラーログで問題がないか確認
- サーバーが[最小要件](/getting-started/installation)を満たしていることを確認
