## 機能をリクエストまたは投票する {#request-or-vote}

私たちが作るものに影響を与える最良の方法は、要望を伝えることです。

- **[Discord](https://wcpos.com/discord)** — 私たちやコミュニティに最も早く連絡できる場所です。`#general` または `#forum` で利用ケースを共有してください。
- **[GitHub Discussions](https://github.com/orgs/wcpos/discussions)** — 機能を提案するか、既存の提案に「+1」と利用ケースを追加してください。多くのロードマップ項目はディスカッションから始まりました。

リクエストするときは、*何をしようとしているのか*、そして*なぜ必要なのか*を教えてください。具体的な利用ケースは、機能名だけよりも設計に大きく影響します。

## 現在取り組んでいること {#what-were-working-on}

開発中のテーマを大まかにまとめたものです。各項目の詳細で最新の状態は、[ロードマップボード](https://github.com/orgs/wcpos/projects/4)で確認してください。

### レジまわり {#at-the-register}

- **在庫管理** — 在庫切れの商品をカートでブロックし、チェックアウト時に在庫を検証して売り越しを防ぎます
- **チェックアウト条件** — 注文を完了する前に、顧客、注文メモ、カスタムフィールドなどを必須にできます
- **分割・一部支払い** — 1つの注文を複数の支払い方法で支払えます（例: 一部を現金、一部をカード）
- **現金管理** — 正確な日次締めレポートのための開始釣銭と正式なシフト開始/終了

### レポートとドキュメント {#reporting-and-documents}

- **レポートテンプレート** — A4 またはサーマル印刷に対応した、設定可能な Z レポートとシフトサマリー
- **メールテンプレート** — 標準の注文メールとは別の、POS 専用メール/レシートレイアウト

### より多くの商品タイプの販売 {#selling-more-product-types}

より多くの WooCommerce 拡張機能との互換性 — **Product Add-Ons**、**Product Bundles**、**Composite Products**、**Bookings**、**Product Options**、および**埋め込み（計量）バーコード**。

### ハードウェアとディスプレイ {#hardware-and-displays}

- **さらに多くのカードリーダー**とリーダー直接連携
- **顧客向けディスプレイ** — 注文内容と合計を顧客に表示する2台目の画面
- **注文通知** — 新しいオンライン注文のリアルタイム通知（レストランに便利）
