WCPOS（ポイントオブセール）システムの重要な側面の一つは、そのデータ管理アプローチです。これは、大量の製品データを効率的に扱う方法を提供します。このガイドでは、WCPOS内での製品のダウンロードと同期方法について説明します。

## 製品のローカルストレージ {#local-storage-of-products}

WCPOSはあなたのWooCommerceストアから製品を取得し、それをデバイスにローカルに保存します。このアプローチでは、一度製品がダウンロードされると、繰り返し取得する必要はありません。POSを使用し続けると、WooCommerceストアからすべての製品を段階的にダウンロードし、製品のローカルデータベースを作成します。

## 製品のバッチダウンロード {#batch-downloading-of-products}

ほとんどのストアでは、一度に約50製品以上をダウンロードしようとすると、サーバーへの負荷が高まり、応答時間が遅くなったり、サーバーがクラッシュすることがあります。これを防ぐために、WCPOSは通常、一度に約50製品の小さなバッチで製品をダウンロードするように設計されています。このプロセスは、すべての在庫がローカルに保存されるまで続き、大量のデータを管理しながらサーバーに過度の負荷をかけないことを保証します。

## 製品検索の使用 {#using-the-product-search}

製品のバッチダウンロードを開始する効果的な方法は、製品検索機能を使用することです。製品検索を実行するたびに、POSシステムは別のバッチの製品を取得して保存します。この戦略は、新しい製品を段階的にダウンロードするのに役立つだけでなく、より多くの製品がローカルに保存されることで検索機能の速度と効率も向上させます。

## 結論 {#conclusion}

すべての製品が一度にPOSシステムに表示されるわけではないことを理解することが重要であり、これは完全に正常なことです。時間が経つにつれて、製品検索やその他のPOS操作を使用することで、すべての製品をダウンロードします。WCPOSの運用についての詳細な情報は、[アーキテクチャ](/reference/architecture)を参照してください。

WCPOSは、サーバーの負荷と運用効率を最適化するように設計されています。段階的な製品ダウンロード機能は、スムーズで効率的、かつユーザーフレンドリーな小売管理体験を保証します。

:::tip
通常の同期では解決できない持続的な同期の問題が発生している場合は、[すべてのローカルデータをクリア](/support/troubleshooting/clear-local-data)して、ストア全体の新しいダウンロードを強制できます。
:::

## よくある質問 {#faq}

<Accordion>
  <AccordionItem question="どうすればもっと製品をロードできますか?">

  POSでの製品の同期は、いくつかの方法でトリガーされることがあります。

  - POSが最初にロードされたとき。
  - 製品検索を実行するたびに。
  - 例えば、カテゴリやタグでフィルターを適用します。
  - 製品リストの下部にスクロールすること。
  - 手動で同期ボタンを押す（短押しで同期、**長押し**でクリアとリフレッシュオプション）。
  - さらに、POSは10分ごとに自動的に同期されます。

  すべての製品がダウンロードされると、今後の同期では前回の同期以降に更新された製品のみが取得されます。

  </AccordionItem>

  <AccordionItem question="下にスクロールしてもすべての製品が表示されません。">

  すべての製品が表示されない理由はいくつか考えられます。

  - POSはデフォルトで在庫切れ品を非表示にします。在庫切れのアイテムを表示したい場合は、製品表示設定でこの設定を変更できます。
  - WooCommerce REST APIは、標準のWooCommerce製品タイプ（シンプル、バリアブル、グループ、外部）にのみ対応しています。カスタム製品タイプを使用している場合、それらがPOSに表示されない可能性があります。

  </AccordionItem>
</Accordion>
