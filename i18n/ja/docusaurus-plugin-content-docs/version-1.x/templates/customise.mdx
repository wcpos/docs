レシートの見た目を変更するには、3つの方法があります。必要な機能を満たす最も簡単な方法を選んでください。ほとんどの店舗では最初の方法で十分です。

## 3つのカスタマイズ方法 {#three-ways-to-customise}

<FeatureGrid>
  <FeatureCard icon="grid" title="1. 別のテンプレートを選ぶ">
    ギャラリーにある既製のテンプレートを使用します。**コードは一切不要です。** 最適な用途：レイアウトの変更、価格の非表示、A4インボイス、キッチンチケット。
  </FeatureCard>
  <FeatureCard icon="sparkles" title="2. AIに調整を依頼する">
    テンプレートをChatGPTまたはClaudeに貼り付けて、希望する変更内容を説明します。**コーディングスキルは不要です** — 自然な言葉で説明するだけです。最適な用途：文言、色、レイアウトの微調整。
  </FeatureCard>
  <FeatureCard icon="code" title="3. 手動で編集する">
    アプリ内エディターでテンプレートを直接変更できます。最適な用途：細かい調整が必要な場合、またはHTMLの知識がある場合。
  </FeatureCard>
</FeatureGrid>

3つの方法はすべて同じ場所から始めます：**WP Admin → POS → テンプレート**。ページは2つの部分で構成されています — 上部に**使用中のテンプレート**（現在使用しているもの）、その下に**テンプレートギャラリー**（スターターライブラリ）があります。

## オプション 1 — 別のテンプレートを選ぶ {#option-1-pick-a-different-template}

最も簡単な方法で、ほとんどのニーズに対応できます。

<Steps>
  <StepCard number="1" title="テンプレートギャラリーを開く">

WP Admin で **POS → テンプレート** に移動します。*マイテンプレート* を通り過ぎて **テンプレートギャラリー** セクションまでスクロールします — これがスターターライブラリです。

  </StepCard>
  <StepCard number="2" title="閲覧とプレビュー">

**カテゴリ**（レシート、請求書、ギフトレシート、キッチンチケット、見積書 / 発注書）、**フォーマット**（HTML はブラウザ印刷用、ESC/POS はサーマルプリンター用）、または**方向**（左から右 または 右から左）でフィルタリングできます。カードのサムネイル、またはフッターの **プレビュー** リンクをクリックすると、店舗の実データを使用したライブプレビューが表示されます。

  </StepCard>
  <StepCard number="3" title="使用する" isLast>

カードの **テンプレートを使用** をクリックします。WCPOS が編集可能なコピーを作成し、ページ上部の **マイテンプレート** に追加します。行の **有効** トグルをオンにするとレシートで使用が開始されます。行のグリップハンドルをドラッグして順序を変更できます。複数のテンプレートを同時に有効にでき、レジで担当者が選択できます。

  </StepCard>
</Steps>

:::tip 「テンプレートを使用」は既存のテンプレートを置き換えません
**テンプレートを使用** をクリックすると、常に新しいコピーが作成されます。ギャラリーの元のテンプレートはそのまま残るため、いつでも別の出発点を選び直すことができます。複数のレシートテンプレートが有効な場合、レシート画面にドロップダウンが表示され、担当者がその場で切り替えることができます。
:::

### 同梱テンプレート {#the-bundled-templates}

| テンプレート | 形式 | 用途 |
|---|---|---|
| **標準レシート** | HTML | デフォルト — ロゴ、商品、合計、支払い。ほとんどの店舗に対応 |
| **標準レシート（RTL）** | HTML | 標準と同内容で、アラビア語／ヘブライ語／ペルシア語／ウルドゥー語向けに右から左へ反転 |
| **ミニマル／モダン** | HTML | 標準と同じ情報を、より少ない縦幅にまとめたレイアウト |
| **詳細レシート** | HTML | 完全な税務請求書 — SKU列、単価、税率別の内訳、住所を含む |
| **ギフトレシート** | HTML | 商品のみ — 価格は非表示。ギフトメッセージと返品ポリシーを含む |
| **請求書** | HTML | A4／レターサイズの全ページ請求書。未払い注文には「支払い方法」パネルを追加 |
| **納品書** | HTML | 倉庫向け — 商品＋数量、配送先のみ、価格なし |
| **見積書** | HTML | 価格と条件を記載した販売前の書類 — 支払いセクションなし |
| **ナローレシート** | HTML | 狭い用紙またはHTML対応サーマルプリンター向けの等幅レシート |
| **シンプルサーマルレシート（58mm）** | Thermal | すっきりした58mmサーマルレイアウト |
| **シンプルサーマルレシート（80mm）** | サーマル | シンプルな80mmサーマルレイアウト — 最も一般的 |
| **シンプルサーマルレシート 80mm（RTL）** | サーマル | 80mm用のRTL版。アラビア語コードページ対応プリンターが必要 |
| **詳細サーマルレシート（58mm / 80mm）** | サーマル | 税内訳、住所、返金、支払い、利用規約、バーコードを追加 |
| **キッチンチケット** | サーマル | 商品のみ、大きいフォント、価格なし — 調理場向け |

同梱テンプレートのほとんどは**店舗の税設定に自動的に対応します** — 税込設定の店舗では総額と「税込」の行が表示され、税抜設定の店舗では税抜価格に税額が別行として追加されます。**詳細**ファミリーは、設定に関係なく常に完全な税内訳を表示します。

## 店舗ごとの割り当て {#per-store-assignments}

複数の[店舗](/stores)がある場合（Pro）、各店舗はサイト全体のデフォルトとは別に、**独自のテンプレート選択と並び順**を設定できます。近くのカフェには別のロゴと住所の小型サーマルレシートを、倉庫には納品書を、本店には標準レシートを設定できます — すべて同じテンプレートギャラリーから選択できます。

**WP Admin → POS → 店舗** から設定します。設定したい店舗を開いてください。**店舗の編集**ページには、*「店舗固有のレシートテンプレート」*トグルのある**レシートテンプレート**セクションがあります：

- **トグルオフ** *（デフォルト）* — 店舗はメインの **POS → テンプレート** ページからサイト全体のテンプレートリストを継承します。
- **トグルオン** — 店舗はサイト全体のデフォルトとは別に、独自のテンプレート選択と並び順を設定できます。ドラッグハンドルによる並べ替えも同様に機能します。

同じ店舗の編集ページには、各店舗の**レターヘッド**も設定されています（ロゴ、住所、連絡先、および*レシートメッセージ*ブロック — 結語、返品ポリシー、フッター）。同梱テンプレートはこれらの店舗ごとのフィールドから情報を取得するため、1つの「標準レシート」テンプレートで店舗ごとに異なるブランディングを表示できます。

レジ担当者が店舗にサインインすると、その店舗の有効なテンプレートのみがレシートのドロップダウンに表示されます。

:::tip サイト全体と店舗ごとの設定
WP Admin の**テンプレート**ページでは、サイト全体のデフォルトを設定します。店舗ごとの上書き機能を使えば、同じテンプレート（例：標準レシート）でも店舗ごとに異なるブランディングを適用したり、特定の店舗だけ別のレイアウトを使用したりできます。すべての店舗で同じテンプレートを使用する場合は、店舗ごとの割り当てを空のままにしておけば、サイト全体のデフォルトが適用されます。
:::

## オプション 2 — AIに調整を依頼する {#option-2-ask-ai-to-tweak-it}

ギャラリーのテンプレートがほぼ希望通りだけれど完全ではない場合、AIアシスタントを使えば数分で変更できます。HTMLの知識は必要ありません。

<Steps>
  <StepCard number="1" title="テンプレートをコピーする">

**WP Admin → POS → テンプレート**で編集したいテンプレートを開き、エディタをクリックして左側のテキストをすべて選択します（Ctrl/Cmd + A）。コピーしてください。

  </StepCard>
  <StepCard number="2" title="ChatGPTまたはClaudeに貼り付ける">

[ChatGPT](https://chat.openai.com)または[Claude](https://claude.ai)を開きます。テンプレートを貼り付けてから、変更したい内容を自然な言葉で書いてください：

  </StepCard>
  <StepCard number="3" title="変更内容を説明する">

AIに変更してほしい内容を具体的に伝えてください。うまくいく例：

- *「店舗名をもっと大きくして中央揃えにして。」*
- *「下部にイタリック体でお礼のメッセージを追加してください。」*
- *「顧客名を非表示にして、注文番号の下に電話番号を表示してください。」*
- *「バーコードを返品ページにリンクするQRコードに変更してください。」*
- *「店舗名の下に『1987年創業の家族経営』というキャッチフレーズを追加してください。」*

AIが修正済みのテンプレートを返します。

  </StepCard>
  <StepCard number="4" title="貼り付ける" isLast>

AIの回答をコピーします。WCPOSテンプレートエディタに戻り、すべてを選択（Ctrl/Cmd + A）して新しいバージョンを貼り付け、**更新**をクリックします。右側のプレビューが更新され、変更結果を確認できます。意図した表示にならない場合は、何が問題だったかを説明してAIに修正を依頼してください。

  </StepCard>
</Steps>

:::tip ベストプラクティス
ギャラリーで**テンプレートを使用**をクリックするたびに、新しい編集可能なコピーが作成されるため、オリジナルは安全に保持されます。試行錯誤する場合は、同じギャラリーテンプレートを複数回使用できます。コピーに名前を付け*（レシート v1、レシート v2）*、決定するまで切り替えてお試しください。
:::

:::info 変数について
`{{store.name}}` や `{{order.number}}` などは、実際のデータの**プレースホルダー**です。AIはこれらを理解しているため、詳しく知る必要はありません。利用可能なすべてのプレースホルダーを確認したい場合は、[レシートデータリファレンス](/templates/receipt-data)を参照してください。
:::

## オプション 3 — 手動で編集する {#option-3-edit-it-by-hand}

HTMLの知識がある場合（または開発者と協力している場合）、アプリ内エディタでテンプレートを直接編集できます。エディタにはライブプレビュー、シンタックスハイライト、検索可能なフィールドピッカー、元に戻す／やり直し、検索と置換の機能があります。

エンジンを選択してください：

- **[HTMLテンプレート](/templates/html-templates)** — Mustache形式の`{{variable}}`プレースホルダーを使用します。クライアントサイドでレンダリングされ、オフラインでも動作します。**ほとんどの店舗におすすめです。**
- **[サーマルテンプレート](/templates/thermal-templates)** — ESC/POSサーマルプリンター用のXMLです。同じテンプレートから画面プレビューとプリンター出力の両方が生成されます。
- **[レシートデータリファレンス](/templates/receipt-data)** — 使用可能なすべてのプレースホルダーをセクション別にまとめています。

:::note レガシーPHPテンプレート
以前テーマ内のPHPファイル(`yourtheme/woocommerce-pos/receipt.php`)でレシートをオーバーライドしていた場合、その方法は引き続き使用できます。現在はギャラリーで**レガシーPHPテンプレート**として表示され、新しいロジックレスエンジンやサーマルエンジンと並んで配置されています。新しいカスタマイズにはギャラリーまたはアプリ内エディタの使用を推奨します。オフラインで動作し、ライブプレビューが可能で、サーバーへのラウンドトリップも不要です。
:::

## よくあるカスタマイズ {#common-customisations}

よく寄せられる質問への簡単な回答です。

<Accordion>
  <AccordionItem question="店舗ロゴを追加するには？">

ロゴはテンプレート自体ではなく、店舗設定から取得されます。**WP Admin → POS → 設定 → 店舗** に移動し、店舗を編集してロゴをアップロードしてください。ロゴを表示する同梱テンプレートはすべて、そのロゴを自動的に使用します。

テンプレート内でロゴの*表示位置*を変更したい場合は、テンプレートを編集して `{{#store.logo}}<img src="{{store.logo}}" ...>{{/store.logo}}` ブロックを目的の位置に移動してください。

  </AccordionItem>

  <AccordionItem question="フッターのテキスト（例：「お買い上げありがとうございます！」）を変更するには？">

2つの方法があります：

1. **最も簡単な方法** — **WP Admin → POS → 設定 → 店舗 → 店舗詳細 → レシートフッター / 個人メモ** で、すべてのレシートに対して一度だけ設定します。同梱テンプレートはこの設定を自動的に反映します。フッターが設定されていない場合は、*「お買い上げありがとうございます！」* のようなデフォルトメッセージが表示されます。
2. **単一のテンプレートで変更する場合** — テンプレートを編集し、フッターテキストを直接置き換えます。`{{store.personal_notes}}` またはお礼メッセージの行を探してください。

  </AccordionItem>

  <AccordionItem question="店舗名の下にキャッチフレーズやスローガンを追加するには？">

テンプレートを編集し、`{{store.name}}` の下に行を追加します：

```html
<div style="font-style: italic; color: #6b7280;">Family-owned since 1987</div>
```

サーマルテンプレートの場合：

```xml
<align mode="center"><text>Family-owned since 1987</text></align>
```

  </AccordionItem>

  <AccordionItem question="（ギフトレシート用に）価格を非表示にするには？">

ギャラリーの**ギフトレシート**カードで**テンプレートを使用**をクリックします。価格や合計をすべて非表示にし、商品、SKU、属性、ギフトメッセージのみを表示します。編集は不要です。

独自の価格なしレシートを作成する場合は、任意のテンプレートをコピーし、`{{...total...}}`、`{{...price...}}`、`{{#totals}}...{{/totals}}` ブロックを削除してください。

  </AccordionItem>

  <AccordionItem question="バーコードをQRコードに変更するには？">

テンプレート内の `<barcode>` 要素を見つけて、 `type` 属性：

```html
<!-- Default: Code 128 barcode of the order number -->
<barcode type="code128">{{order.number}}</barcode>

<!-- Switch to a QR code -->
<barcode type="qrcode">{{order.number}}</barcode>

<!-- Or encode something else, like a returns URL -->
<barcode type="qrcode">https://example.com/returns?order={{order.number}}</barcode>
```

同じ `<barcode>` 構文はHTMLテンプレートとサーマルテンプレートの両方で使用できます。その他のサポートされているタイプには `ean13`、 `ean8`、 `upca`、 `pdf417`、および [bwip-jsがサポートするすべて](https://github.com/metafloor/bwip-js/wiki/Supported-Barcode-Types)があります。

  </AccordionItem>

  <AccordionItem question="特定のプリンターに別のテンプレートを送信するには？">

POSアプリで**設定 → 印刷ルーティング**に移動します。有効なテンプレートごとにプリンターのドロップダウンが表示されます。使用するプリンターを選択するか、**自動**のままにしてください。

- **自動**はテンプレートとプリンターを自動的にマッチングします — サーマルテンプレートはサーマルプリンターに、HTMLテンプレートはシステムの印刷ダイアログに送信されます。
- **特定のプリンター**を指定すると自動設定が上書きされ、そのテンプレートは常に指定したプリンターに送信されます。
- 印刷時に、レシート画面のプリンタードロップダウンから上記いずれの設定もレジ担当者が変更できます。

ルーティングはデバイスごとに保存されるため、各 iPad やコンピューターで個別の設定が可能です。

  </AccordionItem>

  <AccordionItem question="編集してもレシートに古いバージョンが表示される">

テンプレート編集画面で WordPress の**更新**ボタンをクリックしてください。エディターは自動保存されないため、**更新**をクリックしない限り変更は保存されません。

**レガシー PHP テンプレート**の場合、エディター内のプレビューには入力中の内容ではなく*最後に保存された*バージョンが表示されます。まず保存してから、プレビューを更新してください。

  </AccordionItem>

  <AccordionItem question="プレビューが空白、または「POS注文が見つかりません」と表示される">

これは**レガシー PHP テンプレート**でのみ発生します。プレビューに実際の注文データが必要なためです。POS 注文を1件処理すれば（$0 のテスト販売でも可）、プレビューが正常に表示されるようになります。

ロジックレス（HTML）テンプレートおよびサーマルテンプレートには常にフォールバック用のサンプルデータがあるため、新規店舗でも問題なくプレビューできます。

  </AccordionItem>

  <AccordionItem question="ぐちゃぐちゃにしてしまった — 最初からやり直すには？">

3つのセーフティネット：

1. エディターには**元に戻す**機能（Ctrl/Cmd + Z）があり、セッション中の変更を取り消せます。
2. 保存するたびに WordPress の**リビジョン**が作成されます。編集画面の**リビジョン**を開くと、過去のバージョンを比較・復元できます。
3. ギャラリーテンプレートから作成した場合は、*マイテンプレート*でコピーの**削除**をクリックし、同じギャラリーカードの**テンプレートを使用**を再度クリックしてください。未編集の新しいコピーが取得できます。

  </AccordionItem>
</Accordion>

## サポートが必要な場合 {#when-to-ask-for-help}

- テンプレートエディターが読み込まれない、または保存が反映されない。
- レシートがあるデバイスでは正常に印刷されるが、別のデバイスでは印刷されない。
- 特定の国（イタリア、ブラジル、スペインなど）向けの財務・法的レイアウトが必要な場合 — これらは通常、[WCPOS Pro](/getting-started/pro-license) または国別の連携機能で対応します。
- カスタム処理を行おうとしているが、AIでは思い通りの結果が得られない。

[サポートチケット](https://wcpos.com/support)を作成し、作業中のテンプレートを貼り付けてください。それだけでサポートに必要な情報がすべて揃います。
