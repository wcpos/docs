WCPOSには、レシート、請求書、ギフトレシート、梱包明細、キッチンチケットなどを扱う完全なテンプレートシステムがあります。すべて **WP Admin → POS → テンプレート** で管理します。

:::tip レシートだけを変更したい場合
まず **[レシートをカスタマイズ](/templates/customise)** を開いてください。別の付属テンプレートを選ぶ、AIに調整を依頼する、手動で編集する、という3つの簡単な方法を説明しています。
:::

## テンプレートエンジン

WCPOSは3種類のテンプレートエンジンをサポートしています。エンジンはテンプレート作成時、またはギャラリーカードの **テンプレートを使用** をクリックした時に選ばれ、**後から変更できません**。切り替えたい場合は新しいテンプレートを作成してください。

<FeatureGrid>
  <FeatureCard icon="code" title="ロジックなしHTML（推奨）">
    通常のHTMLの中にMustache風のプレースホルダー <code>{"{{variable}}"}</code> を置きます。クライアント側でレンダリングされるため、レシートは**オフラインでも動作**します。ブラウザー印刷やPDFレシートに使用します。
  </FeatureCard>
  <FeatureCard icon="print" title="サーマルXML">
    画面プレビューとサーマルレシートプリンター向けの **ESC/POSコマンド** の両方を生成するXMLテンプレートです。1つのテンプレートで2つの出力に対応します。Epson、Star、その他のサーマルプリンターに使用します。
  </FeatureCard>
  <FeatureCard icon="terminal" title="レガシーPHP">
    WooCommerce関数を使うサーバー側PHPテンプレートです。既存の <code>yourtheme/woocommerce-pos/receipt.php</code> 上書きとの互換性のために残されています。レンダリングにはサーバー接続が必要です。
  </FeatureCard>
</FeatureGrid>

## テンプレートギャラリー

ギャラリーには、そのまま使うこともカスタマイズすることもできる既製テンプレートが含まれています。**カテゴリー**（レシート、請求書、ギフトレシート、キッチンチケット、見積 / 購入注文）、**形式**（HTML / ESC/POS）、**方向**（LTR / RTL）で絞り込み、店舗の実データでプレビューできます。各カードの下部には **プレビュー** リンクと **テンプレートを使用** ボタンがあり、クリックすると編集可能なコピーが *あなたのテンプレート* に作成されます。

| テンプレート | エンジン | 説明 |
|----------|--------|-------------|
| 標準レシート | HTML | 既定のレシート。ロゴ、店舗情報、商品、合計、支払いを表示 |
| 標準レシート（RTL） | HTML | アラビア語、ヘブライ語、ペルシア語、ウルドゥー語向けの右から左の版 |
| ミニマル / モダン | HTML | 標準と同じ主要情報を、より短い縦幅で表示 |
| 詳細レシート | HTML | SKU列、単価、税率別の税内訳、住所を含む完全な税務請求書 |
| ギフトレシート | HTML | 商品のみで価格は非表示。ギフトメッセージと返品ポリシーを含む |
| 請求書 | HTML | 未払い注文向けの「支払い方法」欄を任意で含むA4/Letter全ページ請求書 |
| 梱包明細 | HTML | 倉庫用の補助書類。商品と数量、配送先を表示し、価格は表示しない |
| 見積 / 概算 | HTML | 支払い欄なしで価格と条件を示す販売前ドキュメント |
| 細幅レシート | HTML | 細い用紙やHTML対応サーマルプリンター向けの等幅フォントレシート |
| シンプルサーマルレシート（58 mm） | サーマル | 58 mm向けのすっきりしたサーマルレイアウト |
| シンプルサーマルレシート（80 mm） | サーマル | 最も一般的な80 mm向けのすっきりしたサーマルレイアウト |
| シンプルサーマルレシート80 mm（RTL） | サーマル | 80 mmのRTL版。アラビア語コードページ（CP864 / Windows-1256）対応プリンターが必要 |
| 詳細サーマルレシート（58 mm） | サーマル | 住所、税内訳、返金、支払い、条件、バーコードを含む多機能58 mm版 |
| 詳細サーマルレシート（80 mm） | サーマル | 住所、税内訳、返金、支払い、条件、バーコードを含む多機能80 mm版 |
| キッチンチケット | サーマル | 商品のみ、大きな文字、価格なし。調理ステーション向け |

### サードパーティ書類テンプレート

WP Overnightの **PDF Invoices & Packing Slips for WooCommerce** が有効な場合、WCPOSはレシートテンプレート一覧に **請求書（WP Overnight）** と **梱包明細（WP Overnight）** も表示します。これらのサーバーレンダリングHTMLテンプレートはWP OvernightのドキュメントAPIに処理を委譲し、請求書と梱包明細の番号、レイアウト、ブランド設定、法務・税務フィールド、同プラグインで設定したテンプレートカスタマイズを再利用します。

これらはWP Overnightプラグインが有効な間だけ利用でき、レンダリングにはサーバー接続が必要です。オフライン印刷が必要な場合は、WCPOSに付属するHTMLテンプレートまたはサーマルテンプレートを使用してください。

### 税表示の自動調整

付属テンプレートの多く（標準、標準RTL、ミニマル/モダン、細幅、請求書、見積、シンプルサーマル各種）は、**WooCommerceの税表示設定に自動で合わせます**。

- **税込表示の店舗**（EU/英国/AU）では、税込価格と「税込」行が表示されます。
- **税抜表示の店舗**（米国/カナダ）では、税抜価格に税額が別行で追加されます。
- 総合計には、実際に請求された税込金額が常に表示されます。

**詳細** 系テンプレートは正式な税務請求書であり、店舗設定にかかわらず、税抜行と明示的な税内訳で税額を常に表示します。

## 使い方

1. **テンプレートを追加**。ギャラリーカードの **テンプレートを使用** をクリックするか、最初から自作します。
2. **カスタマイズ**。ライブプレビュー付きのアプリ内エディターを使うか、ChatGPT / Claudeに貼り付けて希望する変更を依頼します。
3. **有効化**。*あなたのテンプレート* で **有効** トグルをオンにします。有効なレシートテンプレートは、POSのレシート画面のドロップダウンに表示されます。
4. **印刷または表示**。会計後にレシートが開き、印刷、メール送信、画面表示を選べます。

テンプレートは、店舗情報、明細、合計、税、支払い、返金などのセクションを持つ標準化された [レシートデータペイロード](/templates/receipt-data) に対してレンダリングされます。通貨フィールドには、`29.99` ではなく `$29.99` のような整形済みの `_display` 版が含まれています。

## Pro機能

[WCPOS Pro](/getting-started/pro-license) では、各店舗が独自のテンプレート割り当て、ブランド設定（ロゴ、住所、連絡先）、テンプレート順序を持てます。複数拠点で、それぞれの店舗が独自のレターヘッドを印刷する場合に便利です。
