プリンター設定は POS の **設定 > プリンター** にあります。各デバイスが独自のプリンター構成を管理します — プリンターはローカルに保存され、デバイス間では同期されません。

## プラットフォーム別の対応プリンター {#supported-printers-by-platform}

**プリンターを追加**フォームは使用中のアプリに合わせて変わります — 各プラットフォームには、実際に利用できる接続とオプションだけが表示されます。

| プラットフォーム | 対応プリンター | 接続 |
|----------|--------------------|-------------|
| **Web アプリ** | 内蔵 Web サーバーを備えた Epson および Star プリンター | ネットワーク (HTTP) |
| **デスクトップアプリ** | 任意のネットワーク対応レシートプリンター | ネットワーク (raw TCP) |
| **iOS アプリ** | Epson および Star | ネットワーク、Bluetooth |
| **Android アプリ** | Epson および Star | ネットワーク、Bluetooth、USB |

:::info Web で Epson と Star だけなのはなぜですか？
Web ブラウザーは raw TCP 接続を開けません。Epson と Star のレシートプリンターには HTTP で印刷ジョブを受け付ける内蔵 Web サーバーがあり、ブラウザーから ESC/POS コマンドを送信する唯一の方法です。デスクトップアプリは TCP に直接接続できるため、この制限はありません。

**モバイルアプリ**では、Bluetooth と USB の印刷は Epson と Star の公式 SDK を経由するため、これらの接続にも Epson または Star プリンターが必要です。汎用プリンターはネットワーク経由でのみ動作します。
:::

## プリンターを追加する {#adding-a-printer}

<Steps>
  <StepCard number="1" title="プリンター設定を開く">

POS で **設定 > プリンター** に移動します。まだプリンターが設定されていない場合は、**プリンターを追加**ボタンのある空の状態が表示されます。各プリンターには **プリンター名** が付きます — これは単なるラベルで、自動的に「レシートプリンター」（次は「レシートプリンター 2」など）として入力されます — 必要に応じてプリンターを見分けやすい名前に変更できます。

  </StepCard>
  <StepCard number="2" title="プリンターの接続方法を選択する">

この手順はプラットフォームによって変わります：

<ul>
  <li><strong>Web アプリ</strong> — <strong>ベンダー</strong>を選択します：Epson または Star。Web ではこれが最初の選択肢です。ブラウザーが到達できるのは、この 2 ブランドの内蔵 Web サーバーだけだからです。</li>
  <li><strong>デスクトップアプリ</strong> — プリンターはネットワーク経由で接続します。ここで追加で選ぶものはありません。</li>
  <li><strong>モバイルアプリ</strong> — <strong>接続タイプ</strong>を選択します：<strong>ネットワーク</strong>、<strong>Bluetooth</strong>、または <strong>USB</strong>（USB は Android のみ）。下のフィールドは選択内容に合わせて変わります。</li>
</ul>

  </StepCard>
  <StepCard number="3" title="プリンターを識別する">

**ネットワークプリンター** — **IP アドレス**（例：`192.168.1.100`）を入力します。WCPOS はそのアドレスを調べ、Epson または Star デバイスかどうかを検出し、ポート、コマンド言語、ベンダーを自動入力します。検出が完了すると「検出済み: Epson」または「検出済み: Star」というラベルが表示されます。Web アプリでは、接続先となる正確なエンドポイント URL も表示されます。

**Bluetooth または USB プリンター**（モバイル）— 検出されたデバイス一覧からプリンターを選択します。デバイスを選ぶと、名前とベンダーが自動入力されます。プリンターがまだ一覧にない場合は、**プリンターをスキャン**（Bluetooth）または **更新**（USB）をタップします。

自動検出で正しく設定されない項目がある場合は、**詳細設定**で全て調整できます。

  </StepCard>
  <StepCard number="4" title="保存してテストする" isLast>

**保存**をクリックします。WCPOS は最初に**テスト印刷**を送信し、成功した場合だけプリンターを保存します。

テスト印刷は短い診断です — 番号付きの列定規、中央揃えの通常サイズと倍サイズの文字、左右揃えの行、用紙カットが含まれます。プリンターに到達できるだけでなく、用紙幅、配置、サイズ、カットが正しく設定されていることを確認するためのものです。

テストに失敗した場合は、エラーとともに **テストせずに保存** オプションが表示され、プロファイルをそのまま保存できます。

  </StepCard>
</Steps>

## 接続タイプ（モバイルアプリ）{#connection-types-mobile-apps}

iOS と Android アプリでは、プリンターは 3 つの方法で接続できます。プリンター追加フォーム上部の **接続タイプ** を選ぶと、接続フィールドがそれに合わせて更新されます。

### ネットワーク {#network}

プリンターはデバイスと同じ Wi-Fi ネットワーク上にあります。IP アドレスを入力します — [プリンターの IP アドレスを確認する](#finding-your-printers-ip-address)を参照してください。ネットワークプリンターは Epson、Star、または汎用にできます。

### Bluetooth {#bluetooth}

プリンターは Bluetooth でデバイスにペアリングされています。**プリンターをスキャン**をタップして一覧から選択します — 名前とベンダーが自動入力されます。Bluetooth 印刷は Epson と Star のネイティブ SDK を使用するため、プリンターは **Epson または Star** モデルである必要があります。

### USB（Android のみ）{#usb-android-only}

プリンターは USB ケーブルで接続されています。**更新**をタップし、接続済みデバイス一覧から選択します。Bluetooth と同様、USB 印刷には **Epson または Star** プリンターが必要です。iOS は一般的な USB 周辺機器をサポートしていないため、このオプションは Android のみに表示されます。

:::note
**ポート**設定はネットワークプリンターにのみ適用されます — Bluetooth と USB 接続では非表示になります。
:::

## 詳細設定 {#advanced-settings}

**詳細設定**を展開して、プリンター構成を細かく調整します。これらの多くは、ベンダーが分かると自動入力されます。

| 設定 | オプション | 説明 |
|---------|---------|-------------|
| **ベンダー** | Epson, Star, Generic | プリンターのメーカーです。**Web アプリ**では、ベンダーはフォーム上部の主要な選択項目（Epson または Star のみ）であり、詳細設定ではありません。Generic はデスクトップおよびモバイルアプリの**ネットワーク**プリンターで利用できますが、Bluetooth または USB 接続では利用できません。 |
| **ポート** | 自動入力 | プリンターのネットワークポートです。ネットワークのみ — Bluetooth と USB では非表示です。 |
| **言語** | ESC/POS, StarPRNT, Star Line Mode | プリンターが使用するコマンドプロトコルです。検出されたベンダーから自動入力されます。 |
| **プリンターテキスト幅** | 58mm（32 文字）、80mm 標準（42 文字）、80mm ワイド（48 文字） | 1 行に収まる文字数です。用紙とプリンターに合わせてください — 多くの 80mm プリンターは 1 行 42 文字、一部は 48 文字です。 |
| **レシート全体をラスター化** | オフ / オン | レシート全体をテキストではなく画像として印刷します — 下の[レシート全体をラスター化](#full-receipt-raster)を参照してください。 |

## プリンターオプション {#printer-options}

これらの切り替えは、プリンター追加フォームの下部にあります：

| オプション | 説明 |
|--------|-------------|
| **用紙を自動カット** | 各レシートの後に用紙を自動でカットします |
| **キャッシュドロワーを自動で開く** | 印刷後、接続されたキャッシュドロワーを開くキックコマンドを送信します |
| **デフォルトに設定** | これをデフォルトプリンターにします — 特定のプリンターにルーティングされていないレシートで使用されます |

:::note 「会計後に自動印刷」を探していますか？
販売完了時にレシートを自動印刷する機能は**カート設定**であり、プリンター設定ではありません — POS カート設定で **レシートを自動印刷** をオンにしてください。*どの*プリンターを使うかは、デフォルトプリンターとテンプレートごとのプリンタールーティングで決まります。
:::

## 印刷ルーティング {#print-routing}

複数のテンプレート — たとえば感熱レシート**と** A4 請求書 — を使用する場合、印刷ルーティングによって各テンプレートの送信先プリンターが決まります。ルーティングは次の順序で 3 層を確認します：

1. **ジョブごとの上書き。** レシート画面では、テンプレート切り替えの横にプリンターのドロップダウンがあります。ここでプリンターを選ぶと、その 1 回の印刷ジョブだけ全てを上書きします。テンプレートを切り替えると **自動** に戻ります。
2. **設定による上書き。** **設定 → 印刷ルーティング** に移動して、各テンプレートに特定のプリンターを割り当てます。たとえば、感熱レシートを Epson ネットワークプリンターに、HTML 請求書をシステム印刷ダイアログに送れます。テンプレートを **自動** に戻すと上書きが削除されます。
3. **自動一致。** 上書きが設定されていない場合、WCPOS は自動的に一致させます：
   - **感熱テンプレート**は文字幅が一致する感熱プリンターに送られます。58mm の感熱テンプレートは 32 桁プリンターを優先し、80mm テンプレートは 42 桁または 48 桁プリンターを優先します。
   - **HTML テンプレート**はシステム印刷ダイアログに送られます。
   - 複数のプリンターが一致する場合は、**デフォルト**プリンターが優先されます。

互換性のないプリンターにテンプレートを手動送信した場合 — たとえば感熱テンプレートをシステムダイアログに送った場合 — レシート画面に琥珀色の**不一致警告**が表示されます。印刷は続行されますが、出力が正しく描画されない可能性があります。

ルーティングの上書きは**デバイスごと**に保存されます。各 iPad、電話、コンピューターが独自のルーティングを管理します — サーバー側の同期はありません。

:::note
印刷ルーティング設定ページは、少なくとも 1 台のプリンターを追加した後にだけ表示されます。プリンターが未設定の場合、全てのテンプレートは暗黙的にシステム印刷ダイアログを使用します。
:::

## レシート全体をラスター化 {#full-receipt-raster}

既定では、WCPOS はレシートを感熱プリンターに**テキスト**として送信します — 高速でコンパクト、プリンター内蔵フォントを使うためくっきりしています。ただし、感熱プリンターのフォントは限られた文字セットしか対応していません。プリンターにフォントがない文字体系 — 多くの非ラテン文字、右から左へ書くテキスト、一部の記号 — は空白の四角や文字化けとして出力されることがあります。

**レシート全体をラスター化**すると、この問題を解決できます。オンにすると、WCPOS はレシート全体を画像としてレンダリングし、その画像をプリンターに送信するため、印刷結果は画面表示と完全に一致します — どの言語や文字体系でも同じです。
