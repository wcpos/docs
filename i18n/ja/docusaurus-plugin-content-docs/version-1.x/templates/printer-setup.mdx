プリンター設定は POS の **設定 > プリンター** にあります。各端末がそれぞれのプリンター設定を管理します。プリンターは端末内に保存され、端末間では同期されません。

## プラットフォーム別の対応プリンター {#supported-printers-by-platform}

**プリンターを追加** フォームは、使用中のアプリに合わせて変わります。各プラットフォームでは、実際に使える接続方法とオプションだけが表示されます。

| プラットフォーム | 対応プリンター | 接続 |
|----------|--------------------|-------------|
| **Web アプリ** | 内蔵 Web サーバーを備えた Epson および Star プリンター | ネットワーク (HTTP) |
| **デスクトップアプリ** | 任意のネットワーク対応レシートプリンター | ネットワーク (raw TCP) |
| **iOS アプリ** | Epson と Star | ネットワーク、Bluetooth |
| **Android アプリ** | Epson と Star | ネットワーク、Bluetooth、USB |

:::info Web で Epson と Star のみなのはなぜですか?
Web ブラウザーは raw TCP 接続を開けません。Epson と Star のレシートプリンターには、HTTP で印刷ジョブを受け付ける内蔵 Web サーバーがあります。ブラウザーから ESC/POS コマンドを送るにはこれが唯一の方法です。デスクトップアプリは TCP で直接接続できるため、この制限はありません。

**モバイルアプリ** では、Bluetooth と USB 印刷は Epson と Star の公式 SDK を通します。そのため、これらの接続でも Epson または Star のプリンターが必要です。汎用プリンターはネットワーク経由でのみ動作します。
:::

## プリンターを追加する {#adding-a-printer}

<Steps>
  <StepCard number="1" title="プリンター設定を開く">

POS で **設定 > プリンター** に移動します。まだプリンターが設定されていない場合は、**プリンターを追加** ボタンがある空の状態が表示されます。各プリンターには **プリンター名** が付きます。これは単なるラベルで、最初は "レシートプリンター"、次は "レシートプリンター 2" のように自動入力されます。プリンターを区別しやすい名前に変更できます。

  </StepCard>
  <StepCard number="2" title="プリンターの接続方法を選ぶ">

この手順はプラットフォームによって変わります。

<ul>
  <li><strong>Web アプリ</strong> — <strong>メーカー</strong> として Epson または Star を選びます。ブラウザーはこの 2 ブランドの内蔵 Web サーバーにしか接続できないため、Web では最初に選ぶ項目です。</li>
  <li><strong>デスクトップアプリ</strong> — プリンターはネットワーク経由で接続します。ここで追加選択する項目はありません。</li>
  <li><strong>モバイルアプリ</strong> — <strong>接続タイプ</strong> として <strong>ネットワーク</strong>、<strong>Bluetooth</strong>、または <strong>USB</strong> を選びます (USB は Android のみ)。下のフィールドは選択内容に合わせて変わります。</li>
</ul>

  </StepCard>
  <StepCard number="3" title="プリンターを識別する">

**ネットワークプリンター** — **IP アドレス** を入力します (例: `192.168.1.100`)。WCPOS はそのアドレスを調べ、Epson または Star の端末かどうかを検出し、ポート、コマンド言語、メーカーを自動入力します。検出が完了すると、"検出: Epson" または "検出: Star" のようなラベルが表示されます。Web アプリでは、接続先となる正確なエンドポイント URL も表示されます。

**Bluetooth または USB プリンター** (モバイル) — 検出された端末の一覧からプリンターを選びます。端末を選択すると、名前とメーカーが自動入力されます。プリンターがまだ表示されない場合は、Bluetooth では **プリンターをスキャン**、USB では **更新** をタップします。

自動検出が正しくない場合は、**詳細設定** ですべて調整できます。

  </StepCard>
  <StepCard number="4" title="保存してテスト" isLast>

**保存** をクリックします。WCPOS はまず **テスト印刷** を送信し、成功した場合だけプリンターを保存します。

テスト印刷は短い診断用の出力です。番号付きの列ルーラー、中央揃えの通常サイズと倍サイズのテキスト、左右揃えの行、用紙カットを印刷します。プリンターに到達できるだけでなく、用紙幅、配置、サイズ、カットが正しく設定されているか確認できます。

テストに失敗した場合は、エラーとともに **テストせずに保存** オプションが表示され、プロファイルだけを保存できます。

  </StepCard>
</Steps>

## 接続タイプ (モバイルアプリ) {#connection-types-mobile-apps}

iOS と Android アプリでは、プリンターは 3 つの方法で接続できます。プリンター追加フォームの上部で **接続タイプ** を選ぶと、接続フィールドがそれに合わせて更新されます。

### ネットワーク {#network}

プリンターが端末と同じ Wi-Fi ネットワーク上にあります。IP アドレスを入力します。詳しくは [プリンターの IP アドレスを見つける](#finding-your-printers-ip-address) を参照してください。ネットワークプリンターは Epson、Star、または汎用を使用できます。

### Bluetooth {#bluetooth}

プリンターは Bluetooth で端末にペアリングされています。**プリンターをスキャン** をタップし、一覧から選びます。名前とメーカーは自動入力されます。Bluetooth 印刷は Epson と Star のネイティブ SDK を使用するため、プリンターは **Epson または Star** モデルである必要があります。

### USB (Android のみ) {#usb-android-only}

プリンターは USB ケーブルで接続されています。**更新** をタップし、接続済み端末の一覧から選びます。Bluetooth と同様に、USB 印刷にも **Epson または Star** のプリンターが必要です。iOS には汎用 USB 周辺機器のサポートがないため、このオプションは Android でのみ表示されます。

:::note
**ポート** 設定はネットワークプリンターにのみ適用されます。Bluetooth と USB 接続では非表示になります。
:::

## 詳細設定 {#advanced-settings}

**詳細設定** を開くと、プリンター構成を細かく調整できます。これらの多くは、メーカーが分かると自動入力されます。

| 設定 | オプション | 説明 |
|---------|---------|-------------|
| **メーカー** | Epson、Star、汎用 | プリンターのメーカーです。**Web アプリ** では、メーカーは詳細設定ではなくフォーム上部の主要な選択項目になります (Epson または Star のみ)。汎用はデスクトップアプリとモバイルアプリの **ネットワーク** プリンターで利用できますが、Bluetooth または USB 接続では利用できません。 |
| **ポート** | 自動入力 | プリンターのネットワークポートです。ネットワークのみで、Bluetooth と USB では非表示です。 |
| **言語** | ESC/POS、StarPRNT、Star Line Mode | プリンターが使用するコマンドプロトコルです。検出されたメーカーから自動入力されます。 |
| **プリンターテキスト幅** | 58mm (32 文字)、80mm 標準 (42 文字)、80mm ワイド (48 文字) | 1 行に入る文字数です。用紙とプリンターに合わせてください。多くの 80mm プリンターは 1 行 42 文字、一部は 48 文字です。 |
| **レシート全体のラスター化** | オフ / オン | レシート全体をテキストではなく画像として印刷します。下の [レシート全体のラスター化](#full-receipt-raster) を参照してください。 |

## プリンターオプション {#printer-options}

これらの切り替えは、プリンター追加フォームの下部にあります。

| オプション | 説明 |
|--------|-------------|
| **用紙を自動カット** | 各レシートの後に用紙を自動でカットします |
| **キャッシュドロワーを自動で開く** | 印刷後、接続されたキャッシュドロワーを開くキックコマンドを送信します |
| **既定に設定** | このプリンターを既定にします。特定のプリンターに振り分けられていないレシートに使われます |

:::note 「会計後に自動印刷」を探していますか?
販売完了時にレシートを自動印刷するかどうかは **カート設定** であり、プリンター設定ではありません。POS のカート設定で **レシートを自動印刷** をオンにしてください。*どの* プリンターを使うかは、既定のプリンターとテンプレートごとのプリンター振り分けで決まります。
:::

## 印刷ルーティング {#print-routing}

複数のテンプレート、たとえばサーマルレシート **と** A4 請求書を使う場合、印刷ルーティングによって各テンプレートの出力先プリンターが決まります。ルーティングは次の順序で 3 段階に確認されます。

1. **ジョブごとの上書き。** レシート画面では、テンプレート切り替えの横にプリンタードロップダウンがあります。ここでプリンターを選ぶと、その 1 回の印刷ジョブだけすべてを上書きします。テンプレートを切り替えると **自動** に戻ります。
2. **設定での上書き。** **設定 → 印刷ルーティング** に移動して、各テンプレートに特定のプリンターを割り当てます。たとえば、サーマルレシートは Epson ネットワークプリンターへ、HTML 請求書はシステムの印刷ダイアログへ送れます。上書きを外すにはテンプレートを **自動** に戻します。
3. **自動一致。** 上書きがない場合、WCPOS が自動的に一致させます。
   - **サーマルテンプレート** は、文字幅が合うサーマルプリンターに送られます。58mm のサーマルテンプレートは 32 桁プリンターを優先し、80mm テンプレートは 42 桁または 48 桁プリンターを優先します。
   - **HTML テンプレート** はシステムの印刷ダイアログへ送られます。
   - 複数のプリンターが一致する場合は、**既定** のプリンターが選ばれます。

互換性のないプリンターへ手動でテンプレートを送った場合、たとえばサーマルテンプレートをシステムダイアログへ送ると、レシート画面に琥珀色の **不一致警告** が表示されます。印刷は続行されますが、出力が正しく表示されない場合があります。

ルーティングの上書きは **端末ごと** に保存されます。各 iPad、電話、コンピューターが独自のルーティングを管理します。サーバー側の同期はありません。

:::note
印刷ルーティング設定ページは、少なくとも 1 台のプリンターを追加した後にだけ表示されます。プリンターが未設定の場合、すべてのテンプレートは暗黙的にシステムの印刷ダイアログを使用します。
:::

## レシート全体のラスター化 {#full-receipt-raster}

既定では、WCPOS はサーマルプリンターへレシートを **テキスト** として送信します。これは高速で軽量、かつプリンター内蔵フォントにより鮮明です。ただし、サーマルプリンターのフォントは対応文字が限られています。プリンターにフォントがない文字体系、たとえば多くの非ラテン文字、右から左へ書く言語、一部の記号は、空白の四角や文字化けとして出ることがあります。

**レシート全体のラスター化** はこれを解決します。オンにすると、WCPOS はレシート全体を画像として描画し、その画像をプリンターへ送信します。そのため、印刷結果は画面表示と正確に一致し、どの言語や文字体系でも正しく出力できます。

プリンターごとに **詳細設定** でオンにします。知っておくべき点:

- **遅くなります。** 画像はテキスト 1 行よりかなり大きいため、送信と印刷に時間がかかります。
- **必要な場合だけ使ってください。** レシートがテキストで問題なく印刷できる場合は、オフのままにします。
- ネットワーク、Bluetooth、USB のどの接続でもサーマルプリンターに適用されます。すでにページ全体のレンダリングから印刷するシステム印刷ダイアログには適用されません。

## プリンターを検出する {#discovering-printers}

**モバイルアプリ** では、プリンター追加フォームに **Bluetooth** (iOS と Android) と **USB** (Android のみ) 接続で近くのプリンターが表示されます。**プリンターをスキャン** または **更新** をタップし、一覧から選びます。名前とメーカーは自動入力されます。

任意のプラットフォームの **ネットワーク** プリンターでは、IP アドレスを直接入力します。下の [プリンターの IP アドレスを見つける](#finding-your-printers-ip-address) を参照してください。Web アプリにはネットワーク検出がありません。ブラウザーはネットワークをスキャンできないため、常に手動の IP 入力を使います。

## プリンターの IP アドレスを見つける {#finding-your-printer-s-ip-address}

多くのレシートプリンターは、IP アドレスを含む自己テストページを印刷できます。

- **Epson**: 電源投入時にフィードボタンを押し続けます
- **Star**: 電源が入った状態でフィードボタンを 5 秒間押し続けます

または、ルーターの接続端末一覧やプリンターの設定ユーティリティを確認してください。

### 固定 IP を推奨 {#static-ip-recommended}

レシートプリンターは、再起動時にアドレスが変わらないよう **固定 IP アドレス** を使用するのがおすすめです。次のいずれかで設定します。

- プリンターの内蔵 Web インターフェイスで設定します。通常は `http://<printer-ip>` からアクセスできます
- ルーターの DHCP 設定で、プリンターの MAC アドレスに予約 IP を割り当てます

## Epson プリンター {#epson-printers}

ePOS 対応の Epson プリンターは HTTP で通信します。WCPOS はプリンターの内蔵 Web サーバーへ SOAP/XML リクエストを送信します。

**接続の詳細:**
- ポート `8008` (HTTP) または `8043` (HTTPS)
- エンドポイント: `/cgi-bin/epos/service.cgi`

**動作確認済みモデル:**
- Epson TM-T70-i 2 (ネットワーク)
- Epson TM-m30iii (Printus ミドルウェア使用)

:::tip
Epson プリンターの設定で ePOS が有効になっていることを確認してください。`http://<printer-ip>` でプリンターの Web インターフェイスを開き、ePOS 設定を確認して有効化します。
:::

**モバイルアプリ** では、Epson プリンターは Epson のネイティブ SDK を使って Bluetooth または USB でも接続できます。IP アドレスや ePOS Web サーバーは不要です。

## Star プリンター {#star-printers}

WebPRNT 対応の Star プリンターは HTTP で通信します。WCPOS は Star WebPRNT エンドポイントへコマンドを送信します。

**接続の詳細:**
- HTTPS または HTTP
- エンドポイント: `/StarWebPRNT/SendMessage`

**モバイルアプリ** では、Star プリンターも Star のネイティブ SDK を使って Bluetooth または USB で接続できます。

## 汎用プリンター {#generic-printers}

**デスクトップアプリ** は raw TCP を使って任意のネットワーク対応レシートプリンターをサポートします。プリンターの IP アドレスとポート (通常は `9100`) を入力します。ESC/POS コマンドを受け付ける限り、メーカーを問わず多くのサーマルプリンターで動作します。

**モバイルアプリ** でも汎用プリンターはサポートされますが、**ネットワーク** 接続のみです。Bluetooth または USB では使えません (その場合は Epson または Star プリンターが必要です)。**Web アプリ** は汎用プリンターをまったくサポートしません。

## キャッシュドロワー {#cash-drawers}

レシートプリンターに接続されたキャッシュドロワーは、各販売後に自動で開けられます。

1. プリンターオプションで **キャッシュドロワーを自動で開く** を有効にします
2. POS は印刷後に ESC/POS のキックコマンドを送信します

:::note
キャッシュドロワー制御にはサーマルプリンタープロファイルが必要です。ブラウザーの印刷ダイアログからキャッシュドロワーを開くことはできません。
:::

## トラブルシューティング {#troubleshooting}

<Accordion>
  <AccordionItem question="設定中にプリンターが検出されない">

  - プリンターの電源が入っており、端末と同じネットワークに接続されていることを確認します
  - ネットワークプリンターの場合: IP アドレスが正しいか確認します (自己テストページを印刷します)
  - Web アプリの場合: プリンターが ePOS/WebPRNT 対応の Epson または Star モデルであることを確認します
  - デスクトップアプリの場合: ポート 9100 で汎用メーカーのオプションを試します
  - Bluetooth の場合: 先に端末の Bluetooth 設定でプリンターをペアリングし、その後 **プリンターをスキャン** をタップします
  - USB (Android) の場合: ケーブルを確認し、USB 権限の確認が出たら許可します
  - ファイアウォールが接続をブロックしていないことを確認します

  </AccordionItem>

  <AccordionItem question="プリンターは検出されるが何も印刷されない">

  - 用紙ロールを確認します。空か、詰まっている可能性があります
  - **プリンターテキスト幅** 設定が用紙 (58mm または 80mm) と一致しているか確認します
  - プリンター本体から自己テストを印刷し、動作することを確認します
  - コマンド言語がプリンターに合っているか確認します (Epson は ESC/POS、Star は StarPRNT または Star Line Mode)

  </AccordionItem>

  <AccordionItem question="レシートが文字化けする、または間違った文字になる">

  - コマンド言語の設定が間違っている可能性があります。ESC/POS、StarPRNT、Star Line Mode を切り替えて試します
  - **プリンターテキスト幅** 設定が正しいことを確認します
  - レシートが非ラテン文字や右から左へ書く言語の場合は、詳細設定で **レシート全体のラスター化** をオンにします。レシートを画像として印刷するため、どの言語でも正しく表示されます

  </AccordionItem>

  <AccordionItem question="キャッシュドロワーが開かない">

  - プリンターオプションで **キャッシュドロワーを自動で開く** が有効になっていることを確認します
  - ドロワーが別電源ではなく、プリンターの DK ポートに接続されていることを確認します
  - すべてのプリンターがドロワーのキックコマンドに対応しているわけではありません。プリンターの資料を確認してください

  </AccordionItem>

  <AccordionItem question="Web アプリから接続できない">

  - Web アプリは、内蔵 Web サーバーを持つ Epson および Star プリンターにのみ接続できます
  - 汎用プリンターと USB プリンターは Web ではサポートされません。代わりにデスクトップアプリまたはモバイルアプリを使用してください
  - サイトが HTTPS を使用していて、プリンターが HTTP のみの場合、ブラウザーが混在コンテンツとして接続をブロックすることがあります

  </AccordionItem>

  <AccordionItem question="Bluetooth プリンターが接続されない (モバイル)">

  - WCPOS に追加する前に、端末のシステム Bluetooth 設定でプリンターをペアリングします
  - Bluetooth 印刷は Epson と Star プリンターのみ対応です。汎用 Bluetooth プリンターは対応していません
  - プリンターが一覧に表示されない場合は、もう一度 **プリンターをスキャン** をタップし、電源が入っていて通信範囲内にあることを確認します

  </AccordionItem>
</Accordion>
