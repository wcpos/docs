# 翻訳: WCPOS WP Multilang {#wcpos-wp-multilang}

[WP Multilang](https://wordpress.org/plugins/wp-multilang/) を WCPOS と統合し、POS が単一言語の商品のみを提供するようにします。Polylang や WPML とは異なり、WP Multilang はすべての翻訳を同じ投稿内に保存します。このプラグインにより、WCPOS が正しい翻訳を読み取り、検索結果にエントリが重複しないようになります。

## 機能概要 {#what-it-does}

- WCPOS の商品およびバリエーションの REST レスポンスを、単一の WP Multilang 言語にフィルタリングします。
- WCPOS の**高速同期**ルートをインターセプトし、アクティブな言語のコンテンツのみがクライアントに送信されるようにします。
- 無料版では WP Multilang のデフォルト言語が使用されます。
- Pro 版では、店舗エディタから店舗ごとに言語を固定できます。

## インストール {#installation}

<Steps>
  <StepCard number="1" title="JA: WP Multilang のインストール">

[WP Multilang](https://wordpress.org/plugins/wp-multilang/) をインストールし、サイトの言語を設定してください。少なくとも1つの言語をデフォルトとして設定する必要があります。

  </StepCard>
  <StepCard number="2" title="JA: WCPOS WP Multilang のインストール">

`WP Admin > POS > 設定 > 拡張機能` からインストールするか、[GitHub リリースページ](https://github.com/wcpos/wcpos-wp-multilang/releases)から最新版をダウンロードし、`プラグイン > 新規追加 > プラグインのアップロード` からアップロードしてください。

  </StepCard>
  <StepCard number="3" title="JA: （Pro）店舗ごとに言語を固定" isLast>

WCPOS Pro では、`POS > 店舗` から店舗を編集し、**言語**サイドバーセクションから言語を選択してください。*デフォルト*のままにすると、WP Multilang のデフォルト言語が使用されます。

  </StepCard>
</Steps>

## 要件 {#requirements}

<RequirementsList>
  <RequirementItem label="JA: WooCommerce">WooCommerce がインストールされ、有効化されていること</RequirementItem>
