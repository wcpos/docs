WCPOSは、POSに新しい機能を追加する拡張機能に対応しています。拡張機能ディレクトリでは、利用可能な拡張機能の閲覧、POS設定からの直接インストール、およびアップデートの管理が行えます。

:::info Pro機能
拡張機能のインストールと管理には[WCPOS Pro](/getting-started/pro-license)が必要です。無料版では拡張機能カタログは表示されますが、インストールおよび有効化のコントロールは無効になります。
:::

## 拡張機能を見る {#browsing-extensions}

`POS Settings > Extensions`（バージョンによっては **プラグイン** と表示されます）から拡張機能ディレクトリを開きます。

ディレクトリには、利用可能な拡張機能がカード形式のグリッドで表示されます。各カードには次の項目が表示されます。

- **アイコン**（拡張機能でアイコンが提供されていない場合はパズルピースの代替アイコン）
- **名称とバージョン**
- **説明**
- **カテゴリーバッジ**
- **ステータス** — 有効、無効、更新可能、または未インストール

### フィルタリングと検索 {#filtering-and-search}

上部の**カテゴリーピルボタン**を使用して、拡張機能をカテゴリー別にフィルタリングできます。**検索フィールド**を使用して、名前、説明、またはタグで拡張機能を検索することもできます。

## 拡張機能のインストール {#installing-an-extension}

1. `POS Settings > Extensions` を開きます。
2. 目的の拡張機能を見つけて、**インストール**をクリックします。
3. 拡張機能は WordPress プラグインインストーラーを使用してダウンロードおよびインストールされます。
4. インストール後、**有効化**をクリックして有効にします。

内部的には、WCPOS は WordPress ネイティブの `Plugin_Upgrader` を使用してインストールを処理するため、拡張機能は通常の WordPress プラグインと同じ手順でインストールされます。

## 有効化と無効化 {#activating-and-deactivating}

インストール済みの各拡張機能のカードには、**有効化**ボタンと**無効化**ボタンが表示されます。

- **有効化** すると、拡張機能が POS で動作するようになります。
- **無効化** すると、アンインストールせずに拡張機能を停止できます。拡張機能のファイルはサーバー上に残り、いつでも再有効化できます。

## 拡張機能の更新 {#updating-extensions}

インストール済みの拡張機能に新しいバージョンが利用可能になると、カードに **更新あり** バッジと **更新** ボタンが表示されます。

### 自動更新 {#auto-updates}

ディレクトリからインストールした拡張機能は、**自動更新がデフォルトで有効** になっています。自動更新は、ディレクトリ内の各拡張機能のカードからオン/オフを切り替えできます。自動更新が有効な場合、自動更新が有効化されているプラグインと同様に、WordPress が新しいバージョンを自動的に適用します。

## トラブルシューティング {#troubleshooting}

### ボタンに「Pro が必要」と表示される {#requires-pro-message-on-buttons}

WCPOS の無料版では、インストール、有効化、更新の各ボタンが無効になっています。拡張機能を管理するには、[WCPOS Pro](/getting-started/pro-license) にアップグレードしてください。

### 拡張機能のインストールに失敗する {#extension-fails-to-install}

- WordPress サーバーが `wp-content/plugins` ディレクトリへの書き込み権限を持っているか確認してください。
- サーバーが外部への HTTPS リクエストを実行できることを確認してください（一部のホストでは外部ダウンロードがブロックされている場合があります）。
- `WP Admin > POS > Support > Logs` でエラーの詳細を確認してください。

### インストール後に拡張機能が表示されない {#extension-not-appearing-after-install}

- POS を再読み込みしてください。拡張機能のリストは最大 12 時間キャッシュされます。
- 拡張機能が有効化されているか確認してください（インストール済みでも無効な拡張機能は動作しません）。

### カタログが読み込まれない {#catalog-not-loading}

拡張機能カタログはリモートソースから取得され、ローカルに 12 時間キャッシュされます。カタログが読み込まれない場合は次を確認してください。

- サーバーのインターネット接続を確認してください。
- キャッシュの有効期限が切れてから再試行するか、サーバーの transient キャッシュをクリアしてください。

## 開発者向け {#for-developers}

### POS 拡張機能の作成 {#creating-a-pos-extension}

WCPOS 拡張機能は、WCPOS のフックと API を通じて POS と連携する標準的な WordPress プラグインです。作成手順は次のとおりです。

1. **WordPress プラグインとして作成します。** 拡張機能には、通常の WooCommerce や WordPress プラグインと同様に、標準のプラグインヘッダーとエントリファイルが必要です。
2. **WCPOS と統合します。** WCPOS が提供するフックとフィルターを使用して、POS インターフェースまたはバックエンドに機能を追加します。
3. **GitHub でリリースをホストします。** 拡張機能ディレクトリは GitHub Releases を使用してバージョンを管理し、アップデートを配信します。

### ディレクトリへの登録 {#submitting-to-the-directory}

拡張機能カタログは [`wcpos/extensions`](https://github.com/wcpos/extensions) GitHub リポジトリで管理されています。拡張機能を登録するには、次の手順に従ってください。

1. リポジトリの README でカタログのフォーマットとメタデータの要件を確認します。
2. プルリクエストを作成し、拡張機能のメタデータを `catalog.json` に追加します。
3. マージされると、拡張機能はすべての WCPOS Pro ユーザー向けのディレクトリに表示されます。

### GitHub リリースの規約 {#github-release-conventions}

アップデートのライフサイクルは GitHub Releases に依存します。

- **バージョンのタグ付け** にはセマンティックバージョニングを使用します（例: `v1.0.0`、`v1.2.3`）。
- **プラグインの zip ファイルを添付します** — リリースアセットとして添付されたこのファイルが、ユーザーがインストールまたはアップデートする際にダウンロードされます。
- ディレクトリで検出できるよう、**リリースを公開**してください（ドラフトではなく）。

新しいリリースを公開すると、拡張機能をインストール済みのユーザーには、拡張機能ディレクトリにアップデートが表示されます。自動更新が有効な場合は、自動的に適用されます。

カタログのスキーマと申請手続きの詳細については、[`wcpos/extensions`](https://github.com/wcpos/extensions) リポジトリを参照してください。
