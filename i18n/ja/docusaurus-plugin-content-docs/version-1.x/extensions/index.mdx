WCPOS は、販売時点管理に新しい機能を追加する拡張機能をサポートしています。拡張機能ディレクトリでは、利用可能な拡張機能を閲覧し、POS 設定から直接インストールして、更新を管理できます。

:::info Pro 機能
拡張機能のインストールと管理には [WCPOS Pro](/getting-started/pro-license) が必要です。無料版では拡張機能カタログは表示されますが、インストールと有効化のコントロールは無効になります。
:::

## 拡張機能の閲覧 {#browsing-extensions}

`POS Settings > 拡張機能` から拡張機能ディレクトリを開きます（一部のバージョンでは **プラグイン** とも表示されます）。

ディレクトリには利用可能な拡張機能がカードグリッドで表示されます。各カードには次の情報が表示されます:

- **アイコン**（拡張機能が提供していない場合はパズルピースの代替アイコン）
- **名前とバージョン**
- **説明**
- **カテゴリーバッジ**
- **ステータス** — 有効、無効、更新あり、または未インストール

### 絞り込みと検索 {#filtering-and-search}

上部の**カテゴリーピルボタン**を使用して、カテゴリー別に拡張機能を絞り込めます。**検索フィールド**を使用して、名前、説明、タグで拡張機能を検索することもできます。

## 拡張機能のインストール {#installing-an-extension}

1. `POS Settings > 拡張機能` を開きます。
2. 目的の拡張機能を見つけて、**インストール**をクリックします。
3. 拡張機能は WordPress プラグインインストーラーを使用してダウンロードおよびインストールされます。
4. インストールが完了したら、**有効化**をクリックして有効にします。

内部では、WCPOS はインストール処理に WordPress ネイティブの `Plugin_Upgrader` を使用するため、拡張機能は通常の WordPress プラグインと同じ手順に従います。

## 有効化と無効化 {#activating-and-deactivating}

インストール済みの各拡張機能には、カード上に **有効化** ボタンと **無効化** ボタンがあります。

- **有効化** により、拡張機能が POS で実行できるようになります。
- **無効化** は、アンインストールせずに拡張機能を無効にします。拡張機能のファイルはサーバー上に残り、いつでも再有効化できます。

## 拡張機能の更新 {#updating-extensions}

インストール済み拡張機能の新しいバージョンが利用可能になると、カードに **更新可能** バッジと **更新** ボタンが表示されます。

### 自動更新 {#auto-updates}

ディレクトリからインストールされた拡張機能では、**自動更新がデフォルトで有効** になっています。ディレクトリ内の拡張機能カードから、拡張機能ごとに自動更新のオンとオフを切り替えられます。自動更新がオンの場合、WordPress は自動更新が有効なプラグインと同様に、新しいバージョンを自動的に適用します。

## トラブルシューティング {#troubleshooting}

### ボタンの「Pro が必要」メッセージ {#requires-pro-message-on-buttons}

WCPOS の無料版では、インストール、有効化、更新ボタンは無効になっています。拡張機能を管理するには、[WCPOS Pro](/getting-started/pro-license) にアップグレードしてください。

### 拡張機能をインストールできない {#extension-fails-to-install}

- WordPress サーバーに `wp-content/plugins` ディレクトリへの書き込み権限があることを確認してください。
- サーバーが外部への HTTPS リクエストを実行できることを確認してください（一部のホストでは外部ダウンロードがブロックされます）。
- `WP Admin > POS > サポート > ログ` でエラーの詳細を確認してください。

### インストール後に拡張機能が表示されない {#extension-not-appearing-after-install}

- POS を更新してください。拡張機能リストは最大 12 時間キャッシュされます。
- 拡張機能が有効化されていることを確認してください（インストール済みでも無効な拡張機能は実行されません）。

### カタログが読み込まれない {#catalog-not-loading}

拡張機能カタログはリモートソースから取得され、ローカルに 12 時間キャッシュされます。カタログが読み込まれない場合:

- サーバーのインターネット接続を確認してください。
- キャッシュの有効期限が切れてから再試行するか、サーバーの transient キャッシュをクリアしてください。

## 開発者向け {#for-developers}

### POS拡張機能の作成 {#creating-a-pos-extension}

WCPOS拡張機能は、WCPOSのフックとAPIを通じてPOSと連携する標準のWordPressプラグインです。作成手順は次のとおりです。

1. **WordPressプラグインから始めます。** WooCommerceやWordPressのプラグインと同様に、拡張機能には標準のプラグインヘッダーとエントリーファイルが必要です。
2. **WCPOSと連携します。** WCPOSが提供するフックとフィルターを使用して、POSインターフェースまたはバックエンドに機能を追加します。
3. **リリースをGitHubでホストします。** 拡張機能ディレクトリでは、GitHub Releasesを使用してバージョンを追跡し、更新を配信します。

### ディレクトリへの登録 {#submitting-to-the-directory}

拡張機能カタログは、[`wcpos/extensions`](https://github.com/wcpos/extensions) GitHubリポジトリで管理されています。拡張機能を掲載するには、次の手順を行います。

1. リポジトリのREADMEで、カタログ形式とメタデータ要件を確認します。
2. プルリクエストを作成して、拡張機能のメタデータを`catalog.json`に追加します。
3. マージされると、拡張機能はすべてのWCPOS Proユーザー向けのディレクトリに表示されます。

### GitHubリリースの規約 {#github-release-conventions}

更新ライフサイクルは GitHub Releases に依存しています:

- **タグのバージョン**にはセマンティック バージョニングを使用します (e.g., `v1.0.0`, `v1.2.3`).
- **プラグインの zip** をリリースアセットとして添付します — これは、ユーザーがインストールまたは更新するときにダウンロードされるファイルです。
- ディレクトリが検出できるように、**リリースを公開**します (下書きではありません)。

新しいリリースを公開すると、その拡張機能をインストールしているユーザーの拡張機能ディレクトリに、利用可能な更新として表示されます。自動更新が有効な場合は、自動的に適用されます。

カタログスキーマと提出手順の詳細については、[`wcpos/extensions`](https://github.com/wcpos/extensions) リポジトリを参照してください。
