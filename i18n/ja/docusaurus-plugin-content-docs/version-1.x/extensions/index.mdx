WCPOS は、POS に新しい機能を追加する拡張機能をサポートしています。拡張機能ディレクトリでは、利用可能な拡張機能を参照し、POS 設定から直接インストールして、更新を管理できます。

:::info Pro 機能
拡張機能のインストールと管理には [WCPOS Pro](/getting-started/pro-license) が必要です。無料版では拡張機能カタログは表示されますが、インストールおよび有効化のコントロールは無効になります。
:::

## 拡張機能を見る {#browsing-extensions}

`POS Settings > Extensions`（バージョンによっては **プラグイン** と表示されます）から拡張機能ディレクトリを開きます。

ディレクトリには利用可能な拡張機能がカードグリッド形式で表示されます。各カードには次の情報が含まれます：

- **アイコン**（拡張機能側で提供されていない場合はパズルピースの代替アイコン）
- **名称とバージョン**
- **説明**
- **カテゴリバッジ**
- **ステータス** — 有効、無効、更新あり、または未インストール

### フィルタリングと検索 {#filtering-and-search}

上部の**カテゴリピルボタン**を使用して、カテゴリ別に拡張機能をフィルタリングできます。**検索フィールド**を使用して、名前、説明、またはタグから拡張機能を検索することもできます。

## 拡張機能のインストール {#installing-an-extension}

1. `POS Settings > Extensions` を開きます。
2. 目的の拡張機能を見つけて、**インストール**をクリックします。
3. 拡張機能は WordPress プラグインインストーラを使用してダウンロードおよびインストールされます。
4. インストール後、**有効化**をクリックして有効にします。

内部的には、WCPOS は WordPress のネイティブな `Plugin_Upgrader` を使用してインストールを処理するため、拡張機能は通常の WordPress プラグインと同じ手順で導入されます。

## 有効化と無効化 {#activating-and-deactivating}

インストール済みの各拡張機能のカードには、**有効化**ボタンと**無効化**ボタンが表示されます。

- **有効化**すると拡張機能が POS で動作するようになります。
- **無効化**すると、アンインストールせずに拡張機能を停止できます。拡張機能のファイルはサーバー上に残るため、いつでも再度有効化できます。

## 拡張機能の更新 {#updating-extensions}

インストール済みの拡張機能に新しいバージョンがある場合、カードに**更新が利用可能**バッジと**更新**ボタンが表示されます。

### 自動更新 {#auto-updates}

ディレクトリからインストールした拡張機能は、**デフォルトで自動更新が有効**になっています。自動更新は、ディレクトリ内の各拡張機能のカードからオン／オフを切り替えられます。自動更新が有効な場合、自動更新が有効化されたプラグインと同様に、WordPress が新しいバージョンを自動的に適用します。

## トラブルシューティング {#troubleshooting}

### ボタンに「Pro が必要」と表示される {#requires-pro-message-on-buttons}

WCPOS の無料版では、インストール、有効化、更新の各ボタンは無効になっています。拡張機能を管理するには、[WCPOS Pro](/getting-started/pro-license) にアップグレードしてください。

### 拡張機能のインストールに失敗する {#extension-fails-to-install}

- WordPress サーバーが `wp-content/plugins` ディレクトリへの書き込み権限を持っていることを確認してください。
- サーバーから外部への HTTPS リクエストが可能であることを確認してください（一部のホストでは外部ダウンロードがブロックされています）。
- `WP Admin > POS > Support > Logs` でエラーの詳細を確認してください。

### インストール後に拡張機能が表示されない {#extension-not-appearing-after-install}

- POS を再読み込みしてください — 拡張機能の一覧は最大 12 時間キャッシュされます。
- 拡張機能が有効化されていることを確認してください（インストール済みでも無効な拡張機能は動作しません）。

### カタログが読み込まれない {#catalog-not-loading}

拡張機能カタログはリモートソースから取得され、ローカルに 12 時間キャッシュされます。カタログが読み込まれない場合は、次を確認してください。

- サーバーのインターネット接続を確認してください。
- キャッシュの有効期限が切れた後に再試行するか、サーバーのトランジェントキャッシュをクリアしてください。

## 開発者向け {#for-developers}

### POS 拡張機能の作成 {#creating-a-pos-extension}

WCPOS の拡張機能は、WCPOS のフックと API を通じて POS と連携する標準的な WordPress プラグインです。作成するには次の手順に従います。

1. **WordPress プラグインとして開始します。** 拡張機能には、WooCommerce や WordPress のプラグインと同様に、標準的なプラグインヘッダーとエントリーファイルが必要です。
2. **WCPOS と統合します。** WCPOS が提供するフックとフィルターを使用して、POS インターフェースまたはバックエンドに機能を追加します。
3. **GitHub でリリースをホストします。** 拡張機能ディレクトリは、バージョン管理と更新の配信に GitHub Releases を使用します。

### ディレクトリへの登録 {#submitting-to-the-directory}

拡張機能カタログは [`wcpos/extensions`](https://github.com/wcpos/extensions) の GitHub リポジトリで管理されています。拡張機能を掲載するには:

1. リポジトリの README でカタログ形式とメタデータの要件を確認します。
2. プルリクエストを作成し、`catalog.json` に拡張機能のメタデータを追加します。
3. マージされると、拡張機能はすべての WCPOS Pro ユーザー向けのディレクトリに表示されます。

### GitHub リリースの規約 {#github-release-conventions}

更新のライフサイクルは GitHub Releases に依存しています:

- **バージョンにタグを付けます。** セマンティックバージョニングを使用します (例: `v1.0.0`, `v1.2.3`)。
- **プラグインの zip を添付します。** リリースアセットとして添付してください。これは、ユーザーがインストールまたは更新する際にダウンロードされるファイルです。
- ディレクトリで検出できるよう、リリースを**公開**してください（ドラフトのままにしないでください）。

新しいリリースを公開すると、拡張機能をインストールしているユーザーの拡張機能ディレクトリにアップデートが表示されます。自動更新が有効になっている場合は、自動的に適用されます。

カタログのスキーマおよび提出プロセスの詳細については、[`wcpos/extensions`](https://github.com/wcpos/extensions) リポジトリを参照してください。
