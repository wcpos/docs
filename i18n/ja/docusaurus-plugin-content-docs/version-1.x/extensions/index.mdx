WCPOSは、POSに新しい機能を追加する拡張機能をサポートしています。拡張機能ディレクトリでは、利用可能な拡張機能を閲覧したり、POS設定から直接インストールしたり、更新を管理したりできます。

:::info Pro機能
拡張機能のインストールと管理には[WCPOS Pro](/getting-started/pro-license)が必要です。無料版では拡張機能カタログは表示されますが、インストールと有効化のコントロールは無効になります。
:::

## 拡張機能の閲覧 {#browsing-extensions}

`POS Settings > Extensions`（バージョンによっては **Plugins** と表示されます）から拡張機能ディレクトリを開きます。

ディレクトリには、利用可能な拡張機能がカードグリッドで表示されます。各カードには次の情報が表示されます。

- **アイコン**（拡張機能がアイコンを提供していない場合はパズルピースの代替アイコン）
- **名称とバージョン**
- **説明**
- **カテゴリーバッジ**
- **ステータス** — 有効、無効、アップデート可能、または未インストール

### フィルタリングと検索 {#filtering-and-search}

上部の**カテゴリーピルボタン**を使用して、拡張機能をカテゴリーで絞り込めます。**検索フィールド**を使用して、名前、説明、またはタグで拡張機能を検索することもできます。

## 拡張機能のインストール {#installing-an-extension}

1. `POS Settings > Extensions` を開きます。
2. インストールしたい拡張機能を見つけて、**インストール**をクリックします。
3. 拡張機能は、WordPressのプラグインインストーラーを使用してダウンロードおよびインストールされます。
4. インストールが完了したら、**有効化**をクリックして使用を開始します。

WCPOSは内部でWordPress標準の`Plugin_Upgrader`を使用してインストールを処理するため、拡張機能は通常のWordPressプラグインと同じプロセスに従います。

## 有効化と無効化 {#activating-and-deactivating}

インストール済みの各拡張機能のカードには、**有効化**ボタンと**無効化**ボタンがあります。

- **有効化** を実行すると、拡張機能が POS で利用できるようになります。
- **無効化** はアンインストールせずに拡張機能を停止します。拡張機能のファイルはサーバー上に残り、いつでも再有効化できます。

## 拡張機能の更新 {#updating-extensions}

インストール済み拡張機能の新しいバージョンが利用可能になると、カードに **更新あり** バッジと **更新** ボタンが表示されます。

### 自動更新 {#auto-updates}

ディレクトリからインストールした拡張機能は、**デフォルトで自動更新が有効** になっています。自動更新は、ディレクトリ内の各拡張機能のカードから個別にオン／オフを切り替えられます。自動更新が有効な場合、WordPress は自動更新が有効なプラグインと同様に、新しいバージョンを自動的に適用します。

## トラブルシューティング {#troubleshooting}

### ボタンに「Requires Pro」と表示される {#requires-pro-message-on-buttons}

WCPOS の無料版では、インストール・有効化・更新の各ボタンは無効化されています。拡張機能を管理するには [WCPOS Pro](/getting-started/pro-license) にアップグレードしてください。

### 拡張機能のインストールに失敗する {#extension-fails-to-install}

- WordPress サーバーが `wp-content/plugins` ディレクトリへの書き込み権限を持っているか確認してください。
- サーバーから外部への HTTPS リクエストが可能か確認してください（一部のホストでは外部からのダウンロードがブロックされています）。
- `WP Admin > POS > Support > Logs` でエラーの詳細を確認してください。

### インストール後に拡張機能が表示されない {#extension-not-appearing-after-install}

- POS を再読み込みしてください。拡張機能の一覧は最大 12 時間キャッシュされます。
- 拡張機能が有効化されていることを確認してください（インストール済みでも無効な拡張機能は動作しません）。

### カタログが読み込まれない {#catalog-not-loading}

拡張機能カタログはリモートソースから取得され、ローカルに 12 時間キャッシュされます。カタログが読み込まれない場合は、次を確認してください。

- サーバーのインターネット接続を確認してください。
- キャッシュの有効期限が切れた後に再試行するか、サーバーの一時キャッシュ（transient cache）をクリアしてください。

## 開発者向け {#for-developers}

### POS 拡張機能の作成 {#creating-a-pos-extension}

WCPOS 拡張機能は、WCPOS のフックと API を介して POS と統合する標準的な WordPress プラグインです。作成するには次の手順に従います。

1. **WordPressプラグインとして開始します。** 拡張機能には、WooCommerceやWordPressの通常のプラグインと同様に、標準的なプラグインヘッダーとエントリーファイルが必要です。
2. **WCPOSと統合します。** WCPOSが提供するフックやフィルターを使用して、POSインターフェースまたはバックエンドに機能を追加します。
3. **GitHubでリリースを公開します。** 拡張機能ディレクトリは、バージョン管理とアップデート配信にGitHub Releasesを利用します。

### ディレクトリへの登録 {#submitting-to-the-directory}

拡張機能カタログは、[`wcpos/extensions`](https://github.com/wcpos/extensions) GitHubリポジトリで管理されています。拡張機能を掲載するには、次の手順に従ってください。

1. リポジトリのREADMEで、カタログの形式とメタデータの要件を確認します。
2. `catalog.json`に拡張機能のメタデータを追加するプルリクエストを作成します。
3. マージされると、拡張機能はすべてのWCPOS Proユーザー向けのディレクトリに表示されます。

### GitHubリリースの規約 {#github-release-conventions}

アップデートのライフサイクルはGitHub Releasesに依存しています。

- **バージョンのタグ付け**にはセマンティックバージョニングを使用します（例：`v1.0.0`、`v1.2.3`）。
- **プラグインのZIPファイルを添付します**。リリースアセットとして添付したファイルが、ユーザーのインストールやアップデート時にダウンロードされます。
- ディレクトリで検出できるよう、リリースを**公開**してください（ドラフトのままにしないでください）。

新しいリリースを公開すると、拡張機能をインストールしているユーザーには、拡張機能ディレクトリに更新が表示されます。自動更新が有効になっている場合は、自動的に適用されます。

カタログスキーマと提出プロセスの詳細については、[`wcpos/extensions`](https://github.com/wcpos/extensions) リポジトリを参照してください。
