# WCPOS ATUM 連携 {#wcpos-atum-integration}

[ATUM Multi-Inventory](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/) を [WCPOS Pro](/getting-started/pro-license) と統合し、POS でロケーション別の在庫、価格、SKU を利用できるようにします。

ATUM Multi-Inventory を使うと、商品の在庫を倉庫や店舗など複数の在庫ロケーションに分割できます。本プラグインは、これらの ATUM ロケーションを WCPOS Pro の[店舗](/stores)と連携させ、各 POS 端末が物理ロケーションに応じた正しい在庫数、価格、SKU を参照できるようにします。

## 機能 {#features}

<FeatureGrid>
  <FeatureCard icon="box" title="ロケーション別在庫">
    各店舗は、WooCommerce の合算在庫ではなく、割り当てられた ATUM 在庫ロケーションから在庫数を取得します。
  </FeatureCard>
  <FeatureCard icon="store" title="柔軟な価格設定">
    WooCommerce のデフォルト価格、WCPOS Pro の店舗別価格、または ATUM のロケーション別価格から選択できます。
  </FeatureCard>
  <FeatureCard icon="barcode" title="ロケーション別 SKU">
    POSで商品のメインSKUをATUMロケーション固有のSKUに任意で置き換えることができます。
  </FeatureCard>
  <FeatureCard icon="sync" title="監査対応の在庫移動">
    注文時に正しいATUMロケーションで在庫が減算・復元され、すべての履歴が `atum_inventory_orders` に記録されます。
  </FeatureCard>
  <FeatureCard icon="arrow-right" title="商品編集の書き戻し">
    POSで編集した在庫数、価格、SKUは、該当ロケーションにマッピングされたATUMの在庫レコードに同期されます。
  </FeatureCard>
</FeatureGrid>

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ATUMとMulti-Inventoryをインストール">

[ATUM Inventory Management](https://wordpress.org/plugins/atum-stock-manager-for-woocommerce/) と [ATUM Multi-Inventory アドオン](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/) をインストールし、ATUMで在庫ロケーションを設定します。

  </StepCard>
  <StepCard number="2" title="WCPOS ATUM連携をインストール">

`WP Admin > POS > Settings > Extensions` からインストールするか、[GitHub リリースページ](https://github.com/wcpos/wcpos-atum/releases) から最新版をダウンロードし、`Plugins > Add New > Upload Plugin` からアップロードしてください。

  </StepCard>
  <StepCard number="3" title="店舗をATUMロケーションにマッピング" isLast>

`POS > Stores` に移動して店舗を編集し、サイドバーの **ATUM Inventory** セクションを設定します。店舗で使用する在庫ロケーションを選択し、価格ソースを指定し、必要に応じてSKUオーバーライドを有効にしてください。

  </StepCard>
</Steps>

## 店舗設定 {#store-configuration}

このプラグインは、WCPOS Pro 店舗エディターのサイドバーに **ATUM 在庫** セクションを追加し、店舗ごとに3つの設定を提供します。

- **在庫ロケーション** — この店舗が在庫を取得する ATUM ロケーション。
- **価格ソース** — 商品価格の取得元：
  - *デフォルト* — 標準の WooCommerce 価格
  - *WCPOS Pro* — WCPOS Pro で設定された店舗別価格
  - *ATUM* — ATUM 在庫からのロケーション別価格
- **SKU オーバーライド** — 商品のメイン SKU の代わりに、ATUM のロケーション別 SKU を必要に応じて使用します。

## POS の動作 {#pos-behavior}

店舗に ATUM ロケーションが割り当てられている場合、POS に提供される商品データは自動的に調整されます。

- **在庫数** は、WooCommerce の合計在庫ではなく、該当ロケーションの在庫を反映します。
- **在庫ステータス** は、ロケーションの在庫数に基づいて再計算されます。
- **価格**は、設定された価格ソースから取得されます。
- **SKU** は、上書きが有効になっている場合、ATUM ロケーションの SKU に置き換えられます。

すべての調整は WCPOS REST API を介して透過的に行われ、POS アプリ側での変更は不要です。POS から行われた商品の編集も、マッピングされた ATUM 在庫行に書き戻されます。詳細は以下の[商品編集の書き戻し](#product-edit-write-back)を参照してください。

## 在庫管理 {#stock-management}

マッピングされた ATUM ロケーションを持つ店舗で行われた POS 注文について、プラグインは ATUM のネイティブな在庫減算フローに書き込みを処理させますが、正しいロケーションへ誘導します:

1. **REST ペイロードの挿入。** POS が注文を作成または更新する際、プラグインは各明細に `mi_inventories` エントリを挿入し、ATUM がどのロケーションから引き当てるべきかを認識できるようにします。これがないと、ATUM はメイン在庫にフォールバックします。
2. **ロケーションに絞り込んだ在庫フィルター。** プラグインは ATUM の候補在庫リストを、店舗にマッピングされたロケーション用語に紐づくものだけにフィルタリングし、減算時と復元時の両方で正しい在庫が選択されることを保証します。

ATUM 自体が注文および返金時に実際の在庫変更を実行し、`atum_inventory_orders` に実際の `order_id` を持つ行を書き込むことで、ATUM の監査証跡が保持されます。

## 商品編集の書き戻し {#product-edit-write-back}

レジ担当者やマネージャーが POS から商品やバリエーションを編集すると、その変更はメインの WooCommerce 商品だけでなく、その店舗のロケーションにマッピングされた ATUM 在庫行へも書き戻されます。これにより、`WP Admin` で手動更新を行うことなく、各ロケーションの在庫、価格、SKU が ATUM と同期された状態に保たれます。

この書き戻しは、WCPOS の商品およびバリエーションに対する REST 更新 (`POST`, `PUT`, `PATCH` の `/wcpos/v1/products/...` への送信) のうち、`store_id` を含むものでトリガーされます。プラグインは店舗にマッピングされた ATUM ロケーションを参照し、そのロケーションの在庫行のみを更新します。他のロケーションは変更されません。

### 同期される項目 {#what-syncs}

書き戻しは各店舗の設定を尊重するため、ATUMデータはその店舗が実際にデータを所有している場合にのみ変更されます:

| フィールド | 同期されるタイミング |
| --- | --- |
| **在庫数量** | 常時 — ATUMロケーションがマッピングされたすべての店舗が、そのロケーション行を同期し続けます。 |
| **通常価格 / セール価格 / 価格** | 店舗の**価格ソース**が*ATUM*に設定されている場合のみ。 |
| **SKU** | 店舗で**SKU上書き**が有効になっている場合のみ。 |

店舗が*デフォルト*または*WCPOS Pro*の価格設定を使用している場合、ATUMの価格フィールドはそのまま残されるため、ATUMは正規の情報源ではなく参照価格として機能し続けます。SKUについても、上書きがオフの場合は同様です。

### 書き戻しがトリガーされない条件 {#what-doesn-t-trigger-write-back}

- 製品の作成(書き戻されるのは更新のみ — 新規製品はWooCommerceの通常の保存パスに渡されます)。
- `store_id` を含まないリクエスト — POS はどのロケーションに書き込むかをプラグインに伝える必要があります。
- ATUMロケーションがマッピングされていない店舗。
- 店舗のロケーションに対する既存のATUM在庫行がない製品 — プラグインは新しい在庫行を作成せず、既存の行のみを更新します。
