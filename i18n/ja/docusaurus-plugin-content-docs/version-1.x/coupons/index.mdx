:::info Pro機能
クーポン画面およびレジでのクーポン適用機能を利用するには、[WCPOS Pro](/getting-started/pro-license)が必要です。無料ユーザーにはクーポン画面のぼかしプレビューが表示されますが、クーポンの詳細を確認したり、カートでコードを適用したりすることはできません。
:::

クーポン画面では、POS内からWooCommerceクーポンを閲覧・検索でき、カートの**クーポンを追加**ボタンで現在の注文に適用できます。クーポン自体の作成と管理はWooCommerceで行います。POSはクーポンを同期してローカルで検証するため、レジで即座にフィードバックが得られます。

## インターフェース概要 {#interface-overview}

### ヘッダーアクション {#header-actions}

画面上部：

- **検索バー** — コードでクーポンを検索
- **表示設定** (<Icon name="sliders" />) — 表示するカラムを設定します

### クーポンテーブル {#coupons-table}

メインエリアには、同期されたクーポンが以下の情報とともに表示されます：

- **コード** — 顧客が入力するクーポンコード
- **説明** — 内部説明（設定時にはレシートラベルとしても使用されます）
- **割引タイプ** — パーセント、固定カート、または固定商品
- **金額** — 割引額
- **使用回数 / 上限** — 使用回数と全体の使用上限
- **有効期限** — 有効期限日（未設定の場合は空欄）
- **アクション** — 三点メニュー

### フッター {#footer}

- 同期ボタン付きのクーポン数 (<Icon name="arrow-rotate-right" />)。**長押し**でクリアと更新のオプションを表示します。

## クーポンの種類 {#coupon-types}

WCPOSは、WooCommerceの3つの標準クーポンタイプに対応しています：

| 種類 | 説明 | 例 |
|---|---|---|
| **割合割引** | カート小計をパーセンテージで割引します | 注文から `10%` 割引 |
| **カート定額割引** | カート合計を固定額で割引します | 注文から `$5` 割引 |
| **商品定額割引** | 対象商品を1個あたりの固定額で割引します | Tシャツ1枚につき `$2` 割引 |

3つのタイプすべてに同じバリデーションルール（有効期限、使用回数制限、商品/カテゴリの制限など）が適用されます。

## クーポンの作成 {#creating-coupons}

クーポンはPOSではなく、WooCommerceで作成します。**WP Admin → マーケティング → クーポン → クーポンを追加**に移動します：

1. **クーポンコード** — レジでキャッシャーが入力するコードです。大文字・小文字は区別されません。
2. **説明** — 内部メモです。WCPOSは設定されている場合、この内容をレシートの割引ラベルとして使用するため、内部コードではなく「マネージャー割引 10%」のような顧客向けの表現を推奨します。
3. **割引タイプ** — パーセンテージ、固定カート、または固定商品。
4. **クーポン金額** — 割引の値です。
5. **有効期限** — 任意設定です。この日付を過ぎるとクーポンは拒否されます。

### 利用制限 {#usage-restriction}

- **最低利用金額 / 最高利用金額** — 小計が指定した金額以上/以下の場合にのみクーポンが適用されます。
- **個別利用のみ** — このクーポンを他のクーポンと併用できないようにします。
- **セール商品を除外** — すでにセール中の商品（およびキャッシャーがレジで価格を下げた明細行）をスキップします（[POSの価格変更](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)を参照）。
- **対象商品 / 除外商品** — クーポンを特定の商品に限定、または特定の商品を除外します。
- **対象カテゴリ / 除外カテゴリ** — カテゴリで制限します。POSで作成されたその他/カスタム商品にも、POSで割り当てたカテゴリが適用されます。
- **許可メールアドレス** — クーポンを特定の顧客メールアドレスに限定します（`*` ワイルドカード対応）。

### 使用制限 {#usage-limits}

- **クーポンごとの使用回数制限** — すべての顧客を通じて、このコードを使用できる合計回数です。
- **ユーザーごとの使用回数制限** — 1人の顧客が使用できる回数です。
- **使用をX個の商品に制限** — 固定商品クーポンの場合、1回の使用で割引が適用される対象商品の数を制限します。

### POS での活用ヒント {#tips-for-pos-use}

- **「マネージャー 10%」** クーポン（10% オフ、最低利用金額なし、有効期限なし、1回限りの制限オフ）を作成し、マネージャーにコードを共有すれば、スタッフがレジで臨時の調整に適用でき、レポートには追跡可能な割引として記録されます。
- リピーター特典用に **「ロイヤルティ $5」** クーポンを作成します。
- 単発のプロモーションには、短い有効期限を設定しておくと、誤って再利用されることを防げます。
- **説明** にレシートに印刷したいフレーズを設定しておくと、割引に独自のブランド表記を付ける最も簡単な方法になります。

## レジでのクーポン適用 {#applying-a-coupon-at-the-register}

カート内の **クーポンを追加** をタップし、コードを入力するか説明で検索します。クーポンは同期済みデータに基づいて即座に検証され、カート合計の上に削除可能なピルとして表示されます。

検索、クーポンピル、複数クーポンの併用、検証エラーの解決など、レジでの完全なワークフローについては、**[レジでのクーポン適用](/coupons/applying-coupons)** を参照してください。

## バリデーションの仕組み {#how-validation-works}

コードが入力されると、POSはWooCommerceがサーバー上でチェックするのと同じルールをすべて検証します：

- クーポンが存在し、有効期限内であること
- 使用回数の上限を超えていないこと（全体およびユーザーごと）
- 最小/最大利用金額の条件を満たしていること
- 「個別利用」クーポンが適用済みのクーポンと競合しないこと
- 商品/カテゴリの制限がカートの内容と一致すること
- メール制限が選択された顧客と一致すること（ゲスト注文の場合はスキップ）
- **セール商品を除外**が有効な場合、セール商品は除外されること

いずれかのチェックに失敗すると、レジ担当者に具体的なエラーメッセージが表示されます（例：「このクーポンは有効期限切れです」「最低利用金額に達していません」）。注文送信時にサーバー上で同じチェックが再度実行されます — ローカルバリデーションは速度のためであり、信頼性を担保するものではありません。

## 同期の動作 {#sync-behaviour}

クーポンは、WCPOSの標準レプリケーションパイプラインを通じてWooCommerceからデバイスに同期されます：

- WP Admin で作成した新しいクーポンは、次回の同期時に POS に表示されます。
- 既存クーポンの更新（使用回数、有効期限の変更など）は自動的に同期されます。
- クーポンは IndexedDB にローカル保存されるため、オフラインでも利用できます。

WP Admin でクーポンを作成したのにまだ表示されない場合は、同期がまだ実行されていない可能性があります。クーポン画面のフッターから同期アイコン（<Icon name="arrow-rotate-right" />）をタップして更新してください。完全に再取得する必要がある場合は、長押しして**クリアして更新**を選択してください。

## 接続要件 {#connectivity}

- **クーポンの適用**は、同期済みデータによるクライアント側の検証で処理されるため、オフラインでも動作します。
- **会計の完了**にはサーバー接続が必要です。サーバー側で再検証が行われ、最終的な注文合計の正式な算出元となります。
- **クーポンの作成・編集**は WP Admin で行うため、WordPress サイトへの接続が必要です。

## クーポンと POS 割引の関係 {#how-coupons-interact-with-pos-discounts}

レジでスタッフが明細の価格を下げた後にクーポンを適用すると、クーポンは元の価格ではなく**値下げ後の価格**に対して計算されます。POS で値下げされた明細は「セール中」として扱われるため、**セール品を除外**が有効なクーポンはそれらをスキップします。詳細は[割引](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)を参照してください。
