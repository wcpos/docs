国によっては、販売時点情報管理ソフトウェアが税務不正防止のために**認証**を受けること、または国の税務当局と連携することが法的に求められます。このページでは、WCPOS の対応状況をまとめています。利用を前提にする**前に**、自分の管轄区域に適合するか確認してください。

:::warning WCPOS は認証済みの財務 POS ではありません
WCPOS は現在、規制対象市場向けの財務認証を**取得していません**。お使いの国で認証済み POS ソフトウェアが求められる場合は、実際の販売に WCPOS を使用する前に、地域の要件を確認してください。WCPOS は WooCommerce のミドルウェアとして機能します。請求書発行と VAT は最終的に WooCommerce 側で処理されます。
:::

:::info 対応を進めています
財務コンプライアンスは現在も継続的に開発している分野です。私たちの目標は、ユーザーが事業を行うすべての国で WCPOS を準拠させることです。以下の状況は、今後の予定ではなく**現時点**のものです。お住まいの地域でのコンプライアンスが重要な場合は、[コミュニティ](https://wcpos.com/discord)でお知らせください。次に対応する市場の優先順位付けに役立ちます。
:::

## WCPOS での税金の処理方法 {#how-wcpos-handles-tax}

WCPOS は WooCommerce からダウンロードした税率を使用してアプリ内で税金をローカル計算し、その後 WooCommerce に注文を作成します。これは高速でオフラインでも動作しますが、認証済みまたはサーバー側の税務ロジック（一部のコンプライアンスプラグインなど）は、注文がサーバーに到達した時点でのみ実行されることを意味します。**「サーバーに保存」**アクションと、**Web チェックアウト** / **メール請求書**ゲートウェイは、標準の WooCommerce チェックアウトを通じて注文を送信し、そこでサーバー側の税金およびコンプライアンスルールが適用されます。

## 国別ステータス {#country-status}

### フランス — NF525 {#france-nf525}

WCPOS は NF525 の認証を**取得していません**。フランスの法律では POS システムに不正防止認証が義務付けられているため、フランスでの販売可否に影響します。WCPOS は WooCommerce 向けのミドルウェアであるため、請求書発行と VAT コンプライアンスは WooCommerce レベルで処理されます。WooCommerce の [フランスの VAT 不正防止法に関するガイダンス](https://woocommerce.com/posts/french-vat-fraud-prevention-laws-how-to-certify-invoices-with-woocommerce/)を参照してください。

### スペイン — VERIFACTU {#spain-verifactu}

WCPOS は VERIFACTU の認証を**取得していません**。スペインでは POS ソフトウェアの認証が義務付けられていますが、VERIFACTU の期限は**2027年に延期**されています。チームはこの要件を認識していますが、認証の時期は確定していません。

### ノルウェー — Kassasystemloven {#norway-kassasystemloven}

WCPOS はノルウェーのキャッシュレジスター法（_Kassasystemloven_）の認証を**取得しておらず**、サプライヤー宣言もありません。ノルウェーの一部の事業者は POS を使用していますが、自社の法的義務を確認してください。

### チリ — SII {#chile-sii}

検証済み販売レシート（_boletas_）向けのチリ内国歳入庁（Servicio de Impuestos Internos、SII）との**連携はありません**。レシートテンプレートは標準のカスタマイズ可能な WooCommerce テンプレートですが、SII コンプライアンスは組み込まれていません。

### EU — One Stop Shop (OSS) {#eu-oss}

POS は、国境を越えた販売向けの EU One Stop Shop (OSS) VAT スキームをネイティブには**サポートしていません**。ダウンロード済みの税率からローカルで税金を計算するためです。

**回避策:**
- チェックアウト前に **サーバーに保存** を使用して、OSSプラグイン経由でサーバー側の税額再計算を実行するか、
- **Webチェックアウト** または **メール請求書** ゲートウェイを使用して、OSSルールが適用される標準のWooCommerceチェックアウトで支払いを受け付けます。

## その他の規制対象市場 {#other-regulated-markets}

認定POSソフトウェアが必要な法域では、まず現地の要件を確認してください。WCPOSは現在、規制対象市場向けの認証を取得していません。認証状況が購入判断に影響する場合は、決定する前に[コミュニティ](https://wcpos.com/discord)で確認してください。
