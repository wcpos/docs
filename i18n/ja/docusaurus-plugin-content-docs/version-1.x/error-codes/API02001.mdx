# API02001: 無効な認証情報

## この意味

入力されたユーザー名またはパスワードが正しくありません。サーバーは、認証情報が有効なアカウントと一致しないため、ログイン試行を拒否しました。

## 一般的な原因

- **ユーザー名またはパスワードの入力ミス** — 大文字ロックや余分なスペースを確認してください
- **間違ったアカウント** — 別のサイトの認証情報を使用
- **最近パスワードを変更** — パスワードが更新されたが、POSに古いパスワードが残っている
- **アカウント削除** — ユーザーアカウントが既に存在しない

## サーバーエラーマッピング

このエラーコードは、サーバーが以下を返す場合にトリガーされます：

| サーバーコード | ソース |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT認証プラグイン |

## 修正方法

### 1. 認証情報の確認

- ユーザー名（通常はメールアドレスまたはWordPressユーザー名）を再確認
- パスワードが正しいことを確認
- WordPress管理画面に直接ログインして確認

### 2. パスワードのリセット

パスワードを忘れた場合：
1. WordPressログインページに移動
2. 「パスワードを忘れましたか？」をクリック
3. メールアドレスを入力してリセットリンクを受け取る
4. パスワードを更新
5. 新しいパスワードでPOSにログイン

### 3. ユーザーアカウントの確認

WordPress管理画面で：
- ユーザー → すべてのユーザーに移動
- アカウントが存在することを確認
- POS利用に適切な権限があることを確認

### 4. 保存された認証情報のクリア

POSで：
- 完全にログアウト
- 保存または キャッシュされたログイン情報をクリア
- 認証情報を最初から再入力
