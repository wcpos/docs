# API02001: 無効な資格情報

## これは何を意味するか

入力したユーザー名またはパスワードが正しくありません。資格情報が有効なアカウントと一致しないため、サーバーはログイン試行を拒否しました。

## 一般的な原因

- **ユーザー名またはパスワードのタイプミス** — Caps Lockや余分なスペースを確認してください
- **間違ったアカウント** — 別のサイトの資格情報を使用している
- **パスワードが最近変更された** — パスワードが更新されたが、POSは古いパスワードを使用している
- **アカウントが削除された** — ユーザーアカウントはもはや存在しない

## サーバーエラーのマッピング

このエラーコードは、サーバーが次のようなコードを返すときにトリガーされます：

| サーバーコード | ソース |
|----------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT 認証プラグイン |

## 修正方法

### 1. 資格情報を確認する

- ユーザー名（通常はメールアドレスまたはWordPressのユーザー名）が正しいか再確認してください
- パスワードが正しいことを確認してください
- 確認のために直接WordPress管理画面にログインしてみてください

### 2. パスワードをリセットする

パスワードを忘れた場合：
1. WordPressのログインページに行きます
2. 「パスワードをお忘れですか？」をクリックします
3. リセットリンクを受け取るためのメールアドレスを入力します
4. パスワードを更新します
5. 新しいパスワードでPOSにログインします

### 3. ユーザーアカウントを確認する

WordPress管理画面で：
- ユーザー → すべてのユーザーに移動します
- アカウントが存在することを確認します
- 適切なPOSアクセス権限があるか確認します

### 4. 保存された資格情報をクリアする

POSで：
- 完全にログアウトします
- 保存された/キャッシュされたログイン情報をクリアします
- 新たに資格情報を再入力します
