# API03007: リクエストキュー満杯

## この意味

POSには、送信を待っている保留中のリクエストが多すぎます。内部キューが満杯で、一部のリクエストが完了するまで、新しいリクエストを受け入れることができません。

## よくある原因

- **ネットワークの遅さ** — 接続が遅いためにリクエストが滞留
- **サーバーの応答が遅い** — サーバーが応答に時間がかかりすぎている
- **高速な操作** — 処理できないほど速く多くの操作を実行
- **同期の過負荷** — 大規模な同期操作によりキューがブロックされている

## 解決方法

### 1. キューがクリアされるまで待つ

保留中のリクエストが完了するまで待ちます：
- 30〜60秒待つ
- 同期インジケーターが完了するのを確認
- クリアされるまで追加の操作を避ける

### 2. ネットワーク接続を確認

接続が遅いとバックアップが発生します：
- インターネット速度をテスト
- より速い接続を試す
- WiFiルーターの近くに移動

### 3. サーバーの応答時間を確認

サーバーが遅い場合：
- ブラウザでサイトにアクセスしてテスト
- サイトが遅い場合はホスティング先に連絡
- サーバー負荷が減少するまで待つ

### 4. POSを再起動

キューが固まっている場合：
1. POSアプリケーションを閉じる
2. しばらく待つ
3. POSを再度開く
4. 保留中のリクエストを再度開始する必要がある場合がある

### 5. 高速な操作を避ける

同期または遅い期間中：
- 1つのアクションが完了するまで次のアクションを開始しない
- ボタンを繰り返しクリックしない
- ネットワーク操作に対して忍耐強くする

## オフライン時の動作

オフライン状態でキューが満杯になった場合：
- POSは、オンラインに戻るまで保留中のリクエストを保持
- キャッシュされた商品と顧客を参照できる
- 新しい注文を開始し、カートにアイテムを追加できる
- 注文を完了するには、接続が復旧する必要がある
