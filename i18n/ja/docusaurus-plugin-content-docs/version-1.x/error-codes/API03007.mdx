# API03007: リクエストキューが満杯

## これが意味すること

POSには送信待ちの保留中リクエストが多すぎます。内部キューが満杯になっており、いくつかが完了するまで新しいリクエストを受け入れることができません。

## 一般的な原因

- **ネットワークの遅延** — 遅い接続によってリクエストが滞っている
- **サーバーの応答なし** — サーバーが応答するのに時間がかかっている
- **迅速な操作** — 処理できるよりも早く多くの操作を行っている
- **同期過負荷** — 大きな同期操作がキューをブロックしている

## 修正方法

### 1. キューがクリアされるのを待つ

保留中のリクエストが完了するまで待ちます：
- 30～60秒待つ
- 同期インジケーターが完了するのを確認する
- クリアされるまで追加の操作を避ける

### 2. ネットワーク接続を確認する

遅い接続はバックアップを引き起こします：
- インターネット速度をテストする
- より速い接続を試す
- WiFiルーターに近づく

### 3. サーバーの応答時間を確認する

サーバーが遅い場合：
- ブラウザでサイトにアクセスしてテストする
- サイトが遅い場合はホスティングに連絡する
- サーバーの負荷が減少するのを待つ

### 4. POSを再起動する

キューが固まっている場合：
1. POSアプリケーションを閉じる
2. 少し待つ
3. POSを再度開く
4. 保留中のリクエストは再起動が必要になる場合があります

### 5. 迅速な操作を避ける

同期中や遅い時期に：
- 別の操作を開始する前に1つの操作が完了するのを待つ
- 繰り返しボタンをクリックしない
- ネットワーク操作には忍耐を持つ

## オフライン時の挙動

オフライン条件によってキューが満杯になると：
- POSはオンラインに復帰するまで保留中のリクエストを保持します
- キャッシュされた製品や顧客を参照することはできます
- 新しい注文を開始し、カートにアイテムを追加することはできます
- 注文を完了させるには接続を復元する必要があります
