# API04006: リソースが見つかりません

## これは何を意味するか

サーバーが要求されたリソースを見つけることができませんでした。これは通常、HTTP 404ステータスコードに対応し、アクセスしようとした製品、注文、顧客、その他のリソースが存在しないことを示しています。

## よくある原因

- **レコードの削除** — サーバー上でリソースが削除された
- **ID の誤り** — 不正または古い ID が使用されている
- **同期の問題** — ローカルデータが存在しないリソースを参照している
- **URL の設定ミス** — API エンドポイントが正しくない
- **パーマリンクの問題** — WordPress のパーマリンクを更新する必要がある

## サーバーエラーのマッピング

このエラーコードは、サーバーが以下を返した場合にトリガーされます：

| サーバーコード | ソース |
|-------------|--------|
| `rest_no_route` | WordPress REST API |
| HTTP 404 | 任意のサーバーレスポンス |

## 修正方法

### 1. リソースの存在を確認する

WooCommerce サイト上でリソースがまだ存在することを確認：
- WordPress 管理画面にログイン
- 該当するセクション（製品、注文、顧客）に移動
- ID または名前で項目を検索

### 2. ローカルデータを更新

リソースがサーバー側で削除された場合：
1. POS 設定を開く
2. 該当するデータセクションに移動
3. 同期/更新を実行してローカルデータを更新
4. 削除された項目がローカルから削除されるはず

### 3. WordPress パーマリンクを確認

複数のリソースが見つからない場合：
1. WordPress 管理画面で **設定 → パーマリンク** に移動
2. **変更を保存** をクリック（変更しなくても）
3. パーマリンク構造が更新されます

### 4. API ルートを確認

REST API を直接テスト：
```
https://yoursite.com/wp-json/wc/v3/products
```
これが 404 を返す場合、サーバー設定に問題がある可能性があります。

### 5. プラグインの競合を確認

REST API ルートが見つからない場合：
1. WooCommerce が有効であることを確認
2. WCPOS プラグインが有効であることを確認
3. 一時的に他のプラグインを無効にしてテスト
