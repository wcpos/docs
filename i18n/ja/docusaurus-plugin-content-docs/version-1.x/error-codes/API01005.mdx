# API01005: SSLサーティフィケートエラー

## これが意味するもの

ウェブサイトのSSL/TLSサーティフィケートに問題があります。このサーティフィケートは接続をセキュアにする（HTTPS）ものです。POSは、データを保護するために、無効なサーティフィケートのあるサイトには接続しません。

## 一般的な原因

- **期限切れのサーティフィケート** — SSLサーティフィケートは定期的に更新する必要があります
- **自己署名サーティフィケート** — 信頼できる認証局によって発行されていません
- **ドメイン不一致** — サーティフィケートがドメイン名と一致していません
- **不完全な証明書チェーン** — 中間証明書が不足しています
- **混在コンテンツ** — 一部のリソースがHTTPSではなくHTTPで読み込まれています

## 修正方法

### 1. サーティフィケートの状態を確認

ブラウザでサイトにアクセスし、ロックアイコンをクリックして証明書の詳細を表示します。以下を確認します：
- 有効期限
- 発行先（ドメインと一致するか）
- 発行元（認められた機関であるか）

### 2. 期限切れサーティフィケートの更新

期限切れの場合：
- ほとんどのホスティングプロバイダーは無料のLet's Encryptサーティフィケートを提供
- ホスティングプロバイダーに更新を依頼
- 自動更新が有効かどうかを確認

### 3. サーティフィケートの不一致を修正

以下を確実にカバーするサーティフィケート：
- 正確なドメイン（`yourstore.com`）
- 使用している場合はWWWバリアント（`www.yourstore.com`）
- ワイルドカード証明書の検討（`*.yourstore.com`）

### 4. 不足している中間証明書のインストール

SSL診断ツール（SSL Labsなど）を使用して、不足している証明書を特定します。ホスティングプロバイダーがインストールを支援できます。

### 5. HTTPSの強制

WordPress内で以下を確認：
- サイトURLが`https://`を使用
- WooCommerceの設定でSSLが強制されている
