# DB02002: レコードが見つかりません {#db02002-record-not-found}

## これはどういう意味か {#what-this-means}

リクエストされたレコードはローカルデータベースに存在しません。POSは、ローカルに保存されていない製品、顧客、注文、またはその他のレコードにアクセスしようとしました。

## 一般的な原因 {#common-causes}

- **まだ同期されていない** — データがまだダウンロードされていない
- **サーバーで削除された** — WooCommerceでレコードが削除された
- **同期が完了していない** — 最初の同期がまだ実行中
- **フィルターが適用されている** — アクティブなフィルターによってレコードが除外されている

## 修正方法 {#how-to-fix}

### 1. 同期を待つ {#1-wait-for-sync}

最近始めた場合:
- 最初の同期が完了するまで待つ
- 同期進行状況インジケーターを確認
- 同期が完了するとレコードが表示される

### 2. 更新/再同期 {#2-refreshre-sync}

データの更新をトリガーします:
- 更新/同期ボタンを使用する（<Icon name="arrow-rotate-right" />）- 短押しで同期
- **長押し**で同期ボタンを押してクリアとリフレッシュオプションを選択
- サーバーから最新データを取得
- 完了するまで待つ

### 3. WooCommerceを確認 {#3-check-woocommerce}

サーバー上にレコードが存在するか確認します:
- WordPress管理画面にログイン
- 製品、注文、または顧客を確認
- アイテムがまだ存在することを確認

### 4. フィルターを確認 {#4-check-filters}

レコードがフィルターで除外されている可能性があります:
- POS内のアクティブフィルターを確認
- カテゴリーフィルターをチェック
- 可視性設定を確認

### 5. データをクリアし再同期 {#5-clear-and-re-sync}

同期が停止しているように見える場合:
1. ローカルデータをクリア
2. 再度ログイン
3. 新たに同期を完了

## 一般的なシナリオ {#common-scenarios}

- **製品が表示されない:** 発行されており、可視であるか確認
- **顧客が見つからない:** まだ同期されていない可能性がある
- **注文が欠落している:** 同期の期間外かもしれない
