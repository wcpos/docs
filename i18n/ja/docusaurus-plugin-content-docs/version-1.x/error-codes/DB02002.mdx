# DB02002: レコードが見つかりません

## この意味

要求されたレコードがローカルデータベースに存在しません。POSは、まだローカルに保存されていない商品、顧客、注文、またはその他のレコードにアクセスしようとしました。

## 一般的な原因

- **まだ同期されていない** — データがまだダウンロードされていない
- **サーバーで削除された** — レコードがWooCommerceで削除された
- **同期が未完了** — 初期同期がまだ実行中
- **フィルターが適用されている** — レコードが有効なフィルターから除外された

## 解決方法

### 1. 同期を待つ

最近開始した場合:
- 初期同期の完了を待つ
- 同期進捗インジケーターを確認
- 同期されると、レコードが表示されます

### 2. 更新/再同期

データ更新をトリガー:
- 更新/同期ボタンを使用
- サーバーから最新のデータを取得
- 完了を待つ

### 3. WooCommerceを確認

サーバー上にレコードが存在することを確認:
- WordPress管理画面にログイン
- 商品、注文、または顧客を確認
- アイテムがまだ存在することを確認

### 4. フィルターを確認

レコードがフィルターで除外されている可能性があります:
- POS内のアクティブなフィルターを確認
- カテゴリーフィルターを確認
- 表示設定を検証

### 5. クリアして再同期

同期が停止している場合:
1. ローカルデータをクリア
2. 再度ログイン
3. 新しい同期を完了

## 一般的なシナリオ

- **商品が表示されない:** 公開され、表示可能であることを確認
- **顧客が見つからない:** まだ同期されていない可能性がある
- **注文が見つからない:** 同期日付範囲外の可能性がある
