# DB02002: レコードが見つかりません

## これは何を意味するのか

要求されたレコードはローカルデータベースに存在しません。POSはローカルに保存されていない製品、顧客、注文、またはその他のレコードにアクセスしようとしました。

## 一般的な原因

- **まだ同期されていない** — データがまだダウンロードされていません
- **サーバーで削除された** — WooCommerceでレコードが削除されました
- **同期が完了していない** — 初期同期がまだ実行中です
- **フィルターが適用されている** — アクティブなフィルターによりレコードが除外されました

## 修正方法

### 1. 同期を待つ

最近始めた場合：
- 初期同期が完了するまで待つ
- 同期進捗インジケーターを確認する
- 同期が完了するとレコードが表示されるはずです

### 2. 更新/再同期

データの更新をトリガーする：
- 更新/同期ボタンを使用する
- サーバーから最新のデータを取得する
- 完了するまで待つ

### 3. WooCommerceを確認

サーバー上にレコードが存在することを確認する：
- WordPress管理にログイン
- 製品、注文、または顧客を確認
- アイテムがまだ存在することを確認

### 4. フィルターを確認

レコードがフィルタリングされている可能性があります：
- POS内のアクティブなフィルターを確認
- カテゴリフィルターを確認
- 可視性設定を確認

### 5. ローカルデータをクリアして再同期

同期がスタックしている場合：
1. ローカルデータをクリアする
2. 再度ログインする
3. 新たに同期を完了する

## 一般的なシナリオ

- **製品が表示されない:** 公開されていて表示されているか確認する
- **顧客が見つからない:** まだ同期されていない可能性があります
- **注文が欠落:** 同期日付範囲外にある可能性があります
