# API02003: トークンが無効 {#api02003-token-invalid}

## これは何を意味するか {#what-this-means}

使用されている認証トークンがサーバーによって認識されていません。これはトークンが期限切れであることとは異なります — このトークンは有効でなかったか、取り消されています。

## 一般的な原因 {#common-causes}

- **トークンが取り消された** — 管理者がセッションを取り消しました
- **トークンが破損した** — 保存または送信中のデータ破損
- **サーバーリセット** — サーバーの秘密鍵が変更され、すべてのトークンが無効になりました
- **間違ったサーバー** — 別のWordPressインストールからのトークン

## サーバーエラーのマッピング {#server-error-mapping}

このエラーコードは、サーバーが返すときにトリガーされます：

| サーバーコード | ソース |
|----------------|--------|
| `jwt_auth_invalid_token` | JWT認証プラグイン |

## 修正方法 {#how-to-fix}

### 1. 再ログインする {#1-log-in-again}

セッションをクリアし、新たに認証します：
1. 完全にログアウトする
2. POSを閉じて再度開く
3. 認証情報を使ってログインする

### 2. アプリケーションデータをクリアする {#2-clear-application-data}

ログアウトしても効果がない場合：
- POSアプリのキャッシュ/データをクリアする
- ウェブの場合：サイトのブラウザクッキーとローカルストレージをクリアする
- デスクトップの場合："データをクリア"オプションがアプリ設定にあるか確認する

### 3. サーバー設定を確認する {#3-check-server-configuration}

この問題がすべてのユーザーに影響を与える場合：
- WordPressのソルトが変更されたか確認する
- JWTまたは認証プラグインの設定を確認する
- 最近のサーバー変更をレビューする

### 4. 正しいサーバーに接続していることを確認する {#4-verify-youre-on-the-correct-server}

POSが正しいWordPressサイトに接続するように設定されていることを確認します。特に複数のインストールがある場合は注意が必要です。
