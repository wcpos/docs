# API02003: トークン無効

## この意味

サーバーに認識されない認証トークンが使用されています。期限切れトークンとは異なり、このトークンは最初から有効ではないか、無効化されています。

## よくある原因

- **トークン無効化** — 管理者によってセッションが無効化された
- **トークン破損** — 保存または転送中のデータ破損
- **サーバーリセット** — サーバーのシークレットキーが変更され、すべてのトークンが無効化された
- **間違ったサーバー** — 異なるWordPressインストール環境からのトークン

## サーバーエラーマッピング

このエラーコードは、サーバーが以下を返す際にトリガーされます：

| サーバーコード | 送信元 |
|-------------|--------|
| `jwt_auth_invalid_token` | JWおよび認証プラグイン |

## 解決方法

### 1. 再ログイン

セッションをクリアし、新たに認証します：
1. 完全にログアウトする
2. POSアプリを閉じて再起動
3. 資格情報で再ログイン

### 2. アプリケーションデータのクリア

ログアウトで解決しない場合：
- POSアプリのキャッシュ/データをクリア
- Web版：サイトのブラウザクッキーとローカルストレージをクリア
- デスクトップ版：「データクリア」オプションを確認

### 3. サーバー設定の確認

全ユーザーに影響がある場合：
- WordPressソルトが変更されていないか確認
- JWTまたは認証プラグインの設定を検証
- 最近のサーバー変更を確認

### 4. 正しいサーバーであることの確認

特に複数のインストール環境がある場合、POSが正しいWordPressサイトに接続されていることを確認します。
