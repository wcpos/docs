# API06002: APIのURLが見つかりません

## この意味

WooCommerceストアに接続するためのURLが設定されていません。POSはストアの場所を知る必要があります。

## 一般的な原因

- **初回セットアップ** — URLがまだ入力されていない
- **設定のクリア** — 設定がリセットまたは消去された
- **アプリデータのクリア** — アプリケーションデータが削除された
- **移行の問題** — 更新または移行中にURLが失われた

## 修正方法

### 1. ストアのURLを入力

POSアプリで：
1. 設定またはログイン画面を開く
2. URL/サイト設定フィールドを見つける
3. WordPressサイトのURLを入力
4. 例：`https://yourstore.com`

### 2. WordPressのURLを探す

URLが不明な場合：
1. WordPressの管理画面にログイン
2. 設定 → 一般に移動
3. 「WordPressアドレス（URL）」を確認
4. このURLをPOSで使用

### 3. 保存された設定を確認

以前URLが設定されていた場合：
- アプリデータが誤って消去されていないか確認
- 設定のバックアップ/エクスポートオプションを探す
- 必要に応じてURLを再入力

### 4. URLの正確性を確認

保存する前に：
1. ブラウザでURLをテスト
2. WordPressサイトが読み込まれることを確認
3. `https://`または`http://`を含める

## セットアップチェックリスト

WCPOSを初めて設定する際：

1. ☐ WordPressサイトのURLを入力
2. ☐ WCPOSプラグインがインストールされ、有効化されていることを確認
3. ☐ WordPressの認証情報でログイン
4. ☐ 初期データ同期の完了を許可
