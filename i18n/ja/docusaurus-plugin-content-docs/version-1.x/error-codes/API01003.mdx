# API01003: サーバー接続リセット

## この意味

サーバーへの接続が確立されたにもかかわらず、予期せず切断されました。これは通常、通信中に何かが接続を中断した場合に発生します。

## 一般的な原因

- **サーバーのクラッシュ** — リクエスト中にサーバープロセスがクラッシュした可能性
- **メモリ制限** — PHPのメモリが不足し、強制終了された
- **リクエスト中のタイムアウト** — サーバーが処理中にタイムアウトした
- **ネットワークの不安定性** — 断続的なネットワーク問題
- **セキュリティソフトウェア** — 接続を閉じる積極的なセキュリティ設定

## 解決方法

### 1. サーバーエラーログを確認

WordPressのエラーログやホスティングコントロールパネルで、リセット発生時の関連エラーを確認してください。

### 2. PHPメモリ制限を増やす

`wp-config.php`で：

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. プラグインの競合を確認

プラグインがPHPのクラッシュを引き起こしている可能性：
- 一時的にプラグインを無効化
- 原因を特定するために一つずつ再有効化

### 4. サーバーリソースを確認

ホスティングプロバイダに以下を確認：
- PHPプロセスの制限
- エラー発生時のメモリ使用状況
- サーバーの安定性ログ

### 5. ネットワークの安定性をテスト

WiFiの場合は有線接続を試してください。異なるネットワークでも問題が続く場合は、サーバー側の問題の可能性が高いです。
