# PY02001: 支払いゲートウェイエラー

## これは何を意味するか

支払いゲートウェイとの通信中にエラーが発生しました。支払いプロセッサーはリクエストを完了できませんでした。

## 一般的な原因

- **ゲートウェイのダウンタイム** — 支払いサービスに問題が発生しています
- **構成エラー** — ゲートウェイ設定が不正確です
- **API資格情報** — 無効または期限切れのAPIキー
- **ネットワークの問題** — ゲートウェイへの接続の問題

## 修正方法

### 1. トランザクションを再試行する

時々、ゲートウェイに一時的な問題が発生します：
- 少し待ってみてください
- もう一度支払いを処理してみてください
- 問題は自動的に解決する可能性があります

### 2. ゲートウェイの状態を確認する

支払いプロバイダーのステータスページにアクセスしてください：
- Stripe: status.stripe.com
- PayPal: PayPalコミュニティまたはステータスを確認
- Square: status.squareup.com
- 既知の障害がないか確認

### 3. ゲートウェイの構成を確認する

WooCommerce管理画面で：
1. WooCommerce → 設定 → 支払いに移動
2. 支払いゲートウェイをクリック
3. APIキーが正しいことを確認
4. テスト/本番モードがキーに一致していることを確認

### 4. API資格情報を確認する

資格情報が有効であることを確認してください：
- APIキーが期限切れでないこと
- 本番トランザクションには本番キーを使用
- テストトランザクションにはテストキーを使用
- 資格情報が正しいアカウントに一致していること

### 5. 別のゲートウェイでテストする

利用可能な場合：
- 別のゲートウェイで処理を試みる
- フォールバックとして現金支払いを使用
- 販売を完了し、後で調整

## ストアオーナー向け

一般的な構成の問題：
- テストAPIキーと本番APIキーの混在
- 異なるアカウントのAPIキー
- 無効な支払い方法
- 期限切れまたは取り消された資格情報

問題が解決しない場合は、支払いプロバイダーのサポートにお問い合わせください。
