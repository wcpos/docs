# API03001: リクエストフォーマットが無効です

## この意味

サーバーに送信されたリクエストが予期されたフォーマットではありませんでした。サーバーは、POSが要求していることを理解できませんでした。リクエスト構造が不正だったためです。

## 一般的な原因

- **データ破損** — データ送信前に破損していた
- **ソフトウェアバグ** — POSアプリケーションの問題
- **プロキシ干渉** — プロキシまたはファイアウォールがリクエストを変更
- **文字エンコーディングの問題** — 特殊文字が正しくエンコードされていない

## サーバーエラーマッピング

このエラーコードは、サーバーが以下を返す際にトリガーされます：

| サーバーコード | 送信元 |
|-------------|--------|
| HTTP 400 | すべてのサーバーレスポンス（フォールバック） |

## 修正方法

### 1. アクションを再試行

時々これは一時的な障害です：
- しばらく待って再試行
- POSを更新して再試行

### 2. 特殊文字を確認

特殊文字を含むデータを入力する場合：
- 絵文字や特殊な記号を削除
- 商品名などに標準的な文字を使用

### 3. POSを更新

最新バージョンを実行していることを確認：
- アプリの更新を確認
- サーバーのWCPOSプラグインを更新

### 4. ネットワーク設定を確認

プロキシの背後にある場合：
- プロキシがリクエストを変更していないか確認
- ファイアウォールルールを確認
- 別のネットワークからアクセスを試行

### 5. 問題を報告

これが継続的に発生する場合：
- エラーをトリガーするアクションをメモ
- ウェブ版の場合、ブラウザコンソールで詳細を確認
- 再現手順を添えて[GitHub](https://github.com/wcpos)で報告
