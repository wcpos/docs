# API05001: WooCommerceのAPIが無効です

## この意味

WooCommerceのREST APIがサイト上で無効になっています。WCPOSはWooCommerceと通信し、ストアデータにアクセスするためにREST APIを必要とします。

## よくある原因

- **WooCommerceの設定** — REST APIが意図的に無効化されている
- **セキュリティプラグイン** — セキュリティプラグインがAPIアクセスをブロックしている
- **ホスティングの制限** — ホストがREST APIアクセスを無効にしている
- **パーマリンクの問題** — REST API用にパーマリンクが正しく設定されていない

## 修正方法

### 1. WooCommerce REST APIを有効にする

WordPress管理画面で：
1. WooCommerce → 設定 → 詳細設定 → REST API
2. REST APIが有効になっていることを確認
3. APIキーが作成されていることを確認

### 2. WordPress REST APIを確認

WooCommerceのAPIはWordPress REST APIに依存しています：
1. ブラウザで `https://yoursite.com/wp-json/` にアクセス
2. JSONデータが返されるはず
3. 返されない場合は [API05004](./API05004) を参照

### 3. セキュリティプラグインを確認

APIをブロックする可能性のある一般的なセキュリティプラグイン：
- **Wordfence** — ファイアウォール設定を確認
- **iThemes Security** — REST API設定を確認
- **All In One WP Security** — ファイアウォールルールを確認

REST APIエンドポイントまたはPOSアプリケーションをホワイトリストに追加します。

### 4. .htaccessを確認

APIアクセスをブロックしているルールを探します：

```apache
# APIアクセスをブロックするルールを削除または変更
# mod_rewriteが有効になっていることを確認
```

### 5. パーマリンクをリフレッシュ

パーマリンク設定を更新する必要がある場合があります：
1. 設定 → パーマリンクに移動
2. 「変更を保存」をクリック（変更しなくても）
3. リライトルールが再生成されます
