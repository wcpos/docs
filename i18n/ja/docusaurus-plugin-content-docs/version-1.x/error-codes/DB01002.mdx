# DB01002: データベースクエリタイムアウト

## この意味

データベース操作の完了に想定以上の時間がかかりました。ローカルデータベースのクエリが最大許容時間を超過しました。

## 一般的な原因

- **大量のデータ** — 非常に多くの商品またはレコード
- **複雑なクエリ** — 多くの条件を含むクエリ
- **デバイスのパフォーマンス** — デバイスの動作が遅い
- **データベースの断片化** — データベースの最適化が必要

## 解決方法

### 1. 待機して再試行

クエリが再試行で成功する可能性があります：
- POSを閉じて再度開く
- 操作を再度試みる
- 同時に他の負荷の高い操作を避ける

### 2. 他のアプリケーションを閉じる

デバイスのリソースを解放：
- 未使用のブラウザタブを閉じる
- 他のアプリケーションを閉じる
- 必要に応じてデバイスを再起動

### 3. データ範囲を縮小

検索する場合：
- より具体的な検索語を使用
- フィルターを適用して結果を絞る
- 大きな操作を小さな操作に分割

### 4. データベースを最適化

クリアして再同期：
1. ローカルデータをクリア（利用可能な設定オプション）
2. 再ログイン
3. データを新たに同期させる

### 5. デバイスのパフォーマンスを確認

デバイスが要件を満たしていることを確認：
- 十分なRAM
- 適切なプロセッサ速度
- HDDよりSSDを推奨

## 大規模なカタログ

非常に大きな商品カタログがある場合：
- 初回の同期に時間がかかる
- POSでフィルターを使用することを検討
- 一部の操作は自然と時間がかかる
- 最適化のアドバイスについてサポートに連絡
