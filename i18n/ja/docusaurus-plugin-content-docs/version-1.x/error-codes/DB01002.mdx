# DB01002: クエリタイムアウト

## これは何を意味するか

データベース操作が完了するまでに時間がかかりすぎました。ローカルデータベースのクエリが許可された最大時間を超えました。

## 一般的な原因

- **大規模データセット** — 非常に多くの製品やレコード
- **複雑なクエリ** — 多くの条件を含むクエリ
- **デバイスパフォーマンス** — デバイスが遅く動作している
- **データベースの断片化** — データベースの最適化が必要

## 修正方法

### 1. 待機して再試行

クエリは再試行で成功するかもしれません：
- POSを閉じて再度開く
- 操作を再試行する
- 同時に他の集中的な操作を避ける

### 2. 他のアプリケーションを閉じる

デバイスのリソースを解放します：
- 使用していないブラウザタブを閉じる
- 他のアプリケーションを閉じる
- 必要に応じてデバイスを再起動する

### 3. データの範囲を減らす

検索する場合：
- より具体的な検索用語を使用する
- 結果を絞り込むためにフィルターを適用する
- 大きな操作を小さなものに分ける

### 4. データベースを最適化する

クリアして再同期する：
1. ローカルデータをクリアする（設定オプションがある場合）
2. 再度ログインする
3. データを新たに同期させる

### 5. デバイスのパフォーマンスを確認する

デバイスが要件を満たしていることを確認する：
- 十分なRAM
- 適切なプロセッサ速度
- HDDよりもSSDが推奨

## 大規模カタログ

非常に大きな製品カタログがある場合：
- 初期同期には時間がかかる
- POSでフィルターを使用することを検討する
- 一部の操作には自然と時間がかかる
- 最適化のアドバイスについてサポートに連絡する
