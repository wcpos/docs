# API02009: リフレッシュトークン期限切れ

## この意味

リフレッシュトークンの有効期間が経過し、新しいアクセストークンを取得するために使用できなくなりました。アクセストークン（迅速に期限切れになる）とは異なり、リフレッシュトークンは通常数日から数週間有効ですが、最終的には期限が切れます。

## よくある原因

- **長期間の非アクティブ状態** — POSを長時間使用していない
- **トークン有効期間の短さ** — サーバーがリフレッシュトークンの有効期間を短く設定している
- **クロックスキュー** — デバイスまたはサーバーの時刻が大きくずれている

## 解決方法

### 1. 再ログイン

リフレッシュトークンの期限が切れた場合は、再認証する必要があります：
1. POSからログアウトする
2. 認証情報を入力する
3. 新しいリフレッシュトークンが発行される

### 2. POSを定期的に使用

今後これを避けるには：
- POSを定期的に使用してトークンを更新する
- アクティブな状態でトークンが自動的に更新される
- 長期間の非アクティブ状態では再ログインが必要

### 3. 時刻設定を確認

すべてのシステムで正確な時刻を確保：
- デバイスで自動時刻同期を有効にする
- サーバー時刻が正確であることを確認する
- 時刻の違いが早期の期限切れを引き起こす可能性がある

### 4. トークン有効期間の調整（上級者向け）

必要に応じてトークンの有効期間が短すぎる場合、サーバー管理者は認証設定でリフレッシュトークンの有効期間を調整できます。
