# API02008: 更新トークンが無効です

## これは何を意味するか

POSに保存されている更新トークンがサーバーに認識されません。更新トークンは、再度ログインせずに新しいアクセストークンを取得するために使用されます。

## 一般的な原因

- **トークンの取り消し** — 管理者によってセッションが無効化されました
- **サーバーシークレットの変更** — WordPressのセキュリティキーが更新されました
- **データ破損** — トークンが保存中に破損しました
- **複数のログイン** — 他の場所でのログインによりこのトークンが無効になった可能性があります

## 解決方法

### 1. 再ログイン

更新トークンが無効なため、再度認証が必要です：
1. POSからログアウトします
2. ユーザー名とパスワードを入力します
3. 新しい有効なトークンが生成されます

### 2. 取り消されたセッションの確認

管理者によってセッションが取り消された場合：
- これは通常のセキュリティ対策です
- 単に再ログインしてください
- 新しいセッションが正常に機能します

### 3. サーバーの変更を確認

全ユーザーに影響がある場合：
- WordPressのソルト/キーが変更された可能性があります
- JWTプラグインの設定が変更された可能性があります
- サーバー管理者に連絡してください

### 4. 保存されたデータの消去

問題が続く場合：
- POSアプリケーションのデータを消去します
- キャッシュされた認証情報を削除します
- クリーンな状態でログインを開始します
