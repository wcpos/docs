# API02008: リフレッシュトークン無効 {#api02008-refresh-token-invalid}

## これは何を意味するか {#what-this-means}

POSに保存されたリフレッシュトークンがサーバーによって認識されていません。リフレッシュトークンは、再度ログインすることなく新しいアクセストークンを取得するために使用されます。

## 一般的な原因 {#common-causes}

- **トークンが取り消された** — 管理者があなたのセッションを無効にしました
- **サーバーの秘密が変更された** — WordPressのセキュリティキーが更新されました
- **データの破損** — トークンがストレージ内で破損しました
- **複数のログイン** — 他の場所でのログインがこのトークンを無効にした可能性があります

## 修正方法 {#how-to-fix}

### 1. 再度ログインする {#1-log-in-again}

リフレッシュトークンが無効なので、再度認証する必要があります：
1. POSからログアウトします
2. ユーザー名とパスワードを入力します
3. これにより、新しい有効なトークンが生成されます

### 2. 取り消されたセッションを確認する {#2-check-for-revoked-sessions}

管理者がセッションを取り消した場合：
- これは通常のセキュリティ手続きです
- 単に再度ログインしてください
- あなたの新しいセッションは正常に動作します

### 3. サーバーの変更を確認する {#3-check-server-changes}

これが全ユーザーに影響する場合：
- WordPressのソルト/キーが変更された可能性があります
- JWTプラグインの設定が変更された可能性があります
- サーバー管理者に連絡してください

### 4. 保存されたデータをクリアする {#4-clear-stored-data}

問題が続く場合：
- POSアプリケーションデータをクリアします
- キャッシュされた資格情報を削除します
- クリーンなログインから始めます
