# API06001: 無効なURLフォーマット

## 意味

WooCommerceサイト用に設定されたURLが正しい形式ではありません。URLは標準的なウェブアドレスの規則に従う必要があります。

## 一般的な原因

- **プロトコル不足** — URLが `http://` または `https://` で始まっていない
- **URLのタイプミス** — ドメインのスペルミス
- **無効な文字** — URLで許可されていない特殊文字
- **URLの不完全** — ドメイン拡張子などのURLの一部が欠落

## 修正方法

### 1. URLフォーマットを確認

有効なURLは以下のようになります：
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com` (プロトコル不足)
- ❌ `https://yourstore` (拡張子不足)
- ❌ `https://your store.com` (スペースを含む)

### 2. プロトコルを含める

常に `https://` または `http://` を含めます：
- 推奨：`https://` (セキュア)
- 代替：`http://` (HTTPSが利用できない場合)

### 3. 余分な文字を削除

以下のものがないことを確認：
- 末尾のスラッシュ（オプションですが、一貫性を保つ）
- 余分なスペース
- 特殊文字
- URLパラメータ（必要でない限り）

### 4. URLが機能することを確認

ブラウザでURLをテスト：
1. 使用しているURLをコピー
2. ブラウザに貼り付け
3. WordPressサイトが読み込まれること

### 5. POS設定を更新

POSの設定で修正したURLを入力：
- ブラウザで機能するものと完全に一致
- サイトに応じて `www.` を含めるか除外する

## 例

| 無効 | 有効 |
|---------|-------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
