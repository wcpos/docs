# API06003: サイト設定が無効

## この意味

サイト設定が無効または不完全です。これは、不正なURL、認証設定、またはその他の設定の問題が関係している可能性があります。

## 一般的な原因

- **部分的な設定** — 一部の設定が不足している
- **設定の不一致** — 設定がサイトと一致しない
- **設定の破損** — 設定が破損している
- **サイトの変更** — サイトが変更され、POSの設定が更新されていない

## 修正方法

### 1. すべての設定を確認

完全な設定を確認します：
- サイトURLが正しい
- 認証が適切に設定されている
- その他の設定が正しい

### 2. 最初から再設定

設定が破損している場合：
1. 保存されているすべての設定をクリア
2. セットアップ処理を再度開始
3. 新しい設定を入力

### 3. サイトの要件を確認

WordPress サイトが要件を満たしていることを確認：
- WordPress がインストールされ、アクセス可能
- WooCommerce がインストールされ、アクティブ
- WCPOS プラグインがインストールされ、アクティブ
- パーマリンクが有効（「プレーン」ではない）

### 4. サイトアクセスをテスト

これらのURLがブラウザで動作することを確認：
- `https://yoursite.com/` — メインサイト
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. WordPress 設定を確認

WordPress 管理画面で：
1. 設定 → 一般 — URLを確認
2. 設定 → パーマリンク — 「プレーン」でないことを確認
3. WooCommerce → ステータス — 問題がないか確認

## 設定要件

有効な設定には以下が必要：
- **サイトURL** — プロトコル付きの完全なURL
- **認証** — 有効な資格情報またはAPIキー
- **権限** — ユーザーがPOSにアクセス可能
- **プラグイン** — 必要なプラグインがアクティブ
