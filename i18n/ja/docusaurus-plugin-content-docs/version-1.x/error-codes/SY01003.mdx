# SY01003: 権限が拒否されました

## 意味

POSアプリケーションが必要なリソースにアクセスする権限がありません。これは、ファイル、フォルダ、またはシステム機能に関連する可能性があります。

## 一般的な原因

- **ファイル権限** — 必要なファイルの読み取りまたは書き込みができない
- **アプリ権限** — アプリケーションに必要な権限がない
- **保護されたフォルダ** — 制限された場所にアクセスしようとしている
- **セキュリティソフトウェア** — ウイルス対策ソフトがアクセスをブロックしている

## 解決方法

### 1. 管理者として実行（Windows）

初期設定に必要な場合は、アプリを右クリックして「管理者として実行」を選択してください。

### 2. アプリケーション権限を確認

**macOS:**
- システム環境設定 → セキュリティとプライバシー → プライバシー
- アプリに必要な権限が付与されていることを確認

**Windows:**
- 設定 → プライバシー
- アプリの権限を確認

### 3. フォルダ権限を確認

アプリのデータフォルダにアクセスできることを確認：
- ユーザーがアプリデータディレクトリを所有していること
- 書き込み権限が有効であること
- フォルダに読み取り専用のフラグがないこと

### 4. ウイルス対策設定を確認

セキュリティソフトウェアがアクセスをブロックしている可能性：
- ウイルス対策ログでブロックを確認
- POSアプリを許可リストに追加
- アプリデータフォルダをホワイトリストに登録

### 5. アプリケーションを再インストール

権限が破損している場合：
1. POSアプリケーションをアンインストール
2. デバイスを再起動
3. 標準的な場所に再インストール
4. 通常の（管理者以外の）権限で実行

## ブラウザ権限（Webバージョン）

Webバージョンを使用している場合：
- サイトのローカルストレージを許可
- サイトのCookieを有効化
- ブラウザ設定を厳しく制限しない
