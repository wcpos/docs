# SY01003: 権限が拒否されました {#sy01003-permission-denied}

## これが意味すること {#what-this-means}

POSアプリケーションには、必要なリソースにアクセスする権限がありません。これはファイル、フォルダ、またはシステム機能である可能性があります。

## 一般的な原因 {#common-causes}

- **ファイルの権限** — 必要なファイルの読み取りまたは書き込みができない
- **アプリの権限** — アプリケーションに必要な権限が欠如している
- **保護されたフォルダ** — 制限された場所にアクセスしようとしている
- **セキュリティソフトウェア** — アンチウイルスがアクセスをブロックしている

## 修正方法 {#how-to-fix}

### 1. 管理者として実行（Windows） {#1-run-as-administrator-windows}

アプリを右クリックし、初期設定に必要な場合は「管理者として実行」を選択します。

### 2. アプリケーションの権限を確認 {#2-check-application-permissions}

**macOS:**
- システム環境設定 → セキュリティとプライバシー → プライバシー
- アプリが必要な権限を持っているか確認します。

**Windows:**
- 設定 → プライバシー
- アプリの権限を確認します。

### 3. フォルダの権限を確認 {#3-check-folder-permissions}

アプリがデータフォルダにアクセスできることを確認します：
- ユーザーはアプリデータディレクトリの所有者である必要があります。
- 書き込み権限が有効になっていること。
- フォルダに読み取り専用のフラグがないこと。

### 4. アンチウイルス設定を確認 {#4-review-antivirus-settings}

セキュリティソフトウェアがアクセスをブロックしている可能性があります：
- ブロックについてのアンチウイルスのログを確認します。
- POSアプリを許可リストに追加します。
- アプリデータフォルダをホワイトリストに登録します。

### 5. アプリケーションを再インストール {#5-reinstall-the-application}

権限が破損している場合：
1. POSアプリケーションをアンインストールします。
2. デバイスを再起動します。
3. 標準の場所に再インストールします。
4. 通常の（非管理者）権限で実行します。

## ブラウザの権限（Webバージョン） {#browser-permissions-web-version}

Webバージョンを使用している場合：
- サイトのローカルストレージを許可します。
- サイトのクッキーを有効にします。
- 過度に制限的なブラウザ設定を使用しないでください。
