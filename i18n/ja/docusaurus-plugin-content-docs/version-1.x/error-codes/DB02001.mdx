# DB02001: 重複レコード

## 意味

データベースは、既に存在するIDを持つレコードを挿入しようとしました。各レコードには一意の識別子が必要です。

## 一般的な原因

- **同期の競合** — 同じデータが二度同期された
- **レースコンディション** — 複数の操作が同じレコードを作成している
- **データ破損** — IDが破損または重複した
- **インポートの問題** — 既に存在するデータをインポートしている

## 修正方法

### 1. データを更新

サーバーから再同期する:
- 最新のデータを取得
- 同期プロセスが競合を解決するのを待つ
- 新しいデータが優先されるべき

### 2. 操作を再試行

新しいレコードを作成する場合:
- システムは既にそれを作成している可能性がある
- レコードが存在するか確認する
- 送信ボタンを何度もクリックしない

### 3. ローカルキャッシュをクリア

ローカルデータをリセット:
1. ローカルデータベース/キャッシュをクリア
2. 再ログインする
3. データを新たに同期する

### 4. WooCommerceで重複を確認

問題が解決しない場合:
- WooCommerceで重複エントリを確認する
- 同じIDを持つ商品/注文を探す
- 管理画面で重複を整理する

### 5. 恒常的な問題を報告

頻繁に発生する場合:
- 何の操作が原因かメモする
- パターンを確認する
- 詳細をサポートに報告する

## 重要な理由

重複したIDは以下を引き起こします:
- どのレコードが正しいかの混乱
- データの上書き
- 同期の問題

このエラーはデータの破損を防ぎます。
