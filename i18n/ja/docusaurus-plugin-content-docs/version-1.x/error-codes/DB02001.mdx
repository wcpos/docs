# DB02001: レコードの重複

## この意味

データベースは、既に存在するIDを持つレコードの挿入を試みました。各レコードには一意の識別子が必要です。

## 一般的な原因

- **同期の競合** — 同じデータが2回同期された
- **競合状態** — 複数の操作で同じレコードを作成
- **データ破損** — IDが破損または重複
- **インポートの問題** — 既存のデータをインポート

## 修正方法

### 1. データを更新

サーバーから再同期：
- 最新のデータを取得
- 同期プロセスで競合を解決
- 新しいデータが優先されるべき

### 2. 操作を再試行

新しいレコードを作成する場合：
- システムが既に作成している可能性がある
- レコードの存在を確認
- 送信ボタンを複数回クリックしない

### 3. ローカルキャッシュをクリア

ローカルデータをリセット：
1. ローカルデータベース/キャッシュをクリア
2. 再ログイン
3. データを新たに同期

### 4. WooCommerceで重複をチェック

問題が続く場合：
- WooCommerceで重複エントリをチェック
- 同じIDの商品/注文を探す
- 管理画面で重複を削除

### 5. 継続的な問題を報告

頻繁に発生する場合：
- 原因となるアクションをメモ
- パターンを確認
- 詳細をサポートに報告

## なぜこれが重要か

重複IDは以下を引き起こす：
- どのレコードが正しいかの混乱
- データの上書き
- 同期の問題

このエラーはデータ破損を防ぎます。
