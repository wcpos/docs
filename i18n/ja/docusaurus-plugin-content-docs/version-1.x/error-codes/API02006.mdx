# API02006: APIキーが無効です

## この意味

使用されているWooCommerceのREST APIキーが無効です。APIキーはサーバー間認証に使用され、提供されたキーがWooCommerceの有効なキーと一致しません。

## 一般的な原因

- **キー削除済み** — APIキーがWooCommerceから削除された
- **キー入力ミス** — キーが正しく入力されていない
- **キーペアの不一致** — コンシューマーキーとシークレットが一致しない
- **別のサイトのキー** — 別のインストール用に生成されたキーを使用

## 修正方法

### 1. APIキーの確認

WordPress管理画面 → WooCommerce → 設定 → 詳細設定 → REST API:
1. APIキーが存在するか確認
2. 取り消されていないことを確認
3. 権限（読み取り/書き込み/読み書き）を確認

### 2. 新しいAPIキーの生成

キーがない、または無効な場合:
1. WooCommerce → 設定 → 詳細設定 → REST API
2. 「キーを追加」をクリック
3. 説明を入力（例：「WCPOS」）
4. ユーザーを選択
5. 「読み書き」権限を選択
6. 「APIキーを生成」をクリック
7. **コンシューマーキーとコンシューマーシークレットをコピー**（一度しか表示されません！）

### 3. POS設定の更新

POSに新しいAPIキーを入力:
- コンシューマーキー（`ck_`で始まる）
- コンシューマーシークレット（`cs_`で始まる）

### 4. キー権限の確認

十分な権限があることを確認:
- **読み取り** — データの閲覧のみ
- **書き込み** — データの変更のみ
- **読み書き** — 完全アクセス（POSでは推奨）
