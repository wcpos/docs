# API03006: サポートされていないメソッド

## 意味

使用されたHTTPメソッド（GET、POST、PUT、DELETE等）が、このエンドポイントでサポートされていません。サーバーがこのURLに対する特定のタイプのリクエストを受け入れません。

## 一般的な原因

- **バージョンの不一致** — POSとプラグインのバージョンに互換性がない
- **エンドポイントの削除** — APIエンドポイントが非推奨になった
- **サーバー設定** — ウェブサーバーが特定のHTTPメソッドをブロックしている
- **プラグインの競合** — 別のプラグインがREST APIの動作を変更している

## 解決方法

### 1. コンポーネントの更新

互換性を確認:
- WCPOS WordPress プラグインを更新
- POSアプリケーションを更新
- リリースノートで重大な変更を確認

### 2. サーバー設定の確認

一部のサーバーは特定のHTTPメソッドをブロックします:
- PUTおよびDELETEメソッドが許可されていることを確認
- `.htaccess`でメソッド制限を確認
- Nginx設定を確認

### 3. REST APIアクセスの検証

WordPress REST APIをテスト:
1. ブラウザで `https://yoursite.com/wp-json/` にアクセス
2. JSONデータが返されることを確認
3. データが返されない場合、REST APIが無効またはブロックされている可能性がある

### 4. プラグインの競合を確認

一時的に他のプラグインを無効化:
- セキュリティプラグインがメソッドをブロックしている可能性
- 他のREST APIプラグインが競合している可能性
- 問題を特定するため、1つずつ再有効化

### 5. ホスティング制限の確認

一部のホストはHTTPメソッドを制限:
- ホスティングサポートに連絡
- 標準メソッドの有効化を要求
- 制限が厳しすぎる場合はホストの変更を検討
