# PY02002: 支払いタイムアウト {#py02002-payment-timeout}

## これは何を意味するか {#what-this-means}

支払い処理に時間がかかり、タイムアウトしました。支払いゲートウェイは、期待される時間内に応答しませんでした。

## 一般的な原因 {#common-causes}

- **ゲートウェイの過負荷** — 支払い処理業者が高いトラフィックを経験している
- **ネットワーク遅延** — 支払いサービスへの接続が遅い
- **処理の遅延** — 複雑な詐欺チェックや確認
- **サーバーの問題** — 支払いゲートウェイがパフォーマンス問題を抱えている

## 修正方法 {#how-to-fix}

### 1. 支払いが成功したか確認する {#1-check-if-payment-succeeded}

**重要:** 再試行する前に、支払い状況を確認してください：
- 支払いゲートウェイのダッシュボードを確認
- WooCommerceでトランザクションを探す
- 実際に成功した場合は再処理しない

### 2. 注意して再試行する {#2-retry-with-caution}

支払いが確実に通らなかった場合：
- 30〜60秒待つ
- 再試行する
- 重複請求に注意する

### 3. 接続を確認する {#3-check-your-connection}

安定した接続を確認：
- インターネット接続をテスト
- WiFiを使用している場合は、有線接続を試す
- ネットワークの問題を確認する

### 4. 混雑していない時間帯に試す {#4-try-at-a-less-busy-time}

ゲートウェイが過負荷の場合：
- 小売のピーク時に遅延が発生する可能性がある
- すぐに再試行する
- 代替の支払い方法を使用する

### 5. 支払いプロバイダーに連絡する {#5-contact-payment-provider}

タイムアウトが続く場合：
- ステータスページを確認
- サポートに連絡
- 進行中の問題を報告

## 重複請求の回避 {#avoiding-duplicate-charges}

タイムアウトが発生した場合：
1. **すぐに再試行しない** — まず状態を確認
2. **トランザクションをレビュー** — ゲートウェイとWooCommerceの両方を確認
3. **重複を返金** — アクシデンタルな重複に対して返金処理を行う
4. **顧客に通知** — 遅延があった場合は状況を説明する

## 代替アクション {#alternative-actions}

支払いがタイムアウトし続ける場合：
- 現金支払いを受け入れる
- 許されている場合は「後で支払う」として記録する
- 別の支払い方法を使用する
