# PY02002: 支払いタイムアウト

## この意味

支払い処理が長すぎて、タイムアウトしました。決済ゲートウェイが予想される時間内に応答しませんでした。

## 一般的な原因

- **ゲートウェイの過負荷** — 決済プロセッサが高い負荷を経験
- **ネットワーク遅延** — 決済サービスへの接続が遅い
- **処理遅延** — 複雑な不正チェックまたは認証
- **サーバーの問題** — 決済ゲートウェイのパフォーマンス問題

## 解決方法

### 1. 支払いの成功を確認する

**重要:** 再試行する前に、支払いステータスを確認してください：
- 決済ゲートウェイのダッシュボードを確認
- WooCommerceで取引を探す
- 実際に成功した場合は再処理しない

### 2. 慎重に再試行する

支払いが確実に通っていない場合：
- 30〜60秒待つ
- 再度試す
- 重複請求に注意

### 3. 接続を確認する

安定した接続を確保：
- インターネット接続をテスト
- WiFi使用時は有線接続を試す
- ネットワークの問題を確認

### 4. 混雑が少ない時間に試す

ゲートウェイが過負荷の場合：
- 小売のピーク時は遅延の可能性
- 少し後に再試行
- 代替の支払い方法を使用

### 5. 決済プロバイダに連絡

タイムアウトが続く場合：
- ステータスページを確認
- サポートに連絡
- 継続的な問題を報告

## 重複請求の回避

タイムアウト発生時：
1. **すぐに再試行しない** — まずステータスを確認
2. **取引を確認** — ゲートウェイとWooCommerceの両方を確認
3. **重複を返金** — 偶発的な重複があれば返金処理
4. **顧客に説明** — 遅延がある場合は状況を説明

## 代替アクション

支払いが継続的にタイムアウトする場合：
- 現金支払いを受け入れる
- 許可されている場合は「後で支払う」として記録
- 別の支払い方法を使用
