# DB03003: 必須フィールドが見つかりません

## この意味

必須フィールドが提供されませんでした。重要な情報が不足しているため、データベースにレコードを保存できません。

## 一般的な原因

- **不完全なフォーム** — 必須フィールドが入力されていない
- **データ同期の問題** — 必須データが正しく同期されなかった
- **バリデーションの回避** — バリデーションなしでフォームが送信された
- **カスタムフィールド必須** — 必須のカスタムフィールドが空である

## 修正方法

### 1. すべての必須フィールドを完成させる

フォームで確認する項目:
- アスタリスク (*) の付いたフィールド
- 強調表示またはエラー表示されているフィールド
- 値が入力されるべき空のフィールド

### 2. 更新して再入力

フォームの状態が正しくない場合:
1. ページ/画面を更新
2. すべての情報を再入力
3. 送信前にすべてのフィールドを確認

### 3. WooCommerceの設定を確認

カスタムフィールドが必須の場合:
1. WooCommerceのチェックアウト設定を確認
2. 必須のカスタムフィールドを確認
3. POSでこれらのフィールドが提供されていることを確認

### 4. 必須データを同期

関連データが不足している場合:
- 商品/顧客などを更新
- 同期が完了するまで待つ
- 操作を再試行

### 5. 必須フィールドの設定を確認

WooCommerce管理画面で:
- どのフィールドが必須とされているか確認
- POSに本当に必要かを検討
- 必要に応じて要件を調整

## 一般的な必須フィールド

注文で通常必須:
- 最低1つの明細行
- 支払方法
- 顧客（設定による）

商品で通常必須:
- 商品名
- 価格（シンプルな商品の場合）
