# API05003: WCPOSプラグインが古い

## これは何を意味するのか

サーバーにインストールされているWCPOSプラグインは古く、POSアプリケーションのバージョンと互換性がありません。プラグインを更新する必要があります。

## 一般的な原因

- **プラグインが更新されていない** — 自動更新が無効になっている可能性があります
- **更新に失敗した** — 前回の更新試行で失敗しました
- **バージョンの不一致** — POSアプリが更新されたがプラグインは更新されていない
- **ステージング/開発サイト** — 古いバージョンでテスト中

## 修正方法

### 1. プラグインを更新する

WordPress管理画面で:
1. プラグイン → インストール済みプラグインに移動
2. 「WooCommerce POS」を探す
3. 更新が可能な場合は、「今すぐ更新」をクリック

### 2. 自動更新を有効にする

将来の不一致を防ぐために:
1. プラグイン → インストール済みプラグインに移動
2. 「WooCommerce POS」を探す
3. 「自動更新を有効にする」をクリック

### 3. 手動更新

自動更新が失敗した場合:
1. [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/)から最新バージョンをダウンロード
2. 現在のプラグインを無効化
3. 古いプラグインファイルを削除
4. 新しいバージョンをアップロードして有効化

### 4. 更新エラーを確認する

更新が失敗している場合:
- wp-content/pluginsのファイル権限を確認
- ディスクスペースがあるか確認
- PHPメモリの問題を探す
- エラーログをチェックして詳細を確認

### 5. POSアプリをダウングレードする（一時的）

プラグインをすぐに更新できない場合:
- 古いバージョンのPOSアプリを使用
- これは一時的な回避策です
- 早急にプラグインを更新する計画を立ててください

## バージョン互換性

両方のコンポーネントを常に最新の状態に保つ:
- POSアプリケーション
- WCPOS WordPressプラグイン

更新時にはリリースノートで重大な変更を確認してください。
