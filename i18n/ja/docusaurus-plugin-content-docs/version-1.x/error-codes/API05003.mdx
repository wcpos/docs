# API05003: WCPOSプラグインが古くなっています

## この意味

サーバーにインストールされているWCPOSプラグインが古く、POSアプリケーションのバージョンと互換性がありません。プラグインを更新する必要があります。

## 一般的な原因

- **プラグインが更新されていない** — 自動更新が無効になっている可能性
- **更新に失敗** — 以前の更新試行が失敗した
- **バージョンの不一致** — POSアプリは更新されたがプラグインは更新されていない
- **ステージング/開発サイト** — 古いバージョンでテスト中

## 修正方法

### 1. プラグインを更新

WordPress管理画面で:
1. プラグイン → インストール済みプラグイン へ移動
2. 「WooCommerce POS」を探す
3. 更新可能な場合は「今すぐ更新」をクリック

### 2. 自動更新を有効化

今後の不一致を防ぐため:
1. プラグイン → インストール済みプラグイン へ移動
2. 「WooCommerce POS」を探す
3. 「自動更新を有効にする」をクリック

### 3. 手動更新

自動更新に失敗した場合:
1. [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/)から最新版をダウンロード
2. 現在のプラグインを非アクティブ化
3. 古いプラグインファイルを削除
4. 新しいバージョンをアップロードしてアクティブ化

### 4. 更新エラーを確認

更新が失敗する場合:
- wp-content/plugins のファイル権限を確認
- ディスク容量を確認
- PHPメモリの問題を確認
- エラーログの詳細を確認

### 5. POSアプリをダウングレード（一時的）

プラグインをすぐに更新できない場合:
- 古いバージョンのPOSアプリを使用
- これは一時的な回避策のみ
- できるだけ早くプラグインを更新する計画を立てる

## バージョン互換性

常に両方のコンポーネントを最新に保つ:
- POSアプリケーション
- WCPOS WordPress プラグイン

更新時は、破壊的な変更についてリリースノートを確認してください。
