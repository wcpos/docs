# API01004: DNS解決に失敗しました {#api01004-dns-resolution-failed}

## これは何を意味するか {#what-this-means}

POSはあなたのウェブサイトのドメイン名（例: `yourstore.com`）をIPアドレスに変換できませんでした。DNS（ドメインネームシステム）はインターネットの電話帳のようなものであり、リストを見つけられなければ接続を作成できません。

## 一般的な原因 {#common-causes}

- **ドメイン名のタイプミス** — URLにスペルミスがある可能性があります
- **DNSサーバーの問題** — あなたのDNSサーバーが応答していません
- **ドメインの期限切れ** — ドメイン登録が失効している可能性があります
- **DNSの伝播** — 最近のDNS変更がまだ伝播していない
- **ローカルDNSキャッシュ** — あなたのデバイスにキャッシュされた古いDNS情報

## 修正方法 {#how-to-fix}

### 1. ドメイン名を確認する {#1-verify-the-domain-name}

タイプミスがないかURLを再確認してください。ブラウザでサイトにアクセスしてみてください。

### 2. ドメインの状態を確認する {#2-check-domain-status}

WHOIS検索を使用して確認してください：
- ドメインがまだ登録されている
- 期限切れでない
- DNSレコードが正しく設定されている

### 3. DNSキャッシュをクリアする {#3-clear-dns-cache}

**あなたのデバイスで：**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. 代替DNSを試す {#4-try-alternative-dns}

次のようなパブリックDNSサーバーに切り替えます：
- Google: `8.8.8.8` および `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. DNSの伝播を待つ {#5-wait-for-dns-propagation}

最近DNS設定を変更した場合は、完全な伝播のために24〜48時間待ってください。
