# API01004: DNSの解決に失敗

## この意味

POSがウェブサイトのドメイン名（例：`yourstore.com`）をIPアドレスに変換できませんでした。DNS（ドメインネームシステム）はインターネットの電話帳のようなものです — リストが見つからない場合、接続を確立できません。

## 一般的な原因

- **ドメイン名のタイプミス** — URLにスペルミスがある可能性があります
- **DNSサーバーの問題** — DNSサーバーが応答していません
- **ドメインの期限切れ** — ドメイン登録の有効期限が切れています
- **DNS伝播** — 最近のDNS変更がまだ反映されていません
- **ローカルDNSキャッシュ** — デバイス上の古いDNS情報がキャッシュされています

## 修正方法

### 1. ドメイン名を確認

URLのスペルを再確認してください。ブラウザでサイトにアクセスしてみてください。

### 2. ドメインステータスを確認

WHOISルックアップで以下を確認：
- ドメインがまだ登録されているか
- 期限切れになっていないか
- DNSレコードが設定されているか

### 3. DNSキャッシュをクリア

**デバイス上で：**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. 代替DNSを試す

以下のような公開DNSサーバーに切り替えてみてください：
- Google: `8.8.8.8` および `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. DNS伝播を待つ

DNS設定を最近変更した場合は、完全な伝播まで24〜48時間待ってください。
