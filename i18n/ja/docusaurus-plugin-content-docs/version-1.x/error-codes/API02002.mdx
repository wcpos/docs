# API02002: トークンの期限切れ {#api02002-token-expired}

## 意味 {#what-this-means}

あなたの認証セッションは期限切れです。セキュリティ上の理由から、ログインセッションは永久ではありません。POSはあなたの認証を更新する必要があるか、再度ログインする必要があります。

## 一般的な原因 {#common-causes}

- **長期間の非活動** — あなたは長い間POSを使用していません
- **サーバートークンの寿命** — サーバーのトークン有効期限が非常に短く設定されています
- **サーバーの時刻の不一致** — サーバーとデバイスの時計が同期していません

## サーバーエラーのマッピング {#server-error-mapping}

このエラーコードは、サーバーが次を返すときにトリガーされます：

| サーバーコード | ソース |
|-------------|--------|
| `jwt_auth_expired_token` | JWT認証プラグイン |

## 修正方法 {#how-to-fix}

### 1. 再ログイン {#1-log-in-again}

最も簡単な解決策は、ログアウトして再度ログインすることです：
1. POSからログアウトします
2. 再度認証情報を入力します
3. これによって新しいセッションが作成されます

### 2. トークンの更新を確認 {#2-check-token-refresh}

POSは自動的にトークンを更新するはずです。機能していない場合：
- インターネット接続を確認してください
- サーバーにアクセス可能であることを確認してください
- 更新が失敗した場合は[API02007](./API02007)を参照してください

### 3. サーバー時刻を確認 {#3-check-server-time}

トークンの有効期限は正確な時刻に依存します：
- サーバーの時計が正しいことを確認してください
- デバイスの時刻が正確であることを確認してください
- 両方で自動時刻同期を使用します

### 4. トークン設定を確認 {#4-review-token-settings}

トークンがあまりにも早く失効する場合、サーバー設定を調整する必要があるかもしれません（ホスティングプロバイダーまたはWordPress管理者に相談してください）。
