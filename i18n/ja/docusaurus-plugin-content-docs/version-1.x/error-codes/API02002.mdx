# API02002: トークン期限切れ

## この意味

認証セッションの有効期限が切れました。セキュリティ上の理由から、ログインセッションは永久に続きません。POSは認証を再取得するか、再度ログインする必要があります。

## よくある原因

- **長時間の非活動** — POSを長時間使用していない
- **サーバートークンの有効期間** — サーバーのトークン有効期限が非常に短く設定されている
- **サーバー時間の不一致** — サーバーとデバイスの時計が同期していない

## サーバーエラーマッピング

このエラーコードは、サーバーが以下を返した場合にトリガーされます：

| サーバーコード | ソース |
|----------------|--------|
| `jwt_auth_expired_token` | JWTAuthentication プラグイン |

## 修正方法

### 1. 再ログイン

最も簡単な解決策は、ログアウトして再度ログインすることです：
1. POSからログアウト
2. 資格情報を再入力
3. 新しいセッションを作成

### 2. トークン更新の確認

POSは自動的にトークンを更新するはずです。うまく機能していない場合：
- インターネット接続を確認
- サーバーにアクセスできることを確認
- 更新に失敗している場合は[API02007](./API02007)を参照

### 3. サーバー時間の確認

トークンの有効期限は正確な時間に依存します：
- サーバーの時計が正確であることを確認
- デバイスの時間が正確であることを確認
- 両方で自動時間同期を使用

### 4. トークン設定の確認

トークンの有効期限が短すぎる場合、サーバー設定を調整する必要があるかもしれません（ホスティングプロバイダーまたはWordPress管理者に相談してください）。
