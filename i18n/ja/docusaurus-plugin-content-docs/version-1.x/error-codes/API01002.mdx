# API01002: 接続拒否

## これが意味すること

サーバーは接続試行を積極的に拒否しました。これはタイムアウトとは異なります — サーバーは応答しましたが、「いいえ」と言いました。

## 一般的な原因

- **サーバーがダウンしている** — ウェブサーバー（Apache/Nginx）が稼働していない可能性があります
- **ポートが間違っている** — 間違ったポートで接続を試みています
- **ファイアウォールによるブロック** — ファイアウォールが接続をブロックしています
- **IP制限** — あなたのIPアドレスがブロックされている可能性があります

## 修正方法

### 1. サーバーが稼働していることを確認する

ブラウザで直接WordPressサイトにアクセスしてみてください。読み込まれない場合は、ホスティングプロバイダーに連絡してください。

### 2. URL設定を確認する

正しいURLを使用していることを確認してください：
- `http://` と `https://` を確認
- ドメイン名にタイプミスがないか確認
- 非標準ポートを使用している場合は、正しいポートを確認

### 3. ファイアウォール設定を確認する

サーバーにアクセスできる場合：
- ポート80（HTTP）または443（HTTPS）が開いていることを確認
- 自分のIPがホワイトリストに登録されているか確認
- APIアクセスをブロックしている可能性のあるセキュリティプラグインを確認

### 4. ホスティングプロバイダーに連絡する

彼らは以下を確認できます：
- サーバーの状態とログ
- ファイアウォールの設定
- あなたのIPがブロックされているかどうか
