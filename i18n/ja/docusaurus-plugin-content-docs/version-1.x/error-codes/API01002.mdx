# API01002: 接続拒否

## この意味

サーバーが接続試行を積極的に拒否しました。これはタイムアウトとは異なり、サーバーは応答しましたが、「いいえ」と言いました。

## 一般的な原因

- **サーバーがダウン** — Webサーバー（Apache/Nginx）が実行されていない可能性があります
- **ポートの誤り** — 間違ったポートに接続しようとしている
- **ファイアウォールによるブロック** — ファイアウォールが接続をブロックしている
- **IPの制限** — IPアドレスがブロックされている可能性があります

## 解決方法

### 1. サーバーが稼働しているか確認する

ブラウザでWordPressサイトに直接アクセスしてみてください。読み込まれない場合は、ホスティングプロバイダに連絡してください。

### 2. URL設定を確認する

正しいURLを使用していることを確認します：
- `http://`と`https://`を確認
- ドメイン名のスペルミスがないか確認
- 非標準ポートを使用している場合は正しいポートを確認

### 3. ファイアウォールの設定を確認する

サーバーにアクセスできる場合：
- ポート80（HTTP）または443（HTTPS）が開いていることを確認
- IPがホワイトリストに登録されているか確認
- APIアクセスをブロックする可能性のあるセキュリティプラグインを確認

### 4. ホスティングプロバイダに連絡する

以下の点を確認できます：
- サーバーの状態とログ
- ファイアウォールの設定
- IPがブロックされているかどうか
