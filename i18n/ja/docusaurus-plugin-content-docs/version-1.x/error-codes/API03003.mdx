# API03003: 無効なパラメータ値

## この意味

リクエスト内の値のいずれかが無効です。データの形式、タイプ、または値がサーバーの期待と一致しません。

## 一般的な原因

- **間違ったデータ型** — 数値が必要な場所にテキストがある（またはその逆）
- **範囲外** — 値が許可された制限を超えている
- **無効な形式** — メール、電話、またはその他の書式化されたフィールドが正しくない
- **無効な参照** — 存在しないIDを参照している

## サーバーエラーマッピング

このエラーコードは、サーバーが次を返す際にトリガーされます：

| サーバーコード | 送信元 |
|-------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## 修正方法

### 1. 入力値を確認

送信するデータを確認します：
- **価格** — 有効な数値であること
- **数量** — 正の整数であること
- **メール** — 有効なメール形式であること
- **ID** — 既存のレコードを参照していること

### 2. 特定のフィールドエラーを探す

エラーは無効なフィールドを示している可能性があります：
- エラーメッセージの詳細を確認
- 特定のフィールドを修正
- アクションを再試行

### 3. 参照されたデータの存在を確認

製品、顧客、その他のレコードを参照する場合：
- WooCommerceにアイテムが存在することを確認
- 最近削除されていないか確認
- ローカルレコードを更新するためにデータを同期

### 4. フィールドの制約を確認

WooCommerceには制約がある場合があります：
- 最大値/最小値
- 必要な形式
- 選択フィールドの許可されたオプション

### 5. クリアして再入力

データが破損していることがあります：
- 問題のあるフィールドをクリア
- 値を最初から再入力
- 他のソースからのコピー&ペーストを避ける
