# API03005: リクエスト制限超過

## この意味

サーバーに短時間で多すぎるリクエストを行いました。レート制限はサーバーが過負荷になることを防ぎます。

## 一般的な原因

- **高速同期操作** — 複数のデバイスが同時に同期
- **積極的な更新** — データを頻繁に更新
- **セキュリティプラグイン** — プラグインがリクエスト制限を適用
- **ホスティング制限** — ホスティングプランのAPIレート制限

## サーバーエラーマッピング

このエラーコードは、サーバーが次を返す場合にトリガーされます：

| サーバーコード | 送信元 |
|-------------|--------|
| HTTP 429 | すべてのサーバーレスポンス（フォールバック） |

## 解決方法

### 1. 待機してリトライ

レート制限は一時的です：
- 1〜5分待機
- POSは自動的に再試行
- 手動で繰り返し更新しない

### 2. 同時操作を減らす

複数のPOSターミナルを使用している場合：
- デバイス間で初期同期をずらす
- すべてのデバイスを一度に更新しない
- 1つが完了してから次を開始

### 3. セキュリティプラグインを確認

Wordfenceなどのプラグインは、高速リクエストをブロックする場合があります：
- POSアプリケーションをホワイトリスト化
- POSデバイスのIPをホワイトリスト化
- レート制限のしきい値を調整

### 4. ホスティング制限を確認

一部のホストはAPIリクエストを制限します：
- ホスティングプランの詳細を確認
- 制限が低すぎる場合はアップグレードを検討
- 制限の引き上げについてサポートに連絡

### 5. 同期頻度を最適化

POS設定で：
- 自動更新間隔を調整
- 適切な場合は手動同期を使用
- 不要なデータ更新を避ける
