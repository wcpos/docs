WCPOSは、製品、注文、顧客のローカルコピーを保存し、POSが迅速に動作し、オフラインで機能できるようにします。時には、このローカルデータがWooCommerceストアと同期しなくなることがあります。ローカルデータをすべて消去すると、ローカルデータベースが削除され、サーバーからの新しい同期がトリガーされます。

:::warning
これは最後の手段です。ローカルデータを消去すると、デバイスに保存されているすべてが削除され、完全再同期が強制されます。店舗のサイズによっては、これに時間がかかることがあります。他のオプション（通常の [同期](/products/sync) など）で問題が解決されない場合にのみ使用してください。
:::

## 使用するタイミング {#when-to-use-it}

ローカルデータを消去する必要がある場合：

- **同期の問題** — 製品、注文、または顧客が正しく更新されない
- **プラグインの更新後** — 新しいデータ構造との互換性を確保するため
- **古いデータ** — 価格、在庫レベル、または他の情報が古く見える

## 何が消去されるか {#what-gets-cleared}

| 消去される項目 | 消去されない項目 |
|----------------|-----------------|
| ローカル製品データベース | ログインセッション |
| ローカル注文データベース | サーバー側のデータ |
| ローカル顧客データベース | WooCommerce設定 |

ログアウトされることは **ありません**。あなたのWooCommerceストアデータには影響がなく、デバイス上のローカルコピーのみが削除されます。消去後、すぐに新しい同期が自動的に開始します。

## ローカルデータを消去する方法 {#how-to-clear-local-data}

### ウェブ {#web}

1. 右上隅の**ユーザーメニュー**をクリックしてメニューを開く
2. **ローカルデータをすべて消去**をクリック
3. ローカルデータベースが削除され、新しい同期が自動的に開始します

### デスクトップ {#desktop}

1. 上部メニューバーの**WCPOS**アプリケーションメニューを開く
2. **詳細**にマウスをホバーする
3. **アプリデータを消去**をクリック
4. ローカルデータベースが削除され、新しい同期が自動的に開始します
