ログ画面は、POS内のシステム活動を記録します。これは、問題のデバッグと背後で何が起こっているのかを理解するための重要なツールです。

{/* スクリーンショット: logs-screen.png */}

## ログへのアクセス

左側のナビゲーションドロワー（ハートビートアイコン）からログ画面を開きます。

## インターフェース概要

### 検索とフィルタ

- **ログ検索** - メッセージ内容によって特定のログエントリを見つける
- **ログレベルフィルタ** - ログレベル（情報、警告、エラー）でフィルタリング
- **表示設定** - 表示する列を構成

### ログテーブル

各ログエントリには次の情報が表示されます：

| 列      | 説明                             |
|---------|----------------------------------|
| **時間** | イベントが発生した時間         |
| **レベル** | 重症度: 情報（灰色）、警告（オレンジ）、エラー（赤） |
| **メッセージ** | イベントの人間が読みやすい説明 |
| **コンテキスト** | 追加の詳細を含む展開可能なJSON |
| **コード** | エラーコード（ドキュメントへのリンク） |

## ログレベル

### 情報（灰色）

通常の操作及び成功したイベント：
- "製品のすべてのIDを取得しました"
- "新しい顧客を同期しました"
- "税金のすべてのIDを取得しました"

### 警告（オレンジ）

注意が必要な非クリティカルな問題：
- "セッションを更新できません"
- "同期の再試行"

### エラー（赤）

対処する必要がある問題：
- "申し訳ありませんが、リソースをリストすることはできません"（権限の問題）
- "APIリクエストに失敗しました"
- "データベースの同期エラー"

## コンテキストデータ

任意のログエントリの展開矢印をクリックすると、詳細なJSONコンテキストを見ることができます：

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

このデータには次の内容が含まれます：
- **アイテム数** - 影響を受けたレコードの数
- **実行時間** - 操作にかかった時間
- **サーバーロード** - サーバーのパフォーマンスメトリクス
- **リクエスト/レスポンスの詳細** - APIの問題をデバッグするため

## エラーコード

多くのログエントリには**エラーコード**（例：`API02004`、`DB01001`）が含まれています。これらのコードは、[エラーコード](/error-codes)ドキュメントに直接リンクされており、次の内容が提供されます：

- エラーの詳細説明
- 一般的な原因
- トラブルシューティング手順

## 一般的なログエントリ

### 同期操作

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

これらは正常な同期活動を示しています。コンテキストは、同期されたアイテムの数を示します。

### 権限エラー

```
[error] Sorry, you cannot list resources    {...} API02004
```

これは通常、ユーザーがリソースにアクセスする権限を持っていないことを意味します。`WP Admin > POS > 設定 > アクセス`でユーザーの役割を確認してください。

### セッションの問題

```
[warn] Unable to refresh session    {...} API02007
```

POSは認証セッションを更新できませんでした。これにはログアウトして再度ログインする必要があるかもしれません。

## サポート用のログの使用

サポートをリクエストする際には：

1. **問題を再現する** - 問題を引き起こすアクションを実行する
2. **ログを確認する** - エラー（赤）または警告（オレンジ）を探す
3. **コンテキストを展開する** - 完全なJSONの詳細を取得する
4. **エラーコードをメモする** - サポートリクエストにこれを含める
5. **関連するエントリをコピーする** - コンテキストを持つログメッセージを共有する

## パフォーマンスモニタリング

ログには実行時間とサーバーロードのメトリクスが含まれています。次のことに気付いた場合：

- **高い実行時間** - サーバーが過負荷または遅い可能性があります
- **高いサーバーロード値** - サーバーの最適化を検討してください
- **頻繁なタイムアウト** - ネットワーク接続を確認してください
