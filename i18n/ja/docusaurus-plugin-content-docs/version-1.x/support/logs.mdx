ログ画面は、POSシステム内のアクティビティを記録します。デバッグや裏側で起こっていることを理解するための重要なツールです。

{/* Screenshot: logs-screen.png */}

## ログへのアクセス

左のナビゲーションドロワー（ハートビートアイコン）からログ画面を開きます。

## インターフェース概要

### 検索とフィルター

- **ログを検索** - メッセージ内容で特定のログエントリを見つける
- **ログレベルフィルター** - ログレベル（情報、警告、エラー）でフィルタリング
- **表示設定** - 表示する列の設定

### ログテーブル

各ログエントリには以下が表示されます：

| 列 | 説明 |
|--------|-------------|
| **時間** | イベント発生時刻 |
| **レベル** | 重要度：情報（グレー）、警告（オレンジ）、エラー（赤） |
| **メッセージ** | イベントの人間が読める説明 |
| **コンテキスト** | 追加詳細のある展開可能なJSON |
| **コード** | エラーコード（ドキュメンテーションへのリンク） |

## ログレベル

### 情報（グレー）

通常の操作と成功したイベント：
- 「すべての製品IDを取得」
- 「新規顧客を同期」
- 「すべての税金IDを取得」

### 警告（オレンジ）

注意が必要な非重大の問題：
- 「セッションを更新できません」
- 「同期の再試行」

### エラー（赤）

対処が必要な問題：
- 「申し訳ありません。リソースをリストできません」（権限の問題）
- 「APIリクエスト失敗」
- 「データベース同期エラー」

## コンテキストデータ

ログエントリの展開矢印をクリックして詳細なJSONコンテキストを表示：

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

このデータには以下が含まれます：
- **アイテム数** - 影響を受けたレコードの数
- **実行時間** - 操作にかかった時間
- **サーバー負荷** - サーバーのパフォーマンス指標
- **リクエスト/レスポンスの詳細** - APIの問題をデバッグするため

## エラーコード

多くのログエントリには**エラーコード**（例：`API02004`、`DB01001`）が含まれています。これらのコードは直接[エラーコード](/error-codes)ドキュメンテーションにリンクし、以下を提供します：

- エラーの詳細な説明
- 一般的な原因
- トラブルシューティングの手順

## 一般的なログエントリ

### 同期操作

```
[info] すべての製品IDを取得    {...} 3 アイテム
[info] 新規顧客を同期            {...} 1 アイテム
[info] すべての税金IDを取得       {...} 3 アイテム
```

これらは通常の同期アクティビティを示します。コンテキストには同期されたアイテム数が表示されます。

### 権限エラー

```
[error] 申し訳ありません。リソースをリストできません    {...} API02004
```

通常、ユーザーにリソースにアクセスする権限がないことを意味します。`WP管理画面 > POS > 設定 > アクセス`でユーザーロールを確認してください。

### セッションの問題

```
[warn] セッションを更新できません    {...} API02007
```

POSが認証セッションを更新できませんでした。ログアウトして再ログインが必要な場合があります。

## サポートのためのログ使用

サポートをリクエストする際：

1. **問題を再現** - 問題を引き起こすアクションを実行
2. **ログを確認** - エラー（赤）または警告（オレンジ）を探す
3. **コンテキストを展開** - 完全なJSONの詳細を取得
4. **エラーコードをメモ** - サポートリクエストにこれを含める
5. **関連するエントリをコピー** - コンテキスト付きのログメッセージを共有

## パフォーマンスモニタリング

ログには実行時間とサーバー負荷の指標が含まれます。次の点に注意：

- **高実行時間** - サーバーが過負荷または遅い可能性
- **高サーバー負荷の値** - サーバーの最適化を検討
- **頻繁なタイムアウト** - ネットワーク接続を確認
