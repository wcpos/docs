# ストライプターミナルゲートウェイ {#stripe-terminal-gateway}

ストライプターミナルゲートウェイを使用すると、WCPOS内でストライプターミナルハードウェアリーダーを使用して対面で支払いを受け付けることができます。このゲートウェイは、物理的なカードリーダーとテスト用のシミュレータモードの両方をサポートしています。

## 機能 {#features}

<FeatureGrid>
  <FeatureCard icon="credit-card" title="ハードウェア統合">
    インターネット接続を介して物理的なストライプターミナルリーダーを接続
  </FeatureCard>
  <FeatureCard icon="desktop" title="シミュレータモード">
    ストライプのシミュレータを使用してハードウェアなしで支払いをテスト
  </FeatureCard>
  <FeatureCard icon="bolt" title="リアルタイム処理">
    即時の支払い処理と確認
  </FeatureCard>
  <FeatureCard icon="shield" title="安全な取引">
    ストライプを通じたPCI準拠の支払い処理
  </FeatureCard>
  <FeatureCard icon="phone" title="電話注文 (MOTO)">
    リーダーに詳細を入力することで、電話越しにカード支払いを受け付ける
  </FeatureCard>
</FeatureGrid>

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [WooCommerce用ストライプターミナルのリリースページ](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases)にアクセス
2. 最新の**stripe-terminal-for-woocommerce.zip**ファイルをダウンロード
3. WordPress管理画面で`プラグイン > 新規追加 > プラグインのアップロード`に移動
4. zipファイルをアップロードし、プラグインを有効化

  </StepCard>
  <StepCard number="2" title="ストライプ設定の構成">

1. `WP管理 > WooCommerce > 設定 > 支払い`に移動
2. 支払い方法リストから**ストライプターミナル**を見つける
3. **ストライプターミナル**をクリックして設定にアクセス
4. **ストライプシークレットキー**を入力（ストライプダッシュボードから取得可能）
5. 設定を保存

:::note
WooCommerceの設定でストライプターミナルゲートウェイを有効にする必要はありません。次のステップでPOS専用に有効化されます。
:::

  </StepCard>
  <StepCard number="3" title="WCPOSでの有効化" isLast>

1. `WP管理 > POS > 設定 > チェックアウト`に移動
2. リストから**ストライプターミナル**ゲートウェイを見つける
3. POSで使用するためにゲートウェイを有効化
4. 設定を保存

  </StepCard>
</Steps>

## 使用法 {#usage}

### リーダーの接続 {#connecting-a-reader}

POSのチェックアウト中にストライプターミナルゲートウェイを選択すると:

1. **接続方法の選択**: 物理リーダーを接続するか、シミュレータを使用できます
2. **物理リーダー**: 画面の指示に従ってストライプターミナルデバイスを接続
3. **シミュレータ**: ハードウェアなしでさまざまな支払いシナリオをテストするためにシミュレータモードを選択

### 支払い処理 {#processing-payments}

1. **アイテムの追加**: POSに製品をカートに追加
2. **ゲートウェイの選択**: 支払い方法として「ストライプターミナル」を選択
3. **リーダーの接続**: リーダーを接続するか、シミュレータモードを選択
4. **支払い処理**: 取引を完了するための手順に従う
5. **確認**: 成功した支払いで自動的に注文が完了します

### シミュレータでのテスト {#testing-with-simulator}

シミュレータを使用して、さまざまな支払い方法やシナリオをテストできます：

- **カード支払い**: 異なるカードタイプ（Visa、Mastercardなど）をテスト
- **非接触型支払い**: タッチ決済を模擬
- **エラーシナリオ**: 支払い拒否や他のエラー条件をテスト
- **異なる金額**: さまざまな取引額をテスト

### 電話注文 (MOTO) {#phone-orders-moto}

MOTO（郵送注文/電話注文）を使用すると、物理的に現れていない顧客のためにカード支払いを処理できます — たとえば、電話で注文を取るときです。タップや挿入の代わりに、商人はカードの詳細をターミナルリーダーの画面に直接入力します。

#### セットアップ {#setup}

<Steps>
  <StepCard number="1" title="ストライプからMOTOアクセスのリクエスト">

MOTOはデフォルトで有効になっていません。 [ストライプサポート](https://support.stripe.com/contact)に連絡して、アカウントのMOTO権限を有効にするように依頼してください。このプロセスは迅速ですが、ストライプの手動承認が必要です。

  </StepCard>
  <StepCard number="2" title="プラグイン設定での有効化">

1. `WP管理 > WooCommerce > 設定 > 支払い > ストライプターミナル`に移動
2. **電話注文 (MOTO)** チェックボックスをチェック
3. 設定を保存

  </StepCard>
  <StepCard number="3" title="互換性のあるリーダーを接続" isLast>

MOTOは、[対応ターミナル](#supported-terminals-internet-connected)にリストされた互換性のあるインターネット接続リーダーでのみ機能します。他のリーダータイプではトグルは表示されません。

  </StepCard>
</Steps>

#### 電話注文の受け付け {#taking-a-phone-order}

1. 互換性のあるリーダーを接続（[対応ターミナル](#supported-terminals-internet-connected)を参照）
2. 支払い画面で**電話注文**をオンにする
3. **カード支払いを受け取る**をクリック — リーダーはタップ/挿入を促すのではなく、カード番号入力画面を表示します
4. 顧客のカード番号、有効期限、CVVをリーダーに入力
5. 支払いは通常通り処理されます

:::tip
MOTO支払いは、支払い方法タイプとして`card`を使用し、`card_present`ではありません。これは、ストライプの観点からオンライン取引に近い扱いとなり、対面レートではなく通常のオンラインカード処理手数料が適用されます。
:::

:::caution
電話注文のトグルは、次の三つの条件が満たされたときのみ表示されます：MOTO設定がプラグイン設定で有効化され、互換性のあるリーダーが接続され、リーダーがシミュレータでないこと。トグルが表示されない場合は、これらの条件を確認してください。
:::

## 要件 {#requirements}

<RequirementsList>
  <RequirementItem label="ストライプアカウント">ターミナルが有効なアクティブなストライプアカウント</RequirementItem>
  <RequirementItem label="APIキー">ダッシュボードからのストライプシークレットキー</RequirementItem>
  <RequirementItem label="WCPOS">POSチェックアウト用のプロ版が必要</RequirementItem>
  <RequirementItem label="HTTPS">あなたのサイトはセキュリティのためにSSL/HTTPSを使用する必要があります</RequirementItem>
</RequirementsList>

## ハードウェア互換性 {#hardware-compatibility}

:::important 接続要件
この実装はストライプのJavaScript SDKを使用しており、ウェブアプリケーションを介して機能しますが、**インターネット接続のターミナルのみ**が必要です。現在、Bluetoothターミナルはサポートされていません。
:::

### 対応ターミナル（インターネット接続） {#supported-terminals-internet-connected}

- **ストライプリーダー S700/S710** ✅ - イーサネット/WiFi接続のターミナル
- **WisePOS E** ✅ - WiFi接続のターミナル

### 非対応ターミナル（Bluetooth） {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - Bluetoothのみ
- **BBPOS WisePad 3** ❌ - Bluetoothのみ  
- **Verifone P400** ❌ - 廃止、Stripeでサポート終了

:::note 今後のサポート
Bluetoothターミナルのサポートは今後のリリースでiOSおよびAndroidアプリケーションに追加される予定です。これにより、M2やWisePad 3を含むすべてのStripe Terminal認定リーダーがサポートされるようになります。最新情報は公式WCPOSリリースノートまたはStripeのドキュメントを参照してください。
:::

### 一般的な問題 {#common-issues}

<Accordion>
  <AccordionItem question="リーダーが接続できない">

  - [対応インターネット接続ターミナル](#supported-terminals-internet-connected)を使用していることを確認
  - ターミナルがWiFi/Ethernetに接続されており、オンラインであることを確認
  - ストライプアカウントにターミナルが有効化されていることを確認
  - ターミナルがストライプダッシュボードに登録されていることを確認

  </AccordionItem>

  <AccordionItem question="支払いが拒否された">

  - ストライプアカウントがアクティブで適切であることを確認
  - 使用しているカードが有効であることを確認
  - 利用可能な十分な資金があることを確認

  </AccordionItem>

  <AccordionItem question="電話注文トグルが表示されない">

  - `WooCommerce > 設定 > 支払い > ストライプターミナル`で**電話注文 (MOTO)**設定が有効になっていることを確認
  - 互換性のあるリーダーに接続していることを確認（[対応ターミナル](#supported-terminals-internet-connected)を参照） — 他のリーダータイプではトグルが隠されています
  - シミュレータを使用している場合、トグルは表示されません

  </AccordionItem>

  <AccordionItem question="MOTO支払いがエラーで失敗する">

  - ストライプがあなたのアカウントでMOTO権限を有効にしていることを確認 — まだであれば[ストライプサポート](https://support.stripe.com/contact)に連絡
  - リーダーにカードの詳細が正しく入力されていることを再確認
  - MOTO支払いはより厳格な詐欺チェックを行う場合があります — カードが有効であり、十分な資金があることを確認

  </AccordionItem>

  <AccordionItem question="SSL証明書エラー">

  - ストライプターミナルはHTTPSを必要とします - サイトに有効なSSL証明書があることを確認
  - SSL証明書が正しく設定されていることを確認

  </AccordionItem>
</Accordion>
