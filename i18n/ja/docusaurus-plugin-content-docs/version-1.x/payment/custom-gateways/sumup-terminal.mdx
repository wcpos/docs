# SumUpターミナルゲートウェイ

SumUpターミナルゲートウェイを使用すると、WooCommerce POS内でSumUpカードリーダーを使用してカード支払いを受け付けることができます。このゲートウェイは、対面での取引のためにSumUpの決済処理システムとシームレスに統合されています。

## 機能

- **ハードウェア統合**: SumUpカードリーダーをインターネット接続経由でPOSシステムに接続
- **リアルタイム処理**: 即時の支払い処理と注文の完了
- **自動検出**: 支払いが検出されると、自動的に注文が完了
- **安全な取引**: SumUpを介したPCI準拠の支払い処理
- **簡単なペアリング**: ペアリングコードによるシンプルなデバイスペアリングプロセス
- **ウェブベースのSDK**: ウェブアプリケーションの互換性のためにSumUpのJavaScript SDKを使用

## インストール

### ステップ1: ダウンロードとインストール

1. [SumUpターミナル for WooCommerceリリースページ](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)にアクセス
2. 最新の**sumup-terminal-for-woocommerce.zip**ファイルをダウンロード
3. WordPressの管理画面で、`プラグイン > 新規追加 > プラグインのアップロード`に移動
4. zipファイルをアップロードし、プラグインを有効化

### ステップ2: SumUp設定の構成

1. `WP管理 > WooCommerce > 設定 > 支払い`に移動
2. 支払い方法リストで**SumUpターミナル**を見つける
3. **SumUpターミナル**をクリックして設定にアクセス
4. **SumUp APIキー**を入力（SumUpマーチャントダッシュボードから取得可能）
5. 設定を保存

:::note
WooCommerceの設定でSumUpターミナルゲートウェイを有効にする必要はありません。次のステップでPOS用に特別に有効になります。
:::

### ステップ3: SumUpターミナルをペアリング

1. 同じ設定ページで、**リーダーのペアリング**セクションを見つける
2. SumUpデバイスでペアリング画面に移動してペアリングコードを表示
3. SumUpデバイスに表示されたペアリングコードを入力
4. **"リーダーをペアリング"**をクリックして接続を確立
5. リーダーが正常にペアリングされたことを確認するまで待つ

:::warning 重要
支払いを処理する前にリーダーが正常にペアリングされている必要があります。プロセスが完了するまで進まないでください。
:::

### ステップ4: WooCommerce POSで有効化

1. `WP管理 > WooCommerce POS > 設定 > チェックアウト`に移動
2. リストで**SumUpターミナル**ゲートウェイを見つける
3. POSで使用するためにゲートウェイを有効化
4. 設定を保存

## 使用方法

### 支払い処理

1. **アイテムを追加**: POSに商品をカートに追加
2. **ゲートウェイを選択**: 支払い方法として"SumUpターミナル"を選択
3. **支払いを開始**: SumUpデバイスで新しい支払いを開始するためにクリック
4. **顧客の支払い**: 顧客がSumUpターミナルで支払いを完了
5. **自動完了**: 支払いが検出されると自動的に注文が完了

### 支払いコントロール

SumUpターミナルゲートウェイを使用する際には、次のオプションがあります：

- **新規支払いを開始**: 接続されたターミナルで支払いリクエストを開始
- **支払いをキャンセル**: 現在処理中の支払いをキャンセル
- **支払い状況**: 現在の支払いプロセスの状況を監視

### 注文管理

- **自動更新**: 成功した支払い後に注文が自動的に完了としてマークされる
- **支払い追跡**: すべての支払い詳細が注文ノートに記録される
- **領収書の生成**: 成功した支払い後に標準のPOS領収書が生成される

## 要件

- **SumUpアカウント**: アクティブなSumUpマーチャントアカウント
- **APIアクセス**: マーチャントダッシュボードから取得したSumUp APIキー
- **互換性のあるハードウェア**: SumUpカードリーダーデバイス
- **WooCommerce POS**: 無料またはプロバージョンがインストールされて有効化されていること
- **安定した接続**: API通信のための信頼できるインターネット接続

## ハードウェア互換性

:::important 接続要件
この実装はSumUpのJavaScript SDKを使用しており、ウェブアプリケーションを介して動作しますが、**インターネット接続されたターミナル専用**です。Bluetoothターミナルは現在サポートされていません。
:::

### サポートされているターミナル（インターネット接続）

- **SumUp Solo** ✅ - WiFi/Ethernet接続ターミナル

### サポートされていないターミナル（Bluetooth）

- **SumUp Air** ❌ - Bluetooth専用
- **SumUp 3G** ❌ - Bluetooth専用

:::note 今後のサポート
Bluetoothターミナルのサポートは、**2026年第1四半期**にiOSおよびAndroidアプリケーションに追加される予定です。これにより、Airおよび3Gモデルを含むすべてのSumUp認定ターミナルがサポートされるようになります。
:::

## トラブルシューティング

### 一般的な問題

**リーダーがペアリングしない**
- サポートされているインターネット接続ターミナル（Soloのみ）を使用していることを確認
- SumUp SoloデバイスがWiFiに接続され、オンラインであることを確認
- デバイスがペアリングモードになっており、ペアリングコードが正しく入力されていることを確認
- SumUpアカウントにAPIアクセスが有効になっていることを確認
- ペアリング中に安定したインターネット接続を確保

**支払いが処理されない**
- リーダーが正しくペアリングされていることを確認
- SumUpアカウントがアクティブで良好な状態であることを確認
- デバイスに十分なバッテリーと接続があることを確認
- APIキーが正しくアクティブであることを確認

**注文が完了しない**
- インターネット接続の安定性を確認
- SumUpアカウントのWebhook設定を確認
- SumUpターミナルで支払いが成功していることを確認
- API通信の問題についてWordPressエラーログを確認

### APIレート制限

ゲートウェイはAPIコールを減らし、レート制限を避けるよう最適化されています：
- 支払いステータスは効率的にチェックされます
- 不必要なAPIリクエストは最小限に
- 自動再試行ロジックが一時的な障害を処理します

### ヘルプを得る
