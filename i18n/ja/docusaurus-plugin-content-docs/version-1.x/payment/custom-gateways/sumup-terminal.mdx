# SumUpターミナルゲートウェイ

SumUpターミナルゲートウェイにより、WooCommerce POSで直接SumUpカードリーダーを使用してカード決済を受け付けることができます。このゲートウェイは、店舗での取引のためのSumUpの決済システムとのシームレスな統合を提供します。

## 機能

- **ハードウェア統合**: インターネット接続を介してSumUpカードリーダーをPOSシステムに接続
- **リアルタイム処理**: 即時の支払い処理と注文完了
- **自動検出**: 支払いが検出されると、注文が自動的に完了
- **安全な取引**: SumUpを通じたPCI準拠の決済処理
- **簡単なペアリング**: ペアリングコードを使用した簡単なデバイスペアリングプロセス
- **WebベースのSDK**: Webアプリケーションとの互換性のためのSumUpのJavaScript SDKを使用

## インストール

### ステップ1: ダウンロードとインストール

1. [SumUp Terminal for WooCommerceリリースページ](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)にアクセス
2. 最新の**sumup-terminal-for-woocommerce.zip**ファイルをダウンロード
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインをアップロード`に移動
4. zipファイルをアップロードし、プラグインをアクティブ化

### ステップ2: SumUp設定を構成

1. `WP管理画面 > WooCommerce > 設定 > 支払い方法`に移動
2. 支払い方法リストで**SumUpターミナル**を探す
3. **SumUpターミナル**をクリックして設定にアクセス
4. **SumUp APIキー**を入力（SumUpの加盟店ダッシュボードから入手可能）
5. 設定を保存

:::note
WooCommerceの設定でSumUpターミナルゲートウェイを有効にする必要はありません。これは後のステップでPOS専用に有効化されます。
:::

### ステップ3: SumUpターミナルをペアリング

1. 同じ設定ページで、**リーダーをペアリング**セクションを探す
2. SumUpデバイスで、ペアリング画面に移動してペアリングコードを表示
3. SumUpデバイスに表示されているペアリングコードを入力
4. **「リーダーをペアリング」**をクリックして接続を確立
5. リーダーが正常にペアリングされたことの確認を待つ

:::warning 重要
支払いを処理する前に、リーダーを正常にペアリングする必要があります。続行する前にペアリングプロセスを完了してください。
:::

### ステップ4: WooCommerce POSで有効化

1. `WP管理画面 > WooCommerce POS > 設定 > チェックアウト`に移動
2. 支払い方法リストで**SumUpターミナル**ゲートウェイを探す
3. POSでの使用を有効化
4. 設定を保存

## 使用方法

### 支払い処理

1. **商品追加**: POSで商品をカートに追加
2. **ゲートウェイ選択**: 支払い方法として「SumUpターミナル」を選択
3. **支払い開始**: SumUpデバイスで新しい支払いを開始
4. **顧客支払い**: 顧客がSumUpターミナルで支払いを完了
5. **自動完了**: 支払いが検出されると、注文が自動的に完了

### 支払いコントロール

SumUpターミナルゲートウェイを使用する際、以下のオプションがあります：

- **新しい支払いを開始**: 接続されたターミナルで支払いリクエストを開始
- **支払いをキャンセル**: 処理中の支払いをキャンセル
- **支払いステータス**: 支払いプロセスの現在の状態を監視

### 注文管理

- **自動更新**: 支払いが正常に完了すると、注文は自動的に完了としてマークされます
- **支払い追跡**: すべての支払い詳細が注文メモに記録されます
- **領収書生成**: 支払い成功後に標準のPOS領収書が生成されます

## 要件

- **SumUpアカウント**: アクティブなSumUp加盟店アカウント
- **API アクセス**: 加盟店ダッシュボードからのSumup APIキー
- **互換性のあるハードウェア**: SumUpカードリーダーデバイス
- **WooCommerce POS**: フリーまたはプロバージョンがインストールされ、アクティブ化
- **安定した接続**: API通信のための信頼性の高いインターネット接続

## ハードウェア互換性

:::important 接続要件
この実装はSumUpのJavaScript SDKを使用するため、Webアプリケーションを通じて動作しますが、**インターネットに接続されたターミナルのみ**が必要です。Bluetoothターミナルは現在サポートされていません。
:::

### サポートされているターミナル（インターネット接続）

- **SumUp Solo** ✅ - WiFi/Ethernetに接続されたターミナル

### サポートされていないターミナル（Bluetooth）

- **SumUp Air** ❌ - Bluetoothのみ
- **SumUp 3G** ❌ - Bluetoothのみ

:::note 将来のサポート
Bluetoothターミナルのサポートは**2026年第1四半期**にiOSおよびAndroidアプリケーションに追加される予定です。これにより、Air and 3Gモデルを含むすべてのSumUp認定ターミナルのサポートが可能になります。
:::

## トラブルシューティング

### 一般的な問題

**リーダーがペアリングできない**
- サポートされているインターネット接続ターミナル（Soloのみ）を使用していることを確認
- SumUp Soloデバイスが WiFi に接続されオンラインであることを確認
- デバイスがペアリングモードであり、ペアリングコードが正しく入力されていることを確認
- SumUpアカウントでAPIアクセスが有効になっていることを確認
- ペアリング中の安定したインターネット接続を確認

**支払いが処理されない**
- リーダーが正しくペアリングされていることを確認
- SumUpアカウントがアクティブで良好な状態であることを確認
- デバイスのバッテリーと接続性を確認
- APIキーが正確でアクティブであることを確認

**注文が完了しない**
- インターネット接続の安定性を確認
- SumUpアカウントのWebhook設定を確認
- SumUpターミナルで支払いが成功したことを確認
- API通信の問題についてWordPressエラーログを確認

### APIレート制限

ゲートウェイは、APIコールを削減し、レート制限を回避するように最適化されています：
- 支払いステータスが効率的にチェックされます
- 不必要なAPIリクエストが最小限に抑えられます
- 一時的な障害を処理する自動再試行ロジック
