# SumUpターミナルゲートウェイ {#sumup-terminal-gateway}

SumUpターミナルゲートウェイを使用すると、WooCommerce POS内でSumUpカードリーダーを使用してカードでの支払いを受け付けることができます。このゲートウェイは、対面取引のためのSumUpの決済処理システムとのシームレスな統合を提供します。

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [SumUp Terminal for WooCommerce リリースページ](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)にアクセスします。
2. 最新の**sumup-terminal-for-woocommerce.zip**ファイルをダウンロードします。
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインのアップロード`に移動します。
4. zipファイルをアップロードし、プラグインを有効化します。

  </StepCard>
  <StepCard number="2" title="SumUp設定の構成">

1. `WP Admin > WooCommerce > 設定 > 支払い`に移動します。
2. 支払い方法リストから**SumUpターミナル**を見つけます。
3. **SumUpターミナル**をクリックして設定にアクセスします。
4. **SumUp APIキー**を入力します（SumUpマーチャントダッシュボードから入手可能）。
5. 設定を保存します。

:::note
WooCommerce設定でSumUpターミナルゲートウェイを有効にする必要はありません。後のステップで特にPOS用に有効になります。
:::

  </StepCard>
  <StepCard number="3" title="SumUpターミナルのペアリング">

1. 同じ設定ページで、**リーダーをペアリング**セクションを探します。
2. SumUpデバイスでペアリング画面に移動し、ペアリングコードを表示します。
3. SumUpデバイスに表示されたペアリングコードを入力します。
4. **"リーダーをペアリング"**をクリックして接続を確立します。
5. リーダーが正常にペアリングされたことを確認するまで待ちます。

:::warning 重要
支払いを処理する前に、リーダーが正常にペアリングされている必要があります。操作を進める前にペアリングプロセスが完了していることを確認してください。
:::

  </StepCard>
  <StepCard number="4" title="WooCommerce POSで有効化" isLast>

1. `WP Admin > WooCommerce POS > 設定 > チェックアウト`に移動します。
2. リストから**SumUpターミナル**ゲートウェイを見つけます。
3. POSでの使用のためにゲートウェイを有効化します。
4. 設定を保存します。

  </StepCard>
</Steps>

## 使用法 {#usage}

### 支払いの処理 {#processing-payments}

1. **アイテムの追加**: POSでカートに商品を追加します。
2. **ゲートウェイの選択**: 支払い方法として「SumUpターミナル」を選択します。
3. **支払いの開始**: SumUpデバイスで新しい支払いを開始するためにクリックします。
4. **顧客の支払い**: 顧客がSumUpターミナルで支払いを完了します。
5. **自動完了**: 支払いが検出されると、注文が自動的に完了します。

### 支払い制御 {#payment-controls}

SumUpターミナルゲートウェイを使用する場合、以下のオプションがあります：

- **新しい支払いの開始**: 接続されたターミナルでの支払いリクエストを開始します。
- **支払いのキャンセル**: 現在進行中の支払いをキャンセルします。
- **支払い状況**: 支払いプロセスの現在の状況を監視します。

### 注文管理 {#order-management}

- **自動更新**: 支払いが成功すると、注文は自動的に完了としてマークされます。
- **支払い追跡**: すべての支払い詳細が注文ノートに記録されます。
- **領収書の生成**: 成功した支払い後に標準POS領収書が生成されます。

## ハードウェアの互換性 {#hardware-compatibility}

:::important 接続要件
この実装はSumUpのJavaScript SDKを使用しており、ウェブアプリケーションを介して動作しますが、**インターネット接続されたターミナルのみ**を必要とします。Bluetoothターミナルは現在サポートされていません。
:::

### サポートされているターミナル（インターネット接続） {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet接続されたターミナル

### サポートされていないターミナル（Bluetooth） {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Bluetoothのみ
- **SumUp 3G** ❌ - Bluetoothのみ

:::note 今後のサポート
Bluetoothターミナルのサポートは**2026年第1四半期**にiOSおよびAndroidアプリケーションに追加される予定です。これにより、Airおよび3Gモデルを含むすべてのSumUp認定ターミナルのサポートが可能になります。
:::

## トラブルシューティング {#troubleshooting}

### よくある問題 {#common-issues}

<Accordion>
  <AccordionItem question="リーダーがペアリングされない">

  - サポートされているインターネット接続ターミナル（Soloのみ）を使用していることを確認してください。
  - SumUp SoloデバイスがWiFiに接続されてオンラインであることを確認してください。
  - デバイスがペアリングモードになっており、ペアリングコードが正しく入力されていることを確認してください。
  - SumUpアカウントにAPIアクセスが有効になっていることを確認してください。
  - ペアリング中は安定したインターネット接続を確保してください。

  </AccordionItem>

  <AccordionItem question="支払いが処理されない">

  - リーダーが正しくペアリングされていることを確認してください。
  - SumUpアカウントがアクティブで良好な状態であることを確認してください。
  - デバイスのバッテリーと接続が十分であることを確認してください。
  - APIキーが正しく、アクティブであることを確認してください。

  </AccordionItem>

  <AccordionItem question="注文が完了しない">

  - インターネット接続の安定性を確認してください。
  - SumUpアカウントのWebhook設定を確認してください。
  - 支払いがSumUpターミナルで成功したことを確認してください。
  - API通信の問題についてWordPressのエラーログを確認してください。

  </AccordionItem>
</Accordion>

### APIレート制限 {#api-rate-limits}

ゲートウェイはAPI呼び出しの削減とレート制限の回避を目指して最適化されています：
- 支払い状況は効率的に確認されます。
- 不必要なAPIリクエストを最小限にします。
- 一時的な失敗を処理するための自動再試行ロジックがあります。
