# Vipps MobilePay ゲートウェイ {#vipps-mobilepay-gateway}

Vipps MobilePay ゲートウェイを使用すると、WCPOS内で直接電話ベースの支払いを受け入れることができます。顧客は画面上のQRコードをスキャンするか、電話でプッシュ通知を確認することで支払います。カードリーダーのハードウェアは不要です。

## 特徴 {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="QRコード決済">
    顧客がスマートフォンでスキャンして支払うためのQRコードをチェックアウト時に表示します
  </FeatureCard>
  <FeatureCard icon="bolt" title="プッシュ通知">
    顧客の電話番号を入力することで、顧客の電話に直接支払い要求を送信します
  </FeatureCard>
  <FeatureCard icon="globe" title="北欧のカバレッジ">
    ノルウェーのVipps、デンマークとフィンランドのMobilePayに対応しています
  </FeatureCard>
  <FeatureCard icon="credit-card" title="自動キャプチャ">
    認証後すぐに資金をキャプチャするか、手動キャプチャ用に予約します
  </FeatureCard>
</FeatureGrid>

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [WCPOS Vipps MobilePay リリースページ](https://github.com/wcpos/wcpos-vipps/releases)にアクセスします
2. 最新の**wcpos-vipps.zip**ファイルをダウンロードします
3. WordPress管理画面に移動し、`プラグイン > 新規追加 > プラグインのアップロード`に進みます
4. zipファイルをアップロードし、プラグインを有効化します

  </StepCard>
  <StepCard number="2" title="Vipps 認証情報を設定">

1. `WP管理画面 > WooCommerce > 設定 > 支払い`に移動します
2. 支払い方法の一覧から**WCPOS Vipps MobilePay**を見つけます
3. **WCPOS Vipps MobilePay**をクリックして設定にアクセスします
4. [Vippsポータル](https://portal.vipps.no/)から取得した認証情報を入力します：
   - **商人シリアル番号 (MSN)**
   - **クライアントID**
   - **クライアントシークレット**
   - **サブスクリプションキー**
5. 設定を保存します

:::note
POS用にここでゲートウェイを有効化する必要はありません。次のステップでPOS専用に有効化されます。WooCommerce設定で有効化するとオンラインストアのチェックアウトでも利用可能になるため、テストに役立つ場合があります。
:::

:::tip すでにVippsプラグインを使用していますか？
公式の[Checkout with Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/)プラグインがインストールされている場合、このプラグインを有効化すると認証情報が自動的にインポートされます。手動での認証情報の入力をスキップできます。
:::

  </StepCard>
  <StepCard number="3" title="WCPOSで有効化" isLast>

1. `WP管理画面 > WCPOS > 設定 > チェックアウト`に移動します
2. リストから**WCPOS Vipps MobilePay**ゲートウェイを見つけます
3. POSで使用するためにゲートウェイを有効化します
4. 設定を保存します

  </StepCard>
</Steps>

## 使用方法 {#usage}

### 支払いの処理 — QRコード {#processing-payments-qr-code}

1. **アイテムを追加**: POSに製品をカートに追加します
2. **ゲートウェイを選択**: 支払い方法として「Vipps MobilePay」を選択します
3. **QRコードを生成**: 「QRコード生成」ボタンをクリックします
4. **顧客がスキャン**: 顧客は自分のVippsまたはMobilePayアプリでQRコードをスキャンします
5. **顧客が確認**: 顧客はアプリで支払いを確認します
6. **自動完了**: 支払いが認証されると、注文は自動的に完了します

### 支払いの処理 — プッシュ通知 {#processing-payments-push-notification}

1. **アイテムを追加**: POSに製品をカートに追加します
2. **ゲートウェイを選択**: 支払い方法として「Vipps MobilePay」を選択します
3. **電話番号を入力**: 顧客の電話番号を入力します
4. **電話に送信**: 「電話に送信」ボタンをクリックします
5. **顧客が確認**: 顧客はプッシュ通知を受け取り、アプリで支払いを確認します
6. **自動完了**: 支払いが認証されると、注文は自動的に完了します

### 支払いのキャンセル {#cancelling-a-payment}

顧客が確認を待っている間に、**支払いキャンセル**ボタンをクリックして取引を中止できます。これにより、Vipps側の保留中の支払いがキャンセルされ、チェックアウトインターフェースがリセットされます。

### 返金 {#refunds}

返金は標準のWooCommerce返金プロセスを通じて処理されます。注文を開き、**返金**をクリックし、金額を入力すると、返金はVipps APIを通じて自動的に処理されます。

## 対応市場 {#supported-markets}

Vipps MobilePayは、北欧地域で二つのブランドの下で運営されています：

| 地域 | ブランド | 通貨 |
|------|---------|------|
| ノルウェー | Vipps | NOK |
| デンマーク、フィンランド | MobilePay | DKK, EUR |

あなたのマーチャントアカウントによって、利用可能な市場と通貨が決まります。対応する市場の顧客は、地元のVippsまたはMobilePayアプリを使用して支払うことができます。

## 要件 {#requirements}

<RequirementsList>
  <RequirementItem label="Vipps アカウント">API認証情報を持つアクティブなVipps MobilePayマーチャントアカウント</RequirementItem>
  <RequirementItem label="API 認証情報">商人シリアル番号、クライアントID、クライアントシークレット、およびサブスクリプションキー</RequirementItem>
  <RequirementItem label="WCPOS">POSチェックアウトにはプロバージョンが必要です。ゲートウェイは、プロなしでも標準のWooCommerceウェブチェックアウトで機能します。</RequirementItem>
  <RequirementItem label="HTTPS">あなたのサイトはSSL/HTTPSを使用する必要があります（Vipps APIによって要求されます）</RequirementItem>
</RequirementsList>

## トラブルシューティング {#troubleshooting}

### 一般的な問題 {#common-issues}

<Accordion>
  <AccordionItem question="支払いが完了しない">

  - チェックアウトは最大5分間ポーリングします — 顧客がそのウィンドウ内に確認しない場合、支払いはタイムアウトします
  - テストモードと本番の認証情報を混同していないことを確認してください
  - POSデバイスと顧客の電話の両方でインターネット接続が安定していることを確認してください

  </AccordionItem>

  <AccordionItem question="無効な認証情報エラー">

  - 4つの認証フィールド（MSN、クライアントID、クライアントシークレット、サブスクリプションキー）を再確認してください
  - テストモードが有効な場合はテスト認証情報を使用し、無効な場合は本番認証情報を使用していることを確認してください
  - [Vippsポータル](https://portal.vipps.no/)で認証情報を確認してください

  </AccordionItem>

  <AccordionItem question="QRコードが生成されない">

  - あなたのサイトがHTTPSで動作していることを確認してください — Vipps APIはHTTPサイトからのリクエストを拒否します
  - APIエラーの詳細についてWooCommerceログ（`WooCommerce > ステータス > ログ`）を確認してください
  - あなたの商人アカウントがアクティブでePayment APIが有効であることを確認してください

  </AccordionItem>
</Accordion>
