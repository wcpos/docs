# メール請求書ゲートウェイ {#email-invoice-gateway}

メール請求書ゲートウェイは、WCPOSから直接顧客にメールで支払い請求書を送信できる機能です。これは、顧客が後で支払うことを好む場合や、ウェブストアの支払い方法を使用してオンラインで支払いを完了する必要がある状況に最適です。

    店舗での注文を完了するか、リモートで支払いを完了させることができます

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [メール請求書ゲートウェイのリリースページ](https://github.com/wcpos/email-invoice-gateway/releases)にアクセス
2. 最新の **woocommerce-pos-email-invoice-gateway.zip** ファイルをダウンロード
3. WordPress管理画面で `プラグイン > 新規追加 > プラグインのアップロード` に移動
4. zipファイルをアップロードし、プラグインを有効化

  </StepCard>
  <StepCard number="2" title="WCPOSで有効化" isLast>

1. `WP管理 > POS > 設定 > チェックアウト` に移動
2. リストから **WCPOS請求書支払いゲートウェイ** を見つけます
3. POSでの使用のためにゲートウェイを有効化
4. 設定を保存

:::note
このゲートウェイには、追加のAPIキーや外部サービスの設定は必要ありません。既存のWordPressメールシステムとWooCommerceの支払い方法を使用します。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### 請求書の送信 {#sending-invoices}

1. **アイテムを追加**: POSでカートに商品を追加
2. **ゲートウェイを選択**: 支払い方法として「メール請求書」を選択
3. **メールアドレス**: 
   - 既存の顧客の場合: 顧客データからメールが自動的に入力されます
   - ゲスト注文の場合: 顧客のメールアドレスを手動で入力
4. **オプション**: 「請求先住所に保存」をチェックして顧客の請求情報を更新
5. **請求書を送信**: 請求書メールを送信するプロセスを完了

### 顧客体験 {#customer-experience}

請求書を送信すると、顧客は以下を含むメールを受け取ります：

- **注文詳細**: アイテム、数量、価格の詳細な内訳
- **合計金額**: 税金や手数料を含む最終的な金額
- **支払いリンク**: オンラインで支払いを完了するための直接リンク
- **注文情報**: 注文番号と参照の詳細

### 支払い完了 {#payment-completion}

顧客は以下の2つの方法で支払いを完了できます：

1. **オンライン支払い**: メール内の「この注文の支払い」リンクをクリック
2. **店舗での返却**: 他のPOS支払い方法を使用して支払いを完了するために戻る

「この注文の支払い」リンクは、顧客をウェブサイトの専用支払いページに誘導し、そこで有効化されたウェブ支払いゲートウェイ（Stripe、PayPalなど）を使用できます。

## メール管理 {#email-management}

### 既存の顧客 {#existing-customers}

- メールアドレスは顧客プロフィールから自動的に取得されます
- 顧客の請求情報は保持されます
- 注文履歴は顧客アカウントの下で保たれます

### ゲスト顧客 {#guest-customers}

- 一度きりの顧客のためにメールアドレスを手動で入力
- 将来の参照のために請求先住所にメールを保存するオプション
- ゲスト注文はWooCommerceで適切に追跡されます

### メールテンプレート {#email-templates}

このゲートウェイはWooCommerceの標準メールテンプレートを使用します：

- **カスタマイズ可能**: WooCommerceのメール設定を通じてメールの外観を変更
- **ブランド化**: ストアロゴとブランディングを含める
- **プロフェッショナル**: クリーンでプロフェッショナルな請求書フォーマット
- **モバイルフレンドリー**: すべてのデバイスに対応したレスポンシブデザイン

## 使用例 {#use-cases}

### 最適な用途 {#perfect-for}

- **B2B営業**: 会計システムを通じて支払いを処理する必要があるビジネス顧客
- **大口注文**: 承認または処理時間が必要な高額取引
- **リモート顧客**: 店舗を離れた後に支払いを完了する必要がある顧客
- **アカウント顧客**: 確立された支払い条件を持つ常連顧客
- **見積もりから注文へ**: 見積もりや計算書を支払い可能な請求書に変換

## トラブルシューティング {#troubleshooting}

### よくある問題 {#common-issues}

<Accordion>
  <AccordionItem question="メールが送信されない">

  - WordPressメール構成を確認
  - カスタムメールサービスを使用している場合はSMTP設定を確認
  - WordPressメールテストプラグインでテスト
  - テストメールのスパムフォルダを確認

  </AccordionItem>

  <AccordionItem question="支払いリンクが機能しない">

  - WooCommerceが正しく設定されていることを確認
  - ウェブチェックアウト用に少なくとも1つの支払いゲートウェイが有効化されていることを確認
  - SSL証明書が正しくインストールされていることを確認
  - 注文ステータスが支払いを許可していることを確認

  </AccordionItem>

  <AccordionItem question="顧客が支払いを完了できない">

  - ウェブ支払いゲートウェイがアクティブで構成されていることを確認
  - 注文が期限切れまたはキャンセルされていないことを確認
  - 顧客が十分な支払い方法の制限を持っていることを確認
  - 自分自身で支払いプロセスをテスト

  </AccordionItem>
</Accordion>

### メール配信 {#email-delivery}

信頼できるメール配信のために、次のことを考慮してください：

- **SMTPサービス**: SendGrid、Mailgun、Amazon SESなどのサービスを使用
- **メールプラグイン**: より良い配信のためにWordPress SMTPプラグインをインストール
- **ドメイン認証**: SPF、DKIM、およびDMARCレコードを設定
- **監視**: メール配信と開封率を追跡

### ヘルプを得る {#getting-help}

技術サポートが必要な場合は：
- [GitHubリポジトリ](https://github.com/wcpos/email-invoice-gateway)を訪れて問題を報告
- テンプレートのカスタマイズについてWooCommerceのメール文書を確認
- WordPressメールテストツールでメール機能をテスト
