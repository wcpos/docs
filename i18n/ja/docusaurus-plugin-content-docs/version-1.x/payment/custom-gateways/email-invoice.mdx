# メール請求書ゲートウェイ {#email-invoice-gateway}

メール請求書ゲートウェイを使用すると、WooCommerce POSから直接顧客に支払い請求書をメールで送信できます。これは、顧客が後で支払いたい場合や、ウェブストアの支払い方法を使用してオンラインでの支払いを完了する必要がある場合に最適です。

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [メール請求書ゲートウェイのリリースページ](https://github.com/wcpos/email-invoice-gateway/releases)にアクセス
2. 最新の **woocommerce-pos-email-invoice-gateway.zip** ファイルをダウンロード
3. WordPress管理画面に移動し、`プラグイン > 新規追加 > プラグインをアップロード`を選択
4. zipファイルをアップロードし、プラグインを有効化

  </StepCard>
  <StepCard number="2" title="WooCommerce POSでの有効化" isLast>

1. `WP管理 > WooCommerce POS > 設定 > チェックアウト`に移動
2. リスト内の **WooCommerce POS請求書支払いゲートウェイ** を見つける
3. POSで使用するためにゲートウェイを有効にする
4. 設定を保存

:::note
このゲートウェイは追加のAPIキーや外部サービスの設定を必要としません。既存のWordPressメールシステムとWooCommerceの支払い方法を使用します。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### 請求書の送信 {#sending-invoices}

1. **アイテムを追加**: POSでショッピングカートに商品を追加
2. **ゲートウェイを選択**: 支払い方法として「メール請求書」を選択
3. **メールアドレス**:
   - 既存顧客の場合: 顧客データからメールが自動的に入力されます
   - ゲスト注文の場合: 顧客のメールアドレスを手動で入力
4. **オプション**: 「請求先住所に保存」にチェックを入れて顧客の請求情報を更新
5. **請求書を送信**: 請求書メールを送信するためのプロセスを完了

### 顧客体験 {#customer-experience}

請求書を送信すると、顧客は次の内容を含むメールを受信します：

- **注文詳細**: アイテム、数量、価格の完全な内訳
- **合計金額**: 税金や手数料を含む最終支払額
- **支払いリンク**: オンラインで支払いを完了するための直接リンク
- **注文情報**: 注文番号と参照詳細

### 支払いの完了 {#payment-completion}

顧客は次の2つの方法で支払いを完了できます：

1. **オンライン支払い**: メール内の「この注文の支払い」リンクをクリック
2. **店舗での返却**: 他のPOS支払い方法を使用して支払いを完了

「この注文の支払い」リンクは、顧客をあなたのウェブサイト上の専用支払いページに案内し、そちらで有効なウェブ支払いゲートウェイ（Stripe、PayPalなど）を使用できます。

## メール管理 {#email-management}

### 既存顧客 {#existing-customers}

- メールアドレスは顧客プロファイルから自動的に取得されます
- 顧客の請求情報は保持されます
- 注文履歴は顧客アカウントで管理されます

### ゲスト顧客 {#guest-customers}

- 一度きりの顧客のためにメールアドレスを手動で入力
- 将来の参照のために請求先住所にメールを保存するオプション
- ゲスト注文はWooCommerceで適切に追跡されます

### メールテンプレート {#email-templates}

このゲートウェイはWooCommerceの標準メールテンプレートを使用します：

- **カスタマイズ可能**: WooCommerceメール設定を通じてメール外観を変更
- **ブランド化**: ストアのロゴやブランディングを含める
- **プロフェッショナル**: クリーンでプロフェッショナルな請求書フォーマット
- **モバイルフレンドリー**: すべてのデバイスに対応したレスポンシブデザイン

## ユースケース {#use-cases}

### 最適な用途 {#perfect-for}

- **B2B販売**: 会計システムを通じて支払いを処理する必要があるビジネス顧客
- **大口注文**: 承認または処理時間を要する高額取引
- **リモート顧客**: 店を離れた後に支払いを完了する必要がある顧客
- **アカウント顧客**: 確立された支払い条件を持つ定期的な顧客
- **見積もりから注文へ**: 見積もりや概算を支払い可能な請求書に変換

## トラブルシューティング {#troubleshooting}

### よくある問題 {#common-issues}

<Accordion>
  <AccordionItem question="メールが送信されない">

  </AccordionItem>

  <AccordionItem question="支払いリンクが機能しない">

  - ウェブチェックアウト用に少なくとも1つの支払いゲートウェイが有効になっているか確認
  - SSL証明書が適切にインストールされているか確認
  - 注文のステータスが支払いを許可しているか確認

  </AccordionItem>

  <AccordionItem question="顧客が支払いを完了できない">

  - ウェブ支払いゲートウェイがアクティブで設定されていることを確認
  - 注文が期限切れまたはキャンセルされていないか確認
  - 顧客が十分な支払い方法の限度を持っているか確認
  - 自分で支払いプロセスをテスト

  </AccordionItem>
</Accordion>

### メール配信 {#email-delivery}

信頼性の高いメール配信のために次のことを考慮してください：

- **SMTPサービス**: SendGrid、Mailgun、またはAmazon SESのようなサービスを使用
- **メールプラグイン**: より良い配信のためにWordPress SMTPプラグインをインストール
- **ドメイン認証**: SPF、DKIM、及びDMARCレコードを設定
- **モニタリング**: メール配信と開封率を追跡

### ヘルプを取得する {#getting-help}

技術サポートが必要な場合：
- [GitHubリポジトリ](https://github.com/wcpos/email-invoice-gateway)を訪れて問題を報告
- テンプレートカスタマイズのためにWooCommerceのメールドキュメンテーションを確認
- WordPressメールテストツールでメール機能をテスト
