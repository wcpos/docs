# メール請求書ゲートウェイ

メール請求書ゲートウェイを使用すると、WooCommerce POSから直接顧客にメールで支払い請求書を送信できます。これは、顧客が後で支払うことを望む場合や、ウェブストアの支払い方法を使用してオンラインで支払いを完了する必要がある場合に最適です。

## 機能

- **メール統合**: POSインターフェースから直接請求書を送信
- **顧客管理**: 既存の顧客データからメールを自動的に入力するか、新しいメールアドレスを追加
- **ゲスト注文**: 請求先住所の更新オプションを備えたゲスト顧客のサポート
- **オンライン支払い**: 顧客がオンラインで支払いを完了するための「この注文の支払い」リンクを受け取ります
- **フレキシブルなワークフロー**: 店舗で注文を完了するか、リモートでの支払い完了を許可

## インストール

### ステップ 1: ダウンロードとインストール

1. [メール請求書ゲートウェイのリリースページ](https://github.com/wcpos/email-invoice-gateway/releases)にアクセス
2. 最新の**woocommerce-pos-email-invoice-gateway.zip**ファイルをダウンロード
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインのアップロード`に移動
4. Zipファイルをアップロードし、プラグインを有効化

### ステップ 2: WooCommerce POSでの有効化

1. `WP管理 > WooCommerce POS > 設定 > チェックアウト`に移動
2. リストの中から**WooCommerce POS請求書支払いゲートウェイ**を探す
3. POSで使用するためにゲートウェイを有効化
4. 設定を保存

:::note
このゲートウェイは追加のAPIキーや外部サービスの設定を必要としません。既存のWordPressメールシステムとWooCommerceの支払い方法を使用します。
:::

## 使用方法

### 請求書の送信

1. **アイテムを追加**: POSでショッピングカートに商品を追加
2. **ゲートウェイを選択**: 支払い方法として「メール請求書」を選択
3. **メールアドレス**: 
   - 既存の顧客の場合: メールは顧客データから自動的に入力されます
   - ゲスト注文の場合: 顧客のメールアドレスを手動で入力
4. **オプション**: 「請求先住所に保存」チェックボックスをオンにして、顧客の請求情報を更新
5. **請求書を送信**: 請求書メールを送信するためにプロセスを完了

### 顧客体験

請求書を送信すると、顧客は以下を含むメールを受け取ります：

- **注文詳細**: アイテム、数量、価格の完全な内訳
- **注文合計**: 税金と手数料を含む最終的な金額
- **支払いリンク**: オンラインで支払いを完了するための直接リンク
- **注文情報**: 注文番号と参照詳細

### 支払い完了

顧客は2つの方法で支払いを完了できます：

1. **オンライン支払い**: メール内の「この注文の支払い」リンクをクリック
2. **店舗での支払い**: 他のPOS支払い方法を使用して支払いを完了するために戻る

「この注文の支払い」リンクは、顧客が利用できるウェブ支払いゲートウェイ（Stripe、PayPalなど）を使用して支払いを完了できる、あなたのウェブサイトの特定の支払いページに顧客を案内します。

## メール管理

### 既存の顧客

- メールアドレスは顧客プロファイルから自動的に取得
- 顧客の請求情報は保持されます
- 注文履歴は顧客アカウントの下に維持されます

### ゲスト顧客

- 一時的な顧客のためにメールアドレスを手動で入力
- 将来の参照のために請求先住所にメールを保存するオプション
- ゲスト注文はWooCommerceで適切に追跡されます

### メールテンプレート

このゲートウェイはWooCommerceの標準メールテンプレートを使用します：

- **カスタマイズ可能**: WooCommerceのメール設定を通じてメールの外観を変更
- **ブランディング**: ストアのロゴとブランディングを含める
- **プロフェッショナル**: クリーンで専門的な請求書形式
- **モバイルフレンドリー**: すべてのデバイスに対応したレスポンシブデザイン

## 使用例

### 完璧な用途

- **B2B販売**: 会計システムを通じて支払いを処理する必要があるビジネス顧客
- **大口注文**: 承認または処理時間が必要な高額取引
- **リモート顧客**: 店を離れた後に支払いを完了する必要がある顧客
- **口座顧客**: 定期的に支払い条件が確立された顧客
- **見積もりから注文**: 見積もりや概算を支払い可能な請求書に変換

### ワークフローの例

**小売店**
1. 顧客がアイテムを選択しますが、購入の承認を得る必要があります
2. すべての詳細を含む請求書メールを送信
3. 顧客が準備ができ次第、オンラインで支払いを完了
4. 注文が自動的にシステム内で更新されます

**サービス業**
1. サービスを完了し、アイテムをPOSに追加
2. 顧客の会計部門に請求書を送信
3. 顧客が希望の支払い方法でオンラインで支払い
4. 支払い確認を受け取り、注文を完了

## 要件

- **WooCommerce POS**: 無料版またはプロ版がインストールされ、有効化されていること
- **メールシステム**: 稼働するWordPressメール構成
- **ウェブ支払いゲートウェイ**: WooCommerceで少なくとも1つのオンライン支払い方法が有効化されていること
- **SSL証明書**: 安全な支払い処理のためにHTTPSが必要

## トラブルシューティング

### よくある問題

**メールが送信されない**
- WordPressのメール構成を確認
- カスタムメールサービスを使用している場合はSMTP設定を確認
- WordPressメールテストプラグインでテスト
- テストメールがスパムフォルダにないか確認

**支払いリンクが機能しない**
- WooCommerceが適切に構成されていることを確認
- ウェブチェックアウト用に少なくとも1つの支払いゲートウェイが有効化されていることを確認
- SSL証明書が適切にインストールされていることを確認
- 注文ステータスが支払いを許可していることを確認

**顧客が支払いを完了できない**
- ウェブ支払いゲートウェイがアクティブで構成されていることを確認
- 注文が期限切れまたはキャンセルされていないことを確認
- 顧客が十分な支払い方法の上限を持っていることを確認
- 自分で支払いプロセスをテスト

### メール配信

信頼性の高いメール配信のために考慮すること：

- **SMTPサービス**: SendGrid、Mailgun、Amazon SESなどのサービスを使用
- **メールプラグイン**: より良い配信のためにWordPress SMTPプラグインをインストール
- **ドメイン認証**: SPF、DKIM、およびDMARCレコードを設定
- **監視**: メールの配信と開封率を追跡

### ヘルプの取得

技術サポートについて：

- WordPressメールテストツールでメール機能をテスト
