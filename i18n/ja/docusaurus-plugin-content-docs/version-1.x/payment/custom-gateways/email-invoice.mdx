# メール請求書ゲートウェイ {#email-invoice-gateway}

メール請求書ゲートウェイを使用すると、WooCommerce POSから顧客にメールで支払い請求書を直接送信できます。これは、顧客が後で支払いたい場合や、Webストアの支払い方法を使用してオンラインで支払いを完了する必要がある場合に最適です。

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [メール請求書ゲートウェイのリリースページ](https://github.com/wcpos/email-invoice-gateway/releases)にアクセス
2. 最新の **woocommerce-pos-email-invoice-gateway.zip** ファイルをダウンロード
3. WordPress管理画面で `プラグイン > 新規追加 > プラグインのアップロード` に進む
4. zipファイルをアップロードし、プラグインを有効化

  </StepCard>
  <StepCard number="2" title="WooCommerce POSで有効化" isLast>

1. `WP管理 > WooCommerce POS > 設定 > チェックアウト` に進む
2. リストから **WooCommerce POS 請求書支払いゲートウェイ** を見つける
3. POSで使用するためにゲートウェイを有効化
4. 設定を保存

:::note
このゲートウェイは追加のAPIキーや外部サービスの構成を必要としません。既存のWordPressメールシステムとWooCommerceの支払い方法を使用します。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### 請求書の送信 {#sending-invoices}

1. **アイテムを追加**: POSでカートに商品を追加
2. **ゲートウェイを選択**: 支払い方法として「メール請求書」を選択
3. **メールアドレス**: 
   - 既存の顧客の場合: 顧客データからメールが自動的に入力されます
   - ゲスト注文の場合: 顧客のメールアドレスを手動で入力
4. **オプション**: 「請求先住所に保存」をチェックして顧客の請求情報を更新
5. **請求書を送信**: 請求書メールを送信するためにプロセスを完了

### 顧客体験 {#customer-experience}

請求書を送信すると、顧客は以下を含むメールを受け取ります：

- **注文の詳細**: アイテム、数量、および価格の完全な内訳
- **注文合計**: 税金と手数料を含む最終金額
- **支払いリンク**: オンラインで支払いを完了するための直接リンク
- **注文情報**: 注文番号と参照詳細

### 支払い完了 {#payment-completion}

顧客は以下の2つの方法で支払いを完了できます：

1. **オンライン支払い**: メール内の「この注文を支払う」リンクをクリック
2. **店内での返却**: 他のPOS支払い方法を使用して支払いを完了するために戻る

「この注文を支払う」リンクは、顧客をあなたのウェブサイトの専用支払いページに導き、そこでは有効なウェブ支払いゲートウェイ（Stripe、PayPalなど）を使用できます。

## メール管理 {#email-management}

### 既存の顧客 {#existing-customers}

- メールアドレスは顧客プロフィールから自動的に取得
- 顧客の請求情報は保持される
- 注文履歴は顧客アカウントの下に維持される

### ゲスト顧客 {#guest-customers}

- 一度きりの顧客のメールアドレスを手動で入力
- 将来の参考のために請求先住所にメールを保存するオプション
- ゲスト注文はWooCommerceで適切に追跡される

### メールテンプレート {#email-templates}

このゲートウェイはWooCommerceの標準メールテンプレートを使用します：

- **カスタマイズ可能**: WooCommerceのメール設定を通じてメールの外観を変更
- **ブランド化**: ストアのロゴとブランドを含む
- **プロフェッショナル**: クリーンでプロフェッショナルな請求書フォーマット
- **モバイル対応**: すべてのデバイスに対応したレスポンシブデザイン

## ユースケース {#use-cases}

### 適しています {#perfect-for}

- **B2Bセールス**: 会計システムを通じて支払いを処理する必要があるビジネス顧客
- **大口注文**: 承認または処理時間が必要な高額取引
- **リモート顧客**: 店を離れた後に支払いを完了する必要がある顧客
- **アカウント顧客**: 確立された支払い条件を持つ定期的な顧客
- **見積もりから注文**: 見積もりや見積もりを支払可能な請求書に変換

## トラブルシューティング {#troubleshooting}

### 一般的な問題 {#common-issues}

<Accordion>
  <AccordionItem question="メールが送信されない">

  - WordPressメール構成を確認
  - カスタムメールサービスを使用している場合はSMTP設定を確認
  - WordPressメールテストプラグインを使ってテスト
  - テストメールがスパムフォルダにないか確認

  </AccordionItem>

  <AccordionItem question="支払いリンクが機能しない">

  - WooCommerceが適切に設定されているか確認
  - ウェブチェックアウト用に少なくとも1つの支払いゲートウェイが有効であることを確認
  - SSL証明書が正しくインストールされているか確認
  - 注文ステータスが支払いを許可していることを確認

  </AccordionItem>

  <AccordionItem question="顧客が支払いを完了できない">

  - ウェブ支払いゲートウェイがアクティブで設定されているか確認
  - 注文が期限切れになっていないこと、またはキャンセルされていないことを確認
  - 顧客の支払い方法の限度額が十分であることを確認
  - 自分で支払いプロセスをテスト

  </AccordionItem>
</Accordion>

### メール配信 {#email-delivery}

信頼性のあるメール配信のために、考慮すべき点：

- **SMTPサービス**: SendGrid、Mailgun、またはAmazon SESなどのサービスを使用
- **メールプラグイン**: より良い配信のためにWordPress SMTPプラグインをインストール
- **ドメイン認証**: SPF、DKIM、DMARCレコードを設定
- **監視**: メール配信とオープン率を追跡

### ヘルプを得る {#getting-help}

技術的サポートのために：
- 問題を報告するために[GitHubリポジトリ](https://github.com/wcpos/email-invoice-gateway)を訪問
- テンプレートカスタマイズのためにWooCommerceのメールドキュメントを確認
- WordPressメールテストツールでメール機能をテスト
