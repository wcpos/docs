# ウェブチェックアウトゲートウェイ {#web-checkout-gateway}

ウェブチェックアウトゲートウェイは、顧客があなたのウェブストアのチェックアウトシステムを使用してPOS取引を完了できるようにします。このゲートウェイは、店内のPOSとオンライン支払い方法の間にシームレスな橋を作成し、顧客が自分のデバイスを使用して支払うことを好む状況や、ウェブ専用の支払い方法にアクセスする必要があるときに最適です。

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [ウェブチェックアウトゲートウェイのリリースページ](https://github.com/wcpos/web-checkout-gateway/releases)にアクセスします
2. 最新の**woocommerce-pos-web-checkout-gateway.zip**ファイルをダウンロードします
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインのアップロード`に移動します
4. zipファイルをアップロードし、プラグインを有効化します

  </StepCard>
  <StepCard number="2" title="WooCommerce POSでの有効化" isLast>

1. `WP管理 > WooCommerce POS > 設定 > チェックアウト`に移動します
2. リストから**WooCommerce POSウェブチェックアウトゲートウェイ**を見つけます
3. POSで使用するためにゲートウェイを有効化します
4. 設定を保存します

:::note
このゲートウェイは、既存のWooCommerce支払い方法を活用するため、少なくとも1つのウェブ支払いゲートウェイが正しく設定されていることを確認してください（Stripe、PayPalなど）。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### ウェブチェックアウトの開始 {#initiating-web-checkout}

1. **アイテムを追加**: POSでカートに商品を追加します
2. **ゲートウェイを選択**: 支払い方法として「ウェブチェックアウト」を選びます
3. **リンクを生成**: システムが注文のためのユニークなチェックアウトリンクを作成します
4. **顧客のアクセス**: 顧客はリンクをクリックしてウェブチェックアウトにアクセスします
5. **オンライン支払い**: 顧客はウェブストアの支払い方法を使用して支払いを完了します

### 支払いプロセス {#payment-process}

ウェブチェックアウトゲートウェイはこのワークフローに従います：

1. **注文の作成**: POSがWooCommerceにペンディング注文を作成します
2. **チェックアウトリンク**: システムが安全でユニークな支払いリンクを生成します
3. **顧客のリダイレクト**: 顧客がウェブチェックアウトページにリダイレクトされます
4. **支払い方法の選択**: 顧客が利用可能なウェブ支払い方法から選択します
5. **支払い処理**: 標準のWooCommerceチェックアウトプロセスが支払いを処理します
6. **注文の完了**: 支払いが成功すると自動的に注文ステータスが更新されます

### POSワークフロー {#pos-workflow}

ウェブチェックアウトを開始した後：

1. **ステータスを監視**: 支払いステータスを監視するためにPOS注文画面を開いたままにします
2. **顧客の支払い**: 顧客が自分のデバイスまたはあなたのタブレットで支払いを完了します
3. **自動更新**: 支払いが受け取られるとリアルタイムで注文ステータスが更新されます
4. **支払いを処理**: POSで「支払いを処理」ボタンをクリックしてレシートに進みます
5. **レシート生成**: 通常通りレシートを生成して印刷します

## ユースケース {#use-cases}

### 最適なシチュエーション {#perfect-for}

- **顧客の好み**: 自分の支払いアプリやカードを使用することを好む顧客
- **複雑な支払い**: POSで利用できない支払い方法を必要とする取引
- **分割支払い**: 複数の支払い方法を使用したい顧客
- **ロイヤリティプログラム**: ウェブベースのロイヤリティポイント交換のアクセス
- **ギフトカード**: デジタルギフトカードやストアクレジットを持つ顧客
- **国際カード**: ウェブゲートウェイ経由でより良く機能する支払い方法

## 技術的詳細 {#technical-details}

### 注文管理 {#order-management}

- **ペンディング注文**: 注文は「支払い待ち」のステータスで作成される
- **ステータス更新**: 支払いを受け取ると自動的にステータスが変更される
- **注文メモ**: 支払い詳細は注文メモに記録される
- **在庫**: チェックアウトプロセス中に在庫が確保される

### セキュリティ {#security}

- **ユニークリンク**: 各チェックアウトセッションにはユニークで時間制限のあるURLがあります
- **SSL必須**: すべての支払い処理は安全なHTTPS接続で行われる
- **PCI準拠**: 既存のPCI準拠のウェブ支払いゲートウェイを利用する
- **セッション管理**: 安全なセッション処理により不正アクセスを防止

### 支払い方法 {#payment-methods}

ゲートウェイは、あなたの設定済みのWooCommerce支払い方法すべてにアクセスを提供します：

- **クレジット/デビットカード**: Stripe、Square、Authorize.netなど
- **デジタルウォレット**: PayPal、Apple Pay、Google Pay
- **銀行振込**: 直接銀行支払い方法
- **今買って後で払う**: Klarna、Afterpay、Sezzle
- **暗号通貨**: ビットコインおよびその他の暗号支払いゲートウェイ
- **ローカル方法**: 地域特有の支払い方法

## トラブルシューティング {#troubleshooting}

### 一般的な問題 {#common-issues}

<Accordion>
  <AccordionItem question="チェックアウトリンクが機能しない">

  - SSL証明書が正しくインストールされているか確認します
  - WooCommerceのパーマリンクが正しく設定されているかチェックします
  - ウェブ支払いゲートウェイがアクティブで設定されているか確認します
  - 別のブラウザやデバイスでテストします

  </AccordionItem>

  <AccordionItem question="支払いが処理されない">

  - ウェブ支払いゲートウェイが正しく設定されているか確認します
  - 支払いゲートウェイのログでエラーメッセージを確認します
  - 顧客の支払い方法がサポートされているか確認します
  - ウェブチェックアウトプロセスを独立してテストします

  </AccordionItem>

  <AccordionItem question="POSで注文が更新されない">

  - インターネット接続の安定性を確認します
  - WordPressのCronジョブが正しく実行されているか検証します
  - POS注文画面を更新します
  - WooCommerce管理で注文ステータスを確認します

  </AccordionItem>

  <AccordionItem question="顧客がチェックアウトにアクセスできない">

  - チェックアウトリンクが期限切れでないか確認します
  - 顧客のデバイスにインターネット接続があるか確認します
  - 注文がキャンセルまたは完了していないか確認します
  - 別のデバイスでリンクをテストします

  </AccordionItem>
</Accordion>

### パフォーマンスの最適化 {#performance-optimisation}

最適なパフォーマンスを得るために：

- **キャッシング**: キャッシングプラグインを設定してチェックアウトページを除外します
- **CDN**: CDNの設定がチェックアウトプロセスと干渉しないようにします
- **データベース**: 注文処理を高速化するためにデータベースを最適化します
- **ホスティング**: 良好な稼働率の信頼できるホスティングを使用します
