# ウェブチェックアウトゲートウェイ

ウェブチェックアウトゲートウェイにより、顧客は店舗のPOS取引をウェブストアのチェックアウトシステムを使用して完了できます。このゲートウェイは、店舗のPOSとオンライン支払い方法との間にシームレスな橋を作り、顧客が自身のデバイスを使用して支払うことを好む状況や、ウェブ専用の支払い方法にアクセスする必要がある場合に最適です。

## 機能

- **シームレスな統合**: POSからウェブチェックアウトへの直接リンク
- **すべての支払い方法**: ウェブストアで有効なすべての支払いゲートウェイにアクセス
- **顧客のコントロール**: 顧客は自分のデバイスと好みの方法で支払うことができます
- **注文の同期**: POSとウェブストア間での自動注文更新
- **柔軟なワークフロー**: POSのワークフローを維持しながらオンラインで取引を完了

## インストール

### ステップ1: ダウンロードとインストール

1. [ウェブチェックアウトゲートウェイのリリースページ](https://github.com/wcpos/web-checkout-gateway/releases)にアクセス
2. 最新の **woocommerce-pos-web-checkout-gateway.zip** ファイルをダウンロード
3. WordPress管理画面で `プラグイン > 新規追加 > プラグインのアップロード` に進む
4. Zipファイルをアップロードし、プラグインをアクティベート

### ステップ2: WooCommerce POSで有効化

1. `WP Admin > WooCommerce POS > 設定 > チェックアウト` に行く
2. リスト内で **WooCommerce POS Web Checkout Gateway** を見つける
3. POSで使用するためにゲートウェイを有効化
4. 設定を保存

:::note
このゲートウェイは既存のWooCommerce支払い方法を活用するため、少なくとも1つのウェブ支払いゲートウェイが適切に設定されていることを確認してください（Stripe、PayPalなど）。
:::

## 使用方法

### ウェブチェックアウトの開始

1. **アイテムを追加**: POSで製品をカートに追加
2. **ゲートウェイを選択**: 支払い方法として「ウェブチェックアウト」を選択
3. **リンクを生成**: システムが注文用のユニークなチェックアウトリンクを作成
4. **顧客アクセス**: 顧客がリンクをクリックしてウェブチェックアウトにアクセス
5. **オンライン支払い**: 顧客がウェブストアの支払い方法を使用して支払いを完了

### 支払いプロセス

ウェブチェックアウトゲートウェイは次のワークフローに従います：

1. **注文作成**: POSがWooCommerceに保留中の注文を作成
2. **チェックアウトリンク**: システムが安全でユニークな支払いリンクを生成
3. **顧客リダイレクト**: 顧客がウェブチェックアウトページにリダイレクトされる
4. **支払い選択**: 顧客が利用可能なウェブ支払い方法から選択
5. **支払い処理**: 標準のWooCommerceチェックアウトプロセスが支払いを処理
6. **注文完了**: 支払いが成功すると自動で注文ステータスが更新

### POSワークフロー

ウェブチェックアウトを開始した後：

1. **ステータスを監視**: 支払いステータスを監視するためにPOSの注文画面を開いたままにします
2. **顧客支払い**: 顧客が自分のデバイスまたはあなたのタブレットで支払いを完了
3. **自動更新**: 支払いが受信されると注文ステータスがリアルタイムで更新
4. **支払い処理**: 領収書に進むためにPOSの「支払い処理」ボタンをクリック
5. **領収書生成**: 通常通り領収書を生成して印刷

## 使用ケース

### 完璧な用途

- **顧客の好み**: 自分の支払いアプリやカードを使いたい顧客
- **複雑な支払い**: POSでは利用できない支払い方法が必要な取引
- **分割支払い**: 複数の支払い方法を使用したい顧客
- **ロイヤルティプログラム**: ウェブベースのロイヤルティポイントの引き換え
- **ギフトカード**: デジタルギフトカードまたはストアクレジットを持つ顧客
- **国際カード**: ウェブゲートウェイを通じてより適した支払い方法

### ワークフロー例

**小売店**
1. 顧客が店内で買い物をし、チェックアウトに商品を持ってくる
2. 顧客がモバイルウォレットアプリで支払いを希望
3. スタッフがウェブチェックアウトゲートウェイを選択
4. 顧客がQRコードをスキャンするか、電話のリンクをクリック
5. 顧客が好みの方法で支払いを完了
6. スタッフが領収書を処理し、取引を完了

**サービスビジネス**
1. サービスを完了し、POSに料金を追加
2. 顧客がビジネスクレジットカードで支払いたい
3. ウェブチェックアウトリンクを生成
4. 顧客が安全なウェブフォームで支払い詳細を入力
5. 支払いがあなたのウェブゲートウェイを通じて処理
6. 注文が完了し、領収書が生成される

## 技術的詳細

### 注文管理

- **保留中の注文**: 支払い待ちのステータスで注文が作成される
- **ステータス更新**: 支払いが受信されると自動でステータス変更
- **注文メモ**: 支払い詳細が注文メモに記録される
- **在庫**: チェックアウトプロセス中に在庫が予約される

### セキュリティ

- **ユニークリンク**: 各チェックアウトセッションにはユニークで時間制限のあるURLがある
- **SSL必須**: すべての支払い処理は安全なHTTPS接続を介して行われる
- **PCI準拠**: 既存のPCI準拠のウェブ支払いゲートウェイを活用
- **セッション管理**: セキュアなセッション管理により、不正アクセスを防止

### 支払い方法

このゲートウェイではすべての構成済みのWooCommerce支払い方法へのアクセスを提供します：

- **クレジット/デビットカード**: Stripe、Square、Authorize.netなど
- **デジタルウォレット**: PayPal、Apple Pay、Google Pay
- **銀行振込**: 直接銀行支払い方法
- **今すぐ購入、後で支払い**: Klarna、Afterpay、Sezzle
- **暗号通貨**: Bitcoinおよび他の暗号支払いゲートウェイ
- **地域特有の方法**: 地域特有の支払い方法

## 要件

- **WooCommerce POS**: 無料版またはプロ版がインストールおよびアクティベートされていること
- **ウェブ支払いゲートウェイ**: WooCommerceで少なくとも1つのオンライン支払い方法が構成されていること
- **SSL証明書**: 安全な支払い処理にはHTTPSが必要
- **最新のブラウザ**: 顧客のデバイスは最新のWeb標準をサポートする必要があります

## トラブルシューティング

### 一般的な問題

**チェックアウトリンクが機能しない**
- SSL証明書が正しくインストールされていることを確認
- WooCommerceのパーマリンクが正しく設定されているか確認
- ウェブ支払いゲートウェイがアクティブで構成されていることを確認
- 別のブラウザやデバイスでテスト

**支払いが処理されない**
- ウェブ支払いゲートウェイが正しく設定されていることを確認
- 支払いゲートウェイのログでエラーメッセージをチェック
- 顧客の支払い方法がサポートされているか確認
- ウェブチェックアウトプロセスを独立してテスト

**注文がPOSで更新されない**
- インターネット接続の安定性を確認
- WordPressのcronジョブが正しく実行されていることを確認
- POSの注文画面をリフレッシュ
- WooCommerce管理で注文ステータスを確認

**顧客がチェックアウトにアクセスできない**
- チェックアウトリンクが期限切れになっていないことを確認
- 顧客のデバイスにインターネット接続があることを確認
- 注文がキャンセルまたは完了していないことを確認
- 別のデバイスでリンクをテスト

### パフォーマンス最適化

最適なパフォーマンスのために：

- **キャッシュ**: チェックアウトページを除外するようにキャッシュプラグインを設定
- **CDN**: CDN設定がチェックアウトプロセスに干渉しないことを確認
- **データベース**: より速い注文処理のためにデータベースを最適化
- **ホスティング**: 信頼できるホスティングを使用し、稼働時間が良好であること

### ヘルプを受ける

技術サポートについて：
- 問題を報告するために[GitHubリポジトリ](https://github.com/wcpos/web-checkout-gateway)に訪れる
- WooCommerceの支払いゲートウェイのドキュメントを確認
- 定期的にウェブチェックアウトプロセスをテスト
- 問題のために支払いゲートウェイのログを監視
