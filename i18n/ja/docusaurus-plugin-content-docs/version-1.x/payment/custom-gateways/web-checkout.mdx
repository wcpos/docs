# ウェブチェックアウトゲートウェイ

ウェブチェックアウトゲートウェイは、顧客が店舗のPOSトランザクションをウエブストアのチェックアウトシステムを使用して完了できるようにします。このゲートウェイは、店舗のPOSとオンラインの支払い方法との間にシームレスなブリッジを作成し、顧客が自分のデバイスを使用して支払うことを好む場合や、ウェブ専用の支払い方法にアクセスする必要がある場合に最適です。

## インストール

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [ウェブチェックアウトゲートウェイのリリースページ](https://github.com/wcpos/web-checkout-gateway/releases)にアクセスします
2. 最新の **woocommerce-pos-web-checkout-gateway.zip** ファイルをダウンロードします
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインをアップロード` に移動します
4. zipファイルをアップロードし、プラグインを有効化します

  </StepCard>
  <StepCard number="2" title="WooCommerce POSで有効にする" isLast>

1. `WP管理 > WooCommerce POS > 設定 > チェックアウト` に移動します
2. リストで **WooCommerce POSウェブチェックアウトゲートウェイ** を見つけます
3. POSでの使用のためにゲートウェイを有効にします
4. 設定を保存します

:::note
このゲートウェイは、既存のWooCommerce支払い方法を利用するため、少なくとも1つのウェブ支払いゲートウェイが正しく設定されていることを確認してください（Stripe、PayPalなど）。
:::

  </StepCard>
</Steps>

## 使用方法

### ウェブチェックアウトの開始

1. **アイテム追加**: POSに製品をカートに追加します
2. **ゲートウェイ選択**: 支払い方法として「ウェブチェックアウト」を選択します
3. **リンク生成**: システムが注文のためのユニークなチェックアウトリンクを生成します
4. **顧客アクセス**: 顧客はリンクをクリックしてウェブチェックアウトにアクセスします
5. **オンライン支払い**: 顧客はウェブストアの支払い方法を使用して支払いを完了します

### 支払いプロセス

ウェブチェックアウトゲートウェイはこのワークフローに従います：

1. **注文の作成**: POSがWooCommerceに保留中の注文を作成します
2. **チェックアウトリンク**: システムが安全でユニークな支払いリンクを生成します
3. **顧客リダイレクト**: 顧客はウェブチェックアウトページにリダイレクトされます
4. **支払い選択**: 顧客は利用可能なウェブ支払い方法から選択します
5. **支払い処理**: 標準のWooCommerceチェックアウトプロセスが支払いを処理します
6. **注文完了**: 支払いが成功すると、注文のステータスが自動的に更新されます

### POSワークフロー

ウェブチェックアウトを開始した後：

1. **ステータスを監視**: 支払い状況を監視するため、POSの注文画面を開いたままにします
2. **顧客支払い**: 顧客は自分のデバイスまたはあなたのタブレットで支払いを完了します
3. **自動更新**: 支払いが受信されると、注文のステータスがリアルタイムで更新されます
4. **支払い処理**: POSで「支払い処理」ボタンをクリックして、レシートに進みます
5. **レシート生成**: 通常通りにレシートを生成し印刷します

## 使用例

### 完璧な対象

- **顧客の好み**: 自分の支払いアプリやカードを使用したい顧客
- **複雑な支払い**: POSで利用できない支払い方法を必要とするトランザクション
- **分割支払い**: 複数の支払い方法を使用したい顧客
- **ロイヤルティプログラム**: ウェブベースのロイヤルティポイントの引き換えへのアクセス
- **ギフトカード**: デジタルギフトカードやストアクレジットを持つ顧客
- **国際カード**: ウェブゲートウェイを通じてうまく機能する支払い方法

## 技術的詳細

### 注文管理

- **保留中の注文**: 注文は「支払い待ち」ステータスで作成されます
- **ステータス更新**: 支払いが受信されると、ステータスが自動的に変更されます
- **注文ノート**: 支払い詳細が注文ノートに記録されます
- **在庫**: チェックアウトプロセス中にストックが予約されます

### セキュリティ

- **ユニークリンク**: 各チェックアウトセッションにはユニークで時間制限のあるURLがあります
- **SSL必須**: すべての支払い処理は安全なHTTPS接続で行われます
- **PCI準拠**: 既存のPCI準拠のウェブ支払いゲートウェイを活用しています
- **セッション管理**: セキュアなセッション処理により、無許可のアクセスを防止します

### 支払い方法

このゲートウェイは、設定されているすべてのWooCommerce支払い方法にアクセスを提供します：

- **クレジット/デビットカード**: Stripe、Square、Authorize.net、他
- **デジタルウォレット**: PayPal、Apple Pay、Google Pay
- **銀行振込**: 銀行直接決済方法
- **今すぐ購入、後で支払い**: Klarna、Afterpay、Sezzle
- **暗号通貨**: Bitcoinおよび他の暗号支払いゲートウェイ
- **ローカルメソッド**: 地域特有の支払い方法

## トラブルシューティング

### よくある問題

<Accordion>
  <AccordionItem question="チェックアウトリンクが機能しない">

  - SSL証明書が正しくインストールされていることを確認してください
  - WooCommerceのパーマリンクが正しく設定されているか確認してください
  - ウェブ支払いゲートウェイがアクティブで設定されていることを確かめる
  - 別のブラウザやデバイスでテストします

  </AccordionItem>

  <AccordionItem question="支払いが処理されない">

  - ウェブ支払いゲートウェイが正しく設定されていることを確認してください
  - 支払いゲートウェイのログを確認してエラーメッセージを探してください
  - 顧客の支払い方法がサポートされているか確認してください
  - ウェブチェックアウトプロセスを独自にテストします

  </AccordionItem>

  <AccordionItem question="POSで注文が更新されない">

  - インターネット接続の安定性を確認してください
  - WordPressのcronジョブが正しく実行されているか確認してください
  - POSの注文画面をリフレッシュします
  - WooCommerce管理で注文のステータスを確認します

  </AccordionItem>

  <AccordionItem question="顧客がチェックアウトにアクセスできない">

  - チェックアウトリンクが期限切れでないことを確認します
  - 顧客のデバイスにインターネット接続があることを確認してください
  - 注文がキャンセルされたり完了していないことを確認してください
  - 別のデバイスでリンクをテストします

  </AccordionItem>
</Accordion>

### パフォーマンス最適化

最適なパフォーマンスのために：

- **キャッシュ**: キャッシュプラグインを設定してチェックアウトページを除外します
- **CDN**: CDN設定がチェックアウトプロセスに干渉しないことを確認します
- **データベース**: より迅速な注文処理のためにデータベースを最適化します
- **ホスティング**: 良好な稼働時間を持つ信頼できるホスティングを使用します

### ヘルプの受け取り

技術サポートのために：
- [GitHubリポジトリ](https://github.com/wcpos/web-checkout-gateway)にアクセスして問題を報告します
- WooCommerceの支払いゲートウェイのドキュメントを確認します
- 定期的にウェブチェックアウトプロセスをテストします
- 問題のために支払いゲートウェイのログを監視します
