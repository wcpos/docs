# ウェブチェックアウトゲートウェイ {#web-checkout-gateway}

ウェブチェックアウトゲートウェイは、顧客があなたのウェブストアのチェックアウトシステムを使用してPOSトランザクションを完了できるようにします。このゲートウェイは、店舗のPOSとオンラインの支払い方法の間のシームレスなブリッジを作成し、顧客が自分のデバイスを使用して支払うことを好む場合や、ウェブ専用の支払い方法にアクセスが必要な場合に最適です。

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [ウェブチェックアウトゲートウェイのリリースページ](https://github.com/wcpos/web-checkout-gateway/releases)へアクセスします
2. 最新の **woocommerce-pos-web-checkout-gateway.zip** ファイルをダウンロードします
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインをアップロード` に移動します
4. zipファイルをアップロードして、プラグインを有効化します

  </StepCard>
  <StepCard number="2" title="WooCommerce POSで有効にする" isLast>

1. `WP管理 > WooCommerce POS > 設定 > チェックアウト` に移動します
2. リストで **WooCommerce POSウェブチェックアウトゲートウェイ** を見つけます
3. POSで使用するためにゲートウェイを有効化します
4. 設定を保存します

:::note
このゲートウェイは既存のWooCommerceの支払い方法を使用するため、少なくとも1つのウェブ支払いゲートウェイが適切に設定されていることを確実にしてください（例：Stripe、PayPalなど）。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### ウェブチェックアウトの開始 {#initiating-web-checkout}

1. **アイテム追加**: POSのカートに製品を追加します
2. **ゲートウェイ選択**: 支払い方法として「ウェブチェックアウト」を選択します
3. **リンク生成**: システムが注文用のユニークなチェックアウトリンクを生成します
4. **顧客アクセス**: 顧客がリンクをクリックしてウェブチェックアウトにアクセスします
5. **オンライン支払い**: 顧客はウェブストアの支払い方法を利用して支払いを完了します

### 支払いプロセス {#payment-process}

ウェブチェックアウトゲートウェイは次のワークフローに従います：

1. **注文の作成**: POSがWooCommerceで「支払い待ち」のステータスで注文を作成します
2. **チェックアウトリンク**: システムが安全でユニークな支払いリンクを生成します
3. **顧客リダイレクト**: 顧客はウェブチェックアウトページにリダイレクトされます
4. **支払い選択**: 顧客が利用可能なウェブの支払い方法から選択します
5. **支払い処理**: 標準のWooCommerceチェックアウトプロセスが支払いを処理します
6. **注文完了**: 支払いが成功すると、注文のステータスが自動的に更新されます

### POSワークフロー {#pos-workflow}

ウェブチェックアウトを開始後：

1. **ステータスを監視**: 支払い状況を確認するため、POSの注文画面を開いたままにします
2. **顧客支払い**: 顧客は自分のデバイスまたはタブレットで支払いを完了します
3. **自動更新**: 支払いが受信されると、注文のステータスがリアルタイムで更新されます
4. **支払い処理**: POSで「支払い処理」ボタンをクリックして、レシート処理に進みます
5. **レシート生成**: 通常通りにレシートを生成して印刷します

## 使用例 {#use-cases}

### 完璧な対象 {#perfect-for}

- **顧客の好み**: 自分の支払いアプリやカードを使用することを好む顧客
- **複雑な支払い**: POSで利用できない支払い方法が必要なトランザクション
- **分割支払い**: 複数の支払い方法を使いたい顧客
- **ロイヤルティプログラム**: ウェブベースのロイヤルティポイントの引き換えにアクセス
- **ギフトカード**: デジタルギフトカードやストアクレジットを持つ顧客
- **国際カード**: ウェブゲートウェイ経由でより良く機能する支払い方法

## 技術的詳細 {#technical-details}

### 注文管理 {#order-management}

- **保留中の注文**: 注文は「支払い待ち」ステータスで作成されます
- **ステータス更新**: 支払いが受信されると、ステータスが自動的に変更されます
- **注文ノート**: 支払いの詳細が注文ノートに記録されます
- **在庫**: チェックアウトプロセス中にストックが予約されます

### セキュリティ {#security}

- **ユニークリンク**: 各チェックアウトセッションにはユニークで時間制限のあるURLがあります
- **SSL必須**: すべての支払い処理は安全なHTTPS接続で行われます
- **PCI準拠**: 既存のPCI準拠のウェブ支払いゲートウェイを活用しています
- **セッション管理**: セキュリティ的なセッション管理により、無許可のアクセスを防ぎます

### 支払い方法 {#payment-methods}

このゲートウェイは、設定されているすべてのWooCommerce支払い方法にアクセスを提供します：

- **クレジット/デビットカード**: Stripe、Square、Authorize.net、他
- **デジタルウォレット**: PayPal、Apple Pay、Google Pay
- **銀行振込**: 銀行直接決済方法
- **今すぐ購入、後で支払い**: Klarna、Afterpay、Sezzle
- **暗号通貨**: Bitcoinおよび他の暗号通貨の支払いゲートウェイ
- **ローカルメソッド**: 地域特有の支払い方法

## トラブルシューティング {#troubleshooting}

### よくある問題 {#common-issues}

<Accordion>
  <AccordionItem question="チェックアウトリンクが機能しない">

  - SSL証明書が正しくインストールされていることを確認してください
  - WooCommerceのパーマリンクが正しく設定されていることを確認してください
  - ウェブ支払いゲートウェイがアクティブで正しく設定されていることを確認してください
  - 別のブラウザやデバイスでテストします

  </AccordionItem>

  <AccordionItem question="支払いが処理されない">

  - ウェブ支払いゲートウェイが正しく設定されていることを確認してください
  - 支払いゲートウェイのログを確認してエラーメッセージをチェックします
  - 顧客の支払い方法がサポートされていることを確認してください
  - ウェブチェックアウトプロセスを個別にテストします

  </AccordionItem>

  <AccordionItem question="POSで注文が更新されない">

  - インターネット接続の安定性を確認してください
  - WordPressのcronジョブが正しく実行されていることを確認してください
  - POSの注文画面をリフレッシュします
  - WooCommerce管理で注文のステータスを確認します

  </AccordionItem>

  <AccordionItem question="顧客がチェックアウトにアクセスできない">

  - チェックアウトリンクが期限切れでないことを確認します
  - 顧客のデバイスにインターネット接続があることを確認してください
  - 注文がキャンセルされたり完了していないことを確認してください
  - 別のデバイスでリンクをテストします

  </AccordionItem>
</Accordion>

### パフォーマンス最適化 {#performance-optimisation}

最適なパフォーマンスを維持するために：

- **キャッシュ**: キャッシュプラグインを設定してチェックアウトページを除外してください
- **CDN**: CDN設定がチェックアウトプロセスに干渉しないようにします
- **データベース**: より迅速な注文処理のためにデータベースを最適化します
- **ホスティング**: 良好な稼働時間を持つ信頼できるホスティングを選択します

### ヘルプを受ける {#getting-help}

技術サポートのために：
- [GitHubリポジトリ](https://github.com/wcpos/web-checkout-gateway)で問題を報告します
- WooCommerceの支払いゲートウェイのドキュメントを確認します
- 定期的にウェブチェックアウトプロセスをテストします
- 問題を特定するために支払いゲートウェイのログを監視します
