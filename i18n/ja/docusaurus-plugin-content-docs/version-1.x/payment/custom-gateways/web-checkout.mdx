# ウェブチェックアウトゲートウェイ

ウェブチェックアウトゲートウェイは、お客様がウェブストアのチェックアウトシステムを使用してPOSトランザクションを完了できるようにします。このゲートウェイは、店舗内POSとオンライン決済方法の間にシームレスな橋を作り、お客様が自分のデバイスで支払いを希望する場合や、ウェブ限定の決済方法にアクセスする必要がある状況に最適です。

## 機能

- **シームレスな統合**: POSからウェブチェックアウトへの直接リンク
- **全決済方法**: ウェブストアで有効な全決済ゲートウェイへのアクセス
- **顧客コントロール**: 顧客が自分のデバイスと希望の決済方法を使用可能
- **注文同期**: POSとウェブストア間の自動注文更新
- **柔軟なワークフロー**: POSワークフローを維持しながらオンラインで取引を完了

## インストール

### ステップ1：ダウンロードとインストール

1. [ウェブチェックアウトゲートウェイのリリースページ](https://github.com/wcpos/web-checkout-gateway/releases)を訪問
2. 最新の **woocommerce-pos-web-checkout-gateway.zip** ファイルをダウンロード
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインをアップロード`に移動
4. zipファイルをアップロードし、プラグインを有効化

### ステップ2：WooCommerce POSで有効化

1. `WP管理画面 > WooCommerce POS > 設定 > チェックアウト`に移動
2. リストから**WooCommerce POS ウェブチェックアウトゲートウェイ**を見つける
3. POSでゲートウェイを使用可能に設定
4. 設定を保存

:::note
このゲートウェイは既存のWooCommerceの決済方法を活用するため、少なくとも1つのウェブ決済ゲートウェイ（Stripe、PayPalなど）が適切に設定されていることを確認してください。
:::

## 使用方法

### ウェブチェックアウトの開始

1. **商品追加**: POSで商品をカートに追加
2. **ゲートウェイ選択**: 決済方法として「ウェブチェックアウト」を選択
3. **リンク生成**: システムが注文の固有のチェックアウトリンクを作成
4. **顧客アクセス**: 顧客がリンクをクリックしてウェブチェックアウトにアクセス
5. **オンライン決済**: 顧客がウェブストアの決済方法で支払いを完了

### 支払いプロセス

ウェブチェックアウトゲートウェイは、次のワークフローに従います：

1. **注文作成**: POSがWooCommerceに保留中の注文を作成
2. **チェックアウトリンク**: システムが安全で固有の支払いリンクを生成
3. **顧客リダイレクト**: 顧客がウェブチェックアウトページに誘導
4. **決済選択**: 顧客が利用可能なウェブ決済方法を選択
5. **決済処理**: 標準のWooCommerceチェックアウトプロセスが決済を処理
6. **注文完了**: 支払い成功時に注文ステータスが自動更新

### POSワークフロー

ウェブチェックアウト開始後：

1. **ステータス監視**: 注文画面を開いて支払いステータスを監視
2. **顧客支払い**: 顧客が自分のデバイスまたはタブレットで支払いを完了
3. **自動更新**: 支払い受領時にリアルタイムで注文ステータスが更新
4. **支払い処理**: POSの「支払い処理」ボタンをクリックしてレシートに進む
5. **レシート生成**: 通常通りレシートを生成して印刷
