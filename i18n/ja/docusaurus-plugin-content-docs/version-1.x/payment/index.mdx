WCPOS は、異なるビジネスニーズに対応するための複数の支払い方法をサポートしています。チェックアウトは iframe/webview を使用して WooCommerce Order Pay ページを読み込みます。つまり、WooCommerce で機能する支払いゲートウェイはすべて POS でも機能します。

## デフォルト支払い方法

無料版には 2 つの組み込み支払いゲートウェイが含まれています:

### 現金

- 支払い額を入力
- 自動おつり計算
- 追加の設定は不要です

### カード

- 外部カードターミナルで使用するため
- ターミナルで処理した後、支払いを完了としてマークするだけです
- 直接の統合は必要ありません

## 追加の支払いゲートウェイ（Pro）

:::info Pro Feature
追加の支払いゲートウェイには[WCPOS Pro](/getting-started/pro-license)が必要です。
:::

Pro では、POS チェックアウトで追加の WooCommerce 支払いゲートウェイを有効にできます：

- **Stripe Terminal** - Stripe カードリーダーとの直接統合
- **SumUp Terminal** - SumUp カードリーダーとの統合
- **Any WooCommerce Gateway** - ストアにインストールされているゲートウェイを有効にする

## ゲートウェイの有効化と無効化

支払いゲートウェイは WordPress admin で管理されます：

1. に移動 `WP Admin > POS > Settings > Checkout`
2. インストール済みのすべてのWooCommerceペイメントゲートウェイのリストが表示されます
3. 各ゲートウェイをオンラインストアとは独立してPOS用にオン/オフ切り替えできます
4. ドラッグして並べ替え—最初に有効なゲートウェイはチェックアウト時のデフォルトになります

つまり、ゲートウェイはウェブサイト上でアクティブでもPOSでは無効にできます。逆も同様です。詳細は[Checkout Settings](/settings/wp-admin/checkout)をご参照ください。

## 決済のテスト

新しいゲートウェイを本番環境に導入する前に:

1. **Cash gateway**—実際の取引なしでチェックアウトフローをテストするために常に利用できます。
2. **Stripe Terminal**—Stripeはテストモード用に[シミュレーションリーダー](https://docs.stripe.com/terminal/testing)を提供しています。まずStripeダッシュボードでテストモードを有効にしてください。
3. **Third-party gateways**—サンドボックス/テストモードの指示については、ゲートウェイ独自のドキュメントをご参照ください。ほとんどのWooCommerceゲートウェイはテストモード切り替え機能をサポートしています。

:::tip
新しく有効にされたゲートウェイがチェックアウト時に表示されない場合は、POSをリフレッシュしてください。WP Adminでのゲートウェイ変更は次ページロード時に反映されます。
:::

## 一般的な問題

### ゲートウェイがチェックアウトに表示されない

- POSでゲートウェイが有効になっていることを確認してください `WP Admin > POS > Settings > Checkout`。
- ゲートウェイがWooCommerceでも有効になっていることを確認してください `WP Admin > WooCommerce > Settings > Payments` — ゲートウェイがPOSに表示される前に、WooCommerceで有効である必要があります。
- 変更後にPOSをリフレッシュしてください。

### ゲートウェイが正しく表示されない

チェックアウトiframeはあなたのサイトのテーマスタイルを読み込むため、干渉することがあります。チェックアウトモーダルの**Checkout Settings**ボタンを使用してスタイルやスクリプトを選択的に無効にしてください。詳細は[Checkout Troubleshooting](/pos/checkout#checkout-settings-troubleshooting)を参照してください。

### 支払い処理が失敗する

- あなたのサイトのエラーログを確認してください(`WP Admin > POS > Support > Logs`) ゲートウェイ固有エラーについて。
- SSLサーティフィケートが有効であることを確認してください — ほとんどの支払いゲートウェイはHTTPSが必要です。
- ステージング環境またはローカル環境を使用している場合は、ゲートウェイがテスト/サンドボックスモードをサポートしていることを確認してください。
