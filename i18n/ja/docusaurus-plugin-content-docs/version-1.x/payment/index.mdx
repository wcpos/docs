WCPOSは、さまざまなビジネスニーズに対応するための複数の支払い方法をサポートしています。チェックアウトはiframe/webviewを使用してWooCommerceの注文支払いページをロードするため、WooCommerceで機能する任意の支払いゲートウェイがPOSで機能できます。

## デフォルト支払い方法

無料版には2つの組み込み支払いゲートウェイが含まれています：

### 現金

- 支払い金額を入力
- 自動おつり計算
- 追加の設定は不要です

### カード

- 外部カードターミナルで使用するため
- ターミナルで処理した後、支払いを完了済みとしてマークするだけです
- 直接統合は不要です

## 追加の支払いゲートウェイ（Pro）

:::info プロ機能
追加の支払いゲートウェイが必要です [WCPOS Pro](/getting-started/pro-license).
:::

Proを使用すると、POS チェックアウトで追加のWooCommerceペイメントゲートウェイを有効にできます：

- **Stripe Terminal** - Stripe カードリーダーとの直接統合
- **SumUp Terminal** - SumUp カードリーダーとの統合
- **Any WooCommerce Gateway** - ストアにインストールされているあらゆるゲートウェイを有効にする

## ゲートウェイの有効化と無効化

支払いゲートウェイはWordPress管理画面で管理されます:

1. に移動してください `WP Admin > POS > Settings > Checkout`
2. インストールされているすべてのWooCommerce支払いゲートウェイのリストが表示されます
3. 各ゲートウェイをオンラインストアとは独立してPOS用に個別にオン/オフできます
4. ドラッグして並べ替えます — 最初に有効化されたゲートウェイがチェックアウト時のデフォルトになります

つまり、ゲートウェイはウェブサイトでアクティブでもPOSで無効になる場合があります。また、その逆も同じです。 [チェックアウト設定](/settings/wp-admin/checkout) 詳細については

## 支払いのテスト

新しいゲートウェイを本番環境で運用する前に:

1. **現金ゲートウェイ** — 実際のトランザクションなしでチェックアウトフローをテストするために常に利用可能です。
2. **Stripe Terminal** — Stripe が提供する [シミュレートされたリーダー](https://docs.stripe.com/terminal/testing) テストモード用です。最初に Stripe ダッシュボードでテストモードを有効にしてください。
3. **サードパーティゲートウェイ** — ゲートウェイ独自のドキュメントでサンドボックス/テストモードの手順を確認してください。ほとんどの WooCommerce ゲートウェイはテストモード切り替えをサポートしています。

:::tip
新しく有効にされたゲートウェイがチェックアウト時に表示されない場合は、POS を更新してみてください。WP Admin でのゲートウェイの変更は次のページ読み込み時に反映されます。
:::

## よくある問題

### チェックアウト時にゲートウェイが表示されない

- POS でゲートウェイが有効になっていることを確認してください `WP Admin > POS > Settings > Checkout`.
- ゲートウェイが以下でも有効になっていることを確認してください `WP Admin > WooCommerce > Settings > Payments` — ゲートウェイは POS に表示される前に WooCommerce で有効にする必要があります。
- 変更を加えた後、POS をリフレッシュしてください。

### ゲートウェイが正しく表示されない

チェックアウト iframe はサイトのテーマスタイルを読み込み、干渉することがあります。チェックアウトモーダルの **Checkout Settings** ボタンを使用して、スタイルまたはスクリプトを選択的に無効にしてください。詳細は [チェックアウトのトラブルシューティング](/pos/checkout#checkout-settings-troubleshooting) を参照してください。

### 支払い処理に失敗する

- サイトのエラーログを確認してください（`WP Admin > POS > Support > Logs`）ゲートウェイ固有のエラーについて。
- SSL 証明書が有効であることを確認してください — ほとんどのペイメントゲートウェイは HTTPS を必要とします。
- ステージング環境またはローカル環境を使用している場合は、ゲートウェイがテスト/サンドボックスモードをサポートしていることを確認してください。
