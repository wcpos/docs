# Web チェックアウトゲートウェイ {#web-checkout-gateway}

Web チェックアウトゲートウェイを使用すると、顧客は Web ストアのチェックアウトシステムを使って POS 取引を完了できます。このゲートウェイは、店舗内の POS とオンライン決済方法をシームレスにつなぎます。顧客が自分のデバイスで支払いたい場合や、Web 専用の決済方法を利用する必要がある場合に最適です。

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [ウェブチェックアウトゲートウェイのリリースページ](https://github.com/wcpos/web-checkout-gateway/releases)にアクセスします
2. 最新の **woocommerce-pos-web-checkout-gateway.zip** ファイルをダウンロードします
3. WordPress 管理画面で `プラグイン > 新規追加 > プラグインのアップロード` に移動します
4. zip ファイルをアップロードし、プラグインを有効化します

  </StepCard>
  <StepCard number="2" title="WCPOS で有効化" isLast>

1. `WP Admin > POS > 設定 > 支払い` に移動します
2. 一覧で **WCPOS ウェブチェックアウトゲートウェイ** を見つけます
3. POS で使用するゲートウェイを有効化します
4. 設定を保存します

:::note
このゲートウェイは既存の WooCommerce 決済方法を利用するため、Web 決済ゲートウェイ（Stripe、PayPal など）が少なくとも 1 つ正しく設定されていることを確認してください。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### Web チェックアウトの開始 {#initiating-web-checkout}

1. **商品を追加**: POS で商品をカートに追加します
2. **ゲートウェイを選択**: 決済方法として「Web チェックアウト」を選択します
3. **リンクを生成**: システムが注文用の一意のチェックアウトリンクを作成します
4. **顧客アクセス**: 顧客がリンクをクリックして Web チェックアウトにアクセスします
5. **オンライン決済**: 顧客がウェブストアの決済方法を使用して支払いを完了します

### 決済プロセス {#payment-process}

Web チェックアウトゲートウェイは、次のワークフローに従います:

1. **注文の作成**: POS が WooCommerce に保留中の注文を作成します
2. **チェックアウトリンク**: システムが安全で一意の決済リンクを生成します
3. **顧客のリダイレクト**: 顧客は Web チェックアウトページに移動します
4. **決済方法の選択**: 顧客は利用可能な Web 決済方法から選択します
5. **決済処理**: 標準の WooCommerce チェックアウトプロセスが決済を処理します
6. **注文の完了**: 決済が成功すると、注文ステータスが自動的に更新されます

### POS ワークフロー {#pos-workflow}

Web チェックアウトを開始した後:

1. **ステータスの監視**: 決済ステータスを確認するため、POS の注文画面を開いたままにします
2. **顧客の決済**: 顧客が自分のデバイスまたは店舗のタブレットで決済を完了します
3. **自動更新**: 決済を受領すると、注文ステータスがリアルタイムで更新されます
4. **決済の処理**: レシートへ進むには、POS の「決済を処理」ボタンをクリックします
5. **レシート生成**: 通常どおりレシートを生成して印刷します

## ユースケース {#use-cases}

### 最適な用途 {#perfect-for}

- **顧客の希望**: 自分の決済アプリやカードの使用を希望する顧客
- **複雑な決済**: POSで利用できない決済方法が必要な取引
- **分割払い**: 複数の決済方法の使用を希望する顧客
- **ロイヤルティプログラム**: Webベースのロイヤルティポイント利用へのアクセス
- **ギフトカード**: デジタルギフトカードや店舗クレジットを持つ顧客
- **国際カード**: Webゲートウェイ経由の方が適している決済方法

## 技術的詳細 {#technical-details}

### 注文管理 {#order-management}

- **保留中の注文**: 注文は「支払い待ち」ステータスで作成されます
- **ステータス更新**: 支払いを受け取るとステータスが自動的に変更されます
- **注文メモ**: 支払いの詳細が注文メモに記録されます
- **在庫**: チェックアウト処理中に在庫が確保されます

### セキュリティ {#security}

- **一意のリンク**: 各チェックアウトセッションには、一意で時間制限付きのURLがあります
- **SSL必須**: すべての支払い処理は安全なHTTPS接続経由で行われます
- **PCI準拠**: 既存のPCI準拠Web支払いゲートウェイを利用します
- **セッション管理**: 安全なセッション処理により、不正アクセスを防ぎます

### 支払い方法 {#payment-methods}

このゲートウェイでは、設定済みのすべての WooCommerce 決済方法を利用できます:

- **クレジット/デビットカード**: Stripe、Square、Authorize.net など
- **デジタルウォレット**: PayPal、Apple Pay、Google Pay
- **銀行振込**: 直接銀行支払い方法
- **後払い決済**: Klarna、Afterpay、Sezzle
- **暗号資産**: Bitcoin やその他の暗号資産決済ゲートウェイ
- **ローカル決済方法**: 地域固有の決済方法

## トラブルシューティング {#troubleshooting}

### よくある問題 {#common-issues}

<Accordion>
  <AccordionItem question="チェックアウトリンクが機能しない">

  - SSL 証明書が正しくインストールされていることを確認します
  - WooCommerce のパーマリンクが正しく設定されていることを確認します
  - Web 決済ゲートウェイが有効化され、設定されていることを確認します
  - 別のブラウザーまたはデバイスでテストします

  </AccordionItem>

  <AccordionItem question="決済が処理されない">

  - Web 決済ゲートウェイが適切に設定されていることを確認します
  - 決済ゲートウェイのログでエラーメッセージを確認します
  - 顧客の決済方法がサポートされていることを確認します
  - Web チェックアウト処理を個別にテストします

  </AccordionItem>

  <AccordionItem question="注文が POS で更新されない">

  - インターネット接続の安定性を確認します
  - WordPress の cron ジョブが正常に実行されていることを確認します
  - POS の注文画面を更新します
  - WooCommerce 管理画面で注文ステータスを確認します

  </AccordionItem>

  <AccordionItem question="顧客がチェックアウトにアクセスできない">

  - チェックアウトリンクの有効期限が切れていないことを確認します
  - 顧客のデバイスがインターネットに接続できることを確認します
  - 注文がキャンセル済みまたは完了済みでないことを確認します
  - 別のデバイスでリンクをテストします

  </AccordionItem>
</Accordion>

### パフォーマンスの最適化 {#performance-optimisation}

最適なパフォーマンスのために:

- **キャッシュ**: キャッシュプラグインを設定して、購入手続きページを除外します
- **CDN**: CDN 設定が購入手続きプロセスに干渉しないようにします
- **データベース**: 注文処理を高速化するためにデータベースを最適化します
- **ホスティング**: 稼働率の高い信頼できるホスティングを使用します

### ヘルプの入手 {#getting-help}

テクニカルサポートについて:
- 問題を報告するには、[GitHub リポジトリ](https://github.com/wcpos/web-checkout-gateway)にアクセスしてください
- WooCommerce 決済ゲートウェイのドキュメントを確認してください
- ウェブチェックアウトのプロセスを定期的にテストする
- 問題がないか決済ゲートウェイのログを監視する
