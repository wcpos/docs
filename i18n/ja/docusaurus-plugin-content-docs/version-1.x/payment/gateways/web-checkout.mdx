# ウェブチェックアウトゲートウェイ {#web-checkout-gateway}

ウェブチェックアウトゲートウェイは、顧客があなたのウェブストアのチェックアウトシステムを使用してPOS取引を完了できるようにします。このゲートウェイは、店舗のPOSとオンライン決済方法の間にシームレスなブリッジを作成し、顧客が自分のデバイスを使用して支払うことを好む場合や、ウェブ専用の決済方法へのアクセスが必要な場合に最適です。

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [ウェブチェックアウトゲートウェイのリリースページ](https://github.com/wcpos/web-checkout-gateway/releases)にアクセスします
2. 最新の**woocommerce-pos-web-checkout-gateway.zip**ファイルをダウンロードします
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインのアップロード`に移動します
4. zipファイルをアップロードし、プラグインを有効化します

  </StepCard>
  <StepCard number="2" title="WCPOSでの有効化" isLast>

1. `WP管理 > POS > 設定 > チェックアウト`に移動します
2. リスト内に**WCPOSウェブチェックアウトゲートウェイ**を見つけます
3. POSでの使用のためにゲートウェイを有効にします
4. 設定を保存します

:::note
このゲートウェイは、既存のWooCommerce決済方法を利用しているため、少なくとも1つのウェブ決済ゲートウェイが正しく設定されていることを確認してください（Stripe、PayPalなど）。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### ウェブチェックアウトの開始 {#initiating-web-checkout}

1. **アイテムを追加**: POSでカートに商品を追加します
2. **ゲートウェイを選択**: 支払い方法として「ウェブチェックアウト」を選択します
3. **リンクを生成**: システムは注文のためのユニークなチェックアウトリンクを作成します
4. **顧客のアクセス**: 顧客がリンクをクリックしてウェブチェックアウトにアクセスします
5. **オンライン支払い**: 顧客はウェブストアの支払い方法を使用して支払いを完了します

### 支払いプロセス {#payment-process}

ウェブチェックアウトゲートウェイは次のワークフローに従います。

1. **注文の作成**: POSがWooCommerceに保留中の注文を作成します
2. **チェックアウトリンク**: システムが安全でユニークな支払いリンクを生成します
3. **顧客リダイレクト**: 顧客がウェブチェックアウトページにリダイレクトされます
4. **支払いの選択**: 顧客が利用可能なウェブ支払い方法から選択します
5. **支払い処理**: 標準のWooCommerceチェックアウトプロセスが支払いを処理します
6. **注文の完了**: 支払いが完了すると、注文のステータスが自動的に更新されます

### POSワークフロー {#pos-workflow}

ウェブチェックアウトを開始した後：

1. **ステータスの監視**: 支払い状況を監視するためにPOSの注文画面を開いたままにします
2. **顧客の支払い**: 顧客が自分のデバイスまたはタブレットで支払いを完了します
3. **自動更新**: 支払いが受領されると、注文のステータスがリアルタイムで更新されます
4. **支払い処理**: POSで「支払い処理」ボタンをクリックして領収書に進みます
5. **領収書の生成**: 通常通りに領収書を生成して印刷します

## 使用例 {#use-cases}

### 完璧な用途 {#perfect-for}

- **顧客の好み**: 自分の支払いアプリやカードを使用したがる顧客
- **複雑な支払い**: POSで利用できない支払い方法を必要とする取引

- **ロイヤルティプログラム**: ウェブベースのロイヤルティポイントの引き換えへのアクセス
- **ギフトカード**: デジタルギフトカードやストアクレジットを持つ顧客
- **国際カード**: ウェブゲートウェイを通じてより機能する支払い方法

## 技術的詳細 {#technical-details}

### 注文管理 {#order-management}

- **保留中の注文**: 注文は「支払い保留」ステータスで作成されます
- **ステータスの更新**: 支払いが受け入れられると自動的にステータスが変更
- **注文メモ**: 支払い詳細が注文メモに記録されます
- **在庫**: チェックアウトプロセス中に在庫が予約されます

### セキュリティ {#security}

- **ユニークなリンク**: 各チェックアウトセッションにはユニークで時間制限のあるURLがあります
- **SSLが必要**: すべての支払い処理は安全なHTTPS接続で行われます
- **PCI準拠**: 既存のPCI準拠のウェブ決済ゲートウェイを活用
- **セッション管理**: セキュアなセッション管理により不正アクセスを防ぎます

### 支払い方法 {#payment-methods}

このゲートウェイは、すべての設定済みのWooCommerce支払い方法にアクセスを提供します：

- **クレジット/デビットカード**: Stripe、Square、Authorize.netなど
- **デジタルウォレット**: PayPal、Apple Pay、Google Pay
- **銀行振込**: 直接の銀行支払い方法
- **今すぐ購入、後で支払い**: Klarna、Afterpay、Sezzle
- **暗号通貨**: Bitcoinおよびその他の暗号支払いゲートウェイ
- **ローカル方法**: 地域特有の支払い方法

## トラブルシューティング {#troubleshooting}

### 一般的な問題 {#common-issues}

<Accordion>
  <AccordionItem question="チェックアウトリンクが機能しない">

  - SSL証明書が正しくインストールされていることを確認してください
  - WooCommerceのパーマリンクが正しく設定されているか確認します
  - ウェブ決済ゲートウェイがアクティブで設定されていることを確認
  - 別のブラウザやデバイスでテストします

  </AccordionItem>

  <AccordionItem question="支払いが処理されない">

  - ウェブ決済ゲートウェイが正しく設定されているか確認してください
  - 支払いゲートウェイのログにエラーメッセージがないか確認
  - 顧客の支払い方法がサポートされているか確認
  - ウェブチェックアウトプロセスを独立してテストします

  </AccordionItem>

  <AccordionItem question="注文がPOSで更新されない">

  - インターネット接続の安定性を確認します
  - WordPressのcronジョブが正しく実行されていることを確認
  - POSの注文画面を更新します
  - WooCommerce管理画面で注文のステータスを確認

  </AccordionItem>

  <AccordionItem question="顧客がチェックアウトにアクセスできない">

  - チェックアウトリンクが期限切れでないことを確認
  - 顧客のデバイスがインターネットに接続されていることを確認
  - 注文がキャンセルまたは完了していないことを確認
  - 別のデバイスでリンクをテストします

  </AccordionItem>
</Accordion>

### パフォーマンスの最適化 {#performance-optimisation}

最適な性能を得るために：

- **キャッシング**: チェックアウトページを除外するようにキャッシングプラグインを設定
- **CDN**: CDN設定がチェックアウトプロセスに干渉しないように確認
- **データベース**: 注文処理を速めるためにデータベースを最適化
- **ホスティング**: 稼働率の良い信頼性のあるホスティングを使用

### ヘルプを得る {#getting-help}

技術サポートについて：
- [GitHubリポジトリ](https://github.com/wcpos/web-checkout-gateway)にアクセスして問題を報告
- WooCommerceの決済ゲートウェイのドキュメントを確認
- 定期的にウェブチェックアウトプロセスをテスト
- 支払いゲートウェイのログを監視して問題を確認
