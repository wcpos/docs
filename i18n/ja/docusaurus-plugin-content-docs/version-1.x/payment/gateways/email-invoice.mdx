# メール請求書ゲートウェイ {#email-invoice-gateway}

メール請求書ゲートウェイを使用すると、WCPOS から直接、支払い請求書を顧客にメールで送信できます。顧客が後払いを希望する場合や、Web 店舗の支払い方法を使用してオンラインで支払いを完了する必要がある場合に適しています。

  </FeatureCard>
  <FeatureCard icon="users" title="顧客管理">

## インストール {#installation}

<Steps>
  <StepCard number="1" title="メール請求書ゲートウェイをインストール">

`WP Admin > POS > 設定 > 拡張機能`からインストールするか、[GitHub releases page](https://github.com/wcpos/email-invoice-gateway/releases)から最新の**プラグイン zip アセット**（GitHub のソースコード zip や tarball ではありません）をダウンロードし、`プラグイン > 新規追加 > プラグインのアップロード`からアップロードします。

  </StepCard>
  <StepCard number="2" title="WCPOS で有効化" isLast>

1. `WP Admin > POS > 設定 > 支払い`に移動します
2. 一覧で**WCPOS 請求書決済ゲートウェイ**を見つけます
3. POS で使用するゲートウェイを有効化します
4. 設定を保存します

:::note
このゲートウェイでは、追加の API キーや外部サービスの設定は必要ありません。既存の WordPress メールシステムと WooCommerce 決済方法を使用します。
:::

  </StepCard>
</Steps>

## 使用方法 {#usage}

### 請求書の送信 {#sending-invoices}

1. **商品を追加**: POSで商品をカートに追加します
2. **ゲートウェイを選択**: 支払い方法として「メール請求書」を選択します
3. **メールアドレス**: 
   - 既存の顧客の場合: メールアドレスは顧客データから自動入力されます
   - ゲスト注文の場合: 顧客のメールアドレスを手動で入力します
4. **任意**: 「請求先住所に保存」にチェックを入れると、顧客の請求先情報を更新できます
5. **請求書を送信**: 処理を完了して請求書メールを送信します

### 顧客体験 {#customer-experience}

請求書を送信すると、顧客には次の内容を含むメールが届きます:

- **注文詳細**: 商品、数量、価格の完全な内訳
- **注文合計**: 税金と手数料を含む最終的な支払金額
- **支払いリンク**: オンラインで支払いを完了するための直接リンク
- **注文情報**: 注文番号と参照情報

### 支払いの完了 {#payment-completion}

顧客は次の2つの方法で支払いを完了できます:

1. **オンライン支払い**: メール内の「この注文の支払い」リンクをクリックします
2. **店舗での再来店**: 他のPOS支払い方法を使用して支払いを完了するために再来店します

「この注文の支払い」リンクをクリックすると、顧客はWebサイト上の専用支払いページに移動し、有効化されている任意のWeb決済ゲートウェイ（Stripe、PayPalなど）を使用できます。

## メール管理 {#email-management}

### 既存顧客 {#existing-customers}

- メールアドレスは顧客プロフィールから自動的に取得されます
- 顧客の請求先情報は保持されます
- 注文履歴は顧客アカウント内で管理されます

### ゲスト顧客 {#guest-customers}

- 一度限りの顧客のメールアドレスを手動で入力します
- 今後の参照用に、メールアドレスを請求先住所に保存するオプションがあります
- ゲスト注文は WooCommerce で適切に追跡されます

### メールテンプレート {#email-templates}

このゲートウェイは WooCommerce の標準メールテンプレートを使用します:

- **カスタマイズ可能**: WooCommerce のメール設定でメールの外観を変更できます
- **ブランド対応**: 店舗のロゴとブランド要素を含められます
- **プロフェッショナル**: すっきりとした業務向けの請求書形式です
- **モバイル対応**: すべてのデバイスに対応するレスポンシブデザインです

## ユースケース {#use-cases}

### 最適な用途 {#perfect-for}

- **B2B販売**: 会計システムを通じて支払いを処理する必要がある法人顧客
- **大型注文**: 承認または処理時間が必要な高額取引
- **リモート顧客**: 店舗を離れた後に支払いを完了する必要がある顧客
- **掛け取引顧客**: 確立された支払い条件を持つ常連顧客
- **見積もりから注文へ**: 見積書や概算見積もりを支払い可能な請求書に変換

## トラブルシューティング {#troubleshooting}

### よくある問題 {#common-issues}

<Accordion>
  <AccordionItem question="メールが送信されない">

  - WordPress のメール設定を確認してください
  - カスタムメールサービスを使用している場合は、SMTP 設定を確認してください
  - WordPress のメールテストプラグインでテストしてください
  - テストメールが迷惑メールフォルダーに入っていないか確認してください

  </AccordionItem>

  <AccordionItem question="決済リンクが機能しない">

  - WooCommerce が正しく設定されていることを確認してください
  - ウェブチェックアウトで少なくとも 1 つの決済ゲートウェイが有効になっていることを確認してください
  - SSL証明書が正しくインストールされていることを確認します
  - 注文ステータスで支払いが許可されていることを確認します

  </AccordionItem>

  <AccordionItem question="顧客が支払いを完了できない">

  - Web決済ゲートウェイが有効で設定済みであることを確認します
  - 注文が期限切れまたはキャンセル済みでないことを確認します
  - 顧客の支払い方法の利用上限に十分な余裕があることを確認します
  - 支払いプロセスを自身でテストします

  </AccordionItem>
</Accordion>

### メール配信 {#email-delivery}

安定したメール配信のため、次を検討してください:

- **SMTPサービス**: SendGrid、Mailgun、Amazon SESなどのサービスを使用します
- **メールプラグイン**: 配信性能を高めるためにWordPress SMTPプラグインをインストールします
- **ドメイン認証**: SPF、DKIM、DMARCレコードを設定します
- **監視**: メールの配信状況と開封率を追跡します

### ヘルプの利用 {#getting-help}

技術サポートについて:
- 問題を報告するには、[GitHub リポジトリ](https://github.com/wcpos/email-invoice-gateway)にアクセスしてください
- テンプレートのカスタマイズについては、WooCommerce のメールに関するドキュメントを確認してください
- WordPress のメールテストツールでメール機能をテストしてください
