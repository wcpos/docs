# Stripe Terminal ゲートウェイ

Stripe Terminal ゲートウェイを使用すると、WCPOS 内で Stripe Terminal ハードウェアリーダーを使用して対面での支払いを直接受け付けることができます。このゲートウェイは、物理的なカードリーダーとテスト用のシミュレーターモードの両方をサポートしています。

  <FeatureCard icon="credit-card" title="ハードウェア統合">
    インターネット接続を介して物理的な Stripe Terminal リーダーを接続
  </FeatureCard>
  <FeatureCard icon="desktop" title="シミュレーターモード">
    Stripe のシミュレーターを使用してハードウェアなしで支払いをテスト
  </FeatureCard>
  <FeatureCard icon="bolt" title="リアルタイム処理">
    即時の支払い処理と確認
  </FeatureCard>
  <FeatureCard icon="shield" title="安全な取引">
    Stripe による PCI 準拠の支払い処理
  </FeatureCard>
  <FeatureCard icon="phone" title="電話注文 (MOTO)">
    リーダーにカード情報を入力して、電話越しにカード支払いを受け付ける
  </FeatureCard>
</FeatureGrid>

## インストール {#installation}

<Steps>
  <StepCard number="1" title="Stripe Terminal for WooCommerce をインストール">

`WP Admin > POS > Settings > Extensions` からインストールするか、[GitHub リリースページ](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases) から最新の**プラグイン zip アセット**（GitHub のソースコード zip や tarball ではありません）をダウンロードし、`Plugins > Add New > Upload Plugin` からアップロードします。

  </StepCard>
  <StepCard number="2" title="Stripe 設定の構成">

1. `WP Admin > WooCommerce > Settings > Payments` に移動します
2. 支払い方法リストで **Stripe Terminal** を見つけます
3. **Stripe Terminal** をクリックして設定にアクセスします
4. **Stripe Secret Key** を入力します（Stripe ダッシュボードから取得できます）
5. 設定を保存します

:::note
WooCommerce の設定で Stripe Terminal ゲートウェイを有効にする必要はありません。次のステップで POS 専用に有効化されます。
:::

  </StepCard>
  <StepCard number="3" title="WCPOS で有効化" isLast>

1. `WP Admin > POS > Settings > Checkout` に移動します
2. リストで **Stripe Terminal** ゲートウェイを見つけます
3. POS で使用するためにゲートウェイを有効化します
4. 設定を保存します

  </StepCard>
</Steps>

## 使用方法 {#usage}

### リーダーの接続 {#connecting-a-reader}

POS でのチェックアウト中に Stripe Terminal ゲートウェイを選択すると：

1. **接続方法の選択**: 物理リーダーを接続するか、シミュレーターを使用できます
2. **物理リーダー**: 画面の指示に従って Stripe Terminal デバイスを接続します
3. **シミュレーター**: ハードウェアなしでさまざまな支払いシナリオをテストするには、シミュレーターモードを選択します

### 支払いの処理 {#processing-payments}

1. **アイテムの追加**: POS でカートに商品を追加します
2. **ゲートウェイの選択**: 支払い方法として「Stripe Terminal」を選択します
3. **リーダーの接続**: リーダーを接続するか、シミュレーターモードを選択します
4. **支払いの処理**: 取引を完了するためのプロンプトに従います
5. **確認**: 支払いが成功すると、注文が自動的に完了します

### シミュレーターでのテスト {#testing-with-simulator}

シミュレーターを使用すると、さまざまな支払い方法とシナリオをテストできます：

- **カード支払い**: 異なるカードタイプ（Visa、Mastercard など）をテスト
- **非接触型支払い**: タップして支払う取引をシミュレート
- **エラーシナリオ**: 拒否された支払いやその他のエラー条件をテスト
- **異なる金額**: さまざまな取引金額をテスト

### 電話注文 (MOTO) {#phone-orders-moto}

MOTO（Mail Order/Telephone Order：通信販売／電話注文）を使用すると、その場にいない顧客のカード支払いを処理できます。たとえば、電話越しに注文を受ける場合などです。カードをタップまたは挿入する代わりに、加盟店がカード情報をターミナルリーダーの画面に直接入力します。

#### セットアップ {#setup}

<Steps>
  <StepCard number="1" title="Stripe に MOTO アクセスをリクエスト">

MOTO はデフォルトでは有効になっていません。[Stripe サポート](https://support.stripe.com/contact)に連絡し、アカウントの MOTO 権限を有効にするよう依頼してください。これは簡単な手続きですが、Stripe による手動承認が必要です。

  </StepCard>
  <StepCard number="2" title="プラグイン設定で有効化">

1. `WP Admin > WooCommerce > Settings > Payments > Stripe Terminal` に移動します
2. **Phone Orders (MOTO)** チェックボックスをオンにします
3. 設定を保存します

  </StepCard>
  <StepCard number="3" title="対応リーダーを接続" isLast>

MOTO は、[対応ターミナル](#supported-terminals-internet-connected)に記載されている、対応するインターネット接続リーダーでのみ動作します。その他のリーダータイプではトグルが表示されません。

  </StepCard>
</Steps>

#### 電話注文を受ける {#taking-a-phone-order}

1. 対応リーダーを接続します（[対応ターミナル](#supported-terminals-internet-connected)を参照）
2. 支払い画面で **Phone Order** をオンに切り替えます
3. **Collect Card Payment** をクリックします — リーダーはタップ／挿入を促す代わりに、カード番号入力画面を表示します
4. リーダーに顧客のカード番号、有効期限、CVV を入力します
5. そこからは通常どおり支払いが処理されます

:::tip
MOTO 支払いは、支払い方法のタイプとして `card_present` ではなく `card` を使用します。これは、Stripe の観点ではオンライン取引に近い扱いとなるため、対面料金ではなく標準のオンラインカード処理手数料が適用されることを意味します。
:::

:::caution
Phone Order トグルは、次の 3 つの条件がすべて満たされた場合にのみ表示されます。プラグイン設定で MOTO 設定が有効になっていること、対応リーダーが接続されていること、そしてリーダーがシミュレーターでないこと。トグルが表示されない場合は、これらの条件を確認してください。
:::

## ハードウェアの互換性 {#hardware-compatibility}

:::important 接続要件
この実装は Stripe の JavaScript SDK を使用しており、Web アプリケーションを通じて機能しますが、**インターネットに接続されたターミナルのみ**が必要です。Bluetooth ターミナルは現在サポートされていません。
:::

### 対応ターミナル（インターネット接続） {#supported-terminals-internet-connected}

- **Stripe Reader S700/S710** ✅ - イーサネット/WiFi 接続ターミナル
- **WisePOS E** ✅ - WiFi 接続ターミナル

### 非対応ターミナル（Bluetooth） {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - Bluetooth 専用
- **BBPOS WisePad 3** ❌ - Bluetooth 専用  
- **Verifone P400** ❌ - Bluetooth 専用

:::note 今後のサポート
Bluetooth ターミナルのサポートは、今後の iOS および Android アプリのリリースで予定されています。利用可能になると、M2 や WisePad 3 を含むすべての Stripe Terminal 認定リーダーのサポートが可能になります。
:::

### 一般的な問題 {#common-issues}

<Accordion>
  <AccordionItem question="リーダーが接続できない">

  - [対応するインターネット接続ターミナル](#supported-terminals-internet-connected)を使用していることを確認してください
  - ターミナルが WiFi/Ethernet に接続されていてオンラインであることを確認してください
  - Stripe アカウントで Terminal が有効になっていることを確認してください
  - ターミナルが Stripe ダッシュボードに登録されていることを確認してください

  </AccordionItem>

  <AccordionItem question="支払いが拒否された">

  - Stripe アカウントがアクティブで良好な状態であることを確認してください
  - 使用されているカードが有効であることを確認してください
  - 十分な資金があることを確認してください

  </AccordionItem>

  <AccordionItem question="Phone Order トグルが表示されない">

  - `WooCommerce > Settings > Payments > Stripe Terminal` で **Phone Orders (MOTO)** 設定が有効になっていることを確認してください
  - 対応リーダーに接続していることを確認してください（[対応ターミナル](#supported-terminals-internet-connected)を参照）— その他のリーダータイプではトグルが非表示になります
  - シミュレーターを使用しているときはトグルが表示されません

  </AccordionItem>

  <AccordionItem question="MOTO 支払いがエラーで失敗する">

  - Stripe がアカウントで MOTO 権限を有効にしていることを確認してください — まだの場合は [Stripe サポート](https://support.stripe.com/contact)に連絡してください
  - リーダーにカード情報が正しく入力されたか再確認してください
  - MOTO 支払いは不正検出がより厳格な場合があります — カードが有効で十分な資金があることを確認してください

  </AccordionItem>

  <AccordionItem question="SSL 証明書エラー">

  - Stripe Terminal は HTTPS を必要とします - サイトに有効な SSL 証明書があることを確認してください
  - SSL 証明書が正しく構成されていることを確認してください

  </AccordionItem>
</Accordion>
