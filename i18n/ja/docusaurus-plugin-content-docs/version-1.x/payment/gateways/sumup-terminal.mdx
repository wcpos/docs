# SumUpターミナルゲートウェイ {#sumup-terminal-gateway}

SumUpターミナルゲートウェイは、WCPOS内でSumUpカードリーダーを使用してカード支払いを受け付けることを可能にします。このゲートウェイは、対面取引のためのSumUpの決済処理システムとのシームレスな統合を提供します。

## 機能 {#features}

  <FeatureCard icon="credit-card" title="ハードウェア統合">

  </FeatureCard>
</FeatureGrid>

## インストール {#installation}

<Steps>
  <StepCard number="1" title="ダウンロードとインストール">

1. [SumUpターミナルfor WooCommerceのリリースページ](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)にアクセスします
2. 最新の**sumup-terminal-for-woocommerce.zip**ファイルをダウンロードします
3. WordPress管理画面で、`プラグイン > 新規追加 > プラグインのアップロード`に移動します
4. zipファイルをアップロードし、プラグインを有効化します

  </StepCard>
  <StepCard number="2" title="SumUp設定の構成">

1. `WP管理 > WooCommerce > 設定 > 支払い`に移動します
2. 支払い方法リストから**SumUpターミナル**を見つけます
3. **SumUpターミナル**をクリックして設定にアクセスします
4. **SumUp APIキー**を入力します（SumUpマーチャントダッシュボードから入手可能）
5. 設定を保存します

:::note
WooCommerce設定でSumUpターミナルゲートウェイを有効にする必要はありません。これは、後のステップでPOS用に特別に有効化されます。
:::

  </StepCard>
  <StepCard number="3" title="SumUpターミナルをペアリングする">

1. 同じ設定ページで、**リーダーペアリング**セクションを探します
2. SumUpデバイスで、ペアリングコードを表示するためのペアリング画面に移動します
3. SumUpデバイスに表示されたペアリングコードを入力します
4. **"リーダーをペアリング"**をクリックして接続を確立します
5. リーダーが正常にペアリングされたことを確認するまで待ちます

:::warning 注意
支払いを処理する前に、リーダーが正常にペアリングされている必要があります。ペアリングプロセスが完了してから進んでください。
:::

  </StepCard>
  <StepCard number="4" title="WCPOSで有効にする" isLast>

1. `WP管理 > POS > 設定 > チェックアウト`に移動します
2. リストから**SumUpターミナル**ゲートウェイを見つけます
3. POSで使用するためにゲートウェイを有効にします
4. 設定を保存します

  </StepCard>
</Steps>

## 使用方法 {#usage}

### 支払いの処理 {#processing-payments}

1. **アイテムを追加**: POSでカートに商品を追加します
2. **ゲートウェイを選択**: 支払い方法として"SumUpターミナル"を選択します
3. **支払いを開始**: SumUpデバイスで新しい支払いを開始するためにクリックします
4. **顧客の支払い**: 顧客がSumUpターミナルで支払いを完了します
5. **自動完了**: 支払いが検出されると、注文が自動的に完了します

### 支払いコントロール {#payment-controls}

SumUpターミナルゲートウェイを使用する際は、以下のオプションがあります：

- **新しい支払いを開始**: 接続されたターミナル上で支払いリクエストを開始します
- **支払いをキャンセル**: 現在進行中の支払いをキャンセルします
- **支払い状況**: 支払いプロセスの現在の状況を監視します

### 注文管理 {#order-management}

- **自動更新**: 支払いの成功に応じて、注文は自動的に完了とマークされます
- **支払い追跡**: すべての支払い詳細が注文ノートに記録されます
- **領収書生成**: 成功した支払い後に標準POS領収書が生成されます

## ハードウェアの互換性 {#hardware-compatibility}

:::important 接続要件
この実装はSumUpのJavaScript SDKを使用しており、ウェブアプリケーションを介して機能しますが、**インターネット接続されたターミナルのみ**が必要です。Bluetoothターミナルは現在サポートされていません。
:::

### サポートされているターミナル（インターネット接続） {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet接続ターミナル

### サポートされていないターミナル（Bluetooth） {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Bluetoothのみ
- **SumUp 3G** ❌ - Bluetoothのみ

:::note 将来のサポート
Bluetoothターミナルのサポートは**2026年Q1**にiOSおよびAndroidアプリケーションに追加される予定です。これにより、Airおよび3Gモデルを含むすべてのSumUp認定ターミナルがサポートされるようになります。
:::

## トラブルシューティング {#troubleshooting}

### よくある問題 {#common-issues}

<Accordion>
  <AccordionItem question="リーダーがペアリングできない">

  - サポートされているインターネット接続ターミナル（Soloのみ）を使用していることを確認してください
  - SumUp SoloデバイスがWiFiに接続され、オンラインであることを確認してください
  - デバイスがペアリングモードになっており、ペアリングコードが正しく入力されていることを確認してください
  - SumUpアカウントにAPIアクセスが有効になっていることを確認してください
  - ペアリング中に安定したインターネット接続を確認してください

  </AccordionItem>

  <AccordionItem question="支払いが処理されない">

  - リーダーが正しくペアリングされていることを確認してください
  - SumUpアカウントがアクティブで、良好な状態にあることを確認してください
  - デバイスのバッテリーと接続が十分であることを確認してください
  - APIキーが正しくアクティブであることを確認してください

  </AccordionItem>

  <AccordionItem question="注文が完了しない">

  - インターネット接続の安定性を確認してください
  - SumUpアカウントのWebhook設定を確認してください
  - SumUpターミナルで支払いが成功したことを確認してください
  - API通信の問題についてWordPressエラーログを確認してください

  </AccordionItem>
</Accordion>

### APIレート制限 {#api-rate-limits}

ゲートウェイはAPIコールを減らし、レート制限を避けるよう最適化されています：
- 支払い状況は効率的に確認されます
- 不要なAPIリクエストは最小限に抑えられます
- 自動再試行ロジックが一時的な失敗を処理します
