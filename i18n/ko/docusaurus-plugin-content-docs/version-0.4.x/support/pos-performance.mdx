title: POS 성능
---

WCPOS는 [WooCommerce REST API](http://woothemes.github.io/woocommerce-rest-api-docs/)를 통해 서버와 통신하는 단일 페이지 자바스크립트 웹 어플리케이션입니다. 성능 문제는 서버 측과 클라이언트 측의 두 가지 범주로 나눌 수 있습니다.

### 서버 측 성능 {#server-side-performance}

서버 측 성능은 제품 페이지(10개 제품)를 다운로드하거나 주문을 처리하는 것과 같은 작업을 의미합니다. 이러한 작업의 속도에 영향을 미칠 수 있는 문제는 다음과 같습니다:

- 인터넷 연결 속도
- 서버의 속도, 즉: 처리 능력, RAM, 서버 부하 등
- 활성화된 플러그인의 수
- 결제 게이트웨이 처리

[데모 사이트](http://demo.wcpos.com/pos)는 '최상의 경우' 시나리오를 나타냅니다. 데모 사이트는 [매우 좋은 웹 호스트](http://wcpos.com/wpe)를 사용하며 활성화된 플러그인의 수가 매우 적습니다.

| 프로세스                  | [demo.wcpos.com/pos](http://demo.wcpos.com/pos)<sup>\*</sup>의 평균 속도        |
| ------------------------ | ------------------------------------------------------------------------------- |
| 10개 제품 가져오기      | 1 - 2초                                                                        |
| 현금 판매 처리           | 1 - 2초                                                                        |
| Stripe 판매 처리        | 2 - 3초                                                                        |

<small>
  <sup>*</sup> 초고속 인터넷 연결을 위한 평균 시간입니다. 시간은 귀하의 인터넷 속도에 따라 달라질 수 있습니다.
</small>

귀하의 POS가 위의 시간보다 상당히 더 오래 걸린다면 다음 테스트를 수행해 볼 수 있습니다:

1. 테마를 WordPress의 기본 Twenty Fifteen 테마로 변경합니다.
2. WooCommerce와 WCPOS를 제외한 모든 플러그인을 비활성화합니다.
3. [로컬 저장소 데이터 지우기](http://faq.wcpos.com/en/clear-local-data.html)
4. 이제 성능 향상이 있는지 확인하기 위해 POS를 사용해 보십시오.
5. **성능 향상이 있다면:** 테마와 플러그인을 하나씩 다시 활성화하여 어떤 플러그인이 성능에 영향을 미치는지 확인합니다.
6. **성능 향상이 없다면:** 더 나은 웹 호스트로 이동하거나 더 빠른 인터넷 연결에 투자하는 것을 고려할 수 있습니다.

### 클라이언트 측 성능 {#client-side-performance}

WCPOS는 JavaScript, HTML 및 CSS를 사용하여 WooCommerce REST API에서 가져온 데이터를 표시합니다. 즉, [제품](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) 및 [주문](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order)입니다.
클라이언트 측 성능을 개선하기 위해 POS는 [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API)를 사용하여 브라우저에 데이터를 저장합니다.
예를 들어, 제품이 처음으로 검색될 때 서버에 요청이 전송되고, 제품 데이터가 다운로드되면 로컬에 저장되어 이후 검색은 즉시 이루어집니다.

![로컬에 저장된 제품의 예시.](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "로컬에 저장된 제품의 예시")

IndexedDB 데이터는 브라우저를 닫거나 컴퓨터를 재시작할 때도 지속됩니다.
제품 데이터가 어떤 이유로 인해 동기화되지 않는 경우, [로컬 저장소를 지우고](http://faq.wcpos.com/en/clear-local-data.html) 서버에서 새로운 데이터 세트를 가져올 수 있습니다.
