WCPOS는 [WooCommerce REST API](http://woothemes.github.io/woocommerce-rest-api-docs/)를 통해 서버와 통신하는 단일 페이지 자바스크립트 웹 애플리케이션입니다. 성능 문제는 서버 측과 클라이언트 측으로 구분할 수 있습니다.

### 서버 측 성능

서버 측 성능은 제품 페이지 다운로드(10개 제품) 또는 주문 처리와 같은 작업을 의미합니다. 이러한 작업의 속도에 영향을 미치는 요소는 다음과 같습니다:

- 인터넷 연결 속도
- 서버 속도(처리 성능, RAM, 서버 부하 등)
- 활성화된 플러그인 수
- 결제 게이트웨이 처리

[데모 사이트](http://demo.wcpos.com/pos)는 '최상의' 시나리오를 나타냅니다. 데모 사이트는 [매우 좋은 웹 호스트](http://wcpos.com/wpe)를 사용하고 매우 적은 수의 플러그인이 활성화되어 있습니다.

| 프로세스                 | [demo.wcpos.com/pos](http://demo.wcpos.com/pos)의 평균 속도<sup>\*</sup> |
| ------------------------ | ------------------------------------------------------------------------ |
| 10개 제품 불러오기       | 1 - 2초                                                                  |
| 현금 판매 처리           | 1 - 2초                                                                  |
| 스트라이프 판매 처리     | 2 - 3초                                                                  |

<small>
  <sup>*</sup> 광대역 연결 기준 평균 시간. 인터넷 속도에 따라 시간이 달라질 수 있습니다.
</small>

POS의 처리 시간이 위 시간보다 현저히 길다면 다음 테스트를 수행해 볼 수 있습니다:

1.  워드프레스 기본 Twenty Fifteen 테마로 변경
2.  WooCommerce와 WCPOS를 제외한 모든 플러그인 비활성화
3.  [로컬 저장소 데이터 지우기](http://faq.wcpos.com/en/clear-local-data.html)
4.  POS를 사용하여 성능 향상 여부 확인
5.  **성능이 개선된 경우:** 테마와 플러그인을 하나씩 다시 활성화하여 성능에 영향을 미치는 플러그인 찾기
6.  **성능이 개선되지 않은 경우:** 더 나은 웹 호스트로 이전하거나 더 빠른 인터넷 연결에 투자 고려

### 클라이언트 측 성능

WCPOS는 WooCommerce REST API에서 검색된 데이터(예: [제품](http://woothemes.github.io/woocommerce-rest-api-docs/#view-a-product) 및 [주문](http://woothemes.github.io/woocommerce-rest-api-docs/#view-an-order))를 표시하기 위해 JavaScript, HTML, CSS를 사용합니다.
클라이언트 측 성능을 개선하기 위해 POS는 브라우저에서 [IndexedDB](https://en.wikipedia.org/wiki/Indexed_Database_API)를 사용하여 데이터를 저장합니다.
예를 들어, 제품을 처음 불러올 때 서버에 요청이 전송되며, 제품 데이터를 다운로드한 후에는 로컬에 저장되어 이후 검색이 즉시 수행됩니다.

![제품을 로컬에 저장한 예시](https://wcpos.com/wp-content/uploads/2015/07/local-products.png "로컬에 저장된 제품 예시")

IndexedDB 데이터는 브라우저를 닫거나 컴퓨터를 다시 시작해도 유지됩니다.
어떤 이유로 제품 데이터가 동기화되지 않은 것 같다면, [로컬 저장소를 지우고](http://faq.wcpos.com/en/clear-local-data.html) 서버에서 새로운 데이터 세트를 가져올 수 있습니다.
