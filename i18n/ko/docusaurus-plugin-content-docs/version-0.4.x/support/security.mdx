---
title: 보안
description: 고객 정보의 보안을 보장하는 것은 귀하의 책임입니다
---

WCPOS는 설치된 모든 [WooCommerce 결제 게이트웨이](http://www.woothemes.com/product-category/woocommerce-extensions/payment-gateways/)를 사용한 결제를 허용합니다. 결제 게이트웨이가 개인 정보를 수집하는 경우 **고객 정보의 보안을 보장하는 것은 귀하의 책임입니다**. 다음은 POS의 보안을 유지하기 위한 몇 가지 팁입니다:

### SSL 인증서 사용

온라인 스토어를 운영하는 경우 웹사이트용 보안 인증서를 가져야 합니다.
이를 통해 로그인 및 신용카드 번호와 같은 모든 양식 데이터가 암호화됩니다.
보안 인증서를 사용하지 않으면 인터넷을 통해 전송되는 정보가 가로채질 수 있으며, 이는 네트워크를 공유하는 사람이나 컴퓨터와 서버를 연결하는 수십 개의 노드 중 어느 곳에서든 발생할 수 있습니다.
기본 보안 인증서는 [미화 16달러 정도로 저렴](https://www.servertastic.com/rapidssl/)하고 설정에 하루도 채 걸리지 않으므로 보안 인증서가 없을 핑계는 없습니다.

### POS를 방치하지 마세요

인터넷을 통한 엿보기는 보안 위험의 한 방법일 뿐이며, 실제로 더 큰 피해는 악의적인 사용자가 장치에 직접 접근할 때 발생합니다.
이는 종종 컴퓨터나 태블릿을 고객 통행량이 많은 지역에 두는 상점 소유주에게 특히 중요합니다.
최악의 시나리오인 컴퓨터나 장치 도난에 대비하고 다음과 같은 보안 조치를 통해 위험을 최소화하세요:

- 기기를 5분 또는 10분 동안 방치할 때 잠금 화면 사용
- 기기를 떠날 때 WordPress와 WCPOS에서 로그아웃
- 장치가 도난당한 경우 신속하게 비밀번호 재설정
- 장치가 도난당한 경우 원격 초기화

### 최소한의 플러그인만 사용

WordPress의 장점 중 하나는 클릭 한 번으로 수천 개의 플러그인을 사용할 수 있다는 점이지만, 동시에 최악의 점이기도 합니다.
플러그인을 설치할 때마다 해당 플러그인에 웹사이트와 통과하는 모든 데이터에 대한 접근 권한을 부여하는 셈입니다.
모든 플러그인으로 스토어를 '화려하게' 만들고 싶은 유혹이 있겠지만, 이는 종종 사이트를 느리게 만들고 불안전하게 만들 수 있습니다.
모든 플러그인을 설치하기 전에 스스로에게 물어보세요:

1. 정말로 필요한가?
2. 플러그인 개발자를 신뢰할 수 있는가?

WooCommerce 스토어 보안에 대한 자세한 정보는 WooThemes 웹사이트의 [이 훌륭한 포스트](http://www.woothemes.com/2013/09/improve-your-wordpress-security-with-these-10-tips/)를 참고하세요.
