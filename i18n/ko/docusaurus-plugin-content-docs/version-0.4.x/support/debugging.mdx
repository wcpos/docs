:::caution
이 문서는 웹 개발자를 위한 것이며, 평균 사용자가 사용할 필요가 없는 기능이 디버깅입니다.
:::

WCPOS는 주로 JavaScript 웹 애플리케이션이며, 따라서 대부분의 디버깅은 [브라우저 콘솔](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors)을 통해 이루어집니다.
디버깅이 활성화되면 WCPOS는 문제를 진단하는 데 도움이 될 수 있는 다양한 메시지를 콘솔에 기록합니다.

![Mac의 Chrome에서 열린 JavaScript 콘솔](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "Mac의 Chrome에서 열린 JavaScript 콘솔")

### wp-config.php를 사용하여 디버깅 활성화하기 {#turn-on-debugging-using-wp-configphp}

wp-config.php 파일에서 [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) 플래그를 설정하면 WCPOS 디버깅을 활성화할 수 있습니다.
이렇게 하면 축소되지 않은 스크립트 파일이 로드되고 브라우저 콘솔에서 디버그 메시지가 활성화됩니다.

### 브라우저 콘솔을 사용하여 디버깅 활성화하기 {#turn-on-debugging-using-the-browser-console}

wp-config.php 파일에 접근할 수 없는 경우, 로컬 스토리지에 `debug` 플래그를 추가하여 디버그 메시지를 활성화할 수 있습니다.
이는 콘솔에 `localStorage.setItem('debug', '*')`를 입력하여 수행할 수 있습니다.
이제 페이지를 새로 고치면 디버깅이 활성화됩니다. 콘솔 로그는 브라우저 리소스를 소모할 수 있으므로 작업이 끝나면 디버깅을 끄는 것이 좋습니다.
디버깅을 끄려면 간단히 콘솔에 `localStorage.removeItem('debug')`를 입력하고 페이지를 새로 고치면 됩니다.
