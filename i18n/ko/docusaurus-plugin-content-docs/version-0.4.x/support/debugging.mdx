:::caution
이 문서는 웹 개발자를 위한 것이며, 디버깅은 일반 사용자가 사용할 필요가 없는 기능입니다.
:::

WCPOS는 주로 JavaScript 웹 애플리케이션이므로 대부분의 디버깅은 [브라우저 콘솔](http://codex.wordpress.org/Using_Your_Browser_to_Diagnose_JavaScript_Errors)을 통해 수행됩니다.
디버깅이 활성화되면 WCPOS는 문제 진단에 도움이 될 수 있는 다양한 메시지를 콘솔에 기록합니다.

![맥의 Chrome에서 열린 JavaScript 콘솔](https://wcpos.com/wp-content/uploads/2016/06/javascript-console.png "맥의 Chrome에서 열린 JavaScript 콘솔")

### wp-config.php를 사용하여 디버깅 켜기

wp-config.php 파일에서 [SCRIPT_DEBUG](https://codex.wordpress.org/Debugging_in_WordPress#SCRIPT_DEBUG) 플래그를 설정하면 WCPOS 디버깅을 활성화할 수 있습니다.
이렇게 하면 압축되지 않은 스크립트 파일이 로드되고 브라우저 콘솔에서 디버그 메시지가 활성화됩니다.

### 브라우저 콘솔을 사용하여 디버깅 켜기

wp-config.php 파일에 접근할 수 없는 경우, 로컬 스토리지에 `debug` 플래그를 추가하여 디버그 메시지를 활성화할 수 있습니다.
콘솔에 `localStorage.setItem('debug', '*')`를 입력하면 됩니다.
이제 페이지를 새로고침하면 디버깅이 켜집니다. 콘솔 로깅은 브라우저 리소스를 소비할 수 있으므로 작업이 끝나면 디버깅을 끄는 것이 좋습니다.
디버깅을 끄려면 콘솔에 `localStorage.removeItem('debug')`를 입력하고 페이지를 새로고침하면 됩니다.
