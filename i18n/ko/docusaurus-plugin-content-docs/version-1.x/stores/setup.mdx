:::info Pro 기능
다중 매장 기능은 [WCPOS Pro](/getting-started/pro-license)가 필요합니다. Pro 없이는 WCPOS가 WooCommerce 일반 설정을 기반으로 단일 가상 매장으로 운영되며, 매장 화면이 표시되지 않습니다.
:::

이 페이지에서는 매장 위치를 처음부터 끝까지 생성하는 과정을 안내합니다. 개념적 개요(다중 매장 기능, 매장별 가격 설정, ATUM 연동)는 [다중 매장](/stores)에서 확인하세요.

## 시작하기 전에 {#before-you-start}

- **WCPOS Pro가 설치되어 있고 라이선스가 활성화되어 있어야 합니다.** Pro를 활성화하면 WP Admin의 POS 아래에 **매장** 하위 메뉴가 추가됩니다.
- **이 매장에서 근무할 캐셔가 WordPress 계정을 보유하고 있어야 하며**, 해당 계정의 역할에 *Access WooCommerce POS* 권한이 포함되어 있어야 합니다. 아직 설정하지 않았다면 [접근 설정](/settings/wp-admin/access)을 참조하세요.
- *(선택 사항)* **업로드할 로고 파일** — PNG 또는 JPG, 권장 크기 300×300 이상.

## 매장 생성 {#create-the-store}

<Steps>
  <StepCard number="1" title="매장 목록 열기">

WP Admin에서 **POS → 매장**으로 이동합니다. 기존 매장 목록이 **매장**, **주소**, **허가된 사용자** 열과 각 행의 **POS 열기** 버튼과 함께 표시됩니다.

첫 번째 매장이고 Pro가 방금 활성화된 경우 목록이 비어 있습니다. 페이지 헤더에서 **새 매장 추가**를 클릭합니다.

  </StepCard>
  <StepCard number="2" title="매장 이름 지정">

편집기 상단에서 **매장 이름**을 설정합니다. 이 이름은 영수증, 로그인 시 매장 선택기, POS 전체의 매장 필터에 표시됩니다. 필수 항목입니다.

예시: *"메인 스트리트 카페"*, *"창고 — 픽업"*, *"시장 매대 #3"*.

  </StepCard>
  <StepCard number="3" title="기본 열 작성">

편집기 왼쪽에는 영수증에 표시되는 모든 항목이 그룹화되어 있습니다:

- **매장 로고** — WP 미디어 라이브러리를 통해 업로드합니다. 사이트에 사용자 정의하기 로고가 있고 이를 대체 로고로 사용하려는 경우 *"매장 로고가 설정되지 않은 경우 사이트 로고 사용"* 토글을 사용할 수 있습니다.
- **주소** — 주소 1, 주소 2, 시/군/구, 시/도, 우편번호, 국가. 국가는 입력 시 자동 검색되는 드롭다운입니다.
- **연락처** — 웹 주소, 전화번호, 이메일.
- **영수증 메시지** — *맺음 인사*, *반품 정책*, *하단 문구*. 간단한 HTML을 포함한 일반 텍스트가 허용됩니다(`<strong>`, `<em>`, `<b>`, `<i>`, `<br>`, `<p>`).
- **영수증 템플릿** — *"매장별 영수증 템플릿"* 토글을 **끔**으로 두면 **POS → 템플릿**의 사이트 전체 템플릿 목록을 상속받고, **켬**으로 전환하면 이 매장에 사용할 사용자 지정 세트를 선택할 수 있습니다. 사용자 지정 세트를 선택하면 드래그 핸들로 순서를 변경할 수 있습니다.
- **영수증 미리보기** — 템플릿 선택기와 샘플 데이터 / 주문 토글이 포함된 실시간 미리보기로, 저장하기 전에 영수증이 실제로 어떻게 표시되는지 확인할 수 있습니다.

</StepCard>
  <StepCard number="4" title="사이드바 구성">

오른쪽 열은 영수증에 표시되지 않는 매장별 구성 항목입니다:

- **승인된 사용자** — *WooCommerce POS 접근* 권한이 있는 역할을 가진 모든 WordPress 사용자의 체크박스 목록입니다. 이 매장에서 로그인할 수 있어야 하는 계산원을 선택하세요. **목록을 비워 두면 POS 권한이 있는 모든 사용자가 승인됩니다** — 단일 매장 사이트에 편리합니다.
- **세율** — *세금 계산 기준* 드롭다운: **이 매장 주소** *(기본값)* 또는 **기본 위치**. 아래 표에는 선택한 주소에 대한 세금 분류별 적용 세율이 표시되며, 읽기 전용이므로 실제 세율은 **WooCommerce → 설정 → 세금**에서 구성하세요.
- **세금 ID** — 매장별 VAT, ABN, GSTIN 등을 위한 (유형 / 값 / 국가 / 라벨) 반복 행입니다. 비워 두면 사이트 전체 [매장 세금 ID](/settings/wp-admin/store-tax-ids)로 대체됩니다.
- **매장 설정** — 통화, 통화 위치, 로케일, 시간대. 기본값은 사이트 값을 따르며, 해당 매장이 사이트 기본값과 다르게 운영되는 경우에만 재정의하세요.
- **영업 시간** — 요일별 시간대와 자유 텍스트 메모 필드입니다. 선택 사항이며 일부 템플릿에서 사용됩니다.

</StepCard>
  <StepCard number="5" title="저장" isLast>

새로 만든 매장은 처음 저장할 때까지 **만들기** 버튼이 표시됩니다. **만들기**를 클릭하세요.

저장되면 페이지가 자동 저장 모드로 전환됩니다. 편집 내용은 짧은 지연 후 자동으로 저장되며, 모서리에 *저장 중… → 저장됨* 표시가 나타납니다. 언제든지 페이지를 떠나도 변경 사항이 유지되며, 저장되지 않은 변경 사항이 있을 때 브라우저를 닫으려 하면 경고가 표시됩니다.

  </StepCard>
</Steps>

## 캐셔가 로그인 시 보는 화면 {#what-cashiers-see-at-login}

캐셔가 로그인하면 POS에 권한이 부여된 모든 매장 목록이 포함된 **매장 선택기**가 표시됩니다:

- **하나의 매장에 권한 부여됨** — 해당 매장이 자동 선택되고 **POS 열기** 버튼이 즉시 활성화됩니다. 캐셔가 어떤 매장에 입장하는지 확인할 수 있도록 선택기는 계속 표시됩니다.
- **여러 매장에 권한 부여됨** — 선택기에 라디오 목록이 표시됩니다. 매장을 선택한 다음 **POS 열기**를 클릭하세요.
- **권한 부여된 매장 없음** — 선택기가 비어 있습니다. 캐셔를 매장의 *권한 부여된 사용자* 목록에 추가하거나, 해당 목록을 비워 두어 모든 사용자에게 권한을 부여하세요.

선택한 매장은 POS 실행 시 `?store={id}`로 추가되며, 해당 세션 동안 생성되는 모든 주문에 기록됩니다(`_pos_store` 주문 메타). 이를 통해 매장별 보고서, 주문의 매장 필터, 영수증의 올바른 로고 및 주소 표시, 매장별 가격 설정이 가능합니다.

## 매장별 가격 설정 {#per-store-pricing}

WCPOS Pro에는 ATUM과 별도로 작동하는 자체 매장별 가격 설정 기능이 있습니다. 모든 상품(및 변형) 편집 화면에서 **가격** 및 **세금** 탭에 매장별 재정의 옵션이 추가됩니다:

1. WP Admin → 상품 → 편집에서 상품을 엽니다.
2. **가격** 탭에서 재정의할 매장의 *"매장별 가격 설정"* 체크박스를 선택합니다.
3. 해당 매장의 정가(및 선택 사항인 할인가)를 입력합니다.
4. 상품을 저장합니다.

해당 매장에서 계산원이 작업할 때 POS는 매장별 가격을 사용합니다. 재정의가 없는 매장에서 작업할 때는 표준 WooCommerce 가격이 적용됩니다.

동일한 방식이 **세금** 탭에서도 적용되어 매장별 세금 상태/세금 분류를 설정할 수 있습니다. 한 매장은 면세 지역에 있고 다른 매장은 그렇지 않은 경우에 유용합니다.

:::tip ATUM과 자체 매장별 가격 설정 비교
WCPOS의 자체 매장별 가격 설정은 매장마다 다른 판매 가격을 적용하되 재고는 **공유**하고자 할 때 적합한 도구입니다. **매장별 재고 수량**도 필요하다면 [ATUM Multi-Inventory](/extensions/atum) 통합을 설치하세요. 매장 편집기에 *재고 위치* 드롭다운과 *가격 소스* 선택기(기본값 / WCPOS Pro / ATUM)가 추가되어 매장별로 두 시스템을 혼합하여 사용할 수 있습니다.
:::

## 특수 사례 {#edge-cases}

<Accordion>
  <AccordionItem question="What happens to orders that already exist when I add a new store?">

아무 영향도 없습니다. 주문은 판매 시점에 `_pos_store` 메타가 기록됩니다. 새 매장을 추가해도 기존 주문이 소급하여 재할당되지 않습니다. Pro가 활성화되기 전(또는 단일 매장 설정에서) 생성된 주문에는 `_pos_store` 메타가 없으며, 사이트 수준의 기본값으로 계속 표시됩니다.

기존 주문을 다른 매장에 재할당해야 하는 경우, WP Admin → WooCommerce → 주문에서 해당 주문을 열고 Pro가 개별 주문 화면에 추가하는 **매장** 드롭다운을 사용하세요.

  </AccordionItem>

  <AccordionItem question="How do I delete a store?">

매장 편집기의 **휴지통** 버튼(및 매장 목록의 행 작업)은 WordPress 소프트 삭제(`wp_trash_post`)를 수행합니다. 영구 삭제가 아니므로:

- 해당 매장은 계산원 선택기와 필터 드롭다운에서 숨겨집니다.
- 휴지통으로 이동된 매장을 참조하는 기존 주문은 영수증 재인쇄 및 주문 상세 보기에서 여전히 정상적으로 표시됩니다 — 휴지통에 있는 매장의 이름, 주소, 로고는 계속 사용할 수 있습니다.
- 휴지통에서 복원하면 모든 설정이 유지된 채로 매장이 활성 상태로 돌아옵니다.

완전한 영구 삭제(휴지통 → 영구 삭제)만이 매장 레코드를 제거합니다. 영구 삭제된 매장을 참조하는 주문은 *"매장 #ID"*와 사이트 전체 기본값으로 대체됩니다.

  </AccordionItem>

  <AccordionItem question="Can a cashier work at more than one store?">

가능합니다 — 해당 사용자가 접근해야 하는 모든 매장의 *인증된 사용자* 목록에서 이름을 확인하세요. 사용자별 "기본 매장"이라는 개념은 없습니다. 로그인할 때마다 선택기를 통해 운영할 매장을 선택합니다.

  </AccordionItem>

  <AccordionItem question="Can I run different currencies per store?">

매장 사이드바에서 **통화**와 **통화 위치**를 설정하세요. 이 설정은 해당 매장에서 시작된 모든 세션에서 사이트 전체 WooCommerce 통화를 재정의합니다 — 영수증, 가격, 합계 모두 매장의 통화 형식으로 표시됩니다.

이는 표시 형식에만 적용됩니다. 각 통화에 대해 WooCommerce 호환 결제 게이트웨이가 여전히 필요하며, 환율 처리는 매장의 책임입니다.

  </AccordionItem>

  <AccordionItem question="Why is there no Stores submenu when I install Pro?">

두 가지 일반적인 원인:

1. **Pro가 활성화되지 않음** — **POS → 설정 → 라이선스**로 이동하여 라이선스 키가 활성 상태로 표시되는지 확인하세요.
2. **기존 무료 + Pro 레이아웃** — v1.8 이전에는 Pro가 무료 플러그인의 애드온이었습니다. v1.8 이상으로 업그레이드한 경우, 무료 플러그인을 비활성화하고 삭제해야 합니다([WCPOS Pro 설치](/getting-started/pro-license) 참고). 두 플러그인이 모두 설치되어 있으면 매장 메뉴가 표시되지 않을 수 있습니다.

  </AccordionItem>
</Accordion>
