WCPOS 영수증 템플릿은 계산대에서만 사용할 수 있는 것이 아닙니다. 온라인 스토어 고객은 내 계정 또는 주문 확인 페이지에서 주문 영수증이나 청구서를 보고 다운로드할 수 있으며, **WP Admin → POS → 템플릿**에서 만든 맞춤 템플릿으로 렌더링됩니다.

:::info WCPOS 1.9.11 필요
온라인 스토어 영수증 액세스 기능은 WCPOS 1.9.11에 포함되어 있습니다. 이전 버전에서는 POS 내에서만 영수증을 렌더링합니다.
:::

:::note 이 페이지는 고객이 자신의 영수증을 다운로드하는 방법을 설명합니다
계산원과 직원에게는 이 설정이 필요하지 않습니다. POS **주문** 화면에는 온라인 주문을 포함한 모든 WooCommerce 주문이 이미 표시되며, 점 3개 메뉴에서 [영수증을 확인, 인쇄 또는 이메일로 전송](/orders#receipt-reprinting)할 수 있습니다. 이 페이지에서는 **고객**이 웹사이트에서 직접 영수증을 다운로드하도록 설정하는 방법을 설명합니다.
:::

## 내 계정 다운로드 버튼 {#my-account}

영수증 템플릿이 **활성** 상태가 되면 로그인한 고객은 **내 계정 → 주문**에서 해당 주문 옆에 **영수증** 버튼을 볼 수 있습니다. 버튼을 클릭하면 스토어프론트의 기본 영수증 템플릿으로 렌더링된 PDF (`receipt-1234.pdf`)가 다운로드됩니다.

기본적으로 버튼은 결제가 완료된 **처리 중**, **완료됨**, **환불됨** 주문에 표시됩니다. 보류, 취소됨, 실패 주문에는 표시되지 않습니다.

별도로 구성할 사항은 없습니다. 템플릿을 활성화하면 버튼이 즉시 표시됩니다. 표시하지 않으려면 개발자가 [필터로 비활성화할 수 있습니다](#developers).

## `[wcpos_receipt]` 쇼트코드 {#shortcode}

주문 확인("감사합니다") 페이지 또는 **내 계정 → 주문 보기** 페이지에 영수증 링크를 추가하려면 다음을 사용하세요:

```text
[wcpos_receipt]
```

속성이 없으면 쇼트코드는 두 페이지에서 현재 주문을 자동으로 감지하고 *영수증 다운로드 (PDF)* 링크를 표시합니다. 다른 페이지에서는 아무것도 출력하지 않습니다. 명시적으로 지정한 `order_id` 값은 해당 주문을 소유한 로그인 고객에게만 작동합니다.

### 속성 {#shortcode-attributes}

| 속성 | 기본값 | 기능 |
|---|---|---|
| `order_id` | *(자동 감지됨)* | 특정 주문으로 연결합니다. 해당 주문을 소유한 로그인 고객에게만 작동합니다. |
| `link_text` | `Download receipt (PDF)` | 링크 텍스트입니다. |
| `format` | `pdf` | `pdf` 형식은 PDF를 다운로드하고, `html` 형식은 대신 인쇄 가능한 영수증 보기를 엽니다. |
| `template` | *(스토어프런트 기본값)* | 스토어프런트 기본값 대신 특정 템플릿 ID로 렌더링합니다. |
| `class` | `wcpos-receipt-link` | 링크의 CSS 클래스입니다. |
| `id` | *(없음)* | 링크의 HTML `id`. |

### 예시 {#shortcode-examples}

주문 확인 페이지의 다운로드 링크:

```text
[wcpos_receipt link_text="Download your invoice"]
```

PDF 대신 화면에 표시되는(인쇄 가능한) 영수증 링크:

```text
[wcpos_receipt format="html" link_text="View your receipt"]
```

특정 템플릿으로 렌더링합니다. 예를 들어 80mm 계산대 영수증 대신 A4 청구서 템플릿을 사용할 수 있습니다:

```text
[wcpos_receipt template="123" link_text="Download invoice (PDF)"]
```

템플릿 ID는 **WP Admin → POS → 템플릿**에서 템플릿 편집 화면 URL의 숫자(`post=123`)입니다.

## 고객에게 제공되는 템플릿은 무엇인가요? {#template-choice}

스토어프런트 다운로드에는 구성된 영수증 템플릿이 사용되며, 구성된 템플릿이 없으면 첫 번째 활성 템플릿이 사용됩니다. 이 스토어프런트 기본값은 계산원이 계산대 드롭다운에서 선택하는 템플릿과 별개입니다. 스토어프런트 기본값이 좁은 감열지 스타일 영수증이면 PDF도 좁게 표시됩니다.

온라인 주문에는 전체 페이지 문서가 더 적합할 수 있습니다. 다음 두 가지 옵션이 있습니다:

- 전체 페이지 템플릿을 스토어프런트 기본값으로 사용합니다(갤러리의 **청구서** 템플릿은 A4/Letter용으로 설계됨). 또는
- 계산대 템플릿을 활성 상태로 유지하고, 쇼트코드의 `template` 속성 또는 아래의 [개발자 필터](#developers)를 사용해 사이트 전체에서 스토어프런트를 다른 템플릿으로 지정합니다.

템플릿은 POS와 동일한 [영수증 데이터 페이로드](/receipts/receipt-data)로 렌더링되므로 로고, 매장 정보, 세금 설정, 사용자 지정 항목이 모두 적용됩니다.

## 보안 {#security}

영수증 링크에는 주문의 고유 키가 포함됩니다. 이는 WooCommerce에서 주문 확인 페이지에 사용하는 것과 동일한 방식이므로, 계정 없이 결제한 게스트도 확인 페이지의 링크를 계속 사용할 수 있습니다. 링크가 없는 사용자는 주문 고객으로 로그인해야 합니다. 명시적인 `order_id` 쇼트코드는 해당 주문의 로그인한 소유자에게만 표시됩니다.

:::note 고유주소 설정 필요
모든 WCPOS 결제 페이지와 마찬가지로 영수증 URL은 WordPress 다시 쓰기 규칙을 사용합니다. **기본** 고유주소(설정 → 고유주소)를 사용하는 사이트는 다른 고유주소 구조로 변경해야 합니다.
:::

## 개발자용 {#developers}

세 가지 필터가 스토어프런트 화면을 제어합니다:

```php
// Hide the My Account Receipt button.
add_filter( 'woocommerce_pos_storefront_receipt_my_account_button', '__return_false' );

// Change which order statuses show the button (default: processing, completed, refunded).
add_filter( 'woocommerce_pos_storefront_receipt_order_statuses', function ( $statuses, $order ) {
	$statuses[] = 'on-hold';
	return $statuses;
}, 10, 2 );

// Use a different template for storefront PDF downloads only.
// Receives and returns the full template array (null when none is configured).
add_filter( 'woocommerce_pos_storefront_receipt_template', function ( $template, $order ) {
	return \WCPOS\WooCommercePOS\Templates::get_template( 123 );
}, 10, 2 );
```

PDF 경로는 `/wcpos-checkout/wcpos-receipt/{order_id}?key={order_key}&format=pdf` 경로입니다. 이는 POS에서 영수증에 사용하는 것과 동일한 주문 키 제한 엔드포인트이며, `format=pdf` 형식은 화면 보기 대신 렌더링된 PDF를 반환합니다.
