**WooCommerce REST API**는 스토어 소유자가 WooCommerce 스토어를 다른 애플리케이션 및 서비스에 연결할 수 있게 해주는 표준화된 "채널" 세트와 같습니다.

작동 방식은 다음과 같습니다:

1. WooCommerce 스토어와 외부 애플리케이션 사이의 다리 역할을 하여 데이터를 통신하고 공유할 수 있게 합니다.

2. API는 특정 **엔드포인트**로 요청을 보내는 방식으로 작동합니다 - 이를 서로 다른 유형의 스토어 데이터를 위한 "채널"로 생각하면 됩니다. 예를 들어, 상품, 주문, 고객을 위한 엔드포인트가 있습니다.

3. 외부 애플리케이션은 이러한 엔드포인트를 사용하여 다음을 수행할 수 있습니다:

- 정보 가져오기 (상품 목록 조회)
- 새 데이터 추가 (새 주문 생성)
- 기존 정보 업데이트 (상품 가격 변경)
- 데이터 삭제 (오래된 상품 제거)

많은 전통적인 WordPress 플러그인이 PHP 훅과 필터에 의존하는 대신, 이 API는 **JSON**이라는 구조화되고 예측 가능한 형식으로 데이터를 보내고 받습니다.

## POS에서 JSON 데이터 보기

POS가 "보고 있는" 내용을 이해하는 데 도움을 주기 위해, 인터페이스의 여러 부분에 JSON 뷰를 포함했습니다.
예를 들어, POS에서 주문 메타나 장바구니 라인 아이템을 편집할 때 JSON 탭을 볼 수 있습니다 (아래 스크린샷 참조).

## 일부 플러그인이 POS에서 작동하지 않는 이유

WCPOS는 WooCommerce REST API를 통해 상품, 주문, 고객 데이터를 다운로드합니다.
그런 다음 해당 데이터의 로컬 복사본을 유지하여 인터넷 연결이 불안정해도 계속 실행될 수 있게 합니다.
이 접근 방식은 POS를 바쁜 계산대 환경에서 더 빠르고 안정적으로 만듭니다.

하지만 이는 사이트에서 실시간으로 동적으로 발생하는 변경 사항을 POS가 "보지" 못한다는 것을 의미합니다. 예를 들어:

    </td>
  </tr>
  <tr>
    <td>상품 추가 옵션</td>

    </td>
  </tr>
</table>
