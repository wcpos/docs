# WCPOS WP Multilang 통합 {#wcpos-wp-multilang}

[WP Multilang](https://wordpress.org/plugins/wp-multilang/)을 WCPOS와 통합하여 POS가 단일 언어의 상품만 제공하도록 합니다. Polylang이나 WPML과 달리 WP Multilang은 모든 번역을 같은 게시물 안에 저장합니다. 이 플러그인은 WCPOS가 올바른 번역을 읽고 검색 결과에서 항목을 중복 표시하지 않도록 합니다.

## 기능 {#what-it-does}

- WCPOS 상품 및 변형 REST 응답을 단일 WP Multilang 언어로 필터링합니다.
- WCPOS **fast-sync** 라우트를 가로채 활성 언어의 콘텐츠만 클라이언트에 전달되도록 합니다.
- 무료 설치에서는 WP Multilang 기본 언어를 사용합니다.
- Pro 설치에서는 매장 편집기에서 매장별 언어를 고정할 수 있습니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="WP Multilang 설치">

[WP Multilang](https://wordpress.org/plugins/wp-multilang/)을 설치하고 사이트 언어를 구성하되, 하나 이상의 언어를 기본값으로 설정합니다.

  </StepCard>
  <StepCard number="2" title="WCPOS WP Multilang 설치">

`WP Admin > POS > 설정 > 확장` 경로에서 설치하거나, [GitHub 릴리스 페이지](https://github.com/wcpos/wcpos-wp-multilang/releases)에서 최신 릴리스를 다운로드한 뒤 `플러그인 > 새로 추가 > 플러그인 업로드` 경로를 통해 업로드합니다.

  </StepCard>
  <StepCard number="3" title="(Pro) 매장별 언어 고정" isLast>

WCPOS Pro에서는 `POS > 매장` 아래에서 매장을 편집하고 **언어** 사이드바 섹션에서 해당 언어를 선택합니다. WP Multilang 기본 언어를 사용하려면 *기본값*으로 둡니다.

  </StepCard>
</Steps>
