KO: # WCPOS WPML

KO: Integrates [WPML](https://wpml.org/) with WCPOS so the POS only serves products for a single language — translated duplicates stop appearing in product search and the catalog grid. On WCPOS Pro, you can pin a language per store.

## 번역: What It Does {#what-it-does}

- KO: Filters WCPOS product and variation REST queries to a single WPML language.
- KO: Intercepts WCPOS **fast-sync** routes (the lightweight requests the POS uses to refresh its local index) so translated duplicates never reach the client.
- KO: Free installs use the WPML default language.
- KO: Pro installs can override the language per store from the store editor.

## 번역: Installation {#installation}

<Steps>
  <StepCard number="1" title="KO: Install WPML">

KO: Install and configure [WPML](https://wpml.org/) as normal with at least one language set as the default.

  </StepCard>
  <StepCard number="2" title="KO: Install WCPOS WPML">

KO: Install from `WP Admin > POS > Settings > Extensions`, or download the latest release from the [GitHub releases page](https://github.com/wcpos/wcpos-wpml/releases) and upload via `Plugins > Add New > Upload Plugin`.

  </StepCard>
  <StepCard number="3" title="KO: (Pro) Pin a language per store" isLast>

KO: On WCPOS Pro, edit a store under `POS > Stores` and pick its language from the **Language** sidebar section. Leave at *Default* to use the WPML default language.

  </StepCard>
</Steps>

## 번역: Known WPML Compatibility Issues {#known-wpml-compatibility-issues}

KO: These are behaviours of WPML itself rather than the integration — worth knowing before rolling out multilingual in production:

- KO: **POS custom fields don't carry across translations.** WPML translates products but does not copy WCPOS custom fields to translated versions by default. A product marked "POS Only" in the default language may lose that setting on its translations. Configure WPML to copy WCPOS custom fields during translation.
- KO: **POS-only products and 404s on the storefront.** Because WPML generates storefront pages for each language, POS-only products may render as 404s when accessed on the website. This is a known WPML interaction, not a WCPOS bug.

KO: See [POS-Only Products](/products/pos-only-products) for the related POS visibility controls.
