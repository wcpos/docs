# WCPOS ATUM 통합 {#wcpos-atum-integration}

[ATUM Multi-Inventory](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/)를 [WCPOS Pro](/getting-started/pro-license)와 통합하여 POS에서 위치 기반 재고, 가격 책정 및 SKU를 사용할 수 있도록 합니다.

ATUM Multi-Inventory는 창고, 소매 매장 등 여러 재고 위치에 제품 재고를 분할할 수 있게 해줍니다. 이 플러그인은 해당 ATUM 위치를 WCPOS Pro [매장](/stores)과 연결하여 각 POS 단말기가 자신의 물리적 위치에 맞는 정확한 재고 수량, 가격 및 SKU를 확인할 수 있도록 합니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="ATUM 및 Multi-Inventory 설치">

[ATUM 재고 관리](https://wordpress.org/plugins/atum-stock-manager-for-woocommerce/) 및 [ATUM Multi-Inventory 애드온](https://www.stockmanagementlabs.com/addons/atum-multi-inventory/)을 설치합니다. ATUM에서 재고 위치를 구성하세요.

  </StepCard>
  <StepCard number="2" title="WCPOS ATUM 통합 설치">

`WP Admin > POS > Settings > Extensions`에서 설치하거나, [GitHub 릴리스 페이지](https://github.com/wcpos/wcpos-atum/releases)에서 최신 릴리스를 다운로드하여 `Plugins > Add New > Upload Plugin`을 통해 업로드하세요.

  </StepCard>
  <StepCard number="3" title="매장을 ATUM 위치에 매핑" isLast>

`POS > Stores`로 이동하여 매장을 편집한 뒤 **ATUM 재고** 사이드바 섹션을 구성합니다. 매장에서 사용할 재고 위치를 선택하고, 가격 기준을 지정하며, 필요한 경우 SKU 재정의를 활성화하세요.

  </StepCard>
</Steps>

## 매장 구성 {#store-configuration}

이 플러그인은 WCPOS Pro 매장 편집기 사이드바에 **ATUM 재고** 섹션을 추가하며, 매장별로 세 가지 설정을 제공합니다:

- **재고 위치** — 이 매장이 재고를 가져올 ATUM 위치입니다.
- **가격 출처** — 상품 가격을 가져올 출처입니다:
  - *기본* — 표준 WooCommerce 가격
  - *WCPOS Pro* — WCPOS Pro에서 설정한 매장별 가격
  - *ATUM* — ATUM 재고의 위치별 가격
- **SKU 재정의** — 상품의 기본 SKU 대신 ATUM의 위치별 SKU를 선택적으로 사용합니다.

## POS 동작 {#pos-behavior}

매장에 ATUM 위치가 지정되면, POS에 제공되는 상품 데이터가 자동으로 조정됩니다:

- **재고 수량**은 WooCommerce 전체 재고가 아닌 해당 위치의 재고를 반영합니다.
- **재고 상태**는 해당 위치의 수량을 기준으로 재계산됩니다.
- **가격**은 구성된 가격 소스 설정에서 가져옵니다.
- SKU 덮어쓰기가 활성화된 경우 **SKU**는 ATUM 위치 SKU로 교체됩니다.

모든 조정은 WCPOS REST API를 통해 투명하게 처리되며 — POS 앱 측에서는 변경할 사항이 없습니다. POS에서 수행한 제품 편집 내용은 매핑된 ATUM 재고 행에도 반영됩니다. 아래의 [제품 편집 라이트백](#product-edit-write-back)을 참조하세요.

## 재고 관리 {#stock-management}

매핑된 ATUM 위치가 있는 매장에서 발생한 POS 주문의 경우, 플러그인은 ATUM의 기본 재고 차감 흐름이 쓰기 작업을 처리하도록 하되 — 올바른 위치로 안내합니다:

1. **REST 페이로드 삽입.** POS가 주문을 생성하거나 업데이트할 때, 플러그인은 각 라인 항목에 `mi_inventories` 항목을 삽입하여 ATUM이 어떤 위치에서 차감해야 하는지 알 수 있도록 합니다. 이것이 없으면 ATUM은 기본 재고로 폴백합니다.
2. **위치별 재고 필터.** 플러그인은 ATUM의 후보 재고 목록을 매장의 매핑된 위치 용어에 연결된 항목으로만 필터링하여, 차감과 복원 모두에서 올바른 항목이 선택되도록 합니다.

ATUM 자체가 주문 및 환불 시점에 실제 재고 변경을 수행하며, `atum_inventory_orders`에 실제 `order_id`로 행을 기록합니다 — ATUM의 감사 추적이 보존됩니다.

## 제품 편집 라이트백 {#product-edit-write-back}

계산원 또는 관리자가 POS에서 제품이나 변형을 편집하면, 변경 사항이 기본 WooCommerce 제품뿐만 아니라 해당 매장 위치에 매핑된 ATUM 재고 행에도 동기화됩니다. 이를 통해 각 위치의 재고, 가격, SKU가 `WP Admin`에서 수동으로 업데이트하지 않고도 ATUM과 동기화 상태를 유지합니다.

WCPOS 제품 및 변형 REST 업데이트(`POST`, `PUT`, `PATCH` 요청을 `/wcpos/v1/products/...`로 전송)에 `store_id`가 포함되어 있으면 라이트백이 트리거됩니다. 플러그인은 매장에 매핑된 ATUM 위치를 조회하여 해당 위치의 재고 행만 업데이트하며, 다른 위치는 영향을 받지 않습니다.

### 동기화되는 항목 {#what-syncs}

쓰기 반영은 각 매장의 구성에 따라 달라지며, ATUM 데이터는 해당 매장이 실제로 그 데이터를 소유한 경우에만 변경됩니다:

| 필드 | 동기화 시점 |
| --- | --- |
| **재고 수량** | 항상 — 매핑된 ATUM 위치가 있는 모든 매장은 해당 위치 행을 동기화 상태로 유지합니다. |
| **정상가 / 할인가 / 가격** | 매장의 **가격 소스**가 *ATUM*으로 설정된 경우에만 해당됩니다. |
| **SKU** | 매장에 **SKU 재정의**가 활성화된 경우에만 해당됩니다. |

매장이 *기본* 또는 *WCPOS Pro* 가격을 사용하는 경우, ATUM 가격 필드는 변경되지 않으므로 ATUM은 진실의 원천이 아닌 참조 가격으로 계속 사용됩니다. 재정의가 꺼져 있을 때 SKU에도 동일하게 적용됩니다.

### 쓰기 반영을 트리거하지 않는 항목 {#what-doesn-t-trigger-write-back}

- 제품 생성 (업데이트만 쓰기 반영 — 신규 제품은 WooCommerce의 일반 저장 경로를 따릅니다).
- `store_id`가 없는 요청 — POS는 어느 위치에 기록할지 플러그인에 알려야 합니다.
- 매핑된 ATUM 위치가 없는 매장.
- 매장 위치에 대한 기존 ATUM 재고 행이 없는 제품 — 플러그인은 새 재고 행을 생성하지 않으며, 기존 행만 업데이트합니다.
