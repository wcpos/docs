WCPOS는 POS에 새로운 기능을 추가하는 확장 기능을 지원합니다. 확장 기능 디렉터리에서 사용 가능한 확장 기능을 찾아보고, POS 설정에서 직접 설치하며, 업데이트를 관리할 수 있습니다.

:::info Pro 기능
확장 기능 설치 및 관리에는 [WCPOS Pro](/getting-started/pro-license)가 필요합니다. 무료 버전에서는 확장 기능 카탈로그가 표시되지만 설치 및 활성화 컨트롤은 비활성화됩니다.
:::

## 확장 기능 둘러보기 {#browsing-extensions}

`POS Settings > Extensions`에서 확장 기능 디렉터리를 엽니다(일부 버전에서는 **플러그인**으로 표시되기도 함).

디렉터리에는 사용 가능한 확장 기능이 카드 그리드 형태로 표시됩니다. 각 카드에는 다음 항목이 포함됩니다:

- **아이콘**(확장 기능에서 제공하지 않을 경우 퍼즐 조각 모양의 기본 아이콘이 표시됨)
- **이름 및 버전**
- **설명**
- **카테고리 배지**
- **상태** — 활성, 비활성, 업데이트 사용 가능 또는 설치되지 않음

### 필터링 및 검색 {#filtering-and-search}

상단의 **카테고리 알약 버튼**을 사용하여 카테고리별로 확장을 필터링할 수 있습니다. **검색 필드**를 사용하여 이름, 설명 또는 태그로 확장을 찾을 수도 있습니다.

## 확장 설치 {#installing-an-extension}

1. `POS Settings > Extensions`를 엽니다.
2. 원하는 확장을 찾아 **설치**를 클릭합니다.
3. 확장은 WordPress 플러그인 설치 프로그램을 사용하여 다운로드 및 설치됩니다.
4. 설치가 완료되면 **활성화**를 클릭하여 사용을 시작합니다.

내부적으로 WCPOS는 WordPress의 기본 `Plugin_Upgrader`를 사용하여 설치를 처리하므로, 확장은 일반 WordPress 플러그인과 동일한 절차를 따릅니다.

## 활성화 및 비활성화 {#activating-and-deactivating}

설치된 각 확장의 카드에는 **활성화** 및 **비활성화** 버튼이 있습니다.

- **활성화**는 확장을 활성화하여 POS에서 실행할 수 있도록 합니다.
- **비활성화**는 확장을 제거하지 않고 사용 중지합니다. 확장 파일은 서버에 그대로 남아 있으며 언제든지 다시 활성화할 수 있습니다.

## 확장 업데이트 {#updating-extensions}

설치된 확장의 새 버전이 사용 가능한 경우 카드에 **업데이트 사용 가능** 배지와 **업데이트** 버튼이 표시됩니다.

### 자동 업데이트 {#auto-updates}

디렉터리에서 설치한 확장은 **기본적으로 자동 업데이트가 활성화**되어 있습니다. 디렉터리의 각 확장 카드에서 자동 업데이트를 켜거나 끌 수 있습니다. 자동 업데이트가 켜져 있으면, 자동 업데이트가 활성화된 플러그인과 마찬가지로 WordPress가 새 버전을 자동으로 적용합니다.

## 문제 해결 {#troubleshooting}

### 버튼에 "Pro 필요" 메시지가 표시됨 {#requires-pro-message-on-buttons}

WCPOS 무료 버전에서는 설치, 활성화, 업데이트 버튼이 비활성화되어 있습니다. 확장을 관리하려면 [WCPOS Pro](/getting-started/pro-license)로 업그레이드하세요.

### 확장 설치 실패 {#extension-fails-to-install}

- WordPress 서버가 `wp-content/plugins` 디렉터리에 대한 쓰기 권한을 가지고 있는지 확인하세요.
- 서버가 외부 HTTPS 요청을 보낼 수 있는지 확인하세요(일부 호스트는 외부 다운로드를 차단합니다).
- `WP Admin > POS > Support > Logs`에서 오류 세부 정보를 확인하세요.

### 설치 후에도 확장이 표시되지 않는 경우 {#extension-not-appearing-after-install}

- POS를 새로고침하세요. 확장 목록은 최대 12시간 동안 캐시됩니다.
- 확장이 활성화되어 있는지 확인하세요(설치되어 있어도 비활성화된 확장은 실행되지 않습니다).

### 카탈로그가 로드되지 않는 경우 {#catalog-not-loading}

확장 카탈로그는 원격 소스에서 가져와 로컬에 12시간 동안 캐시됩니다. 카탈로그가 로드되지 않을 경우:

- 서버의 인터넷 연결 상태를 확인하세요.
- 캐시가 만료된 후 다시 시도하거나, 서버의 트랜지언트 캐시를 삭제하세요.

## 개발자용 {#for-developers}

### POS 확장 만들기 {#creating-a-pos-extension}

WCPOS 확장은 WCPOS 훅과 API를 통해 POS와 통합되는 표준 WordPress 플러그인입니다. 만들려면:

1. **WordPress 플러그인으로 시작하세요.** 확장 기능은 다른 WooCommerce 또는 WordPress 플러그인과 마찬가지로 표준 플러그인 헤더와 진입 파일이 필요합니다.
2. **WCPOS와 통합하세요.** WCPOS에서 제공하는 훅과 필터를 사용하여 POS 인터페이스 또는 백엔드에 기능을 추가하세요.
3. **GitHub에서 릴리스를 호스팅하세요.** 확장 기능 디렉터리는 GitHub Releases를 사용하여 버전을 추적하고 업데이트를 제공합니다.

### 디렉터리에 제출하기 {#submitting-to-the-directory}

확장 기능 카탈로그는 [`wcpos/extensions`](https://github.com/wcpos/extensions) GitHub 저장소에서 관리됩니다. 확장 기능을 등록하려면 다음과 같이 진행하세요.

1. 저장소의 README에서 카탈로그 형식과 메타데이터 요구 사항을 확인하세요.
2. 풀 리퀘스트를 열어 확장 기능의 메타데이터를 `catalog.json`에 추가하세요.
3. 병합되면 모든 WCPOS Pro 사용자에게 해당 확장 기능이 디렉터리에 표시됩니다.

### GitHub 릴리스 규칙 {#github-release-conventions}

업데이트 수명 주기는 GitHub Releases에 의존합니다.

- 시맨틱 버저닝을 사용하여 **버전에 태그를 지정하세요**(예: `v1.0.0`, `v1.2.3`).
- **플러그인 zip 파일을 첨부하세요** — 사용자가 설치하거나 업데이트할 때 다운로드되는 파일입니다.
- 디렉터리에서 감지할 수 있도록 **릴리스를 게시**하세요(초안이 아님).

새 릴리스를 게시하면 해당 확장 기능을 설치한 사용자는 확장 기능 디렉터리에서 업데이트를 사용할 수 있음을 확인하게 됩니다. 자동 업데이트가 활성화되어 있으면 자동으로 적용됩니다.

카탈로그 스키마 및 제출 프로세스에 대한 자세한 내용은 [`wcpos/extensions`](https://github.com/wcpos/extensions) 저장소를 참고하세요.
