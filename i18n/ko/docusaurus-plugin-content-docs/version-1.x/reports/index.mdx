:::

## 인터페이스 개요

### 필터

화면 상단에 있는 필터:

- **상태 필터** - 주문 상태로 필터링 (완료, 처리 중 등)
- **고객 필터** - 고객별 필터링
- **계산원 필터** - 주문 처리자별 필터링
- **매장 필터** - 매장별 필터링 (다중 매장 설정)
- **날짜 범위** - 보고 기간 선택 (오늘, 이번 주, 사용자 정의)

### 3개 패널 레이아웃

보고서 화면은 세 영역으로 나뉩니다:

#### 1. 매출 차트 (상단)

시간에 따른 매출을 막대 차트로 표시:

- 일일 보고서의 시간별 분석
- 매출 패턴 시각화
- 최대 매출 시간 확인

#### 2. 주문 테이블 (하단 왼쪽)

보고서에 포함된 주문 목록:

- **체크박스** - 주문 선택/선택 취소
- **상태** - 주문 상태 아이콘
- **주문 번호** - 고유 주문 ID
- **생성 날짜** - 주문 접수 시간
- **결제 날짜** - 결제 시간
- **합계** - 주문 금액

디스플레이 설정 아이콘을 클릭하여 열 사용자 정의.

#### 3. 보고서 요약 (하단 오른쪽)

계산된 보고서 합계:

- **보고서 유형** 드롭다운 (기본/오프라인)
- **매장 정보** (이름, ID)
- **보고서 기간** (시작/종료 타임스탬프)
- **계산원** (필터링된 경우)

**매출 요약:**
- 총 주문
- 총 순매출
- 총 세금
- **총 매출**
- 총 할인

**결제 방법:**
- 결제 유형별 분석 (현금, 카드 등)

**인쇄** 버튼으로 물리적 보고서 생성.

## 보고서 유형

### 기본 (오프라인)

기본 보고서는 로컬에 저장된 데이터로 생성됩니다. 이는:

- 서버 연결 없이 작동
- 빠른 보고서 생성
- 이미 기기에 동기화된 데이터 사용

모든 관련 주문이 이미 동기화된 빠른 하루 마감 요약에 이상적입니다.

## 하루 마감 작업 흐름

### 일반적인 일일 정산

1. **날짜 범위 설정** - "오늘" 또는 특정 날짜 선택
2. **계산원 필터링** (선택사항) - 개별 계산원 보고서 생성
3. **주문 검토** - 주문 테이블의 정확성 확인
4. **합계 확인** - 실제 현금 및 카드 단말기와 비교
5. **보고서 인쇄** - 물리적 기록 생성

### 현금 정산

1. 현금 결제만 필터링
2. 보고서의 총 현금 금액 확인
3. 금고의 실제 현금 계산
4. 합계 비교 (일치해야 함)
5. 차이점 문서화

### 카드 정산

1. 카드 결제만 필터링
2. 보고서의 총 카드 금액 확인
3. 카드 단말기의 배치 합계와 비교
4. 배치 정산 전 금액 일치 확인

## 보고서 인쇄

**인쇄** 버튼을 클릭하여 인쇄 가능한 보고서 생성. 포함 내용:

- 매장 정보
- 보고서 기간
- 매출 요약
- 결제 방법 분석
- 선택된 주문 상세 정보
