:::

## 인터페이스 개요

### 필터

화면 상단에서:

- **상태 필터** - 주문 상태로 필터링 (완료, 처리 중 등)
- **고객 필터** - 고객으로 필터링
- **캐셔 필터** - 주문을 처리한 사람으로 필터링
- **상점 필터** - 상점으로 필터링 (다중 상점 설정)
- **날짜 범위** - 보고 기간 선택 (오늘, 이번 주, 맞춤)

### 3패널 레이아웃

보고서 화면은 세 개의 영역으로 나눠져 있습니다.

#### 1. 판매 차트 (상단)

시간에 따른 판매를 보여주는 바 차트:

- 일일 보고서를 위한 시간별 세분화
- 판매 패턴의 시각적 표현
- 정점 판매 시간을 식별하는 데 도움

#### 2. 주문 테이블 (하단 왼쪽)

보고서에 포함된 주문 목록:

- **체크박스** - 포함할 주문 선택/해제
- **상태** - 주문 상태 아이콘
- **주문 번호** - 고유 주문 ID
- **생성 날짜** - 주문이 발생한 날짜
- **지불 날짜** - 결제가 이루어진 날짜
- **총액** - 주문 금액

표시 설정 아이콘을 클릭하여 열을 사용자 정의할 수 있습니다.

#### 3. 보고서 요약 (하단 오른쪽)

계산된 보고서 총계:

- **보고서 유형** 드롭다운 (기본/오프라인)
- **상점 정보** (이름, ID)
- **보고서 기간** (시작/종료 타임스탬프)
- **캐셔** (필터링된 경우)

**판매 요약:**
- 총 주문 수
- 총 순 판매액
- 총 세금 징수액
- **총 판매액**
- 총 할인액

**지불 방법:**
- 지불 유형별 내역 (현금, 카드 등)

**인쇄** 버튼을 클릭하여 실제 보고서를 생성합니다.

## 보고서 유형

### 기본 (오프라인)

기본 보고서는 로컬에 저장된 데이터로 생성됩니다. 이는:

- 서버 연결 없이 작동
- 빠른 보고서 생성 제공
- 이미 장치에 동기화 된 데이터 사용

이는 모든 관련 주문이 이미 동기화된 빠른 일일 요약에 이상적입니다.

## 일일 종료 작업 흐름

### 일반적인 일일 조정

1. **날짜 범위 설정** - "오늘" 또는 특정 날짜 선택
2. **캐셔로 필터링** (선택 사항) - 개별 캐셔 보고서용
3. **주문 검토** - 정확성을 위해 주문 테이블 확인
4. **총계 확인** - 실제 현금 수량 및 카드 단말기와 비교
5. **보고서 인쇄** - 실제 기록 생성

### 현금 조정

1. 현금 결제만 표시하도록 필터링
2. 보고서에서 총 현금 금액 기록
3. 서랍의 물리적 현금 계산
4. 총액 비교 (일치해야 함)
5. 불일치 사항 문서화

### 카드 조정

1. 카드 결제만 표시하도록 필터링
2. 보고서에서 총 카드 금액 기록
3. 카드 단말기의 배치 총액과 비교
4. 배치를 정산하기 전에 금액이 일치하는지 확인

## 보고서 인쇄하기

**인쇄** 버튼을 클릭하여 인쇄 가능한 보고서를 생성합니다. 여기에는 다음이 포함됩니다:

- 상점 정보
- 보고 기간
- 판매 요약
- 지불 방법 내역
- 주문 세부 정보 (선택된 경우)
