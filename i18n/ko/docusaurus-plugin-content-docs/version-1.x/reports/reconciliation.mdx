이 페이지에서는 캐셔(또는 매니저)가 수행하는 마감 절차를 안내합니다. 리포트 화면 자체(필터, 열, 요약 패널의 내용)에 대해서는 [리포트](/reports)를 참조하세요.

:::info Pro 기능
리포트 화면은 [WCPOS Pro](/getting-started/pro-license)가 필요합니다. Pro가 없으면 **WP Admin → WooCommerce → 분석**에서 마감 정산을 수행해야 합니다.
:::

:::note 수동 현금 정산
WCPOS에는 시재금 설정이나 "금전함 열기/닫기" 기능이 내장되어 있지 않습니다 — 교대 세션 출퇴근 기록, 앱 내 서랍 금액 계산, 자동 차이 계산 기능이 없습니다. 리포트 화면은 금전함에 **있어야 할** 금액을 보여주며, 캐셔가 직접 서랍을 세어 수동으로 비교합니다. 이 페이지에서는 해당 절차를 설명합니다.
:::

## 첫 마감 전 준비사항 {#before-your-first-close}

한 번 정하고 모든 교대에 일관되게 적용하세요:

- **시재금 금액.** 일반적인 소매 금전함은 거스름돈을 위해 소액권으로 $100~$200을 넣고 하루를 시작합니다. 금액을 정하고 일관되게 유지하면 일일 정산 계산이 간단해집니다.
- **시재금 기록 위치.** WCPOS는 이를 추적하지 않으므로 금전함 옆에 메모지, 점검표, 또는 스프레드시트를 준비해야 합니다. 어떤 방식을 선택하든 *시재금*과 *예상 금전함 합계*를 동일한 기록에 남겨야 나중에 정산을 감사할 수 있습니다.
- **누가 계수하고 누가 서명하는지.** 두 명이 하는 것이 모범 사례입니다(계산원이 계수하고 관리자가 확인). 1인 매장에서는 이를 생략하고 계산원이 모두 수행합니다 — 이를 정책으로 기록해 두세요.

## 금전함 개시 {#opening-the-till}

교대 시작 시:

1. 금고에서 시재금을 꺼내 금전함에 넣습니다.
2. 정산 시트에 시재금 금액과 개시 시간을 기록합니다.
3. 올바른 매장에서 WCPOS에 로그인합니다(다중 매장 사이트의 경우) — 어떤 매장을 선택해야 할지 모르겠다면 [다중 매장 설정](/stores)을 참조하세요.

이것으로 완료입니다. 이제 WCPOS가 해당 교대의 매출을 사용자 계정과 선택한 매장 기준으로 기록합니다.

## 교대 중 {#during-the-shift}

마감을 더 수월하게 만들어 주는 몇 가지 참고 사항:

- **쿠폰 또는 항목별 할인을 일관되게 사용하세요.** 둘 다 임의 조정에 사용할 수 있지만, 쿠폰은 보고서에서 추적 가능한 할인으로 표시되고 항목별 가격 변경은 표시되지 않습니다. 자세한 차이점은 [할인](/pos/cart/discounts)을 참조하세요.
- **주문을 취소하지 말고 보류하세요.** "POS - Open" 주문(**서버에 저장**으로 저장됨)은 주문 및 보고서에 명확하게 표시되지만, 취소된 장바구니는 아무 흔적도 남기지 않습니다. 고객이 돌아오면 보류된 주문을 이어서 처리할 수 있습니다.
- **시재함이 무거워지면 교대 중간에 초과 현금을 수거하세요.** 금액과 시간을 함께 기록하여 마감 시 정산이 맞도록 합니다. WCPOS는 현금 수거를 기록하지 않으므로 정산 시트에 별도로 관리해야 합니다.

## 마감 절차 {#the-close-procedure}

<Steps>
  <StepCard number="1" title="판매 중지">

출입문을 잠그거나(또는 대기열을 일시 중지하여) 계산 도중 새로운 거래가 발생하지 않도록 합니다. 마감 중 고객이 방문하면, 계산과 보고서를 **먼저** 완료한 후에 결제를 진행하세요 — 보고서 기간 이후에 매출을 추가하면 합계가 맞지 않게 됩니다.

  </StepCard>
  <StepCard number="2" title="시재함 계수">

시재함을 사무실이나 조용한 카운터로 가져가서 모든 권종을 계수합니다. 합계를 기록하세요 — 이것이 **계수 현금**입니다.

교대 중 현금 수거나 현금 지출(환불, 시재함에서의 거래처 지급 등)이 있었다면 해당 금액도 준비해 두세요.

  </StepCard>
  <StepCard number="3" title="보고서 화면 열기">

POS에서 **보고서**로 이동합니다. 보고서가 교대 시간을 정확히 포함하도록 필터를 설정하세요:

- **날짜 범위:** *오늘* (또는 사후에 정산하는 경우 해당 날짜)
- **계산원:** 여러 계산원이 시재함을 사용했고 본인의 매출만 확인하려는 경우 본인 사용자 선택
- **매장:** 현재 매장으로 자동 설정됩니다
- **상태:** 기본값으로 두세요 (일반적으로 *완료됨* + *처리 중*)

오른쪽의 매출 요약 패널은 필터를 변경할 때마다 업데이트됩니다. *결제 수단* 아래의 **현금** 금액을 확인하세요 — 이것이 **예상 현금 매출**입니다.

  </StepCard>
  <StepCard number="4" title="현금 정산">

계산을 수행합니다:

> **예상 시재 합계 = 시재 준비금 + 현금 매출 − 현금 환불 − 현금 출금/지출**

**실제 현금**과 비교합니다. 일치해야 합니다.

- **정확히 일치:** 완벽합니다 — 정산서에 서명하고 날짜를 기입한 후 현금을 보관하면 현금 정산이 완료됩니다.
- **불일치:** 서명하기 전에 아래의 [차이 처리](#variance-handling)를 참조하세요.

</StepCard>
  <StepCard number="5" title="카드 정산">

이제 카드 결제를 확인합니다. 매출 요약에서 **카드** 금액(및 Stripe Terminal, SumUp, Vipps/MobilePay 등 통합 게이트웨이 금액)을 확인하세요.

카드 단말기에서 **일괄 합계**를 실행합니다 (정확한 메뉴는 단말기마다 다릅니다 — 보통 기기의 *보고서* 또는 *마감 정산*에 있습니다). 비교합니다:

- **단말기 배치 합계**와 **WCPOS 카드 합계**를 동일 기간 기준으로 비교합니다 — 두 금액이 일치해야 합니다.
- 수동 배치 정산 단말기를 사용하는 경우, 퇴근 전에 **지금 배치를 정산하세요**. Stripe Terminal과 같은 통합 게이트웨이는 자동으로 정산됩니다. 웹 결제를 통한 Square 설정에 대해서는 [웹 결제](/payment/gateways/web-checkout)를 참조하세요.

</StepCard>
  <StepCard number="6" title="보고서 인쇄">

보고서 화면의 요약 패널에서 **인쇄**를 클릭합니다. 이 작업은 일반적으로 구성된 인쇄 라우팅([인쇄 라우팅](/hardware/printers#print-routing) 참조)을 사용하여 영수증 또는 문서 프린터로 보고서를 전송하고 종이 기록을 생성합니다.

인쇄된 보고서에는 매장 정보, 기간, 항목별 합계, 결제 수단별 내역, 그리고 (주문 테이블에서 주문을 선택한 경우) 주문별 상세 내역이 포함됩니다.

  </StepCard>
  <StepCard number="7" title="현금 보관 및 마감" isLast>

- **내일을 위한 시재금**을 서랍에 다시 넣어 두세요.
- 나머지 현금은 서명된 정산 확인서와 인쇄된 보고서를 함께 스테이플러로 철한 후 안전 보관 봉투에 넣습니다.
- 사용 중인 외부 시스템(스프레드시트, 회계 소프트웨어)을 업데이트합니다.
- WCPOS 앱을 종료하고 시건합니다.

</StepCard>
</Steps>

## 차이 처리 {#variance-handling}

실제 현금 수량이 예상 금액과 일치하지 않는 경우, 마감 서명 전에 다음 항목을 순서대로 확인하세요:

1. **금전함을 다시 세십시오.** 실수는 누구나 합니다. 다시 세어 보십시오. 특히 동전을 확인하십시오.
2. **누락된 현금 거래를 확인하십시오.** 다른 기기나 브라우저 탭에서 판매를 처리했는데 이 기간에 포함되지 않은 건은 없습니까? 보고서의 날짜 범위와 계산원 필터를 다시 확인하십시오.
3. **보류/미결 주문을 확인하십시오.** 주문을 *POS - Open* 상태로 필터링하십시오 — 등록되었지만 결제되지 않은 판매가 여기에 표시됩니다. 지금 완료하거나 차이가 발생할 것으로 예상하십시오.
4. **환불 및 지출.** 오늘의 모든 현금 환불과 현금 지출이 금액과 함께 정산표에 기재되어 있는지 확인하십시오.
5. **쿠폰 및 할인.** 적용된 할인이 합계에 반영되어 있는지 확인하십시오 — 보고서의 *총 할인* 항목은 추적된 쿠폰 할인을 표시하며, 개별 항목 가격 재정의는 *순매출*을 직접 감소시킵니다([할인](/pos/cart/discounts#what-changed-in-v190) 참조).
6. **잘못된 거스름돈.** 가장 흔한 차이 원인입니다. 소액 차이($5 미만)는 대부분 여기에 해당하며, 큰 차이는 조사가 필요합니다.

원인을 찾았든 찾지 못했든 차이를 정산표에 기록하십시오. 소액이라도 문서화된 차이 기록이 추적되지 않은 불일치보다 낫습니다.

## 다중 계산원 또는 다중 매장 시나리오 {#multi-cashier-or-multi-store-scenarios}

<Accordion>
  <AccordionItem question="오늘 여러 계산원이 같은 금전함을 사용한 경우">

두 가지 방법이 있습니다:

- **계산원별 마감** (가장 정확): 각 계산원이 마감 전에 자신의 사용자로 보고서를 필터링합니다. 각자 자신이 받은 금액을 세고 자신의 부분을 정산합니다. 교대 시 금전함을 물리적으로 인수인계할 수 있으며, 각 인수인계가 미니 마감이 됩니다.
- **하루 전체 한 번 마감**: 계산원 필터를 *전체*로 두고 하루가 끝날 때 금전함을 한 번 정산합니다. 더 빠르지만 차이의 원인을 파악하기 어렵습니다.

보고서 화면의 캐셔 필터는 내역 분류에 사용되는 필터와 동일하며, 캐셔별 별도 패널은 없습니다.

  </AccordionItem>

  <AccordionItem question="한 캐셔가 오늘 여러 매장에서 근무한 경우">

한 매장에서 로그아웃한 후 다른 매장에서 로그인하면, 각 세션은 `_pos_store` 주문 메타를 통해 해당 매장에 기록됩니다. 마감 시 보고서 화면에서 **매장** 필터를 전환하여 매장별로 정산을 진행합니다.

하루가 끝날 때 전체 사이트를 마감하는 경우, 매장 필터를 *전체*로 설정하여 종합 마감도 함께 진행하면 매장별 마감 내역을 교차 확인하는 데 유용합니다.

  </AccordionItem>

  <AccordionItem question="매장마다 다른 담당자가 마감하는 경우">

이것이 일반적인 패턴입니다. 각 매장의 마감 담당자가 해당 매장에 로그인한 후, 해당 매장과 당일 날짜 범위로 필터링된 보고서를 실행하고, 해당 위치의 시재를 정산합니다. 사이트 소유자는 다음 날 아침 **WP Admin → WooCommerce → 분석**에서 모든 매장의 내역을 통합하여 확인할 수 있습니다.

  </AccordionItem>

  <AccordionItem question="마감을 시작한 후에 판매가 발생한 경우">

이번 마감에는 포함하지 마십시오. 다음 중 하나를 선택합니다:

- **보고서 기간을 연장하여** 해당 판매를 포함시키고, 실제 현금 계수에도 새 판매 금액을 반영합니다. 또는
- **다음 날 보고서에 포함시키고**, 늦은 판매의 현금을 다음 날 개점 시트에 이월 금액으로 기록합니다.

하나의 방식을 선택하여 일관되게 적용하십시오. 방식을 자주 바꾸면 장부 감사가 어려워집니다.

  </AccordionItem>
</Accordion>

## 모범 사례 {#best-practices}

- **매일 마감하세요.** 하루를 건너뛰면 이틀간의 차이를 추적해야 하므로 훨씬 어려워집니다.
- **한가한 날에도 보고서를 인쇄하고 보관하세요** — 감사나 분쟁 시 이 기록이 보호 수단이 됩니다.
- **사용자 계정을 공유하지 마세요.** 각 캐셔가 개별 WordPress 사용자 계정을 사용해야 캐셔 필터가 제대로 작동하여 개인별 보고서를 확인할 수 있습니다.
- **매주 WooCommerce 분석과 대조하세요** (`WP Admin → WooCommerce → Analytics`) — 동일한 데이터를 다른 형태로 보여줍니다. 합계가 맞지 않는 경우, 대기 중인 주문이 늦게 완료되었거나 환불이 POS가 아닌 WP Admin에서 처리된 경우가 대부분입니다.
- **시재금을 일정하게 유지하세요.** 개점 시재금을 자주 변경하면 계산 오류가 발생하기 쉽습니다.
