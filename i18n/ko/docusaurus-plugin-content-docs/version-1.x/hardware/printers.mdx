프린터 설정은 POS의 **설정 > 프린터**에 있습니다. 각 기기는 자체 프린터 구성을 관리합니다 — 프린터는 로컬에 저장되며 기기 간에 동기화되지 않습니다.

## 플랫폼별 지원 프린터 {#supported-printers-by-platform}

**프린터 추가** 양식은 사용 중인 앱에 맞게 조정됩니다 — 각 플랫폼은 실제로 사용할 수 있는 연결과 옵션만 표시합니다.

| 플랫폼 | 지원 프린터 | 연결 |
|----------|--------------------|-------------|
| **웹 앱** | 내장 웹 서버가 있는 Epson 및 Star 프린터 | 네트워크(HTTP) |
| **데스크톱 앱** | 모든 네트워크 영수증 프린터 | 네트워크(raw TCP) |
| **iOS 앱** | Epson 및 Star | 네트워크, Bluetooth |
| **Android 앱** | Epson 및 Star | 네트워크, Bluetooth, USB |

:::info 웹에서는 왜 Epson과 Star만 지원하나요?
웹 브라우저는 raw TCP 연결을 열 수 없습니다. Epson 및 Star 영수증 프린터에는 HTTP로 인쇄 작업을 받는 내장 웹 서버가 있으며, 이것이 브라우저에서 ESC/POS 명령을 보내는 유일한 방법입니다. 데스크톱 앱은 TCP로 직접 연결할 수 있으므로 이 제한이 없습니다.

**모바일 앱**에서는 Bluetooth 및 USB 인쇄가 Epson과 Star의 공식 SDK를 통해 처리되므로, 이 연결에도 Epson 또는 Star 프린터가 필요합니다. 일반 프린터는 네트워크를 통해서만 작동합니다.
:::

## 프린터 추가 {#adding-a-printer}

<Steps>
  <StepCard number="1" title="프린터 설정 열기">

POS에서 **설정 > 프린터**로 이동합니다. 아직 구성된 프린터가 없으면 **프린터 추가** 버튼이 있는 빈 상태가 표시됩니다. 각 프린터에는 **프린터 이름**이 지정됩니다 — 단순한 라벨이며 "영수증 프린터"(그다음은 "영수증 프린터 2" 등)로 자동 입력됩니다 — 프린터를 구분하기 쉬운 이름으로 변경할 수 있습니다.

  </StepCard>
  <StepCard number="2" title="프린터 연결 방식 선택">

이 단계는 플랫폼별로 달라집니다:

<ul>
  <li><strong>웹 앱</strong> — <strong>공급업체</strong>를 선택합니다: Epson 또는 Star. 웹에서는 이것이 첫 번째 선택 항목입니다. 브라우저는 이 두 브랜드의 내장 웹 서버에만 접근할 수 있기 때문입니다.</li>
  <li><strong>데스크톱 앱</strong> — 프린터는 네트워크를 통해 연결됩니다. 여기서 추가로 선택할 것은 없습니다.</li>
  <li><strong>모바일 앱</strong> — <strong>연결 유형</strong>을 선택합니다: <strong>네트워크</strong>, <strong>Bluetooth</strong> 또는 <strong>USB</strong>(USB는 Android 전용). 아래 필드는 선택에 맞게 변경됩니다.</li>
</ul>

  </StepCard>
  <StepCard number="3" title="프린터 식별">

**네트워크 프린터** — **IP 주소**를 입력합니다(예: `192.168.1.100`). WCPOS는 주소를 확인해 Epson 또는 Star 기기인지 감지하고 포트, 명령 언어, 공급업체를 자동으로 입력합니다. 감지가 완료되면 "감지됨: Epson" 또는 "감지됨: Star" 라벨이 표시됩니다. 웹 앱에서는 연결할 정확한 엔드포인트 URL도 보여줍니다.

**Bluetooth 또는 USB 프린터**(모바일) — 검색된 기기 목록에서 프린터를 선택합니다. 기기를 선택하면 이름과 공급업체가 자동으로 입력됩니다. 프린터가 아직 목록에 없으면 **프린터 검색**(Bluetooth) 또는 **새로 고침**(USB)을 탭합니다.

자동 감지가 모든 항목을 올바르게 설정하지 못하면 **고급 설정**에서 모두 조정할 수 있습니다.

  </StepCard>
  <StepCard number="4" title="저장 및 테스트" isLast>

**저장**을 클릭합니다. WCPOS는 먼저 **테스트 인쇄**를 보내며, 성공한 경우에만 프린터를 저장합니다.

테스트 인쇄는 짧은 진단입니다 — 번호가 매겨진 열 눈금자, 가운데 정렬된 일반 및 두 배 크기 텍스트, 왼쪽/오른쪽 정렬 행, 용지 절단이 포함됩니다. 프린터에 연결되는지만이 아니라 용지 너비, 정렬, 크기, 절단이 올바르게 구성되었는지 확인하도록 설계되었습니다.

테스트가 실패하면 오류와 함께 프로필을 그대로 저장할 수 있는 **테스트 없이 저장** 옵션이 표시됩니다.

  </StepCard>
</Steps>

## 연결 유형(모바일 앱) {#connection-types-mobile-apps}

iOS 및 Android 앱에서 프린터는 세 가지 방식으로 연결할 수 있습니다. 프린터 추가 양식 상단에서 **연결 유형**을 선택하면 연결 필드가 그에 맞게 업데이트됩니다.

### 네트워크 {#network}

프린터가 기기와 같은 Wi-Fi 네트워크에 있습니다. IP 주소를 입력합니다 — [프린터 IP 주소 찾기](#finding-your-printers-ip-address)를 참조하세요. 네트워크 프린터는 Epson, Star 또는 일반 모델일 수 있습니다.

### Bluetooth {#bluetooth}

프린터가 Bluetooth로 기기에 페어링되어 있습니다. **프린터 검색**을 탭하고 목록에서 프린터를 선택합니다 — 이름과 공급업체가 자동으로 입력됩니다. Bluetooth 인쇄는 Epson 및 Star의 네이티브 SDK를 사용하므로 프린터는 **Epson 또는 Star** 모델이어야 합니다.

### USB(Android 전용) {#usb-android-only}

프린터가 USB 케이블로 연결되어 있습니다. **새로 고침**을 탭하고 연결된 기기 목록에서 선택합니다. Bluetooth와 마찬가지로 USB 인쇄에는 **Epson 또는 Star** 프린터가 필요합니다. iOS는 일반 USB 주변기기 지원이 없으므로 이 옵션은 Android에만 표시됩니다.

:::note
**포트** 설정은 네트워크 프린터에만 적용됩니다 — Bluetooth 및 USB 연결에서는 숨겨집니다.
:::

## 고급 설정 {#advanced-settings}

**고급 설정**을 펼쳐 프린터 구성을 세부 조정합니다. 이 필드의 대부분은 공급업체를 알게 되면 자동으로 입력됩니다.

| 설정 | 옵션 | 설명 |
|---------|---------|-------------|
| **공급업체** | Epson, Star, Generic | 프린터 제조사입니다. **웹 앱**에서는 공급업체가 양식 상단의 기본 선택 항목(Epson 또는 Star만)이며 고급 설정이 아닙니다. Generic은 데스크톱 및 모바일 앱의 **네트워크** 프린터에서 사용할 수 있지만 Bluetooth 또는 USB 연결에서는 사용할 수 없습니다. |
| **포트** | 자동 입력 | 프린터의 네트워크 포트입니다. 네트워크 전용 — Bluetooth 및 USB에서는 숨겨집니다. |
| **언어** | ESC/POS, StarPRNT, Star Line Mode | 프린터가 사용하는 명령 프로토콜입니다. 감지된 공급업체를 기준으로 자동 입력됩니다. |
| **프린터 텍스트 너비** | 58mm(32자), 80mm 표준(42자), 80mm 와이드(48자) | 한 줄에 들어가는 문자 수입니다. 용지와 프린터에 맞추세요 — 대부분의 80mm 프린터는 줄당 42자이고, 일부는 48자입니다. |
| **전체 영수증 래스터** | 끔 / 켬 | 전체 영수증을 텍스트가 아닌 이미지로 인쇄합니다 — 아래 [전체 영수증 래스터](#full-receipt-raster)를 참조하세요. |

## 프린터 옵션 {#printer-options}

이 토글은 프린터 추가 양식 하단에 있습니다:

| 옵션 | 설명 |
|--------|-------------|
| **용지 자동 절단** | 각 영수증 후 용지를 자동으로 자릅니다 |
| **금전함 자동 열기** | 인쇄 후 연결된 금전함을 열도록 킥 명령을 보냅니다 |
| **기본값으로 설정** | 이 프린터를 기본 프린터로 만듭니다 — 특정 프린터로 라우팅되지 않은 모든 영수증에 사용됩니다 |

:::note "결제 후 자동 인쇄"를 찾고 있나요?
판매가 완료될 때 영수증을 자동으로 인쇄하는 것은 **장바구니 설정**이지 프린터 설정이 아닙니다 — POS 장바구니 설정에서 **영수증 자동 인쇄**를 켜세요. *어떤* 프린터를 사용할지는 기본 프린터와 템플릿별 프린터 라우팅에 의해 결정됩니다.
:::

## 인쇄 라우팅 {#print-routing}

예를 들어 감열 영수증 **및** A4 인보이스처럼 둘 이상의 템플릿을 사용하는 경우, 인쇄 라우팅은 각 템플릿을 어떤 프린터로 보낼지 결정합니다. 라우팅은 다음 순서로 세 계층을 확인합니다:

1. **작업별 재정의.** 영수증 화면에서 프린터 드롭다운은 템플릿 전환기 옆에 있습니다. 여기서 프린터를 선택하면 해당 인쇄 작업 하나에 대해 모든 설정을 재정의합니다. 템플릿을 전환하면 **자동**으로 다시 설정됩니다.
2. **설정 재정의.** **설정 → 인쇄 라우팅**으로 이동해 각 템플릿에 특정 프린터를 할당합니다. 예를 들어 감열 영수증은 Epson 네트워크 프린터로, HTML 인보이스는 시스템 인쇄 대화상자로 보낼 수 있습니다. 템플릿을 **자동**으로 되돌리면 재정의가 제거됩니다.
3. **자동 매칭.** 재정의가 없으면 WCPOS가 자동으로 매칭합니다:
   - **감열 템플릿**은 문자 너비가 일치하는 감열 프린터로 라우팅됩니다. 58mm 감열 템플릿은 32열 프린터를 선호하고, 80mm 템플릿은 42열 또는 48열 프린터를 선호합니다.
   - **HTML 템플릿**은 시스템 인쇄 대화상자로 라우팅됩니다.
   - 여러 프린터가 일치하면 **기본** 프린터가 우선합니다.

템플릿을 호환되지 않는 프린터로 수동 전송하면 — 예를 들어 감열 템플릿을 시스템 대화상자로 보내면 — 영수증 화면에 호박색 **불일치 경고**가 나타납니다. 인쇄는 계속되지만 출력이 올바르게 렌더링되지 않을 수 있습니다.

라우팅 재정의는 **기기별**로 저장됩니다. 각 iPad, 전화, 컴퓨터가 자체 라우팅을 관리합니다 — 서버 측 동기화는 없습니다.

:::note
인쇄 라우팅 설정 페이지는 프린터를 하나 이상 추가한 후에만 표시됩니다. 구성된 프린터가 없으면 모든 템플릿은 암시적으로 시스템 인쇄 대화상자를 사용합니다.
:::

## 전체 영수증 래스터 {#full-receipt-raster}

기본적으로 WCPOS는 영수증을 감열 프린터에 **텍스트**로 보냅니다 — 빠르고, 작고, 선명하며 프린터의 내장 글꼴을 사용합니다. 문제는 감열 프린터 글꼴이 제한된 문자 집합만 지원한다는 점입니다. 프린터에 글꼴이 없는 문자 체계 — 많은 비라틴 문자, 오른쪽에서 왼쪽으로 쓰는 텍스트, 일부 기호 — 는 빈 상자나 깨진 문자로 출력될 수 있습니다.

**전체 영수증 래스터**는 이 문제를 해결합니다. 켜면 WCPOS가 전체 영수증을 이미지로 렌더링하고 그 이미지를 프린터로 보내므로, 출력물이 화면에 보이는 것과 정확히 일치합니다 — 어떤 언어나 문자 체계에서도 마찬가지입니다.
