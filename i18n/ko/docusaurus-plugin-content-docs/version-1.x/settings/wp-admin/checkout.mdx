결제 설정 페이지(`WP Admin > POS > 설정 > 결제`)에서는 POS에 표시되는 결제 게이트웨이, 주문 처리 방식, POS 판매에 대해 발송되는 알림 이메일을 제어합니다.

## 이메일 알림 {#email-notifications}

페이지 상단에는 **관리자 이메일**, **고객 이메일**, **계산원 이메일**이라는 세 가지 이메일 토글 그룹이 있으며, 사이트 전체 WooCommerce 이메일 설정과 별도로 POS 주문에 대해 어떤 WooCommerce 알림 이메일을 보낼지 제어합니다.

이메일별 토글 기본값, 계산원 수신자 모델, 일반적인 시나리오, 개발자 훅을 포함한 전체 참조는 **[이메일 알림](/settings/wp-admin/email-notifications)**을 참조하세요.

## 결제 게이트웨이 {#payment-gateways}

이 섹션에는 사이트에 설치된 모든 WooCommerce 결제 게이트웨이가 나열됩니다. 각 게이트웨이는 POS에서 독립적으로 활성화하거나 비활성화할 수 있습니다.

| 열 | 설명 |
|--------|-------------|
| **기본값** | 계산원이 결제를 열 때 미리 선택될 게이트웨이를 선택합니다 |
| **게이트웨이** | 결제 게이트웨이 이름 |
| **게이트웨이 ID** | WooCommerce 게이트웨이 식별자(개발자에게 유용) |
| **활성화됨** | POS 결제에서 게이트웨이를 켜거나 끕니다 |
| **주문 상태** | 이 게이트웨이로 결제된 POS 주문에 설정되는 상태입니다(아래 [주문 상태](#order-status) 참조) |

각 행에는 메뉴(⋮)도 있으며, **POS 설정**에서는 게이트웨이의 POS 전용 **제목**과 **설명**을 변경할 수 있고, **WooCommerce 설정**은 WooCommerce의 게이트웨이 전체 설정으로 바로 연결됩니다.

### 게이트웨이 활성화 {#enabling-a-gateway}

1. 목록에서 게이트웨이를 찾습니다
2. **활성화됨** 스위치를 켭니다
3. 다음 페이지 로드 시 POS 결제에 게이트웨이가 표시됩니다

:::note 무료와 Pro
무료 버전에서는 기본 제공 **현금** 및 **카드** 게이트웨이만 편집할 수 있습니다. POS에서 설치된 다른 WooCommerce 게이트웨이를 활성화하려면 [WCPOS Pro](/getting-started/pro-license)가 필요합니다.
:::

### 게이트웨이 순서 변경 {#reordering-gateways}

게이트웨이를 드래그하여 POS 결제 화면의 표시 순서를 변경합니다. 계산원이 결제 모달을 열 때 미리 선택될 게이트웨이를 **기본값** 열에서 선택합니다.

### POS와 온라인 매장 {#pos-vs-online-store}

여기의 게이트웨이 설정은 WooCommerce 결제 설정과 **독립적**입니다. 게이트웨이는 다음과 같이 설정할 수 있습니다:

- **POS에서는 활성화, 온라인에서는 비활성화** — 웹 주문에는 적용되지 않는 현금 결제에 유용합니다
- **온라인에서는 활성화, POS에서는 비활성화** — 대면 판매에 적합하지 않은 PayPal 같은 게이트웨이에 유용합니다
- **둘 다 활성화** — 게이트웨이가 모든 곳에서 작동합니다

:::note
게이트웨이가 POS 결제 설정 목록에 표시되려면 먼저 `WP Admin > WooCommerce > 설정 > 결제` 에 설치되어 활성화되어 있어야 합니다. 게이트웨이가 누락된 경우 플러그인이 설치되어 있고 WooCommerce 에서 게이트웨이가 활성화되어 있는지 확인합니다.
:::

## 주문 상태 {#order-status}

주문 상태는 게이트웨이 표의 **주문 상태** 열에서 **게이트웨이별로** 설정됩니다 — 따라서 판매는 결제 방식에 따라 서로 다른 상태로 처리될 수 있습니다:

- **현금** 및 **카드**는 판매 시점에 결제가 수금되므로 기본값이 **완료됨**입니다.
- **BACS**(직접 은행 송금) 또는 **수표**처럼 나중에 입금되는 게이트웨이는 기본값이 **대기 중**입니다.

각 게이트웨이에 대해 원하는 WooCommerce 주문 상태를 선택합니다. (이전 버전에서는 전역 기본값 하나를 사용했으며, 업그레이드 시 해당 값이 각 게이트웨이로 마이그레이션되었습니다.)
