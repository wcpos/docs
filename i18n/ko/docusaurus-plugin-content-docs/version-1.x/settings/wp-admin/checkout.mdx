결제 설정 페이지(`WP Admin > POS > Settings > Checkout`)에서는 POS에 표시되는 결제 게이트웨이와 주문 처리 방식을 제어합니다.

## 결제 게이트웨이 {#payment-gateways}

이 섹션에는 사이트에 설치된 모든 WooCommerce 결제 게이트웨이가 표시됩니다. 각 게이트웨이는 POS에서 개별적으로 활성화하거나 비활성화할 수 있습니다.

| 열 | 설명 |
|--------|-------------|
| **활성화됨** | POS 결제에서 게이트웨이를 켜거나 끕니다 |
| **게이트웨이** | 결제 게이트웨이 이름 |
| **게이트웨이 ID** | WooCommerce 게이트웨이 식별자(개발자에게 유용) |

### 게이트웨이 활성화 {#enabling-a-gateway}

1. 목록에서 게이트웨이를 찾습니다
2. **활성화** 스위치를 켭니다
3. 다음 페이지 로드 시 POS 결제 화면에 게이트웨이가 표시됩니다

### 게이트웨이 순서 변경 {#reordering-gateways}

POS 결제 화면에서 표시 순서를 변경하려면 게이트웨이를 드래그합니다. 캐셔가 결제 모달을 열면 활성화된 첫 번째 게이트웨이가 기본으로 선택됩니다.

### POS와 온라인 스토어 비교 {#pos-vs-online-store}

이곳의 게이트웨이 설정은 WooCommerce 결제 설정과 **독립적**입니다. 게이트웨이는 다음과 같이 구성할 수 있습니다:

- **POS에서 활성화, 온라인에서 비활성화** — 웹 주문에는 적용되지 않는 현금 결제에 유용합니다
- **온라인에서 활성화, POS에서 비활성화** — 대면 판매에는 적합하지 않은 PayPal과 같은 게이트웨이에 유용합니다
- **양쪽 모두 활성화** — 게이트웨이가 모든 환경에서 작동합니다

:::note
게이트웨이를 사용하려면 먼저 다음 위치에 설치되고 활성화되어 있어야 합니다: `WP Admin > WooCommerce > Settings > Payments` 그래야 POS 결제 설정 목록에 표시됩니다. 게이트웨이가 보이지 않는 경우, 플러그인이 설치되어 있고 WooCommerce에서 해당 게이트웨이가 활성화되어 있는지 확인하세요.
:::

## 주문 상태 {#order-status}

완료된 POS 주문의 기본 주문 상태를 구성합니다. 기본적으로 POS 주문은 판매 시점에 결제가 이루어지므로 **완료됨** 상태로 설정됩니다.
