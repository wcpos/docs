---
title: POS 접근하기
sidebar_label: 접근
---

WCPOS 기능에 대한 접근을 제어하기 위해 WordPress 권한 및 사용자 역할을 활용할 수 있습니다. 사용자에게 특정 역할을 할당하고 해당 권한을 구성함으로써 다양한 POS 사용자의 접근 수준을 맞춤 설정할 수 있습니다. WCPOS 플러그인은 기존 WordPress 역할과 통합되며 캐셔를 위한 특정 역할을 도입합니다.

다음 표는 POS의 주요 사용자 역할과 해당 권한을 요약합니다:

| 역할          | 생성 주체      | 설명                                                                                                                                                                                                                                                                                                                                                    |
| ------------- | --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 관리자 | WordPress       | 기본적으로 모든 권한이 활성화된 사이트 소유자입니다.                                                                                                                                                                                                                                                                                                                    |
| 상점 관리자  | WooCommerce     | POS에 접근하고 POS 설정을 관리할 수 있는 사용자를 위한 역할입니다. 상점 관리자는 플러그인 설치 또는 다른 관리 작업을 수행할 권한이 없을 수 있습니다.                                                                                                                                                                                             |
| 캐셔       | WCPOS | WCPOS 플러그인에서 추가된 역할로, 주로 POS를 사용할 사용자를 위한 것입니다. 캐셔는 일반적으로 제한된 권한을 가지며 WP 관리자에서 POS 설정을 수정하거나, 플러그인을 설치하거나, 블로그 게시물을 작성할 수 없습니다. 경우에 따라 캐셔가 POS에서 고객을 추가하거나 편집하는 것을 제한할 수도 있습니다. |

사용자 역할 및 권한을 할당하고 관리하려면 **`WP 관리자 > POS > 접근`**으로 이동할 수 있습니다.

다음 표는 WCPOS 플러그인과 관련된 WordPress 권한을 제공합니다:

| 권한                  | 설명                                                            | 사용 예시                                                                                                                                    |
| --------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `access_woocommerce_pos`    | WCPOS 인터페이스에 접근할 수 있습니다.                        | 이 옵션을 활성화하면 캐셔에게 POS에 접근하고 사용할 수 있는 권한을 부여하여 판매 시점 거래를 수행할 수 있게 됩니다.          |
| `manage_woocommerce_pos`    | WCPOS 설정 및 구성을 관리할 수 있습니다. | 이 옵션을 활성화하면 캐셔가 결제 방법이나 영수증 템플릿과 같은 POS 관련 설정을 수정할 수 있게 됩니다.     |
| `create_users`              | 새 WordPress 사용자 계정을 생성할 수 있습니다.              | 이 옵션을 비활성화하면 캐셔가 POS에서 새 고객 계정을 생성할 수 없게 됩니다.                                                  |
| `edit_others_shop_orders`   | 다른 캐셔가 생성한 상점 주문을 편집할 수 있습니다.              | 이 옵션을 비활성화하면 캐셔가 다른 캐셔가 POS에서 생성한 상점 주문을 수정할 수 없게 됩니다.                              |
| `edit_product`              | 상품 세부 정보를 편집할 수 있습니다.                            | 이 옵션을 비활성화하면 캐셔가 POS에서 사용 가능한 상품의 세부 정보를 수정할 수 없게 됩니다.                                     |
| `edit_published_products`   | 게시된 상품을 편집할 수 있습니다.                         | 이 옵션을 비활성화하면 캐셔가 POS에서 이미 게시된 상품을 변경할 수 없게 됩니다.                           |
| `edit_shop_orders`          | 상점 주문을 편집할 수 있습니다.                                | 이 옵션을 비활성화하면 캐셔가 POS에서 주문 상태나 고객 세부 정보와 같은 상점 주문 세부 정보를 수정할 수 없게 됩니다. |
| `edit_users`                | 기존 사용자 계정을 편집할 수 있습니다.                     | 이 옵션을 비활성화하면 캐셔가 POS에서 청구 주소와 같은 고객 계정 세부 정보를 수정할 수 없게 됩니다.            |
| `list_users`                | 사용자 계정 목록을 볼 수 있습니다.                    | 이 옵션을 비활성화하면 캐셔가 POS에서 고객을 검색하거나 선택할 수 없게 됩니다.                                                |
| `manage_product_terms`      | 상품 카테고리와 태그를 관리할 수 있습니다.              | 이 옵션을 비활성화하면 캐셔가 POS에서 상품 카테고리나 태그를 생성, 편집 또는 삭제할 수 없게 됩니다.                       |
| `publish_shop_orders`       | 상점 주문을 게시할 수 있습니다.                             | 이 옵션을 비활성화하면 캐셔가 POS에서 새 상점 주문을 완료하고 처리할 수 없게 됩니다.                                       |
| `read_private_products`     | 비공개 상품의 세부 정보에 접근할 수 있습니다.          | 이 옵션을 비활성화하면 캐셔가 POS에서 사용 가능한 비공개 상품의 세부 정보를 볼 수 없게 됩니다.                               |
| `read_private_shop_coupons` | 비공개 상점 쿠폰의 세부 정보에 접근할 수 있습니다.      | 이 옵션을 비활성화하면 캐셔가 POS에서 비공개 상점 쿠폰의 세부 정보를 볼 수 없게 됩니다.                                     |
| `read_private_shop_orders`  | 비공개 상점 주문의 세부 정보에 접근할 수 있습니다.       | 이 옵션을 비활성화하면 캐셔가 POS에서 비공개 상점 주문의 세부 정보를 볼 수 없게 됩니다.                                      |
| `read`                      | WordPress 관리자에 접근할 수 있습니다.                                  | 이 옵션을 비활성화하면 캐셔가 WordPress 대시보드와 사용자 프로필 페이지에 접근할 수 없게 됩니다.                                                   |

각 역할에 적절한 권한을 할당함으로써 POS의 다른 사용자에 대한 접근 및 권한을 미세 조정할 수 있습니다. 이를 통해 특정 작업을 수행하고, 민감한 정보를 보고, 상점에 변경을 가할 수 있는 사용자를 제어할 수 있습니다.

WordPress에서 사용자 역할 및 권한 관리에 대한 자세한 정보는 [WordPress Codex](https://codex.wordpress.org/Roles_and_Capabilities)를 참조할 수 있습니다.
