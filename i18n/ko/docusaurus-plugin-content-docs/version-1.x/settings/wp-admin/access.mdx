---
title: POS 접근
sidebar_label: 접근
---

WCPOS 기능에 대한 접근을 제어하기 위해 WordPress의 권한 및 사용자 역할을 활용할 수 있습니다. 특정 역할을 사용자에게 할당하고 그들의 권한을 설정함으로써 다양한 POS 사용자에 대한 접근 수준을 맞춤 설정할 수 있습니다. WCPOS 플러그인은 기존의 WordPress 역할과 통합되며, 캐셔를 위한 특정 역할을 추가합니다.

다음 표는 POS의 주요 사용자 역할과 그에 따른 권한을 간략히 설명합니다:

| 역할          | 생성자          | 설명                                                                                                                                                                                                                                                                                                                                                     |
| ------------- | --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 관리자       | WordPress       | 모든 권한이 기본적으로 활성화된 사이트 소유자입니다.                                                                                                                                                                                                                                                                                                    |
| 상점 관리자  | WooCommerce     | 이 역할은 POS에 접근하고 POS 설정을 관리해야 하는 사용자용으로 설계되었습니다. 상점 관리자는 플러그인을 설치하거나 다른 관리 작업을 수행할 권한이 없을 수 있습니다.                                                                                                                                                                         |
| 캐셔         | WCPOS           | 이 역할은 WCPOS 플러그인에 의해 추가되며, 주로 POS를 사용할 사용자에게 적합합니다. 캐셔는 일반적으로 제한된 권한을 가지며 POS 설정을 수정하거나 플러그인을 설치하거나 블로그 게시물을 생성하기 위해 WP Admin에 접근할 수 없습니다. 경우에 따라 캐셔가 POS에서 고객을 추가하거나 수정하는 것을 제한하고 싶을 수도 있습니다. |

사용자 역할 및 권한을 할당하고 관리하려면 **`WP Admin > POS > 접근`**으로 이동할 수 있습니다.

다음 표는 WCPOS 플러그인과 관련된 WordPress 권한의 개요를 제공합니다:

| 권한                      | 설명                                                            | 예제 사용                                                                                                                                    |
| ------------------------- | ---------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `access_woocommerce_pos`   | WCPOS 인터페이스에 대한 접근을 허용합니다.                    | 이 옵션을 활성화하면 캐셔가 POS에 접근하고 사용할 수 있는 권한이 부여되어 포인트 오브 세일 거래를 수행할 수 있습니다.                     |
| `manage_woocommerce_pos`   | WCPOS 설정 및 구성을 관리할 수 있습니다.                     | 이 옵션을 활성화하면 캐셔가 POS와 관련된 설정을 수정할 수 있는 능력이 부여됩니다. 예를 들어 결제 방법이나 영수증 템플릿과 같습니다.     |
| `create_users`            | 캐셔가 새로운 WordPress 사용자 계정을 생성할 수 있도록 허용합니다. | 이 옵션을 비활성화하면 캐셔가 POS에서 새로운 고객 계정을 생성하는 것을 방지합니다.                                                         |
| `edit_others_shop_orders`  | 다른 캐셔가 생성한 상점 주문을 편집할 수 있도록 허용합니다.    | 이 옵션을 비활성화하면 캐셔가 POS에서 다른 캐셔가 생성한 상점 주문을 수정하는 것을 방지합니다.                                             |
| `edit_product`            | 캐셔가 제품 세부 정보를 편집할 수 있도록 허용합니다.          | 이 옵션을 비활성화하면 캐셔가 POS에서 사용 가능한 제품의 세부 정보를 수정하는 것을 방지합니다.                                           |
| `edit_published_products` | 캐셔가 게시된 제품을 편집할 수 있도록 허용합니다.           | 이 옵션을 비활성화하면 캐셔가 POS에서 이미 게시된 제품을 변경하는 것을 방지합니다.                                                      |
| `edit_shop_orders`        | 캐셔가 상점 주문을 편집할 수 있도록 허용합니다.              | 이 옵션을 비활성화하면 캐셔가 POS에서 주문 상태나 고객 세부 사항과 같은 상점 주문의 세부 정보를 수정하는 것을 방지합니다.                |
| `edit_users`              | 캐셔가 기존 사용자 계정을 편집할 수 있도록 허용합니다.       | 이 옵션을 비활성화하면 캐셔가 POS에서 고객 계정의 세부 정보를 수정하는 것을 방지합니다.                                                  |
| `list_users`              | 캐셔가 사용자 계정 목록을 볼 수 있도록 허용합니다.           | 이 옵션을 비활성화하면 캐셔가 POS에서 고객을 검색하거나 선택하는 것을 방지합니다.                                                        |
| `manage_product_terms`    | 캐셔가 제품 카테고리 및 태그를 관리할 수 있도록 허용합니다.  | 이 옵션을 비활성화하면 캐셔가 POS에서 제품 카테고리 또는 태그를 생성, 편집 또는 삭제하는 것을 방지합니다.                              |
| `publish_shop_orders`     | 캐셔가 상점 주문을 게시할 수 있도록 허용합니다.              | 이 옵션을 비활성화하면 캐셔가 POS에서 새로운 상점 주문을 최종화하고 처리하는 것을 방지합니다.                                            |
| `read_private_products`   | 캐셔가 비공개 제품의 세부 정보를 접근할 수 있도록 허용합니다. | 이 옵션을 비활성화하면 캐셔가 POS에서 비공식적으로 비공개 제품의 세부 정보를 보는 것을 방지합니다.                                        |
| `read_private_shop_coupons` | 캐셔가 비공식 상점 쿠폰의 세부 정보를 접근할 수 있도록 허용합니다. | 이 옵션을 비활성화하면 캐셔가 POS에서 비공식적으로 비공식 쿠폰의 세부 정보를 보는 것을 방지합니다.                                      |
| `read_private_shop_orders`  | 캐셔가 비공식 상점 주문의 세부 정보를 접근할 수 있도록 허용합니다. | 이 옵션을 비활성화하면 캐셔가 POS에서 비공식적으로 비공식 상점 주문의 세부 정보를 보는 것을 방지합니다.                                 |
| `read`                    | WordPress Admin에 접근할 수 있도록 허용합니다.                | 이 옵션을 비활성화하면 캐셔가 WordPress 대시보드와 사용자 프로필 페이지에 접근할 수 없습니다.                                           |

각 역할에 적절한 권한을 할당함으로써 POS의 다양한 사용자에 대한 접근 및 권한을 세부적으로 조정할 수 있습니다. 이를 통해 누구가 특정 작업을 수행할 수 있는지, 민감한 정보에 접근할 수 있는지, 상점 변경을 할 수 있는지를 제어할 수 있습니다.

WordPress에서 사용자 역할 및 권한 관리에 대한 더 자세한 정보를 보려면 [WordPress Codex](https://codex.wordpress.org/Roles_and_Capabilities)를 참조할 수 있습니다.
