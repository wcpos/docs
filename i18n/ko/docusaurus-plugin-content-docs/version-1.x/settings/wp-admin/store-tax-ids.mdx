매장 세금 식별번호는 WCPOS가 POS 영수증과 청구서에 인쇄하는 사업자 세금 식별번호로, VAT 번호, ABN, GSTIN, EIN 또는 각 지역의 사업자등록번호가 해당됩니다.

WooCommerce에는 이러한 항목을 위한 기본 필드가 없으며, 많은 지역에서 두 개 이상의 식별번호가 필요합니다. 예를 들어 독일 사업자는 부가가치세 식별번호(USt-IdNr.), 세금번호(Steuernummer), **그리고** 상업등기번호(HRB)를 인쇄해야 할 수 있고, 이탈리아 사업자는 Codice Fiscale과 Partita IVA를 모두 인쇄합니다. 또한 많은 기업이 VAT ID와 함께 현지 등록번호를 표시합니다. 매장 세금 식별번호 필드를 사용하면 필요한 만큼 항목을 추가할 수 있습니다.

**WP Admin > POS > 설정 > 일반**의 **매장 세금 식별번호** 섹션에서 확인할 수 있습니다.

{/* Screenshot: wp-admin-store-tax-ids.png */}

## 세금 식별번호 추가 {#adding-a-tax-id}

1. **WP Admin > POS > 설정 > 일반**으로 이동하여 **매장 세금 식별번호**까지 스크롤합니다.
2. **세금 식별번호 추가**를 클릭합니다.
3. **유형**을 선택하고 **값**을 입력한 후, 필요에 따라 **국가**와 사용자 지정 **라벨**을 설정합니다.
4. 인쇄해야 하는 각 식별번호에 대해 이 과정을 반복합니다.
5. 행을 삭제하려면 **삭제**를 사용하세요.

:::info 행은 자동으로 저장됩니다
이 섹션에는 저장 버튼이 없습니다. 각 행은 **값** 필드에서 포커스가 벗어날 때 저장됩니다. 필드 바깥을 클릭하거나 탭 키를 눌러 저장하세요.
:::

### 첫 번째 행은 매장 국가를 사용합니다 {#the-first-row-uses-your-store-country}

첫 번째 세금 ID를 추가하면 WCPOS가 **국가**를 WooCommerce 기본 국가로 자동 입력하고 일치하는 **유형**을 선택합니다. 따라서 단일 국내 세금 ID만 필요한 사업체는 별도로 선택할 필요가 없습니다. 기존 행은 변경되지 않습니다.

유형과 국가는 편집 시 동기화됩니다. **유형**을 변경하면 **국가**가 해당 유형의 본국으로 업데이트되고, **국가**를 변경하면 **유형**이 해당 국가의 일반적인 식별자로 전환됩니다.

### 사용 가능한 유형 {#available-types}

유형 목록에는 일반적인 고객 대상 식별자와 사업체가 영수증에 인쇄하는 사업자등록번호가 포함되어 있습니다:

| 지역 | 유형 |
|--------|-------|
| 유럽연합 | EU VAT |
| 영국 | GB VAT, GB 회사 번호 |
| 독일 | 부가가치세 ID(USt-IdNr.), 세금 번호(Steuernummer), 상업등기부 번호(HRB) |
| 프랑스 | SIRET, SIREN |
| 기타 유럽 | 네덜란드 KVK, 이탈리아 세금 코드(Codice Fiscale), 이탈리아 부가가치세 번호(Partita IVA), 스페인 NIF, 스위스 UID |
| 아메리카 | 미국 EIN, 캐나다 GST/HST, 브라질 CPF, 브라질 CNPJ, 아르헨티나 CUIT |
| 아시아 및 중동 | 인도 GSTIN, 사우디아라비아 VAT |
| 호주 | ABN |
| 일반 | 기타 |

**값** 필드에는 형식 예시가 플레이스홀더 텍스트로 표시됩니다(예: 미국 EIN의 경우 `12-3456789`). 이는 참고용일 뿐이며, WCPOS는 예상 형식과 일치하지 않는 값도 거부하지 않으므로 일반적이지 않지만 유효한 식별자도 정상적으로 저장됩니다.

## 영수증에 인쇄되는 내용 {#what-prints-on-receipts}

추가한 각 세금 ID는 영수증과 인보이스에 순서대로 한 줄씩 인쇄됩니다. WCPOS는 유형에 대해 현지화된 라벨("VAT ID", "ABN", "GSTIN" 등)을 사용하며, 사용자 지정 **라벨**을 설정한 경우 해당 라벨이 사용됩니다. 두 행의 값이 동일한 경우 중복 항목은 자동으로 제거됩니다.

기본 제공되는 영수증 템플릿은 이미 전체 목록을 렌더링합니다. 단일 `store.tax_id` 값을 참조하는 **사용자 지정 템플릿**을 사용 중인 경우, 기존대로 작동하지만 첫 번째 항목만 인쇄됩니다. 대신 목록을 반복하도록 변경하세요:

```html
{{#store.tax_ids}}{{label}}: {{value}}{{/store.tax_ids}}
```

## 매장별 세금 ID (Pro) {#per-store-tax-ids-pro}

:::info Pro 기능
[WCPOS Pro](/getting-started/pro-license)를 사용하면 전체 사이트에 한 번만 설정하는 대신 **매장별로** 세금 ID를 설정할 수 있습니다. 각 매장에 자체 VAT/EIN/GSTIN이 있거나, 체인점이 서로 다른 식별자를 사용하는 여러 지역에서 운영되는 경우에 유용합니다.
:::

매장별 세금 ID는 **WP Admin > POS > 매장 > 매장 편집**의 사이드바에 있는 **세금 ID** 섹션에서 설정합니다. 매장의 자체 목록이 비어 있으면 일반 탭에서 여기에 구성한 사이트 전체 매장 세금 ID를 사용합니다. 매장에 항목을 하나라도 추가하면 해당 매장에는 자체 목록만 인쇄됩니다.
