## 바코드 감지 작동 방식 {#how-barcode-detection-works}

대부분의 바코드 스캐너는 기기에 연결된 키보드처럼 동작합니다. 바코드를 스캔하면 POS는 일반적인 타이핑보다 문자가 더 빠르게 입력되었는지 감지합니다. 이러한 "빠른 키 입력"을 사용해 해당 입력을 바코드 스캔으로 식별합니다.

## 설정 {#settings}

| 설정 | 설명 |
|---------|-------------|
| **평균 입력 시간** | 바코드로 간주되기 위해 입력이 얼마나 빨라야 하는지(밀리초)입니다. 값이 낮을수록 더 빠른 입력이 필요합니다 |
| **최소 길이** | 바코드로 처리되기 위해 연속된 문자열이 가져야 하는 길이입니다 |
| **접두사 제거** | 스캔한 바코드의 시작 부분에서 제거할 문자 |
| **접미사 제거** | 스캔한 바코드의 끝 부분에서 제거할 문자 |

## 접두사/접미사 제거 {#prefixsuffix-removal}

스캐너가 접두사나 접미사 같은 추가 문자를 붙이는 경우, 기본 바코드만 남도록 자동으로 제거할 수 있습니다.

**예:** 스캐너가 모든 바코드 앞에 "A"를, 뒤에 "Z"를 추가하는 경우 다음과 같이 설정합니다:
- 접두사 제거: `A`
- 접미사 제거: `Z`

## 스캐너 테스트 {#test-your-scanner}

설정 아래에는 **바코드 스캔 테스트** 패널이 있습니다. 스캐너를 보정하고 접두사/접미사/길이 설정이 올바른지 확인하는 가장 빠른 방법입니다:

- **키 입력 이벤트** — 스캐너가 전송하는 원시 문자를 표시하므로, 스캐너가 추가하는 접두사나 접미사를 확인할 수 있습니다.
- **감지된 바코드** — 설정을 적용한 *후* WCPOS가 감지한 최종 값을 표시합니다.

알고 있는 바코드를 스캔하고 두 필드를 비교합니다. **감지된 바코드**가 비어 있으면 스캔이 인식되지 않은 것입니다 — **평균 입력 시간** 또는 **최소 길이**를 조정한 뒤 다시 시도하십시오. 추가 문자가 표시되면 해당하는 **접두사 제거** / **접미사 제거**를 설정하십시오.

## 문제 해결 {#troubleshooting}

<Accordion>
  <AccordionItem question="바코드가 감지되지 않음">

  - **스캐너가 너무 느림**: "평균 입력 시간" 값을 늘려 보세요
  - **바코드가 너무 짧음**: "최소 길이" 값을 줄여 보세요
  - **추가 문자**: 스캐너가 접두사/접미사를 추가하는지 확인하세요

  </AccordionItem>

  <AccordionItem question="일반 입력이 바코드로 감지됨">

  - **입력이 너무 빠름**: "평균 입력 시간" 값을 줄여 보세요
  - **입력이 너무 김**: "최소 길이" 값을 늘려 보세요

  </AccordionItem>
</Accordion>
