## 바코드 감지 작동 방식 {#how-barcode-detection-works}

대부분의 바코드 스캐너는 기기에 연결된 키보드처럼 작동합니다. 바코드를 스캔하면 POS는 문자가 정상 입력보다 빠르게 입력되었음을 감지합니다. 이러한 "빠른 키 입력"을 사용하여 입력을 바코드 스캔으로 식별합니다.

## 설정 {#settings}

| 설정 | 설명 |
|------|------|
| **평균 입력 시간** | 바코드로 간주되기 위해 입력해야 할 속도(밀리초 단위)입니다. 값이 낮을수록 더 빠른 입력이 필요합니다. |
| **최소 길이** | 연속적인 문자 문자열이 바코드로 간주되기 위해 필요한 길이입니다. |
| **접두사 제거** | 스캔한 바코드의 시작 부분에서 제거할 문자입니다. |
| **접미사 제거** | 스캔한 바코드의 끝 부분에서 제거할 문자입니다. |

## 접두사/접미사 제거 {#prefixsuffix-removal}

스캐너가 추가 문자를 추가하는 경우(접두사 또는 접미사와 같은), 기본 바코드만 남도록 자동으로 제거할 수 있습니다.

**예:** 스캐너가 모든 바코드 앞에 "A"를 추가하고 뒤에 "Z"를 추가하는 경우, 다음과 같이 구성합니다:
- 접두사 제거: `A`
- 접미사 제거: `Z`

## 문제 해결 {#troubleshooting}

<Accordion>
  <AccordionItem question="바코드가 감지되지 않음">

  - **스캐너가 너무 느림**: "평균 입력 시간" 값을 증가시켜 보십시오.
  - **바코드가 너무 짧음**: "최소 길이" 값을 감소시켜 보십시오.
  - **추가 문자**: 스캐너가 접두사/접미사를 추가하고 있는지 확인하십시오.

  </AccordionItem>

  <AccordionItem question="정상 입력이 바코드로 감지됨">

  - **입력이 너무 빠름**: "평균 입력 시간" 값을 감소시켜 보십시오.
  - **입력이 너무 김**: "최소 길이" 값을 증가시켜 보십시오.

  </AccordionItem>
</Accordion>
