:::warning 중요
초기 로드 시 POS는 WooCommerce의 기본 세금 설정을 가져옵니다. 일반적으로 사용자는 특정 필요가 없는 한 WooCommerce와 동일한 설정을 유지해야 합니다. 서버와 다른 세금 설정을 가지면 POS와 WooCommerce가 서로 다른 세금 값을 계산하게 되어 불일치와 주문 동기화 문제를 일으킬 수 있습니다.
:::

## 모든 세금 세율 보기

**모든 세금 세율 보기** 버튼을 클릭하여 POS에 다운로드된 모든 세금 세율을 확인합니다. 여기에는:

- 세금 세율 이름
- 세금 비율
- 적용 지역/위치

이는 올바른 세금 세율이 로드되었는지 확인하고 세금 계산 문제를 디버깅하는 데 유용합니다.

## 세금 계산 설정

### 세금 포함 가격 입력

제품 가격에 세금이 포함되는지 여부를 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **예** | 제품 가격에 세금이 포함됩니다. POS는 총 가격에서 역산하여 세금을 계산합니다. |
| **아니요** | 제품 가격에 세금이 포함되지 않습니다. POS는 입력된 가격에 세금을 추가합니다. |

### 세금 계산 기준

세금 계산에 사용되는 주소를 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **고객의 배송 주소** | 주문이 배송될 위치를 기준으로 한 세금 세율입니다. |
| **고객의 청구 주소** | 고객의 청구 주소를 기준으로 한 세금 세율입니다. |
| **상점 기본 주소** | 상점의 실제 위치를 기준으로 한 세금 세율입니다. |

### 배송 세금 클래스

배송 비용이 세금이 부과되는 방법을 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **장바구니 항목을 기준으로 한 배송 세금 클래스** | 장바구니에 있는 항목의 세금 클래스를 기준으로 배송 세금이 계산됩니다. |
| **기본 세금** | 배송은 기본 세금률로 세금이 부과됩니다. |
| **감세율** | 배송은 감세된 세금률로 세금이 부과됩니다. |
| **세금 없음** | 배송 비용에는 세금이 부과되지 않습니다. |

### 반올림

| 옵션 | 설명 |
|--------|-------------|
| **소계 수준에서 세금 반올림** | 세금은 전체 소계에 대해 계산되고 반올림되며, 각 개별 항목의 세금을 반올림하지 않습니다. |

## 표시 설정

### 상점에서 가격 표시

| 옵션 | 설명 |
|--------|-------------|
| **세금 포함** | 표시되는 가격에 세금 금액이 포함됩니다. |
| **세금 제외** | 표시되는 가격에는 세금이 포함되지 않습니다. |

### 장바구니 및 결제 중 가격 표시

| 옵션 | 설명 |
|--------|-------------|
| **세금 포함** | 장바구니 및 결제 가격에 세금이 포함됩니다. |
| **세금 제외** | 장바구니 및 결제 가격에는 세금이 포함되지 않습니다. |

### 가격 표시 접미사

세금 상태를 나타내기 위해 가격 뒤에 표시되는 텍스트(예: "부가세 포함" 또는 "부가세 제외").

### 세금 총액 표시

| 옵션 | 설명 |
|--------|-------------|
| **단일 총액으로** | 모든 세금이 하나의 총액 줄로 합산됩니다. |
| **항목별** | 각 세금 세율이 별도의 항목으로 표시됩니다. |

## 서버 설정 복원

클릭하여 서버에서 현재 WooCommerce 세금 설정을 가져오고 적용합니다.

:::tip 모범 사례
"서버 설정 복원"을 정기적으로 사용하여 WooCommerce 상점의 세금 구성을 POS와 동기화 상태로 유지하세요. 특히 WordPress 관리에서 세금 설정을 변경한 후에 유용합니다.
:::
