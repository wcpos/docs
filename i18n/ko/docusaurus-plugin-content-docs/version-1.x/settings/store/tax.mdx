:::warning 중요
초기 로드 시, POS는 WooCommerce에서 기본 세금 설정을 가져옵니다. 일반적으로 사용자는 특정한 필요가 없는 한 WooCommerce와 동일한 설정을 유지해야 합니다. 서버와 다른 세금 설정을 가지고 있으면 POS와 WooCommerce가 서로 다른 세금 값을 계산할 가능성이 높아져 불일치와 주문 동기화 문제를 일으킬 수 있습니다.
:::

## 모든 세금율 보기

**모든 세금율 보기** 버튼을 클릭하여 POS에 다운로드된 모든 세금율을 확인합니다. 여기에는 다음이 포함됩니다:

- 세금율 이름
- 세금율 백분율
- 적용 가능한 지역/위치

이것은 올바른 세금율이 로드되었는지 확인하고 세금 계산 문제를 디버깅하는 데 유용합니다.

## 세금 계산 설정

### 세금 포함 가격 입력

상품 가격에 세금이 포함되는지를 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **예** | 상품 가격에 세금이 포함됩니다. POS는 총 가격에서 역으로 작업하여 세금을 계산합니다. |
| **아니오** | 상품 가격에 세금이 포함되지 않습니다. POS는 입력된 가격 위에 세금을 추가합니다. |

### 세금 계산 기준

세금 계산에 사용되는 주소를 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **고객 배송 주소** | 주문이 배송될 주소에 따라 세금율이 결정됩니다. |
| **고객 청구 주소** | 고객의 청구 주소에 따라 세금율이 결정됩니다. |
| **상점 기본 주소** | 상점의 물리적 위치에 따라 세금율이 결정됩니다. |

### 배송 세금 클래스

배송 비용이 과세되는 방식을 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **장바구니 항목에 기반한 배송 세금 클래스** | 배송 세금은 장바구니의 항목 세금 클래스를 기준으로 계산됩니다. |
| **표준** | 배송은 표준 세금율로 과세됩니다. |
| **감면 세율** | 배송은 감면된 세금율로 과세됩니다. |
| **제로 세율** | 배송 비용에는 세금이 적용되지 않습니다. |

### 반올림

| 옵션 | 설명 |
|--------|-------------|
| **소계 수준에서 세금 반올림** | 세금은 전체 소계에 대해 계산된 후 반올림되며, 각 개별 항목의 세금을 반올림하지 않습니다. |

## 표시 설정

### 상점에서 가격 표시

| 옵션 | 설명 |
|--------|-------------|
| **세금 포함** | 표시된 가격은 세금 금액을 포함합니다. |
| **세금 제외** | 표시된 가격은 세금을 포함하지 않습니다. |

### 장바구니 및 결제 중 가격 표시

| 옵션 | 설명 |
|--------|-------------|
| **세금 포함** | 장바구니 및 결제 가격은 세금을 포함합니다. |
| **세금 제외** | 장바구니 및 결제 가격은 세금을 포함하지 않습니다. |

### 가격 표시 접미사

세금 상태를 나타내기 위해 가격 뒤에 나타나는 텍스트 (예: "부가세 포함" 또는 "부가세 제외").

### 세금 합계 표시

| 옵션 | 설명 |
|--------|-------------|
| **단일 합계로 표시** | 모든 세금이 하나의 총 계선으로 결합됩니다. |
| **세부 항목으로 표시** | 각 세금율이 별도의 항목으로 표시됩니다. |

## 서버 설정 복원

클릭하여 현재 WooCommerce 세금 설정을 서버에서 가져오고 적용합니다.

:::tip 모범 사례
"서버 설정 복원"을 정기적으로 사용하여 POS 세금 구성을 WooCommerce 상점과 동기화 상태로 유지하고, 특히 WordPress 관리에서 세금 설정을 변경한 후에 사용하십시오.
:::
