:::warning 중요
초기 로드 시, POS는 WooCommerce의 기본 세금 설정을 가져옵니다. 일반적으로 사용자는 특별한 필요가 없는 한 WooCommerce와 동일한 설정을 유지해야 합니다. 서버와 다른 세금 설정은 POS와 WooCommerce가 서로 다른 세금 값을 계산할 가능성이 거의 확실하며, 이는 주문 동기화 문제를 일으킬 수 있습니다.
:::

## 모든 세금 요율 보기

**모든 세금 요율 보기** 버튼을 클릭하여 POS에 다운로드된 모든 세금 요율을 확인합니다. 이를 통해 다음을 볼 수 있습니다:

- 세금 요율 이름
- 세금 요율 백분율
- 적용 가능한 지역/위치

이는 올바른 세금 요율이 로드되었는지 확인하고 세금 계산 문제를 디버깅하는 데 유용합니다.

## 세금 계산 설정

### 세금이 포함된 가격 입력

제품 가격에 세금이 포함되는지 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **예** | 제품 가격에 세금 포함. POS는 총 가격에서 역으로 세금을 계산 |
| **아니오** | 제품 가격에 세금 미포함. POS는 입력된 가격 위에 세금을 추가 |

### 세금 계산 기준

세금 계산에 사용되는 주소를 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **고객 배송 주소** | 주문이 배송될 곳을 기준으로 한 세금 요율 |
| **고객 청구 주소** | 고객의 청구 주소를 기준으로 한 세금 요율 |
| **매장 기본 주소** | 매장의 물리적 위치를 기준으로 한 세금 요율 |

### 배송 세금 분류

배송 비용에 대한 세금이 부과되는 방식을 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **장바구니 항목 기준 배송 세금 분류** | 배송 세금은 장바구니의 항목 세금 분류에 따라 계산 |
| **표준** | 배송은 표준 세금 요율로 과세 |
| **감면 요율** | 배송은 감면 세금 요율로 과세 |
| **제로 요율** | 배송 비용에 세금 없음 |

### 반올림

| 옵션 | 설명 |
|--------|-------------|
| **소계 수준에서 세금 반올림** | 각 개별 품목이 아닌 전체 소계에 대해 세금을 계산하고 반올림 |

## 표시 설정

### 상점에서 가격 표시

| 옵션 | 설명 |
|--------|-------------|
| **세금 포함** | 세금 금액을 포함한 가격 표시 |
| **세금 제외** | 세금이 제외된 가격 표시 |

### 장바구니 및 결제 중 가격 표시

| 옵션 | 설명 |
|--------|-------------|
| **세금 포함** | 장바구니 및 결제 가격에 세금 포함 |
| **세금 제외** | 장바구니 및 결제 가격에서 세금 제외 |

### 가격 접미사

세금 상태를 나타내는 가격 뒤에 표시되는 텍스트 (예: "부가세 포함" 또는 "부가세 제외").

### 세금 총액 표시

| 옵션 | 설명 |
|--------|-------------|
| **단일 합계로** | 모든 세금을 하나의 합계 라인으로 결합 |
| **항목별** | 각 세금 요율을 별도의 라인 항목으로 표시 |

## 서버 설정 복원

클릭하여 현재 WooCommerce 세금 설정을 서버에서 가져와 적용합니다.

:::tip 모범 사례
WordPress 관리자에서 세금 설정을 변경한 후 특히 POS 세금 구성을 WooCommerce 스토어와 동기화된 상태로 유지하기 위해 "서버 설정 복원"을 정기적으로 사용하세요.
:::
