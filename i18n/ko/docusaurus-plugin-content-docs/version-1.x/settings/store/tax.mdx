:::warning 중요
초기 로드 시 POS는 WooCommerce에서 기본 세금 설정을 가져옵니다. 일반적으로 특별한 필요가 없는 한 WooCommerce와 동일한 설정을 유지하는 것이 좋습니다. 서버와 세금 설정이 다르면 POS와 WooCommerce가 서로 다른 세금 값을 계산하게 되어 주문 동기화에 불일치와 문제가 발생할 수 있습니다.
:::

## 모든 세율 보기 {#view-all-tax-rates}

**모든 세율 보기** 버튼을 클릭하면 POS에 다운로드된 모든 세율을 확인할 수 있습니다. 표시되는 항목은 다음과 같습니다:

- 세율 이름
- 세율 백분율
- 적용 가능한 지역/위치

올바른 세율이 로드되었는지 확인하고 세금 계산 문제를 디버깅하는 데 유용합니다.

## 세금 계산 설정 {#tax-calculation-settings}

### 세금 포함 가격 입력 {#prices-entered-with-tax}

상품 가격에 세금이 포함되어 있는지 여부를 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **예** | 상품 가격에 세금이 포함되어 있습니다. POS는 총 가격에서 역산하여 세금을 계산합니다 |
| **아니오** | 상품 가격에 세금이 포함되어 있지 않습니다. POS는 입력된 가격에 세금을 추가합니다 |

### 세금 계산 기준 {#calculate-tax-based-on}

세금 계산에 사용되는 주소를 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **고객 배송 주소** | 주문이 배송될 주소를 기준으로 세율이 적용됩니다 |
| **고객 청구 주소** | 고객의 청구 주소를 기준으로 세율이 적용됩니다 |
| **매장 기본 주소** | 매장의 실제 위치를 기준으로 세율이 적용됩니다 |

### 배송 세금 클래스 {#shipping-tax-class}

배송비에 대한 세금 적용 방식을 결정합니다:

| 옵션 | 설명 |
|--------|-------------|
| **장바구니 상품 기준 배송 세금 클래스** | 장바구니에 담긴 상품의 세금 클래스를 기준으로 배송 세금이 계산됩니다 |
| **표준** | 표준 세율로 배송비에 세금이 적용됩니다 |
| **경감 세율** | 경감 세율로 배송비에 세금이 적용됩니다 |
| **영세율** | 배송비에 세금이 적용되지 않습니다 |

### 반올림 {#rounding}

| 옵션 | 설명 |
|--------|-------------|
| **소계 수준에서 세금 반올림** | 각 개별 항목의 세금을 반올림하는 대신, 전체 소계에 대해 세금을 계산한 후 반올림합니다 |

## 표시 설정 {#display-settings}

### 매장에서 가격 표시 {#display-prices-in-the-shop}

| 옵션 | 설명 |
|--------|-------------|
| **세금 포함** | 표시되는 가격에 세금이 포함되어 있습니다 |
| **세금 제외** | 표시되는 가격에 세금이 포함되어 있지 않습니다 |

### 장바구니 및 결제 시 가격 표시 {#display-prices-during-cart-and-checkout}

| 옵션 | 설명 |
|--------|-------------|
| **세금 포함** | 장바구니 및 결제 가격에 세금이 포함됩니다 |
| **세금 제외** | 장바구니 및 결제 가격에 세금이 제외됩니다 |

### 가격 표시 접미사 {#price-display-suffix}

세금 상태를 나타내기 위해 가격 뒤에 표시되는 텍스트입니다 (예: "inc. VAT" 또는 "ex. VAT").

### 세금 합계 표시 {#display-tax-totals}

| 옵션 | 설명 |
|--------|-------------|
| **단일 합계로 표시** | 모든 세금이 하나의 합계 항목으로 결합됩니다 |
| **항목별 표시** | 각 세율이 별도의 항목으로 표시됩니다 |

## 서버 설정 복원 {#restore-server-settings}

클릭하면 서버에서 현재 WooCommerce 세금 설정을 가져와 적용합니다.

:::tip 사용 시기 및 방법
WooCommerce는 세금 설정의 기준 소스입니다 — POS는 이를 동일하게 반영해야 합니다. **서버 설정 복원**은 매장의 최신 설정을 가져오는 안전한 방법이며, `WP Admin > WooCommerce > 설정 > Tax` 에서 세금 설정을 변경한 후 POS에 반영하려는 경우 올바른 조치입니다.

한 가지 주의할 점: POS 측 세금 설정을 서버와 다르게 **의도적으로** 변경한 경우(고급 설정), 서버 설정 복원을 사용하면 해당 변경 사항이 덮어쓰기됩니다. 그러한 변경을 한 적이 있는지 확실하지 않다면, 거의 확실히 하지 않은 것입니다.
:::

## 일반적인 문제 {#common-issues}

### POS와 WooCommerce에서 다른 합계가 표시되는 경우 {#pos-and-woocommerce-showing-different-totals}

가장 흔한 세금 문제는 POS 세금 설정과 WooCommerce 세금 설정 간의 불일치입니다. 이 설정이 서로 다르면, 동일한 주문이라도 어디서 처리하느냐에 따라 다른 합계가 계산됩니다. 항상 POS 설정을 동기화 상태로 유지하세요 — `WP Admin > WooCommerce > 설정 > Tax` 에서 변경한 후에는 **서버 설정 복원**을 사용하세요.

### 세금 포함 가격 혼동 {#tax-inclusive-pricing-confusion}

"세금 포함 가격 입력"이 **예**로 설정되어 있지만 "매장 내 가격 표시"가 **세금 제외**로 설정된 경우, 표시되는 가격이 입력한 가격보다 낮게 나타납니다. 이는 정상적인 동작입니다 — POS가 표시를 위해 세금 부분을 제거하는 것입니다 — 하지만 계산원이 표시 가격을 기대하고 있다면 혼란을 줄 수 있습니다.

### 고객 주소와 세금 계산 {#customer-address-and-tax-calculation}

"세금 계산 기준"이 **고객 배송 주소** 또는 **고객 청구 주소**로 설정된 경우, POS에서 올바른 세율을 조회하려면 유효한 주소가 필요합니다. 방문 고객의 주소가 등록되어 있지 않은 경우, WCPOS는 **매장 기본 주소**로 대체하여 계산합니다. 이는 일반적으로 매장 내 판매에 적합한 동작이지만, 특정 고객에 대해 예상치 못한 세율이 적용되는 경우 해당 고객의 주소 정보를 확인하세요.

## 문제 해결 {#troubleshooting}

### 세금 금액이 올바르지 않음 {#tax-amounts-are-incorrect}

1. POS의 **세금 설정**으로 이동하여 **서버 설정 복원**을 클릭하면 WooCommerce에서 최신 구성을 가져옵니다.
2. **모든 세율 보기**를 클릭하여 예상 세율이 표시되는지 확인합니다. 세율이 누락된 경우 `WP Admin > WooCommerce > 설정 > Tax > Standard rates` 에서 올바르게 구성되어 있는지 확인하십시오.
3. 상품의 세금 클래스를 확인하십시오 — "영세율" 세금 클래스가 지정된 상품은 설정에 관계없이 세금이 적용되지 않습니다.

### 세금이 0으로 표시됨 {#tax-is-showing-as-zero}

- WooCommerce에서 세금이 활성화되어 있는지 확인합니다(`WP Admin > WooCommerce > 설정 > General > Enable tax rates and calculations`).
- POS에서 **모든 세율 보기**를 사용하여 세율이 다운로드되었는지 확인합니다. 목록이 비어 있으면 전체 동기화를 시도하십시오.
- 상품의 세금 상태가 "과세"로 설정되어 있는지 확인합니다("없음" 또는 "배송만"이 아닌지 확인).

### 반올림 차이 {#rounding-discrepancies}

"소계 수준에서 세금 반올림"이 한쪽 시스템에서만 활성화되어 있으면 POS와 WooCommerce 간에 소액의 반올림 차이(1~2원)가 발생할 수 있습니다. 이 설정이 양쪽 모두에서 일치하는지 확인하십시오.
