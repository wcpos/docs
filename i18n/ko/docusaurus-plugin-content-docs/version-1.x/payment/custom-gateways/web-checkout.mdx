# 웹 결제 게이트웨이

웹 결제 게이트웨이를 사용하면 고객이 웹 스토어의 체크아웃 시스템을 통해 POS 거래를 완료할 수 있습니다. 이 게이트웨이는 매장 내 POS와 온라인 결제 방법 사이에 원활한 연결을 만들어, 고객이 자신의 기기로 결제하거나 웹 전용 결제 방법을 사용해야 하는 상황에 완벽합니다.

## 기능

- **원활한 통합**: POS에서 웹 체크아웃으로 직접 연결
- **모든 결제 방법**: 웹 스토어의 모든 활성화된 결제 게이트웨이 접근
- **고객 제어**: 고객이 자신의 기기와 선호하는 방법으로 결제
- **주문 동기화**: POS와 웹 스토어 간 자동 주문 업데이트
- **유연한 워크플로**: POS 워크플로를 유지하면서 온라인으로 거래 완료

## 설치

### 1단계: 다운로드 및 설치

1. [웹 체크아웃 게이트웨이 릴리스 페이지](https://github.com/wcpos/web-checkout-gateway/releases) 방문
2. 최신 **woocommerce-pos-web-checkout-gateway.zip** 파일 다운로드
3. WordPress 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일 업로드 및 플러그인 활성화

### 2단계: WooCommerce POS에서 활성화

1. `WP 관리자 > WooCommerce POS > 설정 > 체크아웃`로 이동
2. 목록에서 **WooCommerce POS 웹 체크아웃 게이트웨이** 찾기
3. POS에서 사용할 게이트웨이 활성화
4. 설정 저장

:::note
이 게이트웨이는 기존 WooCommerce 결제 방법을 활용하므로, 최소한 하나의 웹 결제 게이트웨이(Stripe, PayPal 등)가 제대로 구성되어 있는지 확인하세요.
:::

## 사용법

### 웹 체크아웃 시작

1. **상품 추가**: POS에서 장바구니에 상품 추가
2. **게이트웨이 선택**: 결제 방법으로 "웹 체크아웃" 선택
3. **링크 생성**: 시스템이 주문에 대한 고유 체크아웃 링크 생성
4. **고객 접근**: 고객이 링크를 클릭하여 웹 체크아웃 접근
5. **온라인 결제**: 고객이 웹 스토어 결제 방법으로 결제 완료

### 결제 프로세스

웹 체크아웃 게이트웨이는 다음 워크플로를 따릅니다:

1. **주문 생성**: POS가 WooCommerce에 대기 중 주문 생성
2. **체크아웃 링크**: 시스템이 안전하고 고유한 결제 링크 생성
3. **고객 리디렉션**: 고객을 웹 체크아웃 페이지로 안내
4. **결제 선택**: 고객이 사용 가능한 웹 결제 방법 선택
5. **결제 처리**: 표준 WooCommerce 체크아웃 프로세스가 결제 처리
6. **주문 완료**: 성공적인 결제 시 주문 상태 자동 업데이트

### POS 워크플로

웹 체크아웃 시작 후:

1. **상태 모니터링**: 결제 상태를 모니터링하기 위해 POS 주문 화면 열어두기
2. **고객 결제**: 고객이 자신의 기기나 태블릿에서 결제 완료
3. **자동 업데이트**: 결제 수신 시 주문 상태 실시간 업데이트
4. **결제 처리**: POS에서 "결제 처리" 버튼을 클릭하여 영수증으로 진행
5. **영수증 생성**: 일반적으로 영수증 생성 및 인쇄

## 사용 사례

### 완벽한 상황

- **고객 선호도**: 자신의 결제 앱이나 카드를 사용하기 원하는 고객
- **복잡한 결제**: POS에서 사용할 수 없는 결제 방법이 필요한 거래
- **분할 결제**: 여러 결제 방법을 사용하려는 고객
- **로열티 프로그램**: 웹 기반 로열티 포인트 상환 접근
- **기프트 카드**: 디지털 기프트 카드나 스토어 크레딧이 있는 고객
- **국제 카드**: 웹 게이트웨이를 통해 더 잘 작동하는 결제 방법

### 워크플로 예시

**소매점**
1. 고객이 매장에서 쇼핑하고 계산대로 이동
2. 고객이 모바일 지갑 앱으로 결제 선호
3. 직원이 웹 체크아웃 게이트웨이 선택
4. 고객이 QR 코드 스캔 또는 휴대폰에서 링크 클릭
5. 고객이 선호하는 방법으로 결제 완료
6. 직원이 영수증 처리 및 거래 완료

**서비스 사업**
1. 서비스 완료 및 POS에 요금 추가
2. 고객이 회사 신용카드로 결제 희망
3. 웹 체크아웃 링크 생성
4. 고객이 안전한 웹 양식에 결제 세부 정보 입력
5. 웹 게이트웨이를 통해 결제 처리
6. 주문 완료 및 영수증 생성
