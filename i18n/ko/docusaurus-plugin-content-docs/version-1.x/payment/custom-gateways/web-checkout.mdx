# 웹 체크아웃 게이트웨이 {#web-checkout-gateway}

웹 체크아웃 게이트웨이는 고객이 귀하의 웹 스토어 체크아웃 시스템을 사용하여 POS 거래를 완료할 수 있도록 합니다. 이 게이트웨이는 매장 내 POS와 온라인 결제 수단 간의 매끄러운 연결을 생성하며, 고객이 자신의 장치를 사용하여 결제하기를 선호하는 상황이나 웹 전용 결제 방법에 접근해야 할 때에 적합합니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [웹 체크아웃 게이트웨이 릴리스 페이지](https://github.com/wcpos/web-checkout-gateway/releases)에 방문
2. 최신 **woocommerce-pos-web-checkout-gateway.zip** 파일 다운로드
3. WordPress 관리에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일을 업로드하고 플러그인을 활성화

  </StepCard>
  <StepCard number="2" title="WooCommerce POS에서 사용 설정" isLast>

1. `WP 관리자 > WooCommerce POS > 설정 > 체크아웃`으로 이동
2. 목록에서 **WooCommerce POS 웹 체크아웃 게이트웨이** 찾기
3. POS에서 사용하기 위해 게이트웨이 사용 설정
4. 설정 저장

:::note
이 게이트웨이는 기존 WooCommerce 결제 방법을 활용하므로 적어도 하나의 웹 결제 게이트웨이가 제대로 구성되어 있는지 확인하십시오(Stripe, PayPal 등).
:::

  </StepCard>
</Steps>

## 사용법 {#usage}

### 웹 체크아웃 시작하기 {#initiating-web-checkout}

1. **상품 추가**: POS에서 장바구니에 상품 추가
2. **게이트웨이 선택**: 결제 방법으로 "웹 체크아웃" 선택
3. **링크 생성**: 시스템이 주문을 위한 고유 체크아웃 링크 생성
4. **고객 접근**: 고객이 링크를 클릭하여 웹 체크아웃 접근
5. **온라인 결제**: 고객이 웹 스토어 결제 방법으로 결제 완료

### 결제 과정 {#payment-process}

웹 체크아웃 게이트웨이는 다음과 같은 워크플로우를 따릅니다:

1. **주문 생성**: POS가 WooCommerce에서 보류 중인 주문을 생성
2. **체크아웃 링크**: 시스템이 안전하고 고유한 결제 링크 생성
3. **고객 리디렉션**: 고객이 웹 체크아웃 페이지로 리디렉션
4. **결제 선택**: 고객이 사용 가능한 웹 결제 방법 중 선택
5. **결제 처리**: 표준 WooCommerce 체크아웃 프로세스가 결제를 처리
6. **주문 완료**: 결제가 성공적으로 이루어진 후 주문 상태 자동 업데이트

### POS 워크플로우 {#pos-workflow}

웹 체크아웃을 시작한 후:

1. **상태 모니터링**: 결제 상태를 모니터링하기 위해 POS 주문 화면을 계속 열어 두기
2. **고객 결제**: 고객이 자신의 장치 또는 귀하의 태블릿에서 결제를 완료
3. **자동 업데이트**: 결제가 수신되면 주문 상태 실시간 업데이트
4. **결제 처리**: 영수증으로 계속 진행하기 위해 POS에서 "결제 처리" 버튼 클릭
5. **영수증 생성**: 정상적으로 영수증 생성 및 인쇄

## 사용 사례 {#use-cases}

### 완벽한 경우 {#perfect-for}

- **고객 선호**: 자신의 결제 앱이나 카드를 사용하기를 선호하는 고객
- **복잡한 결제**: POS에서 사용할 수 없는 결제 방법이 필요한 거래
- **분할 결제**: 여러 결제 방법을 사용하려는 고객
- **로열티 프로그램**: 웹 기반의 로열티 포인트 사용 가능
- **상품권**: 디지털 상품권이나 스토어 크레딧이 있는 고객
- **국제 카드**: 웹 게이트웨이를 통해 더 좋은 성능을 발휘하는 결제 방법

## 기술 세부정보 {#technical-details}

### 주문 관리 {#order-management}

- **보류 중인 주문**: 주문은 "보류 중 결제" 상태로 생성
- **상태 업데이트**: 결제가 수신되면 자동 상태 변경
- **주문 노트**: 결제 세부정보가 주문 노트에 기록
- **재고**: 체크아웃 과정에서 재고가 예약됨

### 보안 {#security}

- **고유 링크**: 각 체크아웃 세션에는 고유하고 시간 제한이 있는 URL이 있음
- **SSL 필요**: 모든 결제 처리는 안전한 HTTPS 연결을 통해 이루어짐
- **PCI 준수**: 기존의 PCI 준수 웹 결제 게이트웨이를 활용
- **세션 관리**: 안전한 세션 처리는 무단 접근을 방지

### 결제 방법 {#payment-methods}

게이트웨이는 구성된 모든 WooCommerce 결제 방법에 대한 접근을 제공합니다:

- **신용/직불 카드**: Stripe, Square, Authorize.net 등
- **디지털 지갑**: PayPal, Apple Pay, Google Pay
- **은행 이체**: 직접 은행 결제 방법
- **지금 구매, 나중에 결제**: Klarna, Afterpay, Sezzle
- **암호화폐**: 비트코인 및 기타 암호화폐 결제 게이트웨이
- **지역 방법**: 지역별 결제 방법

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="체크아웃 링크가 작동하지 않음">

  - SSL 인증서가 제대로 설치되었는지 확인
  - WooCommerce 퍼머링크가 올바르게 구성되었는지 확인
  - 웹 결제 게이트웨이가 활성화되고 구성되었는지 확인
  - 다른 브라우저나 장치로 테스트

  </AccordionItem>

  <AccordionItem question="결제가 처리되지 않음">

  - 웹 결제 게이트웨이가 제대로 구성되었는지 확인
  - 결제 게이트웨이 로그에서 오류 메시지 확인
  - 고객의 결제 방법이 지원되는지 확인
  - 웹 체크아웃 프로세스를 독립적으로 테스트

  </AccordionItem>

  <AccordionItem question="주문이 POS에서 업데이트되지 않음">

  - 인터넷 연결 안정성 확인
  - WordPress 크론 작업이 제대로 실행되고 있는지 확인
  - POS 주문 화면 새로 고침
  - WooCommerce 관리에서 주문 상태 확인

  </AccordionItem>

  <AccordionItem question="고객이 체크아웃에 접근할 수 없음">

  - 체크아웃 링크가 만료되지 않았는지 확인
  - 고객의 장치에 인터넷 연결이 있는지 확인
  - 주문이 취소되거나 완료되지 않았는지 확인
  - 다른 장치에서 링크 테스트

  </AccordionItem>
</Accordion>

### 성능 최적화 {#performance-optimisation}

최적의 성능을 위해:

- **캐싱**: 체크아웃 페이지를 제외하도록 캐싱 플러그인을 구성
- **CDN**: CDN 설정이 체크아웃 프로세스에 간섭하지 않도록 확인
- **데이터베이스**: 주문 처리를 빠르게 하기 위해 데이터베이스 최적화
- **호스팅**: 안정적인 호스팅을 사용하여 좋은 가동 시간 보장
