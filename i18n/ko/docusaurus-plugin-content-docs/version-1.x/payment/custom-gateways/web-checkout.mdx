# 웹 체크아웃 게이트웨이 {#web-checkout-gateway}

웹 체크아웃 게이트웨이를 사용하면 고객이 웹 상점의 체크아웃 시스템을 이용하여 POS 거래를 완료할 수 있습니다. 이 게이트웨이는 매장 내 POS와 온라인 결제 방법 사이의 원활한 다리를 만들어 주며, 고객이 자신의 장치를 사용하여 결제하기를 선호하거나 웹 전용 결제 방법에 접근해야 할 때 완벽합니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [웹 체크아웃 게이트웨이 출시 페이지](https://github.com/wcpos/web-checkout-gateway/releases) 방문
2. 최신 **woocommerce-pos-web-checkout-gateway.zip** 파일 다운로드
3. WordPress 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일을 업로드하고 플러그인을 활성화

  </StepCard>
  <StepCard number="2" title="WCPOS에서 활성화" isLast>

1. `WP 관리자 > POS > 설정 > 체크아웃`으로 이동
2. 목록에서 **WCPOS 웹 체크아웃 게이트웨이** 찾기
3. POS에서 사용할 게이트웨이를 활성화
4. 설정 저장

:::note
이 게이트웨이는 기존의 WooCommerce 결제 방법을 활용하므로, 적어도 하나의 웹 결제 게이트웨이가 올바르게 구성되어 있는지 확인하세요 (Stripe, PayPal 등).
:::

  </StepCard>
</Steps>

## 사용 방법 {#usage}

### 웹 체크아웃 시작하기 {#initiating-web-checkout}

1. **상품 추가**: POS에서 장바구니에 제품 추가
2. **게이트웨이 선택**: 결제 방법으로 "웹 체크아웃" 선택
3. **링크 생성**: 시스템이 주문에 대한 고유한 체크아웃 링크를 생성
4. **고객 접근**: 고객이 링크를 클릭하여 웹 체크아웃에 접근
5. **온라인 결제**: 고객이 웹 상점 결제 방법을 사용하여 결제 완료

### 결제 과정 {#payment-process}

웹 체크아웃 게이트웨이는 다음의 워크플로를 따릅니다:

1. **주문 생성**: POS에서 WooCommerce에 보류 중인 주문 생성
2. **체크아웃 링크**: 시스템이 안전하고 고유한 결제 링크 생성
3. **고객 리디렉션**: 고객이 웹 체크아웃 페이지로 이동
4. **결제 선택**: 고객이 사용 가능한 웹 결제 방법 중에서 선택
5. **결제 처리**: 표준 WooCommerce 체크아웃 과정이 결제를 처리
6. **주문 완료**: 성공적인 결제 후 주문 상태가 자동으로 업데이트

### POS 워크플로 {#pos-workflow}

웹 체크아웃을 시작한 후:

1. **상태 모니터링**: 결제 상태를 모니터링하기 위해 POS 주문 화면을 열어두기
2. **고객 결제**: 고객이 자신의 장치 또는 태블릿에서 결제 완료
3. **자동 업데이트**: 결제가 수신되면 주문 상태가 실시간으로 업데이트
4. **결제 처리**: 영수증으로 계속 진행하기 위해 POS에서 "결제 처리" 버튼 클릭
5. **영수증 생성**: 정상적으로 영수증을 생성 및 인쇄

## 사용 사례 {#use-cases}

### 완벽한 상황 {#perfect-for}

- **고객 선호**: 자신의 결제 앱이나 카드를 사용하기를 원하는 고객
- **복잡한 결제**: POS에서 사용할 수 없는 결제 방법이 필요한 거래
- **부분 결제**: 여러 결제 방법을 사용하고자 하는 고객
- **충성 프로그램**: 웹 기반의 충성 포인트 교환 접근
- **기프트 카드**: 디지털 기프트 카드나 매장 크레딧이 있는 고객
- **국제 카드**: 웹 게이트웨이를 통해 더 잘 작동하는 결제 방법

## 기술 세부정보 {#technical-details}

### 주문 관리 {#order-management}

- **보류 중인 주문**: "보류 중 결제" 상태로 주문이 생성됨
- **상태 업데이트**: 결제가 수신되면 자동 상태 변경
- **주문 메모**: 결제 세부정보가 주문 메모에 기록됨
- **재고**: 체크아웃 과정 중 재고가 예약됨

### 보안 {#security}

- **고유 링크**: 각 체크아웃 세션에는 고유하고 시간 제한이 있는 URL이 있음
- **SSL 필요**: 모든 결제 처리는 안전한 HTTPS 연결을 통해 이루어짐
- **PCI 준수**: 기존의 PCI 준수 웹 결제 게이트웨이를 활용함
- **세션 관리**: 안전한 세션 처리가 무단 접근을 방지함

### 결제 방법 {#payment-methods}

게이트웨이는 구성된 모든 WooCommerce 결제 방법에 대한 접근을 제공합니다:

- **신용/직불 카드**: Stripe, Square, Authorize.net 등
- **디지털 지갑**: PayPal, Apple Pay, Google Pay
- **은행 송금**: 직접 은행 결제 방법
- **지금 구매, 나중에 결제**: Klarna, Afterpay, Sezzle
- **암호화폐**: 비트코인 및 기타 암호화폐 결제 게이트웨이
- **지역 방법**: 지역 특화 결제 방법

## 문제 해결 {#troubleshooting}

### 일반적인 문제들 {#common-issues}

<Accordion>
  <AccordionItem question="체크아웃 링크가 작동하지 않음">

  - SSL 인증서가 올바르게 설치되었는지 확인
  - WooCommerce의 고유 링크가 올바르게 구성되었는지 확인
  - 웹 결제 게이트웨이가 활성화되고 구성되었는지 확인
  - 다른 브라우저나 장치로 테스트

  </AccordionItem>

  <AccordionItem question="결제가 처리되지 않음">

  - 웹 결제 게이트웨이가 올바르게 구성되었는지 확인
  - 결제 게이트웨이 로그에서 오류 메시지 확인
  - 고객의 결제 방법이 지원되는지 확인
  - 웹 체크아웃 과정을 독립적으로 테스트

  </AccordionItem>

  <AccordionItem question="POS에서 주문이 업데이트되지 않음">

  - 인터넷 연결 안정성 확인
  - WordPress 크론잡이 제대로 작동하는지 확인
  - POS 주문 화면 새로 고침
  - WooCommerce 관리자에서 주문 상태 확인

  </AccordionItem>

  <AccordionItem question="고객이 체크아웃에 접근할 수 없음">

  - 체크아웃 링크가 만료되지 않았는지 확인
  - 고객의 장치가 인터넷에 연결되어 있는지 확인
  - 주문이 취소되거나 완료되지 않았는지 확인
  - 다른 장치에서 링크 테스트

  </AccordionItem>
</Accordion>

### 성능 최적화 {#performance-optimisation}

최적의 성능을 위해:

- **캐싱**: 체크아웃 페이지를 제외하도록 캐싱 플러그인을 구성
- **CDN**: CDN 설정이 체크아웃 과정에 방해가 되지 않도록 확인
- **데이터베이스**: 주문 처리를 더 빠르게 하기 위해 데이터베이스 최적화
- **호스팅**: 좋은 가동 시간을 가진 신뢰할 수 있는 호스팅 서비스 사용
