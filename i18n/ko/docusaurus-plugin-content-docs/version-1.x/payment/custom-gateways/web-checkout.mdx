# 웹 체크아웃 게이트웨이

웹 체크아웃 게이트웨이는 고객이 귀하의 웹 스토어 체크아웃 시스템을 사용하여 POS 거래를 완료할 수 있게 합니다. 이 게이트웨이는 매장 내 POS와 온라인 결제 수단 간의 원활한 연결을 생성하여 고객이 자신의 장치를 사용하여 결제하기를 선호하는 상황이나 웹 전용 결제 수단에 접근해야 할 때 완벽합니다.

## 기능

- **원활한 통합**: POS에서 웹 체크아웃으로의 직접 링크
- **모든 결제 수단**: 웹 스토어에서 활성화된 모든 결제 게이트웨이에 접근
- **고객 제어**: 고객이 자신의 장치와 선호하는 방법으로 결제 가능
- **주문 동기화**: POS와 웹 스토어 간의 자동 주문 업데이트
- **유연한 워크플로우**: POS 워크플로우를 유지하면서 온라인으로 거래 완료

## 설치

### 1단계: 다운로드 및 설치

1. [웹 체크아웃 게이트웨이 릴리스 페이지](https://github.com/wcpos/web-checkout-gateway/releases)에 방문
2. 최신 **woocommerce-pos-web-checkout-gateway.zip** 파일 다운로드
3. WordPress 관리 화면에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일을 업로드하고 플러그인 활성화

### 2단계: WooCommerce POS에서 사용 설정

1. `WP 관리자 > WooCommerce POS > 설정 > 체크아웃`으로 이동
2. 목록에서 **WooCommerce POS 웹 체크아웃 게이트웨이** 찾기
3. POS에서 사용할 수 있도록 게이트웨이 활성화
4. 설정 저장

:::note
이 게이트웨이는 귀하의 기존 WooCommerce 결제 수단을 활용하므로, 적어도 하나의 웹 결제 게이트웨이가 올바르게 구성되어 있는지 확인하세요 (Stripe, PayPal 등).
:::

## 사용법

### 웹 체크아웃 시작하기

1. **상품 추가**: POS에서 상품을 장바구니에 추가
2. **게이트웨이 선택**: 결제 방법으로 "웹 체크아웃" 선택
3. **링크 생성**: 시스템이 주문을 위한 고유 체크아웃 링크 생성
4. **고객 접근**: 고객이 링크를 클릭하여 웹 체크아웃에 접근
5. **온라인 결제**: 고객이 웹 스토어 결제 수단을 사용하여 결제 완료

### 결제 과정

웹 체크아웃 게이트웨이는 다음과 같은 워크플로우를 따릅니다:

1. **주문 생성**: POS가 WooCommerce에서 대기 중인 주문 생성
2. **체크아웃 링크**: 시스템이 안전하고 고유한 결제 링크 생성
3. **고객 리디렉션**: 고객이 웹 체크아웃 페이지로 이동
4. **결제 선택**: 고객이 사용 가능한 웹 결제 수단 중에서 선택
5. **결제 처리**: 표준 WooCommerce 체크아웃 과정이 결제를 처리
6. **주문 완료**: 결제가 성공적으로 이루어지면 주문 상태가 자동으로 업데이트

### POS 워크플로우

웹 체크아웃을 시작한 후:

1. **상태 모니터링**: 결제 상태를 모니터링하기 위해 POS 주문 화면을 열어두기
2. **고객 결제**: 고객이 자신의 장치나 귀하의 태블릿에서 결제 완료
3. **자동 업데이트**: 결제가 수신되면 주문 상태가 실시간으로 업데이트
4. **결제 처리**: POS에서 "결제 처리" 버튼을 클릭하여 영수증으로 진행
5. **영수증 생성**: 정상적으로 영수증을 생성하고 인쇄

## 사용 사례

### 완벽한 대상

- **고객 선호**: 자신의 결제 앱이나 카드를 사용하려는 고객
- **복잡한 결제**: POS에서 사용할 수 없는 결제 수단이 필요한 거래
- **분할 결제**: 여러 결제 수단을 사용하고자 하는 고객
- **충성도 프로그램**: 웹 기반 충성도 포인트 사용 접근
- **기프트 카드**: 디지털 기프트 카드나 스토어 크레딧을 가진 고객
- **국제 카드**: 웹 게이트웨이를 통해 더 잘 작동하는 결제 수단

### 워크플로우 예시

**소매점**
1. 고객이 매장에서 쇼핑 후 체크아웃으로 이동
2. 고객이 모바일 지갑 앱으로 결제하기를 선호
3. 직원이 웹 체크아웃 게이트웨이를 선택
4. 고객이 QR 코드를 스캔하거나 자신의 전화에서 링크 클릭
5. 고객이 선호하는 방법으로 결제 완료
6. 직원이 영수증을 처리하고 거래 완료

**서비스 비즈니스**
1. 서비스를 완료하고 POS에 요금 추가
2. 고객이 비즈니스 신용카드로 결제하기를 원함
3. 웹 체크아웃 링크 생성
4. 고객이 안전한 웹 양식에 결제 정보를 입력
5. 결제가 귀하의 웹 게이트웨이를 통해 처리됨
6. 주문 완료 및 영수증 생성

## 기술 세부정보

### 주문 관리

- **대기 중인 주문**: "대기 중인 결제" 상태로 주문 생성
- **상태 업데이트**: 결제가 수신되면 자동으로 상태 변경
- **주문 메모**: 결제 세부정보가 주문 메모에 기록됨
- **재고**: 체크아웃 과정 중 재고가 예약됨

### 보안

- **고유 링크**: 각 체크아웃 세션에는 고유하고 시간 제한이 있는 URL
- **SSL 요구**: 모든 결제 처리는 보안 HTTPS 연결을 통해 이루어짐
- **PCI 준수**: 귀하의 기존 PCI 준수 웹 결제 게이트웨이를 활용
- **세션 관리**: 안전한 세션 처리가 무단 접근을 방지

### 결제 수단

이 게이트웨이는 구성된 모든 WooCommerce 결제 수단에 접근할 수 있습니다:

- **신용/직불 카드**: Stripe, Square, Authorize.net 등
- **디지털 지갑**: PayPal, Apple Pay, Google Pay
- **은행 송금**: 직접 은행 결제 수단
- **지금 구매, 나중에 결제**: Klarna, Afterpay, Sezzle
- **암호화폐**: 비트코인 및 기타 암호화폐 결제 게이트웨이
- **지역 방법**: 지역 특정 결제 수단

## 요구 사항

- **WooCommerce POS**: 무료 또는 프로 버전이 설치 및 활성화되어 있어야 함
- **웹 결제 게이트웨이**: WooCommerce에서 적어도 하나의 온라인 결제 방법이 구성되어 있어야 함
- **SSL 인증서**: 안전한 결제 처리를 위한 HTTPS 필요
- **최신 브라우저**: 고객 장치는 현대 웹 표준을 지원해야 함

## 문제 해결

### 일반적인 문제

**체크아웃 링크 작동 안 함**
- SSL 인증서가 올바르게 설치되었는지 확인
- WooCommerce 고유 링크 설정이 올바르게 구성되었는지 확인
- 웹 결제 게이트웨이가 활성화되고 제대로 구성되었는지 확인
- 다른 브라우저나 장치로 테스트

**결제 처리 안 됨**
- 웹 결제 게이트웨이가 제대로 구성되었는지 확인
- 결제 게이트웨이 로그에서 오류 메시지 확인
- 고객의 결제 방법이 지원되는지 확인
- 웹 체크아웃 과정을 독립적으로 테스트

**주문이 POS에서 업데이트되지 않음**
- 인터넷 연결의 안정성 확인
- WordPress 크론 작업이 제대로 실행되고 있는지 확인
- POS 주문 화면 새로 고침
- WooCommerce 관리자에서 주문 상태 확인

**고객이 체크아웃에 접근할 수 없음**
- 체크아웃 링크가 만료되지 않았는지 확인
- 고객의 장치가 인터넷에 연결되어 있는지 확인
- 주문이 취소되거나 완료되지 않았는지 확인
- 다른 장치에서 링크 테스트

### 성능 최적화

최적의 성능을 위해:

- **캐싱**: 체크아웃 페이지를 제외하도록 캐싱 플러그인 구성
- **CDN**: CDN 설정이 체크아웃 과정에 방해가 되지 않는지 확인
- **데이터베이스**: 빠른 주문 처리를 위해 데이터베이스 최적화
- **호스팅**: 안정적인 호스팅을 사용하고 좋은 가동시간 확보
