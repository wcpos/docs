# 이메일 청구서 게이트웨이 {#email-invoice-gateway}

이메일 청구서 게이트웨이를 사용하면 WCPOS에서 직접 이메일을 통해 고객에게 결제 청구서를 보낼 수 있습니다. 이는 고객이 나중에 결제하기를 원하거나 귀하의 웹 스토어 결제 방법을 사용하여 온라인으로 결제를 완료해야 하는 경우에 적합합니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [이메일 청구서 게이트웨이 릴리스 페이지](https://github.com/wcpos/email-invoice-gateway/releases) 방문
2. 최신 **woocommerce-pos-email-invoice-gateway.zip** 파일 다운로드
3. 워드프레스 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일을 업로드하고 플러그인을 활성화

  </StepCard>
  <StepCard number="2" title="WCPOS에서 활성화" isLast>

2. 목록에서 **WCPOS 청구서 결제 게이트웨이** 찾기
3. POS에서 사용할 수 있도록 게이트웨이 활성화

:::

  </StepCard>

## 사용법 {#usage}

### 청구서 전송 {#sending-invoices}

1. **상품 추가**: POS에서 장바구니에 제품 추가
2. **게이트웨이 선택**: 결제 방법으로 "이메일 청구서" 선택
3. **이메일 주소**: 
   - 기존 고객의 경우: 고객 데이터에서 이메일이 자동으로 채워집니다.
   - 게스트 주문의 경우: 고객의 이메일 주소를 수동으로 입력
4. **선택 사항**: "청구지 주소에 저장" 체크하여 고객의 청구 정보를 업데이트
5. **청구서 전송**: 청구서 이메일을 전송하기 위해 프로세스 완료

### 고객 경험 {#customer-experience}

청구서를 전송하면 고객은 다음을 포함한 이메일을 받습니다:

- **주문 세부정보**: 항목, 수량 및 가격의 전체 내역
- **주문 총액**: 세금 및 수수료를 포함한 최종 금액
- **결제 링크**: 온라인으로 결제를 완료할 수 있는 직접 링크
- **주문 정보**: 주문 번호 및 참조 세부정보

### 결제 완료 {#payment-completion}

고객은 두 가지 방법으로 결제를 완료할 수 있습니다:

1. **온라인 결제**: 이메일에서 "이 주문 결제" 링크 클릭
2. **매장 내 결제**: 다른 POS 결제 방법을 사용하여 결제를 완료하기 위해 돌아감

"이 주문 결제" 링크는 고객을 귀하의 웹사이트의 전용 결제 페이지로 안내하며, 고객은 귀하의 웹에서 사용 가능한 결제 게이트웨이(Stripe, PayPal 등)를 사용할 수 있습니다.

## 이메일 관리 {#email-management}

### 기존 고객 {#existing-customers}

- 이메일 주소가 고객 프로필에서 자동으로 검색됩니다.
- 고객 청구 정보가 보존됩니다.
- 주문 기록이 고객 계정에 유지됩니다.

### 게스트 고객 {#guest-customers}

- 일회성 고객에 대한 이메일 주소를 수동으로 입력합니다.
- 미래 참조를 위해 청구지 주소에 이메일을 저장하는 옵션
- 게스트 주문은 WooCommerce에서 적절히 추적됩니다.

### 이메일 템플릿 {#email-templates}

이 게이트웨이는 WooCommerce의 표준 이메일 템플릿을 사용합니다:

- **커스터마이즈 가능**: WooCommerce 이메일 설정을 통해 이메일 외관 수정 가능
- **브랜딩**: 귀하의 스토어 로고 및 브랜딩 포함
- **전문적**: 깨끗하고 전문적인 청구서 형식
- **모바일 친화적**: 모든 기기를 위한 반응형 디자인

## 사용 사례 {#use-cases}

### 완벽한 경우 {#perfect-for}

- **B2B 판매**: 회계 시스템을 통해 결제를 처리해야 하는 비즈니스 고객
- **대량 주문**: 승인이나 처리 시간이 필요한 고액 거래
- **원격 고객**: 매장을 떠난 후 결제를 완료해야 하는 고객
- **계정 고객**: 확립된 결제 조건을 가진 정기 고객
- **견적에서 주문으로**: 견적이나 추정치를 결제 가능한 청구서로 변환

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="이메일이 전송되지 않음">

  - 워드프레스 이메일 구성 확인
  - 커스텀 이메일 서비스를 사용하는 경우 SMTP 설정 확인
  - 워드프레스 이메일 테스트 플러그인으로 테스트
  - 테스트 이메일의 스팸 폴더 확인

  </AccordionItem>

  <AccordionItem question="결제 링크 작동하지 않음">

  - WooCommerce가 올바르게 구성되었는지 확인
  - 웹 체크아웃을 위한 결제 게이트웨이가 하나 이상 활성화되었는지 확인
  - SSL 인증서가 제대로 설치되었는지 확인
  - 주문 상태가 결제를 허용하는지 확인

  </AccordionItem>

  <AccordionItem question="고객이 결제를 완료할 수 없음">

  - 웹 결제 게이트웨이가 활성화되고 구성되었는지 확인
  - 주문이 만료되거나 취소되지 않았는지 확인
  - 고객이 충분한 결제 방법 한도를 가지고 있는지 확인
  - 결제 프로세스를 직접 테스트

  </AccordionItem>
</Accordion>

### 이메일 전송 {#email-delivery}

신뢰할 수 있는 이메일 전송을 위해 다음을 고려하세요:

- **SMTP 서비스**: SendGrid, Mailgun 또는 Amazon SES와 같은 서비스 사용
- **이메일 플러그인**: 더 나은 전송을 위해 워드프레스 SMTP 플러그인 설치
- **도메인 인증**: SPF, DKIM 및 DMARC 레코드 설정
- **모니터링**: 이메일 전송 및 열람률 추적
