# 이메일 송장 게이트웨이 {#email-invoice-gateway}

이메일 송장 게이트웨이는 WooCommerce POS에서 고객에게 이메일로 직접 결제 송장을 전송할 수 있도록 합니다. 이는 고객이 나중에 결제하기를 원하거나 웹 스토어의 결제 방법을 사용하여 온라인으로 결제를 완료해야 하는 상황에서 유용합니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [이메일 송장 게이트웨이 릴리스 페이지](https://github.com/wcpos/email-invoice-gateway/releases)를 방문합니다.
2. 최신 **woocommerce-pos-email-invoice-gateway.zip** 파일을 다운로드합니다.
3. WordPress 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동합니다.
4. zip 파일을 업로드하고 플러그인을 활성화합니다.

  </StepCard>
  <StepCard number="2" title="WooCommerce POS에서 활성화" isLast>

1. `WP Admin > WooCommerce POS > 설정 > 체크아웃`으로 이동합니다.
2. 리스트에서 **WooCommerce POS 송장 결제 게이트웨이**를 찾습니다.
3. POS에서 사용할 수 있도록 게이트웨이를 활성화합니다.
4. 설정을 저장합니다.

:::note
이 게이트웨이는 추가 API 키나 외부 서비스 구성이 필요하지 않습니다. 기존 WordPress 이메일 시스템과 WooCommerce 결제 방법을 사용합니다.
:::

  </StepCard>
</Steps>

## 사용법 {#usage}

### 송장 전송 {#sending-invoices}

1. **상품 추가**: POS에서 장바구니에 상품을 추가합니다.
2. **게이트웨이 선택**: 결제 방법으로 "이메일 송장"을 선택합니다.
3. **이메일 주소**: 
   - 기존 고객의 경우: 고객 데이터에서 이메일이 자동으로 채워집니다.
   - 게스트 주문의 경우: 고객의 이메일 주소를 수동으로 입력합니다.
4. **선택 사항**: "청구 주소에 저장"을 체크하여 고객의 청구 정보를 업데이트합니다.
5. **송장 전송**: 송장 이메일을 전송하는 과정을 완료합니다.

### 고객 경험 {#customer-experience}

송장을 전송하면 고객은 다음 내용을 포함하는 이메일을 받습니다:

- **주문 세부사항**: 품목, 수량 및 가격에 대한 전체 세부 사항
- **주문 총액**: 세금 및 수수료를 포함한 최종 금액
- **결제 링크**: 온라인으로 결제를 완료하기 위한 직접 링크
- **주문 정보**: 주문 번호 및 참조 세부정보

### 결제 완료 {#payment-completion}

고객은 두 가지 방법으로 결제를 완료할 수 있습니다:

1. **온라인 결제**: 이메일의 "이 주문에 대해 결제하기" 링크를 클릭합니다.
2. **매장 내 결제**: 다른 POS 결제 방법을 사용하여 결제를 완료하기 위해 돌아옵니다.

"이 주문에 대해 결제하기" 링크는 고객을 웹사이트의 전용 결제 페이지로 안내하여 활성화된 웹 결제 게이트웨이(Stripe, PayPal 등)를 사용할 수 있습니다.

## 이메일 관리 {#email-management}

### 기존 고객 {#existing-customers}

- 이메일 주소는 고객 프로필에서 자동으로 검색됩니다.
- 고객의 청구 정보는 유지됩니다.
- 기존 고객 계정에서 주문 기록이 유지됩니다.

### 게스트 고객 {#guest-customers}

- 일회성 고객을 위해 이메일 주소를 수동으로 입력합니다.
- 향후 참조를 위해 이메일을 청구 주소에 저장하는 옵션이 있습니다.
- 게스트 주문은 WooCommerce에서 제대로 추적됩니다.

### 이메일 템플릿 {#email-templates}

게이트웨이는 WooCommerce의 표준 이메일 템플릿을 사용합니다:

- **커스터마이즈 가능**: WooCommerce 이메일 설정을 통해 이메일 외관 수정
- **브랜드화**: 스토어 로고 및 브랜드 포함
- **전문적**: 깔끔하고 전문적인 송장 형식
- **모바일 친화적**: 모든 장치에 맞게 반응형 디자인

## 사용 사례 {#use-cases}

### 완벽함 {#perfect-for}

- **B2B 판매**: 회계 시스템을 통해 결제를 처리해야 하는 기업 고객
- **대량 주문**: 승인 또는 처리 시간이 필요한 고가 거래
- **원격 고객**: 매장을 떠난 후 결제를 완료해야 하는 고객
- **계정 고객**: 정해진 결제 조건이 있는 정기 고객
- **견적-주문**: 견적이나 예상 금액을 결제 가능한 송장으로 변환

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="이메일이 전송되지 않음">

  - WordPress 이메일 구성을 확인합니다.
  - 사용자 지정 이메일 서비스를 사용하는 경우 SMTP 설정을 확인합니다.
  - WordPress 이메일 테스트 플러그인을 사용하여 테스트합니다.
  - 테스트 이메일에 대해 스팸 폴더를 확인합니다.

  </AccordionItem>

  <AccordionItem question="결제 링크 작동하지 않음">

  - WooCommerce가 올바르게 구성되어 있는지 확인합니다.
  - 웹 체크아웃을 위한 최소 하나의 결제 게이트웨이가 활성화되어 있는지 확인합니다.
  - SSL 인증서가 올바르게 설치되었는지 확인합니다.
  - 주문 상태가 결제를 허용하는지 확인합니다.

  </AccordionItem>

  <AccordionItem question="고객이 결제를 완료할 수 없음">

  - 웹 결제 게이트웨이가 활성화되어 있고 구성되어 있는지 확인합니다.
  - 주문이 만료되었거나 취소되지 않았는지 확인합니다.
  - 고객이 충분한 결제 방법 한도를 가지고 있는지 확인합니다.
  - 결제 프로세스를 직접 테스트합니다.

  </AccordionItem>
</Accordion>

### 이메일 전달 {#email-delivery}

신뢰할 수 있는 이메일 전달을 위해 고려:

- **SMTP 서비스**: SendGrid, Mailgun 또는 Amazon SES와 같은 서비스를 사용합니다.
- **이메일 플러그인**: 더 나은 전달을 위해 WordPress SMTP 플러그인을 설치합니다.
- **도메인 인증**: SPF, DKIM, DMARC 레코드를 설정합니다.
- **모니터링**: 이메일 전달 및 열람률을 추적합니다.
