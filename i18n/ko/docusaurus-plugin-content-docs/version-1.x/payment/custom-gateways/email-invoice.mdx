# 이메일 송장 게이트웨이

이메일 송장 게이트웨이를 사용하면 WooCommerce POS에서 고객에게 직접 결제 송장을 이메일로 보낼 수 있습니다. 고객이 나중에 결제하거나 웹스토어의 결제 방법을 사용하여 온라인으로 결제를 완료해야 하는 상황에 완벽합니다.

## 기능

- **이메일 통합**: POS 인터페이스에서 직접 송장 전송
- **고객 관리**: 기존 고객 데이터에서 자동으로 이메일 채우기 또는 새 이메일 주소 추가
- **비회원 주문**: 선택적 청구 주소 업데이트와 함께 비회원 고객 지원
- **온라인 결제**: 고객은 온라인으로 결제를 완료할 수 있는 "이 주문에 대해 결제" 링크 수신
- **유연한 워크플로**: 매장 내 주문 완료 또는 원격 결제 완료 허용

## 설치

### 1단계: 다운로드 및 설치

1. [이메일 송장 게이트웨이 릴리스 페이지](https://github.com/wcpos/email-invoice-gateway/releases) 방문
2. 최신 **woocommerce-pos-email-invoice-gateway.zip** 파일 다운로드
3. WordPress 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일 업로드 및 플러그인 활성화

### 2단계: WooCommerce POS에서 활성화

1. `WP 관리자 > WooCommerce POS > 설정 > 결제`로 이동
2. 목록에서 **WooCommerce POS 송장 결제 게이트웨이** 찾기
3. POS에서 사용할 게이트웨이 활성화
4. 설정 저장

:::note
이 게이트웨이는 추가 API 키나 외부 서비스 구성이 필요하지 않습니다. 기존 WordPress 이메일 시스템과 WooCommerce 결제 방법을 사용합니다.
:::

## 사용 방법

### 송장 전송

1. **항목 추가**: POS에서 장바구니에 제품 추가
2. **게이트웨이 선택**: 결제 방법으로 "이메일 송장" 선택
3. **이메일 주소**: 
   - 기존 고객: 고객 데이터에서 자동으로 이메일 채우기
   - 비회원 주문: 고객 이메일 주소 수동 입력
4. **선택사항**: "청구 주소에 저장" 확인하여 고객 청구 정보 업데이트
5. **송장 전송**: 송장 이메일 전송 프로세스 완료

### 고객 경험

송장을 보내면 고객은 다음 내용이 포함된 이메일 수신:

- **주문 세부 정보**: 항목, 수량, 가격 전체 내역
- **주문 총액**: 세금 및 수수료 포함 최종 금액
- **결제 링크**: 온라인으로 결제 완료를 위한 직접 링크
- **주문 정보**: 주문 번호 및 참조 세부 정보

### 결제 완료

고객은 두 가지 방법으로 결제 완료 가능:

1. **온라인 결제**: 이메일의 "이 주문에 대해 결제" 링크 클릭
2. **매장 내 반환**: 다른 POS 결제 방법을 사용하여 결제 완료를 위해 돌아오기

"이 주문에 대해 결제" 링크는 고객을 웹사이트의 전용 결제 페이지로 이동시켜 활성화된 모든 웹 결제 게이트웨이(Stripe, PayPal 등) 사용 가능.

## 이메일 관리

### 기존 고객

- 이메일 주소는 고객 프로필에서 자동으로 검색
- 고객 청구 정보 유지
- 주문 이력은 고객 계정에 유지

### 비회원 고객

- 일회성 고객을 위해 이메일 주소 수동 입력
- 향후 참조를 위해 이메일을 청구 주소에 저장하는 옵션
- 비회원 주문은 WooCommerce에서 적절히 추적

### 이메일 템플릿

게이트웨이는 WooCommerce의 표준 이메일 템플릿 사용:

- **사용자 정의 가능**: WooCommerce 이메일 설정을 통해 이메일 모양 수정
- **브랜딩**: 스토어 로고 및 브랜딩 포함
- **전문적**: 깔끔하고 전문적인 송장 형식
- **모바일 친화적**: 모든 기기에 반응형 디자인

## 사용 사례

### 완벽한 상황

- **B2B 영업**: 자체 회계 시스템을 통해 결제를 처리해야 하는 비즈니스 고객
- **대규모 주문**: 승인 또는 처리 시간이 필요한 고가 거래
- **원격 고객**: 매장을 떠난 후 결제를 완료해야 하는 고객
- **계정 고객**: 확립된 결제 조건이 있는 정기 고객
- **견적-주문 변환**: 견적 또는 추정을 지불 가능한 송장으로 변환

### 워크플로 예시

**소매점**
1. 고객이 항목을 선택하지만 구매에 대한 승인이 필요함
2. 모든 세부 정보가 포함된 송장 이메일 전송
3. 고객이 준비되면 온라인으로 결제 완료
4. 주문이 시스템에서 자동으로 업데이트됨

**서비스 업**
1. 서비스 작업 완료 및 POS에 항목 추가
2. 고객의 회계 부서에 송장 전송
3. 고객이 선호하는 결제 방법으로 온라인 결제
4. 결제 확인 및 주문 완료

## 요구 사항

- **WooCommerce POS**: 무료 또는 Pro 버전 설치 및 활성화
- **이메일 시스템**: 작동하는 WordPress 이메일 구성
- **웹 결제 게이트웨이**: WooCommerce에서 최소 하나의 온라인 결제 방법 활성화
- **SSL 인증서**: 안전한 결제 처리를 위해 HTTPS 필요

## 문제 해결

### 일반적인 문제

**이메일 전송 안 됨**
- WordPress 이메일 구성 확인
- 사용자 지정 이메일 서비스의 SMTP 설정 확인
- WordPress 이메일 테스트 플러그인으로 테스트
- 테스트 이메일의 스팸 폴더 확인

**결제 링크 작동 안 함**
- WooCommerce가 제대로 구성되었는지 확인
- 웹 결제를 위해 최소 하나의 결제 게이트웨이가 활성화되었는지 확인
- SSL 인증서가 제대로 설치되었는지 확인
- 주문 상태가 결제를 허용하는지 확인

**고객이 결제를 완료할 수 없음**
- 웹 결제 게이트웨이가 활성화 및 구성되었는지 확인
- 주문이 만료되거나 취소되지 않았는지 확인
- 고객의 결제 방법 한도가 충분한지 확인
- 직접 결제 프로세스 테스트

### 이메일 배달

안정적인 이메일 배달을 위해 고려할 사항:

- **SMTP 서비스**: SendGrid, Mailgun 또는 Amazon SES와 같은 서비스 사용
- **이메일 플러그인**: 더 나은 배달을 위해 WordPress SMTP 플러그인 설치
- **도메인 인증**: SPF, DKIM, DMARC 레코드 설정
- **모니터링**: 이메일 배달 및 열람률 추적

### 도움 받기

기술 지원의 경우:
- 문제 보고를 위해 [GitHub 저장소](https://github.com/wcpos/email-invoice-gateway) 방문
- 템플릿 사용자 정의를 위해 WooCommerce 이메일 문서 확인
- WordPress 이메일 테스트 도구로 이메일 기능 테스트
