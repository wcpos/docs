# 이메일 송장 게이트웨이

이메일 송장 게이트웨이는 WooCommerce POS에서 고객에게 이메일로 지불 송장을 직접 발송할 수 있게 해줍니다. 이는 고객이 나중에 지불하기를 선호하거나 웹 스토어의 결제 방법을 사용하여 온라인으로 지불을 완료해야하는 상황에 적합합니다.

## 기능

- **이메일 통합**: POS 인터페이스에서 직접 송장 발송
- **고객 관리**: 기존 고객 데이터에서 이메일 자동 입력 또는 새로운 이메일 주소 추가
- **비회원 주문**: 선택적인 청구지 주소 업데이트와 함께 비회원 고객 지원
- **온라인 지불**: 고객은 온라인에서 지불을 완료할 수 있는 "이 주문에 대한 지불" 링크 수신
- **유연한 워크플로**: 매장에서 주문을 완료하거나 원격 지불 완료 허용

## 설치

### 단계 1: 다운로드 및 설치

1. [이메일 송장 게이트웨이 릴리스 페이지](https://github.com/wcpos/email-invoice-gateway/releases)에 방문
2. 최신 **woocommerce-pos-email-invoice-gateway.zip** 파일 다운로드
3. WordPress 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일을 업로드하고 플러그인을 활성화

### 단계 2: WooCommerce POS에서 활성화

1. `WP 관리 > WooCommerce POS > 설정 > 체크아웃`으로 이동
2. 목록에서 **WooCommerce POS 송장 결제 게이트웨이** 찾기
3. POS에서 사용하도록 게이트웨이 활성화
4. 설정 저장

:::note
이 게이트웨이는 추가 API 키나 외부 서비스 구성이 필요 없습니다. 기존 WordPress 이메일 시스템 및 WooCommerce 결제 방법을 사용합니다.
:::

## 사용 방법

### 송장 발송

1. **항목 추가**: POS에서 장바구니에 제품 추가
2. **게이트웨이 선택**: 결제 방법으로 "이메일 송장" 선택
3. **이메일 주소**: 
   - 기존 고객의 경우: 고객 데이터에서 이메일이 자동으로 입력됨
   - 비회원 주문의 경우: 고객의 이메일 주소를 수동으로 입력
4. **선택 사항**: "청구지 주소에 저장" 확인하여 고객의 청구 정보를 업데이트
5. **송장 발송**: 송장 이메일 발송을 완료하도록 프로세스 진행

### 고객 경험

송장을 발송하면 고객은 다음 내용을 포함한 이메일을 받습니다:

- **주문 세부사항**: 품목, 수량 및 가격의 완전한 분류
- **주문 총액**: 세금 및 수수료를 포함한 최종 금액
- **지불 링크**: 온라인에서 지불을 완료할 수 있는 직접 링크
- **주문 정보**: 주문 번호 및 참조 세부정보

### 지불 완료

고객은 지불을 두 가지 방법으로 완료할 수 있습니다:

1. **온라인 지불**: 이메일에서 "이 주문에 대한 지불" 링크 클릭
2. **매장 내 반환**: 다른 POS 결제 방법을 사용하여 지불을 완료하기 위해 반환

"이 주문에 대한 지불" 링크는 고객을 웹사이트의 전용 결제 페이지로 안내하며, 고객은 사용 가능한 모든 웹 결제 게이트웨이를 사용할 수 있습니다 (Stripe, PayPal 등).

## 이메일 관리

### 기존 고객

- 이메일 주소는 고객 프로필에서 자동으로 검색됨
- 고객의 청구 정보가 보존됨
- 주문 기록은 고객 계정 아래에 유지됨

### 비회원 고객

- 1회성 고객을 위한 이메일 주소를 수동으로 입력
- 향후 참조를 위해 청구지 주소에 이메일 저장 옵션
- 비회원 주문은 WooCommerce에서 제대로 추적됨

### 이메일 템플릿

이 게이트웨이는 WooCommerce의 표준 이메일 템플릿을 사용합니다:

- **사용자 정의 가능**: WooCommerce 이메일 설정을 통해 이메일 외관 수정
- **브랜딩**: 매장 로고 및 브랜딩 포함
- **전문적**: 깔끔하고 전문적인 송장 형식
- **모바일 친화적**: 모든 장치를 위한 반응형 디자인

## 사용 사례

### 완벽한 경우

- **B2B 판매**: 회계 시스템을 통해 지불을 처리해야 하는 비즈니스 고객
- **대량 주문**: 승인이나 처리 시간이 필요한 고가 거래
- **원거리 고객**: 매장을 떠난 후 지불을 완료해야 하는 고객
- **계정 고객**: 기정된 지불 조건을 가진 정기 고객
- **견적에서 주문으로**: 견적 또는 추정치를 지불할 수 있는 송장으로 변환

### 워크플로 예시

**소매점**
1. 고객이 상품을 선택하지만 구매 승인이 필요함
2. 모든 세부 정보를 포함한 송장 이메일 발송
3. 고객이 준비가 되었을 때 온라인으로 지불 완료
4. 주문이 시스템에서 자동으로 업데이트됨

**서비스 비즈니스**
1. 서비스 작업을 완료하고 POS에 항목 추가
2. 고객의 회계 부서에 송장 발송
3. 고객이 선호하는 결제 방법으로 온라인에서 지불
4. 지불 확인을 받고 주문 완료

## 요구 사항

- **WooCommerce POS**: 무료 또는 프로 버전 설치 및 활성화
- **이메일 시스템**: 제대로 작동하는 WordPress 이메일 구성
- **웹 결제 게이트웨이**: WooCommerce에서 온라인 결제 방법이 최소 하나 활성화되어 있어야 함
- **SSL 인증서**: 안전한 결제 처리를 위한 HTTPS 필수

## 문제 해결

### 일반적인 문제

**이메일 발송되지 않음**
- WordPress 이메일 구성 확인
- 사용자 정의 이메일 서비스를 사용하는 경우 SMTP 설정 확인
- WordPress 이메일 테스트 플러그인으로 테스트
- 테스트 이메일에 대해 스팸 폴더 확인

**지불 링크가 작동하지 않음**
- WooCommerce가 제대로 구성되었는지 확인
- 웹 체크아웃을 위해 최소 하나의 결제 게이트웨이가 활성화되어 있는지 확인
- SSL 인증서가 제대로 설치되었는지 확인
- 주문 상태가 지불을 허용하는지 확인

**고객이 지불을 완료할 수 없음**
- 웹 결제 게이트웨이가 활성화되고 구성되어 있는지 확인
- 주문이 만료되거나 취소되지 않았는지 확인
- 고객이 충분한 지불 방법 한도가 있는지 확인
- 직접 결제 프로세스를 테스트

### 이메일 배달

신뢰할 수 있는 이메일 배달을 위해 고려해야 할 사항:

- **SMTP 서비스**: SendGrid, Mailgun 또는 Amazon SES와 같은 서비스 사용
- **이메일 플러그인**: 더 나은 배달을 위해 WordPress SMTP 플러그인 설치
- **도메인 인증**: SPF, DKIM 및 DMARC 레코드 설정
- **모니터링**: 이메일 배달 및 오픈 비율 추적

기술 지원을 위해:
