# 이메일 인보이스 게이트웨이

이메일 인보이스 게이트웨이를 통해 WooCommerce POS에서 직접 고객에게 결제 인보이스를 이메일로 보낼 수 있습니다. 이는 고객이 나중에 결제하기를 원하거나 웹 스토어의 결제 방법을 사용하여 온라인으로 결제를 완료해야 하는 상황에 적합합니다.

## 설치

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [이메일 인보이스 게이트웨이 릴리스 페이지](https://github.com/wcpos/email-invoice-gateway/releases) 방문
2. 최신 **woocommerce-pos-email-invoice-gateway.zip** 파일 다운로드
3. WordPress 관리 화면에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일을 업로드하고 플러그인 활성화

  </StepCard>
  <StepCard number="2" title="WooCommerce POS에서 활성화" isLast>

1. `WP 관리 > WooCommerce POS > 설정 > 결제`로 이동
2. 목록에서 **WooCommerce POS 인보이스 결제 게이트웨이** 찾기
3. POS에서 사용하기 위해 게이트웨이 활성화
4. 설정 저장

:::

  </StepCard>

## 사용법

### 인보이스 보내기

1. **아이템 추가**: POS에서 장바구니에 제품 추가
2. **게이트웨이 선택**: 결제 방법으로 "이메일 인보이스" 선택
3. **이메일 주소**: 
   - 기존 고객의 경우: 이메일이 고객 데이터에서 자동으로 채워집니다
   - 비회원 주문의 경우: 고객의 이메일 주소를 수동으로 입력
4. **선택 사항**: "청구 주소에 저장"을 체크하여 고객의 청구 정보를 업데이트
5. **인보이스 전송**: 인보이스 이메일 전송을 완료하는 과정

### 고객 경험

인보이스를 전송하면 고객은 다음이 포함된 이메일을 받습니다:

- **주문 세부 사항**: 항목, 수량 및 가격의 전체 내역
- **주문 총액**: 세금 및 수수료를 포함한 최종 요금
- **결제 링크**: 온라인에서 결제를 완료할 수 있는 직접 링크
- **주문 정보**: 주문 번호 및 참조 세부 사항

### 결제 완료

고객은 결제를 두 가지 방법으로 완료할 수 있습니다:

1. **온라인 결제**: 이메일의 "이 주문에 대해 결제하기" 링크 클릭
2. **매장 내 결제**: 매장에서 다른 POS 결제 방법을 사용하여 결제 완료

"이 주문에 대해 결제하기" 링크는 고객을 귀하의 웹사이트의 전용 결제 페이지로 안내하며, 여기에서 활성화된 웹 결제 게이트웨이(Stripe, PayPal 등)를 사용할 수 있습니다.

## 이메일 관리

### 기존 고객

- 이메일 주소는 고객 프로필에서 자동으로 가져옵니다
- 고객의 청구 정보는 유지됩니다
- 주문 내역은 고객 계정 아래에 보관됩니다

### 비회원 고객

- 일회성 고객을 위해 이메일 주소를 수동으로 입력
- 미래 참조를 위해 청구 주소에 이메일 저장 옵션
- WooCommerce에서 비회원 주문은 제대로 추적됩니다

### 이메일 템플릿

이 게이트웨이는 WooCommerce의 표준 이메일 템플릿을 사용합니다:

- **사용자 정의 가능**: WooCommerce 이메일 설정을 통해 이메일 모양 수정
- **브랜딩**: 귀하의 스토어 로고와 브랜딩 포함
- **전문적**: 깔끔하고 전문적인 인보이스 형식
- **모바일 친화적**: 모든 장치에 대응하는 디자인

## 사용 사례

### 완벽한 경우

- **B2B 판매**: 회계 시스템을 통해 결제를 처리해야 하는 비즈니스 고객
- **대량 주문**: 승인이나 처리 시간이 필요한 고가 거래
- **원격 고객**: 매장을 떠난 후 결제를 완료해야 하는 고객
- **계좌 고객**: 정해진 결제 조건의 정기 고객
- **견적-주문**: 견적이나 평가를 지불 가능한 인보이스로 변환

## 문제 해결

### 일반적인 문제

<Accordion>
  <AccordionItem question="이메일이 전송되지 않음">

  - WordPress 이메일 구성 확인
  - 사용자 정의 이메일 서비스를 사용하는 경우 SMTP 설정 확인
  - WordPress 이메일 테스트 플러그인으로 테스트
  - 테스트 이메일을 위해 스팸 폴더 확인

  </AccordionItem>

  <AccordionItem question="결제 링크가 작동하지 않음">

  - WooCommerce가 제대로 구성되어 있는지 확인
  - 웹 결제를 위해 하나 이상의 결제 게이트웨이가 활성화되어 있는지 확인
  - SSL 인증서가 제대로 설치되어 있는지 확인
  - 주문 상태가 결제를 허용하는지 확인

  </AccordionItem>

  <AccordionItem question="고객이 결제를 완료할 수 없음">

  - 웹 결제 게이트웨이가 활성화되고 구성되어 있는지 확인
  - 주문이 만료되거나 취소되지 않았는지 확인
  - 고객이 충분한 결제 방법 제한이 있는지 확인
  - 결제 프로세스를 직접 테스트

  </AccordionItem>
</Accordion>

### 이메일 전달

신뢰할 수 있는 이메일 전달을 위해 고려해야 할 사항:

- **SMTP 서비스**: SendGrid, Mailgun 또는 Amazon SES 와 같은 서비스를 사용
- **이메일 플러그인**: 더 나은 전달을 위해 WordPress SMTP 플러그인 설치
- **도메인 인증**: SPF, DKIM 및 DMARC 레코드 설정
- **모니터링**: 이메일 전달 및 개봉률 추적
