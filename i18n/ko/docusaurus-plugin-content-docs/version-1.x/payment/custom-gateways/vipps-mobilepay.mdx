# Vipps MobilePay 게이트웨이 {#vipps-mobilepay-gateway}

Vipps MobilePay 게이트웨이를 통해 WooCommerce POS 내에서 전화 기반 결제를 직접 수락할 수 있습니다. 고객은 화면의 QR 코드를 스캔하거나 전화에서 푸시 알림을 확인하여 결제합니다. 카드 리더 하드웨어는 필요 없습니다.

## 기능 {#features}

<FeatureGrid>
  <FeatureCard icon="mobile" title="QR 코드 결제">
    고객이 스캔하여 전화로 결제할 수 있도록 체크아웃에서 QR 코드를 표시합니다.
  </FeatureCard>
  <FeatureCard icon="bolt" title="푸시 알림">
    고객의 전화번호를 입력하여 고객의 전화로 직접 결제 요청을 전송합니다.
  </FeatureCard>
  <FeatureCard icon="globe" title="Nordic 커버리지">
    노르웨이의 Vipps, 덴마크와 핀란드의 MobilePay와 함께 작동합니다.
  </FeatureCard>
  <FeatureCard icon="credit-card" title="자동 캡처">

  </FeatureCard>
</FeatureGrid>

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [WCPOS Vipps MobilePay 릴리스 페이지](https://github.com/wcpos/wcpos-vipps/releases)에 방문합니다.
2. 최신 **wcpos-vipps.zip** 파일을 다운로드합니다.
3. WordPress 관리 화면에서 `Plugins > Add New > Upload Plugin`으로 이동합니다.
4. zip 파일을 업로드하고 플러그인을 활성화합니다.

  </StepCard>
  <StepCard number="2" title="Vipps 자격 증명 구성">

1. `WP Admin > WooCommerce > Settings > Payments`로 이동합니다.
2. 결제 수단 목록에서 **WCPOS Vipps MobilePay**를 찾습니다.
3. **WCPOS Vipps MobilePay**를 클릭하여 설정에 접근합니다.
4. [Vipps 포털](https://portal.vipps.no/)에서 자격 증명을 입력합니다:
   - **상인 일련 번호 (MSN)**
   - **클라이언트 ID**
   - **클라이언트 비밀**
   - **구독 키**
5. 설정을 저장합니다.

:::note
여기에서 POS 사용을 위해 게이트웨이를 활성화할 필요는 없습니다. 다음 단계에서 POS를 위해 특별히 활성화됩니다. WooCommerce 설정에서 활성화하면 온라인 상점 체크아웃에서도 사용 가능하게 되어 테스트에 유용할 수 있습니다.
:::

:::tip 이미 Vipps 플러그인을 사용 중인가요?
공식 [Vipps MobilePay로 체크아웃](https://wordpress.org/plugins/woo-vipps/) 플러그인이 설치된 경우, 이 플러그인을 활성화하면 자격 증명이 자동으로 가져와집니다. 수동 자격 증명 입력을 건너뛸 수 있습니다.
:::

  </StepCard>
  <StepCard number="3" title="WooCommerce POS에서 활성화" isLast>

1. `WP Admin > WooCommerce POS > Settings > Checkout`로 이동합니다.
2. 목록에서 **WCPOS Vipps MobilePay** 게이트웨이를 찾습니다.
3. POS 사용을 위해 게이트웨이를 활성화합니다.
4. 설정을 저장합니다.

  </StepCard>
</Steps>

## 사용법 {#usage}

### 결제 처리 — QR 코드 {#processing-payments-qr-code}

1. **상품 추가**: POS에 카트에 제품을 추가합니다.
2. **게이트웨이 선택**: 결제 수단으로 "Vipps MobilePay"를 선택합니다.
3. **QR 코드 생성**: "QR 코드 생성" 버튼을 클릭합니다.
4. **고객 스캔**: 고객이 Vipps 또는 MobilePay 앱으로 QR 코드를 스캔합니다.
5. **고객 확인**: 고객이 앱에서 결제를 확인합니다.
6. **자동 완료**: 결제가 승인되면 주문이 자동으로 완료됩니다.

### 결제 처리 — 푸시 알림 {#processing-payments-push-notification}

1. **상품 추가**: POS에 카트에 제품을 추가합니다.
2. **게이트웨이 선택**: 결제 수단으로 "Vipps MobilePay"를 선택합니다.
3. **전화번호 입력**: 고객의 전화번호를 입력합니다.
4. **전화로 전송**: "전화로 전송" 버튼을 클릭합니다.
5. **고객 확인**: 고객이 푸시 알림을 받고 앱에서 결제를 확인합니다.
6. **자동 완료**: 결제가 승인되면 주문이 자동으로 완료됩니다.

### 결제 취소 {#cancelling-a-payment}

고객이 확인하기를 기다리는 동안, **결제 취소** 버튼을 클릭하여 거래를 중단할 수 있습니다. 이는 Vipps 측에서 보류 중인 결제를 취소하고 체크아웃 인터페이스를 초기화합니다.

### 환불 {#refunds}

환불은 표준 WooCommerce 환불 프로세스를 통해 처리됩니다. 주문을 열고 **환불**을 클릭한 다음 금액을 입력하면 환불이 Vipps API를 통해 자동으로 처리됩니다.

## 지원되는 시장 {#supported-markets}

Vipps MobilePay는 두 브랜드로 북유럽 지역에서 운영됩니다:

| 지역 | 브랜드 | 통화 |
|------|-------|------|
| 노르웨이 | Vipps | NOK |
| 덴마크, 핀란드 | MobilePay | DKK, EUR |

상인 계정에 따라 사용 가능한 시장과 통화가 결정됩니다. 지원되는 시장의 고객은 현지 Vipps 또는 MobilePay 앱을 사용하여 결제할 수 있습니다.

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="결제가 완료되지 않음">

  - 체크아웃은 최대 5분 동안 폴링합니다. 고객이 그 시간 내에 확인하지 않으면 결제가 시간 초과됩니다.
  - 테스트 모드와 프로덕션 자격 증명을 혼동하지 않도록 확인합니다.
  - POS 장치와 고객의 전화 모두에서 인터넷 연결이 안정적인지 확인합니다.

  </AccordionItem>

  <AccordionItem question="자격 증명 오류">

  - 모든 네 개의 자격 증명 필드 (MSN, 클라이언트 ID, 클라이언트 비밀, 구독 키)를 다시 확인합니다.
  - 테스트 모드가 활성화된 경우 테스트 자격 증명을 사용하고 비활성화된 경우 프로덕션 자격 증명을 사용하고 있는지 확인합니다.
  - [Vipps 포털](https://portal.vipps.no/)에서 자격 증명을 확인합니다.

  </AccordionItem>

  <AccordionItem question="QR 코드 생성 안 됨">

  - 사이트가 HTTPS에서 실행되고 있는지 확인합니다. Vipps API는 HTTP 사이트의 요청을 거부합니다.
  - API 오류 세부정보를 위해 WooCommerce 로그를 확인합니다 (`WooCommerce > Status > Logs`).
  - 상인 계정이 활성 상태이며 ePayment API가 활성화되어 있는지 확인합니다.

  </AccordionItem>
</Accordion>
