# Vipps MobilePay 게이트웨이 {#vipps-mobilepay-gateway}

Vipps MobilePay 게이트웨이를 사용하면 WCPOS 내에서 직접 전화 기반 결제를 수락할 수 있습니다. 고객은 화면에 있는 QR 코드를 스캔하거나 전화의 푸시 알림을 확인하여 결제합니다 — 카드 리더 하드웨어가 필요 없습니다.

  <FeatureCard icon="globe" title="북유럽 범위">

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [WCPOS Vipps MobilePay 릴리스 페이지](https://github.com/wcpos/wcpos-vipps/releases)를 방문하세요.
2. 최신 **wcpos-vipps.zip** 파일을 다운로드하세요.
3. WordPress 관리에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동하세요.
4. zip 파일을 업로드하고 플러그인을 활성화합니다.

  </StepCard>
  <StepCard number="2" title="Vipps 자격 증명 구성">

1. `WP Admin > WooCommerce > 설정 > 결제`로 이동하세요.
2. 결제 방법 목록에서 **WCPOS Vipps MobilePay**를 찾으세요.
3. **WCPOS Vipps MobilePay**를 클릭하여 설정에 접근합니다.
4. [Vipps Portal](https://portal.vipps.no/)에서 자격 증명을 입력합니다:
   - **상인 일련 번호 (MSN)**
   - **클라이언트 ID**
   - **클라이언트 비밀**
   - **구독 키**
5. 설정을 저장합니다.

:::note
여기서 POS 사용을 위해 게이트웨이를 활성화할 필요는 없습니다 — 다음 단계에서 POS을 위해 특별히 활성화됩니다. WooCommerce 설정에서 활성화하면 온라인 상점 결제에도 사용할 수 있지만, 이는 테스트에 유용합니다.
:::

:::tip 이미 Vipps 플러그인을 사용 중이신가요?
공식 [Checkout with Vipps MobilePay](https://wordpress.org/plugins/woo-vipps/) 플러그인이 설치되어 있다면, 이 플러그인을 활성화할 때 자격 증명이 자동으로 가져와집니다. 수동으로 자격 증명을 입력할 필요가 없습니다.
:::

  </StepCard>
  <StepCard number="3" title="WCPOS에서 활성화" isLast>

1. `WP Admin > POS > 설정 > 결제`로 이동하세요.
2. 목록에서 **WCPOS Vipps MobilePay** 게이트웨이를 찾으세요.
3. POS에서 사용하기 위해 게이트웨이를 활성화합니다.
4. 설정을 저장합니다.

  </StepCard>
</Steps>

## 사용법 {#usage}

### 결제 처리 — QR 코드 {#processing-payments-qr-code}

1. **제품 추가**: POS에서 장바구니에 제품을 추가합니다.
2. **게이트웨이 선택**: 결제 방법으로 "Vipps MobilePay"를 선택합니다.
3. **QR 코드 생성**: "QR 코드 생성" 버튼을 클릭합니다.
4. **고객 스캔**: 고객이 Vipps 또는 MobilePay 앱으로 QR 코드를 스캔합니다.
5. **고객 확인**: 고객이 앱에서 결제를 확인합니다.
6. **자동 완료**: 결제가 승인되면 주문이 자동으로 완료됩니다.

### 결제 처리 — 전화로 전송 {#processing-payments-send-to-phone}

1. **제품 추가**: POS에서 장바구니에 제품을 추가합니다.
2. **게이트웨이 선택**: 결제 방법으로 "Vipps MobilePay"를 선택합니다.
3. **전화번호 입력**: 고객의 전화번호를 입력합니다.
4. **전화로 전송**: "전화로 전송" 버튼을 클릭합니다.
5. **고객 확인**: 고객이 푸시 알림을 받거나 Vipps 랜딩 페이지로 이동하여 앱에서 결제를 확인합니다.
6. **자동 완료**: 결제가 승인되면 주문이 자동으로 완료됩니다.

플러그인은 고객의 전화로 결제 요청을 보내는 최적의 방법을 자동으로 감지합니다:

- **직접 푸시** (선호됨) — 고객의 Vipps 앱으로 알림을 직접 보냅니다. 이는 가장 빠른 경험이지만 Vipps가 귀하의 판매 단위에서 `PUSH_MESSAGE`를 활성화할 것을 요구합니다 (아래 [직접 푸시 활성화](#enabling-direct-push) 참조).
- **랜딩 페이지 폴백** — 직접 푸시가 활성화되어 있지 않은 경우 플러그인은 새 브라우저 탭에서 Vipps 랜딩 페이지를 엽니다. 랜딩 페이지에서 알림을 전송하는 일을 처리합니다. 이는 특별한 승인 없이 즉시 작동합니다.

플러그인이 처음으로 직접 푸시가 사용 가능하지 않다고 감지하면 "전화로 전송"을 다시 클릭하라는 짧은 메시지가 표시됩니다. 그 후로는 결과를 기억하고 원활하게 작동합니다.

### 직접 푸시 활성화 {#enabling-direct-push}

직접 푸시 흐름은 전화 결제에 최상의 경험을 제공합니다 — 추가 탭도 없고, 랜딩 페이지도 없습니다. 활성화하려면:

1. [portal.vippsmobilepay.com](https://portal.vippsmobilepay.com) 에 로그인합니다.
2. Vipps 키 계정 관리자, 파트너 관리자 또는 고객 서비스에 연락합니다.
3. 다음과 같이 요청합니다: **"POS 통합을 위해 제 MSN에서 PUSH_MESSAGE를 활성화해야 합니다."**

승인되면, 플러그인은 24시간 이내에 자동으로 변경 사항을 감지하고 직접 푸시 흐름으로 전환됩니다. 랜딩 페이지 폴백을 사용할 때 게이트웨이 설정 페이지에 알림이 표시됩니다.

### 결제 취소 {#cancelling-a-payment}

고객이 확인할 때까지 기다리는 동안 **결제 취소** 버튼을 클릭하여 거래를 중단할 수 있습니다. 이렇게 하면 Vipps 측의 보류 중인 결제가 취소되고 체크아웃 인터페이스가 재설정됩니다.

### 환불 {#refunds}

환불은 표준 WooCommerce 환불 프로세스를 통해 처리됩니다. 주문을 열고 **환불**을 클릭한 다음 금액을 입력하면, 환불은 자동으로 Vipps API를 통해 처리됩니다.

## 지원 시장 {#supported-markets}

Vipps MobilePay는 두 브랜드 아래 북유럽 지역에서 운영됩니다:

| 지역        | 브랜드       | 통화      |
|-------------|--------------|-----------|
| 노르웨이    | Vipps        | NOK       |
| 덴마크, 핀란드 | MobilePay    | DKK, EUR  |

귀하의 상인 계정에 따라 사용할 수 있는 시장과 통화가 결정됩니다. 지원되는 모든 시장의 고객은 자신의 지역 Vipps 또는 MobilePay 앱을 사용하여 결제할 수 있습니다.

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="결제가 완료되지 않음">

  - 체크아웃은 최대 5분 동안 폴링합니다 — 고객이 그 창 내에서 확인하지 않으면 결제가 시간 초과됩니다.
  - 테스트 모드와 프로덕션 자격 증명을 혼동하지 않는지 확인하세요.
  - POS 장치와 고객의 전화에서 인터넷 연결이 안정적인지 확인하세요.

  </AccordionItem>

  <AccordionItem question="유효하지 않은 자격 증명 오류">

  - 네 개의 자격 증명 필드(MSN, 클라이언트 ID, 클라이언트 비밀, 구독 키)를 다시 확인하세요.
  - 테스트 모드가 활성화되면 테스트 자격 증명을 사용하고 비활성화되면 프로덕션 자격 증명을 사용하세요.
  - [Vipps Portal](https://portal.vipps.no/)에서 자격 증명을 확인하세요.

  </AccordionItem>

  <AccordionItem question="'전화로 전송'이 직접 전송되는 대신 새 탭을 엽니다">

  이는 귀하의 Vipps 계정에 아직 `PUSH_MESSAGE`가 활성화되어 있지 않다는 의미입니다. 플러그인은 랜딩 페이지 폴백을 사용하고 있어 작동하지만 추가 단계를 추가합니다. 보다 원활한 직접 푸시 경험을 원하신다면 Vipps에 연락하여 귀하의 판매 단위(MSN)에서 PUSH_MESSAGE를 활성화하도록 요청하세요. 위의 [직접 푸시 활성화](#enabling-direct-push)를 참조하세요.

  </AccordionItem>

  <AccordionItem question="QR 코드가 생성되지 않음">

  - 귀하의 사이트가 HTTPS에서 실행되고 있는지 확인하세요 — Vipps API는 HTTP 사이트의 요청을 거부합니다.
  - WooCommerce 로그(`WooCommerce > 상태 > 로그`)에서 API 오류 세부 정보를 확인하세요.
  - 귀하의 상인 계정이 활성화되어 있으며 ePayment API가 활성화되어 있는지 확인하세요.

  </AccordionItem>
</Accordion>
