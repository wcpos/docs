# Stripe Terminal 게이트웨이 {#stripe-terminal-gateway}

Stripe Terminal 게이트웨이를 사용하면 WCPOS 내에서 Stripe Terminal 하드웨어 리더기를 통해 대면 결제를 직접 수락할 수 있습니다. 이 게이트웨이는 물리적 카드 리더기와 테스트용 시뮬레이터 모드를 모두 지원합니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="Stripe Terminal for WooCommerce 설치">

`WP Admin > POS > 설정 > 확장`에서 설치하거나, [GitHub 릴리스 페이지](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases)에서 최신 **플러그인 zip 파일**(GitHub 소스 코드 zip 또는 tarball이 아님)을 다운로드한 후 `플러그인 > 새로 추가 > 플러그인 업로드`를 통해 업로드하세요.

  </StepCard>
  <StepCard number="2" title="Stripe 설정 구성">

1. `WP Admin > WooCommerce > 설정 > 결제`로 이동합니다
2. 결제 수단 목록에서 **Stripe Terminal**을 찾습니다
3. **Stripe Terminal**을 클릭하여 설정에 접근합니다
4. **Stripe Secret Key**를 입력합니다(Stripe Dashboard에서 확인할 수 있습니다)
5. 설정을 저장합니다

:::note
WooCommerce 설정에서 Stripe Terminal 게이트웨이를 활성화할 필요는 없습니다. 다음 단계에서 POS 전용으로 활성화됩니다.
:::

  </StepCard>
  <StepCard number="3" title="WCPOS에서 활성화" isLast>

1. `WP Admin > POS > 설정 > 결제`로 이동하세요
2. 목록에서 **Stripe Terminal** 게이트웨이를 찾으세요
3. POS에서 사용할 수 있도록 게이트웨이를 활성화하세요
4. 설정을 저장하세요

  </StepCard>
</Steps>

## 사용 방법 {#usage}

### 리더기 연결 {#connecting-a-reader}

POS에서 결제 시 Stripe Terminal 게이트웨이를 선택하면:

1. **연결 방법 선택**: 실제 리더기를 연결하거나 시뮬레이터를 사용할 수 있습니다
2. **물리적 리더기**: 화면의 안내에 따라 Stripe Terminal 장치를 연결하세요
3. **시뮬레이터**: 하드웨어 없이 다양한 결제 시나리오를 테스트하려면 시뮬레이터 모드를 선택하세요

### 결제 처리 {#processing-payments}

1. **상품 추가**: POS에서 장바구니에 상품을 추가하세요
2. **게이트웨이 선택**: 결제 수단으로 "Stripe Terminal"을 선택하세요
3. **리더기 연결**: 리더기를 연결하거나 시뮬레이터 모드를 선택하세요
4. **결제 처리**: 안내에 따라 거래를 완료하세요
5. **확인**: 결제가 성공하면 주문이 자동으로 완료됩니다

### 시뮬레이터로 테스트 {#testing-with-simulator}

시뮬레이터를 사용하면 다양한 결제 수단과 시나리오를 테스트할 수 있습니다:

- **카드 결제**: 다양한 카드 유형(Visa, Mastercard 등)을 테스트하세요
- **비접촉 결제**: 탭 투 페이 거래를 시뮬레이션하세요
- **오류 시나리오**: 결제 거부 및 기타 오류 상황을 테스트합니다
- **다양한 금액**: 다양한 거래 금액을 테스트합니다

### 전화 주문 (MOTO) {#requirements}

MOTO(Mail Order/Telephone Order)는 고객이 물리적으로 매장에 없는 경우(예: 전화로 주문을 받는 경우) 카드 결제를 처리할 수 있는 기능입니다. 카드를 탭하거나 삽입하는 대신, 판매자가 터미널 리더 화면에 카드 정보를 직접 입력합니다.

#### 설정 {#hardware-compatibility}

<Steps>
  <StepCard number="1" title="Stripe에 MOTO 액세스 요청">

MOTO는 기본적으로 활성화되어 있지 않습니다. [Stripe 지원팀](https://support.stripe.com/contact)에 문의하여 계정에 대한 MOTO 권한 활성화를 요청하세요. 빠르게 처리되지만 Stripe의 수동 승인이 필요합니다.

  </StepCard>
  <StepCard number="2" title="플러그인 설정에서 활성화">

1. `WP Admin > WooCommerce > 설정 > 결제 > Stripe Terminal`로 이동합니다
2. **전화 주문 (MOTO)** 체크박스를 선택합니다
3. 설정을 저장합니다

  </StepCard>
  <StepCard number="3" title="호환 리더 연결" isLast>

MOTO는 [지원되는 단말기](#supported-terminals-internet-connected)에 나열된 호환 가능한 인터넷 연결 리더기에서만 작동합니다. 다른 리더기 유형에서는 토글이 표시되지 않습니다.

  </StepCard>
</Steps>

#### 전화 주문 처리 {#supported-terminals-internet-connected}

1. 호환 가능한 리더기를 연결합니다([지원되는 단말기](#supported-terminals-internet-connected) 참조)
2. 결제 화면에서 **전화 주문**을 켭니다
3. **카드 결제 받기**를 클릭합니다 — 리더기에 탭/삽입 안내 대신 카드 번호 입력 화면이 표시됩니다
4. 리더기에서 고객의 카드 번호, 유효기간, CVV를 입력합니다
5. 이후 결제는 정상적으로 진행됩니다

:::tip
MOTO 결제는 `card`가 아닌 `card_present`를 결제 수단 유형으로 사용합니다. 이는 Stripe 관점에서 온라인 거래와 유사하게 처리됨을 의미하므로, 대면 결제 수수료가 아닌 일반 온라인 카드 처리 수수료가 적용됩니다.
:::

:::caution
전화 주문 토글은 다음 세 가지 조건이 모두 충족될 때만 표시됩니다: 플러그인 설정에서 MOTO 설정이 활성화되어 있고, 호환 가능한 리더기가 연결되어 있으며, 리더기가 시뮬레이터가 아닌 경우입니다. 토글이 보이지 않으면 이 조건들을 확인하십시오.
:::

## 하드웨어 호환성 {#common-issues}

:::important 연결 요구 사항
이 구현은 Stripe의 JavaScript SDK를 사용하므로 웹 애플리케이션을 통해 작동하지만 **인터넷에 연결된 단말기만** 지원합니다. 블루투스 단말기는 현재 지원되지 않습니다.
:::

### 지원 단말기 (인터넷 연결) {#getting-help}

- **Stripe Reader S700/S710** ✅ - 이더넷/WiFi 연결 단말기
- **WisePOS E** ✅ - WiFi 연결 단말기

### 미지원 단말기 (블루투스) {#screenshots}

- **BBPOS Chipper 2X BT** ❌ - 블루투스 전용
- **BBPOS WisePad 3** ❌ - 블루투스 전용  
- **Verifone P400** ❌ - 블루투스 전용

:::note 향후 지원 예정
블루투스 단말기 지원은 **2026년 1분기**에 iOS 및 Android 애플리케이션에 추가될 예정입니다. 이를 통해 M2 및 WisePad 3을 포함한 모든 Stripe Terminal 인증 리더기를 지원할 수 있게 됩니다.
:::

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="Reader won't connect">

  - [지원되는 인터넷 연결 단말기](#supported-terminals-internet-connected)를 사용하고 있는지 확인하세요
  - 단말기가 WiFi/이더넷에 연결되어 있고 온라인 상태인지 확인하세요
  - Stripe 계정에서 Terminal이 활성화되어 있는지 확인하세요
  - 단말기가 Stripe 대시보드에 등록되어 있는지 확인하세요

  </AccordionItem>

  <AccordionItem question="Payment declined">

  - Stripe 계정이 활성 상태이며 정상적으로 유지되고 있는지 확인하세요
  - 사용 중인 카드가 유효한지 확인하세요
  - 잔액이 충분한지 확인하세요

  </AccordionItem>

  <AccordionItem question="Phone Order toggle not showing">

  - `WooCommerce > 설정 > 결제 > Stripe Terminal`에서 **전화 주문(MOTO)** 설정이 활성화되어 있는지 확인하세요
  - 호환되는 리더기에 연결되어 있는지 확인하세요([지원되는 단말기](#supported-terminals-internet-connected) 참조) — 다른 리더기 유형에서는 토글이 표시되지 않습니다
  - 시뮬레이터를 사용할 때는 토글이 표시되지 않습니다

  </AccordionItem>

  <AccordionItem question="MOTO payment fails with an error">

  - Stripe에서 계정에 MOTO 권한을 활성화했는지 확인하세요 — 아직 요청하지 않았다면 [Stripe 지원팀](https://support.stripe.com/contact)에 문의하세요
  - 리더기에 카드 정보가 올바르게 입력되었는지 다시 확인하세요
  - MOTO 결제는 더 엄격한 사기 방지 검사가 적용될 수 있습니다 — 카드가 유효하고 잔액이 충분한지 확인하세요

  </AccordionItem>

  <AccordionItem question="SSL certificate errors">

  - Stripe Terminal은 HTTPS가 필요합니다 - 사이트에 유효한 SSL 인증서가 설치되어 있는지 확인하세요
  - SSL 인증서가 올바르게 구성되어 있는지 확인하세요

  </AccordionItem>
</Accordion>
