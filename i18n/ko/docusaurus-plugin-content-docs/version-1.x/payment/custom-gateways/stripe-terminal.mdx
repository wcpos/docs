# 스트라이프 터미널 게이트웨이 {#stripe-terminal-gateway}

스트라이프 터미널 게이트웨이를 사용하면 WCPOS 내에서 스트라이프 터미널 하드웨어 리더기를 통해 대면 결제를 수락할 수 있습니다. 이 게이트웨이는 실물 카드 리더기와 테스트용 시뮬레이터 모드를 모두 지원합니다.

  </FeatureCard>

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [WooCommerce용 Stripe 터미널 릴리스 페이지](https://github.com/wcpos/stripe-terminal-for-woocommerce/releases)를 방문하십시오.
2. 최신 **stripe-terminal-for-woocommerce.zip** 파일을 다운로드합니다.
3. WordPress 관리에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동합니다.
4. zip 파일을 업로드하고 플러그인을 활성화합니다.

  </StepCard>
  <StepCard number="2" title="스트라이프 설정 구성">

1. `WP 관리 > WooCommerce > 설정 > 결제`로 이동합니다.
2. 결제 방법 목록에서 **스트라이프 터미널**을 찾습니다.
3. **스트라이프 터미널**을 클릭하여 설정에 접근합니다.
4. **스트라이프 비밀 키**를 입력합니다 (스트라이프 대시보드에서 얻을 수 있습니다).
5. 설정을 저장합니다.

:::note
WooCommerce 설정에서 스트라이프 터미널 게이트웨이를 활성화할 필요가 없습니다. 다음 단계에서 POS를 위해 특별히 활성화됩니다.
:::

  </StepCard>
  <StepCard number="3" title="WCPOS에서 활성화" isLast>

1. `WP 관리 > POS > 설정 > 체크아웃`으로 이동합니다.
2. 목록에서 **스트라이프 터미널** 게이트웨이를 찾습니다.
3. POS에서 사용할 수 있도록 게이트웨이를 활성화합니다.
4. 설정을 저장합니다.

  </StepCard>
</Steps>

## 사용법 {#usage}

### 리더기 연결 {#connecting-a-reader}

POS 체크아웃 중 스트라이프 터미널 게이트웨이를 선택하면:

1. **연결 방법 선택**: 물리적인 리더기를 연결하거나 시뮬레이터를 사용할 수 있습니다.
2. **물리적 리더기**: 화면 지침에 따라 스트라이프 터미널 장치를 연결합니다.
3. **시뮬레이터**: 하드웨어 없이 다양한 결제 시나리오를 테스트하기 위해 시뮬레이터 모드를 선택합니다.

### 결제 처리 {#processing-payments}

1. **항목 추가**: POS에 제품을 장바구니에 추가합니다.
2. **게이트웨이 선택**: 결제 방법으로 "스트라이프 터미널"을 선택합니다.
3. **리더기 연결**: 리더기를 연결하거나 시뮬레이터 모드를 선택합니다.
4. **결제 처리**: 거래를 완료하기 위해 프롬프트를 따릅니다.
5. **확인**: 결제가 성공적으로 완료되면 주문이 자동으로 완료됩니다.

### 시뮬레이터로 테스트 {#testing-with-simulator}

시뮬레이터를 사용하면 다양한 결제 방법과 시나리오를 테스트할 수 있습니다:

- **카드 결제**: 다양한 카드 유형(Visa, Mastercard 등)을 테스트합니다.
- **비접촉 결제**: 터치 결제 거래를 시뮬레이션합니다.
- **오류 시나리오**: 결제가 거부된 경우 및 기타 오류 조건을 테스트합니다.
- **다양한 금액**: 다양한 거래 금액을 테스트합니다.

### 전화 주문 (MOTO) {#phone-orders-moto}

MOTO(메일 주문/전화 주문)는 고객이 물리적으로 존재하지 않을 때 카드 결제를 처리할 수 있게 해줍니다. 예를 들어 전화로 주문을 받을 때입니다. 카드를 탭하거나 삽입하는 대신, 상인은 카드 세부 정보를 직접 터미널 리더의 화면에 입력합니다.

#### 설정 {#setup}

<Steps>
  <StepCard number="1" title="스트라이프에 MOTO 액세스 요청">

MOTO는 기본적으로 활성화되어 있지 않습니다. [스트라이프 지원](https://support.stripe.com/contact)에 연락하여 계정에 대해 MOTO 권한을 활성화해 달라고 요청하십시오. 이 과정은 간단하지만, 스트라이프의 수동 승인이 필요합니다.

  </StepCard>
  <StepCard number="2" title="플러그인 설정에서 활성화">

1. `WP 관리 > WooCommerce > 설정 > 결제 > 스트라이프 터미널`로 이동합니다.
2. **전화 주문 (MOTO)** 확인란을 체크합니다.
3. 설정을 저장합니다.

  </StepCard>
  <StepCard number="3" title="호환 가능한 리더기 연결" isLast>

MOTO는 [지원되는 터미널](#supported-terminals-internet-connected) 목록에 있는 호환 가능한 인터넷 연결 리더기에서만 작동합니다. 다른 리더기 유형에 대해서는 토글이 표시되지 않습니다.

  </StepCard>
</Steps>

#### 전화 주문 수락 {#taking-a-phone-order}

1. 호환 가능한 리더기를 연결합니다 (참조: [지원되는 터미널](#supported-terminals-internet-connected))
2. 결제 화면에서 **전화 주문**을 활성화합니다.
3. **카드 결제 수집** 클릭 — 리더기는 탭/삽입 프롬프트 대신 카드 번호 입력 화면을 표시합니다.
4. 고객의 카드 번호, 만료일 및 CVV를 리더기에 입력합니다.
5. 결제는 그 다음 단계에서 정상적으로 진행됩니다.

:::tip
MOTO 결제는 `card_present` 대신 `card`를 결제 방법 유형으로 사용합니다. 이는 스트라이프 관점에서 온라인 거래와 유사하게 처리되므로, 대면 비율이 아닌 표준 온라인 카드 처리 수수료가 적용됩니다.
:::

:::caution
전화 주문 토글은 모든 세 가지 조건이 충족될 때만 나타납니다: 플러그인 설정에서 MOTO 설정이 활성화되어 있고, 호환 가능한 리더기가 연결되어 있으며, 리더기가 시뮬레이터가 아닐 경우입니다. 토글이 보이지 않으면 이러한 조건을 확인하십시오.
:::

## 하드웨어 호환성 {#hardware-compatibility}

:::important 연결 요구 사항
이 구현에서는 스트라이프의 JavaScript SDK를 사용하므로, 웹 응용 프로그램을 통해 작동하지만 **인터넷에 연결된 터미널만** 필요합니다. 블루투스 터미널은 현재 지원되지 않습니다.
:::

### 지원되는 터미널 (인터넷 연결) {#supported-terminals-internet-connected}

- **스트라이프 리더 S700/S710** ✅ - 이더넷/WiFi 연결 터미널
- **WisePOS E** ✅ - WiFi 연결 터미널

### 지원되지 않는 터미널 (블루투스) {#unsupported-terminals-bluetooth}

- **BBPOS Chipper 2X BT** ❌ - 블루투스 전용
- **BBPOS WisePad 3** ❌ - 블루투스 전용  
- **Verifone P400** ❌ - 블루투스 전용

:::note 향후 지원
2026년 1분기에는 iOS 및 Android 응용 프로그램에 블루투스 터미널 지원이 추가될 것입니다. 이를 통해 M2 및 WisePad 3를 포함한 모든 스트라이프 터미널 인증 리더기를 지원할 수 있게 됩니다.
:::

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="리더기가 연결되지 않음">

  - [지원되는 인터넷 연결 터미널](#supported-terminals-internet-connected)을 사용하고 있는지 확인합니다.
  - 터미널이 WiFi/이더넷에 연결되어 있고 온라인 상태인지 확인합니다.
  - 스트라이프 계정에 터미널이 활성화되어 있는지 확인합니다.
  - 터미널이 스트라이프 대시보드에서 등록되어 있는지 확인합니다.

  </AccordionItem>

  <AccordionItem question="결제 거부됨">

  - 스트라이프 계정이 활성 상태이고 문제가 없는지 확인합니다.
  - 사용 중인 카드가 유효한지 확인합니다.
  - 충분한 자금이 있는지 확인합니다.

  </AccordionItem>

  <AccordionItem question="전화 주문 토글이 표시되지 않음">

  - `WooCommerce > 설정 > 결제 > 스트라이프 터미널`에서 **전화 주문 (MOTO)** 설정이 활성화되어 있는지 확인합니다.
  - 호환 가능한 리더기에 연결되어 있는지 확인합니다(참조: [지원되는 터미널](#supported-terminals-internet-connected)) — 다른 리더기 유형에 대해서는 토글이 숨겨져 있습니다.
  - 시뮬레이터를 사용할 때는 토글이 표시되지 않습니다.

  </AccordionItem>

  <AccordionItem question="MOTO 결제가 오류로 실패함">

  - 스트라이프가 귀하의 계정에서 MOTO 권한을 활성화했는지 확인합니다 — 아직 연락하지 않았다면 [스트라이프 지원](https://support.stripe.com/contact)에게 연락하십시오.
  - 카드 세부 정보가 리더기에 올바르게 입력되었는지 다시 확인합니다.
  - MOTO 결제는 사기 검사 기준이 더 엄격할 수 있습니다 — 카드가 유효하고 충분한 자금이 있는지 확인하십시오.

  </AccordionItem>

  <AccordionItem question="SSL 인증서 오류">

  - 스트라이프 터미널은 HTTPS를 요구합니다 - 사이트에 유효한 SSL 인증서가 있는지 확인하십시오.
  - SSL 인증서가 올바르게 구성되었는지 확인합니다.

  </AccordionItem>
</Accordion>

### 도움 요청하기 {#getting-help}

기술 지원:
- [GitHub 리포지토리](https://github.com/wcpos/stripe-terminal-for-woocommerce)를 방문하여 문제를 신고합니다.
- 하드웨어별 지침을 보려면 [스트라이프 터미널 문서](https://stripe.com/docs/terminal)를 확인합니다.
- 계정 관련 문제는 스트라이프 지원에 문의하십시오.

## 스크린샷 {#screenshots}

스크린샷은 다음과 같은 내용을 보여주기 위해 향후 업데이트에서 추가될 것입니다:
- WooCommerce 결제 설정에서 게이트웨이 구성
- 스트라이프 터미널 선택이 포함된 POS 체크아웃 인터페이스
