# SumUp 터미널 게이트웨이

SumUp 터미널 게이트웨이를 사용하면 WooCommerce POS 내에서 SumUp 카드 리더기를 통해 카드 결제를 수락할 수 있습니다. 이 게이트웨이는 대면 거래를 위한 SumUp의 결제 처리 시스템과 원활하게 통합됩니다.

## 기능

- **하드웨어 통합**: 인터넷 연결을 통해 SumUp 카드 리더기를 POS 시스템에 연결
- **실시간 처리**: 즉각적인 결제 처리 및 주문 완료
- **자동 감지**: 결제가 감지되면 주문이 자동으로 완료됨
- **보안 거래**: SumUp을 통한 PCI 규정 준수 결제 처리
- **쉬운 페어링**: 페어링 코드를 사용한 간단한 장치 페어링 프로세스
- **웹 기반 SDK**: 웹 애플리케이션 호환성을 위한 SumUp의 JavaScript SDK 사용

## 설치

### 1단계: 다운로드 및 설치

1. [SumUp 터미널 for WooCommerce 릴리스 페이지](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases) 방문
2. 최신 **sumup-terminal-for-woocommerce.zip** 파일 다운로드
3. WordPress 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일 업로드 및 플러그인 활성화

### 2단계: SumUp 설정 구성

1. `WP 관리자 > WooCommerce > 설정 > 결제`로 이동
2. 결제 방법 목록에서 **SumUp 터미널** 찾기
3. 설정에 접근하려면 **SumUp 터미널** 클릭
4. **SumUp API 키** 입력 (SumUp 판매자 대시보드에서 확인 가능)
5. 설정 저장

:::note
WooCommerce 설정에서 SumUp 터미널 게이트웨이를 활성화할 필요가 없습니다. 다음 단계에서 POS에 대해 특별히 활성화됩니다.
:::

### 3단계: SumUp 터미널 페어링

1. 동일한 설정 페이지에서 **리더기 페어링** 섹션 찾기
2. SumUp 장치에서 페어링 화면으로 이동하여 페어링 코드 표시
3. SumUp 장치에 표시된 페어링 코드 입력
4. **"리더기 페어링"** 클릭하여 연결 설정
5. 리더기가 성공적으로 페어링되었다는 확인 대기

:::warning 중요
결제를 처리하려면 리더기를 성공적으로 페어링해야 합니다. 계속 진행하기 전에 페어링 프로세스가 완료되었는지 확인하세요.
:::

### 4단계: WooCommerce POS에서 활성화

1. `WP 관리자 > WooCommerce POS > 설정 > 체크아웃`로 이동
2. 목록에서 **SumUp 터미널** 게이트웨이 찾기
3. POS에서 사용할 게이트웨이 활성화
4. 설정 저장

## 사용법

### 결제 처리

1. **항목 추가**: POS에서 장바구니에 상품 추가
2. **게이트웨이 선택**: 결제 방법으로 "SumUp 터미널" 선택
3. **결제 시작**: SumUp 장치에서 새 결제 시작 클릭
4. **고객 결제**: 고객이 SumUp 터미널에서 결제 완료
5. **자동 완료**: 결제가 감지되면 주문 자동 완료

### 결제 제어

SumUp 터미널 게이트웨이를 사용할 때 다음 옵션을 사용할 수 있습니다:

- **새 결제 시작**: 연결된 터미널에서 결제 요청 시작
- **결제 취소**: 진행 중인 결제 취소
- **결제 상태**: 결제 프로세스의 현재 상태 모니터링

### 주문 관리

- **자동 업데이트**: 성공적인 결제 시 주문이 자동으로 완료됨
- **결제 추적**: 모든 결제 세부 정보가 주문 메모에 기록됨
- **영수증 생성**: 성공적인 결제 후 표준 POS 영수증 생성

## 요구 사항

- **SumUp 계정**: 활성 SumUp 판매자 계정
- **API 액세스**: 판매자 대시보드의 SumUp API 키
- **호환 하드웨어**: SumUp 카드 리더기 장치
- **WooCommerce POS**: 무료 또는 프로 버전 설치 및 활성화
- **안정적인 연결**: API 통신을 위한 안정적인 인터넷 연결

## 하드웨어 호환성

:::important 연결 요구 사항
이 구현은 SumUp의 JavaScript SDK를 사용하므로 웹 애플리케이션을 통해 작동하지만 **인터넷에 연결된 터미널만** 필요합니다. 블루투스 터미널은 현재 지원되지 않습니다.
:::

### 지원되는 터미널 (인터넷 연결)

- **SumUp Solo** ✅ - WiFi/이더넷 연결 터미널

### 지원되지 않는 터미널 (블루투스)

- **SumUp Air** ❌ - 블루투스 전용
- **SumUp 3G** ❌ - 블루투스 전용

:::note 향후 지원
블루투스 터미널 지원은 **2026년 1분기**에 iOS 및 Android 애플리케이션에 추가될 예정입니다. 이를 통해 Air 및 3G 모델을 포함한 모든 SumUp 인증 터미널을 지원할 수 있습니다.
:::

## 문제 해결

### 일반적인 문제

**리더기가 페어링되지 않음**
- 지원되는 인터넷 연결 터미널(Solo만) 사용 확인
- SumUp Solo 장치가 WiFi에 연결되고 온라인 상태인지 확인
- 장치가 페어링 모드에 있고 페어링 코드가 올바르게 입력되었는지 확인
- SumUp 계정에 API 액세스가 활성화되어 있는지 확인
- 페어링 중 안정적인 인터넷 연결 확인

**결제가 처리되지 않음**
- 리더기가 제대로 페어링되었는지 확인
- SumUp 계정이 활성 상태이고 정상인지 확인
- 장치의 배터리와 연결 상태 확인
- API 키가 올바르고 활성 상태인지 확인

**주문이 완료되지 않음**
- 인터넷 연결 안정성 확인
- SumUp 계정의 웹훅 설정 확인
- SumUp 터미널에서 결제가 성공했는지 확인
- API 통신 문제에 대해 WordPress 오류 로그 확인

### API 속도 제한

게이트웨이는 API 호출을 줄이고 속도 제한을 방지하도록 최적화되었습니다:
- 결제 상태를 효율적으로 확인
- 불필요한 API 요청 최소화
- 일시적인 장애를 처리하는 자동 재시도 로직

### 도움 받기

기술 지원을 위해:
- 문제를 보고하려면 [GitHub 저장소](https://github.com/wcpos/sumup-terminal-for-woocommerce) 방문
- API 관련 질문은 [SumUp 개발자 문서](https://developer.sumup.com/) 확인
- 계정 및 하드웨어 문제는 SumUp 지원팀에 문의
