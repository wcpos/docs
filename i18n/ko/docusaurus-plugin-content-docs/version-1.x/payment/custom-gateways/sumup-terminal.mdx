# SumUp 단말기 게이트웨이 {#sumup-terminal-gateway}

SumUp 단말기 게이트웨이를 통해 WooCommerce POS 내에서 직접 SumUp 카드 리더기를 사용하여 카드 결제를 수락할 수 있습니다. 이 게이트웨이는 대면 거래를 위한 SumUp의 결제 처리 시스템과 원활하게 통합됩니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [SumUp Terminal for WooCommerce 릴리스 페이지](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)에 방문하세요.
2. 최신 **sumup-terminal-for-woocommerce.zip** 파일을 다운로드하세요.
3. WordPress 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동하세요.
4. zip 파일을 업로드하고 플러그인을 활성화하세요.

  </StepCard>
  <StepCard number="2" title="SumUp 설정 구성">

1. `WP Admin > WooCommerce > 설정 > 결제`로 이동하세요.
2. 결제 방법 목록에서 **SumUp Terminal**을 찾으세요.
3. **SumUp Terminal**을 클릭하여 설정에 접근하세요.
4. **SumUp API 키**를 입력하세요 (SumUp 상인 대시보드에서 확인 가능).
5. 설정을 저장하세요.

:::note
WooCommerce 설정에서 SumUp 단말기 게이트웨이를 활성화할 필요는 없습니다. 나중에 POS에 대해 특별히 활성화됩니다.
:::

  </StepCard>
  <StepCard number="3" title="SumUp 단말기 페어링">

1. 같은 설정 페이지에서 **리더 페어링** 섹션을 찾으세요.
2. SumUp 장치에서 페어링 화면으로 이동하여 페어링 코드를 표시하세요.
3. SumUp 장치에 표시된 페어링 코드를 입력하세요.
4. 연결을 설정하려면 **"리더 페어링"**을 클릭하세요.
5. 리더가 성공적으로 페어링되었다는 확인을 기다리세요.

:::warning 중요
결제를 처리하기 전에 리더가 성공적으로 페어링되어야 합니다. 진행하기 전에 페어링 프로세스가 완료되었는지 확인하세요.
:::

  </StepCard>
  <StepCard number="4" title="WooCommerce POS에서 활성화" isLast>

1. `WP Admin > WooCommerce POS > 설정 > 결제 체크아웃`으로 이동하세요.
2. 목록에서 **SumUp Terminal** 게이트웨이를 찾으세요.
3. POS에서 사용하기 위해 게이트웨이를 활성화하세요.
4. 설정을 저장하세요.

  </StepCard>
</Steps>

## 사용법 {#usage}

### 결제 처리 {#processing-payments}

1. **상품 추가**: POS에서 장바구니에 상품을 추가하세요.
2. **게이트웨이 선택**: 결제 방법으로 "SumUp Terminal"을 선택하세요.
3. **결제 시작**: SumUp 장치에서 새로운 결제를 시작하려면 클릭하세요.
4. **고객 결제**: 고객이 SumUp 단말기에서 결제를 완료하세요.
5. **자동 완료**: 결제가 감지되면 주문이 자동으로 완료됩니다.

### 결제 제어 {#payment-controls}

SumUp 단말기 게이트웨이를 사용할 때 다음 옵션이 있습니다:

- **새로운 결제 시작**: 연결된 단말기에서 결제 요청을 시작하세요.
- **결제 취소**: 현재 진행 중인 결제를 취소하세요.
- **결제 상태**: 결제 프로세스의 현재 상태를 모니터링하세요.

### 주문 관리 {#order-management}

- **자동 업데이트**: 결제가 성공적으로 이루어지면 주문이 자동으로 완료됩니다.
- **결제 추적**: 모든 결제 세부 사항은 주문 메모에 기록됩니다.
- **영수증 생성**: 성공적인 결제 후 표준 POS 영수증이 생성됩니다.

## 하드웨어 호환성 {#hardware-compatibility}

:::important 연결 요구 사항
이 구현은 SumUp의 JavaScript SDK를 사용하므로 웹 애플리케이션을 통해 작동하지만 **인터넷이 연결된 단말기만 필요합니다**. 현재 Bluetooth 단말기는 지원되지 않습니다.
:::

### 지원되는 단말기 (인터넷 연결) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet 연결된 단말기

### 지원되지 않는 단말기 (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Bluetooth만 가능
- **SumUp 3G** ❌ - Bluetooth만 가능

:::note 향후 지원
Bluetooth 단말기 지원은 **2026년 1분기**에 iOS 및 Android 애플리케이션에 추가될 예정입니다. 이는 Air 및 3G 모델을 포함한 모든 SumUp 인증 단말기에 대한 지원을 가능하게 합니다.
:::

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="리더가 페어링되지 않음">

  - 지원되는 인터넷 연결 단말기(Solo 전용)를 사용하고 있는지 확인하세요.
  - SumUp Solo 장치가 WiFi에 연결되어 온라인 상태인지 확인하세요.
  - 장치가 페어링 모드에 있고 페어링 코드가 올바르게 입력되었는지 확인하세요.
  - SumUp 계정이 API 접근이 활성화되어 있는지 확인하세요.
  - 페어링 중 안정적인 인터넷 연결이 이루어졌는지 확인하세요.

  </AccordionItem>

  <AccordionItem question="결제가 처리되지 않음">

  - 리더가 올바르게 페어링되었는지 확인하세요.
  - SumUp 계정이 활성 상태인지 확인하세요.
  - 장치가 충분한 배터리와 연결 상태인지 확인하세요.
  - API 키가 올바르고 활성화되어 있는지 확인하세요.

  </AccordionItem>

  <AccordionItem question="주문이 완료되지 않음">

  - 인터넷 연결의 안정성을 확인하세요.
  - SumUp 계정의 웹훅 설정을 확인하세요.
  - SumUp 단말기에서 결제가 성공적으로 이루어졌는지 확인하세요.
  - API 통신 문제에 대한 WordPress 오류 로그를 확인하세요.

  </AccordionItem>
</Accordion>

### API 속도 제한 {#api-rate-limits}

게이트웨이는 API 호출을 줄이고 속도 제한을 피하도록 최적화되어 있습니다:
- 결제 상태는 효율적으로 확인됩니다.
- 불필요한 API 요청이 최소화됩니다.
- 자동 재시도 로직은 일시적인 실패를 처리합니다.

### 도움받기 {#getting-help}

기술 지원을 위해:
- 문제를 보고하려면 [GitHub 리포지토리](https://github.com/wcpos/sumup-terminal-for-woocommerce)를 방문하세요.
- API 관련 질문은 [SumUp 개발자 문서](https://developer.sumup.com/)를 확인하세요.
