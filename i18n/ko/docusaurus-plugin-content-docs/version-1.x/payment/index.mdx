WCPOS는 다양한 비즈니스 요구를 충족하기 위해 여러 결제 방법을 지원합니다. 체크아웃은 iframe/webview를 사용하여 WooCommerce Order Pay 페이지를 로드하므로 WooCommerce에서 작동하는 모든 결제 게이트웨이가 POS에서 작동할 수 있습니다.

## 기본 결제 방법

무료 버전에는 두 가지 내장 결제 게이트웨이가 포함되어 있습니다:

### 현금

- 결제 금액 입력
- 자동 거스름돈 계산
- 추가 구성이 필요하지 않음

### 카드

- 외부 카드 단말기와 함께 사용하기 위함
- 터미널에서 처리 후 결제를 완료로 표시하기만 하면 됩니다
- 직접 통합이 필요하지 않습니다

## 추가 결제 게이트웨이 (Pro)

:::info Pro Feature
추가 결제 게이트웨이는 [WCPOS Pro](/getting-started/pro-license)가 필요합니다.
:::

Pro를 사용하면 POS 체크아웃에서 추가 WooCommerce 결제 게이트웨이를 활성화할 수 있습니다:

- **Stripe Terminal** - Stripe 카드 리더와의 직접 통합
- **SumUp Terminal** - SumUp 카드 리더와의 통합
- **Any WooCommerce Gateway** - 스토어에 설치된 모든 게이트웨이 활성화

## 게이트웨이 활성화 및 비활성화

결제 게이트웨이는 WordPress 관리자에서 관리됩니다:

1. 로 이동하세요 `WP Admin > POS > Settings > Checkout`
2. 설치된 모든 WooCommerce 결제 게이트웨이 목록이 표시됩니다
3. POS에서 온라인 스토어와 독립적으로 각 게이트웨이를 켜거나 끕니다
4. 드래그하여 순서를 다시 정렬하세요 — 첫 번째 활성화된 게이트웨이가 체크아웃에서 기본값이 됩니다

이는 게이트웨이가 웹사이트에서는 활성화되어 있지만 POS에서는 비활성화될 수 있으며, 그 반대도 가능함을 의미합니다. 자세한 내용은 [Checkout Settings](/settings/wp-admin/checkout)을 참조하세요.

## 결제 테스트

새로운 게이트웨이를 라이브로 전환하기 전에:

1. **Cash gateway** — 실제 거래 없이 체크아웃 플로우를 테스트하기 위해 항상 사용 가능합니다.
2. **Stripe Terminal** — Stripe는 테스트 모드를 위해 [simulated reader](https://docs.stripe.com/terminal/testing)를 제공합니다. 먼저 Stripe 대시보드에서 테스트 모드를 활성화하세요.
3. **Third-party gateways** — 게이트웨이 자체 설명서에서 샌드박스/테스트 모드 지침을 확인하세요. 대부분의 WooCommerce 게이트웨이는 테스트 모드 토글을 지원합니다.

:::tip
새로 활성화된 게이트웨이가 체크아웃에 표시되지 않으면 POS를 새로 고침하세요. WP Admin의 게이트웨이 변경사항은 다음 페이지 로드 시 적용됩니다.
:::

## 일반적인 문제

### 결제 시 게이트웨이가 표시되지 않음

- POS용 게이트웨이가 다음에서 활성화되었는지 확인하세요 `WP Admin > POS > Settings > Checkout`.
- 게이트웨이가 다음에서도 활성화되었는지 확인하세요 `WP Admin > WooCommerce > Settings > Payments` — 게이트웨이는 POS에 표시되기 전에 WooCommerce에서 활성화되어야 합니다.
- 변경 후 POS를 새로 고치세요.

### 게이트웨이가 올바르게 표시되지 않음

결제 iframe이 사이트의 테마 스타일을 로드하여 간섭이 발생할 수 있습니다. 결제 모달의 **Checkout Settings** 버튼을 사용하여 스타일 또는 스크립트를 선택적으로 비활성화하세요. 자세한 내용은 [Checkout Troubleshooting](/pos/checkout#checkout-settings-troubleshooting)을 참조하세요.

### 결제 처리 실패

- 사이트의 오류 로그 확인 (`WP Admin > POS > Support > Logs`) 게이트웨이별 오류에 대해.
- SSL 인증서가 유효한지 확인하십시오 — 대부분의 결제 게이트웨이는 HTTPS가 필요합니다.
- 스테이징 또는 로컬 환경을 사용하는 경우 게이트웨이가 테스트/샌드박스 모드를 지원하는지 확인하십시오.
