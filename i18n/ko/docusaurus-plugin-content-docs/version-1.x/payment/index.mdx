WCPOS는 다양한 비즈니스 요구에 맞게 여러 결제 방법을 지원합니다. 결제 화면은 iframe/webview를 사용해 WooCommerce 주문 결제 페이지를 로드하므로, WooCommerce에서 작동하는 모든 결제 게이트웨이를 POS에서도 사용할 수 있습니다.

## 기본 결제 방법 {#default-payment-methods}

무료 버전에는 두 가지 내장 결제 게이트웨이가 포함되어 있습니다.

### 현금 {#cash}

- 받은 금액 입력
- 거스름돈 자동 계산
- 추가 구성이 필요하지 않음

### 카드 {#card}

- 외부 카드 단말기와 함께 사용
- 터미널에서 처리한 후 결제를 완료로 표시하기만 하면 됩니다
- 직접 통합이 필요하지 않음

## 추가 결제 게이트웨이(Pro) {#additional-payment-gateways-pro}

:::info Pro 기능
추가 결제 게이트웨이를 사용하려면 [WCPOS Pro](/getting-started/pro-license)가 필요합니다.
:::

Pro를 사용하면 POS 결제에서 추가 WooCommerce 결제 게이트웨이를 활성화할 수 있습니다.

- **Stripe Terminal** - Stripe 카드 리더와 직접 통합
- **SumUp Terminal** - SumUp 카드 리더와 통합
- **모든 WooCommerce 게이트웨이** - 매장에 설치된 모든 게이트웨이 활성화

## 게이트웨이 활성화 및 비활성화 {#enabling-and-disabling-gateways}

결제 게이트웨이는 WordPress 관리자에서 관리합니다.

1. 다음 경로로 이동합니다 `WP Admin > POS > 설정 > 결제`
2. 설치된 모든 WooCommerce 결제 게이트웨이 목록이 표시됩니다
3. 온라인 매장과 별도로 POS에서 각 게이트웨이를 켜거나 끕니다
4. 드래그하여 순서를 변경합니다 — 첫 번째로 활성화된 게이트웨이가 결제 시 기본값이 됩니다

즉, 게이트웨이를 웹사이트에서는 활성화하고 POS에서는 비활성화하거나, 그 반대로 설정할 수 있습니다. 자세한 내용은 [결제 설정](/settings/wp-admin/checkout)을 참조하세요.

## 결제 테스트 {#testing-payments}

새 게이트웨이를 운영 환경에 적용하기 전:

1. **현금 게이트웨이** — 실제 거래 없이 결제 흐름을 테스트할 때 항상 사용할 수 있습니다.
2. **Stripe Terminal** — Stripe는 테스트 모드용 [시뮬레이션 리더](https://docs.stripe.com/terminal/testing)를 제공합니다. 먼저 Stripe 대시보드에서 테스트 모드를 활성화하세요.
3. **서드파티 게이트웨이** — 샌드박스/테스트 모드 지침은 해당 게이트웨이의 문서를 확인하세요. 대부분의 WooCommerce 게이트웨이는 테스트 모드 전환 옵션을 지원합니다.

:::tip
새로 활성화한 게이트웨이가 결제 화면에 표시되지 않으면 POS를 새로고침해 보세요. WP Admin에서 변경한 게이트웨이 설정은 다음 페이지 로드 시 적용됩니다.
:::

## 일반적인 문제 {#common-issues}

### 결제 시 게이트웨이가 표시되지 않음 {#gateway-not-appearing-at-checkout}

- 게이트웨이가 POS에 활성화되어 있는지 다음 위치에서 확인합니다 `WP Admin > POS > 설정 > 결제`.
- 게이트웨이가 다음 위치에서도 활성화되어 있는지 확인합니다 `WP Admin > WooCommerce > 설정 > 결제` — 게이트웨이가 POS에 표시되려면 먼저 WooCommerce에서 활성 상태여야 합니다.
- 변경 후 POS를 새로 고칩니다.

### 게이트웨이가 올바르게 표시되지 않음 {#gateway-displays-incorrectly}

결제 iframe은 사이트의 테마 스타일을 로드하므로 간혹 충돌이 발생할 수 있습니다. 결제 모달의 **결제 설정** 버튼을 사용하여 스타일이나 스크립트를 선택적으로 비활성화합니다. 자세한 내용은 [결제 문제 해결](/pos/checkout#checkout-settings-troubleshooting)을 참조하세요.

### 결제 처리가 실패함 {#payment-processing-fails}

- 게이트웨이별 오류는 사이트의 오류 로그(`WP Admin > POS > 지원 > 로그`)에서 확인합니다.
- SSL 인증서가 유효한지 확인합니다 — 대부분의 결제 게이트웨이는 HTTPS를 요구합니다.
- 스테이징 또는 로컬 환경을 사용하는 경우 게이트웨이가 테스트/샌드박스 모드를 지원하는지 확인합니다.
