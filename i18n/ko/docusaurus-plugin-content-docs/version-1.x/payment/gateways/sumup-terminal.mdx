# SumUp 터미널 게이트웨이 {#sumup-terminal-gateway}

SumUp 터미널 게이트웨이를 사용하면 WCPOS 내에서 직접 SumUp 카드 리더기를 사용하여 카드 결제를 받을 수 있습니다. 이 게이트웨이는 대면 거래를 위한 SumUp의 결제 처리 시스템과 원활하게 통합됩니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [SumUp Terminal for WooCommerce 릴리스 페이지](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)에 방문
2. 최신 **sumup-terminal-for-woocommerce.zip** 파일 다운로드
3. WordPress 관리자에서 `플러그인 > 새로 추가 > 플러그인 업로드`로 이동
4. zip 파일을 업로드하고 플러그인을 활성화

  </StepCard>
  <StepCard number="2" title="SumUp 설정 구성">

1. `WP Admin > WooCommerce > 설정 > 결제`로 이동
2. 결제 방법 목록에서 **SumUp 터미널** 찾기
3. **SumUp 터미널**을 클릭하여 설정에 접근
4. **SumUp API 키** 입력 (SumUp 상인 대시보드에서 확인 가능)
5. 설정 저장

:::note
WooCommerce 설정에서 SumUp 터미널 게이트웨이를 활성화할 필요는 없습니다. 이후 단계에서 POS에 대해 특별히 활성화됩니다.
:::

  </StepCard>
  <StepCard number="3" title="SumUp 터미널 페어링">

1. 같은 설정 페이지에서 **리더 페어링** 섹션 찾기
2. SumUp 장치에서 페어링 코드를 표시하기 위해 페어링 화면으로 이동
3. SumUp 장치에 표시된 페어링 코드 입력
4. **"리더 페어링"** 클릭하여 연결 성립
5. 리더가 성공적으로 페어링되었다는 확인 기다리기

:::warning 중요
결제를 처리하기 전에 리더가 성공적으로 페어링되어야 합니다. 진행하기 전에 페어링 프로세스가 완료되었는지 확인하십시오.
:::

  </StepCard>
  <StepCard number="4" title="WCPOS에서 활성화" isLast>

1. `WP Admin > POS > 설정 > 결제`로 이동
2. 목록에서 **SumUp 터미널** 게이트웨이 찾기
3. POS에서 사용하기 위해 게이트웨이 활성화
4. 설정 저장

  </StepCard>
</Steps>

## 사용 방법 {#usage}

### 결제 처리 {#processing-payments}

1. **항목 추가**: POS에서 장바구니에 제품 추가
2. **게이트웨이 선택**: 결제 방법으로 "SumUp 터미널" 선택
3. **결제 시작**: SumUp 장치에서 새 결제 시작 클릭
4. **고객 결제**: 고객이 SumUp 터미널에서 결제를 완료
5. **자동 완료**: 결제가 감지되면 주문이 자동으로 완료됨

### 결제 제어 {#payment-controls}

SumUp 터미널 게이트웨이를 사용할 때 다음과 같은 옵션이 있습니다:

- **새 결제 시작**: 연결된 터미널에서 결제 요청 시작
- **결제 취소**: 현재 진행 중인 결제 취소
- **결제 상태**: 결제 처리의 현재 상태 모니터링

### 주문 관리 {#order-management}

- **자동 업데이트**: 성공적인 결제 후 주문이 자동으로 완료로 표시됨
- **결제 추적**: 모든 결제 세부 정보가 주문 노트에 기록됨
- **영수증 생성**: 성공적인 결제 후 표준 POS 영수증이 생성됨

## 하드웨어 호환성 {#hardware-compatibility}

:::important 연결 요구 사항
이 구현은 SumUp의 JavaScript SDK를 사용하므로 웹 애플리케이션을 통해 작동하지만 **인터넷에 연결된 터미널만 필요합니다**. Bluetooth 터미널은 현재 지원되지 않습니다.
:::

### 지원되는 터미널 (인터넷 연결) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet 연결된 터미널

### 지원되지 않는 터미널 (Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Bluetooth 전용
- **SumUp 3G** ❌ - Bluetooth 전용

:::note 향후 지원
Bluetooth 터미널 지원은 **2026년 1분기**에 iOS 및 Android 애플리케이션에 추가될 예정입니다. 이를 통해 Air 및 3G 모델을 포함한 모든 SumUp 인증 터미널에 대한 지원이 가능해집니다.
:::

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="리더가 페어링되지 않음">

  - 지원되는 인터넷 연결 터미널(Solo만 사용하는지 확인)
  - SumUp Solo 장치가 WiFi에 연결되어 있고 온라인인지 확인
  - 장치가 페어링 모드에 있고 페어링 코드가 정확히 입력되었는지 확인
  - SumUp 계정에서 API 접근이 활성화되어 있는지 확인
  - 페어링 중 안정적인 인터넷 연결이 유지되도록 확인

  </AccordionItem>

  <AccordionItem question="결제가 처리되지 않음">

  - 리더가 올바르게 페어링되었는지 확인
  - SumUp 계정이 활성 상태이고 양호한 상태인지 확인
  - 장치에 충분한 배터리와 연결성이 있는지 확인
  - API 키가 정확하고 활성 상태인지 확인

  </AccordionItem>

  <AccordionItem question="주문이 완료되지 않음">

  - 인터넷 연결 안정성을 확인
  - SumUp 계정에서 웹훅 설정을 확인
  - SumUp 터미널에서 결제가 성공했는지 확인
  - API 통신 문제를 위한 WordPress 오류 로그 확인

  </AccordionItem>
</Accordion>

### API 속도 제한 {#api-rate-limits}

게이트웨이는 API 호출을 줄이고 속도 제한을 피하도록 최적화되었습니다:
- 결제 상태가 효율적으로 확인됩니다
- 불필요한 API 요청이 최소화됩니다
- 자동 재시도 로직이 임시 실패를 처리합니다

### 도움 받기 {#getting-help}

기술 지원을 받으려면:
- 문제를 보고하려면 [GitHub 리포지토리](https://github.com/wcpos/sumup-terminal-for-woocommerce)를 방문하십시오.
- API 관련 질문에 대한 [SumUp 개발자 문서](https://developer.sumup.com/)를 확인하십시오.
