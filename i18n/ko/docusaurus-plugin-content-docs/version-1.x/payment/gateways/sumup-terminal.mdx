# SumUp Terminal 게이트웨이 {#sumup-terminal-gateway}

SumUp Terminal 게이트웨이를 사용하면 WCPOS 내에서 SumUp 카드 리더기로 카드 결제를 받을 수 있습니다. 이 게이트웨이는 대면 거래를 위해 SumUp 결제 처리 시스템과 원활하게 통합됩니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="WooCommerce용 SumUp Terminal 설치">

`WP Admin > POS > 설정 > 확장` 에서 설치하거나, [GitHub 릴리스 페이지](https://github.com/wcpos/sumup-terminal-for-woocommerce/releases)에서 최신 **플러그인 zip 에셋** (GitHub 소스 코드 zip 또는 tarball 아님)을 다운로드한 뒤 `플러그인 > 새로 추가 > 플러그인 업로드` 를 통해 업로드합니다.

  </StepCard>
  <StepCard number="2" title="SumUp 설정 구성">

1. `WP Admin > WooCommerce > 설정 > 결제` 로 이동합니다.
2. 결제 방법 목록에서 **SumUp Terminal**을 찾습니다.
3. **SumUp Terminal**을 클릭하여 설정에 액세스합니다.
4. **SumUp API 키**를 입력합니다(SumUp 판매자 대시보드에서 확인 가능).
5. 설정을 저장합니다.

:::note
WooCommerce 설정에서 SumUp Terminal 게이트웨이를 활성화할 필요는 없습니다. 이후 단계에서 POS 전용으로 활성화됩니다.
:::

  </StepCard>
  <StepCard number="3" title="SumUp Terminal 페어링">

1. 같은 설정 페이지에서 **리더 페어링** 섹션을 찾습니다
2. SumUp 기기에서 페어링 화면으로 이동하여 페어링 코드를 표시합니다
3. SumUp 기기에 표시된 페어링 코드를 입력합니다
4. **"리더 페어링"**을 클릭하여 연결을 설정합니다
5. 리더가 성공적으로 페어링되었다는 확인 메시지가 표시될 때까지 기다립니다

:::warning 중요
결제를 처리하려면 먼저 리더가 성공적으로 페어링되어야 합니다. 계속하기 전에 페어링 과정이 완료되었는지 확인하세요.
:::

  </StepCard>
  <StepCard number="4" title="WCPOS에서 활성화" isLast>

1. `WP Admin > POS > 설정 > 결제` 로 이동합니다
2. 목록에서 **SumUp Terminal** 게이트웨이를 찾습니다
3. POS에서 사용할 게이트웨이를 활성화합니다
4. 설정을 저장합니다

  </StepCard>
</Steps>

## 사용법 {#usage}

### 결제 처리 {#processing-payments}

1. **항목 추가**: POS의 장바구니에 상품을 추가합니다
2. **게이트웨이 선택**: 결제 방법으로 "SumUp Terminal"을 선택합니다
3. **결제 시작**: SumUp 기기에서 새 결제를 시작하려면 클릭합니다
4. **고객 결제**: 고객이 SumUp Terminal에서 결제를 완료합니다
5. **자동 완료**: 결제가 감지되면 주문이 자동으로 완료됩니다

### 결제 제어 {#payment-controls}

SumUp Terminal 게이트웨이를 사용할 때 다음 옵션을 사용할 수 있습니다:

- **새 결제 시작**: 연결된 터미널에서 결제 요청을 시작합니다
- **결제 취소**: 현재 처리 중인 결제를 취소합니다
- **결제 상태**: 결제 프로세스의 현재 상태를 모니터링합니다

### 주문 관리 {#order-management}

- **자동 업데이트**: 결제가 성공하면 주문이 자동으로 완료됨으로 표시됩니다
- **결제 추적**: 모든 결제 세부 정보가 주문 메모에 기록됩니다
- **영수증 생성**: 결제가 성공하면 표준 POS 영수증이 생성됩니다

## 하드웨어 호환성 {#hardware-compatibility}

:::important 연결 요구 사항
이 구현은 SumUp의 JavaScript SDK를 사용하므로 웹 애플리케이션에서 작동하지만 **인터넷에 연결된 단말기만** 필요합니다. Bluetooth 단말기는 현재 지원되지 않습니다.
:::

### 지원되는 단말기(인터넷 연결) {#supported-terminals-internet-connected}

- **SumUp Solo** ✅ - WiFi/Ethernet 연결 단말기

### 지원되지 않는 터미널(Bluetooth) {#unsupported-terminals-bluetooth}

- **SumUp Air** ❌ - Bluetooth 전용
- **SumUp 3G** ❌ - Bluetooth 전용

:::note 향후 지원
Bluetooth 터미널 지원은 향후 iOS 및 Android 앱 릴리스에 포함될 예정입니다. 지원이 제공되면 Air 및 3G 모델을 포함한 모든 SumUp 인증 터미널을 지원할 수 있습니다.
:::

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="리더가 페어링되지 않음">

  - 지원되는 인터넷 연결 터미널(Solo만 해당)을 사용하고 있는지 확인하세요
  - SumUp Solo 기기가 WiFi에 연결되어 있고 온라인 상태인지 확인하세요
  - 기기가 페어링 모드이고 페어링 코드가 올바르게 입력되었는지 확인하세요
  - SumUp 계정에서 API 액세스가 활성화되어 있는지 확인하세요
  - 페어링 중 안정적인 인터넷 연결을 유지하세요

  </AccordionItem>

  <AccordionItem question="결제가 처리되지 않음">

  - 리더가 올바르게 페어링되었는지 확인하세요
  - SumUp 계정이 활성 상태이며 정상적으로 유지되고 있는지 확인하세요
  - 기기의 배터리와 연결 상태가 충분한지 확인하세요
  - API 키가 정확하고 활성 상태인지 확인하세요

  </AccordionItem>

  <AccordionItem question="주문이 완료되지 않음">

  - 인터넷 연결 안정성을 확인하세요
  - SumUp 계정의 웹훅 설정을 확인하세요
  - SumUp Terminal에서 결제가 성공했는지 확인하세요
  - API 통신 문제는 WordPress 오류 로그에서 확인하세요

  </AccordionItem>
</Accordion>

### API 속도 제한 {#api-rate-limits}

이 게이트웨이는 API 호출을 줄이고 속도 제한을 피하도록 최적화되어 있습니다.
- 결제 상태를 효율적으로 확인합니다
- 불필요한 API 요청을 최소화합니다
- 자동 재시도 로직이 일시적인 실패를 처리합니다

### 도움 받기 {#getting-help}

기술 지원:
- 문제를 보고하려면 [GitHub 저장소](https://github.com/wcpos/sumup-terminal-for-woocommerce)를 방문하세요
- API 관련 질문은 [SumUp 개발자 문서](https://developer.sumup.com/)를 확인하세요
- 계정 및 하드웨어 문제는 SumUp 지원팀에 문의하세요
