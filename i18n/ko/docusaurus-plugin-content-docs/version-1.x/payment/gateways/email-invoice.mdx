# 이메일 청구서 게이트웨이 {#email-invoice-gateway}

이메일 청구서 게이트웨이를 사용하면 WCPOS에서 고객에게 직접 이메일로 결제 청구서를 보낼 수 있습니다. 고객이 나중에 결제하기를 원하거나 웹 매장의 결제 방법을 사용해 온라인으로 결제를 완료해야 하는 경우에 적합합니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="이메일 인보이스 게이트웨이 설치">

`WP Admin > POS > 설정 > 확장` 에서 설치하거나, [GitHub 릴리스 페이지](https://github.com/wcpos/email-invoice-gateway/releases)에서 최신 **플러그인 zip 에셋** (GitHub 소스 코드 zip 또는 tarball 아님)을 다운로드한 뒤 `플러그인 > 새로 추가 > 플러그인 업로드` 를 통해 업로드합니다.

  </StepCard>
  <StepCard number="2" title="WCPOS에서 활성화" isLast>

1. `WP Admin > POS > 설정 > 결제` 로 이동합니다
2. 목록에서 **WCPOS 인보이스 결제 게이트웨이**를 찾습니다
3. POS에서 사용할 게이트웨이를 활성화합니다
4. 설정을 저장합니다

:::note
이 게이트웨이는 추가 API 키나 외부 서비스 구성이 필요하지 않습니다. 기존 WordPress 이메일 시스템과 WooCommerce 결제 방법을 사용합니다.
:::

  </StepCard>
</Steps>

## 사용 방법 {#usage}

### 인보이스 보내기 {#sending-invoices}

1. **항목 추가**: POS의 장바구니에 상품을 추가합니다
2. **게이트웨이 선택**: 결제 방법으로 "이메일 인보이스"를 선택합니다
3. **이메일 주소**: 
   - 기존 고객의 경우: 이메일이 고객 데이터에서 자동으로 입력됩니다
   - 비회원 주문의 경우: 고객의 이메일 주소를 수동으로 입력합니다
4. **선택 사항**: 고객의 청구 정보를 업데이트하려면 "청구 주소에 저장"을 선택합니다
5. **인보이스 보내기**: 절차를 완료하여 인보이스 이메일을 보냅니다

### 고객 경험 {#customer-experience}

인보이스를 보내면 고객은 다음 내용이 포함된 이메일을 받습니다.

- **주문 세부 정보**: 항목, 수량, 가격의 전체 내역
- **주문 합계**: 세금과 수수료를 포함한 최종 결제 금액
- **결제 링크**: 온라인에서 결제를 완료하는 직접 링크
- **주문 정보**: 주문 번호와 참조 세부 정보

### 결제 완료 {#payment-completion}

고객은 두 가지 방법으로 결제를 완료할 수 있습니다.

1. **온라인 결제**: 이메일의 "이 주문 결제하기" 링크를 클릭합니다
2. **매장 재방문**: 다른 POS 결제 방법으로 결제를 완료하기 위해 매장으로 돌아옵니다

"이 주문 결제하기" 링크는 고객을 웹사이트의 전용 결제 페이지로 이동시켜, 활성화된 웹 결제 게이트웨이(Stripe, PayPal 등)를 사용할 수 있게 합니다.

## 이메일 관리 {#email-management}

### 기존 고객 {#existing-customers}

- 이메일 주소는 고객 프로필에서 자동으로 가져옵니다
- 고객 청구 정보는 유지됩니다
- 주문 내역은 고객 계정에 보관됩니다

### 비회원 고객 {#guest-customers}

- 일회성 고객의 이메일 주소를 수동으로 입력합니다
- 향후 참고할 수 있도록 이메일을 청구 주소에 저장하는 옵션
- 비회원 주문은 WooCommerce에서 올바르게 추적됩니다

### 이메일 템플릿 {#email-templates}

이 게이트웨이는 WooCommerce의 표준 이메일 템플릿을 사용합니다:

- **사용자 지정 가능**: WooCommerce 이메일 설정에서 이메일 표시 방식을 수정합니다
- **브랜드 적용**: 매장 로고와 브랜딩을 포함합니다
- **전문적**: 깔끔하고 전문적인 송장 형식
- **모바일 친화적**: 모든 기기에 맞는 반응형 디자인

## 사용 사례 {#use-cases}

### 적합한 경우 {#perfect-for}

- **B2B 판매**: 회계 시스템을 통해 결제를 처리해야 하는 비즈니스 고객
- **고액 주문**: 승인 또는 처리 시간이 필요한 고가 거래
- **원격 고객**: 매장을 떠난 후 결제를 완료해야 하는 고객
- **거래 계정 고객**: 기존 결제 조건이 설정된 단골 고객
- **견적에서 주문으로 전환**: 견적서 또는 예상 견적을 결제 가능한 인보이스로 전환

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="이메일이 전송되지 않음">

  - WordPress 이메일 구성을 확인하세요
  - 사용자 지정 이메일 서비스를 사용하는 경우 SMTP 설정을 확인하세요
  - WordPress 이메일 테스트 플러그인으로 테스트하세요
  - 테스트 이메일이 스팸 폴더에 있는지 확인하세요

  </AccordionItem>

  <AccordionItem question="결제 링크가 작동하지 않음">

  - WooCommerce가 올바르게 구성되어 있는지 확인하세요
  - 웹 결제에 하나 이상의 결제 게이트웨이가 활성화되어 있는지 확인하세요
  - SSL 인증서가 제대로 설치되어 있는지 확인합니다
  - 주문 상태에서 결제가 허용되는지 확인합니다

  </AccordionItem>

  <AccordionItem question="고객이 결제를 완료할 수 없음">

  - 웹 결제 게이트웨이가 활성화되고 구성되어 있는지 확인합니다
  - 주문이 만료되었거나 취소되지 않았는지 확인합니다
  - 고객의 결제 수단 한도가 충분한지 확인합니다
  - 결제 과정을 직접 테스트합니다

  </AccordionItem>
</Accordion>

### 이메일 전송 {#email-delivery}

안정적인 이메일 전송을 위해 다음을 고려하세요:

- **SMTP 서비스**: SendGrid, Mailgun 또는 Amazon SES와 같은 서비스를 사용합니다
- **이메일 플러그인**: 더 나은 전송을 위해 WordPress SMTP 플러그인을 설치합니다
- **도메인 인증**: SPF, DKIM, DMARC 레코드를 설정합니다
- **모니터링**: 이메일 전송 및 열람률 추적
