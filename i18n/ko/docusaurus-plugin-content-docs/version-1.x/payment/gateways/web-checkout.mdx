# 웹 결제 게이트웨이 {#web-checkout-gateway}

웹 결제 게이트웨이를 사용하면 고객이 웹 스토어의 결제 시스템으로 POS 거래를 완료할 수 있습니다. 이 게이트웨이는 매장 내 POS와 온라인 결제 방법을 자연스럽게 연결하며, 고객이 자신의 기기로 결제하기를 선호하거나 웹 전용 결제 방법을 사용해야 하는 경우에 적합합니다.

## 설치 {#installation}

<Steps>
  <StepCard number="1" title="다운로드 및 설치">

1. [웹 결제 게이트웨이 릴리스 페이지](https://github.com/wcpos/web-checkout-gateway/releases)를 방문합니다
2. 최신 **woocommerce-pos-web-checkout-gateway.zip** 파일을 다운로드합니다
3. WordPress 관리자에서 다음 경로로 이동합니다: `플러그인 > 새로 추가 > 플러그인 업로드`
4. zip 파일을 업로드하고 플러그인을 활성화합니다

  </StepCard>
  <StepCard number="2" title="WCPOS에서 활성화" isLast>

1. 다음 경로로 이동합니다: `WP Admin > POS > 설정 > 결제`
2. 목록에서 **WCPOS 웹 체크아웃 게이트웨이**를 찾습니다
3. POS에서 사용할 게이트웨이를 활성화합니다
4. 설정을 저장합니다

:::note
이 게이트웨이는 기존 WooCommerce 결제 방법을 활용하므로, 웹 결제 게이트웨이(Stripe, PayPal 등)가 하나 이상 올바르게 구성되어 있는지 확인하세요.
:::

  </StepCard>
</Steps>

## 사용법 {#usage}

### 웹 결제 시작 {#initiating-web-checkout}

1. **항목 추가**: POS의 장바구니에 상품 추가
2. **게이트웨이 선택**: 결제 방법으로 "웹 결제" 선택
3. **링크 생성**: 시스템이 주문에 대한 고유한 결제 링크 생성
4. **고객 접속**: 고객이 링크를 클릭하여 웹 결제에 접속
5. **온라인 결제**: 고객이 웹 스토어 결제 방법을 사용하여 결제 완료

### 결제 절차 {#payment-process}

웹 결제 게이트웨이는 다음 워크플로를 따릅니다:

1. **주문 생성**: POS에서 WooCommerce에 대기 중인 주문을 생성합니다
2. **결제 링크**: 시스템에서 안전하고 고유한 결제 링크를 생성합니다
3. **고객 리디렉션**: 고객이 웹 결제 페이지로 이동합니다
4. **결제 선택**: 고객이 사용 가능한 웹 결제 방법 중에서 선택합니다
5. **결제 처리**: 표준 WooCommerce 결제 프로세스에서 결제를 처리합니다
6. **주문 완료**: 결제가 성공하면 주문 상태가 자동으로 업데이트됩니다

### POS 워크플로 {#pos-workflow}

웹 결제를 시작한 후:

1. **상태 모니터링**: 결제 상태를 확인할 수 있도록 POS 주문 화면을 열어 둡니다
2. **고객 결제**: 고객이 자신의 기기 또는 매장의 태블릿에서 결제를 완료합니다
3. **자동 업데이트**: 결제가 수신되면 주문 상태가 실시간으로 업데이트됩니다
4. **결제 처리**: 영수증으로 계속 진행하려면 POS에서 "결제 처리" 버튼을 클릭합니다
5. **영수증 생성**: 평소처럼 영수증을 생성하고 인쇄합니다

## 사용 사례 {#use-cases}

### 적합한 경우 {#perfect-for}

- **고객 선호**: 본인의 결제 앱이나 카드를 사용하려는 고객
- **복합 결제**: POS에서 사용할 수 없는 결제 수단이 필요한 거래
- **분할 결제**: 여러 결제 수단을 사용하려는 고객
- **로열티 프로그램**: 웹 기반 로열티 포인트 사용 기능 이용
- **기프트 카드**: 디지털 기프트 카드나 매장 크레딧을 보유한 고객
- **해외 카드**: 웹 게이트웨이를 통해 더 잘 처리되는 결제 수단

## 기술 세부 정보 {#technical-details}

### 주문 관리 {#order-management}

- **대기 중인 주문**: 주문은 "결제 대기 중" 상태로 생성됩니다
- **상태 업데이트**: 결제가 수신되면 상태가 자동으로 변경됩니다
- **주문 메모**: 결제 세부 정보가 주문 메모에 기록됩니다
- **재고**: 결제 과정 중 재고가 예약됩니다

### 보안 {#security}

- **고유 링크**: 각 결제 세션에는 고유한 시간 제한 URL이 있습니다
- **SSL 필수**: 모든 결제 처리는 보안 HTTPS 연결을 통해 이루어집니다
- **PCI 준수**: 기존 PCI 준수 웹 결제 게이트웨이를 활용합니다
- **세션 관리**: 안전한 세션 처리로 무단 액세스를 방지합니다

### 결제 방법 {#payment-methods}

이 게이트웨이를 사용하면 구성된 모든 WooCommerce 결제 수단에 접근할 수 있습니다:

- **신용/직불 카드**: Stripe, Square, Authorize.net 등
- **디지털 지갑**: PayPal, Apple Pay, Google Pay
- **은행 이체**: 직접 은행 결제 수단
- **지금 구매 후 나중에 결제**: Klarna, Afterpay, Sezzle
- **암호화폐**: Bitcoin 및 기타 암호화폐 결제 게이트웨이
- **지역별 결제 수단**: 지역별로 제공되는 결제 수단

## 문제 해결 {#troubleshooting}

### 일반적인 문제 {#common-issues}

<Accordion>
  <AccordionItem question="결제 링크가 작동하지 않음">

  - SSL 인증서가 올바르게 설치되었는지 확인합니다
  - WooCommerce 고유주소가 올바르게 구성되어 있는지 확인합니다
  - 웹 결제 게이트웨이가 활성화되어 있고 구성되어 있는지 확인합니다
  - 다른 브라우저 또는 기기로 테스트합니다

  </AccordionItem>

  <AccordionItem question="결제가 처리되지 않음">

  - 웹 결제 게이트웨이가 올바르게 구성되어 있는지 확인하세요
  - 결제 게이트웨이 로그에서 오류 메시지를 확인하세요
  - 고객의 결제 수단이 지원되는지 확인하세요
  - 웹 결제 프로세스를 별도로 테스트하세요

  </AccordionItem>

  <AccordionItem question="POS에서 주문이 업데이트되지 않음">

  - 인터넷 연결 안정성을 확인하세요
  - WordPress cron 작업이 올바르게 실행되고 있는지 확인하세요
  - POS 주문 화면을 새로고침하세요
  - WooCommerce 관리자에서 주문 상태를 확인하세요

  </AccordionItem>

  <AccordionItem question="고객이 결제에 접근할 수 없음">

  - 결제 링크가 만료되지 않았는지 확인하세요
  - 고객의 기기가 인터넷에 연결되어 있는지 확인하세요
  - 주문이 취소 상태 또는 완료 상태가 아닌지 확인합니다
  - 다른 기기에서 링크를 테스트하세요

  </AccordionItem>
</Accordion>

### 성능 최적화 {#performance-optimisation}

최적의 성능을 위해:

- **캐싱**: 결제 페이지가 제외되도록 캐싱 플러그인을 구성하세요
- **CDN**: CDN 설정이 결제 프로세스를 방해하지 않도록 확인하세요
- **데이터베이스**: 더 빠른 주문 처리를 위해 데이터베이스를 최적화하세요
- **호스팅**: 가동 시간이 우수한 안정적인 호스팅을 사용하세요

### 도움말 받기 {#getting-help}
