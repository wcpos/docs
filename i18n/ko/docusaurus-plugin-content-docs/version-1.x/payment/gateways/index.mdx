# 결제 게이트웨이 {#payment-gateways}

WCPOS는 POS 시스템의 기능을 확장하는 여러 맞춤 결제 게이트웨이를 지원합니다. 이러한 게이트웨이는 POS 환경에 맞게 설계되었으며, 다양한 결제 수단 및 서비스와 원활하게 통합됩니다.

## 사용 가능한 맞춤 게이트웨이 {#available-custom-gateways}

### 하드웨어 터미널 게이트웨이 {#hardware-terminal-gateways}

- **[Stripe Terminal](./stripe-terminal)** - Stripe Terminal 하드웨어 리더기를 사용하여 결제를 처리합니다
- **[SumUp Terminal](./sumup-terminal)** - SumUp 카드 리더기를 통해 결제를 처리합니다
- **[Square Terminal](./square-terminal)** - Square Terminal 기기에서 대면 결제를 수금합니다
- **[PayPal Reader (Zettle)](./paypal-reader)** - PayPal Reader (Zettle) 터미널에서 대면 카드 결제를 처리합니다

### 모바일 결제 게이트웨이 {#mobile-payment-gateways}

- **[Vipps MobilePay](./vipps-mobilepay)** - QR 코드 또는 푸시 알림을 통한 모바일 결제 수단

### 디지털 결제 게이트웨이 {#digital-payment-gateways}

- **[이메일 청구서](./email-invoice)** - 이메일을 통해 고객에게 결제 청구서 발송
- **[웹 결제](./web-checkout)** - 온라인 결제 완료를 위해 고객을 웹으로 리디렉션

## 나만의 게이트웨이 만들기 {#creating-your-own-gateway}

특정 요구 사항에 맞는 맞춤 결제 게이트웨이가 필요한 경우, **[게이트웨이 템플릿](/reference/gateway-template)**을 사용하여 자체 WCPOS 결제 게이트웨이를 만들 수 있습니다.

## 설치 개요 {#installation-overview}

모든 맞춤 게이트웨이는 유사한 설치 과정을 따릅니다:

1. 해당 GitHub 저장소에서 최신 릴리스를 **다운로드**합니다
2. `WP Admin > 플러그인 > 새로 추가 > 플러그인 업로드` 를 통해 플러그인을 **설치**합니다
3. `WP Admin > WooCommerce > 설정 > 결제` 에서 게이트웨이 설정을 **구성**합니다
4. `WP Admin > POS > 설정 > 결제` 에서 게이트웨이를 **활성화**합니다
