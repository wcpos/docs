:::info 프로 기능
쿠폰 화면과 계산대에서 쿠폰을 적용하는 기능을 사용하려면 [WCPOS Pro](/getting-started/pro-license)가 필요합니다. 무료 사용자는 쿠폰 화면의 흐릿한 미리보기만 볼 수 있으며, 쿠폰 세부 정보를 확인하거나 장바구니에서 코드를 적용할 수 없습니다.
:::

쿠폰 화면에서는 POS 내에서 WooCommerce 쿠폰을 탐색하고 조회할 수 있으며, 장바구니의 **쿠폰 추가** 버튼을 통해 현재 주문에 쿠폰을 적용할 수 있습니다. 쿠폰 자체는 여전히 WooCommerce에서 생성 및 관리되며, POS에서 동기화하여 계산대에서 즉각적인 피드백을 위해 로컬로 유효성을 검사합니다.

## 인터페이스 개요 {#interface-overview}

### 헤더 작업 {#header-actions}

화면 상단:

- **검색 바** — 코드로 쿠폰 검색
- **표시 설정** (<Icon name="sliders" />) — 표시할 열 구성

### 쿠폰 테이블 {#coupons-table}

메인 영역에 동기화된 쿠폰이 다음 정보와 함께 표시됩니다:

- **코드** — 고객이 입력하는 쿠폰 코드
- **설명** — 내부 설명 (설정 시 영수증 라벨로도 사용됨)
- **할인 유형** — 비율, 고정 장바구니 또는 고정 상품
- **금액** — 할인 값
- **사용 횟수 / 제한** — 사용 횟수 및 전체 사용 제한
- **만료일** — 만료 날짜, 없으면 빈칸
- **작업** — 점 세 개 메뉴

### 푸터 {#footer}

- 동기화 버튼이 있는 쿠폰 수 (<Icon name="arrow-rotate-right" />). 지우기 및 새로고침 옵션을 사용하려면 **길게 누르세요**.

## 쿠폰 유형 {#coupon-types}

WCPOS는 세 가지 표준 WooCommerce 쿠폰 유형을 지원합니다:

| 유형 | 설명 | 예시 |
|---|---|---|
| **비율 할인** | 장바구니 소계를 비율로 할인합니다 | 주문 금액의 `10%` 할인 |
| **고정 장바구니 할인** | 장바구니 합계를 고정 금액으로 할인합니다 | 주문 금액에서 `$5` 할인 |
| **고정 상품 할인** | 일치하는 상품 항목을 단위당 고정 금액으로 할인합니다 | 티셔츠당 `$2` 할인 |

세 가지 유형 모두 동일한 유효성 검사 규칙(만료일, 사용 횟수 제한, 상품/카테고리 제한 등)을 따릅니다.

## 쿠폰 생성 {#creating-coupons}

쿠폰은 POS가 아닌 WooCommerce에서 생성합니다. **WP Admin → 마케팅 → 쿠폰 → 쿠폰 추가**로 이동하세요:

1. **쿠폰 코드** — 계산대에서 캐셔가 입력하는 코드입니다. 대소문자를 구분하지 않습니다.
2. **설명** — 내부 메모입니다. WCPOS는 이 값이 설정되어 있으면 영수증에 할인 라벨로 표시하므로, 내부 코드 대신 "관리자 할인 10%"와 같이 고객 친화적인 문구를 사용하는 것이 좋습니다.
3. **할인 유형** — 비율, 고정 장바구니, 또는 고정 상품 할인입니다.
4. **쿠폰 금액** — 할인 값입니다.
5. **만료일** — 선택 사항입니다. 이 날짜가 지나면 쿠폰이 거부됩니다.

### 사용 제한 {#usage-restriction}

- **최소/최대 지출 금액** — 소계가 지정된 기준 이상/이하인 경우에만 쿠폰이 적용됩니다.
- **개별 사용 전용** — 이 쿠폰이 다른 쿠폰과 함께 사용되는 것을 방지합니다.
- **할인 상품 제외** — 이미 할인 중인 상품(및 캐셔가 계산대에서 가격을 낮춘 항목 — [POS 가격 재정의](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons) 참조)을 건너뜁니다.
- **상품/상품 제외** — 쿠폰을 특정 상품으로 제한하거나 특정 상품을 제외합니다.
- **상품 카테고리/카테고리 제외** — 카테고리별로 제한합니다. POS에서 생성된 기타/맞춤 상품도 POS를 통해 지정된 카테고리가 적용됩니다.
- **허용 이메일** — 쿠폰을 특정 고객 이메일 주소로 제한합니다(`*` 와일드카드 지원).

### 사용 제한 {#usage-limits}

- **쿠폰당 사용 제한** — 모든 고객을 합산하여 이 코드를 사용할 수 있는 총 횟수입니다.
- **사용자당 사용 제한** — 한 고객이 사용할 수 있는 횟수입니다.
- **X개 항목으로 사용 제한** — 고정 상품 쿠폰의 경우, 1회 사용 시 할인이 적용되는 대상 항목 수를 제한합니다.

### POS 사용 팁 {#tips-for-pos-use}

- **"매니저 10%"** 쿠폰(10% 할인, 최소 구매 금액 없음, 만료일 없음, 1회 사용 제한 해제)을 생성하여 매니저에게 코드를 제공하면, 직원이 계산대에서 임의 조정 시 적용할 수 있으며 보고서에 추적 가능한 할인 내역이 남습니다.
- 단골 고객 보상을 위한 **"적립 $5"** 쿠폰을 생성합니다.
- 일회성 프로모션의 경우, 나중에 실수로 재사용되지 않도록 만료일을 짧게 설정합니다.
- **설명**에 영수증에 인쇄할 문구를 입력하면 할인에 브랜드를 가장 빠르게 적용할 수 있습니다.

## 계산대에서 쿠폰 적용하기 {#applying-a-coupon-at-the-register}

장바구니 패널에서:

1. **쿠폰 추가**를 클릭합니다.
2. 쿠폰 코드를 입력합니다
3. **Enter** 키를 누릅니다

POS는 동기화된 데이터를 기반으로 쿠폰을 즉시 검증하며, 서버와의 왕복 통신 없이 장바구니에 할인을 적용합니다. 이후 장바구니 내용을 변경하면 할인이 자동으로 재계산됩니다.

적용된 쿠폰을 제거하려면 장바구니에서 쿠폰 옆의 × 버튼을 클릭합니다.

{/* Screenshot: cart-add-coupon.png */}

## 검증 방식 {#how-validation-works}

코드를 입력하면 POS는 WooCommerce가 서버에서 확인하는 것과 동일한 규칙을 모두 검사합니다:

- 쿠폰이 존재하며 만료되지 않았는지 확인
- 사용 횟수 제한 초과 여부 확인(전체 및 사용자별)
- 최소/최대 지출 금액 충족 여부 확인
- "개별 사용" 쿠폰이 이미 적용된 쿠폰과 충돌하지 않는지 확인
- 상품/카테고리 제한이 장바구니 내용과 일치하는지 확인
- 이메일 제한은 선택한 고객과 일치해야 합니다(게스트 주문의 경우 건너뜀).
- **할인 품목 제외**가 활성화된 경우 할인 품목은 제외됩니다.

검증에 실패하면 캐셔에게 구체적인 오류 메시지가 표시됩니다(예: "이 쿠폰은 만료되었습니다" 또는 "최소 지출 금액에 도달하지 못했습니다"). 주문이 제출될 때 서버에서도 동일한 검증이 다시 실행됩니다. 로컬 검증은 신뢰가 아닌 속도를 위한 것입니다.

## 동기화 동작 {#sync-behaviour}

쿠폰은 표준 WCPOS 복제 파이프라인을 통해 WooCommerce에서 기기로 동기화됩니다:

- WP Admin에서 새로 생성한 쿠폰은 다음 동기화 시 POS에 표시됩니다.
- 기존 쿠폰의 변경 사항(사용 횟수, 만료일 변경 등)은 자동으로 동기화됩니다.
- 쿠폰은 IndexedDB에 로컬로 저장되므로 오프라인에서도 사용할 수 있습니다.

WP Admin에서 쿠폰을 방금 생성했는데 아직 표시되지 않는 경우 동기화가 아직 실행되지 않았을 수 있습니다. 쿠폰 화면 하단에서 동기화 아이콘(<Icon name="arrow-rotate-right" />)을 탭하여 새로고침하세요. 전체를 새로 가져오려면 길게 눌러 **지우고 새로고침**을 실행하세요.

## 연결 상태 {#connectivity}

- **쿠폰 적용**은 동기화된 데이터를 기반으로 클라이언트 측에서 검증하므로 오프라인에서도 작동합니다.
- **결제 완료** 시에는 여전히 서버 연결이 필요합니다 — 서버가 재검증을 수행하며 최종 주문 총액의 권위 있는 출처입니다.
- **쿠폰 생성 또는 편집**은 WP Admin에서 이루어지며 WordPress 사이트에 대한 연결이 필요합니다.

## 쿠폰과 POS 할인의 상호 작용 {#how-coupons-interact-with-pos-discounts}

계산원이 계산대에서 항목 가격을 낮춘 후 쿠폰을 적용하면, 쿠폰은 원래 가격이 아닌 **낮아진 가격**을 기준으로 계산됩니다. POS에서 가격이 낮아진 항목은 "세일 중"으로 처리되며, **세일 상품 제외**가 활성화된 쿠폰은 해당 항목을 건너뜁니다. 자세한 내용은 [할인](/pos/cart/discounts#how-pos-price-changes-interact-with-coupons)을 참조하세요.
