WCPOS(Point of Sale) 시스템의 주요 특징 중 하나는 데이터 관리 방식입니다. 대량의 상품 데이터를 효율적으로 처리할 수 있는 방법을 제공합니다. 이 가이드에서는 WCPOS에서 상품이 다운로드되고 동기화되는 과정을 설명합니다.

## 상품의 로컬 저장 {#local-storage-of-products}

WCPOS는 WooCommerce 스토어에서 상품을 가져와 기기에 로컬로 저장합니다. 이 방식을 통해 한 번 다운로드된 상품은 반복적으로 가져올 필요가 없습니다. POS를 계속 사용하면 WooCommerce 스토어의 모든 상품이 점진적으로 다운로드되어 로컬 상품 데이터베이스가 구축됩니다.

## 상품 일괄 다운로드 {#batch-downloading-of-products}

대부분의 스토어에서 한 번에 약 50개 이상의 상품을 다운로드하려고 하면 서버 부하가 높아지거나 응답 시간이 느려지거나 서버가 다운될 수 있습니다. 이를 방지하기 위해 WCPOS는 보통 한 번에 약 50개씩 소규모 배치로 상품을 다운로드하도록 설계되어 있습니다. 이 과정은 전체 재고가 로컬에 저장될 때까지 계속되며, 서버에 과도한 부담을 주지 않으면서 대량의 데이터를 관리할 수 있습니다.

## 상품 검색 사용 {#using-the-product-search}

상품 일괄 다운로드를 시작하는 효과적인 방법은 상품 검색 기능을 사용하는 것입니다. 상품 검색을 수행할 때마다 POS 시스템이 다음 배치의 상품을 가져와 저장합니다. 이 방법을 통해 새로운 상품을 점진적으로 다운로드할 수 있을 뿐만 아니라, 더 많은 상품이 로컬에 저장됨에 따라 검색 기능의 속도와 효율성도 향상됩니다.

## 결론 {#conclusion}

모든 상품이 POS 시스템에 한꺼번에 표시되지 않는 것은 정상적인 동작입니다. 시간이 지나면서 상품 검색 및 기타 POS 작업을 통해 모든 상품이 다운로드됩니다. WCPOS 작동 방식에 대한 자세한 내용은 [아키텍처](/reference/architecture)를 참조하십시오.

WCPOS는 서버 부하와 운영 효율성을 최적화하도록 설계되었습니다. 점진적 상품 다운로드 기능은 원활하고 효율적이며 사용자 친화적인 매장 관리 경험을 보장합니다.

:::tip
일반 동기화로 해결되지 않는 지속적인 동기화 문제가 발생하는 경우, [로컬 데이터 모두 삭제](/support/troubleshooting/clear-local-data)를 통해 전체 매장 데이터를 새로 다운로드할 수 있습니다.
:::

## 자주 묻는 질문 {#faq}

<Accordion>
  <AccordionItem question="더 많은 상품을 불러오려면 어떻게 하나요?">

  POS에서 상품 동기화는 여러 가지 방법으로 실행할 수 있습니다:

  - POS를 처음 로드할 때.
  - 상품 검색을 실행할 때마다.
  - 카테고리나 태그 등의 필터를 적용할 때.
  - 상품 목록의 하단까지 스크롤할 때.
  - 동기화 버튼을 수동으로 누를 때(짧게 누르면 동기화, **길게 누르면** 초기화 및 새로고침 옵션).
  - 또한 POS는 5분마다 자동으로 동기화됩니다.

  모든 상품이 다운로드되면 이후 동기화에서는 마지막 동기화 이후 업데이트된 상품만 가져옵니다.

  </AccordionItem>

  <AccordionItem question="맨 아래까지 스크롤했는데 여전히 모든 상품이 표시되지 않습니다">

  모든 상품이 표시되지 않는 데에는 몇 가지 이유가 있을 수 있습니다:

  - POS는 기본적으로 재고 소진 항목을 숨깁니다. 재고 소진 항목을 표시하려면 상품 표시 설정에서 이 설정을 변경할 수 있습니다.
  - WooCommerce REST API는 단순, 옵션, 묶음, 외부 상품 등 표준 WooCommerce 상품 유형만 지원합니다. 커스텀 상품 유형을 사용하는 경우 POS에 표시되지 않을 수 있습니다.

  </AccordionItem>
</Accordion>
