## 제품 검색

### 통합 검색 필드

WCPOS는 여러 제품 속성을 동시에 검색하는 단일 검색 필드를 제공합니다:

- **제품 이름** - 제품 제목 및 설명을 검색합니다.
- **SKU** - 제품 postmeta `_sku` 필드와 일치합니다.
- **바코드** - 구성된 바코드 필드를 검색합니다. 이는 어떤 postmeta 필드일 수 있습니다. 구형 설정은 `_sku`이지만, 이는 나중에 `_global_unique_id`로 변경될 것입니다.

"제품 검색" 필드에 검색어를 입력하기만 하면 POS가 이러한 모든 필드에서 결과를 즉시 필터링합니다.

### 토큰화 검색 기술

검색 기능은 고급 토큰화 기능을 가진 [FlexSearch 라이브러리](https://github.com/nextapps-de/flexsearch)를 사용합니다:

- **전방 토큰화** - 시작 부분에서 부분 단어를 일치시킵니다 (예: "blue"를 검색하면 "blueberry"를 찾습니다).
- **성능 최적화** - 빠른 검색 결과를 위해 성능 사전 설정을 사용합니다.
- **언어 인식** - 매장에 설정된 언어에 맞게 조정됩니다.
- **지연 초기화** - 필요할 때만 검색 인덱스를 로드하여 메모리 사용을 최적화합니다.

### 검색 작동 방식

검색 필드에 입력하면 POS가:

1. **토큰화**하여 검색 가능한 용어로 변환합니다.
2. **로컬**에 저장된 제품 데이터를 먼저 검색하여 즉각적인 결과를 제공합니다.
3. **서버에 질의**하여 로컬 일치가 없으면 새로운 제품을 다운로드하여 향후 검색을 위해 저장합니다.
4. **결과를 업데이트**하며 입력하는 대로 실시간으로 변화합니다.

이 접근 방식은 빠른 검색 성능을 보장하며 점진적으로 완전한 로컬 제품 데이터베이스를 구축합니다.

## 제품 필터링

### 필터 바

검색 필드 아래에는 특정 기준에 따라 제품을 좁힐 수 있는 대화형 필터 토글 및 드롭다운 메뉴가 있습니다.

### 사용 가능한 필터

#### 재고 상태
인벤토리 상태에 따라 제품을 필터링합니다:
- **재고 있음** - 사용 가능한 인벤토리가 있는 제품
- **재고 없음** - 재고가 없는 제품
- **예약 주문** - 예약 주문이 가능한 제품

#### 추천 제품
WooCommerce 매장에서 "추천"으로 표시된 제품만 표시하도록 전환합니다.

#### 세일 제품  
현재 세일 중이거나 할인 중인 제품만 표시하도록 필터링합니다.

#### 카테고리
카테고리 드롭다운을 사용하여 할당된 제품 카테고리에 따라 제품을 필터링합니다. 이를 통해 특정 부서나 제품 라인 내에서 제품을 신속하게 찾을 수 있습니다.

#### 태그
제품 태그로 필터링하여 WooCommerce 매장에서 정의한 특정 속성 또는 특성을 가진 항목을 찾습니다.

### 필터 사용하기

- **필터 전환** - 필터 버튼을 클릭하여 활성화합니다 (활성 필터는 강조 표시됩니다).
- **다중 필터** - 여러 필터를 조합하여 검색을 더욱 좁힐 수 있습니다.
- **필터 지우기** - 활성 필터를 다시 클릭하여 비활성화합니다.
- **검색 + 필터** - 정확한 제품 위치를 위해 검색 필드와 함께 필터를 사용합니다.

## 바코드 구성

### 검색 필드

검색 기능은 구성된 바코드 필드를 자동으로 포함합니다. 검색에 사용되는 바코드 필드는 POS 설정 구성에 따라 다릅니다.

## 자주 묻는 질문

<Accordion>
  <AccordionItem question="_global_unique_id 필드는 바코드 용도로 무엇입니까?">

`_global_unique_id` 필드는 WooCommerce에서 최근에 추가한 새로운 바코드 필드로, 매장 간 더 나은 바코드 표준화를 제공합니다.

**핵심 사항:**

- **현대 표준**: 이 필드는 글로벌 바코드 식별을 위해 특별히 설계되었습니다.
- **POS 구성**: POS 설정에서 `_global_unique_id`를 바코드 필드로 사용할 수 있도록 구성할 수 있습니다.
- **구형 대 신형**: 구형 바코드 설정은 `_sku` 필드를 사용하지만, 이는 향후 버전에서 `_global_unique_id`로 변경될 것입니다.
- **유연성**: 서드파티 바코드 플러그인을 사용하는 경우 바코드 필드로 어떤 제품 메타 필드도 구성할 수 있습니다.
- **미래 기본값**: `_global_unique_id`는 향후 POS 업데이트에서 기본 바코드 필드가 될 것입니다.

POS가 바코드에 사용할 필드를 구성하려면 WordPress 관리 영역에서 POS 설정을 방문하십시오.

  </AccordionItem>

  <AccordionItem question="검색할 때 모든 제품이 표시되지 않는 이유는 무엇입니까?">

WCPOS는 성능 유지를 위해 점진적인 제품 다운로드를 사용합니다. 제품이 보이지 않으면:

1. **검색해 보십시오** - 이렇게 하면 POS가 서버에서 해당 제품을 찾게 됩니다.
2. **다운로드 대기** - POS는 제품 및 기타를 소규모 배치로 다운로드합니다.
3. **다시 검색** - 다운로드가 완료되면 제품이 향후 검색에 나타납니다.

이 프로세스는 귀하의 POS가 수천 개의 제품이 있어도 빠르고 응답성이 유지되도록 보장합니다. 자세한 내용은 우리의 [제품 동기화](/products/sync) 가이드를 참조하십시오.

  </AccordionItem>

  <AccordionItem question="부분적인 제품 이름이나 SKU를 검색할 수 있습니까?">

네! 토큰화된 검색은 전방 일치를 사용하므로:

- "blue"를 검색하면 "blueberry", "blue shirt" 등의 제품을 찾습니다.
- "ABC"를 검색하면 "ABC123", "ABC-XYZ"와 같은 SKU를 찾습니다.
- 전체 단어나 코드를 입력할 필요가 없습니다.

검색은 최소한의 입력으로 신속하게 제품을 찾을 수 있도록 설계되었습니다.

  </AccordionItem>
