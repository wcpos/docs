## 제품 검색

### 통합 검색 필드

WCPOS는 여러 제품 속성을 동시에 검색할 수 있는 단일 검색 필드를 제공합니다:

- **제품명** - 제품 제목과 설명 검색
- **SKU** - 제품 메타 `_sku` 필드와 일치
- **바코드** - 구성된 바코드 필드 검색. 어떤 포스트메타 필드든 사용 가능. 레거시 설정은 `_sku`이며, 향후 `_global_unique_id`로 변경될 예정입니다.

"제품 검색" 필드에 검색어를 입력하면 POS가 즉시 모든 필드에서 결과를 필터링합니다.

### 토큰화 검색 기술

검색 기능은 [FlexSearch 라이브러리](https://github.com/nextapps-de/flexsearch)를 사용하여 고급 토큰화 기능을 제공합니다:

- **선행 토큰화** - 단어 시작 부분의 부분 일치 (예: "blue"로 검색하면 "blueberry" 찾기)
- **성능 최적화** - 빠른 검색 결과를 위한 성능 프리셋 사용
- **언어 인식** - 스토어의 구성된 언어에 맞게 조정
- **지연 초기화** - 필요할 때만 검색 인덱스를 로드하여 메모리 사용 최적화

### 검색 작동 방식

검색 필드에 입력하면 POS는 다음과 같이 작동합니다:

1. **토큰화** - 입력을 검색 가능한 용어로 분리
2. **로컬 검색** - 빠른 결과를 위해 로컬에 저장된 제품 데이터 먼저 검색
3. **서버 쿼리** - 로컬 일치 항목이 없으면 서버에 쿼리, 새 제품 다운로드 및 저장
4. **실시간 결과 업데이트** - 입력하는 동안 결과 즉시 갱신

이 접근 방식은 빠른 검색 성능을 보장하면서 점진적으로 완전한 로컬 제품 데이터베이스를 구축합니다.

## 제품 필터링

### 필터 바

검색 필드 아래에는 특정 기준으로 제품을 좁힐 수 있는 대화형 필터 토글과 드롭다운 메뉴가 있습니다.

### 사용 가능한 필터

#### 재고 상태
재고 상태에 따라 제품 필터링:
- **재고 있음** - 사용 가능한 재고가 있는 제품
- **재고 없음** - 재고가 0인 제품
- **백오더** - 백오더 가능한 제품

#### 추천 제품
WooCommerce 스토어에서 "추천" 제품으로 표시된 제품만 표시하는 토글.

#### 할인 제품
현재 할인 중이거나 활성 할인이 있는 제품만 표시.

#### 카테고리
드롭다운을 사용하여 할당된 제품 카테고리별로 제품 필터링. 특정 부서나 제품군의 제품을 빠르게 찾을 수 있습니다.

#### 태그
WooCommerce 스토어에서 정의한 특정 속성이나 특성을 가진 항목을 태그로 필터링.

### 필터 사용 방법

- **필터 토글** - 필터 버튼을 클릭하여 활성화 (활성 필터는 강조 표시됨)
- **다중 필터** - 검색 결과를 더욱 좁히기 위해 여러 필터 조합 가능
- **필터 해제** - 활성 필터를 다시 클릭하여 비활성화
- **검색 + 필터** - 정확한 제품 위치 지정을 위해 검색 필드와 함께 필터 사용

## 바코드 구성

### 검색 필드

검색 기능은 자동으로 구성된 바코드 필드를 포함합니다. 검색에 사용되는 바코드 필드는 POS 설정 구성에 따라 달라집니다.

## 자주 묻는 질문

<details>

<summary>
  _global_unique_id 필드의 용도는 무엇인가요?
</summary>

`_global_unique_id` 필드는 WooCommerce가 최근에 추가한 바코드 필드로, 스토어 간 바코드 표준화를 개선하기 위해 설계되었습니다.

**주요 포인트:**

- **최신 표준**: 전역 바코드 식별을 위해 특별히 설계된 필드
- **POS 구성**: POS 설정에서 `_global_unique_id`를 바코드 필드로 구성 가능
- **레거시 vs 신규**: 레거시 바코드 설정은 `_sku` 필드를 사용하지만, 향후 버전에서 `_global_unique_id`로 변경될 예정
- **유연성**: 서드파티 바코드 플러그인을 사용하는 경우 어떤 제품 메타 필드든 바코드 필드로 구성 가능
- **향후 기본값**: 향후 POS 업데이트에서 `_global_unique_id`가 기본 바코드 필드가 될 예정

바코드에 사용할 필드를 구성하려면 WordPress 관리자 영역의 POS 설정을 방문하세요.

</details>

<details>

<summary>
  검색 시 모든 제품이 보이지 않는 이유는 무엇인가요?
</summary>

WCPOS는 성능 유지를 위해 점진적 제품 다운로드를 사용합니다. 제품이 보이지 않는 경우:

1. **검색해보기** - POS가 서버에서 해당 제품을 찾도록 트리거
2. **다운로드 대기** - POS가 제품을 소량 배치로 다운로드
3. **다시 검색** - 다운로드 후 제품이 향후 검색에 나타남

이 프로세스는 수천 개의 제품이 있어도 POS가 빠르고 응답성 있게 유지되도록 보장합니다. [제품 동기화](/products/sync) 가이드에서 자세히 알아보세요.

</details>

<details>

<summary>
  제품명이나 SKU의 일부로 검색할 수 있나요?
</summary>

네! 토큰화 검색은 선행 일치를 사용하므로:

- "blue"로 검색하면 "blueberry", "blue 셔츠" 등의 제품을 찾습니다.
- "ABC"로 검색하면 "ABC123", "ABC-XYZ" 등의 SKU를 찾습니다.
- 전체 단어나 코드를 입력할 필요가 없습니다.

검색은 최소한의 입력으로 빠르게 제품을 찾도록 설계되었습니다.

</details>
