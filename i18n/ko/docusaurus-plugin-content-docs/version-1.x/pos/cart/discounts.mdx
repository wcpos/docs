WCPOS는 계산원이 주문에 즉석 할인을 적용할 수 있는 여러 방법을 제공합니다. 빠른 백분율 버튼, 직접 품목 가격 편집, 주문 수준 할인 수수료를 사용할 수 있습니다. 사용 규칙이 있는 사전 구성 프로모션은 [쿠폰](/coupons)(Pro)을 참조하세요.

:::info v1.9.0에서 변경됨
POS 가격 변경이 영수증과 보고서에 표시되는 방식이 v1.9.0에서 변경되었습니다. 영수증의 "할인" 합계가 이제 0으로 표시되는 것을 확인했다면 아래의 [v1.9.0에서 변경된 사항](#what-changed-in-v190)을 참조하세요.
:::

## 빠른 할인 {#quick-discounts}

빠른 할인 버튼을 사용하면 한 번의 탭으로 전체 주문에 백분율 할인을 적용할 수 있습니다.

구성하려면 카트 [표시 설정](/pos/cart#display-settings)을 열고 **빠른 할인** 필드에 쉼표로 구분된 백분율 목록을 입력합니다. 예를 들어, `5,10,15,20` 값은 네 개의 바로 가기 버튼을 만듭니다.

빠른 할인 버튼을 탭하면 카트의 모든 품목에 해당 백분율이 적용됩니다.

## 품목별 할인 {#line-item-discounts}

장바구니에서 개별 품목의 가격을 직접 변경할 수 있습니다:

1. 품목의 **가격** 필드를 클릭합니다
2. 새 가격을 입력합니다
3. **Enter**를 눌러 확인합니다

가격 맞춤, 직원 할인 또는 일회성 조정에 유용합니다. 품목의 합계는 수량 × 새 가격을 기준으로 자동 업데이트됩니다.

장바구니 품목 편집에 대한 자세한 내용은 [품목](/pos/cart/line-items)을 참조하세요.

:::tip 품목 분할
고객이 동일한 상품의 일부에 서로 다른 할인을 적용하려는 경우(예: 3개는 정상가, 2개는 할인), 장바구니 [표시 설정](/pos/cart#display-settings)에서 **분할** 옵션을 활성화하여 하나의 품목을 별도 줄로 나눌 수 있습니다.
:::

## 주문 수준 할인 {#order-level-discounts}

개별 품목이 아닌 전체 주문에 고정 할인을 적용하려면 **마이너스 수수료**를 추가합니다:

1. 장바구니 품목 아래의 **수수료 추가**를 클릭합니다
2. 이름을 입력합니다(예: "직원 할인")
3. 할인 금액을 음수로 입력합니다(예: `-5.00`)

수수료는 별도의 라인 항목으로 표시되며 주문 합계를 줄입니다. 필요한 경우 점 세 개 메뉴를 사용해 수수료의 세금 상태를 편집할 수 있습니다.

## POS 할인과 WooCommerce 쿠폰 비교 {#pos-discounts-vs-woocommerce-coupons}

이 페이지의 할인은 계산원이 계산대에서 적용하는 임시 조정입니다. WooCommerce **쿠폰**은 규칙과 추적 기능이 포함된 사전 구성 프로모션입니다. 한눈에 비교하면 다음과 같습니다.

| | POS 할인 | WooCommerce 쿠폰(Pro) |
|---|---|---|
| **적용 방법** | 빠른 할인, 라인 가격 편집 또는 음수 수수료 | 장바구니에 쿠폰 코드 입력 |
| **구성 위치** | 계산원이 즉석에서 설정 | **WP Admin → 마케팅 → 쿠폰**에서 사전 구성 |
| **추적** | 라인 가격으로 기록됨([v1.9.0 변경 사항](#what-changed-in-v190) 참조) | WooCommerce 보고서에서 쿠폰 할인으로 추적됨 |
| **제한 사항** | 없음 — 계산원이 원하는 가격을 설정 | 사용 한도, 상품/카테고리 제한, 최소 지출 금액, 만료일, 이메일 규칙 |
| **적합한 용도** | 임시 조정, 가격 매칭 | 구조화된 프로모션, 추적 가능한 할인 |

:::tip 어떤 것을 사용해야 하나요?
일회성 가격 조정에는 이 페이지의 할인이 더 간단합니다. WooCommerce 보고서에서 할인 사용량을 추적하거나 사용 한도 같은 규칙을 적용해야 하는 경우 [쿠폰](/coupons)을 사용하세요.
:::

## POS 가격 변경이 쿠폰과 상호 작용하는 방식 {#how-pos-price-changes-interact-with-coupons}

계산원이 라인 항목에 사용자 지정 가격을 설정한 뒤(예: $18을 $16로 낮춤) 쿠폰을 적용하면, 쿠폰은 원래 가격($18)이 아니라 **POS에서 할인된 가격**($16)을 기준으로 계산됩니다. 이렇게 하면 계산원 할인과 쿠폰이 원래 가격에 중복 적용되어 고객에게 과도한 할인이 제공되는 것을 방지할 수 있습니다.

- POS에서 할인된 항목은 WooCommerce에서 "할인 중"으로 처리됩니다. 쿠폰에 **할인 상품 제외**가 활성화되어 있으면 일반 할인 상품을 건너뛰는 것과 같은 방식으로 POS에서 할인된 항목도 건너뜁니다. 개발자는 `woocommerce_pos_item_is_on_sale` 필터로 이를 재정의할 수 있습니다.
- 쿠폰을 제거해도 해당 라인은 POS에서 할인된 가격으로 유지됩니다.

:::note 개발자 참고 자료
POS 가격 재정의가 저장되는 방식과 사용 가능한 필터에 대한 기술 세부 정보는 [POS 할인 참고 자료](/reference/pos-discounts)를 참조하세요.
:::

## v1.9.0에서 변경된 사항 {#what-changed-in-v190}

v1.8에서 업그레이드한 후 영수증과 보고서의 **할인** 합계가 이제 **0**으로 표시되는 것을 확인했다면, 이 섹션에서 그 이유와 선택할 수 있는 옵션을 설명합니다.

### 변경 사항 {#the-change}

v1.9.0 이전에는 계산원이 품목 가격을 낮추면(예: $18 → $16) POS가 주문을 `subtotal = $18`, `total = $16` 값으로 기록했습니다. 그러면 WooCommerce의 계산 결과는 `discount_total = $2` 이었으며, 이 금액이 영수증과 보고서에 할인으로 표시되었습니다.

이로 인해 쿠폰을 추가로 적용할 때마다 총액이 잘못 계산되었습니다. 쿠폰이 원래 금액인 $18을 기준으로 계산되어 할인이 과도하게 적용되고 고객에게 실제보다 적게 청구되었습니다.

v1.9.0부터 WCPOS는 WooCommerce가 할인 가격을 처리하는 방식과 일치합니다. 계산대에서 설정한 가격이 곧 품목 소계입니다. WooCommerce는 쿠폰 코드만 "할인"으로 계산합니다(WC가 세일 상품을 처리하는 방식과 동일하며, 이 경우에도 세일 가격은 별도의 할인 항목 없이 소계에 포함됩니다).

### 이 변경이 의미하는 것 {#what-this-means-for-you}

- **영수증**에는 더 이상 품목 가격 변경에 대한 별도의 "할인" 항목이 표시되지 않습니다. 낮아진 가격이 곧 가격입니다.
- **보고서**에는 POS 품목 가격 변경만 사용한 경우 `discount_total = 0` 값이 표시됩니다. 쿠폰 할인만 계산됩니다.
- **쿠폰**은 이제 POS에서 할인된 품목에 중복 적용해도 올바르게 계산됩니다.
- **할인 데이터는 여전히 저장됩니다**. 모든 주문의 품목 메타데이터(`_woocommerce_pos_data`)에 저장되므로, 필요한 경우 과거 수치를 계속 산출할 수 있습니다.

### 할인 표시를 다시 사용하려는 경우 {#if-you-want-discount-visibility-back}

일일 마감 정산에 "오늘 제공한 총 할인" 수치가 필요하다면, 추적 가능한 할인에는 품목 가격 편집 대신 [쿠폰](/coupons)을 사용하는 것이 지원되는 방식입니다. **마케팅 → 쿠폰**에서 "매니저 10%" 또는 "로열티 $5" 쿠폰을 설정한 뒤 장바구니에 적용하세요. 보고서에 기본적으로 표시됩니다.

쿠폰 할인과 별도로 품목 가격 변경을 표시하는 POS 전용 "총 절감액" 수치도 검토 중입니다. 이 기능이 업무 흐름에 중요하다면 [알려주세요](https://wcpos.com/support).

## 알려진 제한 사항 {#known-limitations}

- **자동 할인 규칙 없음** — POS는 "2개 구매 시 1개 무료" 방식의 자동 할인을 지원하지 않습니다. 구조화된 프로모션에는 WooCommerce 쿠폰을 사용하세요.
- **빠른 할인은 비율만 지원** — 고정 금액 할인을 위한 내장 빠른 버튼은 없습니다. 대신 음수 수수료를 사용하거나 개별 가격을 수정하세요.
