WCPOS에서는 계산원이 빠른 비율 버튼, 항목별 직접 가격 수정, 주문 수준 할인 수수료를 사용해 주문에 즉시 할인을 적용할 수 있습니다. 사용 규칙이 있는 사전 구성 프로모션은 [쿠폰](/coupons) (Pro)을 참조하세요.

:::info v1.9.0에서 변경됨
POS 가격 변경이 WooCommerce 할인 합계에 표시되는 방식이 v1.9.0에서 변경되었습니다. 현재 영수증 템플릿에서는 원래 가격과 POS 절감액을 별도로 표시할 수 있으며, 기본 **할인** 합계는 계속 쿠폰용으로 유지됩니다. 자세한 내용은 아래의 [v1.9.0에서 변경된 사항](#what-changed-in-v190)을 참조하세요.
:::

## 빠른 할인 {#quick-discounts}

빠른 할인 버튼을 사용하면 한 번의 탭으로 전체 주문에 비율 할인을 적용할 수 있습니다.

구성하려면 장바구니 [표시 설정](/pos/cart#display-settings)을 열고 **빠른 할인** 필드에 쉼표로 구분한 비율 목록을 입력합니다. 예를 들어 `5,10,15,20` 은 바로 가기 버튼 4개를 생성합니다.

빠른 할인 버튼을 탭하면 해당 비율이 장바구니의 모든 항목에 적용됩니다.

## 항목별 할인 {#line-item-discounts}

장바구니에서 개별 품목의 가격을 직접 변경할 수 있습니다:

1. 품목의 **가격** 필드를 클릭합니다
2. 새 가격을 입력합니다
3. **Enter** 키를 눌러 확인합니다

가격 일치, 직원 할인 또는 일회성 조정에 유용합니다. 품목 합계는 수량 × 새 가격을 기준으로 자동 업데이트됩니다.

장바구니 품목 편집에 대한 자세한 내용은 [품목](/pos/cart/line-items)을 참조하세요.

:::tip 품목 분할
고객이 같은 상품의 일부에 서로 다른 할인을 적용하려는 경우(예: 3개는 정상가, 2개는 할인), 장바구니 [표시 설정](/pos/cart#display-settings)에서 **분할** 옵션을 활성화하여 하나의 품목을 별도의 행으로 나눕니다.
:::

## 주문 전체 할인 {#order-level-discounts}

개별 품목이 아닌 전체 주문에 정액 할인을 적용하려면 **음수 수수료**를 추가합니다:

1. 장바구니 품목 아래의 **수수료 추가**를 클릭합니다
2. 이름을 입력합니다(예: "직원 할인")
3. 할인 금액을 음수로 입력합니다(예: `-5.00`)

수수료는 별도의 항목으로 표시되며 주문 합계를 줄입니다. 필요한 경우 점 3개 메뉴에서 수수료의 세금 상태를 수정할 수 있습니다.

## POS 할인과 WooCommerce 쿠폰 비교 {#pos-discounts-vs-woocommerce-coupons}

이 페이지의 할인은 계산원이 계산대에서 즉시 적용하는 조정이며, WooCommerce **쿠폰**은 규칙과 추적 기능이 미리 설정된 프로모션입니다. 주요 차이점은 다음과 같습니다.

| | POS 할인 | WooCommerce 쿠폰(Pro) |
|---|---|---|
| **적용 방법** | 빠른 할인, 항목 가격 수정 또는 음수 수수료 | 장바구니에 쿠폰 코드를 입력 |
| **설정 위치** | 계산원이 즉시 설정 | **WP Admin → 마케팅 → 쿠폰**에서 미리 설정 |
| **추적** | 항목 가격으로 기록됨([v1.9.0 변경 사항](#what-changed-in-v190) 참조) | WooCommerce 보고서에서 쿠폰 할인으로 추적됨 |
| **제한 사항** | 없음 — 계산원이 원하는 가격을 설정 | 사용 한도, 상품/카테고리 제한, 최소 구매 금액, 만료일, 이메일 규칙 |
| **적합한 용도** | 즉석 조정, 가격 맞춤 | 체계적인 프로모션, 추적 가능한 할인 |

:::tip 어떤 것을 사용해야 하나요?
일회성 가격 조정에는 이 페이지의 할인이 더 간편합니다. WooCommerce 보고서에서 할인 사용을 추적하거나 사용 횟수 제한과 같은 규칙을 적용해야 하는 경우 [쿠폰](/coupons)을 사용하세요.
:::

## POS 가격 변경과 쿠폰의 상호작용 {#how-pos-price-changes-interact-with-coupons}

계산원이 품목에 맞춤 가격을 설정하고(예: $18에서 $16으로 인하) 이후 쿠폰을 적용하면, 쿠폰은 원래 가격($18)이 아닌 **POS 할인 적용 가격**($16)을 기준으로 계산됩니다. 따라서 계산원 할인과 쿠폰을 원래 가격에 중복 적용하여 고객에게 과도한 할인이 제공되는 것을 방지합니다.

- WooCommerce에서는 POS 할인 적용 품목을 "할인 중"으로 처리합니다. 쿠폰에서 **할인 상품 제외**를 활성화하면 일반 할인 품목과 마찬가지로 POS 할인 적용 품목은 제외됩니다. 개발자는 `woocommerce_pos_item_is_on_sale` 필터로 이 동작을 재정의할 수 있습니다.
- 쿠폰을 제거하면 품목은 POS 할인 적용 가격으로 유지됩니다.

:::note 개발자 참조
POS 가격 재정의의 저장 방식과 사용 가능한 필터에 대한 기술적 세부 정보는 [POS 할인 참조](/reference/pos-discounts)를 참조하세요.
:::

## v1.9.0의 변경 사항 {#what-changed-in-v190}

v1.8에서 업그레이드한 후 영수증과 보고서의 **할인** 합계가 이제 **0**으로 표시되는 것을 확인했다면, 이 섹션에서 그 이유와 선택 가능한 방법을 설명합니다.

### 변경 사항 {#the-change}

v1.9.0 이전에는 계산원이 품목 가격을 인하하면(예: $18 → $16), POS는 `subtotal = $18` 및 `total = $16` 으로 주문을 기록했습니다. 그러면 WooCommerce는 `discount_total = $2` 를 계산했고, 이 값은 영수증과 보고서에 할인으로 표시되었습니다.

이로 인해 쿠폰을 추가로 적용할 때마다 총액이 잘못 계산되었습니다. 쿠폰이 원래 가격인 $18을 기준으로 계산되어 과도한 할인이 발생하고 고객에게 적은 금액이 청구되었습니다.

v1.9.0부터 WCPOS는 WooCommerce의 세일 가격 처리 방식에 맞춰 동작합니다. 계산대에서 설정한 가격이 품목 소계입니다. WooCommerce는 쿠폰 코드만 "할인"으로 계산합니다(이는 세일 중인 상품에 대한 WC의 동작과 같습니다. 세일 가격도 별도의 할인 항목 없이 소계에 반영됩니다).

### 의미 {#what-this-means-for-you}

- **영수증**에는 기록된 정가, 정가와 판매가의 차액, 그리고 해당 절감액과 쿠폰을 합산한 **총 절감액**을 표시할 수 있습니다. 가격 절감액은 WooCommerce의 **할인** 필드와 별도로 유지됩니다.
- **보고서**에는 POS 품목 가격 변경만 사용한 경우 `discount_total = 0` 이 표시됩니다. 쿠폰 할인만 계산됩니다.
- **쿠폰**은 이제 POS에서 할인된 품목에 중복 적용해도 올바르게 계산됩니다.
- **기록된 정가와 판매가가 저장됩니다.** POS 품목 메타데이터(`_woocommerce_pos_data`)에 저장되므로, WooCommerce의 할인 총액을 변경하지 않고도 영수증에서 절감액을 계산할 수 있습니다.

### 영수증에 원래 가격과 절감액 표시 {#showing-the-original-price-and-saving-on-receipts}

현재 번들로 제공되는 가격 표시 템플릿은 상품이 세일 중이거나 계산대에서 가격이 변경된 경우 기록된 정가와 절감액을 표시합니다. 또한 WCPOS에서 주문 전체 기준의 완전한 금액을 계산할 수 있으면 **총 절감액**도 표시합니다. 이 총액은 정가 기준 절감액과 쿠폰 할인을 어느 쪽도 중복 계산하지 않고 합산합니다.

갤러리에서 복사한 템플릿은 편집 가능한 스냅샷이며 WCPOS 업데이트로 덮어쓰이지 않습니다. 이 변경 전 영수증을 만들었다면 다음 중 하나를 수행하세요.

1. 번들 템플릿의 새 복사본에서 **템플릿 사용**을 클릭하거나
2. 기존 템플릿을 [영수증 데이터 참조](/receipts/receipt-data#displaying-regular-price-and-savings)의 필드로 업데이트합니다.

주요 영수증 필드는 다음과 같습니다:

| 필드 | 의미 |
|-------|---------|
| `lines[].regular_price_display` | 기록된 정상 단가 |
| `lines[].selling_price_display` | 쿠폰 적용 전 판매 단가 |
| `lines[].unit_savings_display` | 항목당 절감액 |
| `lines[].line_regular_total_display` | 해당 항목의 정상가 합계 |
| `lines[].line_savings_display` | 해당 항목의 총 절감액 |
| `totals.savings_total_display` | 카탈로그 할인 및 계산대 가격 변경으로 인한 정상가 대비 판매가 총 절감액 |
| `totals.discount_total_display` | WooCommerce 할인 총액이며, 일반적으로 현재 주문에 적용된 쿠폰입니다 |
| `totals.total_saved_display` | 정상가 대비 절감액과 WooCommerce 할인의 합계 |
| `totals.total_saved_complete` | 합계가 완전하며 안전하게 표시할 수 있는지 여부 |

사용자 지정 템플릿에서는 이전 가격 데이터가 불완전하거나 절감된 금액이 없을 때 **총 절감액** 행이 표시되지 않도록 처리해야 합니다 — 정확한 패턴은 [영수증 데이터 참조](/receipts/receipt-data#totals)를 확인하세요.

WooCommerce 보고서는 변경되지 않습니다. 기본 할인 합계에는 여전히 현재 판매가 또는 계산대 가격 절감액이 아닌 쿠폰이 포함됩니다. 영수증에는 `totals.total_saved` 를 사용하고, 보고서에서도 절감액을 WooCommerce 할인으로 추적해야 하는 경우 계속해서 [쿠폰](/coupons)을 사용하세요.

## 알려진 제한 사항 {#known-limitations}

- **자동 할인 규칙 없음** — POS는 "2개 구매 시 1개 무료"와 같은 자동 할인을 지원하지 않습니다. 체계적인 프로모션에는 WooCommerce 쿠폰을 사용하세요.
- **빠른 할인은 백분율만 지원** — 고정 금액 할인에 사용할 기본 제공 빠른 버튼은 없습니다. 대신 음수 수수료를 사용하거나 개별 가격을 수정하세요.
