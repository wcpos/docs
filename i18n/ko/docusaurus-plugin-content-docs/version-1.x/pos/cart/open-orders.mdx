WCPOS를 사용하면 여러 주문을 동시에 처리할 수 있습니다. 이는 고객 보류, 거래 간 전환, 중단 복구에 유용합니다.

## 열린 주문 회전판

장바구니 패널 하단에 가로 회전판으로 모든 열린 주문이 표시됩니다:

{/* Screenshot: open-orders-carousel.png */}

- 각 장바구니는 **총 금액**을 표시합니다
- **현재 주문**은 강조 표시됩니다
- 클릭하여 해당 장바구니로 전환
- 좌우로 스크롤하여 더 많은 장바구니 보기

## 새 주문 생성

항상 빈 장바구니를 사용할 수 있습니다. 회전판의 빈 장바구니를 클릭하거나 현재 장바구니가 비어 있을 때 상품을 추가하기 시작하면 됩니다.

## 주문 간 전환

회전판의 주문을 클릭하여 해당 주문으로 전환할 수 있습니다. 장바구니 패널이 선택된 주문의 내용으로 업데이트됩니다.

**사용 사례:**

- 고객이 다른 물건을 가져오는 동안
- 대기 중인 대량 주문 처리 중 빠른 고객 지원
- 주문 간 가격 또는 상품 비교

## 서버에 주문 저장

주문은 두 가지 상태로 존재합니다:

### 로컬에만 저장

기본적으로 새 주문은 로컬 브라우저/앱 데이터베이스에만 저장됩니다. 페이지 새로고침 시 지속되지만:

- WooCommerce 관리자에서 보이지 않음
- 로컬 데이터베이스가 지워지면 손실됨
- 다른 기기에서 접근 불가

### 서버에 저장

**서버에 저장**을 클릭하여 상태가 `pos-open`인 WooCommerce 주문을 생성합니다. 이렇게 하면:

- WooCommerce에 실제 주문 생성
- 로컬 데이터베이스가 지워져도 유지
- 다른 기기에서 접근 가능
- WP 관리자 > WooCommerce > 주문에 표시

:::tip 저장 시기
다음과 같은 경우 주문을 서버에 저장하세요:
- 고객이 나중에 픽업할 주문을 보류하는 경우
- 교대 근무를 마무리하고 다른 계산원이 계속할 경우
- 앱/브라우저 문제에 대비해 백업이 필요한 경우
:::

## 저장된 주문 복구

서버에 저장한 주문은 다음과 같이 다시 접근할 수 있습니다:

1. **주문** 화면 열기 (프로 기능)
2. `pos-open` 상태로 필터링
3. 주문 다시 열기

## 주문 지속성

### 로컬 저장소

WCPOS는 IndexedDB를 사용하여 주문을 로컬에 저장합니다. 이를 통해:

- 브라우저 세션 간 지속
- 네트워크 요청 없이 빠른 접근
- 오프라인 기능 제공

### 서버와 동기화

서버에 저장하거나 체크아웃할 때:

- 주문이 WooCommerce로 전송됨
- 확인 응답 수신
- 로컬 및 서버 데이터 동기화

## 주문 무효화

열린 주문을 제거하려면:

1. 제거할 주문으로 전환
2. **무효화** 버튼 클릭

**발생하는 상황:**

- **저장되지 않은 주문:** 로컬 데이터베이스에서 영구적으로 삭제
- **저장된 주문:** WooCommerce에서 휴지통으로 이동 후 로컬에서 삭제

무효화된 저장된 주문 복구 방법:
1. `WP 관리자 > WooCommerce > 주문 > 휴지통`으로 이동
2. 주문 복원

## 팁

### 주문 정리

여러 열린 주문으로 작업할 때:

- 쉬운 식별을 위해 주문에 고객 이름 추가
- 보류 이유를 설명하는 주문 메모 추가
- 중요한 주문 서버에 저장

### 교대 인계

교대를 마무리할 때:

1. 모든 중요한 주문을 서버에 저장
2. 상태를 설명하는 주문 메모 추가
3. 다음 계산원이 주문 화면에서 접근 가능

### 오프라인 고려사항

연결이 끊어진 경우:

- 로컬 주문은 계속 접근 가능하고 상품 추가 가능
- 연결 복원 시까지 주문 완료/체크아웃 불가
- 연결 복원 시까지 서버에 주문 저장 불가
- 연결 복원 시까지 새 고객 생성 불가
