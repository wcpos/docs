판매를 완료할 준비가 되면 **결제** 버튼을 클릭하여 결제 모달을 엽니다. 여기에서 결제를 처리하고 주문을 완료합니다.

{/* Screenshot: checkout-modal.png */}

## 결제 모달 개요

결제 모달에는 다음이 표시됩니다:

- **주문 번호** - WooCommerce 주문 ID
- **지불 금액** - 총 금액
- **결제 설정** 버튼 - 표시 문제 해결
- **캐셔** - 주문을 처리하는 사람
- **고객** - 이 주문의 고객 (클릭 가능한 링크)
- **쿠폰 코드** 필드 - 할인 코드 적용
- **주문 요약** - 제품, 수량 및 총액
- **지불 방법** - 사용 가능한 지불 옵션
- **취소 / 결제 처리** 버튼

## 지불 방법

### 무료 버전에서 사용 가능

WCPOS의 무료 버전에는 두 개의 지불 게이트웨이가 포함되어 있습니다:

- **현금** - 지불 금액과 거스름돈 계산기
- **카드** - 외부 카드 단말기용

### 추가 게이트웨이 (Pro)

:::info Pro 기능
추가 지불 게이트웨이는 [WCPOS Pro](/getting-started/pro-license)가 필요합니다.
:::

Pro에서는 다음을 활성화할 수 있습니다:

- **Stripe 터미널** - Stripe 카드 리더기와의 직접 통합
- **SumUp 터미널** - SumUp 카드 리더기와의 통합
- **사용자 정의 게이트웨이** - 자신의 지불 통합 생성

게이트웨이 구성에 대한 자세한 내용은 [지불](/payment)을 참조하세요.

### 지불 방법 선택

지불 방법을 클릭하여 선택합니다. 양식이 관련 필드로 업데이트됩니다:

**현금:**

- **입금된 금액** - 고객이 주는 금액 입력
- **거스름돈** - 자동 계산된 거스름돈

{/* Screenshot: checkout-cash-payment.png */}

**카드:**

- 외부 카드 단말기에서 결제 처리
- 결제 처리 클릭하여 완료

## 쿠폰 코드

할인 적용 방법:

1. 필드에 쿠폰 코드를 입력
2. **쿠폰 적용** 클릭
3. 할인 요약에서 표시됨

쿠폰은 WooCommerce에서 구성해야 합니다 (`WP Admin > 마케팅 > 쿠폰`).

## 결제 처리

1. 지불 방법 선택
2. 현금의 경우, 입금된 금액 입력
3. **결제 처리** 클릭
4. 주문이 완료되고 [영수증](/pos/checkout/receipts)이 표시됩니다

## 결제 설정 (문제 해결)

결제 모달은 WooCommerce 주문 결제 페이지를 표시하기 위해 iframe/webview를 사용합니다. 이는 WooCommerce의 기존 결제 인프라를 활용하므로 WooCommerce와 호환되는 모든 지불 게이트웨이가 POS에서 작동해야 합니다.

그러나 테마 및 플러그인 스크립트가 간섭할 수 있습니다. **결제 설정**을 클릭하여 문제를 해결하세요:

### 모든 스타일 및 스크립트 비활성화

:::warning 핵 옵션
이것은 핵 옵션이며, 테스트 용도로만 사용하거나 개발자가 무엇을 하고 있는지 아는 드문 경우에만 사용해야 합니다. 모든 wp_head 스크립트를 비활성화하면 결제 게이트웨이를 확장/축소하는 데 필요한 WooCommerce 스크립트조차 제거되어 결제 기능이 중단될 수 있습니다.
:::

- **wp_head 비활성화** - WordPress 헤더에서 모든 스크립트/스타일을 제거합니다.
- **wp_footer 비활성화** - WordPress 바닥글에서 모든 스크립트/스타일을 제거합니다.

### 선택적 스타일 비활성화

표시 문제를 일으킬 수 있는 CSS를 선택적으로 비활성화합니다:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- 기타.

### 선택적 스크립트 비활성화

지불 게이트웨이에 간섭할 수 있는 JavaScript를 선택적으로 비활성화합니다:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- 기타.

:::tip
지불 게이트웨이가 올바르게 표시되지 않으면:

1. 먼저 테마 스타일 비활성화 시도
2. 필요 없는 WooCommerce 스크립트 비활성화 시도
3. 지불 게이트웨이에 필요한 스크립트를 비활성화하지 않도록 주의하세요.
   :::

## 취소

**취소**를 클릭하여 주문을 완료하지 않고 결제 모달을 닫습니다. 주문은 열린 장바구니로 남습니다.
