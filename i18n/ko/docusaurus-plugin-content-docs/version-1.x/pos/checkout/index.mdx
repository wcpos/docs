판매를 완료할 준비가 되면 **체크아웃** 버튼을 클릭하여 체크아웃 모달을 엽니다. 여기서 결제를 처리하고 주문을 완료합니다.

{/* Screenshot: checkout-modal.png */}

## 체크아웃 모달 개요

체크아웃 모달은 다음을 표시합니다:

- **주문 번호** - WooCommerce 주문 ID
- **지불할 금액** - 총 청구 금액
- **체크아웃 설정** 버튼 - 표시 문제 해결
- **캐셔** - 주문을 처리하는 사람
- **고객** - 이 주문의 고객(클릭 가능한 링크)
- **쿠폰 코드** 필드 - 할인 코드 적용
- **주문 요약** - 제품, 수량 및 총액
- **지불 방법** - 사용 가능한 지불 옵션
- **취소 / 결제 처리** 버튼

## 지불 방법

### 무료 버전에서 제공

WCPOS의 무료 버전에는 두 가지 결제 게이트웨이가 포함되어 있습니다:

- **현금** - 지불 금액과 잔돈 계산기 포함
- **카드** - 외부 카드 단말기용

### 추가 게이트웨이 (프로)

:::info 프로 기능
추가 결제 게이트웨이는 [WCPOS Pro](/getting-started/pro-license)가 필요합니다.
:::

Pro를 이용하면 다음을 활성화할 수 있습니다:

- **스트라이프 터미널** - Stripe 카드 리더와의 직접 통합
- **SumUp 터미널** - SumUp 카드 리더와의 통합
- **커스텀 게이트웨이** - 자신만의 결제 통합 생성

게이트웨이 구성에 대한 자세한 내용은 [지불](/payment)을 참조하세요.

### 지불 방법 선택

지불 방법을 클릭하여 선택합니다. 양식이 업데이트되어 관련 필드를 표시합니다:

**현금:**
- **지불 금액** - 고객이 제공하는 금액을 입력합니다
- **잔돈** - 자동으로 계산된 잔돈

{/* Screenshot: checkout-cash-payment.png */}

**카드:**
- 외부 카드 단말기에서 결제를 처리합니다
- 결제를 완료하려면 결제 처리 버튼을 클릭합니다

## 쿠폰 코드

할인을 적용하려면:

1. 필드에 쿠폰 코드를 입력합니다
2. **쿠폰 적용**을 클릭합니다
3. 할인액이 주문 요약에 표시됩니다

쿠폰은 WooCommerce에서 구성해야 합니다(`WP Admin > 마케팅 > 쿠폰`).

## 결제 처리

1. 지불 방법을 선택합니다
2. 현금의 경우, 지불 금액을 입력합니다
3. **결제 처리**를 클릭합니다
4. 주문이 완료되고 [영수증](/pos/checkout/receipts)이 표시됩니다

## 체크아웃 설정 (문제 해결)

체크아웃 모달은 iframe/webview를 사용하여 WooCommerce 주문 결제 페이지를 표시합니다. 이는 WooCommerce의 기존 결제 인프라를 활용하며, WooCommerce와 호환되는 모든 결제 게이트웨이는 POS에서 작동해야 합니다.

그러나 테마 및 플러그인 스크립트가 간섭할 수 있습니다. 문제 해결을 위해 **체크아웃 설정**을 클릭하십시오:

{/* Screenshot: checkout-settings.png */}

### 모든 스타일 및 스크립트 비활성화

- **wp_head 비활성화** - WordPress 헤더의 모든 스크립트/스타일 제거
- **wp_footer 비활성화** - WordPress 푸터의 모든 스크립트/스타일 제거

### 선택한 스타일 비활성화

디스플레이 문제를 일으킬 수 있는 CSS를 선택적으로 비활성화합니다:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- 등

### 선택한 스크립트 비활성화

결제 게이트웨이에 간섭할 수 있는 JavaScript를 선택적으로 비활성화합니다:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- 등

:::tip
결제 게이트웨이가 제대로 표시되지 않으면:
1. 먼저 테마 스타일 비활성화를 시도하십시오
2. 그 후, 필요하지 않은 WooCommerce 스크립트를 비활성화하십시오
3. 결제 게이트웨이에 필요한 스크립트를 비활성화하지 않도록 주의하십시오
:::

## 취소

**취소**를 클릭하여 주문을 완료하지 않고 체크아웃 모달을 닫습니다. 주문은 열린 장바구니로 남아 있습니다.
