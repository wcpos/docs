판매를 완료할 준비가 되면 **결제** 버튼을 클릭하여 결제 모달을 엽니다. 이 화면에서 결제를 처리하고 주문을 완료합니다.

{/* Screenshot: checkout-modal.png */}

## 결제 모달 개요 {#checkout-modal-overview}

결제 모달에는 다음 정보가 표시됩니다:

- **주문 번호** - WooCommerce 주문 ID
- **결제 금액** - 총 결제 예정 금액
- **결제 설정** 버튼 - 표시 문제 해결
- **계산원** - 주문을 처리하는 담당자
- **고객** - 해당 주문의 고객 (클릭 가능한 링크)
- **쿠폰 추가** 버튼 - 할인 코드 적용
- **주문 요약** - 상품, 수량 및 합계
- **결제 수단** - 사용 가능한 결제 옵션
- **취소 / 결제 처리** 버튼

## 결제 수단 {#payment-methods}

### 무료 버전에서 사용 가능 {#available-in-free-version}

WCPOS 무료 버전에는 두 가지 결제 게이트웨이가 포함되어 있습니다:

- **현금** - 받은 금액 및 거스름돈 계산기 포함
- **카드** - 외부 카드 단말기용

### 추가 게이트웨이 (Pro) {#additional-gateways-pro}

:::info Pro 기능
추가 결제 게이트웨이를 사용하려면 [WCPOS Pro](/getting-started/pro-license)가 필요합니다.
:::

Pro에서는 다음 기능을 활성화할 수 있습니다:

- **Stripe Terminal** - Stripe 카드 리더기와의 직접 연동
- **SumUp Terminal** - SumUp 카드 리더기와의 연동
- **사용자 정의 게이트웨이** - 자체 결제 연동 생성

게이트웨이 설정에 대한 자세한 내용은 [결제](/payment)를 참조하세요.

### 결제 수단 선택 {#selecting-a-payment-method}

결제 수단을 클릭하여 선택합니다. 선택한 결제 수단에 따라 관련 입력 필드가 표시됩니다:

**현금:**

- **받은 금액** - 고객이 지불한 금액을 입력합니다
- **거스름돈** - 반환할 거스름돈이 자동으로 계산됩니다

{/* Screenshot: checkout-cash-payment.png */}

**카드:**

- 외부 카드 단말기에서 결제를 처리합니다
- 결제 처리를 클릭하여 완료합니다

## 쿠폰 {#coupon-codes}

장바구니의 합계 위에 **쿠폰 추가** 입력란이 있습니다(Pro 전용). 코드를 입력하거나 설명으로 검색하면 쿠폰이 로컬에서 유효성 검사를 거친 후 제거 가능한 태그로 표시됩니다. 여러 쿠폰을 순차적으로 적용할 수 있습니다.

검색, 태그, 순차 할인, 유효성 검사 오류 및 해결 방법 표 등 전체 워크플로는 **[계산대에서 쿠폰 적용하기](/coupons/applying-coupons)**를 참조하세요. 쿠폰 유형, 유효성 검사 규칙 및 WooCommerce에서의 설정 방법은 [쿠폰](/coupons)을 참조하세요.

## 결제 처리 {#processing-payment}

1. 결제 수단을 선택합니다
2. 현금 결제의 경우 받은 금액을 입력합니다
3. **결제 처리**를 클릭합니다
4. 주문이 완료되고 [영수증](/pos/checkout/receipts)이 표시됩니다

## 결제 설정 (문제 해결) {#checkout-settings-troubleshooting}

결제 모달은 iframe/webview를 사용하여 WooCommerce 주문 결제 페이지를 표시합니다. 이를 통해 WooCommerce의 기존 결제 인프라를 활용하므로, WooCommerce에서 작동하는 모든 결제 게이트웨이는 POS에서도 작동합니다.

그러나 테마 및 플러그인 스크립트가 간섭을 일으킬 수 있습니다. **결제 설정**을 클릭하여 문제를 해결하세요:

### 모든 스타일 및 스크립트 비활성화 {#disable-all-styles-and-scripts}

:::warning 최후의 수단
이 옵션은 최후의 수단이며, 테스트 목적이거나 개발자가 작업 내용을 정확히 이해하는 드문 경우에만 사용해야 합니다. 모든 wp_head 스크립트를 비활성화하면 결제 게이트웨이를 확장/축소하는 데 필요한 WooCommerce 스크립트까지 제거되어 결제 기능이 손상될 수 있습니다.
:::

- **wp_head 비활성화** - WordPress 헤더에서 모든 스크립트/스타일을 제거합니다
- **wp_footer 비활성화** - WordPress 푸터에서 모든 스크립트/스타일을 제거합니다

### 선택한 스타일 비활성화 {#disable-selected-styles}

표시 문제를 일으킬 수 있는 CSS를 선택적으로 비활성화합니다:

- wp-emoji-styles
- wp-block-library
- classic-theme-styles
- woocommerce-layout
- woocommerce-smallscreen
- woocommerce-general
- 등.

### 선택한 스크립트 비활성화 {#disable-selected-scripts}

결제 게이트웨이에 간섭할 수 있는 JavaScript를 선택적으로 비활성화합니다:

- wc-add-to-cart
- selectWoo
- wc-checkout
- woocommerce
- html5shiv
- 등.

:::tip
결제 게이트웨이가 올바르게 표시되지 않는 경우:

1. 먼저 테마 스타일을 비활성화해 보십시오
2. 그런 다음 필요하지 않은 WooCommerce 스크립트를 비활성화해 보십시오
3. 결제 게이트웨이에 필요한 스크립트를 비활성화하지 않도록 주의하십시오
   :::

## 취소 {#cancel}

**취소**를 클릭하면 주문을 완료하지 않고 결제 모달을 닫을 수 있습니다. 주문은 열린 장바구니 상태로 유지됩니다.
