로그 화면은 POS 내의 시스템 활동을 기록합니다. 문제를 디버깅하고 백그라운드에서 발생하는 일을 이해하는 데 필수적인 도구입니다.

{/* Screenshot: logs-screen.png */}

## 로그 접근

왼쪽 탐색 서랍(하트비트 아이콘)에서 로그 화면을 엽니다.

## 인터페이스 개요

### 검색 및 필터

- **로그 검색** - 메시지 내용으로 특정 로그 항목 찾기
- **로그 레벨 필터** - 로그 레벨별로 필터링 (정보, 경고, 오류)
- **표시 설정** - 표시할 열 구성

### 로그 테이블

각 로그 항목에는 다음이 표시됩니다:

| 열 | 설명 |
|--------|-------------|
| **시간** | 이벤트 발생 시간 |
| **레벨** | 심각도: 정보 (회색), 경고 (주황), 오류 (빨강) |
| **메시지** | 이벤트의 사용자 친화적 설명 |
| **컨텍스트** | 추가 세부 정보가 포함된 확장 가능한 JSON |
| **코드** | 오류 코드 (문서 링크) |

## 로그 레벨

### 정보 (회색)

정상 작동 및 성공한 이벤트:
- "모든 제품 ID 가져오기 완료"
- "새 고객 동기화"
- "모든 세금 ID 가져오기 완료"

### 경고 (주황)

주의가 필요할 수 있는 비치명적 문제:
- "세션 새로고침 불가"
- "동기화 재시도"

### 오류 (빨강)

해결해야 할 문제:
- "죄송합니다, 리소스를 나열할 수 없습니다" (권한 문제)
- "API 요청 실패"
- "데이터베이스 동기화 오류"

## 컨텍스트 데이터

모든 로그 항목에서 확장 화살표를 클릭하여 자세한 JSON 컨텍스트를 확인하세요:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

이 데이터에는 다음이 포함됩니다:
- **항목 수** - 영향받은 레코드 수
- **실행 시간** - 작업 소요 시간
- **서버 부하** - 서버 성능 지표
- **요청/응답 세부 정보** - API 문제 디버깅용

## 오류 코드

많은 로그 항목에는 **오류 코드**(예: `API02004`, `DB01001`)가 포함됩니다. 이러한 코드는 [오류 코드](/error-codes) 문서에 직접 연결되며, 다음을 제공합니다:

- 오류에 대한 자세한 설명
- 일반적인 원인
- 문제 해결 단계

## 일반적인 로그 항목

### 동기화 작업

```
[info] 모든 제품 ID 가져오기 완료    {...} 3개 항목
[info] 새 고객 동기화            {...} 1개 항목
[info] 모든 세금 ID 가져오기 완료       {...} 3개 항목
```

이는 정상적인 동기화 활동을 나타냅니다. 컨텍스트는 동기화된 항목 수를 보여줍니다.

### 권한 오류

```
[error] 죄송합니다, 리소스를 나열할 수 없습니다    {...} API02004
```

이는 보통 사용자에게 리소스에 대한 권한이 없음을 의미합니다. `WP Admin > POS > 설정 > 접근` 메뉴에서 사용자 역할을 확인하세요.

### 세션 문제

```
[warn] 세션 새로고침 불가    {...} API02007
```

POS가 인증 세션을 새로고침할 수 없습니다. 로그아웃 후 다시 로그인해야 할 수 있습니다.

## 지원을 위한 로그 사용

지원을 요청할 때:

1. **문제 재현** - 문제를 발생시키는 작업 수행
2. **로그 확인** - 오류(빨강) 또는 경고(주황) 확인
3. **컨텍스트 확장** - 전체 JSON 세부 정보 확인
4. **오류 코드 메모** - 지원 요청에 포함
5. **관련 항목 복사** - 컨텍스트와 함께 로그 메시지 공유

## 성능 모니터링

로그에는 실행 시간 및 서버 부하 지표가 포함됩니다. 다음 사항을 주의하세요:

- **높은 실행 시간** - 서버가 과부하되거나 느릴 수 있음
- **높은 서버 부하 값** - 서버 최적화 고려
- **빈번한 타임아웃** - 네트워크 연결 확인
