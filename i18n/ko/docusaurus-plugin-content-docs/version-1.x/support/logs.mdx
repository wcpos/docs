로그 화면은 POS 내에서 시스템 활동을 기록합니다. 문제를 디버깅하고 무대 뒤에서 무슨 일이 벌어지고 있는지 이해하는 필수 도구입니다.

{/* 스크린샷: logs-screen.png */}

## 로그 접근하기

왼쪽 탐색 메뉴(심장 박동 아이콘)에서 로그 화면을 엽니다.

## 인터페이스 개요

### 검색 및 필터

- **로그 검색** - 메시지 내용을 통해 특정 로그 항목 찾기
- **로그 레벨 필터** - 로그 레벨(정보, 경고, 오류)로 필터링
- **표시 설정** - 보이는 열 구성

### 로그 테이블

각 로그 항목은 다음을 보여줍니다:

| 열   | 설명                             |
|------|----------------------------------|
| **시간** | 사건이 발생한 시간              |
| **레벨** | 심각도: info (회색), warn (주황), error (빨강) |
| **메시지** | 사건에 대한 인간이 이해할 수 있는 설명 |
| **컨텍스트** | 추가 세부 정보를 포함한 확장 가능한 JSON |
| **코드** | 오류 코드(문서 링크)          |

## 로그 레벨

### Info (회색)

일반 작업 및 성공적인 사건:
- "제품에 대한 모든 ID를 가져왔습니다."
- "새 고객을 동기화했습니다."
- "세금에 대한 모든 ID를 가져왔습니다."

### Warning (주황)

주의가 필요할 수 있는 비치명적인 문제:
- "세션을 새로 고칠 수 없습니다."
- "동기화를 위한 재시도 시도"

### Error (빨강)

해결해야 할 문제:
- "죄송합니다, 자원을 나열할 수 없습니다." (권한 문제)
- "API 요청 실패"
- "데이터베이스 동기화 오류"

## 컨텍스트 데이터

어떤 로그 항목에서든 확장 화살표를 클릭하여 세부 JSON 컨텍스트를 확인하십시오:

```json
{
  "total": "6",
  "execution_time": "54.79 ms",
  "server_load": "[10.51, 10.42, 9.98]"
}
```

이 데이터에는 다음 내용이 포함됩니다:
- **항목 수** - 영향을 받은 레코드 수
- **실행 시간** - 작업에 걸린 시간
- **서버 부하** - 서버 성능 메트릭
- **요청/응답 세부정보** - API 문제 디버깅을 위한

## 오류 코드

많은 로그 항목에는 **오류 코드**(예: `API02004`, `DB01001`)가 포함됩니다. 이러한 코드는 [오류 코드](/error-codes) 문서에 직접 연결됩니다. 이 문서에서는:

- 오류에 대한 자세한 설명
- 일반 원인
- 문제 해결 단계

## 일반 로그 항목

### 동기화 작업

```
[info] Fetched all IDs for products    {...} 3 items
[info] Synced new customers            {...} 1 item
[info] Fetched all IDs for taxes       {...} 3 items
```

이들은 정상적인 동기화 활동을 나타냅니다. 컨텍스트는 몇 개의 항목이 동기화되었는지를 보여줍니다.

### 권한 오류

```
[error] Sorry, you cannot list resources    {...} API02004
```

이는 일반적으로 사용자가 자원에 접근할 권한이 없음을 의미합니다. `WP Admin > POS > 설정 > 접근`에서 사용자 역할을 확인하십시오.

### 세션 문제

```
[warn] Unable to refresh session    {...} API02007
```

POS가 인증 세션을 새로 고칠 수 없었습니다. 로그아웃 후 다시 로그인을 해야 할 수 있습니다.

## 지원을 위한 로그 사용하기

지원 요청 시:

1. **문제를 재현합니다.** - 문제를 일으키는 작업을 수행합니다.
2. **로그를 확인합니다.** - 오류(빨간색) 또는 경고(주황색)를 찾습니다.
3. **컨텍스트를 확장합니다.** - 전체 JSON 세부정보를 가져옵니다.
4. **오류 코드를 기록합니다.** - 이를 지원 요청에 포함합니다.
5. **관련 항목을 복사합니다.** - 컨텍스트와 함께 로그 메시지를 공유합니다.

## 성능 모니터링

로그에는 실행 시간과 서버 부하 메트릭이 포함됩니다. 다음 사항을 주의하십시오:

- **높은 실행 시간** - 서버가 과부하되었거나 느릴 수 있습니다.
- **높은 서버 부하 값** - 서버 최적화를 고려하십시오.
- **빈번한 타임아웃** - 네트워크 연결 상태를 확인합니다.
