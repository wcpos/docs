"'정의되지 않은 속성 읽기(읽기 'data')' 오류"는 POS(판매 시점) 시스템이 서버로부터 데이터를 받기를 기대하지만 대신 빈 응답을 받을 때 발생합니다.

## 가능한 원인

- **플러그인 또는 테마 충돌:** 다른 WordPress 플러그인이나 테마와의 충돌이 데이터 검색을 방해하고 빈 응답을 초래할 수 있습니다.
- **구식 소프트웨어 버전:** 구식 버전의 WordPress, 플러그인 또는 테마를 사용하면 호환성 문제와 예상치 못한 행동이 발생할 수 있습니다. [최소 요구 사항](/getting-started/installation)을 충족하는지 확인하세요.
- **서버 잘못 구성 또는 문제:** 메모리 부족, 제한된 실행 시간 또는 서버 측 오류와 같은 서버의 잘못된 구성이나 문제가 데이터의 적절한 생성 및 전송을 방해할 수 있습니다.

## 문제 해결 단계

### 1. 브라우저의 네트워크 탭 확인

1. 브라우저에서 POS 애플리케이션을 엽니다.
2. 페이지를 마우스 오른쪽 버튼으로 클릭하고 "검사"를 선택하거나 `Ctrl + Shift + I` (macOS의 경우 `Cmd + Option + I`)를 눌러 개발자 도구를 엽니다.
3. 개발자 도구에서 "네트워크" 탭으로 이동합니다.
4. 실패한 요청을 찾아보세요. 보통 빨간색으로 표시되거나 관련 HTTP 상태 코드(예: 500 내부 서버 오류)가 표시됩니다.
5. 실패한 요청을 선택하여 세부정보를 확인합니다.
6. 요청의 응답 섹션을 검토하여 오류 메시지나 JSON 데이터의 문제를 확인합니다.
7. 관련 오류 메시지를 복사하거나 스크린샷을 찍습니다.

### 2. WooCommerce 치명적인 오류 로그 확인

1. WordPress 관리자 영역에 로그인합니다.
2. WordPress 사이드바 메뉴에서 "WooCommerce" > "상태" > "로그"로 이동합니다.
3. "fatal_error" 로그 섹션을 찾습니다.
4. 오류가 발생한 날짜(보통 오늘 날짜)의 로그 항목을 클릭하여 오류의 세부정보를 확인합니다.
5. 이 섹션에 기록된 오류의 원인을 식별합니다.

### 3. POS 로그 확인

POS [로그 화면](/support/logs)에도 문제를 식별하는 데 도움이 되는 관련 오류 정보와 오류 코드가 포함되어 있을 수 있습니다.
