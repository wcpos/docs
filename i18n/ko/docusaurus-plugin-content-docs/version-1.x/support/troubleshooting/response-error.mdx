"정의되지 않은 속성을 읽을 수 없음 (reading 'data')" 오류는 포인트 오브 세일(POS) 시스템이 서버로부터 데이터를 수신할 것으로 예상했지만 대신 빈 응답을 받았을 때 발생합니다.

## 가능한 원인

- **플러그인 또는 테마 충돌:** 다른 WordPress 플러그인 또는 테마와의 충돌로 인해 데이터 검색에 방해가 되고 빈 응답을 초래할 수 있습니다.
- **오래된 소프트웨어 버전:** WordPress, 플러그인 또는 테마의 오래된 버전을 사용하면 호환성 문제와 예상치 못한 동작을 야기할 수 있습니다. [최소 요구 사항](/getting-started/installation)을 충족하는지 확인하세요.
- **서버 설정 또는 문제:** 데이터의 적절한 생성 및 전달을 방해하는 서버 설정 문제나 메모리 부족, 실행 시간 제한, 서버 측 오류 등이 있을 수 있습니다.

## 문제 해결 단계

### 1. 브라우저의 네트워크 탭 확인

1. 브라우저에서 POS 애플리케이션을 엽니다.
2. 페이지를 마우스 오른쪽 버튼으로 클릭하고 "검사"를 선택하거나 `Ctrl + Shift + I`(macOS에서는 `Cmd + Option + I`)를 눌러 개발자 도구를 엽니다.
3. 개발자 도구에서 "네트워크" 탭으로 이동합니다.
4. 빨간색으로 표시되거나 관련 HTTP 상태 코드(예: 500 내부 서버 오류)가 있는 실패한 요청을 찾습니다.
5. 실패한 요청을 선택하여 세부 정보를 봅니다.
6. 요청의 응답 섹션을 검사하여 JSON 데이터의 오류 메시지나 문제를 확인합니다.
7. 관련 오류 메시지를 복사하거나 스크린샷을 찍습니다.

### 2. WooCommerce 치명적 오류 로그 확인

  <Image
    alt="치명적 오류 로그"

  <p class="image-caption"><a target="_blank" rel="noopener noreferrer" href="https://woocommerce.com/document/finding-php-error-logs/">WooCommerce 치명적 오류 로그</a> 예시</p>
</div>

1. WordPress 관리 영역에 로그인합니다.
2. WordPress 사이드바 메뉴에서 "WooCommerce" > "상태" > "로그"로 이동합니다.
3. "fatal_error" 로그 섹션을 찾습니다.
4. 오류가 발생한 날의 로그 항목(보통 오늘 날짜)을 클릭하여 오류 세부 정보를 확인합니다.
5. 이 섹션에 기록된 오류 원인을 식별합니다.

### 3. POS 로그 확인

POS [로그 화면](/support/logs)에도 문제를 식별하는 데 도움이 되는 관련 오류 정보 및 오류 코드가 있을 수 있습니다.
