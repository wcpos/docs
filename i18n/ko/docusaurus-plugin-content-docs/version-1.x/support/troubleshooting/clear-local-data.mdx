WCPOS는 POS가 빠르게 작동하고 오프라인에서도 기능할 수 있도록 제품, 주문 및 고객의 로컬 사본을 저장합니다. 때때로 이 로컬 데이터가 WooCommerce 스토어와 동기화되지 않을 수 있습니다. 모든 로컬 데이터를 초기화하면 로컬 데이터베이스가 제거되고 서버에서 새로운 동기화가 시작됩니다.

:::warning
이것은 최후의 수단입니다. 로컬 데이터를 초기화하면 장치에 저장된 모든 것이 삭제되며 전체 재동기화가 강제됩니다. 스토어의 크기에 따라 시간이 걸릴 수 있습니다. 일반 [동기화](/products/sync)와 같은 다른 옵션으로 문제가 해결되지 않았을 때만 사용하세요.
:::

## 사용 시기 {#when-to-use-it}

로컬 데이터를 초기화해야 할 때는 다음과 같습니다:

- **동기화 문제** — 제품, 주문 또는 고객이 제대로 업데이트되지 않을 때
- **플러그인 업데이트 후** — 새 데이터 구조와의 호환성을 보장하기 위해
- **오래된 데이터** — 가격, 재고 수준 또는 기타 정보가 구식으로 보일 때

## 초기화 되는 항목 {#what-gets-cleared}

| 초기화됨 | 초기화되지 않음 |
|---------|-------------|
| 로컬 제품 데이터베이스 | 로그인 세션 |
| 로컬 주문 데이터베이스 | 서버 측 데이터 |
| 로컬 고객 데이터베이스 | WooCommerce 설정 |

로그인은 해제되지 않습니다. WooCommerce 스토어 데이터에는 영향을 미치지 않으며 — 장치의 로컬 사본만 제거됩니다. 초기화 후 새 동기화가 자동으로 시작됩니다.

## 로컬 데이터 초기화 방법 {#how-to-clear-local-data}

### 웹 {#web}

1. 오른쪽 상단의 **유저 메뉴**를 클릭하여 메뉴를 엽니다.
2. **모든 로컬 데이터 초기화**를 클릭합니다.
3. 로컬 데이터베이스가 삭제되고 새로운 동기화가 자동으로 시작됩니다.

### 데스크탑 {#desktop}

1. 상단 메뉴 바에서 **WCPOS** 애플리케이션 메뉴를 엽니다.
2. **고급** 위에 마우스를 올립니다.
3. **앱 데이터 초기화**를 클릭합니다.
4. 로컬 데이터베이스가 삭제되고 새로운 동기화가 자동으로 시작됩니다.
