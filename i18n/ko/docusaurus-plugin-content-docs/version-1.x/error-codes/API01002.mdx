# API01002: 연결 거부 {#api01002-connection-refused}

## 의미 {#what-this-means}

서버가 연결 시도를 적극적으로 거부했습니다. 이는 타임아웃과 다릅니다. 서버가 응답했지만 "아니요"라고 말했습니다.

## 일반적인 원인 {#common-causes}

- **서버 다운** — 웹 서버(Apache/Nginx)가 실행되고 있지 않을 수 있습니다.
- **잘못된 포트** — 잘못된 포트로 연결을 시도하고 있습니다.
- **방화벽 차단** — 방화벽이 연결을 차단하고 있습니다.
- **IP 제한** — 귀하의 IP 주소가 차단되었을 수 있습니다.

## 해결 방법 {#how-to-fix}

### 1. 서버가 실행되고 있는지 확인 {#1-verify-the-server-is-running}

브라우저에서 직접 WordPress 사이트에 접근해 보세요. 로딩되지 않으면 호스팅 제공업체에 문의하세요.

### 2. URL 구성 확인 {#2-check-the-url-configuration}

정확한 URL을 사용하고 있는지 확인하세요:
- `http://`와 `https://`를 확인하세요
- 도메인 이름에 오타가 있는지 체크하세요
- 비표준 포트를 사용하는 경우 올바른 포트를 확인하세요

### 3. 방화벽 설정 확인 {#3-check-firewall-settings}

서버에 접근할 수 있는 경우:
- 포트 80(HTTP) 또는 443(HTTPS)이 열려 있는지 확인하세요
- 귀하의 IP가 화이트리스트에 있는지 체크하세요
- API 접근을 차단할 수 있는 보안 플러그인을 검토하세요

### 4. 호스팅 제공업체에 문의 {#4-contact-your-hosting-provider}

그들은 다음을 확인할 수 있습니다:
- 서버 상태 및 로그
- 방화벽 구성
- 귀하의 IP가 차단되었는지 여부
