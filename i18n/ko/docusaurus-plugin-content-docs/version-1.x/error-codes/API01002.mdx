# API01002: 연결 거부됨

## 의미

서버가 연결 시도를 적극적으로 거부했습니다. 이는 타임아웃과는 다르며, 서버가 응답했지만 "아니오"라고 말한 상황입니다.

## 일반적인 원인

- **서버 다운** — 웹 서버(Apache/Nginx)가 실행 중이 아닐 수 있음
- **잘못된 포트** — 잘못된 포트로 연결 시도
- **방화벽 차단** — 방화벽이 연결을 차단
- **IP 제한** — IP 주소가 차단되었을 수 있음

## 해결 방법

### 1. 서버 실행 확인

브라우저에서 WordPress 사이트에 직접 접속해 보세요. 로딩되지 않으면 호스팅 제공업체에 문의하세요.

### 2. URL 구성 확인

올바른 URL 사용 여부 확인:
- `http://` vs `https://` 확인
- 도메인 이름 오타 점검
- 비표준 포트 사용 시 올바른 포트 확인

### 3. 방화벽 설정 확인

서버에 접근할 수 있다면:
- 포트 80 (HTTP) 또는 443 (HTTPS) 개방 여부 확인
- IP가 화이트리스트에 있는지 확인
- API 액세스를 차단할 수 있는 보안 플러그인 검토

### 4. 호스팅 제공업체에 문의

다음 사항을 확인할 수 있습니다:
- 서버 상태 및 로그
- 방화벽 구성
- IP가 차단되었는지 여부
