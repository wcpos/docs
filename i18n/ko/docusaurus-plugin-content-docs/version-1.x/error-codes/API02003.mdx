# API02003: 토큰 무효화

## 이 의미

서버에서 인증 토큰을 인식할 수 없습니다. 이는 만료된 토큰과는 다르며, 이 토큰은 처음부터 유효하지 않거나 취소되었습니다.

## 일반적인 원인

- **토큰 취소** — 관리자가 세션을 취소함
- **토큰 손상** — 저장 또는 전송 중 데이터 손상
- **서버 재설정** — 서버 비밀 키 변경으로 모든 토큰 무효화
- **잘못된 서버** — 다른 WordPress 설치의 토큰

## 서버 오류 매핑

다음 서버 응답으로 이 오류 코드가 트리거됩니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT 인증 플러그인 |

## 해결 방법

### 1. 다시 로그인

세션을 완전히 지우고 새로 인증:
1. 완전히 로그아웃
2. POS 앱 닫고 다시 열기
3. 자격 증명으로 로그인

### 2. 애플리케이션 데이터 지우기

로그아웃해도 해결되지 않으면:
- POS 앱 캐시/데이터 지우기
- 웹: 사이트의 브라우저 쿠키 및 로컬 스토리지 지우기
- 데스크톱: "데이터 지우기" 옵션 확인

### 3. 서버 구성 확인

모든 사용자에게 영향을 미치는 경우:
- WordPress 솔트 변경 여부 확인
- JWT 또는 인증 플러그인 설정 검증
- 최근 서버 변경 사항 검토

### 4. 올바른 서버인지 확인

여러 WordPress 설치가 있는 경우, POS가 올바른 WordPress 사이트에 연결되었는지 확인하세요.
