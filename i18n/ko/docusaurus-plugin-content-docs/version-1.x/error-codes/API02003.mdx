# API02003: 토큰 유효하지 않음

## 이것이 의미하는 바

사용 중인 인증 토큰이 서버에서 인식되지 않습니다. 이는 만료된 토큰과는 다른 것으로, 이 토큰은 처음부터 유효하지 않거나 취소되었습니다.

## 일반적인 원인

- **토큰 취소됨** — 관리자가 세션을 취소했습니다.
- **토큰 손상됨** — 저장 또는 전송 중 데이터 손상
- **서버 재설정** — 서버 비밀 키가 변경되어 모든 토큰이 무효화됨
- **잘못된 서버** — 다른 WordPress 설치에서 가져온 토큰

## 서버 오류 매핑

이 오류 코드는 서버가 다음을 반환할 때 발생합니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `jwt_auth_invalid_token` | JWT 인증 플러그인 |

## 해결 방법

### 1. 다시 로그인

세션을 지우고 새로 인증하세요:
1. 완전히 로그아웃합니다.
2. POS를 닫고 다시 엽니다.
3. 자격 증명으로 로그인합니다.

### 2. 애플리케이션 데이터 지우기

로그아웃이 효과가 없을 경우:
- POS 앱 캐시/데이터를 지웁니다.
- 웹에서는: 사이트에 대한 브라우저 쿠키 및 로컬 저장소를 지웁니다.
- 데스크탑에서는: "데이터 지우기" 옵션이 있는지 앱 설정을 확인합니다.

### 3. 서버 구성 확인

문제가 모든 사용자에게 영향을 미치는 경우:
- WordPress 소금이 변경되었는지 확인합니다.
- JWT 또는 인증 플러그인 설정을 검증합니다.
- 최근 서버 변경 사항을 검토합니다.

### 4. 올바른 서버에 연결되어 있는지 확인

POS가 올바른 WordPress 사이트에 연결되도록 설정되어 있는지 확인하세요. 특히 여러 설치가 있는 경우 더욱 중요합니다.
