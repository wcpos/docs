# API02001: 잘못된 자격 증명 {#api02001-invalid-credentials}

## 의미 {#what-this-means}

입력한 사용자 이름 또는 비밀번호가 잘못되었습니다. 자격 증명이 유효한 계정과 일치하지 않아 서버가 로그인 시도를 거부했습니다.

## 일반적인 원인 {#common-causes}

- **사용자 이름 또는 비밀번호의 오타** — Caps lock 또는 추가 공백 확인
- **잘못된 계정** — 다른 사이트의 자격 증명 사용
- **비밀번호가 최근에 변경됨** — 비밀번호가 업데이트되었지만 POS는 이전 것을 사용
- **계정 삭제됨** — 사용자 계정이 더 이상 존재하지 않음

## 서버 오류 매핑 {#server-error-mapping}

서버가 다음을 반환할 때 이 오류 코드가 발생합니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT 인증 플러그인 |

## 해결 방법 {#how-to-fix}

### 1. 자격 증명 확인 {#1-verify-your-credentials}

- 사용자 이름(일반적으로 이메일 또는 WordPress 사용자 이름)을 다시 확인
- 비밀번호가 정확한지 확인
- WordPress 관리자로 직접 로그인하여 확인

### 2. 비밀번호 재설정 {#2-reset-your-password}

비밀번호를 잊어버린 경우:
1. WordPress 로그인 페이지로 이동
2. "비밀번호를 잊으셨나요?" 클릭
3. 재설정 링크를 받기 위해 이메일 입력
4. 비밀번호 업데이트
5. 새로운 비밀번호로 POS에 로그인

### 3. 사용자 계정 확인 {#3-check-the-user-account}

WordPress 관리에서:
- 사용자 → 모든 사용자로 이동
- 계정이 존재하는지 확인
- 적절한 POS 접근 권한이 있는지 확인

### 4. 저장된 자격 증명 지우기 {#4-clear-saved-credentials}

POS에서:
- 완전히 로그아웃
- 저장된/캐시된 로그인 정보를 지우기
- 새롭게 자격 증명을 다시 입력
