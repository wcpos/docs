# API02001: 잘못된 자격 증명

## 의미

입력한 사용자 이름이나 비밀번호가 올바르지 않습니다. 서버가 일치하는 유효한 계정이 없어 로그인 시도를 거부했습니다.

## 일반적인 원인

- **사용자 이름 또는 비밀번호 입력 오류** — 대소문자 또는 추가 공백 확인
- **잘못된 계정** — 다른 사이트의 자격 증명 사용
- **최근 비밀번호 변경** — 비밀번호가 업데이트되었지만 POS에는 이전 비밀번호가 저장됨
- **계정 삭제** — 사용자 계정이 더 이상 존재하지 않음

## 서버 오류 매핑

다음 서버 반환 시 이 오류 코드가 트리거됩니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `woocommerce_rest_authentication_error` | WooCommerce REST API |
| `jwt_auth_failed` | JWT 인증 플러그인 |

## 해결 방법

### 1. 자격 증명 확인

- 사용자 이름(일반적으로 이메일 또는 WordPress 사용자 이름) 다시 확인
- 비밀번호가 맞는지 검증
- WordPress 관리자에 직접 로그인하여 확인

### 2. 비밀번호 재설정

비밀번호를 잊은 경우:
1. WordPress 로그인 페이지로 이동
2. "비밀번호를 잊으셨나요?" 클릭
3. 재설정 링크를 받을 이메일 입력
4. 비밀번호 업데이트
5. 새 비밀번호로 POS에 로그인

### 3. 사용자 계정 확인

WordPress 관리자에서:
- 사용자 → 모든 사용자로 이동
- 계정 존재 여부 확인
- 적절한 POS 액세스 권한 있는지 확인

### 4. 저장된 자격 증명 지우기

POS에서:
- 완전히 로그아웃
- 저장되거나 캐시된 로그인 정보 지우기
- 자격 증명 다시 입력
