# API01005: SSL 인증서 오류 {#api01005-ssl-certificate-error}

## 의미 {#what-this-means}

당신의 웹사이트의 SSL/TLS 인증서에 문제가 있습니다. 이 인증서는 연결을 안전하게 만들어줍니다(HTTPS). POS는 데이터를 보호하기 위해 유효하지 않은 인증서가 있는 사이트에 연결되지 않습니다.

## 일반적인 원인 {#common-causes}

- **인증서 만료** — SSL 인증서는 주기적으로 갱신해야 합니다.
- **자체 서명된 인증서** — 신뢰할 수 있는 기관에서 발급하지 않음
- **잘못된 도메인** — 인증서가 도메인 이름과 일치하지 않음
- **불완전한 인증서 체인** — 중간 인증서가 누락됨
- **혼합된 콘텐츠** — 일부 리소스가 HTTPS 대신 HTTP로 로드됨

## 해결 방법 {#how-to-fix}

### 1. 인증서 상태 확인 {#1-check-certificate-status}

브라우저에서 사이트를 방문하고 자물쇠 아이콘을 클릭하여 인증서 세부정보를 확인합니다. 다음을 확인하세요:
- 만료 날짜
- 발급 대상 (귀하의 도메인과 일치해야 함)
- 발급 기관 (인증된 기관이어야 함)

### 2. 만료된 인증서 갱신 {#2-renew-expired-certificate}

만료된 경우:
- 대부분의 호스팅 제공업체는 무료 Let's Encrypt 인증서를 제공합니다.
- 호스팅 제공업체에 연락하여 갱신합니다.
- 자동 갱신이 활성화되어 있는지 확인합니다.

### 3. 인증서 불일치 수정 {#3-fix-certificate-mismatch}

인증서가 다음을 포함하는지 확인하십시오:
- 귀하의 정확한 도메인 (`yourstore.com`)
- 사용되는 경우 WWW 변형 (`www.yourstore.com`)
- 와일드카드 인증서 (`*.yourstore.com`) 고려

### 4. 누락된 중간 인증서 설치 {#4-install-missing-intermediate-certificates}

SSL 검사 도구(예: SSL Labs)를 사용하여 누락된 인증서를 식별합니다. 호스팅 제공업체가 설치를 도와줄 수 있습니다.

### 5. HTTPS 강제 적용 {#5-force-https}

WordPress에서 다음을 확인하세요:
- 사이트 URL이 `https://`를 사용하고 있는지
- WooCommerce 설정에서 SSL 강제가 활성화되어 있는지
