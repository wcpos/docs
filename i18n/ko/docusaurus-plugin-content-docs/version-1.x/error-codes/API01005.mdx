# API01005: SSL 인증서 오류

## 의미

웹사이트의 SSL/TLS 인증서에 문제가 있습니다. 이 인증서는 연결을 안전하게 만드는(HTTPS) 역할을 합니다. POS는 데이터 보호를 위해 유효하지 않은 인증서가 있는 사이트에 연결하지 않습니다.

## 일반적인 원인

- **만료된 인증서** — SSL 인증서는 주기적으로 갱신해야 합니다
- **자체 서명 인증서** — 신뢰할 수 있는 기관에서 발급하지 않음
- **잘못된 도메인** — 인증서가 도메인 이름과 일치하지 않음
- **불완전한 인증서 체인** — 중간 인증서 누락
- **혼합 콘텐츠** — 일부 리소스가 HTTPS 대신 HTTP로 로드됨

## 해결 방법

### 1. 인증서 상태 확인

브라우저에서 사이트를 방문하고 자물쇠 아이콘을 클릭하여 인증서 세부 정보를 확인하세요. 다음 항목을 확인하세요:
- 만료 날짜
- 발급 대상(도메인과 일치해야 함)
- 발급 기관(인정된 기관이어야 함)

### 2. 만료된 인증서 갱신

만료된 경우:
- 대부분의 호스팅 제공업체에서 무료 Let's Encrypt 인증서 제공
- 호스팅 제공업체에 갱신 요청
- 자동 갱신이 활성화되었는지 확인

### 3. 인증서 불일치 해결

다음 사항을 확인하세요:
- 정확한 도메인 (`yourstore.com`)
- WWW 변형 (`www.yourstore.com`) (사용하는 경우)
- 와일드카드 인증서 고려 (`*.yourstore.com`)

### 4. 누락된 중간 인증서 설치

SSL 검사 도구(SSL Labs 등)를 사용하여 누락된 인증서 확인. 호스팅 제공업체가 설치를 도와줄 수 있습니다.

### 5. HTTPS 강제 적용

WordPress에서 다음 사항을 확인하세요:
- 사이트 URL이 `https://` 사용
- WooCommerce 설정에서 SSL 강제 적용 활성화
