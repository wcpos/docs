# API05003: WCPOS 플러그인 구버전

## 의미

서버에 설치된 WCPOS 플러그인이 구버전이며 POS 애플리케이션 버전과 호환되지 않습니다. 플러그인을 업데이트해야 합니다.

## 일반적인 원인

- **플러그인이 업데이트되지 않음** — 자동 업데이트가 비활성화되어 있을 수 있음
- **업데이트 실패** — 이전 업데이트 시도가 실패했음
- **버전 불일치** — POS 앱은 업데이트되었지만 플러그인은 업데이트되지 않음
- **스테이징/개발 사이트** — 구버전에서 테스트 중임

## 해결 방법

### 1. 플러그인 업데이트하기

WordPress 관리자에서:
1. 플러그인 → 설치된 플러그인으로 이동
2. "WooCommerce POS" 찾기
3. 업데이트가 가능하면 "지금 업데이트"를 클릭

### 2. 자동 업데이트 활성화하기

향후 불일치를 방지하기 위해:
1. 플러그인 → 설치된 플러그인으로 이동
2. "WooCommerce POS" 찾기
3. "자동 업데이트 활성화"를 클릭

### 3. 수동 업데이트

자동 업데이트가 실패하면:
1. [WordPress.org](https://wordpress.org/plugins/woocommerce-pos/)에서 최신 버전을 다운로드
2. 현재 플러그인을 비활성화
3. 이전 플러그인 파일 삭제
4. 새 버전을 업로드하고 활성화

### 4. 업데이트 오류 확인하기

업데이트가 실패하는 경우:
- wp-content/plugins의 파일 권한 확인
- 디스크 공간이 있는지 확인
- PHP 메모리 문제 확인
- 오류 로그에서 세부사항 확인

### 5. POS 앱 다운그레이드 (임시)

플러그인을 즉시 업데이트할 수 없는 경우:
- POS 앱의 이전 버전 사용
- 이는 임시 해결책임
- 곧 플러그인을 업데이트할 계획

## 버전 호환성

항상 두 구성 요소를 업데이트 상태로 유지하십시오:
- POS 애플리케이션
- WCPOS WordPress 플러그인

업데이트 시 파괴적인 변경 사항에 대한 릴리스 노트를 확인하십시오.
