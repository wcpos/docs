# API06003: 잘못된 사이트 구성

## 의미

사이트 구성이 유효하지 않거나 불완전합니다. 잘못된 URL, 인증 설정 또는 기타 구성 문제와 관련될 수 있습니다.

## 일반적인 원인

- **부분 구성** — 일부 설정이 누락됨
- **불일치 설정** — 구성이 사이트와 일치하지 않음
- **손상된 구성** — 설정이 손상됨
- **사이트 변경** — POS 구성을 업데이트하지 않고 사이트 수정됨

## 해결 방법

### 1. 모든 설정 검토

전체 구성 확인:
- 사이트 URL이 정확한지 확인
- 인증이 제대로 구성되었는지 확인
- 추가 설정이 정확한지 확인

### 2. 처음부터 다시 구성

구성이 손상된 경우:
1. 저장된 모든 설정 지우기
2. 설정 프로세스 다시 시작
3. 새로운 구성 입력

### 3. 사이트 요구 사항 확인

WordPress 사이트가 요구 사항을 충족하는지 확인:
- WordPress가 설치되고 접근 가능
- WooCommerce가 설치되고 활성화됨
- WCPOS 플러그인이 설치되고 활성화됨
- 고유 링크가 활성화됨 (일반 링크 아님)

### 4. 사이트 접근 테스트

브라우저에서 다음 URL이 작동하는지 확인:
- `https://yoursite.com/` — 메인 사이트
- `https://yoursite.com/wp-json/` — REST API
- `https://yoursite.com/wp-json/wcpos/v1/` — WCPOS API

### 5. WordPress 설정 확인

WordPress 관리자에서:
1. 설정 → 일반 — URL 확인
2. 설정 → 고유 링크 — 일반 링크 아닌지 확인
3. WooCommerce → 상태 — 문제 확인

## 구성 요구 사항

유효한 구성에 필요한 사항:
- **사이트 URL** — 프로토콜이 포함된 전체 URL
- **인증** — 유효한 자격 증명 또는 API 키
- **권한** — 사용자가 POS 접근 권한 보유
- **플러그인** — 필수 플러그인 활성화
