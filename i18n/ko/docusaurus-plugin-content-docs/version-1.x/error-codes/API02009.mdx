# API02009: 리프레시 토큰 만료

## 의미

리프레시 토큰의 유효 기간이 만료되어 더 이상 새로운 액세스 토큰을 얻는 데 사용할 수 없습니다. 액세스 토큰(빠르게 만료됨)과 달리 리프레시 토큰은 보통 며칠 또는 몇 주 동안 유효하지만 결국 만료됩니다.

## 일반적인 원인

- **장기간 비활성 상태** — POS를 오랫동안 사용하지 않음
- **짧은 토큰 유효 기간** — 짧은 리프레시 토큰 만료로 서버가 구성됨
- **시간 차이** — 장치 또는 서버 시간이 크게 어긋남

## 해결 방법

### 1. 다시 로그인

리프레시 토큰이 만료되면 다시 인증해야 합니다:
1. POS에서 로그아웃
2. 자격 증명을 입력
3. 새로운 리프레시 토큰이 발급됩니다

### 2. POS를 정기적으로 사용

앞으로 이를 피하기 위해:
- 토큰을 신선하게 유지하기 위해 POS를 정기적으로 사용
- 활성 상태일 때 자동으로 토큰이 새로 고쳐짐
- 장기간 비활성 상태일 경우 다시 로그인해야 함

### 3. 시간 설정 확인

모든 시스템에서 정확한 시간을 보장합니다:
- 장치에서 자동 시간 동기화를 활성화
- 서버 시간이 올바른지 확인
- 시간 차이는 조기 만료를 초래할 수 있음

### 4. 토큰 유효 기간 조정(고급)

토큰이 필요에 비해 너무 빨리 만료되는 경우, 서버 관리자는 인증 설정에서 리프레시 토큰 유효 기간을 조정할 수 있습니다.
