# API02009: 새로 고침 토큰 만료됨

## 의미

새로 고침 토큰의 수명이 만료되어 더 이상 새 접근 토큰을 얻는 데 사용할 수 없습니다. 빠르게 만료되는 접근 토큰과 달리, 새로 고침 토큰은 일반적으로 며칠 또는 몇 주 동안 지속되지만 결국 만료됩니다.

## 일반적인 원인

- **장기 비활성** — POS를 오랫동안 사용하지 않음
- **짧은 토큰 수명** — 서버가 짧은 새로 고침 토큰 만료 시간으로 구성됨
- **시계 오차** — 장치 또는 서버 시간이 크게 다름

## 해결 방법

### 1. 다시 로그인

새로 고침 토큰이 만료되면 다시 인증해야 합니다:
1. POS에서 로그아웃
2. 자격 증명 입력
3. 새 새로 고침 토큰이 발급됨

### 2. POS 정기적으로 사용

향후 이를 방지하려면:
- POS를 정기적으로 사용하여 토큰 유효성 유지
- 활성 상태일 때 토큰이 자동으로 갱신됨
- 장기 비활성 기간에는 다시 로그인 필요

### 3. 시간 설정 확인

모든 시스템의 정확한 시간 확인:
- 장치에서 자동 시간 동기화 활성화
- 서버 시간 정확성 확인
- 시간 차이가 조기 만료 원인일 수 있음

### 4. 토큰 수명 조정 (고급)

필요에 따라 토큰이 너무 빨리 만료되는 경우, 서버 관리자가 인증 설정에서 새로 고침 토큰 수명을 조정할 수 있습니다.
