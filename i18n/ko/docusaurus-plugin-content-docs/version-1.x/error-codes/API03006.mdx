# API03006: 지원되지 않는 메서드

## 의미

HTTP 메서드(GET, POST, PUT, DELETE 등)가 이 엔드포인트에 대해 지원되지 않습니다. 서버가 이 URL에 대한 해당 유형의 요청을 수락하지 않습니다.

## 일반적인 원인

- **버전 불일치** — POS와 플러그인 버전이 호환되지 않음
- **엔드포인트 제거** — API 엔드포인트가 더 이상 사용되지 않음
- **서버 구성** — 웹 서버가 특정 HTTP 메서드 차단
- **플러그인 충돌** — 다른 플러그인이 REST API 동작 수정

## 해결 방법

### 1. 모든 구성 요소 업데이트

호환성 확인:
- WCPOS WordPress 플러그인 업데이트
- POS 애플리케이션 업데이트
- 주요 변경 사항에 대한 릴리스 노트 확인

### 2. 서버 구성 확인

일부 서버는 특정 HTTP 메서드를 차단할 수 있습니다:
- PUT 및 DELETE 메서드가 허용되는지 확인
- `.htaccess`의 메서드 제한 확인
- Nginx 구성 검토

### 3. REST API 접근 확인

WordPress REST API 테스트:
1. 브라우저에서 `https://yoursite.com/wp-json/` 방문
2. JSON 데이터가 반환되어야 함
3. 반환되지 않으면 REST API가 비활성화되거나 차단됨

### 4. 플러그인 충돌 확인

일시적으로 다른 플러그인 비활성화:
- 보안 플러그인이 메서드를 차단할 수 있음
- 다른 REST API 플러그인이 충돌 유발 가능
- 문제를 찾기 위해 하나씩 다시 활성화

### 5. 호스팅 제한 검토

일부 호스트는 HTTP 메서드를 제한합니다:
- 호스팅 지원팀에 연락
- 모든 표준 메서드 활성화 요청
- 제한이 너무 엄격한 경우 호스트 변경 고려
