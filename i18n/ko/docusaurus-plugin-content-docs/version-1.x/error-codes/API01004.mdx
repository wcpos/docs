# API01004: DNS 확인 실패

## 의미

POS에서 웹사이트 도메인 이름(예: `yourstore.com`)을 IP 주소로 변환할 수 없었습니다. DNS(도메인 이름 시스템)는 인터넷의 전화번호부와 같아서 해당 항목을 찾을 수 없으면 연결을 할 수 없습니다.

## 흔한 원인

- **도메인 이름 오타** — URL에 철자 오류가 있을 수 있음
- **DNS 서버 문제** — DNS 서버가 응답하지 않음
- **도메인 만료** — 도메인 등록이 만료되었을 수 있음
- **DNS 전파** — 최근 DNS 변경 사항이 아직 전파되지 않음
- **로컬 DNS 캐시** — 기기에 오래된 DNS 정보가 캐시됨

## 해결 방법

### 1. 도메인 이름 확인

URL의 철자를 다시 확인하세요. 브라우저에서 사이트에 접속해 보세요.

### 2. 도메인 상태 확인

WHOIS 조회로 다음을 확인하세요:
- 도메인이 여전히 등록되어 있는지
- 만료되지 않았는지
- DNS 레코드가 구성되어 있는지

### 3. DNS 캐시 지우기

**기기에서:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. 대체 DNS 사용

다음과 같은 공용 DNS 서버로 전환:
- Google: `8.8.8.8` 및 `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. DNS 전파 대기

최근 DNS 설정을 변경했다면 완전한 전파를 위해 24-48시간 대기하세요.
