# API01004: DNS 해결 실패 {#api01004-dns-resolution-failed}

## 의미 {#what-this-means}

POS는 웹사이트의 도메인 이름(예: `yourstore.com`)을 IP 주소로 변환할 수 없었습니다. DNS(도메인 이름 시스템)는 인터넷의 전화번호부와 같습니다. 목록을 찾을 수 없다면 연결을 만들 수 없습니다.

## 일반적인 원인 {#common-causes}

- **도메인 이름의 오타** — URL에 철자 오류가 있을 수 있습니다.
- **DNS 서버 문제** — DNS 서버가 응답하지 않습니다.
- **도메인 만료** — 도메인 등록이 만료되었을 수 있습니다.
- **DNS 전파** — 최근 DNS 변경 사항이 아직 전파되지 않았습니다.
- **로컬 DNS 캐시** — 장치에 저장된 구식 DNS 정보입니다.

## 해결 방법 {#how-to-fix}

### 1. 도메인 이름 확인 {#1-verify-the-domain-name}

오타가 있는지 URL을 다시 확인하십시오. 브라우저에서 사이트에 접근해 보십시오.

### 2. 도메인 상태 확인 {#2-check-domain-status}

WHOIS 조회를 사용하여 다음을 확인하십시오:
- 도메인이 여전히 등록되어 있는지
- 만료되지 않았는지
- DNS 레코드가 구성되어 있는지

### 3. DNS 캐시 지우기 {#3-clear-dns-cache}

**귀하의 장치에서:**

macOS:
```bash
sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
```

Windows:
```bash
ipconfig /flushdns
```

### 4. 대체 DNS 사용 {#4-try-alternative-dns}

다음과 같은 공개 DNS 서버로 전환하십시오:
- Google: `8.8.8.8` 및 `8.8.4.4`
- Cloudflare: `1.1.1.1`

### 5. DNS 전파 대기 {#5-wait-for-dns-propagation}

최근에 DNS 설정을 변경한 경우, 완전한 전파를 위해 24-48시간 기다리십시오.
