# API03005: 요율 제한 초과

## 의미

짧은 시간 내에 서버에 너무 많은 요청을 했습니다. 요율 제한은 서버가 과부하되지 않도록 보호합니다.

## 일반적인 원인

- **빠른 동기화 작업** — 여러 장치가 동시에 동기화
- **강력한 새로 고침** — 데이터를 너무 자주 새로 고침
- **보안 플러그인** — 플러그인이 요청 제한을 시행하고 있음
- **호스팅 제한** — 호스팅 요금제가 API 요율 제한을 가지고 있음

## 서버 오류 매핑

서버가 반환할 때 이 오류 코드가 발생합니다:

| 서버 코드 | 출처 |
|-------------|--------|
| HTTP 429 | 모든 서버 응답 (대체) |

## 해결 방법

### 1. 기다리기 및 재시도

요율 제한은 일시적입니다:
- 1-5분 기다리기
- POS가 자동으로 재시도합니다
- 반복적으로 수동으로 새로 고침을 피하십시오

### 2. 동시 작업 줄이기

여러 POS 단말기를 사용하는 경우:
- 장치 간 초기 동기화를 분산하십시오
- 모든 장치를 한 번에 새로 고치는 것을 피하십시오
- 하나가 완료된 후 다른 작업을 시작하십시오

### 3. 보안 플러그인 확인

Wordfence와 같은 플러그인이 빠른 요청을 차단할 수 있습니다:
- POS 애플리케이션을 허용 목록에 추가하십시오
- POS 장치 IP를 허용 목록에 추가하십시오
- 요율 제한 기준을 조정하십시오

### 4. 호스팅 제한 검토

일부 호스팅 제공자는 API 요청을 제한합니다:
- 호스팅 계획 세부 정보를 확인하십시오
- 제한이 너무 낮으면 업그레이드를 고려하십시오
- 제한을 증가시키는 것에 대해 지원팀에 문의하십시오

### 5. 동기화 빈도 최적화

POS 설정에서:
- 자동 새로 고침 간격을 조정하십시오
- 적절할 때 수동 동기화를 사용하십시오
- 불필요한 데이터 새로 고침을 피하십시오
