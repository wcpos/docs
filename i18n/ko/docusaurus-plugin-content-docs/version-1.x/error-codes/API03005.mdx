# API03005: 요청 제한 초과

## 의미

짧은 시간 내에 서버에 너무 많은 요청을 했습니다. 요청 제한은 서버가 과부하에 걸리지 않도록 보호합니다.

## 일반적인 원인

- **빠른 동기화 작업** — 여러 장치가 동시에 동기화
- **적극적인 새로고침** — 데이터를 너무 자주 새로고침
- **보안 플러그인** — 플러그인이 요청 한도를 강제 적용
- **호스팅 한도** — 호스팅 요금제에 API 요청 한도가 있음

## 서버 오류 매핑

다음과 같이 서버가 응답하면 이 오류 코드가 발생합니다:

| 서버 코드 | 출처 |
|-------------|--------|
| HTTP 429 | 모든 서버 응답 (대체) |

## 해결 방법

### 1. 기다렸다가 다시 시도

요청 제한은 일시적입니다:
- 1-5분 기다리기
- POS가 자동으로 다시 시도할 것입니다
- 수동으로 반복적으로 새로고침 하지 마세요

### 2. 동시 작업 줄이기

여러 POS 단말기를 사용하는 경우:
- 장치 간 초기 동기화를 시차 두고 수행
- 모든 장치를 동시에 새로고침 하지 않기
- 하나가 완료될 때까지 다른 하나를 시작하지 않기

### 3. 보안 플러그인 확인

Wordfence와 같은 플러그인이 급격한 요청을 차단할 수 있습니다:
- POS 애플리케이션을 화이트리스트에 추가
- POS 장치의 IP를 화이트리스트에 추가
- 요청 제한 기준 조정

### 4. 호스팅 한도 검토

일부 호스트는 API 요청을 제한합니다:
- 호스팅 요금제 세부정보 확인
- 한도가 너무 낮으면 업그레이드 고려
- 한도 증가에 대해 지원팀에 문의

### 5. 동기화 빈도 최적화

POS 설정에서:
- 자동 새로고침 간격 조정
- 적절할 때 수동 동기화 사용
- 불필요한 데이터 새로고침 피하기
