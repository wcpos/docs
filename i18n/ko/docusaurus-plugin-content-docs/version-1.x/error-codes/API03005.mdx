# API03005: 요청 속도 제한 초과

## 이 오류의 의미

서버에 짧은 시간 내에 너무 많은 요청을 보냈습니다. 요청 속도 제한은 서버가 과부하되는 것을 방지합니다.

## 일반적인 원인

- **빠른 동기화 작업** — 여러 기기가 동시에 동기화
- **지나친 새로고침** — 데이터를 너무 자주 새로고침
- **보안 플러그인** — 플러그인이 요청 제한을 적용
- **호스팅 제한** — 호스팅 플랜의 API 요청 속도 제한

## 서버 오류 매핑

이 오류 코드는 서버가 다음을 반환할 때 트리거됩니다:

| 서버 코드 | 출처 |
|-------------|--------|
| HTTP 429 | 모든 서버 응답 (대체) |

## 해결 방법

### 1. 대기 후 재시도

속도 제한은 일시적입니다:
- 1-5분 대기
- POS가 자동으로 재시도
- 반복해서 수동으로 새로고침하지 않기

### 2. 동시 작업 줄이기

여러 POS 터미널 사용 시:
- 기기 간 초기 동기화 시차 두기
- 모든 기기를 한 번에 새로고침하지 않기
- 한 기기 완료 후 다른 기기 시작

### 3. 보안 플러그인 확인

Wordfence 같은 플러그인은 빠른 요청을 차단할 수 있습니다:
- POS 애플리케이션 허용 목록에 추가
- POS 기기 IP 허용 목록에 추가
- 속도 제한 임계값 조정

### 4. 호스팅 제한 검토

일부 호스트는 API 요청을 제한합니다:
- 호스팅 플랜 세부 정보 확인
- 제한이 너무 낮으면 업그레이드 고려
- 제한 증가에 대해 지원팀에 문의

### 5. 동기화 빈도 최적화

POS 설정에서:
- 자동 새로고침 간격 조정
- 적절할 때 수동 동기화 사용
- 불필요한 데이터 새로고침 방지
