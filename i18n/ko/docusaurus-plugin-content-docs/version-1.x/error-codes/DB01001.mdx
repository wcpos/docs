# DB01001: 연결 실패 {#db01001-connection-failed}

## 이것이 의미하는 바 {#what-this-means}

POS가 로컬 데이터베이스에 연결할 수 없습니다. WCPOS는 빠른 액세스와 오프라인 기능을 위해 제품, 고객 및 기타 데이터를 저장하기 위해 로컬 데이터베이스를 사용합니다.

## 일반적인 원인 {#common-causes}

- **저장소 가득 참** — 장치에 남은 저장 공간이 없음
- **데이터베이스 손상** — 로컬 데이터베이스 파일이 손상됨
- **권한 문제** — 앱에 저장 권한이 없음
- **브라우저 저장소 비활성화** — IndexedDB가 비활성화됨 (웹 버전)

## 해결 방법 {#how-to-fix}

### 1. 저장 공간 확인 {#1-check-storage-space}

장치에 사용 가능한 저장 공간이 있는지 확인:
- 저장 공간이 가득 찼다면 공간을 확보하십시오
- 사용하지 않는 앱이나 파일 삭제
- 사용 가능한 디스크 공간 확인

### 2. 앱 데이터 지우기 {#2-clear-app-data}

로컬 데이터베이스를 재설정:
- **데스크톱 앱:** 설정에서 "데이터 지우기"를 찾습니다
- **웹 브라우저:** POS 도메인의 사이트 데이터를 지웁니다
- **참고:** 모든 데이터를 다시 동기화해야 합니다

### 3. 브라우저 설정 확인 (웹) {#3-check-browser-settings-web}

웹 버전을 사용하는 경우:
1. IndexedDB가 활성화되어 있는지 확인
2. 개인/시크릿 모드를 사용하지 마십시오 (저장소 제한)
3. 사이트가 데이터를 저장하도록 허용

### 4. 권한 확인 {#4-check-permissions}

데스크톱에서:
- 앱이 데이터 디렉토리에 쓰기 권한이 있는지 확인
- 안티바이러스가 데이터베이스 접근을 차단하고 있는지 확인

### 5. 앱 재설치 {#5-reinstall-the-app}

다른 해결책이 실패한 경우:
1. POS 앱을 제거하십시오
2. 장치를 재시작하십시오
3. 앱을 재설치하십시오
4. 로그인하고 데이터를 동기화하십시오

## 데이터 복구 {#data-recovery}

저장되지 않은 데이터가 있는 경우:
- 앱에 동기화되지 않은 변경 사항이 있을 수 있습니다
- 데이터 복구에 도움이 필요하면 지원팀에 문의하십시오
- 대부분의 경우, 데이터는 서버에서 다시 동기화될 수 있습니다
