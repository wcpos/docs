# DB01003: 트랜잭션 실패

## 의미

데이터베이스 트랜잭션을 완료할 수 없었습니다. 트랜잭션은 여러 작업을 그룹화하며 — 일부가 실패하면 데이터 무결성을 유지하기 위해 모든 것을 롤백합니다.

## 일반적인 원인

- **동시 접근** — 동일한 데이터를 수정하려는 다중 작업
- **저장소 꽉 참** — 새 데이터를 쓸 공간 없음
- **데이터베이스 잠김** — 다른 프로세스가 데이터베이스를 잠그고 있음
- **전원 중단** — 작업이 예기치 않게 중단됨

## 해결 방법

### 1. 작업 다시 시도

문제가 일시적일 수 있습니다:
- 잠시 기다리기
- 작업 다시 시도
- 빠른 반복 시도 피하기

### 2. 충돌 확인

여러 장치나 탭이 열려 있는 경우:
- 한 번에 한 인스턴스 사용
- 중복 브라우저 탭 닫기
- 다중 장치 사용 조정

### 3. 저장소 공간 확인

데이터를 위한 공간 확보:
- 사용 가능한 디스크 공간 확인
- 필요한 경우 공간 확보
- 브라우저 캐시 지우기 (웹 버전)

### 4. 애플리케이션 다시 시작

데이터베이스 상태 초기화:
1. POS 완전히 종료
2. 몇 초 대기
3. 애플리케이션 다시 열기

### 5. 지우고 다시 동기화

트랜잭션이 지속적으로 실패하는 경우:
1. 로컬 데이터 지우기
2. 다시 로그인
3. 서버에서 새로 동기화

## 트랜잭션 안전성

WCPOS는 다음을 보장하기 위해 트랜잭션을 사용합니다:
- 데이터 일관성
- 완전한 작업 (모두 또는 없음)
- 부분 업데이트 방지

트랜잭션이 실패해도 데이터는 일관성을 유지합니다.
