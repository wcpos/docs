# API03003: 잘못된 매개변수 값

## 의미

요청의 값 중 하나가 유효하지 않습니다. 데이터 형식, 유형 또는 값이 서버의 예상과 일치하지 않습니다.

## 일반적인 원인

- **잘못된 데이터 유형** — 숫자가 예상되는 곳에 텍스트(또는 그 반대)
- **범위 초과** — 허용된 한도를 초과하는 값
- **잘못된 형식** — 이메일, 전화번호 또는 기타 형식화된 필드가 잘못됨
- **잘못된 참조** — 존재하지 않는 ID 참조

## 서버 오류 매핑

이 오류 코드는 다음 서버 반환 시 트리거됩니다:

| 서버 코드 | 소스 |
|-------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## 해결 방법

### 1. 입력 값 확인

제출하는 데이터 검토:
- **가격** — 유효한 숫자여야 함
- **수량** — 양의 정수여야 함
- **이메일** — 유효한 이메일 형식이어야 함
- **ID** — 기존 레코드를 참조해야 함

### 2. 특정 필드 오류 확인

오류가 어떤 필드가 유효하지 않은지 나타낼 수 있음:
- 오류 메시지 세부 정보 확인
- 특정 필드 수정
- 작업 다시 시도

### 3. 참조된 데이터 존재 확인

제품, 고객 또는 다른 레코드 참조 시:
- WooCommerce에 해당 항목이 존재하는지 확인
- 최근에 삭제되었는지 확인
- 로컬 레코드 새로 고침을 위해 데이터 동기화

### 4. 필드 제약 조건 확인

WooCommerce에는 다음과 같은 제약 조건이 있을 수 있음:
- 최대/최소값
- 필요한 형식
- 선택 필드의 허용된 옵션

### 5. 지우고 다시 입력

때때로 데이터가 손상될 수 있음:
- 문제가 있는 필드 지우기
- 처음부터 값 다시 입력
- 다른 소스에서 복사-붙여넣기 방지
