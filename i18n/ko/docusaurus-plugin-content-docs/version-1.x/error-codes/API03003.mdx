# API03003: 잘못된 매개변수 값 {#api03003-invalid-parameter-value}

## 의미 {#what-this-means}

요청의 값 중 하나가 유효하지 않습니다. 데이터 형식, 유형 또는 값이 서버에서 예상하는 것과 일치하지 않습니다.

## 일반적인 원인 {#common-causes}

- **잘못된 데이터 유형** — 숫자가 필요한 곳에 텍스트(또는 그 반대)
- **범위를 벗어남** — 값이 허용된 한계를 초과함
- **잘못된 형식** — 이메일, 전화 또는 기타 형식이 잘못됨
- **잘못된 참조** — 존재하지 않는 ID를 참조함

## 서버 오류 매핑 {#server-error-mapping}

이 오류 코드는 서버가 다음을 반환할 때 발생합니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `rest_invalid_param` | WordPress REST API |
| `woocommerce_rest_invalid_id` | WooCommerce REST API |

## 해결 방법 {#how-to-fix}

### 1. 입력 값 확인 {#1-check-input-values}

제출하는 데이터를 검토하세요:
- **가격** — 유효한 숫자여야 함
- **수량** — 양의 정수여야 함
- **이메일** — 유효한 이메일 형식이어야 함
- **ID** — 기존 기록을 참조해야 함

### 2. 특정 필드 오류 확인 {#2-look-for-specific-field-errors}

오류가 어떤 필드가 잘못되었는지를 나타낼 수 있습니다:
- 오류 메시지 세부정보 확인
- 특정 필드 수정
- 작업 다시 시도

### 3. 참조된 데이터 존재 여부 확인 {#3-verify-referenced-data-exists}

제품, 고객 또는 기타 기록을 참조하는 경우:
- WooCommerce에서 항목이 존재하는지 확인
- 최근에 삭제되었는지 확인
- 데이터를 동기화하여 로컬 기록 새로 고침

### 4. 필드 제한 확인 {#4-check-field-constraints}

WooCommerce에는 제한 사항이 있을 수 있습니다:
- 최대/최소 값
- 필수 형식
- 선택 필드에 대한 허용 옵션

### 5. 지우고 재입력하기 {#5-clear-and-re-enter}

때때로 데이터가 손상될 수 있습니다:
- 문제 있는 필드 지우기
- 값을 처음부터 다시 입력
- 다른 소스에서 복사-붙여넣기 피하기
