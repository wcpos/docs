# API02008: 리프레시 토큰 무효화

## 의미

POS에 저장된 리프레시 토큰이 서버에서 인식되지 않습니다. 리프레시 토큰은 다시 로그인하지 않고도 새로운 접근 토큰을 얻는 데 사용됩니다.

## 일반적인 원인

- **토큰 취소** — 관리자가 세션을 무효화함
- **서버 비밀 변경** — WordPress 보안 키가 업데이트됨
- **데이터 손상** — 저장소에서 토큰이 손상됨
- **여러 번 로그인** — 다른 곳에서 로그인하여 이 토큰이 무효화됨

## 해결 방법

### 1. 다시 로그인

리프레시 토큰이 무효화되었으므로 다시 인증해야 합니다:
1. POS에서 로그아웃
2. 사용자 이름과 비밀번호 입력
3. 새로운 유효한 토큰 생성

### 2. 취소된 세션 확인

관리자가 세션을 취소한 경우:
- 이는 정상적인 보안 절차입니다
- 간단히 다시 로그인
- 새 세션이 정상적으로 작동합니다

### 3. 서버 변경 확인

모든 사용자에게 영향을 미치는 경우:
- WordPress 솔트/키가 변경되었을 수 있음
- JWT 플러그인 설정이 수정되었을 수 있음
- 서버 관리자에게 문의

### 4. 저장된 데이터 지우기

지속적인 문제가 있는 경우:
- POS 애플리케이션 데이터 지우기
- 캐시된 자격 증명 제거
- 깨끗한 로그인으로 시작
