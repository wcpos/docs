# API02008: 리프레시 토큰이 유효하지 않음 {#api02008-refresh-token-invalid}

## 의미 {#what-this-means}

POS에 저장된 리프레시 토큰이 서버에 의해 인식되지 않습니다. 리프레시 토큰은 다시 로그인하지 않고도 새 액세스 토큰을 얻는 데 사용됩니다.

## 일반적인 원인 {#common-causes}

- **토큰 철회** — 관리자가 세션을 무효화했습니다
- **서버 비밀 변경** — WordPress 보안 키가 업데이트되었습니다
- **데이터 손상** — 저장소에서 토큰이 손상되었습니다
- **여러 로그인** — 다른 곳에서 로그인하면 이 토큰이 무효화될 수 있습니다

## 해결 방법 {#how-to-fix}

### 1. 다시 로그인하기 {#1-log-in-again}

리프레시 토큰이 유효하지 않으므로 다시 인증해야 합니다:
1. POS에서 로그아웃합니다
2. 사용자 이름과 비밀번호를 입력합니다
3. 이는 새로운 유효한 토큰을 생성합니다

### 2. 철회된 세션 확인 {#2-check-for-revoked-sessions}

관리자가 세션을 철회한 경우:
- 이는 일반적인 보안 관행입니다
- 단순히 다시 로그인합니다
- 새 세션이 정상적으로 작동합니다

### 3. 서버 변경 사항 확인 {#3-check-server-changes}

모든 사용자에게 영향을 미치는 경우:
- WordPress 소금/키가 변경되었을 수 있습니다
- JWT 플러그인 설정이 수정되었을 수 있습니다
- 서버 관리자에게 문의하세요

### 4. 저장된 데이터 지우기 {#4-clear-stored-data}

문제가 계속되는 경우:
- POS 애플리케이션 데이터를 지웁니다
- 캐시된 자격 증명을 제거합니다
- 깨끗한 로그인으로 시작합니다
