# DB02003: 제약 위반

## 의미

저장하려는 데이터가 데이터베이스 규칙을 위반합니다. 제약 조건은 저장할 수 있는 데이터에 대한 규칙을 강제하여 데이터 무결성을 보장합니다.

## 일반적인 원인

- **누락된 필수 데이터** — 필수 필드가 비어 있음
- **잘못된 관계** — 존재하지 않는 레코드 참조
- **데이터 유형 불일치** — 필드에 맞지 않는 데이터 유형
- **범위를 벗어난 값** — 허용된 한도를 초과하는 숫자

## 해결 방법

### 1. 필수 필드 확인

모든 필수 데이터가 제공되었는지 확인:
- 고객 정보 (필요한 경우)
- 상품 세부 정보
- 주문 라인 항목

### 2. 참조 확인

관계와 관련된 오류가 있는 경우:
- 참조된 상품이 존재하는지 확인
- 고객 ID 유효성 검사
- 카테고리 할당 확인

### 3. 데이터 값 검토

잘못된 값 확인:
- 허용되지 않는 음수 수량
- 한도를 초과하는 가격
- 잘못된 상태 값

### 4. 최신 데이터 동기화

참조된 데이터가 동기화되지 않았을 수 있음:
- 서버에서 데이터 새로고침
- 동기화 완료 대기
- 작업 다시 시도

### 5. 지우고 다시 시도

데이터가 손상된 경우:
1. 문제가 있는 양식 지우기
2. 데이터 다시 입력
3. 다시 제출

## 일반적인 제약 조건 예시

- **수량은 양수여야 함** — 0 또는 음수 항목 추가 불가
- **가격은 숫자여야 함** — 가격 필드에 텍스트 허용 안 됨
- **고객은 존재해야 함** — 존재하지 않는 고객에게 주문 할당 불가
