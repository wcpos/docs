# API03007: 요청 대기열이 가득 찼습니다 {#api03007-request-queue-full}

## 의미 {#what-this-means}

POS는 전송 대기 중인 요청이 너무 많습니다. 내부 대기열이 가득 차서 일부 요청이 완료될 때까지 더 이상의 요청을 받을 수 없습니다.

## 일반적인 원인 {#common-causes}

- **느린 네트워크** — 느린 연결로 인해 요청이 쌓임
- **서버 비응답** — 서버가 응답하는 데 시간이 너무 오래 걸림
- **빠른 동작** — 처리할 수 있는 것보다 빠르게 많은 작업 수행
- **동기화 과부하** — 대기열을 차단하는 대규모 동기화 작업

## 해결 방법 {#how-to-fix}

### 1. 대기열이 비워질 때까지 기다리기 {#1-wait-for-queue-to-clear}

대기 중인 요청이 완료될 시간을 줍니다:
- 30-60초 기다리기
- 동기화 지표가 완료될 때까지 지켜보기
- 대기열이 비워질 때까지 추가 작업을 피함

### 2. 네트워크 연결 확인 {#2-check-network-connection}

느린 연결은 백업을 유발합니다:
- 인터넷 속도 테스트
- 더 빠른 연결 시도
- WiFi 라우터와 가까운 곳으로 이동

### 3. 서버 응답 시간 확인 {#3-check-server-response-time}

서버가 느리면:
- 브라우저에서 웹사이트에 접근해 보기
- 웹사이트가 느리면 호스팅 업체에 연락
- 서버 부하가 줄어들 때까지 기다리기

### 4. POS 재시작 {#4-restart-the-pos}

대기열이 멈춘 경우:
1. POS 애플리케이션을 닫습니다
2. 잠시 기다립니다
3. POS를 다시 엽니다
4. 대기 중인 요청은 다시 시작해야 할 수 있습니다

### 5. 빠른 동작 피하기 {#5-avoid-rapid-actions}

동기화 중이거나 느린 시기에는:
- 새로운 작업을 시작하기 전에 하나의 작업이 완료될 때까지 기다리기
- 버튼을 반복적으로 클릭하지 않기
- 네트워크 작업에 인내심을 가지기

## 오프라인 동작 {#offline-behaviour}

대기열이 오프라인 조건으로 가득 찬 경우:
- POS는 온라인으로 복귀할 때까지 대기 중인 요청을 보류합니다
- 캐시된 제품 및 고객을 여전히 탐색할 수 있습니다
- 새로운 주문을 시작하고 장바구니에 항목을 추가할 수 있습니다
- 주문을 완료하려면 연결이 복구되어야 합니다
