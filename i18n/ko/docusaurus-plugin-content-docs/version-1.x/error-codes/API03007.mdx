# API03007: 요청 대기열 가득 참

## 의미

POS에 보내기를 기다리는 보류 중인 요청이 너무 많습니다. 내부 대기열이 가득 차서 일부 요청이 완료될 때까지 더 이상 요청을 받을 수 없습니다.

## 흔한 원인

- **느린 네트워크** — 느린 연결로 인해 요청이 밀림
- **서버 응답 없음** — 서버가 응답하는 데 너무 오래 걸림
- **빠른 작업** — 처리할 수 있는 것보다 더 빠르게 여러 작업 수행
- **동기화 과부하** — 대규모 동기화 작업이 대기열을 막음

## 해결 방법

### 1. 대기열 완료 대기

보류 중인 요청이 완료될 때까지 기다리기:
- 30-60초 대기
- 동기화 표시기가 완료될 때까지 관찰
- 완전히 해결될 때까지 추가 작업 피하기

### 2. 네트워크 연결 확인

느린 연결이 원인일 수 있음:
- 인터넷 속도 테스트
- 더 빠른 연결 시도
- WiFi 라우터에 더 가까이 이동

### 3. 서버 응답 시간 확인

서버가 느리다면:
- 브라우저에서 사이트 접속 테스트
- 사이트가 느리면 호스팅 업체에 문의
- 서버 부하가 감소할 때까지 대기

### 4. POS 다시 시작

대기열이 멈췄다면:
1. POS 애플리케이션 종료
2. 잠시 대기
3. POS 다시 열기
4. 보류 중인 요청 다시 시작 필요할 수 있음

### 5. 빠른 작업 피하기

동기화 또는 느린 기간 동안:
- 한 작업이 완료될 때까지 다른 작업 시작하지 않기
- 버튼 반복 클릭 금지
- 네트워크 작업에 인내심 갖기

## 오프라인 동작

오프라인 상태로 대기열이 가득 차면:
- POS는 온라인 상태로 돌아올 때까지 보류 중인 요청 유지
- 캐시된 제품 및 고객 검색 가능
- 새 주문 시작 및 장바구니에 항목 추가 가능
- 주문 완료는 연결 복원 필요
