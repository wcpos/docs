# SY01003: 권한 거부 {#sy01003-permission-denied}

## 의미 {#what-this-means}

POS 애플리케이션이 필요한 리소스에 접근할 권한이 없습니다. 이는 파일, 폴더 또는 시스템 기능일 수 있습니다.

## 일반적인 원인 {#common-causes}

- **파일 권한** — 필요한 파일을 읽거나 쓸 수 없음
- **앱 권한** — 애플리케이션에 필요한 권한이 없음
- **보호된 폴더** — 제한된 위치에 접근하려고 함
- **보안 소프트웨어** — 안티바이러스가 접근을 차단함

## 해결 방법 {#how-to-fix}

### 1. 관리자 권한으로 실행 (Windows) {#1-run-as-administrator-windows}

앱을 마우스 오른쪽 버튼으로 클릭하고 초기 설정이 필요한 경우 "관리자 권한으로 실행"을 선택합니다.

### 2. 애플리케이션 권한 확인 {#2-check-application-permissions}

**macOS:**
- 시스템 환경설정 → 보안 및 개인 정보 → 개인 정보
- 애플리케이션에 필요한 권한이 있는지 확인합니다.

**Windows:**
- 설정 → 개인 정보
- 앱 권한을 검토합니다.

### 3. 폴더 권한 확인 {#3-check-folder-permissions}

애플리케이션이 데이터 폴더에 접근할 수 있는지 확인합니다:
- 사용자가 앱 데이터 디렉토리를 소유해야 함
- 쓰기 권한이 활성화되어 있어야 함
- 폴더에 읽기 전용 플래그가 없어야 함

### 4. 안티바이러스 설정 검토 {#4-review-antivirus-settings}

보안 소프트웨어가 접근을 차단할 수 있습니다:
- 차단된 내역이 있는지 안티바이러스 로그를 확인합니다.
- POS 애플리케이션을 허용 목록에 추가합니다.
- 앱 데이터 폴더를 화이트리스트에 등록합니다.

### 5. 애플리케이션 재설치 {#5-reinstall-the-application}

권한이 손상된 경우:
1. POS 애플리케이션을 제거합니다.
2. 장치를 재시작합니다.
3. 표준 위치에 재설치합니다.
4. 일반(비관리자) 권한으로 실행합니다.

## 브라우저 권한 (웹 버전) {#browser-permissions-web-version}

웹 버전을 사용하는 경우:
- 사이트에 대한 로컬 저장소 허용
- 사이트에 대한 쿠키 활성화
- 지나치게 제한적인 브라우저 설정 사용 금지
