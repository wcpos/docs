# SY01003: 권한 거부됨

## 의미

POS 애플리케이션이 필요한 리소스에 접근할 권한이 없습니다. 이는 파일, 폴더 또는 시스템 기능에 대한 접근이 거부되었음을 의미합니다.

## 일반적인 원인

- **파일 권한** — 필요한 파일을 읽거나 쓸 수 없음
- **앱 권한** — 애플리케이션에 필요한 권한 부족
- **보호된 폴더** — 제한된 위치에 접근 시도
- **보안 소프트웨어** — 바이러스 백신이 접근 차단

## 해결 방법

### 1. 관리자로 실행 (Windows)

초기 설정에 필요한 경우 앱을 마우스 오른쪽 버튼으로 클릭하고 "관리자 권한으로 실행"을 선택하세요.

### 2. 애플리케이션 권한 확인

**macOS:**
- 시스템 환경설정 → 보안 및 개인정보 → 개인정보 보호
- 앱에 필요한 권한이 있는지 확인

**Windows:**
- 설정 → 개인정보
- 앱 권한 검토

### 3. 폴더 권한 확인

앱이 데이터 폴더에 접근할 수 있는지 확인:
- 사용자가 앱 데이터 디렉토리 소유
- 쓰기 권한 활성화
- 폴더에 읽기 전용 플래그 없음

### 4. 바이러스 백신 설정 검토

보안 소프트웨어가 접근을 차단할 수 있습니다:
- 바이러스 백신 로그에서 차단 내용 확인
- POS 앱을 허용 목록에 추가
- 앱 데이터 폴더 허용

### 5. 애플리케이션 재설치

권한이 손상된 경우:
1. POS 애플리케이션 제거
2. 기기 재시작
3. 표준 위치에 재설치
4. 일반 (관리자 아님) 권한으로 실행

## 브라우저 권한 (웹 버전)

웹 버전 사용 시:
- 사이트의 로컬 저장소 허용
- 사이트의 쿠키 활성화
- 과도하게 제한적인 브라우저 설정 사용 금지
