# API04005: JSON 복구 시도됨

## 이 메시지의 의미

서버가 일부 잘못된 JSON 콘텐츠가 포함된 응답을 보냈지만, POS는 유효한 데이터를 추출하기 위해 복구를 시도했습니다. 이는 심각한 오류가 아닌 정보성 알림입니다.

## 무슨 일이 발생했는지

POS가 다음을 감지했습니다:
1. 응답에 JSON 앞뒤로 추가 콘텐츠가 포함됨
2. 핵심 JSON 데이터는 여전히 식별 가능
3. 유효한 JSON 부분을 추출하여 복구 시도

## 일반적인 원인

- **PHP 알림 출력** — JSON과 혼합된 PHP 경고
- **디버그 출력** — 개발 디버깅이 활성화된 상태
- **플러그인 알림** — 다른 플러그인의 알림 출력
- **공백 문제** — JSON 주변의 추가 공백

## 해결 방법

### 1. PHP 오류 표시 비활성화

`wp-config.php`에서:
```php
define('WP_DEBUG_DISPLAY', false);
ini_set('display_errors', 0);
```

### 2. 대신 오류 로깅 활성화

디버깅을 위해 오류 로깅:
```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
```

### 3. 플러그인 디버그 모드 확인

일부 플러그인에는 추가 콘텐츠를 출력하는 디버그 모드가 있습니다:
- 플러그인 설정 검토
- 디버그/개발 모드 비활성화
- 자세한 로깅 옵션 확인

### 4. 최근 변경 사항 검토

최근에 이 문제가 시작된 경우:
- 서버에서 어떤 변경이 있었나요?
- 플러그인이 업데이트되었나요?
- PHP 설정이 수정되었나요?

## 심각성

POS가 이 문제에서 복구되었지만, 이는 수정해야 할 구성 문제를 나타냅니다. 복구 프로세스는:
- 항상 작동하지 않을 수 있음
- 처리 오버헤드 추가
- 다른 문제를 숨길 수 있음

안정적인 작동을 보장하려면 근본 원인을 해결하세요.
