# API04002: 예상치 못한 응답 코드

## 의미

서버가 요청에 대해 예상치 못한 HTTP 상태 코드를 반환했습니다. 일반적인 코드로는 500(서버 오류), 403(금지됨), 404(찾을 수 없음) 등이 있습니다.

## 일반적인 원인

- **500 내부 서버 오류** — PHP 충돌 또는 오류 발생
- **403 금지됨** — 보안 설정에 의해 접근 거부
- **404 찾을 수 없음** — 엔드포인트가 존재하지 않음
- **502/503/504** — 서버 게이트웨이 또는 가용성 문제

## 해결 방법

### 500 오류 (서버 오류)의 경우

1. PHP 오류 로그 확인
2. PHP 메모리 제한 늘리기
3. 플러그인 충돌 확인
4. `wp-content/debug.log` 확인

### 403 오류 (금지됨)의 경우

1. 보안 플러그인 설정 확인 (Wordfence, Sucuri 등)
2. POS 또는 IP 화이트리스트 추가
3. `.htaccess`의 차단 규칙 확인
4. ModSecurity가 요청을 차단하지 않는지 확인

### 404 오류 (찾을 수 없음)의 경우

1. WCPOS 플러그인이 활성화되어 있는지 확인
2. WordPress 고유 링크 초기화 (설정 → 고유 링크 → 저장)
3. REST API가 활성화되어 있는지 확인
4. URL 구성 확인

### 502/503/504 오류 (게이트웨이 문제)의 경우

1. 호스팅 제공업체에 연락
2. 서버 복구 대기
3. 사이트가 과부하 상태인지 확인
4. 서버 실행 상태 확인

### 일반적인 문제 해결

1. 사이트에 직접 접속 시도
2. 호스팅 제어판에서 문제 확인
3. 서버 접근 로그 검토
4. 필요시 호스팅 지원팀에 문의
