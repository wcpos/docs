# API04002: 예상치 못한 응답 코드 {#api04002-unexpected-response-code}

## 의미 {#what-this-means}

서버가 이 요청에 대해 예상하지 못한 HTTP 상태 코드를 반환했습니다. 일반적인 코드는 500(서버 오류), 403(금지됨), 404(찾을 수 없음) 등이 있습니다.

## 일반적인 원인 {#common-causes}

- **500 내부 서버 오류** — PHP가 충돌했거나 오류가 발생했습니다.
- **403 금지됨** — 보안 설정에 의해 접근이 거부됨
- **404 찾을 수 없음** — 끝점이 존재하지 않음
- **502/503/504** — 서버 게이트웨이 또는 가용성 문제

## 해결 방법 {#how-to-fix}

### 500 오류(서버 오류)의 경우 {#for-500-errors-server-error}

1. PHP 오류 로그 확인
2. PHP 메모리 제한 늘리기
3. 플러그인 충돌 확인
4. `wp-content/debug.log` 확인

### 403 오류(금지됨)의 경우 {#for-403-errors-forbidden}

1. 보안 플러그인 설정 확인 (Wordfence, Sucuri 등)
2. POS 또는 본인의 IP 화이트리스트 추가
3. 차단 규칙이 있는지 `.htaccess` 확인
4. ModSecurity가 요청을 차단하고 있지 않은지 확인

### 404 오류(찾을 수 없음)의 경우 {#for-404-errors-not-found}

1. WCPOS 플러그인이 활성화되었는지 확인
2. 워드프레스 퍼머링크 초기화(설정 → 퍼머링크 → 저장)
3. REST API가 활성화되어 있는지 확인
4. URL 구성 검증

### 502/503/504 오류(게이트웨이 문제)의 경우 {#for-502503504-errors-gateway-issues}

1. 호스팅 제공업체에 문의
2. 서버가 복구될 때까지 기다리기
3. 사이트에 과부하가 걸려 있는지 확인
4. 서버가 실행 중인지 확인

### 일반적인 문제 해결 {#general-troubleshooting}

1. 사이트에 직접 접근해 보기
2. 호스팅 제어판에서 문제 확인
3. 서버 접근 로그 검토
4. 필요시 호스팅 지원에 문의
