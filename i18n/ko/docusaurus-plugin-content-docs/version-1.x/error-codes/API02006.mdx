# API02006: API 키 유효하지 않음

## 의미

사용 중인 WooCommerce REST API 키가 유효하지 않습니다. API 키는 서버 간 인증에 사용되며, 제공된 키가 WooCommerce의 어떤 유효한 키와도 일치하지 않습니다.

## 일반적인 원인

- **키 삭제됨** — WooCommerce에서 API 키가 제거됨
- **키 오타** — 키가 잘못 입력됨
- **잘못된 키 쌍** — 소비자 키와 시크릿이 일치하지 않음
- **다른 사이트의 키** — 다른 설치에서 생성된 키 사용

## 해결 방법

### 1. API 키 확인

WordPress 관리자 → WooCommerce → 설정 → 고급 → REST API:
1. API 키가 존재하는지 확인
2. 취소되지 않았는지 확인
3. 권한 (읽기/쓰기/읽기-쓰기) 확인

### 2. 새 API 키 생성

키가 없거나 유효하지 않은 경우:
1. WooCommerce → 설정 → 고급 → REST API로 이동
2. "키 추가" 클릭
3. 설명 입력 (예: "WCPOS")
4. 사용자 선택
5. "읽기/쓰기" 권한 선택
6. "API 키 생성" 클릭
7. **소비자 키와 소비자 시크릿 복사** (한 번만 표시됨!)

### 3. POS 구성 업데이트

POS에 새 API 키 입력:
- 소비자 키 (`ck_`로 시작)
- 소비자 시크릿 (`cs_`로 시작)

### 4. 키 권한 확인

충분한 권한이 있는지 확인:
- **읽기** — 데이터 보기만
- **쓰기** — 데이터 수정만
- **읽기/쓰기** — 완전한 접근 (POS에 권장)
