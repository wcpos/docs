# API02006: API 키 유효하지 않음 {#api02006-api-key-invalid}

## 의미 {#what-this-means}

사용 중인 WooCommerce REST API 키가 유효하지 않습니다. API 키는 서버 간 인증에 사용되며, 제공된 키가 WooCommerce의 유효한 키와 일치하지 않습니다.

## 일반적인 원인 {#common-causes}

- **키 삭제** — API 키가 WooCommerce에서 제거됨
- **키 입력 오류** — 키가 잘못 입력됨
- **잘못된 키 쌍** — 소비자 키와 비밀번호가 일치하지 않음
- **다른 사이트의 키** — 다른 설치를 위해 생성된 키 사용

## 수정 방법 {#how-to-fix}

### 1. API 키 확인 {#1-verify-the-api-key}

WordPress 관리 → WooCommerce → 설정 → 고급 → REST API:
1. API 키가 존재하는지 확인
2. 키가 취소되지 않았는지 확인
3. 권한(읽기/쓰기/읽기-쓰기) 메모

### 2. 새 API 키 생성 {#2-generate-new-api-keys}

키가 없거나 유효하지 않은 경우:
1. WooCommerce → 설정 → 고급 → REST API로 이동
2. "키 추가" 클릭
3. 설명 입력(예: "WCPOS")
4. 사용자 선택
5. "읽기/쓰기" 권한 선택
6. "API 키 생성" 클릭
7. **소비자 키와 소비자 비밀번호를 모두 복사** (한 번만 표시됨!)

### 3. POS 구성 업데이트 {#3-update-pos-configuration}

POS에 새 API 키 입력:
- 소비자 키 (시작은 `ck_`)
- 소비자 비밀번호 (시작은 `cs_`)

### 4. 키 권한 확인 {#4-check-key-permissions}

키가 충분한 권한을 가지고 있는지 확인:
- **읽기** — 데이터만 보기
- **쓰기** — 데이터만 수정
- **읽기/쓰기** — 전체 접근 (POS에 권장)
