# API02004: 사용자 인증되지 않음 {#api02004-user-not-authorized}

## 의미 {#what-this-means}

로그인되어 있지만, 사용자 계정이 요청된 작업을 수행할 권한이 없습니다. 이는 인증 문제(당신이 누구인가)보다 권한 문제(당신이 할 수 있는 것)입니다.

## 일반적인 원인 {#common-causes}

- **사용자 역할 제한** — WordPress 역할이 POS 접근을 포함하지 않음
- **POS 접근 비활성화** — 계정에 POS 권한이 부여되지 않음
- **기능 제한** — 특정 기능이 특정 역할로 제한됨
- **상점 제한** — 이 특정 상점에 접근할 수 없음

## 서버 오류 매핑 {#server-error-mapping}

서버가 다음을 반환할 때 이 오류 코드가 발생합니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `rest_cannot_view` | WordPress REST API |
| `woocommerce_rest_cannot_view` | WooCommerce REST API |

## 수정 방법 {#how-to-fix}

### 1. 사용자 역할 확인 {#1-check-user-role}

WordPress 관리자 → 사용자:
1. 사용자 계정을 찾습니다
2. 역할을 확인합니다 (예: 상점 관리자, 관리자)
3. 역할에 WooCommerce 기능이 포함되어 있는지 확인합니다

### 2. POS 접근 활성화 {#2-enable-pos-access}

WordPress 관리자 → WooCommerce → POS → 접근:
1. 사용자 또는 역할을 찾습니다
2. POS 접근 권한을 활성화합니다
3. 변경 사항을 저장합니다

### 3. 추가 권한 요청 {#3-request-additional-permissions}

상점 관리자에게 연락하여:
- 역할에 POS 접근 권한 부여
- 적절한 권한이 있는 역할 할당
- 필요한 특정 기능 활성화

### 4. 상점 할당 확인 {#4-check-store-assignment}

여러 상점을 사용하는 경우:
- 올바른 상점에 할당되었는지 확인합니다
- 상점별 권한을 확인합니다

## 필요한 권한 {#required-permissions}

다른 작업은 다른 기능이 필요합니다:
- **제품 보기**: 제품에 대한 읽기 접근
- **주문 생성**: 주문 생성/편집 기능
- **고객 관리**: 고객 관리 기능
- **보고서 접근**: 보고서 보기 기능
