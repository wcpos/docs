# API02004: 사용자 권한 없음

## 의미

로그인되어 있지만, 사용자 계정에 요청한 작업을 수행할 권한이 없습니다. 이는 인증(누구인지) 문제가 아니라 권한 부여(무엇을 할 수 있는지) 문제입니다.

## 일반적인 원인

- **사용자 역할 제한** — WordPress 역할에 POS 접근 권한이 없음
- **POS 접근 비활성화** — 계정에 POS 권한이 부여되지 않음
- **기능 제한** — 특정 기능이 특정 역할로 제한됨
- **스토어 제한** — 해당 스토어에 접근 권한이 없음

## 서버 오류 매핑

이 오류 코드는 서버가 다음을 반환할 때 발생합니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `rest_cannot_view` | WordPress REST API |
| `woocommerce_rest_cannot_view` | WooCommerce REST API |

## 해결 방법

### 1. 사용자 역할 확인

WordPress 관리자 → 사용자:
1. 사용자 계정 찾기
2. 역할 확인 (예: 상점 관리자, 관리자)
3. 역할에 WooCommerce 권한 포함 여부 확인

### 2. POS 접근 활성화

WordPress 관리자 → WooCommerce → POS → 접근:
1. 사용자 또는 역할 찾기
2. POS 접근 권한 활성화
3. 변경 사항 저장

### 3. 추가 권한 요청

스토어 관리자에게 다음을 요청:
- 해당 역할에 POS 접근 권한 부여
- 적절한 권한이 있는 역할 할당
- 필요한 특정 기능 활성화

### 4. 스토어 할당 확인

여러 스토어 사용 시:
- 올바른 스토어에 할당되었는지 확인
- 스토어별 권한 확인

## 필요한 권한

다른 작업에는 다른 권한 필요:
- **제품 보기**: 제품 읽기 권한
- **주문 생성**: 주문 생성/편집 권한
- **고객 관리**: 고객 관리 권한
- **보고서 접근**: 보고서 보기 권한
