# API04006: 리소스를 찾을 수 없음

## 의미

서버가 요청된 리소스를 찾을 수 없습니다. 이는 일반적으로 HTTP 404 상태 코드에 해당하며, 접근하려는 제품, 주문, 고객 또는 기타 리소스가 존재하지 않음을 나타냅니다.

## 일반적인 원인

- **레코드 삭제** — 서버에서 리소스가 삭제됨
- **잘못된 ID** — 부정확하거나 오래된 ID 사용
- **동기화 문제** — 로컬 데이터가 더 이상 존재하지 않는 리소스를 참조
- **URL 잘못 구성** — API 엔드포인트가 올바르지 않음
- **고정 링크 문제** — WordPress 고정 링크를 새로 고침해야 함

## 서버 오류 매핑

이 오류 코드는 서버가 다음을 반환할 때 트리거됩니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `rest_no_route` | WordPress REST API |
| HTTP 404 | 모든 서버 응답 |

## 해결 방법

### 1. 리소스 존재 확인

WooCommerce 사이트에 리소스가 여전히 존재하는지 확인:
- WordPress 관리자에 로그인
- 관련 섹션(제품, 주문, 고객)으로 이동
- ID 또는 이름으로 항목 검색

### 2. 로컬 데이터 새로 고침

리소스가 서버 측에서 삭제된 경우:
1. POS 설정 열기
2. 관련 데이터 섹션으로 이동
3. 데이터를 업데이트하기 위해 동기화/새로 고침 트리거
4. 삭제된 항목이 로컬에서 제거되어야 함

### 3. WordPress 고정 링크 확인

여러 리소스를 찾을 수 없는 경우:
1. WordPress 관리자에서 **설정 → 고정 링크**로 이동
2. **저장** 클릭 (변경 없이도)
3. 이렇게 하면 고정 링크 구조가 새로 고쳐집니다

### 4. API 경로 확인

REST API 직접 테스트:
```
https://yoursite.com/wp-json/wc/v3/products
```
404를 반환하면 서버 구성에 문제가 있을 수 있습니다.

### 5. 플러그인 충돌 확인

REST API 경로가 누락된 경우:
1. WooCommerce가 활성화되어 있는지 확인
2. WCPOS 플러그인이 활성화되어 있는지 확인
3. 테스트를 위해 일시적으로 다른 플러그인 비활성화
