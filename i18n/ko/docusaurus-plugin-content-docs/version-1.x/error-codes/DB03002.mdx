# DB03002: 잘못된 데이터 유형

## 의미

데이터 유형이 데이터베이스가 예상하는 유형과 일치하지 않습니다. 예를 들어, 숫자가 예상되는 곳에 텍스트가 제공되었습니다.

## 일반적인 원인

- **사용자 입력 오류** — 숫자 필드에 텍스트 입력
- **가져오기 문제** — 잘못된 형식의 데이터 가져오기
- **데이터 손상** — 전송 중 값 손상
- **플러그인 충돌** — 다른 플러그인이 데이터 유형 수정

## 해결 방법

### 1. 입력 확인

입력하는 데이터 검토:
- **가격** — 숫자여야 함 (예: `19.99`)
- **수량** — 정수여야 함 (예: `5`)
- **ID** — 숫자여야 함

### 2. 지우고 다시 입력

데이터가 손상된 경우:
- 필드 지우기
- 수동으로 값 다시 입력
- 외부 소스에서 복사 붙여넣기 피하기

### 3. 원본 데이터 확인

WooCommerce에서 동기화하는 경우:
- WordPress 관리자에서 데이터 확인
- 잘못 서식 지정된 필드 찾기
- 원본에서 데이터 수정

### 4. 데이터 다시 동기화

서버에서 새 데이터 가져오기:
- 로컬 캐시 지우기
- 데이터 다시 동기화
- 문제 해결 여부 확인

### 5. 플러그인 충돌 확인

다른 WooCommerce 플러그인 사용 시:
- 예상치 못한 방식으로 데이터 수정 가능
- 테스트를 위해 일시적으로 비활성화
- 비호환성 보고

## 일반적인 예시

| 필드 | 예상 | 잘못된 예 |
|-------|----------|---------|
| 가격 | `19.99` | `$19.99` |
| 수량 | `5` | `five` |
| SKU | `ABC123` | — (모든 형식 가능) |
| 상품 ID | `42` | `product-42` |
