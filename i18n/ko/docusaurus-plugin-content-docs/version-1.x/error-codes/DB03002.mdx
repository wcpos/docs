# DB03002: 잘못된 데이터 유형 {#db03002-invalid-data-type}

## 의미 {#what-this-means}

데이터 유형이 데이터베이스에서 기대하는 것과 일치하지 않습니다. 예를 들어, 숫자가 기대되는 곳에 텍스트가 제공되었습니다.

## 일반적인 원인 {#common-causes}

- **사용자 입력 오류** — 숫자 필드에 텍스트 입력
- **가져오기 문제** — 가져온 데이터의 형식이 잘못됨
- **데이터 손상** — 전송 중 값이 손상됨
- **플러그인 충돌** — 다른 플러그인이 데이터 유형을 변경함

## 해결 방법 {#how-to-fix}

### 1. 입력 내용 확인 {#1-check-your-input}

입력하는 데이터를 검토하세요:
- **가격** — 숫자여야 합니다 (예: `19.99`)
- **수량** — 정수여야 합니다 (예: `5`)
- **ID** — 숫자여야 합니다

### 2. 지우고 다시 입력 {#2-clear-and-re-enter}

데이터가 손상된 경우:
- 필드를 지우세요
- 값을 수동으로 다시 입력하세요
- 외부 소스에서 복사-붙여넣기를 피하세요

### 3. 원본 데이터 확인 {#3-check-source-data}

WooCommerce에서 동기화하는 경우:
- WordPress 관리에서 데이터를 확인하세요
- 잘못 형식이 지정된 필드를 찾아보세요
- 원본 데이터에서 수정하세요

### 4. 데이터 다시 동기화 {#4-re-sync-data}

서버에서 새 데이터를 가져오세요:
- 로컬 캐시를 지우세요
- 데이터를 다시 동기화하세요
- 문제가 해결되는지 확인하세요

### 5. 플러그인 충돌 확인 {#5-check-for-plugin-conflicts}

다른 WooCommerce 플러그인을 사용하는 경우:
- 예상치 못한 방식으로 데이터를 수정할 수 있습니다
- 테스트를 위해 일시적으로 비활성화하세요
- 호환성 문제를 보고하세요

## 일반적인 예시 {#common-examples}

| 필드 | 예상 | 잘못된 |
|-------|----------|---------|
| 가격 | `19.99` | `$19.99` |
| 수량 | `5` | `five` |
| SKU | `ABC123` | — (모든 형식 가능) |
| 제품 ID | `42` | `product-42` |
