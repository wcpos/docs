# API03001: 잘못된 요청 형식

## 의미

서버로 전송된 요청이 예상된 형식과 맞지 않습니다. 서버가 POS에서 요청한 내용을 이해할 수 없었습니다. 요청 구조가 올바르지 않았기 때문입니다.

## 일반적인 원인

- **데이터 손상** — 데이터가 전송 전에 손상됨
- **소프트웨어 버그** — POS 애플리케이션의 문제
- **프록시 간섭** — 프록시 또는 방화벽이 요청을 수정함
- **문자 인코딩 문제** — 특수 문자가 제대로 인코딩되지 않음

## 서버 오류 매핑

다음과 같은 서버 응답에서 이 오류 코드가 트리거됩니다:

| 서버 코드 | 출처 |
|-------------|--------|
| HTTP 400 | 모든 서버 응답 (대체) |

## 해결 방법

### 1. 작업 다시 시도

간혹 일시적인 문제일 수 있습니다:
- 잠시 기다린 후 다시 시도
- POS를 새로고침하고 다시 시도

### 2. 특수 문자 확인

특수 문자로 데이터를 입력할 때:
- 이모지나 특이한 기호 제거
- 제품 이름 등에 표준 문자 사용

### 3. POS 업데이트

최신 버전 실행 확인:
- 앱 업데이트 확인
- 서버의 WCPOS 플러그인 업데이트

### 4. 네트워크 구성 확인

프록시 사용 시:
- 프록시가 요청을 수정하지 않는지 확인
- 방화벽 규칙 확인
- 다른 네트워크에서 접속 시도

### 5. 문제 보고

지속적으로 발생할 경우:
- 오류를 트리거하는 작업 기록
- 웹 버전의 경우 브라우저 콘솔 확인
- [GitHub](https://github.com/wcpos)에 재현 단계와 함께 보고
