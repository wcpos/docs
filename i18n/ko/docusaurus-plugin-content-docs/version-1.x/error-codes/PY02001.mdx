# PY02001: 결제 게이트웨이 오류 {#py02001-payment-gateway-error}

## 이것이 의미하는 바 {#what-this-means}

결제 게이트웨이와 통신하는 동안 오류가 발생했습니다. 결제 프로세서가 요청을 완료할 수 없었습니다.

## 일반적인 원인 {#common-causes}

- **게이트웨이 중단** — 결제 서비스에 문제가 발생하고 있습니다
- **구성 오류** — 게이트웨이 설정이 올바르지 않습니다
- **API 자격 증명** — 잘못되었거나 만료된 API 키
- **네트워크 문제** — 게이트웨이와의 연결 문제

## 해결 방법 {#how-to-fix}

### 1. 거래 재시도 {#1-retry-the-transaction}

가끔 게이트웨이에 일시적인 문제가 발생합니다:
- 잠시 기다리십시오
- 결제를 다시 처리해 보십시오
- 문제는 스스로 해결될 수 있습니다

### 2. 게이트웨이 상태 확인 {#2-check-gateway-status}

결제 제공자의 상태 페이지를 방문하십시오:
- Stripe: status.stripe.com
- PayPal: PayPal 커뮤니티 또는 상태 확인
- Square: status.squareup.com
- 알려진 중단 여부 확인

### 3. 게이트웨이 구성 검증 {#3-verify-gateway-configuration}

WooCommerce 관리자에서:
1. WooCommerce → 설정 → 결제 페이지로 이동
2. 결제 게이트웨이를 클릭하십시오
3. API 키가 올바른지 확인하십시오
4. 테스트/실제 모드가 키와 일치하는지 확인하십시오

### 4. API 자격 증명 확인 {#4-check-api-credentials}

자격 증명이 유효한지 확인하십시오:
- API 키가 만료되지 않았는지
- 실제 거래에 실제 키를 사용하고 있는지
- 테스트 거래에 테스트 키를 사용하고 있는지
- 자격 증명이 올바른 계정과 일치하는지

### 5. 다른 게이트웨이로 테스트 {#5-test-with-a-different-gateway}

가능한 경우:
- 대체 게이트웨이로 결제를 시도하십시오
- 백업으로 현금 결제를 사용하십시오
- 판매를 완료하고 나중에 조정하십시오

## 매장 소유자를 위한 정보 {#for-store-owners}

일반적인 구성 문제:
- 테스트와 실제 API 키 혼합
- 다른 계정의 API 키
- 비활성화된 결제 방법
- 만료되었거나 취소된 자격 증명

문제가 지속되면 결제 제공자의 지원팀에 연락하십시오.
