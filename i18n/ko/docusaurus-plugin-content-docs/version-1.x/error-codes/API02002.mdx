# API02002: 토큰 만료

## 의미

인증 세션이 만료되었습니다. 보안상의 이유로 로그인 세션은 영원히 지속되지 않습니다. POS는 인증을 새로 고치거나 다시 로그인해야 할 수 있습니다.

## 일반적인 원인

- **장기간 비활성** — POS를 오랫동안 사용하지 않음
- **서버 토큰 수명** — 서버의 토큰 만료 시간이 매우 짧게 설정됨
- **서버 시간 불일치** — 서버와 장치 시계가 동기화되지 않음

## 서버 오류 매핑

이 오류 코드는 서버가 다음을 반환할 때 트리거됩니다:

| 서버 코드 | 소스 |
|-------------|--------|
| `jwt_auth_expired_token` | JWT 인증 플러그인 |

## 해결 방법

### 1. 다시 로그인

가장 간단한 해결책은 로그아웃하고 다시 로그인하는 것입니다:
1. POS에서 로그아웃
2. 자격 증명 다시 입력
3. 새 세션 생성

### 2. 토큰 갱신 확인

POS는 토큰을 자동으로 갱신해야 합니다. 작동하지 않는 경우:
- 인터넷 연결 확인
- 서버 접근성 검증
- 갱신 실패 시 [API02007](./API02007) 참조

### 3. 서버 시간 확인

토큰 만료는 정확한 시간에 따라 달라집니다:
- 서버 시계가 정확한지 확인
- 장치 시간이 정확한지 확인
- 양쪽 모두 자동 시간 동기화 사용

### 4. 토큰 설정 검토

토큰이 너무 빨리 만료되는 경우, 서버 설정을 조정해야 할 수 있습니다 (호스팅 제공업체 또는 WordPress 관리자와 상담).
