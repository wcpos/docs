# API02002: 토큰 만료 {#api02002-token-expired}

## 의미 {#what-this-means}

당신의 인증 세션이 만료되었습니다. 보안상의 이유로 로그인 세션은 영원히 지속되지 않습니다. POS는 당신의 인증을 새로 고쳐야 하며, 다시 로그인해야 할 수도 있습니다.

## 일반적인 원인 {#common-causes}

- **오랜 비활성** — POS를 장기간 사용하지 않았습니다
- **서버 토큰 수명** — 서버의 토큰 만료 시간이 매우 짧게 설정됨
- **서버 시간 불일치** — 서버와 장치의 시계가 동기화되어 있지 않음

## 서버 오류 매핑 {#server-error-mapping}

이 오류 코드는 서버가 다음을 반환할 때 발생합니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `jwt_auth_expired_token` | JWT 인증 플러그인 |

## 해결 방법 {#how-to-fix}

### 1. 다시 로그인 {#1-log-in-again}

가장 간단한 해결 방법은 로그아웃 후 다시 로그인하는 것입니다:
1. POS에서 로그아웃
2. 자격 증명을 다시 입력
3. 이렇게 하면 새 세션이 생성됩니다

### 2. 토큰 새로 고침 확인 {#2-check-token-refresh}

POS는 자동으로 토큰을 새로 고쳐야 합니다. 작동하지 않는 경우:
- 인터넷 연결을 확인하세요
- 서버에 접근이 가능한지 확인하세요
- 새로 고침이 실패하는 경우 [API02007](./API02007)를 참조하세요

### 3. 서버 시간 확인 {#3-check-server-time}

토큰 만료는 정확한 시간에 따라 다릅니다:
- 서버의 시계가 정확한지 확인하십시오
- 장치의 시간이 정확한지 확인하십시오
- 두 장치 모두 자동 시간 동기화를 사용하십시오

### 4. 토큰 설정 검토 {#4-review-token-settings}

토큰이 너무 빨리 만료되면 서버 설정을 조정해야 할 수도 있습니다 (호스팅 제공업체나 WordPress 관리자에게 문의).
