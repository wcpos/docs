# API06001: 잘못된 URL 형식

## 의미

WooCommerce 사이트에 대해 구성된 URL이 유효한 형식이 아닙니다. URL은 표준 웹 주소 규칙을 따라야 합니다.

## 일반적인 원인

- **프로토콜 누락** — URL이 `http://` 또는 `https://`로 시작하지 않음
- **URL 오타** — 도메인의 철자 오류
- **잘못된 문자** — URL에 허용되지 않는 특수 문자
- **불완전한 URL** — 도메인 확장자 등 URL의 일부 누락

## 해결 방법

### 1. URL 형식 확인

유효한 URL은 다음과 같아야 합니다:
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com` (프로토콜 누락)
- ❌ `https://yourstore` (확장자 누락)
- ❌ `https://your store.com` (공백 포함)

### 2. 프로토콜 포함

항상 `https://` 또는 `http://`를 포함하세요:
- 권장: `https://` (보안)
- 대체: `http://` (HTTPS를 사용할 수 없는 경우)

### 3. 추가 문자 제거

URL에 다음 항목이 없는지 확인하세요:
- 후행 슬래시 (선택 사항이지만 일관성 유지)
- 추가 공백
- 특수 문자
- URL 매개변수 (필요하지 않은 경우)

### 4. URL 작동 확인

브라우저에서 URL 테스트:
1. 사용 중인 URL 복사
2. 브라우저에 붙여넣기
3. WordPress 사이트가 로드되어야 함

### 5. POS 구성 업데이트

POS 설정에서 수정된 URL 입력:
- 브라우저에서 작동하는 것과 정확히 일치
- 사이트에서 사용하는 대로 `www.` 포함 또는 제외

## 예시

| 잘못된 URL | 올바른 URL |
|---------|-------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
