# API06001: 잘못된 URL 형식 {#api06001-invalid-url-format}

## 의미 {#what-this-means}

WooCommerce 사이트에 구성된 URL이 유효한 형식이 아닙니다. URL은 표준 웹 주소 규칙을 따라야 합니다.

## 일반적인 원인 {#common-causes}

- **프로토콜 누락** — URL이 `http://` 또는 `https://`로 시작하지 않음
- **URL 오타** — 도메인 철자 오류
- **유효하지 않은 문자** — URL에 허용되지 않는 특수 문자
- **불완전한 URL** — 도메인 확장자와 같은 부분이 누락됨

## 수정 방법 {#how-to-fix}

### 1. URL 형식 확인 {#1-check-url-format}

유효한 URL은 다음과 같아야 합니다:
- ✅ `https://yourstore.com`
- ✅ `https://www.yourstore.com`
- ✅ `https://store.yourdomain.com`
- ❌ `yourstore.com` (프로토콜 누락)
- ❌ `https://yourstore` (확장자 누락)
- ❌ `https://your store.com` (공백 포함)

### 2. 프로토콜 포함 {#2-include-the-protocol}

항상 `https://` 또는 `http://`를 포함합니다:
- 선호: `https://` (보안)
- 대체: `http://` (HTTPS를 사용할 수 없는 경우)

### 3. 추가 문자 제거 {#3-remove-extra-characters}

URL에 다음이 없도록 합니다:
- 후행 슬래시 (선택 사항이나 일관되게 유지)
- 추가 공백
- 특수 문자
- URL 매개변수 (필요하지 않은 경우 제외)

### 4. URL 작동 확인 {#4-verify-the-url-works}

브라우저에서 URL을 테스트합니다:
1. 사용하는 URL을 복사합니다
2. 브라우저에 붙여넣습니다
3. WordPress 사이트가 로드되어야 합니다

### 5. POS 구성 업데이트 {#5-update-pos-configuration}

POS 설정에 수정된 URL을 입력합니다:
- 브라우저에서 작동하는 URL과 정확히 일치해야 합니다
- 사이트에 따라 `www.`를 포함하거나 제외합니다

## 예시 {#examples}

| 유효하지 않은 | 유효한 |
|---------------|-------|
| `mystore.com` | `https://mystore.com` |
| `https://my store.com` | `https://mystore.com` |
| `https://mystore` | `https://mystore.com` |
| `htp://mystore.com` | `https://mystore.com` |
