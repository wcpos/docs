# DB02001: 중복 레코드

## 이것이 의미하는 바

데이터베이스가 이미 존재하는 ID로 레코드를 삽입하려고 시도했습니다. 각 레코드는 고유한 식별자를 가져야 합니다.

## 일반적인 원인

- **동기화 충돌** — 동일한 데이터가 두 번 동기화됨
- **경쟁 조건** — 여러 작업이 동일한 레코드를 생성함
- **데이터 손상** — ID가 손상되거나 중복됨
- **가져오기 문제** — 이미 존재하는 데이터를 가져옴

## 해결 방법

### 1. 데이터 새로 고침

서버에서 다시 동기화합니다:
- 최신 데이터 가져오기
- 동기화 프로세스가 충돌을 해결하도록 허용
- 최신 데이터가 우선해야 합니다.

### 2. 작업 다시 시도

새로운 레코드를 생성하는 경우:
- 시스템이 이미 그것을 생성했을 수 있음
- 레코드가 존재하는지 확인
- 여러 번 제출 버튼 클릭을 피함

### 3. 로컬 캐시 지우기

로컬 데이터 초기화:
1. 로컬 데이터베이스/캐시 지우기
2. 다시 로그인
3. 데이터를 새로 동기화

### 4. WooCommerce에서 중복 확인

문제가 지속되는 경우:
- WooCommerce에서 중복 항목 확인
- 동일한 ID를 가진 제품/주문 확인
- 관리자에서 중복 항목 청소

### 5. 지속적인 문제 보고

이런 일이 자주 발생하는 경우:
- 원인을 재확인
- 패턴 확인
- 세부사항과 함께 지원팀에 보고

## 이것이 중요한 이유

중복 ID는 다음과 같은 문제를 일으킬 수 있습니다:
- 어떤 레코드가 정확한지 혼란
- 데이터 덮어쓰기
- 동기화 문제

이 오류는 데이터 손상을 방지합니다.
