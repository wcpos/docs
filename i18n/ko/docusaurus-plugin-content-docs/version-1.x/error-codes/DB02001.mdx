# DB02001: 중복 레코드

## 의미

데이터베이스가 이미 존재하는 ID로 레코드를 삽입하려 했습니다. 각 레코드는 고유한 식별자를 가져야 합니다.

## 일반적인 원인

- **동기화 충돌** — 동일한 데이터가 두 번 동기화됨
- **경쟁 조건** — 여러 작업에서 동일한 레코드 생성
- **데이터 손상** — ID가 손상되거나 중복됨
- **가져오기 문제** — 이미 존재하는 데이터 가져오기

## 해결 방법

### 1. 데이터 새로고침

서버에서 다시 동기화:
- 최신 데이터 가져오기
- 동기화 프로세스가 충돌 해결하도록 함
- 더 최신 데이터가 우선 적용됨

### 2. 작업 다시 시도

새 레코드 생성 시:
- 시스템이 이미 생성했을 수 있음
- 레코드 존재 여부 확인
- 제출 버튼 여러 번 클릭 방지

### 3. 로컬 캐시 지우기

로컬 데이터 초기화:
1. 로컬 데이터베이스/캐시 지우기
2. 다시 로그인
3. 데이터 새로 동기화

### 4. WooCommerce에서 중복 확인

문제가 지속되면:
- WooCommerce에서 중복 항목 확인
- 동일한 ID를 가진 제품/주문 찾기
- 관리자 페이지에서 중복 정리

### 5. 지속적인 문제 보고

자주 발생한다면:
- 발생 원인 기록
- 패턴 확인
- 자세한 내용과 함께 지원팀에 보고

## 왜 중요한가

중복 ID는 다음을 초래할 수 있습니다:
- 어떤 레코드가 정확한지 혼란
- 데이터 덮어쓰기
- 동기화 문제

이 오류는 데이터 손상을 방지합니다.
