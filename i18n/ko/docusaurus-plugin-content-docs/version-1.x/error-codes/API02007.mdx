# API02007: 토큰 새로 고침 실패

## 의미

POS가 인증 세션을 자동으로 갱신하려고 했으나 새로 고침 요청이 실패했습니다. 이는 일반적으로 다시 로그인해야 함을 의미합니다.

## 일반적인 원인

- **새로 고침 토큰 만료** — 새로 고침 토큰이 만료되었습니다.
- **새로 고침 토큰 취소** — 관리자에 의해 모든 세션이 무효화되었습니다.
- **서버 오류** — 서버가 새로 고침 요청을 처리할 수 없었습니다.
- **네트워크 문제** — 새로 고침 시도 중 연결 문제 발생

## 해결 방법

### 1. 다시 로그인하기

가장 신뢰할 수 있는 해결 방법:
1. POS에서 로그아웃합니다.
2. 자격 증명으로 다시 로그인합니다.
3. 이렇게 하면 새 토큰으로 새로운 세션이 생성됩니다.

### 2. 네트워크 연결 확인

네트워크 문제로 새로 고침이 실패하는 경우:
- 인터넷 연결을 확인합니다.
- 브라우저에서 WordPress 사이트에 접근해 봅니다.
- 잠시 대기한 후 다시 시도합니다.

### 3. 서버 상태 확인

서버에 문제가 있을 경우:
- WordPress가 제대로 실행되고 있는지 확인합니다.
- 서버 로그에서 PHP 오류를 확인합니다.
- 인증 플러그인이 활성화되어 있는지 확인합니다.

### 4. 세션 데이터 지우기

문제가 지속될 경우:
- POS 앱 캐시를 지웁니다.
- 저장된 자격 증명을 제거합니다.
- 새 로그인으로 새롭게 시작합니다.

## 자동 토큰 새로 고침

WCPOS는 세션 유지를 위해 토큰이 만료되기 전에 자동으로 새로 고칩니다. 이 과정이 반복적으로 실패하면 구성 문제를 해결해야 할 수 있습니다.
