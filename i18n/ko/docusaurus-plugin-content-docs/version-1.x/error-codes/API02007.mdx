# API02007: 토큰 갱신 실패

## 의미

POS가 자동으로 인증 세션을 갱신하려 했지만 갱신 요청에 실패했습니다. 일반적으로 다시 로그인해야 합니다.

## 일반적인 원인

- **리프레시 토큰 만료** — 리프레시 토큰 자체가 만료됨
- **리프레시 토큰 취소** — 관리자가 모든 세션을 무효화함
- **서버 오류** — 서버가 갱신 요청을 처리할 수 없음
- **네트워크 문제** — 갱신 시도 중 연결 오류

## 해결 방법

### 1. 다시 로그인

가장 확실한 해결 방법:
1. POS에서 로그아웃
2. 자격 증명으로 다시 로그인
3. 새로운 토큰으로 새 세션 생성

### 2. 네트워크 연결 확인

네트워크 문제로 갱신이 실패하는 경우:
- 인터넷 연결 확인
- 브라우저에서 WordPress 사이트 접속
- 잠시 후 다시 시도

### 3. 서버 상태 확인

서버에 문제가 있는 경우:
- WordPress가 제대로 실행 중인지 확인
- 서버 로그의 PHP 오류 점검
- 인증 플러그인이 활성화되어 있는지 확인

### 4. 세션 데이터 지우기

문제가 지속되는 경우:
- POS 앱 캐시 지우기
- 저장된 자격 증명 제거
- 새로운 로그인으로 다시 시작

## 자동 토큰 갱신

WCPOS는 세션을 유지하기 위해 토큰이 만료되기 전에 자동으로 갱신합니다. 이 과정이 반복적으로 실패하면 구성 문제가 있을 수 있습니다.
