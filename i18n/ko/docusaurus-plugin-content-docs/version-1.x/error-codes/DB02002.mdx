# DB02002: 기록을 찾을 수 없음

## 의미

요청한 기록이 로컬 데이터베이스에 존재하지 않습니다. POS는 로컬에 저장되지 않은 제품, 고객, 주문 또는 기타 기록에 액세스하려고 했습니다.

## 일반적인 원인

- **아직 동기화되지 않음** — 데이터가 아직 다운로드되지 않음
- **서버에서 삭제됨** — WooCommerce에서 기록이 제거됨
- **완전한 동기화되지 않음** — 초기 동기화가 계속 실행 중
- **적용된 필터** — 활성 필터에 의해 기록이 제외됨

## 해결 방법

### 1. 동기화 대기

최근 시작한 경우:
- 초기 동기화가 완료될 때까지 기다림
- 동기화 진행 상태 표시기를 확인함
- 동기화가 완료되면 기록이 나타나야 함

### 2. 새로 고침/재동기화

데이터 새로 고침을 트리거합니다:
- 새로 고침/동기화 버튼 사용 (<Icon name="arrow-rotate-right" />) - 짧게 눌러 동기화
- **길게 누름** 동기화 버튼을 눌러 지우기 및 새로 고침 옵션 선택
- 서버에서 최신 데이터 가져오기
- 완료될 때까지 기다림

### 3. WooCommerce 확인

서버에 기록이 존재하는지 확인합니다:
- WordPress 관리자에 로그인
- 제품, 주문 또는 고객 확인
- 항목이 여전히 존재하는지 확인

### 4. 필터 확인

기록이 필터링될 수 있음:
- POS에서 활성 필터 검토
- 카테고리 필터 확인
- 가시성 설정 확인

### 5. 지우기 및 재동기화

동기화가 중단된 것 같으면:
1. 로컬 데이터 지우기
2. 다시 로그인
3. 새 동기화 완료

## 일반적인 시나리오

- **제품이 표시되지 않음:** 게시되고 가시적인지 확인
- **고객을 찾을 수 없음:** 아직 동기화되지 않았을 수 있음
- **주문이 누락됨:** 동기화 날짜 범위를 벗어났을 수 있음
