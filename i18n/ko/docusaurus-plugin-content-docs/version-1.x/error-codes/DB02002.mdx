# DB02002: 레코드를 찾을 수 없음

## 의미

요청한 레코드가 로컬 데이터베이스에 존재하지 않습니다. POS가 로컬에 저장되지 않은 제품, 고객, 주문 또는 다른 레코드에 접근하려고 시도했습니다.

## 일반적인 원인

- **아직 동기화되지 않음** — 데이터가 아직 다운로드되지 않음
- **서버에서 삭제됨** — WooCommerce에서 레코드 제거됨
- **동기화 완료되지 않음** — 초기 동기화가 아직 진행 중
- **필터 적용됨** — 활성 필터로 인해 레코드 제외됨

## 해결 방법

### 1. 동기화 대기

최근에 시작한 경우:
- 초기 동기화 완료 대기
- 동기화 진행 표시기 확인
- 동기화되면 레코드 표시됨

### 2. 새로고침/재동기화

데이터 새로고침 트리거:
- 새로고침/동기화 버튼 사용
- 서버에서 최신 데이터 가져오기
- 완료 대기

### 3. WooCommerce 확인

서버에 레코드 존재 여부 확인:
- WordPress 관리자 로그인
- 제품, 주문 또는 고객 확인
- 항목 존재 확인

### 4. 필터 확인

레코드가 필터링되었을 수 있음:
- POS의 활성 필터 검토
- 카테고리 필터 확인
- 가시성 설정 확인

### 5. 지우고 재동기화

동기화가 멈춘 경우:
1. 로컬 데이터 지우기
2. 다시 로그인
3. 새로운 동기화 완료

## 일반적인 시나리오

- **제품 미표시:** 게시 및 표시 여부 확인
- **고객 찾을 수 없음:** 아직 동기화되지 않았을 수 있음
- **주문 누락:** 동기화 날짜 범위 밖일 수 있음
