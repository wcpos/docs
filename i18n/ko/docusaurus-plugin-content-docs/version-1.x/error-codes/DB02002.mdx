# DB02002: 기록을 찾을 수 없음

## 의미

요청된 기록이 로컬 데이터베이스에 존재하지 않습니다. POS는 로컬에 저장되지 않은 제품, 고객, 주문 또는 기타 기록에 접근하려 했습니다.

## 일반적인 원인

- **아직 동기화되지 않음** — 데이터가 아직 다운로드되지 않았습니다.
- **서버에서 삭제됨** — WooCommerce에서 기록이 제거되었습니다.
- **완전한 동기화가 아님** — 초기 동기화가 아직 진행 중입니다.
- **적용된 필터** — 활성 필터에 의해 기록이 제외되었습니다.

## 해결 방법

### 1. 동기화 대기

최근에 시작된 경우:
- 초기 동기화가 완료될 때까지 기다립니다.
- 동기화 진행 상태 표시기를 확인합니다.
- 동기화가 완료되면 기록이 나타나야 합니다.

### 2. 새로 고침/재동기화

데이터 새로 고침을 트리거합니다:
- 새로 고침/동기화 버튼을 사용합니다.
- 서버에서 최신 데이터를 가져옵니다.
- 완료될 때까지 기다립니다.

### 3. WooCommerce 확인

서버에 기록이 존재하는지 확인합니다:
- WordPress 관리로 로그인합니다.
- 제품, 주문 또는 고객을 확인합니다.
- 항목이 여전히 존재하는지 확인합니다.

### 4. 필터 확인

기록이 필터링되었을 수 있습니다:
- POS에서 활성 필터를 검토합니다.
- 카테고리 필터를 확인합니다.
- 가시성 설정을 확인합니다.

### 5. 삭제 및 재동기화

동기화가 멈춘 것처럼 보이는 경우:
1. 로컬 데이터를 지웁니다.
2. 다시 로그인합니다.
3. 새 동기화를 완료합니다.

## 일반적인 시나리오

- **상품이 표시되지 않음:** 게시되고 가시성이 있는지 확인합니다.
- **고객을 찾을 수 없음:** 아직 동기화되지 않았을 수 있습니다.
- **주문이 누락됨:** 동기화 날짜 범위를 벗어났을 수 있습니다.
