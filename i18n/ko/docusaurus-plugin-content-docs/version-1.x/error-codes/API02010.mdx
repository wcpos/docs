# API02010: 인증 필요

## 의미

수행하려는 작업에는 인증이 필요하지만, 현재 로그인되어 있지 않습니다. POS에서는 이 리소스에 접근하기 위해 유효한 자격 증명이 필요합니다.

## 흔한 원인

- **로그인되지 않음** — 아직 인증되지 않음
- **세션 초기화** — 세션이 지워지거나 만료됨
- **보호된 리소스 접근** — 리소스에 인증이 필요함
- **앱 데이터 삭제** — 저장된 자격 증명이 제거됨

## 서버 오류 매핑

다음과 같은 경우 이 오류 코드가 트리거됩니다:

| 서버 코드 | 출처 |
|-------------|--------|
| `rest_login_required` | WordPress REST API |
| `jwt_auth_no_auth_header` | JWT 인증 플러그인 |
| HTTP 401 | 모든 서버 응답 (대체) |

## 해결 방법

### 1. 로그인

로그인하지 않은 경우:
1. POS 로그인 화면 열기
2. WordPress 자격 증명 입력
3. 인증 프로세스 완료

### 2. 세션 상태 확인

로그인되어 있다고 생각했다면:
- 세션이 만료되었을 수 있음
- 자세한 내용은 [API02002](./API02002) (토큰 만료) 참조
- 액세스 복원을 위해 다시 로그인

### 3. 서버 구성 확인

API 엔드포인트가 제대로 구성되었는지 확인:
- WooCommerce REST API가 활성화되어 있어야 함
- WCPOS 플러그인이 활성 상태여야 함
- 인증 엔드포인트에 접근할 수 있어야 함

### 4. 브라우저/앱 문제 확인

예기치 않게 로그아웃되는 경우:
- 웹 버전인 경우 브라우저 캐시 지우기
- 쿠키/로컬 저장소가 차단되지 않았는지 확인
- 앱이 데이터 저장 권한을 가지고 있는지 확인

## 인증이 필요한 작업

대부분의 POS 작업에는 인증이 필요합니다:
- 제품 및 고객 보기
- 주문 생성 및 편집
- 결제 처리
- 보고서 접근

인증 화면만 인증 없이 접근할 수 있습니다.
