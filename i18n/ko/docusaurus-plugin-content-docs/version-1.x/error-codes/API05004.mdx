# API05004: WordPress REST API 비활성화

## 의미

WordPress REST API가 사이트에서 비활성화되었습니다. 모든 최신 WordPress 기능, WooCommerce 및 WCPOS 포함, 이 API에 의존합니다.

## 일반적인 원인

- **보안 플러그인** — 플러그인이 REST API 접근을 차단
- **호스팅 제한** — 호스트가 REST API를 비활성화
- **사용자 정의 코드** — 테마 또는 플러그인이 API 비활성화
- **방화벽 규칙** — WAF가 REST API 엔드포인트 차단

## 해결 방법

### 1. REST API 테스트

브라우저에서 `https://yoursite.com/wp-json/` 방문:
- 사용 가능한 경로가 있는 JSON 반환
- 오류나 응답이 없으면 차단된 상태

### 2. 보안 플러그인 확인

REST API를 차단하는 일반적인 플러그인:

**Wordfence:**
- 방화벽 → 모든 방화벽 옵션
- "REST API 비활성화" 옵션 해제

**iThemes Security:**
- 보안 → 설정 → WordPress 트윅
- REST API 활성화

**Disable REST API 플러그인:**
- 플러그인 완전히 비활성화

### 3. 사용자 정의 코드 확인

테마의 `functions.php` 또는 사용자 정의 플러그인에서 다음 코드 찾기:

```php
// 이 코드는 REST API를 비활성화 - 제거
add_filter('rest_authentication_errors', function($result) {
    return new WP_Error('rest_disabled', 'REST API 비활성화');
});
```

### 4. .htaccess 확인

`/wp-json/`을 차단하는 규칙 제거:

```apache
# 나쁜 예 - REST API 차단
RewriteRule ^wp-json - [F,L]
```

### 5. 호스팅 제공업체 연락

일부 호스트는 기본적으로 REST API를 차단:
- 활성화 요청
- 보안 제한 문의
- 호스팅 문서 확인

## REST API의 중요성

WordPress REST API는 다음에 필수적입니다:
- 모바일 앱
- 제3자 통합
- WooCommerce 기능
- WCPOS 작동

비활성화하면 많은 기능이 중단됩니다.
