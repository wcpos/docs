# API03004: 요청 크기 초과

## 의미

서버의 크기 제한을 초과하는 요청을 보내고 있습니다. 일반적으로 한 번에 너무 많은 데이터를 전송하려고 할 때 발생합니다.

## 일반적인 원인

- **대규모 배치 작업** — 너무 많은 레코드를 한 번에 동기화하려는 시도
- **대용량 이미지** — 크기가 큰 이미지 업로드
- **항목 과다** — 매우 많은 라인 항목이 있는 주문
- **서버 제한** — PHP 또는 웹 서버의 낮은 업로드 제한

## 해결 방법

### 1. 요청 크기 줄이기

데이터 동기화 시:
- 더 작은 배치로 동기화 시도
- POS가 자동으로 처리해야 함
- 현재 동기화가 완료될 때까지 대기

### 2. 이미지 크기 확인

이미지 업로드 시:
- 업로드 전 이미지 크기 조정
- 압축된 형식 사용 (JPEG vs BMP)
- 대부분의 제품 이미지는 1MB 미만에서 잘 작동

### 3. 대용량 주문 분할

많은 항목이 있는 주문의 경우:
- 여러 주문으로 분할 고려
- 일반 POS 사용에서는 드문 경우

### 4. 서버 제한 늘리기

호스팅 제공업체에 문의하거나 PHP 설정 편집:

```php
// php.ini 또는 .htaccess에서
upload_max_filesize = 64M
post_max_size = 64M
max_input_vars = 5000
```

### 5. 웹 서버 제한 확인

Nginx 또는 Apache에는 자체 제한이 있을 수 있음:
- Nginx의 `client_max_body_size`
- Apache의 `LimitRequestBody`
