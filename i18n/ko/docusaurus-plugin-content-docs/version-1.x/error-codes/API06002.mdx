# API06002: API URL 누락

## 의미

WooCommerce 스토어에 연결할 URL이 구성되지 않았습니다. POS는 스토어의 위치를 알아야 합니다.

## 일반적인 원인

- **최초 설정** — URL이 한 번도 입력되지 않음
- **설정 지워짐** — 설정이 초기화되거나 삭제됨
- **앱 데이터 삭제** — 애플리케이션 데이터가 삭제됨
- **마이그레이션 문제** — 업데이트 또는 마이그레이션 중 URL 손실

## 해결 방법

### 1. 스토어 URL 입력

POS 앱에서:
1. 설정 또는 로그인 화면 열기
2. URL/사이트 구성 필드 찾기
3. WordPress 사이트 URL 입력
4. 예시: `https://yourstore.com`

### 2. WordPress URL 찾기

URL을 모를 경우:
1. WordPress 관리자에 로그인
2. 설정 → 일반으로 이동
3. "WordPress 주소(URL)" 확인
4. POS에서 이 URL 사용

### 3. 저장된 설정 확인

이전에 URL이 구성되었던 경우:
- 앱 데이터가 실수로 삭제되었는지 확인
- 설정 백업/내보내기 옵션 찾기
- 필요한 경우 URL 다시 입력

### 4. URL 정확성 확인

저장하기 전:
1. 브라우저에서 URL 테스트
2. WordPress 사이트가 로드되는지 확인
3. `https://` 또는 `http://` 포함

## 설정 체크리스트

WCPOS를 처음 설정할 때:

1. ☐ WordPress 사이트 URL 입력
2. ☐ WCPOS 플러그인 설치 및 활성화 확인
3. ☐ WordPress 자격 증명으로 로그인
4. ☐ 초기 데이터 동기화 완료 허용
