# API06002: 누락된 API URL

## 의미

WooCommerce 상점에 연결하기 위한 URL이 설정되지 않았습니다. POS는 상점이 위치한 곳을 알아야 합니다.

## 일반적인 원인

- **첫 번째 설정** — URL이 입력되지 않았습니다.
- **구성 초기화** — 설정이 초기화되거나 지워졌습니다.
- **앱 데이터 초기화** — 애플리케이션 데이터가 삭제되었습니다.
- **마이그레이션 문제** — 업데이트 또는 마이그레이션 도중 URL이 손실되었습니다.

## 해결 방법

### 1. 상점 URL 입력하기

POS 앱에서:
1. 설정 또는 로그인 화면을 엽니다.
2. URL/사이트 구성 필드를 찾습니다.
3. WordPress 사이트 URL을 입력합니다.
4. 예: `https://yourstore.com`

### 2. WordPress URL 찾기

URL이 확실하지 않은 경우:
1. WordPress Admin에 로그인합니다.
2. 설정 → 일반으로 이동합니다.
3. "WordPress 주소(URL)"를 확인합니다.
4. POS에서 이 URL을 사용합니다.

### 3. 저장된 설정 확인하기

URL이 이전에 구성된 경우:
- 앱 데이터가 실수로 초기화되었는지 확인합니다.
- 설정 백업/내보내기 옵션을 찾습니다.
- 필요 시 URL을 다시 입력합니다.

### 4. URL이 올바른지 확인하기

저장하기 전에:
1. 브라우저에서 URL을 테스트합니다.
2. WordPress 사이트가 로드되는지 확인합니다.
3. `https://` 또는 `http://`를 포함합니다.

## 설정 체크리스트

WCPOS를 처음 설정할 때:

1. ☐ WordPress 사이트 URL을 입력합니다.
2. ☐ WCPOS 플러그인이 설치되고 활성화되었는지 확인합니다.
3. ☐ WordPress 자격 증명으로 로그인합니다.
4. ☐ 초기 데이터 동기화가 완료되도록 허용합니다.
