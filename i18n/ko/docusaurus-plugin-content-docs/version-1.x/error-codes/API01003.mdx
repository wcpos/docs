# API01003: 연결 재설정 {#api01003-connection-reset}

## 의미 {#what-this-means}

서버와의 연결이 설정되었지만 예기치 않게 닫혔습니다. 이는 일반적으로 연결이 중단되는 중간 통신에서 발생합니다.

## 일반적인 원인 {#common-causes}

- **서버 중단** — 요청 중에 서버 프로세스가 중단되었을 수 있습니다.
- **메모리 한계** — PHP가 메모리를 다 써서 종료되었습니다.
- **요청 중 타임아웃** — 서버가 처리 중에 타임아웃되었습니다.
- **네트워크 불안정성** — 간헐적인 네트워크 문제
- **보안 소프트웨어** — 공격적인 보안 설정이 연결을 종료함

## 해결 방법 {#how-to-fix}

### 1. 서버 오류 로그 확인 {#1-check-server-error-logs}

재설정 시간에 대한 관련 오류를 찾기 위해 WordPress 오류 로그나 호스팅 제어판을 확인하세요.

### 2. PHP 메모리 한도 증가 {#2-increase-php-memory-limit}

`wp-config.php`에서:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. 플러그인 충돌 확인 {#3-check-for-plugin-conflicts}

플러그인이 PHP 중단을 유발할 수 있습니다:
- 플러그인을 일시적으로 비활성화
- 원인을 찾기 위해 하나씩 다시 활성화

### 4. 서버 리소스 검토 {#4-review-server-resources}

호스팅 제공업체에 연락하여 다음을 확인하세요:
- PHP 프로세스 한도
- 오류 발생 시 메모리 사용량
- 서버 안정성 로그

### 5. 네트워크 안정성 테스트 {#5-test-network-stability}

WiFi를 사용 중이라면 유선 연결로 시도해보세요. 여러 네트워크에서 문제가 지속된다면 서버 측에 문제가 있을 가능성이 높습니다.
