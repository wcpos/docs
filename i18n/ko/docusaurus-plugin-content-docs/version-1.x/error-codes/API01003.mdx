# API01003: 연결 리셋

## 이 오류의 의미

서버와의 연결이 설정되었지만 예기치 않게 닫혔습니다. 이는 일반적으로 통신 중간에 연결을 방해하는 무언가가 발생할 때 일어납니다.

## 일반적인 원인

- **서버 크래시** — 요청 중에 서버 프로세스가 중단됨
- **메모리 한계** — PHP 메모리가 부족해서 종료됨
- **요청 중 타임아웃** — 서버가 처리 중에 시간 초과
- **네트워크 불안정** — 간헐적인 네트워크 문제
- **보안 소프트웨어** — 연결을 닫는 공격적인 보안 설정

## 해결 방법

### 1. 서버 오류 로그 확인

WordPress 오류 로그 또는 호스팅 제어판에서 리셋 시간 주변의 관련 오류를 확인하세요.

### 2. PHP 메모리 제한 늘리기

`wp-config.php`에서:

```php
define('WP_MEMORY_LIMIT', '256M');
```

### 3. 플러그인 충돌 확인

플러그인이 PHP 크래시를 유발할 수 있습니다:
- 일시적으로 플러그인 비활성화
- 원인을 찾을 때까지 하나씩 다시 활성화

### 4. 서버 리소스 검토

호스팅 제공업체에 문의하여 다음 사항을 확인하세요:
- PHP 프로세스 제한
- 오류 발생 시 메모리 사용량
- 서버 안정성 로그

### 5. 네트워크 안정성 테스트

WiFi를 사용 중이라면 유선 연결을 시도하세요. 다른 네트워크에서도 문제가 지속되면 서버 측 문제일 가능성이 높습니다.
