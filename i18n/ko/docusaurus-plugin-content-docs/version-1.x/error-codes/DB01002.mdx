# DB01002: 쿼리 타임아웃 {#db01002-query-timeout}

## 이것이 의미하는 바 {#what-this-means}

데이터베이스 작업이 완료되는 데 너무 오랜 시간이 걸렸습니다. 로컬 데이터베이스 쿼리가 허용된 최대 시간을 초과했습니다.

## 일반적인 원인 {#common-causes}

- **대용량 데이터 세트** — 매우 많은 수의 제품 또는 레코드
- **복잡한 쿼리** — 많은 조건이 포함된 쿼리
- **장치 성능** — 장치가 느리게 작동
- **데이터베이스 조각화** — 데이터베이스 최적화 필요

## 해결 방법 {#how-to-fix}

### 1. 기다렸다가 다시 시도하기 {#1-wait-and-retry}

재시도가 성공할 수 있습니다:
- POS를 닫았다가 다시 열기
- 작업을 다시 시도하기
- 동시에 다른 집중적인 작업 피하기

### 2. 다른 응용 프로그램 닫기 {#2-close-other-applications}

장치 리소스를 확보하세요:
- 사용하지 않는 브라우저 탭 닫기
- 다른 응용 프로그램 닫기
- 필요시 장치를 재시작하기

### 3. 데이터 범위 줄이기 {#3-reduce-data-scope}

검색하는 경우:
- 더 구체적인 검색어 사용
- 결과를 좁히기 위한 필터 적용
- 대규모 작업을 더 작은 작업으로 나누기

### 4. 데이터베이스 최적화 {#4-optimise-the-database}

지우고 다시 동기화하기:
1. 로컬 데이터 삭제 (설정 옵션이 있을 경우)
2. 다시 로그인하기
3. 데이터를 새로 동기화하기

### 5. 장치 성능 확인 {#5-check-device-performance}

장치가 요구 사항을 충족하는지 확인하세요:
- 충분한 RAM
- 적절한 프로세서 속도
- SSD가 HDD보다 선호됨

## 대규모 카탈로그 {#large-catalogues}

아주 큰 제품 카탈로그가 있는 경우:
- 초기 동기화가 더 오래 걸림
- POS에서 필터 사용 고려하기
- 일부 작업은 자연적으로 더 많은 시간이 걸림
- 최적화 조언을 위해 지원에 문의하기
