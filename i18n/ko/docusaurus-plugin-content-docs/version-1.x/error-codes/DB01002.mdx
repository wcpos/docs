# DB01002: 쿼리 시간 초과

## 의미

데이터베이스 작업이 완료하는 데 너무 오래 걸렸습니다. 로컬 데이터베이스 쿼리가 허용된 최대 시간을 초과했습니다.

## 일반적인 원인

- **대용량 데이터 세트** — 매우 많은 제품 또는 레코드
- **복잡한 쿼리** — 많은 조건을 포함하는 쿼리
- **장치 성능** — 장치가 느리게 실행됨
- **데이터베이스 조각화** — 데이터베이스 최적화 필요

## 해결 방법

### 1. 대기 및 재시도

쿼리가 재시도 시 성공할 수 있습니다:
- POS 닫고 다시 열기
- 작업 다시 시도
- 동시에 다른 집중적인 작업 피하기

### 2. 다른 애플리케이션 닫기

장치 리소스 확보:
- 사용하지 않는 브라우저 탭 닫기
- 다른 애플리케이션 닫기
- 필요한 경우 장치 재시작

### 3. 데이터 범위 축소

검색 시:
- 더 구체적인 검색어 사용
- 필터 적용하여 결과 좁히기
- 대규모 작업을 더 작은 작업으로 분할

### 4. 데이터베이스 최적화

지우고 다시 동기화:
1. 로컬 데이터 지우기 (가능한 경우 설정 옵션)
2. 다시 로그인
3. 데이터 새로 동기화

### 5. 장치 성능 확인

장치가 요구 사항을 충족하는지 확인:
- 충분한 RAM
- 적절한 프로세서 속도
- HDD보다 SSD 선호

## 대규모 카탈로그

매우 큰 제품 카탈로그가 있는 경우:
- 초기 동기화에 더 오래 걸림
- POS에서 필터 사용 고려
- 일부 작업은 자연스럽게 더 오래 걸림
- 최적화 조언을 위해 지원팀에 문의
