The Checkout Settings page (`WP Admin > POS > Settings > Checkout`) controls which payment gateways appear in the POS and how orders are processed.

## Payment Gateways

This section lists all WooCommerce payment gateways installed on your site. Each gateway can be independently enabled or disabled for the POS.

| Column | Description |
|--------|-------------|
| **Enabled** | Toggle the gateway on or off for POS checkout |
| **Gateway** | The payment gateway name |
| **Gateway ID** | The WooCommerce gateway identifier (useful for developers) |

### Enabling a Gateway

1. Find the gateway in the list
2. Toggle the **Enabled** switch on
3. The gateway will appear at POS checkout on the next page load

### Reordering Gateways

Drag gateways to change their display order in the POS checkout. The first enabled gateway is selected by default when a cashier opens the checkout modal.

### POS vs Online Store

Gateway settings here are **independent** of your WooCommerce payment settings. A gateway can be:

- **Enabled for POS, disabled online** — useful for cash payments which don't apply to web orders
- **Enabled online, disabled for POS** — useful for gateways like PayPal that aren't practical for in-person sales
- **Enabled for both** — the gateway works everywhere

:::note
A gateway must first be installed and active in `WP Admin > WooCommerce > Settings > Payments` before it appears in the POS checkout settings list. If a gateway is missing, check that the plugin is installed and the gateway is enabled in WooCommerce.
:::

## Prevent Overselling

When enabled, this setting prevents cashiers from selling more stock than is available. It provides two layers of protection:

| Layer | What happens |
|-------|-------------|
| **Client-side** | Blocks out-of-stock items from being added to the cart in the POS |
| **Server-side** | Rejects orders during processing if stock is insufficient (e.g., if stock changed between adding to cart and checkout) |

:::info Default Behavior
This setting is **disabled** by default. Existing stores are unaffected until the setting is explicitly turned on.
:::

### Interaction with WooCommerce Backorders

WooCommerce allows backorders to be enabled on a per-product basis (`WP Admin > Products > Edit Product > Inventory > Allow backorders?`). If backorders are enabled for a product, the overselling prevention setting **does not apply** to that product — it can still be added to the cart regardless of stock level.

In other words, overselling prevention only blocks products where backorders are **not allowed**.

### Error Messages

When overselling prevention blocks an action, the cashier will see one of the following messages:

| Situation | Error Message |
|-----------|--------------|
| Adding an out-of-stock item to the cart | "This product is out of stock and cannot be added to the cart." |
| Adding more than available quantity | "Only X items are available for this product." |
| Order rejected at checkout | "Order could not be completed: insufficient stock for one or more items." |

:::tip
If a cashier frequently encounters stock errors, check that your [product sync settings](/products/sync) are configured to keep POS stock data up to date.
:::

## Order Status

Configure the default order status for completed POS orders. By default, POS orders are set to **Completed** since payment is collected at the point of sale.

## Disable Scripts and Styles

The POS checkout uses an iframe/webview to display the WooCommerce Order Pay page. Theme and plugin scripts can sometimes interfere with the checkout display or payment gateway functionality. These settings let you selectively disable problematic resources.

:::warning
Be careful when disabling scripts — payment gateways may require certain scripts to function. If a payment method stops working after disabling a script, re-enable it.
:::

### Disable All Styles and Scripts

| Option | Description |
|--------|-------------|
| **Disable wp_head** | Removes all scripts and styles from the WordPress header |
| **Disable wp_footer** | Removes all scripts and styles from the WordPress footer |

:::warning Nuclear Option
These options remove **all** enqueued scripts and styles, including those required by WooCommerce. Only use for testing or if you know what you are doing.
:::

### Disable Selected Styles

Selectively disable CSS files that cause display issues in the checkout iframe. Common candidates include:

- `wp-emoji-styles`
- `wp-block-library`
- `classic-theme-styles`
- `woocommerce-layout`
- `woocommerce-smallscreen`
- `woocommerce-general`

### Disable Selected Scripts

Selectively disable JavaScript files that interfere with payment gateways. Common candidates include:

- `wc-add-to-cart`
- `selectWoo`
- `wc-checkout`
- `woocommerce`
- `html5shiv`

:::tip
If a payment gateway does not display correctly:

1. Try disabling theme styles first
2. Then try disabling non-essential WooCommerce scripts
3. Do not disable scripts required by your payment gateway
:::

For more details on diagnosing checkout performance issues, see [Checkout Performance](/support/performance/checkout).
