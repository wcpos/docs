Your WCPOS receipt templates aren't limited to the till. Online-store customers can view and download their order receipt or invoice — rendered with the **same custom template** you designed in **WP Admin → POS → Templates** — from their My Account page, the order confirmation page, or anywhere you place the shortcode.

:::info Requires WCPOS 1.9.11+
Storefront receipt access ships with WCPOS 1.9.11. Earlier versions only render receipts inside the POS.
:::

## My Account download button {#my-account}

As soon as a receipt template is **Active**, logged-in customers see a **Receipt** button next to eligible orders in **My Account → Orders**. Clicking it downloads a PDF (`receipt-1234.pdf`) rendered with your active receipt template.

By default the button appears for **Processing**, **Completed**, and **Refunded** orders — statuses where the order has been paid. Pending, cancelled, and failed orders don't show it.

There's nothing to configure: activate a template and the button is live. If you'd rather not show it, a developer can [turn it off with a filter](#developers).

## The `[wcpos_receipt]` shortcode {#shortcode}

To place a receipt link anywhere else — the order confirmation ("thank you") page, a custom account page, or an email template that supports shortcodes — use:

```text
[wcpos_receipt]
```

With no attributes, the shortcode detects the current order automatically on the **order confirmation** page and the **My Account → view order** page, and renders a *Download receipt (PDF)* link. Anywhere else (or when the visitor isn't allowed to see the order) it outputs nothing, so it's safe to leave in a template.

### Attributes {#shortcode-attributes}

| Attribute | Default | What it does |
|---|---|---|
| `order_id` | *(auto-detected)* | Link to a specific order. Only works for logged-in customers who own the order. |
| `link_text` | `Download receipt (PDF)` | The link text. |
| `format` | `pdf` | `pdf` downloads a PDF; `html` opens the printable receipt view instead. |
| `template` | *(active receipt template)* | Render with a specific template ID instead of the active one. |
| `class` | `wcpos-receipt-link` | CSS class for the link. |
| `id` | *(none)* | HTML `id` for the link. |

### Examples {#shortcode-examples}

A download link on the order confirmation page:

```text
[wcpos_receipt link_text="Download your invoice"]
```

A link to the on-screen (printable) receipt rather than a PDF:

```text
[wcpos_receipt format="html" link_text="View your receipt"]
```

Render with a specific template — for example an A4 invoice template instead of your 80mm till receipt:

```text
[wcpos_receipt template="123" link_text="Download invoice (PDF)"]
```

The template ID is the number in the template's edit-screen URL (`post=123`) in **WP Admin → POS → Templates**.

## Which template do customers get? {#template-choice}

Storefront downloads use your **active receipt template** — exactly what the POS prints. If your active template is a narrow thermal-style receipt, the PDF will be a narrow receipt too.

For online orders you may prefer a full-page document. Two options:

- Activate a full-page template (the gallery's **Invoice** template is designed for A4/Letter) as your receipt template, or
- Keep your till receipt active and point the storefront at a different template using the shortcode's `template` attribute, or site-wide with the [developer filter](#developers) below.

Templates render from the same [receipt data payload](/receipts/receipt-data) as the POS, so your logo, store identity, tax settings, and customisations all carry over.

## Security {#security}

Receipt links carry the order's unique key — the same mechanism WooCommerce uses for its order confirmation pages — so guests who checked out without an account can still use the link from the confirmation page. Anyone without the link needs to be logged in as the order's customer. Explicit `order_id` shortcodes only render for the logged-in owner of that order.

:::note Pretty permalinks required
Like all WCPOS checkout pages, the receipt URL uses WordPress rewrite rules. Sites using **Plain** permalinks (Settings → Permalinks) need to switch to any other permalink structure.
:::

## For developers {#developers}

Three filters control the storefront surfaces:

```php
// Hide the My Account Receipt button.
add_filter( 'woocommerce_pos_storefront_receipt_my_account_button', '__return_false' );

// Change which order statuses show the button (default: processing, completed, refunded).
add_filter( 'woocommerce_pos_storefront_receipt_order_statuses', function ( $statuses, $order ) {
	$statuses[] = 'on-hold';
	return $statuses;
}, 10, 2 );

// Use a different template for storefront PDF downloads only.
// Receives and returns the full template array (null when none is configured).
add_filter( 'woocommerce_pos_storefront_receipt_template', function ( $template, $order ) {
	return \WCPOS\WooCommercePOS\Templates::get_template( 123 );
}, 10, 2 );
```

The PDF route itself is `/wcpos-checkout/wcpos-receipt/{order_id}?key={order_key}&format=pdf` — the same order-key-gated endpoint the POS uses for receipts, with `format=pdf` returning the rendered PDF instead of the on-screen view.
